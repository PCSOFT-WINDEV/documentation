
## fExeDir (Function)

***In french: fRepExe***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le répertoire à partir duquel l'exécution est lancée.



**Attention** : La fonction **fExeDir** permet de connaître le répertoire de lancement, alors que la fonction [fRepEnCours](../WDLang1/3036022.md) permet de connaître le répertoire courant.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Example


```wl
// Répertoire de lancement
Info("Cette application a été lancée depuis " + fRepExe())
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntax

`<Résultat> = fRepExe()`
---

**`<Résultat>: (Chaîne de caractères)`**

Répertoire à partir duquel est lancé l'exécutable. Ce chemin ne se termine pas par le caractère "\\". 

- En mode test (GO depuis WINDEV), correspond : 

	- au répertoire de génération donné dans la description de la [configuration de projet](../Editeurs/9000030.md) courante. 

	- sinon au sous-répertoire EXE du répertoire du projet. 




- En exécution, correspond au répertoire contenant l'exécutable en cours.

## Remarks
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Serveur SOAP
<a name="serveur_soap_ELTPARAGRAPHE000116"></a>La fonction **fExeDir** utilisée sur un serveur SOAP retourne le chemin du répertoire contenant la bibliothèque du serveur SOAP en cours d'exécution.
<a name="NOTE0_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![User code (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 

### Application utilisée sous Windows Vista (et supérieur) 
<a name="application_utilisee_sous_windows_vista_superieur_ELTPARAGRAPHE000128"></a>Sous Windows Vista (et supérieur), le [mécanisme de l'UAC](../Editeurs/2025014.md) entraîne certaines contraintes de développement.

Ainsi, il est recommandé pour stocker les fichiers utilisés par l'application (fichiers de données ou les fichiers INI par exemple), d'utiliser les répertoires spécifiques de l'application (au lieu du répertoire de l'exécutable). Ces répertoires sont connus par les fonctions [fRepDonnéesCommun](../WDLang1/3036070.md), [fRepDonnéesUtilisateur](../WDLang1/3036071.md), [fRepGlobalCommun](../WDLang1/3036072.md) et [fRepGlobalUtilisateur](../WDLang1/3036073.md).
<a name="NOTE0_3"></a>




### Chemin d'un répertoire système
<a name="chemin_repertoire_systeme_ELTPARAGRAPHE000153"></a>Pour connaître le chemin d'un répertoire système, utilisez la fonction [SysRep](../WDLang1/3073026.md).

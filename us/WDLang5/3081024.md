


## XMLFirst (Function)

***In french: XMLPremier***



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Positions on the first element (tag or attribute) in the current tree level.

In most cases, **XMLFirst** is used to position in the XML document in order to perform a read loop with [XMLNext](../WDLang5/3081030.md).
![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) The browses are performed in alphabetical order of the attributes.








<a name="Example1"></a>
<a name="sample_code"></a>

## Example


```wl
// Positions on the first element
XMLFirst("XMLDoc")
WHILE XMLOut("XMLDoc") = False
	// Process the data
	XMLNext("XMLDoc")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Syntax

`<Result> = XMLFirst(<XML document>)`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> if an element was found,

- <u><u><u><u>False</u></u></u></u> otherwise. If an error occurs: 

	- the current position remains unchanged. 

	- the [ErrorOccurred](../WDLang1/3087001.md) variable is set to <u><u><u><u>True</u></u></u></u>. To get more details on the error, use [ErrorInfo](../WDLang1/3013008.md).







**`<XML document>: (Character string)`**

Name of the XML document used. This document contains the XML code to study and it was created by [XMLDocument](../WDLang5/3081008.md).

## Remarks


### Handling elements in the XML code
<a name="handling_elements_the_xml_code_ELTPARAGRAPHE000103"></a>

After **XMLFirst**, the current element can be handled by [XMLElementName](../WDLang5/3081026.md), [XMLParentName](../WDLang5/3081035.md), [XMLElementType](../WDLang5/3081031.md) and [XMLData](../WDLang5/3081004.md).
<a name="NOTE0_2"></a>


### Using XMLLast during a search
<a name="using_xmllast_during_search_ELTPARAGRAPHE000124"></a>

If a search is in progress ([XMLFind](../WDLang5/3081025.md)), **XMLFirst** cancels this search.
<a name="NOTE0_3"></a>


### XML functions and threads
<a name="xml_functions_and_threads_ELTPARAGRAPHE000270"></a>

If your application uses threads, the XML document is shared between all these threads. For more details on threads, see [Managing threads](../WDLang1/3077026.md).If the current position in an XML document is modified in a thread, the current position in this XML document is modified for all the threads.

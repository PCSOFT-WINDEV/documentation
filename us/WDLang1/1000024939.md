


## &lt;picLayer variable&gt;.DrawArc (Function)

***In french: &lt;Variable picCalque&gt;.DessineArc***



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Draws a circle arc or an ellipse arc in a variable of type [picLayer](../WDLang1/1000024604.md). The arc is drawn counterclockwise.
![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) **Caution**: A specific configuration is required to use this function in Linux. For more details, see [The drawings](../WDLang1/3029035.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Example


```wl
MyImage is WDPic = "Test.bmp"
MyLayer is picLayer

// Draw a light yellow arc of circle on the layer
MyLayer.DrawArc(10, 10, 50, 50, 20, 20, 40, 50, LightYellow)

IMG_MyDrawing = MyImage
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Syntax

### Drawing an arc

`<picLayer image>.DrawArc(<X1> , <Y1> , <X2> , <Y2> , <X3> , <Y3> , <X4> , <Y4> [, <Line color> [, <Line thickness>]])`
---

**`<picLayer image>: (picLayer variable)`**

Name of the [picLayer](../WDLang1/1000024604.md) variable to be used.

**`<X1>: (Integer)`**

X-coordinate (in pixels) of the upper-left corner of the rectangle containing the circle.

**`<Y1>: (Integer)`**

Y-coordinate (in pixels) of the upper-left corner of the rectangle containing the circle.

**`<X2>: (Integer)`**

X-coordinate (in pixels) of the lower-right corner of the rectangle containing the circle.

**`<Y2>: (Integer)`**

Y-coordinate (in pixels) of the lower-right corner of the rectangle containing the circle.

**`<X3>: (Integer)`**

X-coordinate (in pixels) of the starting point of the arc.

**`<Y3>: (Integer)`**

Y-coordinate (in pixels) of the starting point of the arc.

**`<X4>: (Integer)`**

X-coordinate (in pixels) of the end point of the arc.

**`<Y4>: (Integer)`**

Y-coordinate (in pixels) of the end point of the arc.

**`<Line color>: (Integer or constant (optional))`**

Color of the arc of circle. This color can correspond to:

- an RGB color (returned by [RGB](../WDLang1/3029012.md)),

- an HSL color (returned by [HSL](../WDLang1/3029057.md)),

- [a WLanguage preset color](../WDLang5/3010002.md), 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) a variable of type [Color](../WDLang1/1000019661.md).


If this parameter is not specified, the line color:

- is *Transparent* if [&lt;Image&gt;.Pen](../WDLang1/1000024016.md) has not been used beforehand, 

- corresponds to the color specified in the last call to [&lt;Image&gt;.Pen](../WDLang1/1000024016.md).




**`<Line thickness>: (Optional integer)`**

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Thickness (in pixels) of the arc of circle to draw. If this parameter is not specified, the value taken into account can be:

- the last value specified in the last call to [&lt;Image&gt;.Pen](../WDLang1/1000024016.md).

- 1 if [&lt;Image&gt;.Pen](../WDLang1/1000024016.md) was not used beforehand.

## Remarks


### Coordinates of the arc
<a name="coordinates_the_arc_ELTPARAGRAPHE000482"></a>If ***the start and end points*** are not located on the border of the circle (or on the border of the ellipse), the point taken into account corresponds to the intersection between the circle and the line that joins the specified point and the center of the rectangle (which means the center of the circle).
![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=3&name=RBArc2.gif)
The origin used to calculate the coordinates corresponds to the upper-left corner of the image (coordinates: (0,0)).
<a name="NOTE0_2"></a>


### Drawing a solid figure
<a name="drawing_solid_figure_ELTPARAGRAPHE000494"></a>

To draw a solid figure (a full arc), use [&lt;picLayer variable&gt;.DrawChord](../WDLang1/1000024469.md) or [&lt;picLayer variable&gt;.DrawSlice](../WDLang1/1000024561.md).

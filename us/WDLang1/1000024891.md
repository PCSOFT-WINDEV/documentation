


## <Type Chaîne Ansi>.AnsiVersOem (Function)

***In french: &lt;Chaîne Ansi&gt;.AnsiVersOem***



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Converts an ANSI string (Windows) to an OEM string (DOS). Accented characters are different in both standards. The conversion consists in modifying the accented characters.

**Reminder**:

- ANSI is the character set used in Windows.

- OEM is the character set used in DOS (or Windows console).





<a name="Example1"></a>
<a name="sample_code"></a>

## Example


```wl
Res is string
StringToConvert is string = "Crème brûlée"
Res = StringToConvert.AnsiToOem()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Syntax

`<Result> = <String to convert>.AnsiToOem()`
---

**`<Result>: (ANSI character string)`**

OEM character string.

**`<String to convert>: (ANSI character string)`**

ANSI string to convert.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarks


- **<Type Chaîne Ansi>.AnsiVersOem** does not convert characters with an ASCII code between 0 and 31 (control characters).

- **Caution**: Do not use **<Type Chaîne Ansi>.AnsiVersOem** with a string containing OEM specific characters (accented characters): these characters will be converted and the result will be incorrect.

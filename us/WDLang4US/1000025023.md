


## &lt;Source&gt;.ReadSeek (Function)

***In french: &lt;Source&gt;.LitRecherche***



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Positions on the first file record whose value for a specific item is greater than or equal to a sought value (generic search). The record is read and the corresponding HFSQL variables are updated.

In most cases, **&lt;Source&gt;.ReadSeek** sets the position in the data file to loop through the records that match a condition. [&lt;Source&gt;.ReadNext](../WDLang4/1000025035.md) is used to read the next record corresponding to the condition.Several cases may occur after the call to **&lt;Source&gt;.ReadSeek**:

- a record corresponding to the condition was found, locked (if necessary) and loaded in memory: [&lt;Source&gt;.Found](../WDLang4/1000025076.md) returns <u><u><u>True</u></u></u>.
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Access by JDBC: The management of locks is not available for databases accessed by JDBC.

- the data file is empty or there is no record corresponding to the condition: no reading is performed and [&lt;Source&gt;.Out](../WDLang4/1000024605.md) returns <u><u><u>True</u></u></u>.




This function can be used with the data files, HFSQL views or queries.

**Remarks**:

- By default, [&lt;Source&gt;.ReadSeekFirst](../WDLang4/1000025034.md) and [&lt;Source&gt;.ReadSeekLast](../WDLang4/1000025024.md) are used to perform exact-match searches.

- By default, **&lt;Source&gt;.ReadSeek** is used to perform a generic search on strings. This search is not generic on integers, reals, dates, currencies, etc.





<a name="Example1"></a>
<a name="sample_code"></a>

## Example


```wl
// Find the first record 
// for which the Customer name is MOORE
Customer.ReadSeek(Name, "MOORE")
IF Customer.Found() THEN
	Trace("Customer MOORE found")
ELSE
	Trace("Customer MOORE not found")
END
```

## Syntax

`<Result> = <Source>.ReadSeek(<Item> , <Sought value> [, <Options>])`
---

**`<Result>: (Boolean (optional))`**

Corresponds to: 

- <u><u><u><u>True</u></u></u></u> if the sought record was found and read. The file buffer is loaded with the data of the record found. In this case, [HError](../WDLang4/3044088.md) is set to 0 and [&lt;Source&gt;.Found](../WDLang4/1000025076.md) is set to <u><u><u><u>True</u></u></u></u>. 

- <u><u><u><u>False</u></u></u></u> in the following cases: 

	- error while accessing the file (unable to read for example). [HError](../WDLang4/3044088.md) returns an error code. [HErrorInfo](../WDLang4/3044071.md) returns more details about the error. In this case, [&lt;Source&gt;.Found](../WDLang4/1000025076.md) cannot be used. 

	- the access to the file was performed but no record was found. In this case, [HError](../WDLang4/3044088.md) is set to 0 and [&lt;Source&gt;.Found](../WDLang4/1000025076.md) is set to <u><u><u><u>False</u></u></u></u>. 


 Caution: In this case, the file buffer cannot be used.




**`<Source>: (Type corresponding to the specified source)`**

Name of data file, HFSQL view or query used.

**`<Item>: (Character string)`**

Name of item on which the search will be performed. 
For an exact-match search, this parameter can correspond to a non-key item.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) To perform generic searches on a composite key, all the components of the composite key must be text components. Otherwise, an exact-match search is performed.

**`<Sought value>: (Type corresponding to the value)`**

Value of the sought item.

**`<Options>: (Optional constant (or combination of constants))`**

Used to configure:

- the lock mode applied to the sought record.

- the type of search performed.

### Generic search/Exact-match search
<a name="generic_searchexactmatch_search_ELTPARAGRAPHE000475"></a>

A generic search or an exact-match search will be performed according to &lt;Sought value&gt;.

- **Generic search**: search for all the records that start with the specified value
	To perform a generic search, the desired value must be specified in &lt;Sought value&gt;. 
	Remark: For backward compatibility with WINDEV 5.5, the generic search of an empty string ("") is equivalent to the use of [&lt;Source&gt;.ReadFirst](../WDLang4/1000025022.md).  

- **Exact-match search**: search for all the records that exactly correspond to the specified value.
	To perform an exact-match search, the size of the search argument must be exactly the same as the size of the key. To perform an exact-match search, you also have the ability to use the *hIdentical* constant. 
	To perform an exact-match search, you also have the ability to use [&lt;Source&gt;.ReadSeekFirst](../WDLang4/1000025034.md).




**Caution**: If you are using Hyper File 5.5 files or files in Hyper File 5.5 format migrated to HFSQL Classic, the implementation of generic searches and exact-match searches may change. For more details, see the [Hyper File 5.5 and HFSQL Classic: How to manage the space characters in the searches?](../WDLang4/3044190.md) table

### Performing a search on a composite key
<a name="performing_search_composite_key_ELTPARAGRAPHE000528"></a>

Several methods can be used to perform a search on a composite key:

- Using a list of values, 

- Using [&lt;Source&gt;.BuildKeyValue](../WDLang4/1000024205.md),




**1. Using a list of values**

The following syntax is used to perform a search on a composite key:


```txt
HReadSeek(<File name>, <Name of composite key>,
	[<Search value of first element of composite key>, 
		<Search value of first element of composite key>, ...])
```
Example:

```wl
// Find the record
Customer.ReadSeek(LastName_FirstName, ["MOORE","Vince"])

WHILE Customer.Found()
	// Process
	Customer.ReadNext(LastName_FirstName)
END
```

**2. Using [&lt;Source&gt;.BuildKeyValue](../WDLang4/1000024205.md)**

If the sought item is a composite key, the value to find can be built by [&lt;Source&gt;.BuildKeyValue](../WDLang4/1000024205.md).

![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) To perform generic searches on a composite key, all the components of the composite key must be text components. Otherwise, an exact-match search is performed.

### Looping through records that match a condition
<a name="looping_through_records_that_match_condition_ELTPARAGRAPHE000608"></a>

In most cases, **&lt;Source&gt;.ReadSeek** sets the position in the data file to loop through the records that match a condition. [&lt;Source&gt;.ReadNext](../WDLang4/1000025035.md) and [&lt;Source&gt;.ReadPrevious](../WDLang4/1000025021.md) are used to read the next and previous matching records.

To ignore the search while going to the next or previous record, use one of the following functions:

- [&lt;Source&gt;.CancelSeek](../WDLang4/1000024193.md)

- [&lt;Source&gt;.ReadLast](../WDLang4/1000025020.md)

- [&lt;Source&gt;.ReadFirst](../WDLang4/1000025022.md)




## WDSQL: Structure of a database
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000201"></a>
WDSQL permet de connaître la structure complète de la base de données accédée.

Pour afficher la structure de la base de données, cliquez sur l'icône ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=3&name=WDSQL%20-%20HC%20N%B0006%204.gif)
.

Cette fonctionnalité est disponible :

- sur les bases de données Oracle ou SQL Server,

- sur les autres types de bases de données.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Structure d'une base de données Oracle ou SQL Server
<a name="structure_une_base_donnees_oracle_sql_server_ELTTEXTE000225"></a>
Si la base de données en cours est une base de données Oracle ou SQL Server, lors d'un clic sur l'icône ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=3&name=WDSQL%20-%20HC%20N%B0006%204.gif)
, la fenêtre suivante s'affiche :
![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=3&name=StructureBaseDonnees.gif)
Les informations affichées sont réparties dans différents onglets.

**Remarque** : Les informations affichées concernent tous les utilisateurs de la base de données (et non uniquement l'utilisateur connecté).

| Onglet | Contenu |
| --- | --- |
| Tables | Liste des tables de la base de données créées par l'utilisateur sélectionné (dans la combo "Utilisateur").<br>Pour chaque table, les informations suivantes sont affichées :<br><br>- Le nom des colonnes.<br><br>- Le type HyperFileSQL des colonnes. Ce type correspond au type utilisé sous WINDEV.<br><br>- Le type Oracle des colonnes. Ce type correspond au type des colonnes sous Oracle.<br><br>- La taille des colonnes.<br><br><br> |
| Vues | Liste des vues de la base de données créées par l'utilisateur sélectionné (dans la combo "Utilisateur").<br>Pour chaque vue, les informations suivantes sont affichées :<br><br>- Le nom des colonnes.<br><br>- Le type HyperFileSQL des colonnes. Ce type correspond au type utilisé sous WINDEV.<br><br>- Le type Oracle des colonnes. Ce type correspond au type des colonnes sous Oracle.<br><br>- La taille des colonnes.<br><br><br> |
| Synonymes | Liste des synonymes de la base de données visibles par l'utilisateur sélectionné (dans la combo "Utilisateur").<br>Pour chaque synonyme, les informations suivantes sont affichées :<br><br>- La visibilité du synonyme.<br><br>- Le nom du propriétaire du synonyme.<br><br>- Le nom du synonyme.<br><br>- Le nom d'origine de l'objet.<br><br>- La base de données distante à laquelle l'objet appartient (si nécessaire).<br><br><br> |
| Index | Liste des index de la base de données créés par l'utilisateur sélectionné (dans la combo "Utilisateur").<br>Lors de la sélection d'un index, le nom de la (ou des) colonne(s) associée(s) à cet index s'affiche. |
| Procédures | Liste des procédures stockées, des fonctions et des packages de la base de données créés par l'utilisateur sélectionné (dans la combo "Utilisateur").  L'icône ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=3&name=IconeerreurProcedure.gif)<br> précédant le nom d'une procédure indique que la procédure contient au moins une erreur de compilation.<br>Il est possible de créer, de modifier ou de supprimer une procédure stockée, une fonction ou un package.<br>Pour créer une procédure stockée (une fonction ou un package), il suffit de préciser :<br><br>- Le nom de la procédure stockée (de la fonction ou du package).<br><br>- Le code de la procédure stockée (de la fonction ou du package).<br><br><br>La procédure stockée (la fonction ou le package) est automatiquement créée dans la base de données et compilée. La liste des erreurs détectées à la compilation est affichée si nécessaire.<br>Exemple d'utilisation d'une procédure : la syntaxe suivante permet d'afficher le résultat d'une fonction :<br><br><pre><code>SELECT <NomFonction>(<Paramètres>) from dual</code></pre><br> |
| Liens (uniquement pour les bases de données Oracle) | Liste des différentes bases de données distantes accessibles depuis la base de données.<br>Pour chaque base de données, les informations suivantes sont affichées :<br><br>- Le nom du propriétaire de la base de données distante.<br><br>- Le nom du lien à utiliser pour accéder à la base de données distante.<br><br>- La chaîne de connexion utilisé par Oracle pour accéder à la base de données distante.<br><br>- La visibilité de la base de données distante.<br><br><br>Exemple : la syntaxe suivante permet de sélectionner les données présentes dans une table d'une base de données distante :<br><br><pre><code>SELECT * from <NomTable>@<NomDuLlien></code></pre><br> |
| Utilisateurs | Liste des utilisateurs définis sur la base de données.<br>Pour chaque utilisateur, la date et l'heure de création de cet utilisateur sont affichées. |

## Structure d'une base de données différentes de Oracle et SQL Server
<a name="structure_une_base_donnees_differentes_oracle_sql_server_ELTTEXTE000249"></a>
Si la base de données en cours n'est pas une base de données Oracle ou SQL Server, lors d'un clic sur l'icône ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=3&name=WDSQL%20-%20HC%20N%B0006%204.gif)
, la fenêtre suivante s'affiche :

![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=3&name=WDSQL%20-%20HC%20N%B0010.gif&type=thumb)
Les informations affichées sont réparties dans différents onglets.

| Onglet | Contenu |
| --- | --- |
| Tables | Liste des tables de la base de données.<br>Pour chaque table, les informations suivantes sont affichées :<br><br>- le nom des colonnes,<br><br>- le type des colonnes,<br><br>- la taille des colonnes.<br><br><br> |
| Vues | Liste des vues de la base de données.<br>Pour chaque vue, les informations suivantes sont affichées :<br><br>- le nom des colonnes,<br><br>- le type des colonnes,<br><br>- la taille des colonnes.<br><br><br> |

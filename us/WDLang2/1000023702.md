


## &lt;Looper&gt;.DeleteSelect (Function)

***In french: &lt;Zone répétée&gt;.SupprimeSelect***



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Deletes the selected rows from a Looper control.


<a name="Example1"></a>
<a name="sample_code"></a>

## Example


```wl
// Deletes all selected rows
IF YesNo("Do you really want to delete the selected customers?") THEN 
	LOOP_Display_Customer.DeleteSelect()
END
```

<a name="XSYNTAX"></a>

## Syntax
<a name="SYNTAX1"></a>

`<Looper control>.DeleteSelect()`
---

**`<Looper control>: (Control name)`**

Name of the Looper control to be used.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarks


### Use conditions
<a name="use_conditions_ELTPARAGRAPHE000213"></a>

**&lt;Looper&gt;.DeleteSelect** can be used on: 

- a Looper control based on a data file, 

- a Looper control populated programmatically.



<a name="NOTE0_2"></a>


### Deletion in a Looper control based on a data file
<a name="deletion_looper_control_based_data_file_ELTPARAGRAPHE000228"></a>

**&lt;Looper&gt;.DeleteSelect** deletes the corresponding records from the data file associated with the control. If items are linked to other data files, no modification is automatically performed on these linked data files.
<a name="NOTE0_3"></a>


### Referential integrity and Looper control based on a data file
<a name="referential_integrity_and_looper_control_based_data_file_ELTPARAGRAPHE000240"></a>

If the management of referential integrity is enabled ([&lt;Connection variable&gt;.SetIntegrity](../WDLang4/1000023960.md)), [HErrorIntegrity](../WDLang4/3044086.md) must be called after **&lt;Looper&gt;.DeleteSelect** to check the integrity. If an integrity error is detected, the row is not deleted and it remains visible in the control.
<a name="NOTE0_4"></a>


### Deletion in a Looper control based on a data file and Trigger
<a name="deletion_looper_control_based_data_file_and_trigger_ELTPARAGRAPHE000260"></a>

A trigger can be activated when deleting a record from a Looper control based on a data file. For more details, see the documentation about [&lt;Source&gt;.DescribeTrigger](../WDLang4/1000024216.md).
<a name="NOTE0_5"></a>


### Handling errors
<a name="handling_errors_ELTPARAGRAPHE000275"></a>

The [ErrorOccurred](../WDLang1/3087001.md) variable is set to <u><u><u><u>True</u></u></u></u> if the deletion fails. To get the details of the error, use [ErrorInfo](../WDLang1/3013008.md).
<a name="NOTE0_6"></a>


### Locking the linked data file
<a name="locking_the_linked_data_file_ELTPARAGRAPHE000295"></a>

If the data file is locked in the current process, **&lt;Looper&gt;.DeleteSelect** deletes the specified row and unlocks the data file.

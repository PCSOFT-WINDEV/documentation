## Présentation du champ Applet Java
<a name="presentation_champ_applet_java_ELTTEXTE000140"></a>


### Champ d'une page dynamique ou statique
<a name="champ_une_page_dynamique_statique_ELTPARAGRAPHE000011"></a>Un champ Applet Java permet d'intégrer simplement des Applets Java dans vos sites WEBDEV.

Pour créer un champ Applet Java, il est nécessaire de posséder :

- l'applet JAVA (fichier ".Class").

- les différents paramètres de l'applet.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Création d'un champ Applet Java
<a name="creation_champ_applet_java_ELTTEXTE000164"></a>
Pour créer un champ de type Applet Java : 

1. Sous le volet "Création", dans le groupe "Intégration", cliquez sur "Applet java".

2. Cliquez dans la page à la position où le champ doit être créé. Le champ apparaît en création.




Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ.

<a name="NOTE3_1"></a>


## Caractéristiques spécifiques au champ Applet Java
<a name="caracteristiques_specifiques_champ_applet_java_ELTTEXTE000188"></a>


### Description du champ
<a name="description_champ_ELTPARAGRAPHE000049"></a>Pour configurer le champ Applet Java, il est nécessaire de préciser dans l'onglet "Général" de la description du champ :

- **l'archive Jar** si nécessaire. Cette archive doit être précisée lorsque l'applet Java est intégrée à une archive. 

- le **nom du fichier Applet** (sans son extension) correspondant à l'applet principal à exécuter (fichier ".CLASS").
	Exemple : "Scroll"
	Remarque : Les noms des applets sont "case sensitive". Veillez à respecter les majuscules / minuscules.

- la **base**, c'est-à-dire le nom du répertoire contenant tous les composants de l'applet Java. Cette chaîne doit être précédée et suivie du caractère "/". Il est conseillé de placer tous les objets nécessaires à l'exécution de l'applet dans le répertoire "&lt;NomProjet&gt;_WEB" du projet ou dans un sous-répertoire du répertoire "&lt;NomProjet&gt;_WEB". 
	Exemple : "/SiteJava_web/MesApplets/"
	Remarque : Selon le serveur Web utilisé, les noms des répertoires sont "case sensitive". Veillez à respecter les majuscules / minuscules.

- le **tableau de paramètres**. Ce tableau regroupe tous les paramètres nécessaires à l'exécution de l'applet. La liste des paramètres est fournie par le distributeur de l'Applet. 
	***Pour ajouter un paramètre***, cliquez sur le bouton "Ajouter": Précisez le nom du paramètre (colonne "Nom du Paramètre") et sa valeur (colonne "Valeur"). 
	***Pour supprimer un paramètre***, sélectionnez le paramètre et cliquez sur le bouton "Supprimer".
	***Si l'applet Java a été créée avec WINDEV***, le nom des paramètres correspond au nom des paramètres récupérés par la fonction [LigneCommande](../WDLang1/3013051.md). 
	Exemple : Code d'initialisation du projet WINDEV servant à créer l'applet :
	
	```wl
	P1 est une chaîne
	P1 = LigneCommande("Param1")
	```

	Dans ce cas, le nom du paramètre à utiliser en WEBDEV est "Param1".



<a name="NOTE3_2"></a>


### Répertoire en cours
<a name="repertoire_cours_ELTPARAGRAPHE000084"></a>Pour indiquer la base et les paramètres, le chemin est relatif au répertoire en cours. Le répertoire en cours est le répertoire où est généré le fichier HTM correspondant à la page.

- **Dans le cas d'une page dynamique**, le répertoire en cours est le sous-répertoire du répertoire du projet correspondant à la langue (FR, UK, etc.). 
	Exemple : Si le paramètre est un fichier présent dans le répertoire "&lt;NomProjet&gt;_WEB", la valeur du paramètre correspond à "/&lt;NomProjet&gt;_WEB/&lt;Nom du fichier&gt;".

- **Dans le cas d'une page statique**, le répertoire en cours est le sous-répertoire du répertoire &lt;NomProjet&gt;_WEB correspondant à la langue (FR, UK, etc.).




**Exemple** : Si le paramètre est un fichier présent dans le répertoire "&lt;NomProjet&gt;_WEB", la valeur du paramètre correspond à "../&lt;Nom du fichier&gt;".




## Les pages AWP
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000120"></a>
Vous voulez faire référencer plusieurs pages de votre site WEBDEV ?

Vous souhaitez envoyer à un ami, l'adresse d'une des pages de votre site ?

Deux solutions : 

- Utiliser des pages AWP dans vos sites WEBDEV. 

- utiliser des pages référençables en mode session. Pour plus de détails, consultez [Page référençable en mode session](../WDChamp/2030055.md).  




Les pages en mode AWP permettent également de faire des Webhooks ou encore d'obtenir des URL pouvant être fournies à JQuery.

<a name="NOTE2_1"></a>


## Qu'est-ce qu'une page AWP ?
<a name="questce_quune_page_awp_ELTTEXTE000144"></a>


### Page AWP
<a name="page_awp_ELTPARAGRAPHE000032"></a>Une page AWP (fichier .AWP) est une page dynamique WEBDEV **sans contexte**. 
Cette page est accessible **à tout moment** par un lien du type : 

```txt
http://www.monsite.com/mapageAWP.awp
```


Une page AWP peut évidemment recevoir des paramètres dans son URL.
Exemple : Cette page permettra d'afficher le produit numéro 3254 correspondant à la catégorie 23. Même avec ces paramètres, cette page est accessible à tout moment.

```txt
http://www.monsite.com/mapageAWP.awp?idProduit=3254&idCategorie=23
```


Les pages AWP sont des pages autonomes (ces pages ne nécessitent pas de bibliothèque (WDL) pour fonctionner). Seules les images utilisées par la page sont nécessaires sur le serveur.

**Dès l'enregistrement de la page AWP, les informations spécifiées sont utiles pour le référencement de la page**. Ainsi le titre et le nom du fichier généré sont utilisés par les moteurs de recherche. Pour optimiser le référencement de votre page, utilisez un titre et un nom de fichier (ce dernier est utilisé pour construire le nom du fichier généré en lien avec les informations affichées dans la page). 


<a name="NOTE2_2"></a>


### Site WEBDEV en mode Session avec des pages AWP
<a name="site_webdev_mode_session_avec_des_pages_awp_ELTPARAGRAPHE000053"></a>Un site WEBDEV en mode Session peut contenir N pages AWP. Chacune de ces pages est accessible indépendamment et directement (sans passer par la première page en mode Session du projet).

Il est alors possible de :

- **faire référencer chacune des pages AWP directement par les moteurs de recherche**. Dans l'exemple ci-dessus, les moteurs de recherche référenceront le produit affiché par la page.

- **fournir l'adresse d'une ou plusieurs pages AWP à un ami**. Dans notre exemple, la page affichée sera directement celle du produit voulu.

- **mettre une ou plusieurs pages AWP dans les favoris**.

- **optimiser la gestion des ressources sur le serveur**.



<a name="NOTE2_3"></a>


### ATTENTION: Le mode AWP est destiné aux développeurs maîtrisant WEBDEV.
<a name="attention_mode_awp_est_destine_aux_developpeurs_maitrisant_webdev_ELTPARAGRAPHE000067"></a>La connaissance préalable de WEBDEV, et des concepts de sessions et contextes, est nécessaire à l'utilisation de ce mode avancé. Dans ce mode, il est en particulier nécessaire de gérer les contextes par programmation.
Certaines fonctionnalités liées aux contextes automatiques peuvent donc ne plus fonctionner.
Certaines limitations existent également lors du développement de ce type de pages.

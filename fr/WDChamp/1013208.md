
## Manipuler un ActiveX par programmation
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000114"></a>
WINDEV permet de manipuler un champ de type ActiveX en WLangage par l'intermédiaire :

- de propriétés et de méthodes

- d'événements




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Propriétés et méthodes de l'ActiveX
<a name="proprietes_methodes_activex_ELTTEXTE000138"></a>
Un ActiveX est associé à des propriétés et des méthodes. Ces propriétés et méthodes peuvent être utilisées en WLangage pour manipuler directement le champ ActiveX. Il suffit d'utiliser la syntaxe Automation du WLangage.

Par exemple :


```wl
// LecteurAVI est un champ ActiveX
LecteurAVI>>Zoom=12 // Affectation d'une propriété
LecteurAVI>>Pause(10) // Appel de méthode
```



<a name="NOTE2_2"></a>
Pour connaître les propriétés et méthodes associées à un ActiveX, vous pouvez :

- soit consulter directement la documentation du contrôle ActiveX.

- soit utiliser l'utilitaire [WDXVIEW](../WDXView/3526002.md) livré avec WINDEV.

- soit utiliser la complétion automatique de WINDEV. Les propriétés et méthodes de l'ActiveX sont automatiquement proposées par l'éditeur de code. Par exemple, pour l'ActiveX "Navigateur WEB Microsoft", on obtient :

## Événements associés à l'ActiveX
<a name="evenements_associes_activex_ELTTEXTE000168"></a>
Le WLangage permet de gérer les événements générés par un contrôle ActiveX grâce à la fonction [ActiveXEvénement](../WDLang1/3025010.md). Cette fonction permet d'associer une procédure de votre projet à l'événement voulu.

Par exemple :


```wl
// Code d'initialisation du champ ActiveX LecteurAVI
ActiveXEvénement("Ma_procédure","LecteurAVI","OnClick")
// "ma procédure" sera appelée à chaque fois que l'événement 
// "OnClick" sera déclenché par le contrôle "LecteurAVI".
```


Pour connaître les événements associés à un ActiveX, vous pouvez :

- soit consulter directement la documentation du contrôle ActiveX.

- soit utiliser l'utilitaire [WDXVIEW](../WDXView/3526002.md) livré avec WINDEV.

## Présentation
<a name="presentation_ELTTEXTE000112"></a>


### Champ d'une page dynamique ou statique
<a name="champ_une_page_dynamique_statique_ELTPARAGRAPHE000014"></a>Le Champ SilverLight permet d'afficher une animation de type SilverLight 2 dans une page WEBDEV.

Pour créer un champ de type SilverLight : 

1. Sous le volet "Création", dans le groupe "Intégration", déroulez "Plug-in" puis cliquez sur "Champ Silverlight".

2. Cliquez dans la page à la position où le champ doit être créé. Le champ apparaît en création.

3. Dans la description du champ, associez l'animation (fichier ".xap") au champ SilverLight.




Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ. 


### Remarque : Localisation de l'animation SilverLight 2
<a name="remarque_localisation_animation_silverlight_2_ELTPARAGRAPHE000035"></a>Il est conseillé de placer l'animation SilverLight 2 dans le répertoire des images (répertoire &lt;Nom_Projet&gt;_WEB). Les fichiers de dépendances de l'animation (images, fichiers texte, ...) doivent être copiés également dans le répertoire &lt;Nom_Projet&gt;_WEB.

Si l'animation utilise des fichiers Javascript (.js), il est nécessaire d'inclure ces fichiers dans la page (onglet "Avancé" de la description de la page).

## Affichage d'une page contenant un champ SilverLight
<a name="affichage_une_page_contenant_champ_silverlight_ELTTEXTE000142"></a>


### Principe
<a name="principe_ELTPARAGRAPHE000046"></a>Pour afficher un champ SilverLight, il est nécessaire de posséder le framework Microsoft sur le poste. Si l'internaute ne possède pas ce framework sur son poste, une fenêtre s'affiche lui proposant de télécharger et d'installer "Microsoft Silverlight". Après l'installation, il suffit de réactualiser la page pour que celle-ci s'affiche correctement.
<a name="NOTE2_2"></a>


### Cas particulier : Poste de développement ou serveur Vista (ou supérieur) sous IIS7 avec l'UAC activé : 
<a name="cas_particulier_poste_developpement_serveur_vista_superieur_sous_iis7_avec_uac_active_ELTPARAGRAPHE000053"></a>

- Si le poste utilise Windows Vista (ou supérieur), il est nécessaire d'exécuter l'installation de "Microsoft Silverlight" en tant qu'administrateur.

- Dans certains cas, il est nécessaire de réaliser une modification sous IIS 7 pour que le fichier ".xap" soit interprété sur le serveur ou sur les postes de développement (lors des tests). Il est nécessaire d'ajouter le type MIME correspondant.

	1. Ouvrez le gestionnaire de service Internet Microsoft (Option "Démarrer .. Exécuter .. Inetmgr.exe" depuis le bureau de Windows).

	2. Double-cliquez sur l'option "Type MIME".

	3. Sélectionnez l'option "Ajouter".

	4. Indiquez :

		- l'extension du fichier : ".xap".

		- le type MIME : "application/x-silverlight-app"




5. Validez.

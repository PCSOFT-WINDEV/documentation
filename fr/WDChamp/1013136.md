


## Manipuler un champ Image par programmation
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000300"></a>
WINDEV, WEBDEV et WINDEV Mobile permettent de manipuler un champ Image directement par programmation. Il suffit d'utiliser directement la variable du champ Image dans le code.

La variable du champ Image correspond au nom du champ Image. Cette variable est de type texte.  

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Afficher un fichier image dans un champ Image
<a name="afficher_fichier_image_dans_champ_image_ELTTEXTE000324"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) 

### Dans une fenêtre
<a name="dans_une_fenetre_ELTPARAGRAPHE000022"></a>Pour afficher un fichier image dans un champ Image, une simple affectation suffit.

Quelques exemples :

- affectation directe : 
	
	```wl
	IMG_MonImage = "C:\Temp\MonImage.bmp"
	```


- affectation par variable :
	
	```wl
	CheminImage = fRepEnCours() + "\MonImage.bmp"
	IMG_MonImage = CheminImage
	```

	Remarque : Un champ Image peut aussi être initialisé à partir d'une variable de type buffer ou de type texte contenant la valeur de l'image en binaire.




**Remarque** : si le champ est associé à une rubrique d'un fichier de données, l'affectation est automatiquement réalisée en utilisant la fonction [FichierVersEcran](../WDLang4/3044210.md). Pour plus de détails, consultez [Lier un champ Image à une rubrique](../WDChamp/1013135.md).

## Récupérer le nom du fichier image affiché dans le champ Image
<a name="recuperer_nom_fichier_image_affiche_dans_champ_image_ELTTEXTE000354"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) 

### Dans une fenêtre
<a name="dans_une_fenetre_ELTPARAGRAPHE000095"></a>**Pour récupérer le nom de l'image affichée dans un champ Image :**

- réalisez une simple affectation. Par exemple :
	
	```wl
	SAI_ContenuActuelImage = IMG_Image1
	```


- récupérez une partie du nom, grâce à la fonction [fExtraitChemin](../WDLang1/3036026.md). Par exemple :
	
	```wl
	NomFichierImage = fExtraitChemin(IMG_Image1, fFichier + fExtension)
	```


- utilisez la propriété [Valeur](../Proprietes/2510130.md)




**Remarque** : Pour récupérer l'image (ou son nom) dans la rubrique associée au champ Image : 

1. Si le champ est associé à une rubrique de type texte, la récupération de la valeur saisie est automatiquement réalisée en utilisant la fonction [EcranVersFichier](../WDLang4/3044146.md). Il est aussi possible d'utiliser la notation CLIENT.NOM = SAI_Saisie1.

2. Si le champ est associé à un mémo binaire, utilisez la fonction [HAttacheMémo](../WDLang4/3044173.md) pour associer le fichier image à la rubrique.

3. Si le champ Image a été affecté à partir d'une variable de type buffer ou de type chaîne, la propriété [Valeur](../Proprietes/2510130.md) renvoie une chaîne vide.
	Pour plus de détails, consultez [Lier un champ Image à une rubrique](../WDChamp/1013135.md).

## Récupérer la position de la souris dans une image cliquable
<a name="recuperer_position_souris_dans_une_image_cliquable_ELTTEXTE000384"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Pour récupérer la position de la souris dans une zone de cliquage, il suffit d'utiliser les fonctions suivantes dans le code de clic de l'image :


|   |   |
| --- | --- |

| [SourisPosY](../WDLang1/3071008.md) | Retourne la coordonnée verticale du clic souris. |

## Vider ou réinitialiser le contenu d'un champ Image
<a name="vider_reinitialiser_contenu_champ_image_ELTTEXTE000408"></a>
Pour vider ou réinitialiser un champ Image, il suffit d'affecter une chaîne vide au champ Image. Par exemple : 

```wl
IMG_MonImage = ""
```

**Remarque** : Il est également possible d'utiliser les fonctions [dDébutDessin](../WDLang1/3029009.md)/[dFinDessin](../WDLang1/3029019.md) sur le champ Image.

## Propriétés spécifiques aux champs Image
<a name="proprietes_specifiques_aux_champs_image_ELTTEXTE000432"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Les propriétés suivantes sont spécifiques à la gestion des caractéristiques d'un champ Image par programmation.


|   |   |
| --- | --- |
| [Animation](../Proprietes/2510006.md) | Permet de connaître l'état actuel de l'image animée et de lancer ou d'arrêter l'animation d'une image animée (par exemple l'animation d'une image de type GIF animée). |
| [AnimationInitiale](../Proprietes/2510012.md) | Permet de connaître l'état initial de l'animation dans un champ Image. |
| [CadrageHorizontal](../Proprietes/2510023.md) | Permet de connaître et de modifier le cadrage horizontal du libellé de l'image. |
| [CadrageVertical](../Proprietes/2510054.md) | Permet de connaître et de modifier le cadrage vertical du libellé de l'image. |
| [ImageMode](../Proprietes/2510058.md) | Permet de connaître et de modifier le mode d'affichage d'une image dans un champ Image. |


Pour connaître la liste complète des propriétés du WLangage pouvant être utilisées avec un champ de type Image, consultez [Propriétés associées à un champ Image](../WDChamp/1013138.md).

## Présentation du champ Traitement de texte
<a name="presentation_champ_traitement_texte_ELTTEXTE000270"></a>
Le champ "Traitement de texte" est un champ évolué permettant à l'utilisateur final de créer et d'éditer des documents au format docx standard directement dans son application WINDEV. Les avantages sont immédiats : gain de temps, interface homogène, aucun autre logiciel à installer/piloter/gérer. 

Grâce à ce champ, il est possible de : 

- Ouvrir, créer, modifier, enregistrer des fichiers "docx",

- Convertir des docx en PDF, en html pour envoyer des mails,

- Gérer les images et les tableaux,

- Gérer les zones de texte, 

- Sauvegarder les documents dans les bases de données HFSQL,

- Utiliser un correcteur orthographique,

- Réaliser des impressions, ...




Le ruban associé à ce champ permet de réaliser les manipulations les plus courantes. Pour plus de détails, consultez [Traitement de texte : utiliser le ruban](../WDChamp/1410087025.md). 

De nombreux raccourcis clavier sont également disponibles : 


| Raccourci | Effet | Raccourci | Effet |
| --- | --- | --- | --- |
| Ctrl + A | Sélectionne tout le texte. | Ctrl + C | Copie la sélection dans le presse-papiers. |
| Ctrl + F | Lance la recherche. | Ctrl + G | Passe la sélection en gras |
| Ctrl + H | Lance le rechercher/remplacer | Ctrl + I | Passe la sélection en italique |
| Ctrl + S | Sauvegarde le document | Ctrl + U | Passe la sélection en souligné |
| Ctrl + V | Colle le contenu du presse-papiers. | Ctrl + X | Coupe la sélection et la met dans le presse-papiers. |
| Ctrl + Y | Refait la dernière action. | Ctrl + Z | Annule la dernière action. |
| Ctrl + molette de la souris | Change le zoom. | Ctrl + 0 | Zoom de document à 100% |
| Ctrl + origine | Remonte au début du document. | Ctrl + Fin | Affiche la fin du document. |
| Page Up | Affiche la page précédente. | Page down | Affiche la page suivante. |

## Créer un champ Traitement de texte
<a name="creer_champ_traitement_texte_ELTTEXTE000294"></a>
**Pour créer un champ de type Traitement de texte** :

1. Sous le volet "Création", dans le groupe "Champs graphiques", déroulez "Bureautique" puis cliquez sur "Traitement de texte". 

2. Cliquez dans la fenêtre à la position où le champ doit être créé.




Remarque : Les dimensions du champ créé sont optimisées pour occuper l'espace disponible à la position indiquée. Si la taille du champ ne vous convient pas, utilisez la combinaison de touches Ctrl + Z : le champ retrouvera sa taille par défaut.

Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ.

## Caractéristiques du champ Traitement de texte
<a name="caracteristiques_champ_traitement_texte_ELTTEXTE000322"></a>
L'onglet "Général" de la fenêtre de description du champ Traitement de texte propose les options suivantes : 

- **Document initial** : Permet d'indiquer le document initial affiché dans le champ Traitement de texte.  

- **Avec ruban** : Indique si le ruban doit être affiché ou non. Si le ruban doit être affiché, il est possible de spécifier le ruban utilisé grâce à l'option "FI source du ruban".
	Si l'option **"FI source du ruban"** correspond à : 

	- "Aucune" : le ruban proposé par défaut sera utilisé. 

	- "Fenêtre prédéfinie" : le ruban proposé par défaut pourra être personnalisé.
			Il suffit de sélectionner la fenêtre prédéfinie par défaut dans la zone "FI source du ruban". Dans ce cas, la fenêtre "FI_WinDevFAA_BarreTT" est automatiquement intégrée dans votre projet. Cette fenêtre interne (disponible en français et anglais) peut être modifiée selon vos besoins. Cette fenêtre contient tout le code nécessaire à la gestion du champ Traitement de texte. Nous vous conseillons de vérifier les erreurs d'UI ainsi que le bon fonctionnement de la fenêtre après toute modification. 

	- Fenêtre interne du projet : Permet d'utiliser une fenêtre interne pour proposer un ruban spécifique.
			Vous pouvez utiliser n'importe quelle fenêtre interne de votre projet pour réaliser ce bandeau.


 Il est également possible d'appliquer le gabarit du projet. 

- **Autoriser le changement de document (Ouvrir, Nouveau, ... )** : Adapte le ruban pour permettre à l'utilisateur de manipuler des documents externes. Cette option permet d'afficher les boutons "Nouveau", "Ouvrir", "Enregistrer" du ruban. Si cette option est cochée, l'ouverture du document et son enregistrement doivent être effectués par l'application. 
	Remarque : Cette option n'est pas prise en compte si le ruban est personnalisé entièrement (avec une fenêtre interne de l'application par exemple).  

- **Limiter l'édition pour faciliter l'export HTML** :  Adapte le ruban et l'affichage du document pour n'afficher que les options prises en compte par l'export HTML. Si cette option est cochée : 

	- les options suivantes ne sont pas accessibles : 

		- Gestion des en-têtes et pieds de pages, 

		- Gestion de la mise en page, 

		- Gestion des sauts de pages. 




- l'affichage du document est forcé en mode "Affichage pour le Web".

- **Activer [Vérifier l'orthographe] par défaut** : Permet d'indiquer si le correcteur orthographique doit être activé. Pour plus de détails, consultez [Vérification d'orthographe basée sur les dictionnaires Hunspell](../WDChamp/1410087030.md). 

- **Détecter et transformer automatiquement les liens** : Permet d'activer la détection automatique des liens (de type http) présents dans le document affiché dans le champ. Dans ce cas, les liens sont détectés pendant la saisie, après l'appui sur les touches Espace, Tabulation ou Entrée. Si les liens sont des liens existants, le lien peut être ouvert en utilisant la combinaison de touches Ctrl + Clic de la souris.
	Remarque : Cette option est cochée par défaut.  




**Attention** : Ce champ ne gère pas l'affichage et l'écriture des caractères de droite à gauche.


## Champ cliquable dans un état
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000094"></a>
L'éditeur d'états permet de proposer des champs cliquables dans le visualisateur de rapports.

Lors de l'impression d'une facture, l'utilisateur souhaite disposer de toutes les informations concernant le client, ou de toutes les caractéristiques des produits commandés ? Rien de plus simple, il suffit de renseigner le code de clic du champ pour réaliser l'opération voulue.

Dans le visualisateur de rapports, les zones cliquables sont facilement identifiables grâce au signe ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Etat_ChampClicablesEtoile.gif)
. De plus, ces zones se colorent en jaune lors de leur survol.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Exemple
<a name="exemple_ELTTEXTE000118"></a>
Un état liste les clients affichés par code postal. L'utilisateur veut voir la fiche détaillée du client en cliquant sur son nom. La fiche client existe déjà dans le projet, il suffit de la lier au clic sur le champ Nom dans l'état.

**Pour ouvrir directement la fenêtre depuis le visualisateur de rapports** :

1. Ouvrez l'état sous l'éditeur d'états.

2. Sélectionnez le champ à cliquer et affichez les différents événements disponibles (option "Code" du menu contextuel).

3. Saisissez le code suivant dans l'événement "Clic sur xxx" : 
	
	```wl
	Ouvre(FEN_FicheClient, "Modif", MaSource.IDClient)
	```

	Ce code permet :

	- d'ouvrir la fenêtre "FEN_FicheClient" en mode modification

	- de passer en paramètre à la fenêtre l'identifiant du client cliqué (cet identifiant est présent dans la requête intégrée à l'état). Le mot-clé [MaSource](../Motscles/1511001.md) permet de manipuler la source de l'état.







**Attention** : Si une requête est associée à l'état, la requête est automatiquement fermée à la fin de l'exécution de l'état. Les rubriques de cette requête ne sont plus accessibles.

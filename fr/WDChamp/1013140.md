## Présentation du Champ Image cliquable
<a name="presentation_champ_image_cliquable_ELTTEXTE000182"></a>


### Champ d'une page
<a name="champ_une_page_ELTPARAGRAPHE000017"></a>Le Champ Image cliquable peut être utilisé pour :

- Créer une image cliquable dans une page : dans ce cas, un code de clic serveur est associé à l'image.

- Connaître les coordonnées du clic de la souris dans une page. Dans ce cas, les coordonnées sont renvoyées au serveur lors du clic sur l'image.

- Définir des zones de cliquage / map area. Il est possible d'associer une action à l'image et à chaque zone de l'image.




Pour plus de détails, consultez [Mode de fonctionnement d'une image cliquable](../WDChamp/1013158.md)

## Créer un champ Image cliquable
<a name="creer_champ_image_cliquable_ELTTEXTE000206"></a>
Pour créer un champ de type Image cliquable : 

1. Sous le volet "Création", dans le groupe "Champs usuels", déroulez "Image" et cliquez sur "Image cliquable".

2. Cliquez dans la page à la position où le champ doit être créé. Le champ apparaît en création.




**Pour afficher les caractéristiques du champ**, sélectionnez l'option "Description" dans le menu contextuel du champ. Les principales informations sont les suivantes : 

- Le bouton [...] dans l'onglet "Général" permet de sélectionner l'image associée au champ Image. En fonction de l'image sélectionnée, un assistant apparaît, permettant de :

	- **compresser automatiquement l'image** (cas des images dont la taille dépasse les 20 Ko). Cette option permet également de convertir l'image à un des formats adapté aux pages Internet. Si cette option n'est pas choisie :
			- Si l'image utilise un format non adapté aux pages internet, l'image sera convertie à chaque génération de la page, ce qui peut ralentir cette génération.
			- Si l'image utilise un format adapté aux pages mais est de taille trop élevée, le temps d'affichage de la page pourra être ralenti.
			Si vous choisissez de réaliser la conversion immédiatement, il est possible de sélectionner le format de l'image et sa qualité. Il est également possible d'utiliser le format ***JPEG progressif***. Ce format affiche une image progressivement pendant que le navigateur charge la page.  

	- **convertir automatiquement l'image** à un des formats adapté aux pages Internet. Si cette option n'est pas choisie, l'image sera convertie à chaque génération de la page, ce qui peut ralentir cette génération. Si vous choisissez de réaliser la conversion immédiatement, il est possible de sélectionner le format de l'image et sa qualité.

	- **définir l'emplacement de l'image** : il est possible de copier l'image dans le répertoire du site (répertoire _WEB) ce qui optimise le temps de génération de la page, ou bien de laisser l'image à son emplacement (option conseillée lors du partage du fichier entre plusieurs intervenants).




- **Choix du type d'image** (onglet "Général") : Pour plus de détails sur le type d'image à choisir (statique, dynamique, ...), consultez [Choisir le type d'image](../WDChamp/1013132.md).
	![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Les images dynamiques ne sont pas gérées.

- **Protection des images** (onglet "Détail") : Si cette option est cochée, le menu contextuel de l'image ne pourra pas être affiché dans le navigateur. Toutes les manipulations de l'image disponibles au niveau du navigateur (enregistrement, impression, ...) ne pourront pas être réalisées. Cette option est également disponible dans les [champs Image](../WDChamp/1013218.md) et les [champs Vignette](../WDChamp/1013275.md).

- **Mode d'affichage** (onglet "Général"): Pour plus de détails sur le mode d'affichage et de transparence de l'image, consultez [Choisir le mode d'affichage de l'image](../WDChamp/1013134.md).

- **Transition** (onglet "Général") : Lorsque la valeur du champ Image est modifiée, une transition peut être sélectionnée comme par exemple Fondu, Recouvrement ou Balayage pour rendre le changement d'image plus harmonieux. Pour plus de détails, consultez [Animer une image](../WDChamp/9500117.md). 

- **Défilement automatique** (onglet "Général") : Cette option est accessible uniquement pour les images de type "dynamiques" ou de type "depuis une base de données : chemin". Pour plus de détails, consultez [Image avec défilement automatique](../WDChamp/9500118.md).

- Il est également possible d'associer une action à l'image (onglet "Général"). Pour plus de détails, consultez [Choisir l'action associée à un champ](../WDChamp/1013165.md).

## Site multilingue
<a name="site_multilingue_ELTTEXTE000230"></a>
Si votre site est multilingue, il est possible de sélectionner une image différente pour chaque langue. Cette option est intéressante par exemple lorsque les images contiennent du texte.

**Pour gérer des images multilingues** : 

1. Affichez la description du champ Image cliquable.

2. Cliquez sur l'icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=mutlti_langue_img.gif). La fenêtre de gestion des images multilingues apparaît. Par défaut, le même fichier est associé à toutes les langues gérées par la page.

3. Pour chaque langue, cliquez sur le bouton [...] et sélectionnez l'image correspondant à la langue. En exécution, en fonction de la langue choisie par la fonction [Nation](../WDLang1/3054014.md), l'image correspondante sera affichée.
	**Attention** : Seuls les fichiers image sont gérés par le multilingue et non les caractéristiques d'affichage de l'image.

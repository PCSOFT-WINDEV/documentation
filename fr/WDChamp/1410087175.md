## Présentation du champ Panneau dockable
<a name="presentation_champ_panneau_dockable_ELTTEXTE000174"></a>
Le champ Panneau dockable permet d'organiser facilement l'interface des logiciels en zones librement déplaçables par l'utilisateur final. Il est ainsi possible de créer des interfaces sophistiquées en toute facilité.

Les utilisateurs finaux seront comblés par les possibilités de personnalisation qui leur sont offertes. Et pour vos équipes: zéro ligne de code !

## Créer un champ Panneau dockable
<a name="creer_champ_panneau_dockable_ELTTEXTE000198"></a>
**Pour créer un champ de type Panneau_dockable** :

1. Sous le volet "Création", dans le groupe "Conteneurs", cliquez sur "Panneau dockable".

2. Cliquez dans la fenêtre à la position où le champ doit être créé. Des flèches placées sur les bord de la fenêtre permettent de docker le panneau dès sa création. 




Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ. 


### Création par refactoring
<a name="creation_par_refactoring_ELTPARAGRAPHE000040"></a>Pour créer un champ Panneau dockable à partir d'une sélection : 

1. Sélectionnez les champs à afficher dans un champ Panneau dockable. 

2. Sous le volet "Modification", dans le groupe "Transformations", déroulez "Refactoring et permutations" et sélectionnez l'option "Créer un panneau dockable avec la sélection". 
	Remarque : Il est également possible d'utiliser le menu contextuel de la sélection (option "Refactoring .. Créer un panneau dockable avec la sélection").

3. Le champ Panneau dockable est créé automatiquement. 




**Astuce** : Comment transformer des fenêtres soeurs en champs Panneau dockable ? 
Votre application utilise des fenêtres affichées par la fonction [OuvreSoeur](../WDLang1/3038045.md) pour afficher des détails relatifs à la fenêtre principale. Pour transformer simplement ces fenêtres en champs Panneau dockables : 

1. Sélectionnez le contenu de la fenêtre soeur. 

2. Créez une fenêtre interne avec le contenu de la sélection (option "Refactoring .. Créer une fenêtre interne avec la sélection" du menu contextuel de la sélection). 

3. Enregistrez la fenêtre interne. 

4. Créer un champ Panneau dockable. 

5. Créez dans le champ Panneau dockable un champ Fenêtre interne lié à la fenêtre interne précédemment créé.

## Caractéristiques du champ Panneau dockable
<a name="caracteristiques_champ_panneau_dockable_ELTTEXTE000228"></a>


### Fenêtre de description du champ 
<a name="fenetre_description_champ_ELTPARAGRAPHE000080"></a>Les principales options du champ Panneau dockable disponibles dans la fenêtre de description du champ sont les suivantes : 

- **Icône** (onglet "Général") : Permet d'associer une icône au champ Panneau dockable. Cette icône sera affichée dans les volets d'onglet, ...

- **Positions de dockage autorisées** (onglet "Détail") : Permet de définir les positions de dockage autorisées sur la fenêtre en cours. Le champ Panneau dockable peut être docké en haut de la fenêtre, en bas, à gauche et / ou à droite de la fenêtre. 

- **Avec barre de titre** (onglet "Détail") : Si cette option est cochée, le champ Panneau dockable possédera une barre de titre contenant le titre défini dans l'onglet "Général"). Il est possible d'indiquer la hauteur de la barre de titre ainsi que la présence ou non des boutons permettant de fermer et de replier le panneau dockable (options Avec bouton  et Avec bouton . 

- **Autoriser le dockage avec un autre panneau (onglet)** (onglet"Détail") : Si cette option est cochée, le champ Panneau dockable en cours pourra être docké avec un autre champ Panneau dockable. Dans ce cas, il est possible de définir la position du volet représentant le panneau dockable. 

- **Redimensionnable** : Permet à l'utilisateur final de redimensionner le champ Panneau dockable. 




### Style du champ 
<a name="style_champ_ELTPARAGRAPHE000119"></a>L'onglet "Style" de la fenêtre de description du champ permet notamment de personnaliser : 

- le cadre du séparateur, 

- l'image des volets, 

- l'image des flèches et des boutons d'action (c'est-à-dire l'image des flèches de dockage et les pictos des boutons &lt;Fermer&gt; et &lt;Replier&gt;). Cette image correspond à une planche d'images de 5 x 2. La première ligne correspond aux 5 flèches possibles pour l'état "Repos" et la seconde ligne correspond aux 5 flèches possibles pour l'état "Survol".

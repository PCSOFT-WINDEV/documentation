
## Exécution automatique d'un bouton (Fenêtre)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000098"></a>
Pour forcer une action sur une fenêtre, il suffit de définir un bouton de cette fenêtre en mode "Exécution automatique". Le principe est très simple, si au bout d'une certaine durée, aucun bouton de la fenêtre n'a été utilisé, le traitement du bouton "Exécution automatique" sera automatiquement exécuté.

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fenetre-btn_Auto.gif)


En exécution, la durée est automatiquement décrémentée. A zéro, le code du bouton est automatiquement exécuté. Ce mécanisme est automatiquement arrêté dans les cas suivants :

- si l'utilisateur clique sur un bouton.

- si l'utilisateur modifie un champ.

- si l'utilisateur appuie sur n'importe quelle touche du clavier.




**Remarque** : L'utilisateur peut lui-même ajouter une minuterie sur bouton. C'est une des Fonctionnalités Automatiques de vos Applications (FAA). Pour plus de détails, consultez [Minuterie sur bouton](../Editeurs/9500100.md).

## Mise en place
<a name="mise_place_ELTTEXTE000122"></a>


### Définition du bouton d'exécution automatique
<a name="definition_bouton_execution_automatique_ELTPARAGRAPHE000031"></a>La définition du bouton d'exécution automatique peut se faire :

- soit dans la description de la fenêtre (onglet "Détail"): 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Btn_automatique.gif)


- soit dans la description du champ Bouton (onglet "Détail") : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Btn_automatique_btn.gif)



Un message indique si plusieurs boutons d'exécution automatique ont été définis dans la fenêtre.

## Programmation
<a name="programmation_ELTTEXTE000146"></a>
Il est possible également de définir un bouton de validation automatique grâce à la fonction WLangage [DélaiAvantFermeture](../WDLang1/3038040.md).

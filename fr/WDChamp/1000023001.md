## Présentation du champ Publicité
<a name="presentation_champ_publicite_ELTTEXTE000165"></a>
Le champ Publicité permet de monétiser une application Android en affichant un bandeau de publicité. Ce bandeau va générer des revenus en fonction du nombre de clics effectués par les utilisateurs de l'application. 

Le protocole utilisé pour l'affichage des publicités est "AdMob" (Google). 

**Attention** : L'utilisation du champ Publicité est soumise à des conditions d'utilisations spécifiques. Pour plus de détails, consultez la page d'aide [Conditions d'utilisation du champ Publicité](../WDChamp/1000023005.md).

## Créer un champ Publicité
<a name="creer_champ_publicite_ELTTEXTE000189"></a>


### Pour créer un champ de type Publicité :
<a name="pour_creer_champ_type_publicite_ELTPARAGRAPHE000025"></a>

1. Sous le volet "Création", dans le groupe "Champs graphiques", cliquez sur "Publicité".

2. Cliquez dans la fenêtre à la position où le champ doit être créé.




**Pour afficher les caractéristiques du champ**, sélectionnez l'option "Description" dans le menu contextuel du champ. 


### Taille des publicités
<a name="taille_des_publicites_ELTPARAGRAPHE000045"></a>Le menu déroulant à droite de l'icône de création du champ propose quatre tailles possibles : 

- 320 x 50, pour les téléphones et les tablettes. 

- 300 x 250, pour les tablettes uniquement. 

- 468 x 60, pour les tablettes uniquement.

- 768 x 90, pour les tablettes uniquement.




**Attention** : 

- La taille choisie sera la taille minimale du champ. Le champ pourra être agrandi en édition ou en exécution. Dans ce cas, la bannière restera centrée dans le champ. 

- Si le champ Publicité n'a pas une taille suffisante pour afficher la publicité, aucune publicité ne sera affichée.

## Fonctionnement du champ Publicité
<a name="fonctionnement_champ_publicite_ELTTEXTE000219"></a>


### Mode Test
<a name="mode_test_ELTPARAGRAPHE000064"></a>AdMob n'autorise pas l'éditeur de l'application à cliquer sur ses propres annonces ou à augmenter artificiellement le nombre d'affichages de publicités. 

Le mode test du champ Publicité permet d'afficher de "fausses publicités" sur lesquelles le clic n'engendre pas de revenu.

Le mode test est automatiquement : 

- activé lorsque l'application est lancée sur l'émulateur. 

- désactivé en exécution sur le téléphone ou la tablette.




**Remarques** : 

- En exécution dans le simulateur, le champ Publicité affichera une image fixe de la taille de la bannière de publicité.

- La propriété [ModeTest](../Proprietes/1000020179.md) permet de savoir si le mode test est actif ou non.





### Chargement et rafraîchissement des publicités
<a name="chargement_rafraichissement_des_publicites_ELTPARAGRAPHE000084"></a>Une publicité est automatiquement chargée dans le champ Publicité à l'ouverture de la fenêtre. 

De nouvelles publicités peuvent être ensuite chargées régulièrement dans le champ selon la fréquence de rafraîchissement définie dans le paramétrage de l'application sur le site d'AdMob. 

Les publicités sont chargées en tâche de fond depuis Internet. L'appareil doit donc être connecté pour que les publicités puissent être affichées. **Un délai allant de quelques secondes à plusieurs minutes peut être nécessaire entre l'ouverture de la fenêtre et l'affichage effectif de la première publicité**. Ce délai dépend entre autres de la qualité de la connexion et de la taille de la publicité. 

Il est conseillé d'utiliser les événements "Chargement d'une publicité" et "Erreur de chargement d'une publicité" pour contrôler l'affichage des publicités dans le champ. Pour plus de détails, consultez [Evénements associés au champ Publicité](../WDChamp/1000023002.md). 

**Remarque** : Il est possible de savoir si une publicité est affichée dans le champ grâce à la propriété [PublicitéChargée](../Proprietes/1000020180.md).


### Clic sur une publicité
<a name="clic_sur_une_publicite_ELTPARAGRAPHE000103"></a>Les conséquences d'un clic sur une publicité affichée dans le champ Publicité dépendent du type de la publicité. Les actions les plus courantes sont : 

- Ouverture du market pour télécharger l'application correspondant à la publicité.

- Ouverture du navigateur sur le site correspondant à la publicité.

## Modification des permissions
<a name="modification_des_permissions_ELTTEXTE000255"></a>
L'utilisation d'un champ Publicité entraîne l'ajout automatique des permissions suivantes à la liste des permissions requises par l'application : 

- INTERNET

- ACCES_NETWORK_STATE




## Les graphes de type "Histogramme"
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000227"></a>
Les données sont interprétées comme des "barres", dont la hauteur est proportionnelle à la valeur de la donnée.

Les graphes de type histogramme permettent de représenter l'évolution des données sur une période donnée. Ils facilitent également les comparaisons entre des éléments.

Les catégories sont organisées horizontalement et les valeurs verticalement afin de mettre en évidence leur variation dans le temps.

**Remarque** : Un graphe de type Histogramme horizontal est également disponible.

## Caractéristiques d'un histogramme
<a name="caracteristiques_histogramme_ELTTEXTE000251"></a>

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Genhisto.gif&type=thumb)
Dans ce graphe :

**1. Valeurs des données.** 
Ce paramètre est affiché si l'option "Afficher les valeurs" est cochée dans l'onglet "Détail" de la description du champ Graphe.

**2. Titre de l'axe vertical**. 
Ce paramètre est affiché si le titre de l'axe a été défini dans l'onglet "Général" de la description du champ Graphe. Ce paramètre peut être défini également par programmation grâce à la fonction [grTitreAxe](../WDLang3/3042057.md). Il est également possible de définir un titre pour l'axe horizontal.

**3. Graduation de l'axe vertical**. 
Par défaut, la graduation est calculée automatiquement. L'onglet "Général" de la description du champ permet :

- de définir une graduation spécifique.

- de définir une échelle des abscisses en fonction du temps. Cette option fonctionne uniquement avec des données de type Date ou DateHeure provenant d'un champ Table ou d'une rubrique HFSQL. Cette option ne peut pas être utilisée avec une série remplie par programmation.

- de définir une échelle des ordonnées logarithmique.

- d'indiquer l'orientation des graduations.




Il est également possible de demander à utiliser l'échelle secondaire pour une ou plusieurs séries (onglet "Séries").

**4. Titre du graphe**. 
Les caractéristiques du titre du graphe sont saisies dans l'onglet "Détail" de la description du champ (bouton "Editer le titre du graphe").

Il est possible de définir le titre du graphe, sa position et son style.

**5. Légende**. 
Les caractéristiques de la légende sont saisies dans l'onglet "Détail" de la description du champ (bouton "Editer la légende du graphe").

Il est possible de définir sa position et son style.

**6. Etiquettes de séries**. 
Le libellé de la légende correspond au libellé des différentes séries affichées dans le graphe (définie dans l'onglet "Séries").

**7. Couleur de série**. 
Pour chaque série, il est possible de définir une couleur dans l'onglet "Séries" de la description du champ. Par défaut la couleur utilisée est automatique.

**8. Etiquette de catégorie**. 
Les étiquettes de catégorie peuvent être définies dans l'onglet "Séries" de la description du champ.

**9. Espacement entre les barres de l'histogramme**. 
Cet espacement est défini dans l'onglet "Général" de la description du champ.

**10. Quadrillage**. 
Il est possible de définir le quadrillage horizontal et vertical dans l'onglet "Général" de la description du champ.

### Caractéristiques supplémentaires
<a name="caracteristiques_supplementaires_ELTPARAGRAPHE000125"></a>

- ***Disposition des barres de l'histogramme*** : les barres de l'histogramme peuvent être groupées, empilées ou en profondeur (onglet "Général" de la description du champ). Les histogrammes empilés ne gèrent pas les valeurs négatives.

- Lors d'un affichage en 3 dimensions, il est possible d'utiliser des ***barres cylindriques***.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Un graphe de type Histogramme peut autoriser le ***zoom ***(horizontal et/ou vertical). Dans ce cas, l'utilisateur peut zoomer dans le graphe pour visualiser une donnée.

- Il est possible d'afficher ***une image en fond du graphe***. Cette image est sélectionnée dans l'onglet "Détail" de la description du champ.

- Il est possible d'utiliser une ***image pour les barres d'histogramme*** (onglet "Séries" de la fenêtre de description d'un graphe).

- Il est possible de définir ***une couleur spécifique pour la bordure des barres ***d'histogrammes :

	1. Vérifiez que la couleur de bordure n'est pas automatique (option "Couleurs de bordure automatiques" dans l'onglet "Détail" de la description du champ).

	2. Dans l'onglet "Style", sélectionnez l'élément "Couleur de bordure" et sélectionnez la couleur voulue.




- L'onglet "Style" permet de paramétrer le style de nombreux éléments : couleur du quadrillage, des axes, police du titre, des axes, ...

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Les graphes de type "Histogramme" peuvent être affichés avec une animation. Pour animer ce type de graphe, il suffit de cocher l'option "Activer les animations" dans l'onglet "Détail" de la description du champ. 

- **Graphe de type Histogramme empilé** : Par défaut, une bulle d'aide affiche la valeur de chaque série. 
	Il est possible d'afficher le total de la pile en plus de total de chaque série en utilisant la fonction [grBulle](../WDLang3/3042010.md). Exemple :  
	
	```wl
	grBulle(MoiMême, grFormatBulleEntêteMire, "[%CATEGORIE%], Somme = [%VALEUR%]")
	```


- **Agrégation des données** : 

	- Pour les graphes sans échelle temporelle, l'agrégation n'est pas activée par défaut.

	- Pour les graphes avec échelle temporelle, l'agrégation par défaut est la somme des barres.




- Il est possible de définir l'algorithme d'agrégation via la fonction [grHistoAgrégation](../WDLang3/1000023015.md). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Les graphes de type "Histogramme" peuvent être affichés en temps réel. Pour plus de détails, consultez [Graphe avec affichage en temps réel](../WDChamp/1000021037.md).

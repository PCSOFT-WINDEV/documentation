


## Le champ Cube
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000147"></a>
Le champ Cube est un champ de décor, mobile, qui effectue automatiquement des rotations sur tous les axes. Ce champ peut par exemple être présent en petite taille dans un coin d'une fenêtre.

**Attention** : Ce champ nécessite le framework DotNet 3.0 ou supérieur pour fonctionner. Ce framework est inclus sous Vista (et supérieur) et téléchargeable pour Windows XP (sur le site de Microsoft). La première utilisation d'un champ utilisant ce framework (champ Carrousel, champ Cube ou champ Xaml) demande le chargement de ce framework, ce qui peut entraîner un délai (indépendant de notre volonté).

**Pour créer un champ de type "Cube"** : 

1. Sous le volet "Création", dans le groupe "Autres champs", cliquez sur "Cube".

2. Cliquez dans la fenêtre à la position où le champ doit être créé. Le champ apparaît en création.




Pour afficher les caractéristiques du champ , sélectionnez l'option "Description" dans le menu contextuel du champ.

Il est nécessaire d'associer le champ Cube à une image. Cette image sera automatiquement animée par le champ Cube. Cette image peut être sélectionnée dans l'onglet "Général" de la description du champ.

## Manipuler le champ Cube par programmation
<a name="manipuler_champ_cube_par_programmation_ELTTEXTE000171"></a>
WINDEV permet de manipuler un champ Cube directement par programmation. Il suffit d'utiliser directement la variable du champ Cube dans le code. Cette variable est de type numérique.

La variable du champ Cube correspond au nom du champ Cube.

**Fonctions spécifiques à la gestion du champ Cube**
Les fonctions suivantes permettent de manipuler les champs Cube :



|   |   |
| --- | --- |
| [XamlDésactive](../WDLang1/1000015004.md) | Simule l'absence du framework DotNet 3.0. |




**Propriétés spécifiques à la gestion des champs Cube**
Les propriétés suivantes permettent de manipuler les champs Cube :



|   |   |
| --- | --- |
| [PériodeAnimation](../Proprietes/2510152.md) | La propriété **PériodeAnimation** permet de connaître ou modifier la vitesse de rotation de l'animation des champs Cube ou Carrousel. |




Pour connaître la liste complète des propriétés du WLangage pouvant être utilisées avec un champ de type Cube, consultez [Propriétés associées au champ Cube](../WDChamp/1016101.md).

## Notes
<a name="notes_ELTTEXTE000213"></a>


### Limitations
<a name="limitations_ELTPARAGRAPHE000070"></a>Les limitations du champ Cube sont les suivantes :

- Le framework DotNet 3.0 ou supérieur doit être installé sur le poste. Ce framework est inclus sous Vista et téléchargeable pour Windows XP (sur le site de Microsoft). 
	La première utilisation d'un champ utilisant ce framework (Carrousel, Cube, Xaml) demande le chargement de ce framework, ce qui peut entraîner un délai (indépendant de notre volonté).

- L'application ne peut pas être lancée depuis le réseau.

- Le GFI (Grisage des Fenêtres Inaccessibles) est désactivé sur toutes fenêtres contenant un champ Cube.

- Ce champ ne fonctionne pas sous TSE.

- Ce champ utilise la librairie wd280xaml.dll. Cette librairie ne peut pas être renommée lors de la création de l'exécutable.



<a name="NOTE3_2"></a>


### Simuler l'absence du framework DotNet 3.0
<a name="simuler_absence_framework_dotnet_30_ELTPARAGRAPHE000087"></a>Lors du développement d'une application contenant un champ Cube, il peut être utile de savoir comment réagira l'application sur une machine ne disposant pas du framework DotNet 3.0 ou supérieur.

Pour simuler l'absence du framework sur une machine :

- soit utilisez la fonction [XamlDésactive](../WDLang1/1000015004.md).

- soit renommez la clé de registre suivante :
	 "HKEY_LOCAL_MACHINE\\SOFTWARE\\Microsoft\\.NETFramework\\v3.0".

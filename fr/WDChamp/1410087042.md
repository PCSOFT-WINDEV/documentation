## Présentation du champ Conteneur natif
<a name="presentation_champ_conteneur_natif_ELTTEXTE000091"></a>
Le champ Conteneur natif permet de gérer les champs qui ne sont pas créés avec WINDEV ou WINDEV Mobile. Il est ainsi facile d'inclure dans une fenêtre des champs qui proviennent d'un module tiers, comme un SDK. 

Le champ Conteneur renvoie le handle (UIView en iOS, Identifiant de View en Android) attendu par le SDK. Ce handle peut ensuite être utilisé directement pour manipuler le champ.

<a name="NOTE2_1"></a>


## Créer un champ Conteneur natif
<a name="creer_champ_conteneur_natif_ELTTEXTE000115"></a>
Pour créer un champ de type Conteneur natif :

1. Sous le volet "Création", dans le groupe "Autres champs", cliquez sur "Conteneur natif". 

2. Cliquez dans la fenêtre à la position où le champ doit être créé. Le champ apparaît en création.
	Remarque : Les dimensions du champ créé sont optimisées pour occuper l'espace disponible à la position indiquée. Si la taille du champ ne vous convient pas, redimensionnez le champ grâce à ses poignées. 




Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ.

Pour manipuler le champ Conteneur natif, consultez [Manipuler un champ Conteneur natif par programmation](../WDChamp/1410087045.md).




## Filtres dans les champs Table fichier
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000235"></a>
Pour filtrer les enregistrements affichés dans un champ de type Table fichier, WINDEV, WEBDEV et WINDEV Mobile mettent plusieurs moyens à votre disposition :

- Filtre sur la clé de parcours, défini dans la fenêtre de description du champ Table.

- Propriété [Filtre](../Proprietes/2510042.md).

- Fonction [HFiltre](../WDLang4/3044100.md).

- Evénement optionnel de filtrage des enregistrements d'un champ Table. Pour plus de détails, consultez [Evénements associés aux champs Table](../WDChamp/1013189.md).

## Description du champ Table
<a name="description_champ_table_ELTTEXTE000259"></a>

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Champ_Table_contenu.gif)
L'option "Afficher les enregistrements dont la rubrique de parcours commence par" permet de décrire un filtre sur un champ de type Table fichier.

Le filtre a les caractéristiques suivantes :

- Filtre de type "Commence par". La condition est saisie directement sous l'éditeur.

- Filtre uniquement pour l'affichage des champs de type Table fichier.

- Filtre n'influant pas sur le parcours du fichier de données (fonctions [HLitPremier](../WDLang4/3044051.md), ...).

### Option "Utiliser la fonction HFiltre pour parcourir le fichier" décochée
<a name="option_utiliser_fonction_hfiltre_pour_parcourir_fichier_decochee_ELTPARAGRAPHE000064"></a>Si l'option "Utiliser la fonction HFiltre pour parcourir le fichier" est décochée, le filtre a les caractéristiques suivantes :

- Filtre de type "Commence par". La condition est saisie directement sous l'éditeur.

- Filtre uniquement pour l'affichage des champs de type Table fichier. 

- Filtre n'influant pas sur le parcours du fichier de données (fonctions [HLitPremier](../WDLang4/3044051.md), ...)

- Seule la colonne associée à la rubrique de parcours pourra être triable (option "Triable" dans l'onglet "Général").

- Aucune recherche ne pourra être effectuée par l'utilisateur dans le champ Table (option "Recherche (loupe/filtre)" dans l'onglet "Général").

### Option "Utiliser la fonction HFiltre pour parcourir le fichier" cochée
<a name="option_utiliser_fonction_hfiltre_pour_parcourir_fichier_cochee_ELTPARAGRAPHE000102"></a>Si l'option "Utiliser la fonction HFiltre pour parcourir le fichier" est cochée, le filtre a les caractéristiques suivantes :

- Filtre de type "Commence par". La condition est saisie directement sous l'éditeur.

- Filtre pour l'affichage des champs de type Table fichier.

- Filtre influant sur le parcours du fichier de données (fonctions [HLitPremier](../WDLang4/3044051.md), ...).

- L'utilisateur pourra trier les informations affichées dans les colonnes associées à des rubriques clé (option "Triable" dans l'onglet "Général").

- L'utilisateur pourra effectuer des recherches dans les colonnes associées à des rubriques clé (option "Recherche (loupe/filtre)" dans l'onglet "Général").




**Cas particulier** : Avant l'ouverture de la fenêtre contenant le champ Table, si un filtre a été défini par la fonction [HFiltre](../WDLang4/3044100.md) sur le fichier de base du champ, le champ Table filtrera automatiquement les enregistrements affichés (filtre saisi directement sous l'éditeur ou par la propriété [Filtre](../Proprietes/2510042.md)). Lors de la fermeture de la fenêtre, plus aucun filtre ne sera actif sur le fichier de données.

**Attention** : Vérifiez quel est le filtre actif lors des parcours de fichier de données. Deux filtres ne pouvant pas être activés simultanément, seul le dernier filtre créé avec la fonction [HFiltre](../WDLang4/3044100.md) est conservé.

## Propriété Filtre
<a name="propriete_filtre_ELTTEXTE000295"></a>
La propriété [Filtre](../Proprietes/2510042.md) permet de paramétrer dynamiquement les caractéristiques du filtre appliqué au champ Table.
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Le filtre appliqué prend en compte l'option "Utiliser la fonction HFiltre pour parcourir le fichier" définie sous l'éditeur.

<a name="NOTE4_1"></a>


## Fonction HFiltre
<a name="fonction_hfiltre_ELTTEXTE000319"></a>
La fonction [HFiltre](../WDLang4/3044100.md) permet de filtrer le fichier de base du champ Table. La fonction [HFiltre](../WDLang4/3044100.md) permet d'utiliser tout type de filtre (y compris les filtres de type "Commence par").

Si la fonction [HFiltre](../WDLang4/3044100.md) n'est pas utilisée dans le code d'initialisation du champ Table, il est nécessaire de ré-afficher le contenu du champ Table avec la fonction [TableAffiche](../WDLang1/3074003.md).

**Remarque** : Lorsqu'un champ de type Table fichier est rempli automatiquement (par WINDEV, WEBDEV ou WINDEV Mobile), lors de l'exécution du code d'initialisation, le champ Table n'est pas encore rempli. Il est possible de filtrer les enregistrements avec la fonction [HFiltre](../WDLang4/3044100.md).

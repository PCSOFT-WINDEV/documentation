
## Manipuler un champ par programmation dans un état
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000210"></a>
Il est possible de manipuler tous les champs d'un état directement par programmation. Il suffit d'utiliser directement la variable du champ dans le code. La variable du champ correspond au nom du champ. Il est ainsi possible :

- d'affecter une valeur à un champ d'un état (ou de modifier sa valeur)

- de récupérer la valeur d'un champ d'un état (cas plus rare, la valeur ne pouvant pas être modifiée par l'utilisateur)




**Remarque** : Pour plus de détails sur la manipulation d'un champ d'un état dans un traitement WLangage, consultez [Manipuler un élément dans un traitement WLangage](../WDChamp/1014007.md).





<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Initialiser ou modifier la valeur d'un champ
<a name="initialiser_modifier_valeur_champ_ELTTEXTE000234"></a>


### Cas général
<a name="cas_general_ELTPARAGRAPHE000042"></a>Pour initialiser une valeur, une simple affectation de la variable du champ suffit.

Par exemple, pour un champ calculé :


```wl
// Calcul du prix TTC du produit en cours
CALC_MonChampCalculé = Produit.Prix * 1.196
```


**Remarque sur les champs calculés** : Si le champ est rempli automatiquement par l'état (calcul depuis un champ ou une rubrique de l'état), il est déconseillé d'affecter par programmation une valeur au champ "Calculé".


<a name="NOTE2_2"></a>


### Cas particulier : Champ Interrupteur
<a name="cas_particulier_champ_interrupteur_ELTPARAGRAPHE000056"></a>Pour initialiser un interrupteur, une simple affectation de la variable du champ "Interrupteur" suffit. Par défaut, à l'affichage d'un interrupteur, aucune option n'est cochée.

**Pour cocher une option d'un interrupteur**, utilisez la syntaxe suivante :


```txt
<Nom de l'interrupteur> = 1
<Nom de l'interrupteur> = Vrai
```


**Pour décocher une option d'un interrupteur**, utilisez la syntaxe suivante :


```txt
<Nom de l'interrupteur> = 0
<Nom de l'interrupteur> = Faux
```


Par exemple :


```wl
// Affectation de la valeur d'un interrupteur
INT_MonInterrupteur = Vrai
```



<a name="NOTE2_3"></a>


### Cas particulier : Champ Image
<a name="cas_particulier_champ_image_ELTPARAGRAPHE000076"></a>Pour afficher un fichier image dans un champ "Image", une simple affectation suffit.

Par exemple :

- affectation directe :
	
	```wl
	IMG_MonImage = "C:\Temp\MonImage.BMP"
	```


- affectation par variable : 
	
	```wl
	CheminImage = fRepEnCours() + "\MonImage.BMP"
	IMG_MonImage = CheminImage
	```

## Récupérer la valeur d'un champ
<a name="recuperer_valeur_champ_ELTTEXTE000270"></a>


### Pour récupérer le contenu d'un champ :
<a name="pour_recuperer_contenu_champ_ELTPARAGRAPHE000096"></a>

- réalisez une simple affectation. Par exemple :
	
	```wl
	ContenuActuelChampCalculé = CALC_MonChampCalculé
	```


- manipulez directement le champ. Par exemple :
	
	```wl
	SI CALC_MonChampCalculé>10 ALORS ...
	```


- utilisez la propriété [Valeur](../Proprietes/2510130.md).





<a name="NOTE3_2"></a>


### Cas particulier : Champ Libellé
<a name="cas_particulier_champ_libelle_ELTPARAGRAPHE000113"></a>Pour récupérer le contenu d'un champ "Libellé" :

- réalisez une simple affectation. Par exemple :

- manipulez directement le champ "Libellé". Par exemple :

- utilisez la propriété [Valeur](../Proprietes/2510130.md) ou la propriété [Libellé](../Proprietes/2510053.md).





<a name="NOTE3_3"></a>


### Cas particulier : Champ prédéfini
<a name="cas_particulier_champ_predefini_ELTPARAGRAPHE000131"></a>Il n'est pas possible de récupérer le nombre total de pages d'un état. Ce nombre est connu uniquement lors de l'impression de la dernière page de l'état.


<a name="NOTE3_4"></a>


### Cas particulier : Champ Interrupteur
<a name="cas_particulier_champ_interrupteur_ELTPARAGRAPHE000139"></a>Pour connaître la valeur d'un champ "Interrupteur", utilisez la syntaxe suivante :


```txt
<Valeur_Champ> = <Nom de l'interrupteur>
```


Dans ce cas, le paramètre &lt;Valeur_Champ&gt; est un booléen qui vaut :

- <u><u><u><u>Vrai</u></u></u></u> (ou 1) si la case est cochée

- <u><u><u><u>Faux</u></u></u></u> (ou 0) si la case n'est pas cochée.




Il est possible par exemple de :

- réaliser un test sur la valeur d'un champ "Interrupteur". Par exemple :
	
	```wl
	SI INT_MonInterrupteur = Vrai ALORS
		// Le champ MonChamp devient visible
		MonChamp.Visible = Vrai
	FIN
	```


- manipuler directement le champ "Interrupteur". Par exemple : 
	
	```wl
	Info(INT_MonInterrupteur)
	```

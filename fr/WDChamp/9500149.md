## Les différents types de champs de saisie
<a name="les_differents_types_champs_saisie_ELTTEXTE000300"></a>


### Champ de saisie de type Texte
<a name="champ_saisie_type_texte_ELTPARAGRAPHE000011"></a>Ce type de champ de saisie permet d'afficher ou de saisir une chaîne de caractères (nom, adresse, ...).

Remarques : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Les champs de saisie de type Texte peuvent afficher un bouton de suppression (une croix). Ce bouton permet à l'utilisateur de supprimer le contenu du champ. Ce bouton peut être affiché dans la zone de saisie : 

	- uniquement lorsqu'une saisie est effectuée dans le champ. 

		- uniquement lorsque le champ perd le focus. 
						

		- toujours (que le curseur de saisie soit dans le champ ou non). 




- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Lorsqu'un champ de saisie texte est agrandi sous l'éditeur, il se transforme automatiquement en champ de saisie multiligne.



<a name="NOTE1_2"></a>


### Champ de saisie de type Texte multiligne
<a name="champ_saisie_type_texte_multiligne_ELTPARAGRAPHE000041"></a>Ce type de champ de saisie permet d'afficher ou de saisir plusieurs lignes séparées par un "Retour Chariot".

Remarques : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Si les lignes sont séparées par un "Retour Chariot", il est possible d'afficher un ascenseur vertical ou horizontal.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Il est également possible d'afficher les adresses internet et les chemins de fichiers sous forme de lien.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Lorsqu'un champ de saisie texte est agrandi sous l'éditeur, il se transforme automatiquement en champ de saisie multiligne.

### Champ de saisie de type Texte RTF
<a name="champ_saisie_type_texte_rtf_ELTPARAGRAPHE000062"></a>Ce type de champ de saisie permet de saisir et d'afficher des informations au format RTF. Pour plus de détails, consultez [Manipuler un champ de saisie au format RTF](../WDChamp/1013190.md).

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Pour les applications fonctionnant sous Windows Vista (ou supérieur) ou sous Tablet PC, il est possible d'intégrer l'option "Saisie manuscrite (Tablet PC)". Cette option permet à l'utilisateur de saisir directement le texte dans le champ à l'aide du stylet ou de la souris. Cette fonctionnalité existe par défaut dans les applications fonctionnant sous Mobile.

### Champ de saisie de type Visualisation Markdown
<a name="champ_saisie_type_visualisation_markdown_ELTPARAGRAPHE000104"></a>Ce type de champ de saisie permet d'afficher des informations au format Markdown. Ces informations en affichage peuvent être sélectionnées ou scrollées. 

Rappel : L'option "Support du Markdown" présente dans l'onglet "UI" de la fenêtre de description des champs de saisie s'applique uniquement au libellé, à la bulle et aux jetons associés au champ de saisie. Cette propriété ne s'applique pas au contenu du champ de saisie.

Pour modifier le type du champ de saisie (passer par exemple d'un champ de saisie de type Texte à un champ de saisie de type Visualisation Markdown), utilisez la propriété [VisualisationMarkdown](../Proprietes/1410089587.md).
<a name="NOTE1_5"></a>


### Champ de saisie de type Mot de passe
<a name="champ_saisie_type_mot_passe_ELTPARAGRAPHE000122"></a>Avec ce type de champ de saisie, lors de la saisie d'informations dans le champ, tous les caractères seront remplacés par des étoiles à l'écran. 

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) L'utilisateur a la possibilité de vérifier le mot de passe saisi (FAA). Pour plus de détails, consultez [Voir en clair un mot de passe (FAA)](../WDChamp/9500129.md). 
<a name="NOTE1_6"></a>


### Champ de saisie de type Numérique, Monétaire ou Monétaire + Euro
<a name="champ_saisie_type_numerique_monetaire_monetaire_euro_ELTPARAGRAPHE000138"></a>Ces différents types de champs permettent de manipuler des valeurs numériques : 

- Le champ de saisie Numérique permet d'afficher ou de saisir une valeur numérique (poids, ...). Si le champ n'est pas assez grand pour afficher le chiffre en entier, les caractères "+++" sont affichés.

- Le champ de saisie Monétaire permet d'afficher ou saisir des valeurs monétaires.

- Le champ de saisie Monétaire + Euro permet de gérer des prix par exemple.
	![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Ce type de champ de saisie n'est pas disponible.




![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Les champs de saisie numériques, monétaires et monétaires + Euro proposent d'afficher un bouton **calculatrice** pour permettre à l'utilisateur de réaliser un calcul (option "Afficher le bouton calculatrice"). Il existe également diverses FAA (Fonctionnalités Automatiques de vos Applications) sur ce type de champ (modification de la valeur à l'aide de la roulette de la souris par exemple). Pour plus de détails, consultez [FAA sur les champs numériques](../Editeurs/9500122.md).
<a name="NOTE1_7"></a>


### Champ de saisie de type Date
<a name="champ_saisie_type_date_ELTPARAGRAPHE000166"></a>Ce type de champ de saisie permet d'afficher ou de saisir des dates.

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) L'utilisateur pourra utiliser les touches "+" et "-" du clavier numérique pour modifier la date.
Si le champ n'est pas dans une popup, il est possible d'afficher un bouton calendrier pour permettre à l'utilisateur de saisir la date (option "Afficher un bouton calendrier"). Dans ce cas, il est possible de définir les caractéristiques du calendrier. 

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) ***FAA (Fonctionnalité Automatique de vos Applications)*** : Si le bouton calendrier n'est pas affiché, l'utilisateur pourra tout de même saisir la date dans un calendrier (option "Calendrier" du menu contextuel du champ). Pour supprimer cette FAA, utilisez la fonction [FAADésactive](../WDLang1/1000022018.md) avec la constante *faaCalendrier*. Pour plus de détails, consultez [FAA sur les champs Date](../Editeurs/9500124.md).

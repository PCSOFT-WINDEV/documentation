
## Normalisation W3C
			






	<B><B>Avertissement</B></B>

	A partir de la version <B><B>17</B></B>,  toutes les pages générées par WEBDEV respectent la norme W3C. Cette page d'aide est donc obsolète à partir de cette version : les manipulations présentées dans cette page sont disponibles uniquement pour les versions inférieures à la version 17.  



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000116"></a>
Vous devez respecter la norme W3C (HTML 4.01) dans vos projets WEBDEV ? WEBDEV offre la possibilité de générer un code HTML respectant la norme W3C.

Vous pouvez envoyer les pages générées avec WEBDEV au service de validation de la norme W3C ([http://validator.w3.org](http://validator.w3.org)). Pour cela, il suffit d'enregistrer les pages de votre site avant de les envoyer.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000140"></a>


### Générer des pages HTML respectant la norme W3C
<a name="generer_des_pages_html_respectant_norme_w3c_ELTPARAGRAPHE000028"></a>Pour générer des pages HTML respectant la norme W3C :

1. Affichez la description du projet (option "Projet .. Description du projet").

2. Dans l'onglet "Avancé", cliquez sur le bouton "Avancé..." : la liste des options de génération du code HTML s'affiche.

3. Sélectionnez l'option "Respecter la norme W3C transitional".

4. Regénérez les pages de votre site.




**Remarque** : Pour obtenir un affichage correspondant à la norme W3C par tous les navigateurs, il est nécessaire de fournir le fichier DTD correspondant à la norme W3C (option "Inclure la référence au fichier DTD"). Cependant, cette option peut être décochée si des problèmes d'affichage sont rencontrés.


<a name="NOTE2_2"></a>


### Configurer une page à la norme W3C
<a name="configurer_une_page_norme_w3c_ELTPARAGRAPHE000043"></a>Pour qu'une page soit à la norme W3C :

1. Ouvrez la page sous l'éditeur.

2. Affichez la description de la page.

3. Dans l'onglet "Avancé", sélectionnez l'option "Personnalisée" et cliquez sur le bouton [...].

4. Sélectionnez l'option "Respecter la norme W3C transitional".

5. Validez.




A la prochaine génération de la page, cette option sera prise en compte.


<a name="NOTE2_3"></a>


### Limitations
<a name="limitations_ELTPARAGRAPHE000065"></a>

- Une page avec l'option W3C peut ne pas passer la validation W3C si cette page contient une page interne qui n'a pas l'option W3C.

- Une page avec les options W3C et "incluant la référence au fichier DTD" peut ne pas s'afficher correctement si cette page contient une page interne qui n'a pas les options W3C et "incluant la référence au fichier DTD". Si une page contient des pages internes de composants externes dont on ne sait pas si elles ont été générées avec les options W3C et "incluant la référence au fichier DTD" et que la page ne s'affiche pas correctement, décochez l'option "incluant la référence au fichier DTD" de la page.

## Présentation des pages
<a name="presentation_des_pages_ELTTEXTE000273"></a>
Les pages constituent l'interface principale d'un site WEBDEV. En effet, c'est grâce aux pages que les internautes vont pouvoir consulter et manipuler des informations.

Ces informations peuvent être :

- Dynamiques et provenir par exemple de fichiers de données, requêtes, etc. Ces informations varient en fonction des demandes des internautes.

- Statiques. Ces informations sont fixées et ne changent pas.

## Création d'une page
<a name="creation_une_page_ELTTEXTE000297"></a>


### Comment le faire ? 
<a name="comment_faire_ELTPARAGRAPHE000023"></a>Pour créer une page : 

1. Cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_nouveau.gif) parmi les boutons d'accès rapide. 

	- La fenêtre de création d'un nouvel élément s'affiche : cliquez sur "Page" puis sur "Page".

	- L'assistant de création d'une page se lance.




2. Sélectionnez le type de page à créer :

	- **l'option "Standard"** permet de créer : 

		- une page vierge ne contenant aucun champ. 
						Le mode d'édition de cette page est définit dans l'onglet "Général" de la fenêtre de description de la page : 

			- **Responsive (zoning et agencements)** (par défaut) : Page vierge utilisant le zoning et les agencements.
										Grâce aux agencements définis sur bureau et sur mobile, ce type de page permet de réaliser rapidement un site "Mobile-friendly". 

			- **Tranches (Responsive historique pour compatibilité)** : Page vierge utilisant les tranches du Responsive Web Design. 
										Ce type de page est conservé par compatibilité pour réaliser des sites "Mobile-friendly". Ce type de page 
										Pour plus de détails, consultez [Créer une page Responsive Web Design](../WDChamp/9000168.md). 

			- **Flux HTML** : Page vierge de type Flux HTML. 




- une page basée sur un des modèles du projet, si le projet contient un modèle de pages.  

- une page basée sur un modèle prédéfini (2 ou 3 colonnes, encart à droite ou à gauche, ...). 
						Il est possible de générer le modèle de pages et la page ou uniquement la page seule. 
						La page générée contient des champs pour présenter une mise en page possible. Ces champs peuvent être ensuite modifiés ou supprimés. 

- **l'option "Pages internes"** permet de créer : 

	- une page interne vierge, pouvant être incluse (avec son code) dans une autre page via les champs de type "Page interne". 

	- des pages internes spécifiques à la gestion des boîtes de dialogue. 




Pour afficher les caractéristiques de la page créée, utilisez une des méthodes suivantes : 

- Sélectionnez l'option "Description" dans le menu contextuel de la page.

- Sous le volet "Modification", dans le groupe "Edition rapide", cliquez sur "Description".

### Fichiers correspondant aux pages
<a name="fichiers_correspondant_aux_pages_ELTPARAGRAPHE000105"></a>Les pages correspondent à des fichiers WEBDEV ".WWH". Par défaut, ces fichiers sont créés dans le répertoire principal du projet.

Lors de la génération des pages du site, les fichiers ".WWH" sont automatiquement convertis en fichiers ".HTM". Ces fichiers HTML sont créés dans des répertoires différents selon le type de la page :

- ***Page dynamique*** : sous-répertoires des langues du projet (par exemple, sous-répertoires "FR", "US", ...).

- ***Page statique*** : sous-répertoires des langues présents dans le répertoire "&lt;NomProjet&gt;_WEB" du projet (par exemple "MonSite_Web/FR").

<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=RepPageDynStat.gif)
Pour plus de détails sur les répertoires d'un projet, consultez [Répertoires d'un projet](../Editeurs/2030003.md).

### Description d'une page
<a name="description_une_page_ELTPARAGRAPHE000123"></a>La fenêtre de description d'une page contient plusieurs onglets permettant de définir toutes les caractéristiques de la page :

- Onglet "**Général**" : Il permet de définir : 

	- le nom de la page, 

	- sa description, 

	- son titre, 

	- son type (Session, AWP, statique, PHP, etc.), 

	- son mode d'édition (Responsive, Tanches, etc.).




- Onglet "**UI**" : Il permet de définir : 

	- les caractéristiques de la jauge automatique Ajax,

	- diverses options d'affichage : 

		- ordre de navigation, 

		- gestion des plans, 

		- [gestion du bouton précédent](../WDChamp/1012001.md),




- [l'adaptation pour les mobiles](../Editeurs/2030057.md), 

- la gestion des WebApps iOS, 

- le curseur de survol de la page. 

- la taille de la page. 

- les options d'ancrage de la page. 

- Onglet "**Détail**" :  Il permet de définir :

	- le contexte HFSQL utilisé,

	- [le référencement de la page](../Editeurs/2030037.md),

	- [les options de validité de la page](../WDChamp/1012021.md),

	- [les fichiers JavaScript (.js) à ajouter à la page](../Editeurs/1012027.md). Ces fichiers seront utilisés par la page HTML générée.

	- les fichiers CSS à ajouter à la page. Ces fichiers seront utilisés par la page HTML générée.




- Onglet "**Frame**" : Il permet de définir les caractéristiques des Frames.

- Onglet "**Langue**" : Il permet de définir les langues disponibles dans la page.

- Onglet "**Note**" : Il permet de définir le titre des notes et leur contenu. Ces informations seront imprimées dans la documentation technique.

- Onglet "**Avancé**" : Il permet de :

	- définir les [options avancées de génération des pages](../WDChamp/1012030.md).

	- ajouter du code HTML à l'entête de la page.




- Onglet "**Style**" : Il permet de définir :

	- les caractéristiques de l'ambiance associée à la page. Pour plus de détails, consultez [Gestion des ambiances](../Editeurs/2016020.md). 

	- la feuille de styles associée à la page.

	- les options de styles de la page : 

		- Fond du navigateur. 

		- Cadre et fond de la page. Par exemple, l'image de fond associée à la page. Pour plus de détails, consultez [Page et image de fond](../WDChamp/1012031.md). 

		- Couleurs par défaut : Cette option permet de définir les différentes couleurs utilisées pour le texte de la page, les liens, les liens visités, ... Ces informations modifient directement le code de la balise &lt;BODY&gt; de la page HTML.

### Création d'une page à partir d'une page interne et inversement
<a name="creation_une_page_partir_une_page_interne_inversement_ELTPARAGRAPHE000195"></a>WEBDEV permet de créer une page à partir d'une page interne et inversement.

- Pour créer une page interne à partir de la page en cours, sous le volet "Accueil", dans le groupe "Général", déroulez "Enregistrer" et sélectionnez "Enregistrer la page comme une page interne".

- Pour créer une page à partir de la page interne en cours, sous le volet "Accueil", dans le groupe "Général", déroulez "Enregistrer" et sélectionnez "Enregistrer la page interne comme une page".





- ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=WW_NuageTags.gif) ***Composants (WEBDEV)*** : **WW_NuageTags** <br>Cet exemple montre comment afficher un nuage de tags dans une application WEBDEV dynamique.<br>En effet, sur Internet, une page d'un site répond plus ou moins à un ensemble de mots-clés.<br>L'objectif de cet exemple est donc de présenter comment afficher les mots-clés d'un site dans une zone de la page en indiquant de manière graphique les mots-clés importants.<br>Les mots-clés se retrouvent alors sous forme de liens de différentes tailles : plus le mot-clé est important dans le site, plus sa police est élevée.

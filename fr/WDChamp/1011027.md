


## Impression lors du test d'un état sous l'éditeur
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000147"></a>
Lors du test d'un état (bouton "Go" depuis l'éditeur d'états), plusieurs options d'impression sont proposées : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GO_Etat_choix.gif)


**Remarque** : Par défaut, le mode d'impression de l'état est automatiquement demandé lors du test de chaque état. 

**Pour ne pas afficher cette fenêtre lors du test de l'état**, il suffit de cocher l'option "Ne plus afficher".

Dans ce cas, l'état testé sera directement visualisé dans le visualisateur de rapports.

Remarque : Il est possible de réactiver l'affichage de cette fenêtre. Pour plus de détails, consultez [Ne plus afficher un message](../Editeurs/2020011.md).



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Détail des modes d'impression d'un état proposés lors du test
<a name="detail_des_modes_impression_etat_proposes_lors_test_ELTTEXTE000171"></a>


### Visualisateur de rapports
<a name="visualisateur_rapports_ELTPARAGRAPHE000038"></a>Le visualisateur de rapports permet de visualiser la présentation d'un état sans l'imprimer. La mise en page et le contenu de l'état peuvent ainsi être vérifiés. Il est bien entendu possible de lancer l'impression de l'état.

Pour plus de détails, consultez [Visualisateur de rapports](../WDChamp/1011085.md).
<a name="NOTE2_2"></a>


### Imprimante
<a name="imprimante_ELTPARAGRAPHE000050"></a>L'état est directement imprimé sur l'imprimante par défaut du poste en cours, avec les options par défaut de l'imprimante. 
<a name="NOTE2_3"></a>


### Fichier RTF
<a name="fichier_rtf_ELTPARAGRAPHE000057"></a>L'état est enregistré et visualisé au format RTF (Rich Text Format). Le fichier créé est visualisé directement dans un des éditeurs de texte installé sur le poste en cours (Word, Open Office ou NotePad par exemple). 
Le fichier généré est nommé "&lt;NomEtat&gt;.RTF". Ce fichier est créé dans le sous-répertoire EXE du répertoire du projet.

Remarques :

- Un éditeur de texte doit être installé sur le poste en cours pour visualiser le fichier généré.

- Pour enregistrer un fichier RTF, le fichier "RICHED20.DLL" doit être présent sur le poste en cours. Le fichier "RICHED20.DLL" est généralement présent dans le répertoire système de Windows.

- L'impression des textes orientés (à 90° par exemple) n'est pas supportée.

- Les tableaux sont gérés par le format RTF. La couleur de la police et la couleur de fond des cellules sont conservées.

- La superposition des éléments (par exemple, superposition d'images et de textes) n'est pas gérée. 



<a name="NOTE2_4"></a>


### Fichier HTML
<a name="fichier_html_ELTPARAGRAPHE000074"></a>L'état est enregistré et visualisé au format HTML. Le fichier créé est visualisé directement dans le navigateur par défaut installé sur le poste en cours.
Le fichier généré est nommé "&lt;NomEtat&gt;.HTM". Ce fichier est créé dans le sous-répertoire EXE du répertoire du projet.

Remarques :

- Le fichier HTML créé utilise des feuilles de styles (CSS). 

- Un navigateur Internet doit être installé sur le poste en cours pour visualiser le fichier généré.



<a name="NOTE2_5"></a>


### Fichier PDF ou PDF générique
<a name="fichier_pdf_pdf_generique_ELTPARAGRAPHE000088"></a>L'état est enregistré et visualisé au format PDF (Portable Document Format). Le fichier créé est visualisé directement dans un logiciel de visualisation des fichiers PDF (Adobe Acrobat Reader par exemple) installé sur le poste en cours.

Le fichier généré est nommé "&lt;NomEtat&gt;.PDF". Ce fichier est créé dans le sous-répertoire EXE du répertoire du projet. 

Remarques : 

- Si l'option "Fichier PDF" est sélectionné, le fichier PDF créé dépend de l'imprimante courante. Si l'option "Fichier PDF générique" est sélectionnée, le fichier PDF est indépendant de l'imprimante courante. 

- Un logiciel de visualisation des fichiers PDF doit être installé sur le poste en cours pour visualiser le fichier généré.

- Dans cette version, il est possible de rencontrer des problèmes connus. Pour plus de détails, consultez [Impression au format PDF : Problèmes connus](../WDChamp/1011016.md)

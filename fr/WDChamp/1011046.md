


## Bloc Pied de page (Bas de page)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000249"></a>
Le bloc **Pied de page** est un des blocs disponibles dans un état créé sous l'éditeur d'état. Le bloc **Pied de page** est exécuté à la fin de chaque page.

Le bloc **Pied de page** contient des informations qui doivent figurer en bas de chaque page, par exemple des compteurs et/ou des totaux (générés lors de la création du document ou ajoutés par la suite) ou le numéro de la page en cours par rapport au nombre de pages totales.

Attention : L'auto-extensibilité des champs n'est pas gérée dans le bloc "Pied de page". 



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Création du bloc Pied de page
<a name="creation_bloc_pied_page_ELTTEXTE000273"></a>
Pour créer un bloc Pied de page :

1. Effectuez une des manipulations suivantes sous l'éditeur d'états :

	- sous le volet "Création", dans le groupe "Structure", déroulez "Blocs". 

	- sélectionnez l'option "Ajouter un bloc" du menu contextuel de l'état (clic droit sur l'état en cours de création).

	- cliquez sur le bouton "Ajouter un bloc" dans l'onglet "Blocs" de la description de l'état.




2. Sélectionnez le type "Pied de page".




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Impression du bloc Pied de page
<a name="impression_bloc_pied_page_ELTTEXTE000297"></a>


### Imprimer le bloc Pied de page avant ou après le bloc Fin de document
<a name="imprimer_bloc_pied_page_avant_apres_bloc_fin_document_ELTPARAGRAPHE000056"></a>Par défaut, le bloc *Pied de page* est imprimé après le bloc *Fin de document* sur la dernière page de l'état. L'éditeur d'états permet de modifier cet ordre d'impression par défaut.

Pour imprimer le bloc Pied de page avant le bloc Fin de document sur la dernière page :

- Méthode 1 : 

	1. Affichez l'onglet "Général" de la fenêtre de description du bloc "Pied de page" (option "Description du bloc" du menu contextuel).

	2. Cochez l'option "Imprimer le bloc "Fin de document" APRES le bloc "Pied de page" sur la dernière page".




- Méthode 2 : 

	1. Affichez l'onglet "Blocs" de la fenêtre de description de l'état (option "Description de l'état" du menu contextuel).

	2. Dans la section "Fin de document", cochez l'option "APRES le bloc "Pied de page" ".






<a name="NOTE3_2"></a>


### Bloc Pied de page collé aux blocs précédents
<a name="bloc_pied_page_colle_aux_blocs_precedents_ELTPARAGRAPHE000082"></a>Par défaut, le bloc *Pied de page* est imprimé au bas de chaque page. L'éditeur d'états permet de modifier ce fonctionnement par défaut.

Pour ne pas imprimer obligatoirement le bloc Pied de page au bas de chaque page :

1. Affichez l'onglet "Général" de la fenêtre de description du bloc "Pied de page" (option "Description du bloc" du menu contextuel).

2. Cochez l'option "Le bloc "Pied de page" ne s'imprime pas en bas de la page, il se colle aux blocs précédents". 
	Le bloc *Pied de page* ne sera pas imprimé obligatoirement au bas de chaque page. Il sera possible que le bloc *Pied de page* soit suivi d'un espace vide.



<a name="NOTE3_3"></a>




### Bloc Fin de document collé au bloc Pied de page
<a name="bloc_fin_document_colle_bloc_pied_page_ELTPARAGRAPHE000101"></a>Par défaut, le bloc *Fin de document* est collé au bloc *Corps* ou au bloc *Complément de corps* s'il existe. Il est possible que le bloc *Fin de document* soit obligatoirement collé au bloc *Pied de page*.

Pour coller le bloc Fin de document au bloc Pied de page :

- Méthode 1 : 

	1. Affichez l'onglet "Général" de la fenêtre de description du bloc "Pied de page" (option "Description du bloc" du menu contextuel).

	2. Cochez l'option "Le bloc "Fin de document" est collé au bloc "Pied de page"". 
			Le bloc *Fin de document* sera imprimé juste avant ou après le bloc *Pied de page*.







- Méthode 2 : 

	1. Affichez l'onglet "Blocs" de la fenêtre de description de l'état (option "Description de l'état" du menu contextuel).

	2. Dans la section "Fin de document", cochez l'option "Collé au bloc "Pied de page"". 
			Le bloc *Fin de document* sera imprimé juste avant ou après le bloc *Pied de page*.






<a name="NOTE3_4"></a>


### Bloc Fin du document à la place du bloc Pied de page sur la dernière page de l'état
<a name="bloc_fin_document_place_bloc_pied_page_sur_derniere_page_etat_ELTPARAGRAPHE000132"></a>Par défaut, le bloc *Pied de page* est imprimé sur chaque page (y compris la dernière page de l'état). L'éditeur d'états permet de remplacer le bloc *Pied de page* par le bloc *Fin de document* sur la dernière page de l'état.

Pour remplacer le bloc Pied de page par le bloc Fin de document sur la dernière page de l'état :

- Méthode 1 : 

	1. Affichez l'onglet "Général" de la fenêtre de description du bloc "Pied de page" (option "Description du bloc" du menu contextuel).

	2. Cochez l'option "Imprimer le bloc "Fin de document" A LA PLACE du bloc "Pied de page" sur la dernière page". 




- Méthode 2 : 

	1. Affichez l'onglet "Blocs" de la fenêtre de description de l'état (option "Description de l'état" du menu contextuel).

	2. Dans la section "Fin de document", cochez l'option "A LA PLACE du bloc "Pied de page"". 







Dans ce cas :

- le bloc *Fin de document* remplacera le bloc *Pied de page* sur la dernière page de l'état.

- l'option "Le bloc "Fin de document" est collé au bloc "Pied de page"" sera ignorée sur la dernière page de l'état.

- l'option "Le bloc "Pied de page" ne s'imprime pas en bas de la page, il se colle aux blocs précédents" sera prise en compte pour le bloc *Fin de document* sur la dernière page de l'état.




**Remarque** : Pour éviter des effets de superposition, le bloc de *Fin de document* doit avoir la même hauteur que le bloc *Pied de page*.

<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Visibilité du bloc Pied de page
<a name="visibilite_bloc_pied_page_ELTTEXTE000339"></a>
Par défaut, le bloc *Pied de page* est visible sur toutes les pages de l'état. L'éditeur d'états permet de spécifier les pages de l'état sur lesquelles le bloc *Pied de page* doit être affiché.

Pour spécifier les pages sur lesquelles le bloc Pied de page doit être affiché :

1. Affichez l'onglet "UI" de la fenêtre description du bloc (option "Description du bloc" du menu contextuel).

2. Spécifiez la visibilité du bloc :

	- **Toujours visible** : Le bloc et ses champs seront affichés sur toutes les pages de l'état.

	- **Toujours invisible** : Le bloc et ses champs ne seront pas affichés dans l'état.

	- **Visible uniquement sur la première page** : Le bloc et ses champs seront affichés uniquement sur la première page de l'état.

	- **Invisible uniquement sur la première page** : Le bloc et ses champs seront affichés sur toutes les pages de l'état sauf sur la première page.






<a name="NOTE4_2"></a>


### Conserver la place du bloc Pied de page lorsqu'il est invisible
<a name="conserver_place_bloc_pied_page_lorsquil_est_invisible_ELTPARAGRAPHE000191"></a>Par défaut, la place occupée par le bloc Pied de page est toujours conservée dans les pages de l'état (que le bloc soit visible ou non). L'éditeur d'états permet de modifier ce fonctionnement par défaut.

Pour ne pas conserver la place du bloc Pied de page lorsqu'il est invisible :

1. Affichez l'onglet "UI" de la fenêtre description du bloc (option "Description du bloc" du menu contextuel).

2. Décochez l'option "Conserver la place du pied de page lorsqu'il est invisible.". La place occupée par le bloc Pied de page sera conservée uniquement dans les pages où le bloc est visible. Cette option permet d'optimiser le remplissage des pages de l'état.


## Associer une image aux éléments d'un champ Arbre
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Image associée à un élément
<a name="image_associee_element_ELTTEXTE000132"></a>
Tout élément d'un champ Arbre est précédé d'une image. Cette image peut être différente en fonction de l'état de l'élément (ou du niveau) en cours. Il est ainsi possible d'afficher une image :

- soit lorsque le niveau est déroulé,

- soit lorsque le niveau est enroulé (ou lorsque le niveau correspond à une feuille).




Exemple d'un champ Arbre dans une application WINDEV : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Champ_Arbre_1.gif)


Image associée au niveau enroulé : par exemple : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ImageEnroule.gif)


Image associée au niveau déroulé : par exemple : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ImageDeroulee.gif)


**Remarque** : Il est également possible d'associer un identifiant à un élément. Pour plus de détails, consultez [Associer un identifiant à un élément](../WDChamp/1013033.md).

## Associer une ou plusieurs images à un élément
<a name="associer_une_plusieurs_images_element_ELTTEXTE000156"></a>
Pour associer une ou plusieurs images à un élément d'un champ Arbre, il suffit de préciser le nom (et si nécessaire le chemin) de l'image dans les fonctions de programmation suivantes :



|   |   |
| --- | --- |
| [ArbreAjoute](../WDLang1/3018012.md) | Ajoute un élément "feuille" (et tous les noeuds nécessaires) dans un champ Arbre. |
| [ArbreInsère](../WDLang1/3018023.md) | Insère une feuille à une position spécifique dans un champ Arbre. |
| [ArbreModifie](../WDLang1/3018013.md) | Modifie un élément (noeud ou feuille) dans un champ Arbre. |





Ces fonctions sont également disponibles en syntaxe préfixée : 



|   |   |
| --- | --- |
| [&lt;Arbre&gt;.Ajoute](../WDLang1/1000023422.md) | Ajoute un élément "feuille" (et tous les noeuds nécessaires) dans un champ Arbre. |
| [&lt;Arbre&gt;.Insère](../WDLang1/1000023761.md) | Insère une feuille à une position spécifique dans un champ Arbre. |
| [&lt;Arbre&gt;.Modifie](../WDLang1/1000023763.md) | Modifie un élément (noeud ou feuille) dans un champ Arbre. |





Remarque : Si aucune image spécifique n'est associée aux éléments d'un champ Arbre, les icônes "dossier fermé" (![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ImageEnroule.GIF)
) et "dossier ouvert" (![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ImageDeroulee.GIF)
) sont utilisées.

## Format des images
<a name="format_des_images_ELTTEXTE000220"></a>
Les formats d'images supportés sous Windows sont :

- Bitmap (\*.BMP)

- Graphics Interchange Format (\*.gif)

- Joint Picture Experts Group (\*.jpg ; \*.jpeg)

- Kodak Photo CD (\*.pcd)

- PaintBrush (\*.pcx)

- Adobe Photoshop Format (\*.psd)

- TrueVision TARGA (\*.tga)

- Tagged Image File Format (\*.tif ; \*.tiff)
	Remarque : l'option "tiff mosaïque" de la norme tiff 6.0 n'est pas supportée. Dans ce cas, il est conseillé d'enregistrer l'image à la norme tiff 5.0. 

- Portable Network Graphics (\*.png)

- Windows MetaFiles (\*.emf ; \*.wmf)

- Icônes (\*.ico ; \*.icw)
	Remarques : 

	- Affichage HQ (Haute Qualité) disponible.

	- Gestion des planches au format BMP transparent (32 bits).




- Curseurs (\*.cur)

- Portable Document Format (\*.pdf)

- Scalable Vector Graphics (\*.svg)

- Images WINDEV (générées par l'éditeur d'images, \*.WDPIC).








Les images employées peuvent être :

- des fichiers image.

- des rubriques d'un fichier HFSQL de type "Memo Binaire" contenant une image.

- des dessins réalisés à l'aide des [fonctions de dessin dans les listes et dans les combos](../WDLang1/3029028.md).






Remarques :

- L'image spécifiée est réduite proportionnellement : la hauteur de l'image correspond à la hauteur de la ligne du champ Arbre. Il est conseillé d'utiliser des images 16 x 16 pixels (taille des petites icônes de WINDEV).

- La couleur Magenta présente dans les images est considérée comme transparente en exécution.

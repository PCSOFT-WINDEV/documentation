


## Page spécifique pour afficher les erreurs du site
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000147"></a>
En cas d'erreur dans le site dynamique WEBDEV, par défaut, une page d'erreur standard est affichée. Cette page informe l'internaute de l'erreur rencontrée dans le site. 

Pour personnaliser ce message d'erreur, il suffit d'inclure directement dans votre site la page à utiliser pour afficher les erreurs. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000171"></a>


### Créer une page de gestion des erreurs
<a name="creer_une_page_gestion_des_erreurs_ELTPARAGRAPHE000020"></a>La création d'une page de gestion des erreurs se fait grâce à l'assistant de création de pages. 

Pour créer une page de gestion des erreurs : 

1. Créez une nouvelle page :

	- Cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_nouveau.gif)
 parmi les boutons d'accès rapide. 

	- La fenêtre de création d'un nouvel élément s'affiche : cliquez sur "Page" puis sur "Page". 

	- L'assistant de création d'une page se lance.




2. Cliquez dans l'onglet "Pages d'erreur" de l'assistant : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Erreur_SiteWB.gif&type=thumb)


3. Sélectionnez une des pages d'erreur proposées : 

	- Erreur générique : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ErreurWB_Generique.gif)
Cette page d'erreur générique contient uniquement un libellé et une image et peut être personnalisée par exemple en utilisant le gabarit de votre site, en modifiant le message, ...

	- Erreur simple : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ErreurWB_Simple.gif)
Cette page d'erreur simple contient un libellé, une image et le titre de l'erreur WEBDEV. La balise [%TITLE%] est remplacée en exécution par le titre de l'erreur. Cette page peut être personnalisée par exemple en utilisant le gabarit de votre site, en modifiant le message, ... 

	- Erreur détaillée : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=erreurWB_detail.gif)
Cette page d'erreur détaillée contient : 

		- une image, 

		- le titre de l'erreur WEBDEV. La balise [%TITLE%] est remplacée en exécution par le titre de l'erreur. 

		- le message d'erreur complet. La balise [%MSG%] est remplacée en exécution par le message de l'erreur. 

		- le message système. La balise [%SYS%] est remplacée en exécution par le message système de l'erreur. 
						Cette page peut être personnalisée par exemple en utilisant le gabarit de votre site, ... 




4. Validez l'assistant. 

5. Indiquez le nom de la page à créer et validez. La page statique correspondant au type de page sélectionné est automatiquement créé. 



<a name="NOTE2_2"></a>


### Mettre en place la gestion personnalisée des erreurs
<a name="mettre_place_gestion_personnalisee_des_erreurs_ELTPARAGRAPHE000071"></a>Pour indiquer la page du projet à utiliser pour la gestion des erreurs : 

1. Affichez la fenêtre de description de la configuration en cours : sous le volet "Projet", dans le groupe "Configuration de projet", cliquez sur "Configuration courante". 

2. Dans l'onglet "Général", sélectionnez la page d'erreur voulue : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Erreur_SiteWB_Def.gif&type=thumb)
 Remarque : seules les pages statiques du projet sont proposées. 

3. Validez.



<a name="NOTE2_3"></a>


### Mode de fonctionnement
<a name="mode_fonctionnement_ELTPARAGRAPHE000096"></a>Si une page de gestion des erreurs personnalisée a été définie dans le projet, lors du déploiement du site, cette page est automatiquement affichée en cas d'erreur. 

**Cas particulier** : Si une personnalisation des erreurs avait été précédemment mise en place via l'administrateur WEBDEV pour ce site (onglet "Administration" de l'administrateur WEBDEV), cette personnalisation n'est pas conservée. Cependant, si un libellé personnalisé a été spécifié dans l'administrateur WEBDEV (pour une erreur spécifique par exemple), ce libellé sera affiché dans la page d'erreur.

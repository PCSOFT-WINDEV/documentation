


## Le champ TreeMap
			





## Présentation
<a name="presentation_ELTTEXTE000137"></a>


### Champ d'une fenêtre
<a name="champ_une_fenetre_ELTPARAGRAPHE000008"></a>Le champ TreeMap permet de représenter dynamiquement des données, éventuellement arborescentes, de manière pondérée. 

Pour remplir ce type de champ, il suffit d'indiquer : 

- le nom de l'élément. 

- éventuellement, le chemin arborescent de l'élément. 

- le poids de l'élément. 




Ce type de champ est très utile pour permettre par exemple une visualisation intuitive de volumes de données. 



## Créer un champ TreeMap
<a name="creer_champ_treemap_ELTTEXTE000161"></a>
Pour créer un champ TreeMap : 

1. Sous le volet "Création", dans le groupe "Champs graphiques", cliquez sur "TreeMap".

2. Cliquez dans la fenêtre à la position où le champ doit être créé. Le champ est automatiquement créé. 




Remarque : Les dimensions du champ créé sont optimisées pour occuper l'espace disponible à la position indiquée. Si la taille du champ ne vous convient pas, utilisez la combinaison de touches Ctrl + Z : le champ retrouvera sa taille par défaut.

Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ. Le champ TreeMap peut être manipulé par programmation grâce aux [fonctions TreeMap](../WDLang1/1000018995.md) (ou aux [fonctions TreeMap (syntaxe préfixée)](../WDLang1/1000019801.md)).



## Caractéristiques du champ TreeMap
<a name="caracteristiques_champ_treemap_ELTTEXTE000189"></a>


### Fenêtre de description du champ
<a name="fenetre_description_champ_ELTPARAGRAPHE000051"></a>La fenêtre de description du champ TreeMap permet de saisir les caractéristiques d'affichage du champ. 

L'onglet "Général" permet de sélectionner : 

- le mode d'affichage du libellé des éléments du champ : libellé sur les éléments ou sur les groupes d'éléments. 

- le contenu du libellé d'information sur la pondération. 




L'onglet "UI" permet d'autoriser la sélection dans le champ (cette option est proposée par défaut). 

L'onglet "Style" permet de définir les options de polices, couleurs et autres concernant les différents éléments du TreeMap. 


### Vocabulaire associé au champ TreeMap
<a name="vocabulaire_associe_champ_treemap_ELTPARAGRAPHE000065"></a>Pour manipuler un champ TreeMap, il est nécessaire de connaître le vocabulaire suivant : 

| Nom | Définition |
| --- | --- |
| Elément | Elément affiché dans le champ TreeMap. |
| Chemin de l'élément | Le chemin d'un élément correspond au chemin et au libellé de l'élément affiché dans le champ TreeMap. L'utilisation d'un chemin lors de l'ajout de l'élément dans le champ TreeMap permet d'organiser les éléments et de les regrouper. |
| Groupe d'éléments | Ensemble d'éléments utilisant un chemin commun. |
| Pondération | Poids de l'élément affiché dans le champ TreeMap. Cette pondération peut être affichée ou non dans le champ TreeMap. Cette pondération permet de visualiser l'importance de l'élément par rapport aux autres éléments présents dans le champ TreeMap. |

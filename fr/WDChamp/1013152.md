
## Champ Interrupteur lié à une rubrique
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Caractéristiques de la rubrique
<a name="caracteristiques_rubrique_ELTTEXTE000099"></a>
Un champ Interrupteur peut correspondre à une rubrique d'un fichier de données, d'une requête, ...

Les types de rubriques pouvant être associées à un champ Interrupteur sont :

- Numérique,

- Booléen,

- Interrupteur.




**Si l'interrupteur ne contient qu'une seule option**, la rubrique prendra pour valeur :

- 1 si la case est cochée

- 0 si la case n'est pas cochée.




**Si l'interrupteur contient plusieurs options**, une rubrique tableau est nécessaire. Chaque élément de la rubrique tableau correspondra à une option de l'interrupteur, et prendra pour valeur :

- 1 si la case est cochée

- 0 si la case n'est pas cochée.

## Récupérer le contenu du champ Interrupteur dans une rubrique
<a name="recuperer_contenu_champ_interrupteur_dans_une_rubrique_ELTTEXTE000123"></a>
**1er cas : les champs de la fenêtre, la page ou l'état sont associés à des rubriques d'un fichier de données** (Binding)
Toutes les rubriques du fichier de données (ou d'un fichier relié) associées à un champ de la fenêtre, de l'état ou de la page seront mises à jour par la fonction [EcranVersFichier](../WDLang4/3044146.md).

**2ème cas : les champs de la fenêtre, l'état ou la page ne sont pas associés à des rubriques d'un fichier de données**
Il est nécessaire pour chaque rubrique de récupérer la valeur de chaque champ.

1. Le champ Interrupteur possède une seule option : utilisez la syntaxe suivante pour mettre à jour la rubrique : 
	
	```txt
	<Fichier de données>.<Rubrique> = <Champ Interrupteur>
	```


2. Le champ Interrupteur possède plusieurs options et est relié à une rubrique tableau : utilisez la syntaxe suivante pour mettre à jour la rubrique :
	
	```txt
	<Fichier de données>.<Rubrique>[Indice de l'option] = <Champ Interrupteur>[Indice de l'option]
	```

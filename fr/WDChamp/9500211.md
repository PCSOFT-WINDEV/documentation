## Présentation
<a name="presentation_ELTTEXTE000268"></a>


### Champ d'une page
<a name="champ_une_page_ELTPARAGRAPHE000011"></a>Le champ Multimédia permet d'afficher de façon simple une vidéo dans le navigateur.
 Remarque : Dans les versions précédentes, ce champ s'appelait le champ Vidéo.

Le champ prend en charge la recherche du meilleur lecteur vidéo présent sur le poste client en fonction du format du fichier vidéo à afficher. 

Si le navigateur utilisé supporte la norme HTML 5, la vidéo sera automatiquement jouée sans lecteur vidéo spécifique. 

La vidéo peut être pilotée en code navigateur grâce aux [fonctions Multimédia](../WDLang1/1000015005.md).


### Créer un champ Multimédia
<a name="creer_champ_multimedia_ELTPARAGRAPHE000029"></a>Pour créer un champ Multimédia : 

1. Sous le volet "Création", dans le groupe "Champs usuels", déroulez "Image" et sélectionnez "Champ multimédia".

2. Cliquez dans la page à la position où le champ doit être créé. Le champ apparaît en création.




Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ.

Dans les caractéristiques du champ, il est possible de choisir directement le fichier vidéo à afficher. Le fichier peut également être affecté en programmation (en code serveur).

## Caractéristiques d'un champ Multimédia
<a name="caracteristiques_champ_multimedia_ELTTEXTE000298"></a>


### Source de la vidéo (Fichier vidéo)
<a name="source_video_fichier_video_ELTPARAGRAPHE000057"></a>Le fichier vidéo peut être affecté au champ en code serveur uniquement.

Le fichier vidéo peut être :

- soit un chemin et un nom de fichier : dans ce cas, le fichier sera recherché dans le répertoire _WEB (ou un de ses sous-répertoires), le chemin fourni au champ doit être relatif à ce répertoire.

- soit une URL (commençant par http:// ou https://) finissant par le nom d'un fichier vidéo (par exemple : http://serveurvideo.masociete.com/films/presentation.wmv)

- soit une URL (commençant par http:// ou https://) ne finissant pas par le nom d'un fichier (par exemple : http://serveurvideo.masociete.com/film.awp?id=51452). Dans ce cas, il est nécessaire de renseigner la propriété [LecteurMultimédia](../Proprietes/1000019055.md).

- soit une URL complète d'une vidéo hébergée hors du site (vidéo YouTube par exemple). 
	Remarque : Dans le cas d'une vidéo YouTube, la personnalisation des sous-titres n'est pas disponible.




Le lecteur multimédia utilisé par le champ dépend du format de fichier et du système d'exploitation.
Si le navigateur utilisé supporte la norme HTML 5, la vidéo sera automatiquement jouée sans lecteur vidéo spécifique. 
Le tableau suivant récapitule les principaux cas :

| Extension du fichier | Lecteur multimédia utilisé par le champ vidéo |
| --- | --- |
| *Format Windows Média* :<br><br>- .wmv<br><br>- .wvx<br><br>- .wma<br><br>- .wax<br><br>- .asf<br><br>- .asx<br><br>- .avi<br><br>- .wav<br><br><br> | Sous Windows, le lecteur Windows Média est utilisé.<br><br>Sur les autres systèmes d'exploitation, le choix du lecteur à employer est laissé au navigateur. |
| *Vidéo Flash* :<br><br>- .flv<br><br><br> | Si le logiciel Flash est présent, il est utilisé. Sinon, le choix du lecteur vidéo à employer est laissé au navigateur. |
| *MPEG-4 Part 14 (ISO/IEC 14496-14:2003)* :<br><br>- .mp4<br><br><br> | Si le logiciel Quicktime est présent, il est utilisé. Sinon, le choix du lecteur vidéo à employer est laissé au navigateur. |
| *MPEG* :<br><br>- .mpg<br><br>- .mpeg<br><br>- .mp3<br><br><br> | Sous Windows avec Internet Explorer, le lecteur Windows Média est utilisé.<br><br>Sur les autres systèmes d'exploitation ou avec d'autres navigateurs, le choix du lecteur à employer est laissé au navigateur. |
| *Apple QuickTime* :<br><br>- .mov<br><br>- .aiff<br><br><br> | Si le logiciel Quicktime est présent, il est utilisé. Sinon, le choix du lecteur vidéo à employer est laissé au navigateur. |
| *Ogg Vobis* :<br><br>- .ogg<br><br>- .ogm<br><br>- .oga<br><br>- .ogv<br><br><br> | Sous Firefox (version 3.5 et supérieures), une balise "&lt;VIDEO&gt;" à la norme HTML 5 est employé.<br><br>Avec les autres navigateurs, le choix du lecteur vidéo à employer est laissé au navigateur. |
| Autres cas | Le choix du lecteur vidéo à employer est laissé au navigateur. |


Il est possible de savoir quel lecteur vidéo a été utilisé grâce à la propriété [LecteurMultimédia](../Proprietes/1000019055.md).

**Remarques** : 

- Le champ Multimédia ne contient pas de codecs de lecture. Les codecs appropriés doivent être installés sur le poste de l'internaute.

- Si la détermination du lecteur multimédia employé a été laissée à la charge du navigateur, [les fonctions Multimédia](../WDLang1/1000015005.md) ne peuvent pas être utilisées.

- Selon le lecteur multimédia effectivement utilisé, certaines fonctionnalités peuvent ne pas être disponibles (par exemple, le lecteur Flash de Adobe ne supporte pas le paramètre "Autorise le mode plein écran").

- Si le serveur Web utilisé est IIS, il est nécessaire d'ajouter pour chaque type de vidéo, un type MIME sur le serveur Web. 

- La navigateur utilisé peut ne pas gérer tous les formats. Pour vérifier si le navigateur gère le format voulu (par exemple AVI), il suffit de dropper le fichier voulu dans le navigateur. 





### Sous-titrage
<a name="soustitrage_ELTPARAGRAPHE000167"></a>Pour ajouter des sous-titres à un fichier multimédia, il suffit de : 

1. Cliquer sur le bouton "+". 

2. Dans le tableau, sélectionnez : 

	- le type des sous-titres intégrés (dialogues, commentaires, ...). 

	- la langue des sous-titres. 

	- le libellé des sous-titres. Ce libellé permettra à l'utilisateur de sélectionner le type de sous-titre en exécution. 

	- le fichier contenant les sous-titres. Ce fichier peut être un fichier d'extension "vtt", "ttml" ou "srt". 







Attention : 

- Cette fonctionnalité est une fonctionnalité HTML 5. La page doit être au format HTML 5. 

- Cette fonctionnalité est directement implémentée par le navigateur. Selon le navigateur utilisé, certaines fonctionnalités peuvent être prises en charge ou non. 





### Autres options
<a name="autres_options_ELTPARAGRAPHE000185"></a>

- **Démarrage automatique** : Cette option permet de démarrer automatiquement la vidéo à l'affichage de la page.

- **Répétition automatique** : Cette option permet de relancer en boucle la vidéo.  

- **Autoriser le mode 'plein écran'** : Cette option permet à l'utilisateur de passer en mode plein écran. 

- **Barre MultiMédia** : Cette option permet d'afficher la barre de gestion de la vidéo : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Barre_multimedia.gif)





**Attention** : Les options de ce champ peuvent varier en fonction du type de vidéo jouée.

## Manipuler un champ Multimédia par programmation
<a name="manipuler_champ_multimedia_par_programmation_ELTTEXTE000334"></a>


### Modification de la vidéo dynamiquement par programmation 
<a name="modification_video_dynamiquement_par_programmation_ELTPARAGRAPHE000216"></a>Pour changer dynamiquement le fichier vidéo associé au champ, il suffit d'affecter le champ avec le nouveau fichier. Exemple : 

```wl
// Modification de la vidéo affichée
// La nouvelle vidéo se trouve dans un sous-répertoire "Video" du répertoire _WEB du projet
VIDEO_Video1 = "/" + RépertoireWeb() + "/Video/Wildlife.wmv"
```

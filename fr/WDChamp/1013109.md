## Présentation du champ Code-barres
<a name="presentation_champ_codebarres_ELTTEXTE000227"></a>


### Champ d'une fenêtre, d'une page ou d'un état
<a name="champ_une_fenetre_une_page_etat_ELTPARAGRAPHE000011"></a>Le champ Code-barres permet d'afficher très simplement des codes-barres dans vos fenêtres, vos pages ou vos états. 

La valeur d'un champ Code-barres correspond à une chaîne alphanumérique.

**Attention** : Pour lire directement sur l'écran la valeur d'un code-barres à l'aide d'un appareil spécialisé, il est nécessaire d'utiliser du matériel adapté.







<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Création d'un champ Code-barres
<a name="creation_champ_codebarres_ELTTEXTE000251"></a>
Pour créer un champ Code-barres : 

1. Sous le volet "Création" : 

	- sous l'éditeur de fenêtres ou de pages : dans le groupe "Champs graphiques", cliquez sur "Code-barres".

	- sous l'éditeur d'états : dans le groupe "Données", cliquez sur "Code-barres".




2. Cliquez dans la fenêtre, la page ou l'état à la position où le champ doit être créé. Le champ apparaît en création.




Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ. Vous pouvez alors choisir le type de code-barres (onglet "Général"). Ce type ne peut pas être modifié par programmation.

**Astuce** : Il est également possible de créer directement un champ Code-barres "pré-formaté" en déroulant l'option "Code-barres". La liste des différents types de codes-barres disponibles avec leur aperçu au gabarit du projet apparaît. Il suffit de cliquer sur le champ voulu pour le créer dans l'éditeur. 


### Remarque : taille d'un QR Code
<a name="remarque_taille_code_ELTPARAGRAPHE000095"></a>Un code-barres de type QR Code est une matrice de points dont la taille évolue en fonction des données contenues et du taux de correction. 

La norme de gestion des QR codes oblige les points à être tous de la même taille. 

Afin d'être toujours lisible, la taille du QR Code n'est donc pas systématiquement proportionnelle à celle du champ. D'un QR Code à l'autre, une marge à l'intérieur du champ peut apparaître, afin que le code soit conforme à la norme et toujours lisible par les lecteurs.

## Caractéristiques du Champ Code-barres
<a name="caracteristiques_champ_codebarres_ELTTEXTE000281"></a>


### Caractéristiques du champ Code-barres
<a name="caracteristiques_champ_codebarres_ELTPARAGRAPHE000108"></a>


|   |   |
| --- | --- |
| Le champ Code-barres est constitué :<br><br>- du libellé du champ Code-barres.<br><br>- de l'image du code-barres.<br><br>- de la valeur du code-barres (qui correspond à une chaîne alphanumérique).<br><br><br> | <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=codeBarre_Detail.gif)<br> |

**L'éditeur de fenêtres ou l'éditeur de pages** permet de sélectionner :

- le type de code-barres à afficher (avec si nécessaire ses spécificités).

	- Code-barres avec vérification : Le dernier caractère du code-barres correspond à un caractère de vérification.

	- Code-barres étendu : La valeur du code-barres peut contenir les 128 caractères du code ASCII.

	- Taille proportionnelle : La hauteur du code-barres est proportionnelle à sa largeur (pour les types UPC et EAN).

	- Valeur du code-barres visible : Affiche ou non la valeur du code-barres sous son dessin.
			Pour connaître les différents types de codes-barres gérés par WINDEV, WINDEV Mobile ou WEBDEV, consultez [Les différents types de codes-barres](../WDLang5/3046064.md).




- l'orientation et la position du code-barres (onglet "Détail").

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) le cadre du code-barres (onglet "Style" de l'éditeur de fenêtres).




**L'éditeur d'états** permet d'indiquer :

- la valeur initiale du code-barres. Par exemple, il est possible de saisir l'adresse Internet liée à un code-barres de type QR Code. 

- le type du code-barres à afficher (avec si nécessaire ses spécificités).

	- Avec vérification : Le dernier caractère du code-barres correspond à un caractère de vérification.

	- Code étendu : La valeur du code-barres peut contenir les 128 caractères du code ASCII.

	- Valeur du code-barres visible : Affiche ou non la valeur du code-barres sous son dessin.

	- Taille proportionnelle : La hauteur du code-barres est proportionnelle à sa largeur (pour les types UPC et EAN).

	- Ignorer les caractères de formatage : Ignore les parenthèses d'un code 128 ou EAN-128.

	- La valeur contient des caractères de contrôle : Code-barres contenant des caractères de contrôle (valable uniquement pour les codes-barres de type CODE 128, DATAMATRIX, EAN 128 et PDF 417). Par défaut, les caractères de contrôle sont encadrés par les caractères "&lt;" et "&gt;". Pour ne pas interpréter ces caractères comme caractères de contrôle, il suffit de les doubler.

	- Jeux de caractères : Code-barres utilisant un ou plusieurs jeux de caractères utilisés pour la compression du code-barres. Ce jeu de caractères doit être défini dans la valeur du code-barres.

	- Caractères de vérification : Code-barres contenant des caractères de vérification (valable uniquement pour les codes-barres de type CODE 128, DATAMATRIX, EAN 128 et PDF 417).
			Pour connaître les différents types de codes-barres gérés par WINDEV ou WEBDEV, consultez [Les différents types de codes-barres](../WDLang5/3046064.md).




- l'orientation et la position du code-barres.

- le cadre du code-barres (onglet "Style").

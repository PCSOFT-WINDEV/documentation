
## Champ Planning lié à un fichier de données ou une variable
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000143"></a>
Il est possible de créer des champs Planning directement liés aux données. Vous pouvez créer : 

- un **champ Planning Fichier** : le champ Planning sera lié à un fichier de données ou à une requête. Dans ce cas, le fichier de données contient les caractéristiques des rendez-vous. 

- un **champ Planning sur variable** : le champ Planning sera lié à une variable (le plus souvent une instance de classe). Dans ce cas, la classe liste les rendez-vous avec leurs caractéristiques. 




La programmation des champs Planning est grandement simplifiée. 





## Comment le faire ?
<a name="comment_faire_ELTTEXTE000167"></a>


### Créer un champ Planning fichier
<a name="creer_champ_planning_fichier_ELTPARAGRAPHE000020"></a>Pour créer un champ Planning fichier, il est possible : 

- soit d'utiliser l'assistant de création du champ Planning. Il suffit de sélectionner l'option "Afficher les données d'un fichier ou d'une requête existante". Répondez ensuite aux questions de l'assistant et validez. 

- soit de modifier la description du champ Planning. 




Pour modifier les caractéristiques d'un champ Planning existant :

1. Affichez la fenêtre de description du champ (option "Description" du menu contextuel). 

2. Dans l'onglet "Contenu", sélectionnez l'option "Fichier" et sélectionnez la source de données utilisée. Cette source de données peut être : 

	- un fichier de données défini dans l'analyse,

	- une requête du projet,

	- une requête intégrée à la fenêtre ou la page. 




3. Associez les différentes rubriques de la source de données aux caractéristiques du rendez-vous. Les éléments précédés d'une étoile sont obligatoires. 
	**Remarque** : Si le fichier de données associé ne contient pas directement la rubrique correspondante, il est possible de réaliser une liaison multi-fichiers. 

4. Validez la fenêtre de description du champ Planning. Le champ Planning est un planning fichier. 







### Créer un champ Planning sur variable
<a name="creer_champ_planning_sur_variable_ELTPARAGRAPHE000042"></a>Pour créer un champ Planning sur variable, il est possible : 

- soit d'utiliser l'assistant de création du champ Planning. Il suffit de sélectionner l'option "Afficher les données d'une variable existante". Répondez ensuite aux questions de l'assistant et validez. 

- soit de modifier la description du champ Planning. 




Pour modifier les caractéristiques d'un champ Planning existant :

1. Affichez la fenêtre de description du champ (option "Description" du menu contextuel). 

2. Dans l'onglet "Contenu", sélectionnez l'option "Variable" et sélectionnez la source de données utilisée (classe, ...). 

3. Associez les différents éléments (membres de la classe par exemple) de la source de données aux caractéristiques du rendez-vous. Les éléments précédés d'une étoile sont obligatoires. 

4. Validez la fenêtre de description du champ Planning. Le champ Planning est un planning sur variable. 










## Manipuler un champ Planning Fichier ou un champ Planning sur variable
<a name="manipuler_champ_planning_fichier_champ_planning_sur_variable_ELTTEXTE000197"></a>


### Champ Planning Fichier
<a name="champ_planning_fichier_ELTPARAGRAPHE000060"></a>Un champ Planning fichier se manipule très simplement : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Si l'utilisateur manipule directement le planning via le menu contextuel du champ, les ajouts, suppressions, modifications des rendez-vous sont automatiquement reportés dans le fichier de données lié au champ. 

- Si l'utilisateur manipule les rendez-vous via une fenêtre ou une page spécifique, il suffit de modifier directement le fichier de données lié au planning. Les données affichées seront automatiquement mises à jour. Il est également possible d'utiliser la fonction [PlanningAffiche](../WDLang1/1000019426.md) pour remettre à jour les données affichées dans le champ. 




**Remarque** : Si le champ est lié à une source de données, la suppression réalisée dans le champ est effectuée en même temps dans la source de données liée. 


### Champ Planning sur variable
<a name="champ_planning_sur_variable_ELTPARAGRAPHE000078"></a>Un champ Planning sur variable se manipule très simplement : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Si l'utilisateur manipule directement le planning via le menu contextuel du champ, les ajouts, suppressions, modifications des rendez-vous sont automatiquement reportés dans la source de données du champ.

- Si l'utilisateur manipule les rendez-vous via une fenêtre ou une page spécifique, il suffit de modifier directement les éléments lié au planning (membres de classe par exemple). Les données affichées seront automatiquement mises à jour. Il est également possible d'utiliser la fonction [PlanningAffiche](../WDLang1/1000019426.md) pour remettre à jour les données affichées dans le champ.




**Remarque** : Si le champ est lié à une source de données, la suppression réalisée dans le champ est effectuée en même temps dans la source de données liée.

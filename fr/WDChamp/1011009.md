
## Définir un nom de fichier unique pour le fichier généré
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000113"></a>
Lors de la génération d'un fichier sur le serveur Web (au format PDF, HTML, ...), il est nécessaire de définir un nom de fichier unique.

WEBDEV propose plusieurs méthodes pour définir un nom de fichier unique.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000137"></a>
Pour définir un nom de fichier unique, il est possible :

- soit de concaténer le résultat des fonctions [DateSys](../WDLang1/3027026.md) et [HeureSys](../WDLang1/3027019.md). 
	Par exemple :
	
	```wl
	UnFichier est une chaîne = fRepWeb() + "\" + DateSys() + HeureSys() + ".htm"
	```


- soit d'utiliser un identifiant unique présent dans la base de données du site.
	Par exemple :
	
	```wl
	UnFichier est une chaîne = fRepWeb() + "\" + Client.Id + Commande.Num + ".htm"
	```


- soit d'utiliser la fonction [fFichierTemp](../WDLang1/3036011.md).
	Par exemple :
	
	```wl
	UnFichier est une chaîne 
	UnFichier = fFichierTemp()
	UnFichier = fExtraitChemin(UnFichier, fFichier)
	UnFichier = fRepWeb() + "\" + UnFichier + ".htm"
	```






<a name="NOTE2_2"></a>


### A ne pas faire
<a name="pas_faire_ELTPARAGRAPHE000052"></a>Les fonctions suivantes ne permettent pas d'obtenir un nom de fichier unique :

- [DonneIdentifiant](../WDLang1/3013028.md).

- [ConnexionOccurence](../WDLang2/3012004.md).

- [Hasard](../WDLang1/3013037.md).

- ...

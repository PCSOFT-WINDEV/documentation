<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000133"></a>
Voici les principaux points de la manipulation en WLangage des champs Table remplis par programmation. 

Le WLangage met à votre disposition de nombreuses fonctions permettant de gérer par programmation ce type de champ. N'hésitez pas à consulter la documentation de ces fonctions. 

Pour plus de détails, consultez [Fonctions de gestion des champs Table](../WDLang1/1000024319.md).

<a name="NOTE2_1"></a>


## Initialisation d'un champ Table par programmation
<a name="initialisation_champ_table_par_programmation_ELTTEXTE000176"></a>
La création d'une ligne d'un champ Table n'est pas automatique. Elle doit être demandée explicitement par un appel aux fonctions :

- [&lt;Table&gt;.Ajoute](../WDLang1/1000023600.md) ou [&lt;Table&gt;.AjouteLigne](../WDLang1/1000023404.md) : ajoute une ligne à la fin du champ Table. 

- [&lt;Table&gt;.Insère](../WDLang1/1000024260.md) ou [&lt;Table&gt;.InsèreLigne](../WDLang1/1000023407.md) : ajoute une ligne à l'indice spécifié ou à l'indice en cours. 




**Remarque** : si l'option "Saisie en cascade (Ajout)" n'est pas sélectionnée :

- A sa création, un champ Table ne comporte aucune ligne : 
	TABLE_MaTable.[Occurrence](../WDLang1/3025009.md) = 0. 
	Dans ce cas, aucune saisie n'est possible et la propriété [Vide](../Proprietes/2510126.md) est à <u><u><u><u>Vrai</u></u></u></u>.

- Pour insérer automatiquement une ligne si le champ Table est vide, il faut utiliser les lignes de code suivantes dans l'événement "Initialisation" du champ Table : 
	
	```wl
	SI TABLE_MaTable.Vide = Vrai ALORS TABLE_MaTable.Ajoute()
	```

## Ajout de lignes dans un champ Table par programmation
<a name="ajout_lignes_dans_champ_table_par_programmation_ELTTEXTE000253"></a>
L'ajout de lignes dans un champ Table par programmation est réalisé :

- soit avec les fonctions [&lt;Table&gt;.AjouteLigne](../WDLang1/1000023404.md) et [&lt;Table&gt;.InsèreLigne](../WDLang1/1000023407.md) : dans chaque ligne, la valeur de chaque colonne est séparée par des virgules.

- soit avec les fonctions [&lt;Table&gt;.Ajoute](../WDLang1/1000023600.md) et [&lt;Table&gt;.Insère](../WDLang1/1000024260.md) : dans une ligne, chaque valeur de colonne est séparée par TAB.


![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Si une nouvelle ligne a été ajoutée dans le champ Table avec [&lt;Table&gt;.Ajoute](../WDLang1/1000023600.md), [&lt;Table&gt;.Insère](../WDLang1/1000024260.md), [&lt;Table&gt;.AjouteLigne](../WDLang1/1000023404.md) ou [&lt;Table&gt;.InsèreLigne](../WDLang1/1000023407.md), la propriété [Nouveau](../Proprietes/2510073.md) est à <u><u><u><u>Vrai</u></u></u></u> (sinon elle est à <u><u><u><u>Faux</u></u></u></u>).

## Gestion des colonnes calculées
<a name="gestion_des_colonnes_calculees_ELTTEXTE000337"></a>
La formule de calcul d'une colonne calculée doit être décrite dans l'événement "Affichage d'une ligne" du champ Table. Par exemple : 

```wl
// COL_PHT : colonne calculée
// COL_PUHT et COL_QTE : colonnes remplies par programmation
COL_PHT = COL_PUHT * COL_QTE
```

## Modification d'une ligne ou d'une colonne
<a name="modification_une_ligne_une_colonne_ELTTEXTE000365"></a>
Le contenu des lignes et des colonnes d'un champ Table peut être modifié :

- par l'utilisateur, en saisissant directement dans les colonnes. Les modifications sont mémorisées automatiquement dans le champ Table, sans programmation complémentaire. La propriété [Modifié](../Proprietes/2510080.md) est à <u><u><u><u>Vrai</u></u></u></u>.

- par programmation :

	- avec la fonction [&lt;Table&gt;.Modifie](../WDLang1/1000024273.md) pour modifier le contenu de la ligne en cours ou d'une ligne donnée.
			Par exemple :
			
		```wl
		TABLE_CLIENT.Modifie("PY" + TAB + "Jean" + TAB + "Nîmes")
		```

			**Remarque** : il est également possible d'utiliser directement le nom du champ Table : 
			
		```wl
		// Modifier la ligne en cours
		TABLE_CLIENT = "DURAND" + TAB +"Sophie" + TAB + "Nîmes"
		// Modifier la ligne 3
		TABLE_JOUR.Modifie("Mercredi" + TAB + "Vaqué", 3)
		```


	- directement en donnant le nom de la colonne (comme pour un champ de saisie) pour modifier le contenu de la colonne. Pour modifier une colonne d'une ligne précise, il faut indiquer le numéro de la ligne (indice).
			Par exemple :
			
		```wl
		COL_NOM[Indice] = SAI_NomClient
		```

			Pour modifier une colonne de la ligne en cours, il n'est pas nécessaire de préciser l'indice. Par exemple : 
			
		```wl
		COL_NOM = SAI_NomClient
		```

			La propriété [Modifié](../Proprietes/2510080.md) est à <u><u><u><u>Faux</u></u></u></u> (elle n'est mise à <u><u><u><u>Vrai</u></u></u></u> que par une saisie dans le champ Table).

### Forcer la saisie
<a name="forcer_saisie_ELTPARAGRAPHE000455"></a>La fonction [DonneFocusEtRetourUtilisateur](../WDLang1/1410088107.md) peut être utilisée pour se replacer en saisie sur une colonne de la ligne en cours.

Par exemple :
```wl
// Traitement d'entrée colonne COL_QTE
// COL_QTE ne peut pas être saisi, 
// si COL_PRODUIT n'est pas saisi
SI SansEspace(COL_PRODUIT) = "" ALORS
	Erreur("La colonne Produit doit être saisie en premier")
	DonneFocusEtRetourUtilisateur(COL_PRODUIT)
FIN
```

### Ajout ou suppression d'une colonne
<a name="ajout_suppression_une_colonne_ELTPARAGRAPHE000473"></a>Il est possible de :

- Ajouter une colonne dans un champ Table par programmation grâce à la fonction [ChampClone](../WDLang1/3025017.md).

- Supprimer une colonne dans un champ Table par programmation grâce à la fonction [ChampSupprime](../WDLang1/3025037.md).

## Opérations sur les lignes et les cellules
<a name="operations_sur_les_lignes_les_cellules_ELTTEXTE000487"></a>


### Suppression d'une ligne
<a name="suppression_une_ligne_ELTPARAGRAPHE000510"></a>La suppression d'une ligne doit être demandée explicitement à l'aide de la fonction [&lt;Table&gt;.Supprime](../WDLang1/1000024289.md). La syntaxe utilisée est :

```txt
<Champ Table>.Supprime([<Indice>])
```
Si l'indice est précisé, la fonction [&lt;Table&gt;.Supprime](../WDLang1/1000024289.md) supprime la ligne correspondant à l'indice. Sinon, elle supprime la ligne en cours. Par exemple :

```wl
TABLE_CLIENT.Supprime()
```

<a name="NOTE6_2"></a>
La sélection d'une ligne s'effectue avec la fonction [&lt;Table&gt;.SelectPlus](../WDLang1/1000024287.md).

La syntaxe utilisée est :

```txt
<Champ Table>.SelectPlus([<Indice>])
```


<a name="NOTE6_3"></a>


### Récupération d'une ligne ou d'une cellule
<a name="recuperation_une_ligne_une_cellule_ELTPARAGRAPHE000564"></a>Le contenu d'un champ Table rempli par programmation peut être récupéré :

- soit la ligne entière. 

- soit cellule par cellule.



<a name="NOTE6_4"></a>


### Valeur d'une cellule de la ligne en cours
<a name="valeur_une_cellule_ligne_cours_ELTPARAGRAPHE000577"></a>Pour récupérer la valeur d'une colonne (ou cellule) de la ligne en cours, la syntaxe est la même que pour un champ de saisie simple.

```txt
<Valeur> = <Nom Colonne>
```
Exemple : 

```wl
// COL_QTE est une colonne du champ Table
SI COL_QTE < 10 ALORS
	INFO("Qté insuffisante")
FIN
```

<a name="NOTE6_5"></a>


### Valeur d'une cellule pour une ligne spécifique
<a name="valeur_une_cellule_pour_une_ligne_specifique_ELTPARAGRAPHE000590"></a>Pour récupérer la valeur d'une colonne qui n'est pas sur la ligne en cours, il faut passer l'indice de la ligne.

```txt
<Valeur> = <Nom Colonne>[<Indice>]
```
Par exemple : 

```wl
// Totaliser le Prix PHT pour toutes les lignes de commande
PrixTotal = 0
POUR Indice = 1 _A_ TABLE_COMMANDE.Occurrence
	PrixTotal = PrixTotal + COL_PHT[Indice]
FIN
```

<a name="NOTE6_6"></a>


### Contenu de la ligne en cours
<a name="contenu_ligne_cours_ELTPARAGRAPHE000604"></a>Pour récupérer le contenu de la ligne en cours, la syntaxe est la suivante :

```txt
<Valeur> = <Champ Table>
```

<a name="NOTE6_7"></a>


### Contenu d'une ligne spécifique
<a name="contenu_une_ligne_specifique_ELTPARAGRAPHE000615"></a>Pour récupérer le contenu de la ligne d'indice &lt;Indice&gt;, la syntaxe est la suivante :

```txt
<Valeur> = <Champ Table>[<Indice>]
```

**Remarque** : L'indice de la ligne en cours peut être connu par la fonction [&lt;Table&gt;.Select](../WDLang1/1000024284.md). 
Par exemple :

```wl
// Récupérer la ligne 10 dans le champ Table TABLE_CLIENT
LigneEnCours = TABLE_CLIENT[10]
 
// Nom du client sélectionné
NomClient = COL_NOM[TABLE_CLIENT.Select()]
 
// Récupérer la ligne en cours dans le champ Table TABLE_CLIENT
LigneEnCours = TABLE_CLIENT
```

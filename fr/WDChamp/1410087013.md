## Présentation du champ Signature
<a name="presentation_champ_signature_ELTTEXTE000149"></a>


### Champ d'un état
<a name="champ_etat_ELTPARAGRAPHE000011"></a>Le champ Signature est un champ d'un état permet de signer : 

- les états générés sous forme de fichier PDF. 

- les duplicatas d'états. 




Ce champ Signature permet de mettre en place 2 types de signatures : 

- **Signature d'approbation** : Une signature d'approbation est une signature numérique appliquée à un document. Un document PDF peut contenir plusieurs signatures d'approbation successives, réalisées avec le logiciel adéquat. Lorsque le document contient une signature d'approbation, un bandeau avec une coche verte est affiché sous Adobe Acrobat. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ChampSignature%20-%20HC%20N%B0001.gif)


- **Signature de certification** : Par rapport à la signature d'approbation, la signature de certification garantit un niveau supérieur de contrôle des documents. Lorsque le document est certifié, un bandeau bleu avec une icône de certification est affiché sous Adobe Acrobat. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ChampSignature%20-%20HC%20N%B0002.gif)

## Créer un champ Signature
<a name="creer_champ_signature_ELTTEXTE000173"></a>
**Pour créer un champ Signature** :

1. Affichez si nécessaire un état sous l'éditeur d'états. 

2. Sous le volet "Création", dans le groupe "Certification", déroulez "Signature" et sélectionnez le type de signature à créer : signature d'approbation ou de certification.

3. Cliquez dans l'état à la position où le champ doit être créé.




Pour afficher les caractéristiques du champ, sélectionnez l'option "Description" dans le menu contextuel du champ.

Remarque : un seul champ Signature peut être créé dans un état.

## Caractéristiques du champ Signature
<a name="caracteristiques_champ_signature_ELTTEXTE000197"></a>
Le champ Signature peut être paramétré directement sous l'éditeur grâce à sa fenêtre de description.

Les principales caractéristiques du champ sont disponibles dans l'onglet "Général" : 

- **Type de signature** : Type de signature à générer : signature d'approbation ou de certification. 
	Remarque : Il est possible de modifier le type de la signature par programmation grâce à la propriété [TypeSignature](../Proprietes/1000021995.md). 

- **Libellé** : Libellé affiché dans le champ Signature. Ce libellé peut contenir par exemple des informations sur la personne ayant signé l'état. 
	Remarque : Il est possible de modifier le libellé de la signature par programmation grâce à la propriété [Libellé](../Proprietes/2510053.md). 

- **Image** : Image affichée dans le champ Signature. 
	Remarque : Il est possible de modifier l'image de la signature par programmation grâce à la propriété [Image](../Proprietes/2510034.md). 

- **Certificat par défaut** : Certificat utilisé pour signer l'état. Les différents certificats installés sur le poste en cours sont proposés.
	Remarque : Il est possible de modifier le certificat utilisé pour la signature par programmation grâce à la propriété [Certificat](../Proprietes/1000021684.md). 




Remarques : 

- Le libellé et l'image sont des informations optionnelles.

- Si le champ Signature est invisible (option disponible sous l'onglet "UI" de la fenêtre de description), l'état PDF généré (ou le duplicata) ne sera pas signé. 

- Si le certificat associé au champ Signature n'est pas trouvé, l'état PDF généré (ou le duplicata) ne sera pas signé.




- **Attention** : Pour qu'un fichier PDF signé soit reconnu par Acrobat, l'organisme émetteur du certificat de signature doit être membre de la liste AATL ([Adobe Authorized Trust List](https://helpx.adobe.com/acrobat/kb/trust-services.html)) ou du programme CDS (Certified Document Services). 
	Si le certificat n'est pas reconnu, Acrobat affichera un bandeau "Une signature au moins présente un problème...".

## Créer une signature dans un état imprimé en WLangage
<a name="creer_une_signature_dans_etat_imprime_wlangage_ELTTEXTE000230"></a>
Il est également possible de signer les impressions réalisées en WLangage. Il suffit d'utiliser la fonction [iImprimeSignature](../WDLang5/1000021690.md).


## Manipuler un champ Editeur d'images par programmation
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000163"></a>
WINDEV permet de manipuler un champ Editeur d'images par programmation. Il suffit d'utiliser directement la variable du champ dans le code.

Un champ Editeur d'images peut également être manipulé par programmation grâce aux [fonctions Picxxx](../WDLang1/1000023383.md). Ces fonctions permettent de créer, sauver, ouvrir et fermer une image dans un champ Editeur d'images. 

Cette page d'aide détaille comment manipuler un champ Editeur d'images par programmation. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Afficher une image dans un champ Editeur d'images
<a name="afficher_une_image_dans_champ_editeur_images_ELTTEXTE000187"></a>
Pour afficher une image dans un champ Editeur d'images, il est possible de : 

- utiliser une affectation (directe ou par variable) avec chemin du fichier image. 
	
	```wl
	// Affectation directe
	EI_MonImage = "C:\Temp\MonImage.bmp"
	// Affectation par variable
	CheminImage = fRepEnCours() + "\MonImage.bmp"
	EI_MonImage = CheminImage
	```


- utiliser une affectation avec une variable de type [Image](../WDLang1/1000019650.md).
	
	```wl
	MonImage est une Image
	// Chargement à partir d'un fichier
	MonImage = dChargeImage("image.png")
	// Affectation du champ Editeur d'images
	EI_MonImage = MonImage
	```





Remarque : Pour connaître le chemin du fichier image affiché dans un champ Editeur d'images, utilisez la propriété [CheminFichier](../Proprietes/1000023104.md). 

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Vider ou réinitialiser le contenu d'un champ Editeur d'images
<a name="vider_reinitialiser_contenu_champ_editeur_images_ELTTEXTE000211"></a>
Pour vider ou réinitialiser un champ Editeur d'images, il suffit d'affecter une chaîne vide au champ Editeur d'images. Par exemple :

```wl
EI_MonImage = ""
```
Remarque : Il est également possible d'utiliser la fonction [PicFerme](../WDLang1/1000023382.md). 

<a name="NOTE3b"></a>
<a name="NOTE3b_1"></a>


## Définir les options par défaut d'un champ Editeur d'images
<a name="definir_les_options_par_defaut_champ_editeur_images_ELTTEXTE000235"></a>
Il est possible de : 

- définir l'outil utilisé par défaut à l'ouverture du champ Editeur d'image grâce à la propriété [OutilInitial](../Proprietes/1410088609.md). 

- définir les dimensions par défaut proposées lors de la création d'une nouvelle image dans le champ Editeur d'images grâce aux propriétés [HauteurImageInitiale](../Proprietes/1410088607.md) et [LargeurImageInitiale](../Proprietes/1410088608.md). 






<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Manipuler l'image affichée dans le champ Editeur d'images
<a name="manipuler_image_affichee_dans_champ_editeur_images_ELTTEXTE000259"></a>
Pour manipuler l'image présente dans le champ Editeur d'images, il est conseillé d'utiliser une variable de type [Image](../WDLang1/1000019650.md). Toutes les [fonctions de dessin](../WDLang1/3029033.md) peuvent ensuite être utilisées sur cette variable de type [Image](../WDLang1/1000019650.md). Pour mettre à jour le champ Editeur d'images, il suffit d'affecter le champ avec la variable de type [Image](../WDLang1/1000019650.md). 

Il est également possible d'utiliser une variable de type [WDPic](../WDLang1/1000023503.md). Cette variable est notamment utilisée pour gérer l'affichage des calques dans un fichier ".wdpic".

## Propriétés WLangage associées
<a name="proprietes_wlangage_associees_ELTTEXTE000283"></a>


### Propriétés spécifiques à la gestion d'un champ Editeur d'images
<a name="proprietes_specifiques_gestion_champ_editeur_images_ELTPARAGRAPHE000106"></a>Les propriétés suivantes sont spécifiques à la gestion des caractéristiques d'un champ Editeur d'images par programmation : 



|   |   |
| --- | --- |
| [CheminFichier](../Proprietes/1000023104.md) | La propriété **CheminFichier** permet de connaître : <br><br>- le nom du fichier xlsx associé à un champ Tableur.<br><br>- le nom du fichier associé à un champ Editeur d'images. <br><br>- le nom du fichier PDF associée à un champ Lecteur PDF. <br><br>- le nom du fichier DOCX associé à un champ Traitement de texte. <br><br>- le nom du fichier wddiag associé à un champ Editeur de diagrammes. <br><br><br> |
| [HauteurImageInitiale](../Proprietes/1410088607.md) | La propriété **HauteurImageInitiale** permet de connaître ou de modifier la hauteur par défaut de l'image créée dans le champ Editeur d'images. |
| [ImageType](../Proprietes/1000023516.md) | La propriété **ImageType** permet de connaître le type de l'image en cours d'édition dans un champ Editeur d'images (icône ou image). |
| [LargeurImageInitiale](../Proprietes/1410088608.md) | La propriété **LargeurImageInitiale** permet de connaître ou de modifier la largeur par défaut de l'image créée dans le champ Editeur d'images. |
| [OutilInitial](../Proprietes/1410088609.md) | La propriété **OutilInitial** permet de connaître ou de modifier l'outil de dessin initial sélectionné dans le champ Editeur d'images. |
| [VoletVisible](../Proprietes/1000023515.md) | La propriété **VoletVisible** permet de : <br><br>- savoir si le volet du champ Editeur d'images est visible ou non, <br><br>- rendre visible ou invisible le volet du champ Editeur d'images.<br><br><br> |





Pour connaître la liste complète des propriétés du WLangage pouvant être utilisées avec un champ de type Editeur d'images, consultez [Propriétés associées au champ Editeur d'images](../WDChamp/1410087033.md).

## Evénements gérés par défaut
<a name="evenements_geres_par_defaut_ELTTEXTE000134"></a>
WINDEV et WINDEV Mobile gèrent par défaut les événements suivants :


| Evénement | Condition d'exécution |
| --- | --- |
| Initialisation | Exécuté à l'ouverture de la fenêtre. (\*) |
|   | Exécuté lors du clic sur le champ.<br><br>Cet événement permet par exemple de déclencher la prise de photo ou l'enregistrement du clip (puis son arrêt) en cliquant sur la prévisualisation.<br><br><br>	<B><B><B><B>Avertissement</B></B></B></B><br><br>	 Cet événement est disponible et exécuté uniquement si le champ Caméra est en mode "Compatible 26".<br>
 |
|   | Exécuté lors de la modification du zoom sur le champ Caméra<br><br>**Remarque : Cet événement est disponible uniquement pour les Champs Caméra créés avec la version 27 ou supérieurs (champ n'ayant pas l'option "Compatible 26 cochée" dans l'onglet "Général" de la fenêtre de description du champ).** |
|   | Exécuté lors de la détection d'un code-barres par la caméra. <br><br>**Cet événement est disponible uniquement si le décodage des codes-barres a été configuré dans l'onglet "Détail" de la fenêtre de description du champ.** <br><br>Cet événement est de la forme : <br><br><pre><code>PROCEDURE DécodeCodeBarre(cb est un CodeBarres)</code></pre><br><br><br>Il est possible de récupérer les informations lues via la variable cb de type [CodeBarres](../WDLang5/1000019451.md). <br><br>Pour plus de détails, consultez [Champ Caméra : décoder les codes-barres](../WDChamp/1013304.md). |


(\*) Par défaut, l'événement "Initialisation" de chaque champ est exécuté selon l'ordre de création des différents champs dans la fenêtre. Pour modifier cet ordre d'exécution : 

1. Sous le volet "Fenêtre", dans le groupe "Ordre", cliquez sur "Initialisation".

2. Dans la fenêtre qui s'affiche, utilisez les flèches situées sur la droite pour modifier l'ordre d'initialisation des champs.

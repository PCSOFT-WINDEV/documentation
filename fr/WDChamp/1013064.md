
## Les différents états d'un champ Bouton dans une fenêtre
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000155"></a>
Dans une application WINDEV ou WINDEV Mobile, les champs Bouton peuvent avoir plusieurs états. Ces états permettent de gérer les différentes actions réalisées par l'utilisateur sur le champ Bouton.

La représentation d'un état du champ Bouton peut être gérée :

- soit par une image.

- soit par le style du champ.




Les différents états sont les suivants :

| Etat | Description |
| --- | --- |
| Normal | Le champ Bouton est au repos. |
| Enfoncé | L'utilisateur a cliqué sur le champ Bouton, mais n'a pas encore re-lâché le bouton de la souris. Le champ Bouton est enfoncé. |
| Grisé | Le champ Bouton est inactif, grisé, et l'utilisateur ne peut pas le sélectionner. |
| En cours | Le champ Bouton a le focus, il est sélectionné par l'utilisateur. |
| Survol | Le champ Bouton est survolé par le curseur de la souris. |





<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Gestion des états grâce au style
<a name="gestion_des_etats_grace_style_ELTTEXTE000179"></a>
L'onglet "Style" permet de gérer simplement les différents états d'un champ Bouton affichant un texte. Il suffit de sélectionner l'état voulu et l'élément à configurer.


![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Btn_Style_1.gif)


L'état "Général" correspond au style général du champ Bouton. Ce style général est reporté dans tous les états du champ ("Repos", "Survol" et "Enfoncé").


<a name="NOTE2_2"></a>


### Configurer un des états du champ Bouton
<a name="configurer_des_etats_champ_bouton_ELTPARAGRAPHE000061"></a>Pour configurer un des états du champ Bouton :

1. Affichez l'onglet "Style" de la description du champ.

2. Sélectionnez l'état voulu.

3. Sélectionnez l'élément à configurer en fonction de l'état

4. Configurez les caractéristiques de l'élément pour l'état sélectionné (couleur de fond, cadre, ...).




**Remarque** : Dans ce cas, l'état "Grisé" correspond uniquement au libellé grisé.

## Gestion des états par les images
<a name="gestion_des_etats_par_les_images_ELTTEXTE000209"></a>
Pour configurer les états d'un champ Bouton affichant des images (ou dont la forme est redéfinie), il est nécessaire de gérer les différents états par l'intermédiaire d'images.

Chaque état correspond à un dessin spécifique. Tous les dessins sont enregistrées dans une seule image. Cette image est associée au champ Bouton : il suffit de préciser le nombre d'états (c'est-à-dire de dessins) présents dans le fichier image.

**Exemple** : fichier image BoutonBleu.Gif

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Bleu0208.gif)


Champ Bouton au repos enfoncé grisé sélectionné survolé

WINDEV propose deux types d'images pour les champs Bouton :

- **l'image du cadre du champ** : cette image permet de définir la forme du champ Bouton. Elle peut contenir 1 à 5 dessins, correspondant aux différents états du champ.
	

- **l'image associée au champ** : cette image sera affichée sur le champ Bouton, superposée au cadre du champ s'il existe. Elle peut contenir 1 à 5 dessins correspondant aux différents états du champ.
	Il est possible de gérer séparément :

	- les états de l'image associée au champ Bouton

	- les états de l'image du cadre du champ Bouton.
			L'image associée au champ Bouton pourra alors être ré-utilisée quelle que soit la forme du champ.







Pour plus de détails sur la gestion des images avec les champs Bouton, consultez [Images et boutons](../WDChamp/1013075.md).

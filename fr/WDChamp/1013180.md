
## Champ Liste lié à un fichier de données ou une requête
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000080"></a>
Un champ Liste ou un champ Combo peut être utilisé pour afficher le contenu d'un fichier de données ou d'une requête, par exemple pour qu'un utilisateur ou un internaute sélectionne un enregistrement.

Lorsqu'un champ Liste est lié à un fichier de données (ou une requête), le champ Liste pourra être initialisé avec le contenu du fichier de données (ou de la requête):

- soit ***automatiquement*** : aucun code spécifique n'est nécessaire pour initialiser le champ Liste. Les caractéristiques du parcours du fichier de données lié au champ Liste sont décrites dans l'onglet "Contenu" de la description du champ Liste.

- soit par ***programmation*** : Il est nécessaire de saisir le code de parcours du fichier de données lié au champ Liste dans le code d'initialisation du champ Liste. Cette méthode est déconseillée.




**Remarque** : Lors de la création d'un champ Liste, l'assistant demande les différents renseignements nécessaires à la création d'un champ Liste lié à un fichier de données ou à une requête.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Code d'initialisation d'un champ Liste programmé
<a name="code_initialisation_champ_liste_programme_ELTTEXTE000104"></a>
Si vous utilisez un champ Liste basé sur un fichier de données avec initialisation programmée, il est nécessaire de saisir le code de parcours du fichier de données dans le code d'initialisation du champ Liste.

Par exemple : Initialisation d'un champ Liste avec les clients du fichier de données CLIENT.


```wl
// -- Code d'initialisation d'un champ Liste
HLitPremier(CLIENT, NomCli)
TANTQUE PAS HEnDehors()
	ListeAjoute(LISTE_Liste1, Client.NomCli)
	HLitSuivant(CLIENT, NomCli)
FIN
```

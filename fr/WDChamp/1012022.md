
## Les contextes de pages
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000130"></a>
Dans une application Windows, toutes les informations relatives à la fenêtre sont conservées avec la fenêtre.

Dans un site dynamique WEBDEV, pour chaque page affichée sur le navigateur de l'internaute, un contexte de page existe sur le serveur. 

Le contexte de page regroupe toutes les informations relatives à la page :

- contenu des champs,

- variables locales,

- variables globales,

- code WLangage "serveur", ...






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Fonctionnement
<a name="fonctionnement_ELTTEXTE000154"></a>


### Gestion automatique
<a name="gestion_automatique_ELTPARAGRAPHE000028"></a>Par défaut, WEBDEV gère automatiquement les contextes de pages dynamiques :

- Un contexte de page est ouvert lors de l'affichage de la page dynamique sur le navigateur.

- Le contexte d'une page est mis à jour en fonction des informations saisies par l'internaute sous le navigateur. Cette mise à jour se fait lors de la validation de la page (à l'aide d'un bouton de type "Envoi au serveur" ou de la fonction [PageValide](../WDLang2/3058020.md)).

- Les contextes de page existants sont fermés lors de l'utilisation des fonctions [PageUtilise](../WDLang2/3058016.md) ou [FramesetUtilise](../WDLang2/3058012.md). Les contextes de page correspondants aux pages à afficher sont alors ouverts.





<a name="NOTE2_2"></a>


### Gestion avancée
<a name="gestion_avancee_ELTPARAGRAPHE000049"></a>WEBDEV offre la possibilité de réaliser une gestion avancée des contextes grâce aux fonctions ContexteXXX.

Il est ainsi possible de :

- ouvrir le contexte d'une page sans en lancer l'affichage sur le navigateur ([ContexteOuvre](../WDLang2/3058004.md)),

- tester l'existence du contexte d'une page sur le serveur ([ContexteExiste](../WDLang2/3058002.md)),

- fermer le contexte d'une page inutilisée ([ContexteFerme](../WDLang2/3058003.md)).




Cette gestion avancée des contextes de page peut être utilisée pour :

- Économiser l'espace mémoire du serveur (par exemple, en fermant les contextes des pages précédemment ouvertes mais inutilisées).

- Préparer des pages destinées à dialoguer avec d'autres applications (par exemple, la page recevant les informations en retour d'un site de paiement sécurisé).

- Manipuler des pages à partir d'autres pages.





<a name="NOTE2_3"></a>


### Cas particulier
<a name="cas_particulier_ELTPARAGRAPHE000078"></a>Lors de l'utilisation du bouton "Précédent" du navigateur, une désynchronisation peut exister entre la page et son contexte. Pour plus de détails, consultez [La gestion du bouton "Précédent" du navigateur.](../WDChamp/1012001.md)

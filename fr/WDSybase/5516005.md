
## Connecteur Natif Sybase pour WINDEV et WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000130"></a>
En option, WINDEV et WEBDEV proposent un Connecteur Natif à Sybase (également appelé Accès Natif à Sybase). Ce Connecteur Natif permet d'accéder à une base Sybase sans driver externe depuis un programme en WLangage.

Cette version permet également :

- D'exécuter plusieurs fois une même requête.

- De récupérer la valeur de sortie d'une procédure stockée exécutée sur le serveur




Ces fonctionnalités sont disponibles à partir de la version 12.5 de Sybase Adaptive Server Enterprise (ASE).

**Attention** : Cette version du Connecteur Natif Sybase ne fonctionne qu'avec WINDEV 28 et/ou WEBDEV 28.





<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Configuration nécessaire
<a name="configuration_necessaire_ELTTEXTE000154"></a>


### Conditions nécessaires à l'utilisation du Connecteur Natif Sybase
<a name="conditions_necessaires_utilisation_connecteur_natif_sybase_ELTPARAGRAPHE000032"></a>

- Sybase Adaptive Server Enterprise (ASE) version 10 ou supérieure (version 12.5 pour récupérer le résultat d'une procédure stockée).

- Couche client Sybase installée.

- La DLL ***LIBSYBDB.DLL*** doit être accessible :

	- à partir du poste de développement.

	- à partir du poste de déploiement.






<a name="NOTE3_2"></a>


### Les fichiers nécessaires
<a name="les_fichiers_necessaires_ELTPARAGRAPHE000044"></a>Les fichiers nécessaires à l'utilisation du Connecteur Natif Sybase sont :

- la DLL ***LIBSYBDB.DLL***. Cette DLL est installée par la couche client de Sybase. Pour les conditions de diffusion de cette DLL, reportez-vous à la licence de Sybase.

- ***wd280syb.dll*** : Nécessaire au Connecteur Natif à une base Sybase. Ce fichier est livré avec le Connecteur Natif WINDEV/WEBDEV pour Sybase.

## Utilisation
<a name="utilisation_ELTTEXTE000184"></a>


### Utiliser le Connecteur Natif Sybase dans vos applications WINDEV et vos sites WEBDEV
<a name="utiliser_connecteur_natif_sybase_dans_vos_applications_windev_vos_sites_webdev_ELTPARAGRAPHE000058"></a>Pour utiliser Sybase en natif, il suffit de programmer :

- soit avec les fonctions "SQL" du WLangage (voir [Programmation à l'aide des fonctions SQL](../WDSybase/5516001.md)).

- soit avec les fonctions "HFSQL" du WLangage (voir [Programmation à l'aide des fonctions HFSQL](../WDSybase/5516006.md)).




**Remarques** : 

- N'oubliez pas de consulter les [limitations concernant l'utilisation du Connecteur Natif Sybase](../WDSybase/5516004.md).

- L'option "Créer les fichiers de données automatiquement au moment de leur ouverture" (disponible dans l'onglet "Fichiers" de la description du projet) est prise en compte par le Connecteur Natif SQL Sybase.

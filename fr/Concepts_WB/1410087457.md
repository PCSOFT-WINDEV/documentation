
## 18. Gestion des règles métier

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
WEBDEV permet de gérer des règles métier. Une règle métier permet de définir un mode opératoire précis ou de préciser un traitement particulier. Par exemple : le calcul d'un taux de TVA avec ses particularités, les règles de changement de statut d'un client, la formule de calcul d'un montant de frais de port, une commission commerciale, un taux de remise, un coefficient de vétusté, etc.<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P3-Gestion%20des%20r%E8gles%20m%E9tier.gif)
Une règle métier peut être simple ou élaborée.

Les règles métier peuvent provenir :

- du cahier des charges (correspondant aux exigences).

- de suggestions faites par les utilisateurs de l'application.




Pendant le développement, les règles métier définies pour le projet sont affichées directement dans le volet "Règles métier" de l'environnement de développement. Ce volet présente le nombre d'éléments du projet auxquels les règles métier s'appliquent et le pourcentage de règle actuellement réalisé.

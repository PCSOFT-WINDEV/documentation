
## 19. Envoyer des emails

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Dans un site Web, les emails permettent :

- de simplifier la communication entre les internautes : envoi d'emails au responsable du site, site de mailing-list, etc.

- de valider une prise de commande : dans un site commercial, un email est envoyé aux internautes pour valider leur commande.

- de transférer des données d'un poste à un autre : les nouvelles commandes saisies sur un site commercial sont transférées par email au siège de la société pour traitement, etc.




WEBDEV permet d'envoyer des emails depuis un site par 2 méthodes.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## 1. Envoi de l'email depuis le poste de l'internaute
<a name="1_envoi_email_depuis_poste_internaute_ELTTEXTE000149"></a>
**Utilisez cette méthode lorsque l'internaute doit envoyer un email à une adresse définie : auteur du site, support technique, service commercial, etc.**

La fonction navigateur [EmailOuvreMessagerie](../WDLang2/3032021.md) permet d'ouvrir le logiciel de messagerie de l'internaute. Certains paramètres peuvent être remplis par défaut : adresse email du destinataire, etc. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P4-Envoyer%20des%20emails.gif)
Cette méthode permet d'utiliser directement la messagerie de l'internaute : aucun traitement spécifique n'est à effectuer dans le site WEBDEV.

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## 2. Envoi de l'email depuis le serveur (pages Session ou AWP uniquement)
<a name="2_envoi_email_depuis_serveur_pages_session_awp_uniquement_ELTTEXTE000173"></a>
Dans ce cas, c'est le serveur qui gère l'envoi des emails.

L'envoi des emails peut être réalisé :

- directement dans le site WEBDEV (par exemple pour un transfert de données par email, ou pour un email de validation de commande) ;

- par le spooler d'email (livré avec WEBDEV).

<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P4-Envoi%20de%20l'email%20depuis%20le%20serveur.gif)
Utilisez cette méthode pour une gestion personnalisée des emails :

- un contrôle sur le contenu des emails envoyés par les internautes (cas d'un site de type "liste de diffusion"),

- la vérification de la validité de l'adresse du destinataire,

- le cryptage des informations envoyées par emails,

- la validation d'un traitement, etc.

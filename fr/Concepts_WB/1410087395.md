## 2. Installation sur un serveur Windows avec un accès à Internet par UNIX
<a name="2_installation_sur_serveur_windows_avec_acces_internet_par_unix_ELTTEXTE000169"></a>
La configuration est la suivante : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P8-Installation%20sur%20un%20serveur%20Windows.gif)


1. Ouverture d'une page statique. La page statique est présente sur le serveur UNIX.

2. Lancement du site dynamique WEBDEV (présent sur le serveur Windows) par un lien présent dans la page HTML statique.

3. Fonctionnement normal du site WEBDEV.




**Remarque** : la même configuration peut être adoptée avec un serveur Linux.

## 3. Installation sur un serveur Windows ou Linux avec Load Balancer
<a name="3_installation_sur_serveur_windows_linux_avec_load_balancer_ELTTEXTE000193"></a>
Le système Load Balancer permet de répartir la charge de connexion sur un site dynamique WEBDEV sur différents postes.
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P8-Installation%20sur%20un%20serveur%20Windows%20ou%20Linux.gif)

<a name="NOTE3_2"></a>


### Solution 1 : Installation du site dynamique WEBDEV (Session ou AWP) uniquement sur le serveur de données
<a name="solution_1_installation_site_dynamique_webdev_session_awp_uniquement_sur_serveur_donnees_ELTPARAGRAPHE000044"></a>Cette solution consiste à installer :

- sur chaque serveur Web, le serveur d'application WEBDEV,

- sur le serveur de données, les sites dynamiques WEBDEV et les fichiers de données.



<a name="NOTE3_3"></a>


### Solution 2 : Installation du site dynamique WEBDEV (Session ou AWP) sur tous les serveurs Web
<a name="solution_2_installation_site_dynamique_webdev_session_awp_sur_tous_les_serveurs_web_ELTPARAGRAPHE000056"></a>Cette solution consiste à installer :

- sur chaque serveur Web, le serveur d'application WEBDEV et les sites dynamiques WEBDEV.

- sur le serveur de données, les fichiers de données des sites dynamiques WEBDEV.

## 4. Installation sur un cluster WEBDEV
<a name="4_installation_sur_cluster_webdev_ELTTEXTE000229"></a>
Le système d'un Cluster WEBDEV permet de répartir la charge de connexion sur un site dynamique WEBDEV sur différents postes Serveur présents dans un cluster WEBDEV.

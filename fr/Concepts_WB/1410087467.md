<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Plusieurs outils sont disponibles pour tester un site WEBDEV :

- l'éditeur principal de WEBDEV.

- l'administrateur WEBDEV.
	Ce module est indispensable à l'exécution d'un site dynamique WEBDEV (en mode Session ou en mode AWP). Sur le poste de développement, il est possible de tester les sites dynamiques depuis une page de test créée par l'administrateur.

- WDTestSite, l'outil de test de montée en charge.




Le tableau ci-dessous présente quand et comment utiliser ces différents modes de test.

| Mode de test | Type de test | Lancement du test |
| --- | --- | --- |
| Editeur | Le lancement du test depuis l'éditeur permet par exemple de tester :<br><br>- les fonctionnalités du site,<br><br>- le code utilisé dans les différents événements et traitements. Un puissant débogueur permet de suivre pas à pas l'exécution des différents événements et traitements.<br><br>- l'utilisation du site avec différents navigateurs (installés sur le poste de développement).<br><br><br><br><br>Ce type de test : <br><br>- permet l'utilisation du débogueur.<br><br>- est pris en compte par le "Code Coverage".<br><br><br> | <br><br>- Sous le volet "Projet", dans le groupe "Mode test", déroulez "Mode test" et sélectionnez "Déboguer le projet".<br><br>- Sous le volet "Projet", dans le groupe "Mode test", déroulez "Mode test" et sélectionnez "Déboguer le projet depuis la page d'accueil".<br><br><br><br><br>Les droits de l'internaute (droits d'accès, d'écriture et de lecture) correspondent aux droits du développeur. |
| Administrateur WEBDEV<br><br>(site dynamique WEBDEV uniquement) | Le lancement du test depuis l'administrateur permet par exemple de tester :<br><br>- les fonctionnalités du site,<br><br>- les fonctions spécifiques au Web (cookies, ...),<br><br>- l'utilisation du site dans des conditions réelles.<br><br><br> | Lancez l'administrateur WEBDEV (menu "Démarrer") puis cliquez sur le bouton "Page de test" (onglet "Avancé").<br><br>Ce test permet d'utiliser le site WEBDEV dans des conditions identiques à celles d'un internaute.<br><br>Les droits de l'internaute (accès, écriture et lecture) correspondent aux droits de l'utilisateur par défaut défini dans le serveur Web utilisé. |
| WDTestSite<br><br>(site dynamique WEBDEV uniquement) | WDTestSite permet de réaliser des tests de montée en charge.<br><br>WDTestSite permet de lancer plusieurs connexions simultanées à un site WEBDEV.<br><br>Chaque connexion effectue une succession d'actions dans le site WEBDEV (scénario prédéfini). <br><br>Ce test doit être effectué sur un site WEBDEV installé sur un poste de déploiement, prêt à être mis en service. | <br><br>1. Créez un scénario de test (avec WDTestSite).<br><br>2. Installez WDTestSite et le scénario sur différents postes pour simuler un grand nombre d'accès.<br><br>3. Spécifiez le nombre de connexions à effectuer.<br><br>4. Lancez le scénario de test sur chaque poste.<br><br><br><br><br>Remarques :<br><br>- Chaque poste effectue un nombre de connexions défini. <br><br>- Chaque connexion effectue le scénario sélectionné.<br><br><br> |
| Tests automatiques | Les tests automatiques permettent de tester les procédures et les classes d'un site à différents niveaux de développement.<br><br>Les scénarios de test sont générés en WLangage à partir du test de la procédure ou de la classe. Les scénarios peuvent être modifiés sous l'éditeur de code.<br><br>Ce type de test est pris en compte par le "Code Coverage" . | <br><br>1. Affichez la procédure ou la classe dans le volet "Explorateur de projet".<br><br>2. Affichez le menu contextuel de l'élément (clic droit) et sélectionnez l'option "Créer un test unitaire".<br><br><br> |


## Administrateur SaaS - Tableau de bord
			



## Présentation
<a name="presentation_ELTTEXTE000196"></a>
Le tableau de bord de l'Administrateur SaaS permet de visualiser rapidement les principales informations concernant les sites SaaS d'un Serveur d'Application WEBDEV.

Il est constitué des éléments suivants :

- Les liens d'administration : ces liens proposent les actions les plus fréquentes.

- Un graphe affichant les 5 sites SaaS ayant le plus grand nombre de licences affectées.

- La liste des sites SaaS. 








## Fonctionnement du modèle SaaS
<a name="fonctionnement_modele_saas_ELTTEXTE000220"></a>
Le schéma ci-dessous présente les six principaux éléments en jeu dans le modèle SaaS :
<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=SaaS_Fonctionnement.gif)


Ces éléments sont :

| Elément | Rôle |
| --- | --- |
| *Site SaaS* | Le **Site SaaS** est un site WEBDEV déployé sur un Serveur d'Application WEBDEV. Il est associé à un Webservice SaaS permettant le contrôle de ce site. |
| *Service* | Les **Services** sont les différentes fonctionnalités d'un site. |
| *Compte Client* | Un **Compte Client** représente une société utilisatrice du **Site SaaS**.<br>Chaque **Compte Client** dispose de sa propre base de données et de sa propre personnalisation du **Site SaaS** (logo, graphismes, etc.). |
| *Utilisateur* | Les **Utilisateurs** sont les membres de la société représentée par le **Compte Client**.<br>Les utilisateurs sont les personnes physiques qui utilisent le **Site SaaS**. |
| *Tarification* | Une **Tarification** représente un ensemble de **Services** accessibles par un **Utilisateur**.<br>Une **Tarification** peut inclure un ou plusieurs **Services**.<br>Un **Service** peut être proposé dans une ou plusieurs **Tarifications**. |
| *Abonnement* | Un **Abonnement** représente la mise à disposition d'un certain nombre de licences pour une **Tarification** (et donc des **Services** associés) pour un **Compte Client**.<br>Ces licences doivent ensuite être réparties entre les **Utilisateurs**.<br>Les **Abonnements** servent à réaliser le suivi de l'utilisation du **Site SaaS** et la facturation. |









## Menu principal de l'Administrateur SaaS
<a name="menu_principal_administrateur_saas_ELTTEXTE000244"></a>
Le menu principal donne accès à l'ensemble des opérations d'administration :

- Tableau de bord : Retour à la vision synthétique des sites SaaS du Serveur d'Application WEBDEV.

- Sites

	- Ajouter un site : Permet de déclarer un nouveau site SaaS.

	- Gérer un site : Permet de définir les Services, les Tarifications et les paramètres d'un site SaaS.




- Clients

	- Ajouter un compte client : Permet de déclarer un nouveau Compte Client Saas.

	- Gérer un compte client : Permet de manipuler les utilisateurs associés au compte client ainsi que les Abonnements.




- Relevés : Permet de consulter les statistiques d'utilisation des sites SaaS. 










## Utilisation de l'Administrateur SaaS
<a name="utilisation_administrateur_saas_ELTTEXTE000268"></a>
Afin de pouvoir exploiter un site SaaS, les opérations suivantes sont nécessaires :

1. Déploiement du site SaaS sur un Serveur d'Application WEBDEV. 
	Pour plus de détails sur cette opération, consultez : [Déploiement d'un site](../Editeurs/2028039.md).
	Lors du déploiement, il est possible de préciser l'adresse du Webservice SaaS qui prendra en charge le site.
	Le Webservice SaaS peut être situé sur le même Serveur d'Application que le site (cas par défaut) ou sur un Serveur d'Application différent.
	Les opérations de paramétrage de SaaS se feront ensuite à l'aide de l'Administrateur SaaS situé sur le même Serveur d'Application que le Webservice.

2. Déclaration du site dans l'Administrateur SaaS. 
	Pour plus de détails, consultez [Ajout d'un site SaaS dans l'Administrateur](../SaaSAdminWeb/1600002.md).

3. Déclaration des services proposés par le site dans l'Administrateur SaaS. 
	Pour plus de détails, consultez [Gestion d'un site SaaS dans l'Administrateur](../SaaSAdminWeb/1600003.md).

4. Création des tarifs disponibles pour le site dans l'Administrateur SaaS. 
	Pour plus de détails, consultez [Gestion d'un site SaaS dans l'Administrateur](../SaaSAdminWeb/1600003.md).

5. Ajout de comptes clients dans l'Administrateur SaaS. 
	Pour plus de détails, consultez [Ajout d'un compte client SaaS](../SaaSAdminWeb/1600004.md).

6. Ajout d'un ou plusieurs utilisateurs pour les comptes clients SaaS. 
	Pour plus de détails, consultez [Gestion des comptes clients SaaS](../SaaSAdminWeb/1600005.md).

7. Abonnement des comptes clients SaaS aux tarifications du site SaaS et allocation de licences. 
	Pour plus de détails, consultez [Gestion des Abonnements SaaS](../SaaSAdminWeb/1600006.md).




Remarque : Si vous proposez l'envoi du code de sécurité à l'utilisateur (en cas d'oubli du mot de passe), il est nécessaire de paramétrer le serveur d'email SMTP associé à l'administrateur SaaS. Cette configuration peut être effectuée : 

- par programmation, via la fonction [SaaSAdminConfigureSMTP](../WDLang3/1000022592.md). 

- dans l'administrateur SaaS, via le lien "Configuration SMTP" présent dans le tableau de bord.

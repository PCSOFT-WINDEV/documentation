


## Administrateur SaaS - Gérer un site
			





## Présentation
<a name="presentation_ELTTEXTE000135"></a>
La gestion d'un site SaaS permet de modifier les trois principales caractéristiques du site :

- Les Services SaaS utilisés pour définir les permissions d'accès aux différentes parties du site.

- Les Tarifications permettant de regrouper un ou plusieurs services en vue de les fournir à un compte client.

- Les paramètres du site qui permettent la personnalisation du site en fonction du client.






## Gestion des Services SaaS
<a name="gestion_des_services_saas_ELTTEXTE000159"></a>
Les Services SaaS définissent les blocs fonctionnels du site. Lors de l'utilisation d'un site par un client, les Services SaaS permettent de limiter l'accès à certaines parties du site ou à certaines fonctionnalités en fonction de l'identité du client. 

Cette limitation doit être mise en place dans le code en utilisant la fonction [SaaSVérifieService](../WDLang3/1000018961.md).

Un Service SaaS est simplement défini par son nom. Pour définir un nouveau Service SaaS, il suffit de cliquer sur le lien "Ajouter un service".

Il n'y a pas de limite au nombre de Services SaaS que peut supporter un site SaaS.

Voici quelques exemples de définitions de Services SaaS pour un site de petites annonces :

- Accès au site / Lecture des annonces

- Rédaction d'annonces textes

- Rédaction d'annonces avec images

- Modération des annonces






## Gestion des Tarifications
<a name="gestion_des_tarifications_ELTTEXTE000183"></a>
Une Tarification permet de regrouper un ensemble de Services SaaS et de leur associer un prix afin de les mettre à disposition des utilisateurs du site SaaS (au travers d'un Abonnement).


### Création d'une nouvelle tarification
<a name="creation_une_nouvelle_tarification_ELTPARAGRAPHE000039"></a>Pour créer une Tarification, il suffit de cliquer sur le lien "Ajouter une tarification". Une nouvelle Tarification (dont le nom par défaut est "Nouveau") est alors ajoutée dans la liste des Tarifications du site.


### Edition des propriétés d'une tarification
<a name="edition_des_proprietes_une_tarification_ELTPARAGRAPHE000044"></a>Pour éditer les paramètres de la Tarification, il faut cliquer sur son nom. Une boîte de dialogue s'ouvre alors permettant de saisir :

- Le nom de la tarification,

- Son prix (la devise utilisée par les sites SaaS peut être définie dans la page "Tableau de Bord").

- La périodicité du prix : par mois, par semaine, par an ou illimité (si le prix est payable une seule fois).

- Le mode de facturation : par utilisateur ou forfaitairement (quel que soit le nombre d'utilisateurs).





### Associer une tarification et des services
<a name="associer_une_tarification_des_services_ELTPARAGRAPHE000054"></a>Pour utiliser une tarification, il est nécessaire de l'associer à un ou plusieurs services. Pour cela, il suffit de cocher dans la table des tarifications les lignes correspondant aux services à relier à la tarification puis de cliquer sur le bouton "Valider".

**Remarque** : L'administrateur SaaS affiche les différentes tarifications associées à un site. Si le site comprend plus de 5 tarifications, une flèche permet d'accéder aux tarifications suivantes. Il est également possible de proposer un plus grand nombre de tarifications par programmation en utilisant la fonction [SaaSAdminAjouteTarification](../WDLang3/1000019127.md).



## Gestion des paramètres du site
<a name="gestion_des_parametres_site_ELTTEXTE000225"></a>
Les paramètres d'un site SaaS sont une série de valeurs déterminées par le développeur du site pour permettre sa personnalisation en fonction des clients.

Par défaut, à chaque site déclaré dans l'Administrateur SaaS est associé un paramètre nommé "Nom du client" permettant de personnaliser l'affichage du site avec le nom du client en cours.

En programmation, la valeur d'un paramètre pour le client en cours peut être relue grâce à la fonction [SaaSLitParamètreSite](../WDLang3/1000019037.md).

Il est possible de définir la liste des paramètres d'un site en cliquant sur le lien "Définir les paramètres du site".

A chaque paramètre est associé une valeur par défaut qui sera utilisée pour les comptes clients du site. La valeur par défaut pourra être modifiée pour chaque compte client via la fonction [SaaSEcritParamètreSite](../WDLang3/1000019039.md).

**Note** : Si tous les utilisateurs du compte client peuvent lire la valeur, il faut être un utilisateur avec droit d'administration pour écrire.

Quelques exemples de paramètres :

- Nom du client

- Logo

- Images

- Couleurs

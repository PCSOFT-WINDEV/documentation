


## FenêtreSiNouveau (Propriété)

***En anglais : WindowIfNew***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **FenêtreSiNouveau** permet de connaître et de modifier le nom de la fenêtre interne à ouvrir si l'utilisateur final ouvre un nouveau volet dans un champ Onglet dynamique. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ONG_OngletDyn.FenêtreSiNouveau = "FI_Document"
ONG_OngletDyn.LibelléSiNouveau = "Nouvel onglet"
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Connaître le nom de la fenêtre interne utilisée pour un nouveau volet dynamique

`<Résultat> = <Champ Onglet>.FenêtreSiNouveau`
---

**`<Résultat> : (Chaîne de caractères)`**

Nom de la fenêtre interne utilisée lors de la création d'un nouveau volet dans un onglet dynamique. 

**`<Champ Onglet> : (Nom de champ)`**

Nom du champ Onglet à manipuler. 


<a name="SYNTAXE2"></a>

### Modifier le nom de la fenêtre interne utilisée pour un nouveau volet dynamique

`<Champ Onglet>.FenêtreSiNouveau = <Nouvelle fenêtre interne>`
---

**`<Champ Onglet> : (Nom de champ)`**

Nom du champ Onglet à manipuler. 

**`<Nouvelle fenêtre interne> : (Chaîne de caractères)`**

Nom de la fenêtre interne à utiliser lors de la création d'un nouveau volet dans un onglet dynamique. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le libellé du nouveau volet créé est défini : 

	- soit dans la fenêtre de description du champ (onglet "Détail"). 

	- soit par programmation grâce à la propriété [LibelléSiNouveau](../Proprietes/1000021380.md). 




- Si la fenêtre interne attend des paramètres, dans l'événement "Création d'un volet", utilisez la fonction [OngletOuvre](../WDLang1/1000021369.md) pour définir la fenêtre interne à ouvrir.

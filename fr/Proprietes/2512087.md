


## SensiblePonctuation (Propriété)

***En anglais : PunctuationSensitive***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **SensiblePonctuation** permet de paramétrer la prise en compte de la ponctuation (virgule, point, etc.) dans les différentes recherches effectuées sur la rubrique (avec les fonctions [HLitRecherche](../WDLang4/3044050.md), [HLitRecherchePremier](../WDLang4/3044036.md) et [HLitRechercheDernier](../WDLang4/3044034.md) par exemple). Il est ainsi possible de :

- Savoir si la ponctuation est prise en compte ou non dans les recherches sur cette rubrique (définie sous l'éditeur d'analyses ou définie par programmation).

- Définir la gestion de la ponctuation pour une rubrique (lors de la description de la rubrique par programmation uniquement).





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Définir la gestion de la ponctuation de la rubrique Commande.Nom
Nom.SensiblePonctuation = Vrai
...
// Valide la description du fichier de données
HDécritFichier(Commande)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Savoir si la ponctuation est prise en compte

`<Résultat> = <Fichier de données>.<Rubrique>.SensiblePonctuation`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la ponctuation est prise en compte pour les recherches effectuées sur cette rubrique, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique manipulée. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Rubrique](../WDLang4/1514071.md).  


<a name="SYNTAXE2"></a>

### Définir la gestion de la ponctuation (rubrique décrite par programmation)

`<Rubrique>.SensiblePonctuation = <Nouvelle gestion>`
---

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique manipulée. Ce nom vient d'être défini grâce au type [Description de Rubrique](../WDLang4/1514071.md).

**`<Nouvelle gestion> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la ponctuation doit être prise en compte pour les recherches effectuées sur cette rubrique, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Recherche réalisée sur une rubrique "sensible à la ponctuation"
<a name="recherche_realisee_sur_une_rubrique_sensible_ponctuation_ELTPARAGRAPHE000099"></a>Si une recherche est réalisée sur une rubrique "sensible à la ponctuation", seuls les mots identiques du point de vue de la ponctuation seront trouvés. Ainsi, si le nom de client enregistré est "M.Jérémy", la recherche effectuée sur :

- le nom de client "M.Jérémy" trouvera l'enregistrement,

- le nom de client "M Jérémy" ne trouvera pas l'enregistrement.



<a name="NOTE0_2"></a>


### Paramétrer les différentes options de recherche pour une rubrique
<a name="parametrer_les_differentes_options_recherche_pour_une_rubrique_ELTPARAGRAPHE000109"></a>

- Pour gérer l'accentuation lors des recherches, utilisez la propriété [SensibleAccentuation](../Proprietes/2512077.md).

- Pour gérer la casse lors des recherches, utilisez la propriété [SensibleCasse](../Proprietes/2512085.md).

- Pour gérer la ponctuation lors des recherches, utilisez la propriété **SensiblePonctuation**.


## Détaché (Propriété)

***En anglais : Undocked***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **Détaché** permet de savoir si un volet d'un champ Onglet ou d'un champ Ruban est détaché. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
//  Si le volet 2 du champ Ruban est détaché alors on rend actif le volet suivant
SI RUBAN_Menu[2].Détaché ALORS
	RUBAN_Menu = 3
SINON
	RUBAN_Menu = 2
FIN

// Savoir si un volet d'un champ Onglet est détaché
POUR i = 1 A ONG_Choix.Occurrence
      SI ONG_Choix[i].Détaché ALORS RENVOYER Vrai
FIN
RENVOYER Faux
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Savoir si un volet est détaché

`<Résultat> = <Champ manipulé>.Détaché`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le volet est détaché, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 




**`<Champ manipulé> : (Nom de champ)`**

Nom volet du champ Onglet ou Ruban à manipuler. La notation suivante doit être utilisée : 

```txt
<Nom du champ>[Indice de l'élément]
```
Par exemple : 
```txt
RUBAN_Menu[2]
```




<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Un volet est détaché s'il est dans une fenêtre libre à coté de la fenêtre où se trouve le champ.

- Pour pouvoir détacher un volet du champ Onglet, ce volet doit être dynamique. 

- Pour pouvoir détacher un volet d'un champ Ruban, l'option "Volets détachables" doit être cochée dans la fenêtre de description du champ.

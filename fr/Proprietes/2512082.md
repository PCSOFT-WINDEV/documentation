


## Sécurisé (Propriété)

***En anglais : Secure***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **Sécurisé** permet de paramétrer le niveau de sécurité du cryptage des fichiers de données. Le niveau de sécurité peut être :

- défini par défaut,

- élevé. Attention : dans ce cas, il n'existera aucun moyen de retrouver le mot de passe du fichier de données.




Il est ainsi possible de :

- Connaître le niveau de sécurité du cryptage (fichier de données défini sous l'éditeur d'analyses ou défini par programmation).

- Définir le niveau de sécurité du cryptage (lors de la description du fichier de données par programmation uniquement).







<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Définir le niveau de sécurité du cryptage du fichier de données Commande
Commande.Sécurisé = Vrai
...
// Valide la description du fichier de données
HDécritFichier(Commande)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le niveau de sécurité du cryptage

`<Niveau de sécurité> = <Fichier de données>.Sécurisé`
---

**`<Niveau de sécurité> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le niveau de sécurité est maximal, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).  


<a name="SYNTAXE2"></a>

### Définir le niveau de sécurité du cryptage (fichier décrit par programmation)

`<Fichier de données>.Sécurisé = <Niveau de sécurité>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Niveau de sécurité> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le niveau de sécurité doit être maximal, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Cette propriété correspond à l'option "Activer la sécurité renforcée " disponible dans la fenêtre de description du fichier de données sous l'éditeur d'analyse (onglet "Détail").

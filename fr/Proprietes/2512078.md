


## Répertoire (Propriété)

***En anglais : Directory***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **Répertoire** permet de gérer le répertoire physique des fichiers de données HFSQL. Cette propriété tient compte des éventuels changements de répertoires réalisés grâce aux fonctions [HChangeRep](../WDLang4/3044168.md) ou [HSubstRep](../WDLang4/3044028.md). Il est ainsi possible de :

- Connaître le répertoire physique d'un fichier de données défini sous l'éditeur d'analyses ou défini par programmation.

- Définir un répertoire physique pour un fichier de données (lors de la description du fichier par programmation uniquement).













<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connaître le nom et le chemin complet du fichier de données Client
NomCheminCompletFichier est une Chaîne
NomCheminCompletFichier = Client.Répertoire + "\" + ...
	Client.NomPhysique + Client.Extension
// NomCheminCompletFichier correspond à "C:\MesFichiers\Client.Fic"
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le répertoire physique d'un fichier de données 

`<Répertoire actuel> = <Fichier de données>.Répertoire`
---

**`<Répertoire actuel> : (Chaîne de caractères)`**

Nom du répertoire du fichier de données. Ce nom tient compte des éventuels changements de répertoires réalisés grâce aux fonctions [HChangeRep](../WDLang4/3044168.md) ou [HSubstRep](../WDLang4/3044028.md).

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).  


<a name="SYNTAXE2"></a>

### Définir un répertoire physique pour un fichier de données défini par programmation

`<Fichier de données>.Répertoire = <Nouveau répertoire>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Nouveau répertoire> : (Chaîne de caractères)`**

Nom physique du répertoire.

![HFSQL Client/Serveur](https://doc.pcsoft.fr/ext/images/fr/HFCS.png) Le répertoire du fichier de données doit être relatif à la base de données présente sur le serveur. Par exemple : ".\\Donnees"

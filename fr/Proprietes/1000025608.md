
## FenêtreInternePopup (Propriété)

***En anglais : InternalWindowPopup***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **FenêtreInternePopup** permet de connaître et de modifier le nom de la fenêtre interne utilisée pour personnaliser le contenu de la popup affichée pour les marqueurs du champ Carte. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
CARTE_Carte.FenêtreInternePopup = "FI_Popup"
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître la fenêtre interne utilisée pour personnaliser la popup affichée pour les marqueurs du champ Carte

`<Résultat> = <Champ Carte>.FenêtreInternePopup`
---

**`<Résultat> : (Chaîne de caractères)`**



- Nom de la fenêtre interne utilisée pour la popup,

- Chaîne vide ("") si la popup utilise un affichage par défaut.




**`<Champ Carte> : (Nom de champ)`**

Nom du champ Carte à manipuler.   


<a name="SYNTAXE2"></a>

### Modifier la fenêtre interne utilisée pour personnaliser la popup affichée pour les marqueurs du champ Carte

`<Champ Carte>.FenêtreInternePopup = <Nouvelle fenêtre interne>`
---

**`<Champ Carte> : (Nom de champ)`**

Nom du champ Carte à manipuler.

**`<Nouvelle fenêtre interne> : (Chaîne de caractères)`**



- Nom de la fenêtre interne utilisée pour la popup,

- Chaîne vide ("") si la popup utilise un affichage par défaut.










<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si un nom de fenêtre interne a été renseigné pour la propriété **FenêtreInternePopup**, la fenêtre interne sera chargée à chaque affichage d'une popup dans le champ Carte. La variable [Marqueur](../WDLang3/1000019940.md) correspondant à la popup affichée est passée à l'événement "Déclarations globales" de la fenêtre interne à ouvrir. Ce paramètre est passé par référence et il est considéré comme une variable globale à la fenêtre.

- La fenêtre interne spécifiée est ouverte en mode "affichage uniquement" : aucune interaction avec l'utilisateur n'est possible dans les champs de la fenêtre interne.

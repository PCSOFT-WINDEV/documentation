


## MémoriserLaValeur (Propriété)

***En anglais : StoreTheValue***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **MémoriserLaValeur** permet de :

- Savoir si un champ conserve sa valeur entre deux lancements de l'application via la persistance. 

- Modifier un champ pour qu'il conserve ou non sa valeur entre deux lancements de l'application via la persistance. 




Cette propriété correspond à l'option "Mémoriser la valeur" disponible dans l'onglet "Détail" de la fenêtre de description des champs : 

- Interrupteur / Sélecteur, 

- Combo / Liste,

- Champ de saisie,

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Onglet statique / Onglet dynamique.



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
c est un Champ
c <- ChampCrée("SAI_Date_1", typDate, 0, 0, 100, 20)
c.Libellé = "Date du jour"
c.MémoriserLaValeur = Vrai
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Savoir si un champ conserve sa valeur entre deux lancements de l'application

`<Résultat> = <Champ manipulé>.MémoriserLaValeur`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le champ conserve sa valeur entre deux lancements de l'application, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 




**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler. Ce champ peut être : 

- un champ Interrupteur ou Sélecteur, 

- un champ Combo ou Liste,

- un champ de saisie,

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) un champ Onglet statique ou Onglet dynamique.





<a name="SYNTAXE2"></a>

### Modifier un champ pour qu'il conserve ou non sa valeur entre deux lancements de l'application

`<Champ manipulé>.MémoriserLaValeur = <Nouvelle gestion>`
---

**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler. Ce champ peut être : 

- un champ Interrupteur ou Sélecteur, 

- un champ Combo ou Liste,

- un champ de saisie,

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) un champ Onglet statique ou Onglet dynamique.




**`<Nouvelle gestion> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le champ doit conserver sa valeur entre deux lancements de l'application, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si cette propriété est mise à <u><u><u><u>Vrai</u></u></u></u> après la phase d'initialisation, le champ ne reprendra pas la valeur précédente.

- Cette propriété est surtout utile pour paramétrer les champs créés via les fonctions [ChampCrée](../WDLang1/1000020876.md) et [ChampClone](../WDLang1/3025017.md). 

- Cette propriété correspond à l'utilisation de l'option "Mémoriser la valeur" du menu contextuel des FAA (Fonctionnalités Automatiques de vos Applications). Lorsqu'un champ est créé avec la fonction [ChampCrée](../WDLang1/1000020876.md), l'utilisateur final dispose du menu des FAA : il peut ajouter a posteriori la gestion de la persistance (sauf si l'option est désactivée avec la fonction [FAADésactive](../WDLang1/1000022018.md).


## NbCarteMax (Propriété)

***En anglais : MaxNbCard***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **NbCarteMax** permet de connaître et de modifier le nombre maximum de cartes présentes dans un champ Kanban. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Nombre maximum de cartes dans la liste LISTE_A_Valider
// Il sera impossible de dropper une carte supplémentaire dans la liste
// lorsque ce nombre est atteint. 
LISTE_A_Valider.NbCarteMax = 2
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Connaître le nombre maximal de cartes autorisées

`<Résultat> = <Liste de Kanban>.NbCarteMax`
---

**`<Résultat> : (Entier)`**



- Nombre maximal de cartes pouvant être affichées dans la liste de Kanban,  

- 0, aucune carte ne peut être ajoutée dans la liste de Kanban, 

- -1, le nombre de cartes pouvant être affichées est illimité. 




**`<Liste de Kanban> : (Nom de champ)`**

Champ Liste de Kanban à manipuler. 


<a name="SYNTAXE2"></a>

### Modifier le nombre maximal de cartes autorisées

`<Liste de Kanban>.NbCarteMax = <Valeur maximale>`
---

**`<Liste de Kanban> : (Nom de champ)`**

Champ Liste de Kanban à manipuler. 

**`<Valeur maximale> : (Entier)`**



- Nombre maximal de cartes pouvant être affichées dans la liste de Kanban,  

- 0, aucune carte ne peut être ajoutée dans la liste de Kanban, 

- -1, le nombre de cartes pouvant être affichées est illimité. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Si la liste de Kanban atteint le nombre maximum de cartes autorisées, il n'est plus possible de "dropper" des cartes dans cette liste de Kanban. Le pied de liste (s'il existe) est alors grisé.

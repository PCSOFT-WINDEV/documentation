


## RépertoireJournal (Propriété)

***En anglais : LogDirectory***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **RépertoireJournal** permet de gérer le répertoire du fichier journal décrit dans l'analyse. Il est ainsi possible de :

- Connaître le répertoire du journal pour un fichier de données défini sous l'éditeur d'analyses ou défini par programmation.

- Définir le répertoire du journal pour un fichier de données (lors de la description du fichier de données par programmation uniquement).



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Définir le répertoire journal du fichier de données Commande défini par programmation
Commande.RépertoireJournal = "C:\MesFichiers\MesJournaux"
...
// Valide la description du fichier de données
HDécritFichier(Commande)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le répertoire du journal d'un fichier de données

`<Nom actuel> = <Fichier de données>.RépertoireJournal`
---

**`<Nom actuel> : (Chaîne de caractères)`**



- Nom du répertoire du fichier journal associé au fichier de données, 

- "." si le répertoire correspond au répertoire des fichiers de données en cours.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).  


<a name="SYNTAXE2"></a>

### Définir un répertoire de journal pour un fichier défini par programmation

`<Fichier de données>.RépertoireJournal = <Nouveau nom>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Nouveau nom> : (Chaîne de caractères)`**

Nom du répertoire du journal.




## LibelléSiNouveau (Propriété)

***En anglais : CaptionIfNew***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **LibelléSiNouveau** permet de connaître et de modifier le libellé du nouveau volet ouvert par l'utilisateur final dans un champ Onglet dynamique. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ONG_OngletDyn.FenêtreSiNouveau = "FI_Document"
ONG_OngletDyn.LibelléSiNouveau = "Nouvel onglet"
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Connaître le libellé du nouveau volet dynamique

`<Résultat> = <Champ Onglet>.LibelléSiNouveau`
---

**`<Résultat> : (Chaîne de caractères)`**

Libellé utilisé lors de la création d'un nouveau volet dans un onglet dynamique. 

**`<Champ Onglet> : (Nom de champ)`**

Nom du champ Onglet à manipuler. 


<a name="SYNTAXE2"></a>

### Modifier le libellé du nouveau volet dynamique

`<Champ Onglet>.LibelléSiNouveau = <Nouveau libellé>`
---

**`<Champ Onglet> : (Nom de champ)`**

Nom du champ Onglet à manipuler. 

**`<Nouveau libellé> : (Chaîne de caractères)`**

Nouveau libellé utilisé lors de la création d'un nouveau volet dans un onglet dynamique. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le nom de la fenêtre interne à utiliser pour le nouveau volet d'onglet est défini : 

	- soit dans la fenêtre de description du champ (onglet "Détail"). 

	- soit par programmation grâce à la propriété [FenêtreSiNouveau](../Proprietes/1000021381.md). 




- Si la fenêtre interne attend des paramètres, dans l'événement "Création d'un volet", utilisez la fonction [OngletOuvre](../WDLang1/1000021369.md) pour définir le libellé.


## AfficheOrphelin (Propriété)

***En anglais : DisplayOrphan***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **AfficheOrphelin** permet de : 

- Savoir si une ligne ou une colonne d'un champ Tableau croisé dynamique est affichée lorsqu'elle n'a pas de parent. 

- Modifier le mode d'affichage d'une ligne ou d'une colonne d'un champ Tableau croisé dynamique lorsqu'elle n'a pas de parent. 




Remarque : Si l'intégrité de la base de données correspond à celle définie dans l'analyse, cette propriété est inutile. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// TCD_Vente_Produit est un tableau croisé dynamique 
// affichant les ventes de produits par an. 
// Les produits sont regroupés par catégorie. 

// Affiche uniquement les produits associés à une catégorie
TCD_Vente_Produit.COL_Produit.AfficheOrphelin = aoJamais 
TCDCalculeTout(TCD_Vente_Produit)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Connaître le mode d'affichage actuel d'une ligne ou colonne

`<Résultat> = <Ligne ou colonne>.AfficheOrphelin`
---

**`<Résultat> : (Constante de type Entier)`**

Mode d'affichage des éléments orphelins : 


|   |   |
| --- | --- |
| *aoAuto*<br>(Valeur par défaut) | Le mode d'affichage est automatique en fonction de la cardinalité de la liaison définie dans l'analyse. |
| *aoJamais* | Seuls les éléments ayant un parent sont affichés. Par exemple, seuls les produits associés à une catégorie seront affichés. |
| *aoToujours* | Tous les éléments seront affichés, même ceux n'ayant pas de parents. Par exemple, tous les produits seront affichés, même ceux associés à aucune catégorie. |





**`<Ligne ou colonne> : (Nom de champ)`**

Nom de la ligne ou de la colonne du champ Tableau croisé dynamique. 


<a name="SYNTAXE2"></a>

### Modifier le mode d'affichage d'une ligne ou colonne

`<Ligne ou colonne>.AfficheOrphelin = <Nouveau mode d'affichage>`
---

**`<Ligne ou colonne> : (Nom de champ)`**

Nom de la ligne ou de la colonne du champ Tableau croisé dynamique à manipuler. 

**`<Nouveau mode d'affichage> : (Constante de type Entier)`**

Mode d'affichage des éléments orphelins : 


|   |   |
| --- | --- |
| *aoAuto* | Le mode d'affichage est automatique en fonction de la cardinalité de la liaison définie dans l'analyse. |
| *aoJamais* | Seuls les éléments ayant un parent sont affichés. Par exemple, seuls les produits associés à une catégorie seront affichés. |
| *aoToujours* | Tous les éléments seront affichés, même ceux n'ayant pas de parents. Par exemple, tous les produits seront affichés, même ceux associés à aucune catégorie. |

<a name="NOTE0_1"></a>

## Remarques


### Cas particulier
<a name="cas_particulier_ELTPARAGRAPHE000077"></a>Pour modifier le mode d'affichage des éléments orphelins, la propriété **AfficheOrphelin** doit être utilisée avant le calcul du contenu du champ Tableau croisé dynamique (fonction [TCDCalculeTout](../WDLang1/1000020539.md)).  


### Exemple
<a name="exemple_ELTPARAGRAPHE000088"></a>Le champ Tableau croisé dynamique affiché contient des produits non reliés à des catégories : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=TCD_Orphelin1.gif)


Le code utilisé pour afficher ce champ Tableau croisé dynamique est le suivant : 


```wl
TCD_Vente.COL_NomProduit.AfficheOrphelin = aoToujours
TCDCalculeTout(TCD_Statistiques)
```


Le même champ Tableau croisé dynamique peut être affiché sans les produits non reliés à des catégories : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=TCD_Orphelin2.gif)


Le code suivant est alors utilisé : 


```wl
TCD_Vente.COL_NomProduit.AfficheOrphelin = aoJamais
TCDCalculeTout(TCD_Statistiques)
```




## NomPhysique (Propriété)

***En anglais : PhysicalName***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **NomPhysique** permet de gérer le nom physique des fichiers de données HFSQL. Cette propriété tient compte des éventuels changements de noms réalisés grâce à la fonction [HChangeNom](../WDLang4/3044141.md). Il est ainsi possible de :

- Connaître le nom physique d'un fichier de données défini sous l'éditeur d'analyses ou défini par programmation.

- Définir un nom physique pour un fichier de données (lors de la description du fichier de données par programmation uniquement).









<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connaître le nom physique du fichier de données Client
ResNomPhys = Client.NomPhysique
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le nom physique d'un fichier de données

`<Nom actuel> = <Fichier de données>.NomPhysique`
---

**`<Nom actuel> : (Chaîne de caractères)`**

Nom physique du fichier de données (sans extension). Ce nom tient compte des éventuels changements de noms (fonction [HChangeNom](../WDLang4/3044141.md)).

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).

### Définir un nom physique pour un fichier de données créé par programmation

`<Fichier de données>.NomPhysique = <Nouveau nom>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Nouveau nom> : (Chaîne de caractères)`**

Nom physique du fichier de données (sans extension).

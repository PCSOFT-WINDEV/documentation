


## HauteurMaxVignette (Propriété)

***En anglais : MaxThumbnailHeight***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **HauteurMaxVignette** permet de :

- connaître la hauteur maximale d'une vignette d'une rubrique de type Image (mémo binaire). Cette rubrique a été définie sous l'éditeur d'analyses ou définie par programmation.

- définir la hauteur maximale d'une vignette d'une rubrique de type Image (mémo binaire), lors de la description de la rubrique par programmation uniquement.









<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la rubrique Photo
Photo est une Description de Rubrique
...
// Définir la hauteur maximale de la première 
// vignette de la description de rubrique Photo
Photo.HauteurMaxVignette[1] = 50
// équivalent à Photo.HauteurMaxVignette = 50
...
// Valider la description du fichier de données
HDécritFichier(Client)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître la hauteur maximale d'une vignette d'une rubrique de type Image (mémo binaire)

`<Hauteur maximale actuelle> = <Fichier de données>.<Rubrique>.HauteurMaxVignette[<N>]`
---

**`<Hauteur maximale actuelle> : (Entier)`**

Hauteur maximale (en pixels) de la vignette spécifiée.

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique de type mémo binaire manipulée. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Rubrique](../WDLang4/1514071.md).

**`<N> : (Entier optionnel)`**

Numéro de la vignette dont on recherche la hauteur maximale. Si ce paramètre n'est pas spécifié, la hauteur maximale de la première vignette est retournée.  


<a name="SYNTAXE2"></a>

### Modifier la hauteur maximale d'une vignette d'une rubrique de type Image (mémo binaire)

`<Rubrique>.HauteurMaxVignette[<N>] = <Nouvelle hauteur maximale>`
---

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique de type mémo binaire manipulée. Ce nom vient d'être défini grâce au type [Description de Rubrique](../WDLang4/1514071.md).

**`<N> : (Entier optionnel)`**

Numéro de la vignette dont on veut modifier la hauteur maximale. Si ce paramètre n'est pas spécifié, la hauteur maximale de la première vignette sera modifiée.

**`<Nouvelle hauteur maximale> : (Entier)`**

Nouvelle hauteur maximale (en pixels) de la vignette spécifiée. Cette hauteur doit être comprise entre 1 et 65535.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La propriété **HauteurMaxVignette** est disponible :

- en HFSQL Classic : sur les rubriques d'un fichier de données, d'une vue ou d'une requête.

- en HFSQL Client/Serveur : sur les rubriques d'un fichier de données uniquement.




![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) La propriété **HauteurMaxVignette** est disponible uniquement en HFSQL Client/Serveur sur les rubriques d'un fichier de données.

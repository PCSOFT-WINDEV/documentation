


## LiaisonsDétailLigne (Propriété)

***En anglais : RowDetailsBinding***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **LiaisonsDétailLigne** permet de connaître et de modifier la liaison fichier utilisée pour afficher le détail d'une ligne dans un champ Table fichier. 

**Rappel** :  Le détail d'une ligne est une "sous-table" affichée en dessous de la ligne lorsque l'utilisateur clique sur [+] devant la ligne ou double-clique sur la ligne du champ Table. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Affiche les commandes lors du clic sur la ligne affichant le client
TABLE_CLIENT.LiaisonsDétailLigne = "IDCLIENT:COMMANDE.IDCLIENT"
```



```wl
//  Affiche le nom de sa ville lors du clic sur la ligne affichant le client
TABLE_CLIENT.LiaisonsDétailLigne = "CP:VILLE.CP:NOM"
```



```wl
// Affiche les commandes et la ville + département lors du clic sur la ligne affichant un client 
TABLE_CLIENT.LiaisonsDétailLigne = [
IDCLIENT:COMMANDE.IDCLIENT
CP:VILLE.CP:NOM,DEPARTEMENT
]
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Connaître la liaison fichier utilisée pour afficher le détail d'une ligne dans un champ Table fichier

`<Résultat> = <Champ Table fichier>.LiaisonsDétailLigne`
---

**`<Résultat> : (Constante ou chaîne de caractères)`**

Liaison actuelle : 



|   |   |
| --- | --- |
| *ldlAucune* | Aucune liaison n'a été définie. <br>Aucun détail n'est affiché dans le champ Table fichier lors du clic sur le "[+]" ou lors du double-clic sur la ligne. |
| *ldlAutomatique* | Les liaisons définies dans l'analyse sont utilisées pour trouver le ou les fichiers de données reliés à afficher. <br>Si plusieurs fichiers de données sont liés dans l'analyse, l'utilisateur pourra choisir le fichier à afficher dans le détail via un menu contextuel. |
| Chaîne de caractères | La liaison est personnalisée. La fenêtre de détail affiche les données spécifiées. <br>Le format de cette chaîne est le suivant : <br><br><pre><code>RUB:FICA.RUBA:RubAff1,RubAff2,RubAff3<RC><br>RUB2:FICB.RUBB</code></pre><br>où : <br><br>- RUB,RUB2 sont les rubriques du fichier de parcours principal du champ Table,<br><br>- FICA, FICB sont les fichiers de données reliés à afficher en détail,<br><br>- RUBA, RUBB sont les rubriques reliées aux rubriques RUB, RUB2 dans ces fichiers reliés,<br><br>- RubAff1, RubAff2, RubAff3 sont les rubrique de FICA à afficher dans le détail. Si cet élément est non renseigné, toutes les rubriques visibles de l'analyse seront affichées.<br><br><br><br><br>Par exemple, la ligne : <br><br><pre><code>RefCommande:REQ_LigneCommande.ParamRefCommande:VISUEL,REF,<br>LIBELLE,PRIXUNIT,QUANTITE,TOTALHT</code></pre><br> fait une liaison entre la rubrique RefCommande du champ Table et la rubrique ParamRefCommande de la requête REQ_LigneCommande. <br>Les rubriques affichées dans la fenêtre de détail sont les rubriques VISUEL, REF, LIBELLE, PRIXUNIT, QUANTITE, TOTALHT du résultat de la requête. |



**`<Champ Table fichier> : (Nom de champ)`**

Nom du champ Table fichier à manipuler.  


<a name="SYNTAXE2"></a>

---

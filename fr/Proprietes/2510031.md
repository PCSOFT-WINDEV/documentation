


## Détection (Propriété)

***En anglais : Detection***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **Détection** permet de connaître et de modifier la détection automatique des flux entrants. Si la détection de flux est activée, le flux permet la réception. 

Par défaut, la détection automatique des flux est active.

Remarque : Cette propriété est équivalente à l'option "Détection des flux entrants actif" dans la fenêtre de description des flux. Pour afficher la fenêtre de description des flux, sous le volet "Fenêtre", dans le groupe "Edition", déroulez "Autres actions" et sélectionnez "Editer les flux". Cette option est disponible si un champ Conférence est présent dans la fenêtre.<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Flux_Detection.gif)
A partir de la version 20, les champs Conférence sont supportés sous Windows Vista et supérieur. Si vous souhaitez utiliser le champ Conférence avec une version antérieure de Windows : 

1. Affichez la fenêtre de description du champ. 

2. Dans l'onglet "UI", cochez l'option "Utilisez les API TAPI 3.x / Windows XP (pour compatibilité)". 

3. Validez.



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Autorise la détection de flux entrants
MonFlux.Détection = Actif
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le mode de détection des flux entrants

`<Résultat> = <Nom du flux>.Détection`
---

**`<Résultat> : (Constante de type entier)`**

Mode de gestion des flux entrants actuellement utilisé :


|   |   |
| --- | --- |
| *Actif* | Détection automatique des flux entrants. La procédure de détection automatique des flux entrants est exécutée à chaque nouveau flux entrant. |
| *Inactif* | Aucune détection automatique des flux entrants. La procédure de détection automatique des flux entrants est ignorée à chaque nouveau flux entrant. |



**`<Nom du flux> : (Chaîne de caractères)`**

Nom du Flux ou nom du champ Conférence associé au flux.  


<a name="SYNTAXE2"></a>

### Modifier le mode de détection des flux entrants

`<Nom du flux>.Détection = <Nouveau mode>`
---

**`<Nom du flux> : (Chaîne de caractères)`**

Nom du Flux ou nom du champ Conférence associé au flux.

**`<Nouveau mode> : (Constante de type Entier)`**

Nouveau mode de gestion des flux entrants :


|   |   |
| --- | --- |
| *Actif* | Détection automatique des flux entrants. La procédure de détection automatique des flux entrants est exécutée à chaque nouveau flux entrant. |
| *Inactif* | Aucune détection automatique des flux entrants. La procédure de détection automatique des flux entrants est ignorée à chaque nouveau flux entrant. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Exemple d'utilisation
<a name="exemple_utilisation_ELTPARAGRAPHE000082"></a>Une application permet à l'utilisateur de ne pas être dérangé pendant une Visio-Conférence. Cette option utilise la propriété **Détection** pour bloquer la détection automatique des appels entrants.

Dans ce cas, tous les appels entrants sont ignorés. La procédure automatique de détection des appels entrants n'est pas exécutée.

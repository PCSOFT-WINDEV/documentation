


## GranularitéDurée (Propriété)

***En anglais : GranularityDuration***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **GranularitéDurée** permet de connaître et de modifier la taille de la grille utilisée pour le redimensionnement :

- des rendez-vous d'un champ Agenda.

- des rendez-vous d'un champ Planning.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) des événements d'un champ TimeLine.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) des tâches d'une colonne de type Diagramme de Gantt.




**Remarque** : Pour connaître et modifier la taille de la grille utilisée pour le déplacement, utilisez la propriété [GranularitéDéplacement](../Proprietes/1000019642.md). 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Passer la grille à 15 minutes 
AGD_Agenda1.GranularitéDurée = 15
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Durée des éléments : Connaître la taille de la grille

`<Résultat> = <Champ manipulé>.GranularitéDurée`
---

**`<Résultat> : (Entier)`**

Résolution de la grille du champ pour la durée : 

- des rendez-vous (en minutes). La valeur par défaut est de : 

	- 30 minutes dans un champ Agenda.

	- 24 heures dans un champ Planning. 




- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) des événements dans un champ TimeLine. La valeur par défaut est de 60 (secondes, millisecondes ou microsecondes). L'unité dépend de l'unité d'affichage utilisée pour le champ. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) des tâches dans une colonne de type Diagramme de Gantt. La valeur par défaut est de 1 heure.  




Pour un champ Planning ou Agenda, cette résolution peut être comprise entre : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Pour un champ Planning, entre 1 (une minute) et 43200 (1 mois). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Pour un champ Agenda : 1 (une minute) et 1440 (24 heures).


Cette valeur doit être un multiple de 60 (120, 240, 320, ...) ou un sous-multiple de 60 (1, 2, 5, 10, 15, 30, ...). 
Les rendez-vous ou les événements commencent sur un multiple de cette valeur.

**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler : 

- champ Agenda.

- champ Planning.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) champ TimeLine.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) colonne de type Diagramme de Gantt.





<a name="SYNTAXE2"></a>

### Durée des éléments : Modifier la taille de la grille

`<Champ manipulé>.GranularitéDurée = <Valeur>`
---

**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler : 

- champ Agenda.

- champ Planning.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) champ TimeLine.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) colonne de type Diagramme de Gantt.




**`<Valeur> : (Entier)`**

Nouvelle résolution de la grille du champ pour la durée : 

- des rendez-vous dans un champ Planning ou Agenda. Cette résolution est exprimée en minutes. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) des événements dans un champ TimeLine. L'unité dépend de l'unité d'affichage utilisée pour le champ. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) des tâches dans une colonne de type Diagramme de Gantt. La valeur par défaut est de 1 heure.  


Cette valeur doit être un multiple de 60 (120, 240, 320, ...) ou un sous-multiple de 60 (1, 2, 5, 10, 15, 30, ...). 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Les événements ayant une taille inférieure à la valeur définie avec la propriété **GranularitéDurée** ne sont pas affichés. 

- Cette propriété n'est pas disponible pour les champs Agenda et pour les champs Planning présents dans les états.

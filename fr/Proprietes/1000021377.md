


## AvecBoutonNouveau (Propriété)

***En anglais : WithNewButton***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **AvecBoutonNouveau** permet de : 

- Savoir si un champ Onglet dynamique propose une croix d'ajout.  

- Configurer un champ Onglet dynamique pour proposer ou non une croix d'ajout. 




Rappel : La croix d'ajout permet à l'utilisateur final d'ouvrir un nouvel onglet. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ONG_OngletDyn.AvecBoutonNouveau = Vrai
ONG_OngletDyn.FenêtreSiNouveau = "FI_Document"
ONG_OngletDyn.LibelléSiNouveau = "Nouvel onglet"
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Savoir si un champ Onglet dynamique propose une croix d'ajout

`<Résultat> = <Champ Onglet>.AvecBoutonNouveau`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si tous les volets d'un champ Onglet dynamique possèdent une croix d'ajout,

- <u><u><u><u>Faux</u></u></u></u> si les volets d'un champ Onglet dynamique ne possèdent pas une croix d'ajout. 




**`<Champ Onglet> : (Nom de champ)`**

Nom du champ Onglet à manipuler. 


<a name="SYNTAXE2"></a>

### Configurer un champ Onglet dynamique pour proposer ou non une croix d'ajout

`<Champ Onglet>.AvecBoutonNouveau = <Croix d'ajout>`
---

**`<Champ Onglet> : (Nom de champ)`**

Nom du champ Onglet à manipuler. 

**`<Croix d'ajout> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si les volets d'un champ Onglet dynamique doivent afficher une croix d'ajout,

- <u><u><u><u>Faux</u></u></u></u> si les volets d'un champ Onglet dynamique ne doivent pas afficher une croix d'ajout. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si les volets possèdent une croix d'ajout, lorsque l'utilisateur clique sur cette croix, l'événement "Création d'un volet" est exécuté. Par défaut, un nouveau volet est créé. Cependant, l'événement "Création d'un volet" peut empêcher la création du volet via l'instruction "RENVOYER Faux" ou la fonction [DonneFocusEtRetourUtilisateur](../WDLang1/1410088107.md).  

- Le libellé et la fenêtre interne utilisés par défaut pour le nouveau volet peuvent être définis : 

	- dans la fenêtre de description du champ Onglet (onglet "Détail"). 

	- par programmation grâce aux propriétés [FenêtreSiNouveau](../Proprietes/1000021381.md) et [LibelléSiNouveau](../Proprietes/1000021380.md).




- Si les volets ne possèdent pas de croix d'ajout, la Fonctionnalité Automatique de vos Applications (FAA) "Nouveau" présente dans le menu contextuel du volet est désactivée.




## MémoriserLaConfiguration (Propriété)

***En anglais : StoreTheConfiguration***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **MémoriserLaConfiguration** permet de : 

- Savoir si la configuration des volets d'un champ Onglet dynamique ou d'un champ Ruban est automatiquement sauvée et restaurée. 

- Modifier la configuration des volets d'un champ Onglet dynamique ou d'un champ Ruban pour qu'elle soit automatiquement sauvée et restaurée ou non.




Remarque :  Cette propriété correspond à l'option "Mémoriser la configuration des volets" disponible dans l'onglet "Détail" de la fenêtre de description du champ. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ONG_Dynamique.MémoriserLaConfiguration = Vrai
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Savoir si la configuration des volets est automatiquement sauvée et restaurée

`<Résultat> = <Champ manipulé>.MémoriserLaConfiguration`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la configuration des volets du champ est automatiquement sauvée et restaurée,

- <u><u><u><u>Faux</u></u></u></u> si la configuration des volets du champ n'est pas automatiquement sauvée et restaurée. 




**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler : 

- Champ Onglet dynamique. 

- Champ Ruban.





<a name="SYNTAXE2"></a>

### Paramétrer la sauvegarde des volets

`<Champ manipulé>.MémoriserLaConfiguration = <Nouvelle configuration>`
---

**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler : 

- Champ onglet dynamique. 

- Champ Ruban.




**`<Nouvelle configuration> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la configuration des volets du champ est automatiquement sauvée et restaurée,

- <u><u><u><u>Faux</u></u></u></u> si la configuration des volets du champ n'est pas automatiquement sauvée et restaurée. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si cette propriété est mise à <u><u><u><u>Vrai</u></u></u></u> après la phase d'initialisation, le champ ne reprendra pas la configuration précédente.

- Cette propriété est surtout utile pour paramétrer les champs créés via les fonctions [ChampCrée](../WDLang1/1000020876.md) et [ChampClone](../WDLang1/3025017.md).

- Pour un champ Onglet dynamique, la configuration comprend les volets dynamiques et leur position (c'est-à-dire leur ordre). 

	- Si des volets dynamiques étaient détachés, ils seront restaurés "détachés" à la même taille et position.

	- Si la fenêtre interne prend des paramètres simples (chaînes, entiers, booléens), ils sont mémorisés.

	- Si la fenêtre interne prend des paramètres complexes (objets, connexions HFSQL, ...), la sauvegarde n'est pas effectuée.

	- Si la fenêtre interne utilise des variables globales (par exemple la position d'un enregistrement), la restauration ne donnera pas le même résultat qu'à la sauvegarde.




- Pour un champ Ruban, la configuration comprend les volets et leur position (c'est-à-dire leur ordre). Si des volets étaient détachés, ils seront restaurés "détachés" à la même taille et position.

- Champ Onglet dynamique : Il est possible de sauver ou restaurer manuellement la configuration des volets dynamiques via les fonctions [OngletSauveConfiguration](../WDLang1/1000021371.md)/[OngletChargeConfiguration](../WDLang1/1000021370.md).

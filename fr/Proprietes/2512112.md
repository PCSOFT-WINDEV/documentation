


## LargeurMaxVignette (Propriété)

***En anglais : MaxThumbnailWidth***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **LargeurMaxVignette** permet de :

- connaître la largeur maximale d'une vignette d'une rubrique de type Image (mémo binaire). Cette rubrique a été définie sous l'éditeur d'analyses ou définie par programmation.

- définir la largeur maximale d'une vignette d'une rubrique de type Image (mémo binaire), lors de la description de la rubrique par programmation uniquement.









<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la rubrique Photo
Photo est une Description de Rubrique
...
// Définir la largeur maximale de la première 
// vignette de la description de la rubrique Photo
Photo.LargeurMaxVignette[1] = 50
// équivalent à Photo.LargeurMaxVignette = 50
...
// Valider la description du fichier
HDécritFichier(Client)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître la largeur maximale d'une vignette d'une rubrique de type Image (mémo binaire)

`<Largeur maximale actuelle> = <Fichier de données>.<Rubrique>.LargeurMaxVignette[<N>]`
---

**`<Largeur maximale actuelle> : (Entier)`**

Largeur maximale (en pixels) de la vignette spécifiée.

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique de type mémo binaire manipulée. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Rubrique](../WDLang4/1514071.md).

**`<N> : (Entier optionnel)`**

Numéro de la vignette dont on recherche la largeur maximale. Si ce paramètre n'est pas spécifié, la largeur maximale de la première vignette est retournée.  


<a name="SYNTAXE2"></a>

### Modifier la largeur maximale d'une vignette d'une rubrique de type Image (mémo binaire)

`<Rubrique>.LargeurMaxVignette[<N>] = <Nouvelle largeur maximale>`
---

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique de type mémo binaire manipulée. Ce nom vient d'être défini grâce au type [Description de Rubrique](../WDLang4/1514071.md).

**`<N> : (Entier optionnel)`**

Numéro de la vignette dont on veut modifier la largeur maximale. Si ce paramètre n'est pas spécifié, la largeur maximale de la première vignette sera modifiée.

**`<Nouvelle largeur maximale> : (Entier)`**

Nouvelle largeur maximale (en pixels) de la vignette spécifiée. Cette largeur doit être comprise entre 1 et 65535.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La propriété **LargeurMaxVignette** est disponible :

- en HFSQL Classic : sur les rubriques d'un fichier de données, d'une vue ou d'une requête.

- en HFSQL Client/Serveur : sur les rubriques d'un fichier de données uniquement.




![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) La propriété **LargeurMaxVignette** est disponible uniquement en HFSQL Client/Serveur sur les rubriques d'un fichier de données.

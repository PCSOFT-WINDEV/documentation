


## Avant (Propriété)

***En anglais : Before***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **Avant** remplace ou insère une procédure avant les autres procédures dans : 

- une variable de type [Procédure](../Motscles/1000018907.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) un événement associé à un champ.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
p est une Procédure
p = MaProcédure
p.Après = MaProcédureAppeléeAprès
p.Avant = MaProcédureAppeléeAvant

// L'ordre d'appel des procédures sera :
// - MaProcédureAppeléeAvant
// - MaProcédure
// - MaProcédureAppeléeAprès
p()
```


<a name="Exemple2"></a>
![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) 
```wl
// Renseigne l'événement à exécuter sur un clic
BTN_BOUTON1.Traitement[trtClic] = maProcédure1

// Ajoute une procédure à exécuter sur un clic
BTN_BOUTON1.Traitement[trtClic] += maProcédure2

// Ajoute une procédure à exécuter avant le traitement de clic
BTN_BOUTON1.Traitement[trtClic].Avant += maProcédure3
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Remplacer les procédures existantes avec la propriété Avant (opérateur =)

`<Elément>.Avant = <Procédure WLangage>`
---

**`<Elément> : (Chaîne de caractères)`**

Elément à manipuler : 

- Nom d'une variable de type [Procédure](../Motscles/1000018907.md). 

- ![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Evénement d'un champ. Les événements d'un champ peuvent être manipulés avec la propriété [Traitement](../Proprietes/1000020878.md). 




**`<Procédure WLangage> : (Nom de procédure)`**

Nom de la procédure WLangage qui remplacera les procédures existantes. 


<a name="SYNTAXE2"></a>

### Insérer une procédure avant les procédures existantes (opérateur +=)

`<Elément>.Avant += <Procédure WLangage>`
---

**`<Elément> : (Chaîne de caractères)`**

Elément à manipuler : 

- Nom d'une variable de type [Procédure](../Motscles/1000018907.md). 

- ![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Evénement d'un champ. Les événements d'un champ peuvent être manipulés avec la propriété [Traitement](../Proprietes/1000020878.md).




**`<Procédure WLangage> : (Nom de procédure)`**

Nom de la procédure WLangage à placer "avant".

## Remarques


### Sortir de l'enchaînement des procédures
<a name="sortir_enchainement_des_procedures_ELTPARAGRAPHE000109"></a>Si plusieurs procédures s'enchaînent, il est possible d'interrompre l'enchaînement des procédures à partir du code d'une des procédures en utilisant la fonction [DonneFocusEtRetourUtilisateur](../WDLang1/1410088107.md). 

Exemple : 

```wl
// Clic sur BTN_Bouton1
BTN_Bouton2.Traitement[trtclic].Avant += MaProc1
BTN_Bouton2.Traitement[trtclic].Avant += MaProc2
```



```wl
// Clic sur BTN_Bouton2
Trace("Clic sur Bouton 2")
```



```wl
PROCEDURE MaProc1()
Trace(dbgInfo(dbgTraitement))
DonneFocusEtRetourUtilisateur(MoiMême.Nom)
```



```wl
PROCEDURE MaProc2()
Trace(dbgInfo(dbgTraitement))
```

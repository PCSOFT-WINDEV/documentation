


## NomDécrit (Propriété)

***En anglais : DescribedName***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **NomDécrit** permet de gérer le nom logique des fichiers de données HFSQL (sans tenir compte des éventuelles modifications effectuées par la fonction [HAlias](../WDLang4/3044176.md)). Il est ainsi possible de :

- Connaître le nom logique d'un élément :

	- fichier de données défini sous l'éditeur d'analyses ou défini par programmation,

	- vue,

	- requête




- Définir le nom logique d'un fichier de données (lors de la description du fichier par programmation uniquement).









<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connaître le nom logique du fichier de données Client
ResNom = Client.NomDécrit
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le nom logique d'un élément

`<Nom logique> = <Elément>.NomDécrit`
---

**`<Nom logique> : (Chaîne de caractères)`**

Nom logique de l'élément :

- fichier de données défini sous l'éditeur d'analyses ou défini par programmation,

- vue, 

- requête.




**`<Elément> : (Chaîne de caractères)`**

Elément manipulé (par indirection par exemple).

### Définir le nom logique d'un fichier de données créé par programmation

`<Fichier de données>.NomDécrit = <Nouveau nom>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Nouveau nom> : (Chaîne de caractères)`**

Nom logique de l'élément.

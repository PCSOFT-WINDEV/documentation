


## NullSupporté (Propriété)

***En anglais : NullSupported***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **NullSupporté** permet de gérer la valeur NULL dans des fichiers de données HFSQL.

Cette propriété permet de :

- Connaître le mode de gestion de la valeur NULL pour un fichier de données décrit sous l'éditeur d'analyses.

- Définir que la valeur NULL va être gérée lors de la description d'un fichier de données par programmation.




**Remarque** : Si un fichier de données supporte la valeur NULL et si la rubrique autorise le NULL (propriété [NullAutorisé](../Proprietes/1000019492.md)), il sera possible d'utiliser la propriété [Null](../Proprietes/2512064.md) pour par exemple affecter NULL à la rubrique pour l'enregistrement en cours.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Définir la gestion de la valeur NULL dans le fichier de données Client
Client.NullSupporté = Vrai
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Savoir si le fichier de données permet la gestion de la valeur NULL

`<Résultat> = <Fichier de données>.NullSupporté`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier permet la gestion de la valeur NULL, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).  


<a name="SYNTAXE2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 
### Définir la gestion de la valeur NULL

`<Fichier de données>.NullSupporté = <Mode de gestion>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Mode de gestion> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> pour gérer la valeur NULL, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Cas particulier : Vue et requête
<a name="cas_particulier_vue_requete_ELTPARAGRAPHE000097"></a>La propriété **NullSupporté** utilisée sur une vue ou une requête renvoie <u><u><u><u>Vrai</u></u></u></u> si et seulement si au moins un des fichiers source supporte la gestion du NULL.


### Cas d'erreurs
<a name="cas_erreurs_ELTPARAGRAPHE000109"></a>L'accès en écriture sur la propriété [Null](../Proprietes/2512064.md) échoue si la propriété **NullSupporté** a pour valeur <u><u><u><u>Faux</u></u></u></u>.

L'erreur 70409 ("Vous avez appelé la fonction [HModifie](../WDLang4/3044042.md). Le fichier &lt;XXXX&gt; ne supporte pas l'accès à la propriété Null. Veuillez activer la gestion du Null dans la description du fichier.") apparaît si les conditions suivantes sont réunies :

- une rubrique d'une requête ou d'une vue est mise à NULL,

- cette rubrique provient d'un fichier qui ne gère pas la valeur NULL

- la fonction [HModifie](../WDLang4/3044042.md) est utilisée sur la requête ou la vue.

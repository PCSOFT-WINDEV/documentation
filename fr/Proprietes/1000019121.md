


## SécuritéHtml (Propriété)

***En anglais : SecurityHtml***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **SécuritéHtml** permet de connaître et de modifier l'état du mécanisme de sécurité des champs suivants : 

- champs d'affichage HTML. 

- champs Editeur HTML.


Le mécanisme de sécurité de ces types de champs bloque tous les contenus potentiellement dangereux dans le champ (plugins, ActiveX, code Javascript, etc.). 

**Remarques** : 

- Il est également possible de gérer le mécanisme de sécurité : 

	- du champ d'affichage HTML grâce à l'option "Interdit l'exécution des scripts JavaScript" (onglet "Détail" de la fenêtre de description du champ). Si cette option est cochée, le mécanisme de sécurité est activé.

	- du champ Editeur HTML grâce à l'option "Autoriser l'exécution du code javascript dans le document édité" (onglet "Général" de la fenêtre de description du champ). Si cette option est décochée, le mécanisme de sécurité est activé.




- Le champ d'affichage HTML correspond au nouveau nom du champ de saisie HTML présent dans les versions 25 et précédentes. 



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupération d'un document sur Internet
HTTPRequête(gsURL)

// Activation du mécanisme de sécurité du champ d'affichage HTML
HTML_SaisieDoc.SécuritéHtml = secHtmlMaxi

// Initialisation du champ avec le document
HTM_SaisieDoc = HTTPDonneRésultat()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Savoir si le mécanisme de sécurité d'un champ au format HTML est activé

`<Résultat> = <Champ manipulé>.SécuritéHtml`
---

**`<Résultat> : (Constante de type Booléen ou Booléen)`**

Mécanisme de sécurité en cours : 



|   |   |
| --- | --- |
| *secHtmlAucune* (ou <u><u><u><u>Faux</u></u></u></u>) | Les scripts sont exécutés. Les ActiveX ou plugins sont affichés et exécutés. <br><br>**Remarque** : Les scripts ne s'exécuteront pas dans le champ si la propriété [Etat](../Proprietes/2510039.md) du champ correspond à *Inactif* ou *Grisé*. |
| *secHtmlMaxi* (ou <u><u><u><u>Vrai</u></u></u></u>)<br>(Valeur par défaut) | Aucun script n'est exécuté (javascript, vbscript, ...). Aucun ActiveX ou plugin n'est affiché ou exécuté. |



**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler : 

- Champ d'affichage HTML. 

- Champ Editeur HTML. 





<a name="SYNTAXE2"></a>

### Activer/désactiver le mécanisme de sécurité des champs de saisie au format Html

`<Champ manipulé>.SécuritéHtml = <Valeur>`
---

**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler : 

- Champ d'affichage HTML. 

- Champ Editeur HTML. 




**`<Valeur> : (Constante de type Booléen ou Booléen)`**

Mécanisme de sécurité à appliquer : 



|   |   |
| --- | --- |
| *secHtmlAucune* (ou <u><u><u><u>Faux</u></u></u></u>) | Les scripts sont exécutés. Les ActiveX ou plugins sont affichés et exécutés. <br><br>**Remarque** : Les scripts ne s'exécuteront pas dans le champ si la propriété [Etat](../Proprietes/2510039.md) du champ correspond à *Inactif* ou *Grisé*. |
| *secHtmlMaxi* (ou <u><u><u><u>Vrai</u></u></u></u>)<br>(Valeur par défaut) | Aucun script n'est exécuté (javascript, vbscript, ...). Aucun ActiveX ou plugin n'est affiché ou exécuté. |




## CryptageNdx (Propriété)

***En anglais : NdxEncryption***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **CryptageNdx** permet de paramétrer le mode de cryptage du fichier d'index (d'extension .NDX) associé au fichier de données. Il est ainsi possible de :

- Connaître le mode de cryptage d'un fichier d'index associé à un fichier de données (fichier défini sous l'éditeur d'analyses ou défini par programmation).

- Définir le mode de cryptage d'un fichier d'index associé à un fichier de données lors de sa description par programmation.







<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connaître le mode de cryptage du fichier d'index associé au fichier de données Client
ResCryptage = Client.CryptageNdx
```

<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le mode de cryptage d'un fichier d'index

`<Mode de cryptage actuel> = <Fichier de données>.CryptageNdx`
---

**`<Mode de cryptage actuel> : (Constante de type entier)`**

Les modes de cryptage sont :


|   |   |
| --- | --- |
| *hCryptageAES128* | Le cryptage est effectué uniquement si un mot de passe est défini sur le fichier de données. Dans ce cas, le fichier d'index est crypté selon un algorithme de cryptage AES 128 bits avec un mode d'opération OFB. |
| *hCryptageAES192* | Le cryptage est effectué uniquement si un mot de passe est défini sur le fichier de données. Dans ce cas, le fichier d'index est crypté selon un algorithme de cryptage AES 192 bits avec un mode d'opération OFB. |
| *hCryptageAES256* | Le cryptage est effectué uniquement si un mot de passe est défini sur le fichier de données. Dans ce cas, le fichier d'index est crypté selon un algorithme de cryptage AES 256 bits avec un mode d'opération OFB. |
| *hCryptageAuto* | Le cryptage est effectué uniquement si un mot de passe est défini sur le fichier de données. Dans ce cas, le fichier d'index est crypté selon un algorithme de cryptage optimisé RC5 16 boucles sur 128 bits. |
| *hCryptageNon* | Aucun cryptage du fichier d'index n'est effectué. |
| *hCryptageRC5_12* | Le fichier d'index est crypté selon un algorithme de cryptage optimisé RC5 12 boucles sur 128 bits. |
| *hCryptageRC5_16* | Le fichier d'index est crypté selon un algorithme de cryptage optimisé RC5 16 boucles sur 128 bits. |
| *hCryptageStandard* | Le fichier d'index est crypté selon un algorithme de cryptage optimisé, sur 128 bits. |



**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).  


<a name="SYNTAXE2"></a>

## Remarques


### Cryptage AES (128 bits, 192 bits et 256 bits) avec un mode d'opération OFB
<a name="cryptage_aes_128_bits_192_bits_256_bits_avec_mode_operation_ofb_ELTPARAGRAPHE000130"></a>

- Si le fichier d'index est crypté selon l'algorithme AES, il est nécessaire d'utiliser la même méthode pour crypter les mémos (propriété [CryptageMmo](../Proprietes/2512014.md)) et les données (propriété [CryptageFic](../Proprietes/2512046.md)). 

- Si un cryptage selon l'algorithme AES est réalisé, l'option "Activer la sécurité renforcée " est automatiquement appliquée.




## NomPhysiqueDécrit (Propriété)

***En anglais : DescribedPhysicalName***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **NomPhysiqueDécrit** permet de gérer le nom physique des fichiers de données HFSQL.

Cette propriété ne tient pas compte des éventuels changements de noms réalisés grâce à la fonction [HChangeNom](../WDLang4/3044141.md). Il est ainsi possible de :

- Connaître le nom physique d'un fichier de données défini sous l'éditeur d'analyses ou défini par programmation.

- Définir un nom physique pour un fichier de données (lors de la description du fichier de données par programmation uniquement).









<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connaître le nom physique du fichier de données Client
ResNomPhys = Client.NomPhysiqueDécrit
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le nom physique d'un fichier de données

`<Nom physique actuel> = <Fichier de données>.NomPhysiqueDécrit`
---

**`<Nom physique actuel> : (Chaîne de caractères)`**

Nom physique du fichier de données (sans extension). Ce nom ne tient pas compte des éventuels changements de noms (fonction [HChangeNom](../WDLang4/3044141.md)).

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).

### Définir un nom physique pour un fichier de données créé par programmation

`<Fichier de données>.NomPhysiqueDécrit = <Nouveau nom physique>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Nouveau nom physique> : (Chaîne de caractères)`**

Nom physique du fichier de données (sans extension).

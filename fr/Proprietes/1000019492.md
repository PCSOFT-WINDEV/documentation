


## NullAutorisé (Propriété)

***En anglais : NullAllowed***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **NullAutorisé** permet de gérer la valeur NULL dans les rubriques des fichiers de données HFSQL.

Cette propriété permet de :

- Connaître le mode de gestion de la valeur NULL pour une rubrique d'un fichier de données décrit sous l'éditeur d'analyses.

- Définir le mode de gestion de la valeur NULL lors de la description d'une rubrique d'un fichier de données par programmation.




Si un fichier de données supporte la valeur NULL et si sa rubrique autorise la gestion du NULL, il sera possible d'utiliser la propriété [Null](../Proprietes/2512064.md) pour par exemple affecter NULL à la rubrique pour l'enregistrement en cours.

La propriété **NullAutorisé** est utile par exemple pour ne pas autoriser la valeur NULL pour certaines rubriques d'un fichier de données défini par programmation (ce fichier supportant la valeur NULL).  






<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Savoir si la rubrique permet la gestion de la valeur NULL

`<Résultat> = <Fichier de données>.<Rubrique>.NullAutorisé`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la rubrique permet la gestion de la valeur NULL, 

- <u><u><u><u>Faux</u></u></u></u> si la rubrique ne permet pas la gestion de la valeur NULL.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique manipulée. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Rubrique](../WDLang4/1514071.md).


<a name="SYNTAXE2"></a>

### Définir la gestion de la valeur NULL pour la rubrique

`<Rubrique>.NullAutorisé = <Mode de gestion>`
---

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique manipulée. Ce nom vient d'être défini grâce au type [Description de Rubrique](../WDLang4/1514071.md).

**`<Mode de gestion> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> pour gérer la valeur NULL, 

- <u><u><u><u>Faux</u></u></u></u> pour ne pas gérer la valeur NULL.







## Remarques


### Comment savoir si un fichier de données supporte la gestion de la valeur NULL ? 
<a name="comment_savoir_fichier_donnees_supporte_gestion_valeur_null_ELTPARAGRAPHE000094"></a>Pour savoir si un fichier de données autorise ou non la gestion de la valeur NULL, utilisez la propriété [NullSupporté](../Proprietes/2512061.md).

Attention :

- L'affectation de la propriété **NullAutorisé** sur un fichier de données ou une rubrique qui ne supporte pas la gestion de la valeur NULL est interdite : une erreur fatale est affichée.

- La consultation de la propriété **NullAutorisé** sur un fichier de données ou une rubrique qui ne supporte pas la gestion de la valeur NULL renvoie toujours <u><u><u><u>Faux</u></u></u></u>.



<a name="NOTE0_2"></a>


### Limites
<a name="limites_ELTPARAGRAPHE000119"></a>Il n'est pas possible d'utiliser la valeur NULL :

- sur les rubriques tableau.

- sur les identifiants automatiques.

- sur les clés composées.

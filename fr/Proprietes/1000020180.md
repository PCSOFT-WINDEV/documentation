


## PublicitéChargée (Propriété)

***En anglais : AdLoaded***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **PublicitéChargée** permet de savoir si une publicité est actuellement chargée (et affichée) dans le champ Publicité. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI PUB_MaPub.ModeTest = Faux _ET_ PUB_MaPub.PublicitéChargée = Vrai ALORS
	NbPubVue = NbPubVue + 1
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = <Champ Publicité>.PublicitéChargée`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si une publicité est actuellement chargée (et affichée) dans le champ spécifié. 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  




**`<Champ Publicité> : (Nom de champ)`**

Nom du champ Publicité à manipuler. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Fonctionnement des publicités
<a name="fonctionnement_des_publicites_ELTPARAGRAPHE000040"></a>Une publicité est automatiquement chargée dans le champ Publicité à l'ouverture de la fenêtre. 

De nouvelles publicités peuvent être ensuite chargées régulièrement dans le champ selon la fréquence de rafraîchissement définie dans le paramétrage de l'application sur le site d'AdMob. 

Les publicités sont chargées en tâche de fond depuis Internet. L'appareil doit donc être connecté pour que les publicités puissent être affichées. Un délai allant de quelques secondes à plusieurs minutes peut être nécessaire entre l'ouverture de la fenêtre et l'affichage effectif de la première publicité. Ce délai dépend entre autres de la qualité de la connexion et de la taille de la publicité. 

Il est conseillé d'utiliser les événements "Chargement de publicité" et "Erreur de chargement" pour contrôler l'affichage des publicités dans le champ.

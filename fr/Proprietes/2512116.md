


## RépertoireSauvegardeJournal (Propriété)

***En anglais : LogBackupDirectory***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **RépertoireSauvegardeJournal** permet de gérer le répertoire de sauvegarde des fichiers journal. Il est ainsi possible de :

- Connaître le répertoire de sauvegarde du journal pour un fichier de données défini sous l'éditeur d'analyses ou défini par programmation.

- Définir le répertoire de sauvegarde du journal pour un fichier de données (lors de la description du fichier par programmation uniquement).




**Remarque** : Ce chemin est utilisé lors de la modification automatique des fichiers de données pour archiver les journaux. La fonction [HHistoriqueModification](../WDLang4/3044344.md) recherchera également l'historique des modifications dans ces fichiers.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Définit le répertoire journal du fichier de données Commande
Commande.RépertoireSauvegardeJournal = "C:\MesFichiers\MesJournaux_SVG"
...
// Valide la description du fichier
HDécritFichier(Commande)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le répertoire de sauvegarde du journal d'un fichier

`<Nom actuel> = <Fichier de données>.RépertoireSauvegardeJournal`
---

**`<Nom actuel> : (Chaîne de caractères)`**

Nom du répertoire de sauvegarde du fichier journal associé au fichier de données. Par défaut, ce chemin correspond à : &lt;répertoire des fichiers JNL&gt;\\__JNLBackup. Si ce paramètre correspond à un chemin relatif (et commence par un point), le chemin sera relatif aux fichiers de données.

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).  


<a name="SYNTAXE2"></a>

### Définir un répertoire de sauvegarde du journal pour un fichier créé par programmation

`<Fichier de données>.RépertoireSauvegardeJournal = <Nouveau nom>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Nouveau nom> : (Chaîne de caractères)`**

Chemin des sauvegarde des fichiers journaux correspondant au fichier de données spécifié. Par défaut, ce chemin correspond à : &lt;répertoire des fichiers JNL&gt;\\__JNLBackup. 
Si ce paramètre correspond à une chaîne vide (""), l'emplacement actuel n'est pas modifié. 
Si ce paramètre correspond à un chemin relatif (et commence par un point), le chemin sera relatif aux fichiers de données.

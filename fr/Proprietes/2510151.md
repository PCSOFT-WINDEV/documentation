


## InterprèteEtCommercial (Propriété)

***En anglais : InterpretAmpersand***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **InterprèteEtCommercial** permet de connaître et de modifier le mode d'interprétation du caractère '&' dans :

- le libellé d'un champ Libellé.

- les éléments d'un champ Liste.

- les éléments d'un champ Combo.

- les valeurs d'une colonne d'un champ Table.




**Remarque** : Par défaut, si une chaîne de caractères contient le caractère '&', la lettre qui suit le caractère '&' est interprétée comme lettre d'appel. Pour plus de détails, consultez [Raccourci clavier et lettre d'appel](../WDChamp/1014018.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ne pas interpréter le caractère '&' comme une lettre d'appel
LISTE_ListeLivre.InterprèteEtCommercial = Faux
// Ajout d'un titre de livre
ListeAjoute(LISTE_ListeLivre, "Guerre&Paix")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le mode d'interprétation du caractère '&'

`<Résultat> = <Champ manipulé>.InterprèteEtCommercial`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la lettre suivant le caractère '&' est interprétée comme lettre d'appel, 

- <u><u><u><u>Faux</u></u></u></u> si le caractère '&' n'est pas interprété.




**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler.  


<a name="SYNTAXE2"></a>

### Modifier le mode d'interprétation du caractère '&'

`<Champ manipulé>.InterprèteEtCommercial = <Nouveau mode d'interprétation>`
---

**`<Champ manipulé> : (Nom de champ)`**

Nom du champ à manipuler. 

**`<Nouveau mode d'interprétation> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la lettre suivant le caractère '&' doit être interprétée comme lettre d'appel, 

- <u><u><u><u>Faux</u></u></u></u> si le caractère '&' ne doit pas être interprété.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Caractère '&' interprété comme lettre d'appel
<a name="caractere_interprete_comme_lettre_appel_ELTPARAGRAPHE000064"></a>Si la lettre suivant le caractère '&' est interprétée comme lettre d'appel, pour utiliser le caractère '&' dans un libellé, il est nécessaire de le doubler. Par exemple :

- le libellé "Guerre&Paix" sera "Guerre<u><u><u><u>P</u></u></u></u>aix" (avec Alt + P comme lettre d'appel).

- le libellé "Guerre&&Paix" sera "Guerre&Paix".



<a name="NOTE0_2"></a>


### Limitations
<a name="limitations_ELTPARAGRAPHE000078"></a>La propriété **InterprèteEtCommercial** peut uniquement être utilisée sur :

- un champ Libellé.

- un champ Liste.

- un champ Combo.

- une colonne d'un champ Table.




## CryptageFic (Propriété)

***En anglais : FicEncryption***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **CryptageFic** permet de paramétrer le mode de cryptage du fichier de données. Il est ainsi possible de :

- Connaître le mode de cryptage d'un fichier de données (.FIC) (fichier défini sous l'éditeur d'analyses ou défini par programmation).

- Définir le mode de cryptage d'un fichier de données lors de sa description par programmation.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connaître le mode de cryptage du fichier de données Client
ResCryptage = Client.CryptageFic
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le mode de cryptage d'un fichier de données

`<Mode de cryptage actuel> = <Fichier de données>.CryptageFic`
---

**`<Mode de cryptage actuel> : (Constante de type entier)`**

Les modes de cryptage sont :



|   |   |
| --- | --- |
| *hCryptageAES128* | Le cryptage est effectué uniquement si un mot de passe est défini sur le fichier de données. Dans ce cas, le fichier de données est crypté selon un algorithme de cryptage AES 128 bits avec un mode d'opération OFB. |
| *hCryptageAES192* | Le cryptage est effectué uniquement si un mot de passe est défini sur le fichier de données. Dans ce cas, le fichier de données est crypté selon un algorithme de cryptage AES 192 bits avec un mode d'opération OFB. |
| *hCryptageAES256* | Le cryptage est effectué uniquement si un mot de passe est défini sur le fichier de données. Dans ce cas, le fichier de données est crypté selon un algorithme de cryptage AES 256 bits avec un mode d'opération OFB. |
| *hCryptageAuto* | Le cryptage est effectué uniquement si un mot de passe est défini sur le fichier de données. Dans ce cas, le fichier de données est crypté selon un algorithme de cryptage optimisé RC5 16 boucles sur 128 bits. |
| *hCryptageNon* | Aucun cryptage du fichier de données n'est effectué. |
| *hCryptageRC5_12* | Le fichier de données est crypté selon un algorithme de cryptage optimisé RC5 12 boucles sur 128 bits. |
| *hCryptageRC5_16* | Le fichier de données est crypté selon un algorithme de cryptage optimisé RC5 16 boucles sur 128 bits. |
| *hCryptageStandard* | Le fichier de données est crypté selon un algorithme de cryptage optimisé, sur 128 bits. |



**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).  


<a name="SYNTAXE2"></a>

### Définir le mode de cryptage d'un fichier de données

`<Fichier de données>.CryptageFic = <Nouveau mode de cryptage>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

## Remarques


### Cryptage AES (128 bits, 192 bits et 256 bits) avec un mode d'opération OFB
<a name="cryptage_aes_128_bits_192_bits_256_bits_avec_mode_operation_ofb_ELTPARAGRAPHE000130"></a>

- Si le fichier de données est crypté selon l'algorithme AES, il est nécessaire d'utiliser la même méthode pour crypter les mémos (propriété [CryptageMmo](../Proprietes/2512014.md)) et les index (propriété [CryptageNdx](../Proprietes/2512010.md)). 

- Si un cryptage selon l'algorithme AES est réalisé, l'option "Activer la sécurité renforcée " est automatiquement appliquée. Dans ce mode, un mot de passe doit être défini sur le fichier de données. Ce mot de passe devra être saisi à chaque modification automatique du fichier de données.

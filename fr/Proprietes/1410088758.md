
## NullIndexé (Propriété)

***En anglais : NullableIndex***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **NullIndexé** permet de paramétrer le mode de gestion de la valeur "Null" pour une rubrique clé. Il est ainsi possible de :  

- Savoir si l'index peut supporter la valeur "Null" pour une rubrique clé. 

- Définir le mode de gestion de la valeur "Null" pour une rubrique clé (lors de la description du fichier par programmation uniquement). 




Remarque : Cette propriété est équivalente à l'option "NULL indexé (le fichier sera incompatible avec les applications en version inférieure à 28)" présente dans l'onglet "Détail" de la fenêtre de description du fichier de données, dans l'éditeur d'analyse. 

**Attention : Les fichiers possédant l'option "Null indexé" activée, sont incompatibles avec les applications en version inférieure à la version 28.** 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
UnFichier est une Description de Fichier
UneRubrique est une Description de Rubrique

// Description du fichier de données "CLIENT"
UnFichier.Nom = "CLIENT"
UnFichier.Type = hFichierNormal
UnFichier.NullIndexé = Vrai

// Description de la rubrique 
UneRubrique.Nom = "Message"
UneRubrique.Type = hRubTexte
UneRubrique.Taille = 40
HDécritRubrique(UnFichier, UneRubrique)

// Valide la description du fichier de données 
HDécritFichier(UnFichier)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Savoir si l'index peut supporter la valeur "Null" pour une rubrique clé 

`<Résultat> = <Fichier de données>.NullIndexé`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier de données est configuré pour gérer la valeur "Null" pour une rubrique clé,

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).


<a name="SYNTAXE2"></a>

### Définir le mode de gestion de la valeur "Null" pour une rubrique clé (fichier défini par programmation)

`<Fichier de données>.NullIndexé = <Gestion de la valeur Null>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Gestion de la valeur Null> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier de données doit être configuré pour autoriser la valeur "Null" pour une rubrique clé,

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Gestion de la valeur "Null" pour les rubriques clé
<a name="gestion_valeur_null_pour_les_rubriques_cle_ELTPARAGRAPHE000071"></a>Si le fichier de données supporte la valeur "Null" pour les clés : 

- Il est possible d'utiliser les fonctions de recherche avec la valeur "Null". 

- Plusieurs clés du fichier peuvent avoir la valeur "Null".




## Null (Propriété)

***En anglais : Null***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **Null** permet de gérer la valeur NULL dans les rubriques des fichiers de données HFSQL ou dans les rubriques des fichiers de données accédés par les Connecteurs Natifs.

Cette propriété permet de :

- Affecter NULL à la rubrique pour l'enregistrement en cours.

- Savoir si la rubrique est associée à la valeur NULL pour l'enregistrement en cours.

- Définir la valeur par défaut d'une rubrique lors de la description d'un fichier de données par programmation.









<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Affecter NULL à la rubrique Client.Portable
Client.Portable.Null = Vrai
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Savoir si la rubrique est associée à la valeur NULL

`<Résultat> = <Fichier de données>.<Rubrique>.Null`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la rubrique associée à l'enregistrement en cours a pour valeur NULL, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique manipulée.

### Affecter NULL à une rubrique

`<Fichier de données>.<Rubrique>.Null = <Valeur nulle>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique manipulée.

**`<Valeur nulle> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> pour affecter NULL à la rubrique spécifiée pour l'enregistrement en cours.  

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.

### Définir la valeur par défaut d'une rubrique d'un fichier défini par programmation

`<Fichier de données>.<Rubrique>.Null = <Valeur nulle>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom logique du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Rubrique> : (Chaîne de caractères)`**

Nom de la rubrique manipulée.

**`<Valeur nulle> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la valeur par défaut de la rubrique doit être nulle, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Comment savoir si un fichier de données supporte la gestion de la valeur NULL ? 
<a name="comment_savoir_fichier_donnees_supporte_gestion_valeur_null_ELTPARAGRAPHE000119"></a>

- Pour savoir si un fichier de données autorise ou non la gestion de la valeur NULL, utilisez la propriété [NullSupporté](../Proprietes/2512061.md).

- Pour savoir si une rubrique autorise ou non la gestion de la valeur NULL, utilisez la propriété [NullAutorisé](../Proprietes/1000019492.md). 


Attention :

- L'affectation de la propriété **Null** sur un fichier de données ou une rubrique qui ne supporte pas la gestion de la valeur NULL est interdite : une erreur fatale est affichée.

- La consultation de la propriété **Null** sur un fichier de données ou une rubrique qui ne supporte pas la gestion de la valeur NULL renvoie toujours <u><u><u><u>Faux</u></u></u></u>.



<a name="NOTE0_2"></a>


### Limites
<a name="limites_ELTPARAGRAPHE000150"></a>

- Il n'est pas possible d'utiliser la valeur NULL :

	- sur les rubriques tableau.

	- sur les identifiants automatiques.

	- sur les clés composées.




- Il n'est pas possible d'affecter la valeur NULL plusieurs fois à une rubrique prenant en compte la gestion des doublons.

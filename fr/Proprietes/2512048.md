
## CxAccèsDécrit (Propriété)

***En anglais : CtDescribedAccess***
	



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La propriété **CxAccèsDécrit** permet de gérer le type d'accès au fichier de données lors de la connexion OLE DB à une table spécifique. Il est possible de :

- Connaître le type d'accès au fichier de données utilisé pour la connexion à une table par OLE DB. Ce type d'accès a été décrit sous l'éditeur d'analyses ou lors de la description par programmation du fichier de données.
	Attention : La propriété **CxAccèsDécrit** ne tient pas compte des éventuelles modifications réalisées avec la fonction [HConnecte](../WDLang4/3044263.md).
	Cette propriété peut être utilisée pour un fichier de données défini sous l'éditeur d'analyses ou défini par programmation.

- Définir le type d'accès au fichier de données utilisé pour la connexion à une table par OLE DB (lors de la description par programmation du fichier de données uniquement). Cette propriété est équivalente à la propriété [CxAccès](../Proprietes/2512015.md). 




**Rappel** : La fonction [HConnecte](../WDLang4/3044263.md) permet de redéfinir un ou plusieurs paramètres de la connexion OLE DB sur une table. Par défaut, les caractéristiques de la connexion OLE DB sont définies :

- soit sous l'éditeur d'analyses lors de la description du fichier de données,

- soit en programmation lors de la description du fichier de données.



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connaître le type d'accès au fichier de données Client
ResTypeAccès = Client.CxAccèsDécrit
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le type d'accès au fichier de données par OLE DB

`<Type d'accès> = <Fichier de données>.CxAccèsDécrit`
---

**`<Type d'accès> : (Constante)`**

Renvoie le type d'accès au fichier de données spécifié. 
Cette information a été définie lors de la description du fichier de données (description sous l'éditeur d'analyses ou description par programmation).


|   |   |
| --- | --- |
| *hOLecture* | Le fichier de données est accessible uniquement en lecture. Seule la lecture des enregistrements est possible. Il n'est pas possible d'en ajouter ou d'en supprimer. |
| *hOLectureEcriture* | Le fichier de données est accessible en lecture et en écriture. Il est possible aussi bien de lire des enregistrements que d'écrire de nouveaux enregistrements. |



**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom a été défini sous l'éditeur d'analyses ou grâce au type [Description de Fichier](../WDLang4/1514065.md).  


<a name="SYNTAXE2"></a>

### Définir le type d'accès au fichier par OLE DB

`<Fichier de données>.CxAccèsDécrit = <Type d'accès>`
---

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données manipulé. Ce nom vient d'être défini grâce au type [Description de Fichier](../WDLang4/1514065.md).

**`<Type d'accès> : (Constante)`**

Définit le type d'accès au fichier de données.


|   |   |
| --- | --- |
| *hOLecture* | Le fichier de données est accessible uniquement en lecture. Seule la lecture des enregistrements est possible. Il n'est pas possible d'en ajouter ou d'en supprimer. |
| *hOLectureEcriture* | Le fichier de données est accessible en lecture et en écriture. Il est possible aussi bien de lire des enregistrements que d'écrire de nouveaux enregistrements. |







<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La propriété [CxAccès](../Proprietes/2512015.md) permet de connaître le type d'accès actuellement utilisé pour la connexion OLE DB. Cette propriété permet également de définir le type d'accès utilisé (lors de la description par programmation du fichier de données uniquement).

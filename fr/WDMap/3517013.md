
## WDMap : Caractéristiques des fichiers présents dans une base de données
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000280"></a>
WDMap permet d'obtenir diverses caractéristiques spécifiques :

- à chaque fichier présent dans une base de données manipulée par un projet WINDEV, WEBDEV ou WINDEV Mobile,

- à l'ensemble des fichiers présents dans une base de données manipulée par un projet WINDEV, WEBDEV ou WINDEV Mobile.






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000310"></a>


### WDMap lancé depuis l'éditeur
<a name="wdmap_lance_depuis_editeur_ELTPARAGRAPHE000023"></a>Pour obtenir des informations sur les fichiers présents dans une base de données :

1. Sous WDMap, sélectionnez l'analyse WINDEV, WEBDEV ou WINDEV Mobile (fichier ".WDD") contenant la description du fichier de données voulu (utilisez si nécessaire le bouton "Parcourir").

2. Précisez si nécessaire le mot de passe de l'analyse (mot de passe en exécution défini dans la description de l'analyse).

3. Sélectionnez l'option "Affichage .. Informations générales sur la base de données" (ou cliquez sur l'icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=IconeInfo.gif)
). La fenêtre des caractéristiques de la base de données s'affiche.



<a name="NOTE2_2"></a>


### WDMAP lancé directement par le programme "WDMAP.EXE"
<a name="wdmap_lance_directement_par_programme_wdmapexe_ELTPARAGRAPHE000036"></a>Pour obtenir des informations sur les fichiers présents dans une base de données :

1. Dans l'assistant, sélectionnez l'option "Sélectionnez une analyse" et passez à l'étape suivante.

2. Sélectionnez l'analyse WINDEV, WEBDEV ou WINDEV Mobile (fichier ".WDD") contenant la description du fichier de données voulu (utilisez si nécessaire le bouton "Parcourir") et passez à l'étape suivante. WDMap se lance.

3. Précisez si nécessaire le mot de passe de l'analyse (mot de passe en exécution défini dans la description de l'analyse).

4. Sélectionnez l'option "Affichage .. Informations générales sur la base de données" (ou cliquez sur l'icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=IconeInfo.gif)
). La fenêtre des caractéristiques de la base de données s'affiche.






<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Quelles sont les caractéristiques affichées ?
<a name="quelles_sont_les_caracteristiques_affichees_ELTTEXTE000340"></a>
Une base de données manipulée par un projet WINDEV, WEBDEV ou WINDEV Mobile peut être composée aussi bien de fichiers de type HFSQL que de fichiers de type non HFSQL.

La fenêtre contenant les caractéristiques est divisée en deux :

- La partie haute contient les caractéristiques spécifiques aux fichiers de données de type HFSQL.

- La partie basse contient les caractéristiques spécifiques aux autres types de fichiers de données (format non HFSQL) accédés via un provider OLE DB, un driver ODBC ou un accès natif.




**Remarque** : Si la base de données en cours est composée uniquement de fichiers HFSQL, seule la partie haute sera affichée. Si la base de données en cours est composée uniquement de fichiers non HFSQL, seule la partie basse sera affichée.

### Caractéristiques spécifiques aux fichiers de données de type HFSQL
<a name="caracteristiques_specifiques_aux_fichiers_donnees_type_hfsql_ELTPARAGRAPHE000068"></a>Le tableau suivant présente les caractéristiques affichées par WDMap. Ces caractéristiques sont spécifiques aux fichiers de données de type HFSQL présents dans la base de données du projet WINDEV, WEBDEV ou WINDEV Mobile :

| Nom | Commentaire |
| --- | --- |
| Nom logique | Nom du fichier défini dans l'analyse. |
| Nom physique | Nom du fichier physique associé au fichier logique. Ce nom est défini dans l'analyse. |
| Type | HFSQL Classic ou Hyper File 5.5. |
| Taille totale | Taille actuelle (en Ko) du fichier de données, du fichier d'index et du fichier mémo correspondant au fichier logique. |
| Nombre d'enregistrements actifs | Un enregistrement actif est un enregistrement pouvant être manipulé directement (avec la fonction [HLit](../WDLang4/3044047.md) par exemple). |
| Nombre d'enregistrements rayés | Un enregistrement rayé est un enregistrement supprimé logiquement (avec la fonction [HRaye](../WDLang4/3044121.md)). Cet enregistrement pourra éventuellement être récupéré (grâce aux fonctions [HModifie](../WDLang4/3044042.md) ou [HEcrit](../WDLang4/3044092.md)). Cet enregistrement peut être supprimé physiquement (avec la fonction [HSupprime](../WDLang4/3044018.md)). |
| Nombre d'enregistrements supprimés | Un enregistrement supprimé est un enregistrement supprimé logiquement et physiquement (avec la fonction [HSupprime](../WDLang4/3044018.md)). |
| Nombre d'enregistrements total | Nombre d'enregistrements actifs, rayés et supprimés dans le fichier. |
| Mot de passe | Mot de passe du fichier défini soit à l'aide des fonctions [HPasse](../WDLang4/3044108.md), [HCréation](../WDLang4/3044255.md) ou [HCréationSiInexistant](../WDLang4/3044254.md), soit lors de la [modification automatique des données](../WDLang4/3044195.md). |
| Cryptage | Indique si le fichier est crypté. |
| Journal | Indique si le fichier est journalé.<br>[Le journal](../WDLang4/3044184.md) est un fichier particulier dans lequel sont automatiquement mémorisées toutes les opérations réalisées sur un ou plusieurs fichiers de données depuis un moment donné. |
| Réplication | Indique si le fichier est répliqué.<br>[La réplication](../WDLang4/3044266.md) est l'opération permettant de maintenir à jour des bases de données distantes de structures identiques, et sur lesquelles des opérations différentes sont effectuées. |
| **Informations supplémentaires liées au fichier physique lors d'un double-clic sur un fichier de données de type HFSQL** |
| Répertoire du fichier | Répertoire du fichier physique. |
| Date de la dernière modification |   |
| Chemin de l'analyse 5.5 | Uniquement si le fichier sélectionné provient d'une analyse 5.5. |
| Répertoire du journal | Répertoire du fichier journal associé au fichier sélectionné (si le fichier journal existe). |
| Taille du fichier journal | Taille actuelle du fichier journal en Ko (si le fichier journal existe). |

Pour l'ensemble des fichiers de données de type HFSQL présents dans la base de données, les calculs suivants sont automatiquement effectués :

- La taille totale (en Ko) de l'ensemble des fichiers de données.

- La taille totale (en Ko) des fichiers journal associés aux fichiers de données.

- Le nombre d'enregistrements présents dans l'ensemble des fichiers de données.

### Caractéristiques spécifiques aux autres types de fichiers de données (format non HFSQL)
<a name="caracteristiques_specifiques_aux_autres_types_fichiers_donnees_format_non_hfsql_ELTPARAGRAPHE000193"></a>Le tableau suivant présente les caractéristiques affichées par WDMap. Ces caractéristiques sont spécifiques aux fichiers de données de type non HFSQL présents dans la base de données du projet WINDEV, WEBDEV ou WINDEV Mobile :

| Nom | Commentaire |
| --- | --- |
| Fichier | Nom logique du fichier. |
| Libellé | Libellé de la connexion utilisée pour accéder aux différents fichiers de données présents dans l'analyse en cours (spécifié lors du paramétrage de la connexion sous l'éditeur d'analyses). |
| Connexion par | Mode d'accès à la base de données en cours : provider OLE DB, driver ODBC ou accès natif. |
| Source de données | Source de données de la connexion :<br><br>- soit nom et chemin complet de la base de données,<br><br>- soit répertoire du fichier xBase,<br><br>- soit nom ou alias du serveur utilisé.<br><br><br> |
| Utilisateur | Nom d'utilisateur nécessaire pour se connecter à la base de données. |
| Accès | Type d'accès au fichier : lecture seule ou lecture et écriture. |
| Nombre d'enregistrements | Nombre d'enregistrements dans le fichier. |


## WDMap : Visualiser les enregistrements d'un fichier de données
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000207"></a>
WDMap permet de visualiser les enregistrements présents dans un **fichier de données**. 

Il est ainsi possible d'obtenir une vision rapide et synthétique des différents enregistrements présents dans un fichier de données, pendant le développement d'une application.

Remarque : WDMap ne permet pas de visualiser le résultat d'une requête. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Affichage par défaut
<a name="affichage_par_defaut_ELTTEXTE000231"></a>
Par défaut, WDMap affiche les enregistrements d'un fichier de données en appliquant les règles suivantes :

- Tous les enregistrements présents dans le fichier de données sont affichés.

- Les enregistrements sont triés selon leur ordre de création dans le fichier de données.

- Toutes les rubriques du fichier de données sont affichées (sauf les rubriques de type "Clé composée").

- Les rubriques sont classées par ordre logique. L'ordre logique correspond à l'ordre d'affichage défini sous l'éditeur d'analyses.

- L'intégrité est gérée. 
	**Attention** : Si les fichiers liés au fichier sélectionné ne sont pas automatiquement trouvés par WDMap, il sera possible uniquement de visualiser les enregistrements. Pour ajouter ou modifier ces enregistrements, localisez manuellement les fichiers manquants (icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=IconeGestionIntegrite.gif)).

- Les doublons sont gérés.




Pour plus de détails sur l'affichage du numéro d'enregistrement, des rubriques mémo ou clé composée, consultez [Cas particuliers](../WDMap/3517005.md).


Cependant, il est possible de modifier à tout moment :

- La sélection des informations à afficher. En effet, WDMap permet de sélectionner :

	- les enregistrements à visualiser,

	- les rubriques à visualiser.




- Les options d'affichage de la table de visualisation.




Pour plus de détails, consultez [Modifier le mode d'affichage des enregistrements](../WDMap/3517011.md).



<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000255"></a>
Pour visualiser les enregistrements d'un fichier de données, le mode opératoire peut varier selon le mode de lancement de WDMap :

- Lancement depuis l'explorateur ou depuis l'éditeur d'analyses.

- Lancement depuis WINDEV ou WEBDEV.

- Lancement directement par le programme "WDMAP.EXE".





<a name="NOTE3_2"></a>


### WDMap lancé depuis l'explorateur (double-clic sur un fichier de données) ou depuis l'éditeur d'analyses (option "Éditer le fichier de données avec WDMap" du menu contextuel)
<a name="wdmap_lance_depuis_explorateur_doubleclic_sur_fichier_donnees_depuis_editeur_analyses_option_editer_fichier_donnees_avec_wdmap_menu_contextuel_ELTPARAGRAPHE000069"></a>Pour visualiser les enregistrements présents dans le fichier de données sélectionné :

1. Précisez si nécessaire :

	- Le mot de passe du fichier sélectionné (mot de passe défini soit à l'aide des fonctions [HPasse](../WDLang4/3044108.md), [HCréation](../WDLang4/3044255.md) ou [HCréationSiInexistant](../WDLang4/3044254.md), soit lors de la [modification automatique des données](../WDLang4/3044195.md)).

	- Le chemin complet de l'analyse 5.5 d'origine et le mot de passe de cette analyse (si le fichier sélectionné provient d'une analyse 5.5).

	- Les paramètres de connexion (si le fichier est manipulé via un provider OLE DB, un driver ODBC ou un accès natif).




2. Le contenu du fichier de données s'affiche dans WDMap.





<a name="NOTE3_3"></a>


### WDMap lancé depuis WINDEV ou WEBDEV ("Outils")
<a name="wdmap_lance_depuis_windev_webdev_outils_ELTPARAGRAPHE000096"></a>Pour visualiser les enregistrements présents dans un fichier de données :

1. Précisez si nécessaire le mot de passe de l'analyse (mot de passe en exécution défini dans la description de l'analyse).
	Remarque : L'application en cours sous WINDEV doit être associée à une analyse. Dans le cas contraire, les opérations à effectuer sont celles décrites dans le paragraphe [WDMap lancé directement par le programme "WDMAP.EXE"](#NOTE3_4).

2. Sélectionnez le fichier logique à visualiser.

3. Précisez si nécessaire :

	- Le mot de passe du fichier sélectionné (mot de passe défini soit à l'aide des fonctions [HPasse](../WDLang4/3044108.md), [HCréation](../WDLang4/3044255.md) ou [HCréationSiInexistant](../WDLang4/3044254.md), soit lors de la [modification automatique des données](../WDLang4/3044195.md)).

	- Le chemin complet de l'analyse 5.5 d'origine et le mot de passe de cette analyse (si le fichier sélectionné provient d'une analyse 5.5).

	- Les paramètres de connexion (si le fichier de données est manipulé via un provider OLE DB, un driver ODBC ou un accès natif).

	- Le chemin du fichier physique correspondant au fichier logique sélectionné.
			**Remarque** : Si l'application en cours gère le fichier ".REP", il est possible de simplifier la sélection du répertoire du fichier physique : cliquez sur le bouton "Fichier .REP" et sélectionnez le fichier ".REP" de l'application en cours. WDMAP listera automatiquement les chemins des fichiers physiques correspondant au fichier logique à visualiser. Pour plus de détails sur le fichier ".REP", consultez la [Gestion du ".REP"](../WDLang4/3044186.md).




4. Le contenu du fichier de données s'affiche dans WDMap.





<a name="NOTE3_4"></a>


### WDMap lancé directement par le programme "WDMAP.EXE"
<a name="wdmap_lance_directement_par_programme_wdmapexe_ELTPARAGRAPHE000136"></a>Pour visualiser les enregistrements présents dans un fichier de données :

1. Dans l'assistant, sélectionnez l'option "Visualiser un fichier" et passez à l'étape suivante. Un sélecteur de fichiers s'affiche.

2. Sélectionnez le fichier physique à visualiser (utilisez si nécessaire le bouton "Parcourir").

3. Précisez si nécessaire :

	- Le mot de passe du fichier sélectionné (mot de passe défini soit à l'aide des fonctions [HPasse](../WDLang4/3044108.md), [HCréation](../WDLang4/3044255.md) ou [HCréationSiInexistant](../WDLang4/3044254.md), soit lors de la [modification automatique des données](../WDLang4/3044195.md)).

	- Le chemin complet de l'analyse 5.5 d'origine et le mot de passe de cette analyse (si le fichier sélectionné provient d'une analyse 5.5).

	- Les paramètres de connexion (si le fichier est manipulé via un provider OLE DB, un driver ODBC ou un accès natif).




4. Le contenu du fichier de données s'affiche dans WDMap.

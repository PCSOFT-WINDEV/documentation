## Présentation
<a name="presentation_ELTTEXTE000185"></a>
WDMap est un utilitaire permettant de visualiser et de manipuler un fichier de données lors du développement d'une application. **Cet utilitaire est disponible uniquement en 32 bits.** 

WDMap peut être utilisé par exemple pour :

- Vérifier rapidement le contenu d'un enregistrement.

- Tester rapidement une application avec des données réalistes.




Le fichier de données manipulé peut être : 

- au format HFSQL (Classic ou Client/Serveur). 

- au format Hyper File 5.5

- accédé via un provider OLE DB, un driver ODBC ou un accès natif. 


Il est également possible de manipuler un alias d'un fichier de données HFSQL.

**Important** : L'utilitaire WDMap est disponible uniquement en 32 bits. Si WINDEV, WEBDEV ou WINDEV Mobile sont installés en 64 bits, WDMap est toujours exécuté en 32 bits. Lors d'un accès via un provider OLE DB, un driver ODBC ou un Connecteur Natif, il est nécessaire d'utiliser une des méthodes suivantes : 

- avoir un client 32 bits pour la base de données, 

- utiliser le [Centre de Contrôle HFSQL](../CCHF/3540001.md).

## Lancement de WDMap
<a name="lancement_wdmap_ELTTEXTE000215"></a>


### Lancement de WDMap
<a name="lancement_wdmap_ELTPARAGRAPHE000041"></a>WDMap peut être lancé :

- depuis l'éditeur d'analyses : sous le volet "Analyse", dans le groupe "Fichier", cliquez "Éditer avec WDMap",

- depuis le ruban : sous le volet "Outils", dans le groupe "Base de données", cliquez sur "WDMap",

- en lançant directement le programme "WDMAP.EXE".




**Remarque** : Si WDMap est lancé depuis l'éditeur d'analyses :

- Le fichier physique correspondant à la description du fichier de données sélectionné sera créé si nécessaire.

- Pour prendre en compte toutes les modifications effectuées sur la description du fichier de données, il est conseillé de générer l'analyse avant de visualiser le contenu du fichier de données.



<a name="NOTE2_2"></a>


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000081"></a>WDMap est un outil non redistribuable. WDMap ne peut pas être installé avec les applications développées avec WINDEV ou WEBDEV.

Pour utiliser WDMap , WINDEV, WEBDEV ou WINDEV Mobile doit obligatoirement être installé sur le poste en cours.

## Notes
<a name="notes_ELTTEXTE000269"></a>


- WDMap permet de visualiser un seul fichier de données à la fois.

- Si l'analyse est protégée par un mot de passe (mot de passe en exécution défini dans la description de l'analyse), ce mot de passe sera demandé lors de la sélection de l'analyse sous WDMap.

- Si le fichier de données est protégé par un mot de passe (mot de passe défini soit à l'aide des fonctions [HPasse](../WDLang4/3044108.md), [HCréation](../WDLang4/3044255.md) ou [HCréationSiInexistant](../WDLang4/3044254.md), soit lors de la [modification automatique des données](../WDLang4/3044195.md)), ce mot de passe sera demandé lors de la sélection du fichier physique sous WDMap.

- Il est possible de visualiser les données d'un fichier de données HFSQL Client/Serveur : par exemple, si les fichiers de l'analyse sont définis comme étant en Client/Serveur, avec une connexion, l'option "Editer le fichier de données avec WDMAP" ouvre le fichier de données du serveur selon la connexion définie.

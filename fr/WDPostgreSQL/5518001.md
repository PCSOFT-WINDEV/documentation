


## Connecteur Natif PostgreSQL pour WINDEV et WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000145"></a>
WINDEV et WEBDEV 28 proposent un Connecteur Natif à PostgreSQL (également appelé accès natif à PostgreSQL), disponible en 32 et 64 bits. Ce Connecteur Natif permet d'accéder à une base PostgreSQL sans drivers externes depuis un programme en WLangage.

**Attention** : Cette version du Connecteur Natif PostgreSQL ne fonctionne qu'avec WINDEV et WEBDEV version 28.

**Remarque** : Le Connecteur Natif PostgreSQL est un module optionnel et gratuit disponible en téléchargement sur [le site de PC SOFT](https://www.pcsoft.fr).

## Configuration nécessaire
<a name="configuration_necessaire_ELTTEXTE000169"></a>


### Conditions nécessaires à l'utilisation du Connecteur Natif PostgreSQL
<a name="conditions_necessaires_utilisation_connecteur_natif_postgresql_ELTPARAGRAPHE000029"></a>Il est conseillé d'utiliser une couche cliente PostgreSQL récente, de préférence de la même version que votre serveur.

Pour cela, vous devez :

- Copier les DLLs de la couche client PostgreSQL dans le répertoire d'installation de WINDEV ou WEBDEV du poste de développement. 
	Remarque : Il est nécessaire de copier également toutes les dépendances livrées avec la couche cliente.

- Copier les DLLs de la couche client PostgreSQL dans le répertoire de l'exécutable ou dans le répertoire de Windows du poste de déploiement. 
	Remarque : Il est nécessaire de copier également toutes les dépendances livrées avec la couche cliente. 

- Installer les DLLs du runtime Visual (package téléchargeable sur le site de Microsoft).
	Remarque : La version du runtime visual dépend de la couche cliente de PostgreSQL.




**En 32 bits**, en l'absence de ces dlls, le Connecteur Natif PostgreSQL utilisera une couche cliente intégrée correspondant à la version 7.4.23 de PostgreSQL. Elle contient toutes les fonctionnalités nécessaires au Connecteur Natif et est compatible avec les serveurs jusqu'à la version 9.6. Si vous utilisez un serveur dans une version supérieure, installez une version de la couche cliente compatible avec votre serveur. 

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Il est nécessaire d'installer la couche client PostgreSQL avec le gestionnaire de paquets de la distribution Linux. Les dépendances seront également installées. 
Exemple (Debian) : 

```txt
apt-get install libpq5
```


**Remarque** : Pour utiliser les connexions SSL, il faut installer une couche cliente PostgreSQL version 9.1 ou supérieure.
<a name="NOTE3_2"></a>


### Fichier spécifique livré avec le Connecteur Natif PostgreSQL
<a name="fichier_specifique_livre_avec_connecteur_natif_postgresql_ELTPARAGRAPHE000060"></a>Pour utiliser le Connecteur Natif PostgreSQL : 

- en 32 bits, seule la DLL ***wd280postgresql.DLL*** est nécessaire. Ce fichier est livré avec le Connecteur Natif PostgreSQL pour WINDEV et WEBDEV.

- en 64 bits, seule la DLL ***wd280postgresql64.DLL*** est nécessaire. Ce fichier est livré avec le Connecteur Natif PostgreSQL pour WINDEV et WEBDEV.

## Utilisation
<a name="utilisation_ELTTEXTE000199"></a>


### Utiliser le Connecteur Natif PostgreSQL dans vos applications WINDEV et sites WEBDEV
<a name="utiliser_connecteur_natif_postgresql_dans_vos_applications_windev_sites_webdev_ELTPARAGRAPHE000078"></a>Pour utiliser PostgreSQL en natif, il faut programmer dans l'application WINDEV ou dans le site WEBDEV :

- soit avec les fonctions "SQL" du WLangage (voir [Programmation à l'aide des fonctions SQL](../WDPostgreSQL/5518004.md)). 

- soit avec les fonctions "HFSQL" du WLangage (voir [Programmation à l'aide des fonctions HFSQL](../WDPostgreSQL/5518003.md)). 




**Remarques** : 

- N'oubliez pas de consulter les [limitations concernant l'utilisation de le Connecteur Natif PostgreSQL.](../WDPostgreSQL/5518002.md)

- L'option "Créer automatiquement les fichiers de données s'ils n'existent pas" (disponible dans l'onglet "Fichiers" de la description du projet) est prise en compte par le Connecteur Natif PostgreSQL.


## WDAdminHF : Gestion d'une base de données
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000179"></a>
L'administrateur distant HFSQL propose une gestion complète des bases de données HFSQL Client/Serveur. 

**Pour afficher les caractéristiques d'une base de données HFSQL Client/Serveur** : 

1. Connectez-vous au serveur HFSQL voulu à l'aide de l'administrateur HFSQL distant. 

2. Sélectionnez l'option "Bases de données". 

3. Sélectionnez la base de données voulue et cliquez sur le lien "Accéder à la base de données sélectionnée". 

4. Les options affichées sont les options disponibles sur la base de données en cours.

<a name="NOTE2_1"></a>


## Détail des options disponibles
<a name="detail_des_options_disponibles_ELTTEXTE000209"></a>


### Fichiers
<a name="fichiers_ELTPARAGRAPHE000026"></a>Cette option permet d'obtenir des informations générales sur les fichiers de la base de données : nom, taille, nombre d'enregistrements. Il est également possible de supprimer le fichier de données. 

En cliquant sur le nom du fichier de données, il est possible d'accéder aux options de [gestion des fichiers de données](../WDAdminHF/1000019703.md). 


### Connexions
<a name="connexions_ELTPARAGRAPHE000036"></a>Cette option liste les différents utilisateurs connectés sur la base de données en cours.

La liste des connexions actives contient les informations suivantes :

- le nom de l'utilisateur,

- le nom de la machine,

- le nom de l'application qui accède à la base, ...




Il est possible de gérer les connexions depuis cette page : interdire ou autoriser les connexions, déconnecter les utilisateurs, ou encore envoyer un message aux utilisateurs.


### Blocages
<a name="blocages_ELTPARAGRAPHE000049"></a>Cette option permet de gérer les blocages. Les informations affichées sont les suivantes :

- le nom du fichier de données sur lequel le blocage est posé,

- le numéro de l'enregistrement bloqué,

- le nom de l'utilisateur qui bloque,

- le nom de la machine qui bloque,

- l'adresse IP de la machine qui bloque,

- le nom de l'application qui bloque,

- la date de début de blocage,

- la durée du blocage.





### Sauvegardes
<a name="sauvegardes_ELTPARAGRAPHE000063"></a>Cette option permet de réaliser et de restaurer des sauvegardes des fichiers de données. Pour plus de détails, consultez [Réaliser des sauvegardes avec l'administrateur distant HFSQL](../WDAdminHF/1000019704.md). 


### Droits
<a name="droits_ELTPARAGRAPHE000071"></a>Cette option permet de gérer les droits sur le serveur pour l'utilisateur ou le groupe sélectionné.


### Maintenance
<a name="maintenance_ELTPARAGRAPHE000076"></a>Cette option permet de réindexer les fichiers de données de la base de données.

Pour effectuer une opération de maintenance sur les fichiers de données de la base :

1. Cochez dans la table les fichiers de données à traiter. Indiquez si nécessaire le mot de passe des fichiers à traiter.

2. Sélectionnez les options de réindexation : 

	- **Supprimer les enregistrements inactifs (supprimés ou rayés)**
			Si cette option est sélectionnée, les enregistrements rayés sont définitivement supprimés.
			Rappel : Lorsqu'un enregistrement est rayé, il est supprimé logiquement et pourra éventuellement être récupéré par la suite. L'enregistrement est encore présent dans le fichier de données. La suppression des enregistrements rayés supprime définitivement ces enregistrements et permet ainsi d'optimiser la taille du fichier de données.

	- **Supprimer les enregistrements endommagés**
			Si cette option est cochée, les enregistrements endommagés sont automatiquement supprimés.

	- **Densité de l'index**
			Correspond au taux de remplissage des index. Par défaut, ce taux a pour valeur 80. 
			Plus ce taux est important, plus l'index est dense et de petite taille. Dans ce cas, les parcours, recherches, filtres et requêtes sont plus rapides. Les ajouts d'enregistrements et les modifications d'enregistrements pourront être ralentis.
			Plus ce taux est faible, moins l'index sera dense et plus sa taille sera importante. Dans ce cas, les parcours, recherches, filtres et requêtes seront ralentis. Les ajouts d'enregistrements et les modifications d'enregistrements seront plus rapides.

	- **Alphabet**
			Par défaut, l'alphabet du fichier de données est conservé. Mais il est possible d'effectuer une ré-indexation en changeant l'alphabet du fichier de données. Dans ce cas, le nouvel alphabet sera pris en compte lors de la ré-indexation. Les tris, recherches, ... sur des clés de type chaîne (chaînes, caractères, date et heure) seront effectuées selon cet alphabet.

	- **Essayer de récupérer les données du mémo s'il est endommagé**
			Si cette option est sélectionnée, la réindexation tente de récupérer le mémo. Dans le cas contraire, les enregistrements endommagés sont récupérés sans le mémo associé.
			Attention : La récupération du mémo peut être partielle. Vérifiez vos fichiers de données.




3. Sélectionnez le type d'optimisation à réaliser : 

	- **1. Vérifier les index** : Vérifie la cohérence entre l'index et les fichiers de données.

	- **2. Optimiser la vitesse des index** (Recalculer les statistiques) : Optimise les index en calculant les statistiques sur les index. Ces statistiques permettent d'optimiser les filtres, les requêtes et les vues HFSQL.
			Remarque : Plus le fichier de données est modifié et moins les statistiques reflètent le contenu du fichier de données HFSQL. Plus le fichier de données contient d'enregistrements et moins la modification d'un enregistrement a d'impact sur les statistiques.

	- **3. Réindexer** : Optimise l'accès aux enregistrements du fichier de données et re-calcule les statistiques. Tous les enregistrements rayés sont automatiquement supprimés.

	- **4. Réviser les index et les mémos** : Optimise la totalité du fichier de données, de son index et ses mémos.










### Procédures stockées
<a name="procedures_stockees_ELTPARAGRAPHE000119"></a>Cette option permet de rechercher les collections de procédures et les procédures stockées. Il est possible de supprimer si nécessaire une collection de procédures. 


### Triggers
<a name="triggers_ELTPARAGRAPHE000124"></a>Cette option permet de visualiser les triggers définis sur la base de données et d'en ajouter si nécessaire. 


### Liaisons
<a name="liaisons_ELTPARAGRAPHE000129"></a>Cette option permet de visualiser les liaisons définies sur la base de données et d'en supprimer si nécessaire.

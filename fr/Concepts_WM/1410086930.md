
## 3. Test d'une application WINDEV Mobile

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Le test de l'ensemble du projet permet de simuler le lancement de l'application. Il est ainsi possible de tester l'application dans son ensemble, même si son développement n'est pas terminé.

Plusieurs types de tests du projet sont disponibles sous WINDEV Mobile :


|   |   |
| --- | --- |
| <br><br> | <br><br>- **Test sur le poste de développement**. Ce test réalise une simulation du périphérique mobile sur le poste de développement. Lors de ce test, aucune connexion à un périphérique n'est nécessaire. Ce test permet l'utilisation du débogueur. Cependant, le test étant réalisé sur un PC et non sur un l'appareil réel, certaines différences de comportement de l'application peuvent être notées.<br><br><br> |
| <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Logo_IOS.gif)<br> | <br><br>- **Test en réel (iPhone ou iPad)**. Il est possible de tester l'application iOS directement sur un appareil mobile. Il suffit de : <br><br>	- télécharger sur le périphérique l'application WMDEV (disponible sur le store). <br><br>	- scanner le code-barres affiché par WINDEV Mobile et correspondant à l'application à tester. <br><br><br> Cette option permet de tester directement l'application sur un appareil réel. Le débogueur est disponible. <br><br><br> |
| <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Logo_IOS.gif)<br> | <br><br>- **Test sur simulateur iPhone/iPad**. Il est possible de tester l'application iOS directement sur le Mac lors de la compilation du projet sous Xcode. Cette option évite de devoir disposer d'un véritable périphérique iPhone ou iPad pour réaliser les tests mais fournit une exécution plus fidèle que le mode "simulateur".<br><br><br> |
| <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=LOGO_Android.gif)<br> | <br><br>- **Test et débogage sur le mobile (Android) connecté au poste de développement**. Ce test permet l'utilisation du débogueur tout en étant réalisé directement sur le mobile.<br><br><br><br><br> |
| <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=LOGO_Android.gif)<br> | <br><br>- **Test sur l'émulateur Android**. Le SDK Android est livré avec un émulateur d'appareil Android. Il est possible de tester l'application dans l'émulateur. Cette option évite de devoir disposer d'un véritable périphérique Android pour réaliser les tests mais fournit une exécution plus fidèle que le mode "simulateur".<br><br><br> |
| <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=logo_Windows_10.gif)<br> | <br><br>- **Test en réel (PC ou appareil mobile)**. Il est possible de tester l'application Universal Windows 10 App en réel sur un poste Windows 10. Cette option permet d'exécuter votre projet en mode tuile, directement depuis Windows 10. L'application Universal Windows 10 App est automatiquement générée.<br><br><br> |

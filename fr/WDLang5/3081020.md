


## XMLTrouve (Fonction)

***En anglais : XMLFound***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Vérifie si l'élément en cours correspond au parcours ou à la recherche en cours.








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Rechercher la balise qui commence par "pri"
XMLRecherche("DocXML", "pri", XMLBalise + XMLSousElément, XMLCommencePar)
TANTQUE XMLTrouve("DocXML") = Vrai
	Info("Balise trouvée " + XMLNomElément("DocXML"))
	// Elément suivant dans la recherche
	XMLSuivant("DocXML")
FIN
```
<a name="Exemple2"></a>

```wl
// Parcours d'un document
XMLPremier("DocXML")
// Balise trouvée ?
SI XMLTrouve("DocXML") = Vrai ALORS
	Info("Balise courante : " + XMLNomElement("DocXML"))
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = XMLTrouve(<Document XML>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si une fonction de parcours XML a pu se positionner sur un élément, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire (la position en cours dans le document XML n'est alors pas modifiée). La fonction **XMLTrouve** renvoie toujours le résultat opposé de la fonction [XMLEnDehors](../WDLang5/3081009.md).




**`<Document XML> : (Chaîne de caractères)`**

Nom du document XML manipulé. Ce document contient le code XML à analyser et a été créé avec la fonction [XMLDocument](../WDLang5/3081008.md).

## Remarques


### Fonctions modifiant XMLTrouve
<a name="fonctions_modifiant_xmltrouve_ELTPARAGRAPHE000085"></a>Les fonctions suivantes modifient la valeur retournée par **XMLTrouve** : 

- [XMLPremier](../WDLang5/3081024.md), [XMLDernier](../WDLang5/3081013.md),

- [XMLSuivant](../WDLang5/3081030.md), [XMLPrécédent](../WDLang5/3081023.md),

- [XMLFils](../WDLang5/3081012.md), [XMLParent](../WDLang5/3081022.md), [XMLRacine](../WDLang5/3081019.md),

- [XMLRecherche](../WDLang5/3081025.md).



<a name="NOTE0_2"></a>


### Fonctions XML et threads
<a name="fonctions_xml_threads_ELTPARAGRAPHE000248"></a>Si votre application utilise des threads, le document XML est partagé entre tous ces threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads](../WDLang1/3077026.md).Si la position courante dans un document XML est modifiée dans un thread, la position courante dans ce document XML est modifiée pour tous les threads.

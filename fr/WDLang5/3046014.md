
## iPolice (Fonction)

***En anglais : iFont***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Sélectionne la police par défaut. Cette police sera utilisée par les fonctions [iImprime](../WDLang5/3046031.md), [iImprimeMot](../WDLang5/3046034.md), [iLargeurTexte](../WDLang5/3046033.md), [iHauteurTexte](../WDLang5/3046070.md), etc.

**Remarques** :

- Cette police doit avoir été préalablement créée avec la fonction [iCréePolice](../WDLang5/3046045.md).

- Cette police est active tant que la fonction **iPolice** n'est pas réutilisée.

- ![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Seules les polices TrueType sont manipulables.






![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Imprimer sous Linux](../Editeurs/9000078.md). 




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
iImprimeMot(iPolice(2) + CL.Nom + iPosX(200) + CL.Prénom)
iSauteLigne
iImprime(iPolice(1) + iPosX(100) + CL.Société)
iFinImprime
```

## Syntaxe

### Utilisation d'une police dynamique

`<Résultat> = iPolice(<Nom de la police> [, <Calcul immédiat>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Chaîne de contrôle décrivant l'utilisation de la police spécifiée.

**`<Nom de la police> : (Police)`**

Nom de la police à utiliser par défaut. Ce paramètre correspond à une variable de type Police. Les caractéristiques de la police ont été définies grâce à la fonction [PoliceCrée](../WDLang1/3060002.md) ou grâce aux [propriétés de gestion des polices](../Proprietes/2515045.md).

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Seules les polices TrueType sont manipulables.

**`<Calcul immédiat> : (Booléen optionnel)`**

Permet de spécifier si la sélection de la police doit être réalisée immédiatement (<u><u><u><u>Vrai</u></u></u></u> par défaut) ou doit être effectuée uniquement lors de l'impression (<u><u><u><u>Faux</u></u></u></u>, cas de combinaison de polices par exemple). Pour plus de détails, consultez [les remarques](#NOTE0_1).

### Combinaison de polices
<a name="combinaison_polices_ELTPARAGRAPHE000160"></a>Lors de la combinaison de polices, des effets inattendus peuvent être observés. Par exemple, le code :


```wl
iPolice(2)
iImprime("Première partie" + iPolice(1) + "Seconde partie")
```
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Police_2.GIF)


n'est pas équivalent à :


```wl
iImprime(iPolice(2) + "Première partie" + iPolice(1) + "Seconde partie")
```
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Police_1.GIF)


Dans le premier cas, toute la chaîne de caractères sera imprimée en police 1. En effet, dans ce cas, la fonction **iPolice**(1) est exécutée lors de la construction de la chaîne à imprimer et donc avant l'impression de la chaîne "Première partie".

Pour obtenir un résultat identique, il suffit d'utiliser (dans la première syntaxe) la fonction **iPolice** avec le paramètre <u><u><u><u>Faux</u></u></u></u> : la fonction **iPolice**(2, Faux) ne sera réellement exécutée que lors de l'impression.




## SFExécuteTraitement (Fonction)

***En anglais : SFExecuteProcess***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Demande l'exécution d'un traitement spécifique. Il est possible de :

- Soumettre des enregistrements au processus d'approbation.

- Effectuer une action d'approbation.







<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = SFExécuteTraitement(<Connexion Salesforce> , <Traitement à effectuer>)`
---

**`<Résultat> : (Variable de type Tableau)`**

Tableau de variables de type [sfRésultatExécuteTraitement](../WDLang5/1000018559.md) contenant les données résultat.

**`<Connexion Salesforce> : (Variable de type sfConnexion)`**

Nom de la variable de type [sfConnexion](../WDLang5/1000017579.md) à utiliser

**`<Traitement à effectuer> : (Tableau de variants)`**

Tableau de variants contenant les traitements à exécuter. Ce tableau peut contenir des objets de type :

- [sfRequêteTraitementTâche](../WDLang5/1000018520.md).

- [sfRequêteTraitementSoumission](../WDLang5/1000018516.md).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Cas d'erreurs possibles : 
<a name="cas_erreurs_possibles_ELTPARAGRAPHE000074"></a>

- La connexion Internet n'est pas valide.

- L'authentification n'a pas pu être réalisée correctement.

- La réponse XML du serveur n'est pas lisible.




Pour savoir si une erreur est survenue, utilisez la variable [ErreurDétectée](../WDLang1/3087001.md).

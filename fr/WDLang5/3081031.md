


## XMLTypeElement (Fonction)

***En anglais : XMLElementType***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le type de l'élément en cours (balise ou attribut). Pour sélectionner un élément, utilisez les fonctions de recherche et de parcours XML ([XMLPremier](../WDLang5/3081024.md), [XMLDernier](../WDLang5/3081013.md), [XMLFils](../WDLang5/3081012.md), ...).








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
XMLPremier("DocXML")
SELON XMLTypeElement("DocXML")
	CAS XMLBalise : Info(XMLNomElément("DocXML") + " est de type XMLBalise")
	CAS XMLAttribut : Info(XMLNomElément("DocXML") + " est de type XMLAttribut")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = XMLTypeElement(<Document XML>)`
---

**`<Résultat> : (Constante de type Entier)`**

Type de l'élément en cours.


|   |   |
| --- | --- |
| *XMLAttribut* | L'élément en cours est un attribut. |
| *XMLBalise* | L'élément en cours est une balise. |
| *XMLErreur* | Aucun élément n'est en cours de lecture. La variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. |



**`<Document XML> : (Chaîne de caractères)`**

Nom du document XML manipulé. Ce document contient le code XML à analyser et a été créé avec la fonction [XMLDocument](../WDLang5/3081008.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Caractéristiques de l'élément en cours
<a name="caracteristiques_element_cours_ELTPARAGRAPHE000099"></a>

- La valeur de l'élément en cours peut être connue avec la fonction [XMLDonnée](../WDLang5/3081004.md).

- Le nom de l'élément en cours peut être connu avec la fonction [XMLNomElément](../WDLang5/3081026.md).



<a name="NOTE0_2"></a>


### Fonctions XML et threads
<a name="fonctions_xml_threads_ELTPARAGRAPHE000240"></a>Si votre application utilise des threads, le document XML est partagé entre tous ces threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads](../WDLang1/3077026.md).

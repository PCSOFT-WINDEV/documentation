
## iPoliceDidot (Fonction)

***En anglais : iDidotFont***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Sélectionne l'unité à utiliser pour la hauteur des polices (créées avec la fonction [iCréePolice](../WDLang5/3046045.md)) : point DIDOT ou point PICA.

Par défaut, le point PICA est utilisé en standard.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
iPoliceDidot(Vrai)
// Crée une police de 48 points Didot
iCréePolice(1, 48, iGras + iItalique, iRoman)
iPoliceDidot(Faux)
// Crée une police de 32 points Pica
iCréePolice(2, 32, iGras + iItalique, iRoman)
// Imprime un texte avec cette police
iImprime(iPolice(1) + "Texte de taille 48 points Didot")
// Termine l'impression
iFinImprime()
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`iPoliceDidot([<Unité Didot>])`
---

**`<Unité Didot> : (Booléen optionnel)`**

Nouvelle unité à utiliser :

- <u><u><u><u>Vrai</u></u></u></u> (valeur par défaut) : point Didot.

- <u><u><u><u>Faux</u></u></u></u> : point Pica.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- En principe, la fonction **iPoliceDidot** ne sera appelée qu'une seule fois, en début de projet.

- 1 point PICA = 0,353 mm et 1 point DIDOT = 0,3759 mm (valeurs approximatives).

- La fonction **iPoliceDidot** n'a aucun effet : 

	- sur les impressions réalisées avec l'éditeur d'états. 

	- sur les impressions PDF. 
			Ces impressions utilisent le point PICA comme unité standard.

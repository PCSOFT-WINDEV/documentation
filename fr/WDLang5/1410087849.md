
## HTMLActiveDébogueur (Fonction)

***En anglais : HTMLEnableDebugger***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active le débogueur HTML/Javascript sur les champs HTML basé sur CEF (Chromium) : champ d'affichage HTML ou champ Editeur HTML. 

Pour déboguer les champs, il suffira de : 

- Lancer le navigateur Chrome puis de taper l'adresse suivante : 
	
	```txt
	http://localhost:8088/
	```


- Cliquer sur un des liens apparaissant dans chrome (1 lien par champ). 



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code d'initialisation du projet
SI ChargeParamètre("Debug") = "1" ALORS
      HTMLActiveDébogueur(8081)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`HTMLActiveDébogueur(<Numéro de port>)`
---

**`<Numéro de port> : (Entier)`**

Numéro de port utilisé pour le débogage. Par défaut, ce numéro correspond à 8088. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Cette fonction doit être appelée avant la création de tout champ HTML (champ d'affichage HTML ou champ Editeur HTML), lors de l'initialisation du projet par exemple. 

- Le débogueur HTML/Javascript (Devtools) peut être affiché avec l'URL suivante : 
	
	```txt
	http://nom_machine:num_port/
	```
Par exemple : 
	
	```txt
	http://localhost:8088/
	```


- Le débogueur est activé par défaut en GO mais il n'est pas activé durant l'exécution de l'application. Il permet par exemple de tracer le code Javascript ajouté avec la fonction [ExécuteJS](../WDLang1/1000023014.md).

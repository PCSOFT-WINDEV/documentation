


## XMLPremier (Fonction)

***En anglais : XMLFirst***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Positionne sur le premier élément (balise ou attribut) du niveau en cours de l'arborescence.

En général, la fonction **XMLPremier** permet de se positionner dans le document XML pour effectuer une boucle de lecture avec la fonction [XMLSuivant](../WDLang5/3081030.md).
![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Les parcours sont effectués dans l'ordre alphabétique des attributs.








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Positionne sur le premier élément
XMLPremier("DocXML")
TANTQUE XMLEnDehors("DocXML") = Faux
	// Traitements des données
	XMLSuivant("DocXML")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = XMLPremier(<Document XML>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si un élément a été trouvé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. En cas d'erreur : 

	- la position courante reste inchangée. 

	- la variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. Pour obtenir plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).







**`<Document XML> : (Chaîne de caractères)`**

Nom du document XML manipulé. Ce document contient le code XML à analyser et a été créé avec la fonction [XMLDocument](../WDLang5/3081008.md).

## Remarques


### Manipulation des éléments dans le code XML
<a name="manipulation_des_elements_dans_code_xml_ELTPARAGRAPHE000104"></a>Après la fonction **XMLPremier**, l'élément en cours peut être manipulé avec les fonctions [XMLNomElément](../WDLang5/3081026.md), [XMLNomParent](../WDLang5/3081035.md), [XMLTypeElément](../WDLang5/3081031.md) et [XMLDonnée](../WDLang5/3081004.md).
<a name="NOTE0_2"></a>


### Utilisation de la fonction XMLDernier lors d'une recherche
<a name="utilisation_fonction_xmldernier_lors_une_recherche_ELTPARAGRAPHE000126"></a>Si une recherche est en cours (fonction [XMLRecherche](../WDLang5/3081025.md)), la fonction **XMLPremier** annule cette recherche.
<a name="NOTE0_3"></a>


### Fonctions XML et threads
<a name="fonctions_xml_threads_ELTPARAGRAPHE000273"></a>Si votre application utilise des threads, le document XML est partagé entre tous ces threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads](../WDLang1/3077026.md).Si la position courante dans un document XML est modifiée dans un thread, la position courante dans ce document XML est modifiée pour tous les threads.




## XMLFils (Fonction)

***En anglais : XMLChild***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Positionne sur le premier élément fils de l'élément en cours. Pour sélectionner un élément, utilisez les fonctions de recherche et de parcours XML ([XMLPremier](../WDLang5/3081024.md), [XMLDernier](../WDLang5/3081013.md), ...).

**Rappel** : Un élément correspond soit à une balise soit à un attribut.

![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Les parcours sont effectués dans l'ordre alphabétique des attributs.








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003081012&name=xmlfils_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Création du document XML
XMLDocument("CdeXML", SourceMonXML)
// Parcours effectué avec XMLPremier, XMLSuivant
// Positionne sur l'élément "<commande>"
XMLPremier("CdeXML")
// Descend d'un niveau et positionne sur l'élément "<numero>"
XMLFils("CdeXML")
// Récupère le numéro de la commande
NumCde est un entier = Val(XMLDonnée("CdeXML")) // contient 1
```

## Syntaxe

`<Résultat> = XMLFils(<Document XML>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si un élément a été trouvé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. En cas d'erreur : 

	- la position courante reste inchangée. 

	- la variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. Pour obtenir plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).







**`<Document XML> : (Chaîne de caractères)`**

Nom du document XML manipulé. Ce document contient le code XML à analyser et a été créé avec la fonction [XMLDocument](../WDLang5/3081008.md).

## Remarques


### Manipulation des éléments dans le code XML
<a name="manipulation_des_elements_dans_code_xml_ELTPARAGRAPHE000110"></a>Après l'utilisation de la fonction **XMLFils**, l'élément en cours peut être manipulé avec les fonctions [XMLNomElément](../WDLang5/3081026.md), [XMLNomParent](../WDLang5/3081035.md), [XMLTypeElément](../WDLang5/3081031.md) et [XMLDonnée](../WDLang5/3081004.md).

**Remarque** : il est possible d'utiliser la fonction [XMLExtraitDocument](../WDLang5/3081010.md) pour parcourir uniquement le contenu d'une balise et être averti de la fin du parcours avec la fonction [XMLEnDehors](../WDLang5/3081009.md).
<a name="NOTE0_2"></a>


### Utilisation de la fonction XMLFils lors d'une recherche
<a name="utilisation_fonction_xmlfils_lors_une_recherche_ELTPARAGRAPHE000140"></a>Si une recherche est en cours (fonction [XMLRecherche](../WDLang5/3081025.md)), la fonction **XMLFils** annule cette recherche.
<a name="NOTE0_3"></a>


### Fonctions XML et threads
<a name="fonctions_xml_threads_ELTPARAGRAPHE000287"></a>Si votre application utilise des threads, le document XML est partagé entre tous ces threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads](../WDLang1/3077026.md).Si la position courante dans un document XML est modifiée dans un thread, la position courante dans ce document XML est modifiée pour tous les threads.




## XMLParent (Fonction)

***En anglais : XMLParent***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Positionne sur l'élément parent de l'élément en cours (balise ou attribut). Pour sélectionner un élément, utilisez les fonctions de recherche et de parcours XML ([XMLPremier](../WDLang5/3081024.md), [XMLDernier](../WDLang5/3081013.md), [XMLFils](../WDLang5/3081012.md), ...).








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003081022&name=xmlparent_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Parcours effectué avec XMLPremier, XMLSuivant ...
// Positionne sur l'élément "<commande>"
XMLDernier("CdeXML")
// remonte d'un niveau et positionne sur l'élément "<numero>"
XMLParent("CdeXML")
// Récupère le numéro de la commande
NumCde est un entier = Val(XMLDonnée("DocXML")) // contient 1
// Positionne sur l'élément suivant
XMLSuivant("CdeXML")
// Récupère la date de la commande
DateCde est une chaîne = XMLDonnée("CdeXML") // "20/11/2012"
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = XMLParent(<Document XML>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si un élément a été trouvé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. En cas d'erreur : 

	- La position courante reste inchangée. 

	- La variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. Pour obtenir plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).







**`<Document XML> : (Chaîne de caractères)`**

Nom du document XML manipulé. Ce document contient le code XML à analyser et a été créé avec la fonction [XMLDocument](../WDLang5/3081008.md).

## Remarques


### Manipulation des éléments dans le code XML
<a name="manipulation_des_elements_dans_code_xml_ELTPARAGRAPHE000099"></a>Après la fonction **XMLParent**, l'élément en cours peut être manipulé avec les fonctions [XMLNomElément](../WDLang5/3081026.md), [XMLNomParent](../WDLang5/3081035.md), [XMLTypeElément](../WDLang5/3081031.md) et [XMLDonnée](../WDLang5/3081004.md).
<a name="NOTE0_2"></a>


### Utilisation de XMLParent lors d'une recherche
<a name="utilisation_xmlparent_lors_une_recherche_ELTPARAGRAPHE000121"></a>Si une recherche est en cours, **XMLParent** annule cette recherche (voir la fonction [XMLRecherche](../WDLang5/3081025.md)).
<a name="NOTE0_3"></a>


### Fonctions XML et threads
<a name="fonctions_xml_threads_ELTPARAGRAPHE000268"></a>Si votre application utilise des threads, le document XML est partagé entre tous ces threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads](../WDLang1/3077026.md).Si la position courante dans un document XML est modifiée dans un thread, la position courante dans ce document XML est modifiée pour tous les threads.


## iTraitV (Fonction)

***En anglais : iVLine***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Imprime un trait vertical aux coordonnées spécifiées. Contrairement à la fonction [iLigne](../WDLang5/3046035.md), le bout du trait est carré (et non arrondi). Le trait est tracé à l'emplacement du curseur (paramétré grâce à la fonction [iPosX](../WDLang5/3046015.md)).

L'impression n'est pas lancée (seule la fonction [iFinImprime](../WDLang5/3046053.md) permet de lancer l'impression).

**Remarque** : L'origine (0,0) est située dans le coin supérieur gauche de la feuille. Elle tient compte des marges physiques de l'imprimante.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Trace un trait vertical entre la ligne 5 (mm)
// et la ligne 10 (mm), d'épaisseur 2 points Pica
iTraitV(5, 10, 2)
iFinImprime()
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = iTraitV(<Y1> , <Y2> [, <Epaisseur> [, <Couleur> [, <Style>]]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le trait a été imprimé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Y1> : (Réel)`**

Coordonnée verticale (ligne ou ordonnée) du début du trait (en millimètres). Valeur comprise entre 0 et la hauteur de la page (donnée par [iHauteurPage](../WDLang5/3046052.md)).

**`<Y2> : (Réel)`**

Coordonnée verticale (ligne ou ordonnée) de la fin du trait (en millimètres). Valeur comprise entre 0 et la hauteur de la page (donnée par [iHauteurPage](../WDLang5/3046052.md)).

**`<Epaisseur> : (Réel optionnel)`**

Epaisseur du trait (en points pica). Par défaut, cette épaisseur est égale à 1 (un point Pica = 0,353 mm).
Pour obtenir une épaisseur de 1 pixel, &lt;Épaisseur&gt; doit être égale à 0. Lorsque l'épaisseur augmente, elle est répartie proportionnellement des deux côtés du trait.

**`<Couleur> : (Entier optionnel)`**

Couleur utilisée pour le trait. Si ce paramètre n'est pas précisé, la couleur utilisée est le noir. 
Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md).




**`<Style> : (Constante optionnelle de type entier)`**

Type du trait.


|   |   |
| --- | --- |
| *iContinu*<br>(valeur par défaut) | Trait continu |
| *iMixte* | Trait mixte (-.-.) |
| *iPointillé* | Trait pointillé (....) |
| *iTiret* | Trait tiret (-----) |


![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) **Attention** : Pour avoir un trait en pointillé, il faut que le paramètre &lt;Epaisseur&gt; soit égal à 0. Si &lt;Epaisseur&gt; est supérieure ou égal à 1, le type du trait sera toujours continu. Ce paramètre peut être ignoré si le driver d'imprimante ne gère pas ce paramètre.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le trait est tracé à la position horizontale du curseur (paramétrée avec la fonction [iPosX](../WDLang5/3046015.md)).

- La position du curseur n'est pas modifiée par le tracé du trait.

- L'utilisation du type de trait pointillé (constante *iPointillé*) ou mixte (constante *iMixte*) combinée avec un cadre épais peut provoquer des effets disgracieux.


## iEnchaînementAjouteDiagramme (Fonction)

***En anglais : iSequencingAddDiagram***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute une impression d'un document de type Diagramme dans un enchaînement d'impressions d'états. 

Remarque : L'enchaînement d'impressions d'états permet d'imprimer des états différents à la suite, visibles comme une seule impression dans le visualisateur de rapports. Il est possible d'ajouter divers types de documents dans cet enchaînement. Pour plus de détails, consulter [Enchaîner des états](../WDChamp/9000128.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MonDiagramme est un Diagramme = "c:\temp\Diagramme_archi.wddiag"

// Impression dans le visualisateur de rapports
iDestination(iVisualisateur)
// Ajout des états dans l'enchaînement
iEnchaînementAjoute(ETAT_Etat1)
iEnchaînementAjoute(ETAT_Etat2, 3)
// Ajout du diagramme
iEnchaînementAjouteDiagramme(MonDiagramme)
iEnchaînementImprime()
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`iEnchaînementAjouteDiagramme(<Diagramme>)`
---

**`<Diagramme> : (Type de l'élément)`**

Diagramme à imprimer. Ce document correspond : 

- au nom d'un champ Editeur de diagrammes. 

- à une variable de type [Diagramme](../WDLang1/1410088055.md).

- au chemin d'un fichier de type Diagramme (fichier wddiag).  

- à un mémo HFSQL correspondant à un document de type Diagramme.  

- à une variable de type Buffer correspondant à un document de type Diagramme.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Les documents de type Diagramme sont réellement ajoutés lors de l'appel de la fonction [iEnchaînementImprime](../WDLang5/1000019903.md). 

- Dans cette version, les états et les documents enchaînés conservent leur propre numérotation.

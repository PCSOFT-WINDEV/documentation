
## iRAZNbPages (Fonction)

***En anglais : iResetNbPages***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Réinitialise le nombre de pages courant, utilisé dans les champs prédéfinis de type "Nombre de pages" et "Numéro et nombre de pages". 

Cette fonction peut être utilisée par exemple lors de l'impression d'une série de factures, pour numéroter les pages de chaque facture de façon indépendante.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Dans l'événement "Après impression du bloc de bas de rupture" 
// Réinitialise le nombre de pages pour que le nombre de pages soit affiché par rupture
iRAZNbPages()
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`iRAZNbPages()`
---



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Cette fonction ne doit être appelée que dans les états générés par l'éditeur d'états. Elle ne doit pas être appelée dans les impressions programmées (fonctions iImprimeXXX).

- Cette fonction peut être appelée au milieu d'enchaînement d'états (fonctions [iEnchaînementAjoute](../WDLang5/1000020059.md), [iEnchaînementImprime](../WDLang5/1000019903.md)).

- Cette fonction est équivalente à l'option "Réinitialiser le nombre de pages après impression du bloc" présente dans l'onglet "UI" de la fenêtre de description d'un bloc.

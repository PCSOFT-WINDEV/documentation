


## XMLEnDehors (Fonction)

***En anglais : XMLOut***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de savoir si l'élément (balise ou attribut) sur lequel on veut se positionner est en dehors du document XML.

L'utilisation de la fonction **XMLEnDehors** n'est pas obligatoire. En effet, les fonctions de parcours XML renvoient un résultat qui permet de savoir si elles ont réussi ou échoué. L'utilisation de la fonction **XMLEnDehors** est conseillée pour plus de clarté dans votre code.








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Lecture du premier élément
XMLPremier("DocXML")

// Balise trouvée ?
SI XMLEnDehors("DocXML") = Faux ALORS
	Info("Balise courante : " + XMLNomElement("DocXML"))
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = XMLEnDehors(<Document XML>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si une fonction de parcours n'a pas pu se positionner sur un élément, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire (la position en cours dans le document XML n'est alors pas modifiée). 
	La fonction **XMLEnDehors** renvoie toujours le résultat opposé de la fonction [XMLTrouve](../WDLang5/3081020.md).




**`<Document XML> : (Chaîne de caractères)`**

Nom du document XML manipulé. Ce document contient le code XML à analyser et a été créé avec la fonction [XMLDocument](../WDLang5/3081008.md).

## Remarques


### Fonctions modifiant XMLEnDehors
<a name="fonctions_modifiant_xmlendehors_ELTPARAGRAPHE000093"></a>Les fonctions suivantes modifient la valeur retournée par la fonction **XMLEnDehors** :

- [XMLPremier](../WDLang5/3081024.md), [XMLDernier](../WDLang5/3081013.md),

- [XMLSuivant](../WDLang5/3081030.md), [XMLPrécédent](../WDLang5/3081023.md),

- [XMLFils](../WDLang5/3081012.md), [XMLParent](../WDLang5/3081022.md), [XMLRacine](../WDLang5/3081019.md),

- [XMLRecherche](../WDLang5/3081025.md).



<a name="NOTE0_2"></a>


### Fonctions XML et threads
<a name="fonctions_xml_threads_ELTPARAGRAPHE000256"></a>Si votre application utilise des threads, le document XML est partagé entre tous ces threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads](../WDLang1/3077026.md).Si la position courante dans un document XML est modifiée dans un thread, la position courante dans ce document XML est modifiée pour tous les threads.

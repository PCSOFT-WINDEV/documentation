
## Centre de Suivi de Projets : Gestion des sprints
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000171"></a>
Un Sprint est un cycle de développement (également appelé "itération"). En développement Agile (Scrum), un projet est découpé en un ensemble de sprints. 

Chaque sprint comprend des exigences, tâches ou incidents à réaliser entre deux dates. 

Dès que l'ensemble des éléments liés au sprint est réalisé, le sprint peut être défini comme terminé. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000195"></a>


### Créer un sprint
<a name="creer_sprint_ELTPARAGRAPHE000020"></a>La création d'un sprint s'effectue depuis le gestionnaire de sprints. 

**Pour créer un sprint** : 

1. Affichez ou créez un onglet de type "Sprint". 

2. Dans le ruban de l'onglet, dans le groupe "Sprints", cliquer sur "Nouveau sprint". 

3. Indiquez les caractéristiques du sprint à créer : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0001.gif)


	- Nom du sprint, 

- Objectif,

- Responsable,

- Durée avec une date de début.

4. Validez. Le sprint est créé et apparaît à gauche de la fenêtre. 




**Pour associer des éléments (items) au sprint** : 

1. Dans le ruban de l'onglet, dans le groupe "Backlog", cliquez sur l'élément à ajouter : Exigences, Tâches ou Incidents. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0002.gif)


2. Une liste correspondant à l'élément sélectionné apparaît dans la partie basse de la fenêtre. 

3. Utilisez les filtres pour sélectionner l'item voulu. 

4. Affichez si nécessaire le volet "Items du sprint". ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0003%204.gif)


5. Glissez/déposez les éléments voulus vers le volet "Items du sprint". L'item sélectionné disparaît de la partie basse de la fenêtre (backlog) pour être affichée dans le volet "Items du sprint". 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0004.gif&type=thumb)



Remarque : Il est également possible d'utiliser les boutons suivants pour lier les différents éléments au sprint en cours : 

- ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0003%201.gif)
 pour lier une nouvelle exigence ou une exigence existante. 

- ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0003%202.gif)
 pour lier une nouvelle tâche ou une tâche existante. 

- ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0003%203.gif)
 pour lier un nouvel incident ou un incident existant. 




L'onglet "Tableau de bord" permet de voir les différents éléments du sprint sélectionné : date de fin, avancement, etc. Comme pour le tableau de bord général, les différents widgets affichés sont paramétrables. 
<a name="NOTE2_2"></a>


### Modifier un sprint
<a name="modifier_sprint_ELTPARAGRAPHE000062"></a>Pour modifier : 

- la description du sprint : 

	1. Sélectionnez le sprint voulu dans l'onglet "Sprints". 

	2. Dans le ruban, dans le groupe "Sprints", cliquez sur "Editer". 

	3. Effectuez les modifications voulues. 

	4. Validez. 




- les items associés au sprint : 

	1. Sélectionnez l'item voulu dans le volet "Items du projet". Il est possible d'éditer, de supprimer l'item sélectionné. 

	2. Ajoutez si nécessaire de nouveaux items grâce aux boutons d'ajout.  






<a name="NOTE2_3"></a>


### Terminer un sprint
<a name="terminer_sprint_ELTPARAGRAPHE000080"></a>La terminaison d'un sprint est une tâche obligatoirement manuelle, qui ne peut être automatisée. 

Pour terminer un sprint : 

1. Sélectionnez le sprint à terminer. 

2. Dans le ruban de l'onglet, dans le groupe "Sprints", cliquez sur "Terminer".

3. Si certains items du sprint ne sont pas terminés, une fenêtre s'affiche permettant d'effectuer une action sur les éléments non terminés. Il est par exemple possible de déplacer une tâche non terminée vers un nouveau sprint. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0007.gif&type=thumb)


4. Validez la fenêtre. 

5. Le sprint est terminé. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0008.gif&type=thumb)





Remarque : Lorsqu'un sprint est terminé, l'option "Editer" permet également d'afficher la fenêtre de terminaison de sprint.

### Planning des sprints
<a name="planning_des_sprints_ELTPARAGRAPHE000099"></a>Pour visualiser le planning des sprints : 

1. Dans le ruban, dans le groupe "Planning", cliquez sur "Planning des sprints". 

2. La partie basse de l'onglet affiche le planning des sprints affiche le planning sous forme de diagramme de Gantt. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=cc_suivi_sprint%20-%20HC%20N%B0005.gif&type=thumb)




<a name="NOTE2_5"></a>


### Tableau de bord des sprints
<a name="tableau_bord_des_sprints_ELTPARAGRAPHE000110"></a>Dans le volet "Sprints", l'onglet "Tableau de bord" affiche les caractéristiques du sprint sous forme de Widgets. Les différents Widgets disponibles sont : 

- Date de fin du sprint. 

- Avancement du sprint. 

- Incidents ouverts. 

- Répartition des incidents ouverts. 

- Incidents en attente. 

- Répartition des durées et des tâches par intervenant pour le sprint. 

- Items du sprint. 

- Exigences du sprint. 




Remarque : Les widgets "Exigences" et "Répartition des durées des tâches" s'adaptent en fonction du mode de visualisation choisi (vision "Général", "Développement", "test" ou "Documentation" sélectionné dans le ruban de l'onglet "Sprints").

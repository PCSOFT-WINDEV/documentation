


## Je suis développeur : Comment gérer mes tâches et mon planning ? 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000274"></a>
Grâce aux Centres de Contrôle, WINDEV, WEBDEV et WINDEV Mobile proposent plusieurs outils permettant à un développeur de gérer son planning de développement et les différentes tâches associées à ce planning.

Pour gérer les tâches et le planning, il faut :

1. [Créer la base de données des Centres de Contrôle et de la configurer (si ce n'est déjà fait)](#NOTE2).

2. [Paramétrer le Centre de Suivi de Projets (horaires, congés, ...)](#NOTE3).

3. [Travailler sur une tâche](#NOTE4).

4. [Travailler sur un incident](#NOTE5).




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Etape 1 : Base de données des Centres de Contrôle
<a name="etape_1_base_donnees_des_centres_controle_ELTTEXTE000298"></a>
La base de données des Centres de Contrôle regroupe de nombreuses informations partagées par différents modules présents dans WINDEV, WEBDEV et WINDEV Mobile.

Le paramétrage de la base de données des Centres de Contrôle consiste à indiquer l'emplacement de cette base de données et le format des fichiers utilisé. **Cette étape est très importante**.

Pour plus de détails, consultez [Configurer la base de données des Centres de Contrôle](../CCSuivi/3540720.md).

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Etape 2 : Paramétrer le Centre de Suivi de Projets
<a name="etape_2_parametrer_centre_suivi_projets_ELTTEXTE000322"></a>
Pour gérer les tâches et le planning, il est nécessaire de lancer et de paramétrer le Centre de Suivi de Projets.

**Remarque** : Dans cette partie, seule la configuration nécessaire à l'utilisation des tâches et du planning est abordée. On considère que les intervenants, le planning, les tâches et les exigences ont été mises en place par le Chef de projet. Pour plus de détails, consultez [Comment gérer un projet de développement](../CCSuivi/3540722.md).

**Pour lancer le Centre de Suivi de Projets** :

1. Lancez le Centre de Suivi de Projets depuis WINDEV, WEBDEV ou WINDEV Mobile : sous le volet "Outils", dans le groupe "Centres de Contrôle", cliquez sur "Suivi de Projets".

2. Identifiez-vous pour lancer le Centre de Suivi de Projets à l'aide de votre identifiant et mot de passe.

3. Le Centre de Suivi de Projets s'affiche avec 4 onglets par défaut : 

	- Le Tableau de bord. 

	- La liste des tâches saisie par le Chef de Projet. 

	- Les incidents. 

	- Les exigences.







**Conseil** : Pour chaque projet, il est possible de créer une tâche de débogage spécifique (avec une durée estimée à 0). Cette tâche permet d'utiliser la gestion du temps lors des phases de débogage de l'application.
<a name="NOTE3_2"></a>


### Paramétrage des horaires et des congés
<a name="parametrage_des_horaires_des_conges_ELTPARAGRAPHE000082"></a>Pour configurer vos horaires :

1. Déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=CC_Suivi_Configuration%20-%20HC%20N%B0001.gif) puis sélectionnez "Horaires et congés".

2. Indiquez les jours ouvrés et les heures ouvrées (jours et heures pendant lesquelles vous travaillez). Pour plus de détails, consultez [Configuration des horaires](../CCSuivi/3540706.md).

3. Validez.




Si vous connaissez vos congés, saisissez les périodes de congés correspondantes.

1. Déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=CC_Suivi_Configuration%20-%20HC%20N%B0001.gif) puis sélectionnez "Horaires et congés".

2. Indiquez les jours ouvrés et les heures ouvrées (jours et heures pendant lesquelles vous travaillez). Pour plus de détails, consultez [Configuration des congés](../CCSuivi/3540706.md).

3. Validez.



<a name="NOTE3_3"></a>


### Paramétrage de l'aide à la gestion du temps
<a name="parametrage_aide_gestion_temps_ELTPARAGRAPHE000109"></a>Le mode de calcul du temps passé sur une tâche est différent si l'aide à la gestion du temps est activée ou non.

**Si l'aide à la gestion du temps est activée** : Le calcul du temps passé sur une tâche prend en compte :

- les informations indiquées dans la roue des tâches et dans la fenêtre de gestion de la tâche en cours. Les horaires et les jours de travail indiqués dans les options du Centre de Suivi de Projets ne sont pas pris en compte.

- le blocage automatique de la tâche en cours lors de la fermeture de la session Windows.

- le blocage automatique de la tâche en cours lors de la fermeture du Centre de Suivi de Projets de WINDEV, WEBDEV et WINDEV Mobile.




Dans ce cas, les informations saisies dans la configuration des horaires sont prises en compte uniquement lors de l'établissement du planning. Le planning passé prend en compte les horaires réellement réalisés.

**Si l'aide à la gestion du temps n'est pas activée** : Le calcul du temps passé sur une tâche prend en compte uniquement les horaires et les jours travaillés spécifiés dans la configuration des horaires. Les heures "supplémentaires" ne sont pas prises en compte.

**Pour paramétrer l'aide à la gestion du temps** : 

1. Déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=CC_Suivi_Configuration%20-%20HC%20N%B0001.gif) puis sélectionnez "Options".

2. Cliquez sur "Aide à la gestion du temps" et indiquez si l'aide à la gestion du temps doit être activée ou non. Pour plus de détails, consultez [Fonctionnement de l'aide à la gestion du temps](../CCSuivi/3540711.md).

## Etape 3 : Travailler sur une tâche
<a name="etape_3_travailler_sur_une_tache_ELTTEXTE000358"></a>
Pour travailler sur une tâche : 

1. Affichez le Centre de Suivi de Projets en utilisant une des méthodes suivantes : 

	- sous le volet "Outils", dans le groupe "Centres de Contrôle", cliquez sur "Suivi de Projets". 

	- affichez le volet "Centre de Suivi de Projets" : sous le volet "Accueil", dans le groupe "Environnement", déroulez "Volets" et sélectionnez "Volets" puis "Centre de Suivi de Projets".




2. Sélectionnez la tâche voulue et sélectionnez l'option "Démarrer la tâche" du menu contextuel.

3. Si une tâche est déjà en cours, le Centre de Suivi de Projets propose automatiquement de suspendre ou de terminer la tâche en cours.

4. Lorsque la tâche est terminée, il suffit de l'indiquer (option "Réaliser la tâche" du menu contextuel).

## Etape 4 : Travailler sur un incident Client
<a name="etape_4_travailler_sur_incident_client_ELTTEXTE000382"></a>
Pour travailler sur un incident Client : 

1. Affichez le Centre de Suivi de Projets en utilisant une des méthodes suivantes : 

	- sous le volet "Outils", dans le groupe "Centres de Contrôle", cliquez sur "Suivi de Projets". 

	- affichez le volet "Centre de Suivi de Projets" : sous le volet "Accueil", dans le groupe "Environnement", déroulez "Volets" et sélectionnez "Volets" puis "Centre de Suivi de Projets".




2. Pour chaque projet, créez une tâche de débogage spécifique (avec une durée estimée à 0). Cette tâche permet d'utiliser la gestion du temps lors des phases de débogage de l'application.

3. Démarrez la tâche de débogage correspondant au projet du Client.

4. Corrigez l'incident ou les incidents. Les incidents sont visibles dans les volets de type "Incidents" du Centre de Suivi de projets.

5. Dans l'onglet de type "Incidents", passez l'incident en corrigé.
	Conseil : Il est conseillé de détailler les informations concernant la correction lors de la validation de l'incident. En effet, le texte indiqué sera repris automatiquement dans l'aide des nouveautés lors de la création de l'installation de l'application.

6. Si nécessaire, changez de tâche.



<a name="NOTE5_2"></a>


### Remarque : Créer une tâche à partir d'un incident Client
<a name="remarque_creer_une_tache_partir_incident_client_ELTPARAGRAPHE000218"></a>Dans certains cas, un incident signalé par un client peut nécessiter une vrai tâche de développement. 

Dans ce cas, il est possible de créer une tâche de développement à partir d'un incident client directement depuis le volet de type "Incidents".

Pour plus de détails, consultez [Gestion des incidents](../CCSuivi/3540723.md).

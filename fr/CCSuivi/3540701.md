


## Centre de Suivi de Projets : Présentation

<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000326"></a>
Le Centre de Suivi de Projets permet de gérer le planning de réalisation d'un projet, de sa création à sa livraison finale. Il est ainsi possible de connaître l'ensemble des tâches à réaliser sur un projet, les développeurs concernés, les durées nécessaires, l'état d'avancement du projet, etc.

Après avoir listé l'ensemble des tâches d'un projet, le Centre de Suivi de Projets s'occupe de tout. La saisie du temps passé est quasi automatique, ne demande aucune action spécifique et ne génère aucune contrainte particulière. Pour plus de détails, consultez [Gestion du temps](../CCSuivi/3540713.md).

Lors de l'ouverture du projet concerné, le Centre de Suivi de Projets demande ou indique la tâche en cours. Dès qu'une tâche est réalisée, il suffit d'indiquer que cette tâche est terminée et de spécifier la nouvelle tâche.

Une tâche peut être reliée à un élément du projet (fenêtre, état, etc.). Chaque fois que l'élément concerné est ouvert, le temps passé sur cet élément est décompté et mémorisé dans le Centre de Suivi de Projets. Inversement, depuis la liste des tâches, il est possible d'ouvrir automatiquement l'élément correspondant à la tâche que vous voulez réaliser.

Chaque développeur peut également visualiser directement sa liste de tâches dans le volet "Centre de Suivi de Projets".

## Lancement
<a name="lancement_ELTTEXTE000350"></a>
Le Centre de Suivi de Projets peut être lancé :

- depuis WINDEV, WINDEV Mobile ou WEBDEV : sous le volet "Outils", dans le groupe "Centres de Contrôle", cliquez sur "Suivi de Projets".

- en exécutant directement le programme "CCSuivi.EXE" présent dans le sous-répertoire "Programmes" du répertoire d'installation de WINDEV, WINDEV Mobile ou WEBDEV.




Le Centre de Suivi de Projets permet à l'utilisateur d'afficher les données qui le concernent via plusieurs types de vues : 

- Tableau de bord. 

- [Tâches](../CCSuivi/3540704.md).

- [Incidents](../CCSuivi/3540723.md).

- [Exigences](../CCSuivi/3540716.md). 

- [Planning](../CCSuivi/3540705.md). 

- [Diagramme de Gantt](../CCSuivi/3540724.md). 

- [Demandes](../CCSuivi/3540719.md). 

- [Statistiques](../CCSuivi/3540714.md). 

- [Messages](../CCSuivi/3540726.md).

- [Sprints](../CCSuivi/3540730.md).




Pour créer une nouvelle vue, vous pouvez : 

- méthode 1 : cliquer sur le bouton "+" à droite des vues existantes. 

- méthode 2 : dans le menu contextuel d'un onglet d'une vue, sélectionner l'option "Nouvelle vue". 


Il est possible de créer plusieurs vues du même type mais affichant des données différentes. 

**Astuce** : Par défaut, le libellé affiché dans l'onglet de la vue correspond au type du contenu (Tâches, Incidents, Planning, etc.) suivi du filtre discriminant utilisé dans le volet (nom de l'exigence, responsable de la tâche, développeur de l'incident, etc.). 
Si vous affichez un grand nombre de vues et/ou que certaines vues ont un nom par défaut "identiques" (par exemple, 2 vues "Incidents" affichant 2 lots différents auront le même libellé), il peut être intéressant de renommer les volets des vues pour les rendre plus explicites et gagner de la place. Pour renommer un volet d'une vue : 

1. Sélectionnez la vue voulue. 

2. Affichez le menu contextuel de l'onglet de la vue (clic droit). 

3. Dans le menu contextuel qui apparaît, sélectionnez l'option "Personnaliser le titre de la vue active". 

4. Dans la fenêtre qui s'affiche, modifiez le titre de la vue et validez. 




**Faire une recherche globale dans toutes les données du Centre de Suivi de projet** : 
Pour faire une recherche globale parmi les tâches, incidents, exigences : 

1. Déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=CC_Suivi_Configuration%20-%20HC%20N%B0001.gif) et sélectionnez l'option "Rechercher des items (Ctrl + Maj + F)". 

2. Dans la fenêtre qui s'affiche, indiquez l'élément recherché : 

	- Un mot tout seul : Le texte doit contenir un mot commençant par le mot indiqué.

	- Un mot entouré de guillemets : Le texte doit contenir le mot exact.

	- Plusieurs mots séparés par un espace : Le texte doit contenir tous les mots indiqués.

	- Un mot précédé du signe "-" : Le mot indiqué ne doit pas être présent dans le texte.

	- Un mot suivi de "int:&lt;Intervenant&gt;" : Le mot est recherché parmi les résultats de l'intervenant. Exemple : "aide int:doc"

### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000125"></a>Le Centre de Suivi de Projets n'est pas un outil redistribuable. Le Centre de Suivi de Projets ne peut pas être installé avec les applications développées avec WINDEV, WEBDEV ou WINDEV Mobile.

Pour utiliser le Centre de Suivi de Projets, WINDEV, WEBDEV ou WINDEV Mobile doivent obligatoirement être installés sur le poste en cours.

## Fonctionnalités
<a name="fonctionnalites_ELTTEXTE000380"></a>


### Deux niveaux d'utilisation
<a name="deux_niveaux_utilisation_ELTPARAGRAPHE000136"></a>Le Centre de Suivi de Projets permet de suivre la réalisation d'un projet.

Le Centre de Suivi de Projets propose deux niveaux d'utilisation :

- **Utilisation par un administrateur** (par défaut, le login et le mot de passe correspondent à "ADMIN"). 
	L'administrateur définit les projets, les intervenants, les tâches à réaliser. Seul l'administrateur peut donner les droits d'administrateur ou supprimer une tâche, etc.
	Remarque : Si vous utilisez une version précédente du Centre de Suivi de Projets, le login de l'administrateur peut correspondre à "SUPERVISEUR" (sans mot de passe).  

- **Utilisation par un intervenant** : 
	L'intervenant peut voir son planning, ajouter des tâches, les déplacer, configurer ses horaires de travail, modifier l'état d'une tâche, etc.




**Remarque** : Un intervenant peut créer un projet et être ***responsable de ce projet***. Dans ce cas, sur ce projet, le responsable de projet peut définir les tâches à réaliser, les intervenants, etc.
<a name="NOTE3_2"></a>


### Tâches réalisables par l'administrateur
<a name="taches_realisables_par_administrateur_ELTPARAGRAPHE000156"></a>Pour suivre la réalisation d'un projet, l'administrateur doit définir : 

- le [projet](../CCSuivi/3540702.md).

- les [intervenants du projet](../CCSuivi/3540703.md). 

- les [différentes tâches à réaliser par chacun des intervenants](../CCSuivi/3540704.md).

- les [spools de demandes](../CCSuivi/3540707.md).




Lorsque tous ces paramètres ont été définis, il est possible d'afficher les statistiques sur le suivi du projet (éléments réalisés et éléments à faire).

Cette configuration des projets se fait la plupart du temps par l'administrateur (par défaut, le login et le mot de passe correspondent à "ADMIN") ou par le responsable de projet.
<a name="NOTE3_3"></a>


### Utilisation
<a name="utilisation_ELTPARAGRAPHE000184"></a>Le Centre de Suivi de Projets peut être utilisé par chaque intervenant d'un projet. Il permet à l'intervenant de :

- [Voir son planning et l'imprimer](../CCSuivi/3540705.md).

- [Voir sa liste des tâches pour un ou plusieurs projets](../CCSuivi/3540705.md).

- [Définir la tâche en cours](../CCSuivi/3540705.md).

- [Définir ses options de travail (jours, horaires de travail, etc.)](../CCSuivi/3540706.md).




Pour plus de détails sur l'utilisation du Centre de Suivi de Projets pour la gestion d'un planning et de ses tâches, consultez [Gestion du planning et des tâches](../CCSuivi/3540713.md).

### Mode "Invité"
<a name="mode_invite_ELTPARAGRAPHE000225"></a>Le Centre de Suivi de Projets propose un mode "Invité". Ce mode permet à un utilisateur non identifié de visualiser l'ensemble des informations contenues dans le Centre de Suivi de Projets (liste des tâches, plannings, liste des projets, etc.).

Pour lancer le Centre de Suivi de Projets en mode "Invité", il suffit de cliquer sur le bouton "Invité" dans la fenêtre de login.

**Pour autoriser le mode "Invité"** :

1. Déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=CC_Suivi_Configuration%20-%20HC%20N%B0001.gif) puis sélectionnez "Options".

2. Sélectionnez "Options réservées aux administrateurs" puis sélectionnez l'option "Autoriser le mode invité".




**Remarque** : Seul un administrateur peut autoriser le mode "Invité". Le mode "Invité" n'est pas disponible si les fichiers de données des Centres de Contrôle sont au format HFSQL Client/Serveur.

### Accéder aux données du Centre de Suivi de Projets
<a name="acceder_aux_donnees_centre_suivi_projets_ELTPARAGRAPHE000252"></a>Pour accéder aux données du Centre de Suivi de Projets, il est possible d'intégrer le composant **AccesExterne** dans votre projet.

Ce composant permet de consulter les données des Centres de Contrôle. Vous pouvez vous connecter à la base de données des Centres de Contrôle (HFSQL Classic, HFSQL Client/Serveur, PCSCloud ou GDSDrive) pour exporter au format JSON les données concernant les projets, les intervenants, les tâches, les incidents, les exigences. 

Pour importer le composant **CCAccesExterne** dans le projet en cours : 

1. Sous le volet "Projet", dans le groupe "Projet", déroulez "Importer" et sélectionnez "Un composant Externe .. Composant utilitaire ou exemple".

2. La liste des composants utilitaires et exemples est affichée. 

3. Sélectionnez le composant **CCAccesExterne** et validez la fenêtre. 
	Remarque : Vous pouvez également double-cliquer sur le composant pour l'importer dans votre projet. 

4. La documentation associée à ce composant permet de connaître tous les détails de son mode de fonctionnement et de ses possibilités.


## Centre de Suivi de Projets : Tableau de bord
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000299"></a>
Le Centre de Suivi de Projets regroupe toutes les informations sur les projets dans un tableau de bord. 

Ce tableau de bord contient plusieurs widgets permettant de suivre l'évolution, les tâches, les incidents, les corrections d'un ou de plusieurs projets. 

Chaque intervenant peut personnaliser son tableau de bord selon ses objectifs. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000323"></a>


### Afficher un onglet de type Tableau de bord
<a name="afficher_onglet_type_tableau_bord_ELTPARAGRAPHE000020"></a>Si le tableau de bord n'est pas affiché dans le Centre de Suivi de Projets, il suffit de créer un nouvel onglet de type Tableau de bord : 

1. Cliquez sur le bouton "+" à droite des onglets existants. 

2. Sélectionnez le type d'onglet à créer : "Tableau de bord". 

3. L'onglet "Tableau de bord" est affiché. 




Remarque : Il est possible de créer plusieurs onglets du même type mais affichant des données différentes. 

Par défaut, le tableau de bord contient quelques widgets. Il est possible d'ajouter ou de supprimer des widgets, de déplacer ou de redimensionner des widgets. 
<a name="NOTE2_2"></a>


### Configurer les données affichées par le Tableau de bord
<a name="configurer_les_donnees_affichees_par_tableau_bord_ELTPARAGRAPHE000035"></a>Par défaut, les widgets du tableau de bord affichent les données sélectionnées par les options de filtre présentes dans le ruban de l'onglet. Il est possible de sélectionner : 

- la liste d'exigences.

- le lot d'incidents.

- le spool.




Le ruban permet également de définir la vision du tableau de bord : 

- **Vision générale** : Dans ce mode : 

	- toutes les exigences de la liste d'exigences sélectionnée sont prises en compte. 

	- tous les incidents non corrigés de la liste d'exigences sélectionnée sont pris en compte

	- l'avancement des tâches correspond à l'avancement des tâches de développement. 




- **Vision développement** : Dans ce mode : 

	- toutes les exigences de développement de la liste d'exigences sélectionnée sont prises en compte. 

	- tous les incidents non corrigés de la liste d'exigences sélectionnée sont pris en compte




- **Vision test** : Dans ce mode : 

	- toutes les exigences de test de la liste d'exigences sélectionnée sont prises en compte. 

	- tous les incidents créés dans la liste d'exigences sélectionnée sont pris en compte




- **Vision documentation** : Dans ce mode : 

	- toutes les exigences de documentation de la liste d'exigences sélectionnée sont prises en compte. 

	- tous les incidents non corrigés de la liste d'exigences sélectionnée sont pris en compte

	- l'avancement des tâches correspond à l'avancement des tâches de documentation.  







**Remarque** : Par défaut, les widgets affichent les données correspondant aux filtres spécifiés dans le ruban. Il est cependant possible pour chaque widget d'afficher des informations spécifiques. Il suffit de dérouler la flèche présente à droite du widget et de sélectionner l'option "Sélectionner ...". 
<a name="NOTE2_3"></a>


### Ajouter/Masquer des widgets
<a name="ajoutermasquer_des_widgets_ELTPARAGRAPHE000069"></a>**Pour ajouter des widgets** : 

1. Affichez le menu contextuel du tableau de bord (clic droit dans une zone ne correspondant pas à un widget). 

2. Sélectionnez l'option "Ajouter un widget". La liste des widgets disponibles apparaît dans le menu contextuel. 

3. Sélectionnez le widget voulu. 




**Pour masquer des widgets** : 

1. Passez le tableau de bord en mode "Edition" : 

	- Dans le groupe "Affichage" du ruban, cliquez sur "Mode édition". 

	- Affichez le menu contextuel du tableau de bord (clic droit dans une zone ne correspondant pas à un widget) et sélectionnez l'option "Mode édition". 




2. Les différents widgets passent en mode édition. 

3. Affichez le menu contextuel du widget à supprimer/masquer. 

4. Sélectionnez l'option "Masquer le widget". 

5. Lorsque les widgets voulus sont masqués, il suffit de quitter le mode "Edition" : 

	- Dans le groupe "Affichage" du ruban, cliquez sur "Mode édition". 

	- Affichez le menu contextuel du tableau de bord (clic droit dans une zone ne correspondant pas à un widget) et sélectionnez l'option "Quitter le mode édition". 








<a name="NOTE2_4"></a>


### Déplacer et redimensionner des widgets
<a name="deplacer_redimensionner_des_widgets_ELTPARAGRAPHE000095"></a>Pour déplacer ou redimensionner des widgets : 

1. Passez le tableau de bord en mode "Edition" : 

	- Dans le groupe "Affichage" du ruban, cliquez sur "Mode édition". 

	- Affichez le menu contextuel du tableau de bord (clic droit dans une zone ne correspondant pas à un widget) et sélectionnez l'option "Mode édition". 




2. Les différents widgets passent en mode édition. Il est possible de les sélectionner, de les déplacer, de les agrandir ou de diminuer leur taille. 

3. Lorsque les modifications sont effectuées, il suffit de quitter le mode "Edition" : 

	- Dans le groupe "Affichage" du ruban, cliquez sur "Mode édition". 

	- Affichez le menu contextuel du tableau de bord (clic droit dans une zone ne correspondant pas à un widget) et sélectionnez l'option "Quitter le mode édition". 






<a name="NOTE2_5"></a>


### Les types de widgets disponibles
<a name="les_types_widgets_disponibles_ELTPARAGRAPHE000112"></a>Les types de widgets disponibles sont les suivants : 

- **Widget Intervenant**. Affiche les caractéristiques de l'intervenant en cours : nom, photo, poste bureau, mail. 
	Un clic sur la photo permet d'accéder à la fiche complète de l'intervenant. 
	Le menu déroulant du widget permet de changer l'intervenant affiché. 
	Ce widget est affiché par défaut et ne peut pas être masqué. 

- **Widget Avancement**. Affiche le pourcentage d'avancement des tâches de développement pour l'intervenant en cours. 
	Un clic sur la jauge permet d'afficher la liste d'exigences associée : un nouvel onglet de type Exigences est affiché. 
	Le menu déroulant du widget permet de sélectionner : 

	- une liste d'exigences différente de la liste d'exigences globale au tableau de bord.

	- un sprint différent du sprint global au tableau de bord.


 Ce widget est affiché par défaut et ne peut pas être masqué.

- **Widget Incidents**. Affiche le nombre d'incidents correspondant à la liste d'exigences spécifiée. 
	Un clic sur le nombre d'incidents permet d'ouvrir un nouvel onglet de type Incidents listant les incidents concernés. 
	Le menu déroulant de ce widget permet : 

	- de filtrer les incidents sur le lot d'incidents global au tableau de bord. 

	- de filtrer les incidents sur un lot d'incidents spécifique.

	- de filtrer les incidents sur le sprint global au tableau de bord.

	- de filtrer les incidents sur un sprint spécifique.

	- de supprimer le filtre. 


Le menu contextuel de ce widget (clic droit) permet de le masquer. 

- **Widget Répartition des incidents**. Affiche la répartition des incidents par intervenant. En vision "Test", affiche la répartition des incidents créés par intervenant. 
	Un clic sur le graphique permet d'afficher un onglet de type Incidents pour l'intervenant sélectionné. Cet onglet affiche soit les incidents affectés à l'intervenant, soit les incidents créés par l'intervenant sélectionné. 
	Le menu déroulant de ce widget permet : 

	- de filtrer les incidents sur le lot d'incidents global au tableau de bord. 

	- de filtrer les incidents sur un lot d'incidents spécifique.

	- de filtrer les incidents sur le sprint global au tableau de bord.

	- de filtrer les incidents sur un sprint spécifique.

	- de supprimer le filtre. 


 Le menu contextuel de ce widget (clic droit) permet de configurer le graphe affiché dans le widget.  

- **Widget Exigences à contrôler**. Affiche le nombre d'exigences associées à l'intervenant à développer, tester ou documenter (selon le mode de vision choisi). Affiche également le nombre d'exigences en dépassement, bloquées ou à retester. 
	Un clic sur un des liens permet d'afficher la liste des exigences concernées dans un onglet de type Exigences. 
	Le menu déroulant de ce widget permet : 

	- de filtrer les exigences sur la liste des exigences globales au tableau de bord. 

	- de filtrer les exigences sur une liste d'exigences spécifique.

	- de filtrer les exigences selon le sprint global au tableau de bord.

	- de filtrer les exigences selon un sprint spécifique.


 Ce widget est affiché par défaut et ne peut pas être masqué.

- **Widget Exigences** : Affiche la liste des exigences associées à l'intervenant sous forme de TreeMap. Les exigences terminées sont affichées en vert, les exigences planifiées sont en bleu et les exigences bloquées en rouge. Un clic sur une exigence affiche la description de l'exigence. 
	Le menu déroulant de ce widget permet : 

	- de filtrer les exigences sur la liste des exigences globales au tableau de bord. 

	- de filtrer les exigences sur une liste d'exigences spécifique.

	- de filtrer les exigences selon le sprint global au tableau de bord.

	- de filtrer les exigences selon un sprint spécifique.


 Ce widget est affiché par défaut et ne peut pas être masqué.

- **Widget Incidents en attente** : Affiche la liste des incidents en attente associées à l'intervenant. 
	Le menu contextuel d'un incident permet de éditer, corriger, archiver ou réaffecter la fiche. Un champ de recherche permet de réaliser une recherche de type Contient parmi les incidents listés. 
	Le menu déroulant de ce widget permet : 

	- de filtrer les incidents sur le lot d'incidents global au tableau de bord. 

	- de filtrer les incidents sur un lot d'incidents spécifique.

	- de filtrer les incidents sur le sprint global au tableau de bord.

	- de filtrer les incidents sur un sprint spécifique.


 Ce widget est affiché par défaut et ne peut pas être masqué.

- **Widget Demandes** : Affiche le nombre de demandes à traiter. 
	Un clic sur le nombre de demandes permet d'afficher les demandes du spool dans un onglet de type Demandes. 
	Le menu déroulant de ce widget permet d'utiliser le filtre global ou de sélectionner un spool spécifique.
	Le menu contextuel de ce widget (clic droit) permet de le masquer. 

- **Widget Tâche en cours** : Affiche la tâche en cours de l'intervenant (ainsi que sa durée depuis la dernière activation). 
	Un clic sur ce widget ouvre un menu contextuel permettant de voir le détail de la tâche ou de modifier la tâche en cours. 
	Le menu contextuel de ce widget (clic droit) permet de le masquer. 

- **Widget Date de fin d'exigences**. Permet d'afficher la date de fin de la liste d'exigences sélectionnée. 
	Un clic sur le widget permet d'ouvrir la liste d'exigences dans un onglet de type Exigences.
	Le menu déroulant de ce widget permet : 

	- d'utiliser le filtre global ou de sélectionner une liste d'exigences spécifique.

	- d'utiliser le sprint global ou de sélectionner un sprint spécifique.




- Le menu contextuel de ce widget (clic droit) permet de le masquer. 

- **Widget Liste des tâches**. Permet d'afficher la liste des tâches pour la liste d'exigences sélectionnée. Les tâches affichées sont les tâches 'en attente', 'en cours' et 'bloquées'. 
	Un clic sur une tâche permet d'ouvrir la tâche sélectionnée dans une nouvelle fenêtre. Le menu déroulant des tâches (clic droit) permet de démarrer, modifier ou terminer une tâche.
	Le menu déroulant de ce widget permet : 

	- de filtrer les tâches sur la liste d'exigences globale au tableau de bord. 

	- de filtrer les tâches sur une liste d'exigences spécifique.

	- de filtrer les tâches sur le sprint global au tableau de bord.

	- de filtrer les tâches sur un sprint spécifique.

	- de supprimer le filtre.

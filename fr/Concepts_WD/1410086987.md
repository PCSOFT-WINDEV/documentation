## Présentation
<a name="presentation_ELTTEXTE000094"></a>
Pour simplifier vos développements en équipe, WINDEV propose un gestionnaire de sources. 
Ce gestionnaire de sources permet à plusieurs développeurs de travailler simultanément sur le même projet et de partager des éléments entre différents projets.
<br>![Une base de sources regroupe les éléments du projet, chaque poste possède une copie en local des éléments nécessaires au développement](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P34-GDS.svg)


Il est possible de partager les éléments présents dans le GDS :

- via [PCSCloud (solution payante, clé en main, proposant un serveur dédié)](https://pcscloud.net/FR/cloud_developpement.awp),

- via [GDSDrive (solution payante, clé en main, utilisant un serveur mutualisé)](https://pcscloud-drive.net/FR/GDSDrive/index.awp).

- via un réseau,

- via Internet, 

- en mode déconnecté. Dans ce cas, les éléments sur lesquels un travail spécifique devra être effectué seront extraits du GDS par exemple lors de la connexion du portable au système général.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Fonctionnement du Gestionnaire de Sources
<a name="fonctionnement_gestionnaire_sources_ELTTEXTE000143"></a>
Voici un exemple d'utilisation du Gestionnaire de Sources :
<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GestionnaireDeSources.svg)


Si un élément (fenêtre, état, ...) du projet est extrait, ce même élément ne pourra pas être extrait une seconde fois.

Après modification des éléments extraits, il est nécessaire de réintégrer ces éléments pour que le projet de base prenne en compte ces modifications. En effet, la base de sources conserve un historique de tous les éléments du projet depuis leur création.

A chaque réintégration d'un élément, le numéro de version du projet de base est incrémenté de 1.

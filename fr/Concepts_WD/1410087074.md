
## 12. Les différents tests disponibles

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
WINDEV propose plusieurs méthodes pour tester votre application. Le tableau ci-dessous présente ces différents modes.

| Type de test | Description |
| --- | --- |
| Test de l'ensemble du projet ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_GO_Projet_WB_bl.gif) | Simule le lancement de l'application par l'exécutable. Permet de tester l'application dans son ensemble, même si son développement n'est pas terminé.<br><br>Dès qu'un problème apparaît dans l'exécution du projet, vous pouvez lancer le débogueur pour connaître et solutionner le problème rencontré.<br><br>Remarque : Si le chiffre "64" est présent sur l'icône "Go", cela signifie que le test sera effectuée en 64 bits dans un éditeur 32 bits. |
| Test de l'élément en cours ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ICO_GO_Fenetre_WD_bl.gif) | Permet de tester le fonctionnement de cet élément dès que son développement est terminé.<br>Pour les états ou les fenêtres, il est possible de lancer le débogueur dès qu'un problème est rencontré.<br><br>Remarque : Si le chiffre "64" est présent sur l'icône "Go", cela signifie que le test sera effectuée en 64 bits dans un éditeur 32 bits. |
| Test d'un projet pas à pas<br>(grâce au débogueur) | Permet de lancer le débogueur au lancement de l'application. Cette solution permet de suivre méticuleusement le déroulement de l'application à l'aide du débogueur.<br><br>Il est également possible de débogueur un exécutable spécifique, un exécutable déjà lancé ou un exécutable lancé sur un autre poste. |
| Test unitaire | Permet de tester les fenêtres, les procédures et les classes d'une application à différents niveaux de développement.<br><br>Les scénarios de test sont automatiquement générés en WLangage à partir du test de l'application ou du test d'un élément. Les scénarios peuvent être directement modifiés sous l'éditeur de code. |
| Test de non-régression | Permet de tester la non-régression d'un élément entre les différentes versions d'une application.<br>Il suffit de sélectionner (ou de créer) le scénario de test à exécuter. |

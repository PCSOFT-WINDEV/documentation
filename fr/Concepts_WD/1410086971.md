
## 5. Fenêtres modales, non modales ou MDI

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Les applications créées avec WINDEV peuvent utiliser plusieurs types de fenêtres (et mélanger chacun de ces types).
<a name="NOTE1_2"></a>


### Fenêtres modales
<a name="fenetres_modales_ELTPARAGRAPHE000017"></a>Une fenêtre modale s'ouvre en avant-plan de l'application et bloque l'accès aux autres fenêtres déjà ouvertes (s'il y en a). Les fenêtres modales sont généralement utilisées comme :

- fenêtres principales d'une application,

- boîtes de dialogue dont la validation provoque une action.


Avec WINDEV, lorsqu'une fenêtre modale s'ouvre, une ombre grise apparaît sur les fenêtres en arrière-plan : c'est le GFI ou Grisage des Fenêtres Inaccessibles.

<a name="NOTE1_3"></a>


### Fenêtres non modales
<a name="fenetres_non_modales_ELTPARAGRAPHE000036"></a>Une fenêtre non modale s'ouvre en parallèle des autres fenêtres de l'application. Chacune des fenêtres reste accessible et manipulable. Les fenêtres non modales servent généralement :

- de palettes d'outils,

- de boîtes de dialogue dont la modification affecte directement la fenêtre principale. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P1_Fen%EAtres%20non%20modales%20-%20HC%20N%B0001.gif&type=thumb)



Dans cet exemple, chaque fenêtre de description d'un fauteuil permet d'ouvrir la fenêtre de description d'un autre fauteuil. Toutes les fenêtres ouvertes restent accessibles.
<a name="NOTE1_4"></a>


### Fenêtres MDI
<a name="fenetres_mdi_ELTPARAGRAPHE000051"></a>Dans une application MDI, toutes les fenêtres de l'application sont affichées dans une seule et même fenêtre principale : la fenêtre mère MDI. Les fenêtres des applications MDI peuvent donc être de deux types : 


|   |   |
| --- | --- |
| <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P1_Fenetres%20MDI%20-%20HC%20N%B0001.gif&type=thumb)<br> | **Fenêtre mère MDI** : Toutes les autres fenêtres de l'application sont affichées dans cette fenêtre. Cette fenêtre permet à l'utilisateur de manipuler le menu principal de l'application. |
| **Fenêtre fille MDI** : Fenêtre de l'application affichée dans la fenêtre mère. C'est dans ces fenêtres que l'utilisateur pourra saisir des informations. |


## ActiveXEvénement (Fonction)

***En anglais : ActiveXEvent***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Associe une procédure écrite en WLangage à un événement d'un contrôle ActiveX.

Après l'exécution de la fonction **ActiveXEvénement**, la procédure sera lancée à chaque fois que l'événement choisi sera déclenché.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// -- Initialisation du champ ActiveX AX_LecteurAVI
ActiveXEvénement("Ma_Procédure", AX_LecteurAVI, "OnClick")
```

```wl
// -- Procédure Ma_Procédure
PROCEDURE Ma_Procédure(XCoord, YCoord)
Info("Coordonnées du clic : " + XCoord + "," + YCoord)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`ActiveXEvénement(<Procédure WLangage> , <Champ ActiveX> , <Evénement>)`
---

**`<Procédure WLangage> : (Nom de procédure)`**

Nom de la procédure WLangage appelée lors du déclenchement de l'événement. 
Cette procédure doit comporter le nombre de paramètres fournis par l'événement et peut éventuellement renvoyer un résultat (se référer à la documentation du contrôle ActiveX).

**`<Champ ActiveX> : (Nom de champ)`**

Nom du champ ActiveX.

**`<Evénement> : (Chaîne de caractères)`**

Nom de l'événement associé à l'ActiveX déclenchant la procédure.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Il est conseillé de placer l'appel à la fonction **ActiveXEvénement** dans l'événement "Initialisation" du champ ActiveX.

- **Important** : PC SOFT n'assure aucun support sur le détail du fonctionnement des contrôles ActiveX.
	Vous pouvez trouver les différents événements générés par le contrôle ActiveX dans sa documentation.


## ChampErreur (Fonction)

***En anglais : ControlError***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Personnalise la valeur affichée dans les champs en cas d'impossibilité d'afficher la valeur.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Personnalisation de l'erreur ceDépassement
ChampErreur(ceDépassement, "Le masque du champ est trop petit")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`ChampErreur(<Type d'erreur> , <Message d'erreur>)`
---

**`<Type d'erreur> : (Constante)`**

Type d'erreur concerné :


|   |   |   |
| --- | --- | --- |
| *ceDépassement* | La valeur du champ est supérieure au masque de saisie/d'affichage du champ. | Valeur affichée par défaut (correspond à 5 caractères plus) : "+++++" |
| *ceLectureRub* | Erreur de lecture d'une rubrique : la rubrique n'existe pas, le fichier de données est vide, ... | Valeur affichée par défaut (correspond à 4 caractères dièse) : "####" |
| *ceLiaisonRompue* | La valeur de la rubrique liée n'est pas trouvée (la valeur liée n'existe pas ou erreur pendant sa lecture). | Valeur affichée par défaut (correspond à 3 caractères dièse) : "###" |
| *ceNull* | La rubrique liée est une rubrique HFSQL qui contient la valeur NULL. | Valeur affichée par défaut (chaîne vide) : "" |



**`<Message d'erreur> : (Chaîne de caractères)`**

Nouveau message d'erreur associé au type d'erreur spécifié.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Modifications des valeurs affichées
<a name="modifications_des_valeurs_affichees_ELTPARAGRAPHE000081"></a>Les nouvelles valeurs affichées seront effectives uniquement pour les erreurs à venir. Les valeurs affectées aux champs avant l'utilisation de la fonction **ChampErreur** ne sont pas modifiées.

Le changement est global au programme en cours.

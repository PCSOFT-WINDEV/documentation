


## ChampSuivant (Fonction)

***En anglais : ControlNext***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nom du champ suivant en saisie. 

Attention : Cette fonction doit être utilisée uniquement dans l'événement "Sortie" d'un champ. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Identification du champ suivant en saisie
ResChamp = ChampSuivant()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ChampSuivant()`
---

**`<Résultat> : (Chaîne de caractères)`**

Nom du champ suivant en saisie. La casse du nom du champ (majuscules/minuscules) correspond à celle utilisée lors de la création du champ sous l'éditeur.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction **ChampSuivant** ne peut être appelée que dans l'événement "Sortie" d'un champ. Si la fonction **ChampSuivant** est appelée dans un autre événement ou traitement, le nom du champ retourné n'aura pas de sens.

- Lorsque l'utilisateur utilise la touche TAB, la fonction **ChampSuivant** retourne le nom du champ qui suit celui en cours dans l'ordre de tabulation défini sous l'éditeur. 
	Rappel : Pour définir l'ordre de tabulation, sous le volet "Fenêtre", dans le groupe "Ordre", déroulez "Navigation" et sélectionnez "Éditer".

- Par contre, lorsque l'utilisateur clique sur un champ, la fonction **ChampSuivant** retourne le nom du champ sur lequel l'utilisateur a cliqué.

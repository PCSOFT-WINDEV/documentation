
## Créer une application gérant l'Euro
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Les différentes étapes
<a name="les_differentes_etapes_ELTTEXTE000101"></a>
La création d'une application gérant le passage à l'Euro est aussi simple que la création d'une application standard WINDEV. Les étapes à suivre sont les suivantes :

1. Créez votre projet.

2. Créez l'analyse associée. Les rubriques devant stocker des monétaires doivent être de type "Monétaire".

3. Générez l'analyse.

4. Décrivez les fenêtres, états, ... Pour chaque valeur monétaire devant être gérée :

	- spécifiez que le champ associé (champ de saisie, colonne de table ou champ d'un état) doit être de type "Monétaire+Euro",

	- spécifiez la devise d'affichage,

	- spécifiez la devise mémorisée.




5. Initialisez les taux de change des devises non utilisées dans le programme à l'aide de la fonction [eInitDevise](../WDLang1/3033003.md) : utilisez un taux de change égal à 0.

6. Testez votre application.

7. Créez la bibliothèque si nécessaire, puis l'exécutable.

8. Créez l'installation de l'application et diffusez vos programmes.

## L'Euro dans vos fichiers de données
<a name="euro_dans_vos_fichiers_donnees_ELTTEXTE000125"></a>


### Votre application gère des Euros et des francs (cas le plus fréquent)
<a name="votre_application_gere_des_euros_des_francs_cas_plus_frequent_ELTPARAGRAPHE000035"></a>Pour une plus grande sécurité, il est conseillé de conserver toutes les valeurs monétaires en francs comme en Euros. Ainsi, si votre fichier gère des montants, il est conseillé d'utiliser les rubriques suivantes :

- montant en francs,

- montant en Euros,

- monnaie de référence (dans laquelle la transaction est effectuée),

- différence d'arrondi.




Cette méthode de stockage des données a pour avantage de faciliter tous les types de traitements futurs.
<a name="NOTE2_2"></a>


### Votre application gère uniquement des Euros
<a name="votre_application_gere_uniquement_des_euros_ELTPARAGRAPHE000049"></a>Votre application est destinée à fonctionner après le 28 février 2002. L'affichage en francs est donné à titre purement indicatif : vos fichiers de données peuvent stocker uniquement des valeurs en Euro.

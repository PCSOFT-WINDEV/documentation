
## Constantes Tâches parallèles
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


Les constantes utilisées par les fonctions de gestion des tâches parallèles sont les suivantes : 

| Constante | Description | Utilisée par les fonctions |
| --- | --- | --- |
| **tpeAnnulationDemandée** | Etat de la tâche parallèle : Une demande d'annulation a été faite sur la tâche parallèle (fonction [TâcheParallèleDemandeAnnulation](../WDLang1/1000021213.md)). | Variable de type [TâcheParallèle](../WDLang1/1000021220.md) |
| **tpeAnnulée** | Etat de la tâche parallèle : La tâche parallèle est annulée (fonction [TâcheParallèleAnnule](../WDLang1/1000021278.md)). | Variable de type [TâcheParallèle](../WDLang1/1000021220.md) |
| **tpeAttenteExécution** | Etat de la tâche parallèle : La tâche parallèle est en attente d'exécution. | Variable de type [TâcheParallèle](../WDLang1/1000021220.md) |
| **tpeAttentePrécédente** | Etat de la tâche parallèle : La tâche parallèle attend l'exécution d'une tâche parallèle précédente. | Variable de type [TâcheParallèle](../WDLang1/1000021220.md) |
| **tpeExécutionEnCours** | Etat de la tâche parallèle : La tâche parallèle est en cours d'exécution. | Variable de type [TâcheParallèle](../WDLang1/1000021220.md) |
| **tpeNonPlanifiée** | Etat de la tâche parallèle : La tâche parallèle n'est pas planifiée. | Variable de type [TâcheParallèle](../WDLang1/1000021220.md) |
| **tpeTerminée** | Etat de la tâche parallèle : La tâche parallèle est terminée. | Variable de type [TâcheParallèle](../WDLang1/1000021220.md) |
| **tpoCopieComplèteContexteHFSQL** | Provoque la copie immédiate du contexte HFSQL courant.<br>Conseillé par exemple si la tâche parallèle doit tenir compte des positions en cours dans les fichiers et requêtes du contexte de l'appelant. | [Description de TâcheParallèle](../WDLang1/1000021586.md), variable de type [TâcheParallèle](../WDLang1/1000021220.md), <br>[TâcheParallèleExécute](../WDLang1/1000021211.md), <br>[TâcheParallèleExécuteAprès](../WDLang1/1000021214.md), <br>[TâcheParallèleExécuteAprèsToutes](../WDLang1/1000021217.md)<br>[TâcheParallèleExécuteApresUne](../WDLang1/1000021216.md) |
| **tpoCopieDifféréeContexteHFSQL** | Provoque une copie légère du contexte HFSQL **lors du premier accès aux données HFSQL**. | [Description de TâcheParallèle](../WDLang1/1000021586.md), variable de type [TâcheParallèle](../WDLang1/1000021220.md), <br>[TâcheParallèleExécute](../WDLang1/1000021211.md), <br>[TâcheParallèleExécuteAprès](../WDLang1/1000021214.md), <br>[TâcheParallèleExécuteAprèsToutes](../WDLang1/1000021217.md)<br>[TâcheParallèleExécuteApresUne](../WDLang1/1000021216.md) |
| **tpoCopieLégèreContexteHFSQL** | Provoque la copie immédiate d'une partie du contexte HFSQL courant.<br>Seuls les répertoires où se trouvent les fichiers de données en HFSQL Classic et/ou les connexions en HFSQL Client/Serveur sont mémorisés. | [Description de TâcheParallèle](../WDLang1/1000021586.md), variable de type [TâcheParallèle](../WDLang1/1000021220.md), <br>[TâcheParallèleExécute](../WDLang1/1000021211.md), <br>[TâcheParallèleExécuteAprès](../WDLang1/1000021214.md), <br>[TâcheParallèleExécuteAprèsToutes](../WDLang1/1000021217.md)<br>[TâcheParallèleExécuteApresUne](../WDLang1/1000021216.md) |
| **tpoThreadPrincipal** | Exécute la tâche dans le thread principal.<br>Attention : Cette constante ne peut pas être utilisée pour exécuter une tâche parallèle dans le thread principal depuis le thread principal. | [Description de TâcheParallèle](../WDLang1/1000021586.md), variable de type [TâcheParallèle](../WDLang1/1000021220.md), <br>[TâcheParallèleExécute](../WDLang1/1000021211.md), <br>[TâcheParallèleExécuteAprès](../WDLang1/1000021214.md), <br>[TâcheParallèleExécuteAprèsToutes](../WDLang1/1000021217.md)<br>[TâcheParallèleExécuteApresUne](../WDLang1/1000021216.md) |
| **tppNombreMaximalThread** | Nombre maximal de threads pouvant être exécutés en même temps. | [TâcheParallèleParamètre](../WDLang1/1000021284.md) |

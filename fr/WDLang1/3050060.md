
## MatNbLigne (Fonction)

***En anglais : MatNbLine***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nombre de lignes d'une matrice. Le nombre de lignes d'une matrice varie dynamiquement. Il correspond au nombre de lignes initialisées. Le nombre de lignes peut varier après l'utilisation des fonctions :

- [MatEcrit](../WDLang1/3050077.md),

- [MatRemplit](../WDLang1/3050057.md).







<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003050060&name=matnbligne_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Nombre de lignes ?
ResNbLigne = MatNbLigne("MaMatrice")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = MatNbLigne(<Matrice>)`
---

**`<Résultat> : (Entier)`**

Nombre de lignes dans la matrice spécifiée.

**`<Matrice> : (Chaîne de caractères)`**

Nom de la matrice à manipuler, défini avec la fonction [MatCrée](../WDLang1/3050058.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Existence de la matrice
<a name="existence_matrice_ELTPARAGRAPHE000085"></a>L'existence de la matrice n'est pas vérifiée. Pour vérifier l'existence d'une matrice, utilisez la fonction [MatExiste](../WDLang1/3050076.md).


<a name="NOTE0_2"></a>




### Gestion des erreurs
<a name="gestion_des_erreurs_ELTPARAGRAPHE000097"></a>**Attention** : La fonction **MatNbLigne** ne retourne pas de code d'erreur. Pour savoir si des erreurs ont été générées par le renvoi du nombre de lignes, utilisez la fonction [MatErreur](../WDLang1/3050044.md). Le libellé de l'erreur peut être connu grâce à la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.

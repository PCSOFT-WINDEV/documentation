


## &lt;Editeur HTML&gt;.RécupèreDocument (Fonction)

***En anglais : &lt;HTML editor&gt;.GetDocument***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère en mode asynchrone le document HTML en cours dans un champ Editeur HTML. 

**Remarque** : Cette fonction est notamment utile pour les applications Android et iOS pour lesquelles la propriété WLangage [Document](../Proprietes/1000026145.md) n'est pas disponible en lecture.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
EDHTML_MonEditeurHTML.RécupèreDocument(EditeurHTMLRécupèreDocument_Callback)
PROCÉDURE INTERNE EditeurHTMLRécupèreDocument_Callback(doc est un htmlDocument)
	LIB_Titre = doc.title
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Editeur HTML>.RécupèreDocument(<Procédure WLangage>)`
---

**`<Champ Editeur HTML> : (Nom de champ)`**

Nom du champ Editeur HTML manipulé.

**`<Procédure WLangage> : (Nom de procédure)`**

Procédure WLangage (également nommée "callback") exécutée lorsque le document a été récupéré. Cette procédure est de la forme : 

```txt
PROCEDURE <Nom de la procédure>(MonDocument est un htmlDocument)
```
où &lt;MonDocument&gt; est une variable de type [htmlDocument](../WDLang5/1000025910.md) correspondant au document HTML affiché dans le champ Editeur HTML.

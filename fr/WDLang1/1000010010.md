


## TableTypeElément (Fonction)

***En anglais : TableTypeItem***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Teste l'existence d'un élément d'un champ Table hiérarchique et renvoie son type (noeud ou feuille) si l'élément existe.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Type de l'élément de la ligne 3 
// dans le champ Table hiérarchique "TABLEH_TableRecette"
SI TableTypeElément(TABLEH_TableRecette, 3) = taNoeud ALORS
	// Dérouler ce noeud
	TableDéroule(TABLEH_TableRecette, 3)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Tester l'existence d'un élément en spécifiant le numéro de la ligne

`<Résultat> = TableTypeElément(<Champ Table hiérarchique> [, <Numéro de la ligne>])`
---

**`<Résultat> : (Constante)`**

Type de l'élément : 



|   |   |
| --- | --- |
| *taErreur* | L'élément spécifié n'existe pas. |
| *taFeuille* | L'élément spécifié est une feuille. |
| *taNoeud* | L'élément spécifié est un noeud. |



**`<Champ Table hiérarchique> : (Nom de champ)`**

Nom du champ Table hiérarchique à manipuler.

**`<Numéro de la ligne> : (Entier optionnel)`**

Numéro de la ligne contenant l'élément à manipuler. Si ce paramètre n'est pas spécifié, le type de l'élément contenu dans la ligne en cours est renvoyé.  


<a name="SYNTAXE2"></a>

### Tester l'existence d'un élément en spécifiant son nom

`<Résultat> = TableTypeElément(<Champ Table hiérarchique> [, <Nom de l'élément>])`
---

**`<Résultat> : (Constante)`**

 Type de l'élément : 



|   |   |
| --- | --- |
| *taErreur* | L'élément spécifié n'existe pas. |
| *taFeuille* | L'élément spécifié est une feuille. |
| *taNoeud* | L'élément spécifié est un noeud. |



**`<Champ Table hiérarchique> : (Nom de champ)`**

Nom du champ Table hiérarchique à manipuler.

**`<Nom de l'élément> : (Chaîne de caractères optionnelle)`**

Nom de l'élément à tester. Ce paramètre est de la forme :

```txt
"<Nom de la racine>" + TAB + ["<Nom du 1er noeud>" + TAB + ...
["<Nom du 2ème noeud>" + TAB + [...]]]"<Nom de l'élément>"
```
Si ce paramètre ne correspond pas à un élément existant, une erreur WLangage est générée.

Si ce paramètre n'est pas spécifié, le type de l'élément contenu dans la ligne en cours est renvoyé.

<a name="NOTE0_1"></a>

La vitesse d'exécution de la syntaxe 1 est plus rapide que celle de la syntaxe 2.




## zmlSupprime (Fonction)

***En anglais : mlzDelete***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime une ligne dynamique dans un champ Zone multiligne. 




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Clic sur [supprimer]
nLigne est un entier = ZM_Réseau.Valeur
ZmlSupprime(ZM_Réseau, nLigne)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`zmlSupprime(<Champ Zone multiligne> [, <Position>])`
---

**`<Champ Zone multiligne> : (Nom de champ)`**

Nom du champ Zone multiligne à manipuler.

**`<Position> : (Entier optionnel)`**

Indice de la ligne à supprimer. 

- Si ce paramètre n'est pas précisé, la suppression sera effectuée sur la ligne en cours (si elle est dynamique). Si aucune ligne dynamique n'est sélectionnée, la fonction **zmlSupprime** provoque une erreur fatale.

- Si &lt;Position&gt; est supérieur au nombre de lignes, la fonction **zmlSupprime** provoque une erreur fatale.

- L'indice doit correspondre à l'indice d'une ligne dynamique (ajoutée par la fonction [zmlAjouteLigne](../WDLang1/1000020199.md) ou [zmlInsèreLigne](../WDLang1/1000020207.md)). Si le champ Zone multiligne contient des lignes statiques et des lignes dynamiques, il est nécessaire de prendre en compte le nombre de lignes statiques.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La taille du champ Zone multiligne est modifiée pour prendre en compte la ligne supprimée : le champ se réduit par le bas. Si des champs sont situés sous le champ Zone multiligne, ils sont déplacés vers le haut. La hauteur utile de la fenêtre est diminuée en conséquence.

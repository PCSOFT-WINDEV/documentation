
## dbgOptionLog (Fonction)

***En anglais : dbgLogOption***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère ou modifie les options du log d'exécution.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connaître les options du log d'exécution
SI ETBinaire(dbgOptionLog(), LogFonctionsWLangage) = LogFonctionsWLangage  ALORS
	Trace("Log avec fonctions")
FIN
SI ETBinaire(dbgOptionLog(), LogParamètres) = LogParamètres ALORS
	Trace("Log avec paramètres")
FIN
SI ETBinaire(dbgOptionLog(), LogTraitementUniquement) = LogTraitementUniquement ALORS
	Trace("Log avec traitements")
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Obtenir les options du log d'exécution

`<Résultat> = dbgOptionLog()`
---

**`<Résultat> : (Constante de type Entier)`**

Options du log d'exécution :


|   |   |
| --- | --- |
| *LogFonctionsWLangage* | Les fonctions WLangage sont inscrites dans le log d'exécution. |
| *LogParamètres* | Les paramètres des fonctions sont inscrits dans le log d'exécution. |
| *LogTout* | Le log d'exécution récupère :<br><br>- les fonctions WLangage<br><br>- les paramètres des fonctions<br><br>- les traitements<br><br><br> |
| *LogTraitementUniquement* | Le log d'exécution récupère uniquement les informations des traitements. |






<a name="SYNTAXE2"></a>

### Modifier les options du log d'exécution

`dbgOptionLog(<Anciennes options> , <Nouvelles options>)`
---

**`<Anciennes options> : (Constante de type Entier)`**

|   |   |
| --- | --- |
| *LogFonctionsWLangage* | Les fonctions WLangage sont inscrites dans le log d'exécution. |
| *LogParamètres* | Les paramètres des fonctions sont inscrits dans le log d'exécution. |
| *LogTout* | Le log d'exécution récupère :<br><br>- les fonctions WLangage<br><br>- les paramètres des fonctions<br><br>- les traitements<br><br><br> |
| *LogTraitementUniquement* | Le log d'exécution récupère uniquement les informations des traitements. |





**`<Nouvelles options> : (Constante de type Entier)`**

Options du log d'exécution : 


|   |   |
| --- | --- |
| *LogFonctionsWLangage* | Les fonctions WLangage sont inscrites dans le log d'exécution. |
| *LogParamètres* | Les paramètres des fonctions sont inscrits dans le log d'exécution. |

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Fonctionnement du log d'exécution
<a name="fonctionnement_log_execution_ELTPARAGRAPHE000131"></a>Par défaut, le log d'exécution est créé avec toutes les options : fonctions WLangage listées et paramètres des fonctions. Ces options permettent de détailler les actions faites, mais le log créé a vite une taille importante.

Pour limiter la taille du log d'exécution (par exemple lors d'un log réalisé sur une durée importante), il est possible de ne pas prendre en compte ces options.

Pour limiter la taille du log d'exécution, il est également possible d'activer le log d'exécution uniquement sur la partie de l'application posant problème avec la fonction [dbgEtatLog](../WDLang1/1000017138.md).

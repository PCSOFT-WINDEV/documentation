
## Procédure WLangage appelée par la fonction ConfirmerAsynchrone



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Procédure (également appelée "Callback") appelée par la fonction [ConfirmerAsynchrone](../WDLang1/1000025305.md) lorsque l'utilisateur clique sur l'un des boutons de la boîte de dialogue. Cette procédure permet de connaître le bouton cliqué et permet d'exécuter si nécessaire un traitement spécifique. 
Remarque : Cette procédure peut être une procédure locale, globale, interne ou lambda. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ConfirmerAsynchrone("Quel bouton choisissez vous ?", Procédure_Confirmer)

PROCEDURE INTERNE Procédure_Confirmer(nRéponse entier)
	SELON nRéponse
		CAS Oui : ToastAffiche("Bouton Oui")
		CAS Non : ToastAffiche("Bouton Non")
		CAS Annuler : ToastAffiche("Bouton Annuler")
	FIN
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`ConfirmerAsynchrone_Callback(<Bouton sélectionné>)`
---

**`<Bouton sélectionné> : (Entier ou constante de type entier)`**

Identifie le bouton sélectionné par l'utilisateur :



|   |   |   |
| --- | --- | --- |
| *Annuler* | 2 | L'utilisateur a choisi le bouton [ANNULER]. |
| *Non* | 0 | L'utilisateur a choisi le bouton [NON]. |
| *Oui* | 1 | L'utilisateur a choisi le bouton [OUI]. |

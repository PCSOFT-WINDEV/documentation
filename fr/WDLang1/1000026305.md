


## &lt;Editeur HTML&gt;.RécupèreSélection (Fonction)

***En anglais : &lt;HTML editor&gt;.GetSelection***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère en mode asynchrone la sélection effectuée dans un champ Editeur HTML. 

**Remarque** : Cette fonction est notamment utile pour les applications Android et iOS pour lesquelles la propriété WLangage [Sélection](../Proprietes/1000023033.md) n'est pas disponible en lecture.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Grise ou non le bouton Ouvrir selon la sélection
EDHTML_MonEditeurHTML.RécupèreSélection(EditeurHTMLRécupèreSélection_Callback)
PROCEDURE INTERNE EditeurHTMLRécupèreSélection_Callback(Sélection est un edtHTMLSélection)
   	BTN_Ouvrir.Grisé = PAS Gauche(sélection.Texte, "https:\\" )
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Editeur HTML>.RécupèreSélection(<Procédure WLangage>)`
---

**`<Champ Editeur HTML> : (Nom de champ)`**

Nom du champ Editeur HTML manipulé.

**`<Procédure WLangage> : (Nom de procédure)`**

Procédure WLangage (également nommée "callback") exécutée lorsque la sélection a été récupérée. Cette procédure est de la forme : 

```txt
PROCEDURE <Nom de la procédure>(MaSélection est un edtHTMLSélection)
```
où &lt;MaSélection&gt; est une variable de type [edtHTMLSélection](../WDLang1/1000026093.md) correspondant à la sélection en cours dans le champ Editeur HTML.




## HTMLSurErreurJS (Fonction)

***En anglais : HTMLOnErrorJS***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique la procédure à appeler en cas d'erreur dans l'exécution du Javascript dans un champ d'affichage HTML. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HTMLSurErreurJS(HTM_Affichage, HTMLSurErreurJS_Callback)
PROCEDURE INTERNE HTMLSurErreurJS_Callback()
	Erreur(ErreurInfo())
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`HTMLSurErreurJS(<Champ d'affichage HTML> , <Procédure WLangage>)`
---

**`<Champ d'affichage HTML> : (Nom de champ)`**

Nom du champ d'affichage HTML à manipuler. 

**`<Procédure WLangage> : (Nom de procédure)`**

Procédure WLangage (également nommée "callback") appelée lorsqu'une erreur est rencontrée dans le code Javascript de la page. Cette procédure est de la forme : 

```txt
PROCEDURE <Nom de la procédure>(sErreur est une chaîne)
```
où &lt;sErreur&gt; est une chaîne de caractères correspondant à l'erreur Javascript rencontrée.
Dans le code de cette procédure, la fonction [ErreurInfo](../WDLang1/3013008.md) permet de connaître l'erreur rencontrée.

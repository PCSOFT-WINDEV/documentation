
## dbgEcritWarningAudit (Fonction)

***En anglais : dbgWriteWarningAudit***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ecrit un warning d'exécution personnalisé dans l'audit dynamique. Si l'audit n'est pas actif, la fonction **dbgEcritWarningAudit** est sans effet.

**Remarque** : L'audit dynamique peut être activé grâce à la fonction [dbgActiveAudit](../WDLang1/1000018835.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Vérifie qu'une fenêtre est bien ouverte
SI FenEtat("FenêtreToujoursOuverte") = Inexistant ALORS
	dbgEcritWarningAudit(1, "La fenêtre FenêtreToujoursOuverte a été fermée")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`dbgEcritWarningAudit(<Code> [, <Texte du warning>])`
---

**`<Code> : (Entier)`**

Code du warning à écrire.
Ce numéro (compris entre 1 et 1000) est défini par le développeur.

**`<Texte du warning> : (Chaîne de caractères optionnelle)`**

Message explicatif associé au warning.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La fonction [dbgEtatAudit](../WDLang1/1000018836.md) permet de savoir si l'audit est actif afin de : 

- n'effectuer les vérifications que dans ce cas. 

- ne pas réduire les performances de l'application en exécution normale.

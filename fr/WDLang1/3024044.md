
## DécomplèteRep (Fonction)

***En anglais : UncompleteDir***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Enlève le caractère '/' ou '\\' à la fin d'une chaîne si ce caractère est présent.

Cette fonction est utile pour la manipulation de noms de fichiers complets en cas d'incertitude sur le format du chemin (par exemple si l'utilisateur saisit un chemin de fichier).






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 
```wl
Res = DécomplèteRep("c:\temp\")         // Renvoie "c:\temp"
Res = DécomplèteRep("c:\temp")          // Renvoie "c:\temp"
Res = DécomplèteRep("\\CCS\Rep2\User\") // Renvoie "\\CCS\Rep2\User"
Res = DécomplèteRep("/root/file/")      // Renvoie "/root/file"
Res = DécomplèteRep("")                 // Renvoie ""
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = DécomplèteRep(<Chaîne à vérifier>)`
---

**`<Résultat> : (Chaîne de caractères)`**



- Chaîne de caractères &lt;Chaîne à vérifier&gt; non terminée par le caractère '\\' ou '/',

- Chaîne vide ("") si le paramètre &lt;Chaîne à vérifier&gt; correspond à une chaîne vide ("").




**`<Chaîne à vérifier> : (Chaîne de caractères)`**

Chaîne de caractères à vérifier.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Espaces
<a name="espaces_ELTPARAGRAPHE000068"></a>**Attention** : Les espaces avant et après &lt;Chaîne à vérifier&gt; sont supprimés (identique à la fonction [SansEspace](../WDLang1/3024034.md)).


<a name="NOTE0_2"></a>




### Fonction DécomplèteRep et UNICODE
<a name="fonction_docparampagetitleshort_unicode_ELTPARAGRAPHE000080"></a>Si la chaîne passée en paramètre est au format ANSI, le résultat sera au format ANSI.

Si la chaîne passée en paramètre est au format UNICODE, le résultat sera au format UNICODE.

**Remarque** : Si le résultat de la fonction **DécomplèteRep** utilisée sur une chaîne ANSI est affecté à une chaîne UNICODE (et inversement), la conversion ANSI/UNICODE sera implicitement réalisée.

Pour plus de détails, consultez [Gestion de l'UNICODE](../WDLang1/3024038.md).

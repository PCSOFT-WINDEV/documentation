
## AutoLanceSupprime (Fonction)

***En anglais : AutoRunDelete***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet d'arrêter le lancement automatiquement d'une application WINDEV. Ce lancement automatique a été préalablement défini avec la fonction [AutoLanceAjoute](../WDLang1/3019002.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// -- A partir d'une application WINDEV standard
// Lancement automatique de l'application "MonAppli" au démarrage
AutoLanceAjoute(alDémarrage, "MonAppli", fRepExe() + "\GestionClients.exe")
...
// Arrêt du lancement automatique de l'application "MonAppli"
AutoLanceSupprime(alDémarrage, "MonAppli")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`AutoLanceSupprime(<Événement du lancement> , <Application>)`
---

**`<Événement du lancement> : (Constante de type entier)`**

Événement qui provoquait le lancement de l'application :


|   |   |
| --- | --- |
| *alDémarrage* | Lancement de l'application lors du démarrage du poste PC en cours. |



**`<Application> : (Chaîne de caractères)`**

Nom de l'application défini avec la fonction [AutoLanceAjoute](../WDLang1/3019002.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Gestion des erreurs
<a name="gestion_des_erreurs_ELTPARAGRAPHE000164"></a>**Attention** : La fonction **AutoLanceSupprime** ne retourne pas de code d'erreur. Pour savoir si cette fonction a généré une erreur, utilisez la variable [ErreurDétectée](../WDLang1/3087001.md). En cas d'erreur, il est possible de connaître le détail de l'erreur grâce à la fonction [ErreurInfo](../WDLang1/3013008.md).




## &lt;Table&gt;.Modifie (Fonction)

***En anglais : &lt;Table&gt;.Modify***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie une ligne dans : 

- un champ Table, 

- un champ Table hiérarchique, 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) une table affichée dans un champ Combo.




**Remarque** : Il est conseillé d'utiliser la fonction [&lt;Table&gt;.ModifieLigne](../WDLang1/1000024275.md). La fonction [&lt;Table&gt;.ModifieLigne](../WDLang1/1000024275.md) permet d'ajouter :

- des caractères TAB dans les colonnes,

- des valeurs de n'importe quel type dans les colonnes.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Modification des éléments "Paul" et "Dubois" de la cinquième
// ligne du champ "TABLE_TableClient"
TABLE_TableClient.Modifie("Paul" + TAB + "Dubois", 5)
```
<a name="Exemple2"></a>

```wl
// Modification dans une colonne d'un champ Table
COL_NOM.Modifie("Dubois", 5)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Champ Table>.Modifie(<Ligne> [, <Indice>])`
---

**`<Champ Table> : (Nom de champ)`**

Nom du champ à manipuler. Ce champ peut correspondre à : 

- un champ Table.

- un champ Table hiérarchique. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) un champ Combo avec table.




**`<Ligne> : (Chaîne de caractères)`**

Ligne modifiée remplaçant la ligne spécifiée. Cette ligne est de la forme : 

```txt
<Elément de la colonne 1> + TAB + <Elément de la colonne 2> + TAB + ...
```


**`<Indice> : (Entier optionnel)`**

Indice de la ligne à modifier. La première ligne a pour valeur 1.


|   |   |
| --- | --- |
| Indice non précisé ou égal à -1 | La ligne en cours est modifiée. Cette constante n'a aucun effet si aucune ligne n'est sélectionnée. |
| Indice supérieur au nombre de lignes du champ | Une erreur du WLangage est générée. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000287"></a>La fonction **&lt;Table&gt;.Modifie** peut être utilisée indifféremment sur :

- un champ Table ou Table hiérarchique fichier.

- un champ Table ou Table hiérarchique par programmation.

- un champ monosélection ou multisélection.
	



<a name="NOTE0_2"></a>


### Caractéristiques d'une ligne
<a name="caracteristiques_une_ligne_ELTPARAGRAPHE000331"></a>

- Chaque élément de la ligne est converti dans le type de la colonne associée.

- Si un des éléments correspond à une chaîne vide ("") ou à 0 (selon le type de la colonne), sa valeur sera écrasée. 
	Par exemple :
	
	```wl
	TABLE_TableClient.Modifie("Dubois" + TAB + "Pierre" + TAB + "" + ...
			TAB + "21/06/72")
	```


- Si un élément correspond au résultat d'un calcul, il faut mettre l'expression numérique entre parenthèses. 
	Par exemple :
	
	```wl
	TABLE_TableCalcul.Modifie((53+29) + TAB + (83-21))
	```


- Pour modifier uniquement la valeur d'un élément dans une ligne, utilisez la syntaxe suivante :
	
	```txt
	NomTable.NomColonne[IndiceLigne] = NouvelleValeur
	```

### Champ Table/Table hiérarchique fichier : Modifications dans le fichier de données lié
<a name="champ_tabletable_hierarchique_fichier_modifications_dans_fichier_donnees_lie_ELTPARAGRAPHE000366"></a>**Si la saisie en cascade est activée**, le fichier de données lié au champ est automatiquement modifié lorsque l'utilisateur passe à la ligne suivante.

**Si la saisie en cascade n'est pas activée**, le fichier de données lié au champ n'est pas modifié. Pour écrire l'enregistrement dans le fichier de données lié, il est nécessaire d'utiliser la fonction [&lt;Table&gt;.Enregistre](../WDLang1/1000024249.md) après la fonction **&lt;Table&gt;.Modifie**.

**Remarques** : 

- L'option "Saisie en cascade" peut être sélectionnée dans l'onglet "Détail" de la fenêtre de description du champ.

- ![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) La saisie en cascade n'est pas disponible sur les champs Table fichier à accès direct. Cette option est disponible uniquement sur les champs Table par programmation et sur les champs Table fichier chargé en mémoire.



<a name="NOTE0_4"></a>


### Modifications réalisées dans un champ Table/Table hiérarchique fichier
<a name="modifications_realisees_dans_champ_tabletable_hierarchique_fichier_ELTPARAGRAPHE000398"></a>Pour modifier une ligne d'un champ Table/Table hiérarchique fichier, il est conseillé de :

- modifier directement l'enregistrement dans le fichier de données avec la fonction [&lt;Source&gt;.Modifie](../WDLang4/1000025038.md),

- utiliser la fonction [&lt;Table&gt;.Affiche](../WDLang1/1000023505.md) pour ré-actualiser le champ.

### Fonction &lt;Table&gt;.Modifie et propriété Modifié
<a name="fonction_docparampagetitleshort_propriete_modifie_ELTPARAGRAPHE000440"></a>La fonction **&lt;Table&gt;.Modifie** n'a aucune influence sur la propriété [Modifié](../Proprietes/2510080.md). La propriété [Modifié](../Proprietes/2510080.md) permet de savoir si la ligne en cours a été modifiée directement par l'utilisateur (saisie directe dans le champ).

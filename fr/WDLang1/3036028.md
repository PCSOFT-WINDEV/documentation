
## fCheminLong (Fonction)

***En anglais : fLongPath***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le chemin complet long d'un fichier ou d'un répertoire.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003036028&name=fcheminlong_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Récupération du chemin long d'un fichier
ResCheminLong = fCheminLong("C:\MesRep~1\Compte~1.doc")
// fCheminLong renvoie "C:\MesRépertoires\Compte-Rendu Mars 2000.doc"
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = fCheminLong(<Chemin du fichier> [, <Option>])`
---

**`<Résultat> : (Chaîne de caractères)`**



- Nom long correspondant au chemin complet du fichier ou du répertoire. Si le paramètre &lt;Chemin du fichier&gt; se termine par le caractère "\\", ce caractère est conservé.

- Chaîne vide ("") en cas d'erreur (par exemple si le fichier ou le répertoire spécifié n'existe pas). Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.




**`<Chemin du fichier> : (Chaîne de caractères)`**

Nom et chemin complet (ou relatif) du fichier ou du répertoire (260 caractères maximum). Un chemin de type UNC peut être utilisé. 
Ce paramètre peut être au format Ansi ou Unicode.

**`<Option> : (Constante de type Entier)`**

Permet de spécifier le format du résultat de la fonction. 


|   |   |
| --- | --- |
| *fCheminUNICODE* | &lt;Résultat&gt; sera une chaîne de caractères au format Unicode. |


**Remarque** : Si la constante *fCheminUNICODE* n'est pas spécifiée et si la fonction doit renvoyer une chaîne de caractères Unicode, &lt;Résultat&gt; correspondra au chemin court du fichier.

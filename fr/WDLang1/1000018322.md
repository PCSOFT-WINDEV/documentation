


## Défile (Fonction)

***En anglais : Dequeue***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Défile un élément d'une file. L'élément est supprimé de la file et renvoyé dans la variable spécifiée.  




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MaFile est une File d'entiers

// Enfile les valeurs 1, 2 et 3
Enfile(MaFile, 1)
Enfile(MaFile, 2)
Enfile(MaFile, 3)

// Affichage de la trace : 1, 2, 3
x est un entier
TANTQUE Défile(MaFile, x)
	Trace(x)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Récupérer l'élément au début de la file

`<Résultat> = Défile(<File> [, <Elément>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'élément a été défilé, 

- <u><u><u><u>Faux</u></u></u></u> si la file est vide.




**`<File> : (File)`**

Nom de la variable de type [File](../WDLang1/1514021.md) à manipuler.

**`<Elément> : (Type d'éléments manipulés par la file, paramètre optionnel)`**

Nom de la variable dans laquelle la valeur de l'élément défilé est renvoyée. Si ce paramètre n'est pas spécifié, l'élément est défilé mais sa valeur n'est pas récupérée.


<a name="SYNTAXE2"></a>

### Récupérer l'élément au début de la file (syntaxe multithread)

`<Résultat> = Défile(<File> [, <Elément> [, <Options> [, <Attente>]]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'élément a été défilé, 

- <u><u><u><u>Faux</u></u></u></u> si le temps d'attente est dépassé (paramètre &lt;Attente&gt;) ou si la propriété [AjoutTerminé](../Proprietes/1000018388.md) passe à <u><u><u><u>Vrai</u></u></u></u>.




**`<File> : (File)`**

Nom de la variable de type [File](../WDLang1/1514021.md) à manipuler.

**`<Elément> : (Type d'éléments manipulés par la file, paramètre optionnel)`**

Nom de la variable dans laquelle la valeur de l'élément défilé est renvoyée. Si ce paramètre n'est pas spécifié, l'élément est défilé mais sa valeur n'est pas récupérée.

**`<Options> : (Constante optionnelle de type Entier)`**

Option de gestion de la file :



|   |   |
| --- | --- |
| *AttendSiVide* | Attend qu'un élément soit ajouté à la file par un autre thread. |



**`<Attente> : (Entier optionnel ou Durée optionnelle)`**

Temps d'attente maximal exprimé en centièmes de seconde. Ce paramètre peut correspondre à : 

- un entier correspondant au nombre de centièmes de seconde, 

- une variable de type Durée, 

- l'indication directe de la durée (par exemple 1 s ou 10 ms). 


Par défaut, ce paramètre correspond à la constante *Infini* (attente sans aucun délai maximal).

## Remarques


### Fonctionnement dans le cas d'une gestion multithread
<a name="fonctionnement_dans_cas_une_gestion_multithread_ELTPARAGRAPHE000147"></a>Si un élément est présent dans la file, il est supprimé de la file et renvoyé par le paramètre &lt;Elément&gt; par référence.

Si la file est vide, la fonction **Défile** se bloque en attendant qu'un élément soit ajouté dans la file. Le temps d'attente maximal peut être spécifié par le paramètre &lt;Attente&gt;.

La fonction renvoie <u><u><u><u>Vrai</u></u></u></u> si un élément a été défilé, <u><u><u><u>Faux</u></u></u></u> si le temps d'attente est dépassé ou si la propriété [AjoutTerminé](../Proprietes/1000018388.md) passe à <u><u><u><u>Vrai</u></u></u></u>.

La propriété [NbThreadAttente](../Proprietes/1000018387.md) renvoie le nombre de threads en attente.

L'affectation de la propriété [AjoutTerminé](../Proprietes/1000018388.md) à <u><u><u><u>Vrai</u></u></u></u> provoque la fin de l'attente pour tous les threads en attente. La fonction **Défile** renvoie alors <u><u><u><u>Faux</u></u></u></u>.

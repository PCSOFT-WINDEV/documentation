


## EnModeAWP (Fonction)

***En anglais : InAWPMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de savoir si la page en cours d'exécution est une page AWP. 



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code exécuté depuis une page AWP ? 
SI EnModeAWP() = Vrai ALORS
	// Mode AWP
	// Traitements à effectuer
	...
SINON
	// Mode non AWP
	// Traitements à effectuer
	...
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = EnModeAWP()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la page exécutée est une page AWP WEBDEV, 

- <u><u><u><u>Faux</u></u></u></u> dans tous les autres cas.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000207"></a>La fonction **EnModeAWP** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeAWP** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.

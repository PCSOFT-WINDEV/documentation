
## FenDésactiveEffet (Fonction)

***En anglais : WinDisableEffect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Désactive des effets visuels du moteur graphique de WINDEV (pour une utilisation sous TSE par exemple). Cette fonction est utile pour permettre à une application de fonctionner sur un poste ayant des capacités graphiques limitées ou pour le mode TSE si la bande passante est limitée.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI EnModeTSE() = Vrai ALORS
	FenDésactiveEffet(effetAnimation + effetCadreTranslucide + effetGFI + effetHalo)
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`FenDésactiveEffet(<Effet à désactiver>)`
---

**`<Effet à désactiver> : (Constante de type Entier ou combinaison de constantes)`**

L'effet à désactiver correspond à une ou plusieurs des constantes suivantes :


|   |   |
| --- | --- |
| *effetAnimation* | Désactive les animations des fenêtres. |
| *effetCadreTranslucide* | Désactive le cadre translucide des fenêtres (cadre utilisé uniquement sous Windows XP ou 2003). |
| *effetGFI* | Désactive le grisage des fenêtres inaccessibles. |
| *effetHalo* | Désactive le halo autour des boutons. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction **FenDésactiveEffet** est équivalente :

	- à la fonction [StyleDessin](../WDLang1/3014030.md) utilisée pour le paramétrage de l'effet translucide des fenêtres.

	- à la fonction [FenEffetInactif](../WDLang1/3038055.md) pour le paramétrage du GFI.




- En TSE, par défaut : 

	- les animations des fenêtres sont désactivées. 

	- l'animation liée au GFI (Grisage automatique des fenêtres) est désactivée. Le GFI est activé par défaut en TSE.

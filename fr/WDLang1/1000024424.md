
## EvénementExiste (Fonction)

***En anglais : EventExist***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique si une procédure spécifique est exécutée lorsqu'un événement Windows est envoyé à un champ ou une fenêtre.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Demande à recevoir le clic si pas encore fait
WM_LBUTTONDOWN est un entier = 513
SI PAS EvénementExiste(SAI_MonChamp, WM_LBUTTONDOWN) ALORS
   Evénement(ClicSouris, "SAI_MonChamp", WM_LBUTTONDOWN)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = EvénementExiste(<Elément> , <Evénement Windows>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si : 

	- La fonction [Evénement](../WDLang1/3015011.md) a été appelée sur le champ, 

	- Un événement optionnel a été ajouté en édition sous l'éditeur de code pour le champ. 




- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 




**`<Elément> : (Nom de champ ou nom de fenêtre)`**

Nom du champ ou de la fenêtre pour lequel l'événement est recherché. 

**`<Evénement Windows> : (Entier)`**

Numéro ou nom de l'événement Windows à gérer. Consultez la [liste des événements Windows](../AppelDLL/6510001.md). 
Remarque : Il est possible d'intégrer les constantes Windows directement dans votre code WLangage grâce au mot-clé [EXTERNE](../Motscles/1511013.md).




## EnModeWeb (Fonction)

***En anglais : InWebMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie le mode de lancement du traitement en cours :

- depuis un exécutable WINDEV.

- depuis le moteur d'exécution de WEBDEV (AWP).




**Remarque** : Pour les codes partagés entre une application WINDEV et un site WEBDEV (code de classe, composant, ...), cette fonction permet de connaître le mode d'exécution, et de lancer le traitement approprié.

![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Dans cette version, cette fonction n'a aucun effet.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Mode de lancement ?
SI EnModeWeb() = Vrai ALORS
	// Traitement lancé depuis le moteur d'exécution de WEBDEV : 
	// Appel de la procédure "WebEnvoiePage"
	WebEnvoiePage("Info.htm")
SINON
	// Traitement lancé depuis un exécutable WINDEV 
	// Affichage d'une boîte de dialogue
	Info("Action réalisée")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = EnModeWeb()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté avec le moteur d'exécution de WEBDEV, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.


![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Ce paramètre correspond toujours à <u><u><u><u>Faux</u></u></u></u>.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Partages de composants et de classes entre WINDEV et WEBDEV
<a name="partages_composants_classes_entre_windev_webdev_ELTPARAGRAPHE000112"></a>La fonction **EnModeWeb** permet de partager simplement des classes ou des composants réalisés en WLangage entre un site WEBDEV et une application WINDEV. Selon si l'exécution est réalisée en mode Web ou non, il est possible d'utiliser des fonctions différentes, spécifiques à l'application en cours.
<a name="NOTE0_2"></a>


### Exécution des traitements contenant des erreurs de compilation
<a name="execution_des_traitements_contenant_des_erreurs_compilation_ELTPARAGRAPHE000254"></a>Par défaut, le mécanisme de sécurité du WLangage se déclenche uniquement si la ligne contenant l'erreur de compilation est exécutée.

Par exemple, si une fonction interdite dans le produit en cours est présente dans une boucle SI, le mécanisme de sécurité du WLangage se déclenchera uniquement si la ligne contenant cette fonction s'exécute.
<a name="NOTE0_3"></a>


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000266"></a>La fonction **EnModeWeb** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeWeb** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.

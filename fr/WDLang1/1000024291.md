


## &lt;Table&gt;.SupprimeSelect (Fonction)

***En anglais : &lt;Table&gt;.DeleteSelect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime la ou les lignes sélectionnées dans un champ Table ou Table hiérarchique.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Supprime toutes les lignes sélectionnées
SI OuiNon("Voulez-vous vraiment supprimer les clients sélectionnés ?") ALORS
	TABLE_Affichage_Client.SupprimeSelect()
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Table>.SupprimeSelect()`
---

**`<Champ Table> : (Nom de champ)`**

Nom du champ à manipuler. Ce champ peut correspondre à : 

- un champ Table.

- un champ Table hiérarchique.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000209"></a>La fonction **&lt;Table&gt;.SupprimeSelect** peut être utilisée indifféremment sur : 

- un champ Table ou Table hiérarchique fichier.

- un champ Table ou Table hiérarchique par programmation.

- un champ monosélection ou multisélection. 
	



<a name="NOTE0_2"></a>


### Suppression dans un champ Table ou Table hiérarchique fichier
<a name="suppression_dans_champ_table_table_hierarchique_fichier_ELTPARAGRAPHE000237"></a>La fonction **&lt;Table&gt;.SupprimeSelect** supprime les enregistrements correspondants dans le fichier de données associé au champ. Si des rubriques sont reliées à d'autres fichiers de données, aucune modification n'est effectuée automatiquement sur ces fichiers de données liés.
<a name="NOTE0_3"></a>


### Intégrité référentielle et champ Table ou Table hiérarchique fichier
<a name="integrite_referentielle_champ_table_table_hierarchique_fichier_ELTPARAGRAPHE000250"></a>Si la gestion de l'intégrité référentielle est active (fonction [&lt;Variable Connexion&gt;.GèreIntégrité](../WDLang4/1000023960.md)), testez la fonction [HErreurIntégrité](../WDLang4/3044086.md) après l'appel de la fonction **&lt;Table&gt;.SupprimeSelect** pour vérifier l'intégrité. Si une erreur d'intégrité est détectée, la ligne n'est pas supprimée et reste visible dans le champ.
<a name="NOTE0_4"></a>


### Suppression dans un champ Table fichier et Trigger
<a name="suppression_dans_champ_table_fichier_trigger_ELTPARAGRAPHE000271"></a>Il est possible de déclencher un trigger lors de la suppression d'un enregistrement dans un champ Table fichier. Pour plus de détails, consultez la documentation de la fonction [&lt;Source&gt;.DécritTrigger](../WDLang4/1000024216.md).
<a name="NOTE0_5"></a>


### Gestion des erreurs
<a name="gestion_des_erreurs_ELTPARAGRAPHE000286"></a>En cas d'échec de la suppression, la variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. Pour plus de détails sur l'erreur, il est possible d'utiliser la fonction [ErreurInfo](../WDLang1/3013008.md).
<a name="NOTE0_6"></a>


### Blocage du fichier de données lié
<a name="blocage_fichier_donnees_lie_ELTPARAGRAPHE000306"></a>Si le fichier de données est bloqué dans le traitement en cours, la fonction **&lt;Table&gt;.SupprimeSelect** supprime la ligne spécifiée et débloque le fichier de données.




## &lt;Kanban&gt;.Affiche (Fonction)

***En anglais : &lt;Kanban&gt;.Display***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Rafraîchit l'affichage d'un champ Kanban ou d'une carte d'un champ Kanban.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Tache.Ajoute()
KANBAN_Taches.Affiche()
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Kanban>.Affiche([<Carte>])`
---

**`<Champ Kanban> : (Nom de champ)`**

Nom du champ Kanban à rafraîchir. 

**`<Carte> : (Variable de type kbCarte optionnel)`**

Nom de la variable de type [kbCarte](../WDLang1/1410089167.md) correspondant à la carte pour laquelle l'affichage doit être rafraichi. Si ce paramètre n'est pas spécifié, la totalité du champ Kanban est rafraîchi.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Cas 1 : Seul le champ Kanban est précisé
<a name="cas_1_seul_champ_kanban_est_precise_ELTPARAGRAPHE000180"></a>

- Si le champ Kanban est lié à une source de données (fichier ou tableau WLangage), le champ est réinitialisé à partir de cette source. 

- Si le champ Kanban est manipulé par programmation, les événements "Demande de mise à jour d'affichage" de chaque fenêtre interne (ou page interne) affichée sont ré-exécutés.



<a name="NOTE0_2"></a>


### Cas 2 : Le champ Kanban et la carte sont précisés
<a name="cas_2_champ_kanban_carte_sont_precises_ELTPARAGRAPHE000191"></a>

- Si la carte ne se trouve pas dans une liste de Kanban, la fonction n'a aucun effet. 

- Si le champ Kanban est lié à une source de données (fichier ou tableau WLangage), la carte est réinitialisée à partir de la source. 

- Si le champ est manipulé par programmation, l'événement "Demande de mise à jour d'affichage" de la fenêtre interne (ou de la page interne) liée est ré-exécuté.




## BarrePosition (Fonction)

***En anglais : ToolbarPosition***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique le mode d'affichage d'un champ Barre d'outils.

**Rappel** : Un champ Barre d'outils peut être "flottant" ou ancré aux bords de la fenêtre par un de ses côtés.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI BarrePosition(BAR_Barre1) = barLibre ALORS
	BarreAttache(BAR_Barre1, barGauche)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = BarrePosition(<Champ Barre d'outils>)`
---

**`<Résultat> : (Constante optionnelle)`**

Position du champ Barre d'outils : 


|   |   |
| --- | --- |
| *barBas* | Le champ Barre d'outils est ancré en bas de la fenêtre principale. |
| *barDroite* | Le champ Barre d'outils est ancré à droite de la fenêtre principale. |
| *barGauche* | Le champ Barre d'outils est ancré à gauche de la fenêtre principale. |
| *barHaut*<br>(Valeur par défaut) | Le champ Barre d'outils est ancré en haut de la fenêtre principale. |
| *barLibre* | Champ Barre d'outils "flottant". |



**`<Champ Barre d'outils> : (Nom de champ)`**

Nom du champ Barre d'outils à manipuler.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Le mot-clé [MaFenêtre](../Motscles/1511004.md) ne doit pas être utilisé : 

- dans les événements associés au champ Barre d'outils. 

- dans les événements associés aux champs présents dans la barre d'outils.

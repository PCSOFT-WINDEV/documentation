
## fSéparateur (Fonction)

***En anglais : fSeparator***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie un chemin de fichier avec des séparateurs normalisés (séparateurs correspondants à ceux de la plateforme en cours par exemple). 

**Note** : Cette fonction est très utile pour utiliser un code WLangage exécuté sur plusieurs plateformes (Windows/Linux par exemple) et qui manipule un chemin de fichier.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Renvoie un chemin de type Unix
Trace(fSéparateur(fRepEnCours(), fSéparateurUnix))

// Renvoie un chemin de type Windows
Trace(fSéparateur(fRepEnCours(), fSéparateurWindows))
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = fSéparateur(<Chemin initial> [, <Séparateur>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Chemin normalisé. 

**Remarque** : &lt;Résultat&gt; est de même type que le paramètre &lt;Chemin initial&gt;. Par exemple, si le &lt;Chemin initial&gt; est une chaîne ANSI, &lt;Résultat&gt; sera une chaîne de caractères ANSI. 

**`<Chemin initial> : (Chaîne de caractères (ANSI ou UNICODE))`**

Chemin du répertoire ou du fichier à normaliser. 

**`<Séparateur> : (Chaîne de caractères optionnelle ou constante de type Entier optionnelle)`**

Séparateur à utiliser pour la normalisation du chemin. Peut correspondre : 

- soit à une chaîne de caractères : tous les séparateurs (\\ et /) trouvés dans le paramètre &lt;Chemin initial&gt; seront remplacés par cette chaîne. 

- soit à une des constantes suivantes : 
	


|   |   |
| --- | --- |
| *fSéparateurPlateforme*<br>(Valeur par défaut) | Les séparateurs ('\\' et '/') seront remplacés par le séparateur correspondant à la plateforme en cours. |
| *fSéparateurUnix* | Les séparateurs ('\\' et '/') seront remplacés par le séparateur '/'. |
| *fSéparateurWindows* | Les séparateurs ('\\' et '/') seront remplacés par le séparateur '\\'. |

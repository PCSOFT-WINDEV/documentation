


## EnModeServeurWebSocket (Fonction)

***En anglais : InWebSocketServerMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de savoir si l'exécution en cours est effectuée dans le serveur de WebSocket.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI EnModeServeurWebSocket() ALORS
	// Code spécifique au serveur de WebSocket
SINON
	// Code à exécuter dans les autres cas
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = EnModeServeurWebSocket()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté dans le serveur de WebSocket, 

- <u><u><u><u>Faux</u></u></u></u> dans tous les autres cas. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000160"></a>La fonction **EnModeServeurWebSocket** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeServeurWebSocket** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.

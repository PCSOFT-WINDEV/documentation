
## &lt;Variable WDPic&gt;.DessinePolygone (Fonction)

***En anglais : &lt;WDPic variable&gt;.DrawPolygon***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Dessine un polygone dans une variable de type [WDPic](../WDLang1/1000023503.md).![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les dessins](../WDLang1/3029035.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MonImageWDPic est un WDPic = "Test.bmp"
// Dessin d'un polygone à 4 côtés dont le 
// fond est jaune clair et le bord bleu clair
MonImageWDPic.DessinePolygone(4, 10, 20, 20, 50, 40, 60, 50, 10, JauneClair, BleuClair)
IMG_MonDessin = MonImageWDPic
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Construction d'un polygone point par point

`<Image WDPic>.DessinePolygone(<Nombre de côtés> , <X1> , <Y1> , <X2> , <Y2> [, <X3> , <Y3> [... [, <Xn> , <Yn>]]] [, <Couleur du fond> [, <Couleur des côtés>]])`
---

**`<Image WDPic> : (Variable de type WDPic)`**

Nom de la variable de type [WDPic](../WDLang1/1000023503.md) à manipuler. Seul le calque d'arrière-plan sera manipulé.

**`<Nombre de côtés> : (Entier)`**

Nombre de côtés du polygone (23 côtés maximum). Ce paramètre détermine le nombre de couples &lt;X&gt;, &lt;Y&gt; à spécifier dans la fonction.

**`<X1> : (Entier)`**

Abscisse du premier point du polygone. Ces coordonnées sont exprimées en pixels.

**`<Y1> : (Entier)`**

Ordonnée du premier point du polygone. Ces coordonnées sont exprimées en pixels.

**`<X2> : (Entier)`**

Abscisse du second point du polygone. Ces coordonnées sont exprimées en pixels.

**`<Y2> : (Entier)`**

Ordonnée du second point du polygone. Ces coordonnées sont exprimées en pixels.

**`<X3> : (Entier)`**

Abscisse du troisième point du polygone. Ces coordonnées sont exprimées en pixels.

**`<Y3> : (Entier)`**

Ordonnée du troisième point du polygone. Ces coordonnées sont exprimées en pixels.



**`<Xn> : (Entier optionnel)`**

Abscisse du nième point du polygone. Ces coordonnées sont exprimées en pixels.

**`<Yn> : (Entier optionnel)`**

Ordonnée du nième point du polygone. Ces coordonnées sont exprimées en pixels.

**`<Couleur du fond> : (Entier ou constante (optionnel))`**

Couleur du fond du polygone. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md).




Si ce paramètre n'est pas précisé, la couleur du fond :

- est *Transparent* si la fonction [&lt;Variable WDPic&gt;.Fond](../WDLang1/1000024481.md) n'a pas été utilisée auparavant,

- correspond à la couleur précisée dans le dernier appel à la fonction [&lt;Variable WDPic&gt;.Fond](../WDLang1/1000024481.md).




**`<Couleur des côtés> : (Entier ou constante (optionnel))`**

Couleur des côtés du polygone. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md).




Si ce paramètre n'est pas précisé, la couleur du trait :

- correspond à la couleur précisée dans le dernier appel à la fonction [&lt;Variable WDPic&gt;.Stylo](../WDLang1/1000024636.md),

- est identique à la couleur utilisée pour le fond si la fonction [&lt;Variable WDPic&gt;.Stylo](../WDLang1/1000024636.md) n'a pas été utilisée.

## Remarques


### Coordonnées
<a name="coordonnees_ELTPARAGRAPHE000787"></a>Tracé d'un polygone :

```wl
MonImage est WDPic
MonImage.DessinePolygone(5, 80, 10, 10, 150, 100, 200, 160, 80, 140, 40)
```
<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=rbPolygo1.gif)
Les coordonnées sont indiquées par rapport au coin haut gauche de l'image (coordonnées (0,0)).

**Important** : Il n'est pas nécessaire de fermer le polygone (les coordonnées du dernier point n'ont pas à être égales aux coordonnées du premier point). La "fermeture" du polygone est réalisée automatiquement.

**Limites** : un polygone peut avoir au maximum 100 000 points en WINDEV, WEBDEV et Java, et 1000 points sous Windows CE.
<a name="NOTE0_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 

### Dessin avec opacité ou anti-aliasing
<a name="dessin_avec_opacite_antialiasing_ELTPARAGRAPHE000822"></a>Dans le cas d'un dessin avec opacité (fonction [&lt;Variable WDPic&gt;.DébutDessin](../WDLang1/1000024471.md) avec la constante *dAvecOpacité*) ou anti-aliasing (fonction [&lt;Variable WDPic&gt;.ChangeMode](../WDLang1/1000024405.md) avec la constante *dessinAntiAliasing*), le contour est dessiné par-dessus l'intérieur du polygone : le trait est donc en partie combiné avec l'intérieur du polygone. Si l'épaisseur du trait est importante, le trait s'affichera en deux couleurs.


## DéchargeWDL (Fonction)

***En anglais : UnloadWDL***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Décharge la bibliothèque précédemment chargée en mémoire avec la fonction [ChargeWDL](../WDLang1/3013020.md). La mémoire est libérée.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code de fermeture de la première fenêtre du projet
DéchargeWDL(hWnd) // Fermeture de la bibliothèque
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`DéchargeWDL(<Identifiant de la bibliothèque>)`
---

**`<Identifiant de la bibliothèque> : (Entier)`**

Identifiant de la bibliothèque à décharger, renvoyée par la fonction [ChargeWDL](../WDLang1/3013020.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La bibliothèque principale du projet est toujours chargée et ne peut pas être déchargée.

- Lorsqu'une bibliothèque est fermée grâce à la fonction **DéchargeWDL**, les fichiers HFSQL ouverts dans la bibliothèque ne sont pas automatiquement fermés. Dans ce cas, tout accès à ces fichiers HFSQL provoquera l'erreur "La WDL xxx a été déchargée".
	Il est conseillé d'utiliser la fonction [HFerme](../WDLang4/3044073.md) sur les fichiers ouverts d'une bibliothèque qui va être déchargée.




## &lt;Table&gt;.Position (Fonction)

***En anglais : &lt;Table&gt;.Position***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Affiche un champ Table ou Table hiérarchique à partir d'une ligne spécifiée ou renvoie l'indice de la première ligne affichée dans un champ Table ou Table hiérarchique.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Indice de la première ligne affichée dans le champ "TABLE_Produit"
ResPosition = TABLE_Produit.Position()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Afficher un champ Table ou Table hiérarchique à partir d'une ligne spécifiée

`<Champ Table>.Position(<Indice> [, <Option>])`
---

**`<Champ Table> : (Nom de champ)`**

Nom du champ Table ou Table hiérarchique à manipuler.

**`<Indice> : (Entier)`**

Indice de la ligne qui doit être affichée dans le champ. Par défaut, cette ligne est affichée en premier. 
Si &lt;Indice&gt; est supérieur au nombre de lignes présentes dans le champ (connu avec la fonction [&lt;Table&gt;.Occurrence](../WDLang1/1000024277.md)), la dernière ligne est sélectionnée par défaut.

**`<Option> : (Constante de type Entier)`**

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Position de la ligne dans le champ : 


|   |   |
| --- | --- |
| *taPremier*<br>(Valeur par défaut) | La ligne spécifiée par &lt;Indice&gt; est affichée en premier dans le champ Table/Table hiérarchique. |
| *taVisible* | La ligne spécifiée par &lt;Indice&gt; devient visible dans le champ Table/Table hiérarchique. Elle est affichée : <br><br>- en premier (si &lt;Indice&gt; est inférieur à la ligne en cours). <br><br>- en dernier (si &lt;Indice&gt; est supérieur à la ligne en cours).<br><br><br>Si la ligne est déjà visible, l'affichage du champ Table/Table hiérarchique ne change pas. |




<a name="SYNTAXE2"></a>

### Connaître l'indice de la première ligne affichée dans un champ Table ou Table hiérarchique

`<Résultat> = <Champ Table>.Position()`
---

**`<Résultat> : (Entier)`**

Indice de la première ligne affichée dans le champ spécifié.


**`<Champ Table> : (Nom de champ)`**

Nom du champ Table ou Table hiérarchique à manipuler.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000295"></a>La fonction **&lt;Table&gt;.Position** peut être utilisée indifféremment :

- un champ Table ou Table hiérarchique fichier.

- un champ Table ou Table hiérarchique par programmation.

- un champ monosélection ou multisélection.





### Afficher une ligne en premier (syntaxe 1)
<a name="afficher_une_ligne_premier_syntaxe_1_ELTPARAGRAPHE000321"></a>La ligne affichée en premier peut être différente de celle demandée. En effet, la fonction **&lt;Table&gt;.Position** essaie d'afficher un nombre maximum de lignes dans le champ.

La fonction **&lt;Table&gt;.Position** n'affichera la ligne spécifiée que si cette ligne est suivie d'un nombre suffisant de lignes non vides. Dans l'exemple suivant, la ligne "Durand" ne peut pas être affichée au début du champ Table.<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=TABLEPOSITION.gif)

<a name="NOTE0_2"></a>


### Bandeau de sélection et fonction &lt;Table&gt;.Position
<a name="bandeau_selection_fonction_docparampagetitleshort_ELTPARAGRAPHE000340"></a>La fonction **&lt;Table&gt;.Position** ne modifie pas la ligne en cours (c'est-à-dire la ligne sélectionnée à l'aide du bandeau de sélection).

Ainsi, si le champ Table a le focus lorsque la fonction **&lt;Table&gt;.Position** est exécutée, la ligne en cours devra être visible en même temps que la ligne spécifiée. Si la ligne en cours et la ligne spécifiée ne peuvent pas être affichées en même temps, le contenu du champ sera affiché à partir de la ligne en cours (et non à partir de la ligne &lt;Indice&gt;).

### Champ Table fichier : limitations de la fonction &lt;Table&gt;.Position
<a name="champ_table_fichier_limitations_fonction_docparampagetitleshort_ELTPARAGRAPHE000362"></a>La fonction **&lt;Table&gt;.Position** ne fonctionne pas pour les champs Table fichier en mode "manuel" qui utilisent un ascenseur à rebond (option "Ascenseur proportionnel" non cochée dans l'onglet "Détail" de la description du champ).

**Rappel** :

- Un champ Table fichier en mode "manuel" est un champ qui est relié par programmation à un fichier de données.

- Avec un ascenseur à rebond, il est possible de faire défiler l'ascenseur en gardant le bouton de la souris enfoncé sur le bouton de l'ascenseur ou en gardant la touche flèche haut ou flèche bas appuyée.



<a name="NOTE0_4"></a>


### Evénement "Entrée en saisie d'une ligne"
<a name="evenement_entree_saisie_une_ligne_ELTPARAGRAPHE000382"></a>

- La fonction **&lt;Table&gt;.Position** ne force pas l'exécution de l'événement "Entrée en saisie d'une ligne".

- La fonction **&lt;Table&gt;.Position** ne peut pas être utilisée dans l'événement "Entrée en saisie d'une ligne" du champ (le résultat retourné est incorrect).

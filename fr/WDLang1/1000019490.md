


## EnModeFabrique (Fonction)

***En anglais : InFactoryMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie si le code est exécuté par un plan d'actions de la fabrique logicielle. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code exécuté dans un plan d'actions 
SI EnModeFabrique() = Vrai ALORS
	// Code spécifique à la fabrique logicielle
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = EnModeFabrique()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté par un plan d'actions de la fabrique logicielle, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000204"></a>La fonction **EnModeFabrique** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeFabrique** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.

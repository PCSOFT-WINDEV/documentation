


## Dépile (Fonction)

***En anglais : Pop***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Dépile un élément d'une pile. L'élément au sommet de la pile est supprimé et renvoyé dans la variable spécifiée.  




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MaPile est une Pile d'entiers

// On empile les valeurs 1, 2 et 3
Empile(MaPile, 1)
Empile(MaPile, 2)
Empile(MaPile, 3)

// Affichage de la trace : 3, 2, 1
x est un entier
TANTQUE Dépile(MaPile, x) = Vrai
	Trace(x)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Récupérer l'élément au sommet de la pile

`<Résultat> = Dépile(<Pile> [, <Elément>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'élément a été dépilé, 

- <u><u><u><u>Faux</u></u></u></u> si la pile est vide.




**`<Pile> : (Pile)`**

Nom de la variable de type [Pile](../WDLang1/1514022.md) à manipuler.

**`<Elément> : (Type d'éléments manipulés par la pile, paramètre optionnel)`**

Nom de la variable dans laquelle la valeur de l'élément dépilé est renvoyée. Si ce paramètre n'est pas spécifié, l'élément est dépilé mais sa valeur n'est pas récupérée.


<a name="SYNTAXE2"></a>

### Récupérer l'élément au sommet de la pile (syntaxe multithread)

`<Résultat> = Dépile(<Pile> [, <Elément> [, <Options> [, <Attente>]]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'élément a été dépilé, 

- <u><u><u><u>Faux</u></u></u></u> si le temps d'attente est dépassé (paramètre &lt;Attente&gt;) ou si la propriété [AjoutTerminé](../Proprietes/1000018388.md) passe à <u><u><u><u>Vrai</u></u></u></u>.




**`<Pile> : (Pile)`**

Nom de la variable de type [Pile](../WDLang1/1514022.md) à manipuler.

**`<Elément> : (Type d'éléments manipulés par la pile, paramètre optionnel)`**

Nom de la variable dans laquelle la valeur de l'élément dépilé est renvoyée. Si ce paramètre n'est pas spécifié, l'élément est dépilé mais sa valeur n'est pas récupérée.

**`<Options> : (Constante optionnelle de type Entier)`**

Option de gestion de la pile :


|   |   |
| --- | --- |
| *AttendSiVide* | Attend qu'un élément soit ajouté à la pile par un autre thread. |



**`<Attente> : (Entier optionnel)`**

Temps d'attente maximal exprimé en centièmes de seconde. Ce paramètre peut correspondre à : 

- un entier correspondant au nombre de centièmes de seconde, 

- une variable de type Durée, 

- l'indication directe de la durée (par exemple 1 s ou 10 ms). 


Par défaut, ce paramètre correspond à la constante *Infini* (attente sans aucun délai maximal).

## Remarques


### Fonctionnement dans le cas d'une gestion multithread
<a name="fonctionnement_dans_cas_une_gestion_multithread_ELTPARAGRAPHE000146"></a>Si la pile est vide, la fonction **Dépile** se bloque en attendant qu'un élément soit ajouté dans la pile. Le temps d'attente maximal peut être spécifié par le paramètre &lt;Attente&gt;.

La fonction renvoie <u><u><u><u>Vrai</u></u></u></u> si un élément a été dépilé, <u><u><u><u>Faux</u></u></u></u> si le temps d'attente est dépassé ou si la propriété [AjoutTerminé](../Proprietes/1000018388.md) passe à <u><u><u><u>Vrai</u></u></u></u>.

La propriété [NbThreadAttente](../Proprietes/1000018387.md) renvoie le nombre de threads en attente.

L'affectation de la propriété [AjoutTerminé](../Proprietes/1000018388.md) à <u><u><u><u>Vrai</u></u></u></u> provoque la fin de l'attente pour tous les threads en attente. La fonction **Dépile** renvoie alors <u><u><u><u>Faux</u></u></u></u>.




## &lt;Editeur HTML&gt;.Recherche (Fonction)

***En anglais : &lt;HTML editor&gt;.Search***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Recherche une chaîne de caractères dans un champ Editeur HTML. La recherche ne tient pas compte de la mise en forme de la chaîne de caractères.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Compte le nombre de WINDEV
// (y compris si le WIN est en gras)
nbMot est entier = 0
EDHTML_MonEditeurHTML.Recherche("WINDEV",   ...
		EditeurHTMLRecherche_Callback, MotComplet + SansCasse)

PROCEDURE INTERNE EditeurHTMLRecherche_Callback(nPos est un entier)
	SI nPos = 0 ALORS
		// Fin du parcours
		Info("Nombre de WINDEV : " + nbMot)
		RETOUR 
	FIN
	// Trouvé
	nbMot++
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Editeur HTML>.Recherche(<Texte à rechercher> , <Procédure WLangage> [, <Options> [, <Nombre maximum>]])`
---

**`<Champ Editeur HTML> : (Nom de champ)`**

Nom du champ Editeur HTML manipulé.

**`<Texte à rechercher> : (Chaîne de caractères)`**

Texte à rechercher dans le champ Editeur HTML.

**`<Procédure WLangage> : (Nom de procédure)`**

Procédure WLangage (également nommée "callback") exécutée pour chaque élément trouvé. Cette procédure est de la forme : 

```txt
PROCEDURE <Nom de la procédure>(nPosition est un entier)
```
où &lt;nPosition&gt; est un entier correspondant à la position du texte recherché par rapport au texte sans balise. Cette valeur peut être utilisée : 

- avec la propriété [Curseur](../Proprietes/2510025.md), 

- avec la fonction [&lt;Editeur HTML&gt;.Sélectionne](../WDLang1/1000026307.md). 


A la fin du parcours, &lt;nPos&gt; vaut 0.

**`<Options> : (Constante (ou combinaison de constantes) optionnelle)`**

Options de recherche :


|   |   |
| --- | --- |
| *MotComplet* | Recherche le mot complet (entouré par des caractères de ponctuation ou des espaces). |
| *SansCasse* | Recherche sans tenir compte de la casse (majuscules/minuscules) et des accents. |


Par défaut, la recherche est sensible à la casse : le texte recherché doit avoir la même casse que le texte à trouver. Pour effectuer une recherche ne tenant pas compte de la casse, utilisez la constante *SansCasse*.

**`<Nombre maximum> : (Entier optionnel)`**

Nombre maximum d'éléments à trouver. Permet de limiter la recherche (par exemple, permet de parcourir uniquement les 5 premiers éléments trouvés).

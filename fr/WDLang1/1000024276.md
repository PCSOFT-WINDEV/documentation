


## &lt;Table&gt;.NumEnr (Fonction)

***En anglais : &lt;Table&gt;.RecNum***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le numéro de l'enregistrement en cours dans un champ Table fichier.

**Remarque** : La fonction **&lt;Table&gt;.NumEnr** peut être utilisée indifféremment sur un champ Table monosélection ou multisélection.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Numéro d'enregistrement en cours dans le champ Table "TABLE_TableProduit"
ResNumEnr = TABLE_TableProduit.NumEnr()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Champ Table>.NumEnr()`
---

**`<Résultat> : (Entier)`**



- Numéro de l'enregistrement en cours dans le fichier de données lié au champ Table,

- 0 s'il n'y a pas de ligne en cours ou si la ligne en cours n'est pas encore enregistrée dans le fichier de données lié au champ Table.




**`<Champ Table> : (Nom de champ)`**

Nom du champ Table à manipuler.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si la requête a été exécutée avec la constante *hAvecFiltre* (option cochée dans l'éditeur de requêtes), la fonction **&lt;Table&gt;.NumEnr** renvoie le numéro de l'enregistrement en cours dans la requête.

- Si la requête a été exécutée sans la constante *hAvecFiltre*, la fonction **&lt;Table&gt;.NumEnr** renvoie un résultat indéterminé.

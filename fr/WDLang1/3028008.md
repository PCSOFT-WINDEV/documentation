
## DDEExécute (Fonction)

***En anglais : DDEExecute***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Envoie un ordre à exécuter sur le serveur.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MaConnexion = DDEConnecte("Excel", "System")
DDEExécute(MaConnexion, "[Erreur(Faux)]")
// Terminer Excel
DDEExécute(MaConnexion, "[QUITTER()]")
DDEDéconnecte(MaConnexion)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = DDEExécute(<Identifiant connexion> , <Ordre à exécuter> [, <Temporisation>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'ordre a été exécuté, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur (la fonction [DDEErreur](../WDLang1/3028006.md) permet d'obtenir le compte-rendu de l'erreur).




**`<Identifiant connexion> : (Entier)`**

Identifiant de la connexion renvoyé par la fonction [DDEConnecte](../WDLang1/3028015.md).

**`<Ordre à exécuter> : (Chaîne de caractères)`**

Ordre à exécuter sur le serveur DDE.

**`<Temporisation> : (Entier optionnel ou Durée optionnelle)`**

Durée maximale d'attente (en secondes). Si l'ordre n'a pas pu être exécuté :

- Si &lt;Temporisation&gt; est précisé, il y a une attente de &lt;Temporisation&gt; secondes avant de quitter la fonction **DDEExécute**.

- Si &lt;Temporisation&gt; n'est pas précisé, il y a une attente infinie. 


Remarque : Ce paramètre peut correspondre à : 

- un entier correspondant au nombre de secondes, 

- une variable de type Durée, 

- la durée en clair (par exemple '1s').




## TâcheParallèleAttend (Fonction)

***En anglais : ParallelTaskWait***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Attend la fin de l'exécution d'une tâche parallèle.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Déclenche l'exécution d'une tâche et attend que la tâche soit terminée
t est une TâcheParallèle = TâcheParallèleExécute(Proc)
TâcheParallèleAttend(t)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = TâcheParallèleAttend(<Tâche> [, <Durée maximale>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la tâche se termine normalement. 

- <u><u><u><u>Faux</u></u></u></u> si la tâche ne se termine pas normalement. Par exemple :

	- la durée maximale de la tâche est atteinte,

	- la tâche a été annulée. 







**`<Tâche> : (Variable de type TâcheParallèle)`**

Nom de la variable de type [TâcheParallèle](../WDLang1/1000021220.md) à manipuler. 

**`<Durée maximale> : (Entier optionnel ou Durée optionnelle)`**

Durée maximale d'exécution de la tâche, exprimée en centièmes de seconde.

- Si ce paramètre est précisé, la fonction **TâcheParallèleAttend** bloque l'application jusqu'à ce que l'exécution de la tâche soit terminée ou que le temps d'attente atteigne la durée maximale. 

- Si ce paramètre n'est pas précisé, la fonction **TâcheParallèleAttend** bloque l'application jusqu'à ce que l'exécution de la tâche soit terminée.


Ce paramètre peut correspondre à : 

- un entier correspondant au nombre de centièmes de seconde, 

- une variable de type Durée, 

- l'indication de la durée en clair (par exemple '1 s' ou '10 cs'). 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Si la fonction **TâcheParallèleAttend** est appelée depuis le thread principal, la tâche à attendre ne doit pas s'exécuter dans le thread principal. Dans le cas contraire, un blocage se produira. En effet, le thread principal ne peut pas simultanément attendre la fin de la tâche et l'exécuter. 

Une erreur WLangage signale le problème avant l'apparition du blocage.


## FAAExporteParamètre (Fonction)

***En anglais : AAFExportParameter***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Sauve toutes les personnalisations réalisées via les FAA (Fonctionnalités Automatiques de vos Applications) dans un fichier. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Clic sur " Exporter mes paramètres..."
nomFichier est chaîne 
nomFichier = fSélecteur("", "", "Sélectionnez un fichier...", ...
	"paramètres" + TAB + "*.param" + RC + ...
	"Tous les fichiers (*.*)" + TAB + "*.*", "*.*", fselCrée + fselExiste)
SI nomFichier = "" ALORS RETOUR
FAAExporteParamètre(nomFichier)
SI ErreurDétectée ALORS
	Erreur()
SINON
	ToastAffiche("Paramètres exportés dans " + RC + nomFichier)
	// Ouvre le répertoire
	LanceAppliAssociée(fExtraitChemin(nomFichier, fDisque + fRépertoire))
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`FAAExporteParamètre(<Fichier d'export>)`
---

**`<Fichier d'export> : (Chaîne de caractères)`**

Nom et chemin complet du fichier dans lequel les personnalisations des FAA seront sauvées.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Personnalisations sauvegardées
<a name="personnalisations_sauvegardees_ELTPARAGRAPHE000032"></a>Toutes les personnalisations réalisées par l'utilisateur sont sauvegardées notamment : 

- Les tailles et positions des fenêtres, 

- Les valeurs mémorisées des champs,

- L'historique des saisies, 

- Les perso-notes, 

- La personnalisation des champs Tableau de bord, 

- La personnalisation des panneaux dockables, ...




Ces paramètres peuvent ensuite être restaurées par la fonction [FAAImporteParamètre](../WDLang1/1000024377.md) ou via la Fonctionnalité Automatique de vos Application d'import des paramètres (option disponible dans le menu contextuel d'une fenêtre de l'application).

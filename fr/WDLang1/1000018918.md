
## AppliInstalleVersion (Fonction)

***En anglais : AppInstallVersion***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Installe une version spécifique de l'application.

**Remarque** : Cette fonction n'est utilisable qu'avec une installation réseau sur un réseau local. Elle ne peut être employée avec une installation par IP.




<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`AppliInstalleVersion(<Numéro de version> [, <Chemin de l'application>])`
---

**`<Numéro de version> : (Chaîne de caractères)`**

Numéro de version de l'application à installer. 

**Remarque** : Les numéros de version disponibles peuvent être connus par la fonction [AppliListeVersion](../WDLang1/1000018843.md).

**`<Chemin de l'application> : (Chaîne de caractères optionnelle)`**

Chemin complet de l'exécutable à manipuler. Si ce paramètre n'est pas précisé, l'application manipulée correspond à l'application en cours.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'installation de la version
<a name="conditions_installation_version_ELTPARAGRAPHE000035"></a>La validité de l'installation est testée. Il n'est pas possible d'installer : 

- Une version interdite. 

- Une version identique à la version en cours. 

- Une version si l'historique n'est pas activé. 




**Si l'installation n'est pas possible**, l'application continue de fonctionner. Il est possible d'obtenir plus de détails sur la cause du problème grâce à la fonction [ErreurInfo](../WDLang1/3013008.md). 

**Si l'installation est possible**, les opérations suivantes sont effectuées : 

- l'application est fermée. 

- la version demandée est installée. 

- l'application est relancée.


## ChampSansEspace (Fonction)

***En anglais : ControlNoSpace***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique si les espaces à droite de la valeur présente dans le champ sont supprimés.

Par défaut, les espaces à droite présents dans le champ ne sont pas supprimés.

**Remarque** : Cette fonction est utilisable sur les champs d'un état, d'une fenêtre ou d'une page. Cette fonction est conservée pour compatibilité avec les fichiers Hyper File au format 5.5 avec espaces.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Suppression des espaces à droite des champs
ResSuppEspace = ChampSansEspace(Vrai)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ChampSansEspace(<Suppression des espaces>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si les espaces à droite des champs étaient précédemment supprimés (avant l'appel de la fonction **ChampSansEspace**), 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Suppression des espaces> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si les espaces à droite des champs (et dans chaque colonne de table) doivent être supprimés, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction **ChampSansEspace** agit sur tous les champs du projet jusqu'au prochain appel de **ChampSansEspace**

- La fonction **ChampSansEspace** n'agit ni sur les combos (avec et sans saisie), ni sur les listes.

- La fonction **ChampSansEspace** ne s'applique pas lors de l'affectation des colonnes d'une table avec la fonction [TableAjoute](../WDLang1/3074017.md).

- La fonction **ChampSansEspace** ne s'applique pas après une saisie de l'utilisateur : si l'utilisateur tape des espaces à la fin du champ, ces espaces sont récupérés.

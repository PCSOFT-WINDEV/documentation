
## Procédure WLangage appelée par la fonction OuvreAsynchrone



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Procédure WLangage (appelée "callback") appelée par la fonction [OuvreAsynchrone](../WDLang1/1410087804.md) après la fermeture de la fenêtre. Cette procédure peut être une procédure locale, globale ou interne.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouverture asynchrone sans passage de paramètres 
OuvreAsynchrone(FEN_FenChoixDate, (), onFermeChoixDate)
PROCEDURE INTERNE onFermeChoixDate(DateChoisie)
	SAI_DATE = DateChoisie
FIN
```

```wl
// Ouverture asynchrone sans passage de paramètres et lambda
OuvreAsynchrone(FEN_FenChoixDate, (), (DateChoisie) => {SAI_DATE = DateChoisie})
```

```wl
// Ouverture asynchrone avec 2 paramètres
OuvreAsynchrone(FEN_FenChoixDate, (DateSys(), "Sélectionner une date"), onFermeChoixDate)
PROCEDURE INTERNE onFermeChoixDate(DateChoisie)
	SAI_DATE = DateChoisie
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`OuvreAsynchrone_Callback(<Valeur de retour>)`
---

**`<Valeur de retour> : (Type de la valeur de retour)`**

Correspond à la valeur qui sera renvoyée par la fenêtre. Cette valeur peut être renvoyée par exemple : 

- via l'instruction [RENVOYER](../Motscles/1510007.md) dans le code de fermeture de la fenêtre, 

- via la fonction [Ferme](../WDLang1/3038018.md) (syntaxe Ferme("", Valeur)), 

- via la propriété [ValeurRenvoyée](../Proprietes/2510134.md) de la fenêtre.

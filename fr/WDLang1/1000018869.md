
## SysDocRécentListe (Fonction)

***En anglais : SysRecentDocList***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie la liste des documents récemment ouverts.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Affiche dans la liste LISTE_DocRécent la liste des documents récemment ouverts
LISTE_DocRécent = SysDocRécentListe()
```



```wl
// Affiche dans la liste LISTE_DocRécent 
// la liste des documents et des répertoires récemment ouverts
// triée par date et avec la date affichée
Trace(SysDocRécentListe(sdrlTriDate + sdrlDate))
// Affiche par exemple
// C:\MesDoc	20180322105130
// C:\MesDoc\Poster.jpg	20180322105130
// C:\MesDoc\Poster2.jpg	20180322095020
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = SysDocRécentListe([<Options>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste des documents récemment ouverts. Par défaut, cette liste est de la forme :

```txt
<Chemin document 1> + RC + <Chemin document 2> + RC + ... + <Chemin document N>
```
Selon les options utilisées, cette liste peut également afficher : 

- la date du dernier accès au format AAAAMMJJHHmmSS.  

- le nom des répertoires accédés.




**`<Options> : (Constantes combinables de type Entier)`**



- Mode de traitement des chemins : 
	


|   |   |
| --- | --- |
| *fCheminUNICODE* | &lt;Résultat&gt; sera une chaîne de caractères au format Unicode.<br><br>ATTENTION : Si cette option n'est pas spécifiée, les chemins de documents contenant des chaînes non traduisibles en ANSI seront remplacés par le chemin court correspondant. |



- Informations affichées dans la liste : 
	


|   |   |
| --- | --- |
| *sdrlDate* | &lt;Résultat&gt; contient en plus des informations demandées, la date du dernier accès au fichier sous la forme AAAAMMJJHHmmSS (séparée du chemin de l'élément par une tabulation). |
| *sdrlDéfaut* | &lt;Résultat&gt; contient la liste des derniers documents ouverts (chemin du fichier correspondant au document ouvert). |
| *sdrlTriDate* | &lt;Résultat&gt; contient la liste des derniers répertoires accédés et des derniers documents ouverts (séparés par RC), triée du plus récent au plus ancien. <br><br>Cette constante peut être combinée avec la constante *sdrlDate* : la date du dernier accès est alors ajoutée (séparée du chemin par un RC). |

## Remarques


### Associer une application à une extension de fichier
<a name="associer_une_application_une_extension_fichier_ELTPARAGRAPHE000073"></a>Pour associer une application à une extension de fichier par la base de registre, il faut créer les clés suivantes :

- "HKEY_CLASSES_ROOT\\.ext" : 
	où **ext** est l'extension. 
	La valeur par défaut correspond au type de fichier (chaîne libre).

- "HKEY_CLASSES_ROOT\\nom_type" : 
	où **nom_type** est le nom du type donné à l'extension.
	La valeur par défaut correspond à la description qui sera affichée dans la colonne "Type" de l'explorateur Windows.

- "HKEY_CLASSES_ROOT\\nom_type\\shell\\open\\command" : 
	La valeur par défaut est la commande qui sera exécuté par la fonction [LanceAppliAssociée](../WDLang1/3035007.md) sur ce type de fichier. Dans la ligne de commande, la chaîne %1 sera remplacée par le chemin du fichier.


## FichierVersListeMémoire (Fonction)

***En anglais : FileToMemoryList***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Remplit un champ Liste par programmation (ou Combo) avec tous les enregistrements d'un fichier de données ou d'une requête. Le champ Liste ou Combo ne sont pas vidés avant d'être remplis avec le contenu du fichier ou de la requête.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajouter l'élément "<Aucun>" dans le champ Liste "LISTE_Client"
// Lorsque l'utilisateur sélectionnera cet élément, 
// la valeur "-1" sera retournée
ListeAjoute(LISTE_Client, "<Aucun>" + gValeurMémorisée("-1"))
// Ajouter à la suite de l'élément "<Aucun>",
// les noms des clients contenus dans le fichier "CLIENT"
FichierVersListeMémoire(LISTE_Client, CLIENT, NOM, NOM, IDCLIENT)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`FichierVersListeMémoire(<Champ Liste> [, <Nom du fichier> [, <Nom de la rubrique affichée> [, <Nom de la rubrique de tri> [, <Nom de la rubrique mémorisée>]]]])`
---

**`<Champ Liste> : (Chaîne de caractères)`**

Nom du champ Liste ou du champ Combo à remplir.

**`<Nom du fichier> : (Chaîne de caractères optionnelle)`**

Nom du fichier de données (ou de la requête) utilisé. Si ce paramètre n'est pas spécifié, le nom du fichier de données utilisé correspond à celui spécifié lors de la création du champ (onglet "Contenu" dans la fenêtre de description du champ).<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=FichierVersListeM%E9moire%20-%20HC%20N%B0001.gif&type=thumb)


**`<Nom de la rubrique affichée> : (Chaîne de caractères optionnelle)`**

Nom de la rubrique du fichier de données (ou de la requête) qui sera affichée dans le champ Liste ou Combo. 
Si ce paramètre n'est pas spécifié, le nom de la rubrique affichée correspond à celui spécifié lors de la création du champ (onglet "Contenu" dans la fenêtre de description du champ). 
Si ce paramètre n'est pas spécifié et si l'onglet "Contenu" n'est pas renseigné, la rubrique affichée correspond :

- soit à la dernière rubrique manipulée du fichier de données.

- soit à la première colonne de la requête.




**`<Nom de la rubrique de tri> : (Chaîne de caractères optionnelle)`**

Nom de la rubrique utilisée pour le tri des éléments du champ Liste ou Combo. Si ce paramètre n'est pas spécifié, le nom de la rubrique de tri utilisée correspond à celui spécifié lors de la création du champ (onglet "Contenu" dans la fenêtre de description du champ).

**`<Nom de la rubrique mémorisée> : (Chaîne de caractères optionnelle)`**

Nom de la rubrique qui sera utilisée en programmation lors de la sélection d'un élément dans le champ Liste ou Combo. 
Si ce paramètre est précisé, la valeur ajoutée dans le champ Liste ou Combo est : 

```txt
<Nom de la rubrique affichée> + gValeurMémorisée(<Nom de la rubrique mémorisée>)
```

Si ce paramètre n'est pas précisé, &lt;Nom de la rubrique mémorisée&gt; correspond à &lt;Nom de la rubrique affichée&gt;.





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Fonctionnement de FichierVersListeMémoire
<a name="fonctionnement_fichierverslistememoire_ELTPARAGRAPHE000067"></a>La fonction **FichierVersListeMémoire** remplit le champ Liste avec tous les enregistrements de &lt;Nom du fichier&gt; triés sur &lt;Nom de la rubrique de tri&gt;. Si un filtre est présent sur cette rubrique, ce filtre est pris en compte.


<a name="NOTE0_2"></a>




### Modification des propriétés associées au champ Liste
<a name="modification_des_proprietes_associees_champ_liste_ELTPARAGRAPHE000079"></a>Cette fonction modifie les propriétés :

- [FichierParcouru](../Proprietes/2510040.md),

- [RubriqueAffichée](../Proprietes/2510123.md),

- [RubriqueParcourue](../Proprietes/2510111.md),

- [RubriqueMémorisée](../Proprietes/2510110.md).





<a name="NOTE0_3"></a>




### Divers
<a name="divers_ELTPARAGRAPHE000101"></a>

- Le champ Liste (ou Combo) n'est pas vidé avant son remplissage.

- Cette fonction ne peut pas être utilisée sur un champ Table : il est conseillé d'utiliser la fonction [FichierVersTableMémoire](../WDLang1/3074005.md).

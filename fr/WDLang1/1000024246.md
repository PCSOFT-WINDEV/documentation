


## &lt;Table&gt;.DésactiveFiltre (Fonction)

***En anglais : &lt;Table&gt;.DisableFilter***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Annule un filtre utilisateur sur une colonne ou toutes les colonnes d'un champ Table ou Table hiérarchique. Ce filtre a été activé par l'utilisateur ou grâce à la fonction [TableActiveFiltre](../WDLang1/1000017149.md). 








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Annule le filtre sur toutes les colonnes du champ Table
TABLE_Client.DésactiveFiltre()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Champ Table ou Colonne>.DésactiveFiltre()`
---

**`<Champ Table ou Colonne> : (Nom de champ)`**

Correspond :

- soit au nom du champ Table ou Table hiérarchique. 
	Dans ce cas, tous les filtres utilisateur spécifiés sur le champ sont annulés.

- soit au nom d'une colonne d'un champ Table ou Table hiérarchique. Ce nom est de la forme : 
	```txt
	<Nom champ Table>.<Nom colonne>
	```

	Par exemple : TABLE_TableClient.COL_NomSociété. 
	Dans ce cas, seul le filtre activé sur cette colonne est annulé.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Avantage des filtres utilisateur
<a name="avantage_des_filtres_utilisateur_ELTPARAGRAPHE000215"></a>Les filtres utilisateur sont modifiables par l'utilisateur final (à la différence d'un filtre mis en place par la fonction [&lt;Source&gt;.Filtre](../WDLang4/1000024960.md) ou par la propriété [Filtre](../Proprietes/2510042.md)).
<a name="NOTE0_2"></a>


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000233"></a>Cette fonction est disponible sur : 

- un champ Table ou Table hiérarchique fichier.

- un champ Table ou Table hiérarchique par programmation.

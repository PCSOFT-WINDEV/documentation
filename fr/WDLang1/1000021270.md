
## SignalAutomatique (Type de variable)

***En anglais : AutomaticEvent***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **SignalAutomatique** permet de gérer un signal automatique. Un signal automatique permet de faciliter la synchronisation entre plusieurs threads.

Après son ouverture, un signal de type "Automatique" est automatiquement refermé dès qu'un unique thread est débloqué.

**Remarque** : Il est préférable d'utiliser une variable de type **SignalAutomatique** ou [SignalManuel](../WDLang1/1000021275.md) pour créer un signal plutôt que le nommer en utilisant une chaîne de caractères. En effet, dans le cas de l'utilisation d'une variable, il est possible de : 

- gérer la portée de la variable, 

- éviter que 2 signaux aient le même nom dans 2 threads différents.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// THREAD PRINCIPAL
//------------------
// Définition du signal
s est un SignalAutomatique
// Lancement du thread
ThreadExécute("Thread", threadNormal, ProcThread, s)

// Code en parallèle du thread
...
// On attend le signal déclenché par le thread
SignalAttend(s)
```

```wl
// THREAD SECONDAIRE
// ----------------------
PROCEDURE ProcThread(s)

// Code dans le thread
...
// Déblocage du thread principal
SignalOuvre(s)
```







## Déclaration
<a name="declaration_ELTTEXTE000179"></a>
<a name="SYNTAXE1"></a>

### Déclarer un signal automatique (syntaxe simple)

`MaVariable est un SignalAutomatique`
---

Dans ce cas, le signal automatique créé est dans l'état fermé.


<a name="SYNTAXE2"></a>

### Déclarer un signal automatique (syntaxe avancée)

`MaVariable est un SignalAutomatique(<Etat initial>)`
---

**`<Etat initial> : (Constante de type Entier)`**

Etat du signal à sa création : 



|   |   |
| --- | --- |
| *signalOuvert* | Le signal est ouvert lors de sa création. |
| *signalFermé* | Le signal est fermé lors de sa création. |















<a name="NOTE0"></a>

## Remarques
<a name="NOTE0_1"></a>


### Utilisation des variables de type SignalAutomatique
<a name="utilisation_des_variables_type_signalautomatique_ELTPARAGRAPHE000071"></a>Les variables de type **SignalAutomatique** peuvent être utilisées pour synchroniser des threads entre eux dans les fonctions : 

- syntaxe classique : 
	


|   |   |
| --- | --- |
| [SignalAttend](../WDLang1/3077007.md) | Bloque le thread en cours en attendant que le signal spécifié soit ouvert. |
| [SignalFerme](../WDLang1/1000021272.md) | Ferme un signal de synchronisation entre plusieurs threads. |
| [SignalOuvre](../WDLang1/1000021271.md) | Ouvre un signal de synchronisation entre plusieurs threads. |

- syntaxe préfixée : 
	


|   |   |
| --- | --- |
| [&lt;Variable SignalAutomatique&gt;.Attend](../WDLang1/1000024380.md) | Bloque le thread en cours en attendant que le signal spécifié soit ouvert. |
| [&lt;Variable SignalAutomatique&gt;.Ferme](../WDLang1/1000024383.md) | Ferme un signal de synchronisation entre plusieurs threads. |
| [&lt;Variable SignalAutomatique&gt;.Ouvre](../WDLang1/1000024385.md) | Ouvre un signal de synchronisation entre plusieurs threads. |




Lors d'un appel à la fonction [SignalOuvre](../WDLang1/1000021271.md) (ou [&lt;Variable SignalAutomatique&gt;.Ouvre](../WDLang1/1000024385.md)) : 

- s'il y a au moins un thread en attente, un seul thread est libéré. 

- s'il n'y a aucun thread en attente, le signal reste ouvert jusqu'à ce qu'un thread appelle la fonction [SignalAttend](../WDLang1/3077007.md) (ou [&lt;Variable SignalAutomatique&gt;.Attend](../WDLang1/1000024380.md)).

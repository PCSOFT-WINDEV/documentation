
## fSupprimeFichierRépertoireWeb (Fonction)

***En anglais : fDeleteFileWebFolder***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime un fichier image du répertoire "_WEB" de l'application (ou de un de ses sous-répertoires).

**Attention** : Pour utiliser cette fonction, l'option "Autoriser les fonctions fCopieFichierRépertoireWeb et fSupprimeFichierRépertoireWeb" doit être cochée dans l'administrateur WEBDEV (option présente dans l'onglet "Avancé" de l'administrateur WEBDEV). 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
fSupprimeFichierRépertoireWeb("images\produit.png")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = fSupprimeFichierRépertoireWeb(<Nom du fichier>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier a été supprimé. 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Nom du fichier> : (Chaîne de caractères)`**

Nom du fichier à supprimer. Ce paramètre peut correspondre à : 

- Un nom de fichier avec son chemin complet. 

- Un nom de fichier avec son chemin relatif au répertoire "_WEB" de l'application. Le répertoire "_WEB" de l'application peut être connu grâce à la fonction [fRepWeb](../WDLang1/3036052.md). 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction supprime un seul fichier à la fois. 

- Les caractères jokers (\*, ?) sont interdits dans le nom du fichier.

- Seuls les fichiers image peuvent être supprimés. Les extensions acceptées sont : jpg, jpeg, png, gif, bmp, tif, tiff et svg.


**Attention** : La suppression est prise en charge par l'administrateur de WEBDEV sur le serveur (WD280ADMIN.EXE). Le compte Windows qui l'exécute doit disposer de droits suffisants sur l'emplacement cible de la suppression.




## TâcheParallèleExécuteAprèsUne (Fonction)

***En anglais : ParallelTaskExecuteAfterOne***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique une tâche parallèle de continuation qui sera exécutée après la première tâche terminée d'un tableau de tâches parallèles.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Lance deux tâches
t1 est une TâcheParallèle = TâcheParallèleExécute(Proc1)
t2 est une TâcheParallèle = TâcheParallèleExécute(Proc2)
// Lorsqu'une des deux tâches sera terminée, la procédure ProcContinuation sera exécutée
TâcheParallèleExécuteAprèsUne([t1,t2], ProcContinuation)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Indiquer une procédure à exécuter après la première tâche précédente

`<Résultat> = TâcheParallèleExécuteAprèsUne(<Tâches précédentes> , <Procédure de continuation> [, <Paramètres> [, <Options>]])`
---

**`<Résultat> : (Variable de type TâcheParallèle)`**

Variable de type [TâcheParallèle](../WDLang1/1000021220.md) correspondant à la tâche de continuation créée.

**`<Tâches précédentes> : (Tableau de TâchesParallèles)`**

Nom du tableau de variables de type [TâcheParallèle](../WDLang1/1000021220.md) correspondant aux tâches précédentes.

**`<Procédure de continuation> : (Chaîne de caractères ou variable de type Procédure)`**

Procédure de continuation à exécuter. Ce paramètre peut correspondre :

- au nom de la procédure de continuation à exécuter.

- au nom de la variable [Procédure](../Motscles/1000018907.md) correspondant à la procédure de continuation à exécuter.




**`<Paramètres> : (Liste de valeurs entre parenthèses et séparées par des virgules)`**

Paramètres de la procédure à exécuter. Cette liste de paramètres est de la forme : 

```txt
(<Paramètre 1>, ..., <Paramètre N>)
```
où : 

- &lt;Paramètre 1&gt; : Premier paramètre de la procédure.

- ...

- &lt;Paramètre N&gt; : Nième paramètre de la procédure.


Le mot-clé ***ValeurRenvoyéeTâchePrécédente*** permet de passer la ou les valeurs de retour de la première tâche précédente terminée en paramètre de la procédure de continuation.

**`<Options> : (Constante ou combinaison de constantes optionnelle de type Entier)`**

Options de la tâche à exécuter :


|   |   |
| --- | --- |
| *tpoCopieComplèteContexteHFSQL* | Provoque la copie immédiate du contexte HFSQL courant.<br>Conseillé par exemple si la tâche parallèle doit tenir compte des positions en cours dans les fichiers et requêtes du contexte de l'appelant. |
| *tpoCopieDifféréeContexteHFSQL*<br>(Valeur par défaut) | Provoque une copie légère du contexte HFSQL **lors du premier accès aux données HFSQL**. |
| *tpoCopieLégèreContexteHFSQL* | Provoque la copie immédiate d'une partie du contexte HFSQL courant. <br>Seuls les répertoires où se trouvent les fichiers de données en HFSQL Classic et/ou les connexions en HFSQL Client/Serveur sont mémorisés. |
| *tpoThreadPrincipal* | Exécute la tâche dans le thread principal. <br>Cette constante peut être combinée avec une des autres constantes. |

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction **TâcheParallèleExécuteAprèsUne** est non bloquante.

- Si la fonction est appelée alors qu'une des tâches est déjà terminée, la continuation sera exécutée dès que possible.

- Si la fonction est appelée alors que toutes les tâches ont été annulées, la continuation ne sera pas exécutée.

- Si toutes les tâches précédentes sont annulées alors qu'une continuation a été créée, la continuation ne sera pas exécutée.

- Dans le code de la tâche de continuation, le mot-clé **MaTâcheParallèlePrécédente** permet de manipuler la première tâche précédente terminée.

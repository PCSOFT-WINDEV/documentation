


## &lt;Variable FichierDisque&gt;.NomFichier (Fonction)

***En anglais : &lt;DiskFile variable&gt;.FileName***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nom d'un fichier externe actuellement ouvert. Ce fichier a été ouvert grâce à une des fonctions suivantes : [fOuvre](../WDLang1/3036036.md), [fCrée](../WDLang1/3036017.md) ou [fOuvreFichierTemp](../WDLang1/1000017289.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
NomFichier est une chaîne 
MonFichier est un FichierDisque
MonFichier = fCrée("C:\temp\TestCreation.txt")
NomFichier = MonFichier.NomFichier()
Info("Le fichier " + fExtraitChemin(NomFichier, fFichier + fExtension) + ...
	" a été créé dans le répertoire " + fExtraitChemin(NomFichier, fDisque + fRépertoire))
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Fichier manipulé>.NomFichier()`
---

**`<Résultat> : (Chaîne de caractères)`**

Nom et chemin complet du fichier externe. 
&lt;Résultat&gt; est une chaîne de caractères du même type que celle passée aux fonctions [fOuvre](../WDLang1/3036036.md), [fCrée](../WDLang1/3036017.md) ou [fOuvreFichierTemp](../WDLang1/1000017289.md) : 

- Si ces fonctions ont manipulé une chaîne de caractères Ansi, &lt;Résultat&gt; sera au format Ansi. 

- Si ces fonctions ont manipulé une chaîne de caractères Unicode, &lt;Résultat&gt; sera au format Unicode.




**`<Fichier manipulé> : (Variable de type FichierDisque)`**

Nom de la variable de type [FichierDisque](../WDLang1/1410088883.md) initialisée avec la fonction [fOuvre](../WDLang1/3036036.md), [fCrée](../WDLang1/3036017.md) ou [fOuvreFichierTemp](../WDLang1/1000017289.md).

### Fonctionnement sous Windows Vista (et supérieur)
<a name="fonctionnement_sous_windows_vista_superieur_ELTPARAGRAPHE000286"></a>Si cette fonction ne fonctionne pas correctement sous Windows Vista (et supérieur), il est nécessaire de vérifier si le fichier ou le répertoire manipulé ne se trouve pas dans un des répertoires système (répertoire de Windows ou répertoire "Program Files").

**En effet, sous Windows Vista (et supérieur), avec le [mécanisme de l'UAC](../Editeurs/2025014.md) (contrôle des comptes utilisateurs) activé**, il est nécessaire d'avoir les privilèges administrateur pour manipuler / modifier des fichiers ou des répertoires présents dans les répertoires système (répertoire de Windows ou répertoire "Program Files").

**Conseil de programmation** : Si vous devez manipuler / modifier des fichiers ou des répertoires, sans avoir besoin de privilèges administrateur, il est conseillé :

- de ne pas écrire dans le répertoire de Windows ou dans le répertoire "Program Files",

- d'utiliser le répertoire système correspondant à l'application (connu par la fonction [SysRep](../WDLang1/3073026.md) par exemple avec la constante *srAppDataCommun*).

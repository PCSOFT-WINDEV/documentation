
## MatErreur (Fonction)

***En anglais : MatError***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie le type de la dernière erreur due à une fonction Matxxx ([fonctions de gestion des matrices](../WDLang1/3050051.md)).




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003050044&name=materreur_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Identification du type d'erreur
ResErreur = MatErreur("MaMatrice")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = MatErreur(<Matrice>)`
---

**`<Résultat> : (Entier)`**



- 0 si aucune erreur n'est survenue sur la matrice, 

- un code d'erreur (valeur supérieure à 0) dans le cas contraire.




**`<Matrice> : (Chaîne de caractères)`**

Nom de la matrice manipulée, défini avec la fonction [MatCrée](../WDLang1/3050058.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Code d'erreur
<a name="code_erreur_ELTPARAGRAPHE000082"></a>Les codes d'erreur pouvant survenir sur une matrice sont :

- 8 : Pas assez de mémoire disponible.

- 9 : La position demandée est en dehors de la matrice.

- 10 : Le nom de la matrice est invalide.

- 11 : Les matrices ont des dimensions différentes.

- 12 : Le déterminant est nul. Impossible de calculer l'inverse de la matrice.




**Attention** : Les erreurs 9, 10 et 11 génèrent une erreur du WLangage. L'application est alors arrêtée.


<a name="NOTE0_2"></a>


### Existence de la matrice
<a name="existence_matrice_ELTPARAGRAPHE000108"></a>L'existence de la matrice n'est pas vérifiée. Pour vérifier l'existence d'une matrice, utilisez la fonction [MatExiste](../WDLang1/3050076.md).

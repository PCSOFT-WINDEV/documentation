
## SignalManuel (Type de variable)

***En anglais : ManualEvent***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **SignalManuel** permet de gérer un signal manuel. Un signal manuel permet de faciliter la synchronisation entre plusieurs threads.

Un signal manuel est modifié uniquement par la fonction [SignalModifie](../WDLang1/3077017.md).

**Remarque** : Il est préférable d'utiliser une variable de type [SignalAutomatique](../WDLang1/1000021270.md) ou **SignalManuel** pour créer un signal plutôt que le nommer en utilisant une chaîne de caractères. En effet, dans le cas de l'utilisation d'une variable, il est possible de : 

- gérer la portée de la variable, 

- éviter que 2 signaux aient le même nom dans 2 threads différents.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// THREAD PRINCIPAL
//------------------
// Définition du signal
s est un SignalManuel
// Lancement du thread
ThreadExécute("Thread", threadNormal, ProcThread, s)
// Code en parallèle du thread
...
// On attend le signal déclenché par le thread
SignalAttend(s)
```

```wl
// THREAD SECONDAIRE
// ----------------------
PROCEDURE ProcThread(s)

// Code dans le thread
...
// Débloque le thread principal
SignalOuvre(s)
```





## Déclaration
<a name="declaration_ELTTEXTE000182"></a>
<a name="SYNTAXE1"></a>

### Déclarer un signal manuel (syntaxe simple)

`MaVariable est un SignalManuel`
---

Dans ce cas, le signal manuel créé est dans l'état fermé.




<a name="SYNTAXE2"></a>

### Déclarer un signal manuel (syntaxe avancée)

`MaVariable est un SignalManuel(<Etat initial>)`
---

**`<Etat initial> : (Constante de type Entier)`**

Etat du signal à sa création :


|   |   |
| --- | --- |
| *signalOuvert* | Le signal est ouvert lors de sa création. |
| *signalFermé* | Le signal est fermé lors de sa création. |















<a name="NOTE0"></a>

## Remarques
<a name="NOTE0_1"></a>


### Utilisation des variables de type SignalManuel
<a name="utilisation_des_variables_type_signalmanuel_ELTPARAGRAPHE000073"></a>Les variables de type **SignalManuel** peuvent être utilisées pour synchroniser des threads entre eux dans les fonctions : 

- syntaxe classique : 
	


|   |   |
| --- | --- |
| [SignalAttend](../WDLang1/3077007.md) | Bloque le thread en cours en attendant que le signal spécifié soit ouvert. |
| [SignalFerme](../WDLang1/1000021272.md) | Ferme un signal de synchronisation entre plusieurs threads. |
| [SignalOuvre](../WDLang1/1000021271.md) | Ouvre un signal de synchronisation entre plusieurs threads. |

- syntaxe préfixée : 
	


|   |   |
| --- | --- |
| [&lt;Variable SignalManuel&gt;.Attend](../WDLang1/1000024381.md) | Bloque le thread en cours en attendant que le signal spécifié soit ouvert. |
| [&lt;Variable SignalManuel&gt;.Ferme](../WDLang1/1000024382.md) | Ferme un signal de synchronisation entre plusieurs threads. |
| [&lt;Variable SignalManuel&gt;.Ouvre](../WDLang1/1000024384.md) | Ouvre un signal de synchronisation entre plusieurs threads. |




Lors d'un appel à la fonction [SignalOuvre](../WDLang1/1000021271.md) (ou [&lt;Variable SignalManuel&gt;.Ouvre](../WDLang1/1000024384.md)), le signal reste ouvert : 

- tous les threads en attente sont libérés, 

- tous les threads qui atteindront le signal plus tard passeront sans attente.


Lors d'un appel à la fonction [SignalFerme](../WDLang1/1000021272.md) (ou [&lt;Variable SignalManuel&gt;.Ferme](../WDLang1/1000024382.md)), le signal est fermé : tous les threads qui atteindront le signal resteront bloqués.




## &lt;Planning&gt;.AjouteRessource (Fonction)

***En anglais : &lt;Scheduler&gt;.AddResource***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute une nouvelle ressource dans un champ Planning.



![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) En mode Universal Windows 10 App, cette fonction permet uniquement de manipuler les champs Planning présents dans un état. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajoute une ressource (ici une salle de réunion) dans le champ Planning
// Reunion est le fichier de données qui décrit les différentes salles disponibles
PLN_OccupationSalles.AjouteRessource(Reunion.Nom + gValeurMémorisée(Reunion.IDReunion))
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = <Champ Planning>.AjouteRessource(<Ressource> [, <Image>])`
---

**`<Résultat> : (Entier)`**



- Indice de la ressource ajoutée, 

- -1 en cas d'erreur. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'identifier l'erreur.




**`<Champ Planning> : (Nom de champ)`**

Nom du champ Planning dans lequel la ressource doit être ajoutée. Ce champ peut correspondre : 

- au champ Planning d'une fenêtre, 

- au champ Planning d'une page,

- au champ Planning d'un état.




**`<Ressource> : (Chaîne de caractères)`**

Nom de la ressource ajoutée. Il est possible d'ajouter une valeur invisible avec la fonction [gValeurMémorisée](../WDLang1/1410088363.md).

**`<Image> : (Chaîne de caractères optionnelle)`**

Chemin et nom de l'image de fond du titre de la ressource. Cette image est affichée sous l'entête de la ligne ou de la colonne contenant le nom de la ressource.
![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Ce paramètre n'est pas disponible.


## Fonctions de gestion des champs TreeMap (syntaxe préfixée)
			

Les fonctions permettant de gérer les champs TreeMap (via une syntaxe préfixée) sont les suivantes :



|   |   |
| --- | --- |
| [&lt;TreeMap&gt;.Ajoute](../WDLang1/1000019740.md) | Ajoute un élément dans un champ TreeMap. |
| [&lt;TreeMap&gt;.Cherche](../WDLang1/1000019742.md) | Recherche l'indice d'un élément dans un champ TreeMap à partir du chemin de l'élément. |
| [&lt;TreeMap&gt;.InfoXY](../WDLang1/1000019743.md) | Renvoie l'indice de l'élément affiché à une position donnée dans un champ TreeMap. |
| [&lt;TreeMap&gt;.Occurrence](../WDLang1/1000019752.md) | Renvoie le nombre d'éléments présents dans un champ TreeMap. |
| [&lt;TreeMap&gt;.SelectGroupe](../WDLang1/1000019793.md) | Sélectionne visuellement un groupe d'éléments dans un champ TreeMap. |
| [&lt;TreeMap&gt;.Supprime](../WDLang1/1000019799.md) | Supprime un élément dans un champ TreeMap. |
| [&lt;TreeMap&gt;.SupprimeTout](../WDLang1/1000019800.md) | Supprime tous les éléments d'un champ TreeMap. |






- ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=LechampTreemap.gif) ***Exemples unitaires (WINDEV)*** : **Le champ Treemap** <br>Utilisation du champ Treemap.<br>Le champ TreeMap permet de représenter dynamiquement des données, éventuellement arborescentes, de manière pondérée.<br>Ce type de champ est très utile notamment pour permettre une visualisation intuitive de volumes de données. <br><br>Par exemple, dans l'exemple, il permet de visualiser le contenu d'un répertoire et d'analyser la taille des fichiers et sous-répertoires qu'il contient.

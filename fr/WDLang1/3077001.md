
## SectionCritiqueDébut (Fonction)

***En anglais : CriticalSectionStart***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Signale le début d'une section critique : aucun autre thread ne pourra exécuter le code tant que le thread courant ne sera pas sorti de la section critique.

Pour sortir de la section critique, utilisez la fonction [SectionCritiqueFin](../WDLang1/3077005.md).

![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Lorsque les appels aux fonctions [SectionCritiqueDébut](../WDLang1/3077001.md) et [SectionCritiqueFin](../WDLang1/3077005.md) sont situés dans le même bloc de code, il est conseillé d'utiliser le type [SectionCritique](../WDLang1/1000023972.md) et la syntaxe [UTILISER ... DANS](../Motscles/1510025.md). 
Exemple : 

```wl
// Utilisation d'une variable de type Section Critique
S est une SectionCritique

// Utilisation pour un bloc de code
//---------------------------
UTILISER S DANS
	// code protégé par la section critique
	// un seul thread pourra exécuter ce code à un moment donné
FIN
```

## Syntaxe

### Débuter une section critique sur une section de code : utilisation du type SectionCritique

`SectionCritiqueDébut(<Section critique>)`
---

**`<Section critique> : (Variable de type SectionCritique)`**

Nom de la variable de type [SectionCritique](../WDLang1/1000023972.md) permettant d'identifier la section critique. 


<a name="SYNTAXE2"></a>
![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) 
### Débuter une section critique sur une variable

`SectionCritiqueDébut([<Nom de la variable>])`
---

**`<Nom de la variable> : (Tout type)`**

Nom de la variable WLangage à protéger dans une section critique. Cette variable doit être définie avec l'attribut &lt;section critique&gt;.


<a name="SYNTAXE3"></a>

### Débuter une section critique nommée sur une section de code

`SectionCritiqueDébut([<Nom de la section>])`
---

**`<Nom de la section> : (Chaîne de caractères optionnelle)`**

Identifie la section critique. Ce paramètre permet de distinguer les sections de codes protégées. 

Si ce paramètre n'est pas précisé, une section critique automatique est mise en place. 
Remarque : la section critique automatique et la section critique correspondant à une chaîne vide ("") sont différentes.

## Remarques
<a name="NOTE0_1"></a>
<a name="NOTE0_2"></a>


### Divers
<a name="divers_ELTPARAGRAPHE000157"></a>

- Pour plus de détails sur les sections critiques, consultez [Gérer les sections critiques](../WDLang1/1000021292.md). 

- Si un thread est détruit alors qu'il est dans une section critique, la section critique est automatiquement libérée.

- Il ne peut y avoir qu'un seul thread dans toutes les sections du même nom.

- Si des sections utilisent un nom différent, un thread peut exécuter le code d'une section pendant qu'un autre thread exécute le code d'une autre section.

- Pour plus de détails sur les sections critiques sur les variables, consultez [Associer une section critique à une variable](../WDLang1/1000021292.md).





<a name="NOTE0_4"></a>




### Equivalence
<a name="equivalence_ELTPARAGRAPHE000178"></a>
```wl
SémaphoreDébut("NomSémaphore")
```


est équivalent à : 


```wl
SectionCritiqueDébut("MaSectionCritique")
```


En effet, par défaut, la fonction [SémaphoreDébut](../WDLang1/3077008.md) crée un sémaphore limité à un seul thread.

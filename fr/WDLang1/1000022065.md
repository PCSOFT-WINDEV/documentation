


## EnModeAppleWatch (Fonction)

***En anglais : InAppleWatchMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie si le code en cours d'exécution est en mode Apple Watch.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI EnModeAppleWatch() ALORS
	// Code spécifique à la plateforme Apple Watch
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = EnModeAppleWatch()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté en mode Apple Watch,

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000189"></a>La fonction **EnModeAppleWatch** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeAppleWatch** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.
<a name="NOTE0_2"></a>


### Exécution des traitements contenant des erreurs de compilation
<a name="execution_des_traitements_contenant_des_erreurs_compilation_ELTPARAGRAPHE000205"></a>Par défaut, le mécanisme de sécurité du WLangage se déclenche uniquement si la ligne contenant l'erreur de compilation est exécutée.

Par exemple, si une fonction interdite dans le produit en cours est présente dans une boucle SI, le mécanisme de sécurité du WLangage se déclenchera uniquement si la ligne contenant cette fonction s'exécute.

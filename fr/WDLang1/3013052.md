


## TraceDébut (Fonction)

***En anglais : TraceStart***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ouvre la fenêtre de trace ou enregistre les différentes informations de trace dans un fichier texte.

**Remarques** :

- Si la fenêtre de trace est déjà ouverte, la fonction **TraceDébut** n'a aucune action.

- La fonction **TraceDébut** est optionnelle. Le premier appel à la fonction [Trace](../WDLang1/3013050.md) ou à la fonction [TraceConstruit](../WDLang1/1000021683.md) ouvre automatiquement la fenêtre de trace.

<a name="SYNTAXE1"></a>

## Syntaxe

`TraceDébut([<Caractéristique> [, <Fichier de trace>]])`
---

**`<Caractéristique> : (Constante optionnelle)`**

Caractéristique de la Trace :



|   |   |
| --- | --- |
| *trCréation* | Vide le fichier de trace. |
| *trDésactive* | Tous les appels à la fonction [Trace](../WDLang1/3013050.md) seront ignorés. |
| *trDessusTout* | La fenêtre de trace sera affichée au-dessus de toutes les fenêtres du système.<br>![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Cette constante n'est pas disponible. |
| *trFenêtre* <br>(Valeur par défaut) | La fenêtre de trace s'affiche en haut à gauche de l'écran.<br><br><br> |
| *trFenêtre* + *trFichier* | La fenêtre de trace s'affichera et le contenu de la fenêtre de trace sera également enregistré dans le fichier &lt;Fichier de trace&gt;.<br> |
| *trFichier* | La fenêtre de trace ne s'affichera plus et les informations contenues dans la fenêtre de trace seront enregistrées dans le fichier &lt;Fichier de trace&gt;.<br>![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Seule cette constante est disponible. Par défaut, ce fichier est créé dans le répertoire temporaire d'UNIX, sous le nom "Trace_&lt;Projet&gt;.txt".<br> |



**`<Fichier de trace> : (Chaîne de caractères optionnelle)`**

Nom et chemin complet du fichier dans lequel les informations contenues dans la fenêtre de trace doivent être enregistrées. Si ce paramètre n'est pas spécifié, le fichier de trace sera créé dans à la racine du lecteur "C" et son nom sera de la forme "Trace_&lt;NomDuProjet&gt;.txt".

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Si ce paramètre n'est pas spécifié, le fichier de trace sera créé dans le répertoire temporaire d'UNIX, sous le nom "Trace_&lt;NomDuProjet&gt;.txt".![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Si ce paramètre n'est pas précisé, le fichier est créé dans le répertoire en cours de l'application (connu par la fonction [fRepEnCours](../WDLang1/3036022.md)) sous le nom "Trace_&lt;NomDuProjet&gt;.txt". 
![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Si ce paramètre n'est pas précisé, le fichier est créé dans le répertoire des données de l'application (connu par la fonction [fRepDonnées](../WDLang1/3036001.md)) sous le nom "Trace_&lt;NomDuProjet&gt;.txt".

## Remarques


### Conditions d'utilisation selon les plateformes
<a name="conditions_utilisation_selon_les_plateformes_ELTPARAGRAPHE000184"></a>![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) La fonction **TraceDébut** permet d'enregistrer les informations demandées uniquement dans un fichier. Par défaut, ce fichier est créé dans le répertoire temporaire d'UNIX, sous le nom "Trace_&lt;Projet&gt;.txt".
<a name="NOTE0_2"></a>


### Fermeture de la fenêtre de trace
<a name="fermeture_fenetre_trace_ELTPARAGRAPHE000226"></a>La fonction [TraceFin](../WDLang1/3013044.md) permet de fermer la fenêtre de trace.

Si la constante *trFichier* est spécifiée, la fonction [TraceFin](../WDLang1/3013044.md) arrête l'enregistrement dans le fichier de trace.

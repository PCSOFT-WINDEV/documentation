
## SaisieInvalideAfficheErreur (Fonction)

***En anglais : InvalidInputShowMessage***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Affiche une erreur de saisie invalide pour le champ spécifié. Aucune vérification de la saisie n'est effectuée par la fonction.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI PAS bVerifMotDePasse(SAI_MDP) ALORS
	SaisieInvalideAfficheErreur(SAI_MDP, ...
		"Le mot de passe doit contenir au moins un chiffre, une majuscule et un symbole.")
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`SaisieInvalideAfficheErreur(<Champ manipulé> [, <Texte>])`
---

**`<Champ manipulé> : (Nom de champ)`**

Nom du champ concerné par l'erreur. 

**`<Texte> : (Chaîne de caractères optionnelle)`**

Message d'erreur à afficher. 

Si ce paramètre n'est pas renseigné, la valeur de la propriété [TexteSaisieInvalide](../Proprietes/1000026245.md) du champ sera utilisée ou, à défaut, un message générique.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- L'état "En erreur" d'un champ est annulé dès lors que la saisie du champ est modifiée par l'utilisateur ou par programmation.

- Il est possible de personnaliser le mode d'affichage de l'erreur dans la fenêtre de description du champ.

- Pour vérifier la saisie des champs, utilisez les fonctions WLangage [SaisieInvalideDétecte](../WDLang1/1000026180.md) ou [SaisieInvalideListeChamp](../WDLang1/1000026244.md).




## &lt;Planning&gt;.Affiche (Fonction)

***En anglais : &lt;Scheduler&gt;.Display***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Réinitialise le contenu d'un champ Planning à partir de sa source de données (fichier de données HFSQL par exemple). Tous les rendez-vous sont supprimés du champ Planning et sont re-créés à partir de la source de données.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajout d'un rendez-vous dans un fichier de données HFSQL
HAjoute(Fic_RendezVous)
// Réinitialisation du champ Planning
PLN_Planning.Affiche()
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Planning>.Affiche([<Options>])`
---

**`<Champ Planning> : (Nom de champ)`**

Nom du champ Planning à manipuler. 
Le champ Planning doit être lié à une source de données. Dans le cas contraire (par exemple si le champ est rempli par programmation), la fonction **&lt;Planning&gt;.Affiche** n'a aucun effet.

**`<Options> : (Constante de type Entier)`**

Option d'affichage du champ Planning : 


|   |   |
| --- | --- |
| *plnSansExécuteRequête* | Permet de ne pas ré-exécuter automatiquement la requête liée au champ Planning (cas d'un champ Planning lié à une requête). |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si la source de données liée au champ Planning est une requête, cette requête est réinitialisée. 

- La position dans le champ Planning n'est pas modifiée.

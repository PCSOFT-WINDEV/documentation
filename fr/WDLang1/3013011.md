


## EnModeTest (Fonction)

***En anglais : InTestMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie le mode de lancement de l'application / du site :

- depuis l'éditeur de WINDEV ou de WEBDEV,

- depuis l'exécutable ou le navigateur.




![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Dans cette version, cette fonction n'a aucun effet.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI EnModeTest() = Vrai ALORS 
	Trace("Variable NumMenu : " + NumMenu)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Vérifier le mode de lancement de l'application

`<Résultat> = EnModeTest()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'application est exécutée :

	- soit depuis l'éditeur de WINDEV (Go d'une fenêtre ou du projet).

	- soit depuis l'éditeur de Etats & Requêtes (Go d'un état).

	- soit en débogage sur Mobile.

	- soit en débogage de composant via exécutable.

	- soit depuis l'éditeur de WEBDEV (Go d'une page ou du projet).




- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




Cas particuliers : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Dans un Webservice REST, ce paramètre correspond toujours à <u><u><u><u>Faux</u></u></u></u>.




- ![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Ce paramètre correspond toujours à <u><u><u><u>Faux</u></u></u></u>.





<a name="SYNTAXE2"></a>

### Vérifier l'origine du mode de lancement du test de l'application

`<Résultat> = EnModeTest([<Type d'exécution>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le test est effectué depuis l'élément demandé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**Remarque** : Le mode de lancement pris en compte est celui de l'origine du test. Par exemple, si le test de l'état est lancé depuis une fenêtre affichée par le GO de projet, l'origine du test sera le GO de projet. 

**`<Type d'exécution> : (Constante ou combinaison de constantes de type Entier)`**

Origine de l'exécution du test : 


|   |   |
| --- | --- |
| *mtEtat* | Test effectué depuis un Go d'un état. |
| *mtFenêtre* | Test effectué depuis un Go d'une fenêtre. |
| *mtPage* | Test effectué depuis un Go d'une page. |
| *mtProcédure* | Test effectué depuis un Go d'une procédure. |
| *mtProjet* | Test effectué depuis un Go d'un projet. |
| *mtRequête* | Test effectué depuis un Go d'une requête. |
| *mtTest* | Test effectué depuis un Go d'un test automatique. |

## Remarques
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) 

### Autres possibilités
<a name="autres_possibilites_ELTPARAGRAPHE000194"></a>

- La fonction [EnModeSimulateur](../WDLang1/3014033.md) permet de différencier le mode test sur le mobile et sous le simulateur.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) La fonction [EnModeEtatsEtRequêtes](../WDLang1/3013068.md) permet de différencier le mode test depuis l'éditeur d'états intégré et le mode test depuis le logiciel Etats & Requêtes.



<a name="NOTE0_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 

### GO depuis le logiciel Etats & Requêtes
<a name="depuis_logiciel_etats_requetes_ELTPARAGRAPHE000212"></a>Quand un test d'un état est lancé depuis le logiciel "Etats & Requêtes" (bouton "GO"), les fonctions **EnModeTest** et [EnModeEtatsEtRequêtes](../WDLang1/3013068.md) renvoient <u><u><u><u>Vrai</u></u></u></u>.

Il est conseillé de remplacer le test traditionnel :

```wl
SI EnModeTest() = Vrai ALORS
	Trace("...")
FIN
```
Par le test suivant :

```wl
SI EnModeTest() = Vrai ET EnModeEtatsEtRequêtes() = Faux ALORS
	Trace("...")
FIN
```

<a name="NOTE0_3"></a>


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000373"></a>La fonction **EnModeTest** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeTest** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.

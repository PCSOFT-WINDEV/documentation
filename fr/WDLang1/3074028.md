


## TableModifieLigne (Fonction)

***En anglais : TableModifyLine***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie une ligne dans : 

- un champ Table, 

- un champ Table hiérarchique. 





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Modification des éléments "Paul" et "Dubois" 
// de la cinquième ligne du champ Table "TABLE_TableClient"
TableModifieLigne(TABLE_TableClient, 5, "Paul", "Dubois")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`TableModifieLigne(<Champ Table> , <Indice> [, <Élément colonne 1> [... [, <Élément colonne N>]]])`
---

**`<Champ Table> : (Nom de champ)`**

Nom du champ à manipuler. Ce champ peut correspondre à : 

- un champ Table.

- un champ Table hiérarchique.


Si ce paramètre correspond à une chaîne vide (""), le champ manipulé est le champ auquel appartient l'événement en cours.

**`<Indice> : (Entier)`**

Indice de la ligne à modifier. La première ligne a pour indice 1.


|   |   |
| --- | --- |
| Indice égal à -1 | La ligne en cours est modifiée. Aucun effet si aucune ligne n'est sélectionnée. |
| Indice supérieur au nombre de lignes du champ | Une erreur du WLangage est générée. |



**`<Élément colonne 1> : (Chaîne de caractères optionnelle)`**

Premier élément de la ligne modifiée. La valeur de cet élément remplace la valeur existante. Chaque élément correspond à une colonne. Chaque type d'élément modifié doit être compatible avec le type de la colonne concernée.
Si aucun des paramètres &lt;Elément colonne X&gt; n'est spécifié, la fonction **TableModifieLigne** n'a aucun effet.



**`<Élément colonne N> : (Chaîne de caractères optionnelle)`**

Nème élément de la ligne modifiée. La valeur de cet élément remplace la valeur existante. Chaque élément correspond à une colonne. Chaque type d'élément modifié doit être compatible avec le type de la colonne concernée.
Si aucun des paramètres &lt;Elément colonne X&gt; n'est spécifié, la fonction **TableModifieLigne** n'a aucun effet.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000122"></a>La fonction **TableModifieLigne** peut être utilisée indifféremment sur :

- un champ Table ou Table hiérarchique fichier.

- un champ Table ou Table hiérarchique par programmation.

- un champ monosélection ou multisélection. 
	



<a name="NOTE0_2"></a>


### Caractéristique des éléments de la ligne
<a name="caracteristique_des_elements_ligne_ELTPARAGRAPHE000165"></a>

- Pour modifier uniquement la valeur d'un élément dans une ligne, utilisez la syntaxe suivante : 
	
	```txt
	ChampTable.NomColonne[IndiceLigne] = NouvelleValeur
	```


- Si un élément correspond au résultat d'un calcul, mettez l'expression numérique entre parenthèses. 
	Par exemple :
	
	```wl
	TableModifieLigne(TABLE_TableCalcul, (53+29),(83-21))
	```

### Champ Table/Table hiérarchique fichier : Modifications dans le fichier de données lié
<a name="champ_tabletable_hierarchique_fichier_modifications_dans_fichier_donnees_lie_ELTPARAGRAPHE000188"></a>

- **Si la saisie en cascade est activée**, le fichier de données lié au champ est automatiquement modifié lorsque l'utilisateur passe à la ligne suivante.

- **Si la saisie en cascade n'est pas activée**, le fichier de données lié au champ n'est pas modifié. Pour écrire le contenu de la ligne modifiée dans le fichier de données lié,  il est nécessaire d'utiliser la fonction [TableEnregistre](../WDLang1/3074008.md) après la fonction **TableModifieLigne**.




**Remarques** : 

- L'option "Saisie en cascade" peut être sélectionnée dans l'onglet "Détail" de la description du champ.

- ![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) La saisie en cascade n'est pas disponible sur les champs Table fichier à accès direct. Cette option est disponible uniquement sur les champs Table par programmation et Table fichier chargé en mémoire.



<a name="NOTE0_4"></a>


### Modifications réalisées dans un champ Table/Table hiérarchique fichier
<a name="modifications_realisees_dans_champ_tabletable_hierarchique_fichier_ELTPARAGRAPHE000216"></a>Pour modifier une ligne d'un champ Table/Table hiérarchique fichier, il est conseillé de :

- modifier directement l'enregistrement dans le fichier de données avec la fonction [HModifie](../WDLang4/3044042.md),

- utiliser la fonction [TableAffiche](../WDLang1/3074003.md) pour ré-actualiser le champ.

### Fonction TableModifieLigne et propriété Modifié
<a name="fonction_docparampagetitleshort_propriete_modifie_ELTPARAGRAPHE000236"></a>La fonction **TableModifieLigne** n'a aucune influence sur la propriété [Modifié](../Proprietes/2510080.md). La propriété [Modifié](../Proprietes/2510080.md) permet de savoir si la ligne en cours a été modifiée directement par l'utilisateur (saisie directe dans le champ).

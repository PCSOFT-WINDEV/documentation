
## ErreurDétectée (Variable)

***En anglais : ErrorOccurred***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
**ErreurDétectée** est une variable prédéfinie du  WLangage indiquant le résultat de la dernière fonction WLangage utilisée.

**ErreurDétectée** correspond à :

- <u><u><u>Faux</u></u></u> si la dernière fonction WLangage utilisée a réussi.

- <u><u><u>Vrai</u></u></u> si la dernière fonction WLangage utilisée a provoqué une erreur non fatale.




Remarque : Des informations concernant l'erreur rencontrée peuvent être récupérées grâce aux fonctions [ErreurInfo](../WDLang1/3013008.md) ou [HErreurInfo](../WDLang4/3044071.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
nFichier est un entier
nFichier = fOuvre("C:\MesRépertoires\Fichier.txt", foLectureEcriture)
SI nFichier <> -1 ET ErreurDétectée = Faux ALORS
	...
SINON
	Erreur(ErreurInfo())
FIN
```



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Pour gérer les erreurs fatales, utilisez [QUAND EXCEPTION](../WDLang1/3034001.md).

- Les erreurs fatales non gérées provoquent l'arrêt immédiat de l'application. Un message d'erreur est automatiquement affiché.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Ce message peut être paramétré lors de la création de l'exécutable.

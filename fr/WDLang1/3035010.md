
## ExePriorité (Fonction)

***En anglais : ExePriority***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie ou modifie la priorité d'une application en cours d'exécution.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Sélection de l'application à manipuler
nIndice est un entier = TableSelect(TABLE_Table1)
SI nIndice <> -1 ALORS
	// Priorité de l'application sélectionnée ?
	i est un entier = ExePriorité(TABLE_Table1.PID[nIndice])
	SELON i
		CAS exePrioritéBasse
			Info("Priorité basse")
		CAS exePrioritéHaute
			Info("Priorité haute")
		CAS exePrioritéInférieureNormale
			Info("Priorité inférieure à la normale")
		CAS exePrioritéNormale
			Info("Priorité normale")
		CAS exePrioritéSupérieureNormale
			Info("Priorité supérieure à la normale")
		CAS exePrioritéTempsRéel
			Info("Priorité temps réel")
	FIN 
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Modifier la priorité d'une application en cours d'exécution

`ExePriorité(<Identifiant de l'application> , <Nouvelle priorité>)`
---

**`<Identifiant de l'application> : (Constante de type Entier)`**

Identifiant de l'application à manipuler. Cet identifiant peut être connu grâce à la fonction [ExeListeProcessus](../WDLang1/3035008.md).

**`<Nouvelle priorité> : (Constante de type Entier)`**

Nouvelle priorité de l'application spécifiée :


|   |   |
| --- | --- |
| *exePrioritéBasse* | Priorité basse. |
| *exePrioritéHaute* | Priorité haute. |
| *exePrioritéInférieureNormale* | Priorité supérieure à *exePrioritéBasse* et inférieure à *exePrioritéNormale*<br>Cette constante est disponible uniquement sous Windows 2000 et XP. |
| *exePrioritéNormale* | Priorité supérieure à *exePrioritéInférieureNormale* et inférieure à *exePrioritéSupérieureNormale*. |
| *exePrioritéSupérieureNormale* | Priorité supérieure à *exePrioritéNormale* et inférieure à *exePrioritéTempsRéel*.<br>Cette constante est disponible uniquement sous Windows 2000 et XP. |




<a name="SYNTAXE2"></a>

### Connaître la priorité d'une application en cours d'exécution

`<Résultat> = ExePriorité(<Identifiant de l'application>)`
---

**`<Résultat> : (Constante de type Entier)`**

Priorité de l'application spécifiée :


|   |   |
| --- | --- |
| *exePrioritéBasse* | Priorité basse |
| *exePrioritéHaute* | Priorité haute |
| *exePrioritéInférieureNormale* | Priorité supérieure à *exePrioritéBasse* et inférieure à *exePrioritéNormale*.<br>Cette constante est disponible uniquement sous Windows 2000 et XP. |
| *exePrioritéNormale* | Priorité supérieure à *exePrioritéInférieureNormale* et inférieure à *exePrioritéSupérieureNormale*. |
| *exePrioritéSupérieureNormale* | Priorité supérieure à *exePrioritéNormale* et inférieure à *exePrioritéTempsRéel*.<br>Cette constante est disponible uniquement sous Windows 2000 et XP. |
| *exePrioritéTempsRéel* | Priorité temps réel. |



**`<Identifiant de l'application> : (Constante de type Entier)`**

Identifiant de l'application à manipuler. Cet identifiant peut être connu grâce à la fonction [ExeListeProcessus](../WDLang1/3035008.md).

 

## ChargeProcédure (Fonction)

***En anglais : LoadProcedure***
	

<a name="XUtilisation"></a>
<a name="Utilisation"></a>
|  |   |
| --- | --- |



|  
 | ButUtiliser les procédures locales d'une fenêtre comme des procédures globales au projet.Syntaxe&lt;Résultat&gt; = ChargeProcédure(&lt;NomFenetre&gt;, &lt;Message&gt;)<br><br>- &lt;Résultat&gt; est un booléen permettant de savoir si les procédures ont été chargées<br><br>- &lt;NomFenetre&gt; est une chaîne de caractères contenant le nom de la fenêtre ou de l'alias de la fenêtre dans laquelle les procédures internes ont été décrites.<br><br>- &lt;Message&gt; est un booléen permettant d'indiquer si un message d'erreur doit apparaître (Vrai par défaut) ou non (Faux) lors de l'exécution, en cas de non chargement des procédures.<br><br><br>DétailLes procédures internes de la fenêtre &lt;NomFenetre&gt; deviennent des procédures globales au projet.<br><br>&lt;Résultat&gt; est à Vrai si le chargement a été correctement effectué. Si &lt;Message&gt;=Vrai , un message signale si la fenêtre n'a pas été trouvée à l'exécution.<br><br>Si aucun projet n'est en cours, un message d'erreur le signale.Notes<br><br>- **ChargeProcédure	** permet de charger en même temps les procédures de 16 fenêtres maximum.<br><br>- Pour information : lors de l'exécution d'une procédure en WLangage, celle-ci est recherchée :<br><br>	- en premier dans les procédures internes de la fenêtre,<br><br>	- ensuite dans les différents "modules" chargés,<br><br>	- enfin dans les procédures globales.<br><br><br><br><br>- Les procédures chargées par **ChargeProcédure	** sont automatiquement déchargées lors de la fermeture du projet. Elles peuvent être déchargées par [DéchargeProcedure](../WDLang1/3013018.md).<br><br>- **ChargeProcédure	** ne charge pas :<br><br>	- les variables globales et le code d'ouverture de la fenêtre.<br><br>	- le code des champs de la fenêtre.<br><br><br><br><br>- Le code d'initialisation de la fenêtre n'est pas exécuté. Le chargement de procédure ne permet donc pas d'ajouter des variables globales.<br><br>- Une procédure d'une fenêtre appelée par **ChargeProcédure	** ne peut pas être appelée par Timer.<br><br>- Attention : si une procédure de la fenêtre appelée par **ChargeProcédure	** a le même nom qu'une procédure interne de la fenêtre en cours ou que d'une procédure globale, cette procédure ne sera pas chargée. Il est conseillé de ne pas charger des procédures ayant le même nom que des procédures existantes dans la fenêtre ou dans le projet. Vous pouvez renommer la procédure que vous voulez charger.<br><br><br><br><br> |

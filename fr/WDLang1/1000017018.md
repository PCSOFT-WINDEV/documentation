
## SysListeRésolutionEcran (Fonction)

***En anglais : SysListScreenResolution***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie les résolutions possibles pour un écran.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajoute dans une table contenant 3 colonnes 
// la liste des résolutions disponibles pour l'écran principal
ListeAjoute(TABLE_Résolution, SysListeRésolutionEcran())
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = SysListeRésolutionEcran([<Nom de l'écran>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Chaîne de caractères contenant la liste des résolutions disponibles, chaîne vide ("") en cas d'erreur. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de renseignements sur le problème rencontré.

Cette chaîne est de la forme :

```txt
<Résolution Horizontale 1> + TAB + <Résolution Verticale 1> + TAB +
<Nombre de bits par pixels 1> + TAB + <Fréquence de rafraîchissement> + RC + ... + 
<Résolution Horizontale N> + TAB + <Résolution Verticale N> + TAB + 
<Nombre de bits par pixels N> + TAB + <Fréquence de rafraîchissement N>
```


**`<Nom de l'écran> : (Chaîne de caractères optionnelle)`**

Nom de l'écran pour lequel les résolutions disponibles sont recherchées. Si ce paramètre n'est pas spécifié, les résolutions disponibles sont recherchées pour l'écran principal. Si le nom de l'écran n'existe pas, &lt;Résultat&gt; correspond à chaîne vide ("").

**Remarque** : La liste des noms des écrans peut être connue grâce à la fonction [SysListeEcran](../WDLang1/1000017014.md).

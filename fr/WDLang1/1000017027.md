


## EnModeMCU (Fonction)

***En anglais : InUMCMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie si le code est exécuté depuis un code écrit dans une macro utilisateur. Permet par exemple de savoir dans le code d'une procédure si celle-ci a été lancée par une Macro Code Utilisateur ou non.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code exécuté depuis un code MCU ?
PROCEDURE Vérification()
...
SI EnModeMCU() = Vrai ALORS
	// Procédure exécutée depuis un code MCU
	...
SINON
	// Cas normal
	// Traitement à effectuer
	...
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = EnModeMCU()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté depuis une Macro Code Utilisateur, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000223"></a>La fonction **EnModeMCU** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeMCU** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.

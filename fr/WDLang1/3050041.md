
## FinNbVersement (Fonction)

***En anglais : FinPaymentNb***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nombre de versements nécessaires pour rembourser un capital selon un taux.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003050041&name=finnbversement_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Nombre de versements nécessaires : le montant des versements
// est de 7 500 Euros,
// le taux d'intérêt est de 5% et le montant emprunté est 
// de 42 000 Euros.
ResNbVersement = FinNbVersement(-7500, 5, 42000)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = FinNbVersement([<Valeur d'un versement> [, <Taux> [, <Capital>]]])`
---

**`<Résultat> : (Réel)`**

Nombre de versements à réaliser. Ce nombre peut être :

- négatif : un paiement (ou décaissement) doit alors être effectué;

- positif : un encaissement doit alors être effectué.




**`<Valeur d'un versement> : (Réel ou monétaire optionnel)`**

Valeur d'un versement (0 par défaut). Pour un remboursement, cette valeur doit être négative.

**`<Taux> : (Réel optionnel)`**

Taux d'emprunt exprimé en pourcentage (1 par défaut). Par exemple, pour un taux d'emprunt de 5%, ce paramètre sera égal à 5.

**`<Capital> : (Réel ou monétaire optionnel)`**

Montant de l'emprunt (1 par défaut).

<a name="NOTE0_1"></a>

## Remarques
**Attention** : La fonction **FinNbVersement** ne retourne pas de code d'erreur. Pour savoir si cette fonction a généré des erreurs, utilisez la fonction [FinErreur](../WDLang1/3050071.md). Le libellé de l'erreur peut être connu grâce à la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.

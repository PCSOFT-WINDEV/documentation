


## &lt;Toast&gt;.Affiche (Fonction)

***En anglais : &lt;Toast&gt;.Display***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Affiche un message de type "Toast".

Le Toast est un message furtif qui apparaît durant quelques secondes à l'écran avant de disparaître avec un effet de fondu sans intervention de l'utilisateur. Un message Toast ne prend pas le focus, il n'est pas modal et il n'interrompt pas l'exécution de l'application.

L'affichage des Toasts est recommandé pour transmettre des informations à l'utilisateur sans bloquer l'exécution de l'application (exemple : affichage d'un message pour indiquer la fin d'un téléchargement).<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ToastAffiche.gif)
![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Cette fonction est disponible uniquement pour les sites WEBDEV Linux.

<a name="Exemple1"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) 
```wl
MonToast est un Toast
MonToast.Titre = "Le saviez-vous ?"
MonToast.Texte = "Les FAA sont un truc génial."
MonToast.Bouton[1].Libellé = "En savoir plus..."
MonToast.Bouton[1].ActionClic = FoncAide
MonToast.ActionFermeture = clicFerme
MonToast.Affiche()
PROCÉDURE INTERNE FoncAide
	Ouvre("FEN_Aide")
FIN
PROCÉDURE INTERNE clicFerme(nBouton)
	Trace("Clic fermeture, bouton " + nBouton)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE2"></a>

### Afficher un toast basé sur une variable de type Toast

`<Toast>.ToastAffiche()`
---

**`<Toast> : (Variable de type Toast)`**

Nom de la variable de type [Toast](../WDLang1/1000024638.md) décrivant le toast à afficher.

## Remarques


### Gestion des threads
<a name="gestion_des_threads_ELTPARAGRAPHE000214"></a>La fonction **&lt;Toast&gt;.Affiche** peut être appelée depuis un thread WLangage (par exemple pour notifier l'utilisateur d'événements survenus durant l'exécution du thread).


### Appels successifs
<a name="appels_successifs_ELTPARAGRAPHE000227"></a>Si la fonction **&lt;Toast&gt;.Affiche** est appelée plusieurs fois, chaque toast est affiché lorsque le toast précédent a disparu. Ce comportement peut être modifié en utilisant la propriété **Superposable** de la variable de type [Toast](../WDLang1/1000024638.md). Cette propriété permet de retrouver le comportement de la version 24 :  les toasts se superposent l'un au-dessus de l'autre.

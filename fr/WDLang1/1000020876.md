


## ChampCrée (Fonction)

***En anglais : ControlCreate***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Crée un nouveau champ de n'importe quel type dans une fenêtre par programmation. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
c est un Champ
c <- ChampCréé("BOUTON_1", typBouton, 30, 40, 140, 20)
c.Libellé = "Cliquez moi"
c.Traitement[trtClic] = maProcédure
```
<a name="Exemple2"></a>

```wl
ChampCréé("SAISIE_1", typDate, 30, 40, 140, 20)
chmp est un Champ
chmp <- ChampCrée("DATE", typDate, 150, 150, 80, 20)
chmp.LargeurLibellé = 40
chmp.Libellé = "Date"
```
<a name="Exemple3"></a>

```wl
// Création d'un champ Table avec ses colonnes
cTable est un Champ
cTable <- ChampCrée("cTable", typTable, 0, 0, 300, 250, Vrai)
cTable.Ancrage = ancreLargeur + ancreHauteur
col1 est un Champ <- ChampCrée("cTable.COL1", typColonne)
col1.TypeSaisie = typSaisieTexte
col2 est un Champ <- ChampCrée("cTable.COL2", typColonne)
col2.TypeSaisie = typSaisieHeure
col3 est un Champ <- ChampCrée("cTable.COL3", typColonne)
col3.TypeSaisie = typSaisieNum
TableAjouteLigne(cTable, "Col 1", "1224", "25")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = ChampCrée(<Nom> , <Type> [, <X> [, <Y> [, <Largeur> [, <Hauteur> [, <Visible>]]]]])`
---

**`<Résultat> : (Variable de type Champ)`**

Le champ créé peut être manipulé via une variable de type [Champ](../WDLang1/1000020967.md). Dans ce cas, l'affectation doit être réalisée avec l'opérateur **'<-'**. 

Si la fonction **ChampCrée** est utilisée directement, le champ est créé dans la fenêtre et est visible par défaut. 

**`<Nom> : (Chaîne de caractères)`**

Nom à donner au nouveau champ. Ce nom permettra de manipuler le champ par programmation. 
Si ce nom existe déjà, une erreur WLangage est affichée. 
Si ce paramètre correspond à une chaîne vide (""), un nom unique est automatiquement créé. 

Cas particuliers : 

- Création d'un champ dans un champ "Parent" : il suffit d'indiquer le nom complet du champ à créer. Par exemple : "Onglet[1].Bouton2".

- Création d'une colonne de saisie dans un champ Table : il suffit d'indiquer le type *typColonne* et le nom du champ Table avant le nom de la colonne. Par exemple : 
	
	```wl
	// Colonne de type Saisie + Date 
	c <- ChampCrée("TABLE1.COL3", typColonne)
	c.Largeur = 100
	c.TypeSaisie = typSaisieDate
	```


- Création d'une colonne simple dans un champ Table : il suffit d'indiquer le type de la colonne voulu et le nom du champ Table avant le nom de la colonne. Par exemple :
	
	```wl
	// Colonne de type Image 
	c2 est un Champ <- ChampCrée("Table1.Col3", typImage)
	c2.Largeur = 50
	```


- Création de volets d'onglet  : il suffit d'indiquer le type *typVoletOnglet* et le nom de l'onglet. Par exemple :
	
	```wl
	ChampCrée("ONGLET1", typVoletOnglet)
	```


- Création d'un champ de saisie de type Jeton de texte : Il suffit de créer un champ de saisie de type texte puis d'activer la gestion des jetons. Par exemple : 
	
	```wl
	MonChamp est un Champ <- ChampCrée("test", typTexte, 10, 10, 250, 25)
	MonChamp.JetonActif = Vrai
	```

**`<X> : (Entier optionnel)`**

Abscisse du champ à créer (position sur l'axe des X) en pixels. Position horizontale du coin haut gauche du champ par rapport au coin haut gauche de la zone client de la fenêtre (c'est-à-dire la fenêtre sans les barres de titre et de menu ni le cadre). Ce paramètre correspond à la propriété [XInitial](../Proprietes/2510140.md). 

Si ce paramètre n'est pas précisé, l'abscisse du champ est 0. 

Il est ensuite possible de modifier cette valeur avec la propriété [X](../Proprietes/2510135.md).

**`<Y> : (Entier optionnel)`**

Ordonnée du champ à créer (position sur l'axe des Y) en pixels. Position verticale du coin haut gauche du champ par rapport au coin haut gauche de la zone client de la fenêtre (c'est-à-dire la fenêtre sans les barres de titre et de menu ni le cadre). Ce paramètre correspond à la propriété [YInitial](../Proprietes/2510087.md). 

Si ce paramètre n'est pas précisé, l'ordonnée du champ est 0. 

Il est ensuite possible de modifier cette valeur avec la propriété [Y](../Proprietes/2510137.md).

**`<Largeur> : (Entier optionnel)`**

Largeur du champ à créer (exprimée en pixels). Ce paramètre correspond à la propriété [LargeurInitiale](../Proprietes/2510067.md). 

Si ce paramètre n'est pas précisé, la largeur du champ est 0. 

Il est ensuite possible de modifier cette valeur avec la propriété [Hauteur](../Proprietes/2510050.md).

**`<Hauteur> : (Entier optionnel)`**

Hauteur du champ à créer (exprimée en pixels). Ce paramètre correspond à la propriété [HauteurInitiale](../Proprietes/2510043.md). 

Si ce paramètre n'est pas précisé, la hauteur du champ est 0. 

Il est ensuite possible de modifier cette valeur avec la propriété [Largeur](../Proprietes/2510059.md).

**`<Visible> : (Booléen optionnel)`**



- <u><u><u><u>Vrai</u></u></u></u> (valeur par défaut) si le champ doit être visible, 

- <u><u><u><u>Faux</u></u></u></u> pour créer un champ invisible. 




Il est ensuite possible de modifier cette valeur avec la propriété [Visible](../Proprietes/2510138.md).

## Remarques


- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Si plusieurs champs sont créés dans la même fenêtre en une seule opération, il est conseillé de désactiver la propriété [AffichageActif](../Proprietes/2510015.md) avant la création des champs puis de la réactiver après : l'affichage des nouveaux champs sera optimisé et l'effet d'apparition progressive des champs sera supprimé. 

- Pour paramétrer les caractéristiques du champ créé, utilisez les propriétés associées au type de champ créé : 

	- Si le champ est créé directement, il est possible d'utiliser l'opérateur d'indirection pour manipuler le champ par son nom : 
			
		```wl
		ChampCrée("BOUTON_1", typBouton, 30, 40, 140, 20) 
		
		{"BOUTON_1"}..Libellé = "Envoyer un email"
		{"BOUTON_1"}..Traitement[trtClic] = maProcédure
		```


	- Si le champ créé est associé à une variable de type [Champ](../WDLang1/1000020967.md), il suffit d'utiliser directement les propriétés du champ sur la variable : 
			
		```wl
		c est un Champ
		c <- ChampCréé("BOUTON_1", typBouton, 30, 40, 140, 20) 
		c.Libellé = "Cliquez moi"
		c.Traitement[trtClic] = maProcédure
		```





- Pour changer le style du champ créé : 

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) utilisez la fonction [ChangeStyle](../WDLang1/1000017171.md). Il suffit d'affecter au champ un style présent dans la feuille de styles du projet. 

	- utilisez la propriété [Style](../Proprietes/1000021472.md). Cette propriété permet de copier le style d'un champ existant dans un autre champ. 




- Pour définir les différents traitements/événements associés au champ, utilisez la propriété [Traitement](../Proprietes/1000020878.md). 

- Pour supprimer un champ créé avec la fonction **ChampCrée**, utilisez la fonction [ChampSupprime](../WDLang1/3025037.md).

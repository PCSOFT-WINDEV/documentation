
## JSONValide (Fonction)

***En anglais : JSONValid***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Teste la validité d'une chaîne au format JSON. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Chaîne JSON valide ? 
SI JSONValide(sJSON) ALORS
	// Oui => initialisation variable de type JSON
	oJSON est un JSON = ChaîneVersJSON(sJSON)
	...
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = JSONValide(<Chaîne JSON>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la chaîne correspond à un format JSON valide, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. En cas d'erreur, il est possible de connaître le libellé de l'erreur avec la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Chaîne JSON> : (Chaîne de caractères)`**

Chaîne de caractères (ANSI encodé en ASCII 7 bits + encodage JSON ou UTF-8) contenant les données au format JSON.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Encodage
<a name="encodage_ELTPARAGRAPHE000044"></a>L'encodage utilisé correspond au standard JSON : 

- utilisation d'un encodage en ASCII 7 bits pour les 128 premiers caractères, les caractères non accentués, 

- utilisation d'un encodage JSON des autres caractères : "\\u" suivi du code du caractère en hexadécimal sur 4. Exemple : pour "é" (code ASCII 233, code hexadécimal E9), l'encodage correspond à "\\u00E9".




## &lt;Variable Image&gt;.DessinePortion (Fonction)

***En anglais : &lt;Image variable&gt;.DrawSlice***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Dessine une portion de cercle ou d'ellipse dans une variable de type [Image](../WDLang1/1000019650.md).

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les dessins](../WDLang1/3029035.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MonImage est une Image = "Test.bmp"
// Dessin d'une portion de cercle dont le
// fond est jaune clair et le bord bleu clair
MonImage.DessinePortion(10, 10, 50, 50, 20, 20, 40, 50, JauneClair, BleuClair)
IMG_MonDessin = MonImage
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Variable Image>.DessinePortion(<X1> , <Y1> , <X2> , <Y2> , <X3> , <Y3> , <X4> , <Y4> [, <Couleur du fond> [, <Couleur du trait>]])`
---

**`<Variable Image> : (Variable de type Image)`**

Nom de la variable de type [Image](../WDLang1/1000019650.md) à manipuler. 

**`<X1> : (Entier)`**

Abscisse du coin haut gauche du rectangle dans lequel le cercle est inscrit. Ces coordonnées sont exprimées en pixels.

**`<Y1> : (Entier)`**

Ordonnée du coin haut gauche du rectangle dans lequel le cercle est inscrit. Ces coordonnées sont exprimées en pixels.

**`<X2> : (Entier)`**

Abscisse du coin bas droit du rectangle dans lequel le cercle est inscrit. Ces coordonnées sont exprimées en pixels.

**`<Y2> : (Entier)`**

Ordonnée du coin bas droit du rectangle dans lequel le cercle est inscrit. Ces coordonnées sont exprimées en pixels.

**`<X3> : (Entier)`**

Abscisse du point de départ de la portion de cercle. Ces coordonnées sont exprimées en pixels.

**`<Y3> : (Entier)`**

Ordonnée du point de départ de la portion de cercle. Ces coordonnées sont exprimées en pixels.

**`<X4> : (Entier)`**

Abscisse du point d'arrivée de la portion de cercle. Ces coordonnées sont exprimées en pixels.

**`<Y4> : (Entier)`**

Ordonnée du point d'arrivée de la portion de cercle. Ces coordonnées sont exprimées en pixels.

**`<Couleur du fond> : (Entier ou constante (optionnel))`**

Couleur du fond de la portion de cercle. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md),

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) une variable de type [Couleur](../WDLang1/1000019661.md). 




Si ce paramètre n'est pas précisé, la couleur du fond :

- est *Transparent* si la fonction [&lt;Variable Image&gt;.Fond](../WDLang1/1000024056.md) n'a pas été utilisée auparavant,

- correspond à la couleur précisée dans le dernier appel à la fonction [&lt;Variable Image&gt;.Fond](../WDLang1/1000024056.md).




**`<Couleur du trait> : (Entier ou constante (optionnel))`**

Couleur du trait de la portion de cercle. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md),

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) une variable de type [Couleur](../WDLang1/1000019661.md). 




Si ce paramètre n'est pas précisé, la couleur du trait :

- correspond à la couleur précisée dans le dernier appel à la fonction [&lt;Variable Image&gt;.Stylo](../WDLang1/1000024240.md),

- est identique à la couleur utilisée pour le fond si la fonction [&lt;Variable Image&gt;.Stylo](../WDLang1/1000024240.md) n'a pas été utilisée.

<a name="NOTE0_1"></a>

## Remarques


### Coordonnées
<a name="coordonnees_ELTPARAGRAPHE000489"></a>Si *les points de départ et d'arrivée* ne sont pas situés sur le bord du cercle (ou de l'ellipse), le point pris en compte est l'intersection du cercle et de la droite joignant le point précisé et le centre du rectangle (c'est-à-dire le centre du cercle).<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=RbPortion.gif)
Les coordonnées sont indiquées par rapport au coin haut gauche de l'image (coordonnées (0,0)).

Les portions de dessin sont dessinées dans le sens trigonométrique.
<a name="NOTE0_2"></a>


### Figure non-pleine
<a name="figure_nonpleine_ELTPARAGRAPHE000519"></a>Pour dessiner une figure non-pleine (sans prendre en compte le centre du cercle), utilisez la fonction [&lt;Variable Image&gt;.DessineCorde](../WDLang1/1000024040.md).
<a name="NOTE0_3"></a>

### Dessin avec opacité ou anti-aliasing
<a name="dessin_avec_opacite_antialiasing_ELTPARAGRAPHE000536"></a>Dans le cas d'un dessin avec opacité (fonction [&lt;Variable Image&gt;.DébutDessin](../WDLang1/1000024349.md) avec la constante *dAvecOpacité*) ou anti-aliasing (fonction [&lt;Variable Image&gt;.ChangeMode](../WDLang1/1000024039.md) avec la constante *dessinAntiAliasing*), le contour est dessiné par-dessus l'intérieur de la portion: le trait est donc en partie combiné avec l'intérieur de la portion. Si l'épaisseur du trait est importante, le trait s'affichera en deux couleurs.

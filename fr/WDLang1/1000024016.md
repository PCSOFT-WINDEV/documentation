


## &lt;Image&gt;.Stylo (Fonction)

***En anglais : &lt;Image&gt;.Pen***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Déclare la couleur et le style par défaut du tracé des traits, des rectangles, des cercles, ... contenus dans un champ Image.



![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les dessins](../WDLang1/3029035.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Déclaration de la couleur et du style par défaut des différents tracés
IMG_Dessin.Stylo(VertClair, 3)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Champ Image>.Stylo(<Couleur> [, <Style du trait> [, <Largeur du trait> [, <Opacité>]]])`
---

**`<Champ Image> : (Nom de champ)`**

Nom du champ Image à manipuler.

**`<Couleur> : (Entier ou constante)`**

Couleur par défaut des différents tracés. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md).

- une variable de type [Couleur](../WDLang1/1000019661.md). Dans ce cas, l'opacité spécifiée dans la variable est prise en compte.
	




**`<Style du trait> : (Entier optionnel)`**

Style par défaut des différents tracés :

- 0 (par défaut) : Trait plein

- 1 : Tiret (---)

- 2 : Trait pointillé (...)

- 3 : Pointillé tiret (.-.-.)

- 4 : Tiret deux pointillés (-..-..-)

- 5 : Trait invisible




**`<Largeur du trait> : (Entier optionnel)`**

Largeur du trait en pixels (1 par défaut).

**`<Opacité> : (Entier optionnel)`**

Opacité de la couleur, comprise entre 0 (transparent) et 255 (opacité complète). Une opacité de 128 permet de mélanger 50% de la couleur avec la couleur existante dans le champ.

**Remarques** : 

- Cette opacité sera prise en compte si le dessin dans le champ Image a été initialisé avec la constante *dAvecOpacité* (fonction [&lt;Image&gt;.DébutDessin](../WDLang1/1000024326.md)).

- Si le paramètre &lt;Couleur&gt; correspond à une variable de type [Couleur](../WDLang1/1000019661.md), ce paramètre ne doit pas être spécifié : l'opacité spécifiée dans la variable sera prise en compte.

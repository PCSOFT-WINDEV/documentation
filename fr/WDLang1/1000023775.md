


## &lt;Arbre&gt;.TypeElément (Fonction)

***En anglais : &lt;TreeView&gt;.TypeItem***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Teste l'existence d'un élément d'un champ Arbre et renvoie son type (noeud ou feuille) si l'élément existe.

Exemple de champ Arbre :<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Arbre_Vocabulaire.gif)

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Type de l'élément "Desserts" dans le champ Arbre "ARBRE_RecetteTV"
Res = ARBRE_RecetteTV.TypeElément("Recettes" + TAB + "Desserts")
SELON Res 
	CAS aErreur : Info("Elément inexistant")
	CAS aFeuille : Info("L'élément est une feuille")
	CAS aNoeud : Info("L'élément est un noeud")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Champ Arbre>.TypeElément(<Chemin de l'élément>)`
---

**`<Résultat> : (Constante)`**

Constante permettant d'identifier le type de l'élément : 


|   |   |
| --- | --- |
| *aErreur* | L'élément spécifié n'existe pas. |
| *aFeuille* | L'élément spécifié est une feuille. |
| *aNoeud* | L'élément spécifié est un noeud. |



**`<Champ Arbre> : (Nom de champ)`**

Nom du champ Arbre à manipuler.

**`<Chemin de l'élément> : (Chaîne de caractères)`**

Chemin complet de l'élément à manipuler. Ce paramètre est de la forme :

```txt
"<Nom de la racine>" + TAB + ["<Nom du 1er noeud>" + TAB + ...
["<Nom du 2ème noeud>" + TAB + [...]]]"<Nom de la feuille>"
```
En cas de doublons sur un élément du chemin, ce chemin peut contenir l'identifiant de l'élément (indiqué à la suite du nom grâce à la fonction [ArbreID](../WDLang1/3018027.md)).

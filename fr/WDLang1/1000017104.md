
## ServiceListe (Fonction)

***En anglais : ServiceList***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Liste les services présents sur le poste en cours ou sur le poste distant indiqué.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
sService_Liste est une chaîne

// Vide la table
TableSupprimeTout(TABLE_Service)
// TABLE_Service contient 2 colonnes texte: "Nom" et "Description"

// Liste les services
sService_Liste = ServiceListe()
SI sService_Liste <> "" ALORS
	POUR TOUTE CHAINE sServiceInfo DE sService_Liste SEPAREE PAR RC
		TableAjoute(TABLE_Service, sServiceInfo)
	FIN

SINON 
	Erreur(ErreurInfo(errComplet))
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ServiceListe([<Poste distant>])`
---

**`<Résultat> : (Chaîne de caractères)`**



- Liste des services, renvoyée selon le format suivant : 
	
	```txt
	<Nom du service 1> + TAB + <Description du service 1> + RC + ... 
	+ <Nom du service N> + TAB + <Description du service N>
	```


- Chaîne vide ("") en cas d'erreur. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md). 




**`<Poste distant> : (Chaîne de caractères optionnelle)`**

Adresse IP du poste distant au format xxx.xxx.xxx.xxx (obtenu par la fonction [NetAdresseIP](../WDLang3/3056023.md)) ou nom du poste distant (obtenu par la fonction [NetNomMachine](../WDLang3/3056014.md)). 
Ce paramètre permet de lister les services présents sur un autre poste. 
Pour utiliser cette option, l'utilisateur peut ne pas être administrateur de la machine distante (il doit au moins posséder les droits permettant de lister les services sur cette machine). 
Si ce paramètre n'est pas précisé ou correspond à une chaîne vide (""), le service est manipulé en local.

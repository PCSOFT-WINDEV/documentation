


## SignalFerme (Fonction)

***En anglais : EventClose***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ferme un signal de synchronisation entre plusieurs threads.


<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`SignalFerme(<Signal>)`
---

**`<Signal> : (Chaîne de caractères ou variable de type SignalAutomatique ou SignalManuel)`**

Nom du signal à fermer. Ce nom peut correspondre : 

- à une chaîne de caractères, 

- au nom d'une variable de type [SignalManuel](../WDLang1/1000021275.md), 

- au nom d'une variable de type [SignalAutomatique](../WDLang1/1000021270.md). 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Cette fonction est équivalente à la syntaxe suivante de la fonction [SignalModifie](../WDLang1/3077017.md) : 
	
	```txt
	SignalModifie(<Signal>, signalFermé)
	```


- **Cas d'un signal automatique** : Le signal reste fermé. 
	**Nouveaux threads** : Si d'autres threads arrivent, ils sont mis en attente.

- **Cas d'un signal manuel** : Le signal se ferme. 
	**Nouveaux threads** : Si d'autres threads arrivent, ils sont mis en attente.




## La structure du fichier de contrôle à distance
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000116"></a>
La gestion du contrôle à distance utilise un fichier au format INI. Ce fichier de contrôle permet de gérer simplement le mode d'exécution d'une application et les messages affichés en fonction du mode. Par défaut, ce fichier s'appelle WDUPDATE.NET.

L'application WDADMINEXE permet de gérer simplement aussi bien la création que la maintenance de ce fichier de contrôle.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Structure du fichier de contrôle
<a name="structure_fichier_controle_ELTTEXTE000140"></a>
La structure du fichier de contrôle est la suivante :

```txt
[<Nom du fichier EXE à contrôler>]
MODE = 
MSG_INTERDIT = 
MSG_AVERTISSEMENT =
MSG_ARRET =
DELAI =
TEMPS_AFFICHAGE =
```
Cette structure est répétée pour chaque exécutable à contrôler par le fichier de contrôle.

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Les différents mots-clés du fichier de contrôle
<a name="les_differents_motscles_fichier_controle_ELTTEXTE000164"></a>
Le fichier de contrôle est composé des éléments suivants :


|   |   |
| --- | --- |
| Nom du fichier EXE à contrôler | Nom de l'exécutable à contrôler. Si l'exécutable est WDGESCOM.EXE, le nom du fichier EXE à contrôler sera WDGESCOM. |
| Mode | Mode de l'application contrôlée :<br><br>- 0 : fonctionnement normal de l'application<br><br>- 1 (mode *acInterdit*) : accès interdit à l'application<br><br>- 2 (mode *acAvertissement*) : avertissement d'un arrêt imminent<br><br>- 3 (mode *acArrêt*) : arrêt de l'application<br><br><br> |
| MSG_INTERDIT | Message à afficher si l'accès à l'application est interdit (mode *acInterdit*).<br>Ce message sera affiché à tout nouvel utilisateur qui tente de se connecter à l'application. |
| MSG_AVERTISSEMENT | Message à afficher pour avertir d'un arrêt imminent de l'application (mode *acAvertissement*).<br>Ce message sera affiché à intervalle de temps régulier pour avertir l'utilisateur et lui permettre de terminer la tâche en cours avant l'arrêt de l'application. |
| MSG_ARRET | Message à afficher pour avertir de l'arrêt de l'application (mode *acArrêt*).<br>Ce message sera affiché avant l'arrêt complet de l'application. |
| DELAI | Délais de vérification du fichier de contrôle (exprimé en minutes). Par défaut, ce délai est de 5 minutes. |
| TEMPS_AFFICHAGE | Temps d'affichage (exprimé en secondes) des fenêtres d'avertissement. Si aucune action n'est réalisée par l'utilisateur, la fenêtre sera automatiquement fermée. Par défaut, ce temps est de 10 secondes. |

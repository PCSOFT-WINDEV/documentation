
## EntierVersHeure (Fonction)

***En anglais : IntegerToTime***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Transforme un entier en une heure au format HHMMSSCC.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
EntierVersHeure(0)	 // Retourne "00000000"

// Attente de 4 secondes
Départ, t sont des entiers
Départ = HeureVersEntier(HeureSys())
BOUCLE
	t = HeureVersEntier(HeureSys()) - Départ
	// Affiche le temps écoulé entre 0 et 4s
	Trace(EntierVersHeure(t))
	// Sort si les 4 secondes sont écoulées
	SI t > 400 ALORS SORTIR
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = EntierVersHeure(<Valeur>)`
---

**`<Résultat> : (Chaîne de caractères ou variable de type Heure)`**

Chaîne de caractères au format HHMMSSCC où HH représente les heures, MM les minutes, SS les secondes et CC les centièmes de seconde.

**`<Valeur> : (Entier)`**

Nombre de centièmes de seconde écoulés depuis minuit (00:00:00:00). L'entier 0 correspond à minuit. &lt;Valeur&gt; ne peut pas dépasser 8.639.999 (24 heures - 1 centième de seconde).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Format HHMMSSCC
<a name="format_hhmmsscc_ELTPARAGRAPHE000082"></a>Pour obtenir l'heure sous un format différent de HHMMSSCC, il faut formater la chaîne avec les fonctions [HeureVersChaîne](../WDLang1/3027023.md), [Droite](../WDLang1/3024016.md), [Gauche](../WDLang1/3024001.md), etc.

Le format HHMMSSCC a été choisi car il permet de comparer deux heures au centième de seconde près.


<a name="NOTE0_2"></a>




### Validité de l'heure
<a name="validite_heure_ELTPARAGRAPHE000102"></a>La validité de l'heure passée en paramètre est vérifiée. Un message est affiché en cas d'heure invalide. La validité de l'heure peut être vérifiée avec la fonction [HeureValide](../WDLang1/3027021.md).


## ArbreID (Fonction)

***En anglais : TreeID***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de construire le chemin des éléments d'un champ Arbre en différenciant les doublons. Cette fonction est utilisable dans toutes les fonctions de manipulation des éléments d'un champ Arbre.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
RésultatArbre est un Booléen
RésultatArbre = ArbreAjoute(ARBRE_Arbre1, "Racine" + TAB + "Feuille", ...
		aDéfaut, aDéfaut, "Id unique feuille")
RésultatArbre = ArbreAjoute(ARBRE_Arbre1, "Racine" + TAB + "Feuille", ...
		aDéfaut, aDéfaut, "Id unique feuille 2", aAccepteDoublon)
// Suppression du 2ème doublon
ArbreSupprime(ARBRE_Arbre1, "Racine" + TAB + "Feuille" + ...
		ArbreID("Id unique feuille 2"))
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ArbreID(<Identifiant>)`
---

**`<Résultat> : (Chaîne de caractères)`**

Chaîne de contrôle à ajouter au nom de l'élément lorsque cet élément est manipulé par les fonctions de gestion des champs Arbre.

**`<Identifiant> : (Chaîne de caractères)`**

Identifiant de l'élément. Cet identifiant est utilisé pour construire la chaîne de contrôle de l'élément. Cet identifiant doit être identique à celui utilisé dans les fonctions [ArbreAjoute](../WDLang1/3018012.md), [ArbreInsère](../WDLang1/3018023.md) ou [ArbreModifie](../WDLang1/3018013.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Manipulation d'éléments doublons dans un champ Arbre
<a name="manipulation_elements_doublons_dans_champ_arbre_ELTPARAGRAPHE000042"></a>La gestion des doublons dans les champs Arbre est basée sur la gestion des identifiants. Chaque partie du chemin d'un élément d'un champ Arbre peut être associée à un identifiant. Cette association est réalisée avec les fonctions [ArbreAjoute](../WDLang1/3018012.md), [ArbreInsère](../WDLang1/3018023.md) ou [ArbreModifie](../WDLang1/3018013.md).

La fonction **ArbreID** permet de manipuler les éléments doublons à partir de leur identifiant. Il suffit d'ajouter la chaîne de contrôle correspondante à l'élément du chemin voulu.

Exemple :


```wl
ArbreSupprime(ARBRE_Arbre1, "Racine" + TAB + "Feuille" + ...
		ArbreID("Id unique feuille 2"))
```


Dans cet exemple, il y a deux éléments "Feuille". Pour identifier celui à supprimer, la fonction **ArbreID** a été utilisée.


<a name="NOTE0_2"></a>


### Quand utiliser la fonction ArbreID ? 
<a name="quand_utiliser_fonction_docparampagetitleshort_ELTPARAGRAPHE000073"></a>La fonction **ArbreID** peut être utilisée :

- dans toutes les syntaxes des fonctions de gestion des champs Arbre, dès qu'il est nécessaire d'indiquer le chemin d'un élément qui peut être un doublon.

- dans la syntaxe **&lt;Nom du champ Arbre&gt;[&lt;Chemin de élément&gt;]** qui permet de manipuler simplement un élément d'un champ Arbre.




L'élément manipulé est celui dont le chemin et l'identifiant correspondent.

**Remarque** : Il est possible de préciser un identifiant pour chaque partie du chemin d'un élément.




## Fonctions de gestion du presse-papiers
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>




Les fonctions de gestion du presse-papiers sont les suivantes :



|   |   |
| --- | --- |
| [PressePapier](../WDLang1/3063004.md) | Récupère le texte ou l'image contenu dans le presse-papiers du système. |
| [PressePapierFormat](../WDLang1/3063002.md) | Identifie le format des informations contenues dans le presse-papiers. |
| [PressePapierHauteur](../WDLang1/3063006.md) | Calcule la hauteur de l'image (au format Bitmap) contenue dans le presse-papiers. |
| [PressePapierLargeur](../WDLang1/3063007.md) | Calcule la largeur de l'image (au format Bitmap) contenue dans le presse-papiers. |
| [PressePapierRTF](../WDLang1/1000017200.md) | Récupère une chaîne RTF dans le presse-papiers (le contenu du presse-papiers est au format RTF). |
| [PressePapierUnicode](../WDLang1/1000018858.md) | Retourne le texte contenu dans le presse-papiers au format Unicode. |
| [VersPressePapier](../WDLang1/3063005.md) | Écrit des informations de type texte ou image dans le presse-papiers du système. |
| [VersPressePapierRTF](../WDLang1/1000017199.md) | Écrit une chaîne de caractères au format RTF dans le presse-papiers. |



Les fonctions suivantes permettent également de manipuler le presse-papiers : 


|   |   |
| --- | --- |
| [TableVersPressePapier](../WDLang1/1000017062.md) | Copie dans le presse-papiers le contenu d'un champ Table (champ Table hiérarchique, Table fichier ou Table par programmation). Le contenu du champ est copié dans un format texte : le contenu est converti sous forme de chaîne de caractères, et cette chaîne est copiée dans le presse-papiers. |
| [TwainVersPressePapier](../WDLang6/3078011.md) | Permet :<br><br>- soit de copier le document provenant du périphérique Twain dans le presse-papiers.<br><br>- soit de visualiser le document provenant du périphérique Twain dans l'interface utilisateur du périphérique.<br><br><br> |

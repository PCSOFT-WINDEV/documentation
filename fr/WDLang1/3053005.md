
## MemCrée (Fonction)

***En anglais : MemCreate***

<a name="Utilisation"></a>
<a name="description"></a>
Crée une zone mémoire. Cette zone mémoire peut contenir des informations de type texte ou numérique.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Création d'une zone mémoire
MemCrée("MaZoneMémoire")
 
// Lors de la saisie des fonctions MemXXX suivantes, 
// "MaZoneMémoire" sera automatiquement proposée dans la complétion
```


<a name="Exemple2"></a>



```wl
sChaîne est une chaîne = "A"
sChaîne = Complète(sChaîne, 10000, "B")
Trace(Taille(sChaîne))

MemCrée("ZM")
MemAjoute("ZM", "1", sChaîne)
sChaîne = MemPremier("ZM")
Trace(Taille(sChaîne))
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`MemCrée(<Zone mémoire>)`
---

**`<Zone mémoire> : (Chaîne de caractères)`**

Nom de la zone mémoire à créer (20 caractères maximum). Ce nom sera utilisé par toutes les fonctions de gestion des zones mémoire.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Existence de la zone mémoire
<a name="existence_zone_memoire_ELTPARAGRAPHE000080"></a>L'existence de la zone mémoire n'est pas vérifiée. Si la zone mémoire existait auparavant, cette zone mémoire est écrasée. Pour vérifier l'existence d'une zone mémoire, utilisez la fonction [MemExiste](../WDLang1/3053006.md).


<a name="NOTE0_2"></a>




### Complétion automatique sur le nom de la zone mémoire
<a name="completion_automatique_sur_nom_zone_memoire_ELTPARAGRAPHE000092"></a>Sous l'éditeur de code, dès qu'une zone mémoire a été manipulée avec une fonction Memxxx (par exemple la fonction **MemCrée**), ce nom sera automatiquement proposé lors de la saisie de fonctions WLangage acceptant le nom de la zone mémoire en paramètre.


<a name="NOTE0_3"></a>




### Taille des informations
<a name="taille_des_informations_ELTPARAGRAPHE000104"></a>Si la taille des informations à stocker dans la zone mémoire est plus importante que la place mémoire disponible, un "swap disque" est utilisé : un ou plusieurs fichiers temporaires sont créés afin de stocker le surplus d'informations. Ces fichiers temporaires seront créés sur le disque local, dans le répertoire indiqué par la variable d'environnement "TEMP".




## ChangeStyle (Fonction)

***En anglais : ChangeStyle***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie dynamiquement le style d'un champ.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Modification du style du champ SAI_Saisie1
ChangeStyle(SAI_Saisie1, "Saisie_Sans_libellé__Affichage")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ChangeStyle(<Champ manipulé> , <Style à appliquer> [, <Gabarit à appliquer>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le changement de style a été effectué, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur.




**`<Champ manipulé> : (Nom de champ)`**

Nom du champ dont le style doit être modifié.

**`<Style à appliquer> : (Chaîne de caractères)`**

Nom du style défini en édition. Ce style doit être applicable au type du champ spécifié.

**`<Gabarit à appliquer> : (Chaîne de caractères optionnelle)`**

Nom du gabarit à appliquer (nom et chemin si nécessaire du fichier "\*.WDY"). L'extension ne doit pas être précisée. 

Si ce paramètre correspond à une chaîne vide ("") :

- le gabarit appliqué sera celui défini par défaut sous l'éditeur de fenêtres. **Attention** : Si un gabarit a été dynamiquement appliqué avec la fonction [ChangeGabarit](../WDLang1/3013026.md), ce gabarit dynamique sera conservé.

- les fenêtres déjà ouvertes conserveront le gabarit actuel.




Si ce paramètre n'est pas précisé, le fichier gabarit (\*.WDY) associé au projet en cours est utilisé.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Fichier ".WDY"
<a name="fichier_wdy_ELTPARAGRAPHE000053"></a>La description d'un gabarit est contenue dans un fichier ".WDY". Les fichiers ".WDY" correspondant aux gabarits proposés par WINDEV sont livrés avec WINDEV.

Pour appliquer un style personnalisé ou un style du projet, la syntaxe à utiliser est : 

```txt
ChangeStyle(<Champ>, <StylePerso>, <NomProjet.wdy>)
```
 Sauf s'il a été renommé, le .WDY contenant les styles du projet porte le même nom que le projet lui-même. 

Remarques :

- Lors de la création d'un gabarit, le fichier ".WDY" est automatiquement créé.

- Pour générer le fichier ".WDY" d'un gabarit existant : 

	- Ouvrez le fichier "&lt;NomDuGabarit&gt;.STY" sous WINDEV : sous le volet "Accueil", dans le groupe "Général", cliquez sur "Ouvrir". 

	- Enregistrez ce fichier : sous le volet "Accueil", dans le groupe "Général", cliquez sur "Enregistrer".






<a name="NOTE0_2"></a>


### Déploiement d'une application
<a name="deploiement_une_application_ELTPARAGRAPHE000097"></a>Lors de la création du programme d'installation de votre application, les éléments suivants doivent être ajoutés dans la liste des éléments à installer :

- le fichier ".WDY" du projet.

- le fichier ".WDY" correspondant au gabarit appliqué par la fonction **ChangeStyle**.

- les fichiers images utilisés par le nouveau gabarit appliqué par la fonction **ChangeStyle**. Attention : les images utilisées par les différents gabarits doivent avoir des noms différents, sinon ces images ne seront pas modifiées.




**Remarque** : Pour ajouter un ou des éléments à la liste des éléments d'un projet :

1. Sous le volet "Projet", dans le groupe "Projet", déroulez "Liste des éléments".

2. Sélectionnez l'option "Liste des éléments du projet".

3. Cliquez sur le bouton "Ajouter".

4. Sélectionnez le ou les fichiers à ajouter et validez.



<a name="NOTE0_3"></a>


### Divers
<a name="divers_ELTPARAGRAPHE000133"></a>

- Il est également possible de copier le style d'un champ vers un autre grâce à la propriété [Style](../Proprietes/1000021472.md). 

- Lors du changement de style, le positionnement des libellés n'est pas modifié. En effet, le changement du positionnement des libellés pourrait provoquer des modifications de taille du champ. Le champ risquerait d'entrer en conflit avec d'autres champs. 

- Champs Bouton : 

	- Si le style appliqué possède une image, cette image écrase l'image d'origine du bouton. 

	- Si le style appliqué ne possède pas d'image, l'image d'origine du bouton est conservée.




## EnModeProcédureDifférée (Fonction)

***En anglais : InDelayedProcedureMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie si le code est exécuté depuis une procédure différée ou une tâche planifiée du serveur d'application WEBDEV.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI PAS EnModeProcédureDifférée() ALORS
	...
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = EnModeProcédureDifférée()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code courant : 

	- est dans une procédure lancée par la fonction [ExécuteProcédureDifférée](../WDLang2/1000020682.md) ou lancée comme tâche planifiée par le serveur d'application WEBDEV.

	- est appelé directement ou indirectement par une procédure lancée par la fonction [ExécuteProcédureDifférée](../WDLang2/1000020682.md) ou lancée comme tâche planifiée par le serveur d'application WEBDEV.




- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000196"></a>La fonction **EnModeProcédureDifférée** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeProcédureDifférée** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.


## dbgActiveAssertion (Fonction)

***En anglais : dbgEnableAssert***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active ou désactive la programmation défensive. La programmation défensive a été mise en place grâce à la fonction [dbgAssertion](../WDLang1/3014022.md).




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
dbgActiveAssertion(Vrai)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = dbgActiveAssertion([<Activation>])`
---

**`<Résultat> : (Booléen)`**

Si le paramètre &lt;Activation&gt; n'est pas précisé, &lt;Résultat&gt;  correspond au fonctionnement actuel : 

- &lt;Résultat&gt; = <u><u><u><u>Vrai</u></u></u></u> si la programmation défensive est activée, 

- &lt;Résultat&gt;  = <u><u><u><u>Faux</u></u></u></u> si la programmation défensive est désactivée. 




Si le paramètre &lt;Activation&gt; est précisé, &lt;Résultat&gt;  correspond à l'ancien mode de fonctionnement : 

- &lt;Résultat&gt; = <u><u><u><u>Vrai</u></u></u></u> si la programmation défensive est activée, 

- &lt;Résultat&gt;  = <u><u><u><u>Faux</u></u></u></u> si la programmation défensive est désactivée. 




**`<Activation> : (Booléen optionnel)`**



- <u><u><u><u>Vrai</u></u></u></u> pour activer la programmation défensive, 

- <u><u><u><u>Faux</u></u></u></u> pour la désactiver (valeur par défaut).






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Par défaut, en mode test, la programmation défensive est activée.

- Par défaut, en exécution, la programmation défensive est désactivée.

- Si le mode JITc est activé (mode par défaut), cette fonction doit être appelée dans le code d'initialisation du projet. Pour plus de détails, consultez [Technologie JITc](../Editeurs/2020009.md).

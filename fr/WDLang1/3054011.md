## Transformer un projet en projet multilingue
<a name="transformer_projet_projet_multilingue_ELTTEXTE000852"></a>
Cette page propose un résumé des principales manipulations à effectuer pour transformer vos projets en projet multilingues : 

- [Choix des différentes langues](#NOTE1_2). 

- [Prise en compte de la langue lors de la création d'un champ](#NOTE1_3). 

- Traduction des différents éléments : 

	- [Sans outil spécifique](#NOTE1_4).

	- [Avec WDINT et WDMSG](#NOTE1_5).




- [Affichage de l'interface dans une langue spécifique sous les éditeurs](#NOTE1_6). 

- [Test dans les différentes langues](#NOTE1_7). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) [Création de l’exécutable](#NOTE1_8).




Attention : Il est également nécessaire de gérer les cas particuliers correspondant aux différentes fenêtres et pages proposées par défaut. Pour plus de détails, consultez [Cas particulier : Traduction des fenêtres et pages proposées par défaut](#NOTE2_1).
<a name="NOTE1_2"></a>


### 1. Choix des différentes langues
<a name="1_choix_des_differentes_langues_ELTPARAGRAPHE000052"></a>


|   |   |
| --- | --- |
| <br><br>- **Projet**<br><br><br> | La sélection des langues du projet est réalisée dans la description du projet :<br><br>1. Sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".<br><br>2. Sélectionnez l'onglet "Langues" et utilisez le bouton "Ajouter". <br><br>3. Les langues sélectionnées sont prises en compte pour tous les éléments (les éléments ouverts sont automatiquement fermés).<br><br><br> |
| <br><br>- **Fenêtres, pages, états et champs (texte, masques de saisie et images)**<br><br><br> | Gestion automatique des langues pour les langues gérées par le projet. <br>Cas des éléments partagés entre plusieurs projets qui n'utilisent pas les mêmes langues : lors de l'ouverture de l'élément dans un des projets, les langues supplémentaires sont automatiquement ajoutées dans l'élément. |
| <br><br>- **Système d'aide (hlp ou chm)**<br><br><br> | Gestion automatique des langues pour les langues gérées par le projet. <br>Cas des aides partagées entre plusieurs projets qui n'utilisent pas les mêmes langues : lors de l'ouverture de l'élément dans un des projets, les langues supplémentaires sont automatiquement ajoutées dans l'élément. |
| <br><br>- **Messages du code (code, procédures, classes)**<br><br><br> | Gestion automatique des langues pour les langues gérées par le projet.<br>Cas des classes et procédures partagées entre plusieurs projets : lors de l'ouverture de l'élément dans un des projets, les langues supplémentaires sont automatiquement ajoutées dans l'élément. . |

### 2. Création d'un nouveau champ
<a name="2_creation_nouveau_champ_ELTPARAGRAPHE000096"></a>


|   |   |
| --- | --- |
| <br><br>- **Fenêtres, pages, états et champs (texte, masques de saisie et images)**<br><br><br> | Par défaut, seul le libellé de la langue d'exécution est indiqué. Il est nécessaire de traduire le libellé du champ dans les différentes langues gérées. |
| <br><br>- **Système d'aide (hlp ou chm)**<br><br><br> | Le titre des nouvelles pages et les mots-clés sont automatiquement créés en US. |

### 3. Traduction sans outil spécifique (directement dans WINDEV)
<a name="3_traduction_sans_outil_specifique_directement_dans_windev_ELTPARAGRAPHE000110"></a>


|   |   |
| --- | --- |
| <br><br>- **Fenêtres, pages, états et champs (texte, masques de saisie et images)**<br><br><br> | La traduction doit être réalisée dans toutes les fenêtres de description des fenêtres, des pages, des états et des champs. |
| <br><br>- **Système d'aide (hlp ou chm)**<br><br><br> | Pour le sommaire et les index : la traduction doit être réalisée dans les fenêtres de description de la page. <br>Pour le contenu de la page d'aide : la traduction doit être réalisée après le changement de la langue d'affichage. |
| <br><br>- **Messages du code (code, procédures, classes)**<br><br><br> | La traduction doit être réalisée sous l'éditeur de messages (Ctrl + T). |

### Affichage sous l'éditeur dans une langue
<a name="affichage_sous_editeur_dans_une_langue_ELTPARAGRAPHE000145"></a>


|   |   |
| --- | --- |
| <br><br>- **Projet**<br><br><br> | Changement de la langue d'affichage de tous les éléments du projet : sous le volet "Affichage", dans le groupe "Options", déroulez "Langue affichée"<br>**Attention : la langue par défaut en exécution du projet n'est pas modifiée.** |
| <br><br>- **Fenêtres, pages et champs (texte, masques de saisie et images)**<br><br><br> | Changement de la langue d'affichage automatique sous l'éditeur si la langue d'affichage du projet a été modifiée. <br>Pour modifier la langue d'affichage de tous les éléments du projet : sous le volet "Affichage", dans le groupe "Options", déroulez "Langue affichée" (\*) |
| <br><br>- **États et champs (texte, masque d'affichage et images)**<br><br><br> | Changement de la langue d'affichage automatique sous l'éditeur si la langue d'affichage du projet a été modifiée. <br>Pour modifier la langue d'affichage de tous les éléments du projet : sous le volet "Affichage", dans le groupe "Options", déroulez "Langue affichée" (\*) |
| <br><br>- **Système d'aide (hlp ou chm)**<br><br><br> | Changement de la langue d'affichage automatique sous l'éditeur si la langue d'affichage du projet a été modifiée <br>Pour modifier la langue d'affichage de tous les éléments du projet : sous le volet "Affichage", dans le groupe "Options", déroulez "Langue affichée" (\*) |
| <br><br>- **Messages du code (code, procédures, classes)**<br><br><br> | Changement de la langue d'affichage automatique sous l'éditeur si la langue d'affichage du projet a été modifiée. <br>Pour modifier la langue d'affichage de tous les éléments du projet : sous le volet "Affichage", dans le groupe "Options", déroulez "Langue affichée" (\*) |

### Test
<a name="test_ELTPARAGRAPHE000241"></a>


|   |   |
| --- | --- |
| <br><br>- **Projet**<br><br><br> | Test du projet effectué dans la langue choisie comme "langue par défaut en exécution" du projet. La langue par défaut en exécution est sélectionnée dans la fenêtre de description du projet (onglet "Langues"). |
| <br><br>- **Fenêtres, pages et champs (texte, masques de saisie et images)**<br><br><br> | Test de la fenêtre effectué dans la langue par défaut du projet. |
| <br><br>- **États et champs (texte, masque d'affichage et images)**<br><br><br> | Test de l'état effectué dans la langue par défaut du projet. |
| <br><br>- **Système d'aide (hlp ou chm)**<br><br><br> | Compilation et test dans la langue d'affichage de l'aide. <br>Il faut créer autant de fichiers d'aide que de langues. |
| <br><br>- **Messages du code (code, procédures, classes)**<br><br><br> | Lors d'un test (fenêtre ou projet), les messages du code sont affichés dans la langue par défaut du projet. |
| <br><br>- **Messages WINDEV**<br><br><br> | Définissez le fichier WDM associé à chaque langue dans la description du projet :<br><br>1. Sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".<br><br>2. Sélectionnez l'onglet "Langues".<br><br>3. Sélectionnez la langue voulue. <br><br>4. Dans l'onglet "Divers", sélectionnez le fichier WDM de traduction. <br><br><br> |

| <br><br>- **Messages WINDEV**<br><br><br> | Définissez le fichier WDM associé à chaque langue dans la description du projet :<br><br>1. Sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".<br><br>2. Sélectionnez l'onglet "Langues".<br><br><br> |

## Cas particulier : Traduction des fenêtres et pages proposées par défaut
<a name="cas_particulier_traduction_des_fenetres_pages_proposees_par_defaut_ELTTEXTE000924"></a>
Les éléments suivants peuvent être intégrés automatiquement à vos applications / sites : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) fenêtre d'aperçu avant impression.

- fenêtres HFSQL de gestion automatique des erreurs de programmation HFSQL.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) fenêtres ou pages du groupware utilisateur. Ces éléments permettent de gérer la connexion d'un utilisateur à une application, et les différents niveaux d'accès pour chaque utilisateur.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) fenêtres des FAA (Fonctionnalités Automatiques de vos Applications). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) le programme d'installation, permettant à l'utilisateur d'installer son application ou site.

### Intégrer et traduire les fenêtres et pages proposées par défaut
<a name="integrer_traduire_les_fenetres_pages_proposees_par_defaut_ELTPARAGRAPHE000502"></a>**Par défaut, ces fenêtres et pages sont disponibles en français et en anglais**. Pour gérer des langues supplémentaires, les manipulations suivantes doivent être effectuées :




### Visualisateur de rapports
<a name="visualisateur_rapports_ELTPARAGRAPHE000508"></a>


|   |   |
| --- | --- |
| <br><br>- **But**<br><br><br> | Afficher l'état en mode "Aperçu" à l'écran et permettre à l'utilisateur de lancer l'impression sur le support voulu (imprimante, fichier XML, ...). |
| <br><br>- **Intégration dans le projet**<br><br><br> | Demandez l'intégration des fenêtres dans votre projet :<br><br>1. Affichez la description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".<br><br>2. Affichez l'onglet "Style".<br><br>3. Dans l'option "Visualisateur de rapports", sélectionnez l'option "Personnalisé (intégré dans le projet)".<br><br>4. Sélectionnez le type de personnalisation à effectuer :<br><br>	- &lt;Visualisateur de rapports standard (sans gabarit)&gt; : cette option importe directement les fenêtres standard d'impression dans votre projet.<br><br>	- &lt;Visualisateur de rapports standard au gabarit du projet&gt; : cette option importe directement les fenêtres d'impression correspondant au gabarit de votre projet.<br><br><br><br><br><br> |
| <br><br>- **Traduction sans outil spécifique ou avec WDMSG**<br><br><br> | Voir paragraphe précédent. |
| <br><br>- **Utilisation dans la langue choisie**<br><br><br> | L'utilisation de la langue est automatique en fonction de la langue par défaut en exécution du projet. |
| <br><br>- **Changement de la langue avec la fonction [Nation](../WDLang1/3054014.md)**<br><br><br> | Le changement de langue est automatique. |

- [Impression automatique des champs Agenda](../WDChamp/1016327.md) : Possibilité de personnaliser et de traduire la fenêtre de configuration de l'impression.





- ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=WDListeChainee.gif) ***Exemples didactiques (WINDEV)*** : **WD ListeChaînée** <br>Cet exemple fournit les outils pour construire des listes chaînées à chaînage simple. Une liste à chaînage simple permet de stocker et parcourir de manière ordonnée une collection de valeurs quelconques.<br><br>Dans l'exemple, qui est autoexplicatif, la liste chaînée permet de stocker des boutons. <br>- La liste est ensuite utilisée pour masquer/déplacer les boutons dans l'écran, créant un menu dynamique de type Sidebar.<br>- La liste est aussi utilisée pour traduire les boutons - à la volée - c'est à dire sans besoin de recharger la fenêtre. Pour cela, on stocke aussi pour chaque bouton le libellé français et le libellé anglais. L'avantage de procéder ainsi est de conserver l'état de la fenêtre, notamment le menu dans son aspect déroulé, le texte explicatif, etc .<br><br>Particularité de cet exemple:<br>- il est totalement bilingue (français / anglais).

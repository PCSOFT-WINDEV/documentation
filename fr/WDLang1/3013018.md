 

## DéchargeProcédure (Fonction)

***En anglais : UnloadProcedure***
	

<a name="XUtilisation"></a>
<a name="Utilisation"></a>
|  |   |
| --- | --- |



|  
 | ButDécharger les procédures internes d'une fenêtre qui ont été rendues globales par [ChargeProcédure](../WDLang1/3013019.md).Syntaxe&lt;Résultat&gt; = DéchargeProcédure(&lt;NomFenetre&gt; [, &lt;Message&gt;])<br><br>- &lt;Résultat&gt; est un booléen permettant de savoir si les procédures ont été déchargées<br><br>- &lt;NomFenetre&gt; est une chaîne de caractères contenant le nom ou l'alias de la fenêtre dans laquelle les procédures ont été décrites.<br><br>- &lt;Message&gt; est un booléen permettant d'afficher un message d'erreur si les procédures de la fenêtre ne sont pas trouvées.<br><br><br>Détail&lt;Résultat&gt; renvoie Vrai si le déchargement a été correctement effectué, Faux sinon.<br><br>Si &lt;Message&gt; = Vrai (valeur par défaut), un message d'erreur signale si &lt;Nom Fenêtre&gt; n'est pas trouvée à l'exécution.<br><br>Si &lt;Message&gt; = Faux, aucun message d'erreur signale l'absence de &lt;NomFenêtre&gt;. Il est nécessaire de tester la valeur de &lt;Résultat&gt; par programmation.Notes<br><br>- **DéchargeProcédure	** doit être utilisée pour charger les procédures internes d'autres fenêtres avec [ChargeProcédure](../WDLang1/3013019.md).<br><br>- Les procédures internes de 16 fenêtres (au maximum) peuvent être chargées par [ChargeProcédure](../WDLang1/3013019.md).<br><br>- Lors de l'exécution d'une procédure, celle-ci est recherchée :<br><br>	- en premier dans les procédures internes de la fenêtre,<br><br>	- ensuite dans les différents "modules" chargés,<br><br>	- enfin dans les procédures globales.<br><br><br><br><br>- Les procédures chargées par [ChargeProcédure](../WDLang1/3013019.md) sont automatiquement déchargées lors de la fermeture du projet.<br><br><br><br><br> |
| --- | --- |

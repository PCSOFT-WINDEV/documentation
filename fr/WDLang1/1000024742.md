
## MasqueCodePostal (Fonction)

***En anglais : MaskZipCode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère le masque de saisie du code postal adapté à un pays donné.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Utilise le masque du code postal espagnol sur le champ SAI_CodePostal
SAI_CodePostal.MasqueSaisie = MasqueCodePostal("ES")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = MasqueCodePostal(<Pays> [, <Option>])`
---

**`<Résultat> : (Variable de type MasqueDeSaisie)`**

Variable de type [MasqueDeSaisie](../WDLang1/1000024886.md) contenant les caractéristiques du masque de saisie du code postal pour le pays spécifié. Ce résultat peut être affecté directement à la propriété [MasqueSaisie](../Proprietes/2510033.md).  

**`<Pays> : (Chaîne de caractères ou variable de type Pays)`**

Pays pour lequel le masque de saisie du code postal doit être récupéré. Ce paramètre peut correspondre à : 

- Le code ISO alphabétique ou numérique du pays voulu (par exemple, le code ISO alphabétique de la France est "FR").

- Une variable de type [Pays](../WDLang1/1000024921.md). 


Si ce pays est vide ou inconnu, une exception est déclenchée. 

**`<Option> : (Entier optionnel)`**

Mode de gestion de la sortie du champ : 



|   |   |
| --- | --- |
| *mcpBloqueSortie* | Interdit de sortir du champ tant que la valeur saisie n'est pas valide (par exemple, 5 chiffre pour le code postal de la France).  |
| *mcpSortieLibre*<br>(Valeur par défaut) | Autorise la sortie du champ même si le masque de saisie n'est pas vérifié. |


Ce paramètre est pris en compte uniquement si l'option "Bloquer la sortie du champ" est sélectionnée dans l'onglet "Détail" de la fenêtre de description du champ. Si cette option n'est pas sélectionnée, la sortie du champ n'est jamais bloquante et n'affiche rien quelle que soit la valeur saisie dans le champ.

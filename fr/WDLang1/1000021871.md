
## FenEdSelect (Fonction)

***En anglais : WinEdSelect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie l'un des champs sélectionnés lors de la modification de l'interface par l'utilisateur final (mode "Modification de l'UI par l'utilisateur final" activé). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Désélectionne le champ en cours
X est un Champ
X <- FenEdSelect(FEN_Formulaire) 
FenEdSelectMoins(FEN_Formulaire, X)
```


<a name="Exemple2"></a>

```wl
// Enumération des champs sélectionnés
X est un Champ
POUR i = 1 _A_ FenEdSelectOccurrence(FEN_Formulaire)
	X <- FenEdSelect(FEN_Formulaire, i)
	Trace(X.Nom)	
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = FenEdSelect(<Fenêtre> [, <Indice>])`
---

**`<Résultat> : (Champ)`**



- Si la sélection multiple n'est pas activée, ce paramètre correspond au champ sélectionné. 

- Si la sélection multiple est activée, ce paramètre correspond au Nième champ de la sélection (paramètre &lt;Indice&gt;). 


Ce résultat peut être affecté à une variable de type [Champ](../WDLang1/1000020967.md) pour être manipulé. 

**`<Fenêtre> : (Nom de fenêtre)`**

Nom de la fenêtre ou de la fenêtre interne à manipuler. Cette fenêtre doit être en mode modification (fonction [FenEdActive](../WDLang1/1000021843.md)). 

**`<Indice> : (Entier optionnel)`**

Indice de l'élément dans le cas d'une sélection multiple. Si ce paramètre n'est pas spécifié, le premier champ sélectionné est pris en compte.

***En anglais : InUniversalWindowsPlatformSimulatorMode***

	<IMG style="float:left;margin-right:10px;" src="https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=warning_big.png"><B>Avertissement</B><BR>

<a name="Utilisation"></a>
<a name="description"></a>
Identifie si le code est exécuté dans le simulateur pour les applications en mode Universal Windows Platform (Universal Windows 10 App) ou non.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code exécuté dans le simulateur Universal Windows 10 App ?
SI EnModeSimulateurUniversalWindowsPlatform() = Vrai ALORS
	// Code spécifique pour le simulateur
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = EnModeSimulateurUniversalWindowsPlatform()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté dans le simulateur pour les applications en mode Universal Windows Platform (Universal Windows 10 App), 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.







## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000165"></a>La fonction **EnModeSimulateurUniversalWindowsPlatform** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeSimulateurUniversalWindowsPlatform** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.


### Exécution des traitements contenant des erreurs de compilation
<a name="execution_des_traitements_contenant_des_erreurs_compilation_ELTPARAGRAPHE000181"></a>Par défaut, le mécanisme de sécurité du WLangage se déclenche uniquement si la ligne contenant l'erreur de compilation est exécutée.

Par exemple, si une fonction interdite dans le produit en cours est présente dans une boucle SI, le mécanisme de sécurité du WLangage se déclenchera uniquement si la ligne contenant cette fonction s'exécute.

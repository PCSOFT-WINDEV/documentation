
## InfoBitmap (Fonction)

***En anglais : BitmapInfo***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère les caractéristiques d'un fichier image ou d'un fichier PDF :

- format,

- largeur et hauteur,

- nombre de bits par pixel.




![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Les fichiers PDF ne sont pas gérés.  







![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les dessins](../WDLang1/3029035.md). 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 
```wl
InfoImage, Format sont des chaînes
Hauteur, Largeur sont des entiers
// Récupération des caractéristiques de l'image
InfoImage = InfoBitmap("C:\Windows\CARREAU.BMP")
Format = ExtraitChaîne(InfoImage, 1)
Largeur = Val(ExtraitChaîne(InfoImage, 2))
Hauteur = Val(ExtraitChaîne(InfoImage, 3))
// Affichage des caractéristiques
Info("Format de l'image : " + Format, "Largeur : " + Largeur, "Hauteur : " + Hauteur)
```

## Syntaxe

`<Résultat> = InfoBitmap(<Nom du fichier image>)`
---

**`<Résultat> : (Chaîne de caractères)`**



- Caractéristiques de l'image renvoyées sous la forme :
	
	```txt
	<Format> + TAB + <Largeur> + TAB + <Hauteur> + TAB + <Bits/pixel>
	```

	


|   |   |   |
| --- | --- | --- |
| &lt;Format&gt; | Chaîne de caractères | <br><br>	- Trois caractères permettant d'identifier le format de l'image : "BMP", "GIF", "PDF", ...<br><br>	- "BAD" si le nom de l'image est un fichier inexistant ou si le format est inconnu<br><br><br> |
| &lt;Largeur&gt; | Entier | Largeur de l'image en pixels.<br>Remarque : La largeur et la hauteur peuvent être inversées en fonction de la position du point (0,0) de l'image. Pour connaître la largeur et la hauteur en prenant en compte l'orientation de l'image, utilisez une variable de type  [Image](../WDLang1/1000019650.md). |
| &lt;Hauteur&gt; | Entier | Hauteur de l'image en pixels.<br>Remarque : La largeur et la hauteur peuvent être inversées en fonction de la position du point (0,0) de l'image. Pour connaître la largeur et la hauteur en prenant en compte l'orientation de l'image, utilisez une variable de type  [Image](../WDLang1/1000019650.md). |
| &lt;Bits/pixel&gt; | Chaîne de caractères | Nombre de bits par pixel :<br><br>	- 1 (Image monochrome)<br><br>	- 4 (palette 16 couleurs)<br><br>	- 8 (palette 256 couleurs)<br><br>	- 24 (pas de palette, 16 millions de couleurs)<br><br>	- 32 (pas de palette, couleur vrai)<br><br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Le nombre de bits par pixels est toujours 0.<br><br>![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Le nombre de bits par pixels est toujours 32. |



- En cas d'erreur, la chaîne renvoyée est : 
	```txt
	"BAD" + TAB + TAB + TAB
	```





**`<Nom du fichier image> : (Chaîne de caractères)`**



- Nom du fichier image ou du fichier PDF avec son chemin complet ou relatif (260 caractères maximum). Un chemin de type UNC peut être utilisé.

- Nom du champ lié à une rubrique de type mémo image.




![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Les fichiers PDF ne sont pas gérés.

## Remarques




### Divers
<a name="divers_ELTPARAGRAPHE000152"></a>

- La fonction **InfoBitmap** peut être utilisée avec les fichiers au format PDF. 
	Attention : [le Framework GDI +](../Editeurs/2020007.md) doit être installé sur le poste.
	![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Cette fonctionnalité n'est disponible. 

- Pour les images au format WMF ou EMF, &lt;Bits/pixel&gt; est égal à 0.

- Les formats des fichiers image reconnus sont :

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Tous les formats décrits dans la fonction [fFiltreImage](../WDLang1/3036009.md).

	- ![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) JPEG, PNG, GIF ou BMP.

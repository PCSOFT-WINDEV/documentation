
## Multitâche (Fonction)

***En anglais : Multitask***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de :

- effectuer une temporisation,

- rendre la main à Windows,

- rendre la main à Windows et au WLangage.













<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Boucle de calcul
POUR i = 1 A 10000
	ProcédureCalcul(i)
	// On rend la main au système pour afficher les résultats
	Multitâche()
FIN
// Attendre 1 seconde 
Multitâche(100)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`Multitâche([<Temporisation>])`
---

**`<Temporisation> : (Entier optionnel ou Durée optionnelle)`**



- **Effectuer une temporisation** : &lt;Temporisation&gt; = entier positif. 
	L'exécution de l'application est suspendue durant &lt;Temporisation&gt; 100ème de seconde sans rendre la main au système. Les clics de souris et les actions sur les touches du clavier sont ignorés par l'application. Par exemple, la fenêtre peut être déplacée mais pas agrandie.
	Les [traitements optionnels](../WDChamp/1014004.md) des champs sont toujours exécutés.
	**Remarque** : Si &lt;Temporisation&gt; = 1, l'application rend directement la main à Windows, sans attendre 1 centième de seconde.
	![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) La modification de la taille de la fenêtre n'est pas ignorée. Par contre, les clics souris et les actions sur les touches du clavier sont ignorés.
	![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Les valeurs positives ne sont pas autorisées.

- **Rendre la main au système** : &lt;Temporisation&gt; = 0 ou n'est pas indiqué. 
	Le système finit les traitements en cours, rend la main à l'application WINDEV, WEBDEV ou WINDEV Mobile, puis prend en compte les clics de souris et les actions sur les touches du clavier.
	![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Si &lt;Temporisation&gt; = 0, la fonction **Multitâche** est sans effet.
	

- **Rendre la main à Windows et au WLangage** : &lt;Temporisation&gt; = entier négatif. 
	L'exécution de l'application est suspendue durant &lt;Temporisation&gt; 100ème de seconde. D'autres traitements peuvent être exécutés durant cette période de temps (ré-affichage ou exécution d'un code de clic par exemple). Les interactions avec l'utilisateur (clics de souris, frappe au clavier, ...) sont gérées normalement. 
	**Remarque** : Si &lt;Temporisation&gt; = -1, l'application rend directement la main à Windows, sans attendre 1 centième de seconde.


Remarque : Ce paramètre peut correspondre à : 

- un entier correspondant au nombre de centièmes de seconde, 

- une variable de type Durée, 

- l'indication directe de la durée (par exemple '1s' ou '10cs').

## Remarques
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) 

### Exemple de temporisation
<a name="exemple_temporisation_ELTPARAGRAPHE000134"></a>L'utilisateur appuie sur la touche Echap avant ou pendant l'appel de la fonction **Multitâche** :

- si &lt;Temporisation&gt; est positif : La touche Echap n'est pas prise en compte.

- si &lt;Temporisation&gt; est égal à 0 : La touche Echap est prise en compte à la fin des traitements WINDEV en cours.

- si &lt;Temporisation&gt; est négatif : La touche Echap est prise en compte immédiatement.



<a name="NOTE0_2"></a>




### Conseils
<a name="conseils_ELTPARAGRAPHE000149"></a>

- Pour rafraîchir l'affichage, il est conseillé d'utiliser la fonction [MultitâcheRepeint](../WDLang1/3038004.md).

- Les événements et timers lancés avant l'exécution de la fonction **Multitâche** sont gérés lors de la temporisation.

- Dans une application de type Service, la fonction **Multitâche** doit être remplacée par la fonction [ServiceTemporise](../WDLang1/1000018712.md).





<a name="NOTE0_3"></a>




### Note
<a name="note_ELTPARAGRAPHE000172"></a>Il est conseillé d'utiliser la fonction [ThreadPause](../WDLang1/3077029.md) à la place des fonctions **Multitâche** ou [Temporisation](../WDLang1/3015012.md) lorsque plusieurs threads sont utilisés (y compris pour le thread principal de l'application) si la temporisation n'a pas besoin de traiter les actions de l'utilisateur.

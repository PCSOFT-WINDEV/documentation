
## MultitâcheRepeint (Fonction)

***En anglais : MultitaskRedraw***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Re-dessine immédiatement les champs de la fenêtre nécessitant une mise à jour graphique.

**Remarque** : Cette fonction est identique à la fonction [FenRepeint](../WDLang1/3038006.md).



![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Cette fonction est disponible uniquement pour simplifier la Webisation de projets WINDEV/WINDEV Mobile. Dans un site WEBDEV, cette fonction n'a aucun effet.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
i est un entier
POUR i = 1 A 1000
	TraitementLong()
	MultitâcheRepeint()
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`MultitâcheRepeint([<Fenêtre ou page>])`
---

**`<Fenêtre ou page> : (Nom de fenêtre ou nom de page)`**

Nom de la fenêtre à re-dessiner. Si ce paramètre n'est pas précisé, la fenêtre en cours est re-dessinée. 





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Contrairement à la fonction [Multitâche](../WDLang1/3015004.md), l'appel de la fonction **MultitâcheRepeint** n'interrompt pas le code en cours (les fonctions [Timer](../WDLang1/3015008.md) ne sont pas appelées par exemple).

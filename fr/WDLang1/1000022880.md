


## &lt;Agenda&gt;.Affiche (Fonction)

***En anglais : &lt;Organizer&gt;.Display***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Réinitialise le contenu d'un champ Agenda à partir de sa source de données (fichier de données HFSQL par exemple). Tous les rendez-vous sont supprimés du champ Agenda et sont re-créés à partir de la source de données. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajout d'un rendez-vous dans un fichier de données HFSQL
FIC_RendezVous.Ajoute()
// Réinitialisation du champ Agenda
AGD_Agenda.Affiche()
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Agenda>.Affiche()`
---

**`<Champ Agenda> : (Nom de champ)`**

Nom du champ Agenda à manipuler. Ce champ peut correspondre : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) au champ Agenda d'une fenêtre. 


Le champ Agenda doit être lié à une source de données. Dans le cas contraire (par exemple si le champ est rempli par programmation), la fonction **&lt;Agenda&gt;.Affiche** n'a aucun effet.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si la source de données liée au champ Agenda est une requête, cette requête est réinitialisée.

- La position dans le champ Agenda n'est pas modifiée.

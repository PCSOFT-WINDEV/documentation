
## CombinaisonDepuisNom (Fonction)

***En anglais : CombinationFromName***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie une combinaison d'une ou de plusieurs options connues par leurs noms.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la combinaison
UneCombinaison est une combinaison
	Option1
	Option2
FIN

o est UneCombinaison

// o est affecté avec l'option "Option1"
o = CombinaisonDepuisNom(UneCombinaison, "Option1")

// o est affecté avec la combinaison des options "Option1" et "Option2"
o est UneCombinaison = CombinaisonDepuisNom(UneCombinaison, "Option1 + Option2")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = CombinaisonDepuisNom(<Combinaison> , <Options> [, <Paramètres>])`
---

**`<Résultat> : (Variable de type Combinaison)`**

Variable de type [Combinaison](../Motscles/1514074.md) contenant la combinaison d'options. 

**`<Combinaison> : (Variable de type Combinaison)`**

Nom de la variable de type [Combinaison](../Motscles/1514074.md) à manipuler.  

**`<Options> : (Chaîne de caractères)`**

Options du type Combinaison à affecter à la variable de type [Combinaison](../Motscles/1514074.md). Si plusieurs options sont manipulées, elles doivent être séparées par l'opérateur '+'. Exemple : "Option1 + Option2". 

**`<Paramètres> : (Entier optionnel)`**

Mode de gestion des options inconnues : 


|   |   |
| --- | --- |
| *combinaisonErreurOptionInconnue* | L'utilisation d'une option inconnue provoquera une erreur WLangage. |
| *combinaisonIgnoreOptionInconnue* | Les options inconnues seront ignorées. |

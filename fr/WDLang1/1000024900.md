


## &lt;Lecteur PDF&gt;.Sauve (Fonction)

***En anglais : &lt;PDF Reader&gt;.Save***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Sauve le contenu d'un champ Lecteur PDF dans un fichier.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Sauve le fichier 
sNomFichier est une chaîne = "C:\temp\MonFichier.pdf"

PDF_MonLecteurPDF.Sauve(sNomFichier)

SI ErreurDétectée ALORS
	Erreur(ChaîneConstruit("Le fichier %1 n'a pas pu être enregistré.", ...
			sNomFichier), ErreurInfo())
	RETOUR
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = <Champ Lecteur PDF>.Sauve([<Chemin du fichier>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier a été sauvé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Champ Lecteur PDF> : (Nom de champ)`**

Nom du champ Lecteur PDF à manipuler.

**`<Chemin du fichier> : (Chaîne de caractères optionnelle)`**

Chemin complet et nom du fichier "PDF".

**Si ce paramètre est précisé,** le fichier est sauvegardé à l'emplacement indiqué (le fichier existant est écrasé si nécessaire). 

- Si la sauvegarde est réussie, le fichier "PDF" reste ouvert. 

- Si le document "PDF" est enregistré avec un chemin différent de celui d'origine (correspond à un "Enregistrer sous"), le fichier d'origine est fermé, le PDF est enregistré dans le nouveau fichier et le nouveau fichier "PDF" reste ouvert.  


**Si ce paramètre n'est pas précisé**, le chemin pris en compte est le chemin du fichier PDF actuellement affiché dans le champ Lecteur PDF. Si ce chemin n'existe pas, une erreur est affichée.

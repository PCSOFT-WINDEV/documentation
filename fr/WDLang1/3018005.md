


## ArbrePosition (Fonction)

***En anglais : TreePosition***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Affiche un champ Arbre à partir d'un élément (noeud ou feuille) spécifié ou renvoie le nom du premier élément affiché dans un champ Arbre.

Exemple de champ Arbre :<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Arbre_Vocabulaire.gif)

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Nom du premier élément affiché dans le champ Arbre "ARBRE_RecetteTV"
ResNomElément = ArbrePosition(ARBRE_RecetteTV)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le nom du premier élément affiché dans un champ Arbre

`<Résultat> = ArbrePosition(<Champ Arbre> [, <Identifiant doublons>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Chemin complet du premier élément affiché dans le champ Arbre. 
Ce résultat est de la forme :

```txt
"<Nom de la racine>" + TAB + ["<Nom du 1er noeud>" + TAB + ...
["<Nom du 2ème noeud>" + TAB + [...]]]"<Nom de la feuille>"
```
En cas de doublons sur un élément du chemin, ce chemin peut contenir l'identifiant de l'élément (indiqué à la suite du nom grâce à la fonction [ArbreID](../WDLang1/3018027.md)).

**`<Champ Arbre> : (Nom de champ)`**

Nom du champ Arbre à manipuler.

**`<Identifiant doublons> : (Booléen optionnel)`**

Indique le mode de gestion des doublons dans les éléments du &lt;Résultat&gt; :

- <u><u><u><u>Faux</u></u></u></u> (par défaut) : aucune gestion n'est effectuée.

- <u><u><u><u>Vrai</u></u></u></u> : les noms des éléments du chemin sont construits de manière à contenir leur identifiant. Pour plus de détails, consultez [Gestion des doublons dans les champs Arbre](../WDChamp/1013053.md).


![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Ce paramètre n'est pas pris en compte.


<a name="SYNTAXE2"></a>

### Afficher un champ Arbre à partir d'un élément spécifié

`<Résultat> = ArbrePosition(<Champ Arbre> , <Chemin du noeud>)`
---

**`<Résultat> : (Chaîne de caractères)`**

Chemin complet du premier élément affiché dans le champ Arbre après exécution de la fonction **ArbrePosition**. 
Ce résultat est de la forme :

```txt
"<Nom de la racine>" + TAB + ["<Nom du 1er noeud>" + TAB + ...
["<Nom du 2ème noeud>" + TAB + [...]]]"<Nom de la feuille>"
```


**`<Champ Arbre> : (Nom de champ)`**

Nom du champ Arbre à manipuler.

**`<Chemin du noeud> : (Chaîne de caractères)`**

Chemin complet du noeud à afficher en premier dans le champ Arbre. 
Ce paramètre est de la forme :

```txt
"<Nom de la racine>" + TAB + ["<Nom du 1er noeud>" + TAB + ...
["<Nom du 2ème noeud>" + TAB + [...]]]"<Nom de la feuille>"
```
En cas de doublons sur un élément du chemin, ce chemin peut contenir l'identifiant de l'élément (indiqué à la suite du nom grâce à la fonction [ArbreID](../WDLang1/3018027.md)).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Afficher un élément au début du champ Arbre (syntaxe 2)
<a name="afficher_element_debut_champ_arbre_syntaxe_2_ELTPARAGRAPHE000109"></a>L'élément affiché en premier peut être différent de celui demandé. En effet, s'il n'y a pas assez d'éléments affichés en dessous de l'élément &lt;Chemin du noeud&gt;, cet élément ne pourra pas être affiché en début du champ Arbre.

Par exemple, le noeud "Entrées froides" ne peut pas être affiché en début du champ Arbre : il n'y a pas assez d'éléments affichés en dessous de ce noeud.<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ArbrePosition.gif)

<a name="NOTE0_2"></a>


### Sélectionner l'élément de la ligne &lt;Indice&gt;
<a name="selectionner_element_ligne_ELTPARAGRAPHE000121"></a>Pour sélectionner l'élément de la ligne &lt;Indice&gt;, utilisez :

- soit l'affectation directe : 
	
	```txt
	<Champ Arbre> = Indice
	```
Si la ligne est affichée dans le champ Arbre, l'élément correspondant est sélectionné.

- soit la fonction [ArbreSelectPlus](../WDLang1/3018002.md) :
	
	```txt
	ArbreSelectPlus(<Champ Arbre>, <Chemin complet de l'élément>)
	```

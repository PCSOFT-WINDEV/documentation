## Qu'est-ce que le "Drag and Drop" ?
<a name="questce_que_drag_and_drop_ELTTEXTE000130"></a>
Le "Drag and Drop" (ou "Glisser/Déplacer") permet de transférer des données entre plusieurs champs grâce à la souris. Ces données peuvent être déplacées entre plusieurs champs d'une même application ou d'applications différentes.

Il faut distinguer :

- La source, objet contenant les données à copier ou à couper.

- La cible, objet dans lequel les données doivent être copiées.




Le "Drag and Drop" consiste à :

1. Sélectionner les données de la source avec la souris.

2. Cliquer sur le bouton gauche de la souris et le maintenir enfoncé pendant le déplacement vers la destination.

3. Relâcher le bouton de la souris pour valider le transfert des données.




Si les données doivent être copiées, la touche Ctrl doit être maintenue enfoncée pendant l'opération.

## WINDEV et le "Drag and Drop"
<a name="windev_drag_and_drop_ELTTEXTE000154"></a>
WINDEV permet plusieurs types de "Drag and Drop" dans vos applications :

- ["Drag and Drop" automatique](../WDLang1/3030012.md) dans les champs de saisie, les listes (y compris liste image), les arbres, les tables et les tables hiérarchiques.

- ["Drag and Drop" programmé](../WDLang1/3030013.md) dans de nombreux types de champs.

- ["Drag and Drop" programmé entre l'explorateur Windows](../WDLang1/3030014.md) et une application WINDEV.




![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) En Java, seul le Drag and Drop automatique est disponible.


### Cas particulier : Connecteur Natif SQL Server et Drag and Drop
<a name="cas_particulier_connecteur_natif_sql_server_drag_and_drop_ELTPARAGRAPHE000058"></a>Lors de l'utilisation du Connecteur SQL Server (via OLE DB ou via ODBC), le Drag and Drop peut ne pas fonctionner. 
Pour retrouver un fonctionnement correct, il est conseillé d'ajouter les lignes de code suivantes dans le code d'initialisation du projet : 

```wl
ChargeDLL("ole32")
API("ole32","CoInitializeEx",0,2)
```

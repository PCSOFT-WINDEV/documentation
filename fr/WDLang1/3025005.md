
## EnumèreChamp (Fonction)

***En anglais : EnumControl***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nom du Nème champ de l'élément considéré. Cet élément doit être affiché. 

**Remarque** : Dans la plupart des cas, il est conseillé d'utiliser la fonction [gpwEnumèreChamp](../WDLang6/1000020536.md) qui liste tous les champs directement.

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) L'élément considéré peut être une fenêtre, un état, un groupe de champs, un champ Onglet, un champ Zone répétée ou un superchamp. Les champs Barre d'outils et Boîte à outils ne sont pas gérés. 





Utilisée dans une boucle, cette fonction peut permettre d'énumérer les champs de l'élément considéré.

<a name="sample_code"></a>

## Exemple

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 
```wl
// Nom du champ 7 de la fenêtre "FEN_FenêtreSaisie"
ResChamp = EnumèreChamp(FEN_FenêtreSaisie, 7)
```

```wl
	ListeAjoute(COMBO_ListeChamp, ResChamp)

```

## Syntaxe

`<Résultat> = EnumèreChamp(<Objet "parent"> , <Numéro du champ> [, <Type de l'énumération>])`
---

**`<Résultat> : (Chaîne de caractères)`**



- Nom du champ trouvé, 

- Chaîne vide ("") si aucun champ n'a été trouvé.




**`<Objet "parent"> : (Chaîne de caractères)`**

Nom de l'objet "parent" (contenant le champ). Ce paramètre correspond au nom d'une fenêtre, d'un champ Zone répétée, d'une rupture d'un champ Zone répétée, d'un groupe, d'un champ Onglet, d'un superchamp ou d'un état.

Si l'objet "parent" est un champ Onglet, la notation suivante doit être utilisée : &lt;Nom du champ Onglet&gt;[&lt;Numéro du volet d'onglet&gt;]. Par exemple : ONG_Onglet1[2].



**`<Numéro du champ> : (Entier)`**

Numéro du champ dont le nom est recherché. Chaque champ est associé à un numéro. Ce numéro correspond :

- soit à l'ordre de création des champs (par défaut) si le type d'énumération est effectué avec la constante *parOrdreDeCréation*

- soit à l'ordre de navigation (c'est-à-dire l'ordre de prise de focus des champs dans l'objet "parent") si le type d'énumération est effectué avec la constante *parOrdreDeTabulation*. 
	La touche F5 permet de visualiser l'ordre de navigation sous l'éditeur de fenêtres / de pages.




**`<Type de l'énumération> : (Constante optionnelle de type Entier)`**

Ordre dans lequel l'énumération est réalisée :



|   |   |
| --- | --- |
| *parAltitude* | Énumération effectuée selon l'altitude des champs, du plus bas au plus haut. |
| *parOrdreDeCréation*<br>(Valeur par défaut) | Énumération effectuée selon l'ordre de création des champs (sous l'éditeur ou avec la fonction [ChampClone](../WDLang1/3025017.md)). |
| *parOrdreDeTabulation* | Énumération effectuée selon l'ordre de navigation avec la touche de tabulation (Tab). Les champs qui ne sont pas accessibles par la touche de tabulation (Tab) sont listés à la fin. |

## Remarques




### Appartenance des champs
<a name="appartenance_des_champs_ELTPARAGRAPHE000157"></a>Seuls les champs appartenant directement à l'objet "parent" spécifié sont énumérés.

Par exemple, si le paramètre &lt;Objet "parent"&gt; correspond au nom d'une fenêtre (ou d'une page), la fonction **EnumèreChamp** fonctionne uniquement pour les champs appartenant directement à cette fenêtre (à cette page). La fonction **EnumèreChamp** ne permet pas de récupérer les noms des champs appartenant aux champs Onglet ou aux superchamps de cette fenêtre (de cette page).








### Utilisation des champs récupérés
<a name="utilisation_des_champs_recuperes_ELTPARAGRAPHE000173"></a>Lorsque le nom d'un champ obtenu par la fonction **EnumèreChamp** est contenu dans une variable, le champ lui-même peut être utilisé grâce aux [opérateurs d'indirection (les accolades { et })](../Motscles/1512005.md). Exemple :


```wl
// Nom du champ 7 de la page
NomChamp est une chaîne = EnumèreChamp(MaPage, 7)
// Libellé du champ
ToastAffiche({NomChamp}..Libellé)
```





![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) 

### Cas des champs Onglet avec volets dynamiques
<a name="cas_des_champs_onglet_avec_volets_dynamiques_ELTPARAGRAPHE000199"></a>Si le paramètre &lt;Objet "parent"&gt; correspond au nom d'un champ Onglet avec des volets dynamiques, les alias des volets dynamiques sont énumérés. 
<a name="NOTE0_2"></a>




### Erreur du WLangage
<a name="erreur_wlangage_ELTPARAGRAPHE000207"></a>Une erreur du WLangage est générée si :

- Le paramètre &lt;Numéro du champ&gt; est négatif.

- Le paramètre &lt;Numéro du champ&gt; est plus grand que le nombre de champs contenu dans l'&lt;Objet "parent"&gt; +1. Cela permet d'éviter les boucles sans fin.

- L'&lt;Objet "parent"&gt; n'est ni une fenêtre, ni une page, ni un état, ni un groupe, ni un champ Onglet, ni un superchamp.




## EnModeTSE (Fonction)

***En anglais : InTSEMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie si le code est exécuté :

- depuis un poste en TSE ou depuis un bureau à distance.

- depuis un poste directement.  




**Rappel** : Le TSE (Terminal Server Emulation) permet de brancher des terminaux sur un serveur, c'est-à-dire d'utiliser des clients légers pour accéder à une session ouverte sur le serveur.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code exécuté depuis TSE ? 
...
SI EnModeTSE() = Vrai ALORS
	// Procédure exécutée depuis un TSE
	...
SINON
	// Cas normal
	// Traitement à effectuer
	...
FIN
```
<a name="Exemple2"></a>

```wl
// Code exécuté depuis TSE : désactivation des effets non supportés 
...
SI EnModeTSE() = Vrai ALORS
	FenDésactiveEffet(effetAnimation + effetCadreTranslucide + ...
		effetGFI + effetHalo)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = EnModeTSE()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté depuis une application exécutée sur un poste en TSE ou par un bureau à distance, 

- <u><u><u><u>Faux</u></u></u></u> dans les autres cas.


![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Correspond toujours à <u><u><u><u>Faux</u></u></u></u>.   




<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000238"></a>La fonction **EnModeTSE** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeTSE** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.


## fRepCache (Fonction)

***En anglais : fCacheDir***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le chemin du répertoire à utiliser pour stocker les données en cache de l'application sur l'espace de stockage interne de l'application.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Sauvegarde d'un fichier en cache sur l'espace de stockage interne
sRepCache est une chaîne = fRepCache()
SI sRepCache <> "" ALORS
	fSauveTexte(sRepCache + "fichier.txt", MaChaîne)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = fRepCache()`
---

**`<Résultat> : (Chaîne de caractères)`**

Chemin du répertoire cache de l'application sur l'espace de stockage interne de l'application.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Gestion du répertoire cache interne de l'application par le système
<a name="gestion_repertoire_cache_interne_application_par_systeme_ELTPARAGRAPHE000032"></a>Les données contenues dans ce répertoire seront automatiquement supprimées par le système : 

- si le système nécessite de l'espace disque supplémentaire. 

- si l'utilisateur demande à supprimer les données en cache de l'application.

- si l'application est désinstallée.



<a name="NOTE0_2"></a>


### Conseils d'utilisation du répertoire cache interne de l'application
<a name="conseils_utilisation_repertoire_cache_interne_application_ELTPARAGRAPHE000043"></a>

- Il est recommandé de limiter le volume des données stockées en cache et de supprimer ces données dès qu'elles ne sont plus utilisées.

- Les données stockées dans le répertoire cache de l'application sur l'espace de stockage interne ne sont pas accessibles par les autres applications.

- La fonction [SysCacheStockageExterne](../WDLang1/1000021815.md) permet de connaître le chemin du répertoire cache de l'application sur la mémoire externe de l'appareil.


## DiagrammeDégroupe (Fonction)

***En anglais : DiagramUngroup***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Dissocie les différentes formes d'un groupe dans un champ Editeur de diagrammes. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Si la sélection correspond à un groupe, dégroupe les formes de ce groupe
SI DiagrammeDégroupe(EDIAG_MonDiagramme) ALORS
	ToastAffiche("Groupement dissocié - CTRL+Z pour annuler")
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = DiagrammeDégroupe(<Diagramme> [, <Groupe>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si les éléments du groupe ont été dissociés, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de détails sur l'erreur.




**`<Diagramme> : (Nom de champ)`**

Nom du champ Editeur de diagrammes à manipuler.  

**`<Groupe> : (Variable de type diagGroupe optionnelle)`**

Nom de la variable de type [diagGroupe](../WDLang1/1410088127.md) à manipuler. Si ce paramètre n'est pas précisée, le groupe sélectionné dans le champ Editeur de diagrammes est manipulé.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si le paramètre &lt;Groupe&gt; n'est pas une variable de type [diagGroupe](../WDLang1/1410088127.md) ou si la sélection ne correspond pas à un groupe, la fonction **DiagrammeDégroupe** n'a aucun effet et renvoie <u><u><u><u>Faux</u></u></u></u>. 

- La dissociation d'un groupe est annulable par l’utilisateur final (via le "undo") si le diagramme est en cours d’édition.

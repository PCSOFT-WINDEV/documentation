


## &lt;Variable picCalque&gt;.ChangeMode (Fonction)

***En anglais : &lt;picLayer variable&gt;.ChangeMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Change le mode de dessin sur une variable de type [picCalque](../WDLang1/1000024604.md). Ce mode de dessin est utilisé par les fonctions de dessin (fonctions [&lt;Variable picCalque&gt;.DessineLigne](../WDLang1/1000024495.md), [&lt;Variable picCalque&gt;.DessineRectangle](../WDLang1/1000024564.md), ...).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MonImage est un WDPic = "Test.bmp"
MonCalque est un picCalque

// Dessine un rectangle dans un calque puis l'efface
MonCalque.ChangeMode(dessinXOR)
// Affiche le rectangle (en mode XOR)
MonCalque.DessineRectangle(0, 0, 100, 100)
Multitâche(10)
// Efface le rectangle
MonCalque.DessineRectangle(0, 0, 100, 100)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Image picCalque>.ChangeMode(<Nouveau mode>)`
---

**`<Image picCalque> : (Variable de type picCalque)`**

Nom de la variable de type [picCalque](../WDLang1/1000024604.md) à manipuler.

**`<Nouveau mode> : (Constante de type entier)`**

Mode de dessin dans l'image :



|   |   |
| --- | --- |
| *dessinAntiAliasing* | Utilise le GDI+ et/ou les fonctionnalités de la carte graphique pour dessiner les lignes et les cercles sans crénelage. Ce mode de dessin est plus lent mais le résultat est meilleur qu'avec le mode standard.<br> |
| *dessinNormal* | Couleurs normales. |
| *dessinXOR* | Couleurs appliquées via une opération de "ou exclusif" avec le fond. Permet d'effacer un dessin si cette option est utilisée 2 fois.<br>ATTENTION : Ce mode de dessin n'est pas compatible avec la gestion de l'opacité (le dessin ne s'efface plus).<br>**Attention** : <br><br>- Le dessin en mode XOR est disponible uniquement sur les champs Image. Le dessin en mode XOR n'est pas disponible sur les variables de type [Image](../WDLang1/1000019650.md). <br><br>- La constante *dessinXOR* n'est pas combinable avec la constante *dessinAntiAliasing*. <br><br><br> |

### Anti-aliasing
<a name="antialiasing_ELTPARAGRAPHE000422"></a>L'anti-aliasing consiste à dessiner les traits et les cercles en évitant l'effet de crénelage. Cette fonctionnalité nécessite le framework GDI + (c'est-à-dire le framework de WINDEV).
Exemple :<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=dchangemode.gif)
Limites :

- La fonction [&lt;Variable picCalque&gt;.DessineCorde](../WDLang1/1000024469.md) n'est pas supportée dans ce mode.

- Pour éviter l'effet de crénelage, les couleurs et les traits des éléments sont lissés. Certaines fonctions de dessin peuvent avoir un comportement inattendu car les couleurs spécifiées ne sont plus retrouvées. Ainsi, si vous spécifiez une couleur de bordure pour la fonction [&lt;Variable picCalque&gt;.Remplissage](../WDLang1/1000024588.md), cette couleur n'est pas retrouvée : le remplissage s'effectue sur toute la zone.

- Dans le cas d'un dessin avec anti-aliasing, les fonctions [&lt;Variable picCalque&gt;.DessinePolygone](../WDLang1/1000024504.md) et [&lt;Variable picCalque&gt;.DessinePortion](../WDLang1/1000024561.md) ont le comportement suivant. Le contour est dessiné par-dessus l'intérieur du polygone : le trait est donc en partie combiné avec l'intérieur du polygone. Si l'épaisseur du trait est importante, le trait s'affichera en deux couleurs.

- Pour plus de détails sur les conditions d'utilisation du GDI+, consultez [Framework GDI+](../Editeurs/2020007.md).

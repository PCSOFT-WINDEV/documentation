
## Arrondi (Fonction)

***En anglais : Round***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie la valeur arrondie d'une valeur numérique selon le nombre de décimales précisé.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupération de la valeur arrondie de -16.238167
ResArrondi = Arrondi(-16.238167, 2)
// Renvoie -16.24
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = Arrondi(<Valeur numérique> [, <Nombre de décimales>])`
---

**`<Résultat> : (Entier ou réel)`**

Valeur arrondie de la valeur numérique spécifiée.

**`<Valeur numérique> : (Réel ou monétaire)`**

Valeur numérique à arrondir. Le type monétaire est conseillé pour obtenir des résultats fiables.

**`<Nombre de décimales> : (Entier optionnel)`**

Nombre de décimales à retourner. Si ce paramètre n'est pas précisé, aucune décimale n'est retournée.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Valeur numérique
<a name="valeur_numerique_ELTPARAGRAPHE000077"></a>Si &lt;Valeur numérique&gt; est comprise entre :

- 0 et 0.49 : le nombre est arrondi à 0. Par exemple, 3.2 est arrondi à 3 (aucune décimale).

- 0.5 et 0.99 : le nombre est arrondi à 1. Par exemple, 3.7 est arrondi à 4 (aucune décimale).




Attention : Les arrondis avec le type "Réel" ne sont pas précis. En effet,  les opérations avec le type "réel" ne sont pas précises du fait de la représentation informatique des réels. Pour obtenir un arrondi exact, il est conseillé d'utiliser le type [Monétaire](../Motscles/1514043.md) ou [Numérique](../Motscles/1514072.md) qui utilise une représentation mémoire exacte. Pour plus de détails, consultez le type [réel](../Motscles/1514048.md).
<a name="NOTE0_2"></a>




### Séparateur décimal
<a name="separateur_decimal_ELTPARAGRAPHE000099"></a>Le séparateur décimal des nombres réels est "." (point).

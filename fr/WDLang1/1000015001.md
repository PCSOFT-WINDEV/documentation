


## XamlEvénement (Fonction)

***En anglais : XamlEvent***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Associe une procédure écrite en WLangage à un événement d'un champ Xaml.

Après l'exécution de la fonction **XamlEvénement**, la procédure sera lancée à chaque fois que l'événement choisi sera déclenché.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// -- Initialisation du champ Xaml DOTNET_Xaml_OK
MaChaîne est une chaîne = [
<Button Name="Bouton1" Width="20">OK</Button>
]
DOTNET_Xaml_OK = MaChaîne
XamlEvénement(DOTNET_Xaml_OK, "Bouton1", "Click", MaProcédureSurClic)
```

```wl
// -- Procédure MaProcédureSurClic
PROCEDURE MaProcédureSurClic
Info("Clic sur le bouton OK")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`XamlEvénement(<Champ Xaml> [, <Nom du contrôle .Net>] , <Nom de l'événement> , <Procédure>)`
---

**`<Champ Xaml> : (Nom de champ)`**

Nom du champ Xaml.

**`<Nom du contrôle .Net> : (Chaîne de caractères optionnelle)`**

Nom du contrôle .Net à associer à la procédure &lt;Procédure&gt;. Si ce paramètre n'est pas spécifié, la procédure &lt;Procédure&gt; sera associée uniquement au champ Xaml indiqué.

Ce nom est sensible à la casse.

**`<Nom de l'événement> : (Chaîne de caractères)`**

Nom de l'événement associé au champ Xaml et/ou au contrôle .Net déclenchant la procédure.

**`<Procédure> : (Chaîne de caractères)`**

Nom de la procédure WLangage appelée lors du déclenchement de l'événement. 
Cette procédure doit comporter le nombre de paramètres fournis par l'événement et peut éventuellement renvoyer un résultat (se référer à la documentation du contrôle .Net).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Il est conseillé de placer l'appel à la fonction **XamlEvénement** dans le code d'initialisation du champ Xaml.

- **Important** : PC SOFT n'assure aucun support sur le détail du fonctionnement des contrôles .Net.
	Vous pouvez trouver les différents événements générés par les contrôles .Net dans leur documentation.

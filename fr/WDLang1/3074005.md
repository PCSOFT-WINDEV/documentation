
## FichierVersTableMémoire (Fonction)

***En anglais : FileToMemoryTable***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Remplit un champ de type "Table par programmation" (également appelé "Table mémoire") avec tous les enregistrements d'un fichier de données, d'une vue HFSQL ou d'une requête (requête créée avec l'éditeur de requêtes ou avec la fonction [HExécuteRequêteSQL](../WDLang4/3044084.md)).

Le champ Table par programmation doit contenir les différentes colonnes à associer aux rubriques. La fonction **FichierVersTableMémoire** va : 

- vider le champ Table. 

- associer les colonnes du champ Table aux rubriques du fichier de données, d'une vue HFSQL ou d'une requête. Les rubriques vont être associées selon leur ordre physique défini dans le fichier. 

- remplir le champ Table avec le contenu du fichier de données ou de la requête.




**ATTENTION** : Cette opération peut être très longue et saturer la mémoire si le fichier de données est volumineux.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Remplir le champ TABLE_MaTable avec le contenu de REQ_Requête1
FichierVersTableMémoire(TABLE_MaTable, REQ_Requête1)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`FichierVersTableMémoire(<Champ Table> , <Fichier de données>)`
---

**`<Champ Table> : (Nom de champ)`**

Nom du champ Table par programmation à remplir.

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données, de la vue HFSQL ou de la requête utilisé.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Les différents types de requêtes
<a name="les_differents_types_requetes_ELTPARAGRAPHE000059"></a>La fonction **FichierVersTableMémoire** peut être utilisée avec les requêtes créées sous l'éditeur de requêtes ou avec la fonction [HExécuteRequêteSQL](../WDLang4/3044084.md).

Pour associer une requête créée avec la fonction [SQLExec](../WDLang4/3072007.md) à un champ Table par programmation, utilisez la fonction [SQLTable](../WDLang4/3072031.md).


<a name="NOTE0_2"></a>




### Parcours du fichier de données
<a name="parcours_fichier_donnees_ELTPARAGRAPHE000082"></a>Le parcours du fichier de données se fait sur le contenu de la propriété [RubriqueParcourue](../Proprietes/2510111.md).

Pour que la fonction **FichierVersTableMémoire** prenne en compte le filtre appliqué au fichier de données, il faut :

1. Modifier la propriété [RubriqueParcourue](../Proprietes/2510111.md) en lui affectant la rubrique renvoyée par la fonction [HFiltre](../WDLang4/3044100.md).

2. Utiliser la fonction **FichierVersTableMémoire**.





<a name="NOTE0_3"></a>




### Fonction FichierVersTableMémoire et propriété Nouveau
<a name="fonction_docparampagetitleshort_propriete_nouveau_ELTPARAGRAPHE000111"></a>La fonction **FichierVersTableMémoire** affecte la propriété [Nouveau](../Proprietes/2510073.md) à <u><u><u><u>Vrai</u></u></u></u>.

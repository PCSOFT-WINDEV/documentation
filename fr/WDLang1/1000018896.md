
## HaspHLLit (Fonction)

***En anglais : HaspHLRead***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Lit des données (chaînes de caractères ou buffers) dans la mémoire interne de la clé Hasp.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupère la clé initiale d'activation dans la mémoire de la clé Hasp
CléInitiale est une chaîne 
CléInitiale = HaspHLLit(SAI_ID_Session, haspFichierPrincipal, 0, 30)

// Compare la clé initiale à la clé d'activation reçue
CléCompareClé(CléInitiale, CléActivation)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = HaspHLLit(<Identifiant de la session> , <Identifiant du fichier> , <Position> , <Taille>)`
---

**`<Résultat> : (Chaîne de caractères)`**

Informations lues dans la mémoire de la clé Hasp.

**`<Identifiant de la session> : (Entier)`**

Identifiant de la session Hasp renvoyé par la fonction [HaspHLConnecte](../WDLang1/1000018891.md).

**`<Identifiant du fichier> : (Entier ou constante de type Entier)`**

Identifiant du fichier dans lequel les données doivent être lues. Cet identifiant peut correspondre :

- à un entier. 

- à une des constantes suivantes :
	


|   |   |
| --- | --- |
| *haspFichierLicence* | Lit dans le fichier "zone de licence" de la clé. |
| *haspFichierPrincipal* | Lit dans le fichier "zone de mémoire principale" de la clé. |






**`<Position> : (Entier)`**

Position en octets à laquelle les données seront lues dans le fichier.

**Attention** : Le début du fichier commence à la position 0.

**`<Taille> : (Entier)`**

Taille en octets des données à lire.


## dbgConnecte (Fonction)

***En anglais : dbgConnect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de connecter un débogueur à distance sur une application, un service, un Webservice, ...
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Lance le débogueur à distance
dbgConnecte("1.1.1.1")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = dbgConnecte(<Débogueur>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la connexion a réussi, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 




**`<Débogueur> : (Chaîne de caractères)`**

Nom ou adresse IP de la machine de développement.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le débogueur doit être en attente de connexion avant que l'application à déboguer n'appelle la fonction **dbgConnecte**. Pour passer le débogueur en attente, utilisez au choix : 

	- l'option "Attendre une connexion par dbgConnecte" dans le menu du bouton GO de projet.

	- sous le volet "Projet", dans le groupe "Mode test", déroulez "Mode test" et sélectionnez l'option "Attendre une connexion par dbgConnecte".




- La communication est effectuée via le port 27280 : ce port doit donc être accessible depuis la machine distante.

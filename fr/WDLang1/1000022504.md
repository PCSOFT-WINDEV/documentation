


## &lt;Tableau de bord&gt;.ChangeTailleWidget (Fonction)

***En anglais : &lt;Dashboard&gt;.ResizeWidget***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Change les dimensions d'un Widget présent dans un champ Tableau de bord. Ce Widget est déjà configuré ou affiché. 

Rappel : Un Widget correspond à : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) une fenêtre interne.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Passe le Widget numéro 1 sur deux colonnes afin d'afficher les détails
TDB_MonTableauDeBord.ChangeTailleWidget(1, 2, 1)
LIB_Détails.Visible = Vrai
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE2"></a>

### Changer la taille d'un Widget identifié par son indice

`<Champ Tableau de bord>.ChangeTailleWidget(<Indice du Widget> , <Largeur> , <Hauteur> [, <Options>])`
---

**`<Champ Tableau de bord> : (Nom de champ)`**

Nom du champ Tableau de bord à manipuler.

**`<Indice du Widget> : (Entier)`**

Indice du Widget à manipuler. Cet indice correspond :

- au numéro d'ajout du Widget défini sous l'éditeur de fenêtres ou de pages.

- au numéro d'ajout du Widget renvoyé par la fonction [&lt;Tableau de bord&gt;.AjouteWidget](../WDLang1/1000022502.md).




**`<Largeur> : (Entier)`**

Largeur du Widget en nombre de cases ou en pixels. La taille minimale correspond à une case. La valeur en pixels est arrondie pour correspondre à un nombre entier de cases.

**`<Hauteur> : (Entier)`**

Hauteur du Widget en nombre de cases ou en pixels. La taille minimale correspond à  une case. La valeur en pixels est arrondie pour correspondre à un nombre entier de cases.

**`<Options> : (Constante optionnelle de type Entier)`**

Permet de définir l'unité utilisée et le mode de réorganisation : 


|   |   |
| --- | --- |
| *tdbSansRéorganisation* | Change la taille du Widget sans réorganiser les Widgets adjacents.<br><br>Attention : L'utilisation de la constante *tdbSansRéorganisation* peut conduire à un affichage incorrect (avec des Widgets superposés). En utilisant cette constante, il est important de réorganiser entièrement le champ Tableau de bord à l'aide des fonctions **&lt;Tableau de bord&gt;.ChangeTailleWidget** et [&lt;Tableau de bord&gt;.DéplaceWidget](../WDLang1/1000022527.md). |
| *tdbUnitéCase*<br>(Valeur par défaut) | L'unité utilisée pour la taille du Widget est un nombre de cases. |
| *tdbUnitéPixel* | L'unité utilisée pour la taille du Widget est le pixel. |

<a name="NOTE0_1"></a>

## Remarques


### Règle utilisée pour le calcul des cases
<a name="regle_utilisee_pour_calcul_des_cases_ELTPARAGRAPHE000232"></a>Pour un Widget de taille 100 px par 100 px, avec une marge de 10 px, voici quelques exemples de redimensionnement : 


```wl
TDB_FuseauxHoraires.ChangeTailleWidget(1, 80, 80, tdbUnitéPixel)
// le widget sera de 1 x 1

TDB_FuseauxHoraires.ChangeTailleWidget(1, 200, 200, tdbUnitéPixel)
// le widget sera de 2 x 2

TDB_FuseauxHoraires.ChangeTailleWidget(1, 201, 201, tdbUnitéPixel)
// le widget sera de 3 x 3

TDB_FuseauxHoraires.ChangeTailleWidget(1, 400, 500, tdbUnitéPixel)
// le widget sera de 4 x 5
```

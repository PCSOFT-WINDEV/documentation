


## Fonctions de gestion des événements Windows
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


Les fonctions de gestion des événements sont les suivantes : 



|   |   |
| --- | --- |
| [Evénement](../WDLang1/3015011.md) | Intercepte un événement Windows sur un champ, un groupe de champs ou une fenêtre WINDEV. |
| [EvénementExiste](../WDLang1/1000024424.md) | Indique si une procédure spécifique est exécutée lorsqu'un événement Windows est envoyé à un champ ou une fenêtre. |
| [FinEvénement](../WDLang1/3015010.md) | Annule l'interception d'un événement Windows sur un champ ou une fenêtre WINDEV (mis en place grâce à la fonction [Evénement](../WDLang1/3015011.md)). |
| [FinTimer](../WDLang1/3015007.md) | Termine l'exécution d'un timer déclenché par la fonction [Timer](../WDLang1/3015008.md). |
| [FinTimerSys](../WDLang1/3015009.md) | Termine l'exécution d'un timer déclenché par la fonction [TimerSys](../WDLang1/3015006.md). |
| [Handle](../WDLang1/3015005.md) | Renvoie le "Handle" système (HWND) d'un champ ou d'une fenêtre WINDEV. |
| [HandleParentSuivant](../WDLang1/1000017312.md) | Force le parent de la prochaine fenêtre à ouvrir. |
| [Multitâche](../WDLang1/3015004.md) | La fonction Multitâche permet de : <br><br>- effectuer une temporisation,<br><br>- rendre la main à Windows, <br><br>- rendre la main à Windows et au WLangage.<br><br><br> |
| [PostMessage](../WDLang1/3015001.md) | Envoie un message Windows à un champ ou à une fenêtre. |
| [SendMessage](../WDLang1/3015003.md) | Envoie un message Windows à un champ ou à une fenêtre. |
| [Temporisation](../WDLang1/3015012.md) | Stoppe temporairement l'exécution du programme. |
| [Timer](../WDLang1/3015008.md) | Appelle périodiquement et automatiquement une procédure du WLangage. |
| [TimerSys](../WDLang1/3015006.md) | Appelle périodiquement et automatiquement une procédure du WLangage. |

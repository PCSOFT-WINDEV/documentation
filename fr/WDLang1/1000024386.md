


## Fonctions de gestion des threads (syntaxe préfixée)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Les fonctions de gestion des threads sont les suivantes : 



|   |   |
| --- | --- |
| [&lt;Thread&gt;.DemandeArrêt](../WDLang1/1000024924.md) | Envoie une demande d'arrêt à un thread. |
| [&lt;Thread&gt;.EnvoieSignal](../WDLang1/1000024926.md) | Le thread en cours envoie un signal au thread spécifié pour le débloquer. |
| [&lt;Variable SignalAutomatique&gt;.Attend](../WDLang1/1000024380.md) | Bloque le thread en cours en attendant que le signal spécifié soit ouvert. |
| [&lt;Variable SignalAutomatique&gt;.Ferme](../WDLang1/1000024383.md) | Ferme un signal de synchronisation entre plusieurs threads. |
| [&lt;Variable SignalAutomatique&gt;.Ouvre](../WDLang1/1000024385.md) | Ouvre un signal de synchronisation entre plusieurs threads. |
| [&lt;Variable SignalManuel&gt;.Attend](../WDLang1/1000024381.md) | Bloque le thread en cours en attendant que le signal spécifié soit ouvert. |
| [&lt;Variable SignalManuel&gt;.Ferme](../WDLang1/1000024382.md) | Ferme un signal de synchronisation entre plusieurs threads. |
| [&lt;Variable SignalManuel&gt;.Ouvre](../WDLang1/1000024384.md) | Ouvre un signal de synchronisation entre plusieurs threads. |
| [&lt;Variable Thread&gt;.Attend](../WDLang1/1000024880.md) | Attend la fin de l'exécution du thread désigné. |





Les types de variables suivants sont disponibles pour gérer les signaux : 



|   |   |
| --- | --- |
| [SignalAutomatique](../WDLang1/1000021270.md) | Le type **SignalAutomatique** permet de gérer un signal automatique. |
| [SignalManuel](../WDLang1/1000021275.md) | Le type **SignalManuel** permet de gérer un signal manuel. |




## ArbreSelectMoins (Fonction)

***En anglais : TreeSelectMinus***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Dé-sélectionne un élément dans un champ Arbre. 

Exemple de champ Arbre : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Arbre_Vocabulaire.gif)

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Désélectionne le noeud "Desserts" dans le champ Arbre "ARBRE_RecetteTV"
Res = ArbreSelectMoins(ARBRE_RecetteTV, "Recettes" + TAB + "Desserts")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = ArbreSelectMoins(<Champ Arbre> , <Chemin de l'élément>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'élément a été désélectionné, 

- <u><u><u><u>Faux</u></u></u></u> si l'élément spécifié n'existe pas.




**`<Champ Arbre> : (Nom de champ)`**

Nom du champ Arbre à manipuler.

**`<Chemin de l'élément> : (Chaîne de caractères optionnelle)`**

Chemin complet de l'élément à manipuler. Ce paramètre est de la forme : 

```txt
"<Nom de la racine>" + TAB + ["<Nom du 1er noeud>" + TAB + ...
["<Nom du 2ème noeud>" + TAB + [...]]]"<Nom de la feuille>"
```
En cas de doublons sur un élément du chemin, ce chemin peut contenir l'identifiant de l'élément (indiqué à la suite du nom grâce à la fonction [ArbreID](../WDLang1/3018027.md)). 

Si ce paramètre est spécifié, l'élément spécifié est désélectionné. 

Si ce paramètre n'est pas spécifié, l'élément en cours est désélectionné. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Arbre multisélection
<a name="arbre_multiselection_ELTPARAGRAPHE000072"></a>Dans le cas d'un champ multisélection : 

- si le paramètre &lt;Chemin de l'élément&gt; est spécifié, seul l'élément spécifié est désélectionné. 

- si le paramètre &lt;Chemin de l'élément&gt; n'est pas spécifié, tous les éléments sélectionnés sont désélectionnés.

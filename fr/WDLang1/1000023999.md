


## &lt;Image&gt;.Fond (Fonction)

***En anglais : &lt;Image&gt;.Background***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Déclare la couleur de fond par défaut des rectangles, des cercles (et autres) utilisés dans un champ Image.

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les dessins](../WDLang1/3029035.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Déclaration de la couleur de fond des différents dessins
// réalisés dans le champ Image IMG_Dessin
IMG_Dessin.Fond(VertClair)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Champ Image>.Fond(<Couleur> [, <Style de remplissage> [, <Type de hachures> [, <Opacité>]]])`
---

**`<Champ Image> : (Nom de champ)`**

Nom du champ Image à manipuler.

**`<Couleur> : (Entier ou constante)`**

Couleur du fond par défaut des différents dessins. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md).

- une variable de type [Couleur](../WDLang1/1000019661.md). Dans ce cas, l'opacité spécifiée dans la variable est prise en compte.
	




**`<Style de remplissage> : (Entier ou constante de type Entier optionnel)`**

Style de remplissage du fond par défaut des différents dessins :


|   |   |
| --- | --- |
| *FondHachuré* | Remplissage du fond par des hachures. <br>Correspond à la valeur 2 pour compatibilité. |
| *FondPlein*<br>(Valeur par défaut) | Fond plein. <br>Correspond à la valeur 0 pour compatibilité. |
| *FondTransparent* | Fond transparent (le paramètre &lt;Couleur&gt; est alors ignoré).<br>Correspond à la valeur 1 pour compatibilité. |


![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Cette valeur est ignorée. Dans cette version, il n'est pas possible de hachurer le fond d'un dessin.

**`<Type de hachures> : (Entier optionnel)`**

Type de hachures du fond utilisé par défaut dans les différents dessins. Pour utiliser ce paramètre, la valeur du paramètre &lt;Style de remplissage&gt; doit être égale à 2. Si la valeur du paramètre &lt;Style de remplissage&gt; est différente de 2, le paramètre &lt;Type de hachures&gt; est ignoré. Les différents types de hachures sont :

- 0 : Hachures horizontales

- 1 : Hachures verticales

- 2 : Hachures diagonales à 60%

- 3 : Hachures diagonales à 120%

- 4 (par défaut) : Hachures croisées

- 5 : Hachures diagonales croisées


![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Cette valeur est ignorée. Dans cette version, il n'est pas possible de hachurer le fond d'un dessin.

**`<Opacité> : (Entier optionnel)`**

Opacité de la couleur, comprise entre 0 (transparent) et 255 (opacité complète). Une opacité de 128 permet de mélanger 50% de la couleur avec la couleur existante dans le champ.

**Remarques** : 

- Cette opacité sera prise en compte si le dessin a été initialisé avec la constante *dAvecOpacité* (fonction [&lt;Image&gt;.DébutDessin](../WDLang1/1000024326.md)).

- Si le paramètre &lt;Couleur&gt; correspond à une variable de type [Couleur](../WDLang1/1000019661.md), ce paramètre ne doit pas être spécifié : l'opacité spécifiée dans la variable sera prise en compte.

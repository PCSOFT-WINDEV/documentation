


## &lt;Diagramme de Gantt&gt;.AjouteCongé (Fonction)

***En anglais : &lt;Gantt chart&gt;.AddHoliday***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Déclare une nouvelle période de congés pour une ligne dans une colonne d'un champ Table (ou Table hiérarchique) de type Diagramme de Gantt.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Congés pour la ligne 4, le 4 février 2022
COL_Gantt.AjouteCongé(4, "20220204", "20220204")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Colonne Gantt>.GanttAjouteCongé(<Ligne> , <Date de début> , <Date de fin>)`
---

**`<Colonne Gantt> : (Nom de champ)`**

Nom de la colonne (d'un champ Table ou Table hiérarchique) de type Diagramme de Gantt à manipuler.

**`<Ligne> : (Entier)`**

Numéro de la ligne où les congés doivent être déclarés. 
Remarque : La ligne associée à une tâche peut être connue par la propriété **Ligne** de la variable de type [TâcheGantt](../WDLang1/1000020881.md).

**`<Date de début> : (Chaîne de caractères ou variable de type Date ou DateHeure)`**

Date de début de la période de congés. Cette date est considérée comme étant incluse dans les congés.

**`<Date de fin> : (Chaîne de caractères ou variable de type Date ou DateHeure)`**

Date de fin de la période de congés. Cette date est considérée comme étant incluse dans les congés.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La nouvelle période de congés est considérée comme non ouvrable pour la ligne.

- La durée des tâches est ajustée selon la période de congés spécifiée.

- Par défaut, les jours non ouvrés sont samedi et dimanche.

- Les jours fériés définis avec la fonction [JourFériéAjoute](../WDLang1/1000017304.md) sont considérés comme des jours non ouvrés.

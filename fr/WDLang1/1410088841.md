


## &lt;Fenêtre&gt;.AgencementEnCours (Fonction)

***En anglais : &lt;Window&gt;.CurrentLayout***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie l'indice de l'agencement en cours dans la fenêtre ou dans la fenêtre interne spécifiée.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI FEN_Détail.AgencementEnCours() = 2 ALORS
	LIB_DETAIL = CLIENT.DETAIL
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE2"></a>

## Syntaxe

### Connaître l'agencement d'une fenêtre spécifique

`<Résultat> = FenAgencementEnCours(<Fenêtre manipulée>)`
---

**`<Résultat> : (Entier)`**



- Indice de l'agencement en cours.

- 0 si la fenêtre interne ne possède aucun agencement. 




**`<Fenêtre manipulée> : (Nom de fenêtre)`**

Nom de la fenêtre ou de la fenêtre interne manipulée. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Les agencements sont numérotés dans leur ordre de création en édition.

- Si le terminal change d'orientation ou si la fonction [&lt;Fenêtre&gt;.ChangeAgencement](../WDLang1/1410088842.md) est exécutée, l'agencement change.




## &lt;Colonne Table hiérarchique&gt;.ChercheFils (Fonction)

***En anglais : &lt;TreeView Table column&gt;.SearchChild***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Recherche une valeur dans une colonne d'un champ Table hiérarchique, dans une branche spécifiée.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Recherche l'élément "Chemise" dans la colonne "COL_Produit"
// Recherche générique dans les lignes de niveau 2
ResCherche = COL_Produit.ChercheFils("Chemise", Faux, 2)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Recherche en spécifiant l'indice de l'élément parent

`<Résultat> = <Colonne>.ChercheFils(<Élément recherché> [, <Type de recherche> [, <Indice de l'élément parent> [, <Ligne de départ>]]])`
---

**`<Résultat> : (Entier)`**



- Indice de l'élément trouvé, 

- -1 si aucun élément ne correspond à la recherche.




**`<Colonne> : (Nom de champ)`**

Nom de la colonne dans laquelle la recherche doit être effectuée.

**`<Élément recherché> : (Type de l'élément recherché)`**

Élément à rechercher dans la colonne spécifiée. Si l'élément recherché est de type :

- "Date" ou "Heure" : la recherche est effectuée sur la valeur retournée.

- "Monétaire + Euro" : la recherche est effectuée sur la devise mémorisée.




**`<Type de recherche> : (Booléen optionnel)`**



- <u><u><u><u>Vrai</u></u></u></u> (valeur par défaut) si la recherche doit être à l'identique, 

- <u><u><u><u>Faux</u></u></u></u> si la recherche doit être générique.




**`<Indice de l'élément parent> : (Entier optionnel)`**

Indice de la ligne parente dans la hiérarchie.

**`<Ligne de départ> : (Entier optionnel)`**

Numéro de la ligne de départ de la recherche dans la colonne. Si ce paramètre n'est pas spécifié, la recherche est effectuée dans toute la colonne. Ce paramètre permet de rechercher toutes les occurrences d'une chaîne dans une colonne.


<a name="SYNTAXE2"></a>

### Recherche en spécifiant l'élément parent

`<Résultat> = <Colonne>.ChercheFils(<Élément recherché> [, <Type de recherche> [, <Élément parent> [, <Ligne de départ>]]])`
---

**`<Résultat> : (Entier)`**



- Indice de l'élément trouvé, 

- -1 si aucun élément ne correspond à la recherche.




**`<Colonne> : (Nom de champ)`**

Nom de la colonne dans laquelle la recherche doit être effectuée.

**`<Élément recherché> : (Type de l'élément recherché)`**

Élément à rechercher dans la colonne spécifiée. Si l'élément recherché est de type :

- "Date" ou "Heure" : la recherche est effectuée sur la valeur retournée.

- "Monétaire + Euro" : la recherche est effectuée sur la devise mémorisée.




**`<Type de recherche> : (Booléen optionnel)`**



- <u><u><u><u>Vrai</u></u></u></u> (valeur par défaut) si la recherche doit être à l'identique, 

- <u><u><u><u>Faux</u></u></u></u> si la recherche doit être générique.




**`<Élément parent> : (Entier optionnel)`**

Contenu de la ligne parente dans la hiérarchie. Par défaut, ce paramètre correspond à NULL : permet une recherche parmi les racines.

**`<Ligne de départ> : (Entier optionnel)`**

Numéro de la ligne de départ de la recherche dans la colonne. Si ce paramètre n'est pas spécifié, la recherche est effectuée dans toute la colonne. Ce paramètre permet de rechercher toutes les occurrences d'une chaîne dans une colonne.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Type de recherche
<a name="type_recherche_ELTPARAGRAPHE000268"></a>

- ***Recherche à l'identique*** : Seuls les éléments ayant une valeur strictement égale à &lt;Élément recherché&gt; sont retournés par la fonction **&lt;Colonne Table hiérarchique&gt;.ChercheFils**. Par exemple :
	
	```wl
	// Retourner tous les "Dupond" de la colonne "COL_Nom"
	COL_Nom.ChercheFils("Dupond")
	```


- ***Recherche générique*** : Les éléments commençant par &lt;Élément recherché&gt; sont retournés par la fonction **&lt;Colonne Table hiérarchique&gt;.ChercheFils**. Par exemple :
	
	```wl
	// Retourner tous les noms commençant par "DUP"
	COL_Nom.ChercheFils("DUP", Faux)
	```




<a name="NOTE0_2"></a>


### Divers
<a name="divers_ELTPARAGRAPHE000299"></a>

- L'élément trouvé n'est pas sélectionné. La fonction [&lt;Table&gt;.Position](../WDLang1/1000024278.md) permet de se positionner sur la ligne et la fonction [&lt;Table&gt;.SelectPlus](../WDLang1/1000024287.md) permet de sélectionner la ligne.

- La recherche effectuée avec la fonction **&lt;Colonne Table hiérarchique&gt;.ChercheFils** n'est pas sensible à la casse (majuscules/minuscules).


## ChampPrécédent (Fonction)

***En anglais : ControlPrevious***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique le nom du champ en saisie avant le champ en cours.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code d'entrée SAI_VILLE
// si on vient du champ "SAI_CP"
SI ChampPrécédent() = SAI_CP ALORS
	// On rempli le nom en fonction du CP
	SAI_VILLE = CPVersVille(SAI_CP)
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ChampPrécédent()`
---

**`<Résultat> : (Chaîne de caractères)`**



- Nom du champ précédent en cours de saisie (dans la fenêtre en cours). 

- Chaîne vide ("") si le champ en cours est le premier champ de la fenêtre.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction [ChampEnCours](../WDLang1/3025019.md) permet de connaître le nom du champ en cours de saisie.

- Le nom du champ est toujours relatif à la fenêtre en cours.

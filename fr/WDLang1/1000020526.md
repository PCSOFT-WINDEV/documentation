
## dbgErreur (Fonction)

***En anglais : dbgError***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Provoque : 

- une assertion lors du test de l'élément ou du projet (mode Go). 

- une erreur en mode test automatique. 





<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`dbgErreur(<Commentaire 1> [, <Commentaire 2> [... [, <Commentaire N>]]])`
---

**`<Commentaire 1> : (Chaîne de caractères)`**

Texte à afficher dans la fenêtre d'assertion ou dans le compte-rendu du test unitaire. 

**`<Commentaire 2> : (Chaîne de caractères optionnelle)`**

Texte à afficher dans la fenêtre d'assertion ou dans le compte-rendu du test unitaire. 



**`<Commentaire N> : (Chaîne de caractères optionnelle)`**

Texte à afficher dans la fenêtre d'assertion ou dans le compte-rendu du test unitaire. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Lorsque la fonction **dbgErreur** est appelée :

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) en mode test automatique, une erreur est ajoutée dans le compte-rendu du test (équivalent de la fonction [TestEcritRésultat](../WDLang1/1000013001.md)).

	- en mode test manuel, une assertion est affichée (équivalent de la fonction [dbgAssertion](../WDLang1/3014022.md)).

	- en mode exécution, une assertion est affichée si les assertions ont été activées (équivalent de la fonction [dbgAssertion](../WDLang1/3014022.md)).

***En anglais : fCompare***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Compare bit à bit le contenu de deux fichiers.

## Syntaxe

`<Résultat> = fCompare(<Chemin du fichier 1> , <Chemin du fichier 2>)`
---

**`<Résultat> : (Entier)`**



- 0 si les deux fichiers sont identiques ou si les deux chemins de fichier sont identiques,

- 1 si les deux fichiers sont différents,

- un code d'erreur (valeur inférieure à 0) dans les autres cas.




**`<Chemin du fichier 1> : (Chaîne de caractères)`**

Nom et chemin complet (ou relatif) du premier fichier à comparer (260 caractères maximum). Un chemin de type UNC peut être utilisé.

![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Ce paramètre peut être au format Ansi ou Unicode.



**`<Chemin du fichier 2> : (Chaîne de caractères)`**

Nom et chemin complet (ou relatif) du second fichier à comparer (260 caractères maximum). Un chemin de type UNC peut être utilisé.
![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Ce paramètre peut être au format Ansi ou Unicode.

## Remarques


### Code d'erreur
<a name="code_erreur_ELTPARAGRAPHE000126"></a>Les codes d'erreur retournés sont les suivants :

- -1 : Le &lt;Chemin du fichier 1&gt; n'a pas été trouvé.

- -2 : Le &lt;Chemin du fichier 2&gt; n'a pas été trouvé.

- -3 : Erreur pendant la comparaison :

	- un des fichiers à comparer est ouvert, bloqué ou n'existe pas;

	- le chemin de l'un des fichiers à comparer est invalide;

	- l'utilisateur n'a pas les droits en lecture sur l'un des fichiers à comparer;

	- pas assez de mémoire disponible pour effectuer la comparaison.







Le message correspondant au code d'erreur peut être connu grâce à la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.

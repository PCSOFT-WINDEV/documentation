


## PlanningInfoXY (Fonction)

***En anglais : SchedulerInfoXY***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le rendez-vous, la ressource ou la date se trouvant à une position donnée dans un champ Planning.

**Attention** :  Les fonctions [SourisPosX](../WDLang1/3071007.md) et [SourisPosY](../WDLang1/3071008.md) retournent une information pertinente uniquement dans les traitements liés à la souris ou au stylet (code de clic, code de survol, code de bouton gauche ou droit enfoncé, ...). 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Affiche dans la trace la date se trouvant à l'endroit survolé 
// (événement optionnel WM_MOUSEMOVE)
// Le résultat serait également obtenu dans le code de bouton gauche relâché (WM_LBUTTONUP)
// Ne pas utiliser dans le code "Sélection d'une période" du champ planning 
// qui n'est pas relié à la souris

MaDate est une chaîne
MaDate = PlanningInfoXY(PLN_Planning1, plnDateHeure + plnOrigineEcran, SourisPosX(), SourisPosY())
Trace(MaDate)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = PlanningInfoXY(<Champ Planning> , <Type d'informations> , <X> , <Y>)`
---

**`<Résultat> : (Chaîne de caractères ou Entier)`**

Information demandée : 

- Indice du rendez-vous au point spécifié, -1 si il n'y a pas de rendez-vous. 

- Date et heure (au format AAAAMMJJHHSSCC) du rendez-vous au point spécifié, chaîne vide ("") si il n'y a pas de date. 

- Nom de la ressource au point spécifié, chaîne vide ("") si il n'y a pas de ressource. 




**`<Champ Planning> : (Nom de champ)`**

Nom du champ Planning à manipuler.
Si ce paramètre correspond à une chaîne vide (""), la fonction **PlanningInfoXY** manipule le champ Planning auquel appartient l'événement en cours.

**`<Type d'informations> : (Constante ou combinaison de constantes de type Entier)`**

Type d'information demandée :


|   |   |
| --- | --- |
| *plnDateHeure* | Date et heure de du rendez-vous correspondant aux coordonnées fournies. |
| *plnOrigineEcran* | Par défaut, le point (0,0) correspond au point haut gauche du champ Planning manipulé (les ascenseurs étant situés à l'origine).<br>Si la constante *plnOrigineEcran* est combinée avec les autres constantes, le point (0,0) correspond à l'origine de l'écran.<br>Cette constante ne peut pas être utilisée seule.<br><br> |
| *plnRendezVous* | Indice du rendez-vous correspondant aux coordonnées fournies. |
| *plnRessource* | Nom de la ressource correspondant aux coordonnées fournies. |



**`<X> : (Entier)`**

Coordonnée X (en pixels) à analyser. Cette coordonnée est relative au champ Planning (si la constante *plnOrigineEcran* n'est pas précisée).

**`<Y> : (Entier)`**

Coordonnée Y (en pixels) à analyser. Cette coordonnée est relative au champ Planning (si la constante *plnOrigineEcran* n'est pas précisée).


## HexaVersBuffer (Fonction)

***En anglais : HexaToBuffer***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Convertit une chaîne représentant des octets en hexadécimal en un buffer binaire.

**Remarque** : Pour convertir un buffer en chaîne hexadécimale, utilisez la fonction [BufferVersHexa](../WDLang1/1000019916.md).




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
s est chaîne = "61 00 62 00 64 00 63 00 65 00 66 00 67 00 68 00<\r><\n>69 00 6A 00 6B 00 6C 00"
buf est un Buffer 
buf = HexaVersBuffer(s)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = HexaVersBuffer(<Valeur hexadécimale> [, <Nb octets par mot> [, <Option>]])`
---

**`<Résultat> : (Buffer)`**

Résultat de la conversion.

**`<Valeur hexadécimale> : (Chaîne de caractères)`**

Valeur hexadécimale à convertir. 

**`<Nb octets par mot> : (Entier ou constante optionnel)`**

Format de regroupement des valeurs dans le paramètre &lt;Valeur hexadécimale&gt;. Peut correspondre à : 

- 1 (valeur par défaut) : les valeurs sont regroupées par octet. Par exemple : 61 00 62 12.

- 2 : les valeurs sont regroupées par mot de 2 octets. Par exemple : 6100 6212. 

- 4 : les valeurs sont regroupées par mot double de 4 octets. Par exemple : 61006212. 

- la constante *SansRegroupement* : aucun regroupement ne sera effectué. Tous les codes hexadécimaux sont collés dans le paramètre &lt;Valeur hexadécimale&gt;. Exemple : 61002345A1. 




**`<Option> : (Constante optionnelle de type Entier)`**

Par défaut, le paramètre &lt;Valeur hexadécimale&gt; est considéré avec un regroupement effectué au format Little-Endian (octets de poids fort à la fin, comme en x86). 
Pour gérer un regroupement au format Big-Endian, utilisez la constante *BigEndian*.


## ParentObjet (Fonction)

***En anglais : ParentObject***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie le "parent" d'un objet graphique : champ, colonne, fenêtre, page, ...






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 
```wl
// COL_Colonne1 est dans TABLE_Table1, qui est dans la fenêtre FEN_Fenêtre2,
// qui est fille de FEN_Fenêtre1
ParentObjet(COL_Colonne1)    // Renvoie "TABLE_Table1"
ParentObjet(TABLE_Table1)      // Renvoie "FEN_Fenêtre2"
ParentObjet(FEN_Fenêtre2)    // Renvoie "FEN_Fenêtre1"
ParentObjet(FEN_Fenêtre1)    // Renvoie ""
```


<a name="Exemple2"></a>



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ParentObjet(<Nom de l'objet>)`
---

**`<Résultat> : (Chaîne de caractères)`**



- Si &lt;Nom de l'objet&gt; est un champ, &lt;Résultat&gt; correspond au nom de la fenêtre (ou de la page) à laquelle appartient le champ (même si le champ est dans un onglet).

- Si &lt;Nom de l'objet&gt; est une colonne, &lt;Résultat&gt; correspond au nom du champ Table.

- Si &lt;Nom de l'objet&gt; est une fenêtre, &lt;Résultat&gt; correspond au nom de la fenêtre mère (ou à une chaîne vide ("") si le paramètre &lt;Nom de l'objet&gt; n'a pas de fenêtre mère).

- Si &lt;Nom de l'objet&gt; est une page, &lt;Résultat&gt; correspond au nom du frameset (ou à une chaîne vide ("") si le paramètre &lt;Nom de l'objet&gt; n'a pas de frameset).




**`<Nom de l'objet> : (Chaîne de caractères)`**

Nom de l'objet (champ, fenêtre, page, colonne) dont le parent est recherché. Si l'objet est inconnu, une erreur WLangage est affichée.

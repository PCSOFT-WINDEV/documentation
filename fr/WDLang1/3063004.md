
## PressePapier (Fonction)

***En anglais : Clipboard***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère le texte ou l'image contenu dans le presse-papiers du système.

Remarques : 

- Pour identifier le contenu du presse-papiers, utilisez la fonction [PressePapierFormat](../WDLang1/3063002.md).

- Pour récupérer du texte au format RTF, utilisez la fonction [PressePapierRTF](../WDLang1/1000017200.md).

- Pour récupérer un texte au format Unicode, utilisez la fonction [PressePapierUnicode](../WDLang1/1000018858.md).




![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Il est possible uniquement de récupérer le texte contenu dans le presse-papiers.










<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Texte est une chaîne
SI PressePapierFormat(ppfTexte) = Vrai ALORS
	Texte = PressePapier()  // Contenu du presse-papiers dans Texte
FIN
```

## Syntaxe

### Récupérer du texte

`<Résultat> = PressePapier()`
---

**`<Résultat> : (Chaîne de caractères)`**

Texte contenu dans le presse-papiers.  


<a name="SYNTAXE2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 
### Récupérer une image dans un champ Image

`<Résultat> = PressePapier(<Champ Image>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'image contenue dans le presse-papiers a été récupérée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Champ Image> : (Nom de champ)`**

Nom du champ Image (présent dans la fenêtre en cours) dans lequel l'image présente dans le presse-papiers doit être copiée.  


<a name="SYNTAXE3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 
### Récupérer une image dans une variable de type Image

`<Résultat> = PressePapier(<Image>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'image contenue dans le presse-papiers a été récupérée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Image> : (Variable type Image)`**

Nom de la variable de type [Image](../WDLang1/1000019650.md) dans laquelle l'image présente dans le presse-papiers doit être copiée.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction **PressePapier** lit l'information de type texte ou image contenue dans le presse-papiers du système en cours. Les informations autres que celles de type texte ou image (binaire, etc.) contenues dans le presse-papiers ne peuvent pas être lues par la fonction **PressePapier**. Pour lire du texte au format RTF, utilisez la fonction [PressePapierRTF](../WDLang1/1000017200.md).

- La fonction **PressePapier** peut être utilisée pour gérer le couper/copier/coller d'informations de type texte ou image.

- Si un champ Image doit être affecté plusieurs fois avec le contenu du presse-papiers, il faut utiliser la fonction [dFinDessin](../WDLang1/3029019.md) entre les différentes affectations.

- La fonction [VersPressePapier](../WDLang1/3063005.md) permet d'écrire dans le presse-papiers.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Si les informations de transparence de l'image ont été conservées lors de la copie de l'image dans le presse-papiers (fonction [VersPressePapier](../WDLang1/3063005.md)), ces informations sont également conservées lors de la copie vers le champ Image et/ou vers la variable de type [Image](../WDLang1/1000019650.md). 

- ![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Seules les informations de type Texte peuvent être récupérées (Syntaxe 1).


## Constantes de gestion des threads
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>




| Constante | Description | Utilisée par les fonctions |
| --- | --- | --- |
| **DestructionSansModification** | Par défaut un signal est automatiquement modifié lors de sa destruction. Si le signal est créé avec cette constante, il ne sera pas modifié au moment de l'appel de la fonction SignalDétruit ou de la destruction automatique du signal à la fermeture de l'application. | [SignalCrée](../WDLang1/3077009.md) |
| **Infini** | Attente infinie | [ThreadAttend](../WDLang1/3077020.md), [ThreadArrête](../WDLang1/3077019.md), <br>[ThreadAttendSignal](../WDLang1/3077022.md), [SémaphoreDébut](../WDLang1/3077008.md)<br>[MutexDébut](../WDLang1/1000019023.md) |
| **partageAucun** | Le signal, le sémaphore ou le mutex est propre à l'application. Si les fonctions [SignalDétruit](../WDLang1/3077025.md), [SémaphoreDétruit](../WDLang1/3077011.md) ou [MutexDétruit](../WDLang1/1000019025.md) ne sont pas appelées, le signal, le sémaphore ou le mutex est détruit à la fin de l'application. | [SémaphoreCrée](../WDLang1/3077012.md), [SignalCrée](../WDLang1/3077009.md), <br>[MutexCrée](../WDLang1/1000019022.md) |
| **partageGlobal** | Le signal, le sémaphore ou le mutex est partagé avec toutes les applications de la machine. | [SémaphoreCrée](../WDLang1/3077012.md), [SignalCrée](../WDLang1/3077009.md), <br>[MutexCrée](../WDLang1/1000019022.md) |
| **partageUtilisateur** | Le signal, le sémaphore ou le mutex est partagé avec toutes les applications de la session de l'utilisateur qui exécute l'application. | [SémaphoreCrée](../WDLang1/3077012.md), [SignalCrée](../WDLang1/3077009.md), <br>[MutexCrée](../WDLang1/1000019022.md) |
| **PrioritéBasse** | Thread de priorité basse. | [ThreadPriorité](../WDLang1/3077027.md) |
| **PrioritéElevée** | Thread de priorité élevée. | [ThreadPriorité](../WDLang1/3077027.md) |
| **PrioritéNormale** | Thread de priorité normale. | [ThreadPriorité](../WDLang1/3077027.md) |
| **signalAutomatique** | Après son ouverture, le signal est automatiquement refermé lorsque tous les threads sont débloqués. | [SignalCrée](../WDLang1/3077009.md) |
| **signalFermé** | Le signal est fermé. | [SignalCrée](../WDLang1/3077009.md), [SignalModifie](../WDLang1/3077017.md) |
| **signalOuvert** | Le signal est ouvert. | [SignalCrée](../WDLang1/3077009.md), [SignalModifie](../WDLang1/3077017.md) |
| **signalOuvertEtFermé** | Ouvre le signal, débloque tous les threads en attente et referme le signal. | [SignalModifie](../WDLang1/3077017.md) |
| **signalManuel** | Le signal est modifié uniquement par la fonction [SignalModifie](../WDLang1/3077017.md). | [SignalCrée](../WDLang1/3077009.md) |
| **threadArrêtDemandé** | Etat du thread : L'arrêt a été demandé pour ce thread. | [ThreadEtat](../WDLang1/3077016.md), <br>Type [Thread](../WDLang1/1000024879.md) |
| **threadAttendDémarrage** | Mode de lancement du thread : Attend le démarrage effectif du thread avant de continuer l'exécution. | [ThreadExécute](../WDLang1/3077024.md), <br>Type [Thread](../WDLang1/1000024879.md) |
| **threadAuto** | Gestion automatique des threads. | [ThreadMode](../WDLang1/3077028.md) |
| **threadContexteGlobal** | Mode de lancement du thread : Force l'utilisation du contexte global du projet si le thread est exécuté depuis une fenêtre. <br>Par défaut, le contexte de la fenêtre est utilisé. | [ThreadExécute](../WDLang1/3077024.md), <br>Type [Thread](../WDLang1/1000024879.md) |
| **threadEnCours** | Le thread est en cours d'exécution. | [ThreadEtat](../WDLang1/3077016.md), <br>Type [Thread](../WDLang1/1000024879.md) |
| **threadInexistant** | Le thread n'a pas été créé ou a été détruit. | [ThreadEtat](../WDLang1/3077016.md) |
| **threadMonoProcesseur** | Force l'application à s'exécuter sur un seul processeur sur les machines multiprocesseur, hyperthread ou dual core. | [ThreadMode](../WDLang1/3077028.md) |
| **threadMultiProcesseur** | Retour à l'utilisation normale des processeurs sur les machines multiprocesseur, hyperthread ou dual core. | [ThreadMode](../WDLang1/3077028.md) |
| **threadNonExécuté** | Lancement du thread en mode normal. | Type [Thread](../WDLang1/1000024879.md) |
| **threadNormal** | Lancement du thread en mode normal. | [ThreadExécute](../WDLang1/3077024.md) |
| **threadPrincipal** | Manipule le thread principal. | [ThreadArrête](../WDLang1/3077019.md), <br>[ThreadEnvoieSignal](../WDLang1/3077021.md), <br>Type [Thread](../WDLang1/1000024879.md) |
| **threadSectionCritique** | Gestion manuelle des sections critiques. | [ThreadMode](../WDLang1/3077028.md) |
| **threadSécurisé** | Lance un thread secondaire en mode sécurisé. | [ThreadExécute](../WDLang1/3077024.md), <br>Type [Thread](../WDLang1/1000024879.md) |
| **threadSuspendu** | Le thread est suspendu. | [ThreadEtat](../WDLang1/3077016.md)<br>Type [Thread](../WDLang1/1000024879.md) |
| **threadTerminé** | Le thread est terminé. | Type [Thread](../WDLang1/1000024879.md) |
| **threadUtiliseHyperFile**<br>**threadUtiliseHFSQL**<br>**threadCopieComplèteContexteHFSQL** | Provoque la copie immédiate du contexte HFSQL. <br>Conseillé par exemple si le thread doit tenir compte des positions en cours dans les fichiers et requêtes du contexte de l'appelant. | [ThreadExécute](../WDLang1/3077024.md), <br>Type [Thread](../WDLang1/1000024879.md) |
| **threadCopieLégèreContexteHFSQL** | Provoque la copie immédiate d'une partie du contexte HFSQL.<br>Seuls les répertoires où se trouvent les fichiers de données en HFSQL Classic et/ou les connexions en HFSQL Client/Serveur sont mémorisés. | [ThreadExécute](../WDLang1/3077024.md), <br>Type [Thread](../WDLang1/1000024879.md) |

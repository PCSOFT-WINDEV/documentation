
## ChaîneCompare (Fonction)

***En anglais : StringCompare***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Compare caractère par caractère deux chaînes :

- selon l'ordre des caractères ASCII.

- selon l'ordre lexicographique.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MaChaîne1 est une chaîne = "Le développement facile"
MaChaîne2 est une chaîne = "Le développement simple"
RésultatCompare est un entier = ChaîneCompare(MaChaîne1, MaChaîne2)
// RésultatCompare = -1 : le caractère 'f' est inférieur au caractère 's'
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ChaîneCompare(<Première chaîne> , <Seconde chaîne> [, <Options>])`
---

**`<Résultat> : (Entier)`**



- 0 si les chaînes de caractères sont égales.

- -1 si les caractères de &lt;Première chaîne&gt; sont inférieurs aux caractères de &lt;Seconde chaîne&gt; dans l'ordre des caractères ASCII ou dans l'ordre lexicographique.

- 1 si les caractères de &lt;Première chaîne&gt; sont supérieurs aux caractères de &lt;Seconde chaîne&gt; dans l'ordre des caractères ASCII ou dans l'ordre lexicographique.




**`<Première chaîne> : (Chaîne de caractères)`**

Chaîne de caractères à comparer.

**`<Seconde chaîne> : (Chaîne de caractères)`**

Chaîne de caractères à comparer.

**`<Options> : (Constante (ou combinaison de constantes) optionnelle)`**

Options de comparaison :


|   |   |
| --- | --- |
| *ccNormal*<br>(valeur par défaut) | Comparaison normale, similaire à l'opérateur '='. |
| *ccOrdreLexicographique* | Comparaison des chaînes de caractères selon l'ordre lexicographique (par exemple, le caractère 'é' est compris entre les caractères 'e' et 'f'). <br><br> |
| *ccRespecteNumérique* | Comparaison en tenant compte des numériques au sein des chaînes (dans ce cas, "10" est après "9"). |
| *ccSansAccent* | Comparaison sans tenir compte des accents.<br><br> |
| *ccSansCasse* | Comparaison sans tenir compte de la casse des caractères (majuscules/minuscules). |
| *ccSansEspace* | Comparaison sans tenir compte des espaces avant et après les chaînes de caractères à comparer. |
| *ccSansEspaceIntérieur* | Comparaison sans tenir compte des espaces présents à l'intérieur des chaînes de caractères à comparer.<br><br>![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Cette constante n'est pas disponible. |
| *ccSansPonctuationNiEspace* | Comparaison sans tenir compte de la ponctuation ni des espaces (voir Notes pour plus de détails).<br><br> |

## Remarques




### Équivalences entre la fonction ChaîneCompare et les opérateurs de comparaison
<a name="equivalences_entre_fonction_docparampagetitleshort_les_operateurs_comparaison_ELTPARAGRAPHE000130"></a>

- **Opérateur =** : 
	L'opérateur = est équivalent au code suivant : ChaîneCompare(&lt;Chaîne1&gt;, &lt;Chaîne 2&gt;, ccNormal)

- **Opérateur ~=** : 
	L'opérateur ~= est équivalent au code suivant : ChaîneCompare(&lt;Chaîne1&gt;, &lt;Chaîne 2&gt;, ccSansCasse + ccSansAccent + ccSansEspace)
	

- **Opérateur ~~** : 
	L'opérateur ~~ est équivalent au code suivant : ChaîneCompare(&lt;Chaîne1&gt;, &lt;Chaîne 2&gt;, ccSansCasse + ccSansAccent + ccSansPonctuationNiEspace)
	

- **Opérateur &lt;, &gt;, &lt;= et &gt;=** : 
	Ces opérateurs sont équivalents au code suivant : ChaîneCompare(&lt;Chaîne1&gt;, &lt;Chaîne 2&gt;, ccOrdreLexicographique)
	




Pour plus de détails sur les opérateurs de comparaison, consultez les [Opérateurs de comparaison](../Motscles/1512006.md).

### Comparaison des numériques
<a name="comparaison_des_numeriques_ELTPARAGRAPHE000170"></a>La constante *ccRespecteNumérique* effectue le tri en tenant compte de la valeur numérique représentée par les chiffres et non pas simplement de l'ordre alphabétique des chiffres.

Ainsi sans la constante *ccRespecteNumérique*, les chaînes suivantes seront classées dans cet ordre : "chaîne1", "chaîne10", "chaîne2"

Avec la constante *ccRespecteNumérique*, les mêmes chaînes seront classées dans cet ordre : "chaîne1", "chaîne2", "chaîne10"

### Caractères pris en compte pour la ponctuation et les espaces
<a name="caracteres_pris_compte_pour_ponctuation_les_espaces_ELTPARAGRAPHE000448"></a>Les caractères pris en compte pour la ponctuation et les espaces sont fournis par le système. Pour obtenir la liste de ces caractères, il suffit d'écrire le code WLangage suivant :

```wl
s est une chaîne
POUR i = 0 A 255
	SI Caract(i) <> ChaîneFormate(Caract(i), ccSansPonctuationNiEspace) ALORS
		 s += Caract(i)
	FIN
FIN
Info(s)
VersPressePapier(s)
```

### Fonction ChaîneCompare et UNICODE
<a name="fonction_docparampagetitleshort_unicode_ELTPARAGRAPHE000310"></a>Il est possible d'utiliser des chaînes ANSI, Unicode et des buffers dans les différents paramètres de la fonction.

La règle de conversion utilisée est la suivante pour les systèmes Ansi (Windows ou Linux) :

- Si l'une au moins des chaînes est un buffer, toutes les chaînes sont converties en buffer, et l'opération est réalisée avec des buffers.

- Si la première condition n'est pas remplie, si l'une au moins des chaînes est Unicode, toutes les chaînes sont converties en Unicode et l'opération est réalisée en Unicode (si nécessaire, la conversion est réalisée avec l'alphabet courant).

- Sinon, l'opération est réalisée en Ansi.




La règle de conversion utilisée est la suivante pour les systèmes Unicode :

- Si l'une au moins des chaînes est un buffer, toutes les chaînes sont converties en buffers et l'opération est réalisée avec des buffers.

- Sinon, l'opération est réalisée en Unicode.




Pour plus de détails sur l'UNICODE, consultez [Gestion de l'UNICODE](../WDLang1/3024038.md).

Rappel : Les paramètres linguistiques utilisés sont définis lors de l'appel de la fonction [ChangeAlphabet](../WDLang1/3054001.md).

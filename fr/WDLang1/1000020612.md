
## EvénementTimeline (Type de variable)

***En anglais : EventTimeline***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **EvénementTimeline** permet de définir toutes les caractéristiques avancées d'un événement. Cet événement sera affiché dans un champ TimeLine. Les caractéristiques de cet événement peuvent être définies et modifiées à l'aide de différentes propriétés WLangage. 

L'événement correspondant peut ensuite être utilisé avec les fonctions WLangage de gestion des champs TimeLine.

**Remarque** : Pour plus de détails sur la déclaration de ce type de variable et l'utilisation des propriétés WLangage, consultez [Déclaration d'une variable](../Motscles/1514032.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
TimelineSupprimeTout(TL_Automates)
Evt est un EvénementTimeline

TimelineAjoutePiste(TL_Automates, "Automate 1")
Evt.Piste = "Automate 1"
Evt.Titre = "Démarrage"
Evt.Début = 10
Evt.Fin = 150
Evt.CouleurFond = VertClair
TimelineAjouteEvénement(TL_Automates, Evt)

Evt.Piste = "Automate 1"	
Evt.Titre = "Traitement spécial"
Evt.Début = 150
Evt.Fin = 450
Evt.CouleurFond = BleuPastel
TimelineAjouteEvénement(TL_Automates, Evt)

Evt.Piste = "Automate 1"	
Evt.Titre = "Pause"
Evt.Début = 450
Evt.Fin = 700
Evt.CouleurFond = RougeClair
TimelineAjouteEvénement(TL_Automates, Evt)

Evt.Piste = "Automate 1"	
Evt.Titre = "Arrêt"
Evt.Début = 700
Evt.Fin = 750
Evt.CouleurFond = RougePastel
TimelineAjouteEvénement(TL_Automates, Evt)
```

## Remarques
<a name="NOTE0_1"></a>


### Propriétés spécifiques à la description des variables de type EvénementTimeline
<a name="proprietes_specifiques_description_des_variables_type_evenementtimeline_ELTPARAGRAPHE000030"></a>Les propriétés suivantes peuvent être utilisées pour manipuler un événement :

| **Nom de la propriété** | **Type manipulé** | **Effet** |
| --- | --- | --- |
| Auteur | Chaîne de caractères | Nom de l'auteur de l'événement. Si cette propriété n'est pas précisée, l'événement n'a pas d'auteur. |
| AvecRépétition | Booléen | <br><br>- <u><u><u><u>Vrai</u></u></u></u> si l'événement doit être répété, <br><br>- <u><u><u><u>Faux</u></u></u></u> si l'événement n'a lieu qu'une seule fois. <br><br><br>Si l'événement est répété, la propriété **Répétition** permet de spécifier la fréquence de la répétition. <br><br>Par défaut, cette propriété est à <u><u><u><u>Faux</u></u></u></u>. |
| Bulle | Chaîne de caractères | Bulle d'aide affichée lors du survol de l'événement. Par défaut, cette valeur est composée de l'heure de début et fin de l'événement ainsi que du titre de l'événement. |
| Catégorie | Chaîne de caractères | Catégorie de l'événement. Si cette propriété n'est pas précisée, l'événement n'est associé à aucune catégorie. |
| Contenu | Chaîne de caractères | Détail de la description de l'événement. Le contenu est affiché dans le champ TimeLine. <br><br>Si cette propriété n'est pas précisée, l'événement n'a pas de description. |
| CouleurFond | Entier | Couleur de fond utilisée pour l'affichage de l'événement dans le champ TimeLine.<br>Cette couleur peut correspondre à :<br><br>- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)), <br><br>- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)), <br><br>- une couleur prédéfinie (voir [les couleurs prédéfinies du WLangage](../WDLang5/3010002.md)).<br><br><br>Si cette propriété n'est pas précisée, une couleur sera automatiquement calculée. Les événements de la même catégorie auront la même couleur.<br><br> |
| Début | Entier | Début de l'événement. Cette propriété doit être dans l'unité utilisée pour l'affichage du champ TimeLine (seconde, milliseconde ou macroseconde).<br>Cette propriété est obligatoire. |
| Fin | Entier | Fin de l'événement.  Cette propriété doit être dans l'unité utilisée pour l'affichage du champ TimeLine (seconde, milliseconde ou macroseconde).<br>Cette propriété est obligatoire. |
| ID | Chaîne de caractères | Identifiant associé à l'événement. Permet de stocker l'identifiant d'un élément à associer à l'élément pour l'utiliser en programmation plus tard. <br><br>Par exemple, cet identifiant peut correspondre à l'identifiant d'un enregistrement dans une base de données. |
| Image | Chaîne de caractères | Image associée à l'événement. Cette image est affichée dans le champ TimeLine (en plus du pictogramme représentant un élément important si nécessaire). <br><br>Cette propriété peut correspondre : <br><br>- au chemin vers une image accessible depuis le poste en cours,<br><br>- à un fichier image présent dans la bibliothèque de l'application,<br><br>- à un champ Image contenant une image,<br><br>- à un dessin réalisé dans un champ Image grâce aux fonctions de dessin et sauvegardé en mémoire.<br><br><br>Si cette propriété n'est pas précisée, l'événement n'a pas d'image associée.<br><br>**Remarque** : Il est nécessaire d'utiliser une fenêtre interne pour afficher une image dans l'événement. Le style simple (proposé par défaut) ne permet pas d'afficher une image. |
| Importance | Entier | Importance de l'événement. <br><br>Si cette propriété est supérieure ou égale à 1, un pictogramme (![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=agRendezVous_IconeImportant.gif)<br>) sera affiché dans le champ TimeLine pour indiquer un événement important. <br><br>Si cette propriété n'est pas précisée, sa valeur vaut 0. |
| Note | Chaîne de caractères | Note associée à l'événement. <br><br>Si cette propriété n'est pas précisée, aucune note n'est associée à l'événement. |
| Piste | Chaîne de caractères | Piste associée à l'événement. |
| Répétition | Variable de type [Répétition](../WDLang1/1000019262.md) | Paramètres avancés de la répétition. Cette propriété est prise en compte uniquement si la propriété **AvecRépétition** est égale à <u><u><u><u>Vrai</u></u></u></u>. |
| Titre | Chaîne de caractères | Titre de l'événement. Le titre est affiché dans le champ TimeLine. <br><br>Si cette propriété n'est pas précisée, l'événement n'a aucun titre. |

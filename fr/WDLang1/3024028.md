
## Inverse (Fonction)

***En anglais : Reverse***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie pour chaque caractère d'une chaîne de caractères le complément à 255. La chaîne de caractères est ainsi inversée.

Pour une chaîne de caractères Unicode (ou en WINDEV Mobile), le complément est calculé par rapport à 65535.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SAI_Saisie2 = Inverse(SAI_Saisie1)
// Si le champ SAI_Saisie1 contient TEST, le champ SAI_Saisie2 contiendra "º¬"
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = Inverse(<Chaîne initiale>)`
---

**`<Résultat> : (Chaîne de caractères)`**

Contient la chaîne de caractères inversée.

**`<Chaîne initiale> : (Chaîne de caractères)`**

Chaîne de caractères à inverser. Cette chaîne n'est pas modifiée.

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Cette chaîne peut être une chaîne de caractères au format Unicode.

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Les chaînes de caractères au format Unicode ne sont pas gérées.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Fonction Inverse et Unicode
<a name="fonction_docparampagetitleshort_unicode_ELTPARAGRAPHE000077"></a>Si la chaîne passée en paramètre est au format ANSI, le résultat sera au format ANSI.

Si la chaîne passée en paramètre est au format UNICODE, le résultat sera au format UNICODE.

**Remarque** : Si le résultat de la fonction **Inverse** utilisée sur une chaîne ANSI est affecté à une chaîne UNICODE (et inversement), la conversion ANSI/UNICODE sera implicitement réalisée.

Pour plus de détails, consultez [Gestion de l'UNICODE](../WDLang1/3024038.md).

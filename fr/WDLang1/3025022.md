


## ChampPremier (Fonction)

***En anglais : ControlFirst***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique le nom du premier champ spécifié dans l'ordre de navigation pour la fenêtre spécifiée. Ce champ correspond au premier champ en saisie à l'ouverture de la fenêtre.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Reprendre la saisie sur le premier champ
RAZ()
DonneFocusEtRetourUtilisateur(ChampPremier())
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ChampPremier([<Fenêtre manipulée>])`
---

**`<Résultat> : (Chaîne de caractères)`**



- Nom du premier champ en saisie dans la fenêtre spécifiée (ce champ peut prendre le focus et il n'est ni invisible, ni en affichage seulement). 

- Chaîne vide ("") si aucun champ ne peut prendre le focus ou si la fenêtre spécifiée n'existe pas.




**`<Fenêtre manipulée> : (Nom de fenêtre optionnel)`**

Nom de la fenêtre manipulée. 
Si ce paramètre n'est pas spécifié ou correspond à une chaîne vide (""), la fenêtre manipulée correspond à la fenêtre en cours.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Premier champ dans l'ordre de navigation
<a name="premier_champ_dans_ordre_navigation_ELTPARAGRAPHE000056"></a>L'ordre de navigation dans les champs d'une fenêtre correspond à l'ordre utilisé pour passer au champ suivant par la touche TAB du clavier.

Pour définir l'ordre de navigation sous l'éditeur de fenêtres, sous le volet "Fenêtre", dans le groupe "Ordre", déroulez "Navigation" et sélectionnez "Éditer".

Les champs pris en compte dans l'ordre de navigation ne sont pas obligatoirement des champs de saisie : les champs Combo, Sélecteur, Liste, etc. peuvent être pris en compte dans l'ordre de navigation.




## &lt;Champ d'affichage HTML&gt;.ExécuteProcédureNavigateur (Fonction)

***En anglais : &lt;HTML Display control&gt;.ExecuteBrowserProcedure***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Exécute une procédure WLangage navigateur WEBDEV dans un champ d'affichage HTML avec une page créée par WEBDEV.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Exécuter la procédure navigateur VérifieSaisie()
HTM_AffichageHTML.ExecuteProcédureNavigateur("VérifieSaisie")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ d'affichage HTML>.ExécuteProcédureNavigateur(<Procédure navigateur> [, <Paramètre 1> [, <Paramètre N>]])`
---

**`<Champ d'affichage HTML> : (Nom de champ)`**

Nom du champ d'affichage HTML manipulé.

**`<Procédure navigateur> : (Chaîne de caractères)`**

Nom de la procédure WLangage navigateur WEBDEV à exécuter. Cette procédure doit être une procédure locale de la page WEBDEV affichée dans le champ.

**`<Paramètre 1> : (Type quelconque optionnel)`**

Premier paramètre optionnel à passer à la procédure.
$~ ... |

**`<Paramètre N> : (Type quelconque optionnel)`**

Nème paramètre optionnel à passer à la procédure.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Le nom réel de la procédure utilisé par WEBDEV pour les procédures Javascript ne correspond pas au nom de la procédure navigateur saisi sous l'éditeur. La fonction **&lt;Champ d'affichage HTML&gt;.ExécuteProcédureNavigateur** effectue automatiquement la liaison entre les différents noms. Cette fonction doit donc être préférée à la fonction [&lt;Champ d'affichage HTML&gt;.ExécuteJS](../WDLang1/1000024372.md).




## AnalyseurDébut (Fonction)

***En anglais : ProfilerStart***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Lance la "collecte d'informations" pour l'[analyseur de performances](../Editeurs/2030035.md).

**Rappel** : Pour lancer l'analyseur de performances depuis une application, utilisez la combinaison de touches : Maj + Windows + F10. 

A partir de cette fonction, tous les traitements/événements exécutés sont enregistrés dans un fichier WPF.

Ce fichier pourra être ouvert :

- avec l'[analyseur de performances](../Editeurs/2030035.md). 

- directement sous l'éditeur de WINDEV, WEBDEV ou WINDEV Mobile : sous le volet "Accueil", dans le groupe "Général", déroulez "Ouvrir" et sélectionnez "Ouvrir".





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
AnalyseurDébut("Test_ProcédureLancement.wpf")
ProcédureLancement()
AnalyseurFin()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`AnalyseurDébut([<Nom du fichier WPF>])`
---

**`<Nom du fichier WPF> : (Chaîne de caractères optionnelle)`**

Emplacement et nom du fichier WPF à créer. Le nom du fichier WPF peut être construit avec les éléments suivants :


|   |   |
| --- | --- |
| [%RepExe%] | Répertoire de l'exécutable, toujours complété par le caractère "\\". Ce répertoire est équivalent au résultat de la fonction [fRepExe](../WDLang1/3036049.md). |
| [%NomExe%] | Nom de l'exécutable. |
| [%Date%] | Date de lancement de la collecte d'informations (format AAAAMMJJ). |
| [%Heure%] | Heure de lancement de la collecte d'informations (format HHMMSS). |
| [%NomUtilisateur%] | Nom de l'utilisateur courant. |
| [%NomMachine%] | Nom de la machine. |
| [%RepDonnees%] | Répertoire des fichiers de données HFSQL spécifié par l'utilisateur lors de l'installation de l'application. Ce répertoire est équivalent au résultat de la fonction [fRepDonnées](../WDLang1/3036001.md). |
| [%RepDonneesUtilisateur%] | Répertoire pour les données de l'utilisateur courant de l'application courante. Ce répertoire est équivalent au résultat de la fonction [fRepDonnéesUtilisateur](../WDLang1/3036071.md). |

Par défaut : 

- l'extension du fichier WPF est ".wpf". 

- le nom du fichier WPF correspond à : "[%RepDonneesUtilisateur%][%NomExe%]_[%Date%]_[%Heure%]". 
	Par exemple : "C:\\Users\\Doc\\AppData\\Roaming\\Applications WinDev\\MonAppli\\MonAppli_20161023_130812.wpf".




Attention : Si vous utilisez la construction dynamique des chaînes (option "Autoriser "[% %]" dans les chaînes" dans l'onglet "Compilation" de la description du projet, une erreur de compilation apparaît (identificateur inconnu). Dans ce cas, il est nécessaire de faire précéder chaque chaîne par '-%'. Exemple : 

```wl
AnalyseurDébut("[%RepDonneesUtilisateur%][%NomExe%]_[%Date%]_[%Heure%]")
```
 devient
```wl
AnalyseurDébut(-%"[%RepDonneesUtilisateur%][%NomExe%]_[%Date%]_[%Heure%]")
```




<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Si la fonction **AnalyseurDébut** est utilisée plusieurs fois dans le même traitement ou événement (sans appel à la fonction [AnalyseurFin](../WDLang1/3013002.md)), une erreur WLangage est générée.

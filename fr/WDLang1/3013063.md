
## ExécuteAncêtre (Fonction)

***En anglais : ExecuteAncestor***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Dans le cas d'une surcharge de code, permet d'exécuter l'événement du modèle correspondant à l'événement du champ en cours.

Par exemple :

## Syntaxe

`ExécuteAncêtre()`

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La fonction **ExécuteAncêtre** est automatiquement insérée dans le code en cas de surcharge de code d'un champ d'un modèle. Cette fonction peut être déplacée ou supprimée du code.

Si un événement existe dans la fenêtre modèle, cet événement apparaît en double dans la fenêtre utilisant le modèle. En effet, sont affichés :

- l'événement issu du modèle. Cet événement apparaît en grisé et son intitulé est suivi de "(modèle &lt;Nom du modèle&gt;)".

- l'événement de la fenêtre / page.




**Cette fonction ne doit pas être utilisée dans les événements associés à la fenêtre ou à la page** (code d'initialisation, code d'ouverture, code de fermeture, ...), car les codes des ancêtres sont automatiquement exécutés, dans un ordre spécifique. Le code saisi dans un des événements de la page sera exécuté de la manière suivante :  

- l'événement "Déclarations globales" est fusionné entre le modèle et la page (ou la fenêtre) utilisant le modèle. Il est donc nécessaire de ne pas utiliser des variables de même nom dans cet événement.

- pour tous les événements (sauf celui de fermeture de la page ou de la fenêtre) : 

	- les événements des modèles appliqués à la page (ou à la fenêtre) sont exécutés en suivant l'ordre d'inclusion des modèles dans la page (ou la fenêtre), 

	- puis l'événement correspondant à la page (ou à la fenêtre) est exécuté.




- pour l'événement de fermeture de la page (ou de la fenêtre) : l'événement est exécuté, puis les événements des modèles appliqués à la page (ou à la fenêtre) sont exécutés en suivant l'ordre inverse d'inclusion des modèles dans la page (ou la fenêtre).

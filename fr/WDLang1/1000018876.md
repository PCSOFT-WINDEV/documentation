


## fCréeLien (Fonction)

***En anglais : fCreateLink***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Crée un lien sur un fichier à l'emplacement spécifié. Ce lien peut être : 

- un lien symbolique  (fichier lnk). 

- un lien physique. Ce lien peut avoir ou non le même nom que le fichier d'origine. Le lien physique permet de faire apparaître un même fichier à plusieurs emplacements. 




**Remarques** : 

- Pour supprimer un lien (symbolique ou physique), utilisez la fonction [fSupprime](../WDLang1/3036029.md). 

- Pour créer un raccourci sur le bureau de Windows, dans le menu "Démarrer" de Windows ou dans un répertoire spécifique, utilisez la fonction [CréeRaccourci](../WDLang1/3014008.md). 





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Création d'un lien symbolique (fichier lnk)
fCréeLien("C:\temp\lisezmoi.txt", "C:\lisezmoi.lnk", flienSymbolique)

// Création d'un lien physique
fCréeLien("C:\temp\lisezmoi.txt", "C:\lisezmoi.doc", flienPhysique)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = fCréeLien(<Fichier pointé> , <Destination du lien> , <Type de lien> [, <Ligne de commande> [, <Répertoire de travail>]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le lien a été créé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Fichier pointé> : (Chaîne de caractères)`**

Chemin complet du fichier vers lequel le lien doit pointer.

**`<Destination du lien> : (Chaîne de caractères)`**

Chemin et nom du fichier correspondant au lien à créer. L'extension du lien doit être précisée.

**`<Type de lien> : (Constante de type Entier)`**

Type de lien à créer : 


|   |   |
| --- | --- |
| *fLienPhysique* | Création d'un lien physique. <br>Un lien physique est un fichier accessible par plusieurs chemins. La suppression du fichier est possible uniquement lorsque tous les liens physiques ont été supprimés.<br>**Attention** : Le &lt;Fichier pointé&gt; et la &lt;Destination du lien&gt; doivent être sur le même disque. |
| *fLienSymbolique* | Création d'un lien symbolique. <br>Un lien symbolique est un "pointeur" vers un fichier. La suppression du lien symbolique n'affecte pas le fichier pointé. La suppression du fichier pointé rend le lien symbolique invalide.<br><br>![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Seul le propriétaire du fichier peut créer un lien symbolique.<br><br><br><br>![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png) Sous Windows 10, pour créer un lien symbolique, l'utilisateur doit être administrateur du poste. |



**`<Ligne de commande> : (Chaîne de caractères optionnelle)`**

Ligne de commande du lien. Ce paramètre est pris en compte uniquement lors de la création d'un lien symbolique. 
![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Ce paramètre n'est pas disponible.

**`<Répertoire de travail> : (Chaîne de caractères optionnelle)`**

Répertoire de travail du lien. Ce paramètre est pris en compte uniquement lors de la création d'un lien symbolique.
![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Ce paramètre n'est pas disponible.

<a name="NOTE0_1"></a>

## Remarques

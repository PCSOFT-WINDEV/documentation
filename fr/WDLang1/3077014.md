


## ThreadSuspend (Fonction)

***En anglais : ThreadSuspend***

	<IMG style="float:left;margin-right:10px;" src="https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=warning_big.png"><B>Avertissement</B><BR>

<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Suspend temporairement l'exécution du thread désigné. Le traitement en cours n'est pas bloqué. Pour relancer l'exécution du thread, utilisez la fonction [ThreadReprend](../WDLang1/3077023.md).

**Rappel** : Un thread est un processus lancé en parallèle de l'application en cours ("thread" principal). Il est par exemple possible de lancer l'exécution d'une tâche en traitement de fond (sauvegarde, ...).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// -- Clic sur le bouton BTN_Enregistre
// Suspend l'impression en tâche de fond
ThreadSuspend("Impression_fond")
// Enregistrement
...
// Reprise de l'impression
ThreadReprend("Impression_fond")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`ThreadSuspend(<Nom du Thread>)`
---

**`<Nom du Thread> : (Chaîne de caractères)`**

Nom du thread à suspendre. Ce nom est donné lors de l'exécution du thread (fonction [ThreadExécute](../WDLang1/3077024.md)). 
Si le nom du thread n'existe pas, aucune erreur n'est affichée. 
**Attention** : le nom du thread ne peut pas être une chaîne vide : il est impossible de suspendre le thread courant.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Pour que le thread courant attende un thread, utilisez la fonction [ThreadAttendSignal](../WDLang1/3077022.md).


## RegistreCopieClé (Fonction)

***En anglais : RegistryCopyKey***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Copie une clé de registre avec toutes ses sous-clés et ses valeurs. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
RegistreCopieClé("HKEY_LOCAL_MACHINE\SOFTWARE\MonAppli", ...
	"HKEY_LOCAL_MACHINE\SOFTWARE\MonAppli2")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = RegistreCopieClé([<Mode d'accès>, ] <Chemin de la clé source> , <Chemin de la clé destination>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la copie a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 




**`<Mode d'accès> : (Constante de type Entier)`**

Mode d'accès à la base de registre :



|   |   |
| --- | --- |
| *registreMode32* | Mode d'accès forcé pour accéder à la base de registre comme un programme 32 bits. |
| *registreMode64* | Mode d'accès forcé pour accéder à la base de registre comme un programme 64 bits. |
| *registreModeAuto*<br>(Valeur par défaut) | Mode d'accès à la base de registre automatique : <br><br>- une application 32 bits exécutée sur un système 32 bits manipule la base de registre comme un programme 32 bits.<br><br>- une application 32 bits exécutée sur un système 64 bits manipule la base de registre à partir de la branche suivante :<br>	<br><pre><code>HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node</code></pre><br><br><br>- une application 64 bits exécutée sur un système 64 bits manipule la base de registre comme un programme 64 bits.<br><br><br> |



**`<Chemin de la clé source> : (Chaîne de caractères)`**

Chemin complet de la clé à copier. 

**`<Chemin de la clé destination> : (Chaîne de caractères)`**

Chemin complet de la clé destination. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction **RegistreCopieClé** effectue une copie récursive.

- Sous Windows Vista (et supérieurs), les opérations liées à la copie sont effectuées dans une transaction. En cas d'échec d'une opération, toutes les opérations précédentes sont annulées.

- Sur les versions de Windows antérieures à Windows Vista, les opérations liées à la copie ne sont pas effectuées en transaction. En cas d'échec d'une opération (copie d'une sous-clé par exemple), les opérations précédentes (copie des autres sous-clés par exemple) ne sont pas annulées.


## SysInstance (Fonction)

***En anglais : SysInstance***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie l'instance d'un module.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Instance de l'application en cours
ResInstance = SysInstance()
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = SysInstance([<Nom du module>])`
---

**`<Résultat> : (Entier)`**

Instance du module spécifié.

**`<Nom du module> : (Chaîne de caractères optionnelle)`**

Correspond :

- soit au nom et au chemin complet (ou relatif) du fichier exécutable (extension ".EXE") de l'application en cours. Un chemin de type UNC est autorisé.

- soit au nom et au chemin complet (ou relatif) d'une librairie WINDEV (fichier ".DLL") actuellement chargée en mémoire (fonction [ChargeDLL](../WDLang1/3014006.md)). Un chemin de type UNC est autorisé.




Si ce paramètre n'est pas spécifié, l'instance de l'application en cours est retournée.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Paramètre "Nom du module"
<a name="parametre_nom_module_ELTPARAGRAPHE000043"></a>Le paramètre &lt;Nom du module&gt; ne doit pas contenir le nom et le chemin d'un fichier exécutable différent de celui de l'application en cours.

Pour savoir si un  exécutable est actuellement lancé, utilisez la fonction [ExeLancé](../WDLang1/3035003.md).


<a name="NOTE0_2"></a>




### Équivalence
<a name="equivalence_ELTPARAGRAPHE000057"></a>
```wl
ResInstance = SysInstance()
```


est équivalent à 


```wl
ResInstance = Instance()
```

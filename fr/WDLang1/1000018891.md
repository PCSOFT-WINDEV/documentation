<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = HaspHLConnecte(<Identifiant de la zone> , <Code vendeur> [, <Options>])`
---

**`<Résultat> : (Entier)`**



- Identifiant de session. Cet identifiant sera utilisé dans les [fonctions de gestion des clés Hasp](../WDLang1/1000018999.md).

- -1 en cas d'erreur. En cas d'erreur, la fonction [ErreurInfo](../WDLang1/3013008.md) permet d'identifier l'erreur. 




**`<Identifiant de la zone> : (Entier)`**

Numéro de la zone de la clé Hasp à laquelle l'application doit se connecter.

**`<Code vendeur> : (Chaîne de caractères)`**

Code vendeur obtenu auprès du fournisseur de la clé.

**`<Options> : (Combinaison optionnelle de constantes de type Entier)`**

Mode de connexion à utiliser : 


|   |   |
| --- | --- |
| *haspIgnoreTSE* | Cette constante permet de désactiver la vérification de l'environnement "Terminal Server". |
| *haspLicenceLocale* | Seules les clés HASP connectées au poste en cours seront utilisées. |
| *haspLicenceParProcessus* | Les licences seront décomptées pour chaque processus lancé. Par exemple, deux instances de la même application utiliseront deux licences. |
| *haspLicenceRéseau* | Seules les clés HASP réseau accessibles par le poste en cours seront utilisées. |
| *haspModeCompatible* | Cette constante permet d'utiliser les clés HASP de type HASP3 et HASP4. |


Par défaut, aucune constante n'est utilisée.

**Attention** : La combinaison de constantes *haspLicenceLocale* + *haspLicenceRéseau* n'est pas autorisée.

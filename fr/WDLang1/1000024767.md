
## FAAActive (Fonction)

***En anglais : AAFEnable***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Réactive une fonctionnalité automatique qui a été précédemment désactivée par la fonction [FAADésactive](../WDLang1/1000022018.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Clic sur le champ Bouton "Ne pas sauvegarder"
// => les valeurs saisies par l'utilisateur ne seront pas sauvées
FAADésactive(MaFenêtre, faaSauveValeur)
```

```wl
// Clic sur le champ Bouton "Ré-activer sauvegarde"
// => les valeurs saisies par l'utilisateur seront sauvées
FAAActive(MaFenêtre, faaSauveValeur)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`FAAActive([<Champ ou Fenêtre>, ] <Fonctionnalité à réactiver>)`
---

**`<Champ ou Fenêtre> : (Nom de champ ou nom de fenêtre optionnel)`**

Nom du champ ou de la fenêtre concerné par la fonctionnalité automatique. Il est également possible d'utiliser un nom d'alias de fenêtre.
Si ce paramètre n'est pas spécifié, la fonctionnalité automatique indiquée sera désactivée sur l'ensemble de l'application en cours.

**`<Fonctionnalité à réactiver> : (Constante de type Entier)`**

Action automatique à réactiver. Cette action dépend du type de champ manipulé. Si le type de champ est invalide pour la constante sélectionnée, la fonction n'a aucun effet.




## &lt;Variable WDPic&gt;.DessineCercle (Fonction)

***En anglais : &lt;WDPic variable&gt;.DrawCircle***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Dessine un cercle ou une ellipse dans une variable de type [WDPic](../WDLang1/1000023503.md).![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les dessins](../WDLang1/3029035.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MonImage est un WDPic = "Test.bmp"
// Dessin d'un cercle dont le fond est jaune clair et 
// le bord bleu clair dans une variable WDPic
MonImage.DessineCercle(10, 10, 50, 50, JauneClair, BleuClair)
IMG_MonDessin = MonImage
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Image WDPic>.DessineCercle(<X1> , <Y1> , <X2> , <Y2> [, <Couleur du fond> [, <Couleur du trait>]])`
---

**`<Image WDPic> : (Variable de type WDPic)`**

Nom de la variable de type [WDPic](../WDLang1/1000023503.md) à manipuler. Seul le calque d'arrière-plan sera manipulé.

**`<X1> : (Entier)`**

Abscisse du coin haut gauche du rectangle dans lequel le cercle est inscrit. Ces coordonnées sont exprimées en pixels.

**`<Y1> : (Entier)`**

Ordonnée du coin haut gauche du rectangle dans lequel le cercle est inscrit. Ces coordonnées sont exprimées en pixels.

**`<X2> : (Entier)`**

Abscisse du coin bas droit du rectangle dans lequel le cercle est inscrit. Ces coordonnées sont exprimées en pixels.

**`<Y2> : (Entier)`**

Ordonnée du coin bas droit du rectangle dans lequel le cercle est inscrit. Ces coordonnées sont exprimées en pixels.

**`<Couleur du fond> : (Entier ou constante (optionnel))`**

Couleur du fond du cercle. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md), 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) une variable de type [Couleur](../WDLang1/1000019661.md).


Si ce paramètre n'est pas précisé, la couleur du fond :

- est *Transparent* si la fonction [&lt;Variable WDPic&gt;.Fond](../WDLang1/1000024481.md) n'a pas été utilisée auparavant,

- correspond à la couleur précisée dans le dernier appel à la fonction [&lt;Variable WDPic&gt;.Fond](../WDLang1/1000024481.md).




**`<Couleur du trait> : (Entier ou constante (optionnel))`**

Couleur du trait du cercle. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- [une couleur prédéfinie du WLangage](../WDLang5/3010002.md), 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) une variable de type [Couleur](../WDLang1/1000019661.md).


Si ce paramètre est précisé, les caractéristiques indiquées avec la fonction [&lt;Variable WDPic&gt;.Stylo](../WDLang1/1000024636.md) sont ignorées. Le trait a donc pour épaisseur 1. 
Si ce paramètre n'est pas précisé, la couleur du trait :

- correspond à la couleur précisée dans le dernier appel à la fonction [&lt;Variable WDPic&gt;.Stylo](../WDLang1/1000024636.md),

- est identique à la couleur utilisée pour le fond si la fonction [&lt;Variable WDPic&gt;.Stylo](../WDLang1/1000024636.md) n'a pas été utilisée.

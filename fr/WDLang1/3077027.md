
## ThreadPriorité (Fonction)

***En anglais : ThreadPriority***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie ou modifie le niveau de priorité d'un thread.




<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Récupérer le niveau de priorité d'un thread

`<Résultat> = ThreadPriorité(<Nom du thread>)`
---

**`<Résultat> : (Constante)`**



- -1 si le thread n'existe pas

- Niveau de priorité actuel du thread : 
	


|   |   |
| --- | --- |
| *PrioritéBasse* | Priorité basse. |
| *PrioritéElevée* | Priorité élevée. |
| *PrioritéNormale* | Priorité normale. |






**`<Nom du thread> : (Chaîne de caractères)`**

Nom du thread dont on veut connaître la priorité. Ce nom est donné lors de l'exécution du thread (fonction [ThreadExécute](../WDLang1/3077024.md)).  


<a name="SYNTAXE2"></a>

### Modifier le niveau de priorité d'un thread

`<Résultat> = ThreadPriorité(<Nom du thread> , <Priorité>)`
---

**`<Résultat> : (Constante)`**

Niveau de priorité du thread avant l'exécution de la fonction :


|   |   |
| --- | --- |
| *PrioritéBasse* | Priorité basse. |
| *PrioritéElevée* | Priorité élevée. |
| *PrioritéNormale* | Priorité normale. |



**`<Nom du thread> : (Chaîne de caractères)`**

Nom du thread dont la priorité doit être modifiée. Ce nom est donné lors de l'exécution du thread (fonction [ThreadExécute](../WDLang1/3077024.md)).

**`<Priorité> : (Constante)`**

Nouveau niveau de priorité du thread :


|   |   |
| --- | --- |
| *PrioritéBasse* | Priorité basse. |
| *PrioritéElevée* | Priorité élevée. |
| *PrioritéNormale* | Priorité normale. |

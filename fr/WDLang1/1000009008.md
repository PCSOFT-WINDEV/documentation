
## PiloteOuiNon (Fonction)

***En anglais : EmulateYesNo***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Pilote le prochain appel de la fonction [OuiNon](../WDLang1/3021005.md), puis clique automatiquement sur le bouton indiqué en paramètre.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Fermeture automatique de la prochaine boîte de dialogue OuiNon
// La réponse sur lequel le clic aura lieu est Oui
PiloteOuiNon(Oui)

// Traitement 
// ...

SELON Confirmer("Souhaitez-vous enregistrer les modifications ?")
	CAS Oui
	// Enregistrement des modifications
	CAS Non
	// Annulation des modifications
FIN
// Le message est fermé automatiquement, par un clic sur Oui
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`PiloteOuiNon(<Bouton>)`
---

**`<Bouton> : (Constante de type entier)`**

Indique le bouton de la fenêtre OuiNon sur lequel le clic doit avoir lieu :


|   |   |
| --- | --- |
| *Non* | Clic sur le bouton [NON] |
| *Oui* | Clic sur le bouton [OUI] |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Vérification de la fonction
<a name="verification_fonction_ELTPARAGRAPHE000055"></a>L'existence d'une fonction de pilotage ([PiloteAvertissement](../WDLang1/1000009001.md), [PiloteErreur](../WDLang1/1000009005.md), [PiloteInfo](../WDLang1/1000009006.md), ...) est vérifiée lors de l'appel de la fonction de dialogue associée ([Avertissement](../WDLang1/3021009.md), [Erreur](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ...).


<a name="NOTE0_2"></a>




### Différence de type de message
<a name="difference_type_message_ELTPARAGRAPHE000082"></a>L'ordre des fonctions [PiloteXXX](../WDLang1/1000009002.md) doit respecter l'ordre d'affichage des fonctions [Erreur](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ... Dans le cas contraire, une erreur fatale sera déclenchée.

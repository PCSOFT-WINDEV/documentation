


## &lt;Organigramme&gt;.Affiche (Fonction)

***En anglais : &lt;Organization Chart&gt;.Display***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Réinitialise le contenu d'un champ Organigramme à partir de sa source de données (fichier de données HFSQL par exemple). Tous les éléments sont supprimés du champ Organigramme et sont re-créés à partir de la source de données. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajout d'un élément dans un fichier de données HFSQL
HAjoute(FIC_COLLABORATEURS)
// Réinitialisation du champ Organigramme
ORGA_MonOrganigramme.Affiche()
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Organigramme>.Affiche()`
---

**`<Champ Organigramme> : (Nom de champ)`**

Nom du champ Organigramme à manipuler. Le champ Organigramme doit être lié à une source de données. Dans le cas contraire (par exemple si le champ est rempli par programmation), la fonction **&lt;Organigramme&gt;.Affiche** n'a aucun effet.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si la source de données liée au champ Organigramme est une requête, cette requête est réinitialisée. 

- La position dans le champ Organigramme n'est pas modifiée.


## PiloteInfo (Fonction)

***En anglais : EmulateInfo***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Pilote le prochain appel de la fonction [Info](../WDLang1/3021011.md), puis clique automatiquement sur le bouton "OK".
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Fermeture automatique de la prochaine boîte de dialogue Info
PiloteInfo()

// Traitement 
// ...

Info("L'enregistrement a été validé.")
// Le message est fermé automatiquement
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`PiloteInfo()`
---



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Vérification de la fonction
<a name="verification_fonction_ELTPARAGRAPHE000034"></a>L'existence d'une fonction de pilotage ([PiloteAvertissement](../WDLang1/1000009001.md), [PiloteErreur](../WDLang1/1000009005.md), **PiloteInfo**, ...) est vérifiée lors de l'appel de la fonction de dialogue associée ([Avertissement](../WDLang1/3021009.md), [Erreur](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ...).


<a name="NOTE0_2"></a>




### Différence de type de message
<a name="difference_type_message_ELTPARAGRAPHE000061"></a>L'ordre des fonctions [PiloteXXX](../WDLang1/1000009002.md) doit respecter l'ordre d'affichage des fonctions [Erreur](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ... Dans le cas contraire, une erreur fatale sera déclenchée.

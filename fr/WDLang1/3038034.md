
## FenEnExécution (Fonction)

***En anglais : WinInput***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie la fenêtre contenant le code du WLangage en cours d'exécution.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Nom de la fenêtre dont le code est en cours d'exécution
ResFenEnCoursExéc = FenEnExécution()
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = FenEnExécution()`
---

**`<Résultat> : (Chaîne de caractères)`**



- Nom (ou alias) de la fenêtre si le code en cours d'exécution est :

	- un code de fenêtre,

	- un code de champ,

	- une procédure locale à la fenêtre.




- Chaîne vide ("") si le code en cours d'exécution est :

	- un code associé au projet,

	- une procédure globale,

	- un code d'initialisation ou de terminaison d'une classe,

	- une méthode d'objet.  









<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Fenêtre en cours d'exécution
<a name="fenetre_cours_execution_ELTPARAGRAPHE000086"></a>

- La fonction **FenEnExécution** renvoie une valeur invariante, quelle que soit la façon d'appeler le code contenant cette fonction.

- La fonction **FenEnExécution** ne dépend ni du mode d'ouverture de la fenêtre, ni du type de la fenêtre (modale, non modale, mère MDI ou fille MDI).

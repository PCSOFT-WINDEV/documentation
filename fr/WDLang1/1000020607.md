


## EnModeSimulateurWinRT (Fonction)

***En anglais : InWinRTSimulatorMode***




	<B><B>Avertissement</B></B>

	A partir de la version <B><B>18 (46)</B></B>,  utilisez la fonction [EnModeSimulateurWindowsStoreApp](../WDLang1/1000020696.md).

<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie si le code est exécuté dans le simulateur WinRT (simulateur pour les applications en mode Windows Store apps) ou non.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code exécuté dans le simulateur WinRT ?
SI EnModeSimulateurWinRT() = Vrai ALORS
	// Code spécifique pour le simulateur
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = EnModeSimulateurWinRT()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté dans le simulateur WinRT, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.







## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000166"></a>La fonction **EnModeSimulateurWinRT** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnModeSimulateurWinRT** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées.


### Exécution des traitements contenant des erreurs de compilation
<a name="execution_des_traitements_contenant_des_erreurs_compilation_ELTPARAGRAPHE000182"></a>Par défaut, le mécanisme de sécurité du WLangage se déclenche uniquement si la ligne contenant l'erreur de compilation est exécutée.

Par exemple, si une fonction interdite dans le produit en cours est présente dans une boucle SI, le mécanisme de sécurité du WLangage se déclenchera uniquement si la ligne contenant cette fonction s'exécute.


## Constantes de sérialisation
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Les constantes de sérialisation sont les suivantes :

| Constante | Description | Fonction |
| --- | --- | --- |
| **psdBinaire** | Sérialisation binaire. | [Désérialise](../WDLang1/3013066.md)<br>[Sérialise](../WDLang1/3013065.md) |
| **psdBinaireFormat16** | Sérialisation binaire au format compatible 16. | [Désérialise](../WDLang1/3013066.md)<br>[Sérialise](../WDLang1/3013065.md) |
| **psdJSON** | Sérialisation au format JSON. | [Désérialise](../WDLang1/3013066.md)<br>[Sérialise](../WDLang1/3013065.md) |
| **psdMiseEnForme** | Sérialisation au format JSON avec retours à la ligne et indentation. | [Sérialise](../WDLang1/3013065.md)<br>[VariantVersJSON](../WDLang1/1000021051.md) |
| **psdXML** | Sérialisation XML avec référence aux sous-objets. Ce type de sérialisation permet d'utiliser le format XML comme moyen de stockage et d'échange entre des applications écrites en WLangage. | [Désérialise](../WDLang1/3013066.md)<br>[Sérialise](../WDLang1/3013065.md) |
| **psdXML_AncienFormat** | Jusqu'à la version 100050, l'utilisation de la constante psdXML ne générait pas un fichier XML correct. Le format généré posait un problème essentiellement sur les postes Mobiles. Ce problème a été corrigé.<br><br>Un fichier XML créé avec une version supérieure à la version 100050 n'est pas lisible par un programme compilé avec une version antérieure. Pour conserver le fonctionnement de la version 100050, il suffit d'utiliser la constante psdXML_AncienFormat. | [Désérialise](../WDLang1/3013066.md)<br>[Sérialise](../WDLang1/3013065.md) |
| **psdXMLAgrégé** | Sérialisation XML avec agrégation directe des sous-objets. Ce type de sérialisation permet de générer rapidement un fichier XML dans un format classique, utilisé pour des échanges avec d'autres systèmes. | [Désérialise](../WDLang1/3013066.md)<br>[Sérialise](../WDLang1/3013065.md) |

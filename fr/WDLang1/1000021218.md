


## ThreadDemandeArrêt (Fonction)

***En anglais : ThreadRequestStop***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Envoie une demande d'arrêt à un thread.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ThreadExécute("T1", threadNormal, ProcédureThread)
ThreadDemandeArrêt("T1")
```
<a name="Exemple2"></a>

```wl
// Construit un thread
t est un Thread(Proc, ("Premier paramètre", 2))
// Déclenche l'exécution du thread
ThreadExécute(t)
...
ThreadDemandeArrêt(t)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Demander l'arrêt d'un thread identifié par son nom

`ThreadDemandeArrêt(<Nom du thread>)`
---

**`<Nom du thread> : (Chaîne de caractères)`**

Nom du thread à arrêter. Ce nom est donné lors de l'exécution du thread (fonction [ThreadExécute](../WDLang1/3077024.md)). 


<a name="SYNTAXE2"></a>

### Demander l'arrêt d'un thread identifié par une variable de type Thread

`ThreadDemandeArrêt(<Thread>)`
---

**`<Thread> : (Variable de type Thread)`**

Nom de la variable de type [Thread](../WDLang1/1000024879.md) correspondant au thread à arrêter. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le thread ne sera pas arrêté automatiquement. C'est au code du thread de vérifier si l'arrêt est demandé grâce à la fonction [ThreadArrêtDemandé](../WDLang1/1000021219.md) et de se terminer proprement.

- La fonction est non bloquante et n'attend pas l'arrêt du thread. Il est nécessaire d'utiliser la fonction [ThreadAttend](../WDLang1/3077020.md) pour attendre l'arrêt effectif du thread.


## fRepParent (Fonction)

***En anglais : fParentDir***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le chemin du répertoire parent du répertoire spécifié.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Affiche le répertoire parent du répertoire en cours
Trace(fRepParent(fRepEnCours()))
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = fRepParent(<Répertoire de base> [, <Options>])`
---

**`<Résultat> : (Chaîne de caractères ANSI ou UNICODE)`**

Peut correspondre à : 

- Chemin du répertoire parent de &lt;Répertoire de base&gt;.

- Chaîne vide ("") si le &lt;Répertoire de base&gt; n'a pas de parent. 

- ".." si le &lt;Répertoire de base&gt; vaut ".". 

- "..\\.." si le &lt;Répertoire de base&gt; vaut "..". 




**Remarque** : &lt;Résultat&gt; est du même type que le paramètre &lt;Répertoire de base&gt;. Par exemple, si &lt;Répertoire de base&gt; est une chaîne de caractères ANSI, &lt;Résultat&gt; sera une chaîne de caractères ANSI.

**`<Répertoire de base> : (Chaîne de caractères ANSI ou UNICODE)`**

Chemin de répertoire de base pour lequel le répertoire parent est recherché. Ce chemin peut être absolu ou relatif, local ou UNC.



![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Le chemin du répertoire correspond à un chemin relatif par rapport au répertoire en cours. Le répertoire en cours correspond au répertoire des données de l'application. 
**Rappel** : Seuls les répertoires présents dans le répertoire des données associé à l'application peuvent être manipulés.



**`<Options> : (Constante optionnelle de type Entier)`**

Options prises en compte : 



|   |   |
| --- | --- |
| *fCheminRelatif* | Le chemin renvoyé sera toujours un chemin relatif (même si &lt;Chemin de base&gt; correspond à un chemin absolu). |


![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Ce paramètre n'est pas pris en compte.

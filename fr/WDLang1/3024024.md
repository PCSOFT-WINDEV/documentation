
## NumériqueVersChaîne (Fonction)

***En anglais : NumToString***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Transforme un numérique (entier, réel ou monétaire) en une chaîne de caractères selon le format spécifié.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
NumériqueVersChaîne(123)	// Renvoie "123"
NumériqueVersChaîne(1.23)	// Renvoie "1.23"
NumériqueVersChaîne(12345.5, "+10.2f") // Renvoie " +12345.50"
NumériqueVersChaîne(12345.5, "-+10.2f") // Renvoie "+12345.50 "
NumériqueVersChaîne(12345.5, "+010.2f") // Renvoie "+012345.50"
NumériqueVersChaîne(12345.5, "010.2f") // Renvoie "0012345.50"

NumériqueVersChaîne(12345.5, "10.2fS")
// Renvoie " 12 345.50" (le séparateur des milliers est un " ")

NumériqueVersChaîne(12345.5, "10,2fS")
// Renvoie " 12 345,50" (le séparateur décimal est une ",")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = NumériqueVersChaîne(<Nombre> [, <Format>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Nombre exprimé sous forme de chaîne de caractères en fonction du format choisi. 
Si le paramètre &lt;Nombre&gt; est une chaîne, cette chaîne est restituée telle quelle. Le format est ignoré.



**`<Nombre> : (Entier ou réel)`**

Numérique à convertir sous forme de chaîne de caractères (selon le format spécifié).

**`<Format> : (Chaîne de caractères optionnelle ou constante)`**



- Format de la chaîne de caractères, exprimé sous la forme : 
	
	```txt
	"[<Cadrage>][<Signe>][<Zéro>][<Taille>][<.Précision>]<Type>[<Millier>]"
	```

	
	Pour plus de détails, consultez [les remarques](#NOTE0_1). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Constante permettant de définir un format système :
	


|   |   |
| --- | --- |
| *maskMonétaireSystème* | Utilisation du masque monétaire utilisé par le système. |
| *maskNombreSystème* | Utilisation du masque numérique utilisé par le système. |

<a name="NOTE0_1"></a>

## Remarques




### Format détaillé de la chaîne de caractères à obtenir
<a name="format_detaille_chaine_caracteres_obtenir_ELTPARAGRAPHE000113"></a>Pour convertir un numérique en une chaîne, il est nécessaire de spécifier le format de la chaîne de caractères à obtenir. Ce format est exprimé sous la forme : 


```txt
"[<Cadrage>][<Signe>][<Zéro>][<Taille>][<.Précision>]<Type>[<Millier>]"
```




Les différentes valeurs possibles sont les suivantes : 


|   |   |
| --- | --- |
| &lt;Cadrage&gt; | Si ce paramètre est spécifié, justifie le nombre à gauche (&lt;Cadrage&gt; = "-"). <br>Par défaut, le nombre est justifié à droite. |
| &lt;Signe&gt; | Si ce paramètre est spécifié, force le signe pour les chiffres positifs (&lt;Signe&gt; = "+"). Ce signe apparaîtra à gauche du premier chiffre significatif. <br>Par défaut, seuls les chiffres négatifs ont un signe "-" à gauche du premier chiffre significatif.<br>Cas particulier : si ce paramètre est spécifié et si le nombre vaut 0, le signe "+" sera affiché. De même si un nombre négatif devient égal à 0. |
| &lt;Zéro&gt; | Si ce paramètre est spécifié (&lt;Zéro&gt; ="0"), le nombre sera complété par des zéro à gauche du premier chiffre significatif si la partie significative du nombre est plus petite que la taille demandée et si le nombre est justifié à droite. <br>Par défaut, le nombre n'est pas complété à gauche par des "0".<br>Par exemple : NumériqueVersChaîne(123,"05d") donnera "00123". |
| &lt;Taille&gt; | Fixe la largeur totale de la chaîne résultat. La largeur totale correspond aux éléments suivants : signe, séparateur de milliers, point décimal, exposant, parties entières et décimales du nombre. Cette taille doit être inférieure à 100. <br>Si la taille indiquée est trop petite pour contenir la partie significative du nombre, cette taille sera ignorée. <br>Par défaut, le nombre aura la taille de sa partie significative (il ne pourra pas être complété par des zéro). Idem si &lt;Taille&gt; = 0. <br>Si la taille indiquée est plus grande que la partie significative du nombre :<br><br>- la chaîne résultat sera complétée par des espaces à droite si le nombre est justifié à gauche,<br><br>- la chaîne résultat sera complétée par des espaces ou zéros à gauche si le nombre est justifié à droite.<br><br><br><br><br> |
| &lt;.Précision&gt; | Fixe le séparateur décimal (caractères "." ou ",") et le nombre de décimales (uniquement pour les &lt;Types&gt; flottant ("f") et exposant ("e")). <br><br>**Le séparateur décimal** : <br><br>- Si le séparateur des décimales est une virgule, le séparateur des décimales sera celui défini : <br><br>	- dans les paramètres linguistiques de Windows.<br><br><br><br><br>- Si le séparateur des décimales indiqué est un point, le séparateur décimal utilisé sera un point.  <br><br><br><br><br>Remarques : <br><br>- La fonction [ChangeSéparateur](../WDLang1/3054004.md) n'a pas d'effet sur ce séparateur. En effet, cette fonction agit uniquement sur le masque d'affichage des numériques dans les champs. <br><br><br><br><br>**Nombre de décimales** : <br><br>- Le nombre de décimal doit être inférieur à 100. <br><br>- Si le nombre de décimales du nombre est plus grand que &lt;Précision&gt;, la partie décimale sera arrondie au nombre de décimales demandées.<br><br>- Si le nombre de décimales du nombre est plus petit que &lt;Précision&gt;, la partie décimale sera complétée par des "0" à droite.<br><br>- La valeur de &lt;Précision&gt; peut être plus grande que &lt;Taille&gt;. Dans ce cas &lt;Taille&gt; est ignoré.<br><br><br><br><br> |
| &lt;Type&gt; | Type du résultat, paramètre obligatoire : <br><br>- d : **Entier** : <br>	Un nombre réel sera arrondi à l'entier le plus proche.<br><br>- e : **Notation exponentielle** :<br>	Si &lt;Précision&gt; n'est pas indiqué, 6 chiffres par défaut.<br><br>- f : **Notation décimale** :<br>	Si &lt;Précision&gt; n'est pas indiqué, 6 chiffres par défaut.<br><br>- x et X : **Notation hexadécimale** (X pour des lettres majuscules) :<br>	Seule la partie entière du nombre est prise en compte (pas d'arrondi).<br>	Cas particulier :<br>	1. Taille non précisée et nombre négatif : 20 caractères hexadécimaux pour un monétaire, 8 caractères pour tous les autres numériques.<br>	2. Nombre négatif : la taille de la chaîne résultat sera au minimum la taille permettant d'indiquer le bit de signe. <br>	Par exemple : -32767 dans le format "3X" donnera "8001"<br><br>- o : **Notation octale** : <br>	Seule la partie entière du nombre est prise en compte (pas d'arrondi). <br>	Cas particulier :<br>	1. Taille non précisée et nombre négatif : 27 caractères pour un monétaire, 11 caractères pour tous les autres numériques.<br>	2. Nombre négatif : la taille de la chaîne résultat sera au minimum la taille permettant d'indiquer le bit de signe. <br>	Par exemple : -32767 dans le format "3o" donnera "700001"<br><br><br> |
| &lt;Millier&gt; | Séparateur de milliers (&lt;Millier&gt; = "S" ou "s"). Les chiffres de la partie entière du nombre seront regroupés par 3 et séparés par le caractère séparateur de milliers défini dans Windows. Si le signe est présent, il sera toujours immédiatement à gauche du premier chiffre significatif. Il ne peut pas y avoir de séparateur des milliers entre le signe et le premier chiffre.<br><br><br> |

### Limites
<a name="limites_ELTPARAGRAPHE000268"></a>

- Format "f" donne un résultat correct pour les nombres allant de -1e+17 à 1e+17.

- Formats "x", "d" ou "o" : la partie entière ne doit pas dépasser 99 999 999 999 999 999.

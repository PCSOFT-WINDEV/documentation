
## ImageInfoFerme (Fonction)

***En anglais : ImageInfoClose***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ferme le fichier image (au format JPEG ou TIFF) utilisé par la fonction [ImageInfoValeur](../WDLang1/3091003.md) et/ou la fonction [ImageInfoListe](../WDLang1/3091002.md) et libère l'image de la mémoire.

**Remarques** : 

- Cette fonction n'a aucun effet si l'image utilisée par la fonction [ImageInfoValeur](../WDLang1/3091003.md) et/ou la fonction [ImageInfoListe](../WDLang1/3091002.md) provient d'un champ Image, d'un mémo image ou d'une chaîne binaire.

- Pour manipuler les marqueurs Exif d'une image, il est également possible d'utiliser la propriété **Exif** des variables de type [Image](../WDLang1/1000019650.md). Cette propriété peut être utilisée pour les applications Android. 



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
CheminImg est une chaîne = "C:\Mes Images\MonImage.JPG" 
// Connaître la date de réalisation de cette image 
Info("Date de réalisation : " + ImageInfoValeur(CheminImg, imgDate))
// Fermeture du fichier image 
ImageInfoFerme(CheminImg)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`ImageInfoFerme(<Fichier image>)`
---

**`<Fichier image> : (Chaîne de caractères)`**

Nom et chemin complet (ou relatif) du fichier image à fermer (260 caractères maximum). Un chemin de type UNC peut être utilisé.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La fonction **ImageInfoFerme** ne retourne pas de code d'erreur. Pour savoir si cette fonction a généré une erreur, utilisez la variable [ErreurDétectée](../WDLang1/3087001.md). En cas d'erreur, il est possible de connaître le détail de l'erreur grâce à la fonction [ErreurInfo](../WDLang1/3013008.md).

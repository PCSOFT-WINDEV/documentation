
## EnMode64bits (Fonction)

***En anglais : In64bitMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Identifie si le code est exécuté en mode 64 bits ou non.  




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code exécuté en 64 bits ? 
SI EnMode64Bits() = Vrai ALORS
	// Code spécifique pour une exécution 64 bits
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = EnMode64Bits()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le code est exécuté depuis un traitement exécuté en mode 64 bits, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  










<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Disponibilité de la fonction
<a name="disponibilite_fonction_ELTPARAGRAPHE000069"></a>La fonction **EnMode64bits** est disponible dans tous les produits pour permettre l'écriture d'un code multi-produits. Des fonctions ou procédures utilisant la fonction **EnMode64bits** peuvent être partagées entre plusieurs produits, sans que des erreurs de compilation soient générées. 
<a name="NOTE0_2"></a>


### Exécution des traitements contenant des erreurs de compilation
<a name="execution_des_traitements_contenant_des_erreurs_compilation_ELTPARAGRAPHE000084"></a>Par défaut, le mécanisme de sécurité du WLangage se déclenche uniquement si la ligne contenant l'erreur de compilation est exécutée.

Par exemple, si une fonction interdite dans le produit en cours est présente dans une boucle SI, le mécanisme de sécurité du WLangage se déclenchera uniquement si la ligne contenant cette fonction s'exécute.

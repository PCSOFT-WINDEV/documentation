


## TélémétrieAjoutePassage (Fonction)

***En anglais : TelemetryAddPassage***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Signale l'exécution du code au niveau d'un capteur de passage.

Il est ainsi possible d'établir des statistiques d'utilisation d'une fonctionnalité de l'application : 

- nombre total ou pourcentage d'utilisateurs qui ont utilisé la  fonctionnalité sur une période, 

- nombre minimal, moyen ou maximal  d'utilisateurs qui ont utilisé la  fonctionnalité chaque jour, ...





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Un capteur de passage "AnnulationCommande" a été défini dans l'éditeur.
// A chaque annulation de commande, on enregistre l'utilisation de la fonctionnalité 
// par l'utilisateur
TélémétrieAjoutePassage(AnnulationCommande)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`TélémétrieAjoutePassage(<Capteur> [, <Valeur de la catégorie>])`
---

**`<Capteur> : (Chaîne de caractères)`**

Nom du capteur de passage défini sous l'éditeur. 

**`<Valeur de la catégorie> : (Chaîne de caractères optionnelle)`**

Valeur pour la catégorie définie sous l'éditeur.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Les capteurs doivent être définis dans une télémétrie personnalisée.

- Si le capteur n'est pas un capteur de passage, une erreur WLangage sera affichée.

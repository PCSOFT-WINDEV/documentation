
## Comment ouvrir une page ? 
			

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>
Il existe plusieurs fonctions pour afficher une page : 

- [ PageAffiche](#NOTE1_1),

- [ PageActualise](#NOTE2_1),

- [ PageUtilise](#NOTE3_1),

- [ PageAfficheDialogue](#NOTE4_1),

- [ PopupAffiche](#NOTE5_1),

- [ CelluleAfficheDialogue](#NOTE6_1).







<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Fonction PageAffiche
<a name="fonction_pageaffiche_ELTTEXTE000219"></a>
La fonction [PageAffiche](../WDLang2/3058008.md) permet d'afficher une page dans un navigateur. 

- Dans un premier temps, le contexte de la page est créé ou recréé sur le serveur. 

- La page est ensuite envoyée à l'internaute pour être affichée dans son navigateur. 




Cette fonction est utilisée pour gérer l'enchaînement des pages d'un site Web. La page affichée remplace la page précédente dans le navigateur.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Fonction PageActualise
<a name="fonction_pageactualise_ELTTEXTE000243"></a>
La fonction [PageActualise](../WDLang2/3058011.md) permet d'afficher une page dans un navigateur. 

Contrairement à la fonction [PageAffiche](../WDLang2/3058008.md), le contexte de la page n'est pas recréé sur le serveur. Le contexte existant sur le serveur est réutilisé pour préparer la page sur le serveur avant que celle-ci ne soit envoyée à l'internaute et affichée dans son navigateur.

Le contexte de la page qui a déjà été créé sur le serveur est réutilisé par la fonction [PageAffiche](../WDLang2/3058008.md).

Il est aussi possible de créer un contexte de page directement sur le serveur par les fonctions de gestion des contextes ([ContexteOuvre](../WDLang2/3058004.md), etc.). 

**Cette méthode de programmation est assez complexe et nécessite une connaissance approfondie de la programmation en WLangage.**

La page affichée remplace la page précédente dans le navigateur.

## Fonction PageUtilise
<a name="fonction_pageutilise_ELTTEXTE000267"></a>
La fonction [PageUtilise](../WDLang2/3058016.md) permet d'afficher une page dans un navigateur. 

- Dans un premier temps, le contexte de la page est créé ou recréé sur le serveur. 

- La  page est ensuite envoyée à l'internaute et affichée dans son navigateur. 




Cette fonction supprime tous les précédents contextes de page sur le serveur. 
Cette fonction donne l'impression que le site vient de démarrer en affichant la première page.

## Fonction PageAfficheDialogue
<a name="fonction_pageaffichedialogue_ELTTEXTE000291"></a>
La fonction [PageAfficheDialogue](../WDLang2/1000017124.md) permet d'afficher une page dans un navigateur par dessus une page précédente.

- Dans un premier temps, le contexte de la page est créé ou recréé sur le serveur. 

- La page est ensuite envoyée à l'internaute et affichée dans son navigateur par dessus la page précédente.




Cette page a généralement une taille plus petite que les pages normales du site. Tout ce qui se trouve autour de la page est plus ou moins opaque afin de gérer un effet de transparence par dessus la page précédente.

Ce type d'affichage de page permet de conserver une partie de la page précédente visible et de :

- compléter un affichage,

- faire une saisie complémentaire, ...




Remarque : Il faut prévoir dans cette page un bouton de fermeture afin de revenir en saisie dans la page précédente. La fonction [PageFermeDialogue](../WDLang2/1000017166.md) permet de gérer la fermeture de ce type de page.

Ce type de page est créé dans l'éditeur de WEBDEV de la même manière qu'une page normale.

## Fonction PopupAffiche
<a name="fonction_popupaffiche_ELTTEXTE000315"></a>
La fonction [PopupAffiche](../WDLang2/1000019788.md) permet d'afficher une page dans un navigateur par dessus une  page précédente.

- Dans un premier temps, le contexte de la page est créé ou recréé sur le serveur. 

- La page est ensuite envoyée à l'internaute et affichée dans son navigateur par dessus la précédente.




Cette page a généralement une taille plus petite que les pages normales du site. Tout ce qui est autour de la page est plus ou moins opaque afin de gérer un effet de transparence par dessus la page précédente.

Ce type de page est créé dans l'éditeur de WEBDEV. Elle est générée dans la même page que la page dans laquelle elle va s'afficher. 

Il faut créer une page spéciale de type Popup pour pouvoir l'afficher par cette fonction.

## Fonction CelluleAfficheDialogue
<a name="fonction_celluleaffichedialogue_ELTTEXTE000339"></a>
La fonction [CelluleAfficheDialogue](../WDLang2/1000017139.md) est "l'ancêtre" de la fonction [PopupAffiche](../WDLang2/1000019788.md). Le comportement est similaire à une page de type Popup.

Il n'y a pas de création de nouvelle page. Il faut créer dans la page en cours une cellule (champ Cellule). Les cellules sont affichées ou non en superposition.

Ce mode d'affichage est moins pratique que les pages Popup. Il est préférable d'utiliser des pages de type Popup et d'utiliser la fonction [PopupAffiche](../WDLang2/1000019788.md).


## Comment donner le focus à un champ d'une fenêtre ?
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Principe
<a name="principe_ELTTEXTE000103"></a>
Pour se positionner en saisie (ou donner le focus) à un champ, vous disposez de 2 méthodes :

- Utilisation de la fonction [DonneFocusEtRetourUtilisateur](../WDLang1/1410088107.md) ou [DonneFocus](../WDLang1/3025001.md). 

- Utilisation de la fonction [EcranPremier](../WDLang1/3025039.md). 




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Première méthode : fonction DonneFocusEtRetourUtilisateur ou DonneFocus
<a name="premiere_methode_fonction_donnefocusetretourutilisateur_donnefocus_ELTTEXTE000127"></a>
La fonction [DonneFocusEtRetourUtilisateur](../WDLang1/1410088107.md) (ou la fonction [DonneFocus](../WDLang1/3025001.md)) donne le focus au champ indiqué et permet de revenir directement en saisie sur la fenêtre **sans exécuter le code WLangage placé après l'appel de la fonction**.

Exemple de code : 


```wl
SI SAI_CODE = "" ALORS	
	DonneFocusEtRetourUtilisateur(SAI_CODE)
	// Code non exécuté
SINON
	DonneFocusEtRetourUtilisateur(SAI_INTITULE)
	// Code non exécuté
FIN
// Code non exécuté
```




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Seconde méthode : fonction EcranPremier
<a name="seconde_methode_fonction_ecranpremier_ELTTEXTE000151"></a>
La fonction [EcranPremier](../WDLang1/3025039.md) permet de mémoriser le champ qui doit prendre le focus. Le code WLangage suivant l'appel de la fonction continue de s'exécuter. Il est ainsi possible de changer le champ sur lequel se positionner avant de retourner dans la saisie sur la fenêtre.

Exemple de code : 


```wl
// En fonction des retours des traitements 1 et 2 
// on se positionne à la fin du traitement sur le champ SAI_QTE ou SAI_CODEREMISE

// Exécution traitement partie 1
...
SI nResPartie1 = 0 alors EcranPremier(SAI_QTE)
...
// Exécution traitement partie 2
...
SI bResPartie2 = Faux alors EcranPremier(SAI_CODEREMISE)
...
```

## Menu déroulant
<a name="menu_deroulant_ELTTEXTE000215"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Les menus présentés sous forme de menu déroulant existent depuis les débuts de Windows. C'est une façon simple et facile à mettre en place dans une application. Ce type de menu est presque tout le temps présent sur la première fenêtre du programme.

**Sous WINDEV**, ce type de menu est positionné en haut de la fenêtre, sous la barre de titre et occupe un simple bandeau, soit très peu de place. Par contre, ce type d'interface est plutôt vieillissant et a été depuis remplacé par les menus de type Ruban.

**Sous WEBDEV**, ce type de menu peut être positionné en haut de la page ou bien sur le coté (généralement à gauche). Positionné dans le modèle de pages, il est affiché dans chaque page du site. Le champ Chemin de navigation est souvent associé à un menu en Web afin de mémoriser le cheminement de l'utilisateur dans le site Web.

Pour plus de détails, consultez : 

- [Menus principaux d'une page](../WDChamp/1010041.md). 

- [Menus principaux des fenêtres](../WDChamp/1010010.md).

## Barre d'icônes
<a name="barre_icones_ELTTEXTE000239"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) La barre d'icônes est toujours associée à un menu déroulant. Il est très rare de voir une barre d'icônes seule.

La barre d'icônes représente graphiquement sous la forme de petits boutons des raccourcis rapides sur les options les plus utilisées du menu déroulant. D'ailleurs, l'icône qui est dessinée est la même que celle qui est associée à l'option équivalente du menu déroulant.

Souvent, la barre d'icônes est composée de plusieurs petites barres d'icônes, afin de classer par thème les actions. Les barres d'icônes peuvent être flottantes, cadrées en haut, en bas, à droite, à gauche.

On voit souvent des barres d'icônes dans des applications multifenêtrages ou MDI. Dans ce cas, la fenêtre principale (qui accueille le menu déroulant et les barres d'icônes) est en fond et sert de support aux fenêtres filles.

## Ruban
<a name="ruban_ELTTEXTE000263"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Le champ Ruban est un concept récent pour organiser les actions d'une application. Il remplace le couple "Menu déroulant / Barres d'icônes". Un ruban est composé de volets et de regroupements. Un regroupement contient des boutons simple ou bouton flèche pour proposer une action ou plusieurs actions.

Ce type d'interface est plus moderne mais plus contraignant car cela demande de s'habituer à une nouvelle organisation pour les utilisateurs qui sont habitués aux menus déroulants.

Pour plus de détails, consultez [Le champ Ruban](../WDChamp/1000021001.md).

## Boutons
<a name="boutons_ELTTEXTE000287"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Il est possible de réaliser des menus avec de simples boutons. 

Ce type d'interface est très adapté dans un environnement tactile (écran tactile - application de type caisse, tablette ou smartphone). Il faut juste faire des gros boutons (boutons dont la taille est supérieure à la normale) car le pointage se fait par le doigt plutôt que le curseur souris.

Avantage de cette méthode : très facile à mettre en place et à créer. Si vous avez besoin de gérer plusieurs niveaux de boutons, il suffit de les répartir dans des fenêtres différentes ou dans des plans différents.

Pour plus de détails, consultez [Le champ Bouton](../WDChamp/1013076.md).

## Liens
<a name="liens_ELTTEXTE000311"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Les liens sont des sortes de boutons relookés. Ce type de champ est plutôt utilisé dans un site Web, mais on peut aussi créer des liens dans une application WINDEV. 

Le mode de fonctionnement rejoint celui des boutons. Entre un bouton ou un lien, il n'y a que très peu de différences, mis à part l'aspect graphique.

Pour plus de détails, consultez [Le champ Lien](../WDChamp/1013166.md).

## Version nécessaire

### Version de WINDEV/WEBDEV nécessaire
<a name="version_windevwebdev_necessaire_ELTPARAGRAPHE000011"></a>**Attention** : Cette version du Connecteur Natif Informix ne fonctionne qu'avec WINDEV et WEBDEV 28.

<a name="NOTE2"></a>

## Remarques générales
<a name="remarques_generales_ELTTEXTE000244"></a>


### Rubriques Unicode
<a name="rubriques_unicode_ELTPARAGRAPHE000022"></a>Les rubriques Unicode ne sont pas gérées par le Connecteur Natif Informix. Il est conseillé d'utiliser des rubriques texte au format Ansi et de réaliser si nécessaire la conversion avec la fonction [UnicodeVersAnsi](../WDLang1/3024025.md).

La fonction [HCréation](../WDLang4/3044255.md) affichera un message d'erreur lors de la création de fichiers contenant des rubriques au format Unicode.
<a name="NOTE2_2"></a>


### Configuration
<a name="configuration_ELTPARAGRAPHE000037"></a>Dans les paramètres de configuration de la couche client Informix, il est conseillé de positionner le paramètre "DELIMIDENT" à "YES".

Ce paramétrage est nécessaire si les noms des tables ou des colonnes de tables contiennent des accents (exemple "ProduitsCommandés").
<a name="NOTE2_3"></a>


### Recherches, filtres et requêtes
<a name="recherches_filtres_requetes_ELTPARAGRAPHE000046"></a>Il n'est pas possible d'effectuer des recherches ou de définir des filtres sur une rubrique utilisée dans la clause HAVING (fonctions **HLitRecherchexx** et [HFiltre](../WDLang4/3044100.md)).

En effet, Informix n'autorise pas l'utilisation d'une rubrique à la fois dans la clause HAVING et dans la clause WHERE.
<a name="NOTE2_4"></a>




### Gestion des dates, des heures et des dateheure
<a name="gestion_des_dates_des_heures_des_dateheure_ELTPARAGRAPHE000059"></a>Le Connecteur Natif Informix utilise les fonctions suivantes pour formater les valeurs des types date, heure et dateheure dans les requêtes envoyées au serveur :

- Date : MDY(MM,DD,YYYY)

- Heure : DATETIME(HH:MM) HOUR TO MINUTE

- Dateheure : DATETIME(YYYY-MM-DD HH:MM:SS.ccm) YEAR TO FRACTION




**Cas particulier des serveurs Informix version 7.23 ou antérieure**
Ces serveurs ne gèrent pas la fonction MDY. Pour contourner cette limitation, le Connecteur Natif Informix peut échanger les dates sous forme de chaînes de caractères au format Y4MD. Dans ce cas, il faut définir sur la machine cliente la variable d'environnement DBDATE à Y4MD avec Setnet32.

**Note** : Si le Connecteur Natif Informix détecte que la version du client est inférieure ou égale à 7.23, il vérifie dans la base de registre que la valeur chaîne DBDATE est bien définie à Y4MD à l'emplacement suivant si la valeur DEFAULT_USER_SETTING est définie à CU :


```txt
HKEY_CURRENT_USER\Software\Informix\Environment
```


sinon à l'emplacement suivant :


```txt
HKEY_LOCAL_MACHINE\Software\Informix\Environment
```

## Spécificités du Connecteur Natif Informix
<a name="specificites_connecteur_natif_informix_ELTTEXTE000286"></a>


### Options de l'analyse
<a name="options_analyse_ELTPARAGRAPHE000084"></a>

- Les options de l'analyse concernant les index full-text ne sont pas gérées par le Connecteur Natif.

- Les options de l'analyse concernant l'indexation partielle des mémos texte ne sont pas gérées par le Connecteur Natif.



<a name="NOTE3_2"></a>


### Enregistrements identiques
<a name="enregistrements_identiques_ELTPARAGRAPHE000092"></a>Le Connecteur Natif Informix n'est pas capable de différencier deux enregistrements identiques (dont les valeurs de toutes les rubriques sont égales, sans tenir compte des rubriques de type mémo).

En cas de modification ou de suppression d'enregistrement, tous les enregistrements identiques seront modifiés ou supprimés.
<a name="NOTE3_3"></a>


### Forcer l'identifiant automatique
<a name="forcer_identifiant_automatique_ELTPARAGRAPHE000101"></a>La constante *hForceIDAuto* n'est pas disponible avec la fonction [HModifie](../WDLang4/3044042.md). Cette constante n'est utilisable qu'avec la fonction [HAjoute](../WDLang4/3044147.md). 
<a name="NOTE3_4"></a>


### Requêtes de type INSERT et Identifiant automatique
<a name="requetes_type_insert_identifiant_automatique_ELTPARAGRAPHE000114"></a>Les requêtes de type INSERT (créées par programmation ou grâce à l'éditeur de requêtes) ne gèrent pas les identifiants automatiques. Pour utiliser un identifiant automatique, utilisez la fonction [HAjoute](../WDLang4/3044147.md) sur la table.
<a name="NOTE3_5"></a>


### Blocages
<a name="blocages_ELTPARAGRAPHE000124"></a>Il n'est PAS possible d'utiliser les blocages avec les fonctions HFSQL dans cette version.
<a name="NOTE3_6"></a>


### Types non supportés
<a name="types_non_supportes_ELTPARAGRAPHE000131"></a>Les rubriques de type ROW, MULTISET et LIST ne sont pas supportées par le Connecteur Natif Informix.
<a name="NOTE3_7"></a>


### Restauration des mémos
<a name="restauration_des_memos_ELTPARAGRAPHE000138"></a>Les mémos ne sont pas restaurés lors de l'utilisation de la fonction [HRetourPosition](../WDLang4/3044137.md) sur les requêtes et dans les états.

**Conséquence** : Dans un champ "Table fichier" basé sur une requête, les mémos n'apparaissent plus : 

- lors d'un clic sur une ligne du champ Table, 

- lors du déplacement de l'ascenseur dans le champ Table.



<a name="NOTE3_8"></a>


### Liaisons
<a name="liaisons_ELTPARAGRAPHE000153"></a>

- La fonction [HListeLiaison](../WDLang4/3044138.md) n'est pas supportée dans cette version.

- Les liaisons entre les tables ne sont pas supportées dans cette version.



<a name="NOTE3_9"></a>


### Alias et caractères accentués
<a name="alias_caracteres_accentues_ELTPARAGRAPHE000164"></a>Dans cette version, les alias (de noms de tables et/ou de colonnes de tables) contenant un caractère accentué ne sont pas supportés.
<a name="NOTE3_10"></a>


### Recherche générique
<a name="recherche_generique_ELTPARAGRAPHE000171"></a>Une recherche générique sur une chaîne de caractères contenant le caractère "\*" peut échouer. Le driver Informix affiche alors une erreur "General Error".

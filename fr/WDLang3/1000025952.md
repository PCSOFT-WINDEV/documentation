
## websocketClient (Type de variable)

***En anglais : websocketClient***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **websocketClient** permet de connaître et définir toutes les caractéristiques avancées d'un client connecté à un serveur de WebSocket. Les caractéristiques de ce client peuvent être définies et modifiées à l'aide de différentes propriétés WLangage. 

**Remarque** : Pour plus de détails sur la déclaration de ce type de variable et l'utilisation des propriétés WLangage, consultez [Déclaration d'une variable](../Motscles/1514032.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupère la liste des clients
tabClients est un tableau de websocketClient = WebSocketListeClientConnecté()
// Pour envoyer un message à chacun
POUR TOUT oClient DE tabClients
	WebSocketEnvoie(oClient, bufMessage)
FIN
```

## Remarques
<a name="NOTE0_1"></a>


### Propriétés spécifiques à la description des variables de type websocketClient
<a name="proprietes_specifiques_description_des_variables_type_websocketclient_ELTPARAGRAPHE000029"></a>Les propriétés suivantes peuvent être utilisées pour manipuler un client connecté à un serveur de WebSocket :

| Nom de la propriété | Type manipulé | Effet |
| --- | --- | --- |
| Connecté | Booléen | <br><br>- <u><u><u><u>Vrai</u></u></u></u> si le client est encore connecté, <br><br>- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.<br><br><br><br><br>**Cette propriété est disponible en lecture seulement**. |
| IP | Chaîne de caractères | Adresse IP du client.<br><br>**Cette propriété est disponible en lecture seulement**. |
| Note | Tous types | Données métier quelconques associées à ce client. Ces données sont écrites par programmation, pour être réutilisés plus tard. La valeur par défaut est une chaîne vide ("").<br><br>Grâce à cette propriété, le contexte global ou un autre client peut obtenir des informations sur ce client.  |

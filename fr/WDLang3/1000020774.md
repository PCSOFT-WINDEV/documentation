
## emailEntête (Type de variable)

***En anglais : emailHeader***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **emailEntête** permet de récupérer ou de fixer les entêtes supplémentaires de l'email. Les caractéristiques d'un entête peuvent être définies et modifiées à l'aide de différentes propriétés WLangage. 

Ce type de variable est utilisé par les variables de type [Email](../WDLang3/1000018713.md). 

**Remarque** : Pour plus de détails sur la déclaration de ce type de variable et l'utilisation des propriétés WLangage, consultez [Déclaration d'une variable](../Motscles/1514032.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
cMonEmail est un Email
cUneEntête est un emailEntête
cUneEntête.Nom = "List-Unsubscribe"
cUneEntête.Valeur = "<mailto:unsubscribe@domain.com>"
Ajoute(cMonEmail.Entête, cUneEntête)
EmailEnvoieMEssage(MaSession, cMonEmail, emailOptionEncodeEntête)
```

## Remarques
<a name="NOTE0_1"></a>


### Propriétés spécifiques à la description des variables de type emailEntête
<a name="proprietes_specifiques_description_des_variables_type_emailentete_ELTPARAGRAPHE000051"></a>Les propriétés suivantes peuvent être utilisées pour manipuler un entête d'email :

| **Nom de la propriété** | **Type manipulé** | **Effet** |
| --- | --- | --- |
| Nom | Chaîne de caractères | Nom de l'entête. |
| Valeur | Chaîne de caractères | Valeur de l'entête. <br><br>**Remarque** : Si la fonction [EmailEnvoieMessage](../WDLang3/3032005.md) utilise la constante *emailOptionEncodeEntête*, la valeur de l'entête sera encodée en base 64. |

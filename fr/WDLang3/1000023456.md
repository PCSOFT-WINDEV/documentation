
## TâcheEnArrièrePlanListe (Fonction)

***En anglais : BackgroundTaskList***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie la liste des tâches en arrière-plan de l'application.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Retourne Vrai si la tâche en arrière-plan de traitement des emails a été ajoutée 
PROCEDURE bTacheTraitementEmailActive
sListeTache est une chaîne = TâcheEnArrièrePlanListe()
POUR TOUTE CHAINE sTache de sListeTache SEPAREE PAR RC
	SI ExtraitChaine(sTache, 2, TAB) = "ProcTraitementEmailEnArrièrePlan" ALORS
		RENVOYER Vrai
	FIN
FIN
RENVOYER Faux
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = TâcheEnArrièrePlanListe()`
---

**`<Résultat> : (Chaîne de caractères)`**



- Liste des tâches en arrière-plan de l'application séparées par RC (Retour chariot). Cette liste est de la forme :
	
	```txt
	<Identifiant> + TAB + <Procédure> + TAB + <Fréquence> + TAB + <Auto> + RC +  ...
	```
 où : 

	- &lt;Identifiant&gt; est un entier correspondant à l'identifiant de la tâche, 

	- &lt;Procédure&gt; est une chaîne de caractères correspondant au nom de la procédure WLangage appelée lors de l'exécution de la tâche, 

	- &lt;Fréquence&gt; est un entier correspondant à la fréquence d'exécution de la tâche (en minutes), 

	- &lt;Auto&gt; est un booléen : 

		- <u><u><u><u>Vrai</u></u></u></u> si la tâche a été ajoutée automatiquement via le paramétrage des automatismes de la procédure (non disponible dans cette version), 

		- <u><u><u><u>Faux</u></u></u></u> si elle a été ajoutée avec la fonction [TâcheEnArrièrePlanAjoute](../WDLang3/1000023454.md).




- Chaîne vide ("") si aucune tâche en arrière-plan n'a été définie pour l'application.


## AppleTrackingEtatAutorisation (Fonction)

***En anglais : AppleTrackingAuthorizationStatus***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de savoir si l'application est autorisée à réaliser un suivi d'activité sur les données de l'utilisateur (iOS uniquement). 

Si la permission de suivi d'activité est autorisée, les données permettant d'identifier l'utilisateur pourront être associées aux données provenant d'autres applications ou sites Web, par exemple à des fins de publicités. 

Cette autorisation est nécessaire pour utiliser la fonction [AppleIdentifiantPublicitaire](../WDLang3/1410087936.md).
<a name="Exemple1"></a>

```wl
SELON AppleTrackingEtatAutorisation()
     CAS ateaIndéterminé
             AppleTrackingDemandeAutorisationAsynchrone(OnTracking)
     CAS ateaAutorisé
             PisteUtilisateur(AppleIdentifiantPublicitaire())
FIN
```


<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = AppleTrackingEtatAutorisation()`
---

**`<Résultat> : (Constante de type Entier)`**

Etat de l'autorisation : 



|   |   |
| --- | --- |
| *ateaAutorisé* | L'utilisateur consent à l'utilisation de données pour suivre l'utilisateur ou l'appareil |
| *ateaIndéterminé* | L'utilisateur n'a pas encore donné son consentement ou son opposition. Dans ce cas, il est possible de demander l'autorisation à l'utilisateur via la fonction [AppleTrackingDemandeAutorisationAsynchrone](../WDLang3/1410087935.md). |
| *ateaInterdit* | L'utilisateur refuse l'utilisation de données pour suivre l'utilisateur ou l'appareil. |
| *ateaRestreint* | L'appareil est géré par un "Profil de gestion des appareils" qui interdit le tracking. |

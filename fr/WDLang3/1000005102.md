
## NotesOuvreConnexion (Fonction)

***En anglais : NotesOpenConnection***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ouvre une connexion avec un serveur Domino. Cette session pourra être fermée avec les fonctions [FermeSession](../WDLang3/3032124.md) ou [EmailFermeSession](../WDLang3/3032006.md).

**Remarques** :

- Pour accéder aux emails, contacts, groupes de contacts, tâches et/ou rendez-vous gérés par Lotus Notes, il est nécessaire de spécifier la base Lotus Notes à utiliser (seconde syntaxe de la fonction [EmailOuvreSessionNotes](../WDLang3/3032121.md)).

- Pour accéder aux documents gérés par Lotus Notes, il est nécessaire de spécifier la base Lotus Notes à utiliser (fonction [NotesOuvreBase](../WDLang3/1000005101.md)).



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouverture de la connexion
IDSession est un entier
IDSession = NotesOuvreConnexion("MotDePasse", "ServeurMars", ...
		"C:\Users\Julie\AppData\Local\Lotus\Notes\Data\user.id")
// Spécification de la base Lotus Notes à utiliser
SI NotesOuvreBase(IDSession, "mail/julie.nsf") <> 0 ALORS
	Info("Session ouverte et base spécifiée")
SINON
	Erreur(ErreurInfo())
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = NotesOuvreConnexion(<Mot de passe> , <Serveur Domino> [, <Chemin du certificat>])`
---

**`<Résultat> : (Entier)`**



- Identifiant de la session. 

- 0 en cas d'erreur (ou <u><u><u><u>Faux</u></u></u></u>). Pour obtenir le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.




**`<Mot de passe> : (Chaîne de caractères)`**

Mot de passe de l'utilisateur. Ce mot de passe est fourni par le fournisseur de services ou par l'administrateur réseau.

**`<Serveur Domino> : (Chaîne de caractères)`**



- Nom du serveur Domino, 

- Chaîne vide ("") si la base est locale.




**`<Chemin du certificat> : (Chaîne de caractères optionnelle)`**

Chemin du certificat permettant d'authentifier l'utilisateur (fichier ID). Ce fichier est délivré par l'administrateur du serveur Domino. Ce fichier est obligatoire pour se connecter au serveur.
Si ce paramètre n'est pas spécifié, le chemin du certificat sera lu dans le fichier de configuration du client Lotus Notes (fichier "Notes.INI"). Le certificat utilisé correspondra alors au dernier certificat de l'utilisateur du client Lotus Notes du poste en cours.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
L'accès à Lotus Notes est possible uniquement sous Lotus Notes version 6.5 ou supérieure.

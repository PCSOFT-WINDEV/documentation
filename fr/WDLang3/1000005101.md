
## NotesOuvreBase (Fonction)

***En anglais : NotesOpenDatabase***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet d'accéder aux documents gérés par Lotus Notes.

Il est nécessaire d'ouvrir préalablement une connexion avec le serveur Domino (fonction [NotesOuvreConnexion](../WDLang3/1000005102.md)).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouverture de la connexion
IDSession est un entier
IDSession = NotesOuvreConnexion("MotDePasse", "ServeurMars", ...
		"C:\Users\Julie\AppData\Local\Lotus\Notes\Data\user.id")
...
// Spécification de la base Lotus Notes à utiliser
SI NotesOuvreBase(IDSession, "mail/julie.nsf") <> 0 ALORS
	Info("Session ouverte et base spécifiée")
SINON
	Erreur(ErreurInfo())
FIN
...
// Fermeture de la base
NotesFermeBase(IDSession)
// Fermeture de la connexion
FermeSession(IDSession)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = NotesOuvreBase(<Identifiant de la session> , <Base Lotus Notes>)`
---

**`<Résultat> : (Entier)`**



- Identifiant de la session ouverte.

- 0 en cas d'erreur (ou <u><u><u><u>Faux</u></u></u></u>). Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.




**`<Identifiant de la session> : (Chaîne de caractères)`**

Identifiant de la session à ouvrir. Cet identifiant est retourné par la fonction [NotesOuvreConnexion](../WDLang3/1000005102.md).

**`<Base Lotus Notes> : (Chaîne de caractères)`**

Nom de la base Lotus Notes contenant les emails de l'utilisateur. Ce nom correspond au chemin du fichier ".NSF".

Si la base Lotus Notes est située sur un serveur Domino, il est nécessaire d'utiliser un chemin relatif.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
L'accès à Lotus Notes est possible uniquement sous Lotus Notes version 6.5 ou supérieure.

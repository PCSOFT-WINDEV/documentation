


## &lt;Variable zipArchive&gt;.FichierEnCours (Fonction)

***En anglais : &lt;zipArchive variable&gt;.CurrentFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nom du fichier archivé en cours de manipulation par les fonctions d'ajout et d'extraction de fichiers. Cette fonction est utilisable sur les archives au format CAB, RAR, WDZ, ZIP, TAR et TGZ (TAR.GZ).








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) 
```wl
Archive est une zipArchive

POUR Indice = 1 A 5
	// Extraction des 5 premiers fichiers de l'archive
	ResExtraitFichier = Archive.ExtraitFichier(Indice)
	...
	// Recherche du nom du fichier en cours d'extraction
	ResFichierEnCours = Archive.FichierEnCours()
	// Affichage du nom du fichier dans la barre de message
	Message(ResFichierEnCours)
	...
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Archive>.FichierEnCours()`
---

**`<Résultat> : (Chaîne de caractères)`**

Fichier en cours de manipulation par les fonctions [&lt;Variable zipArchive&gt;.AjouteFichier](../WDLang3/1000019050.md), [&lt;Variable zipArchive&gt;.AjouteRépertoire](../WDLang3/1000019078.md), [&lt;Variable zipArchive&gt;.ExtraitFichier](../WDLang3/1000019082.md) ou [&lt;Variable zipArchive&gt;.ExtraitTout](../WDLang3/1000019085.md).

**`<Archive> : (Variable de type zipArchive)`**

Nom d'une variable de type [zipArchive](../WDLang3/1000018679.md) correspondant à l'archive à manipuler.

### Erreur pendant les traitements d'ajout et d'extraction de fichiers
<a name="erreur_pendant_les_traitements_ajout_extraction_fichiers_ELTPARAGRAPHE000294"></a>En cas d'erreur pendant les traitements d'ajout ou d'extraction, la fonction **&lt;Variable zipArchive&gt;.FichierEnCours** contient le nom du fichier qui n'a pas pu être lu ou écrit dans l'archive.

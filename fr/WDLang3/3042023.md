


## grAjouteDonnée (Fonction)

***En anglais : grAddData***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute une donnée dans un graphe. 

**Attention** : Pour les types de graphes "avancés", il existe une fonction WLangage spécifique pour ajouter les données (voir paragraphe ["Remarques"](#NOTE0_2)). 

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 










<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajout d'une donnée dans le champ "GRF_MonGraphe". 
// Cette donnée de valeur 10, est ajoutée à la fin de la première série.
grAjouteDonnée(GRF_MonGraphe, 1, 10)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`grAjouteDonnée(<Nom du graphe> , <Numéro de la série> [, <Indice>] , <Valeur>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md). Ce nom doit être entre guillemets. 

- soit au nom du champ Graphe (présent dans l'éditeur de fenêtres, de pages ou d'états). Ce nom ne doit pas être entre guillemets. 




**`<Numéro de la série> : (Entier)`**

Numéro de la série dans laquelle une valeur doit être ajoutée. 
Par exemple, si le champ Graphe affiche 2 courbes, ce paramètre permet d'identifier la courbe pour laquelle la donnée doit être ajoutée. 

Si le numéro spécifié n'existe pas, les séries comprises entre la dernière série existante et le numéro spécifié sont automatiquement créées et initialisées à 0.

**`<Indice> : (Entier optionnel)`**

Indice de la donnée dans la série où une valeur doit être ajoutée. 
Par exemple, pour un graphe de type Courbe, ce paramètre correspond à l'abscisse de la valeur à ajouter. 

Si ce paramètre :

- n'est pas spécifié, la valeur est ajoutée à la fin de la série.

- est spécifié, l'ancienne valeur est remplacée.




**`<Valeur> : (Entier ou réel)`**

Valeur à ajouter. 

Par exemple, pour un graphe de type Courbe, ce paramètre correspond à l'ordonnée de la valeur à ajouter.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Ajout de données
<a name="ajout_donnees_ELTPARAGRAPHE000152"></a>

- Les données ajoutées avec la fonction **grAjouteDonnée** seront prises en compte lors du prochain appel à la fonction [grDessine](../WDLang3/3042026.md).

- Pour un graphe de type Secteur (constante *grSecteur*), seules les données de la première série sont utilisées. Il est impossible de dessiner un graphe de type Secteur si toutes les données de la première série sont nulles.

- Pour les graphes de type "Histogramme empilé", si la valeur de la première série est négative, la barre correspondante est orientée vers le bas. 



<a name="NOTE0_2"></a>


### Ajout de données dans les différents types de graphes
<a name="ajout_donnees_dans_les_differents_types_graphes_ELTPARAGRAPHE000169"></a>

- Pour les graphes de type Nuage de points, il est conseillé d'utiliser la fonction [grNuageAjouteDonnéeXY](../WDLang3/3042032.md).

- Pour les graphes boursiers, il est conseillé d'utiliser la fonction [grBourseAjouteDonnée](../WDLang3/3042009.md).

- Pour les graphes Heatmap, il est conseillé d'utiliser la fonction [grHMAjouteDonnée](../WDLang3/1000021628.md).

- Pour les graphes Secteur Multiniveau, il est conseillé d'utiliser la fonction [grSMNAjouteDonnée](../WDLang3/1000021285.md). 

- Pour les graphes de type Surface, il est conseillé d'utiliser la fonction [grSurfaceAjouteDonnée](../WDLang3/1000020425.md).

- Pour les graphes circulaires de progrès, il est conseillé d'utiliser la fonction [grCPAjouteDonnée](../WDLang3/1410087792.md).

- Pour ajouter une valeur à une donnée d'un graphe, utilisez la fonction [grIncrémenteDonnée](../WDLang3/3042030.md).

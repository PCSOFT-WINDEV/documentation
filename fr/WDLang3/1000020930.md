


## grWaterfallTypeCatégorie (Fonction)

***En anglais : grWaterfallCategoryType***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique le type de la catégorie d'un graphe de type Waterfall. 




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// CA 2017
grAjouteDonnée(GRF_Waterfall, 1, 1, 298000)
grEtiquetteCatégorie(GRF_Waterfall, 1, "CA 2017")

// Ajout des bénéfices
grAjouteDonnée(GRF_Waterfall, 1, 2, 420000)
grEtiquetteCatégorie(GRF_Waterfall, 2, "Bénéfices" + RC + "produits")
grAjouteDonnée(GRF_Waterfall, 1, 3, 210000)
grEtiquetteCatégorie(GRF_Waterfall, 3, "Bénéfices" + RC + "services")

// Coûts
grAjouteDonnée(GRF_Waterfall, 1, 4, -170000)
grEtiquetteCatégorie(GRF_Waterfall, 4, "Coûts fixes")
grAjouteDonnée(GRF_Waterfall, 1, 5, -140000)
grEtiquetteCatégorie(GRF_Waterfall, 5, "Coûts variables")

// CA 2018
grAjouteDonnée(GRF_Waterfall, 1, 6, 0)
grEtiquetteCatégorie(GRF_Waterfall, 6, "CA 2018")
grWaterfallTypeCatégorie(GRF_Waterfall, 6, grSousTotal)

// CA Cumulé
grAjouteDonnée(GRF_Waterfall, 1, 7, 0)
grEtiquetteCatégorie(GRF_Waterfall, 7, "CA cumulé")
grWaterfallTypeCatégorie(GRF_Waterfall, 7, grTotal)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`grWaterfallTypeCatégorie(<Nom du graphe> , <Catégorie> , <Type de la catégorie>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- soit au nom du champ Graphe présent dans l'éditeur de fenêtres, d'états ou de pages.




**`<Catégorie> : (Entier)`**

Numéro de la catégorie à manipuler. 

**`<Type de la catégorie> : (Constante de type Entier)`**

Type de la catégorie : 



|   |   |
| --- | --- |
| *grDéfaut* | Valeurs du graphe, positives ou négatives. |
| *grSousTotal* | Valeurs de type sous-total.<br><br>Si la catégorie correspond à un sous-total, la valeur stockée pour cette catégorie est ignorée. Pour cette catégorie, le graphe Waterfall affiche une barre correspondant à la somme de toutes les catégories précédentes à partir du dernier total ou sous-total. |
| *grTotal* | Valeurs de type total.<br><br>Si la catégorie correspond à un total, la valeur stockée pour cette catégorie est ignorée. Pour cette catégorie, le graphe Waterfall affiche une barre correspondant à la somme de toutes les catégories précédentes à partir du début du graphe. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- **Cas particulier : la première catégorie**
	La première catégorie est toujours une catégorie de type "total", mais la valeur spécifiée est affichée : c'est la valeur de départ. 

- Cette fonction s'applique uniquement aux graphes de type Waterfall.

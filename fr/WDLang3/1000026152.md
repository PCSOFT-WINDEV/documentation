


## zipSélectionFichier (Fonction)

***En anglais : zipSelectFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Définit une sélection des fichiers contenus dans l'archive. Cette sélection de fichiers sera traitéé par les fonctions WLangage de gestion des archives. 

Il est ainsi possible par exemple de sélectionner uniquement les fichiers ".doc" d'une archive. Dans ce cas, toutes les fonctions WLangage permettant de manipuler la totalité de l'archive (fonction [zipExtraitTout](../WDLang3/3082011.md) par exemple) n'agiront que sur les fichiers sélectionnés (fichiers ".doc" dans notre exemple).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Archive est un zipArchive
// Ouverture d'une archive
SI zipOuvre(Archive, "C:\Archives\Archive.zip") = 0 ALORS
	// On veut extraire uniquement les documents
	sAncienneSélection est une chaîne = zipSélectionFichier(Archive, "*.doc")
	// Extraction de tous les documents de l'archive
	zipExtraitTout(Archive, zipDisque)
	// Restitution de la sélection précédente
	zipSélectionFichier(Archive, sAncienneSélection)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = zipSélectionFichier(<Archive> [, <Sélection>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Ancienne sélection appliquée sur l'archive.

**`<Archive> : (Chaîne de caractères ou variable de type zipArchive)`**

Nom de l'archive à manipuler.
Ce nom peut correspondre :

- soit au nom d'une archive défini avec la fonction [zipOuvre](../WDLang3/3082044.md) ou [zipCrée](../WDLang3/3082003.md).

- soit au nom d'une variable de type [zipArchive](../WDLang3/1000018679.md). 
	![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Les variables de type [zipArchive](../WDLang3/1000018679.md) ne sont pas disponibles en JAVA.




**`<Sélection> : (Chaîne de caractères optionnelle)`**

Nom générique des fichiers à sélectionner. 
Les caractères génériques (\* et ?) sont autorisés. 
Si ce paramètre n'est pas précisé ou vaut chaîne vide (""), la sélection est supprimée et tous les fichiers seront traités.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Principe
<a name="principe_ELTPARAGRAPHE000113"></a>Après l'appel de la fonction **zipSélectionFichier**, les prochaines fonctions de gestion des archives manipulant l'ensemble des fichiers de l'archive respecteront la sélection : ces fonctions manipuleront uniquement la sélection de fichiers spécifiée. 

Pour manipuler à nouveau l'ensemble des fichiers de l'archive, il suffit d'appeler la fonction **zipSélectionFichier** sans spécifier le paramètre &lt;Sélection&gt;.

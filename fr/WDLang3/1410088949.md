
## SocketParamètre (Fonction)

***En anglais : SocketConfigure***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de paramétrer le mode d'utilisation des fonctions Socket. 

Remarque : Cette fonction peut être utile si la version de Windows sur le poste client ne gère pas les derniers protocoles sécurisés via SChannel (exemple : TLS 1.1 en Windows Vista, ou TLS 1.3 avant Windows 11).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SocketParamètre(socketParamètreMode, 1)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`SocketParamètre(<Option à paramétrer> , <Valeur>)`
---

**`<Option à paramétrer> : (Constante de type Entier)`**

Option à paramétrer : 


|   |   |
| --- | --- |
| *socketParamètreMode* | Choix du mode d'utilisation des sockets. |



**`<Valeur> : (Entier)`**

Valeur correspondant à l'option à paramétrer. 

Si &lt;Option à paramétrer&gt; correspond à *socketParamètreMode*, les valeurs possibles sont les suivantes : 

- 0 (Valeur par défaut) : Dans ce cas, les sockets TCP sécurisées utiliseront le module Schannel de Windows.

- 1 : Dans ce cas, les sockets TCP sécurisées utiliseront OpenSSL (comportement commun avec les autres plateformes : Linux, iOS, Android, etc.).

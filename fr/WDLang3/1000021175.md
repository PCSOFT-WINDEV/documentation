


## grNuageRécupèreDonnéeXY (Fonction)

***En anglais : grScatterGetDataXY***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère une donnée dans un graphe "Nuage de points". 

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Affichage d'une donnée dans le champ "GRF_MonGraphe" de type "Nuage de points". 
// Cette donnée est la première abscisse de la première série.
Info(grNuageRécupèreDonnéeXY(GRF_MonGraphe, 1, 1, grValeurX))
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = grNuageRécupèreDonnéeXY(<Nom du graphe> , <Numéro de la série> , <Indice> , <Valeur à récupérer>)`
---

**`<Résultat> : (Réel)`**

Valeur récupérée.

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md),

- soit au nom du champ Graphe (présent dans l'éditeur de fenêtres, de pages ou d'états).




**`<Numéro de la série> : (Entier)`**

Numéro de la série pour laquelle une valeur doit être récupérée. 
Si la série spécifiée n'existe pas, la valeur récupérée est 0.

**`<Indice> : (Entier)`**

Indice de la donnée dans la série pour laquelle la valeur doit être récupérée.
Si la donnée correspondant à l'indice spécifié n'existe pas dans la série, la valeur récupérée est 0.

**`<Valeur à récupérer> : (Constante de type Entier)`**

Indique la valeur à récupérer :


|   |   |
| --- | --- |
| *grValeurX* | Valeur X de la série. |
| *grValeurY* | Valeur Y de la série. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Pour récupérer une donnée dans un graphe Boursier, il est conseillé d'utiliser la fonction [grBourseRécupèreDonnée](../WDLang3/1000021174.md).

- Pour récupérer une donnée dans un graphe Surface, il est conseillé d'utiliser la fonction [grSurfaceRécupèreDonnée](../ErrCompil/1000001036.md).

- Pour récupérer une donnée dans un autre type de graphe (Secteur, Histogramme, Courbe, etc.), utilisez [grRécupèreDonnée](../WDLang3/1000021171.md).

- Pour récupérer une donnée dans un graphe de type Secteur multiniveau, il est conseillé d'utiliser la fonction [grSMNRécupèreDonnée](../WDLang3/1000021287.md). 

- Pour récupérer une donnée dans un graphe de type Heatmap, il est conseillé d'utiliser la fonction [grHMRécupèreDonnée](../WDLang3/1000021632.md).

- Il est préférable d'utiliser la fonction **grNuageRécupèreDonnéeXY** pour récupérer des données dans des graphes de type "Nuage de points". Il est également possible d'utiliser la fonction [grRécupèreDonnée](../WDLang3/1000021171.md).
	La syntaxe suivante :
	
	```wl
	grNuageRécupèreDonnéeXY(GRF_MonGraphe, i, 1, grValeurX)
	grNuageRécupèreDonnéeXY(GRF_MonGraphe, i, 1, grValeurY)
	```
est équivalente à la syntaxe : 
	
	```wl
	grRécupèreDonnée(GRF_MonGraphe, i*2-1, 1)
	grRécupèreDonnée(GRF_MonGraphe, i*2, 1)
	```





- Par défaut, un graphe créé avec la fonction [grCrée](../WDLang3/3042005.md) ne contient aucune donnée.

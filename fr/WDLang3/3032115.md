
## LDAPSupprimeAttribut (Fonction)

***En anglais : LDAPDeleteAttribute***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime un attribut et toutes ses valeurs.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
LDAPDébutModification("NomSession")
LDAPSupprimeAttribut("NomSession", "NomAttribut")
LDAPValideModification("NomSession", "DN")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = LDAPSupprimeAttribut(<Session LDAP> , <Nom de l'attribut>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la suppression a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de détails sur l'erreur. 


La suppression effective ne sera réalisée que lors de l'appel de la fonction [LDAPValideModification](../WDLang3/3032117.md).

**`<Session LDAP> : (Chaîne de caractères)`**

Nom de la session LDAP manipulée.

**`<Nom de l'attribut> : (Chaîne de caractères)`**

Nom de l'attribut à supprimer.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Modifier un objet existant sur un serveur LDAP
<a name="modifier_objet_existant_sur_serveur_ldap_ELTPARAGRAPHE000052"></a>Pour modifier un objet existant dans un serveur LDAP, il est nécessaire de définir les différentes modifications à effectuer.

Pour optimiser la vitesse de modification des objets, les modifications effectuées sont définies localement : une seule communication avec le serveur est établie lors de la validation de la modification. Les étapes sont les suivantes :

1. Initialisation de la modification (fonction [LDAPDébutModification](../WDLang3/3032112.md))

2. Opérations de modification :

	- Ajout des attributs et de leurs valeurs (fonction [LDAPAjouteAttribut](../WDLang3/3032110.md)).

	- Suppression des attributs (fonction **LDAPSupprimeAttribut**) ou de leurs valeurs (fonction [LDAPSupprimeValeurAttribut](../WDLang3/3032116.md)).

	- Remplacement des valeurs d'un attribut (fonction [LDAPRemplaceAttribut](../WDLang3/3032113.md) ou [LDAPRemplaceValeurAttribut](../WDLang3/3032119.md))

	- Renommage de l'attribut (fonction [LDAPRenommeAttribut](../WDLang3/3032120.md))




3. Validation des modifications sur le serveur (fonction [LDAPValideModification](../WDLang3/3032117.md)). Les modifications sont effectuées sur le serveur en suivant l'ordre utilisé dans le code WLangage.




**Attention** : Les opérations de modification étant réalisées localement avant d'être effectuées sur le serveur, il est nécessaire de respecter l'ordre logique des modifications (par exemple, il ne sera pas possible de modifier un attribut supprimé par la ligne de code précédente).

## Archive mono-partie
<a name="archive_monopartie_ELTTEXTE000206"></a>
Une archive mono-partie se présente sous la forme d'un seul fichier de type ".WDZ", ".ZIP", ".CAB" ou ".7z" : un fichier archive regroupe tous les fichiers compressés.

L'archive occupe la place nécessaire pour contenir tous les fichiers compressés.

**Exemple** : Votre programme doit sauvegarder plusieurs fichiers de taille importante. Votre programme peut créer une archive contenant l'ensemble des fichiers à sauvegarder. Cette archive augmentera l'espace disque disponible.

## Archive multi-parties
<a name="archive_multiparties_ELTTEXTE000230"></a>
![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) 

### Présentation
<a name="presentation_ELTPARAGRAPHE000047"></a>Une archive multi-parties se présente sous la forme de plusieurs fichiers (ou sous-archives) de taille identique. La taille de chaque sous-archive est fixée lors de la création de l'archive multi-parties.

La première sous-archive est un fichier de type ".WDZ", ".ZIP" ou "7z". Les sous-archives suivantes sont des fichiers numérotés de 1 à N grâce à leur extension.

Pour les archives au format WINDEV (WDZ), les sous-parties ont pour extension : ".001", ".002", ..., ".NNN".

Pour les archives au format ZIP standard, les sous-parties ont pour extension : ".Z01", ".Z02", ..., ".ZNN".
<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=achive_mono_multi3.gif)


**Remarque** : Les archives au format ".CAB" sont obligatoirement mono-partie.
<a name="NOTE3_2"></a>
![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) 

### Principe d'utilisation
<a name="principe_utilisation_ELTPARAGRAPHE000066"></a>Pour créer une archive multi-parties :

1. Créez une archive ([zipCrée](../WDLang3/3082003.md)).

2. Ajoutez des fichiers à cette archive. Les fichiers ajoutés sont automatiquement compressés ([zipAjouteFichier](../WDLang3/3082008.md), [zipAjouteRépertoire](../WDLang3/3082014.md)).

3. Découpez cette archive en plusieurs sous-archives ([zipDécoupe](../WDLang3/3082020.md)). Les sous-archives sont de taille identique. Vous décidez de la taille des sous-archives.




Une fois l'archive multi-parties créée, vous pouvez stocker les sous-archives sur différents supports (disquettes, CD, ...).

Les sous-archives peuvent être regroupées ([zipRegroupe](../WDLang3/3082046.md)) : l'archive redevient mono-partie.

**Remarque** : Toutes ces manipulations peuvent également être réalisées grâce à [WDZip](../WDZip/3527005.md).

### Attention : Archives mono-parties et multi-parties
<a name="attention_archives_monoparties_multiparties_ELTPARAGRAPHE000103"></a>Certaines fonctions d'archivage sont utilisables uniquement sur des archives mono-partie (ajout de fichiers, suppression de fichiers, création d'un exécutable auto-extractible, ...). Pour effectuer ces opérations sur une archive multi-parties, il suffit de regrouper les sous-archives ([zipRegroupe](../WDLang3/3082046.md)) pour obtenir une archive mono-partie.

A partir d'une archive multi-parties, il est uniquement possible de :

- Regrouper les différentes parties ([zipRegroupe](../WDLang3/3082046.md)),

- Obtenir diverses informations sur l'archive et sur les fichiers de l'archive ([zipExtraitChemin](../WDLang3/3082023.md), [zipInfoFichier](../WDLang3/3082040.md), [zipListeFichier](../WDLang3/3082031.md), ...),

- Extraire les fichiers ([zipExtraitFichier](../WDLang3/3082019.md), [zipExtraitTout](../WDLang3/3082011.md)).

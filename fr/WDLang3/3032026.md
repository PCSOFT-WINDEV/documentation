


## EmailMsgErreur (Fonction)

***En anglais : EmailMsgError***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Retourne le message correspondant à l'identifiant de l'erreur.

Pour connaître le libellé de l'erreur, nous vous conseillons d'utiliser directement la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.
![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Il est conseillé d'utiliser la fonction [ErreurInfo](../WDLang1/3013008.md) à la place de la fonction **EmailMsgErreur**.

Toutes les fonctions de gestion d'emails renvoient un résultat indiquant si la fonction a été effectuée ou non. En cas d'erreur, ces fonctions renvoient 0 ou <u><u><u>Faux</u></u></u>. La variable Email.Erreur contient alors l'identifiant de l'erreur survenue.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003032026&name=emailmsgerreur_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Envoie le mail
SI EmailEnvoieMessage(NomUser) = Faux ALORS
	Erreur(EmailMsgErreur(Email.Erreur))
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = EmailMsgErreur(<Numéro erreur>)`
---

**`<Résultat> : (Chaîne de caractères)`**

Message complet de l'erreur.

**`<Numéro erreur> : (Entier)`**

Identifiant de l'erreur. La variable Email.Erreur contient l'identifiant de la dernière erreur survenue lors de l'utilisation des fonctions de gestion des emails.

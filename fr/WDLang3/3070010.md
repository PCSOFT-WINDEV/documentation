
## SocketRefuse (Fonction)

***En anglais : SocketDeny***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Exécutée sur le poste serveur, cette fonction refuse une demande de connexion sur un socket existant sur le poste serveur.

**Remarque** :

- Si le poste client utilise une application WINDEV pour se connecter à ce socket, cette application devra utiliser la fonction [SocketConnecte](../WDLang3/3070016.md).

- La fonction **SocketRefuse** ne peut pas être utilisée avec un socket de type UDP.

- La fonction **SocketRefuse** peut être utilisée avec un socket SSL.








**Rappel** : Un socket est une ressource de communication utilisée par les applications pour communiquer d'une machine à une autre sans se soucier du type de réseau.


<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = SocketRefuse(<Nom du socket>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la connexion a été refusée, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. Pour obtenir plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Nom du socket> : (Chaîne de caractères)`**

Nom du socket serveur qui refuse la connexion. Ce nom a été défini :

- soit par la fonction [SocketCrée](../WDLang3/3070011.md), [SocketCréeSSL](../WDLang3/1000017041.md), [SocketCréeInfraRouge](../WDLang3/3070012.md) (cas d'un serveur simplifié).

- soit la fonction [SocketAccepte](../WDLang3/3070004.md) (cas d'un serveur standard).




![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Attention : Le nom du socket est "case-sensitive".

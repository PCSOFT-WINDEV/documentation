


## telExécuteTransfert (Fonction)

***En anglais : tapiCompleteTransfer***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Effectue un transfert d'appel (avec possibilité de reprise de l'appel).
<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = telExécuteTransfert(<Identifiant de l'appel à transférer> , <Identifiant de l'appel cible>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le transfert d'appel est effectué, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Identifiant de l'appel à transférer> : (Entier)`**

Identifiant de l'appel qui doit être transféré. Cet identifiant est connu dans la procédure WLangage appelée par la fonction [telDémarreDétectionAppel](../WDLang3/3076029.md).

**`<Identifiant de l'appel cible> : (Entier)`**

Identifiant de l'appel vers lequel le transfert doit être effectué. Cet identifiant est connu dans la procédure WLangage appelée par la fonction [telDémarreDétectionAppel](../WDLang3/3076029.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Fonctionnement du transfert d'appel
<a name="fonctionnement_transfert_appel_ELTPARAGRAPHE000042"></a>Lors d'un transfert d'appel, l'interlocuteur est prévenu qu'un transfert va être fait, puis la personne compose le numéro vers lequel le transfert va être effectué. Si le transfert est accepté, la personne valide le transfert.

Par programmation, le principe est le même :

1. Un appel est en cours. Cet appel a pour identifiant &lt;Identifiant de l'appel à transférer&gt;.

2. Composition du numéro de la personne vers qui le transfert doit être effectué (fonction [telComposeLigne](../WDLang3/3076012.md)). Cet appel a pour identifiant &lt;Identifiant de l'appel cible&gt;. L'appel à transférer est automatiquement mis en attente.

3. Cas 1 : Le transfert est accepté : il suffit d'utiliser la fonction **telExécuteTransfert**.

4. Cas 2 : Le transfert est refusé. Il suffit de raccrocher l'appel cible (fonction [telRaccroche](../WDLang3/3076030.md)). La communication initiale est automatiquement rétablie.



<a name="NOTE0_2"></a>


### Transfert direct
<a name="transfert_direct_ELTPARAGRAPHE000065"></a>Il est également possible de réaliser un transfert d'appel sans se préoccuper de la présence du correspondant cible. Il suffit d'utiliser la fonction [telTransfertDirect](../WDLang3/3076011.md).
<a name="NOTE0_3"></a>


### Configuration nécessaire
<a name="configuration_necessaire_ELTPARAGRAPHE000075"></a>Les fonctions de téléphonie utilisent la technologie TAPI 2.0 et/ou la technologie TAPI 3.1. Ces technologies sont utilisables sur tous les systèmes.

Attention : le transfert d'appel n'est pas disponible pour tous les matériels. Le transfert d'appel est pris en compte uniquement dans le cas d'un PABX.

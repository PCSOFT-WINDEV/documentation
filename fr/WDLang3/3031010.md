
## Application cliente : Exécuter des procédures sur un serveur de services .Net 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000119"></a>
WINDEV met à votre disposition un ensemble de fonctions DotNet. Grâce à ces fonctions, vous pouvez gérer l'exécution de procédures sur un serveur de services Web XML .Net (dotNet) depuis vos applications WINDEV.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principe d'utilisation
<a name="principe_utilisation_ELTTEXTE000143"></a>
L'ensemble des paramètres nécessaires à l'exécution d'une procédure est fourni dans la documentation du serveur de services Web XML DotNet et de ses procédures.

Pour exécuter une procédure sur un serveur de services Web XML DotNet :

1. Initialisez la structure des paramètres de la procédure à exécuter ([voir la structure DotNet](../WDLang3/3031011.md)).
	Remarque : Pour effectuer des vérifications supplémentaires sur le serveur de services Web XML DotNet, ajoutez un entête de procédure (fonction [DotNetAjouteEntête](../WDLang3/3031009.md)).

2. Exécutez la procédure (fonction [DotNetExécute](../WDLang3/3031016.md) ou [DotNetExécuteXML](../WDLang3/3031012.md)). Les opérations suivantes sont automatiquement réalisées :

	- connexion entre le poste en cours et le serveur de services Web XML DotNet.

	- transmission des paramètres de la procédure au serveur de services Web XML DotNet.

	- vérification de l'entête si nécessaire.

	- exécution de la procédure.

	- renvoi du résultat ou de l'erreur de la procédure au poste en cours.




3. Testez le résultat de la procédure.




Le résultat des fonctions [DotNetExécute](../WDLang3/3031016.md) ou [DotNetExécuteXML](../WDLang3/3031012.md) permet de savoir si la connexion a bien été établie.

Si la connexion n'a pas été établie, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) pour connaître les causes de l'erreur. 

Si la connexion a été établie, testez le résultat de la fonction [DotNetDonneRésultat](../WDLang3/3031005.md) :

- si le résultat est différent d'une chaîne vide (""), la procédure a été exécutée avec succès.

- si le résultat correspond à une chaîne vide (""), la procédure n'a pas été exécutée et/ou a retourné une erreur. Pour connaître l'erreur de la procédure, utilisez la fonction [DotNetErreur](../WDLang3/3031004.md).

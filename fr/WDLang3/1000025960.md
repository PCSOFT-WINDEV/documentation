


## WebSocketDéconnecte (Fonction)

***En anglais : WebSocketDisconnect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Déconnecte un client actuellement connecté sur un serveur de WebSocket. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupère la liste des clients
tabClients est un tableau de websocketClient = WebSocketListeClientConnecté()
// Déconnecte certains clients
POUR TOUT oClient de tabClients
	SI bDéconnecte(oClient) ALORS
		WebSocketDéconnecte(oClient)
	FIN
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`WebSocketDéconnecte(<Client>)`
---

**`<Client> : (Variable de type websocketClient)`**

Nom de la variable de type [websocketClient](../WDLang3/1000025952.md) correspondant au client à déconnecter. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Lors de la déconnexion du client, le traitement de déconnexion du client est appelé.

- **Cette fonction est asynchrone**: la fonction ne fait que demander la déconnexion du client. La déconnexion effective du client peut intervenir ultérieurement (et en particulier après l'exécution de la fonction **WebSocketDéconnecte**). La fonction **WebSocketDéconnecte** ne renvoie jamais d'erreur (par exemple si le client s'est/a été déconnecté entre temps).

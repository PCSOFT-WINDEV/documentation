


## &lt;Graphe&gt;.SourceSérie (Fonction)

***En anglais : &lt;Chart&gt;.SourceSeries***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Définit la source d'une série.


![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Champ Graphe rempli à partir d'une colonne d'un champ Table 
SI GRF_Graphe1.SourceSérie(3, grColonneTable, "Table1.Colonne1") = Vrai ALORS
	Info("La source de la série 3 est désormais la colonne MaColonne")
FIN
```
<a name="Exemple2"></a>

```wl
// Champ Graphe basé sur une rubrique d'une requête SQL
MaRequête est une Source de Données

MonCodeSQL est une chaîne = REQ_Requête1..CodeSQL
HExécuteRequêteSQL(MaRequête, hRequêteDéfaut, MonCodeSQL)
GRF_Graphe1.SourceSérie(1, grRubrique, "MaRequête", "TotalHT")
GRF_Graphe1.Dessine()

CAS ERREUR:
	Erreur(HErreurInfo())
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Champ Graphe>.grSourceSérie(<Numéro de la série> , <Source> [, <Détail 1> [... [, <Détail N>]]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la source a été définie, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Champ Graphe> : (Nom de champ)`**

Nom du champ Graphe à manipuler (présent dans l'éditeur de fenêtres, de pages ou d'états).

**`<Numéro de la série> : (Entier)`**

Numéro de la série concernée.

**`<Source> : (Constante de type Entier)`**

Source  de données du graphe :


|   |   |
| --- | --- |
| *grColonneTable* | Série remplie depuis une colonne d'un champ Table. <br><br>Dans ce cas, &lt;Détail 1&gt; correspond au nom d'une colonne d'un champ Table. Ce nom doit correspondre au nom complet de la colonne, indiqué entre guillemets (par exemple "&lt;NomTable&gt;.&lt;NomColonne&gt;").<br>La colonne est parcourue pour remplir la série. Si une valeur n'est pas numérique, un "trou" est créé dans la série. |
| *grFonction* | Série remplie par une fonction / procédure WLangage. <br>Dans ce cas, &lt;Détail 1&gt; correspond au nom de la procédure WLangage à appeler.<br><br>Pour chaque nombre, la procédure est appelée. Cette procédure est du type : <br><br><br><pre><code>PROCEDURE MaProcédure(<Champ Graphe>, <Numéro de la série>, <br><Index dans la série>)</code></pre><br><br>La procédure renvoie le nombre voulu. Pour indiquer la fin de la série, la procédure doit renvoyer <u><u><u><u>Faux</u></u></u></u>.<br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Cette constante n'est pas disponible. |
| *grListe* | Série remplie depuis un champ Liste.<br><br>Dans ce cas, &lt;Détail 1&gt; correspond au nom d'un champ Liste. Le champ Liste est parcouru pour remplir la série. Si une valeur n'est pas numérique, un "trou" est créé dans la série. |
| *grProgrammation* | Série décrite par programmation. <br><br>&lt;Détail 1&gt; peut correspondre à une série de nombres dans une même chaîne de caractères, séparés par des ";". |
| *grRubrique* | Série remplie depuis une rubrique.<br><br>&lt;Détail 1&gt; correspond au nom du fichier de données (ou de la requête), &lt;Détail 2&gt; correspond au nom de la rubrique. La série va parcourir le fichier de données en prenant toutes les valeurs de la rubrique.<br><br>Si une valeur n'est pas numérique, un "trou" est créé dans la série. |
| *grTableau* | Série remplie depuis un tableau WLangage.<br><br>&lt;Détail 1&gt; est le nom du tableau. &lt;Détail 2&gt; peut être l'indice de la colonne (tableau à 2 dimensions) ou un nom de membre (tableau de structures). |



**`<Détail 1> : (Type attendu  (paramètre optionnel))`**

Premier paramètre attendu, en fonction du type de source de données.



**`<Détail N> : (Type attendu  (paramètre optionnel))`**

Dernier paramètre attendu, en fonction du type de source de données.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La propriété [Source](../Proprietes/2510147.md) permet de connaître le type de source de données d'une série d'un graphe (colonne d'un champ Table, champ Liste, ...).

- La propriété [SourceDescription](../Proprietes/2510148.md) permet de connaître le nom de l'élément qui correspond à la source de données d'une série d'un graphe.

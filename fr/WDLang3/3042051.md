


## grRelief (Fonction)

***En anglais : grRaised***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet d'afficher un graphe en relief. Il suffit de spécifier la profondeur du relief voulu. Tous les types de graphes peuvent être affichés avec un relief.

Exemple de graphe en relief : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Relief.gif)




![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Augmentation de la profondeur du relief du graphe "MonGraphe" (créé par programmation)
grRelief("MonGraphe", 5)
```
<a name="Exemple2"></a>

```wl
// Augmentation de la profondeur du relief du graphe GRF_MonGraphe
grRelief(GRF_MonGraphe, 7)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`grRelief(<Nom du graphe> , <Relief>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- soit au nom du champ Graphe présent dans l'éditeur de fenêtres, de pages ou d'états.




**`<Relief> : (Entier)`**

Profondeur du relief du graphe (en pixels). Pour que le graphe reste lisible, il est déconseillé de spécifier un relief trop important.
Si ce paramètre est égal à 0, le graphe est affiché en deux dimensions (sans relief).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Tous les types de graphes peuvent être affichés en relief.

- Les données définies avec la fonction **grRelief** seront prises en compte lors du prochain appel à la fonction [grDessine](../WDLang3/3042026.md).

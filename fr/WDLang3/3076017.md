
## telAppelEstOccupé (Fonction)

***En anglais : tapiCallIsBusy***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de savoir si le numéro appelé est occupé. Cette fonction est utilisable uniquement avec l'appel sortant en cours.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
PROCEDURE ComposerNumTel(sNuméroTel)
// On compose le numéro
SI telCompose(sNuméroTel, "EtatAppel", 4) = Vrai ALORS
	// Communication établie
	Message("La communication est établie") 
	Info("La communication est établie, vous pouvez décrocher") 
	RETOUR
SINON
	SI PAS ErreurInfo() ~= "" ALORS
		Erreur(ErreurInfo())
	FIN
	// On teste si la ligne est occupée
	SI telAppelEstOccupé() = Vrai ALORS
		Info("La ligne est occupée")
		RETOUR
	FIN
	// On teste si le correspondant a répondu
	SI telAppelPasDeRéponse() = Vrai ALORS
		Info("le correspondant n'a pas répondu")
		RETOUR
	FIN
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = telAppelEstOccupé([<Identifiant de l'appel>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le numéro appelé est occupé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Identifiant de l'appel> : (Entier optionnel)`**

Identifiant de l'appel à manipuler. Cet identifiant est connu dans la procédure WLangage appelée par la fonction [telDémarreDétectionAppel](../WDLang3/3076029.md). Si ce paramètre n'est pas spécifié, l'appel en cours (sortant ou entrant) est manipulé.

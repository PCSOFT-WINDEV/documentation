


## &lt;Variable emailSessionIMAP&gt;.ChangeEtat (Fonction)

***En anglais : &lt;emailSessionIMAP variable&gt;.ChangeStatus***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Change l'état d'un email sur un serveur de messagerie.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Passer tous les emails d'une session IMAP en lus
MaSession est un emailSessionIMAP
...
POUR TOUT MessageEmail NON LU DE MaSession
	MaSession.ChangeEtat(MessageEmail, emailEtatLu)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = <Session>.EmailChangeEtat(<Email> , <Nouvel état>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'état a été modifié sur le serveur, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Session> : (Variable de type emailSessionIMAP)`**

Nom de la variable de type [emailSessionIMAP](../WDLang3/1000018957.md) à utiliser.

**`<Email> : (Variable de type Email)`**

Nom de la variable de type [Email](../WDLang3/1000018713.md) correspondant à l'email à manipuler.

**`<Nouvel état> : (Constante de type Entier)`**

Etat à affecter à l'email, les valeurs possibles sont :


|   |   |
| --- | --- |
| *emailEtatLu* | Indique que le message a été lu. |
| *emailEtatNonLu* | Indique que le message n'a pas encore été lu. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Cette fonction n'est pas disponible : 

- sur les sessions Notes (variable de type [emailSessionNotes](../WDLang3/1000018768.md)), 

- sur les sessions POP3 (variable de type [emailSessionPOP3](../WDLang3/1000018759.md)), 

- sur les sessions SMTP (variable de type [emailSessionSMTP](../WDLang3/1000018765.md)).

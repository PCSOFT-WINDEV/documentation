


## grHMCouleur (Fonction)

***En anglais : grHMColor***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Spécifie les couleurs des différentes valeurs d'un graphe de type Heatmap.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Définition des couleurs
grHMCouleur(GRF_HMap, -50, BleuFoncé)
grHMCouleur(GRF_HMap, +50, RougeClair)
POUR i = 1 A 500
	grHMAjouteDonnée(GRF_HMap, Hasard(-100, 100), Hasard(-100, 100), -100)
	grHMAjouteDonnée(GRF_HMap, Hasard(-100, 100), Hasard(-100, 100), 100)
FIN
grDessine(GRF_HMap)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`grHMCouleur(<Nom du graphe> , <Palier> , <Couleur>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom peut correspondre :

- au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- au nom du champ Graphe présent dans l'éditeur de fenêtres ou d'états.




**`<Palier> : (Réel ou Constante)`**

Palier à partir duquel la couleur sera appliquée. Correspond : 

- soit à une valeur réelle,

- soit à la constante *grHMCouleurValeurVide*. Dans ce cas, la couleur sera appliquée pour les données en dehors des paliers ou sans valeur. Par défaut, cette couleur est transparente. 




**`<Couleur> : (Entier ou constante de type Entier)`**

Couleur à donner au palier. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- une couleur prédéfinie (voir [les couleurs prédéfinies du WLangage](../WDLang5/3010002.md)).  

- la constante *CouleurAutomatique*. Dans ce cas, le palier est supprimé.

## Remarques


### Gestion des couleurs
<a name="gestion_des_couleurs_ELTPARAGRAPHE000098"></a>Les couleurs sont définies par palier. Pour les valeurs se situant entre 2 paliers, un dégradé linéaire est appliqué par défaut.

Ce comportement peut être modifié à l'aide de la fonction [grParamètre](../WDLang3/1000018337.md) utilisée avec la constante *grHMAffichageCouleur*. La syntaxe utilisée est la suivante :

```txt
grParamètre(<Nom Graphe>, grHMAffichageCouleur, <Type d'affichage>)
```



 où &lt;Type d'affichage&gt; correspond à une des constantes suivantes :



|   |   |
| --- | --- |
| *grDégradéCouleur*<br>(Valeur par défaut) | Un dégradé linéaire est appliqué pour les valeurs intermédiaires. |
| *grPalierInférieur* | L'affichage des couleurs se fait par palier. Toutes les valeurs inférieures ou égales au palier prendront la couleur fournie par la fonction [grHMCouleur](../WDLang3/1000021629.md). |
| *grPalierSupérieur* | L'affichage des couleurs se fait par palier. Toutes les valeurs supérieures ou égales au palier prendront la couleur fournie par la fonction [grHMCouleur](../WDLang3/1000021629.md). |


Pour les facettes en dehors des bornes de couleurs ou sans valeur, la couleur spécifiée avec la constante *grHMCouleurValeurVide* sera appliquée. Par défaut, cette couleur est transparente.

Remarque : Les couleurs définies avec la fonction **grHMCouleur** seront prises en compte lors du prochain appel à la fonction [grDessine](../WDLang3/3042026.md).

## Présentation
<a name="presentation_ELTTEXTE000156"></a>
La domotique par courant porteur (X10) est aujourd'hui une réalité. Gérer les équipements électriques d'un bâtiment à travers le câblage électrique existant ouvre la voie à un contrôle à distance facile par programmation.

Saviez-vous qu'il est possible de transformer à bas coût une installation électrique classique en une installation domotique ?

Sans câblage supplémentaire, sans onde radio, vous pilotez des équipements électriques grâce à des projets WINDEV, WINDEV Mobile ou WEBDEV. Il suffit :

- soit d'utiliser une prise ou des interrupteurs adaptés (la norme X10 est utilisée).

- soit d'utiliser une multiprise adaptée.




Allumer une lampe, fermer des volets, éteindre un matériel ou connaître son état (éteint, allumé, ...) s'effectue facilement depuis un projet WINDEV, WINDEV Mobile ou WEBDEV.
<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=SchemaDomo.gif)


**Remarque** : Dans cette version, les interfaces de type CM11 (de Marmitek) et les interfaces de type CM15 sont supportées.

## Comment le faire ?
<a name="comment_faire_ELTTEXTE000180"></a>


### Piloter des équipements électriques
<a name="piloter_des_equipements_electriques_ELTPARAGRAPHE000032"></a>Pour piloter des équipements électriques :

1. Ouvrez une connexion avec une interface X10 (fonction [DomoOuvre](../WDLang3/1000003102.md)).

2. Mettez en marche le ou les appareils électriques voulus (fonction [DomoAllume](../WDLang3/1000003101.md) ou [DomoEnvoie](../WDLang3/1000003104.md)).

3. Envoyez la commande à effectuer : mise en marche, augmentation ou diminution de l'intensité, arrêt (fonction [DomoEnvoie](../WDLang3/1000003104.md)).

4. Vérifiez l'état des appareils électriques : éteint ou allumé (fonction [DomoEtat](../WDLang3/1000003105.md)).

5. Arrêtez le ou les appareils électriques voulus (fonction [DomoEteint](../WDLang3/1000003103.md)).

6. Fermez la connexion avec l'interface X10 (fonction [DomoFerme](../WDLang3/1000003106.md)).




**Remarque** : Le bon fonctionnement du protocole X10 par courant porteur requiert une continuité électrique entre les appareils. Par exemple, la présence de transformateur d'isolement empêche la transmission du courant porteur.


<a name="NOTE2_2"></a>


### Pilotage d'un module de type CM15
<a name="pilotage_module_type_cm15_ELTPARAGRAPHE000070"></a>**Pré-requis** : 

1. Installez le logiciel "Active Home" livré avec le module de type CM15.

2. Installez le programme "ahscript_setup.exe". Ce programme est disponible sur le CD "Active Home" livré avec le module de type CM15.


**Remarque** : le câble USB doit être branché avant le câble électrique.

**Mise en place**

1. Ouvrez une connexion avec une interface X10 (fonction [DomoOuvre](../WDLang3/1000003102.md)).

2. Mettez en marche le ou les appareil électriques voulus (fonction [DomoAllume](../WDLang3/1000003101.md) ou [DomoEnvoie](../WDLang3/1000003104.md)).

3. Envoyez la commande à effectuer : mise en marche, augmentation ou diminution de l'intensité, arrêt (fonction [DomoEnvoie](../WDLang3/1000003104.md)).

4. Arrêtez le ou les appareils électriques voulus (fonction [DomoEteint](../WDLang3/1000003103.md)).

5. Fermez la connexion avec l'interface X10 (fonction [DomoFerme](../WDLang3/1000003106.md)).


**Limitation** : La fonction [DomoEtat](../WDLang3/1000003105.md) n'est pas disponible.




## FluxDéconnecte (Fonction)

***En anglais : StreamDisconnect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Arrête la réception ou l'émission d'un flux. La connexion est fermée et la connexion est dissociée du flux et des champs Conférence correspondants.

**Remarque** : Pour arrêter la réception ou l'émission d'un flux sans dissocier ce flux d'un champ Conférence, utilisez la fonction [FluxStop](../WDLang3/3039002.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Arrête la réception du flux et dissocier le flux du champ associé
FluxDéconnecte(FLUX_MonFlux.Valeur)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`FluxDéconnecte(<Identifiant du flux>)`
---

**`<Identifiant du flux> : (Entier)`**

Identifiant du flux à déconnecter. Cet identifiant peut être connu : 

- dans l'événement "Demande de connexion d'un nouveau flux" associé au flux.

- par la propriété [Valeur](../Proprietes/2510130.md) utilisée sur le flux.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Déconnexion d'un flux
<a name="deconnexion_flux_ELTPARAGRAPHE000041"></a>Lorsqu'un flux est déconnecté (par l'appelant ou l'appelé), l'événement "Fin de connexion" est automatiquement exécuté. 


### Procédure de détection de flux
<a name="procedure_detection_flux_ELTPARAGRAPHE000174"></a>Un flux est associé à l'événement "Demande de connexion d'un nouveau flux". Cet événement est exécuté lorsqu'une demande de connexion d'un flux entrant est détectée.

Cet événement est exécuté dans un des deux cas suivants : 

- si la propriété [Détection](../Proprietes/2510031.md) utilisée sur le flux est à <u><u><u><u>Vrai</u></u></u></u>.

- si l'option "Détection des flux entrants actifs" a été cochée dans la fenêtre de description des flux.




**Remarque** : Cet événement est exécuté dans un thread spécifique du WLangage. Il est donc soumis aux limitations sur les threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads avec WINDEV](../WDLang1/3077026.md).

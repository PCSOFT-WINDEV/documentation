
## WebserviceEcritEntêteHTTP (Fonction)

***En anglais : WebserviceWriteHTTPHeader***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute un entête HTTP supplémentaire à une requête à un Webservice REST. Cet entête HTTP sera retournée au client avec la réponse lors du prochain appel au Webservice REST.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Autorise l'appel au webservice depuis n'importe quel site dans un navigateur.
WebserviceEcritEntêteHTTP("Access-Control-Allow-Origin", "*")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`WebserviceEcritEntêteHTTP(<Nom de l'entête HTTP> , <Valeur de l'entête>)`
---

**`<Nom de l'entête HTTP> : (Chaîne de caractères)`**

Nom de l'entête HTTP à écrire.

**`<Valeur de l'entête> : (Chaîne de caractères)`**

Valeur (ou contenu) de l'entête HTTP à écrire.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le serveur d'application WEBDEV force la valeur de certains entêtes HTTP (par exemple "Content-Length"). Dans ce cas, la valeur écrite par la fonction **WebserviceEcritEntêteHTTP** sera ignorée.

- Lors d'appels successifs à cette fonction pour un même nom d'entête HTTP, seule la dernière valeur est prise en compte.


## FluxInfo (Fonction)

***En anglais : StreamInfo***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nom et / ou le numéro de l'émetteur du flux. Il est ainsi possible d'obtenir des informations sur la connexion entrante avant de l'accepter.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupère le nom de l'émetteur du flux
NomAppelant est une chaîne = FluxInfo(IDFlux)
```
<a name="Exemple2"></a>

```wl
// Récupère le numéro de l'émetteur du flux
NumAppelant est une chaîne = FluxInfo(IDFlux, FluxNuméroAppelant)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = FluxInfo(<Identifiant de la connexion> [, <Information demandée>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Nom ou numéro de l'émetteur du flux.

**`<Identifiant de la connexion> : (Entier)`**

Identifiant de la connexion (connu dans l'événement "Demande de connexion d'un nouveau flux").

**`<Information demandée> : (Constante optionnelle de type Entier)`**

Information à renvoyer :


|   |   |
| --- | --- |
| *FluxNomAppelant*<br>(valeur par défaut) | Nom de l'émetteur du flux.<br>En WLangage, ce nom peut être donné grâce à la propriété [Identifiant](../Proprietes/2510036.md). |
| *FluxNuméroAppelant* | Numéro de l'émetteur de flux (par exemple son adresse IP). |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Procédure de détection de flux
<a name="procedure_detection_flux_ELTPARAGRAPHE000174"></a>Un flux est associé à l'événement "Demande de connexion d'un nouveau flux". Cet événement est exécuté lorsqu'une demande de connexion d'un flux entrant est détectée.

Cet événement est exécuté dans un des deux cas suivants : 

- si la propriété [Détection](../Proprietes/2510031.md) utilisée sur le flux est à <u><u><u><u>Vrai</u></u></u></u>.

- si l'option "Détection des flux entrants actifs" a été cochée dans la fenêtre de description des flux.




**Remarque** : Cet événement est exécuté dans un thread spécifique du WLangage. Il est donc soumis aux limitations sur les threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads avec WINDEV](../WDLang1/3077026.md).

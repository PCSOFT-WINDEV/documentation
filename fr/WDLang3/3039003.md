


## FluxAccepte (Fonction)

***En anglais : StreamAccept***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Associe une demande de connexion de flux à un champ Conférence. Cette fonction doit être utilisée dans l'événement "Demande de connexion d'un nouveau flux" associé au flux.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// -- Evénement "Demande de connexion d'un nouveau flux"
// Procédure de détection d'un nouveau flux
// nIDFlux est une variable de type entier contenant l'identifiant du flux détecté
PROCEDURE NouveauFlux(nIDFlux)
// Accepter le flux détecté ? INT_Reponse_Auto est un champ Interrupteur
SI INT_Reponse_Auto[1] = Vrai ALORS
	FluxAccepte(nIDFlux, MoiMême)
SINON
	FluxRefuse(nIDFlux)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = FluxAccepte(<Identifiant de la connexion> , <Champ Conférence>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le flux a été associé au champ Conférence, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire (identifiant incorrect, champ inexistant, ...). Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Identifiant de la connexion> : (Entier)`**

Identifiant de connexion du flux. Cette connexion sera associée à un champ Conférence et donc au flux auquel le champ est associé. Cet identifiant est connu dans l'événement "Demande de connexion d'un nouveau flux" associé au flux.

**`<Champ Conférence> : (Nom de champ)`**

Nom d'un champ Conférence qui accepte la connexion.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### FluxAccepte : Opérations réalisées
<a name="fluxaccepte_operations_realisees_ELTPARAGRAPHE000044"></a>Le flux associé au champ est lié à cette connexion.

Le flux établi les liens vers tous les champs Conférence qui lui sont liés.

L'identifiant de la connexion (et donc du flux) est connu :

- soit avec la propriété [Valeur](../Proprietes/2510130.md).

- soit directement en utilisant le nom du flux.

- soit directement en utilisant le nom du champ Conférence.




Si le champ Conférence est déjà connecté à un flux, le flux est automatiquement déconnecté.
<a name="NOTE0_2"></a>


### Procédure de détection de flux
<a name="procedure_detection_flux_ELTPARAGRAPHE000192"></a>Un flux est associé à l'événement "Demande de connexion d'un nouveau flux". Cet événement est exécuté lorsqu'une demande de connexion d'un flux entrant est détectée.

Cet événement est exécuté dans un des deux cas suivants : 

- si la propriété [Détection](../Proprietes/2510031.md) utilisée sur le flux est à <u><u><u><u>Vrai</u></u></u></u>.

- si l'option "Détection des flux entrants actifs" a été cochée dans la fenêtre de description des flux.




**Remarque** : Cet événement est exécuté dans un thread spécifique du WLangage. Il est donc soumis aux limitations sur les threads. Pour plus de détails sur la gestion des threads, consultez [Gestion des threads avec WINDEV](../WDLang1/3077026.md).

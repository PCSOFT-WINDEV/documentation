


## &lt;Graphe&gt;.SauveBMP (Fonction)

***En anglais : &lt;Chart&gt;.SaveBMP***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Sauvegarde un graphe préalablement dessiné dans un fichier au format BMP. 



![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Sauvegarde le graphe GRF_MonGraphe dans le fichier "C:\Graphes\Graphe.BMP"
GRF_MonGraphe.SauveBMP("C:\Graphes\Graphe.BMP")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Champ Graphe>.SauveBMP(<Nom du fichier BMP>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le graphe a été sauvegardé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Champ Graphe> : (Nom de champ)`**

Nom du champ Graphe à manipuler (présent dans l'éditeur de fenêtres, de pages ou d'états).

**`<Nom du fichier BMP> : (Chaîne de caractères)`**

Nom et chemin complet du fichier BMP à créer. L'extension du fichier est obligatoire. Ce fichier BMP est créé lors de l'appel à la fonction **&lt;Graphe&gt;.SauveBMP**. Si le fichier existait déjà, il est écrasé et recréé.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Enregistrer un graphe
<a name="enregistrer_graphe_ELTPARAGRAPHE000233"></a>

- Pour enregistrer un graphe, le graphe doit avoir été préalablement dessiné (fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md)). Les modifications effectuées depuis le dernier appel à la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md) ne sont pas prises en compte.

- La taille par défaut de l'image est fixée à 639 x 479. Si le graphe n'a jamais été dessiné ou n'est pas visible, ces dimensions sont prises en compte.



<a name="NOTE0_2"></a>


### Sauvegarde au format BMP
<a name="sauvegarde_format_bmp_ELTPARAGRAPHE000254"></a>

- Ce fichier pourra être lu grâce à un outil reconnaissant le format BMP.

- La taille du fichier est identique à la taille de la destination (taille de la fenêtre, du champ ou du fichier).

- Il est conseillé de visualiser le graphe à l'écran avant de l'enregistrer dans un fichier.



<a name="NOTE0_3"></a>


### Différences entre les fonctions grDestinationBMP et &lt;Graphe&gt;.SauveBMP
<a name="differences_entre_les_fonctions_grdestinationbmp_docparampagetitleshort_ELTPARAGRAPHE000266"></a>

- La fonction [grDestinationBMP](../WDLang3/3042018.md) permet de définir la destination du prochain dessin du graphe. Lors de l'utilisation de la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md), le graphe sera dessiné dans le fichier spécifié et toutes les modifications effectuées depuis le dernier dessin du graphe seront prises en compte.

- La fonction **&lt;Graphe&gt;.SauveBMP** permet de sauvegarder le graphe directement dans un fichier au format BMP. L'image enregistrée correspond au graphe lors du dernier appel de la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md). Si des modifications ont été effectuées sur le graphe depuis l'appel de la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md), ces modifications ne seront pas prises en compte.

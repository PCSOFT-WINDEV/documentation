
## HTTPCréeFormulaire (Fonction)

***En anglais : HTTPCreateForm***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Crée un formulaire HTTP. Ce formulaire pourra être envoyé avec la fonction [HTTPEnvoieFormulaire](../WDLang3/1000017190.md) et manipulé avec les fonctions de gestion de formulaires HTTP.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HTTPCréeFormulaire("FORM")
...
HTTPEnvoieFormulaire("FORM", "www.monsite.fr")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`HTTPCréeFormulaire(<Nom du formulaire>)`
---

**`<Nom du formulaire> : (Chaîne de caractères)`**

Nom du formulaire HTTP. Ce nom permet de manipuler le formulaire HTTP par programmation (par exemple avec la fonction [HTTPAjouteParamètre](../WDLang3/1000017189.md)). Si le formulaire existe déjà, il est vidé.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Libération des ressources
<a name="liberation_des_ressources_ELTPARAGRAPHE000075"></a>La déclaration du formulaire peut être annulée par la fonction [HTTPAnnuleFormulaire](../WDLang3/1000017123.md). Cette fonction permet de libérer les ressources allouées. Les ressources allouées seront automatiquement libérées :

- à la fin du thread en cours (ou de l'application s'il s'agit du thread principal).

- lors de l'appel à la fonction [HTTPEnvoieFormulaire](../WDLang3/1000017190.md).

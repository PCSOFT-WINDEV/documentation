
## awEnvoieInfo (Fonction)

***En anglais : awSendInfo***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Envoie une information d'un téléphone iOS vers une montre Apple Watch ou inversement.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MonBuffer est un Buffer = "Buffer Texte"
SI PAS awEnvoieInfo(MonBuffer) ALORS
	Erreur(ErreurInfo())
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = awEnvoieInfo(<Données>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'envoi a réussi

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de détails sur l'erreur.




**`<Données> : (Buffer)`**

Données à envoyer.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le récepteur obtiendra les données envoyées par l'intermédiaire de la procédure passée à la fonction [awProcédureRéceptionInfo](../WDLang3/1000022119.md).

- Si les données n'ont pas changé depuis l'appel précédent, la procédure du récepteur ne sera pas appelée. 

- Les données ou informations envoyées sont considérées comme volatiles : chaque appel successif remplace les données envoyées par l'appel précédent. 

- Les données sont transférées en arrière-plan pour minimiser l'usage de la batterie. Les données peuvent donc être transmises bien après l'appel effectif.

- Les données envoyées doivent (de préférence) être relativement petites (afin d'éviter de surcharger la communication entre la montre et le téléphone).

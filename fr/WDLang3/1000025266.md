


## ChangeThème (Fonction)

***En anglais : ChangeTheme***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de modifier par programmation le thème (sombre ou clair) utilisé par l'application mobile.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Lors de la modification du champ Interrupteur INT_ModeNuit
SI INT_ModeNuit ALORS
	ChangeThème(stThèmeSombre)
SINON 
	ChangeThème(stThèmeClair)
FIN
```

```wl
// Initialisation du thème du projet à celui du système
ChangeThème(SysThème())
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`ChangeThème(<Thème> [, <Option>])`
---

**`<Thème> : (Constante de type Entier)`**

Thème à appliquer : 



|   |   |
| --- | --- |
| *stThèmeAutomatique* | Thème utilisé par défaut par le mobile. Le thème change automatiquement selon le paramétrage du système. |
| *stThèmeClair* | Force l'utilisation du thème clair. |
| *stThèmeSombre* | Force l'utilisation du thème sombre. |



**`<Option> : (Constante de type Entier optionnelle)`**

Indique si les événements de changement de thème doivent être exécutés :


|   |   |
| --- | --- |
| *avecExécutionDeCode*<br>(Valeur par défaut) | Tous les événements WLangage liés à l'application de thème sont exécutés dans l'ordre suivant : <br><br>1. Evénement "Application du thème clair/sombre" du projet. <br><br>2. Evénement optionnel "Application du thème clair/sombre" des fenêtres. <br><br>3. Evénement optionnel "Application du thème clair/sombre" des fenêtres internes. <br><br><br><br><br>Remarque : Si l'application du thème sombre est interdite pour la fenêtre (option "Ne pas appliquer le thème sombre" dans l'onglet "Style" de la fenêtre de description de la fenêtre), l'événement "Application du thème clair/sombre" n'est pas exécuté pour cette fenêtre mais les événements "Application du thème clair/sombre" de toutes les autres fenêtres ouvertes et du projet sont exécutés. |
| *sansExécutionDeCode* | Aucun événement WLangage (du projet et/ou des fenêtres ou fenêtres internes) lié à l'application du thème clair/sombre n'est exécuté. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000069"></a>Pour utiliser la fonction **ChangeThème**, il est nécessaire d'avoir autorisé le passage en thème sombre dans l'application : 

1. Affichez la fenêtre de description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description". 

2. Dans l'onglet "Style", cochez l'option "Autoriser le passage en thème sombre (automatique ou manuellement via la fonction ChangeThème)".

3. Validez.


Si le passage en thème sombre n'a pas été autorisé, la fonction **ChangeThème** renvoie une erreur.  

Pour plus de détails, consultez [Gestion du thème sombre](../WDLang3/1000025338.md). 


### Fonctionnement et cas particuliers
<a name="fonctionnement_cas_particuliers_ELTPARAGRAPHE000107"></a>

- Par défaut, le mode automatique est utilisé.

- Si le thème choisi correspond au thème courant, la fonction **ChangeThème** ne fait rien.

- Lors de l'utilisation de la fonction **ChangeThème**, toutes les fenêtres de l'application changent de thème (sauf celles pour lesquelles l'option "Ne pas appliquer le thème sombre" est cochée dans l'onglet "Style" de la fenêtre de description.

- Les prochaines fenêtres ouvertes utiliseront le thème spécifié avec la fonction **ChangeThème**.

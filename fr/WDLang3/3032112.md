
## LDAPDébutModification (Fonction)

***En anglais : LDAPStartModify***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Initialise la modification d'un objet existant dans un serveur LDAP.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Supprime une valeur donnée (lue sur le serveur LDAP)
// la valeur à supprimer est dans la chaîne sValeurASupprimer
LDAPDébutModification("MaSessionLDAP")
LDAPSupprimeValeurAttribut("MaSessionLDAP", "NomAttribut", sValeurASupprimer)
LDAPValideModification("MaSessionLDAP", "DN")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`LDAPDébutModification(<Session LDAP>)`
---

**`<Session LDAP> : (Chaîne de caractères)`**

Nom de la session LDAP manipulée.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Modifier un objet existant sur un serveur LDAP
<a name="modifier_objet_existant_sur_serveur_ldap_ELTPARAGRAPHE000033"></a>Pour modifier un objet existant dans un serveur LDAP, il est nécessaire de définir les différentes modifications à effectuer.

Pour optimiser la vitesse de modification des objets, les modifications effectuées sont définies localement : une seule communication avec le serveur est établie lors de la validation de la modification. Les étapes sont les suivantes :

1. Initialisation de la modification (fonction **LDAPDébutModification**).

2. Opérations de modification :

	- Ajout des attributs et de leurs valeurs (fonction [LDAPAjouteAttribut](../WDLang3/3032110.md)).

	- Suppression des attributs (fonction [LDAPSupprimeAttribut](../WDLang3/3032115.md)) ou de leurs valeurs (fonction [LDAPSupprimeValeurAttribut](../WDLang3/3032116.md)).

	- Remplacement des valeurs d'un attribut (fonction [LDAPRemplaceAttribut](../WDLang3/3032113.md) ou [LDAPRemplaceValeurAttribut](../WDLang3/3032119.md)).

	- Renommage de l'attribut (fonction [LDAPRenommeAttribut](../WDLang3/3032120.md)).




3. Validation des modifications sur le serveur (fonction [LDAPValideModification](../WDLang3/3032117.md)). Les modifications sont effectuées sur le serveur en suivant l'ordre utilisé dans le code WLangage.




**Attention** : Les opérations de modification étant réalisées localement avant d'être effectuées sur le serveur, il est nécessaire de respecter l'ordre logique des modifications (par exemple, il ne sera pas possible de modifier un attribut supprimé par la ligne de code précédente).

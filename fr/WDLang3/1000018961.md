
## SaaSVérifieService (Fonction)

***En anglais : SaaSCheckService***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Vérifie les droits d'accès de l'utilisateur à un service du site SaaS. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI SaaSConnecte("login", "motdepasse") = Vrai ALORS
	SI SaaSVérifieService("CRM") = Faux ALORS
		Info("Vous n'êtes pas autorisé à utiliser le service CRM")
	FIN
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Vérifier les droits d'accès à un service SaaS

`<Résultat> = SaaSVérifieService([<Service SaaS>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'utilisateur est autorisé à utiliser le service indiqué, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire (ou si aucun utilisateur n'est connecté). La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de renseignements sur les causes de l'erreur.




**`<Service SaaS> : (Chaîne de caractères optionnelle)`**

Nom du service SaaS pour lequel les droits doivent être vérifiés. Ce nom a été défini dans l'interface d'administration SaaS.
Si ce paramètre n'est pas spécifié, la fonction renvoie <u><u><u><u>Vrai</u></u></u></u> si l'utilisateur est autorisé à utiliser au moins un service du site SaaS.


<a name="SYNTAXE2"></a>

### Vérifier les droits d'accès à un ensemble de services SaaS

`<Résultat> = SaaSVérifieService(<Nom des services SaaS>)`
---

**`<Résultat> : (Tableau)`**

Tableau de Booléens permettant de savoir si le service correspondant est autorisé ou non pour l'utilisateur : 

- <u><u><u><u>Vrai</u></u></u></u> si l'utilisateur est autorisé à utiliser le service indiqué, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire (ou si aucun utilisateur n'est connecté). La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de renseignements sur les causes de l'erreur.




**`<Nom des services SaaS> : (Tableau)`**

Nom du tableau contenant les noms des différents services SaaS pour lesquels les droits doivent être vérifiés. Ces noms ont été définis dans l'interface d'administration SaaS.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Restriction d'accès aux services d'un site SaaS
<a name="restriction_acces_aux_services_site_saas_ELTPARAGRAPHE000071"></a>Quand un client s'abonne à une tarification, il acquiert des licences d'utilisation des services. Il fournit ensuite ces licences à ses utilisateurs.
Un utilisateur peut donc ne pas avoir accès à ce service, car son responsable (utilisateur administrateur) ne lui a pas activé de licence d'utilisation.




## &lt;Graphe&gt;.SauveEMF (Fonction)

***En anglais : &lt;Chart&gt;.SaveEMF***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Sauvegarde un graphe préalablement dessiné dans un fichier au format vectoriel EMF.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Sauvegarde le graphe "GRF_MonGraphe" dans le fichier "C:\Graphes\Graphe.EMF"
GRF_MonGraphe.SauveEMF("C:\Graphes\Graphe.EMF")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Champ Graphe>.SauveEMF(<Nom du fichier EMF>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le graphe a été sauvegardé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Champ Graphe> : (Nom de champ)`**

Nom du champ Graphe présent dans l'éditeur de fenêtres, de pages ou d'états.

**`<Nom du fichier EMF> : (Chaîne de caractères)`**

Nom et chemin complet du fichier EMF à créer. L'extension du fichier est obligatoire. Ce fichier EMF est créé lors de l'appel à la fonction **&lt;Graphe&gt;.SauveEMF**. Si le fichier existait déjà, il est écrasé et recréé.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Enregistrer un graphe
<a name="enregistrer_graphe_ELTPARAGRAPHE000201"></a>

- Pour enregistrer un graphe, le graphe doit avoir été préalablement dessiné (fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md)). Les modifications effectuées depuis le dernier appel à la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md) ne sont pas prises en compte.

- La taille par défaut de l'image est fixée à 639 x 479. Si le graphe n'a jamais été dessiné ou n'est pas visible, ces dimensions sont prises en compte. 



<a name="NOTE0_2"></a>


### Sauvegarde au format EMF
<a name="sauvegarde_format_emf_ELTPARAGRAPHE000231"></a>

- Ce fichier pourra être lu grâce à un outil reconnaissant le format EMF.

- La taille du fichier est identique à la taille de la destination (taille de la fenêtre, du champ ou du fichier).

- Il est conseillé de visualiser le graphe à l'écran avant de l'enregistrer dans un fichier.



<a name="NOTE0_3"></a>


### Différences entre les fonctions grDestinationEMF et &lt;Graphe&gt;.SauveEMF
<a name="differences_entre_les_fonctions_grdestinationemf_docparampagetitleshort_ELTPARAGRAPHE000243"></a>

- La fonction [grDestinationEMF](../WDLang3/3042017.md) permet de définir la destination du prochain dessin du graphe. Lors de l'utilisation de la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md), le graphe sera dessiné dans le fichier spécifié et toutes les modifications effectuées depuis le dernier dessin du graphe seront prises en compte.

- La fonction **&lt;Graphe&gt;.SauveEMF** permet de sauvegarder le graphe directement dans un fichier au format EMF. L'image enregistrée correspond au graphe lors du dernier appel de la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md). Si des modifications ont été effectuées sur le graphe depuis l'appel de la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md), ces modifications ne seront pas prises en compte.

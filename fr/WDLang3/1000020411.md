


## nfcTag (Type de variable)

***En anglais : nfcTag***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **nfcTag** permet de définir toutes les caractéristiques avancées d'un tag NFC (Near Field Communication). Les caractéristiques de ce Tag NFC peuvent être définies et modifiées à l'aide de différentes propriétés WLangage. 

**Remarque** : Pour plus de détails sur la déclaration de ce type de variable et l'utilisation des propriétés WLangage, consultez [Déclaration d'une variable](../Motscles/1514032.md).

### Propriétés spécifiques à la description des variables de type nfcTag
<a name="proprietes_specifiques_description_des_variables_type_nfctag_ELTPARAGRAPHE000042"></a>Les propriétés suivantes peuvent être utilisées pour manipuler un Tag NFC :

| **Nom de la propriété** | **Type manipulé** | **Effet** |
| --- | --- | --- |
| Donnée | Tableau de [nfcDonnée](../WDLang3/1000020406.md) | Tableau contenant : <br><br>- les données lues dans un tag NFC (après un appel à la fonction [NFCLitTag](../WDLang3/1000020416.md)).<br><br><br> |
| Identifiant | Buffer binaire | Identifiant du tag NFC (correspond au "hardware identifiant" d'un tag badgé). Cet identifiant est automatiquement renseigné après une opération de lecture.<br><br>**Cette propriété est disponible en lecture uniquement**.<br><br> |
| LectureSeule | Booléen | <br><br>- <u><u><u><u>Vrai</u></u></u></u> si le tag est en lecture seule, <br><br>- <u><u><u><u>Faux</u></u></u></u> s'il n'est pas protégé en écriture. <br><br><br>**Lors d'une opération de lecture**, cette propriété est automatiquement renseignée. <br><br>**Lors d'une opération d‘écriture**, si la propriété **LectureSeule** vaut <u><u><u><u>Vrai</u></u></u></u>, le tag sera automatiquement protégé en écriture : il ne sera plus possible de modifier ses données. **Cette opération est irréversible**. Tous les tags NFC ne supportent pas ce mode. Pour plus d'informations, consultez l'aide de la fonction [NFCEcritTag](../WDLang3/1000020417.md).<br><br> |
| TailleMax | Entier | Taille maximale (en octets) des données pouvant être stockées dans le tag NFC.<br><br>Cette propriété est automatiquement renseignée après une opération de lecture.<br><br>**Cette propriété est disponible en lecture uniquement**.<br><br> |
| Type | Contante de type Entier | Type du tag correspondant à l'une des valeurs suivantes : <br><br>- *nfcInconnu* : type inconnu. Valeur par défaut si aucune opération de lecture n'a été effectuée avec le tag.<br><br>- *nfcMifareClassic* : NFC Mifare Classic<br><br>- *nfcType1* : NFC Forum Type 1<br><br>- *nfcType2* : NFC Forum Type 2<br><br>- *nfcType3* : NFC Forum Type 3<br><br>- *nfcType4* : NFC Forum Type 4<br><br><br>Le type du tag est automatiquement renseigné après une opération de lecture. <br><br>**Cette propriété est disponible en lecture uniquement**.<br><br> |




<a name="NOTE0_2"></a>


### Remarques générales
<a name="remarques_generales_ELTPARAGRAPHE000161"></a>

- Les fonctions NFC supportent normalement les tags correspondant aux types standardisés 1 à 4 définis par le NFC Forum ([http://www.nfc-forum.org/specs/spec_list/](http://www.nfc-forum.org/specs/spec_list/)) et ne savent lire et écrire que des données encodées au format NDEF (NFC Data Exchange Service).

- Les fonctions NFC ne peuvent pas être utilisées dans le simulateur ou dans l'émulateur.



<a name="NOTE0_3"></a>


### Réinitialisation de la variable
<a name="reinitialisation_variable_ELTPARAGRAPHE000176"></a>Pour réinitialiser le contenu d'une variable de type **nfcTag**, il est possible d'utiliser la fonction [VariableRAZ](../WDLang1/3013069.md).

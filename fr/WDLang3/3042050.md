


## grSauveBMP (Fonction)

***En anglais : grSaveBMP***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Sauvegarde un graphe préalablement dessiné dans un fichier au format BMP. 



![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Sauvegarde le graphe GRF_MonGraphe dans le fichier "C:\Graphes\Graphe.BMP"
grSauveBMP(GRF_MonGraphe, "C:\Graphes\Graphe.BMP")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = grSauveBMP(<Nom du graphe> , <Nom du fichier BMP>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le graphe a été sauvegardé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- soit au nom du champ Graphe présent dans l'éditeur de fenêtres, de pages ou d'états.




**`<Nom du fichier BMP> : (Chaîne de caractères)`**

Nom et chemin complet du fichier BMP à créer. L'extension du fichier est obligatoire. Ce fichier BMP est créé lors de l'appel à la fonction **grSauveBMP**. Si le fichier existait déjà, il est écrasé et recréé.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Enregistrer un graphe
<a name="enregistrer_graphe_ELTPARAGRAPHE000103"></a>

- Pour enregistrer un graphe, le graphe doit avoir été préalablement dessiné (fonction [grDessine](../WDLang3/3042026.md)). Les modifications effectuées depuis le dernier appel à la fonction [grDessine](../WDLang3/3042026.md) ne sont pas prises en compte.

- La taille par défaut de l'image est fixée à 639 x 479. Si le graphe n'a jamais été dessiné ou n'est pas visible, ces dimensions sont prises en compte. 




- Si le graphe a été créé avec la fonction [grCrée](../WDLang3/3042005.md), il est possible de spécifier les dimensions avec la fonction [grFenTaille](../WDLang3/3042016.md).



<a name="NOTE0_2"></a>


### Sauvegarde au format BMP
<a name="sauvegarde_format_bmp_ELTPARAGRAPHE000129"></a>

- Ce fichier pourra être lu grâce à un outil reconnaissant le format BMP.

- La taille du fichier est identique à la taille de la destination (taille de la fenêtre, du champ ou du fichier).

- Il est conseillé de visualiser le graphe à l'écran avant de l'enregistrer dans un fichier.



<a name="NOTE0_3"></a>


### Différences entre les fonctions grDestinationBMP et grSauveBMP
<a name="differences_entre_les_fonctions_grdestinationbmp_docparampagetitleshort_ELTPARAGRAPHE000140"></a>

- La fonction [grDestinationBMP](../WDLang3/3042018.md) permet de définir la destination du prochain dessin du graphe. Lors de l'utilisation de la fonction [grDessine](../WDLang3/3042026.md), le graphe sera dessiné dans le fichier spécifié et toutes les modifications effectuées depuis le dernier dessin du graphe seront prises en compte.

- La fonction **grSauveBMP** permet de sauvegarder le graphe directement dans un fichier au format BMP. L'image enregistrée correspond au graphe lors du dernier appel de la fonction [grDessine](../WDLang3/3042026.md). Si des modifications ont été effectuées sur le graphe depuis l'appel de la fonction [grDessine](../WDLang3/3042026.md), ces modifications ne seront pas prises en compte.

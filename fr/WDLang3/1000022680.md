
## WebserviceLitTypeMIME (Fonction)

***En anglais : WebserviceReadMIMEType***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Retourne le type MIME de la requête reçue dans le cas d'un appel à un Webservice REST.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Accepte un contenu JSON ou XML
SELON WebserviceLitTypeMIME()
	CAS "application/json"
		// Traite le cas JSON
		...
	CAS "application/xml"
		// Traite le cas XML
		...
	AUTRES CAS
		// Traite les autres cas (lance une erreur)
		// Indique que les données reçus sont dans un format non reconnu : 
		// 415 Unsupported Media Type
		WebserviceEcritCodeHTTP(415, sErreur)
		RETOUR
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Type MIME> = WebserviceLitTypeMIME()`
---

**`<Type MIME> : (Chaîne de caractères)`**

Type MIME reçu dans l'entête HTTP Content-Type.
Des centaines de types sont définis dans la norme de communication MIME (disponible sur Internet). Les valeurs les plus courantes et reconnues sont :

- "text/html" : page HTML (\*.htm, \*.html)

- "text/plain" : fichier texte (\*.txt)

- "application/pdf" : document au format PDF (\*.pdf)

- "image/gif" : image au format GIF (\*.gif)

- "image/jpeg" : image au format JPEG (\*.jpg, \*.jpeg)

- "video/mpeg" : vidéo au format MPEG (\*.mpg, \*.mpeg)

- "application/unknown" : permet d'afficher une boîte de dialogue pour proposer le téléchargement du fichier sur le poste de l'internaute.

- "application/msword" : permet d'afficher un fichier Word.

- "application/vnd.ms-excel" : permet d'afficher un fichier Excel.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Cette fonction est disponible uniquement pendant l'appel d'une fonction traitant une requête d'un Webservice REST. Dans tous les autres cas, la fonction retourne une chaîne vide.

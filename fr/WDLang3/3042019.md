
## grDestinationWMF (Fonction)

***En anglais : grDestinationWMF***




	<B><B>Avertissement</B></B>

	A partir de la version <B><B>14</B></B>,  cette fonction est conservée par compatibilité. Il est conseillé d'utiliser la fonction [grDestinationEMF](../WDLang3/3042017.md). 

<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Définit un méta fichier comme destination d'un graphe. Ce méta fichier est créé lors de l'appel à la fonction [grDessine](../WDLang3/3042026.md).

**Remarques** :

- Il est conseillé d'utiliser la fonction [grDestinationEMF](../WDLang3/3042017.md).

- Cette fonction ne peut pas être utilisée avec les champs Graphe. 

- Cette fonction ne peut pas être utilisée sous l'éditeur d'états.



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Définition du fichier "C:\Graphes\Graphe.WMF" comme destination du graphe "MonGraphe"
grDestinationWMF("MonGraphe", "C:\Graphes\Graphe.WMF")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`grDestinationWMF(<Nom du graphe> , <Nom du méta fichier>)`
---

**`<Nom du graphe> : (Chaîne de caractères)`**

Nom du graphe à manipuler, défini avec la fonction [grCrée](../WDLang3/3042005.md).

**`<Nom du méta fichier> : (Chaîne de caractères)`**

Nom du méta fichier dans lequel le graphe doit être enregistré. Ce méta fichier est créé lors de l'appel à la fonction [grDessine](../WDLang3/3042026.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Créer et dessiner un graphe par programmation
<a name="creer_dessiner_graphe_par_programmation_ELTPARAGRAPHE000057"></a>Pour dessiner un graphe :

- créez le graphe (fonction [grCrée](../WDLang3/3042005.md)),

- ajoutez les données (fonctions [grAjouteDonnée](../WDLang3/3042023.md), [grBourseAjouteDonnée](../WDLang3/3042009.md) et [grNuageAjouteDonnéeXY](../WDLang3/3042032.md)),

- définissez la destination du graphe (fonctions [grDestinationBMP](../WDLang3/3042018.md), [grDestinationChamp](../WDLang3/3042020.md), [grDestinationEMF](../WDLang3/3042017.md), [grDestinationFen](../WDLang3/3042031.md) et **grDestinationWMF**),

- dessinez le graphe (fonction [grDessine](../WDLang3/3042026.md)).



<a name="NOTE0_2"></a>


### Fichier WMF pour destination
<a name="fichier_wmf_pour_destination_ELTPARAGRAPHE000099"></a>

- Lors du prochain appel à la fonction [grDessine](../WDLang3/3042026.md), le graphe sera dessiné dans le méta fichier &lt;Nom du méta fichier&gt;.

- Ce fichier pourra être lu grâce à un outil reconnaissant le format WMF.

- Il est possible de définir la taille de ce méta fichier grâce à la fonction [grFenTaille](../WDLang3/3042016.md).

- Il est conseillé de visualiser le graphe à l'écran avant de l'enregistrer dans un fichier.



<a name="NOTE0_3"></a>


### Différences entre les fonctions grDestinationWMF et grSauveWMF
<a name="differences_entre_les_fonctions_grdestinationwmf_grsauvewmf_ELTPARAGRAPHE000115"></a>

- La fonction **grDestinationWMF** permet de définir la destination du prochain dessin du graphe. Lors de l'utilisation de la fonction [grDessine](../WDLang3/3042026.md), le graphe sera dessiné dans le fichier spécifié et toutes les modifications effectuées depuis le dernier dessin du graphe seront prises en compte.

- La fonction [grSauveWMF](../WDLang3/3042044.md) permet de sauvegarder le graphe directement dans un fichier au format WMF. L'image enregistrée correspond au graphe lors du dernier appel de la fonction [grDessine](../WDLang3/3042026.md). Si des modifications ont été effectuées sur le graphe depuis l'appel de la fonction [grDessine](../WDLang3/3042026.md), ces modifications ne seront pas prises en compte.

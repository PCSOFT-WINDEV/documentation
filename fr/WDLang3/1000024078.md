


## &lt;Graphe&gt;.HistoEspacement (Fonction)

***En anglais : &lt;Chart&gt;.ColumnSpacing***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique l'espace avant et après chaque catégorie de données dans : 

- un graphe de type Histogramme, 

- un graphe de type Entonnoir,

- un graphe boursier de type Minimum/Maximum.




Exemple d'histogramme avec espace entre les catégories : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=EspaceHisto.gif)


**Rappel** : Par défaut, l'espace entre deux catégories de données est égal à 10 pixels.

![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Espace entre deux catégories de données dans l'histogramme "GRF_MonGraphe" : 15 pixels
GRF_MonGraphe.HistoEspacement(15)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Champ Graphe>.HistoEspacement(<Espacement>)`
---

**`<Champ Graphe> : (Nom de champ)`**

Nom du champ Graphe à manipuler (présent dans l'éditeur de fenêtres, de pages ou d'états).


**`<Espacement> : (Entier)`**

Espacement (en pixels) entre deux catégories.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Espacement
<a name="espacement_ELTPARAGRAPHE000258"></a>Si la destination est suffisamment grande, l'espacement est exprimé comme suit : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Grespace.gif)


Si la destination ne permet pas d'afficher les espacements, ceux-ci sont réduits à 0 : les barres sont donc "collées" les unes aux autres.
<a name="NOTE0_2"></a>


### Effet de l'espacement
<a name="effet_espacement_ELTPARAGRAPHE000271"></a>

- L'espacement entre deux catégories de données est sans effet sur les types de graphes autres que : 

	- les graphes de type Histogramme,

	- les graphes de type Entonnoir,

	- les graphes Boursier de type Minimum/Maximum.




- Les modifications effectuées avec la fonction **&lt;Graphe&gt;.HistoEspacement** seront prises en compte lors du prochain appel à la fonction [&lt;Graphe&gt;.Dessine](../WDLang3/1000024062.md).

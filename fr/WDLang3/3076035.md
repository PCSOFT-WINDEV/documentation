
## telReprendAppelEnAttente (Fonction)

***En anglais : tapiUnhold***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Reprend un appel en attente. Cet appel a été mis en attente par la fonction [telMetAppelEnAttente](../WDLang3/3076034.md)
<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = telReprendAppelEnAttente(<Identifiant de l'appel>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'appel a été repris, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Identifiant de l'appel> : (Entier)`**

Identifiant de l'appel à reprendre. Cet identifiant est connu dans la procédure WLangage appelée par la fonction [telDémarreDétectionAppel](../WDLang3/3076029.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Limitation
<a name="limitation_ELTPARAGRAPHE000040"></a>Il n'est pas possible d'appeler la fonction **telReprendAppelEnAttente** dans la procédure de détection d'appels (définie grâce à la fonction [telDémarreDétectionAppel](../WDLang3/3076029.md)).


<a name="NOTE0_2"></a>


### Configuration nécessaire
<a name="configuration_necessaire_ELTPARAGRAPHE000165"></a>Les fonctions de téléphonie utilisent la technologie TAPI 2.0 et/ou la technologie TAPI 3.1. Ces technologies sont utilisables sur tous les systèmes.

Attention : cette fonctionnalité n'est pas disponible pour tous les matériels.




## CarteConfigureAPIGgl (Fonction)

***En anglais : MapConfigureGglAPI***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Déclare des paramètres supplémentaires pour le champ Carte qui seront transmis à l'API Google. Cette fonction permet notamment de manipuler des librairies supplémentaires de fonctions, non disponibles dans le champ Carte par défaut. Ces fonctions seront disponibles en code navigateur dans l'objet natif JavaScript retourné par la fonction [CarteRécupèreObjetJS](../WDLang3/1000021474.md).
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Attention : Cette fonction doit être utilisée dans le code d'initialisation du projet. Si cette fonction est utilisée dans un traitement différent, elle n'aura aucun effet.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
CarteConfigureAPIGgl("libraries=geometry,places","language=fr")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`CarteConfigureAPIGgl(<Paramètre 1> [, <Paramètre 2> [... [, <Paramètre N>]]])`
---

**`<Paramètre 1> : (Chaîne de caractères)`**

Premier paramètre transmis à l'API. Le paramètre est de la forme Nom=Valeur.

**`<Paramètre 2> : (Chaîne de caractères optionnelle)`**

Second paramètre optionnel transmis à l'API. Le paramètre est de la forme Nom=Valeur.



**`<Paramètre N> : (Chaîne de caractères optionnelle)`**

Dernier paramètre optionnel transmis à l'API. Le paramètre est de la forme Nom=Valeur.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La configuration est globale au projet. Chaque appel de la fonction remplace les paramètres précédemment définis.

- Si la fonction est appelée sans paramètres, les paramètres précédemment définis sont effacés et aucun paramètre n'est transmis à l'API.

- Si la valeur transmise est invalide, le champ Carte peut ne pas fonctionner.


## géoSuiviActive (Fonction)

***En anglais : geoTrackingEnable***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active la gestion du suivi de position dans une application WINDEV Mobile.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Activation des notifications push
géoSuiviActive(ValideActivationSuivi)
```

```wl
// Gestion du suivi de position
PROCEDURE ValideActivationSuivi(bSuiviActif est booléen, cErreur est une chaîne)
SI PAS bSuiviActif ALORS
	Erreur(cErreur)
	RETOUR
FIN

LIB_Loc = "Localisation active"
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`géoSuiviActive(<Procédure WLangage>)`
---

**`<Procédure WLangage> : (Nom de procédure)`**

Nom de la procédure WLangage appelée : 

- lorsque l'activation a été effectuée, 

- en cas d'erreur : 





Cette procédure est de la forme : 

```txt
PROCEDURE <Nom de la procédure>(<Suivi actif> est un booléen, <Erreur> est une chaîne)
```


- Le paramètre &lt;Suivi actif&gt; est un booléen indiquant si le suivi est actif (<u><u><u><u>Vrai</u></u></u></u>) ou non (<u><u><u><u>Faux</u></u></u></u>). 

- Le paramètre &lt;Erreur&gt; est une chaîne de caractères correspondant au texte de l'erreur d'activation. Ce paramètre est initialisé uniquement si &lt;Suivi Actif&gt; vaut <u><u><u><u>Faux</u></u></u></u>.

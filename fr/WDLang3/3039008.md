


## FluxConnecte (Fonction)

***En anglais : StreamConnect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Connecte un flux (et le champ associé) à une adresse spécifique.

**Remarque** : Cette fonction est identique à la propriété [Adresse](../Proprietes/2510017.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Spécifier le nom de l'appelant
CONF_Conférence1.Identifiant = NomUtilisateur
// Connecter le champ Conférence 
FluxConnecte(CONF_Conférence1, AdresseCorrespondant)
// équivalent à :
// CONF_Conférence1.Adresse = AdresseCorrespondant
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = FluxConnecte(<Champ Conférence> , <Adresse>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la connexion a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Champ Conférence> : (Nom de champ ou chaîne de caractères)`**

Nom du champ Conférence à associer à l'adresse spécifiée.

**`<Adresse> : (Chaîne de caractères)`**

Adresse du correspondant. Cette adresse peut être :

- Une adresse IP au format XXX.XXX.XXX.XXX (par exemple, 125.5.110.100).

- Un nom de serveur (par exemple, pop3.serveur.fr). Cette syntaxe est conseillée. 

- Une chaîne vide (""). Dans ce cas, l'adresse du poste en cours est utilisée (permet notamment de simplifier les tests d'une application).

## Remarques

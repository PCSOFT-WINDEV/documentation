
## OLESauve (Fonction)

***En anglais : OLESave***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Sauve l'objet contenu dans un champ OLE sous forme binaire. Le serveur OLE est invoqué pour sauver l'objet (le serveur n'apparaît pas). Le champ OLE n'est pas mis à jour.

L'objet sauvé pourra ensuite être chargé de nouveau avec la fonction [OLECharge](../WDLang3/3057002.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
OLECréeVide(OLE_OLE1, "PBrush")
// Traitement
// Sauvegarde de l'objet
OLESauve(OLE_OLE1, "OBJET1.OLE")
OLEColle(OLE_OLE1, "PBrush")
// Sauvegarde de l'objet
OLESauve(OLE_OLE1, "OBJET1.OLE")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = OLESauve(<Champ OLE> , <Nom du fichier>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'objet a été sauvegardé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Champ OLE> : (Nom de champ)`**

Nom du champ OLE associé à l'objet OLE. Si ce paramètre correspond à une chaîne vide (""), le champ considéré sera le champ auquel appartient l'événement en cours.

**`<Nom du fichier> : (Chaîne de caractères)`**

Nom et chemin (si nécessaire) du fichier destination. L'extension doit être précisée. Si le chemin n'est pas précisé, le fichier est enregistré dans le répertoire courant.

<a name="NOTE0_1"></a>




## &lt;Graphe&gt;.WaterfallTypeCatégorie (Fonction)

***En anglais : &lt;Chart&gt;.WaterfallCategoryType***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique le type de la catégorie d'un graphe de type Waterfall. 




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// CA 2017
GRF_Waterfall.AjouteDonnée(1, 1, 298000)
GRF_Waterfall.EtiquetteCatégorie(1, "CA 2017")

// Ajout des bénéfices
GRF_Waterfall.AjouteDonnée(1, 2, 420000)
GRF_Waterfall.EtiquetteCatégorie(2, "Bénéfices" + RC + "produits")
GRF_Waterfall.AjouteDonnée(1, 3, 210000)
GRF_Waterfall.EtiquetteCatégorie(3, "Bénéfices" + RC + "services")

// Coûts
GRF_Waterfall.AjouteDonnée(1, 4, -170000)
GRF_Waterfall.EtiquetteCatégorie(4, "Coûts fixes")
GRF_Waterfall.AjouteDonnée(1, 5, -140000)
GRF_Waterfall.EtiquetteCatégorie(5, "Coûts variables")

// CA 2018
GRF_Waterfall.AjouteDonnée(1, 6, 0)
GRF_Waterfall.EtiquetteCatégorie(6, "CA 2018")
GRF_Waterfall.WaterfallTypeCatégorie(6, grSousTotal)

// CA Cumulé
GRF_Waterfall.AjouteDonnée(1, 7, 0)
GRF_Waterfall.EtiquetteCatégorie(7, "CA cumulé")
GRF_Waterfall.WaterfallTypeCatégorie(7, grTotal)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Graphe>.WaterfallTypeCatégorie(<Catégorie> , <Type de la catégorie>)`
---

**`<Champ Graphe> : (Nom de champ)`**

Nom du champ Graphe présent dans l'éditeur de fenêtres, d'états ou de pages.


**`<Catégorie> : (Entier)`**

Numéro de la catégorie à manipuler.

**`<Type de la catégorie> : (Constante de type Entier)`**

Type de la catégorie : 



|   |   |
| --- | --- |
| *grDéfaut* | Valeurs du graphe, positives ou négatives. |
| *grSousTotal* | Valeurs de type sous-total.<br><br>Si la catégorie correspond à un sous-total, la valeur stockée pour cette catégorie est ignorée. Pour cette catégorie, le graphe Waterfall affiche une barre correspondant à la somme de toutes les catégories précédentes à partir du dernier total ou sous-total. |
| *grTotal* | Valeurs de type total.<br><br>Si la catégorie correspond à un total, la valeur stockée pour cette catégorie est ignorée. Pour cette catégorie, le graphe Waterfall affiche une barre correspondant à la somme de toutes les catégories précédentes à partir du début du graphe. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- **Cas particulier : la première catégorie**
	La première catégorie est toujours une catégorie de type "total", mais la valeur spécifiée est affichée : c'est la valeur de départ. 

- Cette fonction s'applique uniquement aux graphes de type Waterfall.

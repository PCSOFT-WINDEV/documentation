


## grLissage (Fonction)

***En anglais : grSmoothing***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Lisse un graphe de type Courbe ou Nuage de points par la méthode des "splines cubiques". 

**Rappel** : Par défaut, un graphe est non lissé.



![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md).








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Lisse le graphe GRF_MonGraphe
grLissage(GRF_MonGraphe, Vrai)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Lisser un graphe

`grLissage(<Nom du graphe> , <Lissage>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- soit au nom du champ Graphe présent dans l'éditeur de fenêtres, de pages ou d'états.




**`<Lissage> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le lissage doit être activé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  





<a name="SYNTAXE2"></a>

### Lisser une série d'un graphe

`grLissage(<Nom du graphe> , <Numéro de la série> , <Lissage>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- soit au nom du champ Graphe présent dans l'éditeur de fenêtres, de pages ou d'états.




**`<Numéro de la série> : (Entier)`**

Numéro de la série à lisser.  

**`<Lissage> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le lissage doit être activé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Le lissage est pris en compte :

- dans les graphes de type courbe (constante *grCourbe*),

- dans les graphes de type nuage de points (constante *grNuage*) dont les points sont reliés (fonction [grNuageReliePoint](../WDLang3/3042040.md)).




**Attention** : Cette fonction n'a aucun effet sur un graphe 3D.

Les paramètres définis avec la fonction **grLissage** seront pris en compte lors du prochain appel à la fonction [grDessine](../WDLang3/3042026.md).


## NotesModifieChamp (Fonction)

***En anglais : NotesModifyItem***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie le champ spécifié du document en cours dans Lotus Notes. Si le champ Lotus Notes spécifié n'existe pas, ce champ est rajouté aux champs du document en cours.

Pour prendre en compte cette modification, il est nécessaire d'enregistrer le document à l'aide de la fonction [NotesEnregistre](../WDLang3/1000005110.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouverture d'une session Lotus Notes
IDSession est un entier
IDSession = EmailOuvreSessionNotes("MotDePasse", ...
			"ServeurMars", "mail\julie.nsf", ...
			"C:\Users\Julie\AppData\Local\Lotus\Notes\Data\user.id")
...
// Lecture du premier document 
NotesDocumentPremier(IDSession)
// Modification du sujet du document
NotesModifieChamp(IDSession, "Subject", "Mon nouveau sujet")
// Enregistrement des modifications
NotesEnregistre(IDSession)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Modifier un champ Lotus Notes de type simple

`<Résultat> = NotesModifieChamp(<Identifiant de la session> , <Champ Lotus Notes> , <Nouvelle valeur>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le champ Lotus Notes a été modifié, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 


En cas d'erreur, la variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.

**`<Identifiant de la session> : (Entier)`**

Identifiant de la session à utiliser. Cet identifiant est renvoyé par la fonction [EmailOuvreSessionNotes](../WDLang3/3032121.md) ou la fonction [NotesOuvreBase](../WDLang3/1000005101.md).

**`<Champ Lotus Notes> : (Chaîne de caractères)`**

Nom du champ Lotus Notes dont on veut modifier la valeur. Si ce champ Lotus Notes n'existe pas, ce champ est rajouté.
Pour connaître le nom et le type des champs du document Notes en cours, utilisez la fonction [NotesListeChamp](../WDLang3/1000005112.md).

**`<Nouvelle valeur> : (Type du champ Lotus Notes modifié)`**

Nouvelle valeur du champ Lotus Notes spécifié.  


<a name="SYNTAXE2"></a>

---

**`<Résultat> : (Booléen)`**

En cas d'erreur, la variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.

**`<Identifiant de la session> : (Entier)`**

Identifiant de la session à utiliser. Cet identifiant est renvoyé par la fonction [EmailOuvreSessionNotes](../WDLang3/3032121.md) ou la fonction [NotesOuvreBase](../WDLang3/1000005101.md).

**`<Champ Lotus Notes> : (Chaîne de caractères)`**

Nom du champ Lotus Notes dont on veut modifier la valeur. Si ce champ Lotus Notes n'existe pas, ce champ est rajouté.
Pour connaître le nom et le type des champs du document Notes en cours, utilisez la fonction [NotesListeChamp](../WDLang3/1000005112.md).

**`<Tableau des nouvelles valeurs> : (Tableau à une dimension, du type du champ Lotus Notes modifié)`**

Tableau contenant les nouvelles valeurs du champ Lotus Notes spécifié.

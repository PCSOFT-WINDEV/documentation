


## grDessine (Fonction)

***En anglais : grDraw***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Dessine un graphe selon les paramètres spécifiés.

**Remarques** : 

- La fonction **grDessine** prend en compte les nouveaux paramètres spécifiés depuis le dernier appel à cette même fonction.

- Cette fonction ne doit pas être utilisée pour les champs Graphe présents dans l'éditeur d'états.


![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Dessin d'un graphe créé par programmation
grDessine("MonGraphe")
```
<a name="Exemple2"></a>

```wl
// Dessin d'un graphe dans un champ Graphe
grDessine(GRF_MonGraphe)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`grDessine(<Nom du graphe>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Ce nom peut correspondre :

- au nom du graphe à dessiner, défini avec la fonction [grCrée](../WDLang3/3042005.md).

- au nom du champ Graphe (présent dans une fenêtre ou une page).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Créer et dessiner un graphe par programmation
<a name="creer_dessiner_graphe_par_programmation_ELTPARAGRAPHE000134"></a>Pour créer et dessiner un graphe :

- créez le graphe (fonction [grCrée](../WDLang3/3042005.md)),

- ajoutez les données (fonctions [grAjouteDonnée](../WDLang3/3042023.md), [grBourseAjouteDonnée](../WDLang3/3042009.md) et [grNuageAjouteDonnéeXY](../WDLang3/3042032.md)),

- définissez la destination du graphe à l'aide des fonctions suivantes : 

	- [grDestinationBMP](../WDLang3/3042018.md), 

	- [grDestinationChamp](../WDLang3/3042020.md), 

	- [grDestinationEMF](../WDLang3/3042017.md), 

	- [grDestinationFen](../WDLang3/3042031.md),

	- [grDestinationImage](../WDLang3/1000020952.md),

	- [grDestinationWMF](../WDLang3/3042019.md). 




- dessinez le graphe (fonction **grDessine**).



<a name="NOTE0_2"></a>


### Graphes et threads
<a name="graphes_threads_ELTPARAGRAPHE000184"></a>Un seul graphe peut être manipulé dans un thread.

Conséquences :

- Deux threads ne peuvent pas manipuler le même graphe.

- Un graphe créé dans un thread est automatiquement détruit à la fin de ce thread. Le graphe existe dans le thread qui contient l'appel à la fonction [grCrée](../WDLang3/3042005.md).




**Remarque** : Il est déconseillé d'appeler la fonction **grDessine** dans un thread. L'appel de cette fonction hors du thread principal peut provoquer des deadlocks et/ou engendrer des comportements aléatoires.




## La structure RendezVous
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000303"></a>
La structure RendezVous est une structure prédéfinie du WLangage (aucune déclaration n'est nécessaire). Cette structure permet de :

- créer ou modifier un rendez-vous.

- récupérer le contenu d'un rendez-vous.


Elle est utilisée pour manipuler les agendas de Outlook et Lotus Notes.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Les différentes variables de la structure RendezVous
<a name="les_differentes_variables_structure_rendezvous_ELTTEXTE000327"></a>
La structure est composée des membres suivants :


|   |   |
| --- | --- |
| **mRendezVous.AvecRépétition** | Booléen.<br>Indique si le rendez-vous est périodique.<br>Cette variable est disponible uniquement en lecture. |
| **mRendezVous.Catégorie** | Chaîne de caractères.<br>Catégorie du rendez-vous |
| **mRendezVous.DateCréation** | Date. <br>Date de création du rendez-vous. |
| **mRendezVous.DateDébut** | Date.<br>Date de début du rendez-vous. |
| **mRendezVous.DateDernièreModification** | Date.<br>Date de dernière modification du rendez-vous. |
| **mRendezVous.DateFin** | Date.<br>Date de fin du rendez-vous<br>Lors de l'ajout d'un rendez-vous (fonction [RendezVousAjoute](../WDLang3/1000011001.md)), si la date de fin de rendez-vous n'est pas spécifiée, la date de fin correspond automatiquement à la date de début.<br>Lors de la modification d'un rendez-vous (fonction [RendezVousModifie](../WDLang3/1000011003.md)), si la date de fin du rendez-vous en cours n'est pas valide, la date de fin correspond automatiquement à la date de début. |
| **mRendezVous.Disponibilité** | Entier.<br>Indique la disponibilité de la personne :<br><br>- 0 : Disponible<br><br>- 1 : Provisoire<br><br>- 2 : Occupé<br><br>- 3 : Absent du bureau<br><br><br>Ces informations sont disponibles sous Outlook uniquement. |
| **mRendezVous.EnDehors** | Booléen.<br>Indique s'il y a un rendez-vous courant (<u><u><u><u>Vrai</u></u></u></u>) ou non (<u><u><u><u>Faux</u></u></u></u>). Cette variable permet par exemple de déterminer si la boucle de lecture des rendez-vous est terminée ou non. |
| **mRendezVous.Enligne** | Booléen.<br>Indique si le rendez-vous est en ligne. |
| **mRendezVous.ID** | Chaîne de caractères.<br>Identifiant unique du rendez-vous. Cette variable est disponible en lecture uniquement. |
| **mRendezVous.JournéeEntière** | Booléen.<br>Indique si le rendez-vous est défini pour la journée entière. |
| **mRendezVous.Lieu** | Chaîne de caractères.<br>Lieu du rendez-vous. |
| **mRendezVous.NbParticipant** | Entier.<br>Nombre de participants au rendez-vous. |
| **mRendezVous.NbParticipantCc** | Entier.<br>Nombre de participants cachés au rendez-vous. |
| **mRendezVous.NbParticipantCci** | Entier.<br>Nombre de participants cachés et invisibles au rendez-vous. |
| **mRendezVous.Notes** | Chaîne de caractères.<br>Notes sur le rendez-vous. |
| **mRendezVous.Objet** | Chaîne de caractères.<br><br><br>Objet du rendez-vous. |
| **mRendezVous.Participant** | Tableau de chaîne de caractères.<br>Nom des participants du rendez-vous (Outlook uniquement, jusqu'à Outlook version 2003).<br><br>Pour accéder aux différents participants du rendez-vous, utilisez le format suivant : <br><br><pre><code>mRendezVous.Participant[<Num participant>]</code></pre><br>Où &lt;Num participant&gt; correspond au numéro du participant voulu. Le nombre de participants du rendez-vous peut être connu grâce à la variable mRendezVous.NbParticipant.<br><br>Remarque : Sous Outlook :<br><br>- Par défaut, sous Outlook, les participants correspondent aux contacts du rendez-vous.<br><br>- Pour gérer les versions récentes de Outlook, il est possible d'indiquer le mode de gestion des contacts dans les fonctions [EmailOuvreSessionOutlook](../WDLang3/3032128.md) et [OutlookOuvreSession](../WDLang3/3032123.md). La variable mRendezVous.Participant contiendra alors soit les contacts des rendez-vous, soit les participants du rendez-vous.<br><br><br> |
| **mRendezVous.ParticipantCc** | Tableau de chaîne de caractères.<br>Nom des participants cachés du rendez-vous. <br><br>Pour accéder aux différents participants cachés du rendez-vous, utilisez le format suivant : <br><br><pre><code>mRendezVous.ParticipantCc[<Num participant>]</code></pre><br>Où &lt;Num participant&gt; correspond au numéro du participant caché voulu. Le nombre de participants cachés du rendez-vous peut être connu grâce à la variable mRendezVous.NbParticipantCc.<br><br>**Remarques** :<br><br>- Cette variable est utilisée pour les agendas Lotus Notes.<br><br>- Pour gérer les versions récentes de Outlook, il est possible d'indiquer le mode de gestion des contacts dans les fonctions [EmailOuvreSessionOutlook](../WDLang3/3032128.md) et [OutlookOuvreSession](../WDLang3/3032123.md). Si le mode de gestion correspond à la constante *outlookRDVAvecParticipant*, la variable mRendezVous.ParticipantCc contiendra alors les participants cachés du rendez-vous.<br><br><br> |
| **mRendezVous.ParticipantCci** | Tableau de chaîne de caractères.<br>Nom des participants cachés et invisibles du rendez-vous. <br><br>Pour accéder aux différents participants cachés et invisibles du rendez-vous, utilisez le format suivant : <br><pre><code>mRendezVous.ParticipantCci[<Num participant>]</code></pre><br>Où &lt;Num participant&gt; correspond au numéro du participant caché et invisible voulu. Le nombre de participants cachés et invisibles du rendez-vous peut être connu grâce à la variable mRendezVous.NbParticipantCci.<br><br>**Remarques** :<br><br>- Cette variable est utilisée pour les agendas Lotus Notes.<br><br>- Pour gérer les versions récentes de Outlook, il est possible d'indiquer le mode de gestion des contacts dans les fonctions [EmailOuvreSessionOutlook](../WDLang3/3032128.md) et [OutlookOuvreSession](../WDLang3/3032123.md). Si le mode de gestion correspond à la constante *outlookRDVAvecParticipant*, la variable mRendezVous.ParticipantCci contiendra alors les participants cachés et invisibles du rendez-vous.<br><br><br> |
| **mRendezVous.Privé** | Booléen.<br>Indique si le rendez-vous est privé. |
| **mRendezVous.RappelAuto** | Booléen.<br>Indique si un rappel automatique doit être effectué pour le rendez-vous. Ce rappel sera réalisé &lt;mRendezVous.RappelAvant&gt; minutes avant l'heure du rendez-vous. |
| **mRendezVous.RappelAvant** | Entier.<br>Si un rappel automatique est demandé, ce rappel sera réalisé &lt;mRendezVous.RappelAvant&gt; minutes avant l'heure du rendez-vous.<br>Cette variable est prise en compte uniquement si un rappel automatique doit être effectué (mRendezVous.RappelAuto). |

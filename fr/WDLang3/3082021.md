


## zipEstMulti (Fonction)

***En anglais : zipIsMulti***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le type de l'archive : archive mono-partie ou archive multi-parties.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003082021&name=zipestmulti_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
MonArchive est un zipArchive

// Ouverture de l'archive "MonArchive"
ResOuvreArchive = zipOuvre(MonArchive, "D:\Sauvegardes\Sauvegarde.zip")

SI ResOuvreArchive = 0 ALORS
	...
FIN
// Archive mono-partie ou multi-parties ?
ResEstMulti = zipEstMulti(MonArchive)
// Affichage du résultat
SI ResEstMulti = Vrai ALORS
	Info("L'archive est multi-parties")
SINON
	Info("L'archive est mono-partie")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = zipEstMulti(<Archive>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'archive est multi-parties, 

- <u><u><u><u>Faux</u></u></u></u> si l'archive est mono-partie.




**`<Archive> : (Chaîne de caractères ou variable de type zipArchive)`**

Nom de l'archive à manipuler.
Ce nom peut correspondre :

- soit au nom d'une archive défini avec la fonction [zipOuvre](../WDLang3/3082044.md) ou la fonction [zipCrée](../WDLang3/3082003.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) soit au nom d'une variable de type [zipArchive](../WDLang3/1000018679.md).






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000121"></a>

- La recherche du type d'archive peut être réalisée uniquement si l'archive existe (la création d'une archive s'effectue avec la fonction [zipCrée](../WDLang3/3082003.md)).

- Cette fonction est disponible pour les archives au format WDZ, ZIP et 7z. 




![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Rappel : Seules les archives au format ZIP, TAR et TGZ (TAR.GZ) sont disponibles.
<a name="NOTE0_2"></a>


### Actions réalisables sur une archive multi-parties
<a name="actions_realisables_sur_une_archive_multiparties_ELTPARAGRAPHE000138"></a>Certaines fonctions d'archivage sont utilisables uniquement sur des archives mono-partie (ajout et suppression de fichiers, création d'un exécutable auto-extractible, ...). Pour effectuer ces opérations, il suffit de regrouper les sous-archives ([zipRegroupe](../WDLang3/3082046.md)) pour obtenir une archive mono-partie.

A partir d'une archive multi-parties, il est uniquement possible de :

- Regrouper les différentes parties (fonction [zipRegroupe](../WDLang3/3082046.md)).

- Obtenir diverses informations sur l'archive et sur les fichiers de l'archive (fonctions [zipExtraitChemin](../WDLang3/3082023.md), [zipInfoFichier](../WDLang3/3082040.md), [zipListeFichier](../WDLang3/3082031.md), ...).

- Extraire les fichiers (fonctions [zipExtraitFichier](../WDLang3/3082019.md), [zipExtraitTout](../WDLang3/3082011.md)).



<a name="NOTE0_3"></a>


### Actions réalisables sur une archive mono-partie
<a name="actions_realisables_sur_une_archive_monopartie_ELTPARAGRAPHE000174"></a>Toutes les fonctions d'archivage sont utilisables sur des archives mono-partie, exceptée la fonction [zipRegroupe](../WDLang3/3082046.md) (permettant de regrouper les différentes parties d'une archive entre elles).




## grDégradé (Fonction)

***En anglais : grGradient***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active ou désactive l'affichage des dégradés de couleurs dans un graphe. 

**Rappel** : Par défaut, les couleurs d'un graphe sont dégradées.
![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Graphe créé par programmation
grCrée("MonGraphe", grChandelier)
...
// Désactive les dégradés de couleurs
grDégradé("MonGraphe", Faux)
```
<a name="Exemple2"></a>

```wl
// Champ Graphe
// Désactive les dégradés de couleurs
grDégradé(GRF_MonGraphe, Faux)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`grDégradé(<Nom du graphe> , <Dégradé>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- soit au nom du champ Graphe présent dans l'éditeur de fenêtres, de pages ou d'états.




**`<Dégradé> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> pour afficher le graphe en utilisant les dégradés de couleurs, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Par défaut, les graphes utilisent les dégradés de couleurs. Tous les types de graphes peuvent afficher des couleurs dégradées.

- Le paramétrage réalisé avec la fonction **grDégradé** sera pris en compte lors du prochain appel à la fonction [grDessine](../WDLang3/3042026.md).

- Lors de l'impression, le mode dégradé ne sera pas pris en compte pour les graphes de type Secteur.

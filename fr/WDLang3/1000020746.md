


## xmlSignatureRéférence (Type de variable)

***En anglais : xmlSignatureReference***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **xmlSignatureRéférence** permet d'ajouter une partie référence dans la signature XML. Les caractéristiques de ce type de variable peuvent être définies et modifiées à l'aide de différentes propriétés WLangage. 

Ce type de variable est utilisé par le type [xmlSignature](../WDLang3/1000020736.md). 

**Remarque** : Pour plus de détails sur la déclaration de ce type de variable et l'utilisation des propriétés WLangage, consultez [Déclaration d'une variable](../Motscles/1514032.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ref est un xmlSignatureRéférence
ref.URI = "#id"
ref.AlgorithmeHash = HA_SHA_160
ref.Transformation[1] = atEnveloppée
ref.Transformation[2] = atC14NExclu
```

<a name="NOTE0"></a>

## Remarques
<a name="NOTE0_1"></a>


### Propriétés spécifiques à la description des variables de type xmlSignatureRéférence
<a name="proprietes_specifiques_description_des_variables_type_xmlsignaturereference_ELTPARAGRAPHE000057"></a>Les propriétés suivantes peuvent être utilisées pour manipuler une variable de type **xmlSignatureRéférence** :

| **Nom de la propriété** | **Type manipulé** | **Effet** |
| --- | --- | --- |
| AlgorithmeHash | Constante de type Entier | Méthode utilisée pour le hachage de la référence : <br><br>- *HA_MD5_128* : 	Fonctions de hachage cryptographique classiques de type MD5.<br><br>- *HA_RIPEMD_160* : Fonctions de hachage cryptographique classiques de type RIPEMD.<br><br>- *HA_SHA_160* : Fonctions de hachage cryptographique classiques de type SHA.<br><br>- *HA_SHA_256* : Fonctions de hachage cryptographique classiques de type SHA.<br><br>- *HA_SHA_384* : Fonctions de hachage cryptographique classiques de type SHA.<br><br>- *HA_SHA_512* : 	Fonctions de hachage cryptographique classiques de type SHA.<br><br><br> |
| Transformation | Tableau de constantes de type Entier | Liste des transformations à appliquer aux données avant de passer l'algorithme de hachage. Les transformations possibles sont : <br><br>- *atAucune* : aucune transformation. <br><br>- *atC14N11Inclus* : Transformation C14N11 inclusive. <br><br>- *atC14N11InclusAvecCommentaire* : Transformation C14N11 inclusive avec commentaires.<br><br>- *atC14NExclu* : Transformation C14N exclusive.<br><br>- *atC14NExcluAvecCommentaire* : Transformation C14N exclusive avec commentaires.<br><br>- *atC14NInclus* : Transformation C14N inclusive.<br><br>- *atC14NInclusAvecCommentaire* : Transformation C14N inclusive avec commentaires.<br><br>- *atEnveloppée* : Transformation par méthode enveloppée.<br><br><br> |
| URI | Chaîne de caractères | Emplacement à signer. |

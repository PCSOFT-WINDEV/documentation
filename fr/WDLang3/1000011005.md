
## RendezVousPremier (Fonction)

***En anglais : AppointmentFirst***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Lit le premier rendez-vous présent dans l'agenda d'une messagerie Lotus Notes ou Outlook.

Lors de la lecture d'un rendez-vous, les données de ce rendez-vous sont mis à jour dans la [structure RendezVous](../WDLang3/1000011010.md). Pour récupérer les données du rendez-vous en cours, il suffit de récupérer les valeurs de la [structure RendezVous](../WDLang3/1000011010.md).

**Rappel** : Cette fonction ne permet pas de lire le premier rendez-vous présent dans l'agenda d'une messagerie Outlook Express.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouverture d'une session Lotus Notes
IDSession est un entier
IDSession = EmailOuvreSessionNotes("MotDePasse", "ServeurMars", "mail/julie.nsf", "C:\Lotus Notes\julie.id")
// Lecture du premier rendez-vous
RendezVousPremier(IDSession)
// Ajout des rendez-vous trouvés dans un champ Liste
TANTQUE mRendezVous.EnDehors = Faux
	ListeAjoute(LISTE_ListeRDV, mRendezVous.Objet + TAB + mRendezVous.Lieu)
	RendezVousSuivant(IDSession)
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = RendezVousPremier(<Identifiant de la session>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le premier rendez-vous a été lu, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*. 




**`<Identifiant de la session> : (Entier)`**

Identifiant de la session à utiliser. Cet identifiant est renvoyé par la fonction [EmailOuvreSessionNotes](../WDLang3/3032121.md), [EmailOuvreSessionOutlook](../WDLang3/3032128.md), [NotesOuvreBase](../WDLang3/1000005101.md) ou [OutlookOuvreSession](../WDLang3/3032123.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La variable mRendezVous.EnDehors permet de savoir s'il y a un rendez-vous courant.

- Sous Outlook, il est possible de spécifier le mode de gestion des contacts/participants lors de l'ouverture de la session (fonction [EmailOuvreSessionOutlook](../WDLang3/3032128.md) ou fonction [OutlookOuvreSession](../WDLang3/3032123.md)).

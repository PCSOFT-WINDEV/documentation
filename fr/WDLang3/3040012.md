
## Constantes FTP
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


| Constante | Description | Utilisée par les fonctions |
| --- | --- | --- |
| **ftpAccès** | Type de date/heure : Date/Heure du dernier accès au fichier | [FTPDate](../WDLang3/3040015.md), <br>[FTPHeure](../WDLang3/3040013.md) |
| **ftpCréation** | Type de date/heure : Date/Heure de création du fichier | [FTPDate](../WDLang3/3040015.md), <br>[FTPHeure](../WDLang3/3040013.md) |
| **ftpErreurIdentification** | Signale une erreur de connexion due à une erreur d'identification | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpesParamètreModeSecurisé** | Permet de modifier le chiffrement en FTPES. Deux modes sont disponibles : <br><br>- Le chiffrement SSL s'applique aux données et aux commandes FTP. Ce mode correspond au mode de fonctionnement par défaut des fonctions en FTPES.<br><br>- Le chiffrement SSL s'applique uniquement aux commandes FTP. Ce mode peut être requis pour certains serveurs FTPES.<br><br><br> | [FTPParamètre](../WDLang3/1000020947.md) |
| **ftpFichier** | Élément à rechercher : recherche effectuée uniquement sur les fichiers | [FTPListeFichier](../WDLang3/3040018.md) |
| **ftpIgnoreCertificatExpiré** | La date du certificat est ignorée. | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpIgnoreCertificatInvalide** | Le certificat est ignoré. | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpIgnoreNomCertificatInvalide** | Le nom du site figurant dans le certificat est ignoré. | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpIgnoreRévocation** | La révocation du certificat est ignorée. | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpModeActif** | **Connexion active** au serveur FTP. <br><br>Lors d'un transfert de fichiers, c'est le client qui prend l'initiative du transfert. Ce type de connexion peut être refusé par certains serveurs FTP protégés. | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpModeASCII** | Transfert de fichiers effectué en mode ASCII : Ce mode est utilisé pour un transfert entre deux systèmes d'exploitation différents, par exemple UNIX vers Windows : le fichier transféré est livré dans le format ASCII du système destinataire. | [FTPEnvoie](../WDLang3/3040014.md), <br>[FTPRécupère](../WDLang3/3040002.md) |
| **ftpModeBinaire** | Transfert de fichiers effectué en mode binaire : le fichier réceptionné est strictement identique au fichier envoyé | [FTPEnvoie](../WDLang3/3040014.md), <br>[FTPRécupère](../WDLang3/3040002.md) |
| **ftpModePassif** | **Connexion passive** au serveur FTP (mode PASV). <br><br>Lors d'un transfert de fichiers, c'est le serveur qui prend l'initiative du transfert. Ce type de connexion permet de franchir certains "firewalls".<br><br><br><br>Remarque : Si la connexion au serveur est en IPV6, alors la connexion se fera en mode passif étendu (car le mode passif ne supporte pas IPV6). | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpModePassifEtendu** | **Connexion passive étendue** au serveur FTP (mode EPSV). <br><br>Lors d'un transfert de fichiers, c'est le serveur qui prend l'initiative du transfert. Ce type de connexion permet de franchir certains "firewalls". <br><br><br><br>Si le serveur ne supporte pas le mode passif étendu (la commande FEAT est envoyée au serveur pour connaître la capacité), la connexion est dégradée en mode passif simple. | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpModePassifIgnoreIP** | **Connexion passive** au serveur FTP (mode PASV). <br><br>Lors d'un transfert de fichiers en mode passif le serveur donne le port et l'adresse IP pour le transfert des données. Avec ce paramètre, seul le port est pris en compte, l'adresse IP est ignorée.<br>Ce type de connexion permet d'utiliser des serveurs FTP qui retournent des adresses IP non routables mais qui sont joignables par l'adresse IP de connexion | [FTPConnecte](../WDLang3/3040016.md) |
| **ftpModification** | Type de date/heure : Date/Heure de la dernière modification du fichier | [FTPDate](../WDLang3/3040015.md), <br>[FTPHeure](../WDLang3/3040013.md) |
| **ftpParamètreMode** | Permet de choisir le mode de fonctionnement du FTP. Deux modes sont disponibles :<br><br>- les fonctions de gestion du FTP utilisent le module "wininet" de Windows. Ce mode correspond au mode de fonctionnement par défaut des fonctions FTP.<br><br>- les fonctions de gestion du FTP utilisent un module indépendant de Windows et supportent l'UTF8. Attention : Dans ce mode, la fonction [FTPCommande](../WDLang3/3040017.md) n'est pas supportée.<br><br><br> | [FTPParamètre](../WDLang3/1000020947.md) |
| **ftpProxyAucun** | Permet d'annuler l'utilisation du proxy FTP. | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxyOpen** | Type de proxy à utiliser : Proxy FTP nécessitant la commande "OPEN". | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxySite** | Type de proxy à utiliser : Proxy FTP nécessitant la commande "SITE". | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxySOCKS5** | Proxy FTP utilisant le protocole SOCKS5. | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxyUtilisateurRegroupé** | Type de proxy à utiliser : Proxy FTP nécessitant la commande "USER" avec l'utilisateur et le mot de passe. | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxyUtilisateurSéparé** | Type de proxy à utiliser : Proxy FTP nécessitant la commande "USER" et la commande "PASS" séparée. | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpRépertoire** | Élément à rechercher : recherche effectuée uniquement sur les répertoires | [FTPListeFichier](../WDLang3/3040018.md) |


## ClavierVisible (Fonction)

***En anglais : SIPVisible***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de rendre le clavier visible.






<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ClavierVisible(<Visible/Invisible>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le clavier en cours était visible avant l'appel de cette fonction, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. 




**`<Visible/Invisible> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> pour rendre le clavier en cours visible, 

- <u><u><u><u>Faux</u></u></u></u> pour rendre le clavier en cours invisible.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Option "Activer le clavier en saisie"
<a name="option_activer_clavier_saisie_ELTPARAGRAPHE000092"></a>L'option "Activer le clavier en saisie" (onglet "Détail" de la fenêtre de description d'un champ de saisie) permet de rendre visible automatiquement le clavier en cours lorsque le champ est en saisie.


<a name="NOTE0_3"></a>


### Gestion des erreurs
<a name="gestion_des_erreurs_ELTPARAGRAPHE000108"></a>**Attention** : La fonction **ClavierVisible** ne retourne pas de code d'erreur. Pour savoir si cette fonction a généré une erreur, utilisez la variable [ErreurDétectée](../WDLang1/3087001.md). En cas d'erreur, il est possible de connaître le détail de l'erreur grâce à la fonction [ErreurInfo](../WDLang1/3013008.md).

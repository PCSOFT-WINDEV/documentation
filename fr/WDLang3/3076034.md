
## telMetAppelEnAttente (Fonction)

***En anglais : tapiHold***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Met un appel en attente. Cet appel sera en attente jusqu'à l'utilisation de la fonction [telReprendAppelEnAttente](../WDLang3/3076035.md).
<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = telMetAppelEnAttente(<Identifiant de l'appel>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'appel a été mis en attente, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Identifiant de l'appel> : (Entier)`**

Identifiant de l'appel à mettre en attente. Cet identifiant est connu dans la procédure WLangage appelée par la fonction [telDémarreDétectionAppel](../WDLang3/3076029.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Limitation
<a name="limitation_ELTPARAGRAPHE000041"></a>Il n'est pas possible d'appeler la fonction **telMetAppelEnAttente** dans la procédure de détection d'appels (définie grâce à la fonction [telDémarreDétectionAppel](../WDLang3/3076029.md)).


<a name="NOTE0_2"></a>


### Configuration nécessaire
<a name="configuration_necessaire_ELTPARAGRAPHE000166"></a>Les fonctions de téléphonie utilisent la technologie TAPI 2.0 et/ou la technologie TAPI 3.1. Ces technologies sont utilisables sur tous les systèmes.

Attention : cette fonctionnalité n'est pas disponible pour tous les matériels.

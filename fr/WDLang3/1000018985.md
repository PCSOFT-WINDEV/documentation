
## HTTPParamètre (Fonction)

***En anglais : HTTPConfigure***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de paramétrer les fonctions utilisant le protocole HTTP ([HTTPRequête](../WDLang3/3043007.md), [HTTPEnvoieFormulaire](../WDLang3/1000017190.md), [SOAPExécute](../WDLang3/3069013.md), [SOAPExécuteXML](../WDLang3/3069014.md), [Webservices importés](../WDLang3/3086001.md), [SFExécuteRequête](../WDLang5/1000018438.md), etc.).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HTTPParamètre(httpParamètreMode, 1)
HTTPRequête("www.pcsoft.fr")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`HTTPParamètre(<Option à modifier> , <Nouvelle valeur>)`
---

**`<Option à modifier> : (Constante de type Entier)`**

Paramètre à modifier : 


|   |   |
| --- | --- |
| *httpParamètreDésactiveCache* | En mode sécurisé (HTTPS), permet de gérer le cache de Internet Explorer. La  associée à cette constante peut être :<br><br>- 0 (<u><u><u><u>Faux</u></u></u></u> par défaut) : Permet de réactiver le cache de Internet Explorer.<br><br>- 1 (<u><u><u><u>Vrai</u></u></u></u>) : Permet de désactiver le cache de Internet Explorer.<br><br><br> |
| *httpParamètreDésactiveCookie* | En mode sécurisé (HTTPS), permet de paramétrer de mode de gestion des cookies HTTP de Internet Explorer. La &lt;Nouvelle valeur&gt; associée à cette constante peut être :<br><br>- 0 (<u><u><u><u>Faux</u></u></u></u> par défaut) : Permet de réactiver la gestion des cookies HTTP de Internet Explorer.<br><br>- 1 (<u><u><u><u>Vrai</u></u></u></u>) : Permet de désactiver la gestion des cookies HTTP de Internet Explorer. <br><br><br> |
| *httpParamètreMode* | Permet de paramétrer le mode de gestion des requêtes. La &lt;Nouvelle valeur&gt; associée à cette constante peut être :<br><br>- 0 (<u><u><u><u>Faux</u></u></u></u> par défaut) : Gestion des requêtes HTTP effectuée par le framework commun aux exécutables WINDEV, et aux sites et webservices WEBDEV. Dans ce mode, si une requête nécessite HTTPS, le mode 1 sera automatiquement utilisé.<br><br>- 1 (<u><u><u><u>Vrai</u></u></u></u>) : Gestion des requêtes HTTP et/ou HTTPS effectuée par l'API Windows qui exécute le processus.<br><br>- 2 : Gestion des requêtes HTTP et/ou HTTPS effectuée par une implémentation multiplateforme commune à tous les frameworks. <br>	Dans ce mode, par exemple, une application WINDEV utilise pour les échanges HTTPS la même implémentation qu'une application WINDEV Mobile pour iOS ou Android. Ce mode peut être utilisé pour remplacer l'API de Windows lorsqu'elle ne permet pas de chiffrer un échange HTTPS.<br><br><br> |





**`<Nouvelle valeur> : (Entier)`**

Nouvelle valeur du paramètre modifié.

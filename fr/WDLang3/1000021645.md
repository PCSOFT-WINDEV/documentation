
## FluxConfigure (Fonction)

***En anglais : StreamConfigure***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Spécifie les paramètres réseau permettant d'utiliser des champs Conférence sur Internet. 

Remarque : Cette fonction doit être appelée avant l'ouverture de la fenêtre contenant les champs Conférence. Dans le cas contraire, cette fonction n'est pas prise en compte. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Utilisation de la syntaxe 1
sConfiguration est une chaîne = [
{
'url': 'stun:stun.l.google.com:19302'
},
{
'url': 'turn:monserveurturn.org'
'username': 'login2'
'credential': motdepasse2'
}
]
FluxConfigure(sConfiguration)
Ouvre(FEN_VideoConférenceInternet)
```



```wl
// Utilisation de la syntaxe 2
FluxConfigure("stun.l.google.com:19302", "monserveurturn.org", "user87", "Secret_87")
Ouvre(FEN_VideoConférenceInternet)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Configuration via une chaîne spécifique

`FluxConfigure(<Paramètres>)`
---

**`<Paramètres> : (Chaîne de caractères)`**

Paramètres permettant de paramétrer les adresses des serveurs relais à utiliser pour que la communication soit possible via Internet. Cette chaîne de caractères est de la forme : 

```txt
{
'url': 'stun:serveurSturn'
},
{
'url': 'turn: serveursTurn'
'username': 'nomutilisateur'
'credential': motdepasse'
}
```


Attention : En cas d'erreur de saisie dans cette chaîne, aucune erreur ne sera renvoyée. 


<a name="SYNTAXE2"></a>

### Configuration via différents paramètres

`FluxConfigure(<Serveur STUN> , <Serveur TURN> , <Login> , <Mot de passe>)`
---

**`<Serveur STUN> : (Chaîne de caractères)`**

Adresse d'un serveur STUN.

**`<Serveur TURN> : (Chaîne de caractères)`**

Adresse d'un serveur TURN. 

**`<Login> : (Chaîne de caractères)`**

Login à utiliser sur le serveur TURN. 

**`<Mot de passe> : (Chaîne de caractères)`**

Mot de passe associé au login pour le serveur TURN.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Si 2 machines sont sur le même réseau local, elles peuvent communiquer directement : la fonction **FluxConfigure** n'est pas nécessaire.

- Si 2 machines sont connectées à Internet via des firewalls (ou via des box par exemple), les communications directes entre ces machines sont souvent bloquées. Il est nécessaire d'utiliser alors la fonction **FluxConfigure**. 



<a name="NOTE0_2"></a>


### Utilisation de Champs Conférence via Internet
<a name="utilisation_champs_conference_via_internet_ELTPARAGRAPHE000064"></a>

- Selon la typologie du réseau et de ses paramètres de sécurité (ports bloqués par un firewall par exemple), il n'est pas forcément nécessaire d'utiliser un serveur TURN et/ou un serveur STUN. 

- Des serveurs STUN sont disponibles en libre accès.

- Les serveurs TURN ne sont pas disponibles en libre accès. Il est nécessaire d'en louer un ou d'en monter un sur Internet.




## &lt;Variable zipArchive&gt;.ChercheFichier (Fonction)

***En anglais : &lt;zipArchive variable&gt;.FindFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Recherche un fichier dans une archive (au format CAB, ZIP, RAR, WDZ, 7z, TAR ou TGZ (TAR.GZ)). La recherche est effectuée dans le chemin mémorisé du fichier.








<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png) 
```wl
// Création de l'archive 
MonArchive est un zipArchive
ResCréation = MonArchive.Crée("C:\Temp\Archive.zip")
SI ResCréation = 0 ALORS
	...
	// Recherche d'un fichier dont le chemin mémorisé contient "Temp"
	ResFichierTrouvé = MonArchive.ChercheFichier("Temp", Faux)
	...
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Archive>.ChercheFichier(<Chemin recherché> [, <Mode de recherche> [, <Début>]])`
---

**`<Résultat> : (Entier)`**



- Indice du fichier recherché dans l'archive, 

- -1 en cas d'erreur ou si le fichier n'est pas trouvé.




**`<Archive> : (Variable de type zipArchive)`**

Nom de la variable de type [zipArchive](../WDLang3/1000018679.md) correspondant à l'archive dans laquelle le fichier doit être recherché.

**`<Chemin recherché> : (Chaîne de caractères)`**

Chemin complet ou partiel du fichier à rechercher. Ce paramètre peut correspondre :

- un nom de fichier,

- un chemin complet de fichier,

- un chemin et un nom de fichier,

- ...


Lorsqu'un chemin est indiqué le séparateur de répertoire est l'anti-slash (\\), et le répertoire racine ne doit pas être précisé.

La recherche ne tient pas compte des majuscules et minuscules.

**`<Mode de recherche> : (Booléen optionnel)`**



- <u><u><u><u>Vrai</u></u></u></u> (valeur par défaut) pour une recherche à l'identique, 

- <u><u><u><u>Faux</u></u></u></u> pour une recherche générique. La recherche ne tient pas compte des majuscules et minuscules.




**Archives au format 7z** : Ce paramètre n'est pas disponible.

**`<Début> : (Entier optionnel)`**

Indice du fichier à partir duquel doit s'effectuer la recherche (1 par défaut, c'est-à-dire le premier fichier intégré à l'archive). 

**Archives au format 7z** : Ce paramètre n'est pas disponible.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Mode de recherche
<a name="mode_recherche_ELTPARAGRAPHE000307"></a>Selon le &lt;Mode de recherche&gt; choisi, le résultat pourra être différent :

- Si la recherche est à l'identique, la fonction **&lt;Variable zipArchive&gt;.ChercheFichier** localisera les fichiers dont le chemin mémorisé est strictement égal au &lt;Chemin recherché&gt;. 
	Par exemple : une archive contient les fichiers "Temp.doc", "Temp1.doc" et "Temp2.doc". La recherche à l'identique de "Temp.doc" permettra de trouver uniquement le fichier "Temp.doc".

- Si la recherche est générique, la fonction **&lt;Variable zipArchive&gt;.ChercheFichier** localisera tous les fichiers dont le chemin mémorisé contient &lt;Chemin mémorisé&gt;.
	Par exemple : une archive contient les fichiers "Temp.doc", "Temp1.doc" et "Temp2.doc". La recherche générique de la chaîne de caractères "Temp" permettra de trouver les trois fichiers.


**Rappel** : Le chemin mémorisé est défini avec la fonction [&lt;Variable zipArchive&gt;.AjouteFichier](../WDLang3/1000019050.md) ou [&lt;Variable zipArchive&gt;.AjouteRépertoire](../WDLang3/1000019078.md).
<a name="NOTE0_2"></a>


### Indice des fichiers dans l'archive
<a name="indice_des_fichiers_dans_archive_ELTPARAGRAPHE000342"></a>Lors de l'ajout d'un fichier dans une archive, un indice est automatiquement attribué au fichier. Cet indice correspond à l'ordre d'intégration des fichiers dans l'archive. Pour sélectionner un fichier de l'archive, il est possible d'utiliser :

- soit l'indice de l'élément.

- soit le chemin mémorisé de l'élément.

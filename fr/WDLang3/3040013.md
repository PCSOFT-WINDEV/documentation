


## FTPHeure (Fonction)

***En anglais : FTPTime***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie les différentes heures (heure de création, de modification ou d'accès) d'un fichier présent sur un serveur FTP (File Transfer Protocol).
![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Renvoie l'heure de modification d'un fichier présent sur un serveur FTP (File Transfer Protocol).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003040013&name=ftpheure_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Connaître l'heure de modification du fichier
// "/Documents/Fichier.txt" présent sur le serveur FTP
ResHeure = FTPHeure(7, "/Documents/Fichier.txt")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Récupérer les différentes heures du dernier fichier consulté

`<Résultat> = FTPHeure([<Type d'heure>])`
---

**`<Résultat> : (Chaîne de caractères)`**



- Heure (au format HHMMSS) demandée du dernier fichier accédé au moyen de l'une des fonctions FTP suivantes : [FTPAttribut](../WDLang3/3040006.md), [FTPDate](../WDLang3/3040015.md), [FTPTaille](../WDLang3/3040007.md), [FTPListeFichier](../WDLang3/3040018.md).

- Chaîne vide ("") en cas d'erreur. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.


![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Heure de modification du dernier fichier consulté.

Contrairement à la syntaxe 2, aucune requête n'est envoyée au serveur. Cette syntaxe est donc à privilégier dès que possible.

**`<Type d'heure> : (Constante optionnelle)`**

Indique l'heure à manipuler :



|   |   |
| --- | --- |
| *ftpAccès* | Heure du dernier accès au fichier. |
| *ftpCréation* | Heure de création du fichier. |
| *ftpModification*<br>(Valeur par défaut) | Heure de dernière modification du fichier. |


![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Ce paramètre n'est pas disponible. Seule l'heure de modification du dernier fichier consulté est disponible.


<a name="SYNTAXE2"></a>

### Récupérer les différentes heures du fichier spécifié

`<Résultat> = FTPHeure(<Identifiant de la connexion> , <Nom du fichier> [, <Type d'heure>])`
---

**`<Résultat> : (Chaîne de caractères)`**



- Heure demandée du fichier spécifié (au format HHMMSS), 

- Chaîne vide ("") en cas d'erreur. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.


![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Heure de modification du fichier spécifié.

**`<Identifiant de la connexion> : (Entier)`**

Identifiant de la connexion, obtenu avec la fonction [FTPConnecte](../WDLang3/3040016.md).


**`<Nom du fichier> : (Chaîne de caractères)`**

Nom et chemin absolu (ou relatif) du fichier à manipuler. Les parties du chemin sont séparées par des "slash" (barre oblique normale, "/").

**`<Type d'heure> : (Constante optionnelle)`**

Indique l'heure à manipuler :



|   |   |
| --- | --- |
| *ftpAccès* | Heure du dernier accès au fichier. |
| *ftpCréation* | Heure de création du fichier. |
| *ftpModification*<br>(Valeur par défaut) | Heure de dernière modification du fichier. |


![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Ce paramètre n'est pas disponible. Seule l'heure de modification du fichier spécifié est disponible.

## Remarques


### Délai
<a name="delai_ELTPARAGRAPHE000148"></a>Par défaut, toutes les fonctions FTP échouent si aucune réponse du serveur FTP n'est reçue après 20 secondes. Ce délai peut être modifié lors de la connexion au serveur FTP avec la fonction [FTPConnecte](../WDLang3/3040016.md).
<a name="NOTE0_3"></a>


### Droits des utilisateurs
<a name="droits_des_utilisateurs_ELTPARAGRAPHE000158"></a>La demande d'information sur des fichiers ou sur des répertoires présents sur un serveur FTP n'est possible que pour un utilisateur ayant les droits en lecture sur le serveur FTP. En règle générale, les utilisateurs "anonymes" ont les droits en lecture.
<a name="NOTE0_4"></a>


### Chemin relatif et absolu
<a name="chemin_relatif_absolu_ELTPARAGRAPHE000165"></a>Les notions de chemin relatif et de chemin absolu sont très importantes dans une application FTP.

- Un chemin commençant par un slash est considéré comme un **chemin absolu** : il s'agit du chemin par rapport à la racine du serveur FTP (paramètre propre au serveur).
	Exemple : /pub/user/JULIE

- Si le chemin ne commence pas par un slash, il est considéré comme un **chemin relatif**, c'est-à-dire le chemin par rapport au répertoire en cours. Ce répertoire en cours peut-être connu ou modifié avec la fonction [FTPRepEnCours](../WDLang3/3040003.md).


Lors de la connexion à un site FTP, le répertoire initial, le "home directory" de l'utilisateur n'est pas forcément la racine du serveur FTP. Il est donc conseillé d'utiliser des chemins relatifs.

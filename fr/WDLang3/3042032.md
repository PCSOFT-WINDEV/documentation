


## grNuageAjouteDonnéeXY (Fonction)

***En anglais : grScatterAddDataXY***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute une donnée dans un graphe de type "Nuage de points". 

**Rappel** : Par défaut, un graphe n'a aucune donnée.



![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajoute une donnée dans le graphe "MonGraphe" de type "Nuage de points".
// Cette valeur (10, 20) est ajoutée à la fin de la première série.
grNuageAjouteDonnéeXY("MonGraphe", 1, 10, 20)
```
<a name="Exemple2"></a>

```wl
// Ajoute une donnée dans le champ GRF_MonGraphe de type "Nuage de points".
// Cette valeur (20, 10) est ajoutée à la fin de la première série.
grNuageAjouteDonnéeXY(GRF_MonGraphe, 1, 20, 10)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`grNuageAjouteDonnéeXY(<Nom du graphe> , <Numéro de la série> [, <Indice>] , <Valeur X> , <Valeur Y>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom correspond :

- soit au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- soit au nom du champ Graphe présent dans l'éditeur de fenêtres, de pages ou d'états.




**`<Numéro de la série> : (Entier)`**

Numéro de la série dans laquelle un point doit être initialisé. Si la série spécifiée n'existe pas, le grapheur crée autant de séries que nécessaire pour obtenir une matrice de données cohérente. Les séries ainsi créées sont toutes initialisées à 0.

**`<Indice> : (Entier optionnel)`**

Indice de la donnée à initialiser dans la série. Si ce paramètre n'est pas spécifié, la valeur est ajoutée à la fin de la série.

**`<Valeur X> : (Réel)`**

Coordonnée horizontale (abscisse) du point à ajouter.

**`<Valeur Y> : (Réel)`**

Coordonnée verticale (ordonnée) du point à ajouter.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Ajout de données dans un graphe de type Nuage de points
<a name="ajout_donnees_dans_graphe_type_nuage_points_ELTPARAGRAPHE000137"></a>Les données définies avec la fonction **grNuageAjouteDonnéeXY** seront prises en compte lors du prochain appel à la fonction [grDessine](../WDLang3/3042026.md).
<a name="NOTE0_2"></a>


### Ajout de données dans un graphe
<a name="ajout_donnees_dans_graphe_ELTPARAGRAPHE000152"></a>La fonction [grAjouteDonnée](../WDLang3/3042023.md) permet d'ajouter des données dans un graphe (secteur, histogramme, ...)

Selon le type de graphe, des fonctions spécifiques permettent également d'ajouter des données : 

- Pour les graphes boursiers, il est conseillé d'utiliser la fonction [grBourseAjouteDonnée](../WDLang3/3042009.md).

- Pour les graphes Heatmap, il est conseillé d'utiliser la fonction [grHMAjouteDonnée](../WDLang3/1000021628.md).

- Pour les graphes Secteur Multiniveau, il est conseillé d'utiliser la fonction [grSMNAjouteDonnée](../WDLang3/1000021285.md). 

- Pour les graphes de type Surface, il est conseillé d'utiliser la fonction [grSurfaceAjouteDonnée](../WDLang3/1000020425.md).


Rappel : Pour ajouter une valeur à une donnée d'un graphe, utilisez la fonction [grIncrémenteDonnée](../WDLang3/3042030.md).
<a name="NOTE0_3"></a>


### Équivalence
<a name="equivalence_ELTPARAGRAPHE000194"></a>Il est préférable d'utiliser la fonction **grNuageAjouteDonnéeXY** pour ajouter des données dans des graphes de type "Nuage de points". Cependant, il est également possible d'utiliser la fonction [grAjouteDonnée](../WDLang3/3042023.md).

La syntaxe suivante :

```wl
grNuageAjouteDonnéeXY(GRF_MonGraphe, i, 1, 10, 20)
```
est équivalente à la syntaxe :

```wl
grAjouteDonnée(GRF_MonGraphe, i*2-1, 1, 10)
grAjouteDonnée(GRF_MonGraphe, i*2, 1, 20)
```

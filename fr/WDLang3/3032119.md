
## LDAPRemplaceValeurAttribut (Fonction)

***En anglais : LDAPReplaceAttributeValue***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Remplace une valeur donnée d'un attribut par une nouvelle valeur donnée.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Modifie une valeur donnée d'un attribut 
// On veut remplacer sValeur2 par sNouvelleValeur
LDAPDébutModification("NomSession")
LDAPRemplaceValeurAttribut("NomSession", "NomAttribut", sValeur2, sNouvelleValeur)
LDAPValideModification("NomSession", "DN")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = LDAPRemplaceValeurAttribut(<Session LDAP> , <Nom de l'attribut> , <Ancienne valeur de l'attribut> , <Nouvelle valeur de l'attribut>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la modification a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de détails sur l'erreur. La modification effective ne sera réalisée que lors de l'appel de la fonction [LDAPValideModification](../WDLang3/3032117.md).




**`<Session LDAP> : (Chaîne de caractères)`**

Nom de la session LDAP manipulée.

**`<Nom de l'attribut> : (Chaîne de caractères)`**

Nom de l'attribut à manipuler.

**`<Ancienne valeur de l'attribut> : (Chaîne de caractères Ansi)`**

Valeur actuelle de l'attribut qui doit être remplacée.

**`<Nouvelle valeur de l'attribut> : (Chaîne de caractères Ansi)`**

Nouvelle valeur de l'attribut qui remplace toutes les &lt;Anciennes valeurs de l'attribut&gt;.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Différences entre LDAPRemplaceAttribut et LDAPRemplaceValeurAttribut
<a name="differences_entre_ldapremplaceattribut_docparampagetitleshort_ELTPARAGRAPHE000054"></a>La fonction [LDAPRemplaceAttribut](../WDLang3/3032113.md) permet de remplacer toutes les valeurs d'un attribut par une seule valeur. Un attribut multi-valué (plusieurs valeurs pour cet attribut) devient alors mono-valué (une seule valeur pour cet attribut).

La fonction **LDAPRemplaceValeurAttribut** permet de remplacer uniquement une certaine valeur d'un attribut par une nouvelle valeur. Un attribut multi-valué reste alors multi-valué.


<a name="NOTE0_2"></a>




### Modifier un objet existant sur un serveur LDAP
<a name="modifier_objet_existant_sur_serveur_ldap_ELTPARAGRAPHE000073"></a>Pour modifier un objet existant dans un serveur LDAP, il est nécessaire de définir les différentes modifications à effectuer.

Pour optimiser la vitesse de modification des objets, les modifications effectuées sont définies localement : une seule communication avec le serveur est établie lors de la validation de la modification. Les étapes sont les suivantes :

1. Initialisation de la modification (fonction [LDAPDébutModification](../WDLang3/3032112.md))

2. Opérations de modification :

	- Ajout des attributs et de leurs valeurs (fonction [LDAPAjouteAttribut](../WDLang3/3032110.md)).

	- Suppression des attributs (fonction [LDAPSupprimeAttribut](../WDLang3/3032115.md)) ou de leurs valeurs (fonction [LDAPSupprimeValeurAttribut](../WDLang3/3032116.md)).

	- Remplacement des valeurs d'un attribut (fonction [LDAPRemplaceAttribut](../WDLang3/3032113.md) ou **LDAPRemplaceValeurAttribut**)

	- Renommage de l'attribut (fonction [LDAPRenommeAttribut](../WDLang3/3032120.md))




3. Validation des modifications sur le serveur (fonction [LDAPValideModification](../WDLang3/3032117.md)). Les modifications sont effectuées sur le serveur en suivant l'ordre utilisé dans le code WLangage.




## &lt;Graphe&gt;.WaterfallCouleur (Fonction)

***En anglais : &lt;Chart&gt;.WaterfallColor***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique la couleur d'un type de catégorie pour un graphe de type Waterfall. 

**Remarque** : Si cette fonction n'est pas utilisée, le graphe est affiché avec des couleurs sélectionnées par défaut. 




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
GRF_Waterfall.WaterfallCouleur(grTotal, RVB(83,129,179))
GRF_Waterfall.WaterfallCouleur(grSousTotal, RVB(103,89,159))
GRF_Waterfall.WaterfallCouleur(grPositif, RVB(115,179,83))
GRF_Waterfall.WaterfallCouleur(grNégatif, RVB(179,84,92))
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Champ Graphe>.WaterfallCouleur(<Type de la catégorie> , <Couleur>)`
---

**`<Champ Graphe> : (Nom de champ)`**

Nom du champ Graphe à manipuler (présent dans l'éditeur de fenêtres, d'états ou de pages).


**`<Type de la catégorie> : (Constante de type Entier)`**

Type de la catégorie pour laquelle la couleur doit être définie : 



|   |   |
| --- | --- |
| *grNégatif* | Valeurs négatives du graphe. |
| *grPositif* | Valeurs positives du graphe. |
| *grSousTotal* | Valeurs de type sous-total du graphe. |
| *grTotal* | Valeurs de type total du graphe. |



**`<Couleur> : (Entier ou constante de type Entier)`**

Couleur du type de catégorie spécifié. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- une couleur prédéfinie (voir [les couleurs prédéfinies du WLangage](../WDLang5/3010002.md)).  

- la constante *CouleurDéfaut* : 

	- Si la constante *CouleurDéfaut* est utilisée pour les types de catégorie *grPositif* et *grNégatif*, les couleurs par défaut des séries sont utilisées. 

	- Si la constante *CouleurDéfaut* est utilisée pour le type de catégorie *grSousTotal* et si des couleurs spécifiques sont utilisées pour les types de catégorie *grPositif* et *grNégatif*, ces couleurs spécifiques sont utilisées pour le sous-total.









<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Les couleurs définies pour les catégories avec la fonction [&lt;Graphe&gt;.CouleurCatégorie](../WDLang3/1000023570.md) sont prioritaires sur les couleurs définies avec la fonction **&lt;Graphe&gt;.WaterfallCouleur**. 

- Cette fonction s'applique uniquement aux graphes de type Waterfall.

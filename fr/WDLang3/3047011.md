
## Application cliente : Exécuter des procédures sur un serveur de services J2EE
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000119"></a>
WINDEV met à votre disposition un ensemble de fonctions J2EE. Grâce à ces fonctions, vous pouvez gérer l'exécution de procédures sur un serveur de services Web XML J2EE depuis vos applications WINDEV.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principe d'utilisation
<a name="principe_utilisation_ELTTEXTE000143"></a>
L'ensemble des paramètres nécessaires à l'exécution d'une procédure est fourni dans la documentation du serveur de services Web XML J2EE et de ses procédures.

Pour exécuter une procédure sur un serveur de services Web XML J2EE:

1. Initialisez la structure des paramètres de la procédure à exécuter ([Structure J2EE](../WDLang3/3047006.md)).
	Remarque : Pour effectuer des vérifications supplémentaires sur le serveur de services Web XML J2EE, ajoutez un entête de procédure (fonction [J2EEAjouteEntête](../WDLang3/3047001.md)).

2. Exécutez la procédure (fonction [J2EEExécute](../WDLang3/3047007.md) ou [J2EEExécuteXML](../WDLang3/3047008.md)). Les opérations suivantes sont automatiquement réalisées :

	- connexion entre le poste en cours et le serveur de services Web XML J2EE.

	- transmission des paramètres de la procédure au serveur de services Web XML J2EE.

	- vérification de l'entête si nécessaire.

	- exécution de la procédure.

	- renvoi du résultat ou de l'erreur de la procédure au poste en cours.




3. Testez le résultat de la procédure.




Le résultat de des fonctions [J2EEExécute](../WDLang3/3047007.md) ou [J2EEExécuteXML](../WDLang3/3047008.md) permet de savoir si la connexion a bien été établie.

Si la connexion n'a pas été établie, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) pour connaître les causes de l'erreur. 
Si la connexion a été établie, testez le résultat de la fonction [J2EEDonneRésultat](../WDLang3/3047003.md) :

- si le résultat est différent d'une chaîne vide (""), la procédure a été exécutée avec succès.

- si le résultat correspond à une chaîne vide (""), la procédure n'a pas été exécutée et/ou a retourné une erreur. Pour connaître l'erreur de la procédure, utilisez la fonction [J2EEErreur](../WDLang3/3047010.md).

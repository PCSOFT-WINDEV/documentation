
## DNSSupprimeAdresse (Fonction)

***En anglais : DNSDeleteAddress***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime une relation Nom/Adresse IP dans un serveur DNS.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ResultDNS est un booléen

// Supprime une adresse
ResultDNS = DNSSupprimeAdresse("testAp.towerqual.local.", "192.168.1.1", "Andromede_2")
SI ResultDNS = Faux ALORS
	Erreur(ErreurInfo(errComplet))
FIN

// Supprime toutes les adresses
ResultDNS = DNSSupprimeAdresse("testAp.towerqual.local.", "", "Andromede_2")
SI ResultDNS = Faux  ALORS
	Erreur(ErreurInfo(errComplet))
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = DNSSupprimeAdresse(<Nom> [, <Adresse IP> [, <Serveur DNS> [, <Utilisateur> [, <Mot de passe>]]]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la suppression a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de renseignements sur le problème rencontré.




**`<Nom> : (Chaîne de caractères Unicode)`**

Nom à supprimer du DNS. Ce paramètre est de la forme : &lt;Nom du poste&gt; + "." + &lt;Domaine&gt;. Exemple : "testdoc.pcsoft.com"

**`<Adresse IP> : (Chaîne de caractères optionnelle)`**

Adresse IP associée au nom à supprimer. Cette adresse IP peut être une adresse IP version 4 (au format xxx.xxx.xxx.xxx) ou version 6 (au format aaaa :bbbb :cccc :dddd :eeee :ffff :gggg :hhhh :iiii :jjjj).

Si ce paramètre correspond à une chaîne vide (""), toutes les associations du nom avec une adresse IP seront supprimées. 

**`<Serveur DNS> : (Chaîne de caractères optionnelle)`**

Adresse du serveur DNS à mettre à jour. L'association "Adresse IP / Nom" sera supprimée de ce serveur. Par défaut, le serveur DNS actuel est utilisé.

**`<Utilisateur> : (Chaîne de caractères Unicode optionnelle)`**

Nom de l'utilisateur utilisé pour signer la requête.

**`<Mot de passe> : (Chaîne de caractères Unicode optionnelle)`**

Mot de passe associé à l'utilisateur pour signer la requête.	 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Cette fonction gère les serveurs DNS Linux et Windows ne nécessitant pas de signature des requêtes.

- Cette fonction gère les serveurs DNS Windows nécessitant une signature des requêtes.

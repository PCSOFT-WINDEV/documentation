


## zipTaille (Fonction)

***En anglais : zipSize***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie la taille de l'archive (au format ZIP, RAR, CAB, WDZ, 7z, TAR ou TGZ (TAR.GZ)) :

- soit la taille de l'ensemble des fichiers compressés présents dans l'archive.

- soit la taille de l'ensemble des fichiers avant compression présents dans l'archive.




Lors de l'extraction des fichiers, il est conseillé de calculer la taille de l'ensemble des fichiers avant compression (présents dans l'archive) pour vérifier si l'espace disque disponible est suffisant.

Rappel : 

- ![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Seules les archives au format ZIP, TAR et TGZ (TAR.GZ) sont disponibles.

- ![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Seules les archives au format ZIP sont disponibles.

## Syntaxe

`<Résultat> = zipTaille(<Archive> [, <Type de taille>])`
---

**`<Résultat> : (Entier)`**

Taille de l'archive.

**`<Archive> : (Chaîne de caractères ou variable de type zipArchive)`**

Nom de l'archive à manipuler.
Ce nom peut correspondre :

- soit au nom d'une archive défini avec la fonction [zipOuvre](../WDLang3/3082044.md) ou [zipCrée](../WDLang3/3082003.md).

- soit au nom d'une variable de type [zipArchive](../WDLang3/1000018679.md). 
	![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Les variables de type [zipArchive](../WDLang3/1000018679.md) ne sont pas disponibles en JAVA.




**`<Type de taille> : (Booléen optionnel)`**

Type de calcul de la taille de l'archive :

- <u><u><u><u>Vrai</u></u></u></u> : Taille de l'ensemble des fichiers compressés présents dans l'archive.

- <u><u><u><u>Faux</u></u></u></u> (valeur par défaut) : Taille de l'ensemble des fichiers avant compression présents dans l'archive.


## LDAPMode (Fonction)

***En anglais : LDAPMode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie le mode de fonctionnement d'une session LDAP.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Procédure de connexion
...

// Désactivation du système de cache
LDAPMode("MaSessionLDAP", LDAP_CacheInactif)

// Récupération du nombre de racines
NbRacine est un entier = LDAPNbValeur("MaSessionLDAP", "namingContexts")

// Affichage des différentes racines du serveur
POUR Indice = 1 A NbRacine
	Info(LDAPValeur("MaSessionLDAP", "namingContexts", Indice)
FIN

// Réactivation du système de cache
LDAPMode("MaSessionLDAP", LDAP_CacheActif)

// Procédure de déconnexion
...
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = LDAPMode(<Session LDAP> , <Option>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la modification est effective, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de détails sur l'erreur.




**`<Session LDAP> : (Chaîne de caractères)`**

Nom de la session LDAP manipulée.

**`<Option> : (Constante de type entier)`**

Indique le mode de fonctionnement de la session LDAP :


|   |   |
| --- | --- |
| *LDAP_CacheActif* | Les différents appels aux fonctions LDAP utilisent le cache interne pour récupérer les différentes valeurs des attributs (mode par défaut). |
| *LDAP_CacheInactif* | Chaque appel aux fonctions [LDAPNbValeur](../WDLang3/3032106.md) et [LDAPValeur](../WDLang3/3032107.md) implique une communication avec le serveur. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
**Attention** : La constante *LDAP_CacheInactif* est à utiliser dans des cas très particuliers.

Pour lire des attributs opérationnels sur un serveur OpenLDAP, il faut utiliser cette option. En effet, par défaut, un serveur OpenLDAP ne renvoie pas les attributs opérationnels d'un objet, et la fonction [LDAPListeAttribut](../WDLang3/3032104.md) ne liste pas non plus ces attributs.

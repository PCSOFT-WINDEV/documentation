


## ContactCherche (Fonction)

***En anglais : ContactSearch***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Recherche un contact dans le carnet d'adresses d'une messagerie Lotus Notes ou Outlook.
**Attention** : Cette fonction **ne permet pas** de rechercher un contact dans le carnet d'adresses d'une messagerie Outlook Express.

La fonction **ContactCherche** ne positionne pas sur le contact recherché. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouverture d'une session Lotus Notes
IDSession est un entier
IDSession = EmailOuvreSessionNotes("MotDePasse", "ServeurMars", ....
		"mail/julie.nsf", "C:\Lotus Notes\julie.id")
...
// Recherche des contacts de la société "Tames"
SI ContactCherche(IDSession, "Tames", mContactSociété) = Vrai ALORS
	// Ajout des contacts trouvés dans un champ Liste
	TANTQUE mContact.EnDehors = Faux
		ListeAjoute(LISTE_ListeContact, mContact.Nom + TAB + mContact.Prénom)
		ContactSuivant(IDSession)
	FIN
SINON
	Info("Aucun contact n'a été trouvé")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Rechercher un contact dans la messagerie Lotus Notes ou Outlook

`<Résultat> = ContactCherche(<Identifiant de la session> , <Valeur recherchée 1> , <Type de la valeur 1> [, <Valeur recherchée 2> [, <Type de la valeur 2>]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si un ou plusieurs contacts correspondent à la recherche, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*. 




**`<Identifiant de la session> : (Entier)`**

Identifiant de la session à utiliser. Cet identifiant est renvoyé par la fonction [EmailOuvreSessionNotes](../WDLang3/3032121.md), [EmailOuvreSessionOutlook](../WDLang3/3032128.md), [NotesOuvreBase](../WDLang3/1000005101.md) ou [OutlookOuvreSession](../WDLang3/3032123.md).

**`<Valeur recherchée 1> : (Type de la valeur recherchée)`**

Première valeur à rechercher.

**`<Type de la valeur 1> : (Constante de type Entier)`**

Type de la première valeur à rechercher :



|   |   |
| --- | --- |
| *mContactEmail* | Recherche dans les emails des contacts |
| *mContactNom* | Recherche dans les noms des contacts |
| *mContactPrénom* | Recherche dans les prénoms des contacts |
| *mContactSociété* | Recherche dans les sociétés des contacts |



**`<Valeur recherchée 2> : (Type de la valeur recherchée, optionnel)`**

Seconde valeur à rechercher. Si ce paramètre et le paramètre &lt;Type de la valeur 2&gt; ne sont pas spécifiés, le contact est recherché uniquement en fonction de la &lt;Valeur recherchée 1&gt;.

**`<Type de la valeur 2> : (Constante optionnelle de type Entier)`**

Type de la seconde valeur à rechercher :



|   |   |
| --- | --- |
| *mContactEmail* | Recherche dans les emails des contacts |
| *mContactNom* | Recherche dans les noms des contacts |
| *mContactPrénom* | Recherche dans les prénoms des contacts |
| *mContactSociété* | Recherche dans les sociétés des contacts |

Si ce paramètre et le paramètre &lt;Valeur recherchée 2&gt; ne sont pas spécifiés, le contact est recherché uniquement en fonction de la &lt;Valeur recherchée 1&gt;.


## Procédure WLangage appelée par la fonction AndroidActivityResultProcédure



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Procédure WLangage (également nommée "callback") appelée par la fonction [AndroidActivityResultProcédure](../WDLang3/1410088881.md) lors de la récupération du résultat renvoyé par une Activity tierce.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
AndroidActivityResultProcédure((resultCode, requestCode, Uri, extras)=>{
	SI resultCode = -1 _ET_ requestCode = 1024 ALORS
		POUR TOUT ELEMENT valeur, clé DE extras 
			...
		FIN
	FIN
})
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`AndroidActivityResultProcédure_Callback(<Résultat> , <Request code> , <Uri> , <Extras>)`
---

**`<Résultat> : (Entier)`**

Résultat de l'opération effectuée par l'Activity qui vient de se terminer.

**`<Request code> : (Entier)`**

"Request code" passé en paramètre à la méthode **startActivityForResult** appelée pour démarrer l'Activity.

**`<Uri> : (Variable de type URI)`**

Nom de la variable de type [URI](../WDLang3/1000024027.md) correspondant à l'URI des données traitées par l'Activity.

**`<Extras> : (Tableau associatif de Variant)`**

Couples clé/valeur reçus comme résultat de l'Activity. Seuls les types suivant de valeur sont gérés : Entier, Chaine de caractères, Booléen, Buffer, Réel, Tableau d'entier, booléen, chaîne ou réel. 
Si un couple clé/valeur d'un type non supporté est reçu, il sera ignoré.

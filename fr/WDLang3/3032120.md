
## LDAPRenommeAttribut (Fonction)

***En anglais : LDAPRenameAttribute***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renomme un attribut. Lors de ce renommage, la valeur des attributs est conservée.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Renomme l'attribut "NomAttribut" en "Nom2"
LDAPDébutModification("NomSession")
LDAPRenommeAttribut("NomSession", "NomAttribut", "Nom2")
LDAPValideModification("NomSession", "DN")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = LDAPRenommeAttribut(<Session LDAP> , <Ancien nom de l'attribut> , <Nouveau nom de l'attribut>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la modification a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de détails sur l'erreur. 


La modification effective ne sera réalisée que lors de l'appel de la fonction [LDAPValideModification](../WDLang3/3032117.md).

**`<Session LDAP> : (Chaîne de caractères)`**

Nom de la session LDAP manipulée.

**`<Ancien nom de l'attribut> : (Chaîne de caractères)`**

Nom actuel de l'attribut à manipuler.

**`<Nouveau nom de l'attribut> : (Chaîne de caractères)`**

Nouveau nom de l'attribut à utiliser.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Renommage
<a name="renommage_ELTPARAGRAPHE000053"></a>Le nouveau nom de l'attribut doit être un attribut valide pour l'objet modifié.

Si le renommage concerne un attribut multi-valué, le nouveau nom doit correspondre à un attribut valide et multi-valué. Dans le cas contraire, une erreur sera rencontrée avec la fonction [LDAPValideModification](../WDLang3/3032117.md).


<a name="NOTE0_2"></a>




### Modifier un objet existant sur un serveur LDAP
<a name="modifier_objet_existant_sur_serveur_ldap_ELTPARAGRAPHE000067"></a>Pour modifier un objet existant dans un serveur LDAP, il est nécessaire de définir les différentes modifications à effectuer.

Pour optimiser la vitesse de modification des objets, les modifications effectuées sont définies localement : une seule communication avec le serveur est établie lors de la validation de la modification. Les étapes sont les suivantes :

1. Initialisation de la modification (fonction [LDAPDébutModification](../WDLang3/3032112.md)).

2. Opérations de modification :

	- Ajout des attributs et de leurs valeurs (fonction [LDAPAjouteAttribut](../WDLang3/3032110.md)).

	- Suppression des attributs (fonction [LDAPSupprimeAttribut](../WDLang3/3032115.md)) ou de leurs valeurs (fonction [LDAPSupprimeValeurAttribut](../WDLang3/3032116.md)).

	- Remplacement des valeurs d'un attribut (fonction [LDAPRemplaceAttribut](../WDLang3/3032113.md) ou [LDAPRemplaceValeurAttribut](../WDLang3/3032119.md)).

	- Renommage de l'attribut (fonction **LDAPRenommeAttribut**)




3. Validation des modifications sur le serveur (fonction [LDAPValideModification](../WDLang3/3032117.md)). Les modifications sont effectuées sur le serveur en suivant l'ordre utilisé dans le code WLangage.

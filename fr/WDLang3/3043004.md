


## Fonctions HTTP
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>






Les fonctions suivantes permettent de manipuler des pages Internet, d'envoyer des formulaires HTTP, ... :



|   |   |
| --- | --- |
| [HTTPAjouteFichier](../WDLang3/1000017191.md) | Ajoute un fichier dans le formulaire HTTP. |
| [HTTPAjouteParamètre](../WDLang3/1000017189.md) | Ajoute un paramètre dans un formulaire HTTP. |
| [HTTPAnnuleFormulaire](../WDLang3/1000017123.md) | Annule une déclaration de formulaire HTTP et libère toutes ses ressources. |
| [HTTPCertificat](../WDLang3/1000018898.md) | Modifie le certificat client utilisé par défaut pour s'identifier sur un serveur. |
| [HTTPCookieEcrit](../WDLang3/1000019234.md) | Ajoute ou modifie un cookie dans une requête HTTP. |
| [HTTPCookieGère](../WDLang3/1000019218.md) | Active ou désactive la gestion des cookies dans une requête HTTP exécutée : <br><br>- soit par la fonction [HTTPRequête](../WDLang3/3043007.md). <br><br>- soit par la fonction [HTTPEnvoie](../WDLang3/1000021183.md).<br><br><br> |
| [HTTPCookieLit](../WDLang3/1000019233.md) | Retourne la valeur d'un cookie reçu suite à une requête HTTP. |
| [HTTPCookieRAZ](../WDLang3/1000019232.md) | Supprime l'ensemble des cookies (globalement ou pour un domaine) stockés : <br><br>- par les appels à la fonction [HTTPRequête](../WDLang3/3043007.md).<br><br>- par les appels à la fonction [HTTPEnvoie](../WDLang3/1000021183.md).<br><br><br> |
| [HTTPCookieRécupère](../WDLang3/1000019228.md) | Récupère les cookies lus par un appel : <br><br>- à la fonction [HTTPRequête](../WDLang3/3043007.md) pour un domaine HTTP.<br><br>- à la fonction [HTTPEnvoie](../WDLang3/1000021183.md) pour un domaine HTTP.<br><br><br> |
| [HTTPCookieRemplace](../WDLang3/1000019231.md) | Remplace l'ensemble des cookies stockés pour un domaine : <br><br>- par la fonction [HTTPRequête](../WDLang3/3043007.md).<br><br>- par la fonction [HTTPEnvoie](../WDLang3/1000021183.md).<br><br><br> |
| [HTTPCréeFormulaire](../WDLang3/1000017119.md) | Crée un formulaire HTTP. |
| [HTTPDestination](../WDLang3/1000017094.md) | Indique le fichier de sauvegarde du résultat de la prochaine requête HTTP, exécutée dans le même thread. |
| [HTTPDonneRésultat](../WDLang3/3043001.md) | Récupère le résultat ou l'entête de la dernière requête HTTP lancée. |
| [HTTPEnvoie](../WDLang3/1000021183.md) | Envoie une requête HTTP et attend la réponse du serveur HTTP. |
| [HTTPEnvoieFormulaire](../WDLang3/1000017190.md) | Envoie un formulaire HTTP. |
| [HTTPJauge](../WDLang3/3043009.md) | Affiche une jauge de progression pendant : <br><br>- la réception de la réponse à une requête HTTP.<br><br>- l'envoi d'un formulaire HTTP (fonction [HTTPEnvoieFormulaire](../WDLang3/1000017190.md)) ou d'une requête HTTP.<br><br><br> |
| [HTTPListeCertificat](../WDLang3/1000018890.md) | Liste les certificats client présents sur la machine pour s'authentifier sur un serveur. |
| [HTTPParamètre](../WDLang3/1000018985.md) | Permet de paramétrer les fonctions utilisant le protocole HTTP. |
| [HTTPRAZFormulaire](../WDLang3/1000017188.md) | Vide un formulaire HTTP en cours d'édition. |
| [HTTPRequête](../WDLang3/3043007.md) | Lance une requête HTTP sur un serveur. |
| [HTTPSimuleRéseau](../WDLang3/1000021595.md) | Simule un réseau lent en limitant la vitesse de téléchargement et d'envoi. |
| [HTTPTimeOut](../WDLang3/3043008.md) | Règle le timeout pour les fonctions WLangage utilisant le protocole HTTP (fonctions [HTTPRequête](../WDLang3/3043007.md), [SOAPExécute](../WDLang3/3069013.md), [DotNetExécute](../WDLang3/3031016.md), [J2EEExécute](../WDLang3/3047007.md), ...). |
| [InternetConnecté](../WDLang3/3043005.md) | Détermine si une connexion Internet est active sur le poste en cours. Sous mobile, il est également possible d'être notifié du changement d'état de la connexion Internet. |
| [OpenAPIVersHTTP](../WDLang3/1410088947.md) | Construit la requête HTTP d'appel à une fonction d'une API. Elle pourra ensuite être envoyée grâce à la fonction [HTTPEnvoie](../WDLang3/1000021183.md). |
| [Proxy](../WDLang3/3043002.md) | Spécifie si les fonctions de communication utilisant le protocole HTTP ou HTTPS doivent ou non passer par un proxy pour exécuter leurs requêtes. Un proxy permet de partager l'accès à Internet entre différents postes d'un réseau. |
| [RESTEnvoie](../WDLang3/1000021476.md) | Envoie une requête REST et attend la réponse du serveur. |



Les types avancés suivants peuvent être utilisés avec les fonctions HTTP :



|   |   |
| --- | --- |
| [Cookie](../WDLang3/1000019222.md) | Le type **Cookie** permet de manipuler les cookies transmis dans des requêtes HTTP. |
| [httpAvancement](../WDLang3/1000021222.md) | Le type **httpAvancement** permet de gérer l'avancement de l'envoi ou de la réception d'une requête HTTP. |
| [httpRéponse](../WDLang3/1000021165.md) | Le type **httpRéponse** permet de récupérer toutes les caractéristiques avancées de la réponse d'une requête HTTP/HTTPS. |
| [httpRequête](../WDLang3/1000021158.md) | Le type **httpRequête** permet de définir toutes les caractéristiques avancées d'une requête HTTP/HTTPS. |
| [restRéponse](../WDLang3/1000021477.md) | Le type **restRéponse** permet de récupérer toutes les caractéristiques avancées de la réponse d'une requête REST. |
| [restRequête](../WDLang3/1000021481.md) | Le type **restRequête** permet de définir toutes les caractéristiques avancées d'une requête REST. |

- ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=WDTraceSocket.gif) ***Composants (WINDEV)*** : **WD Trace Socket** <br>Le principe de cet exemple est de tracer tout ce qui passe sur une communication via socket, sur une adresse donnée, sur un port donné. Cela permet de comprendre ce qu'il se passe dans un dialogue via socket entre deux applications.<br>Ainsi, dans des applications qui dialoguent via socket, même si les développeurs n'ont pas prévu d'option de trace, il sera possible de voir l'ensemble des échanges effectués sur la socket.<br>Cet exemple peut également permettre de mieux comprendre le fonctionnement des protocoles réseau basés sur les sockets : POP3, SMTP, FTP, HTTP, etc.<br>Note pour les utilisateurs avertis en communication réseau : <br>Cet exemple ne permet de tracer que les communications TCP, mais ne permet pas de tracer les paquets UDP.

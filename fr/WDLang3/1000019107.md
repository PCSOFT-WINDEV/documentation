
## SaaSAdminAjouteService (Fonction)

***En anglais : SaaSAdminAddService***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute un service SaaS dans la description d'un site.

**Remarques** : 

- L'utilisation de cette fonction nécessite d'avoir ouvert une session d'administration SaaS au préalable en utilisant la fonction [SaaSAdminConnecte](../WDLang3/1000019098.md).

- L'utilisation de cette fonction est réservée à l'Administrateur SaaS. Un gestionnaire de compte client ne peut pas utiliser cette fonction. 



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ajout du service "Poster un message" dans le site "Forum"
S est un saasSite dynamique 
ServiceMessage est un saasService

S = SaaSAdminChercheSite("Forum")
ServiceMessage.Nom = "Poster un message"

SaaSAdminAjouteService(ServiceMessage, S)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Ajouter un service SaaS (utilisation des variables de type saasxxx)

`<Résultat> = SaaSAdminAjouteService(<Service SaaS> , <Site SaaS>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'ajout a été effectué, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Service SaaS> : (Variable de type saasService)`**

Nom de la variable de type [saasService](../WDLang3/1000019096.md) contenant la description du service à ajouter.

**`<Site SaaS> : (Variable de type saasSite)`**

Nom de la variable de type [saasSite](../WDLang3/1000019068.md) correspondant au site SaaS recevant le nouveau service.


<a name="SYNTAXE2"></a>

### Ajouter un nouveau service SaaS identifié par son nom

`<Résultat> = SaaSAdminAjouteService(<Service SaaS> , <Site SaaS>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'ajout a été effectué, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Service SaaS> : (Chaîne de caractères)`**

Nom du nouveau service SaaS.

**`<Site SaaS> : (Chaîne de caractères)`**

Nom du site SaaS recevant le nouveau service.

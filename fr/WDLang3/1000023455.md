
## TâcheEnArrièrePlanSupprime (Fonction)

***En anglais : BackgroundTaskDelete***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime une tâche en arrière-plan (précédemment ajoutée avec la fonction [TâcheEnArrièrePlanAjoute](../WDLang3/1000023454.md)).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Suppression de la tâche en arrière-plan de traitement des emails 
TâcheEnArrièrePlanSupprime(ProcTraitementEmailEnArrièrePlan)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = TâcheEnArrièrePlanSupprime(<Procédure WLangage>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la tâche a été supprimée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Procédure WLangage> : (Nom de procédure)`**

Nom de la procédure WLangage correspondant à la tâche en arrière-plan à supprimer. Le nom de cette procédure a été spécifiée lors de l'ajout de la tâche avec la fonction [TâcheEnArrièrePlanAjoute](../WDLang3/1000023454.md).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Limite
<a name="limite_ELTPARAGRAPHE000051"></a>Attention : Il n'est pas possible de supprimer par programmation une tâche en arrière-plan ajoutée automatiquement via les automatismes de procédure. 

Pour supprimer ces tâches, il faut désactiver l'option correspondante dans la fenêtre de paramétrage des automatismes de procédure. La suppression sera effective au lancement suivant de l'application.




## grCrée (Fonction)

***En anglais : grCreate***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Crée un graphe d'un type spécifié.

**Remarques** :

- Il est conseillé de créer directement un champ Graphe plutôt que d'utiliser la fonction **grCrée**. En effet, le champ Graphe offre beaucoup plus de possibilités.

- Cette fonction permet uniquement de créer un graphe par programmation. Cette fonction ne peut pas être utilisée dans l'éditeur de fenêtres, de pages ou d'états.




![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Création d'un histogramme 
// (avec sauvegarde des paramètres modifiés par l'utilisateur)
grCrée("MonGraphe", grHisto)
```

<a name="SYNTAXE1"></a>

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Créer et dessiner un graphe par programmation
<a name="creer_dessiner_graphe_par_programmation_ELTPARAGRAPHE000399"></a>Pour créer et dessiner un graphe :

- créez le graphe (fonction **grCrée**),

- ajoutez les données (fonctions [grAjouteDonnée](../WDLang3/3042023.md), [grBourseAjouteDonnée](../WDLang3/3042009.md) et [grNuageAjouteDonnéeXY](../WDLang3/3042032.md)),

- définissez la destination du graphe à l'aide des fonctions suivantes : 

	- [grDestinationBMP](../WDLang3/3042018.md), 

	- [grDestinationChamp](../WDLang3/3042020.md), 

	- [grDestinationEMF](../WDLang3/3042017.md), 

	- [grDestinationFen](../WDLang3/3042031.md),

	- [grDestinationImage](../WDLang3/1000020952.md),

	- [grDestinationWMF](../WDLang3/3042019.md). 




- dessinez le graphe (fonction [grDessine](../WDLang3/3042026.md)).



<a name="NOTE0_2"></a>


### Tracer plusieurs graphes simultanément
<a name="tracer_plusieurs_graphes_simultanement_ELTPARAGRAPHE000447"></a>A un instant donné, il ne peut pas exister plusieurs graphes avec le même nom.

Pour tracer plusieurs graphes en même temps (dans une ou plusieurs fenêtres par exemple), il est nécessaire d'utiliser des noms de graphes différents.
<a name="NOTE0_3"></a>


### Graphes et threads
<a name="graphes_threads_ELTPARAGRAPHE000456"></a>Un seul graphe peut être manipulé dans un thread.

Conséquences :

- Deux threads ne peuvent pas manipuler le même graphe.

- Un graphe créé dans un thread est automatiquement détruit à la fin de ce thread. Le graphe existe dans le thread qui contient l'appel à la fonction **grCrée**.



<a name="NOTE0_4"></a>


### Valeurs par défaut d'un graphe créé par programmation
<a name="valeurs_par_defaut_graphe_cree_par_programmation_ELTPARAGRAPHE000471"></a>Les valeurs par défaut d'un graphe sont les suivantes :

- Menu contextuel activé (paramétrable avec les fonctions [grMenu](../WDLang3/3042035.md) et [grOptionMenu](../WDLang3/3042042.md)).
	

- Graduation automatique des axes (paramétrable avec la fonction [grGraduation](../WDLang3/3042027.md)).

- Les couleurs sont sélectionnées selon un ordre prédéfini parmi un éventail de 13 couleurs (paramétrable avec la fonction [grCouleurSerie](../WDLang3/3042013.md)).

- Les couleurs sont dégradées (paramétrable avec la fonction [grDégradé](../WDLang3/3042029.md)).

- Taille fichier (paramétrable avec la fonction [grFenTaille](../WDLang3/3042016.md)).

- Taille fenêtre (paramétrable avec la fonction [grFenTaille](../WDLang3/3042016.md)).
	

- Pas d'étiquette pour les séries (paramétrable avec la fonction [grEtiquetteSérie](../WDLang3/3042024.md)).

- Pas de libellé pour les éléments (paramétrable avec les fonctions [grEtiquetteCatégorie](../WDLang3/3042015.md) et [grLibellé](../WDLang3/3042046.md)).

- Pas de légende (paramétrable avec la fonction [grLégende](../WDLang3/3042047.md)).

- Pas de titre pour le graphe (paramétrable avec la fonction [grTitre](../WDLang3/3042056.md)).

- Pas de titre pour les axes (paramétrable avec la fonction [grTitreAxe](../WDLang3/3042057.md)).

- Pas de quadrillage de fond (paramétrable avec la fonction [grQuadrillage](../WDLang3/3042052.md)).

- Pas de données (ajout d'éléments avec la fonction [grAjouteDonnée](../WDLang3/3042023.md)).

- Pas de destination (paramétrable avec les fonctions [grDestinationBMP](../WDLang3/3042018.md), [grDestinationChamp](../WDLang3/3042020.md), [grDestinationEMF](../WDLang3/3042017.md), [grDestinationFen](../WDLang3/3042031.md) et [grDestinationWMF](../WDLang3/3042019.md)).

- Pas d'affichage du graphe (pour dessiner le graphe, utilisez la fonction [grDessine](../WDLang3/3042026.md)).

- Pas de relief (paramétrable avec la fonction [grRelief](../WDLang3/3042051.md)).

- Pour les graphes en secteurs, l'angle de départ est égal à 0 (paramétrable avec la fonction [grSecteurAngleDépart](../WDLang3/3042022.md)).

- Pour les histogrammes et les graphes boursiers de type Minimum/Maximum, l'espace entre deux catégories est égal à 10 pixels (paramétrable avec la fonction [grHistoEspacement](../WDLang3/3042014.md)).

- Pour les nuages de points, les points sont reliés et de taille égale à 1 (paramétrable avec les fonctions [grNuageReliePoint](../WDLang3/3042040.md) et [grNuageTaillePoint](../WDLang3/3042041.md)).

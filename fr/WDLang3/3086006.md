## Installer un serveur SOAP avec Netscape iPlanet 4.1
<a name="installer_serveur_soap_avec_netscape_iplanet_41_ELTTEXTE000154"></a>


### Pour installer un serveur SOAP avec Netscape iPlanet 4.1 (le serveur Netscape iPlanet 4.1 doit être installé sur le poste) :
<a name="pour_installer_serveur_soap_avec_netscape_iplanet_41_serveur_netscape_iplanet_41_doit_etre_installe_sur_poste_ELTPARAGRAPHE000027"></a>**1.** Pour trouver les fichiers de configuration :

- Sélectionnez le répertoire d'installation de iPlanet (par défaut C:\\Netscape\\Server4).

- Sélectionnez le sous-répertoire correspondant au serveur à configurer. Le nom du serveur correspond à "https-&lt;nom de l'utilisateur&gt;".

- Sélectionnez le sous-répertoire "config".




**2**. Ouvrez le fichier "mime.types" avec WordPad.

ATTENTION : si vous ouvrez ce fichier avec NotePad.EXE, le format du fichier risque d'être corrompu à la sauvegarde.

**3.** A la fin du fichier, ajoutez la ligne suivante et enregistrez (à la sauvegarde WordPad signale que la mise en forme est perdue ; répondre "Oui") :

```txt
type=application/soap exts=soap
```


**Remarque** : Si l'extension "soap" est déjà associée à un module sur votre serveur, utilisez une autre extension. Dans ce cas, cette nouvelle extension devra être spécifiée dans les fonctions indiquant l'URL du serveur SOAP (paramètre &lt;URL du serveur SOAP&gt; dans la fonction [SoapExécute](../WDLang3/3069013.md) par exemple).

**4.** Ouvrez le fichier obj.conf.

**5.** Recherchez la ligne "&lt;Object name=default&gt;". Insérez avant cette ligne :

- Les lignes suivantes pour les tests depuis votre poste :



```txt
Init fn=load-modules shlib="<Répertoire d'installation de WinDev 15>/Donnees/wdxxxsns.dll" \
funcs="WinDevHandler,ModuleExit,ModuleInit"
Init fn="ModuleInit"
```




- Les lignes suivantes pour installer le serveur soap final :



```txt
Init fn=load-modules shlib="<emplacement de la DLL>/wdxxxsns.dll" \
funcs="WinDevHandler,ModuleExit,ModuleInit"
Init fn="ModuleInit"
```


Attention : Les chemins sont de la forme "d:/mon repertoire/mon fichier.dll". Il faut des quotes et des '/' (et non des '\\').

**6.** Recherchez la première ligne commençant par "Service". Insérez avant cette ligne :

```txt
Service method=(POST) type=application/soap fn=WinDevHandler
```


**7.** Sauvez le fichier.

**8.** Si vous installez un serveur SOAP classique, redémarrez votre serveur. L'installation du serveur est terminée.

Remarque : La DLL wdxxxsns.dll est compatible NSAPI.

## Installer un Webservice
<a name="installer_webservice_ELTTEXTE000178"></a>


### Si vous installez un Webservice
<a name="vous_installez_webservice_ELTPARAGRAPHE000085"></a>

**9.** Ouvrez le fichier "webpub.conf" et recherchez le mot-clé "NS-doc-root".

**10.** Copiez dans le répertoire indiqué après le mot-clé "NS-doc-root" :

- Le fichier XML contenant la description du Webservice.

- Les pages HTML de présentation du Webservice.




**Pour accéder à ce fichier XML**, utilisez la ligne de commande suivante :

"http://&lt;Nom Ou Adresse IP Serveur Soap&gt;/&lt;Nom Fichier XML&gt;.XML"

**Pour tester ce Webservice**, utilisez la ligne de commande suivante :

"http://&lt;Nom Ou Adresse IP Serveur Soap&gt;/index.htm"

**Remarque** : Il est également possible de copier ce fichier .XML dans un autre répertoire. Pour plus de détails, consultez l'aide du serveur HTTP utilisé.

**11.** Redémarrez votre serveur. L'installation du Webservice sur le serveur SOAP est terminée.

Remarque : La DLL wdxxxsns.dll est compatible NSAPI.

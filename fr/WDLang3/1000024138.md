


## &lt;Graphe&gt;.SourceEtiquetteCatégorie (Fonction)

***En anglais : &lt;Chart&gt;.SourceCategoryLabel***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Définit la source des étiquettes de catégorie.


![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) **Attention** : L'utilisation de cette fonction sous Linux nécessite une configuration spécifique. Pour plus de détails, consultez [Les graphes](../WDChamp/3042060.md). 










<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI GRF_MonGraphe.SourceEtiquetteCatégorie(grColonneTable, "MaTable.MaColonne") = Vrai ALORS
	Info("La source des étiquettes de catégorie est désormais la colonne MaColonne")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Champ Graphe>.SourceEtiquetteCatégorie(<Source> [, <Détail 1> [... [, <Détail N>]]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la source a été définie, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Champ Graphe> : (Nom de champ)`**

Nom du champ Graphe à manipuler.

**`<Source> : (Constante de type Entier)`**

Source  de données du graphe :


|   |   |
| --- | --- |
| *grColonneTable* | Etiquettes de catégorie remplies depuis une colonne d'un champ Table. <br><br>Dans ce cas, &lt;Détail 1&gt; correspond au nom d'une colonne d'un champ Table. Ce nom doit correspondre au nom complet de la colonne, indiqué entre guillemets (par exemple "&lt;NomTable&gt;.&lt;NomColonne&gt;"). <br>La colonne est parcourue pour remplir la série. Si une valeur n'est pas numérique, un "trou" est créé dans la série. |
| *grFonction* | Etiquettes de catégorie remplies par une fonction / procédure WLangage. <br>Dans ce cas, &lt;Détail 1&gt; correspond au nom de la procédure WLangage à appeler.<br><br>Pour chaque nombre, la procédure est appelée. Cette procédure est du type : <br><br><pre><code>PROCEDURE MaProcédure(<Champ Graphe>, <br><Numéro de la série>, <Index dans la série>)</code></pre><br><br>La procédure renvoie le nombre voulu. Pour indiquer la fin de la série, la procédure doit renvoyer <u><u><u><u>Faux</u></u></u></u>.<br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png) Cette constante n'est pas disponible. |
| *grListe* | Etiquettes de catégorie remplies depuis un champ Liste.<br><br>Dans ce cas, &lt;Détail 1&gt; correspond au nom d'un champ Liste. Le champ Liste est parcouru pour remplir la série. Si une valeur n'est pas numérique, un "trou" est créé dans la série. |
| *grProgrammation* | Etiquettes de catégorie remplies par programmation.<br><br>&lt;Détail 1&gt; peut correspondre à une série de nombre dans une même chaîne de caractères, séparés par des ";". |
| *grRubrique* | Etiquettes de catégorie remplies depuis une rubrique.<br><br>&lt;Détail 1&gt; correspond au nom du fichier de données (ou de la requête), &lt;Détail 2&gt; correspond au nom de la rubrique. La série va parcourir le fichier de données en prenant toutes les valeurs de la rubrique.<br><br>Si une valeur n'est pas numérique, un "trou" est créé dans la série. |
| *grTableau* | Etiquettes de catégorie remplies depuis un tableau WLangage.<br><br>&lt;Détail 1&gt; est le nom du tableau. &lt;Détail 2&gt; peut être l'indice de la colonne (tableau à 2 dimensions) ou un nom de membre (tableau de structures). |



**`<Détail 1> : (Type attendu (paramètre optionnel))`**

Premier paramètre attendu, en fonction du type de source de données.



**`<Détail N> : (Type attendu (paramètre optionnel))`**

Dernier paramètre attendu, en fonction du type de source de données.

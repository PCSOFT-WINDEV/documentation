
## SNMPChargeMIB (Fonction)

***En anglais : SNMPLoadMIB***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Charge en mémoire et analyse un fichier MIB. Ce fichier contient les définitions d'OID permettant de convertir les identifiants numériques en chaîne. Il contient également un texte décrivant les OID.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MIBChargé est un booléen
CheminMIBs est une chaîne = ComplèteRep(fRepExe()) + "mibs\"
Mib est une chaîne
Mib = "RFC1382-MIB.mib" // MIB du protocole X25
MIBChargé = SNMPChargeMIB(CheminMIBs + Mib)
SI MIBChargé = Faux ALORS 
	Erreur("Erreur MIB :" + ErreurInfo())
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = SNMPChargeMIB(<Nom du fichier MIB>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier a été correctement chargé et analysé, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'obtenir plus de renseignements sur le problème rencontré.




**`<Nom du fichier MIB> : (Chaîne de caractères)`**

Chemin complet du fichier MIB. Ce chemin peut être un chemin absolu ou relatif au répertoire courant.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Sous Windows, un certain nombre de fichiers MIB sont livrés dans le répertoire system32.

- Il est possible de charger plusieurs fichiers MIB. Les recherches seront effectuées sur tous les fichiers MIB chargés.

- Si un fichier MIB accède dans son code à un autre fichier MIB, le chargement du second fichier MIB est automatiquement réalisé (si le nom précisé dans le fichier MIB est identique au nom physique du fichier MIB).

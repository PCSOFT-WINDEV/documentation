


## grSurfaceCouleurAltitude (Fonction)

***En anglais : grSurfaceAltitudeColor***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Définit la couleur utilisée pour une altitude (Z) d'un graphe de type Surface.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000020630&name=grsurfacecouleuraltitude_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
grSurfaceCouleurAltitude(GRF_Surface, 100, RougeClair)
grSurfaceCouleurAltitude(GRF_Surface, 0, BleuPastel)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`grSurfaceCouleurAltitude(<Nom du graphe> , <Altitude> , <Couleur>)`
---

**`<Nom du graphe> : (Nom de champ ou chaîne de caractères)`**

Nom du graphe à manipuler. Ce nom peut correspondre :

- au nom du graphe défini par programmation avec la fonction [grCrée](../WDLang3/3042005.md).

- au nom du champ Graphe (présent dans l'éditeur de fenêtres ou d'états).




**`<Altitude> : (Réel)`**

Altitude pour laquelle la couleur sera appliquée. 

**`<Couleur> : (Entier ou constante de type Entier)`**

Couleur de l'altitude spécifiée. Cette couleur peut correspondre à :

- une couleur RVB (obtenue avec la fonction [RVB](../WDLang1/3029012.md)),

- une couleur TSL (obtenue avec la fonction [TSL](../WDLang1/3029057.md)),

- une couleur prédéfinie (voir [les couleurs prédéfinies du WLangage](../WDLang5/3010002.md)).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Si des couleurs d'altitude sont spécifiées, les facettes se situant à ces altitudes recevront ces couleurs. 

Pour les facettes se situant entre deux couleurs, un dégradé linéaire sera appliqué.

Pour les facettes en dehors des bornes de couleurs, la couleur de la série sera appliquée.

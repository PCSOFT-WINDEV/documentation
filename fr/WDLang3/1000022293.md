


## &lt;Variable EmailSessionOutlook&gt;.MiseAJour (Fonction)

***En anglais : &lt;EmailOutlookSession variable&gt;.Update***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Met à jour les messages de MS Exchange par rapport au serveur d'emails. Les opérations suivantes sont réalisées :

- Les messages présents dans la boîte d'envoi de MS Exchange sont envoyés vers le serveur d'email Internet.

- Les messages en attente sur le serveur Internet sont transférés dans la boîte de réception du client MS Exchange.




La fonction **&lt;Variable EmailSessionOutlook&gt;.MiseAJour** est utilisable uniquement avec le client MS Exchange.

**Rappel** : Pour manipuler des emails, il est nécessaire d'ouvrir une session d'email grâce à la fonction [&lt;Variable EmailSessionOutlook&gt;.OuvreSession](../WDLang3/1000022302.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouverture de la session
IDSession est un EmailSessionOutlook
...
// Récupération des messages dans la boîte de réception de MS Exchange
SI IDSession.MiseAJour() = Vrai ALORS
	Info("Messages récupérés")
SINON
	Erreur()
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Session>.MiseAJour()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la mise à jour a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. En cas d'erreur, la variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.




**`<Session> : (Variable de type emailSessionOutlook)`**

Nom de la variable de type [emailSessionOutlook](../WDLang3/1000018767.md) permettant d'identifier la session.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Erreur lors de la transmission de l'email
<a name="erreur_lors_transmission_email_ELTPARAGRAPHE000219"></a>Si un message n'a pas pu être transmis à cause d'une mauvaise connexion Internet ou d'une adresse incorrecte, le message est retourné au client MS Exchange : le message apparaît donc dans les messages non lus.

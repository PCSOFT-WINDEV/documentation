
## NotesVueListeColonne (Fonction)

***En anglais : NotesViewListColumn***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Liste les colonnes présentes dans la vue en cours dans Lotus Notes.

La vue en cours a été spécifiée à l'aide de la fonction [NotesActiveVue](../WDLang3/1000005118.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1001000005129&name=notesvuelistecolonne_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Ouverture de la connexion
IDConnexion est un entier
IDConnexion = NotesOuvreConnexion("MotDePasse", "ServeurMars", ...
		"C:\Users\Julie\AppData\Local\Lotus\Notes\Data\user.id")
// Spécification de la base Lotus Notes à utiliser
IDBase est un entier
IDBase = NotesOuvreBase(IDConnexion, "julie.nsf")
...
// Activation de la vue
NotesActiveVue(IDBase, "MesClients")
// Liste des colonnes triables de cette vue
ListeColonne est une chaîne
ListeColonne = NotesVueListeColonne(IDBase)
UneColonne est une chaîne
// Pour chaque colonne
POUR TOUTE CHAINE UneColonne DE ListeColonne SEPAREE PAR RC
	// Ajout des caractéristiques de la colonne dans un champ Liste
	ListeAjoute(LISTE_ListeColonneVue, UneColonne)
	// Par exemple, la colonne MaColonne est de type Texte, triable 
	// triée dans l'ordre croissant et sensible à la casse : 
	// "MaColonne+TAB+notesTypeTexte+TAB+1+TAB+1+TAB+1+TAB+0"
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = NotesVueListeColonne(<Identifiant de la session> [, <Colonnes listées>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste des colonnes présentes dans la vue en cours sous la forme :

```txt
<Nom colonne 1> + TAB + <Type colonne 1> + TAB + <Colonne 1 triable> + TAB + 
<Sens tri colonne 1> + TAB + <Sensibilité casse colonne 1> + TAB + 
<Sensibilité accents colonne 1> + RC + <Nom colonne 2> + TAB + 
<Type colonne 2> + TAB + <Colonne 2 triable> + TAB + <Sens tri colonne 2> + TAB + 
<Sensibilité casse colonne 2> + TAB + <Sensibilité accents colonne 2> + ...
```

Où :

- &lt;Nom colonne X&gt; correspond au nom de la colonne.

- &lt;Type colonne X&gt; correspond au type de la colonne. Ce paramètre peut correspondre à :

	- *notesTypeDateHeure* : Colonne de type DateHeure.

	- *notesTypeInconnu* : Type de colonne non supporté.

	- *notesTypeNombre* : Colonne de type Nombre.

	- *notesTypeTexte* : Colonne de type Texte.

	- *notesTypeTexteRiche* : Colonne de type Texte Riche (contenant un formatage et éventuellement des pièces attachées).




- &lt;Colonne X triable&gt; indique si la colonne est triable (1) ou non (0).

- &lt;Sens tri colonne X&gt; indique si la colonne est triée par ordre croissant (1) ou par ordre décroissant (0).

- &lt;Sensibilité casse colonne X&gt; indique si la colonne est sensible à la casse (1) ou non (0).

- &lt;Sensibilité accents colonne X&gt; indique si la colonne est sensible aux accents (1) ou non (0).




Si le paramètre &lt;Colonne X triable&gt; correspond à 0, les paramètres &lt;Sens tri colonne X&gt;, &lt;Sensibilité casse colonne X&gt; et &lt;Sensibilité accents colonne X&gt; ne doivent pas être pris en compte et correspondent toujours à 0.

**`<Identifiant de la session> : (Entier)`**

Identifiant de la session à utiliser. Cet identifiant est renvoyé par la fonction [EmailOuvreSessionNotes](../WDLang3/3032121.md) ou [NotesOuvreBase](../WDLang3/1000005101.md).

**`<Colonnes listées> : (Constante optionnelle de type entier)`**

Mode de sélection des colonnes listées :


|   |   |
| --- | --- |
| *notesLstColonneTous* | Renvoie toutes les colonnes (qu'elles soient triables ou non). |
| *notesLstColonneTriable* <br>(valeur par défaut) | Renvoie uniquement les colonnes triables. |

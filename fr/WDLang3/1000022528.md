
## SaaSEnvoieCodeDeSécurité (Fonction)

***En anglais : SaaSSendSecurityCode***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Envoie le code de sécurité à l'utilisateur pour qu'il puisse changer son mot de passe.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Clic sur "Mot de passe oublié ?"
SI PAS SaaSEnvoieCodeDeSécurité("Marcel") ALORS
	Info("Récupération du mot de passe impossible. " + 
		"Avez vous précisé une adresse mail dans votre profil ?")
	RETOUR
FIN

// Validation du formulaire
SI SaaSConnecte("Marcel", SAI_Code) ALORS
	// Code valide, changer le mot de passe pour un nouveau 
	SaaSChangeMotDePasse(SAI_NouveauMotDePasse)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = SaaSEnvoieCodeDeSécurité(<Login> [, <Webservice>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'envoi du code de sécurité a réussi,

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Login> : (Chaîne de caractères)`**

Login de l'utilisateur.

**`<Webservice> : (Chaîne de caractères optionnelle)`**

Webservice à utiliser. Ce paramètre correspond :

- soit au nom de l'hôte du webservice SaaS (par exemple, "Machine", "Machine.MonReseau.fr" , "http://Mondomaine.com").

- soit à l'URL complète du webservice SaaS (par exemple, "http://domaine.com/WDSAAS280_WEB/WDSaaS280.awws").


Par défaut, la connexion se fera au webservice SaaS installé avec le serveur d'application WEBDEV sur la machine locale.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le code de sécurité est valide tant que la session qui a émis la demande de changement de mot de passe est active.

- Le code de sécurité est envoyé à l'adresse email indiquée dans la variable de type [saasUtilisateur](../WDLang3/1000019083.md) correspondant à l'utilisateur (propriété **Email**). Si cette propriété n'est pas renseignée, l'email est envoyé à son login (si ce login correspond à une adresse email valide). 

- Pour pouvoir envoyer le code de sécurité, il est nécessaire d'avoir configuré les paramètres SMTP de l'administrateur SaaS : 

	- soit via l'administrateur SaaS. 

	- soit via la fonction [SaaSAdminConfigureSMTP](../WDLang3/1000022592.md).


## WidgetProcédure (Fonction)

***En anglais : WidgetProcedure***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de spécifier la procédure qui sera appelée lorsque l'utilisateur effectue un clic dans un Widget iOS 14. Cette fonction doit être utilisée dans l'événement "Initialisation" associé au projet, en spécifiant la cible d'exécution "iOS". 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
<SI CibleExécution=iOS>
	WidgetProcédure(OnWidgetClic)
<FIN>
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`WidgetProcédure(<Procédure WLangage>)`
---

**`<Procédure WLangage> : (Nom de procédure)`**

Procédure WLangage (également nommée "callback") appelée lors d'un clic sur un Widget iOS 14. Cette procédure est de la forme : 

```txt
PROCEDURE <Nom de la procédure> (<Nom de l'élément>)
```
où &lt;Nom de l'élément&gt; correspond : 

- au nom complet du champ cliqué pour les Widgets de taille moyenne ou large, 

- au nom de la fenêtre pour les Widgets de taille petite. 




Cette procédure peut également être une procédure interne.

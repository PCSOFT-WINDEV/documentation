


## ContexteFerme (Fonction)

***En anglais : ContextClose***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ferme un contexte de page. Dans certains cas spécifiques (groupware utilisateur, ...), il est possible de spécifier la valeur à renvoyer.

Attention : La fermeture du contexte de la page en cours est déconseillée.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ContexteFerme(PAGE_Page4)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Fermeture du contexte de la page

`ContexteFerme(<Page>)`
---

**`<Page> : (Nom de page)`**

Nom de la page dont le contexte doit être fermé. Si ce paramètre est égal à une chaîne vide (""), le nom de la page correspond à la page à laquelle appartient le traitement en cours.  


<a name="SYNTAXE2"></a>

### Fermeture du contexte de la page avec retour d'informations

`ContexteFerme(<Page> , <Valeur de retour>)`
---

**`<Page> : (Nom de page)`**

Nom de la page dont le contexte doit être fermé. Si ce paramètre est égal à une chaîne vide (""), le nom de la page correspond à la page à laquelle appartient le traitement en cours.

**`<Valeur de retour> : (Type de la valeur renvoyée)`**

Valeur renvoyée à la page appelante. Ce paramètre est réservé à l'utilisation de certaines pages spécifiques de WEBDEV ouvertes automatiquement (pages du groupware utilisateur, ...). Si ce paramètre est utilisé pour une page n'attendant pas de valeur de retour, la valeur de ce paramètre est ignorée.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Sessions prélancées
<a name="sessions_prelancees_ELTPARAGRAPHE000050"></a>Si votre projet utilise les sessions prélancées, cette fonction ne doit pas être utilisée dans l'événement "Initialisation" du projet. Cette fonction doit être utilisée dans l'événement "Initialisation du projet après connexion au site".

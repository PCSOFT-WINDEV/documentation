
## GglAnalyticsAjouteException (Fonction)

***En anglais : GglAnalyticsAddException***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute une exception auprès de Google Analytics. 

**Remarque** : Au 1er Juillet 2023, Google Analytics passe de Universal Analytics à Google Analytics 4. WEBDEV 28 gère les deux versions. Les fonctions WLangage ont été adaptées. Par défaut, pour les projets existants, le mode géré est Universal Analytics et pour les nouveaux projets, le mode géré est Google Analytics 4. Pour plus de détails, consultez [WEBDEV et Google Analytics](../WDLang2/1000022374.md). 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Message est un Variant
Message.exDescription = ErreurInfo()
Message.exFatal = Faux
GglAnalyticsAjouteException(Message)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`GglAnalyticsAjouteException([<Options>])`
---

**`<Options> : (Variant optionnel)`**

Options supplémentaires dépendant de Google Analytics. Pour connaître les valeurs possibles, consultez la documentation de Google Analytics : 

- Universal Analytics (disponible jusqu'en Juillet 2023) :  [https://developers.google.com/analytics/devguides/collection/analyticsjs/exceptions](https://developers.google.com/analytics/devguides/collection/analyticsjs/exceptions)

- Google Analytics 4 : [https://developers.google.com/tag-platform/gtagjs/reference/events#exception](https://developers.google.com/tag-platform/gtagjs/reference/events#exception)


Par défaut, ce paramètre correspond à Null.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Pour utiliser cette fonction, vous devez posséder et enregistrer dans le projet un identifiant de suivi Google Analytics. Pour plus de détails, consultez [WEBDEV et Google Analytics](../WDLang2/1000022374.md).

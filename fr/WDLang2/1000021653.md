


## &lt;Cellule&gt;.FermeDialogue (Fonction)

***En anglais : &lt;Cell&gt;.CloseDialog***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Masque un champ Cellule précédemment affiché dans la page via la fonction [&lt;Cellule&gt;.AfficheDialogue](../WDLang2/1000021652.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ferme le champ Cellule de question
CELL_Question.FermeDialogue()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Champ Cellule>.FermeDialogue()`
---

**`<Champ Cellule> : (Nom de champ)`**

Nom de la cellule qui sera masquée. Cette cellule est superposable et a été affichée par la fonction [&lt;Cellule&gt;.AfficheDialogue](../WDLang2/1000021652.md). Si ce n'est pas le cas, la fonction **&lt;Cellule&gt;.FermeDialogue** ne fait rien.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Les navigateurs ne rafraîchissent pas l'affichage au milieu d'un code Javascript : les fonctions [&lt;Cellule&gt;.AfficheDialogue](../WDLang2/1000021652.md) et **&lt;Cellule&gt;.FermeDialogue** doivent être utilisées dans des événements/traitements différents. Si ces fonctions sont utilisées dans le même événement/traitement, le champ Cellule n'est pas affiché.

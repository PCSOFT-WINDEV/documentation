


## ZoneRépétéeSupprimeSelect (Fonction)

***En anglais : LooperDeleteSelect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime la ou les lignes sélectionnées dans un champ Zone répétée.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Supprime toutes les répétitions sélectionnées
SI OuiNon("Voulez-vous vraiment supprimer les clients sélectionnés ?") ALORS 
	ZoneRépétéeSupprimeSelect(ZR_Affichage_Client)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`ZoneRépétéeSupprimeSelect(<Champ Zone répétée>)`
---

**`<Champ Zone répétée> : (Nom de champ)`**

Nom du champ Zone répétée à manipuler.
Si ce paramètre est égal à chaîne vide (""), la suppression sera effectuée sur le champ Zone répétée en cours.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000069"></a>La fonction **ZoneRépétéeSupprimeSelect** peut être utilisée indifféremment sur : 

- un champ Zone répétée fichier, 

- un champ Zone répétée par programmation. 



<a name="NOTE0_2"></a>


### Suppression dans un champ Zone répétée fichier
<a name="suppression_dans_champ_zone_repetee_fichier_ELTPARAGRAPHE000084"></a>La fonction **ZoneRépétéeSupprimeSelect** supprime les enregistrements correspondants dans le fichier de données associé au champ. Si des rubriques sont reliées à d'autres fichiers de données, aucune modification n'est effectuée automatiquement sur ces fichiers de données liés.
<a name="NOTE0_3"></a>


### Intégrité référentielle et champ Zone répétée fichier
<a name="integrite_referentielle_champ_zone_repetee_fichier_ELTPARAGRAPHE000096"></a>Si la gestion de l'intégrité référentielle est active (fonction [HGèreIntégrité](../WDLang4/3044058.md)), testez la fonction [HErreurIntégrité](../WDLang4/3044086.md) après l'appel de la fonction **ZoneRépétéeSupprimeSelect** pour vérifier l'intégrité. Si une erreur d'intégrité est détectée, la répétition n'est pas supprimée et reste visible dans le champ.
<a name="NOTE0_4"></a>


### Suppression dans un champ Zone répétée fichier et Trigger
<a name="suppression_dans_champ_zone_repetee_fichier_trigger_ELTPARAGRAPHE000114"></a>Il est possible de déclencher un trigger lors de la suppression d'un enregistrement dans un champ Zone répétée fichier. Pour plus de détails, consultez la documentation de la fonction [HDécritTrigger](../WDLang4/3044074.md).
<a name="NOTE0_5"></a>


### Gestion des erreurs
<a name="gestion_des_erreurs_ELTPARAGRAPHE000126"></a>En cas d'échec de la suppression, la variable [ErreurDétectée](../WDLang1/3087001.md) est à <u><u><u><u>Vrai</u></u></u></u>. Pour plus de détails sur l'erreur, il est possible d'utiliser la fonction [ErreurInfo](../WDLang1/3013008.md).
<a name="NOTE0_6"></a>


### Blocage du fichier de données lié
<a name="blocage_fichier_donnees_lie_ELTPARAGRAPHE000145"></a>Si le fichier de données est bloqué dans le traitement en cours, la fonction **ZoneRépétéeSupprimeSelect** supprime la ligne spécifiée et débloque le fichier de données.


## DynamicServingConfigure (Fonction)

***En anglais : DynamicServingConfigure***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique l'environnement à utiliser par le "Dynamic Serving" pour choisir le jeu de pages à afficher.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Option de menu
// Toujours afficher la version Ordinateur des pages
DynamicServingConfigure(dsDesktop)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`DynamicServingConfigure(<Environnement>)`
---

**`<Environnement> : (Constante de type Entier)`**

Type d'environnement à utiliser : 


|   |   |
| --- | --- |
| *dsAutomatique* | Le choix des pages à afficher est automatiquement réalisé par WEBDEV en fonction de la plateforme en cours. |
| *dsDesktop* | Les pages en mode "Ordinateur" sont forcées. |
| *dsMobile* | Les pages en mode "Mobile" sont forcées. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
L'appel à la fonction **DynamicServingConfigure** déclenche un appel de la fonction [PageUtilise](../WDLang2/3058016.md) pour toutes les pages actuellement ouvertes : 

- qui sont associées au Dynamic Serving,

- et dont le mode d'affichage ne correspond pas à celui choisi (y compris la page courante).

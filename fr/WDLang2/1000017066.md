
## ConfigureContexteAWP (Fonction)

***En anglais : ConfigureAWPContext***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Configure le mode de fonctionnement du contexte AWP. 

Pour plus de détails sur les modes de fonctionnement possibles, consultez [Gestion des contextes AWP](../WDChamp/3539060.md). 

**Remarque** : Si la fonction **ConfigureContexteAWP** n'est pas utilisée, le mode de gestion des contextes utilisé par défaut est la gestion des contextes par cookies. 

**Attention** : Cette fonction est destinée aux développeurs maîtrisant WEBDEV.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Gestion du contexte sur un fichier sur le serveur
// avec transmission de l'identifiant par cookie
ConfigureContexteAWP(ctxDisque, ctxIDCookie)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`ConfigureContexteAWP(<Mode de mémorisation du contenu> [, <Mode de transmission de l'identifiant>])`
---

**`<Mode de mémorisation du contenu> : (Constante de type entier)`**

Mode de mémorisation du contenu du contexte :


|   |   |
| --- | --- |
| *ctxCookie* | Le contenu est transmis dans un cookie (comportement compatible avec la version 11).<br><br>**Attention** : Cette constante ne doit pas être utilisée pour stocker des données sensibles. En effet, ces informations sont stockées "en clair". |
| *ctxDisque* | Le contenu du contexte est sauvegardé dans un fichier sur le serveur. |



**`<Mode de transmission de l'identifiant> : (Constante de type entier (optionnelle))`**

Défini le mode de transmission de l'identifiant (uniquement pris en compte si le &lt;Mode de mémorisation du contenu&gt; correspond à la constante *ctxDisque*) :


|   |   |
| --- | --- |
| *ctxIDCookie* | L'identifiant est transmis par un cookie. |
| *ctxIDCookieURL*<br>(valeur par défaut) | L'identifiant est transmis en priorité par cookies. Si les cookies ne fonctionnent pas, l'identifiant est transmis par l'URL. Lors de la création du contexte, les deux modes sont utilisés car il est impossible de savoir si les cookies seront acceptés. |
| *ctxIDUrl* | L'identifiant est transmis par l'URL. |









<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Il est conseillé d'utiliser la fonction **ConfigureContexteAWP** dans l'événement "Initialisation" du projet. En effet, il est impossible de modifier ce mode lorsqu'une session a été commencée.

- La fonction **ConfigureContexteAWP** est prise en compte uniquement en mode AWP.

- Par défaut, si la fonction **ConfigureContexteAWP** n'est pas utilisée, le mode de gestion des contextes est la gestion des contextes par cookies. 

- **Rappel** : Le traitement d'initialisation du projet est exécuté à chaque affichage d'une page AWP.

- Si votre projet utilise les sessions prélancées, cette fonction ne doit pas être utilisée dans l'événement "Initialisation" du projet. Cette fonction doit être utilisée dans l'événement "Initialisation du projet après connexion au site". 

- Pour plus de détails sur les pages AWP, consultez [Les pages dynamiques AWP](../WDChamp/3539009.md).

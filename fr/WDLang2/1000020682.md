


## ExécuteProcédureDifférée (Fonction)

***En anglais : ExecuteDelayedProcedure***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Exécute une procédure différée (dans le serveur d'application WEBDEV).

Attention : Cette fonction permet uniquement d'exécuter les procédures différées associées aux tâches différées WEBDEV. Elle ne permet pas d'exécuter des tâches planifiées WEBDEV. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// La procédure CalculCA a été définie 
// en tant que procédure différée sous l'éditeur de code
ExécuteProcédureDifférée(CalculCA, 0)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`ExécuteProcédureDifférée(<Procédure WLangage> [, <Paramètre 1> [... [, <Paramètre N>]]])`
---

**`<Procédure WLangage> : (Nom de procédure)`**

Nom de la procédure WLangage à lancer. Cette procédure doit avoir été définie comme procédure différée sous l'éditeur de code. Pour plus de détails, consultez [Procédures différées](../WDLang2/1000019455.md).

**`<Paramètre 1> : (Type du paramètre)`**

Premier paramètre associé à la procédure.



**`<Paramètre N> : (Type du paramètre)`**

Nième paramètre associé à la procédure.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le lancement de la procédure peut échouer si la procédure est déjà lancée en tâche de fond.

- Si votre projet utilise les sessions prélancées, cette fonction ne doit pas être utilisée dans l'événement "Initialisation" du projet. Cette fonction doit être utilisée dans l'événement "Initialisation du projet après connexion au site".


## GglAnalyticsAjoutePage (Fonction)

***En anglais : GglAnalyticsAddPage***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute une page auprès de Google Analytics. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
GglAnalyticsAjoutePage("/domaine.com/tarif.shtml")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`GglAnalyticsAjoutePage([<Chemin de la page> [, <Options>]])`
---

**`<Chemin de la page> : (Chaîne de caractères optionnelle)`**

Partie de l'adresse de la page représentant le chemin. Cette valeur doit commencer par un caractère slash (/).
Par défaut, ce paramètre correspond au chemin de la page courante. 

**`<Options> : (Variant optionnel)`**

Options supplémentaires dépendant de Google Analytics. Pour connaître les valeurs possibles, consultez la documentation de Google Analytics [https://developers.google.com/analytics/devguides/collection/analyticsjs/pages](https://developers.google.com/analytics/devguides/collection/analyticsjs/pages). 
Par défaut, ce paramètre correspond à Null.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Pour utiliser cette fonction, vous devez posséder et enregistrer dans le projet un identifiant de suivi Google Analytics. Pour plus de détails, consultez [WEBDEV et Google Analytics](../WDLang2/1000022374.md). 

- L'ajout de la page courante est automatiquement réalisé (lors de l'inclusion de analytics.js). 

- **Remarque** : Au 1er Juillet 2023, Google Analytics passe de Universal Analytics à Google Analytics 4. WEBDEV 28 gère les deux versions. Les fonctions WLangage ont été adaptées. Par défaut, pour les projets existants, le mode géré est Universal Analytics et pour les nouveaux projets, le mode géré est Google Analytics 4. Pour plus de détails, consultez [WEBDEV et Google Analytics](../WDLang2/1000022374.md).

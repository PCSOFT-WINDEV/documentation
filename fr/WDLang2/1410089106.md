
## SessionPageDemandée (Fonction)

***En anglais : SessionRequestedPage***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nom de la page explicitement demandée au lancement de la session.

Remarque : Cette fonction est notamment utile dans le code du projet, pour connaître le nom de la page qui va être renvoyée. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SELON (SessionPageDemandée()) 
CAS "PAGE_Admin"
	// Vérifie les paramètres de connexion reçus sur l'URL
	SI PAS ConnecteUtilisateur() ALORS
		PageAffiche(PAGE_ErreurLogin)
	FIN
AUTRES CAS
	// Rien
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = SessionPageDemandée()`
---

**`<Résultat> : (Chaîne de caractères)`**



- Nom de la page. Ce nom peut correspondre aux cas suivants : 

	- Connexion à un site en mode session par une page référençable : nom de la page référençable.

	- Connexion à un site en mode session par la fonction [SiteDynamiqueAffiche](../WDLang2/3058024.md) en précisant une page : nom de la page (uniquement si l'option d'accès appropriée est activée dans la page).

	- Connexion à un site en mode session depuis une application WINDEV qui demande une page particulière : nom de la page (uniquement si l'option d'accès appropriée est activée dans la page).

	- Affichage d'une page AWP : nom de la page AWP.




- Chaîne vide ("") dans les autres cas, ou si la fonction est appelée après la requête de connexion à la session.


## FramesetUtilise (Fonction)

***En anglais : FramesetUse***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Affiche un frameset WEBDEV dans le navigateur de l'internaute et ferme tous les contextes de pages et de framesets en cours.

Si la fonction **FramesetUtilise** est utilisée dans le code d'un bouton, le frameset sera ouvert dans la destination définie pour ce bouton (onglet "Général" de la description du champ ou fonction [ChangeDestination](../WDLang2/3058005.md)).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Passage de l'application en anglais
Nation(nationAnglais)
// Réaffichage du frameset FSET_LOGIN avec libellé en anglais
FramesetUtilise(FSET_LOGIN)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`FramesetUtilise(<Nom du frameset> [, <Paramètre 1> [... [, <Paramètre N>]]])`
---

**`<Nom du frameset> : (Chaîne de caractères)`**

Nom du frameset à afficher. 
Si ce paramètre est égal à une chaîne vide (""), le nom du frameset est celui du frameset auquel appartient l'événement en cours.

**`<Paramètre 1> : (Type de la valeur envoyée à la page (optionnel))`**

Paramètre(s) à passer à l'événement "Déclarations globales" du frameset à ouvrir. Ces paramètres sont passés par référence et sont considérés comme des variables globales au frameset et à toutes les pages du frameset.



**`<Paramètre N> : (Type de la valeur envoyée à la page (optionnel))`**

Paramètre(s) à passer à l'événement "Déclarations globales" du frameset à ouvrir. Ces paramètres sont passés par référence et sont considérés comme des variables globales au frameset et à toutes les pages du frameset.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Récupération des paramètres
<a name="recuperation_des_parametres_ELTPARAGRAPHE000062"></a>Pour récupérer les paramètres, dans l'événement "Déclarations globales" du frameset, indiquez en début de code la ligne suivante :

```txt
PROCEDURE <Nom du frameset>(<Paramètre 1> [,...[, <Paramètre N>]])
```

<a name="NOTE0_2"></a>




### Principe détaillé de l'ouverture d'un frameset
<a name="principe_detaille_ouverture_frameset_ELTPARAGRAPHE000071"></a>La fonction **FramesetUtilise** effectue les opérations suivantes :

- création un contexte de frameset sur le serveur.

- création un contexte de page sur le serveur pour toutes les pages du frameset sur le serveur.

- renvoi du frameset dans le navigateur de l'internaute.





<a name="NOTE0_3"></a>




### Opérations effectuées à l'ouverture du frameset
<a name="operations_effectuees_ouverture_frameset_ELTPARAGRAPHE000087"></a>Les différentes opérations effectuées lors de l'affichage du frameset sont les suivantes :

- Exécution des événements "Déclarations globales" et "Initialisation" du frameset.

- Ouverture des pages du frameset (exécution des événements "Déclarations globales" et "Initialisation" de chaque page).

- Code WLangage suivant l'appel à la fonction **FramesetUtilise** (s'il existe).





<a name="NOTE0_4"></a>




### Sites multilingues
<a name="sites_multilingues_ELTPARAGRAPHE000103"></a>La fonction **FramesetUtilise** peut être utilisée dans le cas de sites multilingues. En effet, cette fonction permet de ré-ouvrir un frameset en prenant en compte la langue spécifiée par la fonction [Nation](../WDLang1/3054014.md).


<a name="NOTE0_5"></a>


### Sessions prélancées
<a name="sessions_prelancees_ELTPARAGRAPHE000119"></a>Si votre projet utilise les sessions prélancées, cette fonction ne doit pas être utilisée dans l'événement "Initialisation" du projet. Cette fonction doit être utilisée dans l'événement "Initialisation du projet après connexion au site". 
<a name="NOTE0_6"></a>




### Divers
<a name="divers_ELTPARAGRAPHE000127"></a>

- Pour ré-afficher un frameset, il est conseillé d'utiliser la fonction [FramesetActualise](../WDLang2/3058014.md).

- Pour afficher un frameset sans fermer les contextes ouverts sur le serveur, il est conseillé d'utiliser la fonction [FramesetAffiche](../WDLang2/3058006.md).

- Il est déconseillé de faire suivre l'appel de la fonction **FramesetUtilise** d'un code WLangage.

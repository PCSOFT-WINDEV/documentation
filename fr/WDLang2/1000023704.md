
## &lt;Zone répétée&gt;.Trie (Fonction)

***En anglais : &lt;Looper&gt;.Sort***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de trier un champ Zone répétée selon un ou plusieurs de ses attributs.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Tri selon un des attributs du champ Zone répétée choisi dans un champ Sélecteur
SELON SEL_SELECT1
	CAS 1: ZR_Client.Trie("NOM")
	CAS 2: ZR_Client.Trie("SOCIETE")
	CAS 3: ZR_Client.Trie("VILLE")
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Trier un champ Zone répétée selon une liste d'attributs 

`<Résultat> = <Champ Zone répétée>.Trie(<Liste d'attributs>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le tri a pu s'exécuter, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Champ Zone répétée> : (Nom de champ)`**

Nom du champ Zone répétée à manipuler.

**`<Liste d'attributs> : (Nom de champ)`**

Nom des attributs selon lesquels le champ Zone répétée doit être trié (jusqu'à 10 attributs maximum). Cette chaîne de caractères doit être de la forme :

```txt
[<Sens>]<Attribut 1> + TAB + [<Sens>]<Attribut 2> + TAB + ...
```
avec :

- &lt;Sens&gt; : sens du tri (optionnel), peut prendre les valeurs suivantes :

	- "+" : le tri sera croissant (valeur par défaut).

	- "-" : le tri sera décroissant.




- &lt;Attribut&gt; : nom de l'attribut utilisé pour le tri.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000200"></a>La fonction **&lt;Zone répétée&gt;.Trie** peut être utilisée indifféremment sur : 

- un champ Zone répétée fichier,

- un champ Zone répétée par programmation. 



<a name="NOTE0_3"></a>


### Tri
<a name="tri_ELTPARAGRAPHE000225"></a>Le prochain ajout d'une répétition dans le champ Zone répétée (fonctions [&lt;Zone répétée&gt;.Ajoute](../WDLang2/1000023469.md), [&lt;Zone répétée&gt;.AjouteLigne](../WDLang2/1000023475.md), [&lt;Zone répétée&gt;.Insère](../WDLang2/1000023509.md) et [&lt;Zone répétée&gt;.InsèreLigne](../WDLang2/1000023511.md)) tient compte du tri.
<a name="NOTE0_4"></a>


### Zone répétée liée à une source de données et utilisant un parcours programmé
<a name="zone_repetee_liee_une_source_donnees_utilisant_parcours_programme_ELTPARAGRAPHE000255"></a>La fonction **&lt;Zone répétée&gt;.Trie** ne fonctionne pas sur un champ Zone répétée ayant les caractéristiques suivantes : 

- champ Zone répétée lié à une source de données, 

- champ Zone répétée utilisant l'option **Parcours programmé** (onglet "Contenu" de la fenêtre de description du champ). 




Pour changer l'ordre d'affichage d'un champ Zone répétée avec **parcours programmé**, il est nécessaire d'adapter le code se trouvant dans les différents traitements de lecture du champ Zone répétée (traitements lecture du premier, du suivant, du précédent, ...).

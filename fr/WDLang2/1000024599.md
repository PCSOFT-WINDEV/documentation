
## PageAgencementEnCours (Fonction)

***En anglais : PageCurrentLayout***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie l'indice de l'agencement en cours : 

- dans la page en cours.

- dans la page spécifiée.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI PageAgencementEnCours() = 2 ALORS
	LIB_Détail = Client.Détail
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Connaître l'agencement de la page en cours

`<Résultat> = PageAgencementEnCours()`
---

**`<Résultat> : (Entier)`**



- Indice de l'agencement en cours.

- 0 si la page ne possède aucun agencement. 

- une des constantes suivantes : 
	


|   |   |
| --- | --- |
| *agencementImpression* | L'agencement spécifique pour l'impression est en cours. |
| *agencementWINDEV* | L'agencement spécifique pour WINDEV est en cours. |







<a name="SYNTAXE2"></a>

### Connaître l'agencement d'une page spécifique

`<Résultat> = PageAgencementEnCours(<Page>)`
---

**`<Résultat> : (Entier)`**



- Indice de l'agencement en cours.

- 0 si la page interne ne possède aucun agencement. 

- une des constantes suivantes : 
	


|   |   |
| --- | --- |
| *agencementImpression* | L'agencement spécifique pour l'impression est en cours. |
| *agencementWINDEV* | L'agencement spécifique pour WINDEV est en cours. |






**`<Page> : (Nom de page)`**

Nom de la page manipulée. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Les agencements sont numérotés dans leur ordre de création en édition.

- Si le terminal change d'orientation ou si la fonction [PageChangeAgencement](../WDLang2/1000024598.md) est exécutée, l'agencement change.

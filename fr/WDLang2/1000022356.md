
## GglAnalyticsAjouteEvénement (Fonction)

***En anglais : GglAnalyticsAddEvent***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute un événement auprès de Google Analytics. 

**Remarque** : Au 1er Juillet 2023, Google Analytics passe de Universal Analytics à Google Analytics 4. WEBDEV 28 gère les deux versions. Les fonctions WLangage ont été adaptées. Par défaut, pour les projets existants, le mode géré est Universal Analytics et pour les nouveaux projets, le mode géré est Google Analytics 4. Pour plus de détails, consultez [WEBDEV et Google Analytics](../WDLang2/1000022374.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
GglAnalyticsAjouteEvénement("Vidéo", "play", "démo produit", "poussette")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`GglAnalyticsAjouteEvénement(<Catégorie> , <Action> [, <Libellé> [, <Numéro d'événement> [, <Options>]]])`
---

**`<Catégorie> : (Chaîne de caractères)`**

Nom de la catégorie. Ce paramètre permet de regrouper plusieurs événements (par exemple "Vidéo").

**`<Action> : (Chaîne de caractères)`**

Nom de l'action liée à l'événement (par exemple  "jouer"). 

**`<Libellé> : (Chaîne de caractères optionnelle)`**

Libellé additionnel permettant de simplifier le classement des événements (par exemple "Publicité").
Par défaut, ce paramètre correspond à une chaîne vide. 

**`<Numéro d'événement> : (Entier optionnel)`**

Numéro associé à l'événement. Par défaut, ce paramètre correspond à 0. 

**`<Options> : (Variant optionnel)`**

Options supplémentaires dépendant de Google Analytics. Pour connaître les valeurs possibles, consultez la documentation de Google Analytics [https://developers.google.com/analytics/devguides/collection/analyticsjs/events](https://developers.google.com/analytics/devguides/collection/analyticsjs/events). 
Par défaut, ce paramètre correspond à Null.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Pour utiliser cette fonction, vous devez posséder et enregistrer dans le projet un identifiant de suivi Google Analytics. Pour plus de détails, consultez [WEBDEV et Google Analytics](../WDLang2/1000022374.md).

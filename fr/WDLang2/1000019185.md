
## JSInterruptionEvénement (Fonction)

***En anglais : JSInterruptEvent***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Interrompt le traitement de l'événement en cours.

Le traitement courant est la procédure WLangage spécifique qui a été associée à l'événement par la fonction [JSEvénément](../WDLang2/1000018924.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Intercepte tous les clics sur la page
JSInterruptionEvénement(sEvénementJS, jsInterruptionEvénementAction)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`JSInterruptionEvénement(<Evénement> [, <Options>])`
---

**`<Evénement> : (Chaîne de caractères)`**

Événement en cours à interrompre.
Il s'agit du paramètre reçu par la procédure (voir la fonction [JSEvénement](../WDLang2/1000018924.md) pour plus de détails).

**`<Options> : (Combinaison de constantes de type Entier optionnelle)`**

Indique le niveau d'interruption.
Si aucune constante n'est précisée, la procédure courante est simplement interrompue.

Les constantes possibles sont :


|   |   |
| --- | --- |
| *jsInterruptionEvénementAction* | L'action navigateur associée à l'événement est annulée, en plus de l'interruption de la procédure courante. Par exemple, si l'événement courant est un clic dans une case à cocher, le changement d'état de la case à cocher est annulé. |
| *jsInterruptionEvénementPropagation* | La propagation de l'événement est interrompue, en plus de l'interruption de la procédure courante. Si d'autres procédures sont associées au même événement sur le même objet, ces procédures seront cependant appelées. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La fonction **JSInterruptionEvénement** ne peut être utilisée que dans une procédure WLangage de traitement d'événements déclarée en utilisant la fonction [JSEvénement](../WDLang2/1000018924.md).

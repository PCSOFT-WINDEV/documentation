


## PageActualise (Fonction)

***En anglais : PageRefresh***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Actualise une page affichée dans le navigateur de l'internaute à partir du contexte présent sur le serveur. Cette fonction est principalement utilisée pour les sites en mode Session. 

**Attention** : Le contexte de la page doit exister sur le serveur. Le contexte de la page est créé par les fonctions [PageAffiche](../WDLang2/3058008.md) et [ContexteOuvre](../WDLang2/3058004.md). Tous les événements liés à la page ou aux champs ont déjà été exécutés. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Actualisation de la page PAGE_Page1
PageActualise(PAGE_Page1)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`PageActualise([<Page>])`
---

**`<Page> : (Nom de page)`**

Nom de la page à actualiser.
Si ce paramètre n'est pas spécifié ou est égal à une chaîne vide (""), le nom de la page correspond à la page à laquelle appartient l'événement en cours.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Utilisation de la fonction PageActualise dans un événement en mode AJAX
<a name="utilisation_fonction_docparampagetitleshort_dans_evenement_mode_ajax_ELTPARAGRAPHE000066"></a>**Rappel** : AJAX permet de rafraîchir uniquement les données modifiées dans une page HTML sans réafficher la totalité de la page. Pour plus de détails, consultez [AJAX](../WDLang2/3055101.md).

Si la fonction **PageActualise** est utilisée dans un événement en mode AJAX (terme "AJAX activé" présent dans le bandeau de l'événement) et si cet événement permet uniquement ou systématiquement l'actualisation d'une page, la technologie AJAX deviendra inutile. En mode AJAX, l'utilisation de la fonction **PageActualise** doit être réservée à des cas spécifiques.

Pendant un traitement long, il est possible d'afficher un message permettant de patienter grâce à la fonction [AjaxExécuteAsynchrone](../WDLang2/3055115.md). 


### Sessions prélancées
<a name="sessions_prelancees_ELTPARAGRAPHE000094"></a>Si votre projet utilise les sessions prélancées, cette fonction ne doit pas être utilisée dans l'événement "Initialisation" du projet. Cette fonction doit être utilisée dans l'événement "Initialisation du projet après connexion au site".


## ServeurAppliAjouteTâchePlanifiée (Fonction)

***En anglais : AppServerAddScheduledTask***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute une nouvelle tâche planifiée à l'application (site ou Webservice).

**Attention** : La tâche planifiée créée n'est pas persistante. Elle sera supprimée en cas de redémarrage du serveur d'application WEBDEV ou lors du déploiement du site.
Il est conseillé d'utiliser l'événement du projet "Configuration de site : démarrage du Serveur d'application" pour utiliser la fonction **ServeurAppliAjouteTâchePlanifiée**. La tâche planifiée sera ainsi automatiquement recréée.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Programme la tâche pour s'exécuter toutes les heures
MaTâchePlanifiée est un serveurAppliTâchePlanifiée
MaTâchePlanifiée.Nom = "MaTache"
MaTâchePlanifiée.Procédure = MaProc
MaTâchePlanifiée.Mois = "*"
MaTâchePlanifiée.JourDuMois = "*"
MaTâchePlanifiée.JourDeLaSemaine = "*"
MaTâchePlanifiée.Heure = "*"
MaTâchePlanifiée.Minute = "0"

SI PAS ServeurAppliAjouteTâchePlanifiée(MaTâchePlanifiée) ALORS
	// Cas d'erreur (par exemple si une tâche de même nom existe déjà)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = ServeurAppliAjouteTâchePlanifiée(<Tâche planifiée>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la tâche planifiée a été ajoutée. 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.




**`<Tâche planifiée> : (Variable de type serveurAppliTâchePlanifiée)`**

Nom de la variable de type [serveurAppliTâchePlanifiée](../WDLang2/1000022367.md) correspondant à la description de la tâche planifiée à ajouter.


## ServeurAppliExécuteTâchePlanifiée (Fonction)

***En anglais : AppServerRunScheduledTask***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Lance immédiatement l'exécution d'une tâche planifiée sur un Serveur d'Application WEBDEV. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Programme la tâche pour s'exécuter toutes les heures
MaTâchePlanifiée est un serveurAppliTâchePlanifiée
MaTâchePlanifiée.Nom = "MaTache"
MaTâchePlanifiée.Procédure = MaProc
MaTâchePlanifiée.Mois = "*"
MaTâchePlanifiée.JourDuMois = "*"
MaTâchePlanifiée.JourDeLaSemaine = "*"
MaTâchePlanifiée.Heure = "*"
MaTâchePlanifiée.Minute = "0"

// Lancement immédiat de la tâche
ServeurAppliExécuteTâchePlanifiée(MaTâchePlanifiée)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = ServeurAppliExécuteTâchePlanifiée(<Tâche planifiée>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'exécution de la tâche planifiée a été lancée. 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.




**`<Tâche planifiée> : (Type associé)`**

Tâche planifiée à exécuter. Ce paramètre peut correspondre : 

- à une variable de type [serveurAppliTâchePlanifiée](../WDLang2/1000022367.md). Dans ce cas, seul le nom de la tâche est pris en compte. Les éventuels paramètres de la tâche sont ceux donnés lors de la création de la tâche avec la fonction [ServeurAppliAjouteTâchePlanifiée](../WDLang2/1000022347.md).

- au nom d'une procédure. La tâche planifiée associée au nom de cette procédure sera exécutée. 

- au nom d'une tâche planifiée. La tâche planifiée correspondante sera exécutée. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La tâche est exécutée dans une session indépendante (comme toutes les autres tâches planifiées).

- La tâche n'est pas exécutée si elle est déjà en cours d'exécution.

- La tâche est exécutée même si la tâche est désactivée.

- La tâche planifiée n'est pas exécutable lors d'un GO du projet. Il est nécessaire de déployer le site. 

- Si une erreur se produit durant le lancement ou l'exécution de la tâche, l'erreur ne sera pas remontée par la fonction **ServeurAppliExécuteTâchePlanifiée**. Il est nécessaire de gérer l'erreur dans le traitement, par exemple dans un fichier de LOG.

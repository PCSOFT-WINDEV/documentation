


## ZoneRépétéeNumEnr (Fonction)

***En anglais : LooperRecNum***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le numéro de l'enregistrement en cours dans un champ Zone répétée fichier.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Numéro d'enregistrement en cours dans le champ Zone répétée "ZR_ZoneProduit"
ResNumEnr = ZoneRépétéeNumEnr(ZR_ZoneProduit)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = ZoneRépétéeNumEnr(<Champ Zone répétée>)`
---

**`<Résultat> : (Entier)`**



- Numéro de l'enregistrement en cours dans le fichier de données lié au champ Zone répétée, 

- 0 s'il n'y a pas de répétition en cours ou si la répétition en cours n'est pas encore enregistrée dans le fichier de données lié au champ Zone répétée.




**`<Champ Zone répétée> : (Nom de champ)`**

Nom du champ Zone répétée à manipuler.
Si ce paramètre correspond à une chaîne vide (""), le champ Zone répétée manipulé est le champ Zone répétée auquel appartient l'événement en cours.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Champ Zone répétée basée sur une requête
<a name="champ_zone_repetee_basee_sur_une_requete_ELTPARAGRAPHE000063"></a>Si la requête a été exécutée avec la constante *hAvecFiltre* (option cochée dans l'éditeur de requêtes), la fonction **ZoneRépétéeNumEnr** renvoie le numéro de l'enregistrement en cours dans la requête.

Si la requête a été exécutée sans la constante *hAvecFiltre*, la fonction **ZoneRépétéeNumEnr** renvoie un résultat indéterminé.

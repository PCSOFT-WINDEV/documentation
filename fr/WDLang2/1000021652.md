


## &lt;Cellule&gt;.AfficheDialogue (Fonction)

***En anglais : &lt;Cell&gt;.DisplayDialog***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Affiche un champ Cellule dans la page avec un effet de GFI (Grisage des Fenêtres Inaccessibles). Cette fonction permet de simuler simplement une boîte de dialogue en code navigateur, en utilisant un champ Cellule dans une page.

Pour plus de détails sur les éléments nécessaires et les différents modes de dialogue, consultez [Dialoguer avec l'internaute](../WDChamp/1012026.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Affiche un champ Cellule contenant une question
CELL_Question.AfficheDialogue(celCentre, Vrai, 50)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Champ Cellule>.AfficheDialogue([<Position d'affichage> [, <Prise en compte du GFI> [, <Taux de grisage>]]])`
---

**`<Champ Cellule> : (Nom de champ)`**

Nom du champ Cellule qui sera affiché.

**`<Position d'affichage> : (Constante de type Entier optionnelle)`**

Position d'affichage de la cellule :


|   |   |
| --- | --- |
| *celBasDroite* | Le champ Cellule est placé en bas à droite de la partie visible de la page courante. |
| *celBasGauche* | Le champ Cellule est placé en bas à gauche de la partie visible de la page courante. |
| *celCentre*<br>(Valeur par défaut) | Le champ Cellule est centré dans la partie visible de la page courante. |
| *celHautDroite* | Le champ Cellule est placé en haut à droite de la partie visible de la page courante. |
| *celHautGauche* | Le champ Cellule est placé en haut à gauche de la partie visible de la page courante. |
| *celPositionXY* | Le champ Cellule est laissé à la position courante. |



**`<Prise en compte du GFI> : (Booléen optionnel)`**



- <u><u><u><u>Vrai</u></u></u></u> (valeur par défaut) pour afficher le champ Cellule avec l'effet de GFI (la page affichée derrière le champ Cellule est alors grisée), 

- <u><u><u><u>Faux</u></u></u></u> pour ne pas utiliser l'effet de GFI.




**`<Taux de grisage> : (Entier)`**

Taux de grisage (compris entre 0 et 100) utilisé pour le GFI. 
Si une valeur supérieure à 100 est donnée, la valeur 100 sera utilisée. 

Si une valeur inférieure à 0 est donnée, le GFI n'est pas pris en compte. Le taux de grisage par défaut est 18%.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Caractéristiques du champ Cellule
<a name="caracteristiques_champ_cellule_ELTPARAGRAPHE000222"></a>

- Le champ Cellule doit être superposable (onglet "Détail" de la fenêtre de description du champ Cellule).

- Pour ne pas afficher d'ascenseurs inutiles dans le navigateur, cochez l'option "Positionnement automatique au chargement pour éviter les ascenseurs" (onglet "Détail" de la fenêtre de description du champ Cellule). Cette option permet de positionner automatiquement le champ Cellule à la position (0,0) au chargement de la page. La position d'affichage déterminée par la fonction **&lt;Cellule&gt;.AfficheDialogue** est ensuite prise en compte.




**Remarque** : Pour exécuter un code serveur depuis un champ Cellule, utilisez les fonctions [AJAXExécute](../WDLang2/3055114.md) ou [AJAXExécuteAsynchrone](../WDLang2/3055115.md).
<a name="NOTE0_2"></a>


### Cas particuliers
<a name="cas_particuliers_ELTPARAGRAPHE000248"></a>

- Pour que le champ Cellule soit affiché, il faut utiliser la fonction **&lt;Cellule&gt;.AfficheDialogue** :

	- soit dans un bouton de type "Aucune". Dans ce cas, seul le code navigateur du bouton sera exécuté.

	- soit dans un bouton de type "Submit" et activer AJAX dans le code de ce bouton. Dans ce cas, les codes navigateur et serveur du bouton seront exécutés.




- Le champ Cellule affiché pourra être masqué avec la fonction [&lt;Cellule&gt;.FermeDialogue](../WDLang2/1000021653.md).

- Les navigateurs ne rafraîchissent pas l'affichage au milieu d'un code Javascript : les fonctions **&lt;Cellule&gt;.AfficheDialogue** et [&lt;Cellule&gt;.FermeDialogue](../WDLang2/1000021653.md) doivent être utilisées dans des événements/traitements différents. Si ces fonctions sont utilisées dans le même événement/traitement, le champ Cellule n'est pas affiché.

- Il n'est pas possible d'afficher deux champs Cellule différents en même temps avec la fonction **&lt;Cellule&gt;.AfficheDialogue**.




## ZoneRépétéeTrie (Fonction)

***En anglais : LooperSort***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de :

- Trier un champ Zone répétée selon un ou plusieurs de ses attributs.

- Trier un champ Zone répétée sur tous ses attributs.

- Annuler un tri précédemment effectué (c'est-à-dire ne plus tenir compte du tri).













<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Tri selon un des attributs du champ Zone répétée choisi dans un champ Sélecteur
SELON SEL_SELECT1
	CAS 1: ZoneRépétéeTrie("NOM")
	CAS 2: ZoneRépétéeTrie("SOCIETE")
	CAS 3: ZoneRépétéeTrie("VILLE")
FIN
```
<a name="Exemple2"></a>

```wl
// Tri par nom par ordre croissant et par date par ordre décroissant
ZoneRépétéeTrie("NOM" + TAB + "-DATE")
```
<a name="Exemple3"></a>

```wl
// Tri du champ Zone répétée ZR_Client par ordre décroissant sur tous ses attributs
ZoneRépétéeTrie("-ZR_Client")
```
<a name="Exemple4"></a>

```wl
// Annule le tri précédemment effectué sur le champ Zone répétée ZR_Client
ZoneRépétéeTrie("ZR_Client", Faux)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Trier un champ Zone répétée selon une liste d'attributs 

`<Résultat> = ZoneRépétéeTrie(<Liste d'attributs>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le tri a pu s'exécuter, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Liste d'attributs> : (Chaîne de caractères)`**

Nom des attributs selon lesquels le champ Zone répétée doit être trié (jusqu'à 10 attributs maximum). Cette chaîne de caractères doit être de la forme :

```txt
[<Sens>]<Attribut 1> + TAB + [<Sens>]<Attribut 2> + TAB + ...
```
avec :

- &lt;Sens&gt; : sens du tri (optionnel), peut prendre les valeurs suivantes :

	- "+" : le tri sera croissant (valeur par défaut).

	- "-" : le tri sera décroissant.




- &lt;Attribut&gt; : nom de l'attribut utilisé pour le tri.





<a name="SYNTAXE2"></a>

### Trier ou annuler le tri d'un champ Zone répétée

`<Résultat> = ZoneRépétéeTrie([<Sens>]<Zone répétée> [, <Indicateur>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le tri a pu s'exécuter ou être annulé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<[]> : (Chaîne de caractères)`**

Nom du champ Zone répétée à trier. 

- Si &lt;Sens&gt; est égal à "+", le champ Zone répétée sera trié dans l'ordre croissant (par défaut).

- Si &lt;Sens&gt; est égal à "-", le champ Zone répétée sera trié dans l'ordre décroissant.




**`<Indicateur> : (Booléen optionnel)`**



- <u><u><u><u>Vrai</u></u></u></u> pour effectuer le tri (valeur par défaut), 

- <u><u><u><u>Faux</u></u></u></u> pour annuler un tri précédemment effectué.





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000161"></a>La fonction **ZoneRépétéeTrie** peut être utilisée indifféremment sur : 

- un champ Zone répétée fichier,

- un champ Zone répétée par programmation. 



<a name="NOTE0_2"></a>


### Annulation du tri
<a name="annulation_tri_ELTPARAGRAPHE000170"></a>

- Le tri n'est pas annulé par les fonctions [ZoneRépétéeSupprime](../WDLang2/3083008.md), [ZoneRépétéeSupprimeTout](../WDLang2/3083012.md), [ZoneRépétéeCherche](../WDLang2/3083005.md) et [ZoneRépétéeOccurrence](../WDLang2/3083014.md).

- Si un attribut est modifié par la fonction [ZoneRépétéeModifie](../WDLang2/3083013.md) et si cet attribut sert d'attribut de tri au champ Zone répétée, le tri sur cet attribut est annulé.



<a name="NOTE0_3"></a>


### Tri
<a name="tri_ELTPARAGRAPHE000195"></a>Le prochain ajout d'une répétition dans le champ Zone répétée (fonctions [ZoneRépétéeAjoute](../WDLang2/3083006.md), [ZoneRépétéeAjouteLigne](../WDLang2/3083004.md), [ZoneRépétéeInsère](../WDLang2/3083003.md) et [ZoneRépétéeInsèreLigne](../WDLang2/3083002.md)) tient compte du tri.
<a name="NOTE0_4"></a>


### Zone répétée liée à une source de données et utilisant un parcours programmé
<a name="zone_repetee_liee_une_source_donnees_utilisant_parcours_programme_ELTPARAGRAPHE000216"></a>La fonction **ZoneRépétéeTrie** ne fonctionne pas sur un champ Zone répétée ayant les caractéristiques suivantes : 

- champ Zone répétée lié à une source de données, 

- champ Zone répétée utilisant l'option **Parcours programmé** (onglet "Contenu" de la fenêtre de description du champ). 




Pour changer l'ordre d'affichage d'un champ Zone répétée avec **parcours programmé**, il est nécessaire d'adapter le code se trouvant dans les différents traitements de lecture du champ Zone répétée (traitements lecture du premier, du suivant, du précédent, ...).

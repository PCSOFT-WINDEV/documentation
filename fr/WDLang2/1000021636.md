


## PageEcritEntêteHTTP (Fonction)

***En anglais : PageWriteHeaderHTTP***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute un entête HTTP supplémentaire qui sera retourné au navigateur : 

- lors de la prochaine fonction d'affichage de la page ([PageAffiche](../WDLang2/3058008.md), [PageActualise](../WDLang2/3058011.md), ...),

- lors du renvoi automatique de la page courante à la fin de l'action en cours.



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
PageEcritEntêteHTTP("X-Frame-Options", "deny")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`PageEcritEntêteHTTP(<Entête> , <Valeur>)`
---

**`<Entête> : (Chaîne de caractères)`**

Nom de l'entête à écrire.

**`<Valeur> : (Chaîne de caractères)`**

Valeur (ou contenu) de l'entête à écrire.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le serveur d'application WEBDEV force la valeur de certains entêtes HTTP renvoyés par le serveur au client (par exemple "Content-Length"). Dans ce cas, la valeur écrite par la fonction **PageEcritEntêteHTTP** sera ignorée.
	La liste des entêtes forcés est la suivante : 

	- Content-xxx (content-length, content-type, content-disposition, ...)

	- Location

	- Allow (cas spécifiques)

	- Set-Cookie

	- Access-Control-Expose-Headers




- Un second appel à la fonction **PageEcritEntêteHTTP** avec un même nom d'entête écrasera la précédente valeur.




## PagePrécédente (Fonction)

***En anglais : PreviousPage***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le nom de la page précédente. La page précédente est la page ayant ouvert la page en cours.

En mode AWP, la fonction **PagePrécédente** renvoie toujours une chaîne vide ("").

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Ch est une chaîne
Ch = PagePrécédente()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = PagePrécédente()`
---

**`<Résultat> : (Chaîne de caractères)`**

Nom en majuscules de la page précédente si le code WLangage en cours d'exécution est :

- un code de page (code d'ouverture, de fermeture, de synchronisation)

- un code de champ (initialisation, clic)

- une procédure locale de la page




Ou chaîne vide ("") si la page est une page statique ou si le code en cours d'exécution est :

- le code de projet.

- un code constructeur ou destructeur de classe.

- une méthode d'une classe.

- une procédure globale au projet.


![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) La casse du nom de la page correspond à la casse réelle du nom de la page. Par exemple, si la page précédente correspond à la page "Accueil.WWH", le nom retourné sera "Accueil".



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La fonction **PagePrécédente** peut être utilisée pour gérer l'affichage d'un bouton de type RETOUR dans une page WEBDEV. Il suffit d'indiquer la ligne suivante dans le code de clic serveur du bouton :



```wl
PageAffiche(PagePrécédente())
```


- Pages référençables : Si vous utilisez des pages référençables en mode Session, la fonction **PagePrécédente** ne doit pas être utilisée. En effet, étant donné qu'il est possible d'accéder à la page référençable depuis n'importe quelle page, la page précédente n'existera pas dans ce cas.

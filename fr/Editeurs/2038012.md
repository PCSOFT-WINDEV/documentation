


## Opérations de synchronisation du GDS
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000100"></a>
Lors de la création d'un exécutable, d'une bibliothèque ou d'un composant, le GDS propose de réintégrer et de récupérer la dernière version des éléments présents dans le GDS. L'exécutable, le composant ou la bibliothèque peut ainsi contenir les derniers éléments les plus à jour.
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GDS_Exe.gif)
Les différentes options proposées sont les suivantes :

- **Réintégrer les éléments extraits** : Cette option permet de réintégrer dans le GDS tous les éléments extraits par le développeur en cours. Les éléments sont ainsi mis à disposition de tous les autres développeurs. Cette réintégration est effectuée après la création. Cette option est conseillée.

- **Récupérer la dernière version des éléments du projet** : Cette option permet de récupérer en local la dernière version de tous les éléments du projet. La récupération est effectuée avant la création de l'élément voulu.
	**Attention** : Cette option peut provoquer des erreurs de compilation ou des différences de fonctionnement. Il est conseillé de tester le projet avant de générer l'élément voulu.

- **Récupérer la dernière version des éléments du projet et réintégrer les éléments extraits** : Cette option permet de :

	- réintégrer dans le GDS tous les éléments extraits par le développeur en cours. Les éléments sont ainsi mis à disposition de tous les autres développeurs. Cette réintégration est effectuée après la création.

	- récupérer en local la dernière version de tous les éléments du projet. La récupération est effectuée avant la création de l'élément voulu.
			**Attention** : Cette option peut provoquer des erreurs de compilation ou des différences de fonctionnement. Il est conseillé de tester le projet avant de générer l'élément voulu.




- **Ne rien faire** :  Cette option permet de créer l'élément voulu (exécutable, bibliothèque, composant, ...) avec les éléments du projet présents en local. Aucune réintégration ou synchronisation n'est effectuée.




Par défaut, la génération de l'exécutable, de la bibliothèque ou du composant se fait avec les éléments du projet actuellement présents sur le poste local.

## Mise en place
<a name="mise_place_ELTTEXTE000124"></a>
Par défaut, la synchronisation est automatiquement proposée lors de la création de l'exécutable.

Si vous cochez le choix "Toujours effectuer ce choix sans me poser la question", il est possible de ré-activer cette fenêtre depuis la fenêtre de description du projet :

1. Affichez la fenêtre de description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".

2. Affichez l'onglet "GDS".

3. Sélectionnez l'option "Proposer la réintégration et la récupération de la dernière version des éléments lors de la génération d'exécutables, de bibliothèques, de composants, ...".

4. Validez.

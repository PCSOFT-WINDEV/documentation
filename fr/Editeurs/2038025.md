
## GDS : Migration de projet
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000087"></a>
Lors d'un changement de version de l'outil de développement, les projets présents dans le GDS doivent être migrés vers cette nouvelle version.

Plusieurs cas peuvent se présenter :

- Le projet n'a pas été migré dans le GDS.

- Le projet a déjà été migré dans le GDS.






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000111"></a>
Si vous tentez d'ouvrir un projet présent dans le GDS et réalisé avec une version précédente de votre outil de développement, un écran spécifique s'affiche.

Cet écran permet de :

- **Conserver la connexion actuelle pour le projet migré**. Dans ce cas, tous les éléments du projet vont être extraits sur votre poste, migrés et réintégrés dans le GDS. Une sauvegarde du projet dans l'ancienne version va être effectuée sur votre poste.
	Il est possible si nécessaire de créer une branche dans le GDS avec le projet conservé dans la version précédente.
	**Attention** : Cette opération doit être réalisée par un seul développeur et peut être relativement longue.
	Lorsque le projet sera migré et réintégré dans le GDS, il pourra être utilisé dans cette version par tous les développeurs.

- **Récupérer le projet dans la base qui est déjà au format supérieur**. Cette option est disponible si le projet a déjà été migré dans la base de sources. Dans ce cas, il est en effet inutile de migrer et recompiler le projet local avec la nouvelle version : il suffit de récupérer tous les éléments du projet présents dans la base de sources.

- **Déconnecter le projet du GDS**. Dans ce cas, le projet ne sera plus relié au GDS et sera migré uniquement en local. Cette option peut être utilisée par exemple pour tester le passage du projet dans une version supérieure. Ce projet migré pourra être ajouté ultérieurement dans la base de sources du GDS.




**Rappel** : La migration peut être effectuée uniquement par un poste maître.

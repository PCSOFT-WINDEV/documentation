


## Installation par média physique autonome
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000135"></a>
Une installation de site par média physique autonome permet de fournir un support contenant tous les fichiers nécessaires à l'installation et à l'exécution d'un site. Les éléments suivants sont fournis :

- Les fichiers du site.

- Le serveur d'application WEBDEV gratuit 10 connexions.

- Un serveur Web (sous forme d'une installation MSI). Le serveur sera automatiquement installé et configuré.




**Ce mode d'installation est idéal pour une démonstration d'un site sur un portable par exemple**.

Un assistant permet à la personne déployant le site de paramétrer les différentes options d'installation.

Pour installer votre site dynamique WEBDEV, il est nécessaire de lancer le programme exécutable **Installe.exe** de votre installation.

## Comment le faire ?
<a name="comment_faire_ELTTEXTE000159"></a>


### Pour créer une installation par média physique
<a name="pour_creer_une_installation_par_media_physique_ELTPARAGRAPHE000028"></a>

1. Sous le volet "Projet", dans le groupe "Génération", déroulez "Déployer le site" et sélectionnez "Générer et déployer le site".

2. Suivez l'assistant pour générer la bibliothèque. 

3. Sélectionnez le modèle "Créer une installation par média physique".

4. [Sélectionnez les fichiers à installer](../Editeurs/2028058.md).

5. [Paramétrez la modification automatique](../Editeurs/2028048.md). 

6. Indiquez si la mise à jour doit être effectuée avec ou sans déconnexion des utilisateurs. 

7. [Indiquez les paramètres du site](../Editeurs/2028051.md). 

8. Indiquez qu'un serveur Web doit être inclus dans l'installation (option "Inclure un serveur Web dans l'installation"). Sélectionnez le serveur Web à installer. Par défaut, l'installation du serveur Apache 2 est livrée au format MSI avec WEBDEV.

9. Sélectionnez les informations concernant la personnalisation de l'installation :

	- **L'outil d'installation utilisé** : Par défaut, WBSetup est proposé. WBSetup est un assistant d'installation de sites WEBDEV complet. Ce programme d'installation est proposé en standard en Français et Anglais. Pour que le programme d'installation gère des langues supplémentaires, il est nécessaire de personnaliser le projet "WBSetup" avant de créer le programme d'installation. Ce projet (fourni avec les exemples de WEBDEV) correspond au programme d'installation standard des applications WEBDEV. Ce projet est un projet WINDEV. Pour plus de détails, consultez [Utiliser WBSetup personnalisé](../Editeurs/2028096.md).

	- **Le répertoire d'installation par défaut du site** : Ce répertoire correspond au répertoire où seront installés les fichiers du site (bibliothèques, pages, feuilles de styles, images, ...). Par défaut, le chemin proposé est "&lt;site&gt;\\&lt;Nom du projet&gt;" où &lt;site&gt; correspond au répertoire associé aux différents sites du compte WEBDEV auquel le site sera associé. Il est conseillé d'utiliser des pseudo-répertoires. Ce répertoire pourra être modifié lors de l'installation du site.

	- **Le répertoire d'installation par défaut des données du site** : Ce répertoire correspond au répertoire où seront installés les fichiers de données du site. Par défaut, le chemin proposé est "&lt;data&gt;\\&lt;Nom du projet&gt;" où &lt;data&gt; correspond au répertoire des données associé aux différents sites du compte WEBDEV auquel le site sera associé. Il est conseillé d'utiliser des pseudo-répertoires. Ce répertoire pourra être modifié lors de l'installation du site.
			Les pseudo-répertoires utilisables sont :

		- ***&lt;webroot&gt;*** : Répertoire racine des documents du serveur Web

		- ***&lt;site&gt;*** : Répertoire de base des sites pour le compte utilisateur WEBDEV définit par l'hébergeur. Si un compte vous est attribué, tous vos sites dynamiques seront installés dans ce répertoire.

		- ***&lt;data&gt;*** : Répertoire de base des fichiers de données pour le compte utilisateur WEBDEV définit par l'hébergeur. Si un compte vous est attribué, tous les fichiers de données de vos sites dynamiques seront installés dans ce répertoire.




10. Indiquez les langues gérées par l'installation, ainsi que les paramètres correspondants à chaque langue.
	Par défaut, WBSetup gère le Français et l'anglais. Pour gérer des langues supplémentaires, il est conseillé de personnaliser WBSetup. Pour plus de détails, consultez [Utiliser WBSetup personnalisé](../Editeurs/2028096.md).
	Pour chaque langue proposée lors de l'installation, indiquez :

	- **Le libellé d'introduction**. Ce libellé permet de lancer l'installation dans la langue voulue. Par défaut, il correspond à la langue d'installation.

	- **La licence de votre site**. La licence doit être un fichier au format txt uniquement, elle s'affichera au lancement de l'installation de votre site. Pour que le texte soit dans la langue choisie au début de l'installation, créez un fichier de licence pour chacune des langues. L'installation ne pourra s'effectuer qu'après validation de la licence par la personne chargée de l'installation de votre site dynamique WEBDEV

	- **Lisez-moi de votre site**. Le fichier "Lisez-moi" peut être au format : txt, htm, doc, hlp, pdf 
			Quel que soit le format choisi, l'utilisateur doit posséder un logiciel permettant de visualiser ce fichier. Pour que le fichier "Lisez-moi" soit dans la langue choisie au début de l'installation, créez un fichier "Lisez-moi" pour chacune des langues.




11. Indiquez le répertoire de génération de l'installation. Il est possible de vider ou non le contenu du répertoire lors de la génération.

12. Validez. Le programme d'installation du site est créé. Il est possible de :

	- déployer le site (par exemple pour tester l'installation)

	- ouvrir l'explorateur (par exemple pour copier le répertoire d'installation sur un CD).

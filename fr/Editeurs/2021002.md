


## Manipuler les versions d'une fenêtre, d'un état, d'une requête, d'un projet 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000202"></a>



|   |
| --- |
| **Avertissement de compatibilité** |
| Cette fonctionnalité est conservée par compatibilité avec les anciennes versions.<br><br>Il est recommandé d'utiliser le GDS pour gérer les différentes versions de l'historique des éléments d'un projet.<br><br>Pour plus d'informations consultez : "[Présentation du GDS](../Editeurs/2038001.md)" et "[Configurer un projet pour le GDS](../Editeurs/2038002.md)" . |

Lors de la création et du développement des différents éléments d'un projet (fenêtre, requête, état, projet, ...), il est possible de conserver plusieurs versions d'un élément. Il est ainsi possible d'enregistrer une version avant de réaliser d'importantes modifications dans l'élément (par exemple le remaniement de l'interface d'une fenêtre).

A tout moment il est possible de créer un nouvel élément à partir d'une version précédemment enregistrée.

Ainsi, pour une fenêtre, plus besoin d'enregistrer la même fenêtre sous des noms différents selon la version de la fenêtre : l'éditeur de fenêtres gère automatiquement les différentes versions et les intègre dans le fichier d'enregistrement de la fenêtre. Une fenêtre correspond à un fichier ".WDW". Ce fichier contient la description de la fenêtre et l'ensemble des versions enregistrées pour cette fenêtre. Plus le nombre de versions est important, plus la taille du fichier ".WDW" sera importante.

Le même principe est appliqué pour les requêtes, les pages, les états et le projet (fichier WDP, WWP ou WPP selon le produit utilisé).

A tout moment, il est possible de :

- Enregistrer une nouvelle version de l'élément en cours.

- Ouvrir une version de l'élément en cours.

- Supprimer une version de l'élément en cours.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000226"></a>


### Enregistrer une version
<a name="enregistrer_une_version_ELTPARAGRAPHE000045"></a>L'enregistrement d'une version permet de conserver une sauvegarde à un moment donné.

Pour enregistrer une version de l'élément en cours :

1. Sous le volet "Accueil", dans le groupe "Général", déroulez "Enregistrer" et sélectionnez "Versions". La fenêtre des versions de l'élément en cours s'affiche.

2. Cliquez sur le bouton "Nouvelle". Une fenêtre s'ouvre, permettant de saisir un commentaire personnalisé sur la version qui va être créée.

3. Validez. La nouvelle version est enregistrée et l'élément en cours est automatiquement sauvegardé.




**Remarque** : Les différentes versions d'un élément sont listées par ordre décroissant (de la plus récente à la plus ancienne) selon leur date et heure d'enregistrement.
<a name="NOTE2_2"></a>


### Ouvrir une version
<a name="ouvrir_une_version_ELTPARAGRAPHE000072"></a>L'ouverture d'une version permet de créer un nouvel élément correspondant à la version sélectionnée.

Pour ouvrir une des versions de l'élément en cours :

1. Sous le volet "Accueil", dans le groupe "Général", déroulez "Enregistrer" et sélectionnez "Versions". La fenêtre des versions de l'élément en cours s'affiche.

2. Sélectionnez la version à ouvrir.

3. Cliquez sur le bouton "Ouvrir". Un nouvel élément est créé sous l'éditeur correspondant. Cet élément correspond à la version sélectionnée. Le nom de ce nouvel élément est au format &lt;Nom de l'élément&gt;&lt;Date et heure de l'enregistrement de la version&gt;.



<a name="NOTE2_3"></a>


### Supprimer une version
<a name="supprimer_une_version_ELTPARAGRAPHE000097"></a>La suppression d'une version inutile permet principalement de diminuer la taille du fichier physique ( ".WDW" pour une fenêtre) correspondant à l'élément en cours.

Pour supprimer une version de l'élément en cours :

1. Sous le volet "Accueil", dans le groupe "Général", déroulez "Enregistrer" et sélectionnez "Versions". La fenêtre des versions de l'élément en cours s'affiche.

2. Sélectionnez la version à supprimer.

3. Cliquez sur le bouton "Supprimer". La version sélectionnée est automatiquement supprimée de la fenêtre de gestion des versions. Cette suppression sera effective au prochain enregistrement de l'élément.

## Notes
<a name="notes_ELTTEXTE000262"></a>


### Ouverture de fichiers après un blocage du système
<a name="ouverture_fichiers_apres_blocage_systeme_ELTPARAGRAPHE000124"></a>Pour éviter tout problème lors du blocage du système (coupure de courant par exemple), l'éditeur propose un système de récupération automatique des éléments ouverts sous l'éditeur : 

- Sous le volet "Accueil", dans le groupe "Environnement", déroulez "Options" et sélectionnez "Options générales de WINDEV/WEBDEV/WINDEV Mobile".

- Sous l'onglet "Général", cochez l'option "Enregistrement pour la récupération automatique toutes les" et indiquez si nécessaire la fréquence voulue.




Lorsque ce système est activé :

- Un enregistrement automatique des éléments ouverts est effectué à une période régulière (toutes les 10 minutes par défaut).

- Lors de la réouverture d'un élément après un blocage du système, une fenêtre de récupération automatique apparaît. Cette fenêtre propose de récupérer :

	- soit l'élément dans son état lors de la dernière sauvegarde réalisée par le développeur,

	- soit l'élément sauvegardé automatiquement par WINDEV, WEBDEV ou WINDEV Mobile.







La version sélectionnée est automatiquement ouverte sous l'éditeur. La version non sélectionnée est disponible dans la fenêtre de gestion des versions et peut être ouverte à tout moment.

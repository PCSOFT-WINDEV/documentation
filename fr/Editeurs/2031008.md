
## Type de pages créées par le RAD
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000439"></a>
L'assistant de création de pages propose de nombreux types de pages associées à des fichiers de données HyperFileSQL. Il est possible de créer des :

- **pages dynamiques** : Ces pages sont créées à partir des fichiers de données présents sur le poste de développement. En déploiement, les données sont installées (ou accessibles) depuis le serveur Web. Les pages affichent les données présentes dans les fichiers de données.

- **pages semi dynamiques** : Ces pages sont créées à partir des fichiers de données présent sur le poste de développement. Avant d'installer l'application, ces pages sont converties en pages statiques : les données présentes dans les fichiers de données sont enregistrées dans les pages. Aucun fichier de données n'est installé sur le serveur.




Pour plus de détails sur les pages dynamiques et semi-dynamiques, consultez [Les différents types de pages](../WDChamp/2030020.md).

Les différents types de pages proposés :



|   |   |
| --- | --- |
| **Pages dynamiques** | **Pages semi-dynamiques** |
| Page fiche :<br><br>- simple. <br><br>- avec parcours. <br><br>- avec table reliée. <br><br>- avec table de relation.<br><br><br> | Page Fiche |
| Page table :<br><br>- simple. <br><br>- avec fiche détail.<br><br>- avec table reliée.<br><br><br> | Page Table |
| Page Vision Plus. |   |
| Page Zone répétée. | Page Zone répétée. |


Les paragraphes suivants présentent pour chaque type de page :

- le fonctionnement de la page,

- les différents champs présents dans la page,

- un aperçu de la page,

- des conseils d'utilisation.

## Page fiche
<a name="page_fiche_ELTTEXTE000463"></a>


### Cas général
<a name="cas_general_ELTPARAGRAPHE000074"></a>Une page **fiche** affiche les valeurs d'un enregistrement dans une fiche, à raison d'une valeur par champ. Les enregistrements sont accessibles un par un.

La fiche peut au choix :

- autoriser le passage ou non aux enregistrements suivants et précédents.

- autoriser ou non la création, la modification et la suppression d'enregistrement.





<a name="NOTE2_2"></a>


### Fiche dynamique simple
<a name="fiche_dynamique_simple_ELTPARAGRAPHE000087"></a>Ce type de page permet de visualiser et de modifier un enregistrement.

Ce type de page ne permet pas :

- le parcours sur les différents enregistrements.

- la création et la suppression d'un enregistrement.




Une page "Fiche dynamique simple" est constituée de :

- boutons pour valider, abandonner la saisie.

- bouton pour imprimer l'enregistrement en cours

- champs reliés aux rubriques du fichier.




La page "Fiche dynamique simple" est généralement accessible depuis une page de type "Table dynamique simple" pour visualiser ou modifier l'enregistrement en cours.


<a name="NOTE2_3"></a>


### Fiche dynamique avec parcours
<a name="fiche_dynamique_avec_parcours_ELTPARAGRAPHE000108"></a>Ce type de page permet :

- la modification de l'enregistrement affiché.

- la création d'un enregistrement.

- la suppression d'un enregistrement.

- l'impression de l'enregistrement en cours.

- le passage à un autre enregistrement (suivant ou précédent).




Une page de type "Fiche dynamique avec parcours" est constituée de :

- champs reliés aux rubriques du fichier.

- 4 boutons de parcours.

- boutons permettant de :

	- créer un nouvel enregistrement.

	- supprimer l'enregistrement en cours.

	- imprimer l'enregistrement en cours.

	- fermer la page.







Par rapport à une "Fiche dynamique simple", une "Fiche dynamique avec parcours" propose :

- le passage rapide d'une fiche à une autre.

- la possibilité de créer en chaîne plusieurs enregistrements.




**Conseil** : Pour les fichiers comportant de nombreux enregistrements, il est préférable de sélectionner les enregistrements dans une table et de modifier ou visualiser l'enregistrement sélectionné dans une page de type "Fiche simple". En effet, la sélection dans une table est plus rapide.


<a name="NOTE2_4"></a>


### Fiche dynamique avec table reliée
<a name="fiche_dynamique_avec_table_reliee_ELTPARAGRAPHE000140"></a>Ce type de page permet de visualiser dans la même page le contenu de fichiers liés par une liaison simple (1 - N).

Les valeurs de l'enregistrement du fichier principal sont visualisées dans une fiche. Les enregistrements reliés à l'enregistrement en cours (dans le fichier principal) sont visualisés dans une table.

Ce type de page est constitué de :

- une fiche (dans la partie haute) qui affiche les valeurs de l'enregistrement en cours du fichier de base (principal).

- une table (dans la partie basse) qui affiche les enregistrements reliés à l'enregistrement en cours du fichier principal.

- une réglette permettant d'afficher les enregistrements de la table.

- boutons permettant de :

	- créer un nouvel enregistrement dans le fichier principal.

	- supprimer l'enregistrement en cours du fichier principal.

	- fermer la page.

	- ajouter une ligne dans la table.

	- modifier une ligne dans la table.

	- supprimer une ligne dans la table.







Remarques :

- Si le fichier affiché dans la table est un fichier de relation, la page comporte les boutons nécessaires pour accéder à l'autre fichier en relation.

- Si ce type de page est sélectionné pour un fichier qui n'a pas de fichier lié (avec une liaison 1-N), WEBDEV le signale.





<a name="NOTE2_5"></a>


### Fiche dynamique avec table de relation
<a name="fiche_dynamique_avec_table_relation_ELTPARAGRAPHE000169"></a>Ce type de page permet de visualiser à la fois :

- le contenu d'un fichier dans une table et

- le contenu d'un fichier dans une fiche.




Les enregistrements affichées dans la table et ceux enregistrées dans la fiche sont liés par une liaison complexe (N - N).

La page "Fiche avec table de relation" est constituée de :

- une fiche (dans la partie haute) qui affiche les enregistrements du fichier de base (principal).

- une table (dans la partie basse) qui affiche les enregistrements reliés à l'enregistrement en cours du fichier principal.

- une réglette permettant d'afficher les enregistrements de la table.

- boutons permettant de :

	- créer un nouvel enregistrement dans le fichier principal.

	- supprimer l'enregistrement en cours du fichier principal.

	- fermer la page.

	- ajouter une ligne dans la table.

	- modifier une ligne dans la table.

	- supprimer une ligne dans la table.







La page "Fiche dynamique avec table de relation" est conseillée lorsque le détail du fichier doit être immédiatement affiché.


<a name="NOTE2_6"></a>


### Fiche semi- dynamique
<a name="fiche_semi_dynamique_ELTPARAGRAPHE000198"></a>Ce type de page permet de visualiser un enregistrement et de parcourir les enregistrements à visualiser.

Ce type de page ne permet pas la création et la suppression des enregistrements. En effet, lors de la génération des pages statiques associées, il y aura une page statique créée par fiche.

Ce type de page est constitué :

- de champs affichant le contenu des rubriques du fichier.

- d'une réglette permettant de parcourir les enregistrements.




La page "Fiche semi-dynamique" est généralement accessible depuis une page de type "Table semi-dynamique" pour visualiser l'enregistrement en cours.

## Page table
<a name="page_table_ELTTEXTE000517"></a>


### Cas général
<a name="cas_general_ELTPARAGRAPHE000217"></a>Une page **table** affiche les enregistrements sous forme d'un tableau (lignes et colonnes). Plusieurs enregistrements du fichier peuvent être affichés dans la page.

Les rubriques d'un même enregistrement sont disposées en ligne.


<a name="NOTE3_2"></a>


### Table dynamique simple
<a name="table_dynamique_simple_ELTPARAGRAPHE000227"></a>Ce type de page permet :

- la visualisation des enregistrements dans la table.

- la création d'un enregistrement (dans une page de type fiche)

- la suppression d'un enregistrement.

- la modification d'un enregistrement (dans une page de type fiche)

- l'impression du fichier.




Une page de type "Table dynamique simple" est constituée de :

- une table qui affiche les enregistrements du fichier lié.

- des champs de recherche rapide permettant d'accéder à un des enregistrements de la table.

- une réglette permettant de parcourir les enregistrements de la table.

- des boutons permettant de :

	- créer un nouvel enregistrement.

	- modifier un enregistrement.

	- supprimer un enregistrement.

	- imprimer les enregistrements.

	- fermer la page.







Ce type de page est conseillé dans tous les cas de gestion de fichiers <u><u><u><u>sauf</u></u></u></u> :

- si le détail de l'enregistrement doit être visible immédiatement : préférez la page dynamique avec table et fiche de détail.

- si des rubriques doivent être affichées dans des champs multilignes : préférez la page dynamique avec table et fiche de détail.

- si le fichier contient des images à afficher immédiatement : préférez la page dynamique avec table et fiche de détail.

- si le fichier a beaucoup de rubriques à afficher : préférez la page dynamique avec table et fiche de détail.

- si des rubriques reliées doivent être affichées : préférez la page dynamique fiche avec table reliée.





<a name="NOTE3_3"></a>


### Table dynamique avec fiche détail
<a name="table_dynamique_avec_fiche_detail_ELTPARAGRAPHE000266"></a>Ce type de page permet de visualiser dans la même page le contenu d'un fichier dans une table et dans une fiche. Cette page permet la sélection d'un enregistrement dans la table et la visualisation du détail de l'enregistrement sélectionné dans une fiche.

La page "Table dynamique avec fiche détail" est constituée de :

- une table (dans la partie haute) qui affiche les enregistrements du fichier.

- des champs de recherche rapide pour accéder à un enregistrement de la table

- une fiche (dans la partie basse) qui affiche les valeurs de la ligne sélectionnée dans la table

- une réglette permettant de parcourir les enregistrements de la table.

- plusieurs boutons permettant de :

	- créer un nouvel enregistrement.

	- modifier l'enregistrement en cours.

	- supprimer l'enregistrement en cours.

	- fermer la page.







La page "Table dynamique avec fiche détail" est conseillée lorsque :

- le fichier contient des rubriques images ou multilignes à afficher.

- le détail de l'enregistrement doit être visible immédiatement.

- le fichier a beaucoup de rubriques à afficher.





<a name="NOTE3_4"></a>


### Table dynamique avec table reliée
<a name="table_dynamique_avec_table_reliee_ELTPARAGRAPHE000293"></a>Ce type de page permet d'afficher dans une même page le contenu de fichiers liés par une liaison simple ou complexe. Les tables sont en cascade : la seconde table permet d'afficher les enregistrements reliés à la ligne sélectionné dans la première table.

La page "Table dynamique avec table reliée" est constituée de :

- une table (dans la partie haute) qui affiche les enregistrements du fichier de base.

- une table (dans la partie basse) qui affiche les enregistrements du fichier relié, correspondant à l'enregistrement en cours sélectionné dans la première table.

- deux réglettes permettant de parcourir les enregistrements des tables.

- boutons permettant de :

	- créer un nouvel enregistrement dans le fichier principal.

	- modifier l'enregistrement en cours du fichier principal.

	- supprimer l'enregistrement en cours du fichier principal.

	- fermer la page.

	- ajouter une ligne dans le fichier relié.

	- modifier une ligne dans le fichier relié.

	- supprimer une ligne dans le fichier relié.








<a name="NOTE3_5"></a>


### Table semi-dynamique
<a name="table_semidynamique_ELTPARAGRAPHE000316"></a>Ce type de page permet la visualisation des enregistrements dans la table.

La page "Table semi-dynamique" est constituée de :

- une table qui affiche les enregistrements du fichier lié.

- une réglette permettant de parcourir les enregistrements de la table.




La page "Table semi-dynamique" permet de consulter le détail des enregistrements par un simple clic sur l'enregistrement. Une page de type "Fiche semi-dynamique" est alors affichée.

## Page Zone répétée
<a name="page_zone_repetee_ELTTEXTE000565"></a>


### Cas général
<a name="cas_general_ELTPARAGRAPHE000333"></a>Une page ***Zone répétée*** affiche les enregistrements sous forme de fiche répétée (dans un champ de type Zone répétée). Plusieurs enregistrements peuvent être affichés dans la page


<a name="NOTE4_2"></a>


### Zone répétée dynamique
<a name="zone_repetee_dynamique_ELTPARAGRAPHE000341"></a>Ce type de page permet :

- la visualisation des enregistrements dans une zone répétée fichier.

- la création d'un enregistrement (dans une page de type fiche)

- la suppression d'un enregistrement.

- la modification d'un enregistrement (dans une page de type fiche)




La page "Zone répétée dynamique" est constituée de :

- une zone répétée qui affiche les enregistrements du fichier de base.

- des champs de recherche rapide pour accéder à un enregistrement de la zone répétée.

- une réglette permettant de parcourir les enregistrements de la zone répétée.

- boutons permettant de :

	- créer un nouvel enregistrement

	- modifier un enregistrement

	- supprimer un enregistrement

	- Imprimer un enregistrement

	- fermer la page.








<a name="NOTE4_3"></a>


### Zone répétée semi-dynamique
<a name="zone_repetee_semidynamique_ELTPARAGRAPHE000367"></a>Ce type de page permet de visualiser des enregistrements dans une zone répétée Fichier.

La page "Zone répétée semi dynamique" est constituée de :

- une zone répétée qui affiche les enregistrements du fichier de base.

- un bouton permettant d'afficher le détail de l'enregistrement en cours

- une réglette permettant de parcourir les enregistrements de la zone répétée.

## Page Vision Plus
<a name="page_vision_plus_ELTTEXTE000601"></a>
Les pages Vision Plus permettent de sélectionner un enregistrement dans une table. Par exemple, lors de la saisie d'une commande, des pages Vision Plus peuvent être ouvertes pour sélectionner le client et le produit.

Les pages Vision Plus sont appelées par le bouton qui a pour libellé [...] dans les pages reliées à un fichier.

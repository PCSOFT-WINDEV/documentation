


## Suivi strict d'un projet
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000120"></a>
Grâce au Centre de Suivi de Projets, WINDEV, WEBDEV et WINDEV Mobile permettent de définir les différentes actions nécessaires à la réalisation d'un projet :

- Gestion des exigences,

- Gestion des tâches,

- Gestion des bugs,

- Gestion des règles métier.




Ces différents éléments permettent de planifier et de suivre l'évolution et la réalisation d'un projet.

Pendant le développement, l'intervenant peut indiquer la tâche actuellement en cours (par exemple depuis le Centre de Suivi de Projets).

**Avec le mode "strict", les développeurs ne peuvent pas modifier un élément du projet (fenêtre, code, requête, ...) sans avoir préalablement indiqué la tâche concernée par ces modifications.**

Le mode strict permet aux chefs de projet et aux développeurs d'avoir une traçabilité parfaite de leur travail. De plus, le taux d'avancement des exigences est en permanence le plus proche de la réalité : l'état d'avancement d'un projet est toujours connu.

## Comment le faire ?
<a name="comment_faire_ELTTEXTE000144"></a>


### Activer le mode strict
<a name="activer_mode_strict_ELTPARAGRAPHE000031"></a>Pour activer le mode strict :

1. Affichez la description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".

2. Affichez : 

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) l'onglet "Options".




3. Cochez l'option "Activer le mode strict".

4. Validez.



<a name="NOTE2_2"></a>


### Fonctionnement du mode strict
<a name="fonctionnement_mode_strict_ELTPARAGRAPHE000062"></a>Lorsque le mode strict est activé, si le développeur ouvre un élément du projet qui n'est pas associé à la tâche en cours, cet élément est ouvert en lecture seule et un panneau spécifique est affiché au développeur : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=mode_strict.gif)


En cliquant sur le bouton "Gestion des tâches", le développeur peut choisir de :

- associer la tâche courante à l'élément,

- activer une des tâches associées à l'élément,

- associer une nouvelle tâche à l'élément et l'activer,

- associer une tâche existante à l'élément et l'activer,

- ne rien faire. Dans ce cas, l'élément ne pourra pas être modifié.

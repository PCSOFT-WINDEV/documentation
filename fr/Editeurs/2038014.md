
## Travailler en mode Nomade
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000081"></a>
Lors d'un travail sur un portable, ou un poste temporairement déconnecté du réseau, il peut être intéressant de continuer à travailler :

- sur les données des Centres de Contrôle (suivi du planning, suivi de la qualité, ...).

- sur les projets partagés dans la base de sources du GDS.




WINDEV, WEBDEV et WINDEV Mobile permettent de gérer un mode "Nomade" pour les Centres de Contrôle et pour les projets du GDS. Ce mode permet :

- **pour les Centres de Contrôle** : de se déconnecter de la base de données des Centres de Contrôle et de copier cette base de données en local sur le poste. Ainsi, même déconnecté, vous accédez à toutes vos données. 
	Lors de la reconnexion, les données modifiées localement sont automatiquement mises à jour sur la base de données réseau. En cas de conflit (modification d'une tâche par différentes personnes par exemple), la base du réseau sera toujours prioritaire. 
	Pour plus de détails, consultez [Travailler en mode déconnecté avec les Centres de Contrôle](../Editeurs/3540709.md).

- **pour le GDS** : de se déconnecter de la base de données du GDS et d'utiliser le projet directement en local. Ainsi, même déconnecté, vous accédez à toutes vos données. 
	Lors de la reconnexion, les données modifiées et non extraites du GDS avant la déconnexion seront automatiquement extraites pour être mises à jour. Vous pourrez alors réintégrer vos modifications normalement dans le GDS.
	Pour plus de détails, consultez [Travailler en mode déconnecté avec le GDS](../Editeurs/2038013.md).

## Types de rubriques
<a name="types_rubriques_ELTTEXTE000309"></a>
Les différents types de rubriques proposés dans l'éditeur d'analyses sont les suivants :<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=TypeRubrique%20-%20HC%20N%B0001.gif)


| Type | Permet de stocker : |
| --- | --- |
| Texte | des caractères alphanumériques<br><br>- **Chaîne** : une chaîne de caractères de 65.535 caractères maximum. Cette chaîne de caractères est au format Ansi.<br><br>- **Mémo texte** : une chaîne de caractères de 4 Go maximum. Le format RTF est géré. Ce mémo texte est au format Ansi.<br><br>- **Caractère** : un seul caractère (initialisé par un 0 binaire, Caract(0)).<br><br>- **Unicode** : une chaîne de caractères au format Unicode.<br><br>- **Mémo texte Unicode** : une chaîne de caractères de 4 Go Maximum. Ce mémo texte est au format Unicode.<br><br><br> |
| Numérique | une valeur numérique<br><br>- **Entier signé sur 1 octet** : valeur comprise entre -128 et 127<br><br>- **Entier signé sur 2 octets** : valeur comprise entre -32.768 et 32.767<br><br>- **Entier signé sur 4 octets** : valeur comprise entre <br>	-2.147.483.648 et 2.147.483.647<br><br>- **Entier signé sur 8 octets** : valeur comprise entre -9.223.372.036.854.775.808 et 9.223.372.036.854.775.807<br><br>- **Entier non signé sur 1 octet** : valeur comprise entre 0 et 255<br><br>- **Entier non signé sur 2 octets** : valeur comprise entre 0 et 65.535<br><br>- **Entier non signé sur 4 octets** : valeur comprise entre 0 et 4.294.967.296<br><br>- **Entier non signé sur 8 octets** : valeur comprise entre 0 et 18.446.744.073.709.551.615<br><br>- **Réel sur 4 octets** : valeur comprise entre -3,4e+38 et +3.4e+38<br><br>- **Réel sur 8 octets** : valeur comprise entre -1,79e+308 et +1.79e+308<br><br>- **Décimal**(Numérique WLangage) : un décimal peut avoir jusqu'à 38 chiffres significatifs.<br><br><br> |
| Monétaire | une devise<br>Un monétaire peut avoir jusqu'à 24 chiffres significatifs (18 maximum pour la partie entière, 6 maximum pour la partie décimale) et peut prendre une valeur entre -604 462 909 807 314 587,353 087 et +604 462 909 807 314 587,353 087. |
| Date | une date ou une durée<br><br>- **Date (AAAAMMJJ)** : une date au format AAAAMMJJ (dates comprises entre le 01/01/0001 et le 31/12/9999)<br><br>- **Date et Heure** : une date-heure (comprise entre le 01/01/0001 à 00h00 et le 31/12/9999 à 23h59). La précision est effectuée à la milliseconde.<br><br>- **Durée** : une durée en millisecondes (comprise entre <br>	-9.223.372.036.854.775.808 et 9.223.372.036.854.775.807).<br><br><br> |
| Heure | une heure<br><br>- **Heure (HH)** : Heure comprise entre 00H et 23H<br><br>- **Heure (HHMM)** : Heure comprise entre 00H00 et 23H59. La précision est effectuée à la minute.<br><br>- **Heure (HHMMSS)** : Heure comprise entre 00H00min00ss et 23H59min59ss. La précision est effectuée à la seconde.<br><br>- **Heure (HHMMSSCC)** : Heure comprise entre 00H00min00ss00cs et 23H59min59ss59cs. La précision est effectuée au dixième de seconde.<br><br><br>**Astuce** : Pour conserver une heure avec une précision au centième de seconde, utilisez le type "Date et Heure". |
| Booléen | un booléen (1 : Vrai et 0 : Faux) |
| Son, image, binaire, ... | un fichier ou une chaîne binaire<br><br>- **Image (mémo binaire)** : un fichier image<br><br>- **Son (mémo binaire)** : un fichier son<br><br>- **OLE (mémo binaire)** : un fichier OLE<br><br>- **Autre mémo binaire** : tout type de fichier<br><br>- **Chaîne binaire** : une chaîne au format binaire<br><br><br> |
| Interrupteur | une valeur associée à un champ Interrupteur<br><br>- **Interrupteur** : un booléen (1 : Vrai et 0 : Faux)<br><br>- **Entier non signé sur 1 octet** : une valeur comprise entre 0 et 255<br><br><br> |
| Sélecteur, liste, combo | une valeur associée à un champ Sélecteur, un champ Liste ou un champ Combo. <br><br>Il est possible de sélectionner le sous-type suivant : <br><br>- **Entier sur 1 octet** : un indice correspondant à une liste de valeurs. La valeur de cet indice est comprise entre -128 et 127<br><br>- **Entier sur 2 octets** : un indice correspondant à une liste de valeurs. La valeur de cet indice est comprise entre -32.768 et 32.767<br><br>- **Entier sur 4 octets** : un indice correspondant à une liste de valeurs. La valeur de cet indice est comprise entre <br>	-2.147.483.648 et 2.147.483.647<br><br>- **Entier non signé sur 1 octet** : un indice correspondant à une liste de valeurs. La valeur de cet indice est comprise entre 0 et 255<br><br><br> |
| Id automatique | l'identifiant de chaque enregistrement du fichier. <br>Sa valeur est automatiquement mise à jour par le moteur de base de données HFSQL.<br><br>- **Identifiant automatique (4 octets)** : l'identifiant de chaque enregistrement (valeur comprise entre 0 et 2.147.483.647)<br>	Ce type correspond au type "Identifiant" de WINDEV 5.5<br><br>- **Identifiant automatique (8 octets)** : l'identifiant de chaque enregistrement (valeur comprise entre 0 et 9.223.372.036.854.775.807)<br><br>- **UUID automatique (128 bits)** : l'identifiant de chaque enregistrement correspond à un UUID sur 128 bits. Dans ce cas, l'UUID permet d'obtenir un identifiant unique non séquentiel.<br><br>- **UUID automatique (256 bits)** : l'identifiant de chaque enregistrement correspond à un UUID sur 256 bits. Dans ce cas, l'UUID permet d'obtenir un identifiant unique non séquentiel.<br><br><br><br><br>Remarque : Un même enregistrement peut comporter un UUID automatique et un Identifiant automatique (par exemple, pour simplifier la migration d'un existant). <br><br> |
| XML | une chaîne de caractères XML (4 Go maximum). Cette chaîne est stockée sous forme de mémo texte. |
| Index full-text | les différents composants de l'index full-text. L'index full-text peut être composé de rubriques de type texte ou mémo texte.<br><br>Pour plus de détails sur la création d'un index full-text sous l'analyse, consultez [Recherche et index full-text](../WDLang4/3044375.md). |
| Enumération, Combinaison | <br><br>- Enumération : une valeur entière parmi les valeurs possibles définies dans l'énumération associée à la rubrique.<br><br>- Combinaison : une valeur entière représentant une combinaison des valeurs possibles définies dans la combinaison associée à la rubrique.<br><br><br><br><br>Pour plus de détails sur la création d'une rubrique de type Enumération ou Combinaison, consultez [Rubriques de type Enumération et Combinaison](../Editeurs/9500222.md). |
| Formule | une chaîne de caractères en langage SQL représentant une formule de calcul. La formule peut utiliser aussi bien des fonctions SQL et/ou des fonctions WLangage (WL.). Par défaut, seule la formule est stockée, la valeur est calculée lors de l'accès en lecture à l'enregistrement.<br><br>Par exemple, calculer l'âge à partir de la rubrique DateNaissance du fichier. La valeur sera directement calculée dans le fichier de données. <br><br><pre><code>CAST(TO_CHAR(SYSDATE,'YYYY') AS int)<br>	- CAST(TO_CHAR(datenaissance,'YYYY') AS int)</code></pre><br><br><br>Il est possible toutefois de stocker la valeur calculée dans le fichier afin de ne pas refaire le calcul.<br><br>Il est possible de stocker la formule en Unicode : le calcul sera plus rapide mais la place occupée par la formule sera plus importante.<br><br>Si l'enregistrement est modifié, la formule de calcul est ré-exécutée afin de mettre à jour le buffer du fichier de données avec la nouvelle valeur.<br><br>Attention : La gestion du NULL respecte la norme SQL. Par exemple, si une formule ajoute la valeur NULL à une valeur, le résultat correspond lui-même à la valeur NULL. |
| Horodatage | une DateHeure ou une Date ou une Heure représentant la date de création ou de dernière modification de l'enregistrement.<br><br>Il est possible de choisir : <br><br>- si la date de création ou de dernière modification de l'enregistrement doit être enregistrée.<br><br>- si l'heure locale ou l'heure UTC doit être utilisée.<br><br><br><br><br>Ce type de rubrique est également appelé "Rubrique DateHeure automatique". |
| UUID | un UUID. Un UUID (Universal Unique IDentifier) est un nombre aléatoire réputé unique. Un UUID est utile pour identifier facilement et de manière certaine un élément quelconque.<br><br>Il est possible de choisir : <br><br>- un UUID sur 128 bits. Ce type de rubrique UUID respecte la norme RFC 4122 en 128 bits. Seul le UUID sur 128 bits est normé.<br><br>- un UUID sur 256 bits. <br><br><br><br><br>Pour gérer un identifiant automatique de type UUID (automatiquement initialisé par HFSQL), il est nécessaire d'utiliser une rubrique de type "Identifiant", ayant pour sous-type "UUID automatique". <br><br>Remarque : L'intérêt majeur de l'UUID est de permettre l'anonymisation et la pseudonymisation, utile au respect de la norme RGPD. L'UUID permet également de gérer des identifiants de cryptomonnaie. |
| JSON | une chaîne JSON. Ce type de rubrique peut contenir NULL, une chaîne vide ou un JSON valide. La vérification du JSON est automatiquement réalisée à l'écriture de l'enregistrement. <br><br>Ce type de rubrique est notamment utilisé avec les fonctions SQL JSON. |
| Mot de passe | une chaîne de caractères utilisée comme mot de passe. <br><br>La chaîne de caractères contenue dans cette rubrique est "salée" et "hachée". C'est le résultat de ces opérations qui est stocké dans la rubrique, ainsi que le sel utilisé pour le salage et le type d'algorithme utilisé. Ces informations ne permettent pas de reconstituer le mot de passe d'origine.  <br><br>Ce mot de passe ne peut pas être affiché à l'utilisateur.<br><br>Il est possible de choisir : <br><br>- l'algorithme de dérivation de clé,<br><br>- l'algorithme de hash,<br><br>- le nombre d'itérations,<br><br>- la taille du résultat. <br><br><br><br><br> Pour plus de détails, consultez [Type de rubrique : Mot de passe sécurisé](../Editeurs/9500235.md). |


**Remarque** : Les rubriques Unicode sont gérées par la librairie wdxxxuni.dll. Cette librairie n'est pas disponible sous WINDEV Mobile.




### Rubriques Tableau
<a name="rubriques_tableau_ELTPARAGRAPHE000236"></a>Certains types de rubriques peuvent correspondre à un tableau : le contenu de la rubrique sera indicé.

Pour définir une rubrique de type Tableau : 

1. Créez une rubrique. Les types de rubriques pouvant correspondre à un tableau sont : Texte, Numérique, Monétaire, Date, Heure, Booléen, Interrupteur. 

2. Dans la description de la rubrique : 

	- Cochez l'option "Tableau" et indiquez la dimension du tableau. 
			Attention : La dimension du tableau est fixe. L'espace correspondant est réservé même si la rubrique n'est pas renseignée.

	- La taille réelle de la rubrique est affichée.




3. Indiquez si nécessaire les valeurs par défaut de chaque élément du tableau. 

4. Validez. 




**Remarques** : 

- Le premier indice commence à 1. 

- La syntaxe de manipulation des rubriques tableau est la suivante : &lt;Nom fichier&gt;.&lt;Nom rubrique&gt;[&lt;Indice&gt;]

- Attention : Ce type de rubrique est conservé par compatibilité. La majorité des bases de données SQL ne supportent pas ce type de rubriques.

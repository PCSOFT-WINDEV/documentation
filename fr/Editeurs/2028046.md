


## Etape 1 : Choix du profil de déploiement
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000199"></a>
L'assistant de création du programme d'installation permet de sélectionner le profil de déploiement à utiliser. Ce profil contient les paramètres du serveur d'application ainsi que les paramètres de déploiement du site.

Il est possible :

- soit de choisir un profil précédemment créé (cas de la mise à jour d'un site par exemple). Dans ce cas, les informations concernant le profil sont automatiquement prises en compte. Il suffit de les valider.  

- soit de créer un nouveau profil (cas d'une première installation par exemple).

- soit de ne pas choisir de profil pour l'instant. Dans ce cas, les paramètres du profil de déploiement seront indiqués lors de l'installation du site (cas d'une installation par package ou par média physique).




Dans le cas d'une mise à jour, il est possible de créer de nouveaux profils (en général, un profil par serveur) et de les sélectionner pour la mise à jour. Le site sera déployé sur tous les serveurs sélectionnés : 

- Si le site est déjà installé sur le serveur, l'assistant effectuera une mise à jour.

- Si le site n'est pas installé sur le serveur, l'assistant fera une nouvelle installation.




**Remarque** : Si l'installation échoue sur l'un des serveurs, l'installation se poursuit normalement sur les autres serveurs.

## Paramètres du serveur d'application
<a name="parametres_serveur_application_ELTTEXTE000223"></a>


### Saisie des informations sur le serveur d'application
<a name="saisie_des_informations_sur_serveur_application_ELTPARAGRAPHE000033"></a>Les informations à saisir sont les suivantes : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Info_serveur.gif&type=thumb)


- Adresse du serveur : nom de machine accessible par le réseau, une adresse IP ou une adresse Internet.
	Dans le cas d'une installation sur un Cluster WEBDEV, il est nécessaire d'indiquer l'adresse du proxy associé au Cluster WEBDEV. 

- Caractéristiques du compte de déploiement : Nom d'utilisateur et mot de passe (définis dans le Centre de Contrôle Hébergement, par l'hébergeur).


Remarque : Attention : le nom d'utilisateur peut être précédé du nom du domaine pour éviter des confusions. Par exemple : "monposte\\test" ou "mondomaine\\test".

Le bouton "Paramètres avancés" permet de :

- Définir les paramètres du serveur. 

- Définir les paramètres du site. 

- Indiquer le mode de transfert des fichiers d'installation : 

	- FTP. Il est possible de définir les caractéristiques du serveur FTP (port, adresse IP, mode de transfert).

	- HTTP (mode par défaut à partir de la version 25). 




- Indiquer les commandes d'installation : 

	- utiliser des requêtes authentifiées.

	- utiliser le protocole HTTPS.

	- définir si nécessaire les caractéristiques du proxy.







Pour plus de détails, consultez le paragraphe [Paramètres avancés](#NOTE4_1).

Le bouton "Tester les paramètres" permet d'établir une connexion avec le serveur Web et permet de vérifier que toutes les informations saisies sont correctes.

Le bouton "Exporter les paramètres du serveur" permet de créer un fichier XML contenant les paramètres du serveur. Par défaut, ce fichier est créé dans le sous-répertoire "\\Personnel\\Serveurs WEBDEV" du répertoire d'installation de WEBDEV. Il est ainsi possible de transmettre les informations concernant un serveur d'un projet à un autre. Les paramètres pourront être importés grâce au bouton "Importer les paramètres d'un serveur".

## Paramètres de déploiement du site
<a name="parametres_deploiement_site_ELTTEXTE000247"></a>
L'assistant permet ensuite d'indiquer les paramètres de déploiement du site. Les informations à saisir sont les suivantes : 

- ***Le nom du site***. Par défaut, ce nom correspond au nom du projet. Il est possible d'utiliser un nom différent. Il est ainsi possible de déployer plusieurs instances du même projet sur le même serveur en leur donnant des noms de sites différents. Par exemple, le projet "REWALI" peut être déployé sur un serveur Web :

	- sous le nom REWALI : ce site est destiné à l'agence de voyage X1.

	- sous le nom RESA : ce site est destiné à l'agence de voyage X2.




- ***Le nom de domaine du fichier SiteMap***. Ce paramètre doit être renseigné dans le cas où le nom du serveur utilisé pour le déploiement est différent du nom de domaine permettant d'y accéder. Le fichier SiteMap est un fichier utilisé par Google pour le référencement.

- ***Le sous-répertoire contenant les fichiers de données*** au format HFSQL Classic. Le nom de ce sous-répertoire est lié au nom du site. Il correspond au nom du répertoire où les fichiers de données du site sont installés. Ce répertoire est un sous-répertoire du répertoire de données attribué par l'hébergeur. Ce nom de répertoire doit être modifié uniquement si plusieurs sites partagent les mêmes fichiers de données.

- ***Le nom du profil de déploiement***. Ce nom sera donné au profil de déploiement pour le retrouver facilement (lors de la mise à jour du site par exemple).




**Remarque** : Si vous avez renseigné ces informations dans les paramètres avancés, ces informations sont automatiquement récupérées.

## Paramètres avancés
<a name="parametres_avances_ELTTEXTE000271"></a>
Les paramètres de déploiement du site consistent à indiquer :

- ***Le nom du profil de déploiement***. Ce nom sera donné au profil de déploiement pour le retrouver facilement (lors de la mise à jour du site par exemple).

- ***Les paramètres du serveur*** (onglet "Général") : Adresse du serveur d'application, Compte de déploiement, Compte Utilisateur Système. Ces paramètres sont ceux renseignés dans l'écran précédent : "Paramètres du serveur d'application".

- ***les paramètres du site*** (onglet "Général") : 

	- ***Nom de déploiement du site***. Par défaut, ce nom correspond au nom du projet. Il est possible d'utiliser un nom différent. Il est ainsi possible de déployer plusieurs instances du même projet sur le même serveur en leur donnant des noms de sites différents. Par exemple, le projet "REWALI" peut être déployé sur un serveur Web :

		- sous le nom REWALI : ce site est destiné à l'agence de voyage X1.

		- sous le nom RESA : ce site est destiné à l'agence de voyage X2.




- ***Nom de domaine du fichier SiteMap***. Ce paramètre doit être renseigné dans le cas où le nom du serveur utilisé pour le déploiement est différent du nom de domaine permettant d'y accéder. Le fichier SiteMap est un fichier utilisé par Google pour le référencement.

- ***Répertoire contenant les fichiers de données*** au format HFSQL Classic. Le nom de ce sous-répertoire est lié au nom du site. Il correspond au nom du répertoire où les fichiers de données du site sont installés. Ce répertoire est un sous-répertoire du répertoire de données attribué par l'hébergeur. Ce nom de répertoire doit être modifié uniquement si plusieurs sites partagent les mêmes fichiers de données.

- Si le site doit être déployé dans toutes les langues ou uniquement pour certaines langues (à sélectionner). 

- Le transfert des fichiers d'installation (onglet "Avancé") :

	- Sélection du mode de transfert : 

		- HTTP

		- FTP




- Lors d'un transfert par FTP, il est possible d'indiquer : 

	- L'adresse du serveur FTP, 

	- Le mode FTP : FTP actif ou passif

## Validation
<a name="validation_ELTTEXTE000295"></a>
Si vous avez choisi de déployer le site WEBDEV sur un serveur d'application WEBDEV distant, lors de la validation des paramètres de déploiement du site, WEBDEV tente de se connecter au serveur Web indiqué (et au serveur FTP indiqué si le déploiement est effectué par FTP).

En cas de problème, un message d'erreur est indiqué. Pour plus de détails sur les différentes erreurs, et leur mode de résolution, consultez : [Erreur en cas d'installation par FTP](../Editeurs/2028057.md).

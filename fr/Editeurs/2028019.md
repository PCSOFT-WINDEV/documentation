


## Déployer un site WEBDEV sur un Serveur d'application WEBDEV distant ou un Cluster WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000122"></a>
L'installation d'un site dynamique WEBDEV (en mode Session ou AWP) sur un Serveur d'application distant nécessite :

- Un serveur Web.

- Le Serveur d'Application WEBDEV installé sur le serveur Web de déploiement. 

- La création d'un compte WEBDEV dans l'administrateur WEBDEV, la communication du nom et du mot de passe du compte.




Pour une installation sur un cluster WEBDEV, les éléments suivants sont également nécessaires : 

- Le serveur d'application WEBDEV doit être installé en mode Cluster et le Cluster WEBDEV doit contenir au moins un noeud. Pour plus de détails, consultez [Installation d'un Cluster WEBDEV](../Editeurs/1410086635.md). 

- Un proxy doit être installé et configuré pour être utilisé avec le Cluster WEBDEV. Pour plus de détails, consultez [Cluster WEBDEV : Configuration du proxy](../WDAdminDeploiement/3539085.md). 

- L'installation du site ou Webservice doit se faire via le proxy. Il est donc nécessaire de connaître l'adresse du proxy.  




Pour une installation via FTP, les éléments suivants sont également nécessaires : 

- Un serveur FTP installé sur le serveur Web de déploiement **uniquement si le déploiement est effectué par FTP**. En cas de déploiement par HTTP, le serveur FTP n'est pas nécessaire.

- La création d'un compte FTP, la communication du nom et du mot de passe du compte FTP ainsi que l'adresse IP du serveur.




Remarques sur le Serveur d'application WEBDEV : 

- Si le Serveur d'Application est installé en version 24 ou inférieure, la version de WEBDEV sur le poste de développement peut être quelconque, jusqu'à la 24.

- Si le Serveur d'Application est installé en version 25 ou supérieure, la version de WEBDEV sur le poste de développement doit être au minimum la 24.
	Cas particulier : dans le cas d'un déploiement distant depuis WEBDEV 24 sur un Serveur d'Application en version 25 ou supérieure, il est nécessaire de demander le module WWINST.EXE pour WEBDEV 24 depuis les correctifs des ressources pratiques :  [https://www.pcsoft.fr/st/nouveautes-st.html](https://www.pcsoft.fr/st/nouveautes-st.html), option "Liste des modules correctifs disponibles de WINDEV, WEBDEV, WINDEV Mobile".

## Les différentes étapes
<a name="les_differentes_etapes_ELTTEXTE000146"></a>
Les différentes étapes du déploiement sur un serveur distant sont les suivantes :

1. [Saisie des informations sur le serveur](../Editeurs/2028046.md). 
	Une connexion au serveur est automatiquement effectuée. Les logins et mots de passe sont vérifiés.
	Cette étape permet de définir toutes les options d'accès au serveur (protocole HTTPS, requêtes authentifiées, serveur FTP si nécessaire, ...)  

2. [Sélection des fichiers à installer](../Editeurs/2028058.md).

3. [Modification automatique des fichiers de données](../Editeurs/2028048.md).

4. [Paramétrage de l'administrateur WEBDEV et mode d'installation](../Editeurs/2028051.md).

5. Transfert des fichiers et installation. 
	Des messages d'erreur peuvent apparaître à diverses étapes de l'installation à distance. Pour plus de détails, consultez [Erreurs en cas d'installation par FTP](../Editeurs/2028057.md).

6. Lorsque l'installation est terminée, l'assistant fournit la ligne de commande permettant de lancer le site installé. Cette ligne peut être copiée si nécessaire dans le presse-papiers grâce au menu contextuel.

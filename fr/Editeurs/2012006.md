


## Conditions nécessaires à la manipulation d'un assemblage .NET
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000176"></a>
**Pour créer et utiliser un assemblage .NET sous WINDEV**, il est nécessaire de : 

- Installer le Framework .NET ou le SDK .NET (selon les versions supportées). 

- Installer le Runtime .NET correspondant à la version voulue. 

- Installer la "LibC" correspondant à la version voulue. 

- [Régler le niveau de sécurité .NET](#NOTE3).

- [Rendre accessibles les DLLs nécessaires à l'exécution de l'assemblage .NET](#NOTE4).




**Remarques** : 

- Par défaut, lors de l'utilisation d'un assemblage .NET, c'est le framework/runtime le plus récent qui est utilisé.

- Sur le poste où l'assemblage .NET sera utilisé, il est nécessaire d'installer le runtime correspondant et la LibC.

<a name="NOTE2_1"></a>


## Installer le Framework .NET ou le SDK
<a name="installer_framework_net_sdk_ELTTEXTE000200"></a>


### Vérifier si le Framework .NET est installé sur le poste en cours
<a name="verifier_framework_net_est_installe_sur_poste_cours_ELTPARAGRAPHE000033"></a>Pour vérifier si le Framework .NET est installé sur le poste en cours : 

1. Ouvrez le panneau de configuration de Windows (option "Démarrer .. Paramètres .. Panneau de configuration") et sélectionnez l'option "Ajout / Suppression de programmes".

2. Recherchez dans la liste le programme "Framework .NET".



<a name="NOTE2_2"></a>


### Installer le Framework .NET sur le poste en cours (.Net version 2.0, 3.x, ou 4.x)
<a name="installer_framework_net_sur_poste_cours_net_version_20_3x_4x_ELTPARAGRAPHE000043"></a>Pour installer le Framework .NET sur le poste de développement : 

1. Consultez le site de Microsoft à l'adresse suivante : [https://dotnet.microsoft.com/download/visual-studio-sdks](https://dotnet.microsoft.com/download/visual-studio-sdks).

2. Dans la partie ".NET Framework", recherchez l'installation du Framework .NET voulu (version 2.0, 3.x, ou 4.x).

3. Installez le Framework .NET voulu.

4. Installez le "Runtime" correspondant. 



<a name="NOTE2_3"></a>


### Installer le SDK .NET 5.0
<a name="installer_sdk_net_50_ELTPARAGRAPHE000060"></a>Pour installer le SDK .NET 5.0 sur le poste de développement : 

1. Consultez le site de Microsoft à l'adresse suivante : [https://dotnet.microsoft.com/download/visual-studio-sdks](https://dotnet.microsoft.com/download/visual-studio-sdks)

2. Dans la partie ".NET/.NET Core" : 

	- Téléchargez et installez le SDK .NET voulu (64 bits).

	- Téléchargez et installez le "Runtime" correspondant (64 bits).

## Installer la LibC
<a name="installer_libc_ELTTEXTE000236"></a>
La DLL .NET a besoin de la LibC. Elle ne se charge pas sur des machines sur lesquelles aucune autre application n'a encore installé la LibC. 

La version distribuable de la LibC est librement téléchargeable sur le site de Microsoft : [https://docs.microsoft.com/en-us/cpp/windows/latest-supported-vc-redist?view=msvc-170](https://docs.microsoft.com/en-us/cpp/windows/latest-supported-vc-redist?view=msvc-170)

## Régler le niveau de sécurité .NET
<a name="regler_niveau_securite_net_ELTTEXTE000260"></a>
Pour régler le niveau de sécurité .NET, nous vous conseillons de consulter la documentation Microsoft. Le niveau de sécurité doit être réglé sur "Confiance totale".

## Rendre accessibles les DLLs nécessaires à l'exécution de l'assemblage .NET
<a name="rendre_accessibles_les_dlls_necessaires_execution_assemblage_net_ELTTEXTE000284"></a>
Pour rendre accessibles les DLLs nécessaires à l'exécution de l'assemblage .NET : 

1. Copiez si nécessaire la librairie "WDxxNET.DLL" (cette librairie est disponible dans le sous-répertoire "\\Programmes" du répertoire d'installation) :

	- soit dans le répertoire de l'assemblage .NET.

	- soit dans le sous-répertoire "Assembly" du répertoire d'installation de Windows.

		- Pour un assemblage .NET 2.0, la DLL à copier est wd280net2.dll.

		- Pour un assemblage .NET 2.0 en 64 bits, la DLL à copier est wd280net2_64.dll.

		- Pour un assemblage .NET 4.0, la DLL à copier est wd280net4.dll.

		- Pour un assemblage .NET 4.0 en 64 bits, la DLL à copier est wd280net4_64.dll.

		- Pour un assemblage .NET 5.0, la DLL à copier est wd280net5.dll.

		- Pour un assemblage .NET 5.0 en 64 bits, la DLL à copier est wd280net5_64.dll.




2. Copiez si nécessaire les librairies WINDEV :

	- soit dans le répertoire de l'assemblage .NET.

	- soit dans le PATH.







**Rappel** : Le PATH liste les répertoires dans lesquels les exécutables (fichiers ".EXE") et les librairies (fichiers ".DLL") sont recherchés en priorité.

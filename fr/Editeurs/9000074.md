


## Charte de programmation
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000329"></a>
Une charte de programmation permet de préfixer automatiquement le nom de toutes les variables et de tous les éléments du projet (fenêtre, champ d'un état, classe, ...). Ce préfixage automatique permet de connaître et/ou de rechercher facilement un élément d'un type donné.

Dans l'exemple suivant, le type des champs manipulés est facilement identifiable : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=PrefAuto.GIF)


**Remarque** : Il est également possible de connaître le type d'un élément grâce à la bulle d'aide affichée lors du survol de l'élément : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=PrefAutoType.GIF)

## Comment le faire ?
<a name="comment_faire_ELTTEXTE000353"></a>


### Activer le préfixage automatique
<a name="activer_prefixage_automatique_ELTPARAGRAPHE000029"></a>Pour activer le préfixage automatique :

1. Affichez la description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".

2. Affichez l'onglet "Options".

3. Cochez l'option "Activer le préfixage automatique des variables et des éléments du projet".

4. Choisissez la charte de programmation à utiliser. Par défaut, la charte "Standard" (fournie avec WINDEV, WEBDEV ou WINDEV Mobile) est utilisée.

5. Cochez l'option "Ajouter automatiquement les préfixes" si vous désirez que le préfixe soit géré automatiquement. Dans ce cas, les préfixes seront automatiquement ajoutés sous les différents éditeurs lorsqu'une charte de programmation a été définie. Ainsi, lors de la saisie du code, le préfixe correspondant au type de variable sera automatiquement ajouté devant le nom de la variable. 

6. Cochez l'option "Lever une erreur de compilation d'UI si un champ n'a pas le bon préfixe" pour être informé par une erreur de compilation d'UI lorsque les champs n'utilisent pas le préfixe correspondant à celui défini dans la charte de programmation. 

7. Validez. Lors de la création de nouveaux éléments ou de nouvelles variables, le préfixage correspondant sera automatiquement ajouté.




**Remarques** :

- Autre solution pour activer le préfixage automatique : sous le volet "Projet", dans le groupe "Autres actions", déroulez "Charte" et sélectionnez "Utiliser la charte". 

- A partir de la version 27, lors de la création d'un nouveau projet, seul le préfixage automatique sur les éléments du projet est activé par défaut. Les préfixes ne sont pas ajoutés automatiquement (notamment dans le code WLangage). 
	L'option "Ajouter automatiquement les préfixes à la création des variables" permet d'activer également le préfixage sur les variables dès la création du projet. 

- Les noms de variable de moins de 3 caractères ne sont pas préfixés.



<a name="NOTE2_2"></a>


### Désactiver le préfixage automatique
<a name="desactiver_prefixage_automatique_ELTPARAGRAPHE000097"></a>Pour désactiver le préfixage automatique, utilisez une des méthodes suivantes :

- sous le volet "Projet", dans le groupe "Autres actions", déroulez "Charte" et désélectionnez l'option "Utiliser la charte". 

- dans l'onglet "Options" de la description du projet, décochez l'option "Activer le préfixage automatique des variables et des éléments du projet".




Rappel : Pour afficher la description du projet, sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description". 
<a name="NOTE2_3"></a>


### Créer une charte de programmation
<a name="creer_une_charte_programmation_ELTPARAGRAPHE000137"></a>Pour créer une charte de programmation :

1. Affichez la description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".

2. Affichez l'onglet "Options".

3. Cliquez sur le bouton "Créer une charte de programmation" et validez la fenêtre d'information qui s'affiche. 

4. Sélectionnez l'emplacement de la nouvelle charte de programmation et saisissez son nom (fichier ".CDP").

5. Validez (bouton "Enregistrer"). La nouvelle charte de programmation est créée. 
	Par défaut, cette charte correspond à la charte "Standard" (fournie avec WINDEV, WEBDEV ou WINDEV Mobile).

6. Cliquez sur le bouton "Editer".

7. Définissez les différents préfixages et validez.


**Remarque** : La charte de programmation "Standard" ne peut pas être modifiée.
<a name="NOTE2_4"></a>


### Modifier une charte de programmation
<a name="modifier_une_charte_programmation_ELTPARAGRAPHE000172"></a>Pour modifier une charte de programmation :

1. Affichez la description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".

2. Affichez l'onglet "Options".

3. Sélectionnez la charte de programmation à modifier.

4. Cliquez sur le bouton "Editer la charte". 

5. Définissez les nouveaux préfixages et validez.




**Remarques** :

- Ces modifications seront prises en compte uniquement lors de la création de nouveaux éléments ou de nouvelles variables.

- La charte de programmation "Standard" ne peut pas être modifiée.

- Autre solution pour éditer la charte de programmation en cours : sous le volet "Projet", dans le groupe "Autres actions", déroulez "Charte" et sélectionnez "Editer la charte". 



<a name="NOTE2_5"></a>


### Modifier la charte de programmation utilisée
<a name="modifier_charte_programmation_utilisee_ELTPARAGRAPHE000219"></a>Pour modifier la charte de programmation utilisée :

1. Affichez la description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".

2. Affichez l'onglet "Options".

3. Sélectionnez la charte de programmation à utiliser (dans la liste "Charte de programmation" ou grâce au bouton "..."). 

4. Validez.


**Remarque** : La même charte de programmation (fichier ".CDP") peut être utilisée dans plusieurs projets WINDEV, WEBDEV et WINDEV Mobile.
<a name="NOTE2_6"></a>


### Gestion des préfixes
<a name="gestion_des_prefixes_ELTPARAGRAPHE000245"></a>La gestion des préfixes de la charte de programmation permet : 

- d'ajouter automatiquement les préfixes sous les différents éditeurs lorsqu'une charte de programmation a été définie.

- de générer une erreur de compilation d'UI si un champ n'a pas le bon préfixe. 




Pour gérer les préfixes : 

1. Affichez la description du projet : sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".

2. Affichez l'onglet "Options".

3. Pour ajouter automatiquement les préfixes, sélectionnez l'option "Ajouter automatiquement les préfixes".

4. Pour générer une erreur d'UI en cas de préfixe invalide, sélectionnez l'option "Lever une erreur de compilation d'UI si un champ n'a pas le bon préfixe". 

5. Validez.

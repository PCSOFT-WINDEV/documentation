
## Test d'une application WINDEV et WINDEV Mobile
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000103"></a>
WINDEV et WINDEV Mobile proposent plusieurs méthodes pour tester vos applications :

- [Test de l'ensemble du projet](../Editeurs/2019017.md)
	Le test de l'ensemble du projet permet de simuler le lancement de l'application par l'exécutable. Il est ainsi possible de tester l'application dans son ensemble, même si son développement n'est pas terminé. Dès qu'un problème apparaît dans l'exécution du projet, vous pouvez lancer le débogueur pour connaître et solutionner le problème rencontré.
	

- [Test d'une fenêtre seule](../Editeurs/2019014.md)
	Le test d'une fenêtre seule permet d'exécuter uniquement la fenêtre en cours sous WINDEV ou WINDEV Mobile. Vous pouvez ainsi choisir de tester votre projet à partir d'une fenêtre donnée, ou de tester le fonctionnement d'une fenêtre dès que son développement est terminé. Comme pour le test du projet, il est possible de lancer le débogueur dès qu'un problème est rencontré. 
	

- [Test d'une requête seule](../Editeurs/2019011.md)
	Le test d'une requête seule permet d'exécuter uniquement la requête en cours sous WINDEV ou WINDEV Mobile. Vous pouvez ainsi choisir de tester le fonctionnement d'une requête dès que son développement est terminé.

- [Test d'un état seul](../Editeurs/2019015.md)
	Le test d'un état seul permet d'exécuter uniquement l'état en cours sous WINDEV. Vous pouvez ainsi choisir de tester le fonctionnement d'un état dès que son développement est terminé. Comme pour le test du projet, il est possible de lancer le débogueur dès qu'un problème est rencontré.

- [Exécution du projet pas à pas](../Editeurs/2019001.md)
	L'exécution du projet pas à pas permet de lancer le débogueur au lancement de l'application. Cette solution permet de suivre méticuleusement le déroulement de l'application.

- [Tests de non-régression](../Editeurs/2019025.md)
	Les tests de non-régression sur un élément du projet ou sur le projet dans son ensemble permettent de vérifier qu'une modification (de code, d'interface, ...) n'entraîne pas des dysfonctionnements dans l'application. WINDEV propose d'enregistrer les tests directement dans le projet, afin de pouvoir passer ces tests à chaque nouvelle version de l'application.




Remarques : 

- Lorsque vous réalisez une phase de débogage, tous les éléments sur lesquels vous vous arrêtez (via un point d'arrêt par exemple) sont ouverts sous l'éditeur. Il est possible de relancer le dernier test effectué (fenêtre, requête, état, ...) depuis n'importe quel élément du projet grâce à l'option "GO .. Relancer le dernier test" présente dans les boutons d'accès rapide. ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GO_Relancer_le_dernier_test%20-%20HC%20N%B0001.gif)


- Les lignes de code exécutées par les différents types de test sont automatiquement prises en compte par le [Code coverage](../Editeurs/2019034.md).


## Déploiement manuel d'un site dynamique
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000266"></a>
WEBDEV offre plusieurs possibilités pour le déploiement d'un site dynamique :

- [déploiement par FTP](../Editeurs/2028019.md).

- [déploiement par média physique](../Editeurs/2028090.md), [par média physique autonome](../Editeurs/2028091.md).

- [déploiement par package](../Editeurs/2028089.md).




Dans certains cas, ces solutions peuvent s'avérer inefficaces, et l'installation du site doit être réalisée manuellement.

Cette page d'aide présente les différentes manipulations à réaliser.

### Transfert les fichiers par FTP
<a name="transfert_les_fichiers_par_ftp_ELTPARAGRAPHE000052"></a>Les fichiers à transférer sur le serveur Web sont les suivants :

- Le répertoire &lt;NomProjet&gt;_WEB (et ses sous-répertoires), répertoire des images et des classes java script.

- Le contenu sous-répertoire "EXE" du projet de développement, ou au moins le fichier &lt;NomSite&gt;.WDL (n'oubliez pas de refaire une mise en bibliothèque au préalable pour que ce fichier soit à jour)

- Les répertoires des "pages de bases" des différentes langues : FR, UK (pour les versions inférieures à WEBDEV 11 uniquement)





<a name="NOTE2_3"></a>


### Création de l'alias (ou répertoire virtuel) et du lien pour les URLs courtes
<a name="creation_alias_repertoire_virtuel_lien_pour_les_urls_courtes_ELTPARAGRAPHE000064"></a>Cette opération nécessite d'être sur le poste serveur (ou de pouvoir en prendre le contrôle à distance)

Il faut créer un alias (ou répertoire virtuel) qui se nomme &lt;NOMProjet&gt;_WEB au niveau du serveur Web (IIS, Apache). Cet alias doit "pointer" vers le répertoire du même nom qui a été transféré.

Avec IIS 7 ou supérieur et Apache 2.2 ou supérieur, il faut également créer un lien entre le nom du site et le serveur d'application WEBDEV pour la gestion des [URLs courtes](../Editeurs/2030058.md).

Pour plus de détails, vous trouverez les procédures de configuration pour les sites dynamiques WEBDEV avec les serveurs suivants :

- [Apache 1.3.x, Apache 2.0.x](../Editeurs/2037011.md)

- [Apache 2.2](../Editeurs/2037013.md)

- [FrontPage 97 et 98](../Editeurs/2037009.md)

- [OmniHTTPD professionnel](../Editeurs/2037003.md)

- [IIS version 2 ou 3](../Editeurs/2037002.md)

- [IIS version 4](../Editeurs/2037010.md)

- [IIS version 5](../Editeurs/2037007.md)

- [IIS version 6](../Editeurs/2037008.md)

- [Domino](../Editeurs/2037006.md)

### Ajout du site dynamique dans l'administrateur du "Serveur d'application WEBDEV"
<a name="ajout_site_dynamique_dans_administrateur_serveur_application_webdev_ELTPARAGRAPHE000102"></a>Cette opération nécessite d'être sur le poste serveur (ou de pouvoir en prendre le contrôle à distance)

**Pour ajouter le site WEBDEV au niveau de l'administrateur du Serveur d'Application WEBDEV**, il faut :

1. Sélectionner l'onglet "Sites" de l'administrateur.

2. Cliquer sur le bouton "Ajouter".

3. Sélectionner la bibliothèque du site (&lt;NomSite&gt;.WDL).





<a name="NOTE2_5"></a>


### Remarque
<a name="remarque_ELTPARAGRAPHE000116"></a>Il est possible d'ajouter ou de vérifier la configuration d'un site WEBDEV via la base de registre (REGEDIT). Les paramètres d'un site sont regroupés dans le base de registre du serveur dans la clé : 

[HKEY_LOCAL_MACHINE\\Software\\PC Soft\\WebDev\\28.0\\Applications\\NOMSITE]

où NOMSITE est le nom du site concerné.

Les différentes valeurs sont :

| Nom de la valeur | Type | Signification |
| --- | --- | --- |
| PROJECTNAME | Chaîne | Nom du projet |
| PROJECTPATH | Chaîne | Chemin d'installation du projet |
| VERSION | Chaîne | Version déployée |
| VIMAGEDIR | Chaîne | Nom du répertoire virtuel des images |
| HFPATH | Chaîne | Chemin des fichiers de données HyperFileSQL<br><br> |
| AS400 | DWORD | 1 si le site utilise le Connecteur Natif AS/400 |
| LOCKFORUPDATE | Chaîne | 1 si le site est verrouillé |
| LOGDIR | Chaîne | Chemin des fichiers de log |
| MAXCONNECTAPP | Chaîne | Nombre maximum d'utilisateurs connectés |
| MAXRECONNECTAPP | Chaîne | Nombre maximum d'utilisateurs connectés avec la même adresse |
| TIMEOUT_NORQ | Chaîne | Durée limite (en secondes) d'une requête au site avant déconnexion |

## Déploiement "Manuel" pour une mise à jour
<a name="deploiement_manuel_pour_une_mise_jour_ELTTEXTE000344"></a>
Il suffit de réaliser l'étape "Transfert les fichiers par FTP" du Déploiement "manuel" pour une première installation.

**CONSEIL** : Pour mettre le site "en travaux" le minimum de temps possible, nous vous conseillons de transférer vos fichiers dans de nouveaux répertoires de façon à avoir un répertoire contenant le site en cours, et un répertoire contenant le nouveau site. Il ne sera nécessaire de "bloquer" le site que le temps de renommer les deux répertoires (plus le temps de déplacer).

**ATTENTION** : Cette opération ne lance pas la modification automatique de la structure des fichiers de données si celle-ci a évoluée. Dans le cas de changement de format des fichiers, il est nécessaire de récupérer les fichiers en local, d'effectuer la modification de structure, puis de re-envoyer les fichiers modifiés sur le serveur.

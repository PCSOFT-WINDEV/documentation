


## Analyse : Description des fichiers de données
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000236"></a>
L'analyse permet de décrire la structure des fichiers de données utilisés par le projet.

Il est possible de décrire dans l'analyse :  

- [un nouveau fichier de données.](#NOTE2_1)

- [un fichier de données prédéfini, livré avec WINDEV, WEBDEV et WINDEV Mobile.](#NOTE2_2)

- [un fichier de données importé depuis une base externe](../Editeurs/2011025.md) (appelée aussi "Base tierce").

- [un fichier de données importé depuis une version antérieure de WINDEV](../Editeurs/2011025.md).

- [un fichier de données HFSQL ISAM importé via l'explorateur de fichiers Windows](../Editeurs/2011025.md).




**Remarque** : Pour modifier la description d'un fichier de données ou de ses rubriques : 

- Sélectionnez le fichier de données sous l'éditeur d'analyses. 

- Dans le menu contextuel, sélectionnez une des options suivantes : 

	- "Description du fichier de données".

	- "Description des rubriques".







**Rappel** : Dans une application WINDEV, WEBDEV ou WINDEV Mobile, les termes "MLD" et "Analyse" sont utilisés indifféremment pour définir la structure de la base de données associée à un projet.

Pour plus de détails sur les manipulations d'un fichier ou d'une rubrique sous l'éditeur d'analyses, consultez [Manipulations d'un fichier de données](../Editeurs/2011007.md) et [Manipulations d'une rubrique](../Editeurs/2011009.md).

## Ajout d'un nouveau fichier de données
<a name="ajout_nouveau_fichier_donnees_ELTTEXTE000266"></a>


### Créer un nouveau fichier de données
<a name="creer_nouveau_fichier_donnees_ELTPARAGRAPHE000053"></a>Pour créer un nouveau fichier de données : 

1. Sous le volet "Analyse", dans le groupe "Création", cliquez sur "Nouveau fichier". L'assistant de création d'un fichier de données se lance.

2. Sélectionnez l'option "Créer une nouvelle description d'un fichier de données". Passez à l'étape suivante.

3. Spécifiez :

	- le ***nom logique du fichier***. Ce nom sera utilisé pour manipuler le fichier de données.

	- le ***libellé*** résumant succinctement le sujet du fichier de données.

	- la ***représentation d'un enregistrement*** dans le fichier de données. Cette représentation permet d'améliorer la lisibilité des questions posées lors de la description des liaisons. Cette option doit être précédée d'un article indéfini (UN ou UNE).

	- si le fichier de données possède une ***rubrique de type "Identifiant automatique"***. La valeur de cette rubrique est unique pour chaque enregistrement et est automatiquement calculée par le moteur HFSQL.
			Il est possible d'utiliser : 

		- soit un identifiant sur 8 octets. Ce type d'identifiant est obligatoire dans le cas de la réplication de serveurs HFSQL. 

		- soit un identifiant sur 4 octets. 




- soit un UUID automatique (128 bits). Dans ce cas, l'identifiant de chaque enregistrement correspond à un UUID sur 128 bits (l'UUID permet d'obtenir un identifiant unique non séquentiel). 

- soit un UUID automatique (256 bits). Dans ce cas, l'identifiant de chaque enregistrement correspond à un UUID sur 256 bits (l'UUID permet d'obtenir un identifiant unique non séquentiel).

4. Dans l'étape suivante, indiquez le ***type de base de données*** associé au fichier de données. Selon le type choisi, ce fichier de données sera manipulé par le moteur HFSQL, par un pilote OLE DB ou par un des Connecteurs natifs (SQL Server, Oracle, ...).
	Si le type de base utilisé ne correspond pas à HFSQL Classic (Réseau ou Monoposte), l'assistant demande les caractéristiques de connexion à la base de données.

5. Le fichier de données créé devient le fichier de données en cours. La fenêtre de description des rubriques du fichier de données s'ouvre automatiquement. Vous pouvez décrire les rubriques du fichier de données.



<a name="NOTE2_2"></a>


### Importer une description de fichier de données prédéfinie
<a name="importer_une_description_fichier_donnees_predefinie_ELTPARAGRAPHE000099"></a>Pour importer une description de fichier de données prédéfinie : 

1. Sous le volet "Analyse", dans le groupe "Création", cliquez sur "Nouveau fichier". L'assistant de création d'un fichier de données se lance.

2. Sélectionnez l'option "Sélectionner une description parmi des fichiers de données prédéfinis".

3. Choisissez le ou les fichiers de données prédéfinis à créer. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Analyse_Fichierpredef.gif)
L'assistant propose également les fichiers de données associés qui peuvent être ajoutés sous forme de liens.

4. Pour chaque fichier de données, sélectionnez les rubriques à conserver. Ces rubriques pourront être modifiées par la suite.

5. Le(s) fichier(s) de données prédéfini(s) sont automatiquement insérés dans l'analyse en cours. Les liaisons entre les fichiers prédéfinis créés sont automatiquement réalisées.

## Couleur des descriptions des fichiers de données
<a name="couleur_des_descriptions_des_fichiers_donnees_ELTTEXTE000296"></a>
Pour identifier rapidement les types des fichiers de données sous l'éditeur d'analyses, les couleurs suivantes sont utilisées : 

- Bleu pour les fichiers de données provenant d'un Connecteur Natif.

- Orange pour les fichiers de données HFSQL Client/Serveur.

- Marron pour les fichiers de données HFSQL Classic.

- Vert pour les fichiers de données ODBC. 

- Gris pour les fichiers de données Hyper File 5.5.

## Sécurité d'un fichier de données HFSQL
<a name="securite_fichier_donnees_hfsql_ELTTEXTE000320"></a>
Pour assurer la sécurité d'un fichier de données HFSQL : 

1. Affichez la description du fichier de données : 

	- Sélectionnez le fichier de données sous l'éditeur d'analyses. 

	- Dans le menu contextuel, sélectionnez l'option "Description du fichier de données".




2. Dans l'onglet "Détail" : 

	- Activez l'option de cryptage du fichier de données, des index, des mémos. Plusieurs modes de cryptage sont disponibles. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Base_HFSQLCS_Securite%20-%20HC%20N%B0001.gif)
Par défaut, l'option "Automatique (RC5 16 boucles si mot de passe)" est sélectionnée lors de la création d'un fichier de données. Cette option permet de crypter automatiquement le fichier de données si un mot de passe est défini avec la fonction [HPasse](../WDLang4/3044108.md) sur le fichier de données.

	- Activez l'option "Activer la sécurité renforcée ".
			Remarque : Par défaut, cette option est sélectionnée lors de la création d'un fichier de données. Cette option permet de demander le mot de passe à chaque modification automatique du fichier de données.




3. Générez l'analyse. 

4. Spécifiez un mot de passe pour le fichier de données par programmation avec la fonction [HPasse](../WDLang4/3044108.md). 

5. Chiffrez la connexion à la base de données dans les connexions décrites dans l'analyse, ou par programmation grâce à la propriété [Cryptage](../Proprietes/2512102.md).




Remarque : Cryptage AES (algorithme de cryptage AES 128 bits, 192 bits et 256 bits avec un mode d'opération OFB) : 

- Si le fichier de données est crypté selon l'algorithme AES, il est nécessaire d'utiliser la même méthode pour crypter les mémos et les index. 

- Si un cryptage selon l'algorithme AES est réalisé, l'option "Activer la sécurité renforcée " est automatiquement appliquée.

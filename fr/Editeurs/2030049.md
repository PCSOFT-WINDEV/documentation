## Présentation
<a name="presentation_ELTTEXTE000133"></a>
Les projets créés actuellement sont souvent multicibles.

Par exemple, pour un ERP destiné à fonctionner sous Windows, il est fort probable qu'en plus de l'application qui sera le socle de la solution applicative, des commerciaux seront équipés de mobiles (Android ou iOS), les magasiniers utiliseront des terminaux mobiles pour la gestion des stocks et qu'un site Intranet et un site Internet seront mis en place.

Toutes ces informations sont regroupées dans un projet global.

Par exemple, depuis WINDEV, il est possible d'indiquer les aspects Internet, Intranet, et Mobile. La documentation technique sera exhaustive dès sa conception.

Dans les phases d'utilisation de l'environnement, il est possible de visualiser depuis chaque environnement les éléments de chaque cible. Un projet sous WINDEV affiche les vignettes des pages WEBDEV et les fenêtres WINDEV Mobile par exemple. Cliquer sur une page WEBDEV depuis l'éditeur de projet WINDEV ouvre la page WEBDEV (WEBDEV doit être installé sur le poste). Si vous avez installé WINDEV avec WINDEV Mobile et WEBDEV inclus dans un unique produit, la manipulation est encore plus simple : les éditeurs s'adaptent automatiquement à l'élément ouvert, sans changer d'environnement. 

Sont également partagés : les documents, les descriptions de modélisations, les composants, les classes, ...

## Création d'un projet commun
<a name="creation_projet_commun_ELTTEXTE000157"></a>
Quelque soit le produit avec lequel un projet a été créé, il est possible de l'ouvrir avec les autres produits.

Lorsque vous ouvrez un projet dans un produit différent de celui qui a permis de le créer, un assistant s'affiche, permettant de créer une [configuration de projet](../Editeurs/9000030.md) spécifique au produit utilisé.

Par exemple, si un projet WINDEV est ouvert avec WEBDEV, vous pouvez créer une configuration de projet nommée "SiteWEB", permettant de regrouper tous les éléments nécessaires au site WEBDEV.

## Manipulation d'un projet commun
<a name="manipulation_projet_commun_ELTTEXTE000181"></a>


### Contenu du volet "Explorateur de projet"
<a name="contenu_volet_explorateur_projet_ELTPARAGRAPHE000042"></a>Si un projet a été ouvert sous différents produits (par exemple WINDEV et WEBDEV), le volet "Explorateur de projet" affiche uniquement les types d'éléments correspondant aux produits avec lesquels le projet a été ouvert.

Par exemple, si un projet a été manipulé uniquement avec WINDEV et WEBDEV, les éléments affichés seront : "Fenêtres", "Pages", "Modèles de fenêtres", "Modèles de pages", ...
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P2_Format_compatible%20WB%20-%20HC%20N%B0001.gif)


**Pour profiter de cette fonctionnalité dans vos projets existants** : 

- sous le volet "Accueil", dans le groupe "Général", déroulez "Enregistrer" et sélectionnez "Enregistrer tout". 

- ré-ouvrez ce projet : seuls les éléments appartenant aux produits utilisés pour ouvrir ce projet seront affichés.

## Impression de la documentation technique d'un projet commun
<a name="impression_documentation_technique_projet_commun_ELTTEXTE000205"></a>
Quelque soit le produit manipulant le projet, il est possible d'imprimer une [documentation technique complète du projet](../Editeurs/2015001.md) : sous le volet "Accueil", dans le groupe "Général", déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_imprimer.gif)
 et sélectionnez "Imprimer la documentation technique du projet".

Cette documentation technique contient par défaut les renseignements concernant tous les éléments du projet commun (fenêtres, fenêtres mobiles et pages). Vous pouvez bien sûr sélectionner uniquement les éléments nécessaires.




## Options avancées de l'installeur
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000229"></a>
L'assistant d'installation permet de renseigner des options avancées pour l'installeur. Ces options avancées permettent de :

- [spécifier les options de gestion du framework de l'installeur](#NOTE2_1).

- [personnaliser l'installeur](#NOTE3_1). 

- [spécifier les options de gestion de l'UAC pour l'installeur](#NOTE4_1). 

- [spécifier le plug-in à utiliser](#NOTE5_1). 

- [spécifier les polices à installer](#NOTE6_1). 




Remarque : Ces options avancées sont disponible à la fin de l'assistant de génération de l'installation (bouton "Options avancées de l'installeur" dans l'étape "Support d'installation").

## Framework de l'installeur
<a name="framework_installeur_ELTTEXTE000253"></a>
L'onglet "Framework WINDEV" permet de spécifier le mode de gestion du framework de l'installeur. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Options_avancees_installeur%20-%20HC%20N%B0001.gif)
Il est possible de spécifier les options concernant l'installation du framework : 

- **Copier le framework WINDEV avec l'installation**. Dans ce cas, la taille du programme d'installation pourra être relativement importante. 
	Si cette option est choisie : 

	- l'onglet "Personnaliser le programme de l'installeur" permet si nécessaire de choisir l'installeur utilisé (voir le paragraphe [Personnaliser le programme de l'installeur](#NOTE3_1)). 

	- l'onglet "DLL de l'installateur" permet de voir les DLL prises en compte et si nécessaire d'en ajouter ou d'en supprimer. Cette option peut être nécessaire lors de l'utilisation d'un installeur personnalisé. 




- **Télécharger le framework WINDEV depuis un serveur FTP**. Dans ce cas, l'utilisateur devra avoir un accès à Internet. 
	Si cette option est choisie, l'onglet "Paramètres FTP" permet de définir les options du serveur FTP utilisé (voir le paragraphe [Configurer le serveur FTP](#NOTE2_3)). 

- **Utiliser le framework WINDEV présent sur le poste de l'utilisateur**. Dans ce cas, le téléchargement du framework sera déclenché uniquement si le framework WINDEV n'est pas trouvé sur le poste d'installation. 




Cet onglet permet également de gérer le nettoyage des anciens frameworks. L'option "Supprimer les anciens frameworks présents dans le répertoire de destination" permet de supprimer toutes les anciennes versions du framework WINDEV présent dans le ou les répertoires d'installation.
<a name="NOTE2_3"></a>


### Configurer le serveur FTP (mode Télécharger le framework WINDEV depuis un serveur FTP)
<a name="configurer_serveur_ftp_mode_telecharger_framework_windev_depuis_serveur_ftp_ELTPARAGRAPHE000068"></a>L'onglet "Paramètres FTP" permet de : 

- Configurer le serveur FTP utilisé pour le téléchargement du framework 

- Traduire les messages de l'outil de téléchargement. 




**Serveur FTP :**
 
Par défaut, le serveur FTP de PC SOFT est utilisé pour le téléchargement du framework WINDEV. 

Si un FTP privé est utilisé, il est possible de paramétrer : 

- l'adresse du serveur FTP, 

- le port de connexion, 

- le mode de connexion : connexion passive, connexion anonyme, ...




**Traduction** : 
Le bouton "Traduire" permet de traduire ou de modifier les messages affichés par l'outil de téléchargement des DLL.

## Personnaliser le programme de l'installeur
<a name="personnaliser_programme_installeur_ELTTEXTE000283"></a>
L'onglet "Personnalisation de l'installation" permet de sélectionner l'installeur à utiliser. 

Dans le cas d'une installation individuelle, l'installateur utilisé par le client final peut être personnalisé. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Options_avancees_installeur%20-%20HC%20N%B0002.gif)


Dans le cas d'une installation avec mise à jour en réseau, il est possible de personnaliser : 

- l'installeur utilisé lors de l'installation cliente. 

- l'installeur utilisé pour le déploiement de la version de référence. 




Dans tous les cas, il est possible de : 

- Utiliser l'installateur standard (choix par défaut). 
	L'installeur standard est l'installeur fourni par défaut avec WINDEV pour installer les applications WINDEV. 

- Utiliser un installeur personnalisé. 
	Le programme d'installation standard (appelé "WDSetup") est fourni avec les exemples de WINDEV. Si vous voulez personnaliser le programme d'installation, il suffit de modifier le projet "WDSetup".
	 Il suffit alors de spécifier le répertoire de l'installeur à utiliser. 




**Remarque** : Pour personnaliser le programme d'installation (sous WINDEV) : 

1. Ouvrez le projet "WDSetup". 
	Par défaut, ce projet est présent dans le sous-répertoire "Exemples" du répertoire d'installation de WINDEV.
	Remarque : Cet exemple est également disponible dans les "Exemples complets" affichés dans la page d'accueil.

2. Personnalisez ce projet (modification de l'aspect des fenêtres, gestion de langues supplémentaires, ...).

3. Créez un nouvel exécutable : cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_generer.gif)
 parmi les boutons d'accès rapide.


**Remarque** : Le nom de cet exécutable n'a pas d'importance.

## Gestion de l'UAC pour l'installeur
<a name="gestion_uac_pour_installeur_ELTTEXTE000307"></a>
Par défaut, l'installation d'une application WINDEV déclenche l'UAC. 

L'onglet "UAC" permet de modifier ce mode de fonctionnement : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Options_avancees_installeur%20-%20HC%20N%B0003.gif)


- pour les installations individuelles : l'UAC est désactivée pour l'installation sur le poste de l'utilisateur final. 

- pour les installations réseau avec mise à jour automatique. Dans ce cas, l'UAC peut être désactivée : 

	- pour l'installeur de l'utilisateur final, 

	- pour l'installeur de la version de référence, 

	- pour les deux. 







Attention : L'option "L'installeur NE nécessite PAS les droits administrateur" entraîne certaines restrictions : pas d'installation de fichiers dans les répertoires systèmes (Program Files, Windows...), pas de modifications des fichiers et paramètres systèmes (win.ini, path, ...). 

L'installation échouera si les droits ne sont pas suffisants. Dans la mesure du possible, l'assistant d'installation guidera l'utilisateur dans ses choix pour lui permettre d'installer l'application. 

Remarque : Il est également possible de configurer la prise en charge de l'UAC pour l'installeur dans l'étape ["Répertoire d'installation"](../Editeurs/2028047.md) de l'assistant de création d'installation.

## Plug-in à utiliser
<a name="plugin_utiliser_ELTTEXTE000331"></a>
Dans un programme d'installation, il peut être nécessaire de personnaliser certaines actions, d'ajouter des plans dans l'assistant, d'ajouter de nouveaux traitements, ...

Il est possible de réaliser un plug-in et de modifier ainsi le programme d'installation standard. Ce plug-in est réalisé à partir de l'exemple didactique "WD Plugin WDSETUP" livré avec WINDEV. Pour plus de détails, consultez [Personnaliser le programme d'installation (Plug-in)](../Editeurs/2028109.md). 

L'onglet "Plug-in" permet d'indiquer le chemin du plug-in à inclure dans le programme d'installation standard.


![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Options_avancees_installeur%20-%20HC%20N%B0004.gif)

## Polices à installer
<a name="polices_installer_ELTTEXTE000355"></a>
Dans un programme d'installation, il peut être nécessaire d'installer certaines polices de caractères utilisées par l'application. 
Il suffit d'ajouter les fichiers TTF ou OTF correspondant aux polices à installer. 

Remarque : Veuillez vérifier que vous possédez bien le droit de diffuser ces polices. 

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Options_avancees_installeur%20-%20HC%20N%B0005.gif)




## Modèle Logique des Données (MLD)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000154"></a>
Une Modèle Logique des Données (MLD) permet de décrire les structures de données (fichiers de données, rubriques, ...) utilisées dans une application WINDEV, WEBDEV ou WINDEV Mobile.

**Remarques** :

- Les termes "MLD" et "Analyse" sont utilisés indifféremment pour définir la structure de la base de données associée à un projet.

- Pour plus de détails sur les manipulations d'un MLD, consultez [Manipulations d'un MLD](../Editeurs/2011006.md).




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Création d'un MLD
<a name="creation_mld_ELTTEXTE000184"></a>
Pour créer un MLD : 

1. Cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_nouveau.gif)
 parmi les boutons d'accès rapide. 

	- La fenêtre de création d'un nouvel élément s'affiche : cliquez sur "Données" puis sur "Analyse".

	- L'assistant de création d'une analyse se lance.




2. Spécifiez dans la première étape de l'assistant :

	- le ***nom et le répertoire de l'analyse***. L'analyse correspond à un fichier ".WDA". Par défaut, ce fichier sera créé dans le répertoire de l'analyse du projet (répertoire &lt;Nom du projet&gt;.ANA). Ce répertoire doit être accessible en lecture et en écriture.

	- si l'analyse doit être associée au projet en cours à la place de l'analyse actuelle (cas de la création d'une analyse dans un projet déjà associé à une analyse). 

	- la ***description de l'analyse*** décrivant succinctement le sujet de l'analyse. 




3. L'étape suivante de l'assistant permet de spécifier le type de la base de données manipulée par le projet : 

	- HFSQLDrive (solution payante). Pour plus de détails sur HFSQLDrive, consultez [HFSQLDrive, votre base de données dans le Cloud en 3 clics](https://pcscloud-drive.net/FR/HFSQLDrive/index.awp).

	- PCSCloud (solution payante). Pour plus de détails sur PCSCloud, consultez [Cloud pour base de données HFSQL](https://pcscloud.net/FR/cloud_applications.awp).

	- HFSQL Client/Serveur,

	- HFSQL Classic, 

	- Bases de données externes : SQL Server, Oracle, MySQL, ...
			Remarque : Si l'analyse doit manipuler une base de données externe, l'accès natif correspondant doit être installé sur le poste. 




4. Si la base de données est différente de HFSQL Classic, l'assistant permet de définir la connexion à la base de données (nom, libellé) et de saisir les paramètres avancés (serveur, utilisateur et mot de passe). Ces informations peuvent être également modifiées par la suite, dans l'éditeur d'analyses.

5. Validez. L'analyse créée devient l'analyse en cours.

6. Créez l'ensemble des éléments (fichiers de données, rubriques et liaisons) de votre analyse.
	**Remarque** : Pour plus de détails, consultez [Fichiers de données de l'analyse](../Editeurs/2011010.md), [Rubriques d'un fichier de données](../Editeurs/2011009.md) et [Liaisons de l'analyse](../Editeurs/2011008.md).

7. Pour utiliser l'analyse dans votre projet, générez l'analyse : sous le volet "Analyse", dans le groupe "Analyse", déroulez "Génération" et sélectionnez "Génération". Pour plus de détails, consultez [Génération d'une analyse](../Editeurs/2011012.md).



<a name="NOTE_Optimisation_Analyse"></a>


### Optimisation de l'analyse
<a name="optimisation_analyse_ELTPARAGRAPHE000092"></a>**A partir de la version 20, le format de stockage de l'analyse est optimisé**. Toute nouvelle analyse créée avec la version 20 (ou supérieur) utilise ce nouveau format. 

Il est possible d'enregistrer les analyses existantes avec ce nouveau format. 
**Attention : Une analyse utilisant ce nouveau format sera incompatible avec les versions antérieures à la version 19.**

Pour utiliser le nouveau format sur vos analyses existantes : 

1. Affichez la fenêtre de description de l'analyse (option "Description de l'analyse" du menu contextuel du graphe de l'analyse). 

2. Dans l'onglet "Détail", cliquez sur le bouton "Enregistrer l'analyse dans un format plus compact".

## Création des différentes fenêtres et pages du pattern
<a name="creation_des_differentes_fenetres_pages_pattern_ELTTEXTE000180"></a>


### Principe
<a name="principe_ELTPARAGRAPHE000082"></a>Pour connaître tous les éléments à réaliser pour créer un Pattern RAD pour un projet complet, il est nécessaire de repartir de l'analyse du projet : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=RAD_Pattern.gif)


1. Cachons tout d'abord le fichier de relation RADFichierD : Les fenêtres ou pages à réaliser sont les suivantes :

	- Une fiche pour RADFichierA.

	- Une fiche pour RADFichierB. Une combo et un bouton Vision plus (ou une combo popup) doit permettre de sélectionner un élément du fichier RADFichierA.

	- Une fiche pour RADFichierC. Une combo et un bouton Vision plus (ou une combo popup) doit permettre de sélectionner un élément du fichier RADFichierB.

	- Une table pour RADFichierA.

	- Une table pour RADFichierB.

	- Une table pour RADFichierC.




2. Cachons ensuite le fichier RADFichierA : Les fenêtres ou pages à réaliser sont les suivantes :

	- Une fiche pour RADFichierB. Cette fiche doit montrer la relation avec RADFichierD. C'est donc une Fiche + Table.  

	- Une fiche pour RADFichierC (cet élément a déjà été cité).




3. Il faut ensuite faire si nécessaire les boutons de liaison.

4. Il ne reste plus qu'à faire le menu général.



<a name="NOTE2_2"></a>


### Norme à suivre
<a name="norme_suivre_ELTPARAGRAPHE000105"></a>Les différentes pages concernant les fenêtres et les pages des pattern RAD présentent différentes règles applicables à la conception des pages et des fenêtres du Pattern RAD. Ces règles ont été utilisées pour concevoir les différents Patterns livrés avec WINDEV, WEBDEV ou WINDEV Mobile 16.

Ces règles sont une recommandation. Vous pouvez évidemment mettre en place et utiliser vos propres normes.


## Installation d'une application : installation monoposte ou réseau ? 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000172"></a>
WINDEV propose plusieurs types d'installation pour une application WINDEV : 

- [Installation individuelle](#NOTE2_1).

- Installation avec mise à jour automatique : 

	- [Installation réseau](#NOTE2_2).

	- [Installation HTTP](#NOTE2_3).

	- [Installation multi-sites](#NOTE2_4).

	- [Installation Push](#NOTE2_5).




- Installation dans un Store Privé. 




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Les différents types d'installation
<a name="les_differents_types_installation_ELTTEXTE000196"></a>


### Installation individuelle
<a name="installation_individuelle_ELTPARAGRAPHE000040"></a>L'installation individuelle consiste à créer un programme d'installation qui sera installé sur chaque poste utilisateur grâce à un support d'installation. 

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P5_Installation_classique.gif)


Ce mode d'installation est compatible aussi bien avec : 

- des fichiers de données au format HFSQL Classic installés en local sur le poste utilisateur, 

- des fichiers de données au format HFSQL Classic installés en réseau et accessibles depuis le poste utilisateur,

- des fichiers de données au format HFSQL Client/Serveur installés sur un serveur. 



<a name="NOTE2_2"></a>


### Installation réseau (avec mise à jour automatique)
<a name="installation_reseau_avec_mise_jour_automatique_ELTPARAGRAPHE000055"></a>L'installation réseau consiste à créer un programme d'installation qui sera installé sur un poste Serveur. 

Tous les postes utilisateurs voulant installer l'application lanceront directement l'installation de l'application par le réseau. 

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P5_Installation_reseau.gif)


Si une mise à jour est disponible sur le serveur d'application, la mise à jour de l'application sur le poste utilisateur sera proposée au lancement de l'application. 

Une installation réseau permet un contrôle fin des postes utilisateurs (versions déployées, mises à jour obligatoires, déconnexion des utilisateurs, ...). 

Ce mode d'installation est compatible aussi bien avec : 

- des fichiers de données au format HFSQL Classic installés en local sur le poste utilisateur, 

- des fichiers de données au format HFSQL Classic installés en réseau et accessibles depuis le poste utilisateur,

- des fichiers de données au format HFSQL Client/Serveur installés sur un serveur. 




Remarque : Il est possible de mettre à jour l'installation présente sur le serveur d'application même si une application cliente est en cours de mise à jour. 
<a name="NOTE2_3"></a>


### Installation HTTP (avec mise à jour automatique)
<a name="installation_http_avec_mise_jour_automatique_ELTPARAGRAPHE000080"></a>L'installation HTTP consiste à créer un programme d'installation qui sera installé sur un serveur Web. 

Tous les postes utilisateurs voulant installer l'application lanceront directement l'installation de l'application par une URL HTTP. 

Si une mise à jour est disponible sur le serveur Web, la mise à jour de l'application sur le poste utilisateur sera proposée au lancement de l'application. 

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P5_Installation_HTTP.gif)


Pour plus de détails, consultez [Installation par HTTP (Installation réseau via IP)](../Editeurs/9000026.md). 
<a name="NOTE2_4"></a>


### Installation multi-sites (avec mise à jour automatique)
<a name="installation_multisites_avec_mise_jour_automatique_ELTPARAGRAPHE000098"></a>L'installation multi-sites permet d'installer une application sur plusieurs sites différents (un site représentant par exemple une filiale d'une entreprise).

L'installation multi-sites consiste à créer un programme d'installation qui sera installé sur un serveur Web.

Tous les sites voulant installer l'application lanceront directement l'installation de l'application par une URL HTTP. Dans ce cas, le programme d'installation sera alors installé sur un poste Serveur du site.

Tous les postes utilisateurs voulant installer l'application lanceront l'installation directement par le réseau.

Cette installation combine les avantages de l'installation réseau et de l'installation HTTP en permettant un déploiement à grande échelle et un contrôle fin des installations locales.

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P5_Installation_multi_sites.gif)


Si une mise à jour est disponible sur le serveur Web, la mise à jour de l'application sur le serveur d'application est automatiquement réalisée.


<a name="NOTE2_5"></a>


### Installation réseau de type Push
<a name="installation_reseau_type_push_ELTPARAGRAPHE000118"></a>L'installation réseau consiste à créer un programme d'installation qui sera installé sur un poste Serveur. L'installation sera effectuée depuis le poste Serveur, par l'administrateur réseau sur les postes utilisateurs sélectionnés grâce à WDAdminExe.

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P5_Installation_Push.gif)


Une installation en Push peut ensuite être mise à jour soit en Push, soit dans le cadre d'une installation réseau.

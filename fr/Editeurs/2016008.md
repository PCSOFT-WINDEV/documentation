


## Styles WEBDEV
			



<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000123"></a>
Les styles WEBDEV sont des styles avancés, plus simples à utiliser que les styles CSS.

Un style WEBDEV regroupe pour un champ un ensemble de styles CSS et de caractéristiques.

Par exemple, pour un bouton, le style WEBDEV regroupe :

- L'aspect du bouton,

- L'image ou les images associées au bouton,

- Le style CSS associé au libellé du bouton.




Un champ peut être associé :

- à un ou plusieurs styles CSS. Les styles CSS utilisés par le champ sont indiqués dans les différents éléments de l'onglet "Style" de la fenêtre de description du champ. 

- à un style WEBDEV. Le style WEBDEV utilisé est sélectionné dans l'onglet "Style" de la fenêtre de description du champ. 




**Remarque** : Il est possible de modifier un style WEBDEV depuis la fenêtre de description du champ. Cette modification peut être : 

- locale au champ. Dans ce cas, la redéfinition du style est effectuée uniquement pour le champ. 

- appliquée à tous les champs utilisant le style WEBDEV.

## Gestion des styles WEBDEV
<a name="gestion_des_styles_webdev_ELTTEXTE000147"></a>


### Création et modification d'un style WEBDEV
<a name="creation_modification_style_webdev_ELTPARAGRAPHE000036"></a>La création et la modification d'un style WEBDEV peut être effectuée : 

- depuis la fenêtre de description des styles d'un champ. 

- depuis la fenêtre de gestion des styles WEBDEV. Pour plus de détails, consultez [Edition des feuilles de styles WEBDEV](../Editeurs/2016014.md). 





### Création et modification depuis la fenêtre de description d'un champ
<a name="creation_modification_depuis_fenetre_description_champ_ELTPARAGRAPHE000047"></a>La fenêtre de description des champs permet de gérer le style de ce champ (onglet "Style"). Différents éléments sont gérés par l'onglet "Style". Les éléments pouvant utiliser un style CSS sont identifiés par "(CSS)" à la fin de leur libellé. 
Les autres éléments sont gérés uniquement par le style WEBDEV.

Lors de l'affichage de l'onglet "Style", une fenêtre apparaît à droite de la fenêtre à 7 onglets. Cette fenêtre permet de : 

- **connaître le nom du style WEBDEV associé au champ**. Si aucun style WEBDEV n'est associé au champ, le libellé "Aspect libre : aucun style WEBDEV lié" apparaît. 

- **savoir si les caractéristiques de style du champ sont redéfinies par rapport au style WEBDEV**. Dans ce cas, il existe des "surcharges" de style pour ce champ. Ces surcharges peuvent être listées et supprimées si nécessaire. 

- **avoir un aperçu du style original** du champ. 




Cette fenêtre permet également d'effectuer certaines actions. Il est possible de : 

- **Choisir un style**. Cette option permet de sélectionner un style WEBDEV et de l'associer au champ en cours. Dans ce cas, le champ sera automatiquement mis à jour lorsque le style associé sera mis à jour. 

- **Ajouter ce style au projet**. Cette option permet de créer un nouveau style WEBDEV dans le projet à partir de toutes les caractéristiques actuellement définies pour le champ en cours. Le champ sera automatiquement associé à ce nouveau style WEBDEV. Il suffit alors d'indiquer le nom du nouveau style. 

- **Mettre à jour le style dans le projet**. Cette option apparaît lors de la modification du style du champ. Cette option permet de prendre en compte ces modifications dans tous les champs utilisant le style WEBDEV en cours. Le style WEBDEV est donc mis à jour. 

- **Dissocier le style**. Cette option permet de supprimer la liaison entre le style WEBDEV et le style du champ. Si le style WEBDEV évolue, le champ ne prendra pas en compte ces évolutions. 



<a name="NOTE4_2"></a>


### Editer les styles WEBDEV
<a name="editer_les_styles_webdev_ELTPARAGRAPHE000069"></a>WEBDEV offre la possibilité d'éditer les styles WEBDEV présents dans la feuille de styles associée au projet.

Pour plus de détails, consultez [Edition des styles WEBDEV](../Editeurs/2016014.md).

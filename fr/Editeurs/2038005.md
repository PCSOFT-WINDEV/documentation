


## Créer une base de sources (GDS)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000194"></a>
Pour partager un projet grâce au gestionnaire de sources, il est nécessaire de créer une base de sources. Cette base de sources doit être créée une seule fois sur un poste serveur.

Le système d'exploitation et le système de fichiers sur lequel est installé la base de sources doivent savoir gérer les fichiers de plus de 4 Go.

Il est possible de créer la base de sources : 

- sur le **GDSDrive** (solution payante, clé en main). Pour plus de détails, consultez [GDSDrive](https://pcscloud-drive.net/FR/GDSDrive/index.awp). 
	Dans ce cas, il est nécessaire de préciser : 

	- Votre identifiant (email) et mot de passe, 

	- Votre équipe. 




- sur une plateforme de développement du **PCSCLOUD** (solution payante, clé en main). Pour plus de détails, consultez [PCSCloud](https://pcscloud.net/FR/cloud_developpement.awp). 
	Dans ce cas, il est nécessaire de préciser : 

	- Le nom de votre plateforme,

	- Le nom de l'utilisateur et le mot de passe.




- **sur un serveur HFSQL Client/Serveur préalablement installé** : la base de sources sera au format [HFSQL Client/Serveur](../WDLang4/3044306.md). Dans ce cas, il est nécessaire de préciser :

	- le serveur,

	- la base de données HFSQL,

	- le port utilisé,

	- le nom et le mot de passe d'un administrateur de la base de données HFSQL. 




- **dans un répertoire partagé sur le réseau** : tous les utilisateurs du GDS doivent avoir tous les droits sur ce répertoire. La base de sources sera au format [HFSQL Classic](../CCHF/35400010.md).
	Il suffit de préciser le répertoire réseau où la base de sources doit être créée.




Si vous utilisez une solution personnelle (serveur HFSQL Client/Serveur préalablement installé ou base du GDS au format HFSQL Classic), la machine devra être fiable et sécurisée. Pour plus de détails, consultez [Base de données HFSQL Client/Serveur : Recommandations de sécurité](../WDLang4/1000017310.md)

## Quand doit-on créer une base de sources ?
<a name="quand_doiton_creer_une_base_sources_ELTTEXTE000224"></a>
**La base de sources doit être créée une seule fois.**

WINDEV, WEBDEV et WINDEV Mobile permettent de créer cette base de sources à différents moments :

- lors de l'installation de WINDEV, WEBDEV ou WINDEV Mobile.

- lors de la création d'un projet utilisant le GDS.

- [lors de l'importation d'un projet dans le gestionnaire de sources.](../Editeurs/2038002.md)

- [directement dans l'administrateur du GDS](../Editeurs/2038008.md).




Lorsque la base de sources est créée, il est possible d'importer dans cette base de sources tous les projets WINDEV, WEBDEV et WINDEV Mobile à partager.

## Passer une base de sources existante au format HFSQL Classic en mode Client/Serveur
<a name="passer_une_base_sources_existante_format_hfsql_classic_mode_clientserveur_ELTTEXTE000248"></a>
Si vous avez créé une base de sources en mode HFSQL Classic, pour l'utiliser en mode HFSQL Client/Serveur vous devez réaliser les manipulations suivantes :

1. Optimisez le format de la base de sources HFSQL Classic du GDS. Pour cela : 

	- ouvrez l'administrateur du GDS : sous le volet "GDS", dans le groupe "Base GDS", déroulez "Gérer" et sélectionnez "Administrateur du Gestionnaire de sources". 

	- dans l'administrateur, sous le volet "Outils", dans le groupe "Maintenance", sélectionnez l'option "Migrer au format le plus récent ".




2. Installez si nécessaire un serveur HFSQL sur le poste où la base de sources HFSQL Client/Serveur du GDS doit être installée.

3. Dans l'administrateur du GDS, sous le volet "Outils", dans le groupe "Maintenance", sélectionnez l'option "Passer en Client/Serveur".

4. Tous les utilisateurs peuvent se connecter à la base de sources HFSQL Client/Serveur du GDS en passant les paramètres nécessaires (serveur, base de données, port utilisé, nom et mot de passe d'un administrateur de la base de données HFSQL).

## Sauvegardes
<a name="sauvegardes_ELTTEXTE000272"></a>
Il est indispensable de faire régulièrement des sauvegardes de la base de sources du GDS. Pour cela, il suffit de copier et d'archiver régulièrement les fichiers de données correspondants au GDS.

Cette opération sera automatisée dans une prochaine version.

## Changer le mot de passe de l'administrateur (base de sources en mode Client/Serveur)
<a name="changer_mot_passe_administrateur_base_sources_mode_clientserveur_ELTTEXTE000296"></a>
Le changement du mot de passe de l'administrateur du GDS, dans une base de données Client/Serveur, doit être effectué depuis le Centre de Contrôle HFSQL de la base de données du GDS.

Il suffit de :

1. Sélectionner le serveur correspondant à la base de sources du GDS.

2. Sélectionner l'onglet "Utilisateurs".

3. Sélectionner l'administrateur dans la liste des utilisateurs et cliquer sur "Editer".

4. Modifier le mot de passe et valider.

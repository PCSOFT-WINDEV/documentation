
## Déployer un site sur un serveur de test PC SOFT
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000221"></a>
PC SOFT propose d'héberger vos sites de test. L'installation s'effectue simplement, directement depuis l'environnement. 

Ainsi, une fois votre site terminé, il est possible de le tester "en réel" directement. 

**Remarques** : 

- Le site doit être développé en WEBDEV dynamique ou en mode AWP. 

- Le site doit utiliser uniquement une base de données de type HFSQL Classic ou HFSQL Client/Serveur. 

- Le serveur utilisé est un serveur Windows. Il n'est pas possible de déployer des sites Linux. 

- Il est possible de déployer des Webservices. 

- Les types de sites suivants ne sont pas pris en charge : 

	- Les sites développés en PHP.

	- Les sites SaaS. 

	- Les sites statiques. 







**Limitation** : 100 connexions simultanées sont attribuées par compte développeur. La répartition du nombre de connexions par site ne doit pas dépasser cette limite.

## Les différentes étapes
<a name="les_differentes_etapes_ELTTEXTE000251"></a>
Les différentes étapes de l'installation d'un site dynamique WEBDEV sur le serveur d'hébergement de test de PC SOFT sont les suivantes :

1. [Création si nécessaire d'un compte développeur](#NOTE3_1).

2. [Création si nécessaire d'un compte d'hébergement](#NOTE4_1).

3. [Saisie des paramètres du site](#NOTE5_1).

4. [Sélection des fichiers à installer](../Editeurs/2028058.md). 

5. [Modification automatique](../Editeurs/2028048.md). 

6. [Paramètres du site](../Editeurs/2028051.md). 

7. Transfert des fichiers et installation. 
	Des messages d'erreur peuvent apparaître à diverses étapes de l'installation à distance. Pour plus de détails, consultez [Erreurs en cas d'installation par FTP](../Editeurs/2028057.md).

8. Lorsque l'installation est terminée, l'assistant fournit la ligne de commande permettant de lancer le site installé. Cette ligne peut être copiée si nécessaire dans le presse-papiers grâce au menu contextuel.

## Création d'un compte développeur
<a name="creation_compte_developpeur_ELTTEXTE000275"></a>


### Compte développeur
<a name="compte_developpeur_ELTPARAGRAPHE000070"></a>Un compte développeur peut être créé : 

- soit pour participer aux forums PC SOFT.

- soit pour annoter la documentation en ligne.

- soit pour déployer un site sur le serveur d'hébergement de test de PC SOFT. Un compte d'hébergement de test est également nécessaire. 




Si vous avez déjà créé un compte développeur, vous pouvez réutiliser ce compte pour installer vos sites.


### Eléments nécessaires
<a name="elements_necessaires_ELTPARAGRAPHE000081"></a>**Les éléments nécessaires pour la création du compte développeur** sont les suivants :

1. Une adresse email valide. 
	**Attention** : Lors de la création du compte développeur, l'adresse email fournie ne peut être utilisée qu'une seule fois. Une même adresse email ne peut être utilisée pour créer plusieurs comptes développeur.

2. Le numéro de série de votre produit WEBDEV. 




Lors de la validation de la création du compte développeur, un email de confirmation est envoyé avec un lien permettant d'activer le compte.

## Création d'un compte d'hébergement de test
<a name="creation_compte_hebergement_test_ELTTEXTE000305"></a>


### Compte d'hébergement de test
<a name="compte_hebergement_test_ELTPARAGRAPHE000097"></a>Un compte d'hébergement de test peut être créé pour déployer un site sur le serveur d'hébergement de test de PC SOFT. 

Si vous avez déjà créé un compte d'hébergement de test, vous pouvez réutiliser ce compte pour déployer vos sites.


### Eléments nécessaires
<a name="elements_necessaires_ELTPARAGRAPHE000104"></a>**Les éléments nécessaires pour la création du compte d'hébergement** sont les suivants :

1. Un compte développeur PCSOFT.

2. Un nom de domaine. Ce nom de domaine sera unique. Il correspond à la racine de tous les sites que vous allez déployer.
Par exemple : le domaine PCSOFT donnera comme racine http:\\\\pcsoft.webdev-test.com\\MonsiteTest
	**Attention** : Le nom du domaine doit comporter au minimum 5 caractères.




Lors de la validation de la création du compte d'hébergement, un email de confirmation est envoyé.






### Ressources associées à un compte d'hébergement
<a name="ressources_associees_compte_hebergement_ELTPARAGRAPHE000120"></a>Une fois le compte créé, vous avez accès aux ressources suivantes :

- Un accès FTP personnalisé. 
	L'accès par FTP à votre domaine se fait avec : 

	- le nom du domaine complet (par exemple "pcsoft.webdev-test.com").

	- le nom du domaine (dans notre exemple "pcsoft").

	- le mot de passe que vous aurez renseigné à la création du compte.
			Vous pourrez ainsi envoyer par FTP des fichiers en complément à votre site.




- Un outil Web sécurisé pour administrer vos sites ([https://www.webdev-test.com/WDAdminWeb](https://www.webdev-test.com/WDAdminWeb)). Vous devez saisir votre nom de domaine et votre mot de passe pour vous connecter. Vous pourrez ainsi gérer les différents sites que vous aurez installés :

	- modifier le nombre de connexions maximum autorisé

	- supprimer un site

	- modifier la durée limite pour une déconnexion automatique

	- activer/désactiver la gestion des LOGs pour la gestion des statistiques de fréquentation

	- ...
			Ces options sont identiques à celles disponibles dans le cas d'un déploiement réel.

## Paramètres du site
<a name="parametres_site_ELTTEXTE000341"></a>
Une fois le compte développeur et le compte d'hébergement créés, vous devez renseigner dans l'assistant WEBDEV : 

- le nom de domaine,

- le mot de passe,


et valider pour procéder à l'installation du site. 

Le bouton "Paramètres avancés" permet de définir les paramètres de déploiement : 

- Dans l'onglet "Général", indiquez :

	- le nom de déploiement du site (par défaut, le nom du projet).

	- le nom du domaine associé en cas de gestion d'un fichier sitemap.

	- le nom du dossier contenant les fichiers de données (par défaut, le nom du projet).

	- le nom du profil de déploiement (renseigné par défaut) afin de sauvegarder les paramètres d'installation.




- Dans l'onglet "Avancé", indiquez : 

	- les paramètres FTP,

	- les paramètres de proxy HTTP/HTTPS.

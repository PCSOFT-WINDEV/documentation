


## Collage spécial 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000166"></a>
La commande **Collage Spécial** permet de choisir explicitement le format sous lequel le contenu du presse-papiers doit être collé dans l'éditeur de fenêtres ou de pages. 

Par exemple, si vous copiez un fichier de données de l'analyse, la commande **Collage Spécial** vous permet au choix de :

- Créer une table affichant les enregistrements du fichier de données copié.

- Copier le nom du fichier de données dans un champ Libellé.

- Afficher la représentation graphique du fichier de données copié dans un champ Image et créer un fichier Bitmap (extension ".BMP").




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000190"></a>
Pour utiliser la commande Collage Spécial :

1. Sélectionnez l'élément à coller. 

2. Copiez l'élément à coller : 

	- sous WINDEV, WEBDEV ou WINDEV Mobile : sous le volet "Accueil", dans le groupe "Edition", cliquez sur "Copier". 

	- option "Édition .. Copier" du logiciel utilisé. 

	- Ctrl + C.




3. Affichez la fenêtre ou la page où l'élément doit être collé.

4. Sous le volet "Accueil", dans le groupe "Edition", cliquez sur "Coller" et sélectionnez "Collage spécial".

5. Sélectionnez le format sous lequel l'élément copié doit être collé (voir ci-dessous).

6. Validez le collage.

## Notes
<a name="notes_ELTTEXTE000214"></a>
Les différents formats proposés par WINDEV, WEBDEV et WINDEV Mobile pour le collage spécial sont :

- **Fichier Windows** : Un ou plusieurs fichiers ont été mis dans le presse-papiers. Si cette option est sélectionnée, un ou plusieurs champs vont être créés dans la fenêtre en cours. Le type des champs créés dépend du type des fichiers copiés. 
	Par exemple, si un fichier image est copié depuis l'explorateur, un champ Image affichant l'image sera créé dans la fenêtre en cours. Le champ Image est associé au fichier image copié (chemin d'origine).

- **Fichier ou rubrique de l'analyse** : Un ou plusieurs fichiers de données ou rubriques de l'analyse ont été mis dans le presse-papiers. Si cette option est sélectionnée, un ou plusieurs champs vont être créés dans la fenêtre en cours. 
	Par exemple, si la description du fichier de données "Client" est copiée, une table listant les clients sera créée dans la fenêtre en cours.

- **Groupe d'objets WINDEV, WEBDEV ou WINDEV Mobile** : Un ou plusieurs éléments de WINDEV, WEBDEV ou WINDEV Mobile (une requête, un champ, ...) ont été mis dans le presse-papiers. Si cette option est sélectionnée, un ou plusieurs champs vont être créés dans la fenêtre ou la page en cours. 
	Par exemple, si un bouton présent dans un dictionnaire WINDEV est copié, ce bouton sera créé dans la fenêtre en cours.

- **Image** : Une ou plusieurs images ont été mises dans le presse-papiers. Si cette option est sélectionnée :

	- Un ou plusieurs champs vont être créés dans la fenêtre en cours. Ces champs afficheront les images copiées. Les images copiées **ne seront pas automatiquement associées** à la palette de couleurs.

	- Un ou plusieurs fichiers Bitmap (extension ".BMP") vont être créés. Un sélecteur de fichiers permet de spécifier le nom et le chemin du fichier.
			Par exemple, si une image affichée dans un logiciel de traitement d'images ("Photoshop" par exemple) est copiée :

		- un champ Image affichant l'image copiée sera créé dans la fenêtre en cours,

		- un fichier Bitmap (extension ".BMP") sera créé. Un sélecteur de fichiers permet de spécifier le nom et le chemin du fichier.




- **Image avec palette** : Une ou plusieurs images ont été mises dans le presse-papiers. Si cette option est sélectionnée :

	- Un ou plusieurs champs vont être créés dans la fenêtre en cours. Ces champs afficheront les images copiés. Les images copiées **seront automatiquement associées** à la palette de couleurs.

	- Un ou plusieurs fichiers Bitmap (extension ".BMP") vont être créés. Un sélecteur de fichiers permet de spécifier le nom et le chemin du fichier.
			Par exemple, si une image affichée dans un logiciel de traitement d'images ("Photoshop" par exemple) est copiée :

		- un champ Image affichant l'image copiée sera créé dans la fenêtre en cours,

		- un fichier Bitmap (extension ".BMP") sera créé. Un sélecteur de fichiers permet de spécifier le nom et le chemin du fichier.




- **Image (métafichier amélioré)** : Une ou plusieurs images ont été mises dans le presse-papiers. Si cette option est sélectionnée :

	- Un ou plusieurs champs vont être créés dans la fenêtre en cours.

	- Un ou plusieurs fichiers de type métafichier (extension ".EMF") vont être créés. Un sélecteur de fichiers permet de spécifier le nom et le chemin du fichier.
			Par exemple, si une image affichée dans un logiciel de traitement d'images ("Visio" par exemple) est copiée :

		- un champ Image affichant l'image copiée sera créé dans la fenêtre en cours,

		- un fichier de type métafichier (extension ".EMF") sera créé. Un sélecteur de fichiers permet de spécifier le nom et le chemin du fichier.




- **Texte** : Du texte a été mis dans le presse-papiers. Si cette option est sélectionnée, un champ Libellé multiligne sera créé dans la fenêtre en cours. Ce champ affichera le texte copié **sans** sa mise en forme.
	Par exemple, si du texte saisi sous Word a été copié, un champ Libellé multiligne sera créé dans la fenêtre en cours. Ce champ affichera le texte copié **sans**sa mise en forme.

- **Texte au format RTF** : Du texte mis en forme a été mis dans le presse-papiers. Si cette option est sélectionnée, un champ Libellé multiligne au format RTF sera créé dans la fenêtre en cours. Ce champ affichera le texte copié **avec** sa mise en forme.
	Par exemple, si du texte saisi en gras et en rouge sous Word a été copié, un champ Libellé multiligne au format RTF sera créé dans la fenêtre en cours. Ce champ affichera le texte copié **avec** sa mise en forme (en gras et en rouge).

- **Sélection de l'éditeur de fenêtres** : Un ou plusieurs éléments de l'éditeur de fenêtres (champ de saisie, bouton, fenêtre, ...) ont été mis dans le presse-papiers. Si cette option est sélectionnée, le ou les éléments copiés vont être créés sous l'éditeur de fenêtres. 
	Par exemple, si une sélection de champs a été copiée, ces champs sélectionnés seront créés dans la fenêtre en cours.
	**Remarque** : Cette option correspond à un "copier/coller" standard de champs.

- **Sélection de l'éditeur de pages** : Un ou plusieurs éléments de l'éditeur de pages (champ de saisie, bouton, ...) ont été mis dans le presse-papiers. Si cette option est sélectionnée, le ou les éléments copiés vont être créés sous l'éditeur de pages. 
	Par exemple, si une sélection de champs a été copiée, ces champs sélectionnés seront créés dans la page en cours.
	**Remarque** : Cette option correspond à un "copier/coller" standard de champs.

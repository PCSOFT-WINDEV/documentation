
## Configuration d'un serveur BaaS (Groupware utilisateur et Store Privé)
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000168"></a>
Plusieurs fonctionnalités de WINDEV, WEBDEV et WINDEV Mobile utilisent un serveur BaaS : 

- Groupware utilisateur en mode BaaS (WINDEV, WEBDEV, WINDEV Mobile), 

- Store Privé pour les applications WINDEV et WINDEV Mobile. 




Le serveur BaaS utilisé par ces fonctionnalités peut correspondre : 

- à une plateforme de développement du **PCSCLOUD** (solution payante, clé en main). Pour plus de détails, consultez [PCSCloud](https://pcscloud.net/FR/cloud_developpement.awp). 

- à un serveur que vous configurez vous-même. 




Cette page d'aide présente les différentes étapes permettant de configurer un serveur BaaS : 

- [Etape 1 : Installation d'un serveur HFSQL](#NOTE2_1). 

- [Etape 2 : Installation du serveur réservé BaaS](#NOTE3_1).




Remarques : 

- Un serveur Web (IIS, Apache ou autre) doit être installé sur le poste sur lequel va être installé le serveur réservé. Le serveur Web doit être accessible avec une authentification basique. 

- Dans le cas des applications mobiles (Store Privé ou Groupware utilisateur), le serveur Web doit : 

	- répondre au protocole HTTPS avec un certificat émis par une autorité de confiance reconnue des appareils mobiles iOS et Android,

	- autoriser pour les réponses HTTPS, une 'longueur maximale du contenu' supérieure à la taille des installations (IPA & APK).




- Il est possible d'installer le serveur HFSQL et le serveur réservé sur deux postes différents. 

- Le serveur réservé peut être installé sur un serveur Web hébergeant également un serveur d'application WEBDEV.

## Etape 1 : Installation du serveur HFSQL
<a name="etape_1_installation_serveur_hfsql_ELTTEXTE000192"></a>
Le programme d'installation du serveur HFSQL est disponible dans le package d'installation de WINDEV ou WINDEV Mobile. Si vous ne possédez pas ce package d'installation, l'installation du serveur HFSQL est également disponible sur le site de PC SOFT ([http://www.pcsoft.fr](https://www.pcsoft.fr)).

Pour installer le serveur HFSQL depuis le package d'installation de WINDEV/WINDEV Mobile :

1. Lancez le programme d'installation de WINDEV/WINDEV Mobile.

2. Choisissez l'option "Installation du serveur HFSQL".

3. Sélectionnez ensuite l'option "Installer ou mettre à jour un serveur HFSQL".

4. Acceptez la licence.

5. Choisissez la plateforme (option "Pour Windows sur cette machine").

6. Si des serveurs HFSQL sont déjà installés sur le poste en cours, choisissez l'option "Installer un nouveau serveur".

7. Sélectionnez le répertoire d'installation et indiquez le nom du serveur et le port. Par défaut, le port utilisé est 4900. 

8. Indiquez le nom et le mot de passe de l'utilisateur ayant les droits d'administration qui sera créé par défaut sur le serveur HFSQL. Par défaut, cet utilisateur est "Admin" sans mot de passe.

9. Installez si nécessaire le Centre de Contrôle HFSQL si celui-ci n'est pas présent ou accessible depuis votre poste. 

10. L'assistant propose de paramétrer l'envoi de notifications afin de connaître les dysfonctionnements du serveur en tant réel. Sélectionnez l'option voulue. 

11. L'assistant propose d'activer l'authentification par Active Directory. Activez cette option si nécessaire.

12. Validez (ou non) la participation à l'amélioration du produit installé en autorisant la collecte d'informations relatives à l'utilisation du produit. Cette collecte, facultative et anonyme, permet à PC SOFT d'améliorer les fonctionnalités du produit.

13. L'installation est effectuée. Par défaut (si vous n'avez pas modifié les caractéristiques de l'administrateur), pour se connecter au serveur en mode administrateur, il faut utiliser l'utilisateur "Admin" sans mot de passe.
	Note : Pour des raisons évidentes de sécurité, pensez à changer le mot de passe de l'administrateur.

## Etape 2 : Installation du serveur réservé (WDBaaS)
<a name="etape_2_installation_serveur_reserve_wdbaas_ELTTEXTE000239"></a>
Le serveur réservé BaaS est un Serveur d'application WEBDEV réservé au groupware utilisateur et au Store Privé. La configuration du serveur réservé est commune au groupware utilisateur et Store Privé. 

Pour installer le Serveur réservé du Groupware utilisateur/Store Privé : 

1. Lancez le pack d'installation du serveur réservé (fichier WX280PACKRSRV.exe disponible dans le sous-répertoire "Install\\WDServeurR" du répertoire d'installation de WINDEV ou WINDEV Mobile).

2. Validez la licence. 

3. Sélectionnez le module à installer : "Installer les modules pour le groupware utilisateur et votre Store Privé". Passez à l'étape suivante.<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Install_serveur_store%20-%20HC%20N%B0001.gif)


4. Sélectionnez si nécessaire la version du Serveur d'application réservé à installer (32 ou 64 bits). Passez à l'étape suivante.

5. Sélectionnez le répertoire d'installation. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Install_serveur_store%20-%20HC%20N%B0002.gif)
Passez à l'étape suivante.

6. Spécifiez les caractéristiques du serveur HFSQL utilisé pour les modules WDBaaS (groupware utilisateur/Store Privé). 
	Remarque : Ce serveur HFSQL a été installé précédemment. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Install_serveur_store%20-%20HC%20N%B0003.gif)
Ces caractéristiques sont les suivantes : 

	- Nom du serveur HFSQL, 

	- Port. 

	- Nom et mot de passe de l'administrateur du Serveur HFSQL. Ces informations sont nécessaires pour configurer automatiquement la base de données WDBaaS et l'utilisateur associé. 




7. Testez la connexion pour vérifier vos paramètres. 

8. Indiquez les paramètres spécifiques à la base de données WDBaaS utilisée par le groupware utilisateur/Store Privé : 

	- le nom de la base de données dédiée : WDBAAS par défaut. 

	- le nom de l'utilisateur de cette base et son mot de passe. 




9. Indiquez le mot de passe des fichiers de données WDBaaS utilisés (existants ou à créer).

10. Indiquez le mot de passe de l'administrateur du site WDBaaS (l'utilisateur est "Admin").

11. Cette base de données et son utilisateur vont être automatiquement créés. 

12. Passez à l'étape suivante.

13. spécifiez si une connexion SSL doit être utilisée pour les sites d'administration à distance. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Install_serveur_store%20-%20HC%20N%B0004.gif)


14. Passez à l'étape suivante.

15. Le résumé des informations d'installation est affiché. 

16. Passez à l'étape suivante.   

17. L'assistant vérifie si un serveur Web est présent sur le poste d'installation. 

	- Si aucun serveur Web n'est installé sur le poste d'installation, l'assistant propose d'en installer un et de le configurer pour utiliser le serveur réservé pour le groupware utilisateur et/ou le Store Privé. 

	- Si un serveur Web est présent sur le poste d'installation, l'assistant le configure pour utiliser le serveur réservé pour le groupware utilisateur et/ou le Store Privé. 




18. A la fin de l'installation, l'assistant affiche un récapitulatif de ce type : 
	
	```txt
	Vous avez installé WDBaaS pour le groupware utilisateur et votre Store Privé.
	Token de connexion au serveur WDBaaS à fournir à l'environnement de développement : 
	WDXxxxxxxxxxxxxxxxxxxxxx
	
	Pour information, les paramètres de la connexion HFSQL utilisée sont les suivants : 
	- Serveur : xxx
	- Base de données : xxx
	- Utilisateur xxx - Mot de passe : xxx
	
	Le mot de passe des fichiers est : xxx
	```

	**Copiez et conservez ces informations**. Le token de connexion au serveur WDBaaS doit être utilisé : 

	- dans la fenêtre de paramétrage du groupware utilisateur de votre projet WINDEV.  

	- dans la fenêtre de paramétrage du groupware utilisateur/Store Privé dans votre projet WINDEV Mobile.

	- dans la fenêtre de paramétrage du groupware utilisateur de votre projet WEBDEV.  


 **Important** : La mémorisation des paramètres de configuration du Serveur réservé est enregistrée dans le fichier suivant : "C:\\ProgramData\\WDBaaS\\GPUConnexion.default.xml"
	Ce fichier doit être ajouté au plan de sauvegarde du serveur. En effet, ce fichier devra obligatoirement être restauré en cas de réinstallation du serveur ou de duplication sur une nouvelle machine physique ou virtuelle en reprenant les mêmes réglages.

19. Terminez l'assistant. Vous pouvez directement lancer le site WDBaaS d'administration du Store Privé. 




Pour plus de détails sur l'utilisation du Store Privé, consultez : 

- [Store Privé des applications WINDEV](../Editeurs/9000203.md).

- [Store Privé pour applications mobiles](../Editeurs/1410086652.md).




Remarque : Si le nom de la base de données est erroné, il est possible de relancer l'installation du Serveur réservé pour indiquer le nom correct de la base de données tout en conservant les identifiants.

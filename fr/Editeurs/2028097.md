


## Installation d'un site WEBDEV : Création de la bibliothèque
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000123"></a>
Une bibliothèque (fichier d'extension WDL) permet de regrouper les ressources du projet (pages, requêtes, ...) dans une bibliothèque.
 WEBDEV propose deux méthodes pour créer une bibliothèque :

- Méthode 1 : Si une configuration de projet de type "Bibliothèque" est présente dans le projet, il suffit de sélectionner cette configuration de projet et de la générer : sous le volet "Projet", dans le groupe "Génération", déroulez "Générer la configuration" et sélectionnez "Générer la configuration courante".

- Méthode 2 : Génération automatique de la bibliothèque lors du déploiement d'un site dynamique WEBDEV (cas le plus fréquent).




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Création d'une bibliothèque lors du déploiement d'un site dynamique
<a name="creation_une_bibliotheque_lors_deploiement_site_dynamique_ELTTEXTE000147"></a>


### Création de la bibliothèque lors de la création de l'installation
<a name="creation_bibliotheque_lors_creation_installation_ELTPARAGRAPHE000035"></a>

1. Sous le volet "Projet", dans le groupe "Génération", déroulez "Déployer le site" et sélectionnez "Générer et déployer le site".

2. Dans l'assistant qui se lance, sélectionnez les éléments à intégrer dans la bibliothèque. La bibliothèque aura pour nom le nom du projet en cours. Vous pouvez :

	- **Ajouter des éléments** (bouton "Ajouter") : 
			Tout type de fichier peut être ajouté à la liste des éléments insérés dans la bibliothèque : images, fichiers texte, ...

	- **Enlever des éléments** (bouton "Enlever") : 
			Les fichiers correspondants et leur dépendance ne seront pas pris en compte dans la bibliothèque.

	- **Créer la bibliothèque à partir d'une description de bibliothèque existante** (bouton "WDU") : 
			Lors de la création d'une bibliothèque WEBDEV, un fichier ".WDU" du même nom est automatiquement créé. Ce fichier contient toutes les références des éléments intégrés à la bibliothèque. Il suffit de sélectionner le fichier WDU contenant les éléments à prendre en compte.




3. Si votre projet utilise des composants, l'étape suivante propose d'intégrer ces composants à la bibliothèque. Ainsi, les fichiers des composants ne sont pas directement visibles dans les fichiers installés.

4. Dans l'étape suivante, indiquez la ou les langues à intégrer dans la bibliothèque. Cette option permet par exemple de réduire la taille de la bibliothèque. Par défaut, seule la langue en cours est proposée. Il est possible de sélectionner :

	- Une des langues utilisée par le projet.

	- Toutes les langues du projet.

	- Uniquement la langue en cours sous l'éditeur.




5. Dans l'étape suivante, indiquez les caractéristiques de la bibliothèque. Ces informations sont les suivantes :

	- le nom de la société. Par exemple, le nom de la société éditrice.

	- une brève description du but de l'application. Par exemple : "Gestion des pièces en stock".

	- le numéro de version du site. Ce numéro permet d'identifier les différentes versions des bibliothèques créées pour un même site. Ce numéro de version peut être incrémenté à chaque création de la bibliothèque (option "Incrémentation automatique de la version à chaque création").




6. La bibliothèque est automatiquement générée.

7. Suivez les différentes étapes de [l'assistant de création d'installation](../Editeurs/2028039.md).

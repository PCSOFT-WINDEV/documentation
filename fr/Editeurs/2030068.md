
## Dynamic serving
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000147"></a>
Le "Dynamic serving" est une technique dans laquelle la même adresse (URL) mène à 2 pages différentes : une page pour PC et une page pour mobile.

Selon le matériel (PC ou mobile) qui accède à la page, la page adéquate sera affichée.

C'est une solution alternative (voire complémentaire) au "[Responsive Web Design](../WDChamp/9000165.md)" pour rendre un site traditionnel "mobile friendly". Avec le "Dynamic serving", il n'est pas nécessaire de modifier les pages existantes : il suffit d'en rajouter pour les mobiles.

Cette technique bénéficie d'un excellent référencement par Google.
<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Dynamic%20serving.jpg)


Remarques : le "Dynamic serving" est disponible pour les pages dynamiques et pour les pages AWP (non autonomes). 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principe et fonctionnement
<a name="principe_fonctionnement_ELTTEXTE000171"></a>
Le site dispose de 2 jeux de pages : les pages pour ordinateur et les pages pour mobile. Ces pages sont associées 2 à 2. 

Lorsque l'adresse d'une page est saisie dans le navigateur, l'environnement utilisé est automatiquement détecté et la page correspondante (ordinateur ou mobile) est affichée dans le navigateur. L'adresse affichée dans le navigateur ne change pas : seul le contenu affiché est adapté au système. 

Tous les liens et les pages ouvertes dans le code sont automatiquement redirigés vers la page correspondante dans la plateforme en cours. 



**Remarques** : 

- La détection de l'environnement est réalisée selon les recommandations Google : les tablettes ne sont pas considérées comme des mobiles. 

- Les pages sont vues par Google comme étant en mode "Dynamic serving". Ainsi, les différentes versions des pages sont référencées. Si les pages mobiles sont correctement réalisées, le site est considéré comme "Mobile friendly". 

- Attention, le Dynamic Serving n'est pas disponible sur un serveur PHP. Cependant, pour un site mixte, qui mélangerait dynamique et PHP, alors les pages AWP qui référencent des pages PHP doivent utiliser le Dynamic Serving. Dans ce cas, les pages PHP sont listées par le Dynamic Serving.

## Comment le faire ?
<a name="comment_faire_ELTTEXTE000195"></a>


### Configuration
<a name="configuration_ELTPARAGRAPHE000047"></a>Pour utiliser le "Dynamic serving" dans un site WEBDEV : 

1. Créez les pages spécifiques à l'interface mobile dans votre projet. 

2. Dans le volet "Projet", dans le groupe "Web", déroulez "Référencement" et sélectionnez "Dynamic Serving".

3. La fenêtre qui s'affiche est constituée de trois parties : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Dynamic%20serving%20-%20HC%20N%B0001.gif&type=thumb)


	- 1. La liste des pages communes (affichées quelque soit la plateforme). 

- 2. La liste des pages version ordinateur.

- 3. La liste des pages version mobile.

4. Pour réaliser les associations de pages :

	- Dans la liste "Pages communes", sélectionnez la page correspondant à la page "Version Ordinateur". 

	- Réalisez un "Drag and Drop" de la page vers la zone "Version Ordinateur" ou cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Dynamic%20serving%20-%20HC%20N%B0001%201.gif). 

	- La zone "Version Mobile" correspondante apparaît en rouge.  

	- Dans la liste "Pages communes", sélectionnez la page correspondant à la page "Version Mobile". 

	- Réalisez un "Drag and Drop" de la page vers la zone "Version Mobile" ou cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Dynamic%20serving%20-%20HC%20N%B0001%202.gif).







Lorsque toutes les associations voulues sont réalisées, validez la fenêtre. 


### Remarques
<a name="remarques_ELTPARAGRAPHE000074"></a>

- Les associations sont enregistrées dans les pages. Si vos pages sont présentes dans le GDS, une extraction des pages mobiles est demandée. 

- Les pages associées doivent avoir le même prototype. Si les paramètres attendus ne sont pas les mêmes, les pages sont affichées sur fond rouge dans la fenêtre de gestion des associations. 

- Des pages AWP autonomes ne peuvent pas être associées. 

- Les champs manipulés depuis l'extérieur de la page (syntaxe : &lt;Nom de la page&gt;.&lt;Nom du champ&gt;) doivent être présents dans les deux pages de l'association. 

- La fonction [DynamicServingConfigure](../WDLang2/1000021994.md) permet de faire un lien "Site mobile" ou un lien "Site pour ordinateur" qui force l'affichage du site pour un environnement donné. 





### Tester un site "Dynamic serving"
<a name="tester_site_dynamic_serving_ELTPARAGRAPHE000086"></a>Par défaut, lors du test d'une page ou d'un site, la version "Ordinateur" du site est automatiquement lancée. 

Pour tester un projet ou une page en mode "Mobile", il suffit d'activer l'option de test en mobile : 

1. Dans les boutons d'accès rapide, déroulez la combo près de l'icône "GO" voulue (test de page ou de projet) : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Dynamic_serving_1%20-%20HC%20N%B0003.gif)

2. Sélectionnez l'option "Déboguer la version mobile (Dynamic Serving)". <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Dynamic_serving_1%20-%20HC%20N%B0002.gif)


3. Les prochaines tests de la page ou du projet seront en mode mobile. Pour repasser en mode "Ordinateur", il suffit de sélectionner à nouveau l'option "Déboguer la version mobile (Dynamic Serving)" pour la désactiver. 




Attention : cette option est disponible en mode GO de page et de site. Elle peut être activée/désactivée uniquement pour le GO de page et/ou pour le Go de site.

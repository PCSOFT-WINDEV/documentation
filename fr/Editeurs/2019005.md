


## Gestion des points d'arrêt
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000289"></a>
Lors du test d'un projet sous l'éditeur, les points d'arrêt permettent de lancer automatiquement le débogueur à partir d'une ligne donnée.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000313"></a>


### Insérer un point d'arrêt
<a name="inserer_point_arret_ELTPARAGRAPHE000018"></a>Il existe plusieurs méthodes pour insérer un point d'arrêt :

- Cliquez à gauche de la ligne où doit être inséré le point d'arrêt. Le curseur de la souris prend alors la forme d'une main et le signe ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_PointArret.gif) apparaît.

- Sous le volet "Code", dans le groupe "Points d'arrêt", déroulez "Point d'arrêt" et sélectionnez "Ajouter un point d'arrêt".

- Appuyez sur les touches Ctrl + B.

- Affichez le menu contextuel de la barre de gestion des points d'arrêt et sélectionnez l'option "Ajouter un point d'arrêt".<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Ajout_pointarr%EAt%20-%20HC%20N%B0001.gif)





**Remarque** : Les points d'arrêt sont pris en compte uniquement lors des tests de l'application sous l'éditeur de WINDEV, WEBDEV ou WINDEV Mobile. Lors de l'utilisation de l'exécutable, les points d'arrêt sont ignorés.
<a name="NOTE2_2"></a>


### Désactiver/Activer un point d'arrêt
<a name="desactiveractiver_point_arret_ELTPARAGRAPHE000049"></a>Un point d'arrêt désactivé est toujours présent dans l'éditeur de code mais n'est pas pris en compte lors du débugage de l'application (il devient grisé). 
Il existe plusieurs méthodes pour désactiver le point d'arrêt de la ligne en cours :

- Sous le volet "Code", dans le groupe "Points d'arrêt", déroulez "Point d'arrêt" et sélectionnez "Désactiver le point d'arrêt".

- Appuyez sur les touches Ctrl + Maj + B.

- Affichez le menu contextuel de la barre de gestion des points d'arrêt et sélectionnez "Désactiver le point d'arrêt" : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Code_Menu_PointArret.gif)



Remarque : Le point d'arrêt peut être réactivé à tout moment avec l'option "Activer le point d'arrêt".
<a name="NOTE2_3"></a>


### Supprimer un ou plusieurs points d'arrêt
<a name="supprimer_plusieurs_points_arret_ELTPARAGRAPHE000076"></a>Il existe plusieurs méthodes pour supprimer un point d'arrêt :

- Sous l'éditeur de code, cliquez directement sur le point d'arrêt. Le curseur de la souris prend alors la forme d'une main et le signe ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_PointArret.gif) disparaît.

- Sous le volet "Code", dans le groupe "Points d'arrêt", déroulez "Point d'arrêt" et sélectionnez : 

	- "Supprimer le point d'arrêt" pour supprimer le point d'arrêt de la ligne en cours.

	- "Enlever tous les points d'arrêt" pour enlever tous les points d'arrêt.




- Dans le débogueur, cliquez sur l'icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_paspointarret.gif).

- Affichez le menu contextuel de la barre de gestion des points d'arrêt : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Code_Menu_PointArret.gif)


	- sélectionnez l'option "Supprimer le point d'arrêt" pour supprimer le point d'arrêt présent sur la ligne courante.

- sélectionnez l'option "Enlever tous les points d'arrêt" pour supprimer tous les points d'arrêt. 



<a name="NOTE2_4"></a>


### Lister les points d'arrêt
<a name="lister_les_points_arret_ELTPARAGRAPHE000115"></a>Il existe plusieurs méthodes pour lister les points d'arrêt présents dans le projet en cours :

- Depuis l'éditeur de code : sous le volet "Code", dans le groupe "Points d'arrêt", déroulez "Point d'arrêt" et sélectionnez "Lister les points d'arrêt".

- Dans le débogueur, cliquez sur l'icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_Liste_pointarret.gif).

- Affichez le menu contextuel de la barre de gestion des points d'arrêt et sélectionnez l'option "Lister les points d'arrêt". <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Code_Menu_PointArret.gif)




<a name="NOTE2_5"></a>


### Point d'arrêt conditionnel
<a name="point_arret_conditionnel_ELTPARAGRAPHE000145"></a>Lors du débogage d'une boucle, il peut être utile de déclencher le débogueur uniquement après un certain nombre de passages (par exemple, après la 35ème itération). 
Pour mettre en place un point d'arrêt conditionnel : 

1. Créez un point d'arrêt (paragraphe [Insérer un point d'arrêt](#NOTE2_1)). 

2. Affichez les propriétés du point d'arrêt : 

	- via l'éditeur de code : Sous le volet "Code", dans le groupe "Points d'arrêt", déroulez "Point d'arrêt" et sélectionnez "Propriétés du point d'arrêt".

	- via le menu contextuel du point d'arrêt : Affichez le menu contextuel de la barre de gestion des points d'arrêt et sélectionnez l'option "Propriétés du point d'arrêt".




3. Dans la fenêtre qui s'affiche, dans la zone "Conditions", spécifiez les conditions de prise en compte du point d'arrêt : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Gestion_des_points_d'arr%EAt_2%20-%20HC%20N%B0002.gif&type=thumb)


	- **Toujours** (valeur par défaut) : Le point d'arrêt ne sera pas conditionnel et sera toujours pris en compte. 

- **Quand le nombre de passages sur le point d'arrêt est** : Permet de rendre conditionnel le point d'arrêt. Il suffit de sélectionner une condition et le nombre de passages voulus.

- **Quand l'expression ci-dessous est vérifiée** : Permet de rendre conditionnel le point d'arrêt. Il suffit de saisir directement la condition dans la zone "Si". 

4. Validez. Le point d'arrêt conditionnel est affiché sous l'éditeur (![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Gestion_des_points_d'arr%EAt_2%20-%20HC%20N%B0003.gif)).



<a name="NOTE2_6"></a>


### Point d'arrêt sans arrêt mais enregistré dans la trace du débogage
<a name="point_arret_sans_arret_mais_enregistre_dans_trace_debogage_ELTPARAGRAPHE000197"></a>Dans certains cas, lors du débogage d'une application, il est nécessaire de savoir sur quelle ligne de code le traitement est passé, mais sans pour autant s'arrêter sous le débogueur. Dans ce cas, seule une information doit être affichée dans la fenêtre de trace. 

Pour mettre en place cette fonctionnalité : 

1. Créez un point d'arrêt (paragraphe [Insérer un point d'arrêt](#NOTE2_1)). 

2. Affichez les propriétés du point d'arrêt : 

	- via l'éditeur de code : Sous le volet "Code", dans le groupe "Points d'arrêt", déroulez "Point d'arrêt" et sélectionnez "Propriétés du point d'arrêt".

	- via le menu contextuel du point d'arrêt : Affichez le menu contextuel de la barre de gestion des points d'arrêt et sélectionnez l'option "Propriétés du point d'arrêt".




3. Dans la fenêtre qui s'affiche, dans la zone "Actions", spécifiez l'action à effectuer lors du passage par le point d'arrêt (ces actions sont cumulables) : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Gestion_des_points_d'arr%EAt_2%20-%20HC%20N%B0004.gif&type=thumb)


	- Interrompre l'exécution (valeur par défaut) : le point d'arrêt interrompt l'exécution et lance le débogueur sur la ligne de code en cours. 

- Ajouter un message dans la trace du débogueur : le passage par le point d'arrêt ajoute le message spécifié dans le volet "[Trace du débogueur](../Editeurs/2027028.md)". 
			Remarque : Si aucun message n'est spécifié, le nom du traitement en cours est affiché dans le volet "Trace du débogueur". 

4. Validez. L'icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Gestion_des_points_d'arr%EAt_2%20-%20HC%20N%B0003.gif) présente devant le point d'arrêt indique qu'il possède des propriétés spécifiques.

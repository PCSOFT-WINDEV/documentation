


## Informations partagées
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000146"></a>
Les informations saisies lors de la création des fichiers et des rubriques sont partagées par l'ensemble de l'environnement.

Ces informations sont reprises et permettent de déduire des choix partout où cela est possible, par exemple pour appliquer automatiquement le masque de saisie au champ.

Lorsque vous modifiez les informations concernant un champ lié à une rubrique, il est possible de répercuter ces modifications sur la rubrique, et sur tous les champs de votre projet utilisant cette rubrique.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Informations partagées présentes sous l'éditeur d'analyses
<a name="informations_partagees_presentes_sous_editeur_analyses_ELTTEXTE000170"></a>


### Saisie des informations partagées
<a name="saisie_des_informations_partagees_ELTPARAGRAPHE000022"></a>Les informations partagées sont saisies lors de la description des rubriques dans l'analyse. En effet, lors de la saisie d'une rubrique, toutes les caractéristiques du champ lié à cette rubrique peuvent être saisies directement. Il suffit de : 

- Cliquer sur le lien "Paramètres du champ relié à la rubrique sélectionnée (Informations partagées)" situé au bas de la description des rubriques. 

- Les caractéristiques du champ lié à la rubrique apparaissent. Vous retrouvez les différents onglets des fenêtres de description des champs. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=info_partagee_Analyse_1.gif&type=thumb)





Les champs liés à ces rubriques (fenêtres ou pages créées par le RAD ou champs créés à partir des rubriques de l'éditeur d'analyses), auront toutes les caractéristiques saisies dans ces onglets.
<a name="NOTE2_2"></a>


### Modification des informations partagées
<a name="modification_des_informations_partagees_ELTPARAGRAPHE000039"></a>**Si vous modifiez les informations partagées d'une rubrique sous l'éditeur d'analyses**, lors de l'enregistrement de l'analyse, l'éditeur proposera une synchronisation du projet utilisant cette analyse.

Vous pouvez appliquer les modifications effectuées dans les informations partagées à tous les éléments du projet qui les utilisent.

**Si vous avez modifié des informations partagées d'un champ sous l'éditeur de fenêtres ou sous l'éditeur de pages et si le report des informations partagées est activé**, ces informations partagées sont immédiatement reportées dans l'analyse. Si vous créez un nouveau champ lié à cette rubrique, les nouvelles informations partagées seront prises en compte.

A la prochaine manipulation de l'analyse sous l'éditeur, lorsque l'analyse sera fermée, l'éditeur vous proposera de reporter les modifications effectuées dans les informations partagées dans tous les éléments du projet utilisant ces informations partagées. 
Vous pouvez également reporter ces modifications directement dans tous les éléments du projet : sous le volet "Projet", dans le groupe "Projet", déroulez "Recompiler et synchroniser" et sélectionnez "Synchronisation du projet avec l'analyse".

Vous pouvez :

- Mettre à jour les champs du projet.

- Ne pas mettre à jour les champs du projet.

- Choisir de dissocier le champ des informations partagées. Dans ce cas, le champ ne pourra pas modifier les informations partagées ni être mis à jour en cas de modification.

## Informations partagées présentes sous l'éditeur de fenêtres ou de pages
<a name="informations_partagees_presentes_sous_editeur_fenetres_pages_ELTTEXTE000200"></a>
Les champs présents dans les fenêtres (ou les pages) et liés à des rubriques (soit des champs générés par le RAD, soit des champs créés par "Drag and Drop" du volet "Analyse") peuvent mettre à jour les informations partagées des rubriques liées.

Par défaut, lors de la modification des caractéristiques d'un champ dans la fenêtre de description, les informations partagées de la rubrique liée ne sont pas mises à jour.

Dans les fenêtres de description des champs, un bouton permet de gérer la mise à jour des informations partagées de ce champ :
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Info_partagee_Fenetre.gif)
Dans la fenêtre ouverte par ce bouton, vous pouvez :

- **Mettre à jour les 'Informations Partagées' de la rubrique dans l'analyse**. Dans ce cas, dès la validation des modifications, les informations partagées seront modifiées dans l'analyse.

- **Ne pas mettre à jour les 'Informations Partagées' de la rubrique dans l'analyse**. Dans ce cas, les modifications faites dans la description du champ ne seront pas prises en compte dans l'analyse. Par contre, les modifications des informations partagées effectuées dans l'analyse pourront être reportées dans ce champ.

- **Visualiser l'impact sur le projet** :

	- La couleur du bouton change en fonction du nombre d'éléments du projet modifiés (élément = fenêtres, pages, ...).

	- Le bouton "Détails des éléments impactés" permet de visualiser le détails des éléments du projet qui sont concernés par cette modification. Dans cette liste apparaissent aussi bien les champs ayant le bouton en mode "On" que les champs ayant le bouton en mode "Off". Par contre, les champs dissociés des informations partagées n'apparaissent pas dans cette liste.




## Génération multiple
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000111"></a>
Les configurations de projets permettent de définir simplement les différentes "Cibles" de votre projet. Vous pouvez ainsi définir pour un même projet, des exécutables, bibliothèques, Webservices, assemblages DotNet, ...

Pour générer le résultat de chaque configuration, vous pouvez bien entendu sélectionner la configuration voulue dans le volet "Explorateur de projet" et générer le programme correspondant (l'icône de génération s'adapte automatiquement à la configuration sélectionnée dans les boutons d'accès rapide). 

Un autre moyen plus rapide existe : **la génération multiple**. Vous sélectionnez en une seule opération les configurations à générer et le résultat est immédiat.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment faire une génération multiple ?
<a name="comment_faire_une_generation_multiple_ELTTEXTE000135"></a>
Pour générer le résultat de plusieurs configurations : 

1. Sous le volet "Projet", dans le groupe "Génération", cliquez sur "Génération Multiple".

2. Dans la fenêtre qui s'affiche, sélectionnez les configurations que vous voulez générer.![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P2_WD_Generation_multiple%20-%20HC%20N%B0001.gif)


3. Validez. La génération des différentes configurations est effectuée.



<a name="NOTE2_2"></a>


### Remarques
<a name="remarques_ELTPARAGRAPHE000044"></a>

- Si une génération a déjà été effectuée pour une configuration, ce sont les derniers paramètres de cette génération qui sont pris en compte.

- Si aucune génération n'a été réalisée pour une configuration, les paramètres par défaut sont pris en compte.

- Configuration associée à un composant : le composant doit avoir été créé pour pouvoir être généré grâce à la génération multiple.

- Il est possible d'utiliser des répertoires de génération différents pour chaque configuration. Dans ce cas, chaque répertoire contiendra les fichiers générés par une seule configuration. Il n'y a pas de risque d'écrasement des fichiers. Tous les fichiers d'une configuration peuvent être récupérés facilement. Pour plus de détails, consultez [Configuration de projet](../Editeurs/9000030.md).

### Astuce : Comment connaître par programmation la configuration de projet utilisée ?
<a name="astuce_comment_connaitre_par_programmation_configuration_projet_utilisee_ELTPARAGRAPHE000059"></a>Pour connaître par programmation la configuration de projet utilisée, il suffit d'utiliser la fonction [ProjetInfo](../Editeurs/9000030.md) avec la constante *piConfiguration*.




## Options générales de WINDEV, WEBDEV, WINDEV Mobile
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000817"></a>
Les options générales permettent de configurer l'environnement de travail du produit utilisé.

Ces options concernent :

- Le développeur ([onglet "Utilisateur"](#NOTE2_1)).

- Le démarrage, la sauvegarde des fichiers, l'accélération de l'environnement et le GDS ([onglet "Général"](#NOTE3_1)).

- Le thème de l'éditeur, les volets, l'effet FishEye et la réinitialisation de l'environnement ([onglet "Editeur"](#NOTEEDIT_1)).

- Les paramètres de traduction, permettant de traduire rapidement vos interfaces ([onglet "Traduction"](#NOTE5_1)).

- Les répertoires des différents composants optionnels ([onglet "Répertoires"](#NOTE6_1)).

- La configuration du proxy pour les éditeurs ([onglet "Proxy"](#NOTE_PROXY)). 

- Le mode de gestion de l'aide en ligne ([onglet "Aide"](#NOTE9_1)).

- Les messages qui ne sont plus affichés ([onglet "Les "Ne plus afficher""](#NOTE7_1)).

- Les AAD (Aide au Développement) ([onglet "AAD"](#NOTE8_1)).




![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) **Pour afficher les options de WINDEV** : sous le volet "Accueil", dans le groupe "Environnement", déroulez "Options" et sélectionnez "Options générales de WINDEV".

## Onglet "Utilisateur"
<a name="onglet_utilisateur_ELTTEXTE000841"></a>


### Utilisateur et mot de passe
<a name="utilisateur_mot_passe_ELTPARAGRAPHE000115"></a>L'onglet "Utilisateur" permet de gérer les caractéristiques de l'utilisateur en cours. 

Il est possible de : 

- Connaître l'utilisateur en cours et si nécessaire de changer son mot de passe (bouton "Mot de passe"). Lors du changement de mot de passe, il suffit alors de saisir dans l'écran qui apparaît :

	- le mot de passe actuel,

	- le nouveau mot de passe et sa confirmation.




- **Demander le mot de passe au démarrage** : Cette option permet d'activer ou non la fenêtre de login au démarrage du produit. Dans cette fenêtre, il est possible de saisir le nom de l'utilisateur et le mot de passe associé.

- **Afficher le nom de l'utilisateur dans le titre de l'éditeur** : Cette option permet d'afficher le nom de l'utilisateur en cours dans la barre de titre de tous les éditeurs. 

- **Synchroniser l'environnement dans les Centres de Contrôle** : Cette option permet de sauvegarder l'environnement dans les Centres de Contrôle. Dans ce cas, l'environnement défini par le développeur (position des volets, volets affichés, ...) sera lié au login de connexion à WINDEV, WEBDEV ou WINDEV Mobile. L'utilisateur retrouvera son environnement personnalisé depuis n'importe quel poste lié à la base de données des Centres de Contrôle. 

- **Modifier les paramètres des Centres de Contrôle** (bouton "Paramètres des Centres de Contrôle"). Il est ainsi possible de connaître le paramétrage actuel des fichiers de données des Centres de Contrôle et de redéfinir ces paramètres si nécessaire. Pour plus de détails, consultez [Base de données des Centres de Contrôle](../CCSuivi/3540720.md).

### Dictionnaire de l'utilisateur
<a name="dictionnaire_utilisateur_ELTPARAGRAPHE000156"></a>Il est possible de définir un dictionnaire propre à chaque utilisateur.

Les éléments contenus dans ce dictionnaire (champs, fenêtres, ...) sont alors accessibles directement depuis le [volet "Champs prédéfinis"](../Editeurs/2027035.md).

## Onglet "Général"

### Démarrage
<a name="demarrage_ELTPARAGRAPHE000170"></a>Les options présentes dans la section "Démarrage" sont les suivantes : 

- **Langue de l'éditeur** : Cette option permet de choisir la langue d'affichage de l'éditeur : français, anglais ou espagnol. 

- **Charger le dernier projet** : Cette option permet de charger automatiquement le dernier projet ouvert pour le produit en cours.

- **Derniers projets utilisés** et **Derniers documents utilisés** : Ces options permettent de configurer le nombre de derniers projets et le nombre de derniers éléments affichés dans cette liste. Il suffira de sélectionner l'élément ou le projet voulu pour l'ouvrir directement sous l'éditeur correspondant.
	Les derniers documents et projets utilisés apparaissent à la fin des options permettant d'ouvrir un projet : 

	- ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_ouvrir.gif)
 parmi les boutons d'accès rapide. 

	- sous le volet "Accueil", dans le groupe "Général", déroulez "Ouvrir". Les derniers fichiers utilisés ("Fichiers récents") et les derniers projets utilisés ("Projets récents") sont affichés : 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichiers_Recent.gif)



 Remarques :

- Le nombre maximum de projets pouvant être affichés dans cette liste est 10.

- Le nombre maximum d'éléments pouvant être affichés est 10.

- **Gestion du DPI** : Différents modes de gestion du DPI pour l'affichage des éditeurs de WINDEV, WEBDEV et WINDEV Mobile : 

	- **Laisser Windows gérer le DPI de l'éditeur** : Il est conseillé de sélectionner cette option lorsque le mode de gestion par défaut ne permet pas de visualiser correctement la police de caractères utilisée pour les options de l'éditeur, notamment pour les écrans de petite taille mais avec un DPI important (type Macbook Retina). Lors de l'utilisation de plusieurs écrans utilisant des DPI différents, les images et les polices peuvent être floues.

	- **Utiliser le DPI de l'écran principal** : Si cette option est sélectionnée, les éditeurs de WINDEV, WEBDEV et WINDEV Mobile gèrent spécifiquement leur affichage en fonction de la résolution de l'écran principal. Cette option est conseillée lors de l'utilisation de plusieurs écrans ayant le même DPI que l'écran principal. 

	- **DPI indépendant par écran** (option par défaut) : Cette option permet d'utiliser les éditeurs de WINDEV, WEBDEV et WINDEV Mobile sur plusieurs écrans avec des DPI différents. Quelque soit l'écran d'affichage, les environnements sont nets. Lors du passage d'un écran à l'autre (utilisant des DPI différents), les éditeurs et les éléments affichés se redimensionnent correctement.





**Attention** : ce réglage du DPI concerne uniquement les éditeurs pour le développement. Il est sans effet sur les applications développées. Pour les applications développées, il est possible de modifier ce réglage dans la fenêtre de description des fenêtres (onglet "UI", option "Affichage en grandes polices : Agrandir la fenêtre et les champs").

### Sauvegarde
<a name="sauvegarde_ELTPARAGRAPHE000231"></a>Les options de sauvegarde sont les suivantes : 

- **Proposer l'enregistrement de l'élément dès sa création** : Cette option (cochée par défaut) permet d'afficher la fenêtre de sauvegarde de l'élément dès sa création. Cette sauvegarde permet entre autre d'activer immédiatement la complétion sur les noms des champs, ... Pour retrouver le fonctionnement des versions précédentes, il suffit de décocher cette option. Cette option est disponible à partir de la version 22.

- Si l'option **"Gestion de sauvegarde automatique des éléments du projet (.bk? et .tk?)"** est cochée, deux fichiers archives seront générés pour chaque élément (fenêtre, état, classe, ...) manipulé :

	- **Les fichiers d'extension ".BK\*" (".BKW", ".BKE", ".BKR", ...)**
			Le fichier d'extension BK\* est créé à la première sauvegarde de l'objet après l'ouverture du projet. Il correspond à la version de l'objet lors de l'ouverture du projet.
			Le fichier BK\* permet de récupérer un projet, une fenêtre, ... dans l'état dans lequel il était avant l'ouverture du projet sous l'éditeur.

	- **Les fichiers d'extension ".TK\*" (".TKW", ".TKE", ".TKR", ...)**
			Le fichier d'extension TK\* est créé à chaque sauvegarde de l'objet. Il correspond à l'état N-1 de l'objet.
			Le fichier TK\* permet de récupérer un projet, une fenêtre, ... dans l'état dans lequel il était avant la dernière sauvegarde sous l'éditeur.


 Les fichiers ".BK\*" et ".TK\*" sont enregistrés dans le répertoire "Sauvegarde" de votre projet.

- **Autoriser l'ouverture sans sauvegarde des projets venant d'une version précédente** : Par défaut, une sauvegarde du projet est automatiquement réalisée lors de l'ouverture de projets réalisés avec une version précédente. Cette sauvegarde est réalisée dans le répertoire du projet. Si cette option est cochée, l'ouverture du projet dans une version supérieure demandera si une sauvegarde est nécessaire. 

- **Enregistrement pour la récupération automatique toutes les XXX** : 
	Pour éviter tout problème lors du blocage du système (coupure de courant par exemple), WINDEV, WEBDEV et WINDEV Mobile proposent un système de récupération automatique des éléments (requêtes, états, fenêtres, ...) ouverts sous les différents éditeurs. Ce mécanisme est activé par l'option "Enregistrement pour la récupération automatique toutes les XX minutes".
	Lorsque ce système est activé :

	- Un enregistrement automatique des éléments ouverts est effectué à une période régulière (toutes les 10 minutes par défaut).

	- Lors de la réouverture d'un élément après un blocage du système, une fenêtre de récupération automatique apparaît. Cette fenêtre propose de récupérer :

		- soit l'élément tel qu'il était lors de la dernière sauvegarde réalisée par le développeur.

		- soit l'élément sauvegardé automatiquement par WINDEV, WEBDEV ou WINDEV Mobile.


 La version sélectionnée est automatiquement ouverte sous l'éditeur correspondant. La version non sélectionnée est disponible dans la fenêtre de gestion des versions (option "Fichier .. Versions") et peut être ouverte à tout moment. Pour plus de détails, consultez [Gestion des versions](../Editeurs/2021002.md).

- **Forcer le flush** : Par défaut, toutes les opérations d'écriture réalisées par WINDEV, WEBDEV ou WINDEV Mobile (enregistrement des éléments du projet, ...) ne sont pas immédiatement réalisées sur les disques réseaux.
	Pour forcer cette écriture et éviter l'utilisation des "caches", cochez l'option "Forcer le flush". Cette option risque de ralentir certaines fonctionnalités du produit utilisé (compilation du projet par exemple). Cette option est conseillée lors du partage de projet par plusieurs développeurs.
	**Remarque** : Cette option n'a aucune influence sur l'exécution des applications WINDEV, WEBDEV et WINDEV Mobile.

### Accélérer l'environnement
<a name="accelerer_environnement_ELTPARAGRAPHE000301"></a>Pour accélérer l'environnement, il est possible de :

- Choisir le mode de compilation du projet : Les modes de compilation sont : 

	- **Mono-coeur** : La compilation utilise un seul coeur du système. La compilation des différents éléments du projet est réalisée à l'ouverture du projet. Il est également possible de demander explicitement une recompilation du projet (sous le volet "Projet", dans le groupe "Projet", déroulez "Recompiler et synchroniser" et sélectionnez "Recompiler le projet"). Cette recompilation peut par exemple être effectuée avant de créer un exécutable.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) **En arrière-plan** : La compilation en arrière-plan permet une recompilation non bloquante du projet. Vous pouvez continuer à utiliser votre logiciel pendant la recompilation.

	- **Multi-coeur** : La compilation du projet utilise tous les coeurs du système : elle est donc plus rapide que la compilation standard. Ce mode de recompilation est proposé par défaut à partir de la version 26 U2. Cette recompilation se déclenche dans les cas suivants : 

		- à l'ouverture du projet s'il est nécessaire de réaliser une recompilation complète du projet. 

		- si la recompilation du projet est explicitement demandée : sous le volet "Projet", dans le groupe "Projet", déroulez "Recompiler et synchroniser" et sélectionnez "Recompiler le projet" ou "Réparer le projet".

		- lors du changement d'une configuration de projet, uniquement si cette configuration de projet n'a pas encore été activée sur le poste en cours. 


Remarque : Si l'option "Multi-coeurs" est choisie, par défaut, tous les coeurs du processeur sont utilisés. Pour un paramétrage plus fin, il est possible de choisir le nombre de threads à utiliser. Il est ainsi possible de gérer des cas particuliers ou de permettre à d'autres processus en cours sur le poste de continuer à s'exécuter à vitesse raisonnable.

- **Désactiver les règles métier** : Le chargement des règles métier présentes dans la base de données des Centres de Contrôle peut ralentir l'affichage des éléments sous l'éditeur ou le chargement de la fenêtre de description. Cette option est conseillée dans le cas de l'utilisation d'un réseau lent ou d'un développement via le Cloud. 
	Attention : dans ce cas, les règles métier sont débranchées.

- **Désactiver le correcteur orthographique**. Par défaut, l'environnement utilise le correcteur orthographique de Open Office. Ce correcteur permet d'obtenir en temps réel une correction des fautes d'orthographe en utilisant le dictionnaire Open Office. Il est possible de désactiver cette correction. Pour plus de détails sur la correction orthographique, consultez [Vérifier l'orthographe](../Editeurs/1013146.md).

- **Désactiver la reconnaissance vocale**. Par défaut, l'environnement répond aux commandes vocales (à partir de Windows Vista). Il est possible de désactiver cette reconnaissance vocale. Pour plus de détails, consultez [Commandes vocales dans l'environnement](../Editeurs/2020010.md).

### Gestionnaire de sources (GDS)
<a name="gestionnaire_sources_gds_ELTPARAGRAPHE000375"></a>Les options de WINDEV, WEBDEV et WINDEV Mobile permettent de configurer le mode d'extraction des éléments d'un projet présent dans le GDS.

Deux modes d'extraction des éléments du projet sont disponibles :

- **Le mode classique** : Si vous affichez un élément du GDS non extrait, un panneau indique que cet élément doit être extrait pour être modifié.

- **Le mode automatique** : Si vous tentez de modifier un élément du GDS non extrait, le GDS propose automatiquement d'extraire cet élément. Attention : Ce mode est déconseillé lors d'une utilisation du GDS avec une connexion Internet lente.




L'option "Extraction des éléments à la première modification" permet de passer du mode classique (option non cochée) au mode automatique (option cochée).

Pour plus de détails, consultez [Extraire un élément](../Editeurs/2038006.md).

<a name="telemetrie_ELTPARAGRAPHE000396"></a>Pour améliorer WINDEV, WEBDEV et WINDEV Mobile, il est possible d'activer la télémétrie en cochant l'option "J'accepte de participer à l'amélioration de XXX".

## Onglet "Editeur"
<a name="onglet_editeur_ELTTEXTE000925"></a>


### Thèmes
<a name="themes_ELTPARAGRAPHE000421"></a>Cette option permet de choisir le thème de l'éditeur. Par défaut, le thème clair est utilisé. 

D'autres thèmes sont disponibles : 

- Thème clair, Ruban gris. Dans ce mode, la barre de menu (le ruban) n'est plus colorée : elle est grise. 

- Thème gris. Dans ce mode, la barre de menu (le ruban) est gris et les différentes fenêtres de l'interface sont affichées sur fond gris clair. 

- Thème sombre. Dans ce mode, l'environnement et les différentes fenêtres de l'interface sont affichées sur fond noir ou gris foncé. 




Remarque : Le thème choisi sera pris en compte lors du prochain redémarrage du produit utilisé.

**Astuce** : Dans certains cas, il peut être nécessaire de changer régulièrement de thème en fonction de l'activité courante : par exemple en visio ou vidéoprojection, le thème clair sera privilégié, alors qu'en phase de développement, le thème sombre sera sans doute plus approprié.  Pour lancer WINDEV dans le thème souhaité, sans passer par les options et un "relancement" du produit, il suffit de :

- créer un raccourci vers l'exécutable "WINDEV64.exe"

- ajouter en ligne de commande le paramètre "/THEME" avec le thème souhaité (NOIR, BLANC ou GRIS).




Par exemple : 

```txt
C:\|]\WINDEV 28\Programmes\WINDEV64.exe /THEME=NOIR
```

- via le menu contextuel du ruban.

### Divers
<a name="divers_ELTPARAGRAPHE000472"></a>Les options présentes dans la section "Divers" sont les suivantes :

- **Volets de gauche sur toute la hauteur** : Si cette option est cochée, les volets présents à gauche, seront affichés sur toute la hauteur de l'environnement. 

- **Volets de droite sur toute la hauteur** : Si cette option est cochée, les volets présents à droite seront affichés sur toute la hauteur. 

- **Effet FishEye sur les graphes ( loupe )** : l'effet "FishEye" permet d'agrandir une partie du graphe en survolant une zone. Son action est équivalente à celle d'une loupe. Pour plus de détails, consultez ["FishEye"](../Editeurs/9000067.md). 
	Les modes d'activation du "FishEye" sont les suivants :

	- **Jamais** : le mode "FishEye" ne sera jamais activable.

	- **Lorsque le zoom est entre 25% et 75%** : le mode "FishEye" sera activable en fonction de la valeur du zoom.

	- **Lorsque le zoom est inférieur à 100%** : le mode "FishEye" sera activable en fonction de la valeur du zoom.

	- **Lorsque Ctrl et Maj sont enfoncés** : le mode "FishEye" sera activable en fonction d'une combinaison de touches.

## Onglet "Traduction"
<a name="onglet_traduction_ELTTEXTE001003"></a>


### Paramétrage des options de traduction
<a name="parametrage_des_options_traduction_ELTPARAGRAPHE000571"></a>Si votre projet est multilingue, les caractéristiques d'un élément (titre de la fenêtre, libellé d'un champ, ...) doivent être saisies dans toutes les langues de votre application.

Vous pouvez :

- **Activer le changement automatique des paramètres régionaux d'entrée** : Cette option permet d'activer automatiquement la langue utilisée par le clavier lorsque le curseur de souris est positionné dans une zone de saisie d'une langue utilisant un alphabet spécifique.

- **Sélectionner un moteur de traduction spécifique** : le logiciel de traduction spécifié sera utilisé pour permettre une traduction automatique depuis l'interface du produit des différents éléments à traduire. Le logiciel de traduction utilisé peut être : 

	- [WDDIXIO](../WDDixio/3514003.md), 

	- un logiciel spécifique. 
			Dans ce cas, il est possible de paramétrer le mode de transfert des informations à traduire de WINDEV, WEBDEV ou WINDEV Mobile vers le logiciel utilisé et le mode de récupération des informations traduites depuis le logiciel de traduction.

	- un site Internet.
			 Dans ce cas, lors de l'appui sur le bouton "Traduire", le texte à traduire est mis dans le presse-papiers et le site Web est automatiquement ouvert. Il n'y a pas de retour ou de traduction directe dans ce mode. 

	- Google Traduction. Dans ce cas, il est nécessaire de préciser la clé API attendue par Google Traduction. Pour plus de détails, consultez [Réaliser une traduction avec Google Traduction](../Editeurs/2036003.md). 
			Note : il est bien entendu nécessaire de se conformer aux règles d'utilisation du service par Google. 




- **Sélectionner les langues gérées** (langue d'origine et langue de destination).




**Remarques** : 

- Pour configurer correctement le mode de transfert des informations, consultez la documentation de votre logiciel de traduction.

- [WDDIXIO](../WDDixio/3514003.md) est un dictionnaire multilingue fourni avec WDMSG. Contactez le service commercial de PC SOFT pour plus de détails sur les conditions d'utilisation de ce produit.



<a name="NOTE5_2"></a>


### Utilisation de la traduction automatique
<a name="utilisation_traduction_automatique_ELTPARAGRAPHE000613"></a>Pour utiliser la traduction automatique précédemment paramétrée :

1. Sélectionnez le texte à traduire (par exemple le libellé dans la fenêtre de description du champ).

2. Cliquez sur le bouton "Traduire" (disponible par exemple dans la fenêtre de description du champ).

3. Récupérez la traduction selon les options choisies lors du paramétrage de la traduction automatique (coller dans la zone correspondante à la traduction du texte par exemple).




Pour plus de détails sur le développement d'applications multilingues, consultez [Applications multilingues](../WDLang1/3054008.md).

## Onglet "Proxy"
<a name="onglet_proxy_ELTTEXTE001063"></a>
L'onglet "Proxy" permet de définir les caractéristiques du proxy à utiliser pour permettre à l'environnement d'accéder à Internet. 

Remarque : Dans certains cas, l'environnement peut avoir besoin d'accéder à Internet : 

- Utilisation de Webservices, 

- Plateforme Cloud,

- Téléchargement de Gradle,

- Aide en ligne, 

- ...




En cochant l'option "Utiliser un proxy", il est possible de donner les caractéristiques du proxy à utiliser : 

- Adresse,

- Port,

- Utilisateur et mot de passe (optionnel).




Le bouton "Tester" permet de tester les paramètres spécifiés.

## Onglet "Aide"
<a name="onglet_aide_ELTTEXTE001087"></a>
Cet onglet permet de définir le mode d'utilisation de l'aide en ligne. 

L'aide en ligne est disponible : 

- en local : les fichiers d'aide sont installés sur le poste de développement. L'aide est affichée dans un browser d'aide spécifique. 

- sur Internet : l'aide en ligne Internet est disponible depuis n'importe quel poste disposant d'un accès Internet, sans que le produit soit forcément installé. Cette aide est mise à jour régulièrement.




Pour utiliser : 

- l'aide locale, il suffit de sélectionner l'option "Utiliser l'aide locale". 

- l'aide Internet, il suffit de sélectionner l'option "Utiliser l'aide en ligne".




Cet onglet permet également de filtrer l'aide affichée. Il est possible de : 

- Afficher l'aide pour tous les produits (WINDEV, WEBDEV, WINDEV Mobile et Etats  Requêtes) : l'aide commune à WINDEV, WEBDEV, WINDEV Mobile et Etats et Requêtes sera affichée. Tous les paragraphes spécifiques à un produit seront visibles. Toutes les entrées d'index (même spécifiques à un produit) seront accessibles. 

- N'afficher que l'aide du produit en cours : seules les pages d'aide concernant le produit en cours seront accessibles. Les remarques concernant un autre produit ne seront pas affichées. 




Pour plus de détails, consultez [Comment obtenir de l'aide](../Editeurs/2033001.md). 

Attention : Si vous utilisez le logiciel "Etats et Requêtes", seule l'aide en ligne sera disponible.

## Onglet "Les "Ne plus afficher""
<a name="onglet_les_plus_afficher_ELTTEXTE001111"></a>
Cet onglet permet de lister tous les messages pour lesquels l'option "Ne plus afficher" a été cochée. Les messages présents dans cette liste ne s'affichent plus dans l'environnement.

Pour ré-activer un message, il suffit de décocher ce message.

Pour plus de détails, consultez [Gestion des "Ne plus afficher"](../Editeurs/2020011.md).

## Onglet "AAD"
<a name="onglet_aad_ELTTEXTE001135"></a>
Cet onglet permet de configurer l'[aide au développement (AAD)](../Editeurs/9000036.md).

Il est possible de :

- Indiquer la position de la bulle de l'AAD.

- Demander à être prévenu lors de nouveaux conseils de l'AAD.

- Demander une confirmation avant d'exécuter l'AAD sélectionnée.


La liste des conseils désactivés est affichée. Pour réactiver un conseil, il suffit de cocher la ligne correspondante.

Pour plus de détails, consultez [Aide au développement (AAD)](../Editeurs/9000036.md).

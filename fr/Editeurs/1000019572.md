


## Paramétage et exécution des plans d'action
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000175"></a>
Lorsque les plans d'action sont déployés, il est possible de : 

- paramétrer les plans d'action (si des paramètres ont été définis)

- paramétrer leur mode de déclenchement. 




Ces opérations sont réalisées dans l'administrateur de la Fabrique Logicielle. 

Remarques : 

- Il est également possible d'exécuter directement un plan d'action depuis une application WINDEV ou un site WEBDEV grâce à la fonction [PlanActionExécute](../WDLang6/1000021332.md). 

- Il est également possible de paramétrer l'exécution d'un plan d'action depuis une application WINDEV ou un site WEBDEV grâce à la fonction [PlanActionParamètre](../WDLang6/1000021598.md). 






## Comment le faire ?
<a name="comment_faire_ELTTEXTE000199"></a>


### Paramétrer les plans d'action
<a name="parametrer_les_plans_action_ELTPARAGRAPHE000033"></a>Dans la majorité des cas, les plans d'action sont paramétrables, notamment le nom du projet. Un même plan d'action peut ainsi être réutilisé pour différents projets.  

Pour paramétrer un plan d'action :

1. Lancez l'administrateur de la fabrique logicielle : sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDFAdmin".

2. Dans la fenêtre qui s'affiche, indiquez le nom du coordinateur auquel l'administrateur doit se connecter. 

3. L'administrateur de la fabrique logicielle se lance. Le paramétrage du plan d'action est réalisé depuis l'onglet "Plans d'action". 

4. Sélectionnez le plan d'action à paramétrer et cliquez sur le bouton "Paramétrer le plan d'action". 

5. Dans la fenêtre qui s'affiche, il est possible d'indiquer : 

	- La valeur du ou des paramètres (onglet "Paramètres").

	- La durée maximale de l'exécution du plan d'action, exprimée en minutes (onglet "Paramètres"). 

	- Les destinataires des notifications (onglet "Notifications"). 
			Il est également possible de modifier le nom du plan d'action. En effet, chaque paramétrage d'un plan d'action permet de créer "un sous-plan d'action". Il est conseillé de donner un nom explicite ainsi qu'une description du sous-plan d'action. 




6. Validez. 

7. Le plan d'action paramétré apparaît dans l'onglet "Plans d'action". ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fab_Logicielle_Plan.gif)





**Remarque** : Par défaut, l'onglet "Plans d'action" de l'administrateur de la fabrique logicielle n'affiche que les plans d'action du développeur "en cours". Pour voir les plans d'action de tous les ­développeurs, il suffit de sélectionner "Tous" dans le champ "Plans d'action de ".


### Paramétrer le déclenchement d'un plan d'action
<a name="parametrer_declenchement_plan_action_ELTPARAGRAPHE000071"></a>Pour paramétrer le déclenchement d'un plan d'action :

1. Lancez l'administrateur de la fabrique logicielle : sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDFAdmin".

2. Dans la fenêtre qui s'affiche, indiquez le nom du coordinateur auquel l'administrateur doit se connecter. 

3. L'administrateur de la fabrique logicielle se lance. 

4. Affichez l'onglet "Plans d'action" et sélectionner le plan d'action voulu. 

5. Vous pouvez : 

	- soit lancer directement le plan d'action (bouton "Exécuter le plan d'action"). Dans ce cas, le plan d'action est automatiquement mis dans le spool d'exécution des plans d'action. 

	- soit planifier l'exécution du plan d'action (bouton "Planifier le plan d'action" de l'onglet "Plans d'action"). Un assistant se lance et permet de décrire toutes les options de planification. 
			Remarque : Il est également possible de planifier la planification dans l'écran de paramétrage du plan d'action. 







**Attention** : Si vous avez utilisé des paramètres lors de la description de vos plans d'action, n'oubliez pas de spécifier les paramètres à exécuter (voir paragraphe précédent). 

Si vous avez planifié l'exécution d'un plan d'action, la planification apparaît dans l'onglet "Planifications" de l'administrateur de la fabrique logicielle. Si une coche est affichée devant la planification, cela signifie que la planification est activée. 



## Remarques
<a name="remarques_ELTTEXTE000229"></a>


### Exécution sur un automate
<a name="execution_sur_automate_ELTPARAGRAPHE000104"></a>Pour exécuter un plan d'action, un automate va être choisi. Bien entendu, la machine de compilation (machine de l'automate) doit disposer d'une version de WINDEV, WEBDEV ou WINDEV Mobile correspondant au projet à compiler. 

Lors de l'exécution, WINDEV, WEBDEV ou WINDEV Mobile s'ouvre alors et passe en mode "fabrique logicielle". Dans ce cas, il est important de ne pas manipuler le poste où l'automate s'exécute pour éviter de fausser les résultats (par exemple en déplaçant la souris pendant un test unitaire). 

Remarque : L'administrateur de la Fabrique Logicielle permet de suivre l'exécution des plans d'action grâce au spool d'exécution des plans d'action (onglet "Automates").  Un plan d'action peut être prioritaire. Il est possible de modifier l'ordre des plans d'action dans le spool grâce aux boutons fléchés.


### Réutilisation des plans d'action
<a name="reutilisation_des_plans_action_ELTPARAGRAPHE000116"></a>Un plan d'action paramétrable peut être réutilisé par autant de projets et de développeurs que nécessaire : il suffit de créer un nouveau paramétrage. 

Pour utiliser un plan d'action existant, utilisez le bouton "Ajouter un plan d'action" (onglet "Plans d'action"). 


### Résultat des plans d'action
<a name="resultat_des_plans_action_ELTPARAGRAPHE000123"></a>Dans les paramètres du plan d'action, il est possible de : 

- définir la méthode d'envoi du compte-rendu du plan d'action (messagerie interne des Centres de Contrôle, email, etc.). 

- voir le résultat de la dernière exécution.

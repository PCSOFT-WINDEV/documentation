


## Génération de l'analyse
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000153"></a>
La génération de l'analyse intervient avant la phase de programmation et après la phase de description des fichiers de données. Cette génération permet de :

- valider les modifications effectuées sur l'analyse (MLD).

- créer les modules nécessaires à la programmation.

- mettre à jour automatiquement les fichiers de données si nécessaire.




Chaque génération d'analyse est numérotée. Ce numéro correspond au numéro de génération de l'analyse. Ce numéro est incrémenté lors de la génération de l'analyse uniquement si la description d'un fichier de données ou d'une des rubriques d'un fichier de données a été modifiée. Ce numéro est égal à 0 si l'analyse n'a pas encore été générée.

Tant que l'analyse (le MLD) n'a pas été générée, la description de l'analyse (c'est-à-dire les fichiers de données) ne pourra pas être utilisée dans le projet.

**Remarque** : Dans une application WINDEV, WEBDEV ou WINDEV Mobile, les termes "MLD" et "Analyse" sont utilisés indifféremment pour définir la structure de la base de données associée à un projet.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000177"></a>
La génération comporte trois étapes :

- vérification des modifications effectuées et génération des fichiers de données de la description de l'analyse.

- modification automatique des fichiers de données accessibles (fichiers présents dans le [fichier ".REP"](../WDLang4/3044186.md) ou dans le répertoire "EXE" du projet). Pour plus de détails sur cette opération, consultez [Modification automatique des fichiers de données](../WDLang4/3044195.md).

- synchronisation du projet avec les différentes modifications effectuées dans l'analyse.




**Pour générer l'analyse**, sous le volet "Analyse", dans le groupe "Analyse", déroulez "Génération" et sélectionnez "Génération".

**Remarque** : Il est également possible d'effectuer une **génération avancée de l'analyse** : sous le volet "Analyse", dans le groupe "Analyse", déroulez "Génération" et sélectionnez "Génération avancée". Cette génération avancée permet de définir différents paramètres spécifiques :

- génération d'un squelette pour un programme en langage externe.

- affichage des dernières modifications effectuées sur l'analyse.

- prise en compte avancée des modifications effectuées dans l'analyse du projet (par exemple dans les informations partagées du projet et/ou dans les fichiers de données du projet).

## Numéro de génération d'une analyse ou d'un fichier de l'analyse
<a name="numero_generation_une_analyse_fichier_analyse_ELTTEXTE000201"></a>


### Numéro de génération d'une analyse
<a name="numero_generation_une_analyse_ELTPARAGRAPHE000077"></a>A chaque génération de l'analyse, le numéro de génération de l'analyse est automatiquement incrémenté. Pour plus de détails sur l'utilisation des différentes versions de génération de l'analyse, consultez [Gestion des versions de génération de l'analyse](../Editeurs/2011002.md).

Ce numéro permet uniquement de connaître le nombre de générations effectuées depuis la création de l'application.
<a name="NOTE3_2"></a>


### Numéro de génération d'un fichier de données de l'analyse
<a name="numero_generation_fichier_donnees_analyse_ELTPARAGRAPHE000089"></a>Un numéro de génération est également associé à chaque fichier de données. Ce numéro est incrémenté lors de la génération de l'analyse uniquement si la description du fichier de données ou d'une de ses rubriques a été modifiée. Ce numéro est égal à 0 si l'analyse n'a pas encore été générée.

Le numéro de génération d'un fichier de données est donc inférieur ou égal au numéro de génération de l'analyse.

Le numéro de génération d'un fichier de données peut être :

- connu grâce à la propriété [NuméroGénération](../Proprietes/2512068.md).

- visualisé depuis WDMap (option "Affichage .. Informations générales sur le fichier").




Lors de l'installation de la mise à jour d'une application sur le poste des utilisateurs finaux, le numéro de génération des fichiers de données indique si la [modification automatique des fichiers de données](../WDLang4/3044195.md) est nécessaire ou non.


## Passer un projet WINDEV en Mobile : un audit détaillé
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000177"></a>
Le passage des applications WINDEV en Mobile est très facile.

WINDEV Mobile reprend tous les éléments des projets WINDEV : fenêtres, requêtes, code, ... le tout en restant dans l'éditeur de WINDEV (il est bien sûr nécessaire de posséder une licence WINDEV Mobile).

Il y a bien entendu les fenêtres à retailler pour les adapter à la taille des téléphones et des tablettes.

Et il y a également des adaptations de code à prévoir pour les fonctionnalités non supportées en mobile, par exemple l'absence de souris sur une tablette !

Pour vous aider, un assistant analyse le code de votre projet et indique de manière précise les lignes de code à modifier.

## Comment passer un projet WINDEV en Mobile ?
<a name="comment_passer_projet_windev_mobile_ELTTEXTE000201"></a>


### Comment le faire ? 
<a name="comment_faire_ELTPARAGRAPHE000026"></a>Pour passer un projet WINDEV en mobile, il suffit : 

- Soit d'ouvrir le projet WINDEV sous WINDEV Mobile. WINDEV Mobile propose alors de créer une configuration de projet mobile. 

- Soit si vous utilisez WINDEV et WINDEV Mobile intégré, il suffit de créer une configuration spécifique pour la plateforme voulue. 




L'audit de projet pour le passage en mobile permet d'obtenir tous les conseils nécessaires pour optimiser le passage d'une application WINDEV en Mobile pour les plateformes iOS et Android. 


### Passer un projet WINDEV en mobile (WINDEV avec WINDEV Mobile inclus)
<a name="passer_projet_windev_mobile_windev_avec_windev_mobile_inclus_ELTPARAGRAPHE000036"></a>Pour lancer le passage en mobile du projet WINDEV : 

1. Ouvrez le projet dans WINDEV. 

2. Sous le volet "Projet", dans le groupe "Audit et performances", déroulez "Audit statique" et sélectionnez l'option "Audit mobile". 

3. L'assistant de passage en mobile se lance. 

4. Cet assistant propose de créer une configuration de projet spécifique pour mobile (Android ou iOS). Passez à l'étape suivante. 

5. Sélectionnez le type d'appareils. Passez à l'étape suivante.

6. L'assistant liste des fenêtres WINDEV présentes dans votre projet. 

7. Sélectionnez les fenêtres à dupliquer dans la nouvelle configuration mobile. Toute modification d'IHM nécessaire pour le mobile sera effectuée dans la configuration mobile (ainsi que les modifications de code nécessaires). 

8. Passez à l'étape suivante. 

9. Sélectionnez les éléments (autres que les fenêtres) à intégrer dans la configuration mobile. Ces éléments seront partagés. 

10. Passez à l'étape suivante. 

11. Indiquez la charte graphique utilisée pour la configuration de projet. 

12. Passez à l'étape suivante.

13. Validez l'assistant. 

14. L'audit mobile se lance automatiquement. 




Remarques : 

- Le nom des fenêtres dupliquées est suivi par "_MOBILE". 

- A tout moment, il est possible de faire des copier/coller de code ou d'éléments entre une fenêtre WINDEV et une fenêtre WINDEV Mobile.

- La duplication des fenêtres peut être réalisée uniquement lors de la première création de configuration de projet pour mobile.  





### Charger un projet WINDEV dans WINDEV Mobile
<a name="charger_projet_windev_dans_windev_mobile_ELTPARAGRAPHE000074"></a>Pour lancer le passage du projet WINDEV vers Mobile : 

1. Ouvrez le projet WINDEV dans WINDEV Mobile. 

2. L'assistant de création d'une configuration de projet se lance. 

3. Sélectionnez la plateforme de la nouvelle configuration de projet (iOS ou Android par exemple). Passez à l'étape suivante. 

4. Sélectionnez le type d'appareils. Passez à l'écran suivant. 

5. L'assistant liste des fenêtres WINDEV présentes dans votre projet. 

6. Sélectionnez les fenêtres à dupliquer dans la nouvelle configuration mobile. Toute modification d'IHM nécessaire pour le mobile sera effectuée dans la configuration mobile (ainsi que les modifications de code nécessaires). 

7. Passez à l'étape suivante.

8. Sélectionnez les éléments (autres que les fenêtres) à intégrer dans la configuration mobile. Ces éléments seront partagés. 

9. Passez à l'étape suivante. 

10. Indiquez la charte graphique utilisée pour la configuration de projet.

11. Passez à l'étape suivante.  

12. Validez l'assistant. 

13. L'audit mobile se lance automatiquement. 




Remarques : 

- Le nom des fenêtres dupliquées est suivi par "_MOBILE". 

- A tout moment, il est possible de faire des copier/coller de code ou d'éléments entre une fenêtre WINDEV et une fenêtre WINDEV Mobile.

- La duplication des fenêtres peut être réalisée uniquement lors de la première création de configuration de projet pour mobile. 







### Suivre les conseils de l'audit mobile
<a name="suivre_les_conseils_audit_mobile_ELTPARAGRAPHE000100"></a>L'audit mobile est automatiquement lancé à la fin de l'assistant de création d'une configuration mobile et peut être lancé à tout moment via le menu de WINDEV ou WINDEV Mobile : sous le volet "Projet", dans le groupe "Audit et performances", déroulez "Audit statique" et sélectionnez l'option "Audit mobile".

Cet audit permet de traiter certaines erreurs de compilation en proposant des solutions. 

Toutes les erreurs de compilation rencontrées sont également affichées dans le volet des erreurs. 

Il est nécessaire de traiter toutes ces erreurs pour que l'application soit complètement passée en mobile. Certaines erreurs peuvent concerner des traitements ou des fonctionnalités non disponibles en mobile, des fonctions WLangage non disponibles, ...

Attention : Dans certains cas, de nouveaux développements peuvent être nécessaires pour adapter le fonctionnement de l'application au mobile. 




### Finir le passage en mobile
<a name="finir_passage_mobile_ELTPARAGRAPHE000126"></a>Lorsque toutes les erreurs de compilation et les erreurs de l'audit mobile sont corrigées, les adaptations de gestion réalisées, le passage en mobile est terminé.




## Groupware utilisateur : Applications multilingues
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000145"></a>
Si votre application est multilingue, il est nécessaire de gérer le multilingue également dans les fenêtres/pages du groupware utilisateur. Par défaut, les fenêtres/pages du groupware utilisateur gèrent l'anglais, l'américain et le français.

Cette page d'aide présente comment :

- [Configurer le groupware utilisateur pour l'utiliser dans une application multilingue](#NOTE2_1). 

- [Traduire les différentes fenêtres/pages du groupware utilisateur](#NOTE3_1).

- [Exécuter l'application dans la langue souhaitée](#NOTE4_1).




Pour plus de détails sur les applications multilingues, consultez [Créer une application multilingue](../WDLang1/3054008.md).

## Configurer le groupware utilisateur pour une application multilingue
<a name="configurer_groupware_utilisateur_pour_une_application_multilingue_ELTTEXTE000169"></a>
Pour intégrer les fenêtres ou les pages du groupware utilisateur dans une application multilingue :

1. Sous le volet "Projet", dans le groupe "Projet", cliquez sur "Groupware Utilisateur". 

2. Activez le groupware utilisateur. 

3. Selon les langues gérées, sélectionnez le mode d'intégration : 

	- Pour une application Anglais/Français, dans l'onglet "Options", sélectionnez l'option "Pas de personnalisation". 

	- Pour une application gérant d'autres langues, dans l'onglet "Options", sélectionnez l'option "Personnalisation complète (mode expert)". 




4. Selon le mode de lancement de l'application, définissez le mode de lancement du groupware : 

	- Si l'application doit se lancer dans la langue d'exécution, dans l'onglet "Général", choisissez "Lancement automatique". 

	- Si l'application doit demander à l'utilisateur la langue d'exécution, dans l'onglet "Général", choisissez "Lancement manuel par programmation".




5. Validez.

## Traduction des éléments du groupware utilisateur
<a name="traduction_des_elements_groupware_utilisateur_ELTTEXTE000193"></a>
Par défaut, les éléments (fenêtres, pages, messages, ...) du groupware utilisateur sont disponibles en français et anglais.

Pour traduire ces éléments dans d'autres langues, il suffit de traduire les différents libellés, messages, ... de ces fenêtres ou de ces pages (en utilisant l'outil [WDMSG](../WDMsg/3518040.md) si nécessaire pour extraire les messages à traduire). WDMSG est un outil optionnel de WINDEV, WEBDEV et WINDEV Mobile permettant d'extraire et de réintégrer tous les messages à traduire.
Remarque : Pensez à extraire les chaînes multilingues.

## Exécuter l'application dans la langue souhaitée
<a name="executer_application_dans_langue_souhaitee_ELTTEXTE000217"></a>
Deux cas peuvent se présenter : 

- L'application est automatiquement lancée dans la langue par défaut. L'utilisateur peut changer la langue par la suite. 
	Dans ce cas, les éléments du groupware seront également exécutés dans la langue par défaut. 

- L'application demande à l'utilisateur la langue à utiliser lors de son lancement. Dans ce cas, il faut : 

	1. Demander à l'utilisateur la langue dans laquelle l'application doit s'exécuter (par une fenêtre ou une page spécifique par exemple).

	2. Configurer la langue d'exécution de l'application avec la fonction [Nation](../WDLang1/3054014.md).

	3. Ouvrir la première fenêtre ou page de gestion du groupware utilisateur avec la fonction [gpwOuvre](../WDLang6/3041002.md).







Exemple de lancement personnalisé : 

```wl
// Code d'initialisation du projet
// Ouverture de la fenêtre choix des langues
nLangue est un entier = Ouvre(FEN_Select_Langue)
// Configuration de la langue pour l'application
Nation(nLangue)
// Lancement de la première fenêtre du groupware utilisateur
gpwOuvre()
```

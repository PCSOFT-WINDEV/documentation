## Présentation
<a name="presentation_ELTTEXTE000111"></a>
Le concept de Mashup consiste à agréger des données en provenance de différents logiciels, de différents services, de différentes sources de données, ...

Cette agrégation est réalisée de la manière la plus simple possible, le plus souvent sur une page de site Internet.

L'objectif du Mashup est de permettre une réelle interopérabilité, une liaison entre des logiciels différents. Il devient plus simple de créer des applications composites.

## Mashup spécifique à WEBDEV
<a name="mashup_specifique_webdev_ELTTEXTE000159"></a>
![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/fr/LX.png) Pour simplifier le mashup dans les sites réalisés en WEBDEV, il faut pouvoir s'interfacer avec différents services proposés sur le Web par des sites.

WEBDEV offre la possibilité :

- d'effectuer un mashup au niveau du serveur en utilisant directement les fonctions WLangage.

- d'effectuer un mashup au niveau "navigateur" (sans exécuter de code sur le serveur d'application WEBDEV) :

	- soit en utilisant des API Web.

	- soit en intégrant des composants évolués directement dans la page (partie de pages intégrant des traitements) pour interagir avec eux.







Il n'y a aucune charge induite sur le serveur en fonction des manipulations de l'internaute. La programmation s'effectue directement en WLangage : le code Javascript n'est plus nécessaire.

Les échanges de données proposés par ces services peuvent être structurés en XML ou directement en JSON.

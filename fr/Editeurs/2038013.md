


## GDS : Travailler en mode Nomade
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000138"></a>
Le Gestionnaire de Sources permet très simplement de travailler en mode déconnecté ou nomade.

Ce mode permet par exemple à un développeur utilisant un portable de continuer à travailler sur un projet présent dans la base de sources tout en étant déconnecté de la base de source.

**Le principe est simple** : 

- **avant la déconnexion physique**, vous indiquez que vous vous [déconnectez du GDS](#NOTE2_1). Il est conseillé d'extraire avant la déconnexion les différents éléments qui vont être modifiés (ainsi, ces éléments seront "déjà extraits" pour les autres utilisateurs). Vous pourrez travailler en local sur votre projet. Les différents éléments du projet sont manipulables directement.

- **après la reconnexion physique**, vous vous [reconnectez au GDS](#NOTE3_1) et vous réintégrez les éléments modifiés. 




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Se déconnecter du GDS
<a name="deconnecter_gds_ELTTEXTE000162"></a>
Pour travailler en mode déconnecté (ou nomade) :

1. Ouvrez le projet concerné.

2. Sous le volet "GDS", dans le groupe "Autres actions", déroulez "Travail à distance" et sélectionnez "Se déconnecter pour une utilisation nomade".

3. Dans l'assistant qui se lance, sélectionnez l'option "Je veux me déconnecter du Gestionnaire de sources" (Attention : cet assistant permet également de se déconnecter de la base de données partagée des Centres de Contrôle. Sélectionnez l'option voulue pour les Centres de Contrôle également).

4. Dans l'écran qui apparaît, vous pouvez : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GDS_Deconnexion.gif&type=thumb)


	- Voir les éléments actuellement extraits par vous dans le projet. Ces éléments ne pourront pas être modifiés par les autres développeurs durant toute la durée de votre déconnexion du GDS.

- Extraire de nouveaux éléments (grâce au bouton "Etat des élément"). Les éléments extraits ne pourront pas être modifiés par les autres développeurs durant toute la durée de votre déconnexion du GDS.

- Réintégrer des éléments actuellement extraits : Cette option permet de ne pas bloquer des éléments inutilement pendant votre déconnexion : il suffit de cocher les éléments de la liste à réintégrer (grâce aux boutons "Tous sans conflit", "Tous", "Aucun"). Les éléments cochés seront réintégrés lors de la déconnexion.

5. Vous pouvez également consulter sur les éléments extraits :

	- les modifications effectuées (bouton "Mes Modifs")

	- l'historique des manipulations (bouton "Historique").

	- les propriétés (bouton "Propriétés")

	- la comparaison de l'élément avec celui présent dans la base de sources.




6. La déconnexion est réalisée. Votre projet est disponible en local sur votre poste. Il n'a plus aucun lien avec le GDS. Tous les éléments du projet sont disponibles et peuvent être modifiés.

## Se reconnecter au GDS
<a name="reconnecter_gds_ELTTEXTE000186"></a>
Pour se reconnecter au GDS :

1. Sous le volet "GDS", dans le groupe "Autres actions", déroulez "Travail à distance" et sélectionnez "Se reconnecter et synchroniser".

2. Dans l'assistant qui se lance, sélectionnez l'option "Je veux me reconnecter au Gestionnaire de sources".

3. Lors de la reconnexion, les vérifications suivantes sont effectuées :

	- Extraction des éléments modifiés par le développeur nomade (uniquement si les éléments n'avaient pas été extraits avant la déconnexion).

	- Ajout dans la base de sources du GDS des nouveaux éléments ajoutés par le développeur nomade.







Lors de la reconnexion :

- Les modifications effectuées par les autres développeurs ne sont pas récupérées automatiquement. Il faut synchroniser le projet.

- Les modifications effectuées sur les éléments extraits ne sont pas répercutées. Il faut pour cela réintégrer les éléments extraits dans la base de sources du GDS.

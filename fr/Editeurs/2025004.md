


## Création d'un exécutable : Fonctionnement et bibliothèque 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000112"></a>
Le **mode de fonctionnement** d'un exécutable permet de définir si l'exécutable pourra être utilisé par une ou plusieurs personnes.

La **bibliothèque** principale d'une application regroupe tous les éléments manipulés par cette application (fenêtre, état, classes, collection de procédures, requêtes, fichiers de données...) et leurs dépendances (images utilisées dans les fenêtres par exemple). Pour plus de détails, consultez l'aide sur les [bibliothèques](../Editeurs/2030022.md).
**Remarque** : Pour créer l'exécutable, utilisez une des méthodes suivantes : 

- Sous le volet "Projet", dans le groupe "Génération", cliquez sur "Générer". 

- Déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_generation_exe.gif)
 parmi les boutons d'accès rapide et sélectionnez "Générer l'exécutable Windows 32 bits (exe)" ou "Générer l'exécutable Windows 64 bits (exe)". 




L'assistant de création de l'exécutable permet de définir le mode de fonctionnement de l'exécutable et les modes d'utilisation de la bibliothèque.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Mode de fonctionnement de l'exécutable
<a name="mode_fonctionnement_executable_ELTTEXTE000136"></a>
L'exécutable peut être :

- ***multi-instance***. Un programme multi-instance peut être lancé plusieurs fois depuis un même poste : plusieurs instances du programme seront ouvertes simultanément.

- ***mono-instance***. Un programme mono-instance peut être lancé une seule fois depuis un même poste. Si l'utilisateur tente de lancer une autre instance du programme, l'instance déjà en cours sera automatiquement activée.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Modes d'utilisation de la bibliothèque
<a name="modes_utilisation_bibliotheque_ELTTEXTE000160"></a>
Les modes d'utilisation de la bibliothèque sont les suivants :

- ***intégration de la bibliothèque principale dans l'exécutable***. Cette option permet de créer (ou de modifier) la bibliothèque principale de l'application en cours et d' **intégrer** cette bibliothèque dans le programme exécutable. 
	Pour mettre à jour la bibliothèque principale, il est nécessaire de recréer le programme exécutable.

- ***utilisation externe de la bibliothèque principale***. Cette option permet de créer (ou de modifier) la bibliothèque principale de l'application en cours et d' **associer** cette bibliothèque au programme exécutable. 
	Pour mettre à jour la bibliothèque principale, il suffit de recréer cette bibliothèque.

- ***ne pas utiliser de bibliothèque***. Aucune bibliothèque ne sera associée ou intégrée dans le programme exécutable. Cette option est conservée par compatibilité uniquement. Pour plus de détails, consultez le chapitre [Créer un exécutable sans bibliothèque](../Editeurs/2030025.md).

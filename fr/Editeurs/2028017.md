


## WDInst : paramétrer le répertoire des fichiers de données et des fichiers du groupware utilisateur 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000227"></a>
L'éditeur d'installation WDInst permet d'ajouter des options supplémentaires dans le programme d'installation. Ces options supplémentaires permettent à l'utilisateur de spécifier :

- le répertoire des fichiers de données de l'application.

- le répertoire des fichiers du groupware utilisateur.




Ces options supplémentaires peuvent également être ajoutées grâce à l'assistant de création du programme d'installation.

Rappel : Pour lancer WDInst, sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDInst".

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Proposer le paramétrage du répertoire des fichiers de données
<a name="proposer_parametrage_repertoire_des_fichiers_donnees_ELTTEXTE000251"></a>


### Répertoire de création des fichiers de données
<a name="repertoire_creation_des_fichiers_donnees_ELTPARAGRAPHE000037"></a>Par défaut, les fichiers de données sont créés dans les répertoires spécifiés lors de la description des fichiers dans l'analyse. Cependant, il est possible de modifier ces répertoires :

- lors de la création de l'exécutable de l'application.

- par programmation grâce aux fonctions [HSubstRep](../WDLang4/3044028.md) ou [HChangeRep](../WDLang4/3044168.md).

- lors de l'installation de l'application. Le répertoire de création des fichiers de données sera alors sélectionné par l'utilisateur.



<a name="NOTE2_2"></a>


### Paramétrage lors de l'installation
<a name="parametrage_lors_installation_ELTPARAGRAPHE000054"></a>Si votre application utilise des fichiers de données, vous pouvez ***permettre à l'utilisateur de paramétrer l'emplacement des fichiers de données*** lors de l'installation de l'application.

Le répertoire sélectionné par l'utilisateur sera automatiquement utilisé pour l'ensemble des fichiers de données créés par l'application. Il est mémorisé directement dans l'exécutable, sans utiliser la base de registre ou un fichier de paramètre.
Remarque : Ce répertoire sera proposé par défaut lors de l'installation des mises à jour de l'application.

Cette option permet d'éviter de programmer l'emplacement des fichiers de données (avec la fonction [HSubstRep](../WDLang4/3044028.md) par exemple).

**Rappel** : Dans tous les cas, le répertoire de création des fichiers de données est défini selon l'ordre de priorité suivant :

1. Répertoire spécifié grâce aux fonctions [HSubstRep](../WDLang4/3044028.md) ou [HChangeRep](../WDLang4/3044168.md).

2. Répertoire sélectionné lors de l'installation de l'application (si cette option est proposée à l'utilisateur).

3. Répertoire sélectionné lors de la création de l'exécutable.

4. Répertoire spécifié lors de la description des fichiers dans l'analyse.




Ainsi, le répertoire défini par les fonctions [HSubstRep](../WDLang4/3044028.md) ou [HChangeRep](../WDLang4/3044168.md) est prioritaire sur tous les autres répertoires définis.
<a name="NOTE2_3"></a>


### Cas particulier : des fichiers de données sont installés avec l'application
<a name="cas_particulier_des_fichiers_donnees_sont_installes_avec_application_ELTPARAGRAPHE000091"></a>Ces fichiers de données sont installés dans le répertoire spécifié dans l'éditeur d'installation WDInst (répertoire de destination). 
Dans le cas d'une installation réseau, ces fichiers de données seront installés dans le sous-répertoire "&lt;NomExécutable&gt;" du répertoire d'installation de l'application sur le poste serveur.
<a name="NOTE2_4"></a>


### Proposer le paramétrage du répertoire des fichiers de données
<a name="proposer_parametrage_repertoire_des_fichiers_donnees_ELTPARAGRAPHE000100"></a>Pour proposer le paramétrage du répertoire des fichiers de données : 

1. Sélectionnez l'option "Paramètres de l'installation .. Fichiers HFSQL".

2. Cliquez sur l'onglet "Connexions". 

3. Sélectionnez l'option "Editer l'emplacement des fichiers de données". 

4. Cliquez sur le bouton "Paramètre". 

5. Cochez l'option "Les valeurs saisies lors de l'installation (indiquez ci-dessous les valeurs par défaut)".

## Proposer le paramétrage du répertoire des fichiers du groupware utilisateur
<a name="proposer_parametrage_repertoire_des_fichiers_groupware_utilisateur_ELTTEXTE000293"></a>


### Répertoire de création des fichiers de données du groupware utilisateur
<a name="repertoire_creation_des_fichiers_donnees_groupware_utilisateur_ELTPARAGRAPHE000123"></a>Si votre application gère le [groupware utilisateur](../Editeurs/2018001.md), deux catégories de fichiers de données sont gérées :

- ***les fichiers de données permettant d'identifier les utilisateurs***. Ces fichiers de données peuvent être communs à plusieurs applications.

- ***les fichiers de données permettant de gérer les droits des utilisateurs sur les différentes fenêtres de l'application***. Ces fichiers sont spécifiques à l'application.




Pour plus de détails sur ces fichiers de données, consultez [Les fichiers du groupware utilisateur](../Editeurs/2018002.md).

Par défaut, ces fichiers de données sont créés dans le sous-répertoire "GPW_&lt;NomApplication&gt;" du répertoire d'installation de l'application. Il est cependant possible de modifier ce répertoire. <a name="NOTE3_2"></a>


### Proposer le paramétrage du répertoire des fichiers de données du groupware utilisateur
<a name="proposer_parametrage_repertoire_des_fichiers_donnees_groupware_utilisateur_ELTPARAGRAPHE000171"></a>Pour proposer le paramétrage du répertoire des fichiers de données du groupware utilisateur : 

1. Sélectionnez l'option "Paramètres de l'installation .. Fichiers HFSQL".

2. Cochez l'option "Paramétrer le répertoire des fichiers du groupware lors de l'installation".


## Exécutable autonome 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000095"></a>
L'exécutable sera **autonome** si les éléments suivants sont intégrés dans l'exécutable :

- les librairies WINDEV (fichiers ".DLL") nécessaires à l'application.

- les bibliothèques (fichiers ".WDL") principales et/ou externes.




Seul l'exécutable devra être installé sur le(s) poste(s) des utilisateurs finaux. Cependant, à **chaque** lancement du programme exécutable, les librairies WINDEV (fichiers ".DLL") et les bibliothèques (fichiers ".WDL") seront automatiquement extraites dans le répertoire de lancement du programme exécutable. Les librairies et les bibliothèques présentes dans ce répertoire seront automatiquement remplacées.

Pour éviter de recréer les librairies et les bibliothèques à chaque lancement du programme exécutable, il est conseillé de créer une installation auto-extractible. Ce type d'installation permet d'installer correctement tous les fichiers nécessaires à l'application sur le(s) poste(s) des utilisateurs finaux. L'utilisateur manipule uniquement l'exécutable de l'application. Pour plus de détails sur l'installation auto-extractible, consultez [Programme d'installation](../Editeurs/2028001.md).

## Limitations et remarques
<a name="limitations_remarques_ELTTEXTE000119"></a>


### Remplacement des fichiers existants
<a name="remplacement_des_fichiers_existants_ELTPARAGRAPHE000029"></a>**Attention** : Lors du lancement du programme exécutable, les librairies (fichiers ".DLL") sont automatiquement extraites et remplacent les fichiers existants. Si plusieurs applications WINDEV sont installées sur le même poste, ces remplacements peuvent provoquer des problèmes.

Par exemple, les applications "MonAppli1" et "MonAppli2" sont installées dans le même répertoire. Au lancement de l'exécutable de "MonAppli1", les librairies (fichiers ".DLL") associées à "MonAppli1" peuvent écraser les librairies utilisées par "MonAppli2". Si chaque exécutable de ces applications a été créé avec une version différente de WINDEV, certaines fonctionnalités de "MonAppli2" peuvent ne plus marcher correctement.

**Solution** : Pour éviter d'écraser les librairies (fichiers ".DLL") présentes, il est recommandé de renommer les librairies utilisées par l'application lors de la création du programme exécutable.


<a name="NOTE2_2"></a>


### Remarques
<a name="remarques_ELTPARAGRAPHE000041"></a>

- L'intégration des librairies (fichiers ".DLL") et des bibliothèques (fichiers ".WDL") externes dans l'exécutable permet d'installer et/ou de mettre à jour un seul fichier. Cependant, la taille du fichier exécutable créé est plus importante.

- Il est nécessaire de recréer l'exécutable dans les cas suivants :

	- si une seule des librairies doit être mise à jour.

	- si une seule des bibliothèques externes doit être mise à jour.

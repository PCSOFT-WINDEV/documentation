


## Gestion des versions de l'analyse
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000202"></a>
WINDEV, WEBDEV et WINDEV Mobile permettent de gérer les différentes versions de l'analyse directement depuis l'éditeur d'analyses.

Pour ouvrir l'analyse liée au projet, cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Analyse_bl.gif)
 parmi les boutons d'accès rapide.

La gestion des versions de l'analyse peut être effectuée depuis l'éditeur d'analyses : sous le volet "Analyse", dans le groupe "Analyse", déroulez "Génération" et sélectionnez "Gestion des versions". Chaque version de l'analyse correspond à une génération de l'analyse.

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GestionVersionAnalyse.gif&type=thumb)
Il est alors possible de travailler :

- Soit sur la version actuelle de l'analyse.

- Soit sur une version antérieure de l'analyse.




**Remarque** : Lorsque une analyse est endommagée, il est possible :

- Soit de réparer l'analyse : sous le volet "Analyse", dans le groupe "Analyse", déroulez "Maintenance" et sélectionnez "Réparer l'analyse". Pour plus de détails, consultez [Réparer une analyse](../Editeurs/9500110.md).

- Soit de restaurer une version antérieure de l'analyse.

- Soit de récupérer la dernière version enregistrée de l'analyse : sous le volet "Analyse", dans le groupe "Analyse", déroulez "Génération" et sélectionnez "Dernière version enregistrée".




Toutes les versions de l'analyse sont visualisées sous forme d'un graphique. Le carré jaune indique le numéro de génération de l'analyse et le libellé à droite rappelle la date de la génération.

Des "petits rectangles" sont affichés entre deux versions :

- Un rectangle : peu de modifications ont été effectuées entre les deux versions.

- Plusieurs rectangles : plusieurs modifications ont été effectuées entre les deux versions.




**Pour visualiser le détail des modifications effectuées entre deux versions**, double-cliquez sur la ligne contenant les "petits rectangles" ou cliquez sur le bouton "Modifications".

Les options disponibles dans le gestionnaire de versions de l'analyse sont :

- Restauration d'une analyse.

- Annulation de la dernière génération.

- Remettre à 1 le numéro de version.

## Restauration d'une analyse
<a name="restauration_une_analyse_ELTTEXTE000226"></a>
Pour restaurer une analyse à partir du gestionnaire de versions de l'analyse :

1. Sélectionnez la version à restaurer.

2. Vous pouvez :

	- Soit restaurer la version dans le répertoire spécifié. Une analyse indépendante est créée correspondant à la version choisie. Cette analyse ne sera pas liée au projet en cours. Cette option est sélectionnée par défaut.

	- Soit écraser l'analyse en cours. L'analyse restaurée sera automatiquement liée au projet en cours.




3. Validez. La restauration est effectuée.




**Attention** : Les fichiers de données associés à une analyse dont le numéro de version est supérieur à la version restaurée ne pourront plus être ouverts. Il faut alors :

- Soit restaurer des fichiers de données correspondant à la version de l'analyse restaurée (avec l'outil [WDJournal](../WDJournal/3516012.md) par exemple).

- Soit supprimer les fichiers de données existants pour qu'ils soient recréés.

## Annulation de la dernière génération
<a name="annulation_derniere_generation_ELTTEXTE000250"></a>
L'annulation de la dernière génération de l'analyse permet de restaurer l'analyse dans son état avant la dernière génération. Les modifications effectuées depuis la dernière génération ne sont pas appliquées.

**Pour annuler la dernière génération de l'analyse**, sous le volet "Analyse", dans le groupe "Analyse", déroulez "Génération" et sélectionnez "Annuler la dernière génération".

<a name="NOTE4_1"></a>


## Remettre à 1 le numéro de version
<a name="remettre_1_numero_version_ELTTEXTE000274"></a>
Il est possible de remettre le numéro de version de génération de l'analyse à "1". Dans ce cas :

- Le numéro de version de génération de l'analyse est remis à un. Les utilisateurs ne pourront donc pas voir le nombre de générations qui ont été nécessaires pour mettre au point l'application.

- Les versions des différentes générations de l'analyse sont supprimés.
	**Remarque** : Ces versions sont contenues dans les sous-répertoires "ANAxxx", présents dans le sous-répertoire "&lt;NomDuProjet&gt;.ANA" du projet.




Aucune action spécifique n'est réalisée sur les fichiers de données : le numéro de version des fichiers de données n'est pas modifié.

La remise à 1 du numéro de version de l'analyse peut être fait à tout moment et n'a aucune influence sur les versions de l'analyse diffusées.

**Pour remettre à 1 le numéro de version de l'analyse**, sous le volet "Analyse", dans le groupe "Analyse", déroulez "Génération" et sélectionnez "Remettre à 1 la version de l'analyse".

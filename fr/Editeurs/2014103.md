


## Description d'un composant interne
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000106"></a>
A tout moment, il est possible d'afficher la description d'un [composant interne](../Editeurs/2014100.md). Cette fenêtre regroupe toutes les options concernant le composant interne.

Pour afficher la description d'un composant interne : 

1. Sélectionnez le composant interne dans le volet "Explorateur du projet".

2. Sélectionnez l'option "Description du composant interne" du menu contextuel.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Détail
<a name="detail_ELTTEXTE000130"></a>
Les différents onglets de la fenêtre de description sont les suivants :

- ***Onglet "Description"*** : Cet onglet permet de connaître le nom du composant interne, son libellé et sa description (qui peuvent être directement modifiés).

- ***Onglet "Eléments"*** : Cet onglet liste des éléments du composant interne. Il est possible de :

	- ajouter des éléments dans le composant interne (bouton "Ajouter des éléments"). Il suffit de sélectionner l'élément voulu dans la liste des éléments du projet. 

	- supprimer directement des éléments du composant interne. Il suffit de sélectionner l'élément à supprimer puis de cliquer sur "Supprimer des éléments". Il est possible de supprimer l'élément uniquement du composant interne (il passe alors dans le projet) ou de supprimer l'élément du composant interne et du projet.

	- passer les éléments publics en privés et inversement. Il suffit de cocher les éléments à rendre publics. 




- ***Onglet "Analyse"*** : Cet onglet permet de paramétrer le mode de gestion des données et des contextes d'exécution. Trois modes de gestion des données et des contextes d'exécution sont disponibles :

	- **Utiliser l'analyse du projet ou aucune analyse (partage complet)** : Le composant interne accède aux fichiers de données du projet. Le composant interne utilise également les mêmes contextes d'exécution que le projet. 
			Ce mode correspond au mode par défaut si le composant interne n'utilise pas d'analyse.

	- **Utiliser l'analyse du projet avec contexte d'exécution différent (mode avancé)** : Le composant interne accède aux fichiers de données du projet. Le composant interne utilise des contextes d'exécution distincts de ceux du projet. 
			Ce mode est à utiliser dans des cas très spécifiques.

	- **Utiliser une analyse spécifique** : Le composant interne accède à ses propres fichiers de données : il est autonome. Le composant interne utilise des contextes d'exécution distincts de ceux du projet. 
			Ce mode correspond au mode par défaut si le composant utilise une analyse. 
			Dans ce cas, il est nécessaire d'indiquer l'analyse utilisée par le composant interne et si nécessaire son mot de passe. Il est également possible de créer directement une nouvelle analyse ou importer une analyse du GDS. Cette analyse sera associée au composant interne.


![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Remarque : Si le composant interne possède une analyse, cette analyse ne sera pas prise en compte lors de l'exécution.

- ***Onglet "Live Data"*** : Cet onglet est disponible uniquement si le composant interne possède une analyse spécifique. Il permet de paramétrer le "Live Data" pour le composant interne, c'est-à-dire la possibilité de visualiser le contenu des fichiers de données directement sous l'éditeur.

- ***Onglet "UML"*** : Cet onglet permet de lier un modèle UML au composant interne.

- ***Onglet "Dépendances"*** : Cet onglet permet de lister les dépendances associées au composant interne. 

- ***Onglet "Compilation"*** : Cet onglet permet de définir les normes de programmation à respecter par le composant interne. Pour plus de détails, consultez [Erreurs de norme de programmation](../Editeurs/9000187.md).

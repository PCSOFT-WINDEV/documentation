


## Test d'un site : les différents types de tests disponibles
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000114"></a>
WEBDEV propose plusieurs méthodes pour tester vos sites :

- **Test de l'ensemble du projet** :
	Le test de l'ensemble du projet permet de simuler le lancement du site. Il est ainsi possible de tester le site dans son ensemble, même si son développement n'est pas terminé. 
	Pour plus de détails, consultez : 

	- [Tester un projet dynamique](../Editeurs/2019009.md)

	- [Tester un projet statique](../Editeurs/2019013.md)




- **Test de montée en charge** :
	L'outil WDTestSite permet de réaliser des tests de montée en charge d'un site dynamique. WDTestSite permet de lancer plusieurs connexions simultanées à un même site dynamique WEBDEV. 
	Pour plus de détails, consultez : [L'outil WDTestSite](../WDTestSite/3538003.md).

- **Test d'une page seule** : 
	Le test d'une page seule permet d'exécuter uniquement la page en cours sous WEBDEV. Vous pouvez ainsi choisir de tester votre projet à partir d'une page donnée, ou de tester le fonctionnement d'une page dès que son développement est terminé.
	Pour plus de détails, consultez : 

	- [Tester une page dynamique (en mode Session ou en mode AWP)](../Editeurs/2019016.md).

	- [Tester une page statique](../Editeurs/2019006.md).




- [Test d'une requête seule](../Editeurs/2019011.md) :
	Le test d'une requête seule permet d'exécuter uniquement la requête en cours sous WEBDEV. Vous pouvez ainsi choisir de tester le fonctionnement d'une requête dès que son développement est terminé.

- [Test d'un état seul](../Editeurs/2019015.md) :
	Le test d'un état seul permet d'exécuter uniquement l'état en cours sous WEBDEV. Vous pouvez ainsi choisir de tester le fonctionnement d'un état dès que son développement est terminé. Il est possible de lancer le débogueur dès qu'un problème est rencontré.

- [Exécution du projet pas à pas](../Editeurs/2019001.md) : 
	L'exécution du projet pas à pas permet de lancer le débogueur au lancement du site. Cette solution permet de suivre méticuleusement le déroulement des codes serveur du site.

- [Test et débogage d'un site présent sur un serveur d'application](../Editeurs/2019032.md) :
	Il est possible de tester et de déboguer un site distant présent sur un serveur d'application quelconque. Le site est ainsi débogué directement sur le serveur en exploitation.

- [Attachement à une session en cours sur un serveur d'application](../Editeurs/2019032.md) :
	Il est possible de déboguer une session en cours d'exécution d'un site distant.

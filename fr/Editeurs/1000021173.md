


## Groupware utilisateur : Création de l'exécutable et installation
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Créer l'exécutable
<a name="creer_executable_ELTTEXTE000123"></a>
L'exécutable d'une application gérant le groupware utilisateur est forcément multi-instance.

Pour créer un exécutable gérant le groupware utilisateur : 

1. Lancez l'assistant de génération de l'exécutable à l'aide d'une des méthodes suivantes : 

	- Sous le volet "Projet", dans le groupe "Génération", cliquez sur "Générer". 

	- Déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_generation_exe.gif) parmi les boutons d'accès rapide et sélectionnez "Générer l'exécutable Windows 32 bits (exe)". 




2. Une étape spécifique au groupware utilisateur est affiché dans l'assistant de création de l'exécutable (option "Données et groupware" dans la partie gauche de l'écran). Cette étape permet de définir le chemin par défaut des fichiers du groupware utilisateur. Par défaut, ce répertoire correspond à un sous-répertoire du répertoire de l'exécutable (répertoire EXE\\GPW_&lt;Nom du projet&gt;).
	Remarque : Le chemin des fichiers de données du groupware à utiliser peut également être spécifié lors de l'installation de l'application.

3. Terminez l'assistant. 




Remarques :

- Si vous avez paramétré les fichiers de données du groupware utilisateur pour votre client, il est nécessaire de les sélectionner lors de la préparation de l'installation. Il est donc nécessaire de personnaliser la liste des fichiers de l'installation.

- Si vous n'avez pas paramétré les fichiers du groupware, lors du premier lancement de l'application, seul l'utilisateur Superviseur existera dans l'application.

## Installer une application
<a name="installer_une_application_ELTTEXTE000147"></a>
La création d'un programme d'installation d'une application utilisant le groupware utilisateur se fait comme pour toute application : sous le volet "Projet", dans le groupe "Génération", déroulez "Procédure d'installation" et sélectionnez "Créer la procédure d'installation".

Dans l'assistant de création d'installation, il est possible de demander la personnalisation du chemin des fichiers pour le groupware (option "Personnaliser l'emplacement des fichiers du groupware lors de l'installation" dans l'étape "Paramètres des bases de données").

Dans ce cas, lors de l'installation de l'application, un écran spécifique au groupware utilisateur s'affichera. Cet écran permettra à la personne installant l'application :

- de configurer l'emplacement des fichiers de données propres au groupware utilisateur. 

- d'activer l'historique des connexions.

- d'activer l'authentification des utilisateurs par un annuaire LDAP. Dans ce cas, il sera nécessaire de saisir l'adresse du serveur, le numéro de port et la version de LDAP utilisée.

<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GPU_Exe_install%20-%20HC%20N%B0001.gif)
Remarques :

- Si vous avez paramétré les fichiers de données du groupware utilisateur pour votre client, il est nécessaire de les sélectionner lors de la préparation de l'installation. Il est donc nécessaire de personnaliser la liste des fichiers de l'installation.

- Si vous n'avez pas paramétré les fichiers du groupware, lors du premier lancement de l'application, seul l'utilisateur "Superviseur" existera dans l'application.




**Conseil** : Si vous ne configurez pas les différents niveaux d'utilisation du groupware utilisateur pour vos clients, il est conseillé de leur fournir un document reprenant tous les champs de vos fenêtres/pages pour obtenir une configuration optimisée.

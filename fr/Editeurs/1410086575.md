
## Universal Windows 10 App : Déployer sur le store
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000189"></a>
Votre application Universal Windows 10 App est prête et fonctionnelle. Vous pouvez maintenant la publier sur le Windows Store de Microsoft pour la commercialiser ou en faire profiter la communauté. 

Les différentes étapes du déploiement sont les suivantes : 

1. [Certification locale et critères de certification](#NOTE2_1). 

2. [Création d'un compte développeur Microsoft](#NOTE3_1). 

3. [Soumission pour le déploiement sur le store](#NOTE4_1). 
	Cette étape permet de récupérer les identifiants nécessaires à la génération.

## Certification locale
<a name="certification_locale_ELTTEXTE000213"></a>
Avant tout déploiement, il est nécessaire de valider l'application sur le poste de développement avec le "Kit de certification des applications Windows". 

Ce kit peut être téléchargé à l'adresse suivante : [http://go.microsoft.com/fwlink/?LinkID=263703](http://go.microsoft.com/fwlink/?LinkID=263703). 

**La certification locale de l'application se déroule en plusieurs étapes** : 

1. Dans WINDEV ou WINDEV Mobile, générer l'application. Installez cette application en local sur le poste de développement. 

2. Depuis l'écran de démarrage, lancez l'outil "Windows App Cert Kit". 

3. Dans l'assistant, choisissez l'option "Valider une application Windows Store". 

4. Choisissez votre application. 

5. Cliquez sur le bouton "Suivant" et attendez la fin du test. Il est recommandé de ne pas utiliser la machine pendant la validation. 

6. A la fin de l'assistant, enregistrez le compte-rendu dans un fichier. 

7. Si l'assistant affiche "REUSSITE", vous pouvez soumettre l'application au Store. Dans le cas contraire, le rapport vous indiquera les problèmes rencontrés. 





### Critères de certification
<a name="criteres_certification_ELTPARAGRAPHE000053"></a>Les critères utilisés pour certifier les applications sont présentés à l'adresse suivante : [http://msdn.microsoft.com/fr-fr/library/windows/apps/hh694083.aspx](http://msdn.microsoft.com/fr-fr/library/windows/apps/hh694083.aspx)

N'hésitez pas à vous reporter à cette page pendant le développement de votre application ou si l'assistant de certification locale échoue.

## Création d'un compte développeur Microsoft
<a name="creation_compte_developpeur_microsoft_ELTTEXTE000243"></a>
Pour soumettre une application, vous devez disposer d'un compte développeur Microsoft. 

Il existe deux types de comptes, individuel ou entreprise : 

- Les développeurs individuels sont limités à la proposition d'applications Windows Store Apps. 

- Les développeurs entreprise peuvent soumettre des applications Windows Store Apps mais également des applications de bureau. 




Le site de Microsoft détaille les différences entre les deux types de compte. 

**Pour créer un compte développeur** : 

1. Ouvrez le Centre de développement Windows Store Apps : [http://go.microsoft.com/fwlink/?LinkID=245786](http://go.microsoft.com/fwlink/?LinkID=245786). 

2. Connectez-vous avec votre compte Microsoft (compte Live ou Hotmail). Si vous ne disposez pas d'un compte Microsoft, créez-en un. 

3. Suivez les étapes de l'assistant de création de compte développeur ou individuel. 

	- Choisissez le type de compte : individuel ou entreprise. 
			**Attention : le choix du type de compte est définitif**. 

	- Renseignez les paramètres du compte. En plus des informations personnelles, il est nécessaire de choisir le nom d'affichage de l'éditeur. C'est le nom de l'éditeur qui sera affiché dans le store pour vos applications. Bien entendu, vous devez spécifier un nom pour lequel vous avez des droits. 

	- Acceptez le contrat. L'assistant affiche le prix de l'inscription. 

	- Confirmez l'achat et payez en ligne. 




4. Le compte développeur Microsoft est créé. Il est possible d'accéder au tableau de bord pour soumettre une application. 




**Important** : Si vous souhaitez soumettre des applications payantes, il est nécessaire de configurer un compte de dividendes (compte sur lequel sera versé l'argent de vos ventes).

## Soumission pour un déploiement sur le store
<a name="soumission_pour_deploiement_sur_store_ELTTEXTE000267"></a>


### Récupération des identifiants nécessaires à la génération
<a name="recuperation_des_identifiants_necessaires_generation_ELTPARAGRAPHE000099"></a>**Pour récupérer les identifiants d'une application** : 

1. Affichez le tableau de bord de vos applications. Le tableau de bord de vos applications est disponible à cette adresse : [https://developer.microsoft.com/fr-fr/dashboard/apps/overview](https://developer.microsoft.com/fr-fr/dashboard/apps/overview). 

2. Créez si nécessaire une nouvelle application et réservez un nom de produit. Ce nom est un nom unique. Il correspond au nom utilisé pour votre application dans le Store. Ce nom devra être utilisé dans l'assistant de génération WINDEV ou WINDEV Mobile. 

3. Lorsque l'application est créée, il est possible de connaître les différents identifiants nécessaires à la génération de l'application sous WINDEV Mobile. 

	- Depuis le tableau de bord, affichez les différentes applications. 

	- Sélectionnez votre application. Dans le menu à gauche, dans la partie "Gestion des applications", cliquez sur "Identité des applications". 

	- Les différents identifiants sont affichés. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=W10_Identite_app.gif&type=thumb)








Dans l'assistant de génération WINDEV Mobile : 

- Le nom complet de l'éditeur correspond au "Package/Propriétés/NomCompletÉditeur". 

- L'identifiant de l'éditeur correspond au "Package/Identité/Éditeur"

- Le nom de l'application réservé dans le Windows Store correspond au nom de l'application unique. 

- Le nom complet du package de l'application correspond au "Package/Identité/Nom". 







### Soumettre une application
<a name="soumettre_une_application_ELTPARAGRAPHE000124"></a>Pour soumettre une application au Store : 

1. Cliquez sur le bouton "Démarrer la soumission". La soumission d'une application s'effectue en plusieurs étapes. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=W10_Soumettre_store.gif&type=thumb)
L'étape "Packages" nécessite la fourniture des packages générés par WINDEV Mobile. 

2. Sous WINDEV ou WINDEV Mobile, générez les packages pour le store. Dans l'assistant de génération, choisissez l'option "Génération pour Windows Store". Pour plus de détails, consultez [Générer une application Windows 10](../Editeurs/1410086577.md). 

3. Saisissez dans l'assistant les différents identifiants (voir paragraphe précédent). 

4. A la fin de l'assistant, le fichier &lt;NomAppli&gt;.appxbundle est généré. Ce fichier doit être uploadé sur le site Web de soumission (section "Packages"). 

5. Saisissez toutes les informations demandées pour la soumission. 

6. Pour finaliser le déploiement, cliquez sur le bouton "Soumettre dans le Windows Store". Le compte-rendu de certification vous sera envoyé par email.

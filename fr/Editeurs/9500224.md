
## Fichiers de données et rubriques zombies
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000144"></a>
Lors de la vie d'une application, les rubriques vivent et meurent. Mais avant de mourir, elles peuvent se transformer en zombies.
Cela signifie que la rubrique existe encore dans la description des données, peut être utilisée dans le code existant de l'application, mais que la rubrique ne doit plus être utilisée dans l'écriture de tout nouveau code.
Une rubrique zombie mange de la place et devrait donc être supprimée.
On trouve souvent des rubriques zombies dans le code mort.
En version 23, il est possible d'indiquer les rubriques zombies dans l'analyse; elles seront alors signalées à l'utilisation.

## Définir une fichier ou une rubrique comme étant "zombie"
<a name="definir_une_fichier_une_rubrique_comme_etant_zombie_ELTTEXTE000168"></a>


### Définir un fichier comme étant "zombie"
<a name="definir_fichier_comme_etant_zombie_ELTPARAGRAPHE000024"></a>Pour définir un fichier zombie : 

1. Sous l'éditeur d'analyses, sélectionnez le fichier de données à manipuler. 

2. Affichez la description du fichier (option "Description du fichier de données" du menu contextuel). 

3. Dans l'onglet "Options" : 

	- Cochez l'option "Fichier obsolète ou zombie". 

	- Indiquez si nécessaire un commentaire. Ce commentaire sera affiché dans l'erreur de compilation générée lors de l'utilisation du fichier zombie. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichier_zombie%20-%20HC%20N%B0001.gif)





4. Fermez la fenêtre de description du fichier de données. 

5. Dans le graphe de l'analyse, le fichier zombie est identifié par le pictogramme ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichier_zombie%20-%20HC%20N%B0002%201.gif) : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichier_zombie%20-%20HC%20N%B0002.gif)


6. Dans le volet "Analyse", le fichier zombie apparaît grisé et barré : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichier_zombie%20-%20HC%20N%B0003.gif)


7. Générez l'analyse pour prendre en compte les modifications. Dans les erreurs de compilation, une nouvelle erreur apparaît indiquant les utilisations du fichier zombie. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichier_zombie%20-%20HC%20N%B0003%201.gif)





Remarque : Il est également possible d'effectuer l'opération inverse. 




### Définir une rubrique comme étant "zombie"
<a name="definir_une_rubrique_comme_etant_zombie_ELTPARAGRAPHE000063"></a>Pour définir un fichier zombie : 

1. Sous l'éditeur d'analyses, sélectionnez le fichier de données à manipuler. 

2. Affichez la description des rubriques (option "Description des rubriques" du menu contextuel). 

3. Dans l'onglet "Avancé" : 

	- Cochez l'option "Rubrique zombie (obsolète)". 

	- Indiquez si nécessaire un commentaire. Ce commentaire sera affiché dans l'erreur de compilation générée lors de l'utilisation de la rubrique zombie. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichier_zombie%20-%20HC%20N%B0004.gif&type=thumb)





4. Validez. 

5. Dans le graphe de l'analyse, la rubrique zombie apparaît barrée dans la représentation du fichier de données : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichier_zombie%20-%20HC%20N%B0005.gif&type=thumb)


6. Dans le volet "Analyse", la rubrique zombie apparaît grisée et barrée : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Fichier_zombie%20-%20HC%20N%B0005%201.gif)


7. Générez l'analyse pour prendre en compte les modifications. Dans les erreurs de compilation, une nouvelle erreur apparaît indiquant que la rubrique zombie est utilisée. 




Remarque : Il est également possible d'effectuer l'opération inverse.

## Manipulation d'un fichier ou d'une rubrique zombie
<a name="manipulation_fichier_une_rubrique_zombie_ELTTEXTE000198"></a>
Dans les différents éditeurs manipulant des fichiers ou des rubriques (éditeur de fenêtres, d'états, de requêtes, etc.), les fichiers et les rubriques zombies sont facilement identifiables car ils sont grisés et rayés. 

De plus, une erreur de compilation permet d'identifier rapidement les éléments zombies utilisés.

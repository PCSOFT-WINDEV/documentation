


## Styles CSS (Cascading Style Sheet)
			





## Présentation
<a name="presentation_ELTTEXTE000118"></a>


### Les styles CSS sont les styles standard utilisés sur Internet.
<a name="les_styles_css_sont_les_styles_standard_utilises_sur_internet_ELTPARAGRAPHE000008"></a>Un style CSS permet de définir certaines caractéristiques de style d'un champ. Les styles CSS sont inclus et utilisés par les styles WEBDEV. 

WEBDEV permet de gérer les styles CSS. Il est possible de : 

- Créer un style CSS.

- Modifier un style CSS.

- Associer un style CSS à un champ.

- Surcharger le style CSS associé à un champ.

## Gestion des styles CSS
<a name="gestion_des_styles_css_ELTTEXTE000142"></a>


### Création et modification d'un style CSS
<a name="creation_modification_style_css_ELTPARAGRAPHE000024"></a>La création et la modification d'un style CSS peut être effectuée : 

- depuis la fenêtre de description des styles d'un champ. 

- depuis la fenêtre de gestion des styles CSS. Pour plus de détails, consultez [Edition des feuilles de styles CSS](../Editeurs/2016001.md). 





### Création et modification depuis la fenêtre de description du style d'un champ
<a name="creation_modification_depuis_fenetre_description_style_champ_ELTPARAGRAPHE000035"></a>La fenêtre de description des champs permet de gérer le style de ce champ (onglet "Style"). Différents éléments sont gérés par l'onglet "Style". Les éléments pouvant utiliser un style CSS sont identifiés par "(CSS)" à la fin de leur libellé. Selon le type de champ, plusieurs styles CSS peuvent être utilisés pour différents éléments du champ. Par exemple, pour un champ de type Liste : 

- un style CSS est utilisé pour le libellé du champ 

- un style CSS est utilisé pour les éléments intérieurs du champ.




Le style CSS associé à un élément d'un champ peut être local ou non.

**Si le style est local**, cela signifie que ce style est associé au champ en cours (et ne dépend plus des modifications apportées dans la feuille de styles du projet). Ce style ne sera pas celui défini dans la feuille de styles CSS : ce style sera défini directement dans le fichier HTM de la page.
Il est déconseillé d'utiliser des styles "local à un champ" car cette option augmente la taille des pages HTML générées (et donc le temps de chargement de la page sur le navigateur de l'internaute).

**Si le style CSS correspond au nom d'un style CSS**, cela signifie que le champ utilise un style CSS défini dans la feuille de styles CSS. Si le style CSS est modifié, tous les champs utilisant ce style CSS seront mis à jour. 

L'option "Editer" du bouton ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=VoletCatalogueimage%201.gif) permet d'éditer les caractéristiques du style CSS. Il est possible : 

- d'éditer le style local 

- d'éditer le style CSS utilisé. 

- de créer un nouveau style CSS pour l'associer au champ. 




Pour plus de détails sur la description d'un style CSS, consultez [Fenêtre de description d'un style CSS](../Editeurs/2016007.md). 


### Associer un style CSS à un champ
<a name="associer_style_css_champ_ELTPARAGRAPHE000064"></a>Pour associer un style CSS à un champ, il suffit d'indiquer dans la fenêtre de description du style, le style CSS à utiliser pour l'élément voulu. 


### Surcharger le style CSS associé un champ
<a name="surcharger_style_css_associe_champ_ELTPARAGRAPHE000069"></a>**Attention** : Une modification effectuée dans la partie basse des caractéristiques du style ne modifie pas le style CSS associé au champ : cette modification redéfinit le style CSS localement. Seules les caractéristiques du style CSS du champ en cours sont surchargées. Cette modification n'est pas appliquée à tous les champs utilisant le style CSS.

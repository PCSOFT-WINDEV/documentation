
## Génération d'un diagramme de classes 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000106"></a>
La génération permet de créer les classes WINDEV (fichier ".WDC") qui correspondent aux classes créées sous l'éditeur d'UML.

Il est possible :

- soit de générer une seule classe.

- soit de générer l'ensemble des classes du diagramme.




Il est également possible de générer un diagramme de classes par rétro-ingénierie. Pour plus de détails, consultez [Génération d'un diagramme de classes par rétro-ingénierie](../Editeurs/2035009.md).

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000130"></a>


### Générer une seule classe
<a name="generer_une_seule_classe_ELTPARAGRAPHE000028"></a>Pour générer une seule classe :

1. Sélectionnez la classe à générer.

2. Cliquez sur l'icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=IconeGenerationclasse.gif). La fenêtre de génération d'UML s'ouvre.

3. La classe en cours est :

	- dans le tableau du haut si cette classe n'a jamais été générée. Cette classe va être automatiquement créée par la génération d'UML.

	- dans le tableau du bas si cette classe a déjà été créée. Cette classe va automatiquement être recréée par la génération d'UML.




4. Validez la génération de la classe UML.





<a name="NOTE2_2"></a>


### Générer l'ensemble des classes du diagramme
<a name="generer_ensemble_des_classes_diagramme_ELTPARAGRAPHE000046"></a>Pour générer l'ensemble des classes du diagramme :

1. Sélectionnez l'option "Générer le modèle UML" du menu contextuel de l'éditeur (ou cliquez sur l'icône ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=UML_Ico_generer.gif)). La fenêtre de génération d'UML s'ouvre. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GenerationDiagrammeclasse.gif)


2. Cette fenêtre présente :

	- les classes qui n'ont jamais été générées dans le tableau du haut. Ces classes vont être automatiquement créées par la génération d'UML.

	- les classes qui ont déjà été créées dans le tableau du bas. Ces classes vont automatiquement être recréées par la génération d'UML.




3. Validez la génération du diagramme de classes UML.

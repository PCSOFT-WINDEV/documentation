


## Déploiement page à page
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000093"></a>
WEBDEV permet de déployer des parties de site ou des pages individuelles. Il n'est pas nécessaire de remettre l'ensemble du site à jour systématiquement à chaque modification.

Pendant la mise à jour de la page (et de ses dépendances), l'internaute n'est pas déconnecté.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000117"></a>


### Conditions nécessaires
<a name="conditions_necessaires_ELTPARAGRAPHE000020"></a>

- Pour effectuer un déploiement page à page d'un site, il est bien entendu nécessaire **d'avoir déployé au moins une fois sur un serveur d'application distant**.

- Déployer des pages dynamiques et/ou des pages AWP.



<a name="NOTE2_2"></a>


### Effectuer un déploiement page à page
<a name="effectuer_deploiement_page_page_ELTPARAGRAPHE000028"></a>Pour effectuer un déploiement page à page : 

1. Sélectionnez la page à mettre à jour dans le volet "Explorateur de projet".

2. Dans le menu contextuel, sélectionnez l'option "Déployer".

3. Le déploiement de la page et si nécessaire de ses dépendances est effectué. Le serveur utilisé pour le déploiement de la page est le dernier serveur utilisé lors de la dernière installation du site (voir [Déployer un site sur un serveur d'application distant](../Editeurs/2028019.md)).



<a name="NOTE2_3"></a>


### Limitations
<a name="limitations_ELTPARAGRAPHE000042"></a>

- Seules peuvent être déployées page à page des pages existantes qui ont été modifiées. Si de nouvelles pages sont ajoutées, il est nécessaire de redéployer l'intégralité du site.  

- Si des éléments sont ajoutés au site (requêtes, états, ...), il est nécessaire de redéployer l'intégralité du site.

- Si un code global est modifié, il est nécessaire de redéployer l'intégralité du site.

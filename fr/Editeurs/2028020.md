


## WDInst : options avancées pour l'installation réseau 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000295"></a>
Les options avancées de l'installation réseau permettent de :

- paramétrer les options de contrôle à distance.

- modifier les conditions d'application de la mise à jour.

- afficher l'aide des nouveautés à l'utilisateur lors de l'installation réseau.

- référencer l'application dans les Centres de Contrôle. 




Par défaut :

- aucune aide des nouveautés ne s'affiche.

- les options de contrôle à distance sont les suivantes :

	- délai entre chaque contrôle à distance : 5 minutes.

	- message d'interdiction de se connecter : "Cette application a été verrouillée par son administrateur pour une mise à jour et n'est pas disponible pour le moment."

	- message d'arrêt imminent : "Cette application va être interrompue pour une mise à jour. Veuillez sauver toutes vos modifications et quitter l'application."

	- message d'arrêt forcé : "ATTENTION : Arrêt de l'application pour mise à jour."

	- message de reprise d'installation :"La précédente mise à jour de l'application ne s'est pas terminée correctement et l'application ne peut pas s'exécuter. Voulez-vous relancer la mise à jour ?".




- la mise à jour est automatiquement installée si l'utilisateur n'effectue aucune opération après un délai de 60 secondes.

## Paramétrage des options avancées de l'installation réseau
<a name="parametrage_des_options_avancees_installation_reseau_ELTTEXTE000319"></a>
Le paramétrage des options avancées de l'installation réseau est réalisé sous l'éditeur d'installation WDInst : sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDInst".

Ce paramétrage peut également être effectué grâce à l'assistant de création du programme d'installation.

**Pour accéder aux options avancées de l'installation réseau** :

1. Sélectionnez l'option "Paramètres de l'installation .. Assistant".

2. Cochez l'option "Installation réseau avec mise à jour automatique". Les paramètres pour l'installation réseau apparaissent. 

3. Cliquez sur le bouton "Options avancées".




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Paramétrer les options de contrôle à distance
<a name="parametrer_les_options_controle_distance_ELTTEXTE000343"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000066"></a>Le contrôle à distance d'une application WINDEV permet au responsable d'une application utilisée en réseau de gérer l'arrêt automatique de l'application pour tous les utilisateurs en cours.

Pour plus de détails, consultez le [Principe du contrôle à distance](../WDLang1/3013006.md).

**Remarque** : Le paramétrage des options de contrôle peut être modifié par le responsable de l'application depuis le [programme de contrôle à distance des exécutables WINDEV](../WDAdminExe/2028052.md) (WDADMINEXE) installé sur le poste serveur.
<a name="NOTE3_2"></a>
Le paramétrage des options de contrôle à distance est effectué dans l'onglet "Paramètres de contrôle" des options avancées de l'assistant. 

**Pour paramétrer les options de contrôle à distance :**


1. Spécifiez le délai (en minutes) écoulé entre chaque vérification.

2. Saisissez les différents messages affichés sur le poste de l'utilisateur final : 

	- le message d'interdiction de se connecter. 
			Ce message s'affichera si les utilisateurs tentent de se connecter à l'application alors que l'option "Connexion interdite" est cochée dans le programme d'administration "WDADMINEXE".

	- le message d'arrêt imminent.
			Ce message s'affichera si les utilisateurs sont connectés à l'application et si l'option "Avertissement d'un arrêt imminent" est cochée dans le programme d'administration "WDADMINEXE".

	- le message d'arrêt forcé.
			Ce message s'affichera si les utilisateurs sont connectés à l'application et si l'option "Arrêt forcé" est cochée dans le programme d'administration "WDADMINEXE".

	- le message de reprise d'installation.
			Ce message s'affichera si les utilisateurs tentent de se connecter à l'application alors que l'installation de la dernière mise à jour a échoué.







Remarque : Si aucun message n'est saisi, le message suivant s'affichera : "La précédente mise à jour de l'application ne s'est pas terminée correctement et l'application ne peut pas s'exécuter. Voulez-vous relancer la mise à jour ?".
<a name="NOTE3_3"></a>


### Important : modification des options de contrôle dans WDADMINEXE
<a name="important_modification_des_options_controle_dans_wdadminexe_ELTPARAGRAPHE000109"></a>Si les options de contrôle à distance sont modifiées directement dans le programme d'administration "WDADMINEXE", les nouvelles valeurs définies dans l'administrateur sont conservées par défaut. Le lien "Rétablir les valeurs par défaut" présent dans WDADMINEXE permet de rétablir les dernières valeurs spécifiées dans la dernière installation réalisée.

## Modifier les conditions d'application de la mise à jour
<a name="modifier_les_conditions_application_mise_jour_ELTTEXTE000379"></a>
Pour informer les utilisateurs qu'une mise à jour de l'application est disponible, la fenêtre suivante s'affiche au lancement de l'application. <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=FenMiseAJour.gif)
Cette fenêtre permet selon le cas d'installer ou non la mise à jour.

Les différentes options de validation automatique sont :

- ***Ne pas valider automatiquement les mises à jour***. 
	L'utilisateur devra obligatoirement valider la fenêtre.
	Si la mise à jour est obligatoire, l'utilisateur devra installer cette mise à jour pour utiliser l'application. 
	Si la mise à jour n'est pas obligatoire, l'utilisateur pourra au choix installer la mise à jour ou lancer directement l'application.

- ***Appliquer automatiquement la mise à jour si elle est proposée*** (option par défaut). 
	Un clic sur le bouton "Actualiser" sera effectué à la fin du délai de "Validation automatique".
	Pendant ce délai, l'utilisateur pourra annuler la mise à jour ou lancer directement l'application si la mise à jour n'est pas obligatoire.

- ***Appliquer la mise à jour uniquement si elle est obligatoire***. 
	Si la mise à jour est obligatoire, un clic sur le bouton "Actualiser" sera effectué à la fin du délai de "Validation automatique". Pendant ce délai, l'utilisateur pourra annuler la mise à jour. 
	Si la mise à jour n'est pas obligatoire, un clic sur le bouton "Ignorer" sera effectué à la fin du délai de "Validation automatique". Pendant ce délai, l'utilisateur pourra installer la mise à jour.

- ***Annuler le lancement de l'application si la mise à jour est obligatoire***. 
	Un clic sur le bouton "Annuler" sera effectué à la fin du délai de "Validation automatique".
	Pendant ce délai, l'utilisateur pourra installer la mise à jour ou lancer directement l'application.




**Remarque** : Par défaut, quelle que soit la langue d'exécution du programme d'installation, cette fenêtre s'affiche uniquement en français. Pour afficher cette fenêtre dans une langue autre que le français, il suffit de traduire le projet WDSetup.
<a name="NOTE4_2"></a>
Le paramétrage des conditions d'application de la mise à jour est effectué dans l'onglet "Mise à jour automatique" des options avancées de l'assistant. 

**Pour modifier les conditions d'application de la mise à jour** : 

1. Sélectionnez l'option de validation automatique voulue (voir ci-dessus).

2. Spécifiez le délai (en secondes) après lequel la fenêtre d'information sera validée ou fermée (60 secondes par défaut). 
	Selon l'option de "Mise à jour automatique" sélectionnée :

	- soit la mise à jour sera automatiquement installée.

	- soit l'application sera lancée sans modification.

	- soit le lancement de l'application sera annulé.







Ce délai n'est pas pris en compte si l'option "Ne pas valider automatiquement les mises à jour" est cochée.

Le bouton "Paramètres de la vérification de la présence de mises à jour" permet de configurer : 

- **la fréquence de vérification de la disponibilité des mises à jour**. 
	Par défaut, la vérification de la disponibilité éventuelle d'une mise à jour sur le serveur est effectuée à chaque lancement de l'application. Cette fréquence de vérification est paramétrable. Elle peut être effectuée : 

	- A chaque ouverture de l'application. 

	- Tous les X jours. 




- **les interactions avec l'utilisateur lors de la vérification de la disponibilité d'une mise à jour**. 
	Une application installée via une installation réseau vérifie à chaque lancement si une mise à jour est disponible sur le serveur de référence. 

	- Si une version plus récente est disponible sur le serveur, la mise à jour de l'application est proposée à l'utilisateur final. 

	- Si la connexion réseau n'est pas trouvée à l'ouverture de l'application, le traitement par défaut est d'afficher un message d'avertissement. 
			Ce message peut être inutile dans le cadre d'une application utilisée sur un portable, connecté par intermittence. Dans ce cas, il suffit de sélectionner l'option "Vérification silencieuse : Ne pas informer l'utilisateur". 







Il est possible d'autoriser la modification de ces options pendant l'installation de la version de référence.

## Afficher l'aide des nouveautés
<a name="afficher_aide_des_nouveautes_ELTTEXTE000409"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000198"></a>L'aide des nouveautés permet d'indiquer à l'utilisateur l'ensemble des modifications effectuées dans chaque mise à jour de l'application.

Cette aide sera affichée :

- lors de l'installation de la mise à jour sur les postes utilisateurs (bouton "Détail").

- dans l'onglet "Installation" du [programme de contrôle à distance des exécutables WINDEV](../WDAdminExe/2028052.md) (WDADMINEXE).




Les nouveautés de l'application peuvent être :

- soit saisi directement dans la fenêtre des options avancées de l'installation réseau.

- soit enregistré dans un fichier au format texte ou RTF.

- soit correspondre au fichier d'aide automatiquement généré lors de la création de l'exécutable. Pour plus de détails, consultez [Aide des Nouveautés](../Editeurs/2028084.md). 




Le paramétrage de l'aide des nouveautés est effectué dans l'onglet "Aide des nouveautés" des options avancées de l'assistant. 
<a name="NOTE5_2"></a>


### Saisir directement les nouveautés
<a name="saisir_directement_les_nouveautes_ELTPARAGRAPHE000224"></a>Pour saisir directement les nouveautés (par exemple les dernières modifications effectuées) dans la fenêtre des options avancées de l'installation réseau :

1. Cochez l'option "Texte".

2. Saisissez le texte voulu.

3. Pour insérer automatiquement :

	- le numéro de la version en cours, cliquez sur le bouton "Version".

	- la date du jour, cliquez sur le bouton "Date".






<a name="NOTE5_3"></a>


### Récupérer l'historique enregistré dans un fichier
<a name="recuperer_historique_enregistre_dans_fichier_ELTPARAGRAPHE000238"></a>Pour récupérer l'historique enregistré dans un fichier :

1. Cochez l'option "Fichier".

2. Sélectionnez le fichier contenant l'historique (option "Chemin du fichier de l'historique"). Ce fichier peut être au format texte (".TXT") ou RTF (".RTF" ou ".WRI" par exemple).




**Remarque** : Pour visualiser directement le fichier sélectionné, cliquez sur le bouton "Aperçu". Le fichier s'affichera :

- soit dans le Bloc Notes (fichier au format texte).

- soit dans Word (fichier au format RTF).

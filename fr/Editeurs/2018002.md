## Présentation
<a name="presentation_ELTTEXTE000192"></a>
Deux catégories de fichiers de données sont gérées :

- **les fichiers de données permettant d'identifier les utilisateurs**. Ces fichiers de données peuvent être communs à plusieurs applications. 
	Par défaut, ces fichiers de données sont créés dans un sous-répertoire de votre projet (répertoire EXE\\GPW_&lt;Nom du projet&gt;). Ces fichiers sont utilisés pour les tests de votre application. 
	Pour réaliser les tests de votre application, vous pouvez spécifier un répertoire différent ou utiliser le répertoire des données de vos applications (cette option est conseillée pour des tests sous Windows Vista avec le mécanisme de l'UAC activé). 

- **les fichiers de données permettant de gérer les droits des utilisateurs sur les différentes fenêtres ou pages de l'application**. Ces fichiers sont propres à l'application. 
	Par défaut, ces fichiers de données sont créés dans un sous-répertoire de votre projet (répertoire EXE\\GPW_&lt;Nom du projet&gt;). Ces fichiers sont utilisés pour les tests de votre application. 
	Pour réaliser les tests de votre application, vous pouvez spécifier un répertoire différent ou utiliser le répertoire des données de vos applications (cette option est conseillée pour des tests sous Windows Vista avec le mécanisme de l'UAC activé).




![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) **Remarque** : Lors de la création de l'exécutable (exécutable multi-instances), il est possible de définir l'emplacement des fichiers du groupware utilisateur. Pour plus de détails, consultez [Test et utilisation du groupware utilisateur](../Editeurs/2018009.md).

## Les différents fichiers
<a name="les_differents_fichiers_ELTTEXTE000216"></a>


### Fichiers de données permettant d'identifier les utilisateurs
<a name="fichiers_donnees_permettant_identifier_les_utilisateurs_ELTPARAGRAPHE000042"></a>Ces fichiers peuvent être communs à plusieurs applications :


|   |   |
| --- | --- |
| ***GPWUtilisateur*** | Fichier des utilisateurs. Contient toutes les caractéristiques des utilisateurs (login, nom, prénom, mot de passe, ...) |
| ***GPWUtilisateurConfiguration*** | Fichier des groupes d'utilisateurs. |
| ***GPWHistoriqueConnexion*** | Fichier contenant l'historique des connexions. Ce fichier est présent uniquement si l'historique des connexions a été demandé. |

**Où sont créés ces fichiers ?**

- Par défaut, ces fichiers de données (mode HyperFileSQL Classic) sont créés dans un sous-répertoire de votre projet (répertoire EXE\\GPW_&lt;Nom du projet&gt;). Lors de l'intégration du groupware utilisateur, vous pouvez spécifier un répertoire différent pour réaliser les tests de votre application. Il est possible d'utiliser des fichiers de données HyperFileSQL Client/Serveur. Pour plus de détails, consultez [Fichiers du groupware au format HyperFileSQL Client/Serveur](../Editeurs/2018011.md).
	![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Lors de la création de l'exécutable, il est possible de spécifier un répertoire spécifique pour ces fichiers de données communs.

- Lors de la préparation du programme d'installation, il est possible de permettre à l'utilisateur de paramétrer l'emplacement de ces fichiers (plan intitulé "Modules additionnels"). Dans ce cas, l'emplacement des fichiers du groupware utilisateur sera mémorisé directement dans l'exécutable, et sera mis à jour lors de l'installation de l'exécutable. Il est ainsi possible de sélectionner un répertoire commun à plusieurs applications gérant le groupware utilisateur.

- Si l'option "Lancement manuel" a été sélectionnée, la fonction [gpwOuvre](../WDLang6/3041002.md) permet de lancer la fenêtre ou page de login du groupware utilisateur, mais aussi de personnaliser le répertoire des fichiers de données du groupware utilisateur. Quels que soient les répertoires indiqués dans les différentes étapes de la création et de l'installation de l'application, ce répertoire, s'il est précisé, sera pris en compte.




**Remarque** : si aucun paramétrage spécifique n'est effectué, les fichiers de données Utilisateur (HyperFileSQL Classic) seront créés dans un sous-répertoire du répertoire d'installation de l'exécutable (ou de la bibliothèque dans le cas d'un site WEBDEV) nommé \\GPW_&lt;Nom du projet&gt;.

Pour plus de détails :

- sur la structure de ces fichiers, consultez [Structure des fichiers du groupware utilisateur](../Editeurs/2018005.md)

- sur la gestion des fichiers de données dans une application Client/Serveur, consultez [Groupware utilisateur et Client/Serveur](../Editeurs/2018011.md).

### Fichiers de données permettant de gérer les droits
<a name="fichiers_donnees_permettant_gerer_les_droits_ELTPARAGRAPHE000090"></a>Ces fichiers sont spécifiques à l'application. Ils ne peuvent être partagés avec une autre application.


|   |   |
| --- | --- |
| ***GPWConfiguration*** | Fichier contenant les descriptions des différentes configurations disponibles. Chaque configuration est associée à un groupe d'utilisateurs. |
| ***GPWConfigurationElément*** | Etat de chaque élément de l'application pour chaque configuration. Par exemple permet de stocker si pour l'utilisateur "Secrétaire", l'option "Menu statistiques" doit être grisé ou non. |
| ***GPWElément*** | Fichier content les différents éléments de l'application à configurer (champs des fenêtres ou des pages, options de menu, ...) |

**Où sont créés ces fichiers ?**

- Par défaut, ces fichiers de données (mode HyperFileSQL Classic) sont créés dans un sous-répertoire de votre projet (répertoire EXE\\GPW_&lt;Nom du projet&gt;). Il est possible d'utiliser des fichiers de données HyperFileSQL Client/Serveur. Pour plus de détails, consultez [Fichiers du groupware au format HyperFileSQL Client/Serveur](../Editeurs/2018011.md).

- Lors de la préparation du programme d'installation, il est possible de permettre à l'utilisateur de paramétrer l'emplacement de ces fichiers (plan intitulé "Modules additionnels"). Il est ainsi possible de sélectionner un répertoire spécifique pour les fichiers des droits de l'application.

- Si l'option "Lancement manuel" est sélectionnée, la fonction [gpwOuvre](../WDLang6/3041002.md) permet de lancer la fenêtre ou la page de login du groupware utilisateur, mais aussi de personnaliser le répertoire des fichiers de données du groupware utilisateur. Quels que soient les répertoires indiqués dans les différentes étapes de la création et de l'installation de l'application, ce répertoire, s'il est précisé, sera pris en compte.




**Remarque** : si aucun paramétrage spécifique n'est effectué, les fichiers de données des droits (HyperFileSQL Classic) seront créés dans un sous-répertoire du répertoire d'installation de l'exécutable (ou de la bibliothèque dans le cas d'un site WEBDEV) nommé \\GPW_&lt;Nom du projet&gt;.

Pour plus de détails :

- sur la structure de ces fichiers, consultez [Structure des fichiers du groupware utilisateur](../Editeurs/2018005.md)

- sur la gestion des fichiers de données dans une application Client/Serveur, consultez [Groupware utilisateur et Client/Serveur](../Editeurs/2018011.md).

### Mot de passe des fichiers de données
<a name="mot_passe_des_fichiers_donnees_ELTPARAGRAPHE000134"></a>Par défaut, les fichiers de données du groupware utilisateur ont un mot de passe spécifique. Ce mot de passe est : ***"PCSGPW2001***".

Pour changer ce mot de passe :

1. Affichez la fenêtre de configuration du groupware utilisateur (option "Atelier .. Paramétrer le Groupware utilisateur").

2. Dans l'onglet "Fichiers", saisissez le nouveau mot de passe. Les caractères saisis apparaissent sous forme d'étoiles.




**Attention** : Si les fichiers du groupware existent déjà, leur mot de passe n'est pas modifié. Cette modification est possible uniquement avant la création des fichiers de données.

**Remarque** : Ce mot de passe est utilisé par exemple lors de la ré-indexation des fichiers ou lors de l'ouverture des fichiers avec l'outil WDMAP.

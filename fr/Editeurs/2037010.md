## Configuration du serveur pour le protocole CGI (AWP)
<a name="configuration_serveur_pour_protocole_cgi_awp_ELTTEXTE000122"></a>
Pour configurer le serveur Web :

1. Ouvrez le gestionnaire de service Internet Microsoft.

2. Sélectionnez l'option " Action .. Nouveau .. Répertoire virtuel".

3. Dans l'assistant, indiquez l'alias du protocole CGI : WD280AWP. Cliquez sur Suivant.

4. Sélectionnez le répertoire du protocole CGI (sous-répertoire Programmes\\MOTEUR\\WINDOWS\\AWP du répertoire d'installation de WEBDEV). Cliquez sur Suivant.

5. Donnez les permissions suivantes : Lire - Exécuter.

6. Validez.

## Configuration du serveur pour les sites WEBDEV
<a name="configuration_serveur_pour_les_sites_webdev_ELTTEXTE000146"></a>
Pour configurer le serveur Web : 

1. Ouvrez le gestionnaire de service Internet Microsoft.

2. Sélectionnez l'option " Action .. Nouveau .. Répertoire virtuel".

3. Dans l'assistant, indiquez l'alias du répertoire contenant les images de votre projet : &lt;MonProjet&gt;_WEB. Par exemple MARKET_WEB.

4. Sélectionnez le répertoire contenant les images de votre projet : &lt;Nom Projet&gt;_WEB. 
	Par exemple : C:\\WWprojet\\MARKET\\MARKET_WEB.

5. Donnez les permissions suivantes : Lire - Scripts.

6. Validez.

## Configuration du serveur pour les sites utilisant des pages AWP (Active Web Pages)
<a name="configuration_serveur_pour_les_sites_utilisant_des_pages_awp_active_web_pages_ELTTEXTE000170"></a>
Pour configurer le serveur :

1. Ouvrez le gestionnaire de service Internet Microsoft.

2. Sélectionnez l'option "Action .. Propriétés".

3. Affichez l'onglet "Répertoire de base".

4. Cliquez sur le bouton "Configuration".

5. Cliquez sur le bouton "Ajouter".

6. Dans le champ exécutable, sélectionnez le moteur AWP (sous-répertoire Programmes\\Moteur\\Windows\\AWP\\WD280AWP.exe du répertoire d'installation de WEBDEV).

7. Saisissez l'extension ".awp" et validez.

## Cas particulier : Projets partagés en réseau
<a name="cas_particulier_projets_partages_reseau_ELTTEXTE000194"></a>
Après la première ouverture d'un projet WEBDEV partagé en réseau, il est nécessaire de modifier la configuration du projet sous IIS.

1. Ouvrez le gestionnaire de service Internet Microsoft.

2. Sélectionnez l'alias correspondant à votre projet (par exemple MONSITE_WEB).

3. Sélectionnez l'option "Propriétés" du menu contextuel de l'alias.

4. Dans l'onglet "Répertoire virtuel" : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=IIS_Partage_Reseau.gif)


	- 1 : Cochez l'option "D'un partage situé sur un autre ordinateur".

- 2 : Indiquez le répertoire réseau : Exemple :\\\\MonServeur\\Partage\\MONSITE_WEB

- 3 : Cliquez sur "Se connecter en tant que".

- 4 : Indiquez un nom d'utilisateur qui a les droits d'accès au répertoire réseau spécifié en 2 (et son mot de passe)




## Editeur d'analyses : Décrire une connexion
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000168"></a>
Tout programme réalisé avec WINDEV, WEBDEV ou WINDEV Mobile peut accéder facilement :

- à une base de données HFSQL.

- à une base de données autre que HFSQL.




Dans tous les cas, il suffit de décrire la structure de la base de données dans l'analyse.

Les connexions permettent de lier la description d'un fichier de données avec les données réelles. Par exemple, lors de l'importation dans l'éditeur d'analyses d'un fichier appartenant à une base de données externe, une connexion est automatiquement créée.

Les connexions sont créées :

- soit lors de l'importation d'un fichier de données appartenant à une base de données existante. Pour plus de détails, consultez [Manipuler une base de données externe](../Editeurs/2011025.md).

- soit depuis l'éditeur d'analyses : sous le volet "Analyse", dans le groupe "Connexion", cliquez sur "Connexions".

## Création d'une connexion
<a name="creation_une_connexion_ELTTEXTE000198"></a>


### Créer une connexion (Editeur d'analyses)
<a name="creer_une_connexion_editeur_analyses_ELTPARAGRAPHE000047"></a>Pour créer une connexion dans l'éditeur d'analyses :

1. Sous le volet "Analyse", dans le groupe "Connexion", cliquez sur "Nouvelle connexion".

2. Dans l'assistant de création d'une connexion, sélectionnez le type de connexion à créer :

	- HFSQLDrive (solution payante). 

	- PCSCloud (solution payante).

	- HFSQL Client/Serveur.

	- HFSQL Classic.

	- une base externe.




3. **Lors de la création d'une connexion pour HFSQLDrive**, indiquez dans les différentes étapes de l'assistant les caractéristiques de votre base de données HFSQLDrive (pour plus de détails sur HFDSQLDrive, consultez [HFSQLDrive, votre base de données dans le Cloud en 3 clics](https://pcscloud-drive.net/FR/HFSQLDrive/index.awp)) :

	- le nom ou l'adresse IP du serveur

	- le numéro de port du serveur.

	- le nom d'utilisateur et le mot de passe.

	- le nom de la base de données.

	- le nom de la connexion.




4. **Lors de la création d'une connexion pour PCSCloud**, indiquez dans les différentes étapes de l'assistant les caractéristiques de votre compte PCSCloud (pour plus de détails sur PCSCloud, consultez [Cloud pour base de données HFSQL](https://pcscloud.net/FR/cloud_applications.awp)) :

	- le nom ou l'adresse IP du serveur

	- le numéro de port du serveur.

	- le nom d'utilisateur et le mot de passe.

	- le nom de la base de données.

	- le nom de la connexion.




5. **Lors de la création d'une connexion pour HFSQL Client/Serveur**, indiquez dans les différentes étapes de l'assistant :

	- le nom ou l'adresse IP du serveur

	- le numéro de port du serveur.

	- le nom d'utilisateur et le mot de passe.

	- le nom de la base de données.

	- le nom de la connexion.




6. **Lors de la création d'une connexion pour HFSQL Classic**, indiquez dans les différentes étapes de l'assistant :

	- le nom de la connexion.

	- le répertoire par défaut des fichiers de données. 




7. **Lors de la création d'une connexion à une base externe**, indiquez dans les différentes étapes de l'assistant :

	- le type d'accès utilisé pour la connexion

	- le nom ou l'adresse IP du serveur ou le chemin du fichier.

	- le nom d'utilisateur et le mot de passe.

	- le nom de la base de données.

	- le nom de la connexion.







Une fois créée, la connexion peut être associée à un fichier de données dans la fenêtre de description du fichier de données.

## Intérêt des connexions HFSQL Classic
<a name="interet_des_connexions_hfsql_classic_ELTTEXTE000222"></a>


- L'utilisation d'une connexion pour les fichiers de données HFSQL Classic permet de simplifier les opérations de changement de connexion. Pour utiliser par exemple une base de données HFSQL Client/Serveur ou Oracle, il suffit de changer la connexion définie dans l'analyse (en utilisant par exemple la fonction [HChangeConnexion](../WDLang4/3044150.md)).

- L'utilisation d'une connexion HFSQL Classic est recommandée lors de l'utilisation de la réplication universelle assistée.

## Présentation
<a name="presentation_ELTTEXTE000215"></a>
Le composant **Feedback** est un composant livré avec WINDEV. Ce composant permet aux utilisateurs finaux de vos applications de vous faire parvenir :

- des suggestions sur l'application.

- des problèmes rencontrés sur l'application.

- des règles métier.




La réception des différents éléments est réalisée via les Centres de Contrôle.

Si l'utilisateur final n'est pas connecté au même réseau, les éléments envoyés proviennent par email. L'utilisateur est identifié par le nom indiqué dans l'email.

Si l'utilisateur est connecté au même réseau (c'est-à-dire s'il a accès aux données des Centres de Contrôle, via réseau ou ADSL), les éléments envoyés sont automatiquement ajoutés aux données des Centres de Contrôle.

**Remarque** : Si votre application utilise le menu automatique (menu "?"), le composant Feedback est automatiquement intégré dans votre application si vous sélectionnez l'option "Envoyer une suggestion, un dysfonctionnement ou une règle métier".

**Application multilingue** : Le composant Feedback est disponible en version française et anglaise uniquement. Pour les autres langues, les assistants du composant seront affichés en anglais par défaut. Il est cependant possible de faire traduire les différents messages du composant Feedback. Pour plus de détails, consultez [Traduction des composants](../Editeurs/2014014.md).

Le composant **Feedback**, s'il est utilisé par l'application, sera appelé automatiquement depuis la fenêtre d'erreur du WLangage et permettra de transmettre un dump de débogage de l'application en cours.

## Envoi d'une suggestion, d'un problème ou d'une règle métier avec le composant Feedback
<a name="envoi_une_suggestion_probleme_une_regle_metier_avec_composant_feedback_ELTTEXTE000239"></a>


### Envoi d'une suggestion, d'un problème ou d'une règle métier avec le composant Feedback
<a name="envoi_une_suggestion_probleme_une_regle_metier_avec_composant_feedback_ELTPARAGRAPHE000039"></a>Grâce au composant Feedback, l'utilisateur final peut vous envoyer simplement diverses informations. Il lui suffit de :

1. Lancer l'assistant du composant Feedback :

	- soit grâce au menu automatique (option "? .. Envoyer une suggestion, un dysfonctionnement ou une règle métier").

	- soit grâce à une option de votre application qui gère le composant Feedback.




2. Choisir dans l'assistant le type de demande à effectuer : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=MesIncidents2.gif&type=thumb)


3. Indiquer ses coordonnées (non obligatoire).

4. Indiquer les informations concernant le problème, la suggestion ou la règle métier.

5. Insérer si nécessaire une copie d'écran (avec des commentaires).




Le résumé de la demande est alors affiché. Cette demande peut être enregistrée et/ou imprimée.

Lors de la validation, la demande est directement envoyée :

- Si l'utilisateur est en mode déconnecté (la base de données des Centres de Contrôle n'est pas accessible), la demande est envoyée par email.

- Si l'utilisateur est en mode connecté (la base de données des Centres de Contrôle est accessible via réseau ou internet/ADSL), la demande est directement enregistrée dans les fichiers des Centres de Contrôle.



<a name="NOTE2_2"></a>


### Consultation des demandes envoyées
<a name="consultation_des_demandes_envoyees_ELTPARAGRAPHE000063"></a>Si le menu automatique a été inclus dans l'application WINDEV, l'utilisateur peut simplement consultez les demandes déjà envoyées par l'option de menu "? .. Consulter les demandes envoyées".

Si l'utilisateur est en mode déconnecté, l'identifiant utilisé pour envoyer la demande (nom utilisé dans l'email) est utilisé.

Si l'utilisateur est en mode connecté, l'identifiant utilisé pour envoyer la demande (nom utilisé dans l'assistant) est utilisé.

## Configuration du composant Feedback
<a name="configuration_composant_feedback_ELTTEXTE000269"></a>


### Si les utilisateurs de l'application sont en mode déconnecté
<a name="les_utilisateurs_application_sont_mode_deconnecte_ELTPARAGRAPHE000076"></a>Si vous avez intégré le composant Feedbackgrâce au menu automatique (menu "?"), il est possible d'indiquer l'adresse email utilisée pour l'envoi des suggestions :

- soit directement dans l'assistant de création / modification du menu automatique.

- soit dans le code de l'option du menu automatique.

- soit lors de la création de l'exécutable.




**Conseil** : indiquez une adresse email, cette adresse sera automatiquement utilisée si les utilisateurs ne sont pas connectés (problème d'accès au réseau, ...).
<a name="NOTE3_2"></a>


### Si les utilisateurs de l'application sont en mode connecté
<a name="les_utilisateurs_application_sont_mode_connecte_ELTPARAGRAPHE000089"></a>

- **Base de données des Centres de Contrôle en mode HFSQL Classic** : Aucune configuration spécifique n'est nécessaire.

- **Base de données des Centres de Contrôle en mode HFSQL Client/Serveur** : Si vous avez intégré le composant Feedback grâce au menu automatique (menu "?"), l'utilisateur pris en compte pour la connexion à la base est l'utilisateur en cours. Il est conseillé de :

	- créer un intervenant "Invité" dans le Centre de Suivi de projets (n'ayant pas de droits administrateur). L'envoi des suggestions, règles métier (par exemple) sera réalisé via la connexion de cet utilisateur.

	- utiliser les paramètres de l'intervenant "Invité" dans le code de l'option "? .. Envoyer des suggestions, ...".

		1. Affichez la fenêtre qui utilise le menu automatique. Sous le volet "Fenêtre", dans le groupe "Barres et menus", déroulez "Menu principal" et sélectionnez "Ouvrir le menu '?'".

		2. Editez le code de l'option "Envoyer une suggestion, un dysfonctionnement, une règle métier".

		3. Modifiez la ligne de code permettant d'indiquer le chemin des fichiers de données. La ligne de code à utiliser est la suivante :
						
			```wl
			pFeedback.Configure(fbCheminDonnéesCC, ...
					"hfcs://<utilisateur>:<mot de passe>@<serveur>:<port>/<base de données>")
			```

						Dans ce code :

			- &lt;utilisateur&gt; : Nom d'un utilisateur du Centre de Contrôle.

			- &lt;mot de passe&gt; : Mot de passe associé à l'utilisateur.

			- &lt;serveur&gt; : Adresse IP ou nom du serveur où la base de données des Centres de Contrôle est installée.

			- &lt;port&gt; : Port du serveur.

			- &lt;base&gt; : Nom de la base de données HFSQL Client/Serveur correspondant aux fichiers des centres de contrôle.
										Remarque : si vous mettez à jour le menu automatique de votre application, cette ligne de code sera automatiquement mise à jour avec le nom du développeur en cours.
										Si vous avez intégré le composant Feedback indépendamment du menu automatique (menu "?"), consultez la documentation du composant.

## Récupérer et traiter les retours client
<a name="recuperer_traiter_les_retours_client_ELTTEXTE000305"></a>
Les Centres de Contrôle permettent de récupérer les bugs, suggestions et règles métier envoyés via le composant Feedback par les utilisateurs. Pour plus de détails, consultez [Comment récupérer les bugs et suggestions des clients ?](../CCSuivi/3540719.md).

## Composant Feedback : les différentes fonctions du composant
<a name="composant_feedback_les_differentes_fonctions_composant_ELTTEXTE000329"></a>
Le composant Feedback propose les fonctions suivantes pour personnaliser son utilisation. Pour plus de détails, consultez directement la documentation complète du composant Feedback (Option "Description" du menu contextuel du composant).


|   |   |
| --- | --- |
| **Configure** | Permet de configurer les options du composant Feedback. |
| **DemandeAjoute** | Permet d'ajouter directement une demande dans le spool du Centre de Suivi de Projets. |
| **EnModeTestEmail** | Permet de forcer une application à utiliser l'envoi d'emails même si la base de données des Centres de Contrôle est accessible (envoi d'emails en mode connecté). |
| **ErreurMessage** | Permet de récupérer le message concernant la dernière erreur rencontrée. |
| **NouvelleDemande** | Permet d'ouvrir l'assistant de création d'une suggestion, d'un dysfonctionnement ou d'une règle métier. <br><br>Pour fonctionner en mode déconnecté, il est nécessaire de configurer l'adresse email de réception des retours clients (soit par la fonction **Configure**, soit en renseignant le champ Adresse Email en créant l'exécutable). |

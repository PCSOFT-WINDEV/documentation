


## Installation d'un site WEBDEV : Modification automatique des fichiers de données
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000095"></a>
La modification automatique des fichiers de données permet de mettre à jour la description des fichiers de données présents sur le serveur Web.

Pour plus de détails, consultez la [Modification automatique des fichiers de données](../WDLang4/3044195.md).

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Paramétrer la modification automatique des données
<a name="parametrer_modification_automatique_des_donnees_ELTTEXTE000119"></a>

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=mofiauto.gif&type=thumb)
Pour effectuer la modification automatique des fichiers de données :

- au format HFSQL Classic, cochez l'option "Inclure la modification automatique des fichiers dans l'installation" et vérifiez l'analyse associée au site.

- au format HFSQL Client/Serveur, cochez l'option "Effectuer la modification des fichiers de données HFSQL Client/Serveur". Le bouton "Connexions" permet de décrire les paramètres des connexions HFSQL Client/Serveur concernées.

- au format Hyper File 5.5 vers HFSQL Classic, cochez l'option "Effectuer la modification automatique des données Hyper File 5.5 et la conversion vers HFSQL Classic".



<a name="NOTE2_2"></a>


### Gestion des mots de passe pour les fichiers de données HFSQL Classic
<a name="gestion_des_mots_passe_pour_les_fichiers_donnees_hfsql_classic_ELTPARAGRAPHE000033"></a>Si vos fichiers de données HFSQL Classic sont protégés par mot de passe, cliquez sur le bouton "Mots de passe".

Pour chaque fichier de l'analyse, spécifiez si nécessaire :

- le mot de passe du fichier (s'il existe) dans l'analyse au format HFSQL Classic. 
	**Remarque** : Pour modifier ce mot de passe, cochez l'option "Modifier le mot de passe" et saisissez deux fois le nouveau mot de passe.

- si le site utilise une analyse Hyper File 5.5 migrée :

	- l'analyse d'origine au format Hyper File 5.5 (fichier ".WDA").

	- le nom du fichier dans l'analyse au format Hyper File 5.5.

	- le mot de passe du fichier (s'il existe) dans l'analyse au format Hyper File 5.5.







**Remarque** : Toutes ces informations à l'exception des mots de passe sont automatiquement affichées.




## Analyse : Importer des fichiers dans l'analyse (base externe, fichiers HFSQL, ...) 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000200"></a>
WINDEV et WEBDEV permettent de manipuler simplement une base de données externe (Oracle, SQL Server, Access, ...) en utilisant directement les fonctions du WLangage.

Pour manipuler une base de données externe, il suffit de :

1. Importer les fichiers (ou les tables) de la base de données externe vers l'Analyse. Lors de cette importation, les données peuvent être :

	- soit conservées dans leur format actuel. Aucune conversion n'est nécessaire.
			**Remarque** : Si la base de données évolue, il est possible de prendre en compte ces évolutions. Il suffit de [synchroniser l'analyse avec la base de données externe](../Editeurs/2011024.md).

	- soit converties au format HFSQL.




2. Générer l'analyse.

3. Manipuler les fichiers de données importés grâce aux fonctions du WLangage. Pour plus de détails, consultez [Accès aux bases de données](../WDLang4/3044202.md).




**Remarques** : 

- Il est également possible de convertir uniquement les données d'un fichier de données grâce à l'[outil WDConver](../WDConver/3512006.md). Pour réaliser cette opération, la description du fichier de données doit avoir été importée dans l'analyse.

- L'éditeur d'analyses permet de réaliser les manipulations suivantes : 

	- Importer un fichier de données d'une base externe dans une analyse au format HFSQL Classic.

	- Importer un fichier de données au format HFSQL Classic dans une nouvelle analyse au format HFSQL Classic.

### Importer un fichier de données dans une analyse
<a name="importer_fichier_donnees_dans_une_analyse_ELTPARAGRAPHE000106"></a>Pour importer un fichier de données dans une analyse : 

1. Sous l'éditeur d'analyses, ouvrez l'analyse dans laquelle l'importation doit être effectuée.

2. Sous le volet "Analyse", dans le groupe "Création", déroulez "Importer" et sélectionnez "Importer des descriptions de fichiers/tables". L'assistant d'importation de fichiers de données se lance.

3. Sélectionnez la source de données voulue.

4. **Si vous réalisez une importation d'une base externe** :

	- Pour les sources de données utilisant un Connecteur Natif, une connexion à la base de données externe est automatiquement créée et associée aux fichiers de données importés dans l'analyse. Cette connexion devra être utilisée pour manipuler les fichiers de données par programmation.

	- Précisez si nécessaire les paramètres de la connexion : serveur de données, utilisateur et mot de passe, base de données, nom de la connexion, ...

	- Indiquez le mode d'accès aux données. Il est possible de : 

		- accéder directement aux données dans leur format actuel. Seule la description des tables est importée dans l'analyse. Aucune conversion n'est nécessaire.

		- convertir les données dans le format HFSQL Classic.




- Sélectionnez la (ou les) table(s) (fichiers) à importer.

5. **Si vous réalisez une importation de fichiers de données HFSQL Classic** :

	- Sélectionnez le fichier de données (extension .FIC) au format HFSQL Classic à récupérer.

	- Précisez si nécessaire le mot de passe de ce fichier de données.




6. Validez l'importation. L'analyse en cours est automatiquement générée.

7. Si les données doivent être converties au format HFSQL Classic :

	- Indiquez le répertoire de création des fichiers de données HFSQL Classic.

	- Sauvegardez si nécessaire la description de la conversion (script). Cette sauvegarde crée un fichier d'extension ".WDV" dans le répertoire spécifié. Ce script pourra être ré-utilisé par l'outil [WDConver](../WDConver/3512006.md).

	- Validez la conversion des données.







Remarques : 

- Pour réaliser l'import d'un fichier HFSQL classic dans une analyse, il est également possible de faire un simple "Drag and Drop" du fichier présent dans l'explorateur de fichier Windows vers l'éditeur d'analyse. L'assistant d'importation se lance automatiquement.

- A partir de la version 19, il n'est plus possible de réaliser une importation de fichiers de données Hyper File 5.5. Cette importation ne peut être réalisée qu'avec WINDEV ou WEBDEV 18 ou inférieure.




## Génération d'une description d'un fichier de données à partir d'une fenêtre ou d'une page
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000131"></a>
Lorsque vous concevez une maquette rapide, il arrive souvent de créer rapidement quelques fenêtres (ou quelques pages), sans définir de fichiers de données. Lorsque la maquette est validée, et seulement à ce moment-là, vous décidez de créer les structures des fichiers de données.

Plutôt que de créer ces fichiers de données "à la main", WINDEV, WEBDEV et WINDEV Mobile vous proposent d'exporter le contenu de la fenêtre (ou de la page) pour créer automatiquement la structure du fichier de données correspondante.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000155"></a>
Pour générer une description d'un fichier de données à partir d'une fenêtre ou d'une page : 

1. Ouvrez la fenêtre (ou la page) à partir de laquelle vous souhaitez générer la description du fichier de données.

2. Lancez l'assistant de génération d'une description de fichier : 

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Sous le volet "Fenêtre", dans le groupe "Edition", déroulez "Autres actions" et sélectionnez "Générer une description de fichier". Il est également possible de sélectionner l'option "Générer une description de fichier" du menu contextuel de la fenêtre. 




3. Spécifiez :

	- le nom logique du fichier de données. Ce nom sera utilisé pour manipuler le fichier de données. Par défaut, ce nom correspond au nom de la fenêtre/page.

	- le libellé du fichier résumant succinctement le sujet du fichier de données. Par défaut, ce libellé correspond au titre de la fenêtre/page.

	- la représentation d'un enregistrement dans le fichier de données. Cette représentation permet d'améliorer la lisibilité des questions posées lors de la description des liaisons. Cette option doit être précédée d'un article indéfini (UN ou UNE).




4. Passez à l'étape suivante.

5. Sélectionnez les champs à partir desquels les rubriques du fichier de données doivent être créées. Seuls les champs qui peuvent être liés à une rubrique sont proposés. Passez à l'étape suivante.

6. Indiquez si nécessaire le type de clé de chaque rubrique (non clé, clé unique ou clé avec doublons). Passez à l'étape suivante.

7. L'assistant présente la description du fichier de données à générer. Validez.




Le nouveau fichier de données devient le fichier de données en cours dans l'éditeur d'analyses. Pour le modifier, double-cliquez sur le fichier de données pour afficher la description des rubriques du fichier.
Pour plus de détails sur les manipulations d'un fichier de données ou d'une rubrique, consultez [Manipulations d'un fichier de données](../Editeurs/2011007.md) et [Manipulations d'une rubrique](../Editeurs/2011009.md).

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Correspondance champ - rubrique
<a name="correspondance_champ_rubrique_ELTTEXTE000179"></a>
Tous les champs de la fenêtre / page pouvant être liés à une rubrique (champ de saisie, sélecteur, table, ...) peuvent devenir des rubriques dans la description du fichier de données générée.

Le type et la taille de la rubrique dépend du type d'information affichée dans le champ (texte, numérique, ...) et de la taille de saisie.

L'ensemble des caractéristiques du champ sont reprises dans les informations partagées de la rubrique.


## Les rubriques clé
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000133"></a>
Les clés sont des rubriques spécifiques de votre analyse. Ces rubriques permettent :

- **d'identifier facilement les enregistrements**. Par exemple, dans un fichier produit, la référence du produit permet d'identifier le produit. Cette référence est unique pour chaque produit.

- **d'organiser les données stockées dans un fichier de données**. Par exemple, une clé définie sur la rubrique Nom du fichier client, permettra de trier rapidement les enregistrements du fichier de données par nom.




Plusieurs types de clés peuvent être définies dans un fichier de données :

- [Clé unique](#NOTE2_1).

- [Clé avec doublons](#NOTE3_1).

- Clé primaire.

- [Clé simple ou clé composée](#NOTE4_1).






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Clé unique
<a name="cle_unique_ELTTEXTE000157"></a>
Une clé unique est une clé pour laquelle une seule valeur pourra exister dans le fichier de données.

Par exemple, dans un fichier "PRODUIT", chaque produit est identifié par une référence unique. La rubrique "REFERENCE" doit être une clé unique.

**Remarques** : 

- Si aucune rubrique ne peut être utilisée comme clé unique, utilisez une rubrique de type "Identifiant automatique". Pour chaque enregistrement, la valeur de cette rubrique sera un entier unique.

- Cas d'une clé unique ayant pour valeur par défaut la valeur NULL : La valeur NULL au niveau de l'index est considérée comme une chaîne vide ("") pour les rubriques de type chaîne ou comme la valeur 0 pour les rubriques numériques. Il est donc impossible d'utiliser plusieurs fois la valeur par défaut NULL pour une clé unique. 






<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Clé avec doublons
<a name="cle_avec_doublons_ELTTEXTE000181"></a>
Une clé avec doublons est une clé pour laquelle plusieurs valeurs identiques sont possibles dans le fichier de données.

Par exemple, dans le fichier "CLIENT", il est possible de définir une clé avec doublons sur la rubrique "NOM". Le contenu de cette clé pourra être le même dans différents enregistrements (cas des homonymes). Grâce à cette clé, une recherche pourra être facilement effectuée sur le nom du client.







<a name="NOTE3b"></a>
<a name="NOTE3b_1"></a>


## Clé primaire
<a name="cle_primaire_ELTTEXTE000205"></a>
Une clé primaire est une clé unique qui ne peut pas être nulle. Une seule clé primaire peut être définie pour un fichier de données. 

Par exemple, dans le fichier "PRODUIT", il est possible de définir une clé primaire sur la rubrique "REFERENCE". Chaque produit est identifié par une référence unique et non nulle.



<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Clé simple ou clé composée
<a name="cle_simple_cle_composee_ELTTEXTE000229"></a>
Une clé simple est une clé définie sur une rubrique.

Une clé composée permet de définir une clé sur plusieurs rubriques du même fichier de données. Cette clé va permettre d'accéder aux enregistrements du fichier de données en fonction de la valeur combinée de deux rubriques.

Par exemple, pour simplifier une recherche des commandes, il est possible de définir une clé composée avec les rubriques "Numéro du Client" et "Date de la commande".

Remarque : Une clé composée peut être : 

- Une clé unique, 

- Une clé avec doublons,

- Une clé primaire. Dans ce cas, toutes les composantes de la clé doivent interdire la valeur "Null".

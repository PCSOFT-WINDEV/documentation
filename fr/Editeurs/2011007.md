## Manipulation d'un fichier de données
<a name="manipulation_fichier_donnees_ELTTEXTE000213"></a>


### Dupliquer/Copier un fichier de données
<a name="dupliquercopier_fichier_donnees_ELTPARAGRAPHE000034"></a>La duplication d'un fichier de données permet de disposer de deux fichiers de données identiques de noms différents.

Pour dupliquer/copier un fichier de données :

1. Sélectionnez le fichier de données à dupliquer.

2. Sous le volet "Accueil", dans le groupe "Edition", cliquez sur "Dupliquer". 
	Remarque : Il est également possible de sélectionner l'option "Dupliquer" dans le menu contextuel du fichier. 

3. Spécifiez les informations demandées et validez. Le fichier de données dupliqué est automatiquement créé dans l'analyse (MLD) en cours.



<a name="NOTE2_2"></a>


### Supprimer un fichier de données
<a name="supprimer_fichier_donnees_ELTPARAGRAPHE000061"></a>Pour supprimer un fichier de données :

1. Sélectionnez le fichier de données à supprimer.

2. Affichez le menu contextuel du fichier de données sélectionné (clic droit).

3. Sélectionnez l'option "Supprimer". Le fichier de données est automatiquement supprimé de l'analyse (MLD). Si des liaisons pointaient sur ce fichier de données, ces liaisons sont également supprimées.




**Remarque** : Lors de la création d'un fichier de données dont le nom est identique à celui d'un fichier de données précédemment supprimé, WINDEV, WEBDEV et WINDEV Mobile proposent de restaurer la description du fichier de données supprimé.
<a name="NOTE2_3"></a>


### Renommer un fichier de données
<a name="renommer_fichier_donnees_ELTPARAGRAPHE000074"></a>Pour renommer un fichier de données :

1. Sélectionnez le fichier de données à renommer.

2. Affichez la description du fichier de données : 

	- Sous le volet "Analyse", dans le groupe "Élément courant", cliquez sur "Description".

	- Affichez le menu contextuel du fichier (clic droit) et sélectionnez l'option "Description du fichier de données". 




3. Dans la fenêtre de description, modifiez le nom du fichier de données (et si nécessaire son nom sur disque). 

4. Validez. Le fichier de données est automatiquement renommé.



<a name="NOTE2_4"></a>


### Utiliser un dictionnaire
<a name="utiliser_dictionnaire_ELTPARAGRAPHE000101"></a>Un fichier de données peut simplement être intégré dans un dictionnaire pour être ré-utilisé par exemple dans une nouvelle application. L'intégration du fichier de données dans un dictionnaire se fait par simple "Drag and Drop" du fichier depuis l'éditeur d'analyses, vers le dictionnaire (affiché dans le [volet "Dictionnaire"](../Editeurs/2027014.md)).

Lors de son intégration dans le dictionnaire, la description du fichier de données peut être abonnée aux modifications. Toute modification réalisée dans la description du fichier de données pourra être reportée sur l'objet du dictionnaire et sur toutes les utilisations de cet objet.

Pour plus de détails, consultez le [dictionnaire](../Editeurs/2023002.md).
<a name="NOTE2_5"></a>


### Créer à vide le fichier de données correspondant
<a name="creer_vide_fichier_donnees_correspondant_ELTPARAGRAPHE000118"></a>Pour créer à vide le fichier de données physique correspondant :

1. Sélectionnez le fichier de données pour lequel un fichier physique doit être créé.

2. Affichez le menu contextuel du fichier (clic droit) et sélectionnez l'option "Créer le fichier de données".

3. Sélectionnez le répertoire dans lequel le fichier de données doit être créé et validez.




Les fichiers suivants sont automatiquement créés dans le répertoire donné :

- "&lt;Nom du fichier de données&gt;.FIC" (fichier de données).

- "&lt;Nom du fichier de données&gt;.MMO" (fichier mémo).

- "&lt;Nom du fichier de données&gt;.NDX" (fichier d'index).




Si le fichier de données est journalé, un sous-répertoire "JNL" sera automatiquement créé et contiendra les différents fichiers journalés.

Si le fichier de données contient un index full-text, un fichier "&lt;Nom du fichier&gt;.FTX" sera créé. 

**Remarques** :

- Cette option est identique à l'utilisation de la fonction [HCréationSiInexistant](../WDLang4/3044254.md).

- Si le fichier de données existe déjà dans le répertoire spécifié, cette option n'est pas prise en compte.

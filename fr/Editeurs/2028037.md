


## WDSetup : Gérer des langues supplémentaires
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000247"></a>
Le projet "WDSetup" correspond au programme d'installation standard des applications WINDEV.

Par défaut, ce projet gère le français et l'américain.

**Remarques** : 

- Si vous utilisez WDSetup personnalisé, il est nécessaire de reprendre les évolutions réalisées entre les différentes versions. Pour reporter les évolutions réalisées, utilisez la comparaison de fichiers : sous le volet "Projet", dans le groupe "Autres actions", déroulez "Comparer" puis sélectionnez "Comparer deux projets". Dans l'assistant, sélectionnez votre projet et le projet WDSetup livré avec WINDEV.

- Pour ajouter de nouveaux plans dans l'assistant ou de nouvelles fonctionnalités, il est possible d'utiliser un plug-in. Pour plus de détails, consultez [Personnaliser le programme d'installation (Plug-in)](../Editeurs/2028109.md). 

- Le projet WDSetup est livré avec deux configurations : une configuration 32 bits et une configuration 64 bits.

## Gérer des langues supplémentaires dans WDSetup
<a name="gerer_des_langues_supplementaires_dans_wdsetup_ELTTEXTE000271"></a>
Pour gérer des langues supplémentaires dans WDSetup, vous pouvez : 

- Personnaliser l'exemple WDSetup livré en standard avec le produit. 

- Créer un nouveau projet basé sur l'exemple WDSetup. Cette solution est conseillée car elle évite tout problème d'écrasement du projet lors d'une mise à jour. 




**Pour créer un nouveau projet basé sur WDSetup** : 

1. Depuis la fenêtre d'accueil de WINDEV, cliquez sur "Créer un projet". L'assistant de création de projet se lance. 

2. Donnez un nom à votre projet et indiquez la plateforme d'exécution. 

3. Dans l'étape "Description - Mode de création", choisissez l'option "Créer un projet à partir d'un exemple existant". Passez à l'étape suivante de l'assistant. 

4. Choisissez le projet prédéfini à utiliser : WDSetup. Passez à l'étape suivante. 

5. Indiquez si le projet doit être mis dans le GDS (Gestionnaire de Sources). 

6. Validez. Le projet est automatiquement créé. Il contient tous les éléments de l'exemple "WD Setup" et peut être modifié et personnalisé. 




**Pour ajouter les nouvelles langues au projet personnalisé de WDSetup** : 

1. Ajoutez les langues supplémentaires à gérer dans l'onglet "Langues" de la description du projet. 
	 Pour afficher la description du projet, sous le volet "Projet", dans le groupe "Projet", cliquez sur "Description".

2. Traduisez l'interface du projet (fenêtres, champs, ...) ainsi que les messages de code. 
	**Rappel** : WDMSG permet de faciliter la traduction d'un projet en extrayant l'ensemble des messages d'un projet (libellé des champs, titre des fenêtres, ...) et en réintégrant les messages après traduction. WDMSG est un module complémentaire de WINDEV. Pour plus d'informations sur WDMSG, contactez le service commercial de PC SOFT.

3. Créez un nouvel exécutable pour chaque configuration (configuration 32 bits et configuration 64 bits). 
	Rappel : Pour créer l'exécutable : 

	- activez la configuration voulue (32 bits ou 64 bits). 

	- déroulez ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_generer.gif)
 parmi les boutons d'accès rapide. 

	- sélectionnez "Générer l'exécutable 32 bits (.exe)" ou "Générer l'exécutable 64 bits (.exe)". 
			Dans l'assistant, pensez à intégrer les nouvelles langues dans l'exécutable.

## Tester WDSetup personnalisé
<a name="tester_wdsetup_personnalise_ELTTEXTE000295"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000096"></a>Pour tester un programme d'installation personnalisé (WDSetup personnalisé), il est nécessaire de :

1. Configurer le programme d'installation et générer ce programme d'installation.

2. Tester le projet WDSetup personnalisé sous WINDEV. Pour effectuer ce test, il faut :

	- copier des fichiers dans le répertoire EXE du programme d'installation personnalisé,

	- saisir une ligne de commande spécifique.






<a name="NOTE3_2"></a>


### Configurer et générer le programme d'installation
<a name="configurer_generer_programme_installation_ELTPARAGRAPHE000109"></a>Pour configurer le programme d'installation et générer ce programme d'installation : 

1. Configurez le programme d'installation grâce à [l'assistant de création du programme d'installation](../Editeurs/2028062.md) ou sous [l'éditeur d'installation WDInst](../Editeurs/2028042.md).

2. Si vous utilisez l'assistant de création du programme d'installation, à la fin de l'assistant, générez l'installation puis lancez WDInst.

3. Sous l'éditeur d'installation WDInst, sélectionnez l'option "Paramètres de l'installation .. Options de l'installeur".

4. Dans l'onglet "Framework WINDEV", sélectionnez si nécessaire l'option "Copier le Framework WINDEV avec l'installation". 
	Cliquez ensuite sur l'onglet "Personnalisation de l'installeur". 
	Sélectionnez l'option "Utiliser un installeur personnalisé" et sélectionnez l'exécutable du programme d'installation personnalisé. 
	Remarques : 

	- Selon le type d'installation à réaliser, vous pouvez sélectionner l'exécutable correspondant au programme d'installation 32 bits ou au programme d'installation 64 bits.

	- Il est possible d'associer les librairies (fichiers ".DLL") correspondantes à l'exécutable du programme d'installation personnalisé dans l'onglet "DLL de l'installeur" (cas de librairies renommées par exemple).




5. Validez la fenêtre des options de l'installeur. 

6. Sélectionnez l'option "Fichier .. Générer".

7. Pour le test, il est conseillé de sélectionner un répertoire unique pour le support du programme d'installation.

8. Spécifiez le répertoire de génération du programme d'installation.

9. Dans l'onglet "Avancé", cochez l'option "Conserver le fichier WXF pour tester un WDSetup personnalisé". Pour plus de détails, consultez le [fichier WXF](../Editeurs/2028055.md).

10. Validez la fenêtre pour générer le programme d'installation.



<a name="NOTE3_3"></a>


### Tester WDSetup personnalisé
<a name="tester_wdsetup_personnalise_ELTPARAGRAPHE000147"></a>Pour tester WDSetup personnalisé sous WINDEV :

1. Dans le répertoire "EXE" (sous-répertoire correspondant à la configuration de projet 32 ou 64 bits) du programme d'installation personnalisé (projet "WDSetup" personnalisé), copiez les fichiers suivants :

	- le fichier "&lt;NomApplication&gt;.WXF" (présent dans le répertoire du projet à installer),

	- les fichiers de licence, si une licence est affichée au début de l'installation de l'application,

	- les fichiers image utilisés pour l'interface du programme d'installation (image de fond, logo, ...),

	- les fichiers image utilisés pour les diapositives du programme d'installation (si nécessaire).




2. Sous WINDEV, ouvrez le programme d'installation personnalisé (projet "WDSetup" personnalisé).

3. Sous le volet "Projet", dans le groupe "Mode test", déroulez "Mode test" et sélectionnez "Paramétrage du mode test".

4. Saisissez la ligne de commande suivante : "/Rep=&lt;CheminRépertoireINSTALL&gt;", où &lt;CheminRépertoireINSTALL&gt; correspond au chemin du répertoire INSTALL créé dans le répertoire de génération du programme d'installation. 
	Par exemple, si le répertoire de génération du programme d'installation est : "C:\\Temp", le chemin du répertoire INSTALL sera : "C:\\Temp\\INSTALL".

5. Sélectionnez la configuration de projet correspondant à la version à tester (32 ou 64 bits) et lancez le test du projet (Ctrl + F9).




Attention : Lors de ce test, l'installation est réellement réalisée dans le répertoire spécifié.

## Utiliser WDSetup personnalisé comme programme d'installation
<a name="utiliser_wdsetup_personnalise_comme_programme_installation_ELTTEXTE000331"></a>
La sélection d'un programme d'installation personnalisé est effectuée dans l'éditeur d'installation WDInst.

**Pour utiliser WDSetup personnalisé comme programme d'installation** :

1. Sélectionnez l'option "Paramètres de l'installation .. Options de l'installateur".

2. Dans l'onglet "Framework WINDEV", sélectionnez si nécessaire l'option "Copier le Framework WINDEV avec l'installation". 
	Cliquez ensuite sur l'onglet "Personnalisation de l'installeur". 
	Sélectionnez l'option "Utiliser un installeur personnalisé" et sélectionnez l'exécutable du programme d'installation personnalisé. 
	Remarques : 

	- Selon le type d'installation à réaliser, vous pouvez sélectionner l'exécutable correspondant au programme d'installation 32 bits ou au programme d'installation 64 bits.

	- Il est possible d'associer les librairies (fichiers ".DLL") correspondantes à l'exécutable du programme d'installation personnalisé dans l'onglet "DLL de l'installeur" (cas de librairies renommées par exemple).




3. Validez la fenêtre des options de l'installeur.

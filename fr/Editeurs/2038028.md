


## Recherche dans l'historique du GDS
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000137"></a>
La recherche dans l'historique permet de réaliser une recherche avancée en tenant compte de différents critères dans l'historique des manipulations réalisées dans le GDS. Le résultat de la recherche peut être : 

- Affiché à l'écran, 

- Exporté dans différents formats : HFSQL, document XLSX, document DOCX, vers le presse-papiers, ...




Ce type de recherche peut également être effectué via une ligne de commande. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment réaliser une recherche via l'administrateur du GDS ?
<a name="comment_realiser_une_recherche_via_administrateur_gds_ELTTEXTE000161"></a>
Pour effectuer une recherche dans l'historique : 

1. Lancez l'administrateur du GDS. 

2. Sous le volet "Outils", dans le le groupe "Recherche", cliquez sur "Dans l'historique". 

3. La fenêtre permettant d'effectuer une recherche multi-critère dans l'historique est affichée. 




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Effectuer une recherche via une ligne de commande
<a name="effectuer_une_recherche_via_une_ligne_commande_ELTTEXTE000185"></a>
Il est possible d'effectuer une recherche dans l'historique du GDS via une ligne de commande, en utilisant la syntaxe suivante : 

```txt
WDGDSAdmin.exe /EXPORT 
		/BASE=<Adresse de la base de sources> 
		/LOGIN=<Login> 
		/PASS=<Mot de passe> 
		/DIR=<Nom du répertoire distant> 
		/EXPORTFILE=<Nom complet du fichier d'export>
```
Dans ce code, le paramètre &lt;Adresse de la base de sources&gt; est de la forme :

- Cas HFSQL Client/Serveur : 
	
	```txt
	"CS:\\<Adresse IP du serveur:<Port serveur>\<Nom base de sources GDS>"
	```


- Cas HFSQL Classic : le chemin local de la base doit être indiqué. Par exemple : 
	
	```txt
	"D:\PC SOFT\Base de sources GDS locale\"
	```





Cette ligne de commande accepte des options supplémentaires de filtrage :

- /COMMENTONLY=1 : Permet d'obtenir uniquement les commentaires. 

- /SINCELASTINSTALL=1 : Permet d'obtenir uniquement les modifications effectuées depuis la dernière génération d'une installation. 

- /DATEMIN=AAAMMJJHHMMSS : Permet de définir la date minimale de recherche (incluse dans la recherche). Par exemple : 2017010100000.

- /DATEMAX=AAAMMJJHHMMSS : Permet de définir la date maximale de recherche (incluse dans la recherche). Par exemple : 2018010100000.

- /BUGID=999 : Permet d'obtenir uniquement les modifications liées à l'incident 999. 

- /TASKID=999	: Permet d'obtenir uniquement les modifications liées à la tâche 999. 

- /AUTHOR=Login : Permet d'obtenir uniquement les modifications du développeur &lt;Login&gt;. 

- /FILENAME=xxx : Permet de faire une recherche uniquement sur le fichier &lt;xxx&gt;. Ce paramètre dépend du paramètre "/FILENAMEFILTER". 

- /FILENAMEFILTER=0/1/2/3 : Permet de filtrer le nom du fichier de recherche : 

	- 0 = Commence par xxx, 

	- 1 = Termine par xxx, 

	- 2 = Contient xxx, 

	- 3 = Egal à xxx


 où xxx a été spécifié dans le paramètre /FILENAME

- /LABEL=yyy : Permet de faire une recherche uniquement sur les éléments avec l'étiquette &lt;yyy&gt;. Ce paramètre dépend du paramètre "/LABELFILTER". 

- /LABELFILTER=0/1/2/3 : Permet de filtrer le nom de l'étiquette de recherche : 

	- 0 = Commence par yyy, 

	- 1 = Termine par yyy, 

	- 2 = Contient yyy, 

	- 3 = Egal à yyy, 

	- 4 = tous les éléments avec une étiquette, 


 où yyy a été spécifié dans le paramètre "/LABEL".

- /COMMENT=zzz : Permet d'obtenir uniquement les éléments ayant pour commentaire &lt;zzz&gt;. Ce paramètre dépend du paramètre "/LABELFILTER".

- /COMMENTFILTER=0/1/2/3 : Permet de filtrer le contenu du commentaire : 

	- 0 = Commence par zzz, 

	- 1 = Termine par zzz, 

	- 2 = Contient zzz, 

	- 3 = Egal à zzz,


 où zzz a été spécifié dans le paramètre "/COMMENT".

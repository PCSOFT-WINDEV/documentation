
## Configurer la réplication universelle assistée
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000152"></a>
Si votre application utilise la réplication universelle assistée par l'intermédiaire d'un fichier .WER, l'assistant de création d'installation propose automatiquement de paramétrer l'installation pour installer les différents outils nécessaires à la réplication universelle assistée.

Les différentes étapes de ce paramétrage sont les suivantes :

- Localisation du fichier .WER à manipuler.

- Méthode de réplication.

- Définition du mode connecté.






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Paramètres de la réplication universelle assistée
<a name="parametres_replication_universelle_assistee_ELTTEXTE000176"></a>


### Fichier .WER
<a name="fichier_wer_ELTPARAGRAPHE000025"></a>La première étape du paramétrage de la réplication universelle assistée consiste à donner le chemin du fichier .WER à utiliser.

Depuis l'assistant de création d'installation, il est possible de configurer et de modifier le contenu du fichier .WER à l'aide de l'outil [ReplicEdit](../ReplicEdit/3526500.md).


<a name="NOTE2_2"></a>


### Méthode de réplication
<a name="methode_replication_ELTPARAGRAPHE000038"></a>Lors de la préparation du programme d'installation, il est possible de spécifier la méthode de réplication qui sera utilisée :

- **Réplication à l'aide d'un serveur de réplication**. Cette méthode est conseillée par exemple lorsque la réplication sera réalisée grâce à une connexion Internet. Si cette option est sélectionnée, l'étape suivante de l'assistant permet de configurer les paramètres du serveur de réplication : adresse du serveur, nom de l'utilisateur et mot de passe. Il est également possible :

	- de publier les paramètres de réplication sur le serveur (installation avec mise à jour automatique uniquement) : si cette option est cochée, la réplication sera directement accessible par les applications.

	- d'autoriser la modification de ces paramètres pendant l'installation : la personne utilisant l'application pourra modifier les paramètres du serveur de réplication.

	- d'utiliser ces valeurs même si elles ont été personnalisées lors d'une précédente installation.

	- d'installer un serveur de réplication. Dans ce cas, l'installation de l'application proposera à l'utilisateur d'installer le serveur de réplication sur le poste de son choix.
			**Attention** : pour réaliser cette installation, le programme d'installation doit être livré avec le pack d'installation du serveur de réplication. Le bouton "Avancé" permet de sélectionner le pack nécessaire. Il est également possible de télécharger le dernier pack d'installation disponible depuis le site PC SOFT.




- **Réplication avec accès direct avec la base maître**. Cette méthode est conseillée par exemple lors de l'utilisation d'une application sur un portable, avec une connexion ponctuelle au réseau locale pouvant être effectuée (par exemple, commercial présent au siège de la société tous les Lundi). Si cette option est sélectionnée, il est nécessaire d'indiquer les paramètres de la base maître :

	- Utiliser les connexions définies dans l'analyse : Les paramètres des connexions définies dans l'analyse seront utilisés pour accéder à la base maître.

	- Utiliser les connexions définies dans le fichier .wer : Les paramètres des connexions définies dans le fichier .WER seront utilisés pour accéder à la base maître.

	- Définir les paramètres des connexions maintenant : Les paramètres des connexions sont définies immédiatement dans le plan suivant de l'assistant.

	- Demander les paramètres des connexions pendant l'installation : Les paramètres des connexions sont définies par la personne qui installe l'application.




- **Choix de la méthode de réplication lors de l'installation**. Si cette méthode est sélectionnée, les paramètres de la base maître doivent alors être spécifiés (idem option "Réplication avec accès direct avec la base maître").





<a name="NOTE2_3"></a>


### Définition du mode connecté
<a name="definition_mode_connecte_ELTPARAGRAPHE000062"></a>Lors de la préparation du programme d'installation, il est possible d'indiquer le mode de fonctionnement de l'application en mode connecté.

En effet, lorsque l'utilisateur a accès à la base maître, il est possible d'utiliser automatiquement l'accès direct à la base maître plutôt que d'utiliser la réplication (option "Permettre à l'application de travailler également en mode connecté"). Si ce mode de travail est sélectionné, il est possible de demander la synchronisation automatique des données au lancement et à l'arrêt de l'application. Ce mode permet de travailler en mode nomade avec la base de données la plus à jour possible.

Ces options peuvent être ou non modifiées pendant l'installation de l'application. Il est également possible de forcer l'utilisation de ces options même si elles ont déjà été personnalisées.

## Fichiers installés pour la réplication
<a name="fichiers_installes_pour_replication_ELTTEXTE000212"></a>
Par défaut, les fichiers suivants sont installés sur le poste de l'utilisateur :

|   | Répertoire d'installation par défaut |
| --- | --- |
| Fichiers de données locaux (.fic, .ndx, .mmo) | Documents and settings\\All Users\\Application Data\\&lt;Nom de l'application&gt; |
| Fichiers permettant de réaliser la réplication | Documents and settings\\All Users\\Application Data\\PC SOFT\\RPL\\&lt;Nom de l'application&gt;\\&lt;Nom de l'application&gt; |
| Paramètres de l'installation | Documents and settings\\&lt;Nom user&gt;\\Application Data\\WDSETUP\\&lt;Nom de l'application&gt; |
| Application (Exe et WX) | Program File\\&lt;Nom de l'application&gt; |
| ReplicSynchro et ReplicAdmin | Program Files\\Fichiers communs\\PC SOFT\\28.0\\ReplicSynchro |





**Remarque** : ReplicSynchro peut être lancé directement par l'utilisateur final grâce à l'option "Démarrer .. Outils PC SOFT .. ReplicSynchro".




## WDInst : choisir le type d'installation à générer 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000172"></a>
L'éditeur d'installation WDInst permet de créer plusieurs types d'installation :

- ***Installation standard*** : conseillée pour une application monoposte.
	Tous les fichiers nécessaires à l'application sont installés sur chacun des postes utilisateurs.
	En cas de mise à jour de l'application, le programme de mise à jour doit être exécuté sur chaque poste utilisateur.

- ***Installation réseau avec mise à jour automatique*** : conseillée pour une application multi-postes (utilisée par plusieurs utilisateurs simultanément). 
	Ce type d'installation permet de :

	- gérer automatiquement les mises à jour de l'application.

	- connaître les caractéristiques des utilisateurs connectés à l'application à un moment donné, grâce au programme "WDADMINEXE" installé sur le serveur. Pour plus de détails, consultez le [Contrôle à distance des exécutables WINDEV](../WDAdminExe/2028052.md).

	- paramétrer l'emplacement des fichiers de l'application. Quelques exemples :

		- Installation de tous les fichiers de l'application sur le serveur.

		- Installation de la base de données sur le serveur et des fichiers nécessaires à l'exécution de l'application (".EXE", ".WDL", ".DLL", ...) sur chaque poste utilisateur.

		- Installation d'une partie de la base de données sur le serveur et d'une autre partie sur chaque poste utilisateur. Les fichiers nécessaires à l'exécution de l'application (".EXE", ".WDL", ".DLL", ...) sont également installés sur chaque poste utilisateur.




- ***Installation depuis un Store Privé*** : conseillée pour regrouper les applications utilisées au sein d'une entreprise. Ce type d'installation permet :

	- L'accès à l'installation des applications grâce à un portail. 

	- La mise à jour automatique des applications.   




- ***Installation compacte*** : La taille du programme d'installation généré est optimisé pour être téléchargé par Internet.




**Remarque** : Par défaut, l'installation générée par WINDEV correspond à une installation standard.

## Comment le faire ?
<a name="comment_faire_ELTTEXTE000196"></a>


### Définition du type d'installation à générer
<a name="definition_type_installation_generer_ELTPARAGRAPHE000047"></a>La définition du type d'installation à générer est spécifiée sous l'éditeur d'installation WDInst. Pour lancer WDInst, sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDInst".

L'assistant de création du programme d'installation permet également de définir le type d'installation à générer.

Tableau récapitulatif : [Comment installer une application réalisée avec WINDEV ?](../Editeurs/2028005.md)
<a name="NOTE2_2"></a>


### Générer une installation classique
<a name="generer_une_installation_classique_ELTPARAGRAPHE000072"></a>Pour générer une installation classique :

1. Sélectionnez l'option "Paramètres de l'installation .. Assistant".

2. Cochez l'option "Installation standard".



<a name="NOTE2_3"></a>


### Générer une installation avec mise à jour
<a name="generer_une_installation_avec_mise_jour_ELTPARAGRAPHE000082"></a>Pour générer une installation avec mise à jour :

1. Sélectionnez l'option "Paramètres de l'installation .. Assistant".

2. Cochez l'option "Installation réseau avec mise à jour automatique".

3. Sélectionnez l'exécutable de référence pour la mise à jour réseau.

A chaque lancement de l'application sur le poste utilisateur, la version de l'exécutable de l'application et de l'exécutable de référence (présent sur le poste serveur) sont comparés. Si les versions sont différentes, la mise à jour de l'application est proposée. La gestion des versions est détaillée dans le point suivant. |





**4.** Le numéro de version de l'application à installer est automatiquement récupéré par WDInst. Ce numéro a été spécifié lors de la création de l'exécutable de l'application. 
Saisissez le numéro de version de l'application présente sur les postes utilisateurs à partir duquel la mise à jour ne sera pas obligatoire.


|   |   |
| --- | --- |
| ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=picto_rubrique_exemple.GIF) | **Mise à jour obligatoire / Mise à jour optionnelle**<br>L'installation de la mise à jour :<br><br>- ***sera obligatoire*** si la version de l'application installée sur les postes utilisateurs est inférieure à la dernière version compatible.<br><br>- ***ne sera pas obligatoire*** si la version de l'application installée sur les postes utilisateurs est égale ou supérieure à la dernière version compatible.<br><br><br>**Attention** : Dans le cas d'une mise à jour réseau avec modification automatique des fichiers de données, l'installation de la mise à jour doit être obligatoire. |

**5.** Pour configurer plus précisément la mise à jour de votre application, cliquez sur le bouton "Options avancées". Il sera ainsi possible de :

- spécifier l'historique des modifications.

- paramétrer les options de "Contrôle à distance des exécutables WINDEV".

- modifier les conditions d'application de la mise à jour.




Pour plus de détails, consultez les [Options avancées de l'installation réseau](../Editeurs/2028020.md).

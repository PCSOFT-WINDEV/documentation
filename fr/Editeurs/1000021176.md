


## Groupware utilisateur et fichiers accédés via un Connecteur Natif
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000104"></a>
Une application utilisant le groupware utilisateur peut tout à fait être utilisée avec une base de données manipulée par un Connecteur Natif (également appelé 'Accès Natif').

Pour qu'une application utilisant le groupware utilisateur fonctionne avec un Connecteur Natif, il est nécessaire de configurer correctement le groupware utilisateur afin d'utiliser les fichiers de données présents sur le serveur. Ces manipulations sont présentées dans cette page d'aide.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000128"></a>
Pour utiliser les fichiers du groupware utilisateur avec un Connecteur Natif : 

1. Sous le volet "Projet", dans le groupe "Projet", cliquez sur "Groupware Utilisateur". 

2. Dans la fenêtre de description de la configuration de projet en cours, dans la zone "Groupware Utilisateur : Inactif", cliquez sur le bouton "Modifier". 

3. Dans la fenêtre qui s'affiche, activez le groupware utilisateur.

4. Dans l'onglet "Général", sélectionnez : 

	- Groupware utilisateur classique. 

	- Lancement manuel par programmation. 
			Le groupware utilisateur ne doit pas être lancé en mode "Lancement automatique" dans votre application.




5. Sélectionnez les différentes options du groupware utilisateur puis validez la fenêtre. 
	Pour plus de détails sur les différentes options disponibles, consultez [Groupware utilisateur : Options et paramétrage](../Editeurs/1000021153.md)

6. Dans le code d'initialisation du projet, utilisez la fonction [gpwOuvreConnexion](../WDLang6/3041007.md) pour décrire la connexion à la base de données manipulée via un Connecteur Natif.  
	**Attention** : les mots de passe des fichiers de données du groupware utilisateur ne sont pas pris en compte.

7. Utilisez la fonction [gpwOuvre](../WDLang6/3041002.md) sans indiquer le chemin des fichiers de données. La notion de répertoire n'existe pas dans ce cas.

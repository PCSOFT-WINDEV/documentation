
## Afficher ou non les événements ou traitements vides d'un élément
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000100"></a>
La saisie du code WLangage s'effectue dans l'éditeur de code qui propose les différents événements possibles sur le champ : Entrée, Modification, Sortie, Clic, ...

Une série d'événements standard est systématiquement proposée pour chaque élément : fenêtre, champ, état, ...

A partir de la version 24, il est possible de masquer les événements vides associés à un élément. 

Selon vos habitudes de programmation, il est ainsi possible d'obtenir un éditeur de code épuré. 

Un exemple : les événements non utilisés ont été masqués. La présentation est plus claire.<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Traitements_vides%20-%20HC%20N%B0001.gif&type=thumb)


Remarque : Il existe bien sûr également les [événements optionnels](../WDChamp/1014004.md) qu'il est possible d'activer.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000124"></a>


### Afficher ou non les événements vides
<a name="afficher_non_les_evenements_vides_ELTPARAGRAPHE000031"></a>Pour ne pas afficher les événements vides associé à un élément WINDEV, WEBDEV ou WINDEV Mobile :

1. Affichez un des codes de l'élément voulu (par exemple via la touche F2).

2. Sous le volet "Affichage", dans le groupe "Aide à l'édition", cochez ou décochez l'option "Afficher les traitements vides".<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Traitements_vides%20-%20HC%20N%B0002%201.gif&type=thumb)





Remarque : Cette fonctionnalité permet également de gérer l'affichage des procédures et des propriétés vides.

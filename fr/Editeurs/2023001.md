## Présentation
<a name="presentation_ELTTEXTE000096"></a>
L'abonnement à un élément du dictionnaire est automatiquement proposé :

- Lors de l'intégration d'un élément dans le dictionnaire.

- Lors de la copie d'un élément du dictionnaire dans un projet.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Opérations liées à l'abonnement
<a name="operations_liees_abonnement_ELTTEXTE000120"></a>
Si l'objet abonné n'est pas en phase avec le dictionnaire, un écran de mise à jour de l'objet abonné est proposé à l'ouverture ou la sauvegarde de cet objet.

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Abonnement.gif)


Cet écran permet de :

- Ouvrir l'élément sans mise à jour de l'abonnement (bouton de validation).

- Annuler l'ouverture de l'élément (bouton d'annulation).

- Mettre à jour tous les éléments les plus anciens (bouton "Tout mettre à jour").

- Mettre à jour l'élément du dictionnaire à partir de l'élément abonné sélectionné dans la liste (bouton "Vers le dico").

- Mettre à jour l'élément abonné sélectionné à partir de l'élément du dictionnaire (bouton "Depuis le dico").

- Supprimer l'abonnement (bouton "Supprimer le lien").

- Voir le détail des différences (bouton "Détail").




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Cas particulier : le conflit
<a name="cas_particulier_conflit_ELTTEXTE000144"></a>
Un élément du dictionnaire peut être en conflit avec l'élément abonné : l'élément du dictionnaire et l'élément abonné ont été modifiés différemment et ne sont plus en phase. Le développeur peut :

- Intégrer les évolutions de l'élément du dictionnaire : les dernières évolutions de l'élément abonné seront perdues.

- Mettre à jour l'élément du dictionnaire : ses dernières évolutions seront perdues.


## Paramétrer la réplication universelle assistée
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000138"></a>
Si votre application utilise la réplication universelle assistée par l'intermédiaire d'un fichier .WER, WDInst propose de paramétrer l'installation pour installer les différents outils nécessaires à la réplication universelle assistée.

La configuration de la réplication universelle assistée est effectuée dans l'option "Paramètres de l'installation .. Paramétrage de la réplication universelle assistée".

Les différentes étapes de ce paramétrage sont les suivantes :

- [Mode de fonctionnement de la réplication](#NOTE2_1).

- [Emplacement de la base maître](#NOTE3_1).

- [Emplacement de la base maître en mode connecté](#NOTE4_1).

- [Emplacement de la base locale](#NOTE5_1).






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Mode de fonctionnement de la réplication
<a name="mode_fonctionnement_replication_ELTTEXTE000162"></a>
Une application WINDEV peut :

- Ne pas utiliser la réplication.

- Utiliser la réplication et fonctionner en mode connecté ou répliqué.

- Utiliser la réplication et fonctionner toujours en mode répliqué.




Ces informations peuvent ou non être modifiées lors de l'installation.



<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Emplacement de la base maître
<a name="emplacement_base_maitre_ELTTEXTE000186"></a>
La base maître peut être :

- décrite dans un fichier .WER. Il suffit de sélectionner le fichier .WER correspondant par le bouton "Fichier .WER". Le fichier .WER pourra être édité directement.

- accessible par un serveur de réplication. Il est nécessaire de saisir les informations concernant le serveur de réplication par le bouton "Serveur". Ces informations concernent l'adresse du serveur, nom de l'utilisateur et mot de passe. Il est également possible de :

	- "Publier les paramètres de la réplication sur le serveur". Cette option permet de rendre la réplication utilisable immédiatement.

	- "Autoriser la modification de ces paramètres pendant l'installation" : la personne utilisant l'application pourra modifier les paramètres du serveur de réplication.

	- "Utiliser ces valeurs même si elles ont déjà été personnalisées lors d'une précédente installation".




- configurée uniquement lors de l'installation. Les différents paramètres seront alors demandés à l'utilisateur installant l'application.






<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Emplacement de la base maître en mode connecté
<a name="emplacement_base_maitre_mode_connecte_ELTTEXTE000210"></a>
Si l'application offre la possibilité d'utiliser le mode connecté, il est nécessaire de définir la base maître à utiliser pour le mode connecté. Le bouton "Base maître" permet de définir les caractéristiques des connexions utilisées par l'application.

Si l'application utilise une base de données HFSQL Classic, une connexion est automatiquement créée.

Il est possible de : 

- Utiliser les connexions définies dans l'analyse : Les paramètres des connexions définies dans l'analyse seront utilisés pour accéder à la base maître.

- Utiliser les connexions définies dans le fichier .wer : Les paramètres des connexions définies dans le fichier .WER seront utilisés pour accéder à la base maître.

- Définir les paramètres des connexions maintenant : Les paramètres des connexions sont définis immédiatement dans le plan suivant de l'assistant.

- Demander les paramètres des connexions pendant l'installation : Les paramètres des connexions sont définies par la personne qui installe l'application.






<a name="NOTE5"></a>
<a name="NOTE5_1"></a>


## Emplacement de la base locale
<a name="emplacement_base_locale_ELTTEXTE000234"></a>
Par défaut, la base locale de l'application est créée dans le répertoire "Documents and settings\\All Users\\Application Data\\&lt;Nom de l'application&gt;". Le bouton "Base locale" permet de modifier ce répertoire.

Remarque : Ce répertoire est étudié pour fonctionner avec tous les types de systèmes (y compris sous Windows Vista ou supérieur). Il est déconseillé de modifier ce chemin.

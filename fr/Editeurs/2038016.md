
## GDS : Gestion des branches
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000098"></a>
La gestion des branches permet de gérer en parallèle plusieurs versions (exploitées ou pas) d'une même application.

**Le cas habituel est celui-ci** : Vous avez diffusé une version de votre application et vous travaillez sur la future version applicative de cette application. Mais vous souhaitez dans le même temps, reporter dans la version en clientèle les corrections de bug que vous effectuez dans la nouvelle version, et ainsi créer et diffuser des versions correctives intermédiaires.

**Autre exemple** : Les branches peuvent également servir à gérer des versions spécifiques par client. Chaque branche correspond à une version client. Dès que le projet de base (tronc commun) est modifié, il suffit de reporter ces modifications dans chacune des branches.

Les branches permettent ainsi de diffuser des versions correctives de l'application en clientèle sans effort : vous ne coderez pas deux fois les mêmes choses.

Le risque d'erreurs dû au "report manuel" disparaît également.

Le confort augmente, la sécurité augmente, le frein à la mise à disposition de correctifs disparaît ...

**Remarque** : Les reports de branches sont bidirectionnels. Il est possible de reporter des modifications du projet principal vers la branche ou de la branche vers le projet principal. Le projet modifié est toujours celui qui est ouvert.

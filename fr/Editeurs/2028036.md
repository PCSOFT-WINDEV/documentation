
## Langues gérées par l'installation 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000121"></a>
Si vos applications doivent être installées dans différents pays, l'assistant d'installation propose de générer une installation multilingue.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment proposer à l'utilisateur de lancer l'installation dans plusieurs langues ?
<a name="comment_proposer_utilisateur_lancer_installation_dans_plusieurs_langues_ELTTEXTE000145"></a>
Il suffit de sélectionner les différentes langues qui seront proposées à l'utilisateur lors du lancement du programme d'installation.

Le programme d'installation s'exécutera dans la langue sélectionnée par l'utilisateur.

Seuls le français, l'anglais et l'américain sont gérés automatiquement par WINDEV. Pour que le programme d'installation gère des langues supplémentaires, il est nécessaire de personnaliser le projet "WDSetup" avant de créer le programme d'installation. Ce projet (fourni avec les exemples de WINDEV) correspond au programme d'installation standard des applications WINDEV.

Pour plus de détails, consultez [WDSetup : gérer des langues supplémentaires](../Editeurs/2028037.md).

**Pour toutes les langues sélectionnées, il sera possible de spécifier** :

- une licence, si une licence est affichée au début de l'installation de l'application.

- un fichier "Lisez-Moi", si un fichier "Lisez-Moi" est affiché à la fin de l'installation de l'application.

- une interface du programme d'installation spécifique à chaque langue.




**Application installée en réseau**
Lors de la mise à jour d'une application installée en réseau, une fenêtre s'affiche au lancement de l'application depuis les postes utilisateurs. Cette fenêtre informe les utilisateurs qu'une mise à jour de l'application est disponible. Cette fenêtre permet selon le cas d'installer ou non la mise à jour. Cette fenêtre est également présente dans le projet WDSetup.

## Lancement de l'application multilingue
<a name="lancement_application_multilingue_ELTTEXTE000193"></a>
Par défaut, une application multilingue est exécutée dans la langue principale du projet (option "Langue principale" dans l'onglet "Langues" de la description du projet).

Pour lancer l'application installée dans la langue d'exécution du programme d'installation :

1. Récupérez la langue d'exécution du programme d'installation et enregistrez cette langue dans un fichier ".INI" ou dans la base de registre du poste d'installation. Cette opération peut être réalisée automatiquement lors de l'installation. Pour plus de détails, consultez [Gestion de la base de registre et des fichiers ".INI" lors de l'installation](../Editeurs/2028047.md).

2. Utilisez la fonction [Nation](../WDLang1/3054014.md) pour modifier la langue d'exécution de l'application.




Pour plus de détails sur les applications multilingues, consultez [Développer des applications multilingues](../WDLang1/3054008.md).

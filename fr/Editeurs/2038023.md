


## Administrateur du GDS : outils disponibles
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000529"></a>
L'administrateur de GDS (Gestionnaire de sources) permet de : 

- configurer les [options de l'administrateur et du GDS](#NOTE2_1). 

- [administrer et optimiser la base de sources du GDS](#NOTE3_1). 




L'utilisation de ces outils est disponible uniquement pour un utilisateur de type administrateur. 

Remarque : L'administrateur du GDS permet également de gérer les éléments de la base de sources, etc. Pour plus de détails, consultez [Présentation de l'administrateur du GDS](../Editeurs/2038008.md).

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Options du GDS
<a name="options_gds_ELTTEXTE000553"></a>
Pour configurer les options du GDS, sous le volet "Outils", dans le groupe "Option", cliquez sur "Options".
<a name="NOTE2_2"></a>


### Options générales
<a name="options_generales_ELTPARAGRAPHE000050"></a>L'onglet "Général" des options du GDS permet de configurer l'affichage des boîtes de dialogue lors d'une action réalisée par le GDS. Une boîte de dialogue peut être affichée pour :

- La réintégration sans modification.

- L'ajout de fichiers dans la base de sources.

- La mise à jour des dépendances d'un élément.

- L'extraction demandée explicitement (par la barre d'outils par exemple).

- Signaler l'ajout d'un projet avec succès.




Lors du survol des différentes options, l'image de la boîte de dialogue correspondante est affichée.
<a name="NOTE2_3"></a>


### Options concernant la messagerie
<a name="options_concernant_messagerie_ELTPARAGRAPHE000065"></a>L'onglet "Messagerie" des options du GDS permet d'indiquer si la messagerie intégrée doit être utilisée. Dans ce cas, des messages pourront être envoyés aux autres utilisateurs (demande de déblocage, informations de réintégration, ...).

Si nécessaire, il est également possible de demander à envoyer un message aux utilisateurs du GDS lors de chaque réintégration d'éléments dans le GDS.
<a name="NOTE2_4"></a>


### Options concernant les branches
<a name="options_concernant_les_branches_ELTPARAGRAPHE000074"></a>L'onglet "Branches" permet de paramétrer les chemins créés par défaut lors de l'utilisation des branches.

Cet onglet permet d'indiquer les répertoires distants et locaux utilisés par défaut lors de la création d'une branche. Pour plus de détails sur la gestion des branches, consultez [Gestion des branches](../Editeurs/2038016.md).
<a name="NOTE2_5"></a>


### Options avancées
<a name="options_avancees_ELTPARAGRAPHE000086"></a>L'onglet "Avancé" permet de configurer les performances de la base de sources dans le cas d'une base de sources au format HFSQL Client/Serveur utilisée par ADSL.

L'option "Limiter la taille des paquets pour l'ADSL" permet dans certains cas de gérer les spécificités de certains routeurs.

**Attention** : Si cette option est mal utilisée, elle peut ralentir le GDS.

## Administration du GDS
<a name="administration_gds_ELTTEXTE000607"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000101"></a>Les outils d'administration sont disponibles uniquement lors de la connexion à la base de sources du GDS en tant que Administrateur. Pour se connecter à la base de sources du GDS en tant qu'administrateur, sous le volet "Accueil", dans le groupe "Base de sources", cliquez sur "Connexion ". 

Dans la fenêtre qui s'ouvre :

- Indiquez les caractéristiques de la base de sources utilisée.

- Spécifiez le nom et le mot de passe de l'administrateur de la base de sources :

	- Si vous utilisez une base de sources GDS au format HFSQL Classic, par défaut l'administrateur est "Admin" (sans mot de passe).

	- Si vous utilisez une base de sources GDS au format HFSQL Client/Serveur, par défaut l'administrateur correspond à l'utilisateur de type "Administrateur".







**Remarque** : La connexion à la base de sources du GDS en tant qu'administrateur n'est pas la même chose que "Devenir maître sur le projet". Pour réaliser des manipulations d'administration, il est nécessaire de se connecter en tant qu'administrateur.


<a name="NOTE3_2"></a>


### Paramétrage de l'authentification
<a name="parametrage_authentification_ELTPARAGRAPHE000132"></a>Le paramétrage de l'authentification permet d'utiliser la base de données des Centres de Contrôle pour authentifier les utilisateurs du GDS. Le login et le mot de passe seront ceux spécifiés dans le Centre de Suivi de projets.

Cette option est disponible uniquement si vous travaillez avec une base de sources du GDS au format HFSQL Classic.

Pour activer l'authentification :

1. Sous le volet "Administration", dans le groupe "Sécurité", cliquez sur "Paramétrer l'authentification".

2. Cochez l'option "Activer l'authentification".

3. Indiquez la base de données des Centres de Contrôle utilisée (au format HFSQL Classic).

4. Validez. 



<a name="NOTE3_4"></a>


### Annuler les extractions
<a name="annuler_les_extractions_ELTPARAGRAPHE000160"></a>L'administrateur a la possibilité d'annuler toutes les extractions en cours sur le projet sélectionné : sous le volet "Administration", dans le groupe "Extractions", cliquez sur "Annuler des extractions".

Une fenêtre s'affiche listant tous les éléments extraits. Il est possible également de lister les éléments extraits pour tests ou les éléments extraits dans un répertoire distant différent.

Il suffit de sélectionner les éléments voulus et de cliquer sur le bouton "Annuler l'extraction". L'extraction est annulée. L'élément est présent dans la base de sources dans son état avec l'extraction.

Il est possible de prévenir les utilisateurs (option "Envoyer un message à ceux dont l'extraction a été annulée").
<a name="NOTE3_5"></a>


### Sauvegarde de la base de sources
<a name="sauvegarde_base_sources_ELTPARAGRAPHE000185"></a>L'administrateur a la possibilité de réaliser une sauvegarde de la base de sources du GDS : sous le volet "Administration", dans le groupe "Sauvegardes", cliquez sur "Sauvegarde complète". Cette sauvegarde est effectuée dans un fichier au format .ZIP. 
**Remarque** : Pendant la sauvegarde, la base de sources du GDS est accessible uniquement en lecture seule.

La sauvegarde de la base de sources peut également être réalisée en ligne de commande grâce à la notation suivante :

```txt
WDGDSAdmin.exe /SAVE 
		/BASE=<Adresse de la base de sources> 
		/LOGIN=<Login> 
		/PASS=<Mot de passe> 
		/ZIP=<Nom complet du fichier zip>
```
Dans ce code, le paramètre &lt;Adresse de la base de sources&gt; est de la forme : 

- Cas HFSQL Client/Serveur : "CS:\\\\&lt;Adresse IP du serveur&gt;:&lt;Port serveur&gt;\\&lt;Nom base de sources GDS&gt;"

- Cas HFSQL Classic : le chemin local de la base doit être indiqué. Par exemple : "D:\\PC SOFT\\Base de sources GDS locale\\"




Exemple : 

```wl
LanceAppli("""C:\WinDev\Programmes\WDGDSAdmin.exe"" /SAVE /BASE=CS:\\DOC2:4900\GDS" + ...
		" /LOGIN=admin /PASS=admin /ZIP=C:\Temp\test.zip")
```

<a name="NOTE3_6"></a>


### Sauvegarde et purge de l'historique
<a name="sauvegarde_purge_historique_ELTPARAGRAPHE000216"></a>L'administrateur a la possibilité de supprimer une partie de l'historique de la base de sources du GDS : sous le volet "Administration", dans le groupe "Sauvegardes", cliquez sur "Purge de l'historique avec sauvegarde". La partie supprimée est automatiquement sauvegardée dans un fichier au format ZIP. Cette sauvegarde pourra être ensuite restaurée.

Pour effectuer la sauvegarde et la purge de l'historique, il suffit d'indiquer la date à partir de laquelle l'historique doit être conservé.
<a name="NOTE3_7"></a>


### Restauration d'une sauvegarde complète
<a name="restauration_une_sauvegarde_complete_ELTPARAGRAPHE000237"></a>Pour restaurer une sauvegarde complète de la base de sources précédemment réalisée, sous le volet "Administration", dans le groupe "Sauvegardes", cliquez sur "Restauration complète".
<a name="NOTE3_8"></a>


### Restauration d'une sauvegarde d'historique
<a name="restauration_une_sauvegarde_historique_ELTPARAGRAPHE000256"></a>Pour restaurer une sauvegarde d'historique de la base de sources précédemment réalisée, sous le volet "Administration", dans le groupe "Sauvegardes", cliquez sur "Restauration d'historique".

## Maintenance du GDS
<a name="maintenance_gds_ELTTEXTE000673"></a>
L'administrateur du GDS propose différentes options de maintenance. Ces options permettent à tout moment d'optimiser et de gérer la base de sources du GDS.
<a name="NOTE4_2"></a>


### Migrer la base de sources dans le format le plus récent
<a name="migrer_base_sources_dans_format_plus_recent_ELTPARAGRAPHE000282"></a>Pour migrer la base de sources du GDS dans un format plus récent, sous le volet "Outils", dans le groupe "Maintenance", cliquez sur "Migrer au format le plus récent ". Ce format permet d'optimiser les accès à la base de données.

**Remarque** : Pour utiliser cette option, la base de sources du GDS ne doit pas être installée directement sur un nom de partage.

Par exemple, si les fichiers sont installés sur "\\\\PosteServeur\\GDS" où "GDS" est le nom du répertoire partagé, l'option ne fonctionnera pas. Il est nécessaire que les fichiers soient installés dans un répertoire de type "\\\\PosteServeur\\GDS\\BaseGDS"

Si ce n'est pas le cas, vous pouvez déplacer manuellement les fichiers de données (tous les utilisateurs doivent être déconnectés). A la prochaine ouverture des projets, il suffira de préciser le nouveau répertoire de la base de sources du GDS.

Attention : N'hésitez pas à consulter la page d'aide [Evolutions du format des fichiers du GDS](../Editeurs/2038018.md). 
<a name="NOTE4_3"></a>


### Passer la base de sources en Client/Serveur
<a name="passer_base_sources_clientserveur_ELTPARAGRAPHE000312"></a>Pour lancer l'assistant de passage de la base de sources en mode HFSQL Client/Serveur, sous le volet "Outils", dans le groupe "Maintenance", cliquez sur "Passer en Client/Serveur".<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=GDS_Client_serveur.gif)
Cette opération peut être effectuée uniquement si un moteur HFSQL Client/Serveur est installé sur le poste Serveur où la base de sources du GDS doit être installée.

Il est nécessaire d'indiquer :

- Le nom du serveur. 

- Le nom de la base de données et son port.

- Le nom et le mot de passe d'un administrateur de la base de données HFSQL.




Selon la taille de la base de sources du GDS, la migration de la base de sources peut être relativement longue.

A la fin de cette opération, le GDS propose de créer automatiquement le compte utilisateur correspondant à la personne connectée (si celui-ci n'existe pas).

A la prochaine ouverture du projet, il sera nécessaire d'indiquer les nouveaux paramètres de connexion au GDS.

**Remarque** : La base de sources du GDS en mode "Classic" n'est pas supprimée. Après l'opération de migration au format HFSQL Client/Serveur, il est nécessaire de prévenir tous les utilisateurs afin qu'ils changent leurs paramètres de connexion à la base de sources du GDS.


<a name="NOTE4_4"></a>


### Réindexer et réparer
<a name="reindexer_reparer_ELTPARAGRAPHE000349"></a>Sous le volet "Outils", dans le groupe "Maintenance", cliquez sur "Réindexer et réparer". Cette option permet de :

- Réindexer la base de sources du GDS (qu'elle soit en mode HFSQL Classic ou en mode HFSQL Client/Serveur).

- Annuler les transactions bloquées sur la base de sources du GDS.

- Analyser et réparer la base de sources du GDS.





<a name="NOTE4_5"></a>


### Réparer un répertoire
<a name="reparer_repertoire_ELTPARAGRAPHE000376"></a>Dans certains cas, la réparation d'un répertoire permet de régler certains problèmes d'accès aux données d'un répertoire du GDS. Cette option permet de réparer le répertoire sans bloquer la base de sources du GDS pendant la réparation. 

Pour réparer un répertoire : 

- Sélectionnez le répertoire à réparer dans la liste des répertoires de la base de sources du GDS. 

- Sous le volet "Outils", dans le groupe "Maintenance", cliquez sur "Réparer un répertoire". Cette option permet de réparer la base de sources du GDS pour ce répertoire.




Remarque : Si nécessaire, le GDS propose automatiquement de réparer uniquement le répertoire (au lieu de réparer toute la base de sources du GDS).

## Recherche dans le GDS
<a name="recherche_dans_gds_ELTTEXTE000733"></a>
Plusieurs modes de recherche sont disponibles dans la base de sources du GDS :

- **Recherche par nom** (sous le volet "Outils", dans le groupe "Recherche", cliquez "Par nom") : Cette option permet de rechercher simplement un élément dans la base de sources du GDS. Il suffit d'indiquer le nom de l'élément recherché.

- **Recherche par extraction** (sous le volet "Outils", dans le groupe "Recherche", cliquez "Par extraction") : Cette option permet de rechercher les extractions selon l'utilisateur (utilisateur en cours, utilisateur spécifique, tous les utilisateurs) et le type d'extraction (pour test, exclusive, ...). Il est alors possible de réintégrer un élément ou encore d'annuler une extraction.

- **Recherche par tâche ou incident** : Cette option permet de rechercher le ou les éléments correspondant à la réalisation d'une tâche ou d'un incident. Il est possible de voir les modifications effectuées.

- **Recherche dans l'historique** : Cette option permet de réaliser une recherche dans l'historique des actions réalisées dans le GDS. Il est ainsi possible d'obtenir tous les commentaires saisis entre deux dates et de les exporter dans un fichier. Pour plus de détails, consultez [Recherche dans l'historique du GDS](../Editeurs/2038028.md).

## Options diverses
<a name="options_diverses_ELTTEXTE000757"></a>


### Mes Projets
<a name="mes_projets_ELTPARAGRAPHE000445"></a>Pour lister les projets ayant un répertoire local sur le poste en cours : 

1. Sous le volet "Outils", dans le groupe "Recherche", cliquez sur "Mes projets". 

2. Dans la fenêtre qui s'affiche, spécifiez si nécessaire le login voulu et cliquez sur le bouton "Rechercher". 

3. Pour chaque projet trouvé, le répertoire distant et le répertoire local sont affichés.




Il est possible de supprimer l'association entre le répertoire distant et le répertoire local (bouton "Supprimer l'association"). Cette option est intéressante par exemple lorsqu'un projet du GDS a été ouvert puis supprimé du poste local : en supprimant l'association, l'utilisateur ne recevra plus les messages automatiques du GDS concernant ce projet.
<a name="NOTE6_2"></a>


### Renommage ou changement de machine
<a name="renommage_changement_machine_ELTPARAGRAPHE000470"></a>L'option "Outils .. Renommage ou changement de machine" permet de mettre à jour dans le GDS le nom du poste utilisé. Pour plus de détails, consultez [Renommage et changement de machine](../Editeurs/2038020.md).
<a name="NOTE6_3"></a>


### Statistiques
<a name="statistiques_ELTPARAGRAPHE000480"></a>L'option "Outils .. Statistiques" permet d'obtenir diverses statistiques sur la base de sources du GDS.

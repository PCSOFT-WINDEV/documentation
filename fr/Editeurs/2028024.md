


## WDInst : installer des éléments supplémentaires 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000307"></a>
Par défaut, seuls les fichiers listés dans la fenêtre "Liste des fichiers de l'installation" sont installés sur les postes utilisateurs.

L'éditeur d'installation WDInst offre la possibilité de :

- [Installer des outils redistribuables WINDEV](#NOTE2_1).

- [Distribuer le MDAC](#NOTE3_1).

- [Installer et configurer le driver ODBC](#NOTE4_1).

- [Inclure un désinstallateur automatique](#NOTE5_1).




**Remarque** : Ces options peuvent également être configurées dans l'assistant de création du programme d'installation.

**Installer des éléments supplémentaires**
La gestion des éléments supplémentaires à installer est réalisée sous l'éditeur d'installation WDInst : sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDInst". 
L'assistant de création du programme d'installation permet également d'installer des éléments supplémentaires.

## Installer des outils redistribuables WINDEV
<a name="installer_des_outils_redistribuables_windev_ELTTEXTE000331"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000053"></a>Certains outils WinDev sont diffusables librement et gratuitement avec vos applications :

| Outils | Description |
| --- | --- |
| DplViewer | Permet de visualiser les duplicata. |
| [ReplicSynchro](../ReplicSynchro/3526550.md) | Permet de gérer la Réplication Universelle assistée. |
| [WDConver](../WDConver/3512006.md) | Convertit des fichiers externes vers des fichiers de données HFSQL. |
| [WDDebug](../Editeurs/2019021.md) | Permet de débugger une application à distance |
| [WDJournal](../WDJournal/3516012.md) | Sauvegarde et restaure les fichiers de données au format HFSQL et gère les journaux. |
| [WDOptimiseur](../WDOptimiseur/3519002.md) | Optimise l'accès aux fichiers de données HFSQL (ré-indexation, corrections des mémos, ...). |
| [WDRéplic](../WDReplic/3520016.md) | Gère la réplication des données entre plusieurs postes utilisant la même application. |
| [WDSQL](../WDSQL/3521011.md) | Permet de définir une connexion et de créer des requêtes en code SQL sur cette connexion. |
| WDTelDiagno | Liste les lignes TAPI disponibles, et précise la ligne qui sera utilisée par défaut lors de l'utilisation des fonctions de téléphonie. |
| [WDTrans](../WDTrans/3524005.md) | Permet d'annuler des transactions ou de libérer des enregistrements en transaction. |


Pour limiter la multiplication des applications sur le poste de l'utilisateur final, les fichiers nécessaires à ces outils seront installés dans le répertoire "\\Program Files\\Fichiers communs\\ PC SOFT\\28.0". Le chemin du répertoire des outils optionnels peut être connu grâce à la fonction [SysRep](../WDLang1/3073026.md). Par exemple : SysRep(srProgramFilesCommun)+"\\PC SOFT\\28.0".

Si un raccourci de l'application est créé dans le menu "Démarrer", le raccourci de chacun des outils installés sur les postes utilisateurs sera également ajouté dans le menu "Démarrer" (dans le même groupe de programmes que celui de l'application).

Par défaut, si l'application est associée à une base de données HFSQL, l'optimiseur HFSQL sera installé.
<a name="NOTE2_2"></a>


### Pour installer des outils redistribuables WINDEV sur les postes utilisateurs :
<a name="pour_installer_des_outils_redistribuables_windev_sur_les_postes_utilisateurs_ELTPARAGRAPHE000129"></a>

1. Sélectionnez l'option "Paramètres de l'installation .. Paramètres avancés".

2. Cochez l'option "Inclure des outils optionnels lors de la création de l'installation.".

3. Cliquez sur le bouton "Outils" pour sélectionner les outils à inclure.

4. Sélectionnez les outils à inclure et validez.




**Remarque** : Si vous installez un outil WINDEV sans l'aide de WDInst, il est nécessaire de fournir les fichiers suivants aux utilisateurs :

- Le programme exécutable de l'outil (fichier ".EXE").

- Les librairies WINDEV (fichiers "wd280XXX.DLL").

- Les fichiers composants (fichiers ".WDK") présents dans le sous-répertoire "Programmes" du répertoire d'installation de WINDEV.

## Distribuer le MDAC
<a name="distribuer_mdac_ELTTEXTE000361"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000157"></a>Si votre application manipule une base de données via OLE DB, les éléments suivants doivent être installés sur les postes utilisateurs :

- un provider OLE DB (par exemple "MSDAORA", "OraOLEDB.Oracle", "SQLOLEDB", ...).

- le MDAC (Microsoft Data Access Component) version 2.6 minimum.




Si le MDAC n'est pas installé, seule la connexion à la base OLE DB sera possible : il sera impossible de lire les tables (un message signale que cette interface n'est pas prise en charge).

**Attention** : La licence de redistribution du MDAC précise qu'il est nécessaire de télécharger l'exécutable du MDAC (depuis le site Internet de Microsoft) pour obtenir le droit de le redistribuer avec une application. Avant d'effectuer les manipulations décrites dans les paragraphes suivants, il est donc nécessaire de télécharger l'exécutable du MDAC sur le poste en cours.

Vous pouvez télécharger l'exécutable du MDAC depuis le site Internet de Microsoft aux adresses suivantes (adresse valide lors de la réalisation de cette documentation) : [http://www.microsoft.com/data](http://www.microsoft.com/data)
<a name="NOTE3_2"></a>


### Distribuer l'exécutable du MDAC à vos utilisateurs
<a name="distribuer_executable_mdac_vos_utilisateurs_ELTPARAGRAPHE000174"></a>Pour distribuer l'exécutable du MDAC à vos utilisateurs :

1. Sélectionnez l'option "Paramètres de l'installation .. Fichiers HFSQL".

2. Cochez l'option "Installer le MDAC situé à l'emplacement suivant".

3. Spécifiez le chemin de l'exécutable du MDAC précédemment téléchargé.

## Installer et configurer le driver ODBC
<a name="installer_configurer_driver_odbc_ELTTEXTE000391"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000187"></a>Pour manipuler une base de données HFSQL depuis une application non WINDEV, il est nécessaire d'installer et de configurer le driver ODBC sur HFSQL sur le poste utilisateur. Pour plus de détails, consultez le [Driver ODBC sur HFSQL](../WDLang4/3044179.md).

**Remarque** : Le driver ODBC sur HFSQL permet de **lire et d'écrire** dans une base de données HFSQL depuis une application non WINDEV.
<a name="NOTE4_2"></a>


### Installer et configurer le driver ODBC
<a name="installer_configurer_driver_odbc_ELTPARAGRAPHE000199"></a>Pour installer et configurer le driver ODBC :

1. Sélectionnez l'option "Paramètres de l'installation .. Fichiers HFSQL".

2. Cochez l'option "Installer le driver ODBC pour HFSQL".

3. Saisissez le nom permettant d'identifier la base de données HFSQL lors de l'utilisation du driver ODBC sur HFSQL.

4. Saisissez le chemin de l'analyse utilisée comme source ODBC (fichier .WDD correspondant à l'analyse WINDEV).

## Inclure un désinstallateur automatique
<a name="inclure_desinstallateur_automatique_ELTTEXTE000421"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000213"></a>Pour simplifier la désinstallation de votre application, l'éditeur d'installation WDInst permet d'inclure un désinstallateur dans le programme d'installation.

Dans ce cas, le fichier "uninst.INF" sera créé dans le répertoire d'installation de l'application. Ce fichier liste tous les fichiers de l'application installés sur le poste en cours.

**Remarque** : Pour pouvoir désinstaller l'application, le désinstallateur utilise une entrée dans la base de registre. L'éditeur d'installation WDInst permet de modifier cet identifiant. Pour plus de détails, consultez [Modifier l'identifiant utilisé pour désinstaller l'application](../Editeurs/2028013.md).
<a name="NOTE5_2"></a>


### Désinstaller votre application
<a name="desinstaller_votre_application_ELTPARAGRAPHE000227"></a>Comme pour toutes applications Windows ayant un désinstallateur, pour désinstaller une application, il suffit de :

- afficher la liste des programmes installés sur le système en cours (option "Ajout/Suppression de programmes" dans le panneau de configuration).

- sélectionner l'application dans la liste des programmes.

- cliquer sur le bouton "Modifier/Supprimer".
	Les éléments suivants seront supprimés :

	- les fichiers listés dans le fichier "uninst.INF".

	- les entrées dans la base de registre créées lors de l'installation de l'application.

	- les raccourcis de l'application (sur le bureau et dans le menu "Démarrer") s'ils existent.

	- les fichiers de données (".FIC", ".NDX" et ".MMO") après confirmation de l'utilisateur.

	- les fichiers sauvegardés lors des différentes mises à jour (s'ils existent) après confirmation de l'utilisateur.

	- les répertoires dont les fichiers sont supprimés lors de la désinstallation.







**Cas particulier** : Si l'application crée des fichiers de données dans un répertoire spécifique (répertoire différent des sous-répertoires de l'application), ces fichiers de données ne sont pas supprimés automatiquement lors de la désinstallation.

**Remarque** : Il est possible de traduire les différents messages du désinstalleur. Pour plus de détails, consultez [Traduire le désinstalleur](../Editeurs/2028087.md). 
<a name="NOTE5_3"></a>


### Inclure un désinstallateur dans le programme d'installation
<a name="inclure_desinstallateur_dans_programme_installation_ELTPARAGRAPHE000254"></a>Pour inclure un désinstallateur dans le programme d'installation :

1. Sélectionnez l'option "Paramètres de l'installation .. Fin de l'installation".<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Options%20install_WDINST%20-%20HC%20N%B0005%202.gif)


2. Cochez l'option "Inclure le désinstallateur".

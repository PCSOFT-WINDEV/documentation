## Présentation
<a name="presentation_ELTTEXTE000511"></a>
Il existe différentes versions de la base de sources GDS :

- **Version 10** : version initiale.

- **Version 12 (base de sources au format 12.0.0.205)** créée à partir de WINDEV 12 : 
	Cette version permet de gérer les droits d'accès aux projets et aux éléments du projet. La sécurité des sources est renforcée.

- **Version 15 (base de sources au format 15.0.0.306)** créée à partir de WINDEV 15 : 
	Cette version permet de sauvegarder l'historique des sources par différences binaires. La taille de la base de sources est allégée.

- **Version 17 (base de sources au format 17.0.0.407)** créée à partir de WINDEV 17 : 
	Cette version permet d'ajouter dans la base de sources des éléments dont le nom contient des caractères non latins. 

- **Version 22 (base de sources au format 22.0.0.408.0)** crée à partir de WINDEV 22 et 23 : 
	Cette version permet d'utiliser toutes les possibilités de la gestion des branches.

- **Version 24 Update 2 (base de sources au format 22.0.0.412.9)** créée à partir de WINDEV 24 Update 2 : 
	Cette version permet d'utiliser le partage de répertoires entre différentes bases de sources.

- **Version 24 Update 3 (base de sources au format 24.0.0.412.16)** créée à partir de WINDEV 24 Update 3 : 
	Cette version permet d'utiliser les optimisations pour les opérations réalisées en ADSL (extraction, réintégration, ...).

- **Version 25 (base de sources au format 24.0.0.412.17)** créée à partir de WINDEV 25 : 
	Cette version permet d'utiliser l'optimisation de la vitesse de création des branches et des accès à distance.

- **Version 26 (base de sources au format 24.0.0.414.17)** créée à partir de WINDEV 26.

**Remarque** : **Création de la base de sources GDS** :

- WINDEV 10 et 11 créent une base de sources GDS en version 10.

- WINDEV 12 et 14 créent une base de sources GDS en version 12.

- WINDEV 15 et 16 créent une base de sources GDS en version 15.

- A partir de la version 17 de WINDEV, WEBDEV et WINDEV Mobile, il est possible de créer une base de sources GDS : 

	- en version 15 (par défaut).

	- en version 17 en cochant "Activer le support de l'Unicode" lors de la création de la base de sources.




- A partir de la version 22 de WINDEV, WEBDEV et WINDEV Mobile, il est possible de créer une base de sources en version 22.

- A partir de la version 24 Update 2 de WINDEV, WEBDEV et WINDEV Mobile, il est possible de créer une base de sources en version 24.

- A partir de la version 24 Update 3 de WINDEV, WEBDEV et WINDEV Mobile, il est possible de créer une base de sources en version 24 Update 3.

- A partir de la version 25 de WINDEV, WEBDEV et WINDEV Mobile, il est possible de créer une base de sources en version 25.

- A partir de la version 26 de WINDEV, WEBDEV et WINDEV Mobile, il est possible de créer une base de sources en version 26.

## Comment connaître la version d'une base de sources GDS ?
<a name="comment_connaitre_version_une_base_sources_gds_ELTTEXTE000535"></a>
**Pour connaître la version d'une base de sources GDS :**

1. Lancez l'administrateur du GDS : sous le volet "GDS", dans le groupe "Base GDS", déroulez "Gérer" et sélectionnez "Administrateur du Gestionnaire de Sources". 

2. Dans l'administrateur du GDS, sous le volet "Outils", dans le groupe "Statistiques", cliquez sur "Statistiques".

3. La fenêtre des statistiques du GDS s'affiche. Cette fenêtre contient les informations sur la base de sources : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=P4_Int%E9gration%20dans%20le%20GDS%20-%20HC%20N%B0013.jpg)


	- Le champ "Format de la Base de sources" indique la version de la base de sources du GDS. 

- Le champ "Version des procédures stockées" indique la version des procédures stockées. Le champ "Version des procédures stockées" n'est pas présent dans les anciennes versions de l'administrateur du GDS. Dans ce cas la version des procédures stockées est accolée à droite de la version. Par exemple dans le numéro de version 22.0.0.412.9, 22.0.0.412 correspond à la version, et .9 à la version des procédures stockées de la base GDS.

- Le champ "Version du serveur HFSQL" permet de connaître la version du serveur HFSQL utilisée pour le GDS. Dans le cas d'une base de données locale, le terme "ISAM" est affiché.

## Comment migrer le format de la base de sources du GDS ?
<a name="comment_migrer_format_base_sources_gds_ELTTEXTE000559"></a>
**Pour migrer la base de sources de GDS dans le nouveau format :**

1. Lancez l'administrateur du GDS : sous le volet "GDS", dans le groupe "Base GDS", déroulez "Gérer" et sélectionnez "Administrateur du Gestionnaire de Sources".

2. Connectez-vous à l'administrateur du GDS en tant qu'administrateur.

3. Sous le volet "Outils", dans le groupe "Maintenance", cliquez sur "Migrer la base dans le format le plus récent".




**Attention** :

- **si la base de sources du GDS contient des projets en version 12 qui n'ont pas encore été recompilés dans une version plus récente, dans ce cas, il ne faut pas utiliser les formats récents de la base de sources GDS.**

- **si la base de sources GDS contient des projets utilisés par des versions 14, 15, 16 et 17**, il est possible d'utiliser le format le plus récent de la base de sources proposé dans l'administrateur du GDS de la version 15. Cas particulier pour la version 14, il faut bien installer la mise à jour spécifique au GDS. Pour cela, il est nécessaire de demander le module WX140PACKGDS030f.exe depuis les correctifs des ressources pratiques : [https://www.pcsoft.fr/st/nouveautes-st.html](https://www.pcsoft.fr/st/nouveautes-st.html). 

- **si la base de sources GDS contient uniquement des projets en version 17**, il est possible d'utiliser le format de la base de sources GDS proposé par l'administrateur GDS de la version 17. La base de sources sera alors convertie en UNICODE, il sera alors possible de stocker des éléments, des dossiers, des commentaires dans tous les alphabets.

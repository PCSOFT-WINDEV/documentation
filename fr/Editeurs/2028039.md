


## Déploiement d'un site (depuis le poste de développement)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000191"></a>
Quel que soit le type de site créé avec WEBDEV (statique ou dynamique), WEBDEV version Développement permet d'automatiser l'installation du site sur le serveur.

- **Si vous développez un site statique ou un site PHP** :
	WDDéploie permet de déployer votre site statique ou PHP soit par FTP soit par réseau. Pour lancer WWDéploie, sous le volet "Projet", dans le groupe "Génération", déroulez "Déployer le site" et sélectionnez "Générer et déployer le site". Pour plus de détails, consultez [Aide de WDDéploie](../WDDeploie/3533001.md).
	
	




- **Si vous développez un site dynamique** : 
	Pour lancer l'assistant d'installation, sous le volet "Projet", dans le groupe "Génération", déroulez "Déployer le site" et sélectionnez "Générer et déployer le site".
	Cet assistant vous propose :

	1. La création si nécessaire de la bibliothèque du site dynamique. Pour plus de détails, consultez [Options de création de la bibliothèque](../Editeurs/2028097.md).
			**Rappel** : Une bibliothèque est un fichier de type WDL qui contient toutes les ressources WEBDEV : pages, états, classes, collections de procédures, ... La présence d'une bibliothèque est obligatoire pour un site dynamique.
			

	2. Le choix du type d'installation :

		- [Déploiement dans PCSCloud](../Editeurs/2028107.md) (plus d'informations sur [http://www.pcscloud.net](http://www.pcscloud.net)). 

		- [Déploiement direct du site sur un Serveur d'Application WEBDEV distant ou sur un Cluster WEBDEV.](../Editeurs/2028019.md) Cette installation sera réalisée par FTP ou HTTP.

		- [Création d'un package de déploiement](../Editeurs/2028089.md).

		- [Installation par média physique](../Editeurs/2028090.md).

		- [Installation par média physique autonome](../Editeurs/2028091.md).

		- Installation via une image docker du site. 

		- [Déploiement du site WEBDEV via le service d'hébergement de test de PC SOFT](../Editeurs/2028101.md).









- **Si vous développez un Webservice** : 
	A la fin de l'assistant de génération du Webservice, le déploiement du Webservice est automatiquement proposé. Lors du déploiement sur un serveur d'application WEBDEV, les types d'installation disponibles sont les suivants : 

	- [Déploiement direct du site sur un Serveur d'Application WEBDEV distant ou sur un Cluster WEBDEV.](../Editeurs/2028019.md) Cette installation sera réalisée par FTP ou HTTP.

	- [Déploiement dans PCSCloud](../Editeurs/2028107.md) (plus d'informations sur [http://www.pcscloud.net](http://www.pcscloud.net)). 

	- [Création d'un package de déploiement](../Editeurs/2028089.md).

	- [Installation par média physique](../Editeurs/2028090.md).

	- [Installation par média physique autonome](../Editeurs/2028091.md).

	- [Déploiement du Webservice sur le poste local](../WDLang3/3086002.md). Cette option est proposée uniquement si WEBDEV version Développement est installé sur le poste en cours.

	- [Déploiement du site WEBDEV via le service d'hébergement de test de PC SOFT](../Editeurs/2028101.md).




## Recycler les fenêtres de code
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000119"></a>
Par défaut, une seule fenêtre de code est ouverte par élément ouvert sous l'éditeur.

Par exemple, une fenêtre de code affiche le code d'un bouton d'une fenêtre. Si une nouvelle fenêtre de code est ouverte pour afficher le code d'un champ de cette fenêtre, la fenêtre affichant le code du bouton est utilisée.

C'est le recyclage des fenêtres de code (mode par défaut).

Pour utiliser plusieurs fenêtres de code pour afficher le code de plusieurs objets d'un même élément, il suffit d'annuler le recyclage des fenêtres de code.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000143"></a>


### Mettre en place le recyclage des fenêtres de code
<a name="mettre_place_recyclage_des_fenetres_code_ELTPARAGRAPHE000024"></a>Pour recycler les fenêtres de code d'un fichier WINDEV ou WEBDEV (fenêtre, état, ...) :

1. Affichez un des codes de l'élément voulu.

2. Sous le volet "Affichage", dans le groupe "Aide à l'édition", cochez l'option "Réutiliser la même fenêtre de code".



<a name="NOTE2_2"></a>


### Utiliser plusieurs fenêtres de code pour un même élément
<a name="utiliser_plusieurs_fenetres_code_pour_meme_element_ELTPARAGRAPHE000046"></a>Pour afficher plusieurs fenêtres de code pour un élément WINDEV, WINDEV Mobile ou WEBDEV (fenêtre, état, ...) :

1. Affichez un des codes de l'élément voulu.

2. Sous le volet "Affichage", dans le groupe "Aide à l'édition", décochez l'option "Réutiliser la même fenêtre de code".




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Notes
<a name="notes_ELTTEXTE000173"></a>


### Utiliser plusieurs fenêtres de code pour un même élément WINDEV/WINDEV Mobile/WEBDEV (fenêtre, état, ...)
<a name="utiliser_plusieurs_fenetres_code_pour_meme_element_windevwindev_mobilewebdev_fenetre_etat_ELTPARAGRAPHE000070"></a>Si le code d'un même élément (code d'un bouton, code d'un champ, ...) est demandé plusieurs fois, la fenêtre de code correspondante est ré-activée.

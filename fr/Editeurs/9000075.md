


## Anomalies du projet : code mort et éléments orphelins
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000179"></a>
Lors du développement d'un projet, bien souvent, des codes sont écrits, puis abandonnés pour une raison ou une autre. De même, des fenêtres ou des états peuvent être créés puis abandonnés. 

WINDEV, WEBDEV et WINDEV Mobile permettent de détecter simplement ces "codes morts" et éléments "orphelins". 

Grâce à l'audit statique, vous pouvez obtenir un rapport complet sur les différents éléments posant problème dans votre projet. 

Mais il est également possible de retrouver spécifiquement : 

- les procédures du projet qui ne sont pas utilisées (code mort).

- les procédures de l'élément en cours qui ne sont pas utilisées (code mort).

- les éléments du projets qui ne sont pas utilisés (éléments orphelins).




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000203"></a>


### Afficher le code mort du projet
<a name="afficher_code_mort_projet_ELTPARAGRAPHE000030"></a>Pour afficher le code mort du projet :

1. Sous le volet "Projet", dans le groupe "Audit et performances", déroulez "Audit statique" et sélectionnez "Code mort du projet".

2. La fenêtre listant les codes morts de votre projet s'affiche. 

3. Dans cette fenêtre, il est possible de : 

	- Marquer la procédure comme utilisé (voir ci-dessous). 

	- Supprimer la procédure. 

	- Voir la procédure.

	- Lancer la recherche. 








### Afficher le code mort de l'élément en cours
<a name="afficher_code_mort_element_cours_ELTPARAGRAPHE000058"></a>Pour afficher le code mort de l'élément en cours :

1. Affichez sous l'éditeur l'élément à analyser : fenêtre, état, page, ...

2. Sous le volet "Projet", dans le groupe "Audit et performances", déroulez "Audit statique" et sélectionnez "Code mort de l'élément en cours".

3. La fenêtre listant les codes morts de l'élément en cours s'affiche. 

4. Dans cette fenêtre, il est possible de : 

	- Marquer la procédure comme utilisé (voir ci-dessous). 

	- Supprimer la procédure : Permet de supprimer la procédure. 

	- Voir la procédure : Permet d'afficher le code de la procédure. Il est également possible de double-cliquer sur le nom de la procédure. 

	- Lancer la recherche : Permet de rechercher la procédure dans le projet en cours. 








### Afficher les éléments orphelins du projet
<a name="afficher_les_elements_orphelins_projet_ELTPARAGRAPHE000085"></a>Pour afficher les éléments orphelins du projet :

1. Sous le volet "Projet", dans le groupe "Audit et performances", déroulez "Audit statique" et sélectionnez "Eléments orphelins".

2. La fenêtre listant les éléments orphelins de votre projet s'affiche. 

3. Dans cette fenêtre, il est possible de : 

	- Marquer l'élément comme utilisé (voir ci-dessous).

	- Supprimer l'élément : Permet de supprimer l'élément. 

	- Voir l'élément : Permet de voir l'élément. 

	- Lancer la recherche : Permet de rechercher l'élément dans le projet. 






<a name="NOTE2_2"></a>


### Marquer du code ou un élément comme utilisé
<a name="marquer_code_element_comme_utilise_ELTPARAGRAPHE000113"></a>**Attention** : Dans certains cas, l'audit d'édition du projet ou de l'élément peut lister des éléments qu'il considère être du code mort ou des éléments orphelins. En effet, ces éléments peuvent correspondre à :

- des procédures appelées dans un autre projet.

- des procédures ou éléments appelés en compilation dynamique.

- des éléments dont le nom est construit dynamiquement.




Ces éléments peuvent être marqués comme "utilisés". Ainsi, ces éléments n'apparaîtront plus dans la liste des éléments non vérifiés. Ces éléments apparaîtront dans la liste des "Procédures ou éléments marqués comme utilisés".

**Marquer directement un élément comme inutilisé**
Une fenêtre ou une page de test présente dans votre projet peut être analysée comme "inutilisée". Pour que cet élément soit directement inscrit dans les éléments marqués comme utilisés, il suffit de le préciser dans le code de "Déclaration globales" de l'élément en utilisant la syntaxe suivante : 

```txt
<utile = "Raison">
```
où "Raison" correspond au texte indiquant au développeur la raison de la présence de l'élément dans la liste des éléments marqués comme utilisés. Ce texte sera affiché dans la colonne "Commentaire" de la fenêtre de gestion des éléments orphelins.  

Par exemple : 

```txt
<utile = "Fenêtre de tests internes">
```




## Centres de contrôle : Travailler en mode nomade
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000136"></a>
Lors d'un travail sur un portable ou un poste temporairement déconnecté du réseau, il peut être intéressant de continuer à travailler sur les données des Centres de Contrôle (Centre de Suivi de projets, ...).

WINDEV, WEBDEV et WINDEV Mobile permettent de gérer un mode "Nomade" pour les Centres de Contrôle. Ce mode permet de se déconnecter de la base de données des Centres de Contrôle et de copier cette base de données en local sur le poste. Ainsi, même déconnecté, vous accédez à toutes vos données.

Lors de la reconnexion, les données modifiées localement sont automatiquement mise à jour sur la base de données réseau. En cas de conflit (modification d'une tâche par différentes personnes par exemple), la base du réseau sera toujours prioritaire.

**Remarque** : Lors de la déconnexion, vous pourrez utiliser les outils suivants : Centre de Suivi de Projets, Centre de suivi de qualité, Messagerie. 

**Attention : le mode nomade n'est pas disponible si vous utilisez les Centres de Contrôle avec une base de données HFSQL Client/Serveur.**



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Travailler en mode nomade
<a name="travailler_mode_nomade_ELTTEXTE000166"></a>
Pour travailler en mode "Nomade" : 

1. Sous le volet "GDS", dans le groupe "Autres actions", déroulez "Travail à distance" et sélectionnez "Se déconnecter pour une utilisation nomade".

2. Dans l'assistant qui apparaît, sélectionnez l'option "Je veux me déconnecter de la base partagée des centres de contrôle".
	Attention : Cet assistant gère aussi la déconnexion de la base du GDS pour le projet en cours. Pour plus de détails, consultez [GDS : Travailler en mode Nomade](../Editeurs/2038013.md).

3. Sélectionnez les différentes options de la base locale :

	- Les données optionnelles à ajouter

	- Le chemin de la base de données (il est conseillé de conserver le chemin proposé).




4. Vous utilisez maintenant la base de données locale pour tous vos centres de contrôle.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Se reconnecter
<a name="reconnecter_ELTTEXTE000190"></a>
Pour se reconnecter à la base de données des Centres de Contrôle :

1. Sous le volet "GDS", dans le groupe "Autres actions", déroulez "Travail à distance" et sélectionnez "Se reconnecter et synchroniser".

2. Dans l'assistant qui apparaît, sélectionnez l'option "Je veux me reconnecter de la base partagée des centres de contrôle".
	Attention : Cet assistant gère aussi la reconnexion de la base du GDS pour le projet en cours. Pour plus de détails, consultez [GDS : Travailler en mode Nomade](../Editeurs/2038013.md).

3. Sélectionnez les différentes options :

	- Base partagée à utiliser : la base partagée utilisée précédemment est automatiquement proposée.

	- Le chemin de la base de données locale.
			Si vous avez supprimé les fichiers de la base locale, cochez l'option "J'ai perdu ma base locale". Dans ce cas, vous serez reconnecté à la base partagée, mais aucune synchronisation ne sera effectuée.




4. Vous êtes maintenant reconnecté à la base partagée.




Lors de la reconnexion, les données modifiées localement sont automatiquement mises à jour sur la base de données partagée. En cas de conflit (modification d'une tâche par différentes personnes par exemple), la base du réseau sera toujours prioritaire.




## GDS : Gestion des droits
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000201"></a>
La gestion des droits d'accès aux éléments du GDS permet de définir les différents droits des développeurs sur chaque élément du GDS.

L'accès à chaque élément peut être restreint en fonction des droits du développeur : ces droits sont associés soit au compte de connexion, soit à un groupe (uniquement si la base de sources du GDS est en mode Client/Serveur).

La définition des droits est réalisée depuis l'administrateur du GDS. Pour lancer l'administrateur du GDS, sous le volet "GDS", dans le groupe "Base GDS", déroulez "Gérer" et sélectionnez "Administrateur du Gestionnaire de Sources". 

Le plus souvent, la gestion des droits permet de :

- restreindre les droits accordés à tout le monde, pour accorder des droits spécifiques à certains utilisateurs.

- restreindre les droits de certains utilisateurs (stagiaires par exemple).




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000225"></a>


### Définir les droits sur un répertoire
<a name="definir_les_droits_sur_repertoire_ELTPARAGRAPHE000039"></a>Pour définir les droits sur un répertoire :

1. La définition des droits est réalisée depuis l'administrateur du GDS. Pour lancer l'administrateur du GDS, sous le volet "GDS", dans le groupe "Base GDS", déroulez "Gérer" et sélectionnez "Administrateur du Gestionnaire de Sources".

2. Sélectionnez un répertoire.

3. Dans le menu contextuel du répertoire, sélectionnez l'option "Propriétés". La fenêtre des propriétés du répertoire s'affiche.

4. Sélectionnez l'onglet "Droits". Par défaut, tout le monde a tous les droits.



<a name="NOTE2_2"></a>


### Définir les droits sur un fichier
<a name="definir_les_droits_sur_fichier_ELTPARAGRAPHE000063"></a>Pour définir les droits sur un fichier : 

1. La définition des droits est réalisée depuis l'administrateur du GDS. Pour lancer l'administrateur du GDS, sous le volet "GDS", dans le groupe "Base GDS", déroulez "Gérer" et sélectionnez "Administrateur du Gestionnaire de Sources".

2. Sélectionnez un fichier.

3. Dans le menu contextuel du fichier, sélectionnez l'option "Propriétés". La fenêtre des propriétés du fichier s'affiche.

4. Sélectionnez l'onglet "Droits". Par défaut, tout le monde a tous les droits.



<a name="NOTE2_3"></a>


### Ajouter des droits pour un utilisateur ou un groupe
<a name="ajouter_des_droits_pour_utilisateur_groupe_ELTPARAGRAPHE000087"></a>Pour ajouter des droits pour un utilisateur ou un groupe : 

1. Affichez la fenêtre de gestion des droits du fichier ou du répertoire.

2. Cliquez sur "Ajouter".

3. Indiquez si le droit doit s'appliquer pour :

	- un groupe. Les groupes sont disponibles uniquement si vous utilisez une base de sources du GDS en mode Client/Serveur. Dans ce cas, les groupes correspondent aux groupes définis dans le Centre de Contrôle HFSQL pour la base de sources du GDS.

	- un utilisateur. Vous pouvez saisir n'importe quel nom d'utilisateur.




4. Validez.

5. Sélectionnez l'utilisateur ou le groupe dont les droits doivent être modifiés.

6. Modifiez les droits de cet utilisateur ou de ce groupe dans la partie "Détail des droits". Les droits disponibles sont :

	- ***Contrôle total*** : L'utilisateur peut réaliser toutes les opérations.

	- ***Lecture (L)*** : L'utilisateur peut lire l'élément. Ce droit permet de récupérer une version de l'élément et permet également d'extraire pour test. Si un utilisateur n'a pas le droit de lecture sur un répertoire, les fichiers du répertoire sont invisibles. Le droit d'écriture force le droit de lecture.

	- ***Ecriture*** :

		- ***Extraire et réintégrer*** : Ce droit permet d'extraire et de réintégrer un élément.

		- ***Ajouter (A)*** : Ce droit permet d'ajouter des fichiers et des répertoires, d'ajouter un partage, d'ajouter des étiquettes, de créer des branches.

		- ***Renommer (R)*** : Ce droit permet de renommer un fichier ou un répertoire.

		- ***Supprimer (S)*** : Ce droit permet de supprimer des fichiers, des répertoires ou des partages. Il permet également de remplacer un partage par un autre partage.

		- ***Supprimer définitivement (X)*** : Ce droit permet de supprimer définitivement un fichier ou un répertoire. Ce droit s'applique uniquement pour le dernier partage d'un fichier.




- ***Modifier les droits (M)*** : L'utilisateur peut modifier les permissions des autres utilisateurs.

7. Cliquez sur "Appliquer" pour appliquer les nouveaux droits ou sur "Annuler" pour ne pas les prendre en compte.




**Remarque** : L'administrateur a toujours tous les droits (contrôle total).
<a name="NOTE2_4"></a>


### Héritage des droits
<a name="heritage_des_droits_ELTPARAGRAPHE000150"></a>

- Lorsque des droits sont appliqués à un répertoire, ces droits sont répercutés par défaut à tous les éléments du répertoire.  

- Lorsqu'un nouveau répertoire est créé, ce répertoire hérite automatiquement des droits du répertoire parent.

- Par défaut, une base de sources est créée avec le contrôle total pour tout le monde sur la racine.


## Gestion automatique des URL courtes
			

<a name="NOTE0_0"></a>


## Présentation
<a name="presentation_ELTTEXTE000077"></a>
Pour les sites dynamiques WEBDEV classiques (avec contexte), l'accès au site peut se faire : 

- par une URL "classique" :
	
	```txt
	http://www.monsite.com/WD260AWP/WD260AWP.EXE/CONNECT/NomProjet
	```


- par une URL courte. L'URL courte a l'avantage d'être plus facile à retenir et de rendre transparente la version du serveur d'application WEBDEV utilisé.
	
	```txt
	http://www.monsite.com/NomProjet
	```





**Les deux types d'URL sont automatiquement disponibles lors de l'installation d'un site dynamique WEBDEV classique.**

**Attention** : Les URL courtes fonctionnent uniquement avec les serveurs Web suivants : 

- Apache 2.2 et supérieur,

- IIS 7 et supérieur. 




**Remarque** : Le mécanisme des URL courtes fonctionne uniquement pour les sites dynamiques classiques. Pour les sites AWP, il est possible de masquer le nom du serveur d'application dans l'URL des pages d'un site AWP grâce au [mécanisme d'URL Rewriting](../Editeurs/2030054.md).

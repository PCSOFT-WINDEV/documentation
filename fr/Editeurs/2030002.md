


## Importation d'un projet WINDEV / WINDEV Mobile et de ses éléments
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000234"></a>
Un projet WINDEV et tous ses éléments compatibles avec WINDEV Mobile (analyse, fenêtres, classes, collections de procédures, requêtes, ...) peuvent être ouvert sous WINDEV Mobile. Inversement, un projet WINDEV Mobile peut être ouvert sous WINDEV.

Un projet WINDEV (ou WINDEV Mobile) et tous ses éléments compatibles avec WEBDEV (analyse, classes, collections de procédures, requêtes, ...) peuvent être importés dans un nouveau projet WEBDEV. 
Inversement, un projet WEBDEV peut être importé dans un nouveau projet WINDEV. Pour plus de détails, consultez [Importation d'un projet WEBDEV et de ses éléments](../Editeurs/2030041.md).

### Importer des éléments WINDEV Mobile (fenêtres, états, ...) dans un projet WINDEV
<a name="importer_des_elements_windev_mobile_fenetres_etats_dans_projet_windev_ELTPARAGRAPHE000104"></a>Pour importer des éléments WINDEV Mobile (fenêtres, états, ...) dans un projet WINDEV :

1. Ouvrez le projet WINDEV dans lequel l'import doit être effectué.  

2. Sous le volet "Projet", dans le groupe "Projet", déroulez "Importer" et sélectionnez "Des éléments WINDEV ou WINDEV Mobile". L'assistant d'importation se lance.

3. Sélectionnez le projet WINDEV Mobile contenant les éléments à importer ou directement les fichiers WINDEV Mobile à importer. Passez à l'étape suivante.

4. Sélectionnez les éléments à importer.

5. Si le projet WINDEV Mobile contient des éléments pouvant être partagés avec WINDEV (classes, collections de procédures, états, requêtes), l'assistant d'importation permet :

	- soit de partager ces éléments. Ces éléments ne sont pas recopiés physiquement dans le projet WINDEV, mais seront accessibles depuis le projet.

	- soit de recopier ces éléments dans le projet WINDEV. Ces éléments évolueront indépendamment dans les deux projets.




6. Validez. L'importation est automatiquement réalisée.

## Notes
<a name="notes_ELTTEXTE000318"></a>


### Projet manipulant des codes partagés entre WINDEV, WINDEV Mobile et WEBDEV
<a name="projet_manipulant_des_codes_partages_entre_windev_windev_mobile_webdev_ELTPARAGRAPHE000168"></a>Si les projets partagent des classes ou des collections de procédures, les éléments présents dans le projet original sont directement manipulés. Un seul code source est utilisé.

Il peut être nécessaire d'adapter le code pour fonctionner sur les différentes plateformes. Il est par exemple conseillé d'utiliser les fonctions EnModeXXX ([EnModeAndroid](../WDLang1/1000018990.md), [EnModeIOS](../WDLang1/1000019765.md), ...) et / ou l'instruction &lt;SI CibleExécution=...&gt;pour différencier un code exécutable sur un PC, un site Web et un téléphone. 



<a name="NOTE4_2"></a>


### Dépendance des fichiers
<a name="dependance_des_fichiers_ELTPARAGRAPHE000184"></a>Les éléments sont importés avec leurs dépendances : tous les fichiers liés à l'élément importé sont également importés. Les images sont automatiquement converties dans un format reconnu par WINDEV, WEBDEV ou WINDEV Mobile.

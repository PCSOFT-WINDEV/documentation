


## Version des fichiers dans le répertoire virtuel
			





## Présentation
<a name="presentation_ELTTEXTE000128"></a>
Le diagnostic de configuration affiche l'erreur suivante : "Application : Version des fichiers dans le répertoire virtuel".<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=WebDev_Virtuel.gif)
Cette erreur indique que la version des fichiers contenus le répertoire virtuel du site (répertoire "_WEB") ne correspond pas à la version du serveur d'application.

Cette erreur provient généralement d'un problème de compilation du site.

**En développement**, ces versions doivent correspondre exactement :

- Le site est compilé par WEBDEV : ses fichiers doivent donc avoir la version de WEBDEV.

- Lors d'un GO, WEBDEV utilise le serveur d'application de test inclus dans WEBDEV. WEBDEV et le serveur d'application de test doivent être de la même version.

- WEBDEV, les fichiers du site et le serveur d'application de test doivent donc être de la même version.




**En déploiement**, la version du serveur d'application doit être supérieure ou égale à la version des fichiers présents dans le répertoire virtuel (répertoire "_WEB") de l'application. 
Remarque : Le serveur d'application est compatible avec les versions antérieures de WEBDEV (versions majeures et mineures). 



## Comment diagnostiquer plus précisément le problème ?
<a name="comment_diagnostiquer_plus_precisement_probleme_ELTTEXTE000152"></a>
Le détail de l'erreur permet de connaître la version des fichiers du site (version "lue") et la version du serveur d'application (version "attendue"). 



## Comment corriger le problème ?
<a name="comment_corriger_probleme_ELTTEXTE000176"></a>
Pour corriger le problème, effectuez les opérations suivantes :

- Fermer toutes les instances de WEBDEV sauf celle dans lequel le site est édité. 

- Fermer toutes les instances de l'administrateur WEBDEV.

- Recompiler le site : sous le volet "Projet", dans le groupe "Projet", déroulez "Recompiler et synchroniser" et sélectionnez "Recompiler le projet".

- Relancer l'administrateur WEBDEV : sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDFAdmin".

- Vérifier si le diagnostic est maintenant correct.




Si le diagnostic échoue toujours sur cette erreur :

- Fermer toutes les instances de WEBDEV.

- Fermer toutes les instances de l'administrateur WEBDEV.

- S'assurer que aucune instance n'est encore en mémoire (par exemple avec le gestionnaire de tâches).

- Lancer WEBDEV (dans la version voulue).

- Ouvrir le projet.

- Faire une réparation de projet : sous le volet "Projet", dans le groupe "Projet", déroulez "Recompiler et synchroniser" et sélectionnez "Réparer le projet".

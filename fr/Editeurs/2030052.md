## Présentation
<a name="presentation_ELTTEXTE000186"></a>
La fenêtre de description du projet permet de configurer de nombreux éléments de votre projet. Cette fenêtre est constituée des onglets suivants :

- L'onglet ["Projet"](../Editeurs/2030046.md) qui permet de saisir les informations générales du projet. 

- L'onglet "Analyse" qui permet d'associer une analyse au projet et / ou un modèle UML. 

- L'onglet **"Fichiers"** qui permet de saisir différentes informations sur les fichiers de données gérés par l'application (uniquement disponible si le projet est lié à une analyse).

- L'onglet "Live data" qui permet de configurer l'affichage des données (provenant des fichiers de données) sous les différents éditeurs.

- L'onglet ["Langues"](../Editeurs/9500219.md) qui permet de paramétrer les différentes langues manipulées par le projet ainsi que les options linguistiques à utiliser pour chaque langue.

- L'onglet ["Style"](../Editeurs/9500234.md) qui permet de choisir le thème de gabarit, la feuille de styles personnalisée, et de personnaliser l'aperçu avant impression. Il est également possible de paramétrer la taille des écrans.

- L'onglet "Groupe" qui permet de gérer le travail en groupe (GDS ou GIT).

- L'onglet ["Options"](../Editeurs/9500227.md) qui permet de configurer les options de mise à jour, de sauvegarde, de synchronisation des modèles. Cet onglet permet également d'activer le mode strict et de définir la charte de programmation.

- L'onglet ["Avancé"](../Editeurs/9500236.md) qui permet de gérer l'utilisation de "Etats & Requêtes", les FAA et le verrouillage de l'application.

- L'onglet ["Compilation"](../Editeurs/9500218.md) qui permet de configurer les options de compilation.

- L'onglet ["Télémétrie"](../Editeurs/9500223.md) qui permet de configurer les options de télémétrie pour le projet.

## Onglet Fichiers
<a name="onglet_fichiers_ELTTEXTE000216"></a>


### Répertoire des fichiers à utiliser en mode test
<a name="repertoire_des_fichiers_utiliser_mode_test_ELTPARAGRAPHE000059"></a>Cette option permet de définir le répertoire des fichiers de données à utiliser en mode test : débogage, Go de fenêtre, de page ou de projet. 

Par défaut, le répertoire utilisé est le répertoire de la configuration courante. Il est possible par exemple de spécifier un répertoire spécifique. 

Cette option est intéressante si WINDEV est utilisé sous Windows Vista, avec l'UAC active : il est possible d'utiliser en mode test le répertoire des données de l'application.
<a name="NOTE2_2"></a>


### Automatismes
<a name="automatismes_ELTPARAGRAPHE000070"></a>Les options suivantes sont disponibles sur tous les types de données : 

- **Activer l'assistance automatique HFSQL**
	Cette option permet d'activer la gestion automatique des erreurs par le moteur HFSQL. Pour plus de détails, consultez [Gestion assistée des erreurs](../WDLang4/3044188.md).

- **Créer les fichiers de données automatiquement au moment de leur ouverture** : Cette option permet de créer automatiquement les fichiers de données décrits dans l'analyse si ces fichiers n'existent pas lors de l'utilisation de fonctions nécessitant leur ouverture. Dans ce cas, la fonction [HCréationSiInexistant](../WDLang4/3044254.md) est automatiquement exécutée.
	![Connecteurs Natifs (Accès Natifs)](https://doc.pcsoft.fr/ext/images/fr/AN.png) Cette option est prise en compte par les connecteurs natifs à SQL Server, Informix, Sybase, MySQL, MariaDB et PostgreSQL.



<a name="NOTE2_3"></a>


### Ré-indexation automatique (Uniquement pour HFSQL)
<a name="reindexation_automatique_uniquement_pour_hfsql_ELTPARAGRAPHE000102"></a>La ré-indexation d'un fichier peut être rendue nécessaire par la perte ou l'altération du fichier d'index. Une ré-indexation automatique des fichiers de données peut être mise en place.

L'option "Activer la réindexation automatique" permet d'activer la réindexation automatique des fichiers de données en spécifiant plusieurs paramètres : 

- le nombre d'enregistrements maximum : Si le fichier de données contient moins de XXX enregistrements, la réindexation est automatique en cas d'erreur d'index. Par défaut, cette valeur vaut 100000. Si cette variable vaut 0, la ré-indexation automatique est désactivée.

- l'affichage de la fenêtre de jauge permettant de suivre la ré-indexation en cours.




Remarque : la ré-indexation automatique est disponible uniquement sur les fichiers HFSQL Classic et HFSQL Client/Serveur.
<a name="NOTE2_4"></a>


### Mode de gestion des fichiers (Uniquement pour HFSQL)
<a name="mode_gestion_des_fichiers_uniquement_pour_hfsql_ELTPARAGRAPHE000120"></a>Cette option permet de définir le mode de gestion des fichiers de données de l'application. Par défaut, les applications créées avec WINDEV et WEBDEV sont des applications multi-utilisateurs : les blocages sont gérés de manière automatique.
Il est possible de demander une gestion "Mono-utilisateur : dans ce cas, les fichiers de données sont totalement bloqués en lecture et en écriture dès leur ouverture. 
<a name="NOTE2_5"></a>


### Options pour les Connecteurs Natifs
<a name="options_pour_les_connecteurs_natifs_ELTPARAGRAPHE000131"></a>Si l'option "Copier les connexions pendant la copie de contexte" est cochée, en utilisant un connecteur natif, lors de la copie de contexte, chaque contexte bénéficie en option de sa propre connexion au serveur de la base de données tierce. Cette option est nécessaire notamment dans le cas des transactions. 
 Cette nouveauté est disponible pour les bases : MS SQLServer, Oracle, MySQL, Sybase, Progress, Informix, DB2, SQLite, MariaDB.




## Renommage ou changement de machine
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000138"></a>
Lors d'un développement, il peut être nécessaire de changer le poste utilisé pour le développement (changement de machine, renommage du poste, ...).

Le GDS doit être informé de cette modification : en effet, le GDS mémorise dans la base de sources, le nom de la machine depuis laquelle les différentes opérations sont effectuées. En cas de renommage, cette information doit être mise à jour dans la base de sources.

**Attention** : Si vous changez de poste sans effectuer ces opérations, un message apparaîtra indiquant que l'élément est extrait sur un autre poste (l'ancien poste utilisé).

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000162"></a>


### Prendre en compte le renommage de machine
<a name="prendre_compte_renommage_machine_ELTPARAGRAPHE000022"></a>Pour prendre en compte le renommage de machine :

1. Lancez l'administrateur du GDS : sous le volet "GDS", dans le groupe "Base GDS", déroulez "Gérer" et sélectionnez "Administrateur du Gestionnaire de Sources".

2. Dans l'administrateur du GDS, sous le volet "Outils", dans le groupe "Mobilité", cliquez sur "Renommage ou changement de machine".

3. Indiquez le nouveau nom de la machine.

4. Le bouton "Tester" permet de connaître les conséquences du changement de nom.

5. Validez. Le renommage est effectué. Vous pouvez ré-utiliser vos projets présents dans le GDS.



<a name="NOTE2_2"></a>


### Prendre en compte le changement de machine
<a name="prendre_compte_changement_machine_ELTPARAGRAPHE000059"></a>Pour prendre en compte le changement de machine :

1. Lors du changement de poste, recopiez sur le nouveau poste dans des répertoires identiques, les différents projets utilisés par le GDS. Par exemple si sur le poste 1, les projets partagés dans le GDS sont dans "D:\\MesProjetsGDS", il est nécessaire de recopier ces projets sur le poste 2 dans le répertoire "D:\\MesProjetsGDS".

2. Sur le nouveau poste, lancez l'administrateur du GDS : sous le volet "GDS", dans le groupe "Base GDS", déroulez "Gérer"  et sélectionnez "Administrateur du Gestionnaire de Sources".

3. Dans l'administrateur du GDS, sous le volet "Outils", dans le groupe "Mobilité", cliquez sur "Renommage ou changement de machine".

4. Indiquez le nouveau nom de la machine.

5. Validez. Le changement de poste est pris en compte. Les projets présents dans le GDS peuvent être à nouveau ouverts.

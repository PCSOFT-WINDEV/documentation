


## Analyse d'impact
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000133"></a>
L'analyse d'impact permet de connaître automatiquement les conséquences des modifications en cours. En effet, une fenêtre d'informations s'affiche automatiquement pour vous présenter les ambiguïtés possibles, les modifications à éviter, ...

Les modifications prises en compte sont par exemple :

- le renommage ou la suppression d'un élément manipulé dans le projet,

- le nommage de deux éléments différents avec le même nom,

- l'ajout ou la suppression d'un paramètre d'une procédure,

- ...




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000157"></a>


### Lancer l'analyse d'impact
<a name="lancer_analyse_impact_ELTPARAGRAPHE000025"></a>L'analyse d'impact se lance automatiquement dès qu'une modification pouvant créer des ambiguïtés est réalisée.

Par exemple :

- création d'un champ portant le même nom qu'un autre élément du projet : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=AnalyseImpactNom.GIF)


- suppression d'une procédure appelée dans le projet : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=AnalyseImpactProc.GIF)




<a name="NOTE2_2"></a>


### Connaître les utilisations d'un élément dans le projet
<a name="connaitre_les_utilisations_element_dans_projet_ELTPARAGRAPHE000039"></a>Pour connaître les utilisations d'un élément dans le projet :

1. Sous le volet "Projet", dans le groupe "Audit et performances", déroulez "Audit statique" et sélectionnez "Références croisées".

2. Saisissez le nom de l'élément pour lequel vous voulez connaître les utilisations dans le projet.

3. Cliquez sur "Chercher". La liste des utilisations de l'élément sélectionné s'affiche :![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=AnalyseImpactRech.GIF)


4. Double-cliquez sur une des utilisations de l'élément pour la visualiser.




**Remarque** : Cette fenêtre peut rester ouverte pendant vos modifications dans le projet.
<a name="NOTE2_3"></a>


### Reporter un renommage sur tous les éléments de votre projet
<a name="reporter_renommage_sur_tous_les_elements_votre_projet_ELTPARAGRAPHE000066"></a>Pour reporter un renommage sur tous les éléments de votre projet :

1. Renommer l'élément voulu.

2. A la validation du nouveau nom :

	- les erreurs correspondantes sont affichées dans le volet "Erreurs de compilation".

	- un message apparaît en haut à droite de l'environnement en cours.![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Refactoring.gif)








Ce message propose de réaliser automatiquement le renommage sur tous les éléments nécessaires.

Lors du clic sur ce message, le renommage est automatiquement effectué.

**Remarques** :

- Pour visualiser directement toutes les références qui vont être modifiées, sélectionnez l'option "Analyse d'impact du renommage" du menu contextuel de l'erreur :![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=AnalyseImpact.GIF)


- Il est également possible de lancer le renommage à la validation du nouveau nom, en affichant le menu contextuel de l'erreur :![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Refactoring_Menu.gif)

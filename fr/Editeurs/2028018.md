


## WDInst : Paramétrer la modification automatique des fichiers de données 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000197"></a>
La modification automatique des fichiers de données permet de mettre à jour la description des fichiers de données présents sur les postes utilisateurs.

Pour plus de détails, consultez la [Modification automatique des fichiers de données](../WDLang4/3044195.md).

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Paramétrer la modification automatique des fichiers de données sous WDInst
<a name="parametrer_modification_automatique_des_fichiers_donnees_sous_wdinst_ELTTEXTE000221"></a>
Le paramétrage de la modification automatique des fichiers de données doit être effectué :

- dans les options de l'installation (option "Paramètres de l'installation .. Fichiers HFSQL").

- dans la description de chaque fichier ".EXE" utilisant des fichiers de données à modifier (fenêtre "Description du fichier"),




Pour connaître les différents éléments de l'interface de WDInst, consultez l'[interface de WDInst](../Editeurs/2028022.md)

Les paragraphes ci-dessous détaillent la modification automatique dans les cas suivants :

- la modification automatique des fichiers de données au format HFSQL Classic.

- la modification automatique des fichiers de données au format Hyper File 5.5 ou la migration des fichiers de données Hyper File 5.5 vers HFSQL Classic.



<a name="NOTE2_2"></a>


### Effectuer la modification automatique des fichiers de données au format WINDEV
<a name="effectuer_modification_automatique_des_fichiers_donnees_format_windev_ELTPARAGRAPHE000042"></a>Pour effectuer la modification automatique des fichiers de données au format WINDEV :

1. Sélectionnez l'option "Paramètres de l'installation .. Fichiers HFSQL".

2. Cochez l'option "Exécuter la modification automatique des données". 

3. Spécifiez le fonctionnement de la modification automatique des données :

	- l'option "Modification automatique des données sans IHM" permet de réaliser la modification automatique des données sans que l'utilisateur final ne puisse intervenir. 
			Cette option est proposée par défaut lorsque le programme d'installation de l'application est configuré via l'assistant. 

	- l'option "Demander confirmation de l'exécution de la modification automatique des données" permet d'afficher la liste des fichiers de données modifiés et de demander la confirmation de l'utilisateur final. 

	- l'option "Afficher les options avancées de la modification automatique des données" permet d'afficher une fenêtre spécifique à l'utilisateur final (voir paragraphe [Mise à jour avec modification automatique des fichiers de données](#NOTE3_1)). 




4. Spécifiez si nécessaire le mot de passe associé à l'analyse. 

5. Validez la fenêtre des options de l'installation. 

6. Dans la fenêtre "Liste des fichiers de l'installation", sélectionnez le fichier exécutable (type ".EXE") sur lequel la modification automatique des fichiers de données doit être effectuée. Dans l'onglet "HFSQL", cochez l'option "Gérer la modification automatique des données". 

7. Cliquez sur le bouton "Avancé".

8. Sélectionnez si nécessaire l'analyse au format WINDEV (fichier ".WDD") associée à l'application.

9. Pour chaque fichier de donnée de l'analyse, spécifiez si nécessaire le mot de passe du fichier de données (s'il existe). 
	Pour modifier ce mot de passe, cochez l'option "Modifier le mot de passe" et saisissez deux fois le nouveau mot de passe.


**Remarque** : Si la modification automatique doit être effectuée sur des fichiers de données utilisés par plusieurs fichiers ".EXE", répétez les opérations pour tous les fichiers ".EXE" concernés.
<a name="NOTE2_3"></a>


### Effectuer la modification automatique des fichiers de données au format Hyper File 5.5 ou migrer des fichiers de données Hyper File 5.5 vers HFSQL Classic
<a name="effectuer_modification_automatique_des_fichiers_donnees_format_hyper_file_55_migrer_des_fichiers_donnees_hyper_file_55_vers_hfsql_classic_ELTPARAGRAPHE000080"></a>Pour effectuer la modification automatique des fichiers de données au format Hyper File 5.5 ou pour migrer des fichiers de données Hyper File 5.5 vers HFSQL Classic, les manipulations concernant les options de l'installation sont identiques. 

Dans la fenêtre "Liste des fichiers de l'installation" : 

1. Sélectionnez le fichier exécutable (type ".EXE") sur lequel la modification automatique des fichiers de données doit être effectuée.

2. Dans l'onglet "HFSQL" : 

	- Cochez l'option "Gérer la modification automatique des données".

	- Cochez l'option "Gérer l'existence de fichiers au format Hyper File 5.5".

	- Cliquez sur le bouton "Avancé".




3. Sélectionnez l'analyse au format HFSQL Classic (fichier ".WDD") associée à l'application.

4. Pour chaque fichier de données de l'analyse, spécifiez si nécessaire :

	- le mot de passe du fichier de données (s'il existe) dans l'analyse au format WINDEV.
			Remarque : Pour modifier ce mot de passe, cochez l'option "Modifier le mot de passe" et saisissez deux fois le nouveau mot de passe.

	- si l'application utilise une analyse WINDEV 5.5 migrée :

		- l'analyse d'origine au format WINDEV 5.5 (fichier ".WDA").

		- le nom du fichier de données dans l'analyse au format WINDEV 5.5.

		- le mot de passe du fichier de données (s'il existe) dans l'analyse au format WINDEV 5.5.







**Remarque** : Si la modification automatique doit être effectuée sur des fichiers de données utilisés par plusieurs fichiers ".EXE", répétez les opérations pour tous les fichiers ".EXE" concernés.

## Mise à jour d'une application avec modification automatique des fichiers de données
<a name="mise_jour_une_application_avec_modification_automatique_des_fichiers_donnees_ELTTEXTE000263"></a>
Lors de l'installation d'une mise à jour avec modification automatique des fichiers de données, il est possible de permettre à l'utilisateur de configurer la modification automatique : un bouton "Options avancées" est alors affiché sur le premier plan du programme d'installation. La fenêtre suivante s'affiche alors : <br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Param%E9trer_modif_auto_des_fichiers%20-%20HC%20N%B0001.gif&type=thumb)
Cette fenêtre permet de :

- créer un fichier de compte-rendu des opérations effectuées lors de la modification automatique des données. En cas de problème, ce fichier pourra être transmis au responsable de l'application. 
	Par défaut, ce fichier est nommé "LOGMODAUTO.TXT" et est créé dans le répertoire d'installation de l'application.

- lister les fichiers de données au format HFSQL Classic sur lesquels la modification automatique va être effectuée. Il sera alors possible d'ajouter à cette liste :

	- des fichiers de données supplémentaires.

	- des répertoires supplémentaires contenant des fichiers de données.




- spécifier des répertoires supplémentaires contenant des fichiers de données au format Hyper File 5.5 qui ne sont pas trouvés automatiquement lors de la mise à jour.




**Remarque** : Lors de la mise à jour, les fichiers de données automatiquement trouvés sont :

- les fichiers présents dans le répertoire d'installation de l'application.

- les fichiers listés dans le fichier ".REP". Pour plus de détails sur le fichier ".REP", consultez l'aide sur la [gestion du .REP](../WDLang4/3044186.md).




Pour empêcher l'utilisateur d'accéder aux options de configuration de la modification automatique, l'éditeur d'installation WDInst permet de rendre invisible le bouton "Avancé" du programme d'installation.
<a name="NOTE3_2"></a>


### Rendre le bouton "Avancé" invisible
<a name="rendre_bouton_avance_invisible_ELTPARAGRAPHE000141"></a>Pour rendre le bouton "Avancé" invisible :

1. Sélectionnez l'option "Paramètres de l'installation .. Fichiers HFSQL".

2. Cochez l'option "Modification automatique des données sans IHM".

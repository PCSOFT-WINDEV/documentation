


## WDInst : manipuler les fichiers à installer 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000107"></a>
Par défaut, les fichiers à installer sont :

- récupérés dans le répertoire "EXE" du projet.

- compactés avant d'être copiés sur le support d'installation.

- installés dans le répertoire d'installation de l'application.




L'éditeur d'installation WDInst permet de définir les fichiers à installer. 

Pour lancer l'éditeur d'installation WDInst depuis WINDEV, sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDInst".

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Quelles sont les opérations possibles ?
<a name="quelles_sont_les_operations_possibles_ELTTEXTE000131"></a>
L'éditeur d'installation WDInst permet de manipuler les fichiers à installer. Il est possible de :

- [gérer les fichiers à installer](../Editeurs/2028066.md) :

	- visualiser la liste des fichiers.

	- ajouter un ou plusieurs fichiers.

	- supprimer un ou plusieurs fichiers.




- [paramétrer les caractéristiques des fichiers à installer](../Editeurs/2028033.md) :

	- spécifier le répertoire de destination d'un ou de plusieurs fichiers.

	- spécifier les postes sur lesquels chacun des fichiers doit être installé (cas d'une installation réseau).

	- copier et compacter un ou plusieurs fichiers sur le support d'installation.




- [paramétrer les caractéristiques de l'installation des fichiers](../Editeurs/2028004.md) :

	- créer un raccourci.

	- gérer les collisions.

	- sauvegarder les fichiers remplacés.

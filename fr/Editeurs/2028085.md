
## Mode d'installation
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000138"></a>
Une application WINDEV peut être diffusée de plusieurs façon :

- directement à un utilisateur. L'utilisateur installe l'application sur son poste. Cette application peut être une application monoposte, réseau, ... L'installation est dite autonome.

- installé sur un réseau ou un serveur HTTP. L'utilisateur installe son logiciel depuis le réseau ou une page internet. La mise à jour sera automatiquement proposée à l'utilisateur. Cette application peut être monoposte, réseau, ...




Pour plus de détails, consultez le [schéma d'installation d'une application](../Editeurs/2028008.md). 



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Installation individuelle
<a name="installation_individuelle_ELTTEXTE000162"></a>


### Options disponibles
<a name="options_disponibles_ELTPARAGRAPHE000027"></a>Si vous choisissez une installation individuelle, vous avez le choix entre :

- **Installation standard**. Le programme d'installation utilisé est celui livré avec WINDEV ou une version personnalisée (l'exemple WDSetup correspond aux sources de l'installeur et peut être modifié).

- **Installation compacte**. La taille du programme d'installation généré est optimisée pour un téléchargement par Internet.

- [Installation par un package MSI](../Editeurs/9000025.md). Le programme d'installation utilise l'installeur standard de Windows.

## Installation avec mise à jour automatique
<a name="installation_avec_mise_jour_automatique_ELTTEXTE000186"></a>


### Options disponibles
<a name="options_disponibles_ELTPARAGRAPHE000043"></a>Si vous choisissez une installation avec mise à jour automatique, vous pouvez choisir :

- Installation en réseau local (LAN), avec prise en charge possible du "Push". Une première installation de l'application sera réalisée sur le réseau, dans un répertoire partagé. Ce répertoire contiendra le programme d'installation utilisé par les utilisateurs finaux et la version de référence permettant de gérer les mises à jour.

- [Installation par HTTP](../Editeurs/9000026.md). Une première installation de l'application sera réalisée sur un serveur HTTP. Les utilisateurs pourront se connecter à une page Web pour télécharger le programme d'installation de l'application.

- [Installation multisite](../Editeurs/2028100.md). Cette installation est la combinaison d'une installation réseau local et d'une installation HTTP. L'installation HTTP est utilisée pour la mise à jour de l'installation de référence sur le réseau local. 





<a name="NOTE3_2"></a>


### Contrôle de version
<a name="controle_version_ELTPARAGRAPHE000061"></a>Le contrôle de version consiste à :

- sélectionner l'exécutable de votre application servant de référence pour les mises à jour des applications déjà déployées.

- sélectionner la version minimale autorisée à fonctionner sans mise à jour. Si vous voulez forcer la mise à jour de tous les utilisateurs lors du prochain lancement de leur application, il suffit de recopier la version en cours dans la version minimale.




Pour plus de détails, consultez [Contrôle des versions](../Editeurs/2028035.md).


<a name="NOTE3_3"></a>


### Utilisateurs connectés
<a name="utilisateurs_connectes_ELTPARAGRAPHE000077"></a>La gestion des utilisateurs permet de connaître à partir du poste serveur :

- les utilisateurs connectés actuellement à l'application,

- pour chaque utilisateur connecté :

	- le poste de lancement de l'application,

	- la version de l'application utilisée.







Pour plus de détails, consultez [Gérer les utilisateurs d'une application avec mise à jour réseau](../Editeurs/2028059.md).

## Configuration du serveur pour les sites WEBDEV
<a name="configuration_serveur_pour_les_sites_webdev_ELTTEXTE000119"></a>
Avec Domino, il est nécessaire de créer un serveur virtuel par site dynamique WEBDEV hébergé. 

Pour configurer le serveur :

1. Lancez "Domino Administrator".

2. Connectez-vous à un serveur Web. Pour cela, dans l'onglet "Configuration du serveur", sélectionnez l'option "WEB", puis le serveur Web à paramétrer.

3. Double-cliquez sur le premier serveur (Domino Server).

4. Pour créer un serveur virtuel :

	- Dans le menu "Create WEB (R5)", sélectionnez l'option "Virtual Server".

	- Dans la boîte de dialogue qui s'affiche, sélectionnez l'option "Virtual Host".




5. Pour paramétrer ce serveur virtuel :

	- Double-cliquez dessus.

	- Les paramètres à définir sont répartis entre les onglets "Basics" et "Mapping".




6. Les paramètres à configurer sont les suivants :





|   |   |
| --- | --- |
| HostName | Adresse du site Web (adresse DNS ou adresse IP) |
| Default Home Page | Nom de la page d'accueil du site (par défaut Accueil.htm) |
| Home URL | URL à exécuter si on saisit simplement l'adresse du site (par exemple nom de la page d'accueil) |
| HTML directory | Chemin complet du répertoire _WEB du site |
| Icon directory | Chemin complet du répertoire _WEB du site |
| Icon URL Path | Nom du répertoire virtuel du site (/&lt;Nom site&gt;_WEB) |
| CGI directory | Chemin du répertoire AWP (présent dans le sous-répertoire Programmes du répertoire d'installation de WEBDEV). |
| CGI Url path | /WD280AWP |

### Gestion des pages AWP (Active Web Pages)
<a name="gestion_des_pages_awp_active_web_pages_ELTPARAGRAPHE000069"></a>Sur les serveurs Domino, il n'est pas possible d'utiliser des sites WEBDEV utilisant des pages WEBDEV au format AWP.

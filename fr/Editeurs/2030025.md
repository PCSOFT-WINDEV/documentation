
## Créer un exécutable sans bibliothèque 
			






	<B><B>Avertissement</B></B>

	A partir de la version <B><B>09 (34)</B></B>,  le fonctionnement décrit dans cette page d'aide est conservé par compatibilité avec la version 5.5. 
La création d'un exécutable sans bibliothèque est fortement déconseillée. 





<a name="NOTE1"></a>
<a name="NOTE1_1"></a>




## Présentation
<a name="presentation_ELTTEXTE000115"></a>
Lors de la création d'un exécutable, vous pouvez choisir de ne pas utiliser de bibliothèque ou de ne pas intégrer tous les éléments du projet dans la bibliothèque.

Cette option peut être utilisée par exemple :

- pour des projets présentant la maquette d'une application,

- pour des applications internes, nécessitant des mises à jour rapides. 




Dans ce cas, il est nécessaire de fournir dans le répertoire de l'exécutable les fichiers correspondant aux différents éléments du projet, ainsi que les fichiers compilés associés.

**Remarque** : Pour les fenêtres et les états, il est nécessaire de créer des fenêtres et des états autonomes (c'est-à-dire d'intégrer le code compilé dans le fichier de la fenêtre ou de l'état).



<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Fenêtres et états autonomes
<a name="fenetres_etats_autonomes_ELTTEXTE000139"></a>


### Transformer la fenêtre en cours en une fenêtre autonome
<a name="transformer_fenetre_cours_une_fenetre_autonome_ELTPARAGRAPHE000036"></a>Pour transformer la fenêtre en cours en une fenêtre autonome, dans l'onglet "Détail" de la description de la fenêtre, cochez l'option "Intégrer le code compilé".

Lors de l'enregistrement de la fenêtre, le code compilé sera automatiquement intégré au fichier WDW de la fenêtre.


<a name="NOTE3_2"></a>


### Transformer l'état en cours en un état autonome
<a name="transformer_etat_cours_etat_autonome_ELTPARAGRAPHE000046"></a>Pour transformer l'état en cours en un état autonome, dans l'onglet "Détail" de la description de l'état, cochez l'option "Intégrer le code compilé".

Lors de l'enregistrement de l'état, le code compilé sera automatiquement intégré au fichier WDE de l'état.

**Remarque** : Si cet état est modifié, il est nécessaire de recréer l'exécutable pour prendre en compte ces modifications.



<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Éléments à installer pour un exécutable sans bibliothèque
<a name="elements_installer_pour_executable_sans_bibliotheque_ELTTEXTE000169"></a>
Si votre exécutable ne gère pas de bibliothèque, il est nécessaire d'installer dans le répertoire de l'exécutable :

1. **Pour les fenêtres (fichiers WDW) et les états (fichiers WDE)** :

	- si des fenêtres et des états autonomes ont été créés, copiez-les dans le répertoire de l'exécutable.

	- sinon, copiez dans le répertoire de l'exécutable les fichiers WDW et WDE correspondants ainsi que les fichiers WCW et WCE (présents dans le sous-répertoire CPL de votre projet).




2. **Pour les classes (fichier WDC), les collections de procédures (fichier WDG) et le projet (fichier WDP)** : copiez dans le répertoire de l'exécutable les fichiers WDC, WDG et WDP correspondants ainsi que les fichiers WCC, WCG, et WCP (présents dans le sous-répertoire CPL de votre projet).

3. **Pour les requêtes, copiez directement le fichier correspondant (fichier WDR)** dans le répertoire de l'exécutable.

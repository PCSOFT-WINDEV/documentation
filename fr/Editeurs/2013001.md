


## Complétion automatique 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000200"></a>
Cette complétion est disponible sur les ordres du WLangage, les procédures, les variables, les fichiers de données, les rubriques de fichier de données, les fenêtres, les champs, le type des variables, ...

Cette complétion élimine le risque d'erreurs de frappe et fait gagner énormément de temps : inutile de rechercher le nom exact d'une variable globale ou d'un champ, il est automatiquement proposé !

La complétion automatique permet de :

- ***compléter la saisie en cours*** en proposant une liste intelligente des différentes valeurs (nom de fonction, nom de propriété, ...) possibles. Toutes les valeurs proposées dans la liste contiennent le mot actuellement saisi. 
	A chaque modification saisie, le contenu de la liste est mis à jour.
	Par exemple, lors de la saisie des caractères "OKA", l'ensemble des fonctions, champs, constantes, briques de code, ... contenant ces caractères est proposé.![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=completion.gif)
 **Remarque** : La complétion automatique est également proposée lorsque vous modifiez le nom d'une fonction déjà saisie sous l'éditeur de code. 
	Exemple : vous voulez remplacer "**HLitPremier**" par la fonction "**HLitRecherchePremier**" : lors de la saisie du mot "Recherche" entre "Lit" et "Premier", la complétion propose automatiquement "HLitRecherchePremier". Lors de la sélection de la proposition, l'ancien terme est remplacé par le nouveau sélectionné.  

- ***proposer uniquement la valeur des paramètres possibles*** selon la fonction du WLangage utilisée.
	Par exemple, lors de l'utilisation de la fonction [HLit](../WDLang4/3044047.md), le premier paramètre attendu correspond au nom d'un fichier de données. La complétion automatique proposera uniquement les fichiers de données de l'analyse associée au projet en cours. ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=completion2.gif)


- ***sélectionner les combinaisons de constantes pour un paramètre***. 
	Il suffit de sélectionner les options voulues (avec les touches fléchées) et de valider les options choisies avec la touche Espace. 
	L'ensemble des options sélectionnées est automatiquement intégrée dans le code lors de la validation.
	Remarque : Les constantes sont séparées en différentes zones colorées permettant d'identifier les constantes combinables ou non entre elles (deux constantes présentes dans une zone colorée avec un sélecteur ne peuvent pas être sélectionnées simultanément). ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Completion_cst.gif)


- ***lancer les assistants de code***. Ces assistants permettent de saisir les différents paramètres de la fonction à l'aide de questions simples. 
	L'option &lt;Assistant de la fonction xxx&gt; signale la présence d'un assistant. Pour lancer l'assistant, sélectionnez cette option.![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=AssistantCode.gif)


- ***proposer les différents attributs d'extension disponibles*** sur la ligne de code en cours.
	Un attribut d'extension correspond à une directive passée au compilateur. Ces attributs sont puissants et pratiques. Pour obtenir la liste des attributs d'extension disponibles sur l'élément en cours, il suffit de taper le caractère "&lt;".  ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Completion_Attributs%20-%20HC%20N%B0001.gif)

	Pour plus de détails sur les attributs d'extension disponibles, consultez [Liste des attributs d'extension](../Motscles/1511025.md). 




**Remarques** : 

- Pour utiliser une des propositions, il suffit de : 

	1. Se déplacer dans la liste avec les flèches du clavier. 

	2. Valider la sélection : 

		- avec la touche Entrée.

		- avec la touche Tab.




- Les différentes listes déroulantes affichées sont redimensionnables.

- **Si votre procédure ou fonction utilise un élément du projet (fenêtre, page, état, ...)**, il est possible d'obtenir la liste des éléments voulus dans la complétion. Pour cela, il suffit d'ajouter un attribut d'extension spécifique dans la déclaration des paramètres de la fonction ou de la procédure. Pour plus de détails, consultez [Paramètres d'une fonction ou procédure](../Motscles/1513001.md). 

- **Si vous utilisez une variable de type Source de données**, il suffit d'ajouter un attribut d'extension spécifique pour l'associer à un fichier de données (ou une requête) existant. Dans ce cas, la complétion automatique permettra d'accéder directement aux rubriques. Pour plus de détails, consultez [Variable de type Source de données](../WDLang4/1514053.md). 

- L'éditeur de code propose également une aide à la saisie (aide des paramètres, etc.). Pour plus de détails, consultez [Editeur de code : Aide à la saisie](../Editeurs/2013009.md).

## Comment le faire ?
<a name="comment_faire_ELTTEXTE000224"></a>


### Activer la complétion automatique
<a name="activer_completion_automatique_ELTPARAGRAPHE000093"></a>Pour activer la complétion automatique dans votre projet :

1. Affichez les options de l'éditeur de code : sous le volet "Accueil", dans le groupe "Environnement", déroulez "Options" et sélectionnez "Options de l'éditeur de code".

2. Cochez l'option "Activer l'aide à la saisie après une inactivité de" et spécifiez la durée après laquelle la complétion automatique doit être proposée.




**Remarque** : Pour désactiver la complétion automatique, décochez cette option.

## Remarques
<a name="remarques_ELTTEXTE000248"></a>


### Langue du code
<a name="langue_code_ELTPARAGRAPHE000123"></a>La liste des mots-clés du WLangage proposée par la complétion automatique peut au choix être en français ou en anglais.

Pour modifier la langue de code utilisée par la complétion automatique :

1. Affichez les options de l'éditeur de code : sous le volet "Accueil", dans le groupe "Environnement", déroulez "Options" et sélectionnez "Options de l'éditeur de code".

2. Sélectionnez la langue du code à afficher dans la combo "Langue du code ".



<a name="NOTE3_2"></a>


### Complétion automatique sur le type des variables
<a name="completion_automatique_sur_type_des_variables_ELTPARAGRAPHE000151"></a>Lors de la déclaration d'une nouvelle variable, les différents types sont automatiquement proposés. Il suffit d'écrire : "MaVariable est un ".


## Afficher le code d'un élément
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000137"></a>
WINDEV, WEBDEV et WINDEV Mobile permettent d'utiliser une programmation événementielle et une programmation orientée objet. 

La programmation événementielle proposée associe un ou plusieurs événements à chaque élément utilisé. Ces événements sont affichés dans l'éditeur de code. 

Pour de nombreux éléments : 

- Certains événements sont proposés par défaut : ce sont les événements les plus courants et les plus utilisés. 

- Certains événements peuvent être ajoutés. Ces événements permettent de gérer des cas spécifiques (par exemple, la gestion du Drag and Drop dans certains cas particuliers). Pour plus de détails, consultez [Evénements optionnels](../WDChamp/1014004.md).

<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000161"></a>


### Comment afficher les événements associés à un élément ? 
<a name="comment_afficher_les_evenements_associes_element_ELTPARAGRAPHE000026"></a>Pour afficher les événements associés à un élément (champ, fenêtre, page, etc.), plusieurs méthodes sont disponibles : 

- **Utiliser la touche F2**. Il suffit pour cela de : 

	- Sélectionner l'élément voulu. 

	- Utiliser la touche F2 : l'éditeur de code est automatiquement affiché avec la liste des événements associés par défaut. 




- **Utiliser l'option "Code" du menu contextuel**. Il suffit pour cela de : 

	- Sélectionner l'élément voulu. 

	- Afficher le menu contextuel (clic droit) et de sélectionner l'option "Code". L'éditeur de code est automatiquement affiché avec la liste des événements associés par défaut. 







Remarques : 

- Il est également possible d'afficher les événements associés à un élément depuis la fenêtre de description de l'élément : il suffit de cliquer sur le bouton ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Detail_Description_FEN_btnCode%20-%20HC%20N%B0001%201.gif)
 placé en bas de la fenêtre de description. 

- Pour afficher les événements associés au projet, à une fenêtre, à une page ou un état, il est également possible d'utiliser la barre des éléments ouverts. Il suffit de : 

	- Afficher le menu contextuel du bouton correspondant à l'élément voulu. 

	- Sélectionner l'option "Code de l'élément" dans le menu qui s'affiche. L'éditeur de code est automatiquement affiché avec la liste des événements associés par défaut.






<a name="NOTE2_3"></a>


### Cas particulier : afficher les événements associés à un élément depuis l'éditeur de code
<a name="cas_particulier_afficher_les_evenements_associes_element_depuis_editeur_code_ELTPARAGRAPHE000058"></a>Le code WLangage placé dans les différents traitements ou événements peut manipuler des éléments spécifiques (champs, fenêtres, pages, etc.). 

Il est possible d'afficher directement les événements d'un élément manipulé dans une ligne de code WLangage en cliquant sur le nom de l'élément et en utilisant la touche F2. 

Par exemple : Considérons le code suivant : 

```wl
Ouvre(FEN_Menu)
```
Si la touche F2 est utilisée sur le mot "FEN_Menu", l'éditeur de code affichera les événements associés à l'élément "FEN_Menu".
<a name="NOTE2_4"></a>


### Cas particulier : F2 sur le nom d'un plan
<a name="cas_particulier_sur_nom_plan_ELTPARAGRAPHE000074"></a>Si le code WLangage manipule un plan nommé, l'utilisation de la touche F2 sur le nom du plan dans l'éditeur de code permet d'ouvrir la fenêtre si nécessaire dans l'éditeur de fenêtre et de se positionner sur le plan spécifié. 
<a name="NOTE2_5"></a>


### Astuce : Revenir à la dernière modification de code
<a name="astuce_revenir_derniere_modification_code_ELTPARAGRAPHE000083"></a>Il arrive souvent qu'après avoir tapé un code, on navigue dans le code pour vérifier certains éléments. Et ensuite, il faut revenir au code initial pour continuer ses tâches.

Pour revenir au code initial, utilisez la combinaison de touches Ctrl + Alt + Flèche gauche.

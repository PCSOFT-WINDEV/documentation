
## Site mobile friendly : les méthodes disponibles
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000181"></a>
Pour faire des sites "mobile friendly", WEBDEV propose plusieurs méthodes de création de pages : 

- Le **Dynamic serving** qui consiste à maintenir un jeu de pages pour le mode "Desktop" et un jeu de pages pour le mode "Mobile". 

- Le **Responsive historique**, qui permet de découper les pages en tranches pour s'adapter aux mobiles. 

- Le **Responsive via zoning et agencements** (mode par défaut à partir de WEBDEV 28) qui permet de définir des agencements spécifiques à l'écran d'affichage. Ce mode est désormais conseillé. Pour plus de détails, consultez [Changement de comportement en version 28](#NOTE2_2).   




Cette page d'aide présente les différentes méthodes, ainsi que leurs avantages et leurs inconvénients.

### Changement du mode d'édition des sites mobile friendly en version 28
<a name="changement_mode_edition_des_sites_mobile_friendly_version_28_ELTPARAGRAPHE000095"></a>En version 28, le mode d'édition des pages initialement appelé "Responsive Web Design : mise en page par tranche" devient "Tranches (Responsive historique pour compatibilité)".
Le mode "Responsive (zoning et agencements)" devient le mode recommandé pour les nouvelles créations.

En effet les observations ont montré que les agencements sont le plus couramment retenus pour la création des sites qui doivent être "mobile friendly". Ainsi, les évolutions de l'éditeur de pages ont été concentrées sur l'édition par zones (zoning), avec l'enrichissement des possibilités des agencements :

- les agencements permettent de modifier, masquer les zones (zoning),

- les modèles de champs supportent les agencements,

- les zones de texte riche peuvent avoir un contenu adapté à l'agencement,

- le nouveau champ Flexbox permet de définir des comportements pour le positionnement et le déplacement des champs lorsqu'une page est redimensionnée dans le navigateur,

- un nouvel agencement est disponible pour les pages à intégrer à une application WINDEV ou WINDEV Mobile.




Ces nouveautés des agencements en version 28 viennent compléter les avantages existants des agencements :

- simplicité d'édition,

- même possibilité que dans les applications WINDEV et WINDEV Mobile,

- choix de l'agencement possible par programmation,

- ancrage modifiable par agencement,

- agencement dédié pour les impressions...




**Pour toutes ces possibilités, le mode "Responsive (zoning et agencements)" est donc devenu à partir de WEBDEV 28 le mode recommandé**. Il est possible de convertir une page existante dans le mode "Responsive Web Design : mise en page par tranche", en page "Responsive (zoning et agencements)" : sous le volet "Page", dans le groupe "Agencements", déroulez "Agencements" et sélectionnez l'option "Convertir les tranches en agencements".

Ce mode est destiné à recevoir toutes les évolutions à venir de l'éditeur de pages de WEBDEV.

L'édition par tranches reste conservée pour les sites existants, et les nouveaux sites qui ne peuvent pas bénéficier des agencements. Mais elle ne doit plus recevoir d'évolutions majeures dans les prochaines versions de WEBDEV.

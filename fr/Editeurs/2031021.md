## Création du projet Pattern RAD
<a name="creation_projet_pattern_rad_ELTTEXTE000153"></a>
La création d'un pattern RAD est réalisée à partir d'un projet spécifique, de type "Pattern RAD".
<a name="NOTE2_2"></a>
Pour créer un projet "Pattern RAD" :

1. Cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_nouveau.gif)
 parmi les boutons d'accès rapide. La fenêtre de création d'un nouvel élément s'affiche : cliquez sur "Projet".

2. Dans l'assistant de création du projet, indiquez :

	- Le nom du projet qui servira de base pour créer le pattern.

	- Dans l'écran "Type de génération", sélectionnez l'option "Pattern RAD".

	- Les autres options de la création de projet sont les options classiques. Vous pouvez sélectionner une charte graphique pour votre projet.  




3. Validez l'assistant de création du projet.

4. Le projet est automatiquement créé et associé à une analyse. Cette analyse contient les fichiers nécessaires à la création d'un RAD adaptable à toutes les configurations de fichiers.




Remarque : WEBDEV intégré dans WINDEV : Une configuration de type "Pattern RAD" n'admet que des fenêtres, il n'est pas possible de générer un pattern pour WEBDEV.

## L'analyse du projet de type Pattern RAD
<a name="analyse_projet_type_pattern_rad_ELTTEXTE000183"></a>


### Analyse
<a name="analyse_ELTPARAGRAPHE000054"></a>Les projets de type "Pattern RAD" sont tous associés à une analyse identique. Cette analyse contient tous les types de liaisons et de groupes de fichiers.
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=RAD_Pattern.gif)
Cette analyse comporte :

- un fichier seul (par exemple RADFichierC).

- deux fichiers reliés par une liaison simple (par exemple RADFichierB lié à RADFichierC ou RADFichierA lié à RADFichierB).

- deux fichiers (et un fichier de relation) liés par une liaison complexe (RADFichierB lié à RADFichierC par le fichier de relation RADFichierD).




Toute analyse peut être décomposée en plusieurs ensembles de ces trois groupes de fichiers.
<a name="NOTE3_2"></a>


### Exemple d'application
<a name="exemple_application_ELTPARAGRAPHE000069"></a>Essayons de décomposer l'analyse suivante : ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=RAD_Analyse_Ex.gif)


Les groupes créés sont les suivants :

- Client.

- Commande.

- Produit.

- Fournisseur.

- Client lié à Commande.

- Commande lié à Produit par le fichier de liaison LigneCde.

- Produit lié à Fournisseur.

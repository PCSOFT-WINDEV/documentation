
## Différences selon le mode de test d'un projet dynamique
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000127"></a>
Le test d'un projet dynamique peut être effectué :

- soit depuis l'éditeur de WEBDEV. Pour plus de détails, consultez [Tester un projet dynamique](../Editeurs/2019009.md).

- soit depuis l'administrateur WEBDEV. Pour plus de détails, consultez [Tester un projet dynamique (administrateur)](../Editeurs/2019007.md)




Plusieurs différences peuvent être observées entre ces deux méthodes de test.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Les différences
<a name="les_differences_ELTTEXTE000151"></a>


### Gestion des cookies
<a name="gestion_des_cookies_ELTPARAGRAPHE000029"></a>La gestion des cookies est opérationnelle uniquement en réalisant un test depuis la page de test de l'administrateur. 



<a name="NOTE2_2"></a>


### Droits accordés sur le serveur
<a name="droits_accordes_sur_serveur_ELTPARAGRAPHE000038"></a>

- Lors du test depuis l'éditeur, les droits du site WEBDEV correspondent aux droits de l'utilisateur qui teste le site.

- Lors du test depuis la page de test, les droits utilisés correspondent à ceux de l'utilisateur configuré sur le serveur Web. Ces droits peuvent être plus restrictifs : certaines actions peuvent être interdites (par exemple la suppression de fichiers sur le serveur, ...). 
	



<a name="NOTE2_3"></a>


### Gestion des erreurs de programmation
<a name="gestion_des_erreurs_programmation_ELTPARAGRAPHE000047"></a>

- Lors du test depuis l'éditeur, les erreurs dues à une erreur de programmation sont affichées dans une boîte de dialogue et le débogueur est automatiquement lancé.

- Lors d'un test depuis la page de test de l'administrateur, les erreurs dues à une erreur de programmation sont affichées dans une page sous le navigateur. Cette page d'erreur contient le message "L'application a généré le message suivant : ...". 
	




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Notes
<a name="notes_ELTTEXTE000187"></a>


### Navigateur de test
<a name="navigateur_test_ELTPARAGRAPHE000058"></a>Le navigateur de test est le navigateur défini par défaut sur le poste en cours.

Il est conseillé de tester le projet avec plusieurs navigateurs différents.


<a name="NOTE3_2"></a>


### Lancement depuis l'administrateur
<a name="lancement_depuis_administrateur_ELTPARAGRAPHE000068"></a>L'utilisation du test du site depuis la page de test lancée par l'administrateur WEBDEV permet de tester en configuration réelle le site WEBDEV.


<a name="NOTE3_3"></a>


### Moteur de test et moteur de déploiement
<a name="moteur_test_moteur_deploiement_ELTPARAGRAPHE000076"></a>Le fonctionnement du moteur de test de WEBDEV (livré avec WEBDEV "Version de Développement") et le fonctionnement du moteur de WEBDEV installé chez l'hébergeur (livré avec WEBDEV "Version de Déploiement") peuvent être légèrement différents. Les différences de fonctionnement éventuelles sont mineures, elles proviennent principalement du débogage que permet le moteur de test.

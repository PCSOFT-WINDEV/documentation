## Présentation
<a name="presentation_ELTTEXTE000116"></a>
Le dictionnaire est un fichier pouvant contenir des éléments du projet : description de fichiers de données, de rubriques, fenêtres, etc. Le dictionnaire est un moyen simple de conserver des éléments créés en prévision d'une prochaine réutilisation dans de nouveaux projets.

Lors de cette réutilisation, chaque élément provenant d'un dictionnaire peut être "**abonné**" aux modifications. A chaque modification de l'objet du dictionnaire, tous les objets abonnés pourront intégrer cette modification, et ainsi "suivre" les évolutions du dictionnaire.

Un dictionnaire peut être associé à un utilisateur : c'est le **dictionnaire utilisateur**. Dans ce cas, le dictionnaire peut être affiché de façon permanente dans le volet "Dictionnaire". 

**Un dictionnaire peut aussi être utilisé pour simplifier le travail en groupe** : un dictionnaire partagé entre plusieurs développeurs peut ainsi contenir des procédures, des boutons prédéfinis. Ces objets sont mis en communs, et accessibles par tous les développeurs.

La gestion du groupware développeur sur le dictionnaire et ses éléments permet de restreindre l'accès au dictionnaire (ou à ses objets) pour certains développeurs.

## Principe
<a name="principe_ELTTEXTE000140"></a>
Le dictionnaire est un fichier contenant un ensemble d'éléments : procédures, fenêtres, champs, ... Un dictionnaire peut être manipulé depuis le volet "Dictionnaire".

Lors de la manipulation d'un dictionnaire, il est possible de :

- Créer un nouveau dictionnaire ou ouvrir un dictionnaire existant.

- Ajouter des éléments (fenêtres, états, ...) dans le dictionnaire.

- Copier des éléments depuis le dictionnaire.




Pour plus de détails, consultez [Manipuler un dictionnaire](../Editeurs/2023003.md).

Lorsqu'un élément est ajouté par "Drag and Drop" depuis le dictionnaire ou vers un dictionnaire, une demande d'abonnement est proposée. Si la demande d'abonnement est acceptée :

- A chaque ouverture de l'élément du projet, WINDEV, WEBDEV et WINDEV Mobile vérifient si cet élément est à jour par rapport à l'élément présent dans le dictionnaire et propose éventuellement sa mise à jour.

- A chaque fermeture d'un élément modifié, WINDEV, WEBDEV et WINDEV Mobile proposent de mettre à jour l'objet du dictionnaire.




Avec ce principe, l'abonnement facilite le report des modifications d'un objet abonné dans tous les autres objets abonnés. Pour plus de détails, consultez [Abonnement d'un élément au dictionnaire](../Editeurs/2023001.md).

## Compatibilité avec WINDEV 5.5
<a name="compatibilite_avec_windev_55_ELTTEXTE000164"></a>


### Attention
<a name="attention_ELTPARAGRAPHE000059"></a>**Pour migrer un dictionnaire existant en version 5.5**, il est conseillé de réaliser les opérations suivantes :

- Créer une fenêtre vierge.

- Ajouter dans la fenêtre tous les éléments du dictionnaire.

- Migrer la fenêtre.

- Recréer le dictionnaire sous WINDEV 28.

## Génération d'un pattern RAD
<a name="generation_pattern_rad_ELTTEXTE000144"></a>
Pour générer un pattern RAD :

1. Sous le volet "Projet", dans le groupe "Génération", cliquez sur "Générer".

2. Dans l'assistant qui s'affiche, indiquez :

	- Le nom du pattern ainsi que sa description. 
			Ce nom permettra d'identifier le pattern lors de l'utilisation du RAD application complète ou lors de la génération de fenêtres ou pages. 
			La description permet de connaître les principales fonctionnalités utilisées par le pattern. Cette description est utilisée uniquement lors de l'utilisation du RAD Application Complète.

	- Les éléments qui peuvent être générés seuls. Ces éléments seront proposés lors de la création d'une fenêtre ou d'une page. Pour chaque élément, la flèche située dans la table permet de saisir un libellé et une description. Par exemple :

		- Libellé : Table (avec saisie)

		- Description : Permet de visualiser les enregistrements d'un fichier dans une table et d'en saisir de nouveaux.
						Conseil : Il est conseillé de ne proposer que les fenêtres ou les pages qui portent sur les fichiers RADFichierB et RADFichierD.




- Si un gabarit peut être appliqué aux éléments générés par le RAD. Si le pattern est associé à un gabarit, il est conseillé de ne pas autoriser l'application d'un gabarit. 
			

- Si les tables générées doivent conserver leur mode d'affichage (saisie autorisée ou affichage seulement) ou non. Si les tables ne conservent pas leur mode d'affichage, l'assistant de génération du RAD proposera de gérer les tables en saisie ou en affichage seulement. 
			![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Si le développeur est autorisé ou non à intégrer le menu automatique d'aide dans l'application. Si le menu automatique d'aide est autorisé, lors de la génération du RAD, un plan spécifique proposera d'intégrer le menu "?".

- Si les images des menus peuvent être personnalisées ou non. Si les images des menus sont personnalisables, les images des menus (au format bouton ou Carrousel) utiliseront les images indiquées par le développeur réalisant la génération. Vous pouvez indiquer la taille conseillée pour les images.

- L'image du pattern. Cette image sera utilisée pour identifier le pattern. Par défaut, l'image de la première fenêtre du projet est proposée. Il est possible de choisir une image d'une autre fenêtre, ou une image provenant d'un fichier.

3. Validez. Le pattern RAD est généré. Le fichier &lt;Nom du pattern&gt;.Pattern est créé dans le sous-répertoire EXE du projet.



<a name="NOTE2_2"></a>


### Remarque
<a name="remarque_ELTPARAGRAPHE000071"></a>La caractéristique de visibilité de l'identifiant automatique définie dans le pattern est conservée à la génération. Ainsi, si l'identifiant automatique est invisible sur le pattern d'une fiche, le champ généré dans la fenêtre correspondante sera également invisible.

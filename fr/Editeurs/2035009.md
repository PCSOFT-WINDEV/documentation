


## Génération d'un diagramme de classes par rétro-ingénierie 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000079"></a>
WINDEV, WINDEV Mobile et WEBDEV permettent de créer le diagramme de classes d'une application par rétro-analyse des classes (fichiers ".WDC") présentes dans cette application. Les classes présentes dans le projet sont analysées et le diagramme de classes est construit à partir de ces informations.

**Remarque** : Pour créer un diagramme de classes par rétro-ingénierie, l'application WINDEV doit contenir des classes WINDEV.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000103"></a>
Pour créer un diagramme de classes par rétro-ingénierie :

1. Ouvrez le projet dans lequel le diagramme de classes doit être construit.

2. Cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_nouveau.gif)
 parmi les boutons d'accès rapide.

	- La fenêtre de création d'un nouvel élément s'affiche : cliquez sur "Architecture" puis sur "UML".

	- L'assistant de création d'un modèle UML se lance.




3. Sélectionnez l'option "Diagramme de classes".

4. Spécifiez le modèle UML dans lequel sera inséré le diagramme.

5. Choisissez de construire le diagramme de classes par rétro-analyse du projet.

6. Validez la création du diagramme de classes. Le diagramme de classes créé devient le diagramme de classes en cours.

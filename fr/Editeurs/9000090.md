


## Références croisées dans l'éditeur de code
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000119"></a>
Les références croisées dans l'éditeur de code permettent de savoir :

- quels sont les codes appelant l'élément actuellement sélectionné sous l'éditeur de code.

- quels sont les codes appelés par l'élément actuellement sélectionné sous l'éditeur de code.




Cet élément peut être un champ, une fenêtre, une procédure, une méthode de classe, une fonction WLangage , ...

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Ref_croisees.gif&type=thumb)


<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000143"></a>


### Afficher les références croisées d'un élément
<a name="afficher_les_references_croisees_element_ELTPARAGRAPHE000024"></a>Pour afficher les références croisées d'un élément (champ, fenêtre, procédure, fonction WLangage, ...) présent dans l'éditeur de code :

1. Placez le curseur de la souris sur le nom de l'élément.

2. Utilisez : 

	- soit la combinaison de touches Maj + F2.

	- soit l'option de menu "Références croisées" du menu contextuel. 




3. La liste des références croisées s'affiche.



<a name="NOTE2_2"></a>


### Paramétrer les options d'affichage des références croisées
<a name="parametrer_les_options_affichage_des_references_croisees_ELTPARAGRAPHE000038"></a>Depuis la fenêtre d'affichage des références croisées, il est possible de paramétrer les options d'affichage des références croisées (bouton "Options d'affichage").

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Ref_croisees_option.gif&type=thumb)
Il est possible de lister les éléments suivants :

- ***Références de code*** : ces références peuvent correspondre :

	- aux utilisations de l'élément dans le code. Dans ce cas, dès que le nom de l'élément est utilisé, la référence est automatiquement listée.

	- aux seules modifications de l'élément. Dans ce cas, seules les lignes de code dans lesquelles l'élément est modifié sont listées.

	- aux appels de procédures (pour les procédures locales ou globales). Seuls les appels aux procédures sont listés.




- ***Références d'élément fenêtre ou page interne, modèle, ...***, c'est-à-dire les fenêtres, pages internes ou modèles utilisant l'élément.

- ***Liaisons ***, pour les champs liés à des fichiers de données et rubriques de l'analyse.

- ***Informations partagées de l'analyse***, pour les champs liés à des rubriques.

- ***Liens depuis les modélisations souples*** (par exemple si la fenêtre est utilisée dans une modélisation souple).

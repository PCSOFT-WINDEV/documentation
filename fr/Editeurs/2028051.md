


## Etape 5 : Paramètres du site
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000185"></a>
Les paramètres du site permettent de configurer les options de connexion au site dès son installation. Ces paramètres pourront être modifiés si nécessaire :

- soit par l'administrateur de déploiement présent sur le serveur.

- soit par l'administrateur à distance, utilisable par le gestionnaire du site.

<a name="NOTE2_1"></a>


## Paramétrage du site
<a name="parametrage_site_ELTTEXTE000209"></a>

![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=param_site.gif)
Les paramètres de déploiement sont les suivants :

- **Activer le site immédiatement après son installation** : 
	Le site sera disponible directement après son installation : il sera possible de s'y connecter en utilisant l'adresse adéquate. 
	Si cette option n'est pas cochée, il sera possible d'activer le site grâce à l'administrateur à distance.

- **Faire une sauvegarde du site avant la mise à jour** : 
	Dans le cas d'une mise à jour, permet de sauvegarder le site existant. 
	Cette sauvegarde possède un nom et pourra être complète ou contenir uniquement les fichiers déployés.
	Cette sauvegarde pourra être réactivée depuis l'Administrateur WEBDEV distant. 

- **Programmer le déploiement pour une date ultérieure** : 
	Dans ce cas, le déploiement sera effectué à la date et l'heure indiquées. Pour plus de détails, consultez [Installation immédiate ou différée](#NOTE2_2). 




Les paramètres du site sont les suivants : 

- **Déconnecter les utilisateurs inactifs depuis** : (déploiement d'un site WEBDEV dynamique uniquement)
	Temps maximum de connexion autorisé sans action de la part de l'internaute. 
	Si ce temps est dépassé, l'internaute est automatiquement déconnecté, et un message d'erreur est affiché.

- **Durée des contextes AWP** : (déploiement d'un site WEBDEV AWP uniquement)
	Durée de validité des contextes AWP pour le site. Dès que la durée indiquée est écoulée et si aucune nouvelle requête n'a été effectuée, le fichier des contextes est supprimé. Pour plus de détails sur la gestion des contextes AWP, consultez [AWP : Gestion des contextes](../WDChamp/3539060.md).

- **Durée maximum d'une tâche** : 
	Durée maximale d'exécution des tâches planifiées ou différées. Si les tâches utilisées dépassent cette durée, elles seront automatiquement arrêtées. Pour plus de détails, consultez [WEBDEV : Tâches planifiées et tâches différées](../WDLang2/1000019455.md). 

- **Nombre maximum de connexions simultanées au site** : 
	Correspond au nombre maximum de connexions simultanées autorisées à ce site. Par exemple, si ce chiffre est égal à 250, 250 internautes pourront être connectés en même temps à ce site.

- **Nombre maximum de sessions prélancées du site** : 
	Nombre de sessions prélancées pour le site. 
	Pour plus de détails, consultez [Les sessions prélancées](../WDLang2/1000019485.md).

- **Nombre maximum de connexions par utilisateur** : (déploiement d'un site WEBDEV dynamique uniquement)
	Correspond au nombre maximum de connexions simultanées autorisées d'un même internaute au site. 
	Si ce paramètre est égal à zéro, l'internaute essayant de se connecter simultanément à un même site sera automatiquement déconnecté puis reconnecté. 
	Si ce paramètre est supérieur à zéro, un message d'erreur apparaîtra automatiquement au bout des x connexions simultanées.




Lors d'un déploiement sur plusieurs serveurs simultanément, il est possible de configurer les paramètres du site pour chaque profil. Les éléments pris en compte sont les suivants : 

- Nombre maximum de connexions simultanées au site.

- Nombre maximum de connexions par utilisateur.

- Déconnecter les utilisateurs inactifs depuis.

### Installation immédiate : cas d'une installation sur un serveur d'application WEBDEV distant
<a name="installation_immediate_cas_une_installation_sur_serveur_application_webdev_distant_ELTPARAGRAPHE000100"></a>Par défaut, WEBDEV propose une **installation immédiate** du site. Dans ce cas, les fichiers sont transférés immédiatement par FTP sur le serveur et installés dans le répertoire défini sur le serveur.

Dans certains cas, une **installation différée** peut se révéler intéressante. Quelques exemples :

- installation d'un site de taille importante : une installation différée permet de s'assurer que tous les fichiers sont installés (en installation immédiate, l'installation peut être stoppée selon le time out défini pour le CGI.

- mise à jour d'un site : une installation différée permet de faire une installation de nuit, afin de limiter le nombre d'internautes connectés.

- mise à jour de plusieurs sites partageant une analyse avec modification automatique des données : une mise à jour différée permet de programmer la mise à jour en même temps de tous les sites pour prendre en compte la modification automatique des fichiers de données.




Lors d'une installation différée, les fichiers sont immédiatement transmis par FTP, mais ils ne seront installés réellement que lorsque la date et l'heure indiquées sont atteintes sur le serveur.

**Attention** : la date et l'heure spécifiées sont celles du serveur. Assurez-vous de leur validité.



## Sécurité du site
<a name="securite_site_ELTTEXTE000239"></a>
L'assistant permet ensuite de définir les paramètres de sécurité du site : 

- **Changement d'IP en cours de navigation** : Si cette option est interdite, l'adresse IP associée à la session ne peut pas changer en cours de navigation. Il est ainsi possible d'empêcher les attaques du type "vol de session" (attaque consistant à se faire passer pour un utilisateur légitime connecté au serveur). 

- **Accès aux identifiants de contexte AWP depuis JavaScript** : Dans un site AWP, le contexte du site est stocké sur le serveur. L'identifiant de ce contexte est transmis et mémorisé dans le navigateur par l'intermédiaire d'un cookie. Si cette option est interdite, les cookies utilisés seront de type "HTTPOnly", type ne pouvant pas être lu depuis du code Javascript. Ce mode protège contre les attaques de type XSS (Cross-Site scripting). 
	Par défaut, l'accès aux identifiants de contexte AWP depuis JavaScript est interdit. 






## Génération des statistiques du site
<a name="generation_des_statistiques_site_ELTTEXTE000263"></a>
L'assistant permet ensuite de mettre en place les fichiers de statistiques du site. Ces fichiers sont exploités par l'utilitaire WDStatistique pour connaître, par exemple, la fréquentation du site ou les pages les plus visitées.

Vous pouvez modifier le répertoire des fichiers de statistiques ou ne pas générer ces fichiers en décochant la case "Générer les fichiers de statistiques pour ce site". Le répertoire des fichiers de statistiques est toujours relatif par rapport au répertoire des données.

Pour plus de détails, consultez l'utilitaire [WDStatistiques](../WDStatistique/3536004.md).
Lors d'un déploiement sur plusieurs serveurs simultanément, il est possible de configurer les fichiers de statistiques pour chaque profil.


## Historique des versions (installation réseau)
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000084"></a>
L'assistant d'installation permet de paramétrer l'historique des versions lors d'une installation réseau. 

Le paramétrage de l'historique des versions d'installation permet de laisser à disposition des utilisateurs finaux plusieurs versions différentes d'une même application. 

Le paramétrage de l'historique des installations peut être réalisé : 

- soit à la création de l'installation, 

- soit lors de la mise en place de l'installation de référence (option "Ces paramètres pourront être modifiés au moment du déploiement sur le serveur").  






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Conserver un historique des versions installées
<a name="conserver_historique_des_versions_installees_ELTTEXTE000108"></a>
Si l'option "Conserver un historique des versions installées" est sélectionnée, il est possible de définir : 

- le **nombre maximum de versions conservées**. Ces versions seront conservées sur le serveur de référence. A chaque nouvelle version, les versions obsolètes sont supprimées. 

- la **taille maximale de l'historique** (pour éviter de saturer le disque du serveur de référence). 

- l' **activation immédiate de la version** installée sur le serveur. Dans ce cas, selon les paramètres de vérification spécifiés, l'application proposera une mise à jour. Si cette option n'est pas sélectionnée, il sera possible d'activer la version grâce au [Centre de Contrôle des applications](../WDAdminExe/2028052.md). 


Remarque : Il est possible de forcer la modification des paramètres saisis lors d'un déploiement précédent grâce à l'option "Ces paramètres remplaceront les éventuels paramètres précédemment saisis en déploiement".

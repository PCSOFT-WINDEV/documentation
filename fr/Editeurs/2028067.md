


## WDInst : générer le programme d'installation 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000224"></a>
La génération du programme d'installation permet de créer le programme d'installation selon les caractéristiques spécifiées dans l'éditeur d'installation WDInst. Pour lancer WDInst, sous le volet "Outils", dans le groupe "Utilitaires", cliquez sur "WDInst".

Le programme d'installation sera généré dans un répertoire en vue d'être copié sur différents supports.

Par défaut, le programme d'installation est généré dans le sous-répertoire "Install" du répertoire de votre application.

Pour générer le programme d'installation, utilisez l'option "Fichier .. Générer" de WDInst.

## Support du programme d'installation
<a name="support_programme_installation_ELTTEXTE000248"></a>


### Support du programme d'installation
<a name="support_programme_installation_ELTPARAGRAPHE000036"></a>Les différents supports du programme d'installation proposés par WINDEV sont les suivants :

- ***Répertoire unique***
	Le programme d'installation est créé dans un répertoire unique.

- ***Exécutable auto-extractible***
	Le programme d'installation est créé sous forme d'un exécutable auto-extractible. Lors du lancement de cet exécutable sur le poste des utilisateurs, le programme d'installation est automatiquement décompressé et lancé.
	Cette solution est conseillée si le programme d'installation est transmis par Internet.

- ***Support avec autorun***
	Le programme d'installation est créé dans un répertoire unique en vue d'être copié sur un CD-ROM, DVD-ROM, clé USB, ... 
	Dans ce cas, il est possible de saisir les paramètres de l'exécution automatique. Pour plus de détails, consultez [Inclure un autorun](../Editeurs/2028015.md). 

- ***Répertoires multiples***
	Les fichiers du programme d'installation sont regroupés dans plusieurs sous-répertoires. La taille des fichiers contenus dans chaque sous-répertoire ne dépasse pas la taille spécifiée (1,44 Mo par défaut). Il est ensuite possible de copier le contenu de chaque sous-répertoire.




Pour choisir le support du programme d'installation le plus approprié à vos besoins, consultez le [Tableau récapitulatif sur l'installation d'une application WINDEV](../Editeurs/2028005.md).

Lors de la création du support d'installation, il est possible de :

- **Vider le répertoire avant de générer l'installation** : Si cette option est cochée, le contenu du répertoire dans lequel le programme d'installation est généré sera vidé avant la génération d'une nouvelle installation. Cette option permet de supprimer tout fichier inutile.

- **Protéger l'installation par un mot de passe** : Cette option permet d'indiquer le mot de passe qui sera demandé lors de l'installation de l'application.

- **Signer l'installation avec le certificat** : Pour garantir l'intégrité du pack d'installation, il est possible de signer l'installation. Il suffit de sélectionner le certificat à utiliser. Les certificats disponibles sur le poste en cours, dans le magasin personnel des certificats de Windows, sont automatiquement proposés. 




Remarque : Dans le cas d'une installation via un Store Privé, seul le support de type "Exécutable auto-extractible" est disponible.

## Générer le programme d'installation
<a name="generer_programme_installation_ELTTEXTE000272"></a>


### Générer le programme d'installation
<a name="generer_programme_installation_ELTPARAGRAPHE000094"></a>Pour générer le programme d'installation :

1. Sélectionnez l'option "Fichier .. Générer".

2. Sélectionnez le support du programme d'installation. Les différents supports proposés sont détaillés ci-dessus. Pour choisir le support du programme d'installation le plus approprié à vos besoins, consultez le [Tableau récapitulatif sur l'installation d'une application WINDEV](../Editeurs/2028005.md).

3. A chaque génération du programme d'installation, l'éditeur d'installation WDInst crée un fichier temporaire de type ".WXF". Ce fichier contient la description du programme d'installation. Ce fichier peut être utilisé pour tester un programme d'installation personnalisé ([projet "WDSetup" personnalisé](../Editeurs/2028037.md)). Pour conserver ce fichier, cochez l'option "Conserver le fichier WXF pour tester un WDSetup personnalisé" (onglet "Avancé") 
	Pour plus de détails, consultez le paragraphe [fichier WXF et fichier WDF](#NOTE4_1).

4. Sélectionnez les caractéristiques du programme d'installation en fonction du support d'installation sélectionné : 
	

| Support d'installation | Caractéristiques |
| --- | --- |
| <br><br>	- Répertoire unique<br><br><br> | <br><br>	- Modifiez si nécessaire le répertoire dans lequel le programme d'installation sera généré.<br><br><br> |
| <br><br>	- Exécutable auto-extractible<br><br><br> | <br><br>	- Modifiez si nécessaire le répertoire dans lequel l'exécutable auto-extractible sera généré.<br><br>	- Saisissez le nom de l'exécutable à créer (sans l'extension).<br><br><br> |
| <br><br>	- Support avec autorun<br><br><br> | <br><br>	- Modifiez si nécessaire le répertoire dans lequel le programme d'installation sera généré.<br><br><br>Indiquez si nécessaire les paramètres d'exécution automatique :<br><br>	- Icône du CD.<br><br>	- Action effectuée à l'insertion du CD. Pour plus de détails, consultez [Exécution automatique du CD](../Editeurs/2028015.md).<br><br><br> |
| <br><br>	- Répertoires multiples<br><br><br> | <br><br>	- Modifiez si nécessaire le répertoire de base de la génération du programme d'installation.<br><br>	- Sélectionnez la taille maximale de l'ensemble des fichiers contenus dans chaque sous-répertoire. Les sous-répertoires seront nommés "DISK &lt;N°Répertoire&gt;".<br><br><br> |



5. Validez la génération du programme d'installation.

## Fichier WXF et fichier WDF
<a name="fichier_wxf_fichier_wdf_ELTTEXTE000296"></a>


### Présentation
<a name="presentation_ELTPARAGRAPHE000152"></a>A chaque génération du programme d'installation, l'éditeur d'installation WDInst crée deux fichiers contenant la description du programme de développement :

- **un fichier de type ".WDF"**. Ce fichier permet d'éditer la description du programme d'installation dans WDInst. Par défaut, ce fichier est nommé "&lt;NomApplication&gt;.WDF" et est sauvegardé dans le répertoire de l'application sur le poste de développement.

- **un fichier de type ".WXF"**. Ce fichier permet de tester un programme d'installation personnalisé (projet "WDSetup" personnalisé). Par défaut, ce fichier est compacté et archivé dans le fichier "INSTALL.WDZ", puis supprimé du poste de génération.




Si vous utilisez un programme d'installation personnalisé (projet "WDSetup" personnalisé), il est nécessaire de conserver le fichier ".WXF" pour tester ce programme d'installation personnalisé. En effet, le programme d'installation personnalisé récupère dans le fichier ".WXF" toutes les options du programme d'installation spécifiées dans l'éditeur d'installation WDInst.

Dans ce cas, ce fichier sera nommé "&lt;NomApplication&gt;.WXF" et sera sauvegardé dans le répertoire du projet à installer.

**Remarque** : Pour plus de détails sur l'utilisation de WDSetup (personnalisation, test, utilisation), consultez [Utiliser un WDSetup personnalisé](../Editeurs/2028037.md).
<a name="NOTE4_2"></a>


### Conserver le fichier ".WXF"
<a name="conserver_fichier_wxf_ELTPARAGRAPHE000171"></a>Pour conserver le fichier ".WXF", cochez l'option "Conserver le fichier WXF pour tester un WDSetup personnalisé" lors de la génération du programme d'installation (option "Fichier .. Générer" de WDInst, onglet "Avancé").

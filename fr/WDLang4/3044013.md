


## HInitAbonné (Fonction)

***En anglais : HInitSubscriber***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Initialise la plage d'identifiants automatiques d'un fichier de description d'un réplica abonné (extension ".RPL"). **Cette fonction est utilisable uniquement avec la réplication journalée**. 

L'initialisation de la plage d'identifiants automatiques est réalisée :

- automatiquement, lors de la création d'un fichier de données (fonctions [HCréation](../WDLang4/3044255.md), [HCréationSiInexistant](../WDLang4/3044254.md)).

- manuellement, avec la fonction **HInitAbonné**.




La fonction **HInitAbonné** doit être appelée uniquement lors de l'installation des fichiers de données chez l'abonné.

**Attention** : Les fichiers de données doivent être ouverts avant l'utilisation de la fonction **HInitAbonné**.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Initialise la plage d'ID Auto du fichier de données Client
HInitAbonné("Client", "C:\DATA\Replic\AbonneParis.RPL")

// Initialise la plage d'ID Auto de tous les fichiers de données
HInitAbonné("*", "C:\DATA\Replic\AbonneParis.RPL")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HInitAbonné(<Fichier de données> , <Réplica Abonné>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'initialisation a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données HFSQL à utiliser.
Si ce paramètre est égal à "\*", les plages d'identifiants de tous les fichiers de données décrits comme "répliqués" dans l'analyse seront initialisées.

**`<Réplica Abonné> : (Chaîne de caractères)`**

Nom et chemin complet du réplica abonné. Ce paramètre peut correspondre à un chemin de type UNC.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Plage d'identifiants
<a name="plage_identifiants_ELTPARAGRAPHE000093"></a>Les identifiants automatiques des fichiers des réplicas abonnés utilisent une plage d'identifiants différente de celle du fichier de réplica maître et des autres fichiers de réplicas abonnés.

Cette plage d'identifiants automatiques est inscrite dans l'entête de chaque fichier de données (extension ".FIC").
Cette plage est initialisée :

- lors de la création d'un fichier de données (fonctions [HCréation](../WDLang4/3044255.md), [HCréationSiInexistant](../WDLang4/3044254.md)).

- grâce à l'outil [WDReplic](../WDReplic/3520016.md).

- lors de l'utilisation de la fonction **HInitAbonné**.




La plage d'identifiants peut être connue grâce aux propriétés [IDAutoMin](../Proprietes/2512039.md) et [IDAutoMax](../Proprietes/2512023.md).

Pour plus de détails, consultez les [conséquences de la mise en place de la réplication](../WDLang4/3044246.md). 
<a name="NOTE0_2"></a>


### Réplication et journalisation
<a name="replication_journalisation_ELTPARAGRAPHE000135"></a>Si la fonction **HInitAbonné** est utilisée, il est également nécessaire d'initialiser l'emplacement des journaux et des tables d'opérations (fonction [HChangeRepJNL](../WDLang4/3044164.md), propriétés [RépertoireJournal](../Proprietes/2512075.md) et [RépertoireJournalOpération](../Proprietes/2512076.md)).

**Rappel** : La réplication journalée se base sur les journaux et tables d'opérations des fichiers de données. Si l'emplacement des journaux et des tables d'opérations est erroné, la réplication journalée ne peut pas fonctionner.

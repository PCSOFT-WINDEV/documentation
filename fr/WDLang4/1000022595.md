


## MongoGridFSRécupèreFichierParID (Fonction)

***En anglais : MongoGridFSGetFileByID***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère sur le poste local un fichier (identifié par son identifiant) présent dans un système de fichiers MongoDB.

**Remarque** : Pour récupérer en local un fichier identifié à partir de son nom, utilisez la fonction [MongoGridFSRécupèreFichier](../WDLang4/1000022439.md). 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
POUR TOUT fic de MongoGridFSListeFichier(baseMDB)
	MongoGridFSRécupèreFichierParID(baseMDB, fic.Id, fRepDonnées() + ["\"] + fic.Nom)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = MongoGridFSRécupèreFichierParID(<Système de fichiers> , <Identifiant du fichier à récupérer> , <Nom du fichier Local>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée,

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'identifier l'erreur.




**`<Système de fichiers> : (Variable de type mongoGridFS ou de type mongoBase)`**

Système de fichiers de la base MongoDB à manipuler. Correspond : 

- au nom de la variable de type [mongoGridFS](../WDLang4/1000022416.md) correspondant au système de fichiers Mongo.

- au nom de la variable de type [mongoBase](../WDLang4/1000022410.md) correspondant à la base MongoDB. Dans ce cas, le système de fichiers utilise le préfixe "fs".




**`<Identifiant du fichier à récupérer> : (Variant)`**

Identifiant du fichier présent dans le système de fichiers MongoDB à récupérer sur le poste local.

**`<Nom du fichier Local> : (Chaîne de caractères)`**

Nom et chemin du fichier qui sera récupéré sur le poste local.

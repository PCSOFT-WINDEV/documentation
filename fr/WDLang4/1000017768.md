
## HRepTravailServeur (Fonction)

***En anglais : HServerWorkingDir***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie le chemin d'un répertoire temporaire sur le serveur HFSQL. Ce répertoire est automatiquement créé sur le serveur. Les procédures stockées exécutées sur le serveur auront les droits d'écriture dans ce répertoire. Ce répertoire sera automatiquement vidé et détruit lors de la déconnexion du client.

**Remarque** : Cette fonction peut être exécutée uniquement à partir d'une procédure stockée.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Code d'une procédure stockée
fSauveTexte(ComplèteRep(HRepTravailServeur()) + "MonFichier.txt", MaChaîneTexte)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HRepTravailServeur()`
---

**`<Résultat> : (Chaîne de caractères)`**

Nom complet du répertoire temporaire sur le serveur HFSQL. Une procédure stockée pourra par exemple créer des fichiers texte dans ce répertoire.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Pour forcer la suppression du répertoire et de son contenu, utilisez la fonction [HPurgeRepTravail](../WDLang4/1000017767.md).

- La fonction [HGèreServeur](../WDLang4/3044359.md) renvoie également des informations sur les chemins des différents répertoires présents sur le serveur (chemin du répertoire de la base de données, chemin du répertoire des journaux, chemin du répertoire de sauvegarde des journaux, ...)


## HRplGestionFichier (Fonction)

***En anglais : HRplManageFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Définit les options utilisées pour la réplication universelle d'un fichier de données :

- le sens de réplication.

- le mode de gestion des conflits.




Par défaut, tous les fichiers de données utilisent les caractéristiques de synchronisation définies par la fonction [HSynchroniseRéplica](../WDLang4/3044014.md). La fonction **HRplGestionFichier** permet par exemple d'indiquer simplement qu'un fichier de données est uniquement en réplication du maître vers l'abonné (alors que tous les autres fichiers sont par exemple en réplication bidirectionnelle).

**Cette fonction peut être utilisée uniquement en réplication universelle.**




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Le fichier Produit sera répliqué uniquement du maître vers l'abonné
HRplGestionFichier(Produit, rplGestionConflitRéplication, rplVersAbonné)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`HRplGestionFichier(<Nom du fichier> , <Gestion des conflits> [, <Sens de la réplication>])`
---

**`<Nom du fichier> : (Chaîne de caractères)`**

Nom du fichier de données à paramétrer.

**`<Gestion des conflits> : (Constante de type Entier)`**

Constante indiquant le mode de gestion des conflits :


|   |   |
| --- | --- |
| *rplAbonnéPrioritaire* | Les données présentes dans le fichier de données abonné sont prioritaires lors de la réplication. |
| *rplGestionConflitRéplication*<br>(valeur par défaut) | Le fichier utilisera le mode de gestion de gestion des conflits spécifié avec la fonction [HSynchroniseRéplica](../WDLang4/3044014.md). |
| *rplMaîtrePrioritaire* | Les données présentes dans le fichier de données maître sont prioritaires lors de la réplication. |
| *rplPlusRécentPrioritaire* | Les données les plus récentes sont prioritaires. Attention, les données les plus récentes sont évaluées par rapport à l'heure sur le poste abonné. |



**`<Sens de la réplication> : (Constante optionnelle de type entier)`**

Sens dans lequel la réplication doit être effectuée :


|   |   |
| --- | --- |
| *rplBidirectionnel* <br>(valeur par défaut) | Mise à jour du fichier maître ET du fichier abonné. |
| *rplVersAbonné* | Mise à jour du fichier abonné en fonction de la base de données maître. |
| *rplVersMaître* | Mise à jour du fichier maître en fonction de la base de données abonnée. |

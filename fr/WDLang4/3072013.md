### Opérations nécessaires pour faire fonctionner le Connecteur Natif Access en WINDEV et WEBDEV
<a name="operations_necessaires_pour_faire_fonctionner_connecteur_natif_access_windev_webdev_ELTPARAGRAPHE000011"></a>Le Connecteur Natif (également appelé Accès Natif) aux bases de type ACCESS est fourni dans un objectif d'interopérabilité. Il est conseillé d'utiliser HFSQL le moteur de base de données intégré à WINDEV et WEBDEV.



Pour pouvoir utiliser le Connecteur Natif Access, il est nécessaire d'installer :

- le MDAC.

- le provider Microsoft Jet (installé avec Access ou Internet Explorer 6.0 par exemple).




Remarque : Le Connecteur Natif Access fonctionne pour Access 2007 et supérieur (fichiers au format ACCDB/MDB).

## Créer une application utilisant le Connecteur Natif Access
<a name="creer_une_application_utilisant_connecteur_natif_access_ELTTEXTE000149"></a>
Pour créer une application manipulant des tables Access, il suffit d'importer la structure de ces tables sous l'éditeur d'analyses de WINDEV ou WEBDEV : 

1. Ouvrez l'analyse associée au projet : cliquez sur ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=Ico_Analyse.gif)
 parmi les boutons d'accès rapide.

2. Sous le volet "Analyse", dans le groupe "Création", déroulez "Importer" et sélectionnez "Importer des descriptions de fichiers / tables".




**Si vous choisissez de conserver les données au format Access**, vous pourrez manipuler directement les tables Access grâce aux fonctions WLangage de gestion des fichiers de données (fonctions commençant par la lettre H). Une connexion à la base de données Access est automatiquement réalisée. 
Remarque : si le chemin des tables Access utilisées est différent lors du développement et lors de l'exécution de l'application, utilisez les fonctions [HChangeRep](../WDLang4/3044168.md) et [HChangeNom](../WDLang4/3044141.md).

**Si vous choisissez de convertir les données au format HFSQL**, des fichiers de données au format HFSQL seront créés à partir de la structure des tables Access. Ces fichiers HFSQL pourront être manipulés par les fonctions de gestion des fichiers de données (fonctions commençant par la lettre H).

## J'utilisais le Connecteur Natif Access avec WINDEV 5.5. Comment faire avec une nouvelle version de WINDEV ?
<a name="jutilisais_connecteur_natif_access_avec_windev_55_comment_faire_avec_une_nouvelle_version_windev_ELTTEXTE000173"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Connecteurs Natifs (Accès Natifs)](https://doc.pcsoft.fr/ext/images/fr/AN.png) En WINDEV 5.5, pour utiliser le Connecteur Natif Access, il suffisait d'utiliser la fonction [SQLConnecte](../WDLang4/3072005.md). Une fois connecté à une base de données Access, toutes les fonctions SQL classiques permettaient d'accéder et de modifier les données de la base de données.

Ce fonctionnement est conservé avec les versions supérieures de WINDEV.

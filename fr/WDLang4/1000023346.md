
## MemcachedIncrémente (Fonction)

***En anglais : MemcachedIncrement***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Incrémente la valeur associée à une clé. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MemcachedIncrémente(memConnexion, "compteur", 1, 0, 60s)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = MemcachedIncrémente(<Connexion> , <Clé> [, <Incrémentation> [, <Valeur initiale> [, <Expiration>]]])`
---

**`<Résultat> : (Entier)`**

Valeur précédente de la clé. 

**`<Connexion> : (Variable de type memcachedConnexion)`**

Nom de la variable de type [memcachedConnexion](../WDLang4/1000023339.md) correspondant à la connexion au cache Memcached.  

**`<Clé> : (Chaîne de caractères)`**

Clé à manipuler.

**`<Incrémentation> : (Entier optionnel)`**

Valeur de l'incrémentation (par défaut 1). 

**`<Valeur initiale> : (Entier optionnel)`**

Valeur initiale de la clé si la clé recherchée n'existe pas. En effet, si la clé recherchée n'existe pas, elle est automatiquement créée et elle a pour valeur &lt;Valeur initiale&gt;. 
ATTENTION : pour utiliser ce paramètre, le serveur Memcached doit utiliser le protocole de communication binaire (la propriété **ProtocoleBinaire** de la variable de type [memcachedConnexion](../WDLang4/1000023339.md) doit correspondre à <u><u><u><u>Vrai</u></u></u></u>). 
Si ce paramètre n'est pas renseigné, une erreur est renvoyée.

**`<Expiration> : (Entier ou durée optionnelle)`**

Durée pendant laquelle la clé est conservée dans le cache. Cette durée est exprimée en secondes. Cette durée peut correspondre à : 

- un entier correspondant au nombre de secondes, 

- une variable de type Durée. 

- l'indication directe de la durée (par exemple '10s'). 


Par défaut, cette durée vaut 0. Dans ce cas, la durée est illimitée : s'il n'y a plus de mémoire disponible, les clés les plus anciennes sont supprimées.

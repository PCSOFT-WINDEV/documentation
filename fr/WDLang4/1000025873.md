
## HTriggerEnregistrementAvant (Fonction)

***En anglais : HTriggerRecordBefore***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère la valeur de l'enregistrement en cours avant l'exécution du ou des triggers.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
sEnreg est un Enregistrement de FRAIS
// FRAIS est le nom du fichier sur lequel le trigger a été déclenché
sEnreg = HTriggerEnregistrementAvant()
soit sChaîne = HEnregistrementVersChaîne(sEnreg)
Trace("sEnreg : ", sChaîne)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = HTriggerEnregistrementAvant()`
---

**`<Résultat> : (Variable de type Enregistrement)`**

Variable de type [Enregistrement](../WDLang4/1000020962.md) correspondant à l'enregistrement initial.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Dans le cas de l'ajout d'un nouvel enregistrement dans le fichier de données, l'enregistrement retourné est un enregistrement "vide" marqué comme étant "supprimé". 

- Cas particulier : Si le fichier de données est un fichier Client/Serveur et si le trigger est un trigger "après" exécuté sur le poste client alors la fonction **HTriggerEnregistrementAvant** ne renvoie pas les mémos.

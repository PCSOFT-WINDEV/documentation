


## HModifiePropriétéFichier (Fonction)

***En anglais : HModifyFileProperty***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie les propriétés d'un fichier de données HFSQL situé sur un serveur.

**Remarque** : Il est possible de connaître les propriétés d'un fichier de données HFSQL grâce à la fonction [HInfoPropriétéFichier](../WDLang4/3044324.md).



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
ResModif est un booléen
ResModif = HModifiePropriétéFichier("MaConnexion", ...
		hPropriétaire, "Florence", "MaBase", "Client.fic")
SI ResModif = Vrai ALORS
	Info("Modification effectuée")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HModifiePropriétéFichier(<Connexion> , <Propriété à modifier> , <Nouvelle valeur> , <Base de données> , <Fichier de données>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si les propriétés ont été modifiées,

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Connexion> : (Chaîne de caractères ou variable de type Connexion)`**

Connexion pour laquelle la valeur d'une propriété doit être modifiée. Cette connexion correspond : 

- soit au nom de la connexion définie sous l'éditeur d'analyses ou grâce aux fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

- soit au nom d'une variable de type [Connexion](../WDLang4/1514073.md). 




**`<Propriété à modifier> : (Constante de type Entier)`**

Propriété du fichier de données à modifier : 


|   |   |
| --- | --- |
| *hNotes* | Description du fichier de données ou toute autre information associée au fichier de données. <br>Si cette constante est utilisée, &lt;Nouvelle valeur&gt; est une chaîne de caractères contenant le texte de la note. |
| *hPropriétaire* | Propriétaire du fichier de données. <br>Si cette constante est utilisée, &lt;Nouvelle valeur&gt; est une chaîne de caractères contenant le nom du propriétaire. |



**`<Nouvelle valeur> : (Chaîne de caractères)`**

Nouvelle valeur de la propriété.

**`<Base de données> : (Chaîne de caractères)`**

Nom de la base de données contenant le fichier de données pour lequel les propriétés doivent être modifiées. 
Si ce paramètre correspond à une chaîne vide (""), la base de données utilisée sera celle définie dans la connexion. 
Si aucune base de données n'est définie dans la connexion, une erreur WLangage sera générée.

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données pour lequel les propriétés doivent être modifiées. L'extension du fichier doit être précisée.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Changer le propriétaire d'un fichier de données
<a name="changer_proprietaire_fichier_donnees_ELTPARAGRAPHE000136"></a>

- L'utilisateur spécifié comme nouveau propriétaire doit être un utilisateur existant sur le serveur. Si l'utilisateur n'existe pas, une erreur WLangage est générée.

- Pour modifier le propriétaire, l'utilisateur faisant la modification doit :

	- soit être un "Super utilisateur".

	- soit avoir le droit de changer le propriétaire du fichier de données (droit correspondant à la constante *hChangeDroitPropriétaire*). Pour savoir si l'utilisateur a le droit approprié, utilisez la fonction [HInfoPropriétéFichier](../WDLang4/3044324.md).




## EspaceSignificatif (Fonction)

***En anglais : WithSpace***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute ou supprime les espaces situés à droite d'une rubrique texte lors de sa lecture.

**Attention** : Cette fonction est prise en compte uniquement pour les fichiers de données dont l'option "Mode compatible 5.5 : rubriques texte complétées par des espaces" a été cochée sous l'éditeur d'analyses (onglet "Détail" de la description des fichiers de données).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003044142&name=espacesignificatif_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Gère les espaces significatifs (ajoute des espaces) pour tous les fichiers
EspaceSignificatif(Vrai)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = EspaceSignificatif([<Nom du fichier>] [, <Nom de la rubrique>] , <Options>)`
---

**`<Résultat> : (Booléen)`**

Ancienne gestion des espaces significatifs.

**`<Nom du fichier> : (Chaîne de caractères optionnelle)`**

Nom du fichier de données manipulé. Si ce paramètre n'est pas spécifié, la fonction **EspaceSignificatif** va manipuler **tous les fichiers de données**.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Ce paramètre n'est pas pris en compte.

**`<Nom de la rubrique> : (Chaîne de caractères optionnelle)`**

Nom de la rubrique utilisée. Si ce paramètre n'est pas spécifié, la fonction **EspaceSignificatif** va manipuler **toutes les rubriques** du fichier de données spécifié.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Ce paramètre n'est pas pris en compte.

**`<Options> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> : le contenu des rubriques texte est complété avec des espaces à droite. 

- <u><u><u><u>Faux</u></u></u></u> : les espaces à droite des rubriques texte sont supprimés.


![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png)  La gestion des espaces significatifs sera effectuée pour tous les fichiers de données et pour toutes les rubriques texte de ces fichiers de données.

<a name="NOTE0_1"></a>

## Remarques


### Genéralisation de la gestion des espaces
<a name="generalisation_gestion_des_espaces_ELTPARAGRAPHE000084"></a>

- Pour généraliser le mode de gestion des espaces à **toutes les rubriques d'un fichier de données**, le paramètre &lt;Nom de la rubrique&gt; doit être égal à "\*". 
	Exemple : 
	
	```txt
	EspaceSignificatif(<Nom du fichier>, "*", <Options>)
	```


- Pour généraliser le mode de gestion des espaces à **toutes les rubriques de tous les fichiers de données**, les paramètres &lt;Nom du fichier&gt; et &lt;Nom de la rubrique&gt; doivent être égaux à "\*". 
	Exemple : 
	
	```txt
	EspaceSignificatif("*", "*", <Options>)
	```
Dans ce cas, la fonction **EspaceSignificatif** renvoie <u><u><u><u>Faux</u></u></u></u> si la gestion des espaces était différente sur l'une au moins des rubriques concernées.

- Si seul le paramètre &lt;Options&gt; est précisé, le mode de gestion spécifié s'applique à tous les fichiers de données et à toutes les rubriques.





### Réplication et fonction EspaceSignificatif
<a name="replication_fonction_espacesignificatif_ELTPARAGRAPHE000107"></a>La réplication est incompatible avec l'utilisation de la fonction **EspaceSignificatif** dans le code de l'application cliente. Cette fonction change le comportement de HFSQL. L'utilisation de cette fonction dans l'application cliente peut affecter l'efficacité de la réplication.
Depuis la version 21 update 3 (version 210065), les fonctions de synchronisation (réplication universelle programmée ou assistée) renvoient une erreur à l'application cliente si la fonction **EspaceSignificatif** a été appliquée pour l'un au moins des fichiers répliqués.

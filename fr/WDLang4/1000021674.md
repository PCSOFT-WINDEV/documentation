


## hdfsSupprime (Fonction)

***En anglais : hdfsDelete***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime un fichier ou un répertoire sur un serveur Hadoop.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connexion
MaConnexion est une hdfsConnexion
MaConnexion.namenode = "MonServeur"
MaConnexion.Utilisateur = "Admin"
hdfsOuvreConnexion(MaConnexion)
// Suppression du fichier sur le serveur
hdfsSupprime(MaConnexion, "/tmp/test.data")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = hdfsSupprime(<Connexion> , <Chemin> [, <Options>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la suppression a été réalisée,

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type hdfsConnexion)`**

Nom de la variable de type [hdfsConnexion](../WDLang4/1000021650.md) correspondant à la connexion au serveur Hadoop.

**`<Chemin> : (Chaîne de caractères)`**

Chemin absolu du fichier ou du répertoire à supprimer sur le serveur Hadoop. Ce chemin est sensible à la casse.

**`<Options> : (Constante de type Entier optionnelle)`**

Options de suppression :


|   |   |
| --- | --- |
| *hdfsDéfaut*<br>(Valeur par défaut) | La suppression d'un répertoire non vide ne sera pas réalisée. |
| *hdfsRécursif* | Les contenus du répertoire et des sous-répertoires sont également supprimés. |

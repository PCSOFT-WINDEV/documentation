
## H.LimiteRéindexationAutomatique (Variable)

***En anglais : H.AutoIndexingLimit***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Indique le nombre limite d'enregistrements dans le fichier de données permettant de gérer la ré-indexation automatique.

Tant que ce nombre n'est pas atteint, le moteur HFSQL lance la ré-indexation automatique en cas d'erreur d'index.

Rappel : La ré-indexation automatique n'est pas disponible sur des fichiers de données au format Hyper File 5.5.


<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Valeur de H.LimiteRéindexationAutomatique
<a name="valeur_hlimitereindexationautomatique_ELTPARAGRAPHE000024"></a>

- Par défaut, la variable H.LimiteRéindexationAutomatique vaut 100.000.

- Si H.LimiteRéindexationAutomatique vaut 0, la ré-indexation automatique est désactivée.

- Si H.LimiteRéindexationAutomatique est initialisé avec une valeur négative, la valeur de cette variable correspondra automatiquement à la valeur par défaut.




**Attention** : Si une erreur de blocage survient pendant la ré-indexation (par exemple lorsque le fichier de données ou un enregistrement du fichier de données est bloqué en lecture/écriture), la ré-indexation automatique sera arrêtée et une erreur fatale d'index corrompu sera affichée. Il sera alors nécessaire de ré-indexer le fichier de données.

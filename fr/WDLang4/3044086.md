
## HErreurIntégrité (Fonction)

***En anglais : HErrorIntegrity***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de vérifier si une erreur d'intégrité est survenue.

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Par défaut, WINDEV propose une [gestion automatique des erreurs d'intégrité](../WDLang4/3044188.md). Cette gestion automatique n'est pas disponible dans le code des procédures stockées.

![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Seules les erreurs d'intégrité survenues sur des fichiers de données HFSQL sont signalées.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Gestion d'une erreur d'intégrité
SI HModifie(Client) = Faux ALORS
	SI HErreurIntégrité() = Vrai ALORS 
		Erreur("Erreur d'intégrité : cet enregistrement ne peut être modifié")
	FIN
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HErreurIntégrité()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si une erreur d'intégrité est survenue, 

- <u><u><u><u>Faux</u></u></u></u> sinon (pas d'erreur).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Par défaut, toutes les règles d'intégrité définies dans la description de l'analyse sont actives. Pour activer ou désactiver la gestion d'une contrainte d'intégrité, utilisez la fonction [HGèreIntégrité](../WDLang4/3044058.md).

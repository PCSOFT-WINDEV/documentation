


## HDébloqueFichier (Fonction)

***En anglais : HUnlockFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Débloque les enregistrements d'un fichier de données :

- précédemment bloqués avec la fonction [HBloqueFichier](../WDLang4/3044153.md).

- bloqués individuellement avec une fonction de lecture bloquante.




Le déblocage du fichier de données peut aussi être obtenu en fermant le fichier de données (fonction [HFerme](../WDLang4/3044073.md)).

Remarque : La fonction **HDébloqueFichier** ne permet pas de débloquer un fichier de données bloqué par une autre session ou un autre programme. Il est conseillé de débloquer un fichier de données le plus rapidement possible, afin que les autres sessions puissent accéder à ce fichier de données.
![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Cette fonction peut être utilisée avec des fichiers de données HFSQL Client/Serveur.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Blocage
HBloqueFichier(CLIENT, HBlocageLectureEcriture)
// Modification du fichier
...
// Déblocage
HDébloqueFichier(CLIENT)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HDébloqueFichier([<Fichier de données>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier de données a été débloqué, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème (par exemple le fichier de données n'est pas bloqué, ...). La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Fichier de données> : (Chaîne de caractères optionnelle)`**

Nom du fichier de données manipulé. 
Si ce nom n'est pas spécifié, la fonction **HDébloqueFichier** va manipuler le dernier fichier de données utilisé par la dernière fonction de gestion HFSQL (fonction commençant par la lettre H).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La fonction **HDébloqueFichier** est sans effet si :

- le fichier de données est en mode Mono (voir la fonction [HMode](../WDLang4/3044039.md)).

- le fichier de données n'est pas bloqué.

- le fichier de données a été bloqué par un autre programme.

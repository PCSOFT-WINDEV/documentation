


## &lt;Variable Connexion&gt;.TransactionIsolation (Fonction)

***En anglais : &lt;Connection variable&gt;.TransactionIsolation***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Configure le mode d'isolation des transactions pour une connexion vers un serveur HFSQL.

En fonction du mode sélectionné, une application effectuant des lectures aura accès ou non aux données en cours de modification dans une transaction par une autre application, ou une autre instance de la même application.

Pour plus de détails sur les modes d'isolation, consultez [Transactions Client/Serveur : Modes d'isolation disponibles](../WDLang4/1000017316.md). 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MaConnexion est une Connexion

MaConnexion.TransactionIsolation(hReadCommitted)
MonFichierClient.LitPremier(MaClé)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = <Connexion>.TransactionIsolation(<Mode d'isolation>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> le niveau d'isolation a été mis en place, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion vers un serveur HFSQL.

**`<Mode d'isolation> : (Constante de type Entier)`**

Mode d'isolation choisi :


|   |   |
| --- | --- |
| *hReadCommitted* | Seules les transactions finalisées sont visibles par les autres utilisateurs. |
| *hReadUncommitted* | Les transactions non finalisées sont visibles par les autres utilisateurs. |
| *hRepeatableRead* | L'application en transaction voit un instantané de la base à l'instant où démarre la transaction. |


## RéplicSupprimeAbonnement (Fonction)

***En anglais : ReplicDeleteSubscription***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime l'abonnement d'une réplication sur le serveur de réplication.

Cette fonction est notamment utile dans le cas d'une réplication avec réplicas différés, pour libérer les ressources sur le serveur. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouvre une réplication sur le serveur de réplication
RéplicOuvre("MaRéplication", "", "ServeurRéplication", "Nom", "motdepasse")

// Supprime la réplication
RéplicSupprimeAbonnement("MaRéplication")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = RéplicSupprimeAbonnement(<Nom de la réplication>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'abonnement de la réplication a été supprimé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur le problème rencontré, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Nom de la réplication> : (Chaîne de caractères)`**

Nom de la réplication à manipuler. Ce nom correspond :

- soit au nom du fichier ".wer", créé avec ReplicEdit (dans le cas d'une réplication connectée). Ce fichier doit être présent à côté de l'exécutable.

- soit au nom de publication (dans le cas d'un serveur de réplication).






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- La réplication doit avoir été ouverte avec la fonction [RéplicOuvre](../WDLang4/1000017194.md). 

- Si la réplication n'utilise pas le serveur de réplication (réplication en mode direct), la fonction **RéplicSupprimeAbonnement** n'a aucun effet. 

- Les fichiers locaux associés à la réplication ne sont pas supprimés.

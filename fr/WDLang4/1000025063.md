


## &lt;Source&gt;.Sécurité (Fonction)

***En anglais : &lt;Source&gt;.Security***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active ou désactive le mécanisme de sécurité automatique sur un ou plusieurs fichiers de données.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Client.Sécurité(Faux) // Désactive le mécanisme de protection
Client.Sécurité(Vrai) // Active le mécanisme de protection
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.Sécurité(<Niveau de sécurité>)`
---

**`<Résultat> : (Entier)`**

Ancien niveau de sécurité (avant exécution de la fonction **&lt;Source&gt;.Sécurité**).

**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données manipulé.

**`<Niveau de sécurité> : (Entier ou Booléen)`**

Niveau de sécurité souhaité :

- 0 ou <u><u><u><u>Faux</u></u></u></u> (par défaut) : mécanisme de sécurité désactivé. La vitesse d'écriture dans les fichiers de données est maximale.

- 1 ou <u><u><u><u>Vrai</u></u></u></u> : Mécanisme de sécurité activé : La vitesse d'écriture est plus lente qu'avec l'option HSécurité(0), mais la sécurité est assurée lors de l'écriture dans les fichiers de données.

- 2 : Mécanisme de sécurité maximal : La vitesse d'écriture dans les fichiers de données est plus lente qu'avec l'option HSécurité(1).

<a name="NOTE0_1"></a>

## Remarques


### Fonctions &lt;Source&gt;.Sécurité et [&lt;Source&gt;.ForceEcriture](../WDLang4/1000024981.md)
<a name="fonctions_docparampagetitleshort_forceecriturewdlang41000024981md_ELTPARAGRAPHE000237"></a>Les fonctions **&lt;Source&gt;.Sécurité** et [&lt;Source&gt;.ForceEcriture](../WDLang4/1000024981.md) permettent toutes les deux de forcer l'écriture physique sur le disque.

Cependant :

- La fonction **&lt;Source&gt;.Sécurité** permet de forcer l'écriture dans le ou les fichiers de données manipulés à chaque écriture. Cette écriture est automatiquement réalisée par le moteur HFSQL. Le ou les fichiers de données manipulés ne sont pas fermés lors de cette écriture. La vitesse d'écriture dans les fichiers de données est plus lente.

- La fonction [&lt;Source&gt;.ForceEcriture](../WDLang4/1000024981.md) permet de forcer l'écriture dans le ou les fichiers de données manipulés uniquement lors de l'appel de cette fonction. Le ou les fichiers de données manipulés sont fermés puis ré-ouverts lors de cette écriture. La vitesse d'écriture dans les fichiers de données n'est pas ralentie.

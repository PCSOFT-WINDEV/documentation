


## HChangeClé (Fonction)

***En anglais : HChangeKey***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Change la rubrique de parcours. L'enregistrement en cours reste inchangé.

Après l'exécution de la fonction, la suite du parcours est réalisée à partir de l'enregistrement en cours, suivant la nouvelle rubrique de parcours. La fonction **HChangeClé** peut être utilisée par exemple pour amorcer un parcours du fichier de données, après un positionnement sur un enregistrement en accès direct (fonction [HLit](../WDLang4/3044047.md) par exemple).

**Remarque** : Cette fonction peut être utilisée indifféremment sur un fichier de données, une requête ou une vue HFSQL.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Changement de la clé de parcours
SI HChangeClé(Client, Nom) = Faux ALORS
	Erreur("Sélectionnez un client dans la table")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HChangeClé(<Fichier de données> [, <Rubrique> [, <Mode de parcours>]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la clé de parcours a été modifiée,

- <u><u><u><u>Faux</u></u></u></u> en cas de problème (aucune fonction de lecture n'a précédé le changement de clé, ...) : la clé de parcours n'est pas modifiée. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données, de la vue HFSQL ou de la requête manipulé.

**`<Rubrique> : (Chaîne de caractères optionnelle)`**

Nom de la nouvelle rubrique de parcours. 
Si ce nom n'est pas spécifié, la fonction **HChangeClé** va utiliser la dernière rubrique de parcours spécifiée sur ce fichier par la dernière fonction de gestion HFSQL (fonction commençant par la lettre H).

**`<Mode de parcours> : (Constante optionnelle de type Entier)`**

Caractéristique du parcours du fichier de données sur la nouvelle clé :


|   |   |
| --- | --- |
| *hRespecteFiltre* | Le parcours prendra en compte le filtre mis en place avec la fonction [HFiltre](../WDLang4/3044100.md), même si la clé de parcours utilisée n'est pas optimisée pour le filtre. <br>**Rappel** : la fonction [HFiltre](../WDLang4/3044100.md) renvoie la clé de parcours optimisée pour le filtre.<br>**Attention** : Dans ce cas, sur des fichiers de données de taille importante, des problèmes de performances peuvent survenir. |


![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Le paramètre &lt;Mode de parcours&gt; ne peut pas être utilisé.

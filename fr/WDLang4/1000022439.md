


## MongoGridFSRécupèreFichier (Fonction)

***En anglais : MongoGridFSGetFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Récupère sur le poste local un fichier identifié par son nom et présent dans un système de fichiers MongoDB.

**Remarque** : Pour récupérer en local un fichier identifié à partir de son identifiant, utilisez la fonction [MongoGridFSRécupèreFichierParID](../WDLang4/1000022595.md). 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MongoGridFSRécupèreFichier(cnx.BaseDeDonnées["CRM"], "produit.png", ...
		fRepDonnées() + ["\"] + "produit.png")
```
<a name="Exemple2"></a>

```wl
sRep est une chaîne = fRepSélecteur(SysRep(srTéléchargements), ...
	"Sélectionnez le répertoire de téléchargement")
SI sRep<>"" ALORS
	MongoGridFSRécupèreFichier(gclGridFS, TABLE_GridFS.COL_Nom, ...
		ComplèteRep(sRep) + TABLE_GridFS.COL_Nom)
	ToastAffiche("Récupération terminée avec succès.")
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = MongoGridFSRécupèreFichier(<Système de fichiers> , <Nom du fichier à récupérer> , <Nom du fichier local>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée,

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'identifier l'erreur.




**`<Système de fichiers> : (Variable de type mongoGridFS ou de type mongoBase)`**

Système de fichiers de la base MongoDB à manipuler. Correspond : 

- au nom de la variable de type [mongoGridFS](../WDLang4/1000022416.md) correspondant au système de fichiers Mongo.

- au nom de la variable de type [mongoBase](../WDLang4/1000022410.md) correspondant à la base MongoDB. Dans ce cas, le système de fichiers utilise le préfixe "fs". 




**`<Nom du fichier à récupérer> : (Chaîne de caractères)`**

Nom du fichier présent dans le système de fichiers MongoDB à récupérer sur le poste local.

**`<Nom du fichier local> : (Chaîne de caractères)`**

Chemin et nom du fichier récupéré sur le poste local.

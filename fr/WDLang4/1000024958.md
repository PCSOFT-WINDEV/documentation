


## &lt;Source&gt;.Ferme (Fonction)

***En anglais : &lt;Source&gt;.Close***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ferme soit un fichier de données, soit tous les fichiers de données ouverts par l'utilisateur en cours : tous les fichiers de données physiques correspondant sont fermés pour l'utilisateur en cours. Pour des fichiers de données HFSQL, les fichiers fermés sont :

- le fichier de données (.FIC),

- le fichier d'index (.NDX) s'il existe,

- le fichier Mémo (.MMO) s'il existe.

- le fichier d'index full-text (.FTX) s'il existe.




**A la fermeture du fichier de données** : 

- Les blocages (sur le fichier de données et/ou sur certains de ses enregistrements) sont débloqués.

- Les filtres définis sur le fichier de données avec la fonction [&lt;Source&gt;.Filtre](../WDLang4/1000024960.md) sont automatiquement supprimés. 

- Les parcours en cours sur le fichier de données sont annulés. 

- Les recherches en cours sur le fichier de données sont annulées. 




Après la fermeture du fichier de données, les réassignations effectuées par les fonctions [&lt;Source&gt;.ChangeNom](../WDLang4/1000024201.md), [&lt;Source&gt;.ChangeRep](../WDLang4/1000024202.md) et [HSubstRep](../WDLang4/3044028.md) sont conservées.

Si une requête a été exécutée sur le fichier de données, il est nécessaire d'appeler la fonction [&lt;Source&gt;.AnnuleDéclaration](../WDLang4/1000024192.md) sur la requête pour que la fermeture soit effective.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Fermeture du fichier de données "Fournisseur"
Fournisseur.Ferme()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.Ferme()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier de données a été fermé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [HErreur](../WDLang4/3044088.md) permet d'obtenir plus d'informations sur le problème rencontré.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données à fermer.

<a name="NOTE0_1"></a>

## Remarques


### Fermeture des tables et de la connexion
<a name="fermeture_des_tables_connexion_ELTPARAGRAPHE000271"></a>Lorsque toutes les tables utilisant la même connexion (tables ouvertes sur la même base de données avec les mêmes paramètres de connexion) sont fermées, la connexion est fermée elle aussi automatiquement.

Pour fermer une connexion, utilisez la fonction [&lt;Variable Connexion&gt;.FermeConnexion](../WDLang4/1000023941.md).




## &lt;Source&gt;.JournalRedémarre (Fonction)

***En anglais : &lt;Source&gt;.LogRestart***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Redémarre la journalisation du fichier de données. Cette journalisation a été arrêtée grâce à la fonction [&lt;Source&gt;.JournalStop](../WDLang4/1000025008.md).

**Remarque** : La journalisation n'est pas disponible lors de l'accès à une base de données via .


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Client.JournalInfo("début impression de " + Critère)
Client.JournalStop() // Stopper le journal
Client.LitPremier(Critère)
TANTQUE Client.Trouve()
	// Traitement
	Client.LitSuivant()
FIN

Client.JournalRedémarre() // Redémarrer le journal
Client.JournalInfo("fin impression de" + Critère) // Ecrire
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.JournalRedémarre()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données journalé dont la journalisation doit redémarrer.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Lors du redémarrage du journal, le fichier de données doit être dans son état initial, c'est-à-dire l'état dans lequel il était avant l'utilisation de la fonction [&lt;Source&gt;.JournalStop](../WDLang4/1000025008.md).

- Entre les fonctions [&lt;Source&gt;.JournalStop](../WDLang4/1000025008.md) et **&lt;Source&gt;.JournalRedémarre**, les informations ne seront pas sauvegardées dans le journal.

- La fonction **&lt;Source&gt;.JournalRedémarre** est équivalente à l'utilisation de la fonction [&lt;Source&gt;.GèreJournal](../WDLang4/1000024986.md) avec le paramètre <u><u><u><u>Vrai</u></u></u></u>.

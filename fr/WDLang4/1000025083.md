


## &lt;Source&gt;.VersRubrique (Fonction)

***En anglais : &lt;Source&gt;.ToItem***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Affecte la valeur indiquée à une rubrique de l'enregistrement en cours.

**Remarque** : L'utilisation de cette fonction est très rare. Il est conseillé d'utiliser l'affectation directe des rubriques par la syntaxe &lt;NomFichier&gt;.&lt;NomRubrique&gt; = &lt;Valeur&gt;. 
Par exemple : Client.Nom = "Dupont".

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Affecte à la seconde rubrique du fichier de données client la valeur Dupont
Client.VersRubrique(2, "Dupont")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Affectation d'une valeur à une rubrique (une valeur pour chaque rubrique d'une rubrique tableau)

`<Source>.VersRubrique(<Ordre de la rubrique> [, <Indice>] , <Valeur>)`
---

**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données manipulé.

**`<Ordre de la rubrique> : (Entier)`**

Numéro d'ordre physique de la rubrique dans le fichier de données.

**`<Indice> : (Entier optionnel)`**

Indice de la rubrique tableau à affecter. Si une rubrique non tableau est affectée, &lt;Indice&gt; doit valoir 1.

**`<Valeur> : (Chaîne de caractères ou autre)`**

Valeur à affecter à la rubrique. Si la valeur doit être affectée à une rubrique tableau : 

- si le paramètre &lt;Indice&gt; a été utilisé, la valeur sera affectée à la rubrique indicée spécifiée. 

- si aucun des paramètres précédent n'a été utilisé, &lt;Valeur&gt; doit contenir toutes les valeurs de la rubrique tableau, séparées par TAB.


![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Il est impossible d'affecter des rubriques tableau.


<a name="SYNTAXE2"></a>

### Affectation d'une valeur à une rubrique (une valeur pour une rubrique tableau)

`<Source>.VersRubrique(<Ordre de la rubrique> [, <Séparateur Tableau>] , <Valeur>)`
---

**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données manipulé.

**`<Ordre de la rubrique> : (Entier)`**

Numéro d'ordre physique de la rubrique dans le fichier de données.

**`<Séparateur Tableau> : (Chaîne de caractères optionnelle)`**

Chaîne utilisée pour séparer les valeurs de chaque rubrique tableau. Par défaut, TAB est utilisée.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Il est impossible d'affecter des rubriques tableau.

**`<Valeur> : (Chaîne de caractères ou autre)`**

Valeur à affecter à la rubrique. Si la valeur doit être affectée à une rubrique tableau : 

- si le paramètre &lt;Séparateur Tableau&gt; a été utilisé, &lt;Valeur&gt; doit contenir toutes les valeurs de la rubrique tableau, séparées par le séparateur indiqué

- si aucun des paramètres précédent n'a été utilisé, &lt;Valeur&gt; doit contenir toutes les valeurs de la rubrique tableau, séparées par TAB.


![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Il est impossible d'affecter des rubriques tableau.

## Remarques


### Ordre des rubriques dans le fichier de données
<a name="ordre_des_rubriques_dans_fichier_donnees_ELTPARAGRAPHE000253"></a>L'ordre des rubriques dans le fichier de données commence à 1. L'ordre d'une rubrique peut être obtenu via la propriété [Indice](../Proprietes/2512040.md).

### Langage externe
<a name="langage_externe_ELTPARAGRAPHE000268"></a>

- La &lt;Valeur&gt; doit être passée sous forme de chaîne.

- Il n'est pas possible d'affecter un mémo binaire avec la fonction **&lt;Source&gt;.VersRubrique**.

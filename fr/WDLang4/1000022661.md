


## &lt;Variable Connexion&gt;.FinInterditAccèsBaseDeDonnées (Fonction)

***En anglais : &lt;Connection variable&gt;.EndNoDatabaseAccess***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ré-autorise l'accès (c'est-à-dire la connexion) à une ou plusieurs bases de données accessibles par une connexion.

**Remarque** : L'accès aux bases de données avait été précédemment interdit grâce à la fonction [&lt;Variable Connexion&gt;.InterditAccèsBaseDeDonnées](../WDLang4/1000022690.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion
MaConnexion est une Connexion

SI MaConnexion.InterditAccèsBaseDeDonnées("BaseClient") = Vrai ALORS
	// Traitement spécifique
	...
	MaConnexion.FinInterditAccèsBaseDeDonnées("BaseClient") 
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Connexion>.FinInterditAccèsBaseDeDonnées([<Base de données>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'accès à la base de données spécifiée (ou à toutes les bases de données accédées par la connexion) est rétabli, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion contenant la base de données dont l'accès doit être rétabli.

**`<Base de données> : (Chaîne de caractères optionnelle)`**

Nom de la base de données pour laquelle l'accès doit être rétabli.
Si ce paramètre correspond à "\*", l'accès sera rétabli pour toutes les bases de données.
Si ce paramètre n'est pas spécifié, l'accès sera rétabli pour toutes les bases de données accessibles par &lt;Connexion&gt;.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Connexion à une base de données bloquée
<a name="connexion_une_base_donnees_bloquee_ELTPARAGRAPHE000258"></a>Dans certains cas (coupure de courant par exemple), il se peut que les connexions à une base de données soient refusées. Par exemple, si une coupure de courant survient pendant l'exécution de ce code (avant la fonction [&lt;Variable Connexion&gt;.InterditAccèsBaseDeDonnées](../WDLang4/1000022690.md)) :

```wl
// Description de la connexion
MaConnexion est une Connexion

// Ouverture de la connexion
MaConnexion.OuvreConnexion()
SI MaConnexion.InterditAccèsBaseDeDonnées("BaseClient") = Vrai ALORS
	Info("Il est impossible de se connecter à la base de données")
FIN
```

Pour autoriser à nouveau les accès, il suffit de faire une connexion sans préciser de base de données, puis d'utiliser la fonction **&lt;Variable Connexion&gt;.FinInterditAccèsBaseDeDonnées**.
<a name="NOTE0_2"></a>


### Autoriser l'accès à une base de données bloquée par le Centre de Contrôle HFSQL
<a name="autoriser_acces_une_base_donnees_bloquee_par_centre_controle_hfsql_ELTPARAGRAPHE000285"></a>Lorsque les connexions à une base de données HFSQL Client/Serveur sont refusées, il est également possible d'autoriser à nouveau l'accès via le [Centre de Contrôle HFSQL](../CCHF/35400015.md). Pour cela, il suffit de réaliser les manipulations suivantes :

- Se connecter en tant qu'administrateur au serveur sans indiquer la base de données,

- Afficher l'onglet "Connexions" du serveur,

- Dans le ruban, dans le groupe "Connexions", dérouler le bouton "Blocage des connexions" et sélectionner l'option "Autoriser toutes les connexions".

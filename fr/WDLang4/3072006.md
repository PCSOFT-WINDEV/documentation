
## SQLFormatColonne (Fonction)

***En anglais : SQLColumnFormat***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie les différents types de colonnes supportés :

- soit par la source de données identifiée par la fonction [SQLConnecte](../WDLang4/3072005.md)

- soit par la base de données accédée par une connexion particulière (connexion décrite dans l'analyse ou décrite avec les fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md)).





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Ch est une chaîne
Ch = SQLFormatColonne()
// La chaîne Ch contient par exemple :
// "CHAR" + TAB + "9" + RC + "VARCHAR" + TAB + "20" + RC
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = SQLFormatColonne([<Nom de la connexion>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste des types supportés :

- soit par la source de données (définie par [SQLConnecte](../WDLang4/3072005.md))

- soit par &lt;Nom de la connexion&gt; si ce paramètre est spécifié.




Cette liste est de la forme : &lt;Type de la colonne SQL&gt; + TAB + &lt;Type WINDEV/WEBDEV&gt; + RC. Pour plus de détails, consultez [les remarques](#NOTE0_1).

**`<Nom de la connexion> : (Chaîne de caractères optionnelle)`**

Si ce paramètre est précisé, permet de spécifier la connexion pour laquelle le format des colonnes doit être retourné. Ce nom correspond alors soit à une connexion décrite dans l'analyse, soit à une connexion décrite avec les fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

Si ce paramètre n'est pas précisé, la source de données manipulée sera la source de données définie avec la fonction [SQLConnecte](../WDLang4/3072005.md).

## Remarques




### Liste des types supportés
<a name="liste_des_types_supportes_ELTPARAGRAPHE000096"></a>Le résultat de la fonction **SQLFormatColonne** est de la forme :

```txt
<Type de la colonne SQL> + TAB + <Type WinDev> + RC
```


Le tableau suivant donne les équivalences des types SQL et WINDEV/WEBDEV.

| Type SQL | Correspond à | Type WINDEV/WEBDEV |
| --- | --- | --- |
| SQL_WLONGVARCHAR | Mémo Unicode | -2 |
| SQL_WCHAR | Chaîne Unicode | 0 |
| SQL_BIT | Bit | 1 |
| SQL_TINYINT | Entier | 2 |
| SQL_BIGINT | Entier | 3 |
| SQL_LONGVARBINARY | Mémo de type binaire | 4 |
| SQL_VARBINARY | Chaîne binaire de taille fixe | 5 |
| SQL_BINARY | Binaire | 6 |
| SQL_LONGVARCHAR | Mémo de type chaîne ASCII | 7 |
| SQL_CHAR | Chaîne de taille fixe | 9 |
| SQL_NUMERIC | Numérique | 10 |
| SQL_DECIMAL | Nombre décimal | 11 |
| SQL_INTEGER | Entier | 12 |
| SQL_SMALLINT | Entier | 13 |
| SQL_FLOAT | Réel | 14 |
| SQL_REAL | Réel | 15 |
| SQL_DOUBLE | Réel | 16 |
| SQL_DATE | Date | 17 |
| SQL_TIME | Heure | 18 |
| SQL_TIMESTAMP | Date + Heure | 19 |
| SQL_VARCHAR | Chaîne de taille variable | 20 |

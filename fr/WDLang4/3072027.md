


## SQLListeSource (Fonction)

***En anglais : SQLListSource***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Liste les sources de données déclarées sur un site local. Il n'est pas nécessaire d'être connecté à la base pour réaliser cette opération.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Afficher les sources de données accessibles dans un champ Liste
ListeAjoute(LISTE_ListeData, SQLListeSource())

// Afficher les bases de données accessibles dans un champ Liste 
ListeAjoute(LISTE_ListeData, SQLListeSource("BASE"))
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Lister les sources de données disponibles pour toutes les bases

`<Résultat> = SQLListeSource()`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste de toutes les sources de données disponibles. Les différents éléments sont séparés par des RC et peuvent être ajoutés directement dans une liste (fonction [ListeAjoute](../WDLang1/3049004.md)).
![ODBC](https://doc.pcsoft.fr/ext/images/fr/ODBC.png) Liste de toutes les sources de données définies dans l'administrateur ODBC du poste en cours.


<a name="SYNTAXE2"></a>

### Lister les bases de données disponibles

`<Résultat> = SQLListeSource(<"BASE">)`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste de toutes les types de base disponibles sur le poste en cours. Les différents éléments sont séparés par des RC et peuvent être ajoutés directement dans une liste (fonction [ListeAjoute](../WDLang1/3049004.md)). Les types de base possibles sont :


|   |   |
| --- | --- |
| ACCESS | Le provider Microsoft Jet 4 est installé sur le poste en cours ainsi que le MDAC. |
| DB2 | Le Connecteur Natif DB2 est installé sur le poste en cours. |
| HYPER FILE | Accès aux base de données HFSQL. |
| HIVE | Le Connecteur Natif Hive est installé sur le poste en cours. |
| INFORMIX | Le Connecteur Natif INFORMIX est installé sur le poste en cours. |
| MariaDB | Le Connecteur Natif MariaDB est installé sur le poste en cours. |
| MYSQL | Le Connecteur Natif MYSQL est installé sur le poste en cours. |
| ODBC | La DLL ODBC32 est installée sur le poste en cours. |
| OLEDB | Le MDAC est installé sur le poste en cours. |
| ORACLE | Le Connecteur Natif ORACLE est installé sur le poste en cours. |
| POSTGRESQL | Le Connecteur Natif POSTGRESQL est installé sur le poste en cours. |
| PROGRESS | Le Connecteur Natif PROGRESS est installé sur le poste en cours. |
| SQL AZURE | Le Connecteur Natif SQL AZURE est installé sur le poste en cours. |
| SQL SERVER | Le Connecteur Natif SQL SERVER est installé sur le poste en cours.<br>**Remarque** : La valeur de la variable [H.ModeSQLServer](../WDLang4/3044276.md) est prise en compte. Si l'ancien Connecteur Natif est forcé et n'est pas présent sur le poste, le Connecteur Natif SQL Server ne sera pas listé. Si le nouveau Connecteur Natif est forcé et n'est pas présent sur le poste, le Connecteur Natif SQL Server ne sera pas listé. |
| SYBASE | Le Connecteur Natif SYBASE est installé sur le poste en cours. |



**`<"BASE"> : (Chaîne de caractères)`**

"BASE" : Chaîne de caractères permettant de lister les bases de données disponibles sur le poste en cours.  


<a name="SYNTAXE3"></a>

### Lister les sources de données disponibles pour un type spécifique

`<Résultat> = SQLListeSource(<"BASE"> , <Type de base>)`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste de toutes les sources de données disponibles pour le type spécifié. Les différents éléments sont séparés par des RC et peuvent être ajoutés directement dans une liste (fonction [ListeAjoute](../WDLang1/3049004.md)). Les résultats possibles en fonction du type de base sont les suivants :


|   |   |
| --- | --- |
| Access | Chaîne vide. |
| HYPER FILE | Chaîne vide. Il est conseillé d'utiliser la fonction [HListeAnalyse](../WDLang4/3044049.md) pour obtenir la liste des analyses. |
| ODBC | Liste des bases ODBC définies dans l'administrateur ODBC du poste en cours. |
| OLEDB | Chaîne vide. |



**`<"BASE"> : (Chaîne de caractères)`**

"BASE" : Chaîne de caractères permettant de lister les sources de données disponibles pour un type spécifique.

**`<Type de base> : (Chaîne de caractères)`**

Type des bases recherchées : "ODBC", ...




## &lt;Source&gt;.InitAbonné (Fonction)

***En anglais : &lt;Source&gt;.InitSubscriber***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Initialise la plage d'identifiants automatiques d'un fichier de description d'un réplica abonné (extension ".RPL"). **Cette fonction est utilisable uniquement avec la réplication journalée**. 

L'initialisation de la plage d'identifiants automatiques est réalisée :

- automatiquement, lors de la création d'un fichier de données (fonctions [&lt;Source&gt;.Création](../WDLang4/1000024207.md), [&lt;Source&gt;.CréationSiInexistant](../WDLang4/1000024208.md)).

- manuellement, avec la fonction **&lt;Source&gt;.InitAbonné**.




La fonction **&lt;Source&gt;.InitAbonné** doit être appelée uniquement lors de l'installation des fichiers de données chez l'abonné.

**Attention** : Les fichiers de données doivent être ouverts avant l'utilisation de la fonction **&lt;Source&gt;.InitAbonné**.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Initialise la plage d'ID Auto du fichier de données Client
Client.InitAbonné("C:\DATA\Replic\AbonneParis.RPL")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.InitAbonné(<Réplica Abonné>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'initialisation a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données HFSQL à utiliser.

**`<Réplica Abonné> : (Chaîne de caractères)`**

Nom et chemin complet du réplica abonné. Ce paramètre peut correspondre à un chemin de type UNC.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Plage d'identifiants
<a name="plage_identifiants_ELTPARAGRAPHE000227"></a>Les identifiants automatiques des fichiers des réplicas abonnés utilisent une plage d'identifiants différente de celle du fichier de réplica maître et des autres fichiers de réplicas abonnés.

Cette plage d'identifiants automatiques est inscrite dans l'entête de chaque fichier de données (extension ".FIC").
Cette plage est initialisée :

- lors de la création d'un fichier de données (fonctions [&lt;Source&gt;.Création](../WDLang4/1000024207.md), [&lt;Source&gt;.CréationSiInexistant](../WDLang4/1000024208.md)).

- grâce à l'outil [WDReplic](../WDReplic/3520016.md).

- lors de l'utilisation de la fonction **&lt;Source&gt;.InitAbonné**.




La plage d'identifiants peut être connue grâce aux propriétés [IDAutoMin](../Proprietes/2512039.md) et [IDAutoMax](../Proprietes/2512023.md).

Pour plus de détails, consultez les [conséquences de la mise en place de la réplication](../WDLang4/3044246.md).
<a name="NOTE0_2"></a>


### Réplication et journalisation
<a name="replication_journalisation_ELTPARAGRAPHE000274"></a>Si la fonction **&lt;Source&gt;.InitAbonné** est utilisée, il est également nécessaire d'initialiser l'emplacement des journaux et des tables d'opérations (fonction [&lt;Source&gt;.ChangeRepJnl](../WDLang4/1000024203.md), propriétés [RépertoireJournal](../Proprietes/2512075.md) et [RépertoireJournalOpération](../Proprietes/2512076.md)).

**Rappel** : La réplication journalée se base sur les journaux et tables d'opérations des fichiers de données. Si l'emplacement des journaux et des tables d'opérations est erroné, la réplication journalée ne peut pas fonctionner.

<a name="Utilisation"></a>
<a name="description"></a>
Trie une vue HFSQL en créant un index sur une rubrique de la vue.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HTrieVue(VueClient84, "Ville,Nom")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HTrieVue([<Vue> [, <Rubrique de tri>]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la vue a été triée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [HErreur](../WDLang4/3044088.md) permet d'obtenir plus d'informations sur le problème rencontré.




**`<Vue> : (Chaîne de caractères optionnelle)`**

Nom de la vue manipulée. Si ce nom n'est pas spécifié, la fonction **HTrieVue** va manipuler la dernière vue utilisée par la dernière fonction de gestion HFSQL (fonction commençant par la lettre H).

**`<Rubrique de tri> : (Chaîne de caractères optionnelle)`**

Liste des rubriques de tri de la vue. Les rubriques sont séparées par une virgule ou *RC*. Le sens du tri peut être précisé pour chaque rubrique en la préfixant par "+" (croissant) ou "-" (décroissant). Par défaut, le tri est croissant ("+"). Si ce paramètre n'est pas spécifié, **HTrieVue** se réfère à la dernière rubrique de la vue utilisée.
Les formats possibles de cette liste sont :

- "[+/-]&lt;1ère rubrique&gt; [, [+/-]&lt;2ème rubrique&gt; ]"

- "[+/-]&lt;1ère rubrique&gt; [*RC* [+/-]&lt;2ème rubrique&gt; ]"




Le tri est appliqué d'abord selon la première rubrique de tri, puis selon la seconde, ... L'ordre de tri appliqué est le suivant : dans un tri croissant, "a" &lt; "u" &lt; "é".
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Une seule rubrique de tri doit être spécifiée dans ce paramètre.

## Remarques


### Rubrique de tri
<a name="rubrique_tri_ELTPARAGRAPHE000147"></a>Un index peut être créé sur n'importe quelle rubrique de la vue. Si la rubrique était clé dans le fichier de données, la relation d'ordre qui lui a été donnée sous l'analyse est utilisée (sensibilité à la casse, à la ponctuation, etc.).
<a name="NOTE0_2"></a>

### Trier une vue sur plusieurs rubriques
<a name="trier_une_vue_sur_plusieurs_rubriques_ELTPARAGRAPHE000158"></a>Pour trier une vue sur plusieurs rubriques, il est possible d'afficher les enregistrements de la vue dans un champ Table mémoire et d'utiliser la fonction [TableTrie](../WDLang1/3074035.md).

**Attention** : Cette solution est utilisable uniquement pour les fichiers de données au format HFSQL Classic ou Client/Serveur.

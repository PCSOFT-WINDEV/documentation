


## hdfsCopieDepuis (Fonction)

***En anglais : hdfsCopyFrom***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Copie un fichier situé sur un serveur Hadoop vers la machine locale.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connexion
MaConnexion est une hdfsConnexion
MaConnexion.namenode = "MonServeur"
MaConnexion.Utilisateur = "Admin"
hdfsOuvreConnexion(MaConnexion)
// Copie du serveur vers la machine locale
hdfsCopieDepuis(MaConnexion, "/tmp/test.data", "c:\temp\test.data")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = hdfsCopieDepuis(<Connexion> , <Fichier source> , <Fichier destination> [, <Jauge>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la copie a été réalisée,

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type hdfsConnexion)`**

Nom de la variable de type [hdfsConnexion](../WDLang4/1000021650.md) correspondant à la connexion au serveur Hadoop.

**`<Fichier source> : (Chaîne de caractères)`**

Chemin absolu du fichier situé sur le serveur Hadoop. Ce chemin est sensible à la casse.

**`<Fichier destination> : (Chaîne de caractères)`**

Chemin du fichier local correspondant à la destination (260 caractères maximum). Ce paramètre peut correspondre à :

- un nom de fichier. Ce fichier sera automatiquement créé dans le répertoire en cours (connu avec la fonction [fRepEnCours](../WDLang1/3036022.md)).

- un nom de fichier avec son chemin (complet ou relatif). Un chemin de type UNC peut être utilisé.




**`<Jauge> : (Nom de champ optionnel)`**

Nom du champ Jauge permettant de visualiser l'avancement de la récupération du fichier.

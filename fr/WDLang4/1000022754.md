


## &lt;Variable Connexion&gt;.SimuleRéseau (Fonction)

***En anglais : &lt;Connection variable&gt;.SimulateNetwork***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Simule le fonctionnement de HFSQL Client/Serveur : 

- sur un réseau de type ADSL, 3G, 4G, 5G ou via un cloud. 

- sur un réseau lent avec des paramètres personnalisés.




**Remarque** : Cette fonction permet de tester en local ou en LAN, une application utilisant HFSQL dans les conditions d'un réseau ADSL, d'un réseau 3G, 4G, 5G, cloud ou avec des paramètres personnalisés. Cette fonction implique un ralentissement sur toutes les communications entre le client et le serveur HFSQL.

**Cette fonction s'utilise dans le cadre du débogage et de l'optimisation d'une application.**



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion
MaConnexion est une Connexion

// Simulation du réseau 3G
MaConnexion.SimuleRéseau(h3G)
AnalyseurDébut()
Proc_Traitement() // Traitement dont on veut analyser la performance en 3G
AnalyseurFin()
// Retour à un fonctionnement normal
MaConnexion.SimuleRéseau(hAucun)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Simuler le fonctionnement de HFSQL Client/Serveur sur un réseau prédéfini

`<Résultat> = <Connexion>.SimuleRéseau(<Type de réseau>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la simulation est lancée, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion à utiliser.

**`<Type de réseau> : (Constante de type Entier)`**

Type de réseau à simuler : 


|   |   |
| --- | --- |
| *h3G* | Type de réseau à simuler : 3G<br>Caractéristiques (estimation) : Latence : 130ms, Débit : 1 Mbps |
| *h4G* | Type de réseau à simuler : 4G<br>Caractéristiques (estimation) : Latence : 40ms, Débit : 30 Mbps |
| *h5G* | Type de réseau à simuler : 5G<br>Caractéristiques (estimation) : Latence : 10ms, Débit : 100 Mbps |
| *hADSL* | Type de réseau à simuler : ADSL<br>Caractéristiques (estimation) : Latence : 40ms, Débit : 8 Mbps |
| *hAucun* | Permet de désactiver la simulation d'un réseau spécifique. |
| *hCloud* | Type de réseau à simuler : Cloud.<br>Caractéristiques (estimation) : Latence : 10ms, Débit : 400 Mbps<br>Les valeurs s'appliquent lorsque l'application est utilisée depuis le même continent que le datacenter, avec une connexion internet par fibre. Si l'application est utilisée depuis un autre continent, ou en wifi, il est conseillé d'utiliser la seconde syntaxe de la fonction en précisant la latence et le débit dont dispose la connexion. |




<a name="SYNTAXE2"></a>

### Simuler le fonctionnement de HFSQL Client/Serveur sur un réseau personnalisé

`<Résultat> = <Connexion>.SimuleRéseau(<Latence> , <Débit>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la simulation est lancée, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion à utiliser.

**`<Latence> : (Entier)`**

Latence simulée exprimée en millisecondes. Cette latence est ajoutée à la latence du réseau actuellement utilisé.

**`<Débit> : (Réel)`**

Débit en Mégabits par seconde correspondant au temps nécessaire pour transmettre une quantité de données (en supposant que le réseau actuel à un débit infini).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Cette fonction ne doit pas être utilisée dans une application déployée. Cette fonction s'utilise dans le cadre du débogage et de l'optimisation d'une application.

- Cette fonction augmente la latence et diminue le débit des connexions réseau utilisées pour HFSQL Client/Serveur.

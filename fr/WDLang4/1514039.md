
## Description de Liaison (Type de variable)

***En anglais : Link Description (Type of variable)***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Une variable de type **Description de Liaison** permet de décrire par programmation une liaison entre deux fichiers de données. La description de chaque liaison est validée par la fonction [HDécritLiaison](../WDLang4/3044191.md).

Après cette validation :

- Cette liaison pourra être manipulée comme n'importe quelle liaison décrite dans l'analyse.

- La variable de type "Description de Liaison" est réinitialisée et pourra être utilisée pour décrire par programmation une autre liaison.




**Remarque** : Pour ajouter une liaison dans une base de données HFSQL Client/Serveur, utilisez la fonction [HAjouteLiaison](../WDLang4/3044312.md).




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MaLiaison est une Description de Liaison
MaLiaison1, MaLiaison2 sont des Descriptions de Liaison
```


<a name="Exemple2"></a>



```wl
// Description et validation des fichiers de données Ville et Client
...

// Description de la liaison "HABITE"
MaLiaison est une Description de Liaison
MaLiaison.Nom = "HABITE"
MaLiaison.FichierSource = "VILLE"
MaLiaison.FichierRelié = "CLIENT"
MaLiaison.CléSource = "NOMVILLE"
MaLiaison.CléReliée = "VILLECLI"
// Liaison par défaut (0-1, 0-1)

// Validation de la description de la liaison "HABITE"
HDécritLiaison(MaLiaison)
```

## Remarques




### Propriétés spécifiques à la description de liaisons
<a name="proprietes_specifiques_description_liaisons_ELTPARAGRAPHE000096"></a>Les propriétés spécifiques à la description de liaisons sont les suivantes :

**Remarque** : Les propriétés en gras suivies d'un astérisque doivent obligatoirement être initialisées lors de la description d'une liaison par programmation.

| Nom de la propriété | Effet |
| --- | --- |
| [CardinalitéReliéeMax](../Proprietes/2512011.md) | Cardinalité maximale de la rubrique reliée dans une liaison |
| [CardinalitéReliéeMin](../Proprietes/2512002.md) | Cardinalité minimale de la rubrique reliée dans une liaison |
| [CardinalitéSourceMax](../Proprietes/2512005.md) | Cardinalité maximale de la rubrique source dans une liaison |
| [CardinalitéSourceMin](../Proprietes/2512006.md) | Cardinalité minimale de la rubrique source dans une liaison |
| **[CléReliée](../Proprietes/2512007.md) ( \* )** | Rubrique du fichier relié utilisée dans la liaison |
| **[CléSource](../Proprietes/2512003.md) ( \* )** | Rubrique du fichier source utilisée dans la liaison |
| **[FichierRelié](../Proprietes/2512031.md) ( \* )** | Nom du fichier relié utilisé dans la liaison |
| **[FichierSource](../Proprietes/2512034.md) ( \* )** | Nom du fichier source utilisé dans la liaison |
| **[Nom](../Proprietes/2510082.md) ( \* )** | Nom de la liaison |
| [RègleModification](../Proprietes/2512051.md) | Règle de modification (gestion de l'intégrité) utilisée dans une liaison |
| [RègleSuppression](../Proprietes/2512083.md) | Règle de suppression (gestion de l'intégrité) utilisée dans une liaison |


Par défaut, les cardinalités de la liaison sont (0-1) / (0-1).


<a name="NOTE0_2"></a>




### Comment décrire des fichiers de données par programmation ? 
<a name="comment_decrire_des_fichiers_donnees_par_programmation_ELTPARAGRAPHE000180"></a>Pour décrire des fichiers de données par programmation, il faut :

1. Déclarer des variables de type "[Description de fichier](../WDLang4/1514065.md)", "[Description de Rubrique](../WDLang4/1514071.md)", "[Description d'IndexFullText](../WDLang4/1000017461.md)" et "**Description de Liaison**" (si nécessaire).

2. Pour chaque fichier de données :

	- Décrire les caractéristiques du fichier de données grâce aux [propriétés HFSQL](../Proprietes/2512070.md).

	- Décrire les caractéristiques des rubriques grâce aux [propriétés HFSQL](../Proprietes/2512070.md).

	- Valider la description de chaque rubrique (fonction [HDécritRubrique](../WDLang4/3044233.md)).

	- Décrire si nécessaire les caractéristiques des index full-text grâce aux [propriétés HFSQL](../Proprietes/2512070.md).

	- Valider la description de chaque index full-text (fonction [HDécritIndexFullText](../WDLang4/1000017407.md)).

	- Valider la description du fichier de données (fonction [HDécritFichier](../WDLang4/3044206.md)).




3. Décrire si nécessaire les caractéristiques des liaisons grâce aux [propriétés HFSQL](../Proprietes/2512070.md).

4. Valider si nécessaire la description de chaque liaison (fonction [HDécritLiaison](../WDLang4/3044191.md)).





<a name="NOTE0_3"></a>




### Manipulation des liaisons d'un fichier de données décrit par programmation
<a name="manipulation_des_liaisons_fichier_donnees_decrit_par_programmation_ELTPARAGRAPHE000234"></a>Lors de la création d'une liaison sous l'éditeur d'analyses, le nom de cette liaison est automatiquement reconnu par le compilateur.

Lors de la création d'une liaison à l'aide d'une variable de type "Description de Liaison", le nom de cette liaison est défini grâce à la propriété [Nom](../Proprietes/2510082.md). Ce nom n'est pas automatiquement reconnu par le compilateur. L'utilisation de ce nom pour désigner la liaison provoque une erreur de compilation.

Pour que le nom de la liaison créé par programmation soit reconnu par le compilateur, il est nécessaire de déclarer ce nom grâce au mot-clé [Externe](../Motscles/1511013.md). Cette liaison pourra alors être manipulée par son nom sous l'éditeur de code.

Par exemple :


```wl
MaLiaison est une Description de Liaison
...
MaLiaison.Nom = "HABITE"
...
HDécritLiaison("HABITE")
...
EXTERNE HABITE
...
Cardinalité = HABITE.CardinalitéReliéeMax
```

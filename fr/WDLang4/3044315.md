


## Gestion des groupes et des utilisateurs (HFSQL Client/Serveur)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000350"></a>
Pour gérer simplement les utilisateurs et les groupes d'utilisateurs, le moteur HFSQL Client/Serveur utilise plusieurs variables permettant de définir les caractéristiques des groupes et des utilisateurs.











<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Variables de gestion des utilisateurs
<a name="variables_gestion_des_utilisateurs_ELTTEXTE000380"></a>


### Quelles sont les variables ?
<a name="quelles_sont_les_variables_ELTPARAGRAPHE000058"></a>Les variables suivantes permettent de gérer les utilisateurs :


|   |   |
| --- | --- |
| HUtilisateur.ActiveDirectory | Booléen (Faux par défaut).<br><br>Si cette variable est à Vrai, l'utilisateur utilise une authentification via Active Directory. |
| HUtilisateur.CompteActif | Booléen (Vrai par défaut).<br><br>Si cette variable est à faux, le compte existe mais n'est pas actif. Permet par exemple de ne pas se servir d'un compte utilisateur pendant une période de congés. |
| HUtilisateur.DateExpiration | Date en Lecture seule.<br><br>Date d'expiration du mot de passe. Cette date est déduite de la durée de validité du mot de passe et de la date de sa dernière modification. |
| HUtilisateur.Description | Chaîne de caractères. <br><br>Description de l'utilisateur. |
| HUtilisateur.DuréeDeValidité | Entier (par défaut 0, pas de durée de validité).<br><br>Durée de validité du mot de passe exprimée en jours. Cette durée est prise en compte à partir de la dernière date de modification du mot de passe. |
| HUtilisateur.Email | Chaîne de caractères. <br><br>Adresse email de l'utilisateur. |
| HUtilisateur.Groupes | Chaîne de caractères.<br><br>Nom des groupes auxquels l'utilisateur est associé. Les différents groupes sont séparés par des Retours Chariots (RC). |
| HUtilisateur.InfoCréation | Chaîne de caractères en Lecture seule.<br><br>Nom du créateur et date de la création du compte utilisateur.<br><br>Ces informations sont séparées par une tabulation (TAB). La date de création correspond à une chaîne de 8 caractères. |
| HUtilisateur.IPAutorisée | Chaîne de caractères<br><br>Ensemble d'éléments représentant les adresses IP autorisées pour l'utilisateur. Les différents éléments sont séparés par des Retours Chariots (RC).<br><br>Cette variable peut correspondre à :<br><br>- Une adresse IPv4 (par exemple 68.12.53.26) ou plusieurs adresses IPv4 séparées par des Retours Chariots (RC).<br><br>- Une adresse IPv6 (par exemple feco::40) ou plusieurs adresses IPv6 séparées par des Retours Chariots (RC).<br><br>- Une plage d'adresses IPv4 sous forme de CIDR. Par exemple : 68.12.0.0/16<br><br>- Une plage d'adresses IPv6 sous forme de CIDR. Par exemple : fec0:56::0/32<br><br>- Un nom DNS FQDN (par exemple monserveur.pcsoft.fr) ou plusieurs noms DNS séparés par des Retours Chariots (RC).<br><br>- Un nom de sous-domaine (par exemple .pcsoft.fr : attention au point) ou plusieurs noms de sous-domaines séparés par des Retours Chariots (RC).<br><br><br> |
| HUtilisateur.Login | Chaîne de caractères<br><br>Login de l'utilisateur (50 caractères maximum).<br><br>Le login de l'utilisateur n'est pas sensible à la casse. |
| HUtilisateur.MotDePasse | Chaîne de caractères en Ecriture seule.<br><br>Mot de passe de l'utilisateur. Ce mot de passe est sensible à la casse et aux accents. |
| HUtilisateur.NomComplet | Chaîne de caractères. <br><br>Nom complet de l'utilisateur (50 caractères maximum). |
| HUtilisateur.Photo | Chaîne binaire correspondant à la photo de l'utilisateur.<br><br>Il est conseillé d'utiliser la fonction dSauveImagexxx pour affecter une image à cette variable. |
| HUtilisateur.SuperUtilisateur | Booléen (Faux par défaut)<br><br>Si cette variable est à Vrai :<br><br>- Les droits de cet utilisateur ne sont pas vérifiés lors d'une action. Cet utilisateur peut donc faire toutes les actions.<br><br>- Le mot de passe de cet utilisateur n'expire jamais.<br><br><br> |
| HUtilisateur.Téléphone | Chaîne de caractères. <br><br>Numéro de téléphone de l'utilisateur. |


<a name="NOTE2_2"></a>


### Quand utiliser ces variables ?
<a name="quand_utiliser_ces_variables_ELTPARAGRAPHE000193"></a>Les variables de gestion des utilisateurs sont utilisées :

- lors de l'ajout d'un nouvel utilisateur (fonction [HAjouteUtilisateur](../WDLang4/3044281.md)).

- lors de la recherche d'informations sur un utilisateur (fonction [HInfoUtilisateur](../WDLang4/3044285.md)).

- lors de la modification des informations concernant un utilisateur (fonction [HModifieUtilisateur](../WDLang4/3044309.md)).

- lors de la ré-initialisation de ces variables (fonction [HRazUtilisateur](../WDLang4/3044293.md)).



<a name="NOTE2_3"></a>


### Utilisateur par défaut
<a name="utilisateur_par_defaut_ELTPARAGRAPHE000217"></a>Par défaut, l'utilisateur suivant est automatiquement créé lors de l'utilisation d'une base de données Client/Serveur :

- Login : Admin

- Mot de passe : "" (chaîne vide)

- Utilisateur défini comme super-utilisateur

## Variables de gestion des groupes
<a name="variables_gestion_des_groupes_ELTTEXTE000416"></a>
Les variables suivantes permettent de gérer les groupes d'utilisateurs :


|   |   |
| --- | --- |
| HGroupe.ActiveDirectory | Booléen (Faux par défaut).<br><br>Si cette variable est à Vrai, le groupe utilise une authentification via Active Directory. |
| HGroupe.Description | Chaîne de caractères.<br><br>Description du groupe. |
| HGroupe.IPAutorisée | Chaîne de caractères<br><br>Ensemble d'éléments représentant les adresses IP autorisées pour le groupe. Les différents éléments sont séparés par des Retours Chariots (RC).<br><br>Cette variable peut correspondre à :<br><br>- Une adresse IPv4 (par exemple 68.12.53.26) ou plusieurs adresses IPv4 séparées par des Retours Chariots (RC).<br><br>- Une adresse IPv6 (par exemple feco::40) ou plusieurs adresses IPv6 séparées par des Retours Chariots (RC).<br><br>- Une plage d'adresses IPv4 sous forme de CIDR. Par exemple : 68.12.0.0/16<br><br>- Une plage d'adresses IPv6 sous forme de CIDR. Par exemple : fec0:56::0/32<br><br>- Un nom DNS FQDN (par exemple monserveur.pcsoft.fr) ou plusieurs noms DNS séparés par des Retours Chariots (RC).<br><br>- Un nom de sous-domaine (par exemple .pcsoft.fr : attention au point) ou plusieurs noms de sous-domaines séparés par des Retours Chariots (RC).<br><br><br> |
| HGroupe.Nom | Chaîne de caractères. <br><br>Nom du groupe (50 caractères maximum). |
| HGroupe.Utilisateurs | Chaîne de caractères. <br><br>Liste des utilisateurs associés au groupe. Les noms des utilisateurs sont séparés par des retours chariots (RC). |


<a name="NOTE3_2"></a>


### Quand utiliser ces variables ?
<a name="quand_utiliser_ces_variables_ELTPARAGRAPHE000284"></a>Les variables de gestion des groupes sont utilisées :

- lors de l'ajout d'un nouveau groupe (fonction [HAjouteGroupe](../WDLang4/3044280.md)).

- lors de la recherche d'informations sur un groupe (fonction [HInfoGroupe](../WDLang4/3044283.md)).

- lors de la modification des informations concernant un groupe (fonction [HModifieGroupe](../WDLang4/3044308.md)).

- lors de la ré-initialisation de ces variables (fonction [HRAZGroupe](../WDLang4/3044292.md)).

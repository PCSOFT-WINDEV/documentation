
## HClusterIgnoreSynchro (Fonction)

***En anglais : HClusterIgnoreSynchro***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Désigne un noeud du cluster HFSQL comme source de données pour réaliser la synchronisation du cluster.

**Note** : Cette fonction est d'un usage avancé. L'administration d'un cluster HFSQL doit être réalisée de préférence en utilisant le Centre de Contrôle HFSQL.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Resynchronise le cluster HFSQL à partir de la base de données contenu sur le noeud host3
HClusterIgnoreSynchro("clustermanager.mondomaine.com", "secret", "host3.mondomaine.com")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = HClusterIgnoreSynchro(<Coordinateur> , <Mot de passe> , <Noeud>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la fonction a réussi, 

- <u><u><u><u>Faux</u></u></u></u> sinon. En cas d'erreur, la fonction [ErreurInfo](../WDLang1/3013008.md) retourne le détail de l'erreur.




**`<Coordinateur> : (Chaîne de caractères)`**

Nom DNS de la machine où se trouve le coordinateur du cluster.

**`<Mot de passe> : (Chaîne de caractères)`**

Mot de passe permettant de se connecter au coordinateur. Ce mot de passe a été fourni lors de [l'installation du cluster](../WDLang4/1000019031.md).

**`<Noeud> : (Chaîne de caractères)`**

Nom DNS du noeud dont la base de données servira de référence pour la resynchronisation du cluster.






## Remarques
Lorsque tous les noeuds d'un cluster se sont arrêtés anormalement (par exemple, à la suite d'une coupure de courant ou de réseau), aucun noeud n'est certain de posséder les données les plus à jour. Dans ce cas, la fonction **HClusterIgnoreSynchro** permet de définir manuellement le noeud qui servira de référence pour la resynchronisation du cluster.


## MemcachedModifie (Fonction)

***En anglais : MemcachedModify***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie un couple clé/valeur dans le cache Memcached.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MemcachedModifie(memConnexion, "clé", "valeur")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = MemcachedModifie(<Connexion> , <Clé> , <Valeur> [, <Expiration> [, <Valeur supplémentaire>]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur (par exemple si le couple clé/valeur n'existe pas). Pour connaître le libellé de l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md) avec la constante *errMessage*.




**`<Connexion> : (Variable de type memcachedConnexion)`**

Nom de la variable de type [memcachedConnexion](../WDLang4/1000023339.md) correspondant à la connexion au cache Memcached.  

**`<Clé> : (Chaîne de caractères)`**

Clé à modifier. 

**`<Valeur> : (Chaîne de caractères ou Buffer)`**

Nouvelle valeur à stocker. 

**`<Expiration> : (Entier ou Durée optionnelle)`**

Durée pendant laquelle la clé est conservée dans le cache. Cette durée est exprimée en secondes. Cette durée peut correspondre à : 

- un entier correspondant au nombre de secondes, 

- une variable de type Durée. 

- l'indication directe de la durée (par exemple '10s'). 


Par défaut, cette durée vaut 0. Dans ce cas, la durée est illimitée : s'il n'y a plus de mémoire disponible, les clés les plus anciennes sont supprimées.

**`<Valeur supplémentaire> : (Entier optionnel)`**

Permet de stocker une valeur supplémentaire associée au couple Clé/Valeur. Ce paramètre peut permettre par exemple de décrire la clé ou la valeur.

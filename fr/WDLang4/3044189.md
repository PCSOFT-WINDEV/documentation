
## Fichiers de données HFSQL de plus de 2 Giga 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000118"></a>
Par défaut, le moteur HFSQL permet de gérer des fichiers de données d'une taille très importante (jusqu'à 2 Giga). 

Lorsque la taille d'un fichier de données, de son index, de son index full-text et/ou de son mémo excède 2 Giga, une configuration spécifique est nécessaire au niveau de l'éditeur d'analyses comme au niveau matériel.

En effet, pour gérer les fichiers de données de plus de 2 Giga, il est nécessaire d'utiliser :

- un système d'exploitation gérant les fichiers de plus de 4 Giga. Les systèmes suivants ne gèrent pas les fichiers de plus de 4 Giga : Windows 98 et Windows Me.

- un système de fichiers gérant les fichiers de plus de 4 Giga (par exemple NTFS sous Windows). 




**Attention** : Cette configuration est nécessaire aussi bien sur le poste où les fichiers de données sont installés que sur les postes accédant à ces fichiers de données. Ainsi un poste sous Windows 98 ne pourra pas accéder à un fichier de données supérieur à 2 Giga installé sous Windows 2000.





<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Configurer les fichiers de données (sous l'éditeur d'analyses)
<a name="configurer_les_fichiers_donnees_sous_editeur_analyses_ELTTEXTE000148"></a>
Pour gérer des fichiers de données de plus de 2 Giga : 

1. Affichez la description du fichier de données voulu (option "Description du fichier de données" du menu contextel).

2. Dans l'onglet "Détail", cochez l'option "Gestion des fichiers de données de plus de 2 Go (NTFS uniquement)".




**Attention** : Ces fichiers de données seront utilisables uniquement à partir un poste utilisant un système d'exploitation gérant les fichiers de plus de 4 Giga.

## Utiliser des fichiers de données de plus de 2 Giga
<a name="utiliser_des_fichiers_donnees_plus_2_giga_ELTTEXTE000172"></a>


### Description dynamique de fichiers de données
<a name="description_dynamique_fichiers_donnees_ELTPARAGRAPHE000047"></a>Lors de la description dynamique d'un fichier de données (utilisation des propriétés HFSQL et de la fonction [HDécritFichier](../WDLang4/3044206.md)), il est possible de configurer la gestion de la taille du fichier de données grâce à la propriété [GrosFichier](../Proprietes/2512037.md).


<a name="NOTE3_2"></a>


### Propriété du WLangage
<a name="propriete_wlangage_ELTPARAGRAPHE000061"></a>


|   |   |
| --- | --- |
| [GrosFichier](../Proprietes/2512037.md) | La propriété **GrosFichier** permet de paramétrer la taille maximale d'un fichier de données. |




<a name="NOTE3_3"></a>


### Réindexation
<a name="reindexation_ELTPARAGRAPHE000068"></a>**Attention** : La réindexation de fichiers de données de plus de 2 Giga peut être relativement longue.

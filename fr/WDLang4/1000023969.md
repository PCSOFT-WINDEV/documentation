


## &lt;Source&gt;.ActiveFiltre (Fonction)

***En anglais : &lt;Source&gt;.ActivateFilter***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active le filtre précédemment créé pour le fichier de données (la vue ou la requête) spécifié. Ce filtre a été créé avec la fonction [&lt;Source&gt;.Filtre](../WDLang4/1000024960.md) et désactivé avec la fonction [&lt;Source&gt;.DésactiveFiltre](../WDLang4/1000024218.md). Si aucun filtre n'a été déclaré, la fonction **&lt;Source&gt;.ActiveFiltre** est sans effet.

Cette fonction peut être utilisée indifféremment avec les fichiers de données, les vues HFSQL ou les requêtes.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Déclaration du filtre : sélection des clients
// qui ont entre 20 et 60 ans
Client.Filtre(Age_Client, 20, 60)
...
Client.DésactiveFiltre() // Filtre désactivé
...
// Parcours du fichier de données avec filtre sur l'âge
// Traitement des clients qui ont entre 20 et 60 ans
// (procédure TraiteClient2060)
Client.ActiveFiltre()
Client.LitPremier(Age_Client)
TANTQUE Client.Trouve() = Vrai
	TraiteClient2060()
	Client.LitSuivant(Age_Client)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.ActiveFiltre()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le filtre est activé, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur (fichier ou rubrique inexistant).




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données, de la vue HFSQL ou de la requête manipulé.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Activation et désactivation d'un filtre
<a name="activation_desactivation_filtre_ELTPARAGRAPHE000237"></a>Après la déclaration d'un filtre (fonction [&lt;Source&gt;.Filtre](../WDLang4/1000024960.md)), le filtre est actif. Le filtre peut être désactivé grâce à la fonction [&lt;Source&gt;.DésactiveFiltre](../WDLang4/1000024218.md).
<a name="NOTE0_2"></a>


### Nombre de filtres actifs possibles
<a name="nombre_filtres_actifs_possibles_ELTPARAGRAPHE000257"></a>Pour un fichier de données spécifique, un seul filtre peut être actif à la fois.
<a name="NOTE0_3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Windows](https://doc.pcsoft.fr/ext/images/fr/WINDOWS.png)![OLE DB](https://doc.pcsoft.fr/ext/images/fr/OLEDB.png) 

### Fonctionnement en OLE DB
<a name="fonctionnement_ole_ELTPARAGRAPHE000269"></a>Si la fonction **&lt;Source&gt;.ActiveFiltre** est utilisée lors du parcours d'un fichier de données, le parcours est arrêté. Pour prendre en compte le filtre, il faut ré-initialiser le parcours avec les fonctions [&lt;Source&gt;.LitPremier](../WDLang4/1000025022.md)/[&lt;Source&gt;.Premier](../WDLang4/1000025048.md) ou [&lt;Source&gt;.LitDernier](../WDLang4/1000025020.md)/[&lt;Source&gt;.Dernier](../WDLang4/1000024217.md).

La position en cours et toutes les positions sauvegardées sont perdues. La restauration de la position sera effectuée à partir de la valeur de la meilleure rubrique.

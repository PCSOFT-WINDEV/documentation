


## &lt;Source&gt;.DébloqueNumEnr (Fonction)

***En anglais : &lt;Source&gt;.UnlockRecNum***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Débloque un enregistrement précédemment bloqué avec :

- soit la fonction [&lt;Source&gt;.BloqueNumEnr](../WDLang4/1000024197.md),

- soit une fonction de lecture utilisée avec un paramètre bloquant (par exemple fonction [&lt;Source&gt;.Lit](../WDLang4/1000025019.md) utilisé avec la constante *hBlocageEcriture* ou la constante *hBlocageLectureEcriture*).


**Remarque** : La fonction **&lt;Source&gt;.DébloqueNumEnr** ne permet pas de débloquer un enregistrement bloqué par une autre application.



![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Cette fonction est disponible uniquement pour les fichiers de données HFSQL. Elle n'est pas disponible pour les fichiers de données accédés par JDBC. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Blocage
Client.BloqueNumEnr(hNumEnrEnCours, hBlocageLectureEcriture)
// Modification de l'enregistrement
...
// Déblocage
Client.DébloqueNumEnr(hNumEnrEnCours)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.DébloqueNumEnr([<Numéro d'enregistrement>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'enregistrement a été débloqué, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données manipulé. 

**`<Numéro d'enregistrement> : (Entier optionnel)`**

Numéro de l'enregistrement à débloquer. 
Si ce numéro n'est pas spécifié (égal à 0 ou à la constante *hNumEnrEnCours*), la fonction **&lt;Source&gt;.DébloqueNumEnr** va manipuler l'enregistrement en cours.
![Connecteurs Natifs (Accès Natifs)](https://doc.pcsoft.fr/ext/images/fr/AN.png) La fonction **&lt;Source&gt;.DébloqueNumEnr** débloque uniquement l'enregistrement en cours. Il est impossible de spécifier un numéro d'enregistrement différent de celui en cours (pour désigner l'enregistrement en cours, utilisez la constante *hNumEnrEnCours*).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La fonction **&lt;Source&gt;.DébloqueNumEnr** est sans effet si :

- Le fichier de données est en mode Mono (voir la fonction [&lt;Source&gt;.Mode](../WDLang4/1000025037.md)).

- L'enregistrement n'est pas bloqué.

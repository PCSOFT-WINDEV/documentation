


## HInfoTâche (Fonction)

***En anglais : HInfoTask***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie les caractéristiques d'une tâche planifiée dans une variable de type [hTâchePlanifiée](../WDLang4/1000017448.md). Il est par exemple possible de lire certaines caractéristiques de la tâche planifiée pour les modifier par la suite avec la fonction [HModifieTâche](../WDLang4/1000017374.md).



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MaTâche est une hTâchePlanifiée
// Récupération des caractéristiques de la tâche planifiée
MaTâche = HInfoTâche(MaConnexion, IDTâche)
// Modification de la tâche planifiée
MaTâche.Heure = SAI_Heure.Heure + 1
HModifieTâche(MaConnexion, IDTâche, MaTâche)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HInfoTâche(<Connexion> , <Tâche planifiée>)`
---

**`<Résultat> : (Variable de type hTâchePlanifiée)`**

Variable de type [hTâchePlanifiée](../WDLang4/1000017448.md) contenant les caractéristiques de la tâche planifiée spécifiée.

**`<Connexion> : (Chaîne de caractères ou variable de type Connexion)`**

Connexion pour laquelle une tâche doit être manipulée. Cette connexion correspond : 

- soit au nom de la connexion définie sous l'éditeur d'analyses ou grâce aux fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

- soit au nom d'une variable de type [Connexion](../WDLang4/1514073.md).




**`<Tâche planifiée> : (Entier)`**

Identifiant de la tâche planifiée renvoyé par les fonctions [HAjouteTâche](../WDLang4/1000017091.md) et [HListeTâche](../WDLang4/1000017092.md).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Pour vérifier l'existence d'une tâche, utilisez la fonction [HListeTâche](../WDLang4/1000017092.md) : il suffit de vérifier si l'identifiant de la tâche est présent parmi les tâches renvoyées par la fonction [HListeTâche](../WDLang4/1000017092.md). 

- Pour récupérer les caractéristiques d'une tâche planifiée, il est nécessaire de posséder le droit de connexion (constante *hDroitConnexion* ou *hDroitConnexionCryptée*) sur la base de données contenant la procédure stockée.

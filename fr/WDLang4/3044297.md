


## HActiveFiltreAuto (Fonction)

***En anglais : HActivateAutoFilter***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active un filtre automatique sur les fichiers de données reliés lors du parcours d'un fichier XML. Ce filtre peut être désactivé grâce à la fonction [HDésactiveFiltreAuto](../WDLang4/3044298.md).

**Attention** : Cette fonction est disponible uniquement pour le Connecteur Natif XML.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Activation de tous les filtres auto possibles du fichier de données Client
HActiveFiltreAuto(Client, "*" )
// sauf entre les fichiers de données Client et Commande
HDésactiveFiltreAuto(Client, Commande)

/////////////////////////////

// Parcours des commandes
HLitPremier(Commande, IDCommande)
TANTQUE PAS HEnDehors()
// Parcours des lignes de commande de la commande en cours
	HLitPremier(LigneCde, IDCommande)
	TANTQUE PAS HEnDehors()
		HLitSuivant(LigneCde, IDCommande)
	FIN
	// Commande suivante
	HLitSuivant(Commande, IDCommande)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HActiveFiltreAuto(<Fichier de données de parcours> , <Fichier de données à filtrer>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le filtre est activé, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur (fichier ou rubrique inexistant).




**`<Fichier de données de parcours> : (Chaîne de caractères)`**

Nom du fichier de données utilisé pour le parcours.

Ce paramètre peut correspondre à "\*". Dans ce cas, tous les filtres automatiques possibles seront activés.

Si les paramètres &lt;Fichier de données de parcours&gt; et &lt;Fichier de données à filtrer&gt; correspondent à "\*", &lt;Résultat&gt; vaut <u><u><u><u>Vrai</u></u></u></u> et aucun filtre n'est activé.

**`<Fichier de données à filtrer> : (Chaîne de caractères)`**

Nom du fichier de données à filtrer lors d'un positionnement dans le fichier de données de parcours.

Ce paramètre peut correspondre à "\*". Dans ce cas, tous les filtres automatiques possibles seront activés.

Si les paramètres &lt;Fichier de données de parcours&gt; et &lt;Fichier de données à filtrer&gt; correspondent à "\*", &lt;Résultat&gt; vaut <u><u><u><u>Vrai</u></u></u></u> et aucun filtre n'est activé.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Voici un exemple simple :

```wl
HActiveFiltreAuto(Père, Fils)
```

Ce code indique que quand les enregistrements du fichier de données Fils sont parcourus, ce parcours se fait par rapport au 'contexte' (au noeud) courant du fichier de données Père : tous les 'Fils' du 'Père' courant seront parcourus.


```wl
HDésactiveFiltreAuto(Père, Fils)
```

Ce code indique que le filtrage précédent n'est pas actif : le parcours du fichier de données Fils sera indépendant de celui du fichier de données Père. Cela revient donc à parcourir tous les enregistrements du fichier de données Fils dans le fils donné.

La fonction **HActiveFiltreAuto** permet d'activer/désactiver le lien hiérarchique entre les fichiers de données Père et Fils pendant des parcours.

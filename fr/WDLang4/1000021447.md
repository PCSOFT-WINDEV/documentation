


## HAjoutePlanification (Fonction)

***En anglais : HAddScheduling***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Crée une tâche planifiée sur un serveur HFSQL : 

- procédure stockée, 

- sauvegarde, 

- optimisation, 

- rafraîchissement d'une vue matérialisée.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Recalculer les statistiques des fichiers de données de la base de données "MaBase", 
// tous les samedis à 23H00
Optim est une hOptimisationPlanifiée
Optim.Description = "Recalcul des statistiques des fichiers de données" + ...
	" de la base de données MaBase"

Optim.Planification.Mois = "*"
Optim.Planification.JourDeLaSemaine = "samedi"
Optim.Planification.Heure = "23"
Optim.Planification.Minute = "0"

Optim.Fichier[1].Nom = "MaBase"
Optim.Fichier[1].Option = hOptStat

HAjoutePlanification(MaConnexion, Optim)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

### Ajouter une tâche planifiée

`<Résultat> = HAjoutePlanification(<Connexion> , <Tâche planifiée>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la tâche planifiée a été ajoutée,

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Connexion> : (Chaîne de caractères ou variable de type Connexion)`**

Connexion pour laquelle une tâche planifiée doit être ajoutée. Cette connexion correspond : 

- soit au nom de la connexion définie sous l'éditeur d'analyses ou grâce aux fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

- soit au nom d'une variable de type [Connexion](../WDLang4/1514073.md).




**`<Tâche planifiée> : (Variable de type hTâchePlanifiée)`**

Nom de la variable de type [hTâchePlanifiée](../WDLang4/1000017448.md), décrivant la tâche déclenchant une procédure stockée.

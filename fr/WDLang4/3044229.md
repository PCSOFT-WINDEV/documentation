


## HListeREP (Fonction)

***En anglais : HListREP***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie la liste des affectations des fichiers de données manipulés par l'application en cours, c'est-à-dire la liste des fichiers physiques correspondants aux fichiers de données décrits dans l'analyse.

Pour plus de détails sur cette liste, consultez [Liste des fichiers de données manipulés par une application](../WDLang4/3044186.md).

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
sAffectations est une chaîne
// Récupérer toutes les affectations du fichier logique
// CLIENT dont le nom physique est CLIENT2000 et qui
// existent bien sur disque
sAffectations = HListeREP(".", "CLIENT", "CLIENT2000", hExistant)
```

## Syntaxe

### Liste des affectations à partir du fichier .REP

`<Résultat> = HListeREP([<Nom du fichier REP> [, <Nom logique fichier> [, <Nom physique fichier>]]] [, <Options>])`
---

**`<Résultat> : (Chaîne de caractères)`**



- Liste des affectations, 

- Chaîne vide ("") en cas d'erreur. 


La liste des affectations est de la forme :

```txt
<Nom logique> + TAB + <Nom physique> + TAB + <Répertoire> + RC + ...
```

Où :

- &lt;Nom logique&gt; : Nom du fichier de données dans l'analyse. 

- &lt;Nom physique&gt; : Nom du fichier physique, sans chemin et sans extension.

- &lt;Répertoire&gt; : Répertoire du fichier physique. 


![HFSQL Client/Serveur](https://doc.pcsoft.fr/ext/images/fr/HFCS.png) &lt;Résultat&gt; est complété par 2 nouvelles informations :

- &lt;Nom du serveur&gt; : Nom du serveur HFSQL Client/Serveur.

- &lt;Nom de la base de données&gt; : Nom de la base de données HFSQL Client/Serveur.




**`<Nom du fichier REP> : (Chaîne de caractères optionnelle)`**

Nom complet du fichier .REP examiné. Si ce paramètre correspond à une chaîne vide ou à **"."**, le fichier ".REP" en cours est examiné. Si les trois premiers paramètres optionnels ne sont pas précisés, le fichier ".REP" en cours est également examiné.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Il n'est pas possible de spécifier uniquement ce paramètre. La syntaxe HListeRep(&lt;Nom du fichier .REP&gt;) n'est pas disponible.

**`<Nom logique fichier> : (Chaîne de caractères optionnelle)`**

Nom logique du fichier de données dont les affectations doivent être listées. Si ce paramètre n'est pas précisé ou est égal à **"\*"**, toutes les affectations de tous les fichiers de données sont recherchées.

**`<Nom physique fichier> : (Chaîne de caractères optionnelle)`**

Nom physique du fichier de données dont les affectations doivent être listées. Si ce paramètre n'est pas précisé, toutes les affectations de &lt;Nom logique fichier&gt; sont recherchées.

**`<Options> : (Constante optionnelle de type entier)`**

Par défaut, toutes les affectations sont recherchées. La constante *hExistant* permet de rechercher uniquement les affectations pour lesquelles un fichier de données existe réellement.

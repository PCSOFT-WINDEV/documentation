


## &lt;Source&gt;.Lit (Fonction)

***En anglais : &lt;Source&gt;.Read***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Lit un enregistrement dans un fichier de données en fonction d'un numéro d'enregistrement donné. Ce mode de lecture est appelé "Accès direct". L'enregistrement lu est chargé en mémoire. La fonction [&lt;Source&gt;.Etat](../WDLang4/1000024943.md) permet de connaître l'état de l'enregistrement (actif, rayé ou supprimé).

En général, la fonction **&lt;Source&gt;.Lit** permet de se positionner dans le fichier de données pour effectuer une boucle de lecture en précisant le numéro d'enregistrement.

Après l'exécution de la fonction **&lt;Source&gt;.Lit**, plusieurs cas peuvent se présenter :

- le numéro d'enregistrement spécifié n'existe pas (supérieur au nombre total d'enregistrements ou négatif) : aucune lecture n'est effectuée et la fonction [&lt;Source&gt;.EnDehors](../WDLang4/1000024605.md) renvoie <u><u><u>Vrai</u></u></u>.

- le numéro d'enregistrement spécifié n'existe plus (enregistrement rayé, supprimé, ...) : aucune lecture n'est effectuée et la fonction [&lt;Source&gt;.EnDehors](../WDLang4/1000024605.md) renvoie <u><u><u>Faux</u></u></u>.
	Utilisez la fonction [&lt;Source&gt;.Etat](../WDLang4/1000024943.md) pour connaître l'état de l'enregistrement.

- le fichier de données est vide : la fonction [&lt;Source&gt;.EnDehors](../WDLang4/1000024605.md) renvoie <u><u><u>Vrai</u></u></u>.

- la fonction tente de bloquer un enregistrement déjà bloqué en lecture : la fonction [HErreurBlocage](../WDLang4/3044099.md) renvoie <u><u><u>Vrai</u></u></u> et la fonction [&lt;Source&gt;.EnDehors](../WDLang4/1000024605.md) renvoie <u><u><u>Vrai</u></u></u>.
	![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Accès par JDBC : La gestion des blocages n'est pas disponible sur les bases de données accédées par JDBC.




Cette fonction peut être utilisée indifféremment avec les fichiers de données (indexés ou non) ou les vues HFSQL.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Parcours séquentiel pour afficher le nom de tous les clients
i est un entier
POUR i = 1 _A_ Client.NbEnr(hEtatTous)
	Client.Lit(i)
	SI Client.Etat() = hEtatActif ALORS 
		Info("Nom du client : " + Client.Nom)
	FIN
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.Lit([<Numéro d'enregistrement> [, <Options>]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'enregistrement a été lu, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur (blocage, fin du fichier de données, ...) : l'enregistrement n'est pas lu. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données HFSQL, de la vue ou de la requête manipulé.

**`<Numéro d'enregistrement> : (Entier optionnel)`**

Numéro de l'enregistrement à lire.

Si ce paramètre n'est pas spécifié (ou est égal à 0 ou à la constante *hNumEnrEnCours*), la fonction **&lt;Source&gt;.Lit** va manipuler l'enregistrement en cours.
![OLE DB](https://doc.pcsoft.fr/ext/images/fr/OLEDB.png)![Connecteurs Natifs (Accès Natifs)](https://doc.pcsoft.fr/ext/images/fr/AN.png) La fonction **&lt;Source&gt;.Lit** lit uniquement l'enregistrement en cours. Il est impossible de spécifier un numéro d'enregistrement différent de celui en cours. Pour désigner l'enregistrement en cours, utilisez la constante *hNumEnrEnCours*.![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Accès par JDBC : La fonction **&lt;Source&gt;.Lit** lit uniquement l'enregistrement en cours. Il est impossible de spécifier un numéro d'enregistrement différent de celui en cours. Pour désigner l'enregistrement en cours, utilisez la constante *hNumEnrEnCours*. 

**`<Options> : (Constante optionnelle)`**

Paramètre le blocage réalisé sur l'enregistrement lu par la fonction **&lt;Source&gt;.Lit** :


|   |   |
| --- | --- |
| *hBlocageEcriture* | Blocage en écriture : l'enregistrement en cours de lecture pourra être lu par une autre application mais ne pourra pas être modifié par une autre application. |
| *hBlocageLectureEcriture* | Blocage en lecture/écriture : l'enregistrement en cours de lecture ne pourra ni être lu ni être modifié par une autre application. |
| *hBlocageNon* | Aucun blocage : l'enregistrement pourra être lu ou modifié par une autre application pendant la lecture. |

![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Les options de blocage ne sont pas prises en compte. Utilisez des fonctions de blocage ([&lt;Source&gt;.BloqueNumEnr](../WDLang4/1000024197.md)) conservées par compatibilité.![OLE DB](https://doc.pcsoft.fr/ext/images/fr/OLEDB.png)![Connecteurs Natifs (Accès Natifs)](https://doc.pcsoft.fr/ext/images/fr/AN.png) Si le provider OLE DB ou le Connecteur Natif utilisé ne gère pas les blocages, les options de blocage seront sans effet.![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Accès par JDBC : Ce paramètre est ignoré.

### Enregistrements lus
<a name="enregistrements_lus_ELTPARAGRAPHE000376"></a>La fonction **&lt;Source&gt;.Lit** est la seule fonction permettant d'accéder à un enregistrement rayé ou supprimé. Si l'enregistrement est supprimé, le contenu de l'enregistrement n'est plus valide.
<a name="NOTE0_2"></a>


### Parcours d'un fichier de données
<a name="parcours_fichier_donnees_ELTPARAGRAPHE000389"></a>La fonction **&lt;Source&gt;.Lit** ne permet pas de poursuivre un parcours effectué avec les fonctions [&lt;Source&gt;.LitPremier](../WDLang4/1000025022.md), [&lt;Source&gt;.LitSuivant](../WDLang4/1000025035.md).
<a name="NOTE0_3"></a>


### Mémos
<a name="memos_ELTPARAGRAPHE000412"></a>Les mémos associés à l'enregistrement peuvent être automatiquement lus ou non lors de la lecture de l'enregistrement. La fonction [&lt;Source&gt;.GèreMémo](../WDLang4/1000024987.md) permet de personnaliser cette lecture automatique.

Si les mémos sont gérés, lorsque l'enregistrement est lu, les mémos texte associés sont également lus. Les mémos binaires ne sont lus que lorsqu'ils sont utilisés explicitement (fonction [&lt;Source&gt;.ExtraitMémo](../WDLang4/1000024957.md)).

### Blocages
<a name="blocages_ELTPARAGRAPHE000436"></a>Par défaut (paramètre &lt;Option&gt; non spécifié), aucun blocage de l'enregistrement n'est réalisé.

Si un blocage est demandé (paramètres *hBlocageEcriture* ou *hBlocageLectureEcriture*), la lecture de l'enregistrement sera réalisée uniquement si cet enregistrement n'est pas déjà bloqué.

![OLE DB](https://doc.pcsoft.fr/ext/images/fr/OLEDB.png)![Connecteurs Natifs (Accès Natifs)](https://doc.pcsoft.fr/ext/images/fr/AN.png) Si le provider OLE DB ou le Connecteur Natif utilisé ne gère pas les blocages, les options de blocage seront sans effet.
<a name="NOTE0_5"></a>


### Divers
<a name="divers_ELTPARAGRAPHE000452"></a>

- Le résultat de la fonction [&lt;Source&gt;.Trouve](../WDLang4/1000025076.md) doit être ignoré (la fonction [&lt;Source&gt;.Trouve](../WDLang4/1000025076.md) doit être utilisée pour les recherches réalisées avec les fonctions [&lt;Source&gt;.LitRecherchePremier](../WDLang4/1000025034.md) et [&lt;Source&gt;.LitRechercheDernier](../WDLang4/1000025024.md)).

- Le numéro de l'enregistrement en cours est connu avec la fonction [&lt;Source&gt;.NumEnr](../WDLang4/1000025041.md). Le numéro du premier enregistrement du fichier de données est 1.

- Cette fonction remplace les fonctions **HLitBloque** et **HLitSansBloquer**, conservées par compatibilité avec WINDEV 5.5.




## &lt;Source&gt;.VueVersFichier (Fonction)

***En anglais : &lt;Source&gt;.ViewToFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Enregistre les modifications réalisées dans une vue HFSQL (grâce aux fonctions [&lt;Source&gt;.Modifie](../WDLang4/1000025038.md), [&lt;Source&gt;.Supprime](../WDLang4/1000025071.md) ou [&lt;Source&gt;.Raye](../WDLang4/1000025051.md)) dans le fichier de données correspondant.

**Remarque** : Cette opération ne peut être effectuée que si la vue a été créée avec la constante *hVueBlocage* (c'est-à-dire que les enregistrements importés dans la vue ont été bloqués dans le fichier de données). Après l'exécution de la fonction **&lt;Source&gt;.VueVersFichier**, les enregistrements sont débloqués dans le fichier source de la vue.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Enregistre les modification de la vue Client34
// dans le fichier Client
Client34.VueVersFichier()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.VueVersFichier()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la vue a été transférée dans le fichier de données, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [HErreur](../WDLang4/3044088.md) permet d'obtenir plus d'informations sur le problème rencontré.




**`<Source> : (Type de la Source)`**

Nom de la vue manipulée.   



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
En cas d'erreur pendant la mise à jour du fichier de données, pour connaître les enregistrements qui ont posé problème et les problèmes associés, utilisez la fonction [HErreur](../WDLang4/3044088.md). Il suffit de parcourir la vue à nouveau et de tester la fonction [HErreur](../WDLang4/3044088.md) sur chaque enregistrement de la vue.


## HDBDécritIndex (Fonction)

***En anglais : HDBDescribeIndex***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Décrit par programmation les différents fichiers d'index qui seront créés. Un fichier d'index doit être décrit pour chaque clé de recherche. Le fichier d'index créé sera au format dBase 3.

**Remarque** : Le fichier de données et les fichiers d'index décrits seront créés avec la fonction [HDBCréation](../WDLang4/3044171.md) (au format dBase 3 uniquement). Ainsi, les index de format autre que NDX ne peuvent pas être créés avec la fonction **HDBDécritIndex**.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HDBDécritFichier("CLIENT", "DA", "C:\FIC\CLIENT.DBF")
HDBDécritRubrique("NOM,C,20")
HDBDécritRubrique("PRENOM,C,20")
HDBDécritRubrique("MARIE,L")
HDBDécritRubrique("DATNAISS,D")
HDBDécritRubrique("INFOS,M")
HDBDécritIndex("C:\FIC\FICNOM.NDX", "NOM")
HDBDécritIndex("C:\FIC\FICNP.NDX", "NOMPRE,C,40", "NOM+PRENOM")
HDBCréation()
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HDBDécritIndex(<Nom complet du fichier d'index> , <Nom de la rubrique clé> , <Formule de la clé composée>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Nom complet du fichier d'index> : (Chaîne de caractères)`**

Chemin d'accès au fichier d'index et le nom complet du fichier (avec l'extension).

**`<Nom de la rubrique clé> : (Chaîne de caractères)`**



- Nom de la clé simple : ce nom correspond au nom d'une rubrique du fichier.

- Nom de la clé composée et son type. &lt;Nom de la rubrique Clé&gt; est de la forme "&lt;Nom_CléComp&gt;,&lt;TypeCle&gt;" :
	


|   |   |
| --- | --- |
| &lt;Nom_CléComp&gt; | Nom de la clé composée |
| &lt;TypeClé&gt; | Type de la clé :<br><br>	- C,&lt;Taille&gt; : Rubrique de type texte, &lt;Taille&gt; est la taille de la chaîne de caractères<br><br>	- N,&lt;Taille&gt;,&lt;NbDecimale&gt; : Rubrique de type numérique. &lt;Taille&gt; est la taille du numérique, &lt;NbDecimales&gt; est le nombre de décimales du numérique.<br><br>	- L : Rubrique de type booléen.<br><br>	- D : Rubrique de type date (au format "AAMMJJ").<br><br>	- M : Rubrique de type Mémo texte.<br><br><br> |






**`<Formule de la clé composée> : (Chaîne de caractères)`**

Formule de la clé composée à utiliser.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Utilisation d'espaces dans les paramètres
<a name="utilisation_espaces_dans_les_parametres_ELTPARAGRAPHE000077"></a>Les paramètres ne doivent pas être séparés par des espaces, par exemple il ne faut pas écrire :


```wl
HDBDécritIndex("C:\FIC\FICNP.NDX", "NOMPRE, C, 40", "NOM+PRENOM")
```


Il faut écrire :


```wl
HDBDécritIndex("C:\FIC\FICNP.NDX", "NOMPRE,C,40", "NOM+PRENOM")
```



<a name="NOTE0_2"></a>




### Description d'un index
<a name="description_index_ELTPARAGRAPHE000092"></a>

- Si le fichier est déjà ouvert lors de l'appel de la fonction **HDBDécritIndex**, une erreur 1009 sera générée.

- La taille maximale d'une rubrique :

	- de type texte non clé est de 254 caractères.

	- de type texte clé est de 100 caractères.

	- de type numérique est de 19 chiffres pour la partie entière et 15 chiffres pour la partie décimale.




- Pour utiliser la fonction **HDBDécritIndex**, le fichier doit être décrit avec la fonction [HDBDécritFichier](../WDLang4/3044172.md).





<a name="NOTE0_3"></a>




### Limitations
<a name="limitations_ELTPARAGRAPHE000116"></a>

- Cette fonction n'est pas disponible lors de l'utilisation d'une base de données via .

- Cette fonction n'est pas disponible lors de l'utilisation d'un exécutable autonome.




## &lt;Source&gt;.ListeIndexFullText (Fonction)

***En anglais : &lt;Source&gt;.ListFullTextIndex***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie la liste des index full-text d'un fichier de données reconnu par le moteur HFSQL. Il est possible de savoir :

- la formule de l'index full-text,

- si l'index full-text est sensible à la casse ou aux accents,

- la longueur minimale des mots indexés,

- si l'index full-text gère le HTML ou le RTF,

- si l'index full-text utilise une liste de mots creux

- si l'index full-text utilise une liste de synonymes

- quelle langue est utilisée pour la racination...




Les fichiers définis par les fonctions [&lt;Source&gt;.Déclare](../WDLang4/1000024213.md), [&lt;Source&gt;.DéclareExterne](../WDLang4/1000024214.md) et [&lt;Source&gt;.DécritFichier](../WDLang4/1000024215.md) sont pris en compte.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupérer les index full-text d'un fichier de données de l'analyse en cours
ListeFullText est une chaîne
ListeFullText = Client.ListeIndexFullText(hLstDétail)
// Ajoute la liste des index dans le champ COMBO_COMBO1
COMBO_COMBO1.Ajoute(ListeFullText)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.ListeIndexFullText([<Chemin du WDD> [, <Mot de passe du WDD> [, <Options>]]])`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste des index full-text présents dans le fichier de données spécifié. Les différentes descriptions de chaque index full-text sont séparées par RC (Retour chariot). En fonction de la constante utilisée, le résultat est plus ou moins détaillé :


|   |   |
| --- | --- |
| *hLstDétail* non précisé | &lt;Nom de l'index full-text 1&gt; + RC + &lt;Nom de l'index full-text 2 + RC + ...<br>Le résultat avec la constante *hLstTriée* est identique. |
| *hLstDétail* précisé | <br><br>- **Avant la version 20** : &lt;Nom de l'index full-text 1&gt; + TAB + &lt;Formule&gt; + RC + ...<br><br>- **Après la version 20** : &lt;Nom de l'index full-text 1&gt; + TAB + &lt;Formule&gt; + TAB+ &lt;Sensible à la casse&gt; + TAB + &lt;Sensible aux accents&gt; + TAB + &lt;Longueur minimale&gt; + TAB + &lt;Gestion HTML&gt; + TAB + &lt;Gestion RTF&gt; + TAB + &lt;Mots Creux&gt; + TAB + &lt;Synonymes&gt; + TAB + &lt;Racinisation&gt; + RC + ...<br><br><br> |

Où :


- &lt;Nom de l'index full-text X&gt; : nom de la rubrique index full-text

- &lt;Formule&gt; : liste des rubriques qui composent l'index full-text. Les rubriques sont séparées par le caractère "+".




- &lt;Sensible à la casse&gt; : "1" si l'index full-text est sensible à la casse, "0" sinon.

- &lt;Sensible aux accents&gt; : "1" si l'index full-text est sensible aux accents, "0" sinon.

- &lt;Longueur minimale&gt; : longueur minimale des mots indexés (0 si tous les mots sont indexés).

- &lt;Gestion HTML&gt; : "1" pour que les balises HTML ne soient pas indexées, "0" pour ne pas gérer le HTML.

- &lt;Gestion RTF&gt; : "1" pour que les balises RTF ne soient pas indexées, "0" pour ne pas gérer le RTF.

- &lt;Mots Creux&gt; : "1" si l'index full-text utilise une liste de mots creux, "0" sinon.

- &lt;Synonymes&gt; : "1" si l'index full-text utilise une liste de synonymes, "0" sinon.

- &lt;Racinisation&gt; : entier correspondant à la langue utilisée pour la racinisation, "0" si l'index full-text n'utilise pas la racinisation.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données pour lequel les index full-text doivent être listés.

**`<Chemin du WDD> : (Chaîne de caractères optionnelle)`**

Chemin complet de l'analyse (c'est-à-dire du ficher .WDD) dans laquelle le fichier de données est décrit. Si ce paramètre n'est pas précisé ou est égal à une chaîne vide (""), le fichier de données est recherché dans l'analyse en cours.

**`<Mot de passe du WDD> : (Chaîne de caractères optionnelle)`**

Mot de passe défini pour l'analyse. Si aucun mot de passe n'est défini, ce paramètre doit être égal à une chaîne vide ("").

**`<Options> : (Constante ou combinaison de constantes de type entier (optionnel))`**

Paramètre le type de liste à effectuer :


|   |   |
| --- | --- |
| *hLstDétail* | Informations détaillées. |
| *hLstNormal*<br>(valeur par défaut) | Informations non détaillées. |
| *hLstTriée* | Trie la liste par ordre croissant. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Affichage du résultat dans un champ Liste ou un champ Combo
<a name="affichage_resultat_dans_champ_liste_champ_combo_ELTPARAGRAPHE000298"></a>&lt;Résultat&gt; peut être directement affecté à un champ Liste ou un champ Combo grâce à la fonction [&lt;Liste&gt;.Ajoute](../WDLang1/1000017319.md). Chaque ligne contiendra le nom d'un des index full-text trouvés.
<a name="NOTE0_2"></a>


### Extraire une information du résultat
<a name="extraire_une_information_resultat_ELTPARAGRAPHE000313"></a>Pour extraire une information du résultat détaillé, il est conseillé d'utiliser la fonction [&lt;Type Chaîne&gt;.ExtraitChaîne](../WDLang1/1000024876.md).

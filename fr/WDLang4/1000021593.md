


## HListeSynonyme (Fonction)

***En anglais : HListSynonym***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie la liste des synonymes utilisés par un index full-text. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupère l'ensemble des mots synonymes
soit sEnsembleSynonyme = HListeSynonyme(FichierDeDonnées, MonIndexFullText)
// Pour chaque groupe de mots synonymes
POUR TOUTE CHAÎNE sListeSynonyme DE sEnsembleSynonyme SEPAREE PAR RC
	soit sChaîne = "Les mots suivants sont synonymes : " 	
	// Extrait chaque synonyme
	POUR TOUTE CHAÎNE sSynonyme DE sListeSynonyme SEPAREE PAR TAB
		// Ajoute le mot à la chaîne
		sChaîne += sSynonyme + " "
	FIN
	Trace(sChaîne)
FIN
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = HListeSynonyme(<Fichier de données> , <Index full-text> [, <Chemin du WDD> [, <Mot de passe du WDD>]])`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste des synonymes définis pour l'index full-text spécifié dans le fichier de données spécifié. Cette liste est de la forme : 

```txt
<Mot 1> + TAB + <Synonyme 1A> + TAB + <Synonyme 1B> + RC + ...
<Mot 2> + TAB + <Synonyme 2> + ... + RC + ...
<Mot N> + TAB + <Synonyme N>
```
Dans cette liste : 

- Les synonymes sont séparés par une tabulation (TAB). 

- Les différents groupes de synonymes sont séparés par un Retour Chariot (RC). 


Remarques : 

- Les mots peuvent avoir été mis en majuscules si l'index full-text est aussi insensible à la casse ou les accents retirés si l'index full-text est insensible à l'accentuation.

- Les mots possédant une longueur plus petite que la longueur minimale spécifiée pour l'index full-text peuvent ne pas apparaître dans cette liste.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données contenant l'index full-text.

**`<Index full-text> : (Chaîne de caractères)`**

Nom de l'index full-text manipulé.

**`<Chemin du WDD> : (Chaîne de caractères optionnelle)`**

Chemin complet de l'analyse (c'est-à-dire du ficher .WDD) dans laquelle le fichier de données est décrit. Si ce paramètre n'est pas précisé ou est égal à une chaîne vide (""), le fichier de données est recherché dans l'analyse en cours.

**`<Mot de passe du WDD> : (Chaîne de caractères optionnelle)`**

Mot de passe défini pour l'analyse. Si aucun mot de passe n'est défini, ce paramètre doit être égal à une chaîne vide ("").




## HDébloqueNumEnr (Fonction)

***En anglais : HUnlockRecNum***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Débloque un enregistrement précédemment bloqué avec :

- soit la fonction [HBloqueNumEnr](../WDLang4/3044152.md),

- soit une fonction de lecture utilisée avec un paramètre bloquant (par exemple fonction [HLit](../WDLang4/3044047.md) utilisé avec la constante *hBlocageEcriture* ou la constante *hBlocageLectureEcriture*).


**Remarque** : La fonction **HDébloqueNumEnr** ne permet pas de débloquer un enregistrement bloqué par une autre application.



![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Cette fonction est disponible uniquement pour les fichiers de données HFSQL. Elle n'est pas disponible pour les fichiers de données accédés par JDBC. 
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Blocage
HBloqueNumEnr(CLIENT, hNumEnrEnCours, hBlocageLectureEcriture)
// Modification de l'enregistrement
...
// Déblocage
HDébloqueNumEnr(CLIENT, hNumEnrEnCours)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HDébloqueNumEnr([<Fichier de données> [, <Numéro d'enregistrement>]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'enregistrement a été débloqué, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Fichier de données> : (Chaîne de caractères optionnelle)`**

Nom du fichier de données manipulé. 
Si ce nom n'est pas spécifié, la fonction **HDébloqueNumEnr** va manipuler le dernier fichier de données utilisé par la dernière fonction de gestion HFSQL (fonction commençant par la lettre H).

**`<Numéro d'enregistrement> : (Entier optionnel)`**

Numéro de l'enregistrement à débloquer. 
Si ce numéro n'est pas spécifié (égal à 0 ou à la constante *hNumEnrEnCours*), la fonction **HDébloqueNumEnr** va manipuler l'enregistrement en cours.
![Connecteurs Natifs (Accès Natifs)](https://doc.pcsoft.fr/ext/images/fr/AN.png) La fonction **HDébloqueNumEnr** débloque uniquement l'enregistrement en cours. Il est impossible de spécifier un numéro d'enregistrement différent de celui en cours (pour désigner l'enregistrement en cours, utilisez la constante *hNumEnrEnCours*).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La fonction **HDébloqueNumEnr** est sans effet si :

- Le fichier de données est en mode Mono (voir la fonction [HMode](../WDLang4/3044039.md)).

- L'enregistrement n'est pas bloqué.

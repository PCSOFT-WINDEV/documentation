


## &lt;Variable Connexion&gt;.SupprimeOptimisationPlanifiée (Fonction)

***En anglais : &lt;Connection variable&gt;.DeleteScheduledOptimization***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Supprime une tâche d'optimisation planifiée de fichiers de données HFSQL Client/Serveur.



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Connexion>.SupprimeOptimisationPlanifiée(<Tâche à supprimer>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la tâche a été supprimée, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion pour laquelle la tâche doit être supprimée.

**`<Tâche à supprimer> : (Entier)`**

Identifiant de la tâche à supprimer. Cet identifiant est fourni par les fonctions [&lt;Variable Connexion&gt;.AjouteOptimisationPlanifiée](../WDLang4/1000022641.md) et [&lt;Variable Connexion&gt;.ListeOptimisationPlanifiée](../WDLang4/1000022695.md).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Droits nécessaires
<a name="droits_necessaires_ELTPARAGRAPHE000198"></a>Pour avoir le droit de supprimer une tâche d'optimisation planifiée, il est nécessaire :

- soit d'être le propriétaire de la tâche. 

- soit d'être super-utilisateur.




## &lt;Variable Connexion&gt;.InfoTâche (Fonction)

***En anglais : &lt;Connection variable&gt;.InfoTask***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie les caractéristiques d'une tâche planifiée dans une variable de type [hTâchePlanifiée](../WDLang4/1000017448.md). Il est par exemple possible de lire certaines caractéristiques de la tâche planifiée pour les modifier par la suite avec la fonction [&lt;Variable Connexion&gt;.ModifieTâche](../WDLang4/1000022722.md).



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion
MaConnexion est une Connexion

MaTâche est une hTâchePlanifiée
// Récupération des caractéristiques de la tâche planifiée
MaTâche = MaConnexion.InfoTâche(IDTâche)
// Modification de la tâche planifiée
MaTâche.Heure = SAI_Heure.Heure + 1
MaConnexion.ModifieTâche(IDTâche, MaTâche)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Connexion>.InfoTâche(<Tâche planifiée>)`
---

**`<Résultat> : (Variable de type hTâchePlanifiée)`**

Variable de type [hTâchePlanifiée](../WDLang4/1000017448.md) contenant les caractéristiques de la tâche planifiée spécifiée.

**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion pour laquelle une tâche doit être manipulée.

**`<Tâche planifiée> : (Entier)`**

Identifiant de la tâche planifiée renvoyé par les fonctions [&lt;Variable Connexion&gt;.AjouteTâche](../WDLang4/1000022645.md) et [&lt;Variable Connexion&gt;.ListeTâche](../WDLang4/1000022700.md).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Pour vérifier l'existence d'une tâche, utilisez la fonction [&lt;Variable Connexion&gt;.ListeTâche](../WDLang4/1000022700.md) : il suffit de vérifier si l'identifiant de la tâche est présent parmi les tâches renvoyées par la fonction [&lt;Variable Connexion&gt;.ListeTâche](../WDLang4/1000022700.md). 

- Pour récupérer les caractéristiques d'une tâche planifiée, il est nécessaire de posséder le droit de connexion (constante *hDroitConnexion* ou *hDroitConnexionCryptée*) sur la base de données contenant la procédure stockée.

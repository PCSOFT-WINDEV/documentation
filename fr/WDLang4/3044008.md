


## HInfoMémo (Fonction)

***En anglais : HInfoMemo***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie les caractéristiques des mémo binaire et texte.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HLitPremier(Produit, Nom)
InfoMemo est une chaîne 
InfoMemo = HInfoMémo(Produit, Nom)
// Récupérer le nom du fichier original du mémo
NomFichier est une chaîne 
NomFichier = ExtraitChaîne(InfoMemo, 2, TAB)
```

<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître les caractéristiques d'une rubrique Mémo

`<Résultat> = HInfoMémo(<Fichier de données> , <Rubrique mémo>)`
---

**`<Résultat> : (Chaîne de caractères)`**



- Informations sur le mémo binaire. 

- Chaîne vide ("") si le mémo est vide.




Les informations sont séparées par le caractère TAB. Ces informations sont les suivantes : 

```txt
<Type>+TAB+<Nom>+TAB+<Taille>+TAB+<Date>+TAB+<Heure>+TAB+<Info>
```



|   |   |
| --- | --- |
| Type | Type du mémo. Ce paramètre peut prendre pour valeur "TXT" pour les mémos texte, "IMG", "WAV", "OLE", "BIN" ou "BAD" ("BAD" indique que le mémo binaire est incorrectement stocké) |
| Nom | Nom du fichier d'origine tel qu'il a été donné dans la fonction [HAttacheMémo](../WDLang4/3044173.md). |
| Taille | Taille du fichier d'origine en octets. Pour un mémo texte, cette taille est donnée en caractères. |
| Date | Date du fichier d'origine au format "AAAAMMJJ". |
| Heure | Heure du fichier d'origine au format "HHMMSS" |
| Info | Information utilisateur telle qu'elle a été donnée dans la fonction [HAttacheMémo](../WDLang4/3044173.md) |


**Si le mémo est un mémo texte**, le nom, la date, l'heure et l'information utilisateur ne sont pas renseignés. Le type vaut "TXT". 
Exemple de valeur renvoyée pour un mémo texte de taille 25 : "TXT"+TAB+TAB+25+TAB+TAB+TAB.

**Si le mémo est un mémo binaire** qui n'a pas été affecté par la fonction [HAttacheMémo](../WDLang4/3044173.md), le nom, la date, l'heure et l'information ne sont pas renseignées. Le type vaut "BIN". 
Exemple de valeur renvoyée pour un mémo binaire de taille 1202 : "BIN"+TAB+TAB+1202+TAB+TAB+TAB.
![OLE DB](https://doc.pcsoft.fr/ext/images/fr/OLEDB.png)![Connecteurs Natifs (Accès Natifs)](https://doc.pcsoft.fr/ext/images/fr/AN.png) Les informations sur le mémo ne sont pas enregistrées. La seule information retournée par la fonction **HInfoMémo** correspond à : 

- la taille du mémo, 

- une chaîne vide ("") si il n'y a pas de taille. 


Par exemple : "TXT" (ou "BIN")+TAB+TAB+25+TAB+TAB+TAB.

**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données (de la requête ou de la vue) manipulé.

**`<Rubrique mémo> : (Chaîne de caractères)`**

Nom de la rubrique mémo à manipuler.

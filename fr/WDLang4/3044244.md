
## Convert (Fonction)

***En anglais : Convert***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Convertit une valeur numérique en une chaîne binaire pour effectuer une recherche sur une clé numérique.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// CleComp est composée de REFPROD (texte) et QTE (entier)
// SAI_Quantité et SAI_RefProduit sont deux champs de saisie
Rech est une chaîne
Rech = Complète(SAI_RefProduit, 8) + Convert(SAI_Quantité)
HLitRecherche(Commande, CleComp, Rech)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = Convert(<Valeur numérique>)`
---

**`<Résultat> : (Chaîne de caractères)`**

Résultat de la conversion. Ce résultat est la chaîne binaire correspondant à la valeur de la variable numérique. Cette chaîne binaire ne peut pas être directement affichée. Cette chaîne binaire sert uniquement pour la recherche dans le fichier de données HFSQL.

**`<Valeur numérique> : (Variable de type Numérique)`**

Nom de la variable numérique à convertir  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Le type de la conversion dépend du type de la variable passée en paramètre.

- La fonction **Convert** doit être utilisée pour effectuer des recherches dans un fichier de données HFSQL sur une clé composée numérique. Pour les clés simples numériques, il ne faut pas convertir la valeur numérique en chaîne binaire, la conversion est automatique.

- La fonction **Convert** ne peut pas être utilisée en langage externe. Il faut utiliser la fonction [HConvertX](../WDLang4/3044260.md).

- La fonction [HConvertX](../WDLang4/3044260.md) a la même action que la fonction **Convert**. Avec la fonction **Convert** (et contrairement à la fonction [HConvertX](../WDLang4/3044260.md)), il n'est pas nécessaire d'indiquer le type de la valeur à convertir, le type est automatiquement déduit du type de la variable &lt;Valeur_Numérique&gt;.

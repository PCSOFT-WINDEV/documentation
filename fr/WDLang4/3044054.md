
## HFermeAnalyse (Fonction)

***En anglais : HCloseAnalysis***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ferme l'analyse en cours. Tous les fichiers de données, vues HFSQL et requêtes actuellement ouverts sont fermés et libérés. Après l'exécution de la fonction **HFermeAnalyse**, aucune analyse n'est en cours.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png)![HFSQL Classic](https://doc.pcsoft.fr/ext/images/fr/HF.png) 
```wl
// Ouvre une analyse
HOuvreAnalyse("C:\Gestcom\Gestcom.WDD")
// Traitements 
...
// Ferme l'analyse actuellement utilisée par l'application
HFermeAnalyse()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HFermeAnalyse()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'analyse a été fermée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème : l'analyse n'est pas fermée. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Une seule analyse peut être ouverte à un moment donné.

- Seule une analyse ouverte avec la fonction [HOuvreAnalyse](../WDLang4/3044106.md) peut être fermée avec la fonction **HFermeAnalyse**.

- Si aucune analyse n'est en cours, la fonction **HFermeAnalyse** n'a aucun effet.

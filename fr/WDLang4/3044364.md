


## HCS.TailleTrame (Variable)

***En anglais : HCS.FrameSize***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La variable **HCS.TailleTrame** permet de modifier la taille de la trame de données envoyée. La modification de la taille des trames permet dans certains cas d'adapter le flux des données envoyées et reçues aux performances du matériel utilisé. 

Par exemple, si la connexion entre les postes client et le moteur HFSQL Client/Serveur passe par des installations réseaux (carte réseau, switch, routeur, vpn, ...) aux réglages non standard ou défaillants, des trames sont perdues si elles excédent une certaine taille. Il est alors possible d'utiliser cette variable.  

**Dans le cas général, si le matériel est récent, de qualité et correctement configuré, il n'est pas nécessaire d'intervenir sur la taille des trames.**




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HCS.TailleTrame = hTrameInternet
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`HCS.TailleTrame = <Valeur de la trame>`
---

**`<Valeur de la trame> : (Constante de type Entier ou Entier)`**

Valeur de la trame (en octets). Correspond à une valeur ou à une des constantes prédéfinies suivantes : 


|   |   |
| --- | --- |
| *hTrameActiveSync* | Taille de trame optimisée pour une utilisation avec des Mobiles. |
| *hTrameDéfaut* | Taille de trame standard utilisée par défaut. |
| *hTrameGPRS* | Taille de trame optimisée pour une utilisation par GPRS. |
| *hTrameInternet* | Taille de trame optimisée pour une utilisation par Internet. |
| *hTrameIntranet* | Taille de trame optimisée pour une utilisation par Intranet. |
| *hTrameWifi* | Taille de trame optimisée pour une utilisation par Wi-Fi. |

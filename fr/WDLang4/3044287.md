


## HFinInterditAccèsBaseDeDonnées (Fonction)

***En anglais : HEndNoDatabaseAccess***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ré-autorise l'accès (c'est-à-dire la connexion) à une ou plusieurs bases de données accessibles par une connexion.

**Remarque** : L'accès aux bases de données avait été précédemment interdit grâce à la fonction [HInterditAccèsBaseDeDonnées](../WDLang4/3044286.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI HInterditAccèsBaseDeDonnées("MaConnexion", "BaseClient") = Vrai ALORS
	// Traitement spécifique
	...
	HFinInterditAccèsBaseDeDonnées("MaConnexion", "BaseClient") 
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HFinInterditAccèsBaseDeDonnées(<Connexion> [, <Base de données>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'accès à la base de données spécifiée (ou à toutes les bases de données accédées par la connexion) est rétabli, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Connexion> : (Chaîne de caractères ou variable de type Connexion)`**

Connexion contenant la base de données dont l'accès doit être rétabli. Cette connexion correspond :

- soit au nom de la connexion définie sous l'éditeur d'analyses ou grâce aux fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

- soit au nom d'une variable de type [Connexion](../WDLang4/1514073.md).




**`<Base de données> : (Chaîne de caractères optionnelle)`**

Nom de la base de données pour laquelle l'accès doit être rétabli.
Si ce paramètre correspond à "\*", l'accès sera rétabli pour toutes les bases de données.
Si ce paramètre n'est pas spécifié, l'accès sera rétabli pour toutes les bases de données accessibles par &lt;Connexion&gt;.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Connexion à une base de données bloquée
<a name="connexion_une_base_donnees_bloquee_ELTPARAGRAPHE000126"></a>Dans certains cas (coupure de courant par exemple), il se peut que les connexions à une base de données soient refusées. Par exemple, si une coupure de courant survient pendant l'exécution de ce code (avant la fonction [HInterditAccèsBaseDeDonnées](../WDLang4/3044286.md)) :

```wl
// Description de la connexion
HDécritConnexion("MaConnexion", "Julie", "MotDePasse", "MonServeurHF", ...
		"BaseClient", hAccèsHFClientServeur, HOLecture)
// Ouverture de la connexion
HOuvreConnexion("MaConnexion")
SI HInterditAccèsBaseDeDonnées("MaConnexion", "BaseClient") = Vrai ALORS
	Info("Il est impossible de se connecter à la base de données")
FIN
```

Pour autoriser à nouveau les accès, il suffit de faire une connexion sans préciser de base de données, puis d'utiliser la fonction **HFinInterditAccèsBaseDeDonnées**.
<a name="NOTE0_2"></a>


### Autoriser l'accès à une base de données bloquée par le Centre de Contrôle HFSQL
<a name="autoriser_acces_une_base_donnees_bloquee_par_centre_controle_hfsql_ELTPARAGRAPHE000150"></a>Lorsque les connexions à une base de données HFSQL Client/Serveur sont refusées, il est également possible d'autoriser à nouveau l'accès via le [Centre de Contrôle HFSQL](../CCHF/35400015.md). Pour cela, il suffit de réaliser les manipulations suivantes :

- Se connecter en tant qu'administrateur au serveur sans indiquer la base de données,

- Afficher l'onglet "Connexions" du serveur,

- Dans le ruban, dans le groupe "Connexions", dérouler le bouton "Blocage des connexions" et sélectionner l'option "Autoriser toutes les connexions".

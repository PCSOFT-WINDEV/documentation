


## &lt;Variable Connexion&gt;.ModifiePropriétéBaseDeDonnées (Fonction)

***En anglais : &lt;Connection variable&gt;.ModifyDatabaseProperty***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie les propriétés d'une base de données située sur un serveur HFSQL.

**Remarque** : Il est possible de connaître les propriétés d'une base de données grâce à la fonction [&lt;Variable Connexion&gt;.InfoPropriétéBaseDeDonnées](../WDLang4/1000022673.md).



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion
MaConnexion est une Connexion

ResModif est un booléen
ResModif = MaConnexion.ModifiePropriétéBaseDeDonnées(hPropriétaire, "Florence", "MaBase")
SI ResModif = Vrai ALORS
	Info("Modification effectuée")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Connexion>.ModifiePropriétéBaseDeDonnées(<Propriété à modifier> , <Nouvelle valeur> , <Base de données>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si les propriétés ont été modifiées,

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion pour laquelle la valeur d'une propriété doit être modifiée.

**`<Propriété à modifier> : (Constante de type Entier)`**

Propriété de la base de données à modifier: 


|   |   |
| --- | --- |
| *hNotes* | Description de la base de données ou toute autre information associée à la base de données. <br>Si cette constante est utilisée, &lt;Nouvelle valeur&gt; est une chaîne de caractères contenant le texte de la note. |
| *hPropriétaire* | Propriétaire de la base de données. Le propriétaire est celui qui possède la base de données. Le propriétaire d'une base de données peut changer. La notion de propriétaire impacte la gestion des droits.<br>Si cette constante est utilisée, &lt;Nouvelle valeur&gt; est une chaîne de caractères contenant le nouveau nom du propriétaire. |



**`<Nouvelle valeur> : (Chaîne de caractères)`**

Nouvelle valeur de la propriété.

**`<Base de données> : (Chaîne de caractères)`**

Nom de la base de données pour laquelle les propriétés doivent être modifiées. 
Si ce paramètre correspond à une chaîne vide (""), la base de données utilisée sera celle définie dans la connexion. 
Si aucune base de données n'est définie dans la connexion, une erreur WLangage sera générée.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Changer le propriétaire d'une base de données
<a name="changer_proprietaire_une_base_donnees_ELTPARAGRAPHE000259"></a>

- L'utilisateur spécifié comme nouveau propriétaire doit être un utilisateur existant sur le Serveur. Si l'utilisateur n'existe pas, une erreur WLangage est générée.

- Pour modifier le propriétaire, l'utilisateur faisant la modification doit :

	- soit être un "Super utilisateur".

	- soit avoir le droit de changer le propriétaire de la base de données (droit correspondant à la constante *hDroitChangePropriétaire*).





Pour savoir si l'utilisateur a le droit approprié, utilisez la fonction [&lt;Variable Connexion&gt;.InfoPropriétéBaseDeDonnées](../WDLang4/1000022673.md).

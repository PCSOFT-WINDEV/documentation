


## &lt;Variable Connexion&gt;.AjouteUtilisateur (Fonction)

***En anglais : &lt;Connection variable&gt;.AddUser***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute un utilisateur à une base de données. Toutes les caractéristiques de l'utilisateur doivent être saisies dans les [différentes variables de gestion des utilisateurs](../WDLang4/3044315.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion 
MaConnexion est une Connexion

// Ouverture de la connexion
MaConnexion.OuvreConnexion()
// Association du fichier de données à la connexion
Client.ChangeConnexion(MaConnexion)
// Caractéristiques du nouvel utilisateur
HUtilisateur.Login = "FLO"
HUtilisateur.MotDePasse = "FLOP"
HUtilisateur.NomComplet = "Florence"
HUtilisateur.Email = "Flo@monadresse.fr"
HUtilisateur.Téléphone = "01.67.40.47.58"
HUtilisateur.Groupes = "ADMIN" + RC + "TEST"
// Ajout de l'utilisateur
SI MaConnexion.AjouteUtilisateur() = Vrai ALORS
	Info("Utilisateur ajouté")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Connexion>.AjouteUtilisateur()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'utilisateur a été ajouté,

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion pour laquelle un utilisateur doit être ajouté.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Droits nécessaires pour ajouter un utilisateur
<a name="droits_necessaires_pour_ajouter_utilisateur_ELTPARAGRAPHE000251"></a>Pour ajouter un utilisateur, il est nécessaire de posséder les droits suivants :

- Droit de gestion des utilisateurs (constante *hDroitGestionUtilisateur*).

- Droit de visualisation des utilisateurs (constante *hDroitVisualisationUtilisateur*).


Ces constantes sont utilisées dans les fonctions [&lt;Variable Connexion&gt;.InfoDroitServeur](../WDLang4/1000022668.md) et [&lt;Variable Connexion&gt;.ModifieDroitServeur](../WDLang4/1000022710.md).
<a name="NOTE0_2"></a>


### Gestion des droits
<a name="gestion_des_droits_ELTPARAGRAPHE000276"></a>Par défaut, lors de la création d'un utilisateur (si cet utilisateur n'est associé à aucun groupe), cet utilisateur n'a aucun droit.

Si cet utilisateur est associé à un ou plusieurs groupes, cet utilisateur a alors les droits associés à ce groupe (l'utilisateur "hérite" des droits associés au groupe).

Il est possible de donner des droits à un utilisateur grâce aux fonctions :

- [&lt;Variable Connexion&gt;.ModifieDroitBaseDeDonnées](../WDLang4/1000022707.md) : permet de donner des droits sur une base de données.

- [&lt;Variable Connexion&gt;.ModifieDroitFichier](../WDLang4/1000022709.md) : permet de donner des droits sur un fichier de données.

- [&lt;Variable Connexion&gt;.ModifieDroitServeur](../WDLang4/1000022710.md) : permet de donner des droits sur un serveur.


Pour plus de détails sur la gestion des droits, consultez [Gestion des droits en HFSQL Client/Serveur](../WDLang4/3044333.md).
<a name="NOTE0_3"></a>


### Utilisateur "SuperUtilisateur"
<a name="utilisateur_superutilisateur_ELTPARAGRAPHE000311"></a>Si la variable HUtilisateur.SuperUtilisateur = Vrai, les droits de l'utilisateur ne sont pas vérifiés lors d'une action. Cet utilisateur peut donc faire toutes les actions.




## HJournalRedémarre (Fonction)

***En anglais : HLogRestart***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Redémarre la journalisation du fichier de données. Cette journalisation a été arrêtée grâce à la fonction [HJournalStop](../WDLang4/3044035.md).

**Remarque** : La journalisation n'est pas disponible lors de l'accès à une base de données via .


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HJournalInfo("Client", "début impression de " + Critère)
HJournalStop("Client") // Stopper le journal
HLitPremier("Client", Critère)
TANTQUE HTrouve()
	// Traitement
	HLitSuivant("Client")
FIN

HJournalRedémarre("Client") // Redémarrer le journal
HJournalInfo("Client", "fin impression de" + Critère) // Ecrire
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HJournalRedémarre(<Fichier de données>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données journalé dont la journalisation doit redémarrer.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Lors du redémarrage du journal, le fichier de données doit être dans son état initial, c'est-à-dire l'état dans lequel il était avant l'utilisation de la fonction [HJournalStop](../WDLang4/3044035.md).

- Entre les fonctions [HJournalStop](../WDLang4/3044035.md) et **HJournalRedémarre**, les informations ne seront pas sauvegardées dans le journal.

- La fonction **HJournalRedémarre** est équivalente à l'utilisation de la fonction [HGèreJournal](../WDLang4/3044052.md) avec le paramètre <u><u><u><u>Vrai</u></u></u></u>.

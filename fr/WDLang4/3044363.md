


## HCS.DélaiReponse (Variable)

***En anglais : HCS.Timeout***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
La variable **HCS.DélaiRéponse** permet de modifier le délai de connexion :

- pour les connexions réalisées avec la fonction [HOuvreConnexion](../WDLang4/3044107.md).

- pour les ouvertures automatiques de connexion.




**Remarque** : La variable **HCS.DélaiRéponse** n'est pas prise en compte pour les accès au serveur autres que la connexion. Une requête dont l'exécution serait plus longue que la valeur spécifiée dans la variable **HCS.DélaiRéponse** ne sera pas interrompue par exemple..

Ce délai doit être précisé uniquement dans l'application cliente, avant l'ouverture d'une connexion sur le serveur.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HCS.DélaiRéponse = Intranet
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`HCS.DélaiReponse = <Valeur>`
---

**`<Valeur> : (Constante de type Entier, Entier ou Durée)`**

Valeur du délai de connexion. Correspond à un délai en millisecondes (supérieur à 500) ou à une des constantes prédéfinies suivantes : 


|   |   |
| --- | --- |
| *Internet* | Temps d'attente de réponse (TimeOut) utilisé pour la connexion : Valeur adaptée à l'utilisation d'Internet (temps d'attente de réponse important) |
| *Intranet* | Temps d'attente de réponse (TimeOut) utilisé pour la connexion : Valeur adaptée à l'utilisation d'Intranet (temps d'attente de réponse faible) |


Ce paramètre peut correspondre à : 

- un entier correspondant au nombre de millisecondes, 

- une variable de type Durée, 

- l'indication directe de la durée (par exemple '1s' ou '10ms').

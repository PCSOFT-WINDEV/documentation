
## HGèreREP (Fonction)

***En anglais : HSetREP***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active ou désactive la gestion du fichier .REP. Pour plus de détails sur la gestion du fichier .REP, consultez [Fichier .REP](../WDLang4/3044186.md).



**Remarque** : Par défaut, la gestion du fichier .REP est active.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Désactive la gestion du fichier .REP
HGèreREP(Faux)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HGèreREP(<Gestion> [, <Chemin du .REP> [, <Nom du fichier .REP>]])`
---

**`<Résultat> : (Booléen)`**

Correspond à l'ancien mode de gestion du fichier .REP (avant utilisation de la fonction **HGèreREP**) : 

- <u><u><u><u>Vrai</u></u></u></u> si la gestion du .REP était active, 

- <u><u><u><u>Faux</u></u></u></u> si la gestion du .REP n'était pas active.




**`<Gestion> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la gestion du .REP est activée, 

- <u><u><u><u>Faux</u></u></u></u> si cette gestion doit être désactivée.




**`<Chemin du .REP> : (Chaîne de caractères optionnelle)`**

Si ce paramètre est précisé, la fonction **HGèreREP** utilise le fichier .REP présent dans le répertoire spécifié. 
Si ce paramètre n'est pas précisé, le chemin par défaut utilisé correspond : 


- au précédent chemin du .REP précisé pour l'analyse en cours.

- si aucun chemin n'a été précédemment précisé : 
	1. au chemin de l'analyse (.WDD), 
	2. au chemin de la bibliothèque de l'application (.WDL) 
	3. à défaut, au chemin de l'application (.EXE). C'est le cas le plus courant.




**`<Nom du fichier .REP> : (Chaîne de caractères optionnelle)`**

Si ce paramètre est spécifié, la fonction **HGèreREP** utilise ce nom pour le fichier .REP. 
Si ce paramètre n'est pas précisé, le nom de l'analyse est utilisé comme nom de fichier .REP. 
Dans tous les cas, le fichier .REP a pour extension ".REP".  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques




### Activation du .REP
<a name="activation_rep_ELTPARAGRAPHE000089"></a>

- **Si le paramètre &lt;Gestion&gt; vaut <u><u><u><u>Vrai</u></u></u></u>**, le .REP sera mis à jour à chaque ouverture ou création d'un fichier de données de l'analyse en cours. L'outil de modification automatique des fichiers de données saura alors exactement quels fichiers modifier.

- **Si le paramètre &lt;Gestion&gt; vaut <u><u><u><u>Faux</u></u></u></u>**, aucune mise à jour du .REP n'est effectuée. Mais dans ce cas, il faudra préciser manuellement à l'outil de modification automatique des fichiers de données la localisation des dits fichiers. Si le nom du .REP est précisé, ce nom sera utilisé à la prochaine activation (si un nouveau nom n'est pas précisé entre-temps).

- Le fichier .REP est géré par contexte. Un composant utilisant un contexte HFSQL indépendant ne sera pas impacté par l'appel à la fonction **HGèreREP**.




**Remarque** : Lorsque le .REP n'est pas celui par défaut, sa localisation est stockée dans le fichier **HFRep.ini**. Ce fichier est présent dans le répertoire "Application Data\\PC SOFT" des données partagées de l'application (par compatibilité avec les versions précédentes, ce fichier peut également être présent dans le répertoire de Windows).

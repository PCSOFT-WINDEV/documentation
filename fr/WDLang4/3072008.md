
## SQLDéconnecte (Fonction)

***En anglais : SQLDisconnect***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ferme la connexion en cours et libère l'espace mémoire utilisé par la connexion. La fonction **SQLDéconnecte** doit être appelée systématiquement pour fermer la connexion, même si cette connexion a échoué.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png)![Code Utilisateur (MCU)](https://doc.pcsoft.fr/ext/images/fr/MCU.png)![HFSQL Classic](https://doc.pcsoft.fr/ext/images/fr/HF.png)![HFSQL Client/Serveur](https://doc.pcsoft.fr/ext/images/fr/HFCS.png)![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) 
```wl
// Exemple attaché à l'analyse GESCOM
// TABLE_TableReq est un champ Table par programmation
ResConnect est un entier
ResExec est un booléen
MaBase est une chaîne = "C:\GESCOM.WDD"
ResConnect = SQLConnecte(MaBase, "", "")
SI ResConnect <> 0 ALORS
	ResExec = SQLExec("SELECT NOM, VILLE FROM POP", "REQ1")
	SI ResExec = Vrai ALORS
		// Récupérer les données dans la table
		SQLTable("REQ1", TABLE_TableReq)
	SINON
		// Erreur d'exécution de la requête
		// Récupérer les informations relatives à la requête exécutée
		SQLInfoGene("REQ1")
		Erreur(SQL.MesErreur)
	FIN
	// Libérer la requête dans tous les cas
	SQLFerme("REQ1")
SINON
	 // Erreur de connexion
	 Erreur("La connexion à la base" + MaBase + "a échoué")
FIN
// Libérer la requête dans tous les cas
SQLDéconnecte()
```

## Remarques

### Connexion avec une analyse HFSQL
<a name="connexion_avec_une_analyse_hfsql_ELTPARAGRAPHE000089"></a>Dans le cas d'une connexion avec une analyse HFSQL, la fonction **SQLDéconnecte** ferme l'analyse (équivalent à la fonction [HFermeAnalyse](../WDLang4/3044054.md)). Ainsi, pour continuer à manipuler les fichiers de l'analyse, il faut ouvrir l'analyse avec la fonction [HOuvreAnalyse](../WDLang4/3044106.md).


<a name="NOTE0_2"></a>


### Divers
<a name="divers_ELTPARAGRAPHE000107"></a>

- Une seule connexion peut être en cours à un moment donné.

- La libération de mémoire allouée lors de la connexion (exécution de "SQLFreeConnect") est automatiquement gérée.

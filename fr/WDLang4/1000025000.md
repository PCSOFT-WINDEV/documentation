


## &lt;Source&gt;.JournalInfo (Fonction)

***En anglais : &lt;Source&gt;.LogInfo***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute des commentaires dans le journal lors de l'enregistrement de l'opération journalée. Ces commentaires seront utilisés pour toutes les opérations effectuées dans le contexte HFSQL en cours, jusqu'au prochain appel de la fonction **&lt;Source&gt;.JournalInfo**.

**Remarque** : La journalisation n'est pas disponible lors de l'accès à une base de données via .


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Client.JournalInfo("Ajout de " + Client.NomClient)
Client.Ajoute()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE2"></a>

### Définir un commentaire spécifique par fichier de données journalé

`<Résultat> = <Source>.JournalInfo(<Message>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u>Vrai</u></u></u> si l'opération a été réalisée, 

- <u><u><u>Faux</u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données journalé auquel le message se rapporte. 

**`<Message> : (Chaîne de caractères)`**

Message à insérer dans les informations de journalisation (16 caractères maximum, si la chaîne est plus longue, les informations seront tronquées). Ces informations pourront être récupérées grâce à l'outil [WDJournal](../WDJournal/3516012.md). 
Si ce paramètre n'est pas spécifié, aucun message n'est inséré.


<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Pour insérer des informations différentes lors de chaque opération, il est nécessaire d'appeler la fonction **&lt;Source&gt;.JournalInfo** avant chaque opération.

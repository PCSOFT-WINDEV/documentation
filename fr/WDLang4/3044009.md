


## HJournalInfo (Fonction)

***En anglais : HLogInfo***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute des commentaires dans le journal lors de l'enregistrement de l'opération journalée. Ces commentaires seront utilisés pour toutes les opérations effectuées dans le contexte HFSQL en cours, jusqu'au prochain appel de la fonction **HJournalInfo**.

**Remarque** : La journalisation n'est pas disponible lors de l'accès à une base de données via .


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HJournalInfo("Ajout de " + Client.NomClient)
HAjoute(Client)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Définir un commentaire

`<Résultat> = HJournalInfo([<Message>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Message> : (Chaîne de caractères optionnelle)`**

Message à insérer dans les informations de journalisation (16 caractères maximum, si la chaîne est plus longue, les informations seront tronquées). Ces informations pourront être récupérées grâce à l'outil [WDJournal](../WDJournal/3516012.md). 
Si ce paramètre n'est pas spécifié, aucun message n'est inséré. 
Ce message sera pris en compte pour tous les fichiers de données pour lesquels un commentaire spécifique n'a pas été défini (syntaxe 2).  


<a name="SYNTAXE2"></a>

### Définir un commentaire spécifique par fichier de données journalé

`<Résultat> = HJournalInfo(<Fichier de données> , <Message>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Fichier de données> : (Chaîne de caractères)`**

Nom du fichier de données journalé auquel le message se rapporte. Si ce paramètre vaut "\*", tous les fichiers de données journalés présents dans l'analyse sont pris en compte.

**`<Message> : (Chaîne de caractères)`**

Message à insérer dans les informations de journalisation (16 caractères maximum, si la chaîne est plus longue, les informations seront tronquées). Ces informations pourront être récupérées grâce à l'outil [WDJournal](../WDJournal/3516012.md). 
Si ce paramètre n'est pas spécifié, aucun message n'est inséré.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Pour insérer des informations différentes lors de chaque opération, il est nécessaire d'appeler la fonction **HJournalInfo** avant chaque opération.

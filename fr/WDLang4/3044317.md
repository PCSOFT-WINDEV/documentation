


## HArrêteServeur (Fonction)

***En anglais : HStopServer***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Arrête un serveur HFSQL. Le serveur peut ensuite être redémarré grâce à la fonction [HDémarreServeur](../WDLang4/3044318.md).

**Remarque** : Pour réaliser cette action, l'utilisateur doit posséder un droit spécifique sur le serveur.

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion
HDécritConnexion("MaConnexion", "Julie", "MotDePasse", "MonServeurHF", ...
	"MaBaseDeDonnées", hAccèsHFClientServeur, HOLecture)
// Ouverture de la connexion
HOuvreConnexion("MaConnexion")
...
HArrêteServeur("MaConnexion")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HArrêteServeur(<Connexion>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le serveur a été arrêté, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Connexion> : (Chaîne de caractères ou variable de type Connexion)`**

Connexion pour laquelle le serveur doit être arrêté. Cette connexion correspond :

- soit au nom de la connexion définie sous l'éditeur d'analyses ou grâce aux fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

- soit au nom d'une variable de type [Connexion](../WDLang4/1514073.md). 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Arrêt du serveur
<a name="arret_serveur_ELTPARAGRAPHE000083"></a>Si des applications Clientes sont en cours d'utilisation lors de l'arrêt du serveur, un message d'erreur ("Serveur non disponible") sera affiché lors de la prochaine action effectuée par l'application cliente sur le serveur.

**Remarque** : Suite à un arrêt du serveur, il est possible que le serveur ne puisse pas être redémarré tout de suite. Le système d'exploitation ne ferme pas toujours immédiatement les connexions qui existaient entre les clients connectés et le serveur. Il suffit d'attendre quelques minutes (selon les timeouts du système d'exploitation) et le serveur peut être relancé.
<a name="NOTE0_2"></a>


### Gestion des droits
<a name="gestion_des_droits_ELTPARAGRAPHE000094"></a>

- Pour arrêter le serveur, l'utilisateur ou le groupe d'utilisateurs doit posséder le droit correspondant (constante *hDroitArrêtServeur*).

- Pour savoir si un utilisateur ou un groupe d'utilisateurs possède ce droit, utilisez la fonction [HInfoDroitServeur](../WDLang4/3044322.md).

- Pour modifier les droits d'un utilisateur ou d'un groupe d'utilisateurs sur le serveur, utilisez la fonction [HModifieDroitServeur](../WDLang4/3044328.md).

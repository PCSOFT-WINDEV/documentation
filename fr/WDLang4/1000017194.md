
## RéplicOuvre (Fonction)

***En anglais : ReplicOpen***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ouvre une réplication en mode direct ou avec un serveur de réplication. Cette réplication a été définie par l'éditeur de réplication (ReplicEdit).






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouvre une réplication en mode direct
RéplicOuvre("MaRéplication")
```


<a name="Exemple2"></a>



```wl
// Ouvre une réplication sur le serveur de réplication
RéplicOuvre("MaRéplication", "", "ServeurReplication", "Florence", "motpasse")

// Initialise la réplication en passant le numéro du département 
// à prendre en compte
SI RéplicInfo("MaRéplication", replicInitialisée) = Faux ALORS
	RéplicInitialise("MaRéplication", 47012)
FIN

// Lance la synchronisation
RéplicSynchronise("MaRéplication")

// Traitement des erreurs renvoyées par les fonctions de Réplication
CAS Erreur :
	Erreur(ErreurInfo(errComplet))
	RETOUR
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Ouverture d'une réplication en mode direct

`<Résultat> = RéplicOuvre(<Réplication> [, <Répertoire de travail>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la réplication a été ouverte, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. Pour plus de détails sur le problème rencontré, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Réplication> : (Chaîne de caractères)`**

Nom du fichier de réplication à ouvrir (fichier d'extension ".wer"), sans chemin ni extension. Ce fichier a été créé grâce à l'éditeur de réplication (ReplicEdit). Ce fichier doit être présent à côté de l'exécutable.

**`<Répertoire de travail> : (Chaîne de caractères optionnelle)`**

Répertoire dans lequel la réplication sera effectuée (création des fichiers nécessaires à la réplication). Par défaut (ou si ce paramètre correspond à chaîne vide ("")) : 
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) ce répertoire correspond au répertoire de l'exécutable de l'application (répertoire renvoyé par [fRepExe](../WDLang1/3036049.md)).


<a name="SYNTAXE2"></a>

### Ouverture d'une réplication avec un serveur de réplication

`<Résultat> = RéplicOuvre(<Réplication> , <Répertoire de travail> , <Adresse du serveur> , <Utilisateur> , <Mot de passe>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la connexion au serveur de réplication a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. Pour plus de détails sur le problème rencontré, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md).




**`<Réplication> : (Chaîne de caractères)`**

Nom de la réplication à ouvrir. Ce nom correspond au nom de publication de la réplication.

**`<Répertoire de travail> : (Chaîne de caractères)`**

Répertoire dans lequel la réplication sera effectuée (création des fichiers nécessaires à la réplication). Par défaut (ou si ce paramètre correspond à chaîne vide ("")) : 
![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) ce répertoire correspond au répertoire de l'exécutable de l'application (répertoire renvoyé par [fRepExe](../WDLang1/3036049.md)).

**`<Adresse du serveur> : (Chaîne de caractères)`**

Adresse du serveur de réplication. Cette adresse peut correspondre :

- soit à l'adresse IP du serveur au format xxx.xxx.xxx.xxx

- soit au nom du poste serveur.




**`<Utilisateur> : (Chaîne de caractères)`**

Nom de l'utilisateur. Ce nom correspond à un compte créé sur le serveur de réplication.

**`<Mot de passe> : (Chaîne de caractères)`**

Mot de passe associé à l'utilisateur.

<a name="NOTE0_1"></a>

## Remarques


### Utilisation en mode direct
<a name="utilisation_mode_direct_ELTPARAGRAPHE000137"></a>L'appel de cette fonction est facultatif en mode direct car elle est appelée automatiquement par les autres fonctions de la réplication (avec les paramètres par défaut) si elle n'a pas été appelée avant.


### Ouverture de ports
<a name="ouverture_ports_ELTPARAGRAPHE000142"></a>Cette fonction utilise le serveur de réplication qui utilise le serveur Web sur lequel il est installé. Il est doc nécessaire d'ouvrir le port 80 sur le serveur de réplication.

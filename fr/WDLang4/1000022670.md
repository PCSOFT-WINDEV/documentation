


## &lt;Variable Connexion&gt;.InfoGroupe (Fonction)

***En anglais : &lt;Connection variable&gt;.InfoGroup***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie des informations sur le groupe d'utilisateurs spécifié. Ces informations sont automatiquement affectées [aux variables de gestion des groupes d'utilisateurs](../WDLang4/3044315.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion
MaConnexion est une Connexion

// Ouverture de la connexion
MaConnexion.OuvreConnexion()
// Association du fichier de données à la connexion
Client.ChangeConnexion(MaConnexion)
...
// Recherche des caractéristiques d'un groupe
Res est un booléen
Res = MaConnexion.InfoGroupe("Groupe_Test")
SI Res = Vrai ALORS
	Trace("Structure du groupe mise à jour")
	Trace("Nom du groupe : " + HGroupe.Nom)
	Trace("Description du groupe : " + HGroupe.Description)
	Trace("Utilisateurs du groupe : " + HGroupe.Utilisateurs)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Connexion>.InfoGroupe(<Groupe>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si les informations concernant le groupe ont été lues, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




Remarque : Une erreur "Groupe inconnu" sera renvoyée par la fonction [HErreurInfo](../WDLang4/3044071.md) si l'utilisateur en cours n'a pas le droit de visualisation des utilisateurs (constante *hDroitVisualisationUtilisateur*). Ce droit peut être défini par la fonction [&lt;Variable Connexion&gt;.ModifieDroitServeur](../WDLang4/1000022710.md).

**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion à utiliser.

**`<Groupe> : (Chaîne de caractères)`**

Nom du groupe d'utilisateurs à manipuler.




## &lt;Source&gt;.DébloqueFichier (Fonction)

***En anglais : &lt;Source&gt;.UnlockFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Débloque les enregistrements d'un fichier de données :

- précédemment bloqués avec la fonction [&lt;Source&gt;.BloqueFichier](../WDLang4/1000024196.md).

- bloqués individuellement avec une fonction de lecture bloquante.




Le déblocage du fichier de données peut aussi être obtenu en fermant le fichier de données (fonction [&lt;Source&gt;.Ferme](../WDLang4/1000024958.md)).

Remarque : La fonction **&lt;Source&gt;.DébloqueFichier** ne permet pas de débloquer un fichier de données bloqué par une autre session ou un autre programme. Il est conseillé de débloquer un fichier de données le plus rapidement possible, afin que les autres sessions puissent accéder à ce fichier de données.
![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Cette fonction peut être utilisée avec des fichiers de données HFSQL Client/Serveur.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Blocage
Client.BloqueFichier(HBlocageLectureEcriture)
// Modification du fichier
...
// Déblocage
Client.DébloqueFichier()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.DébloqueFichier()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier de données a été débloqué, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème (par exemple le fichier de données n'est pas bloqué, ...). La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données manipulé. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
La fonction **&lt;Source&gt;.DébloqueFichier** est sans effet si :

- le fichier de données est en mode Mono (voir la fonction [&lt;Source&gt;.Mode](../WDLang4/1000025037.md)).

- le fichier de données n'est pas bloqué.

- le fichier de données a été bloqué par un autre programme.




## HAjouteOptimisationPlanifiée (Fonction)

***En anglais : HAddScheduledOptimization***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute une tâche d'optimisation des fichiers de données HFSQL Client/Serveur.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Recalculer les statistiques des fichiers de la base "MaBase", 
// tous les samedis à 23H00
Optim est une hOptimisationPlanifiée
Optim.Description = "Recalcul des statistiques des fichiers" + ...
	" de la base de données MaBase"

Optim.Planification.Mois = "*"
Optim.Planification.JourDeLaSemaine = "samedi"
Optim.Planification.Heure = "23"
Optim.Planification.Minute = "0"

Optim.Fichier[1].Nom = "MaBase"
Optim.Fichier[1].Option = hOptStat

HAjouteOptimisationPlanifiée(MaConnexion, Optim)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HAjouteOptimisationPlanifiée(<Connexion> , <Tâche d'optimisation planifiée>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si la tâche d'optimisation planifiée a été ajoutée 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Connexion> : (Chaîne de caractères ou variable de type Connexion)`**

Connexion pour laquelle une tâche d'optimisation planifiée doit être ajoutée. Cette connexion correspond : 

- soit au nom de la connexion définie sous l'éditeur d'analyses ou grâce aux fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

- soit au nom d'une variable de type [Connexion](../WDLang4/1514073.md).




**`<Tâche d'optimisation planifiée> : (Variable de type hOptimisationPlanifiée)`**

Nom de la variable de type [hOptimisationPlanifiée](../WDLang4/1000018854.md) manipulée.

<a name="NOTE0_1"></a>

## Remarques


### Droits nécessaires
<a name="droits_necessaires_ELTPARAGRAPHE000097"></a>Pour ajouter une tâche d'optimisation planifiée, il est nécessaire de posséder : 

- le droit correspondant (constante *hDroitGèreTâche*) sur le serveur, 

- pour chaque base concernée, le droit de connexion (constante *hDroitConnexion* ou *hDroitConnexionCryptée*),

- le droit d'exécuter la commande liée à la tâche d'optimisation planifiée.


La tâche sera exécutée avec le compte de l'utilisateur qui a créé la tâche. 

Pour une optimisation du serveur complet, l'utilisateur doit être super-utilisateur.

**Remarque** : Si le serveur est éteint, la tâche d'optimisation planifiée ne sera pas réalisée. La tâche d'optimisation planifiée sera réalisée à la prochaine date disponible lorsque le serveur sera allumé.

Pour savoir si un utilisateur ou un groupe possède ces droits, utilisez la fonction [HInfoDroitServeur](../WDLang4/3044322.md) ou [HInfoDroitDaseDeDonnées](../WDLang4/3044320.md).

Pour modifier les droits d'un utilisateur ou d'un groupe d'utilisateurs, utilisez la fonction [HModifieDroitServeur](../WDLang4/3044328.md) ou [HModifieDroitBaseDeDonnées](../WDLang4/3044326.md).

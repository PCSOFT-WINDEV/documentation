


## Fonctions de liaison (syntaxe préfixée)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Les fonctions de liaison (syntaxe préfixée) sont les fonctions permettant de lier l'interface aux fichiers de données (et inversement). Ces fonctions sont les suivantes :



|   |   |
| --- | --- |
| [&lt;Fenêtre&gt;.DepuisFichier](../WDLang4/1000021387.md) | Initialise automatiquement les champs d'une fenêtre (ou d'une page) avec :<br><br>- les valeurs des rubriques associées dans l'enregistrement en cours (chargé en mémoire) du fichier de données décrit sous l'éditeur d'analyses. <br><br>- les valeurs des variables WLangage associées. <br><br><br>Cette opération est effectuée quel que soit l'état des champs (grisés, inactifs, ou invisibles). |
| [&lt;Fenêtre&gt;.DepuisSource](../WDLang4/1000021388.md) | Initialise automatiquement les champs d'une fenêtre avec : <br><br>- les valeurs des rubriques associées dans l'enregistrement en cours (chargé en mémoire) du fichier de données HFSQL décrit sous l'éditeur d'analyses.<br><br>- les valeurs des variables WLangage associées. <br><br><br> |
| [&lt;Fenêtre&gt;.VersFichier](../WDLang4/1000021385.md) | Initialise automatiquement :<br><br>- la valeur mémoire des rubriques d'un fichier de données avec la valeur des champs de la fenêtre. Les rubriques du fichier de données de base reliées aux champs de la fenêtre sont automatiquement affectées avec les valeurs des champs correspondants.<br><br>- la valeur des variables WLangage avec la valeur des champs de la fenêtre. Les variables WLangage reliées aux champs de la fenêtre sont automatiquement affectées avec les valeurs des champs correspondants.<br><br><br><br><br>Cette fonction est également disponible pour les pages WEBDEV. |
| [&lt;Fenêtre&gt;.VersSource](../WDLang4/1000021386.md) | Initialise automatiquement : <br><br>- la valeur mémoire des rubriques d'un fichier de données avec la valeur des champs de la fenêtre.<br><br>- la valeur des variables WLangage avec la valeur des champs de la fenêtre.<br><br><br><br><br>Cette fonction est également disponible pour les pages WEBDEV. |
| [&lt;Page&gt;.DepuisFichier](../WDLang4/1000021389.md) | Initialise automatiquement les champs d'une page avec :<br><br>- les valeurs des rubriques associées dans l'enregistrement en cours (chargé en mémoire) du fichier de données décrit sous l'éditeur d'analyses. Cette opération est effectuée quel que soit l'état des champs (grisés, inactifs, ou invisibles).<br><br>- les valeurs des variables WLangage associées. Cette opération est effectuée quel que soit l'état des champs (grisés, inactifs, ou invisibles).<br><br><br> |
| [&lt;Page&gt;.DepuisSource](../WDLang4/1000021393.md) | Initialise automatiquement les champs d'une page avec : <br><br>- les valeurs des rubriques associées dans l'enregistrement en cours (chargé en mémoire) du fichier de données HFSQL décrit sous l'éditeur d'analyses.<br><br>- les valeurs des variables WLangage associées.<br><br><br> |
| [&lt;Page&gt;.VersFichier](../WDLang4/1000021390.md) | Initialise automatiquement :<br><br>- la valeur mémoire des rubriques d'un fichier de données avec la valeur des champs de la page. Les rubriques du fichier de données de base reliées aux champs de la fenêtre sont automatiquement affectées avec les valeurs des champs correspondants.<br><br>- la valeur des variables WLangage avec la valeur des champs de la fenêtre. Les variables WLangage reliées aux champs de la page sont automatiquement affectées avec les valeurs des champs correspondants.<br><br><br>Cette opération est effectuée quel que soit l'état des champs (grisés, inactifs, ou invisibles). |
| [&lt;Page&gt;.VersSource](../WDLang4/1000021392.md) | Initialise automatiquement :<br><br>- la valeur mémoire des rubriques d'un fichier de données avec la valeur des champs de la page.<br><br>- la valeur des variables WLangage avec la valeur des champs de la page.<br><br><br> |
| [&lt;Source&gt;.DepuisFenêtre](../WDLang4/1000021403.md) | Initialise automatiquement : <br><br>- la valeur mémoire des rubriques d'un fichier de données avec la valeur des champs de la fenêtre.<br><br>- la valeur des variables WLangage avec la valeur des champs de la fenêtre.<br><br><br>Cette fonction est également disponible pour les pages WEBDEV. |
| [&lt;Source&gt;.DepuisPage](../WDLang4/1000021405.md) | Initialise automatiquement :<br><br>- la valeur mémoire des rubriques d'un fichier de données avec la valeur des champs de la page.<br><br>- la valeur des variables WLangage avec la valeur des champs de la page.<br><br><br> |
| [&lt;Source&gt;.VersFenêtre](../WDLang4/1000021397.md) | Initialise automatiquement les champs d'une fenêtre avec : <br><br>- les valeurs des rubriques associées dans l'enregistrement en cours (chargé en mémoire) du fichier de données HFSQL décrit sous l'éditeur d'analyses.<br><br>- les valeurs des variables WLangage associées. <br><br><br> |
| [&lt;Source&gt;.VersPage](../WDLang4/1000021401.md) | Initialise automatiquement les champs d'une page avec : <br><br>- les valeurs des rubriques associées dans l'enregistrement en cours (chargé en mémoire) du fichier de données HFSQL décrit sous l'éditeur d'analyses. <br><br>- les valeurs des variables WLangage associées. <br><br><br>Cette opération est effectuée quel que soit l'état des champs (grisés, inactifs, ou invisibles). |

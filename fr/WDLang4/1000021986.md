


## &lt;Variable hdfsConnexion&gt;.Renomme (Fonction)

***En anglais : &lt;hdfsConnection variable&gt;.Rename***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie le nom d'un fichier ou d'un répertoire sur un serveur Hadoop.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Connexion
MaConnexion est une hdfsConnexion
MaConnexion.namenode = "MonServeur"
MaConnexion.Utilisateur = "Admin"
MaConnexion.OuvreConnexion()
// Renomme le fichier sur le serveur
MaConnexion.Renomme("/tmp/ancien.txt", "/tmp/nouveau.txt")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = <Connexion>.Renomme(<Ancien nom> , <Nouveau nom> [, <Options>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le renommage a été réalisé,

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [ErreurInfo](../WDLang1/3013008.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type hdfsConnexion)`**

Nom de la variable de type [hdfsConnexion](../WDLang4/1000021650.md) correspondant à la connexion au serveur Hadoop.

**`<Ancien nom> : (Chaîne de caractères)`**

Chemin absolu du fichier ou du répertoire à renommer sur le serveur Hadoop. Ce chemin est sensible à la casse.

**`<Nouveau nom> : (Chaîne de caractères)`**

Nouveau chemin absolu du fichier ou du répertoire sur le serveur Hadoop. Ce chemin est sensible à la casse.

**`<Options> : (Constante de type Entier optionnelle)`**

Options de renommage :


|   |   |
| --- | --- |
| *hdfsDéfaut*<br>(Valeur par défaut) | Si le fichier destination existe déjà, le renommage n'est pas effectué. |
| *hdfsEcraser* | Si le fichier destination existe déjà, il est remplacé par le fichier renommé. |

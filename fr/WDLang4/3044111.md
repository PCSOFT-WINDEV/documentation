
## HPoste (Fonction)

***En anglais : HComputer***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Mémorise un numéro ou un identifiant unique de poste afin d'utiliser le journal et les transactions en réseau. Le poste correspond à l'ordinateur de l'utilisateur. 

Par défaut, le poste est identifié par un numéro unique interne et par le nom du poste (nom défini sous Windows). Pour identifier simplement le poste effectuant les opérations en transaction, **HPoste** permet de définir un identifiant spécifique au poste. Cet identifiant (non obligatoire) remplace le nom du poste et peut être visualisé sous [WDTrans](../WDTrans/3524005.md) et [WDJournal](../WDJournal/3516012.md).




<a name="sample_code"></a>

## Exemple

<a name="Exemple1"></a>



```wl
HPoste("45000")
```


<a name="Exemple2"></a>



```wl
// Affectation du numéro de poste lu dans l'entrée "NUMPOSTE"
// de la section "MONPROGRAMME" du fichier "WIN.INI"
NumPoste est une chaîne
NumPoste = INILit("MONPROGRAMME", "NUMPOSTE", "")
SI NumPoste = "" ALORS
	Erreur("Problème de récupération du numéro de poste")
SINON
	HPoste(NumPoste)
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HPoste(<Identifiant>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le numéro de poste a été mémorisé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [HErreur](../WDLang4/3044088.md) permet d'obtenir plus d'informations sur le problème rencontré.




**`<Identifiant> : (Chaîne de caractères)`**

Identifie le poste en cours (16 caractères maximum). Cet identifiant est visible avec WDTrans et WDJournal. Cet identifiant correspond par défaut au nom du poste client (connu avec la fonction [NetNomMachine](../WDLang3/3056014.md)) ou bien au nom du poste où se trouvent les fichiers de données (dans le cas d'une analyse distante).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- En général, la fonction **HPoste** s'utilise en réseau conjointement à un fichier de paramètre situé localement sur le poste et indiquant son numéro.

- La fonction **HPoste** n'a pas d'intérêt si les fichiers de données ne sont pas journalés et/ou ne sont pas en transaction.

- Si la fonction **HPoste** est utilisée après la fonction [HTransactionDébut](../WDLang4/3044002.md), le numéro de poste sera modifié pour les prochaines opérations effectuées.

- Cette fonction n'est pas disponible lors de l'utilisation d'une base de données via  .




## &lt;Source&gt;.JournalRecrée (Fonction)

***En anglais : &lt;Source&gt;.LogRecreate***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de re-créer un journal à vide. Cette fonction permet par exemple une remise à 0 du journal après une sauvegarde ou une réplication par exemple. Le contenu des fichiers journaux existants est perdu.

**Attention** : cette fonction est une fonction avancée.

**L'utilisation de cette fonction est déconseillée. Il est préférable d'utiliser [WDJournal](../WDJournal/3516012.md) pour gérer le journal.**

**Remarque** : La journalisation n'est pas disponible lors de l'accès à une base de données via .


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Client.JournalRecrée()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.JournalRecrée([<Options de création>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données journalé pour lequel le journal doit être re-créé. 

**`<Options de création> : (Constante ou combinaison de constante optionnelle)`**

Paramètre le fichier journal à re-créer :


|   |   |
| --- | --- |
| *hJournalFichierJNL*<br>(Valeur par défaut) | Seul le fichier &lt;Fichier de données&gt;JNL.FIC est recréé. |
| *hJournalIdentification* | Recrée le fichier "JournalIdentification.fic". |
| *hJournalOpération* | Recrée le fichier "JournalOpération.fic". |
| *hJournalUtilisateur* | Recrée le journal des utilisateurs "JournalUtilisateur.fic".<br>![HFSQL Client/Serveur](https://doc.pcsoft.fr/ext/images/fr/HFCS.png) Ce fichier existe uniquement en HFSQL Client/Serveur. Ce fichier contient les informations (login, machine, ...) sur les utilisateurs effectuant des actions journalées. |


**Remarque** : il est déconseillé de recréer uniquement les fichiers utilisateur et identification sans les autres fichiers. En effet, certains liens entre les fichiers pourraient être rompus et les fichiers recréés pourraient être incohérents.

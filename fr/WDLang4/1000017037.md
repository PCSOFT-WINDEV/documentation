
## HListeTriggerServeur (Fonction)

***En anglais : HListServerTrigger***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Liste les différents triggers disponibles sur une connexion ou sur un des fichiers de données de la connexion.  

**Remarque** : Les fonctions de manipulation des triggers serveur sont des fonctions avancées. Les triggers serveur peuvent être créés directement dans l'éditeur d'analyses. Pour plus de détails, consultez [Triggers serveur](../WDLang4/3044369.md).




<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HListeTriggerServeur(<Connexion> [, <Fichiers physiques HFSQL Client/Serveur>])`
---

**`<Résultat> : (Chaîne de caractères)`**

Liste des triggers serveurs appliqués sur la connexion spécifiée. Les différents triggers sont séparés par des retours chariot (RC).
La liste est de la forme :

```txt
<Trigger> + TAB + <Fichier de données> + TAB + <Procédure WLangage> + TAB + 
<Numéro de la fonction> + TAB + <Type de trigger> + TAB + <Etat du trigger> + RC + ...
```



|   |   |
| --- | --- |
| &lt;Trigger&gt; | Nom du trigger. |
| &lt;Fichier de données&gt; | Nom physique du fichier de données sur lequel s'applique le trigger. |
| &lt;Procédure WLangage&gt; | Nom de la procédure stockée WLangage appelée par le trigger. |
| &lt;Numéro de la fonction&gt; | Identifiant de la fonction WLangage sur laquelle est activée le trigger serveur. <br>Pour retrouver le nom de la fonction WLangage utilisée, il suffit de consulter le fichier ListeDéfinitionHF.WL (présent dans le sous-répertoire "Personnel\\Externe\\" de WINDEV, WINDEV Mobile ou WEBDEV). |
| &lt;Type de trigger&gt; | <br><br>- "A" pour un trigger "AVANT" (la procédure associée au trigger est exécutée **avant** la fonction HFSQL du trigger).<br><br>- "P" pour un trigger "APRES" (la procédure associée au trigger est exécutée **après** la fonction HFSQL du trigger).<br><br><br> |
| &lt;Etat du trigger&gt; | <br><br>- "1" pour un trigger actif (fonction [HActiveTriggerServeur](../WDLang4/1000017034.md)),<br><br>- "0" pour un trigger inactif (fonction [HDésactiveTriggerServeur](../WDLang4/1000017033.md)).<br><br><br> |



**`<Connexion> : (Chaîne de caractères optionnelle ou variable de type Connexion)`**

Connexion à manipuler. Cette connexion correspond : 

- soit au nom de la connexion définie sous l'éditeur d'analyses ou grâce aux fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

- soit au nom d'une variable de type [Connexion](../WDLang4/1514073.md).




**`<Fichiers physiques HFSQL Client/Serveur> : (Chaîne de caractères optionnelle)`**

Nom physique de un ou plusieurs fichiers de données HFSQL Client/Serveur associés à la connexion (par exemple "CLIENT.FIC"). Pour indiquer plusieurs noms, séparez les noms des fichiers de données par une virgule (","). 
Si ce paramètre n'est pas précisé ou est égal à une chaîne vide (""), le trigger sera activé sur tous les fichiers de données de la base.

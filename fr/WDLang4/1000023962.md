


## &lt;Variable Connexion&gt;.OuvreConnexion (Fonction)

***En anglais : &lt;Connection variable&gt;.OpenConnection***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ouvre une connexion à une base de données spécifique. La connexion reste ouverte jusqu'à l'appel de la fonction [&lt;Variable Connexion&gt;.FermeConnexion](../WDLang4/1000023941.md). La fonction **&lt;Variable Connexion&gt;.OuvreConnexion** permet d'ouvrir une connexion définie avec une variable de type [Connexion](../WDLang4/1514073.md).
**Attention** : L'ordre d'utilisation des fonctions permettant de se connecter à une base de données HFSQL Client/Serveur est important. Consultez [Optimiser les temps de connexion aux bases HFSQL Client/Serveur](../WDLang4/9000176.md) pour plus de détails.

## Syntaxe

`<Résultat> = <Connexion>.OuvreConnexion()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème (par exemple, lors d'une connexion décrite par programmation, certains paramètres utilisés dans la fonction [HDécritConnexion](../WDLang4/3044205.md) peuvent être inexacts).




La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.

**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion à une base de données.

### Syntaxe 1 : Ouvrir une connexion prédéfinie : gestion automatique des connexions
<a name="syntaxe_1_ouvrir_une_connexion_predefinie_gestion_automatique_des_connexions_ELTPARAGRAPHE000286"></a>

- La connexion associée à un fichier de données est automatiquement ouverte lors de l'ouverture du fichier de données. La connexion est fermée lorsque le dernier fichier de données utilisant cette connexion est fermé.

- Si une connexion est ouverte avec la fonction **&lt;Variable Connexion&gt;.OuvreConnexion**, cette connexion reste ouverte même si aucun fichier de données ne l'utilise.

- Si l'application effectue de nombreuses ouvertures et fermetures de fichiers de données successives, la gestion automatique des connexions va provoquer de nombreuses ouvertures/fermetures des connexions, entraînant d'importants ralentissements. Il est dans ce cas conseillé d'utiliser la fonction **&lt;Variable Connexion&gt;.OuvreConnexion** pour forcer la conservation de la connexion ouverte.

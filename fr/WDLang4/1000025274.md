
## HTransactionEnCours (Fonction)

***En anglais : HTransactionInProgress***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de savoir si une transaction est en cours.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Recherche si une transaction est en cours
// Base de données ISAM (HFSQL Classic)
bConnexionEnCours est un booléen = HTransactionEnCours()
```



```wl
// Recherche si une transaction avec MaConnexion est en cours
// Base de données Client/Serveur (HFSQL Client/Serveur)
MaConnexion est une Connexion
bConnexionEnCours est un booléen = HTransactionEnCours(MaConnexion)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = HTransactionEnCours([<Connexion>])`
---

**`<Résultat> : (Booléen)`**

**Si une connexion est spécifiée** : 

- <u><u><u><u>Vrai</u></u></u></u> s'il existe une transaction sur la connexion spécifiée ou s'il existe une transaction globale (pour HFSQL Classic (ISAM) et toutes les connexions HFSQL Client/Serveur).

- <u><u><u><u>Faux</u></u></u></u> s'il n'existe pas de transaction en cours sur la connexion spécifiée. 


**Si aucune connexion n'est spécifiée** : 

- <u><u><u><u>Vrai</u></u></u></u> s'il existe une transaction globale (pour HFSQL Classic (ISAM) et toutes les connexions HFSQL Client/Serveur).

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Attention : il peut cependant exister des transactions en cours sur des connexions spécifiques.  




**`<Connexion> : (Chaîne de caractères optionnelle ou variable optionnelle de type Connexion)`**

Connexion à utiliser. Cette connexion correspond :

- soit au nom de la connexion définie sous l'éditeur d'analyses ou grâce aux fonctions [HDécritConnexion](../WDLang4/3044205.md) ou [HOuvreConnexion](../WDLang4/3044107.md).

- soit au nom d'une variable de type [Connexion](../WDLang4/1514073.md).

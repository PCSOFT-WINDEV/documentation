


## &lt;redisConnexion&gt;.PipelineRAZ (Fonction)

***En anglais : &lt;redisConnection&gt;.PipelineReset***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ignore les résultats de l'exécution d'une commande Redis ayant été ajoutée par la fonction [&lt;redisConnexion&gt;.PipelineAjoute](../WDLang4/1410088283.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
gRedis.PipelineAjoute("get p2")
gRedis.PipelineAjoute("get p1")
gRedis.PipelineAjoute("set p3 v3")
gRedis.PipelineAjoute("set p4 v4")

// Récupération des 2 premiers résultats
r1 est un redisRéponse = gRedis.PipelineRécupère()
r2 est un redisRéponse = gRedis.PipelineRécupère()
// Ignore les résultats suivants
gRedis.PipelineRAZ()
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = <Connexion>.PipelineRAZ()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été effectuée,  

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Connexion> : (Variable de type redisConnexion)`**

Nom de la variable de type [redisConnexion](../WDLang4/1000023527.md) correspondant à la connexion au serveur Redis.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Fonctionnement du pipeline
<a name="fonctionnement_pipeline_ELTPARAGRAPHE000182"></a>Lorsque l'ensemble des commandes souhaitées auront été ajoutées dans le pipeline, les résultats pourront être récupérés avec la fonction [&lt;redisConnexion&gt;.PipelineRécupère](../WDLang4/1410088285.md).

Pour ne pas récupérer les résultats, il est possible d'utiliser la fonction **&lt;redisConnexion&gt;.PipelineRAZ**.

Si la fonction **&lt;redisConnexion&gt;.PipelineRAZ** n'a pas été utilisée, il doit y avoir autant d'appels à la fonction [&lt;redisConnexion&gt;.PipelineRécupère](../WDLang4/1410088285.md) que d'appels à la fonction [&lt;redisConnexion&gt;.PipelineAjoute](../WDLang4/1410088283.md).

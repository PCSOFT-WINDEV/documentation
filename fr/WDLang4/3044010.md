
## HInitHF55 (Fonction)

***En anglais : HInitHF55***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet d'accéder aux fonctions spécifiques à la gestion des données xBase (ou dBase) si l'Accès Natif xBase livré avec WINDEV ou WEBDEV n'est pas disponible.

Remarques : 

- Il est conseillé d'utiliser l'Accès Natif xBase livré avec WINDEV.

- Cette fonction n'est pas disponible lors de l'utilisation d'un exécutable autonome. 




**Pour accéder à un fichier xBase sans utiliser l'Accès Natif xBase standard :**

- **Application WINDEV ou WEBDEV sans fichier de données de type Hyper File 5.5 dans l'analyse :**
	Il est nécessaire d'utiliser une analyse créée avec WINDEV 5.5. 
	Vous trouverez une analyse au format WINDEV 5.5 dans le sous-répertoire "Personnel\\Modèles" du répertoire d'installation de votre produit (fichier **ForxBase.WDD**). Cette analyse doit être copiée dans le répertoire de votre projet et associée au projet avec la fonction **HInitHF55**.

- **Application WINDEV ou WEBDEV avec un ou des fichiers de données de type Hyper File 5.5 dans l'analyse :**
	aucune configuration particulière n'est à effectuer.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Application WinDev sans fichiers au format WinDev 5.5
HInitHF55("C:\Fic\ForxBase.WDD")
HDBDécritFichier("CLIENTDB","CD","C :\FIC\CLIENT.DBF")
HDBDécritRubrique("NOM,C,20")
HDBDécritRubrique("PRENOM,C,20")
HDBDécritRubrique("AGE,N,3,0")
HDBDécritRubrique("SOLDE,N,10,2")
HDBDécritRubrique("MARIE,L")
HDBDécritRubrique("DATNAISS,D")
HDBDécritRubrique("INFOS,M")
HDBCréation()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HInitHF55(<Chemin complet du fichier WDD> [, <Mot de passe du fichier WDD>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si aucun problème n'a été rencontré, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. La fonction [HErreur](../WDLang4/3044088.md) permet d'obtenir plus d'informations sur le problème rencontré.




**`<Chemin complet du fichier WDD> : (Chaîne de caractères)`**

Chemin d'accès au fichier .WDD de l'analyse au format Hyper File 5.5. Tous les formats d'analyse reconnus par Hyper File 5.5 peuvent être utilisés (WINDEV 5.5B, 5.5, 5.0 et 4.1). Ce chemin est pris en compte uniquement si aucune analyse n'est ouverte ou si l'analyse ouverte ne contient pas de fichiers de données au format 5.5 d'origine (onglet "Détail" de la description de l'analyse).

**`<Mot de passe du fichier WDD> : (Chaîne de caractères optionnelle)`**



- Mot de passe associé à l'analyse. 

- Chaîne vide ("") si aucun mot de passe n'a été défini.

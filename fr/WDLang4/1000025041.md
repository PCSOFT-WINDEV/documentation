


## &lt;Source&gt;.NumEnr (Fonction)

***En anglais : &lt;Source&gt;.RecNum***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie :

- le numéro de l'enregistrement en cours dans le fichier de données HFSQL.

- le numéro de l'enregistrement en cours dans la vue HFSQL.





<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Enregistrement en cours du fichier de données "Client"
EnrEnCoursClient = Client.NumEnr()
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.NumEnr()`
---

**`<Résultat> : (Entier)`**

Correspond :

- soit au numéro de l'enregistrement en cours dans le fichier de données.

- soit au numéro de l'enregistrement en cours dans la vue ou dans la requête (et non le numéro de l'enregistrement dans le fichier de données). 
	**Attention** : Utilisée sur une vue HFSQL Classic, ce paramètre ne correspond pas à la valeur de la variable [H.NumEnr](../WDLang4/3044076.md).

- soit à la constante *hNumEnrInvalide* si aucun enregistrement n'est en cours (position en dehors du fichier de données).




La fonction [HErreur](../WDLang4/3044088.md) permet d'obtenir plus d'informations sur le problème rencontré.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/fr/HF55.png) Utilisée sur une vue Hyper File 5.5, la fonction **&lt;Source&gt;.NumEnr** renvoie le numéro d'enregistrement en cours dans le fichier de données associé à la vue.

**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données, de la requête ou de la vue HFSQL manipulée. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Fichiers de données non HFSQL
<a name="fichiers_donnees_non_hfsql_ELTPARAGRAPHE000236"></a>

- Il n'y a pas de numéro d'enregistrement sur les fichiers de données.

- L'appel de cette fonction provoque une erreur WLangage.

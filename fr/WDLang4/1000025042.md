


## &lt;Source&gt;.Optimise (Fonction)

***En anglais : &lt;Source&gt;.Optimize***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet d'utiliser des temps morts dans l'utilisation d'un logiciel (période sans traitements) pour optimiser les requêtes et les parcours qui vont être exécutés plus tard. Optimise l'accès aux index des fichiers de données HFSQL Classic (les index des fichiers de données HFSQL Client/Serveur ne peuvent pas être optimisés). 

<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
Client.Optimise(Nom)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Source>.Optimise([<Rubrique clé>])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier a été optimisé, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'obtenir plus de renseignements.




**`<Source> : (Type correspondant à la source spécifiée)`**

Nom du fichier de données HFSQL Classic manipulé.

**`<Rubrique clé> : (Chaîne de caractères optionnelle)`**

Nom de la rubrique clé à optimiser. Si ce paramètre n'est pas précisé, la fonction **&lt;Source&gt;.Optimise** optimise toutes les clés du fichier de données.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Différence entre HOptimise et HOptimiseRequête
<a name="difference_entre_hoptimise_hoptimiserequete_ELTPARAGRAPHE000203"></a>

- La fonction **&lt;Source&gt;.Optimise** charge les caches du système avec tout l'index du fichier de données. Si le fichier de données a une taille importante ou si l'application utilise plusieurs fichiers de données de taille importante, il y a risque de saturation des caches du système. 
	Pour éviter les risques de saturation, évitez d'appeler la fonction **&lt;Source&gt;.Optimise** sur un trop grand nombre de fichiers de données ou de rubriques clé.

- La fonction [HOptimiseRequête](../WDLang4/3044003.md) charge les caches du système avec uniquement les parties d'index nécessaires à la requête spécifiée, pour tous les fichiers de données utilisés par la requête. Cette fonction permet d'optimiser les requêtes.



<a name="NOTE0_2"></a>


### Divers
<a name="divers_ELTPARAGRAPHE000225"></a>

- La fonction **&lt;Source&gt;.Optimise** rend la main immédiatement au programme et est exécutée en tâche de fond.

- La fonction **&lt;Source&gt;.Optimise** n'a aucun effet sur les rubriques non clé ou sur les fichiers de données n'étant pas au format HFSQL Classic.

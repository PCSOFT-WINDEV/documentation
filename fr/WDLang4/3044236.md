
## HChangeRepRpl (Fonction)

***En anglais : HChangeRplDir***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie l'emplacement de la description du réplica abonné (fichier RPL). Cette fonction doit être utilisée sur le poste abonné.

**Cette fonction est une fonction avancée de la réplication journalée**. Elle doit être utilisée par exemple lors de l'utilisation de bases de données non connectées par le réseau. Dans ce cas, le réplica maître et le réplica abonné sont créés sur le même poste (poste maître). Le fichier RPL (fichier du réplica abonné) est ensuite copié sur le poste abonné.

Ce fichier peut être installé :

- soit dans le répertoire de réplication indiqué dans l'analyse (sous-répertoire \\RPL des fichiers de données par défaut).

- soit dans un autre répertoire. Dans ce cas, l'application abonnée doit utiliser la fonction **HChangeRepRpl** pour indiquer où le fichier du réplica abonné doit être recherché. En effet, à l'ouverture du fichier de données abonné, le fichier RPL est lu pour définir la plage des identifiants automatiques utilisés par le fichier de donnés.




Par défaut, les fichiers de réplication sont gérés dans le répertoire spécifié sous l'analyse (option "Description de l'analyse", onglet "Journalisation"). ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=HChangeRepJNL%20-%20HC%20N%B0002.gif)



<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Le fichier RPL se trouve dans le même répertoire que les fichiers de données
HChangeRepRpl(".")

// Le RPL est dans le sous-répertoire des fichiers de données \RPL
HChangeRepRpl(".\RPL") 

// Le RPL est dans le répertoire "X:\Data\RPL"
HChangeRepRpl("X:\Data\RPL")
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HChangeRepRpl(<Emplacement du fichier réplica abonné>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été réalisée, 

- <u><u><u><u>Faux</u></u></u></u> en cas de problème. La fonction [HErreurInfo](../WDLang4/3044071.md) permet d'identifier l'erreur.




**`<Emplacement du fichier réplica abonné> : (Chaîne de caractères)`**

Nouveau chemin d'accès au fichier \*.RPL correspondant au réplica abonné (chemin complet ou relatif). Ce répertoire est utilisé :

- à l'ouverture d'un fichier de données HFSQL en réplication sur le poste abonné. La plage des identifiants automatiques est mise à jour.

- lors de la synchronisation (par exemple [HSynchroniseRéplica](../WDLang4/3044014.md)), si aucun répertoire n'est précisé. 
	**Remarque** : si le répertoire est relatif, le répertoire sera relatif au répertoire par défaut des fichiers de données défini dans l'analyse.

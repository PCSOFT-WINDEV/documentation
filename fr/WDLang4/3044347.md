


## HVersFichier (Fonction)

***En anglais : HToFile***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Copie une source de données (fichier de données, requête, vue, ...) vers un fichier de données physique HFSQL de même description. Ce fichier de données n'est ni crypté, ni protégé par mot de passe.  

Cette fonction permet par exemple de sauvegarder un historique : une requête sélectionne tous les enregistrements de l'année 2019, le résultat est transféré dans un fichier de données, et une nouvelle requête supprime tous les enregistrements de l'année 2019.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
SI HVersFichier(MaVue, "C:\temp\Test.fic") = Vrai ALORS
	Info("Données transférées dans un fichier de données")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HVersFichier(<Source de données> , <Fichier destination>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été effectuée, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire (par exemple le fichier destination existe déjà).




**`<Source de données> : (Chaîne de caractères)`**

Nom logique de la source de données à copier (fichier de données, vue, requête, ...).

**`<Fichier destination> : (Chaîne de caractères)`**

Nom complet du fichier de données physique dans lequel les enregistrements de la source doivent être copiés. Ce fichier de données aura la même description que la source de données. Pour accéder à ce fichier de données, utilisez par exemple la fonction [HDéclareExterne](../WDLang4/3044204.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Source filtrée
<a name="source_filtree_ELTPARAGRAPHE000095"></a>Si un filtre est défini et actif sur la source de données, seuls les enregistrements correspondant au filtre seront copiés.
<a name="NOTE0_2"></a>


### Source de données : Vue ou requête
<a name="source_donnees_vue_requete_ELTPARAGRAPHE000104"></a>Si la source de données est une vue ou une requête, toutes les rubriques seront des clés du fichier de données créé à l'exception :

- des rubriques de type Mémo.

- des rubriques dont la taille est supérieure à 260 caractères et non clé dans le fichier source de la vue ou de la requête.

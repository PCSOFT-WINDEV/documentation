
## HGèreCache (Fonction)

***En anglais : HSetCache***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de paramétrer la gestion des caches dans le moteur HFSQL Client/Serveur.






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
HGèreCache(hTaillePage, 2000)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HGèreCache(<Paramètre à modifier> [, <Fichier de données> [, <Rubrique>]] , <Valeur>)`
---

**`<Résultat> : (Entier)`**

Ancienne valeur du paramètre modifié (valeur avant modification). 

**`<Paramètre à modifier> : (Constante)`**

Paramètre du cache à modifier :


|   |   |
| --- | --- |
| *hTaillePage* | Taille d'un bloc de caches. Ce bloc de caches sera utilisé lors du parcours d'un fichier de données. <br>Plus le bloc de caches est important, plus le parcours sera performant, et plus les transferts réseau et la mémoire consommée seront importants.<br>Ce paramètre correspond au nombre d'enregistrements lus lors du parcours. |



**`<Fichier de données> : (Chaîne de caractères optionnelle)`**

Nom du fichier de données auquel la modification doit être appliquée. 
Si ce paramètre n'est pas spécifié, la modification est appliquée pour tous les fichiers de données.

**`<Rubrique> : (Chaîne de caractères optionnelle)`**

Nom de la rubrique à laquelle la modification doit être appliquée. 
Si ce paramètre n'est pas spécifié, la modification sera appliquée à toutes les rubriques du fichier de données spécifié.

**`<Valeur> : (Entier)`**

Nouvelle valeur à donner au paramètre.

Dans le cas de la constante *hTaillePage*, cette valeur doit être comprise entre 1 et 5000 (la valeur par défaut sur un PC est de 1000 et sur un mobile est de 10).

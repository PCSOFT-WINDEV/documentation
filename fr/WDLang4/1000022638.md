


## &lt;Variable Connexion&gt;.AjouteGroupe (Fonction)

***En anglais : &lt;Connection variable&gt;.AddGroup***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ajoute un groupe d'utilisateurs. Toutes les caractéristiques du groupe doivent être saisies dans [les variables de gestion des groupes](../WDLang4/3044315.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion
MaConnexion est une Connexion

// Description de la connexion
MaConnexion.Utilisateur = "USER"
MaConnexion.MotDePasse = "PASSWORD"
MaConnexion.Serveur = "MONSERVEUR" 
MaConnexion.BaseDeDonnées = "Base de données"
MaConnexion.Provider = hAccèsHFClientServeur
MaConnexion.Accès = hOLecture


// Ouverture de la connexion
MaConnexion.OuvreConnexion()
// Association du fichier de données à la connexion
Client.ChangeConnexion(MaConnexion)
// Caractéristiques du nouveau groupe
HGroupe.Nom = "ADMIN2"
HGroupe.Description = "Administrateurs de la base de données"
HGroupe.Utilisateurs = "Flo" + RC + "Julie"
// Ajout du groupe
SI MaConnexion.AjouteGroupe() = Vrai ALORS
	Info("Groupe ajouté")
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Connexion>.AjouteGroupe()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le groupe a été ajouté, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur. La fonction [HErreur](../WDLang4/3044088.md) permet d'identifier l'erreur.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion pour laquelle le groupe d'utilisateurs doit être ajouté.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Droits nécessaires pour ajouter un groupe d'utilisateurs
<a name="droits_necessaires_pour_ajouter_groupe_utilisateurs_ELTPARAGRAPHE000239"></a>Pour ajouter un groupe d'utilisateurs, il est nécessaire de posséder les droits suivants :

- Droit de gestion des utilisateurs (constante *hDroitGestionUtilisateur*).

- Droit de visualisation des utilisateurs (constante *hDroitVisualisationUtilisateur*).




Ces constantes sont utilisées dans les fonctions [&lt;Variable Connexion&gt;.InfoDroitServeur](../WDLang4/1000022668.md) et [&lt;Variable Connexion&gt;.ModifieDroitServeur](../WDLang4/1000022710.md).
<a name="NOTE0_2"></a>


### Gestion des droits
<a name="gestion_des_droits_ELTPARAGRAPHE000264"></a>A sa création, un groupe n'a aucun droit.

Il est possible de donner des droits à un groupe grâce aux fonctions :

- [&lt;Variable Connexion&gt;.ModifieDroitBaseDeDonnées](../WDLang4/1000022707.md) : permet de donner des droits sur une base de données.

- [&lt;Variable Connexion&gt;.ModifieDroitFichier](../WDLang4/1000022709.md) : permet de donner des droits sur un fichier de données.

- [&lt;Variable Connexion&gt;.ModifieDroitServeur](../WDLang4/1000022710.md) : permet de donner des droits sur un serveur.


Pour plus de détails sur la gestion des droits, consultez [Gestion des droits en HFSQL Client/Serveur](../WDLang4/3044333.md).




## &lt;Source&gt;.ActiveFiltreAuto (Fonction)

***En anglais : &lt;Source&gt;.ActivateAutoFilter***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Active un filtre automatique sur les fichiers de données reliés lors du parcours d'un fichier XML. Ce filtre peut être désactivé grâce à la fonction [&lt;Source&gt;.DésactiveFiltreAuto](../WDLang4/1000024219.md).

**Attention** : Cette fonction est disponible uniquement pour le Connecteur Natif XML.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Activation de tous les filtres auto possibles du fichier de données Client
Client.ActiveFiltreAuto("*" )
// sauf entre les fichiers de données Client et Commande
Client.DésactiveFiltreAuto(Commande)

/////////////////////////////

// Parcours des commandes
Commande.LitPremier(IDCommande)
TANTQUE PAS Commande.EnDehors()
	// Parcours des lignes de commande de la commande en cours
	LigneCde.LitPremier(IDCommande)
	TANTQUE PAS LigneCde.EnDehors()
		LigneCde.LitSuivant(IDCommande)
	FIN
	// Commande suivante
	Commande.LitSuivant(IDCommande)
FIN
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Fichier de données de parcours>.ActiveFiltreAuto(<Fichier de données à filtrer>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le filtre est activé, 

- <u><u><u><u>Faux</u></u></u></u> en cas d'erreur (fichier ou rubrique inexistant).




**`<Fichier de données de parcours> : (Nom de fichier de données)`**

Nom du fichier de données utilisé pour le parcours.





**`<Fichier de données à filtrer> : (Nom de fichier de données)`**

Nom du fichier de données à filtrer lors d'un positionnement dans le fichier de données de parcours.

Ce paramètre peut correspondre à "\*". Dans ce cas, tous les filtres automatiques possibles seront activés.





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques
Voici un exemple simple :

```wl
Père.ActiveFiltreAuto(Fils)
```

Ce code indique que quand les enregistrements du fichier de données Fils sont parcourus, ce parcours se fait par rapport au 'contexte' (au noeud) courant du fichier de données Père : tous les 'Fils' du 'Père' courant seront parcourus.


```wl
Père.DésactiveFiltreAuto(Fils)
```

Ce code indique que le filtrage précédent n'est pas actif : le parcours du fichier de données Fils sera indépendant de celui du fichier de données Père. Cela revient donc à parcourir tous les enregistrements du fichier de données Fils dans le fils donné.

La fonction **&lt;Source&gt;.ActiveFiltreAuto** permet d'activer/désactiver le lien hiérarchique entre les fichiers de données Père et Fils pendant des parcours.

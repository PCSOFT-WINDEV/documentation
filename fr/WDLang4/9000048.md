## Spécificités de cette version
<a name="specificites_cette_version_ELTTEXTE000128"></a>
Dans cette version, les spécificités de la réplication universelle assistée sont les suivantes :

- **Gestion de la modification automatique des données.**
	

	- Réplication entre des bases de données de type HFSQL : La gestion de la modification automatique des données est disponible uniquement lors d'une réplication universelle concernant des bases de données HFSQL. 
			Pour gérer la modification automatique des données lors de la réplication, vous devez le préciser dans l'outil [ReplicEdit](../ReplicEdit/3526501.md) : dans la [description de la réplication](../ReplicEdit/3526502.md), dans l'onglet "Avancé", option **""**. 
			**Attention** : 

		- Pour des réplications existantes, il est nécessaire de recréer une réplication (et notamment les abonnés) pour mettre en place ce mécanisme.

		- Ce mécanisme ne fonctionne pas en cas d'ajout ou de suppression d'une clé unique. Dans ce cas, la modification automatique doit être effectuée de la même manière que pour des bases de données de formats différents.   




- Réplication entre des bases de données de formats différents :  En cas de modification automatique des données, il n'y a aucun mécanisme automatique dans cette version. Il est nécessaire de :

	- faire le maximum de synchronisation d'abonnés avant la modification automatique des données.

	- faire la modification automatique des données sur les fichiers maîtres.

	- faire la modification automatique des données sur les fichiers abonnés.

	- synchroniser les abonnés.

- **Gestion de la réplication pour les fichiers du groupware utilisateur.**
	La réplication universelle assistée ne gère pas automatiquement les fichiers du groupware utilisateur. Il est nécessaire de gérer "à la main" une copie des fichiers du groupware utilisateur sur le poste local.

- **Réplication directe**
	Il est obligatoire de déployer le fichier .WER à côté du fichier exécutable (même dans le cas d'une réplication programmée). Il n'est pas possible d'intégrer le fichier .WER dans une librairie WINDEV (fichier .WDL).

- **Réplication universelle assistée et WINDEV Mobile**
	ReplicSynchro version Mobile n'est pas livré dans cette version. Il est nécessaire de réaliser une réplication universelle assistée en utilisant les ordres de programmation WLangage. Pour plus de détails, consultez [Réplication Universelle Assistée par programmation](../WDLang4/9000047.md).

- **Réplication universelle assistée et WEBDEV**
	Avec WEBDEV, il est nécessaire de réaliser une réplication universelle assistée en utilisant les ordres de programmation WLangage. Pour plus de détails, consultez [Réplication Universelle Assistée par programmation](../WDLang4/9000047.md).

- **Paramètres d'accès aux fichiers maîtres**
	L'outil **ReplicAdmin** ne permet pas de modifier les paramètres d'accès aux fichiers maîtres : il est nécessaire de réaliser une installation avec l'assistant de création d'installation et de faire une mise à jour de l'application.
	Il n'est donc pas possible de déployer une réplication directement sur un poste sans utiliser une application.

- **Volume de données**
	**Attention** : la réplication universelle peut rencontrer des difficultés sur de grands volumes de données. Le principal facteur limitant est la taille de la clé utilisée pour identifier les enregistrements. Par exemple, une clé de type entier (ou identifiant automatique) permettra de répliquer plus d'enregistrements qu'une clé de type chaîne.
	De plus, il est conseillé de ne répliquer qu'un sous ensemble des enregistrements en utilisant les fonctionnalités de filtrage de la réplication assistée. En effet, c'est le volume des données répliquées et non le volume total des données qui est à prendre en compte.




## &lt;Variable Connexion&gt;.ExécuteProcédure (Fonction)

***En anglais : &lt;Connection variable&gt;.ExecuteProcedure***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Exécute une procédure stockée.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Description de la connexion
ConnexionHF est une Connexion

// Exécution de la procédure "VenteParAnnée", avec comme paramètre "2019"
ConnexionHF.ExécuteProcédure("VenteParAnnée", 2019)
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = <Connexion>.ExécuteProcédure(<Procédure stockée> [, <Paramètre 1> [... [, <Paramètre N>]]])`
---

**`<Résultat> : (Type correspondant au résultat)`**



- Résultat de la procédure stockée : enregistrement, tableau d'enregistrements, tableau de types simples, ...

- Chaîne vide ("") si la procédure &lt;Procédure stockée&gt; ne retourne pas de résultat.




**`<Connexion> : (Variable de type Connexion)`**

Nom de la variable de type [Connexion](../WDLang4/1514073.md) décrivant la connexion vers le serveur qui exécute la procédure stockée.

**`<Procédure stockée> : (Procédure WLangage)`**

Nom de la procédure stockée à exécuter.

**`<Paramètre 1> : (Variant optionnel)`**

Premier paramètre à passer à la procédure &lt;Procédure stockée&gt;.



**`<Paramètre N> : (Variant optionnel)`**

Nième paramètre à passer à la procédure &lt;Procédure stockée&gt;.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Gestion automatique des erreurs
<a name="gestion_automatique_des_erreurs_ELTPARAGRAPHE000251"></a>La gestion automatique des erreurs est désactivée dans les procédures stockées. Il est cependant possible d'utiliser la fonction [&lt;Source&gt;.SurErreur](../WDLang4/1000025073.md) pour exécuter une procédure spécifique (procédure stockée).
<a name="NOTE0_2"></a>


### Passage de paramètres
<a name="passage_parametres_ELTPARAGRAPHE000266"></a>Les paramètres passés à la procédure stockée sont passés "par référence" : si ces paramètres sont modifiés dans la procédure stockée, ils seront également modifiés dans l'application qui a exécuté cette procédure stockée.
<a name="NOTE0_3"></a>


### Exécution d'une procédure stockée depuis un code SQL
<a name="execution_une_procedure_stockee_depuis_code_sql_ELTPARAGRAPHE000276"></a>Une procédure stockée peut être appelée : 

- depuis du code SQL, par exemple avec les fonctions [&lt;Source&gt;.ExécuteRequêteSQL](../WDLang4/1000024948.md) ou [SQLExec](../WDLang4/3072007.md), 

- depuis une application tierce utilisant le [driver ODBC pour HFSQL](../WDLang4/9000160.md) ou le [provider OLE DB pour HFSQL](../WDLang4/9000059.md). 




Pour plus de détails, consultez [Procédures et requêtes stockées (HFSQL Client/Serveur)](../WDLang4/3044360.md).

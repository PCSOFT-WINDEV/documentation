
## mongoChercheOption (Type de variable)

***En anglais : mongoFindOption***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **mongoChercheOption** permet de définir toutes les caractéristiques avancées d'une recherche dans une base MongoDB. Les caractéristiques de cette recherche peuvent être définies et modifiées à l'aide de différentes propriétés WLangage. 

**Remarque** : Pour plus de détails sur la déclaration de ce type de variable et l'utilisation des propriétés WLangage, consultez [Déclaration d'une variable](../Motscles/1514032.md).


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MesOptions est un mongoChercheOption
MesOptions.Limite = 5

soit MaRecherche = MongoCherche(CnxMdb.BaseDeDonnées["CRM"].Collection["produits"], "{}", MesOptions)
```
<a name="Exemple2"></a>

```wl
Option est un mongoChercheOption
// Tri selon l'âge	
Option.Trie = "{ ""age"" : -1 }"

sFiltre est une chaîne = "{}"	
soit MonRésultat = MongoCherche(CnxMdb.BaseDeDonnées["CRM"].Collection["Acteurs"], sFiltre, Option)
```

<a name="NOTE0"></a>

## Remarques
<a name="NOTE0_1"></a>


### Propriétés spécifiques à la description des variables de type mongoChercheOption
<a name="proprietes_specifiques_description_des_variables_type_mongochercheoption_ELTPARAGRAPHE000041"></a>Les propriétés suivantes peuvent être utilisées pour manipuler les options de recherche :

| Nom de la propriété | Type manipulé | Effet |
| --- | --- | --- |
| Ignore | Entier | Nombre de documents ignorés. Le premier document renvoyé sera le document correspondant à cette valeur +1. |
| Limite | Entier | Nombre maximal de documents à renvoyer lors de la recherche. |
| Projection | Variant, Buffer ou Chaîne de caractères | Document au format JSON (Chaîne de caractères), BSON (buffer) ou variant définissant les rubriques à renvoyer. |
| Tri | Variant, Buffer ou Chaîne de caractères | Document au format JSON (Chaîne de caractères), BSON (buffer) ou variant définissant les paramètres de tri.<br><br>Pour obtenir un tri ascendant, le nom de la clé doit être suivi de ":1". <br><br>Pour obtenir un tri descendant, le nom de la clé doit être suivi de ":-1". |

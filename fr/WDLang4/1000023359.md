
## memcachedRésultat (Type de variable)

***En anglais : memcachedResult***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le type **memcachedRésultat** permet de définir toutes les caractéristiques d'une valeur lue sur un serveur Memcached. Les caractéristiques de cette valeur peuvent être définies et modifiées à l'aide de différentes propriétés WLangage. 

**Remarque** : Pour plus de détails sur la déclaration de ce type de variable et l'utilisation des propriétés WLangage, consultez [Déclaration d'une variable](../Motscles/1514032.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
R est un memcachedRésultat = MemcachedRécupère(memConnexion, "clé")
```





<a name="NOTE0"></a>

## Remarques
<a name="NOTE0_1"></a>


### Propriétés spécifiques à la description des variables de type memcachedRésultat
<a name="proprietes_specifiques_description_des_variables_type_memcachedresultat_ELTPARAGRAPHE000029"></a>Les propriétés suivantes peuvent être utilisées pour manipuler une variable de type **memcachedRésultat** :


| Nom de la propriété | Type manipulé | Effet |
| --- | --- | --- |
| CAS | Entier | Valeur de la version précédente lue.<br>**Cette propriété est disponible en lecture seulement**.<br><br>Attention : Le serveur Memcached doit supporter le CAS (propriété **SupporteCAS** de la variable de type [memcachedConnexion](../WDLang4/1000023339.md) à <u><u><u><u>Vrai</u></u></u></u>). Le CAS (Check And Set) permet de réaliser une opération de modification des données uniquement si cette donnée n'a pas été modifiée depuis sa dernière lecture. |
| Clé | Chaîne de caractères | Nom de la clé.<br>**Cette propriété est disponible en lecture seulement**. |
| Flag | Entier | Valeur supplémentaire associée au couple Clé/Valeur.<br>**Cette propriété est disponible en lecture seulement**. |
| Valeur | Buffer | Valeur de la clé.<br>**Cette propriété est disponible en lecture seulement**. |

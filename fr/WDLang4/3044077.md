
## HErreurDoublon (Fonction)

***En anglais : HErrorDuplicates***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Permet de vérifier si une erreur de doublons est survenue.

![WINDEV](https://doc.pcsoft.fr/ext/images/fr/WD.png) Cette fonction peut par exemple être utilisée lorsque la gestion automatique des erreurs est désactivée par la fonction [HSurErreur](../WDLang4/3044017.md). Par défaut, WINDEV propose une [gestion automatique des doublons](../WDLang4/3044188.md).






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple
<a class="notetitle" target="_blank" href="$DOC$=1000003044077&name=herreurdoublon_fonction&product=WD">Voir des exemples supplémentaires</a>

```wl
// Gestion d'une erreur de doublons
SI HModifie(Client) = Faux ALORS
	SI HErreurDoublon() = Vrai ALORS 
		sNomRub est une chaîne
		sNomRub = HErreurInfo(hErrRubrique)
		Erreur("Doublon : cet enregistrement ne peut être modifié", ...
					"Rubrique associée à l'erreur : " + sNomRub, ...
					"Valeur de la rubrique : " + {"Client." + sNomRub})
	FIN
FIN
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = HErreurDoublon()`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si une erreur de doublons est survenue, 

- <u><u><u><u>Faux</u></u></u></u> sinon (pas d'erreur).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Par défaut, la vérification des doublons est active. Pour activer ou désactiver la gestion des doublons, utilisez la fonction [HGèreDoublon](../WDLang4/3044057.md).

- Cette fonction remplace la fonction **HDoublon** présente en WINDEV 5.5.

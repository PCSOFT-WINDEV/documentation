
## SQLChangeConnexion (Fonction)

***En anglais : SQLChangeConnection***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Modifie la connexion en cours. La connexion en cours correspond à la dernière connexion ouverte ou à la connexion spécifiée avec la fonction **SQLChangeConnexion**.

**Important** : Les bases de données HFSQL ne permettent pas d'ouvrir plusieurs connexions simultanément. L'imbrication de connexions peut être utilisée uniquement pour les bases utilisant un accès par ODBC, OLE DB, ou par un Connecteur Natif. Le Connecteur Natif AS/400 permet d'imbriquer au maximum le nombre de connexions autorisé par la licence.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Les clients sont sur une base ORACLE
Conn1 est un entier = SQLConnecte("ORACLE", "", "")
// La comptabilité est sur une base SQLSERVER
Conn2 est un entier = SQLConnecte("SQLSERVER", "", "")
...
// Accéder aux clients
SQLChangeConnexion(Conn1)
...
// Accéder à la comptabilité
SQLChangeConnexion(Conn2)
...
// Fermeture de la connexion en cours
SQLDéconnecte()
// Fermeture de la connexion en cours
SQLDéconnecte()
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`<Résultat> = SQLChangeConnexion(<Numéro de la connexion>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le changement de connexion a été réalisé, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Numéro de la connexion> : (Entier)`**

Numéro de la connexion à utiliser, renvoyé par la fonction [SQLConnecte](../WDLang4/3072005.md).

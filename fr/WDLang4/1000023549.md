


## RedisPipelineRAZ (Fonction)

***En anglais : RedisPipelineReset***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ignore les résultats de l'exécution d'une commande Redis ayant été ajoutée par la fonction [RedisPipelineAjoute](../WDLang4/1000023542.md).
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
RedisPipelineAjoute(gRedis, "get p2")
RedisPipelineAjoute(gRedis, "get p1")
RedisPipelineAjoute(gRedis, "set p3 v3")
RedisPipelineAjoute(gRedis, "set p4 v4")

// Récupération des 2 premiers résultats
r1 est un redisRéponse = RedisPipelineRécupère(gRedis)
r2 est un redisRéponse = RedisPipelineRécupère(gRedis)
// Ignore les résultats suivants
RedisPipelineRAZ(gRedis)
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = RedisPipelineRAZ(<Connexion>)`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si l'opération a été effectuée,  

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire.




**`<Connexion> : (Variable de type redisConnexion)`**

Nom de la variable de type [redisConnexion](../WDLang4/1000023527.md) correspondant à la connexion au serveur Redis.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Fonctionnement du pipeline
<a name="fonctionnement_pipeline_ELTPARAGRAPHE000061"></a>Lorsque l'ensemble des commandes souhaitées auront été ajoutées dans le pipeline, les résultats pourront être récupérés avec la fonction [RedisPipelineRécupère](../WDLang4/1000023543.md).

Pour ne pas récupérer les résultats, il est possible d'utiliser la fonction **RedisPipelineRAZ**.

Si la fonction **RedisPipelineRAZ** n'a pas été utilisée, il doit y avoir autant d'appels à la fonction [RedisPipelineRécupère](../WDLang4/1000023543.md) que d'appels à la fonction [RedisPipelineAjoute](../WDLang4/1000023542.md).

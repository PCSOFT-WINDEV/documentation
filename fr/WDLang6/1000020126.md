


## PDFFusionneAvecMotDePasse (Fonction)

***En anglais : PDFMergeWithPassword***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Fusionne plusieurs fichiers PDF existant en un seul fichier PDF. Les mots de passe des différents fichiers PDF sont gérés. Les fichiers sont fusionnés les uns après les autres.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
PDFFusionneAvecMotDePasse("C:\Doc pdf\pdf_fusion.pdf", ...
	"C:\Doc pdf\EtatRequete.pdf", "mdp", "C:\Doc PDF\Concepts.pdf", "mdp2")
```

<a name="XSYNTAXE"></a>

## Syntaxe
<a name="SYNTAXE1"></a>

`<Résultat> = PDFFusionneAvecMotDePasse(<Fichier PDF Résultat> , <Fichier PDF 1> , <Mot de passe PDF 1> , <Fichier PDF 2> , <Mot de passe PDF 2> [, <Fichier PDF 3> , <Mot de passe PDF 3> [... [, <Fichier PDF N> , <Mot de passe PDF N>]]])`
---

**`<Résultat> : (Booléen)`**



- <u><u><u><u>Vrai</u></u></u></u> si le fichier résultat a été généré, 

- <u><u><u><u>Faux</u></u></u></u> dans le cas contraire. Pour plus de détails sur l'erreur, utilisez la fonction [ErreurInfo](../WDLang1/3013008.md). 




**`<Fichier PDF Résultat> : (Chaîne de caractères)`**

Nom et chemin complet du fichier PDF à créer. Ce fichier contiendra la fusion des différents fichiers PDF.

**`<Fichier PDF 1> : (Chaîne de caractères)`**

Nom et chemin complet du premier fichier PDF à fusionner.

**`<Mot de passe PDF 1> : (Chaîne de caractères)`**



- Mot de passe d'ouverture du premier fichier PDF,

- Chaîne vide ("") si le fichier n'a pas de mot de passe.




**`<Fichier PDF 2> : (Chaîne de caractères)`**

Nom et chemin complet du second fichier PDF à fusionner.

**`<Mot de passe PDF 2> : (Chaîne de caractères)`**



- Mot de passe d'ouverture du second fichier PDF, 

- Chaîne vide ("") si le fichier n'a pas de mot de passe.




**`<Fichier PDF 3> : (Chaîne de caractères optionnelle)`**

Nom et chemin complet du troisième fichier PDF à fusionner.

**`<Mot de passe PDF 3> : (Chaîne de caractères optionnelle)`**



- Mot de passe d'ouverture du troisième fichier PDF, 

- Chaîne vide ("") si le fichier n'a pas de mot de passe.






**`<Fichier PDF N> : (Chaîne de caractères optionnelle)`**

Nom et chemin complet du Nième fichier PDF à fusionner.

**`<Mot de passe PDF N> : (Chaîne de caractères optionnelle)`**



- Mot de passe d'ouverture du Nième fichier PDF, 

- Chaîne vide ("") si le fichier n'a pas de mot de passe.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- Il n'est pas possible de fusionner des fichiers PDF protégés par un mot de passe d'administration. 

- Jusqu'à la version 18 : 

	- Cette fonction génère uniquement des fichiers PDF version 1.4. 

	- La fusion n'est pas supportée pour les fichiers PDF version 1.5 et supérieures. 




- A partir de la version 19, les fichiers PDF version 1.5 sont gérés par WINDEV et WEBDEV. 

- Les signets présents dans les PDF ne sont pas conservés par la fusion.

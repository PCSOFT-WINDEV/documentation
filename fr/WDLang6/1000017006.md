
## VerrouAppliDélai (Fonction)

***En anglais : LockAppTimeout***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Renvoie ou modifie le délai d'inactivité avant qu'une application WINDEV passe en mode verrouillé. 

**Remarque** : Le délai d'inactivité correspond au temps écoulé depuis la dernière action de l'utilisateur sur l'application.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Verrouillage de l'application si elle n'est pas utilisée pendant 15 minutes 
VerrouAppliDélai(15)

// Application ne se verrouille plus
VerrouAppliDélai(-1)
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Connaître le délai d'inactivité

`<Résultat> = VerrouAppliDélai()`
---

**`<Résultat> : (Entier)`**



- Délai d'inactivité (exprimé en minutes) avant le verrouillage de l'application, 

- 0 (valeur par défaut) signifie que la fonctionnalité "Verrouillage de l'application" n'a pas été activée.  





<a name="SYNTAXE2"></a>

### Modifier le délai d'inactivité

`VerrouAppliDélai(<Nouveau délai>)`
---

**`<Nouveau délai> : (Entier ou Durée)`**



- Délai d'inactivité avant le verrouillage de l'application (exprimé en minutes). 

- Si cette valeur vaut 0, la fonctionnalité est désactivée (l'application ne sera jamais verrouillée). 


Ce paramètre peut correspondre à : 

- un entier correspondant au nombre de minutes, 

- une variable de type Durée, 

- l'indication directe de la durée (par exemple 1 min).

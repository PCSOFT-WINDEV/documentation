


## Créer une réplication
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000129"></a>
Pour mettre en place une réplication assistée, la première étape consiste à créer la réplication avec ReplicEdit. 

La création de la réplication est réalisée grâce à un assistant. 

Les paramètres définis dans cet assistant pourront à tout moment être modifiés dans la [description de la réplication](../ReplicEdit/3526502.md). 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Création de la réplication
<a name="creation_replication_ELTTEXTE000153"></a>
Pour créer une réplication : 

1. Lancez ReplicEdit depuis WINDEV, WEBDEV ou WINDEV Mobile : sous le volet "Outils", dans le groupe "Base de données", déroulez "Réplication" et sélectionnez "ReplicEdit - Editeur de réplication".

2. Si l'assistant de démarrage se lance, sélectionnez l'option "Créer une nouvelle réplication". 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Ass%20-%20HC%20N%B0001.gif)
Si non, sélectionnez l'option "Fichier .. Nouveau". L'assistant de création de réplication se lance.

3. Dans l'assistant de création de réplication, indiquez le nom et la description de la réplication. Le nom sera donné au fichier de réplication qui sera créé (fichier d'extension .WER).
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Ass%20-%20HC%20N%B0002.gif)
Passez à l'étape suivante. 

4. La description des fichiers de données à synchroniser doit être chargée dans ReplicEdit afin de créer la réplication. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Ass%20-%20HC%20N%B0003.gif)
L'assistant propose 2 possibilités : 

	- **Utiliser une analyse (WDD)**. Il suffit de sélectionner le chemin du fichier ".WDD" correspondant et le mot de passe associé (si nécessaire). 
			**Dans ce cas, il est nécessaire que l'analyse contienne une connexion HFSQL Client/Serveur. Cette connexion doit être associée aux fichiers de données** 
			En effet, ReplicEdit ne permet pas de créer une connexion. Il est possible uniquement de modifier une connexion existante. 

	- **Importer depuis un serveur HFSQL**. Cette option permet d'utiliser directement la description d'une base de données HFSQL Client/Serveur installée sur un serveur (méthode à utiliser par exemple si l'application est déployée). **Cette solution est à préconiser si vos fichiers sont décrits en HFSQL Client/Serveur dans l'analyse**. En effet, ReplicEdit va automatiquement affecter aux fichiers de données à répliquer la connexion que vous aurez utilisée.
			Dans ce cas, l'assistant demande : 

		- les paramètres de connexion : adresse, port, utilisateur et mot de passe. 

		- la base de données à répliquer. 




5. L'étape suivante permet de choisir le sens de la réplication. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Ass%20-%20HC%20N%B0004.gif)
Le sens de la réplication peut être :

	- Bidirectionnel : les données sont synchronisées sur la base de données maître et sur la base de données abonnée.

	- Vers l'abonné uniquement : les données sont synchronisées uniquement du maître vers l'abonné.

	- Vers le maître uniquement : les données sont synchronisées uniquement de l'abonné vers le maître.




6. L'étape suivante permet de choisir la méthode pour gérer les conflits de modification. 
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Ass%20-%20HC%20N%B0005.gif)
La gestion des conflits permet de gérer le cas où le même enregistrement a été modifié à la fois par le maître et par l'abonné. Choisissez la règle à appliquer :  

	- Plus récent prioritaire : seul le dernier enregistrement modifié est pris en compte.

	- Maître toujours prioritaire : l'enregistrement de la base maître est celui pris en compte.

	- Abonné toujours prioritaire : l'enregistrement de la base abonnée est celui pris en compte.
			Attention : pour éviter tout problème, il est important que le serveur et les postes abonnés utilisent la même heure. 




7. A la fin de l'assistant, il est possible d'ajouter un fichier à la réplication ou d'aller directement dans l'éditeur.
![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Ass%20-%20HC%20N%B0006.gif)


## Options de la réplication (ReplicEdit)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000138"></a>
La création d'une réplication avec ReplicEdit permet d'indiquer de nombreux paramètres :

- Les caractéristiques de la réplication (sens de réplication, mode de gestion des conflits, ...)

- Les différents fichiers à répliquer (fichiers de base, fichiers liés, ...).




**Selon le type des fichiers à répliquer, des informations complémentaires doivent être indiquées pour permettre de réaliser une réplication universelle assistée automatique (à l'aide de ReplicSynchro) ou programmée.**

Les options de réplication permettent notamment d'indiquer l'emplacement de la base maître.

<a name="NOTE2_1"></a>


## Réplication d'une base de données HFSQL Classic
<a name="replication_une_base_donnees_hfsql_classic_ELTTEXTE000162"></a>
Lors de la réplication d'une base de données HFSQL Classic, il est nécessaire d'indiquer dans le fichier de réplication la localisation des fichiers maître de la réplication.

Pour définir le répertoire des fichiers de données maître HFSQL Classic : 

1. Affichez la description de la réplication (option "Réplication .. Description de la réplication").

2. Sélectionnez l'onglet "Fichiers maîtres". Dans cet onglet, il est possible de :

	- Indiquer le répertoire de base pour les fichiers maître HFSQL Classic, à l'aide du sélecteur de répertoire.

	- Paramétrer les caractéristiques de chaque fichier. Il suffit de double-cliquer sur le nom du fichier : la fenêtre de description du fichier apparaît. Dans l'onglet "Général", il est possible de préciser :

		- le nom physique du fichier. Ce paramètre peut correspondre à un chemin complet, à un chemin relatif (qui sera ajouté au répertoire de base spécifié) ou à un nom de fichier. Si seul un nom de fichier est spécifié, il sera recherché dans le répertoire de base spécifié.

		- le mot de passe du fichier (si nécessaire).







**Remarque** : Ces informations peuvent également être fournies lors de la préparation de la procédure d'installation ou encore lors de l'installation.

## Réplication d'une base de données Client/Serveur
<a name="replication_une_base_donnees_clientserveur_ELTTEXTE000186"></a>


### Connexions
<a name="connexions_ELTPARAGRAPHE000047"></a>Lors de la réplication d'une application Client/Serveur, par défaut, ReplicEdit utilise les connexions définies dans l'analyse. **Pour des raisons de sécurité, seul le mot de passe n'est pas initialisé.**

Pour donner le mot de passe associé aux connexions (ou pour modifier les connexions), sélectionnez l'option "Réplication .. Description des connexions".

Pour modifier l'association d'un fichier à une connexion :

1. Sélectionnez l'option "Réplication .. Description des fichiers de données".

2. Sélectionnez le fichier voulu et cliquez sur l'icône de recherche.

3. Sélectionnez la connexion voulue.




Remarques :

- Réplication universelle assistée automatique (par ReplicSynchro) : Ces paramètres sont modifiables lors de la préparation de l'installation.

- Réplication universelle assistée par programmation : Seul ReplicEdit permet de modifier ces paramètres.





<a name="NOTE3_2"></a>


### Rubrique de réplication
<a name="rubrique_replication_ELTPARAGRAPHE000068"></a>Lors de la réplication d'une base de données utilisant un accès natif, il est nécessaire de définir une rubrique spécifique de type DateHeure dans chaque fichier répliqué. Cette rubrique sera mise à jour par l'application lors de la modification ou lors de l'ajout d'un enregistrement.

Pour indiquer la rubrique de réplication :

1. Sélectionnez l'option "Réplication .. Description des fichiers de données".

2. Double-cliquez sur le fichier concerné. La fenêtre de description du fichier s'affiche.

3. Dans l'onglet "Rubriques", indiquez la rubrique de datation des enregistrements.

## Sélection des rubriques à répliquer
<a name="selection_des_rubriques_repliquer_ELTTEXTE000216"></a>
ReplicEdit permet également de ne pas prendre en compte certaines rubriques des fichiers de données lors de la réplication.

Pour ne pas répliquer toutes les rubriques :

1. Sélectionnez l'option "Réplication .. Description des fichiers de données".

2. Double-cliquez sur le fichier de données concerné. La fenêtre de description du fichier s'affiche.

3. Dans l'onglet "Rubriques", seules les rubriques cochées seront répliquées.

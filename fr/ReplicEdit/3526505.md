
## Publier la réplication depuis ReplicEdit
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000100"></a>
Une description de réplication créée avec ReplicEdit peut être publiée directement depuis cet outil sur un serveur de réplication. 

Cette manipulation doit être réalisée notamment lors d'une réplication d'une application Android ou iOS. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comment le faire ?
<a name="comment_faire_ELTTEXTE000124"></a>


### Publier une réplication
<a name="publier_une_replication_ELTPARAGRAPHE000018"></a>Pour publier une réplication : 

1. Sélectionnez l'option "Déploiement .. Déployer la réplication sur un serveur". L'assistant se lance.

2. Sélectionnez le serveur sur lequel la réplication doit être déployée : 
	![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Publier.gif)


	- Un serveur de réplication dédié. 

- Un serveur de réplication du Cloud PC  SOFT. Dans ce cas, vous devez posséder un compte Cloud PC SOFT. 

3. Dans le cas d'un serveur de réplication dédié, ce serveur a été installé précédemment. Pour plus de détails, consultez [Serveur de réplication pour la réplication universelle assistée](../WDLang4/1000017227.md). 

4. Indiquez : 

	- l'adresse du serveur (de préférence l'adresse IP), 

	- un login et un mot de passe, 

	- le nom de la réplication (nom du fichier WER). 
			
			![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Publier_Serveur.gif)





5. Dans l'étape suivante, indiquez la périodicité de la préparation des données à répliquer : 
	
	![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Publier_Period.gif)


	- **Préparation immédiate** : la préparation des données à répliquer est effectuée lors d'une demande de réplication. Dans ce cas, la réplication permettra d'obtenir les données les plus à jour. Cependant, la préparation du réplica peut être lente.

- **Préparation périodique** : la préparation des données à répliquer est effectuée régulièrement, selon une périodicité définie. Dans ce cas, lors de la demande de réplication, les dernières données préparées sont automatiquement utilisées. Cette solution est rapide (le temps de préparation des données à répliquer est nul) mais les données peuvent ne pas être à jour. 
			Cette solution est conseillée dans le cas d'une réplication d'une application mobile (Android ou iOS). 

6. Validez. L'assistant propose le code WLangage à placer dans l'application pour synchroniser les données. 
	
	![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ReplicEdit_Publier_Code.gif)

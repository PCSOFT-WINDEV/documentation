## Présentation
<a name="presentation_ELTTEXTE000139"></a>
Un état permet d'obtenir une vue personnalisée d'informations : données saisies dans la base de données, données présentes dans un fichier texte, données provenant d'un champ Table, ... Pour plus de détails, consultez [Les états](../WDChamp/1011023.md).

La création et la manipulation d'un état s'effectuent sous l'éditeur d'états. Pour plus de détails sur l'interface de l'éditeur d'états, consultez [Environnement de l'éditeur d'états](../Editeurs/2029004.md).

Deux versions de l'éditeur d'états sont disponibles :

- ***l'éditeur d'états directement intégré***.
	Cette version est fournie en standard avec WINDEV, WEBDEV ou WINDEV Mobile. Cette version ne peut être utilisée que lors du développement d'une application WINDEV ou WINDEV Mobile, ou d'un site WEBDEV. Pour plus de détails, consultez [l'Editeur d'états](../WDChamp/1011069.md).

- ***l'éditeur d'états Utilisateur*** (appelé également le ***logiciel Etats & Requêtes***).
	Cet éditeur est livré uniquement avec WINDEV.
	Cette version est indépendante de WINDEV et peut être utilisée par un utilisateur final. 
	Certaines fonctionnalités de l'éditeur d'états ne sont pas disponibles dans cette version. Pour connaître les limitations de l'éditeur d'états Utilisateur, consultez [Limitations de l'éditeur d'états Utilisateur](../Presentation/3088005.md).


Pour plus d'informations concernant le logiciel Etats & Requêtes, contactez le service commercial de PC SOFT.

## Éditeur d'états Utilisateur (logiciel Etats & Requêtes)
<a name="editeur_etats_utilisateur_logiciel_etats_requetes_ELTTEXTE000163"></a>
Avec le logiciel Etats & Requêtes, il est possible de créer et d'exécuter des états directement depuis l'éditeur (sans avoir besoin de WINDEV).

Les états ne pourront pas contenir de code de modification des fichiers de données (code comprenant des fonctions d'écriture dans les fichiers).

Les états créés pourront être exécutés depuis l'éditeur d'états ou depuis une application WINDEV (si cette application gère les états créés avec le logiciel Etats & Requêtes).

Certaines fonctionnalités de l'éditeur d'états ne sont pas disponibles dans cette version. Pour connaître les limitations de l'éditeur d'états Utilisateur, consultez [Limitations de l'éditeur d'états Utilisateur](../Presentation/3088005.md).

L'icône suivante ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=picto_nondispo4.gif) indique les différences entre l'éditeur d'états Utilisateur et l'éditeur d'états intégré.

La clé électronique n'est pas nécessaire à l'utilisation du logiciel Etats & Requêtes.

Pour créer des nouveaux états ou pour modifier des états existants avec le logiciel Etats & Requêtes, certaines manipulations sont nécessaires. Pour plus de détails, consultez [Diffuser le logiciel Etats & Requêtes](../Presentation/3088002.md).

## Migration des états créés avec WDEtat Utilisateur
<a name="migration_des_etats_crees_avec_wdetat_utilisateur_ELTTEXTE000187"></a>
Les états créés avec WDEtat Utilisateur peuvent être ouverts sous le logiciel Etats & Requêtes. A leur ouverture, ces états sont automatiquement migrés en version 28.

Pour pouvoir utiliser ces états, nous vous conseillons d'enregistrer ces états sous le logiciel Etats & Requêtes. Pour enregistrer les états, sous le volet "Accueil", dans le groupe "Général", déroulez "Enregistrer" et sélectionnez "Enregistrer".

**Remarque** : Les états créés sous WDQBE ne peuvent pas être ouverts sous le logiciel Etats & Requêtes. Il est nécessaire de recréer ces états avec le logiciel Etats & Requêtes.

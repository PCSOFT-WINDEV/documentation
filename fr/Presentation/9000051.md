## Nouveautés et modifications "Update 3" (version 280085)
<a name="nouveautes_modifications_update_3_version_280085_ELTTEXTE000312"></a>
**Environnement**

- Nouvelle ambiance "Velvet".

- Le "copier / coller" d'un champ est accessible depuis son menu contextuel (en plus des raccourcis Ctrl + C et Ctrl + V).   




**Editeur de pages**

- Amélioration de la présentation des styles d'un champ (option "Choisir un style" du menu contextuel du champ). 

- **Changement de comportement pour les agencements** : dans un agencement : 

	- **les champs masqués** (par le menu contextuel "Agencements .. Visibilité par agencement .. Rendre invisible dans cet agencement") : 
			Ils ne sont jamais visibles (quelle que soit la valeur de la propriété Visible).

	- **les champs non masqués** :
			**A partir de la version 28 Update 3**, ils prennent **toujours** en compte la valeur de la propriété Visible (définie par défaut dans la fenêtre de description du champ, et modifiable par programmation)




- **Changement de comportement : caractéristiques d'un style CSS (onglet "Style" de la fenêtre de description des champs)** : 

	- Jusqu'à la version 28 Update 2, la sélection d'un style CSS associé au champ permettait de connaître les caractéristiques du style CSS du champ. (Aucune distinction n'était faite entre les caractéristiques de base et les caractéristiques surchargées). 

	- A partir de la version 28 Update 3, la sélection d'un style CSS associé au champ permet de distinguer les caractéristiques surchargées et non surchargées du style CSS du champ. Les caractéristiques non surchargées apparaissent avec le libellé &lt;Non défini&gt;.







**PHP**

- Modification de la version minimale conseillée pour les serveurs PHP 5 : 

	- Version 5.3 jusqu'à la version 28 Update 3 niveau 1. 

	- Version 5.6 à partir de la version 28 Update 3 niveau 2. 







Remarque : les textes en bleu correspondent aux améliorations de la version Update 3 niveau 2. Dans cette version, de nombreuses améliorations de fonctionnement ont été également apportées.

## Nouveautés et modifications "Update 2" (version 280075)
<a name="nouveautes_modifications_update_2_version_280075_ELTTEXTE000336"></a>
**Editeur de pages**

- Nouvelle police clipart disponible : 5000 cliparts à inclure dans vos libellés. 

- Interrupteur dans les champs Zone répétée : Il est désormais possible de manipuler par programmation des interrupteurs avec plusieurs options dans les champs Zone répétée. Pour chaque répétition, l'interrupteur peut avoir des valeurs différentes. 

- Modèles : Optimisation de la mise à jour des modèles.




**GDS / GIT**

- Connexion à un serveur Git par SSH (sans avoir à ressaisir ni login, ni mot de passe). 

- Optimisations des performances du GDS : La vitesse des manipulations suivantes ont été optimisées : 

	- Utilisation du GDS dans le Cloud. 

	- Réintégration d'éléments. 

	- Création de branche. 

	- Ouverture de l'administrateur du GDS. 

	- Suppression de fichiers. 

	- Suppression de répertoires. 

	- Gestion des gros projets contenant de nombreux fichiers. 







**Nouveautés WLangage** 

- Fonction **AdminWBCréeCompte** : Pour ajouter des comptes WEBDEV, le compte Windows utilisé doit être administrateur du poste Serveur. 

- Fonction **xlsRecalculeTout** : Recalcule toutes les formules d'un document Excel au format XLSX.

- Fonction **xlsRecupèreFusion** : Permet de savoir si une cellule est fusionnée avec d'autres cellules et d'obtenir la plage des cellules fusionnées.

- Type **xlsCellule** : Il est désormais possible de définir une formule pour une cellule d'un document grâce à la propriété **Cellule**. La fonction xlsRecalculeTout permet de recalculer la valeur de la formule. 




**Code navigateur** 

- La propriété **ValeurMémorisée** permet désormais de connaître la valeur mémorisée des champs Combo et ligne d'un champ Combo, des champs Liste et ligne d'un champ Liste (champs liés à un fichier de données ou champs manipulés par programmation).

- Lorsqu'une erreur Javascript du type "Propriété inexistante" est affichée, le nom de la propriété concernée est affichée dans le message d'erreur.

## Nouveautés et modifications "Update 1" (version 280066)
<a name="nouveautes_modifications_update_1_version_280066_ELTTEXTE000360"></a>
**ATTENTION** : 
**A partir de la version Update 1 (version 280066)**, certains formats internes de cette version ont évolué pour plus d'efficacité, et ne sont pas compatibles avec la précédente version de WINDEV/WEBDEV/WINDEV Mobile 28. Votre projet va donc être migré dans cette version. 
Si vous travaillez en équipe (avec ou sans GDS), l’ensemble de l'équipe doit passer en version U1 (ou supérieure).

**Centre de Suivi de projets**

- Les différents Widgets présents dans le tableau de bord du Centre de Suivi de projets peuvent désormais afficher les données du sprint global au tableau de bord ou d'un sprint spécifique. 




**Editeur de projet**

- Nouvelle ambiance : Maki.  




**Editeur de pages**

- Champ Kanban

	- Binding sur les champs Kanban. 

	- Nouvelles fonctions. 

	- Possibilité de définir des actions autorisées. 

	- Gestion du nombre maximal de cartes par liste. 

	- Agencements : Gestion du mode d'affichage du champ Kanban. 




- Champ Libellé : Accès aux variantes de mises en forme via le menu contextuel.  

- Styles CSS : Sélection du style CSS simplifiée

- Police d'icônes enrichie (5000 icônes supplémentaires) 

- Champ métier : Les champs métier spécifiques aux tables sont affichés lors de la création d'un champ Table.

- Pages WEBDEV dans une application mobile : 

	- Intégrer une page WEBDEV dans une fenêtre WINDEV Mobile (mode téléphone ou tablette). 

	- Interactions : 

		- La fenêtre appelle des procédures de la page. 

		- La page appelle des procédures de la fenêtre.  




- GO de l'agencement WINDEV ou de l'agencement WINDEV Mobile. 

- **Attention** : les pages ouvertes avec la version 28 Update 1 (version 280066) ne pourront plus être ouvertes dans une version inférieure. Une sauvegarde du projet est automatiquement réalisée.  




**Editeur de code**

- Recherche de code dans tous les éléments du projet : les tests automatiques sont désormais pris en compte dans cette recherche. 




**RAD**

- Pour chaque rubrique, il est désormais possible de différencier les informations destinées au RAD de l'analyse selon la plateforme de génération des fenêtres ou des pages. Cette fonctionnalité est disponible uniquement lors de l'utilisation des produits en mode intégré (WINDEV Mobile ou WEBDEV intégré à WINDEV). 




**HFSQL**

- Index nullable : Lors de l'activation de cette fonctionnalité dans l'éditeur d'analyses, le fichier d'index généré n'est plus compatible avec les versions inférieures à la 28. Seules les applications compilées en version 28 pourront accéder à ces fichiers. 

- Index nullable : Gestion des vues matérialisées avec index nullable

- Lors de l'installation du serveur HFSQL, l'équilibrage de charges est désactivé par défaut.

- HFSQL Spare : Sélection des sous-dossiers d'une base de données à prendre en compte par le Spare. Permet d'exclure des dossiers dont le contenu ne varie pas et ainsi de limiter le volume des données à transférer et à stocker. 




**Fabrique logicielle**

- Nouvelle action pour déployer dans le Store Privé. 




**Robot de surveillance** 

- Définition de groupe de destinataires pour les rapports. 

- Possibilité d'envoyer différents types de requêtes HTTP (GET, POST, PUT et DELETE) pour vérifier si un serveur répond.




**Serveur d'application WEBDEV**

- Activation à tout moment du site d'administration de la télémétrie. 

- Cluster WEBDEV : Le déploiement d'un nouveau site ou la mise à jour d'un site est automatiquement propagé sur tous les noeuds du Cluster WEBDEV.

- WebService REST pour Windows permettant de gérer les comptes et les connexions par programmation.




**Déploiement Docker**

- Le processus de création des images Docker est maintenant basé sur la version 11 de Debian.






**WLangage**

- Requête HTTP : Possibilité de connaître la version du protocole TLS et le cipher choisi pour exécuter la requête HTTP (nouvelles propriétés CipherChoisi et VersionSSLChoisie). 

- Connexion OAuth 2.0 en mode "Client Credentials".

- Google Analytics : Au 1er Juillet 2023, Google Analytics passe de Universal Analytics à Google Analytics 4. WEBDEV 28 gère les deux versions. Les fonctions WLangage ont été adaptées. 

- WebService : Génération des logs d'exécution. 

- L'événement "Demande de mise à jour de l'affichage" est désormais disponible en Ajax, avec actualisation des champs. 

- Amélioration de la mise à jour des champs depuis les tâches différées.

- AWP : Le nouvel événement WLangage serveur "Après mise à jour des valeurs navigateur des champs" permet de synchroniser la valeur des champs sur le serveur avec celle sur le navigateur au moment de l'appel Ajax.  

- Nouvelles fonctions : 

	- JSONValide : Teste la validité d'une chaîne au format JSON.

	- KanbanInfoXY : Renvoie pour une position donnée dans un champ Kanban (coordonnées d'un point du champ) soit le numéro de la carte pour la position indiquée, soit le numéro de la liste pour la position indiquée.

	- KanbanFiltre : Affiche uniquement les cartes correspondant au filtre spécifié.

	- KanbanAffiche : Rafraîchit l'affichage d'un champ Kanban ou d'une carte d'un champ Kanban.

	- xlsDéfusionneCellule : Supprime la fusion de la cellule spécifiée.

	- xlsFusionneCellule : Fusionne les cellules spécifiées dans un document Excel.




- Nouveautés navigateur : 

	- Automatismes de procédures

	- Glocales (variables globales de champs) 

	- Procédures de champs. 

	- Propriété Extra. 

	- Interfaces.

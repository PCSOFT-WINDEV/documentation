
## Centre de Contrôle Android
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000200"></a>
Savez-vous dans quel répertoire du téléphone se trouvent les fichiers de log de votre application ? 
Savez-vous enregistrer une vidéo de manipulation de votre appli sur votre téléphone ? Etc. 
Le Centre de Contrôle Android vous aide pour ces fonctions, et bien plus. 


Le Centre de Contrôle Android vous permet de gérer facilement vos matériels Android pour les phases de mise au point. 
Parmi les fonctionnalités offertes par le Centre de Contrôle du Téléphone, vous bénéficierez de : 

- Affichage des informations générales de l'appareil : version, résolution, DPI, processeur, etc. 

- Affichage des logs système (logcat),

- Affichage de la trace du débogueur pour les applications WINDEV Mobile : trace, assert, erreur, etc.

- Capture d'écran,

- Création de vidéos de manipulations,

- Gestion des applications de l'appareil : listage, recherche, infos sur les applications, installation, désinstallation, passage en mode device owner, etc.

- Explorateur de fichiers intuitif et réactif,

- ...

## Lancement du Centre de Contrôle Android
<a name="lancement_centre_controle_android_ELTTEXTE000224"></a>


### Pré-requis
<a name="prerequis_ELTPARAGRAPHE000035"></a>Avant de lancer le Centre de Contrôle Android : 

1. Connectez le périphérique Android au poste de développement. 

2. Assurez-vous que le mode "Débogage USB" est bien activé pour votre poste de développement (PC) dans les paramètres de l'appareil mobile (les options de développement doivent également être activées). 




Remarques : 

- Si plusieurs appareils mobile sont utilisés, le débogage USB doit être autorisé sur chaque appareil mobile.

- Si le débogage USB est activé sur le téléphone, lors de la connexion au poste de développement, il est nécessaire de donner l'autorisation depuis le téléphone (avec une coche "Toujours autorisé sur cet ordinateur"). 

- Si l'autorisation n'est pas mémorisée, cette demande d'autorisation sera demandée à chaque déconnexion / reconnexion du téléphone. 

- Si la demande d'autorisation est annulée, l'option "Débogage USB" est bien active sur le téléphone, mais l'autorisation de déboguer depuis le poste n'est pas activée.





### Lancement du Centre de Contrôle
<a name="lancement_centre_controle_ELTPARAGRAPHE000050"></a>Le Centre de Contrôle Android peut être lancé :

- depuis WINDEV Mobile : sous le volet "Outils", dans le groupe "Outils Mobile", cliquez sur "CCAndroid".

- en exécutant directement le programme "CCAndroid.EXE" présent dans le sous-répertoire "Programmes" du répertoire d'installation de WINDEV Mobile.




Au lancement, le Centre de Contrôle Android détecte le matériel connecté puis affiche les données concernant ce matériel.

Remarque : Si plusieurs appareils sont connectés, une liste permet de sélectionner l'appareil voulu. 
<a name="NOTE2_2"></a>


### Conditions d'utilisation
<a name="conditions_utilisation_ELTPARAGRAPHE000076"></a>Le Centre de Contrôle Android n'est pas un outil redistribuable. Le Centre de Contrôle Android ne peut pas être installé avec les applications développées avec WINDEV, WEBDEV ou WINDEV Mobile.

Pour utiliser le Centre de Contrôle Android, WINDEV, WEBDEV ou WINDEV Mobile doit obligatoirement être installé sur le poste en cours.

## Fonctionnalités
<a name="fonctionnalites_ELTTEXTE000260"></a>
Le Centre de Contrôle Android affiche les données concernant le périphérique sélectionné dans plusieurs onglets : 

- **Onglet "Informations"** : Cet onglet affiche les informations concernant le périphérique Android en cours : constructeur, modèle, numéro de série, API level, etc.
	Remarque : Sur certains téléphones, la liste des applications non-optimisées/optimisées ne se met pas à jour à moins de redémarrer le téléphone.

- **Onglet "Trace"** : Cet onglet permet d'obtenir les traces de votre application Android créée avec WINDEV Mobile. Pour obtenir les traces de votre application : 

	1. Ouvrez votre projet Android sous WINDEV Mobile. 

	2. Lancez votre application en mode test sur le périphérique connecté au poste de développement. 

	3. Utilisez votre application sur votre mobile. 

	4. Dans l'onglet "Trace" du Centre de Contrôle Android, vous pouvez afficher : 

		- la trace de votre application en **mode simple** : tous les messages affichés via la fonction [Trace](../WDLang1/3013050.md) seront affichés dans l'onglet "Trace" du Centre de Contrôle Android. 
						Ce mode correspond aux informations affichées dans la fenêtre de trace lors d'un GO via le simulateur. 

		- la trace de votre application en **mode complet** : Ce mode permet d'afficher : 

			- tous les messages affichés via la fonction [Trace](../WDLang1/3013050.md). Ces messages seront affichés dans l'onglet "Trace" du Centre de Contrôle Android.

			- les exceptions, les erreurs et les assertions rencontrées. 


 Ce mode correspond aux informations affichées dans le volet "Trace du débogueur" lors d'un Go via le simulateur. 

- **Onglet "Logcat"** : Cet onglet affiche tous les logs système enregistrés sur le périphérique Android, selon leur priorité. Il est possible d'effectuer une recherche dans les messages. 
	Cet onglet peut être utile pour rechercher les causes de problèmes de certaines applications Android.
	Remarque : Les priorités minimales peuvent correspondre à l'utilisation de fonctions WLangage : 

	- Priorité Info : Fonction [Trace](../WDLang1/3013050.md). 

	- Priorité Debug : Fonction [dbgSortieStandard](../WDLang1/3014020.md). 

	- Priorité Assert : Fonction [dbgAssert](../WDLang1/3014022.md).

	- Priorité Error : Fonction [ErreurDéclenche](../WDLang1/3034011.md).

	- Priorité Fatal : Fonction [ExceptionDéclenche](../WDLang1/3034005.md). 




- **Onglet "Capture"** : Cet onglet permet de capturer une image ou une vidéo (3 minutes maximum) de l'écran du périphérique. Cette image (ou vidéo) peut être enregistrée sur le poste de développement. Dans le cas d'une capture d'écran, cette capture peut être éditée et modifiée. 

- **Onglet "Applications"** : Cet onglet permet de visualiser la liste des applications installées et en cours d'utilisation sur le périphérique Android. Il est possible de : 

	- Voir le détail de l'application sélectionnée : nom, version, version du sdk minimale, permissions de l'application, ...

	- Installer une application sur le périphérique : Il suffit de sélectionner le fichier APK correspondant depuis le poste de développement. 

	- Désinstaller une application. 

	- Lancer l'application sélectionnée (si le fichier APK correspondant est disponible). 

	- Lancer l'application sélectionnée en lui passant des paramètres (si le fichier APK correspondant est disponible).

	- Arrêter l'application sélectionnée. 

	- Afficher le répertoire de l'application dans l'onglet "Explorateur". Cette option est disponible uniquement pour les applications déboguables.  




- **Onglet "Explorateur"** : Cet onglet affiche les différents dossiers présents dans le périphérique sous la forme d'un explorateur de fichiers.

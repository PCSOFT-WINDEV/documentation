
## WDTestSite : Création d'un scénario
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000104"></a>
Le scénario est un fichier texte (extension ".WCN") contenant toutes les manipulations effectuées pendant l'enregistrement du scénario.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Création d'un scénario
<a name="creation_scenario_ELTTEXTE000128"></a>
Pour créer un scénario, les éléments suivants doivent être installés sur le poste en cours :

- WEBDEV version Développement ou Serveur d'application WEBDEV.

- le site WEBDEV utilisé pour créer le scénario.




Il est conseillé de créer le scénario directement sur le serveur Web où le site WEBDEV est installé (poste de déploiement).


<a name="NOTE2_2"></a>


### Créer un scénario sur le poste de déploiement
<a name="creer_scenario_sur_poste_deploiement_ELTPARAGRAPHE000030"></a>Pour créer un scénario sur le poste de déploiement : 

1. Lancez WDTestSite (double-clic sur WDTestSite.EXE par exemple).

2. Vérifiez que le serveur WEBDEV utilisé correspond à l'option "Machine locale".

3. Vérifiez que l'administrateur WEBDEV est lancé (option "Programmes .. WEBDEV 28 .. Administrateur WEBDEV" du menu "Démarrer").

4. Cliquez sur le bouton "Nouveau".

5. Indiquez le nom du scénario et sa description. La description peut par exemple correspondre à un résumé des fonctionnalités testées dans le scénario. Cette description vous permettra d'identifier facilement le scénario et ses fonctionnalités.
	Attention :

	- La description du scénario ne pourra pas être modifiée par la suite.

	- Si le nom du scénario correspond à un scénario existant sur ce poste, le scénario existant sera remplacé par le nouveau scénario.




6. Cliquez sur le bouton "Enregistrement" : la page de test de l'administrateur de WEBDEV s'affiche.

7. Sélectionnez le site WEBDEV utilisé pour créer le scénario.

8. Effectuez toutes les manipulations nécessaires dans le site WEBDEV.
	Remarques :

	- Pour tester la capacité du serveur à supporter un nombre important de connexions simultanées, il est conseillé de laisser le site connecté après la dernière action : pendant l'enregistrement du scénario, n'utilisez pas de bouton ou de lien permettant de fermer le site.

	- Pour tester la non-régression d'un site ou la gestion des accès concurrentiels, vous pouvez quitter le site (par un lien ou un bouton) lors de l'enregistrement du scénario.




9. Pour terminer l'enregistrement du scénario, cliquez sur le bouton "Terminer" de WDTestSite. 
	Le scénario (fichier d'extension ".WCN") est alors créé dans le sous-répertoire "Scénario" du répertoire contenant le programme "WDTestSite.EXE".

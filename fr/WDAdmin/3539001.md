
## Personnalisation des erreurs
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000207"></a>
L'administrateur WEBDEV version Développement permet de personnaliser les messages d'erreur d'un site WEBDEV.

Les messages personnalisés d'un site sont sauvegardés et automatiquement installés lors du déploiement du site WEBDEV.

Remarque : Il est également possible de personnaliser la page des erreurs du site depuis le projet WEBDEV. Pour plus de détails, consultez [Gestion des erreurs du site](../WDChamp/9000134.md).



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Personnalisation des erreurs
<a name="personnalisation_des_erreurs_ELTTEXTE000231"></a>
Il est possible de personnaliser :

- les messages d'erreur de tous les sites.

- les messages d'erreur d'un site spécifique.



<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=WDADMIN_Onglets%20-%20HC%20N%B0007.gif&type=thumb)

<a name="NOTE2_2"></a>


### Personnalisation des erreurs de tous les sites
<a name="personnalisation_des_erreurs_tous_les_sites_ELTPARAGRAPHE000038"></a>Pour personnaliser les messages d'erreur de tous les sites WEBDEV installés sur le poste en cours (et sur le serveur Web lors de l'installation), choisissez l'option "Tous les sites" dans la combo "Site".

Cette personnalisation concerne :

- le texte des messages d'erreur.

- la page affichant le message d'erreur.





<a name="NOTE2_3"></a>


### Personnalisation des erreurs d'un site spécifique
<a name="personnalisation_des_erreurs_site_specifique_ELTPARAGRAPHE000051"></a>Pour personnaliser les messages d'erreur d'un site spécifique du poste en cours (et sur le serveur Web lors de l'installation), sélectionnez le nom de votre site dans la combo "Site".

Cette personnalisation concerne :

- le texte des messages d'erreur.

- la page affichant le message d'erreur.





<a name="NOTE2_4"></a>


### Ordre de priorité de la personnalisation
<a name="ordre_priorite_personnalisation_ELTPARAGRAPHE000064"></a>L'ordre de personnalisation pris en compte est :

1. la personnalisation de tous les messages de tous les sites installés sur le serveur.

2. la personnalisation des messages d'un site particulier.




**Attention** : La personnalisation des messages d'erreur de tous les sites est déconseillée lors de l'utilisation d'un serveur mutualisé. En effet, cette modification sera appliquée pour tous les sites WEBDEV installés sur le serveur.

## Comment le faire ?
<a name="comment_faire_ELTTEXTE000273"></a>
Pour personnaliser les messages d'erreur : 

1. Sélectionnez le site concerné par la personnalisation (site spécifique ou tous les sites).

2. Pour personnaliser ***la page utilisée pour toutes les erreurs*** :
	Par défaut, les erreurs sont affichées dans une page spécifique. Pour personnaliser cette page :

	- Choisissez l'option "Personnalisé" :<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=WDADMIN_Onglets%20-%20HC%20N%B0007%202.gif&type=thumb)


	- Sélectionnez une page HTML accessible depuis votre poste.
			Cette page HTML doit être une page HTML statique, pouvant contenir des images, ... 
			Deux champs libellé sont obligatoires dans cette page : Ces champs doivent avoir respectivement pour valeur les balises [%MSG%] et [%SYS%]. Ces champs seront remplacés par l'identifiant et le libellé de l'erreur. 
			Il est conseillé de placer cette page HTML dans le répertoire &lt;NomProjet&gt;_WEB du projet.

3. Pour personnaliser ***le message et/ou la page d'une erreur*** :

	- Choisissez l'identifiant de l'erreur : le libellé par défaut s'affiche. 
			Pour obtenir une explication sur ces messages, consultez [Messages d'erreur pouvant être affichés dans le navigateur](../WDAdmin/3539002.md).<br>![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=WDADMIN_Onglets%20-%20HC%20N%B0007%201.gif&type=thumb)


	- En sélectionnant l'option "Libellé personnalisé", vous pouvez saisir le nouveau libellé de l'option. 
			Remarque : Les caractères spéciaux utilisés dans les messages :

	- la balise {RECONNECT} signifie que l'internaute sera reconnecté automatiquement après l'affichage de l'erreur.

	- les caractères %x seront remplacés automatiquement lors de l'affichage de l'erreur.

4. Pour ***appliquer les changements*** :

	- Pour appliquer les changements immédiatement sur le poste en cours, cliquez sur le bouton "Appliquer".

	- Pour sauver les changements dans un fichier ".REG", cliquez sur le bouton "Sauver". 
			Si le site sélectionné est un site spécifique, le fichier de configuration est sauvé dans le répertoire du projet.
			Si le site sélectionné est "Tous les sites", le fichier de configuration est sauvé dans le répertoire de WEBDEV.




5. Pour ***installer les fichiers modifiés*** :

	- Si la personnalisation concerne un site spécifique, l'installation à distance sélectionnera automatiquement le fichier de configuration, et l'exécutera sur le serveur. Si le fichier HTML de personnalisation de la page d'erreur est présent dans le répertoire des images du projet (&lt;Nom_Projet&gt;_Web), il sera automatiquement installé.

	- Si la personnalisation concerne tous les sites présents sur le serveur, il est nécessaire de :

		- Étape optionnelle : copier si nécessaire les fichiers HTML des pages d'erreur et les images associées sur le serveur Web. 
						Ces fichiers doivent se trouver dans un répertoire du serveur pour lequel un alias a été défini.

		- copier et exécuter le fichier ERR_MSG_WebDev.REG sur le serveur Web.







**Attention** : Cette personnalisation est une modification avancée, nécessitant des connaissances dans la gestion de serveurs WEB.

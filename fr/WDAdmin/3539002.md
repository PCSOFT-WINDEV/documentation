## Présentation
<a name="presentation_ELTTEXTE000882"></a>
Cette page d'aide présente les différents messages pouvant être affichés dans le navigateur.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Message d'erreur
<a name="message_erreur_ELTTEXTE000906"></a>
Les messages d'erreur pouvant être affichés dans le navigateur sont les suivants :

| Erreurs | Désignation |
| --- | --- |
| Error opening access log. Check setup | Ce message apparaît au lancement d'un site WEBDEV.<br><br>Ce message est affiché par le serveur OmniHTTPd. Ce message peut avoir plusieurs causes :<br><br>- le serveur OmniHTTPd est installé dans un répertoire utilisant des noms longs. <br>	**Solution** : Ré-installez le serveur OmniHTTPd dans un répertoire n'utilisant pas des noms longs.<br><br>- plusieurs serveurs OmniHTTPd sont ouverts. <br>	**Solution** : Fermez tous les serveurs ouverts et relancez une seule instance de OmniHTTPd.<br><br>- le répertoire "Log File" n'existe pas. <br>	**Solution** : Dans le panneau de configuration du serveur, désactivez tous les répertoires "Log File" et relancez le serveur.<br><br><br> |
| HTTP/1.0 404 Objet non trouvé ou Ressource not found | Ce message apparaît dans le navigateur Internet, lors du lancement du site.<br><br>Ce message est dû à une mauvaise configuration du Serveur Internet. Il signifie que le serveur n'arrive pas à trouver le gestionnaire de protocole CGI.<br><br>**Solution** : Vérifiez la configuration du Serveur Internet utilisé, notamment :<br><br>- le répertoire virtuel (ou Alias), pointant sur le répertoire contenant le gestionnaire de protocole CGI, WD280AWP.EXE<br><br>- la clé de registre du gestionnaire de protocole CGI doit être égale à WD280AWP.EXE.<br><br><br>Rappel : Les informations concernant le paramétrage de WEBDEV sont enregistrées dans la base de registre, dans l'entrée "HKEY_LOCAL_MACHINE\\SOFTWARE\\PC SOFT\\WEBDEV\\28.0". |
| Internet Explorer ne peut ouvrir le site Internet ... | Ce message est affiché par le navigateur Internet lors de l'exécution d'un site WEBDEV si aucun serveur Internet n'est lancé lors de l'exécution du site.<br><br>**Solution** : Lancez un serveur Internet avant de lancer un site WEBDEV. |
| Could not resolve local name, Error code : 11004 | Ce message est affiché au lancement du serveur OmniHTTPD.<br><br>**Solution** :<br><br>1. Ouvrez les propriétés du serveur OmniHTTPD (clic droit sur l'icône du serveur dans la barre de tâches).<br><br>2. Affichez l'onglet "Server".<br><br>3. Décochez l'option "Lookup Site Address (Dynamic)".<br><br>4. Validez et redémarrez le serveur.<br><br><br> |
| RunTime Erreur .WDD absent ou nom invalide JW06 | Si vous utilisez un site WEBDEV en mode développement :<br><br>- Ouvrez votre projet sous WEBDEV.<br><br>- Sauvez votre projet.<br><br><br>Si vous utilisez un site installé sur le serveur d'application WEBDEV :<br><br>- Recompilez votre projet : sous le volet "Projet", dans le groupe "Projet", déroulez "Recompiler et synchroniser" et sélectionnez "Recompiler le projet".<br><br>- Regénérez la bibliothèque.<br><br>- Recréez l'installation de votre site.<br><br>- Ré-installez le site sur le serveur.<br><br><br>**Remarque** :<br><br>- Dans tous les cas, vérifiez que le répertoire d'installation de l'exemple correspond au répertoire utilisé par l'administrateur WEBDEV (onglet "Applications").<br><br>- Dans le cas contraire :<br><br>	- supprimez le site de l'administrateur.<br><br>	- en version de développement, ouvrez votre projet sous WEBDEV et cliquez sur l'icône "GO".<br><br>	- en version de déploiement, ré-installez le site.<br><br><br><br><br><br> |
| 1, ERR_MISSING_PAGE | La requête est incomplète. Elle ne contient pas d'information sur le contexte concerné.<br><br>Ce message d'erreur est affiché lors d'un clic sur un lien n'utilisant pas une syntaxe valide, ou pointant sur une page HTML inexistante.<br><br>**Solution** : Vérifiez la syntaxe utilisée pour le lien et l'existence de la page appelée. |
| 4, ERR_BUTTON_EXPECTED | La requête est erronée. Il manque l'action à effectuer.<br><br>Ce message d'erreur peut apparaître dans plusieurs cas :<br><br>- un lien est spécifié dans la page HTML avec la syntaxe [%URL()%] sans nom de bouton dans les paramètres.<br>	**Solution** : Il faut obligatoirement spécifier dans les paramètres d'un lien le nom du bouton dans la page WEBDEV correspondant à l'action à effectuer.<br><br>- un clic a été effectué sur un bouton existant dans la page HTML uniquement. Ce bouton n'existe pas dans la page. <br>	**Solution** : Il est nécessaire que les boutons présents dans la page HTML aient leur équivalent dans la page WEBDEV correspondante.<br><br>- une validation de formulaire est réalisée sans utiliser un bouton (utilisation de code JavaScript) ou validation réalisée par le bouton ENTREE. <br>	**Solution** : Il est nécessaire d'utiliser un bouton pour réaliser la validation d'un formulaire. <br>	**Astuce** : Pour permettre à l'internaute de valider un formulaire par la touche ENTREE, il suffit de définir un bouton de type "Envoyer la valeur des champs au serveur" dans la page WEBDEV en lui associant la lettre d'appel ENTREE (onglet "Détail" du champ).<br><br><br> |
| 5, ERR_NO_CURRENT_PAGE | Le site a mal fonctionné car il n'a pas renvoyé de page (peut être un appel à la fonction <u><u><u><u>FinProgramme</u></u></u></u> sans avoir renvoyé de page).<br><br>Ce message d'erreur survient lorsqu'un site WEBDEV est fermé, sans utiliser de page HTML signalant la fin de la connexion.<br>Pour éviter ce message, il est nécessaire d'afficher une page de fin de connexion grâce à la fonction du WLangage <u><u><u><u>PageAffiche</u></u></u></u> ou <u><u><u><u>FinProgramme</u></u></u></u>. |
| 6, ERR_PARSER | Erreur de syntaxe dans le fichier HTML &lt;NOM_FICHIER&gt; ligne &lt;NUM_LIGNE&gt;.<br><br>Une mauvaise syntaxe a été utilisée dans la page HTML affichée.<br>Vérifiez le code présent dans les champs HTML, ou le code HTML inséré dans les champs de votre site WEBDEV (onglet "Avancé" de la fenêtre de description des champs). |
| 7, ERR_BAD_ACTION | L'action spéciale &lt;NOMACTION&gt; est inconnue.<br><br>Une action spécifique a été définie à l'aide du paramètre WD_ACTION_ dans la page HTML, mais les paramètres employés ne sont pas corrects. Il est nécessaire de corriger la syntaxe utilisée pour définir l'action.<br>Cette erreur peut survenir par exemple lors de la mise en place d'un lien permettant d'ouvrir ou de fermer une transaction sécurisée. Vérifiez et corrigez la syntaxe utilisée. |
| 8, ERR_NO_PAGE | Vous avez demandé à travailler sur la page &lt;NOM_PAGE&gt;. | 9, ERR_BAD_PARAMETER | La commande &lt;NOM_COMMANDE&gt; ne comporte pas les bons paramètres.<br><br>Une action spécifique a été définie à l'aide du paramètre WD_ACTION_ dans la page HTML, mais les paramètres employés ne sont pas corrects. Il est nécessaire de corriger la syntaxe utilisée pour définir l'action.<br>Cette erreur peut survenir lors de l'utilisation d'une action personnalisée définie pour un bouton ou un lien, ou lors de l'utilisation d'un lien pour ouvrir ou fermer une transaction sécurisée. Vérifiez la syntaxe utilisée. |
| 10, ERR_BAD_FILE | Le fichier &lt;NOM_FICHIER&gt; ne peut être ouvert (et donc renvoyé).<br><br>Le fichier HTML devant être affiché sous le navigateur n'existe pas.<br><br>**Solution** : Vérifiez la présence du fichier dans le répertoire du site. |
| 11, ERR_WWADMIN_EXIST | Le programme d'administration de WEBDEV a déjà été lancé. Il ne peut en exister qu'un seul par machine.<br><br>Il est impossible de lancer plusieurs sessions de l'administrateur WEBDEV (WD280ADMIN.EXE) sur le même poste. |
| 12, ERR_NO_WWADMIN | Le programme d'administration de WEBDEV n'a pas été lancé.<br><br>L'accès à un site dynamique WEBDEV ne peut être effectué que si l'administrateur WEBDEV (WD280ADMIN.EXE) est lancé et s'exécute sur le poste du Serveur Internet.<br><br>**Solution** : Il est nécessaire de vérifier si l'administrateur WEBDEV est bien lancé sur le poste serveur. |
| 13, ERR_OUT_OF_MEMORY | Pas assez de mémoire pour effectuer l'opération.<br><br>Le programme ne peut s'exécuter par manque de mémoire.<br><br>**Rappel : La ressource nécessaire par internaute connecté :**<br>Pour chaque internaute connecté, il est nécessaire de prévoir une consommation :<br><br>- d'environ 400 Ko de mémoire centrale par connexion, en plus de la mémoire nécessaire au serveur.<br><br>- d'environ 1 Mo d'espace disque par connexion.<br><br><br>Par exemple, pour 20 connexions parallèles, il est nécessaire de prévoir environ :<br><br>- Mémoire vive : 67,8 Mo (7,8 Mo de mémoire centrale pour ce site, 60 Mo pour le serveur NT)<br><br>- Mémoire virtuelle : 84 Mo (20 Mo pour les connexion, 64 Mo pour le serveur NT)<br><br><br> |

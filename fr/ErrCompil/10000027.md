
## Erreur 27 : Il manque le FIN associé à ce SINON
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Cause
<a name="cause_ELTTEXTE000103"></a>
Vous avez utilisé une instruction conditionnelle de type SI ALORS SINON. Pour terminer cette instruction, il est nécessaire d'utiliser le mot-clé FIN.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Correction
<a name="correction_ELTTEXTE000127"></a>
Vérifiez le code de votre instruction conditionnelle et ajoutez si nécessaire le mot-clé FIN manquant.



<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Exemples
<a name="exemples_ELTTEXTE000151"></a>


### Utilisation d'une instruction conditionnelle sans FIN
<a name="utilisation_une_instruction_conditionnelle_sans_fin_ELTPARAGRAPHE000027"></a>

**Code provoquant l'erreur** 


```wl
SI TABLE_Cours.Vide ALORS 
	IMG_Dessiner.Etat = Grisé
SINON
	IMG_Dessiner.Etat = Invisible
```




**Correction proposée**
Utilisez le mot-clé FIN pour indiquer que l'instruction conditionnelle est terminée.



```wl
SI TABLE_Cours.Vide ALORS 
	IMG_Dessiner.Etat = Grisé
SINON
	IMG_Dessiner.Etat = Grisé
FIN
```



<a name="NOTE3_2"></a>


### Instructions conditionnelles imbriquées
<a name="instructions_conditionnelles_imbriquees_ELTPARAGRAPHE000044"></a>

**Code provoquant l'erreur** 


```wl
SI SAI_Age<18 ALORS
	LIB_Libellé = "Enfant"
SINON
	SI SAI_Age<60 ALORS
	   LIB_Libellé = "Personne active"
FIN
```




**Correction proposée**
Vérifiez que toutes les instructions imbriquées se terminent correctement (par l'instruction FIN par exemple). Dans notre exemple, la seconde instruction SI n'a pas de FIN associée. Ajoutez une instruction FIN.



```wl
SI SAI_Age<18 ALORS
	LIB_Libellé = "Enfant"
SINON
	SI SAI_Age<60 ALORS
	   LIB_Libellé = "Personne active"
	FIN
FIN
```

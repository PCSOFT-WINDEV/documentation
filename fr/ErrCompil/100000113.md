
## Erreur 113 : Tableau non dynamique sans définition
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Cause
<a name="cause_ELTTEXTE000096"></a>
Vous définissez une variable de type tableau sans préciser les éléments de ce tableau.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Correction
<a name="correction_ELTTEXTE000120"></a>
Complétez la syntaxe de déclaration du tableau en précisant les dimensions du tableau et le type des éléments du tableau.

**Remarque** : Si vous voulez utiliser un tableau dynamique, ajoutez le mot-clé DYNAMIQUE lors de la déclaration du tableau.



<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Exemple
<a name="exemple_ELTTEXTE000144"></a>


### Déclaration d'un tableau
<a name="declaration_tableau_ELTPARAGRAPHE000030"></a>

**Code provoquant l'erreur** 


```wl
Tab_CA est un tableau
```




**Corrections proposées**

Indiquez les dimensions du tableau.



```wl
Tab_CA est un tableau de 10 par 10 réels
```

Utilisez le mot-clé DYNAMIQUE pour indiquer que le tableau est dynamique. Les dimensions seront définies dans la suite du code par la fonction [Dimension](../WDLang1/3013022.md).



```wl
Tab_CA est un tableau dynamique
```

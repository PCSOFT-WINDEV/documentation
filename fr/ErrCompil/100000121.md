
## Erreur 121 : Masquage de la structure
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Vous avez déclaré une structure : une structure du même nom existe déjà dans la fenêtre en cours ou dans le projet.

Si vous conservez ce nom, la structure "globale" ne sera plus visible dans le traitement en cours : c'est la structure locale qui sera utilisée.

Attention : Si vous redéfinissez une structure globale au projet dans le code d'initialisation de la fenêtre, la nouvelle structure sera utilisée dans tous les traitements de la fenêtre et de ses champs.

**Rappel** : Portée des structures

- Les structures déclarées dans le code d'initialisation du projet sont visibles dans tout le projet. 

- Les structures déclarées dans le code d'ouverture des fenêtres sont visibles dans tous les traitements de la fenêtres et de ses champs. 

- Les structures déclarées dans le code de déclaration des classes sont visibles dans tous les méthodes de la classe. 






**Différences avec la version 5.5** :

Dans WINDEV 5.5, dès qu'une structure est déclarée, elle reste publique. Si une structure est déclarée dans le traitement d'ouverture d'une fenêtre, elle reste publique et donc sera reconnue dans tous les autres traitements même si la fenêtre dans laquelle elle a été déclarée est fermée.

A partir de WINDEV 11, cette affirmation n'est plus vraie : une structure déclarée dans le code d'ouverture de la fenêtre est accessible tant que la fenêtre est ouverte.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Conseil
<a name="conseil_ELTTEXTE000107"></a>
Évitez d'utiliser des mêmes identifiants pour des structures différentes.

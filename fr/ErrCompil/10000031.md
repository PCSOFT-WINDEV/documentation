
## Erreur 31 : Le fichier à inclure est introuvable. Vérifier le nom et le chemin d'accès ...
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Cause
<a name="cause_ELTTEXTE000093"></a>
Cette ligne de code utilise le mot-clé EXTERNE avec un nom de fichier. Le fichier n'est pas trouvé car :

- Le nom du fichier n'est pas correct (faute d'orthographe dans le nom du fichier par exemple).

- Le chemin permettant de retrouver le fichier n'est pas correct (le chemin spécifié n'existe pas par exemple).




**Rappel** : Le mot-clé EXTERNE permet d'insérer un fichier dans le code WLangage. Par défaut, si aucun chemin n'est précisé pour le fichier, le fichier est recherché dans le répertoire du projet.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Correction
<a name="correction_ELTTEXTE000117"></a>
Vérifiez le chemin et le nom du fichier à inclure dans le code.

**Remarque** : Le sous-répertoire "\\Personnel\\WLangage" de WINDEV, WEBDEV ou WINDEV Mobile peut être utilisé pour stocker des fichiers à inclure. Le chemin est alors automatique : le fichier est automatiquement recherché dans ce répertoire.



<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Exemple
<a name="exemple_ELTTEXTE000141"></a>


### Utilisation d'un nom de fichier qui n'existe pas dans le répertoire du projet
<a name="utilisation_nom_fichier_qui_nexiste_pas_dans_repertoire_projet_ELTPARAGRAPHE000034"></a>

**Code provoquant l'erreur** 


```wl
EXTERNE "Constantes.txt"
```




**Correction proposée**

Créez le fichier ou modifiez le chemin d'accès au fichier.



```wl
EXTERNE "C:\Constantes\Constantes.txt"
```


## Connecteur Natif Progress : Spécificités et remarques

<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000185"></a>
Les paragraphes suivants présentent les remarques générales et les spécificités du Connecteur Natif Progress.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Remarques générales
<a name="remarques_generales_ELTTEXTE000209"></a>


### Versions nécessaires
<a name="versions_necessaires_ELTPARAGRAPHE000018"></a>**Attention** : Cette version du Connecteur Natif Progress ne fonctionne qu'avec WINDEV 28 et/ou WEBDEV 28.
<a name="NOTE2_2"></a>


### Utilisation d'ordres WLangage dans les requêtes SQL
<a name="utilisation_ordres_wlangage_dans_les_requetes_sql_ELTPARAGRAPHE000029"></a>Dans les requêtes SQL, il n'est pas possible d'utiliser des ordres WLangage. 
<a name="NOTE2_3"></a>


### Rubriques Unicode
<a name="rubriques_unicode_ELTPARAGRAPHE000036"></a>Les rubriques Unicode ne sont pas gérées par le Connecteur Natif Progress. Il est conseillé d'utiliser des rubriques texte au format Ansi, et de réaliser si nécessaire la conversion avec la fonction [UnicodeVersAnsi](../WDLang1/3024025.md).

La fonction [HCréation](../WDLang4/3044255.md) affichera un message d'erreur lors de la création de fichiers de données contenant des rubriques au format Unicode.
<a name="NOTE2_4"></a>


### Table nécessaire
<a name="table_necessaire_ELTPARAGRAPHE000051"></a>La table "sysprogress.systables" doit être accessible en lecture à l'utilisateur connecté. Si ce n'est pas le cas, l'erreur "Version de Progress non supportée" est affichée.
<a name="NOTE2_5"></a>


### Requête de type UPDATE sur un mémo
<a name="requete_type_update_sur_memo_ELTPARAGRAPHE000058"></a>Progress ne permet pas de faire une requête de type UPDATE sur une rubrique de type Mémo (LVARBINARY). Cependant, WINDEV et WEBDEV permettent de mettre à jour le mémo en utilisant la fonction [HModifie](../WDLang4/3044042.md) (l'enregistrement est supprimé puis ajouté).
<a name="NOTE2_6"></a>


### Insertion répétée de plus de 100 enregistrements avec un mémo
<a name="insertion_repetee_plus_100_enregistrements_avec_memo_ELTPARAGRAPHE000068"></a>L'insertion répétée de plus de 100 enregistrements contenant une rubrique mémo provoque l'erreur 9799. Cette erreur est référencée chez Progress (référence P26935).

Pour éviter cette erreur, fermez la connexion avant d'atteindre les 100 enregistrements, puis rouvrez la connexion.
<a name="NOTE2_7"></a>


### Mémo Texte et mémo binaire
<a name="memo_texte_memo_binaire_ELTPARAGRAPHE000077"></a>Tous les mémos sont de type binaires dans une base de données Progress. Il est cependant possible d'enregistrer du texte dans les mémos binaires.

## Spécificités du Connecteur Natif Progress
<a name="specificites_connecteur_natif_progress_ELTTEXTE000269"></a>


### Options de l'analyse
<a name="options_analyse_ELTPARAGRAPHE000086"></a>

- Les options de l'analyse concernant les index full-text ne sont pas gérées par du Connecteur Natif.

- Les options de l'analyse concernant l'indexation partielle des mémos texte ne sont pas gérées par le Connecteur Natif.



<a name="NOTE3_2"></a>




### Requêtes
<a name="requetes_ELTPARAGRAPHE000095"></a>Il n'est pas possible de réaliser des recherches, des filtres ou des tris (ORDER BY) sur des colonnes agrégat (Min, Max, ...).
<a name="NOTE3_3"></a>


### Mémo
<a name="memo_ELTPARAGRAPHE000102"></a>

- Il n'est pas possible d'insérer un mémo de plus de 2 Mo.

- Il n'est pas possible de récupérer un mémo dans une chaîne Texte avec la fonction [SQLLitMémo](../WDLang4/3072020.md) ou la fonction [SQLLitMémoTexte](../WDLang4/3072012.md). Il est nécessaire d'utiliser la fonction [HExtraitMémo](../WDLang4/3044072.md). Ce problème sera corrigé dans une prochaine version.



<a name="NOTE3_4"></a>


### Fermeture de connexion et transaction
<a name="fermeture_connexion_transaction_ELTPARAGRAPHE000119"></a>La fermeture d'une connexion peut entraîner une Erreur Générale de Protection si la transaction n'a pas été terminée. Ce problème sera résolu dans une prochaine version.
<a name="NOTE3_5"></a>


### Accents dans les alias
<a name="accents_dans_les_alias_ELTPARAGRAPHE000126"></a>Si un alias comporte des accents, il est nécessaire de :

- mettre l'alias entre doubles quotes,

- utiliser la constante *hRequêteSansCorrection* lors de l'exécution d'une requête utilisant cet alias.



<a name="NOTE3_6"></a>


### Blocage HFSQL
<a name="blocage_hfsql_ELTPARAGRAPHE000136"></a>Il n'est PAS possible d'utiliser les blocages d'enregistrements avec les fonctions HFSQL dans cette version.


## Paramètres utiles d'une procédure
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Une procédure peut déclarer certains paramètres qui ne sont pas immédiatement utilisés. Dans ce cas, par défaut un warning apparaît sous l'éditeur de code indiquant que la variable locale n'est pas utilisée. 

Pour éviter ce warning, il suffit de faire suivre le nom de la variable d'un attribut d'extension spécifique. La syntaxe à utiliser est la suivante : 


```txt
PROCEDURE <Nom de la procédure>(<Paramètre 1> <utile [="Raison"]>, ...
	<Paramètre optionnel 1> = <Valeur> <utile [="Raison"]>,
	<Paramètre optionnel 2> = <Valeur> <utile [="Raison"]>,...)
```


où "Raison" correspond à une indication donnée par le développeur. 

Par exemple :


```wl
// -- Déclaration de la procédure Dessin
PROCEDURE Dessin(coorLigne <utile = "Coordonnée ligne">, coorColonne <utile = "coordonnée colonne">)
...
```

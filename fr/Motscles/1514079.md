
## Rectangle (Type de variable)

***En anglais : Rectangle***
				



<a name="description"></a>
Une variable de type **Rectangle** est un type structuré qui permet de représenter un rectangle (X, Y, Largeur, Hauteur) en 2 dimensions. 






<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MonRectangle est un Rectangle
MonRectangle.X = 280
MonRectangle.Y = 54
MonRectangle.Largeur = 50
MonRectangle.Hauteur = 150
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Déclarer et initialiser un rectangle

`<Nom du rectangle> est [un] Rectangle
`
---

**`<Nom du rectangle> : ()`**

Nom de la variable Rectangle à déclarer.

<a name="NOTE0_1"></a>

## Remarques


### Propriétés WLangage utilisables avec le type Rectangle
<a name="proprietes_wlangage_utilisables_avec_type_rectangle_ELTPARAGRAPHE000051"></a>La valeur initiale d'un rectangle est (0,0,0,0). 

Les propriétés suivantes peuvent être utilisées pour manipuler une variable de type **Rectangle** : 

| Nom de la propriété | Effet |
| --- | --- |
| Hauteur | Hauteur du rectangle. Cette coordonnée correspond à un entier signé sur 4 octets. |
| Largeur | Largeur du rectangle. Cette coordonnée correspond à un entier signé sur 4 octets. |
| X | Coordonnée X du point haut gauche. Cette coordonnée correspond à un entier signé sur 4 octets. |
| Y | Coordonnée Y du point haut gauche. Cette coordonnée correspond à un entier signé sur 4 octets. |




<a name="NOTE0_2"></a>


### Affectation d'une variable de type Rectangle
<a name="affectation_une_variable_type_rectangle_ELTPARAGRAPHE000082"></a>Une variable de type Rectangle peut être affectée avec : 

- une autre variable de type Rectangle. 
	Par exemple : 
	
	```wl
	MonRectangle est un Rectangle
	MonRectangle.X = 280
	MonRectangle.Y = 54
	MonRectangle.Largeur = 50
	MonRectangle.Hauteur = 150
	MonRectangle1 est un Rectangle = MonRectangle
	```


- un tableau de quatre valeurs convertibles en entier. 
	Par exemple : 
	
	```wl
	MonRectangle est un Rectangle
	MonRectangle = [280, 54, 50, 150]
	```


- deux variables de type [Point](../Motscles/1000018908.md) (correspondant respectivement au coin haut gauche et au coin bas droit du rectangle). 
	Par exemple : 
	
	```wl
	MonRectangle est un Rectangle
	MonRectangle = [[280, 54], [330, 204]]
	```






<a name="NOTE0_3"></a>




### Affectation d'un tableau de Rectangles
<a name="affectation_tableau_rectangles_ELTPARAGRAPHE000110"></a>Un tableau de rectangles peut être affecté avec : 

- un tableau de valeurs contenant des rectangles. 
	Par exemple : 
	
	```wl
	TableauDeRectangles est un tableau de Rectangles
	TableauDeRectangles = [MonRectangle1, MonRectangle2]
	```


- un tableau de quatre valeurs convertibles en entier. 
	Par exemple : 
	
	```wl
	TableauDeRectangles est un tableau de Rectangles
	TableauDeRectangles = [[1,2,3,4], [5,6,7,8]
	```




<a name="NOTE0_4"></a>


### Comparaison et sérialisation
<a name="comparaison_serialisation_ELTPARAGRAPHE000128"></a>

- Comparaison : Les opérateurs de comparaison **=** et **&lt;&gt;** sont disponibles. 

- Sérialisation : Les sérialisations binaires, XML et JSON sont disponibles. 
	Attention : Lors d'une sérialisation XML, il n'est pas possible d'utiliser la constante*xmlAttribut* sur un membre de classe ou de structure de type Rectangle. 



<a name="NOTE0_5"></a>


### Notes
<a name="notes_ELTPARAGRAPHE000138"></a>

- Une variable de type Rectangle peut être stockée dans un variant. 

- Une variable de type Rectangle est jamais nulle : 

	- La condition "VariableRectangle = NULL" renvoie toujours <u><u><u><u>Faux</u></u></u></u>. 

	- La condition "VariableRectangle &lt;&gt; NULL" renvoie toujours <u><u><u><u>Vrai</u></u></u></u>.




- Il n'est pas possible de référencer une variable de type Rectangle. 
	La syntaxe "VariableRectangle2 &lt;- VariableRectangle1" provoque une erreur. 

- Il n'est pas possible de forcer la copie d'une variable de type Rectangle. 
	La syntaxe "VariableRectangle2 &lt;= VariableRectangle1" provoque une erreur.

- Il est possible d'additionner une variable de type [Point](../Motscles/1000018908.md) et une variable de type Rectangle. Cette opération permet de réaliser une translation du rectangle. 

- Il est possible de soustraire une variable de type [Point](../Motscles/1000018908.md) à une variable de type Rectangle. Cette opération permet de réaliser une translation du rectangle.   

- Les variables de type Rectangle peuvent être passées aux API du système.

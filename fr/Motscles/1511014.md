


## STOP  (Mot réservé)

***En anglais : Stop***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le mot-clé **STOP** permet d'appeler le débogueur du WLangage lors d'un test sous l'éditeur. Après exécution de ce mot-clé, le test en cours s'effectuera dans le débogueur.


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Ouverture d'une nouvelle fenêtre
Ouvre(FEN_FenêtreSaisie)
// Lancement du débogueur
STOP
```

<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Lancement du débogueur

`STOP`
---


<a name="SYNTAXE2"></a>

### Lancement du débogueur avec condition

`STOP SI <Condition>`
---

**`<Condition> : ()`**

Condition pour lancer le débogueur. Le débogueur sera lancé uniquement si la condition est vérifiée. 
Attention : cette condition est évaluée uniquement lors du débogage d'un projet. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Modes de lancement du débogueur
<a name="modes_lancement_debogueur_ELTPARAGRAPHE000061"></a>Pour lancer le débogueur lors d'un test, il est possible d'utiliser :

- Le mot-clé **STOP** dans l'éditeur de code.

- Un point d'arrêt ![](https://doc.pcsoft.fr/fr-FR/images/image.awp?langid=5&name=ico_PointArret.gif) dans l'éditeur de code : sous le volet "Code", dans le groupe "Points d'arrêt", déroulez "Point d'arrêt" et sélectionnez "Ajouter un point d'arrêt" (ou raccourci clavier Ctrl + B sous l'éditeur de code).

- La combinaison des touches Ctrl + Pause lors de l'exécution du test.



<a name="NOTE0_2"></a>


### Tracer / Déboguer un projet
<a name="tracer_deboguer_projet_ELTPARAGRAPHE000086"></a>Pour tracer un projet dès le début de son exécution : sous le volet "Projet", dans le groupe "Mode test", déroulez "Mode test" et sélectionnez "Déboguer le projet" (raccourci clavier Ctrl + F9).
<a name="NOTE0_3"></a>


### Exécutable
<a name="executable_ELTPARAGRAPHE000105"></a>Le mot-clé **STOP** présent dans le code d'un exécutable n'a aucune action.

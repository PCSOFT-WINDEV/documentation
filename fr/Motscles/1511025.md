
## Liste des attributs d'extension
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Les attributs d'extension sont des instructions WLangage particulières qui permettent d'ajouter des propriétés ou des fonctionnalités spécifiques à des variables ou à des procédures. Ces attributs d'extension permettent de simplifier le code et de faciliter la maintenance de vos applications.

WINDEV, WEBDEV et WINDEV Mobile proposent de nombreux attributs d'extension. Cette page présente les différents attributs d'extension disponibles regroupés par thème.

Les catégories des attributs d'extension sont les suivantes : 

- [Attributs d'extension divers](#NOTE2_1),

- [Attributs d'extension spécifiques à la POO et au MVP](#NOTE3_1),

- [Attributs d'extension spécifiques aux variables](#NOTE4_1),

- [Attributs d'extension spécifiques aux procédures locales, globales ou internes](#NOTE5_1).

- [Attributs d'extension de gestion des timers](#NOTE6_1). 

- [Attributs d'extension de gestion des threads](#NOTE7_1).




Pour simplifier la saisie des attributs d'extension, l'éditeur de code propose automatiquement la liste des attributs d'extension disponibles lors de la frappe de la touche "&lt;". Pour plus de détails, consultez [Complétion automatique](../Editeurs/2013001.md).

## Attributs d'extension spécifiques à la POO et au MVP
<a name="attributs_extension_specifiques_poo_mvp_ELTTEXTE000756"></a>
Les attributs d'extension spécifiques à la POO et au MVP sont les suivants : 



|   |   |
| --- | --- |
| **Sérialise** | Permet de gérer la sérialisation de chaque membre d'une classe ou d'une structure.  <br><br>Pour plus de détails, consultez : <br><br>- [Sérialise](../WDLang1/3013065.md).<br><br>- [Description de variable](../WDLang1/1000019560.md). <br><br><br> |
| **sans pas à pas** | Permet d'indiquer qu'une méthode sera ignorée par le débogueur lors d'un débogage en mode "Pas à pas". <br><br>Pour plus de détails, consultez l'aide sur comment déclarer des [Classes, membres, méthodes et propriétés](../POO/6010006.md) |
| **présentation** | Permet d'associer une classe de la couche présentation à une vue (fenêtre ou état). <br><br>Pour plus de détails, consultez [RAD MVP](../Editeurs/1000021508.md).<br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/fr/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) Non disponible |
| **mapping** | Permet de spécifier le nom du membre à "mapper" au membre de la structure ou de la classe. <br><br>Pour plus de détails, consultez : <br><br>- [RAD MVP](../Editeurs/1000021508.md). <br><br>- [Opérateurs d'affectation](../Motscles/1515003.md).<br><br>- [Description de variable](../WDLang1/1000019560.md). <br><br><br><br><br> |
| **associé** | Permet d'accéder aux membres, aux méthodes et aux propriétés d'une classe Modèle depuis sa classe Présentation sans avoir à effectuer de "rebonds".<br><br>Pour plus de détails, consultez [RAD MVP](../Editeurs/1000021508.md). |
| **copie** | Permet d'indiquer l'opération à effectuer lors d'une copie d'une instance d'un objet. <br><br>- &lt;Copie=Clone&gt; : utilisé sur un objet dynamique, cet attribut d'extension permet de forcer le clonage de l'objet. <br><br>- &lt;Copie=Faux&gt; : utilisé sur un membre, cet attribut d'extension permet d'empêcher la copie de la valeur du membre lors de la copie d'un objet dans un autre. <br><br><br><br><br>Pour plus de détails, consultez [Copie et clonage](../POO/6010018.md). |
| **force destructeur** | Permet de forcer l'appel du destructeur (libérer l'objet) même si une référence forte a été prise sur cet objet.<br><br>Pour plus de détails, consultez [Gestion avancée des instances de classes](../POO/6010012.md). |
| **faible** | Permet d'indiquer une prise de référence faible. Dans ce cas, les objets seront libérés automatiquement selon leur portée (même si une référence globale a été prise sur ces objets). La référence faible doit être indiquée lors de l'instanciation dynamique de l'objet prenant référence.<br><br>Pour plus de détails, consultez [Gestion avancée des instances de classes](../POO/6010012.md). |
| **note** | Permet d'indiquer un commentaire sur un membre de classe (&lt;note=Commentaire sur le membre&gt;). Le contenu de cette note pourra être connu par la propriété **AttributNote** d'une [Description de variable](../WDLang1/1000019560.md). <br>Rappel : Le type **Description de Variable** permet d'obtenir des informations sur un membre de classe ou de structure. <br><br>Pour plus de détails, consultez [Description de variable](../WDLang1/1000019560.md). |

## Attributs d'extension spécifiques aux variables
<a name="attributs_extension_specifiques_aux_variables_ELTTEXTE000780"></a>
Les attributs d'extension spécifiques aux variables sont les suivants : 



|   |   |
| --- | --- |
| **Contrôleur** | Permet dans le code d'un script de test d'indiquer qu'une variable est de type contrôleur. Cette variable de type contrôleur peut alors être utilisée dans le code du test par exemple pour vérifier le résultat d'une procédure. <br><br>Pour plus de détails, consultez [Gestionnaire de tests automatiques](../Editeurs/2019033.md). |
| **indétectable** | Permet d'obfusquer automatiquement le contenu de la chaîne en mémoire. <br><br>Pour plus de détails, consultez [Chaînes de caractères](../Motscles/1514044.md). |
| **section critique** | Permet d'associer une section critique à une variable. <br><br>Pour plus de détails, consultez [Gérer les sections critiques](../WDLang1/1000021292.md). |
| **synchronisé navigateur** | Permet de synchroniser une variable globale au projet afin de la rendre disponible coté navigateur. Dans ce cas la valeur affectée à une variable globale dans un code navigateur, peut être récupérée par la suite dans un code serveur. <br><br>Pour plus de détails, consultez [Variables globales et locales](../Motscles/1514054.md). |

| **zombie** | Permet d'indiquer qu'une variable ou une procédure est obsolète et ne doit plus être utilisée. <br><br>Pour plus de détails, consultez l'aide sur comment déclarer une [Variable globale et locale](../Motscles/1514054.md). |

## Remarque
<a name="remarque_ELTTEXTE000876"></a>
Selon le type de déclaration et d'objet utilisé, il est possible de combiner plusieurs attributs d'extension à l'aide de la syntaxe: 
```txt
Déclaration de mon objet <Nom Attribut 1=valeur1, Nom Attribut 2=valeur2, ...>
```

Par exemple, dans le cas d'une déclaration d'une variable de type buffer: 
```txt
Classe
	m_buffer est un Buffer <sérialise=faux,mapping="MémoImage">
FIN
```


## Monétaires
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Le type monétaire
<a name="type_monetaire_ELTTEXTE000181"></a>
Un **monétaire** est un réel codé sur 10 octets.

Un **monétaire** peut avoir jusqu'à 23 chiffres significatifs (17 maximum pour la partie entière, 6 maximum pour la partie décimale) et peut prendre une valeur entre -604 462 909 807 314 587,353 087 et +604 462 909 807 314 587,353 087.

Le type **monétaire** est conseillé pour éviter les erreurs d'arrondi dues au codage binaire des réels.

**Remarques** :

- Pour une plus grande précision, il est recommandé d'utiliser les [numériques](../Motscles/1514072.md) (38 chiffres significatifs).

- Pour les autres calculs, il est préférable d'utiliser des [réels](../Motscles/1514048.md), plus rapides.

## Notes
<a name="notes_ELTTEXTE000205"></a>


### Valeur par défaut
<a name="valeur_par_defaut_ELTPARAGRAPHE000071"></a>Une variable de type "Monétaire" déclarée et non initialisée est égale à 0.

**Pour forcer l'affectation d'un monétaire**, utilisez **0m** devant la valeur. Par exemple :


```wl
num1 est un monétaire
num1 = 0m12345678901234567891234567
Trace(num1)
```

<a name="NOTE2_2"></a>


### Codage binaire des réels
<a name="codage_binaire_des_reels_ELTPARAGRAPHE000082"></a>Les [réels](../Motscles/1514048.md) ont tous le même nombre de chiffres significatifs (15).

Si le résultat d'un calcul sur des [réels](../Motscles/1514048.md) met en jeu plus de 15 chiffres significatifs, ce résultat sera automatiquement arrondi à 15 chiffres significatifs. La précision du résultat sera donc plus faible que pour un calcul réalisé avec des **monétaires**.

Les [numériques](../Motscles/1514072.md) permettent d'obtenir une précision encore plus grande (32 chiffres pour la partie entière et 6 chiffres pour la partie décimale).






<a name="NOTE0_9"></a>


### Séparateur des milliers
<a name="separateur_des_milliers_ELTPARAGRAPHE000106"></a>Il est possible d'utiliser le caractère 'Espace' ou le caractère '_' comme séparateur de milliers dans les nombres. Par exemple : 

```wl
num1 est un monétaire
num1 = 123 456.478

num2 est un monétaire
num2 = 568_014.478
```





<a name="NOTE2_3"></a>


### Calculs sur les monétaires
<a name="calculs_sur_les_monetaires_ELTPARAGRAPHE000116"></a>Tous les calculs utilisant au moins un **monétaire** seront effectués sous forme **monétaire**. Pour garantir la précision du résultat, toutes les opérandes seront automatiquement converties en **monétaire**.

Par exemple, lorsque un réel est multiplié par un monétaire, le calcul est effectué sur des monétaires.

Remarque : Pour obtenir une précision plus importante, il est recommandé d'utiliser le [type Numérique](../Motscles/1514072.md) pour effectuer des calculs.


<a name="NOTE2_4"></a>


### Langage externe
<a name="langage_externe_ELTPARAGRAPHE000131"></a>Le type **monétaire** n'est pas disponible en langage externe.

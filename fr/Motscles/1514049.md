**Remarque** : Tous les types avancés (différents du type "Entier") sont conseillés pour l'utilisation des API Windows.

## Notes
<a name="notes_ELTTEXTE000384"></a>


### Valeur par défaut
<a name="valeur_par_defaut_ELTPARAGRAPHE000228"></a>Une variable de type "Entier" déclarée et non initialisée est égale à 0.
<a name="NOTE2_2"></a>


### Valeur Hexadécimale
<a name="valeur_hexadecimale_ELTPARAGRAPHE000235"></a>Pour saisir une valeur hexadécimale dans le code, il suffit de préfixer la valeur hexadécimale par "0x".

```wl
MaVariable est un entier
MaVariable = 0x4D5
```


Remarque : Il est possible d'utiliser le caractère '_' comme séparateur de blocs dans les nombres hexadécimaux. Les blocs peuvent avoir une taille de 2, 4 ou 8 caractères de long. 
Exemple : 

```wl
Nb1 est un entier = 0x_12_15_A4_BC
Nb2 est un entier = 0x_CD4A_DE58
```

<a name="NOTE0_9"></a>


### Séparateur des milliers
<a name="separateur_des_milliers_ELTPARAGRAPHE000253"></a>Il est possible d'utiliser le caractère 'Espace' ou le caractère '_' comme séparateur de milliers dans les nombres. Par exemple :

```wl
num1 est un entier
num1 = 123 456
soit num2 = 246_789
```

### Différences entre WINDEV 5.5 et WINDEV 28
<a name="differences_entre_windev_55_windev_docparamversion_ELTPARAGRAPHE000283"></a>

Les types d'entiers ont été modifiés entre WINDEV 5.5 et WINDEV 28 :

| Type d'entiers avec WINDEV 5.5 | Type d'entiers avec WINDEV 28 |
| --- | --- |
| Entier | Entier sur 2 octets |
| Entier sans signe | Entier sans signe sur 2 octets |
| Entier long | Entier sur 4 octets |
| Entier court | Entier sans signe sur 1 octet |





Par défaut, avec WINDEV 5.5, les entiers étaient des entiers sur 2 octets. Par défaut, depuis les versions 7.x, les entiers sont des entiers sur 4 octets.

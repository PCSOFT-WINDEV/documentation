
## Point (Type de variable)

***En anglais : Point***
				

<a name="description"></a>
Le type **Point** est un type structuré qui permet de représenter une position (X, Y) en 2 dimensions. 


<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
MonPoint est un Point
MonPoint.X = 280
MonPoint.Y = 54
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

### Déclarer et initialiser un point

`<Nom du point> est [un] Point
`
---

**`<Nom du point> : ()`**

Nom de la variable Point à déclarer.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


### Propriétés WLangage utilisables avec le type Point
<a name="proprietes_wlangage_utilisables_avec_type_point_ELTPARAGRAPHE000042"></a>La valeur initiale d'un point est (0,0). 

Les propriétés suivantes peuvent être utilisées pour manipuler une variable de type **Point** : 


| Nom de la propriété | Effet |
| --- | --- |
| X | Coordonnée X du point. Cette coordonnée correspond à un entier signé sur 4 octets. |
| Y | Coordonnée Y du point. Cette coordonnée correspond à un entier signé sur 4 octets. |




<a name="NOTE0_2"></a>


### Affectation d'une variable de type Point
<a name="affectation_une_variable_type_point_ELTPARAGRAPHE000066"></a>Une variable de type **Point** peut être affectée avec : 

- un autre point. 
	Par exemple : 
	
	```wl
	MonPoint est un Point
	MonPoint.X = 280
	MonPoint.Y = 54
	MonPoint1 est un Point = MonPoint
	```


- un tableau de deux valeurs convertibles en entier. 
	Par exemple : 
	
	```wl
	MonPoint est un Point
	MonPoint = [280, 54]
	```




<a name="NOTE0_3"></a>




### Affectation d'un tableau de Points
<a name="affectation_tableau_points_ELTPARAGRAPHE000085"></a>Un tableau de Points peut être affecté avec : 

- un tableau de valeurs contenant des points. 
	Par exemple : 
	
	```wl
	TableauDePoints est un tableau de Points
	TableauDePoints = [MonPoint1, MonPoint2]
	```


- un tableau de deux valeurs convertibles en entier. 
	Par exemple : 
	
	```wl
	TableauDePoints est un tableau de Points
	TableauDePoints = [[1,2],[3,4]]
	```




<a name="NOTE0_4"></a>


### Comparaison et sérialisation
<a name="comparaison_serialisation_ELTPARAGRAPHE000103"></a>

- Comparaison : Les opérateurs de comparaison **=** et **&lt;&gt;** sont disponibles. 

- Sérialisation : Les sérialisations binaires, XML et JSON sont disponibles. 
	Attention : Lors d'une sérialisation XML, il n'est pas possible d'utiliser l'option *xmlAttribut* sur un membre de classe ou de structure de type **Point**. 



<a name="NOTE0_5"></a>


### Notes
<a name="notes_ELTPARAGRAPHE000113"></a>

- Une variable de type **Point** peut être stockée dans un variant. 

- Une variable de type **Point** est jamais nulle : 

	- La condition "VariablePoint = NULL" renvoie toujours <u><u><u><u>Faux</u></u></u></u>. 

	- La condition "VariablePoint &lt;&gt; NULL" renvoie toujours <u><u><u><u>Vrai</u></u></u></u>.




- Il n'est pas possible de référencer une variable de type **Point**. 
	La syntaxe "VariablePoint2 &lt;- VariablePoint1" provoque une erreur. 

- Il n'est pas possible de forcer la copie d'une variable de type **Point**. 
	La syntaxe "VariablePoint2 &lt;= VariablePoint1" provoque une erreur.

- Il est possible d'additionner et de soustraire deux variables de type **Point**.   

- Les variables de type **Point** peuvent être passées aux API du système. 

- Les variables de type **Point** permettent de définir les points des variables de type [Polygone](../WDLang1/1000025624.md).

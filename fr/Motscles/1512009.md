## Opérateurs binaires
<a name="operateurs_binaires_ELTTEXTE000424"></a>


### ET, OU et OU exclusif binaires
<a name="exclusif_binaires_ELTPARAGRAPHE000058"></a>Les syntaxes suivantes peuvent être utilisée :

- ET binaire : &lt;Valeur 1&gt;** &**&lt;Valeur 2&gt;

- OU binaire : &lt;Valeur 1&gt;** |** &lt;Valeur 2&gt;

- OU exclusif binaire : &lt;Valeur 1&gt; **||**&lt;Valeur 2&gt;




Le type de résultat peut être différent selon le type des opérandes :


|   |   |   |   |
| --- | --- | --- | --- |
| Valeur 2<br>Valeur 1 | **Entier sur 4** | **Entier sur 8** | **Autre** |
| **Entier sur 4** | Entier sur 4 | Entier sur 8 | Entier sur 4 |
| **Entier sur 8** | Entier sur 8 | Entier sur 8 | Entier sur 8 |
| **Autre** | Entier sur 4 | Entier sur 8 | Entier sur 8 |




<a name="NOTE2_2"></a>


### NON binaire
<a name="non_binaire_ELTPARAGRAPHE000115"></a>La syntaxe est la suivante : ** ~**&lt;Valeur&gt;

Le type de résultat peut être différent selon le type de l'opérande :


|   |   |
| --- | --- |
| **Opérande** | **Résultat** |
| Entier sur 4 | Entier sur 4 |
| Entier sur 8 | Entier sur 8 |
| Autre | Entier sur 8 |

## Opérateurs de décalage
<a name="operateurs_decalage_ELTTEXTE000454"></a>


### Syntaxes
<a name="syntaxes_ELTPARAGRAPHE000144"></a>

- Décalage vers la gauche : 
	
	```txt
	<Valeur 1> bitDécaleGauche <Valeur 2>
	bitDécaleGauche(<Valeur1>, <Valeur 2>)
	```


- Décalage vers la droite : 
	
	```txt
	<Valeur 1> bitDécaleDroite <Valeur 2>
	bitDécaleDroite(<Valeur1>, <Valeur 2>)
	```




<a name="NOTE3_2"></a>


### Notes
<a name="notes_ELTPARAGRAPHE000156"></a>

- Les bits de &lt;Valeur 1&gt; sont décalés de &lt;Valeur 2&gt; bits vers la droite ou vers la gauche. 
	Par exemple : 
	
	```wl
	bitDécaleGauche(4,1) // Renvoie 8
	```

	En effet, 4 en décimal correspond à 0100 en binaire. Décalé de 1 bit vers la gauche, on obtient en binaire 1000 ce qui correspond à 8 en décimal.
	
	```wl
	bitDécaleDroite(4,2) // Renvoie 1
	```

	En effet, 4 en décimal correspond à 0100 en binaire. Décalé de 2 bits vers la droite, on obtient en binaire 0001 ce qui correspond à 1 en décimal.

- Les bits qui dépassent la taille de &lt;Valeur 1&gt; sont ignorés. Par exemple : 
	
	```wl
	bitDécaleGauche(4,30) // Renvoie 0
	bitDécaleDroite(4,4) // Renvoie 0
	```


- Si &lt;Valeur 2&gt; est supérieur à la taille de &lt;Valeur 1&gt; (32 pour un entier sur 4 et 64 pour un entier sur 8), le résultat est toujours 0. Par exemple :
	
	```wl
	bitDécaleGauche(4,35) // Renvoie 0
	```


- Le type de résultat peut être différent selon le type de l'opérande : 
	


|   |   |
| --- | --- |
| **Opérande Valeur 1** | **Résultat** |
| Entier sur 4 | Entier sur 4 |
| Entier sur 8 | Entier sur 8 |
| Autre | Entier sur 8 |


	![Java](https://doc.pcsoft.fr/ext/images/fr/JAVA.png) L'opérateur de décalage binaire à gauche réinjectera par la droite les bits qui dépassent la taille de l'opérande. L'opérateur de décalage binaire à droite réinjectera par la gauche les bits qui dépassent la taille de l'opérande. Par exemple :
	
	```wl
	e est un entier sur 4 
	res est un entier sur 4
	e = 1
	res = e bitDécaleGauche 32 // Renvoie 0 en WinDev et 1 en Java
	```

## Opérateur d'accès direct aux bits
<a name="operateur_acces_direct_aux_bits_ELTTEXTE000484"></a>


### Accès à un bit
<a name="acces_bit_ELTPARAGRAPHE000207"></a>

**Syntaxe** : &lt;Valeur 1&gt;** [** &lt;n&gt;** ]**

Cette syntaxe permet de lire ou de modifier la valeur du bit &lt;N&gt; dans la valeur &lt;Valeur 1&gt;.

Le comptage des bits commence à 1 de la gauche vers la droite (du poids le plus faible au poids le plus fort) :

- 1 à 32 pour un entier sur 4 octets,

- 1 à 64 pour un entier sur 8 octets.




Si la valeur de &lt;n&gt; est incorrecte, l'opération renvoie 0.

Exemples :


```wl
// Positionne les cinquième et septième bits à 1
n est un entier
n[5] = 1
n[7] = Vrai
// Teste la valeur des bits 4, 5, 6 et 7
SI n[4] ALORS Trace(4) // pas affiché
SI n[5] ALORS Trace(5) // affiche 5
SI n[6] ALORS Trace(6) // pas affiché
SI n[7] ALORS Trace(7) // affiche 7
```



```wl
// Affecter une valeur en modifiant les bits
n est un entier = 0
// On allume le bit numéro 1 : n vaut 1 en décimal
n[1] = 1 
// On allume également le bit numéro 8 : n vaut 129 en décimal (2 puissance 0 + 2 puissance 7)
n[8] = 1
```



<a name="NOTE4_2"></a>


### Accès à un entier sur 1, 2 ou 4 octets
<a name="acces_entier_sur_1_2_4_octets_ELTPARAGRAPHE000243"></a>

**Syntaxes** :

- &lt;Valeur 1&gt;** [** &lt;n&gt;, wlEntier_1** ]**

- &lt;Valeur 1&gt;** [** &lt;n&gt;, wlEntier_2** ]**

- &lt;Valeur 1&gt;** [** &lt;n&gt;, wlEntier_4** ]**




Ces syntaxes permettent de lire ou de modifier la valeur de l'entier sur 1, 2 ou 4 octets dans la valeur &lt;Valeur 1&gt;.

Valeurs possibles pour &lt;n&gt; selon le type de &lt;Valeur 1&gt; :


|   |   |   |   |
| --- | --- | --- | --- |
| **Valeur 1** | **wlEntier_1** | **wlEntier_2** | **wlEntier_4** |
| Entier sur 4 | 1 à 4 | 1 à 2 | 1 |
| Entier sur 8 | 1 à 8 | 1 à 4 | 1 à 2 |


Si la valeur de &lt;n&gt; est incorrecte, l'opération renvoie 0.


<a name="NOTE4_3"></a>


### Accès à la valeur de plusieurs bits
<a name="acces_valeur_plusieurs_bits_ELTPARAGRAPHE000312"></a>

**Syntaxes** :

- &lt;Valeur 1&gt;** [ A** &lt;n&gt;** ]**

- &lt;Valeur 1&gt;** [** &lt;n&gt; **A ]**

- &lt;Valeur 1&gt;** [** &lt;n&gt; **A**&lt;o&gt; **]**

- &lt;Valeur 1&gt;** [** &lt;n&gt; SUR &lt;Nombre&gt;** ]**




Ces syntaxes permettent de lire ou de modifier la valeur correspondant aux bits indiqués.


## MaFenêtre (Mot réservé)

***En anglais : MyWindow***
				



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Le mot-clé **MaFenêtre** permet de manipuler la fenêtre courante. Lors de l'exécution de la fenêtre, le mot-clé **MaFenêtre** est remplacé par la fenêtre et non par le nom de la fenêtre. Le mot-clé **MaFenêtre** s'utilise alors comme une fenêtre.

**Intérêt** : Le mot-clé **MaFenêtre** permet de rendre un code local (champ, bouton, ...) ou global (procédure globale, classe, ...) indépendant de la fenêtre en cours.




<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```wl
// Récupération du titre de la fenêtre en cours
TitreFenêtre = MaFenêtre.Titre
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`MaFenêtre`

<a name="NOTE0_1"></a>

## Remarques




### Manipulation de la fenêtre en cours
<a name="manipulation_fenetre_cours_ELTPARAGRAPHE000058"></a>

- Le mot-clé **MaFenêtre** est toujours remplacé par la fenêtre en cours. Par exemple, utilisation de la propriété [Titre](../Proprietes/2510090.md) :
	
	```wl
	MaFenêtre..Titre = "Saisissez vos données"
	MonTitreEnCours = MaFenêtre..Titre
	 
	// Passage en paramètre :
	AppelProcedure(MaFenêtre)
	```


- Si le mot-clé **MaFenêtre** doit être converti en chaîne, c'est le résultat de la propriété [Valeur](../Proprietes/2510130.md) qui est renvoyé. Pour une fenêtre, la propriété [Valeur](../Proprietes/2510130.md) renvoie le titre de la fenêtre.
	
	```wl
	Trace(MaFenêtre) // Renvoie le titre de la fenêtre
	```

	Exemple à ne pas faire :
	
	```wl
	ExécuteTraitement(MaFenêtre, trtModification)
	```

	Il faut faire :
	
	```wl
	ExécuteTraitement(MaFenêtre..Nom, trtModification)
	```


- Le mot-clé **MaFenêtre** peut être utilisé uniquement dans les traitements manipulant la fenêtre en cours (par exemple, dans les traitements associés à un champ, à une fenêtre, ...). Il n'est pas possible d'utiliser le mot-clé **MaFenêtre** dans un état. Pour désigner l'état en cours dans un état, utilisez le mot-clé [MonEtat](../Motscles/1511010.md)

- Le mot-clé **MaFenêtre** ne permet pas d'utiliser les variables globales déclarées dans la fenêtre.

- Pour manipuler un champ d'une fenêtre en utilisant le mot-clé **MaFenêtre**, il est possible d'utiliser la syntaxe suivante :
	
	```wl
	Info({MaFenêtre..Nom + ".Bouton2"}..Libellé)
	```




<a name="NOTE0_2"></a>




### Utilisation de MaFenêtre dans une procédure
<a name="utilisation_mafenetre_dans_une_procedure_ELTPARAGRAPHE000095"></a>Le mot-clé **MaFenêtre** peut être utilisé dans une procédure (locale ou globale) uniquement si la procédure manipule la fenêtre en cours. Le mot-clé **MaFenêtre** fait alors référence à la fenêtre en cours.

Dans le cas d'une procédure locale, le mot-clé **MaFenêtre** correspond à la fenêtre à laquelle appartient la procédure.


<a name="NOTE0_3"></a>




### Fonctions du WLangage et fenêtre en cours
<a name="fonctions_wlangage_fenetre_cours_ELTPARAGRAPHE000106"></a>Pour désigner la fenêtre en cours dans les fonctions du WLangage acceptant un nom de fenêtre en paramètre, utilisez :

- soit une chaîne vide,

- soit directement le mot-clé **MaFenêtre**.

Le mot-clé **MaFenêtre** peut être utilisé uniquement depuis un traitement ou une procédure de la fenêtre.

Limitation : le mot-clé **MaFenêtre** n'est pas utilisable dans un thread secondaire. Pour plus de détails sur la manipulation des fenêtres dans les threads, consultez [Gérer l'ouverture d'une fenêtre dans un thread secondaire](../WDLang1/3077015.md).

## Installer les fichiers d'un site statique sur un serveur Web
<a name="installer_les_fichiers_site_statique_sur_serveur_web_ELTTEXTE000221"></a>
Pour installer les fichiers d'un site statique sur un serveur Web :

1. Lancez WDDéploie : sous le volet "Outils", dans le groupe "Utilitaires Web", cliquez sur "WDDeploie".

2. Créez ou sélectionnez le profil correspondant à votre site. Les caractéristiques sont saisies dans différents onglets :

	- **Informations générales** :

		- nom du profil. 
						Lors de la création d'un profil, un nom par défaut est automatiquement donné. Il est possible de modifier ce nom.

		- emplacement des fichiers locaux du site (sous-répertoire "&lt;MonProjet&gt;_WEB" pour un site statique WEBDEV).

		- adresse du site WEBDEV.

		- déploiement d'une seule langue pour le site. Si cette option est sélectionnée, vous pouvez déployer votre site uniquement dans une seule des langues gérées par le projet. Attention : dans ce cas, votre site ne doit pas utiliser d'options ou de fonctions permettant de changer de langue ou de gérer plusieurs langues.




- **Connexion**

	- emplacement des fichiers du site déployé (répertoire réseau ou serveur FTP).

	- caractéristiques du serveur FTP utilisé pour la mise à jour du site (uniquement lors d'un déploiement par FTP). 
						Ces caractéristiques sont fournis par l'hébergeur du site. 
						Remarque : Pour utiliser un serveur SFTP, saisissez une adresse du type "sftp://&lt;Adresse du serveur&gt;".

3. Cliquez sur le bouton "Préparer". WDDéploie prépare la liste des fichiers à installer. 
	En cas de mise à jour, WDDéploie compare les fichiers présents sur le poste de développement et les fichiers déjà installés.

4. Spécifiez si nécessaire les différentes actions à effectuer lors de la synchronisation sur les différents fichiers à installer :

	- bouton "Copier en local" : copie les fichiers sélectionnés du site distant vers le site local.

	- bouton "Copier à distance" : copie les fichiers sélectionnés du site local vers le site distant.

	- bouton "Effacer à distance" : supprime les fichiers sélectionnés du site distant.

	- bouton "Ne rien faire" : n'effectue aucune opération sur les fichiers sélectionnés.




5. Cliquez sur le bouton "Synchroniser". Les fichiers du site sont automatiquement copiés à l'emplacement spécifié dans le profil.




Remarques : 

- Le bouton "Options" permet de spécifier des options de synchronisation des fichiers. Pour plus de détails, consultez les Notes.

- Lors de la création d'un nouveau profil, un assistant vous guide dans la saisie des paramètres et propose des profils pré-configurés pour les principaux hébergeurs.

## Compresser et regrouper les fichiers d'un site distant sous forme d'archive
<a name="compresser_regrouper_les_fichiers_site_distant_sous_forme_archive_ELTTEXTE000245"></a>
Pour compresser et regrouper les fichiers d'un site distant sous forme d'archive :

1. Lancez WDDéploie : sous le volet "Outils", dans le groupe "Utilitaires Web", cliquez sur "WDDeploie".

2. Cliquez sur le bouton "Sauvegarder".

3. Saisissez le nom de l'archive à créer (fichier ".WDZ") et spécifiez son répertoire de création. Les fichiers présents dans le site distant spécifié dans le profil sont automatiquement compressés et regroupés dans l'archive créé.

## Restaurer les fichiers contenus dans une archive
<a name="restaurer_les_fichiers_contenus_dans_une_archive_ELTTEXTE000269"></a>
Pour restaurer les fichiers contenus dans une archive :

1. Lancez WDDéploie : sous le volet "Outils", dans le groupe "Utilitaires Web", cliquez sur "WDDeploie".

2. Cliquez sur le bouton "Restaurer".

3. Sélectionnez le fichier archive (fichier ".WDV") à restaurer.

4. Sélectionnez le répertoire dans lequel les fichiers contenus dans l'archive doivent être restaurés et validez.

## Notes
<a name="notes_ELTTEXTE000293"></a>
Les options de WDDéploie sont les suivantes : 

- "**Afficher la bulle d'aide sur la table des fichiers**" : affiche une bulle d'aide décrivant pour chaque fichier survolé :

	- le nom et la date et l'heure de création du fichier sur le poste de développement.

	- le nom et la date et l'heure de création du fichier sur le poste de déploiement.




- "**Ne pas traiter les sous-répertoires**" : ne prend pas en compte les sous-répertoires du site local lors du déploiement des fichiers du site.

- "**Ne pas proposer de modifier le site local, même si les fichiers distants sont plus récents**" : ne met pas à jour les fichiers distants plus récents sur le poste de développement. Lors de la préparation de la synchronisation, WDDéploie ne proposera pas de remplacer les fichiers locaux par les fichiers distants plus récents.

- "**Ne pas afficher les fichiers qui sont déjà synchronisés**" : affiche uniquement les fichiers modifiés depuis la dernière synchronisation.

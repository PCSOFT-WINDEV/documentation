
## Contrôle : Connectivité HTTP
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000102"></a>
Le robot de surveillance va lire l'entête et le contenu d'une page Web.

Le contrôle sera :

- réussi si la page existe et si son entête ne contient aucune erreur.

- en échec dans le cas contraire.






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Paramètres spécifiques au contrôle de la connectivité HTTP
<a name="parametres_specifiques_controle_connectivite_http_ELTTEXTE000126"></a>


### Informations sur la page Web
<a name="informations_sur_page_web_ELTPARAGRAPHE000024"></a>Indiquez : 

- l'adresse de la page Web à tester. 

- le port de connexion utilisé.

- la méthode HTTP à utiliser pour vérifier que le serveur répond : GET, POST, PUT, DELETE, HEAD ou PATCH. 
	Ce paramètre permet notamment de tester certains Webservices.





<a name="NOTE2_2"></a>


### Informations sur le contrôle à effectuer
<a name="informations_sur_controle_effectuer_ELTPARAGRAPHE000040"></a>Il est possible de contrôler la validité du contenu de la page. Dans ce cas, il est également possible de vérifier que le contenu est identique d'un contrôle à l'autre.


<a name="NOTE2_3"></a>


### Gestion des cas d'erreurs personnalisés
<a name="gestion_des_cas_erreurs_personnalises_ELTPARAGRAPHE000048"></a>Il est possible de définir des cas d'erreurs spécifiques en ajoutant des conditions sur l'en-tête et/ou sur le contenu de la page. Le bouton "+" permet d'ajouter des conditions. Ces conditions permettent de définir : 

- la condition d'erreur. 

- le contenu du message d'alerte si la condition est vérifiée.

## Options d'activation du contrôle
<a name="options_activation_controle_ELTTEXTE000252"></a>
Les options d'activation du contrôle permettent de savoir et de définir : 

- si le contrôle est activé ou non, 

- à quelle date le contrôle doit être réactivé automatiquement. 




Remarque : Le menu contextuel du contrôle permet de désactiver simplement un contrôle (option "Désactiver le contrôle"). Cette option permet de désactiver le contrôle : 

- pendant 1 jour, 

- pendant 1 semaine, 

- pendant 1 mois, 

- pendant une durée personnalisée. Dans ce cas, une fenêtre permet de spécifier la date de réactivation du contrôle. 

- indéfiniment.

## Modèle et options de planification
<a name="modele_options_planification_ELTTEXTE000276"></a>
Les options de planification permettent de définir quand le contrôle va être exécuté. 

Ces options de planifications sont définies dans un [modèle de contrôle](../RobotMonitor/3541124.md). Les modèles de contrôle permettent de créer rapidement plusieurs contrôles ayant les mêmes options de planification.

### Contenu du message
<a name="contenu_message_ELTPARAGRAPHE000076"></a>Les options de traitement des erreurs permettent de définir : 

- le message personnalisé pouvant apparaître dans le rapport d'erreurs.

- la solution conseillé pour solutionner le problème.

- le destinataire des rapports d'erreur avec le niveau de gravité pour ce destinataire. 




Exemple de message :

| --- |
| 2 Alerte(s) Réseau \ Priorité maximale : Normale<br>[----------------------------------------------------------------------]<br>Priorité : Normale<br>Catégorie : FTP<br>Contrôle : FTP Etats Et Requetes (xxxx) (n° 38)<br>Erreur :<br>Erreur lors de la connexion au serveur FTP<br>Echec de la connexion au serveur FTP "ftp.pcsoft.com" sur le port 21 avec le nom d'utilisateur.<br>La dernière réponse du serveur est :<br>Solution :<br><br>Détails interne :<br>- Syntaxe : <br>Adresse du serveur FTP : ftp.pcsoft.com<br>Port de connexion : 21<br>Connexion en mode passif : Vrai<br>Fichier à chercher : /wx11/latest/WD280PACKER.exe<br>Nombre d'utilisateurs max pour check : 0<br>Télécharger le fichier : Faux<br> |

### Destinataires des rapports d'erreur
<a name="destinataires_des_rapports_erreur_ELTPARAGRAPHE000138"></a>Pour chaque contrôle (test), il est possible de définir les destinataires des rapports d'erreur, avec le niveau de gravité de l'erreur pour ce destinataire. Ainsi un contrôle en erreur peut être important pour une personne et mineur pour une autre. 
Ces destinataires peuvent correspondre à : 

- un utilisateur. 

- un groupe de destinataires. Dans ce cas, tous les utilisateurs du groupe recevront le rapport d'erreur.




Pour ajouter un destinataire pour un contrôle : 

1. Appuyez sur "+". 

2. Dans la fenêtre qui s'affiche, sélectionnez le groupe ou l'utilisateur et validez. 

3. L'utilisateur ou le groupe sélectionné apparaît dans la liste des destinataires. Par défaut, la gravité de l'utilisateur est "Erreur". 

4. Modifiez si nécessaire le niveau de gravité de l'utilisateur. Les niveaux de gravité sont : 

	- Débogage, 

	- Information,

	- Notification, 

	- Avertissement,

	- Erreur, 

	- Erreur critique. 







**Rappel** : 

- Les caractéristiques des utilisateurs sont définies dans la fenêtre de gestion des utilisateurs (pour plus de détails, consultez [Configuration des utilisateurs](../RobotMonitor/3541104.md)). 

- Les groupes et la liste des utilisateurs associés sont définis dans la fenêtre de gestion des groupes (pour plus de détails, consultez [Configuration des groupes d'utilisateurs](../RobotMonitor/3541126.md)).

### Programme à lancer en cas d'erreur
<a name="programme_lancer_cas_erreur_ELTPARAGRAPHE000179"></a>Pour chaque test, il est possible d'indiquer un programme spécifique à lancer en cas d'erreur. Ce programme doit être accessible depuis le poste d'installation du robot. Il est possible d'indiquer une ligne de commande.

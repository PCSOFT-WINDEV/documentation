
## Contrôle : Log d'un site WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000079"></a>
Le robot de surveillance va récupérer le log du jour d'une application WEBDEV sur le serveur FTP associé. Le contenu du log est analysé pour trouver d'éventuelles erreurs.

Le contrôle sera :

- réussi si le log est récupéré et ne comporte aucune erreur.

- en échec dans le cas contraire.






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Paramètres spécifiques au contrôle du log d'un site WEBDEV
<a name="parametres_specifiques_controle_log_site_webdev_ELTTEXTE000103"></a>


### Informations sur le serveur WEBDEV
<a name="informations_sur_serveur_webdev_ELTPARAGRAPHE000024"></a>Les informations nécessaires pour vérifier les logs d'un serveur WEBDEV sont les suivantes :

- Adresse du serveur.

- Port de connexion (avec si nécessaire une connexion en mode passif).

- L'utilisateur et le mot de passe associé.

- Le répertoire sur le serveur contenant les logs du site voulu.

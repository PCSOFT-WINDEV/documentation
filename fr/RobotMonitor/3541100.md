
## Présentation du robot de surveillance
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Présentation
<a name="presentation_ELTTEXTE000168"></a>
Un robot de surveillance est livré avec WINDEV, WEBDEV et WINDEV Mobile. Ce robot permet de vérifier qu'une application ou un serveur fonctionne. En cas de défaillance de l'élément surveillé (panne matérielle, arrêt de la liaison Internet, arrêt du système, ...), le robot a pour mission de lancer les alertes qui ont été définies.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principe
<a name="principe_ELTTEXTE000192"></a>
Composé de trois exécutables lancés sur différents postes, le robot de surveillance permet d'exécuter différents tests : test Internet, test serveur FTP, test réseau, ...

Le robot de surveillance est un ensemble logiciel composé de : 

- un moniteur permettant de configurer et visualiser l'état du robot et des différents contrôles. 

- un robot de surveillance réalisant de manière périodique des contrôles sur l'infrastructure ou les applications. 

- éventuellement un contre-robot qui vérifie le fonctionnement du robot lui-même.  




En cas de problèmes lors du passage d'un test, le robot de surveillance peut vous avertir de différentes façons :

- Message envoyé aux personnes indiquées dans la messagerie intégrée (WDBal).

- Message email envoyé aux adresses indiquées.

- Message envoyé à une application spécifique.

- Utilisation d'un écran de contrôle (message visuel et/ou sonore).

- Lancement d'une procédure WLangage.

- Exécution d'un programme tiers.

## Comment installer le robot de surveillance ?
<a name="comment_installer_robot_surveillance_ELTTEXTE000216"></a>
Pour utiliser le robot de surveillance, il est nécessaire d'installer les différents programmes du robot de surveillance. Le pack d'installation (WX280PACKROBOT.EXE) est disponible dans le sous-répertoire "Install\\Robot" du produit utilisé.

L'installation des différents modules doit être faite selon l'ordre suivant : 

1. [Installation du robot de surveillance](../RobotMonitor/3541101.md). Le robot de surveillance doit être installé sur un poste qui ne sera jamais éteint.

2. [Installation si nécessaire du contre-robot](../RobotMonitor/3541102.md). Le contre-robot est une sécurité supplémentaire. Il permet de surveiller le bon fonctionnement du robot de surveillance.

3. [Installation du moniteur](../RobotMonitor/3541103.md). Le moniteur peut être installé sur plusieurs postes. Le moniteur permet de configurer les différents tests effectués par le robot de surveillance.




**Remarque** : Les trois programmes doivent être installés sur le même réseau local ou sur le même domaine.

## Comment utiliser le robot de surveillance ?
<a name="comment_utiliser_robot_surveillance_ELTTEXTE000240"></a>


### Utilisation
<a name="utilisation_ELTPARAGRAPHE000067"></a>Pour utiliser le robot de surveillance, le robot doit être lancé (et le contre-robot si nécessaire).  

Le moniteur peut être lancé pour gérer les tests à effectuer.


<a name="NOTE4_2"></a>


### Utilisation du moniteur
<a name="utilisation_moniteur_ELTPARAGRAPHE000077"></a>Le moniteur du robot de surveillance permet d'avoir à tout moment accès aux différents contrôles lancés par le robot de surveillance.

Il permet de :

- [Créer, modifier ou supprimer un contrôle](../RobotMonitor/3541107.md).

- [Créer, modifier ou supprimer un modèle de contrôle](../RobotMonitor/3541124.md).

- Dupliquer un contrôle.

- Rechercher un contrôle.

- Exécuter un contrôle.

- Configurer :

	- [le moniteur](../RobotMonitor/3541104.md)

	- [le robot](../RobotMonitor/3541105.md)

	- [le contre-robot](../RobotMonitor/3541106.md)

	- [les utilisateurs](../RobotMonitor/3541104.md)




- Afficher l'historique.







<a name="NOTE4_3"></a>


### Composant ComposantsRobots
<a name="composant_composantsrobots_ELTPARAGRAPHE000112"></a>Un composant permettant de gérer le robot de surveillance est livré avec WINDEV. Ce composant est un composant utilitaire, nommé "ComposantsRobots". 

Le composant du robot de surveillance permet à l'application :

- de signaler son activité par écriture dans un fichier .ini.

- de parcourir un rapport d'erreur au format XML fourni par le robot de surveillance.

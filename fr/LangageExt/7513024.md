
## Bibli,Disque (Langage externe)

***En anglais : Library,Disk***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Ouvre la bibliothèque du projet qui est stockée sur le disque.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```txt
// En C
APPELWD("Bibli,Disque,Compta.WDL");
APPELWD("Projet,Compta");
```


<a name="Exemple2"></a>



```txt
// En Pascal
APPELWD('Bibli,Disque,Compta.WDL');
APPELWD('Projet,Compta');
```


<a name="Exemple3"></a>



```txt
// En VB
call APPELWD("Bibli,Disque,Compta.WDL")
call APPELWD("Projet,Compta")
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`Bibli,Disque(<Nom de la bibliothèque> [, <"/CR">])`
---

**`<Nom de la bibliothèque> : (Chaîne de caractères)`**

Nom de la bibliothèque avec l'extension et éventuellement le chemin d'accès.

**`<"/CR"> : (Chaîne de caractères (optionnelle))`**

Compte-rendu optionnel retourné par **Bibli,Disque**.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- **Bibli,Disque** doit être appelée avant HInit et Projet.
	Une bibliothèque est créée par l'éditeur de WINDEV et a pour extension WDL. Une seule bibliothèque peut être ouverte à un instant donné.
	Le chargement de la feuille de styles du projet est effectué par la fonction:
	Projet,&lt;nom_projet&gt;

- **Bibli,Disque** est appelée avec le paramètre "/CR":

	- Si une erreur s'est produite lors de l'ouverture de la bibliothèque :

		- le compte-rendu "\*E\*" est retourné dans la variable WdTouche (WDTouche en C)

		- WdEntier (WDEntier en C) contient alors un fichier inexistant, une bibliothèque invalide et une  mémoire insuffisante.




- Si aucune erreur ne s'est produite lors de l'ouverture de la bibliothèque :

	- WdEntier (WDEntier en C) contient 0

	- WdTouche (WDTouche en C) retourne une chaîne vide.

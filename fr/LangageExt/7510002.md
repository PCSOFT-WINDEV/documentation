## Principe
<a name="principe_ELTTEXTE000155"></a>
WINDEV peut également être utilisé pour créer des éléments qui seront gérés dans des programmes écrits en C, C++, Pascal, Visual Basic, Cobol, ...

Il suffit d'appeler les éléments WINDEV depuis le langage externe. Cet appel peut être réalisé depuis un programme (fichier ".EXE") ou depuis une librairie (fichier ".DLL" ou ".OCX") compilé en langage externe.

Pour appeler un élément WINDEV depuis un langage externe, utilisez la fonction [AppelWD](../LangageExt/7510001.md).

Pour gérer un fichier de données HFSQL depuis un langage externe, utilisez les [fonctions HFSQL du WLangage](../WDLang4/3044156.md).

**Remarque** : Il est également possible d'utiliser des traitements écrits dans un langage externe (sous forme de librairies compilées) dans une application WINDEV. Il faut pour cela compiler une librairie (fichier ".DLL" ou ".OCX") en langage externe puis appeler les traitements de cette librairie grâce à la fonction WLangage [API](../WDLang1/3014005.md) ou [AppelDLL32](../WDLang1/3014003.md).


**Attention** : WINDEV permet de compiler uniquement du code WLangage.

Les différents langages disponibles sont les suivants :

- [C et C++](../LangageExt/7513005.md)

- [Visual Basic](../LangageExt/7517001.md)

- [Pascal et Delphi](../LangageExt/7516001.md)

- [Cobol](../LangageExt/7511001.md)

- [Java](../LangageExt/7515001.md)

- [Fortran](../LangageExt/7514001.md)

- [C#](../LangageExt/7512001.md)

## Mise en place nécessaire à l'utilisation d'une interface avec un langage externe
<a name="mise_place_necessaire_utilisation_une_interface_avec_langage_externe_ELTTEXTE000179"></a>
Pour utiliser des éléments développés en WINDEV avec un langage externe, sélectionnez "Langages externes" lors de l'installation de WINDEV.

**Remarque** : Les fichiers nécessaires au "Langages externes" sont automatiquement installés lors d'une installation complète de WINDEV.

**Attention** : Votre compilateur du langage externe utilisé doit obligatoirement être installé sur le poste de développement.

<a name="NOTE3_1"></a>


## Fichiers nécessaires pour exécuter le programme écrit en langage externe
<a name="fichiers_necessaires_pour_executer_programme_ecrit_langage_externe_ELTTEXTE000203"></a>
Pour exécuter le programme écrit en langage externe, vous devez disposer :

- Des librairies WINDEV (Fichiers ".DLL").

- Des librairies spécifiques au langage externe utilisé.

- Du ou des programmes exécutables écrits en langage externe.

- De la ou des bibliothèques de l'application WINDEV.

- De l'analyse associée à l'application WINDEV. Cette analyse peut être intégrée à la bibliothèque.



<a name="NOTE3_2"></a>


### Librairies WINDEV (fichiers ".DLL")
<a name="librairies_windev_fichiers_dll_ELTPARAGRAPHE000080"></a>Le programme écrit en langage externe nécessite les librairies WINDEV pour pouvoir fonctionner (appel des fonctions WLangage, utilisation des objets graphiques, ...). Ces librairies sont listées lors de la création de l'exécutable sous WINDEV. Référez-vous à cette liste pour connaître les différentes librairies utilisées pour l'application WINDEV (fichiers "wdxxx\*.DLL").

Les librairies listées et les librairies wdxxxle.dll et wdxxxcpl.dll doivent être copiées dans le répertoire d'exécution du programme écrit en langage externe.

**Remarques** :

- En Java, il est également nécessaire de copier la librairie wdXXXIJV.DLL.

- En C#, il est également nécessaire de copier la librairie wdXXXICS.DLL.

- En Fortran et en Cobol, il est également nécessaire de copier la librairie wdXXXFL.DLL.



<a name="NOTE3_3"></a>


### Bibliothèques de l'application WINDEV
<a name="bibliotheques_application_windev_ELTPARAGRAPHE000095"></a>La ou les bibliothèques de l'application WINDEV contiennent toutes les fenêtres, les états, les images (et autres fichiers) de l'application WINDEV.

**Remarque** : Si en test d'exécution du programme sous l'éditeur de langage externe, la ou les bibliothèques (fichier ".WDL") ne sont pas trouvées, il est nécessaire de préciser le chemin de la ou des bibliothèques lors de leur ouverture.

**Cas particulier** : En Visual Basic, il est nécessaire d'utiliser les fonctions Visual Basic suivantes : chDir et chDisk pour préciser le chemin de la ou des bibliothèques.

## Exemples livrés avec WINDEV
<a name="exemples_livres_avec_windev_ELTTEXTE000239"></a>
Pour chaque langage disponible, WINDEV propose un exemple complet d'utilisation. Ces exemples sont présents dans le sous-répertoire "LangagesExternes\\&lt;NomDuLangage&gt;" du répertoire d'installation de WINDEV.

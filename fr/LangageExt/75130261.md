
## Ecran,FichierVersEcran (Langage externe)

***En anglais : Screen,FileToScreen***



<a name="XUtilisation"></a>
<a name="Utilisation"></a>
<a name="description"></a>
Initialise les champs de la fenêtre avec les valeurs des rubriques du fichier auxquelles les champs sont reliés.
<a name="Exemple1"></a>
<a name="sample_code"></a>

## Exemple


```txt
// En C
HLitPremier("CLIENT","NOM")
if !HEnDehors() {
	APPELWD("Ecran,FichierVersEcran");
else
	APPELWD("Erreur,Fichier vide");
}
```


<a name="Exemple2"></a>



```txt
// En Pascal
HLitPremier("CLIENT","NOM");
If not HEnDehors() then APPELWD('Ecran,'FichierVersEcran')
else APPELWD('ERREUR,Fichier vide');
```


<a name="Exemple3"></a>



```txt
// En VB
if not HEnDehors() then Call APPELWD("Ecran,FichierVersEcran")
else call APPELWD("ERREUR,Fichier vide")
End If
```



<a name="XSYNTAXE"></a>
<a name="SYNTAXE1"></a>

## Syntaxe

`Ecran,FichierVersEcran()`
---



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Remarques


- **Ecran,FichierVersEcran** n'effectue aucune lecture du fichier. Les champs sont initialisés avec l'enregistrement chargé en mémoire. Un enregistrement doit être nécessairement en mémoire avant d'appeler cette fonction.

- Dans le cas de liaison directe, **Ecran,FichierVersEcran** transfère la valeur des rubriques de l'enregistrement chargé en mémoire dans les champs reliés, sans effectuer de lecture.

- Dans le cas de liaison avec recherche dans un fichier relié, la recherche est effectuée dans le fichier relié avant de transférer la valeur des rubriques dans le champ.

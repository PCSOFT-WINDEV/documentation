## Principaux termes utilisés
<a name="principaux_termes_utilises_ELTTEXTE000103"></a>



|   |
| --- |
| **Accès natif (Connecteur natif)**<br>Méthode de connexion à une base de données depuis un programme.<br><br> |
| **Ajax**<br>AJAX (pour Asynchronous Javascript and XML) est un ensemble de technologies permettant de rafraîchir uniquement les données modifiées dans une page HTML sans réafficher la totalité de la page.<br><br> |
| **Affectation**<br>Opération qui consiste à mettre une valeur dans une variable ou dans un champ. Par exemple : <br><br><br><pre><code>// Affectation de la valeur DUPONT dans la variable Nomclient<br>Nomclient = "DUPONT"</code></pre><br>   <br>Le signe = est l'opérateur d'affectation.<br><br> |
| **Alignement**<br>Façon d'organiser les champs d'une fenêtre ou d'une page. Par exemple, centrer un champ dans une page, mettre plusieurs champs à la même largeur, etc.<br><br> |
| **Ambiance**<br>Charte graphique d'un site WEBDEV.<br><br>Elément dans lequel est définie la forme graphique des pages d'un projet WEBDEV.<br><br> |
| **Analyse**<br>Description de la structure des fichiers de données et de leurs relations.<br><br> |
| **Ancrage**<br>Mécanisme qui consiste à définir des règles de position ou de redimensionnement pour que le contenu d'une fenêtre ou d'une page s'adapte au redimensionnement d'une fenêtre ou d'un navigateur.<br><br> |
| **AWP**<br>Format de page adressable directement (URL fixe), référençable, sans contexte automatique.<br><br> |
| **Base de données**<br>Elément contenant les données d'un programme. Les données sont organisées en fichiers de données (également appelés tables).<br><br> |
| **Bloc (état)**<br>Elément constituant un état. Par exemple, un bloc Haut de page, un bloc Bas de page, un bloc Corps.<br><br> |
| **Champ (fenêtre ou page)**<br>Elément graphique servant à la constitution de l'interface d'un programme ou d'un site.<br><br> |
| **Classe (POO)**<br>Elément défini en Programmation Orientée Objet. Une classe rassemble des méthodes (actions) et des membres (données).<br><br> |
| **Classic (fichier de données)**<br>Type d'accès à un fichier de données HFSQL. Un fichier de données HFSQL est de type Classic lorsque l'on accède à celui-ci directement dans son répertoire.<br><br> |
| **Clé (fichier de données)**<br>Caractéristique d'une rubrique de fichier de données. Une rubrique clé permet d'optimiser les recherches et les tris dans un fichier de données. <br><br>Synonyme : index<br><br> |
| **Client/Serveur (fichier de données)**<br>Type d'accès à un fichier de données HFSQL. Un fichier de données HFSQL est de type Client/Serveur lorsque l'on accède à celui-ci en se connectant à un serveur contenant ce fichier de données via le moteur HFSQL.<br><br> |
| **Code du projet**<br>Code exécuté lors du lancement d'un programme ou d'un site.<br><br> |
| **Composant Externe**<br>Brique logicielle qui permet d'exporter une ou plusieurs fonctions métier à des fins de réutilisabilité.<br><br> |
| **Composant Interne**<br>Conteneur regroupant des éléments d'un projet (fenêtre, page, requête, état, classe, etc.) afin de permettre et de faciliter un partage avec un autre projet.<br><br> |
| **Configuration de projet**<br>Description du format de sortie d'un projet : exécutable Windows, Linux, JAVA, etc.<br><br> |
| **Connecteur natif (Accès natif)**<br>Méthode de connexion à une base de données depuis un programme.<br><br> |
| **Contexte (Page)**<br>Partie en mémoire sur un serveur Web qui a servi à constituer une page qui est visible dans un navigateur.<br><br> |
| **Contrainte d'intégrité**<br>Règle à respecter associée à une rubrique de fichier de données afin d'assurer la cohérence des données dans une base de données.<br><br> |
| **CSS**<br>Langage de description des styles des différents éléments d'une page HTML.<br><br> |
| **Data binding**<br>Méthode qui permet d'associer un élément graphique de l'interface à une donnée (variable, rubrique).<br><br> |
| **Déploiement**<br>Action d'installer un programme sur le poste d'un utilisateur.<br><br> |
| **Editeur**<br>Programme qui permet de créer un élément d'un projet (éditeur de fenêtres, éditeur d'états,..).<br><br> |
| **Etat**<br>Elément d'un projet qui définit une impression à réaliser.<br><br> |
| **FAA**<br>Fonctionnalités Automatiques de vos Applications.<br><br> |
| **Feuille de styles**<br>Contient la liste des styles utilisés dans un projet.<br><br> |
| **Fichier de données (Table)**<br>Elément constituant une base de données. Un fichier de données sert à stocker des données saisies dans un programme.<br><br>Par exemple, un fichier de données CLIENT va contenir les informations des clients qui ont été saisies dans un programme.<br><br> |
| **GDS**<br>Gestionnaire de sources. Outil pour organiser, partager des sources des projets, gérer les droits, etc.<br><br> |
| **Global(e) (variable ou procédure)**<br>Correspond à la portée en mémoire d'une variable ou d'une procédure. Un élément global est accessible depuis n'importe quel autre élément du projet. L'inverse est local.<br><br> |
| **Groupware utilisateur**<br>Outil de description et de gestion des droits d'accès à l'interface pour les utilisateurs d'un programme ou d'un site. Par exemple, empêcher un utilisateur de pouvoir cliquer sur un bouton "Supprimer" en fonction de son login ou de son groupe.<br><br> |
| **Homothétique**<br>Méthode de redimensionnement d'une image afin de maximiser son affichage sans déformer son contenu.<br><br> |
| **HTML**<br>HyperText Markup Language.<br><br>Langage qui permet de décrire les éléments d'une page Web.<br><br> |
| **IHM** (également appelé UI)<br>Interface Homme-Machine (ou Interface Utilisateur). Description des fenêtres ou des pages qui constituent un programme. C'est ce que voit l'utilisateur lors de l'utilisation du programme.<br><br> |
| **Index (fichier de données)**<br>Synonyme : clé<br><br> |
| **Javascript**<br>Langage de programmation qui est utilisé par les navigateurs pour faire des calculs et des traitements dans une page HTML (sur le poste de l'internaute).<br><br> |
| **Liaison (analyse)**<br>Permet de décrire la nature de la relation ou le point commun entre 2 fichiers de données de l'analyse. En fonction de la liaison spécifiée, les règles d'intégrité à respecter lors de l'écriture dans les fichiers de données concernés peuvent être définies.<br><br> |
| **Live Data**<br>Mécanisme qui consiste à afficher des données réelles provenant de la base de données durant la création de l'UI. Ce mécanisme n'est utilisé qui si l'élément est lié au fichier de données.<br><br> |
| **Local(e) (variable ou procédure)**<br>Correspond à la portée en mémoire d'une variable ou d'une procédure. Un élément local n'est accessible que dans le traitement dans lequel il a été défini. L'inverse est global.<br><br> |
| **Membre**<br>Variable faisant partie d'une classe ou d'une structure.<br><br> |
| **Menu contextuel**<br>Menu déroulant contenant des actions possibles dépendant de l'endroit où s'est effectué le clic droit de la souris et du type de l'élément sur lequel ce clic droit a été effectué.<br><br> |
| **Méthode**<br>Procédure faisant partie d'une classe permettant d'agir sur les données (membres) de la classe.<br><br> |
| **Modèle de champs**<br>Conteneur d'un ou plusieurs champs (avec traitements) qui peut être réutilisé indéfiniment dans des pages.<br><br>Caractéristique principale d'un modèle : si le modèle original est modifié, les modifications sont automatiquement répercutées sur les différentes utilisations du modèle.<br><br> |
| **Modèle de pages**<br>Conteneur représentant une page type qui peut être appliqué à une ou plusieurs pages d'un projet.<br><br>Caractéristique principale d'un modèle : si le modèle original est modifié, les modifications sont automatiquement répercutées sur les différentes utilisations du modèle.<br><br> |
| **Modèle d'états**<br>Conteneur représentant un état type qui peut être appliqué à un ou plusieurs états d'un projet.<br><br>Caractéristique principale d'un modèle : si le modèle original est modifié, les modifications sont automatiquement répercutées sur les copies.<br><br> |
| **n-tiers**<br>Méthode de programmation en couches. Chaque couche est indépendante et peut être changée sans impact sur les autres.<br><br>Avantage : Grande souplesse de maintenance.<br><br>Inconvénients : Difficulté et temps de développement.<br><br> |
| **Page**<br>Elément d'un projet WEBDEV dans lequel est définie une partie de l'interface graphique du site à réaliser. Un site est généralement composé de plusieurs pages qui s'enchaînent.<br><br> |
| **Paramètre (fenêtre, page, procédure, méthode, etc.)**<br>Elément attendu dans une fenêtre, page, procédure ou méthode, lors de l'appel de celle-ci. Chaque valeur passée en paramètre doit être affectée dans une variable.<br><br> |
| **POO**<br>Abréviation de Programmation Orientée Objet.<br><br> |
| **Popup**<br>Type de fenêtre (ou page). Une popup est une fenêtre (ou page) qui se superpose à une fenêtre (ou page). Il est ainsi possible de continuer à visualiser le contenu de la fenêtre (ou page) de dessous tout en continuant à saisir dans la popup.<br><br> |
| **Privé(e)**<br>Variable ou procédure utilisable uniquement dans l'élément en cours.<br><br> |
| **Procédure**<br>Elément d'un projet contenant le code d'un traitement à exécuter.<br><br> |
| **Programmation événementielle**<br>Type de programmation. Une action d'un utilisateur sur une fenêtre ou une page induit un code à exécuter. Le code de l'action à exécuter est saisi dans l'événement représentant l'action de l'utilisateur.<br><br>Par exemple, l'événement de "Clic d'un bouton" correspond à un clic avec la souris d'un utilisateur sur ce bouton.<br><br> |
| **Programmation Orientée Objet (POO)**<br>Méthode de programmation avancée, par opposition à la programmation procédurale.<br><br>En POO, on manipule uniquement des objets, c'est-à-dire des ensembles groupés de variables et de méthodes associées à des entités intégrant naturellement ces variables et ces méthodes.<br><br>En programmation procédurale, on définit des fonctions s'appelant mutuellement. Chaque fonction ou procédure est associée à un traitement particulier qui peut être décomposé en sous-traitements jusqu'à obtenir des fonctions basiques.<br><br> |
| **Projet**<br>Elément regroupant tous les éléments constituant un programme ou un site. Un projet contient par exemple, une analyse, des pages, des états, des requêtes, etc.<br><br> |
| **Propriété (champ, fenêtre, etc.)**<br>Mot-clé représentant une caractéristique d'un élément. Les propriétés permettent de manipuler et de modifier les caractéristiques des éléments d'un projet par programmation.<br><br> |
| **Public(que)**<br>Variable ou procédure utilisable depuis tous les éléments.<br><br> |
| **RAD**<br>Acronyme de Rapid Application Development<br><br>Méthode de développement rapide d'un programme à partir d'une analyse (description des fichiers de données).<br><br> |
| **RAD Applicatif**<br>Méthode de développement rapide d'un programme à partir d'un modèle de programme.<br><br> |
| **Requête**<br>Elément écrit en langage SQL qui permet d'accéder en lecture ou écriture au contenu d'une base de données relationnelle.<br><br> |
| **Rubrique**<br>Elément faisant partie de la structure d'un fichier de données (ou table) d'une analyse. Par exemple, un fichier de données CLIENT peut être composé des rubriques Nom et Prénom.<br><br> |
| **Rupture (état)**<br>Mécanisme qui consiste à regrouper et séparer des données suivant une valeur.<br><br>Par exemple, faire une rupture dans un état qui liste des clients par ville. On regroupe donc les clients de la même ville ensemble. Visuellement, on sépare les clients à chaque changement de ville.<br><br> |
| **Site dynamique**<br>Projet développé en WEBDEV qui contient des pages statiques et des pages dynamiques (en mode Session ou AWP). Les pages dynamiques (en mode Session ou AWP) permettent de gérer l'accès à des données stockées dans une base de données.<br><br> |
| **Site statique**<br>Projet développé en WEBDEV qui ne contient que des pages statiques, c'est-à-dire qui n'ont pas d'accès à une base de données.<br><br> |
| **Structure**<br>Type de variable qui est composée de plusieurs sous-variables.<br><br> |
| **Style**<br>Elément qui permet de décrire l'aspect graphique d'un champ dans une fenêtre ou dans une page. Un style est constitué par exemple, d'un type de police de caractères, de la taille des caractères, de la couleur des caractères, etc.<br><br> |
| **Table (fichier de données)**<br>Elément constituant une base de données. Une table sert à stocker des données saisies dans un programme. Par exemple, une table CLIENT va contenir les noms et les adresses des clients qui ont été saisis dans un programme.<br><br> |
| **Table (champ)**<br>Elément graphique présent dans une fenêtre ou une page. Un champ Table est constitué d'une ou plusieurs colonnes ainsi que de plusieurs lignes.<br><br> |
| **Tableau**<br>Type de variable qui contient plusieurs valeurs. Les valeurs sont accessibles au travers d'un indice. Les [] permettent depuis le langage d'accéder aux éléments d'un tableau.<br><br> |
| **UI**<br>Interface Utilisateur. Description des fenêtres ou des pages qui constituent un programme. C'est ce que voit l'utilisateur lors de l'utilisation du programme.<br><br> |
| **Variable**<br>Elément qui permet de stocker en mémoire une valeur du programme. Il existe plusieurs types de variables. Chaque type correspond à la nature de la valeur que l'on veut mémoriser. Par exemple, une variable de type chaîne pour stocker le nom d'une personne, une variable de type monétaire pour stocker un montant.<br><br> |
| **Webservice**<br>Programme installé sur un serveur Web dont les traitements sont accessibles par le web.<br><br> |
| **XML**<br>Langage pour organiser des données afin de normaliser et faciliter l'échange de ces données (utilisé notamment dans les échanges avec les Webservices).<br><br> |
| **Zoning**<br>Méthode pour définir le découpage d'une page Web. |


## Cheque: Actividad de una aplicación
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000115"></a>
El robot de monitorización propone comprobar la actividad de una aplicación WINDEV.

En este caso, el robot comprobará la actividad de la aplicación leyendo la fecha y la hora que se encuentran en un archivo.INI. Este archivo INI es un archivo de actividad, rellenado regularmente por la aplicación WINDEV que se comprueba. La actividad también puede leerse en la base de datos del robot (por ejemplo, para comprobar la actividad del robot o de un contra-robot). 

El cheque será:

- con éxito si la fecha y la hora leídas no superan el tiempo de espera de la alerta.

- sin éxito de otra manera.




**Observación**: El archivo a comprobar puede ser un archivo que se encuentra en un servidor FTP.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Parámetros específicos de la verificación relativos a la actividad de una aplicación
<a name="parametros_especificos_verificacion_relativos_actividad_una_aplicacion_ELTTEXTE000139"></a>


### Detalles de la aplicación
<a name="detalles_aplicacion_ELTPARAGRAPHE000027"></a>

Estos parámetros se refieren a la aplicación WINDEV para comprobar. Debe especificar:

- el nombre de el equipo donde está instalada la aplicación (ordenador accesible por el robot).

- el nombre de la aplicación WINDEV.

- el tiempo de espera antes de la alerta. Es el tiempo máximo permitido entre la fecha y la hora especificadas en el archivo INI y la fecha y la hora de la prueba.



<a name="NOTE2_2"></a>


### Información sobre la actividad
<a name="informacion_sobre_actividad_ELTPARAGRAPHE000038"></a>

Hay tres maneras de probar la actividad de la aplicación: 

- Archivo INI en red: Estos parámetros afectan al fichero INI utilizado para comprobar la actividad de la aplicación. En este caso, se debe especificar:

	- el nombre del directorio donde está instalado el archivo INI (ordenador accesible por el robot).

	- el nombre del archivo INI.

	- la sección que debe leerse en el fichero INI.

	- el nombre de la palabra clave que contiene la fecha y la hora en formato YYYYMMDDHHmmSS.




- Archivo INI por FTP: El archivo INI de la aplicación puede ser accesible en un servidor FTP. En este caso, debe especificar las características del servidor FTP que desea utilizar:

	- la Address del servidor FTP.

	- el número de puerto del servidor y el modo de conexión (pasivo o no).

	- el usuario y la contraseña de associated.




- Base de datos de robots: Estos parámetros también pueden almacenarse en la base de datos del robot, en el archivo de datos "ApplicationActivity".. En este caso, debe especificar el identificador de la aplicación.  



<a name="NOTE2_3"></a>


### Información sobre el reinicio de la aplicación
<a name="informacion_sobre_reinicio_aplicacion_ELTPARAGRAPHE000058"></a>

Esta información se utiliza para intentar reiniciar la aplicación WINDEV. Si la opción "Restart enabled" está marcada, puede especificar: 

- el número máximo de intentos de reinicio. 

- el nombre de usuario y la contraseña. La aplicación se reiniciará con esta información. 

- la aplicación ruta y el comando Line si es necesario.


## Verificar: Prueba de un código WLanguage
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000099"></a>
El robot de monitorización se utiliza para compilar y ejecutar el código WLanguage introducido.

El cheque será:

- exitoso si el código WLanguage ha sido compilado y devuelve <u><u><u><u>True</u></u></u></u>.

- sin éxito de otra manera.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Parámetros específicos de la prueba WLanguage
<a name="parametros_especificos_prueba_wlanguage_ELTTEXTE000123"></a>


### del código WLanguage
<a name="del_codigo_wlanguage_ELTPARAGRAPHE000027"></a>

Introduzca el código WLanguage utilizado para ejecutar la prueba solicitada. Este código se compila automáticamente y los errores de compilación (si existen) se muestran en la parte inferior de la pantalla.
<a name="NOTE2_2"></a>


### Observaciones
<a name="observaciones_ELTPARAGRAPHE000034"></a>

- El código WLanguage introducido debe devolver <u><u><u><u>True</u></u></u></u> durante la ejecución para que se considere válido.

- Se puede especificar un estado específico reporte a través del [[INFORME DE SITUACIÓN]] Variable. Este status reporte se incluirá en el error reporte.

- El botón "Test" sirve para comprobar directamente el código WLanguage. Si se ha utilizado la [[INFORME DE SITUACIÓN]] Variable, su contenido se muestra durante la prueba.

- El robot de supervisión admite las cadenas Unicode en tiempo de ejecución. Por lo tanto, todas las cadenas declaradas en el código WLangage son [Cadenas unicode](../Motscles/1514044.md) por Default. Cuando un control debe intercambiar datos con otra Process (a través de sockets, intercambiar archivos, ...), puede ser necesario declarar las cadenas en ANSI si la otra Process está compilada para soportar cadenas en formato ANSI. En este caso, debe especificar que el string es un string ANSI (declaración como: MyString es una cadena Ansi).


## Cheque: Conectividad HTTP
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000097"></a>
El robot de vigilancia leerá la cabecera y el contenido de una web Page.

El cheque será:

- con éxito si la Page existe y si su cabecera no contiene ningún error.

- sin éxito de otra manera.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Parámetros específicos de la comprobación relativos a la conectividad HTTP
<a name="parametros_especificos_comprobacion_relativos_conectividad_http_ELTTEXTE000121"></a>


### Información sobre la web Page
<a name="informacion_sobre_web_page_ELTPARAGRAPHE000023"></a>

Especifique: 

- la Address de la web Page que se va a probar. 

- el puerto de conexión utilizado.

- el método HTTP utilizado para comprobar que el servidor responde: get, Post, PUT, DELETE, HEAD o PATCH. 
	Este parámetro le permite probar determinados servicios web.



<a name="NOTE2_2"></a>


### Información sobre la verificación a realizar
<a name="informacion_sobre_verificacion_realizar_ELTPARAGRAPHE000038"></a>

Se puede comprobar la validez del contenido de Page. En este caso, puede hacer que se verifique si el contenido es idéntico entre una verificación y otra.
<a name="NOTE2_3"></a>


### Gestión de casos de error personalizados
<a name="gestion_casos_error_personalizados_ELTPARAGRAPHE000045"></a>

Puede definir casos de error específicos añadiendo condiciones en la cabecera y/o en el contenido de Page. El botón "+" permite añadir condiciones. Estas opciones le permiten definir: 

- la condición de error. 

- el contenido del mensaje de alerta si se cumple la condición.

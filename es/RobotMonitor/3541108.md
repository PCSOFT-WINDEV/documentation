## Comprobar las opciones de activación
<a name="comprobar_las_opciones_activacion_ELTTEXTE000230"></a>
Las opciones de activación de cheques le permiten determinar y definir: 

- si la comprobación está activada,

- en qué fecha debe reactivarse automáticamente el cheque. 




Observación: Puede desactivar fácilmente una comprobación a través del menú contextual ("Desactivar comprobación"). Esta opción desactiva la comprobación: 

- durante 1 día,

- durante una semana,

- durante 1 mes,

- para un tiempo personalizado interval. En este caso, una ventana permite especificar cuándo debe reactivarse el control. 

- indefinidamente.

### Contenido del mensaje
<a name="contenido_del_mensaje_ELTPARAGRAPHE000073"></a>

Las opciones de tratamiento de errores permiten definir: 

- el mensaje personalizado que se mostrará en el error reporte.

- la solución recomendada para resolver el problema.

- el destinatario de los informes de error con el nivel de gravedad para ese destinatario.

### Destinatarios de informes de error
<a name="destinatarios_informes_error_ELTPARAGRAPHE000134"></a>

Para cada control (prueba), puede definir los destinatarios de los informes de error, con el nivel de gravedad para cada destinatario. Por ejemplo, un control con errores puede ser importante para una persona y menor para otra. 

Para añadir un destinatario de cheque: 

1. Haga clic en "+". 

2. En la ventana que aparece, seleccione el usuario y valide. 

3. El usuario seleccionado aparece en la lista de destinatarios. Por defecto, el nivel de gravedad es set a "Error". 

4. Cambiar el nivel de gravedad del usuario si es necesario. Los niveles de gravedad son: 

	- Depuración,

	- Información,

	- Notification ,

	- Advertencia,

	- Error, 

	- Error crítico. 







**Recordatorio**: Las características de los usuarios se pueden definir en la ventana de gestión de usuarios.

### Programa para iniciar si se produce un error
<a name="programa_para_iniciar_produce_error_ELTPARAGRAPHE000157"></a>

Para cada prueba, tiene la capacidad de definir un programa específico que se iniciará si ocurre un error.. Este programa debe ser accesible desde el ordenador instalación del robot. Se puede especificar un comando Line.

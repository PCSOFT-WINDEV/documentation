
## Verificar: Registro de un sitio WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000077"></a>
El robot de vigilancia recuperará el registro del día para una aplicación WEBDEV en el servidor FTP de associated. Se analiza el contenido del log para encontrar los posibles errores.

El cheque será:

- con éxito si se recupera el log y si no contiene ningún error.

- sin éxito de otra manera.

<a name="NOTE2_1"></a>


## Parámetros específicos de la verificación del log de un sitio WEBDEV
<a name="parametros_especificos_verificacion_del_log_sitio_webdev_ELTTEXTE000101"></a>


### Información sobre el servidor WEBDEV
<a name="informacion_sobre_servidor_webdev_ELTPARAGRAPHE000023"></a>

Se debe especificar la siguiente información para comprobar los registros de un servidor WEBDEV:

- Dirección del servidor

- Puerto de conexión (con una conexión en modo pasivo si es necesario).

- El usuario y la contraseña de associated.

- El directorio en el servidor que contiene los registros del sitio solicitado.

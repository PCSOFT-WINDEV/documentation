
## Presentación del motor de monitoreo
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000163"></a>
WINDEV, WEBDEV y WINDEV Mobile incluyen un robot de supervisión. Este robot se utiliza para comprobar si una aplicación o un servidor funcionan correctamente.. Si el elemento vigilado falla (fallo de hardware, fallo de conexión de INTERNET, apagado del sistema, etc.), el robot envía las alertas que se hayan definido.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principio
<a name="principio_ELTTEXTE000187"></a>
El motor de monitoreo se compone de tres archivos ejecutables que se ejecutan en diferentes equipos. Este motor permite ejecutar diferentes pruebas: Prueba de INTERNET, prueba de servidor FTP, prueba de red, ...

El robot de vigilancia es una set de programas que incluye: 

- un monitor para configurar y visualizar el estado del robot y los diferentes controles. 

- un robot de supervisión, que realiza comprobaciones periódicas de la infraestructura o las aplicaciones. 

- si es necesario, un contra-robot que compruebe el funcionamiento del propio robot.  




Si se produce un problema durante una prueba, el motor de monitoreo le envía un mensaje utilizando uno de los siguientes métodos:

- Mensaje enviado a los indicados en el software de mensajería integrada (WDBal).

- Email enviado a las direcciones especificadas.

- Mensaje enviado a una aplicación específica.

- Pantalla control (alerta visual y/o sonora).

- Procedimiento WLanguage.

- A través de un programa de terceros.

## ¿Cómo instalar el robot de monitorización?
<a name="¿como_instalar_robot_monitorizacion_ELTTEXTE000211"></a>
Para utilizar el robot de monitorización, debe instalar los diferentes programas del robot de monitorización. El paquete de instalación (WX28 0PACKROBOT.EXE) se encuentra en el subdirectorio "Install\\Robot" del IDE utilizado.

Los diferentes módulos deben instalarse en el siguiente orden: 

1. [Instalar el robot de vigilancia](../RobotMonitor/3541101.md). El robot de vigilancia debe instalarse en un ordenador que no se apague nunca.

2. [Instalar el contra-robot si es necesario](../RobotMonitor/3541102.md). El contra-robot es una herramienta de seguridad adicional. Se utiliza para comprobar si el robot de monitorización funciona correctamente.

3. [Instalar el monitor](../RobotMonitor/3541103.md). El monitor se puede instalar en varios ordenadores. El monitor se utiliza para configurar las diferentes pruebas realizadas por el robot de monitorización.




**Observación**: Los tres programas deben estar instalados en la misma red local o dominio.

## ¿Cómo utilizar el robot de monitorización?
<a name="¿como_utilizar_robot_monitorizacion_ELTTEXTE000235"></a>


### aprovechamiento
<a name="aprovechamiento_ELTPARAGRAPHE000065"></a>

Para utilizar el robot de monitorización, debe arrancar el robot (así como el robot de contador si es necesario)..  

El monitor se puede iniciar para gestionar las pruebas a ejecutar.
<a name="NOTE4_2"></a>


### Uso del monitor
<a name="uso_del_monitor_ELTPARAGRAPHE000074"></a>

El monitor del robot de monitorización le permite acceder a las diferentes comprobaciones iniciadas por el robot de monitorización.

Está acostumbrado:

- [Crear, editar o eliminar un cheque](../RobotMonitor/3541107.md).

- [Crear, editar o eliminar un modelo de cheque](../RobotMonitor/3541124.md).

- Duplicar un cheque.

- Encuentra un cheque.

- Haz una comprobación.

- Configurar:

	- [el monitor](../RobotMonitor/3541104.md)

	- [el robot](../RobotMonitor/3541105.md)

	- [el contra-robot](../RobotMonitor/3541106.md)

	- [los usuarios](../RobotMonitor/3541104.md)




- Visualizar el historial.

### RobotComponent con Component
<a name="robotcomponent_con_component_ELTPARAGRAPHE000109"></a>

Con WINDEV se proporciona un Component para gestionar el robot de vigilancia. Esta Component se llama "RobotComponent". 

La Component del robot de vigilancia permite que la aplicación:

- señalan su actividad escribiendo en un archivo.ini.

- buscar un error reporte en formato XML suministrado por el robot de monitorización.


## 12. Replicación universal

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
La replicación universal permite mantener actualizadas bases de datos del mismo o distinto formato (HFSQL, Oracle, SQL Server, etc.). Por ejemplo, puede sincronizar una base de datos HFSQL Classic con una base de datos Oracle.

La replicación universal utiliza un modelo centralizado: todas las bases de datos están sincronizadas con una base de datos maestra. Luego, la base de datos maestra aplica los cambios a las demás bases de datos.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Replication_Bdd.gif)


La sincronización puede adaptarse a casos especiales. Por ejemplo, puede obtener los registros relacionados con un producto específico o creados en una fecha determinada, gestionar conflictos, mostrar una ventana de configuración, etc.

Estas adaptaciones deben realizarse mediante programación con la función [HRplFilterProcedure](../WDLang4/3044300.md).

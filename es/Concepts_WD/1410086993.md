
## 14. Modos de generación

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Además de las aplicaciones de Windows, WINDEV permite generar otros tipos de proyectos.


|   |   |
| --- | --- |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_modes_generation%20-%20HC%20Application.gif)<br> | **Aplicaciones**<br><br>Las aplicaciones son el modo de generación más común. Las aplicaciones desarrolladas con WINDEV pueden ejecutarse en las siguientes plataformas: Windows (32 o 64 bits) y Linux. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_modes_generation%20-%20HC%20Service.gif)<br> | **Servicios**<br><br>WINDEV permite generar servicios para Windows (32 o 64 bits) y Linux (en Linux, los servicios suelen llamarse daemons). Un servicio es un tipo específico de aplicación, que se inicia al mismo tiempo que el sistema operativo pero sin interfaz. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_modes_generation%20-%20HC%20Bibli.gif)<br> | **Bibliotecas y parches**<br><br>Una biblioteca es un archivo único que agrupa varios elementos de un proyecto WINDEV: ventanas, reportes, etc. Puede generar bibliotecas independientes que pueden ser utilizadas por otras aplicaciones, así como parches para una aplicación que ya ha sido implementada. Esto evita tener que reinstalar toda la aplicación para hacer una corrección menor. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_modes_generation%20-%20HC%20Compo%20ext.gif)<br> | **Componentes externos**<br><br>Los componentes externos son fragmentos de aplicaciones que permiten compartir una o varias funcionalidades específicas entre varias aplicaciones. Un componente externo generado con WINDEV también puede utilizarse en un proyecto WEBDEV o WINDEV Mobile. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_modes_generation%20-%20HC%20WebS.gif)<br> | **Webservices SOAP o REST**<br><br>Un Webservice (también llamado servicio Web XML) puede generarse a partir de un proyecto WINDEV o WEBDEV. Un Webservice utiliza un conjunto de funciones (uno o varios conjuntos de procedimientos) y los hace accesibles a través de la Web (o de una red privada) mediante los protocolos HTTP, SOAP y REST.<br><br>Observación: Un Webservice debe implementarse en un WEBDEV Application Server para ser utilizado. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_modes_generation%20-%20HC%20Net.gif)<br> | **Aplicaciones y DLL de ensamblado .Net**<br><br>WINDEV permite generar aplicaciones y ensamblados para la plataforma .Net de Microsoft. Un ensamblado .Net generado con WINDEV puede importarse en una aplicación .Net desarrollada en otro lenguaje de forma similar a un componente externo compartido entre dos aplicaciones WINDEV. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_modes_generation%20-%20HC%20Java.gif)<br> | **Aplicaciones Java**<br><br>WINDEV permite generar aplicaciones Java. Las aplicaciones Java pueden ejecutarse en cualquier equipo que tenga una máquina virtual Java, independientemente del sistema operativo. Esto permite crear aplicaciones que se ejecutan en Windows, Mac OS o Solaris, por ejemplo. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_modes_generation%20-%20HC%20W%20Uni.gif)<br> | **Aplicaciones universales de Windows 10**<br><br>WINDEV permite generar aplicaciones para la Plataforma universal de Windows. Estas aplicaciones pueden utilizarse en modo de mosaico o distribuirse a través de Windows Store. |

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Cuando su aplicación esté completa (ventanas, reportes, etc.), deberá crear el archivo ejecutable correspondiente.

El archivo ".EXE" se puede ejecutar directamente : no es necesario abrir la aplicación en WINDEV.

Una vez creado el archivo EXE, la aplicación se puede implementar en los equipos de los usuarios finales mediante el programa de instalación

<a name="NOTE2_1"></a>


## Crear el archivo ejecutable de una aplicación WINDEV
<a name="crear_archivo_ejecutable_una_aplicacion_windev_ELTTEXTE000163"></a>
**Para crear el ejecutable**:

1. Seleccione (si es necesario) la configuración de proyecto que corresponde al ejecutable que se creará:

	- Configuración de proyecto para un archivo ejecutable de Windows de 32 bits.

	- Configuración de proyecto para un archivo ejecutable de Windows de 64 bits.

	- Configuración de proyecto para un ejecutable Linux.




2. En la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar".

3. El asistente de creación de ejecutables se abre.

4. Especifique:

	- las ***opciones generales*** del ejecutable (nombre del ejecutable, ícono del ejecutable, splash screen, etc.).

	- el ***modo de funcionamiento*** del ejecutable:

		- Instancia múltiple: El programa puede iniciarse varias veces desde el mismo equipo: se abrirán varias instancias al mismo tiempo.

		- Instancia única: El programa solo puede iniciarse una vez desde el mismo equipo: solo se abrirá una instancia a la vez. Si el usuario intenta ejecutar otra instancia del programa, se activará la instancia actual.




- ***cómo se utilizará la biblioteca***. La biblioteca es opcional. Puede crearse en un archivo separado (extensión "WDL") o incluirse en el ejecutable.
			La ventaja de incluir la biblioteca en el archivo ejecutable es que solo necesita instalar o actualizar un archivo. Sin embargo, esto aumentaría el tamaño del archivo ".EXE".

- el ***mensaje de error*** que se mostrará si se produce un error de programación.

- si se va a activar la ***telemetría***. La telemetría le permite obtener información sobre cómo se utiliza la aplicación. 

- el uso de ***parches para las actualizaciones***. En este caso, solo tiene que copiar el archivo Patch en el directorio del ejecutable.

- los lenguajes del ejecutable.

- los ***archivos que se incluirán en la biblioteca*** (si se utiliza una).

- si los componentes utilizados en el proyecto se incluirán en el ejecutable.

- ***la ubicación de los archivos de datos del Groupware Usuario*** (si la aplicación utiliza el Groupware Usuario)

- la ***ubicación de los archivos de datos*** que utilizará el ejecutable.

- la ***información de la versión***. Esta información estará disponible en la pestaña "Versión" de la ventana de propiedades del ejecutable (accesible desde el explorador de Windows). El ***número de versión*** puede ***incrementar*** automáticamente cada vez que se crea el ejecutable.

- el modo de uso del framework WINDEV (framework específico de la aplicación o común a todas las aplicaciones WINDEV instaladas en el equipo).

- el manifiesto a utilizar (ejecución en Windows Vista y posterior solamente).

- si se debe realizar una ***copia de seguridad***.

5. Valide la descripción del ejecutable. El ejecutable se genera automáticamente a partir de esta información. El ejecutable puede ejecutarse directamente una vez que ha sido creado.

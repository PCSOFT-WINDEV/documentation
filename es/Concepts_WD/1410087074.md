<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
WINDEV incluye varios métodos para probar las aplicaciones. La tabla que aparece a continuación presenta los diferentes modos disponibles.

| Tipo de prueba | Descripción |
| --- | --- |
| Prueba de todo el proyecto ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_GO_Projet_WB_bl.gif) | Ejecuta la aplicación como lo haría el archivo ejecutable. Prueba toda la aplicación, aunque aún no se haya desarrollado por completo.<br><br>Cuando se produce un problema al ejecutar el proyecto, puede iniciar el depurador para identificar y solucionar el problema.<br><br>Observación: Si aparece el número "64" en el ícono "Go", significa que la prueba se ejecutará en modo de 64 bits en un editor de 32 bits. |
| Prueba del elemento actual ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_GO_Fenetre_WD_bl.gif) | Prueba un elemento una vez que ha sido desarrollado.<br>Puede iniciar el depurador cuando se produzca un problema en reportes y ventanas.<br><br>Observación: Si aparece el número "64" en el ícono "Go", significa que la prueba se ejecutará en modo de 64 bits en un editor de 32 bits. |
| Prueba de un proyecto paso a paso<br>(mediante el depurador) | Permite iniciar el depurador cuando se ejecuta la aplicación. Esta solución ayuda a para controlar el modo de funcionamiento de la aplicación mediante el depurador.<br><br>También puede depurar un archivo ejecutable específico, un ejecutable en ejecución o un archivo ejecutable activo en otro equipo. |
| Prueba unitaria | Prueba las ventanas, procedimientos y clases de una aplicación en diferentes etapas de desarrollo.<br><br>Los escenarios de prueba se generan automáticamente en WLanguage a partir de la prueba de la aplicación o de la prueba de un elemento. Los escenarios se pueden modificar directamente en el editor de código. |
| Prueba de regresión | Se utiliza para asegurarse de que un elemento no ha causado ninguna regresión entre diferentes versiones de una aplicación.<br>Para ello, seleccione (o cree) el escenario de prueba que se ejecutará. |

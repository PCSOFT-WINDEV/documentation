<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Una aplicación depende de varios parámetros externos: sitio web, servidor HFSQL, red, etc. Para optimizar la gestión de incidentes, WINDEV permite utilizar un motor de monitoreo.

El motor de monitoreo se compone de tres archivos ejecutables que se ejecutan en diferentes equipos. Este motor permite ejecutar diferentes pruebas: verificación de la conectividad a Internet, pruebas de red, etc.

Si se produce un problema durante una prueba, el motor de monitoreo le envía un mensaje utilizando uno de los siguientes métodos:

- Mensaje enviado a través de la aplicación de mensajería de PC SOFT (WDBAL).

- Correo electrónico.

- A través de un programa de terceros.


Además, se puede implementar una alerta sonora en el monitor.

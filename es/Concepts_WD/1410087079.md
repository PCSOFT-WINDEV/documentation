
## 1. El Framework de WINDEV

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
El Framework de WINDEV es un conjunto de bibliotecas (archivos .DLL) necesarias para ejecutar una aplicación WINDEV.

Al crear el ejecutable, puede elegir:

- un archivo ejecutable dependiente del Framework, 

- un archivo ejecutable independiente,

- el Framework WINDEV común. 




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Utilizar un archivo ejecutable dependiente del Framework
<a name="utilizar_archivo_ejecutable_dependiente_del_framework_ELTTEXTE000159"></a>
Cuando se utiliza un archivo ejecutable dependiente del Framework, las bibliotecas se instalan en el directorio de cada aplicación. Por lo tanto, cada aplicación WINDEV utiliza su propia versión de las bibliotecas. Al utilizar un archivo ejecutable dependiente del Framework, es posible cambiar el nombre de las bibliotecas.

**Ventajas de utilizar un archivo ejecutable dependiente del Framework**:

- Cada aplicación puede utilizar una versión diferente de las DLL del Framework.

- El Framework de una aplicación puede actualizarse sin afectar a otras aplicaciones.

- Se puede cambiar el nombre del Framework.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Utilizar un archivo ejecutable independiente
<a name="utilizar_archivo_ejecutable_independiente_ELTTEXTE000183"></a>
Cuando el Framework está integrado en el archivo ejecutable, se trata de un ejecutable independiente. Las DLL del Framework no se extraen al ejecutar el archivo ejecutable.

**Ventajas de utilizar un archivo ejecutable independiente**:

- Ejecutable independiente. 

- No se extraen las DLL del Framework. 




**Observación** : Dependiendo de las funcionalidades utilizadas por la aplicación, puede que no sea posible utilizar un ejecutable independiente.

<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Utilizar el Framework WINDEV común
<a name="utilizar_framework_windev_comun_ELTTEXTE000207"></a>
Al utilizar el Framework común, las bibliotecas se instalan en un directorio común. Todas las aplicaciones WINDEV instaladas en el mismo equipo comparten estas bibliotecas.

**Ventajas del Framework común**:

- El espacio de disco utilizado por el Framework (unos 30 MB) se comparte entre todas las aplicaciones.

- El Framework se actualiza una sola vez para todas las aplicaciones instaladas. 

- Se puede cambiar el nombre del Framework común. 




**Observación** : Cuando se utiliza el Framework común, se pueden reemplazar una o varias DLL (si se requiere una versión específica, por ejemplo).

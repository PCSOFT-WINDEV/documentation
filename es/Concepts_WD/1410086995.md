## Presentación
<a name="presentacion_ELTTEXTE000260"></a>
WINDEV permite generar aplicaciones Java sin tener experiencia previa en desarrollo con Java. Como todas las aplicaciones WINDEV, la aplicación se desarrolla en WLanguage.
<a name="NOTE1_2"></a>


### ¿Qué es una aplicación Java?
<a name="¿que_una_aplicacion_java_ELTPARAGRAPHE000019"></a>

Una aplicación Java se compila completamente en JAVA. Si la aplicación utiliza una base de datos, la base de datos debe tener un controlador JDBC (MySQL, Oracle, etc.). Este controlador es específico para cada base de datos (puede descargarse del sitio Web del publicador de la base de datos).

Las bases de datos HFSQL se acceden de forma nativa (solo en Windows y Linux) mediante las funciones SQL de WLanguage. No se requiere ningún controlador adicional.

Una aplicación Java corresponde a un archivo ".jar" que requiere Java Runtime para ejecutarse.

**Para compilar (en Java) un proyecto Java creado con WINDEV**, el J2SDK de Oracle (versión 1.4.2 o posterior) debe instalarse en el equipo de desarrollo. Hay una versión disponible en los medios de instalación de WINDEV. Este programa también puede descargarse desde el sitio de Oracle.
<a name="NOTE1_3"></a>


### ¿Por qué desarrollar una aplicación Java con WINDEV?
<a name="¿por_que_desarrollar_una_aplicacion_java_con_windev_ELTPARAGRAPHE000034"></a>

¿Tiene una aplicación WINDEV que funciona en Windows y quiere que sea accesible en cualquier sistema operativo? Genere una aplicación Java para que los usuarios de Mac OS, por ejemplo, puedan utilizar su aplicación.

Esto le permite compartir aplicaciones de red.
<a name="NOTE1_4"></a>


### Generar una aplicación Java con WINDEV
<a name="generar_una_aplicacion_java_con_windev_ELTPARAGRAPHE000045"></a>

Como se ha mencionado anteriormente, no es necesario tener experiencia previa en desarrollo Java para desarrollar una aplicación Java con WINDEV. Puede desarrollar sus aplicaciones en WLanguage como cualquier otra aplicación WINDEV y, al compilar el proyecto, las ventanas y funciones WLanguage se convierten automáticamente a Java.

**Observación**: el número de funciones WLanguage que pueden utilizarse en un proyecto Java está limitado en esta versión. Este número de funciones aumentará en las próximas versiones.

## Funcionalidades de un proyecto Java
<a name="funcionalidades_proyecto_java_ELTTEXTE000302"></a>


### Crear un proyecto Java
<a name="crear_proyecto_java_ELTPARAGRAPHE000058"></a>

Un proyecto Java puede crearse como cualquier proyecto WINDEV:

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P1_ICO_Cr%E9er_cpt.gif) en los botones de acceso rápido. La ventana de creación de nuevos elementos se abre: haga clic en "Proyecto". 

2. El asistente de creación de proyectos se abre.

3. En una de las primeras etapas, el asistente le permite elegir el modo de generación del proyecto. Seleccione "Archivo Java" y continúe.
	De forma predeterminada, un proyecto WINDEV para Java corresponde a una configuración de proyecto específica. El tipo de generación del proyecto puede cambiarse en cualquier momento. Solo debe crear una nueva configuración de proyecto.
	Una vez que el proyecto ha sido configurado como un proyecto Java, todas las herramientas disponibles en WINDEV se configuran automáticamente para el proyecto Java:

	- El RAD de una ventana o el RAD completo es específico para una base de datos manipulada por una aplicación Java.

	- El compilador indica si el proyecto contiene funciones o controles que no pueden utilizarse en Java. Un mensaje en color púrpura aparece en el panel "Errores de compilación".

	- La creación del archivo genera un archivo ".jar" en el directorio EXE del proyecto.

	- El módulo de instalación solo manipula los archivos necesarios para implementar la aplicación Java.





**Observación**: le recomendamos crear un nuevo proyecto WINDEV para desarrollar una aplicación Java en lugar de convertir una aplicación WINDEV a Java.
<a name="NOTE2_2"></a>


### RAD Java
<a name="rad_java_ELTPARAGRAPHE000084"></a>

Como para cualquier proyecto asociado a un análisis, puede utilizar la metodología RAD para generar las ventanas de su aplicación.

La metodología RAD tiene en cuenta todas las características específicas de la generación Java (controles, procesos, funciones que pueden utilizarse en una aplicación WINDEV generada en Java).

Independientemente del tipo de ventana que se genere, la primera etapa del asistente permite introducir el tipo de conexión a la base de datos:

- HFSQL (en Windows o Linux): aparece el asistente estándar de creación de ventanas.

- JDBC: puede especificar los parámetros de conexión a la base de datos que utiliza el controlador JDBC.



<a name="NOTE2_3"></a>


### Probar un proyecto Java en el editor
<a name="probar_proyecto_java_editor_ELTPARAGRAPHE000100"></a>

Cuando se prueba un proyecto Java (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_GO_Projet_WB_bl.gif)), el proyecto se compila y ejecuta en Windows. El proyecto no se compila en Java.

Para ejecutar la aplicación Java, debe ejecutar el archivo ".Jar" ubicado en el directorio EXE del proyecto.
<a name="NOTE2_4"></a>


### Compilar un proyecto Java en WINDEV
<a name="compilar_proyecto_java_windev_ELTPARAGRAPHE000113"></a>

Un proyecto Java se compila para:

- identificar errores de programación en el código WLanguage

- ver controles, eventos y funciones WLanguage que no están disponibles en Java. Estos errores aparecen en púrpura en el panel "Errores de compilación".
	Para obtener más información sobre los elementos de WINDEV que no están disponibles en una aplicación Java, consulte la ayuda en línea.


**Observación**: como se mencionó en la sección [¿Qué es una aplicación Java?](../Concepts_WD/1410086995.md), el J2SDK de Sun debe instalarse en el equipo de desarrollo para compilar el proyecto en Java.
<a name="NOTE2_5"></a>


### Crear un archivo Java
<a name="crear_archivo_java_ELTPARAGRAPHE000132"></a>

WINDEV permite:

- crear y compilar en un compilador de Java los diferentes archivos Java necesarios para la aplicación.

- crear un archivo Java que pueda ejecutarse directamente:

	- una aplicación Java estándar,

	- una aplicación Java Web Start.







Para crear el archivo Java, debe crear el ejecutable: en la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar". Un asistente le ayuda a crear el archivo Java.
<a name="NOTE2_6"></a>


### Implementar una aplicación Java
<a name="implementar_una_aplicacion_java_ELTPARAGRAPHE000161"></a>

Como para cualquier aplicación, WINDEV permite crear un programa de instalación para aplicaciones Java. Este procedimiento de instalación solo está disponible para Windows. El programa de instalación es un ejecutable. Se puede utilizar para instalar los archivos necesarios, así como el Java Runtime Environment (JRE). Para otras plataformas, es necesario implementar los archivos manualmente.

Para abrir el asistente de creación de la instalación, vaya a la pestaña "Proyecto", grupo "Generación" y haga clic en "Procedimiento de instalación".

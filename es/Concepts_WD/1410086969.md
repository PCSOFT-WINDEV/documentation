
## 3. Ciclo de desarrollo de una aplicación

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
WINDEV cubre todo el ciclo de desarrollo de una aplicación:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P1-Cycle%20de%20d%E9veloppement%20d'une%20application.gif)
**Detalles de las diferentes etapas**:

**Etapa de diseño**: Una aplicación puede diseñarse a partir de requisitos específicos, un modelado UML de los procesos o incluso de archivos de datos existentes.

**Etapa de desarrollo**: El proyecto y el análisis se crean a través de asistentes específicos. La aplicación puede desarrollarse en modo RAD (Rapid Application Development) con generación automática de código e interfaz; o puede resultar de la creación manual de los diferentes elementos del proyecto.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P1-Cycle%20de%20d%E9veloppement%20d'une%20application-phases.gif)
**Etapa de pruebas y generación**: WINDEV incluye varias herramientas de pruebas automatizadas para garantizar la fiabilidad de las aplicaciones y asegurarse de que no se produzca ninguna regresión durante las etapas de desarrollo.

**Etapa de implementación**: Una aplicación WINDEV se puede implementar a través de varios métodos: descarga, a través de una red local (incluyendo el modo Push) o a través de Internet. En cualquier caso, los archivos de datos HFSQL se actualizarán automáticamente en función de los cambios realizados en la aplicación.

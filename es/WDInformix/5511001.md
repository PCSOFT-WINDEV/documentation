


## Conector Nativo Informix: Programación con funciones HFSQL
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000190"></a>
Para utilizar una base de datos Informix con Conector Nativo, **la estructura de las tablas Informix debe importarse en el análisis WINDEV o WEBDEV**. 

**Diferencias con respecto a la programación SQL**: Las consultas no son necesarias para manipular los datos de Informix. De hecho, las funciones HFSQL se pueden utilizar directamente para leer datos, realizar búsquedas e implementar filtros., ... Sin embargo, la estructura de los ficheros Informix debe importarse en el análisis WINDEV o WEBDEV.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Importación de la estructura de los ficheros
<a name="importacion_estructura_los_ficheros_ELTTEXTE000220"></a>


### Los pasos de importación
<a name="los_pasos_importacion_ELTPARAGRAPHE000022"></a>

Para realizar esta importación: 

1. Inicie la editor de análisis: haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Ico_Analyse.gif) en los botones de acceso rápido.

2. En la pestaña "Análisis", en el grupo "Creación", despliegue "Importar" y seleccione "Importar descripciones de archivos/tablas...".

3. El asistente se abre. Se establece automáticamente una conexión con la base de datos Informix y associated con los archivos importados en el análisis. Esta conexión se utilizará para manipular el archivo de datos Informix mediante programación.




En los diferentes pasos de la asistente, elija las siguientes opciones:

- Tipo de base de datos: Informix. 

- Características de la conexión: Servidor de datos, usuario y contraseña, base de datos.  

- La aplicación accederá a los datos en el formato actual.

- Una vez establecida la conexión, basta con seleccionar las tablas que se importarán en el análisis WINDEV o WEBDEV.. Estas tablas pueden manejarse mediante programación con las funciones HFSQL de WLanguage. Estas tablas se mostrarán en azul en el editor de análisis.



<a name="NOTE2_2"></a>


### Teniendo en cuenta la evolución de la base de datos Informix
<a name="teniendo_cuenta_evolucion_base_datos_informix_ELTPARAGRAPHE000054"></a>

Para tener en cuenta las evoluciones de las tablas importadas en el editor de análisis, vaya a la pestaña "Análisis", al grupo "Análisis", amplíe "Sincronización " y seleccione "Actualizar análisis a partir de bases de datos externas".

Una asistente comienza, permitiéndole:

- analizar las diferencias para las tablas importadas

- analizar las diferencias para todas las tablas de Informix.

## Programación
<a name="programacion_ELTTEXTE000250"></a>


### 1. Abrir la conexión
<a name="1_abrir_conexion_ELTPARAGRAPHE000080"></a>

Para abrir la conexión con la base de datos Informix, utilice [HOpenConnection](../WDLang4/3044107.md) y especifique el nombre de la conexión que debe abrirse. El nombre de la conexión se definió al importar la estructura de los archivos al análisis.

Para modificar algunos parámetros de esta conexión (nombre de usuario o contraseña, por ejemplo), llame a [HConnect](../WDLang4/3044263.md).

**Observación**: Si intenta leer directamente el archivo de datos ([HReadFirst](../WDLang4/3044051.md), ...), se abrirá automáticamente la conexión asociada al fichero description en el análisis.
<a name="NOTE3_2"></a>


### 2. Ejecutar las consultas
<a name="2_ejecutar_las_consultas_ELTPARAGRAPHE000100"></a>

[HExecuteQuery](../WDLang4/3044080.md) y [HExecuteSQLQuery](../WDLang4/3044084.md) se utilizan para ejecutar consultas en la base de datos actual.

- [HExecuteQuery](../WDLang4/3044080.md): se usa para ejecutar una consulta preestablecida con el editor de consultas.

- [HExecuteSQLQuery](../WDLang4/3044084.md): se utiliza para ejecutar una consulta especificando el código SQL de la consulta y el nombre que se asignará a este código SQL.



```wl
// Initialize the "Customer_84" query
HExecuteQuery(CUSTOMER_84, "InformixDatabase", hQueryWithoutCorrection)
HExecuteSQLQuery(QryCustomer, hQueryWithoutCorrection, ...
	"SELECT NAME FROM CUSTOMER", "InformixDatabase")
```

<a name="NOTE3_3"></a>


### 3. Recuperación del resultado
<a name="3_recuperacion_del_resultado_ELTPARAGRAPHE000121"></a>

El resultado es navegado por las funciones de lectura estándar: [HReadFirst](../WDLang4/3044051.md), [HReadNext](../WDLang4/3044037.md), [HReadSeek](../WDLang4/3044050.md)...

**Observación**: Las constantes *hLockWrite* y *hLockReadWrite* son ignoradas. De hecho, los bloqueos de lectura y/o escritura no son soportados por el Conector Nativo.
<a name="NOTE3_4"></a>


### 4. Cierre de la conexión
<a name="4_cierre_conexion_ELTPARAGRAPHE000139"></a>

[HCloseConnection](../WDLang4/3044095.md) se utiliza para cerrar la conexión con la base de datos una vez que se han ejecutado todas las consultas necesarias.

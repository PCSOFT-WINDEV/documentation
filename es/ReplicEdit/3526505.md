
## Publicación de la réplica desde ReplicEdit
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000100"></a>
Una réplica de description creada con ReplicEdit puede ser publicada desde esta herramienta en un servidor de réplicas. 

Esta operación debe realizarse cuando se replica una aplicación Android o iOS. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000124"></a>


### Publicación de una réplica
<a name="publicacion_una_replica_ELTPARAGRAPHE000018"></a>

Para publicar una réplica: 

1. Seleccione "Deployment .. Deploy the replication on a server".. El asistente se abre.

2. Seleccione el servidor en el que se implementará la replicación: 
	<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Publier.gif)


	- Un servidor de replicación dedicado. 

- Un servidor de replicación de PC SOFT Cloud. En este caso, debe tener una cuenta de PC SOFT Cloud. 

3. Para un servidor de replicación dedicado, este servidor se instaló de antemano. Para obtener más información, consulte [Servidor de replicación para la replicación universal asistida](../WDLang4/1000017227.md). 

4. Especifique: 

	- el servidor Address (el IP Address preferentemente),

	- un nombre de usuario y una contraseña,

	- el nombre de la replicación (nombre del archivo WER). 
			
			<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Publier_Serveur.gif)





5. En el siguiente paso, especifique la periodicidad de la preparación de los datos a replicar: 
	
	<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Publier_Period.gif)


	- **Preparación inmediata**: los datos a replicar se preparan durante una solicitud de replicación. En este caso, la réplica se utilizará para get los datos más actualizados. Sin embargo, la preparación de la réplica puede ser bastante lenta.

- **Preparación periódica**: los datos a replicar se preparan de forma regular, de acuerdo con una periodicidad específica. En este caso, durante la solicitud de reproducción, se utilizan automáticamente los últimos datos preparados.. Esta solución es más rápida (el tiempo de preparación de los datos para replicar es nulo) pero los datos pueden no estar actualizados.. 
			Esta solución se recomienda para replicar una aplicación móvil (Android o iOS). 

6. Validar. La asistente propone el código WLanguage que debe ser incluido en la aplicación para sincronizar los datos. 
	
	<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Publier_Code.gif)

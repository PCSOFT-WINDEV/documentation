
## Descripción de los archivos replicados (ReplicEdit)
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000117"></a>
Una replicación incluye uno o más archivos de datos. RepliEdit permite editar la description de un archivo de datos tomada en cuenta por la réplica. 

Para editar la description de un archivo de datos: 

1. Seleccione el archivo de datos en la lista que aparece en el editor de replicación. 

2. Haga doble clic en su nombre o seleccione "Propiedades" en el menú contextual.

## Las diferentes opciones
<a name="las_diferentes_opciones_ELTTEXTE000141"></a>


### Pestaña "General"
<a name="pestana_general_ELTPARAGRAPHE000023"></a>

La pestaña "General" presenta la siguiente información: 

- Nombre archivo, 

- Modo de acceso al fichero maestro,

- Dirección de replicación, 

- Gestión de conflictos. 




Esta información se introdujo al añadir el archivo a la réplica y puede modificarse en cualquier momento.. 


### "Pestaña Filtro
<a name="pestana_filtro_ELTPARAGRAPHE000035"></a>

La pestaña "Filtro" sirve para filtrar los datos a replicar. 

Veamos un ejemplo simple: Un vendedor desea recuperar la base de datos de clientes localmente.. En lugar de actualizar toda la base de datos, sólo se lleva a los clientes que viven en los EE.UU.. 

Para crear un filtro sobre un archivo de datos: 

1. En la pestaña "Filtro" seleccione "Selección".. 

2. La lista de artículos encontrados en el archivo de datos se muestra. 

3. Seleccione la Line correspondiente al elemento a filtrar. 

4. En la columna "Condición", seleccione la condición que debe aplicarse a la posición. 

5. En la columna "Valor o parámetro", especifique: 

	- El valor solicitado (por ejemplo,"EE.UU.").

	- Un parámetro ("Ciudad" por ejemplo). Este parámetro se inicializará con el valor solicitado durante la sincronización.







**Observación**: También tiene la posibilidad de introducir un filtro manual directamente. 

**Consejos:**

- Filtrar los datos replicados. De hecho, al limitar el número de registros replicados, la replicación será más rápida.

- Dé un nombre explícito a los parámetros.

	- Estos parámetros pueden utilizarse en [ReplicInitialize](../WDLang4/1000017195.md).  

	- Estos parámetros serán vistos por los usuarios finales que iniciarán la replicación a través de ReplicSynchro.








### "Ficha"Artículos
<a name="fichaarticulos_ELTPARAGRAPHE000066"></a>

La pestaña "Items" sirve para configurar los elementos a replicar. Todos los artículos son replicados por Default. 

Esta ficha también se puede utilizar para especificar el elemento de fecha que se utilizará para la replicación..




## Creación de una réplica
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000129"></a>
Para implementar una replicación asistida, el primer paso consiste en crear la replicación con ReplicEdit. 

La réplica se crea a través de una asistente. 

Los parámetros definidos en esta asistente pueden modificarse en la [descripción de la replicación](../ReplicEdit/3526502.md). 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Creación de la réplica
<a name="creacion_replica_ELTTEXTE000153"></a>
Para crear una réplica: 

1. Iniciar ReplicEdit desde WINDEV, WEBDEV o WINDEV Mobile: en la pestaña "Herramientas", en el grupo "Base de datos", despliegue "Replicación" y seleccione "ReplicEdit - Editor de replicación".

2. Si el inicio de asistente comienza, seleccione "Crear una nueva réplica". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Ass%20-%20HC%20N%B0001.gif)
 De lo contrario, seleccione "Archivo... Nuevo". Comienza la creación de la réplica de asistente.

3. En la creación de la réplica asistente, especifique el nombre y description de la réplica. Este nombre se le dará al archivo de replicación que se creará (archivo .WER)<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Ass%20-%20HC%20N%B0002.gif)
 Vaya al siguiente paso. 

4. La description de los archivos de datos a sincronizar debe ser cargada en ReplicEdit para crear la réplica. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Ass%20-%20HC%20N%B0003.gif)
 El asistente ofrece 2 posibilidades: 

	- **Utilizar un análisis (WDD)**. Para ello, seleccione la ruta del archivo ".WDD" correspondiente y la contraseña de associated (si es necesario). 
			**En este caso, el análisis debe contener una conexión HFSQL Client/Server. Esta conexión debe ser associated con los archivos de datos** 
			De hecho, ReplicEdit no se puede utilizar para crear una conexión. Sólo tiene la posibilidad de modificar una conexión existente. 

	- **Importar desde un servidor HFSQL**. Esta opción permite utilizar la description de una base de datos HFSQL Client/Server instalada en un servidor (este método debe utilizarse si la aplicación se despliega, por ejemplo). **Tsu solución se recomienda si sus archivos se describen en HFSQL Client/Server en el análisis**. ReplicEdit asignará automáticamente la conexión utilizada a los archivos de datos para replicar.
			En este caso, la asistente solicita: 

		- los parámetros de conexión: Address, puerto, usuario y contraseña. 

		- la base de datos para replicar. 




5. El siguiente paso le permite elegir la dirección de replicación. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Ass%20-%20HC%20N%B0004.gif)
 La dirección de la replicación puede ser:

	- Bidireccional: Los datos se sincronizan en la base de datos maestra y en la base de datos de los abonados.

	- Solo al esclavo: los datos se sincronizan desde el maestro al suscriptor.

	- Solo al maestro: los datos se sincronizan desde el suscriptor al maestro.




6. El siguiente paso se utiliza para elegir el método de gestión de los conflictos de modificación. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Ass%20-%20HC%20N%B0005.gif)
 La gestión de conflictos permite gestionar los registros que han sido modificados tanto por el maestro como por el abonado. Seleccione la regla a aplicar:  

	- El más reciente tiene prioridad: sólo se tiene en cuenta la última modificación de Record.

	- El maestro siempre tiene prioridad: se tiene en cuenta el Record que se encuentra en la base de datos principal.

	- El esclavo siempre tiene prioridad: se tiene en cuenta el Record que se encuentra en la base de datos de suscriptores.
			Atención: para evitar cualquier problema, el servidor y los ordenadores de los abonados deben utilizar el mismo tiempo. 




7. Al final de la asistente, puede añadir un archivo a la replicación o ir al editor directamente.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ReplicEdit_Ass%20-%20HC%20N%B0006.gif)

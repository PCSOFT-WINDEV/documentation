
## WDLog: Restaurando a un archivo de datos de su bitácora
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000095"></a>
WDLog se utiliza para aplicar a una copia de seguridad de un archivo de datos las diferentes operaciones guardadas en el registro de este archivo.

A partir de una copia de seguridad de un archivo de datos, tienes la posibilidad de volver a aplicar todas las modificaciones realizadas en este archivo (y guardadas en el registro) hasta una fecha determinada.

La regeneración de la un archivo de datos sólo puede realizarse en modo interactivo. Esta operación requiere:

- El archivo de registro associated con el archivo de datos (\*JNL.FIC).

- La contraseña del archivo de registro (si existe).

- El archivo de datos en el que deben realizarse las operaciones de registro. Si este archivo no existe, se puede volver a crear a partir de la información que se encuentra en el registro.

<a name="NOTE2_1"></a>


## Modo interactivo
<a name="modo_interactivo_ELTTEXTE000125"></a>


### Para regenerar los archivos de datos HFSQL
<a name="para_regenerar_los_archivos_datos_hfsql_ELTPARAGRAPHE000028"></a>

1. Inicie el registro WDLog.

2. Seleccione "Archivo... Gestionar los registros" (o haga clic en el libro).

3. Haga clic en el botón "Restaurar un archivo desde su registro".

4. Especifique:

	- El archivo de registro associated con el archivo de datos. 
			El archivo de datos se actualizará a partir de este archivo de registro. Este archivo debe ser un archivo &lt;FileName&gt;JNL.FIC.

	- La contraseña associated con el archivo de registro (si existe).

	- El archivo de datos en el que debe aplicarse el archivo de registro. 
			Este archivo debe ser el último archivo de datos válido que existe. Si este archivo no existe, WDLog puede intentar recrear a el archivo de datos a partir del registro.

	- El directorio en el que se creará el archivo de datos. 
			Este directorio debe ser especificado y debe diferir del directorio que contiene el archivo de copia de seguridad.

	- La fecha y hora límite. 
			Fecha y hora hasta la cual las operaciones encontradas en el archivo de registro se reproducirán de nuevo en el archivo de copia de seguridad.. El archivo de datos se actualizará. Todas las operaciones que se encuentran en el archivo de registro se realizarán si no se especifica fecha y hora.




5. Haga clic en el botón "Restaurar" para restaurar el archivo.

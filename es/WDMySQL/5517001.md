<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000238"></a>
WINDEV y WEBDEV 28 proponen un Conector MySQL Nativo (o Acceso MySQL Nativo). Esta Conector Nativo le permite acceder a una base de datos MySQL desde un programa WLanguage sin usar ningún controlador externo.

**Atención**: Esta versión del Conector Nativo de MySQL funciona con WINDEV y WEBDEV versión 28. El Native MySQL Connector no está disponible para WINDEV Mobile. Si utiliza WINDEV Mobile, el acceso a una base de datos ajena (MySQL, Oracle, SQL Server) debe realizarse a través de una [Webservice](../WDLang4/1000021063.md) proporcionada con WINDEV Mobile. Para obtener más información, consulte la ayuda sobre la función [SQLConnectWS](../WDLang4/1000021031.md). 

**Observación**: El Conector Nativo de MySQL es un módulo opcional y gratuito que se puede descargar de [el sitio de PC SOFT](https://www.windev.com)

### Condiciones de uso del Native MySQL Connector
<a name="condiciones_uso_del_native_mysql_connector_ELTPARAGRAPHE000036"></a>

- Capa de cliente de MySQL (archivo LibMySQL.DLL)

	- Capa de cliente de 32 bits para utilizar el Conector MySQL nativo de 32 bits (versión 3.x a 5.x probada).

	- Capa de cliente de 64 bits para utilizar el Conector Nativo de MySQL de 64 bits (versión 5.x probada).




- Se debe encontrar la DLL ***LibMySQL.DLL*** DLL:

	- En el ordenador de desarrollo: 

		- en el directorio "\\Programas\\Framework\\Win32x86" del directorio instalación de WINDEV o WEBDEV (32 bits).

		- en el directorio "\\Programas\\Framework\\Win64x86" del directorio instalación de WINDEV o WEBDEV (64 bits).




- En el ordenador de implementación, en el directorio de los ejecutables o en el directorio de Windows. Para una implementación en un servidor Web, la DLL debe encontrarse en el directorio de la Conector Nativo o en el directorio de Windows.

### Los archivos necesarios
<a name="los_archivos_necesarios_ELTPARAGRAPHE000057"></a>

Los siguientes archivos son necesarios para usar el Native MySQL Connector:

- El ***LibMySQL.DLL*** DLL. Esta DLL se distribuye con MySQL. Para más detalles, consulte el acuerdo de licencia de MySQL sobre las condiciones de distribución de esta DLL.

	- 32-bit para usar el Conector MySQL Nativo de 32-bit.

	- Archivo de 64 bits para utilizar el conector MySQL nativo de 64 bits. 


A partir de la versión 5.5 del cliente libmysql.dll, el equipo que ejecuta la aplicación debe instalar un paquete **Visual Studio Redistributable** correspondiente a la versión del servidor (para más detalles, consulte el sitio de Oracle).




- El ***wd28 0msql.dll*** DLL: Requerido para la Conector Nativo de 32 bits a una base de datos MySQL. Este archivo se suministra con el Native MySQL Connector para WINDEV o WEBDEV.

- El ***wd28 0msql64.dll*** DLL: Requerido para la Conector Nativo de 64 bits a una base de datos MySQL. Este archivo se suministra con el Native MySQL Connector para WINDEV o WEBDEV.




Para depurar un proyecto de multiconfiguración en configuración de 32 y 64 bits: 

- copy la DLL libmysql de 32 bits en la subcarpeta EXE de configuración de 32 bits del proyecto. 

- copy la DLL libmysql de 64 bits en la subcarpeta EXE de configuración de 64 bits del proyecto.

### Instalando una nueva versión de MySQL
<a name="instalando_una_nueva_version_mysql_ELTPARAGRAPHE000090"></a>

El diálogo entre el conector nativo de MySQL y MySQL utiliza la biblioteca LIBMYSQL.DLL proporcionada con MySQL. 
La Conector Nativo puede dejar de funcionar si se modifica el comportamiento de esta DLL (al cambiar de versión, por ejemplo).

Pronto estará disponible una versión actualizada que tiene en cuenta estos cambios; mientras tanto, todo lo que tiene que hacer es usar la versión anterior de LIBMYSQL.DLL.. Esta DLL funcionará con la nueva versión de MySQL, sin dejar de ser compatible con la Conector Nativo.

### Archivos necesarios para usar Native MySQL Connector en Linux
<a name="archivos_necesarios_para_usar_native_mysql_connector_linux_ELTPARAGRAPHE000103"></a>

Para usar Native MySQL Connector en Linux, se deben realizar las siguientes operaciones:

1. Instalar la capa de cliente de MySQL en el servidor. Compruebe si la DLL ***libmysqlclient.so*** fue copiada en /usr/lib.
	Observaciones: 

	- La capa de cliente MySQL de 32 bits debe instalarse para utilizar el conector MySQL nativo de 32 bits.. 

	- La capa de cliente MySQL de 64 bits debe instalarse para utilizar el conector MySQL nativo de 64 bits.. 


 Si las DLLs copiadas son como libmysqlClient.so.12.0.0, se debe crear un enlace en esta DLL con la siguiente Line de código:
	
	```txt
	ln -s libmysqlclient.so.12.0.0 libmysqlclient.so
	```


2. Si está utilizando un sitio WEBDEV, copy las siguientes DLL al directorio del servidor de aplicaciones WEBDEV:

	- wd28 0hf.so: HFSQL library

	- wd28 0msql.so: biblioteca de Conector Nativo de 32 bits

	- wd28 0msql64.so: biblioteca de Conector Nativo de 64 bits

	- wd28 0sql.so: SQL library




3. Si está usando una aplicación Java o un lote de Linux, copy las siguientes DLLs al directorio de la aplicación:

	- wd28 0hf.so: HFSQL library

	- wd28 0msql.so: biblioteca de Conector Nativo de 32 bits

	- wd28 0msql64.so: biblioteca de Conector Nativo de 64 bits

	- wd28 0sql.so: biblioteca SQL

## aprovechamiento
<a name="aprovechamiento_ELTTEXTE000304"></a>


### Uso del conector MySQL nativo en sus aplicaciones WINDEV y sitios WEBDEV
<a name="uso_del_conector_mysql_nativo_sus_aplicaciones_windev_sitios_webdev_ELTPARAGRAPHE000156"></a>

Para utilizar MySQL en modo nativo, debe programar en la aplicación WINDEV o en el sitio WEBDEV:

- con las funciones "SQL" de WLanguage (véase [Programación con funciones SQL](../WDMySQL/5517004.md)).

- con las funciones "HFSQL" de WLanguage (véase [Programación con funciones HFSQL](../WDMySQL/5517003.md)).


**Observaciones**: 

- No se olvide de leer el documento [limitaciones del conector nativo de MySQL](../WDMySQL/5517002.md).

- El Conector Nativo MySQL tiene en cuenta la opción "Crear automáticamente los archivos de datos cuando se abran" (disponible en la pestaña "Archivos" de la descripción del proyecto). 



<a name="NOTE4_2"></a>


### Optimización de las funciones HFSQL
<a name="optimizacion_las_funciones_hfsql_ELTPARAGRAPHE000188"></a>

- **TLos controles "Browsing tabla" basados en consultas se optimizan**: el contenido de la control Tabla se puede ordenar haciendo clic en una de sus columnas.

- Para evitar volver a ejecutar la misma consulta varias veces al navegar por su resultado, le recomendamos utilizar la constante ***hNoRefresh*** (por ejemplo, si un solo equipo modifica los datos).

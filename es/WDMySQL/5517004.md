
## Conector Nativo MySQL: Programación con funciones SQL
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000125"></a>
Las funciones SQL se pueden utilizar para manipular la base de datos MySQL.

**Diferencias con respecto a la programación HFSQL**: El uso de funciones SQL para los datos de manipular MySQL no requiere la importación de la estructura de archivos en el análisis. Sin embargo, los datos sólo deben tratarse mediante consultas.. El contenido de los archivos MySQL no se puede leer directamente.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Programación
<a name="programacion_ELTTEXTE000155"></a>


### 1. Establecimiento de la conexión
<a name="1_establecimiento_conexion_ELTPARAGRAPHE000022"></a>

[SQLConnect](../WDLang4/3072005.md) abre una conexión a una base de datos MySQL a través de Conector Nativo. El tipo de base de datos utilizada (en este caso MariaDB) debe pasarse como parámetro a la función **SQLConnect**.

**Observación**: El nombre de la fuente de datos MySQL ("MySQLDatabase" en este caso) es el nombre o la IP Address de el equipo donde está disponible la base de datos. En cualquier caso, debe especificarse el nombre de la "Base de datos".


```wl
ConnectionNum = SQLConnect("MySQLDatabase", "", "", "", "MySQL")
// or ConnectionNum = SQLConnect("MySQLDatabase", "User", "Password", "MyDatabase", "MySQL")
 
// Connection to a remote MySQL database
ConnectionNum = SQLConnect("192.168.1.51", "User", "Password", "test", "MySQL")
```

<a name="NOTE2_2"></a>


### 2. Ejecutar las consultas
<a name="2_ejecutar_las_consultas_ELTPARAGRAPHE000035"></a>

Las consultas son realizadas por [SQLExec](../WDLang4/3072007.md). **SQLExec** espera que el texto de la consulta SQL como parámetro, así como el nombre que se le dará a esta consulta.


```wl
SQLExec("SELECT LASTNAME, FIRSTNAME, EXTENSION, PHOTO FROM CUSTOMER", "QRY1")
```

<a name="NOTE2_3"></a>


### 3. Recuperación del resultado
<a name="3_recuperacion_del_resultado_ELTPARAGRAPHE000047"></a>

Se pueden utilizar varios métodos para recuperar el resultado:

- [SQLFirst](../WDLang4/3072017.md): Recupera e itera sobre el resultado con [SQLFirst/SQLCol](../WDLang4/3072025.md).

- [SQLTable](../WDLang4/3072031.md): Recupera el resultado de la consulta en una control Tabla poblada programáticamente en la aplicación WINDEV o en el sitio WEBDEV. El resultado de la consulta puede ser recuperado por Partial Fetch.

- [SQLFetch](../WDLang4/3072011.md): Se utiliza para leer las diferentes líneas del resultado de la consulta. La iteración sobre el resultado de la consulta se realiza con [SQLFetch/SQLGetCol](../WDLang4/3072025.md).




[SQLLock](../WDLang4/3072003.md) se utiliza para bloquear los registros de tabla o los registros seleccionados por la consulta. El tiempo de espera para bloquear o modificar una Record bloqueada es de set a 50s por Default en MySQL. La base de datos devolverá un error cuando se alcance el tiempo límite.
<a name="NOTE2_4"></a>


### 4. Cierre de la conexión
<a name="4_cierre_conexion_ELTPARAGRAPHE000074"></a>

[SQLClose](../WDLang4/3072015.md) se utiliza para cerrar la conexión con la base de datos una vez que se han ejecutado todas las consultas necesarias.

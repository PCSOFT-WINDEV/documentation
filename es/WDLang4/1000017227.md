
## Servidor de replicación para la replicación universal asistida
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000192"></a>
La replicación universal asistida puede utilizar un servidor de replicación.

En este caso, la replicación se realiza mediante el servidor de replicación. La transferencia de réplicas se gestiona automáticamente.

***La replicación a través del servidor se puede realizar con replica inmediata***: en este caso, los datos se sincronizan inmediatamente. La base de datos de suscriptores envía los datos para replicar, se realiza la replicación y la base de datos de suscriptores recupera los datos para actualizarlos.. Esta operación puede llevar bastante tiempo.. Este modo de replicación se recomienda para una sincronización diaria, por ejemplo (un vendedor sincroniza cada noche con la oficina).

***La replicación por servidor se puede realizar con réplica retardada***: en este caso, los datos se sincronizan cuando los intercambios se realizan con el servidor. Cuando el servidor recibe los datos para replicar, prepara automáticamente los datos para devolverlos. Estos datos serán devueltos durante la próxima sincronización.. Este modo de replicación se recomienda para una sincronización periódica (cuando se realiza un pedido, por ejemplo).



Observación: Este servidor de replicación también puede utilizarse para realizar una replicación de datos entre una aplicación Android/iOS y una aplicación WINDEV. Para obtener más información, consulte [Replicación de datos móviles (Android o iOS)](../WDLang4/1000021071.md).

## Instalación del servidor de replicación
<a name="instalacion_del_servidor_replicacion_ELTTEXTE000222"></a>




### Presentación
<a name="presentacion_ELTPARAGRAPHE000035"></a>

Se puede instalar el servidor de replicación:

- a través del paquete instalación disponible en el subdirectorio **"Install\\WDRServer"** de WINDEV o WINDEV Mobile.

- al instalar una aplicación WINDEV.



<a name="NOTE2_2"></a>




### Instalación de un paquete
<a name="instalacion_paquete_ELTPARAGRAPHE000046"></a>

El paquete instalación le permite instalar o desinstalar el servidor de replicación.

Durante la instalación, debe especificar:

- el directorio instalación de los elementos del servidor.

- el directorio implementación de los datos de replicación. Este directorio será el directorio donde se crearán los archivos requeridos por la replicación.




**Observación**: Si hay varios servidores virtuales disponibles, debe especificar aquellos para los que se instalará un servidor de replicación.

Al final de instalación, el sitio para configurar las réplicas comenzará. Este sitio le permite configurar las diferentes replicaciones gestionadas por el servidor de replicación.

**Observación**: Para desinstalar el servidor de replicación, todo lo que tienes que hacer es reiniciar el paquete de instalación.
<a name="NOTE2_3"></a>




### Observación
<a name="observacion_ELTPARAGRAPHE000065"></a>

La interfaz Web del servidor de replicación requiere la creación de un alias en el servidor Web.. Esta creación es automatizada en la mayoría de los casos.

Si este alias no ha sido creado, debe ser creado manualmente. Este alias se llama "SRVRPL_WEB".

**Ejemplo para crear el alias en Apache 2.2**:


```txt
Alias /SRVRPL_WEB/ "D:/PC SOFT/Replication Server 19/SRVRPL/SRVRPL_WEB/"
<Directory "D:/PC SOFT/Replication Server 19/SRVRPL/SRVRPL_WEB/">
Order allow,deny
Allow from all
</Directory>
```






### Permisos de acceso
<a name="permisos_acceso_ELTPARAGRAPHE000076"></a>

El servidor de replicación utiliza una base de datos que se encuentra en la carpeta "\\ProgramData\\ReplicSRV\\".. Por lo tanto, en esta carpeta deben concederse plenos derechos a la cuenta utilizada por el servidor web (INTERNET guest) para que pueda acceder a los discos.

## Uso del sitio para configurar las réplicas
<a name="uso_del_sitio_para_configurar_las_replicas_ELTTEXTE000258"></a>
El sitio para configurar las réplicas se utiliza para: 

- crear los diferentes usuarios de las réplicas,

- configurar las diferentes réplicas gestionadas por el sitio.




Se requiere un nombre y una contraseña para conectarse al sitio.. Por Default, sólo se crea el administrador (nombre de "admin" sin contraseña).
<a name="NOTE3_2"></a>




### Gestión de cuentas
<a name="gestion_cuentas_ELTPARAGRAPHE000097"></a>

La pestaña "Cuentas" sirve para gestionar las diferentes cuentas de los usuarios de la replicación.. Sólo el administrador existe por Default.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ServReplic_Compte.gif)


La opción "Añadir una cuenta" se utiliza para crear una nueva cuenta.. Los elementos necesarios son:

- el nombre de usuario.

- la contraseña y su confirmación.

- el correo electrónico Address.




Desde el tabla que enumera los usuarios, usted tiene la posibilidad de añadir o eliminar una cuenta.
<a name="NOTE3_3"></a>




### Gestión de las publicaciones
<a name="gestion_las_publicaciones_ELTPARAGRAPHE000115"></a>

La pestaña "Publicaciones" se utiliza para gestionar las publicaciones de las réplicas, es decir, las réplicas que puede utilizar el servidor de réplicas.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ServReplic_publication.gif)


La opción "Añadir una publicación" se utiliza para crear una nueva publicación.. Especifique:

- El nombre de la publicación.

- El archivo de replicación a utilizar (archivo.WER). Este archivo fue creado por ReplicEdit.

- El modo de preparación de las réplicas: la preparación es inmediata por parte de Default: las réplicas se crean inmediatamente durante la replicación. Si selecciona una preparación periódica, el servidor de replicación preparará periódicamente la réplica.. La réplica no es la más actualizada, pero el tiempo de replicación está optimizado.




Si selecciona una preparación periódica, especifique los parámetros del período de preparación de la réplica (todos los días, ....).

La publicación se crea al validar la asistente.

Desde el tabla que enumera las publicaciones, usted tiene la posibilidad de añadir o eliminar una publicación.
<a name="NOTE3_4"></a>




### Gestión de las suscripciones
<a name="gestion_las_suscripciones_ELTPARAGRAPHE000137"></a>

La suscripción a una publicación se realiza: 

- al instalar la aplicación WINDEV

- por [ReplicInitialize](../WDLang4/1000017195.md).

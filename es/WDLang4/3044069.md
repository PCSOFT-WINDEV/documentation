


## HExportXML (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Exporta los registros de un archivo de datos (HFSQL u OLE DB), vista o consulta a un archivo XML.

**Observación**: Una hoja de estilo (archivo".XLS") se genera automáticamente al crear el archivo XML..

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Export all customers
HExportXML(CUSTOMER, "C:\TEMP\Customer.xml", hExpCreation)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HExportXML(<Data file> [, <Path of the XML file> [, <Items to export>]] [, <Options> [, <Progress Bar>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos, vista o consulta utilizada. Si no se especifica este nombre, la función **HExportXML** utilizará el último archivo de datos utilizado por la última función HFSQL (que empieza por "H").

**`<Path of the XML file>: (Cadena de caracteres opcional)`**

ruta completa del archivo XML para generar. 
Si no se especifica este parámetro, se generará un archivo &lt;Data file&gt;.XML en el directorio actual.

**`<Items to export>: (Cadena de caracteres opcional)`**

Lista de nombres de artículos a exportar. Los nombres están delimitados por comillas simples y separados por comas o caracteres CR.. Se ignoran los nombres de los elementos clave compuestos, los elementos binarios y los elementos de la nota binaria..
Si este parámetro corresponde a una cadena vacía (""), se exportan todos los elementos de el archivo de datos (excepto los elementos de clave compuesta, binaria y memo binario).

**`<Options>: (Constante opcional de tipo Integer)`**

Modo para crear el archivo XML:



|   |   |
| --- | --- |
| *hExpBreakable* | La exportación puede interrumpirse pulsando Esc. <br>Si se interrumpe la exportación, el fichero XML es válido y sólo contiene los registros exportados antes de la interrupción. |
| *hExpCreation* | Si el fichero de exportación XML ya existe, se sobrescribe. Se produce un error si no se especifica esta opción y si existe el archivo XML. |
| *hExpNormal*<br>(valor predeterminado) | Funcionamiento estándar. |
| *hExpWithoutXSL* | La hoja de estilo (archivo".XSL") no se crea. |



**`<Progress Bar>: (Nombre de control o ventana opcional)`**

Nombre de control Barra de progreso o nombre de la ventana utilizada para mostrar el progreso de la creación del archivo.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Exportación del contenido de la consulta
<a name="exportacion_del_contenido_consulta_ELTPARAGRAPHE000135"></a>

Para exportar el contenido de una consulta, debe inicializarla (función [HExecuteQuery](../WDLang4/3044080.md)) antes de utilizar la función **HExportXML**.
<a name="NOTE0_2"></a>


### Exportar un solo Record
<a name="exportar_solo_record_ELTPARAGRAPHE000149"></a>

Para exportar una única Record a una cadena XML, utilice [HRecordToXML](../WDLang4/3044093.md).
<a name="NOTE0_3"></a>


### Campos de tipo Array
<a name="campos_tipo_array_ELTPARAGRAPHE000161"></a>Si uno de los artículos exportados es un artículo de array, cada elemento del artículo de array se exporta en un nuevo nivel:


```xml
<FILE>
	<ITEM1>data of item 1</ITEM1>
	<ITEM2>data of item 2</ITEM2>
	<ARRITEM>
		<ARRITEM_1>data of element 1</ARRITEM_1>
		<ARRITEM_2>data of element 2</ARRITEM_2>
		<ARRITEM_3>data of element 3</ARRITEM_3>
	</ARRITEM>
...
</FILE>
```

**Recordatorio**: Se ignoran las claves compuestas, las notas binarias o los elementos binarios.
<a name="NOTE0_4"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000174"></a>

La entidad Document del archivo XML generado se llama HF_Document.

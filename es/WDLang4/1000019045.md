
## Preguntas frecuentes sobre HFSQL Cluster
			



## Preguntas/Respuestas
<a name="preguntasrespuestas_ELTTEXTE000275"></a>



|   |
| --- |
| **Atención**: <br>Se requiere un paquete específico de instalación para utilizar HFSQL Cluster. <br>Este paquete puede descargarse sin coste adicional en el sitio de PC SOFT, en la sección de descargas de módulos comunes a la versión 28. |





| Pregunta | ¿Qué es HFSQL Cluster? |
| --- | --- |
| Respuesta | HFSQL Cluster es un sistema de gestión de bases de datos relacionales destinado a ser desplegado en un grupo de servidores (un cluster) con el fin de garantizar una alta disponibilidad.. Para obtener más información, consulte [Descripción general de HFSQL Cluster](../WDLang4/1000019030.md). |





| Pregunta | ¿Cuál es la relación entre HFSQL Cluster , HFSQL Client/Server, HFSQL Classic y HFSQL Mobile? |
| --- | --- |
| Respuesta | Estos términos se utilizan para identificar las diferentes ediciones de la base de datos HFSQL. Todas estas ediciones comparten el mismo formato de archivo, lo que facilita la evolución y la transferencia de datos.. Todas las ediciones comparten los mismos enunciados para acceder a los datos.<br><br>- **HFSQL Cluster**: edición en clúster destinada a arquitecturas críticas que requieren una gran accesibilidad.<br><br>- **HFSQL Client/Server**: Edición Cliente/Servidor. Las bases de datos se ponen a disposición de las aplicaciones cliente mediante un servidor de base de datos.<br><br>- **HFSQL Classic**: Edición local o de red. Las bases de datos se almacenan en archivos simples que pueden ser utilizados por una sola aplicación en un ordenador local o que pueden ser compartidos en una red.<br><br><br> |





| Pregunta | ¿Cuál es la licencia propuesta para usar HFSQL Cluster? |
| --- | --- |
| Respuesta | La licencia HFSQL Client/Server también se aplica a HFSQL Cluster. |





| Pregunta | ¿Requiere HFSQL Cluster una configuración de hardware específica? |
| --- | --- |
| Respuesta | HFSQL Cluster no requiere ninguna inversión especial para ser implementado.. Para obtener más información, consulte [Instalación de HFSQL Cluster](../WDLang4/1000019031.md). |





| Pregunta | ¿El HFSQL CLuster está diseñado para un tipo específico de aplicaciones? |
| --- | --- |
| Respuesta | Todos los tipos de aplicaciones pueden ser tratadas por HFSQL Cluster, especialmente en los campos que requieren alta disponibilidad de datos y/o los campos que requieren balanceo de carga (Web hosting, ....). |





| Pregunta | ¿Tengo que modificar mi código HFSQL Client/Server existente para usar el Cluster HFSQL? |
| --- | --- |
| Respuesta | No. El código es totalmente compatible entre las diferentes ediciones de HFSQL. Sin embargo, en su versión actual, HFSQL Cluster tiene algunas limitaciones en comparación con HFSQL Client/Server (estas limitaciones se enumeran a continuación). Es muy importante ejecutar la prueba de cualquier aplicación que se vaya a implementar en un clúster antes de que se implemente. |





| Pregunta | ¿Cuántos servidores físicos se necesitan para crear una configuración mínima de clúster? |
| --- | --- |
| Respuesta | Se necesitan al menos tres servidores físicos para crear una configuración de clúster HFSQL: dos de ellos serán los nodos del cluster, el tercero correrá al servicio del Coordinador (servicio ClusterManager). |





| Pregunta | ¿Cuál es el número máximo de nodos en un cluster HFSQL? |
| --- | --- |
| Respuesta | Un clúster puede gestionar hasta 128 nodos. |





| Pregunta | ¿Pueden los servidores físicos de un clúster estar ubicados en varias ubicaciones geográficas? |
| --- | --- |
| Respuesta | Debido al rendimiento requerido para realizar una replicación sincrónica de las bases de datos, recomendamos encarecidamente no disociar geográficamente los nodos de un cluster.. Sin embargo, se puede utilizar un mecanismo de replicación asíncrona entre dos clusters geográficamente distantes. |





| Pregunta | ¿Las aplicaciones cliente pueden estar en 32 y 64 bits? |
| --- | --- |
| Respuesta | Las aplicaciones cliente pueden ejecutarse en Windows de 32 o 64 bits, Linux (sólo de 32 bits) y MacOS X (sólo de 32 bits), independientemente de la versión del clúster (32 o 64 bits). |





| Pregunta | ¿Cómo se administra HFSQL Cluster? |
| --- | --- |
| Respuesta | El clúster HFSQL se gestiona a través de [Centro de Control HFSQL](../CCHF/3540001.md). Para obtener más información, consulte [Gestión de HFSQL Cluster](../WDLang4/1000019032.md). |





| Pregunta | ¿Cómo cambiar la versión principal del motor HFSQL en los nodos de un clúster HFSQL? |
| --- | --- |
| Respuesta | La versión principal del motor HFSQL Client/Server se cambia sucesivamente en cada nodo:<br><br>- Detener un nodo,<br><br>- Actualizar el motor HFSQL Client/Server para el nodo correspondiente,<br><br>- Reiniciar, <br><br>- Espere a que el nodo se sincronice antes de realizar la misma operación en otro nodo.<br><br><br> |





| Pregunta | ¿Cómo se controla el estado de un cluster HFSQL? |
| --- | --- |
| Respuesta | El estado de los nodos de un Cluster HFSQL puede ser monitorizado a través de la aplicación [Robot de Monitorización](../RobotMonitor/3541100.md). |





| Pregunta | Después de cambiar la estructura de los archivos en el análisis (añadiendo, cambiando o borrando elementos), ¿cómo aplicar los cambios a los archivos en los diferentes nodos del cluster? |
| --- | --- |
| Respuesta | Simplemente aplique la modificación de la estructura (modificación de datos de automatic) a un nodo del cúmulo.<br>La sincronización del Cluster HFSQL Process transfiere modificaciones de estructura a todos los nodos del cluster. |





| Pregunta | ¿Es posible añadir o eliminar enlaces? |
| --- | --- |
| Respuesta | A partir de la versión 27, el soporte de modificación de datos de automatic es más completo: <br><br>- Se admiten enlaces ([HAddLink](../WDLang4/3044312.md), [HDeleteLink](../WDLang4/3044314.md), etc.)<br><br>- Se admiten archivos seguros. <br><br>- etc.<br><br><br> |





| Pregunta | ¿Tiene HFSQL Cluster alguna limitación? |
| --- | --- |
| Respuesta | La versión actual de HFSQL Cluster tiene las siguientes limitaciones en comparación con HFSQL Client/Server.<br>Estas limitaciones desaparecerán en las próximas versiones.<br><br>- No se pueden definir los límites del identificador de automatic para la replicación.<br><br>- Una copia de seguridad no puede ser restaurada sin detener el cluster.<br><br>- Las siguientes funciones no están permitidas:<br><br>	- HAddScheduledOptimization/HModifyScheduledOptimization/HDeleteScheduledOptimization<br><br>	- HAddScheduledBackup/HModifyScheduledBackup/HDeleteScheduledBackup/HExecuteScheduledBackup<br><br>	- HLoadParameter/HSaveParameter/HDeleteParameter<br><br>	- HCopyFile (sólo la sintaxis de Cliente a Servidor y la sintaxis de Servidor a Servidor)<br><br>	- HCreateServerTrigger/HDescribeServerTrigger/HActivateServerTrigger/HDeactivateServerTrigger/HDeleteServerTrigger<br><br>	- HSetServer (las siguientes opciones no están disponibles: hLogLevel, hMaxLogSize, hActivityStatisticsPeriod, hMaxActivityStatisticsSize, hServerPort, hMode2GB y hDebuggingPort)<br><br>	- HManageTask/HAddTask/HModifyTask/HDeleteTask<br><br>	- HNoDatabaseAccess/HEndNoDatabaseAccess<br><br>	- HNoModif/HEndNoModif<br><br>	- HLogRecreate<br><br>	- HLogRestore<br><br>	- HFree<br><br>	- HModifyDatabaseProperty/HModifyFileProperty/HModifyServerProperty<br><br>	- HBackup/HDeleteBackup<br><br>	- HDeleteFile<br><br>	- HDeleteDirectory<br><br>	- HDeleteAll<br><br><br><br><br><br> |

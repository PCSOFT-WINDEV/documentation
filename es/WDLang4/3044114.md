<a name="Use"></a>
<a name="description"></a>
Ordena una vista HFSQL creando un índice en un elemento de la vista..


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HSortView(ViewCustomer84, "City,Name")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HSortView([<View> [, <Sort item>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ordenó la vista,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HError](../WDLang4/3044088.md) permite obtener más detalles sobre el problema.




**`<View>: (Cadena de caracteres opcional)`**

Nombre de la vista utilizada. Si no se especifica este nombre, la función **HSortView** manipula la última vista utilizada por la última función de gestión HFSQL (que comienza por la letra H).

**`<Sort item>: (Cadena de caracteres opcional)`**

Lista de posiciones de clasificación que se encuentran en la vista. Los elementos se separan con una coma o retorno de carro *CR*. El orden de clasificación se puede especificar añadiendo el prefijo "+" (ascendente) o "-" (descendente) a la cadena.. Por Default, la clasificación se realiza en orden ascendente ("+"). Si no se especifica este parámetro, **HSortView** hace referencia al último elemento de la vista utilizada.
El formato posible para esta lista es:

- "[+/-]&lt;1.° elemento&gt; [, [+/-]&lt;2.° elemento&gt; ]"

- "[+/-]&lt;primer elemento&gt; [*CR* [+/-]&lt;segundo elemento&gt; ]"




La clasificación se realiza según la primera posición de clasificación y, a continuación, según la segunda., ... Se aplica el siguiente orden de clasificación:: para un tipo ascendente, "a" &lt; "u" &lt; "é"..
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) En este parámetro debe especificarse una única posición de clasificación.

## Observaciones


### Ordenar el artículo
<a name="ordenar_articulo_ELTPARAGRAPHE000145"></a>

Se puede crear un índice sobre cualquier elemento de la vista. Si el elemento es clave en el archivo de datos, se utiliza la relación de orden que se definió en el análisis (distinguir mayúsculas y minúsculas, puntuación, etc.).

### Clasificación de una vista en varias posiciones
<a name="clasificacion_una_vista_varias_posiciones_ELTPARAGRAPHE000156"></a>

Para ordenar una vista en varios elementos, puede mostrar los registros de la vista en un control Tabla poblado programáticamente y utilizar [TableSort](../WDLang1/3074035.md).

**Atención**: Esta solución sólo puede utilizarse para archivos de datos HFSQL Classic o Cliente/Servidor.

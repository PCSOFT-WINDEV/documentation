


## Estructura de los archivos de registro (HFSQL Classic)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000234"></a>
Los siguientes archivos se crean automáticamente al registrar un archivo de datos HFSQL Classic:


|   |   |
| --- | --- |
| **JournalOpération.Fic** | Lista de todas las operaciones realizadas en los ficheros de datos HFSQL registrados que utiliza la aplicación. Una operación corresponde a una función HFSQL. |
| **JournalIdentification.Fic** | Lista de las ubicaciones físicas de todos los archivos registrados que se encuentran en la aplicación |
| **xxxJNL.Fic** | Archivo creado para cada archivo registrado. Contiene el valor de los registros utilizados antes y después de cada operación. |

Esta página presenta la estructura de estos archivos.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Archivo JournalOpération.FIC
<a name="archivo_journaloperationfic_ELTTEXTE000264"></a>
Los diferentes artículos que se encuentran en JournalOpération.Fic son los siguientes:


|   |   |
| --- | --- |
| **Id_Operación** | Identificador de la operación realizada. |
| **Número_de_registro** | Número del Record usado en el archivo de datos. |
| **Id_Save_Save_Original_Record** | Número de la bitácora Record correspondiente al Record *antes de la modificación*. Este número de Record corresponde a un Record en xxxxJNL.Fic. |
| **Id_Backup_Editar_Grabación_de_copias_de_identificación** | Número del registro Record correspondiente al Record *after modification*. Este número de Record corresponde a un Record en xxxxJNL.Fic. |
| **ID_FunciónWL** | Identificador de la función WLanguage utilizada en el programa. Para encontrar el nombre de la función WLanguage utilizada, basta con consultar el fichero ListeDéfinitionHF.WL que se encuentra en el subdirectorio "Personal\\External\\" de WINDEV o WEBDEV) |
| **Poste_Nom** | Nombre de el equipo que realizó la operación (últimos 12 caracteres). |
| **Dirección_PostalIP** | IP Address de la computadora que realizó la operación. |
| **Aplicación** | Nombre de la aplicación desde la que se realizó la operación (primeros 12 caracteres) |
| **HComputadoraID** | Identificador del ordenador del usuario, definido mediante programación con [HComputer](../WDLang4/3044111.md). |
| **HJournalInfoMsg** | Comentarios sobre la operación registrada. Estos comentarios pueden ser configurados por [HLogInfo](../WDLang4/3044009.md). |
| **Heure_Poste_Client** | Hora GMT en la que se realizó la operación en el ordenador del usuario (2 horas de diferencia con la hora de verano francesa). |
| **ID_Replica** | Identificador de la réplica. Esta posición es utilizada por la reproducción. |
| **ID_FichierJNL** | Identificador del archivo JNL que contiene la copia de seguridad de Record |

<a name="NOTE3_1"></a>


## Archivo JournalIdentification.fic
<a name="archivo_journalidentificationfic_ELTTEXTE000288"></a>
Los diferentes elementos que se encuentran en JournalIdentification.Fic son los siguientes:


|   |   |
| --- | --- |
| **ID_FichierJNL** | Identificador del fichero utilizado. |
| **GUID_Fichier_Source** | Identificador único del archivo registrado. Este identificador es asignado automáticamente por el motor HFSQL al archivo cuando se crea el archivo de datos. |
| **GUID_JNL_Archivo_de_Archivo** | Identificador único del archivo de registro associated con el archivo de origen. Este identificador es asignado automáticamente por el motor HFSQL al archivo cuando se crea el archivo de datos. |
| **Emplacement_Fichier_Source** | Ruta del archivo de datos utilizada (archivo de datos HFSQL registrado) |


<a name="NOTE3_2"></a>


### GUÍA y registro Process
<a name="guia_registro_process_ELTPARAGRAPHE000123"></a>

El motor HFSQL soporta dos tipos de archivos GUID:

- GUÍA del archivo definida cuando el archivo fue descrito en el archivo editor de análisis. Esta GUID se utiliza para administrar el.REP.

- El archivo GUID definido al crear el archivo de datos físicos. Este GUID es utilizado por el mecanismo de registro para detectar que dos archivos de datos no utilizan el mismo archivo de registro.

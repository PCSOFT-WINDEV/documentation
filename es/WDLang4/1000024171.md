


## &lt;Fuente&gt;.ActivateAutoFilter (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Activa un filtro automático en los archivos de datos relacionados al navegar por un archivo XML. El filtro se puede ser desactivar con la función [&lt;Fuente&gt;.DeactivateAutoFilter](../WDLang4/1000024219.md).

**Atención**: Esta función solo está disponible para el Conector Nativo XML.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Enable all the possible automatic filters on the Customer data file
Customer.ActivateAutoFilter("*" )
// except between Customer and Order data files
Customer.DeactivateAutoFilter(Order)

/////////////////////////////

// Browse orders
Order.ReadFirst(OrderID)
WHILE NOT Order.Out()
	// Browse order lines of current order
	OrdLine.ReadFirst(OrderID)
	WHILE NOT OrdLine.Out()
		OrdLine.ReadNext(OrderID)
	END
	// Next order
	Order.ReadNext(OrderID)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Browse data file>.ActivateAutoFilter(<Data file to filter>)`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se activó el filtro,

- <u><u><u><u>False</u></u></u></u> si se produce un error (archivo o elemento inexistente).




**`<Browse data file>: (Nombre de archivo de datos)`**

Nombre del archivo de datos utilizado para la navegación.





**`<Data file to filter>: (Nombre de archivo de datos)`**

Nombre del archivo de datos a filtrar cuando se establece la posición en el archivo de datos utilizado para la navegación.

Este parámetro puede corresponder a "\*". En este caso, se activarán todos los filtros automáticos posibles.





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Veamos un ejemplo simple:

```wl
Parent.ActivateAutoFilter(Child)
```

Este código indica que cuando se navega por los registros del archivo de datos Hijo, la navegación se realiza con respecto al 'contexto' (nodo) actual del archivo de datos Padre: se navegará por todos los 'Hijo' de 'Padre'.


```wl
Parent.DeactivateAutoFilter(Child)
```

Este código indica que el filtro anterior no está activo: El archivo de datos Hijo y el archivo de datos Padre se navegarán de forma independiente. Esto significa que se navegará por todos los registros del archivo de datos Hijo en el hijo dado.

La función **&lt;Fuente&gt;.ActivateAutoFilter** permite activar/desactivar el enlace jerárquico entre el archivo de datos Padre e Hijo durante la navegación.


## HCancelAlias (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cancela un alias declarado previamente con la función [HAlias](../WDLang4/3044176.md). Después de esta operación, el motor HFSQL no reconoce el archivo de datos asociado al alias.

**Observación**: Si el archivo de alias está abierto, se cierra automáticamente durante la cancelación.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Create 2 aliases on the CUSTOMER data file
HAlias(CUSTOMER, CUSTOMER1)
HAlias(CUSTOMER, CUSTOMER2)
// Cancel the first alias
HCancelAlias(CUSTOMER1)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HCancelAlias(<Alias file>)`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Alias file>: (Cadena de caracteres)`**

Nombre del archivo de alias a cancelar. Si el archivo de alias no existe, se produce un error WLanguage no fatal .  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Cuando se cancela un alias (función **HCancelAlias**): 

- las modificaciones realizadas con las funciones [HChangeName](../WDLang4/3044141.md) y [HChangeDir](../WDLang4/3044168.md) no se cancelan. Si se vuelve a describir el alias, estos parámetros se conservan y se vuelven a aplicar.

- si se creó un archivo de datos con un nombre definido por la función [HChangeName](../WDLang4/3044141.md) y una ubicación dada por la función [HChangeDir](../WDLang4/3044168.md), el archivo de datos no se elimina físicamente. 

	- Para eliminar un archivo de datos creado a partir de un alias, se debe utilizar la función [HDeleteFile](../WDLang4/3044294.md). 

	- Para eliminar una carpeta creada a partir de un alias, se debe utilizar la función [HDeleteDirectory](../WDLang4/1000019001.md).

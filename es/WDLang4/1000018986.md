
## HCreateServerTrigger( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade o modifica un disparador de servidor sobre un servidor HFSQL. Para realizar estas operaciones: 

- El desencadenante debe haberse descrito previamente en el análisis.

- El Procedure almacenado liberado por el disparador debe existir en el servidor. Esta Procedure puede ser actualizada por [HRefreshSet](../WDLang4/3044354.md).




**Tsu función se utiliza para actualizar el servidor HFSQL sin tener que realizar una actualización de datos automatic.**

**Recordatorio**: Los triggers del servidor son procedimientos almacenados ejecutados por el servidor antes o después de una operación de escritura realizada en un archivo de base de datos.. Para obtener más información, consulte [Disparadores de servidor](../WDLang4/3044369.md).  




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HCreateServerTrigger(ClientTrigger)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HCreateServerTrigger(<Trigger>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Trigger>: (Cadena de caracteres)`**

Nombre del servidor que dispara para crear. Este factor desencadenante se describió en el análisis. 
Si este parámetro corresponde a "\*" ", se crearán todos los triggers descritos en el análisis.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Los disparadores del servidor descritos en el análisis pueden ser actualizados mediante la modificación de datos de automatic.




## HFound (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Verifica si el registro corresponde al filtro o búsqueda actual.

**HFound** devuelve <u><u><u>True</u></u></u> si existe el Record actual y si se cumple una de las siguientes condiciones:

- El navegador el archivo de datos está filtrado y el Record actual corresponde a las condiciones del filtro,

- La búsqueda de el archivo de datos se inició con una búsqueda y el actual Record corresponde al valor buscado,

- La búsqueda de el archivo de datos no está filtrada ni iniciada por una búsqueda.




Cuando una operación provoca una salida del filtro o del archivo de datos (es decir, cuando [HOut](../WDLang4/3044094.md) devuelve <u><u><u>True</u></u></u>), **HFound** devuelve <u><u><u>False</u></u></u>.

**Observación**: La función **HFound** corresponde a la variable H.Found.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HReadSeekFirst(Customer, Name, "Moore")
IF HFound() = True THEN
	Info("Record found")
ELSE
	Error("No record found")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HFound([<Data file>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el Record actual corresponde al filtro o a la búsqueda, 

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HError](../WDLang4/3044088.md) permite obtener más detalles sobre el problema.




**`<Data file>: (Cadena de caracteres opcional)`**

Nombre del archivo de datos, vista o consulta utilizada. Si no se especifica este nombre, **HFound** utilizará el último archivo de datos (vista o consulta) utilizado por la última función de gestión HFSQL (función que empieza por "H").

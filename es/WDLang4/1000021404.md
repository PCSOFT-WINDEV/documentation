


## SourceFromPage (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Se actualiza automáticamente:

- el valor de los elementos en un archivo de datos con el valor de los controles en Page. Los elementos del fichero de datos vinculados a los controles Page se actualizan automáticamente con los valores de dichos controles.

- el valor de las variables WLanguage con el valor de los controles Page. Las variables WLanguage vinculadas a los controles Page se actualizan automáticamente con los valores de dichos controles.


Esta operación se realiza independientemente del estado de los mandos (grises, inactivos o Invisible).

**Observaciones**: 

- **SourceFromPage** no se puede usar para escribir en el archivo de datos: sólo se asignan las variables de los elementos de del archivo de datos (es decir, las variables del Record cargadas en la memoria). Para modificar los registros, utilice [HAdd](../WDLang4/3044147.md), [HModify](../WDLang4/3044042.md), [HWrite](../WDLang4/3044092.md), etc.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Click on BTN_ADD
SourceFromPage(Customer, PAGE_Customer)
HAdd(Customer)
IF HErrorDuplicates() = True THEN
	Error("This number already exists")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`SourceFromPage(<Source> [, <Page>])`
---

**`<Source>: (Cadena de caracteres)`**

Nombre de la fuente que contiene los datos a actualizar. Esta fuente puede ser:

- El nombre de un archivo de datos: sólo se tienen en cuenta los controles vinculados a los elementos del archivo de datos.

- El nombre de un WLanguage Variable: sólo se tienen en cuenta los controles vinculados a la lengua de trabajo Variable.
	Observaciones: 

	- Si el nombre de la Variable está entre comillas, el control se vinculará a la Variable especificada . 

	- Si el nombre del Variable no va entre comillas, el control se vinculará a los datos del Variable (por ejemplo, un Variable que contenga el nombre de otro Variable).  







**`<Page>: (Nombre de la página)`**

Nombre de la Page que contiene los controles.
Si este parámetro no se especifica o es una cadena vacía (""), se utiliza la Page que contiene el código actual.




<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Vinculación entre un control y un WLanguage Variable o entre un control y un elemento de un archivo de datos
<a name="vinculacion_entre_control_wlanguage_variable_entre_control_elemento_archivo_datos_ELTPARAGRAPHE000100"></a>

Se puede determinar la vinculación entre un control y un WLanguage Variable o entre un control y un elemento de un archivo de datos:

- En el editor, en la ventana de control description (pestaña "Binding")

- De forma programada, con el programa [DataBinding](../Proprietes/2510060.md) Property.



<a name="NOTE0_2"></a>


### Actualización de los artículos: Casos especiales
<a name="actualizacion_los_articulos_casos_especiales_ELTPARAGRAPHE000115"></a>

- Para asignar automáticamente el valor NULL a un elemento con la función **SourceFromPage**:

	1. Marque "NULL supported" para el elemento en el editor de análisis.

	2. Seleccione la opción "NULL si está vacío" en la Edición control vinculada al artículo.




- Los elementos del archivo de datos que no están vinculados a un control no se actualizan con **SourceFromPage** .

- Si varios controles en Page están vinculados al mismo elemento del archivo de datos, es posible que **SourceFromPage** no actualice correctamente el elemento vinculado. ***No se recomienda vincular varios controles Page al mismo elemento*** .

- **SourceFromPage** no actualiza los elementos con los valores de los controles Image. Utiliza la función [HLinkMemo](../WDLang4/3044173.md).

- **SourceFromPage** no está disponible en las vistas.

- **Vinculación con elementos de "Contraseña segura"**: El valor del elemento "Contraseña" no puede modificarse hasta que el usuario cambie el valor en la control vinculada al elemento.



<a name="NOTE0_3"></a>


### Actualización de datos en la control Tabla (o en los controles del looper)
<a name="actualizacion_datos_control_tabla_los_controles_del_looper_ELTPARAGRAPHE000147"></a>

La función **SourceFromPage** no tiene ningún efecto en el contenido de un control Tabla. 

La transferencia de datos reales de un control Tabla a su fuente de datos (archivo de datos o Variable) se realiza al guardar. Este ahorro se realiza: 

- al finalizar la introducción si está seleccionada la opción "Guardar al salir de la fila (Modif.)" (pestaña "Detalles" de la ventana control Tabla description).

- con la función [TableSave](../WDLang1/3074008.md).




El mismo principio se aplica a los controles Looper vinculados a una fuente de datos.

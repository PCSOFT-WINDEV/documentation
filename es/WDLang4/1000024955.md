


## &lt;Fuente&gt;.ExportXLS (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Exporta los registros de un archivo de datos (HFSQL u OLE DB), vista o consulta a un archivo Excel (XLSX).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Export all customers from the Customer data file
Customer.ExportXLS("C:\TEMP\Customer.xlsx", hExpCreation)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = <Source>.ExportXLS(<Excel file path> , <List of items to export> [, <Options> [, <Progress Bar>]])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos, vista o consulta utilizada. 

**`<Excel file path>: (Cadena de caracteres)`**

ruta completa del fichero Excel (XLSX) que se va a generar. 
Si no se especifica este parámetro, se generará un archivo &lt;Nombre de archivo&gt;.XLSX en el directorio actual.

**`<List of items to export>: (Cadena de caracteres)`**

Lista de nombres de artículos a exportar. Los nombres están delimitados por comillas simples y separados por comas o caracteres CR. Se ignoran los nombres de los elementos clave compuestos, los elementos binarios y los elementos de la nota binaria..
Si este parámetro corresponde a una cadena vacía (""), se exportan todas las partidas de el archivo de datos (excepto las partidas de clave compuesta, binaria y memo binario).
Observación: Excel puede truncar los datos exportados de las notas de texto en función del tamaño de la nota..

**`<Options>: (Constante opcional de tipo Integer)`**

Especifica el modo de creación del fichero Excel:



|   |   |
| --- | --- |
| *hExpBreakable* | La exportación puede interrumpirse pulsando Esc. <br>Si se interrumpe la exportación, el fichero XLSX es válido y sólo contiene los registros exportados antes de la interrupción. |
| *hExpCreation* | Si el fichero de exportación XLSX ya existe, se sobrescribe. Se produce un error si no se especifica esta opción y si existe el archivo XLSX. |
| *hExpNoLineItemName* | La primera Line del archivo XLSX no contendrá los nombres de los artículos. |
| *hExpNormal*<br>(valor predeterminado) | Funcionamiento estándar. |



**`<Progress Bar>: (Nombre de control o ventana opcional)`**

Nombre de control Barra de progreso o nombre de la ventana utilizada para mostrar el progreso de la creación del archivo de Excel.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Exportación del contenido de la consulta
<a name="exportacion_del_contenido_consulta_ELTPARAGRAPHE000232"></a>

Para exportar el contenido de una consulta, debe inicializarla (función [&lt;Fuente&gt;.ExecuteQuery](../WDLang4/1000024944.md)) antes de utilizar la función **&lt;Fuente&gt;.ExportXLS**.
<a name="NOTE0_2"></a>


### Campos de tipo Array
<a name="campos_tipo_array_ELTPARAGRAPHE000249"></a>

Si uno de los elementos exportados es un elemento de tipo array, cada elemento array &lt;ELEMENTOARRAY&gt; se exporta en un elemento &lt;ELEMENTOARRAY&gt;_&lt;Índice del elemento&gt;.

Recordatorio: Se ignoran las claves compuestas, las notas binarias o los elementos binarios.

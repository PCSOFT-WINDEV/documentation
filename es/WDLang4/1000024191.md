


## &lt;Fuente&gt;.Alias (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Crea un alias lógico de un archivo de datos (o una consulta). El alias puede cancelarse con [HCancelAlias](../WDLang4/3044149.md).

**Recordatorio**: Un alias permite trabajar en diferentes archivos de datos físicos que tienen la misma lógica description. Por ejemplo, un archivo de datos se describe en el análisis y la función **&lt;Fuente&gt;.Alias** crea el alias del archivo.

**Observación**: Esta página de ayuda explica cómo crear un alias con **&lt;Fuente&gt;.Alias**. A partir de la versión 27, puede utilizar las variables de [Alias](../WDLang4/1000021446.md). Este tipo de Variable facilita la creación y los alias de manipular. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Data Source variable to handle the alias after its creation
OrderCheck is Data Source <description=Order>
// New feature of version 19: 
// <description=Order> is used to get the code completion 
// for the items in the Order data file

// Create an alias for the "Order" data file
IF Order.Alias(OrderCheck) = True THEN
	// "OrderCheck" can now be used in processes.
	// It behaves like the "Order" file described in the analysis.
	// By default, the name of the associated physical file corresponds to 
	// the name of the data source, therfore "OrderCheck" in this case
	// To handle the same physical file as the initial one, the name must be modified:
    OrderCheck.ChangeName(Order.PhysicalName)
    	// To use a file on another database or in another directory,
	// then use <Source>.ChangeConnection or <Source>.ChangeDir
ELSE
    Error("Failure creating the alias", HErrorInfo())
    RETURN
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.Alias([<Alias>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se creó el alias,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre lógico del archivo de datos (definido en el análisis) o nombre de la consulta (creado con el editor de la consulta). 

**`<Alias>: (Cadena de caracteres opcional o fuente de datos)`**

Nombre del alias, usado para definir la estructura del alias. 
Si no se especifica este parámetro, se cancelan todos los alias creados para &lt;Fuente&gt;.. Los archivos de datos correspondientes se cierran. 
Para optimizar sus aplicaciones, se recomienda utilizar una Variable de tipo [Fuente de datos](../WDLang4/1514053.md) (ver ejemplo). 

![OLE DB](https://doc.pcsoft.fr/ext/images/us/OLEDB.png)![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) El archivo &lt;Alias&gt; utiliza los parámetros de conexión de &lt;Source&gt; (parámetros definidos en el análisis).

## Observaciones


### Archivo de datos físicos associated con un alias
<a name="archivo_datos_fisicos_associated_con_alias_ELTPARAGRAPHE000296"></a>

Por Default, el nombre especificado para el alias se asigna al archivo de datos físicos correspondiente al alias. 

Para asociar el alias a un archivo de datos físico de nombre diferente, debe utilizar [&lt;Fuente&gt;.ChangeConnection](../WDLang4/1000024199.md), [&lt;Fuente&gt;.ChangeName](../WDLang4/1000024201.md) y [&lt;Fuente&gt;.ChangeDir](../WDLang4/1000024202.md).
<a name="NOTE0_2"></a>


### Cancelación de los alias de uno o más archivos de datos
<a name="cancelacion_los_alias_uno_mas_archivos_datos_ELTPARAGRAPHE000323"></a>

La cancelación de un alias ([HCancelAlias](../WDLang4/3044149.md)) cierra el archivo de datos associated.

Para cancelar los alias de un archivo de datos, se debe utilizar la función **&lt;Fuente&gt;.Alias** sin especificar el nombre del alias.


<a name="NOTE0_3"></a>


### Alias y consultas
<a name="alias_consultas_ELTPARAGRAPHE000346"></a>

- **Si la consulta fue creada a través de la programación de** ([&lt;Fuente&gt;.ExecuteSQLQuery](../WDLang4/1000024948.md)), basta con crear la consulta de alias con esta misma función especificando el nombre del alias.

- **If la consulta fue creada por editor de reportes**, para crear un alias, debe hacerlo usted:

	- ejecutar la consulta inicial. 

	- utilice **&lt;Fuente&gt;.Alias** y especifique el nombre de la consulta así como el nombre del alias. 

	- utilizar [&lt;Fuente&gt;.ExecuteQuery](../WDLang4/1000024944.md) con la consulta de alias. El alias se inicializa.


 Por ejemplo:
	
	```wl
	QueryAlias is Data Source
	QRY_Query1.Alias(QueryAlias)
	QueryAlias.ExecuteQuery()
	```

	**Observación**: Al crear el alias, se debe haber ejecutado la consulta inicial.
	Ejemplo:
	
	```wl
	QueryAlias is Data Source 
	IF QRY_Query1.ExecuteQuery() = False THEN
		Error(HError(hErrCurrent)) 
	ELSE
		QRY_Query1.ReadFirst()
		Trace(QRY_Query1.City)
		QRY_Query1.Alias(QueryAlias)
		IF QueryAlias.ExecuteQuery() = False THEN
			Error(HError(hErrCurrent)) 
		ELSE
			FOR EACH AliasQuery
				Trace(QueryAlias.City)
			END
		END
	END
	```




<a name="NOTE0_4"></a>


### Manejo del archivo de alias y sus elementos
<a name="manejo_del_archivo_alias_sus_elementos_ELTPARAGRAPHE000392"></a>

- Para evitar el error "*Identificador desconocido*" al utilizar un archivo de alias y sus elementos, se recomienda utilizar una variable de tipo [Fuente de datos](../WDLang4/1514053.md) para declarar el nombre del alias antes de llamar a la función **&lt;Fuente&gt;.Alias**.
	Por ejemplo: 
	
	```wl
	Order2018 is Data Source
	Order.Alias(Order2018)
	```

	Atención: los **Externo** se puede utilizar una palabra clave, pero ralentiza la ejecución significativamente.

- En [&lt;Fuente&gt;.ReadSeek](../WDLang4/1000025023.md), [&lt;Fuente&gt;.ReadFirst](../WDLang4/1000025022.md), [&lt;Fuente&gt;.ReadNext](../WDLang4/1000025035.md), ... el nombre del elemento alias no puede utilizarse directamente: el nombre del artículo debe estar entre comillas. 
	Por ejemplo:
	
	```wl
	srcAlias.ReadSeekFirst("KeyItem", "Sought Value")
	```


- El nombre de la fuente de datos no debe ir entre comillas, si [Nombramiento automático de las fuentes de datos](../Editeurs/9500218.md) está activado en la ventana del proyecto description.

- En el editor de ventanas o páginas, los controles no pueden vincularse directamente a los elementos de los archivos de alias. Debes hacerlo:

	- asignar un valor a cada control en el código.

	- utilizar la función [ControlAlias](../WDLang1/3025016.md) o la [DataBinding](../Proprietes/2510060.md) property antes de llamar: 

		- la función [.fromFile](../WDLang4/1000021387.md).

		- la función [ArchivoPágina](../WDLang4/1000021389.md).




- Los archivos de alias pueden ser manejados por [&lt;Fuente&gt;.ListItem](../WDLang4/1000025017.md) y [&lt;Fuente&gt;.ListKey](../WDLang4/1000025012.md). Los archivos de alias pueden ser re-indexados.

- De forma predeterminada, la **contraseña de un archivo de alias** es idéntica a la contraseña del archivo de datos inicial. Esta contraseña puede ser modificada por [&lt;Fuente&gt;.Password](../WDLang4/1000025044.md). El nombre del archivo de datos utilizado en esta función corresponde al nombre del alias.

- La modificación de los archivos de datos de automatic puede aplicarse a los archivos de alias.

- Los bloqueos realizados en los archivos de datos o en los registros son efectivos entre los alias de la misma instancia de una aplicación.

### Cambiar el directorio o modificar la conexión
<a name="cambiar_directorio_modificar_conexion_ELTPARAGRAPHE000495"></a>

Si se ha utilizado [&lt;Fuente&gt;.ChangeDir](../WDLang4/1000024202.md) o [HConnect](../WDLang4/3044263.md) con el archivo de datos inicial, estos cambios se ignoran para el archivo de datos &lt;Alias&gt;.. 

Una vez creado el alias, le recomendamos que utilice [&lt;Fuente&gt;.ChangeConnection](../WDLang4/1000024199.md) para definir los parámetros de conexión en el archivo de alias.




## Seguridad (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Activa o desactiva el mecanismo de seguridad automática en uno o más archivos de datos.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HSecurity(False) // Disables the protection mechanism
HSecurity(True) // Enables the protection mechanism
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HSecurity([<Data file>, ] <Security level>)`
---

**`<Result>: (Integro)`**

Nivel de seguridad anterior (antes de ejecutar la función **Seguridad**). 

**`<Data file>: (Cadena de caracteres opcional)`**

Nombre del archivo de datos utilizado. 
Si no se especifica este nombre, la función **Seguridad** manipula todos los archivos de datos descritos en el análisis.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) Este parámetro no se tiene en cuenta. La seguridad está habilitada para todos los archivos de datos que se encuentran en el análisis actual.

**`<Security level>: (Entero o booleano)`**

Nivel de seguridad solicitado:

- 0 o <u><u><u><u>False</u></u></u></u> (predeterminado): el mecanismo de seguridad está desactivado. La velocidad de las operaciones de escritura en archivo es máxima.

- 1 o <u><u><u><u>True</u></u></u></u>: El mecanismo de seguridad está activado: La velocidad de las operaciones de escritura a archivo es más lenta que cuando se utiliza HSecurity(0), pero la seguridad está garantizada cuando se escribe en los archivos de datos.

- 2: Mecanismo de máxima seguridad: La velocidad de las operaciones de escritura a archivo es más lenta que cuando se utiliza HSecurity(1).

<a name="NOTE0_1"></a>

## Observaciones


### Seguridad y [HFlush](../WDLang4/3044004.md)
<a name="docparampagetitleshort_hflushwdlang43044004md_ELTPARAGRAPHE000110"></a>

Las funciones **Seguridad** y [HFlush](../WDLang4/3044004.md) permiten forzar la escritura física en el disco.

Sin embargo:

- **Seguridad** obliga a escribir el archivo de datos (s). Esta operación de escritura es realizada automáticamente por el motor HFSQL. Los archivos de datos utilizados no se cierran durante esta operación de escritura. La velocidad de la operación de escritura a archivo es más lenta.

- [HFlush](../WDLang4/3044004.md) se utiliza para forzar una operación de escritura en los archivos de datos manejados durante la llamada a esta función. Los archivos de datos utilizados se cierran y se vuelven a abrir durante esta operación de escritura.. La velocidad de la operación de escritura en archivo no se ralentiza.

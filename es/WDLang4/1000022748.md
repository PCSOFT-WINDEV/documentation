


## &lt;Variable Connection&gt;.ConnectionQuality (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el nivel de calidad de la conexión: entre más alto sea el nivel, más rápida será la conexión.  



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Checks the quality of connection
// Connection1 is a connection with compression
// Connection2 is a connection without compression
Connection1 is Connection
Connection2 is Connection

Result1 is int = Connection1.ConnectionQuality()
Result2 is int = Connection2.ConnectionQuality()
IF Result1 > Result2 THEN
	Trace("The best connection is Connection1")
ELSE
	Trace("The best connection is Connection2")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Connection>.ConnectionQuality()`
---

**`<Result>: (Integer)`**

Nivel de calidad de la conexión. Entre más alto sea el nivel, más rápida será la conexión.

**`<Connection>: (Variable de tipo Connection)`**

Nombre de la [Conexión](../WDLang4/1514073.md) Variable que describe la conexión a probar.

## Observaciones


### ¿Cómo utilizar &lt;Variable Connection&gt;.ConnectionQuality?
<a name="¿como_utilizar_docparampagetitleshort_ELTPARAGRAPHE000206"></a>

**&lt;Variable Connection&gt;.ConnectionQuality** puede utilizarse con tramas comprimidas. 

**¿Cuándo hay que comprimir las tramas de red?**
La elección debe basarse en 3 parámetros: 

- la velocidad de la red utilizada,

- el rendimiento de la CPU de los ordenadores cliente

- el tipo de datos transmitidos. 




La importancia de cada parámetro depende de los otros dos, por lo que no se dispone de una fórmula estándar.. **Sin embargo, aquí hay algunos consejos para ayudarle a elegir**:

- Si la velocidad de la red utilizada es superior a 10 Mbits/segundo, no es necesario comprimir las tramas.

- Si la red utilizada (tipo GRPS, EDGE o 3G) se factura en función de la cantidad de datos transmitidos, le aconsejamos comprimir las tramas para reducir el coste.. 

- Si se transmite una cantidad importante de datos (memo binario) (imágenes, vídeo, sonido zip, etc.), la compresión de los fotogramas será inútil.. 

- En todos los demás casos:

	- en ordenadores cliente equipados con procesadores de alto rendimiento y si su aplicación utiliza datos de texto o de notas de texto, puede ser interesante comprimir los marcos.. 

	- en ordenadores clientes equipados con procesadores de bajo rendimiento, puede ser muy interesante comprimir los fotogramas si la velocidad de transferencia es bastante lenta.


Puede ser interesante realizar mediciones comparativas utilizando el analizador de rendimiento.




**&lt;Variable Connection&gt;.ConnectionQuality** puede utilizarse para comparar el rendimiento de una conexión comprimida y otra sin comprimir.
<a name="NOTE0_2"></a>


### Nivel de calidad de la conexión
<a name="nivel_calidad_conexion_ELTPARAGRAPHE000243"></a>

El nivel de calidad de la conexión depende de varios aspectos, por ejemplo:

- La carga de trabajo de la CPU del cliente y del servidor.

- El uso de los recursos de la red por parte de otros programas.

- La carga de trabajo de los dispositivos de red (switch, router, etc.).

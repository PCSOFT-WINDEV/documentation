


## HOptimize (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite a manipular los periodos de inactividad de un software (periodo sin procesos) para optimizar las consultas y las navegaciones que se ejecutarán a partir de entonces.. Optimiza el acceso a los índices de los archivos de datos HFSQL Classic (los índices de los archivos de datos HFSQL Client/Server no pueden optimizarse). 

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HOptimize(Customer, Name)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HOptimize([<Data file> [, <Key item>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se optimizó el archivo,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. [HErrorInfo](../WDLang4/3044071.md) devuelve más detalles.




**`<Data file>: (Cadena de caracteres opcional)`**

Nombre del archivo de datos HFSQL Classic utilizado. Si no se especifica este parámetro, **HOptimize** utilizará el último archivo de datos utilizado por la última función de gestión HFSQL (función que empieza por "H").

**`<Key item>: (Cadena de caracteres opcional)`**

Nombre del elemento clave a optimizar. Si no se especifica este parámetro, **HOptimize** optimiza todas las claves en el archivo de datos.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Diferencias entre HOptimize y HOptimizeQuery
<a name="diferencias_entre_hoptimize_hoptimizequery_ELTPARAGRAPHE000071"></a>

- **HOptimize** carga las cachés del sistema con el índice completo del archivo de datos. Existe el riesgo de sobrecargar los cachés del sistema si el archivo de datos es muy grande o si la aplicación utiliza varios archivos de datos de gran tamaño. 
	Para evitar el riesgo de overload, evite llamar a HOptimize en un gran número de archivos de datos o elementos clave.

- [HOptimizeQuery](../WDLang4/3044003.md) carga las cachés del sistema sólo con las partes del índice necesarias para la consulta especificada (para todos los archivos de datos utilizados por la consulta). Esta función se utiliza para optimizar las consultas.



<a name="NOTE0_2"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000089"></a>

- **HOptimize** devuelve inmediatamente control al programa y se ejecuta en el background.

- **HOptimize** no tiene efecto en los elementos no clave o en los archivos de datos que no están en formato HFSQL Classic.


## Los diferentes tipos de navegación disponibles en SQL
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000189"></a>
Se pueden utilizar dos tipos de navegación para examinar el resultado de una consulta realizada por [SQLExec](../WDLang4/3072007.md)

- Tipo de iteración: [SQLFirst](../WDLang4/3072017.md)/[SQLCol](../WDLang4/3072018.md)

- Tipo de iteración: [SQLFetch](../WDLang4/3072011.md)/[SQLGetCol](../WDLang4/3072019.md).




Los siguientes párrafos presentan los beneficios y los inconvenientes de estos dos métodos.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Comparación de los dos métodos
<a name="comparacion_los_dos_metodos_ELTTEXTE000213"></a>
El tabla a continuación presenta las características de los dos métodos de navegación:


|   |   |   |
| --- | --- | --- |
|   | **Examinar SQLFirst/SQLCol** | **Examinar SQLFetch/SQLGetCol** |
| **Resultado de la consulta** | Todos los registros encontrados en el resultado de la consulta se cargan en memoria al ejecutar [SQLFirst](../WDLang4/3072017.md).<br>La primera llamada a [SQLFirst](../WDLang4/3072017.md) puede ser bastante larga. | No hay ningún Record cargado en la memoria. El resultado de la consulta es gestionado por la base de datos. |
| **Número de registros** | El número de Record s correspondientes al resultado de la consulta se calcula automáticamente al leer el primer Record ([SQLFirst](../WDLang4/3072017.md)). El Variable SQL.NbLine se actualiza | No se calcula el número de registros correspondiente al resultado de la consulta. |
| **SQLInfo** | [SQLInfo](../WDLang4/3072028.md) devuelve información sobre la consulta y la navegación actual. | [SQLInfo](../WDLang4/3072028.md) devuelve información general sobre la consulta o la conexión. |
| **Navegando por los registros** | Posibilidad de volver a un elemento de resultado anterior | No es posible volver a un elemento de resultado anterior |
|   | El resultado de la consulta es leído por [SQLCol](../WDLang4/3072018.md), [SQLAssociate](../WDLang4/3072004.md). | El resultado de la consulta es leído por [SQLGetCol](../WDLang4/3072019.md) y [SQLGetMemo](../WDLang4/3072020.md). |
| **Visualización en una tabla** | No se debe utilizar [SQLTable](../WDLang4/3072031.md) (error fatal). | [SQLTable](../WDLang4/3072031.md) sólo puede utilizarse para recuperar el resultado Record s del Record actual. |
| **Manejo de tabulaciones** | Este tipo de navegación no se puede utilizar para recuperar las tabulaciones encontradas en los valores de los registros.. Una tabulación separa dos ítems. Sólo la parte encontrada antes de la tabulación será recuperada por las funciones de lectura.. <br>Para más detalles, consulte la ayuda sobre [SQLCol](../WDLang4/3072018.md) y [SQLAssociate](../WDLang4/3072004.md). | Este tipo de navegación se puede utilizar para recuperar las tabulaciones que se encuentran en los valores de los registros.<br>Para obtener más información, consulte la ayuda sobre la función [SQLGetCol](../WDLang4/3072019.md). |



**Ejemplo de exploración SQLFirst/SQLCol**


```wl
ResExec = SQLExec("SELECT CUSTNAME FROM INV","QRY1")
IF ResExec THEN 
	SQLFirst("QRY1") 
	// Transfer the name into LIST_NAMELIST
	WHILE NOT SQL.Out
		ListAdd(LIST_NAMELIST, SQLCol("QRY1",1))
		SQLNext("QRY1")
	END
ELSE
	// ProcessError
END
SQLClose("QRY1")
```


**Ejemplo de exploración SQLFetch/SQLGetCol**


```wl
i is int = 0
SQLExec("SELECT LASTNAME, FIRSTNAME, EXTENSION, PHOTO FROM CUSTOMER", "QRY1")
// Retrieve the query row by row
WHILE SQLFetch("QRY1") = 0 
	// There is still another line
	i++
	// Retrieve the data
	LASTNAME[i] = SQLGetCol("QRY1", 1)
	FIRSTNAME[i] = SQLGetCol("QRY1", 2)
	EXTENSION[i] = SQLGetCol("QRY1",3)
	{"IMAGE"+i} = SQLGetMemo("QRY1", 4)
END
SQLClose("QRY1")
```


## Replicación universal asistida
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000117"></a>
Para establecer fácilmente la replicación universal entre los datos de una aplicación remota y una base de datos central, WINDEV y WEBDEV le permiten

- **crear una réplica** mediante un editor de réplicas (ReplicEdit). Este editor se utiliza para definir las características de una reproducción.

- **implementar una replicación de automatic sobre los datos de una aplicación**. La replicación es realizada directamente por los usuarios finales a través de dos herramientas:

	- ReplicSynchro, utilizado para realizar una replicación.

	- ReplicAdmin, usado para administrar las replicaciones.
			Para obtener más información, consulte [Implementación de una réplica universal asistida por el automatic](../WDLang4/9000046.md).




- **implementa una replicación programada incluida en una aplicación** a través de funciones de WLanguage. Estas funciones se utilizan para replicar una réplica de manipular y para realizar la sincronización..
	Para obtener más información, consulte [Implementación de una replicación universal asistida programada](../WDLang4/9000047.md).




**Observaciones**:

- Ninguna replicación se puede implementar directamente en un equipo sin utilizar una aplicación..  

- Algunas características no están disponibles en esta versión. Para obtener más información, consulte [Características específicas de la replicación universal asistida](../WDLang4/9000048.md). 

- **Gestión de la modificación automática de datos**: La replicación puede tener en cuenta la modificación de datos de automatic para las bases de datos HFSQL. Para obtener más información, consulte [Características específicas de la replicación universal asistida](../WDLang4/9000048.md). 

- La replicación universal asistida está disponible para la replicación de datos móviles (Android o iOS). Para obtener más información, consulte [Replicación de datos móviles (Android o iOS)](../WDLang4/1000021071.md).

## Principio de la replicación universal asistida
<a name="principio_replicacion_universal_asistida_ELTTEXTE000147"></a>
El propósito de la replicación universal asistida es proponer una replicación rápida para cualquier tipo de aplicación.

La replicación universal asistida propone las siguientes modalidades de replicación:

- **reproducción directa**: La replicación se realiza directamente entre la base de datos local y la base de datos maestra.. Este modo de replicación se recomienda cuando un equipo debe conectarse a la red local, por ejemplo. Este modo de replicación no requiere ninguna configuración específica.

- **replicación por servidor**: la replicación se realiza a través de un servidor de replicación. La transferencia de réplicas se gestiona automáticamente.




***La replicación a través del servidor se puede realizar con replica inmediata***: en este caso, los datos se sincronizan inmediatamente. La base de datos de suscriptores envía los datos para replicar, se realiza la replicación y la base de datos de suscriptores recupera los datos para actualizarlos.. Esta operación puede llevar bastante tiempo.. Este modo de replicación se recomienda para una sincronización diaria, por ejemplo (un vendedor sincroniza cada noche con la oficina).

***La replicación por servidor se puede realizar con réplica retardada***: en este caso, los datos se sincronizan cuando los intercambios se realizan con el servidor. Cuando el servidor recibe los datos para replicar, prepara automáticamente los datos para devolverlos. Estos datos serán devueltos durante la próxima sincronización.. Este modo de replicación se recomienda para una sincronización periódica (cuando se realiza un pedido, por ejemplo).

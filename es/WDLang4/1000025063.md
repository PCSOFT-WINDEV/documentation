


## &lt;Fuente&gt;.Security (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Activa o desactiva el mecanismo de seguridad automática en uno o más archivos de datos.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Customer.Security(False) // Disables the protection mechanism
Customer.Security(True) // Enables the protection mechanism
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.Security(<Security level>)`
---

**`<Result>: (Integer)`**

Nivel de seguridad anterior (antes de ejecutar la función **&lt;Fuente&gt;.Security**).

**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos utilizado.

**`<Security level>: (Integer or boolean)`**

Nivel de seguridad solicitado:

- 0 o <u><u><u><u>False</u></u></u></u> (predeterminado): el mecanismo de seguridad está desactivado. La velocidad de las operaciones de escritura en archivo es máxima.

- 1 o <u><u><u><u>True</u></u></u></u>: El mecanismo de seguridad está activado: La velocidad de las operaciones de escritura a archivo es más lenta que cuando se utiliza HSecurity(0), pero la seguridad está garantizada cuando se escribe en los archivos de datos.

- 2: Mecanismo de máxima seguridad: La velocidad de las operaciones de escritura a archivo es más lenta que cuando se utiliza HSecurity(1).






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### &lt;Fuente&gt;.Security y [&lt;Fuente&gt;.Flush](../WDLang4/1000024981.md)
<a name="docparampagetitleshort_flushwdlang41000024981md_ELTPARAGRAPHE000235"></a>

Las funciones **&lt;Fuente&gt;.Security** y [&lt;Fuente&gt;.Flush](../WDLang4/1000024981.md) permiten forzar la escritura física en el disco.

Sin embargo:

- **&lt;Fuente&gt;.Security** obliga a escribir el archivo de datos (s). Esta operación de escritura es realizada automáticamente por el motor HFSQL. Los archivos de datos utilizados no se cierran durante esta operación de escritura. La velocidad de la operación de escritura a archivo es más lenta.

- [&lt;Fuente&gt;.Flush](../WDLang4/1000024981.md) se utiliza para forzar una operación de escritura en los archivos de datos manejados durante la llamada a esta función. Los archivos de datos utilizados se cierran y se vuelven a abrir durante esta operación de escritura.. La velocidad de la operación de escritura en archivo no se ralentiza.

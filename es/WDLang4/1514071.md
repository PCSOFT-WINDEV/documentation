
## Artículo description (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Una variable **Item description** se usa para describir programando uno o más Item s que se encuentran en un archivo de datos. El description de cada artículo es validado por [HDescribeItem](../WDLang4/3044233.md).

Después de esta validación:

- Esta posición se puede tratar como cualquier otra posición descrita en el análisis.

- La variable "Item description" se reinicia y puede ser usada para describir otro archivo Item por medio de la programación.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyItem is Item Description
MyItem1, MyItem2 are Item Descriptions
```


<a name="Example2"></a>



```wl
// Describe the AFile file
...

// Describe the "NAME" item
MyItem..Name = "NAME"
MyItem..Type = hItemText
MyItem..Size = 40
MyItem..KeyType = hUniqueKey

// Validate the description of "NAME" item
HDescribeItem(AFile, MyItem)

// Validate the description of AFile file
...
```

<a name="SYNTAX1"></a>

**Caso especial: Descripción de una clave compuesta**
Al describir una clave compuesta por programación, no es obligatorio utilizar la propiedad **[..Type](../Proprietes/2510131.md)**. Por el contrario, [..KeyExpression](../Proprietes/2512035.md) debe utilizarse para describir la composición de la clave compuesta.

**Caso especial: Describiendo una combinación o enumeración Variable**: 

- Debe llamar a [..TypeDescripción](../Proprietes/1000021596.md) para definir el tipo utilizado por el artículo. 

- Se debe especificar el tipo con la propiedad **[..Type](../Proprietes/2510131.md)** antes de indicar el valor predeterminado con la propiedad **[..DefaultValue](../Proprietes/2512092.md)**.




**Caso especial: Descripción de un elemento de la marca de tiempo**: 

- El artículo debe ser un DateTime, Fecha o Time artículo. 

- Se debe especificar el tipo de marca de tiempo con la propiedad **[..TimestampType](../Proprietes/1000021954.md)**.




**Caso especial: Describir un elemento de Expresión**:: 
Al describir un elemento de tipo Expression por programación, no se debe utilizar la propiedad **[..Type](../Proprietes/2510131.md)**. Debe utilizar [CalculatedItemExpression](../Proprietes/1000021823.md) para describir la expresión de cálculo.

### ¿Cómo describir los archivos de datos por programación?
<a name="¿como_describir_los_archivos_datos_por_programacion_ELTPARAGRAPHE000308"></a>

Para describir los archivos de datos por programación, usted debe:

1. Para cada archivo de datos:

	- Describa las características de del archivo de datos a través del [Propiedades HFSQL](../Proprietes/2512070.md).

	- Describir las características de los artículos a través de la [Propiedades HFSQL](../Proprietes/2512070.md).

	- Valide la description de cada artículo ([HDescribeItem](../WDLang4/3044233.md)).

	- Describa (si es necesario) las características de los índices de texto completo mediante el [Propiedades HFSQL](../Proprietes/2512070.md).

	- Validar el description de cada índice de texto completo ([HDescribeFullTextIndex](../WDLang4/1000017407.md)).

	- Valide el description del archivo de datos ([HDescribeFile](../WDLang4/3044206.md)).




2. Describir (si es necesario) las características de los enlaces a través del [Propiedades HFSQL](../Proprietes/2512070.md).

3. Valide (si es necesario) la description de cada enlace ([HDescribeLink](../WDLang4/3044191.md).

### Manejar los elementos de un archivo de datos descritos por la programación
<a name="manejar_los_elementos_archivo_datos_descritos_por_programacion_ELTPARAGRAPHE000351"></a>

Cuando se crea un archivo de datos en el editor de análisis, el compilador reconoce automáticamente los nombres de los archivos y elementos de datos.

Cuando se crea un archivo de datos a través de una variable "File description", los nombres del archivo de datos y los elementos son definidos por [..Nombre](../Proprietes/2510082.md). Estos nombres no son reconocidos automáticamente por el compilador. Se produce un error de compilación si estos nombres se utilizan para identificar a el archivo de datos o los artículos.

Para que el compilador reconozca los nombres del archivo de datos y los elementos descritos por la programación, el nombre de este archivo de datos debe ser declarado como [origen de los datos](../WDLang4/1514053.md). Entonces, este archivo de datos y sus elementos pueden ser manejados por su nombre en el editor de código.

Por ejemplo:


```wl
AFile is File Description
...
AFile..Name = "CUSTOMER"
...
HDescribeFile("CUSTOMER")
...
CUSTOMER is Data Source
...
HReadFirst(CUSTOMER, "CUSTNAME")
```


**Observación**: El nombre del archivo de datos también puede ser declarado con la palabra clave [Externo](../Motscles/1511013.md). Sin embargo, la velocidad de ejecución de Process será más lenta.
<a name="NOTE0_4"></a>




### Registro
<a name="registro_ELTPARAGRAPHE000378"></a>

Cuando el archivo de datos es descrito por la programación:

- si se registra el archivo de datos y si no se registra ningún elemento, todos los elementos (excepto las claves compuestas) se registran automáticamente.

- si se registra el archivo de datos y si al menos un elemento se registra, los otros elementos no se modifican.

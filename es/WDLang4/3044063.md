


## HRecNum (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve:

- el número del actual Record en el archivo de datos HFSQL.

- el número de la actual Record en la vista HFSQL.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003044063&name=hrecnum_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Current record in the "Customer" data file
CurrentCustomerRec = HRecNum(Customer)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HRecNum([<Data file>])`
---

**`<Result>: (Integro)`**

Corresponde a:

- el número de la actual Record en el archivo de datos.

- el número del Record actual en la vista o en la consulta (y no el número del Record actual en el archivo de datos). 
	**Atención**: Utilizado en una vista HFSQL Classic, este parámetro no corresponde al valor del [H.NumRec](../WDLang4/3044076.md) Variable.

- la constante *hInvalidRecNum* si no hay Record actual (posición fuera de el archivo de datos).




La función [HError](../WDLang4/3044088.md) permite obtener más detalles sobre el problema.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) Cuando se utiliza en una vista de Hyper File 5.5, **HRecNum** devuelve el número del registro actual en el archivo de datos associated con la vista.

**`<Data file>: (Cadena de caracteres opcional)`**

Nombre del archivo de datos HFSQL, consulta o vista utilizada. 
Si no se especifica este parámetro, **HRecNum** utilizará el último archivo de datos utilizado por la última función de gestión HFSQL (función que empieza por "H").

<a name="NOTE0_1"></a>

## Observaciones


### Archivos de datos no HFSQL
<a name="archivos_datos_hfsql_ELTPARAGRAPHE000109"></a>

- No hay ningún número de Record en los archivos de datos.

- La llamada a esta función desencadena un error WLanguage.

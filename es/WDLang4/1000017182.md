
## HFSQL Client/Server: Principio para guardar las bases de datos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000157"></a>
Como para cualquier base de datos, las copias de seguridad de sus bases de datos HFSQL Client/Server deben realizarse regularmente. 

Esta copia de seguridad se puede realizar mientras ***la base de datos está siendo utilizada***. En este caso, la copia de seguridad es una copia de seguridad "caliente".. Los accesos pueden estar en curso (lectura, escritura, ejecución de consultas, etc.). La copia de seguridad corresponderá al estado de la base de datos cuando se inicie la copia de seguridad.

Existen varios tipos de copias de seguridad disponibles: 

- copia de seguridad completa con o sin copia de seguridad diferencial.

- copia de seguridad inmediata o programada. 




Se puede activar una copia de seguridad:

- en el Centro HFSQL control.

- mediante programación.


Para obtener más información, consulte [¿Cómo realizar una copia de seguridad?](../WDLang4/1000017308.md). 

Una copia de seguridad puede ser restaurada en cualquier momento. Para obtener más información, consulte [¿Cómo restaurar una copia de seguridad?](../WDLang4/1000017309.md).

## Los diferentes tipos de copias de seguridad
<a name="los_diferentes_tipos_copias_seguridad_ELTTEXTE000187"></a>


### Copia de seguridad completa y copia de seguridad diferencial
<a name="copia_seguridad_completa_copia_seguridad_diferencial_ELTPARAGRAPHE000051"></a>

**Una copia de seguridad completa** consiste en guardar completamente los datos encontrados en un servidor, en una base de datos, en un archivo, ... Este tipo de copia de seguridad requiere mucho espacio.  

También puede realizar copias de seguridad **diferential backups**. La copia de seguridad diferencial se utiliza para guardar las secciones de archivo modificadas desde la copia de seguridad completa anterior.. Por lo tanto, el tamaño de las copias de seguridad se reduce considerablemente y la velocidad de las copias de seguridad aumenta.. 

El método común consiste en realizar copias de seguridad completas en fechas fijas y en realizar copias de seguridad diferenciales entre estas fechas fijas.. Las frecuencias dependen de la velocidad de evolución de los datos y de la importancia de los mismos.. Una frecuencia correcta consiste en realizar una copia de seguridad mensual completa y una copia de seguridad diferente cada noche.. 
<a name="NOTE2_2"></a>


### Copias de seguridad inmediatas y programadas
<a name="copias_seguridad_inmediatas_programadas_ELTPARAGRAPHE000062"></a>

El **inminente** las copias de seguridad se utilizan para realizar una copia de seguridad en un momento dado. Sólo se puede realizar una copia de seguridad completa de forma inmediata. 

Las copias de seguridad de ** programadas por** se usan para implementar una política de copias de seguridad de automatic definiendo la frecuencia de las copias de seguridad.. Las copias de seguridad completas y las copias de seguridad diferenciales se pueden programar.

## Observaciones
<a name="observaciones_ELTTEXTE000217"></a>




### Características de la copia de seguridad
<a name="caracteristicas_copia_seguridad_ELTPARAGRAPHE000075"></a>

La copia de seguridad realizada corresponderá al estado de la base de datos al inicio de la copia de seguridad.

Durante la copia de seguridad, no se le permite hacerlo:

- eliminar archivos y bases de datos,

- actualizar los procedimientos almacenados.   



<a name="NOTE3_2"></a>




### Notas
<a name="notas_ELTPARAGRAPHE000088"></a>

- Una copia de seguridad completa se puede realizar inmediatamente o se puede programar. 

- Una copia de seguridad diferencial sólo se puede programar. 

- Le aconsejamos que realice las copias de seguridad cuando la carga del servidor sea ligera.. De hecho, la copia de seguridad utiliza el disco duro y puede ralentizar el servidor.

- Si la copia de seguridad es una copia de seguridad programada y el servidor está apagado cuando se debe realizar la copia de seguridad, la copia de seguridad se realizará en la siguiente fecha especificada, cuando el servidor esté disponible.

- Realizar una copia de seguridad y restaurarla con un nombre diferente le permite duplicar fácilmente una base de datos.. 



<a name="NOTE3_3"></a>




### Derechos de Windows necesarios para realizar copias de seguridad
<a name="derechos_windows_necesarios_para_realizar_copias_seguridad_ELTPARAGRAPHE000100"></a>

La copia de seguridad de una base de datos de HFSQL Client/Server es ejecutada por el Process "manta.exe" (Visible en el administrador de servicios de Windows y llamado "Hyper File Server: NombreServidor").

Por Default, este servicio está gestionado por la "cuenta de sistema local" de Windows, cuyos derechos pueden ser limitados. 

Si una copia de seguridad falla con un error como "Acceso denegado" o "No se puede crear el directorio", se debe modificar la cuenta utilizada para ejecutar el servicio:

1. Abra el administrador de servicios de Windows. 

2. Haga clic con el botón derecho del ratón en "Hyper File Server : ServerName" y seleccione "Properties".. 

3. En el panel "Conexión", sustituya la "Cuenta del sistema local" por un usuario que tenga los derechos de acceso a la ubicación de la copia de seguridad.




## HTransactionIsolation (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Configura el modo de aislamiento de las transacciones para una conexión a un servidor HFSQL.

Dependiendo del modo seleccionado, una aplicación que realice operaciones de lectura tendrá acceso o no a los datos que estén siendo modificados en una transacción por otra aplicación, u otra instancia de la misma aplicación.

Para obtener más información, consulte [Transacciones Cliente/Servidor: Modos de aislamiento disponibles](../WDLang4/1000017316.md). 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyConnection is Connection
HTransactionIsolation(MyConnection, hReadCommitted)
HReadFirst(MyCustomerFile, MyKey)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HTransactionIsolation(<Connection> , <Isolation mode>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se estableció el nivel de aislamiento,

- <u><u><u><u>False</u></u></u></u> si se produce un error. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión a un servidor HFSQL. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).




**`<Isolation mode>: (Constante de tipo Integer)`**

Modo de aislamiento seleccionado:


|   |   |
| --- | --- |
| *hReadCommitted* | Los demás usuarios sólo pueden ver las transacciones realizadas. |
| *hReadUncommitted* | Otros usuarios pueden ver las transacciones incompletas. |
| *hRepeatableRead* | La aplicación en la transacción ve una instantánea de la base de datos cuando se inicia la transacción. |

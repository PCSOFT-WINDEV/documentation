


## FileToPage (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Actualiza automáticamente los controles de una página con: 

- los valores de los elementos vinculados en el Record actual (cargado en memoria) del archivo de datos descritos en el editor de análisis. Esta operación se realiza independientemente del estado de los mandos (grises, inactivos o Invisible).

- los valores de las variables vinculadas de la lengua WL. Esta operación se realiza independientemente del estado de los mandos (grises, inactivos o Invisible).




Observación: **FileToPage** no lee registros en el archivo de datos. El valor de la Record cargado en memoria se utiliza para actualizar los controles. Por lo tanto, hay que cargar previamente un Record en memoria (con [HRead](../WDLang4/3044047.md), [HReadSeek](../WDLang4/3044050.md), [HReadFirst](../WDLang4/3044051.md), etc.).

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HReadSeekFirst(CUSTOMER, NAME, "PAUL")
IF HFound() = True THEN FileToPage()

// Read the record and transfer the items to the controls
HReadFirst(CUSTOMER, NAME)
FileToPage()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`FileToPage([<Page> [, <Source>]])`
---

**`<Page>: (Nombre de la página)`**

Nombre de la página donde se deben actualizar los controles con los valores de la Record actual en el archivo de datos. Si este parámetro no se especifica o es una cadena vacía (""), se utiliza la página del código actual.


**`<Source>: (Cadena de caracteres opcional)`**

Nombre de la fuente a la que están vinculados los controles que se van a actualizar. Esta fuente puede ser:

- El nombre de un archivo de datos: sólo se tienen en cuenta los controles vinculados a los elementos del archivo de datos.

- El nombre de un WLanguage Variable: sólo se tienen en cuenta los controles vinculados a la lengua de trabajo Variable.
	Observaciones: 

	- Si el nombre de la Variable está entre comillas, el control se vinculará a la Variable especificada. 

	- Si el nombre de la Variable no está entre comillas, el control se vinculará a los datos de la Variable (por ejemplo, una Variable que contenga el nombre de otra Variable).





Si no se especifica este nombre, se tendrán en cuenta todos los controles de página vinculados a un elemento de un archivo de datos descrito en la editor de análisis o a un WLanguage Variable.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Vinculación entre un control y un WLanguage Variable o entre un control y un elemento de un archivo de datos
<a name="vinculacion_entre_control_wlanguage_variable_entre_control_elemento_archivo_datos_ELTPARAGRAPHE000086"></a>

Se puede determinar la vinculación entre un control y un WLanguage Variable o entre un control y un elemento de un archivo de datos:

- En el editor, en la ventana de control description (pestaña "Binding").

- De forma programada, con el programa [DataBinding](../Proprietes/2510060.md) Property.



<a name="NOTE0_2"></a>


### Vinculación a un elemento: tipos de enlace
<a name="vinculacion_elemento_tipos_enlace_ELTPARAGRAPHE000101"></a>

Existen dos tipos de vínculos entre una control y un artículo

- *vinculación directa*: En este caso, **FileToPage** transfiere el valor de los elementos del Record cargado en memoria al control vinculado, sin leer ningún valor.

- La vinculación de *mediante la lectura de los registros del archivo de datos associated*.

### Varios
<a name="varios_ELTPARAGRAPHE000117"></a>

- Los caracteres de espacio en los elementos HFSQL se mantienen.

- **FileToPage** actualiza los controles Image vinculados a los elementos de la memoria Image.

- **FileToPage** no está disponible en vistas. Los valores de los elementos deben asignarse a cada control.

- Vinculación de **con los elementos de "Contraseña segura"**:

	- En una edición de control vinculada a un elemento de contraseña, aparecen los caracteres "\*\*\*\*\*".. Si la edición de control es del tipo "Contraseña", se muestran puntos en su lugar. 

	- El valor del elemento "Contraseña" no puede volver a leerse hasta que el usuario cambie el valor de la contraseña.

	- El valor del elemento "Contraseña" puede seguir comparándose con una cadena (por ejemplo, "" o "1234").




## &lt;Fuente&gt;.Open (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Abre un archivo de datos.

**Observación**: La apertura de los archivos de datos se gestiona automáticamente. Sin embargo, en algunos casos, es posible que tenga que abrir explícitamente un archivo de datos (para comprobar la existencia del archivo o la validez de la contraseña, por ejemplo).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Open Customer data file
Customer.Open()
```
<a name="Example3"></a>

```wl
// Check status report of <source>.Open
IF Customer.Open("ABCD", hORead) = False THEN
	Error("Unable to open the Customer data file" + HErrorInfo())
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.Open([<Password> [, <Access mode>]])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si el archivo de datos fue abierto.

- <u><u><u><u>False</u></u></u></u> si se produce un problema (contraseña incorrecta, etc.). La función [HError](../WDLang4/3044088.md) permite identificar el error. La función [HErrorInfo](../WDLang4/3044071.md) devuelve más detalles.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos utilizado (nombre lógico, definido en el análisis). 

**`<Password>: (Cadena de caracteres opcional)`**

Contraseña del archivo de datos. 
Si no hay contraseña es associated con el archivo de datos, tienes la posibilidad de usar una cadena vacía ("").

**`<Access mode>: (Constante opcional de tipo Integer)`**

Configura el modo de acceso a el archivo de datos. 
Si no se especifica este parámetro, el archivo de datos se abrirá según sus características (si el archivo de datos es de sólo lectura, se abrirá en modo de sólo lectura; si el archivo de datos es de lectura/escritura, se abrirá en modo de lectura/escritura).


|   |   |
| --- | --- |
| *hORead* | Se puede acceder al archivo de datos en modo de solo lectura. Solo se pueden leer registros. No se pueden agregar o eliminar. |
| *hOReadWrite* | Se puede acceder al archivo de datos en modo de lectura y escritura. Se pueden leer y escribir nuevos registros. |


![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) Este parámetro no se tiene en cuenta.

<a name="NOTE0_1"></a>

## Observaciones


### Gestión de la contraseña
<a name="gestion_contrasena_ELTPARAGRAPHE000304"></a>

Se puede especificar la contraseña:

- cuando el archivo de datos se abre con **&lt;Fuente&gt;.Open**,

- antes de abrir el archivo de datos con [&lt;Fuente&gt;.Password](../WDLang4/1000025044.md).



<a name="NOTE0_2"></a>


### Creando o abriendo un archivo de datos
<a name="creando_abriendo_archivo_datos_ELTPARAGRAPHE000324"></a>

La función [&lt;Fuente&gt;.CreationIfNotFound](../WDLang4/1000024208.md) puede reemplazar a la función **&lt;Fuente&gt;.Open**. De hecho, si el archivo de datos existe, se abre automáticamente.
<a name="NOTE0_3"></a>


### La apertura de un archivo de datos que ya está abierta
<a name="apertura_archivo_datos_que_esta_abierta_ELTPARAGRAPHE000341"></a>

Si se utiliza **&lt;Fuente&gt;.Open** sobre un archivo de datos ya abierto con un &lt;modo de acceso&gt; diferente, se pierden los filtros, bloqueos y características de la iteración actual.
<a name="NOTE0_4"></a>


### Gestión de la lista de archivos de datos utilizados (.REP)
<a name="gestion_lista_archivos_datos_utilizados_rep_ELTPARAGRAPHE000353"></a>

WINDEV Mobile le permite actualizar una lista de archivos de datos utilizados por la aplicación (equivalente al archivo.REP de WINDEV).

**&lt;Fuente&gt;.Open** especifica la lista de archivos de datos utilizados por la aplicación (.REP para WINDEV y WEBDEV) si se habilitan las actualizaciones para esta lista.

Para obtener más información, consulte [Lista de archivos de datos manipulados por una aplicación](../WDLang4/3044045.md).
<a name="NOTE0_5"></a>


### Gestión de errores fatales
<a name="gestion_errores_fatales_ELTPARAGRAPHE000372"></a>Si se produce un error fatal al abrir un archivo de datos, puede ser interesante continuar el programa mientras se procesa el error. Para ello, le recomendamos que realice la siguiente excepción: Process:

```wl
WHEN EXCEPTION IN
	Parent.Open()
DO
	Error("HFSQL error: " + HError())
ELSE
	Trace("File opened")
END
```

### Apertura de una conexión y la tabla especificada
<a name="apertura_una_conexion_tabla_especificada_ELTPARAGRAPHE000393"></a>

Si es necesario, **&lt;Fuente&gt;.Open** abre la conexión a la base de datos antes de abrir la tabla especificada. Si la conexión ya está abierta (con los mismos parámetros), esta conexión se utiliza automáticamente.

Observación: La conexión utilizada por defecto corresponde a la conexión associated con el tabla en el editor de análisis.

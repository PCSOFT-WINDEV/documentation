
## HConvertX (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Convierte un valor numérico en una cadena binaria para realizar una búsqueda en una clave numérica. Se utiliza para construir una clave compuesta a partir de uno o más elementos numéricos.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=HConvertX%20-%20HC%20N%B0001.gif)


Para los archivos de datos que utilizan el "Modo 7" para gestionar espacios en elementos de texto, se debe utilizar la función [HBuildKeyValue](../WDLang4/3044258.md) para navegar por las claves compuestas. Para obtener más información, consulte [Claves compuestas](../WDLang4/3044178.md).


|   |
| --- |
| **Advertencia**<br><br>Estas funciones se conservan por motivos de compatibilidad para migrar aplicaciones desarrolladas en WINDEV 5.5 y WEBDEV 1.5. |


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// CompKey is made of PRODREF (text) and QTY (integer)
// EDT_Quantity and EDT_ProductRef are two edit controls
Srch is string
HConvertE(Search, EDT_Quantity)
Srch = Complete(EDT_ProductRef, 8) + Srch
HReadSeek(Orders, CompKey, Srch)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HConvertX(<Text Variable> , <Numeric Value>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si no se ha encontrado ningún problema,

- <u><u><u><u>False</u></u></u></u> si se produce un error. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Text Variable>: (Cuerda de caracteres de Ansi Variable)`**

Variable a la que se asignará el resultado de la conversión. Esta variable debe tener un tamaño suficiente y se completará con espacios.

**`<Numeric Value>: (Variable numérica o valor numérico)`**

Valor a convertir.

## Observaciones




### Sintaxis según los valores a convertir
<a name="sintaxis_segun_los_valores_convertir_ELTPARAGRAPHE000063"></a>


|   |   |
| --- | --- |
| HConvertC(&lt;string&gt;,&lt;Short integer&gt;) |   |
| HConvertD(&lt;string&gt;,&lt;Double real&gt;) |   |
| HConvertE(&lt;string&gt;,&lt;Integer&gt;) | Para compatibilidad con WEBDEV 1.5, esta función manipula integer_55 en 2 bytes. |
| HConvertL(&lt;string&gt;,&lt;Long integer&gt;) | Se recomienda utilizar la función [HBuildKeyValue](../WDLang4/3044258.md). |
| HConvertM(&lt;string&gt;,&lt;Currency&gt;) |   |
| HConvertP(&lt;string&gt;,&lt;Turbo real&gt;) | Para la compatibilidad con versiones anteriores, esta función es idéntica a la HConvertD. |
| HConvertR(&lt;string&gt;,&lt;Simple real&gt;) |   |
| HConvertU(&lt;string&gt;,&lt;Unsigned integer&gt;) | Para la compatibilidad con versiones anteriores, esta función maneja el entero_55 en 2 bytes. |
| HConvertV(&lt;string&gt;,&lt;Signed 1-byte integer&gt;) |   |

**Consejo**: Se recomienda utilizar [Convertir](../WDLang4/3044244.md) en lugar de **HConvertX** ya que es mucho más fácil de manipular.


### Uso de la función HConvertX (método conservado por motivos de compatibilidad con WINDEV 5.5)
<a name="uso_funcion_docparampagetitleshort_metodo_conservado_por_motivos_compatibilidad_con_windev_55_ELTPARAGRAPHE000103"></a>

Para construir el valor de una clave compuesta sin utilizar la función [HBuildKeyValue](../WDLang4/3044258.md), es necesario:

- completar los componentes de texto con la constante *hMinVal*.

- convertir los componentes numéricos con la función [HConvert](../WDLang4/3044260.md).




Ejemplo:


```wl
MyCompositeKey = Complete(Customer.LastName, Dimension(Customer.LastName), hMinval) + ...
		Complete(Customer.FirstName, Dimension(Customer.FirstName), hMinVal)
```


Observaciones:

- La constante *hMinVal* equivale a Charact(0).

- La constante *hMaxVal* equivale a Charact(255).




Para más detalles sobre cómo utilizar las teclas compuestas en los filtros, véase [Claves compuestas y filtros](../WDLang4/3044178.md).

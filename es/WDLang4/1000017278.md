
## HReconnect( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Establece una reconexión al servidor para todas las conexiones interrumpidas. Las conexiones ininterrumpidas no se modifican.  

**Observaciones**:

- Para utilizar esta característica, el servidor HFSQL y la aplicación cliente deben estar en la versión 14 o posterior.

- Por Default, cuando se produce un error de conexión, el mecanismo de gestión asistida de errores muestra una ventana o una página que permite al usuario volver a conectarse. Este mecanismo puede ser personalizado por [HOnError](../WDLang4/3044017.md).  











<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HReconnect()`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la reconexión,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HErrorInfo](../WDLang4/3044071.md) permite obtener más detalles sobre el problema.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Reconexión al servidor
<a name="reconexion_servidor_ELTPARAGRAPHE000083"></a>

Una aplicación puede perder la conexión con el servidor por una de las siguientes razones:

- pérdida del enlace GPRS.

- intervención en la red.

- actualización del servidor.

- del servidor.

- en espera del ordenador cliente.




La función **HReconnect** le permite al usuario volver a conectarse al servidor sin reiniciar la aplicación.
<a name="NOTE0_2"></a>




### Límites
<a name="limites_ELTPARAGRAPHE000101"></a>

La reconexión no se puede realizar en los siguientes casos: 

- La aplicación incluye una transacción en curso. De hecho, la transacción fue cancelada automáticamente por el servidor cuando el cliente fue desconectado.

- La aplicación ha bloqueado todo el archivo de datos.

- La aplicación ha realizado un bloqueo de "lectura/escritura" en un archivo de datos.

- La aplicación ha realizado un bloqueo en "modo de escritura" en un archivo de datos y otra aplicación ha modificado este Record mientras la aplicación no estaba conectada.

- La aplicación realizó un bloqueo en modo de escritura con [HFirst](../WDLang4/3044113.md), [HNext](../WDLang4/3044027.md), [HPrevious](../WDLang4/3044112.md) o [HLast](../WDLang4/3044101.md).

- Una vista o una consulta está actualmente llena y se ha iniciado una búsqueda en esta vista o consulta.




En estos casos, el valor de &lt;Resultado&gt; es <u><u><u><u>False</u></u></u></u>.

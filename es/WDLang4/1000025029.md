


## &lt;Fuente&gt;.Track (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia la supervisión de acciones de escritura (agregar, modificar o eliminar) realizadas en un archivo de un servidor HFSQL. En caso de modificación, se puede disparar un Procedure específico. 

Esta pista puede ser detenida por [&lt;Fuente&gt;.TrackStop](../WDLang4/1000025030.md). 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Starts tracking the Order data file
// The ProcessOrder procedure is called whenever a record is added to the data file
Order.Track(ProcessOrder, hRecNumAll, hsAdd)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = <Source>.Track(<WLanguage procedure> [, <Record number> [, <Action to track> [, <Runtime mode>]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se activó el seguimiento,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos HFSQL a utilizar. El archivo de datos es associated con una conexión y con un servidor. 

**`<WLanguage procedure>: (Nombre de Procedure)`**

Nombre del WLanguage Procedure ("callback") llamado cuando se realiza un cambio en el archivo de datos especificado. 
Para obtener más información sobre este procedimiento, consulte [Parámetros del procedimiento utilizado por la función &lt;Fuente&gt;.Track](../WDLang4/1000024804.md).

**`<Record number>: (Entero opcional)`**



- Número de la Record a rastrear. 

- *hRecNumAll* constante para rastrear todos los registros que se encuentran en el archivo de datos (lo que significa todo el archivo de datos).




**`<Action to track>: (Constante o combinación de constantes opcional)`**

Acciones de seguimiento para las que se iniciará el Procedure:  


|   |   |
| --- | --- |
| *hsAdd* | Monitorea las operaciones para agregar registros. |
| *hsCross* | Supervisa las operaciones de cruce de registros. |
| *hsDeletion* | Supervisa las operaciones de borrado de registros.. |
| *hsModification* | Supervisa las operaciones de modificación de registros. |



**`<Runtime mode>: (Constante opcional)`**

Modo de ejecución de WLanguage Procedure:  



|   |   |
| --- | --- |
| *hSecondaryThread* | Ejecutar el lenguaje Procedure en un Thread secundario. |


Por defecto, el WLanguage Procedure se ejecuta en el thread principal.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### WLanguage Procedure ejecutado en el Thread principal
<a name="wlanguage_procedure_ejecutado_thread_principal_ELTPARAGRAPHE000272"></a>

Por defecto, este WLanguage Procedure es llamado en el hilo principal de la aplicación, con el contexto HFSQL que corresponde al Procedure:

- Si el Procedure es un Procedure local de una ventana que utiliza un contexto independiente, se utilizará el contexto de la ventana. 

- Si la Procedure pertenece a un componente que utiliza un contexto independiente, se utilizará el contexto del componente.





|   |
| --- |
| **Atención**:<br>El procedimiento asociado a la función **&lt;Fuente&gt;.Track** se puede llamar en cualquier momento en la aplicación.<br>Si el Procedure lee registros o se mueve entre registros en los archivos de datos, esto tendrá un impacto en la aplicación (como los temporizadores).<br>Por lo tanto, puede ser necesario utilizar: <br><br>- [&lt;Fuente&gt;.SavePosition](../WDLang4/1000025062.md) y [&lt;Fuente&gt;.RestaurarPosición](../WDLang4/1000025061.md) para guardar los contextos de los archivos utilizados.<br><br>- una Variable de tipo [Alias](../WDLang4/1000021446.md) para realizar operaciones en archivos de alias. <br><br><br>[Multitask](../WDLang1/3015004.md) no debe utilizarse en el código Procedure. |


Para obtener más información sobre este procedimiento, consulte [Parámetros del procedimiento utilizado por la función &lt;Fuente&gt;.Track](../WDLang4/1000024804.md).
<a name="NOTE0_2"></a>


### Lenguaje Procedure ejecutado en un Thread secundario
<a name="lenguaje_procedure_ejecutado_thread_secundario_ELTPARAGRAPHE000326"></a>

Si el lenguaje WL Procedure se llama en un thread secundario: 

- el contexto HFSQL se copia la primera vez que se llama a **&lt;Fuente&gt;.Track**. Solo se almacenan los directorios que contienen los archivos de datos en modo HFSQL Classic y/o las conexiones en modo HFSQL Client/Server.

- Los elementos de la interfaz de usuario (ventanas, controles, etc.) no pueden ser manipulados en la Thread secundaria. Cuando un Thread secundario debe interactuar con el usuario o actualizar la UI, debe utilizar un Process iniciado desde el Thread principal. Este proceso puede corresponder a:

	- un procedimiento global del proyecto o un procedimiento local (de una ventana, etc.) llamado con la función [ExecuteMainThread](../WDLang1/1000019862.md),

	- el evento "Solicitud para actualizar la visualización" de una ventana, ejecutado con la función [RequestRefreshUI](../WDLang1/1000023899.md).







Para obtener más información sobre este procedimiento, consulte [Parámetros del procedimiento utilizado por la función &lt;Fuente&gt;.Track](../WDLang4/1000024804.md).
<a name="NOTE0_3"></a>


### Casos especiales
<a name="casos_especiales_ELTPARAGRAPHE000356"></a>

La pista está definida para un archivo de datos y una conexión: 

- Si el nombre físico o la conexión asociada al nombre lógico del archivo de datos cambia, el archivo supervisado por el servidor sigue siendo el que se especificó al llamar a la función **&lt;Fuente&gt;.Track**. 

- Si la conexión está cerrada, la pista se detiene automáticamente.

- El seguimiento continúa si se pierde la conexión y luego se restablece con [Reconexión de automatic](../WDLang4/3044188.md).




Para dejar de seguir a el archivo de datos, utilice [&lt;Fuente&gt;.TrackStop](../WDLang4/1000025030.md). 
Las funciones **&lt;Fuente&gt;.Track** y [&lt;Fuente&gt;.TrackStop](../WDLang4/1000025030.md) se pueden utilizar en un servicio de Windows.

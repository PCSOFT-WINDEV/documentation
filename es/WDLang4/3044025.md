


## HToItem (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Asigna el valor especificado a un campo del registro actual.

**Observación**: Esta función se utiliza muy pocas veces. Se recomienda asignar directamente los ítems con la siguiente sintaxis: &lt;NombreDeArchivo&gt;.&lt;NombreDeItem&gt; = &lt;Valor&gt;. 
Por ejemplo: Apellido del cliente = "Smith".

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Assigns "Smith" to the second item of the Customer data file
HToItem(Customer, 2, "Smith")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Asignar un valor a un campo (un valor por cada campo de un campo array)

`HToItem(<Data file> , <Item order> [, <Index>] , <Value>)`
---

**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado.

**`<Item order>: (Integro)`**

Número de orden físico del artículo en el archivo de datos.

**`<Index>: (Entero opcional)`**

Índice del elemento de array a asignar. Si se asigna un elemento no array, &lt;Index&gt; debe ser set a 1.

**`<Value>: (Cadena de caracteres u otro)`**

Valor que se debe asignar a la posición. Si el valor debe asignarse a una posición de array: 

- si se utilizó &lt;Index&gt;, el valor se asignará al elemento indexado especificado. 

- si no se ha utilizado ninguno de los parámetros anteriores, &lt;Valor&gt; debe contener todos los valores del elemento array, separados por TAB.


![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) Las posiciones de array no se pueden asignar.


<a name="SYNTAX2"></a>

### Asignar un valor a un campo (un valor para un campo array)

`HToItem(<Data file> , <Item order> [, <Array separator>] , <Value>)`
---

**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado.

**`<Item order>: (Integro)`**

Número de orden físico del artículo en el archivo de datos. 

**`<Array separator>: (Cadena de caracteres opcional)`**

String utilizado para separar los valores de cada elemento array. TAB es usado por Default.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) Las posiciones de array no se pueden asignar.

**`<Value>: (Cadena de caracteres u otro)`**

Valor que se debe asignar a la posición. Si el valor debe asignarse a una posición de array: 

- si se ha utilizado &lt;Array separator&gt;, &lt;Value&gt; debe contener todos los valores del elemento array, separados por el separador especificado

- si no se ha utilizado ninguno de los parámetros anteriores, &lt;Valor&gt; debe contener todos los valores del elemento array, separados por TAB.


![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) Las posiciones de array no se pueden asignar.

## Observaciones


### Orden de los artículos en el archivo de datos
<a name="orden_los_articulos_archivo_datos_ELTPARAGRAPHE000117"></a>

El orden de los artículos en el archivo de datos comienza desde el 1. El pedido de un artículo es devuelto por la [Índice](../Proprietes/2512040.md) Property.

### Lenguaje externo
<a name="lenguaje_externo_ELTPARAGRAPHE000131"></a>

- &lt;El valor&gt; debe ser pasado en formato de cadena.

- **HToItem** no asigna valores a las partidas binarias de la memoria.

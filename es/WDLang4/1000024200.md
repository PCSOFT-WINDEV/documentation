


## &lt;Fuente&gt;.ChangeLocation (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica el modo de búsqueda de los archivos de datos físicos (.FIC, .NDX, etc.). Por defecto, los archivos de datos físicos se buscan en la librería de la aplicación y luego en el disco.. 

Recordatorio: Los archivos de datos físicos se pueden encontrar:

- en el disco (caso más común).

- en librerías WINDEV (archivos WDL).

- en el ejecutable.

- en un componente.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Find data files in libraries 4, 6 and 9
CustomerFile.ChangeLocation(hWDL, 4, 6, 9)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.ChangeLocation(<Search mode> [, <Parameters>])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.


![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) Si se utiliza la función **&lt;Fuente&gt;.ChangeLocation**, &lt;Resultado&gt; = <u><u><u><u>True</u></u></u></u>, pero la función no tiene ningún efecto: los archivos de datos utilizados siempre están en el servidor.

**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos (definido en el análisis). 
**Atención**: los archivos de datos correspondientes se cierran automáticamente. 

**`<Search mode>: (Constante)`**

Configura el modo de búsqueda de los archivos de datos:


|   |   |
| --- | --- |
| *hDisk* | Los archivos de datos solo se buscan en el disco. Este es el caso más común. |
| *hWDL*<br>(Valor predeterminado) | Los archivos de datos se buscan:<br><br>1. En la WDL principal; <br><br>2. Si no se especifica &lt;Parámetros&gt;: Busca los archivos de datos en las WDL según el orden de carga y luego en el disco. <br>	Si se especifica &lt;Parámetros&gt;: Busca los archivos de datos en las WDL especificadas, y luego en el disco. <br><br>3. En la WDL principal del componente. <br><br>4. En las WDL secundarias del componente según el orden de carga.<br><br><br> |



**`<Parameters>: (Cadena de caracteres opcional)`**

Identificador de las WDL en las que se debe realizar la búsqueda. Estas WDL se deben cargar con la función [LoadWDL](../WDLang1/3013020.md), que devuelve el identificador de la WDL.

## Observaciones


### ¿Por qué incluir los archivos de datos en una librería?
<a name="¿por_que_incluir_los_archivos_datos_una_libreria_ELTPARAGRAPHE000260"></a>

Incluir los archivos de datos en una librería tiene las siguientes ventajas: 

- protección de datos: solo se puede acceder a los archivos de datos en modo de lectura. 

- agrupamiento de datos: todos los archivos necesarios para la ejecución se agrupan en un solo archivo.




## &lt;Fuente&gt;.ChangeConnection (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cambia dinámicamente la conexión asociada a un archivo de datos. La nueva conexión debe conocerse y definirse durante este cambio.. Esta nueva conexión se utilizará durante la próxima apertura de del archivo de datos.

Esto le permite importar un description de tablas de Oracle en el análisis y usar este description para trabajar en archivos de datos en formato HFSQL. Para más detalles, consulte los ejemplos detallados.

**Consejos**: 

- Para las conexiones manipular, se recomienda utilizar un Variable de tipo [Conexión](../WDLang4/1514073.md).

- **Atención**: El orden en el que se llama a las funciones para conectarse a una base de datos HFSQL Client/Server es importante. Para obtener más información, consulte [Optimizar el tiempo de conexión a las bases de datos HFSQL Client/Server](../WDLang4/9000176.md).





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyConnection is Connection

// Describe the connection
MyConnection.User = "USER"
MyConnection.Password = "PASSWORD"
MyConnection.Server = "MYSERVER" 
MyConnection.Database = "Database"
MyConnection.Provider = hNativeAccessSQLServer
MyConnection.Access = hOReadWrite

// Open the connection
MyConnection.OpenConnection()

// Specify that "Salaries" uses the new connection
Salaries.ChangeConnection(MyConnection)

// Start looping through the "Salaries" table on SQLServer
Salaries.ReadFirst()
...
// Close the connection
MyConnection.CloseConnection()

// Restore the use of connection on Oracle
Salaries.ChangeConnection("")

// Open the "Salaries" table on Oracle
Salaries.Open()
```

<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.ChangeConnection(<New connection>)`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos (definido en el análisis) para el cual debe cambiarse la conexión. Este archivo de datos se cerrará automáticamente para que se modifique su conexión. 

**`<New connection>: (Cadena de caracteres o variable de tipo Connection)`**

Nueva conexión a utilizar. Este parámetro puede corresponder a: 

- el nombre de la conexión a una base de datos. Esta conexión ha sido definida en el editor de análisis o dinámicamente con [HDescribeConnection](../WDLang4/3044205.md). Si este parámetro corresponde a una cadena vacía ("""), se utiliza automáticamente la conexión inicial.

- el nombre de la [Conexión](../WDLang4/1514073.md) Variable que corresponde a la nueva conexión a utilizar.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Debe conocerse la conexión al ejecutar la función **&lt;Fuente&gt;.ChangeConnection**. Si no se conoce el archivo de datos al utilizar la función **&lt;Fuente&gt;.ChangeConnection**, la conexión cambiará al declarar el archivo de datos.

- La función **&lt;Fuente&gt;.ChangeConnection** cierra automáticamente el archivo de datos (función [&lt;Fuente&gt;.Close](../WDLang4/1000024958.md)) cuya conexión debe modificarse.

- La función **&lt;Fuente&gt;.ChangeConnection** no abre ni el archivo de datos, ni la conexión. La conexión definida con **&lt;Fuente&gt;.ChangeConnection** se utilizará la próxima vez que se abra el archivo de datos

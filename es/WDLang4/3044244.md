
## Convert (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Convierte un valor numérico en una cadena binaria para realizar una búsqueda en una clave numérica.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// CompKey is made of PRODREF (text) and QTY (integer)
// EDT_Quantity and EDT_ProductRef are two edit controls
Srch is string
Srch = Complete(EDT_ProductRef, 8) + Convert(EDT_Quantity)
HReadSeek(Orders, CompKey, Srch)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = Convert(<Numeric value>)`
---

**`<Result>: (Cadena de caracteres)`**

Resultado de la conversión. Este resultado es la cadena binaria que corresponde al valor de la variable numérica. Esta cadena binaria no se puede mostrar directamente. Esta cadena binaria solo se utiliza para realizar búsquedas en el archivo de datos HFSQL.

**`<Numeric value>: (Variable de tipo Numeric)`**

Nombre de la variable numérica a convertir



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- El tipo de conversión depende del tipo de la variable pasada como parámetro.

- La función **Convert** debe utilizarse para realizar búsquedas en una clave numérica compuesta, en un archivo de datos HFSQL. Para las claves numéricas simples, no hay necesidad de convertir el valor numérico en cadena binaria, la conversión se realiza automáticamente.

- La función **Convert** no puede utilizarse en lenguaje externo. Se debe utilizar la función [HConvertX](../WDLang4/3044260.md).

- La función [HConvertX](../WDLang4/3044260.md) y **Convert** realizan la misma acción. Con **Convert** (a diferencia de [HConvertX](../WDLang4/3044260.md)), no es necesario indicar el tipo de valor a convertir; el tipo se deduce automáticamente del tipo de variable &lt;Valor_Númerico&gt;


## H.AutoIndexingLimit (Variable)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica el número máximo de registros en el archivo de datos utilizados para gestionar la reindexación automática.

Mientras no se alcance este número, el motor HFSQL inicia la reindexación automática si se produce un error de índice.

Recordatorio: No se dispone de reindexación automática para los archivos de datos en formato Hyper File 5.5.


<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Valor de H.AutoIndexingLimit
<a name="valor_hautoindexinglimit_ELTPARAGRAPHE000024"></a>

- La variable H.AutoIndexingLimit está establecida en 100.000 por defecto.

- Si H.AutoIndexingLimit está ajustado a 0, la reindexación automática está desactivada.

- Si H.AutoIndexingLimit se inicializa con un valor negativo, el valor de esta variable corresponderá automáticamente al valor por defecto.




**Atención**: Si se produce un error de bloqueo durante la operación de reindexación (cuando el archivo de datos o un registro encontrado en el archivo de datos se bloquea en lectura/escritura, por ejemplo), la reindexación automática se detiene y se muestra un error fatal (índice corrupto). Entonces, tendrás que reindexar a el archivo de datos.

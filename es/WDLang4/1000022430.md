


## HTrackStop (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Deja de rastrear a un archivo de datos en un servidor HFSQL. Este tema fue iniciado por [HTrack](../WDLang4/1000022407.md). 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Starts tracking the Order data file
// The ProcessOrder procedure is called whenever a record is added to the data file
HTrack(Order, ProcessOrder, hRecNumAll, hsAdd)

...
// Stops tracking the Order data file
HTrackStop(Order, ProcessOrder, hRecNumAll)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HTrackStop(<Data file> , <WLanguage procedure> [, <Record number>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si de detuvo la supervisión,

- <u><u><u><u>False</u></u></u></u> en caso contrario (por ejemplo, supervisión inexistente). Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Data file>: (Cadena de caracteres opcional)`**

Nombre del archivo de datos HFSQL a utilizar. El archivo de datos es associated con una conexión y con un servidor. 
Si este parámetro es una cadena vacía (""), **HTrackStop** manipula el último archivo de datos utilizado por la última función HFSQL (función que empieza por "H").

**`<WLanguage procedure>: (Nombre de Procedure)`**

El nombre de WLanguage Procedure no se llamará más para rastrear a el archivo de datos. 

**`<Record number>: (Entero opcional)`**

Número de la Record que ya no se puede rastrear. 
Si este parámetro no se especifica o corresponde a la constante *hRecNumAll*, **HTrackStop** dejará de seguir todos los registros de el archivo de datos (es decir, todo el archivo de datos).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Las funciones [HTrack](../WDLang4/1000022407.md) y **HTrackStop** se pueden utilizar en un servicio de Windows.

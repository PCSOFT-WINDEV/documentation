
## Acceso nativo a xBase/FoxPro: Características y observaciones específicas
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000243"></a>
Esta página de ayuda presenta las características específicas relativas a la programación de Native xBase/FoxPro Access. 

Se admiten los siguientes formatos de archivo Image: 

- dBase3+

- dBase4

- FoxPro/FoxBase (hasta la versión 9)

- Clipper5

- Clipper87






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principales observaciones
<a name="principales_observaciones_ELTTEXTE000273"></a>


### Gestión de la integridad y los duplicados
<a name="gestion_integridad_los_duplicados_ELTPARAGRAPHE000028"></a>

Los duplicados y la integridad no están soportados. 
<a name="NOTE2_2"></a>


### Memo files
<a name="memo_files_ELTPARAGRAPHE000035"></a>

Los archivos de memo se abren y gestionan automáticamente con los archivos de datos.
<a name="NOTE2_3"></a>


### Modificación de datos de automatic
<a name="modificacion_datos_automatic_ELTPARAGRAPHE000042"></a>

La modificación automatic de los archivos de datos HFSQL no puede ser usada con los archivos xBase/FoxPro.
<a name="NOTE2_4"></a>


### Bloqueo de los archivos de datos
<a name="bloqueo_los_archivos_datos_ELTPARAGRAPHE000049"></a>

Sólo el archivo de datos completo puede ser bloqueado en modo de escritura y/o en modo de sólo lectura. De hecho, bloquear algunos registros de del archivo de datos no está permitido por el acceso nativo de xBase/FoxPro.
<a name="NOTE2_5"></a>


### Borrar registros
<a name="borrar_registros_ELTPARAGRAPHE000056"></a>

Un Record se puede borrar con [HDelete](../WDLang4/3044018.md) o [HCross](../WDLang4/3044121.md). En los archivos xBase/FoxPro, [HDelete](../WDLang4/3044018.md) y [HCross](../WDLang4/3044121.md) tienen el mismo efecto: La Record está "cruzada".
<a name="NOTE2_6"></a>


### Número de registros
<a name="numero_registros_ELTPARAGRAPHE000075"></a>

Cuando se utiliza con la constante *hStateActive* o *hStateAll*, [HNbRec](../WDLang4/3044006.md) también devuelve el número de registros cruzados o borrados (por compatibilidad con WINDEV 5.5). 

Sin embargo, puede averiguar el número de registros cruzados o borrados utilizando [HNbRec](../WDLang4/3044006.md) con la constante *hStateDel* o *hStateCrossed*. 
<a name="NOTE2_7"></a>


### Búsqueda de registros
<a name="busqueda_registros_ELTPARAGRAPHE000090"></a>

Durante una búsqueda, si el número de caracteres en el valor buscado es mayor que el número máximo de caracteres en el elemento clave en el que se realiza la búsqueda, la búsqueda se realiza en el campo *N* que se encuentran en el valor buscado (donde *N* es el número máximo de caracteres en el elemento clave).

**Ejemplo**: El número máximo de caracteres en el elemento clave llamado ITM de FILE es 3. El elemento clave llamado ITM contiene "abc" y "abd".

Si el valor buscado contiene más de 3 caracteres, sólo se tendrán en cuenta los 3 primeros caracteres de este valor.

- Si el valor buscado es *"abcd"*, la búsqueda se realizará en *"abc"* (se encontrará un Record).

- Si el valor buscado es *"abhd"*, la búsqueda se realizará en *"abh"* (no se encontrará ningún Record).




Observación: En las búsquedas y filtros, las fechas en blanco (cadena vacía) son mayores que los otros valores de fecha.. 
<a name="NOTE2_8"></a>


### description dinámica de archivos de datos
<a name="description_dinamica_archivos_datos_ELTPARAGRAPHE000106"></a>

La description de un archivo xBase se puede hacer en WLanguage (se debe abrir un proyecto associated con un análisis HFSQL o xBase); se debe utilizar

- [HDBDescribeFile](../WDLang4/3044172.md).

- [HDBDescribeField](../WDLang4/3044170.md).

- [HDBDescribeIndex](../WDLang4/3044166.md).

- [HDBCreation](../WDLang4/3044171.md) o [HDBOpen](../WDLang4/3044163.md).




El archivo creado está en **Formato dBase 3**.

El archivo descrito en WLanguage puede contener elementos de notas y claves.

Estos comandos se pueden utilizar para crear archivos temporales.
<a name="NOTE2_9"></a>


### Archivos HFSQL y archivos xBase/FoxPro
<a name="archivos_hfsql_archivos_xbasefoxpro_ELTPARAGRAPHE000135"></a>

El mismo proyecto puede manipular tanto archivos HFSQL como archivos xBase/FoxPro.

**Atención**: En el mismo proyecto, un archivo de datos HFSQL y un archivo de datos xBase/FoxPro no pueden tener el mismo nombre lógico, de lo contrario se producirá un error.

## Características específicas
<a name="caracteristicas_especificas_ELTTEXTE000345"></a>


### Opciones de análisis
<a name="opciones_analisis_ELTPARAGRAPHE000146"></a>

Independientemente de si las opciones "Distinguir mayúsculas y minúsculas", "Distinguir acentos" y "Distinguir puntuación" están marcadas en el editor de análisis, la búsqueda siempre distinguirá mayúsculas y minúsculas, acentos y puntuación. 
<a name="NOTE3_2"></a>


### Campos de tipo Array
<a name="campos_tipo_array_ELTPARAGRAPHE000153"></a>

Los elementos array no están soportados por el Native Access. Si está utilizando un elemento Array, las funciones HFSQL sólo manejarán el valor del índice 1 en la array (en lectura/escritura). 
<a name="NOTE3_3"></a>


### Contexto HFSQL independiente
<a name="contexto_hfsql_independiente_ELTPARAGRAPHE000160"></a>

La gestión de los contextos HFSQL independientes (casilla en la description de las ventanas o informes) no está disponible en esta versión. 
<a name="NOTE3_4"></a>


### Transacciones
<a name="transacciones_ELTPARAGRAPHE000167"></a>

No se pueden realizar transacciones ([HTransactionStart](../WDLang4/3044002.md)) en el Acceso Nativo xBase/FoxPro.
<a name="NOTE3_5"></a>


### Límites de los archivos de datos de xBase
<a name="limites_los_archivos_datos_xbase_ELTPARAGRAPHE000177"></a>

Ver [Archivos en formato xBase: Límites](../WDLang4/3044247.md) a get la lista de limitaciones conocidas para los archivos de datos xBase/FoxPro.
<a name="NOTE3_6"></a>


### Múltiples índices dBase4 (MDX) e índices FoxPro (CDX)
<a name="multiples_indices_dbase4_mdx_indices_foxpro_cdx_ELTPARAGRAPHE000187"></a>

El Native xBase/FoxPro Access no soporta las dos condiciones siguientes utilizadas conjuntamente: 

- Índice basado en una expresión distinta del valor individual del elemento (por ejemplo, SUPERIOR (ITM1) o ITM1 + ITM2).. 

- Nombre del índice lógico idéntico a uno de los elementos que se encuentran en el archivo xBase/FoxPro.


El archivo de índice no será reconocido.

**Solución**: Renombrar el índice lógico.




## HInfoBackup (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve información sobre una o más copias de seguridad realizadas en un servidor HFSQL Client/Server.

Las copias de seguridad han sido realizadas por [HBackup](../WDLang4/1000017081.md) (o a través de HFSQL control Center) y pueden ser restauradas por [HRestoreBackup](../WDLang4/1000017230.md) (o a través de HFSQL control Center).



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Displays information about the backups in a Table control
sReport is string = HInfoBackup(gctConnection)
sLine is string
FOR EACH STRING sLine OF sReport SEPARATED BY EOT
	IF sLine ~="" THEN CONTINUE
	TableAdd(TABLE_Backups)
	nSub is int = TableCount(TABLE_Backups)
	TABLE_Backups.COL_BackupID[nSub] = ExtractString(sLine, 1)
	TABLE_Backups.COL_DateTime[nSub] = ExtractString(sLine, 2)
	TABLE_Backups.COL_Filter[nSub] = ExtractString(sLine, 3)
	TABLE_Backups.COL_Name[nSub] = ExtractString(sLine, 4)
	TABLE_Backups.COL_Description[nSub] = ExtractString(sLine, 5)
	TABLE_Backups.COL_Status[nSub] =  ...
			SWITCH(Val(ExtractString(sLine, 6)) + 1, ...
				"In progress", "Performed", "Canceled", "Error")
	TABLE_Backups.COL_PerThousand[nSub] = ExtractString(sLine, 7)
END
```

## Sintaxis

### Información sobre las copias de seguridad realizadas en el servidor

`<Result> = HInfoBackup(<Connection>)`
---

**`<Result>: (Cadena de caracteres)`**

description de las copias de seguridad realizadas en el servidor especificado. Este description tiene el siguiente formato: 

```txt
<Identifier of Backup 1> + TAB + <Date of Backup 1> + TAB + 
<Files Saved> + TAB + <Destination Path of Backup 1> + TAB + 
<Description> + TAB + <Status> + TAB + <Per Thousand> + TAB + <Error Message> + TAB + 
<Type> + TAB + <Identifier of Full Backup> + TAB + <With Index> + TAB + 
<Compression> + EOT
```
donde:

- &lt;Archivos guardados&gt; es la lista de datos guardados separados por punto y coma (";"). 
	&lt;El estado&gt; corresponde a un número entero (o a una constante entera) utilizado para averiguar el estado de la copia de seguridad:
	


|   |   |   |
| --- | --- | --- |
| *hBackupCanceled* | 2 | Copia de seguridad cancelada |
| *hBackupCompleted* | 1 | Copia de seguridad completada |
| *hBackupError* | 3 | Error durante la copia de seguridad |
| *hBackupInProgress* | 0 | Creando copia de seguridad |



- &lt;Por cada mil&gt; corresponde a la tasa de finalización de la copia de seguridad si la copia de seguridad está en curso (expresada en por mil).

- &lt;Mensaje de error&gt; es el mensaje de error si la copia de seguridad falla.

- &lt;Tipo&gt; corresponde a un número entero (o a una constante entera) utilizado para averiguar el tipo de copia de seguridad.: 
	


|   |   |   |
| --- | --- | --- |
| *hBackupDifferential* | 1 | Copia de seguridad diferencial |
| *hBackupFull* | 0 | Copia de seguridad completa |



- &lt;El identificador de Full Backup&gt; corresponde al identificador de la copia de seguridad completa correspondiente en caso de copia de seguridad diferencial.

- Con índice&gt; &lt;Con índice: Se usa para averiguar si la copia de seguridad contiene los índices: 

	- 0 si no se ha realizado la copia de seguridad de los índices,

	- 1 si la copia de seguridad de los índices se ha realizado. 




- &lt;Compresión&gt;: Constante correspondiente a la compresión de la copia de seguridad:

	- *zipFormatNone*: la copia de seguridad no está comprimida.

	- *zipFormatZIP*: la copia de seguridad está comprimida. 





Observación: el separador utilizado entre los diferentes description es "EOT" (en lugar de "CR") porque los diferentes parámetros de description pueden contener caracteres de Carriage Return (CR). 

**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión con el servidor cuyas copias de seguridad serán listadas. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).

## Observaciones


### Varios
<a name="varios_ELTPARAGRAPHE000277"></a>

- La función **HInfoBackup** permite listar las tareas realizadas o las tareas en curso. No se listan las tareas programadas que no se han realizado.

- Se enumeran las copias de seguridad realizadas por el HFSQL control Center y las realizadas por programación.

- Las fechas y horas se expresan en hora universal (UTC).



<a name="NOTE0_2"></a>


### Derechos necesarios
<a name="derechos_necesarios_ELTPARAGRAPHE000292"></a>

Para get en formación sobre una copia de seguridad, el usuario o el grupo debe tener: 

- para cada base de datos afectada por la copia de seguridad, los permisos para realizar copias de seguridad (constante *hRightsBackup*) o los permisos para conectarse (constante *hRightsConnection* o *hRightsEncryptedConnection*). 

- para una copia de seguridad completa del servidor, los derechos para realizar copias de seguridad (constante *hRightsBackup*) en el servidor. 




Para saber si estos derechos están concedidos a un usuario o a un grupo, utilice [HInfoServerRights](../WDLang4/3044322.md) o [HInfoDatabaseRights](../WDLang4/3044320.md).

Para modificar los derechos concedidos a un usuario o a un grupo de usuarios, utilice [HModifyServerRights](../WDLang4/3044328.md) o [HModifyDatabaseRights](../WDLang4/3044326.md).

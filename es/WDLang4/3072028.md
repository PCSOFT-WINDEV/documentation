
## SQLInfo( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicializa las diferentes variables SQL con información relativa a la última consulta ejecutada.. Esta función puede utilizarse independientemente del tipo de exploración realizada (para más detalles, véase [Tipos de navegación SQL](../WDLang4/3072025.md)).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
ResSQL = SQLExec("SELECT * FROM INV", "QRY1")
IF ResSQL = False THEN
	SQLInfo("QRY1")
	Info("SQL error: " + SQL.MesError)
ELSE
	SQLTable("QRY1", TABLE_Table1)
END
SQLClose("QRY1")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`SQLInfo([<Query name>])`
---

**`<Query name>: (Cadena de caracteres opcional)`**

Nombre de la consulta: 

- creado y dirigido por [SQLExec](../WDLang4/3072007.md),

- dirigido por [SQLExecWDR](../WDLang4/3072010.md). 


Se genera un error si este parámetro corresponde a una consulta que no existe.. Para comprobar la existencia de la consulta, utilice [SQLReqExists](../WDLang4/3072026.md).

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las consultas creadas con el editor de consultas deben especificarse en el formato de una cadena de caracteres.

## Observaciones




### Información actualizada si no se especifica el parámetro &lt;Nombre de pregunta&gt;.
<a name="informacion_actualizada_especifica_parametro_ELTPARAGRAPHE000105"></a>



| Variable | escribir a máquina | description |
| --- | --- | --- |
| *Base de datos SQL* | Cadena de caracteres | Nombre de la base de datos a la que está conectado el proyecto. |
| *Conexión SQL.* | Integro | Identificador de la conexión actual (devuelto por [SQLConnect](../WDLang4/3072005.md)). |
| *Error de SQL.* | Cadena de caracteres | <br><br>- Código de error ("00000" si no hay error). Consulte la documentación sobre el conductor para conocer su significado.. <br><br>- Código de error inexistente para las conexiones con los controladores ORACLE o SQL SERVER nativos de PC SOFT.<br><br><br> |
| *SQL.MesError* | Cadena de caracteres | Subtítulo de error. |
| *SQL.NbCol* | Integro | Número de columnas de resultados. |

Si no se especifica el parámetro &lt;Nombre de consulta&gt;, se debe llamar a la función **SQLInfo** después de la conexión (función [SQLConnect](../WDLang4/3072005.md))
<a name="NOTE0_2"></a>

- En Java y PHP, se asigna el valor de la variable **SQL.NbLine** solo si se llamó a la función [SQLFirst](../WDLang4/3072017.md) antes de la función **SQLInfo**.

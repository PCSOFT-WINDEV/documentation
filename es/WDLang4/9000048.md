## Características específicas de esta versión
<a name="caracteristicas_especificas_esta_version_ELTTEXTE000128"></a>
En esta versión, las características específicas de la replicación universal asistida son las siguientes:

- **dministración de la modificación de datos de automatic.**
	

	- Replicación entre bases de datos HFSQL: La gestión de modificación de datos de automatic sólo está disponible para la replicación universal de las bases de datos HFSQL. 
			Para habilitar la modificación de datos de automatic durante la replicación, debe especificarlo en [ReplicEdit](../ReplicEdit/3526501.md): en la pestaña [réplica description](../ReplicEdit/3526502.md), "Avanzado", **""**. 
			**Atención**: 

		- En el caso de las réplicas existentes, es necesario recrear una réplica (así como los suscriptores) para poner en práctica este mecanismo.

		- Este mecanismo no funciona si se añade o se elimina una clave única. En este caso, la modificación de automatic debe realizarse de la misma manera que para las bases de datos en diferentes formatos.   




- Replicación entre bases de datos en diferentes formatos:  No hay ningún mecanismo automatic disponible para la modificación de datos de automatic. Es necesario:

	- sincronizar el mayor número posible de suscriptores antes de la modificación de los datos del automatic.

	- realizar la modificación de los datos de automatic en los archivos maestros.

	- realizar la modificación de los datos de automatic en los archivos de los suscriptores.

	- sincronizar a los suscriptores.

- **Gestión de la replicación de los archivos de groupware usuario.**
	Los archivos de la Agrupación de Usuarios no son gestionados automáticamente por la replicación universal asistida. Una copy de los archivos de la el groupware usuario debe ser administrada "manualmente" en el ordenador local.

- **Replicación directa**
	El archivo.WER debe desplegarse necesariamente junto al archivo ejecutable (incluso para una replicación programada).. El archivo.WER no se puede incluir en una biblioteca WINDEV (archivo.WDL).

- **Replicación universal asistida y WINDEV Mobile**
	La versión móvil de ReplicSynchro no se suministra en esta versión.. Se debe realizar una replicación universal asistida utilizando las funciones de programación de WLanguage. Para obtener más información, consulte [Replicación universal asistida mediante programación](../WDLang4/9000047.md).

- **Replicación universal asistida y WEBDEV**
	Con WEBDEV, se debe realizar una replicación universal asistida utilizando las funciones de programación WLanguage. Para obtener más información, consulte [Replicación universal asistida mediante programación](../WDLang4/9000047.md).

- **Parámetros para acceder a los ficheros maestros**
	**ReplicAdmin** no se puede utilizar para modificar los parámetros de acceso a los archivos maestros: Debe realizar una instalación con la creación de instalación asistente y actualizar la aplicación.
	Por lo tanto, una replicación no se puede implementar directamente en un equipo sin utilizar una aplicación.

- **Volumen de datos**
	**Atención**: la replicación universal puede encontrar problemas cuando se procesan grandes volúmenes de datos. El principal factor limitante es el tamaño de la clave utilizada para identificar los registros.. Por ejemplo, una clave de número entero (o identificador automatic) le permitirá replicar más registros que una clave de cadena.
	Además, recomendamos que replique una sub-set de registros usando las características de filtrado de la replicación asistida. De hecho, lo que hay que tener en cuenta es el volumen de datos reproducidos y no el volumen total de datos.




## HRSDeleteConfig (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina una replicación entre dos servidores HFSQL. Esta función debe ser ejecutada en un servidor HFSQL maestro de una réplica, o en un servidor de suscriptor huérfano.
 

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Cnt is Connection
...
HRSDeleteConfig(Cnt, ConfigReplication1)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Borrar una réplica identificada por un hHRSConfig Variable

`<Result> = HRSDeleteConfig(<Connection> , <Replication to delete> [, <Type of deletion>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se eliminó la replicación,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión al servidor HFSQL en el que se realiza la replicación. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).




**`<Replication to delete>: (hRSConfig Variable)`**

Nombre de [hRSConfig](../WDLang4/1000020484.md) Variable que describe la réplica a eliminar. 

**`<Type of deletion>: (Constante opcional de tipo Integer)`**

Tipo de reproducción que desea borrar: 


|   |   |
| --- | --- |
| *hrsDefault* <br>(valor predeterminado) | Borra la replicación sólo si el servidor de suscriptores es accesible. |
| *hrsIgnoreSubscriber* | Forza la eliminación incluso si el otro servidor no puede ser contactado.. En ese caso: <br><br>- la información relativa a la replicación no se eliminará del servidor de suscriptores.  <br><br>- para una replicación bidireccional, la replicación no se eliminará del otro servidor. Le aconsejamos que realice este borrado desde este servidor también para evitar un consumo de memoria inútil.<br><br><br> |
| *hrsSubscriber* | Fuerza la eliminación de una réplica huérfana en el lado del suscriptor. En este caso, la variable de tipo [hRSConfig](../WDLang4/1000020484.md) se puede obtener con la función [HRSListConfig](../WDLang4/1000020497.md) junto con la constante *hrsSubscriber*.  |

Observación: Antes de la versión 190040, este parámetro correspondía a un booleano (<u><u><u><u>True</u></u></u></u> equivalente a la constante *hrsIgnoreSubscriber* y <u><u><u><u>False</u></u></u></u> equivalente a la constante *hrsDefault*). 


<a name="SYNTAX2"></a>

### Suprimir una réplica identificada por su identificador

`<Result> = HRSDeleteConfig(<Connection> , <Identifier> [, <Forced deletion>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se eliminó la replicación,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión al servidor HFSQL en el que se realiza la replicación. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).




**`<Identifier>: (Integro)`**

Identificador de la réplica a eliminar. Este identificador corresponde a la propiedad **Identifier** de la variable [hRSConfig](../WDLang4/1000020484.md) que describe la replicación.

**`<Forced deletion>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> para forzar la eliminación, incluso si no se puede contactar con el otro servidor. En ese caso: 

	- la información relativa a la replicación no se eliminará del servidor de suscriptores.  

	- para una replicación bidireccional, la replicación no se eliminará del otro servidor. Le aconsejamos que realice este borrado desde este servidor también para evitar un consumo de memoria inútil..  .  




- <u><u><u><u>False</u></u></u></u> (valor predeterminado) para eliminar la replicación solo si el servidor esclavo es accesible.

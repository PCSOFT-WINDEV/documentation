
## WLanguage Procedure llamado por HTrack



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
WLanguage Procedure ("callback") llamado por [HTrack](../WDLang4/1000022407.md) cuando se realiza un cambio en el archivo de datos especificado.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Starts tracking the Order data file
// The ProcessOrder procedure is called whenever a record is added to the data file
HTrack(Order, ProcessOrder, hRecNumAll, hsAdd)

PROCEDURE ProcessOrder(FileName, RecNum, Action)
IF BinaryAND(nAction, hsAdd) = hsAdd THEN
	Trace("An addition was made in the data file: " + FileName)
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`HTrack_Callback(<Modified data file> , <Record number> , <Action>)`
---

**`<Modified data file>: (Cadena de caracteres)`**

Nombre del archivo de datos HFSQL modificado. 

**`<Record number>: (Integro)`**

Número de la Record modificada (equivalente al número devuelto por [HRecNum](../WDLang4/3044063.md)). 

**`<Action>: (Constante de tipo Integer)`**

Acción que desencadenó el Procedure. Los valores posibles son: 



|   |   |
| --- | --- |
| *hsAdd* | El procedimiento comenzó una vez que se añadió un Record a el archivo de datos. |
| *hsCross* | El procedimiento comenzó una vez que se cruzó un Record en el archivo de datos. |
| *hsDeletion* | El procedimiento comenzó una vez que se borró un Record de el archivo de datos. |
| *hsModification* | El procedimiento comenzó una vez que se modificó un Record en el archivo de datos. |


**Atención**: Si un proceso añade una Record modifica esta Record inmediatamente después, se llama a [HTrack](../WDLang4/1000022407.md) una vez con una combinación de las constantes *hsAdd* y *hsModification*.
BinaryAND debe ser usado para identificar estas acciones en el WLanguage Procedure. Por ejemplo: 

```wl
IF BinaryAND(nAction, hsAdd) = hsAdd THEN ...
```

## Observaciones


### WLanguage Procedure ejecutado en el Thread principal
<a name="wlanguage_procedure_ejecutado_thread_principal_ELTPARAGRAPHE000074"></a>

Por defecto, este WLanguage Procedure es llamado en el hilo principal de la aplicación, con el contexto HFSQL que corresponde al Procedure:

- Si el Procedure es un Procedure local de una ventana que utiliza un contexto independiente, se utilizará el contexto de la ventana. 

- Si la Procedure pertenece a un componente que utiliza un contexto independiente, el contexto utilizado será el del componente.





|   |
| --- |
| **Atención**:<br>El Procedure associated con [HTrack](../WDLang4/1000022407.md) puede ser llamado en cualquier momento en la aplicación.<br>Si el Procedure lee registros o se mueve entre registros en los archivos de datos, esto tendrá un impacto en la aplicación (como los temporizadores).<br>Por lo tanto, es posible que tenga que llamar a [HSavePosition](../WDLang4/3044136.md) y [HRestorePosition](../WDLang4/3044137.md) para guardar los contextos de los archivos utilizados.<br><br>[Multitask](../WDLang1/3015004.md) no debe utilizarse en el código Procedure. |




### Lenguaje Procedure ejecutado en un Thread secundario
<a name="lenguaje_procedure_ejecutado_thread_secundario_ELTPARAGRAPHE000234"></a>

Si el lenguaje WL Procedure se llama en un thread secundario: 

- el contexto HFSQL se copia la primera vez que se llama a **WLanguage Procedure llamado por HTrack**. Solo se almacenan los directorios que contienen los archivos de datos en modo HFSQL Classic y/o las conexiones en modo HFSQL Client/Server.

- Los elementos de la interfaz de usuario (ventanas, controles, etc.) no pueden ser manipulados en la Thread secundaria. Cuando un Thread secundario debe interactuar con el usuario o actualizar la UI, debe utilizar un Process iniciado desde el Thread principal. Este proceso puede corresponder a:

	- un procedimiento global del proyecto o un procedimiento local (de una ventana, etc.) llamado con la función [ExecuteMainThread](../WDLang1/1000019862.md),

	- el evento "Solicitud para actualizar la visualización" de una ventana, ejecutado con la función [RequestRefreshUI](../WDLang1/1000023899.md).







Para obtener más información sobre este procedimiento, consulte [Parámetros del procedimiento utilizado por la función WLanguage Procedure llamado por HTrack](../WDLang4/1000024804.md).

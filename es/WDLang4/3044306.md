## Presentación
<a name="presentacion_ELTTEXTE000202"></a>
Tiene la posibilidad de crear aplicaciones que accedan a las bases de datos de HFSQL Client/Server.

Una aplicación HFSQL Client/Server ejecuta una aplicación en varios ordenadores (llamados ordenadores cliente) y almacena las bases de datos y los procesos en un servidor. Este modo de funcionamiento hace que los tiempos de respuesta sean más rápidos y fiables, y simplifica el mantenimiento de la base de datos.



**Ejemplo con aplicaciones WINDEV**: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P3-HFSQL-Client-Serveur.gif)


**Ejemplo con sitios WEBDEV**: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5-HFSQL-Client-Serveur.gif)


Es posible:

- crear una aplicación HFSQL Client/Server desde cero.

- modificar una aplicación WINDEV, WEBDEV o WINDEV Mobile existente en una aplicación HFSQL Client/Server.




**Observación**: Para comparar el modo HFSQL Client/Server con el modo HFSQL Network, véase [Modo de funcionamiento de la aplicación HFSQL](../WDLang4/3044303.md).

### Compatibilidad entre versiones
<a name="compatibilidad_entre_versiones_ELTPARAGRAPHE000056"></a>

Las diferentes versiones de clientes y servidores son compatibles. Todas las versiones de cliente pueden utilizar todas las versiones del servidor.

Sin embargo, le aconsejamos que mantenga siempre el motor HFSQL Client/Server en su versión más reciente propuesta en [el centro de descargas](https://www.windev.com/ts/download/index.html), aunque todos los clientes conectados (aplicaciones y/o sitios) no estén compilados con la última versión.

**Observación**: Si la DLL cliente utilizada es más reciente que el servidor (por ejemplo, DLL cliente en versión 18 y servidor en versión 12), las nuevas funcionalidades provocarán un error de tipo "El servidor no soporta esta funcionalidad (servidor demasiado antiguo)" si son utilizadas por la aplicación cliente.

### Instalación del servidor HFSQL Client/Server en un servidor LINUX
<a name="instalacion_del_servidor_hfsql_clientserver_servidor_linux_ELTPARAGRAPHE000070"></a>

Para instalar el servidor HFSQL Client/Server en un servidor LINUX: 

1. copy los siguientes archivos al servidor: 

	- manta_install

	- mantainstall.zip

	- README
			|Estos archivos están localizados en el subdirectorio "HFSQL Server\\Linux4" del paquete instalación.. 




2. Conéctese al servidor como "root".

3. Ejecute el archivo manta_install. Es posible: 

	- Instalar un nuevo servidor.

	- Actualizar un servidor existente. 

	- Desinstalar un servidor.




4. Siga las instrucciones que aparecen en la pantalla. 


**Atención**: La licencia debe desplazarse completamente para mostrar las siguientes pantallas. 



Para instalar el servidor HFSQL Client/Server de 64 bits en un servidor LINUX de 64 bits: 

1. copy los siguientes archivos al servidor: 

	- manta_install64

	- mantainstall.zip

	- README
			Estos archivos están localizados en el subdirectorio "HFSQL Server\\Linux44_64" del paquete instalación. 




2. Conéctese al servidor como "root"

3. Run manta_install64. Es posible: 

	- Instalar un nuevo servidor.

	- Actualizar un servidor existente. 

	- Desinstalar un servidor.




4. Siga las instrucciones que aparecen en la pantalla. 




**Atención**: La licencia debe desplazarse completamente para mostrar las siguientes pantallas. 

**Observaciones**: 

- El servidor se instalará como "Daemon (servicio)". 

- Las siguientes versiones de Linux son compatibles con el motor HFSQL para Linux: 

	- Versión Linux4: Debian 7 (y posterior), OpenSuse 10.3 (y posterior)

	- Versión Linux4 (64 bits): Ubuntu 14 (y posteriores)

### Servidor HFSQL Client/Server de 32 y 64 bits
<a name="servidor_hfsql_clientserver_32_64_bits_ELTPARAGRAPHE000122"></a>

Hay dos versiones disponibles para el servidor HFSQL: Modo de 32 bits y modo de 64 bits. El cliente HFSQL también está disponible en modo de 32 bits y en modo de 64 bits.

El servidor en modo de 64 bits reconoce a los clientes en modo de 32 bits y el servidor en modo de 32 bits reconoce a los clientes en modo de 64 bits.




### El servidor de HFSQL Client/Server y IPv6
<a name="servidor_hfsql_clientserver_ipv6_ELTPARAGRAPHE000132"></a>

El servidor HFSQL soporta el protocolo IPv6. Si el ordenador contiene una tarjeta de red configurada en IPv6, el servidor HFSQL aceptará automáticamente las conexiones utilizando este protocolo en todas las tarjetas de red configuradas en IPv6.

El servidor HFSQL soporta simultáneamente las conexiones en IPv6 y en IPv4.

No es necesario modificar el código para usar el protocolo IPv6. Si el nombre del servidor es resuelto por el DNS en un Address IPv6, se utilizará este protocolo.

## Limitaciones
<a name="limitaciones_ELTTEXTE000280"></a>


- El modo de gestión de los archivos de datos puede tener que adaptarse cuando se utiliza una aplicación de HFSQL Client/Server. Para saber más sobre las características específicas que hay que tener en cuenta, consulte [Diferencias y limitaciones específicas de una aplicación de HFSQL Client/Server](../WDLang4/3044302.md).

- El servidor HFSQL está disponible para los siguientes sistemas operativos:

	- Windows de 32 bits

	- Windows de 64 bits

	- Linux de 32 bits

	- Linux de 64 bits


 No está disponible para los otros sistemas operativos.

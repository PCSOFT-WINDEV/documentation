


## &lt;Fuente&gt;.UnlockFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Desbloquea los registros de un archivo de datos:

- bloqueado por [&lt;Fuente&gt;.LockFile](../WDLang4/1000024196.md).

- bloqueados individualmente por una función de lectura bloqueante.




El archivo de datos también puede desbloquearse cerrándolo ([&lt;Fuente&gt;.Close](../WDLang4/1000024958.md)).

Observación: **&lt;Fuente&gt;.UnlockFile** no puede utilizarse para desbloquear un archivo de datos bloqueado por otra sesión o programa. Le aconsejamos que desbloquee un archivo de datos lo antes posible para que las otras sesiones puedan acceder a este archivo de datos.
![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función se puede utilizar con archivos de datos HFSQL Client/Server.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Lock
Customer.LockFile(HLockReadWrite)
// Modify the file
...
// Unlock
Customer.UnlockFile()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.UnlockFile()`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> si se bloqueó el archivo de datos,

- <u><u><u><u>False</u></u></u></u> si se produce un problema (por ejemplo, el archivo de datos no está bloqueado, etc.). La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos utilizado. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
La función **&lt;Fuente&gt;.UnlockFile** no tiene ningún efecto si:

- el archivo de datos está en modo monopuesto (ver [&lt;Fuente&gt;.Mode](../WDLang4/1000025037.md)).

- el archivo de datos no está cerrado.

- el archivo de datos fue bloqueado por otro programa.




## Convertir una base de datos HFSQL Classic en HFSQL Client/Server
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000121"></a>
Para convertir una base de datos HFSQL Classic en una base de datos HFSQL Client/Server, WINDEV, WEBDEV y WINDEV Mobile proponen dos métodos: 

- [Use la editor de análisis](#NOTE2_1) modificar directamente el tipo de archivos de datos: pasar del tipo HFSQL Classic (archivo de datos en un directorio) a un tipo HFSQL Client/Server (conexión a un servidor HFSQL). 

- [Utilizar el HFSQL control Center](#NOTE3_1) para convertir directamente una base de datos HFSQL Classic en HFSQL Client/Server. 




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Conversión en la editor de análisis
<a name="conversion_editor_analisis_ELTTEXTE000145"></a>
Para convertir una base de datos HFSQL Classic en HFSQL Client/Server: 

1. Cargar el proyecto y abrir el análisis que contiene los archivos de datos a convertir. 

2. Selecciona un archivo de datos para convertir en el editor de análisis. 

3. En el menú contextual, seleccione la opción "Descripción del archivo de datos". 

4. En la ventana que se muestra, modifique el tipo de archivo: seleccione " HFSQL Client/Server ". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Convertir_HFSQLClassic%20-%20HC%20N%B0001.gif&type=thumb)


5. La asistente para crear una conexión comienza. 

6. Introduzca el nombre del servidor HFSQL o su IP Address. 

7. Introduzca el nombre de usuario y la contraseña (la primera vez que se conecte, introduzca "Admin" y deje vacía la contraseña).. 

8. Elija o introduzca el nombre de la base de datos a la que se copiará el archivo de datos actual. 

9. Se crea la conexión. 

10. La asistente propone asociar la conexión creada con otros archivos de datos encontrados en el análisis. Por Default, se seleccionan todos los archivos de datos. Puedes desmarcar los que no quieras convertir.

11. La asistente propone a la copy los archivos de datos en el servidor HFSQL. 




Una vez que la operación se completa, los archivos de datos en el modo HFSQL C/S aparecen en amarillo en la editor de análisis.

## Conversión en el Centro HFSQL control
<a name="conversion_centro_hfsql_control_ELTTEXTE000169"></a>
Para convertir una base de datos HFSQL Classic en HFSQL Client/Server: 

1. Abrir el Centro HFSQL control. 

2. Conectarse a un servidor HFSQL existente (usuario "Admin" sin contraseña por Default). 

3. Seleccione el servidor. En el panel de características del servidor, en el grupo "Base de datos", haga clic en "Importar una base de datos HFSQL Classic".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Convertir_HFSQLClassic%20-%20HC%20N%B0002.gif&type=thumb)


4. El asistente de importación se abre automáticamente. 

5. Especifique el directorio donde se encuentran sus archivos de datos HFSQL Classic. 

6. Introduzca el nombre de una base de datos de destino o haga clic en el botón "Nuevo" para crear una nueva base de datos.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Convertir_HFSQLClassic%20-%20HC%20N%B0003.gif&type=thumb)


7. Localizar el análisis (lo que significa la description de los archivos de datos a convertir). Este paso es opcional, pero permite recuperar la description de enlaces y reglas de integridad entre archivos de datos.

8. Compruebe la lista de archivos de datos a convertir e inicie la conversión.

9. El "Código de conexión" permite recuperar, a través del portapapeles, el código de conexión a la base de datos HFSQL Client/Server. Por ejemplo: 
	
	```wl
	HDescribeConnection("HyperFileCS", "admin", "<password>", ...
			"localhost:4900", "CRMDATABASE", hAccessHFClientServer)
	HChangeConnection("*", "HyperFileCS")
	```



Observación: Los nombres de los archivos de datos no se modifican durante la conversión.

<a name="NOTE4_1"></a>


## Nota
<a name="nota_ELTTEXTE000193"></a>
Una vez convertida la base de datos, no olvides incluir el código de conexión en el proyecto existente para poder cambiar el tipo de conexión mediante programación. Por ejemplo, el código de conexión se puede copiar en el código de proyecto.
Ejemplo de código de conexión: 

```wl
ConnectionName is Connection

// Connection parameters
ConnectionName.Provider = hAccessHFClientServer
ConnectionName.User = "UserName"
ConnectionName.Password = "Password"
ConnectionName.Server = "Myserver:port"
ConnectionName.Database = "MyDatabase"
ConnectionName.CryptMethod = hEncryptionNO

// Open the connection
HOpenConnection(ConnectionName)

HChangeConnection("*", ConnectionName)
```

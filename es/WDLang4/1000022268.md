
## ReplicDeleteSubscription( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina la suscripción de una replicación en el servidor de replicación.

Esta función se utiliza principalmente para una replicación con réplicas retrasadas, para liberar recursos en el servidor.. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Opens a replication on the replication server
ReplicOpen("MyReplication", "", "ReplicationServer", "Name", "password")

// Deletes the replication
ReplicDeleteSubscription("MyReplication")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = ReplicDeleteSubscription(<Replication name>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se eliminó la suscripción de la replicación,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para más detalles sobre el problema, utilice [ErrorInfo](../WDLang1/3013008.md).




**`<Replication name>: (Cadena de caracteres)`**

Nombre de la réplica a utilizar. Este nombre corresponde a:

- el nombre del archivo ".wer", creado por ReplicEdit (para una replicación en línea). Este archivo debe encontrarse junto al ejecutable.

- el nombre de la publicación (para un servidor de replicación).






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- La réplica debe haber sido abierta por [ReplicOpen](../WDLang4/1000017194.md). 

- Si la replicación no usa el servidor de replicación (replicación en modo directo), la función **ReplicDeleteSubscription** no tiene ningún efecto. 

- Los archivos locales associated con la réplica no se borran.

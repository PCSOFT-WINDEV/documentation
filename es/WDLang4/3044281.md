


## HAddUser (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Agrega un usuario a una base de datos. Todas las características del usuario deben ser introducidas en el [variables para la gestión de usuarios](../WDLang4/3044315.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Describe the connection 
HDescribeConnection("MyConnection", "Julia", "Password", "MyHFServer", ...
		"MyDatabase", hAccessHFClientServer, HORead)
// Open the connection
HOpenConnection("MyConnection")
// Associate the data file with the connection
HChangeConnection(Customer, "MyConnection")
// Characteristics of the new user
HUser.Login = "FLO"
HUser.Password = "FLOP"
HUser.FullName = "Flora"
HUser.Email = "Flo@myaddress.eu"
HUser.Phone = "01.67.40.47.58"
HUser.Groups = "ADMIN" + CR + "TEST"
// Add the user
IF HAddUser("MyConnection") = True THEN
	Info("User added")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HAddUser(<Connection>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se agregó el usuario,

- <u><u><u><u>False</u></u></u></u> si se produce un error. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión a la que se debe añadir un usuario. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).

## Observaciones


### Derechos necesarios para añadir un usuario
<a name="derechos_necesarios_para_anadir_usuario_ELTPARAGRAPHE000116"></a>

Para añadir un usuario, debe tener los siguientes derechos:

- Permisos para administrar usuarios (constante *hRightsManageUser*).

- Permisos para ver usuarios (constante *hRightsSeeUser*).


Estas constantes se utilizan en [HInfoServerRights](../WDLang4/3044322.md) y [HModifyServerRights](../WDLang4/3044328.md).
<a name="NOTE0_2"></a>


### Gestión de derechos
<a name="gestion_derechos_ELTPARAGRAPHE000136"></a>

No se conceden derechos por defecto a un nuevo usuario (si este usuario es associated sin grupo).

Si este usuario es associated con uno o más grupos, los derechos associated con este grupo se conceden al usuario (el usuario "hereda" los derechos associated con el grupo).

Las siguientes funciones se utilizan para conceder derechos a un usuario:

- [HModifyDatabaseRights](../WDLang4/3044326.md): se utiliza para conceder derechos sobre una base de datos.

- [HModifyFileRights](../WDLang4/3044327.md): se utiliza para conceder derechos sobre un archivo de datos.

- [HModifyServerRights](../WDLang4/3044328.md): se utiliza para conceder derechos sobre un servidor.


Para obtener más información, consulte [Gestión de los derechos en HFSQL Client/Server](../WDLang4/3044333.md).
<a name="NOTE0_3"></a>


### "Usuario"SuperUser
<a name="usuariosuperuser_ELTPARAGRAPHE000164"></a>

Si HUser.SuperUser = True, los derechos de usuario no se verifican durante una acción.. Por lo tanto, todas las acciones pueden ser realizadas por este usuario.

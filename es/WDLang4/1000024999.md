


## &lt;Fuente&gt;.NoModif (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
No permite que se realicen modificaciones en un archivo de datos (para todos los programas, incluyendo el que solicitó que no se puedan realizar modificaciones). Se puede acceder a los registros en modo de sólo lectura.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Invoice.NoModif()
// Process...
Invoice.EndNoModif()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.NoModif()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si no se ha encontrado ningún problema,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HError](../WDLang4/3044088.md) permite obtener más detalles sobre el problema.


Si al cabo de [H.NbRetry](../WDLang4/3044075.md) intentos, el archivo de datos sigue bloqueado, la función [HErrorLock](../WDLang4/3044099.md) devuelve <u><u><u><u>True</u></u></u></u> (la variable H.Locked se establece en <u><u><u><u>True</u></u></u></u>). 
Si se bloqueó el archivo de datos, la función [HErrorLock](../WDLang4/3044099.md) devuelve <u><u><u><u>False</u></u></u></u>.

**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos utilizado.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Cerraduras set de &lt;Fuente&gt;.NoModif
<a name="cerraduras_set_docparampagetitleshort_ELTPARAGRAPHE000217"></a>

La función **&lt;Fuente&gt;.NoModif** puede bloquear **en modo de escritura** un archivo de datos que ya ha sido bloqueado por otro programa (con la función **&lt;Fuente&gt;.NoModif**). Por lo tanto, varios programas pueden prohibir las modificaciones en el mismo archivo de datos al mismo tiempo. Una función de lectura bloqueante ejecutada después de la función **&lt;Fuente&gt;.NoModif** en el mismo programa, lee el registro sin bloquearlo.

La función **&lt;Fuente&gt;.NoModif** no puede bloquear un archivo de datos que ya ha sido bloqueado en **lectura y escritura** por otro programa.
Recordatorio: el archivo de datos no puede bloquearse con **&lt;Fuente&gt;.NoModif** solo si el archivo de datos ha sido bloqueado en lectura y escritura por otro programa.

Observaciones:

- En modo de usuario único (función [&lt;Fuente&gt;.Mode](../WDLang4/1000025037.md)), la función **&lt;Fuente&gt;.NoModif** es equivalente a [&lt;Fuente&gt;.Open](../WDLang4/1000025043.md) (los archivos de datos se bloquean en lectura y escritura al abrirse).

- Tienes la capacidad de bloquear a un archivo de datos cuyos registros son bloqueados por otra aplicación.



<a name="NOTE0_2"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000259"></a>

- El número máximo de programas que pueden solicitar la política de no modificación en un archivo de datos depende del tamaño de un registro en el archivo de datos. Se puede calcular de la siguiente manera: 
	
	```txt
	MaxNbSessionsOrPrograms = RecordSize - 20
	```


- Esta función no está disponible cuando se utiliza una base de datos a través de .

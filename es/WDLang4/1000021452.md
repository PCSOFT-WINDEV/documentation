


## HListScheduling (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Lista las tareas programadas definidas en un servidor HFSQL: 

- procedimiento almacenado, 

- refuerzos,

- optimización, 

- actualización de una vista materializada.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Modify all scheduled backups
arrBckDesc is array dynamic of hBackupDescription 
arrBckDesc = HListScheduling(MyDatabaseConnection, hlpBackup)
hBckDesc is hBackupDescription
FOR EACH hBckDesc OF arrBckDesc
	IF hBckDesc.LimitNumberBackups = 3 THEN 
		hBckDesc.LimitNumberBackups = 5
		HModifyScheduling(MyDatabaseConnection, hBckDesc)
	END
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HListScheduling(<Connection> , <Type of scheduling>)`
---

**`<Result>: (Array)`**

Array que contiene las descripciones listadas. El tipo de los elementos array depende de &lt;Tipo de programación&gt;.. 

**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión a utilizar. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).




**`<Type of scheduling>: (Constante de tipo Integer)`**

Clase de programación a lista: 


|   |   |
| --- | --- |
| *hlpBackup* | Programación de copias de seguridad. |
| *hlpMaterializedView* | Tareas para refrescar las vistas materializadas. |
| *hlpOptimization* | Tareas de optimización de archivos de datos (reindexación, cálculo de estadísticas). |
| *hlpTask* | Tareas programadas que ejecutan un Procedure almacenado. |

## Observaciones


### Derechos necesarios
<a name="derechos_necesarios_ELTPARAGRAPHE000104"></a>

Para listar las tareas de actualización de vistas materializadas o las tareas programadas, es necesario tener permisos de conexión (constante *hRightsConnection* o *hRightsEncryptedConnection*) para cada base de datos que contenga una vista a procesar o un procedimiento almacenado a ejecutar. 

Para saber si estos derechos están concedidos a un usuario o grupo, utilice [HInfoServerRights](../WDLang4/3044322.md) o [HInfoDatabaseRights](../WDLang4/3044320.md).

Para modificar los derechos concedidos a un usuario o grupo de usuarios, utilice [HModifyServerRights](../WDLang4/3044328.md) o [HModifyDatabaseRights](../WDLang4/3044326.md).




## &lt;Variable Connection&gt;.AddUser (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Agrega un usuario a una base de datos. Todas las características del usuario deben ser introducidas en el [variables para la gestión de usuarios](../WDLang4/3044315.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Describe the connection 
MyConnection is Connection

// Open the connection
MyConnection.OpenConnection()
// Associate the data file with the connection
Customer.ChangeConnection(MyConnection)
// Characteristics of the new user
HUser.Login = "FLO"
HUser.Password = "FLOP"
HUser.FullName = "Flora"
HUser.Email = "Flo@myaddress.eu"
HUser.Phone = "01.67.40.47.58"
HUser.Groups = "ADMIN" + CR + "TEST"
// Add the user
IF MyConnection.AddUser() = True THEN
	Info("User added")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Connection>.AddUser()`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> si se agregó el usuario,

- <u><u><u><u>False</u></u></u></u> si se produce un error. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Connection>: (Variable de tipo Connection)`**

Nombre de la [Conexión](../WDLang4/1514073.md) Variable que describe la conexión a la que hay que añadir un usuario.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Derechos necesarios para añadir un usuario
<a name="derechos_necesarios_para_anadir_usuario_ELTPARAGRAPHE000251"></a>

Para añadir un usuario, debe tener los siguientes derechos:

- Permisos para administrar usuarios (constante *hRightsManageUser*).

- Permisos para ver usuarios (constante *hRightsSeeUser*).


Estas constantes se utilizan en [&lt;Variable Connection&gt;.InfoServerRights](../WDLang4/1000022668.md) y [&lt;Variable Connection&gt;.ModifyServerRights](../WDLang4/1000022710.md).
<a name="NOTE0_2"></a>


### Gestión de derechos
<a name="gestion_derechos_ELTPARAGRAPHE000276"></a>

No se conceden derechos por defecto a un nuevo usuario (si este usuario es associated sin grupo).

Si este usuario es associated con uno o más grupos, los derechos associated con este grupo se conceden al usuario (el usuario "hereda" los derechos associated con el grupo).

Las siguientes funciones se utilizan para conceder derechos a un usuario:

- [&lt;Variable Connection&gt;.ModifyDatabaseRights](../WDLang4/1000022707.md): se utiliza para conceder derechos sobre una base de datos.

- [&lt;Variable Connection&gt;.ModifyFileRights](../WDLang4/1000022709.md): se utiliza para conceder derechos sobre un archivo de datos.

- [&lt;Variable Connection&gt;.ModifyServerRights](../WDLang4/1000022710.md): se utiliza para conceder derechos sobre un servidor.


Para obtener más información, consulte [Gestión de los derechos en HFSQL Client/Server](../WDLang4/3044333.md).
<a name="NOTE0_3"></a>


### "Usuario"SuperUser
<a name="usuariosuperuser_ELTPARAGRAPHE000311"></a>

Si HUser.SuperUser = True, los derechos de usuario no se verifican durante una acción.. Por lo tanto, todas las acciones pueden ser realizadas por este usuario.

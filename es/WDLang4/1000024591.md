


## &lt;Fuente&gt;.ChangePassword (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cambia la contraseña de un archivo de datos HFSQL Classic o HFSQL Client/Server. Las aplicaciones que utilizan la antigua contraseña ya no podrán acceder a este archivo de datos: se mostrará un error de contraseña. 

**Atención**: Esta función es una función avanzada. Debe ser usado con mucho cuidado porque puede provocar errores en las aplicaciones del cliente.. 


<a name="sample_code"></a>

## Ejemplo


```wl
// Former password
HPass("*", "Password")
// Modify the password
Customer.ChangePassword("NewPassword")
```

```wl
// Create a data file with password
FileA.Password("TEST")
FileA.Creation()

// Add data
FileA.ItemA = "Test 1"
FileA.Add()

// Read data
FileA.ReadFirst()
Info(FileA.ItemA)

// Change the password
Info("Changing the password...")
FileA.ChangePassword("NW_Test")
Info("Password changed, the first record is read again")

// Read data again
FileA.ReadFirst()
Info(FileA.ItemA)

// Manage the errors
CASE ERROR:
	Error(HErrorInfo())
	RETURN
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.ChangePassword(<New password>)`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> si se modificó la contraseña,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos HFSQL (definido en el análisis) a utilizar.

**`<New password>: (Cadena de caracteres)`**

Nueva contraseña que será usada para el archivo de datos.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si el archivo de datos ya está protegida por una contraseña, ésta debe especificarse con [&lt;Fuente&gt;.Password](../WDLang4/1000025044.md) antes de utilizar **&lt;Fuente&gt;.ChangePassword**. 

- Para modificar la contraseña de un archivo de datos, debe tener los permisos de modificación automática (constante *hRightsAutoModif*). Estos derechos pueden ser definidos por [&lt;Variable Connection&gt;.ModifyFileRights](../WDLang4/1000022709.md). 

- Esta función inicia una modificación de automatic del archivo de datos. Si el archivo de datos fue encriptado con una contraseña diferente, se volverá a encriptar completamente con la nueva contraseña. Esta operación puede llevar bastante tiempo..

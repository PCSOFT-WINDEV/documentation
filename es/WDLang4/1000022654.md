


## &lt;Variable Connection&gt;.DeactivateServerTrigger (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Desactiva un trigger del Servidor HFSQL Client/Server en un servidor. Este trigger está deshabilitado para todos los clientes de este servidor. Este activador puede volver a ser activado por [&lt;Variable Connection&gt;.ActivateServerTrigger](../WDLang4/1000022637.md).  

**Observación**: Las funciones para manejar los disparadores del servidor son funciones avanzadas. Los triggers del servidor se pueden crear directamente en el editor de análisis. Para obtener más información, consulte [Disparadores de servidor](../WDLang4/3044369.md).




<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

### Desactivar un trigger de servidor en archivos HFSQL físicos de un servidor

`<Result> = <Connection>.DeactivateServerTrigger(<Trigger> [, <Physical HFSQL Client/Server files>])`
---

**`<Result>: (Booleano)`**



- <u><u><u>True</u></u></u> si se realizó la operación,

- <u><u><u>False</u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Connection>: (Variable de tipo Connection)`**

Nombre de la variable de tipo [Conexión](../WDLang4/1514073.md) que describe la conexión con el servidor HFSQL.

**`<Trigger>: (Cadena de caracteres)`**

Nombre del disparador para desactivar. Este nombre se definió con [&lt;Variable Connection&gt;.DescribeServerTrigger](../WDLang4/1000022653.md).

**`<Physical HFSQL Client/Server files>: (Cadena de caracteres opcional)`**

Nombre físico de uno o más archivos de datos de HFSQL Client/Server associated con la conexión (por ejemplo "CUSTOMER.FIC"). 
Para especificar varios nombres, los nombres de los archivos de datos deben estar separados por una coma (","). 
Si este parámetro no se especifica o si corresponde a una cadena vacía ("""), el trigger se desactivará en todos los ficheros de datos de las bases de datos..

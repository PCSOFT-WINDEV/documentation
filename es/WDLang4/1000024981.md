


## &lt;Fuente&gt;.Flush (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Obliga al sistema operativo del equipo en el que se encuentran los archivos de datos a escribir los datos directamente en el disco.. El contexto HFSQL del archivo de datos utilizado se almacena (bloqueo, navegación actual, ...).

**Observación**: Utilizada regularmente, esta función minimiza la pérdida de datos causada por la inesperada interrupción del sistema en el equipo, donde se encuentran los archivos de datos (por ejemplo, un corte de energía).

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función sólo está disponible para los archivos de datos HFSQL. No está disponible para los archivos de datos a los que accede con JDBC. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Add a record in a data file
// (record described through programming)
Customer.LastName = "Clark"
Customer.FirstName = "Vince"
Customer.Address = "Palm lane"
Customer.City = "San Francisco"
Customer.Zip = "94102"
Customer.Country = "USA"
Customer.Add()
// Forces a write operation into Customer file
Customer.Flush()
```

`<Source>.Flush()`
---

**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos utilizado.

### Gestión de cierres
<a name="gestion_cierres_ELTPARAGRAPHE000232"></a>

Al llamar a la función **&lt;Fuente&gt;.Flush**, el archivo de datos manipulado se cierra y luego se vuelve a abrir. Si se implementaron bloqueos en este archivo de datos antes de que se cerrara, estos bloqueos se restauran automáticamente cuando se vuelve a abrir.. De hecho, al cerrar un archivo de datos, el sistema operativo borra los bloqueos implementados en este archivo de datos.

**Observación**: No es aconsejable llamar a esta función con demasiada frecuencia:

- archivos de datos en los que se implementa un número importante de bloqueos.

- archivos de datos en los que se lleva a cabo un número importante de operaciones de escritura.



<a name="NOTE0_2"></a>


### &lt;Fuente&gt;.Seguridad y &lt;Fuente&gt;.Flush
<a name="seguridad_docparampagetitleshort_ELTPARAGRAPHE000255"></a>Las funciones [&lt;Fuente&gt;.Security](../WDLang4/1000025063.md) y **&lt;Fuente&gt;.Flush** permiten forzar la escritura física en el disco.

Sin embargo:

- [&lt;Fuente&gt;.Security](../WDLang4/1000025063.md) se utiliza para forzar una operación de escritura en los archivos de datos utilizados durante cada operación de escritura. Esta operación de escritura es realizada automáticamente por el motor HFSQL. Los archivos de datos utilizados no se cierran durante esta operación de escritura. La velocidad de la operación de escritura a archivo es más lenta.

- **&lt;Fuente&gt;.Flush** obliga a escribir el archivo de datos(s) cuando es llamado. Los archivos de datos utilizados se cierran y se vuelven a abrir durante esta operación de escritura.. La velocidad de la operación de escritura en archivo no se ralentiza.

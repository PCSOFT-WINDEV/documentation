


## HCS.FrameSize (Variable)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La variable **HCS.FrameSize** permite modificar el tamaño de la trama de datos enviada. En algunos casos, el redimensionamiento de las tramas puede servir para adaptar el flujo de datos salientes y entrantes al rendimiento del hardware utilizado. 

Por ejemplo, si la conexión entre los ordenadores clientes y el motor de HFSQL Client/Server pasa por redes de instalación (tarjeta de red, switch, router, vpn, ...) con un ajuste no estándar, se pierden los cuadros si superan un tamaño determinado. En este caso, tienes la capacidad de usar este Variable.  

**En la mayoría de los casos, si el hardware es reciente y está configurado correctamente, no es necesario cambiar el tamaño de los marcos.**




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HCS.FrameSize = hInternetFrame
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`HCS.FrameSize = <Frame Value>`
---

**`<Frame Value>: (Constante entera o entero)`**

Valor de la trama (en bytes). Corresponde a un valor o a una de las siguientes constantes predefinidas: 


|   |   |
| --- | --- |
| *hActiveSyncFrame* | El tamaño del marco está optimizado para ser utilizado con dispositivos móviles. |
| *hDefaultFrame* | El tamaño estándar será usado por Default. |
| *hGPRSFrame* | El tamaño del marco está optimizado para su uso a través de GPRS. |
| *hInternetFrame* | El tamaño del cuadro está optimizado para un uso a través de INTERNET. |
| *hIntranetFrame* | El tamaño del cuadro está optimizado para un uso a través de INTRANET. |
| *hWifiFrame* | El tamaño del marco está optimizado para su uso a través de Wi-Fi. |

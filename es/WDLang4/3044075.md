
## H.NbRetry (Variable)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica el número de intentos de bloquear a el archivo de datos o a Record antes de devolver a control al programa. Esta información se aplica a todos los archivos de datos.

De hecho, el el archivo de datos o el Record a bloquear puede haber sido bloqueado por otro programa sólo por unos segundos. Por lo tanto, puede ser interesante repetir la solicitud de bloqueo varias veces.

**H.NbRetry** sólo se puede utilizar en modo multiusuario.

**Atención**: Este valor no se tiene en cuenta en HFSQL Classic cuando el bloqueo fue implementado por otro contexto HFSQL de la misma aplicación. Este caso puede ocurrir cuando se utilizan dos hilos de rosca.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- **H.NbRetry** es igual a 50 por Default.

- El valor de **H.NbRetry** puede modificarse tantas veces como sea necesario, según el tipo de cerraduras.

- No se debe utilizar el valor "0".: no se realiza ninguna modificación.

- El valor "-1" significa que hay un número muy importante de intentos de bloqueo repetidos.. Si se selecciona esta opción, debe evitar necesariamente los riesgos de bloqueo.

- Para cualquier valor negativo (distinto de -1), no se realiza ninguna modificación.

- El valor que se debe asignar a **H.NbRetry** depende de la velocidad de la red. Cuanto más rápida sea la red, menor será el valor de **H.NbRetry** será.




## SourceToScreen (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Actualiza automáticamente los controles de una ventana o Page con:

- los valores de los elementos vinculados en el Record actual (cargado en memoria) del archivo de datos HFSL descrito en el editor de análisis. 

- los valores de las variables vinculadas de la lengua WL. 


Esta operación se realiza independientemente del estado de los mandos (grises, inactivos o Invisible).

**Observaciones**: 

- **SourceToScreen** no lee registros en el archivo de datos. El valor de la Record cargado en memoria se utiliza para actualizar los controles. Por lo tanto, un Record debe ser cargado en la memoria previamente: 

	- [HRead](../WDLang4/3044047.md), [HReadSeek](../WDLang4/3044050.md), [HReadFirst](../WDLang4/3044051.md), etc. 

	- [LooperDisplay](../WDLang2/3083001.md) para un control Looper, etc.










<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Searches for customer in the data file
HReadSeekFirst(CUSTOMER, NAME, "PAUL")
IF HFound() = True THEN
	// Displays the client found in the current window
	SourceToScreen()
END

// Read the record and
// transfer items to the controls in WIN_Customer
HReadFirst(CUSTOMER, NAME)
SourceToScreen (WIN_Customer)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`SourceToScreen([<Window> [, <Source>]])`
---

**`<Window>: (Nombre de ventana)`**

Nombre de la ventana o Page que contiene los controles a actualizar.

Resumen: Se llama cuando una control gana el foco.


**`<Source>: (Cadena de caracteres opcional)`**

Nombre (entre comillas) de la fuente a la que están vinculados los controles a actualizar. Esta fuente puede ser:

- El nombre de un archivo de datos: sólo se tienen en cuenta los controles vinculados a los elementos del archivo de datos.

- El nombre de un WLanguage Variable: sólo se tienen en cuenta los controles vinculados a la lengua de trabajo Variable.
	Observaciones: 

	- Si el nombre de la Variable está entre comillas, el control se vinculará a la Variable especificada . 

	- Si el nombre de la Variable no está entre comillas, el control se vinculará a los datos de la Variable (por ejemplo, una Variable que contenga el nombre de otra Variable).





Si no se especifica este nombre, se tendrán en cuenta todos los controles de ventana vinculados a un elemento de un archivo de datos descrito en la editor de análisis o a un WLanguage Variable.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Vinculación entre un control y un WLanguage Variable o entre un control y un elemento de un archivo de datos
<a name="vinculacion_entre_control_wlanguage_variable_entre_control_elemento_archivo_datos_ELTPARAGRAPHE000129"></a>

Se puede determinar la vinculación entre un control y un WLanguage Variable o entre un control y un elemento de un archivo de datos:

- En el editor, en la ventana de control description (pestaña "Binding").

- De forma programada, con el programa [DataBinding](../Proprietes/2510060.md) Property.



<a name="NOTE0_2"></a>


### Vinculación a un elemento: tipos de fijaciones
<a name="vinculacion_elemento_tipos_fijaciones_ELTPARAGRAPHE000144"></a>

Existen dos tipos de vínculos entre una control y un artículo

- *unión directa*: En este caso, **SourceToScreen** transfiere el valor de los elementos del Record cargado en la memoria al control enlazado f006c00 , sin leer ningún valor.

- *enlace leyendo los registros en el archivo de datos associated*. 



<a name="NOTE0_3"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000158"></a>

- **control Combo Box autocompletado vinculado a un archivo de datos HFSQL**: **SourceToScreen** no debe estar en la ventana "Declaraciones globales" Event. La función debe estar en la Event "Inicialización" de la ventana.

- Los caracteres de espacio en los elementos HFSQL se mantienen.

- **SourceToScreen** actualiza los controles Image con el valor de los elementos de memo Image enlazados.

- **SourceToScreen** admite el valor NULL. Si el contenido del elemento es NULL, se mostrará una cadena vacía en el campo Editar control. Para cambiar este modo de visualización, utilice [ControlError](../WDLang1/3025036.md).

- **SourceToScreen** no está disponible en vistas. Los valores de los elementos deben asignarse a cada control.

- **Vinculación con elementos de "Contraseña segura"**:

	- En una Edición control vinculada a un elemento Contraseña, aparecen los caracteres "\*\*\*\*\*".. Si la Edición control es de tipo "Contraseña", se muestran puntos en su lugar. 

	- El valor del elemento "Contraseña" no puede volver a leerse hasta que el usuario cambie el valor de la contraseña.

	- El valor del elemento "Contraseña" puede seguir comparándose con una cadena (por ejemplo, "" o "1234").


## Optimizar el tiempo de conexión a las bases de datos HFSQL Client/Server
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000103"></a>
Esta página de ayuda explica cómo optimizar los tiempos de conexión a las bases de datos de HFSQL Client/Server limitando el tráfico de red.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000127"></a>
Para que una aplicación se conecte por programación a una base de datos HFSQL Client/Server pueden ser necesarias varias llamadas a funciones: 

- [HOpenConnection](../WDLang4/3044107.md)para especificar el servidor Address, el usuario, la base de datos,

- [HChangeConnection](../WDLang4/3044150.md)para asociar algunas tablas con una conexión o con otra,

- [HPass](../WDLang4/3044108.md)para especificar la contraseña de los archivos encriptados,

- [HChangeDir](../WDLang4/3044168.md)para especificar una carpeta de base de datos concreta,

- ... 




El orden en que se llaman estas funciones es importante: influye en el tiempo necesario para establecer la conexión. La ganancia no es significativa para la conexión de una aplicación de escritorio que accede al servidor a través de una red local. Pero desde una aplicación móvil en iOS o Android, el impacto puede ser significativo si la conexión pasa por una red móvil con latencia. 

Por lo tanto, se debe utilizar la siguiente secuencia: 

1. HChangeConnection 

2. HChangeDir 

3. HPass 

4. HOpenConnection 




en lugar de: 

1. HOpenConnection 

2. HChangeConnection 

3. HChangeDir 

4. HPass 




En efecto, si se llama primero a [HOpenConnection](../WDLang4/3044107.md), la conexión se abre inmediatamente con tráfico de red. A continuación, las demás funciones desencadenarán otros intercambios entre la aplicación y el servidor para modificar los parámetros de la conexión que se acaba de abrir.

Por el contrario, si se llama a [HOpenConnection](../WDLang4/3044107.md) en último lugar, la información necesaria para configurar la conexión se envía de inmediato: tráfico de red limitado, las prestaciones mejoran.




## &lt;Fuente&gt;.FilterIdentical (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Define y activa un filtro utilizado para encontrar el valor exacto de un elemento de cadena. Este filtro puede ser usado en un archivo de datos, una vista o una consulta.

Después de su ejecución, los casos más comunes son los siguientes:

- no se puede crear el filtro: **&lt;Fuente&gt;.FilterIdentical** devuelve una cadena vacía (""). [HError](../WDLang4/3044088.md) devuelve el identificador del error.

- se crea el filtro: la función **&lt;Fuente&gt;.FilterIdentical** devuelve la clave de búsqueda adecuada que se utilizará para navegar por el archivo de datos, la vista o la consulta.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Select all the customers whose last name is Archambault 
// and whose first name is Colin
SearchKey is string
SearchKey = Customer.FilterIdentical(LastNameFirstName, "Archambault", "Colin")
Customer.ReadFirst(SearchKey)
WHILE NOT Customer.Out()
	Trace(Customer.CustomerName, Customer.City)
	Customer.ReadNext(SearchKey)
END
// Disables the filter
Customer.DeactivateFilter()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.FilterIdentical(<Key item> , <Value for key component 1> [, <Value for key component 2> [... [, <Value for key component N>]]])`
---

**`<Result>: (Cadena de caracteres)`**

Campo de navegación. Corresponde a:

- la clave de búsqueda del archivo de datos si el filtro está activado.

- una cadena vacía ("") si no se puede establecer el filtro.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos HFSQL, vista o consulta utilizada.

**`<Key item>: (Cadena de caracteres)`**

Nombre del elemento clave sobre el que se aplicará el filtro. Este elemento puede ser una llave simple (cadena) o una llave compuesta. Atención: este parámetro sólo debe contener letras, caracteres subrayados o dígitos. De lo contrario, el nombre debe estar entre comillas.

**`<Value for key component 1>: (Tipo de cada componente de la clave de búsqueda)`**

Valor buscado para el primer componente de la clave compuesta. 
El tipo de este parámetro debe corresponderse con el tipo de componente del elemento clave sobre el que se define el filtro.

**`<Value for key component 2>: (Tipo de cada componente de la clave de búsqueda)`**

Valor buscado para el segundo componente de la clave compuesta. 
El tipo de este parámetro debe corresponderse con el tipo de componente del elemento clave sobre el que se define el filtro.



**`<Value for key component N>: (Tipo de cada componente de la clave de búsqueda)`**

Valor buscado para el componente N de la clave compuesta. 
El tipo de este parámetro debe corresponderse con el tipo de componente del elemento clave sobre el que se define el filtro.

## Observaciones


### automatic gestión de claves compuestas
<a name="automatic_gestion_claves_compuestas_ELTPARAGRAPHE000257"></a>

La función **&lt;Fuente&gt;.FilterIdentical** gestiona automáticamente los filtros tanto para las claves simples como compuestas.

Ejemplo: Uso de la función **&lt;Fuente&gt;.FilterIdentical** con una clave simple:

```wl
Res = Customer.FilterIdentical(LastName, "Archambault")
// Select the customer whose last name is Archambault
```


Ejemplo: Utilizar **&lt;Fuente&gt;.FilterIdentical** con una clave compuesta

```wl
Res = Customer.FilterIdentical(LastNameFirstName, "Archambault", "Colin")
// Select all the customers whose last name is Archambault and 
// whose first name is Colin
```
 Este Line de código equivale a:

```wl
Customer.Filter(LastNameFirstName, ...
	Customer.BuildKeyValue(LastNameFirstName, "Archambault", "Colin"), ...   
	Customer.BuildKeyValue(LastNameFirstName, "Archambault", "Colin")
```

<a name="NOTE0_2"></a>


### Filtro e iteración filtrada
<a name="filtro_iteracion_filtrada_ELTPARAGRAPHE000296"></a>

Después de ejecutar la función **&lt;Fuente&gt;.FilterIdentical**, la navegación del archivo de datos debe realizarse en el elemento devuelto por la función **&lt;Fuente&gt;.FilterIdentical**. El filtro será ignorado si se utiliza otro elemento para recorrer el archivo de datos.

Cuando se define y habilita un filtro en un archivo de datos (vista o consulta), todos los registros leídos corresponden al filtro. Si ningún otro Record corresponde al filtro durante la iteración: 

- [&lt;Fuente&gt;.Out](../WDLang4/1000024605.md) devuelve <u><u><u><u>True</u></u></u></u>. 

- el actual Record corresponde al último Record leído con el filtro. 




Por ejemplo:

| En un archivo de datos filtrado, después de la función: | el actual Record es: |
| --- | --- |
| **[&lt;Fuente&gt;.ReadFirst](../WDLang4/1000025022.md)** | el primer archivo Record correspondiente al filtro. |
| **[&lt;Fuente&gt;.ReadLast](../WDLang4/1000025020.md)** | el último archivo Record correspondiente al filtro. |
| **[&lt;Fuente&gt;.ReadNext](../WDLang4/1000025035.md)** (o **[&lt;Fuente&gt;.Forward](../WDLang4/1000024195.md)**) | el siguiente Record (o el siguiente n-ésimo Record) correspondiente al filtro. |
| **[&lt;Fuente&gt;.ReadPrevious](../WDLang4/1000025021.md)** (o **[&lt;Fuente&gt;.Backward](../WDLang4/1000025056.md)**) | el anterior Record (o el anterior n-ésimo Record) correspondiente al filtro. |


<a name="NOTE0_3"></a>


### Habilitar/Deshabilitar un filtro
<a name="habilitardeshabilitar_filtro_ELTPARAGRAPHE000376"></a>

- [&lt;Fuente&gt;.DeactivateFilter](../WDLang4/1000024218.md) se utiliza para desactivar un filtro.

- [&lt;Fuente&gt;.ActivateFilter](../WDLang4/1000023969.md) se utiliza para volver a habilitar un filtro.

- El filtro se elimina cuando se cierra el archivo de datos (consulta o vista) ([&lt;Fuente&gt;.Close](../WDLang4/1000024958.md) o [HOpenAnalysis](../WDLang4/3044106.md), por ejemplo).

- Un único filtro puede existir en un momento dado en un archivo de datos (consulta o vista). Si la función **&lt;Fuente&gt;.FilterIdentical** se utiliza varias veces, solo se tendrá en cuenta el último filtro: se borrarán los filtros anteriores. Del mismo modo, si se utilizan varios filtros (funciones HFilter\*), sólo se tendrá en cuenta el último filtro.

### Archivo de datos no en formato HFSQL
<a name="archivo_datos_formato_hfsql_ELTPARAGRAPHE000409"></a>

- El filtro se puede crear a partir de cualquier elemento (clave o no clave).

- El filtro se activa automáticamente cuando se ejecuta **&lt;Fuente&gt;.FilterIdentical**.

- Los cambios realizados con **&lt;Fuente&gt;.FilterIdentical** y [&lt;Fuente&gt;.DeactivateFilter](../WDLang4/1000024218.md) sólo se tienen en cuenta cuando se utilizan las siguientes funciones [&lt;Fuente&gt;.ReadFirst](../WDLang4/1000025022.md)/[&lt;Fuente&gt;.First](../WDLang4/1000025048.md), [&lt;Fuente&gt;.ReadLast](../WDLang4/1000025020.md)/[&lt;Fuente&gt;.Last](../WDLang4/1000024217.md) o [&lt;Fuente&gt;.ReadSeekFirst](../WDLang4/1000025034.md)/[&lt;Fuente&gt;.ReadSeekLast](../WDLang4/1000025024.md).

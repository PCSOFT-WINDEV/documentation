


## &lt;Fuente&gt;.Backward (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Retrocede varios registros en el archivo de datos a partir de la posición actual, según un elemento especificado.

El registro de destino se lee y se carga en la memoria: se actualizan las variables HFSQL (Customer.Name por ejemplo, que significa el elemento *Name* del archivo *Customer*).

Tras la llamada a **&lt;Fuente&gt;.Backward**pueden darse varios casos:

- no existen más registros en el archivo de datos: [&lt;Fuente&gt;.Out](../WDLang4/1000024605.md) devuelve <u><u><u>True</u></u></u>. No se realiza ninguna lectura.

- la función intenta bloquear un registro ya bloqueado en modo de lectura: [HErrorLock](../WDLang4/3044099.md) devuelve <u><u><u>True</u></u></u> y [&lt;Fuente&gt;.Out](../WDLang4/1000024605.md) devuelve <u><u><u>True</u></u></u>.
	
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Acceso con JDBC: La gestión de las cerraduras no está disponible para las bases de datos a las que accede el JDBC.




Esta función se puede utilizar con archivos de datos, vistas HFSQL o consultas.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Go back to the 200th record of the Customer data file
// according to the CustName key
Customer.ReadLast(CustName)
Customer.Backward(CustName, 199)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.Backward([<Item> [, <Step> [, <Options>]]])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se retrocedió en el archivo de datos,

- <u><u><u><u>False</u></u></u></u> si se produce un error (bloqueo, fin del archivo de datos, etc.): el registro no se lee. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos, vista o consulta utilizada. 

**`<Item>: (Cadena de caracteres opcional)`**

Nombre del artículo según el cual se realiza la búsqueda. Si no se especifica este nombre, **&lt;Fuente&gt;.Backward** se encarga del último elemento de el archivo de datos utilizado por la última función HFSQL (función que empieza por "H").
![OLE DB](https://doc.pcsoft.fr/ext/images/us/OLEDB.png)![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) Todos los elementos utilizados para retroceder en el archivo se consideran elementos clave.

**`<Step>: (Entero opcional)`**

Número de pasos hacia atrás (lo que significa número de registros a examinar). Por defecto, &lt;Intervalo> = 1.

**`<Options>: (Constante o combinación de constantes de tipo Integer (opcional))`**

Configura:

- el modo de bloqueo aplicado a la Record leyó. Por Default, el bloqueo realizado corresponde al bloqueo actual.
	


|   |   |
| --- | --- |
| *hLockNo* | Sin bloqeuo: la Record puede ser leída o modificada por otra aplicación durante la lectura. |
| *hLockReadWrite* | Bloqueo de lectura/escritura: el Record que se lee actualmente no puede ser leído o modificado por otra aplicación. |
| *hLockWrite* | Bloqueo de escritura: el Record que se lee actualmente puede ser leído por otra aplicación pero no puede ser modificado por otra aplicación. |


	![OLE DB](https://doc.pcsoft.fr/ext/images/us/OLEDB.png)![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) Las opciones de bloqueo no tendrán efecto si los bloqueos no son soportados por el proveedor OLE DB o por Conector Nativo.

- la gestión de los duplicados durante la navegación. Por Default, todos los duplicados son examinados.
	


|   |   |
| --- | --- |
| *hDistinct* | Cuando se realiza una búsqueda en un elemento clave, se busca un solo Record si se encuentran duplicados. |




![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) &lt;Las opciones&gt; se ignoran.![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Acceso con JDBC: &lt;Las opciones&gt; se ignoran.

## Observaciones


### Lectura o no de la operación según un elemento clave
<a name="lectura_operacion_segun_elemento_clave_ELTPARAGRAPHE000339"></a>

En todos los casos, la operación de navegación debe haber sido inicializada por una de las siguientes funciones: [&lt;Fuente&gt;.ReadFirst](../WDLang4/1000025022.md), [&lt;Fuente&gt;.First](../WDLang4/1000025048.md), [&lt;Fuente&gt;.ReadSeekFirst](../WDLang4/1000025034.md), [&lt;Fuente&gt;.SeekFirst](../WDLang4/1000025055.md), [&lt;Fuente&gt;.ChangeKey](../WDLang4/1000024198.md), [&lt;Fuente&gt;.RestaurarPosición](../WDLang4/1000025061.md).

Si la posición utilizada es una posición clave:

- si la constante *hDistinct* no está activa, la función **&lt;Fuente&gt;.Backward** permite retroceder &lt;Intervalo&gt; valores de la clave a partir de la posición actual.

- si la constante *hDistinct* está activada, **&lt;Fuente&gt;.Backward** se desplaza hacia atrás de &lt;Paso&gt; valores de clave distintos desde la posición actual. Si se encuentran duplicados, se lee un solo duplicado.




Si el elemento utilizado no es un elemento clave:

- la constante *hDistinct* no está disponible,

- La función **&lt;Fuente&gt;.Backward** permite retroceder &lt;Intervalo&gt; registros activos a partir de la posición actual.



<a name="NOTE0_2"></a>


### Memos
<a name="memos_ELTPARAGRAPHE000395"></a>

Los memos asociados con el Record pueden ser leídos automáticamente (o no) cuando se lee el Record. [&lt;Fuente&gt;.SetMemo](../WDLang4/1000024987.md) se utiliza para personalizar esta operación de lectura de automatic.

Si los memos están soportados, los memos de texto asociados se leen cuando se lee la Record. Los memos binarios sólo se leen cuando se utilizan explícitamente ([&lt;Fuente&gt;.ExtractMemo](../WDLang4/1000024957.md)).
<a name="NOTE0_3"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000417"></a>

- El resultado de [&lt;Fuente&gt;.ReadSeekLast](../WDLang4/1000025024.md) debe ignorarse ([&lt;Fuente&gt;.ReadSeekFirst](../WDLang4/1000025034.md) debe utilizarse para las búsquedas realizadas por [&lt;Fuente&gt;.Found](../WDLang4/1000025076.md), [&lt;Fuente&gt;.Found](../WDLang4/1000025076.md), [&lt;Fuente&gt;.ReadSeek](../WDLang4/1000025023.md)).

- El número actual de Record es devuelto por [&lt;Fuente&gt;.RecNum](../WDLang4/1000025041.md).

- **&lt;Fuente&gt;.Backward** respeta el filtro activo actual (definido por [&lt;Fuente&gt;.Filter](../WDLang4/1000024960.md)).

- [&lt;Fuente&gt;.ChangeKey](../WDLang4/1000024198.md) se utiliza para modificar la clave de búsqueda mientras se posiciona en el record actual.

- **&lt;Fuente&gt;.Backward**(Cliente,Nombre) es equivalente a [&lt;Fuente&gt;.ReadPrevious](../WDLang4/1000025021.md)(Cliente,Nombre)

### Cierre
<a name="cierre_ELTPARAGRAPHE000481"></a>

Por defecto (no se especifica el bloqueo Variable), el bloqueo realizado corresponde al modo de bloqueo actual.

Si se solicita un bloqueo (constantes *hLockWrite* o *hLockReadWrite*), se leerá y bloqueará el registro solo si este aún no ha sido bloqueado por otra aplicación.




## HLockFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Bloquea un archivo de datos e impide que los demás sitios o aplicaciones accedan a él. El archivo de datos podrá desbloquearse con la función [HUnlockFile](../WDLang4/3044160.md).

Después de llamar a la función **HLockFile**, se pueden producir varios errores:

- Un registro del archivo de datos ya está bloqueado por otra aplicación: no se puede realizar el bloqueo.

- El archivo de datos ya está bloqueado por otra aplicación: no se puede realizar el bloqueo.




La función [HErrorLock](../WDLang4/3044099.md) devuelve <u><u><u>True</u></u></u> si se produce un problema de bloqueo.

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función se puede utilizar con archivos de datos HFSQL Client/Server.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Lock the CUSTOMER data file
IF HLockFile(CUSTOMER, hLockReadWrite) = False THEN
	IF HErrorLock() = True THEN
		Error("Operation not allowed")
	END
END
// Process on data file ...
...
// Restore access to data file
HUnlockFile()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HLockFile([<Data file> [, <Lock>]])`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> si se realizó el bloqueo,

- <u><u><u><u>False</u></u></u></u> si se produce un problema (por ejemplo, si el archivo ya está bloqueado, etc.). La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Data file>: (Cadena de caracteres opcional (con o sin comillas))`**

Nombre del archivo de datos utilizado. 
Si no se especifica este nombre, **HLockFile** utilizará el último archivo de datos utilizado por la última función de gestión de HFSQL (función que comienza con "H").

**`<Lock>: (Constante opcional de tipo Integer)`**

Permite definir el tipo de bloqueo que se realiza en el archivo de datos:



|   |   |
| --- | --- |
| *hLockWrite*<br>(valor predeterminado) | Bloqueo de escritura: otras aplicaciones pueden leer los registros pero no modificarlos. Ninguna otra aplicación puede definir un bloqueo en el archivo de datos. |
| *hLockReadWrite* | Bloqueo de lectura/escritura: otras aplicaciones no pueden leer ni modificar los registros del archivo de datos. Ninguna otra aplicación puede definir un bloqueo en el archivo de datos. |
| *hLockNo* | Ningún efecto |

<a name="NOTE0_1"></a>

## Observaciones


### Modificar el tipo de bloqueo en un archivo de datos
<a name="modificar_tipo_bloqueo_archivo_datos_ELTPARAGRAPHE000153"></a>

Para modificar el tipo de bloqueo de un archivo de datos, llame a la función **HLockFile** con el tipo de bloqueo deseado. No es necesario desbloquear el archivo de datos.
<a name="NOTE0_2"></a>


### Efecto del modo de bloqueo
<a name="efecto_del_modo_bloqueo_ELTPARAGRAPHE000164"></a>

Si el archivo de datos está en modo de bloqueo *hModeMono* (ver función [HMode](../WDLang4/3044039.md)), la función **HLockFile** no tiene ningún efecto.
<a name="NOTE0_3"></a>


### Error de bloqueo
<a name="error_bloqueo_ELTPARAGRAPHE000178"></a>

Si el archivo de datos (o uno de sus registros) ya está bloqueado, la solicitud de bloqueo se repite [H.NbRetry](../WDLang4/3044075.md) veces (50 por defecto).

Si el archivo de datos no puede bloquearse tras los intentos de ***H.NbRetry***, [HErrorLock](../WDLang4/3044099.md) devuelve <u><u><u><u>True</u></u></u></u>.

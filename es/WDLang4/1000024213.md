


## &lt;Fuente&gt;.Declare (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Declara una descripción del archivo de datos (que esta en el análisis) en el proyecto actual. Esta importación es temporal y se puede cancelar con la función [&lt;Fuente&gt;.CancelDeclaration](../WDLang4/1000024192.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Declare a SupplierWD data file from the SALESMGT analysis
Supplier.Declare("\Examples\SalesMgt\SALESMGT.WDD", "", "SupplierWD")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.Declare(<Full path of the WDD file> [, <WDD password> [, <Alias> [, <Options>]]])`
---

**`<Result>: (: Boolean)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre lógico del archivo de datos a importar, utilizado en el análisis externo.

**`<Full path of the WDD file>: (Cadena de caracteres)`**

Ruta completa de acceso al archivo del análisis (archivo .WDD) que contiene la descripción del archivo de datos HFSQL a importar (hasta 260 caracteres).
Este archivo puede corresponder a un archivo WDD creado con una versión anterior de WINDEV o WEBDEV.

**`<WDD password>: (Cadena de caracteres opcional)`**



- Contraseña asociada al análisis. 

- Cadena vacía ("") si no se utiliza ninguna contraseña.




**`<Alias>: (Cadena de caracteres opcional)`**



**`<Options>: (Constante opcional de tipo Integer)`**

Opciones de búsqueda:


|   |   |
| --- | --- |
| *hDisk*<br>(Valor predeterminado) | El análisis &lt;Ruta completa del WDD&gt; se busca en el disco (caso más común). |
| *hWDL* | El análisis &lt;Ruta completa del WDD&gt; se busca según las siguientes prioridades: <br><br>1. En la biblioteca principal (archivo .WDL)<br><br>2. En las bibliotecas según el orden de carga, y luego en el disco. <br><br>3. En la biblioteca principal del componente. <br><br>4. En las bibliotecas secundarias del componente según el orden de carga.<br><br><br> |

<a name="NOTE0_1"></a>

## Observaciones


### Varios
<a name="varios_ELTPARAGRAPHE000271"></a>

- **No es necesario tener un análisis relacionado con la aplicación actual para importar una descripción del archivo de datos**. 

- Todas las funciones HFSQL se pueden utilizar en el archivo de datos importado, excepto la gestión de la integridad.

- Se pueden importar tantas descripciones como sea necesario.

- Esta función permite utilizar archivos de datos descritos en análisis anteriores de WINDEV y WEBDEV (WINDEV 4.1 a 5.5, WEBDEV 1.0 y 1.5).

### Función &lt;Fuente&gt;.Declare en análisis de versiones anteriores
<a name="funcion_docparampagetitleshort_analisis_versiones_anteriores_ELTPARAGRAPHE000286"></a>

El motor Hyper File versión 4.1 a 5.5 requiere un análisis abierto para funcionar (a diferencia de HFSQL Classic), por lo que el análisis del primer archivo de datos al que se accede se abre en el motor Hyper File 5.

Se debe utilizar un alias para acceder a un archivo de datos con las siguientes características: 

- archivo de datos de otro análisis.

- archivo de datos con el mismo nombre lógico que uno de los archivos de datos del análisis abierto.




**Atención**: Los archivos de datos descritos en análisis anteriores de WINDEV y WEBDEV (WINDEV 4.1 a 5.5, WEBDEV 1.0 y 1.5) son manipulados por el motor Hyper File versión 5.5: WD553HF.DLL.
Este módulo solo existe en la versión de 32 bits. 
Por lo tanto, una aplicación que deba consultar datos en formato 5.5 o anterior debe:

- ser compilada en 32 bits y no en 64 bits,

- limitar sus capacidades a las que están disponibles en WINDEV versión 5.5. Por ejemplo, la constante *hWDL* no existía en la versión 5.5: por lo tanto, no se puede utilizar en un archivo Hyper File 5.5.

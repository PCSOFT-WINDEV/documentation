
## ReplicSynchronize( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Sincroniza los datos correspondientes a una réplica. Esta réplica fue creada por el editor de réplicas (ReplicEdit).  






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Starts the synchronization
ReplicSynchronize("MyReplication")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Sincronización en modo directo

`<Result> = ReplicSynchronize(<Replication name>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la sincronización,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para más detalles sobre el problema, utilice [ErrorInfo](../WDLang1/3013008.md).




**`<Replication name>: (Cadena de caracteres)`**

Nombre de la replicación a sincronizar. Este nombre corresponde al nombre del archivo ".wer" creado por ReplicEdit. Este archivo debe ser encontrado al lado del ejecutable.  


<a name="SYNTAX2"></a>

### Sincronización con un servidor de replicación

`<Result> = ReplicSynchronize(<Replication name> [, <Options>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la sincronización,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para más detalles sobre el problema, utilice [ErrorInfo](../WDLang1/3013008.md).




**`<Replication name>: (Cadena de caracteres)`**

Nombre de la replicación a sincronizar. Este nombre corresponde al nombre de la publicación de un servidor de replicación.

**`<Options>: (Constante opcional de tipo Integer)`**

Tipo de replicación a realizar. Estas opciones se tienen en cuenta para una replicación con un servidor de replicación. 


|   |   |
| --- | --- |
| *replicaImmediate* | La réplica se crea inmediatamente, durante la sincronización.<br><br>Este modo de replicación puede ser bastante largo, pero garantiza que se actualice desde la base de datos maestra. |
| *replicaNoProgressBar* | Por Default, se muestra una barra de progreso durante la sincronización. Esta barra de progreso está adaptada a la plataforma actual. <br>Esta constante le permite no mostrar más la barra de progreso.<br>Esta constante puede combinarse con las constantes *replicaImmediate* y *replicaPeriodic*. |
| *replicaPeriodic*<br> (Valor predeterminado) | La réplica que se tiene en cuenta es la que está disponible en el servidor.<br><br>**If la réplica se crea periódicamente en el servidor**, se tiene en cuenta la última réplica creada.<br><br>**If la réplica no se crea periódicamente en el servidor**, se tiene en cuenta la última réplica generada. Esta última réplica fue generada justo después de la última sincronización.<br><br>Este modo de replicación es rápido, pero no garantiza una "actualización completa" de la base de datos maestra. |


**Observación**: debe utilizar el mismo tipo de réplica que el definido en el servidor. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Función de replicación y WithSpace
<a name="funcion_replicacion_withspace_ELTPARAGRAPHE000131"></a>

La replicación no es compatible con [WithSpace](../WDLang4/3044142.md) en el código de la aplicación cliente. Esta función cambia el comportamiento de HFSQL. El uso de esta función en la aplicación cliente puede afectar a la eficiencia de la replicación.

A partir de la versión 21 actualización 3 (versión 210065), las funciones de sincronización (replicación universal programada o asistida) devuelven un error a la aplicación cliente si se ha aplicado [WithSpace](../WDLang4/3044142.md) a al menos uno de los archivos replicados. 





<a name="NOTE0_2"></a>


### Replicación a través de un servidor de replicación
<a name="replicacion_traves_servidor_replicacion_ELTPARAGRAPHE000149"></a>

Durante una replicación a través de un servidor de replicación (especialmente para el [replicación de datos móviles (Android o iOS)](../WDLang4/1000021071.md)), el servidor web que aloja el servidor de replicación debe estar configurado para permitir la carga y descarga de datos. Por lo tanto, la opción "Longitud máxima autorizada para el contenido" configurada en "Filtrado de solicitudes" de un servidor IIS debe ser mayor que el tamaño de los datos de los suscriptores (RPA).

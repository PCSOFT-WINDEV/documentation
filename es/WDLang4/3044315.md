


## Gestión de grupos y usuarios (HFSQL Client/Server)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000350"></a>
Para simplificar la gestión de usuarios y grupos de usuarios, el motor HFSQL Client/Server utiliza varias variables que permiten definir fácilmente las características de los grupos y usuarios.











<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Variables para la gestión de usuarios
<a name="variables_para_gestion_usuarios_ELTTEXTE000380"></a>


### ¿Qué variables?
<a name="¿que_variables_ELTPARAGRAPHE000058"></a>

Las siguientes variables se utilizan para gestionar los usuarios:


|   |   |
| --- | --- |
| Contraseña de usuario | Cadena de caracteres de sólo lectura.<br><br>Contraseña de usuario. Esta contraseña es sensible al caso y a los caracteres acentuados. |
| Fecha de caducidad de la unidad de manipulación | Fecha de sólo lectura.<br><br>Fecha de expiración de la contraseña. Esta fecha se deduce del período de validez de la contraseña y de la fecha de su última modificación. |
| Grupos de unidades de manipulación | Cadena de caracteres.<br><br>Nombre de los grupos a los que el usuario es associated. Los diferentes grupos están separados por caracteres CR (Carriage Return). |
| HUser. Email | Cadena de caracteres. <br><br>Envía un correo electrónico a Address del usuario. |
| HUser. Phone | Cadena de caracteres. <br><br>Número de Phone del usuario. |
| HUser.ActiveDirectory | Booleana (False de Default).<br><br>Si esta variable es set a True, el usuario está usando una autenticación a través del directorio Active. |
| HUser.cuenta habilitada | Booleana (True de Default).<br><br>Si este Variable es set a False, la cuenta existe pero no está habilitada. Se utiliza, por ejemplo, para desactivar una cuenta de usuario durante un período determinado. |
| HUser.description | Cadena de caracteres. <br><br>El usuario description. |
| HUser.login | Cadena de caracteres<br><br>Login de usuario (hasta 50 caracteres).<br><br>El inicio de sesión de usuario no distingue entre mayúsculas y minúsculas. |
| HUser.photo | Cadena binaria correspondiente a la foto del usuario.<br><br>Le recomendamos que use dSaveImagexxx para asignar una imagen a esta Variable. |
| Información de creación de unidades de manipulación | Cadena de caracteres de sólo lectura.<br><br>Nombre de la persona que creó la cuenta de usuario y fecha de creación.<br><br>Estos detalles están separados por una tabulación (TAB). La fecha de creación corresponde a un string de 8 dígitos. |
| IP autorizada por el fabricante de la unidad de manipulación | Cadena de caracteres<br><br>set de elementos representativos de las direcciones IP permitidas al usuario. Los diferentes elementos están separados por caracteres CR (Carriage Return).<br><br>Este Variable puede corresponder:<br><br>- Una Address IPv4 (68.12.53.26 por ejemplo) o varias Address IPv4 es separada por caracteres CR (Carriage Return).<br><br>- Un Address IPv6 (feco::40 por ejemplo) o varios Address es IPv6 separados por caracteres CR (Carriage Return).<br><br>- Una serie de direcciones de IPv4 en formato CIDR. Por ejemplo: 68.12.0.0/16<br><br>- Una serie de direcciones de IPv6 en formato CIDR. Por ejemplo: fec0:56::0/32<br><br>- Un nombre DNS FQDN (myserver.pcsoft.com por ejemplo) o varios nombres DNS separados por caracteres CR (Carriage Return).<br><br>- Un nombre de subdominio (.pcsoft.com por ejemplo: prestar atención al punto) o a varios nombres de subdominio separados por caracteres CR (Carriage Return).<br><br><br> |
| Nombre completo del usuario | Cadena de caracteres. <br><br>Nombre completo del usuario (hasta 50 caracteres). |
| Período de validez de la unidad de manipulación | Entero (0 por Default, sin período de validez).<br><br>Período de validez de la contraseña (en días). Este período de validez se tiene en cuenta desde la fecha de la última modificación de la contraseña. |
| Superusuario de unidad de manipulación | Booleana (False de Default)<br><br>Si este Variable es set a True:<br><br>- Los derechos otorgados a este usuario no son comprobados. Por lo tanto, todas las acciones pueden ser realizadas por este usuario.<br><br>- La contraseña de este usuario nunca caduca.<br><br><br> |


<a name="NOTE2_2"></a>


### ¿Cuándo debo utilizar estas variables?
<a name="¿cuando_debo_utilizar_estas_variables_ELTPARAGRAPHE000193"></a>

Se utilizan las variables para la gestión de usuarios:

- al añadir un nuevo usuario ([HAddUser](../WDLang4/3044281.md)).

- al buscar información sobre el usuario ([HInfoUser](../WDLang4/3044285.md)).

- al modificar la información sobre un usuario ([HModifyUser](../WDLang4/3044309.md)).

- al reiniciar estas variables ([HResetUser](../WDLang4/3044293.md)).



<a name="NOTE2_3"></a>


### Usuario predeterminado
<a name="usuario_predeterminado_ELTPARAGRAPHE000217"></a>

Por Default, el siguiente usuario se crea automáticamente cuando se utiliza una base de datos Cliente/Servidor:

- Nombre de usuario: Admin

- Contraseña: "" (cadena vacía)

- Usuario definido como superusuario

## Variables para la gestión de grupos
<a name="variables_para_gestion_grupos_ELTTEXTE000416"></a>
Las siguientes variables se utilizan para gestionar grupos de usuarios:


|   |   |
| --- | --- |
| HGroup.ActiveDirectory | Booleana (False de Default).<br><br>Si esta Variable es set a True, el grupo está usando una autenticación a través del Directorio de Active. |
| HGroup.AuthorizedIP | Cadena de caracteres<br><br>set de elementos representativos de las direcciones IP autorizadas para el grupo. Los diferentes elementos están separados por caracteres CR (Carriage Return).<br><br>Este Variable puede corresponder:<br><br>- Una Address IPv4 (68.12.53.26 por ejemplo) o varias Address IPv4 es separada por caracteres CR (Carriage Return).<br><br>- Un Address IPv6 (feco::40 por ejemplo) o varios Address es IPv6 separados por caracteres CR (Carriage Return).<br><br>- Una serie de direcciones de IPv4 en formato CIDR. Por ejemplo: 68.12.0.0/16<br><br>- Una serie de direcciones de IPv6 en formato CIDR. Por ejemplo: fec0:56::0/32<br><br>- Un nombre DNS FQDN (myserver.pcsoft.com por ejemplo) o varios nombres DNS separados por caracteres CR (Carriage Return).<br><br>- Un nombre de subdominio (.pcsoft.com por ejemplo: prestar atención al punto) o a varios nombres de subdominio separados por caracteres CR (Carriage Return).<br><br><br> |
| HGroup.description | Cadena de caracteres.<br><br>Grupo description. |
| Nombre HGroup | Cadena de caracteres. <br><br>Nombre del grupo (hasta 50 caracteres). |
| Usuarios de HGroup. | Cadena de caracteres. <br><br>Lista de usuarios associated con el grupo. Los nombres de los usuarios están separados por caracteres CR (Carriage Return). |


<a name="NOTE3_2"></a>


### ¿Cuándo debo utilizar estas variables?
<a name="¿cuando_debo_utilizar_estas_variables_ELTPARAGRAPHE000284"></a>

Se utilizan las variables para la gestión de grupos:

- al añadir un nuevo grupo ([HAddGroup](../WDLang4/3044280.md)).

- al buscar información sobre el grupo ([HInfoGroup](../WDLang4/3044283.md)).

- al modificar la información de un grupo ([HModifyGroup](../WDLang4/3044308.md)).

- al reiniciar estas variables ([HResetGroup](../WDLang4/3044292.md)).




## &lt;mongoCollection Variable&gt;.ModifyOne (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica una Document en una colección de la base de datos de MongoDB.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Collection is mongoCollection
let sFilter = "{ ""name"": ""Andy"" }"
let sModification = "{ ""$inc"": { ""score"": 1 } }"
let d = Collection.ModifyOne(sFilter, sModification)
```
<a name="Example2"></a>

```wl
gclCollection is mongoCollection

// Find the first contact whose "FirstName" member is set to "Jack" and modify it to "Joey" 
// Search filter 
sFilter is string = "{ ""FirstName"": ""Jack"" }"
// Modification criterion
sModification is string = "{ ""$set"": {""FirstName"": ""Joey"" } }"
gclCollection.ModifyOne(sFilter, sModification)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = <Collection>.ModifyOne(<Filter> , <Modification> [, <Sort>] [, <Option>])`
---

**`<Result>: (Variant)`**

Document antes o después de la modificación (según la opción seleccionada).

**`<Collection>: (Variable de tipo mongoCollection)`**

Nombre de la [mongoCollection](../WDLang4/1000022411.md) Variable que representa la colección MongoDB en la que se realizará la modificación.

**`<Filter>: (Tipo correspondiente al filtro utilizado)`**

description de filtros en formato MongoDB que se aplicará para seleccionar la Document a modificar. Este filtro corresponde a: 

- un Document en formato JSON, en formato de cadena de caracteres. 

- un Document en formato BSON, en formato buffer. 

- una variante.




**`<Modification>: (Tipo correspondiente al formato utilizado)`**

description de las modificaciones que se harán en la Document correspondientes al filtro. Corresponde a: 

- un Document en formato JSON, en formato de cadena de caracteres. 

- un Document en formato BSON, en formato buffer. 

- a variant.




**`<Sort>: (Tipo opcional correspondiente al formato utilizado)`**

Opciones de clasificación utilizadas para definir la primera Document que se modificará cuando varias Document correspondan al filtro. Corresponde a: 

- un Document en formato JSON, en formato de cadena de caracteres. 

- un Document en formato BSON, en formato buffer. 

- una variante.




**`<Option>: (Constante opcional de tipo Integer)`**

Opciones de modificación: 


|   |   |
| --- | --- |
| *mongoAddIfNotFound* | Se utiliza para añadir el Document si no existe en la colección. |
| *mongoReturnValueAfter* | Se utiliza para devolver el Document después de la modificación. <br><br>Por defecto, si no se usa esta constante, el Document devuelto corresponde al Document antes de la modificación. |




## &lt;Variable Connection&gt;.AddLink (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade una regla de integridad entre dos archivos de datos en el servidor. Esta regla de integridad ha sido descrita previamente por una Variable de tipo [Descripción de la relación](../WDLang4/1514039.md). Todas las aplicaciones cliente deben cumplir con estas reglas de integridad.

**Observación**: Tiene la capacidad de describir reglas de integridad entre bases de datos.  

La adición será efectiva para la aplicación que la realizó una vez que la conexión se haya cerrado y reabierto.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Describe the connection
MyConnection is Connection

// Describe the connection
MyConnection.User = "USER"
MyConnection.Password = "PASSWORD"
MyConnection.Server = "MYSERVER" 
MyConnection.Database = "Database"
MyConnection.Provider = hAccessHFClientServer
MyConnection.Access = hORead

// Open the connection
MyConnection.OpenConnection()
...
MyLink is link description
// Describe the "LIVES" link
MyLink.Name = "LIVES"
MyLink.SourceFile = ".\MyDatabase\city.fic"
MyLink.LinkedFile = ".\MyDatabase\customer.fic"
MyLink.SourceKey = "CITYNAME"
MyLink.LinkedKey = "CUSTCITY"
// Default link (0-1, 0-1)
MyConnection.AddLink(MyLink)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source connection>.AddLink([<Linked connection>, ] <Link>)`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Source connection>: (Variable de tipo Connection)`**

Nombre de la [Conexión](../WDLang4/1514073.md) Variable que describe la conexión a la que pertenece el archivo fuente de enlace.

**`<Linked connection>: (Cadena de caracteres opcional o variable de tipo Connection)`**

Conexión a la que pertenece el fichero relacionado del enlace. Esta conexión corresponde a: 

- la conexión definida en el editor de análisis o con el [HDescribeConnection](../WDLang4/3044205.md) o el [&lt;Variable Connection&gt;.OpenConnection](../WDLang4/1000023962.md).

- una variable de tipo [Conexión](../WDLang4/1514073.md). 


Si no se especifica este parámetro, ambos archivos de datos pertenecerán a la misma conexión (&lt;Source connection&gt;).

**`<Link>: (Variable de tipo Link Description)`**

Nombre de la [Descripción de la relación](../WDLang4/1514039.md) Variable que contiene las características del enlace a crear.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Descripción de la relación
<a name="descripcion_relacion_ELTPARAGRAPHE000257"></a>

En el enlace description, los nombres de los ficheros de datos implicados en el enlace (propiedades [SourceFile](../Proprietes/2512034.md) y [LinkedFile](../Proprietes/2512031.md)) deben corresponder a los nombres físicos de los ficheros de datos, relativos a la Base de Datos utilizada. La base de datos de cada archivo es devuelta por &lt;Source connection&gt; y &lt;Linked connection>.
<a name="NOTE0_2"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000273"></a>

- [HDescribeLink](../WDLang4/3044191.md) se utiliza para describir un enlace temporal en una base de datos ISAM (no en modo Cliente/Servidor) o sólo para la aplicación cliente (no en el servidor).

- [&lt;Variable Connection&gt;.DeleteLink](../WDLang4/1000022759.md) se utiliza para eliminar un enlace en el servidor.

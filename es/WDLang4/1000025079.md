


## &lt;Fuente&gt;.CheckMemo (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Verifica la integridad estructural de los memos de un archivo de datos HFSQL.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF Customer.CheckMemo(PROGBAR_ProgBar1) <> "" THEN
	Info("Memo corrupted")
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = <Source>.CheckMemo([<Progress Bar>])`
---

**`<Result>: (Cadena de caracteres)`**



- Vaciar la cadena ("") si no hay problema. 

- En caso de problema, se debe encadenar en el siguiente formato: 
	
	```txt
	<Item 1> + TAB + <Record number 1> + CR + ...
	<Item N> + TAB + <Record number N>
	```
 donde: 

	- &lt;Item X&gt; es la posición de nota, 

	- &lt;número de Record&gt; es el número del Record que contiene el elemento de Memo. 





![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) Si la lista de bloques libres está corrupta, &lt;Resultado&gt; contendrá:

```txt
<TAB> + "-1"
```
&lt;Resultado&gt; tendrá el siguiente formato: 

```txt
<TAB> + "-1"+ CR + <Item 1> + TAB + <Record number 1> + CR + ...
<Item N> + TAB + <Record number N>
```


**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos HFSQL a comprobar.

**`<Progress Bar>: (Nombre de ventana o control opcional)`**



- Nombre de la ventana en la que se mostrará la barra de progreso. 

- Nombre de la control Barra de progreso.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- La lista de bloques libres sólo se verifica en HFSQL Client/Server.

- **&lt;Fuente&gt;.CheckMemo** es más rápido que [&lt;Fuente&gt;.Index](../WDLang4/1000025060.md) con la constante *hCheckMemo*.. 

- Si se detecta un problema, puede utilizar [&lt;Fuente&gt;.Index](../WDLang4/1000025060.md) con la constante *hCheckMemo*.

- A partir de la versión 27, la verificación de los memorandos es mejor y más completa, lo que puede dar lugar a tiempos de procesamiento más largos.

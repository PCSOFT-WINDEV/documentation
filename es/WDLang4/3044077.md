
## HErrorDuplicates (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite verificar si se produjo un error duplicados.

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Esta función se puede utilizar cuando la función [HOnError](../WDLang4/3044017.md) desactiva la gestión automática de errores. De forma predeterminada, WINDEV propone una [gestión automática de duplicados](../WDLang4/3044188.md).






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003044077&name=herrorduplicates_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Manage a duplicate error
IF HModify(Customer) = False THEN
	IF HErrorDuplicates() = True THEN 
		sItemName is string
		sItemName = HErrorInfo(hErrItem)
		Error("Duplicate: this record cannot be modified", ...
					"Item associated with the error: " + sItemName, ...
					"Value of the item: " + {"Customer. " + sItemName})
	END
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HErrorDuplicates()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se produjo un error de duplicados,

- <u><u><u><u>False</u></u></u></u> en caso contrario (sin errores).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Por defecto, la verificación de duplicados está activa. Para activar o desactivar la gestión de duplicados, utilice la función [HSetDuplicates](../WDLang4/3044057.md).

- Esta función reemplaza a la función **HDuplicates**, que se conserva por motivos de compatibilidad con WINDEV 5.5.

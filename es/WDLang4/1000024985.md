


## &lt;Fuente&gt;.SetDuplicates (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Activa o desactiva la gestión de duplicados en una clave única.

**Observaciones**: 

- Por defecto, la verificación de duplicados está activa.

- Esta función no está disponible en las bases de datos a las que accede OLE DB o Conectores Nativos (también denominadas Accesos Nativos).. 



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Disable the management of duplicates on the Customer data file for the "Name" key
Customer.SetDuplicates(Name, False)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.SetDuplicates([<Key item>, ] <Activation>)`
---

**`<Result>: (Booleano)`**

Anterior modo de gestión de duplicados (antes de llamar a la función **&lt;Fuente&gt;.SetDuplicates**): 

- <u><u><u><u>True</u></u></u></u> si la gestión de duplicados estaba activa,

- <u><u><u><u>False</u></u></u></u> si la gestión de duplicados no estaba activa, (o si la clave especificada es una clave con duplicados). 


Si &lt;Elemento clave&gt; = "\*", &lt;Resultado&gt; se establece en <u><u><u><u>True</u></u></u></u> si en el anterior modo de gestión de duplicados, la prueba estaba activa en al menos un elemento. 


**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos utilizado.

**`<Key item>: (Cadena de caracteres opcional)`**

Nombre del elemento clave unívoco utilizado. 
Si no se especifica este parámetro o si corresponde a "\*", se tiene en cuenta la gestión de los duplicados para todos los elementos clave del archivo de datos especificado.

**`<Activation>: (Booleano)`**

Modo de gestión: 

- <u><u><u><u>True </u></u></u></u> si se debe activar la gestión de duplicados en el elemento,

- <u><u><u><u>False STRIMEULB</u></u></u></u> si esta gestión debe ser desactivada. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Gestión de duplicados habilitada
<a name="gestion_duplicados_habilitada_ELTPARAGRAPHE000283"></a>

Si se habilita la gestión de duplicados en una clave definida como clave única, la unicidad se comprueba automáticamente cada vez que se añade o modifica un Record en el archivo de datos.

Por lo tanto, si no se verifica la unicidad al agregar ([&lt;Fuente&gt;.Add](../WDLang4/1000024187.md)) o modificar ([&lt;Fuente&gt;.Modify](../WDLang4/1000025038.md)) un registro, este no se agrega o modifica; la función para agregar/modificar registros devuelve un error y la función [HErrorDuplicates](../WDLang4/3044077.md) devuelve <u><u><u><u>True</u></u></u></u>.

**Observación**: Sólo [&lt;Fuente&gt;.Add](../WDLang4/1000024187.md) y [&lt;Fuente&gt;.Modify](../WDLang4/1000025038.md) pueden generar un error de duplicación.
<a name="NOTE0_2"></a>


### Generalizar la comprobación de duplicados
<a name="generalizar_comprobacion_duplicados_ELTPARAGRAPHE000324"></a>

Para generalizar el modo de gestión de los duplicados a todos los artículos que se encuentran en un archivo de datos, utilice el metacaracter "\*". 
Por ejemplo:

```txt
<Data file>.SetDuplicates("*", <Activation>)
```

<a name="NOTE0_3"></a>


### Desactivar la verificación de duplicados
<a name="desactivar_verificacion_duplicados_ELTPARAGRAPHE000353"></a>

La desactivación de la comprobación de duplicados significa la posibilidad de no cumplir con las reglas de unicidad especificadas durante la description de los archivos de datos en la editor de análisis. Por lo tanto, la desactivación de la comprobación de duplicados debe hacerse con mucho cuidado.: debe usarse con precaución.


## Controlador ODBC para HFSQL Classic y para HFSQL Client/Server (Linux)
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000171"></a>
El controlador ODBC para HFSQL Classic (monopuesto o multipuesto) se utiliza para acceder a una base de datos HFSQL Classic desde un software de base de datos externo que admita los accesos por ODBC.

El controlador ODBC para HFSQL Client/Server permite acceder a una base de datos HFSQL Client/Server desde un software de base de datos externo que admita el acceso mediante ODBC.

El controlador está disponible en lectura/escritura. Una aplicación escrita en lenguaje externo puede leer y escribir en archivos HFSQL.

El controlador ODBC para HFSQL Classic y HFSQL Client/Server es un controlador ODBC de nivel 3. Consulte una documentación específica para obtener más detalles.

**Observaciones**: 

- Esta ayuda Page explica cómo instalar el controlador ODBC para HFSQL Classic y HFSQL Client/Server para Linux. La instalación del controlador ODBC para HFSQL Classic y HFSQL Client/Server para Windows se presenta en el [Controlador ODBC para HFSQL Classic y para HFSQL Client/Server](../WDLang4/3044179.md) Page. 

- Para dar acceso a los datos HFSQL a través del controlador ODBC, el controlador ODBC para Linux debe instalarse manualmente en el equipo donde están instalados los datos HFSQL.

## Instalación
<a name="instalacion_ELTTEXTE000195"></a>


### Condición previa: instalación del iODBC manager
<a name="condicion_previa_instalacion_del_iodbc_manager_ELTPARAGRAPHE000030"></a>

El controlador ODBC en Linux utiliza el gestor iODBC ([http://www.iodbc.org/](http://www.iodbc.org/)). Este programa debe ser instalado antes de usar el controlador. 

Para instalar este controlador en Ubuntu, todo lo que tiene que hacer es instalar el paquete libiodbc-dev: 


```txt
apt-get install libiodbc2-dev
apt-get install iodbc
```



### Instalación del controlador
<a name="instalacion_del_controlador_ELTPARAGRAPHE000042"></a>

El controlador ODBC en Linux se suministra en versiones de 32 y 64 bits en el directorio "INSTALL\\ODBC" del directorio instalación de WINDEV, WEBDEV o WINDEV Mobile: 

- En 32 bits: wxpackodbclinux32.zip

- En 64-bit: wxpackodbclinux64.zip




Para instalarlo: 

- Descomprimir el paquete solicitado en un directorio local ("/opt/wdhfo" por ejemplo). 

- Ejecute el script "install.sh" de la siguiente manera: 
	
	```txt
	>cd /path/to/driver/
	>sudo ./install.sh
	```





Observación: Este script registra el controlador junto a iODBC. Para más detalles, utilice: 

```txt
install.sh -help
```

## Configuración de la base de datos
<a name="configuracion_base_datos_ELTTEXTE000225"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000065"></a>

Se pueden utilizar dos métodos para configurar una fuente de datos: 

- utilizar [la herramienta gráfica de iODBC](#NOTE4_2). 

- realizar la configuración [de forma manual](#NOTE4_3). 



<a name="NOTE4_2"></a>


### Configuración de una fuente de datos mediante la herramienta gráfica de iODBC
<a name="configuracion_una_fuente_datos_mediante_herramienta_grafica_iodbc_ELTPARAGRAPHE000081"></a>

- Iniciar la herramienta "iodbcadm-gtk. 

- La siguiente ventana se abre: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=iodbc1.gif)


- Haga clic en "Agregar". 

- En la lista que se muestra, seleccione el controlador HFSQL y haga clic en "Finish".. La siguiente ventana se abre: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=iodbc2.gif)


- Introducir las teclas manualmente: 

	- Para una conexión ISAM: 

		- FileDir: Directorio de archivos.FIC

		- Análisis: ruta de WDD. Este parámetro es opcional




- Para una conexión Cliente/Servidor: 

	- Nombre del servidor: Nombre del servidor

	- Puerto del servidor: Puerto a utilizar

	- Base de datos: Nombre de la base de datos

	- UID: Nombre de usuario

	- PWD: Contraseña de usuario

	- Análisis: ruta de WDD. Este parámetro es opcional. <br>![Ejemplo de una conexión ISAM](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=iodbc3.gif)

- Validar el diálogo dos veces. 




La fuente está configurada. Se puede probar con el programa "iodbctest"..

```txt
iodbctest "DSN=MyODBCSource"
```

<a name="NOTE4_3"></a>


### Configuración manual
<a name="configuracion_manual_ELTPARAGRAPHE000114"></a>

- Crear un archivo ".odbc.ini" en la raíz del directorio "home. 

- Para cada fuente, añada las siguientes líneas a este archivo: 

	- Fuente ISAM HFSQL:
			
		```txt
		[ODBC Data Sources] 
		<Source Name> = HFSQL 
		[<Source Name>] 
		Driver = /path/vers/WD220hfo64.so 
		FileDir = <path of files>
		```


	- Fuente HFSQL C/S: 
			
		```txt
		[ODBC Data Sources] 
		<Source Name> = HFSQL 
		[<Source Name>] 
		Server Name = <Name of server>
		Server Port = <Port to use>
		Database = <Name of database>
		UID = <Name of user>
		PWD = <Password of user>
		```


## Instalación y configuración de HFSQL Cluster
			



## Configuración necesaria
<a name="configuracion_necesaria_ELTTEXTE000221"></a>
Para instalar un Cluster HFSQL, debe tener: 

- varios ordenadores correspondientes a los nodos del cluster. 

- un ordenador utilizado para alojar al coordinador (este ordenador puede ser uno de los nodos del clúster). 

- to DNS server. De hecho, el acceso al cluster se hace compartiendo el nombre de una máquina virtual associated con las direcciones IP de los diferentes nodos.




**Recordatorio**: Un clúster debe incluir al menos tres ordenadores. Para que el coordinador se instale en el mismo ordenador que un nodo, el clúster debe incluir al menos tres nodos. Si el grupo contiene sólo dos nodos, el coordinador debe encontrarse necesariamente en un tercer ordenador.

**Atención**: Todos los servidores y ordenadores deben estar ejecutando Windows. Los clusters no están disponibles en Linux.

### Consejos
<a name="consejos_ELTPARAGRAPHE000022"></a>

Para evitar diferencias de rendimiento, la configuración de cada uno de los nodos del clúster debe tener el mismo orden de potencia (procesador y memoria). 

**La configuración mínima** necesario para instalar HFSQL Cluster es el siguiente (para cada nodo):

- Sistema operativo: todas las versiones de servidor de Windows soportadas por Microsoft.  Se admiten las ediciones de 32 y 64 bits. 

- Memoria: 512 MB

- Disco: 20 MB para los elementos del cluster + tamaño de las bases de datos

- Red: 100 Mbps entre los nodos. Las direcciones IP de los nodos deben ser fijas.




**La configuración recomendada** para instalar HFSQL Cluster depende del tamaño de las bases de datos, su complejidad y el número de clientes. La siguiente configuración será robusta en la mayoría de los casos (para cada nodo):

- Sistema operativo : Windows Server en una versión actualmente soportada por el editor.
	Se recomienda utilizar la función : 

	- la versión más reciente para incorporar los últimos protocolos de seguridad,

	- una versión que no está al final de su ciclo de vida ([https://docs.microsoft.com/en-us/lifecycle/faq/general-lifecycle](https://docs.microsoft.com/en-us/lifecycle/faq/general-lifecycle)).




- Procesador: se recomienda un procesador multinúcleo

- Memoria: 4 GB

- Disco: un sistema RAID




**Observaciones**: 

- El ordenador que alberga el servicio de coordinación del cluster requiere una configuración menos potente. Típicamente, el coordinador puede ser localizado:

	- en uno de los nodos,

	- en el mismo servidor que el DNS de la red local,

	- en otra computadora.




- El puerto 4997 utilizado por el coordinador del cluster debe estar abierto en el firewall de Windows.

- Los puertos utilizados por los nodos y el puerto 4998 deben estar abiertos en el firewall de Windows de cada servidor.








## Creación de un cluster
<a name="creacion_cluster_ELTTEXTE000266"></a>
Para crear un cluster:

1. Inicie el clúster HFSQL instalación asistente.

2. Acepta los términos del acuerdo de licencia para continuar con el instalación.

3. Seleccione "Crear un nuevo cluster".

4. Un texto breve resume el principio de un grupo temático y las medidas para su aplicación. Ir al siguiente avión. 

5. Puede elegir convertir un servidor HFSQL existente en el primer nodo del nuevo clúster o instalar un clúster en blanco.. Al convertir un servidor existente, especifique el nombre del servidor (y el puerto de escucha de HFSQL).

6. Introduzca los parámetros instalación del coordinador. Estos parámetros son:

	- El nombre de el equipo en el que debe instalarse el servicio del coordinador.

	- La contraseña para gestionar el clúster. Esta contraseña se utilizará en el [Centro de Control HFSQL](../CCHF/3540001.md) o a través de la programación para las operaciones de mantenimiento del cluster (añadir nodos, parar y reiniciar, etc.)

	- El nombre DNS del cluster: es el nombre a partir del cual se podrá acceder al clúster en la red. Para obtener más información, consulte ["Configurar el DNS para el acceso al clúster"](#DNS).




7. Describir la composición del cluster. Puede añadir los diferentes nodos del cluster. 
	Si ha elegido convertir un servidor HFSQL existente, éste se propondrá automáticamente.
	**Recordatorio**: Un clúster debe incluir al menos tres ordenadores. Para que el coordinador se instale en el mismo ordenador que un nodo, el clúster debe incluir al menos tres nodos. Si el grupo contiene sólo dos nodos, el coordinador debe encontrarse necesariamente en un tercer ordenador.

8. Introduzca el número de puerto que se utilizará para las conexiones HFSQL. Este número de puerto será compartido por todos los nodos del cluster. Por lo tanto, debe estar disponible en cada equipo que aloje un nodo de clúster.. Ir al siguiente avión. 

9. Se muestra el resumen de la configuración seleccionada. Puede volver a los pasos anteriores de la asistente para modificar la configuración del cluster.

10. Haga clic en "Siguiente". Se crea automáticamente un paquete instalación personalizado. Es posible: 

	- Continuar instalación. Los diferentes ordenadores incluidos en el cluster deben ser accesibles. 

	- Utilizar un paquete instalación. Este paquete debe ejecutarse en cada ordenador incluido en el cluster. 







**Observación**: Se puede instalar un solo coordinador en un ordenador determinado. Si dos clústeres están instalados en la misma red, sus respectivos coordinadores deben encontrarse en dos ordenadores diferentes.

## Configuración de los DNS para acceder al clúster
<a name="configuracion_los_dns_para_acceder_cluster_ELTTEXTE000290"></a>
Para dar acceso al clúster, se debe agregar un Record en su servidor DNS para cada nodo. Esta operación se realiza automáticamente al instalar el cluster y los nodos. Cuando no es posible la modificación automática del DNS (servidor DNS no soportado, problema de derechos, etc.), la Record debe ser añadida manualmente.

**Observación**: Para garantizar un modo de funcionamiento correcto del sistema DNS, es imprescindible que las direcciones IP de los nodos no cambien.. Si los servidores obtienen sus IP Address es a través de un DHCP, debe ser configurado para que se reserve un Address fijo para cada uno de los nodos del cluster.

**Características de los registros DNS a crear**:

- Record de tipo (A) si los nodos están en IPv4 y Record de tipo (AAAA) si están en IPv6.

- La dirección IP del Record es la del nodo.

- El nombre de la Record es el nombre del DNS del cúmulo. Este nombre puede ser elegido libremente. No debe corresponder al nombre de un ordenador real.




Cuando una aplicación cliente desea conectarse al clúster, recupera los registros DNS correspondientes al nombre elegido para el clúster.: este nombre se especifica como servidor address en el [Variable de tipo Connection](../WDLang4/1514073.md), o en las funciones [HOpenConnection](../WDLang4/3044107.md) o [HDescribeConnection](../WDLang4/3044205.md). El ordenador cliente seleccionará el nodo utilizado para la conexión entre las diferentes direcciones IP.

El DNS Record se puede crear una vez instalado el cluster, utilizando el HFSQL control Center. 


### Configurar el DNS desde el HFSQL control Center
<a name="configurar_dns_desde_hfsql_control_center_ELTPARAGRAPHE000121"></a>

Para configurar el DNS desde el Centro control de HFSQL:

1. Iniciar el Centro de Control HFSQL

2. Conéctese al grupo especificando la Address del coordinador.

3. Si el nombre DNS del clúster aún no está configurado, se muestra un fila en el "Potential problems" tabla.

4. Haga clic en el Icon para la resolución de problemas (al final del fila).

5. Especifique el servidor DNS que desea configurar, el nombre de dominio que desea utilizar (en la mayoría de los casos, el nombre de dominio del equipo local) y los parámetros de autenticación si son necesarios para la conexión con el servidor DNS.

6. Valide la ventana.

7. En el cuadro de diálogo, introduzca el nombre DNS que utilizarán los equipos cliente para conectarse al clúster.

8. Valide el cuadro de diálogo.








## Instalar un nuevo nodo
<a name="instalar_nuevo_nodo_ELTTEXTE000320"></a>
Todos los nodos, excepto el primero, deben ser necesariamente servidores HFSQL nuevos.

También tiene la posibilidad de instalar un nuevo servidor como primer nodo del clúster.. En este caso, el cluster no contendrá ninguna base de datos inicial.

**Observaciones**:

- Tanto los servidores HFSQL de 32 bits como los de 64 bits pueden utilizarse como nodos del mismo clúster.

- Todos los servidores de un cluster deben utilizar el mismo puerto de escucha. Este puerto debe elegirse durante el proceso de instalación.





### Instalación de un nuevo servidor HFSQL en el equipo local como nodo de clúster
<a name="instalacion_nuevo_servidor_hfsql_equipo_local_como_nodo_cluster_ELTPARAGRAPHE000147"></a>

Para instalar un nuevo servidor HFSQL en el equipo local como nodo de clúster: 

1. Inicie el clúster HFSQL instalación asistente.

2. Seleccione "Añadir un nodo a un cluster existente".

3. Introduzca los parámetros para acceder al coordinador: nombre del ordenador y contraseña de gestión.

4. Modificar la composición del cluster. Se pueden añadir varios nodos en una sola operación.

5. Un resumen le permite comprobar la configuración del cluster antes de aplicar las modificaciones.

6. Haga clic en "Siguiente". Se crea automáticamente un paquete instalación personalizado. Es posible: 

	- Continuar instalación. Los diferentes ordenadores incluidos en el cluster deben ser accesibles. 

	- Utilizar un paquete instalación. Este paquete debe ejecutarse en cada ordenador incluido en el cluster. 











## Operación de desinstalación
<a name="operacion_desinstalacion_ELTTEXTE000350"></a>


### Desinstalación de un nodo o cluster
<a name="desinstalacion_nodo_cluster_ELTPARAGRAPHE000164"></a>

Para desinstalar un nodo de clúster o todo el clúster:

1. En el el equipo del coordinador, inicie el cluster HFSQL instalación asistente. 

2. Seleccione "Desinstalar un cluster o nodos".. 

3. Especifique la contraseña del coordinador. 

4. Compruebe los elementos que deben ser desinstalados. 

5. Validar. La operación de desinstalación se realiza. 




**Observación**: Esta operación de desinstalación también puede realizarse desde el HFSQL control Center. En este caso, tiene la posibilidad de excluir temporalmente un nodo del cluster sin tener que desinstalarlo..

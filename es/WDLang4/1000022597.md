
## mongoWriteConcern (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **mongoWriteConcern** permite definir el nivel de acuse de recibo solicitado para las operaciones de escritura en un mongod independiente, en un conjunto de réplicas o en un clúster fragmentado. Las características de estas opciones se pueden definir y cambiar mediante diferentes propiedades WLanguage. 

**Observaciones**: 

- Este tipo de Variable es utilizado por el [mongoCollection](../WDLang4/1000022411.md) Variable s. 

- Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyCollection.WriteConcern.Mode = mwcMajority
```

### Propiedades específicas de la descripción de las variables mongoWriteConcern
<a name="propiedades_especificas_descripcion_las_variables_mongowriteconcern_ELTPARAGRAPHE000046"></a>

Las siguientes propiedades pueden ser utilizadas para manipular las opciones para escribir en una colección Mongo:

| Nombre de la propiedad | Tipo utilizado | Efecto |
| --- | --- | --- |
| Etiqueta | Cadena de caracteres | Nombre de la preconfiguración del servidor. |
| modelar | Constante de tipo Integer | Número mínimo de nodos que deben haber reconocido la Process para considerar que la escritura se realiza. Están disponibles los siguientes valores: <br><br>- *mwcPor defecto*: Un solo nodo debe haber reconocido a la Process para considerar que la escritura se realiza. <br><br>- *mwcMajority*: La mayoría de los nodos deben haber reconocido la Process para considerar que la escritura se realiza. <br><br>- *mwcTag*: El número de nodos es el definido en una preconfiguración definida en el servidor.. En este caso, se debe especificar la propiedad **.Tag**. <br><br>- *mwcUnacknowledged*: Ningún nodo debe haber reconocido a la Process para considerar que la escritura se realiza. Atención, no se devuelve ningún error en este caso.. <br><br>- Número mínimo de nodos que deben haber reconocido la Process (entero estrictamente positivo).<br><br><br> |
| Registro | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> para escribir una traza de la operación de escritura en el registro.<br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario.<br><br><br> |
| Timeout | Entero o Duración | <br><br>- Tiempo máximo de espera para realizar una escritura. Las operaciones de escritura se consideran fallidas después de este tiempo de espera.. <br><br>- 0 por no tener tiempo de espera de escritura.<br><br><br><br><br>Esta propiedad puede corresponder a: <br><br>- un número entero correspondiente al número de segundos,<br><br>- una variable de tipo Duration,<br><br>- la duración en un formato legible (por ejemplo, 1 s o 10 ms). <br><br><br> |

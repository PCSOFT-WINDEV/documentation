


## &lt;Variable Connection&gt;.DeleteServerTrigger (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina el trigger de un servidor. Este disparador de servidor ya no se puede utilizar.

**Observación**: Las funciones para manejar los disparadores del servidor son funciones avanzadas. Los triggers del servidor se pueden crear directamente en el editor de análisis. Para obtener más información, consulte [Disparadores de servidor](../WDLang4/3044369.md).




<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

### Eliminar un trigger servidor en archivos HFSQL físicos asociados a una conexión

`<Result> = <Connection>.DeleteServerTrigger(<Trigger> [, <Physical HFSQL Client/Server files>])`
---

**`<Result>: (Booleano)`**



- <u><u><u>True</u></u></u> si se realizó la operación,

- <u><u><u>False</u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Connection>: (Variable de tipo Connection)`**

Nombre de la variable de tipo [Conexión](../WDLang4/1514073.md) que describe la conexión a utilizar.

**`<Trigger>: (Cadena de caracteres)`**

Nombre del disparador a borrar. Este nombre se definió con [&lt;Variable Connection&gt;.DescribeServerTrigger](../WDLang4/1000022653.md).

**`<Physical HFSQL Client/Server files>: (Cadena de caracteres opcional)`**

Nombre físico de uno o más archivos de datos de HFSQL Client/Server associated con la conexión (por ejemplo "CUSTOMER.FIC"). 
Para especificar varios nombres, los nombres de los archivos de datos deben estar separados por una coma (","). 
Si este parámetro no se especifica o si corresponde a una cadena vacía ("""), el trigger se habilitará en todos los ficheros de datos de la base de datos..

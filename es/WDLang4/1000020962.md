
## Record (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **Record** permite manipular (modificar, guardar, etc.) los valores de un registro. Este Record puede ser un Record que se encuentra en un archivo de datos, una consulta o una vista. La variable asociada al tipo **Record** se estructurará como un registro de la base de datos. 

Ejemplos de uso: Esto le permite almacenar en diferentes variables el contenido de varios registros para compararlos o para comparar algunos artículos.. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// CUSTOMER is a data file described in the analysis
CustomerRec is Record of CUSTOMER

HReadFirst(CUSTOMER, CUSTNUM)
IF NOT HOut() THEN
	CustomerRec = CUSTOMER
	// Display the last and first names of the customer 
	Info(CustomerRec.LASTNAME, CustomerRec.FIRSTNAME)
END
```


<a name="Example2"></a>

```wl
// CUSTOMER is a data file described in the analysis
CustomerRec is Record of CUSTOMER

CustomerRec.LastName = "MOORE"
CustomerRec.FirstName = "Paul"
CustomerRec.City = "PARIS"

// Assigns the content of the CustomerRec variable in the buffer of Customer data file
CUSTOMER = CustomerRec  

HAdd(CUSTOMER)

// You have the ability to do: HAdd(CUSTOMER, CustomerRec)
```

## Observaciones
<a name="NOTE0_1"></a>


### Declarando un Record Variable
<a name="declarando_record_variable_ELTPARAGRAPHE000042"></a>

La siguiente sintaxis se utiliza para declarar una variable de tipo **Record**: 


```txt
<Variable name> is Record [of <Name of data file in the analysis>]
```


- Si se especifica &lt;Nombre del archivo de datos en el análisis&gt;, el Record se asocia a un description de uno de los archivos de datos en el análisis. En este caso, tienes la capacidad de manipular los diferentes artículos de Record. 

- Si no se especifica &lt;Nombre del fichero de datos en el análisis&gt;, la estructura del Record se construirá dinámicamente según el buffer del fichero que se le asigne. 



<a name="NOTE0_2"></a>


### Acceso a los artículos
<a name="acceso_los_articulos_ELTPARAGRAPHE000054"></a>

Para acceder a los elementos de un **Record** Variable, utilice la siguiente sintaxis: 

- Para asignar un elemento de la Variable: 
	
	```txt
	<Variable name>.<Item name> = <Value to assign>
	```


- Para recuperar el valor de los elementos: 
	
	```txt
	<Value> = <Variable name>.<Item name>
	```





**Observación**: Cuando se utiliza una variable **Record** asociada a un archivo de datos del análisis, el editor de código propone automáticamente los diferentes elementos de Record:

<a name="NOTE0_3"></a>


### Lectura y asignación de un Record Variable
<a name="lectura_asignacion_record_variable_ELTPARAGRAPHE000072"></a>

Las variables de tipo **Record** se pueden leer o asignar. La sintaxis es la siguiente: 

- Para almacenar el buffer de un archivo de datos en un **Record** Variable LE: 
	
	```txt
	<Name of Record variable> = <Name of data file>
	```


- Para asignar el buffer de un archivo de datos de un **Record** Variable LE: 
	
	```txt
	<Name of data file> = <Name of Record variable>
	```





Observación: La asignación de una variable de tipo **Record** a un archivo de datos modifica el [NewRecord](../Proprietes/1000017090.md) Property. 
<a name="NOTE0_4"></a>


### Casos especiales
<a name="casos_especiales_ELTPARAGRAPHE000091"></a>

- [HRecordToString](../WDLang4/1000025983.md) utilizado en un **Record** Variable devuelve una cadena de caracteres que contiene el valor de cada elemento separado por un carácter TAB.

- No se produce ningún error de compilación si un **Record** Variable con tipo en un archivo de datos se asigna con el búfer de otro archivo de datos. Se realiza un copy homónimo de los buffers: sólo se asignarán los elementos con el mismo nombre de cada búfer. Los demás elementos no se asignarán.

- Para este tipo de Variable ([serialize](../WDLang1/3013065.md)) sólo está disponible la serialización JSON. No es posible la deserialización.

### Funciones que manipular el tipo Record
<a name="funciones_que_manipular_tipo_record_ELTPARAGRAPHE000105"></a>Las siguientes funciones utilizan el tipo **Record**: 



|   |   |
| --- | --- |
| [HAdd](../WDLang4/3044147.md) | Agrega: <br><br>- el registro que se encuentra en la memoria al archivo de datos (consulta o vista). <br><br>- el registro que se encuentra en una variable de tipo [Record](../WDLang4/1000020962.md) al archivo de datos (consulta o vista).<br><br><br> |
| [HCopyRecord](../WDLang4/3044261.md) | Copia: <br><br>- el contenido del registro en curso (cargado en memoria) en el registro en curso de un archivo de datos.<br><br>- el contenido de una variable de tipo [Record](../WDLang4/1000020962.md) en el registro en curso de un archivo de datos<br><br><br> |
| [HExtractMemo](../WDLang4/3044072.md) | Extrae el contenido de un campo de tipo memo binario de un archivo de datos, consulta o vista HFSQL a un archivo físico (en el disco). |
| [HInfoMemo](../WDLang4/3044008.md) | Devuelve las características de un memo binario y de texto. |
| [HLinkMemo](../WDLang4/3044173.md) | Permite asociar un archivo a un campo de tipo memo binario o cancelar la asociación existente entre un archivo y un campo binario. |
| [HListItem](../WDLang4/3044230.md) | Devuelve la lista de artículos:<br><br>- encontrado en un archivo de datos ( una consulta o una vista) conocido por el motor HFSQL. Los archivos de datos definidos por las funciones [HDeclare](../WDLang4/3044203.md), [HDeclareExternal](../WDLang4/3044204.md) y [HDescribeFile](../WDLang4/3044206.md) se tienen en cuenta.<br><br>- encontrada en un Record Variable. <br><br><br> |
| [HRecordToString](../WDLang4/1000025983.md) | Convierte el contenido de la Record actual en cadena (en un archivo de datos, vista HFSQL, consulta, etc.). |
| [HReset](../WDLang4/3044118.md) | Inicializa: <br><br>- una o todas las variables de los elementos de un archivo de datos con sus valores predeterminados. <br><br>- uno o todos los elementos de una variable de tipo [Record](../WDLang4/1000020962.md) con sus valores predeterminados. <br><br><br> |





Observación: También puede utilizar [VariableReset](../WDLang1/3013069.md) para restablecer el contenido de una variable de tipo **Record**.

### Propiedades específicas de las variables Record
<a name="propiedades_especificas_las_variables_record_ELTPARAGRAPHE000120"></a>

Las siguientes propiedades pueden utilizarse para averiguar las características de un **Record** Variable:

| **Nombre de la propiedad** | **Tipo utilizado** | **Efecto** |
| --- | --- | --- |
| AssociatedFile | Cadena de caracteres | Nombre del archivo de datos HFSQL en el que se basa el Variable. <br><br>**Esta propiedad es de solo lectura**. |
| Contenido | Cadena de caracteres | Contenido de la Record actual (idéntico al resultado devuelto por [HRecordToString](../WDLang4/1000025983.md)). <br><br>**Esta propiedad es de solo lectura**. |

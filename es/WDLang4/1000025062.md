


## &lt;Fuente&gt;.SavePosition (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Almacena el contexto actual de un archivo de datos: registro actual, filtro, punteros. Esto le permite suspender temporalmente la actual Process para que el archivo de datos pueda ser utilizado en otro lugar (para realizar comprobaciones, por ejemplo). El contexto guardado puede ser restaurado por [&lt;Fuente&gt;.RestaurarPosición](../WDLang4/1000025061.md). Entonces, el principal Process puede continuar.

La función **&lt;Fuente&gt;.SavePosition** permite guardar:

- el número de la Record utilizada,

- los diferentes punteros del índice (la posición en el índice se guarda para cada tecla),

- el filtro actual. Se deberá utilizar la constante *hRPFilter* en la función [&lt;Fuente&gt;.RestaurarPosición](../WDLang4/1000025061.md) para restaurar el filtro guardado.  




Esta función se puede utilizar con archivos de datos, vistas HFSQL o consultas.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Find a record
Customer.ReadSeekFirst(Name, Name)
WHILE Customer.Found() = True
	// Saves the current context
	nPos = Customer.SavePosition(Name)
	// Check a customer with another name
	Customer.ReadSeek(Name, Name2)
	IF Customer.Found() = True THEN Counter = Counter + 1
	// Restores the context
	HRestorePosition(nPos)
	Customer.ReadNext(Name)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.SavePosition([<Item>] [, <Options>])`
---

**`<Result>: (Entero)`**



- Número de la posición que se ha grabado,

- -1 si se produce un error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos, vista o consulta utilizada. Si no se especifica este nombre, la función **&lt;Fuente&gt;.SavePosition** utilizará el último archivo de datos utilizado por la última función HFSQL (que empieza por "H").

**`<Item>: (Cadena de caracteres opcional)`**

Nombre del campo utilizado. Si no se especifica este parámetro, **&lt;Fuente&gt;.SavePosition** guarda las posiciones de todos los elementos en bucle.
![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) El parámetro &lt;Item&gt; será ignorado: la posición se guardará para todos los elementos que se recorran en bucle.

**`<Options>: (Constante opcional de tipo Integer)`**

Permite configurar la copia de seguridad:



|   |   |   |
| --- | --- | --- |
| *hSaveItems* | Guarda los valores de los elementos en la memoria utilizando la función **&lt;Fuente&gt;.SavePosition**. Estos valores se restaurarán la próxima vez que se llame a [&lt;Fuente&gt;.RestaurarPosición](../WDLang4/1000025061.md).<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta constante sólo está disponible para los archivos de datos HFSQL. |   |

## Observaciones


### Ejemplo de gestión de los puestos en un archivo de datos
<a name="ejemplo_gestion_los_puestos_archivo_datos_ELTPARAGRAPHE000281"></a>

Las funciones **&lt;Fuente&gt;.SavePosition** y [&lt;Fuente&gt;.RestaurarPosición](../WDLang4/1000025061.md) pueden utilizarse en los siguientes casos:

- formulario utilizado para mostrar una Record y para recorrer el archivo de datos según una clave. Se llama a otra ventana para mostrar los datos en forma de tabla. Las posiciones le permiten volver fácilmente a la Record deseada en la iteración.

- ver un archivo de datos en un control Tabla. Cuando se selecciona, la Record se muestra como un formulario, con la posibilidad de pasar en bucle por los formularios. La gestión de la posición le permite mostrar el control Tabla de nuevo mientras se posiciona correctamente en el el archivo de datos.



<a name="NOTE0_2"></a>


### Reiniciación de posiciones
<a name="reiniciacion_posiciones_ELTPARAGRAPHE000301"></a>

Atención: todas las funciones que abren o cierran un archivo de datos reinicializan los números de contexto ([&lt;Fuente&gt;.Creation](../WDLang4/1000024207.md), [&lt;Fuente&gt;.Close](../WDLang4/1000024958.md), etc.).
<a name="NOTE0_3"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000321"></a>

- El número de llamadas a la función **&lt;Fuente&gt;.SavePosition** no está limitado.

- Se pueden anidar las funciones **&lt;Fuente&gt;.SavePosition** y [&lt;Fuente&gt;.RestaurarPosición](../WDLang4/1000025061.md) para un mismo archivo de datos.

- Si se elimina un registro al que se apunta al llamar a la función **&lt;Fuente&gt;.SavePosition**, ya no se podrá llamar a la función [&lt;Fuente&gt;.RestaurarPosición](../WDLang4/1000025061.md).

### Reposicionamiento por OLE DB y conectores nativos de MySQL, DB2 e Informix
<a name="reposicionamiento_por_ole_conectores_nativos_mysql_db2_informix_ELTPARAGRAPHE000351"></a>

Si la base de datos o el estado de la iteración (p. ej., reinicializado) no permite restablecer la posición directamente en Record , la posición se establece utilizando la memoria caché de Record s read.

Las siguientes funciones reinicializan la iteración:

- [&lt;Fuente&gt;.ReadFirst](../WDLang4/1000025022.md)/[&lt;Fuente&gt;.First](../WDLang4/1000025048.md).

- [&lt;Fuente&gt;.ReadLast](../WDLang4/1000025020.md)/[&lt;Fuente&gt;.Last](../WDLang4/1000024217.md).

- [&lt;Fuente&gt;.SetMemo](../WDLang4/1000024987.md).




Este modo de funcionamiento es el mismo en Java.


## HSetREP( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Activa o desactiva la gestión del archivo .REP. Para obtener más información, consulte [Archivo .REP](../WDLang4/3044186.md).



**Observación**: La gestión del archivo .REP está habilitada por Default.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Disables the management of .REP file
HSetREP(False)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HSetREP(<Management> [, <Path of .REP> [, <Name of .REP file>]])`
---

**`<Result>: (booleano)`**

Corresponde al anterior modo de gestión del archivo .REP (antes de llamar a la función **HSetREP**): 

- <u><u><u><u>True</u></u></u></u> si la gestión del .REP estaba activa,

- <u><u><u><u>False</u></u></u></u> si la gestión del .REP no estaba activa.




**`<Management>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si la gestión del .REP está activada,

- <u><u><u><u>False STRIMEULB</u></u></u></u> si esta gestión debe ser desactivada.




**`<Path of .REP>: (Cadena de caracteres opcional)`**

Si se especifica este parámetro, **HSetREP** utiliza el archivo .REP situado en el directorio especificado. 
Si no se especifica este parámetro, el Default ruta corresponde a: 


- el ruta anterior del archivo .REP especificado para el análisis actual.

- si no se ha especificado previamente ningún ruta: 
	1. la ruta del análisis (.WDD), 
	2. la ruta de la biblioteca de aplicaciones (.WDL) 
	3. por Default, el ruta de la aplicación (.EXE). Este es el caso más común.




**`<Name of .REP file>: (Cadena de caracteres opcional)`**

Si se especifica este parámetro, la función **HSetREP** utiliza este nombre para el archivo .REP. 
Si no se especifica este parámetro, el nombre del análisis se utiliza como nombre para el archivo.REP. 
En todos los casos, la extensión del archivo.REP es ".REP"..

<a name="NOTE0_1"></a>

## Observaciones




### Habilitar el.REP
<a name="habilitar_elrep_ELTPARAGRAPHE000084"></a>

- **Si el parámetro &lt;Gestión&gt; es set a <u><u><u><u>True</u></u></u></u>**, el .REP se actualizará cada vez que se abra o cree un archivo de datos en el análisis actual. Entonces, la herramienta para la modificación de archivos de datos de automatic podrá identificar los archivos que deben ser modificados.

- **Si el parámetro &lt;Gestión&gt; es set a <u><u><u><u>False</u></u></u></u>**, el archivo .REP no se actualiza. En este caso, la ubicación de estos archivos debe ser especificada manualmente a la herramienta de modificación de archivos de datos automatic. Si se especifica el nombre de.REP, este nombre se utilizará para la siguiente activación (si no se especifica un nuevo nombre mientras tanto).

- El archivo.REP se administra por contexto. Un componente que utilice un contexto HFSQL independiente no se verá afectado por la llamada a **HSetREP**.




**Observación**: Cuando el .REP no es el de Default, su ubicación se almacena en el archivo **HFRep.ini**. Este archivo se encuentra en el directorio "Application Data\\PC SOFT" de los datos compartidos de la aplicación (para la compatibilidad con versiones anteriores, este archivo también se puede encontrar en el directorio de Windows).


## Constantes utilizadas por la replicación entre servidores HFSQL
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


| Constante | description | Utilizado por la función |
| --- | --- | --- |
| **hmcrHigherPriority** | Método para resolver conflictos: Durante un conflicto, sólo se tiene en cuenta el valor de prioridad definido para el servidor.. Cuanto menor sea el valor de prioridad definido para el servidor, mayor será la prioridad concedida al servidor. | Variable de tipo [hRSConfig](../WDLang4/1000020484.md) |
| **hmcrMásRecientes** | Método para resolver conflictos: Durante un conflicto, el elemento más reciente tiene prioridad.<br>Atención: Este método requiere una sincronización del reloj en los diferentes servidores. | Variable de tipo [hRSConfig](../WDLang4/1000020484.md) |
| **hmcrProcedureAllItems** | Método para resolver conflictos: Cuando se produce un conflicto, se inicia un Procedure almacenado (definido con el **Procedure ForModificationConflictResolution** Property). En este procedimiento, se podrá acceder a todos los artículos que se encuentran en la Record. | Variable de tipo [hRSConfig](../WDLang4/1000020484.md) |
| **hmcrProcedureWithModifiedItems** | Método para resolver conflictos: Cuando se produce un conflicto, se inicia un Procedure almacenado (definido con el **Procedure ForModificationConflictResolution** Property). En este Procedure, sólo se podrá acceder a los elementos modificados. | Variable de tipo [hRSConfig](../WDLang4/1000020484.md) |
| **hrsDefault** | Borra la replicación sólo si el servidor de suscriptores es accesible. | [HRSDeleteConfig](../WDLang4/1000020493.md) |
| **hrsIgnoreSubscriber** | Forza la eliminación incluso si el otro servidor no puede ser contactado. | [HRSDeleteConfig](../WDLang4/1000020493.md) |
| **hrsInactive** | Papel del servidor en la replicación: Inactive en la replicación entre servidores HFSQL. | [HRSInfo](../WDLang4/1000020574.md), [HRSInit](../WDLang4/1000020541.md) |
| **hrsMaster** | Papel del servidor en la replicación: Ordenador maestro en la replicación entre servidores HFSQL. | [HRSInfo](../WDLang4/1000020574.md), [HRSInit](../WDLang4/1000020541.md), [HRSListConfig](../WDLang4/1000020497.md) |
| **hrsSubscriber** | <br><br>- Papel del servidor en la replicación: Ordenador de abonado en la replicación entre servidores HFSQL.<br><br>- Forza la eliminación de una réplica huérfana en el lado del suscriptor.<br><br><br> | <br><br>- [HRSInfo](../WDLang4/1000020574.md), [HRSInit](../WDLang4/1000020541.md), [HRSListConfig](../WDLang4/1000020497.md)<br><br>- [HRSDeleteConfig](../WDLang4/1000020493.md)<br><br><br> |

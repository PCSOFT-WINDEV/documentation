
## H. ErrIgnorar (Variable)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Especifica el modo para procesar los errores:

- los errores son procesados por WINDEV,

- el programa procesa los errores.




**Atención**: El H.ErrIgnore Variable no tiene ningún efecto para los errores de programación (llamados "errores mayores"). Estos errores no deben ser ignorados porque la ejecución del programa se quedaría en nada.. Estos errores se presentan en [errores HFSQL](../WDLang4/3044177.md).


<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Valor de H.ErrIgnore
<a name="valor_herrignore_ELTPARAGRAPHE000027"></a>

**Si la variable H.ErrIgnore se establece en <u><u><u><u>True</u></u></u></u>**:

- los errores (que pueden ser ignorados) deben ser procesados en el programa, no detienen la ejecución del programa. Si se produce un error, no se ejecuta la función solicitada.

- el global *H.Error* Variable devuelve el número de error al programa para que el error sea procesado.




**Si la variable H.ErrIgnore se establece en <u><u><u><u>False</u></u></u></u> (valor predeterminado)**:

- Todos los errores en tiempo de ejecución muestran automáticamente el número de error, el mensaje de error associated y el valor de los parámetros no válidos.

- La ejecución del programa actual se detiene, pulsando una tecla del teclado se sale del programa.



<a name="NOTE0_2"></a>




### Ejemplo 
<a name="ejemplo_ELTPARAGRAPHE000051"></a>

El siguiente ejemplo se utiliza para comprobar la existencia de un archivo de datos. Si el archivo de datos no existe, se crea; si existe, el programa sigue funcionando.


```wl
// In order for the execution not to be stopped if an error occurs
H.ErrIgnore = True
HOpen(CUSTOMER)
H.ErrIgnore = False
IF H.Error = 3 THEN HCreation(CUSTOMER)
...
```

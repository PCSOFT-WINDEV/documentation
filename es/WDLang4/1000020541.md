


## HRSInit (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Configura un servidor HFSQL para que este servidor sea un servidor maestro o un servidor suscriptor para una replicación entre servidores HFSQL.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HRSInit(Cnt, hrsSubscriber, 1, "pwd", 4996, 5)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HRSInit(<Connection> , <Server role> , <Server identifier> , <Password> , <Port> , <Priority>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la configuración,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión al servidor HFSQL para configurar. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).




**`<Server role>: (Constante de tipo Integer)`**

Papel del servidor en la replicación: 


|   |   |
| --- | --- |
| *hrsInactive* | Inactive en la replicación entre servidores HFSQL. |
| *hrsMaster* | Ordenador maestro en la replicación entre servidores HFSQL. |
| *hrsMaster* + *hrsSubscriber* | Ordenadores maestros y de abonado en la replicación entre servidores HFSQL (para una replicación bidireccional). |
| *hrsSubscriber* | Ordenador de abonado en la replicación entre servidores HFSQL. |



**`<Server identifier>: (Integro)`**

Número utilizado para identificar el servidor HFSQL actual entre todos los servidores implicados en una replicación.. Este identificador debe ser único. Este identificador debe incluirse entre 1 y 32000. 

**`<Password>: (Cadena de caracteres)`**

Contraseña de replicación, utilizada para autenticar los servidores entre ellos.. Esta contraseña debe utilizarse en el otro servidor en la llamada a [HRSAddConfig](../WDLang4/1000020492.md) (es decir, en el **Password** Property del [hRSConfig](../WDLang4/1000020484.md) Variable).

Para utilizar la replicación bidireccional, la contraseña debe ser la misma en los dos servidores.

**`<Port>: (Integro)`**

Número de puerto TCP/IP utilizado para la replicación entre servidores HFSQL. 

**`<Priority>: (Integro)`**

Número de prioridad del servidor. Durante un conflicto de modificación, este número se utiliza si el método para resolver el conflicto corresponde a una resolución por prioridad.. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
**HRSInit** se puede utilizar varias veces en el mismo servidor, para cambiar el papel de un servidor en una replicación, por ejemplo.

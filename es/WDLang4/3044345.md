
## Archivo HFConf.Ini
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000493"></a>
El archivo HFConf.INI se encuentra en la raíz del servidor. Este archivo se utiliza para definir varias características del servidor HFSQL Client/Server.

Este archivo también se utiliza para especificar las características de los archivos de registro creados en el servidor. Los archivos de registro son archivos de texto cuyos elementos están separados por caracteres TAB. La primera línea del archivo contiene la Definition de columnas. Los archivos de registro son creados por HFLogger.exe (hflogger en Linux).



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Contenido del archivo HFConf.Ini
<a name="contenido_del_archivo_hfconfini_ELTTEXTE000523"></a>
El archivo HFConf.INI contiene varias secciones:

- La sección [INIT](#NOTE2_2) que se utiliza para realizar diferentes inicializaciones.

- La sección [ADMINISTRADOR](#NOTE2_3), utilizada sólo en Linux.

- La sección [AUTOANÁLISIS](#NOTE2_4) que se utiliza para configurar la optimización de los índices en la tarea Background.

- La sección [TELEMETRÍA](#NOTE2_5) que se utiliza para configurar la transmisión de datos a PC SOFT con el fin de mejorar el producto (telemetría).

- La sección [ACTIVEDIRECTORY](#NOTE2_6) que se utiliza para configurar la autenticación a través del Directorio Active.

### Sección MANAGER
<a name="seccion_manager_ELTPARAGRAPHE000388"></a>

La sección MANAGER se usa sólo en Linux. Esta sección puede contener las siguientes palabras clave:

- **ReiniciarSobreFallido**: Reinicia el servidor cuando se produce un error fatal.. Esta palabra clave puede ser set a 0 (sin activación) o 1 (activación). Por defecto, esta palabra clave es set a 1 (activación).

- **Período de verificación**: Período (en segundos) tras el cual MantaManager comprueba si el servidor se ha detenido correctamente. 600 por Default (lo que significa cada 10 minutos).

### Sección AUTOANÁLISIS
<a name="seccion_autoanalisis_ELTPARAGRAPHE000400"></a>

La sección de AUTOANÁLISIS que se utiliza para configurar la optimización de los índices en la tarea de Background. Esta sección puede contener la siguiente palabra clave:

**Habilitado**: Permite la optimización de los índices en la tarea de Background. Esta palabra clave puede ser set a 0 (sin activación) o 1 (activación). Por defecto, esta palabra clave es set a 1 (activación).

Si la optimización de los índices en la tarea de Background está activada, los índices de los archivos de datos se optimizan automáticamente si se cumplen las siguientes condiciones: 

- las estadísticas no han sido calculadas para el archivo de datos desde al menos 7 días. 

- el archivo de datos fue creado desde al menos 1 día. 

- el número de operaciones de escritura desde el último cálculo estadístico es superior a 10000 (más el 10% del número de registros del archivo). 




Si la optimización de los índices está activada, esta operación no es una operación de bloqueo..

### Sección TELEMETRÍA
<a name="seccion_telemetria_ELTPARAGRAPHE000419"></a>

La sección TELEMETRIA se utiliza para configurar la implementación de la transmisión de datos para el servidor HFSQL (telemetría).. Puede ayudarnos a mejorar el producto instalado permitiéndonos recopilar información sobre el uso del producto.. Esta recogida opcional y anónima permite a PC SOFT mejorar las características del producto.

Esta sección contiene la siguiente palabra clave:

**Active**: Permite la transmisión de datos sobre el uso del servidor HFSQL. El valor de esta palabra clave puede ser: 

- 1 si la transmisión de datos (telemetría) fue habilitada al instalar el servidor HFSQL,

- 0 si la transmisión de datos (telemetría) fue rechazada al instalar el servidor HFSQL.

- otro valor si no se hizo ninguna elección durante la instalación en cuanto al uso de la telemetría. La pregunta se volverá a hacer durante la próxima instalación del servidor HFSQL.




Observación: Si la sección "TELEMETRÍA" no existe en el archivo HFConf.ini, la transmisión de datos sobre el uso del producto no está habilitada para el servidor HFSQL.

### Sección ACTIVEDIRECTORIO
<a name="seccion_activedirectorio_ELTPARAGRAPHE000439"></a>

La sección Active DIRECTORY se utiliza para configurar la autenticación a través del directorio Active. 

Esta sección puede contener las siguientes palabras clave:

- **Habilitado**: Permite la autenticación a través de Active Directory. Esta palabra clave puede ser set a 0 (sin activación) o 1 (activación). 

- **KerberosKeyFile** (sólo se usa en Linux): Camino a "keytab"..


## HErrorLock( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Se utiliza para verificar si se ha producido un error de bloqueo.. Esta función debe utilizarse para la gestión personalizada de bloqueos y conflictos de modificación.

Observación: Por defecto, se realiza una [gestión automática de errores de bloqueo y de conflictos de modificación](../WDLang4/3044188.md) (excepto en el código de procedimientos almacenados). La gestión automática de errores se puede personalizar o desactivar en cualquier momento con la función [HOnError](../WDLang4/3044017.md).










<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Read the first record
HReadFirst(Customer, Name)
// Lock error?
IF HErrorLock() = True THEN
	Error("A lock error was detected")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HErrorLock()`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se produjo un error de bloqueo,

- <u><u><u><u>False</u></u></u></u> en caso contrario (sin errores).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- No hay gestión de cerraduras automatic disponible para los elementos de Memo o Memo Binario.

- Una vez que se ha leído un Record cerrado, no se realiza ninguna operación de lectura en el archivo de datos:

	- la función **HErrorLock** devuelve <u><u><u><u>True</u></u></u></u>.

	- [HOut](../WDLang4/3044094.md) devuelve <u><u><u><u>True</u></u></u></u>.
			Por lo tanto, si la operación de lectura se realiza mientras se realiza un bucle en un archivo de datos HFSQL, la iteración se interrumpe.
			La variable [H.NbRetry](../WDLang4/3044075.md) indica el número de intentos de bloqueo de el archivo de datos o Record antes de devolver control al programa (sólo en modo multiusuario). 




- Esta función sustituye a **HIsLocked**, que se mantiene por compatibilidad con WINDEV 5.5.




## &lt;Variable Connection&gt;.SimulateNetwork (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Simula el funcionamiento de la HFSQL Client/Server: 

- en una red ADSL, 3G, 4G y 5G o a través de la nube. 

- en una red lenta con ajustes personalizados.




**Observación**: Esta función puede utilizarse para probar una aplicación (localmente o a través de una LAN) que utilice HFSQL en una red ADSL, 3G, 4G o 5G, en la nube o con una configuración personalizada.. Esta función ralentiza todas las comunicaciones entre el cliente y el servidor HFSQL..

**Esta función se utiliza para depurar y optimizar una aplicación.**



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Describe the connection
MyConnection is Connection

// Simulate the 3G network
MyConnection.SimulateNetwork(h3G)
ProfilerStart()
Proc_Process() // Process whose performance in 3G must be analyzed
ProfilerEnd()
// Back to a standard operating mode
MyConnection.SimulateNetwork(hNone)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Simulación del funcionamiento de HFSQL Client/Server en una red previamente configurada

`<Result> = <Connection>.SimulateNetwork(<Type of network>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se inició la simulación,

- <u><u><u><u>False</u></u></u></u> si se produce un error.




**`<Connection>: (Variable de tipo Connection)`**

Nombre de la variable de tipo [Conexión](../WDLang4/1514073.md) que describe la conexión a utilizar.

**`<Type of network>: (Constante de tipo Integer)`**

Tipo de red a simular: 


|   |   |
| --- | --- |
| *h3G* | Tipo de red a simular: 3G<br>Características (estimación): Latencia: 130 ms, Ancho de banda: 1 Mbps |
| *h4G* | Tipo de red a simular: 4G<br>Características (estimación): Latencia: 40ms, Ancho de banda: 30 Mbps |
| *h5G* | Tipo de red a simular: 5G<br>Características (estimación): Latencia: 10ms, Ancho de banda: 100 Mbps |
| *hADSL* | Tipo de red a simular: ADSL<br>Características (estimación): Latencia: 40ms, Ancho de banda: 8 Mbps |
| *hCloud* | Tipo de red a simular: Nube.<br>Características (estimación): Latencia: 10ms, Ancho de banda: 400 Mbps<br>Estos valores se aplican cuando la aplicación se utiliza desde el mismo Continent que el centro de datos, con una conexión de fibra INTERNET. Si la aplicación se utiliza desde otro Continent, o con una conexión Wi-Fi, se recomienda utilizar la segunda sintaxis de la función especificando la latencia y la velocidad de la conexión. |
| *hNone* | Permite desactivar la simulación de una red específica. |




<a name="SYNTAX2"></a>

### Simulación del funcionamiento de HFSQL Client/Server en una red personalizada

`<Result> = <Connection>.SimulateNetwork(<Latency> , <Bandwidth>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se inició la simulación,

- <u><u><u><u>False</u></u></u></u> si se produce un error.




**`<Connection>: (Variable de tipo Connection)`**

Nombre de la variable de tipo [Conexión](../WDLang4/1514073.md) que describe la conexión a utilizar.

**`<Latency>: (Integro)`**

Latencia simulada expresada en milisegundos. Esta latencia se añade a la latencia de la red actual.

**`<Bandwidth>: (Real)`**

Ancho de banda expresado en megabits por segundo. Representa el tiempo necesario para transferir una cantidad determinada de datos (suponiendo que la red actual tenga un ancho de banda Infinite).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Esta función no debe utilizarse en una aplicación implementada. Esta función se utiliza para depurar y optimizar una aplicación.

- Esta función aumenta la latencia y reduce el rendimiento de las conexiones de red utilizadas para HFSQL Client/Server..

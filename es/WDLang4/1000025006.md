


## &lt;Fuente&gt;.LogRecreate (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite volver a crear un registro vacío. Esta función se utiliza para restablecer un registro a 0 después de una copia de seguridad o una réplica, por ejemplo.. Se pierde el contenido de los archivos de registro existentes.

**Atención**: esta función es una función avanzada.

**No se recomienda el uso de esta función.. Le recomendamos que utilice [WDLog](../WDJournal/3516012.md) para gestionar el registro.**

**Observación**: El registro Process no está disponible cuando se accede a una base de datos a través de .


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Customer.LogRecreate()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.LogRecreate([<Creation options>])`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos registrado para el cual el registro debe ser recreado. 

**`<Creation options>: (Constante o combinación de constantes opcional)`**

Configura el archivo de registro para volver a crear:


|   |   |
| --- | --- |
| *hLogIdentification* | Vuelve a crear el archivo "JournalIdentification.fic". |
| *hLogJNLFile*<br>(Valor predeterminado) | Sólo se recrea &lt;fichero de datos&gt;JNL.FIC. |
| *hLogOperation* | Vuelve a crear el archivo "JournalOperation.fic". |
| *hLogUser* | Recrea el registro de usuarios ("UserLog.fic").<br>![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) Este archivo sólo existe en HFSQL Client/Server. Este archivo contiene la información (login, ordenador,...) sobre los usuarios que realizan las acciones registradas. |


**Observación**: no le aconsejamos que vuelva a crear el archivo de usuario y el archivo de identificación solamente. De hecho, los vínculos entre los archivos podrían romperse y los archivos recreados podrían ser incoherentes..




## &lt;Variable Connection&gt;.NotifAddEmailRecipient (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Agrega destinatarios para las notificaciones enviadas por correo electrónico. Estas notificaciones se envían cuando el servidor HFSQL encuentra un problema. 

**Observación**: Los parámetros especificados por [&lt;Variable Connection&gt;.NotifConfigure](../WDLang4/1000022726.md) (servidor Email, etc.) se utilizan para enviar el Email. 

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Describe the connection
ctCSConnection is Connection

ctCSConnection is Connection
ctCSConnection.Provider = hAccessHFClientServer
ctCSConnection.User = "admin"
ctCSConnection.Password = ""
ctCSConnection.Server = "test:4900"
ctCSConnection.OpenConnection()

ctCSConnection.NotifConfigure(hNotifEmail, "FLOWER:25", "test.doc", "doc", "doc@pcsoft.com", False)
ctCSConnection.NotifAddEmailRecipient("test.app@gmail.com", hNotifWarning)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = <Connection>.NotifAddEmailRecipient(<Recipient> , <Minimum level>)`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se agregó el destinatario. 

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Connection>: (Variable de tipo Connection)`**

Nombre de la variable de tipo [Conexión](../WDLang4/1514073.md) que describe la conexión a utilizar.

**`<Recipient>: (Cadena de caracteres)`**

Correo electrónico Address del destinatario.

**`<Minimum level>: (Constante de tipo Integer)`**

El nivel mínimo de gravedad desde el que se envía una Notification: 


|   |   |
| --- | --- |
| *hNotifCritical* | Las notificaciones se enviarán desde el nivel crítico (para los problemas críticos y de error).<br><br>Ejemplo de error crítico: "Error en la preparación de la réplica del patrón". |
| *hNotifError* | Las notificaciones se enviarán sólo para los problemas de error.<br><br>Ejemplo de error: "El archivo estadístico de la actividad del servidor está dañado. Se reiniciará". |
| *hNotifInformation* | Las notificaciones se enviarán desde el nivel de información (para la información, advertencia, problemas críticos y de error).<br><br>Ejemplos de información: <br><br>- Durante la actualización del servidor de automatic: <br><br>	- "El servidor se detendrá para la actualización. Se añadirá otro Notification después de la actualización".<br><br>	- "El servidor fue actualizado a la versión XXX."<br><br><br><br><br>- Durante la búsqueda de las claves necesarias en una base de datos en explotación. <br><br><br> |
| *hNotifWarning* | Las notificaciones se enviarán desde el nivel de alerta (para los problemas de advertencia, críticos y de error).<br><br>Ejemplo de advertencia: El servidor XX y el servidor YY tienen horas diferentes. La resolución de conflictos durante la replicación puede ser incorrecta". |


El orden de gravedad es el siguiente: 

- Información.

- Advertencia.

- Error.

- Crítico.


## HOnServerCall( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Personaliza el mensaje mostrado y la administración de desconexión en el computador cliente.

**Recordatorio**: El envío de un mensaje al ordenador cliente lo realiza [HSendMessageToClient](../WDLang4/3044346.md). La desconexión de un ordenador cliente la realiza [HDisconnectClient](../WDLang4/3044339.md).






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Define the Client messages
// The ClientMessage procedure is called
// whenever HSendMessageToClient is used
HOnServerCall(hMessage, "ClientMessage")
```

```wl
// ClientMessage procedure
PROCEDURE ClientMessage(Type, MessageToDisplay, DurationMessageDisplay)
IF Type = hMessage THEN
	Open(WIN_MyWin, MessageToDisplay)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HOnServerCall(<Type of message> [, <WLanguage procedure> [, <Options>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación correctamente,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HError](../WDLang4/3044088.md) permite obtener más detalles sobre el problema.




**`<Type of message>: (Constante de tipo Integer)`**

Tipo de mensaje a configurar.


|   |   |
| --- | --- |
| *hDisconnection* | Configure el mensaje de desconexión. |
| *hMessage* | Configurar el mensaje de información |



**`<WLanguage procedure>: (Nombre de Procedure)`**

Procedure que debe ser llamado cuando se muestra un &lt;Tipo de mensaje&gt; mensaje.

Este procedimiento tiene el siguiente formato: 

```txt
PROCEDURE <Procedure name>(<Message type>, <Message>, <Display duration>)
```


Si no se especifica este parámetro, el mensaje se mostrará en la ventana de Default. 
Si este parámetro es igual a una cadena vacía (""), la gestión de automatic se desactivará. En caso de desconexión, la conexión será cerrada por el servidor. Se mostrará un error durante el próximo acceso a un archivo de datos.

**`<Options>: (Constante de tipo Integer)`**

Opción para el funcionamiento de la Procedure: 


|   |   |
| --- | --- |
| *hSecondaryThread* | La Procedure se ejecuta en una thread secundaria. |


Si no se especifica esta opción, el Procedure se ejecuta en el thread principal.

## Observaciones


### Personalización de la visualización de mensajes
<a name="personalizacion_visualizacion_mensajes_ELTPARAGRAPHE000129"></a>

Todas las acciones están permitidas en el WLanguage Procedure.

En el caso de un mensaje de desconexión, la desconexión se realizará después de la duración especificada en [HDisconnectClient](../WDLang4/3044339.md) (independientemente de las acciones realizadas en el Procedure).
<a name="NOTE0_2"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000141"></a>

- La función **HOnServerCall** debe utilizarse en el hilo principal.

- La función **HOnServerCall** es global a toda la aplicación, incluyendo los componentes ejecutados. Si se utiliza la función **HOnServerCall** en un componente, se activará para la aplicación host (se tendrá en cuenta la última llamada a la función **HOnServerCall**)

- La función **HOnServerCall** puede utilizarse en un servicio Windows.

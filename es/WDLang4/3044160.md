


## HUnlockFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Desbloquea los registros de un archivo de datos:

- bloqueado por [HLockFile](../WDLang4/3044153.md).

- bloqueados individualmente por una función de lectura bloqueante.




El archivo de datos también puede desbloquearse cerrándolo ([HClose](../WDLang4/3044073.md)).

Observación: **HUnlockFile** no puede utilizarse para desbloquear un archivo de datos bloqueado por otra sesión o programa. Le aconsejamos que desbloquee un archivo de datos lo antes posible para que las otras sesiones puedan acceder a este archivo de datos.
![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función se puede utilizar con archivos de datos HFSQL Client/Server.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Lock
HLockFile(CUSTOMER, HLockReadWrite)
// Modify the file
...
// Unlock
HUnlockFile(CUSTOMER)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HUnlockFile([<Data file>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se bloqueó el archivo de datos,

- <u><u><u><u>False</u></u></u></u> si se produce un problema (por ejemplo, el archivo de datos no está bloqueado, etc.). La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Data file>: (Cadena de caracteres opcional)`**

Nombre del archivo de datos utilizado. 
Si no se especifica este nombre, la función **HUnlockFile** utilizará el último archivo de datos utilizado por la última función HFSQL (que empieza por "H").  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
La función **HUnlockFile** no tiene ningún efecto si:

- el archivo de datos está en modo monopuesto (ver [HMode](../WDLang4/3044039.md)).

- el archivo de datos no está cerrado.

- el archivo de datos fue bloqueado por otro programa.




## ReplicInitialize( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicializa o reinicializa una replicación definida por el editor de replicación (ReplicEdit).. Esta inicialización se utiliza para definir los parámetros de la replicación (valor de los filtros). Los datos necesarios se transfieren a la base de datos local.

**Observación**: La base de datos de abonados debe crearse localmente con [HCreation](../WDLang4/3044255.md) antes de utilizar **ReplicInitialize**.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Initializes the replication 
// by passing the zip code to take into account
IF ReplicInfo("MyReplication", replicInitialized) = False THEN
	ReplicInitialize("MyReplication", 47012)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = ReplicInitialize(<Replication name> [, <Parameter 1> [... [, <Parameter N>]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la inicialización,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para más detalles sobre el problema, utilice [ErrorInfo](../WDLang1/3013008.md).




**`<Replication name>: (Cadena de caracteres)`**

Nombre de la réplica a inicializar. Este nombre corresponde a:

- el nombre del archivo ".wer", creado por ReplicEdit (para una replicación en línea). Este archivo debe encontrarse junto al ejecutable.

- el nombre de la publicación (para un servidor de replicación)




**`<Parameter 1>: (Tipo de parámetro, opcional)`**

Valor del primer parámetro para la replicación. Si este parámetro corresponde a una cadena vacía ("""), se cancelará la inicialización anterior de los parámetros.



**`<Parameter N>: (Tipo de parámetro, opcional)`**

Valor del parámetro Nth para la replicación. Si este parámetro corresponde a una cadena vacía ("""), se cancelará la inicialización anterior de los parámetros.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Modo de funcionamiento
<a name="modo_funcionamiento_ELTPARAGRAPHE000087"></a>

La inicialización de la réplica se utiliza para:

- Inicializar los parámetros de los filtros si es necesario.

- reinicializar los parámetros de los filtros a 0.

- reinicializar una replicación (eliminar y volver a crear los elementos de la replicación: .rpp, .rpa, .syn files, ...). 

- reiniciar la información relativa a los rangos de identificadores. 




Atención: tan pronto como se inicialice una réplica de suscriptor, ya no debe reemplazar/restaurar uno de los archivos maestros (porque contienen información relativa a los rangos de identificadores de las réplicas de suscriptor).. 

	<B><B><B><B>Advertencia</B></B></B></B>

	A partir de la versión <B><B><B><B>17</B></B></B></B>, los archivos de replicación creados con una aplicación o un sitio en la versión 17 (o posterior) no pueden ser gestionados por una aplicación o un sitio en la versión 16 (o anterior).
Las aplicaciones o los sitios en la versión 17 (o posterior) continúan a manipular los ficheros de replicación creados con una aplicación o un sitio en la versión 16 (o anterior). 

<a name="NOTE0_2"></a>


### Inicialización de filtros
<a name="inicializacion_filtros_ELTPARAGRAPHE000106"></a>

Los parámetros deben especificarse en el orden definido en el editor de réplicas (ReplicEdit).. En el editor de réplica, el orden de los parámetros se puede ver (y modificar si es necesario) a través de "Replicación .. Replication description", pestaña "Parámetros".

**Recordatorio**: Para optimizar el rendimiento de la replicación universal, le aconsejamos que defina filtros.




## &lt;Variable Connection&gt;.ActivateServerTrigger (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Vuelve a activar un activador del servidor previamente desactivado por [&lt;Variable Connection&gt;.DeactivateServerTrigger](../WDLang4/1000022654.md).

**Observación**: Las funciones para manejar los disparadores del servidor son funciones avanzadas. Los triggers del servidor se pueden crear directamente en el editor de análisis. Para obtener más información, consulte [Disparadores de servidor](../WDLang4/3044369.md).



<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis
<a name="SYNTAX2"></a>

### Activar un trigger servidor en archivos físicos asociados a una conexión

`<Result> = <Connection>.ActivateServerTrigger(<Trigger> [, <Physical HFSQL Client/Server files>])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Connection>: (Variable de tipo Connection)`**

Nombre de la [Conexión](../WDLang4/1514073.md) Variable que corresponde a la conexión que se va a utilizar. 

**`<Trigger>: (Cadena de caracteres)`**

Nombre del disparador para activar. Este nombre se definió con [&lt;Variable Connection&gt;.DescribeServerTrigger](../WDLang4/1000022653.md).

**`<Physical HFSQL Client/Server files>: (Cadena de caracteres opcional)`**

Nombre físico de uno o más archivos de datos de HFSQL Client/Server associated con la conexión (por ejemplo "CUSTOMER.FIC"). 
Para especificar varios nombres, los nombres de los archivos de datos deben estar separados por una coma (","). 
Si no se especifica este parámetro o si corresponde a una cadena vacía (""), el disparador se activará en todos los archivos de datos de la base de datos associated a la conexión.

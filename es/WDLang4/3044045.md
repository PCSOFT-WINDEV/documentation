


## HListFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve la lista de archivos de datos:

- encontrado en el análisis actual o en un análisis específico reconocido por el motor HFSQL. Se tienen en cuenta los ficheros de datos definidos por [HDeclare](../WDLang4/3044203.md), [HDeclareExternal](../WDLang4/3044204.md) y [HDescribeFile](../WDLang4/3044206.md).

- disponible en una conexión a través de un acceso nativo o a través de un controlador OLE DB.

- para un grupo definido en la editor de análisis.
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) No se soportan los grupos definidos en la editor de análisis.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003044045&name=hlistfile_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Retrieve the data files described in the analysis and through programming
FileList is string
FileList = HListFile(hLstDetail + hLstAll)
// Add the list of data files in COMBO_COMBO1
ListAdd(COMBO_COMBO1, FileList)
// Display in a trace
Trace(FileList)
```

## Sintaxis

### Enumeración de los archivos de datos de un análisis

`<Result> = HListFile([<Path of the WDD file> [, <Password of the WDD file>]] [, <Options>])`
---

**`<Result>: (Cadena de caracteres)`**

Lista de archivos de datos encontrados en el análisis actual o en el análisis especificado. Las descripciones de cada análisis están separadas por un carácter CR (Carriage Return). El resultado es más o menos detallado en función de la constante utilizada:


|   |   |
| --- | --- |
| *hLstDetail* no especificado | &lt;Nombre del archivo de datos 1&gt; + CR + &lt;Nombre del archivo de datos 2&gt; + CR + ... |
| Se especifica *hLstDetail* | &lt;Nombre del archivo de datos 1&gt; + TAB + &lt;Abreviatura&gt; + TAB + &lt;Título&gt; + TAB + &lt;Nombre físico&gt; + CR + ... |


Donde:


- &lt;Nombre del archivo de datos X&gt;: El nombre lógico del archivo de datos (usado en la programación).

- &lt;Abreviatura&gt;: Abreviatura de archivo de datos.

- &lt;Título&gt;: Pie de foto del archivo de datos.

- &lt;Nombre físico&gt;: Nombre físico del archivo de datos (en el disco).




**`<Path of the WDD file>: (Cadena de caracteres opcional)`**

ruta completo de análisis (archivo .WDD) donde se describe el archivo de datos. 
Si este parámetro no se especifica o si es igual a una cadena vacía (""), se busca el archivo de datos en el análisis actual.

**`<Password of the WDD file>: (Cadena de caracteres opcional)`**

Contraseña definida para el análisis. 
Si no se define ninguna contraseña, este parámetro debe ser igual a una cadena vacía ("").

**`<Options>: (Constante o combinación de constantes de tipo Integer (opcional))`**

Configura el tipo de lista a realizar:


|   |   |
| --- | --- |
| *hLstNormal*<br>(valor predeterminado) | Lista no ordenada, no detallada, sin los archivos de datos descritos por programación. |
| *hLstDetail* | Enumera la información detallada (igual que WINDEV 5.5). |
| *hLstAll* | Enumera todos los archivos de datos (incluidos los descritos por [HDeclare](../WDLang4/3044203.md), [HDeclareExternal](../WDLang4/3044204.md) y [HDescribeFile](../WDLang4/3044206.md)).<br>![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) Este parámetro no se tiene en cuenta. |
| *hLstSorted* | Ordena la lista de archivos de datos en orden ascendente.<br>![Hyper File 5.5](https://doc.pcsoft.fr/ext/images/us/HF55.png) Este parámetro no se tiene en cuenta. |

### Enumeración de los archivos de datos de un grupo

`<Result> = HListFile(<Group name> [, <Options>])`
---

**`<Result>: (Cadena de caracteres)`**

Lista de ficheros de datos pertenecientes al grupo. Las descripciones de cada archivo de datos están separadas por un carácter CR (Carriage Return).

**`<Group name>: (Cadena de caracteres)`**

Nombre del grupo del archivo de datos. Este grupo de archivos de datos fue definido en la editor de análisis.

**`<Options>: (Constante o combinación de constantes de tipo Integer (opcional))`**

Configura el tipo de archivos de datos a listar:


|   |   |
| --- | --- |
| *hLstStoredProcedure* | Lista de procedimientos almacenados. |
| *hLstQuery* | Enumera las consultas que se han ejecutado. |
| *hLstTable*<br>(valor predeterminado) | Lista de tablas de usuario.<br>![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) Los nombres de las tablas corresponden a los nombres físicos de los archivos de datos. |
| *hLstSystemTable* | Lista de tablas del sistema.<br>![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) Esta opción no está disponible. |
| *hLstView* | Lista de vistas.<br>![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) **Conector Nativo Oracle**: La constante *hLstView* permite listar las vistas y los sinónimos.<br>![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) Enumera las vistas SQL. |
| *hLstMaterializedView* | ![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) Lista de vistas materializadas. |

### Listado de los archivos de datos de una conexión a través de Conectores Nativos o OLE DB (2)

### Visualización del resultado en un List Box o control Combo Box
<a name="visualizacion_del_resultado_list_box_control_combo_box_ELTPARAGRAPHE000532"></a>

&lt;Resultado&gt; puede asignarse directamente a un control List Box o Combo Box mediante la función [ListAdd](../WDLang1/3049004.md). Cada fila contendrá el nombre completo de uno de los análisis encontrados.

### Función HListFile y el conector Native MySQL o MariaDB
<a name="funcion_hlistfile_conector_native_mysql_mariadb_ELTPARAGRAPHE000544"></a>

- Para el Conector Nativo MySQL, la función **HListFile** ha evolucionado según las versiones: 

	- En la versión 15, **HListFile** devuelve la lista de archivos de la base de datos actual. 

	- En la versión 16, para poder usar las nuevas funcionalidades de MySQL 5, si el Conector Nativo está conectado a un servidor MySQL 5 o posterior, la función **HListFile** devuelve la lista de archivos de todas las bases de datos accesibles. En este caso, el nombre del archivo tiene el siguiente formato: "&lt;Nombre de la base de datos&gt;.&lt;Nombre de archivo>. 

	- A partir de la versión 170069, **HListFile** devuelve la lista de archivos de la base de datos actual. [HListDatabase](../WDLang4/3044288.md) enumera las bases de datos accesibles y sus archivos. 




- Para el Conector Nativo MariaDB, la función **HListFile** solo devuelve la lista de archivos de la base de datos correspondiente. [HListDatabase](../WDLang4/3044288.md) enumera las bases de datos accesibles y sus archivos.

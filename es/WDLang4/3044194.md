


## HUnlockRecNum (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Desbloquea un registro bloqueado con:

- la función [HLockRecNum](../WDLang4/3044152.md), 

- una función de lectura utilizada con un parámetro bloqueante (por ejemplo, función [HRead](../WDLang4/3044047.md) utilizada con la constante *hLockWrite* o *hLockReadWrite*).


**Observación**: **HUnlockRecNum** no desbloquea un record bloqueado por otra aplicación.



![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función sólo está disponible para los archivos de datos HFSQL. No está disponible para los archivos de datos a los que accede con JDBC. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Lock
HLockRecNum(CUSTOMER, hRecNumCurrent, hLockReadWrite)
// Modify the record
...
// Unlock
HUnlockRecNum(CUSTOMER, hRecNumCurrent)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HUnlockRecNum([<Data file> [, <Record number>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se desbloqueó el registro,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Data file>: (Cadena de caracteres opcional)`**

Nombre del archivo de datos utilizado. 
Si no se especifica este nombre, la función **HUnlockRecNum** utilizará el último archivo de datos utilizado por la última función HFSQL (que empieza por "H").

**`<Record number>: (Entero opcional)`**

Número de la Record a desbloquear. 
Si no se especifica este número (o es igual a 0 o a la constante *hCurrentRecNum*), **HUnlockRecNum** leerá el Record actual.
![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) **HUnlockRecNum** desbloquea sólo el record actual. No se puede especificar un número de Record que no sea el actual (para especificar el número actual de Record, utilice la constante *hRecNumCurrent*).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
La función **HUnlockRecNum** no tiene ningún efecto si:

- El archivo de datos está en modo monopuesto (véase [HMode](../WDLang4/3044039.md)).

- La Record no está cerrada.

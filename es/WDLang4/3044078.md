


## HListQueryParameter (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve la lista de parámetros de una consulta creada en el editor de consultas.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Retrieve the parameters of QRY_CustomerZip
ParamList is string
ParamList = HListQueryParameter(QRY_CustomerZip)

// Adds the list of parameters into COMBO_COMBO1
ListAdd(COMBO_COMBO1, ParamList)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HListQueryParameter(<Query>)`
---

**`<Result>: (Cadena de caracteres)`**

Lista de parámetros de consulta. Los diferentes parámetros están separados por caracteres CR (Carriage Return).

**`<Query>: (Cadena de caracteres)`**

Nombre de una consulta creada con el editor de consultas. Este nombre puede corresponder a:

- el nombre lógico de la consulta definida en el editor.

- el nombre y el ruta completo (o relativo) de la consulta (archivo".WDR"). Se puede utilizar una ruta de tipo UNC.

<a name="NOTE0_1"></a>

## Observaciones


### Visualización del resultado en un List Box o control Combo Box
<a name="visualizacion_del_resultado_list_box_control_combo_box_ELTPARAGRAPHE000068"></a>

&lt;Resultado&gt; puede asignarse directamente a un control List Box o Combo Box mediante la función [ListAdd](../WDLang1/3049004.md). Cada Line tendrá el nombre de uno de los parámetros de la consulta.


## Archivo description (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **archivo description** Variable se utiliza para describir mediante programación archivos de datos. La description de cada archivo de datos es validada por [HDescribeFile](../WDLang4/3044206.md).

Después de esta validación:

- Este archivo de datos puede ser manejado como cualquier otro archivo de datos descrito en el análisis.

- El "Fichero description" Variable se reinicializa y puede utilizarse para describir otro fichero de datos.







<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
AFile is File Description
MyFile1, MyFile2 are File Descriptions
```


<a name="Example2"></a>



```wl
// Describe the "CUSTOMER" data file
AFile is File Description

AFile.Name = "CUSTOMER"
AFile.Type = hFileNormal
AFile.FicCryptMethod = hCryptStandard

// Description of the data file items
...

// Validate the description of "CUSTOMER"
HDescribeFile(AFile)
```

### ¿Cómo describir los archivos de datos mediante programación? 
<a name="¿como_describir_los_archivos_datos_mediante_programacion_ELTPARAGRAPHE000327"></a>

Para describir archivos de datos de forma programada:

1. Declare the " **File description** ", " [Artículo description](../WDLang4/1514071.md) ", " [FullTextIndex Description](../WDLang4/1000017461.md) " and " [Enlace description](../WDLang4/1514039.md) " variables (if necessary).

2. Para cada archivo de datos:

	- Describa las características de del archivo de datos utilizando [Propiedades HFSQL](../Proprietes/2512070.md).

	- Describa las características de los artículos utilizando [Propiedades HFSQL](../Proprietes/2512070.md).

	- Validar la description de cada elemento ([HDescribeItem](../WDLang4/3044233.md)).

	- Si es necesario, describa las características de los índices de texto completo utilizando [Propiedades HFSQL](../Proprietes/2512070.md).

	- Validar la description de cada índice de texto completo ([HDescribeFullTextIndex](../WDLang4/1000017407.md)).

	- Validar el archivo de datos description ([HDescribeFile](../WDLang4/3044206.md)).




3. Si es necesario, describa las características de los enlaces utilizando [Propiedades HFSQL](../Proprietes/2512070.md).

4. Si es necesario, valide la description de cada enlace ([HDescribeLink](../WDLang4/3044191.md)).

### Manipulación de los elementos de un archivo de datos descritos de forma programática
<a name="manipulacion_los_elementos_archivo_datos_descritos_forma_programatica_ELTPARAGRAPHE000380"></a>

Cuando se crea un archivo de datos en el editor de análisis, el compilador reconoce automáticamente los nombres de los archivos y elementos de datos.

Al crear un archivo de datos mediante una variable "Descripción de fichero", los nombres del archivo de datos y sus elementos pueden definirse con el [denominar](../Proprietes/2510082.md) Property. Estos nombres no son reconocidos automáticamente por el compilador. Se produce un error de compilación si estos nombres se utilizan para identificar a el archivo de datos o los artículos.

El nombre del archivo de datos debe ser declarado como fuente de datos utilizando la palabra clave [origen de los datos](../WDLang4/1514053.md), para que el compilador reconozca este nombre. Este archivo de datos puede ser manejado por su nombre en el editor de código.

Por ejemplo:


```wl
AFile is File Description
...
AFile.Name = "CUSTOMER"
...
HDescribeFile("CUSTOMER")
...
CUSTOMER is Data Source
...
HReadFirst(CUSTOMER, "CUSTNAME")
```


**Observación**: El nombre del archivo de datos también puede declararse con la palabra clave [Externo](../Motscles/1511013.md). Sin embargo, la ejecución de Process será más lenta.
<a name="NOTE0_4"></a>




### Registro
<a name="registro_ELTPARAGRAPHE000407"></a>

Cuando se describe el archivo de datos de forma programada:

- si se registra el archivo de datos y si no se registra ningún elemento, todos los elementos (excepto las claves compuestas) se registran automáticamente.

- si se registra el archivo de datos y si al menos un elemento se registra, los otros elementos no se modifican.

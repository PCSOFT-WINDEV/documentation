
## Replicación basada en registros
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000177"></a>
La replicación es la operación que permite mantener actualizadas bases de datos remotas de estructuras idénticas en las que se realizan varias operaciones. Las operaciones realizadas en cada una de las bases de datos se aplican a todas las demás bases de datos.

Para obtener más información, consulte [La replicación](../WDLang4/3044266.md).

**WINDEV y WEBDEV proponen dos métodos para la implementación de la replicación basada en logs -**: 

- [**Uso de WDReplic**](../WDReplic/3520002.md). En este caso, no se requiere ninguna programación específica.. Todo lo que tiene que hacer es configurar las diferentes opciones de replicación. Esta herramienta (y los archivos de configuración necesarios) deberán ser instalados junto con las bases de datos a replicar.

- [**Uso de las funciones WLanguage**](../WDLang4/3044279.md). En este caso, la gestión de la replicación se realizará en la aplicación a través de opciones específicas. Estas opciones utilizarán las funciones de WLanguage para implementar la replicación.




**Observación**: La réplica basada en el registro no está disponible en HFSQL Client/Server. Le recomendamos que utilice el programa [replicación universal](../WDLang4/9000032.md).

## Implementación de la replicación basada en registros con WDReplic
<a name="implementacion_replicacion_basada_registros_con_wdreplic_ELTTEXTE000207"></a>


### Etapas
<a name="etapas_ELTPARAGRAPHE000039"></a>

Para implementar la replicación basada en registros con WDREPLIC, debe: 

1. Modificar el análisis (para más detalles, véase [Modificación del análisis para apoyar la replicación basada en registros](../WDLang4/3044246.md)).

2. Crear el ejecutable (para una aplicación WINDEV), luego la instalación Procedure. 
	Observación: Al crear la instalación Procedure, no olvides añadir WDReplic al programa instalación.

3. Instalar la aplicación (y WDREPLIC) en los sitios maestro y de suscriptores.. La aplicación instalación también incluye:

	- WDReplic si la replicación se realiza con esta herramienta

	- una actualización de automatic de los archivos de datos. Esta actualización se utiliza para Log( si es necesario) y para replicar los archivos de datos. Si los archivos ya estaban registrados, se borran los registros existentes..
			Recordatorio: Para implementar la replicación, los archivos de datos deben tener una estructura idéntica (análisis) en todos los ordenadores (ordenador maestro y ordenadores de abonado).




4. Configure WDReplic en el equipo maestro.

5. Instale los archivos requeridos por WDReplic en las computadoras de los suscriptores.




Para obtener más información, consulte [Ayuda sobre WDReplic](../WDReplic/3520002.md).
<a name="NOTE2_2"></a>


### Importante: condiciones de aplicación
<a name="importante_condiciones_aplicacion_ELTPARAGRAPHE000067"></a>

La implementación de la replicación basada en el registro en una aplicación es una operación importante.

Le recomendamos que respete algunas reglas básicas.

Para cada archivo en réplica, le recomendamos que utilice una única clave en sus archivos. Esta clave única debe ser un "identificador de 8 bytes de automatic".

Para obtener más información, consulte [Replicación basada en registros: Modificación del análisis](../WDLang4/3044246.md).

## Implementación de la replicación basada en registros mediante programación
<a name="implementacion_replicacion_basada_registros_mediante_programacion_ELTTEXTE000237"></a>
Para implementar la replicación basada en registros a través de la programación, debe:

1. Modificar el análisis (para más detalles, véase [Replicación basada en registros: Modificación del análisis](../WDLang4/3044246.md)).

2. Modificar la aplicación para gestionar la replicación con las funciones de WLanguage. Para obtener más información, consulte [Replicación basada en registros: Programa en WLanguage](../WDLang4/3044279.md)

3. Crear el ejecutable (para una aplicación WINDEV) y el programa instalación.  

4. Instalar la aplicación en el sitio maestro y en el sitio del suscriptor. La aplicación instalación debe incluir una actualización de los archivos de datos de automatic. Esta actualización se utiliza para Log( si es necesario) y para configurar los archivos de datos para la replicación. Si los archivos ya estaban registrados, los registros existentes se borran.




Recordatorio: Para poner en práctica la replicación basada en registros, los archivos de datos deben tener una estructura idéntica (análisis) en todas las computadoras (computadora principal y computadoras de suscripción).
<a name="NOTE3_2"></a>

Le recomendamos que respete algunas reglas básicas.

Para obtener más información, consulte [Replicación basada en registros: Modificación del análisis](../WDLang4/3044246.md).

## Precaución
<a name="precaucion_ELTTEXTE000267"></a>


### Mover los archivos de datos manualmente
<a name="mover_los_archivos_datos_manualmente_ELTPARAGRAPHE000120"></a>

Atención: Para mover los archivos de datos manualmente, debe cumplir con las siguientes reglas:

1. Actualice todas las réplicas en los ordenadores correspondientes.

2. copy los archivos de datos a los ordenadores correspondientes.

3. Guarde y luego elimine los siguientes archivos:

	- todos los archivos de registro (archivos".JNL").

	- todos los ficheros de tablas de identificación (ficheros"JournalIdentification.\*").

	- todos los archivos de las tablas de operaciones (archivos"JournalOpération.\*").




## <Window>.fromFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Actualiza automáticamente los controles de una ventana o Page con:

- los valores de los elementos vinculados en el Record actual (cargado en memoria) del archivo de datos descritos en el editor de análisis. 

- los valores de las variables vinculadas de la lengua WL. 


Esta operación se realiza independientemente del estado de los mandos (grises, inactivos o Invisible).

**Observaciones**:

- **<Window>.fromFile** no lee registros en el archivo de datos. El valor de la Record cargado en memoria se utiliza para actualizar los controles. Por lo tanto, un Record debe ser cargado en memoria previamente (con [&lt;Fuente&gt;.Read](../WDLang4/1000025019.md), [&lt;Fuente&gt;.ReadSeek](../WDLang4/1000025023.md), [&lt;Fuente&gt;.ReadFirst](../WDLang4/1000025022.md), etc.).

- Esta función es global a la ventana. Si se llama en una ventana interna, la ventana padre y las otras ventanas internas se beneficiarán de ello.. El mecanismo es idéntico para las páginas. 





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HReadSeekFirst(Customer, Name, "PAUL")
IF HFound() = True THEN
	WIN_Customer.FromFile(Customer)
END
// Read the record and transfer the items to the controls
HReadFirst(Customer, Name)
WIN_Customer.FromFile(Customer)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Window>.FromFile([<Source>])`
---

**`<Window>: (Nombre de ventana)`**

Nombre de la ventana o Page que contiene los controles que deben actualizarse. 
Si este parámetro no se especifica o es una cadena vacía (""), se utiliza la ventana del código actual.

**`<Source>: (Cadena de caracteres opcional)`**

Nombre de la fuente a la que están vinculados los controles que se van a actualizar. Esta fuente puede ser:

- El nombre de un archivo de datos: sólo se tienen en cuenta los controles vinculados a los elementos del archivo de datos.

- El nombre de un WLanguage Variable: sólo se tienen en cuenta los controles vinculados a la lengua de trabajo Variable.
	Observaciones: 

	- Si el nombre de la Variable está entre comillas, el control se vinculará a la Variable especificada . 

	- Si el nombre de la Variable no está entre comillas, el control se vinculará a los datos de la Variable (por ejemplo, una Variable que contenga el nombre de otra Variable).





Si no se especifica este nombre, se tendrán en cuenta todos los controles de ventana o de página que estén vinculados a un archivo de datos descritos en la editor de análisis o a un WLanguage Variable.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Vinculación entre un control y un WLanguage Variable o entre un control y un elemento de un archivo de datos
<a name="vinculacion_entre_control_wlanguage_variable_entre_control_elemento_archivo_datos_ELTPARAGRAPHE000285"></a>

Se puede determinar la vinculación entre un control y un WLanguage Variable o entre un control y un elemento de un archivo de datos:

- En el editor, en la ventana de control description (pestaña "Binding").

- De forma programada, con el programa [DataBinding](../Proprietes/2510060.md) Property.



<a name="NOTE0_2"></a>


### Vinculación a un elemento: tipos de fijaciones
<a name="vinculacion_elemento_tipos_fijaciones_ELTPARAGRAPHE000301"></a>

Existen dos tipos de vínculos entre una control y un artículo

- *unión directa*: En este caso, **<Window>.fromFile** transfiere el valor de los elementos del Record cargado en la memoria al control enlazado f006c00 , sin leer ningún valor.

- *enlace leyendo los registros en el archivo de datos associated*.



<a name="NOTE0_3"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000316"></a>

- **control Combo Box autocompletado vinculado a un archivo de datos HFSQL**: **<Window>.fromFile** no debe estar en la ventana "Declaraciones globales" Event. La función debe estar en la Event "Inicialización" de la ventana.

- Los caracteres de espacio en los elementos HFSQL se mantienen.

- **<Window>.fromFile** actualiza los controles Image vinculados a elementos de memo Image .

- **<Window>.fromFile** admite el valor NULL. Si el contenido del elemento es NULL, se mostrará una cadena vacía en el campo Editar control. Para cambiar este modo de visualización, utilice [ControlError](../WDLang1/3025036.md).

- **<Window>.fromFile** no está disponible en vistas. Los valores de los elementos deben asignarse a cada control.

- **Vinculación con elementos de "Contraseña segura"**:

	- En una Edición control vinculada a un elemento Contraseña, aparecen los caracteres "\*\*\*\*\*".. Si la Edición control es de tipo "Contraseña", se muestran puntos en su lugar. 

	- El valor del elemento "Contraseña" no puede volver a leerse hasta que el usuario cambie el valor de la contraseña.

	- El valor del elemento "Contraseña" puede seguir comparándose con una cadena (por ejemplo, "" o "1234").

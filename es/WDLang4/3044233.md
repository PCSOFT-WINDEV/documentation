
## HDescribeItem (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Describe un elemento de archivo mediante programación.

Tras la llamada a **HDescribeItem**:

- El artículo se puede manejar como cualquier otro artículo descrito en el análisis.

- El artículo description ya no puede ser modificado.

- La description Variable del artículo se reinicia. Se puede realizar un nuevo artículo, description.


## Sintaxis

### Describir un elemento de un archivo de datos creado mediante programación

`<Result> = HDescribeItem(<Data file> , <Item>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Data file>: (Variable de tipo File Description)`**

Nombre de la variable [Descripción del archivo](../WDLang4/1514065.md) que corresponde a el archivo de datos en la que se debe crear el elemento. 

**`<Item>: (Artículo description Variable)`**

Nombre de la [Artículo description](../WDLang4/1514071.md) Variable correspondiente al elemento que se va a crear.

### Descripción de un elemento temporal en formato Hyper 5.5 (sólo compatibilidad retroactiva)

`<Result> = HDescribeItem(<Logical name of the Hyper File 5.5 file> , <Item description> [, <Array dimension>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Logical name of the Hyper File 5.5 file>: (Cadena de caracteres)`**

Nombre del archivo de datos temporal en formato Hyper File 5.5 descrito por [HDescribeFile](../WDLang4/3044206.md) (Sintaxis 2).

**`<Item description>: (Cadena de caracteres)`**

description del artículo en el siguiente formato: &lt;nombre del artículo&gt;,&lt;tipo de artículo&gt; (sin espacio entre &lt;nombre del artículo&gt; y &lt;tipo de artículo>).

&lt;Nombre del artículo&gt; contiene el nombre del artículo

&lt;Tipo de elemento&gt; contiene el tipo de elemento y puede tomar uno de los siguientes valores:

- E: para los ítems enteros

- M: para los ítems enteros largos

- C: para los ítems de Entero Corto

- S: para las posiciones reales simples

- D: para los artículos de Double Real

- A: para las posiciones de Fecha en 8 dígitos (AAAAMMDD)

- H: para las posiciones de tiempos (HHMM)

- T,&lt;Tamaño: para los elementos de texto (&lt;Tamaño&gt; es el tamaño de la cadena de caracteres)

- M: para las posiciones de nota de texto

- O: para las posiciones de la nota binaria

- F: para las posiciones de moneda




**`<Array dimension>: (Entero opcional)`**

Tamaño de la array de artículos.

## Observaciones




### ¿Cómo describir un artículo mediante la programación?
<a name="¿como_describir_articulo_mediante_programacion_ELTPARAGRAPHE000127"></a>

Para describir un elemento a través de la programación, debe:

1. Utilice las variables [Descripción de la relación](../WDLang4/1514039.md), [Expediente description](../WDLang4/1514065.md) y [Artículo description](../WDLang4/1514071.md) (si es necesario).

2. Describa las características de del archivo de datos, sus elementos y enlaces utilizando las propiedades HFSQL.

3. Definir la descripción de los elementos con la función **HDescribeItem**.

4. Definir el description del archivo de datos ([HDescribeFile](../WDLang4/3044206.md)) y los enlaces ([HDescribeLink](../WDLang4/3044191.md)).

### Sintaxis 2: Describir un elemento temporal en formato Hyper File 5.5 (compatibilidad con versiones anteriores)
<a name="sintaxis_2_describir_elemento_temporal_formato_hyper_file_55_compatibilidad_con_versiones_anteriores_ELTPARAGRAPHE000167"></a>

Esta sintaxis se utiliza para crear elementos temporales en los archivos de datos temporales en formato Hyper File 5.5.. Esta sintaxis sólo puede utilizarse en una aplicación desarrollada con WINDEV 5.5 o WEBDEV 1.5 y luego migrada a la versión 18.. Para utilizar esta función:

- el análisis en formato WINDEV 5.5 debe ser accesible en el ordenador de desarrollo.

- la ruta del análisis WINDEV 5.5 debe especificarse en las características del análisis (pestaña"Detalles").


## Archivos de datos HFSQL de más de 2 gigabytes 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000114"></a>
Por Default, el motor HFSQL permite gestionar grandes archivos de datos (hasta 2 gigabytes). 

Cuando el tamaño de un archivo de datos (incluyendo su índice, su índice de texto completo y/o su memo) excede los 2 gigabytes, se requiere una configuración específica en el editor de análisis y en el hardware.

De hecho, para gestionar archivos de datos de más de 2 gigabytes, debe utilizar:

- un sistema operativo que soporta archivos de más de 4 gigabytes. Los siguientes sistemas operativos no soportan archivos de más de 4 gigabytes: Windows 98 y Windows Me.

- Un sistema de archivos que soporta archivos de más de 4 gigabytes (NTFS por ejemplo en Windows). 




**Atención**: Esta configuración es necesaria en el equipo donde se instalan los archivos de datos y en las computadoras que acceden a estos archivos de datos. Por lo tanto, un ordenador equipado con Windows 98 no puede acceder a un archivo de datos que exceda los 2 gigabytes instalados en Windows 2000.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Configuración de los ficheros de datos (en el editor de análisis)
<a name="configuracion_los_ficheros_datos_editor_analisis_ELTTEXTE000144"></a>
Para gestionar los archivos de datos que excedan de 2 gigabytes: 

1. Visualizar la description del fichero de datos solicitado ("description del fichero de datos" de el menú contextual).

2. En la pestaña "Detalles", seleccione la opción "Gestionar archivos de datos de más de 2 GB (NTFS únicamente)".




**Atención**: Estos archivos de datos sólo se pueden utilizar desde un ordenador equipado con un sistema operativo que soporte archivos de más de 4 gigabytes.

## Uso de archivos de datos de más de 2 gigabytes
<a name="uso_archivos_datos_mas_2_gigabytes_ELTTEXTE000168"></a>


### description dinámica de archivos de datos
<a name="description_dinamica_archivos_datos_ELTPARAGRAPHE000045"></a>

Durante la dinámica description de un archivo de datos (a través de las propiedades HFSQL y [HDescribeFile](../WDLang4/3044206.md)), el modo de gestión del tamaño del archivo puede ser configurado por [HugeFile](../Proprietes/2512037.md).
<a name="NOTE3_2"></a>


### Propiedad WLanguage
<a name="propiedad_wlanguage_ELTPARAGRAPHE000058"></a>




|   |   |
| --- | --- |
| [HugeFile](../Proprietes/2512037.md) | El **HugeFile** Property se utiliza para configurar el tamaño máximo de un archivo de datos. |




<a name="NOTE3_3"></a>


### Reindexación
<a name="reindexacion_ELTPARAGRAPHE000065"></a>

**Atención**: La reindexación de archivos de datos de más de 2 gigabytes puede llevar bastante tiempo.

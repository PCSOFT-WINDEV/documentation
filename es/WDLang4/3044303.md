## Presentación
<a name="presentacion_ELTTEXTE000125"></a>
Una aplicación HFSQL puede funcionar en modo de red o en modo Cliente/Servidor.

Para saber qué modo de funcionamiento se adapta a sus aplicaciones, comparemos estos dos modos.


|   |   |
| --- | --- |
| **En el modo de red HFSQL** | **En modo HFSQL Client/Server** |
| Una aplicación de red HFSQL se ejecuta en diferentes computadoras de usuario. | Una aplicación HFSQL Client/Server se ejecuta en diferentes ordenadores de usuario (llamados ordenadores cliente). |
| Los archivos de datos se encuentran en un directorio accesible a los ordenadores de los usuarios.. Cada equipo accede físicamente a los archivos de datos. | Los archivos de datos se encuentran en un servidor. Solo el servidor accede físicamente a los archivos de datos. |
| Los procesos (consultas, operación de lectura/agregación en un archivo de datos, ...) se realizan en cada ordenador de usuario. | Todos los procesos (consultas, operaciones de lectura/agregación en un archivo de datos, ...) se realizan en el servidor |

## Ejemplos de uso

### Leyendo una Record
<a name="leyendo_una_record_ELTPARAGRAPHE000050"></a>


|   |   |
| --- | --- |
| <br><br>- 1 acceso a la red para leer los índices (archivo".NDX").<br><br>- 1 acceso a la red para leer el Record (archivo "FIC").<br><br><br> | <br><br>- 1 acceso de red para solicitar el Record hacia el servidor.<br><br><br><br><br>Observación: Cada computadora cliente mantiene una caché que contiene bloques de registros. Por lo tanto, no se requiere un nuevo acceso a la red para leer los siguientes registros. |
| En el modo de red HFSQL | En modo HFSQL Client/Server |

**Ejecutar una simple consulta de selección**:


|   |   |
| --- | --- |
| <br><br>- De manera simplificada, tantos accesos a la red como el número de registros recuperados (para el archivo índice **y** el archivo de datos).<br><br><br><br><br>Por ejemplo, 500 accesos de red si el resultado de la consulta contiene 1000 registros. | <br><br>- 1 acceso a la red para enviar la consulta.<br><br>- 1 acceso a la red para recuperar el resultado de la consulta.<br><br><br><br><br>Por ejemplo, 500 accesos de red si el resultado de la consulta contiene 2 registros. |
| En el modo de red HFSQL | En modo HFSQL Client/Server |


## HClusterIgnoreSynchro( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Define un nodo del cluster HFSQL como fuente de datos para realizar la sincronización del cluster.

**Observación**: Esta es una función avanzada. Le recomendamos que utilice el HFSQL control Center para gestionar un cluster HFSQL.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Resynchronizes the HFSQL cluster from the database found on the host3 node
HClusterIgnoreSynchro("clustermanager.mydomain.com", "secret", "host3.mydomain.com")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HClusterIgnoreSynchro(<Coordinator> , <Password> , <Node>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si la función se ejecuta con éxito,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Coordinator>: (Cadena de caracteres)`**

Nombre DNS del ordenador donde está instalado el coordinador del clúster.

**`<Password>: (Cadena de caracteres)`**

Contraseña utilizada para conectarse con el coordinador. Esta contraseña se dio cuando [instalación del cluster](../WDLang4/1000019031.md).

**`<Node>: (Cadena de caracteres)`**

Nombre DNS del nodo cuya base de datos se utilizará como reference al resincronizar el cluster.






## Observaciones
Cuando todos los nodos de un cluster han sido anormalmente detenidos (por ejemplo, a causa de un corte de energía o de un fallo de red), ningún nodo está seguro de contener los datos más actualizados.. En este caso, **HClusterIgnoreSynchro** permite definir manualmente el nodo que se utilizará como reference al sincronizar el cluster

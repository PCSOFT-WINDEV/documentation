
## Enlace description (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Un enlace **description** Variable se utiliza para describir un enlace entre dos archivos de datos mediante programación. El description de cada enlace es validado por el [HDescribeLink](../WDLang4/3044191.md).

Después de esta validación:

- Este enlace puede ser manejado como cualquier otro enlace descrito en el análisis.

- El "Enlace description" Variable se reinicializa y puede utilizarse para describir otro enlace mediante programación.




**Observación**: Para añadir una relación a una base de datos HFSQL Client/Server, utilice [HAddLink](../WDLang4/3044312.md).




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyLink is Link Description
MyLink1, MyLink2 are Link Descriptions
```


<a name="Example2"></a>



```wl
// Describe and validate City and Customer data files
...

// Describe the "LIVES" link
MyLink is Link Description
MyLink.Name = "LIVES"
MyLink.SourceFile = "CITY"
MyLink.LinkedFile = "CUSTOMER"
MyLink.SourceKey = "CITYNAME"
MyLink.LinkedKey = "CUSTCITY"
// Default link (0-1, 0-1)

// Validate the description of the "LIVES" link
HDescribeLink(MyLink)
```

## Observaciones




### Propiedades específicas de las descripciones de los enlaces
<a name="propiedades_especificas_las_descripciones_los_enlaces_ELTPARAGRAPHE000095"></a>

Las siguientes propiedades son específicas de las descripciones de enlaces:

**Observación**: Las propiedades escritas en negrita seguidas de una estrella deben necesariamente ser inicializadas cuando se describe un enlace por programación.

| Nombre de la propiedad | Efecto |
| --- | --- |
| [MaxLinkedCardinality](../Proprietes/2512011.md) | Máxima cardinalidad del elemento relacionado en un enlace |
| [MinLinkedCardinality](../Proprietes/2512002.md) | Cardinalidad mínima del elemento relacionado en un enlace |
| [MaxSourceCardinality](../Proprietes/2512005.md) | Cardinalidad máxima de la fuente en un enlace |
| [MinSourceCardinalidad](../Proprietes/2512006.md) | Cardinalidad mínima de la fuente en un enlace |
| **[LinkedKey](../Proprietes/2512007.md) ( \* )** | Elemento del fichero relacionado utilizado en el enlace |
| **[SourceKey](../Proprietes/2512003.md) ( \* )** | Elemento del archivo de origen utilizado en el enlace |
| **[LinkedFile](../Proprietes/2512031.md) ( \* )** | Nombre del archivo relacionado utilizado en el enlace |
| **[SourceFile](../Proprietes/2512034.md) ( \* )** | Nombre del archivo fuente utilizado en el enlace |
| **[denominar](../Proprietes/2510082.md) ( \* )** | Nombre del enlace |
| [ModificaciónRegla](../Proprietes/2512051.md) | Regla de modificación (gestión de integridad) utilizada en un enlace |
| [Regla de borrado](../Proprietes/2512083.md) | Regla de borrado (gestión de integridad) utilizada en un enlace |


Por Default, las cardinalidades del enlace son (0-1) / (0-1).

### ¿Cómo describir los archivos de datos mediante programación?
<a name="¿como_describir_los_archivos_datos_mediante_programacion_ELTPARAGRAPHE000178"></a>

Para describir archivos de datos de forma programada:

1. Declare las variables de tipo "[Descripción del archivo](../WDLang4/1514065.md)", "[Artículo description](../WDLang4/1514071.md)", "[FullTextIndex Description](../WDLang4/1000017461.md)" y "**Link description**" (si es necesario).

2. Para cada archivo de datos:

	- Describa las características de del archivo de datos utilizando [Propiedades HFSQL](../Proprietes/2512070.md).

	- Describa las características de los artículos utilizando [Propiedades HFSQL](../Proprietes/2512070.md).

	- Validar la description de cada elemento ([HDescribeItem](../WDLang4/3044233.md)).

	- Si es necesario, describa las características de los índices de texto completo utilizando [Propiedades HFSQL](../Proprietes/2512070.md).

	- Validar la description de cada índice de texto completo ([HDescribeFullTextIndex](../WDLang4/1000017407.md)).

	- Validar el archivo de datos description ([HDescribeFile](../WDLang4/3044206.md)).




3. Si es necesario, describa las características de los enlaces utilizando [Propiedades HFSQL](../Proprietes/2512070.md).

4. Si es necesario, valide la description de cada enlace ([HDescribeLink](../WDLang4/3044191.md)).

### Manejo de los enlaces de un archivo de datos descritos mediante programación
<a name="manejo_los_enlaces_archivo_datos_descritos_mediante_programacion_ELTPARAGRAPHE000231"></a>

Cuando se crea un enlace en la editor de análisis, el compilador reconoce automáticamente el nombre del enlace.

Cuando se crea un enlace a través de una variable "Link description", el nombre de este enlace se define con el [denominar](../Proprietes/2510082.md) Property. Este nombre no es reconocido automáticamente por el compilador. El uso de este nombre para identificar el enlace genera un error de compilación.

Para que el nombre del enlace creado a través de la programación sea reconocido por el compilador, este nombre debe ser declarado a través de la palabra clave [Externo](../Motscles/1511013.md). Este enlace ahora puede ser manejado por su nombre en el editor de código.

Por ejemplo:


```wl
MyLink is Link Description
...
MyLink.Name = "LIVES"
...
HDescribeLink("LIVES")
...
EXTERN LIVES
...
Cardinality = LIVES.MaxLinkedCardinality
```


## HSubstDir( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Asocia el directorio de datos específicado en el análisis con un directorio ubicado en disco. Después de ejecutar la función **HSubstDir**, todos los archivos de datos asociados al directorio especificado en el análisis se manipularán en el directorio indicado.

**Atención**: Los archivos de datos deben cerrarse previamente ([HClose](../WDLang4/3044073.md)).

Para modificar la ruta de un solo archivo de datos, utilice [HChangeDir](../WDLang4/3044168.md).

**Observación**: En modo Cliente/Servidor, utilice [HChangeDir](../WDLang4/3044168.md).

## Sintaxis

`<Result> = HSubstDir(<Directory of files specified in analysis> , <Physical directory of files on disk>)`
---

**`<Result>: (booleano)`**

Este parámetro siempre es igual a <u><u><u><u>True</u></u></u></u>. Si se produce un error, la próxima función que se utilice para acceder al archivo de datos devolverá <u><u><u><u>False</u></u></u></u>. Por ejemplo, si el directorio físico especificado no existe, se producirá un error de directorio durante el primer acceso a el archivo de datos.

**`<Directory of files specified in analysis>: (Cadena de caracteres)`**

Nombre del directorio especificado durante el description del archivo de datos.

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Si este parámetro es igual a "?" o ".", se tiene en cuenta el directorio actual de la aplicación (directorio devuelto por [fCurrentDir](../WDLang1/3036022.md)).

**`<Physical directory of files on disk>: (Cadena de caracteres)`**

Nuevo nombre del directorio (hasta 260 caracteres). Se puede utilizar una ruta de tipo UNC.

La función **HSubstDir** se ignora si el directorio especificado contiene archivos de datos en un formato distinto a HFSQL.

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Si este parámetro corresponde a:

- una cadena vacía ("""): se cancela la sustitución: se tiene en cuenta el directorio de datos descrito en el análisis.

- "?" o ".": el directorio de datos físicos corresponde al directorio actual.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Múltiples sustituciones de directorios
<a name="multiples_sustituciones_directorios_ELTPARAGRAPHE000122"></a>

Se puede llamar a la función **HSubstDir** varias veces con parámetros diferentes. Cuando el directorio físico se redefine varias veces para el mismo directorio lógico (descrito en el análisis), el último directorio físico especificado reemplaza a todos los demás.
<a name="NOTE0_2"></a>




### Sustitución de directorios
<a name="sustitucion_directorios_ELTPARAGRAPHE000132"></a>

[HChangeDir](../WDLang4/3044168.md) se utiliza para asociar un directorio a un archivo de datos. Para sustituir un directorio por otro, utilice **HSubstDir**.

Atención: la sustitución de directorio realizada con la función **HSubstDir** se aplica al directorio definido con la función [HChangeDir](../WDLang4/3044168.md). Por ejemplo:


```wl
HSubstDir("C:\Statistics\January", "C:\New\JanStats")
HChangeDir(Stat, "C:\Statistics\January")
HCreation(Stat) //File created in C:\New\JanStats
```

<a name="NOTE0_3"></a>




### Varios
<a name="varios_ELTPARAGRAPHE000153"></a>

- Si la gestión de ".REP" está habilitada ([HSetREP](../WDLang4/3044061.md)), la modificación de la ruta se guarda en el archivo ".REP" (véase [gestión de archivos .REP](../WDLang4/3044186.md)).

- La nueva ruta se almacena cuando se cierra el archivo de datos.

- El número de sustituciones es ilimitado.

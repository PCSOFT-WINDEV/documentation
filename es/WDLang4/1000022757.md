


## &lt;Variable Connection&gt;.DeleteFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina del servidor los archivos de datos HFSQL (archivos .fic, .ndx, .ftx y .mmo si existen).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyConnection is Connection

// Deletes data files from the server 
MyConnection.DeleteFile(".\backup\customerfile_bck.fic")
```

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) 
```wl

IF Ct_test.DeleteFile(".\Customer.FIC") = False THEN
	Error(HErrorInfo())
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Connection>.DeleteFile(<Data file path> [, <Options>])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se eliminó el archivo de datos,

- <u><u><u><u>False</u></u></u></u> si se produce un error (por ejemplo, si el archivo de datos no existe en el servidor). La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Connection>: (Variable de tipo Connection)`**

Nombre de la [Conexión](../WDLang4/1514073.md) Variable que describe la conexión en la que se debe eliminar el archivo de datos.

**`<Data file path>: (Cadena de caracteres)`**

Nombre físico y ruta del archivo de datos para borrar. El ruta es relativo a la base de datos: el nombre de la base de datos no debe especificarse en el ruta. La extensión ".fic" del archivo de datos debe ser especificada.

**`<Options>: (Constante o combinación de constantes de tipo Integer (opcional))`**

Por Default, al borrar un archivo de datos: 

- los archivos correspondientes a los registros no se eliminan,

- se eliminan los archivos correspondientes a las transacciones. 




Este parámetro se utiliza para modificar este comportamiento:



|   |   |
| --- | --- |
| *hDeleteLog* | Borra el archivo de registro y todas las operaciones registradas associated con el archivo de datos eliminado. |
| *hDestroyLink* | Elimina todos los enlaces relacionados con el archivo de datos eliminado. |
| *hKeepTRS* | Mantiene el archivo de transacciones associated con el archivo de datos eliminado. |

<a name="NOTE0_1"></a>

## Observaciones


### Borrar un archivo de datos
<a name="borrar_archivo_datos_ELTPARAGRAPHE000282"></a>

El archivo de datos sólo puede ser eliminado si no se utiliza actualmente (es decir, si no es utilizado por una consulta, una vista, otra ventana, un thread o un componente con contexto HFSQL independiente).

El archivo de datos se cierra automáticamente en el contexto HFSQL que llama a la función **&lt;Variable Connection&gt;.DeleteFile**. Se produce un error de WLanguage si se utiliza el archivo de datos en un contexto o por una aplicación.
<a name="NOTE0_2"></a>


### Nombres de archivos y directorios en el servidor LINUX
<a name="nombres_archivos_directorios_servidor_linux_ELTPARAGRAPHE000296"></a>

En un servidor LINUX, el nombre de los archivos de datos (archivos.fic,.ftx,.ndx y.mmo si existen) deben escribirse en minúsculas y no deben contener ningún carácter acentuado.




## HExecuteProcedimiento (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ejecuta un procedimiento almacenado.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Run the "SalesPerYear" procedure and pass "2019" in parameter
HExecuteProcedure("HFConnection", "SalesPerYear", 2019)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HExecuteProcedure(<Connection> , <Stored procedure> [, <Parameter 1> [... [, <Parameter N>]]])`
---

**`<Result>: (Tipo del resultado)`**



- Resultado de la Procedure almacenada: Record, array de Record s, array de tipos simples, etc.

- Cadena vacía ("") si &lt;Stored Procedure&gt; no devuelve un resultado.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión al servidor que ejecuta el Procedure almacenado. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).




**`<Stored procedure>: (Procedimiento WLanguage)`**

Nombre de la Procedure almacenada para ejecutar.

**`<Parameter 1>: (Variante opcional)`**

Primer parámetro a pasar a &lt;Stored Procedure&gt;.



**`<Parameter N>: (Variante opcional)`**

El noveno parámetro a pasar a &lt;Stored Procedure&gt;.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Gestión automática de errores
<a name="gestion_automatica_errores_ELTPARAGRAPHE000116"></a>

La gestión de errores de automatic está desactivada en los procedimientos almacenados. Sin embargo, [HOnError](../WDLang4/3044017.md) puede utilizarse para ejecutar un Procedure específico (Procedure almacenado).
<a name="NOTE0_2"></a>


### Pasar parámetros
<a name="pasar_parametros_ELTPARAGRAPHE000128"></a>

Los parámetros pasados al procedimiento almacenado son pasados "por reference": si estos parámetros se modifican en el Procedure almacenado, también se modificarán en la aplicación que ejecutó este Procedure almacenado.
<a name="NOTE0_3"></a>


### Ejecutando un Procedure almacenado desde un código SQL
<a name="ejecutando_procedure_almacenado_desde_codigo_sql_ELTPARAGRAPHE000137"></a>

Un Procedure almacenado puede ser llamado: 

- desde el código SQL, con [HExecuteSQLQuery](../WDLang4/3044084.md) o [SQLExec](../WDLang4/3072007.md) por ejemplo,

- desde una aplicación externa que utilice [Driver ODBC para HFSQL](../WDLang4/9000160.md) o [Proveedor de bases de datos OLE para HFSQL](../WDLang4/9000059.md). 




Para obtener más información, consulte [Procedimientos y consultas almacenados (HFSQL Client/Server)](../WDLang4/3044360.md).

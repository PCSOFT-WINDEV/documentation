


## HDeleteAll (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina todos los registros de un archivo de datos, vista HFSQL o consulta. Los registros se borran lógica y físicamente.. No pueden ser restaurados.

Se realizan las siguientes operaciones:

- los registros se eliminan: los registros ya no están referenciados en el índice. Los datos no se guardan.

- los índices (correspondientes a las claves de los registros) se eliminan del archivo de índices.

- los memos associated con los registros se borran del archivo de memos.

- los identificadores de automatic son reiniciados.


Atención: no se comprueba la integridad.

WINDEV, WINDEV Mobile y WEBDEV proponen una gestión de errores automatic para los siguientes tipos de errores: Error de duplicados, error de integridad, error de contraseña, error de conflicto de modificación, error de estado durante el conflicto de modificación y error de bloqueo. Para obtener más información, consulte [Gestión asistida de errores HFSQL](../WDLang4/3044188.md).

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función está disponible para los archivos de datos o consultas relacionado a una conexión HFSQL (clásica o cliente/servidor). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Clear data file without re-creating it
HDeleteAll(Customer)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HDeleteAll(<Data file>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se eliminaron los registros,

- <u><u><u><u>False</u></u></u></u> si se produce un problema (registro bloqueado, en transacción, etc.). La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos HFSQL, vista o consulta HFSQL utilizada.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Eliminación de registros de una consulta
<a name="eliminacion_registros_una_consulta_ELTPARAGRAPHE000106"></a>

Si se creó la consulta utilizando la constante *hModifyFile*, la función **HDeleteAll** elimina los registros en la fuente de la consulta.
<a name="NOTE0_2"></a>


### Eliminación de registros de una vista HFSQL
<a name="eliminacion_registros_una_vista_hfsql_ELTPARAGRAPHE000117"></a>

La función **HDeleteAll** elimina los registros de la vista, pero no los registros del archivo de datos utilizado para la vista.
<a name="NOTE0_3"></a>


### Borrar registros de una operación
<a name="borrar_registros_una_operacion_ELTPARAGRAPHE000128"></a>

La función **HDeleteAll** no elimina los registros de una transacción (cancelada o validada).
<a name="NOTE0_4"></a>


### Ventajas en comparación con un bucle para borrar registros
<a name="ventajas_comparacion_con_bucle_para_borrar_registros_ELTPARAGRAPHE000139"></a>

Para borrar los registros de un archivo de datos, también puede set un bucle para borrar los registros ([HDelete](../WDLang4/3044018.md)). Por ejemplo:

```wl
FOR EACH CustNum OF Orders
	HDelete()
END
```
 El uso de **HDeleteAll** tiene las siguientes ventajas:

- el Process es más rápido,

- el tamaño del archivo de datos vacío es menor.



<a name="NOTE0_5"></a>


### Ventaja en comparación con [HCreation](../WDLang4/3044255.md)
<a name="ventaja_comparacion_con_hcreationwdlang43044255md_ELTPARAGRAPHE000163"></a>

Para eliminar los registros de un archivo de datos, también tiene la posibilidad de utilizar [HCreation](../WDLang4/3044255.md). 

El uso de **HDeleteAll** tiene las siguientes ventajas:

- El borrado es posible incluso si los usuarios están conectados a el archivo de datos.

- automatic gestión de los errores del HFSQL.

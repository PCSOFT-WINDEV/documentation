


## HRetrieveBackup (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Recupera en el equipo local el contenido de una copia de seguridad comprimida (en formato ZIP) realizada en un servidor HFSQL.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Describe the connection
MyConnection is Connection

HRetrieveBackup(MyConnection, BackupID, "C:\temp\Myfile.zip", PROGBAR_Progress)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HRetrieveBackup(<Connection> , <Backup> , <Destination file> [, <Progress Bar>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se obtuvo la copia de seguridad,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión al servidor que contiene la copia de seguridad a recuperar. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).




**`<Backup>: (Entero o cadena de caracteres)`**

Copia de seguridad para recuperar. Puede corresponder a:

- **Backup identificador**. Este identificador puede corresponder al identificador de la copia de seguridad completa o al identificador de la copia de seguridad diferencial.. 
	Durante una copia de seguridad inmediata, este identificador es devuelto por [HBackup](../WDLang4/1000017081.md). Este identificador también puede ser devuelto por [HInfoBackup](../WDLang4/1000017083.md).
	En una copia de seguridad programada, este identificador corresponde a la propiedad **Identifier** de la variable [hBackupDescription](../WDLang4/1000017455.md). 

- **Ruta y nombre de la copia de seguridad**. Este nombre se especificó durante la copia de seguridad. Este nombre también puede ser devuelto por [HInfoBackup](../WDLang4/1000017083.md). Si este nombre corresponde a un ruta relativo, la copia de seguridad se buscará en el subdirectorio "Backup" del servidor HFSQL. Para especificar el directorio del servicio Manta, basta con utilizar "%%EXE%%".




**`<Destination file>: (Cadena de caracteres)`**

Ruta y nombre del archivo local en el que se recuperará la copia de seguridad.

En el caso de una copia de seguridad completa, el archivo de destino es un archivo zip (la extensión ".zip" debe ser especificada en el nombre del archivo. 

**`<Progress Bar>: (Nombre de control opcional, nombre de ventana opcional o entero opcional)`**



- Nombre de la ventana en la que se mostrará la barra de progreso o nombre de la control Barra de progreso.

- Mango de la ventana que muestra la barra de progreso.

## Observaciones


### Derechos necesarios para recuperar una copia de seguridad
<a name="derechos_necesarios_para_recuperar_una_copia_seguridad_ELTPARAGRAPHE000118"></a>

Para obtener una copia de seguridad, el usuario o el grupo debe tener los permisos para realizar copias de seguridad (constante *hRightsBackup*). Estos derechos se aplican a los servidores y a las bases de datos. Por ejemplo, para realizar una copia de seguridad completa del servidor, debe tener los derechos para realizar copias de seguridad a nivel de servidor.. 

Para saber si estos derechos están concedidos a un usuario o a un grupo, utilice [HInfoServerRights](../WDLang4/3044322.md) o [HInfoDatabaseRights](../WDLang4/3044320.md).

Para modificar los derechos concedidos a un usuario o a un grupo de usuarios, utilice [HModifyServerRights](../WDLang4/3044328.md) o [HModifyDatabaseRights](../WDLang4/3044326.md).

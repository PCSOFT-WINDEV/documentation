


## <redisConnection>PipelineReset (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ignora los resultados de la ejecución de un comando de Redis que fue añadido por [.PipelineAdd](../WDLang4/1410088283.md).<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
gRedis.PipelineAdd("get p2")
gRedis.PipelineAdd("get p1")
gRedis.PipelineAdd("set p3 v3")
gRedis.PipelineAdd("set p4 v4")

// Retrieve the first 2 results
r1 is redisResponse = gRedis.PipelineGet()
r2 is redisResponse = gRedis.PipelineGet()
// Ignore the following results
gRedis.PipelineReset()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = <Connection>.PipelineReset()`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,  

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Connection>: (Variable de tipo redisConnection)`**

Nombre del [redisConnection](../WDLang4/1000023527.md) Variable que corresponde a la conexión del servidor Redis.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Cómo funciona el oleoducto
<a name="como_funciona_oleoducto_ELTPARAGRAPHE000183"></a>

Cuando se han añadido todos los comandos deseados a la cadena, los resultados se pueden recuperar con [PipelineGet](../WDLang4/1410088285.md).

Para no obtener los resultados, se puede utilizar la función **<redisConnection>PipelineReset**.

Si no se utilizó la función **<redisConnection>PipelineReset**, se debe realizar el mismo número de llamadas a la función [PipelineGet](../WDLang4/1410088285.md) y a la función [.PipelineAdd](../WDLang4/1410088283.md).

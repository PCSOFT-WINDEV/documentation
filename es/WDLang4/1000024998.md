


## &lt;Fuente&gt;.InitSubscriber (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicializa la gama de identificadores automáticos para el archivo description de la réplica del abonado (extensión "RPL"). **Esta función sólo puede utilizarse con la replicación basada en registros**. 

El rango de los identificadores de automatic está inicializado:

- automáticamente, al crear un archivo de datos ([&lt;Fuente&gt;.Creation](../WDLang4/1000024207.md), [&lt;Fuente&gt;.CreationIfNotFound](../WDLang4/1000024208.md)).

- manualmente, con la función **&lt;Fuente&gt;.InitSubscriber**.




Se debe llamar a la función **&lt;Fuente&gt;.InitSubscriber** solo al instalar los archivos de datos en el esclavo.

**Atención**: Los archivos de datos deben abrirse antes de utilizar **&lt;Fuente&gt;.InitSubscriber**.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Initialize range of auto IDs for the Customer data file
Customer.InitSubscriber("C:\DATA\Replic\ParisSubscriber.RPL")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.InitSubscriber(<Subscriber Replica>)`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la inicialización,

- <u><u><u><u>False</u></u></u></u> si se produce un error. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos HFSQL a utilizar.

**`<Subscriber Replica>: (Cadena de caracteres)`**

Nombre y ruta completo de la réplica del abonado. Este parámetro puede corresponder a un UNC ruta.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Rango de identificadores
<a name="rango_identificadores_ELTPARAGRAPHE000224"></a>

Los identificadores de automatic para los archivos de las réplicas de los suscriptores utilizan una gama de identificadores que difiere de la utilizada por el archivo de la réplica maestra y por los otros archivos de las réplicas de los suscriptores.

Este rango de identificadores de automatic está escrito en el encabezado de cada archivo de datos (extensión "FIC").
Este rango se inicializa:

- al crear un archivo de datos ([&lt;Fuente&gt;.Creation](../WDLang4/1000024207.md), [&lt;Fuente&gt;.CreationIfNotFound](../WDLang4/1000024208.md)).

- mediante [WDReplic](../WDReplic/3520016.md).

- al utilizar la función **&lt;Fuente&gt;.InitSubscriber**.




La gama de identificadores es devuelta por [MinAutoID](../Proprietes/2512039.md) y [MaxAutoID](../Proprietes/2512023.md).

Para más detalles, consulte la página web de [consecuencias de la implementación de la réplica](../WDLang4/3044246.md).
<a name="NOTE0_2"></a>


### Replicación y registro Process
<a name="replicacion_registro_process_ELTPARAGRAPHE000270"></a>

Si se utiliza la función **&lt;Fuente&gt;.InitSubscriber**, debe inicializarse la ubicación de los registros y tablas de operaciones (función [&lt;Fuente&gt;.ChangeLogDir](../WDLang4/1000024203.md), [LogDirectorio](../Proprietes/2512075.md) y [OperaciónLogDirectorio](../Proprietes/2512076.md)).

**Recordatorio**: La replicación basada en registros utiliza registros y tablas de operación de archivos de datos. La replicación basada en los registros no funciona si la ubicación de los registros y las tablas de operaciones no es válida.




## HViewToFile (Función)

<DIV class="specObsolete">
	<IMG style="float:left;margin-right:10px;" src="https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=warning_big.png"><B>Advertencia</B><BR>
	A partir de la versión <B>20</B>, las vistas SQL y las vistas materializadas están disponibles.<br>Las vistas manipuladas por esta función corresponden al anterior mecanismo de vistas. No corresponden ni a las vistas SQL ni a las vistas materializadas. 
</DIV><a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Guarda las modificaciones realizadas en una vista HFSQL (por [HModify](../WDLang4/3044042.md), [HDelete](../WDLang4/3044018.md) o [HCross](../WDLang4/3044121.md)) en el archivo de datos correspondiente.

**Observación**: Esta operación solo se puede realizar si se creó la vista con la constante *hViewLocking* (es decir, que los registros importados en la vista se bloquearon en el archivo de datos). Tras la llamada a **HViewToFile**, los registros se desbloquean en el archivo fuente de la vista.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003044021&name=hviewtofile_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Saves the modifications of Customer34 view
// into Customer file
HViewToFile(Customer34)
```

<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HViewToFile(<View>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se transfirió la vista al archivo de datos,

- <u><u><u><u>False</u></u></u></u> en caso contrario. La función [HError](../WDLang4/3044088.md) permite obtener más detalles sobre el problema.




**`<View>: (Cadena de caracteres)`**

Nombre de la vista utilizada. Si no se especifica este nombre, **HViewToFile** manipular la última vista utilizada por una función HFSQL (función que empieza por "H").  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Si se produce un error al actualizar el archivo de datos, se utiliza [HError](../WDLang4/3044088.md) para identificar los registros que han causado el problema. Basta con recorrer de nuevo la vista y ejecutar [HError](../WDLang4/3044088.md) en cada Record de la vista.




## HExtractMemo (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Extrae el contenido de un elemento de memo binario de un archivo de datos HFSQL, de una consulta o de una vista HFSQL en un archivo físico (en disco).

## Sintaxis

### Extraer un memo

`<Result> = HExtractMemo(<Data file> , <Memo item> [, <Destination file>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos (consulta de vista) utilizado. Si no se especifica este nombre, la función **HExtractMemo** utilizará el último archivo de datos utilizado por la última función HFSQL (que empieza por "H").

**`<Memo item>: (Cadena de caracteres)`**

Nombre del elemento que contiene la nota binaria a extraer.

**`<Destination file>: (Cadena de caracteres opcional)`**

Ruta y nombre completo del archivo donde se debe guardar el contenido del elemento. Si no se especifica este parámetro, el memo binario se guarda en su ruta y archivo iniciales.. Se restauran la fecha de creación y la fecha de la última modificación de este archivo..
![OLE DB](https://doc.pcsoft.fr/ext/images/us/OLEDB.png)![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) Este parámetro es obligatorio porque el nombre del archivo fuente no se guarda en los memos de OLE DB.

**`<Destination file>: (Cadena de caracteres)`**

### Extrayendo un memorándum de un Record Variable

`<Result> = HExtractMemo(<Memo item> [, <Destination file>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Memo item>: (Variable de tipo Record)`**

Nombre del elemento que contiene la nota binaria a extraer. Utilice la siguiente notación: &lt;Nombre de Record Variable&gt;.&lt;Nombre de Memo item&gt;.

**`<Destination file>: (Cadena de caracteres opcional)`**

Ruta y nombre completo del archivo donde se debe guardar el contenido del elemento. Si no se especifica este parámetro, el memo binario se guarda en su ruta y archivo iniciales.. Se restauran la fecha de creación y la fecha de la última modificación de este archivo..
![OLE DB](https://doc.pcsoft.fr/ext/images/us/OLEDB.png)![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) Este parámetro es obligatorio porque el nombre del archivo fuente no se guarda en los memos de OLE DB.

### Elemento de nota binaria que utiliza miniaturas de HFSQL
<a name="elemento_nota_binaria_que_utiliza_miniaturas_hfsql_ELTPARAGRAPHE000188"></a>

Para extraer el contenido de una miniatura HFSQL de un elemento HFSQL, utilice [Miniaturas en miniatura](../Proprietes/2512115.md).

Para obtener más información, consulte [Miniaturas HFSQL](../WDLang4/3044348.md).
<a name="NOTE0_2"></a>
![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) 

### Gestión de memos en SQLite
<a name="gestion_memos_sqlite_ELTPARAGRAPHE000207"></a>

Se recomienda almacenar información en formato memo para datos cuyo tamaño es inferior a 1 MB. Los datos en formato de archivo externo deben guardarse en la memoria interna o externa del teléfono y la ruta de acceso a estos datos debe almacenarse en la base de datos.

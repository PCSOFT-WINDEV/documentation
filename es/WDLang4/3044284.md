


## HInfoServer (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve la información específica sobre el servidor. El servidor corresponde a el equipo en el que se establece la conexión.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Trace(HInfoServer("MyConnection", hInfoHardware))
// Return "Intel Pentium" + TAB + "639MB" + CR
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HInfoServer(<Connection> [, <Type of information>])`
---

**`<Result>: (Cadena de caracteres)`**

Información solicitada. Si se solicitan varios tipos de información, cada tipo de información se separa por un carácter CR (Carriage Return).

El orden en el que se devuelve la información (si se solicita) es el siguiente:

*hInfoVI* + *hInfoVersion* + *hInfoPlatform* + *hInfoHardware* + *hInfoName* + *hInfoLogicalName* + *hInfoDate*

**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión para la que debe recuperarse la información del ordenador servidor. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).


Se produce un error WLanguage si el nombre especificado no existe.

**`<Type of information>: (Constante o combinación de constantes opcional)`**

Tipo de información solicitada. El resultado depende del tipo de información solicitada.


|   |   |
| --- | --- |
| *hInfoAll*<br>(Valor predeterminado) | Información completa sobre el servidor (corresponde a la combinación * hInfoVi* + *hInfoVersion* + *hInfoPlatform* + *hInfoHardware* + *hInfoName* + *hInfoLogicalName* + *hInfoDate*).<br>Los diferentes valores están separados por caracteres CR (Carriage Return). |
| *hInfoCluster* | Información sobre el cluster HFSQL Client/Server, en el siguiente formato: <br> &lt;Nombre del Cluster Manager&gt;+ CR + &lt;Lista de nodos del clúster&gt;. <br><br>La lista de nodos de cluster tiene el siguiente formato: <br>&lt;Nombre del clúster&gt; + TAB + &lt;Estado del nodo&gt; + TAB + &lt;Versión&gt;. <br>Los estados del nodo pueden ser:<br><br>- 0: Nodo que no pertenece al cluster.<br><br>- 1: Nodo detenido.<br><br>- 2: Nodo en espera de ser iniciado.<br><br>- 3: Nodo que se está iniciando<br><br>- 4: Nodo en espera de la fuente de sincronización.<br><br>- 5: Nodo actualmente sincronizado.<br><br>- 6: Nodo habilitado.<br><br>- 7: Nodo detenido.<br><br>- 8: Estado desconocido.<br><br><br> |
| *hInfoDate* | Fecha y hora del servidor en 8 bytes en formato UTC (hora universal). Esta información es una cadena de caracteres con el siguiente formato "AAAAMMDDHHmmSS". |
| *hInfoFreeDiskSpace* | Espacio disponible en el disco del servidor, expresado en bytes. |
| *hInfoHardware* | Información sobre el hardware utilizado:<br><br>- Tipo de procesador (por ejemplo: Intel Pentium).<br><br>- Tamaño de la RAM del servidor (por ejemplo: 630 MB)<br><br><br>Esta información se devuelve en el siguiente formato:<br>&lt;Tipo de CPU&gt; = TAB + &lt;Tamaño de la RAM del servidor&gt; |
| *hInfoLogicalName* | Nombre del servidor dado en el archivo HFConf.ini o especificado durante la instalación. Este nombre aparece en el Centro HFSQL control. |
| *hInfoName* | Nombre del servidor (DNS) |
| *hInfoPlatform* | Información sobre la plataforma del servidor:<br><br>- Nombre del sistema operativo: <br><br>	- Windows Vista, Windows Vista (64 bits), Windows 7, Windows 8, Windows 8.1, Windows 10, Windows 2008, Windows 2008 R2, Windows 2012, Windows 2012, Windows 2012 R2.<br><br>	- LINUX seguido del nombre de la distribución. Por ejemplo: "LINUX Ubuntu 14"<br><br><br><br><br>- Versión del sistema: corresponde a Service Pack en Windows o cadena vacía si no hay ningún Service Pack instalado. En Linux, corresponde a la versión del núcleo de Linux (2.4.20-8 por ejemplo).<br><br><br>Esta información se devuelve en el siguiente formato:<br><a name="NOTE0"></a>

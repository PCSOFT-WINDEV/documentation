
## HSetCache( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite configurar la gestión de las cachés del motor HFSQL Client/Server.






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HSetCache(hPageSize, 2000)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HSetCache(<Parameter to modify> [, <Data file> [, <Item>]] , <Value>)`
---

**`<Result>: (Integro)`**

Valor anterior del parámetro modificado (valor antes de la modificación). 

**`<Parameter to modify>: (Constante)`**

Parámetro de la caché a modificar:


|   |   |
| --- | --- |
| *hPageSize* | Tamaño de un bloque de cachés. Este bloque de cachés se usará cuando se navegue por un archivo de datos. <br>Un gran bloque de cachés mejora la navegación y aumenta las transferencias a través de la red y el consumo de memoria..<br>Este parámetro corresponde al número de registros leídos durante la navegación. |



**`<Data file>: (Cadena de caracteres opcional)`**

Nombre del archivo de datos al que debe aplicarse la modificación. 
Si no se especifica este parámetro, la modificación se aplica a todos los ficheros de datos.

**`<Item>: (Cadena de caracteres opcional)`**

Nombre del artículo al que debe aplicarse la modificación. 
Si no se especifica este parámetro, la modificación se aplicará a todos los elementos que se encuentran en el archivo de datos especificado.

**`<Value>: (Integro)`**

Nuevo valor que debe darse al parámetro.

Para la constante *hPageSize*, este valor debe estar entre 1 y 5000 (el valor de Default en un PC es 1000 y en un móvil es 10).

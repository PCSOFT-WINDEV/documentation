
## hRSConfig (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **hRSConfig** permite definir las características avanzadas de una replicación entre 2 servidores HFSQL. Las características de esta replicación pueden ser definidas y modificadas por varias propiedades de WLanguage. 

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Cnt is Connection
ConfigReplication1 is hRSConfig

ConfigReplication1.Server = "atlas.mycompany.com:4996"
Add(ConfigReplication1.File, "MyDatabase")
Add(ConfigReplication1.File, "-MyDatabase\MyFilefic")
HRSAddConfig(Cnt, ConfigReplication1, hrsNoCopy)
```

## Observaciones
<a name="NOTE0_1"></a>


### Propiedades específicas de la description de las variables hRSConfig
<a name="propiedades_especificas_description_las_variables_hrsconfig_ELTPARAGRAPHE000028"></a>

Las siguientes propiedades pueden ser utilizadas para manipular una replicación entre servidores HFSQL:


| **Nombre de la propiedad** | **Tipo utilizado** | **Efecto** |
| --- | --- | --- |
| Archivo | Array de cuerdas | Lista de elementos a replicar. <br><br>Los elementos a replicar pueden corresponder a: <br><br>- una base de datos, <br><br>- un archivo de datos, <br><br>- un archivo de datos para excluir de la réplica. En este caso, el nombre del archivo debe ir precedido por "-" (menos).<br><br><br> |
| Bidireccional | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si la replicación debe realizarse en las dos direcciones: la replicación será bidireccional. <br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario (valor predeterminado): la replicación será unidireccional.<br><br><br> |
| Contraseña | Cadena de caracteres | Contraseña específica para la replicación. Esta contraseña debe corresponder a la contraseña especificada en [HRSInit](../WDLang4/1000020541.md) utilizada en el servidor de abonado. <br><br>Este Property está disponible en modo de escritura. |
| description | Cadena de caracteres | description de la replicación. |
| Identificador | Integro | Se utiliza para identificar la replicación manejada.<br><br>**Esta propiedad es de solo lectura**. |
| Inicialización | Integro | Permite seguir la evolución de la copia inicial de los elementos replicados del servidor de conexión (servidor maestro) al otro servidor (servidor esclavo) al realizar una copia en tarea de fondo (función [HRSAddConfig](../WDLang4/1000020492.md) utilizada con la constante *hrsCopyBackgroundTask*). <br><br>Si este Property: <br><br>- se incluye entre 0 y 1000, la réplica se añade actualmente. <br><br>- es set al 1000, la réplica fue añadida. <br><br>- no se especifica, la réplica no se ha añadido.<br><br><br><br><br>**Esta propiedad es de solo lectura**. |
| ModificaciónConflictoResolución | Constante de tipo Integer | Método para resolver conflictos: <br><br>- *hmcrHigherPriority* (valor de Default): Durante un conflicto, sólo se tiene en cuenta el valor de prioridad definido para el servidor.. Cuanto menor sea el valor de prioridad definido para el servidor, mayor será la prioridad concedida al servidor.. <br><br>- *hmcrMásRecientes*: Durante un conflicto, el elemento más reciente tiene prioridad. <br>	Atención: Este método requiere una sincronización del reloj en los diferentes servidores.  <br><br>- *hmcrProcedureWithModifiedItems*: Cuando se produce un conflicto, se inicia un Procedure almacenado (definido con el **Procedure ForModificationConflictResolution** Property). En esta Procedure, sólo los elementos modificados serán accesibles.  <br><br>- *hmcrProcedureAllItems*: Cuando se produce un conflicto, se inicia un Procedure almacenado (definido con el **Procedure ForModificationConflictResolution** Property). En este procedimiento, se podrá acceder a todos los artículos que se encuentran en la Record.<br><br><br> |
| ProcedimientoParaModificaciónConflictoResolución | Cadena de caracteres | Nombre del procedimiento almacenado que se utilizará si el método de resolución de conflictos (**ModificationConflictResolution**) corresponde a las constantes *hmcrProcedureWithModifiedItems* o *hmcrProcedureAllItems*.<br><br>Este Procedure almacenado debe regresar:<br><br>- la constante *opRetry* para validar la replicación de la modificación,<br><br>- la constante *opCancel* para cancelar la replicación de la modificación.<br><br><br>Este Procedure almacenado debe ser encontrado en la base de datos replicada en el servidor del suscriptor. |
| Programación | Variable de tipo [hScheduling](../WDLang4/1000018853.md) | Período de reproducción. <br><br>Si todos los miembros de la [hScheduling](../WDLang4/1000018853.md) Variable quedan vacíos, será una réplica en streaming. |
| Servidor | Cadena de caracteres | Servidor de destino de la replicación. Este servidor se identifica por: <br><br>- su nombre DNS o su IP Address.<br><br>- el puerto de replicación (4996 por Default). <br><br><br>Si el puerto difiere del puerto Default, se debe utilizar la siguiente sintaxis para especificarlo:  "&lt;Nombre del servidor&gt;:&lt;Puerto de réplica&gt;". Este puerto debe corresponder al puerto especificado en [HRSInit](../WDLang4/1000020541.md) utilizado en el servidor de abonado. |
| Spare | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> para realizar una réplica en modo de reserva,<br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario. <br><br><br><br><br>**Observación**: Este Property es incompatible con el **Bidireccional** Property. |

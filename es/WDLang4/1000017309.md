
## Restauración de una base de datos HFSQL Client/Server
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000133"></a>
Una copia de seguridad de una base de datos HFSQL Client/Server puede restaurarse en cualquier momento: 

- si la copia de seguridad realizada es una copia de seguridad completa, todo lo que tiene que hacer es restaurar esta copia de seguridad. 

- si el respaldo realizado es un respaldo diferencial, todo lo que tienes que hacer es restaurar el respaldo completo y el respaldo diferencial de associated. 




**Recordatorio**: La copia de seguridad diferencial contiene todas las modificaciones realizadas desde la copia de seguridad completa anterior.. 



Se puede activar la operación de restauración:

- [en el Centro HFSQL control](#NOTE2).

- [mediante programación](#NOTE3).




**Atención**: La restauración de una copia de seguridad debe realizarse con sumo cuidado. 

- Para realizar esta operación, se deben tener los derechos para realizar una copia de seguridad (constante *hRightsBackup*) así como los derechos para bloquear la base de datos (constante *hRightsLock*).

- Para restaurar todo el servidor (o las bases de datos __jnl y __trs), no se debe conectar ninguna otra aplicación al servidor.. La conexión utilizada no debe estar conectada a una base de datos.

- Para restaurar sólo las bases de datos, no debe haber ninguna conexión en la base de datos a restaurar.




**Observación**: Realizar una copia de seguridad y restaurarla con un nombre diferente le permite duplicar fácilmente una base de datos..

## Restaurar una copia de seguridad en el Centro HFSQL control
<a name="restaurar_una_copia_seguridad_centro_hfsql_control_ELTTEXTE000163"></a>
Para restaurar una copia de seguridad: 

1. Seleccione el servidor HFSQL Client/Server en la estructura de árbol a la izquierda de la pantalla.

2. Seleccione la pestaña "Copias de seguridad".

3. En tabla de las copias de seguridad existentes, seleccione la copia de seguridad que desea restaurar.. 

4. Haga clic en el botón "Restaurar la copia de seguridad. 

5. Validar la operación de restauración. 




**Observaciones**: 

- También tiene la capacidad de restaurar una copia de seguridad que se encuentra en un directorio específico del servidor.. Para ello, haga clic en el botón "Restaurar una copia de seguridad de un directorio. 

- Si la copia de seguridad se restaura en un disco de red (UNC o letra de unidad de red), se deben haber concedido los derechos necesarios al usuario..
	En este caso, para realizar la operación de restauración: 

	- especificar un ruta UNC para la ubicación de la copia de seguridad,

	- modificar las propiedades del servidor HFSQL Client/Server a través del administrador de servicios de Windows especificando un usuario que tenga los derechos necesarios para acceder a la ubicación UNC que contiene la copia de seguridad.

## Restauración de una copia de seguridad mediante programación
<a name="restauracion_una_copia_seguridad_mediante_programacion_ELTTEXTE000187"></a>


### Para restaurar una copia de seguridad de la base de datos
<a name="para_restaurar_una_copia_seguridad_base_datos_ELTPARAGRAPHE000066"></a>

Utilice la función [HRestoreBackup](../WDLang4/1000017230.md).
<a name="NOTE4_3"></a>


### Consejo: Usar una copia de seguridad de copy y renombrar una base de datos
<a name="consejo_usar_una_copia_seguridad_copy_renombrar_una_base_datos_ELTPARAGRAPHE000076"></a>

Para copy y renombrar una base de datos, tienes la posibilidad de usar el siguiente consejo:

1. Realice la copia de seguridad de la base de datos.

2. Restaurar la copia de seguridad de la base de datos con un nombre diferente.




Esta operación puede realizarse con [HRestoreBackup](../WDLang4/1000017230.md).


## HDBOpenNoLock( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
En modo de usuario único, abre un archivo de datos xBase sin bloquearlo. El archivo abierto se puede compartir entre varios programas.

En modo multiusuario, modo de funcionamiento equivalente al de [HDBOpen](../WDLang4/3044163.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HMode("*", hModeMono)
HDBOpenNoLock("ORDERS", "OR", "C:\FILE\ORDERS.DBF")
HDBIndex("ORDERS", "C:\FILE\ORDERS.NTX", "OrdNum")
HReadFirst("ORDERS", "OrdNum")
WHILE HOut() = False
	Display_order
	HReadNext()
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HDBOpenNoLock(<Logical name of xBase file> , <Abbreviation of xBase file> , <Full name of xBase file>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Logical name of xBase file>: (Cadena de caracteres)`**

Nombre lógico dado al archivo xBase. Este nombre se utilizará para manipular el archivo xBase por programación.

**`<Abbreviation of xBase file>: (Cadena de caracteres)`**

Prefijo (2 caracteres) utilizado a través de la programación para identificar un elemento del archivo. Por ejemplo, si el prefijo del fichero "Customer" es CU, la sintaxis para utilizar el elemento "Name" de este fichero es CU.NOMBRE.
Atención: si **HDBOpenNoLock** abre varios archivos, no pueden tener la misma abreviatura, aunque los archivos se hayan cerrado.

**`<Full name of xBase file>: (Cadena de caracteres)`**

Ruta y nombre completo del archivo xBase a abrir (con la extensión).

## Observaciones




### Uso de HDBOpenNoLock
<a name="uso_hdbopennolock_ELTPARAGRAPHE000059"></a>

Atención: La función **HDBOpenNoLock** solo debe utilizarse en programas que no modifican el archivo y que utilizan el modo de usuario único. El archivo abierto no está bloqueado. Por lo tanto, las siguientes opciones no deben utilizarse después de **HDBOpenNoLock**:

- [HAdd](../WDLang4/3044147.md)

- [HWrite](../WDLang4/3044092.md)

- [HFree](../WDLang4/3044064.md)

- [HModify](../WDLang4/3044042.md)

- [HCross](../WDLang4/3044121.md)

- [HDelete](../WDLang4/3044018.md)

- [TableSave](../WDLang1/3074008.md)

- [TableDelete](../WDLang1/3074032.md)



<a name="NOTE0_2"></a>




## &lt;Fuente&gt;.RetrieveItem (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el contenido de un elemento del registro actual (en el archivo de datos, vista o consulta).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
CustomerName is string
// The second item of the "Customer" data file 
// corresponds to the name of the customer
CustomerName = Customer.RetrieveItem(2)
// Equivalent to:
// CustomerName = Customer.Name
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Obtener un elemento especificando el índice

`<Result> = <Source>.RetrieveItem(<Item order> [, <Subscript>])`
---

**`<Result>: (Tipo de elemento)`**



- Valor del artículo,

- Cadena vacía ("") para los artículos con un valor NULL. 


Para una array, todas las posiciones de array se devuelven como strings.. Los elementos se separan mediante el parámetro &lt;Array separator&gt;.

**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos, vista o consulta utilizada.

**`<Item order>: (Entero)`**

Orden físico del artículo en el archivo de datos. El orden de los artículos comienza desde 1. 
Recordatorio: El orden de un elemento se puede obtener mediante sus propiedades (&lt;Archivo de datos&gt;.&lt;Elemento&gt;[Índice](../Proprietes/2512040.md)).

**`<Subscript>: (Entero opcional)`**

Índice del artículo para un artículo de array. 
Para los artículos que no son de array, el índice es opcional. Si se especifica, debe ser set a 1. 
Para los elementos array, si no se especifica el índice, se devuelven todos los elementos array (separados por caracteres TAB).


<a name="SYNTAX2"></a>

### Obtener un elemento especificando el separador de array

`<Result> = <Source>.RetrieveItem(<Item order> [, <Array separator>])`
---

**`<Result>: (Tipo de elemento)`**



- Valor del artículo,

- Cadena vacía ("") para los artículos con valor NULL.


Para una array, todas las posiciones de array se devuelven como strings.. Los elementos se separan mediante el parámetro &lt;Array separator&gt;.

**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos, vista o consulta utilizada.

**`<Item order>: (Entero)`**

Orden físico del artículo en el archivo de datos. El orden de los artículos comienza desde 1. 
Recordatorio: El orden de un elemento se puede obtener mediante sus propiedades (&lt;Archivo de datos&gt;.&lt;Elemento&gt;[Índice](../Proprietes/2512040.md)).

**`<Array separator>: (Cadena de caracteres opcional)`**

Cadena de caracteres utilizada para separar los valores de los elementos (o subelementos) de array en &lt;Resultado&gt;.. El separador de Default es TAB.
Atención: para la compatibilidad con la versión anterior de Hyper File 5.5, los separadores no se indican cuando los elementos de array están vacíos.

<a name="NOTE0_1"></a>

## Observaciones


### Elementos específicos
<a name="elementos_especificos_ELTPARAGRAPHE000265"></a>

- El contenido de los elementos de la nota binaria no se recupera, pero el separador de elementos está presente (para evitar un desplazamiento).

- El contenido de las claves compuestas no se recupera, pero el separador de elementos está presente (para evitar un desplazamiento).

### Lenguaje externo
<a name="lenguaje_externo_ELTPARAGRAPHE000278"></a>

En el lenguaje externo, el &lt;Resultado&gt; siempre corresponde a una cadena de caracteres. El contenido de una nota binaria no puede recuperarse según este método en un lenguaje externo.


## ReplicOpen( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Abre una replicación en modo directo o con un servidor de replicación.. Esta replicación fue definida por el editor de replicación (ReplicEdit).






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Open a replication in direct mode
ReplicOpen("MyReplication")
```


<a name="Example2"></a>



```wl
// Opens a replication on the replication server
ReplicOpen("MyReplication", "", "ReplicationServer", "Flora", "password")

// Initialize the replication by passing the zip code 
// to take into account
IF ReplicInfo("MyReplication", replicInitialized) = False THEN
	ReplicInitialize("MyReplication", 47012)
END

// Starts the synchronization
ReplicSynchronize("MyReplication")

// Process errors returned by Replication functions
CASE Error:
	Error(ErrorInfo(errFullDetails))
	RETURN
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Abrir una réplica en modo directo

`<Result> = ReplicOpen(<Replication> [, <Working directory>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se abrió la replicación,

- <u><u><u><u>False</u></u></u></u> si se produce un error. Para más detalles sobre el problema, utilice [ErrorInfo](../WDLang1/3013008.md).




**`<Replication>: (Cadena de caracteres)`**

Nombre del archivo de replicación a abrir (archivo con extensión ".wer"), sin ruta y sin extensión. Este archivo fue creado por el editor de replicación (ReplicEdit). Este archivo debe encontrarse junto al ejecutable.

**`<Working directory>: (Cadena de caracteres opcional)`**

Directorio donde se realizará la replicación (creación de los archivos requeridos por la replicación). Por Default (o si este parámetro corresponde a una cadena vacía ("")): 
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) este directorio corresponde al directorio del ejecutable de la aplicación (directorio devuelto por [fExeDir](../WDLang1/3036049.md)).


<a name="SYNTAX2"></a>

### Abrir una réplica con un servidor de réplicas

`<Result> = ReplicOpen(<Replication> , <Working directory> , <Server address> , <User> , <Password>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se estableció la conexión con el servidor de replicación,

- <u><u><u><u>False</u></u></u></u> si se produce un error. Para más detalles sobre el problema, utilice [ErrorInfo](../WDLang1/3013008.md).




**`<Replication>: (Cadena de caracteres)`**

Nombre de la réplica a abrir. Este nombre corresponde al nombre de la publicación de la replicación.

**`<Working directory>: (Cadena de caracteres)`**

Directorio donde se realizará la replicación (creación de los archivos requeridos por la replicación). Por Default (o si este parámetro corresponde a una cadena vacía ("")): 
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) este directorio corresponde al directorio del ejecutable de la aplicación (directorio devuelto por [fExeDir](../WDLang1/3036049.md)).

**`<Server address>: (Cadena de caracteres)`**

Address del servidor de replicación. Este Address puede corresponder:

- la IP Address del servidor en formato xxx.xxx.xxx.xxx

- el nombre del servidor.




**`<User>: (Cadena de caracteres)`**

Nombre de usuario. Este nombre corresponde a una cuenta creada en el servidor de replicación.

**`<Password>: (Cadena de caracteres)`**

Contraseña associated con el usuario.

## Observaciones


### Uso en modo directo
<a name="uso_modo_directo_ELTPARAGRAPHE000136"></a>

La llamada a esta función es opcional en modo directo porque es llamada automáticamente por las otras funciones de replicación (con los parámetros de Default) si no fue llamada previamente.


### Puertos de apertura
<a name="puertos_apertura_ELTPARAGRAPHE000141"></a>

Esta función utiliza el servidor de replicación que utiliza el servidor Web en el que está instalado.. Por lo tanto, el puerto 80 debe estar abierto en el servidor de replicación.

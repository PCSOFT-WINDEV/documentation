## Presentación
<a name="presentacion_ELTTEXTE000176"></a>
La replicación de datos es una característica poderosa. La replicación es la operación que permite el mantenimiento de bases de datos remotas con estructuras idénticas.. Cada una de estas bases de datos evoluciona de forma independiente: se realizan diferentes operaciones en estas bases de datos.

Mediante la replicación, las operaciones realizadas en cada una de las bases de datos se aplican a todas las demás bases de datos.

WINDEV, WINDEV Mobile y WEBDEV le permiten realizar fácilmente estas operaciones:

- entre las bases de datos HFSQL Classic. Para obtener más información, consulte [Replicación basada en registros](../WDLang4/3044256.md).

- entre bases de datos heterogéneas (HFSQL y Oracle por ejemplo). Para obtener más información, consulte [Replicación universal](../WDLang4/9000032.md). Esta replicación también está disponible para las bases de datos HFSQL Client/Server.

- entre 2 servidores HFSQL. Para obtener más información, consulte [Replicación entre servidores](../WDLang4/1000020588.md).

### Ejemplo de uso de la replicación de datos
<a name="ejemplo_uso_replicacion_datos_ELTPARAGRAPHE000074"></a>

En una empresa, el fichero "Cliente" debe ser accesible a todos los vendedores, tanto desde sus ordenadores portátiles como desde la sede central.. Cada vendedor actualiza el archivo encontrado en su ordenador portátil y recibe actualizaciones periódicas de la base de datos principal de la empresa.

***En una aplicación que no utiliza la reproducción de datos***la base de datos es accesible por red en la mayoría de los casos. El vendedor trabaja a través de una conexión por módem a la red, con todas las limitaciones en cuanto a tiempo de acceso.... Además, todos los usuarios deben estar conectados a la misma red para acceder a los datos.

***Con la replicación***cada uno de los ordenadores remotos tiene su propia base de datos. Esta base de datos (o réplica) es idéntica a la base de datos "maestra" durante su inicialización.. Regularmente, se utiliza una "Sincronización" de estas bases de datos para tener en cuenta las modificaciones realizadas en cada una de las diferentes bases de datos.

## Vocabulario específico de la réplica
<a name="vocabulario_especifico_replica_ELTTEXTE000212"></a>


### Bases de datos
<a name="bases_datos_ELTPARAGRAPHE000092"></a>

La replicación distingue entre dos tipos de bases de datos:

- **La base de datos maestra**
	Es la base de datos de reference. Todas las actualizaciones se realizan en esta base de datos:

	- modificaciones realizadas por la aplicación ejecutada en este ordenador.

	- modificaciones realizadas en los ordenadores remotos y transmitidas por la replicación.




- **La base de datos de réplicas o base de datos de suscriptores**
	Esta base de datos remota es idéntica a la base de datos "Master. Las modificaciones realizadas por el ordenador remoto se aplican a esta base de datos. La réplica transmite estas modificaciones a la base de datos "master".



<a name="NOTE2_2"></a>


### Tipo de replicación
<a name="tipo_replicacion_ELTPARAGRAPHE000109"></a>

Se pueden implementar dos tipos de replicación:

- **Replicación unidireccional**
	Este tipo de replicación consiste en realizar una actualización desde la base de datos "Maestro" a las bases de datos "Suscriptor" o desde una base de datos "Suscriptor" a la base de datos "Maestro".

- **Replicación bidireccional**
	Este tipo de replicación consiste en realizar una actualización de la base de datos "Master" a las bases de datos "Subscriber" y de las bases de datos "Subscriber" a la base de datos "Master".

### Archivos creados durante una replicación
<a name="archivos_creados_durante_una_replicacion_ELTPARAGRAPHE000123"></a>

Cuando la replicación se implementa en una aplicación WINDEV, se administran varios tipos de archivos específicos para la replicación:

- El **réplica maestra** (Archivo de extensión RPM): este archivo es associated con la base de datos maestra. Este archivo contiene los nombres e identificadores de las diferentes bases de datos de abonados, así como la fecha de la última sincronización.

- El **réplica de abonado** (Archivo con extensión RPL): este archivo es associated con la base de datos de suscriptores. Este archivo contiene el identificador de la base de datos de suscriptores, la fecha de la última sincronización, ....

- El **réplica móvil** (o medios móviles, archivo.RPA): este archivo es necesario para sincronizar dos bases de datos remotas que no están conectadas. La réplica móvil contiene la información necesaria para actualizar la base de datos remota.. Este archivo puede ser transmitido a la base de datos remota a través de FTP, Email o medios físicos.




## &lt;Fuente&gt;.UnlockRecNum (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Desbloquea un registro bloqueado con:

- la función [&lt;Fuente&gt;.LockRecNum](../WDLang4/1000024197.md), 

- una función de lectura utilizada con un parámetro bloqueante (por ejemplo, función [&lt;Fuente&gt;.Read](../WDLang4/1000025019.md) utilizada con la constante *hLockWrite* o *hLockReadWrite*).


**Observación**: **&lt;Fuente&gt;.UnlockRecNum** no desbloquea un record bloqueado por otra aplicación.



![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función sólo está disponible para los archivos de datos HFSQL. No está disponible para los archivos de datos a los que accede con JDBC. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Lock
Customer.LockRecNum(hRecNumCurrent, hLockReadWrite)
// Modify the record
...
// Unlock
Customer.UnlockRecNum(hRecNumCurrent)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.UnlockRecNum([<Record number>])`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> si se desbloqueó el registro,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HError](../WDLang4/3044088.md) permite identificar el error.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos utilizado. 

**`<Record number>: (Entero opcional)`**

Número de la Record a desbloquear. 
Si no se especifica este número (o es igual a 0 o a la constante *hCurrentRecNum*), **&lt;Fuente&gt;.UnlockRecNum** leerá el Record actual.
![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) **&lt;Fuente&gt;.UnlockRecNum** desbloquea sólo el record actual. No se puede especificar un número de Record que no sea el actual (para especificar el número actual de Record, utilice la constante *hRecNumCurrent*).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
La función **&lt;Fuente&gt;.UnlockRecNum** no tiene ningún efecto si:

- El archivo de datos está en modo monopuesto (véase [&lt;Fuente&gt;.Mode](../WDLang4/1000025037.md)).

- La Record no está cerrada.

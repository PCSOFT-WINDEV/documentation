


## HDropView (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina una vista materializada del servidor HFSQL. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Create a materialized view on a server
IF NOT HUpdateView(MyConnection1, MyMaterializedView) THEN
	Trace(HErrorInfo())
END

// Refresh the materialized view on the server
IF NOT HRefreshView(MyConnection1, MyMaterializedView) THEN
	Trace(HErrorInfo())
END

// Remove the materialized view from the server
IF NOT HDropView(MyConnection1, MyMaterializedView) THEN 
	Trace(HErrorInfo())
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HDropView(<Connection> , <View> [, <Cascading deletion>])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión al servidor HFSQL que contiene la vista materializada. Esta conexión corresponde a:

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Connection](../WDLang4/1514073.md).




**`<View>: (Cadena de caracteres)`**

Nombre de la vista materializada a manipular (en el servidor). 

**`<Cascading deletion>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> para habilitar la eliminación en cascada, 

- <u><u><u><u>False</u></u></u></u> (valor predeterminado) en caso contrario. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Si la vista materializada no existe, la función devuelve un error no fatal.


## HChangeRplDir( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica la ubicación de la réplica de la description del suscriptor (archivo RPL). Esta función debe utilizarse en el ordenador del abonado.

**Tes una función avanzada de la replicación basada en el registro**. Se debe utilizar cuando se manejan bases de datos no conectadas por red, por ejemplo. En este caso, la réplica maestra y la réplica de suscriptor se crean en el mismo ordenador (ordenador maestro).. Luego, el archivo RPL (archivo de réplica de suscriptor) se copia en la computadora del suscriptor.

Este archivo se puede instalar:

- en el directorio de réplica especificado en el análisis (subdirectorio\\RPL de los archivos de datos de Default).

- en otro directorio. En este caso, la aplicación **HChangeRplDir** esclava debe utilizar la función **HChangeRplDir** para indicar en dónde se debe buscar el archivo de la réplica esclava. De hecho, cuando se abre el archivo de datos de los suscriptores, se lee el archivo RPL para definir el rango de identificadores de automatic utilizados por el archivo de datos.




Por defecto, los archivos de replicación se gestionan en el directorio especificado en el análisis (pestaña "Análisis description", "Log Process"). <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=HChangeRepJNL%20-%20HC%20N%B0002.gif)



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// The RPL file and the data files are found in the same directory
HChangeRPLDir(".")

// The RPL file is located in the \RPL subdirectory of the data files
HChangeRplDir(".\RPL") 

// The RPL file is found in "X:\Data\RPL"
HChangeRplDir("X:\Data\RPL")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = HChangeRplDir(<Location of subscriber replica file>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un problema. La función [HErrorInfo](../WDLang4/3044071.md) permite identificar el error.




**`<Location of subscriber replica file>: (Cadena de caracteres)`**

Nuevo acceso ruta al archivo \*.RPL correspondiente a la réplica del suscriptor (ruta completo o relativo). Se utiliza este directorio:

- al abrir un archivo de datos HFSQL en la replicación en el ordenador del suscriptor. Se actualiza la gama de identificadores de automatic.

- al realizar la sincronización ([HSynchronizeReplica](../WDLang4/3044014.md) por ejemplo) si no se especifica ningún directorio. 
	**Observación**: si el directorio es un directorio relativo, el directorio será relativo al directorio Default de los archivos de datos definidos en el análisis.

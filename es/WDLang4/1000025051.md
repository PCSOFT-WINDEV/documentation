


## &lt;Fuente&gt;.Cross (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Tacha un registro en un archivo de datos. La Record se borra lógicamente y puede ser recuperada si es necesario.

Se realizan las siguientes operaciones:

- el Record se convierte en Inactive. Los datos se almacenan (lo que no ocurre cuando un Record es eliminado por [&lt;Fuente&gt;.Delete](../WDLang4/1000025071.md)).

- los índices (correspondientes a las claves de Record) se borran del archivo de índices.

- los memos asociados con la Record se almacenan.




Esta función se puede utilizar en los archivos de datos, las vistas de HFSQL Mobile o las consultas.


<a name="sample_code"></a>

## Ejemplo

<a name="Example1"></a>

```wl
// Crosses the record #123
Customer.Cross(123)
```
<a name="Example2"></a>

```wl
// Cross the customer orders
Order.ReadSeekFirst(CUSTNUM, ValCustNum)
WHILE Order.Found() = True
	Order.Cross()
	Order.ReadNext()
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.Crossed([<Record number> [, <Options>]])`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> si se tachó el registro,

- <u><u><u><u>False</u></u></u></u> si se produce un problema (registro bloqueado, problema de integridad, etc.). [HErrorInfo](../WDLang4/3044071.md) devuelve más detalles.




**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos HFSQL, consulta o vista utilizada. 

**`<Record number>: (Entero opcional)`**

Número de la Record a cruzar. 
Si no se especifica este parámetro (igual a 0 o a la constante *hCurrentRecNum*), el registro activo se tachará.

**`<Options>: (Constante opcional)`**

Configura:

- el modo de bloqueo aplicado a la cruzada Record. Si no se especifica una constante de bloqueo, la Record cruzada no está bloqueada.
	


|   |   |
| --- | --- |
| *hLockNo*<br>(valor predeterminado) | Sin bloqeuo: el cruce de Record no se cerrará. |
| *hLockReadWrite* | Bloqueo de lectura/escritura: la corriente cruzada Record no puede ser leída o reactivada por otra aplicación. |
| *hLockWrite* | Bloqueo de escritura: otra aplicación puede leer y consultar la record cruzada actual ([&lt;Fuente&gt;.Read](../WDLang4/1000025019.md)) pero no puede volver a activarla. Sólo la aplicación actual puede volver a activarla. |



- la gestión de la integridad aplicada a la cruzada Record.
	


|   |   |
| --- | --- |
| *hCheckIntegrity* | Comprueba la integridad de la operación, incluso si la comprobación de integridad de automatic está desactivada ([&lt;Variable Connection&gt;.SetIntegrity](../WDLang4/1000023960.md)). |
| *hIgnoreIntegrity* | Ignora la comprobación de integridad para esta operación, incluso si la comprobación de integridad de automatic está activada ([&lt;Variable Connection&gt;.SetIntegrity](../WDLang4/1000023960.md)). |


	Se produce un error de WLanguage si se utilizan las dos constantes al mismo tiempo.. Estas constantes no tienen ningún efecto en el borrado en cascada.

## Observaciones


### Manejando un Record cruzado
<a name="manejando_record_cruzado_ELTPARAGRAPHE000330"></a>

- Un Record cruzado no se leerá:

	- mediante funciones de iteración: [&lt;Fuente&gt;.First](../WDLang4/1000025048.md), [&lt;Fuente&gt;.Last](../WDLang4/1000024217.md), [&lt;Fuente&gt;.Next](../WDLang4/1000025070.md), [&lt;Fuente&gt;.Previous](../WDLang4/1000025047.md), [&lt;Fuente&gt;.ReadFirst](../WDLang4/1000025022.md), [&lt;Fuente&gt;.ReadLast](../WDLang4/1000025020.md), [&lt;Fuente&gt;.ReadNext](../WDLang4/1000025035.md), [&lt;Fuente&gt;.ReadPrevious](../WDLang4/1000025021.md), [&lt;Fuente&gt;.Forward](../WDLang4/1000024195.md), [&lt;Fuente&gt;.Backward](../WDLang4/1000025056.md)

	- por las funciones de búsqueda: [&lt;Fuente&gt;.SeekFirst](../WDLang4/1000025055.md), [&lt;Fuente&gt;.ReadSeekFirst](../WDLang4/1000025034.md), [&lt;Fuente&gt;.SeekLast](../WDLang4/1000025054.md), [&lt;Fuente&gt;.ReadSeekLast](../WDLang4/1000025024.md).




- Una Record cruzada puede ser leída por [&lt;Fuente&gt;.Read](../WDLang4/1000025019.md).

- Un Record cruzado puede volver a activarse mediante [&lt;Fuente&gt;.Modify](../WDLang4/1000025038.md) o [&lt;Fuente&gt;.Write](../WDLang4/1000024331.md).

- [&lt;Fuente&gt;.State](../WDLang4/1000024943.md) sirve para conocer el estado de record (active, cruzado, borrado, etc.).

- La ubicación física de una Record cruzada sólo puede ser reutilizada (por [&lt;Fuente&gt;.Add](../WDLang4/1000024187.md), por ejemplo) después:

	- la ejecución de [&lt;Fuente&gt;.Free](../WDLang4/1000025010.md),

	- una operación de reindexación (con compresión).






<a name="NOTE0_2"></a>


### Número de registros tachados
<a name="numero_registros_tachados_ELTPARAGRAPHE000448"></a>

[&lt;Fuente&gt;.NbRec](../WDLang4/1000025040.md) se utiliza para conocer el número de registros cruzados en el archivo de datos.

Observación: Para la compatibilidad con WEBDEV 1.5/WINDEV 5.5, el número de registros cruzados también puede ser devuelto por el *H.NbDel* Variable.
<a name="NOTE0_3"></a>


### Versión del archivo de datos
<a name="version_del_archivo_datos_ELTPARAGRAPHE000464"></a>

Si **&lt;Fuente&gt;.Cross** tiene éxito, se modifica el número de versión de el archivo de datos. Este número de versión es devuelto por [&lt;Fuente&gt;.Version](../WDLang4/1000025082.md).
<a name="NOTE0_4"></a>


### Cruzar un Record de una consulta
<a name="cruzar_record_una_consulta_ELTPARAGRAPHE000481"></a>

Un registro puede ser tachado:

- en el resultado de la consulta.

- en los archivos tratados por la consulta (*hModifyFile* constante utilizada en [&lt;Fuente&gt;.ExecuteQuery](../WDLang4/1000024944.md) o [&lt;Fuente&gt;.ExecuteSQLQuery](../WDLang4/1000024948.md)). En este caso, la constante *hWithFilter* se selecciona automáticamente.




Independientemente del modo de ejecución de la consulta (con o sin la constante *hWithFilter*), **&lt;Fuente&gt;.Cross** sólo puede utilizarse en consultas de un solo archivo.

Atención: 

- Los registros no pueden cruzarse en una consulta que utilice grupos o agregados.

- La gestión de la integridad y de los duplicados no se realiza durante las operaciones en las consultas multiarchivo.: se recomienda utilizar las transacciones para evitar cualquier problema.



<a name="NOTE0_5"></a>


### Casos especiales
<a name="casos_especiales_ELTPARAGRAPHE000513"></a>

- **Gestión de cierres**: 
	Si se produce un problema de bloqueo al "tachar" un registro (intentar tachar un registro bloqueado, por ejemplo), no se modifica el registro y la función [HErrorLock](../WDLang4/3044099.md) devuelve <u><u><u><u>True</u></u></u></u>.

- **Gestión de la integridad referencial**
	Si la gestión de la integridad referencial está activa (función [&lt;Variable Connection&gt;.SetIntegrity](../WDLang4/1000023960.md)), y no se verifica la integridad durante la eliminación, no se ejecuta la función **&lt;Fuente&gt;.Cross**. Se debe probar la función [HErrorIntegrity](../WDLang4/3044086.md) después de la función **&lt;Fuente&gt;.Cross**.

- **Registros borrados**
	No se debe llamar a la función **&lt;Fuente&gt;.Cross** en un registro eliminado previamente.

### Archivos xBase
<a name="archivos_xbase_ELTPARAGRAPHE000554"></a>

La función [&lt;Fuente&gt;.Delete](../WDLang4/1000025071.md) y **&lt;Fuente&gt;.Cross** tienen el mismo efecto: el Record se cruza pero no se borra. Por lo tanto, la Record puede volver a activarse mediante [&lt;Fuente&gt;.Modify](../WDLang4/1000025038.md). En este caso, la función [&lt;Fuente&gt;.State](../WDLang4/1000024943.md) devuelve la constante *hStateCrossed* en un registro eliminado (y no la constante *hStateDel*).

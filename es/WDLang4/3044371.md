
## Gestión del formato Unicode en los archivos HFSQL
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000179"></a>
El análisis propone dos tipos de elementos Unicode: 

- "Elemento "Texto Unicode

- "Elemento "Unicode Memo. 


Estos elementos se utilizan para almacenar información en formato Unicode.

Estos artículos pueden ser associated con un lenguaje: la clasificación realizada en el artículo tendrá en cuenta el idioma especificado. La asociación con el lenguaje se puede hacer:

- en el editor de análisis.

- mediante la programación a través del [H.LanguageCreationUnicode](../WDLang4/3044382.md) Variable. Este Variable debe utilizarse antes de crear los archivos de datos ([HCreation](../WDLang4/3044255.md) o [HCreationIfNotFound](../WDLang4/3044254.md)).




Los textos introducidos en los elementos Unicode son automáticamente soportados por los diferentes controles que se encuentran en la ventana o en la Page, las tablas, ...

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000209"></a>


### Creación de un elemento Unicode en el análisis
<a name="creacion_elemento_unicode_analisis_ELTPARAGRAPHE000041"></a>

Cuando se crea el artículo en la editor de análisis (para más detalles, véase [Operaciones sobre las partidas](../Editeurs/2011009.md)), basta con especificar el subtipo solicitado para el artículo. Puede ser una posición de texto:

- Un string (corresponde a un string Ansi)

- Un personaje

- Unicode (corresponde a una cadena de caracteres Unicode)

- Nota de texto (corresponde a una nota de texto Ansi)

- Nota de texto Unicode (corresponde a una nota de texto Unicode).




Al seleccionar un tipo Unicode (cadena o nota de texto), editor de análisis le permite especificar las características del idioma que se introducirá en este elemento.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Unicode_Analyse.gif)


Estas características son necesarias para realizar la clasificación de la posición. 

- Si se selecciona un idioma, este idioma se utilizará para realizar las clasificaciones del elemento actual.. Se realizará un ordenamiento lingüístico. La clasificación se realiza según el siguiente orden:

	1. Clasificación realizada en toda la cadena.

	2. Tener en cuenta los caracteres acentuados.

	3. Tener en cuenta los caracteres en mayúsculas.




- Si la opción "Idioma" corresponde a "< Defined at creation >", tiene la posibilidad de especificar las características del idioma mediante programación a través del [H.LanguageCreationUnicode](../WDLang4/3044382.md) Variable. Este Variable debe utilizarse antes de crear los archivos de datos ([HCreation](../WDLang4/3044255.md) o [HCreationIfNotFound](../WDLang4/3044254.md)). Por lo tanto, se puede utilizar el mismo análisis para gestionar varios idiomas Unicode.

- Si la opción "Idioma" corresponde a "", la ordenación se realizará comparando el valor binario de los bytes. Este tipo de clasificación es muy rápida. Se debe utilizar para realizar una búsqueda por coincidencia exacta, por ejemplo. 

- Si la opción "Idioma" corresponde a "", se realizará una comparison carácter por carácter según una ordenación genérica. Se realizará la misma clasificación, independientemente de los caracteres que se encuentren en el elemento.. 


<a name="NOTE2_2"></a>


### Cambio de Unicode a ANSI (y viceversa) en la editor de análisis
<a name="cambio_unicode_ansi_viceversa_editor_analisis_ELTPARAGRAPHE000091"></a>

El subtipo de un elemento puede modificarse cambiando de ANSI a Unicode (y viceversa).. En este caso, se realizará una modificación de los datos de automatic.

La conversión de ANSI a Unicode se realizará según el idioma especificado.

La conversión de Unicode a ANSI se realizará según el idioma actual.

## Para más información
<a name="para_mas_informacion_ELTTEXTE000239"></a>


### Controla la relacionado de los artículos
<a name="controla_relacionado_los_articulos_ELTPARAGRAPHE000104"></a>

Cuando se crean los control s (en una ventana o en una página), tiene la posibilidad de especificar si el control admite o no el formato Unicode (marque "Unicode" en la pestaña "General" de la ventana description del control).

Al crear los controles relacionado a un elemento Unicode, "Unicode" se marca automáticamente en el control description.

El [Unicode](../Proprietes/1000017001.md) Property se utiliza para determinar y especificar si el valor de un control es una cadena Unicode.
<a name="NOTE3_2"></a>


### Programación en WLanguage
<a name="programacion_wlanguage_ELTPARAGRAPHE000118"></a>

El método de programación de los archivos que contienen elementos Unicode es idéntico al método de programación de los archivos que contienen elementos ANSI.. Las opciones de idioma definidas en la editor de análisis se tienen en cuenta durante las búsquedas indexadas y durante las clasificaciones para definir el orden de los registros.

El [escribir a máquina](../Proprietes/2510131.md) Property se utiliza para determinar si un elemento es un elemento Unicode o Ansi.
<a name="NOTE3_4"></a>


### Buscar realizado por HReadSeek
<a name="buscar_realizado_por_hreadseek_ELTPARAGRAPHE000130"></a>

No se debe realizar ninguna búsqueda genérica en los elementos Unicode que distingan entre mayúsculas y minúsculas.. De hecho, al ser la codificación de caracteres específicos en Unicode, este tipo de búsqueda puede dar como resultado resultados inválidos o incompletos..

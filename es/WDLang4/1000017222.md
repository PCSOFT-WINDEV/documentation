## Presentación
<a name="presentacion_ELTTEXTE000147"></a>
Se proponen diferentes tipos de replicación:

- **la replicación asistida**. Este modo de replicación se utiliza para definir fácilmente los registros a replicar mediante ReplicEdit. Luego, los registros son replicados por ReplicSynchro. Este modo de replicación utiliza la replicación universal. Se utilizan funciones específicas de WLanguage para manipular las replicaciones.
	La replicación universal asistida está disponible para las aplicaciones iOS y Android. Para obtener más información, consulte [Replicación de datos móviles (Android o iOS)](../WDLang4/1000021071.md). 

- **la replicación universal**. El propósito de la replicación universal es mantener sincronizadas varias bases de datos. Estas bases de datos pueden tener diferentes tipos (HFSQL, Oracle, ....). La replicación universal utiliza un modelo centralizado: todas las bases de datos están sincronizadas con una base de datos maestra. Luego, la base de datos maestra aplica los cambios a las demás bases de datos. Varias funciones de WLanguage se utilizan para implementar la replicación universal.

- Replicación basada en **log**. La replicación basada en registros utiliza registros para mantener actualizadas las bases de datos HFSQL remotas. Se utilizan varias funciones WLanguage para implementar la replicación basada en el log. También se puede utilizar la función [WDReplic](../WDReplic/3520016.md).

- **la replicación entre servidores HFSQL**. Este modo de replicación se utiliza para actualizar los datos encontrados en los servidores HFSQL relacionado 2 by 2.

## Funciones para la gestión de la replicación asistida
<a name="funciones_para_gestion_replicacion_asistida_ELTTEXTE000177"></a>
Las siguientes funciones se utilizan para implementar la replicación universal asistida (replicación creada a partir de ReplicEdit):



|   |   |
| --- | --- |
| [ReplicDeleteSubscription](../WDLang4/1000022268.md) | Elimina la suscripción de una replicación en el servidor de replicación. |
| [ReplicInfo](../WDLang4/1000017197.md) | Devuelve información sobre una replicación. |
| [ReplicInitialize](../WDLang4/1000017195.md) | Inicializa o reinicializa una replicación definida por el editor de replicación (ReplicEdit). |
| [ReplicOpen](../WDLang4/1000017194.md) | Opens a replication in direct mode or with a replication server. |
| [ReplicSynchronize](../WDLang4/1000017196.md) | Sincroniza los datos correspondientes a una réplica. |

| [HSynchronizeReplica](../WDLang4/3044014.md) | Sincroniza la réplica maestra y la réplica del suscriptor.: las operaciones realizadas en una de las réplicas se transfieren a la otra réplica. |

## Funciones utilizadas para manipular la replicación de servidores HFSQL
<a name="funciones_utilizadas_para_manipular_replicacion_servidores_hfsql_ELTTEXTE000426"></a>
Las siguientes funciones se utilizan para implementar una replicación entre 2 servidores HFSQL. Para obtener más información, consulte [Programación de una replicación de servidores HFSQL](../WDLang4/1000020587.md).



|   |   |
| --- | --- |
| [HRSAddConfig](../WDLang4/1000020492.md) | Añade una replicación entre dos servidores HFSQL en el servidor maestro. |
| [HRSDeleteConfig](../WDLang4/1000020493.md) | Elimina una replicación entre dos servidores HFSQL. |
| [HRSExecute](../WDLang4/1000020556.md) | Ejecuta inmediatamente una replicación recurrente entre servidores HFSQL: la replicación se activa antes de la programación. |
| [HRSInfo](../WDLang4/1000020574.md) | Lee la configuración de la replicación de un servidor HFSQL que participa en una o más replicaciones. |
| [HRSInit](../WDLang4/1000020541.md) | Configura un servidor HFSQL para que este servidor sea un servidor maestro o un servidor suscriptor para una replicación entre servidores HFSQL. |
| [HRSListConfig](../WDLang4/1000020497.md) | Enumera las réplicas disponibles en un servidor HFSQL maestro. |
| [HRSModifyConfig](../WDLang4/1000020495.md) | Modifica algunos parámetros de una replicación existente entre dos servidores HFSQL. |
| [HRSWaitForDataProcess](../WDLang4/1000023446.md) | Espera: <br><br>- los datos de replicación en el servidor maestro para ser enviados al servidor del suscriptor.<br><br>- los datos de replicación recibidos en el servidor del suscriptor para ser aplicados en su totalidad.<br><br><br> |

|   |   |
| --- | --- |
| [hRSConfig](../WDLang4/1000020484.md) | El tipo **hRSConfig** permite definir las características avanzadas de una replicación entre 2 servidores HFSQL. |

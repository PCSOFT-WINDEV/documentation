


## HInfoLock (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve información sobre el bloqueo de un archivo de datos, un registro o todos los registros de un archivo de datos.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
LockList is string
// Lists all the locks of the Customer data file
LockList = HInfoLock(Customer)
// Lists all the locks set on the data files of CSConnection
LockList = HInfoLock(CSConnection)
// Lists all the locks set on the Customer.fic data file of CSConnection
LockList = HInfoLock(CSConnection, ".\customer.fic")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Información de bloqueo de un archivo de datos identificado por su nombre

`<Result> = HInfoLock(<Data file> [, <Record number>])`
---

**`<Result>: (Cadena de caracteres)`**



- Cadena vacía ("") si no hay ningún bloqueo o si el archivo de datos utilizado no es un archivo de datos HFSQL Client/Server.

- Información sobre la cerradura.  Esta información está separada por caracteres TAB y tiene el siguiente formato:
	
	```txt
	<Physical name of the data file relative to the database> + TAB + 
	<Record number> + TAB + 
	<User> + TAB + <Name of client computer> + TAB + 
	<IP address of client computer> + TAB + <Name of client application> + TAB + 
	<Type of lock> + TAB + <Date of lock implementation> + TAB + 
	<Number of seconds since start of lock> + CR
	```


	- &lt;número de Record&gt; es set a -1 si la cerradura era set en el archivo de datos.

	- El parámetro &lt;Tipo de bloqueo&gt; puede corresponder a:

		- W: Bloqueo de escritura.

		- RW: Bloqueo de lectura/escritura.

		- W-T: Bloquear en modo de escritura set por una transacción.

		- RW-T: Bloquear la lectura/escritura de set por una transacción.




- El parámetro &lt;Fecha de inicio del bloqueo&gt; tiene el siguiente formato: AAAAMMDDHHmmSS. Esta fecha está en formato UTC (Universal Time Coordinated).




**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado. 
Si este parámetro es una cadena vacía (""), **HInfoLock** manipula el último archivo de datos utilizado por la última función HFSQL (función que empieza por "H").

**`<Record number>: (Entero opcional)`**

Número de la Record para la que se solicita la información de la cerradura. 
Si no se especifica este parámetro, la función **HInfoLock** lista todos los bloqueos del archivo de datos.  


<a name="SYNTAX2"></a>

### Información de bloqueo de un archivo de datos identificado por su conexión

`<Result> = HInfoLock(<Connection> [, <Data file path>] [, <Record number>])`
---

**`<Result>: (Cadena de caracteres)`**

Información sobre la cerradura. Esta información está separada por caracteres TAB y tiene el siguiente formato:

```txt
<Physical name of file relative to the database> + TAB + <Record number> + 
TAB + <User> + TAB + <Name of client computer> + TAB + 
<IP address of client computer> + TAB + <Name of client application> + 
TAB + <Type of lock> + TAB + <Start date of lock> + TAB + 
<Nb of seconds since start of lock> + CR
```


- &lt;número de Record&gt; es set a -1 si la cerradura era set en el archivo de datos.

- El parámetro &lt;Tipo de bloqueo&gt; puede corresponder a:

	- W: Bloqueo de escritura.

	- W-T: Bloquear en modo de escritura set por una transacción.

	- RW: Bloqueo de lectura/escritura.

	- RW-T: Bloquear la lectura/escritura de set por una transacción.




- El parámetro &lt;Fecha de inicio del bloqueo&gt; tiene el siguiente formato: AAAAMMDDHHmmSS.




**`<Connection>: (Cadena de caracteres o variable de tipo Connection)`**

Conexión para la que se solicita la información de la cerradura. Esta conexión corresponde a: 

- el nombre de la conexión definida en el editor de análisis o con las funciones [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- el nombre de una variable de tipo [Conexión](../WDLang4/1514073.md).


Si este parámetro no es un nombre de conexión, **HInfoLock** considera que corresponde al nombre de un archivo de datos.

**`<Data file path>: (Cadena de caracteres opcional)`**

ruta completa del archivo de datos, relativa a la base de datos definida por &lt;Conexión&gt;. 
Si el parámetro &lt;Conexión&gt; corresponde al nombre lógico de un archivo de datos, se ignora. 
Si no se especifica este parámetro, se devuelven todos los bloqueos de set en todos los archivos de datos de la base de datos.

**`<Record number>: (Entero opcional)`**

Número de la Record para la que se solicita la información de la cerradura. 
Si no se especifica este parámetro, la función **HInfoLock** lista todos los bloqueos del archivo de datos.

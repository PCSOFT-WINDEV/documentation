


## &lt;Conexión Variable&gt;.RSListConfig (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Enumera las replicaciones disponibles en un servidor HFSQL maestro o de suscriptores.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Declare the variables
Cnt is Connection
listRS is array of hRSConfig

// Configuring the connection
...

// List of replications
listRS = MyConnection.RSListConfig()
FOR EACH ConfigReplication of listRS
	Trace(ConfigReplication.Server)
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = <Connection>.RSListConfig([<Option>])`
---

**`<Result>: (Array de hRSConfig)`**

Array de variables [hRSConfig](../WDLang4/1000020484.md) que describen las diferentes réplicas definidas en el servidor HFSQL.

**`<Connection>: (Variable de tipo Connection)`**

Nombre del [Conexión](../WDLang4/1514073.md) Variable que describe la conexión con el servidor HFSQL maestro (en el que se definen las réplicas).

**`<Option>: (Constante de tipo Integer)`**

Se usa para listar las replicaciones para las cuales el servidor actual es maestro o suscriptor.: 


|   |   |
| --- | --- |
| *hrsMaster*<br>(Valor predeterminado) | Enumera las réplicas para las que el servidor actual es maestro. |
| *hrsSubscriber* | Enumera las réplicas para las que el servidor actual es suscriptor.<br><br>Observación: En este caso, la [hRSConfig](../WDLang4/1000020484.md) Variable correspondiente a la replicación tiene las siguientes características:<br><br>- Replicación bidireccional: el identificador corresponde al identificador de la replicación en el servidor de suscriptores.<br><br>- Replicación unidireccional: el identificador es set a 0.<br><br>- La propiedad **Server** corresponde al nombre del servidor maestro.<br><br>- Las propiedades **description* y*Scheduling** no están disponibles.<br><br><br> |

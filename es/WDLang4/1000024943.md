


## &lt;Fuente&gt;.State (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite conocer el estado de un registro. Un Record puede ser Active, cruzado o borrado.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF Customer.State() = hStateActive THEN
	// Processes
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Source>.State([<Record number>])`
---

**`<Result>: (Constante de tipo Integer)`**

Estado de Record: 


|   |   |
| --- | --- |
| *hStateActive* | El Record es Active. |
| *hStateCrossed* | La Record está cruzada. |
| *hStateDel* | El Record se ha borrado. |
| *hStateInvalid* | Estado de Record no encontrado (error). |



**`<Source>: (Tipo de fuente especificada)`**

Nombre del archivo de datos utilizado. 

**`<Record number>: (Entero opcional)`**

Número de la buscada Record. Si no se especifica este número (o es igual a 0), **&lt;Fuente&gt;.State** será manipular el actual Record. El actual Record no se lee de nuevo.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Número de Record no especificado
<a name="numero_record_especificado_ELTPARAGRAPHE000248"></a>

Si no se especifica el parámetro &lt;Número de registro&gt;, la función **&lt;Fuente&gt;.State** no lee el registro.

En una aplicación de red, se recomienda leer el registro antes de llamar a la función **&lt;Fuente&gt;.State** para asegurarse de que el registro no ha sido eliminado. Con [&lt;Fuente&gt;.Read](../WDLang4/1000025019.md), por ejemplo:

```wl
FileName.Read(hRecNumCurrent)
```

### Archivos de datos en formato Hyper File 5.5
<a name="archivos_datos_formato_hyper_file_55_ELTPARAGRAPHE000277"></a>

Con el motor de Hyper File 5, la Record debe ser leída de nuevo para que su estado se refresque. De hecho, si no se lee de nuevo el registro después de las funciones [&lt;Fuente&gt;.Delete](../WDLang4/1000025071.md) o [&lt;Fuente&gt;.Cross](../WDLang4/1000025051.md), la función **&lt;Fuente&gt;.State** siempre devuelve *hStateActive*.

Este modo de funcionamiento ya no está disponible en el motor HFSQL Classic. Después de utilizar las funciones [&lt;Fuente&gt;.Delete](../WDLang4/1000025071.md) o [&lt;Fuente&gt;.Cross](../WDLang4/1000025051.md) en el registro en curso, la función **&lt;Fuente&gt;.State** devuelve el estado actual del registro.

### Archivos xBase
<a name="archivos_xbase_ELTPARAGRAPHE000315"></a>

[&lt;Fuente&gt;.Delete](../WDLang4/1000025071.md) y [&lt;Fuente&gt;.Cross](../WDLang4/1000025051.md) tienen el mismo efecto: el Record se cruza pero no se borra. Por lo tanto, la Record puede volver a activarse mediante [&lt;Fuente&gt;.Modify](../WDLang4/1000025038.md). 

En este caso, **&lt;Fuente&gt;.State** devuelve la constante *hStateCrossed* para un Record eliminado (en lugar de *hStateDel* 000).

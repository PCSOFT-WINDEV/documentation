## Presentación
<a name="presentacion_ELTTEXTE000135"></a>
WEBDEV permite crear aplicaciones/sitios que acceden a bases de datos HFSQL Client/Server.

El modo Client/Server permite alojar las bases de datos en un servidor (puede ser diferente del servidor en el que se implementa el sitio).

Este modo de funcionamiento:

- aumenta la seguridad de los datos,

- permite gestionar fácilmente el modo multiusuario,

- simplifica el mantenimiento,

- permite conexiones remotas (por ejemplo, para actividades administrativas de "back office").



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WB-HFSQL-Client-Serveur.gif)


WEBDEV permite:

- crear un sitio que utilice una base de datos HFSQL Client/Server.

- modificar un sitio que utiliza una base de datos HFSQL Classic para que pueda utilizar una base de datos HFSQL Client/Server.

## ¿Por qué cambiar un sitio a HFSQL Client/Server?
<a name="¿por_que_cambiar_sitio_hfsql_clientserver_ELTTEXTE000159"></a>
Las principales ventajas de un sitio en modo HFSQL Client/Server en comparación con uno en modo HFSQL Classic son las siguientes:

- HFSQL Client/Server es más seguro (puede crear un nombre de usuario y una contraseña, y conceder permisos a los usuarios).

- No es necesario gestionar directorios: todos los archivos de la base de datos están en la misma carpeta.

- Las bases de datos Client/Server pueden utilizarse con una conexión a Internet.

- Gestión del modo multiusuario nativo: El rendimiento se optimiza en el modo multiusuario.

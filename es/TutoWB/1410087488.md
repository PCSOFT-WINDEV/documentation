## Cómo mostrar un sitio WEBDEV en modo Sesión
<a name="como_mostrar_sitio_webdev_modo_sesion_ELTTEXTE000233"></a>
En la parte 3, desarrollaremos la parte "Intranet" del sitio. Esta parte del sitio utiliza un sitio WEBDEV en modo Sesión. 

La siguiente imagen presenta cómo se muestra un sitio WEBDEV en modo Sesión:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=PageAffiche_Ctx.gif)



### Sesión WEBDEV
<a name="sesion_webdev_ELTPARAGRAPHE000071"></a>

Cuando el usuario se conecta a un sitio WEBDEV en modo Sesión, se crea automáticamente una sesión en el servidor.

Esta sesión contiene los contextos de página que corresponden a cada página abierta por el usuario.

La sesión termina cuando el usuario sale del navegador o cuando se supera el tiempo de espera de la sesión. Este tiempo de espera se define en el administrador de WEBDEV (pestaña "Configuración", opción "Desconectar usuarios inactivos después de").
<a name="NOTE3_2"></a>


### Contextos de página
<a name="contextos_pagina_ELTPARAGRAPHE000082"></a>

Por cada página que se muestra en el navegador, se crea automáticamente un contexto de página en el servidor. Este contexto de página contiene todos los elementos necesarios para construir la página que ve el usuario final:

- variables globales,

- variables locales, 

- procesos y eventos del servidor,

- conexiones a las bases de datos,

- contextos de archivos de datos, etc.




Los contextos de página permanecen en la memoria en el servidor hasta el final de la sesión WEBDEV.

Cuando se llama a la misma página varias veces:

- Si se llama a la función [PageDisplay](../WDLang2/3058008.md) para mostrar la página, el contexto de la página se elimina y se vuelve a crear.

- Si se llama a la función [PageRefresh](../WDLang2/3058011.md) para mostrar la página, se vuelve a utilizar el mismo contexto de página. 



<a name="NOTE3_3"></a>


### Programación
<a name="programacion_ELTPARAGRAPHE000108"></a>

De forma predeterminada, la sesión WEBDEV y los contextos de página se gestionan automáticamente. No hay nada por programar.

## Presentación
<a name="presentacion_ELTTEXTE000382"></a>
Un sitio puede ser utilizado por diferentes colaboradores con diferentes perfiles. A veces es necesario definir diferentes niveles de acceso según el usuario (cliente, vendedor, administrador, etc.).

Veamos un ejemplo simple: un sitio web de comercio electrónico que permite a los usuarios:

- Ver precios,

- Modificar precios,

- Agregar pedidos,

- Gestionar clientes.


Los accesos son diferentes según el usuario. Por ejemplo:

- los clientes pueden ver la lista de precios y hacer pedidos.

- los vendedores pueden ver la lista de precios, hacer pedidos y crear nuevos clientes.

- los administradores tienen acceso a todas las opciones.




WEBDEV permite administrar estos diferentes niveles de acceso mediante el Groupware Usuario.

Vamos a habilitar el Groupware Usuario en el sitio "Full_WEBDEV_Site" y a configurarlo para limitar el acceso a la página de adición de productos.



- Abra de nuevo el proyecto en el que trabajó en la lección anterior. 

	1. Vaya a la página de inicio de WEBDEV (Ctrl +&lt;).

	2. En la página de inicio, haga clic en "Tutorial", luego en "Parts 3 to 6", haga doble clic en "Full WEBDEV Site - Exercise".

	3. Un cuadro de diálogo le pide que abra el proyecto en el que trabajó en la lección anterior. Puede abrir la copia local o el proyecto original. Seleccione "Abrir la copia local".

### Incluir la sección de clientes en el sitio web
<a name="incluir_seccion_clientes_sitio_web_ELTPARAGRAPHE000122"></a>

En este ejemplo, vamos a agregar el enlace de inicio de sesión a la página "PAGE_List_of_products".

El enlace de inicio de sesión se proporciona como un modelo de controles.|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | Un modelo de controles es una página específica que contiene varios controles. Esta página puede contener todo tipo de controles. Un modelo de controles es un archivo con extensión "WDT".<br><br>La principal ventaja de un modelo de controles es que se puede reutilizar fácilmente. Un modelo de controles puede reutilizarse en cualquier página del proyecto. Si realiza cambios en el modelo de controles, estos solo deben hacerse una vez; WEBDEV los aplicará automáticamente en todas las páginas que utilicen el modelo de controles.<br><br>Los modelos de controles se pueden sobrescribir: por ejemplo, se puede añadir código y los controles se pueden mover en la página que utiliza el modelo de controles. Los controles también se pueden modificar. |







- Para agregar el enlace de inicio de sesión:

	1. Abra la página "PAGE_List_of_products" en el editor.

	2. Vaya al panel "Explorador de proyectos", y en la carpeta "Componentes internos", expanda "WDGPU_WB_CNT". Este componente interno se incluyó en el proyecto cuando se habilitó el Groupware. Contiene todos los elementos necesarios, especialmente el modelo de controles que permite iniciar sesión.

	3. A continuación, despliegue "Modelo de controles WEBDEV".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5_Groupware%20Utilisateur%20-%20HC%20N%B0002.jpg)


	4. Seleccione el modelo de controles TPLC_GPU_WB_Login y arrástrelo a PAGE_List_of_products. Suelte el modelo de controles arriba de la barra de menú.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5_Groupware%20Utilisateur%20-%20HC%20N%B0003.jpg&type=thumb)

### Crear usuarios y grupos
<a name="crear_usuarios_grupos_ELTPARAGRAPHE000171"></a>

Para configurar el Groupware Usuario, vamos a crear un grupo de gestión de productos. Este grupo incluirá a los usuarios que pueden modificar y agregar productos en el sitio.

A continuación, crearemos un usuario llamado Alison y lo asociaremos al grupo "Product Management".



- Para crear un nuevo grupo de usuarios:

	1. Haga clic en la pestaña "Grupos". 

	2. Haga clic en "Agregar un grupo". La ventana para introducir un nuevo grupo se abre.

	3. Escriba el nombre del grupo: "Product Management".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5_Groupware%20Utilisateur%20-%20HC%20N%B0004.jpg&type=thumb)


	4. Haga clic en "Guardar". El grupo "Product Management" aparece en la lista de grupos definidos en el Groupware Usuario.




- Para crear un usuario:

	1. Haga clic en la pestaña "Usuarios". 

	2. Haga clic en "Agregar un usuario". La ventana para introducir un nuevo usuario se abre.

	3. Introduzca la siguiente información:

		- Nombre de usuario : Alison

		- Nombre: Alison

		- Seleccione la opción "Forzar la siguiente contraseña" e introduzca la contraseña: Alison. 
						Observación: También es posible dejar que los usuarios elijan sus contraseñas la primera vez que inicien sesión.
						|   |   |   |

En este caso, el supervisor podrá modificar usuarios, grupos y permisos.<br><br><br> |


<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5_Groupware%20Utilisateur%20-%20HC%20N%B0005.jpg&type=thumb)







- Para asociar a "Alison" al grupo "Product Management":

	1. Seleccione "Alison" en la página.

	2. Marque la casilla que se encuentra al principio de la fila: los botones "Grupos de usuarios" y "Eliminar usuario" aparecen.

	4. En la ventana emergente que aparece, seleccione el grupo "Product Management" y haga clic en "OK".

### Definición de permisos
<a name="definicion_permisos_ELTPARAGRAPHE000220"></a>

Ahora definiremos los permisos de acceso al menú que permite agregar un producto.

**Por motivos de seguridad, le recomendamos denegar el acceso predeterminado y permitir el acceso solo a los grupos seleccionados**.

- Vamos a definir los permisos de los usuarios que no han iniciado sesión (es decir, no identificados por el Groupware). Estos permisos se utilizarán de forma predeterminada al iniciar el sitio, siempre y cuando el usuario no haya iniciado sesión.




- Defina los permisos:

	1. Para que ningún usuario acceda a la página que permite agregar productos, seleccione el usuario "Visitor (not connected)". Todo usuario que no haya iniciado sesión se asociará automáticamente a este usuario.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5_Groupware%20Utilisateur%20-%20HC%20N%B0006.jpg&type=thumb)


	2. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5_Groupware%20Utilisateur%20-%20HC%20N%B0006%201.jpg) para definir los permisos.

3. La página que aparece permite seleccionar cada página, modelo de páginas o reporte de la aplicación.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5_Groupware%20Utilisateur%20-%20HC%20N%B0007.jpg&type=thumb)




- En nuestro caso, el enlace "Add a product" se encuentra en el modelo de páginas "PAGETPL_Menu":

	1. Seleccione el modelo de páginas "PAGETPL_Menu" de la lista. Los permisos definidos en el modelo de páginas se aplicarán a todas las páginas que utilicen el modelo.

	2. Los nombres de los diferentes controles de la página aparecen a la derecha.

	3. En "Opción de menú Option_2", seleccione "En gris" en el cuadro combinado de "Estado".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P5_Groupware%20Utilisateur%20-%20HC%20N%B0008.jpg&type=thumb)


	4. Cierre el navegador.

## Desactivar el Groupware Usuario
<a name="desactivar_groupware_usuario_ELTTEXTE000628"></a>
**No seguiremos utilizando el Groupware Usuario en este tutorial**. Puede desactivar la gestión de permisos de usuarios en este proyecto y eliminar el modelo de controles que utilizamos:

1. En la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Groupware Usuario".

2. La ventana de configuración de la plataforma se abre. 

3. En "Groupware Usuario", haga clic en "Cambiar". 

4. En la ventana que aparece, haga clic en "Inactivo".

5. Valide.

6. Si es necesario, abra la página "PAGE_List_of_products". 

7. Elimine el modelo de controles "Log in" que agregamos anteriormente. 

8. Guarde y cierre la página. 

9. Recompile el proyecto: en la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Recompilar y sincronizar" y seleccione "Recompilar proyecto y generar páginas HTML". 

10. En el panel "Explorador de proyectos", despliegue "Componentes internos" y seleccione "WDGPU_WB_CNT". Haga clic derecho para abrir el menú contextual y seleccione "Retirar y eliminar del disco". 

11. Valide el mensaje que aparece. 


El Groupware Usuario se elimina del proyecto.

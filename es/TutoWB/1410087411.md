## Presentación
<a name="presentacion_ELTTEXTE000239"></a>
Al diseñar un sitio web, es posible que tenga que manipular datos. Para almacenar los datos, debe crear una "base de datos".

En un proyecto WEBDEV que manipule datos lo primero que se debe hacer es crear un "análisis".

Un "Análisis" contiene la descripción de archivos de datos (o tablas). Los datos de la aplicación se guardan en estos archivos de datos.

Cuando se ejecute el sitio web, estas descripciones se utilizarán para crear la base de datos y/o los archivos de datos. Los datos se almacenarán en esta base de datos o en estos archivos.|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | WEBDEV incluye varias herramientas de mantenimiento de bases de datos HFSQL. Se puede acceder a ellas desde el Centro de Control HFSQL. |




WEBDEV es compatible con la mayoría de formatos de bases de datos (casi todos). Los formatos más comunes son:

- HFSQL, un sistema de base de datos proporcionado con WEBDEV. La base de datos HFSQL está disponible en modo Classic o Client/Server.

- AS/400, Access, Sybase, Informix...

- Oracle, SQL Server, MySQL, etc.

- Cualquier base de datos accesible en lenguaje SQL en Windows.

- Texto (archivos ASCII).




Existen varios métodos para acceder a los datos:

- Conector Nativo (también llamado Acceso Nativo),

- Acceso OLE DB, 

- Acceso ODBC directo,

- Acceso ODBC a través de OLE DB.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## HFSQL
<a name="hfsql_ELTTEXTE000281"></a>
HFSQL es una base de datos potente, rápida y segura.

HFSQL funciona en Windows y Linux, en dispositivos móviles (iOS, Android), en redes de cualquier tamaño y tipo, y gestiona cientos de accesos simultáneos automáticamente.

HFSQL está disponible en modo Classic y Client/Server.

HFSQL se puede distribuir de forma gratuita con los sitios WEBDEV.

HFSQL propone todas las funcionalidades de una base de datos, especialmente:

- [registros](../WDLang4/3044184.md),

- [transacciones](../WDLang4/3044335.md),

- [replicación](../WDLang4/3044266.md),

- [desencadenadores](../WDLang4/3044272.md).




En las diferentes partes de este tutorial, usaremos una base de datos HFSQL Classic, y luego una base de datos HFSQL Client/Server.

## Modos de acceso a las bases de datos
<a name="modos_acceso_las_bases_datos_ELTTEXTE000305"></a>


### Conector Nativo (Acceso Nativo)
<a name="conector_nativo_acceso_nativo_ELTPARAGRAPHE000086"></a>

Un Conector Nativo manipula un formato de base de datos de forma directa y exclusiva. Este tipo de acceso optimizado se desarrolla específicamente para cada formato de base de datos.

En WEBDEV, hay un Conector Nativo para las siguientes bases de datos:

- HFSQL Classic o Client/Server (estándar).

- Access (estándar).

- SQLite (estándar).

- Oracle (opcional).

- AS/400 (opcional).

- SQL Server (opcional).

- Sybase (opcional).

- Informix (opcional).

- DB2 (opcional).

- Progress (opcional).

- MySQL (opcional y gratuito).

- PostgreSQL (opcional y gratuito).

- MariaDB (opcional y gratuito).


Si tiene preguntas sobre estos Conectores Nativos, póngase en contacto con nuestro departamento de ventas.

Las funciones **SQL\*** y **HRead\*** de WLanguage se pueden utilizar con este tipo de acceso. El código es portable e independiente de la base de datos.
<a name="NOTE4_2"></a>


### Acceso ODBC directo
<a name="acceso_odbc_directo_ELTPARAGRAPHE000113"></a>

El acceso a través de ODBC utiliza un estándar de acceso de múltiples bases de datos. La capa ODBC debe estar instalada en su equipo. En la mayoría de los casos, esta capa ya está instalada en las últimas versiones de Windows. Esto se puede comprobar en el panel de control de Windows seleccionando "Origen de datos ODBC" (o "Administrador de origen de datos ODBC" según la versión de Windows utilizada).

Atención: puede que no se pueda acceder a todas las bases de datos a través de este método. Si desea utilizar este tipo de acceso, compruebe si existe un controlador ODBC e instálelo si es necesario.

Solo las funciones **SQL\*** de WLanguage se pueden utilizar con este tipo de acceso.
<a name="NOTE4_3"></a>


### Acceso OLE DB
<a name="acceso_ole_ELTPARAGRAPHE000124"></a>

OLE DB utiliza un estándar de acceso a varias bases de datos. Este tipo de acceso se basa en MDAC (Microsoft Data Access Component).|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=avertissement.png) | ¡Atención! | Si utiliza un acceso OLE DB, MDAC debe estar instalado en los equipos de los usuarios (versión 2.6 o posterior). |



Puede que no se pueda acceder a todas las bases de datos a través de este método. Si desea utilizar este tipo de acceso, compruebe si existe un controlador OLE DB.

Las funciones **SQL\*** y **HRead\*** de WLanguage se pueden utilizar con este tipo de acceso.
<a name="NOTE4_4"></a>


### Acceso ODBC a través de OLE DB
<a name="acceso_odbc_traves_ole_ELTPARAGRAPHE000138"></a>

En resumen, es una "mezcla" de OLE DB y ODBC. Este es el método más "pesado" y menos eficiente en términos de rendimiento. No debe utilizarse en bases de datos pequeñas. 

Las funciones **SQL\*** y **HRead\*** de WLanguage se pueden utilizar con este tipo de acceso.

## Proyecto con análisis
<a name="proyecto_con_analisis_ELTTEXTE000365"></a>
En esta parte del tutorial, vamos a crear y a asociar un proyecto a una base de datos HFSQL.

Veremos algunos elementos clave para desarrollar un sitio con archivos de datos:

- Crear un proyecto WEBDEV.

- Definir los archivos de datos del sitio.




Más adelante en este tutorial, nos centraremos en el desarrollo de los elementos de un sitio web con datos (páginas, reportes, etc.). Este sitio web se desarrollará desde cero, desde la interfaz hasta la distribución final. Abordaremos los puntos principales para desarrollar un sitio.

En esta parte, también crearemos una base de datos para administrar pedidos.

El mismo tipo de base de datos se utilizará en la parte 3 de este tutorial para desarrollar un sitio completo. Utilizaremos HFSQL Classic, una base de datos gratuita proporcionada con WEBDEV. Más adelante veremos cómo utilizar una base de datos HFSQL Client/Server.

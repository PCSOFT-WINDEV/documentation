## Presentación
<a name="presentacion_ELTTEXTE000519"></a>
Una vez desarrollado el sitio WEBDEV, se debe implementar en un servidor para que los usuarios puedan acceder a él.

Vamos a presentar los diferentes pasos necesarios para implementar un sitio WEBDEV.|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | Esta sección presenta la implementación en un servidor Windows. Si utiliza un servidor Linux, consulte la documentación sobre WEBDEV Application Server para Linux. |



Existen varios métodos para implementar un sitio WEBDEV (en modo Sesión o AWP):

- Implementación remota desde el equipo de desarrollo (mediante HTTP o FTP).

- Implementación en [PCSCloud](https://pcscloud.net/UK/cloud_development.awp) (servicio en la nube de PC SOFT).

- Implementación remota desde un equipo de administración (vía HTTP o FTP) utilizando un "paquete de implementación".

- Implementación a través del servicio de alojamiento de pruebas de PC SOFT.

- Implementación a través de un medio físico (CD, etc.). Esta implementación puede ser autónoma: en este caso, la instalación también instala un servidor web y un WEBDEV Application Server limitado.

- Implementación a través de una imagen Docker del sitio.

<a name="NOTE3_1"></a>


## Configuración necesaria
<a name="configuracion_necesaria_ELTTEXTE000561"></a>
Para la implementación en el servidor, se deben instalar y configurar los siguientes elementos:

- Un servidor web,

- Un WEBDEV Application Server (una versión de 10 conexiones se proporciona con WEBDEV).


Observación: Para una instalación remota a través de FTP, también se debe instalar un servidor FTP en el servidor.  

Al instalar WEBDEV, puede instalar la versión de prueba de WEBDEV Application Server. Esto le permite probar la implementación de sus sitios WEBDEV dinámicos (en modo Sesión o AWP).

Para simplificar las operaciones de implementación y probar directamente la implementación del sitio web, vamos a:

- Instalar "WEBDEV Application Server - 10 conexiones" en un equipo Windows. Este servidor de aplicaciones debe instalarse en un equipo diferente al equipo de desarrollo. WEBDEV no debe estar instalado en el mismo equipo.

- Implementar el sitio de forma remota (mediante HTTP).

3. En la siguiente etapa, puede definir la configuración avanzada de WEBDEV Application Server. Conserve la opción "Utilizar parámetros predeterminados".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P6_Install%20Serveur%2010%20Connexions%20-%20HC%20N%B0003.jpg)

	Puede crear una nueva cuenta Windows o usar una cuenta Windows existente. 
	Seleccione una opción y pase a la siguiente etapa.

			Seleccione una opción y pase a la siguiente etapa. 




6. La siguiente etapa permite gestionar el alojamiento de sitios SaaS. La activación de SaaS instala automáticamente un sitio y un Webservice para administrar los sitios SaaS. Para obtener más información, consulte [SaaS](../WDLang3/9500213.md). Valide la etapa.

9. Valide las diferentes etapas de la instalación.

10. Si el equipo contiene varios servidores virtuales, seleccione los servidores web en los que debe instalarse WEBDEV Application Server. Puede conservar el sitio web predeterminado. Valide.

11. El asistente presenta diferentes opciones: seleccione las opciones deseadas y conserve la opción "Iniciar el administrador de WEBDEV". Valide.

 En este ejemplo crearemos una cuenta Windows. Introduzca el nombre de usuario y la contraseña (también puede generar la contraseña. En ese caso, no olvide guardarla).<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P6_Configuration%20CCH%E9bergement%20-%20HC%20N%B0005.jpg&type=thumb)

## Los diferentes modos de implementación
<a name="los_diferentes_modos_implementacion_ELTTEXTE000735"></a>
Ahora que el servidor web está configurado, podemos proceder a la implementación del sitio. Hay varios métodos disponibles. La elección del método depende principalmente de las limitaciones técnicas (posibilidad de utilizar un servidor FTP, acceso al servidor, etc.). 
Un sitio WEBDEV dinámico puede implementarse de acuerdo a uno los siguientes métodos:

- **Implementación remota desde el equipo de desarrollo** (mediante HTTP o FTP). Vamos a implementar el sitio utilizando este método mediante HTTP.
	Observación: la implementación mediante FTP requiere un servidor FTP. 

- **Implementación en PCSCloud**.

- **Implementación remota desde un equipo de administración** (mediante HTTP o FTP) utilizando un "paquete de implementación". Este modo de implementación es similar a la implementación remota directa. Única diferencia: la implementación no se realiza desde el entorno de desarrollo de WEBDEV. En este modo:

	- el desarrollador genera un paquete de implementación.

	- el administrador del sitio (que puede ser una persona distinta del desarrollador) implementa el paquete en el servidor remoto mediante WDDeploy. WDDeploy es una herramienta redistribuible proporcionada con WEBDEV.




- **Implementación a través de un medio físico** (CD, etc.). La implantación a través de un medio físico crea un soporte de instalación (como una aplicación) que debe ejecutarse directamente en el servidor. Este modo de implementación solo está disponible en Windows. 

- **Implementación a través del servicio de alojamiento de pruebas de PC SOFT**.

- **Implementación a través de una imagen Docker del sitio**.




Veamos la implementación mediante HTTP.

## Directorios en el equipo de desarrollo y en el servidor de implementación
<a name="directorios_equipo_desarrollo_servidor_implementacion_ELTTEXTE000801"></a>
Los archivos del sitio se almacenan en directorios diferentes en el equipo de desarrollo y en el servidor de implementación. La siguiente tabla presenta los directorios predeterminados definidos por WEBDEV.


|   |   |
| --- | --- |
| Directorio en el equipo de desarrollo | Directorio en el servidor de implementación |
| &lt;Nombre del proyecto&gt;\\&lt;Nombre del proyecto&gt;_WEB | &lt;sitio&gt;\\&lt;nombre del sitio&gt;\\&lt;nombre del sitio en mayúsculas&gt;_WEB |
| &lt;Nombre del proyecto&gt;\\Exe | Para un sitio:<br><br>- archivos de datos (.fic, .ndx, .mmo, .ftx): &lt;datos&gt;\\&lt;nombre del sitio&gt;<br><br>- otros archivos: &lt;sitio&gt;\\&lt;nombre del sitio&gt;<br><br><br>Para un Webservice: <br><br>- archivos de datos (.fic, .ndx, .mmo, .ftx): &lt;datos&gt;\\&lt;nombre del sitio&gt;<br><br>- otros archivos: &lt;Webservice&gt;\\&lt;nombre del sitio&gt;<br><br><br> |


donde: 

- &lt;sitio&gt;, &lt;datos&gt; y &lt;Webservice&gt; son los directorios definidos al crear la cuenta de alojamiento (véase más arriba).

- &lt;nombre del sitio&gt; es el nombre del sitio. 


Los directorios de los archivos pueden definirse en el asistente de implementación.

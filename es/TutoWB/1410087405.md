## Principio de Navegador/Servidor
<a name="principio_navegadorservidor_ELTTEXTE000371"></a>


### ¿Cómo funciona?
<a name="¿como_funciona_ELTPARAGRAPHE000024"></a>

Un sitio de Internet o intranet funciona de la siguiente manera:

- El cliente (usuario de Internet) utiliza un navegador para acceder al sitio.

- El navegador envía una solicitud al servidor que aloja el sitio. En esta solicitud, se indica la página que debe mostrarse y los diferentes parámetros que permiten al servidor construir la página correspondiente.

- El servidor recibe la solicitud, la procesa y devuelve la página "HTML" correspondiente. HTML (HyperText Markup Language) es el lenguaje utilizado por todos los navegadores para mostrar páginas Web.




Por lo tanto, hay dos tipos de eventos:

- Eventos que se ejecutan a nivel del navegador, en el equipo del usuario.

- Eventos que se ejecutan a nivel del servidor.


El código que se ejecuta a nivel del navegador se llama código JavaScript. Los navegadores solo pueden ejecutar código JavaScript.


### ¿Y en WEBDEV?
<a name="webdev_ELTPARAGRAPHE000040"></a>

Con WEBDEV, todo se desarrolla:

- en WYSIWYG ("What You See Is What You Get") en el editor: sus páginas son visualmente idénticas cuando se crean y en tiempo de ejecución.

- en código WLanguage.




WEBDEV convierte la página creada en el editor en una página HTML que puede ser leída por los navegadores.

El código Servidor se ejecuta en WLanguage.

El código Navegador se convierte en JavaScript. 

Para crear un sitio con WEBDEV, se requiere un solo lenguaje: WLanguage.

Sin embargo, hay dos tipos de código disponibles: código Servidor y código Navegador.

¿Por qué esta distinción entre servidor y navegador? Por motivos de rendimiento. La conexión a Internet se interpone entre el navegador y el servidor, lo que implica tiempos de respuesta, latencia, etc. Algunas operaciones sencillas pueden realizarse directamente en el navegador, sin volver al servidor.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Navigation_internet.gif)

En nuestro ejemplo, todas las verificaciones de entrada se realizan en código Navegador (código verde). Por ejemplo, el control EDT_LastName no debe estar vacío. El código correspondiente es el siguiente:

```wl
// Check whether the "EDT_LastName" control is empty
// (excluding spaces and punctuation)
IF EDT_LastName ~= "" THEN
	// If the control is empty, display an error message to the user
	Error("Enter your last name")
	// The "EDT_LastName" control gains focus again (without running the rest of the code)
	SetFocusAndReturnToUserInput(EDT_LastName)
END
```

Esta verificación se realiza en código Navegador porque no hay necesidad de volver al servidor para comprobar que los controles están completos.

Esto evita idas y vueltas inútiles y reduce los tiempos de espera: la navegación por el sitio web es mucho más fluida.

Una vez que se ejecuta el código Navegador, la página envía los valores introducidos al servidor. Luego, el servidor ejecuta el código Servidor. En el código Servidor, se puede procesar la información recibida.

En nuestro ejemplo, la información recibida se agrega a la base de datos mediante el siguiente código:

```wl
// Reset the customer structure
Customer.Reset()

// Retrieve the values of controls in the customer structure directly
MyPage.ToFile()

// Add the customer to the database
Customer.Add()
```

Esta operación no se puede realizar en código Navegador porque la base de datos es común a todos los usuarios del sitio y por lo tanto se encuentra en el servidor.

## Intranet/Extranet/Internet
<a name="intranetextranetinternet_ELTTEXTE000461"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000158"></a>

Un **sitio Intranet o Extranet** suele considerarse como una aplicación de gestión en modo Web, es decir, una aplicación que se ejecuta en un navegador.

Esta aplicación Web puede contener:

- funcionalidades de la empresa destinadas a usuarios específicos,

- procesos que deben ser seguros: no cualquiera debe tener acceso a la aplicación.


Se puede acceder a la aplicación Web:

- desde la red de una empresa, es decir, un **sitio Intranet**.

- desde Internet, es decir, un **sitio extranet**.


En ambos casos, la aplicación Web está asegurada con nombres de usuario, contraseñas, permisos, etc.

Un **sitio de Internet** es un sitio destinado al uso "público" (usuarios particulares o profesionales).
Un sitio de Internet debe ser fácil de encontrar en la Web. Por ejemplo: sitio de presentación, sitio de comercio electrónico, etc.

Para atraer más usuarios al sitio web, debe ser posicionado en los buscadores. Para que los buscadores posicionen cada página correctamente existe una restricción: se debe poder acceder a las páginas del sitio en todo momento. Pero esta restricción también representa una garantía de simplicidad para los usuarios de Internet: pueden copiar/pegar fácilmente el enlace de una página y utilizarlo siempre que lo necesiten.


### ¿Y en WEBDEV? (Sesión/AWP, PHP, Estático)
<a name="webdev_sesionawp_php_estatico_ELTPARAGRAPHE000180"></a>En WEBDEV, **para crear un sitio de Intranet o Extranet**, el modo "Sesión" es el más adecuado porque incluye las siguientes características: seguridad integrada y contextos automáticos.
El modo Sesión incluye sesiones automáticas. El identificador de la sesión se incluye en la URL. La dirección de las páginas depende de este identificador que cambia en cada conexión.
Inconveniente: Los motores de búsqueda no pueden indexar este sitio.

En WEBDEV, **para crear un sitio de Internet**, puede elegir uno de los siguientes modos:

- El modo AWP (Active WEBDEV Page).

- El modo de generación PHP.

- El modo estático si su sitio contiene solo páginas definidas previamente (sin base de datos).

## Método para crear un sitio
<a name="metodo_para_crear_sitio_ELTTEXTE000533"></a>
Hemos visto los principales conceptos de la Web, ahora veamos cómo crear un sitio WEBDEV.

Para crear un sitio con WEBDEV, utilice el siguiente método:

- Creación de la maqueta del sitio:

	- Maqueta de las páginas (experiencia de usuario).

	- Definición de los estilos CSS (si es necesario).

	- Diseño de la estructura de la base de datos.




- Desarrollo en WEBDEV:

	- Creación de un proyecto que agrupe todos los elementos del sitio.

	- Creación de los modelos de página que definirán el estilo del sitio. Estos modelos se basan en la maqueta de las páginas, estilos CSS y estilos WEBDEV.

	- Definición de la base de datos (Análisis), que puede reutilizar una estructura existente.

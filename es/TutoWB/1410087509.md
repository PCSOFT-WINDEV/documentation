## Importar elementos
<a name="importar_elementos_ELTTEXTE000289"></a>
Puede importar elementos WEBDEV existentes en su proyecto. Puede importar todo tipo de elementos WEBDEV:

- páginas, reportes,

- clases, componentes,

- procedimientos, etc 

- etc.




- Para importar elementos existentes al proyecto actual:

	1. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Importar" y seleccione "Elementos WEBDEV y dependencias...".

	2. Haga clic en "..." y seleccione el directorio que contiene los elementos WEBDEV a importar.

	3. Valide. WEBDEV muestra los elementos ubicados en el directorio especificado que se pueden importar (los subdirectorios se ignoran).<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P9_Import%20Export%20-%20HC%20N%B0002.jpg&type=thumb)


	4. Seleccione los elementos a importar y valide. Los elementos (y todos sus archivos: imágenes, etc.) se importan al proyecto.

| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | El botón "Calcular" (en la ventana de importación) calcula el tamaño de los elementos seleccionados y sus dependencias. |

## Exportar elementos
<a name="exportar_elementos_ELTTEXTE000331"></a>
También puede exportar elementos de su proyecto a otro directorio. Estos elementos podrán reutilizarse en otros proyectos.|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=avertissement.png) | ¡Atención! | Cuando se exporta un elemento no solo se hace una copia. Una exportación es el único método seguro para transferir los elementos de un proyecto con sus dependencias (imágenes, íconos, etc.). Es una solución sencilla para enviar páginas y sus dependencias por correo electrónico, por ejemplo. |







- Para exportar elementos del proyecto:

	1. En la pestaña "Inicio", en el grupo "General", despliegue "Guardar" y seleccione "Exportar .. A un directorio...".

	2. En la ventana que aparece, seleccione los elementos del proyecto que desea exportar.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P9_Import%20Export%20-%20HC%20N%B0001.jpg&type=thumb)


	3. Especifique el nombre del directorio de destino (o selecciónelo con el botón "...").

4. Valide. Los elementos se exportan al directorio especificado. Estos elementos siguen estando disponibles en el proyecto actual.

### Importar un proyecto WINDEV: Webificación de un proyecto
<a name="importar_proyecto_windev_webificacion_proyecto_ELTPARAGRAPHE000094"></a>

A menudo, los desarrolladores necesitan transformar aplicaciones Windows (o partes de aplicaciones) en sitios web dinámicos.

El asistente de webificación de WEBDEV le permite: 

- convertir las ventanas en páginas,

- adaptar el código WLanguage. 




Para probar la webificación de una aplicación WINDEV, utilizaremos un proyecto WINDEV. |   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | Una aplicación WINDEV solo se puede webificar:<br><br>- utilizando una versión de WINDEV 28 que incluye WEBDEV 28.<br><br>- utilizando WEBDEV 28.<br><br><br> |





- Para convertir un proyecto WINDEV existente en un sitio web:

	1. Vaya a la página de inicio de WEBDEV (Ctrl +&lt;). 

	2. En la página de inicio, haga clic en "Tutorial", luego en "Part 9 - Lesson 9.3 - Import/Export", haga doble clic en "Webify a WINDEV project - Exercise".

	3. Hay dos opciones disponibles en función del IDE utilizado: 

		- **Si está utilizando WINDEV**, en la pestaña "Proyecto", en el grupo "Webificación", despliegue "Webificar" y seleccione "Webificar". 

		- **Si está utilizando WEBDEV**, el asistente de webificación se abre e indica que se creará una configuración de proyecto WEBDEV. Pase a la etapa siguiente del asistente. 




4. El asistente muestra los diferentes elementos a convertir. En nuestro caso, seleccione todos los elementos. 
			|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | Para webificar un proyecto WINDEV, se recomienda empezar convirtiendo una o dos ventanas. Puede convertir las otras ventanas más tarde. |





5. Pase a la etapa siguiente. 

6. El asistente muestra los elementos que se pueden integrar en el sitio. En nuestro ejemplo, conserve el reporte. Conserve las opciones predeterminadas y pase a la siguiente etapa. 

7. El asistente ha terminado. Haga clic en "Finalizar". 

8. La auditoría del proyecto para la conversión de páginas inicia automáticamente. Esta auditoría permite corregir los errores de compilación sugiriendo algunas soluciones. Para completar la webificación de la aplicación, se deben corregir todos los errores. Algunos errores pueden estar relacionados con procesos o características que no están disponibles en la web, funciones WLanguage no disponibles, etc. En nuestro ejemplo, la auditoría muestra los siguientes problemas:  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Webisation.jpg&type=thumb)




- Vamos a examinar los diferentes problemas uno por uno: 

	1. La aplicación WINDEV mostraba un reporte en modo "Visor de reportes". Esta opción no está disponible en sitios WEBDEV. Se debe modificar el código para crear un archivo PDF. 

		- Haga doble clic en "Clic (servidor)" junto a "Errores". El código de clic que contiene el error se abre.

		- En el encabezado del evento "Clic en BTN_Print_quote", haga clic en "AJAX activado" **para desactivar el modo Ajax**. 

		- Reemplace el código por: 
						
			```wl
			iDestination(iGenericPDF)
			RPT_OrderForm.Print()
			FileDisplay(iLastFile, typeMimePDF, iLastFile)
			fDelete(iLastFile)
			```


		- Actualice la ventana de auditoría: todos los errores de vista previa de impresión desaparecieron. 




2. El siguiente problema está relacionado con procesos que no son compatibles con sitios web. Simplemente compruebe el comportamiento del elemento y mueva o elimine el código WLanguage correspondiente, si es necesario. 

3. El último problema detectado está relacionado con el uso de la función [fSelect](../WDLang1/3036053.md). Los selectores de archivos no pueden utilizarse en sitios web. Es preferible utilizar un control Upload. Ya hemos visto este tipo de control en la parte 3 (en el párrafo ["Gestión de la imagen del producto"](../TutoWB/1410087490.md)). 




Este ejemplo nos permitió ver la facilidad y la ayuda que proporciona WEBDEV para convertir un proyecto WINDEV en un sitio web. 

Observación: En algunos casos, es posible que tenga que desarrollar nuevas funcionalidades para adaptar el comportamiento de la aplicación a sitios web. Por ejemplo, si la aplicación utiliza un control Procesador de texto, puede ser necesario utilizar un control HTML y adaptar el análisis para gestionar el formato HTML.

Para obtener más información sobre la webificación de un proyecto WINDEV, consulte [Convierta sus aplicaciones WINDEV en sitios web](../Editeurs/9500225.md).

### Importar una página HTML
<a name="importar_una_pagina_html_ELTPARAGRAPHE000167"></a>

Puede importar fácilmente páginas de otros sitios web, incluso si no han sido desarrolladas con WEBDEV.

WEBDEV le ofrece la posibilidad de importar páginas HTML en su proyecto. Cada página importada se convierte en un elemento del sitio WEBDEV. Puede editar la página, si es necesario.|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | La importación de páginas HTML puede ser de gran ayuda para los desarrolladores de sitios WEBDEV. Sin embargo, el objetivo de esta funcionalidad no es "copiar" sitios web.<br><br>Tenga en cuenta que puede haber algunas diferencias entre la página HTML de origen y el resultado de la importación. |







- Para importar una página HTML a un proyecto WEBDEV:

	1. Abra el proyecto WEBDEV (por ejemplo: un proyecto existente o un nuevo proyecto).

	2. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Importar" y seleccione "Una página HTML...". El asistente de importación de páginas HTML se abre.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P9_Import%20Export%20-%20HC%20N%B0003.jpg&type=thumb)
Observación: En una versión de WEBDEV integrada en WINDEV, esta opción es ligeramente diferente: en la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Importar", seleccione "De WEBDEV", y luego seleccione "Una página HTML"). 

	3. Especifique la ubicación de la página HTML. Esta página puede ser un archivo en su equipo (botón "...") o una dirección de Internet (por ejemplo: http://www.misitiowebdev.com/mapping.html). La página HTML se muestra en el cuadro "Vista previa de la página".

	4. Valide. Una nueva página WEBDEV se crea.

	5. Guarde la página. La página se agrega automáticamente a la lista de elementos del proyecto.

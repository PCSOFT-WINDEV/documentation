## Protección de datos y páginas mediante TLS y SSL
<a name="proteccion_datos_paginas_mediante_tls_ssl_ELTTEXTE000202"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000024"></a>

De forma predeterminada, los datos intercambiados entre el equipo del usuario final y el servidor web no están protegidos. Estos datos no se cifran mientras se transfieren por la red.

Existen varios sistemas para proteger los datos. Un sistema común consiste en utilizar los protocolos TLS (Transport Layer Security) y SSL (Secure Socket Layer). 
Los datos no se intercambian a través del protocolo HTTP, sino a través del protocolo HTTPS (por ejemplo: https://clientes.misitiowebdev.com/clientes).
<a name="NOTE2_2"></a>


### Implementación de transacciones seguras a través del protocolo TLS y SSL
<a name="implementacion_transacciones_seguras_traves_del_protocolo_tls_ssl_ELTPARAGRAPHE000034"></a>

Para implementar transacciones seguras a través de los protocolos TLS y SSL, debe instalar un certificado en el servidor web y configurar este último.

Se pueden utilizar dos métodos diferentes para obtener un certificado:

1. Comprarlo a una autoridad certificada.

2. Generar un certificado autofirmado.




Para obtener más información sobre los certificados, consulte [Crear un certificado SSL que se pueda utilizar con SocketCreateSSL](../WDLang3/3070021.md).
<a name="NOTE2_3"></a>


### Transacciones seguras mediante los protocolos TLS y SSL en un sitio WEBDEV
<a name="transacciones_seguras_mediante_los_protocolos_tls_ssl_sitio_webdev_ELTPARAGRAPHE000051"></a>

El modo seguro se implementa cuando una página debe mostrar datos de manera segura (por ejemplo, cuando se introduce información de una tarjeta de crédito).

Solo tiene que llamar a la función [SSLActive](../WDLang2/3012014.md) en el código Navegador del botón que abre la página.

En cuanto se abra la página segura, todas las acciones que se realicen estarán en modo seguro (es decir, cifradas), independientemente de los objetos que se utilicen (controles Enlace, Tabla, Looper, etc.).

Para obtener más información, consulte la ayuda sobre la función [SSLActive](../WDLang2/3012014.md).

## Pago seguro
<a name="pago_seguro_ELTTEXTE000238"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000072"></a>

La mayoría de sitios de comercio electrónico que permiten a los usuarios realizar una compra en línea utilizan un sistema de pago en línea con tarjeta de crédito.

El pago seguro es una característica esencial para un sitio de comercio electrónico. La solución de pago debe generar confianza al usuario ("cliente") y garantizar el pago al sitio de comercio electrónico.

Los datos intercambiados durante esta transacción deben estar protegidos (por ejemplo, mediante el protocolo SSL descrito en el párrafo anterior).

Existen varias soluciones de pago (PayBox, ATOS, CyberMut, CyperPaiement, SPPlus, etc.).

El pago seguro puede cambiar ligeramente de un proveedor a otro, pero en general es el mismo para todos: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WB_Paiement-securise.gif)


- 1 a 4. Preparación del pedido en el sitio de comercio electrónico: los usuarios hacen sus pedidos en el sitio. Durante la operación de pago, el sitio de comercio electrónico transfiere la información necesaria para identificar el pedido al operador de pago (número de proveedor, monto de la factura, etc.).

- 5 a 7. Entrada y verificación del número de la tarjeta de crédito: los usuarios introducen el número de tarjeta de crédito en una página de pago segura. La transmisión de datos está protegida mediante SSL para garantizar la confidencialidad de los datos. El sitio de comercio electrónico no puede ver el número de tarjeta de crédito introducido por el usuario.

- 8. Volver al sitio de comercio electrónico: el operador del pago indica al sitio de comercio electrónico si el pago ha sido validado, cancelado o rechazado.

### Sistema de pago seguro en un sitio WEBDEV
<a name="sistema_pago_seguro_sitio_webdev_ELTPARAGRAPHE000095"></a>

**Procedimiento general**

En la mayoría de los casos, se deben realizar las siguientes operaciones al implementar una solución de pago seguro:

1. Solicitar un kit de desarrollo de software al operador de pagos (PayBox, ATOS, SIPS, CyberMut, etc.).

2. Contactar con el operador de pago para establecer un contrato de venta a distancia. En esta etapa, el banco proporciona un número de proveedor.

3. Contactar con el operador de pago para establecer un contrato que especifique los parámetros del contrato de venta a distancia.

4. Poner en marcha el sitio de comercio electrónico.




**Utilizar el componente proporcionado con WEBDEV: componente "Secure payment"**

WEBDEV incluye varios componentes para realizar pagos en línea.
El componente "Secure payment" incluye varios modos de pago seguro. El componente "Secure payment" se proporciona con el código fuente y un ejemplo de uso.



- Para abrir el ejemplo del componente "Secure payment":

	1. Vaya a la página de inicio de WEBDEV (Ctrl +&lt;). 

	2. Haga clic en "Abrir un ejemplo". 

	3. Escriba "Payment" en el cuadro de búsqueda. 

	4. Haga clic en el ejemplo "WW_SecurePayment": el proyecto correspondiente se abre automáticamente en el editor.

	5. En el panel "Explorador de proyectos", seleccione la configuración "Ejemplo". 

	6. Pruebe el ejemplo.

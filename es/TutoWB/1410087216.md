## Presentación
<a name="presentacion_ELTTEXTE000292"></a>
Ahora que sabemos cómo crear y/o adaptar un sitio para HFSQL Client/Server, veremos cómo administrar la base de datos.

Una base de datos Cliente/Servidor requiere:

- una configuración específica en los equipos (instalar un servidor HFSQL, etc.).

- una administración a través del Centro de Control HFSQL.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Configuración de los equipos
<a name="configuracion_los_equipos_ELTTEXTE000316"></a>
Para utilizar una base de datos HFSQL Client/Server, se debe instalar un servidor HFSQL en el servidor. Se pueden instalar varios servidores HFSQL que usen puertos diferentes en el mismo equipo. 

Por ejemplo, en un mismo equipo, es posible instalar un servidor HFSQL de pruebas (con una base de datos de pruebas) y un servidor HFSQL de producción, utilizando puertos diferentes.

Sin embargo, por razones de rendimiento, no se recomienda esta configuración. Se pueden instalar una o más bases de datos en cada servidor.

<a name="NOTE4_1"></a>


## Centro de Control HFSQL
<a name="centro_control_hfsql_ELTTEXTE000340"></a>
El Centro de Control HFSQL permite administrar de forma completa bases de datos y servidores HFSQL Client/Server.

Vamos a ver las principales funcionalidades.

Primero, vamos a abrir el Centro de Control HFSQL desde el proyecto WEBDEV.

- Para abrir el Centro de Control HFSQL desde WEBDEV y acceder a los datos:

	1. En el menú de WEBDEV, en la pestaña "Herramientas", en el grupo "Base de datos", haga clic en "HFSQL". El Centro de Control HFSQL se abre.

	2. La ventana de inicio del Centro de Control HFSQL aparece. El análisis del proyecto actual se selecciona automáticamente. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0008.jpg&type=thumb)


	3. Valide la pantalla. El Centro de Control HFSQL se abre. Este modo de inicio le permite ver los diferentes archivos de datos relacionados con el análisis del proyecto actual.

4. Haga clic en la pestaña vertical "HFSQL C/S" si es necesario. La lista de archivos de datos HFSQL Client/Server aparece. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0009.jpg&type=thumb)
El Centro de Control muestra la lista de los archivos de datos Cliente/Servidor que se encuentran en el análisis relacionado con el proyecto actual. No hay ninguna conexión: los nombres de los archivos de datos aparecen en gris.

5. Para ver los registros de los archivos de datos, haga doble clic en uno de los archivos de datos de la izquierda (por ejemplo, "Order"). Si el Centro de Control HFSQL no reconoce todos los parámetros de la conexión, una ventana le permite establecer la conexión con el servidor HFSQL Client/Server utilizado. Si aparece esta ventana, especifique la contraseña y valide.

6. La información del archivo de datos seleccionado que utiliza la conexión se muestra en una nueva pestaña "Order". Esta información está disponible en varias pestañas verticales: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0011.jpg&type=thumb)




**Toda la base de datos HFSQL Client/Server se puede administrar desde el Centro de Control HFSQL Client/Server.**

## Crear una cuenta de usuario en el Centro de Control HFSQL
<a name="crear_una_cuenta_usuario_centro_control_hfsql_ELTTEXTE000364"></a>
Al instalar un servidor HFSQL y crear una base de datos se crea una cuenta de usuario: la cuenta del administrador. De forma predeterminada, el administrador puede utilizar "Admin" como nombre de usuario y dejar la contraseña vacía. Recordatorio: la contraseña se puede establecer al instalar el servidor HFSQL. 

Las cuentas de usuario permiten acceder a los datos de forma segura. No todos los usuarios del sitio son administradores. Se pueden conceder permisos específicos a cada usuario (o grupo de usuarios).|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=avertissement.png) | ¡Atención! | Los permisos de usuario especificados en el Centro de Control HFSQL se otorgan para la base de datos y no para acceder al sitio.<br><br>Los permisos concedidos para las bases de datos Cliente/Servidor no deben confundirse con los del Groupware Usuario, que vimos en una lección anterior. |



Puede que algunos usuarios no tengan permisos para escribir en ciertos archivos de datos, por ejemplo.
Para hacer una prueba sencilla, crearemos un usuario y le otorgaremos permisos para ver los registros del archivo de datos Customer.

- Para conectarse directamente a la base de datos del servidor:

	1. Despliegue el menú ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0009%201.jpg) en la parte superior izquierda del Centro de Control HFSQL y seleccione "Conectarse a un servidor HFSQL".

	2. La ventana de inicio del Centro de Control HFSQL aparece. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0012.jpg&type=thumb)


	3. La opción "Conectarse a un servidor HFSQL" está seleccionada de forma predeterminada. Especifique las características del servidor instalado en la lección anterior.

4. Las características del servidor HFSQL se muestran en diferentes paneles:

	- el nombre del servidor HFSQL y la lista de bases de datos se muestran en el panel izquierdo.

	- en la sección derecha de la pantalla, aparece una nueva pestaña que muestra las características del servidor HFSQL. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0013.jpg&type=thumb)




Vamos a conceder permisos al usuario: el usuario "Test" puede conectarse a la base de datos y leer el archivo de datos Customer.



- Para otorgar permisos de conexión a la base de datos:

	1. En el Centro de Control HFSQL, haga doble clic en la base de datos "Full_WEBDEV_Site". Una nueva pestaña aparece.

	2. En la pestaña "Full_WEBDEV_Site", en el grupo "Permisos", haga clic en "Administrar permisos".

	3. Seleccione el usuario "Test" en la lista de usuarios.

	4. En la lista de permisos, para la acción "Permiso para conectarse al servidor (conexión cifrada y no cifrada)", haga clic en la columna "Permiso definido" y seleccione la marca de verificación verde. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0016.jpg&type=thumb)


	5. Haga clic en "Aplicar" en la parte inferior de la ventana. El permiso se hace efectivo.

6. Cierre la ventana de administración de permisos.




- Para conceder permisos de lectura sobre el archivo de datos Customer:

	1. En el Centro de Control HFSQL, despliegue la base de datos "Full_WEBDEV_Site", y luego haga doble clic en el archivo de datos Customer (a la izquierda de la pantalla).

	2. En la pestaña "Customer", en el grupo "Permisos", haga clic en "Administrar permisos".

	3. Seleccione el usuario "Test" en la lista de usuarios. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0017.jpg&type=thumb)


	4. En la lista de permisos, para la acción "Permiso para leer los registros de un archivo", haga clic en la columna "Permiso definido" y seleccione la marca de verificación verde. 

5. Haga clic en "Aplicar" en la parte inferior de la ventana. El permiso se hace efectivo.

6. Cierre la ventana de administración de permisos.




Del mismo modo, se pueden definir permisos:

- sobre el servidor HFSQL, 

- sobre la base de datos, 

- sobre los archivos de la base de datos.




En nuestro ejemplo, el usuario "Test" solo puede navegar por los registros del archivo de datos Customer. Si este usuario intenta realizar otra acción, se mostrará un mensaje: "El usuario Test no tiene permisos suficientes para XXXX" (donde XXXX corresponde a la acción realizada).

Una vez creada la cuenta, se puede utilizar cuando la aplicación se conecta al servidor (cuando se utiliza la función [HOpenConnection](../WDLang4/3044107.md)).|   |   |   |

## Hacer una copia de seguridad de la base de datos
<a name="hacer_una_copia_seguridad_base_datos_ELTTEXTE000442"></a>
Para hacer una copia de seguridad de la base de datos:

1. Vaya a la pestaña que corresponde a la base de datos "Full_WEBDEV_Site".

2. Seleccione la pestaña "Copias de seguridad" a la izquierda.

3. En el menú, en el grupo "Copias de seguridad", despliegue "Nueva copia de seguridad" y seleccione "Nueva copia de seguridad en caliente".
	Observación: Esta opción está disponible en el grupo "Copias de seguridad":

	- en la pestaña que corresponde al servidor HFSQL,

	- en la pestaña que corresponde a la base de datos. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P11_Passage%20analyse%20en%20CS%20WB%20-%20HC%20N%B0018.jpg&type=thumb)





4. Valide la copia de seguridad. 






## Conclusión
<a name="conclusion_ELTTEXTE000466"></a>
El Centro de Control HFSQL es una herramienta completa de administración de bases de datos que permite:

- detener o reiniciar un servidor si se produce un problema,

- administrar usuarios y permisos,

- reindexar archivos de datos si es necesario,

- hacer copias de seguridad de la base de datos.




**El Centro de Control HFSQL es una herramienta redistribuible que se puede instalar en los equipos de los usuarios que trabajan con bases de datos HFSQL Client/Server. El Centro de Control HFSQL debe ser gestionado por el administrador de la base de datos**.

## Principales términos utilizados
<a name="principales_terminos_utilizados_ELTTEXTE000102"></a>



|   |
| --- |
| **Acceso Nativo (Conector Nativo)**<br>Método para conectarse a una base de datos desde un programa.<br><br> |
| **Ajax**<br>AJAX (Asynchronous JavaScript and XML) es una tecnología utilizada para actualizar solo los datos modificados en una página HTML sin tener que volver a mostrar toda la página.<br><br> |
| **Asignación**<br>Operación que consiste en asignar un valor a una variable o control. Por ejemplo: <br><br><br><pre><code>// Assign the value MOORE to CustomerName variable<br>CustomerName = "MOORE"</code></pre><br>   <br>El signo = es el operador de asignación.<br><br> |
| **Alineación**<br>Método para organizar los controles en una ventana o página. Por ejemplo, centrar un control en una página, definir el mismo ancho para varios controles, etc.<br><br> |
| **Tema**<br>Libro de estilo de un sitio WEBDEV.<br><br>Elemento en el que se define el diseño gráfico de las páginas de un proyecto WEBDEV.<br><br> |
| **Análisis**<br>Descripción de la estructura de los archivos de datos y relaciones.<br><br> |
| **Anclaje**<br>Reglas de posición y redimensionamiento definidas para adaptar el contenido de una ventana o página cuando se redimensiona el contenedor (ventana o navegador).<br><br> |
| **AWP**<br>Página a la que se puede acceder directamente a través de una dirección (URL fija), que se puede posicionar en buscadores y que no tiene un contexto automático.<br><br> |
| **Base de datos**<br>Elemento que contiene los datos del programa. Los datos están organizados en archivos de datos (también llamados tablas).<br><br> |
| **Bloque (reporte)**<br>Elemento que define la estructura de un reporte. Por ejemplo, los bloques Encabezado de página, Pie de página y Cuerpo.<br><br> |
| **Control (ventana o página)**<br>Elemento gráfico utilizado para construir la interfaz de un programa o sitio.<br><br> |
| **Clase (POO)**<br>Elemento definido en la Programación Orientada a Objetos. Una clase reúne métodos (acciones) y miembros (datos).<br><br> |
| **Classic (archivo de datos)**<br>Tipo de acceso a un archivo de datos HFSQL. Un archivo de datos HFSQL es de tipo Classic cuando se accede a él directamente en su directorio.<br><br> |
| **Clave (archivo de datos)**<br>Características de un campo de un archivo de datos. Un campo clave optimiza búsquedas y ordenaciones en un archivo de datos. <br><br>Sinónimo: índice<br><br> |
| **Client/Server (archivo de datos)**<br>Tipo de acceso a un archivo de datos HFSQL. Un archivo de datos HFSQL es de tipo Client/Server cuando se accede a él conectándose al servidor en el que se encuentra a través del motor HFSQL.<br><br> |
| **Código del proyecto**<br>Código que se ejecuta al iniciar un programa o sitio.<br><br> |
| **Componente externo**<br>Biblioteca de software utilizada para exportar una o varias funciones empresariales con el fin de reutilizarlas.<br><br> |
| **Componente interno**<br>Contenedor que agrupa los elementos de un proyecto (ventana, página, consulta, reporte, clase, etc.) para permitir y facilitar el intercambio con otros proyectos.<br><br> |
| **Configuración de proyecto**<br>Formato de salida de un proyecto: ejecutable Windows, Linux, JAVA, etc.<br><br> |
| **Conector Nativo (Acceso Nativo)**<br>Método para conectarse a una base de datos desde un programa.<br><br> |
| **Contexto (Página)**<br>Parte almacenada en la memoria de un servidor Web que se utilizó para construir una página que se muestra en un navegador.<br><br> |
| **Restricción de integridad**<br>Regla asociada a un campo de un archivo de datos para asegurar la consistencia de los datos en una base de datos.<br><br> |
| **CSS**<br>Lenguaje de hojas de estilo que describe los diferentes elementos de una página HTML.<br><br> |
| **Enlace de datos (Data binding)**<br>Método que permite enlazar un elemento de la interfaz a una fuente de datos (variable o campo).<br><br> |
| **Implementación**<br>Método a través del cual se proporciona un programa al usuario final.<br><br> |
| **Editor**<br>Programa utilizado para crear un elemento del proyecto (editor de ventanas, editor de reportes, etc ..).<br><br> |
| **Reporte**<br>Elemento del proyecto destinado a ser impreso.<br><br> |
| **AAF**<br>Funcionalidades Automáticas de la Aplicación .<br><br> |
| **Hoja de estilo**<br>Contiene la lista de estilos utilizados en un proyecto.<br><br> |
| **Archivo de datos (Tabla)**<br>Elemento que define la estructura de una base de datos. Un archivo de datos permite almacenar los datos introducidos en un programa.<br><br>Por ejemplo, un archivo de datos CLIENTE contendrá la información del cliente introducida en un programa.<br><br> |
| **SCM**<br>Administrador de Código Fuente. Herramienta para organizar, compartir el código fuente del proyecto, administrar permisos, etc.<br><br> |
| **Global (variable o procedimiento)**<br>Corresponde al alcance de una variable o procedimiento en la memoria. Se puede acceder a un elemento global desde cualquier otro elemento del proyecto. El concepto opuesto es local.<br><br> |
| **Groupware Usuario**<br>Herramienta para definir y administrar permisos de acceso a la interfaz para los usuarios de un programa o sitio. Por ejemplo, evitar que un usuario haga clic en el botón "Eliminar" en función de su nombre de usuario o grupo.<br><br> |
| **Homotético**<br>Método para ampliar las imágenes sin que se distorsionen.<br><br> |
| **HTML**<br>HyperText Markup Language.<br><br>Lenguaje utilizado para describir los elementos de una página Web.<br><br> |
| **GUI** (también llamado UI)<br>Interfaz gráfica de usuario (o interfaz de usuario). Elementos gráficos de un programa, como ventanas o páginas. Es lo que ven los usuarios cuando utilizan un programa.<br><br> |
| **Índice (archivo de datos)**<br>Sinónimo: clave<br><br> |
| **JavaScript**<br>Lenguaje de programación utilizado por los navegadores para realizar cálculos y procesos en una página HTML (de lado del cliente).<br><br> |
| **Relación (análisis)**<br>Describe la naturaleza de la relación o punto común entre 2 archivos de datos del análisis. Las reglas de integridad para escribir en los archivos de datos se pueden definir en función de la relación especificada.<br><br> |
| **Live Data**<br>Mecanismo que consiste en mostrar datos reales procedentes de la base de datos al crear la UI. Este mecanismo solo se utiliza si el elemento está vinculado al archivo de datos.<br><br> |
| **Local (variable o procedimiento)**<br>Corresponde al alcance de una variable o procedimiento en la memoria. Solo se puede acceder a un elemento local en el proceso en el que se definió. El concepto opuesto es global.<br><br> |
| **Miembro**<br>Variable que pertenece a una clase o estructura.<br><br> |
| **Menú contextual**<br>Menú desplegable que contiene diferentes acciones en función de la ubicación del clic derecho y del tipo de elemento sobre el que se hace clic derecho.<br><br> |
| **Método**<br>Procedimiento que pertenece a una clase utilizado para actuar sobre los datos (miembros) de dicha clase.<br><br> |
| **Modelo de controles**<br>Contenedor de uno o varios controles (con procesos) que puede reutilizarse en las páginas tantas veces como sea necesario.<br><br>Característica principal del modelo: si se modifica el modelo inicial, los cambios se aplican automáticamente a los elementos que utilizan el modelo.<br><br> |
| **Modelo de páginas**<br>Contenedor que representa una página estándar que se puede aplicar a una o más páginas del proyecto.<br><br>Característica principal del modelo: si se modifica el modelo inicial, los cambios se aplican automáticamente a los elementos que utilizan el modelo.<br><br> |
| **Modelo de reportes**<br>Contenedor que representa un reporte estándar que se puede aplicar a uno o más reportes del proyecto.<br><br>Característica principal del modelo: si se modifica el modelo inicial, los cambios se aplican automáticamente a los reportes que utilizan dicho modelo.<br><br> |
| **n-capas**<br>Método de programación en capas. Cada capa es independiente y puede modificarse sin afectar a las demás.<br><br>Ventaja: Mantenimiento simplificado.<br><br>Inconvenientes: Dificultad y tiempo de desarrollo.<br><br> |
| **Página**<br>Elemento de un proyecto WEBDEV en el que se define la interfaz gráfica del sitio. En la mayoría de los casos, un sitio incluye varias páginas enlazadas entre sí.<br><br> |
| **Parámetro (ventana, página, procedimiento, método, etc.)**<br>Elemento esperado en una ventana, página, procedimiento o método cuando se llama a alguno de ellos. Cada valor pasado como parámetro debe asignarse a una variable.<br><br> |
| **POO**<br>Abreviatura de Programación Orientada a Objetos.<br><br> |
| **Popup**<br>Tipo de ventana (o página). Una ventana emergente o popup es una ventana (o página) que se abre sobre otra ventana (o página). Puede seguir viendo el contenido de la ventana (o página) que hay debajo mientras escribe en la popup.<br><br> |
| **Privado(a)**<br>Variable o procedimiento que solo puede utilizarse en el elemento actual.<br><br> |
| **Procedimiento**<br>Elemento de un proyecto que contiene el código de un proceso a ejecutar.<br><br> |
| **Programación orientada a eventos**<br>Tipo de programación. Una acción del usuario en una ventana o página desencadena la ejecución de un código. El código de la acción a ejecutar se introduce en el evento que representa la acción del usuario.<br><br>Por ejemplo, el evento "Clic" de un control Botón corresponde a un clic del usuario en el botón.<br><br> |
| **Programación Orientada a Objetos (POO)**<br>Método de programación avanzado, opuesto a la programación por procedimientos.<br><br>En la POO, solo se manipulan objetos, es decir, conjuntos agrupados de variables y métodos asociados a entidades que incluyen dichas variables y métodos.<br><br>En la programación por procedimientos, se definen funciones que se llaman entre sí. Cada función o procedimiento está asociado a un proceso específico que puede dividirse en subprocesos hasta obtener las funciones básicas.<br><br> |
| **Proyecto**<br>Conjunto de elementos que definen la estructura de un programa o sitio. Un proyecto contiene un análisis, páginas, reportes, consultas, etc.<br><br> |
| **Propiedad (control, ventana, etc.)**<br>Palabra clave que representa la característica de un elemento. Las propiedades manipulan y cambian las características de los elementos de un proyecto mediante programación.<br><br> |
| **Público(a)**<br>Variable o procedimiento que se puede utilizar desde cualquier elemento.<br><br> |
| **RAD**<br>Acrónimo de Rapid Application Development (Desarrollo Rápido de Aplicaciones)<br><br>Método de desarrollo rápido de un programa basado en un análisis (descripción de archivos de datos).<br><br> |
| **RAD basado en un ejemplo**<br>Método de desarrollo rápido de un programa basado en un ejemplo.<br><br> |
| **Consulta**<br>Elemento escrito en lenguaje SQL que permite acceder (en modo de solo lectura o lectura/escritura) al contenido de una base de datos relacional.<br><br> |
| **Campo**<br>Elemento de la estructura de un archivo de datos (tabla) en un análisis. Por ejemplo, un archivo de datos CLIENTE puede incluir los campos Nombre y Apellido.<br><br> |
| **Ruptura (reporte)**<br>Mecanismo que consiste en agrupar y separar los datos según un valor.<br><br>Por ejemplo, una ruptura en un reporte que muestre los clientes por ciudad. Los clientes que viven en la misma ciudad se agrupan. Visualmente, los clientes se separan cada vez que se cambia de ciudad.<br><br> |
| **Sitio dinámico**<br>Proyecto desarrollado en WEBDEV que contiene páginas estáticas y dinámicas (en modo Sesión o AWP). Las páginas dinámicas (en modo Sesión o AWP) gestionan el acceso a los datos almacenados en una base de datos.<br><br> |
| **Sitio estático**<br>Proyecto desarrollado en WEBDEV que contiene solo páginas estáticas, es decir, que no tienen acceso a una base de datos.<br><br> |
| **Estructura**<br>Tipo de variable compuesta por varias subvariables.<br><br> |
| **Estilo**<br>Elemento que describe el estilo gráfico de un control en una ventana o página. Un estilo incluye, por ejemplo, una fuente, el tamaño de los caracteres, el color de los caracteres, etc.<br><br> |
| **Tabla (archivo de datos)**<br>Elemento que define la estructura de una base de datos. Una tabla almacena los datos introducidos en un programa. Por ejemplo, una tabla CLIENTE contiene los nombres y direcciones de los clientes que se han introducido en un programa.<br><br> |
| **Tabla (control)**<br>Elemento gráfico en una ventana o página. Un control Tabla incluye una o varias columnas y filas.<br><br> |
| **Array**<br>Tipo de variable que contiene varios valores. Se puede acceder a los valores mediante un índice. Los caracteres [] permiten acceder a los elementos de un array.<br><br> |
| **UI**<br>interfaz de usuario. Elementos gráficos de un programa, como ventanas o páginas. Es lo que ven los usuarios cuando utilizan un programa.<br><br> |
| **Variable**<br>Elemento que almacena un valor del programa en la memoria. Existen varios tipos de variables. Cada tipo corresponde a la naturaleza del valor que será almacenado. Por ejemplo, una variable de tipo String para almacenar el nombre de una persona, una variable de tipo Currency para almacenar una cantidad.<br><br> |
| **Webservice**<br>Programa instalado en un servidor Web cuyos procesos se pueden acceder a través de la Web.<br><br> |
| **XML**<br>Lenguaje utilizado para organizar, estandarizar y facilitar el intercambio de datos (utilizado principalmente para intercambiar datos con los Webservices).<br><br> |
| **Zoning**<br>Método utilizado para definir el diseño de una página Web. |

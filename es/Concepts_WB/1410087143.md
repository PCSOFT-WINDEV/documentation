
## 1. Probar un sitio: Elementos a probar

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Las pruebas desempeñan un papel esencial en el desarrollo de un sitio. Una prueba comprueba el comportamiento y la usabilidad del sitio.

Los principales puntos a comprobar en un sitio WEBDEV son los siguientes:

- ***funcionamiento del sitio***:
	secuencia de las páginas, ejecución del código, etc.




- ***apariencia del sitio***:
	utilice diferentes navegadores (Internet Explorer, Chrome, FireFox, Edge, etc.), diferentes resoluciones de pantalla, cambie el tamaño del navegador, etc.




- ***características específicas de la Web***:
	impresión, cookies, botón "Atrás" del navegador, etc.




- ***acceso al sitio por parte de diferentes usuarios***:
	gestión de nombres de usuario y contraseñas, gestión de accesos simultáneos a los archivos de datos, etc.




- ***regresión de un sitio durante una actualización***:
	validación de los cambios realizados en un sitio y verificación para garantizar que las características no modificadas siguen siendo accesibles.




- ***prueba de carga*** (para un sitio instalado en el servidor):
	validación del número máximo de usuarios que pueden acceder al sitio.




**Observación**: Para determinar la cobertura de las pruebas ejecutadas, WEBDEV incluye un análisis de "Cobertura de código". La cobertura de código mide la cobertura de las pruebas realizadas en un sitio. Cada línea de código ejecutada se considera como "probada".
Atención: La cobertura de código no permite saber si las pruebas tienen éxito: solo permite saber si se ejecutó una línea de código.

La cobertura de código permite detectar las líneas de código que nunca se han ejecutado.

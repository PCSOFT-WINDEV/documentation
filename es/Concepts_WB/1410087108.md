## ¿Qué es una página AWP?
<a name="¿que_una_pagina_awp_ELTTEXTE000144"></a>
Una página AWP (Active WEBDEV Page) es una página WEBDEV dinámica **sin un contexto persistente** en el servidor. El contexto de página AWP es temporal. Se crea en una sesión temporal.

**Recordatorio**: En un sitio WEBDEV en modo Sesión, cada página posee un contexto de página persistente durante toda la vida útil de la sesión en el servidor.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Modo de funcionamiento de sitios AWP
<a name="modo_funcionamiento_sitios_awp_ELTTEXTE000168"></a>
Los siguientes elementos se crean automáticamente en el servidor cuando se muestra una página AWP en el navegador:

- una sesión temporal,

- un contexto de página AWP temporal.




La **sesión temporal** contiene el contexto de página AWP temporal. Cuando se envía la página AWP al usuario, se destruye el contexto de la página temporal y la sesión temporal. No se conserva nada en la memoria del servidor.

El **contexto de página AWP** temporal contiene todos los elementos necesarios para construir la página que ve el usuario:

- variables locales, 

- procesos del servidor,

- conexiones a las bases de datos,

- contextos de archivos de datos, etc.


Cuando se envía la página AWP al usuario, se destruyen estos elementos.

## ¿Cómo compartir información (valores) entre páginas AWP?
<a name="¿como_compartir_informacion_valores_entre_paginas_awp_ELTTEXTE000192"></a>
Existen dos métodos para compartir información (valores) entre páginas AWP: 

- Pasar información en la URL. Este método permite mejorar el SEO.

- Guardar información en los contextos AWP (mediante programación).




**Pasar información (valores) entre dos páginas en la URL**

Se puede pasar información de una página a otra a través de la URL. La URL tiene el siguiente formato "http:\\\\Servidor Web\\...\\mipagina.awp?NombreParam1=Valor1& NombreParam2=Valor2". 

Este método mejora el SEO de la página porque los robots de SEO pueden ver y analizar la información de la URL.

**Guardar información (valores) en los contextos AWP (mediante programación)**. 

Los valores comunes a varias páginas AWP se pueden almacenar en el servidor mediante **contextos AWP**. Un contexto AWP se crea en el disco del servidor. Este contexto permanece disponible mientras se muestren las páginas AWP, y siempre y cuando no se exceda el tiempo de espera de dichos contextos. El tiempo de espera de los contextos AWP se define en el administrador WEBDEV (pestaña "Configuración", opción "Duración de contextos AWP").

Para gestionar los contextos AWP, debe utilizar funciones WLanguage como [DeclareAWPContext](../WDLang2/3058028.md), [FreeAWPContext](../WDLang2/1000020429.md), etc.

Para más información, consulte la ayuda en línea.

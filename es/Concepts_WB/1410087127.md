## ¿Código Servidor o código Navegador?
<a name="¿codigo_servidor_codigo_navegador_ELTTEXTE000143"></a>
Se pueden programar dos tipos de acciones en un sitio WEBDEV:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P2-les%20deux%20types%20de%20code.gif)
**Para gestionar estos dos tipos de acciones**, el editor de código de WEBDEV propone dos tipos de código:

- ***Código Servidor*** (código amarillo o rosado en el editor de código): Este tipo de código se escribe en WLanguage (código amarillo) o en PHP (código rosado, disponible únicamente en páginas PHP). Este código se ejecuta en el servidor.
	Este código solo está disponible en páginas dinámicas (en modo Sesión o AWP).

- ***Código Navegador*** (código verte o azul): Este tipo de código se escribe en WLanguage (código verde) o en JavaScript (código azul).
	Al guardar la página, este código se convierte automáticamente en JavaScript y se integra en las páginas HTML de WEBDEV. Este código se ejecuta en el equipo del usuario y no requiere ninguna acción del servidor.

## Código Navegador: ¿Código WLanguage o código JavaScript?
<a name="codigo_navegador_¿codigo_wlanguage_codigo_javascript_ELTTEXTE000167"></a>
El código que se ejecuta en el equipo del usuario (en el navegador) está representado por una banda verde o azul en el editor de código.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P2_Code_WL_JS_PHP%20-%20HC%20N%B0001.gif&type=thumb)<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P2_Code_WL_JS_PHP%20-%20HC%20N%B0002.gif&type=thumb)


- Banda verde = WLanguage: el símbolo WL aparece en el encabezado del código.

- Banda azul = JavaScript: el símbolo JS aparece en el encabezado del código.


Para pasar del código verde (WLanguage) al código azul (JavaScript), simplemente haga clic en el símbolo WL del encabezado del código (y a la inversa, haga clic en JS), y elija el lenguaje deseado. Al guardar la página, el código WLanguage se convierte automáticamente en JavaScript.

**Le recomendamos desarrollar en WLanguage**.

## Eventos adicionales
<a name="eventos_adicionales_ELTTEXTE000191"></a>
El enlace "Agregar otros eventos" que se encuentra en la ventana de código permite utilizar otros eventos además de los eventos propuestos de forma predeterminada. Simplemente haga clic en "Agregar otros eventos a xxx" en la parte inferior de la ventana de código: la lista de eventos adicionales se muestra en pantalla. 

Seleccione los eventos que desea agregar al editor de código.

Ejemplo de eventos adicionales: doble clic, tecla pulsada, tecla presionada, tecla soltada, botón presionado, movimiento del puntero, etc.

**Recordatorio**: El código Navegador está disponible en páginas dinámicas (en modo Sesión o AWP) y estáticas.

## Sitio dinámico en PHP: Código Servidor: ¿Código WLanguage o código PHP?
<a name="sitio_dinamico_php_codigo_servidor_¿codigo_wlanguage_codigo_php_ELTTEXTE000215"></a>
El código que se ejecuta en el servidor está representado por una banda amarilla o rosada en el editor de código.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P2_Code_WL_JS_PHP%20-%20HC%20N%B0003.gif&type=thumb)<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P2_Code_WL_JS_PHP%20-%20HC%20N%B0004.gif&type=thumb)


- Banda amarilla = WLanguage: el símbolo WL aparece en el encabezado del código.

- Banda rosada = PHP: el símbolo PHP aparece en el encabezado del código.




Para pasar del código amarillo (WLanguage) al código rosado (PHP), simplemente haga clic en el símbolo WL del encabezado del código (y a la inversa, haga clic en PHP), y elija el lenguaje deseado. Al guardar la página PHP, el código WLanguage se convierte automáticamente en PHP.

**Le recomendamos desarrollar en WLanguage**.

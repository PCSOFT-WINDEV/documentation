
## 8. Sitio "mobile friendly": Responsive Web Design

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Los sitios Web se pueden ver en varias plataformas: móviles, tabletas o PCs, ... Dependiendo del dispositivo, el tamaño del navegador y el espacio utilizado para mostrar los datos pueden ser diferentes.

El método "Responsive Web Design" consiste en crear una sola página que se adapta automáticamente según la plataforma en la que se muestra.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Método utilizado
<a name="metodo_utilizado_ELTTEXTE000150"></a>
De forma predeterminada, WEBDEV utiliza el método "Desktop first": los desarrolladores deben crear primero la interfaz de sus sitios Web en modo "Escritorio". Luego, pueden crear la versión en modo "Tableta", y finalmente, en modo "Móvil".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Responsive.gif)
En términos de diseño, el sitio Web se desarrolla para un uso en "escritorio". Al reducir las resoluciones de pantalla (tamaño de la página), el contenido y las funcionalidades se mueven, cambian de tamaño u ocultan.

En el editor de WEBDEV, las diferentes resoluciones están representadas por rangos de resolución. Una rango de resolución representa el tamaño de navegador para un dispositivo específico. WEBDEV contiene 3 rangos de resolución:

- Móvil, que corresponde a teléfonos inteligentes (Smartphones).

- Tableta, que corresponde a las tabletas.

- Escritorio, que corresponde a los PC, MAC, etc.

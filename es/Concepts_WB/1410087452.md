<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Además de los sitios Web, WEBDEV permite generar varios tipos de proyectos.


|   |   |
| --- | --- |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P3_WB%20Mode%20de%20generation%20-%20HC%20N%B0002%201.gif)<br> | **Sitios**<br><br>Los sitios son el modo de generación más común. WEBDEV permite generar:<br><br>- sitios estáticos o dinámicos (Sesión o AWP).<br><br>- sitios PHP.<br><br><br> |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P3_WB%20Mode%20de%20generation%20-%20HC%20N%B0001%202.gif)<br> | **Bibliotecas**<br><br>Una biblioteca es un archivo único que agrupa varios elementos de un proyecto WEBDEV: páginas, reportes, etc. Puede generar bibliotecas independientes que pueden ser utilizadas por otros sitios. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P3_WB%20Mode%20de%20generation%20-%20HC%20N%B0001%201.gif)<br> | **Componentes externos**<br><br>Los componentes externos son fragmentos de aplicaciones que permiten compartir una o varias funcionalidades específicas entre varias aplicaciones. Un componente externo generado con WINDEV también puede utilizarse en un proyecto WEBDEV o WINDEV Mobile. |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P3_WB%20Mode%20de%20generation%20-%20HC%20N%B0002.gif)<br> | **Webservices SOAP o REST**<br><br>Un Webservice (también llamado servicio Web XML) puede generarse a partir de un proyecto WINDEV o WEBDEV. Un Webservice utiliza un conjunto de funciones (uno o varios conjuntos de procedimientos) y los hace accesibles a través de la Web (o de una red privada) mediante los protocolos HTTP, SOAP y REST.<br><br>Observación: Un Webservice debe implementarse en un WEBDEV Application Server para ser utilizado. |

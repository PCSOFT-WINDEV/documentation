


## 4. ¿Cuándo utilizar el depurador?

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
El depurador siempre se puede utilizar cuando se ejecuta una prueba desde el editor. Solo es necesario establecer un punto de interrupción en el punto deseado para depurar el sitio paso a paso.

Los diferentes tipos de pruebas disponibles en el editor son los siguientes:

- ***"Go" de la página***:
	La página actual en el editor se puede probar mediante el ícono ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_Go%20Page_bl.gif) en los botones de acceso rápido de WEBDEV.




- ***"Go" del proyecto***:
	La prueba del proyecto en el equipo de desarrollo se puede ejecutar mediante una de las siguientes opciones:

	- ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_GO_Projet_WB_bl.gif)que se encuentra en los botones de acceso rápido de WEBDEV,

	- en la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Depurar proyecto".







- ***Proyecto implementado***:
	Un proyecto implementado se puede depurar. Esta característica es muy útil para reproducir un problema que ocurre en condiciones específicas.
	Para iniciar el proyecto implementado, en la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Conectarse a un sitio y depurar la conexión".
	**Observación**: Esta opción puede usarse con un servidor de desarrollo o con un servidor de implementación. El servidor debe estar configurado para permitir la depuración remota.




- ***Proyecto implementado que se está utilizando***:
	Un proyecto implementado puede ser depurado mientras se utiliza. Esta característica es muy útil para reproducir un problema que ocurre en condiciones específicas. Caso más común: ocurre un problema mientras se está usando el sitio: puede iniciar el depurador inmediatamente estableciendo un punto de interrupción en el código del proyecto.
	Para utilizar una sesión del proyecto implementado, en la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione la opción "Depurar una conexión existente".
	**Observación**: Esta opción puede usarse con un servidor de desarrollo o con un servidor de implementación. El servidor debe estar configurado para permitir la depuración remota.

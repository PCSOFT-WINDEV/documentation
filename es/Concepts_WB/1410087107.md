## Sesión WEBDEV
<a name="sesion_webdev_ELTTEXTE000128"></a>
Cuando se muestra una página WEBDEV en modo Sesión, se realizan las siguientes operaciones:

1. Solicitud de visualización de la página.

2. Inicio del motor WEBDEV. El motor permanece en el servidor hasta que se cierra la aplicación.

3. Creación del contexto de la aplicación. El contexto permanece en el servidor hasta que se cierra la aplicación.

4. El motor WEBDEV ejecuta el código Servidor y construye la página HTML (por ejemplo, a partir de los datos de la base de datos).

5. Una vez que la página HTML está completamente construida, el servidor transmite el resultado al cliente (el navegador).

<a name="NOTE2_1"></a>


## Contextos de página
<a name="contextos_pagina_ELTTEXTE000152"></a>
Por cada página que se muestra en el navegador, se crea automáticamente un contexto de página en el servidor. Este contexto de página contiene todos los elementos necesarios para construir la página que ve el usuario final:

- variables globales,

- variables locales, 

- procesos del servidor, 

- conexiones a las bases de datos,

- contextos de archivos de datos, etc.




Los contextos de página permanecen en la memoria en el servidor hasta el final de la sesión WEBDEV.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=mode-dynamique-WebDev.gif)


Cuando se llama a la misma página varias veces:

- Si se llama a la función [PageDisplay](../WDLang2/3058008.md) para mostrar la página, el contexto de la página se elimina y se vuelve a crear.

- Si se llama a la función [PageRefresh](../WDLang2/3058011.md) para mostrar la página, se vuelve a utilizar el mismo contexto de página.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Programación
<a name="programacion_ELTTEXTE000176"></a>
De forma predeterminada, la sesión WEBDEV y los contextos de página se gestionan automáticamente. No hay nada por programar.

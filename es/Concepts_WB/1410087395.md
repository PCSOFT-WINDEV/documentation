## 2. Configuración en un servidor Windows con acceso a INTERNET vía UNIX
<a name="2_configuracion_servidor_windows_con_acceso_internet_via_unix_ELTTEXTE000169"></a>
La configuración es la siguiente: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P8-Installation%20sur%20un%20serveur%20Windows.gif)


1. Abrir una página estática. La página estática está en el servidor UNIX.

2. Iniciar el sitio WEBDEV dinámico (ubicado en el servidor Windows) a través de un enlace en la página HTML estática.

3. El modo de funcionamiento estándar del sitio WEBDEV.




**Observación**: la misma configuración puede ser definida en un servidor Linux.

## 3. Configuración en un servidor Windows o Linux con Load Balancer
<a name="3_configuracion_servidor_windows_linux_con_load_balancer_ELTTEXTE000193"></a>
El sistema de balanceo de carga se utiliza para repartir el número de conexiones a un sitio dinámico de WEBDEV en varios ordenadores.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P8-Installation%20sur%20un%20serveur%20Windows%20ou%20Linux.gif)

<a name="NOTE3_2"></a>


### Solución 1: Instalar el sitio dinámico de WEBDEV (Sesión o AWP) sólo en el servidor de datos
<a name="solucion_1_instalar_sitio_dinamico_webdev_sesion_awp_solo_servidor_datos_ELTPARAGRAPHE000044"></a>

Esta solución consiste en instalar:

- el servidor de aplicaciones WEBDEV en cada servidor web,

- los sitios dinámicos de WEBDEV y los archivos de datos en el servidor de datos.



<a name="NOTE3_3"></a>


### Solución 2: Instalando el sitio dinámico de WEBDEV (Session o AWP) en todos los servidores Web
<a name="solucion_2_instalando_sitio_dinamico_webdev_session_awp_todos_los_servidores_web_ELTPARAGRAPHE000056"></a>

Esta solución consiste en instalar:

- el servidor de aplicaciones WEBDEV y los sitios dinámicos WEBDEV en cada servidor web.

- los archivos de datos de los sitios dinámicos de WEBDEV en el servidor de datos.


## 7. Sitio "mobile friendly": Publicación dinámica

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
La "Publicación dinámica" es una técnica en la que la misma dirección (URL) conduce a 2 páginas diferentes: una página para PC y otra para móviles.

La página se mostrará en función del dispositivo (PC o móvil) que la abra.

Puede utilizarse como complemento o alternativa al "Responsive Web Design" para hacer que un sitio Web estándar se adapte a los dispositivos móviles. Con la "Publicación dinámica" ya no es necesario modificar las páginas existentes: simplemente agregue páginas específicas para dispositivos móviles.

Esta técnica mejora el SEO (Google).<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Dynamic%20serving.jpg)


<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principio
<a name="principio_ELTTEXTE000130"></a>
El sitio incluye 2 conjuntos de páginas: Páginas específicas para PC y páginas específicas para móviles. Estas páginas están asociadas en grupos de 2.

Al introducir la dirección de la página en el navegador, se detecta automáticamente el entorno utilizado y se muestra la página correspondiente (PC o dispositivo móvil) en el navegador.. La dirección que se muestra en el navegador no cambia: solo el contenido que se muestra se adapta al sistema.

Todos los enlaces y páginas abiertas en el código se redirigen automáticamente a la página correspondiente en la plataforma actual.

Observaciones:

- El entorno se detecta en función de las recomendaciones de Google: las tabletas no se consideran como dispositivos móviles.

- Google detecta que las páginas están en modo "Publicación dinámica". Por lo tanto, las diferentes versiones de las páginas se optimizan en el buscador. Si las páginas móviles se desarrollan correctamente, el sitio se considera "Mobile friendly".

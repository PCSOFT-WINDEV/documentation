
## 3. Ciclo de desarrollo de un sitio Web

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
WEBDEV cubre todo el ciclo de desarrollo de un sitio:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P2-cycle%20de%20developpement%20d'un%20site.gif)
**Detalles de las diferentes etapas**:

**Etapa de diseño**: Un sitio puede diseñarse a partir de requisitos específicos, del modelado UML de los procesos o incluso de archivos de datos existentes.

**Etapa de desarrollo**: El proyecto y el análisis se crean a través de asistentes específicos. El sitio puede desarrollarse en modo RAD (Rapid Application Development) con generación automática de código e interfaz; o puede resultar de la creación manual de los diferentes elementos del proyecto.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P2-cycle%20de%20developpement%20d'un%20site-2.gif)
**Etapa de pruebas y generación**: WEBDEV incluye varias herramientas de pruebas automatizadas para garantizar la fiabilidad de las aplicaciones y asegurarse de que no se produzca ninguna regresión durante las etapas de desarrollo.

**Etapa de implementación**: Un sitio WEBDEV dinámico (en modo Sesión o AWP) se implementa en un WEBDEV Application Server. Una vez implementado el sitio, puede probarlo antes de hacerlo accesible a los usuarios.

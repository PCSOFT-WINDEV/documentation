
## 17. TLS/SSL: Cifrar información en la Web

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
De forma predeterminada, los datos intercambiados entre el equipo del usuario final y el servidor Web no están cifrados. No se garantiza la autenticidad de la identidad del servidor, la confidencialidad y la integridad de los datos intercambiados.

Para asegurar la confidencialidad, el método más utilizado es el protocolo TLS/SSL (Transport Layer Security, antes llamado Secure Socket Layer). Este protocolo garantiza la identidad del servidor al que se accede por Internet y cifra los datos intercambiados entre el servidor y el navegador.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Implementación de transacciones seguras a través del protocolo TLS/SSL
<a name="implementacion_transacciones_seguras_traves_del_protocolo_tlsssl_ELTTEXTE000135"></a>
Para implementar transacciones seguras a través del protocolo TLS/SSL, debe cumplir con todos los requisitos del protocolo (compra de una clave SSL certificada de una organización específica o generación de una clave autofirmada para uso privado o de prueba). Para obtener más detalles, consulte la ayuda en línea de WEBDEV.

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Transacciones seguras con TLS/SSL en un sitio WEBDEV
<a name="transacciones_seguras_con_tlsssl_sitio_webdev_ELTTEXTE000159"></a>
En la mayoría de los casos, solo se debe asegurar la transferencia de datos sensibles: transferencia de un número de tarjeta de crédito, por ejemplo. El modo seguro es más lento. Sin embargo, un número creciente de sitios está operando enteramente con TLS/SSL gracias al aumento de la potencia de las máquinas, lo que asegura una mayor confidencialidad de las operaciones de los usuarios.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4-TLS-SSL%20Crypter%20les%20informations%20sur%20le%20web.gif)
El modo seguro se establece cuando se abre una nueva página mediante un botón (o enlace). Simplemente llame a la función [SSLActive](../WDLang2/3012014.md) en el código Navegador del botón (o enlace). Una vez se abra la página segura, todas las acciones se realizarán en modo seguro (cifrado).

Para volver al modo no seguro, utilice la función [SSLActive](../WDLang2/3012014.md)(False) en el código Navegador de un botón (o enlace) que abre una nueva página.

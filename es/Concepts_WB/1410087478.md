
## 13. Replicación universal
			

| [Página anterior](../Concepts_WB/1410087439.md) | [Tabla de contenido](../Concepts_WB/1410087102.md) | [Página siguiente](../Concepts_WB/1410087479.md) |

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
La replicación universal permite actualizar bases de datos en formatos similares o diferentes (HFSQL Classic, Oracle, SQL Server, etc.). Por ejemplo, puede sincronizar una base de datos HFSQL Classic con una base de datos Oracle.

La replicación universal utiliza un modelo centralizado: todas las bases de datos están sincronizadas con una base de datos maestra. Luego, la base de datos maestra aplica los cambios a las demás bases de datos. 
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Replication_Bdd.gif)


La sincronización puede adaptarse a casos especiales. Por ejemplo, puede obtener solo los registros relativos a un producto específico, o los registros creados en una fecha determinada, gestionar conflictos, etc.

Estas adaptaciones deben realizarse mediante programación con la función [HRplFilterProcedure](../WDLang4/3044300.md).

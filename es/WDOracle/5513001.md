## Configuración de una conexión
<a name="configuracion_una_conexion_ELTTEXTE000232"></a>
Para configurar la conexión:

1. Inicie WDORAINS (32-bit o 64-bit). 

2. En la ventana que se abre (utilizada para listar las conexiones existentes), pulse el botón "Añadir" para crear una nueva conexión. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Parametrage.gif)


3. Especifique la configuración de la conexión: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Connexion.gif)





- Elija el **nombre de la fuente de datos**. Este nombre se pasará como parámetro a [SQLConnect](../WDLang4/3072005.md) para conectarse a la base de datos. Si la conexión se definió en el editor de análisis de WINDEV o WEBDEV, especifique el nombre de la conexión correspondiente.

- Agregue un título **description**

- Especificar una **cadena de conexión**. Esta cadena de conexión permitirá a SQL\*Net encontrar la base de datos. Este valor debe ser proporcionado por el administrador de la base de datos Oracle. Ejemplo de cadena de conexión: "OracleServer". En este ejemplo, "OracleServer" es el alias del servidor definido en SQL\*Net.

- Especifique si el formato de las fechas debe seguir siendo **compatible con WinDev 5.5**. 
	Recordatorio: Entre la versión 5.5 y la versión 5.5B, el formato de fecha fue modificado. Si marca la casilla, restaurará el formato de fecha devuelto antes de la versión 5.5B (véase el párrafo siguiente).

- Especifique el **formato de las fechas para las consultas INSERTAR y ACTUALIZAR**: esta opción se utiliza para definir el formato utilizado por Default para las fechas en formato de texto. Este formato se utilizará si el formato de fecha no está especificado en la consulta.

- Especifique las **opciones de captación previa** para Oracle 8i o posterior.
	**Recordatorio**: Oracle 8i recupera bloques de registros ("prefetch"). Un bloque grande requiere más memoria pero reduce el tráfico de la red.

	- El número de registros corresponde al número de líneas de cada bloque.

	- La memoria máxima utilizada corresponde al tamaño (en bytes) de cada bloque.. La opción "Sin límite" permite no especificar ningún límite.




- Especificar si las **transacciones distribuidas** deben usarse o no. Por Default, las transacciones distribuidas son utilizadas por las cerraduras y las transacciones manejadas por Conector Nativo Oracle. Marque esta opción si:

	- Las transacciones distribuidas no son soportadas por su servidor.

	- Las transacciones distribuidas no son compatibles con las versiones de las bibliotecas cliente (error ORA_02041).




- La conexión se añade a la lista una vez validados los parámetros.

- WDSQL se utiliza para comprobar si la conexión a la base de datos es correcta.

### Formato de fecha devuelto por el controlador nativo de Oracle (compatibilidad 5.5)
<a name="formato_fecha_devuelto_por_controlador_nativo_oracle_compatibilidad_55_ELTPARAGRAPHE000094"></a>

**En la versión 5.5**: La fecha devuelta por el controlador nativo de Oracle al leer un elemento Date era: DD-MM-YYY

**En la versión 5.5B**: La fecha devuelta por el controlador nativo de Oracle al leer un elemento Date es ahora: AAAAMMDD.

**Razones detrás de esta modificación**: Armoniza la gestión de las fechas en WINDEV. Simplifica la asignación de la fecha leída por el driver de Oracle en una control WINDEV de tipo DATE. No se requiere ninguna Process específica para convertir la fecha en un formato reconocido por WINDEV.

**¿Cómo mantener el modo de funcionamiento anterior?**

Configurar el formato de fecha en WDORAINS: todo lo que tienes que hacer es comprobar "Compatibilidad: devolver las fechas en formato 'DD-MM-YYYY'" al describir la base de datos utilizada.

**Recordatorio**: Cuando escriba en una base de datos a través del controlador nativo de Oracle, le recomendamos que utilice el formato ODBC estándar {d 'AAAA-MM-DD'} (con un espacio entre d y '). El controlador convierte automáticamente la fecha al formato utilizado por la base de datos.

Ejemplo:


```sql
INSERT INTO TABLE(DATE) VALUES ({d '1999-12-25'})
```


**Observación**: A partir de la versión 12, las fechas vacías corresponden a cadenas rellenas de ceros. En las versiones anteriores, las fechas vacías correspondían a cadenas vacías.

## Opciones de conexión
<a name="opciones_conexion_ELTTEXTE000268"></a>


### Opciones para las cerraduras
<a name="opciones_para_las_cerraduras_ELTPARAGRAPHE000119"></a>

[SQLLock](../WDLang4/3072003.md) espera dos parámetros:

- el tiempo de espera: el tiempo de espera durante la llamada a [SQLLock](../WDLang4/3072003.md) puede ser Infinite o no (bloqueo).

- el bloqueo de teclas. Para más detalles, consulte la documentación sobre el servidor.



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=options.gif)


**Observación**: Pueden producirse algunos fallos si la versión del servidor Oracle es anterior a la versión del cliente Oracle (por ejemplo, servidor Oracle 7.3 y cliente Oracle 8).. Para evitar estos problemas, marque "Utilizar siempre un cliente 7.x".

## Prueba de conexión
<a name="prueba_conexion_ELTTEXTE000292"></a>
Para ejecutar la prueba de una conexión desde WDORAINS:

1. Seleccione la conexión cuya prueba debe ejecutarse.

2. Haz clic en el botón "Probar".

3. Especifique el nombre y la contraseña del usuario.

4. Haga clic en "Probar conexión".. Enseguida, se muestra el informe.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=test.gif)

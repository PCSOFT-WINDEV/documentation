<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000195"></a>
WINDEV y WEBDEV soportan un Conector Nativo SQL Server opcional (también llamado Native SQL Server Access). Esta Conector Nativo es compatible con las versiones 7, 2000, 2005, 2008, 2012, 2014, 2016, 2019 y 2022, así como con las versiones Express de SQL Server.. Esta Conector Nativo también gestiona el acceso a las bases de datos SQL Azure. 
Esta Conector Nativo le permite acceder a una base de datos de un servidor SQL desde un programa WLanguage sin usar ningún controlador externo.

La Conector Nativo SQL Server ofrece varios modos de uso:




|   |   |
| --- | --- |
| Servidor SQL vía OLE DB | Servidor SQL vía ODBC |
| La Conector Nativo SQL Server se utiliza para acceder a la capa de clientes a través de la OLE DB. <br><br>Este método permite utilizar las diferentes capas de cliente propuestas por Microsoft:<br><br>- *MSOLEDBSQL* (recomendado)<br><br>- *SQLnCli 11*: Capa de cliente correspondiente a SQL Server 2012. <br><br>- *SQLnCli 10*: Capa de cliente correspondiente a SQL Server 2008. <br><br>- *SQLnCli*: Capa de cliente correspondiente a SQL Server 2005. Es más poderoso que SQLOLEDB pero necesita ser instalado en las computadoras de los usuarios.<br><br>- *SQLOLEDB*: Capa de cliente liberada a partir de SQL Server 2000. Se puede encontrar en los sistemas operativos de Microsoft desde Windows XP.<br><br><br><br><br>**Este método está disponible en Windows de 32 y 64 bits.** | La Conector Nativo SQL Server se utiliza para acceder a la capa de clientes a través de ODBC. <br><br>Este método permite utilizar las diferentes capas de cliente propuestas por Microsoft:<br><br>- *msodbcsql17* (recomendado)<br><br>- *msodbcsql13*<br><br>- *SQLnCli 11*<br><br>- *SQLnCli 10*: Capa de cliente correspondiente a SQL Server 2008. A partir de esta versión, es posible utilizar, por ejemplo, los nuevos tipos de HORA y FECHA (novedades de la versión 17).<br><br>- *SQLnCli*: Capa de cliente correspondiente a SQL Server 2005. Es más poderoso que SQLOLEDB pero necesita ser instalado en las computadoras de los usuarios.<br><br><br><br><br>**Este método está disponible en Windows en 32 y 64 bits, y en Linux en 64 bits.** |
| En las versiones de 32 bits, se requiere **wd28 0sqlserver.dll** para este modo de uso del Conector Nativo a través de OLE DB.<br><br>En las versiones de 64 bits, se requiere **wd28 0sqlserver64.dll** para este modo de uso del Conector Nativo a través de OLE DB. | En las versiones de 32 bits, se requiere **wd28 0sqlserver2.dll** para este modo de uso del Conector Nativo a través de OBDC.<br><br>En las versiones de 64 bits, **wd28 0sqlserver2_64.dll** es necesario para este modo de utilización del Conector Nativo vía ODBC. |


Observación: Esta documentación es común a los dos modos de uso de la Conector Nativo. Se indican acciones o comportamientos diferentes cuando es necesario.
**Estas son algunas de las muchas ventajas de usar la Conector Nativo SQL Server:**

- Las funciones HReadxxx (HReadNext, HModify, ....) se pueden utilizar en las bases de datos de SQL Server.. Lo que significa que una base de datos del Servidor SQL y una base de datos HFSQL son manejadas por los mismos comandos. Se simplifica la adaptación de una aplicación existente a bases de datos externas.. 

- El RAD permite generar el código con las funciones Hxxx (HReadNext, ...) o con las funciones SQLxxx.

- WDMap (el visor de datos) puede ser usado con sus bases de datos SQL Server. 

- Soporte de bases de datos SQL Server en todas las versiones (a partir de la versión 7).

- Mejor rendimiento comparado con un acceso vía ODBC, por ejemplo.

- El Conector Nativo se utiliza para explotar un número importante de características ([procedimientos almacenados](../WDSQLServer/5515009.md), [HPrepareSQLQuery](../WDLang4/3044119.md), ...)

- etc.




El uso de esta Conector Nativo es muy recomendable para el desarrollo de aplicaciones Cliente/Servidor (que acceden a bases de datos SQL Server) con WINDEV o WEBDEV.. Siéntase libre de Contact nosotros para cualquier información adicional. 

**Observación**: El departamento de ventas de Contact PC SOFT si no posee Conector Nativo SQL Server.

## aprovechamiento
<a name="aprovechamiento_ELTTEXTE000225"></a>


### Uso de Conector Nativo SQL Server en sus aplicaciones WINDEV o sitios WEBDEV
<a name="uso_conector_nativo_sql_server_sus_aplicaciones_windev_sitios_webdev_ELTPARAGRAPHE000094"></a>

Para utilizar SQL Server en modo nativo, debe hacerlo:

- configure la conexión: 

	- en el editor de análisis, 

	- mediante [HDescribeConnection](../WDLang4/3044205.md), 

	- en WDSQSINS (paso opcional). 
			Observación: WDSQSINS se utiliza para completar los parámetros definidos en el editor de análisis o con [HDescribeConnection](../WDLang4/3044205.md).




- programa:

	- con las funciones "HFSQL" de WLanguage (véase [Programación con funciones HFSQL](../WDSQLServer/5515011.md)).

	- con las funciones "SQL" de WLanguage (véase [Programación con funciones SQL](../WDSQLServer/5515010.md)).







**Observaciones**: 

- La versión del Conector Nativo se puede elegir a través del [H.SQLServerMode](../WDLang4/3044276.md) Variable. 

- La versión de la capa cliente de SQL SERVER se puede elegir con la palabra clave "WD CLIENT VERSION" (especificada en el [información opcional de conexión](../WDLang4/9000124.md)).

- No se olvide de leer el documento [limitaciones de la Conector Nativo SQL Server.](../WDSQLServer/5515005.md)

- Para desplegar los sitios que utilizan el Conector Nativo SQL Server, este último debe estar instalado en el servidor (Servidor de Aplicaciones WEBDEV). 

- La opción "Crear automáticamente los archivos de datos si no existen" (que se encuentra en la pestaña "Archivos" del proyecto description) es tenida en cuenta por Conector Nativo SQL Server. 

- El controlador Msoledbsql para SQL Server está soportado por SQL Server Conector Nativo.

- Gestión de errores de thread: Cuando se realiza una llamada a SQL Server Conector Nativo
a partir de un hilo, es posible manejar los errores de una manera Normal.

- La Conector Nativo SQL Server apoya a los EE.UU.

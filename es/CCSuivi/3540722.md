


## Trabajo como jefe de proyecto: ¿Cómo gestionar un proyecto de desarrollo?
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000307"></a>
A través de los Centros control, WINDEV, WEBDEV y WINDEV Mobile incluyen varias herramientas que permiten al gestor de proyectos gestionar un proyecto de desarrollo que incluye

- el Definition de los diferentes contribuyentes del proyecto,

- la Definition de los requisitos y tareas así como su asignación a los diferentes contribuyentes,

- el seguimiento del progreso del proyecto.




**Las diferentes etapas de gestión de un proyecto de desarrollo son las siguientes**: 

1. [Crear y configurar la base de datos de los Centros control (si es necesario)](#NOTE2).

2. [Definir los colaboradores](#NOTE3).

3. [Definir los requisitos](#NOTE4).

4. [Definir los proyectos WINDEV, WEBDEV o WINDEV Mobile](#NOTE5).

5. [Definir las tareas y asociarlas a los requisitos y a los desarrolladores](#NOTE6).

6. [Seguir el progreso del proyecto](#NOTE7).




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Etapa 1: Base de datos de los centros control
<a name="etapa_1_base_datos_los_centros_control_ELTTEXTE000331"></a>
La base de datos de los Centros control agrupa diversas informaciones compartidas por los diferentes módulos que se encuentran en WINDEV, WEBDEV y WINDEV Mobile.

La configuración de la base de datos de los Centros control consiste en indicar la ubicación de esta base de datos y el formato de los archivos utilizados. **Este paso es muy importante**.

Para obtener más información, consulte [Configuración de la base de datos de los Centros control](../CCSuivi/3540720.md).

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Etapa 2: Definición de los colaboradores
<a name="etapa_2_definicion_los_colaboradores_ELTTEXTE000355"></a>
Para seguir el progreso de un proyecto de desarrollo, se deben definir los contribuyentes de este proyecto.

Los contribuyentes se definen en la Centro de Seguimiento de Proyectos. El Centro de Seguimiento de Proyectos sirve para definir a todos los contribuyentes gestionados por los Centros control. A continuación, se utiliza el Centro de Seguimiento de Proyectos para asociar a los contribuyentes con los diferentes proyectos gestionados por el Centro de Seguimiento de Proyectos.

Una vez creados los colaboradores en la base de datos de los Centros control, estos colaboradores pueden utilizar su identificador para iniciar WINDEV, WEBDEV o WINDEV Mobile: serán reconocidos automáticamente. También tienen la posibilidad de conectarse a la Centro de Seguimiento de Proyectos para configurar sus horarios y vacaciones, y gestionar sus tareas.
<a name="NOTE3_2"></a>


### Definición de los colaboradores
<a name="definicion_los_colaboradores_ELTPARAGRAPHE000066"></a>

Definir los colaboradores:

1. Inicie la Centro de Seguimiento de Proyectos desde WINDEV, WEBDEV o WINDEV Mobile: en la pestaña "Herramientas", en el grupo "Centros de Control", haga clic en "Seguimiento de Proyectos".

2. Identifíquese para iniciar la Centro de Seguimiento de Proyectos con su identificador y contraseña. Para crear contribuidores, usted debe ser un administrador. Por Default, el usuario "Admin" se crea automáticamente al instalar la base de datos de los Centros control.

3. Despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=CC_Suivi_Configuration%20-%20HC%20N%B0001.gif) y seleccione "Colaboradores".

4. Cree los diferentes contribuidores a través del botón "Nuevo contribuidor. Para obtener más información, consulte [Gestión de colaboradores](../CCSuivi/3540703.md).




Veremos más adelante (paso 4) cómo los contribuyentes pueden ser associated con los proyectos de desarrollo.

## Etapa 3: Definir los requisitos
<a name="etapa_3_definir_los_requisitos_ELTTEXTE000385"></a>
Los requisitos se ponen a disposición del jefe de proyecto para gestionar el desarrollo del proyecto.. ¿Qué es un requisito? Un requisito es una necesidad con respecto a una acción que debe ser realizada por un producto.

Tomemos el caso de una aplicación WINDEV: una necesidad corresponderá a una nueva característica (capacidad de mostrar estadísticas en la aplicación de ventas, por ejemplo). Esta nueva característica agrupa varios elementos:

- Una o más tareas de desarrollo, para incluir la característica en la aplicación.

- Una o más tareas de prueba para ejecutar la prueba de características.

- Una o más tareas de Document ATION a Document la característica.




Esta característica (o requisito) también puede ser relacionado a una o más reglas de negocio o a uno o más incidentes.

Los requisitos se crean y se gestionan desde Centro de Seguimiento de Proyectos. Para obtener más información, consulte [Creación de necesidades](../CCSuivi/3540717.md).

## Etapa 4: Definición de los proyectos WINDEV
<a name="etapa_4_definicion_los_proyectos_windev_ELTTEXTE000409"></a>
Un proyecto de desarrollo puede corresponder a uno o más proyectos WINDEV, WEBDEV o WINDEV Mobile.. Estos proyectos deben estar referenciados en la Centro de Seguimiento de Proyectos.

Para comprobar si los proyectos están referenciados en el sistema Centro de Seguimiento de Proyectos:

1. Inicie la Centro de Seguimiento de Proyectos desde WINDEV, WEBDEV o WINDEV Mobile: en la pestaña "Herramientas", en el grupo "Centros de Control", haga clic en "Seguimiento de Proyectos".

2. Comprobar la presencia del proyecto de desarrollo en la Centro de Seguimiento de Proyectos: expanda ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=CC_Suivi_Configuration%20-%20HC%20N%B0001.gif) y seleccione "Proyectos".. Su proyecto se puede mostrar en un grupo de proyectos, o en la lista "&lt;No asignado a un grupo&gt;".. Puede escribir el nombre del proyecto buscado y hacer clic en el botón de la lupa.




Si el proyecto no existe, puede hacerlo:

- Crear un nuevo proyecto de desarrollo. Este proyecto puede relacionarse o no a un proyecto WINDEV (WDP), WEBDEV (WWP) o WINDEV Mobile (WPP.

- Importar uno o más proyectos WINDEV, WEBDEV o WINDEV Mobile como proyecto de desarrollo.




Para obtener más información, consulte [Creación de un proyecto en Centro de Seguimiento de Proyectos](../CCSuivi/3540702.md).
<a name="NOTE5_2"></a>


### Asociar a los colaboradores a un proyecto de desarrollo
<a name="asociar_los_colaboradores_proyecto_desarrollo_ELTPARAGRAPHE000153"></a>

La asociación entre los contribuyentes que se encuentran en la base de datos de los Centros control y un proyecto de desarrollo puede realizarse a través de la base de datos Centro de Seguimiento de Proyectos.

1. Inicie la Centro de Seguimiento de Proyectos desde WINDEV, WEBDEV o WINDEV Mobile: en la pestaña "Herramientas", en el grupo "Centros de Control", haga clic en "Seguimiento de Proyectos".

2. Encuentre su proyecto de desarrollo: expanda ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=CC_Suivi_Configuration%20-%20HC%20N%B0001.gif) y seleccione "Proyectos".. Puede escribir el nombre del proyecto buscado y hacer clic en el botón de la lupa.

3. Seleccione su proyecto de desarrollo y haga clic en el botón "Colaboradores del proyecto".

4. Asociar a los colaboradores o a los equipos con el proyecto en curso. Usted tiene la capacidad de especificar el papel de cada colaborador o equipo.




**Observación**: Debe definirse al menos un gestor para el proyecto.. El gerente tiene todos los derechos sobre el proyecto, sobre los colaboradores y sobre las tareas que associated con el proyecto.

## Paso 5: Definir las tareas y asociarlas a los requisitos y a los desarrolladores
<a name="paso_5_definir_las_tareas_asociarlas_los_requisitos_los_desarrolladores_ELTTEXTE000441"></a>
Un requisito correspondiente a una característica, debe ser associated con las diferentes tareas y con los diferentes contribuyentes utilizados para realizar el requisito.

Por ejemplo, el requisito "Muestra las estadísticas de ventas" puede verse afectado:

- Una o más tareas de desarrollo:

	- El desarrollador X que trabaja en la aplicación WINDEV y que debe crear la ventana para mostrar las estadísticas.

	- El desarrollador Y que trabaja en el sitio WEBDEV correspondiente.

	- El desarrollador Z que debe desarrollar el Process para el cálculo de las estadísticas.




- Una tarea de prueba para validar el cálculo y la visualización de las estadísticas.

- Una tarea de documentación para visualizar la ayuda correspondiente a las estadísticas visualizadas.

- Una sugerencia hecha por un usuario a get las estadísticas.




La asociación entre las tareas y los requisitos se realiza en el sistema Centro de Seguimiento de Proyectos.
<a name="NOTE6_2"></a>


### Asociar una tarea a una necesidad
<a name="asociar_una_tarea_una_necesidad_ELTPARAGRAPHE000201"></a>

Para asociar una tarea a una necesidad: 

1. Inicie la Centro de Seguimiento de Proyectos desde WINDEV, WEBDEV o WINDEV Mobile: en la pestaña "Herramientas", en el grupo "Centros de Control", haga clic en "Seguimiento de Proyectos".

2. Visualizar o crear una ficha "Requisitos. 

3. Seleccione el requisito solicitado. El menú contextual se utiliza para:

	- vincular la necesidad a una nueva tarea o a una tarea existente.

	- vincular el requisito a un nuevo incidente o a un incidente existente.

	- vincular la necesidad a una nueva norma empresarial o a una norma empresarial existente.







Observación: Al crear una tarea relacionado a la necesidad, no olvide especificar:

- el tipo de tarea (desarrollo, prueba o documentación)

- El contribuyente,

- El proyecto associated con la tarea.




Para obtener más información, consulte [Asignación de las necesidades](../CCSuivi/3540718.md).

## Paso 6: Seguir el progreso del proyecto
<a name="paso_6_seguir_progreso_del_proyecto_ELTTEXTE000471"></a>
El progreso del estado de un proyecto de desarrollo compuesto por varios requisitos puede ser seguido por el sistema Centro de Seguimiento de Proyectos.
<a name="NOTE7_2"></a>


### Seguimiento de un proyecto a través de Centro de Seguimiento de Proyectos
<a name="seguimiento_proyecto_traves_centro_seguimiento_proyectos_ELTPARAGRAPHE000246"></a>

La pestaña "Requisitos" ofrece al gestor una visión global del progreso de una necesidad o grupo de necesidades.. 

Se utilizan varios filtros para seleccionar los requisitos a mostrar. 

La opción "Modo de visualización" que se encuentra en la cinta del panel se utiliza para get: 

- La lista de requisitos (modo de visualización Default). 

- El cuadro de necesidades: Varios cuadros y gráficos muestran el número de requisitos por departamento, por contribuyente, así como su estado.

- El cuadro de tareas: Se utilizan varios gráficos para visualizar el número de tareas afectadas por las necesidades y su progreso.

- El cuadro de incidencias: Se utilizan varios gráficos para mostrar el número de incidentes afectados por los requisitos, así como su progreso.

- Elementos del proyecto: La lista de elementos de proyecto afectados por los requisitos (elementos de un proyecto WINDEV afectados por una tarea o por las reglas de negocio, por ejemplo).




Los tres "Gráficos" pueden ser visualizados de acuerdo a diferentes modos (General, Desarrollo, Prueba y Documentación). El modo Desarrollo, el modo Prueba y el modo Documentación son relacionado al tipo de tarea. Los coeficientes de complejidad también se pueden tener en cuenta. Estos coeficientes se definen a nivel de necesidad para cada tipo de tarea.

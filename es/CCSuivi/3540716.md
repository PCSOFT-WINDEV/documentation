## Presentación
<a name="presentacion_ELTTEXTE000108"></a>
Para gestionar el ciclo de vida del desarrollo del proyecto, Centro de Seguimiento de Proyectos le permite aplicar los requisitos.

Un requisito es una necesidad sobre una acción que debe realizar un producto o un servicio.

Tomemos el caso de una aplicación WINDEV: una necesidad corresponderá a una nueva característica (capacidad de mostrar estadísticas en la aplicación de ventas, por ejemplo). Esta nueva característica agrupa varios elementos:

- Una o más tareas de desarrollo, si incluyen la característica en la aplicación.

- Una o más tareas de prueba para ejecutar la prueba de características.

- Una o más tareas de Document ATION a Document la característica.




Esta característica (o requisito) también puede ser relacionado a una o más reglas de negocio o a uno o más incidentes.

Una vez que todos los elementos de relacionado al requerimiento son completados, el requerimiento es validado.. En nuestro ejemplo, la función puede ponerse a disposición de los usuarios a través de una actualización de la aplicación.

Observación: Puede indicar qué miembro del equipo es el **gerente del requerimiento**. En ese caso: 

- el gerente no puede set una tarea como "completada" hasta que las tareas de los otros miembros se completen. 

- se notificará al responsable cada vez que un miembro del equipo complete su tarea.. 




Por lo tanto, la Centro de Seguimiento de Proyectos cubre todo el ciclo de desarrollo, desde la idea inicial de la característica hasta su implementación.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000132"></a>
Se deben realizar los pasos siguientes para implementar las necesidades:

1. [Creación de una lista de necesidades](../CCSuivi/3540717.md). Esta lista puede agrupar los requisitos necesarios para una nueva versión de software.

2. [Creación de las necesidades](../CCSuivi/3540717.md).

3. [Asignación de las diferentes relaciones a necesidades](../CCSuivi/3540718.md)(tareas, reglas de negocio, etc.).

4. [Supervisión de los requisitos](../CCSuivi/3540718.md).



<a name="NOTE2_2"></a>


### Inicio de la herramienta para la gestión de las necesidades
<a name="inicio_herramienta_para_gestion_las_necesidades_ELTPARAGRAPHE000057"></a>Los requisitos se gestionan en el sistema Centro de Seguimiento de Proyectos. La herramienta para gestionar las necesidades también se puede iniciar directamente.. Para ello, ejecute el ejecutable de Centro de Seguimiento de Proyectos (CCSuivi.exe) pasando el siguiente comando Line:


```txt
CCSuivi.exe /MODE=REQUIREMENT
```

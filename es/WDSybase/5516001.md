
## Conector Nativo Sybase: Programación con funciones SQL
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000127"></a>
El uso de funciones SQL para los datos de la manipular Sybase no requiere la importación de la estructura de archivos en el análisis. Sin embargo, los datos sólo deben tratarse mediante consultas.. El contenido de los archivos de Sybase no se puede leer directamente.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Programación
<a name="programacion_ELTTEXTE000151"></a>


### 1. Establecimiento de la conexión
<a name="1_establecimiento_conexion_ELTPARAGRAPHE000018"></a>

[SQLConnect](../WDLang4/3072005.md) abre una conexión a una base de datos Sybase a través de Conector Nativo. El tipo de base de datos utilizado (SYBASE en este caso) debe pasarse como parámetro a [SQLConnect](../WDLang4/3072005.md).

El nombre de la base de datos corresponde al nombre del servidor Sybase (o a su alias).


```wl
ConnectionNum = SQLConnect("SybaseDatabase", "sa", "", "", "SYBASE")
```

<a name="NOTE2_2"></a>


### 2. Ejecutar las consultas
<a name="2_ejecutar_las_consultas_ELTPARAGRAPHE000034"></a>

Las consultas son realizadas por [SQLExec](../WDLang4/3072007.md). [SQLExec](../WDLang4/3072007.md) espera el texto de la consulta SQL como parámetro, así como el nombre que se le dará a esta consulta.


```wl
SQLExec("SELECT LASTNAME, FIRSTNAME, EXTENSION, PHOTO FROM CUSTOMER", "QRY1")
```

<a name="NOTE2_3"></a>


### 3. Recuperación del resultado
<a name="3_recuperacion_del_resultado_ELTPARAGRAPHE000049"></a>

Se pueden utilizar varios métodos para recuperar el resultado:

- [SQLFirst](../WDLang4/3072017.md): Recupera el resultado e inicia una navegación en [SQLFirst/SQLCol](../WDLang4/3072025.md).

- [SQLTable](../WDLang4/3072031.md): Recupera el resultado de la consulta en una control Tabla poblada programáticamente en la aplicación WINDEV o en el sitio WEBDEV. El resultado de la consulta puede ser recuperado por Partial Fetch.

- [SQLFetch](../WDLang4/3072011.md): Se utiliza para navegar por las diferentes líneas del resultado de la consulta. La navegación realizada en el resultado de la consulta es una navegación [SQLFetch/SQLGetCol](../WDLang4/3072025.md). 




[SQLLock](../WDLang4/3072003.md) se utiliza para bloquear los registros de tabla o los registros seleccionados por la consulta.
<a name="NOTE2_4"></a>


### 4. Cierre de la conexión
<a name="4_cierre_conexion_ELTPARAGRAPHE000076"></a>

[SQLClose](../WDLang4/3072015.md) se utiliza para cerrar la conexión con la base de datos una vez que se han ejecutado todas las consultas necesarias.

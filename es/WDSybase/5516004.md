## Principales observaciones
<a name="principales_observaciones_ELTTEXTE000419"></a>


### Versión necesaria
<a name="version_necesaria_ELTPARAGRAPHE000018"></a>

**Atención**: Esta versión del Conector Nativo Sybase funciona con WINDEV 28 y/o WEBDEV 28.
<a name="NOTE2_2"></a>


### Elementos Unicode
<a name="elementos_unicode_ELTPARAGRAPHE000029"></a>

El conector nativo de DB2 no admite elementos de Sybase. Le aconsejamos que utilice elementos de texto en formato Ansi y que realice la Conversion( si es necesario) con [UnicodeToAnsi](../WDLang1/3024025.md).

[HCreation](../WDLang4/3044255.md) mostrará un mensaje de error al crear archivos que contengan elementos en formato Unicode.
<a name="NOTE2_3"></a>


### Código Page
<a name="codigo_page_ELTPARAGRAPHE000043"></a>

El código de página (charset) utilizado por Native Sybase Connector es cp850 de Default. Si el servidor utiliza un código Page diferente, puede configurar el juego de caracteres utilizado por Conector Nativo.

El código Page (charset) utilizado por el cliente Sybase:

- en WDSYBINS.INI (que se encuentra en el directorio "C:\\Windows"):
	Especifique el conjunto de caracteres a utilizar en la entrada 'CHARSET' de la sección correspondiente al nombre del servidor. Por ejemplo:
	
	```txt
	[MARS]
	CHARSET=iso_1
	```


- en la información de conexión ampliada: CHARSET=iso_1



<a name="NOTE2_4"></a>


### Un fichero se bloquea en una operación
<a name="fichero_bloquea_una_operacion_ELTPARAGRAPHE000059"></a>

La función [HLockFile](../WDLang4/3044153.md) inicia una transacción y bloquea la tabla en modo exclusivo. La función [HUnlockFile](../WDLang4/3044160.md) valida la transacción y desbloquea la tabla.
<a name="NOTE2_5"></a>


### Especificar el tiempo de espera de ejecución de las consultas
<a name="especificar_tiempo_espera_ejecucion_las_consultas_ELTPARAGRAPHE000072"></a>

Para especificar el tiempo de espera de ejecución de las consultas:

- en modo SQL ([SQLExec](../WDLang4/3072007.md)), debe introducir la siguiente Line en la entrada "[ OPTIONS]" del archivo "WDSYBINS.INI" que se encuentra en el directorio de Windows: 
	
	```txt
	TIMEOUT = <Timeout in seconds>
	```
Este tiempo de espera es set a 60 segundos por Default.
	Para una espera interminable, entra en la siguiente Line:
	
	```txt
	NOWAIT = Y
	```


- con las funciones HFSQL ([HAdd](../WDLang4/3044147.md), [HExecuteQuery](../WDLang4/3044080.md), etc.), basta con indicar este tiempo de espera en la ventana description de la conexión utilizada.




También tiene la posibilidad de especificar el tiempo de espera de ejecución de las consultas mediante las opciones "WD Connect Timeout" y "WD Command Timeout" de [HDescribeConnection](../WDLang4/3044205.md) y [HOpenConnection](../WDLang4/3044107.md).
<a name="NOTE2_6"></a>


### INSERTAR consultas e identificador de automatic
<a name="insertar_consultas_identificador_automatic_ELTPARAGRAPHE000112"></a>

Los identificadores automatic no son compatibles con las consultas INSERT (creadas mediante programación o en el editor de consultas). Para utilizar un identificador automatic, llame a [HAdd](../WDLang4/3044147.md) en el tabla.
<a name="NOTE2_7"></a>


### Número de registro
<a name="numero_registro_ELTPARAGRAPHE000122"></a>

No se puede especificar un número de registro distinto de cero en las funciones [HWrite](../WDLang4/3044092.md) y [HRead](../WDLang4/3044047.md). "Cero" corresponde al número de registro actual.

Del mismo modo, las funciones [HLockRecNum](../WDLang4/3044152.md) y [HUnlockRecNum](../WDLang4/3044194.md) solo pueden utilizarse en el registro actual.

## Principales características específicas
<a name="principales_caracteristicas_especificas_ELTTEXTE000479"></a>


### Opciones de análisis
<a name="opciones_analisis_ELTPARAGRAPHE000145"></a>

- Las opciones de análisis relativas a los índices de texto completo no están soportadas por Conector Nativo.

- Las opciones de análisis relativas a la indexación de los memorandos de texto de Partial no están soportadas por el Conector Nativo.

- No importa si las opciones "Sensible a mayúsculas", "Sensible a acentos" y "Sensible a la puntuación" están marcadas en el editor de análisis , ***la búsqueda siempre será sensible a las mayúsculas y minúsculas, a los caracteres acentuados y a la puntuación.***. 



<a name="NOTE3_2"></a>


### Campos de tipo Array
<a name="campos_tipo_array_ELTPARAGRAPHE000154"></a>

Los artículos de Array no están soportados por el Conector Nativo. Si está utilizando un elemento array, sólo el valor del índice 1 en la array será manejado por las funciones HFSQL (lectura/escritura).
<a name="NOTE3_3"></a>


### Contexto HFSQL independiente
<a name="contexto_hfsql_independiente_ELTPARAGRAPHE000161"></a>

La gestión de los contextos independientes de HFSQL (casilla de verificación en la description de ventanas o informes) no está disponible en esta versión.
<a name="NOTE3_4"></a>


### Registros idénticos
<a name="registros_identicos_ELTPARAGRAPHE000168"></a>

El Native Sybase Connector no puede diferenciar entre dos registros idénticos (es decir, con valores idénticos para todos los artículos, excluyendo los artículos memo).

Cuando se modifica o elimina una Record, se modifican o eliminan todas las Record idénticas.
<a name="NOTE3_5"></a>


### Bloqueo de registros
<a name="bloqueo_registros_ELTPARAGRAPHE000177"></a>

Las opciones de bloqueo (***hLockWrite*** y ***hLockReadWrite***) se ignoran en las funciones de lectura (**HReadXXX**, **HReadSeekXXX**). Sin embargo, estas opciones de cierre pueden ser utilizadas por [HAdd](../WDLang4/3044147.md) y [HModify](../WDLang4/3044042.md).

Si se añade o modifica un Record con una opción de bloqueo, este Record se bloquea:

- para todos los demás usuarios

- para el usuario que solicitó la adición o la modificación si se ha llamado a [HReadFirst](../WDLang4/3044051.md) y [HReadSeekFirst](../WDLang4/3044036.md) en este Record. Si se llama a [HRead](../WDLang4/3044047.md) en este mismo Record, el Record no se bloquea para el usuario que solicitó la adición o la modificación.




**Observación**: Para desbloquear la Record (para todos los usuarios), utilice [HUnlockRecNum](../WDLang4/3044194.md)(hCurrentRecNum).

| Limitaciones | Funciones afectadas |
| --- | --- |

| La constante *hDistinct* no se tiene en cuenta: todos los registros (incluyendo los duplicados) serán examinados. | [HForward](../WDLang4/3044154.md), [HBackward](../WDLang4/3044126.md),<br>[HReadPrevious](../WDLang4/3044048.md), [HReadNext](../WDLang4/3044037.md)<a name="?"></a><br>, [HNext](../WDLang4/3044027.md) |
| **Consulta de archivos múltiples.** Las funciones **HAdd**, **HModify** y **HDelete** no están soportadas. | [HAdd](../WDLang4/3044147.md)<br>[HModify](../WDLang4/3044042.md)<br>[HDelete](../WDLang4/3044018.md) |

## Observaciones específicas de la base de datos Sybase
<a name="observaciones_especificas_base_datos_sybase_ELTTEXTE000551"></a>


### Columna de agregados
<a name="columna_agregados_ELTPARAGRAPHE000289"></a>

No se puede utilizar una columna agregada (SUM, MIN, etc.) en una cláusula WHERE (en el editor de consultas, en [HFilter](../WDLang4/3044100.md), [HReadSeek](../WDLang4/3044050.md), [HExecuteSQLQuery](../WDLang4/3044084.md) y [SQLExec](../WDLang4/3072007.md)).

**Solución alterna**: Para usar una columna de agregados, DONDE debe ser reemplazada por HAVING GROUP BY.
<a name="NOTE5_2"></a>


### Columna IDENTIDAD
<a name="columna_identidad_ELTPARAGRAPHE000310"></a>

En modo SQL ([SQLExec](../WDLang4/3072007.md) o [HExecuteSQLQuery](../WDLang4/3044084.md)), para insertar un valor en una columna IDENTITY, debe:

- escriba la siguiente sintaxis delante de la consulta: set IDENTITY_INSERT  ON

- escriba la siguiente sintaxis después de la consulta: set IDENTITY_INSERT  OFF




Por ejemplo, para insertar el valor "12" en la columna "CustomerID" de "Cliente" tabla, utilice la siguiente sintaxis:


```wl
SQLExec("SET IDENTITY_INSERT Customer ON INSERT INTO Customer(CustomerID)" + ...
		" VALUES (12) SET IDENTITY_INSERT Customer OFF", "Qry1")
```
<a name="NOTE5_3"></a>


### Sensibilidad a mayúsculas y minúsculas para los nombres de tablas y columnas
<a name="sensibilidad_mayusculas_minusculas_para_los_nombres_tablas_columnas_ELTPARAGRAPHE000334"></a>

Los nombres de tabla y los nombres de columna distinguen entre mayúsculas y minúsculas:

- en las consultas realizadas por [HExecuteSQLQuery](../WDLang4/3044084.md) o [SQLExec](../WDLang4/3072007.md).

- en el parámetro "Condición de selección" de [HFilter](../WDLang4/3044100.md).

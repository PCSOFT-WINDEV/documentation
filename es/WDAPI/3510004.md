<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000123"></a>
Las dos características más interesantes de WDAPI son:

- la posibilidad de consultar la lista de funciones, estructuras y constantes de Windows API.

- la capacidad de get la correspondiente sintaxis WLanguage para las funciones y las constantes.




Estas funciones están disponibles en los paneles "Funciones", "Estructuras" y "Constantes" de WDAPI.

**Recordatorio**: WDAPI se puede iniciar desde WDTool: en la pestaña "Herramientas", en el grupo "Utilidades", haga clic en "WDTool".

## Consulte la lista de funciones
<a name="consulte_lista_funciones_ELTTEXTE000147"></a>
Para encontrar una función, una constante o una estructura:

1. Posición en el panel deseado ("Funciones", "Estructuras" o "Constantes").

2. Haga clic en la lupa situada en la cabecera de la columna "Nombre de la función" (o "Nombre de la estructura" o "Nombre de la constante").

3. Escriba las primeras letras del nombre de la función, constante o estructura. El tabla se posiciona automáticamente sobre el elemento cuyo nombre es el más cercano al escrito.




**Ejemplo**: Cuando escriba "Mover" en el panel "Funciones", la lista se situará en las funciones cuyo nombre empiece por "Mover"..<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAPI%20-%20HC%20N%B0001.gif)

## Generación y uso del código WLanguage necesario para usar un API de Windows
<a name="generacion_uso_del_codigo_wlanguage_necesario_para_usar_api_windows_ELTTEXTE000173"></a>
Generar y utilizar el código WLanguage correspondiente al API de Windows:

1. Seleccione la función o estructura de Windows API que se utilizará en WLanguage.

2. WDAPI genera automáticamente el código WLanguage necesario para utilizar la función o estructura API..
	Si la función API requiere el uso de una o más estructuras, WDAPI genera automáticamente el código WLanguage correspondiente a la declaración de las estructuras utilizadas.
	Si desea gestionar los errores, seleccione "Añadir el código WLanguage para mostrar un mensaje si se produce un error".

3. Haga clic en el botón "Copiar" associated con la sintaxis WLanguage.

4. En su proyecto WINDEV, seleccione el Process que debe llamar a la función API de Windows.

5. En el editor de código, seleccione "Pegar" en el menú contextual (o pulse Ctrl + V).

6. Adaptar el código copiado a su aplicación: inicializar las variables de acuerdo a su programa.

7. Si se requieren estructuras, el código de declaración de estas estructuras debe copiarse en su código WLanguage.. Para ello:

	- Haga clic en el botón "Copiar" associated con el código de declaración de estructuras WLanguage.

	- En su proyecto WINDEV, seleccione el Process solicitado:

		- Inicialización Process del proyecto (recomendado).

		- declaración Process de las variables globales de la ventana.

		- declaración Process de las variables globales de set de los procedimientos en los que se realiza la llamada a la función API.




- En el editor de código, seleccione "Pegar" en el menú contextual (o pulse Ctrl + V).




**Caso raro**: Puede ocurrir que algunas características relacionado a una función API de Windows no sean interpretadas correctamente por WDAPI.. En este caso, la conversión debe ser realizada por usted mismo.

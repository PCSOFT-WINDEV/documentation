## Vocabulario relacionado a replicar
<a name="vocabulario_relacionado_replicar_ELTTEXTE000120"></a>
Antes de implementar la réplica, debe estar familiarizado con el vocabulario específico de la réplica.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Bases de datos
<a name="bases_datos_ELTTEXTE000144"></a>
La replicación distingue entre dos tipos de bases de datos:

- **La base de datos maestra**
	Es la base de datos de reference. Todas las actualizaciones se realizan en esta base de datos:

	- modificaciones realizadas por la aplicación ejecutada en este ordenador,

	- modificaciones realizadas en los ordenadores remotos y transmitidas por la replicación.




- **La base de datos de suscriptores**
	Esta base de datos remota es idéntica a la base de datos "Master. Las modificaciones realizadas por el ordenador remoto se aplican a esta base de datos. La réplica transmite estas modificaciones a la base de datos "master".




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Tipos de replicación
<a name="tipos_replicacion_ELTTEXTE000168"></a>
Se pueden implementar dos tipos de replicación:

- **Replicación unidireccional**
	Este tipo de replicación consiste en realizar una actualización desde la base de datos "Maestro" a las bases de datos "Suscriptor" o desde la base de datos "Suscriptor" a la base de datos "Maestro".

- **Replicación bidireccional**
	Este tipo de replicación consiste en realizar una actualización de la base de datos "Master" a las bases de datos "Subscriber" y de las bases de datos "Subscriber" a la base de datos "Master".. 
	Durante la replicación maestro-abonado, las modificaciones realizadas en el ordenador del abonado desde la última replicación se guardarán en el ordenador maestro.. 
	Durante la replicación del suscriptor maestro, las modificaciones realizadas en el equipo maestro desde la última replicación se guardarán en el equipo del suscriptor.

## Vocabulario de relacionado a WDReplic
<a name="vocabulario_relacionado_wdreplic_ELTTEXTE000192"></a>
WDReplic utiliza vocabulario específico:

- **El escenario de replicación**
	El escenario de replicación es un archivo SRP que contiene la set de las operaciones a realizar para construir la replicación. 
	Por ejemplo, para una réplica portátil entre la réplica maestra y la réplica de abonado, el escenario consiste en:

	- la ruta de la réplica maestra,

	- la ruta de la réplica portátil (archivo con extensión WDZ) que se debe crear,

	- el modo de transmisión de la réplica portátil y sus características (para una transmisión por FTP o correo electrónico),

	- el identificador del emplazamiento del abonado afectado por la réplica,

	- el modo de gestión de conflictos durante la replicación (si se han modificado dos registros en dos equipos, qué modificación se aplica).




- **Réplica móvil**
	Archivo en formato ZIP que contiene toda la información necesaria para la replicación. Este archivo se crea cuando el sitio maestro y el sitio del suscriptor son remotos y no están conectados.. 
	Durante una sincronización por FTP o Email, WDReplic envía el archivo WDZ a través del medio especificado.

- **Identificador del sitio esclavo**
	Cuando se instala una réplica de un abonado, esta réplica es associated con un identificador (numérico o cadena de caracteres). Este identificador se utiliza al crear un escenario de replicación utilizando medios portátiles.

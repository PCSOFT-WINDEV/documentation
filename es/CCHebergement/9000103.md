
## Importar desde versiones anteriores
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000091"></a>
Para instalar un sitio dinámico WEBDEV en un servidor web, necesitará cuentas implementación. La importación de cuentas de una versión anterior le permite utilizar estas cuentas en la versión actual.

La información del sitio, los servidores WebSocket y los servicios web SOAP y REST también deben ser migrados para poder ser utilizados con una versión posterior del motor WEBDEV. Por ejemplo, para utilizar un sitio creado en WEBDEV 27 con un motor WEBDEV 28, debe importar los detalles del sitio.

El Centro de Hosting control permite importar configuraciones de versiones anteriores: 

- Cuentas de despliegue,

- Sitios, 

- Servicios web SOAP,

- Servicios web REST,

- Servidores WebSocket. 




**Para importar elementos de versiones anteriores**: 

1. Iniciar el Centro control de alojamiento. 

2. En el grupo "Configuración", amplíe "Importar de versiones anteriores" y seleccione los elementos que desea importar.

3. Importar cuentas: 
	En la asistente que se abre, seleccione: 

	- La versión de WEBDEV de la que deben importarse los elementos,

	- La lista de cuentas de usuario a importar.




4. Sitios de importación, servicios web SOAP y REST, servidores WebSocket: 
	Seleccione los elementos que desea importar de la lista. Los sitios, servicios web (SOAP y REST) y servidores WebSocket que no hayan sido importados no funcionarán con una versión más reciente de WEBDEV Application Server. 
	 Observación: Un sitio creado con una versión anterior de WEBDEV puede utilizarse con un motor WEBDEV 28. Sin embargo, la información relacionado al sitio debe ser migrada. 

5. Confirmar la importación. 
	Ahora puede gestionar las cuentas importadas en el Centro control de Alojamiento.

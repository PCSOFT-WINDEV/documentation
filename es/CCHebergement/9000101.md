## Presentación
<a name="presentacion_ELTTEXTE000303"></a>
Los parámetros de hospedaje definen **los valores Default** que serán usados para crear las nuevas cuentas de hospedaje. Estos parámetros pueden ser modificados en cualquier momento, al crear una cuenta de usuario..  

Estos parámetros definen:

- los diferentes directorios utilizados por los sitios WEBDEV,

- los grupos de Windows,

- las cuentas de usuario,

- las bases de datos HFSQL Client/Server,

- los detalles del motor WEBDEV. 




Si es necesario, abra el Centro de Hosting control desde el menú de Inicio de Windows.

## Directorios
<a name="directorios_ELTTEXTE000327"></a>
Para configurar los directorios utilizados por el servidor: 

1. En el grupo "Hosting parameters", haga clic en "Directorios"..
	Observación: Las diferentes opciones de este grupo le permiten definir los parámetros que serán usados por Default para su servidor de alojamiento. 

2. Elija el directorio raíz en el que se crearán los subdirectorios de las cuentas de usuario WEBDEV.
	|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | Use un directorio local de el equipo. Si desea utilizar un directorio de red, debe especificarse necesariamente UNC ruta. El huésped del ordenador INTERNET debe tener acceso a este ruta sin tener que autenticarse. |





3. Especifique:  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Configuration%20CCH%E9bergement%20-%20HC%20N%B0001.gif&type=thumb)


	- el nombre del subdirectorio para sitios **WEBDEV** (por ejemplo, "Sitios").

- el nombre del subdirectorio para **SOAP Webservices** (por ejemplo, "SOAPWebservices").

- el nombre del subdirectorio para **REST Webservices** (por ejemplo, "RESTWebservices").

- el nombre del subdirectorio para los servidores **WebSocket** (por ejemplo, "WebSocketServers").

- el nombre del subdirectorio **data** para los sitios WEBDEV (por ejemplo, archivos de datos HFSQL Classic).

- el nombre del subdirectorio para los sitios estáticos (por ejemplo, "www").

- el nombre del subdirectorio de datos de instalación.

## Grupos Windows
<a name="grupos_windows_ELTTEXTE000369"></a>
Cada cuenta de alojamiento es associated con dos cuentas de usuario del sistema operativo:

- una cuenta para instalar y administrar los sitios.

- una cuenta para ejecutar los sitios.




Estas dos cuentas no tienen los mismos derechos.

Todas las cuentas de administración se pueden agrupar en un grupo de usuarios y todas las cuentas de tiempo de ejecución se pueden agrupar en otro grupo.

Puede especificar los nombres de los grupos que desea utilizar.

**Para definir los grupos**: 

1. En el grupo "Parámetros de alojamiento", haga clic en "Grupos de Windows".

2. Definir a qué grupos se asignarán los usuarios de Windows creados para el implementación. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P6_Configuration%20CCH%E9bergement%20-%20HC%20N%B0002.jpg&type=thumb)


	- Para el implementación, puedes crear un grupo o usar el grupo estándar de "Usuarios de Poder"..
			

- Una buena práctica para el funcionamiento de los sitios consiste en utilizar el grupo "IIS_IUSRS" (en las versiones de Windows donde existe).

3. Haga clic en "Aplicar" para confirmar. 




**Observación**: Se listan todos los grupos definidos en el sistema. Debe crear un grupo de usuarios con los derechos suficientes para que éste se muestre en la lista.

### Derechos de acceso para el uso de un sitio
<a name="derechos_acceso_para_uso_sitio_ELTPARAGRAPHE000081"></a>

Para el funcionamiento del sitio WEBDEV se necesitan varios derechos de acceso. Estos derechos afectan al usuario anónimo INTERNET del servidor (es decir, la cuenta de Windows que ejecuta WD28 0AWP.EXE). La tabla a continuación presenta un resumen de estos derechos.

| Directorio | Derechos |
| --- | --- |
| Directorio de datos | Lectura/Escritura |
| Directorio del sitio | indicar |
| Directorio WEB | indicar |
| Directorio WEBDEV | Leer/Ejecutar |





| Registro | Derechos |
| --- | --- |
| HKEY_LOCAL_MACHINE\\Software\\PC Soft\\WebDev\\28.0 | indicar |





| Ejecutable | Modo de ejecución |
| --- | --- |
| WD28 0AWP.EXE | Ejecución en modo anónimo usando la cuenta de huésped INTERNET |


<a name="NOTE3_3"></a>

<a name="derechos_acceso_para_realizar_una_instalacion_remota_modo_anonimo_ELTPARAGRAPHE000135"></a>

Se necesitan algunos derechos de acceso adicionales para instalar y actualizar una aplicación de forma remota en modo anónimo:

| Directorio | Derechos |
| --- | --- |
| Directorio base de la cuenta FTP utilizada para la instalación | Lectura/Escritura |
| Directorio de cuentas WEBDEV | Lectura/Escritura |
| Directorio de datos | Lectura/Escritura |
| Directorio del sitio | Lectura/Escritura |
| Directorio WEB | Lectura/Escritura |
| Directorio WEBDEV | Leer/Ejecutar |





| Registro | Derechos |
| --- | --- |
| HKEY_LOCAL_MACHINE\\Software\\PC Soft\\WebDev\\28.0\\Aplicaciones | Lectura/Escritura |





| Ejecutable | Modo de ejecución |
| --- | --- |
| WD28 0INSTAWP.EXE | Ejecución en modo autenticado utilizando la autenticación básica. Los derechos de ejecución deben ser concedidos a los usuarios de Windows que realizan la instalación (derechos idénticos a los de la cuenta FTP). |


<a name="NOTE3_4"></a>


### Derechos de acceso para realizar una instalación remota en modo autenticado
<a name="derechos_acceso_para_realizar_una_instalacion_remota_modo_autenticado_ELTPARAGRAPHE000195"></a>

Para instalar y actualizar una aplicación en modo autenticado, los derechos de acceso deben concederse a una segunda cuenta.

La cuenta debe ser una cuenta de Windows con los mismos parámetros que la cuenta FTP utilizada para instalación remota (automatic si el servidor FTP es IIS). Los derechos para asociarse a esta cuenta son los mismos que los de la Above tabla.

## Límites de la cuenta
<a name="limites_cuenta_ELTTEXTE000411"></a>
Puede set dos límites en una cuenta de usuario:

- el número máximo de conexiones asignadas a la cuenta. Este número de conexiones debe ser distribuido entre los diferentes sitios WEBDEV de la cuenta.  

- el número de sitios associated con una cuenta.




**A set los límites**: 

1. En el grupo "Parámetros de alojamiento", haga clic en "Límites". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P6_Configuration%20CCH%E9bergement%20-%20HC%20N%B0003.jpg&type=thumb)


2. Especifique: 

	- El número máximo de conexiones que se compartirán entre sitios (0 = ilimitado),

	- Si hay que limitar el número de sitios associated con una cuenta, etc.

## Base de datos HFSQL Client/Server
<a name="base_datos_hfsql_clientserver_ELTTEXTE000435"></a>
Para definir las características de los servidores HFSQL en modo Cliente/Servidor que pueden ser associated con una cuenta de usuario: 

1. En el grupo "Parámetros de alojamiento", haga clic en "Base de datos HFSQL Client/Server". El Centro de Hosting control permite definir los diferentes servidores HFSQL ubicados en el servidor. Al crear una cuenta de usuario, podrá seleccionar el servidor HFSQL en el que deben copiarse los datos de la aplicación. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P6_Configuration%20CCH%E9bergement%20-%20HC%20N%B0004.jpg&type=thumb)


2. Al crear un servidor HFSQL se pueden especificar las siguientes características:

	- description, 

	- Servidor IP Address,

	- puerto de conexión,

	- el nombre de usuario y la contraseña del administrador.







Observación: Tiene la posibilidad de crear o eliminar un servidor HFSQL.

Recordatorio: Esta información facilita la creación de cuentas de usuario (especialmente cuando se crean muchas cuentas). Son opcionales. La información que falta se puede introducir al crear la cuenta.

## Características del motor WEBDEV
<a name="caracteristicas_del_motor_webdev_ELTTEXTE000459"></a>
El centro de alojamiento de control le permite encontrar rápidamente las características del motor de WEBDEV:

- Directorio de instalación.

- Directorio de archivos para las cuentas de usuario.

- Directorio del módulo AWP.

- Información sobre la versión del motor.




Para acceder a esta información: 

1. Abra el Centro de Hosting control desde el menú de Inicio de Windows.

2. En el grupo "Parámetros de alojamiento", haga clic en "Información".

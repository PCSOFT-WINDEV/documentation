## Gestionar las cuentas de implementación
<a name="gestionar_las_cuentas_implementacion_ELTTEXTE000511"></a>


### Archivo de parámetros de la cuenta de despliegue
<a name="archivo_parametros_cuenta_despliegue_ELTPARAGRAPHE000030"></a>

Las cuentas de WEBDEV implementación se gestionan mediante un archivo que contiene los parámetros de la cuenta. Este archivo es un archivo de texto con cualquier extension. Este fichero tiene la siguiente estructura:


```txt
[MAIN]
NOM=Smith
PRENOM=Alan
SOCIETE=PCSOFT
CONNEXION=66
LOGINOS=Smith

[DIR]
FTP=d:\temp\ftp
APPLI=d:\AppliWW\Smith
FICHIER=d:\DataWW\Smith
```


Detalles de estos parámetros:

| Sección [ MAIN] |   |
| --- | --- |
| NOM | Apellido del usuario de la cuenta |
| PRENOM | Nombre del usuario de la cuenta |
| SOCIEDAD | Empresa del usuario de la cuenta |
| CONEXIÓN | Número total de conexiones permitidas para esta cuenta |
| Email | Enviar un correo electrónico a Address del usuario de la cuenta |
| DIRECCIÓN | Address del usuario de la cuenta |
| TELÉFONO | Phone número de cuenta de usuario |
| VARIOS | Notas adicionales |
| INTERDIT | <br><br>- 0 si la cuenta está habilitada, <br><br>- 1 si la cuenta está deshabilitada.<br><br><br> |
| ADMIN | <br><br>- 2 para una cuenta de administrador exclusiva de implementación,<br><br>- 1 para una cuenta de administrador de implementación,<br><br>- 0 para una cuenta implementación.<br><br><br> |
| LOGINOS | Nombre de la cuenta de Windows utilizada para la instalación (implementación) y la administración remota. La contraseña de associated es la que se utiliza para la cuenta de Windows. |
| LOGINOSIUSR | Nombre de la cuenta de Windows utilizada para ejecutar sitios y servicios web. La contraseña de associated es la que se utiliza para la cuenta de Windows. |
| PWDOS | Contraseña de la cuenta de Windows utilizada para la instalación (implementación) y la administración remota. **Esta contraseña solo se usa en el modo de clúster.** |
| PWDOSIUSR | Contraseña de la cuenta de Windows utilizada para ejecutar sitios y servicios web. **Esta contraseña solo se usa en el modo de clúster.** |
| LOGINOSIUSR | Nombre de la cuenta de Windows utilizada para ejecutar sitios y servicios web. La contraseña de associated es la que se utiliza para la cuenta de Windows. |
| **[ DIR ] sección** |   |
| FTP | Directorio utilizado para la transferencia por FTP. |
| APPLI | Directorio de aplicaciones y sitios WEBDEV associated con esta cuenta. |
| FICHIER | Directorio de archivos de datos para las aplicaciones y sitios WEBDEV associated con esta cuenta. |
| SERVICIO WEB | Directorio de los Webservices SOAP associated con esta cuenta. |
| WEBSERVICEREST | Directorio de los Webservices REST associated con esta cuenta. |
| SERVIDORESWEBSOCKET | Directorio de servidores Websocket associated con esta cuenta. |
| **Sección [HFCS1]** | Esta sección es opcional. Se utiliza para describir el servidor HFSQL cuando se utiliza una base de datos HFSQL Client/Server. |
| SERVIDOR | IP Address del servidor HFSQL |
| PUERTO | Puerto del servidor HFSQL |
| INICIO DE SESIÓN | Login del usuario de la base de datos HFSQL Client/Server |
| PASE | Contraseña del usuario de la base de datos HFSQL Client/Server |
| BASE | Base de datos HFSQL Client/Server |

### Creación de una cuenta WEBDEV implementación
<a name="creacion_una_cuenta_webdev_implementacion_ELTPARAGRAPHE000176"></a>

Para crear una nueva cuenta WEBDEV implementación mediante un comando Line y un archivo de parámetros: 

1. Crear un archivo con los parámetros de la cuenta a crear. 

2. Inicie CCHebergement con el siguiente comando Line: 
	
	```txt
	CCHebergement.exe /CREATEUSER /PARAM="<Full path of parameter file>"
	```





Observación: La contraseña es la que se utiliza para la cuenta de Windows.

Atención: El archivo de parámetros NO se elimina al final de la creación de la cuenta.
<a name="NOTE2_3"></a>


### Modificación de una cuenta WEBDEV implementación
<a name="modificacion_una_cuenta_webdev_implementacion_ELTPARAGRAPHE000194"></a>

Para modificar una cuenta WEBDEV implementación mediante un comando Line y un archivo de parámetros: 

1. Crear un archivo con los parámetros de la cuenta a modificar.  

2. Inicie CCHebergement con el siguiente comando Line: 
	
	```txt
	CCHebergement.exe /MODUSER /PARAM="<Full name of parameter file>"
	```





Los parámetros no especificados en el archivo de parámetros no se modifican.

**Atención**: El archivo de parámetros NO se elimina al final de la creación de la cuenta.
<a name="NOTE2_4"></a>


### Borrar una cuenta de implementación
<a name="borrar_una_cuenta_implementacion_ELTPARAGRAPHE000210"></a>

La siguiente sintaxis permite eliminar una cuenta mediante un comando Line: 

```txt
CCHebergement.exe /DELETEUSER /PARAM="<LOGINOS>"
```

<a name="NOTE2_5"></a>


### Leer la información sobre una cuenta de implementación
<a name="leer_informacion_sobre_una_cuenta_implementacion_ELTPARAGRAPHE000219"></a>

La siguiente sintaxis permite leer los detalles de la cuenta: 

```txt
CCHebergement.exe /INFOUSER /PARAM="<Full name of parameter file>" /USER="<LOGINOS>"
```


El archivo de parámetros se crea con la información de la cuenta de usuario identificada por su LOGINOS.
<a name="NOTE2_6"></a>


### Obtener la lista de cuentas de implementación
<a name="obtener_lista_cuentas_implementacion_ELTPARAGRAPHE000230"></a>

Para leer la información sobre las diferentes cuentas de implementación mediante un comando Line, utilice la siguiente sintaxis:


```txt
CCHebergement.exe /LISTUSER /PARAM="<Full name of parameter file>"
```


El archivo de parámetros se crea con la información sobre las diferentes cuentas de usuario.

## Migración de cuentas
<a name="migracion_cuentas_ELTTEXTE000649"></a>


### Migración de una cuenta de usuario
<a name="migracion_una_cuenta_usuario_ELTPARAGRAPHE000360"></a>

Para migrar una cuenta de usuario a través de un comando Line, utilice la siguiente sintaxis:


```txt
CCHebergement.exe /MIGREUSER [/VERSIONMIGRATION=<Version to migrate>] [/MIGRERESETUSER]
```


 corresponde al número de versión de la cuenta a migrar. 
Si este parámetro corresponde a \*, se migrarán todas las versiones anteriores, de la versión más reciente a la más antigua.. Si la cuenta ya existe, no se reemplaza. 
Si no se especifica este parámetro, sólo se migra la versión anterior a la última.

La palabra clave MIGRERESETUSER se utiliza para eliminar a los usuarios de la versión actual antes de la migración.. Esta palabra clave se utiliza principalmente para eliminar la cuenta "admin" creada automáticamente durante la instalación de instalación.<a name="NOTE5_2"></a>


### Migración de un sitio WEBDEV
<a name="migracion_sitio_webdev_ELTPARAGRAPHE000378"></a>

Para migrar un sitio WEBDEV a través de un comando Line, use la siguiente sintaxis:


```txt
CCHebergement.exe /MIGREAPP [/VERSIONMIGRATION=<Version to migrate>]
```


El parámetro &lt;Número de versión por migrar&gt; corresponde al número de versión del sitio que se va a migrar. 
Si este parámetro corresponde a \*, se migrarán todas las versiones anteriores, de la versión más reciente a la más antigua.. Si el centro ya existe, no se reemplaza.
Si no se especifica este parámetro, sólo se migra la versión anterior a la última.
<a name="NOTE5_3"></a>

Si no se especifica este parámetro, sólo se migra la versión anterior a la última.<a name="NOTE5_3B"></a>

 corresponde al número de versión del servidor WebSocket a migrar.

### Migrando todo
<a name="migrando_todo_ELTPARAGRAPHE000427"></a>

Para migrar usuarios, sitios y Webservices mediante un comando Line, utilice la siguiente sintaxis:


```txt
CCHebergement.exe /MIGREALL 
	[/VERSIONMIGRATION=<Version to migrate>] [/MIGRERESETUSER]
```


 corresponde al número de versión de la cuenta a migrar. 
Si este parámetro corresponde a \*, se migrarán todas las versiones anteriores, de la versión más reciente a la más antigua.. Si la cuenta, el sitio o el servicio web ya existe, no se reemplaza.. 
Si no se especifica este parámetro, sólo se migra la versión anterior a la última.

La palabra clave MIGRERESETUSER se utiliza para eliminar a los usuarios de la versión actual antes de la migración.. Esta palabra clave se utiliza principalmente para eliminar la cuenta "admin" creada automáticamente durante la instalación de instalación.

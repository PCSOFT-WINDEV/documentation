
## Conector nativo MariaDB para WINDEV y WEBDEV
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000223"></a>
WINDEV y WEBDEV 28 incluyen un Conector Nativo MariaDB (también llamado Acceso Nativo MariaDB). Este Conector Nativo le permite acceder a una base de datos MariaDB desde un programa WLanguage sin utilizar ningún controlador externo.

**Atención**: Esta versión del Conector Nativo MariaDB funciona con WINDEV y WEBDEV versión 28.

**Observación**: El conector nativo de MariaDB es un módulo opcional y gratuito que se puede descargar de [el sitio de PC SOFT](https://www.windev.com)

### Condiciones para los conectores MariaDB nativos
<a name="condiciones_para_los_conectores_mariadb_nativos_ELTPARAGRAPHE000030"></a>



- WINDEV 28 o WEBDEV 28.

- Capa de cliente MariaDB (archivo libmariadb.DLL)

	- Capa de cliente de 32 bits para utilizar el conector MariaDB nativo de 32 bits (versión 3.x.x a 10.x.x probada).

	- Capa de cliente de 64 bits para utilizar el conector MariaDB nativo de 64 bits (versión 10.x.x probada).




- El ***LibMariaDB.DLL*** DLL debe ser:

	- En el ordenador de desarrollo: 

		- en el directorio "\\Programas\\Framework\\Win32x86" del directorio instalación de WINDEV o WEBDEV (32 bits).

		- en el directorio "\\Programas\\Framework\\Win64x86" del directorio instalación de WINDEV o WEBDEV (64 bits).




- En el ordenador de implementación, en el directorio de los ejecutables o en el directorio de Windows. Para una implementación en un servidor Web, la DLL debe encontrarse en el directorio de la Conector Nativo o en el directorio de Windows.

### Archivos necesarios
<a name="archivos_necesarios_ELTPARAGRAPHE000056"></a>

Los siguientes archivos son necesarios para usar el Native MariaDB Connector:

- El ***LibMariaDB.DLL*** DLL. Esta DLL se distribuye con MariaDB. Para más detalles, consulte la licencia de MariaDB sobre las condiciones de distribución de esta DLL.

	- 32-bit para usar el Conector Nativo MariaDB de 32-bit.

	- Archivo de 64 bits para usar el Conector MariaDB nativo de 64 bits. 




- El ***wd28 0maria.dll*** DLL: Requerido para la Conector Nativo de 64 bits a una base de datos MariaDB. Este archivo se proporciona con el conector MariaDB nativo para WINDEV y/o WEBDEV.

- El ***wd28 0maria64.dll*** DLL: Requerido para la Conector Nativo de 64 bits a una base de datos MariaDB. Este archivo se proporciona con el conector MariaDB nativo para WINDEV y/o WEBDEV.




Para depurar un proyecto de multiconfiguración en configuración de 32 y 64 bits: 

- copy la DLL libmariadb de 32 bits a la subcarpeta EXE de la configuración del proyecto de 32 bits. 

- copy la DLL libmariadb de 64 bits a la subcarpeta EXE de la configuración del proyecto de 64 bits.

### Instalar una nueva versión de MariaDB
<a name="instalar_una_nueva_version_mariadb_ELTPARAGRAPHE000081"></a>

El diálogo entre Native MariaDB Connector y MariaDB utiliza la DLL LibMariaDB.DLL suministrada con MariaDB. 
La Conector Nativo puede dejar de funcionar si se modifica el comportamiento de esta DLL (al cambiar de versión, por ejemplo).

Pronto estará disponible una versión actualizada que tiene en cuenta estos cambios; mientras tanto, todo lo que tiene que hacer es usar la versión anterior de LibMariaDB.DLL.. Esta DLL funcionará con la nueva versión de MariaDB, siendo compatible con la Conector Nativo.

### Archivos necesarios para usar el Native MariaDB Connector en Linux
<a name="archivos_necesarios_para_usar_native_mariadb_connector_linux_ELTPARAGRAPHE000094"></a>

Para usar el Native MariaDB Connector en Linux, se deben realizar las siguientes operaciones:

**1.** Instalar la capa de cliente MariaDB en el servidor. Compruebe si la DLL ***libmariadbclient.so*** se ha copiado en /usr/lib.

Observación: 

- La capa de cliente MariaDB de 32 bits debe ser instalada para usar el conector MariaDB nativo de 32 bits.. 

- La capa de cliente MariaDB de 64 bits debe ser instalada para usar el Conector MariaDB nativo de 64 bits.. Si las DLLs copiadas son como libmariadbClient.so.12.0.0, se debe crear un enlace en esta DLL a través de la siguiente Line de código:





```txt
ln -s libmariadbclient.so.12.0.0 libmariadbclient.so
```


**2.** Si utiliza un sitio WEBDEV, copy las siguientes DLL en el directorio del servidor de aplicaciones WEBDEV:

- wd28 0hf.so: HFSQL library

- wd28 0maria.so: biblioteca de conectores nativos de 32 bits

- wd28 0maria64.so: biblioteca de conectores nativos de 64 bits

- wd28 0sql.so: SQL library




**3.** Si está usando una aplicación de Linux por lotes, copy las siguientes DLLs en el directorio de la aplicación:

- wd28 0hf.so: HFSQL library

- wd28 0maria.so: biblioteca de conectores nativos de 32 bits

- wd28 0maria64.so: biblioteca de conectores nativos de 64 bits

- wd28 0sql.so: biblioteca SQL

## aprovechamiento
<a name="aprovechamiento_ELTTEXTE000289"></a>


### Uso del conector nativo MariaDB en sus aplicaciones WINDEV y sitios WEBDEV
<a name="uso_del_conector_nativo_mariadb_sus_aplicaciones_windev_sitios_webdev_ELTPARAGRAPHE000142"></a>

Para utilizar Native MariaDB Connector, debe programar en la aplicación WINDEV o en el sitio WEBDEV:

- con las funciones "SQL" de WLanguage (véase [Programación con funciones SQL](../WDMariaDB/1000021529.md)).

- con las funciones "HFSQL" de WLanguage (véase [Programación con funciones HFSQL](../WDMariaDB/1000021528.md)).


**Observaciones**: 

- No se olvide de leer el documento [limitaciones con respecto al uso del Native MariaDB Connector.](../WDMariaDB/1000021521.md)

- La opción "Crear automáticamente los archivos de datos cuando se abran" (pestaña "Archivos" en el proyecto description) es tenida en cuenta por el Conector Nativo MariaDB. 





<a name="NOTE4_2"></a>


### Optimización de las funciones HFSQL
<a name="optimizacion_las_funciones_hfsql_ELTPARAGRAPHE000173"></a>

- **TLos controles "Browsing tabla" basados en consultas se optimizan**: el contenido de la control Tabla se puede ordenar haciendo clic en una de sus columnas.

- Para evitar volver a ejecutar la misma consulta varias veces al navegar por su resultado, le recomendamos utilizar la constante ***hNoRefresh*** (por ejemplo, si un solo equipo modifica los datos).


## WDAdminHF: Gestión de una base de datos
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000177"></a>
El administrador remoto HFSQL propone una gestión completa de las bases de datos HFSQL Client/Server. 

**Para visualizar las características de una base de datos HFSQL Client/Server**: 

1. Conectarse al servidor HFSQL solicitado a través del administrador remoto HFSQL. 

2. Seleccionar"Bases de datos". 

3. Seleccione la base de datos solicitada y haga clic en el enlace "Acceder a la base de datos seleccionada".. 

4. Las opciones mostradas son las opciones disponibles en la base de datos actual.  






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Detalles de las opciones disponibles
<a name="detalles_las_opciones_disponibles_ELTTEXTE000207"></a>


### Archivos
<a name="archivos_ELTPARAGRAPHE000026"></a>

Esta opción se utiliza para get información general sobre los archivos encontrados en la base de datos: nombre, tamaño y número de registros. También tienes la capacidad de borrar el el archivo de datos. 

Haciendo clic en el nombre del archivo de datos, tiene la posibilidad de acceder a las opciones de [gestión de los ficheros de datos](../WDAdminHF/1000019703.md). 


### Conexiones
<a name="conexiones_ELTPARAGRAPHE000036"></a>

Esta opción lista los diferentes usuarios conectados a la base de datos actual.

La lista de conexiones de Active contiene la siguiente información:

- el nombre de usuario, 

- El nombre el equipo,

- el nombre de la aplicación que accede a la base de datos, etc.




Usted tiene la capacidad de administrar las conexiones desde esta página: permitir o prohibir las conexiones, desconectar a los usuarios o enviar un mensaje a los usuarios.


### Bloqueos
<a name="bloqueos_ELTPARAGRAPHE000049"></a>

Esta opción se utiliza para gestionar los bloqueos. Se muestra la siguiente información:

- el nombre de del archivo de datos en el que se aplica la cerradura,

- el número de la Record cerrada,

- el nombre del usuario que está bloqueando,

- el nombre del ordenador de cierre,

- la IP Address del ordenador de bloqueo,

- el nombre de la aplicación de cierre,

- la fecha de inicio del bloqueo,

- la duración del bloqueo.





### copias
<a name="copias_ELTPARAGRAPHE000063"></a>

Esta opción se utiliza para realizar y restaurar las copias de seguridad de los archivos de datos.. Para obtener más información, consulte [Realización de copias de seguridad con el administrador remoto de HFSQL](../WDAdminHF/1000019704.md). 


### Derechos
<a name="derechos_ELTPARAGRAPHE000071"></a>

Esta opción se utiliza para gestionar los derechos en el servidor para el usuario o grupo seleccionado.


### Mantenimiento
<a name="mantenimiento_ELTPARAGRAPHE000076"></a>

Esta opción se utiliza para reindexar los archivos de datos que se encuentran en la base de datos.

Para realizar una operación de mantenimiento en los ficheros de datos de la base de datos:

1. En el tabla, verifique los ficheros de datos que deben procesarse.. Especifique (si es necesario) la contraseña de los archivos de Process.

2. Seleccione las opciones de reindexación: 

	- **Borrar los registros de la Inactive (borrados o tachados)**
			Si se selecciona esta opción, los registros cruzados se eliminan de forma permanente..
			Recordatorio: Cuando se cruza una Record, se borra lógicamente y se puede recuperar más tarde. El Record se encuentra todavía en el archivo de datos. La eliminación de los registros cruzados elimina estos registros de forma permanente y optimiza el tamaño del archivo de datos.

	- **Eliminar los registros dañados**
			Si se selecciona esta opción, los registros dañados se eliminan automáticamente.

	- **Densidad del índice**
			Corresponde a la tasa de densidad de los índices. Esta tasa es set a 80 por Default. 
			Una tasa más alta aumenta la densidad del índice y reduce su tamaño. En este caso, las navegaciones, búsquedas, filtros y consultas son más rápidas. Las adiciones de registros y las modificaciones de registros pueden ser más lentas.
			Una tasa menor reduce la densidad del índice y aumenta su tamaño.. En este caso, las navegaciones, búsquedas, filtros y consultas serán más lentas.. Las adiciones de registros y las modificaciones de registros serán más rápidas.

	- **Conjunto de caracteres**
			El personaje set del archivo de datos es mantenido por Default. Sin embargo, se puede realizar una operación de re-indexación cambiando el carácter set del archivo de datos. En este caso, el nuevo personaje set se tendrá en cuenta durante la operación de re-indexación. El orden, las búsquedas, ... en claves de tipo String (cadenas, caracteres, fecha y hora) se realizarán de acuerdo con este conjunto de caracteres.

	- **Intentar recuperar los datos del memo si este está dañando**
			Si se selecciona esta opción, la operación de reindexación intenta recuperar el memo. De lo contrario, los registros dañados se recuperan sin el memorándum de associated.
			Atención: El memorándum puede que no se haya recuperado por completo. Compruebe sus archivos de datos.




3. Seleccione el tipo de optimización a realizar: 

	- **1. Compruebe los índices**: Comprueba la coherencia entre el índice y los archivos de datos.

	- **2. Optimizar la velocidad de los índices**(recalcular las estadísticas): Optimiza los índices calculando estadísticas sobre los índices. Estas estadísticas se utilizan para optimizar los filtros, las consultas y las vistas HFSQL..
			Observación: Cuanto más se modifique el el archivo de datos, menos las estadísticas reflejarán el contenido del archivo de datos HFSQL. Cuanto más Record s el archivo de datos contiene, menos la modificación de un solo Record impacta las estadísticas generales.

	- **3. índice puro**: Optimiza el acceso a los registros del archivo de datos y recalcula las estadísticas. Todos los registros cruzados se borran automáticamente.

	- **4. Revisa los índices y los memos**: Optimiza todo el archivo de datos, su índice y sus notas.








### Procedimientos almacenados
<a name="procedimientos_almacenados_ELTPARAGRAPHE000118"></a>

Esta opción se utiliza para encontrar los conjuntos de procedimientos y los procedimientos almacenados.. Tiene la capacidad de borrar una set de procedimientos si es necesario. 


### Triggers
<a name="triggers_ELTPARAGRAPHE000123"></a>

Esta opción se utiliza para ver los triggers definidos en la base de datos y para añadir algunos si es necesario.. 


### acoplamientos
<a name="acoplamientos_ELTPARAGRAPHE000128"></a>

Esta opción se utiliza para ver los enlaces definidos en la base de datos y para borrar algunos si es necesario..


## WDAdminHF: Guardar las bases de datos
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000163"></a>
El administrador remoto de HyperFileSQL se utiliza para realizar copias de seguridad de los archivos de datos de HyperFileSQL Client/Server.. 

Esta copia de seguridad se puede realizar: 

- [desde el servidor](#NOTE2_1). La copia de seguridad de la base de datos o la copia de seguridad de todas las bases de datos del servidor se mantienen en el servidor y pueden ser restauradas.. 

- [de una base de datos](#NOTE3_1). La copia de seguridad de los archivos de datos se realiza en un archivo que se puede descargar al final de la operación..

<a name="NOTE2_1"></a>


## Copia de seguridad desde el servidor: ¿Cómo proceder?
<a name="copia_seguridad_desde_servidor_¿como_proceder_ELTTEXTE000193"></a>


### Realizar una copia de seguridad
<a name="realizar_una_copia_seguridad_ELTPARAGRAPHE000028"></a>

Para realizar una copia de seguridad:

1. Conectarse al servidor HFSQL solicitado. 

2. Seleccione "Copias de seguridad".

3. Haga clic en el botón "Nueva copia de seguridad. Comienza la asistente para crear una copia de seguridad.

4. Especifique el contenido de la copia de seguridad:

	- Todo el contenido del servidor.

	- Una base de datos específica.

	- Una selección de bases de datos y/ archivos.




5. Especifique si la copia de seguridad debe contener los archivos de índice (archivos de índice con extensión NDX y archivos de índice de texto completo con extensión FTX).. Si no se guardan los archivos índice, el tamaño de la copia de seguridad se reducirá, pero el tiempo de restauración será mayor.. 
	Observación: si los archivos de índice no se guardan, las contraseñas de los archivos deben especificarse durante la operación de restauración para permitir su reindexación.

6. Especifique el directorio de la copia de seguridad. Si este nombre corresponde a un ruta relativo, la copia de seguridad se realizará en el subdirectorio "Backup" del servidor HFSQL. El asistente se utiliza para definir un subdirectorio de "copia de seguridad" ("Backup" de Default). La fecha y la hora (en formato "Año\\Mes\\Día\\Tiempo-Minuto") se pueden añadir al nombre del directorio para identificar fácilmente la copia de seguridad..
	Observación: Las copias de seguridad se realizan en un subdirectorio numerado ("copia de seguridad\\Copia de seguridad\\1" por ejemplo. Por lo tanto, si se realizan varias copias de seguridad, los archivos correspondientes no serán sobrescritos por una nueva copia de seguridad.

7. Déle un description a su copia de seguridad y valídelo.

8. La copia de seguridad se inicia automáticamente. El progreso de la copia de seguridad se puede supervisar en el HFSQL control Center.




Observación: También tiene la posibilidad de realizar una copia de seguridad desde la opción "Copias de seguridad" de una base de datos.




### Realización de una copia de seguridad programada
<a name="realizacion_una_copia_seguridad_programada_ELTPARAGRAPHE000053"></a>

Para realizar una copia de seguridad programada:

1. Conectarse al servidor HFSQL solicitado. 

2. Seleccione "Tareas programadas".

3. Haga clic en el botón "Crear. Comienza la asistente para crear una tarea programada.

4. Especifique las opciones de programación: 

	- mes de ejecución: cada mes, algunos meses

	- día de ejecución: todos los días, algunos días de la semana o algunos días del mes

	- tiempo de ejecución: a una hora fija o durante el día de forma regular




5. Indique el contenido de la tarea programada: 

	- un Procedure almacenado

	- una copia de seguridad




6. Si elige iniciar un Procedure almacenado, especifique:

	- el nombre de la base de datos que contiene el Procedure almacenado para ejecutarlo,

	- el nombre del conjunto de Procedure s que contiene el Procedure almacenado para ejecutar,

	- el nombre de la Procedure a correr.




7. Valide el asistente. Se crea la tarea programada. 

8. Si decide iniciar una copia de seguridad, especifique el contenido de la copia de seguridad:

	- Todo el contenido del servidor.

	- Una base de datos específica.

	- Una selección de bases de datos y/ archivos.




9. Especifique si la copia de seguridad debe contener los archivos de índice (archivos de índice con extensión NDX y archivos de índice de texto completo con extensión FTX).. Si no se guardan los archivos índice, el tamaño de la copia de seguridad se reducirá, pero el tiempo de restauración será mayor.. Observación: si los archivos de índice no se guardan, las contraseñas de los archivos deben especificarse durante la operación de restauración para permitir su reindexación.

10. Especifique el directorio de la copia de seguridad. Si este nombre corresponde a un ruta relativo, la copia de seguridad se realizará en el subdirectorio "Backup" del servidor HFSQL. El asistente se utiliza para definir un subdirectorio de "copia de seguridad" ("Backup" de Default). La fecha y la hora (en formato "Año\\Mes\\Día\\Tiempo-Minuto") se pueden añadir al nombre del directorio para identificar fácilmente la copia de seguridad..
	Observación: Las copias de seguridad se realizan en un subdirectorio numerado ("copia de seguridad\\Copia de seguridad\\1" por ejemplo. Por lo tanto, si se realizan varias copias de seguridad, los archivos correspondientes no serán sobrescritos por una nueva copia de seguridad.

11. Déle un description a su copia de seguridad y valídelo.

12. La copia de seguridad se iniciará automáticamente de acuerdo con la programación especificada.. La copia de seguridad puede ser monitorizada en el administrador remoto de HFSQL.





### Operaciones en las copias de seguridad
<a name="operaciones_las_copias_seguridad_ELTPARAGRAPHE000088"></a>

El administrador remoto de HFSQL le permite manipular las diferentes copias de seguridad realizadas. Es posible:

- Realice una copia de seguridad.

- Reinicie una copia de seguridad.

- Restaurar una copia de seguridad.




Para realizar estas operaciones:

1. Conectarse al servidor HFSQL solicitado. 

2. Seleccione "Copias de seguridad".

3. Seleccione la copia de seguridad en manipular y haga clic en el botón correspondiente a la acción solicitada.

## Copia de seguridad de una base de datos: ¿Cómo proceder?
<a name="copia_seguridad_una_base_datos_¿como_proceder_ELTTEXTE000229"></a>
Para realizar una copia de seguridad:

1. Conectarse al servidor HFSQL solicitado. 

2. Seleccionar"Bases de datos". 

3. Seleccione la base de datos solicitada y haga clic en el enlace "Acceder a la base de datos seleccionada".. 

4. Seleccione "Copias de seguridad".

5. Especifique el nombre del archivo comprimido (archivo Zip) que contendrá los archivos de datos guardados.. 

6. Especifique si el fichero de archivo debe descargarse en el ordenador del usuario al final de la operación.. 

7. Seleccione los ficheros de datos que desea guardar. 

8. Haga clic en el botón "Guardar" para iniciar la copia de seguridad.




## Estaba usando Visual Basic con WINDEV 5.5. ¿Cómo proceder en WINDEV 28?
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000123"></a>
Usted tiene la habilidad de usar lenguajes externos con WINDEV. Para más detalles sobre las diferentes formas de utilizar elementos desarrollados en WINDEV con un lenguaje externo, véase [idiomas externos en WINDEV](../LangageExt/7510002.md).

Este párrafo presenta las operaciones necesarias para migrar una aplicación WINDEV 5.5 que utiliza Visual Basic a WINDEV 28.

**Observación**: Las llamadas DDE (a través de CallDDE) en modo de 16 bits ya no están disponibles en esta versión de la interfaz externa.


<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Gestión de objetos y archivos HFSQL en Visual Basic
<a name="gestion_objetos_archivos_hfsql_visual_basic_ELTTEXTE000153"></a>


### Para migrar una aplicación WINDEV 5.5 que gestiona los objetos en Visual Basic
<a name="para_migrar_una_aplicacion_windev_55_que_gestiona_los_objetos_visual_basic_ELTPARAGRAPHE000028"></a>

1. Migre su proyecto de WINDEV 5.5 a WINDEV 7.5 y ábralo con WINDEV 28.

2. Generar el esqueleto de su aplicación: en el editor de análisis, en la pestaña "Análisis", en el grupo "Análisis", amplíe "Generación" y seleccione "Generación avanzada". Atención: No sobreescriba el esqueleto utilizado por su aplicación

3. Recrear la biblioteca de su aplicación WINDEV: expanda la generación Icon y seleccione "Otras generaciones... Generar la biblioteca (.wdl)".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=genererbibliotheque.gif)


4. Borre los archivos WinDevxx.Bas y WDHFxx.Bas del directorio de sus códigos fuente (para evitar cualquier confusión).

5. En tu proyecto Visual Basic, borra el reference de dependencia a los archivos WDHF.bas y WINDEV.bas.

6. copy los siguientes archivos en el directorio de su aplicación.

	- WDHF.bas (se encuentra en el subdirectorio "ExternalLanguages\\Basic" de WINDEV).

	- WINDEV.Bas (se encuentra en el subdirectorio "ExternalLanguages\\Basic" de WINDEV).

	- &lt;Bas generado por WINDEV (encontrado en el directorio de análisis del proyecto WINDEV).

	- &lt;NombreAnálisis&gt;.gbl generado por WINDEV (encontrado en el directorio de análisis del proyecto WINDEV).




7. Añada estos 4 archivos al proyecto de Visual Basic.

8. Borrar las llamadas a CALLDDE desde su aplicación Visual Basic. Esta función sólo se puede utilizar en las aplicaciones en modo de 16 bits.

9. La gestión de los menús se puede realizar mediante uno de los siguientes métodos:

	- Recuperación de los accesos directos de los menús con WDKEY = "\*M\*":
			
		```txt
		While WDKey <> "ESC"
				' perform the input of the menu
				Call callwd("SCREEN,input")
				' the status report WDKey is set to *M* when a menu choice
				' was selected
				If (WDKey = "*M*") Then
					'---------------------------------------
					' Test of the selected option.
					' WDString contains the sequence of shortcut letters
					' that lead to the selected menu choice
					'---------------------------------------
					if( WDString="FQ" ) then WDKey = "ESC" ' Exit>
					If WDString = "RN" Then Call SearchCity  ' Search by City
					If WDString = "RD" Then Call SearchState  ' Search by State
					If WDString = "DD" Then Call LstState  ' Display list.
					If WDString = "DI" Then Call LstPrint ' Print.
					If WDString = "DC" Then Call LstConfig ' Configure printer
				End If
			Wend
		```


	- Recuperación de los accesos directos de los menús mediante la gestión directa del acceso directo de la opción:

		1. En el código de cada opción del menú de Process, asigna el atajo a la palabra clave **WDKey**. Por ejemplo, para Archivo...Abrir:
						
			```txt
			WDKey="FO";
			```


		2. Modifique el código de Visual Basic de la siguiente manera:
						
			```txt
			' the program loops until the File Exit option
					' is selected
					While WDKey <> "ESC"
						' perform the input of the menu
						Call callwd("SCREEN,input")
						'---------------------------------------
						' Test of the selected option.
						'---------------------------------------
						If WDKey = "FE" then WDKey = "ESC" ' Exit
						If WDKey = "RN" Then Call FindCity ' Find City
						If WDKey = "RD" Then Call FindState ' Find State
						If WDKey = "DD" Then Call ListState	' Display list.
						If WDKey = "DI" Then Call LstPrint ' Print.
						If WDKey = "DC" Then Call LstConfig ' Configure printer
					Wend
			```





10. Recompila tu proyecto de Visual Basic. 
	Atención: el archivo .Bas generado por WINDEV contiene un procedimiento "Main". Mantén tu propio "Principal" Procedure en su lugar.

11. copy las DLLs WINDEV necesarias en el directorio del ejecutable de su aplicación. Atención: wdxxxle.dll es obligatorio.




**Observación**: Para conocer la lista de DLLs necesarias, cree el ejecutable de su aplicación en WINDEV.

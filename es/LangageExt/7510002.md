## Principio
<a name="principio_ELTTEXTE000155"></a>
WINDEV también puede ser utilizado para crear elementos que serán administrados en los programas escritos en C, C++, Pascal, Visual Basic, Cobol,...

Todo lo que tiene que hacer es llamar los elementos WINDEV desde el idioma externo. Esta llamada puede realizarse desde un programa (archivo".EXE") o desde una biblioteca (archivo".DLL" u ".OCX") compilada en lenguaje externo.

Para llamar a un elemento WINDEV desde un lenguaje externo, utilice [Llame a CallWD](../LangageExt/7510001.md).

Para gestionar un archivo de datos HFSQL desde un lenguaje externo, utilice el programa [Funciones HFSQL de WLanguage](../WDLang4/3044156.md).

**Observación**: También tiene la capacidad de utilizar procesos escritos en lenguaje externo (como bibliotecas compiladas) en una aplicación WINDEV.. Para ello, debe compilar una biblioteca (archivo ".DLL" o ".OCX") en lenguaje externo y llamar a los procesos de esta biblioteca mediante la función [API](../WDLang1/3014005.md) o [CallDLL32](../WDLang1/3014003.md) de WLanguage.


**Atención**: WINDEV sólo se puede utilizar para compilar código WLanguage.

Los idiomas disponibles son los siguientes:

- [C y C++](../LangageExt/7513005.md)

- [Visual Basic](../LangageExt/7517001.md)

- [Pascal y Delfos](../LangageExt/7516001.md)

- [Cobol](../LangageExt/7511001.md)

- [Java](../LangageExt/7515001.md)

- [fortran](../LangageExt/7514001.md)

- [C#](../LangageExt/7512001.md)

## Implementación necesaria para utilizar una interfaz con un idioma externo
<a name="implementacion_necesaria_para_utilizar_una_interfaz_con_idioma_externo_ELTTEXTE000179"></a>
Para utilizar los elementos desarrollados en WINDEV con un lenguaje externo, seleccione "Lenguajes externos" al instalar WINDEV.

**Observación**: Los archivos requeridos por los "Idiomas externos" se instalan automáticamente durante una instalación completa de WINDEV.

**Atención**: Su compilador del lenguaje externo utilizado debe estar necesariamente instalado en el ordenador de desarrollo.

## Archivos necesarios para ejecutar el programa escritos en un lenguaje externo
<a name="archivos_necesarios_para_ejecutar_programa_escritos_lenguaje_externo_ELTTEXTE000203"></a>
Para ejecutar el programa escrito en lenguaje externo, debe tener:

- Las librerías WINDEV (archivos".DLL").

- Las bibliotecas específicas del lenguaje externo utilizado.

- Los programas ejecutables escritos en lenguaje externo.

- Las librerías de la aplicación WINDEV.

- El análisis associated con la aplicación WINDEV. Este análisis puede ser incluido en la biblioteca.



<a name="NOTE3_2"></a>


### Librerías WINDEV (archivos".DLL")
<a name="librerias_windev_archivosdll_ELTPARAGRAPHE000080"></a>

El programa escrito en lenguaje externo requiere las librerías WINDEV para funcionar (llamada a las funciones WLanguage, uso de objetos gráficos, ....). Estas librerías se listan cuando el ejecutable se crea en WINDEV. Consulte esta lista para conocer las diferentes bibliotecas utilizadas por los archivos de la aplicación WINDEV ("wdxxx\*.DLL").

Las bibliotecas listadas, así como las bibliotecas wdxxxle.dll y wdxxxcpl.dll deben copiarse en el directorio de tiempo de ejecución del programa escrito en lenguaje externo.

**Observaciones**:

- En Java, también se necesita para copy la biblioteca wdXXXIJV.DLL.

- En C#, también necesitas copy la biblioteca wdXXXICS.DLL.

- En Fortran y Cobol, también necesitas copy la biblioteca wdXXXFL.DLL.



<a name="NOTE3_3"></a>


### Bibliotecas de la aplicación WINDEV
<a name="bibliotecas_aplicacion_windev_ELTPARAGRAPHE000095"></a>

Las librerías de la aplicación WINDEV contienen todas las ventanas, informes, imágenes (y otros archivos) de la aplicación WINDEV.

**Observación**: Si las bibliotecas (archivo".WDL") no se encuentran al ejecutar el test de ejecución del programa en el editor del lenguaje externo, se debe especificar el ruta de las bibliotecas cuando se abren.

**Caso especial**: En Visual Basic, debe utilizar las siguientes funciones de Visual Basic: chDir y chDisk para especificar la ruta de las bibliotecas.

## Ejemplos suministrados con WINDEV
<a name="ejemplos_suministrados_con_windev_ELTTEXTE000239"></a>
WINDEV propone un ejemplo completo para cada idioma disponible. Estos ejemplos se encuentran en el subdirectorio "ExternalLanguages\\&lt;LanguageName&gt;" del directorio instalación de WINDEV.

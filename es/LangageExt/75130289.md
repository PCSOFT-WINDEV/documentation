
## Lista,get (Idioma externo)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Recupera el valor de un elemento dado o el valor del elemento seleccionado en una lista desplegable o en una lista ampliada.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```txt
// In C
CALLWD("List,Get,ProdList,3");
strcpy(Value,WDString);
```


<a name="Example2"></a>



```txt
// In Pascal
CALLWD('List,Get,ProdList,3');
Value:=WdString;
```


<a name="Example3"></a>



```txt
// In VB
call CALLWD("List,Get,ProdList,3")
Value=WdString
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`List,Get(<List name> [, <Subscript>])`
---

**`<List name>: (Cadena de caracteres)`**

Nombre de la lista a utilizar.

**`<Subscript>: (Cadena de caracteres opcional)`**

Índice del elemento que se va a recuperar.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si no se especifica &lt;Índice&gt;, el valor del elemento seleccionado se encuentra en WdString. La índice del elemento (si existe) está contenida en WdInt.
	[Lista,Seleccionar,?](../LangageExt/75130290.md) debe utilizarse para recuperar el índice del elemento seleccionado.

- Si no se selecciona ningún elemento en la lista o si la lista está vacía, **List, Get** devuelve -1 en WdInt.

- Si se especifica &lt;Índice&gt;, **List, get** devuelve en WdString el valor del elemento cuyo índice se especifica y devuelve en WdInt el tamaño del elemento.

- Para los controles Combo Box con campo de entrada, **List, Get** siempre devuelve WdString que contiene el valor del control Campo de entrada del Combo Box.

- WdInt no tiene sentido.


## Abierto (Lenguaje externo)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Abre una ventana.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```txt
// In C
CALLWD("Open,WIN_Window1.WDW");
```


<a name="Example2"></a>



```txt
// In Pascal
CALLWD('Open,WIN_Window1.WDW');
```


<a name="Example3"></a>



```txt
// In VB
call CALLWD("Open,WIN_Window1.WDW")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`Open(<Window name> [, <"/CR"> [, <Y> [, <X> [, <"+TAB+"Parameter 1"+TAB+"Parameter 2"+...>]]]])`
---

**`<Window name>: (Cadena de caracteres)`**

 Nombre de la ventana a abrir. Este parámetro incluye el ruta, el nombre y la extensión de la ventana (la extensión de las ventanas WINDEV es ".WDW" de Default).

**`<"/CR">: (Parámetro opcional)`**

Le permite detectar un error.

**`<Y>: (Entero (opcional))`**

 Ordenada (en pixeles) de la ventana (con respecto a la esquina superior izquierda de la pantalla o ventana padre).

**`<X>: (Entero (opcional))`**

Coordenada X (en píxeles) de la ventana (en relación con la Corner superior de la pantalla).

**`<"+TAB+"Parameter 1"+TAB+"Parameter 2"+...>: (Parámetros opcionales)`**

 Parámetros a pasar a la ventana cuando se abre.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si se especifica el parámetro /CR, el código de estado "E\*" se devuelve en WdKey en caso de error.

- Le recomendamos utilizar la función **Open** en lugar de la función Use cuando debe restaurarse el contexto antes abrir la ventana.

- La ventana abierta se apila sobre la ventana anterior (si existe).

- Los controles que se encuentran en la ventana abierta son Active. Los controles que se encuentran en la ventana anterior (si existe) son Inactive.

- Para modificar el título de la ventana, debes usar Pantalla,Siguiente o Pantalla,Actual.

- WdKey contiene "\*E\*" si se ha producido un error durante la apertura.

- Si una ventana llama a procedimientos globales, el proyecto debe ser ejecutado por Proyecto antes de abrir la ventana, de lo contrario los procedimientos globales y la hoja de estilo no serán reconocidos.

- Se puede abrir una ventana aunque no haya ningún proyecto en curso. En este caso, la ventana no debe estar conectada a ningún proyecto.

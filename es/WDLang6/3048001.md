
## JavaLoad( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Especifica la ubicación de las clases de usuario necesarias para ejecutar la aplicación Java o el applet Java.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Specifies the location of .class files of the Java application to start
// Note: Specify the names of folders/directories and the names of .jar or .zip
// (and not just the name of .class)

JavaLoad("D:\sources\Example\JavaTest")
```


<a name="Example2"></a>





```wl
// Defines the java.library.path variable to the c:\tmp value
JavaLoad(".", "-Djava.library.path=c:\tmp")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = JavaLoad(<Location> [, <Parameters>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si existen los caminos especificados,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Location>: (Cadena de caracteres)`**

Lista de ubicaciones separadas por: 

- punto y coma (";") en Windows,

- dos puntos (":") en Linux. 


Las ubicaciones pueden corresponder a: 

- ruta completa o relativa de los archivos.jar,

- ruta completa o relativa de los directorios que contienen los archivos.class. 
	Atención: no especifique la ruta completa de un archivo.class sino sólo la ruta del directorio que la contiene. 




**`<Parameters>: (Cadena de caracteres opcional)`**

Opciones para inicializar la biblioteca de Java. Estas cadenas deben tener el siguiente formato: 

```txt
"-D<Name>=<Value>"
```

Si se deben especificar varias opciones, las cadenas correspondientes deben estar separadas por una coma.

## Observaciones




### Llamada a JavaLoad
<a name="llamada_docparampagetitleshort_ELTPARAGRAPHE000062"></a>

Una sola llamada a **JavaLoad** debe realizarse en una aplicación WINDEV o en un sitio WEBDEV.

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) **JavaLoad** se puede llamar tantas veces como sea necesario.
<a name="NOTE0_2"></a>




### Java versión 1.2 (o sub-versión), 1.3 (o sub-versión), 1.4 (o sub-versión), Java 6
<a name="java_version_12_subversion_13_subversion_14_subversion_java_6_ELTPARAGRAPHE000079"></a>

- Para utilizar **JavaLoad**, el entorno de ejecución Java (J2SE JRE) o el entorno de desarrollo (J2SE SDK) deben estar instalados en el equipo.

- El mensaje de error "Fallo al cargar el JVM.DLL" significa que el archivo Java.DLL no se encontró en el ordenador actual. El directorio que contiene el archivo JVM.DLL debe ser añadido al PATH Variable del ordenador actual.




**Observación**: la búsqueda del archivo JVM.DLL en Windows y del archivo "libjvm.soc" en Linux se realiza de la siguiente manera: 

1. En Windows, busque en las claves del registro:

	- HKEY_LOCAL_MACHINE\\SOFTWARE\\JavaSoft\\JRE: "CurrentVersion" valor, entonces "RuntimeLib" valor en &lt;versión&gt; subclave.

	- HKEY_MÁQUINA_LOCAL\\SOFTWARE\\Java Soft\\Java Runtime Environment: "CurrentVersion" valor, entonces "RuntimeLib" valor en &lt;versión&gt; subclave.

	- HKEY_LOCAL_MACHINE\\SOFTWARE\\JavaSoft\\JDK:: "CurrentVersion" valor, entonces "RuntimeLib" valor en &lt;versión&gt; subclave.

	- HKEY_MÁQUINA_LOCAL\\SOFTWARE\\Java Soft\\Java Kit de desarrollo: "CurrentVersion" valor, entonces "RuntimeLib" valor en &lt;versión&gt; subclave.




2. Para todas las plataformas (Windows y Linux), busque en el directorio dado por el entorno "JAVA_HOME" Variable.

3. Para Linux, busque en el directorio "/etc/alternative/jre".. 

4. Para todas las plataformas (Windows y Linux), busque en PATH para encontrar bibliotecas del sistema operativo.


## ActionPlanInfo( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Recupera la información relativa a la ejecución de un plan de acción.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Checks the execution of the action plan for compiling my project
bCompilOK is boolean = ActionPlanInfo("FacCoord", "admin", ...
	"fackey", nPlanCompilMyProjectID, apStatusCompletedOK)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = ActionPlanInfo(<Coordinator> , <User> , <Password> , <Identifier of Instance of Action Plan> [, <Information>])`
---

**`<Result>: (Tipo de información solicitada)`**

Información solicitada. 

**`<Coordinator>: (Cadena de caracteres)`**

Nombre de el equipo utilizado para coordinar la ejecución de los planes de acción.

**`<User>: (Cadena de caracteres)`**

Nombre del usuario. Este usuario debe poder ejecutar el plan de acción en el coordinador.. 

**`<Password>: (Cadena de caracteres)`**

Contraseña de usuario. 

**`<Identifier of Instance of Action Plan>: (Integro)`**

Identificador de la instancia del plan de acción a utilizar. Este identificador es devuelto por [ActionPlanExecute](../WDLang6/1000021332.md).

**`<Information>: (Constante opcional de tipo Integer)`**

Información solicitada sobre el plan de acción. 



|   |   |
| --- | --- |
| *apInfoCancelationInProgress* | &lt;Resultado&gt; es un booleano y es set a: <br><br>- <u><u><u><u>True</u></u></u></u> si la instancia del plan de acción está actualmente cancelada, <br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario. <br><br><br> |
| *apInfoComputer* | &lt;Resultado&gt; es una cadena de caracteres que contiene el nombre de el equipo que ejecuta el plan de acción. |
| *apInfoEnd* | &lt;Resultado&gt; es una Variable DateTime correspondiente a la fecha y hora de finalización de la ejecución del plan de acción. |
| *apInfoLog* | &lt;Resultado&gt; es una cadena de caracteres que contiene el registro de tiempo de ejecución del plan de acción. |
| *apInfoProgress* | &lt;Resultado&gt; es un número entero que corresponde al porcentaje de progreso en la ejecución del plan de acción. |
| *apInfoStart* | &lt;Resultado&gt; es una Variable DateTime correspondiente a la fecha y hora de inicio de la ejecución del plan de acción. |
| *apInfoStatus*<br>(Valor predeterminado) | &lt;Resultado&gt; es una constante entera que corresponde al estado del plan de acción: <br><br>- *apStatusDisconnected*: El robot utilizado para ejecutar el plan de acción ya no responde.<br><br>- *apStatusPending*: El plan de acción está a la espera de ser ejecutado.<br><br>- *apStatusInProgress*: El plan de acción se está ejecutando actualmente.<br><br>- *apStatusStartup*: El coordinador va a iniciar el plan de acción.<br><br>- *apStatusCompletedFailure*: La ejecución del plano finalizó con error.<br><br>- *apStatusCompletedOK*: La ejecución del plano finalizó sin errores.<br><br><br> |
| *apInfoUser* | &lt;Resultado&gt; es una cadena de caracteres que contiene el nombre del usuario que inició la ejecución del plan de acción. |


## TileAdd( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade una nueva personalización de Tile para la aplicación actual de la aplicación universal de Windows 10. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyTile is Tile
MyTile.Type = tileWide
MyTile.Model = tileTemplateWideImageTop
Add(MyTile..Images, "photo.jpg")
MyTile.Text = "Important"
TileDeleteAll()
TileAdd(MyTile)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Añadiendo un azulejo definido por un Azulejo Variable

`TileAdd(<Tile>)`
---

**`<Tile>: (Azulejo Variable)`**

Nombre de la variable [Mosaico](../WDLang6/1000020713.md) que contiene el description del Tile a añadir. 


<a name="SYNTAX2"></a>

### Añadiendo un Tile definido por un archivo XML

`TileAdd(<URI> , <Interval>)`
---

**`<URI>: (Cadena de caracteres)`**

URI Address apuntando al contenido XML del Tile para añadir. 

**`<Interval>: (Constante de tipo Integer)`**

El tiempo interval entre las actualizaciones de Tile: 


|   |   |
| --- | --- |
| *tileEveryDay* | El Tile se actualiza todos los días. |
| *tileEveryHalfHour* | El Tile se actualiza cada 30 minutos. |
| *tileEveryHour* | El Tile se actualiza cada hora. |
| *tileEverySixHours* | El Tile se actualiza cada 6 horas. |
| *tileEveryTwelveHours* | El Tile se actualiza cada 12 horas. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Tile añadido del tipo Tile
<a name="tile_anadido_del_tipo_tile_ELTPARAGRAPHE000072"></a>

El azulejo associated con la aplicación está personalizado. Tiene la posibilidad de añadir hasta 5 personalizaciones diferentes. Estas personalizaciones pueden ser eliminadas por [TileDeleteAll](../WDLang6/1000020720.md).
<a name="NOTE0_2"></a>


### Tile añadido de un URI
<a name="tile_anadido_uri_ELTPARAGRAPHE000082"></a>

Un Tile añadido desde una URI se utiliza para crear un Tile con actualización, incluso si la aplicación está cerrada. Al final de cada período, el azulejo get el contenido del archivo XML que se encuentra en el URI para realizar la udpate.

Para obtener más información, consulte [http://msdn.microsoft.com/en-us/library/windows/apps/hh779724.aspx](http://msdn.microsoft.com/en-us/library/windows/apps/hh779724.aspx).

Observación: El **XML** Property del tipo [Mosaico](../WDLang6/1000020713.md) recupera un código XML válido. 

Una Tile recurrente se puede borrar con [TileDeleteAll](../WDLang6/1000020720.md).
<a name="NOTE0_3"></a>


### Caso especial
<a name="caso_especial_ELTPARAGRAPHE000104"></a>

La primera adición de una Tile en una aplicación eliminará las personalizaciones existentes (si las hay).

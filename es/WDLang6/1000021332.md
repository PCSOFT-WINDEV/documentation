


## ActionPlanExecute( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ejecuta una instancia del plan de acción en la fábrica de software desde una aplicación WINDEV o desde un sitio WEBDEV.. Esto le permite automatizar el lanzamiento de planes de acción.. Por ejemplo, se puede iniciar un plan de acción para ejecutar pruebas desde una aplicación por lotes.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Runs the instance of action plan for compiling my project
InstanceID is int
InstanceID = ActionPlanExecute("FacCoord", "admin", "fackey", "PlanCompilMyProject")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = ActionPlanExecute(<Coordinator> , <User> , <Password> , <Instance of action plan> [, <WLanguage procedure>])`
---

**`<Result>: (Integro)`**



- Identificador de la instancia del plan de acción si se ha iniciado,

- 0 no se ha iniciado la instancia del plan de acción. El mensaje de error correspondiente es devuelto por [ErrorInfo](../WDLang1/3013008.md).




**`<Coordinator>: (Cadena de caracteres)`**

Nombre de el equipo utilizado para coordinar la ejecución de los planes de acción.

**`<User>: (Cadena de caracteres)`**

Nombre de usuario. Este usuario debe poder ejecutar el plan de acción en el coordinador.. 

**`<Password>: (Cadena de caracteres)`**

Contraseña de usuario. 

**`<Instance of action plan>: (Cadena de caracteres)`**

Nombre de la instancia del plan de acción a ejecutar.
Para ejecutar una instancia de plan de acción de una biblioteca dada, el nombre de la instancia debe tener el siguiente formato: 

```txt
<Library name> + TAB + <Name of instance of action plan>
```
Si no se especifica el nombre de la biblioteca, se ejecutará la primera instancia del plan de acción encontrado correspondiente al nombre especificado.

**`<WLanguage procedure>: (Nombre opcional de Procedure)`**

Nombre del lenguaje WLanguage Procedure llamado cuando se ejecuta el plan de acción. Esta Procedure se utiliza para seguir la ejecución del plan de acción y para saber si su ejecución se terminó correctamente. 
Este procedimiento tiene el siguiente formato: 


```txt
PROCEDURE ProcedureName(<Instance of action plan>, <Current step>, 
			<Progress>, <Status>)
```
donde: 

- &lt;Instancia del plan de acción&gt; es una cadena que corresponde al nombre de la instancia ejecutada. 

- &lt;Paso actual&gt; es una cadena que corresponde a la leyenda del paso actual.. 

- &lt;Progreso&gt; es un número entero correspondiente al porcentaje de progreso de la ejecución (entre 0 y 100).. 

- &lt;Status&gt; es una constante entera que indica el estado de ejecución:
	


|   |   |
| --- | --- |
| *apStatusCompletedFailure* | La ejecución del plano finalizó con error. |
| *apStatusCompletedOK* | La ejecución del plano finalizó sin errores. |
| *apStatusDisconnected* | El robot utilizado para ejecutar el plan de acción ya no responde. |
| *apStatusInProgress* | El plan de acción se está ejecutando actualmente. |
| *apStatusPending* | El plan de acción está a la espera de ser ejecutado. |
| *apStatusStartup* | El coordinador va a iniciar el plan de acción. |

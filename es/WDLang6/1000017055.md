


## gpwOpenLDAPSession( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Informa al User Groupware que la autenticación de los usuarios se hará a través de un directorio LDAP. El nombre y la contraseña de los usuarios que se conectarán a groupware usuario se comprobarán en el directorio LDAP directamente.

**Observación**: Esta función es inútil si la configuración del directorio LDAP se realizó en la ventana de configuración de groupware usuario. Para configurar el Groupware Usuario, en la pestaña "Proyecto", en el grupo "Proyecto", seleccione "Groupware Usuario".
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// User Groupware used in automatic mode 
// Manual start. No data files. 

// Project code
gpwOpenLDAPSession("gemini.myserver.com", 389, 3, False)

// Identification of a user without login window
IF gpwLogin("FP", "FP") <> gpwOK THEN
	Error("Invalid user or password")
	EndProgram()
END
```


<a name="Example2"></a>

```wl
// Use a base DN 
gpwOpenLDAPSession("myserver", 389, 3, False, "DC=groupcom,DC=local")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`gpwOpenLDAPSession(<Server address> [, <Port number> [, <Server version> [, <With files> [, <Base DN> [, <Management of SSL>]]]]])`
---

**`<Server address>: (Cadena de caracteres)`**

Address del servidor LDAP. Este parámetro puede corresponder a:

- IP Address del servidor en formato xxx.xxx.xxx.xxx

- Nombre del servidor




**`<Port number>: (Entero opcional)`**

Número de puerto del servidor LDAP (389 por Default).

**`<Server version>: (Entero opcional)`**

Versión del servidor LDAP (2 o 3). Corresponde al 3 de Default.

**`<With files>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> (valor predeterminado) para usar los archivos de datos del groupware,

- <u><u><u><u>False</u></u></u></u> para realizar una autenticación LDAP.




**`<Base DN>: (Cadena de caracteres opcional)`**

DN de la base a utilizar. Este parámetro tiene el siguiente formato: "DC=xxx, DC=yyyy, DC=zzzz, ....".  
Si no se especifica este parámetro, se buscará automáticamente en la base de datos.. 

**`<Management of SSL>: (Constante o combinación de constantes de tipo Integer)`**

Modo de gestión de SSL: 



|   |   |
| --- | --- |
| *ldapDefaultOption*<br>(Valor predeterminado) | LDAP estándar, sin gestión SSL. |
| *optionSSL* | LDAP con gestión de la conexión SSL de secure. |
| *optionSSL* + *auNegotiate* | LDAP con gestión de la conexión y autentificación secure SSL. La conexión se establece a través de "Generic Security Services" (GSS) (Snego). El método de autenticación utilizado es el más apropiado que se encuentra en una lista de posibles autenticaciones.. |


Observación: El modo de conexión utilizado debe ser soportado por el servidor LDAP.

## Observaciones


### Cheques
<a name="cheques_ELTPARAGRAPHE000092"></a>

Se realizan las comprobaciones relativas a la conexión con el servidor LDAP, el nombre de usuario y la contraseña:

- durante la llamada a [gpwLogin](../WDLang6/3041001.md).

- al validar la ventana de inicio de sesión del groupware.

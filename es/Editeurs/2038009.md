


## Añadir un elemento al repositorio de SCM
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000126"></a>
Cuando se añade un elemento al repositorio de SCM (Administrador de Código Fuente), se añade al proyecto reference en el servidor de SCM. Este elemento se pondrá a disposición de todos los usuarios de SCM durante su próxima sincronización.

**Observación**: Mediante Default, cuando se añade un elemento al proyecto local, el Administrador de Código Fuente propone automáticamente añadirlo al repositorio SCM cuando se está guardando.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ajout_element_base%20-%20HC%20N%B0001.gif&type=thumb)

## Agregar un elemento
<a name="agregar_elemento_ELTTEXTE000150"></a>
Para añadir un elemento específico al depósito de SCM:

1. Seleccione el elemento en el panel "Explorador de proyectos".

2. Seleccione "Añadir a Administrador de Código Fuente" en el menú contextual.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ajout_element_base%20-%20HC%20N%B0002.gif)





Para añadir un elemento a la SCM, también puede ir a la pestaña "SCM", grupo "Elemento actual", y hacer clic en "Agregar".

Cuando se añade un elemento a la SCM se muestra la siguiente pantalla:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ajout_element_base%20-%20HC%20N%B0003.gif&type=thumb)
Es posible:

- En la pestaña "Tareas", especifique que el nuevo elemento corresponde a la ejecución de una tarea o a la finalización de una tarea.

- En la pestaña "Incidentes", especifique que el nuevo elemento corrige un incidente.

- En la pestaña "comment", introduzca un comment sobre el nuevo elemento. Por ejemplo, puede especificar que esta nueva ventana o Page se utilice para escribir los datos de acceso de los usuarios.




Las opciones disponibles son las siguientes: 

- **Extraer elementos**: Esta opción se utiliza para poner el elemento a disposición de los demás usuarios mientras se mantiene seleccionado para su modificación.

- **No permitir extracciones múltiples**: Esta opción permite que el elemento sea revisado por un solo usuario a la vez.. El elemento no puede ser modificado por varios usuarios al mismo tiempo.

- **No guardar historial**: Si esta opción está marcada, no se almacenará ningún historial de check-in / check-out para este elemento.
	Observación: El historial del elemento se puede activar posteriormente a través de las propiedades del elemento. Para ver las propiedades del elemento actual, vaya a la pestaña "SCM", grupo "Elemento actual" y haga clic en "Propiedades".

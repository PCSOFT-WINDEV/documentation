## Presentación
<a name="presentacion_ELTTEXTE000186"></a>
La ventana del proyecto description le permite configurar varios elementos de su proyecto. Esta ventana incluye las siguientes pestañas:

- La pestaña ["Proyecto"](../Editeurs/2030046.md), que permite introducir información general sobre el proyecto. 

- La pestaña "Análisis", que permite asociar un análisis al proyecto y/o a un modelo UML.. 

- La pestaña **"Archivos"** que permite introducir diferente información sobre los archivos de datos utilizados por la aplicación (disponible solo si el proyecto está asociado a un análisis)

- La pestaña "Datos en vivo", que permite configurar cómo se muestran los datos de los ficheros de datos en los distintos editores.

- La pestaña ["Idiomas"](../Editeurs/9500219.md), que se utiliza para configurar los diferentes idiomas utilizados por el proyecto, así como las opciones que se utilizarán para cada idioma.

- La pestaña ["Estilo"](../Editeurs/9500234.md), que permite elegir el plantilla, personalizar la vista previa de impresión y seleccionar una hoja de estilo.. También puedes configurar el tamaño de la pantalla.

- La pestaña "SCM", que sirve para gestionar el trabajo en equipo (SCM o GIT).

- La pestaña ["Opciones"](../Editeurs/9500227.md), que permite configurar las opciones de actualización, copia de seguridad y sincronización de modelo. Esta pestaña también se utiliza para habilitar el modo estricto y para definir el estilo de código.

- La pestaña ["Avanzado"](../Editeurs/9500236.md), que permite configurar cómo utilizar "Reports & Queries", las AAF y cómo bloquear la aplicación.

- La pestaña ["Recopilación"](../Editeurs/9500218.md), que permite configurar las opciones de compilación.

- La pestaña ["Telemetría"](../Editeurs/9500223.md), que permite configurar las opciones de telemetría del proyecto.

## Pestañas de archivos
<a name="pestanas_archivos_ELTTEXTE000216"></a>


### Directorio de archivos que se utilizará en modo de prueba
<a name="directorio_archivos_que_utilizara_modo_prueba_ELTPARAGRAPHE000059"></a>

Esta opción permite definir el directorio de los archivos de datos que se utilizarán en el modo de prueba.: depuración, Ir a una ventana, Page o proyecto. 

El directorio usado por Default es el directorio de la configuración actual. Puede especificar un directorio concreto. 

Esta opción es useful si se usa WINDEV en Windows Vista mientras la UAC está activada: el directorio de los datos de la aplicación se puede utilizar en modo de prueba.
<a name="NOTE2_2"></a>


### Automatizaciones
<a name="automatizaciones_ELTPARAGRAPHE000070"></a>

Las siguientes opciones están disponibles para todos los tipos de datos: 

- **Activar asistencia automática HFSQL**
	Esta operación se utiliza para permitir la gestión de errores del automatic por el motor HFSQL. Para obtener más información, consulte [Gestión asistida de errores](../WDLang4/3044188.md).

- **Crear automáticamente los archivos de datos cuando se abran**: Esta opción se utiliza para crear automáticamente los ficheros de datos descritos en el análisis si estos ficheros no existen cuando se utilizan funciones que requieren su apertura.. En este caso, [HCreationIfNotFound](../WDLang4/3044254.md) se ejecuta automáticamente.
	![Conectores Nativos](https://doc.pcsoft.fr/ext/images/us/AN.png) Esta opción es tomada en cuenta por los conectores nativos a SQL Server, Informix, Sybase, MySQL, MariaDB y PostgreSQL.



<a name="NOTE2_3"></a>


### Reindexación automática (solo HFSQL)
<a name="reindexacion_automatica_solo_hfsql_ELTPARAGRAPHE000102"></a>

Un fichero puede tener que ser reindexado cuando el fichero índice se pierde o se daña. Se puede implementar una operación de reindexación de automatic para los archivos de datos.

La opción "Activar reindexación automática" permite reindexar automáticamente los ficheros de datos especificando varios parámetros: 

- el número máximo de registros: Si el archivo de datos contiene menos de XXX registros, la operación de reindexación se realiza automáticamente si se produce un error de índice. Por defecto, este valor se establece en 100000. La operación de reindexación de automatic se desactiva si este Variable es set a 0.

- la visualización de una ventana de progreso que muestra el estado de la operación de reindexación actual.




Observación: la reindexación de automatic sólo está disponible para los archivos HFSQL Classic y HFSQL Client/Server.
<a name="NOTE2_4"></a>


### Modo de gestión de archivos (solo HFSQL)
<a name="modo_gestion_archivos_solo_hfsql_ELTPARAGRAPHE000120"></a>

Esta opción se utiliza para definir el modo de gestión de los ficheros de datos que se encuentran en la aplicación. Por Default, las aplicaciones creadas por WINDEV y WEBDEV son aplicaciones multiusuario: los bloqueos se gestionan automáticamente.
Puede set un modo "Monopuesto: en este caso, la apertura de un archivo de datos impide automáticamente a los demás usuarios leer o escribir. 
<a name="NOTE2_5"></a>


### Opciones para Conectores Nativos
<a name="opciones_para_conectores_nativos_ELTPARAGRAPHE000131"></a>

Cuando se copian contextos utilizando un Conector Nativo, si la opción "Copiar conexiones al copiar el contexto" está marcada, cada contexto puede tener su propia conexión con el servidor de base de datos externo. Esta opción es particularmente useful para las transacciones. 
 Esta novedad está disponible para las siguientes bases de datos: MS SQLServer, Oracle, MySQL, Sybase, Progress, Informix, DB2, SQLite, MariaDB.




## Probar y depurar un proyecto en el ordenador de desarrollo
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000262"></a>
La prueba de todo el proyecto permite simular el inicio de la aplicación con el ejecutable. Esto le permite probar toda la aplicación, incluso si todavía se está desarrollando. Cuando se produce un problema al ejecutar el proyecto, puede iniciar el depurador para identificar y solucionar el problema.

WINDEV incluye un amplio conjunto de pruebas de aplicaciones que pueden ejecutarse desde el equipo de desarrollo:

- [Ejecutar la prueba del proyecto actual y depurarla en el editor](#NOTE2).

- [Ejecutar y depurar el ejecutable correspondiente al proyecto actual](#NOTE3).

- [Depurar un ejecutable ya ejecutado en el equipo de desarrollo](#NOTE4).




**Observación**: La prueba de un proyecto puede ejecutarse independientemente del elemento actual en el editor.

## Probar y depurar el proyecto actual
<a name="probar_depurar_proyecto_actual_ELTTEXTE000286"></a>


### Probar un proyecto
<a name="probar_proyecto_ELTPARAGRAPHE000032"></a>

Para probar un proyecto desde el editor: 

1. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Depurar proyecto" (Ctrl + F9). También tiene la posibilidad de hacer clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_GO_Projet_WD.gif) en los botones de acceso rápido.

2. Si no se ha definido la primera ventana de proyecto, se muestra una ventana que le permite seleccionar la primera ventana de proyecto.. 

3. El editor se minimiza automáticamente y el proyecto se ejecuta.




Al comienzo de la prueba del proyecto, antes de abrir la primera ventana del proyecto, se ejecutan los siguientes eventos WLanguage:

- declarando clases,

- inicialización del proyecto,

- declarando las variables globales de la primera ventana,

- inicializando la primera ventana,

- inicializar los controles que se encuentran en la primera ventana,

- ganando el foco (primera ventana),

- entrando en el primer control de la primera ventana (si este control está asociado a una entrada de WLanguage Event).





### Observación: Minimizar (o no) el editor durante la prueba
<a name="observacion_minimizar_editor_durante_prueba_ELTPARAGRAPHE000317"></a>

Durante una prueba, el editor es minimizado por Default. Dependiendo de la configuración del hardware (pantalla ancha, 2 pantallas, etc.), puede que no sea necesario minimizar el editor. 

Para mantener abierto el editor: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones del editor de código". 

2. En la pestaña "Depurador", deseleccione la opción "Minimizar editor durante el modo de prueba". 

3. Validar. 



<a name="NOTE2_2"></a>


### Configuración del modo de prueba
<a name="configuracion_del_modo_prueba_ELTPARAGRAPHE000070"></a>

Para configurar el modo de prueba, vaya a la pestaña "Proyecto", grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Configurar modo de prueba".

Para más detalles sobre las diferentes opciones, véase [Opciones del modo de prueba](../Editeurs/2019020.md).

## Ejecutar y depurar el ejecutable en el ordenador de desarrollo
<a name="ejecutar_depurar_ejecutable_ordenador_desarrollo_ELTTEXTE000350"></a>
Algunos problemas solo se pueden reproducir cuando se prueba el ejecutable en condiciones de uso real. Este modo de prueba permite depurar un archivo ejecutable una vez que se instala en el equipo de desarrollo. En este caso, el código ejecutado proviene de las bibliotecas (y no del código de los elementos en el equipo). Por lo tanto, el código del proyecto actual y el código del ejecutable pueden no estar sincronizados: el código que no está sincronizado se resalta en rosado durante la depuración.

**Para ejecutar y depurar el ejecutable**: 

1. Abre el proyecto associated con el ejecutable en el editor.

2. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba", vaya a "En un ejecutable" y seleccione "Ejecutar".

3. Especifique las opciones que deben tenerse en cuenta durante la prueba:

	- el ejecutable a ejecutar. Este ejecutable debe corresponder al proyecto abierto en el editor.

	- el comando Line para ejecutar (si es necesario).

	- la cuenta de Windows que debe utilizarse para iniciar el ejecutable. Si esta cuenta difiere del usuario actual, el usuario debe haber sido definido en el ordenador actual.. Debe especificarse su nombre de usuario, su contraseña y su dominio.




4. Validar. Se realiza la prueba.




Observación: En este modo, [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>False</u></u></u></u>.

## Depurar un ejecutable ya ejecutado en el equipo de desarrollo
<a name="depurar_ejecutable_ejecutado_equipo_desarrollo_ELTTEXTE000374"></a>
Algunos problemas solo se pueden reproducir cuando se prueba el ejecutable en condiciones de uso real. Este modo de prueba permite encontrar un problema que solo ocurre en tiempo de ejecución (y no en modo de prueba). En este caso, el código ejecutado proviene de las bibliotecas (y no del código de los elementos en el equipo). Por lo tanto, el código del proyecto actual y el código del ejecutable pueden no estar sincronizados: el código que no está sincronizado se resalta en rosado durante la depuración.

**Para depurar un ejecutable que ya se está ejecutando**: 

1. Abre el proyecto associated con el ejecutable en el editor.

2. Se puede incluir un punto de interrupción en el código que activa el depurador.

3. En Proyecto pestaña " Modo de prueba ", en el " Modo de prueba " Ejecutar ".

4. La ventana que se muestra enumera todos los ejecutables de WINDEV que se están ejecutando actualmente. Seleccione el ejecutable a depurar.

5. Validar. El depurador es relacionado para la ejecución ejecutable.




**Observación**: En este modo, [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>False</u></u></u></u>.

## Iniciar el depurador y detener la prueba
<a name="iniciar_depurador_detener_prueba_ELTTEXTE000398"></a>


### Inicio del depurador
<a name="inicio_del_depurador_ELTPARAGRAPHE000171"></a>

Se pueden utilizar varios métodos para iniciar el depurador durante la prueba del ejecutable:

- **Presionando Ctrl + Pausa**. La acción realizada una vez que se han presionado las teclas Ctrl + Pausa (haga clic en un botón, ...) iniciará el depurador.. Este método se puede utilizar antes de hacer clic en un botón, por ejemplo.
	Este método también se puede utilizar para iniciar el depurador cuando una larga Process (que contiene declaraciones multitarea) parece repetirse sin fin.

- **Iniciar el depurador desde el editor**. En este caso, realice las siguientes operaciones:

	- Haga clic en el WINDEV Icon en la barra de tareas.

	- En "¿Desea detener la prueba?", seleccione "No".

	- Vaya al panel "Depurador", grupo "Depurar", y haga clic en el botón ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=main_debog.gif) Icon.  







También se puede incluir un punto de ruptura en el editor de código delante de la deseada Line de código. El depurador se mostrará automáticamente cuando se ejecute el correspondiente Line.

Una vez que el depurador está funcionando: 

- el editor de código WINDEV muestra la pestaña "Depurador".. El texto de la ficha se muestra en rojo.  

- la barra de estado se muestra en naranja. 




Para más detalles sobre el depurador, véase [Ejecución de un proyecto paso a paso](../Editeurs/2019001.md). 
<a name="NOTE5_2"></a>


### Detener la prueba
<a name="detener_prueba_ELTPARAGRAPHE000203"></a>

Existen varios métodos para detener la prueba:

- Método 1: Cierre la aplicación que se está probando. WINDEV vuelve al editor que estaba abierto cuando se inició la prueba.

- Método 2: Vuelve al editor con la barra de tareas (o presiona Alt + Tab) y confirma el final de la prueba. WINDEV vuelve al editor que estaba abierto cuando se inició la prueba.




**Observación**: Si se han realizado cambios en la UI durante la prueba del proyecto (por ejemplo, cambiar el tamaño o mover los controles), al final de la prueba, WINDEV le permite cambiar la interfaz de las ventanas correspondientes. Para obtener más información, consulte [Usando la modificación de la UI durante una prueba del proyecto](../WDLang1/1000021927.md).

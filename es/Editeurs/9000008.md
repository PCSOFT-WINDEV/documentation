
## Manejo de una base de datos de una aplicación de Java
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>




## Presentación
<a name="presentacion_ELTTEXTE000333"></a>
Puede empezar a desarrollar su aplicación tan pronto como el proyecto WINDEV se defina como proyecto Java.
Todos los tipos de bases de datos pueden manejarse en WLanguage:

- Base de datos HFSQL.

- base de datos no en formato HFSQL. Esta base de datos puede ser manejada por un controlador JDBC o por un acceso nativo.


Para manipular estas bases de datos, usted tiene la posibilidad de utilizar:

- [Las funciones HFSQL](#NOTE2)que le permiten manipular los archivos de datos directamente (base de datos HFSQL, base de datos MySQL accedida por JDBC, base de datos SQL Server accedida por JDBC).

- [Las funciones SQL](#NOTE5) que le permiten manipular los archivos de datos directamente (base de datos HFSQL o base de datos accedida por JDBC (MySQL, SQL Server, Oracle, DB2, ....)).  

- [El Java RAD](#NOTE8)que permite desarrollar rápidamente ventanas que manipular archivos de datos.

## Acceso a los archivos de datos con las funciones HFSQL
<a name="acceso_los_archivos_datos_con_las_funciones_hfsql_ELTTEXTE000363"></a>
Una aplicación Java desarrollada con WINDEV le permite acceder a los archivos de datos de manipular independientemente de la plataforma en la que se ejecute la aplicación Java (Windows, Linux o Mac).

El uso de una base de datos de una aplicación Java puede realizarse mediante las funciones HFSQL. Las librerías del Framework WINDEV están disponibles en estos entornos, por lo que el acceso a la base de datos HFSQL a través de las funciones HFSQL es inmediato.

**Observación**: Si se marca la casilla "Crear automáticamente los archivos de datos si no se encuentran al abrirlos" (en la ventana description del proyecto, pestaña "Archivos"), se crearán automáticamente los archivos de datos correspondientes si es necesario.

## Las funciones HFSQL en un proyecto Java
<a name="las_funciones_hfsql_proyecto_java_ELTTEXTE000387"></a>


### Acceso a archivos HFSQL
<a name="acceso_archivos_hfsql_ELTPARAGRAPHE000050"></a>

El método para manejar los archivos HFSQL en un proyecto Java es el mismo que el método para manejar los archivos HFSQL en un proyecto de Windows.

Dependiendo de la plataforma de tiempo de ejecución (Windows, Linux o MacOSX), las bibliotecas necesarias para acceder a las bases de datos HFSQL pueden variar. Para obtener más información, consulte [Gestión de bases de datos en las aplicaciones WINDEV Java](../Editeurs/9000018.md). 

**Observación**: Por Default, los archivos de datos se instalan con el archivo Java. Sin embargo, los archivos de datos pueden instalarse en otro directorio. En este caso, debe especificar la nueva ruta en la aplicación con [HChangeDir](../WDLang4/3044168.md).


### Configuración del modo de ejecución de las aplicaciones de Java usando HFSQL en MacOSX
<a name="configuracion_del_modo_ejecucion_las_aplicaciones_java_usando_hfsql_macosx_ELTPARAGRAPHE000065"></a>

La versión mínima de MacOSX requerida para utilizar las bases de datos HFSQL con una aplicación Java es 10.5.8.

### Acceso a una base de datos MySQL, SQL Server o SQLite por JDBC
<a name="acceso_una_base_datos_mysql_sql_server_sqlite_por_jdbc_ELTPARAGRAPHE000075"></a>

Para manipular los archivos de datos descritos en el análisis de un proyecto de Java, deben:


1. Describa una conexión con [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md). Al definir esta conexión, debe especificar el controlador JDBC que desea utilizar.

2. Asociar esta conexión a los diferentes archivos de análisis con [HChangeConnection](../WDLang4/3044150.md).
	Ejemplo de conexión a una base de datos MySQL:
	
	```wl
	HOpenConnection("MyMySQLConnection", "Login", "Password", ...
		"jdbc:mysql://MyServer/MyDatabase", "MyDatabase", ...
		hAccesJDBCMySQL, 0, "com.mysql.jdbc.Driver")
	HChangeConnection("*", "MyMySQLConnection")
	```

	Ejemplo de conexión a una base de datos SQLite:
	
	```wl
	SQLiteConnection is Connection
	
	// Connection parameters
	SQLiteConnection.Provider = hAccessJDBCSQLite
	SQLiteConnection.Server = "jdbc:sqlite:mydatabase.db"
	SQLiteConnection.Access = hOReadWrite
	SQLiteConnection.ExtendedInfo = "org.sqlite.JDBC"
	// Open the connection
	HOpenConnection(SQLiteConnection)
	```

	Notas sobre el acceso a SQLite: 

	- [denominar](../Proprietes/2510082.md) y [Contraseña](../Proprietes/2510086.md) no tienen ningún efecto,

	- La [Servidor](../Proprietes/2512105.md) Property le permite set el nombre y la ubicación del archivo de la base de datos (en este caso: el archivo se llama mydatabase.db y se encuentra en el directorio de ejecución de la aplicación).




3. Cuando se define la conexión, las funciones HFSQL pueden utilizarse para manipular los registros encontrados en los archivos de datos.

### Acceso a archivos HFSQL
<a name="acceso_archivos_hfsql_ELTPARAGRAPHE000133"></a>

Para manipular los archivos de datos HFSQL descritos en el análisis de un proyecto Java, debes:

1. Describa una conexión con [SQLConnect](../WDLang4/3072005.md). Al definir esta conexión, se debe especificar el análisis utilizado (archivo.WDD) así como la base de datos (HFSQL)..
	Ejemplo:
	
	```wl
	nConnectionNum is int
	nConnectionNum = SQLConnect("JavaClient.wdd", "", "", "", "HYPER FILE", "", "")
	```


2. Cuando se define la conexión, las funciones HFSQL pueden utilizarse para manipular los registros encontrados en los archivos de datos HFSQL.




**Observaciones**:

- Por Default, los archivos de datos se instalan con el archivo Java. Sin embargo, los archivos de datos pueden instalarse en otro directorio. En este caso, debe especificar la nueva ruta en la aplicación con [HChangeDir](../WDLang4/3044168.md).

- Para acceder a los archivos HFSQL, las siguientes librerías y el archivo Java de la aplicación deben encontrarse en el mismo directorio (son copiados automáticamente por WINDEV durante la generación de Java):

	- **en Windows**: wd28 0hf.dll, wd28 0sql.dll, wd28 0jav.dll, wd28 0vm.dll y wd28 0uni.dll. Las bibliotecas de Windows se encuentran en el subdirectorio "Programas" del directorio de instalación de WINDEV.

	- **en Linux**: wd28 0hf.so, wd28 0sql.so, wd28 0jav.so, wd28 0vm.so y wd28 0uni.so. Las bibliotecas Linux se encuentran en el subdirectorio "Programas\\Framework" del directorio de instalación de WINDEV.

	- **en MacOSX**: wd28 0hf.dylib, wd28 0sql.dylib, wd28 0jav.dylib y wd28 0vm.dylib. Las librerías MacOSX se encuentran en el subdirectorio "Programas\\Framework" del directorio de instalación de WINDEV.




- El archivo de análisis (archivo WDD) especificado en la sintaxis de [SQLConnect](../WDLang4/3072005.md) debe ser accesible desde la aplicación Java.

### Acceso a una base de datos que no está en formato HFSQL por JDBC
<a name="acceso_una_base_datos_que_esta_formato_hfsql_por_jdbc_ELTPARAGRAPHE000196"></a>

Para manipular los archivos de datos descritos en el análisis de un proyecto de Java, deben:

1. Describa una conexión con [SQLConnect](../WDLang4/3072005.md). Al definir esta conexión, se debe especificar el análisis utilizado (archivo.WDD) así como la base de datos (HFSQL)..
	Ejemplo de conexión a una base de datos MySQL:
	
	```wl
	nConnectionNum is int
	nConnectionNum = SQLConnect("jdbc:mysql://Venus/test", "test", ...
			"mysql", "", "JDBC", DriverNameMysqlName)
	```


2. Una vez establecida la conexión, utilice las funciones SQL para manipular los registros encontrados en los ficheros de datos.

### Disponible RAD
<a name="disponible_rad_ELTPARAGRAPHE000218"></a>

Como para cualquier proyecto WINDEV associated con un análisis, el RAD puede ser usado para generar las ventanas de su aplicación Java.
Tiene la capacidad de crear ventanas de formulario, ventanas tabla, ... El RAD de Java tiene en cuenta todas las características específicas de la generación de Java (controles, procesos, funciones que pueden utilizarse en una aplicación WINDEV generada en Java).
Dos tipos de patrones RAD pueden ser usados directamente en Java:

- El simple patrón RAD. 

- El patrón RAD'Compatible 11. 


El código generado por los otros patrones debe ser adaptado para operar en Java.

### Patrón RAD simple
<a name="patron_rad_simple_ELTPARAGRAPHE000236"></a>

Este patrón está adaptado para generar una aplicación de Java. Como para cualquier proyecto WINDEV associated con un análisis, el RAD puede ser usado para generar las ventanas de su aplicación Java.

**Observación**: Por Default, el simple patrón RAD maneja archivos HFSQL. Si desea utilizar una conexión por JDBC, debe adaptar el código generado. Para ello, describa la conexión a utilizar ([HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md)) especificando el controlador JDBC a utilizar. Esta conexión es associated con los diferentes archivos de análisis de [HChangeConnection](../WDLang4/3044150.md).

Ejemplo de conexión a una base de datos MySQL:


```wl
HOpenConnection("MyMySQLConnection", "Login", "Password", ...
	"jdbc:mysql://MyServer/MyDatabase", "MyDatabase", ...
	hAccesJDBCMySQL, 0, "com.mysql.jdbc.Driver")
HChangeConnection("*", "MyMySQLConnection")
```

### Errores de compilación
<a name="errores_compilacion_ELTPARAGRAPHE000282"></a>

Un RAD de Java desarrollado en WINDEV 10 o anterior puede provocar errores de compilación en la versión 18 para los controles que se encuentran en las pestañas.

Para resolver estos errores, el nombre de la control debe ir precedido del nombre de la ficha a la que pertenece. Por ejemplo, para un control llamado "Mycontrol" que pertenece a la pestaña llamada "MyTab", use la siguiente anotación: "MyTab.Mycontrol".

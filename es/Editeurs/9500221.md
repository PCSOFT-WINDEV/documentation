## Presentación
<a name="presentacion_ELTTEXTE000461"></a>
El proyecto cuadro de mando ofrece una visión general del estado del proyecto mediante diferentes indicadores. La cuadro de mando incluye varios elementos (Widgets) que proporcionan una visión general del estado del proyecto. ## Configurando la cuadro de mando
<a name="configurando_cuadro_mando_ELTTEXTE000485"></a>
Puede utilizar un menú contextual para configurar y personalizar su proyecto cuadro de mando (haga clic con el botón derecho del ratón en una zona vacía de cuadro de mando). <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=TDB_Prj_MenuCtx.gif)
 Este menú contextual se utiliza principalmente para: 

- Cambiar al modo de edición: El modo "Modificación" permite configurar los diferentes widgets. Es posible:

	- mover los widgets: Simplemente seleccione el widget deseado, mantenga el botón izquierdo del ratón pulsado y mueva el widget al lugar deseado. 

	- mostrar nuevos widgets: basta con volver a abrir el menú contextual de la cuadro de mando y seleccionar "Añadir".. Se proponen los widgets disponibles y se pueden seleccionar. 

	- ocultar los widgets inútiles: todo lo que tienes que hacer es abrir el menú contextual del widget a ocultar y seleccionar "Ocultar". El widget se borra inmediatamente de la cuadro de mando. 

	- restaurar la configuración inicial (antes de cambiar al modo de edición). 


 Observación: Para salir del modo "Modificación", basta con volver a seleccionar "Modo Modificación" en el menú contextual de cuadro de mando.

- Animar los widgets: Esta opción se utiliza para get un efecto de animación cuando se actualiza el widget. 

- Refrescar la cuadro de mando: Esta opción se utiliza para actualizar la cuadro de mando según las últimas modificaciones realizadas en el proyecto. 




La mayoría de los widgets proponen sus propias opciones.

### Información del proyecto
<a name="informacion_del_proyecto_ELTPARAGRAPHE000111"></a>

Este widget se utiliza para ver rápidamente las características del proyecto: 

- Proyecto encontrado o no el SCM (Administrador de Código Fuente). Si el proyecto no se encuentra en la SCM, un enlace le permite añadirlo. 

- Número de configuraciones encontradas en el proyecto. Al expandir el botón de flecha, tiene la posibilidad de seleccionar una configuración de proyecto o de crear una nueva.. 




En este Widget se encuentran diferentes enlaces y menús: 

- Un clic en el directorio del proyecto se utiliza para abrir este directorio en el explorador de Windows.. 

- La opción "Código" permite visualizar el código del proyecto en el editor de código. 

- La opción "description" permite visualizar la ventana description del proyecto.




El botón de la flecha en la parte superior derecha del widget se utiliza para abrir un menú desplegable con las principales opciones del proyecto: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=TDB_Wid_Prj.gif)

<a name="NOTE3_3"></a>


### Análisis del proyecto
<a name="analisis_del_proyecto_ELTPARAGRAPHE000130"></a>

Este widget permite saber si el proyecto es associated con un análisis. 

Si el proyecto está asociado a un análisis, se muestra el nombre del análisis y el número de archivos de datos que se encuentran en el análisis. El botón de flecha se utiliza para abrir el análisis en el editor. 

Si el proyecto no está relacionado a un análisis, el enlace "Crear un análisis" permite iniciar la creación del análisis asistente. 
<a name="NOTE3_4"></a>


### Elementos del proyecto
<a name="elementos_del_proyecto_ELTPARAGRAPHE000141"></a>

Este widget permite conocer el número de elementos del proyecto actual (número de ventanas, informes, clases,...). Con un clic en la lupa se abre la ventana para una búsqueda rápida de los elementos del proyecto.. Para obtener más información, consulte [Ventana de selección rápida](../Editeurs/2030051.md). 

Este widget ofrece un menú contextual (botón derecho del ratón) que permite: 

- Crear un nuevo elemento en el proyecto. 

- Importar un elemento existente en el proyecto. Para obtener más información, consulte [Importación de elementos en un proyecto](../Editeurs/2030006.md). 



<a name="NOTE3_5"></a>


### Administrador de Código Fuente
<a name="administrador_codigo_fuente_ELTPARAGRAPHE000159"></a>

Cuando se incluye un proyecto en un repositorio, este widget permite get: 

- el modo de gestión actual (automatic o Maestro/Suscriptor). 

- el número de elementos de la caja. Haga clic en este número para abrir la ventana que le permite registrar elementos en el repositorio. 

- el número de elementos a facturar. Haga clic en este número para abrir la ventana que le permite recuperar elementos del repositorio.

- el tamaño del proyecto en la Administrador de Código Fuente.




Para obtener más información, consulte [Trabajar con la Administrador de Código Fuente](../Editeurs/2038004.md). 
<a name="NOTE3_6"></a>


### Selección rápida
<a name="seleccion_rapida_ELTPARAGRAPHE000178"></a>

Este widget permite acceder rápidamente a los elementos del proyecto. Todo lo que tienes que hacer es escribir una palabra en la búsqueda control para mostrar todos los elementos del proyecto que contienen esta palabra. 

Un menú contextual sobre los elementos encontrados permite realizar la acción deseada (abrir, excluir de la configuración, eliminar del proyecto, ...).

- iniciar la auditoría estática del proyecto. Todo lo que tienes que hacer es pulsar la flecha gris del widget. 




Para obtener más información, consulte [Auditoría estática](../Editeurs/1014501.md).

### Componentes externos
<a name="componentes_externos_ELTPARAGRAPHE000202"></a>

Este widget se utiliza para conocer el número de componentes externos que se encuentran en el proyecto. 

Un menú desplegable (que se muestra mediante un clic en la flecha) se utiliza para: 

- Importar un componente externo a partir de un archivo. 

- Importar un componente externo a partir del SCM. 

- Importar un componente externo utilitario o ejemplo.

- Visualizar la lista de componentes externos importados en el proyecto. 

- Reubicar un componente.




Para obtener más información, consulte [Component externa de WINDEV, WEBDEV o WINDEV Mobile](../Editeurs/2014011.md).

### codificar
<a name="codificar_ELTPARAGRAPHE000226"></a>

Este Widget se utiliza para codificar las estadísticas del proyecto get: 

- Número total de líneas. 

- Autor de líneas de código. Un clic en la lupa sirve para realizar una búsqueda en todas las líneas de código del autor. 

- Número de líneas de comentarios. Un clic en la lupa se utiliza para realizar una búsqueda en todas las líneas de comment. 

- Número de procesos que contienen líneas de código. 

- Número de acciones predefinidas. Al hacer clic en la lupa se realiza una búsqueda en todas las acciones preestablecidas. 

- Número de errores de gestión de automatic. Un clic en la lupa permite realizar una búsqueda sobre estos procesos.

### Ejecución
<a name="ejecucion_ELTPARAGRAPHE000240"></a>

Este widget (también llamado "Auditoría dinámica") se utiliza para iniciar una auditoría dinámica. La auditoría dinámica se utiliza para realizar una auditoría durante la prueba de la aplicación.. Usted puede averiguar: 

- la huella de memoria de los elementos utilizados por la aplicación. 

- las imágenes no encontradas, ....




Un clic en la flecha del widget sirve para ampliar el menú del widget. Este menú se utiliza para: 

- permitir o no la auditoría dinámica. 

	- Si la auditoría dinámica está habilitada, se realizará para cada proyecto GO. El archivo de auditoría correspondiente se guarda en el directorio del proyecto. 

	- Si el widget está desactivado, se utiliza un enlace para volver a activarlo. 




- iniciar un proyecto detallado GO. 

- abrir una auditoría. 




Para obtener más información, consulte [Auditoría dinámica](../Editeurs/1014502.md). 
<a name="NOTE3_11"></a>


### Rendimiento
<a name="rendimiento_ELTPARAGRAPHE000264"></a>

Este widget se utiliza para iniciar el perfilador de rendimiento en el proyecto. El perfilador de rendimiento se utiliza para comprobar y optimizar el tiempo de ejecución de la aplicación.. 

Un clic en la flecha del widget sirve para ampliar el menú del widget. Este menú se utiliza para: 

- habilitar (o no) el perfil de rendimiento. 

	- Si el perfil de rendimiento está activado, se realizará para cada proyecto GO. El archivo correspondiente se guarda en el directorio del proyecto. 

	- Si el widget está desactivado, se utiliza un enlace para volver a activarlo. 




- iniciar el perfil de rendimiento. 

- abrir el perfil de rendimiento. 




Para obtener más información, consulte [Analizador de rendimiento](../Editeurs/2030035.md).

### Pruebas automatizadas
<a name="pruebas_automatizadas_ELTPARAGRAPHE000285"></a>

Este widget se utiliza para conocer el porcentaje de pruebas automatizadas superadas, con errores o no superadas. 

Un clic en la flecha del widget sirve para ampliar el menú del widget. Este menú se utiliza para: 

- realizar todas las pruebas del proyecto.  

- ejecutar todas las pruebas del proyecto que no hayan sido superadas.

- realizar todas las pruebas del proyecto en construcción.

- ejecutar todas las pruebas que detectaron un error en el proyecto.

- mostrar los resultados. 

- Record un nuevo escenario de prueba. 




Para obtener más información, consulte [Pruebas automatizadas](../Editeurs/2019025.md).

### Portada
<a name="portada_ELTPARAGRAPHE000307"></a>

Este widget muestra un gráfico con los porcentajes de código probado y no probado. 

Un clic en la flecha del widget sirve para ampliar el menú del widget. Este menú se utiliza para: 

- Mostrar el resultado del último GO. En este caso, sólo se muestra la cobertura del código con respecto al último GO de la configuración actual.. 

- Mostrar resultados personales. En este caso, se muestra la cobertura del código con respecto al desarrollador actual en la configuración actual.. Se tienen en cuenta los datos de las pruebas GO y automatizadas. 

- Mostrar resultados del equipo. En este caso, se muestra la cobertura del código de todos los desarrolladores que trabajan en la configuración actual.. Se tienen en cuenta los datos de las pruebas GO y automatizadas.

- Activar o desactivar el Widget (para mostrar o no los datos de cobertura de código en el cuadro de mando del proyecto. 

- Mostrar cobertura en el código. Se utiliza para visualizar las líneas cuya prueba se ejecutó en el editor de código.

### Servidores HFSQL
<a name="servidores_hfsql_ELTPARAGRAPHE000369"></a>

Este widget se utiliza para crear accesos directos a los servidores HFSQL. 

El botón "+" se utiliza para añadir un nuevo servidor HFSQL al widget. Todo lo que tiene que hacer es especificar los parámetros de conexión necesarios: servidor, puerto, nombre de usuario y contraseña. 

El menú emergente del servidor propone : 

- conectarse al servidor: El Centro HFSQL control se pone en marcha. 

- eliminar el servidor del widget.

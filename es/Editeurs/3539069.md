
## Tipo y versión del servidor web
			



## Presentación
<a name="presentacion_ELTTEXTE000090"></a>
El diagnóstico de configuración muestra el siguiente error: **"HTTP server: Versión**



Este error indica que se ha detectado un problema con el servidor web.. 

Este error se muestra en los siguientes casos: 

- **Se rechaza la conexión**.

	- No hay ningún servidor Web instalado localmente o el servidor Web no se ha iniciado..
			**Solución**: Instalar, configurar e iniciar un servidor Web en el ordenador local. Esto le permite comprobar localmente los sitios creados con WEBDEV. 

	- El servidor web no está escuchando en el puerto 80. 
			**Solución**: Por Default, los servidores Web están escuchando en el puerto 80. WEBDEV intenta conectarse al puerto porque no hay una configuración específica.
			Si el servidor web no está escuchando en este puerto, el puerto debe añadirse a la pestaña "Advanced" del administrador de WEBDEV, opción "Host Name/IP".: debe añadir, después del nombre de host/IP, el carácter ":" que sigue al puerto.
			Los puertos alternativos más comunes de los servidores Web son 81 y 8080.
			Por ejemplo, con el nombre de host de Default "localhost", si el servidor web está escuchando en el puerto 81, el valor se convierte en "localhost:81".




- **La conexión es aceptada pero la respuesta es inválida**. Este modo de operación puede corresponder a uno de los siguientes casos: 

	- Un programa de tres partidos monopoliza (o bloquea) este puerto.. El puerto no estaba disponible, el servidor web no es accesible.

	- El servidor web aceptó la conexión pero no respondió (o no respondió lo suficientemente rápido).. 




- **El servidor Web detectado no es el especificado en el administrador de WEBDEV** (ficha"Avanzado", botón"Servidor").
	Solución: En la mayoría de los casos, una inversión entre IIS y Apache explica la Error( cuando ambos servidores se utilizan para ejecutar las pruebas por ejemplo). 




**Reminder: Primer diagnóstico**
El primer diagnóstico consiste en iniciar un navegador y conectarse a http://localhost/ (localhost puede ser reemplazado por el host/IP adecuado con un puerto si es necesario).

Normalmente:

- IIS muestra una página con un simple Image que contiene IIS.

- Apache muestra "It works!".

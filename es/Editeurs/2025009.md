
## Ejecutable autónomo
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000092"></a>
El ejecutable será **stand-alone** si los siguientes elementos están incluidos en el ejecutable:

- las librerías WINDEV (archivos".DLL") requeridas por la aplicación.

- las bibliotecas principales y/o externas (archivos".WDL").




Sólo el ejecutable debe estar instalado en los ordenadores de los usuarios finales.. Sin embargo, las DLLs WINDEV (archivos".DLL") y las librerías (archivos".WDL") se registrarán automáticamente en el directorio de lanzamiento del programa ejecutable **whenever** al iniciar el programa ejecutable.. Las DLLs y las bibliotecas que se encuentran en este directorio serán reemplazadas automáticamente.

Para evitar recrear las DLL y las bibliotecas cada vez que se inicie el programa ejecutable, le aconsejamos que cree una instalación autoextraíble. Este tipo de instalación se utiliza para instalar todos los archivos requeridos por la aplicación en los ordenadores de los usuarios finales. El usuario sólo maneja el ejecutable de la aplicación. Para obtener más información, consulte [Programa de instalación](../Editeurs/2028001.md).

## Limitaciones y notas
<a name="limitaciones_notas_ELTTEXTE000116"></a>


### Sustitución de los archivos existentes
<a name="sustitucion_los_archivos_existentes_ELTPARAGRAPHE000028"></a>

**Atención**: Al ejecutar el programa ejecutable, las librerías (archivos".DLL") se extraen automáticamente y sustituyen a las existentes.. Estos reemplazos pueden provocar problemas si se instalan varias aplicaciones WINDEV en el mismo equipo.

Por ejemplo, las aplicaciones denominadas "MyApp1" y "MyApp2" se instalan en el mismo directorio. Cuando se ejecuta el ejecutable de "MyApp1", las librerías (archivos ".DLL") associated con "MyApp1" pueden sobreescribir las librerías usadas por "MyApp2". Si cada ejecutable de estas aplicaciones fue creado con una versión diferente de WINDEV, es posible que algunas características de "MyApp2" no funcionen correctamente.

**Solución**: Para evitar sobrescribir las DLL existentes, le recomendamos que cambie el nombre de las DLL utilizadas por la aplicación al crear el programa ejecutable.
<a name="NOTE2_2"></a>


### Observaciones
<a name="observaciones_ELTPARAGRAPHE000039"></a>

- Incluir bibliotecas (archivos".DLL") y bibliotecas externas (archivos".WDL") en el ejecutable le permite instalar y/o actualizar un solo archivo.. Sin embargo, el tamaño del archivo ejecutable creado es mayor.

- El ejecutable debe volver a crearse en los siguientes casos:

	- si sólo una de las DLLs debe ser actualizada.

	- si una de las bibliotecas externas debe actualizarse.

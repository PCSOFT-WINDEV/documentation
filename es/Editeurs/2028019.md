


## Despliegue de un sitio WEBDEV en un servidor de aplicaciones WEBDEV remoto o en un clúster WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000122"></a>
La instalación de un sitio WEBDEV dinámico (en modo Sesión o AWP) en una servidor de aplicaciones remota requiere:

- Un servidor web.

- El servidor de aplicaciones WEBDEV instalado en el servidor web implementación. 

- La creación de una cuenta WEBDEV en el administrador de WEBDEV, la comunicación del nombre de la cuenta y la contraseña.




Para una instalación en un clúster WEBDEV, también se requieren los siguientes elementos: 

- El servidor de aplicaciones WEBDEV debe estar instalado en modo clúster y el clúster WEBDEV debe contener al menos un nodo. Para obtener más información, consulte [Instalación de un clúster WEBDEV](../Editeurs/1410086635.md). 

- Debe instalarse y configurarse un proxy para utilizarlo con el Cluster WEBDEV. Para obtener más información, consulte [WEBDEV Cluster: Configuración de Proxy](../WDAdminDeploiement/3539085.md). 

- El sitio o servicio web debe instalarse a través del proxy. Por lo tanto, es necesario el apoderamiento de Address.  




Para una instalación vía FTP, también se requieren los siguientes elementos: 

- Un servidor FTP instalado en el servidor Web implementación ** sólo para un implementación de FTP**. El servidor FTP no es necesario para un implementación HTTP.

- La creación de una cuenta FTP, la comunicación del nombre y la contraseña de la cuenta FTP así como la IP Address del servidor.




Notas sobre el servidor de aplicaciones WEBDEV: 

- Para un Servidor de Aplicaciones de la versión 24 o anterior, se puede utilizar cualquier versión de WEBDEV (hasta la versión 24) en el ordenador de desarrollo.

- Para un Servidor de Aplicaciones de la versión 25 o posterior, debe utilizarse WEBDEV 24 o posterior en el ordenador de desarrollo.
	Caso especial: para un implementación remoto desde WEBDEV 24 en un Servidor de Aplicaciones de la versión 25 o posterior, solicite el módulo WWINST.EXE para WEBDEV 24 en la página de recursos de useful:  [https://www.windev.com/ts/useful_resources.htm](https://www.windev.com/ts/useful_resources.htm), "Actualizaciones disponibles para WINDEV, WEBDEV o WINDEV Mobile".

## Etapas a seguir
<a name="etapas_seguir_ELTTEXTE000146"></a>
Los diferentes pasos para la implementación en un servidor remoto son los siguientes:

1. [Introducir información sobre el servidor](../Editeurs/2028046.md). 
	Se establece automáticamente una conexión con el servidor. Se comprueban los nombres de usuario y las contraseñas.
	En este paso se definen todas las opciones de acceso al servidor (protocolo HTTPS, peticiones autenticadas, servidor FTP, etc.).

2. [Selección de los archivos a instalar](../Editeurs/2028058.md).

3. [Modificación automática de archivos de datos](../Editeurs/2028048.md).

4. [Configurando el administrador de WEBDEV y el modo instalación](../Editeurs/2028051.md).

5. Transferencia de archivos y instalación. 
	Los mensajes de error pueden ser mostrados en diferentes pasos de la instalación remota. Para obtener más información, consulte [Errores durante una instalación por FTP](../Editeurs/2028057.md).

6. Cuando la instalación se completa, la asistente provee el comando Line usado para iniciar el sitio. Este Line puede ser copiado (si es necesario) en el portapapeles a través de el menú contextual.

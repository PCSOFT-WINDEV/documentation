


## Convierta sus aplicaciones WINDEV en sitios web
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000215"></a>
A menudo, los desarrolladores necesitan transformar aplicaciones Windows (o partes de aplicaciones) en sitios web dinámicos.

Como ya sabe, WEBDEV permite recuperar el código, las consultas, los reportes, las clases, las ventanas, ... existentes de una aplicación WINDEV. 

WEBDEV va mucho más allá gracias a la webificación asistente.

<a name="NOTE2_1"></a>


## ¿Cómo webificar un proyecto WINDEV?
<a name="¿como_webificar_proyecto_windev_ELTTEXTE000239"></a>


### Consejos y requisitos previos
<a name="consejos_requisitos_previos_ELTPARAGRAPHE000022"></a>

Una aplicación WINDEV solo se puede webificar: 

- A partir de la versión 24,

- usando WINDEV 64 bits con WEBDEV 64 bits incluido. 
	Atención: Debe tener ambas licencias, WINDEV y WEBDEV para realizar este tipo de instalación. Para utilizar la versión integrada WINDEV de WEBDEV, tanto los dongles WEBDEV como WINDEV deben estar conectados al ordenador de desarrollo.

- usando WEBDEV 64 bits. 




Antes de webificar una aplicación WINDEV, se recomienda: 

- Conocer cuál es el objetivo de la aplicación webificada. ¿Son necesarias todas las características de la aplicación en la versión Web? 
	Por ejemplo: 

	- Si la aplicación WINDEV se ocupa de la gestión de sistemas, ¿es necesario webizarla?

	- Si la aplicación WINDEV utiliza funcionalidades que no están disponibles en la web (Scanner, comunicación Bluetooth, tratamiento de textos, etc.), éstas no podrán ser migradas y requerirán un desarrollo específico si es necesario. 




- Verificar la partición de código Business/GUI de la aplicación. Esto puede ser verificado usando WINDEV. 
	La función de división "Lógica empresarial/código GUI" simplifica la migración y evita la duplicación de código en las ventanas y en las páginas.. 
	Si esta acción no se realiza antes de la webificación, es posible hacerlo después.. 

- Asegúrese de que los componentes utilizados por la aplicación sean multiproducto.. 





### Iniciar la webificación de asistente de WINDEV (con WEBDEV incluido)
<a name="iniciar_webificacion_asistente_windev_con_webdev_incluido_ELTPARAGRAPHE000048"></a>

Para iniciar la webificación del proyecto WINDEV: 

1. Abrir el proyecto en WINDEV. 

2. En la pestaña "Proyecto", en el grupo "Webificación", despliegue "Webificar" y seleccione "Webificar". La webificación de asistente comienza. Esta asistente creará una configuración de proyecto específica para los elementos del sitio. 

3. Seleccione los elementos a convertir, especialmente las ventanas de su aplicación. 
	Observaciones: 

	- Si algunos artículos ya han sido convertidos, la asistente enumera los artículos convertidos y no convertidos. 

	- Comience convirtiendo una o dos ventanas de su aplicación. Puede convertir las otras ventanas más tarde. 




4. Pase a la etapa siguiente. 

5. Anule la selección, si es necesario, de los elementos del proyecto que no deben incluirse en la configuración del proyecto del centro.: informes, consultas, clases, ... Los elementos comprobados se integran automáticamente en la obra. 

6. Pase a la etapa siguiente del asistente: se creará la configuración del sitio y las ventanas seleccionadas se "webificarán".. 

7. Valide el asistente. 
	 Observación: Esta operación puede ser relativamente larga, dependiendo del número de elementos seleccionados (5 a 10 segundos por elemento).. 

8. La auditoría de webificación del proyecto se ejecuta automáticamente. 




Observación: En cualquier momento: 

- La webificación de asistente se puede ejecutar a través del menú WINDEV: en la pestaña "Proyecto", en el grupo "Webificación", despliegue "Webificar" y seleccione "Webificar".

- Es posible webificar una ventana mediante la opción "Webificar" del menú contextual de la ventana en el explorador de proyectos. 

- Es posible copy/pegar código o elementos entre una ventana y un Page. 





### Inicie la webificación asistente de WEBDEV
<a name="inicie_webificacion_asistente_webdev_ELTPARAGRAPHE000099"></a>

Para iniciar la webificación del proyecto WINDEV: 

1. Abrir el proyecto WINDEV en WEBDEV. 

2. La webificación de asistente comienza. Esta asistente creará una configuración de proyecto específica para los elementos del sitio. 

3. Seleccione los elementos a convertir, especialmente las ventanas de su aplicación. 
	Observaciones: 

	- Si algunos artículos ya han sido convertidos, la asistente enumera los artículos convertidos y no convertidos. 

	- Comience convirtiendo una o dos ventanas de su aplicación. Puede convertir las otras ventanas más tarde. 




4. Pase a la etapa siguiente. 

5. Anule la selección, si es necesario, de los elementos del proyecto que no deben incluirse en la configuración del proyecto del centro.: informes, consultas, clases, ... Los elementos comprobados se integran automáticamente en la obra. 

6. Pase a la etapa siguiente del asistente: se creará la configuración del proyecto de sitio y se webificarán las ventanas seleccionadas. 

7. Valide el asistente. 
	 Observación: Esta operación puede ser relativamente larga, dependiendo del número de elementos seleccionados (5 a 10 segundos por elemento).

8. La auditoría de webificación del proyecto se ejecuta automáticamente. 




Observación: La webificación de asistente puede ser ejecutada en cualquier momento a través del menú de WEBDEV: en la pestaña "Proyecto", en el grupo "Webificación", despliegue "Webificar" y seleccione "Webificar".


### Siga los consejos de la auditoría de webificación
<a name="siga_los_consejos_auditoria_webificacion_ELTPARAGRAPHE000134"></a>

La auditoría de webificación se inicia automáticamente después de que la webificación asistente termina, y puede ser ejecutada en cualquier momento a través del menú WINDEV: en la pestaña "Proyecto", en el grupo "Webificación", amplíe "Webificar" y seleccione "Auditoría de Webificación".
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Webisation%20-%20HC%20N%B0001.gif&type=thumb)


Esta auditoría permite corregir los errores de compilación sugiriendo algunas soluciones. 

Todos los errores de compilación encontrados también se muestran en el panel de errores.. 

Para completar la webificación de la aplicación, se deben corregir todos los errores. Algunos errores pueden estar relacionados con procesos o funciones que no están disponibles en la Web, funciones de WLanguage no disponibles, etc.

Atención: En algunos casos, es posible que tenga que desarrollar nuevas funcionalidades para adaptar el comportamiento de la aplicación a sitios web. Por ejemplo: 

- Si la aplicación usó un control de edición de Image, puede ser necesario usar un control Upload, que permitirá descargar el Image modificado . 

- Si la aplicación utilizaba un control Procesador de texto, puede ser necesario utilizar un control HTML y adaptar el análisis para gestionar el HTML. 





### Terminar la webificación
<a name="terminar_webificacion_ELTPARAGRAPHE000164"></a>

Cuando se corrigen todos los errores de compilación y de auditoría de webificación y se realizan las adaptaciones de gestión, se completa la webificación.. 

Otras funcionalidades pueden ser añadidas más tarde, como Responsive Web Design. Puede ser una buena idea usar Dynamic Serving temporalmente para reemplazar las diferentes páginas a medida que avanza.

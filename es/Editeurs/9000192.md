
## Mostrar el código de un elemento
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000134"></a>
WINDEV, WEBDEV y WINDEV Mobile permiten la programación orientada a eventos y objetos. 

La programación basada en eventos asocia uno o más eventos a cada elemento utilizado. Estos eventos se muestran en el editor de código. 

Para un gran número de elementos: 

- Algunos eventos se proponen de forma predeterminada: los eventos más comunes y usados. 

- Se puede agregar algunos eventos. Estos eventos permiten gestionar casos específicos (por ejemplo, la acción de arrastrar y soltar). Para obtener más información, consulte [Eventos opcionales](../WDChamp/1014004.md).

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000158"></a>


### ¿Cómo ver los eventos asociados a un elemento?
<a name="¿como_ver_los_eventos_asociados_elemento_ELTPARAGRAPHE000026"></a>

Existen varios métodos para ver los eventos asociados a un elemento (control, ventana, página, etc.): 

- **Utilizar la tecla F2**. Para ello: 

	- Seleccione el elemento deseado. 

	- Presione F2: el editor de código se abre automáticamente con la lista de eventos predeterminados. 




- **Utilizar la opción "Código" del menú contextual**. Para ello: 

	- Seleccione el elemento deseado. 

	- Haga clic derecho para abrir el menú contextual y seleccione "Código". El editor de código se abre automáticamente con la lista de eventos predeterminados. 







Observaciones: 

- También se pueden ver los eventos asociados a un elemento desde la ventana de descripción: simplemente haga clic en el ícono ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Detail_Description_FEN_btnCode%20-%20HC%20N%B0001%201.gif), en la parte inferior de la ventana de descripción. 

- Para ver los eventos asociados al proyecto, a una ventana, página o reporte, también puede utilizar las pestañas de documentos abiertos. Para ello: 

	- Abra el menú contextual del elemento deseado. 

	- Seleccione "Código del elemento" en el menú que aparece. El editor de código se abre automáticamente con la lista de eventos predeterminados.






<a name="NOTE2_3"></a>


### Caso especial: abrir los eventos asociados a un elemento desde el editor de código
<a name="caso_especial_abrir_los_eventos_asociados_elemento_desde_editor_codigo_ELTPARAGRAPHE000058"></a>

El código WLanguage de los diferentes procesos o eventos manipula elementos específicos (controles, ventanas, páginas, etc.). 

Puede abrir directamente los eventos de un elemento manipulado en una línea de código haciendo clic en el elemento y pulsando F2. 

Por ejemplo: Para el siguiente código: 

```wl
Open(WIN_Menu)
```
Si se pulsa F2 cuando el cursor está sobre "WIN_Menu", el editor de código abrirá los eventos asociados a esa ventana.
<a name="NOTE2_4"></a>


### Caso especial: F2 sobre el nombre de un plano
<a name="caso_especial_sobre_nombre_plano_ELTPARAGRAPHE000074"></a>

Si el código WLanguage manipula un plano con nombre, pulse F2 sobre el nombre del plano para abrir la ventana correspondiente e ir al plano especificado. 
<a name="NOTE2_5"></a>


### Consejo: Volver al último código modificado
<a name="consejo_volver_ultimo_codigo_modificado_ELTPARAGRAPHE000083"></a>

Después de escribir una línea de código, solemos navegar por el código para verificar ciertos elementos. Luego, tenemos que volver al código inicial para continuar con nuestra tarea.

Para volver al código inicial, presione Ctrl + Alt + Flecha izquierda.

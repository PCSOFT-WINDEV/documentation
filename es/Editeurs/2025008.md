


## Creación de un ejecutable: Información y número de versión
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000136"></a>
La información de la versión de un ejecutable corresponde a la información de Visible en el explorador de archivos de Windows. Esta información se muestra en la pestaña "Versión" o "Detalles" de la ventana de propiedades del archivo ejecutable.

**Observación**: Para crear el ejecutable, utilice uno de los siguientes métodos: 

- En la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar". 

- Despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_exe.gif) en los botones de acceso rápido y seleccione "Generar el ejecutable de Windows de 32 bits (exe)" o "Generar el ejecutable de Windows de 64 bits (exe)".

## Información del ejecutable
<a name="informacion_del_ejecutable_ELTTEXTE000160"></a>
Esta información se puede recuperar mediante la programación con [ExeInfo](../WDLang1/3035001.md). Esta información es la siguiente:

- el nombre del **compañía**. Por ejemplo, el nombre del editor.

- un breve **description** del propósito de la aplicación. Por ejemplo: "Gestión de inventario".

- los **derechos de autor** de la solicitud

- **la URL del sitio web**: Puede especificar la URL de su sitio, por ejemplo.

- **el correo electrónico Address**: Envía un correo electrónico a Address donde los usuarios de la aplicación puedan Contact arte.




Este Address también será utilizado por el componente de retroalimentación para enviar sugerencias y notificar problemas relacionados con su solicitud. 
**Recordatorio**: El componente Feedback es añadido automáticamente a su aplicación por el programa [Menú automatic? (menú de ayuda de la aplicación)](../Editeurs/9000070.md).

## Versión del ejecutable
<a name="version_del_ejecutable_ELTTEXTE000184"></a>
El **versión de un ejecutable** se utiliza para identificar las características disponibles (o no) en la aplicación utilizada por el usuario. Pero esta versión también es muy importante porque si la aplicación se utiliza en red, la versión le permite saber si una actualización está disponible y es necesaria.. Este número de versión se puede recuperar mediante la programación con [ExeInfo](../WDLang1/3035001.md)

Hay dos formatos disponibles para los números de versión:

- **Formato para la compatibilidad.** Este formato se utilizó hasta la versión 12 para las aplicaciones WINDEV.
	**Observación**: Todos los caracteres (dígitos y letras) están permitidos. Por ejemplo: "1.01A".
	Si la aplicación debe instalarse en red, le aconsejamos que lo haga:

	- utilizar un número de versión diferente cada vez que se cree el programa ejecutable. De hecho, cuando se instala una actualización, este número de versión se utiliza para comprobar si la actualización es obligatoria o no.

	- incrementar automáticamente los últimos caracteres (dígitos o letras) del número de versión cada vez que se crea el programa ejecutable (marque "Incrementar automáticamente la versión en cada generación"). 
			Por ejemplo:
			- el número de versión "1.01A" se convertirá en "1.01B"..
			- el número de versión "A15Z" se convertirá en "A16A".




- **Formato estándar.** Este formato es el formato estándar gestionado por Windows. Este formato corresponde a una set de 4 números separados por puntos. 

	- El primer grupo de 4 dígitos corresponde al número de cambios importantes realizados en la aplicación.

	- El segundo grupo de 4 dígitos corresponde al número de cambios menores realizados en la aplicación.

	- El tercer grupo de 4 dígitos corresponde al número de generación. 
			Le aconsejamos que incremente automáticamente este número en cada generación. Esta operación de incremento se realiza automáticamente si se selecciona la opción "Incrementar automáticamente la versión en cada generación".

	- El último grupo de 4 dígitos corresponde al número de revisiones (ramas en la SCM por ejemplo).

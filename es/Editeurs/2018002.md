## Presentación
<a name="presentacion_ELTTEXTE000192"></a>
Se gestionan dos categorías de archivos de datos:

- **los ficheros de datos utilizados para identificar a los usuarios**. Estos archivos de datos pueden ser comunes a varias aplicaciones. 
	Por Default, estos archivos de datos se crean en un subdirectorio de su proyecto (EXE\\GPW_&lt;Nombre del proyecto&gt; directorio). Estos archivos se utilizan para ejecutar la prueba de su aplicación. 
	Para ejecutar la prueba de su aplicación, puede especificar un directorio diferente o utilizar el directorio que contiene los datos de la aplicación (esta opción se recomienda para las pruebas ejecutadas en Windows Vista con el UAC habilitado).. 

- **los ficheros de datos utilizados para gestionar los derechos concedidos a los usuarios en las diferentes ventanas o páginas que se encuentran en la aplicación**. Estos archivos son específicos de la aplicación. 
	Por Default, estos archivos de datos se crean en un subdirectorio de su proyecto (EXE\\GPW_&lt;Nombre del proyecto&gt; directorio). Estos archivos se utilizan para ejecutar la prueba de su aplicación. 
	Para ejecutar la prueba de su aplicación, puede especificar un directorio diferente o utilizar el directorio que contiene los datos de la aplicación (esta opción se recomienda para las pruebas ejecutadas en Windows Vista con el UAC activado).




![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Observación**: Al crear el ejecutable (ejecutable multiinstancia), tiene la posibilidad de definir la ubicación de los archivos de groupware usuario. Para obtener más información, consulte [Ejecutando la prueba de el groupware usuario y usándola](../Editeurs/2018009.md).

## Los diferentes archivos
<a name="los_diferentes_archivos_ELTTEXTE000216"></a>


### Archivos de datos utilizados para identificar a los usuarios
<a name="archivos_datos_utilizados_para_identificar_los_usuarios_ELTPARAGRAPHE000042"></a>

Estos archivos pueden ser comunes a varias aplicaciones:


|   |   |
| --- | --- |
| ***gpwUser*** | Fichero de usuarios. Contiene todas las características de los usuarios (login, apellido, nombre, contraseña,...) |
| ***Configuración del GPWUser*** | Archivo de grupos de usuarios. |
| ***GPWConnectionHistoria*** | Archivo que contiene el historial de conexiones. Este archivo sólo se encuentra si se ha solicitado el historial de conexiones. |

**¿Dónde se crean estos archivos?**

- Mediante Default, estos archivos de datos (modo HyperFileSQL Classic) se crean en un subdirectorio de su proyecto (directorio EXE\\GPW_<:Nombre del proyecto&gt;). Cuando se incluye el User Groupware, se puede especificar un directorio diferente para ejecutar las pruebas de su aplicación. Usted tiene la capacidad de usar los archivos de datos de HyperFileSQL Client/Server. Para obtener más información, consulte [Archivos de Groupware en formato HyperFileSQL Client/Server](../Editeurs/2018011.md).
	![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Al crear el ejecutable, se puede especificar un directorio específico para estos archivos de datos comunes.

- Al crear el programa instalación, puede permitir a los usuarios configurar la ubicación de estos archivos (plano "Módulos adicionales"). En este caso, la ubicación de los archivos de groupware usuario se almacenará en el ejecutable y se actualizará al instalar el ejecutable. Por lo tanto, se puede seleccionar un directorio común para varias aplicaciones que manejan el groupware usuario.

- Si se ha seleccionado "Inicio manual", [gpwOpen](../WDLang6/3041002.md) se utiliza para iniciar la ventana o página de inicio de sesión del User Groupware, así como para personalizar el directorio de los archivos de datos que se encuentran en el User Groupware. Independientemente de los directorios especificados al crear e instalar la aplicación, este directorio se tendrá en cuenta si se especifica.




**Observación**: si no se realiza ninguna configuración específica, los archivos de datos de usuario (HyperFileSQL Classic) se crearán en un subdirectorio del directorio instalación del ejecutable (o biblioteca para un sitio WEBDEV) llamado \\GPW_&lt;Nombre del proyecto&gt;

Para más detalles:

- sobre la estructura de estos archivos, véase [Estructura de los archivos de groupware usuario](../Editeurs/2018005.md)

- sobre la gestión de archivos de datos en una aplicación Cliente/Servidor, véase [Grupo de usuarios y cliente/servidor](../Editeurs/2018011.md).

### Archivos de datos utilizados para la gestión de los derechos
<a name="archivos_datos_utilizados_para_gestion_los_derechos_ELTPARAGRAPHE000090"></a>

Estos archivos son específicos de la aplicación. No se pueden compartir con otra aplicación.


|   |   |
| --- | --- |
| ***GPWConfiguración*** | Archivo que contiene la description de las diferentes configuraciones disponibles. Cada configuración es associated con un grupo de usuarios. |
| ***GPWElementConfiguración*** | Status de cada elemento de aplicación para cada configuración. Por ejemplo, permite almacenar si el "menú de estadísticas" debe ser Grayed para el usuario "asistente". |
| ***GPWElement*** | Archivo que contiene los diferentes elementos de la aplicación a configurar (controles de ventana o página, opciones de menú, ....) |

**¿Dónde se crean estos archivos?**

- Mediante Default, estos archivos de datos (modo HyperFileSQL Classic) se crean en un subdirectorio de su proyecto (directorio EXE\\GPW_<:Nombre del proyecto&gt;). Usted tiene la capacidad de usar los archivos de datos de HyperFileSQL Client/Server. Para obtener más información, consulte [Archivos de Groupware en formato HyperFileSQL Client/Server](../Editeurs/2018011.md).

- Al crear el programa instalación, puede permitir a los usuarios configurar la ubicación de estos archivos (plano "Módulos adicionales"). Tiene la posibilidad de seleccionar un directorio específico para los archivos de derechos de la aplicación.

- Si se selecciona "Ejecución manual", [gpwOpen](../WDLang6/3041002.md) se utiliza para iniciar la ventana o página de inicio de sesión del User Groupware y para personalizar el directorio de archivos de datos para el User Groupware. Independientemente de los directorios especificados al crear e instalar la aplicación, este directorio se tendrá en cuenta si se especifica.




**Observación**: si no se realiza ninguna configuración específica, los archivos de datos de derechos (HyperFileSQL Classic) se crearán en un subdirectorio del directorio instalación del ejecutable (o biblioteca para un sitio WEBDEV) llamado \\GPW_&lt;Nombre del proyecto&gt;

Para más detalles:

- sobre la estructura de estos archivos, véase [Estructura de los archivos de groupware usuario](../Editeurs/2018005.md)

- sobre la gestión de archivos de datos en una aplicación Cliente/Servidor, véase [Grupo de usuarios y cliente/servidor](../Editeurs/2018011.md).

### Contraseña de los archivos de datos
<a name="contrasena_los_archivos_datos_ELTPARAGRAPHE000134"></a>

Por Default, los archivos de datos que se encuentran en el Grupo de Usuarios tienen una contraseña específica. Esta contraseña es: ***"PCSGPW2001***".

Para cambiar esta contraseña:

1. Mostrar la ventana de configuración de el groupware usuario ("Workshop .. Configure el groupware usuario").

2. En la pestaña "Ficheros" introduzca la nueva contraseña. Los caracteres escritos se muestran como estrellas.




**Atención**: Si los archivos de software de trabajo en grupo ya existen, su contraseña no se modifica.. Esta modificación sólo puede realizarse antes de que se creen los archivos de datos.

**Observación**: Esta contraseña se utiliza cuando los archivos son re-indexados o cuando los archivos son abiertos por WDMAP.




## WDInst: configurar el directorio de los archivos de datos y el directorio de los archivos de groupware usuario
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000227"></a>
El editor WDInst de instalación le permite agregar opciones adicionales al programa de instalación. Estas opciones adicionales permiten al usuario especificar:

- el directorio de los ficheros de datos de la aplicación.

- el directorio de los archivos de groupware usuario.




Estas opciones también pueden ser añadidas usando el programa de creación de instalación asistente.

Recordatorio: Para abrir WDInst, en la pestaña "Herramientas", en el grupo "Utilidades", haga clic en "WDInst".

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Permitir al usuario configurar el directorio de archivos de datos
<a name="permitir_usuario_configurar_directorio_archivos_datos_ELTTEXTE000251"></a>


### Directorio donde se crean los archivos de datos
<a name="directorio_donde_crean_los_archivos_datos_ELTPARAGRAPHE000037"></a>

Por Default, los archivos de datos se crean en los directorios especificados al describir los archivos en el análisis. Sin embargo, estos directorios pueden ser modificados:

- al crear el ejecutable de la aplicación.

- mediante la programación con [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md).

- al instalar la aplicación. El directorio para crear los archivos de datos será seleccionado por el usuario.



<a name="NOTE2_2"></a>


### Ajuste durante la instalación
<a name="ajuste_durante_instalacion_ELTPARAGRAPHE000054"></a>

Si su aplicación utiliza archivos de datos, puede ***permitir al usuario que set la ubicación de los archivos de datos*** al instalar la aplicación.

El directorio seleccionado por el usuario se utilizará automáticamente para todos los ficheros de datos creados por la aplicación.. Se almacena directamente en el ejecutable, sin usar el registro o un archivo de parámetros..
Observación: Este directorio será propuesto por Default al instalar las actualizaciones de la aplicación.

Esta opción permite evitar la programación de la ubicación de los archivos de datos (con [HSubstDir](../WDLang4/3044028.md), por ejemplo).

**Recordatorio**: En cualquier caso, el directorio para la creación de los ficheros de datos se define según el siguiente orden de prioridad:

1. Directorio especificado por [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md).

2. Directorio seleccionado al instalar la aplicación (si esta opción se propone al usuario).

3. Directorio seleccionado al crear el ejecutable.

4. Directorio especificado al describir los archivos en el análisis.




Por lo tanto, el directorio definido por [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md) tiene prioridad sobre todos los demás directorios.
<a name="NOTE2_3"></a>


### Caso especial: los archivos de datos se instalan con la aplicación
<a name="caso_especial_los_archivos_datos_instalan_con_aplicacion_ELTPARAGRAPHE000091"></a>

Estos archivos de datos se instalan en el directorio especificado en WDInst (directorio de destino). 
Para una red instalación, estos archivos de datos se instalarán en el subdirectorio "&lt;ExecutableName&gt;" del directorio de la aplicación instalación en el servidor.
<a name="NOTE2_4"></a>


### Permitir al usuario configurar el directorio de archivos de datos
<a name="permitir_usuario_configurar_directorio_archivos_datos_ELTPARAGRAPHE000100"></a>

Para permitir al usuario configurar la ubicación de los archivos de datos: 

1. Seleccione "Parámetros de configuración ... archivos HFSQL".

2. Haga clic en la pestaña "Conexiones. 

3. Seleccione la opción "Editar ubicación de los archivos de datos". 

4. Haga clic en el botón "Parámetro. 

5. Seleccione la opción "Los valores introducidos durante la instalación (indique los valores predeterminados)".

## Permitir al usuario configurar el directorio de los archivos de groupware usuario
<a name="permitir_usuario_configurar_directorio_los_archivos_groupware_usuario_ELTTEXTE000293"></a>


### Directorio para la creación de los archivos de datos de la Agrupación de Usuarios
<a name="directorio_para_creacion_los_archivos_datos_agrupacion_usuarios_ELTPARAGRAPHE000123"></a>

Si su aplicación utiliza el [Groupware Usuario](../Editeurs/2018001.md), se admiten dos categorías de archivos de datos:

- ***los ficheros de datos utilizados para identificar a los usuarios***. Estos archivos de datos pueden ser comunes a varias aplicaciones.

- ***los ficheros de datos utilizados para gestionar los derechos concedidos a los usuarios en las diferentes ventanas de la aplicación***. Estos archivos son específicos de la aplicación.




Para obtener más información, consulte [Los archivos de el groupware usuario](../Editeurs/2018002.md).

Por Default, estos archivos de datos se crean en el subdirectorio "GPW_&lt;ApplicationName&gt;" del directorio de la aplicación instalación. Sin embargo, este directorio puede ser modificado. ### Permitir al usuario configurar el directorio de los archivos de datos de groupware usuario
<a name="permitir_usuario_configurar_directorio_los_archivos_datos_groupware_usuario_ELTPARAGRAPHE000171"></a>

Para permitir que se configure el directorio de los archivos de datos de groupware usuario: 

1. Seleccione "Parámetros de configuración ... archivos HFSQL".

2. Seleccione la opción "Configurar el directorio de los archivos del groupware durante la instalación".

## Presentación
<a name="presentacion_ELTTEXTE000816"></a>
Las opciones generales se utilizan para configurar el entorno de trabajo del producto utilizado.

Estas opciones afectan a:

- El promotor (["Ficha "Usuario](#NOTE2_1)).

- El inicio, la copia de seguridad del archivo, la velocidad del entorno y la SCM ([Pestaña "General"](#NOTE3_1)).

- El tema del editor, los paneles, el efecto FishEye y el reinicio del entorno (["Editor" tab](#NOTEEDIT_1)).

- Los ajustes de traducción, para traducir fácilmente sus interfaces (["Pestaña "Traducción](#NOTE5_1)).

- Los directorios de los componentes opcionales (["Pestaña "Directorios](#NOTE6_1)).

- La configuración del proxy para los editores ([pestaña "Proxy"](#NOTE_PROXY)). 

- El modo de gestión de la ayuda en línea ([pestaña "Ayuda"](#NOTE9_1)).

- Los mensajes que ya no se muestran (["Pestaña "No volver a mostrar](#NOTE7_1)).

- El desarrollo de aplicaciones asistidas (["Pestaña "AAD](#NOTE8_1)).




![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Para abrir las opciones de WINDEV**: en la pestaña "Opciones", en el grupo "Inicio", despliegue "Entorno" y seleccione "Opciones generales de WINDEV".

## "Ficha "Usuario
<a name="ficha_usuario_ELTTEXTE000840"></a>


### Usuario y contraseña
<a name="usuario_contrasena_ELTPARAGRAPHE000115"></a>

La pestaña "Usuario" permite gestionar las características del usuario actual. 

Es posible: 

- Identificar al usuario actual y cambiar su contraseña si es necesario (botón "Contraseña"). Para cambiar una contraseña, basta con escribir:

	- la contraseña actual, 

	- la nueva contraseña y su confirmación.




- **Solicitar contraseña al iniciar**: Esta opción se utiliza para habilitar (o no) la ventana de inicio de sesión al iniciar el producto.. Esta ventana se utiliza para introducir el nombre de usuario y la contraseña.

- **Mostrar nombre de usuario en el título del editor**: Permite mostrar el nombre del usuario actual en la barra de título de todos los editores. 

- **Sincronizar entorno en los Centros de Control**: Esta opción se utiliza para salvar el medio ambiente en los Centros control. En este caso, el entorno definido por el desarrollador (posición de los paneles, paneles mostrados,...) será relacionado al login para conectarse a WINDEV, WEBDEV o WINDEV Mobile.. El usuario podrá acceder a su entorno personalizado desde cualquier ordenador relacionado a la base de datos del Centro control. 

- **Modificar los parámetros de los centros de control** (botón "Parámetros de los Centros de Control"). Esto permite conocer la configuración actual de los archivos de datos que se encuentran en los Centros control y redefinir estos parámetros si es necesario. Para obtener más información, consulte [Base de datos de los Centros de Control](../CCSuivi/3540720.md).

### Inicio
<a name="inicio_ELTPARAGRAPHE000170"></a>

En la sección "Inicio" aparecen las siguientes opciones: 

- **Idioma del editor**: Esta opción permite elegir el idioma de visualización del editor: Inglés, español o francés. 

- **Cargar último proyecto**: Esta opción se utiliza para cargar automáticamente el último proyecto abierto para el producto actual.

- **Últimos proyectos utilizados** y **Últimos documentos utilizados**: Estas opciones permiten configurar el número de últimos proyectos y el número de últimos elementos mostrados en esta lista. Simplemente seleccione el elemento o proyecto deseado para abrirlo directamente en el editor correspondiente.
	Los últimos documentos y proyectos utilizados se muestran al final de las opciones utilizadas para abrir un proyecto.: 

	- ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_ouvrir.gif) en los botones de acceso rápido. 

	- en la pestaña "Inicio", en el grupo "General", despliegue "Abrir". Se muestran los últimos archivos utilizados ("Archivos recientes") y los últimos proyectos utilizados ("Proyectos recientes").: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Fichiers_Recent.gif)



 Observaciones:

- El máximo número de proyectos que se pueden mostrar en esta lista es de set a 10.

- El máximo número de elementos que se pueden mostrar es set a 10.

- **gestión de DPI**: Diferentes modos de gestión de DPI para la visualización de los editores WINDEV, WEBDEV y WINDEV Mobile: 

	- **Permitir que Windows administre el DPI**: Esta opción se recomienda cuando el modo Default no muestra correctamente la fuente utilizada para las opciones del editor, especialmente en pantallas pequeñas con un alto DPI (por ejemplo, Macbook Retina). Cuando se utilizan varios monitores con diferentes ajustes de DPI, las imágenes y las fuentes pueden aparecer borrosas.

	- **Utilizar DPI de la pantalla principal**: Si se selecciona esta opción, los editores WINDEV, WEBDEV y WINDEV Mobile se muestran según la resolución de la pantalla principal. Esta opción se recomienda cuando se utilizan varias pantallas con la misma configuración de PPP que la pantalla principal. 

	- **DPI independiente por pantalla** (opción Default ): Esta opción le permite utilizar los editores WINDEV, WEBDEV y WINDEV Mobile en varias pantallas con diferentes ajustes de DPI. Los entornos se muestran claramente, independientemente de la pantalla. Cuando se cambia entre diferentes pantallas (con diferentes DPIs), los editores y los elementos mostrados cambian de tamaño correctamente.





**Atención**: este ajuste de DPI sólo afecta a los editores para el desarrollo solamente. No tiene ningún efecto sobre las aplicaciones desarrolladas. Para las aplicaciones desarrolladas, esta configuración se puede modificar en la ventana description (pestaña " UI ", "Mostrar en fuentes grandes: Ampliar la ventana y los controles").

### reserva
<a name="reserva_ELTPARAGRAPHE000231"></a>

Las opciones de copia de seguridad son las siguientes: 

- **Proponer que se guarde un elemento en el momento en que se cree**: Esta opción (marcada por Default) se utiliza para abrir la ventana de guardado del elemento en cuanto se crea. Esta copia de seguridad se utiliza para permitir inmediatamente la realización de los nombres de los controles., ... Para volver al comportamiento de las versiones anteriores, basta con desmarcar esta opción. Esta opción está disponible a partir de la versión 22.

- Si se marca la opción **" Gestionar copia de seguridad automática de los elementos del proyecto (¿.bk? y ¿.tk?) "** , se generarán dos archivos de almacenamiento para cada elemento manejado (ventana, reporte , clase, etc.):

	- **Archivos ".BK\*" (".BKW", ".BKE", ".BKR", etc.)**
			El archivo BK\* se crea durante la primera copia de seguridad del objeto después de abrir el proyecto. Corresponde a la versión del objeto cuando se abrió el proyecto.
			El archivo BK\* le permite recuperar un proyecto, ... en el estado en que se encontraba antes de abrir el proyecto en el editor.

	- **Archivos ".TK\*" (".TKW", ".TKE", ".TKR", etc.)**
			El archivo TK\* se crea cada vez que se guarda el objeto. Corresponde al status N-1 del objeto..
			El archivo TK\* se utiliza para recuperar un proyecto, una ventana, ... en el estado en que se encontraba antes de la última copia de seguridad realizada en el editor.


Los archivos ".BK\*" y ".TK\*" se guardan en el directorio "Backup" del proyecto.

- **Permitir que los proyectos procedentes de una versión anterior se abran sin copia de seguridad**: Por Default, se realiza una copia de seguridad del proyecto automáticamente cuando se abren los proyectos desarrollados con una versión anterior. Esta copia de seguridad se realiza en el directorio del proyecto. Si esta opción está marcada, aparecerá un aviso de copia de seguridad al abrir el proyecto en una versión posterior. 

- **Copia de seguridad para la recuperación automática cada XXX**: 
	Para evitar problemas cuando se produce un bloqueo del sistema (por ejemplo, un corte de energía), WINDEV, WINDEV y WINDEV Mobile proponen un sistema para recuperar automáticamente los elementos (consultas, informes, ventanas, etc.) abiertos en los diferentes editores.. Este mecanismo se activa mediante la opción "Copia de seguridad para recuperación de automatic cada XX minutos"..
	Cuando este sistema está habilitado:

	- Un respaldo de automatic de los elementos abiertos se realiza en intervalos de tiempo específicos (cada 10 minutos por Default).

	- Cuando se reabre un elemento después de un bloqueo del sistema, aparece una ventana de recuperación de automatic. Esta ventana permite recuperar:

		- el elemento tal y como estaba durante la última copia de seguridad realizada por el desarrollador.

		- el elemento guardado automáticamente por WINDEV, WEBDEV o WINDEV Mobile.


 La versión seleccionada se abre automáticamente en el editor correspondiente.. La versión no seleccionada está disponible en la ventana de gestión de versiones ("Archivo .. Versiones") y puede abrirse en cualquier momento. Para obtener más información, consulte [Gestión de versiones](../Editeurs/2021002.md).

- **Forzar el flush**: Por Default, todas las operaciones de escritura realizadas por WINDEV, WEBDEV o WINDEV Mobile (guardando los elementos del proyecto, ...) no se realizan inmediatamente en los discos de la red.
	Para forzar la escritura y evitar el uso de "cachés", marque "Forzar el flush". Esta opción puede ralentizar algunas características del producto utilizado (compilación del proyecto, por ejemplo).. Le recomendamos que utilice esta opción para los proyectos compartidos entre varios desarrolladores.
	**Observación**: Esta opción no tiene efecto en la ejecución de las aplicaciones WINDEV, WEBDEV y WINDEV Mobile.

### Acelerar el medio ambiente
<a name="acelerar_medio_ambiente_ELTPARAGRAPHE000300"></a>

Para acelerar el ambiente, usted puede hacerlo:

- Elija el modo de compilación del proyecto: Los modos de compilación son: 

	- **Mononúcleo**: La compilación utiliza un solo núcleo. Los diferentes elementos del proyecto se compilan cuando se abre el proyecto. También puede solicitar una recompilación del proyecto (vaya a la pestaña "Proyecto", grupo "Proyecto", despliegue "Recompilar y sincronizar" y seleccione "Recompilar proyecto").. Esta recompilación puede realizarse antes de crear el ejecutable, por ejemplo.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **En segundo plano**: La compilación en Background permite una recompilación sin bloqueo del proyecto. Puede seguir utilizando su software durante la recompilación.

	- **Multinúcleo**: Todos los núcleos se utilizan para compilar el proyecto: por lo que es más rápido que la compilación estándar. A partir de la versión 26 U2, este modo de recompilación es seleccionado por Default. Esta recompilación se activa en los siguientes casos: 

		- cuando se abre el proyecto, si es necesaria una recompilación completa del proyecto. 

		- si se solicita explícitamente la recompilación del proyecto: en la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Recompilar y sincronizar" y seleccione "Recompilar proyecto" o "Reparar proyecto".

		- al cambiar la configuración de un proyecto, sólo si esta configuración de proyecto aún no ha sido activada en el ordenador actual. 


Observación: En Default, se utilizarán todos los núcleos del procesador si se elige la compilación "Multi-core".. Para una configuración más específica, puede elegir el número de hilos a utilizar. Esto permite manipular casos especiales y permite que otros procesos sigan funcionando a una velocidad razonable.

- **Desactivar reglas de negocio**: Cargar las reglas de negocio que se encuentran en la base de datos de los Centros de Control puede ralentizar la visualización de los elementos en el editor o la carga de la ventana de description. Esta opción se recomienda cuando se utiliza una red lenta o durante un desarrollo a través de la nube.. 
	Atención: En este caso, las reglas de negocio están deshabilitadas.

- **Desactivar corrector ortográfico**. Por Default, el ambiente utiliza el corrector ortográfico Open Office. Este corrector te permite revisar la ortografía en tiempo real de get usando el diccionario Open Office. Usted tiene la posibilidad de desactivar esta ficha. Para más detalles sobre el corrector ortográfico, véase [Revisar ortografía](../Editeurs/1013146.md).

- **Desactivar reconocimiento de voz**. El entorno responde a los comandos de voz (desde Windows Vista). El reconocimiento de voz se puede desactivar. Para obtener más información, consulte [Comandos de voz en el entorno](../Editeurs/2020010.md).

### Administrador de Código Fuente (SCM)
<a name="administrador_codigo_fuente_scm_ELTPARAGRAPHE000374"></a>

Las opciones de WINDEV, WEBDEV y WINDEV Mobile le permiten configurar el modo de check-out para los elementos de un proyecto que se encuentran en el SCM.

Hay dos modos de pago disponibles para los elementos del proyecto:

- **El modo estándar**: Si abre un elemento del SCM que no ha sido extraído, se abre una ventana que indica que el elemento debe extraerse para poder modificarlo.

- **El modo automatic**: Si intenta modificar un elemento del SCM que no ha sido extraído, el SCM propone extraerlo automáticamente. Atención: Este modo no se recomienda cuando se utiliza el SCM con una conexión lenta a Internet.




La opción "Comprobación de elementos durante la primera modificación" sirve para pasar del modo estándar (opción no marcada) al modo automatic (opción marcada).

Para obtener más información, consulte [Comprobación de un elemento](../Editeurs/2038006.md).

## "Editor" tab

### Temas
<a name="temas_ELTPARAGRAPHE000420"></a>

Esta opción se utiliza para elegir el tema del editor. El tema claro se utiliza de forma predeterminada. 

Otros temas están disponibles: 

- Tema claro, cinta de opciones gris. En este modo, la barra de menús (la cinta) ya no está coloreada.: está en gris. 

- Tema gris. En este modo, la barra de menús (ribbon) es gris y las diferentes ventanas de la interfaz se muestran en un gris claro de Background. 

- Tema oscuro. En este modo, el entorno y las diferentes ventanas de la interfaz se muestran en un Background negro o gris oscuro . 




Observación: El tema seleccionado se tendrá en cuenta cuando se reinicie el producto utilizado.

**Consejo**: En algunos casos, puede ser necesario cambiar el tema regularmente en función de la actividad actual: por ejemplo, cuando se realiza una videollamada o se utiliza un videoproyector, se preferirá el tema claro, mientras que en la fase de desarrollo será más apropiado el tema oscuro. Para iniciar WINDEV en el tema deseado sin pasar por las opciones y reiniciar el producto:

- crear un acceso directo al archivo "WINDEV64.exe

- en el comando Line, especifique el parámetro "/TEMA" con el tema deseado (OSCURO, CLARO o GRIS).




Por ejemplo: 

```txt
C:\|]\WINDEV 28\Programs\WINDEV64.exe /THEME=DARK
```

	- **Cuando el zoom es inferior al 100%**: el modo "Ojo de Pez" se activará en función del valor del zoom.

	- **Cuando se presionan Ctrl y Shift**: el modo "Ojo de Pez" se activará al pulsar una determinada combinación de teclas.

## "Pestaña "Traducción
<a name="pestana_traduccion_ELTTEXTE001002"></a>


### Configuración de las opciones de traducción
<a name="configuracion_las_opciones_traduccion_ELTPARAGRAPHE000570"></a>

Si su proyecto es multilingüe, las características de un elemento (título de la ventana, leyenda de una control, ...) deben introducirse en todos los idiomas de su aplicación.

Es posible:

- **Cambiar automáticamente los parámetros regionales según el idioma de entrada**: Esta opción permite activar automáticamente el idioma utilizado por el teclado cuando el cursor se sitúa en un área de entrada de un idioma que utiliza un carácter específico set.

- **Seleccione un motor de traducción específico**: el software de traducción especificado se utilizará para permitir una traducción de automatic desde la interfaz del producto de los diferentes elementos para traducir. El software de traducción utilizado puede ser: 

	- [WDDIXIO](../WDDixio/3514003.md), 

	- un software específico. 
			En este caso, usted tiene la capacidad de configurar el modo para transferir la información a traducir desde WINDEV, WEBDEV o WINDEV Mobile al software utilizado, así como el modo para recuperar la información traducida desde el software de traducción.

	- un sitio de INTERNET.
			 En este caso, al pulsar el botón "Traducir", el texto a traducir se copia en el portapapeles y el sitio Web se abre automáticamente.. No hay devolución o traducción directa en este modo. 

	- Google Translate. En este caso, debe especificar la clave API esperada por Google Traduction. Para obtener más información, consulte [Realizar una traducción con Google Traduction](../Editeurs/2036003.md). 
			note: debe cumplir con las reglas de uso del servicio a través de Google. 




- **Seleccione los idiomas admitidos** (idiomas de origen y de destino)




**Observaciones**: 

- Para configurar correctamente el modo de transferencia, consulte la documentación suministrada con el software de traducción.

- [WDDIXIO](../WDDixio/3514003.md) es un diccionario multilingüe suministrado con WDMSG. Póngase en contacto con el Departamento de ventas de PC SOFT para más información sobre las condiciones de uso de este producto.



<a name="NOTE5_2"></a>


### Usando la traducción de automatic
<a name="usando_traduccion_automatic_ELTPARAGRAPHE000612"></a>

Para usar la traducción de automatic que fue configurada de antemano:

1. Seleccione el texto a traducir (por ejemplo, el título de la ventana del control description).

2. Haga clic en el botón "Traducir" (disponible en la ventana de control description, por ejemplo).

3. Recuperar la traducción según las opciones elegidas al configurar la traducción de automatic (pegar en el área traducida por ejemplo).




Para obtener más información, consulte [Aplicaciones multilingües](../WDLang1/3054008.md).

## pestaña "Ayuda"
<a name="pestana_ayuda_ELTTEXTE001086"></a>
Esta pestaña se utiliza para definir el modo de uso de la ayuda en línea. 

La ayuda en línea está disponible: 

- en la región: los archivos de ayuda se instalan en el ordenador de desarrollo. La ayuda se muestra en un navegador de ayuda especial. 

- en INTERNET: la ayuda en línea de INTERNET está disponible desde cualquier ordenador equipado con un acceso INTERNET, sin que el producto esté necesariamente instalado. Esta ayuda se actualiza frecuentemente.




Para usar: 

- la ayuda local, seleccione la opción "Utilizar la ayuda local". 

- la ayuda de Internet, seleccione la opción "Utilizar la ayuda en línea".




Esta pestaña también se utiliza para filtrar la ayuda mostrada. Usted puede: 

- Mostrar ayuda para todos los productos (WINDEV, WEBDEV, WINDEV Mobile y Reports  Queries): se mostrará la ayuda común a WINDEV, WEBDEV, WINDEV Mobile y Reports and Queries. Se mostrarán todos los párrafos específicos de un producto. Todas las entradas del índice (incluso las específicas de un producto determinado) serán accesibles.. 

- Mostrar solo la ayuda del producto actual: sólo las páginas de ayuda relevantes para el producto actual serán accesibles. No se mostrarán notas sobre otros productos. 




Para obtener más información, consulte [Cómo acceder a la ayuda](../Editeurs/2033001.md). 

Atención: Si utiliza "Reports and Queries", sólo estará disponible la ayuda en línea.

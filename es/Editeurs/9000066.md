


## Snippets de código
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000204"></a>
El editor de código le permite introducir "fragmentos de código". Estos "fragmentos de código" son secciones del código que se utilizan con frecuencia. Puedes crear todos tus fragmentos de código: siempre estarán disponibles para ayudarle a ahorrar tiempo.

Estos fragmentos de código pueden contener parámetros: el nombre del archivo de datos, el nombre del elemento de búsqueda o el nombre de la ventana se pueden pasar como parámetro a su fragmento de código. Cuando se utiliza, el editor de código le pedirá que introduzca los parámetros necesarios: no hay riesgo de error!

Los fragmentos de código se pueden mover fácilmente de un proyecto a otro, de un ordenador a otro, etc.

Cada fragmento de código es associated con una palabra clave que simplifica la búsqueda.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000228"></a>


### Administrador de snippets de código
<a name="administrador_snippets_codigo_ELTPARAGRAPHE000029"></a>

El editor de código incluye un gestor de fragmentos de código. Este gerente está acostumbrado:

- Añade un fragmento de código.

- Modificar un fragmento de código.

- Usa un fragmento de código.

- Borrar un fragmento de código.




Para visualizar este gestor:  

- En el editor de código, seleccione "Fragmentos de código... Mostrar todos los fragmentos de código" en el menú contextual.

- En la pestaña "Inicio", en el grupo "Entorno", amplíe "Paneles", seleccione "Paneles" y, a continuación, "Bloques de código". 

<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=BriqueCode_Gestionnaire16.gif)
 
**Observación**: Por Default, el administrador de fragmentos de código está vacío. Si se definieron fragmentos de código personalizado en las versiones anteriores, estos fragmentos de código se importan automáticamente. Para visualizar los elementos estándar (los atajos propuestos por Default en las funciones WLanguage), seleccione "Visualizar los recortes estándar" del botón de flecha. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=BriqueCode_Menu.gif)

### Creación de un fragmento de código
<a name="creacion_fragmento_codigo_ELTPARAGRAPHE000063"></a>

Se puede crear un fragmento de código:

- desde el gestor de fragmentos de código. Para ello, haga clic en el botón "+" e introduzca la información solicitada.

- desde el editor de código. Para ello, seleccione el código correspondiente al fragmento de código y realice un arrastrar y soltar hacia el gestor de fragmentos de código.. Se muestra la ventana del fragmento de código description, con la información solicitada. Si la primera línea de código corresponde a una comment, esta comment se convierte automáticamente en el título del fragmento de código.




La ventana de description de un fragmento de código contiene los siguientes elementos: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=BriqueCode_Description.gif)


1. Título del fragmento de código. Este título se muestra en el gestor de fragmentos de código.

2. Abreviatura associated con el fragmento de código. Esta abreviatura puede ser una palabra o una set de palabras. Esta abreviatura simplifica el uso del fragmento de código: todo lo que tiene que hacer es introducir la abreviatura en el editor de código y pulsar F5: el fragmento de código correspondiente se inserta automáticamente.

3. Código correspondiente al fragmento de código.

4. Botón utilizado para insertar los controles del automatic. Se puede insertar la siguiente información: 

	- **la fecha y la hora** cuando el fragmento fue insertado en el código (al formato definido en los parámetros de idioma de Windows)

	- **automatic inserción del código seleccionado**: en este caso, cuando el fragmento de código se "suelta" del gestor de códigos al editor de códigos, el código actualmente seleccionado en el editor de códigos se inserta en la ubicación especificada. Esta opción es útil para un fragmento de código que se utiliza para realizar un Process iterativo (bucle, etc.).

	- **de acceso** del usuario que inserta el fragmento de código

	- **parameter**: esta opción se utiliza para reemplazar el código seleccionado en el fragmento de código por §&lt;Nombre del parámetro&gt;§. &lt;Nombre del parámetro&gt; puede ser sustituido por el nombre solicitado. Si el fragmento de código contiene esta opción, se mostrará automáticamente una ventana para introducir los parámetros cuando se utilice el fragmento de código.

	- **parámetro con valor Default**: esta opción se utiliza para reemplazar el código seleccionado en el fragmento de código por §&lt;Nombre del parámetro&gt;=&lt;Valor&gt;§. &lt;Nombre del parámetro&gt; se puede sustituir por el nombre solicitado y &lt;Valor&gt; por el valor Default del parámetro. Si el fragmento de código contiene esta opción, se mostrará automáticamente una ventana para introducir los parámetros cuando se utilice el fragmento de código.

	- **cursor position**: esta opción permite definir la posición del cursor una vez que se ha utilizado el fragmento de código. El cursor se puede posicionar en la sección de código para escribir, por ejemplo.







**Localización de fragmentos de código**: 
Por Default, los fragmentos de código añadidos se almacenan en "Personal\\Code snippets", en el directorio de instalación del producto utilizado. Este directorio puede ser modificado en las opciones de WINDEV, WEBDEV o WINDEV Mobile.: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile".

2. Mostrar la pestaña "Directorios" y modificar el directorio de fragmentos de código. 




**Observación**: El gestor de fragmentos de código se utiliza para gestionar todas las abreviaturas del editor de código.. Cada abreviatura permite acceder a una función, una Procedure, una Property o una set de líneas de código. El gestor de fragmentos de código incluye el glosario que se encuentra en las versiones anteriores. Puede crear un fragmento de código correspondiente a un nombre de la función WLanguage.

### Usando fragmentos de código
<a name="usando_fragmentos_codigo_ELTPARAGRAPHE000109"></a>

Varios métodos le permiten utilizar los fragmentos de código:

- Arrastrar y soltar desde el gestor de fragmentos de código.

- Pulsando F5 sobre la palabra clave associated con el fragmento de código.




**Observaciones**: 

- Al escribir el acceso directo correspondiente a un fragmento de código, la entrada asistida propone el fragmento de código correspondiente. Todo lo que tienes que hacer es seleccionarla y validarla para incluirla en el código actual.. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=BriqueCode_Code.gif)


- La opción "Fragmento de código" (menú contextual del editor de código) devuelve la lista de los diez últimos fragmentos de código, lo que permite utilizarlos directamente.



<a name="NOTE2_4"></a>


### Compartir fragmentos de código
<a name="compartir_fragmentos_codigo_ELTPARAGRAPHE000127"></a>

¿Desea compartir sus fragmentos de código entre desarrolladores o entre WINDEV, WEBDEV y WINDEV Mobile?

Todo lo que tienes que hacer es configurar el directorio de fragmentos de código en las opciones del producto utilizado: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile". 

2. Mostrar la pestaña "Directorios" y modificar el directorio de fragmentos de código. 



<a name="NOTE2_5"></a>


### Recuperación de los datos de un glosario personalizado
<a name="recuperacion_los_datos_glosario_personalizado_ELTPARAGRAPHE000151"></a>

Si su glosario fue personalizado en una versión anterior, puede recuperar estos datos:  

1. Seleccione "Importar un glosario" del botón de flecha. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=BriqueCode_Menu.gif)


2. Seleccione el archivo "WX_Gloss.fic" que se encuentra en el subdirectorio "Custom" del producto utilizado.. El archivo se importa automáticamente.

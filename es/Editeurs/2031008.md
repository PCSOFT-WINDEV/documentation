## Presentación
<a name="presentacion_ELTTEXTE000423"></a>
La asistente para la creación de páginas propone varios tipos de páginas associated con archivos de datos HyperFileSQL. Tienes la habilidad de crear:

- **páginas dinámicas**: Estas páginas se crean a partir de los archivos de datos que se encuentran en el ordenador de desarrollo. En implementación, los datos están instalados (o accesibles) desde el servidor Web. Las páginas muestran los datos encontrados en los archivos de datos.

- **páginas semidinámicas**: Estas páginas se crean a partir de los archivos de datos que se encuentran en el ordenador de desarrollo. Antes de instalar la aplicación, estas páginas se convierten en páginas estáticas: los datos encontrados en los archivos de datos se guardan en las páginas. No hay ningún archivo de datos instalado en el servidor.




Para obtener más información, consulte [Diferentes tipos de páginas](../WDChamp/2030020.md).

Los tipos de páginas disponibles:



|   |   |
| --- | --- |
| **Páginas dinámicas** | **Páginas semidinámicas** |
| Página de formulario:<br><br>- simple. <br><br>- con navegar. <br><br>- con relacionado tabla. <br><br>- con la relación tabla.<br><br><br> | Página de formulario |
| Página de la tabla:<br><br>- simple. <br><br>- con un formulario detallado.<br><br>- con relacionado tabla.<br><br><br> | Página de la tabla |
| Página de Vision Plus. |   |
| Página del looper. | Página del looper. |


Los siguientes párrafos se presentan para cada tipo de página:

- el modo de funcionamiento de la página,

- los controles que se encuentran en la página,

- una vista previa de la página,

- consejos.

## Página de formulario
<a name="pagina_formulario_ELTTEXTE000447"></a>


### Caso general
<a name="caso_general_ELTPARAGRAPHE000073"></a>

Una página **form** muestra los valores de un Record en una forma, en base a un valor por control. Se puede acceder a los registros uno por uno.

El formulario puede:

- permitir (o no) el paso a los registros anteriores y siguientes.

- permitir (o no) la creación, modificación y eliminación de registros.



<a name="NOTE2_2"></a>


### Forma dinámica simple
<a name="forma_dinamica_simple_ELTPARAGRAPHE000085"></a>

Este tipo de página se utiliza para ver y modificar un Record.

Este tipo de página no se lo permite:

- navegar por los diferentes registros.

- crear o borrar un Record.




Se compone de una página "Simple dynamic form":

- botones utilizados para validar o Cancel la entrada.

- botones utilizados para imprimir la actual Record

- controla relacionado a las posiciones de fichero.




En la mayoría de los casos, se puede acceder a la página de "Forma dinámica simple" desde una página de "tabla dinámica simple" para ver o modificar el Record actual.
<a name="NOTE2_3"></a>


### Forma dinámica con navegación
<a name="forma_dinamica_con_navegacion_ELTPARAGRAPHE000105"></a>

Este tipo de página está acostumbrada a:

- modificar la Record que se muestra.

- crear un registro.

- eliminar un registro.

- imprimir la actual Record.

- pasar a otra Record (siguiente o anterior).




Una página "Dynamic form with browse" (Forma dinámica con navegación) incluye:

- controla relacionado a las posiciones de fichero.

- 4 botones de navegación.

- que se usaban antes:

	- crear un nuevo Record.

	- borrar la actual Record.

	- imprimir la actual Record.

	- cerrar la página.







En comparación con una "Forma dinámica simple", una "Forma dinámica con navegación" propone:

- una manera fácil de pasar de una forma a otra.

- la capacidad de crear varios registros en un fila.




**Consejo**: Para los archivos que contienen varias Record s, le aconsejamos seleccionar las Record s en una tabla y modificar o ver la Record seleccionada en una página de "Forma simple".. De hecho, la selección es más rápida en una tabla.
<a name="NOTE2_4"></a>


### Forma dinámica con relacionado tabla
<a name="forma_dinamica_con_relacionado_tabla_ELTPARAGRAPHE000136"></a>

Este tipo de ventana se utiliza para ver el contenido de los archivos relacionado mediante un simple enlace (1 - N) en la misma página.

Los valores de la Record que se encuentran en el archivo principal se muestran en una forma. Los Record s relacionado al Record actual (en el archivo principal) se muestran en un tabla.

Este tipo de página incluye:

- un formulario (en la sección superior) utilizado para mostrar los valores de la Record actual que se encuentra en el archivo base (archivo principal).

- un tabla (en la sección inferior) utilizado para mostrar el Record s relacionado al Record actual en el archivo principal.

- un buscapersonas usado para mostrar el tabla Record.

- que se usaban antes:

	- crear un nuevo Record en el archivo principal.

	- eliminar la actual Record del archivo principal.

	- cerrar la página.

	- añadir una fila en el tabla.

	- modificar un fila en el tabla.

	- borrar un fila del tabla.







Observaciones:

- Si el archivo que se muestra en tabla es un archivo de relación, la página contiene los botones necesarios para acceder al otro archivo de relación.

- Si se selecciona este tipo de página para un archivo sin archivo relacionado (enlace 1-N), se señala con WEBDEV.



<a name="NOTE2_5"></a>


### Forma dinámica con relación tabla
<a name="forma_dinamica_con_relacion_tabla_ELTPARAGRAPHE000164"></a>

Este tipo de página se utiliza para ver al mismo tiempo:

- el contenido de un fichero en una tabla y

- el contenido de un archivo en un formulario.




Los registros que aparecen en la tabla y los que se guardan en el formulario son relacionado por un enlace complejo (N - N).

La página "Formulario con relación tabla" incluye:

- un formulario (en la sección superior) utilizado para visualizar los registros encontrados en la carpeta base (carpeta principal).

- un tabla (en la sección inferior) utilizado para mostrar el Record s relacionado al Record actual en el archivo principal.

- un buscapersonas usado para mostrar el tabla Record.

- que se usaban antes:

	- crear un nuevo Record en el archivo principal.

	- eliminar la actual Record del archivo principal.

	- cerrar la página.

	- añadir una fila en el tabla.

	- modificar un fila en el tabla.

	- borrar un fila del tabla.







La página "Formulario con relación tabla" se recomienda cuando los detalles del archivo deben ser mostrados inmediatamente.
<a name="NOTE2_6"></a>


### Forma semidinámica
<a name="forma_semidinamica_ELTPARAGRAPHE000192"></a>

Este tipo de página se utiliza para ver un Record y para navegar por los Record s para ver.

Este tipo de página no le permite crear o borrar registros. De hecho, al generar las páginas estáticas del associated, se creará una página estática por formulario.

Este tipo de página incluye:

- que se utilizan para mostrar el contenido de los elementos del archivo.

- una barra de navegación utilizada para navegar por los registros.




En la mayoría de los casos, se puede acceder a la página de "Forma semidinámica" desde la página de "tabla semidinámica" para ver el Record actual.

## Página de la tabla
<a name="pagina_tabla_ELTTEXTE000501"></a>


### Caso general
<a name="caso_general_ELTPARAGRAPHE000210"></a>

Una página **table** muestra los registros como una tabla (en filas y columnas). Se pueden mostrar varios registros de archivo en la página.

Los artículos de la misma Record se muestran en fila.
<a name="NOTE3_2"></a>


### Simple dinámica tabla
<a name="simple_dinamica_tabla_ELTPARAGRAPHE000219"></a>

Este tipo de página está acostumbrada a:

- ver los registros en un tabla.

- crear una Record (en una página de formulario)

- eliminar un registro.

- modificar una Record (en una página de formulario)

- imprimir el archivo.




Una página "Simple dinámica tabla" incluye:

- un tabla utilizado para mostrar los registros encontrados en el archivo relacionado.

- controles de búsqueda rápida utilizados para acceder a uno de los registros de tabla.

- un localizador utilizado para navegar por los registros de tabla.

- que se usaban antes:

	- crear un nuevo Record.

	- modificar un registro.

	- eliminar un registro.

	- imprimir los registros.

	- cerrar la página.







Este tipo de página se recomienda en todos los casos de gestión de ficheros<u><u><u><u>excepto</u></u></u></u>:

- si los detalles de la Record deben ser mostrados inmediatamente: prefieren una página dinámica con tabla y un formulario detallado.

- si algunos elementos deben mostrarse en los controles multilínea: prefieren una página dinámica con tabla y un formulario detallado.

- si el archivo contiene imágenes que deben mostrarse inmediatamente: prefieren una página dinámica con tabla y un formulario detallado.

- si el archivo contiene muchos elementos para mostrar: prefieren una página dinámica con tabla y un formulario detallado.

- si se deben visualizar los artículos de relacionado: prefieren una página dinámica con formulario y relacionado tabla.



<a name="NOTE3_3"></a>


### tabla dinámica con forma detallada
<a name="tabla_dinamica_con_forma_detallada_ELTPARAGRAPHE000257"></a>

Este tipo de página se utiliza para ver en la misma página el contenido de un archivo en un tabla y en un formulario.. Esta página se utiliza para seleccionar una Record en la tabla y para ver los detalles de la Record seleccionada en un formulario.

La página "tabla dinámica con formulario detallado" incluye:

- un tabla (en la sección superior) que se utiliza para visualizar los registros de fichero.

- controles de búsqueda rápida utilizados para acceder a un Record en el tabla

- un formulario (en la sección inferior) utilizado para mostrar los valores de la fila seleccionada en el tabla

- un localizador utilizado para navegar por los registros de tabla.

- varios botones que solían hacerlo:

	- crear un nuevo Record.

	- modificar la actual Record.

	- borrar la actual Record.

	- cerrar la página.







La página "tabla dinámica con formulario detallado" se recomienda cuando:

- el archivo contiene elementos de Image o elementos de varias líneas para mostrar.

- los detalles de la Record deben ser inmediatamente Visible.

- el archivo contiene muchos elementos para mostrar.



<a name="NOTE3_4"></a>


### tabla dinámico con relacionado tabla
<a name="tabla_dinamico_con_relacionado_tabla_ELTPARAGRAPHE000283"></a>

Este tipo de ventana se utiliza para mostrar en la misma página el contenido de los archivos relacionado mediante un enlace simple o complejo.. Las tablas son tablas en cascada: el segundo tabla se utiliza para mostrar los registros relacionado al fila seleccionado en el primer tabla.

La página "tabla dinámica con relacionado tabla" incluye:

- un tabla (en la sección superior) utilizado para visualizar los registros encontrados en el fichero base.

- una tabla (en la sección inferior) utilizada para mostrar las Record s encontradas en el archivo relacionado, correspondientes a la Record actual seleccionada en la primera tabla.

- dos localizadores utilizados para navegar por los registros de las tablas.

- que se usaban antes:

	- crear un nuevo Record en el archivo principal.

	- modificar la actual Record en el archivo principal.

	- eliminar la actual Record del archivo principal.

	- cerrar la página.

	- añadir una fila en el archivo relacionado.

	- modificar una fila en el archivo relacionado.

	- borrar un fila del archivo relacionado.






<a name="NOTE3_5"></a>


### tabla semidinámica
<a name="tabla_semidinamica_ELTPARAGRAPHE000305"></a>

Este tipo de página se utiliza para ver los registros de la tabla.

La página "tabla Semi-dinámica" incluye:

- un tabla utilizado para mostrar los registros encontrados en el archivo relacionado.

- un localizador utilizado para navegar por los registros de tabla.




La página "tabla semidinámica" se utiliza para ver los detalles de los Record s mediante un clic realizado en el Record. A continuación, se muestra una página "Forma semidinámica".

## Página del looper
<a name="pagina_del_looper_ELTTEXTE000549"></a>


### Caso general
<a name="caso_general_ELTPARAGRAPHE000321"></a>

Una página de ***Looper*** muestra los registros de forma repetida (en un control Looper). Se pueden mostrar varios registros en la página
<a name="NOTE4_2"></a>


### Looper dinámico
<a name="looper_dinamico_ELTPARAGRAPHE000328"></a>

Este tipo de página está acostumbrada a:

- ver los registros en un looper de navegación.

- crear una Record (en una página de formulario)

- eliminar un registro.

- modificar una Record (en una página de formulario)




La página "Looper dinámico" incluye:

- un looper usado para mostrar los registros encontrados en el archivo base.

- controles de búsqueda rápida utilizados para acceder a un looper Record.

- un localizador que control usó para buscar los registros encontrados en el looper.

- que se usaban antes:

	- crear un nuevo Record

	- modificar un registro

	- eliminar un registro

	- imprimir un Record

	- cerrar la página.






<a name="NOTE4_3"></a>


### Garfio semidinámico
<a name="garfio_semidinamico_ELTPARAGRAPHE000353"></a>

Este tipo de página se utiliza para ver los registros en un looper de navegación.

La página "Looper semidinámico" incluye:

- un looper usado para mostrar los registros encontrados en el archivo base.

- un botón que se utiliza para mostrar los detalles de la actual Record

- un localizador que control usó para buscar los registros encontrados en el looper.

## Página de Vision Plus
<a name="pagina_vision_plus_ELTTEXTE000585"></a>
Las páginas de Vision Plus se utilizan para seleccionar una Record en una tabla. Por ejemplo, al introducir un pedido, se pueden abrir algunas páginas de Vision Plus para seleccionar el cliente y los productos.

Las páginas de Visión Plus son llamadas por el botón cuya leyenda es [...] en las páginas relacionado a un archivo.

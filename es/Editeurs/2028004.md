


## WDInst: configuración de las características del fichero instalación 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000218"></a>
WDInst se utiliza para configurar la instalación de archivos:

- Crear un atajo para cada archivo instalado (con un comando Line si es necesario).

- Gestionar las colisiones al actualizar la aplicación (reemplazar o no los archivos existentes).

- Guardar los archivos reemplazados cuando se actualiza la aplicación (automatic copia de seguridad de los archivos existentes).




Las características del archivo instalación se configuran en el editor de instalación WDInst. Para abrir WDInst, en la pestaña "Herramientas", en el grupo "Utilidades", haga clic en "WDInst".

A diferencia del programa de creación instalación asistente , WDInst incluye opciones avanzadas para configurar los archivos a instalar.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Creación de un acceso directo
<a name="creacion_acceso_directo_ELTTEXTE000242"></a>
Para cada archivo a instalar, WDInst le permite crear:

- Un atajo en el escritorio de la computadora de instalación.

- Un atajo en un grupo de programas.

- Un atajo en la barra de lanzamiento.




Por Default, por cada ejecutable instalado, se creará un atajo:

- en el escritorio de la computadora de instalación.

- en un grupo de programas.

- en la barra de tareas.




**Observación**: El pie de foto de los atajos se creará en el lenguaje de ejecución del programa instalación.
<a name="NOTE2_2"></a>


### Creación de un acceso directo
<a name="creacion_acceso_directo_ELTPARAGRAPHE000055"></a>

Para crear un acceso directo: 

1. Seleccione el archivo deseado en la ventana "Lista de archivos instalación".

2. Vaya a la pestaña "Acceso directo" de la ventana "Archivo description".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Parametrer_caract_installation_fichier%20-%20HC%20N%B0002.gif)


3. Marque la opción deseada: 

	- "Crear un acceso directo en el escritorio" para crear el acceso directo de la aplicación en el escritorio del equipo del usuario. 

	- "Crear un acceso directo en un grupo de programas" para crear el acceso directo en el grupo especificado. 

	- "Crear un acceso directo en la barra de tareas" para crear el acceso directo en la barra de tareas de Windows. 
			Todas estas opciones se pueden acumular. 




4. Introduzca el título del atajo en cada uno de los idiomas soportados por el programa instalación.

5. Introduzca (si es necesario) el grupo de programas donde se debe crear el acceso directo..
	**Observación**: Este grupo de programas se creará si no existe en el ordenador del usuario.

6. Especifique (si es necesario) los parámetros del acceso directo. La ejecución de este atajo iniciará un comando Line. 
	Por ejemplo, el comando Line: "C:\\MyDirectory\\MyApplication.EXE /Login = Bill" contiene:

	- La ruta completa del archivo associated con el atajo.
			En nuestro ejemplo: "C:\\MyDirectory\\MiAplicación.EXE".

	- Los parámetros de acceso directo.
			En nuestro ejemplo: "/Login = Bill".

## Gestión de las colisiones
<a name="gestion_las_colisiones_ELTTEXTE000272"></a>
Por Default, al actualizar una aplicación, si ya existen ficheros con el mismo en el directorio de instalación, estos ficheros se sustituyen (excepto si los ficheros ya encontrados en el equipo son más recientes que los encontrados en el instalación). 
No se muestra ninguna ventana de advertencia para confirmar estos reemplazos.

WDInst se utiliza para configurar el programa instalación para:

- Instale los archivos aunque los archivos con el mismo nombre y más recientes se encuentren en el directorio instalación.

- Evite instalar los archivos si se encuentran archivos con el mismo nombre en el directorio de instalación.

- Pida al usuario que confirme si los archivos deben ser reemplazados o no.

- Renombrar los archivos existentes.



<a name="NOTE3_2"></a>


### Gestión de las colisiones
<a name="gestion_las_colisiones_ELTPARAGRAPHE000115"></a>

Para gestionar las colisiones:

1. Seleccione el archivo o archivos deseados en la ventana "Lista de archivos instalación"..
	Para seleccionar varios archivos simultáneamente, mantenga pulsada la tecla Ctrl o Mayúsculas al seleccionar archivos.

2. Visualizar la pestaña "Versión" de la ventana "Fichero description".: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Parametrer_caract_installation_fichier%20-%20HC%20N%B0003.gif)


	- Para evitar instalar los archivos seleccionados si se encuentran archivos con el mismo nombre en el directorio instalación, marque "No instalar si el archivo ya existe (no se realizará ninguna otra comprobación)".

- Para reemplazar los archivos existentes (incluso si son más recientes) seleccione la opción "Reemplazar el archivo incluso si es más reciente". 

- Para solicitar la confirmación del usuario para reemplazar el/los archivo(s) seleccionado(s), seleccione la opción "Solicitar confirmación antes de reemplazar el archivo".

- Para cambiar el nombre de los archivos existentes en el directorio de instalación antes de instalar archivos nuevos, marque "Cambiar el nombre del archivo sustituido por el nombre especificado a continuación".. Introduzca el nombre y la extension del fichero sustituido.
			Por ejemplo: MiImagen_Bak.GIF".
			**Observación**: Esta operación se debe realizar para cada archivo reemplazado que se deba renombrar.

## Guardar los archivos reemplazados
<a name="guardar_los_archivos_reemplazados_ELTTEXTE000302"></a>
Cuando se actualiza una aplicación, se puede incluir un avión adicional en el programa instalación. Este plano adicional permite al usuario guardar los archivos reemplazados en un directorio de copia de seguridad.. Este directorio de copia de seguridad se denominará "Backup__" por Default.

Esta copia de seguridad puede ser usada si la instalación falla por ejemplo. Para utilizar esta copia de seguridad, el usuario tendrá que copy los archivos que se encuentran en el directorio de la copia de seguridad en el directorio de instalación de la aplicación.

La copia de seguridad de los archivos reemplazados es propuesta por Default a los usuarios.<a name="NOTE4_2"></a>


### Proponer la copia de seguridad de los archivos sustituidos
<a name="proponer_copia_seguridad_los_archivos_sustituidos_ELTPARAGRAPHE000160"></a>

Proponer la copia de seguridad de los archivos sustituidos: 

1. En el editor de instalación WDInst: seleccione "Configuración ... Asistente".

2. Marque "Proponer guardar los elementos sustituidos con el programa instalación". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Info_affiches_install%20-%20HC%20N%B0001.gif)

	Al instalar la aplicación, el usuario podrá hacerlo:

	- Seleccione esta opción para guardar los archivos reemplazados.

	- Modificar el directorio de respaldo propuesto por Default.

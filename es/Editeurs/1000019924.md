
## Prueba de una aplicación iOS en un dispositivo móvil
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000148"></a>
Cuando se desarrolla una aplicación para iOS (iPhone o iPad), WINDEV Mobile permite probar la aplicación que se está desarrollando sin tener que recompilarla en Mac, sino directamente en el iPhone o iPad.

Simplemente descarga WMDev, disponible de forma gratuita en el Apple Store.

**Observación**: Tenga en cuenta que esta funcionalidad está sujeta a la discreción de Apple y Apple puede decidir eliminar la aplicación para descargar de su tienda.
Por favor acepte nuestras disculpas por adelantado si Apple retira la aplicación . En ese caso, esta página se actualizará con el Procedure a seguir para mantener esta funcionalidad..

<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000172"></a>


### Condición previa
<a name="condicion_previa_ELTPARAGRAPHE000022"></a>

Para probar una aplicación iOS directamente en un iPhone o iPad, deben cumplirse las siguientes condiciones: 

- el iPhone/iPad y el equipo de desarrollo deben estar en la misma red (Wifi, por ejemplo).

- WMDev debe descargarse en el iPhone/iPad (https://itunes.apple.com/us/app/WMDev/id1437792304). 

- los siguientes puertos de red deben estar abiertos: 

	- 64000 (64000 a 64005 si se están ejecutando varios WINDEV Mobile de forma simultánea). 

	- 27280


 Si se utiliza un firewall, debe estar configurado para permitir el uso de estos puertos. 





### Esta acción debe realizarse una sola vez
<a name="esta_accion_debe_realizarse_una_sola_vez_ELTPARAGRAPHE000035"></a>

Para conectar el ordenador de desarrollo y el dispositivo iOS, basta con: 

1. Abrir un proyecto iOS en WINDEV Mobile. 

2. Si es necesario, seleccione la IP Address del ordenador de desarrollo que corresponde a la red a la que está conectado el iPhone o el iPad: 

	- En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones del editor de código".

	- Vaya a la pestaña "Depurador" y seleccione la IP Address en la zona "Dirección IP utilizada para la depuración".




3. Seleccione "GO .. Depurar en nuevo iPhone/iPad".. La ventana de conexión a un dispositivo iOS se abre. Esta ventana contiene un código QR específico. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Test_Appli_iOs%20-%20HC%20N%B0001.gif)


4. Abra WMDev en el dispositivo móvil (iPhone o iPad). 

	- Haga clic en "+" y luego en el botón que muestra un código de barras. 

	- Escanee el código de barras que se aparece en WINDEV Mobile en la ventana de conexión de un dispositivo iOS. 




5. La aplicación móvil se abre en el dispositivo iOS. 





### Ejecución de la prueba
<a name="ejecucion_prueba_ELTPARAGRAPHE000069"></a>

Una vez conectados el ordenador de desarrollo y el dispositivo iOS, es posible lanzar la prueba de la aplicación directamente en el dispositivo móvil. Simplemente seleccione "GO .. Depurar en ".

Observaciones: 

- La aplicación WMDev debe ser lanzada en el móvil para que la opción esté disponible.. 

- No es posible ejecutar varios depuradores simultáneamente.

- WMDEV no puede ejecutar código nativo.  

- Las siguientes características no pueden ser probadas: 

	- Notificaciones Push. 

	- Compras In-App. 

	- Tareas de fondo. 

	- Etiquetas de balizamiento.

	- Geolocalización. 




- Las fuentes personalizadas añadidas al proyecto se utilizan durante las pruebas. 




### Posibles problemas
<a name="posibles_problemas_ELTPARAGRAPHE000092"></a>

Aparece el siguiente mensaje: 
"No se puede conectar a este dispositivo: la versión de WMDev es demasiado antigua para usarse con el depurador"


Solución: Busque actualizaciones de WMDev en el App Store (https://itunes.apple.com/us/app/wmdev/id1437792304) y actualice la aplicación en el dispositivo iOS.

Atención: Después del mensaje, la actualización de WMDEV puede tardar varios días en estar disponible (ya que la aplicación es validada por Apple)..

## Configurar la aplicación
<a name="configurar_aplicacion_ELTTEXTE000148"></a>
El Grupo de Usuarios es configurado por el supervisor de la aplicación. Esta configuración consiste en crear usuarios y grupos de usuarios, y en concederles derechos específicos para cada control que se encuentre en cada ventana o página de la aplicación. Estos permisos permiten deshabilitar controles, grupos de controles u opciones de menú, y hacerlos visibles o invisibles.

**Para configurar el programa de grupo de usuarios,** debe:

1. Inicie la aplicación (ejecutable o test desde WINDEV o WEBDEV) y conéctese como supervisor:

	- denominar: SUPERVISOR

	- Contraseña: SUPERVISOR




2. Seleccione la opción "Configurar groupware".

## Gestión de usuarios
<a name="gestion_usuarios_ELTTEXTE000172"></a>
Para crear un nuevo usuario, debe especificar:

- el apellido del usuario (obligatorio)

- el nombre del usuario

- el nombre de usuario. El nombre de usuario es el identificador utilizado para iniciar sesión en la aplicación.

- la contraseña del usuario. Esta contraseña es opcional y puede ser introducida por el propio usuario durante su primer acceso (marque la opción correspondiente).




**El usuario puede definirse como un supervisor de la aplicación**.

También puede modificar o eliminar usuarios. Cuando se elimina un usuario, se puede eliminar todo el usuario o solo los permisos otorgados.

**TLos usuarios pueden ser agrupados.** Cuando se crea un grupo, se tiene la posibilidad de copy los derechos que fueron definidos previamente para otro grupo o usuario.

Un grupo puede ser creado o eliminado. Al eliminar un grupo, tiene la posibilidad de eliminar el grupo y sus usuarios o de eliminar sólo el grupo.. En este caso, los usuarios ya no son associated con un grupo.
<a name="NOTE2_2"></a>


### Observaciones
<a name="observaciones_ELTPARAGRAPHE000050"></a>

- La contraseña del supervisor debe cambiarse durante el primer uso.

- El grupo "Default" es el grupo propuesto por Default en la Agrupación de Usuarios. Por Default, este grupo tiene todos los derechos sobre la solicitud.

- El grupo &lt;None&gt; y el grupo Default no pueden ser borrados.

- Si utiliza un directorio LDAP, puede importar los usuarios desde el directorio para administrar los permisos. Para obtener más información, consulte [Groupware de usuario y LDAP](../Editeurs/2018013.md).

## Gestión de los derechos
<a name="gestion_los_derechos_ELTTEXTE000202"></a>
El supervisor puede gestionar los derechos concedidos a cada usuario (o grupo de usuarios) sobre los elementos que se encuentran en las ventanas, plantillas de ventanas, páginas y plantillas de páginas del proyecto.

Para cada asociación entre un usuario y una ventana/página, se puede definir un estado específico para todos los elementos que se encuentran en la ventana o en la página.. Esto se puede hacer para las plantillas de ventana o plantillas de página.

Los elementos gestionados en las ventanas, ventanas internas, páginas, páginas internas, plantillas de ventanas y plantillas de páginas son los siguientes:

- los controles

- los grupos de control

- las opciones de menú






Los siguientes estados están disponibles para cada elemento:

- **Default**: el comportamiento del elemento corresponde al comportamiento predeterminado, definido en la aplicación.

- **Inactive**: el elemento se muestra pero no se puede realizar ninguna entrada.

- **Grayed**: el elemento se muestra en gris. No se puede realizar ninguna entrada.

- **Invisible**: el elemento no se muestra.



<a name="NOTE3_2"></a>


### Caso especial: el supervisor no tiene acceso a las plantillas de ventana
<a name="caso_especial_supervisor_tiene_acceso_las_plantillas_ventana_ELTPARAGRAPHE000089"></a>

Si las ventanas de groupware usuario han sido incluidas en su proyecto, puede permitir que el supervisor configure las ventanas internas y las plantillas de ventanas.

Para ello, modifique el código de la ventana GPWConfigurationDetails o el código de la página GPWConfigurationDetails_WB.

**Modificación a realizar en WINDEV**: Modifique el código de inicialización del cuadro combinado COMBO_Window de la siguiente manera:


```wl
// Enumerate the windows
WindowName is string = EnumElement(enumWindow + enumInternalWindow + enumWindowTemplate)
WHILE WindowName <> ""
	ListAdd(COMBO_Window, WindowName) 
	// Next window
	WindowName = EnumElement() 
END
Window = 1
```


**Modificación para realizar en WEBDEV**: Modifique el código de inicialización del cuadro combinado "Combo_Page" como sigue


```wl
// Enumerate the windows
WindowName is string = EnumElement(enumDynamicPage + enumInternalPage + enumPageTemplate)
WHILE WindowName <> ""
	ListAdd(COMBO_Window, WindowName) 
	// Next window
	WindowName = EnumElement() 
END
Window = 1
```




## Groupware Usuario Mobile (Android y iOS)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000267"></a>
WINDEV Mobile incluye un Groupware Usuario para Android y iOS. 

Es muy sencillo de usar: 

- Cuando se inicia la aplicación, una ventana específica permite a los usuarios iniciar sesión (o crear una nueva cuenta). 

- Cuando se valida la ventana de inicio de sesión, la aplicación comprueba los datos de acceso consultando a un Webservice. 

- Si los datos de inicio de sesión son correctos, los usuarios pueden utilizar la aplicación con los permisos que les han sido concedidos. Una base de datos local de permisos se copia en el dispositivo móvil, lo que permitirá conectarse, incluso en modo sin conexión. 




**Observación**: Esta página de ayuda presenta la implementación del "Groupware Usuario". Para implantar un sistema de tienda privada, consulte [Tienda privada para aplicaciones móviles](../Editeurs/1410086652.md).

<a name="NOTE2_2"></a>


### Etapa 1: Configuración del servidor
<a name="etapa_1_configuracion_del_servidor_ELTPARAGRAPHE000049"></a>

Pueden presentarse dos casos: 

- **Si utiliza una plataforma PCSCloud**: los elementos necesarios se instalan de forma predeterminada en su plataforma. 
	Recordatorio: PCSCloud es una solución paga lista para usar. Para obtener más información, consulte [PCSCloud](https://pcscloud.net/UK/cloud_development.awp).

- **Si no utiliza una plataforma PCSCloud**: instale los siguientes elementos: 

	- un servidor HFSQL,

	- el servidor reservado, incluidos los módulos para el Groupware Usuario de dispositivos móviles (WDBaaS).


 Para obtener más información, consulte [Configuración de un servidor BaaS (Groupware Usuario y Tienda privada)](../Editeurs/1000021311.md).



<a name="NOTE2_3"></a>


### Etapa 2: Active el Groupware Usuario en una configuración del proyecto móvil
<a name="etapa_2_active_groupware_usuario_una_configuracion_del_proyecto_movil_ELTPARAGRAPHE000071"></a>

Para activar el Groupware Usuario en una configuración del proyecto WINDEV Mobile: 

1. Abra el proyecto en WINDEV Mobile. 

2. Si es necesario, seleccione la configuración del proyecto en la que debe activarse el Groupware Usuario. 

3. En la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "GPU / Tienda privada". 

4. En la ventana de descripción de la configuración del proyecto actual, en la sección "Groupware Usuario: Desactivado", haga clic en "Cambiar". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GPU_Mobile%20-%20HC%20N%B0001.gif&type=thumb)


5. En la ventana que aparece, en la pestaña "General": 

	- Active los servicios MBaaS: haga clic en "Activo". 

	- Seleccione la funcionalidad deseada: "GPU: Gestión de permisos de acceso de los usuarios". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GPU_Mobile%20-%20HC%20N%B0002.gif&type=thumb)


	- En la sección "Servidor de aplicaciones del GPU y de la tienda privada (webservice y sitio de administración)", indique el servidor a utilizar. En el caso de un nuevo servidor, seleccione el tipo de servidor a utilizar: 

	- **Plataforma PCSCloud**. En este caso, especifique las características de la plataforma [PCSCloud](https://pcscloud.net/UK/cloud_development.awp). 
						Recordatorio: Esta opción permite utilizar un servidor web, un servidor dedicado y un motor HFSQL Client/Server ya instalados y configurados. 

	- **Servidor de red o de Internet** con el protocolo utilizado. Introduzca el nombre del servidor, usuario y contraseña. Esta información le fue proporcionada durante la instalación del servidor reservado para el Groupware Usuario. 

6. En la pestaña "GPU", especifique:  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GPU_Mobile%20-%20HC%20N%B0003.gif&type=thumb)


	- El modo de personalización del Groupware Usuario: 

	- **"Ventana de inicio de sesión predeterminada (recomendado)"**:  WINDEV Mobile se encarga de todo. Simplemente inicie la aplicación para utilizar el Groupware Usuario.

	- **"Personalizar ventana de inicio de sesión"**: Todos los elementos del groupware usuario se integran en el proyecto a través de un componente interno. Puede personalizar los diferentes elementos del Groupware Usuario. Sin embargo, las evoluciones del producto no se integrarán.

7. En la pestaña "Opciones", especifique: 

	- El mensaje de los correos electrónicos enviados a los usuarios para validar su registro y gestionar las contraseñas olvidadas. 
			Observación: Los parámetros del servidor utilizado para el envío de correos electrónicos deben especificarse en el sitio de gestión del Groupware Usuario. 

	- El idioma de los archivos: De forma predeterminada, los archivos de datos específicos del Groupware Usuario están en inglés, pero también puede utilizarlos en francés. 




8. Valide la ventana de configuración del Groupware Usuario. 




**Atención: Al validar la ventana del Groupware Usuario, se realizan las siguientes acciones:** 

- Adición del componente interno que contiene las ventanas del Groupware Usuario (por ejemplo, ventana de inicio de sesión personalizada del Groupware Usuario). 

- Declaración de la aplicación en curso en el servidor de aplicaciones reservado para el Groupware Usuario. 

- Activación de sitios de gestión de usuarios y permisos. 

- Actualización del archivo XML que contiene la descripción de las ventanas de la aplicación en el servidor. 

- Definición de un usuario predeterminado, con el nombre de usuario y la contraseña "supervisor".

### Etapa 3: Configure los usuarios y permisos
<a name="etapa_3_configure_los_usuarios_permisos_ELTPARAGRAPHE000169"></a>

Para configurar usuarios y permisos: 

1. En la pestaña "General" de la ventana de gestión del Groupware Usuario, abra un sitio de administración a través de uno de los siguientes enlaces: 

	- la URL que corresponde a la aplicación implementada. Los usuarios y contraseñas definidos se utilizarán al implementar la aplicación. 

	- la URL que corresponde al modo de prueba. Los usuarios y contraseñas definidos se utilizarán al realizar una prueba en el teléfono ("Go").  




2. Inicie sesión en el sitio de administración. Introduzca "supervisor" como nombre de usuario y contraseña. 

3. Puede: 

	- definir los diferentes usuarios, permisos y modos de acceso. 

	- definir los parámetros del servidor SMTP utilizado para el envío de correos electrónicos (validación de la inscripción, etc.). 







Observaciones: 

- Solo el "Supervisor" puede acceder al sitio de gestión de usuarios y permisos. 

- Una banda específica permite saber si el sitio de gestión de usuarios y permisos se utiliza en modo de prueba. 

- La lista de ventanas y controles de la aplicación se actualizan en el servidor (para la gestión de permisos) al realizar las siguientes acciones: 

	- Validación de la ventana de configuración del groupware usuario. 

	- Inicio de la aplicación en modo de prueba. 

	- Generación de la aplicación. 


 Para configurar una nueva ventana en los permisos de los usuarios, se debe generar la aplicación para que la ventana aparezca en el sitio de gestión de permisos.

### Etapa 4: Pruebe la aplicación
<a name="etapa_4_pruebe_aplicacion_ELTPARAGRAPHE000197"></a>

Para probar la aplicación: 

- Pruebe el proyecto a través de los botones de acceso rápido. 

	- En Android, la prueba se realiza a través del simulador o en su móvil. 

	- En iOS, la prueba se realiza a través del simulador o en su móvil usando WMDEV. 




- Inicie sesión con el nombre de usuario y la contraseña definidos en el sitio de administración de la prueba. La aplicación utilizará la base de datos de los usuarios y permisos de prueba. 




La aplicación se implementa igual que las aplicaciones Android o iOS. La aplicación implementada utilizará la base de datos de usuarios y permisos de implementación. 

Observación: 

- Cuando se inicia la aplicación en modo de prueba (Go en el simulador, o emulador Android, o WMDev), la base de datos de los usuarios y de los permisos de prueba se selecciona automáticamente. 

- Después de implementar un APK en un dispositivo o emulador de Android, la base de datos de los usuarios y de los permisos de producción se utiliza automáticamente.




Para modificar la base de datos utilizada (base de datos de usuarios y permisos de prueba o producción) mediante programación, seleccione: 

- un inicio manual del Groupware Usuario. 

- la función [gpwOpen](../WDLang6/3041002.md).

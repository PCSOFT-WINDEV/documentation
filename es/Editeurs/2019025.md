## Presentación
<a name="presentacion_ELTTEXTE000150"></a>
Existen varias herramientas de prueba disponibles para garantizar la calidad de sus aplicaciones:

- El modo de prueba (Ir a un proyecto o Ir a una ventana) se utiliza para comprobar inmediatamente una modificación en su aplicación.




Para automatizar estas pruebas y aumentar la calidad de sus aplicaciones, tiene la capacidad de ejecutar pruebas de la unidad **automatic**. Estas pruebas se utilizan para comprobar fácilmente todas las características propuestas por sus aplicaciones.

Cada prueba contiene un escenario que se puede tratar directamente en la interfase del producto.. Este escenario está escrito en WLanguage y puede ser modificado en cualquier momento.

Estas pruebas se pueden ejecutar antes de crear el ejecutable para comprobar el modo de funcionamiento de una aplicación.

Observación: Las pruebas automatizadas son tenidas en cuenta por "[Cobertura de código](../Editeurs/2019034.md)".

## ¿Qué elementos se pueden comprobar automáticamente?
<a name="¿que_elementos_pueden_comprobar_automaticamente_ELTTEXTE000174"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) WINDEV y WINDEV Mobile le permiten ejecutar pruebas automatizadas en:

- [Windows](../Editeurs/2019026.md).

- [Conjuntos de procedimientos](../Editeurs/2019027.md).

- [Clases](../Editeurs/2019027.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [Un ejecutable](../Editeurs/2019029.md).




Tiene la capacidad de crear una prueba: 

- para una ventana específica,

- para toda la aplicación,

- no relacionado a una ventana, Procedure o a una clase.




![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) También tiene la posibilidad de importar escenarios de test creados por los usuarios.. Para obtener más información, consulte [Pruebas automatizadas creadas por el usuario](../Editeurs/2019031.md).

## Observaciones
<a name="observaciones_ELTTEXTE000204"></a>


- Cada prueba es associated con un código de lenguaje WLanguage: el escenario de prueba. Este escenario se puede ver en el editor de código. El código de las pruebas puede ser modificado.

- Las pruebas (y el código associated) no se incluyen en el ejecutable y no se entregan a los usuarios. El número de pruebas para una aplicación no tiene ninguna incidencia en el tamaño de la aplicación suministrada a los usuarios.

- Las pruebas automatizadas no funcionan en una sesión de TSE no interactiva. 

- El widget "Pruebas" que se encuentra en la [Proyecto cuadro de mando](../Editeurs/9500221.md) permite conocer rápidamente el porcentaje de pruebas superadas, no superadas, ... También se puede utilizar para realizar las diferentes pruebas.

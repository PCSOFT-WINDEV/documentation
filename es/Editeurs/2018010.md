### Modo de gestión del groupware
<a name="modo_gestion_del_groupware_ELTPARAGRAPHE000017"></a>

Se pueden utilizar dos métodos para incluir el Grupo de Usuarios:

- **la integración de Default**: toda la programación del Groupware Usuario se integra automáticamente en la aplicación.
	La ventana o página de inicio de sesión se incluye automáticamente en el proyecto.. La ventana o página de inicio de sesión se muestra cuando se inicia la aplicación, lo que permite al usuario conectarse a la aplicación.. Puede personalizar esta ventana o página aplicando el plantilla de su aplicación, por ejemplo.

- **integración personalizada**: toda la programación del Groupware Usuario se integra automáticamente en la aplicación. 
	Todas las ventanas o páginas necesarias para gestionar el User Groupware (ventana o página de acceso, ventanas o páginas de gestión de usuarios) se añaden automáticamente a su proyecto. Puede personalizar todas las ventanas o páginas utilizadas por el groupware usuario.

### Las diferentes ventanas de groupware usuario
<a name="las_diferentes_ventanas_groupware_usuario_ELTPARAGRAPHE000035"></a>

Dependiendo del modo de integración de groupware usuario en su aplicación, se pueden incluir una o varias ventanas en su proyecto. Estas ventanas son las siguientes.

**Observación**: el modo de funcionamiento de estas ventanas se presenta en [Configuración de groupware usuario](../Editeurs/2018006.md).


|   |   |   |
| --- | --- | --- |
| **Ventana de inicio de sesión** | ***GPWLOGIN.WDW*** | Esta ventana permite a todos los usuarios de la aplicación conectarse a la aplicación. En esta ventana, el usuario debe introducir el nombre y la contraseña (opcional) que fueron definidos previamente por el supervisor de la aplicación..<br><br>Durante el arranque, tiene la posibilidad de utilizar el login "SUPERVISOR" y la contraseña "SUPERVISOR"..<br><br>**Observación**: A partir de la versión 120044, la ventana de acceso ya no propone la lista de usuarios de Default. Para mostrar la lista de usuarios, la variable ***gbSecurity*** debe establecerse en <u><u><u><u>False</u></u></u></u> en el código de declaración de las variables globales de la ventana GPWLOGIN. Para habilitar esta función, la ventana de inicio de sesión debe volver a incluirse en el proyecto. |
| **Supervisor menu** | **GPWSupervisorMenú. JMTD** | Esta ventana sólo está disponible en modo personalizado.<br><br>Esta ventana permite al supervisor iniciar la aplicación o configurar los derechos. |
| **Formulario de usuario** | **Formulario GPWUserForm.WDW** | Esta ventana sólo está disponible en modo personalizado.<br><br>Esta ventana permite al supervisor introducir un nuevo usuario y sus datos personales. |
| **Asociar usuarios con configuraciones** | **GPWAssociationConfiguración. JMTD** | Esta ventana sólo está disponible en modo personalizado.<br><br>Esta ventana permite al supervisor gestionar los usuarios, los grupos y los derechos. |
| **Detalles de una configuración** | **GPWConfigurationDetails.WDW** | Esta ventana sólo está disponible en modo personalizado.<br><br>Esta ventana permite al supervisor configurar para cada ventana los derechos concedidos a un usuario (o a un grupo) sobre todos los controles, grupos de controles y opciones de menú que se encuentran en la ventana. |
| **Elegir una configuración** | **GPWConfigurationChoice. JMTD** | Esta ventana sólo está disponible en modo personalizado.<br><br>Esta ventana permite al supervisor de copy una configuración definida de antemano (para un usuario o para un grupo). |
| **Formulario de configuración** | **Formulario GPWConfigurationForm.WDW** | Esta ventana sólo está disponible en modo personalizado.<br><br>Esta ventana permite al supervisor crear un usuario o un grupo. |
| **Historial de conexiones** | **GPWConnectionHistory.WDW** | Esta ventana sólo está disponible en modo personalizado.<br><br>Esta ventana permite al supervisor ver el historial de conexiones (y borrar este historial si es necesario). |

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000100"></a>
Al crear un ejecutable, una biblioteca o un componente, el SCM propone comprobar en la última versión de los elementos que se encuentran en el SCM. Por lo tanto, el ejecutable, el componente o la librería contendrá los elementos más actualizados.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GDS_Exe.gif)
 Las diferentes opciones son las siguientes:

- **Comprobación de los elementos de la caja**: Esta opción se utiliza para comprobar todos los elementos que han sido comprobados por el desarrollador actual.. Los elementos se pondrán a disposición de los demás desarrolladores. Esta operación de facturación se realiza después de la creación. Se recomienda esta opción.

- **Obtener la última versión de los elementos del proyecto**: Esta opción le permite a get la última versión de todos los elementos del proyecto. La operación se realiza antes de crear el elemento solicitado.
	**Atención**: Esta opción puede provocar errores de compilación o diferencias de comportamiento. Le aconsejamos que realice la prueba del proyecto antes de generar el elemento.

- **get la última versión de los elementos del proyecto y el check in de los elementos del check-out**: Esta opción permite:

	- comprobar todos los elementos que han sido comprobados por el desarrollador actual. Los elementos se pondrán a disposición de los demás desarrolladores. Esta operación de facturación se realiza después de la creación.

	- get la última versión de todos los elementos del proyecto. La operación se realiza antes de crear el elemento solicitado.
			**Atención**: Esta opción puede provocar errores de compilación o diferencias de comportamiento. Le aconsejamos que realice la prueba del proyecto antes de generar el elemento.




- **Ninguna acción**:  Esta opción permite crear el elemento solicitado (ejecutable, biblioteca, componente,...) con los elementos de proyecto encontrados localmente.. No se realiza ninguna operación de facturación ni sincronización.




Por Default, el ejecutable, la biblioteca y el componente se generan con los elementos del proyecto que se encuentran actualmente en el ordenador local.

## Implementación
<a name="implementacion_ELTTEXTE000124"></a>
La sincronización es propuesta automáticamente por Default al crear el ejecutable.

Si marca "Realizar siempre esta elección sin preguntarme", tendrá la posibilidad de volver a activar esta ventana desde la ventana de description del proyecto:

1. Abra la ventana de descripción del proyecto: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. Visualizar la pestaña "SCM".

3. Seleccione "Proponga a get la última versión de los elementos al generar ejecutables, bibliotecas, componentes, ...".

4. Valide.




## Instalación de herramientas de desarrollo de Android
			





## Presentación
<a name="presentacion_ELTTEXTE000384"></a>
Mediante Default, cuando se genera una aplicación Android por primera vez, **la generación asistente propone descargar e instalar automáticamente todos los elementos necesarios**: 

- Abrir el JDK (en un ordenador nuevo que utilice la generación Android). 

- El SDK Android de Google,

- Gradle.


Por lo tanto, estas herramientas están siempre disponibles en la versión adaptada a la versión WINDEV Mobile que se utiliza actualmente.. 
Atención: La primera descarga puede ser bastante larga. 

Observación: Puedes configurar el Proxy que se usará para la descarga de automatic: 

- en la generación de aplicaciones Android asistente. 

- en las opciones de WINDEV Mobile (pestaña "Proxy"). 




Estos elementos también se pueden instalar manualmente. Este tipo de instalación no se recomienda porque las versiones de las diferentes herramientas pueden estar desfasadas con la versión esperada por WINDEV Mobile. 



## Manual instalación (no recomendado): los elementos necesarios
<a name="manual_instalacion_recomendado_los_elementos_necesarios_ELTTEXTE000408"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000027"></a>

Para desarrollar aplicaciones Android con WINDEV Mobile, debe instalar:

- [El JDK](#NOTE2_1): El JDK (Java Development Kit) distribuido por Oracle se utiliza para compilar los archivos Java generados.

- [El SDK Android de Google](#NOTE1_3): El SDK de Android (Software Development Kit) es un conjunto de archivos y aplicaciones distribuidos por Google que permiten compilar aplicaciones para Android.

- [Gradle](#NOTE3_1): Esta herramienta es necesaria para compilar y generar aplicaciones Android. 

- En algunos dispositivos, se deben instalar archivos adicionales para que el SDK y WINDEV Mobile detecten correctamente el dispositivo.


Al crear un proyecto o una configuración de proyecto para Android, se debe especificar el ruta del SDK de Android.
<a name="NOTE1_3"></a>


### SDK de Android
<a name="sdk_android_ELTPARAGRAPHE000049"></a>

**¿En dónde puedo obtener el SDK de Android?**
El SDK de Android puede descargarse desde el sitio de Google dedicado al desarrollo para Android ([http://developer.android.com](http://developer.android.com)).
Para desarrollar aplicaciones Android con WINDEV Mobile, solo se necesita el "Stand-alone SDK". 

La versión para Windows del SDK "Stand-alone" se encuentra en la siguiente dirección: [http://developer.android.com/sdk/installing/index.html?pkg=tools](http://developer.android.com/sdk/installing/index.html?pkg=tools).

- Haga clic en "Download the SDK Now". 

- Descargue el SDK para Windows. 


**Importante**: Estas direcciones son válidas en el momento de la publicación de esta documentación.

**¿Cómo instalar el SDK de Android?**
La Procedure es la siguiente: 

1. Descomprima el archivo descargado en una carpeta en la raíz de la unidad "C:" para evitar una virtualización de Windows. Por ejemplo: "C:\\Android\\". 

2. Abra la carpeta con el explorador de Windows. 

3. Inicie "SDK Manager.exe". Este programa sirve para instalar las versiones existentes del SDK de Android, así como sus actualizaciones.




**Observación**: Puede que sea necesario agregar la ruta del subdirectorio "Tools" en el PATH de su equipo. Para ello: 

- Mostrar las propiedades del ordenador en funcionamiento. 

- En la pestaña "Avanzado" haga clic en el botón "Variables de entorno". 

- Seleccione la opción "PATH" en las variables del sistema y haga clic en "Modificar". 

- Agregue la ruta completa de directorio "Tools". 




**Observación**: El programa "SDK Manager.exe" puede mostrar el siguiente mensaje de error: 
"Failed to fetch URL https://dl-ssl.google.com/android/repository/repository.xml, reason: HTTPS SSL error. You might want to force download through HTTP in the settings.". 
Este mensaje indica que el gestor del SDK de Android no ha conseguido establecer una conexión HTTPS para descargar los componentes del SDK. Para resolver este problema, debe hacerlo:

- Hacer clic en "Settings" en la lista a la izquierda de la ventana. 

- Seleccionar la casilla "Force https://... source to be fetched using http://". 

- Hacer clic en "Save & Apply". 




Observación: Si no ha elegido la opción de actualización de automatic propuesta por Gradle y el SDK Android, este ruta tendrá que ser especificado en [la generación de aplicaciones Android asistente](../Editeurs/9000109.md). 


### Versión de Android SDK necesaria para WINDEV Mobile
<a name="version_android_sdk_necesaria_para_windev_mobile_ELTPARAGRAPHE000101"></a>

WINDEV Mobile necesita el SDK de Android versión 4.0 (o posterior). Esta versión permite desarrollar aplicaciones para Android 4.0 y posteriores.
Si se instala una versión más reciente, WINDEV Mobile utilizará esta versión. Las aplicaciones generadas seguirán siendo compatibles con la versión 4.0 de Android.

**Observación**: Para utilizar la detección de la plataforma Android desde WINDEV Mobile, también le aconsejamos instalar el paquete "USB Driver". 

**Atención: Si no utiliza la descarga de automatic de Android SDK**, para generar las aplicaciones Android con WINDEV Mobile, debe: 

- Actualización de las herramientas del SDK de Android. 

- instalar los dos siguientes complementos: "Android Support Repository" y "Google Repository".

### JDK utilizado para el desarrollo de Android
<a name="jdk_utilizado_para_desarrollo_android_ELTPARAGRAPHE000268"></a>**¿En dónde puedo obtener el JDK?**
Puede descargar: 

- el Oracle JDK, que puede descargarse del sitio de Oracle en: [http://www.oracle.com/technetwork/java/javase/downloads/index.html](http://www.oracle.com/technetwork/java/javase/downloads/index.html).

- OpenJDK, disponible para su descarga en: [https://openjdk.Java.net/](https://openjdk.java.net/)





Versión de **JDK para instalar:** 


|   |   |   |   |
| --- | --- | --- | --- |
|   | WINDEV Mobile 24 y 25 hasta la versión "Update 1" | WINDEV Mobile 25 de la versión "Update 2" y posterior | WINDEV Mobile 27 |
| Versión JDK | 1.8 o 9 | 1,8 o 10 | 11 (disponible sólo en modo de 64 bits) |

**Observaciones**: 

- El JDK es suficiente. No es necesario descargar las versiones que incluyen módulos adicionales (JavaFX o NetBeans por ejemplo).

- El JDK 8 a veces se llama JDK 1.8. Estos dos nombres identifican el mismo producto.

- La versión de 64 bits del JDK debe ser instalada.

- Si varios JDK se instalan a satisfacer los requisitos específicos, el Android aplicación de generación de asistente le permite seleccionar el JDK de usar para la compilación en el: 

	1. Haga clic en "Configuración avanzada". 

	2. En la ventana que aparece, en "Comando del compilador Java, seleccione "Personalizar...".

	3. Seleccione el compilador de Java para usar. 

	4. Valide la ventana y continúe con la generación de la aplicación asistente.

### Gradle
<a name="gradle_ELTPARAGRAPHE000322"></a>

**¿Cómo obtener Gradle?**
Puede compilar proyectos Android en WINDEV Mobile 27 utilizando la versión 7.0.2 de Gradle.

Gradle puede ser descargada desde el siguiente sitio: [http://gradle.org/gradle-download/](http://gradle.org/gradle-download/). La versión actualmente disponible para su descarga es más reciente que la versión 7.0.2, necesaria para WINDEV Mobile. 

- En "versiones anteriores", haga clic en "versiones Page".

- En el Page que aparece, seleccione la versión 5.4.1. 

- Haga clic en "Binary-only". 

- Se descarga un archivo Zip. 

- Descomprimir los archivos comprimidos en un directorio del ordenador de desarrollo.

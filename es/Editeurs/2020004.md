## Presentación
<a name="presentacion_ELTTEXTE000178"></a>
WINDEV y WEBDEV permiten realizar múltiples tipos de procesos desde un comando Line

Es posible:

- Compilar y generar el ejecutable (EXE) associated con uno o más proyectos.

- Crear la biblioteca (WDL) associated con uno o más proyectos.

- Generar los componentes (WDK) associated con uno o más proyectos.






**Observación**: También tiene la capacidad de realizar procesos por lotes multiproyecto.. En este caso, tiene la posibilidad de configurar las diferentes operaciones realizadas durante la compilación o al crear el ejecutable.. Para obtener más información, consulte [Procesos batch multiproyecto](../Editeurs/2020013.md).

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000202"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000041"></a>

Todo lo que tienes que hacer es crear un archivo de texto que contenga la ruta completa de los proyectos correspondientes a la acción a realizar. Este archivo de texto tiene el siguiente formato:


```txt
C:\My Projects\My Sales Management\Customer\Customer.WDP
C:\My Projects\My compatibility\Compatibility.WDP
```


A continuación, especifique mediante un comando Line la acción a realizar sobre los proyectos referidos en este archivo (ver los siguientes párrafos para más detalles).
<a name="NOTE2_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Generación automática de ejecutables
<a name="generacion_automatica_ejecutables_ELTPARAGRAPHE000054"></a>

Para generar automáticamente los ejecutables:

1. Cree el archivo de texto que contiene la ruta de los proyectos para los que deben crearse los ejecutables.

2. Ejecute el siguiente comando Line: WINDEV28.EXE /CREATEEXE=&lt;Ruta del archivo de texto&gt;
	Por ejemplo: WINDEV28.EXE /CREATEEXE="C:\\Temp\\Test.txt"

3. El editor correspondiente se inicia automáticamente. Cada proyecto especificado en el archivo de texto se abre y se compila, luego se crea el ejecutable con todas las opciones seleccionadas por Default (o con las opciones seleccionadas durante la última creación del ejecutable para este proyecto).. El editor se cierra automáticamente una vez generado el ejecutable.




Para más detalles sobre los diferentes pasos para generar un archivo ejecutable, véase [Crear un ejecutable](../Editeurs/2025002.md).
<a name="NOTE2_3"></a>


### Creación automática de la biblioteca principal de la aplicación
<a name="creacion_automatica_biblioteca_principal_aplicacion_ELTPARAGRAPHE000076"></a>

Para crear automáticamente la biblioteca principal de la aplicación:

1. Cree el archivo de texto que contiene la ruta de los proyectos para los que debe crearse la biblioteca.

2. Ejecute el siguiente comando Line:
	WINDEV28.EXE /CREATEWDL=&lt;Ruta del archivo de texto&gt;
	Por ejemplo: WINDEV28.EXE /CREATEWDL="C:\\Temp\\Test.txt"

3. Cada proyecto especificado en el archivo de texto se abre y se compila, luego se crea la biblioteca principal con todas las opciones seleccionadas por Default (o con las opciones seleccionadas durante la última creación de la biblioteca para este proyecto).. El editor se cierra automáticamente una vez creada la biblioteca.

4. El editor correspondiente se inicia automáticamente. Cada proyecto especificado en el archivo de texto se abre y se compila, luego se crea la biblioteca principal con todas las opciones seleccionadas por Default (o con las opciones seleccionadas durante la última creación de la biblioteca para este proyecto).. El editor se cierra automáticamente una vez creada la biblioteca.




Para más detalles sobre las bibliotecas, véase [Biblioteca: Definition](../Editeurs/2030022.md).

### Generar automáticamente una Component a partir de un proyecto:
<a name="generar_automaticamente_una_component_partir_proyecto_ELTPARAGRAPHE000101"></a>

Para generar automáticamente una Component a partir de un proyecto:

1. Cree el fichero de texto que contiene los ruta de los proyectos para los que debe crearse el Component asociado. Si el proyecto está asociado a varios Component s, sólo se generará el primer Component.

2. Ejecute el siguiente comando Line:

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) En WINDEV:
			WINDEV28.EXE /CREATEWDK=&lt;Ruta del archivo de texto&gt;
			Por ejemplo: WINDEV28.EXE /CREATEWDK="C:\\Temp\\Test.txt"




3. El editor se inicia automáticamente. Cada proyecto especificado en el fichero de texto se abre y se compila, luego se genera el primer Component encontrado asociado al proyecto con todas las opciones seleccionadas por defecto (o con las opciones seleccionadas durante la última generación del Component para este proyecto). El editor se cierra automáticamente una vez generada la Component.




Para más detalles sobre los distintos pasos para generar una Component, consulte [Creación de una Component](../Editeurs/2014002.md).

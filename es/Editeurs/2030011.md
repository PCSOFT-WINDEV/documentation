## Eventos de Default
<a name="eventos_default_ELTTEXTE000314"></a>
WINDEV, WEBDEV y WINDEV Mobile manipular los siguientes eventos por Default (orden de aparición en el editor de código):

| Event | Condición de ejecución |
| --- | --- |
| Inicialización | Ejecutar al iniciar la aplicación. Este es el orden en el que se ejecutan los eventos al iniciar una aplicación (también al ejecutar una prueba):<br><br>- Inicialización de los componentes "cargados al iniciar la aplicación".<br><br>	- Inicialización de los subcomponentes (\*).<br><br>	- Inicialización de clases de componentes.<br><br>	- Inicialización de los conjuntos de procedimientos del componente.<br><br>	- Inicialización del proyecto componente (\*).<br><br><br><br><br>- Inicialización de clases.<br><br>- Inicialización de conjuntos de procedimientos.<br><br>- Inicialización del proyecto.<br><br>- Inicialización de la primera ventana de proyecto o del elemento cuya prueba se ejecuta.<br><br><br>(\*) La inicialización Event de un componente se ejecuta sólo cuando el componente utiliza el mismo contexto de ejecución que el proyecto. Para más detalles sobre las opciones de generación de componentes, véase [Crear y generar un componente externo](../Editeurs/2014002.md).<br><br>Si se ejecuta [aprovechamiento](../WDLang1/3038044.md), se interrumpe la inicialización del proyecto Event (todas las líneas de código después de [aprovechamiento](../WDLang1/3038044.md) no se ejecutarán).<br><br>**Ejemplos**<br>Algunos ejemplos de cómo usar este Event:<br><br>- Inicialización de las variables globales del proyecto.<br><br>- Creación de los archivos de datos ([HCreation](../WDLang4/3044255.md) y [HCreationIfNotFound](../WDLang4/3044254.md)).<br><br>- Definición de directorios para crear archivos de datos ([HChangeDir](../WDLang4/3044168.md) y [HSubstDir](../WDLang4/3044028.md)).<br><br>- Descripción de las conexiones temporales a una base de datos ([HDescribeConnection](../WDLang4/3044205.md)).<br><br><br> |
|   | Se utiliza si el sitio se ejecuta con sesiones en caché. <br><br>Para gestionar las sesiones prelanzadas, dos eventos son associated con el proyecto:<br><br>- "Iniciando el proyecto", ejecutado antes de que la solicitud llegue a la WEBDEV servidor de aplicaciones. Este código se ejecuta al iniciar la sesión prelanzada. <br><br>- "Iniciando el proyecto después de la conexión al sitio", se ejecuta cuando el WEBDEV servidor de aplicaciones procesa la solicitud. <br><br><br><br><br>Atención: Algunas funciones de WLanguage no deben ser usadas en el Event "Iniciando el proyecto". Deben ser trasladados a la Event "Iniciando el proyecto después de la conexión al sitio". <br><br>Para obtener más información, consulte [Sesiones preiniciadas](../WDLang2/1000019485.md). |
| Cierre | Ejecutar cuando se cierra la aplicación (cuando se cierra la última ventana de la aplicación, por ejemplo).<br>Este es el orden en el que se ejecutan los eventos para cerrar el proyecto:<br><br>- Cierre del proyecto.<br><br>- Event para terminar los conjuntos de procedimientos.<br><br>- Event para terminar las clases.<br><br>- Cerrar los componentes (cerrando el proyecto del componente, luego sus conjuntos de procedimientos, luego sus clases, luego sus subcomponentes).<br><br><br>**Este Event no se ejecutará:**<br><br>- Cuando se ejecuta [EndProgram](../WDLang1/3013033.md) y el parámetro "Código de cierre" no se especifica o se fija en "False".<br><br>- Al ejecutar la función [AppControl](../WDLang1/3013003.md) con el parámetro "Ejecutar código de cierre del proyecto" no especificado o igual a "False".<br><br><br> |
| ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Cierre de la sesión | Ejecutar en los siguientes casos si la aplicación se utiliza actualmente:<br><br>- Al cerrar la sesión de usuario actual.<br><br>- Cuando se detiene el sistema de la computadora actual.<br><br>- Al reiniciar el ordenador actual.<br><br><br>En estos casos, sólo este Event se ejecutará. Los eventos de cierre del proyecto, así como los eventos de finalización de las clases y los conjuntos de procedimientos no se llevarán a cabo..<br>**Observación**: Dependiendo del sistema operativo del ordenador actual, el sistema operativo puede intentar detener abruptamente la aplicación después de un tiempo..<br><br>**Ejemplos**<br>Algunos ejemplos de cómo usar este Event:<br><br>- Liberación de las bibliotecas cargadas en memoria por [FreeDLL](../WDLang1/3014009.md) (si se han cargado bibliotecas en memoria).<br><br>- Liberación de clases, objetos OLE Automation y arrays dinámica (palabra clave [Libre](../Motscles/1511012.md)).<br><br>- Anulación de las operaciones en curso ([HTransactionCancel](../WDLang4/3044001.md)).<br><br>- Cierre de los archivos HFSQL ([HClose](../WDLang4/3044073.md)).<br><br>- Envío de mensajes de información a otros ordenadores ([SendMessage](../WDLang1/3015003.md)).<br><br>- Ejecutar el Event para el cierre del proyecto con [EndProgram](../WDLang1/3013033.md) con el "Código de cierre" fijado en "True".<br><br><br> |
| ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) <br>Prueba automatizada | Opcional Event.<br><br>Para habilitar esta Event, vaya a la pestaña "Pruebas automatizadas", grupo "Pruebas", haga clic en el lanzador del cuadro de diálogo (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_regroup.gif)) y seleccione "Habilitar el código de prueba automatizado del proyecto".<br><br>Para desactivar esta Event, vaya a la pestaña "Pruebas automatizadas", grupo "Pruebas", haga clic en el lanzador de cuadros de diálogo (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_regroup.gif)) y seleccione "Desactivar el código de prueba automatizado del proyecto".<br><br>En el caso de una prueba automatizada, esta Event se ejecuta antes del código de inicialización del proyecto. Este código se utiliza para Process algunos casos especiales en los que la prueba automatizada puede estar bloqueada: abrir una ventana en el código de inicialización del proyecto (ventana de configuración, ventana de licencia, ventana de inicio de sesión personalizada, ....).<br><br>El código de prueba automatizado del proyecto permite utilizar [EmularVentana](../WDLang1/1000009009.md) para validar la ventana. |
|   | Ejecutar cuando la aplicación se mueva a la Background. Esta Event se utiliza, por ejemplo, para guardar los datos.<br><br>Observación: Este Event se ejecuta DESPUÉS de la ventana opcional Event "Mover aplicación al Background". |
|   | Ejecutar cuando la aplicación se habilite de nuevo y cuando se mueva al primer plano. Este Event se utiliza, por ejemplo, para actualización los datos visualizados.<br><br>Observaciones:<br><br>- Este Event no se ejecuta cuando se inicia la aplicación.<br><br>- Este Event se ejecuta ANTES de la ventana opcional Event "Mover la aplicación al primer plano".<br><br><br> |
|   | Se ejecuta en los siguientes casos: <br><br>- Al iniciar la aplicación (después del evento "Inicialización" del proyecto).<br><br>- Cuando se cambia el tema en el código con [ChangeTheme](../WDLang3/1000025266.md) sólo si se ha especificado la constante *withCodeExecution*.<br><br>- Cuando el sistema cambia automáticamente el tema: modificación de los ajustes del sistema, cambio de hora día/noche, activación/desactivación del ahorrador de batería, etc.<br><br><br><br><br>Observación: Este Event sólo está disponible si se activa la gestión de temas oscuros. Para obtener más información, consulte [Gestión del tema oscuro en dispositivos móviles](../WDLang3/1000025338.md). |
|   | Se ejecuta cuando se inicia el servidor de aplicaciones WEBDEV o la aplicación.<br><br>Observación: Dado que las tareas programadas no son persistentes en WEBDEV, esta Event permite añadir tareas programadas de forma programada ([AppServerAddScheduledTask](../WDLang2/1000022347.md)). |
|   | Se ejecuta cuando se instala, actualiza o desbloquea una aplicación. |
|   | Se ejecuta cuando se detiene el servidor de aplicaciones WEBDEV. |

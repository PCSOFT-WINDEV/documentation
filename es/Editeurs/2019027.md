


## Pruebas automatizadas de procedimientos y clases
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000109"></a>
Las pruebas automatizadas de Procedure o de la clase se utilizan para probar las diferentes características propuestas por el Procedure o por la clase.

**Observación: Uso en Windows Vista (y posterior)**: Para Record y repetir las pruebas de la unidad automatic, el mecanismo UAC debe ser desactivado.

<a name="NOTE2_1"></a>


## ¿Cómo crear la prueba?
<a name="¿como_crear_prueba_ELTTEXTE000133"></a>


### Para crear una prueba automatizada sobre una Procedure (o sobre un método de clase):
<a name="para_crear_una_prueba_automatizada_sobre_una_procedure_sobre_metodo_clase_ELTPARAGRAPHE000020"></a>

***1er método***

1. Si es necesario, muestre el panel "Explorador de proyectos".

2. En el panel "Explorador de proyectos", seleccione el nombre del método o procedimiento (global o local) a probar.

3. En el menú contextual, seleccione la opción "Pruebas unitarias" .. "Crear una prueba unitaria".

4. Se crea automáticamente un nuevo escenario de test. El escenario de test contiene la llamada a la función. El código requerido para ejecutar la prueba de la Procedure o la prueba de método debe ser escrito por el desarrollador. Si los parámetros son esperados por el método o por el Procedure, tienes la capacidad de crear un set de datos para la prueba.




***2º método***

1. Si es necesario, muestre el panel "Código".

2. En el panel "Código", seleccione el nombre del método o Procedure cuya prueba debe ser ejecutada.

3. En el menú contextual, seleccione la opción "Pruebas unitarias" .. "Crear una prueba unitaria".

4. Se crea automáticamente un nuevo escenario de test. El escenario de test contiene la llamada a la función. El código requerido para ejecutar la prueba de la Procedure o la prueba de método debe ser escrito por el desarrollador. Si los parámetros son esperados por el método o por el Procedure, tienes la capacidad de crear un set de datos para la prueba.




***3º método***

1. Mostrar la Procedure o el método en el editor de códigos.

2. En la pestaña "Pruebas automatizadas", en el grupo "Pruebas", despliegue "Nuevo" y seleccione "Grabar un nuevo escenario".

3. Se crea automáticamente un nuevo escenario de test. El escenario de test contiene la llamada a la función. El código requerido para ejecutar la prueba de la Procedure o la prueba de método debe ser escrito por el desarrollador. Si los parámetros son esperados por el método o por el Procedure, tienes la capacidad de crear un set de datos para la prueba.




Observación: Para crear una set de datos de prueba, utilice [TestAddIteration](../WDLang1/1000017015.md). Para obtener más información, consulte [Código del escenario](../Editeurs/2019033.md).




## Componentes externos multiproducto
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000130"></a>
Los componentes externos son una herramienta notablemente useful para los equipos de desarrollo. La posibilidad de reutilizar el código de secure es una gran ventaja.

Las posibles diferencias entre los distintos objetivos se pueden tener en cuenta automáticamente: Windows, INTERNET-INTRANET, aplicaciones móviles, etc.

Estos componentes externos pueden contener ventanas o páginas y pueden ser utilizados en proyectos WINDEV, WEBDEV y WINDEV Mobile.

Cualquier diferencia en el código o en UI se tiene en cuenta automáticamente.

En tiempo de ejecución, el componente externo se adaptará a la plataforma de destino.

<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000154"></a>
Tienes la capacidad de desarrollar proyectos comunes a los diferentes productos: WINDEV, WEBDEV y WINDEV Mobile. Estos proyectos pueden contener ventanas WINDEV, ventanas WINDEV Mobile y/o páginas WEBDEV.

**Crear un componente externo multiproducto a partir de un proyecto común es muy fácil**:

1. Cree los diferentes elementos de su componente (ventana, páginas,...).
	Recordatorio: Las declaraciones [COMPILAR IF](../Motscles/1510021.md) o [IF Objetivo de ejecución](../Motscles/1510028.md) permiten utilizar los procesos independientemente del entorno, la plataforma o el tipo de generación. También puede utilizar [InComponentMode](../WDLang1/3013023.md), [InJavaMode](../WDLang1/3013058.md), [InPHPMode](../WDLang1/3013057.md), [InWebMode](../WDLang1/3013042.md), etc.

2. Crear la configuración del proyecto "Componente: en la pestaña "Proyecto", en el grupo "Configuración del proyecto", amplíe "Nueva configuración" y seleccione "Componente externo (WDK)".

3. En la creación de la configuración asistente:

	- En el paso "Compatibilidad de la configuración", especifique: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Composant_factory%20-%20HC%20N%B0001.gif&type=thumb)


		- El tipo de configuración en la que se puede utilizar el componente. Puede seleccionar varias configuraciones. Por ejemplo, el componente puede utilizarse en aplicaciones WINDEV e iOS.   

- Las plataformas para las que se deben habilitar los errores de compilación. 

- En el siguiente paso, seleccione los elementos que se integrarán en la configuración del proyecto. Puede incluir todos los elementos requeridos por su componente (ventanas, páginas, procedimientos,...).

- En el siguiente paso, seleccione los componentes internos a integrar (si es necesario).

4. Validar. Se crea la nueva configuración del proyecto. Puede generar el componente externo. Para obtener más información, consulte [Crear y generar un componente externo](../Editeurs/2014002.md). 




Observaciones: 

- Si utiliza versiones independientes de WINDEV, WEBDEV o WINDEV Mobile, algunas combinaciones de plataformas no están disponibles. 

- También puede crear un proyecto común de "Componentes". La creación del proyecto asistente permite elegir las plataformas en las que se puede utilizar el componente. 

- Atención: Si utiliza una versión integrada WINDEV-WEBDEV-WINDEV Mobile para crear un componente que pueda utilizarse en Android y/o iOS, debe crear una configuración "Android" y una "iOS" en el proyecto "Componente. Estas configuraciones le permitirán crear ventanas móviles. Estas ventanas (associated con las configuraciones "Android" o "iOS") deben ser también associated con la configuración del proyecto "Componente.  

- Al generar un componente multiplataforma se crea una única set de archivos (WDK, WDI, WDZ) que puede importarse en los proyectos WINDEV, WEBDEV y WINDEV Mobile correspondientes a las plataformas de destino.

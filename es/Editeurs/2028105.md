
## Historia de las versiones (red instalación)
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000080"></a>
La instalación asistente permite configurar el historial de versiones durante una red instalación. 

La configuración de la historia de las versiones de instalación se utiliza para poner a disposición de los usuarios finales varias versiones de la misma aplicación. 

El historial de las configuraciones se puede configurar: 

- al crear la instalación,

- al implementar la reference instalación ("Estos parámetros pueden ser modificados durante la implementación en el servidor").  




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Mantener un historial de las versiones instaladas
<a name="mantener_historial_las_versiones_instaladas_ELTTEXTE000104"></a>
Si se selecciona "Mantener un historial de las versiones instaladas", se puede definir: 

- los **número máximo de versiones almacenadas**. Estas versiones se almacenarán en el servidor de reference. Las versiones obsoletas se borrarán cada vez que se almacene una nueva versión.. 

- el **tamaño máximo de la historia** (para evitar la saturación del disco del servidor reference). 

- los **activación inmediata de la versión** instalado en el servidor. En este caso, de acuerdo con los parámetros de comprobación especificados, la aplicación propondrá una actualización. Si esta opción no está seleccionada, la versión se puede habilitar a través de [Centro de aplicaciones control](../WDAdminExe/2028052.md). 


Observación: Para forzar la modificación de los parámetros introducidos durante un implementación anterior, seleccione "Estos parámetros sustituirán a los eventuales parámetros introducidos anteriormente en implementación".




## WDSetup: Gestión de idiomas adicionales
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000247"></a>
El proyecto "WDSetup" corresponde al programa estándar instalación de aplicaciones WINDEV.

Este proyecto apoya el francés y el inglés (EE.UU.) de Default.

**Observaciones**: 

- Si está utilizando un WDSetup personalizado, debe tener en cuenta las evoluciones realizadas entre las diferentes versiones. Para aplicar las evoluciones, utilice el archivo comparison: en la pestaña "Proyecto", en el grupo "Otras acciones", despliegue "Comparar" y seleccione "Comparar dos proyectos". En asistente, seleccione su proyecto y el proyecto WDSetup proporcionado con WINDEV.

- Puedes usar un plug-in para añadir nuevas características o nuevos pasos a la asistente. Para obtener más información, consulte [Personalizar el programa instalación (Plug-in)](../Editeurs/2028109.md). 

- El proyecto WDSetup viene con dos configuraciones: una configuración de 32 bits y una configuración de 64 bits.

## Gestión de idiomas adicionales en WDSetup
<a name="gestion_idiomas_adicionales_wdsetup_ELTTEXTE000271"></a>
Para gestionar idiomas adicionales en WDSetup, puede: 

- Personalice el ejemplo "WDSetup", que se suministra de serie con el producto.. 

- Crear un nuevo proyecto basado en el ejemplo "WDSetup".. Esta solución se recomienda porque elimina el riesgo de sobrescribir el proyecto durante una actualización.. 




**To crear un nuevo proyecto basado en WDSetup**: 

1. Desde la ventana de inicio de WINDEV, haga clic en "Crear un proyecto".. El asistente de creación de proyectos se abre. 

2. Dé un nombre a su proyecto e indique la plataforma de tiempo de ejecución. 

3. En "description - Modo de creación", elija "Crear un proyecto a partir de un ejemplo existente". Pase a la etapa siguiente del asistente. 

4. Elija el proyecto preestablecido que desea utilizar: WDSetup. Pase a la etapa siguiente. 

5. Indique si el proyecto debe guardarse en el archivo SCM (Administrador de Código Fuente). 

6. Validar. El proyecto se crea automáticamente. Contiene todos los elementos del ejemplo "WD Setup" y puede ser modificado y personalizado.. 




**To añadir los nuevos idiomas al proyecto personalizado WDSetup**: 

1. Añade los idiomas adicionales en la pestaña "Idiomas" del proyecto description. 
	 Para abrir la descripción del proyecto, en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. Traducir la interfaz del proyecto (ventanas, controles,...) así como los mensajes de código.. 
	**Recordatorio**: El WDMSG se utiliza para simplificar la traducción de un proyecto, revisando todos los mensajes del proyecto (títulos de control, títulos de las ventanas, ...) y volviéndolos a revisar una vez que han sido traducidos.. WDMSG es un módulo adicional de WINDEV. Contact PC SOFT Departamento de Ventas para más detalles sobre WDMSG.

3. Crear un nuevo ejecutable para cada configuración (configuración de 32 bits y configuración de 64 bits). 
	Recordatorio: Para crear el ejecutable: 

	- habilitar la configuración deseada (32 bits o 64 bits). 

	- despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generer.gif) en los botones de acceso rápido. 

	- seleccione "Generar el ejecutable de 32 bits (.exe)" o "Generar el ejecutable de 64 bits (.exe)".. 
			En la asistente, no olvides incluir los nuevos lenguajes en el ejecutable.

## Ejecución de la prueba de WDSetup personalizado
<a name="ejecucion_prueba_wdsetup_personalizado_ELTTEXTE000295"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000096"></a>

Para ejecutar la prueba de un programa personalizado de instalación (WDSetup personalizado), debe:

1. Configure el programa instalación y genere este programa instalación.

2. Ejecute la prueba del proyecto WDSetup personalizado en WINDEV. Para hacer esta prueba, debes:

	- copy los archivos en el directorio EXE del programa personalizado instalación,

	- introducir un comando específico Line.






<a name="NOTE3_2"></a>


### Configurando y generando el programa instalación
<a name="configurando_generando_programa_instalacion_ELTPARAGRAPHE000109"></a>

Para configurar el programa instalación y generar este programa instalación: 

1. Configurar el programa instalación con [la creación del programa instalación asistente](../Editeurs/2028062.md) o en [el editor de instalación (WDInst)](../Editeurs/2028042.md).

2. Si está utilizando la creación del programa instalación asistente, genere el instalación e inicie WDInst.

3. En el editor WDInst de instalación, seleccione "Configuración de las opciones del instalador".

4. En la pestaña "Framework WINDEV", seleccione "Copiar el framework WINDEV con la instalación". 
	A continuación, haga clic en la pestaña "Personalizar el instalador. 
	Seleccione "Usar un instalador personalizado" y seleccione el ejecutable del programa personalizado de instalación. 
	Observaciones: 

	- Dependiendo del tipo de instalación a realizar, se puede seleccionar el ejecutable correspondiente al programa instalación de 32 bits o al programa instalación de 64 bits.

	- Las bibliotecas correspondientes (archivos ".DLL") pueden ser associated con el ejecutable del programa instalación personalizado en la pestaña "DLLs del instalador" (caso de bibliotecas renombradas, por ejemplo).




5. Validar la ventana para las opciones del instalador. 

6. Seleccione "Archivo... Generar".

7. Para ejecutar la prueba, recomendamos que seleccione un directorio único para el apoyo del programa instalación.

8. Especifique el directorio para generar el programa instalación.

9. En la pestaña "Avanzado", marque "Guardar el archivo WXF para ejecutar la prueba de un WDSetup personalizado".. Para obtener más información, consulte [Archivo WXF](../Editeurs/2028055.md).

10. Valide la ventana para generar el programa instalación.



<a name="NOTE3_3"></a>


### Ejecución de la prueba de WDSetup personalizado
<a name="ejecucion_prueba_wdsetup_personalizado_ELTPARAGRAPHE000147"></a>

Para ejecutar la prueba de un WDSetup personalizado en WINDEV:

1. En el directorio "EXE" (subdirectorio correspondiente a la configuración de 32 bits o 64 bits) del programa instalación personalizado (proyecto "WDSetup" personalizado), copy los siguientes archivos:

	- el archivo "&lt;ApplicationName&gt;.WXF" (que se encuentra en el directorio del proyecto a instalar), 

	- los archivos de licencia, si se muestra un acuerdo de licencia al principio de la aplicación instalación,

	- los archivos Image utilizados para la interfaz del programa instalación (fondo Image, logotipo, ...),

	- los archivos Image utilizados para las diapositivas del programa instalación (si es necesario).




2. En WINDEV, abre el programa personalizado instalación (proyecto personalizado "WDSetup").

3. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Configurar modo de prueba".

4. Introduzca el siguiente comando Line: "/Dir=&lt;PathINSTALLDirectory&gt;", donde &lt;PathINSTALLDirectory&gt; corresponde a la ruta del directorio INSTALL creado en el directorio de generación del programa instalación. 
	Por ejemplo, si el directorio de generación del programa instalación es: "C:\\Temp", la ruta del directorio INSTALL será: "C:\\Temp\\INSTALL".

5. Seleccione la configuración del proyecto correspondiente a la versión a comprobar (32 o 64 bits) y ejecute el test del proyecto (Ctrl + F9).




Atención: Cuando se ejecuta esta prueba, la instalación se realiza en el directorio especificado.

## Usando un WDSetup personalizado como programa de instalación
<a name="usando_wdsetup_personalizado_como_programa_instalacion_ELTTEXTE000331"></a>
El programa personalizado de instalación se selecciona en el editor de instalación WDInst.

**Usar un WDSetup personalizado como instalación program**:

1. Seleccione "Parámetros de configuración ... Opciones del instalador".

2. En la pestaña "Framework WINDEV", seleccione "Copiar el framework WINDEV con la instalación". 
	A continuación, haga clic en la pestaña "Personalizar el instalador. 
	Seleccione "Usar un instalador personalizado" y seleccione el ejecutable del programa personalizado de instalación. 
	Observaciones: 

	- Dependiendo del tipo de instalación a realizar, se puede seleccionar el ejecutable correspondiente al programa instalación de 32 bits o al programa instalación de 64 bits.

	- Las bibliotecas correspondientes (archivos ".DLL") pueden ser associated con el ejecutable del programa instalación personalizado en la pestaña "DLLs del instalador" (caso de bibliotecas renombradas, por ejemplo).




3. Validar la ventana para las opciones del instalador.




## Probar un proyecto: ajuste
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000284"></a>
La prueba de todo el proyecto simula cómo se inicia el sitio o la aplicación (por ejemplo, a través del ejecutable de una aplicación WINDEV). Esto permite probar el sitio o la aplicación en su totalidad, incluso si su desarrollo aún no se ha completado. Cuando se produce un problema al ejecutar el proyecto, puede iniciar el depurador para identificar y solucionar el problema.

Se utilizan diferentes parámetros:

- Pruebe un proyecto especificando un comando Line.

- Pruebe un proyecto especificando un directorio de ejecución específico.

- Probar un proyecto con un manifiesto (sólo en Windows Vista).

- Probar un proyecto utilizando los derechos concedidos a otro usuario. Este ajuste se utiliza para comprobar el modo de funcionamiento de la aplicación simulando el mecanismo UAC en Windows Vista.




**Observación**: La prueba de un proyecto se puede ejecutar independientemente del elemento actual en el editor.

## Configuración del modo de prueba
<a name="configuracion_del_modo_prueba_ELTTEXTE000308"></a>


### Línea de comandos
<a name="linea_comandos_ELTPARAGRAPHE000027"></a>

Para probar un proyecto mediante un comando Line, basta con especificar este comando Line: 

1. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Configurar modo de prueba" (Ctrl + Shift + F9).

2. Escriba el comando Line.

	- El comando personalizado **Line** corresponde a los parámetros personalizados utilizados para ejecutar la prueba del proyecto. 

	- El comando común **Line** corresponde a los parámetros comunes utilizados para ejecutar la prueba del proyecto. Estos parámetros se guardan en el proyecto y se tienen en cuenta cuando el proyecto se comparte a través de SCM. 


Se muestra el comando Line utilizado durante la prueba (concatenación de comando común y personalizado Line s).


Los diferentes elementos del comando Line se pasarán como parámetros al programa actual.

**Recordatorio**: estos parámetros se pueden recuperar mediante la programación con [CommandLine](../WDLang1/3013051.md).
<a name="NOTE3_2"></a>


### Directorio de archivos utilizados para la prueba
<a name="directorio_archivos_utilizados_para_prueba_ELTPARAGRAPHE000064"></a>

Para configurar los ficheros de datos que se utilizan al ejecutar la prueba de aplicación en el editor:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. 1. En la pestaña "Ficheros", seleccione la ubicación de los ficheros de prueba:

	- **Directorio de la configuración actual o directorio actual**: Los archivos de datos utilizados estarán en el subdirectorio correspondiente a la configuración del proyecto (subdirectorio del directorio EXE del proyecto).

	- **Directorio específico**: Los archivos de datos se encontrarán en el directorio especificado.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Directorio de los datos de la aplicación** (recomendado cuando se utiliza la aplicación en Windows XP, Windows Vista y posterior). Este directorio es diferente si los datos son comunes a todos los usuarios de la aplicación o si los datos son específicos de cada usuario.







![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Observación: Grupo de usuarios**
Si su aplicación utiliza el User Groupware, también debe set la ubicación de los archivos de datos del User Groupware. Para realizar esta configuración: 

1. Abrir la ventana de description del User Groupware: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Groupware Usuario".

2. En la pestaña "GPU / Private Store" de la configuración actual, haga clic en "Cambiar" en la opción "User Groupware". 

3. En la ventana que aparece, haga clic en la pestaña "Archivos". 




Para obtener más información, consulte [Groupware Usuario: Opciones y ajustes](../Editeurs/1000021153.md).

### Directorio de tiempo de ejecución
<a name="directorio_tiempo_ejecucion_ELTPARAGRAPHE000121"></a>

WINDEV le permite ejecutar la prueba del proyecto en un directorio diferente al directorio "EXE" del proyecto.. Por lo tanto, la prueba se realiza en condiciones similares a las que se encuentran en los ordenadores de los usuarios finales.

**Para modificar el directorio de ejecución de la aplicación**:

1. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Configurar modo de prueba".

2. En la ventana que aparece, amplíe "Modos avanzados (Windows Vista, UAC, ....)".. 

3. En la pestaña "Directorio de tiempo de ejecución" marque "Iniciar el modo de prueba en el directorio de tiempo de ejecución real".: ".

4. Seleccione el directorio de tiempo de ejecución deseado.

5. Los ficheros de datos que se tendrán en cuenta son los ficheros de datos que se encuentran en los directorios especificados:

	- en la descripción del proyecto. 

	- en el description de el groupware usuario si la aplicación utiliza el groupware usuario.




6. Haga clic en el botón "copy los archivos". Para cada archivo de datos, puede elegir el directorio en el que debe copiarse este archivo:

	- un directorio propuesto por Default:

		- ***&lt;RuntimeDir&gt;&gt;Dirección del tiempo de ejecución***: Directorio de tiempo de ejecución estándar de la aplicación. Este directorio es idéntico al elegido en la pestaña "Runtime directory".

		- ***&lt;DirDataUser&gt;&gt;DirDataUser***: Directorio estándar de los datos de la aplicación para un usuario específico.. En Windows XP/Vista, este directorio tiene el siguiente formato: "C:\\Documentos y ajustes\\&lt;Usuario&gt;\\Datos de la aplicación\\&lt;Nombre de la empresa&gt;\\&lt;Nombre de la aplicación&gt;".

		- ***Datos comunes&gt; &lt;DirCommonData***: Directorio estándar de datos de aplicación para todos los usuarios del mismo ordenador. En Windows XP/Vista, este directorio tiene el siguiente formato: "C:\\Documentos y ajustes\\Todos los usuarios\\Datos de la aplicación\\&lt;Nombre de la empresa&gt;\\&lt;Nombre de la aplicación&gt;".

		- ***GlobalDirUser&gt; &lt;GlobalDirUser&gt;***: Directorio estándar de los datos compartidos por varias aplicaciones para un usuario específico.. En Windows XP/Vista, este directorio tiene el siguiente formato: "C:\\Documentos y ajustes\\&lt;Usuario&gt;\\Datos de aplicación".

		- ***&lt;GlobalDirCommon&gt;***: Directorio estándar de datos compartidos por varias aplicaciones para todos los usuarios del mismo ordenador. En Windows XP/Vista, este directorio tiene el siguiente formato: "C:\\Documentos y ajustes\\Todos los usuarios\\Datos de la aplicación".




- en un directorio seleccionado mediante el selector (opción [...]).

### Manifest view
<a name="manifest_view_ELTPARAGRAPHE000169"></a>

Si Windows Vista (o posterior) es su sistema operativo, WINDEV le permite ejecutar la prueba de su aplicación utilizando un manifiesto. Este manifiesto puede incluirse en el ejecutable de su proyecto al crear el ejecutable.

El uso de un manifiesto permite especificar el nivel de derechos necesarios para ejecutar la aplicación.

Los manifiestos disponibles son los siguientes:

- Ningún manifiesto

- Ningún privilegio

- Máximos privilegios de usuario

- Privilegios de administrador

- Como en la generación. Esta opción le permite reutilizar automáticamente las características de manifiesto especificadas durante la última generación de la aplicación.




Para más detalles sobre los diferentes manifiestos, véase [Elegir un manifiesto para Vista](../Editeurs/2025013.md).

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Otra cuenta de usuario
<a name="otra_cuenta_usuario_ELTPARAGRAPHE000195"></a>

WINDEV le permite ejecutar la prueba del proyecto en un directorio utilizando una cuenta de usuario diferente a la actual.. Esto le permite ejecutar la prueba del proyecto utilizando derechos distintos de los que tiene el usuario actualmente conectado.

De hecho, en la mayoría de los casos, usted es el administrador de su ordenador cuando desarrolla una aplicación.. En la mayoría de los casos, los usuarios de sus aplicaciones son usuarios estándar. Este modo de funcionamiento es aún más cierto en Windows Vista.

WINDEV le permite ejecutar la prueba de sus aplicaciones como un usuario estándar.: puede identificar fácilmente los problemas de sus aplicaciones causados por los derechos y corregirlos si es necesario.

**Observación**: Le aconsejamos que combine esta opción con:

- el uso de un directorio en tiempo de ejecución. Para Windows Vista, el directorio de tiempo de ejecución y el directorio de datos deben corresponder a los estándares de Vista.

- el uso de un manifiesto (si su ordenador de desarrollo se ejecuta en XP).




**Atención**: **La gestión de derechos se tiene en cuenta para los discos que utilizan un sistema de archivos NTFS**. Para los sistemas de ficheros FAT32, no se tiene en cuenta la gestión de derechos.

**Para ejecutar las pruebas utilizando otra cuenta de usuario:**

1. Crear (si es necesario) uno o más usuarios en su ordenador. Se pueden crear nuevos usuarios desde el panel control de Windows ("Cuentas de usuario").  

2. En la pestaña "Otra cuenta de usuario", especifique las características del usuario: Nombre, Contraseña, Dominio.

3. La próxima prueba de aplicación se ejecutará con este usuario.




Observación: No puede utilizar un manifiesto de administrador con un usuario local que pertenezca al grupo de administradores.

### Opciones de depuración
<a name="opciones_depuracion_ELTPARAGRAPHE000225"></a>

WINDEV le permite configurar las opciones de depuración específicas del modo de prueba. 

Es posible: 

- ignorar los temporizadores: Esta opción se utiliza para desactivar los temporizadores durante la depuración.. 

- detener sistemáticamente las excepciones, errores o afirmaciones no verificadas. En este caso, en el modo de depuración: 

	- la ventana de error no se muestra,

	- el depurador se sitúa directamente sobre la correspondiente Line de código. 







Estas opciones también están disponibles en el panel "Depurador" de la cinta, durante el modo de prueba.

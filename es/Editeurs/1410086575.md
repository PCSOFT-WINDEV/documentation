## Presentación
<a name="presentacion_ELTTEXTE000185"></a>
Su aplicación Universal Windows 10 App está lista para ser utilizada. Ahora puede ser publicado en el Windows Store de Microsoft para ser vendido o compartido con la comunidad.. 

Los pasos de implementación son los siguientes: 

1. [Certificación local y criterios de certificación](#NOTE2_1). 

2. [Creación de una cuenta de desarrollador de Microsoft](#NOTE3_1). 

3. [Presentación para el implementación en la tienda](#NOTE4_1). 
	Este paso se utiliza para get los identificadores requeridos por la generación.

## Certificación local
<a name="certificacion_local_ELTTEXTE000209"></a>
Antes de cualquier implementación, la aplicación debe ser validada en el ordenador de desarrollo con el "Windows App Cert Kit". 

Este kit puede ser descargado de la siguiente Address: [http://go.microsoft.com/fwlink/?LinkID=263703](http://go.microsoft.com/fwlink/?LinkID=263703). 

**La certificación local de la aplicación se realiza en varios pasos**: 

1. Generar la aplicación en WINDEV o WINDEV Mobile. Instalar esta aplicación localmente en el ordenador de desarrollo. 

2. Desde la pantalla de inicio, inicie "Windows App Cert Kit".. 

3. En la asistente, seleccione "Validar una aplicación de Windows Store". 

4. Elija su aplicación. 

5. Haga clic en el botón "Siguiente" y espere a que finalice la prueba.. El ordenador no debe utilizarse durante la validación. 

6. Al final de la asistente, guarde el estado reporte en un archivo. 

7. Si la asistente muestra "ÉXITO", puede enviar la solicitud a la Tienda. De lo contrario, el reporte indicará los problemas que se han producido.

## Creación de una cuenta de desarrollador de Microsoft
<a name="creacion_una_cuenta_desarrollador_microsoft_ELTTEXTE000239"></a>
Se requiere una cuenta de desarrollador de Microsoft para presentar una solicitud.. 

Existen dos tipos de cuentas disponibles, individuales o de empresa: 

- Los desarrolladores individuales sólo pueden enviar aplicaciones de Windows Store Apps. 

- Los desarrolladores de la empresa pueden enviar aplicaciones de Windows Store Apps así como aplicaciones de escritorio.. 




El sitio de Microsoft presenta las diferencias entre los dos tipos de cuentas. 

**Para crear una cuenta de desarrollador**: 

1. Abrir el centro de desarrollo de Windows Store Apps: [http://go.microsoft.com/fwlink/?LinkID=245786](http://go.microsoft.com/fwlink/?LinkID=245786). 

2. Conéctese con su cuenta de Microsoft (cuenta de Live o Hotmail). Si no tiene una cuenta de Microsoft, cree una. 

3. Siga los pasos de la asistente para crear una cuenta de desarrollador o individual. 

	- Elija el tipo de cuenta: persona física o jurídica. 
			**Atención: la elección del tipo de cuenta es permanente**. 

	- Especifique los parámetros de la cuenta. Además de los datos personales, debe elegir el nombre para mostrar del editor. Este es el nombre del editor que se mostrará en la tienda para sus aplicaciones.. Debe especificar un nombre para el que se le han concedido derechos.. 

	- Aceptar el contrato. La asistente muestra el costo de la inscripción. 

	- Confirmar la compra y pagar en línea. 




4. Se crea la cuenta de desarrollador de Microsoft. Tiene la posibilidad de acceder a la cuadro de mando para presentar una solicitud. 




**Importante**: Si desea presentar solicitudes basadas en tarifas, debe configurar una cuenta de dividendos (cuenta en la que se abonarán sus ingresos por ventas)..

## Presentación para un implementación en la tienda
<a name="presentacion_para_implementacion_tienda_ELTTEXTE000263"></a>


### Recuperación de los identificadores requeridos por la generación
<a name="recuperacion_los_identificadores_requeridos_por_generacion_ELTPARAGRAPHE000099"></a>

**Para recuperar los identificadores de una aplicación**: 

1. Mostrar la cuadro de mando de sus aplicaciones. La cuadro de mando de sus solicitudes está disponible en la siguiente Address: [https://developer.microsoft.com/fr-fr/cuadro de mando/apps/overview](https://developer.microsoft.com/fr-fr/dashboard/apps/overview). 

2. Cree una nueva aplicación si es necesario y reserve un nombre de producto. Este nombre es un nombre único. Corresponde al nombre utilizado para su aplicación en la Tienda. Este nombre tendrá que ser usado en la generación asistente de WINDEV o WINDEV Mobile. 

3. Cuando se crea la aplicación, usted tiene la habilidad de encontrar los diferentes identificadores requeridos para generar la aplicación en WINDEV Mobile.. 

	- En la cuadro de mando, se muestran las diferentes aplicaciones. 

	- Seleccione su aplicación. En el menú de la izquierda, en la sección "Gestión de aplicaciones", haga clic en "Identidad de las aplicaciones".. 

	- Se visualizan los diferentes identificadores. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=W10_Identite_app.gif&type=thumb)








En la asistente para la generación de WINDEV Mobile: 

- El nombre completo del editor corresponde a "Package/Properties/FullEditorName".. 

- El identificador del editor corresponde a "Paquete/Identidad/Editor".

- El nombre de la aplicación reservado en Windows Store corresponde al nombre único de la aplicación. 

- El nombre completo del paquete de aplicación corresponde a "Paquete/Identidad/Nombre".. 







### Envío de una solicitud
<a name="envio_una_solicitud_ELTPARAGRAPHE000124"></a>

Para enviar una solicitud a la Tienda: 

1. Haga clic en el botón "Iniciar el envío. La presentación de una solicitud se realiza en varios pasos. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=W10_Soumettre_store.gif&type=thumb)
 El paso "Paquetes" requiere proporcionar los paquetes generados por WINDEV Mobile. 

2. En WINDEV o WINDEV Mobile, genere los paquetes para la tienda. En la generación asistente, seleccione "Generación para la tienda de Windows". Para obtener más información, consulte [Generación de una aplicación de Windows 10](../Editeurs/1410086577.md). 

3. En la asistente, introduzca los diferentes identificadores (ver párrafo anterior). 

4. El archivo &lt;AppName&gt;.appxbundle se genera al final de asistente. Este archivo debe ser cargado en el sitio web de envío (sección"Paquetes"). 

5. Introduzca la información solicitada para la presentación. 

6. Para finalizar la implementación, haga clic en el botón "Enviar a la tienda de Windows".. El estado de certificación de reporte le será enviado por correo electrónico..

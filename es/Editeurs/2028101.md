
## Despliegue de un sitio en un servidor de prueba PC SOFT
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000217"></a>
PC SOFT le propone alojar sus sitios de prueba. La instalación se realiza fácilmente, directamente desde el entorno. 

Una vez que su sitio está desarrollado, usted tiene la capacidad de ejecutar su prueba directamente. 

**Observaciones**: 

- El sitio debe estar desarrollado en WEBDEV dinámico o en modo AWP. 

- El sitio debe utilizar una base de datos HFSQL Classic o HFSQL Client/Server. 

- El servidor utilizado es un servidor Windows. No se puede desplegar ningún sitio Linux. 

- Tiene la capacidad de implementar Webservices. 

- No se tienen en cuenta los siguientes tipos de centros: 

	- Los sitios desarrollados en PHP.

	- Los sitios SaaS. 

	- Los sitios estáticos. 







**Limitación**: Se asignan 100 conexiones simultáneas a cada cuenta de desarrollador. La distribución del número de conexiones por emplazamiento no debe superar este límite.

## Etapas a seguir
<a name="etapas_seguir_ELTTEXTE000247"></a>
Los diferentes pasos para instalar un sitio WEBDEV dinámico en el servidor de alojamiento de prueba de PC SOFT son los siguientes:

1. [Creación de una cuenta de desarrollador si es necesario](#NOTE3_1).

2. [Creación de una cuenta de hosting si es necesario](#NOTE4_1).

3. [Introducción de los parámetros del sitio](#NOTE5_1).

4. [Selección de los archivos a instalar](../Editeurs/2028058.md). 

5. [Modificación automática](../Editeurs/2028048.md). 

6. [Parámetros del sitio](../Editeurs/2028051.md). 

7. Transferencia de archivos y instalación. 
	Los mensajes de error pueden ser mostrados en diferentes pasos de la instalación remota. Para obtener más información, consulte [Errores durante una instalación por FTP](../Editeurs/2028057.md).

8. Cuando la instalación se completa, la asistente provee el comando Line usado para iniciar el sitio. Este Line puede ser copiado (si es necesario) en el portapapeles a través de el menú contextual.

## Creación de una cuenta de desarrollador
<a name="creacion_una_cuenta_desarrollador_ELTTEXTE000271"></a>


### Cuenta del desarrollador
<a name="cuenta_del_desarrollador_ELTPARAGRAPHE000070"></a>

Se puede crear una cuenta de desarrollador: 

- para participar en los foros de PC SOFT.

- para escribir anotaciones en la documentación online.

- para desplegar un sitio en el servidor de alojamiento de prueba de PC SOFT. También se requiere una cuenta de hosting para la prueba. 




Si ya tiene una cuenta de desarrollador, esta cuenta puede ser reutilizada para instalar sus sitios.


### Elementos necesarios
<a name="elementos_necesarios_ELTPARAGRAPHE000081"></a>

**Los siguientes elementos son necesarios para crear la cuenta de desarrollador**:

1. Un correo electrónico válido Address. 
	**Atención**: Al crear la cuenta de desarrollador, el correo electrónico Address suministrado puede ser usado una sola vez. El mismo correo electrónico Address no puede ser usado para crear varias cuentas de desarrollador.

2. El número de serie de su producto WEBDEV. 




Al validar la creación de la cuenta de desarrollador, se envía una confirmación Email con un enlace que se utiliza para habilitar la cuenta.

## Creación de una cuenta de alojamiento de prueba
<a name="creacion_una_cuenta_alojamiento_prueba_ELTTEXTE000301"></a>


### Prueba de cuenta de hosting
<a name="prueba_cuenta_hosting_ELTPARAGRAPHE000097"></a>

Se puede crear una cuenta de alojamiento de prueba para implementar un sitio en el servidor de alojamiento de prueba de PC SOFT. 

Si ya tiene una cuenta de prueba de hosting, esta cuenta puede ser reutilizada para desplegar sus sitios.


### Elementos necesarios
<a name="elementos_necesarios_ELTPARAGRAPHE000104"></a>

**Los siguientes elementos son necesarios para crear la cuenta de hosting**:

1. Una cuenta de desarrollador de PCSOFT.

2. Un nombre de dominio. Este nombre de dominio será único. Corresponde a la raíz de todos los sitios que serán desplegados..
	Por ejemplo: la raíz dada por el dominio PCSOFT será http:\\_ASLASH_ pcsoft.webdev-test.com\\MyTestSite
	**Atención**: El nombre de dominio debe contener al menos 5 caracteres.




Al validar la creación de la cuenta de alojamiento, se envía una confirmación Email.




### Recursos associated con una cuenta de alojamiento
<a name="recursos_associated_con_una_cuenta_alojamiento_ELTPARAGRAPHE000119"></a>

Una vez creada la cuenta, usted tiene la posibilidad de acceder a los siguientes recursos:

- Un acceso FTP personalizado. 
	El acceso por FTP a su dominio se realiza con: 

	- el nombre de dominio completo ("pcsoft.webdev-test.com", por ejemplo).

	- el nombre de dominio ("pcsoft" en nuestro ejemplo).

	- la contraseña que se especificó al crear la cuenta.
			Esto le permite enviar archivos por FTP a su sitio.




- Una herramienta web segura para gestionar sus sitios ([https://www.webdev-test.com/WDAdminWeb](https://www.webdev-test.com/WDAdminWeb)). Debe introducir su nombre de dominio y su contraseña para conectarse. Esto le permite administrar los diferentes sitios instalados:

	- modificar el número máximo de conexiones autorizadas

	- borrar un sitio

	- modificar el tiempo de espera de una desconexión de automatic

	- habilitar/deshabilitar la gestión de LOGs para las estadísticas de tráfico

	- ...
			Estas opciones son idénticas a las disponibles para un verdadero implementación.

## Parámetros del sitio
<a name="parametros_del_sitio_ELTTEXTE000337"></a>
Una vez creada la cuenta de desarrollador y la cuenta de hospedaje, debe especificar en el WEBDEV asistente: 

- el nombre de dominio,

- la contraseña,


y validar para instalar el sitio. 

El botón "Parámetros avanzados" se utiliza para definir los parámetros de implementación: 

- En la pestaña "General", especifique:

	- el nombre de implementación del sitio (el nombre del proyecto por Default).

	- el nombre del dominio associated para gestionar un archivo de mapa de sitio.

	- el nombre de la carpeta que contiene los archivos de datos (el nombre del proyecto de Default).

	- el nombre del perfil de implementación (especificado por defecto) para guardar los parámetros de instalación.




- En la pestaña "Avanzado", especifique: 

	- los parámetros FTP,

	- los parámetros del proxy HTTP/HTTPS.

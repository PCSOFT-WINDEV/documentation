## Presentación
<a name="presentacion_ELTTEXTE000215"></a>
El **Feedback** Component es un Component proporcionado con WINDEV. Esta Component permite a los usuarios finales de sus aplicaciones enviar:

- sugerencias sobre la aplicación.

- problemas que ocurrieron en la aplicación.

- reglas de negocio.




Los diferentes elementos se reciben a través de los centros de control.

Si el usuario final no está conectado a la misma red, los elementos son enviados por Email. El usuario se identifica con el nombre especificado en la Email.

Si el usuario está conectado a la misma red (es decir, si puede acceder a los datos de los Centros control a través de la red o del ADSL), los elementos enviados se añaden automáticamente a los datos de los Centros control.

**Observación**: Si su aplicación utiliza el menú de automatic ("?"), el componente de Feedback es aliado de automatic en su aplicación cuando selecciona "Enviar una sugerencia, un problema o una regla de negocio".

**Aplicación multilingüe**: El Feedback Component está disponible en francés y en inglés. Para los demás idiomas, los asistentes de la Component se mostrarán en inglés por Default. Sin embargo, los diferentes mensajes que se encuentran en el Feedback Component pueden ser traducidos. Para obtener más información, consulte [Traducción de los componentes](../Editeurs/2014014.md).

El **Feedback** Component, si es utilizado por la aplicación, será llamado automáticamente desde la ventana de error de WLanguage y permitirá transmitir un archivo de volcado a la aplicación actual.

## Enviar una sugerencia
<a name="enviar_una_sugerencia_ELTTEXTE000239"></a>


### Enviar una sugerencia, un problema o una regla de negocio con el Feedback Component
<a name="enviar_una_sugerencia_problema_una_regla_negocio_con_feedback_component_ELTPARAGRAPHE000039"></a>

El Feedback Component permite al usuario final enviar fácilmente diversas informaciones. Para ello:

1. Iniciar la retroalimentación Component asistente:

	- a través del menú de automatic ("... Enviar una sugerencia, un problema o una norma empresarial").

	- a través de una opción de su aplicación que gestiona el Feedback Component.




2. En la asistente, elija el tipo de solicitud a realizar: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=MesIncidents2.gif&type=thumb)


3. Introduzca los datos personales (opcional).

4. indicar la información sobre el problema, la sugerencia o la regla de negocio.

5. Inserte (si es necesario) una captura de pantalla (con comentarios).




Se visualiza un resumen de la solicitud. Esta solicitud se puede grabar y/o imprimir.

Durante la validación, la solicitud se envía directamente:

- Si el usuario está en modo offline (la base de datos de los Centros de Control no es accesible), la solicitud es enviada por Email.

- Si el usuario está en modo online (la base de datos de los Centros control si es accesible a través de la red o INTERNET/ADSL), la solicitud se guarda directamente en los archivos de los Centros control.



<a name="NOTE2_2"></a>


### Mostrar las solicitudes salientes
<a name="mostrar_las_solicitudes_salientes_ELTPARAGRAPHE000063"></a>

Si el menú de automatic fue incluido en la aplicación WINDEV, el usuario puede ver fácilmente las solicitudes ya enviadas a través de "? .. Mostrar las solicitudes salientes".

Si el usuario está en modo offline, se utiliza el identificador utilizado para enviar la solicitud (nombre especificado en la Email).

Si el usuario se encuentra en modo online, se utiliza el identificador utilizado para enviar la solicitud (nombre especificado en la asistente).

## Configuración de la retroalimentación Component
<a name="configuracion_retroalimentacion_component_ELTTEXTE000271"></a>


### Si los usuarios de la aplicación están en modo offline
<a name="los_usuarios_aplicacion_estan_modo_offline_ELTPARAGRAPHE000076"></a>

Si el componente de Feedback se incluyó a través del menú de automatic ("?"), tiene la posibilidad de especificar el correo electrónico que Address utilizó para enviar las sugerencias:

- en la asistente para crear/modificar el menú de la automatic.

- en el código de la opción del menú de automatic.

- al crear el ejecutable.




**Consejo**: especificar un correo electrónico Address, esta Address se utilizará automáticamente si los usuarios no están conectados (red caída, ...).
<a name="NOTE3_2"></a>


### Si los usuarios de la aplicación están en modo interactivo
<a name="los_usuarios_aplicacion_estan_modo_interactivo_ELTPARAGRAPHE000089"></a>

- **Base de datos de los centros control en modo HFSQL Classic**: No se requiere ninguna configuración específica.

- **Base de datos de los centros control en modo HFSQL Client/Server**: Si el componente Feedback se ha incluido a través del menú automatic ("?"), el usuario que se tendrá en cuenta para la conexión a la base de datos es el usuario actual. Se recomienda:

	- crear un colaborador "invitado" en la Centro de Seguimiento de Proyectos (sin derechos de administrador). Las sugerencias, reglas de negocio (por ejemplo) serán enviadas a través de la conexión de este usuario.

	- utilizar los parámetros del colaborador "Invitado" en el código de "?... Enviar sugerencias, ...".

		1. Mostrar la ventana que utiliza el menú de automatic. En la pestaña "Ventana", en el grupo "Barras y menús", despliegue "Menú principal" y seleccione "Abrir menú '?'".

		2. Editar el código de la opción "Enviar una sugerencia, un incidente, una regla de negocio".

		3. Modificar la Line del código utilizado para especificar la ruta de los archivos de datos. Utilice la siguiente Line de código:
						
			```wl
			pFeedback.Configure(fbPathCCData, ...
					"hfcs://<user>:<password>@<server>:<port>/<database>")
			```

						En este código:

			- &lt;usuario&gt;: Nombre de un usuario del Centro control.

			- &lt;contraseña&gt;: Contraseña associated con el usuario.

			- &lt;servidor&gt;: IP Address o nombre del servidor donde está instalada la base de datos de los Centros control.

			- &lt;Puerto&gt;&gt;&gt; de: Puerto del servidor.

			- Base de datos &lt;base de datos: Nombre de la base de datos HFSQL Client/Server correspondiente a los archivos de los Centros control.
										Observación: si actualiza el menú automatic de su aplicación, esta línea de código será automatic aliado actualizado con el nombre del desarrollador actual.
										Si la retroalimentación Component se incluyó independientemente del menú automatic ("?"), consulte la documentación sobre el Component.

## Recuperación y procesamiento de la retroalimentación del usuario
<a name="recuperacion_procesamiento_retroalimentacion_del_usuario_ELTTEXTE000307"></a>
Los Centros de Control permiten recuperar los errores, sugerencias y reglas de negocio enviados por los usuarios a través del Feedback Component. Para obtener más información, consulte [¿Cómo recuperar los errores y sugerencias hechas por los usuarios?](../CCSuivi/3540719.md).

## Comentarios Component: las diferentes funciones de Component
<a name="comentarios_component_las_diferentes_funciones_component_ELTTEXTE000331"></a>
El Feedback Component propone las siguientes funciones para personalizar su uso. Para más detalles, consulte la documentación completa del Feedback Component ("description" en el menú contextual del Component).


|   |   |
| --- | --- |
| **AñadirSolicitar** | Se utiliza para añadir una solicitud al carrete de Centro de Seguimiento de Proyectos. |
| **Configurar** | Permite configurar las opciones de Feedback Component. |
| **ErrorMessage** | Permite recuperar el mensaje del último error que se ha producido. |
| **InEmailTestMode** | Forzar a una aplicación a utilizar la función de envío de correos electrónicos incluso si la base de datos de los Centros control es accesible (envío de correos electrónicos en modo online). |
| **NuevaSolicitud** | Se utiliza para abrir la asistente para crear una sugerencia, un incidente o una regla de negocios. <br><br>Para funcionar en modo desconectado, se debe configurar la dirección de correo electrónico para recibir los mensajes de los usuarios (con la función **Configure**, o introduciendo el control Dirección de correo electrónico al crear el ejecutable). |

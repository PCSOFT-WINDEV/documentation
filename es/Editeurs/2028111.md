
## Instalación de un sitio WEBDEV con actualización sin tiempo de inactividad
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000145"></a>
A partir de la versión 24, es posible actualizar un sitio de tipo Session o AWP sin desconectar a los usuarios que están conectados al sitio.

La actualización de una sesión WEBDEV o de un sitio AWP no afecta a los usuarios conectados. Los usuarios conectados permanecen en la versión antigua. Sólo tendrán acceso a la nueva versión del sitio la próxima vez que se conecten.. 

Este modo implementación permite desplegar sitios de sesión sin tiempo de inactividad.. 
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=D%E9ploiement_sans_deconnexion%20-%20HC%20N%B0002.gif)


ATENCIÓN: este modo **no se recomienda** para: 

- sitios que contengan páginas de Active WEBDEV sin contexto.

- sitios con páginas estáticas. 

- sitios PHP. 


Al utilizar este modo implementación, las URL cambian cada vez que se despliega una nueva actualización, lo que afecta al SEO de estas páginas e impide que los usuarios las recuerden. 

Observación: Esta función sólo está disponible para sitios desplegados con WEBDEV versión 24 y posteriores..

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000169"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000033"></a>

Para evitar desconectar a los usuarios, el servidor mantiene varias versiones del sitio. 

Por Default, 4 versiones anteriores se mantienen en el servidor. 




### ¿Cómo proceder?
<a name="¿como_proceder_ELTPARAGRAPHE000041"></a>

Para instalar un sitio sin desconectar a los usuarios durante la actualización: 

1. **Caso de un primer sitio implementación**: 

	- Despliegue su sitio por primera vez. 
			En el plan de actualización sin tiempo de inactividad, seleccione la opción "Desplegar sin desconectar a los usuarios".. 
			Observación: La instalación es similar a la del modo estándar (como el sitio nunca se ha desplegado, no hay usuarios conectados).

	- En futuras actualizaciones, el sitio se instalará automáticamente sin tener que preocuparse de si los usuarios están conectados o no.




2. **Caso de un sitio ya desplegado**: Si el sitio ya estaba instalado con el método "clásico: 

	- Durante la primera instalación utilizando la opción "Desplegar sin desconectar usuarios", TODOS los usuarios serán desconectados.. 

	- Durante las siguientes instalaciones/despliegues, los usuarios no serán desconectados. 


Este nuevo modo se aplica a partir de la segunda instalación.







### Importante: Gestión de directorios y archivos de datos
<a name="importante_gestion_directorios_archivos_datos_ELTPARAGRAPHE000061"></a>

Si utiliza una instalación sin desconectar a los usuarios durante la actualización: 

- utilizar [fDataDirUser](../WDLang1/3036071.md) para manipular o acceder a los archivos de datos del sitio. Cuando escriba, no utilice [fWebDir](../WDLang1/3036052.md) o [fExeDir](../WDLang1/3036049.md) porque los datos no se sincronizarán entre las diferentes actualizaciones del sitio. Sólo [fDataDirUser](../WDLang1/3036071.md) garantiza el uso de un único directorio compartido entre las diferentes versiones del sitio. 

- para manipular el directorio _WEB, utilice [fWebDir](../WDLang1/3036052.md) y/o [FolderWeb](../WDLang2/3012015.md). No utilice el nombre del directorio directamente (_WEB no debe aparecer en su código). 






### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000092"></a>

**Atención**: No se puede volver a una instalación estándar después de una instalación sin tiempo de inactividad.. Si desea reinstalar su sitio utilizando el método "clásico", debe: 

1. Desinstale el sitio.

2. Eliminar del servidor web (IIS o Apache) los directorios virtuales que se han creado para este sitio.

3. Reinstalar el sitio en modo clásico.


## Configuración de la replicación universal asistida
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000146"></a>
Si su aplicación utiliza la replicación universal asistida a través de un archivo .WER, la creación de instalación asistente propone configurar el instalación para instalar las diferentes herramientas necesarias para este tipo de replicación.

Los diferentes pasos de esta configuración son los siguientes:

- Ubicación del archivo.WER en manipular.

- Método de replicación.

- Definition del modo conectado.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Parámetros de la replicación universal asistida
<a name="parametros_replicacion_universal_asistida_ELTTEXTE000170"></a>


### Archivo.WER
<a name="archivower_ELTPARAGRAPHE000024"></a>

El primer paso a la hora de configurar la replicación universal asistida consiste en dar a conocer la ruta del fichero.WER.

El contenido del archivo .WER puede ser configurado y modificado por [ReplicEdit](../ReplicEdit/3526500.md) en la creación de instalación asistente.
<a name="NOTE2_2"></a>


### Método de replicación
<a name="metodo_replicacion_ELTPARAGRAPHE000036"></a>

Al preparar el programa instalación, puede especificar el método de replicación que se utilizará:

- **Replication a través de un servidor de replicación**. Este método se recomienda cuando la replicación se realiza a través de una conexión INTERNET. Si se selecciona esta opción, el siguiente paso de asistente permite configurar los parámetros del servidor de replicación: servidor Address, nombre de usuario y contraseña. También tienes la capacidad de hacerlo:

	- publicar los parámetros de replicación en el servidor (instalación con actualización de automatic solamente): si esta opción está seleccionada, las aplicaciones podrán acceder directamente a la replicación.

	- permitir que estos parámetros se modifiquen durante la instalación: el usuario de la aplicación podrá modificar los parámetros del servidor de replicación.

	- utilizar estos valores incluso si ya han sido personalizados durante una instalación anterior.

	- instalar un servidor de replicación. En este caso, la instalación de la aplicación propondrá instalar el servidor de replicación en la el equipo seleccionada por el usuario.
			**Atención**: para realizar esta instalación, el programa instalación debe ser suministrado con el paquete instalación del servidor de replicación. Con el botón "Advanced" se selecciona el paquete necesario. El último paquete de instalación disponible también se puede descargar de la página web de PC SOFT.




- **Replication con acceso directo a la base de datos principal**. Este método se recomienda, por ejemplo, cuando se utiliza una aplicación en un ordenador portátil, con una conexión limitada a la red local (por ejemplo, un vendedor que visita la sede central una vez a la semana).. Si se selecciona esta opción, se deben especificar los parámetros de la base de datos maestra:

	- Utilizar las conexiones definidas en el análisis: Los parámetros de las conexiones definidas en el análisis se utilizarán para acceder a la base de datos maestra.

	- Utilice las conexiones definidas en el archivo.WER: Los parámetros de las conexiones definidas en el archivo.WER se utilizarán para acceder a la base de datos maestra.

	- Definir los parámetros de conexión ahora: Los parámetros de conexión se definen inmediatamente en el siguiente paso de la asistente.

	- Solicitar los parámetros de conexión durante la instalación: Los parámetros de las conexiones son definidos por la persona que instala la aplicación.




- **Elija el método de replicación durante la instalación**. Si se selecciona este método, se deben especificar los parámetros de la base de datos maestra (igual que con "Replicación con acceso directo a la base de datos maestra").



<a name="NOTE2_3"></a>


### Definition del modo conectado
<a name="definition_del_modo_conectado_ELTPARAGRAPHE000059"></a>

Cuando se crea el programa instalación, se tiene la posibilidad de especificar el modo de funcionamiento de la aplicación en modo conectado.

De hecho, cuando el usuario tiene acceso a la base de datos maestra, el acceso directo a la base de datos maestra puede utilizarse automáticamente en lugar de la replicación ("Allow the application to operate in connected mode").. Si se selecciona este modo de funcionamiento, se puede solicitar la sincronización de datos de automatic al iniciar y detener la aplicación. Este modo le permite trabajar en modo offline con la base de datos más actualizada.

Estas opciones pueden ser modificadas o no al instalar la aplicación. También puede forzar el uso de estas opciones incluso si ya han sido personalizadas..

## Archivos instalados para la replicación
<a name="archivos_instalados_para_replicacion_ELTTEXTE000206"></a>
Por Default, los siguientes archivos se instalan en el ordenador del usuario:

|   | Directorio de configuración de Default |
| --- | --- |
| Archivos de datos locales (.fic,.ndx,.mmo) | Documentos y ajustes\\Todos los usuarios\\Datos de la aplicación\\&lt;Nombre de la aplicación&gt; |
| Archivos que permiten realizar la replicación | Documentos y ajustes\\Todos los usuarios\\Datos de la aplicación\\_ASLASH_ PC SOFT\\RPL\\&lt;Nombre de la aplicación&gt;\\&lt;Nombre de la aplicación&gt; &lt;Nombre de la aplicación |
| Configuración de la instalación | Documentos y ajustes\\&lt;Nombre de usuario&gt;\\Datos de la aplicación\\WDSETUP\\&lt;Nombre de la aplicación&gt; |
| Aplicación (Exe y WX) | Fichero de programa\\&lt;Nombre de la aplicación &gt;&gt;. |
| ReplicSynchro y ReplicAdmin | Archivos de programa\\Archivos comunes\\PC SOFT\\28.0\\ReplicSynchro |

**Observación**: El usuario final puede iniciar ReplicSynchro a través de "Start .... PC SOFT tools ... ReplicSynchro".

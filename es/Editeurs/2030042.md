


## Crear un proyecto WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000235"></a>
Un proyecto es un conjunto lógico de elementos y procesos cuyo propósito es lograr un objetivo determinado. El proyecto gestiona:

- las relaciones entre los elementos (páginas, reportes, etc.)

- las relaciones entre los elementos y los archivos de datos (descritos en el análisis relacionado con el proyecto).




Un proyecto solo puede estar relacionado con un análisis.

Un mismo análisis puede asociarse a varios proyectos.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000259"></a>


### Crear un proyecto WEBDEV
<a name="crear_proyecto_webdev_ELTPARAGRAPHE000025"></a>

Para crear un proyecto WEBDEV:

1. Inicie el asistente de creación de proyectos mediante una de las siguientes opciones: 

	- Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. En la ventana que aparece, haga clic en "Proyecto". El asistente de creación de proyectos se abre.

	- En la página de inicio de WEBDEV, haga clic en "Crear un proyecto" y elija el tipo de proyecto que desea crear. 


 Observaciones: 

- Las etapas necesarias para crear el proyecto pueden variar en función del tipo de proyecto elegido.

- El nuevo proyecto contendrá una configuración específica para ese tipo de proyecto. Las características de la configuración del proyecto se pueden modificar en cualquier momento. El mismo proyecto se puede asociar a varias configuraciones de proyecto. Para obtener más información, consulte [Configuración de proyecto](../Editeurs/9000030.md). 

2. Seleccione "Sitio" y vaya a la siguiente etapa. 

3. Especifique el modo de creación del sitio:

	- [Proyecto en blanco](#NOTE2_2). 

	- [Proyecto basado en la metodología RAD. En este caso, seleccione el modelo deseado](#NOTE2_3).







<a name="NOTE2_2"></a>


### Sitio web basado en un proyecto en blanco
<a name="sitio_web_basado_proyecto_blanco_ELTPARAGRAPHE000055"></a>

**Si desea crear un proyecto en blanco**: 

1. Especifique el tipo de sitio que desea crear:

	- Sitio WEBDEV. En este caso, indique si el sitio está: 

		- En modo Sesión (Classic): Este tipo de sitio gestiona un contexto automático. Las páginas no se optimizan en los motores de búsqueda.
						Este tipo de sitio se recomienda para sitios de Intranet. 

		- En modo AWP: Este tipo de sitio propone un contexto que debe ser gestionado mediante programación. Las páginas se optimizan en los motores de búsqueda. 
						Este tipo de sitio se recomienda para sitios de Internet. 




- Sitio PHP (requiere un motor PHP).

- Sitio estático (funcionalidades limitadas, sin código Servidor...).  

2. Pase a la etapa siguiente.

3. Especifique las opciones del proyecto:

	- ***El nombre y la ubicación***. Estas opciones no se pueden modificar. En WEBDEV, el proyecto corresponde a un archivo ".WWP". Este archivo se puede abrir con WINDEV y WINDEV Mobile. Para obtener más información, consulte [Proyecto común](../Editeurs/2030049.md). Todos los objetos asociados al proyecto se crearán en el directorio especificado ("C:\\My Projects\\&lt;Nombre del proyecto&gt;" de forma predeterminada).

	- ***La descripción*** que resume el propósito del proyecto. Esta descripción se utiliza en la documentación del proyecto.




4. Pase a la etapa siguiente.

5. Especifique los documentos asociados al proyecto. Estos documentos pueden corresponder a archivos de texto, imágenes, documentos escaneados, ... Para obtener más información, consulte [Asociar documentos al proyecto](../Editeurs/9000092.md). 
	Pase a la etapa siguiente.

6. Indique si el proyecto debe incluirse en el [Administrador de Código Fuente (SCM)](../Editeurs/2038001.md). El Administrador de Código Fuente simplifica el trabajo en equipo y también puede ser útil para el trabajo individual. Pase a la etapa siguiente.

7. Indique si desea utilizar el estilo de código. El estilo de código corresponde a un estándar de programación que puede aplicarse a toda la aplicación. Este estándar permite añadir prefijos a los diferentes elementos del proyecto, las variables, etc, ... Para obtener más información, consulte [Estilo de código](../Editeurs/9000074.md). Pase a la etapa siguiente.

8. Elija el tema y la paleta de colores utilizada para definir el tema gráfico (puede probar las diferentes opciones en una vista previa). Este tema gráfico se propondrá de forma predeterminada al crear una página o reporte y se aplicará a los nuevos controles. Esta opción se puede modificar en la ventana de descripción del proyecto (pestaña "Estilo"). 
	Pase a la etapa siguiente. 
	

9. Seleccione los ***idiomas del proyecto***. Estos idiomas se propondrán de forma predeterminada cuando se deba traducir la opción de un elemento (control, página, reporte , etc.). Seleccione el **idioma predeterminado en tiempo de ejecución** (opción "Idioma predeterminado" del menú contextual de la lista de idiomas). Este idioma se utilizará para ver el proyecto en tiempo de ejecución. 
	Pase a la etapa siguiente.

10. Para sitios WEBDEV o PHP, especifique si el proyecto utilizará o no **una base de datos**.
	Si su proyecto utiliza una base de datos, puede ser una base de datos existente o una nueva base de datos. Si utiliza una nueva base de datos, un asistente de creación de bases de datos iniciará automáticamente cuando haya creado el proyecto.

11. Valide. El proyecto que se acaba de crear permanece abierto.




**Observación**: Si ha elegido:

- usar el SCM, [el asistente para agregar proyectos a un repositorio](../Editeurs/2038002.md) se abre automáticamente.

- crear una base de datos, [el asistente correspondiente](../Editeurs/2011011.md) se abre automáticamente.

### Sitio web basado en un ejemplo (Aplicación RAD)
<a name="sitio_web_basado_ejemplo_aplicacion_rad_ELTPARAGRAPHE000128"></a>

Si ha elegido un sitio de Internet basado en un ejemplo: 

1. Seleccione el ejemplo que se utilizará para crear el proyecto. 

2. Especifique las opciones del proyecto:

	- ***El nombre y la ubicación***. Estas opciones no se pueden modificar. En WEBDEV, el proyecto corresponde a un archivo ".WWP". Este archivo se puede abrir con WINDEV y WINDEV Mobile. Para obtener más información, consulte [Proyecto común](../Editeurs/2030049.md). Todos los objetos asociados al proyecto se crearán en el directorio especificado ("C:\\My Projects\\&lt;Nombre del proyecto&gt;" de forma predeterminada).

	- ***La descripción*** que resume el propósito del proyecto. Esta descripción se utiliza en la documentación del proyecto.




3. Pase a la etapa siguiente.

4. Especifique los documentos asociados al proyecto. Estos documentos pueden corresponder a archivos de texto, imágenes, documentos escaneados, ... Para obtener más información, consulte [Asociar documentos al proyecto](../Editeurs/9000092.md). 
	Pase a la etapa siguiente.

5. Indique si el proyecto debe incluirse en el [Administrador de Código Fuente (SCM)](../Editeurs/2038001.md). El Administrador de Código Fuente simplifica el trabajo en equipo y también puede ser útil para el trabajo individual. Pase a la etapa siguiente.

6. Valide. El proyecto que se acaba de crear permanece abierto.




**Observaciones**: 

- Si eligió usar el SCM, [el asistente para agregar proyectos a un repositorio](../Editeurs/2038002.md) se abre automáticamente.

- Crear un proyecto a partir de un ejemplo le permite crear un nuevo proyecto con todas las páginas, reportes y códigos del ejemplo. Solo debe personalizar el ejemplo según sus necesidades. 
	Este método de creación tiene varias ventajas: 

	- Puede elegir el nombre del proyecto. 

	- Trabaja con una versión independiente del ejemplo: no hay riesgo de que se sobrescriban las modificaciones. Si desea seguir las evoluciones del ejemplo, simplemente compare su proyecto con el del ejemplo. Para obtener más información, consulte [Comparador de proyectos](../Editeurs/2030024.md). 

	- Puede crear o eliminar elementos sin modificar el ejemplo.

## Notas

### Archivo del proyecto
<a name="archivo_del_proyecto_ELTPARAGRAPHE000175"></a>

En WEBDEV, el proyecto corresponde a un archivo ".WWP". Este archivo contiene:

- El código de inicialización del proyecto, que se ejecuta al abrir el proyecto.

- La lista de objetos relacionados (páginas, reportes, etc.). La secuencia de estos objetos se puede ver en el editor de proyectos.

- El nombre de la página de inicio y de la primera página dinámica del proyecto. Estas páginas se abren al ejecutar el proyecto o cuando se ejecuta una prueba del proyecto.

- El nombre del análisis asociado al proyecto (si existe) y su contraseña. Los archivos de datos descritos en el análisis pueden ser manipulados por los elementos del proyecto (ventanas, páginas, reportes, etc.).

- El nombre del modelo UML asociado al proyecto (si existe).

- La hoja de estilo asociada al proyecto (si existe). La hoja de estilo contiene todos los estilos utilizados por los elementos del proyecto.

- El modelo predefinido asociado al proyecto (si existe este modelo).

- El directorio de la lista de tareas a realizar en el proyecto en curso.

- El directorio de los archivos de datos (archivos ".FIC", ".MMO" y ".NDX") utilizados durante una prueba del proyecto. 

- La lista de idiomas del proyecto y el idioma principal, que se utilizará de forma predeterminada para ver el proyecto en tiempo de ejecución en el editor.

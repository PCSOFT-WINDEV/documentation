## Presentación
<a name="presentacion_ELTTEXTE000219"></a>
La ventana del proyecto description le permite configurar varios elementos de su proyecto. Esta ventana incluye las siguientes pestañas:

- La pestaña ["Proyecto"](../Editeurs/2030046.md), que permite introducir información general sobre el proyecto

- La pestaña "Análisis", que permite asociar un análisis al proyecto y/o a un modelo UML.

- La pestaña ["Archivos"](../Editeurs/2030052.md), que permite introducir información sobre los archivos de datos utilizados por la aplicación (sólo si el proyecto está vinculado a un análisis).

- La pestaña "Datos en vivo", que permite configurar cómo se muestran los datos de los ficheros de datos en los distintos editores.

- La pestaña ["Idiomas"](../Editeurs/9500219.md), que se utiliza para configurar los diferentes idiomas utilizados por el proyecto, así como las opciones que se utilizarán para cada idioma.

- La pestaña ["Estilo"](../Editeurs/9500234.md), que permite personalizar la vista previa de impresión y elegir el tema y la hoja de estilo personalizada de plantilla. También puedes configurar el tamaño de la pantalla.

- La pestaña "SCM", que sirve para gestionar el trabajo en equipo (SCM o GIT).

- La pestaña **"Opciones"**, que permite configurar las opciones de actualización, copia de seguridad y sincronización de modelos. Esta pestaña también se utiliza para habilitar el modo estricto y para definir el estilo de código.

- La pestaña ["Avanzado"](../Editeurs/9500236.md), que permite configurar cómo utilizar "Reports & Queries", las AAF y cómo bloquear la aplicación.

- La pestaña ["Recopilación"](../Editeurs/9500218.md), que permite configurar las opciones de compilación.

- La pestaña ["Telemetría"](../Editeurs/9500223.md), que permite configurar las opciones de telemetría del proyecto.

### Varios
<a name="varios_ELTPARAGRAPHE000055"></a>

Las siguientes son diversas opciones: 

- "Preguntar la ubicación de los nuevos elementos en el proyecto": Si se selecciona esta opción, se solicita automáticamente la ubicación de un nuevo elemento para guardarlo en el proyecto. 

- "Generar todas las configuraciones en el directorio "Exe" (no recomendado si hay varias configuraciones)": Si se selecciona esta opción, todas las configuraciones se generarán en el mismo directorio. Si no se selecciona esta opción, cada configuración del proyecto se generará en un subdirectorio del directorio 'Exe' del proyecto. Este subdirectorio de generación tendrá el mismo nombre que el subdirectorio de configuración del proyecto.

- "Incluir nuevos elementos solo en la configuración actual": Si se selecciona esta opción, todos los elementos nuevos del proyecto se incluirán automáticamente **solo** en la configuración del proyecto actual en el editor. 

- "Generar una sola aplicación para la tienda privada (opción para compatibilidad)": Esta opción indica si el servidor que aloja la aplicación instalación para la Tienda privada utilizará ajustes diferentes para cada configuración de proyecto. Si esta opción está marcada, la configuración del servidor será la misma para todas las configuraciones del proyecto. Si esta opción no está marcada (Default), los ajustes de Tienda privada pueden ser diferentes para cada configuración de proyecto. 

- **Formato predeterminado para guardar elementos**: Puede elegir entre formato binario o de texto. El formato de texto es necesario para compartir archivos a través de Git. Para obtener más información, consulte [Compartir proyectos a través de Git](../Editeurs/9500230.md).

### Corrector ortográfico
<a name="corrector_ortografico_ELTPARAGRAPHE000092"></a>

Esta opción permite: 

- Seleccione el corrector ortográfico utilizado. Este corrector ortográfico será usado por Default para corregir la ortografía en los controles de edición y en los controles del procesador de textos. 

- "Copiar automáticamente los diccionarios en el directorio de generación": Si se selecciona esta opción, los diccionarios correspondientes a los idiomas de la aplicación se copiarán automáticamente en el directorio de generación cuando se genere la aplicación.. 







### Sincronización de plantillas
<a name="sincronizacion_plantillas_ELTPARAGRAPHE000105"></a>

La opción "Sincronizar modelos durante la generación" permite automatizar la sincronización de las plantillas de controles, ventanas, páginas o reporte al generar la aplicación. 






### Desarrollo en modo estricto
<a name="desarrollo_modo_estricto_ELTPARAGRAPHE000123"></a>

El desarrollo en modo estricto consiste en utilizar las tareas definidas en el Centro de Seguimiento de Proyectos para el desarrollo. 

Si se selecciona la opción "Activar modo estricto", el programador no podrá modificar un elemento del proyecto sin seleccionar previamente la tarea en la que se tendrán en cuenta las modificaciones. 




### Estilo de código
<a name="estilo_codigo_ELTPARAGRAPHE000135"></a>

El estilo de código permite agregar prefijos automáticamente al nombre de todas las variables y elementos del proyecto (ventanas, controles de reportes, clases, etc.). Este prefijo automatic permite identificar y/o encontrar fácilmente un elemento de un tipo determinado.

Para utilizar el estilo de código en el proyecto, basta con seleccionar "Activar prefijos automáticos de variables y elementos del proyecto".. Usted puede: 

- elegir el estilo de código deseado. 
	 Los botones "Crear un estilo de código" y "Editar estilo de código" le permiten crear y personalizar su propio estilo de código. 

- añadir automáticamente (o no) los prefijos (opción "Agregar prefijos automáticamente").

- generar un error UI si se utiliza un prefijo no válido ("Generar un error de compilación de UI si un control no tiene el prefijo correcto"). 




Para obtener más información, consulte [Estilo de código](../Editeurs/9000074.md).

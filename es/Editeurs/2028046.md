## Presentación
<a name="presentacion_ELTTEXTE000199"></a>
La creación del programa instalación asistente le permite seleccionar el perfil de implementación a utilizar. Este perfil contiene los parámetros de la servidor de aplicaciones así como los parámetros del sitio implementación.

Es posible:

- elegir un perfil creado de antemano (al actualizar un sitio, por ejemplo). En este caso, la información relativa al perfil se tiene en cuenta automáticamente.. Simplemente valide esta información.  

- crear un nuevo perfil (durante una primera instalación por ejemplo).

- elegir no perfil. En este caso, los parámetros del perfil de implementación se especificarán al instalar el sitio (instalación vía paquete o vía medios físicos).




Para una actualización, tiene la posibilidad de crear nuevos perfiles (en la mayoría de los casos, un perfil por servidor) y de seleccionarlos para la actualización.. El sitio se desplegará en todos los servidores seleccionados: 

- Si el sitio ya está instalado en el servidor, la asistente realizará una actualización.

- Si el sitio no está instalado en el servidor, el servidor realizará una nueva instalación.




**Observación**: Si la instalación falla en uno de los servidores, la instalación continúa normalmente en los otros servidores.

## Parámetros del servidor de aplicaciones
<a name="parametros_del_servidor_aplicaciones_ELTTEXTE000223"></a>


### Introducción de información sobre la servidor de aplicaciones
<a name="introduccion_informacion_sobre_servidor_aplicaciones_ELTPARAGRAPHE000033"></a>

Se debe especificar la siguiente información: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Info_serveur.gif&type=thumb)


- Dirección del servidor: nombre del ordenador accesible a través de la red, una IP Address o una INTERNET Address.
	En el caso de una instalación en un cluster WEBDEV, debe indicar la Address del proxy associated con el cluster. 

- Características de la cuenta implementación: Nombre de usuario y contraseña (definidos por el gestor de alojamiento en el Centro de Alojamiento control).


Observación: Atención: el nombre de usuario puede ir precedido del nombre del dominio para evitar confusiones. Por ejemplo: "miequipo\\prueba" o "midominio\\prueba".

A través de "Parámetros avanzados", puedes:

- Definir los parámetros del servidor. 

- Definir parámetros del sitio. 

- Especificar cómo transferir los archivos de instalación: 

	- FTP. Puede definir las características del servidor FTP (puerto, IP Address, método de transferencia).

	- HTTP (modo Default, a partir de la versión 25). 




- Especificar los comandos de instalación: 

	- usar peticiones autenticadas.

	- usar el protocolo HTTPS.

	- definir (si es necesario) las características del proxy.







Para obtener más información, consulte [Parámetros avanzados](#NOTE4_1).

" Probar parámetros" establece una conexión con el servidor Web y permite comprobar que toda la información introducida es correcta.

" Exportar parámetros del servidor" permite crear un fichero XML con los parámetros del servidor. Por Default, este fichero se crea en el subdirectorio "\\Personal\\WEBDEV servers" del directorio WEBDEV instalación. Por lo tanto, la información relativa a un servidor puede ser transmitida de un proyecto a otro.. Los parámetros pueden importarse mediante el botón "Importar los parámetros de un servidor".

## Parámetros de implementación del sitio
<a name="parametros_implementacion_del_sitio_ELTTEXTE000247"></a>
Entonces, la asistente permite introducir los parámetros de la implementación del sitio. Se debe especificar la siguiente información: 

- ***El nombre del sitio***. Por Default, este nombre corresponde al nombre del proyecto. Puede utilizar un nombre diferente. Esto le permite implementar varias instancias del mismo proyecto en el mismo servidor, dándoles diferentes nombres de sitio.. Por ejemplo, el proyecto "REWALI" puede desplegarse en un servidor web:

	- como REWALI: este sitio está destinado a la agencia de viajes X1.

	- como RESA: este sitio está destinado a la agencia de viajes X2.




- ***El nombre de dominio del archivo SiteMap***. Este parámetro debe ser completado en el caso de que el nombre del servidor utilizado para implementación sea diferente del nombre de dominio utilizado para acceder a él. El archivo SiteMap es un archivo utilizado por Google para hacer referencia.

- ***El subdirectorio que contiene los archivos de datos*** en formato HFSQL Classic. El nombre de este subdirectorio es relacionado al nombre del sitio. Corresponde al nombre del directorio donde están instalados los ficheros de datos del sitio.. Este directorio es un subdirectorio del directorio de datos asignado por el anfitrión. Este nombre de directorio debe ser modificado sólo si varios sitios comparten los mismos archivos de datos.

- ***El nombre del perfil de implementación ***. Se dará este nombre al perfil de implementación para poder encontrarlo fácilmente (al actualizar el sitio por ejemplo).




**Observación**: Si ha especificado alguna información en la configuración avanzada, esta información se recuperará automáticamente.

## Parámetros avanzados
<a name="parametros_avanzados_ELTTEXTE000271"></a>
Los parámetros del sitio implementación son los siguientes:

- ***El nombre del perfil de implementación ***. Se dará este nombre al perfil de implementación para poder encontrarlo fácilmente (al actualizar el sitio por ejemplo).

- ***Los parámetros del servidor*** (pestaña "General"): Servidor de aplicaciones Address, Cuenta de despliegue, Cuenta de usuario del sistema. Estos parámetros son los especificados en la pantalla anterior: "Configuración del servidor de aplicaciones".

- ***los parámetros del sitio*** (pestaña "General"): 

	- ***Nombre de implementación del sitio***. Por Default, este nombre corresponde al nombre del proyecto. Puede utilizar un nombre diferente. Esto le permite implementar varias instancias del mismo proyecto en el mismo servidor, dándoles diferentes nombres de sitio.. Por ejemplo, el proyecto "REWALI" puede desplegarse en un servidor web:

		- como REWALI: este sitio está destinado a la agencia de viajes X1.

		- como RESA: este sitio está destinado a la agencia de viajes X2.




- ***Nombre de dominio del archivo SiteMap***. Este parámetro debe ser completado en el caso de que el nombre del servidor utilizado para implementación sea diferente del nombre de dominio utilizado para acceder a él. El archivo SiteMap es un archivo utilizado por Google para hacer referencia.

- ***Directorio que contiene los archivos de datos*** en formato HFSQL Classic. El nombre de este subdirectorio es relacionado al nombre del sitio. Corresponde al nombre del directorio donde están instalados los ficheros de datos del sitio.. Este directorio es un subdirectorio del directorio de datos asignado por el anfitrión. Este nombre de directorio debe ser modificado sólo si varios sitios comparten los mismos archivos de datos.

- Si el sitio debe desplegarse en todos los idiomas o sólo en determinados idiomas (a seleccionar). 

- Transferencia de ficheros instalación (pestaña "Avanzado"):

	- Selección del método de transferencia: 

		- HTTP

		- FTP




- Al transferir por FTP, puede especificar: 

	- El Address del servidor FTP, 

	- El modo FTP: Active o FTP pasivo

## Validación
<a name="validacion_ELTTEXTE000295"></a>
Si ha elegido desplegar el sitio WEBDEV en un Servidor de Aplicaciones WEBDEV remoto, WEBDEV intenta conectarse al servidor Web especificado (y al servidor FTP especificado si la implementación se realiza por FTP) cuando se validan los parámetros implementación del sitio.

Si se produce un problema, aparece un mensaje de error.. Para obtener más información, consulte: [Error al realizar una instalación por FTP](../Editeurs/2028057.md).

## Presentación
<a name="presentacion_ELTTEXTE000344"></a>
El principio para trabajar con la Administrador de Código Fuente es el siguiente:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GestionnaireDeSources_synch.gif)


- Todos los desarrolladores trabajan en un copy local del proyecto. Este proyecto y todos sus elementos son sólo de lectura por Default.

- Cuando se abre un elemento de un proyecto, este elemento es de sólo lectura por Default: no se puede modificar. Para modificarla, hay que sacarla de la Administrador de Código Fuente, modificarla y volver a registrarla.

- Sólo el maestro puede modificar las características del proyecto (análisis, proyecto description, etc.).




**Observación**: El SCM sólo se puede utilizar con los sistemas operativos que soportan archivos de datos de más de 4 GB. La SCM no se puede utilizar en los ordenadores que ejecutan los siguientes sistemas operativos: Windows 98 y Me. Si el repositorio SCM está en modo HFSQL Client/Server, SCM puede utilizarse con estos sistemas operativos (en los ordenadores cliente).

Esta página de ayuda explica cómo trabajar con un proyecto que se encuentra en SCM. Para más información sobre el SCM, consulte [Crear un repositorio](../Editeurs/2038005.md).

## Apertura del proyecto desde SCM (sólo para la primera apertura del proyecto)
<a name="apertura_del_proyecto_desde_scm_solo_para_primera_apertura_del_proyecto_ELTTEXTE000368"></a>


### Trabajando en un proyecto que se encuentra en la Administrador de Código Fuente
<a name="trabajando_proyecto_que_encuentra_administrador_codigo_fuente_ELTPARAGRAPHE000030"></a>

Para trabajar en un proyecto que se encuentra en la Administrador de Código Fuente: 

1. Debes get un copy local del proyecto que se encuentra en el Administrador de Código Fuente. Para ello, abra el proyecto desde la página web de Administrador de Código Fuente.
	**Esta operación debe ser realizada una sola vez por cada promotor que utilice el proyecto.. El promotor que ha añadido el proyecto a Administrador de Código Fuente no tiene ninguna operación que realizar.**
	Importante: No copy los archivos localmente a través del explorador de Windows.

2. Despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_ouvrir.gif) en los botones de acceso rápido y seleccione "Abrir un proyecto desde el SCM". 
	Observación: También tienes la posibilidad de usar la página de inicio para abrir un proyecto que se encuentra en la SCM.

3. En la asistente, especifique:

	- la ruta del repositorio (ordenador de la red o directorio compartido)

	- el nombre de usuario y la contraseña (si es necesario).




4. Seleccione el directorio del repositorio que corresponde al proyecto deseado.

5. Especifique el directorio local en el que se copiará el proyecto.

6. Validar. El proyecto se copia localmente y se abre automáticamente de acuerdo con el modo especificado cuando se introdujo en el repositorio.




**Observación**: La próxima vez, podrá abrir un proyecto en el SCM como cualquier otro proyecto: solo debe abrir el proyecto (archivo ".WDP") que corresponde a la copia local.

### Opciones de proyectos relacionados con el SCM
<a name="opciones_proyectos_relacionados_con_scm_ELTPARAGRAPHE000057"></a>

Existen varias opciones para configurar un proyecto agregado al Administrador de Código Fuente. Estas opciones están disponibles en la pestaña "SCM" de la descripción del proyecto. Estas opciones están seleccionadas de forma predeterminada.
**Recordatorio**: Para abrir la descripción del proyecto, en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción". 

- **Proponer obtener la última versión de los elementos al abrir el proyecto**:
	Al abrir un proyecto ubicado en el SCM, esta opción propone obtener la última versión de los elementos del proyecto. De forma predeterminada, la última versión de los elementos se incorpora automáticamente.

- **Proponer la reintegración de los elementos al cerrar el proyecto**:
	Al cerrar el proyecto, esta opción muestra una lista de los elementos extraídos para reintegrar algunos o todos los elementos.
	De forma predeterminada, los elementos extraídos no se reintegran al cerrar el proyecto.

- **Extraer/Reintegrar automáticamente el proyecto**:
	Esta opción permite gestionar automáticamente el "archivo de proyecto". Esta opción se utiliza para desactivar la gestión "Master/Guest" en el proyecto..
	Si esta opción está marcada: 

	- El archivo de proyecto se comprueba sólo si la acción realizada lo requiere.. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Int%E9gration%20dans%20le%20GDS%20-%20HC%20N%B0007.gif)


	- Una vez realizada la acción en el proyecto, se propone automáticamente el registro del archivo del proyecto..
			Si la opción "Comprobar automáticamente el proyecto al final de la operación" se ha comprobado durante la comprobación, el archivo de proyecto se comprobará automáticamente (sin ventana de confirmación)..

### Los diferentes modos de pago
<a name="los_diferentes_modos_pago_ELTPARAGRAPHE000139"></a>

El SCM incluye dos modos para extraer los elementos del proyecto:

- **modo estándar**: si abre un elemento del SCM que no ha sido extraído, se abre una ventana que indica que el elemento debe extraerse para poder modificarlo. El elemento se puede extraer inmediatamente (con el botón de extracción de la ventana).

- **el modo automatic**: si intenta modificar un elemento del SCM que no ha sido extraído, el SCM propone extraerlo automáticamente. Una vez validada la extracción, se puede modificar el elemento. 
	Observación: este modo no se recomienda cuando se utiliza el SCM con una conexión lenta a Internet.




El modo de check-out de los elementos SCM es una opción de WINDEV, WEBDEV y WINDEV Mobile.

**Para cambiar el modo de extracción**: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile".

2. Seleccione la pestaña "General. La opción "Comprobar los elementos durante la primera modificación" se utiliza para cambiar todos los proyectos abiertos del producto actual al modo automatic.




Atención: En esta versión, el modo automatic se apoya en las ventanas, las páginas, las consultas y los informes.

## Modo de gestión del proyecto
<a name="modo_gestion_del_proyecto_ELTTEXTE000428"></a>
El SCM propone dos modos de gestión:

- [Gestión del proyecto en modo Maestro/Invitado](#NOTE4_2)

- [Gestionar el proyecto en modo automatic](#NOTE4_3)



<a name="NOTE4_2"></a>


### Gestión del proyecto en modo Maestro/Invitado
<a name="gestion_del_proyecto_modo_maestroinvitado_ELTPARAGRAPHE000184"></a>

El Administrador de Código Fuente distingue entre 2 tipos de usuarios:

- el maestro: el maestro es el usuario que inicialmente almacenó el proyecto en el Administrador de Código Fuente.

- los invitados: los invitados son los desarrolladores que manipulan el proyecto desde el Administrador de Código Fuente.
	No es necesario estar conectado en modo maestro en un proyecto. El modo "Maestro" solo es necesario para:

- cambiar las características del proyecto y reintegrar los cambios realizados en el repositorio.

- reintegrar todos los elementos para crear el ejecutable y el programa de instalación de la aplicación.




**Para cambiar del modo maestro al modo huésped**: en la pestaña "SCM", en el grupo "Proyecto", expanda "Maestro/Invitado" y seleccione "Convertirse en invitado en el proyecto (y comprobar todo en)".
En este caso, Administrador de Código Fuente propone comprobar todos los elementos del proyecto (incluido el archivo .WDP).

**Para pasar del modo huésped al modo maestro**: en la pestaña "SCM", en el grupo "Proyecto", amplíe "Maestro/Invitado" y seleccione "Convertirse en maestro en el proyecto".

**Modificar las opciones del proyecto**
Todos los usuarios del proyecto (maestro o invitados) pueden modificar las características del proyecto (primera ventana del proyecto, animación, estilo de código, etc.).
Sin embargo, solo el maestro del proyecto podrá reintegrar estos cambios en el Administrador de Código Fuente.
Las modificaciones realizadas por un invitado se perderán cuando el proyecto se actualice desde el repositorio.
<a name="NOTE4_3"></a>


### Gestionar el proyecto en modo automatic
<a name="gestionar_proyecto_modo_automatic_ELTPARAGRAPHE000227"></a>

Con el modo automático, el archivo de proyecto se extrae solo si la acción realizada lo requiere (independientemente del usuario). Una vez completadas las acciones en el proyecto, el archivo de proyecto se reintegra automáticamente.

El modo automatic le evita manejar el modo "Maestro/Inquilino" en el proyecto.

**To habilitar el modo automatic**: 

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. En la pestaña "SCM", seleccione "Check out/Check in the project automatically".

## Casos especiales
<a name="casos_especiales_ELTTEXTE000464"></a>


### Modificación del análisis
<a name="modificacion_del_analisis_ELTPARAGRAPHE000255"></a>

El análisis se abre en modo de lectura por Default. Por lo tanto, no puede modificarse.

**Para modificar el análisis, debe ser revisado**.

Para comprobar el análisis:

1. Abre el análisis en el editor de análisis.

2. Haga clic en el botón "Check out" de la barra de herramientas de SCM.

3. Validar la comprobación del análisis. El análisis se abre en modo de modificación. Se puede modificar.




Los cambios realizados en el análisis pueden afectar a diferentes elementos del proyecto (páginas, ventanas, informes, etc.).

**Antes de generar el análisis**, le recomendamos que get la última versión de todos los elementos del proyecto. De hecho, los elementos comprobados no pueden modificarse y, por lo tanto, no pueden sincronizarse con el análisis.. Por lo tanto, asegúrese de que:

- todos los elementos del proyecto están registrados,

- la última versión de los elementos del proyecto se encuentra en el equipo donde se genera el análisis.

## Archivos a instalar
<a name="archivos_instalar_ELTTEXTE000293"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000025"></a>

Uno de los principales pasos al crear una instalación es la selección de los archivos a instalar.

Mediante Default, el programa instalación selecciona automáticamente todos los archivos que se encuentran en el subdirectorio EXE de su proyecto excluyendo los archivos de datos (".FIC", ".NDX", ".MMO"). Los archivos seleccionados se instalarán en los ordenadores de los usuarios.

Los archivos que aparecen en verde corresponden a los nuevos archivos a instalar desde la última generación de instalación.

**Para añadir archivos a la instalación**haga clic en el botón "Añadir" y seleccione los archivos que desee añadir.

**Para borrar archivos de la instalación**, seleccione el archivo y haga clic en el botón "Remove".

**Para configurar las propiedades de instalación de cada archivo**, seleccione el archivo y haga clic en el botón "Propiedades".. Tiene la posibilidad de crear un atajo con una línea de comando para uno o más archivos, administrar las versiones de los archivos al actualizar la aplicación, administrar la modificación de datos de automatic, ... Para obtener más información, consulte [Propiedades del archivo](#NOTE4_1).



### Para una red instalación
<a name="para_una_red_instalacion_ELTPARAGRAPHE000044"></a>

Cuando se realiza una red instalación, el programa de creación de instalación asistente permite elegir el equipo en el que cada archivo debe ser instalado. Cada archivo puede ser instalado:

- sólo en el servidor (archivos de datos compartidos por todos los usuarios, por ejemplo).

- en los ordenadores de los usuarios únicamente (el ejecutable de la aplicación, por ejemplo).

- tanto en el servidor como en las computadoras de los usuarios (un archivo ".INI" por ejemplo, utilizado para configurar la aplicación instalada en el servidor con los parámetros de Default y las aplicaciones instaladas en las computadoras de los usuarios con parámetros específicos de los usuarios).




Para especificar los equipos en los que se debe instalar cada uno de los archivos, sólo tiene que marcar la casilla:

- "Servidor" para los archivos a instalar en el servidor. Estos archivos se instalarán cuando el programa instalación se ejecute en el servidor.

- "Cliente" para que los archivos se instalen en los ordenadores de los usuarios. Estos archivos se instalarán cuando el programa instalación se ejecute en los ordenadores de los usuarios.

- "Servidor" y "Cliente" para que los archivos se instalen tanto en el servidor como en los ordenadores de los usuarios.

## Framework WINDEV
<a name="framework_windev_ELTTEXTE000329"></a>


### Modo de configuración de la framework
<a name="modo_configuracion_framework_ELTPARAGRAPHE000063"></a>

El WINDEV framework corresponde a todos los WINDEV DLLs. Por Default, el WINDEV framework está incluido en el programa de instalación. Todos los elementos de framework necesarios para el funcionamiento de la aplicación se instalan automáticamente con la misma.

Para crear programas "más pequeños" de instalación, puedes..:

- descargue el WINDEV framework del sitio de PC SOFT. 

- instalar no framework. Se utilizará el framework instalado en los ordenadores de los usuarios.. Esta framework se descargará si no existe.




**Observaciones**: 

- WINDEV framework puede descargarse incluso si su aplicación utiliza DLLs renombradas.

- La descarga de framework se realiza mediante HTTPS. Si la descarga vía HTTPS falla, la descarga se completa vía FTP. 



<a name="NOTE3_2"></a>


### Descargar la framework
<a name="descargar_framework_ELTPARAGRAPHE000085"></a>

La opción "Descargar la framework WINDEV desde el sitio de PC SOFT" se utiliza para crear una instalación más pequeña.

Si su aplicación contiene ejecutables que usan un nombre común renombrado framework, tiene la capacidad de especificar el directorio instalación del framework (siguiente paso del asistente). Por defecto, el directorio de instalación es "&lt;srCommonProgramFiles&gt;\\ &lt;Nombre de la empresa&gt; \\Marco\\".

Al instalar la aplicación, el usuario tendrá que estar conectado a la INTERNET. La siguiente ventana se mostrará tan pronto como el programa instalación se inicie: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install_Framework.gif)


Siempre que se ejecute el ejecutable, todos los archivos de WINDEV framework se descargarán automáticamente del sitio de PC SOFT.. La versión de framework descargada corresponderá a la versión de WINDEV que se utilizó para crear el ejecutable.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install_Framework2.gif)


**Observación**: Para traducir los diferentes elementos que se encuentran en esta ventana, haga clic en el botón "Ajustes FTP" y en el botón "Traducir".
<a name="NOTE3_3"></a>


### Ubicación de la estructura
<a name="ubicacion_estructura_ELTPARAGRAPHE000102"></a>

Por Default, el framework común se instala en el directorio "&lt;Setup directory of elements common to several applications&gt;\\PC SOFT\\&lt;WINDEV version&gt;\\Framework",

Dónde:

- "&lt;Directorio de elementos comunes a varias aplicaciones&gt;" corresponde al directorio devuelto por [SysDir](../WDLang1/3073026.md) (con la constante *srCommonProgramFiles*).

- "&lt;WINDEV version&gt;" corresponde a la versión WINDEV que se utilizó para crear el ejecutable ("21.0" si el ejecutable se creó con WINDEV 21, por ejemplo).

## Propiedades de los archivos a instalar
<a name="propiedades_los_archivos_instalar_ELTTEXTE000365"></a>


### "Ficha Vía de acceso
<a name="ficha_via_acceso_ELTPARAGRAPHE000119"></a>

Esta pestaña se utiliza para especificar:

- el directorio instalación del archivo. Se pueden utilizar los siguientes accesos directos:

	- **&lt;SetupDir&gt;&gt;SetupDir**: Directorio de instalación elegido por el usuario durante la instalación

	- **Configuración de la unidad&gt; &lt;SetupDrive**: del directorio de instalación elegido por el usuario durante la instalación.

	- **&lt;WinDir&gt;&gt;Dirección del viento**: Directorio Windows de ordenadores de usuario

	- **&lt;Archivo de configuración&gt;&gt;.**: El directorio de archivos de datos de instalación. La creación del programa instalación asistente permite configurar este directorio. Para obtener más información, consulte [Configuración del directorio de archivos de datos](../Editeurs/2028086.md).

	- **&lt;srProgramFiles&gt;**: Directorio"Archivos de programa" de los ordenadores de los usuarios.




- el nombre del archivo instalado. Por Default, el archivo tiene el mismo nombre. Sin embargo, se puede dar un nombre diferente al archivo durante la instalación.



<a name="NOTE4_2"></a>


### "Ficha"Accesos directos
<a name="fichaaccesos_directos_ELTPARAGRAPHE000138"></a>

Esta ficha se utiliza para crear para el archivo seleccionado:

- un acceso directo en el menú "Inicio

- y/o un acceso directo en el escritorio.

- y/o atajo en la barra de inicio rápido.




Para crear un acceso directo: 

1. Seleccione el tipo de acceso directo que desea crear.

2. Introduzca el título del atajo en cada uno de los idiomas soportados por el programa instalación.

3. Introduzca el grupo de programas en el que se debe crear el acceso directo.
	**Observación**: Este grupo de programas se creará si no existe en el ordenador del usuario.

4. Especifique (si es necesario) los parámetros del acceso directo. La ejecución de este atajo iniciará un comando Line. 
	Por ejemplo, el comando Line: "C:\\MyDirectory\\MyApplication.EXE /Login = Bill" contiene:

	- la ruta completa del archivo associated con el atajo.
			En nuestro ejemplo: "C:\\MyDirectory\\MiAplicación.EXE".

	- los parámetros del acceso directo.
			En nuestro ejemplo: "/Login = Bill".







**Observación**: Por Default, por cada ejecutable instalado, se creará un atajo:

- en el menú "Inicio" de la computadora de instalación.

- en el escritorio de la computadora de instalación.



<a name="NOTE4_3"></a>


### "Pestaña"Versión
<a name="pestanaversion_ELTPARAGRAPHE000172"></a>

Por Default, al actualizar una aplicación, si ya existen archivos con el mismo nombre en el directorio de instalación, estos archivos son reemplazados (incluso si los archivos reemplazados son más recientes que los archivos instalados). No se muestra ninguna ventana de advertencia para confirmar estos reemplazos.

Para gestionar las colisiones:

- Para instalar el archivo seleccionado aunque ya se encuentre un archivo más reciente con el mismo nombre en el directorio de instalación, marque "Reemplazar el archivo aunque sea más reciente".

- Para evitar la instalación de los archivos seleccionados si se encuentra un archivo con el mismo nombre en el directorio de instalación, marque "No instalar si el archivo ya existe".

- Para pedir al usuario que confirme si los archivos seleccionados deben ser reemplazados o no, marque "Confirmar la sustitución del archivo".

- Para renombrar el archivo existente, marque "Renombrar el archivo reemplazado con el nombre de abajo", e introduzca el nombre y la extensión del archivo reemplazado. Por ejemplo: MyImage_Bak.GIF.

### Ficha"Framework" (sólo para los ejecutables)
<a name="fichaframework_solo_para_los_ejecutables_ELTPARAGRAPHE000214"></a>

Esta pestaña se utiliza para especificar el modo de uso de framework. La información que se muestra en esta pestaña no puede ser modificada y depende de la información especificada en la pantalla de asistente. Las diferentes posibilidades son las siguientes:

- **Don't install the framework for this executable**. En este caso, si el framework no está instalado en el ordenador actual, será descargado por el usuario.

- **Utilizar el framework WINDEV común**. Utilizar el framework común instalado en el ordenador del usuario. Si este framework común no está instalado, será descargado por el usuario.

- **Usar el framework común renombrado**: Use el común y renombrado framework instalado en la computadora del usuario.. Si este framework no está instalado, será descargado por el usuario.

- **Usar un framework específico del ejecutable**: Utilice el framework especificado al crear el ejecutable.

- **Usar un framework específico del ejecutable (DLL renombradas)**: Utilice el framework especificado al crear el ejecutable.

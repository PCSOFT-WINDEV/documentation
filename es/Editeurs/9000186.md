


## Código Navegador (WLanguage): Framework 2
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000107"></a>
A partir de la versión 22, WEBDEV propone un nuevo WLanguage framework para el código del navegador. Esta nueva versión propone un comportamiento cercano al código del servidor y completa las características del código del navegador.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Activación de framework V2
<a name="activacion_framework_ELTTEXTE000131"></a>


### ¿Cómo proceder?
<a name="¿como_proceder_ELTPARAGRAPHE000018"></a>

El nuevo WLanguage framework está habilitado globalmente para un proyecto: 

1. Abra la ventana de descripción del proyecto: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción". 

2. En la pestaña "Avanzado", para la opción "Navegador WLanguage", seleccione "Versión 2". 

3. Validar. 




Importante: Para utilizar WLanguage framework versión 2, el estándar de generación de páginas debe ser "HTML 5".. Para definir la norma de generación: 

1. Abra la ventana de descripción del proyecto. 

2. En la pestaña "Avanzado: 

	- Seleccione el estándar "HTML 5. 

	- En las opciones de generación (botón "..."), marque "Forzar Internet Explorer en modo HTML5 en intranet (X-UA-compatible)". 


Si una de estas condiciones no se cumple, la opción de "Navegador de idiomas" será Grayed y forzada en la "Versión 1".





### ¿Cuándo debo usar (o no) la nueva framework?
<a name="¿cuando_debo_usar_nueva_framework_ELTPARAGRAPHE000052"></a>

El nuevo navegador framework propone un comportamiento cercano al servidor WLanguage. Por lo tanto, se modifican algunos comportamientos del código del navegador, especialmente el método para pasar parámetros a los procedimientos.

**No se recomienda modificar el modo de generación de un proyecto en producción** para evitar modificar el comportamiento del código y evitar las regresiones.
Para utilizar la nueva versión de framework en un proyecto existente (para beneficiarse de las nuevas características), le aconsejamos que realice pruebas completas de diferentes códigos de proyecto para asegurarse de que el cambio de framework no desencadena ninguna regresión.

Al crear un nuevo proyecto, puede ser interesante utilizar el framework en la versión 2 para beneficiarse de la comodidad de programación de las nuevas características.

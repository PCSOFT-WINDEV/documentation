## Presentación
<a name="presentacion_ELTTEXTE000133"></a>
Se pueden utilizar varios métodos para distribuir una aplicación WINDEV:

- a un usuario directamente. El usuario instala la aplicación en su ordenador. Esta aplicación puede ser una aplicación de un solo usuario, una aplicación multiusuario, ... La instalación es una instalación independiente.

- instalado en una red o en un servidor HTTP. El usuario instala su aplicación desde la red o desde una página de INTERNET. La actualización se propondrá automáticamente al usuario. Esta aplicación puede ser una aplicación de un solo usuario, una aplicación multiusuario, ....




Para obtener más información, consulte [Diagram para instalar una aplicación](../Editeurs/2028008.md).

## Instalación individual
<a name="instalacion_individual_ELTTEXTE000157"></a>


### Opciones disponibles
<a name="opciones_disponibles_ELTPARAGRAPHE000026"></a>

Si selecciona una instalación individual, puede elegir entre:

- **Estándar instalación**. El programa instalación utilizado es el que se suministra con WINDEV o una versión personalizada (el ejemplo de WDSetup corresponde al código fuente del instalador y puede ser modificado).

- **Compacta instalación**. El tamaño del programa instalación generado se optimiza para ser descargado de INTERNET.

- [Instalación mediante un paquete MSI](../Editeurs/9000025.md). El programa instalación utiliza el instalador estándar de Windows.

## Configuración con la actualización de automatic
<a name="configuracion_con_actualizacion_automatic_ELTTEXTE000181"></a>


### Opciones disponibles
<a name="opciones_disponibles_ELTPARAGRAPHE000041"></a>

Si seleccionas una instalación con actualización de la automatic, puedes elegir:

- Configuración en red de área local (LAN) con "Push" posiblemente activado. Una primera aplicación instalación se realizará en la red, en un directorio compartido. Este directorio contendrá el programa instalación utilizado por los usuarios finales y la versión de reference utilizada para gestionar las actualizaciones.

- [Configuración por HTTP](../Editeurs/9000026.md). Una primera instalación de la aplicación se realizará en un servidor HTTP. Los usuarios se conectarán a una página web para descargar el programa de aplicación instalación.

- [Instalación multisitio](../Editeurs/2028100.md). Esta instalación es la combinación entre una red local instalación y una HTTP instalación. El HTTP instalación se utiliza para actualizar la referencia instalación en la red local. 



<a name="NOTE3_2"></a>


### Comprobación de versiones
<a name="comprobacion_versiones_ELTPARAGRAPHE000058"></a>

Comprobar los medios de la versión:

- seleccione el ejecutable de su aplicación que se utilizará como reference para actualizar las aplicaciones que ya están desplegadas.

- seleccionar la versión mínima permitida para operar sin actualización. Para forzar a todos los usuarios a actualizar la próxima vez que inicien su aplicación, todo lo que tienen que hacer es copy la versión actual en la versión mínima.




Para obtener más información, consulte [Comprobación de las versiones](../Editeurs/2028035.md).
<a name="NOTE3_3"></a>


### Usuarios conectados
<a name="usuarios_conectados_ELTPARAGRAPHE000073"></a>

La gestión de los usuarios le permite informarse desde el servidor:

- los usuarios actualmente conectados a la aplicación,

- para cada usuario conectado:

	- el equipo solía iniciar la aplicación,

	- la versión de la aplicación utilizada.







Para obtener más información, consulte [Gestión de los usuarios de una aplicación con actualización de red](../Editeurs/2028059.md).

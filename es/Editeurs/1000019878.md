<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000151"></a>
Cuando el proyecto WINDEV Mobile para iPhone/iPad se genera en el ordenador de desarrollo, los archivos generados deben transferirse a un Mac para poder ser compilados.. 

Para obtener más información, consulte: 

- [Desarrollo de un proyecto para iPhone/iPad](../Editeurs/1000019818.md). 

- [Desarrollo de una aplicación para Apple Watch](../Editeurs/1000019919.md).

- [Generación de la aplicación iPhone/iPad](../Editeurs/1000019908.md).






## Versión de Xcode a utilizar
<a name="version_xcode_utilizar_ELTTEXTE000175"></a>
Los proyectos Xcode generados por **WINDEV Mobile 26** están adaptados a Xcode 11.2. Esta versión de Xcode debe usarse para probar e implementar aplicaciones para iOS. 
Atención: los proyectos generados a partir de WINDEV Mobile 26 sólo pueden ser utilizados en dispositivos iOS versión 11 o posterior.



## Compilación en Xcode en el MAC
<a name="compilacion_xcode_mac_ELTTEXTE000199"></a>


### Transferencia del proyecto WINDEV Mobile a MAC
<a name="transferencia_del_proyecto_windev_mobile_mac_ELTPARAGRAPHE000037"></a>

Para transferir el proyecto WINDEV Mobile al MAC:

1. Copie toda la carpeta generada en el directorio EXE en un medio externo (memoria USB, disco duro externo, directorio compartido con el MAC en la red). **El nombre del directorio es &lt;Nombre del proyecto&gt;.xcode.gen**. 

2. Pegue la carpeta en el MAC que va a compilar la aplicación.

3. Abra la carpeta en el MAC y abra el archivo "Project_name.xcodeproj".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IOS_selection_projet_xCode1.gif)


4. El proyecto se abre en Xcode.





### Gestión de firmas en Xcode 8 y superiores
<a name="gestion_firmas_xcode_8_superiores_ELTPARAGRAPHE000050"></a>

La aplicación de generación de iOS asistente permite gestionar las firmas en Xcode.

Si ha elegido esta opción en la [generación asistente](../Editeurs/1000019923.md), Xcode usará el modo de firma "automatic" de Default al abrir el proyecto. Se selecciona la opción "Gestionar automáticamente la firma".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=XCode_signature1.gif)
 TLa opción "Equipo" debe ser especificada antes de compilar el proyecto en Xcode. 
Observación: las características "Equipo" se especifican en las cuentas Xcode. El "Perfil de aprovisionamiento" se selecciona automáticamente. 

Puede elegir el modo de firma "manual" desmarcando la opción "Gestionar automáticamente la firma".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=XCode_signature2.gif)
 En este caso, Xcode te pide que llenes las características de la firma. Todo lo que tienes que hacer es seleccionar el "perfil de aprovisionamiento" deseado para los modos "Debug" y "Release".. Estos archivos pueden ser importados si no se encuentran en Xcode. 


### Compilar el proyecto en Xcode
<a name="compilar_proyecto_xcode_ELTPARAGRAPHE000068"></a>

WINDEV Mobile genera automáticamente un proyecto Xcode para compilar en iOS. Para simplificar la implementación de aplicaciones, WINDEV Mobile genera un "Scheme" para Xcode.

Para compilar el proyecto en Xcode: 

1. Seleccione las opciones de compilación en la lista desplegable de la parte superior izquierda de Corner. Seleccione el esquema que corresponde a su aplicación, y luego la plataforma de compilación (dispositivo conectado o simulador). 
	
	<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IOS_Choix_cible1.gif)<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IOS_selection_cible1.gif)


2. Para iniciar la compilación, seleccione la opción del menú "Product .. Clean" y luego "Product .. Build". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IOS_compilation_build.gif)


3. En la parte superior aparece un estado de compilación ("Exitoso", o el número de advertencias y errores). Puede hacer clic en los símbolos para ver la lista de errores/advertencias.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IOS_resultat_Compilation.gif)


4. Una vez compilado el programa sin errores, puede iniciar la simulación (seleccione "Producto... Ejecutar"). La ventana de simulación aparece con la aplicación.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IOS_simulation_iPhone.gif)





A continuación, puede probar su aplicación directamente en su iPhone o iPad. Para obtener más información, consulte [Desplegando la aplicación](../Editeurs/1000019864.md). 

Observaciones: 

- **Atención**: Desde abril de 2020, implementación a App Store requiere WINDEV Mobile 25.

- Generación de 32 y 64 bits: Las aplicaciones iOS son generadas por Default en 32 y 64 bits (impuestas por Apple). Sin embargo, en Xcode en el simulador de Apple, las aplicaciones SIEMPRE se ejecutarán en modo de 32 bits.

- En algunos casos, un error de enlace puede aparecer en Xcode 11.2. Este error corresponde a un problema referenciado en Xcode (petición en curso a Apple). Puede hacer lo siguiente a get su solicitud: 

	- Seleccione "Producto... Limpiar carpeta de compilación". 

	- Seleccione "Xcode .. Salir de Xcode" (no cerrar Xcode con la "X"). 

	- Reabrir el proyecto en XCode y reiniciar la compilación.

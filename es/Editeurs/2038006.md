<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000166"></a>
Para trabajar en un elemento de proyecto que se encuentra en la Administrador de Código Fuente, debe comprobar el elemento antes de que pueda ser modificado.

Hay dos modos de pago disponibles:

- Salida manual: Un elemento debe ser comprobado antes de que pueda ser modificado.

- Extracción automática: El elemento se comprueba automáticamente durante el primer intento de modificación.




**Para cambiar el modo de extracción**: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile".

2. Seleccione la pestaña "General. La opción "Comprobar los elementos durante la primera modificación" se utiliza para cambiar todos los proyectos abiertos del producto actual al modo automatic. 
	Atención: En esta versión, el modo automatic se apoya en las ventanas, las páginas, las consultas y los informes.

3. Validar.

## Comprobación de un elemento
<a name="comprobacion_elemento_ELTTEXTE000190"></a>


### ¿Cómo comprobar un elemento?
<a name="¿como_comprobar_elemento_ELTPARAGRAPHE000043"></a>

- **1ª solución: check-out solicitado** (disponible independientemente del modo de pago):

	1. Seleccione el elemento en el panel "Explorador de proyectos".

	2. Seleccione "Check out" en el menú contextual del elemento. 
			Otra solución: en la pestaña "SCM", en el grupo "Elemento actual", haga clic en "Extraer" (ícono ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GDS_ico_Extraire.gif)).




- **2ª solución: salida manual** ("Check-out durante la primera modificación" sin marcar):

	1. Abrir el elemento en el editor. Un panel naranja indica que el elemento no ha sido comprobado y que no puede ser modificado.

	2. En el panel naranja, haga clic en la caja Icon para modificar el elemento.




- **3ª solución: automatic check-out** ("Check-out durante la primera modificación" comprobado):

	1. Abrir el elemento en el editor.

	2. Realice la modificación deseada. Al validar la modificación, la SCM propone automáticamente comprobar el elemento para modificarlo.. Si se valida el check-out, la modificación se realiza en el elemento check-out. Si se rechaza la salida, el elemento no se modifica.






<a name="NOTE2_2"></a>


### Comprobación de un elemento
<a name="comprobacion_elemento_ELTPARAGRAPHE000073"></a>

Independientemente del método utilizado para comprobar el elemento, SCM propone las siguientes opciones durante el proceso de comprobación: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Int%E9gration%20dans%20le%20GDS%20-%20HC%20N%B0011.gif)
 El modo de salida puede ser:

- ***Exclusivo***: nadie puede extraer el elemento si no se ha reintegrado. El elemento solo se puede extraer para realizar pruebas.

- ***Para la prueba***: el elemento se puede modificar, pero las modificaciones no se pueden reintegrar.

- ***Múltiple***: otros usuarios también pueden extraer el elemento. En este caso, se pueden ver las diferencias de las versiones del elemento al reintegrarlo.




Validar. El elemento se abre. La barra de título indica que se ha extraído el elemento.

**Observaciones**:

- Si intenta modificar el código fuente de un elemento que no está marcado, Administrador de Código Fuente le propone automáticamente que compruebe el elemento.  

- Si marca la opción "No mostrar más este mensaje en este caso (haga clic en 'check out')", podrá volver a mostrar esta ventana a través de la ventana [opciones del administrador de SCM](../Editeurs/2038023.md).



<a name="NOTE2_3"></a>


### Comprobación de un elemento ya comprobado en modo exclusivo
<a name="comprobacion_elemento_comprobado_modo_exclusivo_ELTPARAGRAPHE000096"></a>

Si el elemento a retirar ya fue retirado en modo exclusivo por otro usuario, la siguiente ventana es mostrada por el Administrador de Código Fuente: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GDS_Deja_Extrait.gif)


Es posible:

- Ver las características del elemento de caja ("Propiedades")

- Enviar un mensaje a la persona que verificó el elemento (botón"Enviar un mensaje"). 
	**Recordatorio**: WINDEV, WEBDEV y WINDEV Mobile incluyen una herramienta de mensajería. Esta herramienta utiliza los parámetros especificados en los centros control (usuarios, ubicación de los archivos, etc.). Para abrir la mensajería, en la pestaña "Herramientas", en el grupo "Centros de Control", haga clic en "WDBal - Mensajería".

- Compruebe el elemento sólo para la prueba. En este caso, las modificaciones hechas al elemento serán ignoradas en el proyecto reference (que se encuentra en el servidor de SCM).




## Cobertura de código
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000223"></a>
La cobertura del código es la medida relativa a la cobertura de las pruebas ejecutadas en una aplicación o sitio.. Cada línea de código que se ha ejecutado se considera como "probada". Se tienen en cuenta las pruebas "manuales" (líneas de código ejecutadas durante una GO, por ejemplo) y las pruebas automatizadas.

**Atención**: La cobertura del código no le permite saber si las pruebas son exitosas.: sólo permite saber si se ha ejecutado la Line de código.

La cobertura de código se utiliza para detectar las líneas de código de una aplicación que nunca se han ejecutado.

Estas líneas de código se encuentran en todos los elementos de la aplicación:

- ventana,

- control,

- procedimiento, 

- method, 

- "código de sección",

- ...


La cobertura del código da una buena idea del nivel de prueba con respecto al código de la aplicación.

Observación: La cobertura del código recoge: 

- datos individuales en un archivo ".wdcoverage" que se encuentra en el directorio del proyecto. 

- datos del equipo en la base de datos de los Centros control. Por lo tanto, se comparte entre los diferentes usuarios de los Centros control. Esto le permite a get estadísticas sobre el equipo que está trabajando en el mismo proyecto.

## ¿Cómo ver la cobertura del código?
<a name="¿como_ver_cobertura_del_codigo_ELTTEXTE000247"></a>
La cobertura del código se puede ver desde varias ubicaciones: 

- Proyecto cuadro de mando,

- Explorador de proyectos, 

- Editor de código. 



<a name="NOTE2_2"></a>


### Proyecto cuadro de mando
<a name="proyecto_cuadro_mando_ELTPARAGRAPHE000047"></a>

La cobertura del código se ve en el proyecto cuadro de mando a través de un widget específico llamado "Cobertura". 
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Code%20coverage%20-%20HC%20N%B0001.gif)


Este widget muestra un gráfico con los porcentajes de código probado y no probado. 

Un clic en la flecha del widget sirve para ampliar el menú del widget. Este menú se utiliza para: 

- Mostrar el resultado del último GO. En este caso, sólo se muestra la cobertura del código con respecto al último GO de la configuración actual.. 

- Mostrar resultados personales. En este caso, se muestra la cobertura del código con respecto al desarrollador actual en la configuración actual.. Se tienen en cuenta los datos de las pruebas GO y automatizadas. 

- Mostrar resultados del equipo. En este caso, se muestra la cobertura del código de todos los desarrolladores que trabajan en la configuración actual.. Se tienen en cuenta los datos de las pruebas GO y automatizadas.

- Activar o desactivar el Widget (para mostrar o no los datos de cobertura de código en el cuadro de mando del proyecto. 

- Mostrar cobertura en el código. Se utiliza para visualizar las líneas cuya prueba se ejecutó en el editor de código.


 
Observación: Estas acciones también se pueden realizar desde la pestaña "Pruebas automatizadas". Para obtener más información, consulte [Configuración de los datos visualizados por cobertura de código](#NOTE3_1). 
<a name="NOTE2_3"></a>


### Explorador de proyectos
<a name="explorador_proyectos_ELTPARAGRAPHE000073"></a>El explorador del proyecto muestra la cobertura de código para cada elemento encontrado en el explorador. Se muestra un gráfico para cada elemento correspondiente al porcentaje de código ejecutado para este elemento.. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Code%20coverage%20-%20HC%20N%B0002.gif)


Una información específica se muestra en un tooltip cuando el cursor del ratón pasa por encima de este gráfico.. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Code%20coverage%20-%20HC%20N%B0003.gif)


La información mostrada en el explorador de proyectos se refiere a la configuración actual.. 

Tiene la posibilidad de configurar la información mostrada. Para obtener más información, consulte [Configuración de los datos visualizados por cobertura de código](#NOTE3_1). 
<a name="NOTE2_4"></a>


### Editor de código
<a name="editor_codigo_ELTPARAGRAPHE000091"></a>

En el editor de código, tiene la posibilidad de ver el código cuya prueba se ejecutó. Para habilitar esta función: 

- en el widget "Cobertura", seleccione "Mostrar la cobertura en el código". 

- en la pestaña "Pruebas automatizadas", en el grupo "Cobertura de código", haga clic en "Vista en el código". Si esta opción está marcada, la visualización en el código está activada.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Code%20coverage%20-%20HC%20N%B0001%201.gif)





El código cuya prueba se realizó se muestra en un Background azul.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Code%20coverage%20-%20HC%20N%B0004.gif)


Observación: La visualización en el código depende de: 

- la configuración actual,

- las opciones de visualización seleccionadas en la pestaña "Pruebas automatizadas" (último GO, ...). Para obtener más información, consulte [Configuración de los datos visualizados por cobertura de código](#NOTE3_1).

## Configuración de los datos visualizados por cobertura de código
<a name="configuracion_los_datos_visualizados_por_cobertura_codigo_ELTTEXTE000289"></a>
Los datos que muestra la cobertura del código se configuran a través de la pestaña "Pruebas automatizadas" de la cinta de opciones. 

Para seleccionar el tipo de datos a mostrar: 

1. En la pestaña "Pruebas automatizadas", en el grupo "Cobertura de código", despliegue la primera opción. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Code%20coverage%20-%20HC%20N%B0002%201.gif)


2. Seleccione los datos a visualizar: 

	- Último GO. En este caso, sólo se muestra la cobertura del código con respecto al último intento de configuración actual.. 

	- Total personal. En este caso, se muestra la cobertura del código con respecto al desarrollador actual en la configuración actual.. Se tienen en cuenta los datos de las pruebas GO y automatizadas. 

	- Total equipo. En este caso, se muestra la cobertura del código de todos los desarrolladores que trabajan en la configuración actual.. Se tienen en cuenta los datos de las pruebas GO y automatizadas.

	- nono. No se muestra la cobertura del código.




3. Los datos visualizados se actualizan automáticamente.  




Para reiniciar los datos: 

- Datos personales: En la pestaña "Pruebas automatizadas", en el grupo "Cobertura de código", haga clic en "Reinicializar datos personales". Se borran los datos personales del proyecto y de la configuración actual.. 

- Datos del equipo: En la pestaña "Pruebas automatizadas", en el grupo "Cobertura de código", haga clic en "Reinicializar datos personales". Los datos del equipo se eliminan de la base de datos de los Centros control para el proyecto y la configuración actual.

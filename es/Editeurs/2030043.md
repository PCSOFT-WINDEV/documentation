<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000190"></a>
Un proyecto es un conjunto lógico de elementos y procesos cuyo propósito es lograr un objetivo determinado. El proyecto gestiona:

- las relaciones entre los elementos (páginas, reportes, etc.)

- las relaciones entre los elementos y los archivos de datos (descritos en el análisis relacionado con el proyecto).




Un proyecto solo puede estar relacionado con un análisis.

Un mismo análisis puede asociarse a varios proyectos.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000214"></a>


### Crear un proyecto WINDEV Mobile
<a name="crear_proyecto_windev_mobile_ELTPARAGRAPHE000025"></a>

Para crear un proyecto WINDEV Mobile:

1. Inicie el asistente de creación de proyectos mediante una de las siguientes opciones: 

	- Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. En la ventana que aparece, haga clic en "Proyecto". El asistente de creación de proyectos se abre. Elija el tipo de proyecto a crear.

	- En la página de inicio de WINDEV Mobile, haga clic en "Crear un proyecto" y elija el tipo de proyecto que desea crear. 


 Observaciones: 

- Las etapas necesarias para crear el proyecto pueden variar en función del tipo de proyecto elegido.

- El nuevo proyecto contendrá una configuración específica para ese tipo de proyecto. Las características de la configuración del proyecto se pueden modificar en cualquier momento. El mismo proyecto se puede asociar a varias configuraciones de proyecto. Para obtener más información, consulte [Configuración de proyecto](../Editeurs/9000030.md). 

2. Para un proyecto "Android/iOS", indique: 

	- Si la aplicación será multiplataforma, solo Android o solo iOS. 

	- Especifique si el proyecto debe ser: 

		- un nuevo proyecto. 

		- un proyecto basado en un ejemplo. En este caso, la etapa siguiente permite seleccionar el proyecto de ejemplo utilizado para crear el nuevo proyecto. Solo se conserva del uso del SCM. A continuación, se crea el proyecto. Para obtener más información, consulte [Crear un proyecto basado en un ejemplo](#NOTE3_2).




3. Indique a qué tipo de dispositivos está destinado el proyecto. 

4. Especifique las opciones del proyecto:

	- ***El nombre y la ubicación***. Estas opciones no se pueden modificar. En WINDEV Mobile, el proyecto corresponde a un archivo ".WPP". Este archivo se puede abrir con WEBDEV y WINDEV. Para obtener más información, consulte [Proyecto común](../Editeurs/2030049.md).
			Todos los objetos asociados al proyecto se crearán en el directorio especificado.

	- ***La descripción del proyecto*** que resume el propósito del proyecto. Esta descripción se utiliza en la documentación del proyecto.




5. Pase a la etapa siguiente.

6. Especifique los documentos asociados al proyecto. Estos documentos pueden corresponder a archivos de texto, imágenes, documentos escaneados, ... Para obtener más información, consulte [Asociar documentos al proyecto](../Editeurs/9000092.md). Pase a la etapa siguiente.

7. Indique si el proyecto debe incluirse en el [Administrador de Código Fuente (SCM)](../Editeurs/2038001.md). El Administrador de Código Fuente simplifica el trabajo en equipo y también puede ser útil para el trabajo individual. 

8. Pase a la etapa siguiente.

9. Indique si desea utilizar el estilo de código. El estilo de código corresponde a un estándar de programación que puede aplicarse a toda la aplicación. Este estándar permite añadir prefijos a los diferentes elementos del proyecto, las variables, etc, ... Para obtener más información, consulte [Estilo de código](../Editeurs/9000074.md). Pase a la etapa siguiente.

10. Seleccione el **tema gráfico** utilizado si es necesario. Este tema gráfico se propondrá de forma predeterminada al crear una ventana o reporte y se aplicará a los nuevos controles. Esta opción se puede modificar en la ventana de descripción del proyecto (pestaña "Estilo"). Pase a la etapa siguiente.

11. Seleccione los ***idiomas del proyecto***. Estos idiomas se propondrán de forma predeterminada cuando se deba traducir la opción de un elemento (control, ventana, reporte , etc.). Seleccione el **idioma predeterminado en tiempo de ejecución**. Este idioma se utilizará para ver el proyecto en tiempo de ejecución. Pase a la etapa siguiente.

12. Indique si el proyecto debe utilizar una **base de datos**.
	Si su proyecto utiliza una base de datos, puede ser una base de datos existente o una nueva base de datos. Si utiliza una nueva base de datos, un asistente de creación de bases de datos iniciará automáticamente cuando haya creado el proyecto.

13. Valide. El proyecto que se acaba de crear permanece abierto.




**Observación**: Si ha elegido:

- usar el SCM, [el asistente para agregar proyectos a un repositorio](../Editeurs/2038002.md) se abre automáticamente.

- crear una base de datos, [el asistente correspondiente](../Editeurs/2011011.md) se abre automáticamente.

## Notas

### Archivo del proyecto
<a name="archivo_del_proyecto_ELTPARAGRAPHE000101"></a>

En WINDEV Mobile, el proyecto corresponde a un archivo ".WPP". Este archivo contiene:

- El código de inicialización del proyecto, que se ejecuta al abrir el proyecto.

- La lista de objetos relacionados (ventanas, reportes, etc.). La secuencia de estos objetos se puede ver en el editor de proyectos.

- El nombre de la primera ventana que se abre (si existe) al ejecutar el proyecto o cuando se ejecuta la prueba del proyecto.

- El nombre del análisis asociado al proyecto (si existe) y su contraseña. Los archivos de datos descritos en el análisis pueden ser manipulados por los elementos del proyecto (ventanas, reportes, etc.).

- El nombre del modelo UML asociado al proyecto (si existe).

- La hoja de estilo asociada al proyecto (si existe). La hoja de estilo contiene todos los estilos utilizados por los elementos del proyecto.

- El directorio de la lista de tareas a realizar en el proyecto en curso.

- El directorio de los archivos de datos (archivos ".FIC", ".MMO" y ".NDX") utilizados durante la prueba de un proyecto (en la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Depurar en").

- La lista de idiomas del proyecto y el idioma principal, que se utilizará de forma predeterminada para ver el proyecto en tiempo de ejecución en el editor.

### Crear un proyecto basado en un ejemplo
<a name="crear_proyecto_basado_ejemplo_ELTPARAGRAPHE000132"></a>

Crear un proyecto a partir de un ejemplo le permite crear un nuevo proyecto con todas las ventanas, reportes y códigos del ejemplo. Solo debe personalizar el ejemplo según sus necesidades. 

Este método de creación tiene varias ventajas: 

- Puede elegir el nombre del proyecto. 

- Trabaja con una versión independiente del ejemplo: no hay riesgo de que se sobrescriban las modificaciones. Si desea seguir las evoluciones del ejemplo, simplemente compare su proyecto con el del ejemplo. Para obtener más información, consulte [Comparador de proyectos](../Editeurs/2030024.md). 

- Puede crear o eliminar elementos sin modificar el ejemplo.

## Crear las diferentes ventanas de patrones
<a name="crear_las_diferentes_ventanas_patrones_ELTTEXTE000165"></a>


### Los ficheros a crear
<a name="los_ficheros_crear_ELTPARAGRAPHE000040"></a>

Recordatorio: Para construir un patrón RAD utilizado para crear un proyecto completo, se requieren los siguientes archivos:

- Un formulario para RADFileA.

- Un formulario para RADFileB. Un cuadro combinado y un botón Vision Plus (o un cuadro combinado emergente) deben permitirle seleccionar un elemento de RADFileA.

- Un formulario para RADFileC. Un cuadro combinado y un botón Vision Plus (o un cuadro combinado emergente) deben permitirle seleccionar un elemento de RADFileB.

- Una tabla para RADFileA.

- Una tabla para RADFileB.

- Un tabla para RADFileC.

- Una ventana de relación RADFileB/RADFileD.




Estos diferentes elementos deben contener botones de enlace. También tiene la posibilidad de proporcionar el menú principal de la aplicación.

## Los botones de enlace
<a name="los_botones_enlace_ELTTEXTE000195"></a>
Las ventanas Formulario o Tabla pueden incluir botones de enlace. Estos botones de enlace se utilizan para mostrar un tabla, un formulario o un formulario tabla + en ficheros relacionado, por ejemplo.

Ejemplo simple: ver los pedidos de un cliente desde el formulario del cliente.

**Un botón de enlace debe ser nombrado**: 

- BTN_TableLink_RADFileX para un enlace a una ventana tabla

- BTN_FormLink_RADFileX para un enlace a una ventana de formulario

- BTN_RelationLink_RADFileX para un enlace a una ventana de relación.




**Observación**: BTN_ es el prefijo del estilo de código: no es necesario si no se utiliza ningún estilo de código.

Se puede utilizar cualquier leyenda. Sin embargo, le recomendamos que utilice un título simple (por ejemplo,"RADFileB tabla" o"RADFileA form").

**Code**: A continuación encontrará un ejemplo de código que puede utilizarse en un botón de relación, que permite abrir una tabla en el archivo ***RADFileB*** desde el formulario que se encuentra en ***RADFileA*/b>:


```wl
// Implement the filter on the link key 
HFilter(RADFileB,IDRADFileARel, RADFileA.IDRADFileA)
// Open the table window of RADFileB
Open(WIN_Table_RADFileB)
// Disable filter 
HDeactivateFilter(RADFileB)
```


**Posición de los botones de enlace y botones de enlace adicionales**

Los botones de enlace se pueden colocar en cualquier lugar de la ventana. Al generar la ventana, los botones de enlace se crearán en la ubicación especificada. Dependiendo del tamaño del análisis utilizado por el RAD, se pueden encontrar más de 3 botones de enlace en una ventana. Por lo tanto, le recomendamos que planifique botones de enlace adicionales.

Estos botones de adición se utilizan para posicionar los botones de enlace. De hecho, los botones de enlace se pueden alinear horizontalmente, verticalmente o incluso en círculo.

Los botones de enlace adicionales deben tener un nombre específico: BTN_LINK_n (BTN_ es el prefijo en el estilo de código: no es necesario si no utiliza un estilo de código), donde n es el número del botón (por ejemplo, BTN_LINK_1). Estos botones no deben contener ningún código y deben ser associated sin acción preestablecida.

**Observación**: Cuando se realiza la generación RAD, si el número de botones de enlace es mayor que el número especificado en el patrón, los botones adicionales generados se superpondrán a la posición del último botón de enlace colocado en el patrón.

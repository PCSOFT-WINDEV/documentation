


## Creación y despliegue de un programa instalación: ¿Cómo proceder?
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000229"></a>
La creación del programa instalación permite desplegar una aplicación WINDEV en los ordenadores de los usuarios finales.

La creación del programa instalación consiste en especificar todos los parámetros necesarios para instalar y utilizar su aplicación WINDEV.

**WINDEV le permite crear un programa completo de instalación**. Este programa de instalación puede ser usado para instalar:

- una aplicación WINDEV.

- otros archivos. Instalar golosinas, por ejemplo (imágenes, vídeos,...).




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Diferentes tipos de instalación para una aplicación WINDEV
<a name="diferentes_tipos_instalacion_para_una_aplicacion_windev_ELTTEXTE000253"></a>
Hay varios tipos de instalación disponibles:

- ***Instalación estándar***: El programa de instalación es independiente de la aplicación. Todos los archivos necesarios para la aplicación se instalan en los equipos de los usuarios. Para realizar una actualización, esta debe ejecutarse en cada equipo cliente.

- ***Instalación compacta***: Realiza una instalación estándar. Sin embargo, el programa de instalación es más pequeño porque no utiliza archivos DLL específicos y algunas opciones de personalización no están disponibles.

- ***Instalación con actualización automática (instalación de red)***: Esta instalación permite:

	- gestionar la actualización automática de la aplicación instalada en la red.

	- obtener las características de los usuarios conectados a la aplicación en un momento determinado.

	- configurar la ubicación de los archivos necesarios para la aplicación. Se pueden gestionar diferentes casos, por ejemplo:

		- Instalar todos los archivos de la aplicación en el servidor.

		- Instalar la base de datos en el servidor y los archivos necesarios para ejecutar la aplicación (".EXE", ".WDL", ".DLL", etc.) en cada equipo cliente.

		- Instalar parte de la base de datos en el servidor y otra parte en cada ordenador de usuario. Los archivos necesarios para ejecutar la aplicación (".EXE", ".WDL", ".DLL", etc.) también se instalan en cada equipo cliente.




- ***Configuración de un Tienda privada***: Esta instalación permite instalar la aplicación en una Tienda privada. Este tipo de instalación permite: 

	- usuarios para acceder a un portal con las diferentes aplicaciones disponibles. 

	- actualización automática de las aplicaciones descargadas desde la página web de Tienda privada.

## ¿Cómo crear un programa de instalación?
<a name="¿como_crear_programa_instalacion_ELTTEXTE000277"></a>
Para crear un programa de instalación que permita:

- ***implementar una aplicación WINDEV***, es posible:

	- utilizar [la creación del programa instalación asistente](../Editeurs/2028062.md) de WINDEV: en la pestaña "Proyecto", en el grupo "Generación", despliegue "Procedimiento de instalación" y seleccione "Crear procedimiento de instalación". Esta asistente le permite crear fácilmente los tres tipos de programas de la instalación.

	- utilizar [el editor de instalación (WDInst)](../Editeurs/2028042.md): en la pestaña "Herramientas", en el grupo "Utilidades", haga clic en "WDInst". Este editor permite personalizar el programa de instalación.




- ***proporcionan otros archivos***, debe utilizar [la creación del programa instalación asistente](../Editeurs/2028062.md), iniciado desde WDInst ("Archivo .. Nuevo" de WDInst).
	Recordatorio: Para abrir WDInst, en la pestaña "Herramientas", en el grupo "Utilidades", haga clic en "WDInst".




Para ver las diferentes opciones de configuración del programa instalación propuestas por el asistente y/o por el WDInst, consulte el [Diferentes opciones de configuración propuestas por la asistente y el WDInst](../Editeurs/2028016.md).

## ¿Cómo instalar una aplicación WINDEV?
<a name="¿como_instalar_una_aplicacion_windev_ELTTEXTE000301"></a>


### Principio para la instalación estándar o compacta
<a name="principio_para_instalacion_estandar_compacta_ELTPARAGRAPHE000115"></a>

Para instalar la aplicación, inicie el programa de instalación de la aplicación en el equipo cliente. Los archivos necesarios para la aplicación se instalan en el directorio de instalación especificado.

Para instalar una actualización de la aplicación, ejecute el programa de instalación de la actualización en el equipo cliente. La actualización reemplaza los archivos necesarios en la aplicación.

Observación: El programa de instalación propone: 

- una "instalación rápida": en este caso, la aplicación se instala según los parámetros preestablecidos. No se solicita información al usuario.

- una "instalación personalizada": en este caso, el usuario sigue los diferentes pasos de la instalación asistente para personalizar los parámetros de la instalación. 




Tabla de resumen: [¿Cómo instalar una aplicación desarrollada con WINDEV?](../Editeurs/2028005.md)
<a name="NOTE4_2"></a>


### Principio para la red instalación
<a name="principio_para_red_instalacion_ELTPARAGRAPHE000135"></a>

Para instalar su aplicación:

1. Inicie el programa instalación del servidor desde un ordenador local (Install_server.exe). Los archivos necesarios para la aplicación de referencia se instalan en el directorio de instalación especificado. La aplicación de referencia corresponde a un programa de instalación cliente.

2. Ejecute el programa cliente instalación desde cada ordenador de usuario. Los archivos necesarios para la aplicación cliente se instalan en el directorio de instalación especificado.




Para instalar una actualización de su aplicación:

1. Inicie su actualización de instalación en el servidor. La actualización reemplaza los archivos necesarios de la aplicación de referencia. Puede guardar los archivos reemplazados.

2. Cuando la aplicación se inicia desde un ordenador de usuario, la actualización de la aplicación se propone automáticamente.




Observación: El programa de instalación propone: 

- una "instalación rápida": en este caso, la aplicación se instala según los parámetros preestablecidos. No se solicita información al usuario.

- una "instalación personalizada": en este caso, el usuario sigue los diferentes pasos de la instalación asistente para personalizar los parámetros de la instalación. 




Tabla de resumen: [¿Cómo instalar una aplicación desarrollada con WINDEV?](../Editeurs/2028005.md)

**automatic comprobar las actualizaciones**
La aplicación comprueba regularmente si hay una actualización de la red disponible. 
Recordatorio: la hora de comprobación puede ser set desde WDAdminExe o con [AppChangeParameter](../WDLang1/1000017260.md). 

Si hay una actualización disponible, se mostrará una Notification de Windows sólo si el usuario está utilizando la aplicación.

- **Para las actualizaciones opcionales**, el mensaje Notification es el siguiente: 
	Una actualización opcional para '&lt;nombre de la aplicación&gt;' está disponible. Para instalar esta actualización, cierre y reinicie la aplicación.

- **Para las actualizaciones obligatorias**, el mensaje Notification es el siguiente: 
	Existe una actualización obligatoria para '&lt;nombre de la aplicación&gt;'.. Para instalar esta actualización, cierre y reinicie la aplicación.


Estos mensajes no se pueden modificar ni configurar. 

La comprobación de las actualizaciones disponibles es siempre Active. 
**Observaciones**: 

- Si el usuario ha elegido no instalar una actualización opcional al iniciar la aplicación, esa versión no será propuesta por el mecanismo de Notification. 

- Las notificaciones de actualización obligatoria se muestran cada 5 minutos si la actualización no se ha instalado. 

- Las notificaciones opcionales de actualización no se vuelven a mostrar si la actualización no está instalada. Sólo aparecerá la próxima vez que se inicie la aplicación.






Gestión de las notificaciones: 

- Si la aplicación tiene un icono en el área de Notification, el icono se utilizará para resaltar el Notification. 

- Si la aplicación no tiene un icono en el área de Notification, se creará un icono para resaltar el Notification. Se utilizará un Icon diferente según si la actualización es opcional u obligatoria.


[AppDetectUPDAvailable](../WDLang1/1410087705.md) le permite personalizar la comprobación de actualizaciones de automatic.

### Instalación silenciosa
<a name="instalacion_silenciosa_ELTPARAGRAPHE000170"></a>

Para instalar silenciosamente una aplicación WINDEV (sin mostrar las diferentes pantallas el programa instalación), pase el parámetro /SILENT al programa INSTALL.EXE.
Observación: Esta opción no se puede utilizar con una MSI instalación o con una compacta instalación.

Observación: Por Default, la instalación silenciosa no crea los accesos directos correspondientes. Para forzar la creación de accesos directos, basta con pasar el parámetro /FORCEICONE=1 después de /SILENT en el comando Line. 
Por ejemplo: 

```txt
INSTALL.EXE /SILENT /FORCEICONE=1
```

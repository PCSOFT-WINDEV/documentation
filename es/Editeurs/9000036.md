## Desarrollo de aplicaciones asistido
<a name="desarrollo_aplicaciones_asistido_ELTTEXTE000202"></a>
El Desarrollo de Aplicaciones Asistido (AAD) es una ayuda permanente en el entorno. Esta ayuda sugiere acciones siempre que sea apropiado.

Para realizar la acción sugerida, todo lo que tiene que hacer es hacer clic en el tooltip.

El AAD afecta a varias áreas (GUI, código fuente, ...), sugiere el uso de un Component, propone mejorar un algoritmo o da consejos generales.


### Gestión y configuración del desarrollo de aplicaciones asistido
<a name="gestion_configuracion_del_desarrollo_aplicaciones_asistido_ELTPARAGRAPHE000033"></a>

Para configurar el desarrollo de aplicaciones asistido: 

1. En la pestaña "Inicio", en el grupo "Entorno", amplíe "Opciones" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile".

2. En la pestaña "AAD" puede hacerlo:

	- Especifique la posición de la información sobre la herramienta AAD (en la parte superior o inferior).

	- Pida que se le notifiquen los nuevos consejos.

	- Pida confirmación antes de ejecutar el DAA seleccionado.

	- Vuelva a activar un AAD.







Para volver a habilitar un AAD: 

1. En la pestaña "Inicio", en el grupo "Entorno", amplíe "Opciones" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile".

2. Seleccionar la pestaña "AAD. Aparece la lista de consejos desactivados.

3. Compruebe la Line correspondiente a la punta que debe ser reactivada.

4. Valide.





### No mostrar más un AAD
<a name="mostrar_mas_aad_ELTPARAGRAPHE000077"></a>

Se muestra un consejo sobre Desarrollo de Aplicaciones Asistido mientras que usted prefiere no hacerlo? ¿No quieres ver más este mensaje? Todo lo que tienes que hacer es desactivar el AAD.

Para desactivar un AAD:

1. Abra el menú contextual del AAD (haga clic con el botón derecho del ratón en la información sobre la herramienta).

2. Seleccione "No mostrar esta categoría de nuevo".




Observación: El menú emergente del AAD también se puede utilizar para configurar el AAD ("Configurar el AAD").

### Visualización de características no soportadas por plataforma
<a name="visualizacion_caracteristicas_soportadas_por_plataforma_ELTPARAGRAPHE000118"></a>

Los editores (ventanas, páginas...) proponen una ayuda automatic que permite ver las características no soportadas para cada plataforma de la configuración actual en la ventana de 7 pestañas.

Las características no soportadas están cubiertas por un tono azul.

Por ejemplo, en la ventana de description de una control Combo Box en una configuración de Java, los siguientes elementos no están soportados y por lo tanto no son accesibles:

- Imán eYe.

- Sangría izquierda.

- Alineación vertical.

- Opacidad.

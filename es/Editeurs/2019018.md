
## Prueba de una aplicación WINDEV y WINDEV Mobile
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000103"></a>
WINDEV y WINDEV Mobile ofrecen múltiples métodos para probar las aplicaciones:

- [Prueba de todo el proyecto](../Editeurs/2019017.md)
	Al probar el proyecto, puede simular cómo se ejecutará la aplicación cuando se lance desde un ejecutable. Esto le permite probar toda la aplicación, incluso si no está completa. Cuando se produce un problema al ejecutar el proyecto, puede iniciar el depurador para identificar y solucionar el problema.
	

- [Ensayo de una ventanilla única](../Editeurs/2019014.md)
	Probar una sola ventana le permite ejecutar la ventana actual en WINDEV o WINDEV Mobile. Esto le permite probar su proyecto desde una ventana determinada o probar una ventana una vez terminada. Al igual que para la prueba del proyecto, el depurador se puede iniciar tan pronto como se produzca un problema.. 
	

- [Prueba de una consulta única](../Editeurs/2019011.md)
	Probar una sola consulta le permite ejecutar la consulta actual en WINDEV o WINDEV Mobile. Esto permite comprobar el modo de funcionamiento de una consulta tan pronto como se haya completado su desarrollo.

- [Prueba de una sola reporte](../Editeurs/2019015.md)
	Probar una sola reporte permite ejecutar la reporte actual en WINDEV. Esto le permite verificar la implementación de una reporte tan pronto como se desarrolla.. Al igual que para la prueba del proyecto, el depurador se puede iniciar tan pronto como se produzca un problema.

- [Ejecución del proyecto paso a paso](../Editeurs/2019001.md)
	Ejecutar el proyecto paso a paso le permite iniciar el depurador al iniciar la aplicación.. Esta solución le permite monitorear el progreso de la aplicación.

- [Pruebas de regresión](../Editeurs/2019025.md)
	Las pruebas de regresión de un elemento de proyecto o de todo el proyecto permiten comprobar si una modificación (código, interfaz, etc.) no afecta al modo de funcionamiento de la aplicación.. WINDEV propone guardar las pruebas en el proyecto directamente para poder ejecutar estas pruebas para cada nueva versión de la aplicación.




Observaciones: 

- Durante una operación de depuración, todos los elementos en los que se detiene (por ejemplo, mediante un breakpoint) se abren en el editor.. La última prueba (ventana, consulta, reporte,...) se puede volver a ejecutar desde cualquier elemento del proyecto a través de "GO... Vuelva a ejecutar la última prueba" que se encuentra en los botones de acceso rápido. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GO_Relancer_le_dernier_test%20-%20HC%20N%B0001.gif)


- Las líneas de código ejecutadas por los distintos tipos de pruebas son tenidas en cuenta automáticamente por [Cobertura de código](../Editeurs/2019034.md).




## Depuración de un Component externo
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000181"></a>
¿Ha desarrollado un Component externo y quiere depurarlo? ¿El usuario de un Component externo envía un ejecutable indicando que tu Component externo no funciona? ¿Estás utilizando tu Component externo en otro proyecto y quieres depurarlo?

Nada es más fácil. WINDEV, WEBDEV y WINDEV Mobile le permiten depurar fácilmente sus componentes externos.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000205"></a>


### Depurar un componente
<a name="depurar_componente_ELTPARAGRAPHE000020"></a>

Para depurar un Component:

1. Abra el proyecto correspondiente a la Component externa.

2. En el panel "Explorador de proyectos", seleccione y active la configuración del proyecto correspondiente a la Component externa ("Activar esta configuración" en el menú contextual).  

3. Para configurar el modo de depuración del Component actual, vaya a la pestaña "Proyecto", grupo "Modo de prueba", expanda "Modo de prueba" y seleccione "Configurar modo de prueba".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Composant_debog.gif)


4. Seleccione la opción deseada. Es posible:

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Ejecutar prueba del componente desde el proyecto** (Default):
			Esta opción se utiliza para ejecutar la prueba de la Component desde su propio proyecto. Tiene la capacidad de especificar un comando Line.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Ejecutar prueba del componente ejecutando otro proyecto**: 
			Esta opción se utiliza para ejecutar la prueba de la Component ejecutando primero el proyecto que llama a la Component. La Component debe haber sido incluida en el proyecto. Esta opción es útil si has creado una Component y si quieres utilizarla en otro proyecto.
			Para utilizar esta opción, debe especificar el nombre y la ruta completa del proyecto donde se incluyó la Component. También tiene la capacidad de especificar un comando Line para iniciar el proyecto.
			Observación: el proyecto que llama a la Component debe haber sido abierto al menos una vez y recompilado desde el ordenador actual.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Ejecutar prueba del componente desde un ejecutable** (sólo WINDEV): 
			Esta opción se utiliza para realizar la prueba de la Component ejecutando primero un ejecutable. El Component debe haber sido incluido en este ejecutable. Esta opción es útil si uno de sus clientes tiene un problema con una Component y si sólo le envía el ejecutable por ejemplo.
			Para utilizar esta opción, debe especificar el nombre y la ruta completa del ejecutable en el que se incluyó la Component. También puede especificar un comando Line para iniciar el ejecutable.




5. En su proyecto, incluya los puntos de interrupción necesarios para depurar su Component.

6. Cuando se inicie la prueba del proyecto, se utilizará la configuración de depuración seleccionada. Cuando se encuentra un punto de interrupción (o la palabra clave [PARAR](../Motscles/1511014.md)), el depurador se iniciará automáticamente.



<a name="NOTE2_2"></a>


### Component multiproducto para WEBDEV
<a name="component_multiproducto_para_webdev_ELTPARAGRAPHE000095"></a>

Para depurar la parte WEBDEV de un Component multiplataforma (por ejemplo, un Component para WINDEV y WEBDEV), debe generar el Component que tiene el mismo nombre que el Component multiplataforma real generado desde WINDEV.
<a name="NOTE2_3"></a>


### Limitación
<a name="limitacion_ELTPARAGRAPHE000102"></a>

No se pueden depurar los Component s que utilizan el contexto de ejecución del proyecto "cliente" (es decir, el proyecto que utiliza el Component).

**Recordatorio**: En las opciones de Component se puede elegir si se utiliza o no el contexto de ejecución del proyecto "cliente": 

1. Abra la ventana de descripción de la configuración de tipo "Componente": 

	- Si la configuración del componente ya está abierta: en la pestaña "Proyecto", en el grupo "Configuración del proyecto", haga clic en "Configuración actual".

	- En el panel "Explorador de proyectos", abra el menú contextual de la configuración del componente y seleccione "Descripción". 




2. En la ventana que aparece, en la pestaña "General", haga clic en "Opciones del componente". 

3. En la pestaña "Derechos de acceso", elija cómo accede Component a los datos del proyecto del cliente. Para utilizar el contexto del proyecto cliente, seleccione "El componente accederá a los archivos de datos, ventanas, páginas y reportes del proyecto cliente.".

4. Validar.

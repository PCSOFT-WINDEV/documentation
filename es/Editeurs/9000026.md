


## Configuración por HTTP (Red instalación vía IP)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000268"></a>
La red instalación vía HTTP consiste en instalar y actualizar una aplicación WINDEV a través de un servidor HTTP (Live update). Este servidor puede ser un servidor Windows o un servidor Linux.

Sólo la versión de la aplicación reference está instalada en el servidor. La aplicación reference corresponde a la instalación que será ejecutada por los usuarios finales.

Durante una primera instalación, el usuario final instala su aplicación a través de INTERNET, a través de una página específica.

Cada vez que se ejecuta la aplicación, se realiza automáticamente una petición HTTP para comprobar si la aplicación se actualizó en el servidor.. Si hay una actualización disponible, la aplicación se actualizará automáticamente.

<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000292"></a>


### Creando una instalación a través de un acceso HTTP
<a name="creando_una_instalacion_traves_acceso_http_ELTPARAGRAPHE000024"></a>

Para crear una instalación a través de un acceso HTTP: 

1. Crear el ejecutable de la aplicación: haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_exe.gif) en los botones de acceso rápido.

2. Empieza a crear la instalación Procedure: en la pestaña "Proyecto", en el grupo "Generación", despliegue "Procedimiento de instalación" y seleccione "Crear procedimiento de instalación". 

3. En la asistente, haga clic en "2 - Setup" y seleccione "Acceder a las opciones de la instalación".

4. Elige "Configuración con actualización de automatic (Live Update)" y selecciona "Configuración por HTTP (en el servidor del propietario o en la NUBE)".

5. Se presentan los diferentes tipos de implementación: 

	- Crear un paquete de implementación. Esta opción se utiliza para realizar una instalación en un servidor HTTP(S) o en la NUBE para aplicaciones PC SOFT. 

	- Implementar inmediatamente en el CLOUD. Esta opción se utiliza para desplegar la aplicación en CLOUD para aplicaciones PC SOFT y requiere los identificadores de la plataforma CLOUD creada de antemano.




6. Seleccione "Crear un paquete de implementación". 

7. Especificar los parámetros para HTTP implementación. Es posible: 

	- Utilice un **conexión segura o no**. Es posible: 

		- definir el tipo de conexión al instalar la versión de reference en el servidor. 

		- usar sólo una conexión segura. 




- Usar **parámetros de autenticación**. Es posible: 

	- definir los parámetros de autenticación cuando se instala la versión de reference en el servidor. 

	- utilizar una conexión sólo con autenticación. En este caso, el usuario final tendrá que escribir un nombre de usuario y una contraseña para conectarse al servidor HTTP e instalar su actualización.. 

8. Siga los diferentes pasos de asistente y valide la creación del programa instalación. Para obtener más información, consulte [Asistente de instalación: Los pasos para crear la instalación de una aplicación WINDEV](../Editeurs/2028062.md). 



<a name="NOTE2_2"></a>


### Instalando la versión de reference en el servidor HTTP
<a name="instalando_version_reference_servidor_http_ELTPARAGRAPHE000074"></a>

Una vez que la instalación Procedure es creada, tienes la capacidad de ejecutar el programa instalación. Este programa también se puede ejecutar directamente a través del archivo INSTALL.EXE.

**Cuando se ejecuta el programa instalación**, la asistente se propone:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install_IP_Assistant_Instcl.gif)


- implementar la aplicación localmente, en el equipo actual.

- implementar la aplicación de forma remota en el servidor HTTP (en la empresa de alojamiento). 

- desplegar la instalación en la nube de PC SOFT.




**Caso #1: Despliegue de la aplicación localmente, en el equipo actual**. 
Este modo de instalación no está disponible durante la instalación en un servidor Linux.

- Si se instala un servidor HTTP en el ordenador actual, este servidor se configura y los archivos de la aplicación reference se copian en el directorio especificado.

- Si no hay un servidor HTTP instalado en el ordenador actual, se instala un servidor Default (WDADMINEXE) y se configura automáticamente. Los archivos de la aplicación reference se instalan en el directorio especificado.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install_IP_Assistant_Ins3.gif)

	En las opciones de instalación, si ha especificado que: 

	- el tipo de conexión se define durante la instalación, un cuadro combinado permite especificar el protocolo utilizado. 

	- la elección de la autenticación se realiza durante la instalación, tiene la posibilidad de especificar que se requiere una autenticación.   

	- se debe utilizar un Address externo específico, este Address se selecciona en el "Access IP Address" control. 





Haga clic en "Siguiente".

**Caso #2: Despliegue de la aplicación de forma remota, en el servidor HTTP** (en la empresa de alojamiento). En este caso, se debe utilizar una conexión FTP para enviar los elementos al servidor.

- Escriba la URL de su sitio en la empresa de alojamiento: esta información corresponde a la Address que será introducida por los usuarios para acceder a la página de la instalación de su aplicación. La instalación de la aplicación reference se encuentra en esta URL.
	En las opciones de instalación, si ha especificado que: 

	- el tipo de conexión se define durante la instalación, un cuadro combinado permite especificar el tipo de Address (http o https). 

	- la elección de la autenticación se realiza durante la instalación, tiene la posibilidad de especificar que se requiere una autenticación.




- Introduzca las características del servidor FTP que se utilizará para instalar la aplicación reference en el servidor HTTP.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install_IP_Assistant_Ins2.gif)

### Actualización de la aplicación
<a name="actualizacion_aplicacion_ELTPARAGRAPHE000138"></a>

Para actualizar su aplicación:

1. Crear el ejecutable de la aplicación: haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generer.gif) en los botones de acceso rápido.

2. Empieza a crear la instalación Procedure: en la pestaña "Proyecto", en el grupo "Generación", despliegue "Procedimiento de instalación" y seleccione "Crear procedimiento de instalación".

3. En la asistente, haga clic en "2 - Setup" y seleccione "Acceder a las opciones de la instalación".

4. Elija "Configuración con actualización de automatic (Live Update)" y seleccione "Configuración por HTTP (en el servidor del propietario o en la NUBE)". 

5. Validar la creación del programa instalación.

6. Instala la versión de reference en el servidor.




Una vez que la versión de reference se actualiza en el servidor, las aplicaciones de los usuarios finales se actualizarán de acuerdo con las opciones de actualización especificadas (durante su próximo inicio por Default).

## Observaciones
<a name="observaciones_ELTTEXTE000328"></a>


### Actualización del equipo del usuario final
<a name="actualizacion_del_equipo_del_usuario_final_ELTPARAGRAPHE000170"></a>

Según las opciones de gestión de actualizaciones (siempre que la aplicación se inicia por defecto), se envía una solicitud HTTP al servidor para comprobar si la aplicación reference se ha modificado desde la última actualización. En caso de modificación, la actualización de la aplicación se propone automáticamente al usuario final.
<a name="NOTE3_2"></a>


### Protocolos utilizados
<a name="protocolos_utilizados_ELTPARAGRAPHE000177"></a>

La instalación y las actualizaciones de la aplicación reference en el servidor se realizan mediante el protocolo FTP.

La instalación y las actualizaciones de la aplicación en los ordenadores de los usuarios finales de la aplicación reference se realizan: 

- por el protocolo HTTP. 

- por el protocolo HTTPS.




Las peticiones HTTP descargan los archivos (extensiones".tmp", ".net" y ".zip") desde el servidor.. Por lo tanto, estas extensiones deben ser aceptadas por el servidor web que aloja la aplicación reference. Para obtener más información, consulte [FAQ #3102](https://faq.pcsoft.fr/faqread.awp?idfaq=3102&lang=es).
<a name="NOTE3_3"></a>


### Cómo transformar una aplicación de red en una aplicación de red mediante HTTP
<a name="como_transformar_una_aplicacion_red_una_aplicacion_red_mediante_http_ELTPARAGRAPHE000196"></a>

La aplicación debe reinstalarse en el ordenador del usuario final.
<a name="NOTE3_4"></a>


### Página de descargas
<a name="pagina_descargas_ELTPARAGRAPHE000203"></a>

Cuando se despliega la aplicación, se crea automáticamente una página HTML con un enlace al archivo instalación de la aplicación cliente (Install.exe).

Puede reutilizar esta página y adaptarla a sus necesidades, por ejemplo: 

- modificando la página HTML en el servidor web después de haberla desplegado,

- personalizando la página de modelo en WINDEV en el ordenador de desarrollo. Así, todas las configuraciones generadas posteriormente utilizarán esta página personalizada. Puede encontrar la página modelo ("InstIP.html") en "\\Programas\\Datos\\Modelos\\WDInst\\InstIP\\Estándar\\" en el directorio WINDEV instalación.




Puede trasladar esta página al servidor para integrarla en un sitio estático, por ejemplo. Para ello, actualice el enlace de descarga de la página.
<a name="NOTE3_5"></a>


### Administración de un servidor proxy
<a name="administracion_servidor_proxy_ELTPARAGRAPHE000217"></a>

Tiene la posibilidad de desplegar un instalación HTTP en una red que utiliza un proxy. 

La aplicación instalación y las actualizaciones en los ordenadores clientes utilizarán automáticamente el proxy sin tener que realizar ninguna configuración adicional.

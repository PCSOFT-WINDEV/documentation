
## Manual implementación de un sitio dinámico
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000259"></a>
WEBDEV propone varios métodos para desplegar un sitio dinámico:

- [implementación por FTP](../Editeurs/2028019.md).

- [implementación por medios físicos](../Editeurs/2028090.md), [por medios físicos independientes](../Editeurs/2028091.md).

- [implementación por paquete](../Editeurs/2028089.md).




En algunos casos, estas soluciones pueden ser ineficientes y el sitio debe ser instalado manualmente.

Esta página de ayuda presenta las diferentes operaciones a realizar.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## "Manual" implementación para una primera instalación
<a name="manual_implementacion_para_una_primera_instalacion_ELTTEXTE000283"></a>


### Etapas a seguir
<a name="etapas_seguir_ELTPARAGRAPHE000035"></a>

1. [Transferencia de archivos por FTP](#NOTE2_2).

2. [Creación del alias correspondiente al sitio en el servidor y el enlace para las URLs cortas](#NOTE2_3).

3. [Añadir la aplicación al administrador de "WEBDEV servidor de aplicaciones".](#NOTE2_4).



<a name="NOTE2_2"></a>


### Transferencia de archivos por FTP
<a name="transferencia_archivos_por_ftp_ELTPARAGRAPHE000050"></a>

Los siguientes archivos deben ser transferidos al servidor web:

- El directorio &lt;ProjectName&gt;_WEB (y sus subdirectorios), directorio de imágenes y archivos JavaScript.

- El contenido del subdirectorio "EXE" del proyecto de desarrollo, o al menos el archivo &lt;SiteName&gt;.WDL (no olvide crear previamente una biblioteca para que este archivo se actualice)

- Los directorios de las "home pages" para los diferentes idiomas: FR, UK (sólo para las versiones anteriores a WEBDEV 11)



<a name="NOTE2_3"></a>


### Creación del alias (o directorio virtual) y el enlace para las URLs cortas
<a name="creacion_del_alias_directorio_virtual_enlace_para_las_urls_cortas_ELTPARAGRAPHE000061"></a>

Para realizar esta operación, debe estar localizado en el servidor (o debe ser capaz de control a distancia)

Debe crear un alias (o directorio virtual) llamado &lt;ProjectName&gt;_WEB en el servidor Web (IIS, Apache). Este alias debe "apuntar" al directorio del mismo nombre que fue transferido.

Con IIS 7 (o posterior) y Apache 2.2 (o posterior), también debe crearse un enlace entre el nombre del sitio y el WEBDEV servidor de aplicaciones para gestionar el [URLs cortos](../Editeurs/2030058.md).

Para obtener más detalles, consulte los procedimientos para configurar los sitios WEBDEV dinámicos con los siguientes servidores:

- [Apache 1.3.x, Apache 2.0.x](../Editeurs/2037011.md)

- [Apache 2.2](../Editeurs/2037013.md)

- [FrontPage 97 y 98](../Editeurs/2037009.md)

- [Profesional de OmniHTTPD](../Editeurs/2037003.md)

- [IIS versión 2 o 3](../Editeurs/2037002.md)

- [IIS versión 4](../Editeurs/2037010.md)

- [IIS versión 5](../Editeurs/2037007.md)

- [IIS versión 6](../Editeurs/2037008.md)

- [IIS versión 7 y 7.5](../Editeurs/2037012.md)

- [Domino](../Editeurs/2037006.md)



<a name="NOTE2_4"></a>


### Añadir el sitio dinámico al administrador de "WEBDEV servidor de aplicaciones".
<a name="anadir_sitio_dinamico_administrador_webdev_servidor_aplicaciones_ELTPARAGRAPHE000098"></a>

Para realizar esta operación, debe estar localizado en el servidor (o debe ser capaz de control a distancia)

**Para agregar el sitio WEBDEV al administrador del Servidor de Aplicaciones WEBDEV**, solo debe:

1. Seleccione la pestaña "Sitios" en el administrador.

2. Haga clic en el botón "Agregar".

3. Seleccione la biblioteca del sitio (&lt;SiteName>.WDL).



<a name="NOTE2_5"></a>


### Observación
<a name="observacion_ELTPARAGRAPHE000111"></a>

La configuración de un sitio WEBDEV puede ser añadida o comprobada a través del registro (REGEDIT).. Los parámetros de un sitio se agrupan en el registro del servidor en la siguiente clave: 

[ HKEY_LOCAL_MACHINE\\Software\\PC Soft\\WebDev\\28.0\\Aplicaciones\\SITENAME]

donde SITENAME es el nombre del sitio correspondiente.

Los diferentes valores son:

| Nombre del valor | escribir a máquina | Significado |
| --- | --- | --- |
| NOMBRE DEL PROYECTO | Cadena | Nombre del proyecto |
| PROYECTO | Cadena | Proyecto instalación ruta |
| VERSIÓN | Cadena | Versión desplegada |
| VIMAGEDIR | Cadena | Nombre del directorio virtual de imágenes |
| HFPATH | Cadena | Ruta de los archivos de datos de HyperFileSQL<br><br> |
| AS400 | PALABRA | 1 si el sitio utiliza el conector nativo de AS/400 |
| BLOQUEO PARA ACTUALIZAR | Cadena | 1 si el sitio está bloqueado |
| LOGDIR | Cadena | Ruta de los archivos de registro |
| MAXCONNNECTAPP | Cadena | Número máximo de usuarios conectados |
| MAXRECONNECTAPP | Cadena | Número máximo de usuarios conectados con el mismo Address |
| TIMEOUT_NORQ | Cadena | Duración máxima (en segundos) de una solicitud antes de la desconexión |

## "Manual" implementación para una actualización
<a name="manual_implementacion_para_una_actualizacion_ELTTEXTE000337"></a>
Todo lo que tienes que hacer es realizar el paso "Transferir los archivos por FTP" del "manual" de implementación para una primera instalación.

**SUGERENCIA**: Con el fin de "perturbar" el sitio lo menos posible, le aconsejamos que transfiera sus archivos a nuevos directorios para tener un directorio que contenga el sitio actual y un directorio que contenga el nuevo sitio.. Sólo tendrá que "bloquear" el sitio para renombrar los dos directorios (y moverlos).

**ATENCIÓN**: Esta operación no inicia la modificación de la estructura de los archivos de datos de automatic si esta estructura ha evolucionado. Si se ha cambiado el formato de archivo, debe recuperar los archivos localmente, realizar la modificación de estructura y, a continuación, volver a enviar los archivos modificados al servidor.


## Generación de una aplicación para Universal Windows 10 App
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000325"></a>
La generación de una aplicación en modo Universal Windows 10 App agrupa todos los elementos del proyecto (ventanas, código, etc.) en un archivo para que la aplicación se despliegue en un dispositivo Windows 10 y se distribuya.

Para más detalles sobre cómo crear una aplicación universal de Windows 10, consulte [Desarrollo de aplicaciones en el modo Universal Windows 10 App](../Editeurs/1410086576.md).

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000349"></a>


### Generación de una aplicación en modo Universal Windows 10 App
<a name="generacion_una_aplicacion_modo_universal_windows_10_app_ELTPARAGRAPHE000022"></a>

Para generar una aplicación en el modo Universal Windows 10 App: 

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Ico_Generation_W10.gif) en los botones de acceso rápido. Se inicia la asistente para generar la aplicación en modo Universal Windows 10 App.
	Observación: Si no están instalados todos los módulos necesarios para generar una aplicación en modo Universal Windows 10 App, la asistente indica los elementos que faltan. Para obtener más información, consulte [Configuración necesaria](../Editeurs/1410086574.md). 

2. Seleccione el modo de generación: 

	- [Generación e implementación de la aplicación](#NOTE2_2). 

	- [Generación de la aplicación para Windows Store](#NOTE2_3). 






<a name="NOTE2_2"></a>


### Generación e implementación de la aplicación
<a name="generacion_implementacion_aplicacion_ELTPARAGRAPHE000046"></a>

Se inicia la generación asistente y luego la implementación asistente. 

Los diferentes pasos de la generación asistente son los siguientes: 

1. Especifique la información general sobre la aplicación: 

	- Nombre de la solicitud,

	- Descripción de la aplicación, 

	- Editor de software

	- Copyright. 




2. Pase a la etapa siguiente. 

3. Especifique el número de versión de la aplicación en formato estándar. Este formato corresponde a una set de 4 números separados por puntos. 

	- El primer grupo de 4 dígitos corresponde al número de cambios importantes realizados en la aplicación.

	- El segundo grupo de 4 dígitos corresponde al número de cambios menores realizados en la aplicación.

	- El tercer grupo de 4 dígitos corresponde al número de generación. 
			Le aconsejamos que incremente automáticamente este número en cada generación. Esta operación de incremento se realiza automáticamente si se selecciona la opción "Incrementar automáticamente la versión en cada generación".

	- El último grupo de 4 dígitos corresponde al número de revisiones (ramas en la SCM por ejemplo).




4. Pase a la etapa siguiente. 

5. Elija las diferentes imágenes associated con la aplicación. Por Default, WINDEV Mobile genera imágenes con el formato esperado. Puedes personalizar estas imágenes o usar tus propias imágenes. 
	Las imágenes esperadas son: 

	- El logo estándar de la aplicación para el cuadrado Tile (150 x 150 píxeles)

	- El pequeño logo para el cuadrado Tile y la lista de todas las aplicaciones (44 x 44 píxeles)

	- El logo para el cuadrado Tile (71 x 71 píxeles)

	- El logo para el cuadrado Tile (310 x 310 píxeles)

	- El gran logo de la aplicación para el Tile rectangular (310 x 150 píxeles)

	- El pequeño logo para el jefe de tareas, las miniaturas y los brindis de Notification (30 x 30 píxeles)

	- La imagen del splash screen (620 x 300 pixeles)

	- La Image de la aplicación mostrada en Windows Store (50 x 50 píxeles)
			Consejo: tienes la posibilidad de seleccionar un Image en el catálogo Image de WINDEV Mobile. Las imágenes esperadas se generarán automáticamente a partir de la imagen seleccionada y los archivos correspondientes se associated en la asistente. 




6. Pase a la etapa siguiente. 

7. Seleccionar: 

	- el fondo Color del Tile.

	- el modo de visualización de la Tile.  




8. Pase a la etapa siguiente.

9. Introduzca el mensaje de error que se mostrará si se produce un error de programación.. Este mensaje de error se puede escribir en todos los idiomas soportados por el proyecto actual.. Pase a la etapa siguiente. 

10. Especifique si los archivos deben incluirse en el paquete. Puedes incluir:

	- recursos específicos.

	- imágenes específicas.

	- archivos de datos. 




11. Especifique la opción "Escribir: 

	- **Si la opción "Escribir" está marcada para los archivos** , estos archivos se extraerán automáticamente (por ejemplo, los archivos que deben ser modificados por la aplicación). En este caso, especifique el directorio de la aplicación en el cual este archivo debe ser revisado (directorio de datos de Default). 

	- **Si la opción "Write" no está seleccionada para los archivos**estos archivos pueden ser:

		- usado directamente (archivos para leer por ejemplo: archivos de sonido, archivos de Image, etc.). Estos archivos están incluidos en la aplicación y no serán extraídos. [fDirección de recursos](../WDLang1/1000020166.md) se utiliza para acceder a estos archivos en modo de lectura.  

		- extraído a través de la programación con [fExtractResource](../WDLang1/1000019509.md). 




12. Pase a la etapa siguiente. 

13. Si su proyecto contiene varios idiomas, especifique los idiomas que se encuentran en la aplicación. Pase a la etapa siguiente. 

14. Seleccione las características utilizadas por la aplicación. 
	Observación: **If usted está utilizando las características relacionado a APIs**, el siguiente paso le permite declarar los ficheros utilizados por las APIs y sus características. 

15. Pase a la etapa siguiente. 

16. La firma de la solicitud es obligatoria. La asistente propone generar una Certificate o utilizar una Certificate existente. En este último caso, todo lo que tienes que hacer es seleccionar el Certificate solicitado. 
	Para generar un nuevo Certificate: 

	- Especificar el nombre de Certificate para generar. 

	- El siguiente paso de asistente se utiliza para crear la Certificate. 

		- Haga clic en el botón "Crear certificado". 

		- El explorador de Windows muestra el Certificate creado. 

		- En el explorador, haga clic derecho para abrir el menú contextual del archivo y seleccione "Instalar certificado". La importación de Certificate asistente comienza. 

		- En el asistente, seleccione la ubicación de almacenamiento: "Ordenador local". Haga clic en "Siguiente". 

		- Seleccione "Colocar todos los certificados en la siguiente tienda".. 

		- Haga clic en el botón "Examinar" y seleccione la tienda "Trust root certification authorities".. 

		- Haga clic en el botón "Siguiente" hasta el final de asistente. Aparece un mensaje indicando que la importación se ha realizado correctamente.. 




- En el asistente, marque "El Certificate fue instalado con éxito" y vaya al siguiente paso. 

17. Se visualiza la pantalla para generar la aplicación. En esta pantalla, tiene la posibilidad de declarar una extensión "windows.protocol".. Para iniciar la aplicación se utiliza una extensión de "protocolo Windows":

	- a través de un comando Line (presione Windows + R simultáneamente y ejecute el intérprete de comandos escribiendo "CMD").

	- desde un navegador. 
			To declare a "windows.protocol" extension: 

		- Haga clic en el botón "Avanzado",

		- En la ventana que se abre, introduzca el nombre que se debe dar al protocolo. El comando Line y el URL para los navegadores se crean automáticamente.




- Por ejemplo, si el protocolo se llama "wdorganizer", se generarán las siguientes sintaxis:

	- para un comienzo en el comando Line: "start wdorganizer:"

	- para empezar desde un navegador: " wdorganizer: "<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=W10_Protocole.gif)

18. Validar la generación de la aplicación Universal Windows 10 App.  

19. El implementación asistente se pone en marcha y propone seleccionar la plataforma implementación: 

	- el ordenador local. 

	- un Phone conectado en USB. 

	- una red Phone. 
			Observación: La aplicación generada puede instalarse en una tableta de Windows 10. Para obtener más información, consulte [Cómo instalar una aplicación en una tableta](../Editeurs/1410086579.md). 







Para más detalles sobre la implementación asistente, véase [Aplicación Universal de Windows 10: Plataforma de implementación](../Editeurs/1410086578.md).

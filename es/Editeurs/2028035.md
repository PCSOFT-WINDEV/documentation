
## Condiciones para la actualización automática de una aplicación (Live update)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000155"></a>
Cada vez que se modifica la aplicación en el ordenador de desarrollo (modificaciones de ventanas, código, análisis,...), se debe recrear el ejecutable e instalar una actualización de la aplicación en los ordenadores del usuario.

En el caso de una instalación individual, todo lo que tienes que hacer es instalar la actualización en el ordenador del usuario.

Para una "red" instalación, la instalación de la actualización en los ordenadores de los usuarios puede ser obligatoria u opcional. De hecho, si las modificaciones no afectan a la estructura de los ficheros de datos, los usuarios pueden seguir trabajando con la versión ya instalada.

Esta página de ayuda presenta **la actualización de una aplicación instalada en la red (actualización en vivo)**.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cuándo es obligatoria una actualización?
<a name="¿cuando_obligatoria_una_actualizacion_ELTTEXTE000179"></a>
La actualización es obligatoria:

- si el análisis fue modificado.

- para que todos los usuarios puedan utilizar la misma versión de la aplicación.

- si se han añadido nuevas características esenciales.




**Observación**: Si se utiliza una nueva versión de WINDEV para crear el ejecutable de su aplicación, le recomendamos que haga obligatoria la actualización.

<a name="NOTE3_1"></a>


## reference ejecutable para comprobar las versiones
<a name="reference_ejecutable_para_comprobar_las_versiones_ELTTEXTE000203"></a>
Este paso de la asistente permite especificar la versión de la aplicación desde la cual es obligatorio instalar la instalación.

Se debe especificar la siguiente información:

- ***el número de versión actual***: este número corresponde al número de versión del ejecutable que se instalará. Este número es recuperado automáticamente por el programa de creación de instalación asistente.
	Observación: El número de versión se especifica cuando se crea el ejecutable.. Este número puede incrementarse automáticamente cada vez que se crea el ejecutable.

- ***el número mínimo de versión permitido para operar sin actualización***: este número corresponde al número de versión del ejecutable que se encuentra en los equipos del usuario desde los que la actualización es opcional. 
	Por Default, este número corresponde al número de versión actual.




Instalación de la actualización:

- ***será obligatoria*** si la versión del ejecutable instalado en los equipos de los usuarios es **anterior**a la versión mínima permitida para funcionar sin la actualización.

- ***no será obligatoria*** si la versión del ejecutable instalado en los equipos de los usuarios es **igual o posterior**a la versión mínima permitida para funcionar sin la actualización.




Para personalizar la ventana de actualización de su aplicación, haga clic en el botón "Avanzado. Es posible:

- introduzca las modificaciones realizadas en la aplicación para esta actualización (historial de modificaciones),

- configurar las opciones de [Comprobación remota de los ejecutables de WINDEV](../WDAdminExe/2028052.md),

- modificar las condiciones de aplicación de la actualización. 
	Para informar a los usuarios de que una actualización de la aplicación está disponible, se muestra una ventana cuando se inicia la aplicación.. Esta ventana permite al usuario instalar (o no) la actualización. Las condiciones para aplicar la actualización le permiten configurar esta ventana.

## Instalación de la actualización en los equipos de los usuarios
<a name="instalacion_actualizacion_los_equipos_los_usuarios_ELTTEXTE000227"></a>
Cuando se inicia la aplicación:

- ***si la actualización es obligatoria***: Los usuarios tendrán que instalar esta actualización. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=FenMiseAJourOblig.gif)


- ***si la actualización es opcional***: Los usuarios podrán elegir si desean instalar la actualización o iniciar la aplicación directamente.. Esta actualización se propondrá en los próximos arranques de la aplicación.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=FenMiseAJour.gif)
 Esta ventana se muestra en el lenguaje de ejecución del programa instalación. Sólo el inglés y el francés están disponibles en Default. Para utilizar otro idioma, el proyecto WDSetup debe ser traducido. Para obtener más información, consulte [WDSetup: Gestión de idiomas adicionales](../Editeurs/2028037.md).



<a name="NOTE4_2"></a>


### Ejemplo
<a name="ejemplo_ELTPARAGRAPHE000083"></a>

En enero de 2020, se instaló la versión 1.01A de su aplicación en los ordenadores de los usuarios finales.

En febrero de 2020, se modificó la description de un archivo de datos y se creó la versión 1.01B. La actualización de esta nueva versión es obligatoria.

Posteriormente, en marzo de 2020, se modificó el aspecto de las ventanas y se creó la versión 1.01C. La actualización de esta versión es opcional.

Cuando la aplicación se inicia en los ordenadores de los usuarios:

- los usuarios que estén utilizando la versión 1.01A tendrán que instalar necesariamente la actualización.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=MAJAppliOblig.gif)


- los usuarios que utilicen la versión 1.01B podrán elegir si quieren instalar la actualización o iniciar directamente la aplicación.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=MAJAppliNonOblig.gif)

### automatic comprueba las actualizaciones disponibles
<a name="automatic_comprueba_las_actualizaciones_disponibles_ELTPARAGRAPHE000103"></a>

La aplicación comprueba regularmente si hay una actualización de la red disponible. 
Recordatorio: la hora de comprobación puede ser set desde WDAdminExe o con [AppChangeParameter](../WDLang1/1000017260.md). 

Si hay una actualización disponible, se mostrará una Notification de Windows sólo si el usuario está utilizando la aplicación.

- **Para las actualizaciones opcionales**, el mensaje Notification es el siguiente: 
	Una actualización opcional para '&lt;nombre de la aplicación&gt;' está disponible. Para instalar esta actualización, cierre y reinicie la aplicación.

- **Para las actualizaciones obligatorias**, el mensaje Notification es el siguiente: 
	Existe una actualización obligatoria para '&lt;nombre de la aplicación&gt;'.. Para instalar esta actualización, cierre y reinicie la aplicación.


Estos mensajes no se pueden modificar ni configurar. 

La comprobación de las actualizaciones disponibles es siempre Active. 
**Observaciones**: 

- Si el usuario ha elegido no instalar una actualización opcional al iniciar la aplicación, esa versión no será propuesta por el mecanismo de Notification. 

- Las notificaciones de actualización obligatoria se muestran cada 5 minutos si la actualización no se ha instalado. 

- Las notificaciones opcionales de actualización no se vuelven a mostrar si la actualización no está instalada. Sólo aparecerá la próxima vez que se inicie la aplicación.






Gestión de las notificaciones: 

- Si la aplicación tiene un icono en el área de Notification, el icono se utilizará para resaltar el Notification. 

- Si la aplicación no tiene un icono en el área de Notification, se creará un icono para resaltar el Notification. Se utilizará un Icon diferente según si la actualización es opcional u obligatoria.


[AppDetectUPDAvailable](../WDLang1/1410087705.md) permite personalizar la comprobación de actualizaciones de automatic.

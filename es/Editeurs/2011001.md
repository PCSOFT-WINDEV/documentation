## Enlace paralelo
<a name="enlace_paralelo_ELTTEXTE000271"></a>




### Definición
<a name="definicion_ELTPARAGRAPHE000036"></a>

Para una relación paralela, **cada registro** de un archivo de datos ("Producto") está relacionado con **un registro** en otro archivo de datos ("Información"), y **viceversa**.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LiaisonParallele.gif)


Para gestionar un enlace paralelo, el identificador del archivo de datos "Producto" se copia en el archivo de datos "Información".. Este identificador también es una clave única en el archivo de datos "Info".

El archivo de datos del "Producto" es el archivo del propietario. El archivo de datos "Info" es el archivo de miembros.
<a name="NOTE2_2"></a>




### Observaciones
<a name="observaciones_ELTPARAGRAPHE000050"></a>

- Este tipo de enlace es raro porque los dos archivos de datos se pueden agrupar en un solo archivo.

- El paralelismo de los registros en el archivo de datos se respeta si las siguientes operaciones se ejecutan de forma simultánea en los dos archivos de datos:

	- creando un Record

	- borrar un Record

	- reindexación con compresión

## Enlace opcional
<a name="enlace_opcional_ELTTEXTE000301"></a>




### Definición
<a name="definicion_ELTPARAGRAPHE000065"></a>

Para un enlace opcional, **e cada Record** que se encuentra en un archivo de datos ("Categoría") se asocia ** con ningún Record o con un solo Record** en otro archivo de datos ("Grupo"). **Cada Record** del otro archivo de datos ("Grupo") está asociado a ** con ningún Record o con un solo Record** en el primer archivo de datos ("Categoría").
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LiaisonFacultative.gif)


Un enlace opcional se realiza copiando el identificador de cada archivo de datos en el otro archivo de datos.

## Enlace de complemento
<a name="enlace_complemento_ELTTEXTE000325"></a>




### Definición
<a name="definicion_ELTPARAGRAPHE000079"></a>

Para un enlace de complemento, **e cada Record** que se encuentra en un archivo de datos ("Producto") se asocia ** con ningún Record o con un solo Record** en otro archivo de datos ("Detalles"). **Cada registro** del otro archivo de datos ("Detalles") está **obligatoriamente asociado con un registro** en el primer archivo de datos ("Producto").
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Liaisoncomplement.gif)


Las relaciones de complemento son bastante comunes. Se utilizan cuando un registro puede tener información adicional opcional.

Para gestionar un enlace de complemento, la clave del archivo de datos "Producto" se copia en el archivo de datos "Detalles".. Para asegurar que la cardinalidad máxima sea 1, sigue siendo una clave única. Como esta llave es única, no es posible insertar más de un Record en "Detalles" para un Record en "Producto".

El archivo propietario es "Producto", el archivo miembro es "Detalles".

## Relación compartida
<a name="relacion_compartida_ELTTEXTE000349"></a>




### Definición
<a name="definicion_ELTPARAGRAPHE000097"></a>

Para una relación compartida, el **mismo registro** de un archivo de datos ("Proveedor") puede estar **compartido por varios registros** en otro archivo de datos ("Producto").

Para gestionar un enlace compartido, la clave del archivo de datos del "Proveedor" se copia en el archivo de datos del "Producto".. Esta se convierte en una clave múltiple para optimizar la velocidad al verificar la integridad.

El archivo propietario es "Proveedor", el archivo miembro es "Producto".

Dependiendo de la cardinalidad, se pueden distinguir cuatro tipos de relaciones compartidas:

- Relación compartida con una cardinalidad 0,n - 0,1

- Relación compartida con una cardinalidad 0,n - 1,1

- Relación compartida con una cardinalidad 1,n - 0,1

- Relación compartida con una cardinalidad 1,n - 1,1



<a name="NOTE5_2"></a>




### Enlace compartido con una cardinalidad 0,N - 0,1
<a name="enlace_compartido_con_una_cardinalidad_0n_01_ELTPARAGRAPHE000116"></a>

Con este tipo de relación:

- un propietario puede no tener ningún miembro (un proveedor no necesariamente tiene un producto).

- un miembro puede no tener un propietario (un producto no necesariamente tiene un proveedor).



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LiaisonPartagee_0N01.gif)

<a name="NOTE5_3"></a>




### Enlace compartido con una cardinalidad de 0,N - 1,1
<a name="enlace_compartido_con_una_cardinalidad_0n_11_ELTPARAGRAPHE000129"></a>

Con este tipo de relación:

- un propietario puede no tener ningún miembro (un proveedor puede no tener ningún producto).

- cada miembro tiene un único propietario (cada producto tiene un único proveedor).



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LiaisonPartagee_0N11.gif)


**Observación**: Este tipo de relación es bastante común. WINDEV le permite crear automáticamente las ventanas utilizadas para gestionar los ficheros de datos relacionado mediante un enlace 0,N - 1,1.
<a name="NOTE5_4"></a>




### Enlace compartido con una cardinalidad de 1,N - 0,1
<a name="enlace_compartido_con_una_cardinalidad_1n_01_ELTPARAGRAPHE000144"></a>

Con este tipo de relación:

- cada propietario tiene al menos un miembro (un proveedor tiene al menos un producto).

- un miembro puede no tener propietario (un producto puede no tener proveedor).



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LiaisonPartagee_1N01.gif)

<a name="NOTE5_5"></a>




### Enlace compartido con una cardinalidad 1,N - 1,1
<a name="enlace_compartido_con_una_cardinalidad_1n_11_ELTPARAGRAPHE000157"></a>

Con este tipo de relación:

- cada propietario tiene al menos un miembro (cada proveedor tiene al menos un producto).

- cada miembro tiene un único propietario (cada producto tiene un único proveedor).



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LiaisonPartagee_1N11.gif)

## Relación compleja
<a name="relacion_compleja_ELTTEXTE000397"></a>


### Definición
<a name="definicion_ELTPARAGRAPHE000171"></a>

Para un enlace "complejo", debe gestionar un archivo de enlace llamado **archivo de relación**.

El archivo de relación se creará automáticamente. Este contendrá una clave única compuesta por las dos claves de los archivos de datos relacionados.

El archivo de relación también puede contener información específica de la relación.

Una relación compleja incluye dos relaciones compartidas.
<a name="NOTE6_2"></a>




### Ejemplo de enlace complejo
<a name="ejemplo_enlace_complejo_ELTPARAGRAPHE000185"></a>

Un pedido (fichero"Pedidos") puede contener uno o varios productos. Un producto (ficha"Producto") puede ser utilizado en varios pedidos.

En este caso, se requiere un archivo de enlace (archivo"OrderLine").

El archivo "OrderLine" contiene:

- una clave única compuesta por las claves de Product y Orders.

- el número de productos pedidos.




## Ejecutar la prueba de una reporte
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000202"></a>
La prueba de una sola reporte le permite ejecutar la reporte actual en WINDEV, WEBDEV o WINDEV Mobile.. Al igual que con la prueba del proyecto, el depurador puede iniciarse si se produce un problema.

**Observación**: La prueba de la reporte se ejecuta sólo en el ordenador de desarrollo. Los datos utilizados para la prueba corresponden a los datos encontrados en el ordenador de desarrollo.

Los siguientes eventos WLanguage se ejecutan cuando se prueba una reporte:

- Inicialización del proyecto.

- Eventos associated con el reporte.

- Eventos associated con los bloques del reporte.

- Eventos associated con los controles del reporte.




Para más detalles, véase [Secuencia de eventos associated con informes](../WDChamp/1011004.md), [Eventos associated con un reporte](../WDChamp/1011037.md), [Eventos associated con los bloques de un reporte](../WDChamp/1011042.md) y [Eventos associated con los controles de un reporte](../WDChamp/1013047.md).

**Observación**: La prueba del reporte está disponible para los informes basados en una consulta, un archivo de datos, un archivo de texto y para los informes associated sin fuente de datos. Para los demás tipos de informes, debe ejecutar la prueba del elemento (ventana, Page o proyecto) desde el que se imprime el reporte.|   |   |   |

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000244"></a>


### Prueba de la actual reporte
<a name="prueba_actual_reporte_ELTPARAGRAPHE000049"></a>

Para probar la actual reporte: 

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_GO_Fenetre_WD_bl.gif) en los botones de acceso rápido (tecla F9).

2. Una pantalla permite elegir el modo de impresión reporte: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Go_Etat_Choix.gif)
 Las opciones disponibles son:

	- **Visor de reportes**: se utiliza para obtener una vista previa de la reporte a medida que se imprime.

	- **Impresora**: el reporte se imprime en la impresora predeterminada.

	- **Archivo RTF**: el reporte se guarda en un archivo RTF y se muestra en un editor de texto (Word por ejemplo).

	- **Archivo HTML**: la reporte se guarda en un archivo HTML y se visualiza en el navegador de Internet instalado en la el equipo.

	- **Archivo PDF**: el reporte se guarda en un archivo PDF y se muestra en un visor de PDF (Acrobat Reader por ejemplo).

	- **Archivo PDF Genérico**: el reporte se guarda en un archivo PDF genérico (independiente de la impresora) y se muestra en un visor de PDF (Acrobat Reader por ejemplo).


Para más detalles sobre estos diferentes modos, véase [Modos de impresión de un reporte](../WDChamp/1011031.md).

3. Seleccione el modo de impresión y valide. Si su reporte no se guarda, WINDEV, WEBDEV o WINDEV Mobile propone automáticamente guardarla bajo el nombre dado en la creación de reporte asistente.

4. Especifique la información adicional si es necesario:

	- Si ***el reporte tiene límites***, se abre una ventana para introducir los límites del reporte. Para obtener más información, consulte [reporte encuadernada](../WDChamp/1011052.md).

	- Si ***el reporte se basa en una consulta con parámetros*** , una ventana le permite ingresar los parámetros de la consulta. Para obtener más información, consulte [Reporte a partir de una consulta con parámetros](../WDChamp/1011013.md).

	- Si ***el reporte espera parámetros***, indique un valor predeterminado para los parámetros en la declaración de parámetros. Para obtener más información, consulte [Pasar parámetros a un reporte](../WDChamp/1011008.md).





Observación: La última prueba realizada (ventana, consulta, reporte, etc.) puede volver a ejecutarse desde cualquier elemento del proyecto a través de la opción "GO .. Volver a ejecutar la última prueba" en los botones de acceso rápido. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GO_Relancer_le_dernier_test%20-%20HC%20N%B0001.gif)

### Observación: Minimizar (o no) el editor durante la prueba
<a name="observacion_minimizar_editor_durante_prueba_ELTPARAGRAPHE000275"></a>

Durante una prueba, el editor es minimizado por Default. Dependiendo de la configuración del hardware (pantalla ancha, 2 pantallas, etc.), puede que no sea necesario minimizar el editor. 

Para mantener abierto el editor: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones del editor de código". 

2. En la pestaña "Depurador", deseleccione la opción "Minimizar editor durante el modo de prueba". 

3. Validar. 



<a name="NOTE2_3"></a>


### Detener la prueba de una reporte
<a name="detener_prueba_una_reporte_ELTPARAGRAPHE000101"></a>

Existen varios métodos para detener la prueba:

- **método 1**: Cierre el visor de la reporte que muestra a la reporte siendo probada. Aparece el editor actual en el momento en que se inició la prueba.

- **método 2**: Vuelve al editor con la barra de tareas (o presiona Alt + Tab) y confirma el final de la prueba. Aparece el editor actual en el momento en que se inició la prueba.

- **Método 3**: Vuelva al editor a través de la barra de tareas o con Alt + Tab y haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Partie2_PileDesAppels%202.gif) ("Finalizar la prueba") en el panel "Depurador".

## Notas
<a name="notas_ELTTEXTE000308"></a>


### Directorio de los archivos de datos utilizados para las pruebas
<a name="directorio_los_archivos_datos_utilizados_para_las_pruebas_ELTPARAGRAPHE000116"></a>

Para especificar los archivos de datos utilizados al ejecutar la prueba de la aplicación en el editor:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. En la pestaña "Archivos", seleccione la ubicación de los archivos de datos de prueba:

	- ***Directorio de la configuración actual o directorio actual*** ( Default ): Los archivos de datos se ubicarán en el subdirectorio correspondiente a la configuración actual (subdirectorio del directorio EXE del proyecto).

	- ***Directorio específico***: Los archivos de datos se ubicarán en el directorio especificado.






<a name="NOTE3_2"></a>


### Gestión de contraseñas (análisis o archivo de datos)
<a name="gestion_contrasenas_analisis_archivo_datos_ELTPARAGRAPHE000145"></a>

No se solicita contraseña cuando se ejecuta la prueba de un reporte relacionado a un archivo de datos con contraseña protected.

Las contraseñas necesarios se debe especificar mediante programación en el evento "Apertura" del reporte:

- Si el reporte es relacionado a uno o más archivos de datos protegidos por contraseña, la contraseña debe especificarse con [HPass](../WDLang4/3044108.md).

- Se produce un error de WLanguage durante la prueba si no se especifican las contraseñas en la Event "Abrir" de la reporte.

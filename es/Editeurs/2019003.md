


## Prueba de sitio: los diferentes tipos de pruebas
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000114"></a>
WEBDEV ofrece varios métodos para probar sus sitios:

- **Prueba de todo el proyecto**:
	La prueba de todo el proyecto permite simular la puesta en marcha del sitio. Esto permite probar todo el sitio, incluso si su desarrollo aún no está completo. 
	Para obtener más información, consulte: 

	- [Probar un proyecto dinámico](../Editeurs/2019009.md)

	- [Probar un proyecto estático](../Editeurs/2019013.md)




- **Prueba de esfuerzo**:
	La herramienta WDTestSite se utiliza para realizar pruebas de esfuerzo para un sitio dinámico.. WDTestSite le permite establecer varias conexiones simultáneas a un sitio WEBDEV dinámico. 
	Para obtener más información, consulte: [WDTestSite](../WDTestSite/3538003.md).

- **Prueba de una sola página**: 
	La prueba de una sola página le permite ejecutar la página actual en WEBDEV. Esto le permite probar su proyecto a partir de una página determinada, o comprobar si una página se ejecuta correctamente una vez finalizado su desarrollo.
	Para obtener más información, consulte: 

	- [Probar una página dinámica (en modo Sesión o en modo AWP)](../Editeurs/2019016.md).

	- [Probar una página estática](../Editeurs/2019006.md).




- [Prueba de una consulta única](../Editeurs/2019011.md):
	Probar una sola consulta le permite ejecutar la consulta actual en WEBDEV. Esto permite comprobar el modo de funcionamiento de una consulta una vez desarrollada.

- [Prueba de una sola reporte](../Editeurs/2019015.md):
	Probar una sola reporte le permite ejecutar la reporte actual en WEBDEV. Esto permite probar un reporte una vez que se ha desarrollado. El depurador se puede iniciar tan pronto como se produzca un problema.

- [Probar el proyecto paso a paso](../Editeurs/2019001.md): 
	Probar el proyecto paso a paso le permite iniciar el depurador al lanzar el sitio. Esta solución le permite monitorizar cómo se ejecutan los códigos del servidor del sitio.

- [Prueba y depuración de un sitio encontrado en una servidor de aplicaciones](../Editeurs/2019032.md):
	Puede probar y depurar un sitio remoto presente en cualquier servidor de aplicaciones. Por lo tanto, el sitio se depura directamente en el servidor activo.

- [Adjuntar a una sesión que se ejecuta actualmente en un servidor de aplicaciones](../Editeurs/2019032.md):
	Puede depurar una sesión que se esté ejecutando en un sitio remoto.

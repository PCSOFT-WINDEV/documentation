## Presentación
<a name="presentacion_ELTTEXTE000470"></a>
Una presentación permite definir varias vistas de una Page sin duplicarla. 

Mediante el uso de diseños en un proyecto WEBDEV, puede utilizar muchas características: 

- crear sitios "aptos para móviles. En este caso, puede definir dos diseños: 

	- una vista específica para el modo de escritorio,

	- una vista específica para el modo móvil, etc.. 




- imprimir una versión "limpia" de la Page. En este caso, puede crear un diseño específico para imprimir el contenido de Page.

- Integrar una Page en una aplicación WINDEV o WINDEV Mobile. En este caso, puede crear un diseño específico para integrar el Page en una ventana. 




Importante: **En un proyecto WEBDEV, los diseños son otra forma de hacer un sitio optimizado para dispositivos móviles**. A diferencia del modo de diseño web receptivo (RWD), los diseños no tienen restricciones en el orden de los controles.
Por lo tanto, los diseños le permiten: 

- hacer diseños que no se pueden hacer fácilmente en RWD. 

- crear sin esfuerzo elementos en un diseño y esconderlos en otro diseño.


Para más información sobre las ventajas de los diseños para sitios web adaptados a dispositivos móviles, consulte [Sitio web adaptado a dispositivos móviles: métodos disponibles](../Editeurs/1410086641.md).

## Creación de un diseño en WEBDEV
<a name="creacion_diseno_webdev_ELTTEXTE000494"></a>


### Creación de un nuevo diseño para un sitio WEBDEV Page (Mobile-friendly website)
<a name="creacion_nuevo_diseno_para_sitio_webdev_page_mobilefriendly_website_ELTPARAGRAPHE000070"></a>

Para crear un nuevo diseño para una Page: 

1. Abra la Page correspondiente en el editor. 

2. En la pestaña "Página", en el grupo "Layouts", despliegue "Layouts" y seleccione "Agregar layout a partir de la vista activa".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page%20-%20HC%20N%B0001.gif&type=thumb)


3. Aparece la ventana de creación del diseño: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page%20-%20HC%20N%B0002.gif&type=thumb)


4. Especificar el tipo de pantalla para el que debe crearse la disposición. También se puede especificar el ancho mínimo de la pantalla. 

5. Validar. 

6. A la derecha, el editor muestra las diferentes disposiciones disponibles para Page. Haciendo doble clic en la insignia del diseño deseado se puede abrir en el editor y modificarlo. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page%20-%20HC%20N%B0003.gif&type=thumb)





**Atención**: Los diseños no están disponibles en las plantillas internas Page s, Page s o Page s en modo Responsive Web Design.

### Creación de un nuevo diseño para imprimir una Page
<a name="creacion_nuevo_diseno_para_imprimir_una_page_ELTPARAGRAPHE000115"></a>

Para crear un nuevo diseño para imprimir el contenido de una Page: 

1. Abra la Page correspondiente en el editor. 

2. En la pestaña "Página", en el grupo "Layouts", despliegue "Layouts" y seleccione "Añadir un diseño de impresión".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_impression%20-%20HC%20N%B0001.gif&type=thumb)


3. Se crea automáticamente un diseño "Imprimir. A la derecha, el editor muestra las diferentes disposiciones disponibles para Page. Puede hacer doble clic en el diseño "Imprimir" para abrirlo y modificarlo en el editor. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_impression%20-%20HC%20N%B0003.gif&type=thumb)
 Esto permite mantener sólo los controles necesarios para la impresión en esta disposición. 




**Observaciones**

- Los diseños no están disponibles en las plantillas internas Page s, Page s o Page s en modo Responsive Web Design.

- El diseño de impresión se utilizará automáticamente cuando el usuario imprima la correspondiente Page. Por ejemplo, los controles innecesarios (botones, enlaces, etc.) pueden ocultarse en el diseño de impresión.

### Creación de un nuevo diseño para integrar una Page WEBDEV en una ventana WINDEV
<a name="creacion_nuevo_diseno_para_integrar_una_page_webdev_una_ventana_windev_ELTPARAGRAPHE000148"></a>

Para crear un nuevo diseño para integrar una Page WEBDEV en una ventana WINDEV: 

1. Abra la Page correspondiente en el editor. 

2. En la pestaña "Página", en el grupo "Layouts", despliegue "Layouts" y seleccione "Añadir diseño para integrar Page en WINDEV".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WB_dans_WD%20-%20HC%20N%B0002.gif&type=thumb)


3. Se crea automáticamente un diseño "Para WINDEV. A la derecha, el editor muestra las diferentes disposiciones disponibles para Page. Para editar la presentación "Para WINDEV", haga doble clic en su miniatura. Esta disposición permite conservar en la aplicación WINDEV únicamente los controles necesarios para la visualización de Page.  




Para obtener más información, consulte [Integrar un sitio WEBDEV en una aplicación WINDEV](../Editeurs/1410086640.md). 

**Observaciones**

- Los diseños no están disponibles en las plantillas internas Page s, Page s o Page s en modo Responsive Web Design.

- La maquetación para aplicaciones WINDEV se utilizará automáticamente cuando la página se integre en una "Página WEBDEV" control en el proyecto WINDEV.

### Creación de un nuevo diseño para integrar una Page WEBDEV en una ventana WINDEV Mobile
<a name="creacion_nuevo_diseno_para_integrar_una_page_webdev_una_ventana_windev_mobile_ELTPARAGRAPHE000184"></a>

Para crear un nuevo diseño para integrar una WEBDEV Page en una ventana WINDEV Mobile: 

1. Abra la Page correspondiente en el editor. 

2. En la pestaña "Página", en el grupo "Layouts", despliegue "Layouts" y seleccione: 

	- "Agregar un layout para integrar la página en WINDEV Mobile (Teléfono)".

	- "Agregar un layout para integrar la página en WINDEV Mobile (Tableta)".




3. Se crea automáticamente un diseño "WINDEV Mobile (Tablet)" o "WINDEV Mobile (Phone). A la derecha, el editor muestra las diferentes disposiciones disponibles para Page. Puede hacer doble clic en la miniatura de la maqueta de WINDEV Mobile para abrirla y editarla. Esta disposición permite conservar en la aplicación WINDEV Mobile únicamente los controles necesarios para visualizar la Page.




**Observaciones**

- Los diseños no están disponibles en las plantillas internas Page s, Page s o Page s en modo Responsive Web Design.

- La maquetación para aplicaciones WINDEV se utilizará automáticamente cuando la página se integre en una "Página WEBDEV" control en el proyecto WINDEV.

## Uso de diseños en el editor
<a name="uso_disenos_editor_ELTTEXTE000536"></a>


### Ver dos diseños simultáneamente en el editor
<a name="ver_dos_disenos_simultaneamente_editor_ELTPARAGRAPHE000240"></a>

En Default, sólo se puede ver en el editor Page la maqueta que se está editando en ese momento. 

Para ver (y editar) dos diseños simultáneamente: 

1. Haga clic con el botón derecho en la miniatura de la maqueta que desea abrir. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_MenuCtx.gif)


2. Seleccione "Editar (vista dividida)". 

3. Ambos diseños se muestran en el editor. 





### Gestión de controles en los layouts
<a name="gestion_controles_los_layouts_ELTPARAGRAPHE000252"></a>

Las maquetas se disocian mediante Default (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_manip%20-%20HC%20N%B0001.gif)): 

- Una control creada en una maqueta se crea en todas las maquetas, con las mismas características (tamaño, posición, etc.). 

- Una control desplazada o redimensionada en una presentación sólo se desplaza o redimensiona en esa presentación. 

- La disociación automatic está habilitada por Default para todos los controles. Con este modo, cuando se desplaza o redimensiona una control en una de las páginas de la maqueta, los cambios no se aplicarán en las demás páginas. 




Si desea que los cambios de tamaño o posición se apliquen en todas las demás maquetas, puede desactivar la disociación automatic (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_manip%20-%20HC%20N%B0002.gif)): los próximos cambios de tamaño o posición se aplicarán en todas las maquetas. Puede aparecer un triángulo rojo cuando se selecciona una control. Indica que la control ha sido disociada, y que cualquier cambio realizado en ella se aplicará sólo en la disposición actual. 

**Consejo**: Los mensajes Toast aparecen regularmente para recordarle si los cambios realizados en un control se aplican en otras maquetas, en función de las opciones de disociación control.

### Menú contextual de controles específicos de los diseños
<a name="menu_contextual_controles_especificos_los_disenos_ELTPARAGRAPHE000271"></a>

El menú contextual "Layouts" también permite disociar características específicas de los controles. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_manip%20-%20HC%20N%B0003.gif)
. Las opciones son las siguientes:  

- **Disociar posición**: Si se selecciona esta opción, la control sólo se desplazará en la disposición actual. Si no se selecciona esta opción, la control se desplazará en todas las disposiciones. 

- **Disociar tamaño**: Si se selecciona esta opción, la control se redimensionará sólo en la disposición actual. Si no se selecciona esta opción, la control se redimensionará en todas las disposiciones.  

- **Disociar anclaje**: Si se selecciona esta opción, los anclajes de la control sólo se modificarán en la disposición actual. Si no se selecciona esta opción, los anclajes de la control se modificarán en todas las disposiciones. 

- **Disociar estilo**: Si se selecciona esta opción, los cambios realizados en el estilo de la control se aplicarán sólo en la presentación actual. Si no se selecciona esta opción, los cambios realizados en el estilo de la control se aplicarán en todas las maquetas.

- **Disociar contenido enriquecido**: Si se selecciona esta opción, el contenido enriquecido de la control sólo se modificará en la presentación actual. Si no se selecciona esta opción, el contenido enriquecido de la control se modificará en todas las disposiciones.




Observación: Cuando se desactiva el modo de disociación automatic, y se desactiva una disociación específica a través del menú contextual, los cambios realizados en la maqueta actual se automatic aplicarán en todas las maquetas. Por ejemplo, si la opción "Disociar posición" se activó y luego se desactivó para una control determinada, el siguiente cambio realizado en la posición de la control se aplicará en todas las disposiciones. 

El menú contextual "Layouts" incluye otras opciones: 

- **Editar propiedades disociadas**: Muestra la lista de propiedades disociadas para la control seleccionada , con los valores correspondientes para la disposición actual. Puede cambiar algunos valores del diseño actual. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_manip%20-%20HC%20N%B0004.gif&type=thumb)


- **Visibilidad por layout**: Muestra u oculta la control seleccionada en determinados diseños. 

- **Ocultar en este layout**: Oculta la control seleccionada en la presentación actual. 

- **Mostrar en todos los layouts**: Muestra la control actual en todas las disposiciones. 

- **Mostrar controles invisibles**: Muestra los controles ocultos en la presentación actual. Estos controles aparecen Grayed fuera. Desaparecen la próxima vez que se visualiza el diseño. 




Observación: Este menú contextual está disponible para una selección de controles. 

Limitación: Para los controles de Gráfico, las propiedades del título y la leyenda no pueden disociarse entre diseños.

### Menú contextual de diseños
<a name="menu_contextual_disenos_ELTPARAGRAPHE000302"></a>

Puedes gestionar los diseños a través del menú contextual de las miniaturas.

Usted puede: 

- Ver y editar el diseño description. Para obtener más información, consulte [Ajustes de diseño](#NOTE4_1).

- Ver la lista de controles disociados. 

- Abrir un diseño para editarlo. 

- Editar la presentación en el modo de vista dividida: los dos diseños pueden visualizarse y editarse simultáneamente. 

- Crear un nuevo diseño a partir del diseño actual.

- Desactivar este layout. Aparece una "X" roja en la maqueta Icon y la maqueta no puede utilizarse en tiempo de ejecución.. Sin embargo, seguirá estando disponible en el editor. 
	Observación: Esta opción puede ser useful cuando un Page hereda una o más plantillas Page con diseños. Esto le permite mantener sólo los diseños deseados.

- Borrar el layout seleccionado. 

- Mostrar miniaturas proporcionales.

- Actualizar automáticamente: Se utiliza para actualización la disposición tan pronto como se realiza una modificación en la ventana..  

- Actualizar este layout: Se utiliza para actualización inmediatamente la disposición. 

- Actualizar todo: Utilizado para actualización inmediatamente todos los layouts.

### Controles específicos (sitios web aptos para móviles)
<a name="controles_especificos_sitios_web_aptos_para_moviles_ELTPARAGRAPHE000336"></a>

Algunos controles avanzados (Looper, control Tabla, Menú horizontal, etc.) requieren acciones más específicas que la simple disociación del tamaño o los anclajes. En este caso, puede hacer clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_manip%20-%20HC%20N%B0006%201.gif) para gestionar las opciones que pueden disociarse. Esto le permite set un valor específico para cada diseño, si es necesario. 

**Para control Tabla** , puede agrupar columnas verticalmente para adaptar tabla a dispositivos móviles. 

**Para control Looper** , puede set un número específico de columnas por diseño. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_manip%20-%20HC%20N%B0006.gif&type=thumb)


**Para los menús manipular** , WEBDEV le proporciona un control específico para diseños: el control Barra de navegación. El control Barra de navegación permite crear un área de menú que puede adaptarse fácilmente al tamaño del navegador. Esta área puede contener todo tipo de controles. Las características de este control también pueden definirse desde la ventana description.

## Gestionar los diseños en tiempo de ejecución
<a name="gestionar_los_disenos_tiempo_ejecucion_ELTTEXTE000590"></a>


### Modo de funcionamiento Default
<a name="modo_funcionamiento_default_ELTPARAGRAPHE000379"></a>

La disposición inicial se selecciona automáticamente en función de: 

- el tamaño del navegador,

- el tamaño de la Page (en el editor).




El diseño se modifica automáticamente en función del tamaño del navegador. 

Los ajustes de la maqueta pueden definirse en función del tamaño del navegador ("description" en el menú contextual de la barra de título de la maqueta).  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Page_manip%20-%20HC%20N%B0005.gif&type=thumb)


Estos parámetros se tienen en cuenta para elegir la disposición en tiempo de ejecución. 

Observación: Si la disposición elegida no es adecuada, puede forzar el uso de una disposición determinada mediante programación. 
<a name="NOTE4_2"></a>


### Caso especial: Diseño de impresión
<a name="caso_especial_diseno_impresion_ELTPARAGRAPHE000400"></a>

Si se ha creado un diseño "Imprimir" en un Page, este diseño se utilizará automáticamente cuando el usuario seleccione la opción "Imprimir" en el menú contextual del navegador (o pulse Ctrl + P). En este caso, sólo se imprimirán los controles mostrados en el diseño. 
<a name="NOTE4_3"></a>




## Referencias cruzadas en el editor de código
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000119"></a>
Las referencias cruzadas en el editor de código se utilizan para identificar:

- los códigos que llaman al elemento actualmente seleccionado en el editor de códigos.

- los códigos llamados por el elemento actualmente seleccionado en el editor de códigos.




Este elemento puede ser un control, una ventana, una Procedure, un método de clase, una función WLanguage , ...<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Ref_croisees.gif&type=thumb)

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000143"></a>


### Visualización de las referencias cruzadas de un elemento
<a name="visualizacion_las_referencias_cruzadas_elemento_ELTPARAGRAPHE000024"></a>

Para mostrar las referencias cruzadas de un elemento (control, ventana, Procedure, función WLanguage, ...) que se encuentran en el editor de código:

1. Sitúe el cursor del ratón sobre el nombre del elemento.

2. aprovechamiento: 

	- la combinación de teclas Mayúsculas + F2.

	- "Referencias cruzadas" en el menú contextual. 




3. Se visualiza la lista de referencias cruzadas.



<a name="NOTE2_2"></a>


### Configuración de las opciones de visualización de referencias cruzadas
<a name="configuracion_las_opciones_visualizacion_referencias_cruzadas_ELTPARAGRAPHE000038"></a>Puede configurar las opciones de visualización de las referencias cruzadas en la ventana de visualización ("Opciones de visualización").<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Ref_croisees_option.gif&type=thumb)
 Se pueden enumerar los siguientes elementos:

- ***Referencias del código fuente***: a las que estas referencias pueden corresponder:

	- los usos del elemento en el código. En este caso, la reference se lista automáticamente tan pronto como se utiliza el nombre del elemento.

	- las modificaciones del elemento únicamente. En este caso, sólo se listan las líneas de código en las que se modifica el elemento.

	- la llamada a los procedimientos (para los procedimientos locales o globales). Sólo se listan las llamadas a los procedimientos.




- ***Referencias de elemento: ventana o página interna, modelo, etc.***, es decir, las ventanas, páginas internas o modelos que utilizan el elemento.

- ***Relaciones ***, para controles relacionados con archivos de datos y elementos del análisis.

- ***Información compartida del análisis***, para controles relacionados con elementos.

- ***Enlaces desde los modelados flexibles*** (por ejemplo, si la ventana se utiliza en un modelado flexible).

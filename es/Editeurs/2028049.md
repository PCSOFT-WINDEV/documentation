


## WDInst: eligiendo el tipo de instalación para generar
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000171"></a>
WDInst le permite crear varios tipos de instalación:

- ***Instalación estándar***: Recomendado para una aplicación monopuesto.
	Todos los archivos necesarios para la aplicación se instalan en los equipos de los usuarios.
	Al actualizar la aplicación, el programa de actualización debe ejecutarse en cada ordenador de usuario.

- Red ***instalación con automatic update***: recomendado para una aplicación multiusuario (utilizado por varios usuarios simultáneamente). 
	Este tipo de instalación le permite:

	- gestionar automáticamente las actualizaciones de la aplicación.

	- conocer las características de los usuarios conectados a la aplicación en un momento dado, a través del programa "WDADMINEXE" instalado en el servidor. Para obtener más información, consulte [Comprobación remota de los ejecutables de WINDEV](../WDAdminExe/2028052.md).

	- configurar la ubicación de los archivos de la aplicación. Por ejemplo:

		- Instalar todos los archivos de la aplicación en el servidor.

		- Instalar la base de datos en el servidor y los archivos necesarios para ejecutar la aplicación (".EXE", ".WDL", ".DLL", etc.) en cada equipo cliente.

		- Instalar parte de la base de datos en el servidor y otra parte en cada ordenador de usuario. Los archivos necesarios para ejecutar la aplicación (".EXE", ".WDL", ".DLL", etc.) también se instalan en cada equipo cliente.




- ***Configuración desde un almacén privado***: recomendada para agrupar las aplicaciones utilizadas en una empresa. Este tipo de instalación permite:

	- Acceso a la configuración de las aplicaciones a través de un portal. 

	- Actualizaciones de la aplicación automatic.   




- ***Instalación compacta***: El tamaño del programa instalación generado se optimiza para ser descargado de INTERNET.




**Observación**: Por Default, la instalación generada por WINDEV corresponde a una instalación estándar .

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000195"></a>


### Definiendo el tipo de instalación a generar
<a name="definiendo_tipo_instalacion_generar_ELTPARAGRAPHE000047"></a>

El tipo de instalación a generar se especifica en WDInst. Para abrir WDInst, en la pestaña "Herramientas", en el grupo "Utilidades", haga clic en "WDInst".

La creación del programa instalación asistente también puede ser usada para definir el tipo de instalación a generar.

Tabla de resumen: [¿Cómo instalar una aplicación desarrollada con WINDEV?](../Editeurs/2028005.md)
<a name="NOTE2_2"></a>


### Generando una instalación estándar
<a name="generando_una_instalacion_estandar_ELTPARAGRAPHE000072"></a>

Para generar una instalación estándar:

1. Seleccione "Ajustes de configuración ... Asistente".

2. Marque "instalación estándar".



<a name="NOTE2_3"></a>


### Generando una instalación con actualización
<a name="generando_una_instalacion_con_actualizacion_ELTPARAGRAPHE000082"></a>

Para generar una instalación con actualización:

1. Seleccione "Ajustes de configuración ... Asistente".

2. Marque "Red instalación con actualización de automatic".

3. Selecciona el ejecutable reference para la actualización de la red.

Introduzca el número de versión de la aplicación que se encuentra en los equipos de los usuarios desde los que la actualización no será obligatoria.

**5.** Para configurar la actualización de su aplicación de manera más específica, haga clic en el botón "Opciones avanzadas". Tendrás la capacidad de hacerlo:

- especificar el historial de modificaciones.

- configurar las opciones de "control remoto de los ejecutables de WINDEV".

- modificar las condiciones de aplicación de la actualización




Para obtener más información, consulte los [Opciones avanzadas de la red instalación](../Editeurs/2028020.md).

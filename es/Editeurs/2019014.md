


## Probar una ventana
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000180"></a>
Probar una sola ventana le permite ejecutar la ventana actual en WINDEV o WINDEV Mobile. Esto le permite probar el proyecto desde una ventana determinada o probar una ventana una vez terminada.

Al igual que con la prueba del proyecto, el depurador puede iniciarse si se produce un problema.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000204"></a>


### Prueba de la ventana actual
<a name="prueba_ventana_actual_ELTPARAGRAPHE000020"></a>

Para probar la ventana actual desde el editor: 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_go_fenetre_WD_bl.gif) en los botones de acceso rápido. El editor se minimiza automáticamente y la ventana se ejecuta.




Los siguientes eventos de WLanguage se ejecutan al inicio de una prueba de ventana:

- declarando clases,

- inicialización del proyecto,

- declarando las variables globales de la ventana actual,

- inicializar los controles que se encuentran en la ventana actual,

- inicialización de la ventana actual,

- ...ajustando el foco en la ventana actual,

- entrando en el primer control de la ventana actual (si este control tiene una entrada Process).




Durante la prueba, se pueden ejecutar todas las características de la ventana. Podrá abrir otras ventanas, por ejemplo.

**Observaciones**: 

- Para ejecutar la prueba de una ventana que espera uno o más parámetros, se debe dar un valor Default a los parámetros en su declaración. De lo contrario, aparece una ventana que le permite introducir los parámetros esperados.. Para obtener más información, consulte [Probar una ventana con parámetros](../WDChamp/1010026.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Si se han realizado modificaciones en la interfaz durante la prueba de la ventana (redimensionamiento de los controles, modificación de su posición, etc.), al final de la prueba, el editor de ventanas propone modificar la interfaz de la ventana correspondiente.. Para obtener más información, consulte [Uso de la modificación de la interfaz gráfica de usuario durante una prueba de ventana](../WDLang1/1000021927.md).

- La última prueba realizada (ventana, consulta, reporte, etc.) puede volver a ejecutarse desde cualquier elemento del proyecto a través de la opción "GO .. Volver a ejecutar la última prueba" en los botones de acceso rápido. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GO_Relancer_le_dernier_test%20-%20HC%20N%B0001.gif)






### Observación: Minimizar (o no) el editor durante la prueba
<a name="observacion_minimizar_editor_durante_prueba_ELTPARAGRAPHE000235"></a>

Durante una prueba, el editor es minimizado por Default. Dependiendo de la configuración del hardware (pantalla ancha, 2 pantallas, etc.), puede que no sea necesario minimizar el editor. 

Para mantener abierto el editor: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones del editor de código". 

2. En la pestaña "Depurador", deseleccione la opción "Minimizar editor durante el modo de prueba". 

3. Validar.

### Prueba en un simulador
<a name="prueba_simulador_ELTPARAGRAPHE000074"></a>

Para las aplicaciones de Android, iOS o Windows Store, WINDEV Mobile permite probar la ventana (y la aplicación) en el ordenador de desarrollo a través del modo de simulación. Esta prueba simula un dispositivo de la plataforma utilizado en el ordenador de desarrollo. Esta prueba es useful si el desarrollador no tiene un dispositivo (Android, por ejemplo) o un ordenador Mac para compilar la aplicación iOS. Sin embargo, algunas funciones no están disponibles en modo simulador. 
<a name="NOTE2_3"></a>


### Detener la prueba de la ventana
<a name="detener_prueba_ventana_ELTPARAGRAPHE000081"></a>

Existen varios métodos para detener la prueba:

- Método 1: Cierre la aplicación que se está probando. WINDEV vuelve al editor que estaba abierto cuando se inició la prueba.

- Método 2: Vuelve al editor con la barra de tareas (o presiona Alt + Tab) y confirma el final de la prueba. WINDEV o WINDEV Mobile muestra el editor que se utilizó al principio de la prueba.

## Notas
<a name="notas_ELTTEXTE000274"></a>


### Directorio de archivos utilizados para la prueba
<a name="directorio_archivos_utilizados_para_prueba_ELTPARAGRAPHE000093"></a>

Para definir la ubicación de los archivos de datos utilizados al probar una aplicación en el editor:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. En la pestaña "Archivo", seleccione la ubicación de los archivos de prueba:

	- Directorio actual: Los archivos de datos se ubicarán en el subdirectorio EXE del proyecto.

	- Directorio específico: Los archivos de datos se ubicarán en el directorio especificado.






<a name="NOTE3_2"></a>


### Gestión de contraseñas (análisis o archivo de datos)
<a name="gestion_contrasenas_analisis_archivo_datos_ELTPARAGRAPHE000118"></a>

No se solicita ninguna contraseña durante la prueba de la ventana, independientemente de si la ventana está asociada a un análisis protected en tiempo de ejecución o a un fichero de datos protected con contraseña.

Las contraseñas necesarias deben especificarse mediante programación en la "Inicialización" Event associated con el proyecto:

- Si la ventana utiliza un archivo de datos de un análisis contraseña-protected en tiempo de ejecución, esta contraseña debe especificarse con [HOpenAnalysis](../WDLang4/3044106.md).

- Si la ventana utiliza uno o varios archivos de datos con contraseña protected, la contraseña debe especificarse con [HPass](../WDLang4/3044108.md).




Se produce un error WLanguage al probar la ventana si no se especifican las contraseñas en la Event "Inicialización" del proyecto.

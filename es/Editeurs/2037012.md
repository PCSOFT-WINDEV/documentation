## Instalación de IIS 7
<a name="instalacion_iis_7_ELTTEXTE000216"></a>
El servidor IIS 7 se suministra con Windows Vista/7. Para configurar IIS 7 para WEBDEV, se deben instalar algunos elementos de IIS 7.

Para instalar IIS 7: 

1. Abra el panel de control de Windows ("Inicio .. Panel de control").

2. Haga doble clic en "Programas y características".

3. Haga clic en "Activar/Desactivar las características de Windows".

4. En la ventana que aparece, haga clic en la opción "Servicios de Internet (IIS)" y seleccione las siguientes opciones:




Para Windows Vista y 7, las opciones se presentan de la siguiente manera:

- **Para Windows Vista:** <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IIS_Version7.gif)


- **Para Windows 7:** <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IIS7_SevenFR.gif)







Las siguientes opciones deben estar marcadas:


|   |   |
| --- | --- |
| Compatibilidad con el IIS metabase y con la configuración del IIS 6 | Permite que el IIS 7 sea configurado por la metabase.<br><br>Esta opción es obligatoria. |
| Consola para la gestión de IIS | Permite configurar IIS 7<br>Esta opción es opcional, pero es necesaria si desea configurar IIS 7 manualmente. |
| Consola para la gestión de FTP | Permite configurar el servidor FTP |
| Servidor FTP | Requerido para realizar un implementación remoto. |
| Registro HTTP | Requerido para usar WDStatistic |
| CGI | Permite visualizar las páginas dinámicas de WEBDEV<br><br>Opción obligatoria. |
| Contenido estático | Utilizado para gestionar los archivos estáticos (.CSS, imágenes, ....). <br><br>Opción obligatoria. |
| Por defecto Document | Se utiliza para gestionar la página de Default del sitio |
| Autenticación básica | Requerido para realizar un implementación remoto. |

Observación: Atención: Si está utilizando un antivirus, es posible que tenga que desactivarlo para que funcione la prueba de un sitio WEBDEV con IIS 7.

## Configurando el servidor para el protocolo CGI (AWP)
<a name="configurando_servidor_para_protocolo_cgi_awp_ELTTEXTE000240"></a>
Para configurar el servidor web:

1. Abra el administrador de servicios de Internet de Microsoft (Opción "Iniciar .. Ejecutar .. Inetmgr.exe" desde el escritorio de Windows).

2. Sitúese en la raíz del equipo en el TreeView de la izquierda y haga clic en el botón "Restricciones ISAPI y CGI". Haga clic en "Agregar".

3. Especifique la ruta:

	- del archivo WD28 0AWP.EXE situado en el subdirectorio Programs\\ENGINE\\WINDOWS\\AWP. Esta ruta no debe contener ningún carácter de espacio.

	- del archivo WD28 0INSTAWP.EXE situado en el subdirectorio Programs\\ENGINE\\WINDOWS\\AWP. Esta ruta no debe contener ningún carácter de espacio.




4. Seleccione la opción "Permitir ejecución de la ruta de la extensión".

5. Valide.

6. En el TreeView de la izquierda, haga doble clic en "Sitios web" y seleccione la opción "Sitio web predeterminado".. Seleccione "Añadir un directorio virtual" de el menú contextual de esta opción.

7. En la pantalla que se visualiza:

	- Especifique el alias del protocolo CGI: wd28 0awp.

	- Seleccione el directorio del protocolo CGI (Programas\\MOTOR\\VENTANAS\\AWP subdirectorio del directorio WEBDEV instalación). Esta ruta no debe contener ningún carácter de espacio. Haga clic en Aceptar.




8. Seleccione WD28 0AWP en la vista de árbol.

9. Haga doble clic en el botón "Asignaciones de controlador".

10. Seleccione la opción "Modificar los permisos del controlador".

11. En la ventana que aparece, seleccione las opciones "Lectura", "Script", "Ejecución".

12. Valide.

## Configurar el servidor para los sitios WEBDEV
<a name="configurar_servidor_para_los_sitios_webdev_ELTTEXTE000264"></a>
Para configurar el servidor web:

1. Abra el administrador de servicios de Internet de Microsoft (opción "Iniciar .. Ejecutar .. Inetmgr.exe" desde el escritorio de Windows.

2. En el TreeView de la izquierda, haga doble clic en "Sitios" y seleccione la opción "Sitio Web predeterminado" (o seleccione el sitio Web predeterminado en el que deben instalarse los sitios dinámicos WEBDEV).. Seleccione "Añadir un directorio virtual" de el menú contextual de esta opción.

3. En la pantalla que se visualiza:

	- Especifique el alias del directorio que contiene las imágenes de su proyecto: &lt;MiProyecto&gt;_WEB. Por ejemplo: MARKET_WEB.

	- Seleccione el directorio que contiene las imágenes de su proyecto: &lt;Nombre Proyecto&gt;_WEB. Por ejemplo: C:\\WWproject\\MARKET\\MARKET_WEB.




4. Haga clic en Aceptar.

5. Seleccione el nombre de su alias en la vista de árbol de la izquierda.

6. Haga doble clic en el botón "Asignaciones de controlador".

7. Seleccione la opción "Modificar las autorizaciones de la función" en la lista de enlaces a la derecha.

8. En la ventana que aparece, seleccione las opciones "Lectura", "Script" y valide.

9. Seleccione de nuevo el sitio Web en el TreeView de la izquierda ("Sitio Web predeterminado" si hay un solo sitio).

10. Haga doble clic en el botón "Asignaciones de controlador".

11. Seleccione la opción "Agregar una asignación de scripts" en la lista de enlaces a la derecha.

12. En la ventana que aparece:

	- En el control "Ruta de solicitudes", indique el nombre del proyecto precedido de un slash. Por ejemplo: "/MARKET_WEB". 

	- En el "ejecutable" control, especifique el ruta de WD28 0AWP.EXE que se encuentra en el subdirectorio "Programas\\ENGINE\\WINDOWS\\AWP".. Utilice el botón "..." para seleccionar el archivo. Ejemplo: "C:\\WEBDEV\\AWP\\WD28 0awp.exe".
			Atención: 

		- Por defecto, el selector de archivos filtra solo las bibliotecas ("\*.dll") en la parte inferior derecha. Cambie el filtro en los ejecutables ("\*.exe") para seleccionar el archivo. 

		- Si el ruta contiene espacios en blanco, añada comillas al principio y al final.




- El contenido del control "Nombre" no tiene importancia. 

- Haga clic en el botón "Restricción de solicitudes. 

	- En el panel "Asignación", desactive la casilla "Llamar al controlador si se ha asignado una solicitud a".

	- En el panel "Verbos", verifique que el selector está en "Todos los verbos". 

	- En el panel "Acceso", verifique que el selector está situado en "Script".

## Configuración del servidor para los sitios que utilizan Active WEBDEV Pages
<a name="configuracion_del_servidor_para_los_sitios_que_utilizan_active_webdev_pages_ELTTEXTE000288"></a>
Para configurar el servidor: 

1. Abrir el administrador de los servicios de información de INTERNET.

2. En el TreeView de la izquierda, haga doble clic en "Sitios web" y seleccione la opción "Sitio Web predeterminado" (o seleccione el sitio Web predeterminado en el que deben instalarse los sitios dinámicos WEBDEV).

3. Haga doble clic en la opción "Asignaciones de controlador".

4. Añade un guión mapping:

	- la ruta de las solicitudes es: \*.awp

	- especificar el ruta del archivo WD28 0AWP.EXE situado en el subdirectorio Programs\\ENGINE\\WINDOWS\\AWP. Esta ruta no debe contener ningún carácter de espacio.

	- introduzca cualquier nombre.




5. Valide y responda "No" al mensaje.

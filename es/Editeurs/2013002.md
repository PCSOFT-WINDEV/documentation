## Presentación
<a name="presentacion_ELTTEXTE000119"></a>
WEBDEV le permite crear páginas estáticas y páginas dinámicas. Se pueden programar dos tipos de acciones en un sitio WEBDEV: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Deux_types_de_code.gif)


Para gestionar estos dos tipos de acciones, el editor de código de WEBDEV distingue entre dos tipos de código:

- **Código del servidor** (Código amarillo en el editor de código)
	Este código está escrito en WLanguage. Este código se ejecuta en el servidor.
	Este código sólo está disponible en las páginas dinámicas.

- **Código del navegador** (código verde o azul)
	Este código está escrito en WLanguage (verde) o en JavaScript/TypeScript (azul).
	Cuando se guarda la Page, el código WLanguage se convierte automáticamente en JavaScript y se incluye en la Page HTML de WEBDEV s.
	Este código se ejecuta en el equipo del usuario y no requiere ninguna acción del servidor.

## Código Navegador: ¿Código WLanguage o código JavaScript/TypeScript?
<a name="codigo_navegador_¿codigo_wlanguage_codigo_javascripttypescript_ELTTEXTE000143"></a>
En el editor de código, el código ejecutado en el navegador del usuario se representa mediante una cabecera verde o azul.


|   |   |
| --- | --- |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Page2_bis.gif&type=thumb)<br> | <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Page2_1.gif&type=thumb)<br> |



- Banda verde = WLanguage: el símbolo WL aparece en el encabezado del código.

- Cabecera azul = JavaScript / TypeScript: 

	- código JavaScript: el símbolo JS aparece en el encabezado del código.

	- código TypeScript: el símbolo TS aparece en la cabecera del código.







**Para cambiar de código verde (WLanguage) a código azul (JavaScript o TypeScript)** , simplemente: 

- haga clic en el símbolo **WL** en el encabezado del código.

- elija el lenguaje deseado (JavaScript o TypeScript). 




Le recomendamos que desarrolle en WLanguage.

**Recordatorio**: El código del navegador está disponible tanto en páginas dinámicas como estáticas.

### Notas sobre el código JavaScript
<a name="notas_sobre_codigo_javascript_ELTPARAGRAPHE000061"></a>

- Los procesos de controles y páginas no deben contener ningún prototipo: simplemente escriba el código JavaScript deseado

- Los procedimientos de las páginas deben contener el prototipo y las llaves requeridas por el código JavaScript.

- Los procedimientos de los supercontroles deben contener el prototipo y las llaves requeridas el código JavaScript.. 
	Además, el nombre de la función JavaScript generada debe ir precedido por el alias del supercontrol.

- Los accesos a los controles deben realizarse a través de sus alias.

- "_Page_" se utiliza para acceder al formulario que contiene los controles de la Page actual 

- Se pueden utilizar varias funciones de WLanguage para manipular las propiedades y los métodos de los objetos JavaScript en el código del navegador:
	


|   |   |
| --- | --- |
| [JSEndEvent](../WDLang2/1000018925.md) | Elimina la asociación entre una función de navegador de WLanguage y una Event( implementada por [JSEvent](../WDLang2/1000018924.md)). |
| [JSEvent](../WDLang2/1000018924.md) | Asocia una Procedure del navegador con una Event en un objeto del código del navegador. |
| [JSInfoEvent](../WDLang2/3036067.md) | Usó para manipular las propiedades de JavaScript del navegador Event que desencadenó la ejecución del código. |
| [JSInterruptEvent](../WDLang2/1000019185.md) | Interrumpe la Process de la actual Event. |
| [JSMethod](../WDLang2/3012002.md) | Se utiliza para ejecutar un método JavaScript en un elemento que se encuentra en la página actual. |
| [JSProperty](../WDLang2/3012019.md) | Se utiliza para las características específicas de manipular en los objetos que se encuentran en la página actual. |

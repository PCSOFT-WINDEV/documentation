## Presentación
<a name="presentacion_ELTTEXTE000097"></a>
El objetivo principal de un equipo de desarrollo es crear aplicaciones optimizadas, capaces de satisfacer las necesidades cambiantes de los usuarios.

El modelado de una aplicación permite especificar la estructura y el comportamiento esperado de un sistema. Ayuda a comprender su organización y a detectar las oportunidades de simplificación y reutilización, así como a gestionar los posibles riesgos.

Un modelo es una simplificación de la realidad. Le permite entender mejor el sistema a desarrollar.

Un diagrama es la representación gráfica de un conjunto de elementos que componen un sistema. Para ver el sistema desde diferentes perspectivas, el Lenguaje Unificado de Modelado (UML) define nueve diagramas diferentes. Cada diagrama representa un estado del sistema.

WINDEV, WEBDEV y WINDEV Mobile le permiten crear los siguientes nueve tipos de modelos UML:

- [Diagrama de clases](../Editeurs/2035011.md): describe la estructura general de un sistema.

- [Diagrama de casos de uso](../Editeurs/2035004.md): representa las funcionalidades del sistema desde el punto de vista del usuario.

- [Diagrama de objetos](../Editeurs/2035012.md): representa un conjunto de objetos y sus relaciones en un momento determinado.

- [Diagrama de componentes](../Editeurs/2035020.md): describe la arquitectura física y estática de una aplicación informática.

- [Diagrama de actividades](../Editeurs/2035005.md): representa el comportamiento de un método o el progreso de un caso de uso.

- [Diagrama de secuencia](../Editeurs/2035017.md): representa el orden cronológico de los mensajes enviados y recibidos por un conjunto de objetos.

- [Diagrama de colaboración](../Editeurs/2035002.md): representa la organización estructural de objetos que envían y reciben mensajes.

- [Diagrama de transición de estados](../Editeurs/2035023.md): representa una secuencia de estados.

- [Diagrama de implementación](../Editeurs/2035019.md): muestra la intersección de los dispositivos (nodos) utilizados en un sistema, así como la asociación entre los programas ejecutables y estos dispositivos.




Esta página de ayuda presenta una visión general del modelado UML. Para obtener más información sobre el UML, consulte la documentación específica.

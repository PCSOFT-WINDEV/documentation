## Presentación
<a name="presentacion_ELTTEXTE000214"></a>
La prueba de una sola página le permite ejecutar la página actual en WEBDEV. Esto le permite ejecutar la prueba de su proyecto desde una página determinada o comprobar el modo de funcionamiento de una página tan pronto como esté completamente desarrollada.

Recordatorio: Una página semidinámica se utiliza para generar páginas estáticas que contienen datos. Antes de ejecutar la prueba de una página semidinámica, debe generar las páginas HTML estáticas correspondientes.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000238"></a>


### Ejecutar la prueba de la página semidinámica actual en el editor
<a name="ejecutar_prueba_pagina_semidinamica_actual_editor_ELTPARAGRAPHE000023"></a>

Para ejecutar la prueba de la página semidinámica actual en el editor:

1. Compruebe si los archivos de datos utilizados se encuentran en el directorio de los archivos utilizados para la prueba (consulte las Notas).

2. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_Go_Page.gif) en los botones de acceso rápido (o presione F9).
	**Consejo**: Para seleccionar el navegador utilizado al ejecutar la prueba de la página, haga clic en la prueba Icon de la página. La flecha de la derecha amplía un menú que le permite elegir el navegador que se utilizará para la prueba.. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GO_PAGE_WB.gif)
 TLa elección del navegador Default también puede realizarse en el [Opciones de WEBDEV](../Editeurs/2036001.md).

3. Validar (si es necesario) la generación de las páginas HTML correspondientes.
	Recordatorio: Los siguientes códigos se ejecutan cuando se generan las páginas semidinámicas:

	- Declarar las variables globales de las páginas.

	- Inicializando páginas.




4. El editor se minimiza automáticamente y se ejecuta la página.



<a name="NOTE2_2"></a>


### Ejecución de la prueba de una secuencia de páginas semidinámicas
<a name="ejecucion_prueba_una_secuencia_paginas_semidinamicas_ELTPARAGRAPHE000067"></a>

Para ejecutar la prueba de una secuencia de páginas semidinámicas:

1. Compruebe si los archivos de datos utilizados se encuentran en el directorio de los archivos utilizados para la prueba (consulte las Notas).

2. Iniciar la herramienta para generar las páginas estáticas: en la pestaña "Herramientas", en el grupo "Utilidades web", haga clic en "WDGenPage".

3. Seleccione la página semidinámica que desea generar.

4. Haga clic en el botón "Generar.
	Los siguientes códigos se ejecutan cuando se generan las páginas semidinámicas:

	- Declarar las variables globales de la página

	- Inicialización de la página




5. Una vez finalizada la generación, seleccione la página cuya prueba debe ejecutarse y haga clic en [ Run the test]. Puede ejecutar la prueba de la primera página estática generada. El editor se minimiza automáticamente y la página se muestra en un navegador.




Durante la prueba de una página semidinámica, se ejecutan los siguientes procesos:

- cargar la primera página (código del navegador)

- entrando en el primer control de la primera página (si este control tiene una entrada Process) (código de navegador)




Durante la prueba, se pueden ejecutar todas las funciones de la página. Tendrá la posibilidad de abrir otras páginas, por ejemplo.

Observación: La última prueba (ventana, consulta, reporte,...) se puede volver a ejecutar desde cualquier elemento del proyecto a través de "GO... Vuelva a ejecutar la última prueba" que se encuentra en los botones de acceso rápido. 
<a name="NOTE2_3"></a>


### Detener la prueba de una página semidinámica
<a name="detener_prueba_una_pagina_semidinamica_ELTPARAGRAPHE000107"></a>

Se pueden utilizar varios métodos para detener la prueba:

- 1er método: Cerrar el sitio cuya prueba se está ejecutando actualmente. WEBDEV muestra el editor utilizado al principio de la prueba.
	Si la prueba se ejecutó desde WDGenPage, debe salir de esta herramienta.

- 2º método: Vuelva al editor con la barra de tareas (o Alt + Tab) y confirme que la prueba debe ser detenida.. WEBDEV muestra el editor utilizado al principio de la prueba.

## Notas
<a name="notas_ELTTEXTE000274"></a>


### Directorio de archivos utilizados para la prueba
<a name="directorio_archivos_utilizados_para_prueba_ELTPARAGRAPHE000121"></a>

Para configurar los archivos de datos utilizados al ejecutar la prueba de sitio en el editor:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción"

2. 1. En la pestaña "Archivo", seleccione la ubicación de los archivos de prueba:

	- Directorio actual: Los archivos de datos estarán en el subdirectorio del proyecto EXE.

	- Directorio específico: Los archivos de datos se encontrarán en el directorio especificado.






<a name="NOTE3_2"></a>


### Códigos de generación
<a name="codigos_generacion_ELTPARAGRAPHE000139"></a>

Dos códigos de generación están disponibles en una página semidinámica. Estos códigos son códigos de servidor y se utilizan para introducir la información necesaria para la generación.

Por ejemplo:

- Si el archivo base de la página (o el análisis) está protegido por contraseña, la contraseña se puede especificar en el código de generación.

- Si el archivo base de la página es una consulta con parámetros, los parámetros esperados por la consulta pueden ser especificados en este código.

- También tiene la posibilidad de llamar a una clase en este código.



<a name="NOTE3_3"></a>


### Gestión de contraseñas (análisis o archivo)
<a name="gestion_contrasenas_analisis_archivo_ELTPARAGRAPHE000152"></a>

No se solicita ninguna contraseña durante la prueba de la página, independientemente de que la página sea associated con un análisis protegido en tiempo de ejecución o con un archivo de datos protegido por contraseña.

Las contraseñas necesarias deben ser especificadas por programación en el código de inicialización del proyecto (si existe) o en el código de inicialización de las variables globales de la página semidinámica.

- Si la página utiliza un archivo de datos a partir de un análisis protegido por contraseña, ésta debe especificarse con [HOpenAnalysis](../WDLang4/3044106.md).

- Si la página utiliza uno o varios archivos de datos protegidos por contraseña, ésta debe especificarse con [HPass](../WDLang4/3044108.md).




Si no se especifican las contraseñas, se produce un error al generar la página.

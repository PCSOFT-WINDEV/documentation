## Presentación
<a name="presentacion_ELTTEXTE000236"></a>
El análisis se utiliza para definir la estructura de los archivos de datos utilizados por el proyecto.

Puede describir en el análisis:  

- [un nuevo archivo de datos.](#NOTE2_1)

- [un archivo de datos preestablecido, proporcionado con WINDEV, WEBDEV y WINDEV Mobile.](#NOTE2_2)

- [un archivo de datos importado de una base de datos externa](../Editeurs/2011025.md)(también denominada "Base de datos de terceros").

- [un archivo de datos importado de una versión anterior de WINDEV](../Editeurs/2011025.md).

- [un archivo de datos ISAM HFSQL importado a través del Explorador de archivos de Windows](../Editeurs/2011025.md).




**Observación**: Para modificar la description de un archivo de datos (o la description de sus artículos): 

- Selecciona el archivo de datos en el editor de análisis. 

- Seleccione una de las siguientes opciones de el menú contextual: 

	- "Descripción del archivo de datos".

	- "Descripción de los campos".







**Recordatorio**: En una aplicación WINDEV, WEBDEV o WINDEV Mobile, los términos "MLD" y "Analysis" se utilizan indistintamente para definir la estructura de la base de datos asociada a un proyecto.

Para más detalles sobre cómo manipular un archivo de datos o elemento en el editor de análisis, ver [Operaciones en un archivo de datos](../Editeurs/2011007.md) y [Operaciones en una posición](../Editeurs/2011009.md).

## Añadir un nuevo archivo de datos
<a name="anadir_nuevo_archivo_datos_ELTTEXTE000266"></a>


### La creación de un nuevo archivo de datos
<a name="creacion_nuevo_archivo_datos_ELTPARAGRAPHE000053"></a>

Para crear un nuevo archivo de datos: 

1. En la pestaña "Análisis", en el grupo "Creación", haga clic en "Nuevo archivo". El asistente para crear un nuevo archivo de datos se abre.

2. Seleccione "Crear una nueva descripción de archivo de datos". Pase a la etapa siguiente.

3. Especifique:

	- el ***nombre del archivo lógico***. Este nombre se usará para manipular el archivo de datos.

	- el ***título*** que describe brevemente el tema del archivo.

	- la ***representación de un registro*** en el archivo de datos. Esta representación facilita las preguntas formuladas al describir las relaciones. Esta opción debe ir precedida de un artículo indefinido (UNO o UNA).

	- si el archivo de datos contiene un campo de tipo ***"Identificador automático"***. El valor de este artículo es único para cada Record y es calculado automáticamente por el motor HFSQL.
			Es posible utilizar: 

		- un ID de 8 bytes. Este tipo de identificador es obligatorio para la replicación de servidores HFSQL.. 

		- un ID de 4 bytes. 




- un UUID de automatic (128 bits). En este caso, el identificador de cada Record corresponde a un UUID de 128 bits (el UUID proporciona un identificador único no secuencial). 

- un UUID de automatic (256 bits). En este caso, el identificador de cada Record corresponde a un UUID de 256 bits (el UUID proporciona un identificador único no secuencial).

4. En la siguiente etapa, indique el ***tipo de base de datos*** asociado al archivo de datos. Dependiendo del tipo seleccionado, este archivo de datos será manejado por el motor HFSQL, un controlador OLE DB o un Conector Nativo (SQL Server, Oracle, etc.).
	Si el tipo de base de datos utilizada no corresponde a HFSQL Classic (Red o Monopuesto), el asistente solicita las características de conexión a la base de datos.

5. El nuevo archivo de datos se convierte en el archivo de datos actual. La ventana de descripción de los campos del archivo de datos se abre automáticamente. Esta ventana le permite definir los campos del archivo de datos.

### Importar un description preestablecido de archivo de datos
<a name="importar_description_preestablecido_archivo_datos_ELTPARAGRAPHE000099"></a>

Para importar un archivo preestablecido description: 

1. En la pestaña "Análisis", en el grupo "Creación", haga clic en "Nuevo archivo". El asistente para crear un nuevo archivo de datos se abre.

2. Seleccione la opción "Seleccionar una descripción entre los archivos de datos predefinidos".

3. Elija el o los archivos de datos preestablecidos para crear. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Analyse_Fichierpredef.gif)
 La asistente también propone archivos de datos associated que pueden añadirse como enlaces.

4. Para cada archivo de datos, seleccione los elementos a mantener. Puede modificar los campos más adelante.

5. Los archivos de datos preestablecidos se insertan automáticamente en el análisis actual.. Los enlaces entre los archivos preestablecidos se establecen automáticamente.

## color para las descripciones de los ficheros de datos
<a name="color_para_las_descripciones_los_ficheros_datos_ELTTEXTE000296"></a>
Los siguientes colores se utilizan para identificar rápidamente los tipos de archivos de datos en la editor de análisis: 

- Azul para los archivos de datos procedentes de una Conector Nativo.

- Naranja para los archivos de datos de HFSQL Client/Server.

- Marrón para los archivos de datos de HFSQL Classic.

- Verde para los archivos de datos ODBC. 

- Gris para los archivos de datos de Hyper File 5.5.

## Seguridad de un archivo de datos HFSQL
<a name="seguridad_archivo_datos_hfsql_ELTTEXTE000320"></a>
Para garantizar la seguridad de un archivo de datos HFSQL: 

1. Abra el archivo description: 

	- Selecciona el archivo de datos en el editor de análisis. 

	- En el menú contextual, seleccione la opción "Descripción del archivo de datos".




2. En la pestaña "Detalles: 

	- Habilitar la opción de encriptación para el archivo de datos, índices, memorandos. Varios modos de encriptación están disponibles. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Base_HFSQLCS_Securite%20-%20HC%20N%B0001.gif)
 La opción "automatic (RC5 16 rondas si contraseña)" se selecciona por defecto al crear un archivo de datos. Esta opción se utiliza para cifrar automáticamente el archivo de datos si se define una contraseña por [HPass](../WDLang4/3044108.md) en el archivo de datos.

	- Active la opción "Activar alto nivel de seguridad ".
			Observación: Esta opción es seleccionada por Default cuando se crea un archivo de datos. Esta opción le permite solicitar la contraseña en cada modificación de automatic del archivo de datos.




3. Generar el análisis. 

4. Especifique una contraseña para el archivo de datos con [HPass](../WDLang4/3044108.md). 

5. Encriptar la conexión a la base de datos en las conexiones descritas en el análisis, o programáticamente con el programa [Cifrado](../Proprietes/2512102.md) Property.




Observación: Estándar de cifrado avanzado (algoritmo AES de 128, 192 y 256 bits con OFB): 

- Si el archivo de datos está encriptada con el algoritmo AES, hay que utilizar el mismo método para encriptar los memos y los índices. 

- Si hay un cifrado que utiliza el algoritmo AES, la opción "Activar alto nivel de seguridad " se aplica automáticamente.

## Creación de un proyecto para iOS con WINDEV Mobile
<a name="creacion_proyecto_para_ios_con_windev_mobile_ELTTEXTE000156"></a>


### Creación de un proyecto de iPhone/iPad con WINDEV Mobile
<a name="creacion_proyecto_iphoneipad_con_windev_mobile_ELTPARAGRAPHE000028"></a>

Para crear un proyecto de iPhone/iPad con WINDEV Mobile: 

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- Aparece la ventana de creación: haga clic en "Proyecto". 

	- El asistente de creación de proyectos se abre. 




2. Elija el tipo de proyecto a crear: en nuestro caso, seleccione "Android o aplicación iOS". 

3. Pase a la etapa siguiente del asistente. 

4. Elija la plataforma deseada. Para un proyecto iOS, seleccione "sólo iOS" y vaya al siguiente paso. 

5. Especifique si el nuevo proyecto debe estar en blanco o basado en un ejemplo. Pase a la etapa siguiente. 

6. La asistente propone seleccionar el tipo de dispositivos afectados por el proyecto: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Developper_pour_ios%20-%20HC%20N%B0001.gif)

	Observación: Si la aplicación está destinada a ser utilizada en varios dispositivos iOS (teléfonos con diferentes tamaños de pantalla o resoluciones, por ejemplo), WINDEV Mobile muestra la resolución más pequeña por Default para crear las ventanas de la aplicación. El uso de anclajes permitirá que la aplicación opere en todos los dispositivos.

7. Elija el nombre y la ubicación de su proyecto. Tienes la capacidad de entrar en un corto description del proyecto. Pase a la etapa siguiente.
	Observación: no se debe utilizar ningún carácter acentuado en el nombre del proyecto. 

8. Añadir (si es necesario) documentos de diseño al proyecto. Pase a la etapa siguiente.

9. Indique si el proyecto debe incluirse en el [Administrador de Código Fuente (SCM)](../Editeurs/2038001.md). El Administrador de Código Fuente simplifica el trabajo en equipo y también puede ser útil para el trabajo individual. Pase a la etapa siguiente.

10. Seleccione el estilo de código de su aplicación. Pase a la etapa siguiente.

11. Seleccione el tema gráfico de su aplicación. Pase a la etapa siguiente.

12. Seleccione los idiomas soportados. Estos idiomas serán propuestos por Default tan pronto como una opción de un elemento (control, ventana, ...) pueda ser traducida. Especificar (si es necesario) las opciones que deben tenerse en cuenta para cada idioma (gestión de números, fechas, ....

	- Para seleccionar el idioma de Default que aparece en el editor, seleccione el idioma solicitado y seleccione "Default language" de el menú contextual.

	- Para seleccionar el idioma Default en tiempo de ejecución, seleccione el idioma solicitado y elija "Idioma Default en tiempo de ejecución". Este idioma se utilizará para ver el proyecto en tiempo de ejecución.




13. Especificar si el proyecto utiliza una base de datos. De hecho, el proyecto creado puede ser associated con un análisis. Los archivos de datos pueden tener el siguiente formato: 

	- HFSQL Classic. 

	- HFSQL Client/Server.


 Observación: No se debe utilizar ningún carácter acentuado en los nombres de los archivos de datos.. 

14. La asistente para la creación de proyectos ha terminado. La asistente para la creación de análisis se inicia si se solicitó la creación de una base de datos. 

15. El proyecto se desarrolla igual que un proyecto WINDEV Mobile. Sin embargo, hay algunos aspectos específicos del desarrollo de iOS. Para obtener más información, consulte [Características específicas de aplicaciones iOS](../Editeurs/1000019865.md). 




**Observaciones**:

- Para insertar Image s e iconos en sus aplicaciones iPhone/iPad, el catálogo Image propone Image s específicos. Todo lo que tiene que hacer es seleccionar el tema "iOS" durante su búsqueda. 

- Para tus aplicaciones de iPhone y iPad, usa el "ActivPhone 4" o "ActivPhone 7" plantilla.

- iOS utiliza algunos tipos de letra específicos de Apple. Estas fuentes son propiedad de Apple. Estas fuentes se pueden utilizar en el editor. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Developper_pour_ios%20-%20HC%20N%B0002.gif&type=thumb)
 Si esta fuente no está disponible en Windows y es seleccionada por un control encontrado en una ventana de iOS, se utilizará temporalmente una fuente de sustitución para la visualización en el editor y en el simulador del ordenador de desarrollo. Se utilizará la fuente Apple cuando se ejecute la aplicación.

- Atención: Los proyectos de iOS están en modo "Unicode. La Line del código: 
	
	```wl
	MyString is string
	```
significa que MyString es una cadena Unicode.

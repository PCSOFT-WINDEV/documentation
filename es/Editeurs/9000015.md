


## Análisis de impacto
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000129"></a>
El análisis de impacto identifica inmediatamente las consecuencias de los últimos cambios. Se muestra automáticamente una ventana de información para presentar las posibles ambigüedades, las modificaciones a evitar....

Las modificaciones que se han tenido en cuenta son las siguientes:

- renombrar o borrar un elemento manejado en el proyecto, 

- renombrando dos elementos diferentes con el mismo nombre, 

- agregar o eliminar el parámetro de un procedimiento,

- ...




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000153"></a>


### Inicio del análisis de impacto
<a name="inicio_del_analisis_impacto_ELTPARAGRAPHE000025"></a>

El análisis de impacto se inicia automáticamente cuando se produce un cambio potencialmente ambiguo.

Por ejemplo, crear una control con el mismo nombre que otro elemento del proyecto: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=AnalyseImpactNom.GIF)

<a name="NOTE2_2"></a>


### Descubrir los usos de un elemento del proyecto
<a name="descubrir_los_usos_elemento_del_proyecto_ELTPARAGRAPHE000035"></a>

Conocer los usos de un elemento del proyecto:

1. En la pestaña "Proyecto", en el grupo "Auditoría y rendimiento", despliegue "Auditoría estática" y seleccione "Referencias cruzadas".

2. Introduzca el nombre del elemento para el que desea averiguar los usos en el proyecto.

3. Haga clic en "Find". Se muestra la lista de usos del elemento seleccionado:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=AnalyseImpactRech.GIF&type=thumb)


4. Haga doble clic en uno de los usos del elemento para visualizarlo.




**Observación**: Esta ventana puede permanecer abierta mientras se realizan las modificaciones en el proyecto.
<a name="NOTE2_3"></a>


### Aplicando un cambio de nombre a todos los elementos que se encuentran en su proyecto
<a name="aplicando_cambio_nombre_todos_los_elementos_que_encuentran_proyecto_ELTPARAGRAPHE000062"></a>

Aplicar un cambio de nombre a todos los elementos que se encuentran en su proyecto:

1. Renombrar el elemento solicitado.

2. Al validar el nuevo nombre:

	- los errores correspondientes se muestran en el panel "Errores de compilación".

	- se muestra un mensaje en la parte superior derecha Corner del entorno actual.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Refactoring.gif)








Este mensaje propone renombrar automáticamente todos los elementos necesarios.

La operación de renombrar se realiza automáticamente cuando se hace clic en este mensaje.

**Observaciones**:

- Para ver todas las referencias que se modificarán, seleccione "Análisis de impacto del cambio de nombre" en el menú contextual del error:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=AnalyseImpact.GIF&type=thumb)


- También puede iniciar la operación de renombramiento cuando se valida el nuevo nombre, mostrando el menú contextual del error:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Refactoring_Menu.gif&type=thumb)

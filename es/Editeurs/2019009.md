


## Prueba de un proyecto dinámico (sesión o modo AWP) de WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000301"></a>
La prueba de todo el proyecto permite simular la puesta en marcha del sitio. Esto permite probar todo el sitio, incluso si su desarrollo aún no está completo. Tan pronto como se produce un problema en la ejecución del proyecto (sólo código del servidor), se puede iniciar el depurador para identificar y solucionar el problema.

**Observación**: La prueba de un proyecto puede ejecutarse independientemente del elemento actual en el editor.

Esta ayuda Page explica cómo probar un proyecto desde el editor WEBDEV. También es posible probar un proyecto correspondiente a un sitio en modo de sesión desde el administrador de WEBDEV.

Para obtener más información, consulte: 

- [Prueba de un proyecto WEBDEV (sitio en sesión o en modo AWP) desde el administrador de WEBDEV (versión de desarrollo)](../Editeurs/2019007.md).

- [Probando un proyecto WEBDEV (sitio en sesión o en modo AWP) desde un servidor de pruebas](../Editeurs/2019012.md).

- [Las diferencias entre probar un proyecto desde el editor y desde el administrador](../Editeurs/2019008.md).

- [Probando y depurando un sitio en modo de sesión a distancia](../Editeurs/2019032.md).

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000325"></a>


### Probando un sitio en modo de sesión/AWP (o una parte de un sitio en modo de sesión/AWP)
<a name="probando_sitio_modo_sesionawp_una_parte_sitio_modo_sesionawp_ELTPARAGRAPHE000037"></a>

Para probar un sitio WEBDEV (modo sesión/AWP) desde el editor:

1. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Depurar proyecto". 
	También puede hacer clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_GO_Projet_WB_bl.gif) en los botones de acceso rápido o pulsar Ctrl + F9.

2. El editor se minimiza automáticamente y el proyecto se ejecuta en el navegador Default.

3. Aparece la primera Page del sitio en modo sesión.




Al inicio de la prueba del proyecto, antes de abrir la primera Page en modo sesión del proyecto, se ejecutan los siguientes eventos WLanguage:

- declarando clases,

- inicialización del proyecto,

- declarar las variables globales de la primera Page en modo sesión,

- inicialización de la primera Page,

- inicializar los controles que se encuentran en la primera Page,

- cargando la primera Page,

- entrando en el primer control de la primera página (si este control tiene una entrada de WLanguage Event).



<a name="NOTE2_2"></a>


### Ejecución de la prueba de las características del sitio
<a name="ejecucion_prueba_las_caracteristicas_del_sitio_ELTPARAGRAPHE000074"></a>

Se recomienda probar todas las características del sitio así como el uso del botón de regreso del navegador. Para más información sobre el botón de retroceso del navegador, véase [Gestión del botón Atrás](../WDChamp/1012001.md). 
<a name="NOTE2_3"></a>


### Detener la prueba del proyecto
<a name="detener_prueba_del_proyecto_ELTPARAGRAPHE000084"></a>

Existen varios métodos para detener la prueba:

- Método 1: Cerrar el navegador en el que se ejecuta la prueba. WEBDEV muestra el editor que se encontraba abierto al iniciar la prueba.

- Método 2: Vuelve al editor con la barra de tareas (o presiona Alt + Tab) y confirma el final de la prueba. WEBDEV muestra el editor que se encontraba abierto al iniciar la prueba.

## Notas
<a name="notas_ELTTEXTE000361"></a>


### Diferentes módulos se iniciaron automáticamente al probar un sitio WEBDEV (modo sesión/AWP)
<a name="diferentes_modulos_iniciaron_automaticamente_probar_sitio_webdev_modo_sesionawp_ELTPARAGRAPHE000096"></a>

Los siguientes módulos se inician automáticamente durante una prueba:

- El **servidor Web** que se instaló en el equipo y se configuró para WEBDEV al instalar la versión de desarrollo de WEBDEV. 
	La prueba no se puede ejecutar si no se inicia el servidor Web.

- El **administrador de WEBDEV** (WD 28 0Admin.EXE).
	El administrador permite administrar las conexiones al servidor Web y configurar los sitios en el modo de sesión WEBDEV.

- El **motor WEBDEV** (WD 28 0AWP.EXE).
	El motor WEBDEV se utiliza para gestionar las peticiones realizadas por los usuarios de la Web desde su navegador y devolver el correspondiente HTML dinámico Page.

- El **navegador INTERNET**.
	El navegador INTERNET muestra las páginas HTML del sitio WEBDEV.



<a name="NOTE3_2"></a>


### Navegador de prueba
<a name="navegador_prueba_ELTPARAGRAPHE000120"></a>

Para seleccionar el navegador de prueba: 

1. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba".

2. Seleccione "Navegador de prueba" y elija el navegador deseado.


**Observación**: El navegador Default está definido en las opciones de WEBDEV: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones generales de WEBDEV". 

2. En la ventana de opciones de WEBDEV, haga clic en la pestaña "Web".. Para obtener más información, consulte [opciones de WEBDEV](../Editeurs/2036001.md).




**Consejo**: Para seleccionar el navegador utilizado para probar el proyecto, haga clic en la prueba del proyecto Icon los botones de acceso rápido. La flecha de la derecha despliega un menú para elegir el navegador que se utilizará para la prueba.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GO_Projet_WB_deroule.gif)

### Probar un sitio desde una Phone
<a name="probar_sitio_desde_una_phone_ELTPARAGRAPHE000167"></a>

Puede probar sitios WEBDEV directamente desde un Phone. Esto le permite ver cómo se comporta el sitio en un tamaño de pantalla determinado. 

Para probar un proyecto en un Phone: 

1. Despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_GO_Projet_WB_bl.gif) en los botones de acceso rápido.

2. En la lista que aparece, seleccione "Probar navegador .. Phone (a través de un código QR)". 

3. Ejecutar la prueba del proyecto haciendo clic en el "GO" Icon.

4. Aparece una ventana con un código QR. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P8_Agencement_simple_1%20-%20HC%20N%B0016.jpg)


5. Escanear el código QR con el dispositivo. 

6. El proyecto figura en la lista Phone. 




Observaciones: 

- Asegúrese de que el dispositivo móvil esté conectado a la misma red que el PC.

- Los puntos de interrupción están activados. El modo de depuración estándar sigue estando disponible: ejecución paso a paso, posibilidad de ver y modificar el contenido de las variables, etc.




Para obtener más información, consulte [Probar un sitio WEBDEV en un dispositivo móvil](../Editeurs/2019035.md).

### Vista y UAC: Ejecución de una prueba
<a name="vista_uac_ejecucion_una_prueba_ELTPARAGRAPHE000196"></a>

**Vista con UAC (cuenta de usuario control ) habilitado**: Si WEBDEV se inicia como administrador (opción en el menú contextual o "Ejecutar este programa como administrador" en las propiedades del ejecutable), la cuenta de invitado utilizada por IIS7 debe formar parte del grupo de administradores. De lo contrario, se mostrará un error en el navegador.

**Vista sin el UAC habilitado**: Si el usuario de Vista es un administrador o un usuario del grupo de administradores, la cuenta de invitado usada por IIS 7 debe ser parte del grupo de administradores.. De lo contrario, se mostrará un error en el navegador.

Para get y cambiar la cuenta de invitado utilizada, véase [Identificar la cuenta que ejecuta el protocolo AWP](../Editeurs/2037001.md).
<a name="NOTE3_4"></a>


### Directorio de los archivos de datos utilizados para las pruebas
<a name="directorio_los_archivos_datos_utilizados_para_las_pruebas_ELTPARAGRAPHE000210"></a>

Para configurar los archivos de datos utilizados al ejecutar la prueba de sitio en el editor:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. En la pestaña "Archivos", seleccione la ubicación de los archivos de datos de prueba:

	- Directorio actual: Los archivos de datos se ubicarán en el subdirectorio EXE del proyecto.

	- Directorio específico: Los archivos de datos se ubicarán en el directorio especificado. 








### Comprobación del entorno de tiempo de ejecución antes de la prueba
<a name="comprobacion_del_entorno_tiempo_ejecucion_antes_prueba_ELTPARAGRAPHE000237"></a>

Antes de ejecutar una prueba de proyecto, WEBDEV comprueba si el entorno de prueba está configurado correctamente.

Si se detecta un problema, WEBDEV le pedirá que: 

- "Diagnosticar" abre la ventana de diagnóstico detallado.

- "Ejecutar prueba" ignora el diagnóstico y ejecuta la prueba. Esta opción debe utilizarse si el diagnóstico es incorrecto.

- " Cancel" Cancel s la prueba.




La ventana para el diagnóstico detallado es la siguiente: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Configuration%20CCH%E9bergement%20Compte%20WB%20-%20HC%20N%B0004.gif)


Esta ventana realiza una set de pruebas para comprobar si el entorno de la prueba está correctamente configurado.

Para cada error detectado, un botón "Solucionar" le permite resolver automáticamente el problema.

El botón "Ayuda" se activará si hay disponible una ayuda adicional.




## Historia del código
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000258"></a>
El historial de códigos le permite:

- llevar la cuenta de todos los cambios realizados en el código en los diferentes procesos/eventos.

- restaurar el código a partir de una fecha determinada, si este código fue borrado o modificado por error desde entonces.




El historial de códigos está disponible para los eventos o procesos con associated:

- el proyecto,

- ventanas y páginas,

- reportes, 

- preguntas,

- clases, 

- conjuntos de procedimientos.


Observación: Si el proyecto está en Administrador de Código Fuente (SCM), puede ver rápidamente qué elementos se han modificado. Para obtener más información, consulte [Historia de una Line de código](#NOTE2_5).

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000282"></a>


### Activación del historial de códigos
<a name="activacion_del_historial_codigos_ELTPARAGRAPHE000035"></a>

Para habilitar el historial de código en su proyecto:

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones del editor de código".

2. En la pestaña "Código", marque "Guardar historial de cambios del código".




**Observaciones**:

- Para desactivar el historial de códigos, desactive esta opción.

- Si se realizan varias modificaciones en el código de un proyecto, el tamaño de los archivos que contienen el historial de códigos puede llegar a ser bastante grande.



<a name="NOTE2_2"></a>


### Visualización del historial de códigos
<a name="visualizacion_del_historial_codigos_ELTPARAGRAPHE000066"></a>

Para ver el historial de código de su proyecto:

1. Mostrar la Process cuya historia debe ser vista.

2. En la pestaña "Código", en el grupo "Otras acciones", amplíe "Historial" y seleccione "Mostrar historial de modificaciones". Aparece la ventana del historial.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Historique-de-code.gif)


3. Haga clic en las versiones a la izquierda de la ventana para desplazarse por los cambios realizados en la Process actual. Esta ventana le permite:

	- ver las diferencias entre el elemento seleccionado y la versión actual (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Btn_Diff.gif)).

	- imprimir la versión mostrada (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=btn_imp.gif)).

	- borrar la modificación actual.






<a name="NOTE2_3"></a>


### Restaurar un código en el historial
<a name="restaurar_codigo_historial_ELTPARAGRAPHE000098"></a>

Para restaurar un código en el historial:

1. Mostrar el historial de la Process actual: en la pestaña "Código", en el grupo "Otras acciones", amplíe "Historial" y seleccione "Mostrar historial de modificaciones".

2. Encuentra el código a restaurar.

3. Seleccione el código a restaurar (usando el ratón, Ctrl + A, etc.).

4. copy el código seleccionado al portapapeles (Ctrl + C).

5. Cierre la ventana del historial de códigos.

6. En la ventana de código, seleccione el código a sustituir (con el ratón, Ctrl + A, etc.).

7. Pegue el contenido del portapapeles (Ctrl + V).



<a name="NOTE2_4"></a>


### Borrar el historial de códigos
<a name="borrar_historial_codigos_ELTPARAGRAPHE000125"></a>

Este método permite borrar las modificaciones específicas de un Process.

Para borrar el historial de códigos:

1. Vaya al proceso o a la Event de la que desea borrar el historial.

2. En "Código", en el grupo "Otras acciones", amplíe "Historial" y seleccione "Borrar historial".




**Atención**: Una vez borrado, el historial de un proceso o de Event ya no se puede ver. Los futuros cambios en este proceso o en Event se guardarán automáticamente si el historial de códigos está activado.

**Observación**: En la ventana del historial de códigos, también puede eliminar los cambios actuales (botón ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IconeSupprime.gif)).
<a name="NOTE2_5"></a>


### Historia de un código Line (proyecto encontrado en el SCM (Administrador de Código Fuente))
<a name="historia_codigo_line_proyecto_encontrado_scm_administrador_codigo_fuente_ELTPARAGRAPHE000155"></a>

Cuando un proyecto está en la SCM, el editor de código propone un historial avanzado de la Line de código. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Historique_Code_GDS.gif)
 Durante un clic en la fecha, un tooltip le permite: 

- Averiguar desde cuándo se realizó la modificación. 

- Ver la modificación de código que se realizó: En este caso, se muestra la ventana SCM utilizada para comparar las versiones de código.. 

- Ver todas las modificaciones: En este caso, se muestra la ventana SCM utilizada para comparar las versiones de la unidad actual.. 

- Acceder a la historia del elemento: En este caso, se muestra la ventana SCM utilizada para listar el historial de las modificaciones actuales.. 




Condiciones de uso: Para utilizar esta característica, deben cumplirse las siguientes condiciones: 

- Opción"Fecha y Autor" habilitada: En la pestaña "Vista", en el grupo "Ayuda edición", marque "Mostrar fecha y autor" (F6). 

- Las líneas de código deben haber sido modificadas y revisadas en la versión 19 (o posterior). 




Observación: Si se introdujo un comment cuando el elemento modificado se registró de nuevo, este comment se muestra en el tooltip.

### Guardar el historial de códigos
<a name="guardar_historial_codigos_ELTPARAGRAPHE000190"></a>

Cuando se activa la historia, se crea un subdirectorio "Historia" en el directorio del proyecto. Este directorio contiene los siguientes subdirectorios:

- WDP: historia de los acontecimientos associated con el proyecto. 

- WDW: historia de los acontecimientos associated con las ventanas.

- WWH: historia de los acontecimientos associated con las páginas (sólo WEBDEV).

- WDC: código de historia de las clases.

- WDE: historia de los eventos associated con los informes.

- WDR: historia de los eventos associated con las consultas.

- GDTD: historial de códigos de conjuntos de procedimientos (o procedimientos de servidores).

- RED MUNDIAL DE BOSQUES (WWN): historial de códigos de conjuntos de procedimientos de los navegadores (sólo WEBDEV).




En cada subdirectorio, los archivos ".FIC", ".NDX" y ".MMO" correspondientes al nombre de las ventanas, proyectan, ... contener el historial de códigos de cada elemento. Estos archivos pueden ocupar una gran cantidad de espacio después de un rato.. Pueden serlo:

- eliminarlos directamente en el Explorador de Windows.

- borrado del editor de código.




**Atención**: El historial de códigos ya no se mostrará si estos archivos se eliminan o se borran.. Si el historial de códigos está habilitado:

- las siguientes modificaciones hechas al código se guardarán automáticamente,

- los archivos serán recreados si es necesario.

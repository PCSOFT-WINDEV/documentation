
## SCM: Gestión de sucursales
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000096"></a>
La gestión de sucursales se utiliza para gestionar varias versiones de la misma aplicación en paralelo.

**El caso más común es el siguiente**: Ha desplegado una versión de su aplicación y está trabajando en la siguiente versión de la misma.. Mientras tanto, le gustaría aplicar las correcciones de errores realizadas en la versión actual a la versión desplegada y, por lo tanto, crear y desplegar versiones intermedias.

**Otro ejemplo**: Las sucursales también se pueden utilizar para gestionar versiones específicas para cada cliente.. Cada sucursal corresponde a una versión de cliente. Tan pronto como se modifica el proyecto base (tronco común), todo lo que hay que hacer es aplicar las modificaciones en cada una de las ramas.

Las sucursales se utilizan para distribuir versiones de parches de la aplicación a los clientes sin ningún esfuerzo.: no hay necesidad de codificar lo mismo dos veces.

El riesgo de errores debido a la "notificación manual" desaparece.

El confort aumenta, la seguridad aumenta, los obstáculos a la disponibilidad de correcciones de errores desaparecen....

**Observación**: Las modificaciones de rama son bidireccionales. Las modificaciones se pueden aplicar desde el proyecto principal a la rama o desde la rama al proyecto principal.. El proyecto modificado es siempre el proyecto que se abre.

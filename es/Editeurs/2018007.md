## Personalizar las ventanas y las páginas
<a name="personalizar_las_ventanas_las_paginas_ELTTEXTE000302"></a>


### Personalizar las ventanas y las páginas
<a name="personalizar_las_ventanas_las_paginas_ELTPARAGRAPHE000053"></a>

***La ventana/página de inicio de sesión***

Es posible que desee personalizar la ventana/página de inicio de sesión que aparece al iniciar la aplicación. Si esta ventana o página está incluida en su proyecto, el estilo o las plantillas de tema utilizadas por su aplicación pueden aplicarse a esta ventana o página.

**Atención**: Esta ventana/página no debe ser renombrada.

***Todas las ventanas/páginas de groupware usuario***

Si todas las ventanas/páginas de groupware usuario han sido incluidas en su proyecto, todas pueden ser personalizadas. Sin embargo, sólo la ventana/página de inicio de sesión será vista por todos los usuarios.. Las ventanas/páginas para configurar el groupware sólo pueden ser vistas por el supervisor de la aplicación.

**Atención**:

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Las ventanas GPWLogin y GPWSupervisorMenu no deben ser renombradas.




Para obtener más información, consulte [Las ventanas y las páginas de groupware usuario](../Editeurs/2018010.md).
<a name="NOTE2_2"></a>

### Modificar la primera ventana o la primera página según el usuario
<a name="modificar_primera_ventana_primera_pagina_segun_usuario_ELTPARAGRAPHE000092"></a>

Para cambiar la primera ventana de la aplicación o la primera página del sitio según el usuario, se debe llamar a [gpwGetUserInfo](../WDLang6/3041004.md) en el código de inicialización de su proyecto. Esta opción le permite get información sobre el usuario como su nombre o contraseña por ejemplo.

Dependiendo de su nombre, usted tiene la posibilidad de abrir una ventana o una página determinada.

Ejemplo:


```wl
IF gpwGetUserInfo(gpwLogin) = "FP" THEN
	Open(WIN_Management)
ELSE
	Open(WIN_Menu)
END
```

<a name="NOTE2_4"></a>


### Iniciar la ventana/página de inicio de sesión desde una ventana de la aplicación o desde la página de un sitio
<a name="iniciar_ventanapagina_inicio_sesion_desde_una_ventana_aplicacion_desde_pagina_sitio_ELTPARAGRAPHE000108"></a>

Cuando el User Groupware se incluye en una aplicación, puede elegir si el User Groupware se iniciará automáticamente o no. Esta opción puede ser modificada en cualquier momento.

Por defecto (inicio de automatic), la ventana/página de inicio de sesión es la primera ventana/página de la aplicación. Se ejecuta con el código de inicialización del proyecto.

**En algunos casos, es posible que desee abrir una ventana de configuración antes de la ventana de inicio de sesión.**. Para ello, es necesario:

1. Seleccione "Arranque manual".

2. Abra la ventana o página de acceso en cualquier parte del código con [gpwOpen](../WDLang6/3041002.md).

## Definición de los derechos en los menús

### Definir los derechos en los menús emergentes
<a name="definir_los_derechos_los_menus_emergentes_ELTPARAGRAPHE000129"></a>

Para definir los derechos en los menús emergentes, hay que incluir todas las ventanas de groupware usuario. A continuación, puede modificar la ventana de gestión de derechos (***GPWConfigurationDetails.WDW***). 

Para obtener la lista de menús contextuales, utilice [EnumSubElement](../WDLang1/3064002.md) (con las constantes *enumFirst + enumContextualMenu*).
<a name="NOTE3_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) 

### Definir los derechos en el menú de automatic (menú "¿"?")
<a name="definir_los_derechos_menu_automatic_menu_ELTPARAGRAPHE000143"></a>

Algunas opciones del menú de automatic deben ser usadas con mucho cuidado. Las opciones son las siguientes:

- Optimizar velocidad de acceso a la base de datos

- Guardar datos

- Guardar aplicación

- Reparar base de datos.




El modo de funcionamiento de estas opciones con el User Groupware es el siguiente:

- Por Default, estas opciones son Grayed si el usuario no es el supervisor.

- Para permitir que un usuario no supervisor acceda a estas opciones, modifique los derechos de la ventana interna llamada IW_MenuHWinDevHelp que contiene el menú contextual.



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=menu_auto_gpw.gif)


**Observación**: Este comportamiento fue implementado desde la versión 100050 de WINDEV 10. Si ya has personalizado el groupware e incluido la ventana para modificar los derechos en tu aplicación, debes copy:

- la ventana GPWAssociationConfiguration.wdw. Esta ventana se encuentra en el directorio Programas\\Datos\\Modelos\\GPW\\WD de WINDEV.

- el archivo wdxxxgpu.dll que se encuentra en el directorio de Programas de WINDEV.

## Configuración de los archivos de datos de groupware usuario
<a name="configuracion_los_archivos_datos_groupware_usuario_ELTTEXTE000374"></a>
El User Groupware utiliza dos tipos de archivos de datos:

- Archivos de datos de los usuarios

- Ficheros de datos de derechos




La ubicación de estos archivos de datos se puede configurar:

- Al incluir el groupware usuario para la prueba ("Taller .. Configurar el groupware usuario").

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Al crear el ejecutable.

- Al instalar la aplicación (por la persona que realiza la instalación).

- Mediante programación.




Para obtener más información, consulte [Archivos del Groupware usuario](../Editeurs/2018002.md).

## Reinicialización de los archivos de datos
<a name="reinicializacion_los_archivos_datos_ELTTEXTE000398"></a>


### Para reiniciar los archivos de datos de groupware usuario
<a name="para_reiniciar_los_archivos_datos_groupware_usuario_ELTPARAGRAPHE000193"></a>

Lo único que hay que hacer es borrar los ficheros de datos específicos del User Groupware (fichero de usuarios y fichero de derechos). Estos archivos se recrean automáticamente al iniciar la aplicación.
<a name="NOTE5_2"></a>


### Observaciones
<a name="observaciones_ELTPARAGRAPHE000200"></a>

- Si se borran los archivos de usuario (gpwUser Configuración y gpwUser), sólo el supervisor podrá iniciar la aplicación.

- Si se eliminan los archivos de derechos (GPWElementConfiguration, GPWConfiguration y GPWElement), todos los usuarios tendrán todos los derechos sobre la aplicación completa.




Para obtener más información, consulte [Archivos del Groupware usuario](../Editeurs/2018002.md).

**Atención**: El grupo &lt;None&gt; y el grupo Default no pueden ser borrados.

## Modificación de los archivos de datos
<a name="modificacion_los_archivos_datos_ELTTEXTE000428"></a>
La estructura de los archivos de datos específicos de el groupware usuario no puede ser modificada.

**Observación**: Estos archivos contienen información confidencial y, por lo tanto, están encriptados.
<a name="NOTE6_2"></a>


### Gestión de los análisis
<a name="gestion_los_analisis_ELTPARAGRAPHE000224"></a>

Si se abre un análisis específico al configurar el User Groupware, se debe ejecutar de nuevo [gpwInitAnalysis](../WDLang6/3041005.md) (utilizado para abrir el análisis del User Groupware).

**Recordatorio**: En tiempo de ejecución, no se pueden abrir varios análisis al mismo tiempo.

## Presentación
<a name="presentacion_ELTTEXTE000113"></a>
Una aplicación que utiliza el groupware usuario puede ser utilizada con una base de datos HyperFileSQL en modo Cliente/Servidor.

Para que una aplicación que utiliza el User Groupware funcione en modo Cliente/Servidor, debe:

- convertir la aplicación existente en modo Cliente/Servidor. Para obtener más información, consulte [Cliente/Servidor](../WDLang4/3044306.md).

- configurar el User Groupware para utilizar los archivos de datos que se encuentran en el servidor. Estas operaciones se presentan en esta página de ayuda.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000137"></a>


### Modo automático
<a name="modo_automatico_ELTPARAGRAPHE000032"></a>

Para convertir los archivos de datos de groupware usuario en modo Cliente/Servidor: 

1. Seleccione "Taller .. Configurar el"User Groupware".

2. En la pestaña "Ficheros" seleccione "HyperFileSQL Client/Server".

3. Especifique:

	- el nombre del servidor utilizado

	- el número de puerto utilizado

	- el nombre de usuario utilizado para la base de datos.

	- la contraseña de associated

	- el nombre de la base de datos

	- los directorios de los ficheros de datos




4. Si existen los archivos de datos específicos de el groupware usuario (gpwUser, gpwUser Configuration, GPWConfiguration, GPWElement Configuration, GPWElement), copy estos archivos de datos en el servidor.



<a name="NOTE2_2"></a>


### Modo manual
<a name="modo_manual_ELTPARAGRAPHE000051"></a>

Para convertir los archivos de datos de groupware usuario en modo Cliente/Servidor: 

1. El Grupo de Usuarios no debe utilizar el modo "automatic start" en su aplicación.
	Asegúrese de que "automatic ejecutar" está desmarcado en la ventana de configuración de el groupware usuario ("Taller .. Configurar el groupware usuario").

2. En el código de inicialización del proyecto, utilice [gpwOpenConnection](../WDLang6/3041007.md) para describir la conexión a la base de datos HyperFileSQL Cliente/Servidor.

3. Si existen los archivos de datos específicos de el groupware usuario (gpwUser, gpwUser Configuration, GPWConfiguration, GPWElement Configuration, GPWElement), copy estos archivos de datos en el servidor.

4. Llame a [gpwOpen](../WDLang6/3041002.md) para especificar la posición de los archivos de datos utilizados para gestionar los derechos, así como los archivos de datos utilizados para identificar a los usuarios. Los directorios deben especificarse en relación con el directorio del servidor.

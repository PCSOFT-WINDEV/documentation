
## Consultas a múltiples bases de datos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000096"></a>
Una consulta (tecleada en código SQL) puede operar en diferentes bases de datos: HFSQL, Oracle, SQL Server,...

Puede realizar una unión entre una consulta en una base de datos de un tipo y una consulta en una base de datos de otro tipo.

Por lo tanto, los datos se pueden encontrar en diferentes bases de datos. Usted tiene la capacidad de extraer y agregar datos de cualquier fuente.

Esta característica es muy useful para la gestión de los sistemas de información existentes.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000126"></a>
Para comprender mejor el modo de funcionamiento de las consultas de bases de datos múltiples, veamos un ejemplo sencillo:

1. En una base de datos Oracle, [HExecuteSQLQuery](../WDLang4/3044084.md) ejecuta la siguiente consulta (denominada MyQry000032). Esta consulta se utiliza para seleccionar los números de Record de los clientes.
	
	```sql
	Select Customer.CUUniqueKey+10 as CUUniqueKey10 FROM Customer
	```


2. En una base de datos HFSQL, [HExecuteSQLQuery](../WDLang4/3044084.md) ejecuta la siguiente consulta (denominada MyQry000038). Esta consulta se utiliza para seleccionar los números de Record de los clientes y su número de pedido.
	
	```sql
	SELECT CUUNIQUEKEY,Orders.OrderNumber AS ORDERNUM from ORDERS
	```


3. La consulta de base de datos múltiple ejecutada es la siguiente:
	
	```sql
	SELECT * FROM MyQry1,MyQry2
		WHERE MyQry2.CUUniqueKey=MyQry1.CUUniqueKey10
		AND (MyQry2.ORDERNUM = '37348' OR MyQry1.CUUniqueKey10=80)
	```

	Esta consulta es ejecutada por [HExecuteSQLQuery](../WDLang4/3044084.md).

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000307"></a>
Con Default, sólo se instalan en los ordenadores de los usuarios los archivos que se encuentran en la ventana "Lista de archivos instalación".

El editor de instalación WDInst te da la posibilidad de:

- [Instalar herramientas WINDEV distribuibles](#NOTE2_1).

- [Distribuir MDAC](#NOTE3_1).

- [Instalar y configurar el controlador ODBC](#NOTE4_1).

- [Incluir un desinstalador de automatic](#NOTE5_1).




**Observación**: Estas opciones también pueden ser configuradas en el programa de creación de instalación asistente.

**Instalación de elementos adicionales**
Los elementos adicionales a instalar se gestionan en WDInst: en la pestaña "Herramientas", en el grupo "Utilidades", haga clic en "WDInst". 
El programa de creación de instalación asistente también permite instalar elementos adicionales.

## Instalación de herramientas WINDEV distribuibles
<a name="instalacion_herramientas_windev_distribuibles_ELTTEXTE000331"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000053"></a>

Varias herramientas WINDEV pueden ser distribuidas gratuitamente con sus aplicaciones:

| Herramientas | description |
| --- | --- |
| DplViewer | Permite visualizar los duplicados. |
| [ReplicSincro](../ReplicSynchro/3526550.md) | Permite gestionar la Replicación Universal asistida. |
| [WDConver](../WDConver/3512006.md) | Convierte archivos externos a archivos de datos HFSQL. |
| [WDDebug](../Editeurs/2019021.md) | Se utiliza para depurar una aplicación de forma remota |
| [WDLog](../WDJournal/3516012.md) | Guarda y restaura los archivos de datos en formato HFSQL y gestiona los registros. |
| [WDOptimizer](../WDOptimiseur/3519002.md) | Optimiza el acceso a los archivos de datos HFSQL (reindexación, correcciones de notas, etc.). |
| [WDReplic](../WDReplic/3520016.md) | Gestiona la replicación de datos entre ordenadores que utilizan la misma aplicación. |
| [WDSQL](../WDSQL/3521011.md) | Define una conexión y crea consultas SQL en esta conexión. |
| WDTelDiagno | Devuelve la lista de Line s TAPI disponibles y especifica la Line que será utilizada por las funciones de telefonía. |
| [WDTrans](../WDTrans/3524005.md) | Utilizado para las transacciones de Cancel o para liberar registros en la transacción. |


Para no saturar el ordenador del usuario final, los archivos necesarios para estas herramientas se instalan en "\\Archivos de programa\\Archivos comunes\\ PC SOFT\\28.0".. La ruta del directorio de herramientas opcional es devuelta por [SysDir](../WDLang1/3073026.md). Por ejemplo: SysDir(srCommonProgramFiles)+"\\PC SOFT\\28.0".

Si se crea un acceso directo a una aplicación en el menú "Inicio", el acceso directo de cada herramienta instalada en el ordenador del usuario también se añadirá al menú "Inicio" (en el mismo grupo de programas que la aplicación).

El optimizador HFSQL se instalará por defecto si la aplicación es associated con una base de datos HFSQL.
<a name="NOTE2_2"></a>


### Instalar las herramientas WINDEV distribuibles en los ordenadores de los usuarios:
<a name="instalar_las_herramientas_windev_distribuibles_los_ordenadores_los_usuarios_ELTPARAGRAPHE000129"></a>

1. Seleccione "Configuración ... Parámetros avanzados".

2. Seleccione "Incluir herramientas opcionales al crear la instalación.".

3. Haga clic en el botón "Herramientas" para seleccionar las herramientas que desea incluir.

4. Seleccione las herramientas a incluir y validar.




**Observación**: Si instala una herramienta WINDEV sin usar WDInst, los siguientes archivos deben ser suministrados a los usuarios:

- El programa ejecutable de la herramienta (archivo ".EXE").

- Las bibliotecas WINDEV (archivos "wd28 0XXX.DLL").

- Los ficheros Component (ficheros ".WDK") situados en el subdirectorio "Programas" del directorio de instalación de WINDEV.

## Distribuir MDAC
<a name="distribuir_mdac_ELTTEXTE000361"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000157"></a>

Si su aplicación utiliza una base de datos a través de OLE DB, los siguientes elementos deben ser instalados en los ordenadores de los usuarios:

- un proveedor OLE DB ("MSDAORA", "OraOLEDB.Oracle", "SQLOLEDB", ...).

- MDAC (Microsoft Data Access Component) versión 2.6 o posterior.




Si MDAC no está instalado, sólo será posible la conexión a la base de datos OLE DB.: no será posible leer las tablas (un mensaje indica que esta interfaz no es compatible).

**Atención**: La licencia de distribución de MDAC especifica que el ejecutable debe descargarse (del sitio web de Microsoft) para que get tenga los derechos de redistribuirlo con una aplicación. Antes de realizar las operaciones descritas en los siguientes párrafos, debe descargar el ejecutable de MDAC en la computadora actual.

El ejecutable MDAC puede descargarse del sitio web de Microsoft en la siguiente dirección (válida en el momento de redactar esta Document):: [http://www.microsoft.com/data](http://www.microsoft.com/data)
<a name="NOTE3_2"></a>


### Distribución del ejecutable MDAC a los usuarios
<a name="distribucion_del_ejecutable_mdac_los_usuarios_ELTPARAGRAPHE000174"></a>

Distribuir el ejecutable de MDAC a los usuarios:

1. Seleccione "Configuración .. Archivos HFSQL".

2. Marque "Instalar MDAC encontrado en la siguiente ubicación".

3. Especifique el ruta del ejecutable MDAC que se descargó de antemano.

## Instalación y configuración del controlador ODBC
<a name="instalacion_configuracion_del_controlador_odbc_ELTTEXTE000391"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000187"></a>

Para manipular una base de datos HFSQL de una aplicación no WINDEV, el controlador ODBC en HFSQL debe ser instalado y configurado en el ordenador del usuario. Para obtener más información, consulte [Controlador ODBC en HFSQL](../WDLang4/3044179.md).

**Observación**: El controlador ODBC en HFSQL solo se puede usar para **leer y escribir** en una base de datos HFSQL desde una aplicación que no sea WINDEV.
<a name="NOTE4_2"></a>


### Instalación y configuración del controlador ODBC
<a name="instalacion_configuracion_del_controlador_odbc_ELTPARAGRAPHE000199"></a>

Para instalar y configurar el controlador ODBC:

1. Seleccione "Configuración .. Archivos HFSQL".

2. Marque "Instalar el controlador ODBC para HFSQL".

3. Introduzca el nombre utilizado para identificar la base de datos HFSQL cuando utilice el controlador ODBC en HFSQL.

4. Introduzca el ruta del análisis utilizado como fuente ODBC (archivo.WDD correspondiente al análisis WINDEV).

## Incluido un desinstalador de automatic
<a name="incluido_desinstalador_automatic_ELTTEXTE000421"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000213"></a>

Para simplificar la desinstalación de Process de su aplicación, WDInst le permite incluir un desinstalador en el programa instalación.

En este caso, se creará el archivo "uninst.INF" en el directorio de instalación de la aplicación. Este archivo enumera todos los archivos de aplicación instalados en el ordenador actual.

**Observación**: Para desinstalar la aplicación, el desinstalador utiliza una entrada en el Registro. El editor WDInst de instalación le permite modificar este identificador. Para obtener más información, consulte [Modificar el identificador utilizado para desinstalar la aplicación](../Editeurs/2028013.md).
<a name="NOTE5_2"></a>


### Desinstalación de la aplicación
<a name="desinstalacion_aplicacion_ELTPARAGRAPHE000227"></a>

Como para cualquier aplicación de Windows que incluya un desinstalador, para desinstalar una aplicación, debe hacerlo:

- abrir la lista de programas instalados en el sistema actual (opción "Agregar o quitar programas" en el Panel de control).

- seleccionar la aplicación en la lista de programas.

- haga clic en el botón "Modificar/Eliminar".
	Se suprimirán los siguientes elementos:

	- los archivos enumerados en el archivo "uninst.INF".

	- las entradas en el registro creadas al instalar la aplicación.

	- los accesos directos a las aplicaciones (en el escritorio y en el menú "Inicio"), si existen.

	- los ficheros de datos (ficheros ".FIC", ".MMO" y ".NDX") tras la confirmación del usuario.

	- los archivos guardados durante las diferentes actualizaciones (si existen) después de la confirmación del usuario.

	- los directorios cuyos archivos se borran durante la desinstalación de Process.







**Caso especial**: Si la aplicación crea archivos de datos en un directorio específico (diferente de los subdirectorios de la aplicación), estos archivos de datos no se borrarán automáticamente durante la desinstalación de Process.

**Observación**: Usted tiene la capacidad de traducir los diferentes mensajes del desinstalador. Para obtener más información, consulte [Traducción del desinstalador](../Editeurs/2028087.md). 
<a name="NOTE5_3"></a>

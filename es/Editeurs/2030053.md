


## Archivo de sitemaps para Google
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000119"></a>
Para analizar mejor el contenido de los sitios, el motor de búsqueda de Google utiliza un archivo XML llamado "Sitemap" (si está disponible).

Este archivo, que se encuentra en el servidor que aloja el sitio, describe los enlaces a las páginas de reference. Este archivo se genera automáticamente cuando WEBDEV instala un sitio.. Este archivo está disponible para los sitios AWP, para los sitios estáticos y para los sitios PHP.

El archivo es un complemento a la referencia asistente y se utiliza para mejorar las páginas del sitio reference. Por ejemplo, si una página no incluye ningún enlace que pueda ser utilizado por un robot (por ejemplo, un enlace que contenga código JavaScript), la página se describe en este archivo (el enlace se convierte en un formato que puede ser analizado por los motores) y se hace referencia a él.

<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000143"></a>
Para crear un archivo de sitemaps: 

1. Crea tu sitio estático, PHP o AWP.

2. Para cada página de su sitio, especifique:

	- si la página debe ser referenciada o no ("No permitir SEO" en la pestaña "Detalles" de la página description). Si la página debe estar referenciada, aparecerá en el archivo Sitemap.

	- las expresiones y palabras clave asociadas con cada página referenciable del sitio (pestaña "Detalles" de la ventana description de la página). Estas palabras clave serán utilizadas por los robots para reference las páginas.

	- la frecuencia de actualización de su página (todos los días, todas las semanas, ....). Si no lo sabe, le aconsejamos que seleccione "Indefinido" en lugar de una información incorrecta.

	- la prioridad interna de la página (alta, baja, ....). Esta prioridad le permite proponer una página de su sitio en lugar de otra al realizar búsquedas.




3. Instale su sitio a través de las opciones de instalación suministradas por WEBDEV:

	- para un sitio WEBDEV AWP, utilice el asistente para la creación de instalación: en la pestaña "Proyecto", en el grupo "Generación", despliegue "Implementar sitio" y seleccione "Generar e implementar sitio".

	- para un sitio WEBDEV estático o para un sitio PHP, utilice WDDeploy: en la pestaña "Herramientas", en el grupo "Web utilities", haga clic en "WDDeploy".







El archivo "Sitemap.xml" se genera automáticamente en el servidor cuando se realiza una instalación en la raíz del directorio &lt;Nombre_del_proyecto&gt;_WEB y en la raíz de cada directorio del idioma.

Observaciones: 

- Se genera un archivo "_Sitemap.xml" en el ordenador de desarrollo. Este archivo contiene las referencias a las páginas incluidas en el archivo SiteMap.. Este archivo no se puede utilizar porque se llena/modifica automáticamente al instalar el sitio.

- Si el sitio contiene una [mapa del sitio](../WDChamp/1013300.md) con URLs con parámetros, estas URLs se referencian automáticamente en el archivo SiteMap.

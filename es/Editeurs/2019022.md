


## Depuración de una conexión a un servidor HFSQL
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000194"></a>
Los procedimientos almacenados y los desencadenantes se depuran desde el entorno, con ejecución real en el servidor.. Es posible:

- ejecutar la prueba de un Procedure almacenado: por sí mismo desde el editor (véase [Procedimiento almacenado](../WDLang4/3044360.md)).

- conectar el depurador a una conexión existente para rastrear un problema en tiempo real.




Este último caso será presentado en esta página de ayuda.


<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000224"></a>


### Probar una conexión HFSQL Client/Server
<a name="probar_una_conexion_hfsql_clientserver_ELTPARAGRAPHE000027"></a>

Para probar una conexión HFSQL Client/Server:

1. Abra el proyecto deseado en el editor.

2. Mostrar el análisis relacionado al proyecto: haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Ico_Analyse.gif) en los botones de acceso rápido.

3. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Depurar en un servidor HFSQL".

4. En la pantalla que se visualiza, seleccione:

	- el usuario identificado en el servidor para la depuración.

	- el modo para ejecutar la prueba y el depurador.







Hay tres modos de arranque disponibles:

- Iniciar el proyecto y depurar las siguientes conexiones a la base de datos.

- Depurar las siguientes conexiones a la base de datos procedentes de un usuario específico para una aplicación específica.

- Depurar una conexión existente.



<a name="NOTE2_2"></a>


### Moda 1: Inicio del proyecto y depuración de las siguientes conexiones a la base de datos
<a name="moda_1_inicio_del_proyecto_depuracion_las_siguientes_conexiones_base_datos_ELTPARAGRAPHE000062"></a>

Esta opción permite:

- iniciar el proyecto actual (sin el depurador),

- depurar los procedimientos almacenados y los desencadenantes utilizados en el proyecto en una base de datos. Se ejecutan los elementos desplegados en el servidor.




**Para usar este modo de depuración**: 

- El puerto utilizado para la depuración es el puerto 27281 de Default. Este puerto debe abrirse si se utiliza un cortafuegos.. De hecho, la comunicación entre el ordenador de desarrollo y el ordenador del usuario final se realiza a través de un socket. Este puerto puede modificarse en el programa [Archivo HFConf.INI](../WDLang4/3044345.md).

- El usuario que maneje la base de datos debe tener los derechos para depurar la base de datos.




**Observaciones**: 

- La combinación Ctrl + Pausa no se puede utilizar para iniciar el depurador.. Tiene la capacidad de utilizar puntos de ruptura.

- Las teclas F7 no pueden utilizarse en [HExecuteProcedure](../WDLang4/3044358.md) o [HExecuteQuery](../WDLang4/3044080.md).

- [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>False</u></u></u></u>.



<a name="NOTE2_3"></a>


### Moda 2: Depuración de las siguientes conexiones a la base de datos procedentes de un usuario específico para una aplicación específica
<a name="moda_2_depuracion_las_siguientes_conexiones_base_datos_procedentes_usuario_especifico_para_una_aplicacion_especifica_ELTPARAGRAPHE000098"></a>

Esta opción le permite depurar los procedimientos almacenados y los disparadores utilizados desde cualquier conexión que cumpla con los criterios de filtrado.. Si los criterios de filtrado no son lo suficientemente restrictivos, puede haber demasiadas llamadas al depurador.. Se ejecutan los elementos desplegados en el servidor.

**Para usar este modo de depuración**: 

- El puerto utilizado para la depuración es el puerto 27281 de Default. Este puerto debe abrirse si se utiliza un cortafuegos.. De hecho, la comunicación entre el ordenador de desarrollo y el ordenador del usuario final se realiza a través de un socket. Este puerto puede modificarse en el programa [Archivo HFConf.INI](../WDLang4/3044345.md).

- El usuario que maneja la base de datos debe tener:

	- los derechos para depurar la base de datos,

	- los derechos para depurar cualquier conexión.





**Observación**: [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>False</u></u></u></u>.

<a name="moda_3_depuracion_una_conexion_existente_ELTPARAGRAPHE000125"></a>

**Para usar este modo de depuración**: 

- El puerto utilizado para la depuración es el puerto 27281 de Default. Este puerto debe abrirse si se utiliza un cortafuegos.. De hecho, la comunicación entre el ordenador de desarrollo y el ordenador del usuario final se realiza a través de un socket. Este puerto puede modificarse en el programa [Archivo HFConf.INI](../WDLang4/3044345.md).

- El usuario que maneja la base de datos debe tener:

	- los derechos para depurar la base de datos,

	- los derechos para depurar cualquier conexión.





**Observación**: [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>False</u></u></u></u>.

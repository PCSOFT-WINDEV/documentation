## Presentación
<a name="presentacion_ELTTEXTE000277"></a>
El archivo MDL es el archivo utilizado para generar una ventana RAD. Este archivo contiene toda la description del patrón a generar. Esta description está compuesta de dos partes distintas:

- la description del patrón,

- la description de la ventana.




Esta página de ayuda sólo presenta el código utilizado para describir el patrón. Para presentar este código en detalle, el script de una **Formulario.mdl** se utiliza el patrón.

**Atención**: **En la versión 12, el RAD fue reconstruido completamente. Los archivos MDL se guardan para compatibilidad con versiones anteriores. Para obtener más información, consulte [Nuevos patrones RAD](../Editeurs/2031020.md).**

## Las diferentes partes del patrón description
<a name="las_diferentes_partes_del_patron_description_ELTTEXTE000301"></a>


### 1. Las características del patrón reutilizado en la asistente:
<a name="1_las_caracteristicas_del_patron_reutilizado_asistente_ELTPARAGRAPHE000028"></a>


```txt
[Pattern] 
############################# 
# name of the pattern 
############################# 
Name=Form with browse 
Title=Form 
Number=2 
############################# 
# description of the pattern 
############################# 
Description=Beginning 
Form pattern with browse buttons. 
Used to enter, modify and delete records. 
Used to view records. 
Description=End 
############################# 
# classes for OOP 
############################# 
Class=CWDHandleFile 
Class=CWDForm 
############################# 
# group of files 
############################# 
GPFile=A 
#accepts the queries 
Query=Yes 
############################# 
# group of controls 
############################# 
ControlGroup=GP_CONTROL 
BrowseGroup=GP_BROWSE 
Group=GP_BROWSE 
Group=GP_MODIF 
LinkGroup=GP_LINK 
Group=GP_APPLY
```

<a name="NOTE2_2"></a>
Los diferentes elementos utilizados en este código son los siguientes:


|   |   |
| --- | --- |
| **[ Patrón]** | Señala el comienzo de la sección para el patrón description |
| **denominar** | Nombre del patrón, mostrado en la asistente para la creación de la ventana. |
| **Título** | Título de la ventana generada |
| **Number** | Número del patrón RAD. Once patrones RAD están disponibles en Default |
| **description = Comienzo** | Comienzo de description de las características del patrón. Estas características se mostrarán en la asistente para la creación de la ventana |
| **Description = End** | El final de description de las características del patrón. |
| **Clase** | La ventana se puede crear en modo OOP. La opción correspondiente será Visible en la asistente.<br>Nombre de las clases encontradas en el patrón _EASLASH_\\RAD que deben ser copiadas al directorio del proyecto para que la ventana funcione correctamente. |
| **GPFile=A** | Indica que se debe elegir un solo tipo de archivo en la asistente.<br>**GpFile=A(N1)B** indica que hay que seleccionar dos ficheros en el asistente. Estos dos archivos deben ser relacionado en el análisis por un enlace 1-n. |
| **Query=Yes** | Indica que, para este patrón RAD, la asistente puede proponer tanto los archivos definidos en el análisis como las consultas creadas en el editor de consultas.<br>Si no se deben utilizar las consultas, indique **Query=No** |
| **control Group=GP_control**<br>**ExaminarGrupo=GP_BROWSE**<br>**Grupo=GP_BROWSE**<br>**Grupo=GP_MODIF**<br>**LinkGroup=GP_LINK**<br>**Grupo=GP_APPLY** | Define los nombres de los diferentes grupos de controles a crear.<br>**Grupo de control**: para los controles de edición en la ventana<br>**ExaminarGrupo**: grupo para los controles Visible en modo de navegación<br>**Grupo**: grupo de controles<br>**LinkGroup**: para los botones de enlace |

### 2. Código del proyecto
<a name="2_codigo_del_proyecto_ELTPARAGRAPHE000111"></a>


```txt
WinCode=Beginning
%define PCODEPRJ OpenPrj
%ifdef FILE
@// Create the files
IF NOT HCreationIfNotFound({RAD_FILE_A}) THEN Info("Unable to create 
or to initialize the access to {RAD_FILE_A}",HErrorInfo())
@
%endif
%enddef PCODEPRJ
```

<a name="NOTE2_4"></a>
Los diferentes elementos utilizados en este código son los siguientes:


|   |   |
| --- | --- |
| **WinCode=Inicio** | Señala el comienzo de la sección de código para la ventana. El final del código se indica con **WinCode=End** |
| **%definir** | Señala el comienzo de un área de código Definition. El final de este Definition está marcado por la palabra clave **%enddef** |
| **ifdef** | Señala el comienzo de un código condicional. La declaración completa es la siguiente:<br>**ifdef .... %else %endif** |
| **PCODEPRJ** | Define el tipo de expresión que se describe actualmente:<br><br>- **LOCPROC**: description de una Procedure local<br><br>- **GLOBPROC**: description de una Procedure global<br><br>- **CÓDIGO**: description de una Process específica<br><br>- **PCODEPRJ**: description de una Process associated con el proyecto<br><br><br> |
| **OpenPrj** | El tipo de Process actualmente descrito<br><br>- "**Apertura**": Código de declaración de variables globales de ventana<br><br>- "**FinalizaciónInicialización**": Código para abrir la ventana<br><br>- "**Cierre**": Código de cierre de la ventana<br><br>- "**Modificación**": Código cada vez que se modifica la ventana<br><br>- "**OpenPrj**": Código de inicialización del proyecto<br><br>- "**CerrarPrj**": Código de cierre del proyecto<br><br>- "**Initialization**": Código de inicialización de un botón<br><br>- "**Haga clic**": Haga clic en el código de un botón<br><br>- "**Entrada de línea**": Código para introducir un tabla fila<br><br>- "**Selección de línea**": Código para seleccionar un tabla fila<br><br><br> |
| **FILE** | Se utiliza para saber si un archivo fue seleccionado por el usuario en la asistente para la creación de la ventana.<br>**QUERY**: utilizado para averiguar si una consulta fue seleccionada por el usuario en la asistente para la creación de la ventana. |
| **@** | Marca el principio y el final del código para insertarlo en el Process especificado al generar la ventana. |
| **{RAD_FILE_A}** | Etiqueta utilizada para identificar el archivo seleccionado por el usuario en la asistente. A identifica el primer fichero, B el segundo, ... Esta etiqueta se reemplaza automáticamente en el código generado por el nombre del archivo.<br>Las otras etiquetas utilizadas son:<br><br>- **RAD_ARCHIVO_DE_RAD**: Archivo Variable<br><br>- **Formulario_**: Ventana del formulario Variable<br><br>- **LLAVE_RAD**: clave Variable<br><br>- **RAD_SOURCE_KEY**: Nombre del elemento antes de que se utilice en la consulta (antes del alias)<br><br>- **RAD_KEY_control_**: Controla a associated con la llave<br><br>- **RAD_VENTANA**: ventana Variable<br><br>- **RAD_LINKKEY**: clave de enlace entre 2 ventanas<br><br>- **RAD_SOURCE_LINKKEY**: Nombre del elemento clave del enlace antes de que se utilice en la consulta (antes del alias)<br><br>- **RAD_ORIGIN_LINKKEY**: Nombre del elemento clave del enlace en el fichero actual (no en el fichero relacionado)<br><br>- **RAD_ORIGEN_FUENTE_DE_ENLACE**: Nombre del elemento clave del enlace en el archivo actual antes de que se utilice en la consulta (antes del alias)<br><br>- **RAD_LINKKEY_control**: Controla associated con la tecla de enlace entre 2 ventanas<br><br>- **INFORME_RAD**: reporte Variable<br><br>- **RAD_VISION_COMBOBOX**: Variable que contiene el nombre del cuadro combinado auto-llenado<br><br>- **RAD_FICHERO_VISIÓN**: Variable que contiene el nombre del archivo para los botones de Vision+.<br><br>- **RAD_VISION_KEY**: Variable que contiene el nombre de la tecla para los botones de Vision+.<br><br>- **RAD_VISIÓN_VENTANA**: Variable que contiene el nombre de la ventana Vision+ a abrir<br><br>- **Reporte_Form_**: Variable que contiene el nombre de una forma reporte<br><br>- **Reporte_Tabla_**: Variable que contiene el nombre de un tabla reporte<br><br>- **RAD_ITEM_ITEM**: artículo Variable<br><br>- **RAD_LAST_ITEM_ITEM**: Variable que contiene el nombre del último elemento del archivo<br><br>- **WWW.SUBS-TEAM.TV P R E S E N T A.**: Variable que contiene el nombre de una consulta de selección<br><br>- **COTIZAR_CONTRATO**: para las variables que deben estar necesariamente entre comillas<br><br>- **NOQUOTE_SOLICITAR**: prefijo para las variables que no deben estar necesariamente entre comillas<br><br><br> |
| **%endif** | Señala el final de un bucle condicional |
| **%deferencia** | Señala el final de un código Definition |

### 3. Código de la ventana
<a name="3_codigo_ventana_ELTPARAGRAPHE000210"></a>


```txt
%define PCODE Open 
%ifdef PROC 
.... 
@ 
%else 
%ifdef SQL 
@ 
... 
@ 
%endif 
%endif
```

<a name="NOTE2_6"></a>
Los diferentes elementos utilizados en este código son los siguientes:


|   |   |
| --- | --- |
| **PROC** | Permite saber si el modo de programación del procedimiento fue elegido por el usuario. |
| **SQL** | Permite saber si el modo de programación SQL ha sido elegido por el usuario. |
| **ODBC** | Permite saber si el modo de programación ODBC ha sido elegido por el usuario. |

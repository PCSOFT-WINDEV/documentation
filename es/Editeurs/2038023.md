


## Administrador del SCM: herramientas disponibles
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000531"></a>
El administrador de SCM (Administrador de Código Fuente) le permite: 

- configurar la [opciones de administrador y SCM](#NOTE2_1). 

- [gestionar y optimizar el repositorio SCM](#NOTE3_1). 




Estas herramientas sólo pueden ser utilizadas por un administrador. 

Observación: El administrador de SCM también puede utilizarlo para gestionar los elementos del repositorio, etc.. Para obtener más información, consulte [Vista general del administrador de SCM](../Editeurs/2038008.md).

<a name="NOTE2_1"></a>


## Opciones de SCM
<a name="opciones_scm_ELTTEXTE000555"></a>
Para configurar las opciones de SCM, en la pestaña "Herramientas", en el grupo "Opción", haga clic en "Opciones".
<a name="NOTE2_2"></a>


### Opciones generales
<a name="opciones_generales_ELTPARAGRAPHE000050"></a>

La pestaña "General" de las opciones de SCM se utiliza para configurar la visualización de los cuadros de diálogo durante una acción realizada por el SCM. Se puede visualizar una ventana de diálogo en la que se indica cuándo:

- Registrarse sin modificaciones.

- Añadir archivos al repositorio.

- Actualización de las dependencias de un elemento.

- Solicitar una salida explícitamente (a través de la barra de herramientas, por ejemplo).

- Señalización de que un proyecto ha sido añadido con éxito.




La Image del cuadro de diálogo correspondiente se muestra cuando las diferentes opciones son pasadas por el cursor del ratón.
<a name="NOTE2_3"></a>


### Opciones en cuanto a la mensajería
<a name="opciones_cuanto_mensajeria_ELTPARAGRAPHE000065"></a>

La pestaña "Mensajería" de las opciones de SCM se utiliza para especificar si se debe utilizar la mensajería integrada.. En este caso, se pueden enviar mensajes a los demás usuarios (solicitud de desbloqueo, información sobre las operaciones de facturación, etc.).

Si es necesario, puede pedir que se envíe un mensaje a los usuarios de SCM cada vez que se vuelva a comprobar un elemento en el SCM.
<a name="NOTE2_4"></a>


### Opciones sobre las sucursales
<a name="opciones_sobre_las_sucursales_ELTPARAGRAPHE000074"></a>

La pestaña "Ramas" sirve para configurar la ruta creada por Default cuando se utilizan las ramas.

Esta pestaña permite especificar los directorios remotos y locales utilizados por Default al crear una rama. Para obtener más información, consulte [Gestión de ramas](../Editeurs/2038016.md).
<a name="NOTE2_5"></a>


### Opciones avanzadas
<a name="opciones_avanzadas_ELTPARAGRAPHE000086"></a>

La pestaña "Avanzado" sirve para configurar el rendimiento del repositorio para un repositorio HFSQL Client/Server al que se accede por ADSL.

En algunos casos, "Limitar el tamaño de los paquetes para ADSL" se utiliza para gestionar las características específicas de algunos routers.

**Atención**: Si esta opción no se utiliza correctamente, puede ralentizar la SCM.

## Gestión de la SCM
<a name="gestion_scm_ELTTEXTE000609"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000101"></a>

Las herramientas de administración están disponibles sólo cuando se conecta al repositorio SCM como Administrador. Para conectarse al repositorio SCM como administrador, en la pestaña "Inicio", en el grupo "Repositorio", haga clic en "Conexión ".. 

En la ventana que aparece:

- Especifique las características del depósito utilizado.

- Especifique el nombre y la contraseña del administrador del repositorio:

	- Si utiliza un repositorio SCM en formato HFSQL Classic, el administrador corresponde a "Admin" de Default (sin contraseña).

	- Si está utilizando un repositorio SCM en formato HFSQL Client/Server, el administrador corresponde al usuario "Administrator" de Default.







**Observación**: Establecer una conexión con el repositorio SCM como administrador difiere de "Convertirse en maestro del proyecto".. Para realizar las operaciones de gestión, debe estar conectado como administrador.


<a name="NOTE3_2"></a>


### Configuración de la autenticación
<a name="configuracion_autenticacion_ELTPARAGRAPHE000132"></a>

La configuración de la autentificación permite utilizar la base de datos de los Centros control para autentificar a los usuarios de SCM. El nombre de usuario y la contraseña son los que se especifican en el programa Centro de Seguimiento de Proyectos.

Esta opción sólo está disponible si se trabaja con un repositorio SCM en formato HFSQL Classic.

Para habilitar la autenticación:

1. En la pestaña "Administración", en el grupo "Seguridad", haga clic en "Configurar autenticación".

2. Marque "Activar autenticación".

3. Especifique la base de datos de los Centros control utilizados (en formato HFSQL Classic).

4. Validar. 



<a name="NOTE3_4"></a>


### Cancelación de salidas
<a name="cancelacion_salidas_ELTPARAGRAPHE000160"></a>

El administrador puede Cancel todas las salidas en curso en el proyecto seleccionado: en la pestaña "Administración", en el grupo "Extracciones", haga clic en "Deshacer extracciones".

Una ventana lista todos los elementos comprobados. También puede listar los elementos comprobados para las pruebas o los elementos comprobados en un directorio remoto diferente.

Simplemente seleccione los elementos deseados y haga clic en "Deshacer el check-out". La salida se cancela. El elemento está en el depósito en su estado antes de la salida.

Tiene la posibilidad de informar a los usuarios ("Enviar un mensaje a aquellos cuyo check-out fue cancelado").
<a name="NOTE3_5"></a>


### Copia de seguridad del repositorio
<a name="copia_seguridad_del_repositorio_ELTPARAGRAPHE000185"></a>

El administrador puede hacer una copia de seguridad copy del repositorio SCM: en la pestaña "Administración", en el grupo "Copias de seguridad", haga clic en "Copia de seguridad completa". Esta copia de seguridad copy se guarda en un archivo .ZIP. 
**Observación**: Durante la copia de seguridad, se puede acceder al repositorio SCM en modo de sólo lectura.

La copia de seguridad del repositorio también se puede realizar en un comando Line a través de la siguiente anotación:


```txt
WDGDSAdmin.exe /SAVE 
		/BASE=<Address of the repository> 
		/LOGIN=<Login> 
		/PASS=<Password> 
		/ZIP=<Full name of zip file>
```
En este código, &lt;Address del repositorio&gt; tiene el siguiente formato: 

- Para HFSQL Client/Server: "CS:\\\\&lt;Dirección IP del servidor&gt;:&lt;Puerto servidor&gt;\\&lt;Nombre repositorio SCM&gt;"

- Para HFSQL Classic: se debe especificar el ruta local del repositorio. Por ejemplo: "D:\\PC SOFT\\Repositorio local de SCM _ASLASH_"




Ejemplo: 


```wl
ExeRun("""C:\WINDEV\Programs\WDSCMAdmin.exe"" /SAVE /BASE=CS:\\DOC2:4900\GDS " + ...
		" /LOGIN=admin /PASS=admin /ZIP=C:\Temp\test.zip")
```

<a name="NOTE3_6"></a>


### Guardar y borrar el historial
<a name="guardar_borrar_historial_ELTPARAGRAPHE000218"></a>

El administrador tiene la posibilidad de borrar parte del historial del repositorio de SCM.: en la pestaña "Administración", en el grupo "Copias de seguridad", haga clic en "Borrar historial con copia de seguridad". La parte eliminada se guarda automáticamente en un archivo en formato.ZIP. Esta copia de seguridad se puede restaurar posteriormente.

Para grabar y borrar el historial, especifique la fecha a partir de la cual se debe almacenar el historial.
<a name="NOTE3_7"></a>


### Restauración de una copia de seguridad completa
<a name="restauracion_una_copia_seguridad_completa_ELTPARAGRAPHE000239"></a>

Para restaurar una copia de seguridad completa del repositorio, en la pestaña "Administración", en el grupo "Copias de seguridad", haga clic en "Restauración completa".
<a name="NOTE3_8"></a>


### Restauración de una copia de seguridad del historial
<a name="restauracion_una_copia_seguridad_del_historial_ELTPARAGRAPHE000258"></a>

Para restaurar una copia de seguridad del historial del repositorio, en la pestaña "Administración", en el grupo "Copias de seguridad", haga clic en "Restaurar historial".

## Mantenimiento de la SCM
<a name="mantenimiento_scm_ELTTEXTE000675"></a>
El administrador de SCM propone varias opciones de mantenimiento. Estas opciones le permiten optimizar y gestionar el repositorio SCM en cualquier momento.
<a name="NOTE4_2"></a>


### Migración del repositorio al formato más reciente
<a name="migracion_del_repositorio_formato_mas_reciente_ELTPARAGRAPHE000284"></a>

Para migrar el repositorio SCM a un formato más reciente, en la pestaña "Herramientas", en el grupo "Mantenimiento", haga clic en "Migrar al formato más reciente ". Este formato optimiza los accesos a la base de datos.

**Observación**: Para utilizar esta opción, el repositorio SCM no debe estar instalado directamente en un nombre compartido.

Por ejemplo, si los archivos están instalados en "\\_ASLASH_\\Server\\SCM" donde "SCM" es el nombre del directorio compartido, la opción no funcionará.. Los archivos deben estar instalados en un directorio como "\\_ASLASH_ Server\\SCM\\SCMRepository".

Si no es así, los archivos de datos se pueden mover manualmente (todos los usuarios deben ser desconectados).. La próxima vez que se abra un proyecto, tendrá que especificar el nuevo directorio del repositorio SCM.

Atención: No dude en consultar la página de ayuda de [Evolución del formato de los archivos SCM](../Editeurs/2038018.md). 
<a name="NOTE4_3"></a>


### Cambiar el repositorio a Cliente/Servidor
<a name="cambiar_repositorio_clienteservidor_ELTPARAGRAPHE000314"></a>

Para abrir el asistente que permite cambiar el repositorio al modo HFSQL Client/Server, en la pestaña "Herramientas", en el grupo "Mantenimiento", haga clic en "Cambiar a Cliente/Servidor".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GDS_Client_serveur.gif)
 Esta operación sólo puede realizarse si hay un motor HFSQL Client/Server instalado en el servidor donde debe instalarse el repositorio SCM.

Debe especificar:

- El nombre del servidor. 

- El nombre de la base de datos y el puerto.

- El nombre y la contraseña de un administrador de la base de datos HFSQL.




Dependiendo del tamaño del repositorio SCM, la migración del repositorio puede llevar bastante tiempo.

Al final de esta operación, SCM propone crear automáticamente la cuenta de usuario correspondiente al usuario conectado (si esta cuenta no existe).

La próxima vez que se abra el proyecto, tendrá que especificar los nuevos parámetros para conectarse a SCM.

**Observación**: El repositorio SCM en modo "Clásico" no se borra. Una vez finalizada la migración al formato HFSQL Client/Server, debe informar a todos los usuarios para que puedan cambiar sus parámetros de conexión al repositorio SCM.


<a name="NOTE4_4"></a>


### Re-índice y reparación
<a name="reindice_reparacion_ELTPARAGRAPHE000351"></a>

En la pestaña "Herramientas", en el grupo "Mantenimiento", haga clic en "Reindexar y reparar". Esta opción permite:

- Reindexar el repositorio SCM (ya sea en modo HFSQL Classic o en modo HFSQL Client/Server).

- Cancel las transacciones bloqueadas en el depósito de SCM.

- Analizar y reparar el repositorio SCM.





<a name="NOTE4_5"></a>


### Reparar un directorio
<a name="reparar_directorio_ELTPARAGRAPHE000378"></a>

En algunos casos, la reparación de un directorio se utiliza para solucionar algunos problemas relacionados con el acceso a los datos que se encuentran en un directorio de SCM. Esta opción se utiliza para reparar el directorio sin bloquear el repositorio SCM durante esta operación. 

Para reparar un directorio: 

- Seleccionar el directorio a reparar en la lista de directorios del repositorio SCM. 

- En la pestaña "Herramientas", en el grupo "Mantenimiento", haga clic en "Reparar un directorio". Esta opción se utiliza para reparar el repositorio de SCM para este directorio.




Observación: Si es necesario, SCM propone automáticamente reparar sólo el directorio (en lugar de reparar todo el repositorio de SCM)..

## Realizar una búsqueda en el SCM
<a name="realizar_una_busqueda_scm_ELTTEXTE000735"></a>
Hay varios modos de búsqueda disponibles en el repositorio de SCM:

- **Buscar por nombre** (en la pestaña "Herramientas", en el grupo "Buscar", haga clic en "Por nombre"): Esta opción se utiliza para encontrar fácilmente un elemento en el repositorio SCM. Simplemente especifique el nombre del elemento a buscar.

- **Buscar por extracción** (en la pestaña "Herramientas", en el grupo "Buscar", haga clic en "Por extracción"): Esta opción se utiliza para encontrar las cajas según el usuario (usuario actual, usuario específico, todos los usuarios) y el tipo de caja (para prueba, exclusiva, etc.).. Tiene la capacidad de registrar la entrada de un elemento o la salida de Cancel.

- **Búsqueda por tarea o incidente**: Esta opción se utiliza para encontrar los elementos correspondientes a la ejecución de una tarea o incidente.. Puedes ver los cambios realizados.

- **Búsqueda en el historial**: Esta opción se utiliza para realizar una búsqueda en el historial de acciones realizadas en esta SCM. Esto le permite a get todos los comentarios escritos entre dos fechas y exportarlos a un archivo. Para obtener más información, consulte [Buscar en la historia de SCM](../Editeurs/2038028.md).

## Varias opciones
<a name="varias_opciones_ELTTEXTE000759"></a>


### Mis Proyectos
<a name="mis_proyectos_ELTPARAGRAPHE000447"></a>

Para listar los proyectos con un directorio local en el ordenador actual: 

1. En la pestaña "Herramientas", en el grupo "Buscar", haga clic en "Mis proyectos". 

2. En la ventana que aparece, especifique el acceso deseado si es necesario y haga clic en "Buscar". 

3. El directorio remoto y el directorio local se muestran para cada proyecto encontrado.




Tiene la posibilidad de eliminar la asociación entre el directorio remoto y el directorio local (botón"Eliminar la asociación").. Esta opción es useful cuando se abrió un proyecto de SCM y luego se borró del ordenador local: Al eliminar la asociación, el usuario ya no recibirá los mensajes de automatic de SCM sobre este proyecto.
<a name="NOTE6_2"></a>


### Ordenador renombrado o cambiado
<a name="ordenador_renombrado_cambiado_ELTPARAGRAPHE000472"></a>

La opción "Herramientas .. Renombrar o cambiar ordenador" se utiliza para actualizar el nombre de el equipo utilizado en la SCM. Para obtener más información, consulte [Renombrar o cambiar de ordenador](../Editeurs/2038020.md).
<a name="NOTE6_3"></a>


### Estadísticas
<a name="estadisticas_ELTPARAGRAPHE000482"></a>

La opción "Herramientas .. Estadísticas" se utiliza para los cálculos estadísticos de get sobre el repositorio SCM.

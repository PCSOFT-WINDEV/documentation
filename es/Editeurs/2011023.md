
## Campos clave
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000130"></a>
Las claves son campos específicos del análisis. Estos campos permiten:

- **identificar fácilmente los registros**. Por ejemplo, en una ficha de producto, la referencia identifica el producto. La referencia es única para cada producto.

- **organizar los datos almacenados en un archivo de datos**. Por ejemplo, una clave definida en el campo Name del archivo de datos Customer le permitirá ordenar rápidamente los registros del archivo de datos por nombre.




Se pueden definir varios tipos de claves en un archivo de datos

- [Clave única](#NOTE2_1).

- [Clave con duplicados](#NOTE3_1).

- Clave primaria.

- [Clave simple o clave compuesta](#NOTE4_1).




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Clave única
<a name="clave_unica_ELTTEXTE000154"></a>
Una clave única es una clave para la que solo puede haber un valor en el archivo de datos.

Por ejemplo, en un archivo "PRODUCT", cada producto está identificado con una referencia única. El campo "REFERENCIA" debe ser una clave única.

**Observaciones**: 

- Si no se puede definir ningún campo como clave única, utilice una campo de tipo "ID automático". Para cada registro, el valor de este campo será un número entero único.

- Clave única con NULL como valor predeterminado: El valor NULL se considera como una cadena vacía ("") cuando se utiliza en el índice de campos de tipo String, y como 0 cuando se utiliza en campos numéricos. Por lo tanto, las claves únicas no pueden contener valores NULL. 




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Clave con duplicados
<a name="clave_con_duplicados_ELTTEXTE000178"></a>
Una clave con duplicados es una clave para la que puede haber varios valores idénticos en el archivo de datos.

Por ejemplo, en el archivo de datos "CUSTOMER", se puede definir una clave con duplicados en el campo "NAME". El contenido de esta clave puede ser el mismo en diferentes registros (por ejemplo, para los homónimos). Esta clave permite realizar una búsqueda por nombre de cliente.





<a name="NOTE3b"></a>
<a name="NOTE3b_1"></a>


## Clave primaria
<a name="clave_primaria_ELTTEXTE000202"></a>
Una clave primaria es una clave única que no puede contener valores NULL. Solo se puede definir una clave primaria para un archivo de datos. 

Por ejemplo, en el archivo de datos "PRODUCT", se puede definir una clave primaria en el campo "REFERENCE". Cada producto está identificado por una referencia única que no contiene valores NULL.



<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Clave simple o clave compuesta
<a name="clave_simple_clave_compuesta_ELTTEXTE000226"></a>
Una clave simple es una clave definida en un campo.

Una clave compuesta permite definir una clave en varios campos del mismo archivo de datos. Esta clave permitirá acceder a los registros del archivo de datos en función del valor combinado de dos campos.

Por ejemplo, para simplificar la búsqueda de pedidos, puede definir una clave compuesta que contenga los campos "Customer Number" y "Order Date".

Observación: Una clave compuesta puede ser: 

- Una clave única,

- Una clave con duplicados,

- Una clave primaria. En este caso, ninguno de los componentes de la clave debe aceptar valores NULL.

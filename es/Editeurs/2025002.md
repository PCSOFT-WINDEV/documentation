


## Crear un ejecutable
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000204"></a>
Una vez que haya terminado de desarrollar su aplicación WINDEV (ventanas, reportes, consultas, etc.), puede crear el archivo ejecutable correspondiente. WINDEV permite crear el archivo ejecutable para aplicaciones Windows. 

El archivo ".EXE" se puede ejecutar directamente: no es necesario abrir la aplicación en WINDEV.

Una vez creado el archivo EXE, la aplicación se puede implementar en los equipos de los usuarios finales mediante el programa de instalación. Para obtener más información, consulte [Programa de instalación de una aplicación WINDEV](../Editeurs/2028001.md)

Observación: Los ejecutables creados con WINDEV pueden ser verificados por el Kit de certificación de aplicaciones de Windows (Windows App Certification Kit). El kit de certificación de aplicaciones de Windows prueba y valida si la aplicación está lista para obtener la certificación y ser publicada en Microsoft Store.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000228"></a>


### Para crear un ejecutable WINDEV
<a name="para_crear_ejecutable_windev_ELTPARAGRAPHE000028"></a>

1. Inicie el asistente de generación de ejecutables mediante uno de los siguientes métodos: 

	- En la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar". 

	- Despliegue ![Generación de ejecutable](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_exe.gif) en los botones de acceso rápido y seleccione "Generar ejecutable Windows de 32 bits (.exe)". 




2. Especifique las [opciones generales del ejecutable](../Editeurs/2025006.md) (nombre del ejecutable, ícono, etc.).

3. Seleccione:

	- [cómo utilizar la biblioteca](../Editeurs/2025004.md) (biblioteca incluida en el ejecutable, biblioteca externa, etc.).

	- el [modo de funcionamiento del ejecutable](../Editeurs/2025004.md) (programa de instancias múltiples o instancia única).




4. Introduzca el mensaje de error que se mostrará si se produce un error de programación. Este mensaje de error se puede escribir en todos los idiomas del proyecto actual.

5. Especifique si la aplicación debe enviar datos para la telemetría. 
	Atención: para implementar la telemetría en una aplicación se debe implementar un servidor de telemetría. Para obtener más información, consulte las siguientes páginas: 

	- [Telemetría: Presentación](../WDLang1/1410086580.md).

	- [Telemetría: Instalación del servidor](../WDLang1/1410086581.md). 




6. Especifique si el archivo ejecutable utilizará parches para las actualizaciones. En este caso, solo tiene que copiar el archivo Patch en el directorio del ejecutable. Para obtener más información, consulte [Parche de aplicación](../Editeurs/2030031.md).

7. Especifique los [idiomas que se incluirán](../Editeurs/2025010.md) en el archivo ejecutable.

8. Especifique [los archivos a incluir en la biblioteca principal](../Editeurs/2025012.md) de la aplicación.

9. Si su proyecto utiliza componentes externos, especifique si deben incluirse en el archivo ejecutable.

10. Si su aplicación es compatible con el [Groupware Usuario](../Editeurs/2018001.md), especifique el [directorio en el que se crearán los archivos de datos del Groupware Usuario](../Editeurs/2025007.md).

11. Especifique el [directorio en el que se crearán los archivos de datos utilizados por la aplicación](../Editeurs/2025007.md).

12. Especifique la información del ejecutable disponible en la pestaña "Versión" o "Detalles" de la descripción del archivo en el explorador:

	- [Información del ejecutable](../Editeurs/2025008.md)(descripción de la aplicación, copyright, etc.).

	- **Firma del ejecutable**: Si se debe firmar el ejecutable, especifique el certificado a utilizar. Este certificado debe estar en Microsoft Store (más concretamente en el Store del usuario y no en el Store del equipo). El estándar de firma utilizado es PKCS7.
			Esta firma permite: 

		- asegurarse de que el ejecutable no ha sido modificado desde que se generó.

		- mostrar el nombre del editor en la ventana de descarga de Windows. 




13. Especifique el [número de versión del ejecutable](../Editeurs/2025008.md).

14. Seleccione el modo de utilización del [Framework WINDEV](../Editeurs/2025003.md) (archivos .DLL). Especifique si las DLL del framework WINDEV deben incluirse o no en el ejecutable. Seleccione las DLL que se van a utilizar.

15. Si su aplicación se utiliza en Windows Vista (o posterior), especifique si se debe incluir un [manifiesto](../Editeurs/2025013.md) en el archivo ejecutable. En este caso, especifique el nivel de privilegios que debe gestionarse.

16. Seleccione las [bibliotecas externas (archivos ".WDL") utilizadas por el archivo ejecutable](../Editeurs/2025003.md).

17. Valide la creación del archivo ejecutable. Una vez creado el archivo ejecutable, WINDEV le pide que lo ejecute.




Observaciones: 

- Si el ejecutable y el procedimiento de instalación se crean al mismo tiempo, el asistente también le permite [documentar las nuevas funcionalidades](../Editeurs/2028084.md).

- El directorio de generación se puede abrir en cualquier momento: en la pestaña "Inicio", en el grupo "General", despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Rep_Projet.gif) y seleccione "Abrir directorio de generación".

## Caso de error
<a name="caso_error_ELTTEXTE000252"></a>
Si se produce un error de "Acceso denegado" durante la generación del archivo ejecutable, significa que un proceso lo está bloqueando.

Se debe identificar el programa que genera el bloqueo. Los programas a examinar son los que están configurados para realizar una acción en todos los archivos ubicados en el disco (copia de seguridad, sincronización de datos, antivirus, etc.). Una herramienta como "Process Explorer" puede ayudarle a encontrar el proceso correspondiente.

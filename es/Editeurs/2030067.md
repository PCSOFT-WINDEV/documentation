## Presentación
<a name="presentacion_ELTTEXTE000132"></a>
En un sitio dinámico de WEBDEV, los datos de la página se actualizan durante una acción del usuario. 

En algunos casos, puede ser necesario para actualización alguna información que se encuentra en esta página, sin la intervención del usuario.: por ejemplo, un contador de correos electrónicos entrantes o un nivel de existencias. Hay varios métodos disponibles: 

- Utiliza un temporizador del navegador para actualización la página o llama a un servidor Procedure en [Ajax](../WDLang2/3055116.md).

- Utilice la opción para automatic actualización de la página. Este mecanismo se basa en la tecnología de "Eventos enviados por el servidor" (SSE). Esta tecnología permite que la sesión (lado del servidor) inicie la transmisión de datos hacia el navegador una vez que la conexión inicial se haya implementado.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000156"></a>


### Definiendo un automatic datos actualización
<a name="definiendo_automatic_datos_actualizacion_ELTPARAGRAPHE000036"></a>

Para definir un automatic datos actualización en una página: 

1. Abra la página deseada en el editor de páginas. 

2. Abra la ventana de descripción de la página: en la pestaña "Página", en el grupo "Descripción", haga clic en "Descripción". 

3. En la pestaña "General", marque "" y especifique el periodo de actualización (en segundos).

4. Validar. 

5. Mostrar los eventos associated con la página (F2 en la página por ejemplo). 

6. Dos nuevos eventos están disponibles: 

	- **RRefresca los datos de la página (código del servidor)**: Este Event se ejecuta cuando la página se actualiza. Se utiliza, por ejemplo, para actualización los controles deseados: llenando una lista, ... Este es un Ajax Event. 

	- **Después de refrescar los datos de la página (código del navegador)**: Este Event se ejecuta después del Event "Refrescando los datos de la página (código de servidor)". Se utiliza para realizar una Process adicional en el navegador.


 Observación: Si el evento "Actualización de los datos de la página (código servidor)" devuelve <u><u><u><u>True</u></u></u></u> (RESULT True), se ejecuta el código navegador (caso predeterminado). De lo contrario, el código del navegador no se ejecutará.




**Atención**: Para que el código actualización se ejecute, el periodo de actualización especificado debe ser inferior al tiempo de espera de la vida del sitio.
Recordatorio: La duración de la sesión corresponde al parámetro "Desconectar los usuarios inactivos desde..." especificado en el administrador de la WEBDEV servidor de aplicaciones (pestaña "Configuración").


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000076"></a>

Esta opción está disponible para los sitios WEBDEV dinámicos desarrollados en HTML5 y que funcionan en los siguientes tipos de navegadores: 

- Google Chrome: Todas las versiones

- Mozilla Firefox: Desde Firefox 6.0

- Opera: De Opera 11

- Safari: Desde Safari 5.0

- Internet Explorer: No soportado (incluyendo IE 11)

- Borde: No soportado

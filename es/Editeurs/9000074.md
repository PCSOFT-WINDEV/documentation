## Presentación
<a name="presentacion_ELTTEXTE000329"></a>
El estilo de código permite agregar prefijos automáticamente al nombre de todas las variables y elementos del proyecto (ventanas, controles de reportes, clases, etc.). Este prefijo automatic permite identificar y/o encontrar fácilmente un elemento de un tipo determinado.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000353"></a>


### Habilitando el prefijo automatic
<a name="habilitando_prefijo_automatic_ELTPARAGRAPHE000029"></a>

Para activar el prefijo automatic:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. Visualizar la pestaña "Opciones".

3. Seleccione "Activar prefijos automáticos de variables y elementos del proyecto".

4. Elija el estilo de código a utilizar. El estilo "Estándar" (proporcionado con WINDEV, WEBDEV o WINDEV Mobile) es utilizado por Default.

5. Marque "Agregar prefijos automáticamente" si desea que los prefijos se añadan automáticamente. En este caso, los prefijos se utilizarán automáticamente en los diferentes editores si se ha definido un estilo de código. Por lo tanto, al escribir el código, el prefijo que corresponde al tipo de Variable se añadirá automáticamente antes del nombre de la Variable. 

6. Marque "Generar un error de compilación de UI si un control no tiene el prefijo correcto" para lanzar un error UI si los controles no utilizan el prefijo definido en el estilo de código. 

7. Validar. Al crear nuevos elementos o nuevas variables, el prefijo correspondiente se añadirá automáticamente.




**Observaciones**:

- Otra solución para habilitar el prefijo automatic: en la pestaña "Proyecto", en el grupo "Otras acciones", amplíe "Estilo de código" y seleccione "Usar estilo de código". 

- A partir de la versión 27, al crear un nuevo proyecto, Default sólo habilita los prefijos automatic para los elementos del proyecto. Los prefijos no se añaden automáticamente (especialmente en el código WLanguage). 
	La opción "Agregar prefijos automáticamente a las nuevas variables" permite habilitar también los prefijos para las variables, desde la creación del proyecto. 

- Los nombres de las variables que contienen menos de 3 caracteres no están prefijados.



<a name="NOTE2_2"></a>


### Desactivando el prefijo automatic
<a name="desactivando_prefijo_automatic_ELTPARAGRAPHE000097"></a>

Para desactivar el prefijo automatic, utilice uno de los siguientes métodos:

- en la pestaña "Proyecto", en el grupo "Otras acciones", despliegue "Estilo de código" y deseleccione la opción "Utilizar estilo de código". 

- en la pestaña "Opciones" de la descripción del proyecto, deseleccione la opción "Activar prefijos automáticos de variables y elementos del proyecto".




Recordatorio: Para abrir la descripción del proyecto, en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción". 
<a name="NOTE2_3"></a>


### Crear una carta de programación
<a name="crear_una_carta_programacion_ELTPARAGRAPHE000137"></a>

Para crear un estilo de código:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. Visualizar la pestaña "Opciones".

3. Haga clic en "Crear un estilo de código" y valide la ventana que aparece. 

4. Seleccione la ubicación del nuevo estilo de código e introduzca su nombre (archivo".CDP").

5. Validar (botón"Guardar"). Se crea el nuevo estilo de código. 
	Por Default, este estilo es el estilo "Estándar" (provisto con WINDEV, WEBDEV o WINDEV Mobile).

6. Haga clic en el botón "Editar".

7. Definir los diferentes prefijos y validar.


**Observación**: El estilo de código "Estándar" no puede modificarse.
<a name="NOTE2_4"></a>


### Modificar un estilo de código
<a name="modificar_estilo_codigo_ELTPARAGRAPHE000172"></a>

Para modificar un estilo de código:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. Visualizar la pestaña "Opciones".

3. Seleccione el estilo de código que desea modificar.

4. Haga clic en "Editar estilo de código". 

5. Definir los nuevos prefijos y validar.




**Observaciones**:

- Estas modificaciones se tendrán en cuenta a la hora de crear nuevos elementos o nuevas variables.

- El estilo de código "Estándar" no puede modificarse.

- Otra solución para editar el estilo de código actual: en la pestaña "Proyecto", en el grupo "Otras acciones", expanda "Estilo de código" y seleccione "Editar estilo de código". 



<a name="NOTE2_5"></a>


### Modificar el estilo de código utilizado
<a name="modificar_estilo_codigo_utilizado_ELTPARAGRAPHE000219"></a>

Modificar el estilo de código utilizado:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. Visualizar la pestaña "Opciones".

3. Seleccione el estilo de código a utilizar (en la lista "Estilo de código" o a través del botón "..."). 

4. Valide.


**Observación**: El mismo estilo de código (archivo".CDP") puede utilizarse en varios proyectos WINDEV, WEBDEV y WINDEV Mobile.
<a name="NOTE2_6"></a>


### Gestión de los prefijos
<a name="gestion_los_prefijos_ELTPARAGRAPHE000245"></a>

En el estilo de código, puede: 

- añadir automáticamente prefijos en los diferentes editores cuando se ha definido un estilo de código.

- generar un error de compilación de la UI si una control tiene un prefijo incorrecto. 




Para gestionar los prefijos: 

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. Visualizar la pestaña "Opciones".

3. Para añadir los prefijos automáticamente, seleccione "Agregar prefijos automáticamente".

4. Para generar un error UI si se utiliza un prefijo no válido, seleccione "Generar un error de compilación de UI si un control no tiene el prefijo correcto". 

5. Valide.

## Presentación
<a name="presentacion_ELTTEXTE000169"></a>
La compilación del proyecto es un Process utilizado para:

- Verifique la consistencia de las sintaxis utilizadas en la base de datos de sintaxis autorizadas de WLanguage.

- Comprobar la existencia y la coherencia de los identificadores utilizados.

- Preparar el código optimizado que se utilizará en tiempo de ejecución.




Cuando se realizan cambios en un elemento, éste se recompila.

Para recompilar todo el proyecto, en la pestaña "Proyecto", en el grupo "Proyecto", amplíe "Recompilar y sincronizar" y seleccione "Recompilar proyecto".

Durante la compilación, el directorio "CPL" del proyecto es borrado y recreado.. Este directorio contiene todos los códigos compilados de los elementos del proyecto.

**Observación**: Reparar (o actualizar) el proyecto se utiliza para volver a guardar todos los elementos del proyecto.. Todos los elementos del proyecto se guardan de nuevo en el formato de la versión actualmente utilizada.. A continuación, se recompila el proyecto.

**Consejo**: Le aconsejamos que vuelva a compilar completamente el proyecto antes de crear el ejecutable o la biblioteca para comprobar la coherencia de todo el proyecto.

## Reconstrucción del proyecto automatic
<a name="reconstruccion_del_proyecto_automatic_ELTTEXTE000193"></a>
Las diferentes opciones para la compilación del proyecto se agrupan en el proyecto description: 

1. En la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción". 

2. Mostrar la pestaña "Compilación".




La opción "Recompilar proyecto antes de generar las aplicaciones, bibliotecas, componentes, etc." permite forzar la recompilación del proyecto antes de cualquier operación importante. Esta opción se recomienda para una generación múltiple de un proyecto.

Esta opción también está disponible en todos los asistentes utilizados para realizar una operación de implementación (creación de ejecutables, creación de bibliotecas, generación múltiple, etc.).

Para obtener más información, consulte [Opciones de compilación del proyecto](../Editeurs/9500218.md).

## Los diferentes modos de compilación del proyecto
<a name="los_diferentes_modos_compilacion_del_proyecto_ELTTEXTE000217"></a>
Existen tres modos de compilación de proyectos: 

- Compilación de un solo núcleo (modo estándar): 
	Este modo de compilación utiliza un solo núcleo. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Compilación en segundo plano. 
	Este modo de compilación le permite seguir trabajando en el proyecto durante la compilación. 

- Compilación multinúcleo. 
	Este modo de compilación utiliza todos los núcleos disponibles por Default. Por lo tanto, es más rápido que la compilación estándar. 
	Puede seleccionar el número de hilos (núcleos) a utilizar. Esto permite ajustar el rendimiento a los casos especiales de manipular y permitir que otros procesos sigan funcionando a una velocidad razonable. 




Para cambiar el modo de compilación del proyecto: 

1. En la pestaña Inicio", en el grupo "Entorno", amplíe "Opciones" y seleccione "Opciones generales de WINDEV/WINDEV Mobile". 

2. En la pestaña "General", seleccione: 

	- el modo de compilación deseado.  

	- el número de hilos utilizados, si es necesario.




3. Validar.

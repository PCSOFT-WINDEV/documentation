


## Importar una ventana que no sea WINDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000111"></a>
¿Está interesado en una ventana que se encuentra en una aplicación que no es WINDEV?

¿Quieres recuperar su interfaz?

Nada es más fácil. WINDEV le permite importar una ventana abierta en el entorno actual.

La ventana se importa al proyecto actual. Cada elemento de la ventana se convierte automáticamente en un elemento WINDEV (control estática , control Campo de entrada, botón, ...).

**Observación**: Todos los elementos de la ventana son importados pero puede que tengas que modificar la ventana para que get la interfaz solicitada.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000135"></a>
Para importar una ventana a un proyecto WINDEV:

1. Abra el proyecto WINDEV en el que debe importarse la ventana.

2. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- La ventana de creación de nuevos elementos se abre: haga clic en "Ventana", y luego en "Ventana".

	- El asistente de creación de ventanas se abre.




3. Mostrar la pestaña "Importar. Las ventanas que se muestran actualmente en el ordenador actual se enumeran automáticamente. Estas ventanas se identifican por su título (que se muestra en la barra de título). 
	Observación: La opción "Generar imágenes para controles no reconocidos." hace que la interfaz final se parezca más a la interfaz de la ventana que se va a importar.

4. Validar. Se muestra la vista previa de la ventana para importar.

5. Modifique (si es necesario) el tipo de control creado al importar la ventana (botón "Detalles").

6. Validar. La ventana se incluye automáticamente en el proyecto WINDEV actual.

## Limitaciones
<a name="limitaciones_ELTTEXTE000159"></a>
No se importan:

- El código de los elementos.

- Los controles distintos de los siguientes:

	- controles estándar de Windows, 

	- Controles Delphi, 

	- Controles de Visual Basic.




- Imágenes e imágenes Background (excepto si está marcada la opción "Generar imágenes para controles no reconocidos.").

- La color de los elementos.

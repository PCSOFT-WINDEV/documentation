


## Generación del análisis
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000153"></a>
El análisis se genera antes del ciclo de programación y después de describir los archivos de datos. Esta generación permite:

- validar los cambios realizados en el análisis (MLD).

- crear los módulos necesarios para la programación.

- actualizar automáticamente los archivos de datos, si es necesario.




Cada generación de análisis está numerada. Este número corresponde al número de generación del análisis. Este número se incrementa cuando el análisis se genera sólo si el description de un archivo de datos (o el description de uno de sus artículos) fue modificado. Este número es set a 0 si el análisis no se ha generado todavía.

Mientras no se haya generado el análisis (el LDM), el description del análisis (es decir, los archivos de datos) no puede ser utilizado en el proyecto.

**Observación**: En una aplicación WINDEV, WEBDEV o WINDEV Mobile, los términos "MLD" y "Analysis" se utilizan indistintamente para definir la estructura de la base de datos asociada a un proyecto.

<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000177"></a>
La generación incluye tres etapas:

- verificación de las modificaciones y generación de los archivos de datos del análisis description.

- Modificación de automatic de los archivos de datos accesibles (archivos que se encuentran en el ["Archivo ".REP](../WDLang4/3044186.md) o en el directorio "EXE" del proyecto). Para más detalles sobre esta operación, véase [Modificación automática de archivos de datos](../WDLang4/3044195.md).

- sincronizar el proyecto con las modificaciones realizadas en el análisis.




**Para generar el análisis**, en la pestaña "Análisis", en el grupo "Análisis", amplíe "Generación" y seleccione "Generación".

**Observación**: También tiene la capacidad de realizar una **generación avanzada del análisis**: en la pestaña "Análisis", en el grupo "Análisis", despliegue "Generación" y seleccione "Generación avanzada". Esta generación avanzada se utiliza para definir parámetros específicos:

- generando un esqueleto para un programa en lenguaje externo.

- mostrando las últimas modificaciones realizadas en el análisis.

- teniendo en cuenta las modificaciones avanzadas realizadas en el análisis del proyecto (en la información compartida del proyecto y/o en los archivos de datos del proyecto, por ejemplo).

## Número de generación de un análisis o número de generación de un fichero de análisis
<a name="numero_generacion_analisis_numero_generacion_fichero_analisis_ELTTEXTE000201"></a>


### Número de generación de un análisis
<a name="numero_generacion_analisis_ELTPARAGRAPHE000077"></a>

El número de generación del análisis se incrementa automáticamente cada vez que se genera el análisis.. Para obtener más información, consulte [Gestión de las versiones de generación del análisis](../Editeurs/2011002.md).

Este número permite conocer el número de generaciones realizadas desde la creación de la aplicación.
<a name="NOTE3_2"></a>


### El número de generación de un archivo de datos encontrado en el análisis
<a name="numero_generacion_archivo_datos_encontrado_analisis_ELTPARAGRAPHE000089"></a>

Un número de generación es también associated con cada archivo de datos. Este número se incrementa cuando el análisis se genera sólo si el description del archivo de datos (o el description de uno de sus artículos) fue modificado. Este número es set a 0 si el análisis no se ha generado todavía.

Por lo tanto, el número de generación de un archivo de datos es menor o igual al número de generación del análisis.

El número de generación de un archivo de datos puede ser:

- devuelto por [GenerationNumber](../Proprietes/2512068.md).

- visto desde WDMap ("Mostrar... Información general sobre el archivo").




Cuando se instala la actualización de una aplicación en los ordenadores de los usuarios finales, el número de generación de los archivos de datos indica si se necesita o no la [automatic modificación de los archivos de datos](../WDLang4/3044195.md).

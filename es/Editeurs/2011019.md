


## Modelo de Datos Conceptuales (MDL)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000100"></a>
En la metodología MERISE, el modelo conceptual de datos define los objetos y las acciones que deben tenerse en cuenta en la base de datos.. Este modelo sólo representa las estructuras de información y los intercambios de información para el sistema utilizado, independientemente de las limitaciones físicas relacionado a la naturaleza de la base de datos utilizada.

**Por ejemplo**un sistema de facturación simplificado puede describirse de la siguiente manera:

- Un cliente corresponde a un nombre de cliente, a una empresa cliente y a un cliente Address.

- Un pedido corresponde a un número de pedido, a una fecha de pedido, a un importe de pedido y a una lista de productos pedidos.

- Un cliente puede "hacer" un pedido en una fecha determinada.

## Creación de un MDL
<a name="creacion_mdl_ELTTEXTE000124"></a>
Para crear un MDL:

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido: 

	- La ventana de creación de nuevos elementos se abre: haga clic en "Datos" y luego en "Análisis (CDM)".

	- Comienza la asistente para crear un MDL.




2. Especifique:

	- los ***nombre y directorio del MDL***. El MDL corresponde a un fichero ".MCD. Este archivo se creará en el directorio del análisis del proyecto de forma predeterminada (directorio &lt;Nombre del proyecto&gt;.ANA. Este directorio debe ser accesible en modo lectura/escritura.

	- los ***Leyenda MDL*** que describe brevemente el tema del MDL.

	- si el MDL debe ser directamente ***associated con un análisis***. Este análisis puede ser un análisis existente o un nuevo análisis que se creará.




3. El MDL creado se convierte en el actual MDL. Ahora puede crear la primera [entidad](../Editeurs/2011005.md).

4. Cree todos los elementos ([instancias](../Editeurs/2011005.md), [asociaciones](../Editeurs/2011004.md), [propiedades](../Editeurs/2011018.md) y [relaciones](../Editeurs/2011013.md)) de su MDL.

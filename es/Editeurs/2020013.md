
## Procesos multiproyecto: utilizando lotes
			






	<B><B>Advertencia</B></B>

	A partir de la versión <B><B>16</B></B>, la creación de procesos por lotes se mantiene para la compatibilidad con versiones anteriores. Le recomendamos que utilice el [Fábrica de software](../Editeurs/1000019563.md) para automatizar el Process para crear aplicaciones. 





<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000177"></a>
WINDEV, WEBDEV y WINDEV Mobile permiten ejecutar una set de procesos (compilación, generación, ...) que se aplicarán a un proyecto o a un grupo de proyectos. Estos procesos se agrupan en un "lote".

Los lotes pueden ser ejecutados inmediatamente o por las tareas programadas de Windows (compilación de un grupo de proyectos cada noche, por ejemplo).

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000201"></a>


### Creación de un archivo por lotes
<a name="creacion_archivo_por_lotes_ELTPARAGRAPHE000029"></a>

Para crear un archivo batch:

1. Seleccione "Taller .. Lote multiproyecto... Crear un lote". Comienza la asistente para la creación de lotes.

2. Especificar el nombre del lote, su ubicación y su description.
	**Observación**: La extensión del archivo correspondiente a un lote es ".WDBatch".

3. Seleccione los diferentes proyectos de manipular en el lote a través de los botones "Añadir un proyecto", "Añadir varios proyectos" y "Eliminar de la lista"..
	**Observaciones**:

	- El orden de los proyectos es muy importante. Por ejemplo, un proyecto que genera un componente utilizado por un segundo proyecto debe procesarse primero.

	- El mismo proyecto se puede visualizar varias veces en la lista de proyectos. De hecho, las diferentes configuraciones de proyecto pueden ser procesadas. Todas las configuraciones del proyecto son tenidas en cuenta por Default. La elección de la configuración a tener en cuenta se realiza en el lote description.




4. Especificar los procesos que debe ejecutar el lote. Una lista de procesos es propuesta por Default. Tiene la posibilidad de añadir o eliminar procesos.
	Ahora puede configurar las operaciones que se realizarán automáticamente durante la ejecución del lote mediante el botón "Editar la hoja de ruta".. Estas operaciones afectan a:

	- la actualización de los componentes de automatic.

	- la sincronización del proyecto con el análisis.

	- sincronizar las plantillas.

	- la actualización del proyecto.

	- la gestión de los errores de compilación (se ignoran los errores de compilación de UI). Si se selecciona "Iniciar la generación independientemente de los errores", se escribirá un "aviso" en el status reporte siempre que se produzca un error de compilación.

	- la gestión de las pruebas automatizadas. Si se selecciona la opción "Continuar la ejecución del lote independientemente de los errores", se escribirá una "advertencia" en el estado reporte cada vez que se produzca un error en las pruebas de automatic.

	- las opciones específicas de los proyectos que se encuentran en la SCM.




5. Especificar las acciones del lote Post a ejecutar. Tienes la capacidad de ejecutar un ejecutable después del lote, de cerrar el editor, de apagar el equipo, ... También se puede enviar un status reporte a la mensajería de WDBal.

6. La asistente está terminada. Es posible:

	- Editar descripción del batch. La ventana description del lote se utiliza para configurar las características del lote con mayor precisión.

	- Ejecutar batch inmediatamente.

	- Programar la ejecución del lote mediante la creación de una tarea programada en Windows.. Comienza la asistente para crear una tarea programada.






<a name="NOTE3_2"></a>


### Consejo
<a name="consejo_ELTPARAGRAPHE000064"></a>

Para Process los proyectos multiproducto (proyecto con una configuración en WINDEV y una configuración en WEBDEV por ejemplo), le aconsejamos crear un lote para cada producto.

Por ejemplo, el lote WINDEV se utilizará para Process la configuración WINDEV del proyecto y el lote WEBDEV se utilizará para Process la configuración WEBDEV del proyecto.
<a name="NOTE3_3"></a>


### Status reporte de un lote
<a name="status_reporte_lote_ELTPARAGRAPHE000073"></a>

Al ejecutar un lote, se crea automáticamente un status reporte (fichero.wbl).. Este status reporte puede abrirse en el editor:

- a través de "Taller... Multi-proyecto de lote .. Abrir un lote reporte ".

- a través de "Ficha .... Abierto".

## Describir un lote
<a name="describir_lote_ELTTEXTE000237"></a>
La description de un lote existente se puede visualizar en cualquier momento. Para ello, abra el lote:

- a través de "Workshop .. Lote multiproyecto... Abrir un lote".

- A través de "Ficha .... Abierto".




La ventana de description de un lote se utiliza para:

- Crear un nuevo lote (botón"Nuevo").

- Seleccionar un batch existente.

- Inicie la ejecución del lote (botón"Iniciar la ejecución") o programe la ejecución del lote en Windows (botón"Programar la ejecución").

- Definir para cada proyecto procesado por el lote:

	- la configuración de Process. Todas las configuraciones del proyecto son procesadas por Default.

	- la lista de procesos a realizar. La lista principal del lote se aplica a todos los proyectos de Default. Tiene la posibilidad de crear una nueva lista de procesos. Cada proyecto puede ser associated con una lista diferente de procesos. La lista de procesos se visualiza en la parte inferior de la pantalla.. Se puede definir una hoja de ruta específica para cada lista de procesos.




- Definir las acciones del lote Post a realizar (botón "Acciones del lote Post").

## Ejecutando un lote en el comando Line
<a name="ejecutando_lote_comando_line_ELTTEXTE000261"></a>
La siguiente sintaxis permite ejecutar un lote en el comando Line:

En WINDEV:


```txt
WinDev17.EXE /NAME="<Initials>" /PASS="<Password>" /BATCH="<Full name of batch>"
```


En WINDEV Mobile:


```txt
WD17Mobile.EXE /NAME="<Initials>" /PASS="<Password>" /BATCH="<Full name of batch>"
```


En WEBDEV:


```txt
WEBDEV17.EXE /NAME="<Initials>" /PASS="<Password>" /BATCH="<Full name of batch>"
```


donde

&lt;Iniciales&gt; corresponde a las iniciales utilizadas en la ventana de inicio de sesión de WINDEV, WEBDEV y WINDEV Mobile.

&lt;Contraseña&gt; corresponde a la contraseña utilizada en la ventana de inicio de sesión de WINDEV, WEBDEV y WINDEV Mobile.

&lt;El nombre completo del lote corresponde a la ruta completa del archivo batch a ejecutar..

Por ejemplo:


```txt
WinDev17.EXE /NAME="FP" /BATCH="C:\MyProjects\Test.WDBatch"
```

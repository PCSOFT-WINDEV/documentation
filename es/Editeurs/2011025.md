


## Análisis: Importar archivos al análisis (base de datos externa, archivos HFSQL, etc.)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000200"></a>
WINDEV y WEBDEV permiten manipular fácilmente bases de datos externas (Oracle, SQL Server, Access, etc.) mediante diferentes funciones WLanguage.

Para manipular una base de datos externa, es necesario:

1. Importar los archivos de datos (o tablas) de la base de datos externa al análisis. Durante esta importación, los datos pueden:

	- almacenarse en su formato actual. No se requiere ninguna conversión.
			**Observación**: Si se realizan cambios en la base de datos, estos pueden tenerse en cuenta. Solo debe [sincronizar el análisis con la base de datos externa](../Editeurs/2011024.md).

	- convertirse a HFSQL.




2. Generar el análisis.

3. Manipular los archivos de datos importados con funciones WLanguage. Para obtener más información, consulte [Acceder a las bases de datos](../WDLang4/3044202.md).




**Observaciones**: 

- También es posible convertir solo los datos de un archivo de datos específico con [WDConver](../WDConver/3512006.md). Para realizar esta operación, se debe haber importado la descripción del archivo de datos al análisis.

- El editor de análisis permite realizar las siguientes operaciones: 

	- Importar un archivo de datos desde una base de datos externa a un análisis HFSQL Classic.

	- Importar un archivo de datos HFSQL Classic en un nuevo análisis HFSQL Classic.

### Importar a un archivo de datos a un análisis
<a name="importar_archivo_datos_analisis_ELTPARAGRAPHE000106"></a>

Para importar un archivo de datos a un análisis: 

1. En el editor de análisis, abra el análisis en el que se debe realizar la importación.

2. En la pestaña "Análisis", en el grupo "Creación", despliegue "Importar" y seleccione "Importar descripciones de archivos/tablas...". El asistente de importación de archivos de datos se abre.

3. Seleccione la fuente de datos deseada.

4. **Si desea importar una base de datos externa**:

	- Para las fuentes de datos que utilizan un Conector Nativo, se crea automáticamente una conexión a la base de datos externa y se asocia a los archivos de datos importados al análisis. Esta conexión se utilizará para manipular los archivos de datos mediante programación.

	- Especifique los parámetros de la conexión, si es necesario: servidor de datos, usuario y contraseña, base de datos, nombre de conexión, etc.

	- Indique el modo de acceso a los datos. Es posible: 

		- acceder directamente a los datos en su formato actual. Solo la descripción de las tablas se importa al análisis. No se requiere ninguna conversión.

		- convertir los datos a HFSQL Classic.




- Seleccione las tablas (archivos de datos) que desea importar.

5. **Si desea importar archivos de datos HFSQL Classic**:

	- Seleccione el archivo de datos HFSQL Classic deseado (extensión .FIC).

	- Especifique la contraseña del archivo de datos, si es necesario.




6. Valide la importación. El análisis actual se genera automáticamente.

7. Si los datos se deben convertir a HFSQL Classic:

	- Indique el directorio en el que se crearán los archivos de datos HFSQL Classic.

	- Guarde (si es necesario) la descripción de la conversión (script). Esta operación crea un archivo ".WDV" en el directorio especificado. El script puede volver a ser utilizado por [WDConver](../WDConver/3512006.md).

	- Valide la conversión de los datos.







Observaciones: 

- Para importar un archivo HFSQL Classic en un análisis, arrastre el archivo desde el Explorador de archivos de Windows y suéltelo en el editor de análisis. El asistente de importación se abre automáticamente.

- A partir de la versión 19, no se pueden importar archivos de datos Hyper File 5.5. Los archivos de datos en este formato solo pueden importarse con WINDEV o WEBDEV 18 (u otras versiones anteriores).

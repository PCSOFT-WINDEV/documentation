
## Sitio web adaptado a dispositivos móviles: métodos disponibles
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000178"></a>
WEBDEV incluye varios métodos de creación de Page que permiten crear sitios web adaptados a dispositivos móviles: 

- **Servicio dinámico** , que consiste en mantener un set de páginas para escritorio y un set de páginas para móvil. 

- **Legacy Responsive Web Design** , que permite dividir las páginas en rangos de resolución o "segmentos" para adaptarse a los dispositivos móviles. 

- **Diseño web receptivo a través de zonificación y diseños** , que le permite definir diseños específicos para una pantalla determinada. A partir de WEBDEV 28, este modo es utilizado por Default. Ahora se recomienda este modo. Para obtener más información, consulte [Nuevo comportamiento en la versión 28](#NOTE2_2).   




Esta ayuda Page presenta los diferentes métodos, sus ventajas e inconvenientes.

### Nuevo modo de edición para sitios web adaptados a dispositivos móviles en la versión 28
<a name="nuevo_modo_edicion_para_sitios_web_adaptados_dispositivos_moviles_version_28_ELTPARAGRAPHE000093"></a>

En la versión 28, el "Diseño Web Responsivo: diseño por rango de resolución " el modo se convierte en "Segmentos (modo receptivo heredado)".
"Responsive (zonificación y diseños)" es ahora el modo recomendado para cualquier proyecto nuevo.

Los diseños se han convertido en el método más popular para crear sitios adaptados a móviles. Así, la mayoría de las mejoras introducidas en el editor Page se centran en el modo de edición por zonas (zonificación), que amplía las posibilidades de los diseños:

- le permiten cambiar y ocultar zonas,

- Las plantillas control admiten diseños,

- Las áreas de texto enriquecido pueden tener contenido adaptado al diseño,

- el nuevo Flexbox control permite definir cómo reaccionan los control s cuando se cambia el tamaño de un Page en el navegador,

- un nuevo diseño está disponible para las páginas que se integren en una aplicación WINDEV o WINDEV Mobile.




Estas nuevas funciones de la versión 28 se suman a las ventajas ya existentes de los diseños:

- edición sencilla,

- mismas posibilidades que en las aplicaciones WINDEV y WINDEV Mobile,

- posibilidad de elegir diseños mediante programación,

- capacidad de set diferentes anclajes en cada disposición, 

- diseño de impresión específico...




**Debido a sus numerosas ventajas, "Responsive (zonificación y diseños)" es el modo recomendado a partir de WEBDEV 28 en adelante**. Puede convertir un Page en "Diseño web receptivo: diseño por rango de resolución "modo a" Responsivo (zonificación y diseños)": en la pestaña "Página", en el grupo "Layouts", despliegue "Layouts" y seleccione "Convertir cortes en diseños".

Todas las mejoras futuras del editor WEBDEV Page se centrarán en este modo.

El modo de edición basado en rangos de resolución (slices) sigue estando disponible para los sitios existentes y para los nuevos sitios en los que no se pueden utilizar diseños.. Sin embargo, este modo no verá grandes mejoras en futuras versiones de WEBDEV.

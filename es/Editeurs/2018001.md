## Presentación
<a name="presentacion_ELTTEXTE000164"></a>
Una aplicación Windows, un sitio de Internet o Intranet requieren que se defina el papel de los diferentes colaboradores. Se pueden establecer varios niveles de acceso según el usuario. Todos los usuarios tienen diferentes responsabilidades y requisitos. Por lo tanto, suele ser necesario limitar sus campos de acción.

Veamos un ejemplo simple: una aplicación de gestión de ventas que propone las siguientes funcionalidades:

- Ver la lista de precios

- Modificar la lista de precios

- Introducir pedidos

- Agregar clientes.




El acceso puede variar según el usuario. Por ejemplo:

- el personal de secretaría puede ver la lista de precios y crear pedidos

- los vendedores pueden ver la lista de precios, hacer pedidos y crear nuevos clientes.

- los directores de ventas tienen acceso a todas las opciones.




Para gestionar estos niveles de acceso en las aplicaciones WINDEV o sitios WEBDEV, se debe incluir el Groupware Usuario en la aplicación o sitio. Con unos pocos clics, una aplicación estándar puede transformarse en una aplicación que gestiona varios niveles de acceso. Esta característica se puede implementar fácilmente, tanto en una aplicación WINDEV como en un sitio WEBDEV.

Al iniciar la aplicación o el sitio, el administrador podrá crear usuarios (identificados por su nombre de usuario y contraseña) y darles acceso a algunas características del sitio o de la aplicación.

**Observación**: El User Groupware está disponible para las aplicaciones WINDEV y para los sitios WEBDEV dinámicos que se ejecutan en Windows o Linux. El programa de grupo de usuarios no está disponible en sitios estáticos, semidinámicos, PHP o Ajax.

## Notas
<a name="notas_ELTTEXTE000218"></a>


### Modo de funcionamiento del Groupware Usuario
<a name="modo_funcionamiento_del_groupware_usuario_ELTPARAGRAPHE000081"></a>

Una aplicación que utiliza el Groupware Usuario tiene dos niveles:

- nivel de usuario

- nivel de supervisor




**El nivel de usuario**
El usuario se conecta a la aplicación a través de una ventana/página de inicio de sesión y puede acceder a las funciones que se le han permitido.

**El nivel de supervisor**
El supervisor se conecta a la aplicación a través de una ventana/página de inicio de sesión y accede a un menú avanzado para configurar los usuarios y sus derechos, o para lanzar la aplicación. La ventana/página de configuración permite al supervisor:

- Inicie la aplicación.

- configurar el groupware.




La configuración del groupware consiste en crear usuarios y grupos, asociar usuarios a grupos, y administrar los permisos otorgados a cada usuario (o grupo) en cada ventana/página de la aplicación. Los permisos se pueden definir para las opciones de menú, grupos de controles y controles. El supervisor puede inhabilitar, desactivar o hacer invisibles estos elementos. Estas configuraciones tienen prioridad sobre cualquier configuración definida en el programa.

Para más información sobre la configuración del groupware definida por el supervisor, consulte [Configurar el programa de grupo de usuarios](../Editeurs/2018006.md).

**Observación**: Los detalles sobre los usuarios y sus derechos se almacenan en archivos en formato HyperFileSQL.. Los archivos relativos a los usuarios pueden ser comunes a varias aplicaciones. Los archivos que contienen los derechos concedidos a los usuarios sobre las diferentes ventanas/páginas que se encuentran en la aplicación son específicos de la aplicación y no pueden ser compartidos.

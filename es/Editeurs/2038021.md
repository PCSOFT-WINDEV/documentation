


## Administrador del SCM: Gestionar el depósito
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000469"></a>
El administrador de SCM (Administrador de Código Fuente) le permite manipular directamente los códigos fuente de los diferentes proyectos incluidos en el programa Administrador de Código Fuente. Es posible:

- administrar repositorios (crear o conectarse a un repositorio).

- gestionar los archivos y directorios que se encuentran en un proyecto del repositorio (Añadir, borrar, renombrar archivos y directorios).

- administrar los archivos del repositorio (extraer, reintegrar, compartir etc.).




El administrador de SCM también puede ser utilizado para realizar operaciones de mantenimiento en el repositorio., ... Para obtener más información, consulte [Vista general del administrador de SCM](../Editeurs/2038008.md).

## Gestionar el depósito
<a name="gestionar_deposito_ELTTEXTE000493"></a>


### Conexión a un repositorio
<a name="conexion_repositorio_ELTPARAGRAPHE000027"></a>

Para conectarse a un depósito: 

1. En el administrador del SCM, en la pestaña "Inicio", en el grupo "Repositorio", haga clic en "Conexión ".

2. Especificar las características del repositorio (ubicación, usuario, etc.). Este repositorio al que se accede puede ser: 

	- un repositorio local (HFSQL Classic). 

	- un depósito de HFSQL Client/Server SCM. 

	- un repositorio SCMDrive (solución llave en mano de pago, en un servidor compartido). Para crear este repositorio, conéctese a [SCMDrive](https://pcscloud-drive.net/UK/SCMDrive/index.awp). 

	- un repositorio PCSCloud (solución llave en mano de pago, en un servidor dedicado). Para crear este repositorio, se debe crear una cuenta en la nube. Para más detalles, conéctese a [PCSCloud](https://pcscloud.net/UK/cloud_development.awp).




3. Validar. Si el repositorio SCM es accesible, se conecta automáticamente a este repositorio SCM.



<a name="NOTE2_2"></a>


### Crear un repositorio
<a name="crear_repositorio_ELTPARAGRAPHE000065"></a>

Para crear un repositorio SCM: 

1. En el administrador del SCM, en la pestaña "Inicio", en el grupo "Repositorio", haga clic en "Nuevo repositorio".

2. Especificar las características del repositorio SCM: repositorio en modo red compartida o repositorio en modo HFSQL Client/Server. Para obtener más información, consulte [Crear un repositorio.](../Editeurs/2038005.md)

3. Validar. Si el repositorio SCM es accesible, se conecta automáticamente a este repositorio SCM.

## Gestión de archivos y directorios
<a name="gestion_archivos_directorios_ELTTEXTE000523"></a>


### Gestión de archivos
<a name="gestion_archivos_ELTPARAGRAPHE000092"></a>

El administrador de SCM está acostumbrado a:

- **Agregar un archivo**: en la pestaña "Inicio", en el grupo "Archivo", haga clic en "Agregar". Esta opción permite añadir un archivo que se encuentra en el proyecto local en el proyecto reference.

- **Renombrar un archivo**: en la pestaña "Inicio", en el grupo "Archivo", haga clic en "Renombrar". Esta opción le permite renombrar el archivo actualmente seleccionado en el administrador de SCM. Este archivo se encuentra en el proyecto reference (que se encuentra en el servidor SCM).
	Observación: Esta operación debe hacerse con mucho cuidado.. De hecho, el archivo renombrado no estará disponible para los usuarios de SCM hasta que se realice una sincronización completa del proyecto.. Le aconsejamos que cambie el nombre del archivo en el proyecto local y luego actualizar el proyecto reference.

- **Eliminar un archivo**: en la pestaña "Inicio", en el grupo "Archivo", haga clic en "Eliminar". Esta opción le permite borrar el archivo seleccionado del administrador de SCM. Este archivo se encuentra en el proyecto reference (que se encuentra en el servidor SCM).
	Observaciones: 

	- Esta operación debe hacerse con mucho cuidado.. Sólo se pueden eliminar los archivos que no son utilizados por el proyecto. De lo contrario, la eliminación de un archivo de la SCM puede provocar problemas con el proyecto.

	- En el caso de los archivos compartidos, sólo se elimina el archivo especificado: se guardan copias compartidas en otras carpetas.







**Para restaurar un archivo eliminado**: 

1. Abra las propiedades del directorio del proyecto en Administrador de Código Fuente ("Propiedades" en el menú contextual del directorio).

2. Seleccionar la pestaña "Elementos borrados: tiene la posibilidad de restaurar los archivos eliminados.




Observación: También puede eliminar o renombrar un archivo seleccionándolo y eligiendo la opción correspondiente en su menú contextual. 
<a name="NOTE3_2"></a>


### Gestión de directorios
<a name="gestion_directorios_ELTPARAGRAPHE000153"></a>

El administrador de SCM está acostumbrado a:

- **Añadir un directorio existente**: en la pestaña "Inicio", en el grupo "Directorio", haga clic en "Importar un directorio local". Esta opción permite importar un directorio que se encuentra en el proyecto local en el proyecto reference por ejemplo.
	Observación: No utilice esta opción para añadir un proyecto a la SCM. Para añadir un proyecto a la SCM, véase [Configurar un proyecto para la SCM](../Editeurs/2038002.md).

- **Crear un nuevo directorio**: en la pestaña "Inicio", en el grupo "Directorio", haga clic en "Nuevo". Esta opción permite crear un nuevo directorio en el proyecto reference.
	Observación: También puede crear un directorio a través del menú contextual del explorador de SCM ("Crear un nuevo directorio"). 

- **Eliminar un directorio**: en la pestaña "Inicio", en el grupo "Directorio", haga clic en "Eliminar directorio". Esta opción puede usarse para borrar un directorio del repositorio SCM por ejemplo. También tiene la posibilidad de borrar un proyecto de la SCM (para ello, borre el directorio correspondiente).. El directorio eliminado es el directorio seleccionado en el explorador SCM..
	Atención: Esta operación no se puede cancelar.
	Observación: También puede eliminar un directorio a través del menú contextual del explorador de SCM ("Eliminar").

- **Renombrar un directorio**: en la pestaña "Inicio", en el grupo "Directorio", haga clic en "Renombrar". Esta opción le permite renombrar un directorio en el repositorio SCM. El directorio utilizado es el seleccionado en el explorador de SCM.
	Atención: Esta operación no se puede cancelar.
	Observación: También puede renombrar un directorio a través del menú contextual del explorador de SCM ("Renombrar").

- **Define el directorio local associated**: en la pestaña "Directorios y proyectos", en el grupo "Directorio local", haga clic en "Cambiar directorio local ". Esta opción permite especificar el nombre del directorio local associated con el proyecto que se encuentra en el repositorio SCM para el usuario actual.
	Observación: También puede definir el directorio local associated a través del menú contextual del explorador SCM ("Definir el directorio local associated").

- **Mover un directorio**: También puede mover un directorio a través del menú contextual del explorador de SCM ("Mover..."). Sólo hay que especificar la nueva ubicación de la carpeta seleccionada en el SCM. Esta operación tiene en cuenta las ramas. Si el directorio trasladado corresponde a una rama, el proyecto en la nueva ubicación seguirá correspondiendo a la misma rama.

### Gestión de favoritos
<a name="gestion_favoritos_ELTPARAGRAPHE000246"></a>

El administrador de SCM le permite definir proyectos "favoritos. 

To delete a favorite porject: 

1. Seleccione el proyecto deseado en la sección izquierda del administrador de SCM.

2. Seleccione "Añadir a favoritos" en el menú contextual.

3. El proyecto aparece en la lista de proyectos favoritos en la sección izquierda del administrador de SCM.




Observación: También tiene la posibilidad de utilizar el menú del administrador de SCM para definir un proyecto favorito.: en la pestaña "Directorios y proyectos", en el grupo "Favoritos", haga clic en "Agregar a favoritos".

Para eliminar un favorito: 

1. Seleccione el proyecto deseado en el área de favoritos en la sección izquierda del administrador de SCM.

2. Seleccione "Eliminar" en el menú contextual.



<a name="NOTE3_4"></a>


### Restauración de una versión de proyecto
<a name="restauracion_una_version_proyecto_ELTPARAGRAPHE000278"></a>

El administrador de SCM le permite restaurar una versión específica de un proyecto (correspondiente a una versión anterior del proyecto, almacenada en el historial).

Para restaurar una versión de un proyecto: 

1. Seleccione el proyecto deseado en la sección izquierda del administrador de SCM.

2. Seleccione "Historia" en el menú contextual. Aparece la ventana "Historial de un directorio".

3. En la tabla, seleccione la versión deseada del proyecto.

4. Haga clic en el botón "Guardar como".

5. Seleccione el directorio donde se debe guardar la versión especificada del proyecto.

6. Validar. La versión deseada del proyecto se guarda automáticamente en el directorio especificado.

## Gestión de los ficheros del repositorio SCM
<a name="gestion_los_ficheros_del_repositorio_scm_ELTTEXTE000565"></a>
El administrador de SCM le permite realizar todas las operaciones de check-in y check-out., ... También tiene la posibilidad de mostrar el historial de las modificaciones realizadas a cada elemento, ver las diferencias o compartir los elementos entre varios proyectos.. 
<a name="NOTE4_2"></a>


### Operaciones de check-in y check-out, ....
<a name="operaciones_checkin_checkout_ELTPARAGRAPHE000301"></a>

El administrador de SCM puede ser utilizado para realizar las siguientes operaciones:

- get la última versión del archivo seleccionado.

- Compruebe el archivo seleccionado en el directorio local del proyecto.

- Compruebe el archivo seleccionado para la prueba

- Verificar el elemento seleccionado.




Todas estas operaciones se pueden realizar en WINDEV, WEBDEV o WINDEV Mobile.. Le aconsejamos que realice estas operaciones directamente desde el proyecto correspondiente.
<a name="NOTE4_3"></a>


### Historia y diferencias
<a name="historia_diferencias_ELTPARAGRAPHE000315"></a>

Para ver **el historial** de los cambios realizados en un archivo (operaciones de entrada y salida), debe: 

- seleccione el archivo deseado. 

- en la pestaña "Archivos", en el grupo "Archivo", haga clic en "Historial ".


La ventana que se muestra está acostumbrada a:

- get una versión anterior.

- Guardar una versión anterior con un nuevo nombre.

- Compare dos versiones.

- Fusionar.




Para comparar la versión local con la versión encontrada en el repositorio de SCM: 

- seleccione el archivo deseado. 

- en la pestaña "Archivos", en el grupo "Archivo", haga clic en "Comparar".



<a name="NOTE4_4"></a>


### Compartir archivos
<a name="compartir_archivos_ELTPARAGRAPHE000361"></a>

El administrador de SCM le permite compartir archivos entre varios proyectos (compartiendo conjuntos de procedimientos, compartiendo análisis, etc.).

Esta operación sólo se puede realizar en el administrador de SCM.

Es posible:

- **Compartir un archivo**: en la pestaña "Archivos", en el grupo "Compartir", haga clic en "Uso compartido". Esta opción se utiliza para compartir el archivo seleccionado con otro proyecto que se encuentra en SCM.

- **Descompartir un archivo**: en la pestaña "Archivos", en el grupo "Compartir", haga clic en "Eliminar elem. compartido". Esta opción se utiliza para descompartir el archivo actualmente seleccionado en el administrador de SCM.. Este archivo se encuentra en el proyecto reference (que se encuentra en el servidor SCM). El archivo ya no será compartido entre los diferentes proyectos de SCM. El archivo permanece en cada proyecto. 
	Observación: Esta operación debe hacerse con mucho cuidado.. De hecho, si se elimina el recurso compartido, los archivos evolucionarán de forma independiente en cada proyecto.




**How para compartir desde el administrador de SCM**: 

1. Abra el administrador del SCM: en la pestaña "SCM", en el grupo "Repositorio", despliegue "Administrar" y seleccione "Administrador del SCM".

2. En el administrador de SCM:

	- Seleccione los archivos que desea compartir en el directorio del proyecto fuente.

	- Arrastre y suelte estos archivos al directorio del segundo proyecto que comparte estos elementos.

	- Aceptar para compartir los archivos.







**Importante**: el administrador de SCM no le permite compartir las dependencias de los elementos. La compartición de archivos debe implementarse desde la interfaz de WINDEV, WEBDEV o WINDEV Mobile.. Para obtener más información, consulte [Compartir archivos SCM](../Editeurs/2038002.md).

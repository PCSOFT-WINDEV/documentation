
## La estructura de los patrones RAD (conservados para compatibilidad con versiones anteriores)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000112"></a>
**Atención**: en la versión 12, el RAD fue completamente reconstruido. El método explicado en esta página se mantiene para la compatibilidad con versiones anteriores.. Para obtener más información, consulte [Nuevos patrones RAD](../Editeurs/2031020.md). 

Los archivos de patrones RAD (archivos MDL) se utilizan para describir:

- Las diferentes opciones mostradas o Grayed en la asistente para la creación de ventanas.

- Los ficheros necesarios para el funcionamiento de la ventana (por ejemplo, las clases RAD que se deben copiar en el directorio de generación de la ventana).

- Los grupos de controles que se encontrarán en la ventana.

- El código de la ventana y el código de los procedimientos locales o globales.

- Las características de las diferentes áreas donde se mostrarán los controles. Cada área de control es associated con un área de botones y con un área de botones de navegación. Para cada área de botones, el patrón contiene la description de los botones (tamaño, nombre, leyenda, ...) y su código.




Esta información se presenta en el script según una estructura de patrón definida.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Estructura de los archivos
<a name="estructura_los_archivos_ELTTEXTE000136"></a>
La estructura de los archivos MDL se puede dividir en dos secciones:

1. **Descripción del pattern**

	- Características del patrón que se muestra en la asistente

	- Características del patrón requerido en la programación (clases, grupos de controles,...)

	- El código de la ventana y el código de los procedimientos locales o globales




2. **Ventana description**

	- Características de las áreas de control (repetidas para cada área de control)

	- Características de las áreas de botones (repetidas para cada área de botones)

	- Características de un botón (repetidas para cada botón)

	- Código del botón

## Ficheros MDE
<a name="ficheros_mde_ELTTEXTE000160"></a>
Los archivos MDE son archivos MDL utilizados para compartir scripts comunes entre varios archivos MDL. El contenido del archivo MDE es idéntico al del archivo MDL. Sin embargo, el código se encuentra una sola vez en el archivo MDE. Los archivos MDL que utilicen este código sólo tendrán que realizar una simple llamada.
<a name="NOTE3_2"></a>


### Llamada al archivo MDE desde un archivo MDL
<a name="llamada_archivo_mde_desde_archivo_mdl_ELTPARAGRAPHE000055"></a>Se debe utilizar la siguiente sintaxis:


```txt
%extern <Name of .MDE file>
```

Ejemplo:


```txt
%extern Declare1file.mde
```


**Observación**: En el archivo MDE, le aconsejamos que termine el archivo con un Carriage Return (un Line vacío) para evitar problemas al insertar el código en el archivo MDL.

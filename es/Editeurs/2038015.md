## Presentación
<a name="presentacion_ELTTEXTE000199"></a>
WINDEV, WEBDEV y WINDEV Mobile le permiten crear y publicar fácilmente componentes externos.. Esta operación se puede realizar:

- ***en proyectos no compartidos***: el componente externo está disponible en el grafo, por ejemplo, y puede ser utilizado por cualquier proyecto. Para obtener más información, consulte [Creación y distribución de un componente externo](../Editeurs/2014002.md).

- ***on los proyectos encontrados en el repositorio (SCM)***: el componente externo se puede crear a partir de un proyecto que se encuentra en la SCM y se puede poner a disposición de los otros proyectos que también se encuentran en la SCM.




Esta página de ayuda presenta las operaciones a realizar para crear y distribuir un componente externo mediante SCM.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## SCM: Creación y distribución de un componente externo
<a name="scm_creacion_distribucion_componente_externo_ELTTEXTE000223"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000026"></a>

Para distribuir un componente externo mediante SCM, este componente debe haberse creado y generado a partir de un proyecto encontrado en SCM.

Observación: Un proyecto SCM puede también utilizar un componente externo que no ha sido publicado en la SCM.
<a name="NOTE2_2"></a>


### Creación del componente externo
<a name="creacion_del_componente_externo_ELTPARAGRAPHE000035"></a>

El método para crear un componente externo para SCM es el mismo que el método para crear un componente estándar.. Se puede crear un componente externo: 

- desde un proyecto existente, utilizando una configuración de "Componente externo".. Esta configuración contiene los elementos del proyecto que el componente necesita para funcionar correctamente. 

- de un proyecto de "componente externo". Por Default, este proyecto sólo contiene una configuración de "Componente externo".




El proyecto utilizado para crear el componente externo debe estar ubicado en la SCM.

Para obtener más información, consulte [Crear un componente externo](../Editeurs/2014002.md).
<a name="NOTE2_3"></a>


### Generación del componente externo
<a name="generacion_del_componente_externo_ELTPARAGRAPHE000052"></a>

Cuando se crean todos los elementos de componente, se puede generar el componente externo.: 

1. Seleccione (si es necesario) la configuración del proyecto correspondiente al componente (en el panel "Project explorer"). 

2. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_Composant.gif) en los botones de acceso rápido y seleccione "Generar un componente".

3. Comprobar (si es necesario) los elementos comprobados del proyecto.

4. Especifique las opciones solicitadas en la generación de componentes externos asistente. Para obtener más información, consulte [Generación de un componente externo](../Editeurs/2014002.md).
	Centrémonos en las opciones de **para publicar el componente externo** (paso "Publicar componente"): 

	- Por Default, el componente externo se genera en el directorio EXE del proyecto (o en el directorio correspondiente a la configuración del proyecto del componente). Los archivos correspondientes al componente externo pueden ser utilizados por cualquier proyecto.

	- El asistente le pide que "publique" el componente externo en el SCM, es decir, que copy los archivos del componente externo en un directorio específico del SCM. En este caso, el componente externo puede incluirse fácilmente en otros proyectos SCM.





Para publicar el componente externo en el SCM: 

1. Seleccione la opción "Publicar el componente en el siguiente directorio del SCM".

2. Seleccione el directorio de publicación. Este directorio es relativo al repositorio SCM. Por ejemplo, puede agrupar todos los componentes externos que se encuentran en la SCM en un solo directorio. Puede crear un directorio específico mediante el botón "Nuevo directorio" de la ventana de selección de directorios. 
	Observación: Un componente externo publicado en SCM no puede ser incluido en el Centro de Reutilización. 

3. En el siguiente paso, asistente le pide que etiquete las fuentes para encontrar la versión de los elementos utilizados durante la creación del componente. 

4. Validar la creación del componente externo.   

5. Un cuadro de diálogo confirma que la generación del componente se ha completado. Publique el componente en el SCM (botón "Publicar en el SCM").

6. Aparece una ventana que muestra los archivos de componentes que se han añadido al repositorio (\*.wdi, \*.wdk y \*.wdz, si es necesario). Valide la ventana. 

7. El componente ya está publicado y puede ser utilizado por los proyectos de SCM.

## SCM: Utilizando un componente externo que se encuentra en el SCM
<a name="scm_utilizando_componente_externo_que_encuentra_scm_ELTTEXTE000259"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000092"></a>

WINDEV, WEBDEV y WINDEV Mobile le permiten incluir en un proyecto encontrado en el SCM un componente externo procedente de:

- un archivo, 

- la SCM.


El siguiente párrafo explica cómo importar un componente publicado en el SCM.
<a name="NOTE3_2"></a>


### Utilizando un componente externo que se encuentra en el SCM
<a name="utilizando_componente_externo_que_encuentra_scm_ELTPARAGRAPHE000104"></a>

Importar un componente externo en un proyecto de la SCM:

1. Abrir proyecto.

2. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Importar" y seleccione "Un componente externo .. Desde el SCM".

3. La pantalla le permite hacerlo:

	- encontrar todos los componentes externos encontrados en el repositorio SCM (pestaña"Buscar"). Atención: esta operación puede llevar bastante tiempo.

	- Navegue por los diferentes directorios de la SCM para encontrar el componente externo (pestaña "Navegar"). Esta operación es recomendable pero debe conocer el directorio de generación del componente en SCM.




4. Seleccione el componente externo solicitado y valide.

5. El componente externo se incluye en el proyecto y se muestra la ventana de description del componente. Valide.

6. El componente externo puede utilizarse en el proyecto.





<a name="NOTE3_3"></a>


### Caso especial: Sustitución de un componente externo por un componente externo procedente de la SCM
<a name="caso_especial_sustitucion_componente_externo_por_componente_externo_procedente_scm_ELTPARAGRAPHE000139"></a>

Su proyecto en la SCM utiliza un componente externo que fue incluido a través del método estándar (archivo disponible en la red por ejemplo) y usted quiere reemplazarlo con un componente externo ubicado en la SCM: 

1. Abrir proyecto.

2. En el panel "Explorador de proyectos": 

	- Seleccionar la carpeta "Componentes externos. 

	- Abra el menú contextual de la carpeta (clic derecho).

	- Seleccione la opción "Lista de componentes internos importados al proyecto".




3. La ventana que se muestra lista los componentes externos que se encuentran en el proyecto.

4. Para sustituir el componente existente por el componente externo que se encuentra en la SCM, haga clic en el botón "Importar (SCM)" y seleccione el componente externo.


**Observación**: El botón "Re-localizar" no puede utilizarse para sustituir el componente externo incluido en el proyecto por el mismo componente que se encuentra en el SCM.

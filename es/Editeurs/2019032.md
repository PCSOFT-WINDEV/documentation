


## Prueba y depuración de un sitio remoto
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000210"></a>
WEBDEV ofrece varios métodos para probar y depurar un sitio en el ordenador de desarrollo (véase [Los diferentes tipos de pruebas WEBDEV](../Editeurs/2019003.md)). Sin embargo, en algunos casos, es posible que tenga que depurar el sitio directamente en los equipos de los usuarios.

Desde su oficina en Londres, tiene la posibilidad de depurar un sitio que se ejecuta en un servidor Web en Taiwán.. La operación de depuración se realiza sin tener que ir a ninguna parte, en la configuración final directamente.

Hay dos opciones disponibles:

- Iniciando y depurando el sitio en un servidor de aplicaciones remoto.

- Depurando un sitio actualmente usado en un servidor de aplicaciones remoto.




Para estas dos opciones, se requiere una configuración específica del equipo remoto.

**Atención**: Esta característica no está disponible para los sitios desplegados en un servidor de aplicaciones Linux. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Configurando el remoto WEBDEV servidor de aplicaciones
<a name="configurando_remoto_webdev_servidor_aplicaciones_ELTTEXTE000234"></a>
Para depurar un sitio WEBDEV dinámico en un servidor de aplicaciones remoto, debes:

- **abrir el puerto principal y los puertos de sesiones** si se utiliza un cortafuegos. El puerto principal es el 27271 de Default y los puertos de las sesiones son los 27280 a 27289 de Default. 
	Estos puertos deben estar abiertos porque la comunicación con el servidor de aplicaciones WEBDEV remoto se establece mediante sockets.
	Los puertos utilizados pueden ser modificados en el WEBDEV servidor de aplicaciones (pestaña "Configuración" del administrador).

- ** permitir la depuración remota** en el WEBDEV servidor de aplicaciones. Para ello, marque "Permitir la depuración remota" en la pestaña "Configuración" del administrador..

## Iniciando y depurando un sitio en una remota servidor de aplicaciones
<a name="iniciando_depurando_sitio_una_remota_servidor_aplicaciones_ELTTEXTE000258"></a>
Para ejecutar y depurar un sitio en un servidor de aplicaciones remoto:

1. Abra el proyecto correspondiente en WEBDEV.

2. Compruebe la configuración del servidor remoto.

3. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Conectarse a un sitio y depurar conexión".

4. En la ventana que se visualiza, introduzca:

	- el nombre o IP Address del WEBDEV remoto servidor de aplicaciones.

	- el puerto principal de depuración.




5. Haga clic en el botón "Actualizar. Se muestran los nombres de los sitios que pueden ser depurados.

6. Selecciona el lugar deseado y valídalo.

7. El sitio se inicia en modo debug en el ordenador de desarrollo. Puede manipular el sitio y supervisar el código ejecutado a través del depurador (insertando puntos de interrupción en las secciones del código que causan problemas, por ejemplo).




Observaciones:

- Si la operación de depuración se realiza en un servidor de desarrollo, los elementos que se están editando se ejecutan y [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>True</u></u></u></u>.

- Si la operación de depuración se realiza en un WEBDEV servidor de aplicaciones para implementación, la ejecución del código es el código que se encuentra en la biblioteca (archivo WDL). Por lo tanto, el código del proyecto actual en el editor y el código del sitio pueden quedar fuera de sincronización.: el código que no está sincronizado se resalta en rosado durante la depuración. [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>True</u></u></u></u>.

## Depuración de un sitio actualmente utilizado en un servidor de aplicaciones remoto
<a name="depuracion_sitio_actualmente_utilizado_servidor_aplicaciones_remoto_ELTTEXTE000282"></a>
Para depurar un sitio ya iniciado en un servidor de aplicaciones remoto:

1. Abra el proyecto correspondiente en WEBDEV.

2. Compruebe la configuración del servidor remoto.

3. En la pestaña "Proyecto", en el grupo "Modo de prueba", despliegue "Modo de prueba" y seleccione "Depurar una conexión existente".

4. En la ventana que se visualiza, introduzca:

	- el nombre o IP Address del WEBDEV remoto servidor de aplicaciones.

	- el puerto principal de depuración.




5. Haga clic en el botón "Actualizar. Se muestran los nombres de los sitios que pueden ser depurados, con el identificador del cliente conectado.

6. Selecciona el lugar y la conexión deseada y valídalo.

7. El proyecto está en modo de depuración.




Observaciones:

- Si la operación de depuración se realiza en un servidor de desarrollo, los elementos que se están editando se ejecutan y [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>True</u></u></u></u>.

- Si la operación de depuración se realiza en un WEBDEV servidor de aplicaciones para implementación, la ejecución del código es el código que se encuentra en la biblioteca (archivo WDL). Por lo tanto, el código del proyecto actual en el editor y el código del sitio pueden quedar fuera de sincronización.: el código que no está sincronizado se resalta en rosado durante la depuración. [InTestMode](../WDLang1/3013011.md) devuelve <u><u><u><u>True</u></u></u></u>.

## Iniciar el depurador y detener la prueba
<a name="iniciar_depurador_detener_prueba_ELTTEXTE000306"></a>


### Inicio del depurador
<a name="inicio_del_depurador_ELTPARAGRAPHE000144"></a>

Para iniciar el depurador mientras se ejecuta la prueba del sitio, utilice el editor:

- En la pestaña "Depurador", haga clic en "Pausa" (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=main_debog.gif)).

- También se puede incluir un punto de ruptura en el editor de código delante de la deseada Line de código. El depurador se mostrará automáticamente cuando se ejecute el correspondiente Line.



<a name="NOTE5_2"></a>


### Detener la prueba
<a name="detener_prueba_ELTPARAGRAPHE000156"></a>

Existen varios métodos para detener la prueba:

- 1er método: Cierre el navegador. WEBDEV muestra el editor abierto cuando se inició la prueba.

- 2º método: En el editor, detenga el depurador (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatIconeArretTest.gif) o Shift + F5).

## Generación de un patrón RAD
<a name="generacion_patron_rad_ELTTEXTE000144"></a>
Para generar un patrón RAD:

1. En la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar".

2. En la asistente que se muestra, especifique:

	- El nombre del patrón así como su description. 
			Este nombre se usará para identificar el patrón cuando se utilice el RAD completo de la aplicación o cuando se generen las ventanas o páginas.. 
			La description se utiliza para describir las principales características del patrón. Este description es usado sólo por la aplicación completa RAD.

	- Los elementos que se pueden generar por sí solos. Estos elementos se propondrán al crear una ventana o una página. Para cada elemento, la flecha que se encuentra en la tabla permite introducir una leyenda y una description. Por ejemplo:

		- Pie de foto: Tabla (con edición)

		- description: Le permite ver los registros de archivos en una tabla e introducir nuevos registros..
						Consejo: Le aconsejamos que sólo proponga las ventanas o las páginas relacionadas con los archivos RADFileB y RADFileD.




- Si se puede aplicar una plantilla a los elementos generados por el RAD. Si el patrón es associated con un plantilla, recomendamos que no permita que se aplique el plantilla. 
			

- Si las tablas generadas deben mantener su modo de visualización (tratable o de sólo lectura) o no.. Si las tablas no mantienen su modo de visualización, la asistente para la generación de RAD propondrá gestionar la tabla en edición o en sólo lectura. 
			![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Si el desarrollador puede o no incluir el menú de ayuda de automatic en la aplicación. Si se permite el menú de ayuda de automatic, un avión específico propondrá incluir el menú "?" durante la generación del RAD.

- Si las imágenes de los menús se pueden personalizar o no. Si las imágenes de los menús pueden ser personalizadas, las imágenes del menú (en formato de botón o Carrusel) utilizarán las imágenes especificadas por el desarrollador que realiza la generación.. Puede especificar el tamaño recomendado para las imágenes.

- La Image del patrón. Esta Image se usará para identificar el patrón. La imagen de la primera ventana del proyecto es propuesta por Default. Tienes la posibilidad de elegir un Image de otra ventana o un Image proveniente de un archivo.

3. Validar. Se genera el patrón RAD. El archivo &lt;Pattern Name&gt;.Pattern se crea en el subdirectorio EXE del proyecto.



<a name="NOTE2_2"></a>


### Observación
<a name="observacion_ELTPARAGRAPHE000071"></a>

La característica de visibilidad del identificador automatic definido en el patrón se mantiene durante la generación. Por lo tanto, si el identificador de automatic es Invisible en el patrón de una forma, el control generado en la página correspondiente también será Invisible.




## Generar un description de un archivo de datos desde una ventana o desde una página
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000131"></a>
Al diseñar un prototipo rápido, a menudo se crean ventanas (o páginas) sin definir ningún archivo de datos.. Cuando se valida el prototipo, y sólo entonces, se decide crear las estructuras de los ficheros de datos.

En lugar de crear estos archivos de datos "manualmente", WINDEV, WEBDEV y WINDEV Mobile permiten exportar el contenido de la ventana (o página) para crear automáticamente la estructura correspondiente del archivo de datos.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000155"></a>
Para generar un description de un archivo de datos desde una ventana o desde una página: 

1. Abra la ventana (o la página) desde la que se generará el archivo de datos de la description.

2. Inicie el archivo description generación asistente: 

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) En la pestaña "Ventana", en el grupo "Edición", despliegue "Otras acciones" y seleccione "Generar una descripción de archivo". También puede seleccionar la opción "Generar una descripción de archivo" en el menú contextual de la ventana. 




3. Especifique:

	- el nombre lógico del archivo de datos. Este nombre se usará para manipular el archivo de datos. Por Default, este nombre corresponde al nombre de la ventana o página.

	- el pie de foto del archivo que resume brevemente el tema del archivo de datos. Por Default, esta leyenda corresponde al título de la ventana o página.

	- la representación de un Record en el archivo de datos. Esta representación facilita las preguntas formuladas al describir las relaciones. Esta opción debe ir precedida de un artículo indefinido (UNO o UNA).




4. Pase a la etapa siguiente.

5. Seleccione los controles a partir de los cuales deben crearse los elementos del archivo de datos. Sólo se proponen los controles que pueden ser relacionado para una posición.. Pase a la etapa siguiente.

6. Especifique (si es necesario) el tipo de clave para cada posición (no clave, clave unívoca o clave con duplicados).. Pase a la etapa siguiente.

7. La asistente presenta la descripción de la del archivo de datos para generar. Valide.




El nuevo archivo de datos se convierte en el archivo de datos actual en la editor de análisis. Para modificarlo, haga doble clic en el archivo de datos para mostrar el description de los elementos del archivo.
Para más detalles sobre cómo manipular un archivo de datos o elemento, véase [Operaciones en un archivo de datos](../Editeurs/2011007.md) y [Operaciones en una posición](../Editeurs/2011009.md).

## Correspondencia entre control y el artículo
<a name="correspondencia_entre_control_articulo_ELTTEXTE000179"></a>
Todos los controles de la ventana/página que pueden ser relacionado a un ítem (control Campo de entrada, botón de radio, tabla, ...) pueden convertirse en ítems en el description del archivo de datos generado.

El tipo y el tamaño de los elementos depende del tipo de información que se muestra en la control (texto, numérico, etc.) y del tamaño de la entrada.

Todas las características de la control se utilizan en la información compartida del artículo.




## Manejar las versiones de una ventana, reporte, consulta o proyecto
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000202"></a>



|   |
| --- |
| **Advertencia de compatibilidad** |
| Esta característica se mantiene para que sea compatible con las versiones anteriores.<br><br>Le recomendamos que utilice el SCM para gestionar las diferentes versiones del historial de los elementos de proyecto.<br><br>Para obtener más información, consulte: "[Resumen de SCM](../Editeurs/2038001.md)" y "[Configurar un proyecto para la SCM](../Editeurs/2038002.md)" . |

Al crear y desarrollar los diferentes elementos de un proyecto (ventana, consulta, reporte, proyecto, etc.), tiene la posibilidad de mantener varias versiones de un elemento. Esto le permite guardar una versión antes de realizar modificaciones importantes en el elemento (por ejemplo, cambiar la interfaz de una ventana).

Se puede crear un nuevo elemento a partir de una versión que se haya grabado previamente.

Por lo tanto, para una ventana, no es necesario guardar la misma ventana con nombres diferentes según la versión de la ventana.: el editor de ventanas gestiona automáticamente las diferentes versiones y las incluye en el archivo de copia de seguridad de la ventana. Una ventana corresponde a un archivo ".WDW. Este archivo contiene la description de la ventana así como todas las versiones guardadas para esta ventana. El tamaño del archivo ".WDW" es proporcional al número de versiones guardadas.

El mismo principio se aplica a las consultas, páginas, informes y proyectos (archivo WDP, WWP o WPP dependiendo del producto utilizado).

En cualquier momento, puedes hacerlo:

- Guardar una nueva versión del elemento actual.

- Abrir una versión del elemento actual.

- Borrar una versión del elemento actual.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000226"></a>


### Grabación de una versión
<a name="grabacion_una_version_ELTPARAGRAPHE000045"></a>

Guardar una versión se utiliza para mantener una copia de seguridad en un momento dado.

Para guardar una versión del elemento actual:

1. En la pestaña "Inicio", en el grupo "General", despliegue "Guardar" y seleccione "Versiones". Se muestra la ventana de las versiones del elemento actual.

2. Haga clic en el botón "Nuevo. Una ventana le permite escribir su propia comment sobre la versión a crear.

3. Validar. La nueva versión se guarda y el elemento actual se guarda automáticamente.




**Observación**: Las diferentes versiones de un elemento se muestran en orden descendente (de la versión más reciente) según su fecha y hora de copia de seguridad.
<a name="NOTE2_2"></a>


### Abrir una versión
<a name="abrir_una_version_ELTPARAGRAPHE000072"></a>

Abrir una versión le permite crear un nuevo elemento correspondiente a la versión seleccionada.

Para abrir una de las versiones del elemento actual:

1. En la pestaña "Inicio", en el grupo "General", despliegue "Guardar" y seleccione "Versiones". Se muestra la ventana de las versiones del elemento actual.

2. Seleccione la versión que desea abrir.

3. Haga clic en el botón "Abrir. Se crea un nuevo elemento en el editor correspondiente. Este elemento corresponde a la versión seleccionada. El nombre de este nuevo elemento tiene el siguiente formato: &lt;Nombre del elemento&gt; &lt;Fecha y hora de la copia de seguridad de la versión&gt;.



<a name="NOTE2_3"></a>


### Borrar una versión
<a name="borrar_una_version_ELTPARAGRAPHE000097"></a>

La eliminación de las versiones inútiles reduce el tamaño del archivo físico (".WDW" para una ventana) correspondiente al elemento actual.

Para borrar una versión del elemento actual:

1. En la pestaña "Inicio", en el grupo "General", despliegue "Guardar" y seleccione "Versiones". Se muestra la ventana de las versiones del elemento actual.

2. Seleccione la versión que desea borrar.

3. Haga clic en el botón "Eliminar. La versión seleccionada se borra automáticamente de la ventana para la gestión de versiones.. Esta eliminación será efectiva durante la siguiente copia de seguridad del elemento.

## Notas
<a name="notas_ELTTEXTE000262"></a>


### Abrir archivos después de un bloqueo del sistema
<a name="abrir_archivos_despues_bloqueo_del_sistema_ELTPARAGRAPHE000124"></a>

Para evitar problemas cuando se produce un bloqueo del sistema (por ejemplo, un corte de corriente), el editor propone un sistema para recuperar automáticamente los elementos abiertos en el editor: 

- En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile".

- En la pestaña "General", marque "Copia de seguridad para la recuperación automática cada" e indique la frecuencia deseada si es necesario.




Cuando este sistema está habilitado:

- Un respaldo de automatic de los elementos abiertos se realiza en intervalos de tiempo específicos (cada 10 minutos por Default).

- Cuando se reabre un elemento después de un bloqueo del sistema, aparece una ventana de recuperación de automatic. Esta ventana permite recuperar:

	- el elemento tal y como estaba durante la última copia de seguridad realizada por el desarrollador,

	- el elemento guardado automáticamente por WINDEV, WEBDEV o WINDEV Mobile.







La versión seleccionada se abre automáticamente en el editor. La versión no seleccionada está disponible en la ventana de gestión de versiones y se puede abrir en cualquier momento.

## Creación del proyecto RAD Pattern
<a name="creacion_del_proyecto_rad_pattern_ELTTEXTE000153"></a>
El patrón RAD se crea a partir de un proyecto "Patrón RAD".
<a name="NOTE2_2"></a>
Para crear un proyecto "Patrón RAD":

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. La ventana de creación de nuevos elementos se abre: haga clic en "Proyecto".

2. En la creación del proyecto asistente:

	- En "Tipo de proyecto", seleccione "Patrón RAD".

	- En el siguiente paso, introduzca el nombre del proyecto que se utilizará para crear el patrón.

	- Las demás opciones de creación de proyectos son opciones estándar. Se puede seleccionar un libro de estilo para su proyecto.  




3. Validar la creación del proyecto asistente.

4. El proyecto se crea automáticamente y associated con un análisis. Este análisis contiene los archivos necesarios para crear un RAD que se puede adaptar a todas las configuraciones de archivos.




Observación: WEBDEV integrado en WINDEV: Una configuración de "patrón RAD" sólo admite ventanas, no se pueden generar patrones para WEBDEV.

## El análisis del proyecto RAD Pattern
<a name="analisis_del_proyecto_rad_pattern_ELTTEXTE000183"></a>


### Análisis
<a name="analisis_ELTPARAGRAPHE000054"></a>Todos los proyectos "Patrón RAD" son associated con el mismo análisis. Este análisis contiene todos los tipos de enlaces y grupos de archivos.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Projet_Pattern_RAD%20-%20HC%20N%B0001.gif)
 Este análisis contiene:

- un único fichero (RADFileC por ejemplo).

- dos archivos relacionado mediante un simple enlace (RADFileB relacionado a RAD FileC o RADFileA relacionado a RADFileB, por ejemplo).

- dos ficheros (y un fichero de relación) relacionado mediante un enlace complejo (RADFileB relacionado a RADFileC mediante el fichero de relación RADFileD).




Cualquier análisis puede dividirse en varios conjuntos de estos tres grupos de archivos.
<a name="NOTE3_2"></a>


### Ejemplo de aplicación
<a name="ejemplo_aplicacion_ELTPARAGRAPHE000069"></a>

Intentemos dividir el siguiente análisis:  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RAD_Analyse_Ex.gif)


Se crean los siguientes grupos:

- Cliente.

- Ordenar.

- Producto.

- Proveedor.

- Cliente relacionado a Pedidos.

- Ordena relacionado a Product por el archivo de enlace llamado OrdLine.

- Producto relacionado a Proveedor.




## Página por página implementación
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000093"></a>
WEBDEV le permite desplegar secciones del sitio o páginas individuales. No hay necesidad de actualizar todo el sitio cada vez que se realiza una modificación.

Los usuarios de la Web no se desconectan al actualizar la página (y sus dependencias).

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000117"></a>


### Condiciones necesarias
<a name="condiciones_necesarias_ELTPARAGRAPHE000020"></a>

- Para realizar un implementación página por página de un sitio, el sitio **debe haber sido desplegado al menos una vez en un servidor de aplicaciones** remoto.

- Desplegando páginas dinámicas y/o páginas de AWP.



<a name="NOTE2_2"></a>


### Realizando un implementación página por página
<a name="realizando_implementacion_pagina_por_pagina_ELTPARAGRAPHE000028"></a>

Para realizar un implementación página por página: 

1. Seleccione la página que desea actualizar en el panel "Explorador de proyectos".

2. Seleccione "Desplegar" de el menú contextual.

3. La implementación de la página (y las dependencias si es necesario) se realiza. El servidor utilizado para desplegar la página es el último servidor utilizado durante la última instalación del sitio (ver [Despliegue de un sitio en un servidor de aplicaciones remoto](../Editeurs/2028019.md)).



<a name="NOTE2_3"></a>


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000042"></a>

- Sólo las páginas existentes que han sido modificadas pueden ser desplegadas página por página.. Si se añaden nuevas páginas, se debe redistribuir todo el sitio..  

- Si se añaden elementos al sitio (consultas, informes,...), se debe redistribuir todo el sitio.

- Si se modifica un código global, se debe redistribuir todo el sitio.

## Presentación
<a name="presentacion_ELTTEXTE000178"></a>
La instalación de dos niveles (o instalación multisitio) combina:

- un reference [instalación de red](../Editeurs/2028085.md) desplegado en una red local.

- un [Instalación HTTP](../Editeurs/9000026.md) utilizado para actualizar el instalación de referencia en la red local.




La instalación se realiza en 3 pasos:

- **Descargue una instalación de referencia en un servidor local a través de un servidor HTTP**. Durante una primera instalación, el usuario final inicia la instalación mediante INTERNET, a través de una Page específica.

- **Instale este reference instalación en un servidor local**. La aplicación reference corresponde al programa de aplicación instalación en los ordenadores de los usuarios. 
	Al final de esta instalación, se crea una tarea programada en el servidor local. Esta tarea programada se utiliza para comprobar y actualizar (cada 15 minutos) la versión de la reference instalación según la versión del HTTP instalación. 

- **La aplicación final se instala en cada computadora de usuario ejecutando el programa instalación que se encuentra en el servidor**. Las actualizaciones se realizarán automáticamente cuando la aplicación reference cambie en el servidor




Al iniciar la aplicación, se realizará una actualización de automatic (según la versión de la aplicación instalada en el ordenador del usuario)

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000202"></a>


### Creación de un multisitio instalación
<a name="creacion_multisitio_instalacion_ELTPARAGRAPHE000039"></a>

Para crear un multisitio instalación: 

1. Cree el ejecutable de la aplicación (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_Generation_Exe.gif) en los iconos de acceso rápido). 

2. Empieza a crear la instalación Procedure: 

	- ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_Generation_Exe.gif) entre los iconos de acceso rápido. 

	- Seleccione "Crear la instalación Procedure del ejecutable".




3. En el asistente, seleccione "Configuración".

4. Seleccione "Configuración con actualización automatic (Live Update)".. 

5. Elija "Multisitio instalación".

6. Elija si desea crear un paquete implementación o desplegarlo inmediatamente en una plataforma PCS Cloud. 

7. Siga los diferentes pasos de la asistente. 



<a name="NOTE2_2"></a>


### Para instalar la versión reference
<a name="para_instalar_version_reference_ELTPARAGRAPHE000060"></a>

Una vez que la instalación Procedure es creada, tienes la capacidad de ejecutar el programa instalación. Este programa también se puede ejecutar directamente a través del archivo INSTALL.EXE.

1. Inicie el programa instalación que fue creado con anterioridad. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Installation_a_2_niveaux%20-%20HC%20N%B0001.gif&type=thumb)


2. Elija el servidor de destino en el que se instalará la reference instalación: 

	- Ordenador actual (opción "Desplegar instalación en este ordenador"). Este modo de instalación no está disponible durante la instalación en un servidor Linux.

		- Si se instala un servidor HTTP en el ordenador actual, este servidor se configura y los archivos de la aplicación reference se copian en el directorio especificado. 

		- Si no hay un servidor HTTP instalado en el ordenador actual, se instala un servidor Default (WDADMINEXE) y se configura automáticamente. Los archivos de la aplicación reference se instalan en el directorio especificado. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Installation_a_2_niveaux%20-%20HC%20N%B0002.gif&type=thumb)







Observaciones: 

- En algunos casos, puede que tenga que configurar el servidor HTTP. Por ejemplo, puede crear un directorio virtual (alias) en el sitio web que apunta al directorio donde se realizó el FTP implementación. 

- En cualquier caso, está disponible una Page HTML que permite al usuario final descargar la aplicación instalación. Esta Page puede incluirse en su sitio.




**3.** Valide el instalación de la versión reference .

Al final de esta instalación, se crea una tarea programada en el equipo. Esta tarea programada se utiliza para comprobar y actualizar (cada 15 minutos) la versión de la reference instalación según la versión del HTTP instalación. 

El nombre de la tarea programada creada se especifica en el último avión instalación. Este nombre tiene el siguiente formato: UPD__. Ejemplo: UPD_WDGoldLite_2169562193.

### Protocolos utilizados
<a name="protocolos_utilizados_ELTPARAGRAPHE000111"></a>

La instalación y las actualizaciones de la aplicación reference en el servidor se realizan mediante el protocolo FTP.

La instalación y las actualizaciones de la aplicación en los ordenadores de los usuarios finales de la aplicación reference se realizan mediante el protocolo HTTP. 

Las peticiones HTTP descargan los archivos (extensiones ".tmp", ".net" y ".zip") del servidor.. Por lo tanto, estas extensiones deben ser aceptadas por el servidor web que aloja la aplicación reference. Para obtener más información, consulte [FAQ #3102](https://faq.pcsoft.fr/faqread.awp?idfaq=3102&lang=es).
<a name="NOTE3_4"></a>


### Descargar Page
<a name="descargar_page_ELTPARAGRAPHE000125"></a>

Al desplegar la aplicación, se crea automáticamente un htm Page con un enlace al archivo instalación de la aplicación cliente (enlace a Install.exe).

Esta Page puede modificarse y personalizarse según sus propias necesidades.

Esta Page se puede mover. Para ello, actualice el enlace de descarga del programa Page.

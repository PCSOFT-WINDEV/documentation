## Presentación
<a name="presentacion_ELTTEXTE000186"></a>
Los parámetros del sitio sirven para configurar las opciones de conexión al sitio desde el momento de su instalación.. Estos parámetros pueden ser modificados si es necesario:

- por el administrador de implementación que se encuentra en el servidor.

- por el administrador remoto, que puede ser utilizado por el administrador del sitio.

## Configurar el sitio
<a name="configurar_sitio_ELTTEXTE000210"></a><br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=param_site.gif)
Los parámetros de implementación son los siguientes:

- **Activar sitio inmediatamente después de instalación**: 
	El sitio estará disponible una vez que se haya instalado: podrás conectarte usando la Address adecuada. 
	Si esta opción no está seleccionada, el sitio puede ser activado a través del administrador remoto.

- **Realice una copia de seguridad del sitio antes de la actualización**: 
	Para una actualización, le permite guardar el sitio existente. 
	Esta copia de seguridad copy debe tener un nombre y puede ser una copia de seguridad completa o contener sólo los archivos desplegados.
	Esta copia de seguridad puede reactivarse desde el Administrador remoto de WEBDEV. 

- **Programar implementación para una fecha posterior**: 
	En este caso, la implementación se realizará en la fecha y hora especificadas. Para obtener más información, consulte [Inmediato o retrasado instalación](#NOTE2_2). 




Los parámetros del sitio son los siguientes: 

- **Desconectar usuarios inactivos después de**: (sólo cuando se despliega un sitio WEBDEV dinámico)
	Tiempo máximo de conexión permitido sin que el usuario realice ninguna acción. 
	Cuando se excede este tiempo, el usuario Web se desconecta automáticamente y aparece un mensaje de error.

- **Duración de contextos AWP**: (sólo cuando se despliega un sitio AWP WEBDEV)
	Periodo de validez de los contextos de AWP para el sitio. Tan pronto como finalice la duración especificada y si no se ha realizado ninguna nueva solicitud, se elimina el archivo context. Para más detalles sobre los contextos AWP, véase [AWP: Gestión de contextos](../WDChamp/3539060.md).

- **Duración máxima de la tarea**: 
	Duración máxima para ejecutar las tareas programadas o retrasadas. Si las tareas utilizadas superan esta duración, se detendrán automáticamente.. Para obtener más información, consulte [WEBDEV: Tareas programadas y tareas retrasadas](../WDLang2/1000019455.md). 

- **Número máximo de conexiones simultáneas al sitio**: 
	Corresponde al número máximo de conexiones simultáneas permitidas para este sitio.. Por ejemplo, si este número es set a 250, 250 usuarios de la web podrán conectarse a este sitio al mismo tiempo.

- **Número máximo de sesiones en caché para el sitio**: 
	Número de sesiones de caché para el sitio. 
	Para obtener más información, consulte [Sesiones preiniciadas](../WDLang2/1000019485.md).

- **Número máximo de conexiones por usuario**: (sólo cuando se despliega un sitio WEBDEV dinámico)
	Corresponde al número máximo de conexiones simultáneas al sitio permitidas para el mismo usuario web.. 
	Si este parámetro es set a cero, el usuario que intente conectarse simultáneamente al mismo sitio será desconectado automáticamente y luego reconectado. 
	Si este parámetro es mayor que cero, se mostrará automáticamente un mensaje de error después de las x conexiones simultáneas.




Durante un implementación en varios servidores simultáneamente, tienes la posibilidad de configurar los parámetros del sitio para cada perfil. Se tienen en cuenta los siguientes elementos: 

- Número máximo de conexiones simultáneas al sitio.

- Número máximo de conexiones por usuario.

- Desconecte los usuarios inactivos desde entonces.

### Inmediato instalación: caso de una instalación en un remoto WEBDEV servidor de aplicaciones
<a name="inmediato_instalacion_caso_una_instalacion_remoto_webdev_servidor_aplicaciones_ELTPARAGRAPHE000101"></a>

WEBDEV propone una **instalación inmediata** del sitio de forma predeterminada. En este caso, los archivos se transfieren inmediatamente por FTP al servidor y se instalan en el directorio definido en el servidor.

En algunos casos, un **instalación retrasado** puede ser useful. Por ejemplo:

- instalación de un sitio grande: un instalación retardado permite comprobar si todos los archivos están instalados (en instalación inmediato, el instalación puede detenerse según el tiempo de espera definido para el CGI.

- actualización de un sitio: una instalación retrasada le permite realizar su instalación por la noche, para reducir el número de usuarios de la Web conectados.

- actualizando varios sitios que comparten un análisis con la modificación de datos de automatic: una actualización retardada permite actualizar todos los sitios al mismo tiempo para tener en cuenta la modificación de los archivos de datos de automatic.




Durante una instalación diferida, los archivos se transmiten inmediatamente por FTP, pero sólo se instalan realmente cuando se alcanzan la fecha y la hora especificadas en el servidor.

**Atención**: la fecha y la hora especificadas son las del servidor. Asegúrese de que son válidos.



## Seguridad de sitios Web
<a name="seguridad_sitios_web_ELTTEXTE000240"></a>
Entonces, la asistente le permite definir los parámetros de seguridad del sitio: 

- **Cambio de IP durante la navegación**: Si esta opción no está permitida, la dirección IP de associated con la sesión no puede cambiar mientras se navega. Se utiliza para protegerse de los ataques de "secuestro de sesión" (ataque que consiste en hacerse pasar por un usuario legítimo conectado al servidor).. 

- **Acceso a los identificadores de contexto AWP desde JavaScript**: En un sitio de AWP, el contexto del sitio se almacena en el servidor. El identificador de este contexto es enviado y almacenado en el navegador a través de un Cookie. Si no se permite esta opción, las cookies utilizadas serán cookies "HTTPOnly", que no se pueden leer desde código JavaScript. Este modo protege contra ataques XSS (Cross-Site scripting). 
	Por Default, el acceso a los identificadores de contexto de AWP no está permitido desde JavaScript. 






## Generación de estadísticas del sitio
<a name="generacion_estadisticas_del_sitio_ELTTEXTE000264"></a>
Entonces, la asistente le permite implementar los archivos estadísticos del sitio. Estos archivos son explotados por WDStatistic para conocer el tráfico del sitio o las páginas más visitadas, por ejemplo.

Puede modificar el directorio del archivo estadístico o puede elegir no generar estos archivos desmarcando "Generar los archivos estadísticos para este sitio".. El directorio de los ficheros estadísticos es siempre relativo en relación con el directorio de datos.

Para obtener más información, consulte [WDStatistic](../WDStatistique/3536004.md).
Durante un implementación en varios servidores simultáneamente, tienes la posibilidad de configurar los archivos estadísticos para cada perfil.

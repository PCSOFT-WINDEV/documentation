<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000154"></a>
Un Modelo de Datos Lógicos (LDM) permite describir las estructuras de datos (archivos de datos, elementos,...) utilizadas en una aplicación WINDEV, WEBDEV o WINDEV Mobile.

**Observaciones**:

- Los términos "LDM" y "Análisis" se utilizan indistintamente para definir la estructura de la base de datos associated con un proyecto.

- Para obtener más información, consulte [Manipulación en LDM](../Editeurs/2011006.md).

## Crear un LDM
<a name="crear_ldm_ELTTEXTE000184"></a>
Para crear un LDM: 

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- La ventana de creación de nuevos elementos se abre: haga clic en "Datos" y luego en "Análisis".

	- El asistente de creación de análisis se abre.




2. En el primer paso de la asistente, indique:

	- los ***nombre y directorio del análisis***. El análisis corresponde a un archivo ".WDA". Este archivo se creará en el directorio del análisis del proyecto de forma predeterminada (directorio &lt;Nombre del proyecto&gt;.ANA. Este directorio debe ser accesible en modo lectura/escritura.

	- si el análisis debe ser associated con el proyecto actual en lugar del análisis actual (cuando se crea un análisis en un proyecto ya associated con un análisis). 

	- la ***descripción del análisis*** que describe brevemente el tema del análisis. 




3. El siguiente paso de asistente es especificar el tipo de base de datos que utilizará el proyecto: 

	- HFSQLDrive (solución de pago). Para más detalles sobre HFSQLDrive, véase [HFSQLDrive, su base de datos en la nube en 3 clics](https://pcscloud-drive.net/UK/HFSQLDrive/index.awp).

	- PCSCloud (solución de pago). Para más detalles sobre PCSCloud, consulte [Nube para bases de datos HFSQL](https://pcscloud.net/UK/cloud_applications.awp).

	- HFSQL Client/Server, 

	- HFSQL Classic, 

	- Bases de datos externas: SQL Server, Oracle, MySQL, ...
			Observación: Si el análisis debe manipular una base de datos externa, el correspondiente acceso nativo debe ser instalado en el equipo. 




4. Para cualquier base de datos que no sea HFSQL Classic, el asistente permite definir la conexión a la base de datos (nombre, título) y especificar la configuración avanzada (servidor, usuario y contraseña). Esta información puede editarse posteriormente en el programa editor de análisis.

5. Validar. El análisis creado se convierte en el análisis actual.

6. Cree todos los elementos del análisis (archivos de datos, campos y relaciones).
	**Observación**: Para más detalles, consulte [Archivos de datos de análisis](../Editeurs/2011010.md), [Artículos de un archivo de datos](../Editeurs/2011009.md) y [Enlaces de análisis](../Editeurs/2011008.md).

7. Para utilizar el análisis en su proyecto, genere el análisis: en la pestaña "Análisis", en el grupo "Análisis", despliegue "Generación" y seleccione "Generación". Para obtener más información, consulte [Generación de un análisis](../Editeurs/2011012.md).

### Optimización del análisis
<a name="optimizacion_del_analisis_ELTPARAGRAPHE000092"></a>

**A partir de la versión 20, se optimiza el formato de almacenamiento del análisis.**. Cualquier nuevo análisis creado con la versión 20 (o posterior) utiliza este nuevo formato. 

Tiene la posibilidad de grabar los análisis existentes con este nuevo formato. 
**Atención: Un análisis que utilice este nuevo formato no será compatible con versiones anteriores a la 19.**

Para utilizar el nuevo formato en sus análisis existentes: 

1. Abra la ventana de análisis description ("Análisis description" en el menú contextual del gráfico de análisis). 

2. En la pestaña "Detalles", haga clic en "Guardar el análisis en un formato más compacto".

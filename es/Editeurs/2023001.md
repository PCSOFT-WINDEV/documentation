## Presentación
<a name="presentacion_ELTTEXTE000096"></a>
La suscripción a un elemento del diccionario se propone automáticamente:

- Cuando un elemento es registrado en el diccionario.

- Cuando un elemento del diccionario se copia en un proyecto.

## Operaciones relacionado a la suscripción
<a name="operaciones_relacionado_suscripcion_ELTTEXTE000120"></a>
Si el objeto del suscriptor no está sincronizado con el diccionario, se muestra una pantalla para actualizar el objeto del suscriptor al abrir o guardar este objeto.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Abonnement.gif)


Esta pantalla está acostumbrada a:

- Abrir el elemento sin actualizar la suscripción (botón de validación).

- Cancel la apertura del elemento (botón de Cancel ATION).

- Actualizar todos los elementos anteriores (botón"Actualizar todos").

- Actualizar el elemento del diccionario desde el elemento del suscriptor seleccionado en la lista (botón"A diccionario").

- Actualizar el elemento de abonado seleccionado desde el elemento del diccionario (botón"Desde el diccionario").

- Eliminar la suscripción (botón"Eliminar el enlace").

- Mostrar los detalles de las diferencias (botón"Detalles").

## Caso especial: el conflicto
<a name="caso_especial_conflicto_ELTTEXTE000144"></a>
Un elemento del diccionario puede estar en conflicto con el elemento del suscriptor: el elemento diccionario y el elemento suscriptor se han modificado por separado y ya no están sincronizados. El desarrollador puede:

- Incluir las evoluciones del elemento diccionario: se perderán las últimas evoluciones del elemento abonado.

- Actualizar el elemento del diccionario: sus últimas evoluciones se perderán.

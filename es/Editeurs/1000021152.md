


## Groupware Usuario
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000203"></a>
Una aplicación Windows, un sitio de Internet o Intranet requieren que se defina el papel de los diferentes colaboradores. Se pueden establecer varios niveles de acceso según el usuario. Todos los usuarios tienen diferentes responsabilidades y requisitos. Por lo tanto, suele ser necesario limitar sus campos de acción.

Veamos un ejemplo simple: una aplicación de gestión de ventas que propone las siguientes funcionalidades:

- Ver la lista de precios.

- Modificar la lista de precios.

- Agregar pedidos.

- Agregar clientes.




El acceso puede variar según el usuario. Por ejemplo:

- el personal de secretaría puede ver la lista de precios y crear pedidos.

- los vendedores pueden ver la lista de precios, hacer pedidos y crear nuevos clientes.

- los administradores tienen acceso a todas las opciones.




Para gestionar estos niveles de acceso en las aplicaciones WINDEV o sitios WEBDEV, se debe incluir el Groupware Usuario en la aplicación o sitio. Con unos pocos clics, una aplicación estándar puede transformarse en una aplicación que gestiona varios niveles de acceso. Esta característica se puede implementar fácilmente, tanto en una aplicación WINDEV como en un sitio WEBDEV.

Al ejecutar la aplicación o el sitio, el administrador podrá crear usuarios (identificados por su nombre de usuario y contraseña) y darles acceso a determinadas funcionalidades.

**Condiciones de uso**: 

- El groupware usuario está disponible: 

	- en aplicaciones WINDEV (Windows y Linux).

	- en sitios WEBDEV dinámicos (en modo Sesión o AWP) ejecutados en Windows y Linux. 

	- en un sitio mixto AWP/Estándar. En este caso (proyecto con páginas AWP y páginas en modo Estándar), la conexión desde una página AWP se transmite a la sección estándar. 




- El Groupware Usuario no está disponible en sitios estáticos, PHP o Ajax.

- El cifrado de contraseñas del Groupware Usuario se activa de forma predeterminada al integrar el Groupware Usuario en un proyecto. Para activar esta opción en un groupware existente, consulte [Cifrado de contraseñas](../Editeurs/1000021283.md).

### Modo de funcionamiento del Groupware Usuario
<a name="modo_funcionamiento_del_groupware_usuario_ELTPARAGRAPHE000112"></a>

Una aplicación que utiliza el Groupware Usuario tiene dos niveles:

- nivel de usuario
	El usuario inicia sesión en la aplicación o el sitio mediante una ventana o página y accede a las funcionalidades disponibles.

- nivel de supervisor
	El supervisor inicia sesión en la aplicación mediante una ventana o página y accede a un menú avanzado para configurar los usuarios y sus permisos, o para iniciar la aplicación. La ventana o página de configuración permite al supervisor:

	- iniciar la aplicación.

	- configurar el groupware.
			La configuración del groupware consiste en crear usuarios y grupos, asociar usuarios a grupos, y administrar los permisos otorgados a cada usuario (o grupo) en cada ventana/página de la aplicación. Los permisos se pueden definir para las opciones de menú, grupos de controles y controles. El supervisor puede inhabilitar, desactivar o hacer invisibles estos elementos. Estas configuraciones tienen prioridad sobre cualquier configuración definida en el programa.







Para más información sobre la configuración del groupware definida por el supervisor, consulte: 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [Configurar la aplicación (definir usuarios, grupos, etc.)](../Editeurs/1000021154.md).




**Observaciones**: 

- La información sobre los usuarios y sus permisos se almacena en archivos HFSQL. Estos archivos pueden ser comunes a varias aplicaciones. 

- El Groupware Usuario de la versión en francés no es compatible con el Groupware Usuario de la versión internacional: análisis diferente, nombres de archivos diferentes, variables con nombres diferentes, etc.

- Si ya ha configurado el Groupware Usuario con una versión anterior (posterior a la 17), no es necesario migrar los datos para cambiar el modo de gestión del Groupware Usuario. Los datos se conservan.

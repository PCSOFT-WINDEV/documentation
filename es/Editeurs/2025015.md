
## Windows: Qué se debe tener en cuenta si los usuarios están trabajando con Vista (y más tarde)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000081"></a>
Windows Vista (y posteriores) propone varias mejoras. Estas mejoras traen consigo algunas modificaciones en cuanto a la programación, la creación del ejecutable o las aplicaciones instaladas.

Entre las mejoras que hay que tener en cuenta:

- La cuenta de usuario control (UAC)

- El administrador del dispositivo Windows Mobile, necesario para sincronizar los dispositivos móviles.

<a name="NOTE2_1"></a>


## UAC y creación de ejecutables
<a name="uac_creacion_ejecutables_ELTTEXTE000105"></a>
Para aumentar la seguridad contra gusanos y virus, Windows Vista/7 implementó el mecanismo UAC (User Account control).

Por Default, este mecanismo considera que todos los usuarios (incluidos los administradores) son usuarios estándar con derechos reducidos.

Para ejecutar una aplicación que requiere derechos adicionales, el mecanismo UAC solicita confirmación.

Para más detalles sobre este sistema, y las consecuencias para sus ejecutables, consulte [El mecanismo de la UAC](../Editeurs/2025014.md).

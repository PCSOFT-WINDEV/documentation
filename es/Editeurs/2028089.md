


## Paquete de implementación
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000106"></a>
Este paquete de implementación se utiliza para crear un archivo ZIP que contiene la instalación del sitio. Este archivo se puede entregar al proveedor que instalará el sitio con WDDeploy.

**Observación**: WDDDeploy es una herramienta WEBDEV que permite:

- a la empresa de hosting para desplegar un sitio dinámico desde un paquete.

- para implementar sitios WEBDEV estáticos.

- para implementar sitios PHP.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000130"></a>
Para crear un paquete de implementación: 

1. Iniciar la asistente para crear la instalación de un sitio WEBDEV: en la pestaña "Proyecto", en el grupo "Generación", despliegue "Implementar sitio" y seleccione "Generar e implementar sitio".

2. Generar la biblioteca del sitio. 

3. Seleccione "Crear un paquete de implementación remoto". 

4. [Introduzca la información sobre el servidor](../Editeurs/2028046.md).

5. [Seleccione los archivos a instalar](../Editeurs/2028058.md).

6. [Configurar la modificación de automatic](../Editeurs/2028048.md)

7. [Configurar el sitio](../Editeurs/2028051.md).

8. Especifique las características del paquete:

	- El directorio de generación del paquete. Puede elegir borrar (o no) el contenido del directorio durante la generación.

	- El nombre del paquete de implementación.




9. Validar la creación del paquete. Una vez que la creación del paquete se completa, WEBDEV propone:

	- para iniciar el implementación del sitio (vía WDDeploy).

	- para abrir el explorador.

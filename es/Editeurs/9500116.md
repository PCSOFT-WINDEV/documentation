


## automatic para copiar formularios
			





## Presentación
<a name="presentacion_ELTTEXTE000095"></a>
Al escribir varias copias de un formulario una tras otra, parte de la información que se debe escribir puede ser la misma para cada instancia del formulario.. Por ejemplo, cuando se teclea un conjunto de artículos, el nombre y la reference de los artículos cambian cada vez pero el nombre del proveedor puede permanecer igual durante varias entradas.

Para agilizar las operaciones, puede pedir a la ventana actual que copy la información introducida en los diferentes controles en el portapapeles para pegarla cada vez que se realice una nueva entrada.



## Copiar los datos del formulario
<a name="copiar_los_datos_del_formulario_ELTTEXTE000119"></a>
Para copy los datos de la ventana actual al portapapeles, utilice:

- La opción "Formulario copy" en el menú contextual de la ventana.

- El atajo Ctrl + Alt + C.

- La función [Formulario WinCopy](../WDLang1/1000018992.md).

- [AAFExecute](../WDLang1/1000022099.md) como sigue:
	
	```wl
	AAFExecute(MyWindow, aafCopyForm)
	```







## Pegar los datos del formulario
<a name="pegar_los_datos_del_formulario_ELTTEXTE000143"></a>
Para pegar los datos del portapapeles en la ventana actual, utilice:

- La opción "Pegar el formulario" de el menú contextual de la ventana.

- El atajo Ctrl + Alt + V.

- La función [Formulario WinPaste](../WDLang1/1000018993.md).






## Observaciones
<a name="observaciones_ELTTEXTE000167"></a>
Sólo los controles en edición de la ventana se modifican al pegar un formulario.. Los controles de sólo lectura o los controles de Grayed mantienen sus valores.

Todos los controles se copian en el portapapeles cuando se copia un formulario.. Puede pegar el resultado en un bloc de notas, por ejemplo. Por el contrario, no se puede copy texto de un bloc de notas para pegarlo en una ventana.

Durante la operación de pegado, no se ejecuta ningún Process (modificación Process o salida de control por ejemplo). No se informa a la ventana de que los valores de sus controles han cambiado.. Si esta característica de automatic está activada en una aplicación, debe comprobar la validez de los control s al validar la ventana (y no en la salida de control).

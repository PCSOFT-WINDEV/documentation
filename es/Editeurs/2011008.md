


## Enlaces de análisis
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000146"></a>
Los enlaces se utilizan para enlazar los ficheros de datos descritos en un análisis entre ellos.

Hay varios tipos de enlaces disponibles. Para obtener más información, consulte [Tipos de relaciones](../Editeurs/2011001.md).

**Recordatorio**: Los términos "LDM" y "Análisis" se utilizan indistintamente para definir la estructura de la base de datos associated con un proyecto.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Creación de un enlace
<a name="creacion_enlace_ELTTEXTE000170"></a>
Para crear un enlace:

1. En la pestaña "Análisis", en el grupo "Creación", haga clic en "Nueva relación". El puntero se convierte en un lápiz.

2. Haga clic en los dos archivos de datos que deben ser relacionado. El enlace description asistente se abre automáticamente.

3. Para definir las cardinalidades entre los dos archivos:

	- seleccione las cardinalidades deseadas (0, 1; 1, 1; 0, N; 1, N).

	- responda a las preguntas. Las cardinalidades se actualizarán automáticamente.




4. Para definir las cardinalidades avanzadas, seleccione la opción "Mostrar cardinalidades avanzadas" y responda a las preguntas.

5. Especifique el título de la relación y describa brevemente el propósito de la misma. Pase a la etapa siguiente. 

6. Especifique las claves que se relacionarán. Pase a la etapa siguiente. 

7. Defina las reglas de integridad. Estas reglas garantizan la integridad de los datos cuando se modifica o elimina una de las claves de la relación.

8. Validar. La relación se crea automáticamente.




**Observaciones**: 

- Para más detalles sobre las cardinalidades, véase [Cardinalidades](../Editeurs/2011014.md).

- Si los archivos de datos a vincular contienen una clave compuesta única idéntica (y también contienen los elementos incluidos en la clave compuesta), se puede realizar un enlace en esta clave compuesta.

## Notas
<a name="notas_ELTTEXTE000194"></a>


### Archivo propietario y archivo miembro
<a name="archivo_propietario_archivo_miembro_ELTPARAGRAPHE000066"></a>

Cuando hay un enlace entre dos archivos de datos, hay un archivo propietario y un archivo de miembro:

- el archivo de propietario es el propietario de la clave.

- el archivo de miembros es un miembro de los archivos de análisis que contiene un copy de clave.




Para gestionar el enlace entre dos archivos de datos, la clave del archivo propietario se copia en el archivo de miembro.

Por ejemplo, la clave de la ficha "Proveedor" se copia en cada Record de la ficha "Producto".. Varios registros del archivo "Producto" pueden tener la misma clave que el archivo "Proveedor":

- el archivo propietario es"Proveedor",

- el archivo de miembro es "Producto".

### Integridad referencial
<a name="integridad_referencial_ELTPARAGRAPHE000086"></a>

La integridad referencial en una base de datos es un conjunto de restricciones que mantiene la coherencia de los datos en una relación entre dos archivos de datos.

La integridad referencial verifica que:

- si se borra un Record del archivo del propietario, los Record correspondientes también se borran de los archivos de los miembros,

- si se agrega un registro a un archivo de datos miembro, existe un registro relacionado en el archivo de datos propietario,

- si se modifica un Record en el archivo del propietario, la clave única no se modifica,

- ...




La integridad referencial se comprueba en función de la naturaleza de la relación entre los archivos de datos.

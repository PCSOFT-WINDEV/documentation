


## Software de grupo para usuarios (WINDEV y WEBDEV): Opciones y ajustes
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000381"></a>
Una aplicación a menudo está destinada a ser utilizada por varios usuarios conectados a través de una red.. Del mismo modo, un sitio siempre está destinado a ser utilizado por varios usuarios de la Web. En la mayoría de los casos, se deben definir diferentes niveles de acceso en función del usuario o del usuario Web.

Para gestionar estos niveles de acceso en las aplicaciones WINDEV o sitios WEBDEV, se debe incluir el Groupware Usuario en la aplicación o sitio. Con unos pocos clics, una aplicación estándar puede transformarse en una aplicación que gestiona varios niveles de acceso. Esta funcionalidad puede implementarse fácilmente.

Para más detalles sobre el programa de grupo de usuarios, véase [Visión general de los grupos de usuarios](../Editeurs/1000021152.md).

## Las opciones de la Agrupación de Usuarios
<a name="las_opciones_agrupacion_usuarios_ELTTEXTE000405"></a>


### Implantación del programa de grupo de usuarios en su aplicación
<a name="implantacion_del_programa_grupo_usuarios_aplicacion_ELTPARAGRAPHE000025"></a>

Para implantar el User Groupware en su aplicación o sitio: 

1. En la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Groupware Usuario". 

2. En la ventana de descripción de la configuración del proyecto actual, en la sección "Groupware Usuario: Desactivado", haga clic en "Cambiar". 

3. En la ventana que aparece, habilite el programa de grupo de usuarios. Puede configurar el User Groupware para su aplicación o sitio en las diferentes pestañas de esta ventana. Esta configuración se presenta en los siguientes párrafos. 

4. Validar. El programa de grupo de usuarios se integra en su proyecto. 

5. Se creará una cuenta de administrador durante la primera ejecución de la aplicación o sitio bajo el nombre de "supervisor".. A continuación, se puede especificar la contraseña correspondiente.

## Opciones de la pestaña "General
<a name="opciones_pestana_general_ELTTEXTE000429"></a>
La pestaña "General" permite habilitar o deshabilitar la Agrupación de Usuarios en la configuración actual del proyecto. Esta pestaña incluye las siguientes opciones: 

- Tipo de programa de grupo,

- Modo de inicio

- Cuenta de usuario. 


Veamos con más detalle estas opciones. 


### Tipo de software de grupo
<a name="tipo_software_grupo_ELTPARAGRAPHE000061"></a>

Existen varios tipos de groupware: 

- **Groupware Usuario estándar**: El Groupware Usuario se integrará en su versión "estándar": el Grupo de Usuarios será associated sólo a su aplicación o sitio, y los derechos serán definidos desde la aplicación o sitio. 
	Si se selecciona esta opción, aparecen dos pestañas específicas: 

	- la pestaña "Archivos", que permite especificar el tipo de base de datos y de autenticación. 

	- la pestaña "OAuth", que permite especificar los servicios de conexión disponibles para los usuarios. 




- **Groupware Usuario BaaS**: Esta opción permite integrar un Groupware Usuario BaaS (Backend as a Service) y gestionar los permisos mediante un servidor. 
	Si se selecciona esta opción, aparecen dos pestañas específicas: 

	- la pestaña "BaaS", que permite especificar las características del servidor BaaS, las cuentas de usuario y las URL de los sitios de administración. 

	- la pestaña "Correo", que permite personalizar los mensajes que se envían a los usuarios (correos de validación del registro o de restablecimiento de la contraseña). 







**Groupware Usuario BaaS: Configuración necesaria**: 
Pueden presentarse dos casos: 

- **Si utiliza una plataforma PCSCloud**: los elementos necesarios se instalan de forma predeterminada en su plataforma. 
	Recordatorio: PCSCloud es una solución paga lista para usar. Para obtener más información, consulte [PCSCloud](https://pcscloud.net/UK/cloud_development.awp).

- **Si no utiliza una plataforma PCSCloud**: instale los siguientes elementos: 

	- un servidor HFSQL,

	- el servidor reservado, incluidos los módulos para el Groupware Usuario de dispositivos móviles (WDBaaS).


 Para obtener más información, consulte [Configuración de un servidor BaaS (Groupware Usuario y Tienda privada)](../Editeurs/1000021311.md).





### Modo de inicio
<a name="modo_inicio_ELTPARAGRAPHE000097"></a>

Existen varios modos de inicio: 

- La opción "automatic start" indica que el User Groupware se inicia al mismo tiempo que la aplicación o el sitio.
	La ventana o página de inicio de sesión del User Groupware se muestra antes que cualquier otra ventana/página de la aplicación. El código de inicialización del proyecto se ejecuta cuando se abre y valida la ventana/página de inicio de sesión.

- La opción "Inicio manual (mediante programación)" permite definir el momento en el que se debe iniciar el Groupware Usuario.
	La ventana o página de acceso sólo se abrirá si se utiliza [gpwOpen](../WDLang6/3041002.md) (o [gpwLogin](../WDLang6/3041001.md)). Esta opción se utiliza para ejecutar el código de inicialización del proyecto antes de abrir la ventana/página de inicio de sesión.. Esta opción es útil para una aplicación multilingüe, por ejemplo: Se puede mostrar una ventana o una página para solicitar el idioma de ejecución de la aplicación.

- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) La opción "Inicio manual a través de un control modelo" se utiliza para integrar la página de conexión de User Groupware en el sitio. Esto le permite trabajar con el programa de grupo de usuarios de automatic (y beneficiarse de sus actualizaciones), al tiempo que puede personalizar la página de conexión.
	Si se selecciona esta opción, el componente interno denominado **WDGPU_WBB_CNX** está incluido en su proyecto. Este componente contiene un control modelo ("TPLC_GPU_Connection") que se utiliza para gestionar la conexión. Simplemente incluya esta control modelo en las páginas (o plantillas) deseadas mediante arrastrar y soltar. 
	Después de la integración, tienes la posibilidad de personalizar este control modelo: 

	- sobrecargando los controles y/o los procesos para pequeñas modificaciones,

	- modificando el control modelo para modificaciones más importantes.

## Opciones en la pestaña "Opciones"
<a name="opciones_pestana_opciones_ELTTEXTE000465"></a>


### Personalización del software de grupo
<a name="personalizacion_del_software_grupo_ELTPARAGRAPHE000129"></a>

Hay varios modos de personalizar el User Groupware en su aplicación o sitio:

- **Sin personalización**: 
	WINDEV y WEBDEV se encargan de todo. Todo lo que tiene que hacer es iniciar la aplicación o el sitio para configurar el acceso de los usuarios. 
	![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Las ventanas del Groupware Usuario pueden utilizar automáticamente la plantilla de la aplicación. Simplemente seleccione la opción "Aplicar plantilla del proyecto a las ventanas no personalizadas durante la ejecución".
	Para que todas las ventanas de la aplicación tengan la misma apariencia, se recomienda marcar siempre esta opción si se utiliza el Groupware Usuario en modo no personalizado.
	

- **Personalizar UI de conexión**: 
	La ventana o página de inicio de sesión/registro se integra automáticamente en su aplicación.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) El componente interno **WDGPU** se incluye en el proyecto. Este componente contiene la ventana de inicio de sesión y el procedimiento asociado. Así, podrá personalizar la conexión al Groupware Usuario.




- **Personalización completa (modo experto)** - Solo Groupware Usuario estándar: 
	La programación del Groupware Usuario se incluye automáticamente en su aplicación.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) El componente interno **WDGPU** se incluye en el proyecto. Este componente contiene un análisis, así como todas las ventanas, consultas, clases y procedimientos necesarios para gestionar el Groupware Usuario. Esto permite personalizar todas las ventanas utilizadas por el Groupware Usuario.






<a name="NOTE3_4"></a>


### Seguridad
<a name="seguridad_ELTPARAGRAPHE000174"></a>

Las opciones de seguridad son las siguientes: 

- **Permitir la apertura de ventanas y reportes de forma predeterminada**: Esta opción permite que todos los usuarios abran ventanas y reportes de forma predeterminada. En este caso, al configurar los permisos, basta con especificar los elementos a los que no se podrá acceder. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Permitir la edición de reportes y consultas en "Reports and Queries"**: Si se selecciona esta opción, la aplicación podrá utilizar "Reports and Queries" para todos los reportes y consultas. 
	Observación: Esta opción es idéntica a "Habilitar 'Reports and Queries'", en la pestaña "Avanzado" de la descripción del proyecto. 



<a name="NOTE3_5"></a>


### Inicio de sesión automático en modo de prueba
<a name="inicio_sesion_automatico_modo_prueba_ELTPARAGRAPHE000188"></a>

Especifique el nombre de usuario y la contraseña que se utilizará en el modo [prueba automatizada](../Editeurs/2019025.md). Esta información se utilizará si se ejecuta una prueba automatizada en la aplicación.
<a name="NOTE3_6"></a>


### Idioma de los archivos
<a name="idioma_los_archivos_ELTPARAGRAPHE000198"></a>

Por Default, los archivos de datos específicos del User Groupware están en inglés, pero se pueden utilizar archivos de datos en francés. 
<a name="NOTE_3_7"></a>


### Contraseña de los archivos de datos
<a name="contrasena_los_archivos_datos_ELTPARAGRAPHE000205"></a>

Por Default, los archivos de datos que se encuentran en el Grupo de Usuarios tienen una contraseña específica. Esta contraseña es: ***"PCSGPW2001***".

Para cambiar esta contraseña, introduzca la nueva contraseña. Los caracteres escritos están ocultos.

**Observaciones**: 

- La nueva contraseña sólo se tiene en cuenta si se cambia antes de crear los archivos de datos del programa de grupo de usuarios. 

- Esta contraseña se utiliza cuando los archivos son re-indexados o cuando los archivos son abiertos por WDMAP.

## Pestaña "Archivos" (Grupo de usuarios estándar)
<a name="pestana_archivos_grupo_usuarios_estandar_ELTTEXTE000513"></a>


### Tipo de base de datos (solo en modo "Inicio automático")
<a name="tipo_base_datos_solo_modo_inicio_automatico_ELTPARAGRAPHE000221"></a>

El programa de grupo de usuarios maneja los archivos de datos utilizados para identificar a los usuarios y definir sus derechos. 

Estos archivos pueden ser:

- en formato HFSQL Classic (por Default). Se debe especificar el directorio del archivo de datos. 

- en formato HFSQL Client/Server. Se deben especificar las características de la conexión al servidor HFSQL así como el directorio de las diferentes categorías de archivos.




Importante: La información de localización se aplicará sólo en el modo "Inicio de automatic".. Si ha elegido el modo "Inicio manual (mediante programación)" para el programa de grupo, llame a [gpwOpen](../WDLang6/3041002.md) y [gpwOpenConnection](../WDLang6/3041007.md).
<a name="NOTE5_2"></a>


### Tipo de autenticación
<a name="tipo_autenticacion_ELTPARAGRAPHE000241"></a>

Existen diferentes tipos de autenticación: 

- Autenticación totalmente gestionada por el Groupware Usuario. 

- Autenticación a partir de un directorio LDAP o Active Directory. 
	Esta opción evita la duplicación de nombres de usuario y contraseñas utilizando los usuarios del directorio LDAP. En este caso, deben especificarse las características de la conexión (véase más abajo). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Uso de la autenticación de Windows. 
	Esta opción requiere que se creen los usuarios en la base de datos de User Groupware. Sólo el supervisor (contraseña de supervisor) es creado por Default. Durante la primera ejecución, se debe: 

	- conectar con el usuario supervisor. 

	- crear los usuarios que existen en Windows. 
			Observación: No es necesario especificar una contraseña para estos usuarios: su contraseña de Windows será comprobada para su autenticación.







**Para una autenticación a partir de un directorio LDAP o de un directorio Active**, debe configurar las características del servidor LDAP utilizado para las pruebas (y en tiempo de ejecución). La información solicitada es:

- Dirección del servidor

- Número de puerto del servidor.

- Versión del servidor LDAP contactado.

- Base de datos


En el código, basta con utilizar [gpwOpenLDAPSession](../WDLang6/1000017055.md).

Observaciones: 

- El programa de grupo de usuarios funciona con el directorio Active. No funciona con openLDAP.

- Cuando el puerto del servidor corresponde al puerto 636, la conexión se realiza automáticamente en modo "Negociar".



<a name="NOTE6_2"></a>


### Opciones
<a name="opciones_ELTPARAGRAPHE000280"></a>

Existen varias opciones en función del tipo de autenticación elegido: 

- **Ventana de inicio de sesión únicamente (sin gestión de permisos)**: Esta opción solo está disponible para la autenticación LDAP y Windows. 
	 En este caso, no se creará ningún archivo de groupware.. Si la aplicación se inicia en modo automático, el usuario deberá iniciar sesión. Si el usuario está guardado en el directorio LDAP o en Windows, la aplicación se iniciará; de lo contrario, se cerrará.. Un supervisor no puede configurar los derechos en las ventanas o páginas. Si esta opción no está marcada, la identificación se realiza con la gestión de derechos.: sólo los usuarios que se encuentren en el directorio LDAP podrán conectarse. Los derechos se pueden configurar en las ventanas o páginas.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Autenticación única (SSO)**: Esta opción permite al usuario identificarse una sola vez (en la primera ejecución de la aplicación).. Las demás aplicaciones que utilicen el User Groupware se conectarán automáticamente con el usuario especificado. Si se utiliza una autenticación de Windows con esta opción, los usuarios no tendrán que escribir ningún nombre de usuario.: la conexión es automatic. En este caso, para configurar el groupware, la sesión de Windows debe haber sido iniciada por un usuario que sea supervisor.

## Pestaña "OAuth" (Grupo de usuarios estándar)
<a name="pestana_oauth_grupo_usuarios_estandar_ELTTEXTE000549"></a>
En el Grupo de Usuarios "Estándar", la pestaña "OAuth" permite autorizar el uso de un servicio de conexión específico. Una vez seleccionado el servicio, se deben especificar los identificadores correspondientes (proporcionados al registrar la aplicación en el servicio).

<a name="NOTE7"></a>
<a name="NOTE7_1"></a>


## Pestaña "BaaS" (BaaS User Groupware)
<a name="pestana_baas_baas_user_groupware_ELTTEXTE000573"></a>
La pestaña "BaaS" permite definir todas las opciones necesarias para gestionar los derechos de un sitio específico:  

- En "GPU servidor de aplicaciones (servicio web y sitio de administración)", especifique el servidor a utilizar. En el caso de un nuevo servidor, seleccione el tipo de servidor a utilizar: 

	- **Plataforma PCSCloud**. En este caso, indique las características de su plataforma PCSCloud. 
			Recordatorio: Esta opción le permite utilizar un servidor web, un servidor dedicado y un motor HFSQL Client/Server ya instalado y configurado. 

	- Red **o servidor INTERNET** con el protocolo utilizado. Introduzca el nombre del servidor, usuario y contraseña. Esta información se le proporcionó durante la instalación del servidor reservado para el Grupo de Usuarios/Tienda Privada. 




- Especifique el tipo de cuentas de usuario gestionadas: 

	- **Cuentas compartidas entre aplicaciones**: Esta opción se utiliza para desplegar la aplicación con otras aplicaciones que serán accesibles a los mismos usuarios. Esta opción se recomienda para las implantaciones en diferentes empresas. 

	- **Cuentas específicas de la aplicación**: Esta opción se utiliza para almacenar la aplicación con su base de datos de usuarios. 
						Esta opción se recomienda para desplegar aplicaciones en un almacén.

## Pestaña " Email" (Programa de grupo de usuarios de BaaS)
<a name="pestana_email_programa_grupo_usuarios_baas_ELTTEXTE000597"></a>
La pestaña "Email" permite configurar las opciones de personalización de Email: 

- Validación de registro: Si es necesario, personalice los mensajes que se envían a los nuevos usuarios registrados. 

- Contraseña olvidada: Si es necesario, personalice los mensajes que se envían a los usuarios que olvidan su contraseña.

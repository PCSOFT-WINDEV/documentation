## Presentación
<a name="presentacion_ELTTEXTE000402"></a>
El programa de creación de instalación asistente tiene en cuenta la instalación de la base de datos con la aplicación y propone varios elementos opcionales. 

Los módulos adicionales para utilizar una base de datos permiten añadir opciones adicionales al programa instalación. El usuario podrá acceder a los parámetros avanzados de instalación. 

Es posible:

- [Configurar la sincronización de los datos desplegados (automatic modificación de los archivos de datos)](#NOTE2_1).

- [Indexar archivos HFSQL durante la instalación](#NOTE3_1).

- [Incluir la instalación de un servidor de la HFSQL Client/Server](#NOTE4_1).

- [Configurar la ubicación de los archivos de datos de la aplicación](#NOTE5_1).

- [Implementar los archivos de datos en un servidor HFSQL](#NOTE6_1).

- [Establecer la ubicación de los archivos del groupware durante la instalación](#NOTE7_1).

- [Incluir el MDAC en la instalación](#NOTE8_1).

## Configuración de la sincronización de los datos desplegados (modificación de los archivos de datos de automatic)
<a name="configuracion_sincronizacion_los_datos_desplegados_modificacion_los_archivos_datos_automatic_ELTTEXTE000432"></a>
Si la opción "Configurar la sincronización de los datos desplegados (auto modif.)" está marcada, la modificación de los datos automatic se realizará al instalar la aplicación actual.

La modificación de datos de la automatic se utiliza para actualizar la description de los archivos de datos encontrados en los ordenadores de los usuarios. Para obtener más información, consulte:

- [La modificación de los datos del automatic](../WDLang4/3044195.md).

- [Configuración de la modificación de los archivos de datos de automatic](../Editeurs/2028043.md).




**Observaciones**: 

- si esta opción está marcada, se realizará la modificación de los datos de la automatic (tanto si es necesaria como si no). La modificación de automatic no tendrá ningún efecto sobre la aplicación si se realiza sobre datos ya actualizados.

- Por defecto, se realiza una modificación "silenciosa" de los datos automatic.. Para que el usuario final pueda realizar cambios en la modificación de datos de automatic, es necesario utilizar el editor instalación .

## Indexación de ficheros HFSQL durante instalación
<a name="indexacion_ficheros_hfsql_durante_instalacion_ELTTEXTE000456"></a>
La opción "Indexar archivos HFSQL durante la instalación" elimina los archivos de índice del programa de instalación para reducir su tamaño. Estos archivos de índice se volverán a crear automáticamente en el ordenador del usuario final durante la aplicación instalación.

Tiene la posibilidad de seleccionar los archivos para los que deben eliminarse los archivos de índice.

**Observaciones**:

- Esta función no está disponible para los archivos de datos con contraseña protected.

- Esta función sólo está disponible para los archivos de datos de HFSQL Classic. Para los ficheros HFSQL Client/Server, la configuración se realiza a través de "Despliegue de ficheros de datos en un servidor HFSQL".

## Configuración de la ubicación de los archivos de datos de la aplicación
<a name="configuracion_ubicacion_los_archivos_datos_aplicacion_ELTTEXTE000504"></a>
La opción "Configurar la ubicación de los archivos de datos de la aplicación" permite definir la ubicación de los archivos de datos que se encuentran en la aplicación. 

Se enumeran las siguientes conexiones:

- La conexión llamada "***Ubicación predeterminada***": Esta conexión corresponde a la ubicación de los archivos HyperFileSQL Classic encontrados en el análisis y associated sin conexión. Por Default, los archivos de datos se crean en los directorios especificados al describir los archivos en el análisis.

- Las conexiones definidas en el análisis.




La información de conexión utilizada por la aplicación se encontrará en el archivo ".WX" que se encuentra junto al ejecutable.

El botón "Parámetros" permite configurar estas conexiones en caso necesario:  

- Si la opción "Todas las conexiones utilizan los mismos parámetros" está marcada, se puede configurar una única conexión.

- Si la opción "Cada conexión se configura de forma independiente" está marcada, todas las conexiones pueden configurarse una a una.




La pantalla para configurar la conexión le permite hacerlo:

- utilizar los valores dados en el análisis de proyecto.

- usar los valores dados en el archivo.WER (para una replicación universal).

- usar nuevos valores. En este caso, tiene la posibilidad de definir una nueva conexión con todas las características.

- utilizar los valores introducidos durante la instalación. En este caso, el usuario especificará la ubicación de los datos cuando se instale la aplicación.. Los valores de Default propuestos al usuario se pueden escribir en la pantalla de configuración.

- guardar los parámetros del servidor. Esta opción está disponible sólo si la instalación de un servidor HFSQL se incluyó en la aplicación instalación.



<a name="NOTE5_2"></a>


### Observaciones
<a name="observaciones_ELTPARAGRAPHE000125"></a>

- Si los parámetros de la conexión se modifican entre dos configuraciones, tiene la posibilidad de forzar los nuevos parámetros mediante "Los parámetros Above sustituirán a los valores introducidos durante un instalación anterior".

- El ***SetupFile*** Variable le permite usar el ruta de archivos de datos para archivos HFSQL Classic. El ruta utilizado es el definido en las propiedades de HFSQL Classic de una conexión a una base de datos HFSQL Classic (ISAM).

### Instalación de una aplicación que crea archivos de datos HFSQL Classic
<a name="instalacion_una_aplicacion_que_crea_archivos_datos_hfsql_classic_ELTPARAGRAPHE000133"></a>

Por Default, los archivos de datos se crean en los directorios especificados al describir los archivos en el análisis. Sin embargo, estos directorios pueden ser modificados:

- al crear el ejecutable de la aplicación.

- mediante la programación con [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md).

- al crear el programa de instalación (ajustes de la conexión "***Ubicación predeterminada***")

- durante la instalación. El directorio para crear los archivos de datos será seleccionado por el usuario.




Si su aplicación utiliza archivos de datos, puede ***permitir que el usuario set la ubicación de los archivos de datos*** al instalar la aplicación. El directorio seleccionado por el usuario se utilizará automáticamente para todos los ficheros de datos creados por la aplicación.

**Observación**: Este directorio será propuesto por Default al instalar las actualizaciones de la aplicación. Esta opción permite evitar la programación de la ubicación de los archivos de datos (con [HSubstDir](../WDLang4/3044028.md), por ejemplo).

**Recordatorio**: En cualquier caso, el directorio para crear los archivos de datos de HFSQL Classic se define según el siguiente orden de prioridad:

1. Directorio especificado por [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md).

2. Directorio seleccionado al instalar la aplicación (si esta opción se propone al usuario).

3. Directorio seleccionado al crear el ejecutable.

4. Directorio especificado al describir los archivos en el análisis.




Por lo tanto, el directorio definido por [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md) tiene prioridad sobre todos los demás directorios.

## Despliegue de los archivos de datos en un servidor HFSQL
<a name="despliegue_los_archivos_datos_servidor_hfsql_ELTTEXTE000546"></a>
Al crear el programa instalación de la aplicación, también puede desplegar los archivos de datos HFSQL Client/Server en el servidor deseado mediante "Desplegar los archivos de datos en un servidor HFSQL".

Para cada fichero de datos de la aplicación, el botón "Propiedades" permite especificar:

- la conexión que debe ser associated con este archivo de datos. 

- la ubicación actual del archivo de datos a instalar (ubicación del archivo de datos de origen).

- si el índice debe copiarse o no. Si no se copia el índice, se reconstruirá automáticamente durante la instalación. Esta opción se utiliza para limitar el tamaño del programa instalación cuando el archivo índice es bastante grande.




**Observación**: La ubicación actual de los archivos de datos puede corresponder a los archivos de datos encontrados en un servidor accesible o a los archivos de datos copiados localmente.

## Fijación de la ubicación de los archivos del programa de grupo durante instalación

### Grupo de usuarios después de la versión 18
<a name="grupo_usuarios_despues_version_18_ELTPARAGRAPHE000207"></a>

Si su aplicación utiliza el [Groupware Usuario](../Editeurs/1000021152.md), la opción "set la ubicación de los archivos de groupware durante instalación" le permite set la ubicación de los archivos de datos de User Groupware. 

Por Default, estos ficheros de datos se crean en el subdirectorio "GPW_" del directorio de instalación de la aplicación. Sin embargo, este directorio puede ser modificado. El tabla a continuación presenta los diferentes métodos propuestos por WINDEV para modificar este directorio.

**¿Cómo modificar el directorio de creación de estos archivos?**

- Al configurar el programa de grupo de usuarios (precaución: este directorio se usa sólo para las pruebas de la aplicación).

- Al crear el ejecutable de la aplicación.

- Al instalar la aplicación. Este directorio es seleccionado por el usuario.




**¿Cuál es el orden de prioridad al definir el directorio de creación de estos archivos?**

1. El directorio seleccionado por el usuario al instalar la aplicación (si el programa instalación permite al usuario seleccionar este directorio).

2. El directorio seleccionado al crear el ejecutable de la aplicación.

3. El directorio Default (subdirectorio "GPW_" del directorio de instalación de la aplicación).


Por lo tanto, el directorio seleccionado por el usuario tiene prioridad sobre los otros directorios.

Para permitir que los usuarios establezcan la ubicación de los archivos del Groupware Usuario, seleccione "Establecer la ubicación de los archivos del groupware durante la instalación".

## Incluir el MDAC en la instalación
<a name="incluir_mdac_instalacion_ELTTEXTE000600"></a>
Si su aplicación utiliza una base de datos a través de OLE DB, los siguientes elementos deben ser instalados en los ordenadores de los usuarios:

- Un proveedor OLE DB ("MSDAORA", "OraOLEDB.Oracle", "SQLOLEDB", ...).

- MDAC (Microsoft Data Access Component) versión 2.6 o posterior.




Si MDAC no está instalado, sólo será posible la conexión a la base de datos OLE DB.: no será posible leer las tablas (un mensaje indica que esta interfaz no es compatible).

**Atención**: La licencia de distribución de MDAC especifica que el ejecutable debe descargarse (del sitio web de Microsoft) para que get tenga los derechos de redistribuirlo con una aplicación. Antes de realizar las operaciones descritas en los siguientes párrafos, debe descargar el ejecutable de MDAC en la computadora actual.

El ejecutable MDAC puede descargarse del sitio web de Microsoft en la siguiente dirección (válida en el momento de redactar esta Document):: [http://www.microsoft.com/data](http://www.microsoft.com/data)

**Para distribuir MDAC a los usuarios:**

1. Seleccione la opción "Incluir el MDAC en la instalación".

2. Especificar la ruta del ejecutable del MDAC. Un enlace permite descargar el ejecutable MDAC del sitio web de Microsoft.




Observación: El tamaño del programa de la instalación se incrementará si se incluye el MDAC en la instalación. El programa instalación del MDAC será el que se encuentre en el ordenador de desarrollo.

## Instalación del controlador ODBC para HFSQL
<a name="instalacion_del_controlador_odbc_para_hfsql_ELTTEXTE000624"></a>
Para manipular una base de datos HFSQL de una aplicación no WINDEV, el controlador ODBC en HFSQL debe ser instalado y configurado en el ordenador del usuario. Para obtener más información, consulte [El controlador ODBC en HFSQL](../WDLang4/3044179.md).

**Observación**: El controlador ODBC en HFSQL solo se puede usar para **leer y escribir** en una base de datos HFSQL desde una aplicación que no sea WINDEV.

**Para incluir el controlador ODBC en el programa instalación**:

1. Marque "Instalar el controlador ODBC para HFSQL".

2. En los siguientes pasos de la asistente, introduzca el nombre utilizado para identificar la base de datos HFSQL cuando se utiliza el controlador ODBC en HFSQL.


## Configurando el servidor para el protocolo CGI/AWP
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000112"></a>
**Para usar WEBDEV, hay que encontrar un servidor web en el equipo donde está instalado WEBDEV.**

Diferentes servidores Web están disponibles para Windows: IIS, Apache, etc, ... 

El modo de configuración de estos servidores depende de:

- editor

- la versión del servidor,

- el sistema operativo,

- y otros parámetros.




El uso de un servidor Web para ejecutar la prueba de sitios WEBDEV dinámicos con la versión de desarrollo de WEBDEV requiere una configuración específica para el protocolo CGI.

Esta configuración se realiza al instalar WEBDEV. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Instalación de la versión de desarrollo de WEBDEV
<a name="instalacion_version_desarrollo_webdev_ELTTEXTE000136"></a>
Al instalar WEBDEV, el servidor Web para el protocolo CGI se configura automáticamente para los siguientes servidores Web:

- Apache 2.4.x.

- IIS versión 7.




Si su servidor se encuentra en esta lista y se produce un problema de configuración, el servidor web elegido al instalar la versión de desarrollo de WEBDEV no se corresponde con el servidor web utilizado.. En este caso, le recomendamos que reinstale WEBDEV.

Si su servidor no aparece en esta lista, el servidor Web debe estar configurado para el protocolo CGI.

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Configuración manual de servidores Web para WEBDEV
<a name="configuracion_manual_servidores_web_para_webdev_ELTTEXTE000160"></a>
Veamos la Procedure que sigue:

1. Cree un alias en el protocolo CGI de WEBDEV.

2. Este alias debe tener las siguientes características:




- denominar: WD28 0AWP

- Directorio asociado: del motor WEBDEV (Programas\\ENGINE\\WINDOWS\\AWP subdirectorio del directorio instalación de WEBDEV).

- Derechos de associated: Leer, Ejecutar

## Creación de las diferentes ventanas y páginas del patrón
<a name="creacion_las_diferentes_ventanas_paginas_del_patron_ELTTEXTE000180"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000082"></a>

Para averiguar todos los elementos que deben crearse para construir un patrón RAD para un proyecto completo, debe partir del análisis del proyecto: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RAD_Pattern.gif)


1. Primero, ocultemos el archivo de relación llamado RADFileD: Se deben crear las siguientes ventanas o páginas:

	- Un formulario para RADFileA.

	- Un formulario para RADFileB. Un cuadro combinado y un botón Vision Plus (o un cuadro combinado emergente) deben permitirle seleccionar un elemento de RADFileA.

	- Un formulario para RADFileC. Un cuadro combinado y un botón Vision Plus (o un cuadro combinado emergente) deben permitirle seleccionar un elemento de RADFileB.

	- Una tabla para RADFileA.

	- Una tabla para RADFileB.

	- Un tabla para RADFileC.




2. Ahora, vamos a esconder RADFileA: Se deben crear las siguientes ventanas o páginas:

	- Un formulario para RADFileB. Este formulario debe mostrar la relación con RADFileD. Por lo tanto, es una Forma + Tabla.  

	- Un formulario para RADFileC (este elemento ya fue mencionado).




3. A continuación, se deben crear los botones de enlace.

4. Ahora, todo lo que tenemos que hacer es crear el menú principal.

### Estándar a seguir
<a name="estandar_seguir_ELTPARAGRAPHE000105"></a>

Las diferentes páginas relativas a las ventanas y a las páginas de los patrones RAD presentan diferentes reglas que se pueden aplicar al crear las páginas y ventanas del patrón RAD. Estas reglas se han utilizado para crear los diferentes patrones suministrados con WINDEV, WEBDEV o WINDEV Mobile 16.

Estas reglas son recomendaciones. Puede implementar y utilizar su propio estándar.




## Descripción del componente interno
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000106"></a>
La descripción de un [componente interno](../Editeurs/2014100.md) se puede abrir en cualquier momento. Esta ventana contiene todas las opciones específicas del componente interno.

Para abrir la descripción del componente interno: 

1. Seleccione el componente interno en el panel "Explorador de proyectos".

2. Seleccione "Descripción del componente interno" en el menú contextual.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Detalles
<a name="detalles_ELTTEXTE000130"></a>
Las diferentes pestañas de la ventana de descripción son las siguientes:

- ***Pestaña "Descripción"***: Esta pestaña muestra el nombre, el título y la descripción del componente interno. Puede cambiar esta información directamente.

- ***Pestaña "Elementos"***: Esta pestaña muestra la lista de elementos del componente interno. Es posible:

	- agregar elementos al componente interno (botón "Agregar elementos"). Simplemente seleccione el elemento deseado en la lista de elementos del proyecto. 

	- eliminar elementos del componente interno. Simplemente seleccione los elementos que desea eliminar y haga clic en "Eliminar elementos". El elemento puede ser eliminado solo del componente interno o del componente interno y del proyecto.

	- cambiar elementos públicos a privados y viceversa. Simplemente seleccione los elementos que desea hacer públicos. 




- ***Pestaña "Análisis"***: Esta pestaña permite configurar el modo de gestión de los datos y de los contextos de ejecución. Existen tres modos para gestionar los datos y los contextos de ejecución:

	- **Utilizar análisis del proyecto o ningún análisis (compartir todo)**: El componente interno accede a los archivos de datos del proyecto. El componente interno y el proyecto utilizan los mismos contextos de ejecución. 
			Este es el modo predeterminado si el componente interno no utiliza ningún análisis.

	- **Utilizar análisis del proyecto con contextos de ejecución diferentes (modo avanzado)**: El componente interno accede a los archivos de datos del proyecto. El componente interno y el proyecto utilizan diferentes contextos de ejecución. 
			Este modo está reservado para casos específicos.

	- **Utilizar un análisis específico**: El componente interno accede a sus propios archivos de datos: es independiente. El componente interno y el proyecto utilizan diferentes contextos de ejecución. 
			Este es el modo predeterminado si el componente utiliza un análisis. 
			En este caso, se debe especificar el análisis utilizado por el componente interno y su contraseña (si la hay). También puede crear un nuevo análisis directamente o importar un análisis del SCM. Este análisis se asociará al componente interno.


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Observación: Si el componente interno incluye un análisis, éste será ignorado en tiempo de ejecución.

- ***Pestaña "Live Data"***: Esta pestaña solo está disponible si el componente interno tiene un análisis específico. Permite ver el contenido de los archivos de datos del componente interno directamente en el editor (Live Data).

- ***Pestaña "UML"***: Esta pestaña permite vincular un modelo UML al componente interno.

- ***Pestaña "Dependencias"***: Esta pestaña muestra la lista de dependencias asociadas al componente interno. 

- ***Pestaña "Compilación"***: Esta pestaña contiene las opciones para definir el estándar de programación del componente interno. Para obtener más información, consulte [Errores de estándar de programación](../Editeurs/9000187.md).

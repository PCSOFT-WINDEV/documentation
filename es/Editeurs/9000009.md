## Presentación
<a name="presentacion_ELTTEXTE000271"></a>
Cuando su aplicación Java está desarrollada, puede crear un archivo Java que puede ser ejecutado directamente. En este caso, hay varias soluciones:

- [generar una aplicación estándar de Java](#NOTE2_1).

- [generar una aplicación para Java Web Start](#NOTE4_1).




Para empezar a crear un archivo Java: 

- Visualizar la configuración del proyecto Java en el editor. 

- En la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar". 




**Observación**: La configuración actual del proyecto debe corresponder a la configuración del proyecto Java que se generará.

Observación: **Para Compile( en Java) un proyecto Java creado con WINDEV**, debe instalarse un kit de desarrollo Java en el ordenador de desarrollo (OpenJDK, por ejemplo).

## Generación de un archivo estándar Java
<a name="generacion_archivo_estandar_java_ELTTEXTE000295"></a>
Un archivo Java estándar permite iniciar la aplicación haciendo doble clic en el archivo .jar generado. También se genera un archivo.bat si el doble clic no funciona.
<a name="NOTE2_2"></a>
Para crear el archivo Java: 

1. Inicie el asistente para crear el archivo Java: 

	- Visualizar la configuración del proyecto Java en el editor. 

	- En la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar".




2. Seleccione "Generar una aplicación estándar de Java". Pase a la etapa siguiente.

3. Introduzca el nombre del archivo y la información que se encuentra en el manifiesto (Empresa, description y Copyright). Pase a la etapa siguiente.

4. Especifique el número de versión del ejecutable Java. Hay dos formatos disponibles para los números de versión:

	- **Formato estándar.** Este formato es el formato estándar gestionado por Windows. Este formato corresponde a una set de 4 números separados por puntos. 

		- El primer grupo de 4 dígitos corresponde al número de cambios importantes realizados en la aplicación.

		- El segundo grupo de 4 dígitos corresponde al número de cambios menores realizados en la aplicación.

		- El tercer grupo de 4 dígitos corresponde al número de generación. 
						Le aconsejamos que incremente automáticamente este número en cada generación. Esta operación de incremento se realiza automáticamente si se selecciona la opción "Incrementar automáticamente la versión en cada generación".

		- El último grupo de 4 dígitos corresponde al número de revisiones (ramas en la SCM por ejemplo).




- **Formato para la compatibilidad.** Este formato fue usado hasta la versión 12 por Default.
			**Observación**: Todos los caracteres (dígitos y letras) están permitidos. Por ejemplo: "1.01A".
			Si la opción "Incrementar automáticamente la versión en cada generación" está seleccionada:

	- el número de versión "1.01A" se convertirá en "1.01B".

	- el número de versión "A14Z" se convertirá en "A15A".

5. Especifique si el WINDEV framework para Java debe incluirse en el archivo .jar. Pase a la etapa siguiente.

6. Especifique los otros archivos que deben incluirse en el archivo ".jar. Puede seleccionar los controladores JDBC utilizados para acceder a una base de datos, por ejemplo. Pase a la etapa siguiente.

7. Especifique:

	- el comando del compilador de Java para usar. Para personalizar este parámetro (si se instalan varios compiladores en el equipo, por ejemplo), seleccione "Custom".. En esta pantalla, también puede especificar el comando Line que se utilizará (para más detalles, consulte la documentación sobre el compilador).

	- el comando para crear el archivo comprimido. La opción "Personalizado" permite definir la herramienta que se utilizará para crear el archivo.. También puede especificar el comando Line que se utilizará (para más detalles, consulte la documentación del compilador) y el contenido del manifiesto del archivo. 

	- el comando para incluir otros archivos. La opción "Personalizar" permite definir la herramienta que se utilizará para extraer el archivo así como su comando Line.




8. Se crea el archivo ".jar. 


El archivo correspondiente se creó en un subdirectorio correspondiente a la configuración del proyecto Java (subdirectorio del directorio EXE del proyecto). 
Para abrir un explorador de archivos en este directorio, en la pestaña "Inicio", en el grupo "General", expanda ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Rep_Projet.gif) y seleccione "Abrir directorio de generación". 

La ejecución del archivo Java se inicia utilizando el jvm del jdk utilizado para la compilación.

También encontrará un archivo ".BAT" en el directorio de generación del archivo Java. Este archivo contiene el comando Line necesario para iniciar la aplicación y puede ser ejecutado en un entorno Windows.

**Observación**: se pueden instalar varios jvm en el mismo ordenador (por ejemplo, JVM de 32 y 64 bits) mientras que el explorador asocia los archivos.jar con otro jvm. En este caso, ejecutar el archivo comprimido haciendo doble clic en el archivo.jar puede presentar algunas diferencias.

**Para los otros entornos**debe crear un archivo utilizado para iniciar el archivo ".jar".

Para Linux, puedes crear un archivo ".sh" que contiene el siguiente comando Line:

```txt
"java -jar MyProject.jar"
```
"MyProject.jar" corresponde al archivo ".jar" generado a partir de su proyecto.

**Observación**: Para obtener más información, consulte [Java y base de datos](../Editeurs/9000008.md)

## Generar una aplicación para Java Web Start
<a name="generar_una_aplicacion_para_java_web_start_ELTTEXTE000325"></a>
La aplicación Java se despliega en un servidor web y se inicia desde un navegador en los ordenadores cliente. Sin embargo, se ejecuta como cualquier aplicación estándar.

Cuando la aplicación se actualiza en el servidor web, también se actualiza en los ordenadores cliente.

Esta opción de generación simplifica la implementación y la actualización de una aplicación Java. 
<a name="NOTE4_2"></a>


### Para generar una aplicación para Java Web Start: 
<a name="para_generar_una_aplicacion_para_java_web_start_ELTPARAGRAPHE000145"></a>

1. Inicie el asistente para crear el archivo Java: 

	- Visualizar la configuración del proyecto Java en el editor. 

	- En la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar".




2. Seleccione "Generar una aplicación para Java Web Start". Pase a la etapa siguiente.

3. Introduzca el nombre del archivo y la información que se encuentra en el manifiesto (Empresa, description y Copyright). Pase a la etapa siguiente.

4. Especifique el número de versión del ejecutable Java. Hay dos formatos disponibles para los números de versión:

	- **Formato estándar.** Este formato es el formato estándar gestionado por Windows. Este formato corresponde a una set de 4 números separados por puntos. 

		- El primer grupo de 4 dígitos corresponde al número de cambios importantes realizados en la aplicación.

		- El segundo grupo de 4 dígitos corresponde al número de cambios menores realizados en la aplicación.

		- El tercer grupo de 4 dígitos corresponde al número de generación. 
						Le aconsejamos que incremente automáticamente este número en cada generación. Esta operación de incremento se realiza automáticamente si se selecciona la opción "Incrementar automáticamente la versión en cada generación".

		- El último grupo de 4 dígitos corresponde al número de revisiones (ramas en la SCM por ejemplo).




- **Formato para la compatibilidad.** Este formato fue usado hasta la versión 12 por Default.
			**Observación**: Todos los caracteres (dígitos y letras) están permitidos. Por ejemplo: "1.01A".
			Si la opción "Incrementar automáticamente la versión en cada generación" está seleccionada: 

	- el número de versión "1.01A" se convertirá en "1.01B".

	- el número de versión "A14Z" se convertirá en "A15A".

5. Especifique los otros archivos que deben incluirse en el archivo ".jar. Puede seleccionar los controladores JDBC utilizados para acceder a una base de datos, por ejemplo. Pase a la etapa siguiente.

6. Defina los parámetros de firma del archivo Java.

	- Archivo de claves.

	- Contraseña de associated.

	- Nombre de la clave en el archivo.

	- Contraseña de clave.




7. Defina los parámetros de despliegue de la aplicación Java Web Start:

	- URL de implementación.

	- Directorio correspondiente a la URL.




8. Especifique:

	- el comando del compilador de Java para usar. Para personalizar este parámetro (si se instalan varios compiladores en el equipo, por ejemplo), seleccione "Custom".. En esta pantalla, también puede especificar el comando Line que se utilizará (para más detalles, consulte la documentación sobre el compilador).

	- el comando para crear el archivo comprimido. La opción "Personalizado" permite definir la herramienta que se utilizará para crear el archivo.. También puede especificar el comando Line que se utilizará (para más detalles, consulte la documentación del compilador) y el contenido del manifiesto del archivo. 

	- el comando para incluir otros archivos. La opción "Personalizar" permite definir la herramienta que se utilizará para extraer el archivo así como su comando Line.

	- el comando para generar el fichero de archivo. La opción "Custom" permite definir las herramientas que se utilizarán para generar el fichero de claves.

	- el comando para firmar el archivo comprimido. La opción "Personalizar" permite definir la herramienta que se utilizará para firmar el archivo así como su comando Line.




9. La creación del applet Java se ha completado. 


El archivo correspondiente se creó en un subdirectorio correspondiente a la configuración del proyecto Java (subdirectorio del directorio EXE del proyecto).
Para abrir un explorador de archivos en este directorio, en la pestaña "Inicio", en el grupo "General", expanda ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Rep_Projet.gif) y seleccione "Abrir directorio de generación". 

Para ejecutar la prueba de este applet, inicie el HTML Page para la prueba. Este Page se utiliza para instalar la aplicación cliente Java.

**Observación**: Para obtener más información, consulte [Java y base de datos](../Editeurs/9000008.md).

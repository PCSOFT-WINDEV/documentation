
## Mostrar u ocultar eventos o procesos vacíos de un elemento
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000098"></a>
Todo el código de WLanguage está escrito en el editor de código, que ofrece varios eventos en un determinado control : Entrar, Modificación, Salir, Clic, ...

Siempre se propone una serie estándar de eventos para cada elemento: ventana, control, reporte, etc.

A partir de la versión 24, es posible ocultar eventos vacíos associated con un elemento. 

Esto permite un editor de código limpio, de acuerdo con sus hábitos de programación. 

Ejemplo: se han ocultado los eventos no utilizados. La presentación es mucho más clara.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Traitements_vides%20-%20HC%20N%B0001.gif&type=thumb)


Observación: También hay [eventos opcionales](../WDChamp/1014004.md) que puedes habilitar, por supuesto.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000122"></a>


### Mostrar u ocultar eventos vacíos
<a name="mostrar_ocultar_eventos_vacios_ELTPARAGRAPHE000031"></a>

Para ocultar eventos vacíos associated a un elemento en WINDEV, WEBDEV o WINDEV Mobile:

1. Abra uno de los códigos del elemento deseado (por ejemplo, a través de la tecla F2).

2. En la pestaña "Vista", en el grupo "Ayuda edición", marque o desmarque "Mostrar procesos vacíos".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Traitements_vides%20-%20HC%20N%B0002%201.gif&type=thumb)





Observación: Esta función también le permite mostrar u ocultar procedimientos y propiedades vacíos..

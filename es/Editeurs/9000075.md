


## Anomalías del proyecto: código muerto y elementos huérfanos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000181"></a>
Al desarrollar un proyecto, algunas secciones del código están escritas y ya no se utilizan.. Del mismo modo, se pueden crear ventanas o informes que ya no se utilizan.. 

WINDEV, WEBDEV y WINDEV Mobile le permiten detectar fácilmente estos "códigos muertos" y elementos "huérfanos".. 

La auditoría estática devuelve una reporte completa con respecto a los elementos que causan problemas en su proyecto.. 

También tiene la capacidad de identificar específicamente: 

- los procedimientos del proyecto que no se utilizan (código muerto).

- los procedimientos del elemento de corriente que no se utilizan (código muerto).

- los elementos del proyecto que no se utilizan (elementos huérfanos).

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000205"></a>


### Visualización del código muerto del proyecto
<a name="visualizacion_del_codigo_muerto_del_proyecto_ELTPARAGRAPHE000030"></a>

Para visualizar el código muerto del proyecto:

1. En el panel "", en el grupo "", amplíe "" y seleccione "Código muerto del proyecto".

2. Aparece la ventana que lista los códigos muertos de su proyecto.. 

3. En esta ventana, puede: 

	- Marque la Procedure como se usa (ver abajo). 

	- Borrar el Procedure. 

	- Vea el Procedure.

	- Iniciar búsqueda. 








### Visualización del código muerto de la etapa de corriente
<a name="visualizacion_del_codigo_muerto_etapa_corriente_ELTPARAGRAPHE000058"></a>

Para visualizar el código muerto de la etapa actual:

1. En el editor, muestre el elemento a estudiar: ventana, reporte, página,....

2. En el panel "", en el grupo "", expanda "" y seleccione "Código muerto del elemento actual".

3. Aparece la ventana que lista los códigos muertos de la unidad actual.. 

4. En esta ventana, puede: 

	- Marque la Procedure como se usa (ver abajo). 

	- Borrar el Procedure: Le permite borrar la Procedure. 

	- Ver el Procedure: Le permite mostrar el código de Procedure. También tienes la capacidad de hacer doble clic en el nombre de Procedure. 

	- Iniciar búsqueda: Le permite encontrar el Procedure en el proyecto actual. 








### Visualización de los elementos huérfanos del proyecto
<a name="visualizacion_los_elementos_huerfanos_del_proyecto_ELTPARAGRAPHE000085"></a>

Para mostrar los elementos huérfanos del proyecto:

1. En el panel "", en el grupo "", amplíe "" y seleccione "Elementos huérfanos".

2. Aparece la ventana utilizada para listar los elementos huérfanos de su proyecto.. 

3. En esta ventana, puede: 

	- Marque el elemento como se utiliza (ver abajo).

	- Eliminar elemento: Permite borrar el elemento. 

	- Ver el elemento: Permite ver el elemento. 

	- Iniciar búsqueda: Permite encontrar el elemento en el proyecto.

### Marcar una sección de código o un elemento como utilizado
<a name="marcar_una_seccion_codigo_elemento_como_utilizado_ELTPARAGRAPHE000113"></a>

**Atención**: En algunos casos, la auditoría de edición de un proyecto o elemento puede listar elementos que se consideran como código muerto o elementos huérfanos.. De hecho, estos elementos pueden corresponder a:

- procedimientos llamados en otro proyecto.

- procedimientos o elementos llamados en compilación dinámica.

- cuyo nombre se construye dinámicamente.




Estos elementos pueden ser marcados como "usados".. Por lo tanto, estos elementos ya no se mostrarán en la lista de elementos no verificados.. Estos elementos se mostrarán en la lista de "Procedimientos o elementos marcados como utilizados".

**Marcar un elemento como no utilizado**
Una ventana o una página de prueba encontrada en su proyecto puede ser analizada como "sin usar".. Para que este elemento se registre directamente en los elementos marcados como no utilizados, basta con especificarlo en el "Código de declaración de variables globales" del elemento mediante la siguiente sintaxis: 


```txt
<useful = "Reason">
```
donde "Reason" corresponde al texto que indica al desarrollador por qué el elemento se encuentra en la lista de elementos marcados como no utilizados.. Este texto se mostrará en la columna "Comentario" de la ventana de gestión de los elementos huérfanos..  

Por ejemplo: 


```txt
<useful = "Window for internal tests">
```

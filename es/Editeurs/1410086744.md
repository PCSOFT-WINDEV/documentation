


## RGPD: una auditoría de los datos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000195"></a>
Se dispone de una auditoría de la GDPR para identificar y supervisar fácilmente los datos personales en una aplicación WINDEV, WEBDEV o WINDEV Mobile.

Para obtener más información, consulte [RGPD: Soluciones concretas](../Editeurs/1410086740.md).

### Definición de elementos con datos personales
<a name="definicion_elementos_con_datos_personales_ELTPARAGRAPHE000041"></a>

Los elementos con datos relevantes para el GDPR pueden definirse en el sistema editor de análisis. 

Para definir los datos personales: 

1. Abrir el análisis associated con el proyecto.

2. Si un elemento del fichero corresponde a datos personales: 

	- Seleccione el archivo en el gráfico de análisis (o en el panel "Análisis"). 

	- Abra la ventana description de las partidas (seleccione "description de las partidas" en el menú contextual del fichero). 

	- En la tabla que presenta la lista de artículos, la marca de verificación "GDPR" sirve para indicar si un artículo contiene datos personales: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RGPD_Audit%20-%20HC%20N%B0002.gif&type=thumb)


	- Marque "GDPR" para los elementos pertinentes. 

- Valide.

3. Generar el análisis. 




**Observaciones**: 

- Si "Activar la opción "datos personales" al crear los campos" está activada en la ventana el archivo de datos description (pestaña "Opciones"), la opción "GDPR" se activará para cualquier nuevo elemento en el archivo de datos. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RGPD_Audit%20-%20HC%20N%B0001.gif&type=thumb)
 La opción "GDPR" puede desmarcarse en cualquier momento a nivel de artículo si no es necesaria.. 

- La Property RGPD se utiliza para determinar si la opción "RGPD" está marcada para un artículo. 




**Atención**: La aplicación de la opción "GDPR" en el análisis lo hace incompatible con las versiones anteriores.. Esta funcionalidad está disponible a partir de la versión 23 Update 2. Si el análisis se comparte entre varios proyectos, una vez implementada la opción "GDPE", este análisis sólo podrá ser abierto por un producto de la versión 23 o posterior.

### Inicio de la auditoría GDPR
<a name="inicio_auditoria_gdpr_ELTPARAGRAPHE000076"></a>

Para iniciar la auditoría GDPR en la configuración actual del proyecto: 

1. En la pestaña "Proyecto", en el grupo "Auditoría y rendimiento", despliegue "Auditoría estática" y seleccione "Auditoría de datos personales (RGPD)". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RGPD_Audit%20-%20HC%20N%B0003.gif)
 **Observación**: La auditoría del GDPR también puede iniciarse desde el editor de análisis: en la pestaña "Análisis", en el grupo "Análisis", haga clic en "Auditoría RGPD". 

2. La ventana que se muestra incluye varias pestañas: 

	- Auditoría por fuente de datos: Esta vista se utiliza para encontrar todos los datos afectados por GDPR, así como consejos (requisitos de cifrado, visibilidad, etc.).. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RGPD_Audit%20-%20HC%20N%B0004.gif&type=thumb)


	- Auditoría por elemento de proyecto: Esta vista se utiliza para verificar los datos personales a los que accede un elemento (una ventana, por ejemplo).. Esto le permite verificar, si es necesario, el requisito de acceso y realizar las modificaciones (eliminación o control del acceso, por ejemplo. También se enumeran los elementos basados en una consulta que contiene datos afectados por el GDPR. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RGPD_Audit%20-%20HC%20N%B0006.gif&type=thumb)




**Atención**: La auditoría de GDPR enumera los usos de los datos personales para el proyecto actual (y la configuración del proyecto), relacionado al análisis para el que se definieron los datos de GDPR.. Si el análisis se comparte entre varios proyectos, la auditoría debe realizarse en cada uno de ellos.. 

Recordatorio: La auditoría GDPR sirve para averiguar el uso de los datos "personales" definidos en el análisis. Para cumplir con la normativa, los datos personales también deben ser protegidos. Puede, por ejemplo: 

- Cifrar los datos (para las bases de datos HFSQL, todo lo que tienes que hacer es especificarlo en el editor de análisis).

- Proteger las bases de datos mediante identificadores: todo lo que tienes que hacer es definir usuarios/contraseñas desde el Centro control de HFSQL.

- Proteger la apertura de los archivos de datos mediante contraseña ([HPass](../WDLang4/3044108.md) en bases de datos HFSQL).

- Cifre las comunicaciones (HFSQL le permite restringir la apertura de conexiones a conexiones cifradas) y utilice SSL (https).

- etc.

### Implementación de los registros de las actividades de tratamiento e impresión de la carpeta
<a name="implementacion_los_registros_las_actividades_tratamiento_impresion_carpeta_ELTPARAGRAPHE000130"></a>

En la auditoría GDPR, la pestaña "Información adicional" se utiliza para: 

- Crear o incluir una Record de actividades de procesamiento.
	Para ello, despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RGPD_Audit%20-%20HC%20N%B0008%201.gif) y seleccione la opción deseada: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RGPD_Audit%20-%20HC%20N%B0007.gif)
 Puede: 

	- seleccionar un registro de actividades de tratamiento existente (opción "Examinar"). Se pueden utilizar todos los formatos de archivo. Los archivos en formato docx y xlsx se abrirán automáticamente y se incluirán en la carpeta impresa de GDPR.. 

	- crear un Record de actividades de procesamiento (de un modelo o de un Document vacío). 




- Pr&int la documentación del GDPR. 
	Sólo tiene que hacer clic en el botón "Imprimir la carpeta".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RGPD_Audit%20-%20HC%20N%B0009.gif&type=thumb)
Observación: Si se especificó una Record de actividades de procesamiento, se incluye en la carpeta impresa. Esta carpeta presenta el description y el uso de elementos que contienen datos personales en el proyecto.

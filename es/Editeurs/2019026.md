


## Pruebas automatizadas de la ventana
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000126"></a>
Las pruebas automatizadas de la ventana se utilizan para comprobar las diferentes características propuestas por la ventana.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo crear la prueba?
<a name="¿como_crear_prueba_ELTTEXTE000150"></a>


### Creación de una prueba automatizada en una ventana
<a name="creacion_una_prueba_automatizada_una_ventana_ELTPARAGRAPHE000018"></a>

Para crear una prueba automatizada en una ventana: 

1. Abra la ventana cuya prueba debe ejecutarse en el editor de ventanas.

2. En la pestaña "Pruebas automatizadas", en el grupo "Pruebas", despliegue "Nuevo" y seleccione "Grabar un nuevo escenario".

	- Aparece un cuadro de diálogo de información que indica que la ventana se ejecutará en modo de prueba y que todas las operaciones realizadas a partir de ahora se registrarán en WLanguage en el escenario de prueba.. 

	- Haga clic en el botón "Iniciar grabación" para validar.
			**Observación**: Si la ventana espera parámetros, escriba los valores de prueba de la ventana.




3. Realice las operaciones necesarias en su ventana.

4. Para detener la prueba, cierre la ventana.

5. Al final de la prueba, se muestra la ventana de prueba description. Tiene la posibilidad de especificar el nombre de la prueba y su description (pestaña "General"). La información disponible en las otras pestañas es especificada por Default. Esta información puede ser modificada en cualquier momento.

6. El código correspondiente a la prueba se muestra en el editor de códigos.



<a name="NOTE2_3"></a>


### Observaciones
<a name="observaciones_ELTPARAGRAPHE000049"></a>

- Si se abre una ventana desde la Event "Inicializando" del proyecto, también se crea una prueba para esta ventana. La "prueba automatizada" del proyecto Event se modifica para tener en cuenta la prueba de esta ventana.

- Si su aplicación utiliza el groupware usuario:

	- en modo manual: si se utiliza [gpwOpen](../WDLang6/3041002.md), esta función debe sustituirse por [gpwLogin](../WDLang6/3041001.md) en el código de prueba.

	- en modo automatic: en la ventana de configuración de groupware usuario, especifique el nombre de usuario y la contraseña que se utilizarán para el modo de prueba de automatic.




- Si la ventana cuya prueba se está ejecutando abre otra ventana (o una ventana interna), las operaciones realizadas en las diferentes ventanas se registran en el mismo escenario.

- Si la ventana abre otra ventana que se encuentra en un componente, las operaciones realizadas en la ventana del componente también se registrarán en el mismo escenario.




**Atención**: No se puede crear una prueba automatizada si se abre una ventana procedente de un componente en el código de inicialización del proyecto.
<a name="NOTE2_4"></a>


### Límites
<a name="limites_ELTPARAGRAPHE000070"></a>

- No se puede hacer ninguna prueba con el control Carrusel.

- No se puede ejecutar ninguna prueba en las ventanas que WINDEV muestra automáticamente (ventanas para la gestión de errores HFSQL, ventanas de impresión, etc.).. Si estas ventanas se incluyen en el proyecto para ser personalizadas, se puede ejecutar la prueba de automatic.

- La operación utilizada para arrastrar y soltar un control a otro no puede ser reproducida en una prueba de automatic.

- La grabación de un escenario ignora la persistencia de los controles. De hecho, de no ser así, el contexto y, por tanto, el resultado podrían ser diferentes entre dos pruebas superadas en dos momentos diferentes.

- Cuando se ejecuta una prueba automatizada en un organizador control, la selección del periodo con el ratón debe realizarse sin desplazamiento. 

- No se puede ejecutar el test de una impresión mediante la presentación preliminar.. El código generado automáticamente para gestionar la ventana de "Previsión" debe ser eliminado del código de la prueba.

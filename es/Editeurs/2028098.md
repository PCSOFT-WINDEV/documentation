


## Configuraciones teniendo en cuenta el Push
			





## Presentación
<a name="presentacion_ELTTEXTE000157"></a>
La instalación teniendo en cuenta el Push se utiliza para desplegar una aplicación en todos los ordenadores clientes de una red desde un único ordenador de gestión.

Este modo de instalación es una opción de la instalación en red local con actualización de automatic (Live Update). Para obtener más información, consulte: [Editor de instalaciones WDInst](../Editeurs/2028042.md).



## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000181"></a>


### Creando la instalación en modo Push
<a name="creando_instalacion_modo_push_ELTPARAGRAPHE000017"></a>

Para crear una red instalación teniendo en cuenta el Push:

1. Iniciar asistente de creación de instalaciones: en la pestaña "Proyecto", en el grupo "Generación", despliegue "Procedimiento de instalación" y seleccione "Crear procedimiento de instalación".

2. Seleccione "Configuración en red local (LAN) teniendo en cuenta el Push" en la creación de instalación asistente.

3. Generar la instalación

4. Realice la reference instalación de la red instalación (ejecutando Install_Server.exe).

5. Al final de la reference instalación, la instalación asistente permite iniciar "WDAdminExe". Esta herramienta se utiliza para realizar la instalación de los ordenadores clientes en Push. Marque la casilla "Start WDAdminEXE" para iniciarla.. 


**Observación**: "WDAdminExex" se encuentra en el directorio "INSTALL" de la reference instalación de la aplicación. WDAdminExe puede ser redistribuido.  


### Implementación en Push en los equipos cliente
<a name="implementacion_push_los_equipos_cliente_ELTPARAGRAPHE000044"></a>

Para desplegar en empuje en los equipos cliente:

1. Iniciar WDAdminExe (ver Above).

2. Mostrar la pestaña "Push" de la ventana principal: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdminExe_2%20-%20HC%20N%B0002.gif&type=thumb)


3. En "Lista de ordenadores" (1), introduzca el nombre de usuario y la contraseña de una cuenta de Windows con derechos suficientes para enumerar los ordenadores de la red.
	**Observaciones**:

	- En un directorio Active, el nombre de usuario debe tener uno de los siguientes formatos: DOMINIO\\USUARIO y/o USUARIO@DOMINIO.

	- Esta cuenta no tiene que ser un administrador.

	- Tiene la capacidad de almacenar la información de conexión. Si se utiliza esta opción, es importante no almacenar la contraseña de un administrador.

	- También tiene la posibilidad de cargar una lista de equipos desde un archivo de texto.. El archivo de texto puede contener: 

		- nombres de ordenador o direcciones IP.

		- comment s en formato "// comment". 


Cada línea debe contener un nombre (o dirección IP) y un comment. Por ejemplo: 
			
		```txt
		VIB //Vincent's computer
		FP //Florence's computer
		```


4. Haga clic en "Listar los ordenadores" para validar la información de conexión y empezar a enumerar los ordenadores cliente.
	**Atención**: dependiendo de las políticas LDAP definidas en el Directorio Active, la importación puede estar limitada a 1.000 usuarios. En este caso, para eliminar esta limitación, es necesario modificar el parámetro LDAP MaxPageSize. Para obtener más información, consulte [https://support.microsoft.com/kb/315071](https://support.microsoft.com/kb/315071). 

5. Los criterios (2) y la lista de grupos (3) pueden utilizarse para filtrar los ordenadores para mostrar. La lista de grupos está disponible sólo en un dominio del Directorio Active.

6. El tabla de los ordenadores (4) muestra: 

	- los ordenadores cliente encontrados,

	- la comment si se encuentra un archivo de texto.

	- el número de versión de la aplicación si ya está instalada,

	- el resultado de las configuraciones Push anteriores.




7. Seleccione los equipos cliente en los que debe desplegarse la aplicación en modo Push.

8. Haga clic en "Iniciar el PUSH instalación..." para iniciar el instalación.
	La ventana para iniciar la instalación se muestra: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdminExe_2%20-%20HC%20N%B0003.gif&type=thumb)


9. Si se encuentran varias versiones de la aplicación en la reference instalación, seleccione la versión que debe ser desplegada en el Push (1).

10. El resumen (2) muestra la lista de ordenadores objetivo de la instalación. Algunos ordenadores pueden ser eliminados de esta lista a través de el menú contextual.

11. Si es necesario, puede utilizar una cuenta de Windows distinta de la que se utilizó para enumerar los equipos (3).

12. Puedes programar el Push instalación para que se realice después (4). Esta opción le permite actualizar automáticamente una set de ordenadores clientes fuera de las horas de trabajo por ejemplo.

13. La validación de esta ventana inicia las configuraciones Push. El progreso puede ser monitoreado en el tabla de las computadoras de la ventana principal de WDADMINEXE.




**Observación**: Un botón "Advanced" (Avanzado) se utiliza para configurar el tiempo de espera para las configuraciones Push.. Si varias configuraciones fallan con errores de tiempo de espera, es posible que tenga que ajustar este parámetro.


### Servicios requeridos en los ordenadores de destino
<a name="servicios_requeridos_los_ordenadores_destino_ELTPARAGRAPHE000096"></a>

**Observación**: Dependiendo de las versiones de Windows utilizadas, los nombres de estos servicios pueden cambiar en el panel de control.

Los siguientes servicios deben ser iniciados en las computadoras de destino de la instalación en modo Push:

- Compartir archivos (servicio"lanmanserver")

- Acceso al registro remoto (servicio"RemoteRegistry")




Además, la parte administrativa (ADMIN$) debe existir. Este recurso compartido se puede ver en la herramienta de gestión denominada "Gestión informática", en "Herramientas del sistema... Carpetas compartidas... Acciones".


### Servicios requeridos en el equipo que realiza el Push instalación
<a name="servicios_requeridos_equipo_que_realiza_push_instalacion_ELTPARAGRAPHE000108"></a>

El servicio de "lanmanworkstation" debe ser iniciado en el equipo que realiza de la instalación en modo Push:.

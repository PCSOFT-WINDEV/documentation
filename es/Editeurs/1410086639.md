
## WEBDEV Cluster: Configuración de Proxy

## Presentación
<a name="presentacion_ELTTEXTE000100"></a>
Para utilizar un cluster WEBDEV, es necesario distribuir la carga entre los diferentes nodos del cluster (cada nodo corresponde a un servidor de aplicaciones WEBDEV). Para ello, debe instalar y utilizar un proxy o un equilibrador de carga.

Este servidor proxy (o equilibrador de carga) debe implementarse antes de que los nodos del clúster. Actuará como puerta de enlace entre INTERNET y el clúster WEBDEV. Este servidor llevará a los usuarios a los sitios web en diferentes nodos del clúster.

Observaciones: 

- Esta Document no explica cómo instalar y configurar un proxy.  

- Las plataformas PCSCloud permiten utilizar clusters WEBDEV. Un proxy está disponible y está listo para ser utilizado con los clusters de WEBDEV.

## Proxy y seguridad
<a name="proxy_seguridad_ELTTEXTE000148"></a>
El proxy se conecta a los nodos del cluster WEBDEV a través del puerto 80. 

Mediante Default, los nombres de usuario y las contraseñas se envían a los nodos del clúster en un formato legible. 

El nivel de seguridad dependerá de su infraestructura: 

- Si utiliza una red privada (LAN, VPN), los riesgos son muy bajos porque la red no está conectada a INTERNET. 

- Si utiliza nodos públicos (porque tiene varios proveedores, por ejemplo), existen diferentes métodos para asegurar el acceso y los datos: 

	- Instalar un proxy en cada nodo y utilizar el filtrado IP Address. Se recomienda este método. 

	- Instalar un TLS Certificate en cada nodo, para cifrar los intercambios de datos. Esta solución requiere un Certificate diferente en cada nodo, ya que el primero es associated con el segundo.

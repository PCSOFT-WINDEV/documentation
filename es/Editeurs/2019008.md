
## Diferencias según el modo de prueba de un proyecto dinámico
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000122"></a>
Se puede realizar la prueba de un proyecto dinámico:

- en el editor WEBDEV. Para obtener más información, consulte [Ejecución de la prueba de un proyecto dinámico](../Editeurs/2019009.md).

- desde el administrador WEBDEV. Para obtener más información, consulte [Probar un proyecto dinámico (administrador)](../Editeurs/2019007.md)




Se observan varias diferencias entre estos dos métodos de prueba.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Las diferencias
<a name="las_diferencias_ELTTEXTE000146"></a>


### Gestión de cookies
<a name="gestion_cookies_ELTPARAGRAPHE000028"></a>

La gestión de cookies es operativa cuando se ejecuta una prueba desde la página de prueba del administrador.. 
<a name="NOTE2_2"></a>


### Derechos concedidos en el servidor
<a name="derechos_concedidos_servidor_ELTPARAGRAPHE000036"></a>

- Cuando la prueba se ejecuta desde el editor, los derechos del sitio WEBDEV corresponden a los derechos del usuario que ejecuta la prueba del sitio.

- Cuando la prueba se ejecuta desde la página de prueba, los derechos utilizados corresponden a los derechos del usuario configurados en el servidor web.. Estos derechos pueden ser más restrictivos: algunas acciones pueden no estar permitidas (borrar archivos del servidor, ....). 
	



<a name="NOTE2_3"></a>


### Gestión de los errores de programación
<a name="gestion_los_errores_programacion_ELTPARAGRAPHE000045"></a>

- Cuando se ejecuta la prueba desde el editor, los errores de programación se muestran en un cuadro de diálogo y el depurador se inicia automáticamente.

- Cuando la prueba se ejecuta desde la página de prueba del administrador, los errores de programación se muestran en una página del navegador.. Esta página de error contiene el mensaje: "El siguiente mensaje fue generado por la aplicación: ...". 
	




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Notas
<a name="notas_ELTTEXTE000182"></a>


### Navegador de prueba
<a name="navegador_prueba_ELTPARAGRAPHE000056"></a>

El navegador de prueba es el navegador definido por Default en el ordenador actual.

Le aconsejamos que realice la prueba del proyecto con varios navegadores.
<a name="NOTE3_2"></a>


### Comenzar desde el administrador
<a name="comenzar_desde_administrador_ELTPARAGRAPHE000065"></a>

El uso de la prueba de sitio desde la página de prueba iniciada por el administrador de WEBDEV le permite ejecutar la prueba del sitio WEBDEV en configuración real.
<a name="NOTE3_3"></a>


### Motor de prueba y motor implementación
<a name="motor_prueba_motor_implementacion_ELTPARAGRAPHE000072"></a>

El modo de funcionamiento del motor de prueba de WEBDEV (proporcionado con la "Versión de Desarrollo" de WEBDEV) y el modo de funcionamiento del motor de WEBDEV instalado en el host (proporcionado con la "Versión de Despliegue" de WEBDEV) pueden diferir ligeramente. Las posibles diferencias en el modo de funcionamiento son menores, debido principalmente a la depuración permitida por el motor de prueba..

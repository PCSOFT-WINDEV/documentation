


## Centros de Control: Trabajar sin conexión
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000136"></a>
Cuando se trabaja en un ordenador portátil o en un ordenador sin conexión, se puede seguir trabajando con los datos en los Centros control (Centro de Seguimiento de Proyectos, etc.).

WINDEV, WEBDEV y WINDEV Mobile permiten gestionar un modo "Offline" para los Centros control. Este modo se utiliza para desconectarse de la base de datos de los Centros control y copy esta base de datos localmente en el equipo. Sus datos siguen siendo accesibles incluso en modo offline.

Durante la reconexión, los datos modificados localmente se actualizan automáticamente en la base de datos de la red.. Si se produce un conflicto (tarea modificada por varios usuarios, por ejemplo), la base de datos de la red tiene prioridad sobre los otros equipos.

**Observación**: En el modo offline se pueden utilizar las siguientes herramientas: Centro de Seguimiento de Proyectos, Centro de Control de Calidad y Versiones, Mensajería. 

**Atención: el modo offline no está disponible si se utilizan los Centros control con una base de datos HFSQL Client/Server.**



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Trabajar sin conexión
<a name="trabajar_sin_conexion_ELTTEXTE000166"></a>
Para trabajar en modo "Offline: 

1. En la pestaña "SCM", en el grupo "Otras acciones", despliegue "Trabajo remoto" y seleccione "Desconectarse para trabajar sin conexión".

2. En la asistente, seleccione "Quiero desconectarme de la base de datos compartida del centro de la control".
	Atención: Esta asistente también gestiona la desconexión del repositorio del proyecto actual. Para obtener más información, consulte [SCM: Trabajar sin conexión](../Editeurs/2038013.md).

3. Seleccione las diferentes opciones para la base de datos local:

	- Los datos adicionales a añadir

	- La ruta de la base de datos (le aconsejamos que conserve la ruta propuesta).




4. Ahora estás usando la base de datos local para todos tus centros de control.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Reconectando
<a name="reconectando_ELTTEXTE000190"></a>
Para reconectarse a la base de datos de los centros de control:

1. En la pestaña "SCM", en el grupo "Otras acciones", despliegue "Trabajo remoto" y seleccione "Volver a conectarse y sincronizar".

2. En el asistente, seleccione "Quiero volver a conectarme a la base de datos compartida del centro control".
	Atención: Esta asistente también gestiona la reconexión al repositorio del proyecto actual. Para obtener más información, consulte [SCM: Trabajar sin conexión](../Editeurs/2038013.md).

3. Seleccione las diferentes opciones:

	- Base de datos compartida que se utilizará: la base de datos compartida que se utilizó anteriormente se propone automáticamente.

	- La ruta de la base de datos local.
			Si los archivos han sido borrados de la base de datos local, marque "He perdido mi base de datos local".. En este caso, volverá a conectarse a la base de datos compartida, pero no se realizará ninguna sincronización.




4. Ahora está de nuevo conectado a la base de datos compartida.




Durante la reconexión, los datos modificados localmente se actualizan automáticamente en la base de datos compartida.. Si se produce un conflicto (tarea modificada por varios usuarios, por ejemplo), la base de datos de la red tiene prioridad sobre los otros equipos.




## Sangrar el código
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000188"></a>
Al escribir sentencias compuestas ("Switch", "For" y "While", por ejemplo), el código se indenta automáticamente para resaltar la estructura de los bucles utilizados en el programa.

En cuanto se escribe la primera sentencia, se inserta automáticamente la palabra clave "END" y el cursor se sitúa en la sentencia compuesta.

La indentación de automatic también permite alinear por el signo "=" y la palabra "es"..

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000212"></a>


### Sangrado de un código existente
<a name="sangrado_codigo_existente_ELTPARAGRAPHE000024"></a>

Se pueden utilizar varias opciones para sangrar un código existente o el código que se está escribiendo:

- Para sangrar la Line actual del código **a la izquierda** vaya a la pestaña "Código", grupo "Edición" y haga clic en "Disminuir sangría".

- Para sangrar la Line actual del código **a la derecha** vaya a la pestaña "Código", grupo "Edición" y haga clic en "Aumentar sangría".

- Para que **automáticamente la sangría de todo el código**, en la pestaña "Código", en el grupo "Edición", haga clic en "Sangría auto." (o pulse Ctrl + R). 




Para configurar las opciones de reindentación de la automatic: 

- Posicione el cursor en el editor de código (pero no en una Line que contenga código de WLanguage). 

- En la pestaña "Código", en el grupo "Edición", despliegue "Sangría auto." (o pulse las teclas Ctrl + R). 

- Una ventana permite configurar el alcance de la sangría de automatic. El alcance puede corresponder a:

	- **código actual**: sólo la Process que se está editando (donde se encuentra el cursor) está reindicada.

	- **elemento actual**: todos los eventos associated con el elemento actual en el editor de código son reindentados (por ejemplo, todos los códigos de la ventana, controles y menús associated con la ventana).

	- **todo el código que se muestra en la ventana de código activa**: todos los eventos o procesos que se muestran actualmente en la ventana del código serán reindicados.

	- **todo el código WLanguage del proyecto**: todos los eventos de todos los elementos del proyecto.




- La opción "Alinear por '=', 'is', '//'" permite alinear todos los textos y códigos que se encuentran a la derecha de los símbolos '=', 'is' o '//'. 
	Esta opción se tendrá en cuenta como aliada de automatic para todas las reindentaciones posteriores de automatic con Ctrl + R.

### Configurando la reindentación de automatic de un código pegado
<a name="configurando_reindentacion_automatic_codigo_pegado_ELTPARAGRAPHE000119"></a>

Para reindentar automáticamente el código durante una copy/pegado:

1. Abrir las opciones del editor de código: en la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones del editor de código".

2. En la pestaña "Código", seleccione la opción "Sangría (el formato y la sangría del código pegado se modifican para ajustarse al diseño de la página)".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Coller-Code.gif)

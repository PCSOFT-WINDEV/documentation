
## Docker Image del servidor de aplicaciones WEBDEV
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000150"></a>
A partir de la versión 24, se puede encontrar un contenedor Docker con el Servidor de Aplicaciones WEBDEV en Docker Hub:
[https://hub.docker.com/r/windev/webdev](https://hub.docker.com/r/windev/webdev)


La instalación de un servidor de Aplicación WEBDEV para Linux se vuelve extremadamente simple: ¡Una Line para escribir a máquina o con guión!
Dependiendo del tipo de servidor (10 conexiones simultáneas o ilimitadas), se le pedirá el número de serie del servidor.

Observación: Sólo está disponible la versión de 64 bits. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000174"></a>


### Configuración necesaria
<a name="configuracion_necesaria_ELTPARAGRAPHE000027"></a>

- El sitio debe ser [adecuado para Linux implementación](../Editeurs/9000055.md). La opción "Habilitar errores de compilación para compatibilidad con Linux" debe estar habilitada.. Esta opción está disponible en la pestaña "Compatibilidad", en la ventana description de la configuración actual.

- La distribución de Linux utilizada no importa. Sin embargo, para utilizar el contenedor Docker que contiene el Servidor de Aplicaciones WEBDEV, es necesario estar familiarizado con Docker y, en particular:

	- saber cómo instalar un nuevo contenedor desde el Docker Image del Servidor de Aplicaciones WEBDEV,

	- gestionar al menos un volumen necesario para la persistencia de los datos,

	- gestionar la configuración de red dentro de Docker. Por ejemplo, debe poder manipular el caso en que uno o varios contenedores con el Servidor de Aplicaciones WEBDEV Image necesiten acceder a los datos alojados en un servidor HFSQL Client/Server.










### La ejecución de la Image (creación del contenedor)
<a name="ejecucion_image_creacion_del_contenedor_ELTPARAGRAPHE000041"></a>

- El nombre del preajuste Image es: 
	
	```txt
	"windev/webdev:<Version>"
	```


- Debes activar la servidor de aplicaciones pasando el número de serie en el comando Line. El parámetro a pasar debe tener el siguiente formato: 
	
	```txt
	--configuration="<Serial number>-<Activation key>"
	```
Si no se especifica este parámetro, se usará un servidor de aplicaciones de 10 conexiones.  

- Para persistir en las aplicaciones y sus datos (por ejemplo, si el Image cambia), debe: 

	- Usar un volumen. El comando para crear un volumen: 
			
		```txt
		"docker volume create <VolumeName>"
		```


	- Monte el volumen en el contenedor en 
			
		```txt
		"/var/lib/WEBDEV/<major.minor version>/"
		```





- Se creará una cuenta WEBDEV y una cuenta Linux llamada "webdevuser":

	- Se generará una contraseña y se les asignará.

	- Es importante ver la salida del contenedor: 

		- realizando un inicio interactivo con "-it",

		- consultando los logs de contenedor con la sintaxis: 
						
			```txt
			"docker logs <container ID>"
			```








Ejemplo de comando Line: 

```txt
docker run -p 80:80 -v WB:/var/lib/WEBDEV/24.0/ 
windev/webdev:FR240054h --configuration="<Serial number>-<Activation key>"
```



### Notas (importante)
<a name="notas_importante_ELTPARAGRAPHE000073"></a>

- Si un nuevo contenedor utiliza datos de un volumen existente (por ejemplo, datos de un contenedor anterior después de una actualización de Image: Cambio de versión de WEBDEV (menor):

	- Esta operación sólo es posible con la misma versión principal.

	- Atención: Se crea una nueva contraseña. Esta contraseña se aplica a las cuentas "webdevuser" WEBDEV y Linux, por lo que deberá cambiar la contraseña para futuros despliegues del sitio.




- Para conectar con el administrador remoto "http[s]://&lt;servidor&gt;/WDAdminWeb":

	- La cuenta de WEBDEV implementación: 
		```txt
		webdevuser/<Password generated and displayed>
		```





- La biblioteca libqtgui5 está instalada: las imágenes generadas y las impresiones están disponibles. Sin embargo, sólo se instala una fuente.

- Reminder de los comandos básicos del Docker:

	- Detener un contenedor: 
			
		```txt
		"docker stop <ID>"
		```


	- Reiniciar un contenedor: 
			
		```txt
		"docker start <ID>"
		```


	- Precaución! Sin inicio interactivo: 
			
		```txt
		"docker start -i <ID>"
		```
Esto envía la señal SIGWINCH que Apache interpreta como un comando de parada.




## Errores de compilación de UI
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000207"></a>
Los errores de compilación de UI indican los posibles problemas detectados en sus interfaces cada vez que se guarda una ventana, una Page o una reporte o cada vez que se recompila el proyecto.

Los errores de compilación de UI también están disponibles en los informes.
Por Default, los errores de compilación de UI se habilitan para cada nuevo proyecto.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000231"></a>


### Habilitar los errores de compilación de UI
<a name="habilitar_los_errores_compilacion_ELTPARAGRAPHE000021"></a>

Los errores de compilación de UI pueden ser activados: 

- del proyecto description. 

- desde el panel de errores de compilación. 




Para permitir errores de compilación de UI del proyecto description:

1. Abra la ventana de descripción del proyecto (en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción").

2. En la pestaña "Compilación: 

	- En la sección "Errores de UI", seleccione la opción "Activar compilación de UI".

	- El botón "Editar los idiomas" permite elegir los idiomas del proyecto que serán tenidos en cuenta por la compilación UI.




3. Validar. 




Para habilitar los errores de compilación de UI desde el panel de errores de compilación:

1. Mostrar el panel de errores de compilación: en la pestaña "Inicio", en el grupo "Entorno", expanda "Paneles", seleccione "Paneles" y, a continuación, "Errores de compilación". 

2. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_Erreur_IHM.gif).




Se comprueban los errores de UI:

- al guardar una ventana, una Page o una reporte.

- en el momento de recompilar el proyecto.



<a name="NOTE2_2"></a>


### Corregir un error de UI
<a name="corregir_error_ELTPARAGRAPHE000080"></a>

Los errores de UI están listados en el panel de errores de compilación.

Para arreglar los errores de UI:

1. Haga doble clic en el error que se muestra en el panel. El elemento que contiene la control donde ocurre el problema se abre en el editor. Se muestra la description del control correspondiente.

2. Arregla el problema.



<a name="NOTE2_3"></a>


### Desactivando un error de compilación de UI
<a name="desactivando_error_compilacion_ELTPARAGRAPHE000092"></a>

Para desactivar un error de compilación de UI: 

1. En el panel de errores de compilación, seleccione el error que desea deshabilitar. 

2. Mostrar el menú contextual del error: 

	- "Desactivar este error" sirve para desactivar el error seleccionado. 

	- la opción "Desactivar este error y todos los idénticos" sirve para desactivar el error y todos los errores del mismo tipo en el proyecto actual.







Para ver la lista de errores de compilación de UI desactivados: 

1. Haga clic con el botón derecho del ratón en el panel de errores de compilación para abrir el menú contextual.  

2. Seleccione "Lista de errores desactivados". 


Para volver a habilitar (si es necesario) uno o más errores en esta lista, todo lo que tiene que hacer es desmarcar los errores que deben volver a habilitarse y validar la ventana..

## Algunos ejemplos de errores detectados de UI
<a name="algunos_ejemplos_errores_detectados_ELTTEXTE000267"></a>
Los problemas que se pueden detectar son los siguientes:

- Comprobando la existencia de los Image utilizados por los controles de Image.

- Detección de los subtítulos truncados en algunos tipos de controles.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Comprobación de la existencia de las ventanas internas utilizadas por los controles de la ventana interna.

- Comprobación de si un tabla anclado contiene una columna anclada.

- Comprobación de los enlaces con los ítems de los ficheros de datos o con las variables.

- Comprobando la existencia de las leyendas de control en todos los idiomas.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Comprobar si un acceso directo no existe varias veces en la misma ventana.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Comprobación de los enlaces dañados en las rupturas de tabla.

- Comprobación de la presencia de filas vacías al final del contenido inicial del list box y del combo box.

- Comprobación de las faltas de ortografía.

- Error si el texto que se muestra en un idioma es siempre el texto Default.

- Comprobando si la leyenda de un botón no excede el tamaño de la control.

- Comprobación de los mismos enlaces de enlace de la base de datos.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Comprobación del uso de menús con estilos diferentes ("sistema" y "no sistema") en la misma ventana

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Comprobación de los consejos de herramientas en una ventana. Error si varios controles tienen el mismo tooltip.

- Verificación de las dependencias del proyecto. Error si faltan dependencias. Por ejemplo: hojas de estilo, hojas CSS externas, imágenes de carpetas personalizadas, estilo de código, fuentes personalizadas del proyecto, etc.




## Importar un proyecto WEBDEV a WINDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000196"></a>
En la mayoría de los casos, un sitio es relacionado para una aplicación. 

WINDEV propone una función para importar sitios WEBDEV o páginas WEBDEV.

Tiene la posibilidad de importar todo el proyecto o una sola página.. Las páginas se convierten en ventanas. Los códigos de Servidor y Navegador están agrupados.

Los elementos sin equivalente en WINDEV (código JavaScript, por ejemplo) se importan como comentarios o como elementos separados.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000220"></a>


### Importar un proyecto WEBDEV
<a name="importar_proyecto_webdev_ELTPARAGRAPHE000024"></a>

Para importar un proyecto WEBDEV:

1. Abra WINDEV.

2. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Importar" y seleccione "De WEBDEV .. Un proyecto WEBDEV".. El asistente de importación se abre automáticamente.

3. Seleccione el proyecto WEBDEV que desea importar. Pase a la etapa siguiente.

4. Se creará un nuevo proyecto WINDEV a partir del proyecto WEBDEV importado.. El proyecto WEBDEV no será modificado. Especifique las características del nuevo proyecto WINDEV:

	- Nombre del proyecto.

	- Descripción del proyecto.

	- Directorio del proyecto.




5. Pase a la etapa siguiente.

6. Si el proyecto WEBDEV es relacionado para un análisis, puede hacerlo:

	- Compartir el análisis con el proyecto inicial: el mismo análisis será utilizado por el proyecto WEBDEV y por el nuevo proyecto WINDEV.

	- Copiar el análisis en el directorio del nuevo proyecto: el análisis se duplicará. Cada proyecto (WINDEV y WEBDEV) evolucionará con su propio análisis

	- No usar ningún análisis: el análisis es ignorado.




7. Pase a la etapa siguiente.

8. Si el proyecto WEBDEV contiene elementos que pueden ser compartidos con WINDEV (clases, conjuntos de procedimientos, informes, consultas), la importación de asistente le permite:

	- comparten estos elementos. Estos elementos no se copiarán físicamente en el nuevo proyecto WEBDEV pero serán accesibles desde el proyecto.

	- copy estos elementos en el nuevo proyecto WEBDEV. Estos elementos evolucionarán de forma independiente en cada proyecto.




9. Validar. La importación se realiza automáticamente: se crea el nuevo proyecto WINDEV.



<a name="NOTE2_2"></a>


### Importación de elementos WEBDEV
<a name="importacion_elementos_webdev_ELTPARAGRAPHE000064"></a>

Para importar elementos WEBDEV (páginas,....):

1. Abrir el proyecto WINDEV donde se debe realizar la importación.  

2. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Importar" y seleccione "De WEBDEV .. Elementos WEBDEV". El asistente de importación se abre automáticamente.

3. Seleccione el proyecto WEBDEV que contiene los elementos a importar o seleccione los archivos WEBDEV a importar.. Pase a la etapa siguiente.

4. Seleccione las páginas que desea importar.

5. Si el proyecto WEBDEV contiene elementos que pueden ser compartidos con WINDEV (clases, conjuntos de procedimientos, informes, consultas), la importación de asistente le permite:

	- comparten estos elementos. Estos elementos no se copiarán físicamente en el proyecto WEBDEV pero serán accesibles desde el proyecto.

	- copy estos elementos en el proyecto WEBDEV. Estos elementos evolucionarán de forma independiente en cada proyecto.




6. Validar. La importación se realiza automáticamente.

## Conversiones realizadas al importar un proyecto WEBDEV
<a name="conversiones_realizadas_importar_proyecto_webdev_ELTTEXTE000250"></a>
Cuando se importan proyectos WEBDEV o páginas WEBDEV en un proyecto WINDEV:

- no se importan los siguientes controles: IFrame, frameset, mapa del sitio ruta, paginador, ... Todos los controles no disponibles en WINDEV son sustituidos por una leyenda que indica "&lt;No importado&gt;"

- las miniaturas, las imágenes en las que se puede hacer clic y las imágenes se importan como imágenes.

- las celdas se importan como imágenes o como pies de foto.

- los enlaces se convierten en botones

- las acciones para la apertura de páginas se sustituyen automáticamente por el código de apertura de la ventana correspondiente.

- las páginas internas se convierten en ventanas internas.

- se comentan los códigos no soportados en WINDEV (código JavaScript, código PHP,...)

- las siguientes funciones (específicas de WEBDEV) se sustituyen automáticamente por su equivalente en WINDEV:
	


|   |   |
| --- | --- |
| **Función WEBDEV** | **Función WINDEV** |
| PageRefresh | WinRedraw |
| PageDisplay | entablar |
| FramesetDisplay | entablar |
| FramesetRefresh | entablar |
| ArchivoPágina | FileToScreen |
| PageParameter | CommandLine |


	Las funciones específicas de WEBDEV que no se sustituyen desencadenan errores de compilación. El código de la aplicación debe ser revisado y modificado para que funcione en Windows.

### Importar un proyecto WEBDEV PHP a un proyecto WINDEV
<a name="importar_proyecto_webdev_php_proyecto_windev_ELTPARAGRAPHE000142"></a>

Al importar un proyecto WEBDEV PHP a un proyecto WINDEV, los siguientes elementos no se importan:

- El código de los botones,

- Código del proyecto.




Al importar una tabla, pueden surgir problemas de visualización.

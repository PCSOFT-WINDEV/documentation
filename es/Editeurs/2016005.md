


## Creación de una plantilla de reporte
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Etapas a seguir
<a name="etapas_seguir_ELTTEXTE000205"></a>
Se requieren varios pasos para crear una plantilla de reporte:

- [**Etapa 1**: Creación de un proyecto que agrupa los diferentes modelos de informes de tema](#NOTE2_1). 

- [**Etapa 2**: Creación de un tema-modelo reporte para todos los tipos de informes](#NOTE3_1).

- [**Etapa 3**: Copia de los informes tema-modelo](#NOTE4_1).

- [**Paso 4 (opcional)**: Creación de una vista previa de reporte](#NOTE5_1). 




Observaciones:

- Uno de los métodos para crear una plantilla de reporte se presenta aquí. Se recomienda el método que se presenta a continuación, pero no es obligatorio.

- La versión de usuario de la editor de reportes (Reports & Queries) no le permite crear sus propios modelos de informes tema.

- La ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=i_blue_sm.jpg) Icon indica una punta. Las instrucciones descritas en estos consejos no son obligatorias pero le ayudarán a crear un reporte plantilla.

## Etapa 1: Creación de un proyecto plantilla
<a name="etapa_1_creacion_proyecto_plantilla_ELTTEXTE000229"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000039"></a>

En la mayoría de los casos, un plantilla incluye varias plantillas reporte (hasta 4). Para simplificar la creación y la modificación de sus plantillas tema, le recomendamos que utilice un proyecto para cada plantilla.

El proyecto plantilla se agrupará:

- un tema-modelo reporte para cada tipo de reporte. WINDEV propone 4 tipos de informes, por lo tanto el proyecto tema-modelo contendrá 4 informes tema-modelo.

- una vista previa de reporte.




**Observación**: Si se utilizan plantillas de tema para personalizar sus ventanas, sus plantillas de tema de ventana pueden ser creadas en este proyecto. Para obtener más información, consulte [Creando una ventana plantilla](../Editeurs/2016009.md).
<a name="NOTE2_2"></a>


### Creación de un proyecto plantilla
<a name="creacion_proyecto_plantilla_ELTPARAGRAPHE000056"></a>

Para crear un proyecto tema-modelo, crear un nuevo proyecto: 

- Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

- Aparece la ventana de creación: haga clic en "Proyecto".Este proyecto es relacionado a ningún análisis y no utiliza ningún plantilla.


![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=i_blue_sm.jpg) Para identificar fácilmente sus proyectos plantilla, se recomienda utilizar la siguiente notación: "SkinTemplate_&lt;NameOfSkinTemplate&gt;". Por ejemplo, si la plantilla se llama Caribe, el proyecto se llamará "SkinTemplate_Caribbean".

## Etapa 2: Creación de un tema-modelo reporte
<a name="etapa_2_creacion_temamodelo_reporte_ELTTEXTE000259"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000073"></a>

Cada tema-modelo reporte corresponde a un tipo de reporte. Por lo tanto, se debe crear un tema-modelo reporte para cada tipo de reporte utilizado. Para obtener más información, consulte [Los tipos de informes propuestos por WINDEV](../WDChamp/1011054.md).
<a name="NOTE3_2"></a>


### Creación de un tema-modelo reporte
<a name="creacion_temamodelo_reporte_ELTPARAGRAPHE000083"></a>

Para crear un tema-modelo reporte:

1. Crear una Record libre: 

	- Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- Aparece la ventana de creación: haga clic en "Reporte", y luego en "Reporte". El asistente de creación de reportes se abre.

	- En la asistente, seleccione "Blanco reporte" y valide.




2. Este reporte:

	- no se imprimirá en un formulario,

	- no utilizará ninguna fuente de datos,

	- tendrá las dimensiones de Default.




3. Especifique el nombre de la reporte según la siguiente anotación: "&lt;NameOfSkinTemplate&gt;_&lt;ReportType&gt;. Por ejemplo, "Caribbean_Label".

4. En las pestañas "Fondo" y "Detalles" de la description de los bloques de reporte ("Bloque description" en el menú contextual), especifique el estilo deseado (fondo Color, espesor Border, ...).

5. Para personalizar su reporte, puede añadir:

	- ***controles específicos de la plantilla***. 
			Estos controles se mostrarán en todos los informes que utilicen este plantilla. Por ejemplo, puede añadir una control preestablecida para mostrar la fecha y la hora del sistema, o una control Imagen para mostrar el logotipo de una empresa.

	- ***controles con los datos del reporte***.
			Al crear un reporte, el editor de reportes crea automáticamente todos los controles necesarios para utilizar el reporte. Estos controles utilizan el estilo por defecto de los controles reporte (sin fondo Color, sin Border, fuente "Arial", tamaño "11"). 
			Para utilizar un estilo diferente de control, añada estos control s (véase la lista de control s) y defina su estilo en el tema-modelo reporte. Estos mandos se pueden posicionar en cualquier bloque.
			Si se añaden controles personalizados en un tema-modelo reporte, estos controles se usarán automáticamente cuando se cree el reporte.




6. Guarde el reporte (Ctrl + S).

## Etapa 3: Copia de los informes tema-modelo
<a name="etapa_3_copia_los_informes_temamodelo_ELTTEXTE000289"></a>
Para usar una plantilla personalizada de tema en sus aplicaciones, los informes de tema-modelo deben ser copiados en el directorio "Personal\\Template\\&lt;NameOfSkinTemplate&gt;" de WINDEV o WEBDEV

**Observación**: Una vez copiados los informes de tema-modelo, se debe reiniciar WINDEV o WEBDEV para tener en cuenta estas nuevas plantillas de informes de tema.

## Etapa 4: Creación de la vista previa de la plantilla
<a name="etapa_4_creacion_vista_previa_plantilla_ELTTEXTE000313"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000129"></a>

Para simplificar el uso de un plantilla personalizado, le aconsejamos que cree vistas previas. Estas vistas previas se utilizarán cuando se aplique una plantilla a una reporte.

Cada vista previa de reporte corresponde a un tipo de reporte. De hecho, la vista previa difiere según el tipo de reporte (o según el reporte actual). Por lo tanto, se debe crear una vista previa de reporte para cada tipo de reporte utilizado.
<a name="NOTE5_2"></a>


### Crear una vista previa de plantilla
<a name="crear_una_vista_previa_plantilla_ELTPARAGRAPHE000138"></a>

Para crear una vista previa de plantilla:

1. En un proyecto associated con un análisis, crear una nueva reporte: 

	- Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- Aparece la ventana de creación: haga clic en "Reporte", y luego en "Reporte". El asistente de creación de reportes se abre.

	- Cree un "formulario" reporte, por ejemplo.




2. Seleccione la plantilla de reporte que se creó de antemano. En la zona "Preview" no se muestra ninguna vista previa de plantilla.

3. Validar la creación de reporte. La reporte creada se convierte en la reporte actual.

4. copy la reporte creada: en la pestaña "Inicio", en el grupo "Editar", haga clic en "copy" (o pulse Ctrl + C).

5. Pegue el contenido del portapapeles en un software de dibujo ("Paint" por ejemplo).

6. Guardar imagen:

	- de acuerdo con la siguiente notación: "&lt;NameOfSkinTemplate&gt;_&lt;ReportType&gt;.BMP".
			Por ejemplo, "Caribbean_Form.BMP".

	- en el directorio "Personal\\Template\\&lt;NameOfSkinTemplate&gt;" de WINDEV o WEBDEV.

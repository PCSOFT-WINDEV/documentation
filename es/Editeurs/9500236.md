## Presentación
<a name="presentacion_ELTTEXTE000333"></a>
La ventana del proyecto description le permite configurar varios elementos de su proyecto. Esta ventana incluye las siguientes pestañas:

- La pestaña ["Proyecto"](../Editeurs/2030046.md), que permite introducir información general sobre el proyecto

- La pestaña "Análisis", que permite asociar un análisis al proyecto y/o a un modelo UML.

- La pestaña ["Archivos"](../Editeurs/2030052.md), que permite introducir información sobre los archivos de datos utilizados por la aplicación (sólo si el proyecto está vinculado a un análisis).

- La pestaña "Datos en vivo", que permite configurar cómo se muestran los datos de los ficheros de datos en los distintos editores.

- La pestaña ["Idiomas"](../Editeurs/9500219.md), que se utiliza para configurar los diferentes idiomas utilizados por el proyecto, así como las opciones que se utilizarán para cada idioma.

- La pestaña ["Estilo"](../Editeurs/9500234.md), que permite personalizar la vista previa de impresión y elegir el tema y la hoja de estilo personalizada de plantilla. También puedes configurar el tamaño de la pantalla.

- La pestaña "SCM", que sirve para gestionar el trabajo en equipo (SCM o GIT).

- La pestaña ["Opciones"](../Editeurs/9500227.md), que permite configurar las opciones de actualización, copia de seguridad y sincronización de modelo. Esta pestaña también se utiliza para habilitar el modo estricto y para definir el estilo de código.

- La pestaña **"Avanzado"**, que permite habilitar o deshabilitar el uso de "Reports & Queries", configurar las FAA y definir el bloqueo de la aplicación.

- La pestaña ["Recopilación"](../Editeurs/9500218.md), que permite configurar las opciones de compilación.

- La pestaña ["Telemetría"](../Editeurs/9500223.md), que permite configurar las opciones de telemetría del proyecto.

### Reports and Queries
<a name="reports_and_queries_ELTPARAGRAPHE000058"></a>

La opción "Activar "Reports and Queries"" permite activar o desactivar "Reports and Queries" para el proyecto en curso.. En este caso, el usuario podrá iniciar "Reports and Queries" desde el visor de reporte. 
<a name="NOTE2_2"></a>


### Características de la aplicación automatic (AAF)
<a name="caracteristicas_aplicacion_automatic_aaf_ELTPARAGRAPHE000065"></a>

Las opciones disponibles son: 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Activar OVI (Oscurecimiento de Ventanas Inactivas) para oscurecer las ventanas cuando están inactivas: Con esta opción, puede set las ventanas en el proyecto para que se apaguen automáticamente cuando Inactive. Si esta opción está marcada, las ventanas de fondo aparecen automáticamente en gris, lo que permite al usuario identificar rápidamente la ventana Active. 
	Para obtener más información, consulte [Oscurecimiento de ventanas inactivas](../Editeurs/9000071.md). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) "Permitir que el usuario final modifique la UI": Esta opción permite a los usuarios finales modificar algunos elementos de UI. El botón "Opciones" permite definir los cambios que se pueden realizar. 
	Para obtener más información, consulte [UI modificable por el usuario final](../WDLang1/1000021853.md). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Macro-Código de Usuario: El botón "Macrocódigo de usuario" le permite decidir si autoriza a los usuarios finales a escribir código. 
	Para obtener más información, consulte [UMC: Implantación y gestión de los "macrocódigos de usuario"](../WDLang1/9000020.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Animaciones de ventanas: El botón "Animaciones de ventanas" permite configurar las animaciones que se aplican automáticamente en las distintas ventanas del proyecto. 
	Para obtener más información, consulte [Animación de las ventanas de una aplicación WINDEV](../WDLang1/3038054.md). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Animaciones de controles: El botón "Animaciones de control s" permite activar y configurar diferentes animaciones para cada tipo de control. 
	Para obtener más información, consulte [Animación de controles](../WDLang1/3038060.md). 

- Menú de FAA: Cada control de un proyecto WINDEV contiene un menú contextual automatic, con opciones Default. El botón "Menú AAF" permite configurar las opciones que se mostrarán u ocultarán en el menú contextual de cada tipo de control del proyecto. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Mostrar los AAF disponibles: Este botón abre un archivo PDF que contiene algunos de los AAF disponibles en una aplicación WINDEV.

### Seguridad de la aplicación
<a name="seguridad_aplicacion_ELTPARAGRAPHE000126"></a>

WINDEV le ofrece la posibilidad de bloquear sus aplicaciones (tras una duración determinada, por ejemplo). Todas las ventanas de aplicación se minimizan y se hacen Invisible. Se requiere una contraseña cuando el usuario quiere restaurar la aplicación.

La opción "Bloquear automáticamente la aplicación..." le permite implementar esta funcionalidad. 

Para obtener más información, consulte [Bloqueo de una aplicación](../WDLang6/3061101.md).

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Contar el tiempo
<a name="contar_tiempo_ELTPARAGRAPHE000142"></a>

La opción "Activar función para contar el tiempo invertido en los elementos del proyecto" le permite set la gestión del tiempo. Para cada elemento del proyecto (ventanas, informes, etc.), se calcula el tiempo total empleado:

- para todos los colaboradores del proyecto, 

- para cada colaborador del proyecto.




Para obtener más información, consulte [Gestión del tiempo](../Editeurs/2030017.md).

### Unidades de tamaños y posiciones para las nuevas ventanas
<a name="unidades_tamanos_posiciones_para_las_nuevas_ventanas_ELTPARAGRAPHE000187"></a>

Esta opción permite elegir el sistema de unidades que se utilizará al editar el tamaño y la posición de los controles, las fuentes, etc.

- Compatible con escritorio Y móviles: Utiliza una unidad lógica compatible entre plataformas. 
	Se utilizará si la ventana se muestra en dispositivos de escritorio y móviles. 

- Plataforma única escritorio O móvil: El desarrollador debe adaptar las coordenadas en función de la plataforma. 
	 Se debe utilizar si la plataforma es sólo móvil o sólo de escritorio.

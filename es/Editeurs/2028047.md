
## Directorio de instalación
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000313"></a>
La creación de instalación asistente permite especificar el directorio instalación. 

En el paso "Directorio de configuración" de asistente, tiene la capacidad de especificar: 

- [la configuración básica](#NOTE2_1), 

- [la configuración avanzada](#NOTE3_1), 

- [la implementación de UAC para el instalador](#NOTE4_1). 




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Ajuste básico
<a name="ajuste_basico_ELTTEXTE000337"></a>


### Default ruta de instalación en el ordenador del usuario final
<a name="default_ruta_instalacion_ordenador_del_usuario_final_ELTPARAGRAPHE000031"></a>

La configuración de instalación se utiliza para seleccionar el directorio instalación propuesto por Default. El usuario podrá modificar este directorio al instalar la aplicación.

Este directorio es "C:\\Archivos de programa\\&lt;Nombre de la aplicación&gt;" de Default.


### Por defecto ruta de referencia instalación (red instalación solamente)
<a name="por_defecto_ruta_referencia_instalacion_red_instalacion_solamente_ELTPARAGRAPHE000040"></a>

La configuración de instalación se utiliza para seleccionar el directorio instalación de reference instalación propuesto por Default. El usuario podrá modificar este directorio al instalar la aplicación.

Esta ruta es opcional.

<a name="configuracion_avanzada_del_directorio_instalacion_ELTTEXTE000367"></a>

### Permitir al usuario guardar los archivos reemplazados
<a name="permitir_usuario_guardar_los_archivos_reemplazados_ELTPARAGRAPHE000082"></a>

Cuando se actualiza una aplicación, se puede incluir un avión adicional en el programa instalación. Este plano adicional permite al usuario guardar los archivos reemplazados en un directorio de copia de seguridad.. Este directorio de respaldo se llamará "Backup_&lt;BackupDate&gt;_&lt;BackupTime&gt;" por Default.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Info_affiches_install%20-%20HC%20N%B0003.gif)


Esta copia de seguridad puede ser usada si la instalación falla por ejemplo. Para utilizar esta copia de seguridad, el usuario tendrá que copy los archivos encontrados en el directorio de copia de seguridad en el directorio de la aplicación instalación.

La copia de seguridad de los archivos reemplazados es propuesta por Default a los usuarios.
<a name="NOTE3_3"></a>


### Ejecute un programa al final de instalación
<a name="ejecute_programa_final_instalacion_ELTPARAGRAPHE000095"></a>

Por Default, el programa instalación propone al usuario ejecutar la aplicación ejecutable al final de instalación. La opción "Iniciar el programa" está marcada.. El ejecutable se ejecuta automáticamente al hacer clic en el botón "Listo".

La creación del programa instalación asistente permite especificar el ejecutable que debe ser ejecutado y asociar parámetros a este ejecutable. Sólo se puede ejecutar un ejecutable (archivo".EXE").

El ejecutable a ejecutar corresponde a:

- el nombre y la ruta completa del archivo ejecutable. Se pueden utilizar las siguientes palabras clave:

	- **&lt;SetupDir&gt;&gt;SetupDir**: Directorio de instalación elegido por el usuario durante la instalación.

	- **Configuración de la unidad&gt; &lt;SetupDrive**: del directorio de instalación elegido por el usuario durante la instalación.

	- **&lt;Archivo de configuración&gt;&gt;.**: El directorio de archivos de datos de instalación. La creación del programa instalación asistente permite configurar este directorio. Para obtener más información, consulte [Configuración del directorio de archivos de datos](../Editeurs/2028045.md).

	- **&lt;WinDir&gt;&gt;Dirección del viento**: Directorio Windows de ordenadores de usuario

	- **&lt;SysDir&gt; &lt;SysDir&gt;**: directorio del sistema.

	- **&lt;RefDir&gt;&gt;RefDir**: directorio de referencia para las configuraciones con actualización de automatic solamente.




- los parámetros de tiempo de ejecución del programa instalado (si es necesario).




Por ejemplo: "&lt;SetupDir&gt;\\MyApp.EXE /Param".

Observación: Si se realizan modificaciones en el archivo "AutoExec.Bat", no se debe ejecutar ningún programa al final de la instalación. De hecho, las modificaciones realizadas en este archivo serán tenidas en cuenta cuando se reinicie el equipo.
<a name="NOTE3_Silent"></a>


### Configurar una instalación silenciosa
<a name="configurar_una_instalacion_silenciosa_ELTPARAGRAPHE000125"></a>

Las configuraciones de la versión de reference y la versión cliente pueden realizarse en modo silencioso. En este caso, la instalación se hará sin ninguna intervención del usuario. Las opciones definidas por Default serán usadas automáticamente. 

Las opciones disponibles son: 

- **Instalación cliente silenciosa**: la aplicación instalación para el usuario final no requiere ninguna acción por parte del usuario. 

- **Silenciosa implementación de la versión de referencia** (disponible durante una actualización de instalación con automatic): la versión reference de la red instalación se instalará sin intervención del usuario.  



<a name="NOTE3_4"></a>


### Modificar el identificador utilizado para desinstalar la aplicación
<a name="modificar_identificador_utilizado_para_desinstalar_aplicacion_ELTPARAGRAPHE000137"></a>

Si ha elegido permitir que su aplicación se desinstale, se creará automáticamente un identificador específico en el registro.. Este identificador puede ser modificado a través del programa de creación de instalación asistente.

Por Default, este identificador es "&lt;NombreDeAplicaciónSinEspacio&gt;". Por ejemplo, "WDCustomer" para la aplicación "WD Customer".. Este identificador puede (o no) contener espacios en blanco.

Este identificador se creará en la clave:

- "HKEY_CURRENT_USER\\Software\\Microsoft\\Windows\\CurrentVersion\\Uninstall" en Windows 2000 y NT.

- "HKEY_LOCAL_MACHINE\\Software\\Microsoft\\Windows\\CurrentVersion\\Uninstall" en Windows 98 y Me.



<a name="NOTE3_5"></a>


### Modificar o crear el archivo "AutoExec.BAT".
<a name="modificar_crear_archivo_autoexecbat_ELTPARAGRAPHE000151"></a>

Tienes la capacidad de modificar o crear el archivo "AutoExec.BAT" en las computadoras de instalación. Este archivo se modificará o creará (si es necesario) cuando se reinicie el equipo. En este caso, se propone automáticamente un reinicio al final de la aplicación instalación.

El archivo "AutoExec.BAT" se utiliza para:

- añadir el directorio del ejecutable de la aplicación en el PATH.

- inicie la aplicación cuando inicie la computadora de instalación.

- ...




Recordatorios:

- El archivo "AutoExec.BAT" se ejecuta automáticamente cuando se inicia el equipo.

- El PATH lista los directorios en los que se buscan prioritariamente los ejecutables (archivos".EXE") y las bibliotecas (archivos"DLL").



<a name="NOTE3_6"></a>


### Modificar o crear los archivos ".INI
<a name="modificar_crear_los_archivos_ini_ELTPARAGRAPHE000169"></a>

Cuando se instala una aplicación WINDEV, se tiene la posibilidad de modificar o crear los archivos ".INI" en los ordenadores de instalación. Estos archivos serán associated con la aplicación instalada.

Los archivos ".INI" están acostumbrados a:

- personalizar la aplicación según el usuario conectado.

- definir los parámetros de Default de la aplicación.

- ...




Entonces, los archivos ".INI" pueden ser leídos y modificados desde la aplicación directamente ([INIRead](../WDLang1/3013043.md) y [INIWrite](../WDLang1/3013039.md)).

**Ejemplo de modificación**
Por ejemplo, para guardar el lenguaje de ejecución del programa instalación en el archivo "MyAppli.INI" (en el directorio de la aplicación instalación):

- "Archivo ".INI: &lt;Dirección de instalación&gt;\\MyApp.INI

- Sección: IDIOMA

- paramétrico: INSTALAR_IDIOMA

- Valor: &lt;Idioma&gt;




La palabra clave **&lt;Language&gt;** se reemplazará automáticamente durante la instalación por el número del idioma de ejecución del programa de instalación (este número puede utilizarse con la función [Nación](../WDLang1/3054014.md)).

En este ejemplo, si el número del lenguaje de ejecución es 3, el siguiente parámetro se añadirá al archivo "MyApp.INI".: 


```txt
[LANGUAGE]
LANGUAGE_INSTALL=3
```

<a name="NOTE3_7"></a>


### Modificar o crear valores en el registro
<a name="modificar_crear_valores_registro_ELTPARAGRAPHE000208"></a>

Al instalar una aplicación WINDEV, tiene la posibilidad de modificar o crear entradas en el registro.

El registro se lo permite:

- personalizar la aplicación según el usuario conectado.

- definir los parámetros de Default de la aplicación.

- ...




Entonces el registro puede ser leído y modificado desde la aplicación directamente ([RegistrySetValue](../WDLang1/3065006.md), [RegistryQueryValue](../WDLang1/3065012.md), ...).

Ejemplo de modificación
Por ejemplo, para guardar el lenguaje de ejecución del programa instalación en el registro, escriba:

- Clave: HKEY_LOCAL_MACHINE\\Software\\MyCompany\\MyApp\\1.0\\

- denominar: Idioma

- Valor: &lt;Idioma&gt;


La palabra clave **&lt;Language&gt;** se reemplazará automáticamente durante la instalación por el número del idioma de ejecución del programa de instalación (este número puede utilizarse con la función [Nación](../WDLang1/3054014.md)).

## Implementación del UAC para el instalador
<a name="implementacion_del_uac_para_instalador_ELTTEXTE000433"></a>
Por Default, la instalación de una aplicación WINDEV activa la UAC.

Para acceder a la configuración relativa a la implementación de la UAC, haga clic en el botón "Implementar la UAC".

La ventana que se muestra permite modificar este modo de funcionamiento a través de "El instalador NO necesita los derechos de administrador".: 

- para las configuraciones individuales: la UAC está desactivada para la instalación en el ordenador del usuario final. 

- para las configuraciones de la red con la actualización de automatic. En este caso, el UAC puede ser desactivado: 

	- para el instalador del usuario final,

	- para el instalador de la versión reference,

	- para ambos. 







Atención: La opción "No requiere derechos de administrador" tiene algunas limitaciones: no se pueden instalar archivos en los directorios del sistema (Archivos de programa, Windows...), no se pueden modificar los archivos y parámetros del sistema (win.ini, ruta, etc.). 

La instalación fracasará si los derechos no son suficientes. Siempre que sea posible, la instalación asistente guiará al usuario en sus elecciones para que pueda instalar la aplicación. 

Observación: La implementación del UAC para el instalador también puede configurarse en el [opciones avanzadas del instalador](../Editeurs/2028108.md). Estas opciones son accesibles en el paso "Medio" de la creación de instalación asistente.

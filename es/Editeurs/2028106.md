


## WDInst: Personalizando el programa de instalación
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000255"></a>
El editor de instalación WDInst propone un programa estándar de instalación.

Este instalador puede ser personalizado para proponer una instalación personalizada. 

Esta personalización está disponible: 

- para las configuraciones autónomas: instalación estándar, instalación compacto o instalación a través del paquete MSI. 

- para las configuraciones con automatic update (Live Update). 




Para personalizar el instalador utilizado en WDInst, seleccione "Parámetros de configuración... Opciones del instalador". 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Elección del tipo de instalador a utilizar
<a name="eleccion_del_tipo_instalador_utilizar_ELTTEXTE000279"></a>
La ventana de opciones del instalador le permite elegir el tipo de instalador que desea utilizar: 

- instalador estándar.

- Instalador MSI.

- instalador compacto. 

- instalador personalizado. 

- sin instalador. 




El tipo de instalador se selecciona a través del enlace "Cambiar" que se encuentra en el título de la ventana.

## Instalador estándar o personalizado
<a name="instalador_estandar_personalizado_ELTTEXTE000303"></a>


### Instalador estándar
<a name="instalador_estandar_ELTPARAGRAPHE000044"></a>

El instalador estándar es el instalador suministrado con WINDEV para instalar las aplicaciones WINDEV.. 

La pestaña **"Framework WINDEV** permite especificar el modo de gestión del framework del instalador. Es posible: 

- copy el WINDEV framework con el instalación. En este caso, el tamaño del programa de instalación puede ser bastante importante. Tiene la posibilidad de personalizar el instalador (pestaña"Personalizar el instalador"). 

- descargar el WINDEV framework desde un servidor FTP. En este caso, el usuario debe tener acceso a INTERNET. Si se selecciona esta opción, se utiliza la pestaña "Parámetros FPT" para configurar la descarga de la framework. 

- utilizar el WINDEV framework que se encuentra en el ordenador del usuario. En este caso, la framework se descargará sólo si el WINDEV framework no se encuentra en el ordenador de la instalación. 




Esta pestaña también se puede utilizar para limpiar las estructuras anteriores. La opción "Borrar los marcos anteriores encontrados en el directorio de destino" se utiliza para borrar todas las versiones anteriores de la WINDEV framework que se encuentran en los directorios de la instalación.

El **"Ficha"UAC** se utiliza para especificar si el instalador necesita el UAC o no. 

La **pestaña "Plug-in"** permite indicar la ruta del plug-in que se incluirá en el programa de instalación estándar. Para obtener más información, consulte [Personalizar el programa instalación (Plug-in)](../Editeurs/2028109.md). 

El **"Ficha"Fuentes** se utiliza para especificar las fuentes de caracteres que se implementarán e instalarán en el equipo cliente.. Tiene la posibilidad de añadir los archivos TTF u OTF correspondientes a las fuentes a instalar.
Observación: Compruebe que tiene derecho a distribuir estas fuentes.


### Instalador personalizado: caso del individuo instalación
<a name="instalador_personalizado_caso_del_individuo_instalacion_ELTPARAGRAPHE000076"></a>

El programa estándar de instalación (llamado "WDSetup") se suministra con los ejemplos de WINDEV. Para personalizar el programa de instalación, todo lo que tienes que hacer es modificar el proyecto "WDSetup".

**Observación**: Si está usando un programa personalizado de instalación (proyecto personalizado "WDSetup"), debe mantener el archivo ".WXF" para poder ejecutar la prueba de este programa personalizado de instalación. Para obtener más información, consulte ["Archivo ".WXF](../Editeurs/2028055.md).

Para personalizar el programa instalación (en WINDEV): 

1. Abrir el proyecto "WDSetup. 
	Por Default, este proyecto se encuentra en el subdirectorio "Ejemplos" del directorio WINDEV instalación.
	Observación: Este ejemplo también está disponible en los "Ejemplos completos" que se muestran en la página principal de WINDEV (Ctrl + <).

2. Personalizar este proyecto (modificar el diseño de las ventanas, gestionar idiomas adicionales, ....).

3. Crear un nuevo ejecutable: haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_exe.gif) en los botones de acceso rápido.


**Observación**: El nombre de este ejecutable no importa.

Para usar un programa personalizado de instalación (en WDInst): 

1. Seleccione "Parámetros de configuración ... Opciones del instalador".

2. Seleccione "Instalador estándar" (a través del enlace "Cambiar" si es necesario).

3. En la pestaña "Framework WINDEV", seleccione "Copiar el framework WINDEV con la instalación".

4. En la pestaña "Personalizar el instalador", seleccione "Usar un instalador personalizado" y seleccione el ejecutable del programa personalizado de instalación.

5. En la pestaña "DLL del instalador", seleccione las bibliotecas (archivos ".DLL") associated con el programa personalizado instalación.




**Observación**: Estas bibliotecas (archivos "DLL") se encuentran en el subdirectorio "EXE" del proyecto personalizado instalación.


### Instalación personalizada: caso de una instalación con actualización de la red
<a name="instalacion_personalizada_caso_una_instalacion_con_actualizacion_red_ELTPARAGRAPHE000112"></a>

El programa estándar instalación se utiliza para crear la referencia instalación y la instalación de la aplicación cliente. 

Ahora tiene la posibilidad de usar un instalador diferente para la reference instalación y para la instalación de la aplicación cliente. 

Recordatorio: 

- La aplicación instaladora (llamada "WDSetup") se suministra con los ejemplos de WINDEV. Para personalizar el programa de instalación, todo lo que tienes que hacer es modificar el proyecto "WDSetup".

- Si está usando un programa personalizado de instalación (proyecto personalizado "WDSetup"), debe mantener el archivo ".WXF" para poder ejecutar la prueba de este programa personalizado de instalación. Para obtener más información, consulte ["Archivo ".WXF](../Editeurs/2028055.md).




Para personalizar el programa instalación (en WINDEV): 

1. Abrir el proyecto "WDSetup. 
	Por Default, este proyecto se encuentra en el subdirectorio "Ejemplos" del directorio WINDEV instalación.
	Observación: Este ejemplo también está disponible en los "Ejemplos completos" que se muestran en la página de inicio.

2. Personalizar este proyecto (modificar el diseño de las ventanas, gestionar idiomas adicionales, ....).

3. Crear un nuevo ejecutable: haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_exe.gif) en los botones de acceso rápido.


**Observación**: El nombre de este ejecutable no importa.

Usar un programa personalizado de instalación para una aplicación con actualización de la red (en WDInst): 

1. Seleccione "Parámetros de configuración ... Opciones del instalador".

2. Seleccione "Instalador estándar" (a través del enlace "Cambiar" si es necesario).. 

3. En la pestaña "Framework WINDEV", seleccione "Copiar el framework WINDEV con la instalación".

4. En la pestaña "Personalización
del instalador", puede: 

	- personalizar (o no) el instalador de la aplicación cliente. Para personalizar el instalador, seleccione "Usar un instalador personalizado" y seleccione el ejecutable del programa personalizado de instalación.

	- personalizar (o no) el instalador de reference. Para personalizar el instalador, seleccione "Usar un instalador personalizado" y seleccione el ejecutable del programa personalizado de instalación. 
			 Atención: se debe utilizar un instalador estándar para desplegar una reference instalación en la Nube de aplicaciones PC SOFT. 




5. En la pestaña "DLL del instalador", seleccione las bibliotecas (archivos ".DLL") associated con el programa personalizado instalación.


**Observación**: Estas bibliotecas (archivos "DLL") se encuentran en el subdirectorio "EXE" del proyecto personalizado instalación.


### Gestión de la UAC
<a name="gestion_uac_ELTPARAGRAPHE000166"></a>

Por Default, la instalación de una aplicación WINDEV activa la UAC. 

La pestaña " UAC" permite modificar este modo de funcionamiento: 

- para las configuraciones individuales: la UAC está desactivada para la instalación en el ordenador del usuario final. 

- para las configuraciones de la red con la actualización de automatic. En este caso, el UAC puede ser desactivado: 

	- para el instalador del usuario final,

	- para el instalador de la versión reference,

	- para ambos. 







Atención: La opción "El instalador NO requiere derechos de administrador" tiene algunas restricciones: no se pueden instalar archivos en los directorios del sistema (Archivos de programa, Windows...), no se pueden modificar los archivos y parámetros del sistema (win.ini, ruta, etc.). 

La instalación fracasará si los derechos no son suficientes. Siempre que sea posible, la instalación asistente guiará al usuario en sus elecciones para que pueda instalar la aplicación.

## Instalador MSI
<a name="instalador_msi_ELTTEXTE000345"></a>
Para personalizar un programa MSI instalación (en WDInst): 

1. Seleccione "Parámetros de configuración ... Opciones del instalador".

2. Seleccione "Servicio de Windows Installer (.msi)" (a través del enlace "Cambiar" si es necesario).

3. En la pestaña "Propiedades del producto instalado", especifique las propiedades de la aplicación a instalar. 

4. En la pestaña "Propiedades del archivo MSI", especifique las propiedades del archivo MSI que se generará.

## Instalador compacto
<a name="instalador_compacto_ELTTEXTE000369"></a>
Para personalizar un programa compacto de instalación (en WDInst): 

1. Seleccione "Parámetros de configuración ... Opciones del instalador".

2. Seleccione "Instalador estándar" (a través del enlace "Cambiar" si es necesario).

3. El botón "Traducir los mensajes del instalador compacto" se utiliza para personalizar el instalador proporcionando las traducciones de los mensajes mostrados durante instalación.

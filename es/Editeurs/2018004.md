## Presentación
<a name="presentacion_ELTTEXTE000235"></a>
Una aplicación a menudo está destinada a ser utilizada por varios usuarios conectados a través de una red.. Del mismo modo, un sitio siempre está destinado a ser utilizado por varios usuarios de la Web. En la mayoría de los casos, se deben definir diferentes niveles de acceso en función del usuario o del usuario Web.

Para gestionar estos niveles de acceso en las aplicaciones WINDEV o sitios WEBDEV, se debe incluir el Groupware Usuario en la aplicación o sitio. Con unos pocos clics, una aplicación estándar puede transformarse en una aplicación que gestiona varios niveles de acceso. Esta funcionalidad puede implementarse fácilmente.

Para más detalles sobre el programa de grupo de usuarios, véase [Visión general de los grupos de usuarios](../Editeurs/2018001.md).

## Las opciones de la Agrupación de Usuarios

### Implantación del programa de grupo de usuarios en su aplicación
<a name="implantacion_del_programa_grupo_usuarios_aplicacion_ELTPARAGRAPHE000031"></a>

Para implementar el Groupware Usuario en la aplicación: 

1. Seleccione "Taller... Configurar el groupware usuario".

2. Seleccione la opción "Activar la gestión del groupware usuario"

3. Valide.

## Opciones de la pestaña "General
<a name="opciones_pestana_general_ELTTEXTE000289"></a>


### Modo de integración del groupware
<a name="modo_integracion_del_groupware_ELTPARAGRAPHE000053"></a>

Hay dos modos disponibles para incluir el User Groupware en su aplicación:

- **Integración predeterminada**: 
	La programación del Groupware Usuario se incluye automáticamente en su aplicación. 
	***Se incluye una nueva ventana o página en su proyecto: La ventana/página de inicio de sesión***. Se muestra cuando se inicia la aplicación y permite al usuario conectarse a la misma.. Puede personalizar esta ventana/página aplicando el plantilla de su aplicación, por ejemplo.

- **Integración personalizada**: 
	La programación del Groupware Usuario se incluye automáticamente en su aplicación.
	***Todas las ventanas o páginas necesarias para la gestión de la Agrupación de Usuarios (login y gestión de usuarios) se incluyen automáticamente en su proyecto***. Puede personalizar todas las ventanas o páginas utilizadas por el groupware usuario.




Las diferentes ventanas o páginas utilizadas por el User Groupware se presentan en [Las ventanas y páginas del programa de grupo de usuarios](../Editeurs/2018010.md).
<a name="NOTE3_2"></a>


### Iniciar el programa de grupo de usuarios
<a name="iniciar_programa_grupo_usuarios_ELTPARAGRAPHE000076"></a>

Existen dos modos para iniciar el Groupware Usuario:

- ***Inicio automático***: La ventana o página de inicio de sesión del User Groupware se muestra antes que cualquier otra ventana/página de la aplicación. El código de inicialización del proyecto se ejecuta cuando se abre y valida la ventana/página de inicio de sesión.

- ***Arranque manual***: La ventana o página de acceso sólo se abrirá si se utiliza [gpwOpen](../WDLang6/3041002.md). Esta opción se utiliza para ejecutar el código de inicialización del proyecto antes de abrir la ventana/página de inicio de sesión.. Esta opción es útil para una aplicación multilingüe, por ejemplo: Se puede mostrar una ventana o una página para solicitar el idioma de ejecución de la aplicación.



<a name="NOTE3_3"></a>


### Inicio de sesión automático en modo de prueba
<a name="inicio_sesion_automatico_modo_prueba_ELTPARAGRAPHE000089"></a>

Especifique el nombre de usuario y la contraseña que se utilizará en el modo [prueba automatizada](../Editeurs/2019025.md). Esta información se utilizará si se ejecuta una prueba automatizada en la aplicación.
<a name="NOTE3_4"></a>


### Uso de un directorio LDAP para autenticar a los usuarios
<a name="uso_directorio_ldap_para_autenticar_los_usuarios_ELTPARAGRAPHE000099"></a>

Si la empresa de su cliente utiliza un directorio LDAP, puede activar el Groupware Usuario en este directorio. Por lo tanto, la cuenta y la contraseña encontradas en el directorio LDAP se preguntarán automáticamente al usuario.

Hay dos modos disponibles:

- **Sin gestión de derechos**: En este caso, no se creará ningún archivo de groupware.. Si la aplicación se inicia en modo automático, el usuario deberá iniciar sesión. Si el usuario está guardado en el directorio LDAP, la aplicación se iniciará; de lo contrario, se cerrará. Un supervisor no puede configurar los derechos en las ventanas o páginas.

- **Con gestión de derechos**: Sólo los usuarios que se encuentran en el directorio LDAP pueden conectarse. Los derechos se pueden configurar en las ventanas o páginas.




Para obtener más información, consulte [Grupo de usuarios y directorio LDAP](../Editeurs/2018013.md).

Observación: El programa de grupo de usuarios funciona con el directorio Active. No funciona con openLDAP.

## "Ficha"Archivos
<a name="fichaarchivos_ELTTEXTE000331"></a>


### Contraseña de los archivos de datos
<a name="contrasena_los_archivos_datos_ELTPARAGRAPHE000120"></a>

Por Default, los archivos de datos que se encuentran en el Grupo de Usuarios tienen una contraseña específica. Esta contraseña es: ***"PCSGPW2001***".

Para cambiar esta contraseña, introduzca la nueva contraseña. Los caracteres escritos están ocultos.

**Observación**: Esta contraseña se utiliza cuando los archivos son re-indexados o cuando los archivos son abiertos por WDMAP.

Para obtener más información, consulte [Archivos del Groupware usuario](../Editeurs/2018002.md).
<a name="NOTE4_2"></a>


### Activación del historial de conexiones
<a name="activacion_del_historial_conexiones_ELTPARAGRAPHE000136"></a>

Si el historial de conexiones está habilitado:

- Se crea un archivo de datos adicional: GPW_CONNECTIONLOG. Este archivo contiene el historial de conexiones.

- Al configurar la aplicación, el supervisor puede acceder al historial de conexiones a través del botón "Historial de conexiones".. 
	Este historial se muestra en una ventana/página específica. Esta página sirve para identificar a todos los usuarios que se han conectado a una aplicación determinada durante un periodo de tiempo determinado.. Una tabla indica el número de conexiones por hora durante el día. Un botón está disponible para borrar el historial si es necesario.



<a name="NOTE4_3"></a>


### Tipo de base de datos
<a name="tipo_base_datos_ELTPARAGRAPHE000148"></a>

El Grupo de Usuarios maneja varias categorías de archivos:

- **los ficheros de datos utilizados para identificar a los usuarios**. Estos usuarios pueden utilizar varias aplicaciones. Por lo tanto, estos archivos pueden ser compartidos entre varias aplicaciones.

- **los ficheros de datos utilizados para gestionar los derechos de los usuarios** en las ventanas/páginas de la aplicación.




Estos archivos pueden ser:

- en formato HyperFileSQL Classic (por Default). Puede especificar el directorio en el que se creará cada categoría de archivos de usuario (específico de el groupware usuario) y se utilizará para la prueba de su aplicación.

- en formato HyperFileSQL Client/Server. Se deben especificar las características de la conexión al servidor HyperFileSQL así como el directorio de las diferentes categorías de archivos.




Para obtener más información, consulte [Los archivos de datos de groupware usuario](../Editeurs/2018002.md).

## "Ficha"LDAP
<a name="fichaldap_ELTTEXTE000367"></a>


### Características del servidor LDAP
<a name="caracteristicas_del_servidor_ldap_ELTPARAGRAPHE000170"></a>

Si el User Groupware fue configurado para utilizar un directorio LDAP, la pestaña LDAP se utiliza para configurar las características del servidor LDAP utilizado para la prueba (y en tiempo de ejecución).

La información solicitada es:

- Dirección del servidor

- Número de puerto del servidor.

- Versión del servidor LDAP contactado.




En el código, basta con utilizar [gpwOpenLDAPSession](../WDLang6/1000017055.md).

Para obtener más información, consulte [Groupware de usuario y LDAP](../Editeurs/2018013.md).

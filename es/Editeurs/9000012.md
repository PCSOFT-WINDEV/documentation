## Presentación
<a name="presentacion_ELTTEXTE000190"></a>
Como para cualquier aplicación, WINDEV permite crear un programa de instalación para aplicaciones Java. Este procedimiento de instalación solo está disponible para Windows. El programa de instalación es un ejecutable. Se puede utilizar para instalar los archivos necesarios, así como el Java Runtime Environment (JRE). Para otras plataformas, es necesario implementar los archivos manualmente.

Para abrir el asistente de creación de instalaciones, en la pestaña "Proyecto", en el grupo "Generación", despliegue "Procedimiento de instalación" y seleccione "Crear procedimiento de instalación".

## Despliegue de una aplicación Java (generada por WINDEV)
<a name="despliegue_una_aplicacion_java_generada_por_windev_ELTTEXTE000220"></a>


1. Instalar JRE (Java Runtime Environment) en el ordenador de destino.
	Observación: El JRE para Windows, Linux, Apple Mac OS X y Solaris puede descargarse de [http://www.oracle.com/technetwork/java/javase/downloads/index.html](http://www.oracle.com/technetwork/java/javase/downloads/index.html). 

2. copy el archivo generado por WINDEV en el ordenador objetivo.
	También debe copy todos los archivos utilizados por la aplicación (imágenes, archivos de texto, ...).

3. Si el Java framework de WINDEV no se incluyó en el archivo generado:
	copy el archivo wd28 0java.jar en el mismo directorio que el archivo generado por WINDEV.

4. Si la aplicación utiliza un controlador JDBC que no estaba incluido en el archivo generado:
	copy el archivo (o los archivos de clase) del controlador JDBC en el mismo directorio que el archivo generado por WINDEV.

5. Si la aplicación utiliza el motor HFSQL: 
	copy los siguientes elementos en el directorio del archivo generado por WINDEV:

	- el análisis (archivo.WDD).

	- los archivos de datos. 
			Recordatorio: En Linux y Mac, los archivos de datos HFSQL deben escribirse en minúsculas sin caracteres acentuados.

	- las bibliotecas nativas para acceder a HFSQL:

		- para una ejecución en Windows: wd28 0jav.dll, wd28 0hf.dll, wd28 0sql.dll y wd28 0vm.dll. Estos archivos están disponibles en el subdirectorio Programas del directorio de instalación de WINDEV.

		- para una ejecución en Linux: wd28 0jav.so, wd28 0hf.so, wd28 0sql.so, wd28 0vm.so. Estos archivos están disponibles en el subdirectorio "Programas\\Data" del directorio de instalación de WINDEV. 
						Observación: los módulos para acceder a Hyper File 5 así como el Conectores Nativos no están disponibles para Linux en esta versión. Debe utilizar HFSQL Classic o un acceso JDBC.





Observación: Mac OS X 10.9 mínimo.

## Ejecutar la aplicación
<a name="ejecutar_aplicacion_ELTTEXTE000244"></a>
En la mayoría de los casos, el archivo generado por WINDEV es un archivo autoejecutable (el manifiesto del archivo contiene la información necesaria para su ejecución).

Un doble clic en el archivo comprimido inicia su ejecución.

Sin embargo, es posible que el doble clic no funcione en algunos casos.

**Lista de casos conocidos en los que el doble clic no funciona**: 

- el ejecutable "Java" no es associated con la extensión JAR.

- el nombre del controlador JDBC es demasiado largo y no se ha copiado correctamente en el archivo manifest.mf generado por la herramienta para crear archivos jar.exe, lo que hace que el manifiesto sea inutilizable.




En este caso, el archivo puede ejecutarse utilizando el comando Line copiado en el archivo .bat de .

Este fichero es generado automáticamente por WINDEV al generar el archivo de la aplicación.

ATENCIÓN: Este comando Line fue generado para la computadora de compilación. Este Line puede tener que ser adaptado para el ordenador objetivo.<a name="NOTE3_2"></a>


### Ejemplo de comando Line: 
<a name="ejemplo_comando_line_ELTPARAGRAPHE000113"></a>


```txt
"C:\j2sdk1.4.2_04\bin\java.exe" -classpath MyProject.jar;
WD280java.jar;mysql-connector-java-3.0.14-production-bin.jar GWDPMyProject
```


Detalles del comando Line:


|   |   |
| --- | --- |
| "C:\\j2sdk1.4.2_04\\bin\\Java.exe" | ruta completa del ejecutable Java para ejecutar el archivo generado. |
| classpath MyProject.jar;wd280Java.jar;mysql-connector-Java-3.0.14-production-bin.jar | Lista de archivos (.JAR) necesarios para ejecutar la aplicación.<br><br>MyProject.jar es el nombre del archivo generado por WINDEV.<br><br>En este ejemplo, ni el controlador framework ni el controlador JDBC de MySQL se han incluido en el archivo generado.: se añaden al classpath, separados por ';' (punto y coma). |
| GWDPMyProject | Este es el nombre de la clase Java que contiene el método "main", punto de partida para la ejecución de la aplicación.<br>Este nombre es generado automáticamente por WINDEV a partir del nombre del proyecto.<br>Para más detalles, consulte la documentación del ejecutable "Java". |

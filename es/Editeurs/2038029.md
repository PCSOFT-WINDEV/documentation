


## Creación de una sucursal
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000149"></a>
Se utiliza una rama para crear una copy del proyecto actual con el fin de realizar modificaciones, adiciones, ... en dos versiones del mismo proyecto.

Por ejemplo: crear una rama le permite preparar una nueva versión de su aplicación mientras corrige los problemas en la versión desplegada. El proyecto principal será el proyecto correspondiente a la nueva versión y el proyecto de rama será el proyecto desplegado, en el que se realizará la corrección de errores.

Observación: Con portafolios de proyectos, puedes crear una rama en un set de proyectos en una sola operación. Para obtener más información, consulte [Portafolio de proyectos](../Editeurs/9500231.md). 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000173"></a>
**Para crear una rama**: 

1. Abra su proyecto.

2. En la pestaña "SCM", en el grupo "Proyecto", despliegue "Ramas" y seleccione "Crear una rama".

3. El asistente de creación de ramas se abre. Esta asistente le permite crear una copy de la rama del proyecto, localmente en su ordenador y en el repositorio SCM.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GDS_Creer_Branche%20-%20HC%20N%B0001.gif&type=thumb)


4. Especifique un nombre para la sucursal. Este nombre puede incluir letras, números, ... Este nombre es importante porque será utilizado:

	- para identificar los directorios de los proyectos afectados por la versión,

	- para identificar las diferentes versiones en la historia del proyecto, ....




5. Pase a la etapa siguiente.

6. Especifique el nombre del directorio a crear para la rama. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GDS_Creer_Branche%20-%20HC%20N%B0002.gif&type=thumb)
 Por Default:

	- se crea un nuevo directorio en la página web de SCM. Por Default, este directorio está compuesto por "Ramas\\&lt;Nombre de la rama&gt;\\&lt;Proyectos WX&gt;\\&lt;Nombre del proyecto&gt;". El proyecto SCM se duplicará en este directorio.

	- se crea un nuevo directorio localmente. Por Default, este directorio se crea en "Mis Proyectos.Ramas\\&lt;Nombre de la rama&gt;\\&lt;Nombre del proyecto&gt;". Este directorio contendrá el proyecto local de la sucursal.


 **Observación**: Los nombres propuestos son los del directorio Default. Para modificar estos nombres: 

- Visualizar las opciones del gestor de código fuente: en la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones del Administrador de Código Fuente". 

- En la pestaña "Ramas", especifique la ubicación de Default para las ramas. Esta ubicación se basa en la ubicación del proyecto Default especificada en la base de datos [opciones generales del editor](../Editeurs/2036001.md).

7. Pase a la etapa siguiente. 

8. Especifique el comment associated con la creación de la rama y seleccione las opciones para crear la rama: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GDS_Creer_Branche%20-%20HC%20N%B0003.gif&type=thumb)


	- **Use mi versión local como reference para la rama**: Se utiliza para crear la rama a partir de la versión de proyecto que se encuentra en SCM (recomendado) o a partir de la versión local.

- **Buscar y volver a crear las acciones en la sucursal**: Permite recrear todas las partes entre los diferentes proyectos de la misma rama.. Observación: esta opción no añade automáticamente en la rama el proyecto que tiene una parte. La opción vuelve a crear las acciones al añadir los otros proyectos.

- **Abrir el nuevo proyecto una vez creada la rama**: Permite abrir directamente el proyecto correspondiente a la rama. Se ejecutará una nueva instancia de WINDEV, WEBDEV o WINDEV Mobile.

9. Validar. Los proyectos especificados se crean localmente y en la SCM.

### Observaciones
<a name="observaciones_ELTPARAGRAPHE000087"></a>

- **Sub-ramas**
	Tiene la capacidad de crear sub-ramas (rama creada a partir de una rama).
	Las modificaciones realizadas en una subdivisión no se pueden recuperar directamente en la versión actualmente desarrollada.. Las modificaciones de la sub-rama deben ser recuperadas en la rama y luego las modificaciones de la rama deben ser recuperadas en la versión actualmente desarrollada.

- **Creación de una sucursal a partir de entonces**
	Tiene la posibilidad de crear ramas en una versión anterior del proyecto utilizando el historial.
	Para crear una sucursal a partir de entonces: 

	1. Abre la historia del proyecto.

	2. Seleccione la versión del historial que desee.

	3. Haz clic en "Crear una sucursal".

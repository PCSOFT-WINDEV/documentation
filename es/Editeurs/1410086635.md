
## Instalar un WEBDEV Cluster
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000189"></a>
Un cluster WEBDEV es una set de servidores (o granja de servidores). Cada servidor contiene todos sus sitios y servicios web. 
Esto permite soportar un número ilimitado de conexiones a estos sitios y servicios web.

Esta Page presenta los diferentes pasos para instalar un cluster WEBDEV y sus nodos. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Etapas de instalación del WEBDEV Cluster
<a name="etapas_instalacion_del_webdev_cluster_ELTTEXTE000213"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000024"></a>

La instalación de un cluster WEBDEV implica lo siguiente: 

1. Instalación del Servidor de Aplicaciones WEBDEV en modo "Cluster" en un primer ordenador (Ordenador 1). 

2. Instalación del Servidor de Aplicaciones WEBDEV en modo "Nodo Cluster" en un segundo ordenador (Ordenador 2). 

3. Vinculación del clúster con el nodo del clúster. 





### 1. Instalación del servidor de aplicaciones WEBDEV en modo cluster (ordenador 1)
<a name="1_instalacion_del_servidor_aplicaciones_webdev_modo_cluster_ordenador_1_ELTPARAGRAPHE000033"></a>

Las etapas de la instalación son las siguientes:

1. Seleccione el idioma de instalación de WEBDEV Application Server y valide. 

2. Seleccione "Instalar versión completa" e introduzca la clave de activación. Pase a la etapa siguiente del asistente. 

3. Acepte los términos de la licencia. Pase a la etapa siguiente del asistente. 

4. Seleccione la ruta de instalación del servidor de aplicaciones (ruta predeterminada: "C:\\WEBDEV 28"). Pase a la etapa siguiente del asistente. 

5. Seleccione el modo de funcionamiento de WEBDEV Application Server: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install%20Serveur%20WB%20Cluster%20-%20HC%20N%B0001.gif)


	- Para instalar un servidor de aplicaciones WEBDEV estándar, seleccione "Servidor autónomo".. 

- Para instalar un cluster de servidores WEBDEV, seleccione "Cluster".. Seleccione la última opción y pase al siguiente paso de la asistente. 

6. Seleccione el modo de instalación del nodo de clúster. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install%20Serveur%20WB%20Cluster%20-%20HC%20N%B0002.gif)
Es posible: 

	- Instalar un nuevo clúster. 

	- Unirse a un clúster. 


 Seleccione "Nuevo cluster" y vaya al paso siguiente de la asistente. 

7. En el siguiente paso, puede describir el primer nodo del clúster que se va a crear. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install%20Serveur%20WB%20Cluster%20-%20HC%20N%B0003.gif)
 Indicar el nombre DNS del nodo. Este nombre será utilizado por los otros nodos del cluster para Contact este nodo. Pase a la etapa siguiente del asistente. 

8. La siguiente etapa permite especificar el directorio raíz para la implementación de sitios y Webservices. Pase a la etapa siguiente. 

9. En la siguiente etapa, puede definir la configuración avanzada de WEBDEV Application Server. es posible: 

	- Utilizar parámetros predeterminados: Los valores de los parámetros avanzados son definidos por Default. También puede cambiar estas opciones después de la instalación. 

	- Definir manualmente los valores de los parámetros. En este caso, la asistente permite definir la configuración avanzada del Servidor de Aplicaciones WEBDEV. 




10. Pase a la etapa siguiente del asistente. 

11. Especifique la cuenta de Windows que se utilizará como administrador de WEBDEV. Esta cuenta también se llamará "Cuenta de alojamiento". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install%20Serveur%20WB%20Cluster%20-%20HC%20N%B0004.gif)

	Para un clúster, tendrá que crear una nueva cuenta de Windows. El Servidor de Aplicaciones WEBDEV debe conocer la contraseña del usuario para conectarse a los distintos nodos del cluster. Pase a la etapa siguiente.  

12. Seleccione los sitios de administración remota que se instalarán con el servidor de aplicaciones WEBDEV. Puede instalar: 

	- El sitio de administración remota del servidor de aplicaciones WEBDEV. Este sitio le permite gestionar fácilmente todos los sitios y servicios web desplegados, configurar el servidor de aplicaciones, etc.

	- Sitio de administración HFSQL: Este sitio permite gestionar remotamente las bases de datos HFSQL. 


 Seleccione las opciones deseadas y vaya al siguiente paso. 

13. La siguiente etapa permite gestionar el alojamiento de sitios SaaS. La activación de SaaS instala automáticamente un sitio y un Webservice para administrar los sitios SaaS. Para obtener más información, consulte [SaaS](../WDLang3/9500213.md). Valide la etapa.

14. El siguiente paso permite definir la necesidad de una conexión segura para acceder a todos los sitios de administración. Si selecciona una conexión segura, especifique el nombre de dominio utilizado para acceder a los sitios de administración. 

15. En el siguiente paso, puede instalar el controlador ODBC para HFSQL. Este controlador permite el acceso directo a los archivos HFSQL Classic y Cliente/Servidor desde una aplicación externa.  

16. Valide las diferentes etapas de la instalación.

17. Se comprueba el servidor web. Si el equipo contiene varios servidores virtuales, seleccione los servidores web en los que debe instalarse WEBDEV Application Server. Puede conservar el sitio web predeterminado. Valide.

18. El servidor de aplicaciones WEBDEV en modo clúster está instalado. 

19. Al final de la instalación, asistente le pide que registre el servidor de aplicaciones WEBDEV. 

20. La asistente muestra diferentes opciones en el siguiente paso: seleccione las opciones deseadas y conserve la opción "Iniciar el administrador de WEBDEV". Valide.

21. El "Servidor de aplicaciones WEBDEV" se configura automáticamente. Los sitios WEBDEV pueden implementarse inmediatamente. 

22. El administrador WEBDEV y el Centro de Control de Alojamiento se inician automáticamente.





### 2. Instalación de un nodo del clúster (ordenador 2)
<a name="2_instalacion_nodo_del_cluster_ordenador_2_ELTPARAGRAPHE000088"></a>

Para instalar un nodo de Cluster:

1. Seleccione el idioma de instalación de WEBDEV Application Server y valide. 

2. Seleccione "Instalar versión completa" e introduzca la clave de activación. Pase a la etapa siguiente del asistente. 

3. Acepte los términos de la licencia. Pase a la etapa siguiente del asistente. 

4. Seleccione la ruta de instalación del servidor de aplicaciones (ruta predeterminada: "C:\\WEBDEV 28"). Pase a la etapa siguiente del asistente. 

5. Seleccione el modo de funcionamiento de WEBDEV Application Server: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install%20Serveur%20WB%20Noeud%20Cluster%20-%20HC%20N%B0001.gif)


	- Para instalar un servidor de aplicaciones WEBDEV estándar, seleccione "Servidor autónomo".. 

- Para instalar un cluster de servidores WEBDEV, seleccione "Cluster".. Seleccione la última opción y pase al siguiente paso de la asistente. 

6. Seleccione el modo de instalación del nodo de clúster. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install%20Serveur%20WB%20Noeud%20Cluster%20-%20HC%20N%B0002.gif)
Es posible: 

	- Instalar un nuevo clúster. 

	- Unirse a un clúster. 


 Seleccione "Unirse a un cluster existente" y vaya al siguiente paso de asistente. 

7. La siguiente etapa permite especificar el directorio raíz para la implementación de sitios y Webservices. Pase a la etapa siguiente. 

8. En el siguiente paso, puede instalar el controlador ODBC para HFSQL. Este controlador permite el acceso directo a los archivos HFSQL Classic y Cliente/Servidor desde una aplicación externa.  

9. Valide las diferentes etapas de la instalación.

10. Se comprueba el servidor web. Si el equipo contiene varios servidores virtuales, seleccione los servidores web en los que debe instalarse WEBDEV Application Server. Puede conservar el sitio web predeterminado. Valide.

11. El servidor de aplicaciones WEBDEV en modo de nodo clúster está instalado. 

12. Al final de la instalación, asistente le pide que registre el servidor de aplicaciones WEBDEV. 

13. La asistente muestra diferentes opciones en el siguiente paso: seleccione las opciones deseadas y conserve la opción "Iniciar el administrador de WEBDEV". Valide.

14. El "Servidor de aplicaciones WEBDEV" se configura automáticamente. Se muestra una ventana para añadir el servidor a un clúster (ver más abajo). 





### Vinculación del clúster con un nodo del clúster
<a name="vinculacion_del_cluster_con_nodo_del_cluster_ELTPARAGRAPHE000120"></a>

Después de instalar un nodo de cluster, el servidor de aplicaciones WEBDEV muestra la siguiente ventana: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Install%20Serveur%20WB%20Noeud%20Cluster%20-%20HC%20N%B0004.gif)
 Si el nodo no es relacionado con el cluster, esta ventana aparecerá cada vez que se inicie el administrador de WEBDEV. 

Para enlazar el cluster (Ordenador 1) y el nodo del cluster (Ordenador 2):

1. copy la seguridad Token que aparece en la ventana al final de la instalación del nodo de cluster (Ordenador 2). 

2. Abra el administrador de WEBDEV en el ordenador 1 utilizando uno de los siguientes métodos: 

	- Administrador remoto iniciado a través de una Address como "http://cluster_server_Address/WDWebAdmin". En la pestaña "Cluster", seleccione "Añadir un nodo".. En la ventana emergente que se abre, introduzca el nombre DNS del nodo, el token de seguridad y la description del nodo. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Cluster_Admin_wb_jeton.GIF&type=thumb)


	- Administrador local del ordenador 1 (o de uno de los ordenadores del cluster). En la pestaña "Cluster: 

	- haga clic en "Añadir un nodo a este clúster".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Cluster_Admin.GIF&type=thumb)




Una vez que el nodo se relacionado al cluster, la ventana del administrador de WEBDEV que muestra el Token desaparece automáticamente (en el el equipo correspondiente al nodo). 

Observación: En un clúster WEBDEV, los nodos se sincronizan automáticamente entre sí. Los nodos sincronizan periódicamente la configuración del servidor de aplicaciones WEBDEV, así como la configuración y el contenido de los sitios y servicios web desplegados.

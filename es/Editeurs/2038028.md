## Presentación
<a name="presentacion_ELTTEXTE000138"></a>
La búsqueda en el historial se utiliza para realizar una búsqueda avanzada en el historial de operaciones realizadas en SCM.. El resultado de la búsqueda puede ser: 

- Se muestra en la pantalla.

- Exportación en diferentes formatos: HFSQL, documento XLSX, documento DOCX, al portapapeles, ....




Este tipo de búsqueda también se puede realizar a través de una línea de comandos. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo realizar una búsqueda a través del administrador de SCM?
<a name="¿como_realizar_una_busqueda_traves_del_administrador_scm_ELTTEXTE000162"></a>
Para realizar una búsqueda en el historial: 

1. Iniciar el administrador de SCM. 

2. En la pestaña "Herramientas", en el grupo "Buscar", haga clic en "En el historial". 

3. Aparece la ventana utilizada para realizar una búsqueda multicriterio en el historial..

<a name="NOTE3_1"></a>


## Realizar una búsqueda a través de una línea de comandos
<a name="realizar_una_busqueda_traves_una_linea_comandos_ELTTEXTE000186"></a>
Se puede realizar una búsqueda en el historial de SCM a través de una línea de comandos, utilizando la siguiente sintaxis: 


```txt
WDGDSAdmin.exe /EXPORT 
		/BASE=<Address of the repository> 
		/LOGIN=<Login> 
		/PASS=<Password> 
		/DIR=<Name of remote directory> 
		/EXPORTFILE=<Full name of export file>
```
En este código, &lt;Dirección del repositorio&gt; tiene el siguiente formato:

- Para HFSQL Client/Server: 
	
	```txt
	"CS:\\<IP Address of server:<Server port>\<Name of SCM repository>"
	```


- Para HFSQL Classic: se debe especificar el ruta local del repositorio. Por ejemplo: 
	
	```txt
	"D:\PC SOFT\Local SCM repository\"
	```





Esta línea de comandos acepta opciones de filtrado adicionales:

- /COMENTARIOS=1: Utilizado para obtener comentarios solamente. 

- /SINCELASTINSTALL=1: Utilizado para realizar las modificaciones desde la última generación de configuración.. 

- /MINDATE=YYYMMDDHHMMSS: Permite definir la fecha mínima de búsqueda (incluida en la búsqueda). Por ejemplo: 2017010100000.

- /MAXDATE=YYMMDDHHHHMMSS: Permite definir la fecha máxima de búsqueda (incluida en la búsqueda). Por ejemplo: 2018010100000.

- /BUGID=999: Utilizado para obtener las modificaciones relacionado al incidente 999. 

- /TASKID=999	: Se utiliza para obtener las modificaciones relacionado a la tarea 999. 

- /AUTHOR=Inicio de sesión: Utilizado para obtener las modificaciones de &lt;Login&gt; developer. 

- /FILENAME=xxx: Se usa para realizar una búsqueda sólo en el archivo &lt;xxx&gt;.. Este parámetro depende de "/FILENAMEFILTER".. 

- /FILTRO DE NOMBRE = 0/1/2/3: Se utiliza para filtrar el nombre del archivo de búsqueda: 

	- 0 = Comienza con xxx,

	- 1 = Termina en xxx,

	- 2 = Contiene xxx,

	- 3 = Es igual a xxx


donde xxx fue especificado en el parámetro /FILENAME

- /ETIQUETA=aaaa: Se utiliza para realizar una búsqueda en los elementos con la etiqueta &lt;yyyy&gt; solamente.. Este parámetro depende de "/LABELFILTER".. 

- /LABELFILTER=0/1/2/3: Se utiliza para filtrar el nombre de la etiqueta de búsqueda: 

	- 0 = Comienza con yyyy,

	- 1 = Termina por yyyy,

	- 2 = Contiene yyyyy,

	- 3 = Es igual a yyyy,

	- 4 = todos los elementos con etiqueta,


donde se especificó yyyy en el parámetro "/LABEL".

- /COMENTARIO=zzzz: Se utiliza para obtener los elementos cuyo comentario es &lt;zzzz&gt;&gt;.. Este parámetro depende de "/LABELFILTER".

- /FILTRO DE COMENTARIO=0/1/2/3: Se utiliza para filtrar el contenido de los comentarios: 

	- 0 = Comienza con zzz,

	- 1 = Termina en zzz,

	- 2 = Contiene zzz,

	- 3 = Es igual a zzz,


donde zzzz se especificó en el parámetro "/COMMENT".

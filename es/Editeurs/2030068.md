
## Publicación dinámica
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000144"></a>
"Servicio dinámico" es una técnica en la que el mismo Address (URL) conduce a 2 páginas diferentes: una página para PC y una página para móviles.

La página adecuada se mostrará según el dispositivo (PC o móvil) que acceda a la página.

Se trata de una solución alternativa (incluso complementaria) a "[Responsive Web Design](../WDChamp/9000165.md)" para hacer que un sitio estándar sea "mobile friendly". Con "Dynamic serving", no es necesario modificar las páginas existentes: todo lo que tienes que hacer es añadir algo para los dispositivos móviles.

Esta técnica se beneficia de un excelente SEO de Google.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Dynamic%20serving.jpg)


Observaciones: "Servir dinámicamente" está disponible para las páginas dinámicas y para las páginas de Active WEBDEV (no autónomas).

## Principio y modo de funcionamiento
<a name="principio_modo_funcionamiento_ELTTEXTE000168"></a>
El sitio incluye 2 juegos de páginas: las páginas para el ordenador y las páginas para el dispositivo móvil. Estas páginas son associated 2 por 2. 

Cuando se teclea la página Address en el navegador, se detecta automáticamente el entorno utilizado y se muestra la página correspondiente (ordenador o dispositivo móvil) en el navegador. La Address que aparece en el navegador no cambia: sólo el contenido mostrado se adapta al sistema. 

Todos los enlaces y las páginas abiertas en el código se redirigen automáticamente a la página correspondiente en la plataforma actual.. 



**Observaciones**: 

- La detección del entorno se realiza según las recomendaciones de Google: las pastillas no se consideran dispositivos móviles. 

- Las páginas son identificadas por Google como en modo "Servicio dinámico".. Por lo tanto, las diferentes versiones de la página están optimizadas. Si las páginas móviles se desarrollan correctamente, el sitio se considera como "Mobile friendly".. 

- Precaución, el Servidor Dinámico no está disponible en un servidor PHP. Sin embargo, para un sitio mixto que mezclara dinámica y PHP, las Páginas Activas de WEBDEV que reference PHP deben utilizar el Servido Dinámico. En este caso, las páginas PHP están listadas por Dynamic Serving.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000192"></a>


### Configuración
<a name="configuracion_ELTPARAGRAPHE000047"></a>

Para utilizar "Dynamic serving" en un sitio WEBDEV: 

1. Cree las páginas específicas de la interfaz móvil en su proyecto. 

2. En la pestaña "Proyecto", en el grupo "Web", amplíe "SEO" y seleccione "Servidor dinámico".

3. La ventana que se muestra incluye tres secciones: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Dynamic%20serving%20-%20HC%20N%B0001.gif&type=thumb)


	- 1. La lista de páginas comunes (mostradas independientemente de la plataforma). 

- 2. La lista de páginas para el ordenador.

- 3. La lista de páginas para móviles.

4. Para realizar asociaciones de páginas:

	- En la lista "Páginas comunes", seleccione la página correspondiente a la página "Versión ordenador".. 

	- Arrastre y suelte la página en la sección "Versión Computador" o haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Dynamic%20serving%20-%20HC%20N%B0001%201.gif). 

	- La sección "Versión móvil" correspondiente aparece en rojo.  

	- En la lista "Páginas comunes", seleccione la página correspondiente a la página "Versión móvil. 

	- Arrastre y suelte la página a la sección "Versión móvil" o haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Dynamic%20serving%20-%20HC%20N%B0001%202.gif).







Cuando se realizan todas las asociaciones, valide la ventana. 


### Observaciones
<a name="observaciones_ELTPARAGRAPHE000074"></a>

- Las asociaciones se guardan en las páginas. Si sus páginas se encuentran en el SCM, se solicita la salida de las páginas móviles.. 

- Las páginas de associated deben tener el mismo prototipo. Si los parámetros esperados no son los mismos, las páginas se muestran en una Background roja en la ventana de gestión de las asociaciones. 

- Las páginas WEBDEV activas independientes no pueden ser associated. 

- Los controles utilizados desde el exterior de la página (sintaxis: &lt;Nombre de página&gt;.&lt;Nombre de control&gt;) deben estar en las dos páginas de la asociación. 

- [DynamicServingConfigure](../WDLang2/1000021994.md) se utiliza para crear un enlace "Sitio móvil" o un enlace "Sitio para ordenador" que fuerza la visualización del sitio para un entorno determinado. 





### Probar un sitio de "servicio dinámico"
<a name="probar_sitio_servicio_dinamico_ELTPARAGRAPHE000085"></a>

Por Default, cuando se ejecuta la prueba de una página o sitio, la versión "Computer" del sitio se inicia automáticamente. 

Para ejecutar la prueba de un proyecto o de una página en modo "Móvil", basta con activar la opción de prueba en modo móvil.: 

1. En los botones de acceso rápido, amplíe el Combo Box que se encuentra al lado del Icon "GO" solicitado (prueba de página o prueba de proyecto): ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Dynamic_serving_1%20-%20HC%20N%B0003.gif)

2. Selecciona "Depurar la versión móvil (Dynamic Serving)".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Dynamic_serving_1%20-%20HC%20N%B0002.gif)


3. La siguiente página o prueba del proyecto se ejecutará en modo móvil. Para volver al modo "Ordenador", basta con seleccionar de nuevo "Depurar la versión móvil (Dynamic Serving)" para desactivarla.. 




Atención: Esta opción está disponible en el modo GO para la página y el sitio.. Puede activarse/desactivarse para la página GO y/o para el sitio GO solamente..

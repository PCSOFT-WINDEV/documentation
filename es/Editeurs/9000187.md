


## Errores de estándar de programación
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000228"></a>
Los errores estándar de programación le informan de los posibles problemas detectados al guardar ventanas, páginas, informes o al recompilar el proyecto: 

- Funciones conservadas para compatibilidad.

- Acceso a HFSQL desde la interfaz.

- Uso de las funciones de objeto WLanguage. 




Por Default, los errores estándar de programación no están habilitados para cada nuevo proyecto creado.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000252"></a>


### Activar el error estándar de programación
<a name="activar_error_estandar_programacion_ELTPARAGRAPHE000024"></a>

Se pueden habilitar los errores estándar de programación: 

- del proyecto description. 

- desde el panel de errores de compilación. 




Para habilitar los errores estándar de programación del proyecto description:

1. Abra la ventana de descripción del proyecto (en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción"). 

2. En la pestaña "Compilación", en "Estándar de programación", marque las opciones deseadas: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Compilation_Norme%20-%20HC%20N%B0001.gif)


	- **Sintaxis estándar o prefijada**: Es posible: 

	- Permita ambos...: en este caso, no se mostrará ningún error estándar de programación. 

	- Indicar uso de sintaxis con prefijo: Aparecerá un error estándar de programación si se utiliza la sintaxis del prefijo (por ejemplo, "ListControl.Add"). 

	- Indicar uso de sintaxis estándar: Aparecerá un error estándar de programación si se utiliza la sintaxis estándar en lugar de una sintaxis de prefijo disponible. 

3. Validar. 




Para mostrar los errores estándar de programación desde el panel de errores de compilación:

1. Mostrar el panel de errores de compilación: en la pestaña "Inicio", en el grupo "Entorno", amplíe "Paneles", seleccione "Paneles" y, a continuación, seleccione "Errores de compilación". 

2. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Compilation_Norme%20-%20HC%20N%B0002.gif).




Se comprueban los errores estándar de programación:

- siempre que se guarde una ventana, una página o un reporte.

- en el momento de recompilar el proyecto.



<a name="NOTE2_2"></a>


### Corrección de un error estándar de programación
<a name="correccion_error_estandar_programacion_ELTPARAGRAPHE000122"></a>

Los errores estándar de programación se enumeran en el panel de errores de compilación.

Para corregir los errores estándar de programación:

1. Haga doble clic en el error que se muestra en el panel. El elemento que contiene la control o el código donde se produce el problema se abre en el editor. Se muestra la description del control correspondiente.

2. Arregla el problema.



<a name="NOTE2_3"></a>


### Anular un error estándar de programación
<a name="anular_error_estandar_programacion_ELTPARAGRAPHE000134"></a>

Por Default, el estándar de programación definido para las opciones del proyecto se aplica a todos los elementos del proyecto. Sin embargo, puede override esta norma para cada elemento (ventana, reporte, página, Component interno ). 

**A override el estándar de programación de una ventana o página**:

1. Abrir la ventana del elemento description. 

2. En la pestaña "Detalles", haga clic en "Estándar de programación". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Compilation_Norme%20-%20HC%20N%B0003.gif&type=thumb)


3. En la ventana que se muestra, seleccione el modo Process de cada opción: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Compilation_Norme%20-%20HC%20N%B0004.gif)


	- Como el proyecto: el estándar de programación que se aplica es el del proyecto. 

- Indicar: el estándar de programación se aplica siempre a este elemento. 

- No indicar: el estándar de programación nunca se aplica a este elemento. 

4. Validar la ventana estándar de programación. 

5. Validar la ventana description. 




**Para sobrescribir el estándar de programación de un reporte**:

1. Abre la ventana de reporte description. 

2. En la pestaña "Opciones", haga clic en "Estándar de programación". 

3. En la ventana que se muestra, seleccione el modo Process de cada opción: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Compilation_Norme%20-%20HC%20N%B0004.gif)


	- Como el proyecto: el estándar de programación que se aplica es el del proyecto. 

- Indicar: el estándar de programación se aplica siempre a este elemento. 

- No indicar: el estándar de programación nunca se aplica a este elemento. 

4. Validar la ventana estándar de programación. 

5. Validar el reporte description. 



**A override la norma de programación de un Component interno**: 

1. Abrir el description del Component interno (opción "description" del Component interno que aparece en el explorador de proyectos). 

2. En la pestaña "Compilación", especifique las opciones de la norma de programación que serán anuladas: 

	- Como el proyecto: el estándar de programación que se aplica es el del proyecto. 

	- Indicar: el estándar de programación se aplica siempre a este elemento. 

	- No indicar: el estándar de programación nunca se aplica a este elemento. 




3. Validar la description de la Component interna.

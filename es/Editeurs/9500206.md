


## Seguimiento estricto de un proyecto
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000120"></a>
A través de Centro de Seguimiento de Proyectos, WINDEV, WEBDEV y WINDEV Mobile permiten definir las diferentes acciones necesarias para crear un proyecto:

- Gestión de requisitos, 

- Gestión de tareas, 

- Manejo de errores,

- Gestionar las reglas de negocio.




Estos diferentes elementos se utilizan para programar y monitorear la evolución y la creación de un proyecto.

Durante el desarrollo, el colaborador puede especificar la tarea actual (desde la Centro de Seguimiento de Proyectos, por ejemplo).

**En modo "estricto", los desarrolladores no pueden modificar un elemento del proyecto (ventana, código, consulta,...) sin haber especificado previamente la tarea afectada por estas modificaciones..**

El modo estricto permite al director de proyecto y al desarrollador controlar fácilmente su trabajo.. Además, el estado de avance de los requisitos se aproxima a la realidad: el estado de avance de un proyecto es siempre conocido.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000144"></a>


### Activar modo estricto
<a name="activar_modo_estricto_ELTPARAGRAPHE000031"></a>

Para activar el modo estricto:

1. Abrir el proyecto description: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. Pantalla: 

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) la pestaña "Opciones".




3. Marque "Activar modo estricto".

4. Valide.



<a name="NOTE2_2"></a>


### Modo de funcionamiento del modo estricto
<a name="modo_funcionamiento_del_modo_estricto_ELTPARAGRAPHE000062"></a>

Cuando se activa el modo estricto, si el desarrollador abre un elemento del proyecto que no es associated con la tarea actual, este elemento se abre en modo de sólo lectura y se muestra un panel específico al desarrollador: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=mode_strict.gif)


Haciendo clic en el botón "Gestionar las tareas", el desarrollador puede elegir:

- asociar la tarea actual con el elemento,

- habilitar una de las tareas de associated con el elemento,

- asociar una nueva tarea al elemento y habilitarla,

- asociar una tarea existente con el elemento y habilitarla,

- no realizar ninguna acción. En este caso, el elemento no puede ser modificado.

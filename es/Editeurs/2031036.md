## ¿Qué ventanas se deben crear?
<a name="¿que_ventanas_deben_crear_ELTTEXTE000357"></a>


### Los ficheros a crear
<a name="los_ficheros_crear_ELTPARAGRAPHE000043"></a>

Recordatorio : Para construir un patrón RAD utilizado para crear un proyecto completo, se requieren los siguientes archivos:

- Un formulario para RADFileA.

- Un formulario para RADFileB. Un cuadro combinado y un botón Vision Plus (o un cuadro combinado emergente) deben permitirle seleccionar un elemento que se encuentre en el archivo RADFileA.

- Un formulario para RADFileC. Un cuadro combinado y un botón Vision Plus (o un cuadro combinado emergente) deben permitirle seleccionar un elemento que se encuentre en el archivo RADFileB.

- Una tabla para RADFileA.

- Una tabla para RADFileB.

- Un tabla para RADFileC.

- Una ventana de relación RADFileB/RADFileD.




Estos diferentes elementos deben contener botones de enlace. También tiene la posibilidad de proporcionar el menú principal de la aplicación.
<a name="NOTE2_2"></a>


### Diferencia entre Móvil y Smartphone
<a name="diferencia_entre_movil_smartphone_ELTPARAGRAPHE000060"></a>

Crear una aplicación para Móvil y una aplicación para Smartphone son dos cosas diferentes. En un Smartphone, la pantalla es más pequeña y un menú desplegable le permite validar (o no) las pantallas.

Esta diferencia también se aplica a los patrones RAD. Esta página de ayuda explica cómo crear una ventana de formulario para Móvil y para Smartphone.
<a name="NOTE2_3"></a>


### Estándar a seguir
<a name="estandar_seguir_ELTPARAGRAPHE000069"></a>

Esta página de ayuda presenta las diferentes reglas que deben aplicarse al crear las páginas y ventanas del patrón RAD. Estas reglas se han utilizado para crear los diferentes patrones suministrados con WINDEV Mobile 28.

Estas reglas son recomendaciones. Puede implementar y utilizar su propio estándar.

## Las ventanas de formularios para móviles
<a name="las_ventanas_formularios_para_moviles_ELTTEXTE000393"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000082"></a>

Una ventana de formulario incluye los siguientes elementos:

- A supercontrol : Este supercontrol contiene los controles del formulario a mostrar. Este supercontrol es relacionado al archivo a mostrar.

- Los controles de los botones utilizados para realizar las principales acciones : "Validar" e "Imprimir" si es necesario.

- Opciones de enlace para visualizar los cuadros correspondientes, por ejemplo (sólo para los formularios relativos al RADFileA y al RADFileB).

- Controles del botón Vision Plus utilizados para seleccionar elementos de otro archivo de datos en una ventana específica (sólo para los formularios relativos a RADFileB y RADFileC)




Ejemplo de ventana de formulario para el patrón RAD simple : <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Pattern_RAD_WMFiche.gif)

### Creación de una ventana de formulario
<a name="creacion_una_ventana_formulario_ELTPARAGRAPHE000097"></a>

Para crear una ventana de formulario:

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- Aparece la ventana de creación : haga clic en "Ventana", y luego en "Ventana". El asistente de creación de ventanas se abre. 

	- Seleccione "Wizard" (Asistente). 

	- En la asistente, mantenga las opciones de la Default.




2. En la ventana de description, introduzca la siguiente información:

	- El nombre de la ventana. Este nombre debe contener necesariamente el nombre del archivo que aparece en el formulario (algunos ejemplos : WIN_Form_RADFileA, WIN_For_RADFileA, ....).

	- El título de la ventana. Si el título debe mostrar el nombre del archivo, utilice el nombre del archivo "RADFileXX" (por ejemplo, "RADFileA : Formulario de entrada").




3. Grabe la ventana.

4. Crear un supercontrol en la ventana : en la pestaña "Creación", en el grupo "Contenedores", haga clic en "Supercontrol". Este supercontrol mostrará los controles que se encuentran en el formulario. Posicione esta control en la ventana y defina su tamaño. Las barras de desplazamiento se mostrarán automáticamente si algunos de los controles no caben en el tamaño especificado.

5. En la ventana del supercontrol description, especifique el nombre del supercontrol. Este nombre debe ser necesariamente SC_Form (SC_ es el prefijo definido en el código de estilo : no es necesario si no se utiliza un estilo de código).

6. En la pestaña "Contenido" de la ventana del supercontrol description, especifique:

	- el archivo navegado : RADFileXX ("RADFileA" por ejemplo).

	- el artículo consultado : IDRADFileXX ("IDRADFileA" por ejemplo).




7. Cree y posicione los diferentes botones de su ventana (validación, impresión, navegación, ....). Estos botones pueden tener cualquier nombre.
	Observación : En un patrón RAD móvil, no hay un botón Cancel para no abarrotar las ventanas. El usuario está haciendo clic en el botón para cerrar la ventana.
	Atención : si el usuario puede aplicar un plantilla a las ventanas creadas por RAD, debe definir el tipo de botón que se debe utilizar en el plantilla. Esta información se puede definir en el modificador (categoría"Plantilla").

8. Caso de ***botón de validación*** (botón "OK" o "Validate" por ejemplo).
	Para gestionar simplemente el botón de validación, tiene la posibilidad de utilizar una acción predefinida. Le recomendamos que utilice la acción "Cerrar la ventana y devolver un valor", con "Valor devuelto : el valor especificado en el código por ..ReturnedValue".
	En el código del botón de validación, le aconsejamos que habilite la gestión de errores de automatic ("Si el error : ", con el automatic Process "Mostrar el error y volver a editar").
	El código del botón de validación puede ser (para un formulario en RADFileA):
	
	```wl
	// Read the information entered 
	ScreenToFile()
	// If it's a new record
	IF RADFileA..NewRecord = True THEN 
		// Add it
		HAdd(RADFileA) 
	ELSE 
		// Modify it 
		HModify(RADFileA) 
	END
	// Return value of the window 
	MyWindow..ReturnedValue = True
	```


9. Introduzca el código de inicialización de la ventana : Veamos un ejemplo : 
	
	```wl
	// Print the order form
	FileName = fDataDir() + [fSep()] + StringBuild("ORF_%1.PDF", OrderNumber)
	iDestination(iGenericPDF, FileName)
	iPrintReport(RPT_OrderForm, OrderNumber, OrderNumber)
	
	// Check the file existence
	IF fFileExist(FileName) = False THEN
		Error("The PDF file was not generated")
		RETURN
	END
	
	// Confirmation to display the PDF
	IF YesNo(Yes, "Display the generated order form (PDF format)?") = Yes THEN	
		// Display the generated order form
		ShellExecute(FileName)
	ELSE
		iPrintPDF(FileName)
	END
	```

	Este código se utiliza para mostrar el valor de los registros en el formulario y para devolver un valor.

10. Añadir la [opciones del menú de enlaces](../Editeurs/2031039.md) si es necesario.

11. Añada la [Botones Vision Plus](../Editeurs/2031040.md) si es necesario (para los formularios RADFileB y RADFileC).



<a name="NOTE3_3"></a>


### Caso de los botones de impresión
<a name="caso_los_botones_impresion_ELTPARAGRAPHE000162"></a>

Un patrón RAD puede contener botones utilizados para iniciar una impresión. El código de estos botones (o la opción preestablecida associated) debe permitirle iniciar la impresión del reporte.

Los informes deben ser creados en el proyecto del patrón RAD:

- El nombre de estos informes debe contener RADFileX. Para una mejor legibilidad, le recomendamos que utilice en nombre de la reporte:

	- "Detalles" para los informes del formulario,

	- "Lista" para los informes de tabla.




- Este nombre debe ser usado en el código del botón "Imprimir" (o en la acción preestablecida associated con el botón).




ATENCIÓN : No hay necesidad de personalizar los informes encontrados en su proyecto del patrón RAD. En efecto, durante la generación, sólo se conservará el nombre de la reporte.. Si el botón "Imprimir" se encuentra en una ventana "Form", la reporte impresa será un Formulario reporte. Si el botón "Imprimir" se encuentra en una ventana "Tabla", la reporte impresa será una Tabla reporte.

- Opciones de menú de enlace utilizadas para visualizar las tablas correspondientes, por ejemplo (sólo para los formularios que afectan a RADFileA y RADFileB).

### Creación de una ventana de formulario para Smartphone
<a name="creacion_una_ventana_formulario_para_smartphone_ELTPARAGRAPHE000213"></a>

Para crear una ventana de formulario para Smartphone : 

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- Aparece la ventana de creación : haga clic en "Ventana", y luego en "Ventana". El asistente de creación de ventanas se abre. 

	- Seleccione "Wizard" (Asistente). 

	- En la asistente, mantenga las opciones de la Default.




2. En la ventana de description, introduzca la siguiente información:

	- El nombre de la ventana. Este nombre debe contener necesariamente el nombre del fichero visualizado en el formulario (algunos ejemplos : WIN_Form_RADFileA, WIN_For_RADFileA, ....).

	- El título de la ventana. Si el título debe mostrar el nombre del archivo, utilice el nombre del archivo "RADFileXX" (por ejemplo, "RADFileA : Forma").




3. Grabe la ventana.

4. Crear un supercontrol en la ventana : en la pestaña "Creación", en el grupo "Contenedores", haga clic en "Supercontrol". Este supercontrol mostrará los controles que se encuentran en el formulario. Posicione esta control en la ventana y defina su tamaño. Las barras de desplazamiento se mostrarán automáticamente si algunos de los controles no caben en el tamaño especificado.

5. En la ventana del supercontrol description, especifique el nombre del supercontrol. Este nombre debe ser necesariamente SC_Form (SC_ es el prefijo definido en el código de estilo : no es necesario si no se utiliza un estilo de código).

6. En la pestaña "Contenido" de la ventana del supercontrol description, especifique:

	- el archivo navegado : RADFileXX ("RADFileA" por ejemplo).

	- el artículo consultado : IDRADFileXX ("IDRADFileA" por ejemplo).




7. Modificar el menú que se encuentra en la ventana:

	- Una opción de menú debe permitirle validar la ventana.

	- Una opción de menú debe permitirle acceder al fichero relacionado (para acceder a RADFileB desde el formulario RADFileA, por ejemplo).

	- Una opción de menú debe permitirle salir del formulario sin validar.




8. Las opciones del menú deben tener nombres específicos:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Pattern_RAD_MenuFiche_smart.gif)


	- MAINMENU_xxxxxx para las opciones principales.

- MENULIA_xxx para las opciones de enlace.

9. Caso de la opción de menú para la validación.
	En el código de la opción de menú para la validación, le aconsejamos que habilite la gestión de errores de automatic ("Si el error : ", con el automatic Process "Mostrar el error y volver a editar").
	El código del botón de validación puede ser (para un formulario en RADFileA):
	
	```wl
	// Read the information entered 
	ScreenToFile()
	// If it's a new record
	IF RADFileA..NewRecord = True THEN 
		// Add it
		HAdd(RADFileA) 
	ELSE 
		// Modify it 
		HModify(RADFileA) 
	END
	// Return value of the window 
	MyWindow..ReturnedValue = True
	```


10. Caso de las opciones de enlace:
	Desde el formulario RADFileA, puede tener una opción de enlace usada para mostrar el RADFileB tabla por ejemplo.
	Desde el formulario RADFileB, puede tener una opción de enlace utilizada para mostrar el formulario RADFileA, el RADFileC tabla y la ventana de relaciones por ejemplo.
	Ejemplo de código utilizado para abrir la ventana Tabla RADFileB desde la ventana del formulario RADFileA:
	
	```wl
	// Implement the filter on the link key
	HFilter(RADFileB, IDRADFileARel, RADFileA.IDRADFileA)
	// Open the table window of RADFileB
	Open(WIN_Table_RADFileB)
	// Disable filter
	HDeactivateFilter(RADFileB)
	```


11. Caso de la opción "Cancel
	
	```wl
	// Return value of the window
	MyWindow..ReturnedValue = False
	// Close the window
	Close()
	```


12. Introduzca el código de inicialización de la ventana. Ejemplo:
	
	
	```wl
	// Display the records 
	FileToScreen()
	// Return value returned by default
	MyWindow..ReturnedValue = False
	```

	Este código se utiliza para mostrar el valor de los registros en el formulario y para devolver un valor.

13. Añada los botones [Visión +](../Editeurs/2031040.md) si es necesario (para los formularios RADFileB y RADFileC).

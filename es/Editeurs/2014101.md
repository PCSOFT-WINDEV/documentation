


## Compartir componentes internos (mediante el SCM)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000135"></a>
Un componente interno es un conjunto de elementos de un proyecto. Este conjunto se utiliza para:

- **Organizar un proyecto**: Puede crear componentes internos para agrupar los elementos de un proyecto, por funcionalidad, por ejemplo. El proyecto se puede encontrar (o no) en la SCM.

- **Compartir elementos entre varios proyectos**: Los elementos que se encuentran en una Component interna pueden ser compartidos entre varios proyectos a través de la SCM.




Una de las ventajas de un componente interno en comparación con un componente externo es que puede depurarse directamente desde el proyecto que lo utiliza.

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Compartir un componente interno
<a name="compartir_componente_interno_ELTTEXTE000159"></a>
Los componentes internos pueden compartirse entre varios proyectos a través del SCM.

Para compartir un componente interno a través del SCM, el proyecto que contiene el componente interno debe estar en el SCM. Es posible:

- crear el componente interno a partir de un proyecto ubicado en el SCM. El componente interno se guardará automáticamente en el SCM.

- importar un proyecto que contenga uno o varios componentes internos en el SCM. Los componentes internos se incluirán automáticamente en el SCM.




Para obtener más información, consulte [Crear un componente interno](../Editeurs/2014100.md).
Cuando se importa un Component interno en un proyecto, la ventana Component description presenta las dependencias (otros Component internos o externos) necesarias para que el Component importado funcione correctamente.
<a name="NOTE3_2"></a>


### Utilización de un Component interno en un proyecto encontrado en el SCM
<a name="utilizacion_component_interno_proyecto_encontrado_scm_ELTPARAGRAPHE000041"></a>

Para utilizar un Component interno en un proyecto que se encuentra en el SCM:

1. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Importar" y seleccione "Un componente interno .. Desde el SCM". El archivo del proyecto se comprueba automáticamente si es necesario.

2. Aparece la ventana para añadir un Component interno. Se visualiza la lista de componentes internos encontrados en la SCM.. Tiene la posibilidad de ver las acciones ya realizadas para cada Component disponible.

3. Seleccione una Component interna y valídela.

4. El Component interno y todos sus elementos se incluyen automáticamente en el proyecto. En el proyecto local (en el ordenador de desarrollo) se crea un subdirectorio con el nombre de Component interno. Este directorio contiene todos los elementos de Component. Este subdirectorio también se crea en el directorio del proyecto en la base de datos SCM.




Observaciones:

- También tiene la posibilidad de importar un Component interno sin utilizar el SCM: en la pestaña "Proyecto", en el grupo "Importar", despliegue "" y seleccione "Un componente interno .. Desde un archivo". El contenido de Component interno se copiará en el proyecto. Los elementos no son compartidos.

- También puede utilizar un Component interno que se encuentra en el SCM desde la ventana de gestión de los Component s internos ("Gestionar los Component s internos del proyecto" desde el el menú contextual de la carpeta "Component s internos" que se encuentra en el explorador de proyectos).



<a name="NOTE3_3"></a>


### Manejo de una Component interna compartida
<a name="manejo_una_component_interna_compartida_ELTPARAGRAPHE000078"></a>

Tanto si la Component interna es compartida como si no, las operaciones que se pueden realizar en la Component interna son idénticas. Para obtener más información, consulte [Gestión de una Component interna](../Editeurs/2014104.md).

Atención: cuando el Component interno es compartido, todos los desarrolladores que utilizan el SCM pueden modificar el Component interno y sus características. Puede limitar las modificaciones aplicando una gestión de derechos a los componentes internos y a sus ficheros. Para obtener más información, consulte [Gestión de los derechos en la SCM](../Editeurs/2038019.md).

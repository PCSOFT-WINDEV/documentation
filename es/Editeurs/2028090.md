
## Instalación a través de un medio físico
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000135"></a>
Una instalación por medios físicos permite dar a la empresa de hospedaje un medio que contiene todos los archivos necesarios para instalar el sitio en un servidor. Para este tipo de implementación, la versión del servidor de aplicaciones debe ser idéntica a la versión del medio físico. Si la versión de la servidor de aplicaciones es más reciente, se debe realizar una implementación por FTP.

Una asistente permite a la persona que la despliega configurar las diferentes opciones de la instalación.

**Observación**: Para instalar su sitio WEBDEV dinámico, la empresa de alojamiento debe:

1. Instalar el WEBDEV servidor de aplicaciones en el servidor Web (si es necesario).
	Observación: Se requiere una licencia de WEBDEV servidor de aplicaciones para cada sitio WEBDEV dinámico instalado.

2. Instale su sitio WEBDEV ejecutando el ejecutable **Install.exe** de su instalación.




**Atención**: Este modo implementación no asegura la compatibilidad.: Un sitio desarrollado en la versión N no puede ser desplegado en un servidor en la versión N+1, N+2, etc.. Por ejemplo, un sitio desarrollado en la versión 20 no puede desplegarse en un servidor de aplicaciones de la versión 28.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000159"></a>
Para crear una instalación por medios físicos: 

1. En la pestaña "Proyecto", en el grupo "Generación", despliegue "Implementar sitio" y seleccione "Generar e implementar sitio".

2. Cree la biblioteca si es necesario. 

3. Seleccione "Setup by physical media" (Configuración por medio físico). 

4. [Seleccione los archivos a instalar](../Editeurs/2028058.md).

5. [Configurar la modificación de automatic](../Editeurs/2028048.md)

6. [Configurar el sitio](../Editeurs/2028051.md).

7. Seleccione la información relativa a la personalización de la instalación:

	- **La herramienta instalación utilizada**: WBSetup es propuesto por Default. WBSetup es una asistente para instalar sitios WEBDEV. Este programa de instalación está disponible en inglés y francés. Para que el programa instalación pueda soportar idiomas adicionales, el proyecto "WDSetup" debe ser personalizado antes de crear el programa instalación. Este proyecto (suministrado con los ejemplos de WEBDEV) corresponde al programa estándar instalación de las aplicaciones de WEBDEV. Este proyecto es un proyecto WINDEV. Para obtener más información, consulte [Utilización de un ajuste PEP personalizado](../Editeurs/2028096.md).

	- **El directorio Default instalación del sitio**: Este directorio corresponde al directorio donde se instalarán los archivos del sitio (bibliotecas, páginas, hojas de estilo, imágenes, ....).. Se propone por defecto el "&lt;sitio&gt;\\&lt;Nombre del proyecto&gt;" ruta, donde &lt;sitio&gt; corresponde al directorio associated con los diferentes sitios de la cuenta WEBDEV a los que se destinará el sitio associated. Le recomendamos que utilice pseudo-directorios. Este directorio se puede modificar al instalar el sitio.

	- **El directorio Default instalación del sitio data**: Este directorio corresponde al directorio donde se instalarán los archivos de datos del sitio.. Se propone por defecto el "&lt;data&gt;\\&lt;Nombre del proyecto&gt;" ruta donde &lt;data&gt; corresponde al directorio de datos associated con los diferentes sitios de la cuenta WEBDEV a los que se destinará el sitio associated. Le recomendamos que utilice pseudo-directorios. Este directorio se puede modificar al instalar el sitio.
			Se pueden utilizar los siguientes pseudo-directorios:

		- ***&lt;cebohidratos&gt;***: Directorio raíz de documentos del servidor web

		- ***Sitio web&gt; &lt;Sitio web&gt;***: Directorio de inicio de los sitios para la cuenta de usuario WEBDEV definida por la empresa de hosting. Si se le asigna una cuenta, todos sus sitios dinámicos se instalarán en este directorio.

		- ***&lt;datos&gt; &lt;datos***: Directorio principal de los ficheros de datos de la cuenta de usuario WEBDEV definida por la empresa de hosting. Si se le asigna una cuenta, todos los archivos de datos de sus sitios dinámicos se instalarán en este directorio.




8. Especificar los idiomas soportados por la instalación así como los parámetros correspondientes a cada idioma.
	Por Default, WBSetup apoya el francés y el inglés. Para soportar idiomas adicionales, le recomendamos que personalice WBSetup. Para obtener más información, consulte [Utilización de un ajuste PEP personalizado](../Editeurs/2028096.md).
	Para cada lenguaje propuesto durante la instalación, especifique:

	- **El pie de foto de la introducción**. Esta leyenda se utiliza para iniciar la instalación en el idioma deseado. Corresponde al lenguaje instalación de Default.

	- **La licencia de su sitio web**. La licencia debe ser un archivo en formato txt, se mostrará al iniciar la instalación de su sitio. Para que el texto se muestre en el idioma elegido al principio de instalación, se debe crear un archivo de licencia para cada idioma. La instalación se realizará sólo la licencia es validada por la persona que está instalando su sitio WEBDEV dinámico

	- **Léame el archivo de su sitio**. El archivo "Léame" puede tener uno de los siguientes formatos: txt, htm, doc, hlp, pdf 
			Independientemente del formato seleccionado, el usuario debe disponer de un software que le permita visualizar este archivo. Para que el archivo "Léame" se muestre en el idioma elegido al principio de instalación, se debe crear un archivo "Léame" para cada idioma.




9. Especificar el directorio de generación de la instalación. Puede elegir borrar (o no) el contenido del directorio durante la generación.

10. Validar. Se crea el programa instalación del sitio. Es posible:

	- desplegar el sitio (para ejecutar la prueba de instalación por ejemplo)

	- abrir el explorador (a copy el directorio de instalación a un CD por ejemplo).

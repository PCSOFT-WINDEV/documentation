## Presentación
<a name="presentacion_ELTTEXTE000148"></a>


### ¿Qué es una aplicación Java?
<a name="¿que_una_aplicacion_java_ELTPARAGRAPHE000011"></a>

Una aplicación de Java está enteramente compilada en Java. Si la aplicación utiliza una base de datos, la base de datos debe tener un controlador JDBC (MySQL, Oracle, etc.). Este controlador es específico para cada base de datos (puede descargarse del sitio INTERNET del editor de la base de datos).

El acceso a una base de datos HFSQL se realiza de forma nativa (sólo en Windows y Linux). En este caso no se requiere ningún conductor adicional.

Una aplicación Java corresponde a un archivo ".jar" que requiere Java Runtime para ejecutarse.

## Desarrollando una aplicación de Java con WINDEV
<a name="desarrollando_una_aplicacion_java_con_windev_ELTTEXTE000178"></a>


### ¿Por qué desarrollar una aplicación Java con WINDEV?
<a name="¿por_que_desarrollar_una_aplicacion_java_con_windev_ELTPARAGRAPHE000025"></a>

¿Ha desarrollado una aplicación WINDEV para Windows y quiere que esta aplicación sea accesible independientemente del sistema operativo? Genere una aplicación Java para que los usuarios que trabajen con Mac OS por ejemplo, también puedan utilizar su aplicación.

Esto le permite compartir sus aplicaciones de red.

Observación: Mac OS X 10.9 mínimo.
<a name="NOTE2_2"></a>


### Generar una aplicación Java con WINDEV
<a name="generar_una_aplicacion_java_con_windev_ELTPARAGRAPHE000036"></a>

WINDEV le permite generar una aplicación Java sin siquiera conocer el lenguaje Java. El método para desarrollar la aplicación es idéntico al método para desarrollar cualquier otra aplicación WINDEV.

Cuando se compila el proyecto WINDEV, las ventanas y las funciones WLanguage se convierten automáticamente en Java.

### Java y la gestión de la memoria
<a name="java_gestion_memoria_ELTPARAGRAPHE000091"></a>

Por defecto, la cantidad de memoria asignada a la máquina virtual de Java (JVM) para ejecutar aplicaciones es de set a 64 MB. 

Esta cantidad puede no ser suficiente si la aplicación requiere una gran cantidad de memoria (para imágenes manipular por ejemplo).. Los errores ("OutOfMemoryException" o "Java Heap Space") pueden ocurrir si se excede la cantidad de memoria al ejecutar una aplicación Java. 

Para aumentar la cantidad de memoria asignada a la JVM, especifique **"-Xmx&lt;tamaño&gt;"** cuando se inicia la aplicación. Por ejemplo, para asignar 256 MB de memoria a una aplicación, especifique el siguiente comando Line al iniciar la aplicación:


```txt
java -jar -Xmx256m MyArchive.jar
```

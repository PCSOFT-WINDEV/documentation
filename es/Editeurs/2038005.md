


## Creación de un repositorio SCM
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000194"></a>
Para compartir un proyecto mediante el Administrador de Código Fuente, se debe crear un repositorio. El repositorio debe crearse una sola vez en el servidor.

El sistema operativo y el sistema de archivos en el que está instalado el repositorio SCM deben soportar archivos de más de 4 GB.

Se puede crear el repositorio SCM: 

- en **SCMDrive** (solución paga lista para usar). Para obtener más información, consulte [SCMDrive](https://pcscloud-drive.net/UK/SCMDrive/index.awp). 
	En este caso, se debe especificar: 

	- Su nombre de usuario (Email) y su contraseña,

	- Su equipo. 




- en una plataforma de desarrollo **PCSCLOUD** (solución paga lista para usar). Para obtener más información, consulte [PCSCloud](https://pcscloud.net/UK/cloud_development.awp). 
	En este caso, se debe especificar: 

	- El nombre de su plataforma,

	- El nombre de usuario y la contraseña.




- **en un servidor HFSQL Client/Server previamente instalado**: el repositorio estará en formato [HFSQL Client/Server](../WDLang4/3044306.md). En este caso, se debe especificar:

	- el servidor, 

	- la base de datos HFSQL,

	- el puerto utilizado,

	- el nombre y la contraseña de un administrador de la base de datos HFSQL. 




- **en un directorio compartido de la red**: todos los usuarios del SCM deben tener permisos sobre este directorio. El repositorio estará en formato [HFSQL Classic](../CCHF/35400010.md).
	Especifique el directorio de red en el que se creará el repositorio.




Si utiliza su propia solución (servidor HFSQL Client/Server previamente instalado, o repositorio SCM en formato HFSQL Classic), la máquina debe ser fiable y segura. Para obtener más información, consulte [Base de datos HFSQL Client/Server: Recomendaciones de seguridad](../WDLang4/1000017310.md)

## ¿Cuándo debe crearse un repositorio SCM?
<a name="¿cuando_debe_crearse_repositorio_scm_ELTTEXTE000224"></a>
**El repositorio SCM debe crearse una sola vez.**

WINDEV, WEBDEV y WINDEV Mobile le permiten crear este repositorio SCM:

- al instalar WINDEV, WEBDEV o WINDEV Mobile.

- al crear un proyecto que utiliza el SCM.

- [al importar un proyecto al Administrador de Código Fuente.](../Editeurs/2038002.md)

- [directamente en el administrador del SCM](../Editeurs/2038008.md).




Una vez creado el repositorio SCM, todos los proyectos compartidos de WINDEV, WEBDEV y WINDEV Mobile pueden ser importados a este repositorio SCM..

## Conversión de un repositorio SCM existente de HFSQL Classic a HFSQL Client/Server
<a name="conversion_repositorio_scm_existente_hfsql_classic_hfsql_clientserver_ELTTEXTE000248"></a>
Las siguientes operaciones permiten convertir un repositorio SCM existente de HFSQL Classic a HFSQL Client/Server:

1. Optimizar el formato del repositorio HFSQL Classic SCM. Para ello: 

	- abrir el administrador de SCM: en la pestaña "SCM", en el grupo "Repositorio", despliegue "Administrar" y seleccione "Administrador del SCM". 

	- en el administrador, en la pestaña "Herramientas", en el grupo "Mantenimiento", seleccione la opción "Migrar al formato más reciente ".




2. Instalar (si es necesario) un servidor HFSQL en el equipo donde se debe instalar el repositorio SCM HFSQL Client/Server.

3. En el administrador del SCM, en la pestaña "Herramientas", en el grupo "Mantenimiento", seleccione "Cambiar a Cliente/Servidor".

4. Todos los usuarios pueden conectarse al repositorio HFSQL Client/Server SCM pasando los parámetros necesarios (servidor, base de datos, puerto utilizado, nombre y contraseña de un administrador de base de datos HFSQL).

## copias
<a name="copias_ELTTEXTE000272"></a>
Le recomendamos hacer copias de seguridad del repositorio de forma regular. Para ello, todo lo que tienes que hacer es copy y archivar los archivos de datos correspondientes al SCM.

Esta operación se realizará automáticamente en una próxima versión.

## Cambiar la contraseña del administrador (repositorio SCM en modo Cliente/Servidor)
<a name="cambiar_contrasena_del_administrador_repositorio_scm_modo_clienteservidor_ELTTEXTE000296"></a>
La contraseña del administrador de SCM, en una base de datos Cliente/Servidor, debe ser cambiada en el Centro de Control HFSQL del repositorio de SCM.

Para ello:

1. Seleccione el servidor que corresponde al repositorio SCM.

2. 1. Seleccione la pestaña "Usuarios".

3. Seleccione el administrador en la lista de usuarios y haga clic en "Editar".

4. Modificar la contraseña y validar.

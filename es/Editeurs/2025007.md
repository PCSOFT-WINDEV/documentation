## Directorio para la creación de los archivos de datos de la Agrupación de Usuarios
<a name="directorio_para_creacion_los_archivos_datos_agrupacion_usuarios_ELTTEXTE000225"></a>
Se gestionan dos categorías de archivos:

- **los ficheros de datos utilizados para identificar a los usuarios**. Estos archivos de datos pueden ser comunes a varias aplicaciones.

- **los ficheros de datos utilizados para gestionar los derechos concedidos a los usuarios en las diferentes ventanas de la aplicación**. Estos archivos son específicos de la aplicación.




Para obtener más información, consulte [Los archivos de el groupware usuario](../Editeurs/2018002.md).

En Default, estos archivos de datos se crean en el subdirectorio "&lt;NombreDeLaAplicación&gt;_GPW" del directorio instalación de la aplicación. Sin embargo, este directorio puede ser modificado. Este directorio puede corresponder a:

- un directorio específico

- el directorio de los datos de la aplicación. Esta elección se adapta al mecanismo UAC de Windows Vista. Si los usuarios trabajan con Windows Vista (o posterior) y si el mecanismo UAC está habilitado, podrán usar la aplicación sin derechos de administrador específicos.

## Directorio para crear los archivos de datos de la aplicación HFSQL Classic
<a name="directorio_para_crear_los_archivos_datos_aplicacion_hfsql_classic_ELTTEXTE000249"></a>
Por Default, los archivos de datos HFSQL se crean en los directorios especificados al describir los archivos en el análisis. Sin embargo, estos directorios pueden ser modificados:

- **al crear el ejecutable de la aplicación**.
	Hay varias opciones disponibles: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=D%E9ployer%20l'application%20-%20HC%20N%B0003.gif)


	- **Automático (recomendado)**: Esta opción permite instalar los archivos de datos: 

	- en el directorio de datos de la aplicación (si el ejecutable está instalado en "Program files"). 

	- en el directorio del ejecutable. 

- **mediante la programación de** con [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md).

- **al instalar la aplicación**. El directorio para crear los archivos de datos será seleccionado por el usuario.




En cualquier caso, el directorio para la creación de los ficheros de datos se define según el siguiente orden de prioridad:

1. Directorio especificado por [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md).

2. Directorio seleccionado al instalar la aplicación (si esta opción se propone al usuario).

3. Directorio seleccionado al crear el ejecutable.

4. Directorio especificado al describir los archivos en el análisis.




Por lo tanto, el directorio definido por [HSubstDir](../WDLang4/3044028.md) o [HChangeDir](../WDLang4/3044168.md) tiene prioridad sobre todos los demás directorios.

**Caso especial: archivos de datos instalados con la aplicación**
Estos archivos de datos se instalan en el directorio especificado al crear el programa instalación (directorio de destino especificado en el paso "Archivos instalados").
El mismo directorio debe utilizarse para instalar los archivos de datos suministrados con la aplicación y para crear los nuevos archivos de datos.




## Creación de un ejecutable: Framework WINDEV 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000258"></a>
Cada función de WINDEV es associated con un **WINDEV DLL**. Para que el ejecutable funcione correctamente, todas las DLLs necesarias deben instalarse con el ejecutable. Todas estas DLLs corresponden al Framework WINDEV.

El asistente para [Creación de un ejecutable](../Editeurs/2025002.md) incluye diferentes modos para cargar el Framework. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>

<a name="NOTE3_1"></a>


### Utilice las DLLs de framework junto al ejecutable
<a name="utilice_las_dlls_framework_junto_ejecutable_ELTPARAGRAPHE000049"></a>

Las DLLs de WINDEV necesarias se copian automáticamente al subdirectorio EXE del proyecto cuando se crea el ejecutable. 

Si selecciona esta opción: 

- El botón "Siguiente" enumera las diferentes DLL de framework necesarias para ejecutar su aplicación (véase [Descripción del Framework](#NOTE3_5)). 

- Si su ejecutable utiliza bibliotecas externas específicas, la lista de bibliotecas se propone también con la lista de DLLs de framework.



<a name="NOTE3_2"></a>


### Utilice el framework incluido en el ejecutable
<a name="utilice_framework_incluido_ejecutable_ELTPARAGRAPHE000066"></a>

El ejecutable es independiente. Las DLLs no serán extraídas durante el arranque. 

Si se selecciona esta opción, se recompila el proyecto para comprobar las posibles incompatibilidades.. 

**Atención**: Todos los framework DLLs se incluyen sistemáticamente en el ejecutable. El tamaño del ejecutable puede ser muy importante. 

**Observación**: Las siguientes características no están permitidas: 

- Conectores Nativos

- .Net

- HTML control en los informes

- Funciones de grabación (característica permitida a partir de la versión 200051)

- Video control Conferencia

- CM15 (domótica vía USB)

- Control «xaml»

- Gráficos DirectX 3D

- Gestión de los archivos del CAC



<a name="NOTE3_3"></a>


### Usando el programa común de framework
<a name="usando_programa_comun_framework_ELTPARAGRAPHE000109"></a>

Esta opción permite:

- limitar el tamaño del programa a instalar en el ordenador del usuario final. De hecho, el WINDEV framework no se copia en el directorio EXE del proyecto y no se instala en el ordenador del usuario final.

- utilizar el mismo WINDEV framework para todas las aplicaciones WINDEV utilizadas por el usuario final. Los archivos de framework sólo se encuentran una vez en el equipo (y no se copian con cada ejecutable).




Es posible: 

- utilice el **Marco de trabajo común de WINDEV**: Si se selecciona esta opción, tiene la posibilidad de configurar los diferentes mensajes que se muestran al usuario final: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Exe_Framework_telech.gif)
**Notas**: 

	- En las versiones anteriores, el framework sólo podía descargarse por FTP. A partir de la versión 26, esta descarga utiliza principalmente HTTPS (resuelve los problemas de los puertos que hay que abrir en los cortafuegos). 

	- En el ordenador del usuario final, el PC SOFT Framework se encuentra en "Archivos de programa\\Archivos comunes\\PC SOFT\\28.0\\Framework".




- usar un **renombrado framework**: Esta opción no está disponible si está utilizando las bibliotecas XAML o.NET en su aplicación..
	Si se selecciona esta opción, el botón "Siguiente" lista los diferentes framework DLLs necesarios para ejecutar su aplicación.. Puede seleccionar las DLLs, su modo de carga y el prefijo del nuevo nombre.
	**Observación**: Al crear la instalación (paso "Setup files 3/3"), puedes: 

	- elegir entre proporcionar la framework o permitir que el usuario final la descargue. 

	- definir la framework instalación ruta. En el ordenador del usuario final, la ubicación del framework se almacenará en la clave del registro "\\HKEY_LOCAL_MACHINE\\SOFTWARE\\PC SOFT\\Framework\\28.0\\&lt;WD28 0WM.DLL&gt;" con un valor "PATH" (REG_SZ) que contenga el ruta completo del **Common renombrado framework**. Por ejemplo "C:\\Archivos de programa\\Archivos comunes\\Marco\\".






<a name="NOTE3_4"></a>


### Opciones conservadas por motivos de compatibilidad con versiones anteriores
<a name="opciones_conservadas_por_motivos_compatibilidad_con_versiones_anteriores_ELTPARAGRAPHE000144"></a>

Las siguientes opciones se mantienen para la compatibilidad con versiones anteriores: 

- **Include the necessary framework DLLs in the executable** (las DLLs se extraen durante el inicio)
	Las DLLs necesarias se incluirán en el ejecutable. En este caso, el tamaño del ejecutable aumentará. Cuando el ejecutable se ejecuta por primera vez, los diferentes framework DLLs se extraerán del ejecutable. 
	Si selecciona esta opción, el botón "Next" (Siguiente) enumera las diferentes DLL de framework necesarias para ejecutar su aplicación (véase [Descripción del Framework](#NOTE3_5)). Si su ejecutable utiliza bibliotecas externas específicas, la lista de bibliotecas se propone también con la lista de DLLs de framework.

- **Include part of the framework en el ejecutable** (las DLLs se extraen al inicio)
	Esta opción se utiliza para seleccionar las diferentes DLLs que se incluirán en el ejecutable. Las otras DLLs utilizadas por la aplicación estarán disponibles junto al archivo.EXE de la aplicación.. Cuando el ejecutable se ejecuta por primera vez, los diferentes framework DLLs incluidos en el ejecutable se extraerán del ejecutable.. 
	Haga clic en el botón ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Framework_select.gif) y seleccione las DLL WINDEV que desea incluir.



<a name="NOTE3_5"></a>


### Descripción del Framework
<a name="descripcion_del_framework_ELTPARAGRAPHE000167"></a>

Para cada DLL de WINDEV o biblioteca externa, puede hacerlo:

- ***renombrar este archivo*** (columna"Nuevo nombre"). Si se instalan varias aplicaciones WINDEV en el mismo ordenador, cada ejecutable utiliza sus propias DLLs.
	El botón "Prefijo" le permite sustituir "wd28 0" por el prefijo que desee. 
	Observación: No cambie el nombre de las DLLs WINDEV de los programas creados con un lenguaje externo.

- ***especifique el modo de carga de este archivo***:

	- *al iniciar la aplicación*. En este caso, la DLL de WINDEV estará disponible siempre y cuando se utilice la aplicación.. Sin embargo, el tiempo requerido para iniciar la aplicación puede aumentar de acuerdo al número y tamaño de DLLs WINDEV a cargar.. La cantidad de memoria utilizada también puede aumentar.

	- *cuando se utiliza por primera vez* (modo recomendado). La DLL WINDEV se cargará en la memoria sólo durante la primera llamada a una función que utilice esta biblioteca. Este modo de carga se recomienda para las DLLs que se utilizan raramente. Este modo de carga optimiza el tiempo necesario para iniciar la aplicación y la cantidad de memoria utilizada.

## Biblioteca externa
<a name="biblioteca_externa_ELTTEXTE000336"></a>


### Librerías externas (archivos".WDL") utilizadas por el programa ejecutable
<a name="librerias_externas_archivoswdl_utilizadas_por_programa_ejecutable_ELTPARAGRAPHE000188"></a>

Para compartir recursos entre varias aplicaciones, WINDEV le permite utilizar bibliotecas creadas por otras aplicaciones WINDEV.. Estas bibliotecas se denominan "bibliotecas externas".. Asociar estas librerías con la aplicación:

- seleccione "Taller .. Biblioteca.... Bibliotecas externas". Por Default, estas bibliotecas serán propuestas automáticamente por la creación ejecutable asistente.

- utilice [LoadWDL](../WDLang1/3013020.md) en su código WLanguage. Las bibliotecas externas deben suministrarse junto con su aplicación.




**Observación**: Para obtener más información, consulte [Uso de varias bibliotecas en una aplicación](../Editeurs/2030027.md).

Para cada biblioteca externa disponible (archivo".WDL"), puede hacerlo:

- ***renombrar esta biblioteca*** (columna"Nuevo nombre").

- ***incluir esta librería en el ejecutable***. Si las librerías WINDEV requeridas por la aplicación y las librerías externas están incluidas en el ejecutable, el ejecutable será independiente.. No es necesario instalar las bibliotecas y las bibliotecas externas en los ordenadores de los usuarios finales.. Para obtener más información, consulte [Ejecutable independiente](../Editeurs/2025009.md).
	**Observación**: Si las librerías (archivos ".WDL") no están incluidas en el ejecutable, las librerías se copiarán automáticamente en el subdirectorio "EXE" del proyecto cuando se cree el ejecutable.

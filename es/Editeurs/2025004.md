


## Creación de un ejecutable: Modo de funcionamiento y biblioteca
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000112"></a>
El **modo de funcionamiento** de un ejecutable permite definir si puede ser utilizado por una o varias personas.

La biblioteca principal **library** de una aplicación agrupa todos los elementos manejados por esta aplicación (ventanas, informes, clases, set de procedimientos, consultas, archivos de datos) y sus dependencias (imágenes utilizadas en las ventanas por ejemplo). Para más detalles, consulte la ayuda de [libraries](../Editeurs/2030022.md).
**Observación**: Para crear el ejecutable, utilice uno de los siguientes métodos: 

- En la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar". 

- Despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_exe.gif) en los botones de acceso rápido y seleccione "Generar el ejecutable de Windows de 32 bits (exe)" o "Generar el ejecutable de Windows de 64 bits (exe)". 




La creación del ejecutable asistente permite definir el modo de funcionamiento del ejecutable y los modos de utilización de la biblioteca.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Modo de funcionamiento del ejecutable
<a name="modo_funcionamiento_del_ejecutable_ELTTEXTE000136"></a>
El ejecutable puede ser:

- ***instancia múltiple***. Un programa multi-instancia puede iniciarse varias veces desde el mismo ordenador.: varias instancias del programa se abrirán al mismo tiempo.

- ***instancia única***. Un programa de una sola instancia puede iniciarse una sola vez desde el mismo ordenador.. Si el usuario intenta iniciar otra instancia del programa, la instancia actual se activará automáticamente.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Modos de uso de la biblioteca
<a name="modos_uso_biblioteca_ELTTEXTE000160"></a>
Los modos de uso de la biblioteca son los siguientes:

- ***integración de la biblioteca principal en el ejecutable***. Esta opción se utiliza para crear (o modificar) la biblioteca principal de la aplicación actual y para **incluir** esta biblioteca en el programa ejecutable. 
	Para actualizar la biblioteca principal, el programa ejecutable debe ser recreado.

- ***uso externo de la biblioteca principal***. Esta opción se utiliza para crear (o modificar) la biblioteca principal de la aplicación actual y para asociar **associate** a esta biblioteca con el programa ejecutable.. 
	Para actualizar la biblioteca principal, esta biblioteca debe ser recreada.

- ***no utilizar ninguna biblioteca***. Ninguna biblioteca será associated con o incluida en el programa ejecutable. Esta opción se mantiene para la compatibilidad con versiones anteriores. Para obtener más información, consulte [Creación de un ejecutable sin biblioteca](../Editeurs/2030025.md).

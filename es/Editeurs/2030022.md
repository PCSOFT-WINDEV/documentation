## Caso general: aplicación o sitio asociado a una biblioteca
<a name="caso_general_aplicacion_sitio_asociado_una_biblioteca_ELTTEXTE000116"></a>
Una biblioteca es un archivo WDL que contiene todos los recursos de WINDEV o WEBDEV: ventanas, páginas, reportes, clases, conjuntos de procedimientos, archivos de datos, etc.

Este archivo es abierto por el ejecutable o por el sitio. Los elementos necesarios se cargan automáticamente cuando se utilizan. 

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Al crear el ejecutable, se genera la biblioteca que corresponde al proyecto. Esta se puede incluir (o no) en el ejecutable. Para obtener más información, consulte [Crear un ejecutable](../Editeurs/2025002.md). 
**Observación**: Para guardar la compatibilidad con la versión 5.5, puede crear un ejecutable sin biblioteca. Para obtener más información, consulte [Crear un ejecutable sin biblioteca](../Editeurs/2030025.md).

## Caso especial: Dividir una aplicación o sitio en varias bibliotecas
<a name="caso_especial_dividir_una_aplicacion_sitio_varias_bibliotecas_ELTTEXTE000140"></a>
Para reducir los tiempos de carga cuando se trabaja con proyectos grandes (por ejemplo, cientos de ventanas), puede dividir el proyecto en: 

- un proyecto principal de tipo "Ejecutable" o "Sitio". 

- uno o varios proyectos de tipo "Biblioteca". Cada proyecto de tipo "Biblioteca" contiene los elementos del proyecto asociados a un tema. 




El proyecto principal crea la biblioteca principal. Esta biblioteca se crea al crear el ejecutable. La biblioteca contiene los elementos específicos del proyecto: análisis, proyecto, etc. La biblioteca principal no se puede compartir entre varios proyectos. Es única para cada proyecto.

Los proyectos de tipo "Biblioteca" crean las bibliotecas secundarias de la aplicación agrupando los elementos de la misma por temas. 

Por ejemplo, un proyecto de gestión empresarial agrupa varios cientos de ventanas y reportes. Para optimizar el tiempo de carga del proyecto en el editor, éste se divide en: 

- un proyecto principal. Este proyecto crea la biblioteca principal y utiliza las bibliotecas secundarias. 

- un proyecto de "Gestión de clientes". Este proyecto se utiliza para crear una biblioteca secundaria. 

- un proyecto de "Gestión de pedidos". Este proyecto se utiliza para crear una biblioteca secundaria. 

- etc.




Para obtener más información, consulte [Usar varias bibliotecas](../Editeurs/2030027.md). 

**Observación**: Para compartir recursos entre aplicaciones o sitios, le recomendamos utilizar componentes externos o internos. Son muy fáciles de usar. Para obtener más información, consulte: 

- [Componentes externos](../Editeurs/2014011.md).

- [Componentes internos](../Editeurs/2014100.md).

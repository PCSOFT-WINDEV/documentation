
## 4. Modos de depuración

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Existen dos modos diferentes para depurar una aplicación WINDEV Mobile:

- Depuración en el simulador,

- Depuración en el dispositivo móvil (disponible en Android e iOS). 




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Depuración en el simulador
<a name="depuracion_simulador_ELTTEXTE000136"></a>
Este modo inicia la aplicación en un simulador de dispositivos móviles. Sin embargo, la aplicación se ejecuta en el PC, en el entorno de Windows.

Este modo se utiliza para depurar rápidamente una nueva función o probar un algoritmo, pero puede tener un comportamiento diferente en comparación con el dispositivo real:

- las rutas de los archivos son las de Windows,

- de forma predeterminada, las cadenas están en formato ANSI (no en Unicode),

- las funciones específicas de los dispositivos móviles (por ejemplo, SMS) no están disponibles.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Depuración en el dispositivo móvil
<a name="depuracion_dispositivo_movil_ELTTEXTE000160"></a>
Este modo de depuración hace que el comportamiento de la aplicación sea más preciso, ya que el depurador ejecuta la aplicación directamente en el dispositivo móvil (Android e iOS). Se pueden utilizar todas las funciones específicas (excepto las que requieren ejecutables firmados digitalmente).

Este modo es ligeramente más lento que el simulador. Añade una fase de comunicación entre el entorno de desarrollo y el depurador del dispositivo.

En este modo, el dispositivo móvil debe estar conectado al equipo de desarrollo.


## 3. Prueba de una aplicación WINDEV Mobile

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Al probar el proyecto puede simular cómo se ejecutará la aplicación. Esto le permite probar toda la aplicación, incluso si todavía se está desarrollando.

WINDEV Mobile incluye varios tipos de pruebas de proyecto:


|   |   |
| --- | --- |
|   | <br><br>- **Prueba en el equipo de desarrollo**. Esta prueba simula un dispositivo móvil en el equipo de desarrollo. No se requiere ninguna conexión a un dispositivo para esta prueba. Puede utilizar el depurador. Sin embargo, como la prueba se ejecuta en un PC y no en el dispositivo, la aplicación puede comportarse de forma diferente.<br><br><br> |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_IOS.gif)<br> | <br><br>- **Prueba real (iPhone o iPad)**. Puede probar la aplicación iOS directamente en un dispositivo móvil. Para ello: <br><br>	- descargue la aplicación WMDEV (disponible en la tienda) al dispositivo. <br><br>	- escanee el código de barras que se muestra en WINDEV Mobile y que corresponde a la aplicación que se va a probar. <br><br><br> Esta opción permite probar la aplicación directamente en el dispositivo. Puede utilizar el depurador. <br><br><br> |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Logo_IOS.gif)<br> | <br><br>- **Prueba en un simulador de iPhone/iPad**. La aplicación iOS se puede probar directamente en el Mac al compilar el proyecto en Xcode. Esta opción no requiere un iPhone o iPad para ejecutar las pruebas, pero proporciona una ejecución más precisa que el modo "simulador".<br><br><br> |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LOGO_Android.gif)<br> | <br><br>- **Prueba y depuración en un dispositivo (Android) conectado al equipo de desarrollo**. Esta prueba permite utilizar el depurador aunque se ejecute en el dispositivo móvil.<br><br><br> |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LOGO_Android.gif)<br> | <br><br>- **Prueba en el emulador Android**. El SDK de Android incluye un emulador Android. La aplicación se puede probar en el emulador. Esta opción no requiere un dispositivo Android para realizar las pruebas, pero proporciona una ejecución más precisa que el modo "simulador".<br><br><br> |
| <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=logo_Windows_10.gif)<br> | <br><br>- **Prueba real (PC o dispositivo móvil)**. Puede probar una aplicación universal de Windows 10 en un equipo con Windows 10. Esta opción permite ejecutar el proyecto desde el modo de iconos, desde Windows 10 directamente. La aplicación universal de Windows 10 se genera automáticamente.<br><br><br> |


## 13. Arquitectura de 3 niveles

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
La arquitectura de 3 niveles es un modelo de arquitectura de aplicaciones.

Su principio básico consiste en dividir una aplicación en 3 niveles:

- Nivel de presentación: la interfaz de usuario,

- Nivel de aplicación (o procesos): lógica de negocio de la aplicación,

- Nivel de datos: para acceder a los datos persistentes.




El objetivo de esta separación es hacer que cada nivel sea independiente, para facilitar el mantenimiento y los futuros desarrollos de la aplicación. Esto proporciona una mayor seguridad porque solo el Nivel aplicación permite acceder a la base de datos. También optimiza el trabajo en equipo y el desarrollo multiplataforma. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=3tiers.gif)
La arquitectura de 3 niveles es totalmente compatible con el desarrollo de aplicaciones o sitios utilizando WINDEV, WEBDEV o WINDEV Mobile.

## Presentación
<a name="presentacion_ELTTEXTE000132"></a>
Para simplificar el trabajo en equipo, WINDEV Mobile incluye un Administrador de Código Fuente. 

El Administrador de Código Fuente permite a los desarrolladores trabajar juntos en un mismo proyecto al mismo tiempo y compartir elementos entre varios proyectos.
<br>![Un repositorio contiene una colección de elementos del proyecto. Cada equipo tiene una copia local de los elementos necesarios para el desarrollo](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P34-GDS.gif)


Los elementos del SCM se pueden compartir:

- mediante [PCSCloud (solución paga lista para usar, en un servidor dedicado)](https://pcscloud.net/UK/cloud_development.awp), 

- mediante [SCMDrive (solución paga lista para usar, en un servidor compartido)](https://pcscloud-drive.net/UK/SCMDrive/index.awp).

- a través de una red,

- a través de Internet, 

- en modo sin conexión. En este caso, los elementos necesarios se extraerán del SCM cuando el portátil se conecte al sistema principal, por ejemplo.

## Modo de funcionamiento del Administrador de Código Fuente
<a name="modo_funcionamiento_del_administrador_codigo_fuente_ELTTEXTE000156"></a>
El siguiente ejemplo presenta el Administrador de Código Fuente:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GestionnaireDeSources.gif)


Un elemento del proyecto (ventana, reporte, etc.) solo puede ser extraído por un desarrollador a la vez.

Una vez que se modifican los elementos extraídos, se deben volver a reintegrar para que el proyecto fuente tenga en cuenta los cambios. El repositorio almacena un historial de todos los elementos del proyecto desde que se crearon.

Cada vez que se vuelve a reintegrar un elemento, el número de versión del proyecto fuente se incrementa en 1.

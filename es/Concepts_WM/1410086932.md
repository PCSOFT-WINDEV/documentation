


## 5. Pruebas unitarias

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Las pruebas unitarias (también llamadas pruebas automatizadas) se utilizan para probar las ventanas, procedimientos y clases de una aplicación en diferentes niveles de desarrollo.

Para ello, seleccione (o cree) el escenario de prueba que se ejecutará.

Los escenarios de prueba se generan en WLanguage y pueden modificarse directamente.

Estos escenarios se encuentran en el editor de pruebas. El editor de pruebas analiza el resultado de las pruebas unitarias y calcula el índice de validación de la aplicación.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P69_tests_unitaires.gif)


Para crear una prueba unitaria en la ventana actual, en la pestaña "Pruebas automatizadas", en el grupo "Pruebas", despliegue "Nuevo" y seleccione "Grabar un nuevo escenario".

Para crear una prueba unitaria en un procedimiento o en un método de clase, seleccione el procedimiento o el método de clase en el panel "Explorador de proyectos" y seleccione "Pruebas unitarias .. Crear una prueba unitaria" en el menú contextual.


## 12. Sincronización de datos

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
WINDEV Mobile permite sincronizar los registros utilizados por varias aplicaciones en diferentes plataformas.

Así, las aplicaciones manipulan los mismos datos de forma independiente. Cuando se sincronizan los datos, los cambios realizados en la base de datos utilizada por el PC se reflejan automáticamente en el dispositivo móvil, y viceversa.

Esta sincronización se realiza automáticamente a través de la **Replicación Universal**.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Replicación universal
<a name="replicacion_universal_ELTTEXTE000129"></a>
La replicación universal actualiza las bases de datos (con o sin el mismo formato) utilizadas por las aplicaciones. Por ejemplo, puede sincronizar una base de datos HFSQL con una base de datos Oracle.

La replicación universal utiliza un modelo centralizado: todas las bases de datos están sincronizadas con una base de datos maestra. Luego, la base de datos maestra aplica los cambios a las demás bases de datos.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Replication_Bdd.gif)


La sincronización puede adaptarse a casos especiales. Por ejemplo, puede obtener los registros relacionados con un producto específico o creados en una fecha determinada, gestionar conflictos, mostrar una ventana de configuración, etc.

Estas adaptaciones deben realizarse mediante programación con la función [HRplFilterProcedure](../WDLang4/3044300.md).

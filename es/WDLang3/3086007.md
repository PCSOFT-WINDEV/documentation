## Configuración del administrador de SOAP
<a name="configuracion_del_administrador_soap_ELTTEXTE000099"></a>
**Para configurar el administrador de SOAP**: 

1. Inicie el administrador SOAP (WDSOAPConfig.EXE). Este administrador se encuentra en el directorio instalación de su aplicación (o en un directorio específico si ya está instalado). En el ordenador de desarrollo, este administrador está disponible en el directorio "Programs _ASLASH_Data" de WINDEV. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ConfigSOAP.gif)


2. Configurar los diferentes elementos:

	- **Tiempo antes de descargar una BDM no utilizada**: Si se encuentran varias bibliotecas (archivo WDL) correspondientes a aplicaciones de servidor SOAP (o a servicios web XML) en el equipo, esta opción permite configurar el tiempo máximo de espera antes de borrar de la memoria una biblioteca no utilizada. El código final del proyecto se ejecutará durante esta eliminación.

	- **Número máximo de WDLs en la memoria**: Si se encuentran varias bibliotecas (archivo WDL) correspondientes a aplicaciones de servidor SOAP (o a servicios web XML) en el equipo, esta opción sirve para configurar el número de WDLs cargados en memoria al mismo tiempo. Cuando se alcanza este número, la biblioteca menos reciente se cierra automáticamente. El código final del proyecto se ejecutará durante este cierre.

	- **Guardando un Log(.LOG)**: Esta opción se utiliza para guardar en un archivo de texto todas las operaciones realizadas en el servidor SOAP.. La fecha y la hora se especifican para cada operación. Este archivo puede contener, por ejemplo, los siguientes mensajes: 
			
		```txt
		Loading the WDL X
		Receiving a request: calling the X function of WDL Y
		The call to the X function failed
		The call to the X function was successful
		The limit of WDL in memory was reached
		Unloading WDL X
		```





- **Archivo de registro**: Directorio del servidor en el que se debe crear el archivo de registro.

- **Localización de WDL**: Directorio del servidor donde se encuentran las librerías de las aplicaciones del servidor SOAP. 
			Observación: si el directorio no existe, las BDM se buscarán en el directorio C:\\soapmodules.

- **Ubicación de las DLLs de WINDEV**: Directorio del servidor donde se encuentran las DLLs WINDEV utilizadas por las aplicaciones del servidor SOAP.. 
			Observación: si el directorio no existe, las DLLs se buscarán en el directorio C:\\soapmodules.

- **Ubicación de los archivos en tiempo de ejecución**: Directorio de los archivos HyperFileSQL. 
			Al cargar la biblioteca del servidor SOAP:

	- El análisis associated con el servidor SOAP se cargará automáticamente.

	- El directorio de datos especificado en el análisis corresponderá automáticamente a este directorio.




## EmailGetTimeOut( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Lee el valor del "timeout" para conectarse a los servidores de mensajería SMTP, POP3 e IMAP. El valor del "tiempo de espera" es set a 10 minutos por Default. Este valor puede ser modificado por [EmailSetTimeOut](../WDLang3/3032001.md).

**Recordatorio**: El tiempo de espera es el tiempo máximo permitido mientras se espera una respuesta del servidor de mensajería SMTP, POP3 o IMAP. Si el servidor no responde dentro de este tiempo de espera, la conexión falla.








<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = EmailGetTimeOut()`
---

**`<Result>: (Integro)`**

Valor de tiempo de espera en segundos. 
**Observación**: Siempre se define un "tiempo de espera" en el servidor. Por lo tanto, la función **EmailGetTimeOut** no puede devolver errores.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Funciones de conexión y tiempo de espera
<a name="funciones_conexion_tiempo_espera_ELTPARAGRAPHE000072"></a>

Las funciones de conexión ([EmailStartSession](../WDLang3/3032028.md), [EmailStartPOP3Session](../WDLang3/3032022.md) y [EmailStartSMTPSession](../WDLang3/3032025.md)) son funciones de bloqueo durante toda la conexión. Si el servidor no responde, estas funciones son funciones de bloqueo durante el tiempo de espera.

**Atención**: El valor del tiempo de espera definido por [EmailSetTimeOut](../WDLang3/3032001.md) no tiene correlación con la duración de [EmailStartSession](../WDLang3/3032028.md), [EmailStartPOP3Session](../WDLang3/3032022.md) y [EmailStartSMTPSession](../WDLang3/3032025.md). De hecho, es la duración de la falta de respuesta del servidor. A esta duración hay que añadirle el tiempo de la resolución de Address.

Por ejemplo, con un servidor que responde instantáneamente, la conexión puede tardar de 3 a 4 segundos si el DNS es lento (incluso si el tiempo de espera es de set a 1 segundo).

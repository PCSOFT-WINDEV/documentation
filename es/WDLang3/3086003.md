## Presentación
<a name="presentacion_ELTTEXTE000184"></a>
WINDEV le permite administrar fácilmente el protocolo SOAP permitiéndole crear:

- [Aplicaciones cliente SOAP](../WDLang3/3069008.md). Estas aplicaciones pueden ser utilizadas por cualquier servidor SOAP.

- Aplicaciones de servidor SOAP. Existen dos tipos de aplicaciones de servidor SOAP disponibles:

	- [Aplicación de servidor SOAP estándar](../WDLang3/3069004.md)instalado en un servidor.

	- [Webservice](../WDLang3/3086002.md)instalado en un servidor.





En estos dos últimos casos, la configuración del servidor SOAP es casi idéntica. Esta ayuda Page presenta las diferentes operaciones que deben realizarse para instalar el servidor SOAP.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo configurar el servidor WINDEV SOAP?
<a name="¿como_configurar_servidor_windev_soap_ELTTEXTE000208"></a>
Para instalar su aplicación en un servidor SOAP:

1. Cree la biblioteca WINDEV.

	- Para crear un servidor SOAP estándar: Cree la biblioteca (archivo.WDL) de su aplicación Server: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_bibli_WD.gif)


	- Para crear un servicio web: generar el Webservice: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_generation_WebService.gif)

			Se inicia el asistente para generar el Webservice. Para obtener más información, consulte [Generación de un servicio web](../WDLang3/3086002.md).
			**Atención**: Las DLLs WINDEV no deben ser renombradas.




2. Seleccione los archivos que se instalarán en el servidor SOAP.

	- La biblioteca WINDEV de su aplicación de servidor o Webservice (archivo\*.WDL).

	- Para un Webservice, el archivo XML que contiene la description del Webservice.

	- La aplicación WDSOAPConfig.EXE se encuentra en el directorio "Data" del directorio instalación de WINDEV.

	- Las DLLs WINDEV necesarias para ejecutar su aplicación. Compruebe si se han seleccionado las siguientes DLLs: wd28 0img.dll, wd28 0obj.dll, wd28 0pnt.dll, wd28 0std.dll y wd28 0vm.dll.

	- Las DLLs WINDEV específicas para el servidor HTTP utilizado: wd28 0sapa.dll para Apache 1.3, wd28 0sapa2.dll para Apache 2.0 y wd28 0siis para IIS, etc.


**Observación**: Tiene la posibilidad de crear una instalación de su aplicación: en la pestaña "Proyecto", en el grupo "Generación", amplíe "Configuración Procedure" y seleccione "Crear instalación Procedure". Sin embargo, con una aplicación de servidor SOAP, esta instalación sólo puede utilizarse para copy los archivos necesarios al servidor. La configuración del servidor debe realizarse posteriormente.

3. Instalar la aplicación en el servidor.  

4. Configure el servidor INTERNET para su aplicación WINDEV:

	- [Configuración para Apache](../WDLang3/3086004.md)

	- [Configuración para IIS](../WDLang3/3086005.md)

	- [Configuración para Netscape iPlanet 4.1](../WDLang3/3086006.md)




5. Configurar el administrador de SOAP. Para obtener más información, consulte [Configuración del administrador](../WDLang3/3086007.md).

6. Si está utilizando una aplicación cliente SOAP: para que el cliente SOAP Contact el servidor, el Address del servidor utilizado en [SOAPRun](../WDLang3/3069013.md) debe tener el siguiente formato: 
	
	```txt
	<IP address of server>\Server.soap
	```





**Observación**: Si varias aplicaciones de servidor SOAP o Webservices están instaladas en el mismo ordenador, debe instalarlas:

- todas las bibliotecas (WDL) en el mismo directorio,

- todas las DLLs WINDEV utilizadas en el mismo directorio.




Se instalará y utilizará un único administrador SOAP.

**Atención**: Los derechos concedidos al usuario del sitio http son muy importantes. Si un servidor WINDEV SOAP debe escribir en un archivo, los derechos de escritura deben ser otorgados al usuario del sitio (huésped de INTERNET).

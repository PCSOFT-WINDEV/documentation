## SOAP
<a name="soap_ELTTEXTE000105"></a>
SOAP (Simple Object Access Protocol) es un protocolo de comunicación utilizado para ejecutar procedimientos en un servidor remoto.. Este protocolo se basa principalmente en los protocolos HTTP y XML. También se puede utilizar con el protocolo SMTP.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Soap.gif)


Los datos se transmiten entre el ordenador local y el servidor SOAP como texto en formato XML (Extensible Markup Language).

**Beneficios del protocolo SOAP**
La principal ventaja del protocolo SOAP es que se basa en dos protocolos estándar:

- XML: para la estructura de los mensajes.

- HTTP: para la transferencia de datos.




Por esta razón, el protocolo SOAP es independiente de los sistemas operativos y de los lenguajes de programación.. El protocolo SOAP promueve la interoperabilidad.

Además, el uso del protocolo HTTP (para la transferencia de datos) permite que el protocolo SOAP pase a través de cortafuegos sin ningún problema.

## WINDEV y el protocolo SOAP
<a name="windev_protocolo_soap_ELTTEXTE000129"></a>
WINDEV permite:

- crear una **[Aplicación cliente SOAP](../WDLang3/3069008.md)**. Esta aplicación utiliza las funciones SOAP de WLanguage para hacer que los procedimientos de ejecución del servidor SOAP y recuperar su resultado.

- crear una **[Aplicación de servidor SOAP](../WDLang3/3069004.md)**. Esta aplicación incluye procedimientos que serán iniciados por la aplicación cliente.




**Observación**: Las transferencias no son seguras en este momento (los datos transferidos no están encriptados). El protocolo SOAP no debe utilizarse para transferir datos sensibles.

La versión SOAP utilizada: 

- para el servidor es 1.1 (WEBDEV servidor de aplicaciones).

- para el uso de los Webservices es 1.1 y 1.2.


## FTPSize( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el tamaño (en bytes) de un archivo encontrado en un servidor FTP (File Transfer Protocol).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003040007&name=ftpsize_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Find out the size of "/Documents/File.Doc" file 
// found on the FTP server
ResSize = FTPSize(7, "/Documents/File.DOC")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Averiguar el tamaño del último archivo consultado

`<Result> = FTPSize()`
---

**`<Result>: (Integro)`**



- Tamaño (en bytes) del último archivo consultado a través de una de las siguientes funciones FTP: [FTPAttribute](../WDLang3/3040006.md), [FTPDate](../WDLang3/3040015.md), [FTPTime](../WDLang3/3040013.md), [FTPListFile](../WDLang3/3040018.md), [FTPGet](../WDLang3/3040002.md).

- -1 si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.


**Observación: A diferencia de la sintaxis 2, no se envía ninguna petición al servidor. Por lo tanto, se recomienda utilizar esta sintaxis siempre que sea posible.**  


<a name="SYNTAX2"></a>

### Averiguar el tamaño del archivo especificado

`<Result> = FTPSize(<Connection identifier> , <File name>)`
---

**`<Result>: (Integro)`**



- Tamaño (en bytes) del archivo especificado,

- -1 si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Connection identifier>: (Integro)`**

Identificador de conexión, devuelto por la función [FTPConnect](../WDLang3/3040016.md).



**`<File name>: (Cadena de caracteres)`**

Nombre y ruta completa (o relativa) del archivo al que se accede. Las secciones de la ruta están separadas por barras oblicuas ("/").

## Observaciones


### Timeout
<a name="timeout_ELTPARAGRAPHE000097"></a>

De forma predeterminada, todas las funciones FTP fallan si no se recibe ninguna respuesta del servidor FTP después de 20 segundos. Este tiempo de espera se puede modificar al conectarse al servidor FTP con la función [FTPConnect](../WDLang3/3040016.md).
<a name="NOTE0_2"></a>
<a name="NOTE0_3"></a>




### Permisos de usuarios
<a name="permisos_usuarios_ELTPARAGRAPHE000126"></a>

Solo un usuario con permisos de lectura en el servidor FTP puede obtener información sobre los archivos o directorios que se encuentran en dicho servidor. En la mayoría de los casos, los usuarios "anónimos" tienen permisos de lectura.
<a name="NOTE0_4"></a>




### Ruta relativa o absoluta
<a name="ruta_relativa_absoluta_ELTPARAGRAPHE000134"></a>

Las nociones de ruta relativa y ruta absoluta son de gran importancia en una aplicación FTP.

- Una ruta que comienza con un slash se considera como una ruta **absoluta**:  es la ruta con respecto a la raíz del servidor FTP (parámetro específico del servidor).
	ej: /publicidad/usuario/JULIA

- Si la ruta no comienza con un slash, se considera como una **ruta relativa**, es decir, una ruta con respecto al directorio en curso. El directorio en curso puede conocerse o modificarse con la función [FTPCurrentDir](../WDLang3/3040003.md).


Al conectarse a un sitio FTP, el directorio inicial ("home directory" del usuario) no necesariamente es la raíz del servidor FTP. Por lo tanto, se recomienda utilizar rutas relativas.


## nfcData (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **nfcData** se utiliza para describir y manipular los datos de un Variable de tipo [nfcTag](../WDLang3/1000020411.md). Las características de estos datos pueden definirse y modificarse mediante diferentes propiedades de WLanguage.

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo






```wl
// Read the data of an NFC tag
tag is nfcTag
tag = nfcReadTag()
IF NOT ErrorOccurred = True THEN
	FOR EACH data of tag.Data
		SWITCH data.Type
			CASE nfcText
				sLanguage is string = LanguageToName(data.Language)
				sText is string = data.Content
			CASE nfcURI
				IF StringStartsWith(data.Content, "http") THEN
					BrowserRunApp(data.Content)
				END
			CASE nfcMedia
				IF data.MIMEType ~= "image/png" THEN
					fSaveBuffer(CompleteDir(SysDirStorageCard()) +  ...
						"image.png", data.Content)
				END
		END
	END
END
```

## Observaciones
<a name="NOTE0_1"></a>


### Propiedades específicas de la description de las variables nfcData
<a name="propiedades_especificas_description_las_variables_nfcdata_ELTPARAGRAPHE000047"></a>

Las siguientes propiedades pueden utilizarse para manipular los datos de una etiqueta NFC:

| **Nombre de la propiedad** | **Tipo utilizado** | **Efecto** |
| --- | --- | --- |
| Contenido | Binario/Buffer | Data content. Su tipo depende del valor de **Tipo** Property.<br><br>Si la propiedad **Type** corresponde a: <br><br>- la constante *nfcText*: <br><br><br><br><br>- la constante *nfcURI*, la propiedad **Content** corresponde a una cadena ANSI. <br><br>- la constante *nfcMedia*, la propiedad **..Content** corresponde a un buffer.<br>	La interpretación de los datos de la memoria intermedia depende del tipo de MIME asociado al **nfcData** Variable. <br><br>- la constante *nfcEmpty*: Se producirá un error fatal si se intenta modificar la propiedad **Content**.<br><br>- la constante *nfcUnknown*: Se producirá un error fatal si se intenta modificar la propiedad **Content**.<br><br><br><br><br> |
| escribir a máquina | Constante de tipo Integer | Tipo de datos, correspondientes a una de las siguientes constantes: <br><br>- *nfcText*: Datos de texto.<br><br>- *nfcURI*: Datos de URI (Uniform Resource Identifier). Puede corresponder a: <br><br>	- la url de una página web (http://www.windev.com),<br><br>	- un FTP Address (ftp://ftp.windev.com/),<br><br>	- un número de Phone (tel:0123456789),<br><br>	- un correo electrónico Address (mailto:adresse@mail.com), etc..<br>			Tiene la posibilidad de utilizar [ShellExecute](../WDLang1/3035007.md) con este URI (por ejemplo, para abrir el navegador cuando el URI corresponde al Address de un sitio web).<br><br><br><br><br>- *nfcMedia*: Media data. Puede corresponder a cualquier Resource identificado por un tipo de MIME (imagen, video, Document, etc.).<br><br>- *nfcEmpty*: Tipo de vacío. Este tipo se utiliza cuando se crea una variable de tipo **nfcData** siempre que no se haya especificado ningún tipo de datos.<br><br>- *nfcUnknown*: Tipo desconocido.<br><br><br>**Importante note**: Al modificar el tipo de una variable **nfcData**, se reinicializa el contenido. Por lo tanto, no olvide especificar el tipo antes del contenido.<br><br> |
| Idioma | Integro | Idioma del texto almacenado en el **Content** Property.<br><br>Esta propiedad debe utilizarse únicamente cuando el tipo de la variable **nfcData** sea *nfcText*. Se producirá un error fatal si se intenta leer o modificar la **Lenguaje** Property de una variable de tipo **nfcData** o de un tipo diferente. <br><br>Por defecto, el **Language** Property se inicializa con el idioma actual de la aplicación.<br><br>Para averiguar: <br><br>- el nombre del idioma, utilice [IdiomaNombre](../WDLang1/3054020.md).<br><br>- la nación correspondiente a la lengua, utilice [LanguageToNation](../WDLang1/3054019.md).<br><br><br><br><br> |
| MIMEType | Cadena de caracteres | Tipo MIME del recurso almacenado en la propiedad **..Content**. <br><br>El tipo MIME especificado puede corresponder a: <br><br>- un tipo MIME estándar. Más de 150 tipos están definidos en el estándar de comunicación MIME (disponible en INTERNET). Los valores más comunes son: <br><br>	- "solicitud/pdf": Document en formato PDF (\*.pdf)<br><br>	- "text/html": Página HTML (\*.htm,\*.html)<br><br>	- "text/plain": archivo de texto (\*.txt)<br><br>	- "Image/gif": Image en formato GIF (\*.gif)<br><br>	- "Image/jpeg": Image en formato JPEG (\*.jpg, \*.jpeg)<br><br>	- "video/mpeg": vídeo en formato MPEG (\*.mpg,\*.mpeg)<br><br>	- "solicitud / palabra clave": archivo de Word. <br><br>	- "application/vnd.ms-excel": Excel file.<br><br><br><br><br>- un tipo personalizado. <br><br><br>El **MIMEType** Property debe utilizarse sólo cuando la variable **nfcData** es del tipo *nfcMedia*. Se producirá un error fatal si se intenta leer o modificar el **MIMEType** Property de una variable de tipo **nfcData** o de otro tipo.<br><br> |


<a name="NOTE0_2"></a>


### Reinicio
<a name="reinicio_ELTPARAGRAPHE000176"></a>

Puede utilizar [VariableReset](../WDLang1/3013069.md) para restablecer el contenido de una variable de tipo **nfcData**.

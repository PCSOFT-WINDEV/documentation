


## EmailConfigure( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite configurar las funciones de WLangage para la gestión de Email. 

Esta función es useful si se utiliza un código multiplataforma. Esto le permite utilizar el mismo modo de gestión de los correos electrónicos para sus aplicaciones Windows y Linux, para sus aplicaciones móviles (Android/iOS) y para sus sitios WEBDEV. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Enable the multi-platform implementation
EmailConfigure(emailParameterMode, 1)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`EmailConfigure(<Option> , <Option value>)`
---

**`<Option>: (Constante de tipo Integer)`**

Opción para definir: 


|   |   |
| --- | --- |
| *emailParameterHTML* | Cuando se leen correos de un servidor POP o IMAP, la parte HTML consiste en un encabezado y el contenido HTML mismo. El conjunto de caracteres que indica la codificación de los caracteres (UTF-8, ISO 8859-1, ISO-xxxx, etc.) puede estar en una o ambas partes.<br><br>- Si &lt;Valor de opción&gt; es set a 0 (valor por defecto), las funciones de Email sólo usarán el charset especificado en el encabezado Email para manipular el contenido HTML. <br><br>- Si &lt;Valor de opción&gt; es set a 1, las funciones de correo electrónico usarán el charset especificado en el encabezado del correo electrónico a manipular el contenido HTML, si no se ha especificado ningún charset en la propia parte HTML. En algunos casos, esto evita los errores de visualización de los caracteres acentuados y especiales. <br><br><br> |
| *emailParameterMode* | Permite elegir el modo de funcionamiento de los correos electrónicos.<br><br>- Si &lt;Valor de opción&gt; se pone a 0, las funciones para la gestión del correo electrónico utilizan el módulo "wininet" de Windows para los protocolos IMAP, POP3 y SMTP en modo SSL (constante *optionSSL*) o en modo STARTTLS (constante *emailOptionSecuredTLS*). Este modo corresponde al modo de funcionamiento de las funciones de Default para la gestión de Email.<br><br>- Si &lt;Valor de opción&gt; es set a 1, las funciones para la gestión de Email utilizan una implementación que soporta la multiplataforma. Por lo tanto, la gestión de Email es la misma para sus plataformas Windows, Linux y móvil (Android/iOS).<br><br><br> |



**`<Option value>: (Integro)`**

Nuevo valor de la opción.

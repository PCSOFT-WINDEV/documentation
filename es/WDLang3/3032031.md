## Descripción general de los protocolos POP3/SMTP
<a name="descripcion_general_los_protocolos_pop3smtp_ELTTEXTE000233"></a>


### Los protocolos POP3 y SMTP
<a name="los_protocolos_pop3_smtp_ELTPARAGRAPHE000011"></a>

- El protocolo POP3 (Post Office Protocol) se utiliza para recibir correos electrónicos.

- El protocolo SMTP (Simple Mail Transfer Protocol) se utiliza para enviar correos electrónicos.


Observaciones: 

- La recepción de correos electrónicos también puede tenerse en cuenta mediante el uso de la [protocolo IMAP](../WDLang3/1000019172.md).

## Principio
<a name="principio_ELTTEXTE000257"></a>


### Envío de mensajes utilizando el protocolo SMTP
<a name="envio_mensajes_utilizando_protocolo_smtp_ELTPARAGRAPHE000058"></a>

Para enviar mensajes utilizando el protocolo SMTP, debe hacerlo:

1. Inicie una sesión SMTP:

	- utilizando un [emailSMTPSession](../WDLang3/1000018765.md) variable y [EmailStartSession](../WDLang3/3032028.md). 

	- utilizando [EmailStartSMTPSession](../WDLang3/3032025.md).




2. Construye el mensaje a enviar en un Variable de tipo [Email](../WDLang3/1000018713.md) o en la estructura [Email](../WDLang3/3032029.md).

3. Envía el mensaje con [EmailSendMessage](../WDLang3/3032005.md).

4. Cierre la sesión SMTP con [EmailCloseSession](../WDLang3/3032006.md).





### Recepción de mensajes utilizando el protocolo POP3
<a name="recepcion_mensajes_utilizando_protocolo_pop3_ELTPARAGRAPHE000092"></a>

Para recibir correos electrónicos utilizando el protocolo POP3, debe hacerlo:

1. Inicie una sesión POP3:

	- utilizando un [emailPOP3Session](../WDLang3/1000018759.md) variable y [EmailStartSession](../WDLang3/3032028.md). 

	- utilizando [EmailStartPOP3Session](../WDLang3/3032022.md).


**Ejemplo para iniciar una sesión utilizando un emailPOP3Session Variable:**
	
	```wl
	// Start the messaging session
	MySession is emailPOP3Session
	MySession.ServerAddress = "pop.mycompany.us"
	MySession.Name = "user"
	MySession.Password = "secret"
	EmailStartSession(MySession)
	```


2. Lea los mensajes en el servidor de mensajería:

	- con la función [EmailGetAll](../WDLang3/1000018727.md).

	- con un bucle como:

		- **Ejemplo que está usando un correo electrónico Variable:**
						
			```wl
			MyMessage is Email
			EmailReadFirst(MySession, MyMessage)
			WHILE NOT MyMessage.Out
				// Place the process of the message read...
				...
				
				EmailReadNext(MySession, MyMessage)
			END
			```


		- **Ejemplo que utiliza la estructura Email:**
						
			```wl
			EmailReadFirst(MySession)
			WHILE NOT Email.Out
				// Place the process of the message read...
				...
				
				EmailReadNext(MySession)
			END
			```





- o con un bucle como:
			
		```wl
		MyMessage is Email
		FOR EACH MyMessage OF MySession
			// Place the process of the message read...
			...
		END
		```


3. Cierre la sesión POP3 con [EmailCloseSession](../WDLang3/3032006.md).




**Observación**: [EmailStartSession](../WDLang3/3032028.md) puede iniciar simultáneamente una sesión POP3 y una sesión SMTP.

### Principio y características específicas de WEBDEV
<a name="principio_caracteristicas_especificas_webdev_ELTPARAGRAPHE000154"></a>
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Pop3_WEBDEV.gif)


En un sitio WEBDEV, los correos electrónicos son enviados y recibidos por el servidor de aplicaciones.

El envío puede ser realizado por un spooler local para no ralentizar un sitio WEBDEV si el servidor SMTP tarda en responder.. Esta función es administrada automáticamente por el servidor de aplicaciones WEBDEV.

### Conexión con el proveedor de servicios
<a name="conexion_con_proveedor_servicios_ELTPARAGRAPHE000171"></a>

Se necesita una conexión INTERNET para gestionar los correos electrónicos. Pueden ocurrir varios casos:

- El usuario utiliza un módem para conectarse a INTERNET: [NetOpenRemoteAcceso remoto](../WDLang3/3056026.md) establece la conexión con INTERNET.

- El usuario utiliza una conexión directa a INTERNET (cable o ADSL): no se requiere ninguna operación específica.




**Observación**: Para limitar el tiempo de conexión:

- Al leer los correos electrónicos, le aconsejamos que lea todos los correos entrantes y que los guarde en una base de datos HFSQL, por ejemplo.

- Al enviar los correos electrónicos, le aconsejamos que los prepare de antemano.

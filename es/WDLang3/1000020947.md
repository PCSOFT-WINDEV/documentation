
## FTPConfigure( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica el comportamiento de las funciones FTP estándar. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Enable the native implementation of FTP
FTPConfigure(ftpConfigureMode, 1)
```


<a name="Example2"></a>

```wl
// Enable the Windows implementation ("wininet") of FTP
FTPConfigure(ftpConfigureMode, 0)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`FTPConfigure(<Option> , <Option value>)`
---

**`<Option>: (Constante de tipo Integer)`**

Opción para definir: 


|   |   |
| --- | --- |
| *ftpConfigureMode* | Selecciona el modo de funcionamiento del FTP.<br><br>- **Si &lt;Valor de la opción&gt; es set a 0**, las funciones de gestión FTP utilizan el módulo "wininet" de Windows excepto para los protocolos seguros ([FTPIS, FTPES y SFTP](../WDLang3/3040016.md)). Este modo corresponde al modo de funcionamiento de Default de las funciones FTP.<br><br>- **Si &lt;valor de la opción&gt; es set a 1**, las funciones de gestión de FTP utilizan una implementación nativa e independiente de Windows de FTP, con soporte UTF-8. <br>	**Atención**:  En este modo, [FTPCommand](../WDLang3/3040017.md) no es compatible. Este modo se utiliza sistemáticamente para los protocolos seguros ([FTPIS, FTPES y SFTP](../WDLang3/3040016.md)).<br><br><br> |
| *tpesParameterSecureMode* | Cambia la encriptación FTPES.<br><br>- **Si &lt;valor de la opción&gt; es set a 0**, la encriptación se aplica a los datos y comandos FTP. Este es el modo Default para las funciones en FTPES.<br><br>- **Si &lt;valor de la opción&gt; es set a 1**, el cifrado SSL se aplica sólo a los comandos FTP. Este modo puede ser necesario para algunos servidores FTPES.<br><br><br> |



**`<Option value>: (Integro)`**

Nuevo valor de la opción. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Las funciones FTPxxx de WLanguage utilizan por defecto las funciones FTP de Windows API( WINET.DLL). En Windows 8.1, estas APIs ya no devuelven los resultados esperados:

- transferencia no completada,

- "incapaz de modificar el directorio a.".




Mientras se esperan las funciones FTP estándar para ser restauradas por Windows Update, el comportamiento de las funciones FTPxxx de WLanguage puede modificarse para que ya no se basen en las funciones FTP de Windows API. Para ello, inserte la siguiente llamada en la transferencia FTP Process, antes de la llamada existente a [FTPConnect](../WDLang3/3040016.md): 


```wl
FTPConfigure(ftpConfigureMode, 1)
```


Observación: La función **FTPConfigure** se aplica a todas las funciones FTPxxx de WLanguage, excepto a la función [FTPCommand](../WDLang3/3040017.md). Si se ha utilizado éste, puede sustituirse por [FTPSend](../WDLang3/3040014.md) o [FTPGet](../WDLang3/3040002.md) según el caso.

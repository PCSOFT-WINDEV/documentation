
## LDAPStartAdd( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicializa la adición de un nuevo objeto en un servidor LDAP.

**Recordatorio**: Para añadir un nuevo objeto a un servidor LDAP, debe definir los diferentes atributos de este objeto así como sus valores.

Para optimizar la velocidad de creación de los objetos, los atributos y sus valores se definen localmente: se establece una comunicación única con el servidor al validar la adición. Los pasos son:

1. Inicializar la adición (función **LDAPStartAdd**).

2. Añadir atributos y sus valores [Atributo LDAPAddAttribute](../WDLang3/3032110.md)). Esta operación se puede realizar tantas veces como sea necesario.

3. Asociar los atributos y sus valores a un objeto ([LDAPValidateAdd](../WDLang3/3032111.md)).


**Atención**: Al realizarse estas operaciones localmente y luego en el servidor, debe respetarse el orden lógico de las modificaciones (no se puede, por ejemplo, modificar un atributo que ya fue borrado por la Line de código anterior).  
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Add a Test user into an "Active Directory"
LDAPStartAdd("MyLDAPSession")
LDAPAddAttribute("MyLDAPSession", "objectClass", "top")
LDAPAddAttribute("MyLDAPSession", "objectClass", "person")
LDAPAddAttribute("MyLDAPSession", "objectClass", "organizationalPerson")
LDAPAddAttribute("MyLDAPSession", "objectClass", "user")
LDAPAddAttribute("MyLDAPSession", "objectCategory", ...
	"CN=Person, CN=Schema, CN=Configuration, DC=company, DC=us")
LDAPAddAttribute("MyLDAPSession", "instanceType", "4")
LDAPAddAttribute("MyLDAPSession", "cn", "Test")
LDAPAddAttribute("MyLDAPSession", "sAMAccountName", "Test")
LDAPValidateAdd("MyLDAPSession", "cn=Test,cn=Users,dc=company,dc=us")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`LDAPStartAdd(<LDAP session>)`
---

**`<LDAP session>: (Cadena de caracteres)`**

Nombre de la sesión LDAP utilizada.

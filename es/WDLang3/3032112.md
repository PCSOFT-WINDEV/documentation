
## LDAPStartModify( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia la modificación de un objeto existente en un servidor LDAP.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Deletes a given value (read on the LDAP server)
// the value to delete is found in the sValueToDelete string
LDAPStartModify("MyLDAPSession")
LDAPDeleteAttributeValue("MyLDAPSession", "AttributeName", sValueToDelete)
LDAPValidateModify("MyLDAPSession", "DN")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`LDAPStartModify(<LDAP session>)`
---

**`<LDAP session>: (Cadena de caracteres)`**

Nombre de la sesión LDAP utilizada.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Modificación de un objeto existente en un servidor LDAP
<a name="modificacion_objeto_existente_servidor_ldap_ELTPARAGRAPHE000032"></a>

Para modificar un objeto existente en un servidor LDAP, debe definir las diferentes modificaciones a realizar.

Para optimizar la velocidad de modificación de los objetos, las modificaciones se definen localmente: se establece una comunicación única con el servidor al validar la modificación. Los pasos son:

1. Inicializar la modificación (función **LDAPStartModify**).

2. Operaciones de modificación:

	- Añadir atributos y sus valores [Atributo LDAPAddAttribute](../WDLang3/3032110.md)).

	- Borrar atributos ([LDAPDeleteAttribute](../WDLang3/3032115.md)) o sus valores ([LDAPDeleteAttributeValue](../WDLang3/3032116.md)).

	- Sustituir los valores de un atributo ([Atributo LDAPReplaceAttribute](../WDLang3/3032113.md) o [LDAPReplaceAttributeValue](../WDLang3/3032119.md)).

	- Cambiar el nombre de los atributos ([LDAPRenameAttribute](../WDLang3/3032120.md)).




3. Validar las modificaciones en el servidor ([LDAPValidateModify](../WDLang3/3032117.md)). Las modificaciones se realizan en el servidor según el orden utilizado en el código de WLanguage.




**Atención**: Al realizarse estas operaciones localmente y luego en el servidor, debe respetarse el orden lógico de las modificaciones (no se puede, por ejemplo, modificar un atributo que ya fue borrado por la Line de código anterior).

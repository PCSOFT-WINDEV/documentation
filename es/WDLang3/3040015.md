


## FTPDate (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve las diferentes fechas de un archivo (fecha de creación, modificación o acceso) ubicado en un servidor FTP (File Transfer Protocol).
![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Devuelve la fecha de la última modificación de un archivo ubicado en un servidor FTP (File Transfer Protocol).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003040015&name=ftpdate_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Get the modification date of
// "/Documents/File.txt" on the FTP server
ResDate = FTPDate(ConnectionNum, "/Documents/File.txt")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Obtener las fechas del último archivo al que se accedió

`<Result> = FTPDate([<Type of date>])`
---

**`<Result>: (Cadena de caracteres)`**



- Fecha (en formato AAAAMMDD) del último archivo al que se accedió mediante una de las siguientes funciones FTP: [FTPAttribute](../WDLang3/3040006.md), [FTPTime](../WDLang3/3040013.md), [FTPSize](../WDLang3/3040007.md), [FTPListFile](../WDLang3/3040018.md).

- Cadena vacía ("") si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.


![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Fecha de modificación del último archivo al que se accedió.**A diferencia de la sintaxis 2, no se envía ninguna solicitud al servidor. Por lo tanto, se recomienda utilizar esta sintaxis siempre que sea posible.**

**`<Type of date>: (Constante opcional)`**

Indica la fecha a manipular:


|   |   |
| --- | --- |
| *ftpAccess* | Fecha en la que se accedió al archivo por última vez. |
| *ftpCreation* | Fecha de creación del archivo. |
| *ftpModification*<br>(Valor predeterminado) | Fecha de la última modificación del archivo. |

![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Este parámetro no está disponible. Solo se devuelve la fecha de la última modificación del archivo.


<a name="SYNTAX2"></a>

### Obtener las fechas del archivo especificado

`<Result> = FTPDate(<Connection identifier> , <File name> [, <Type of date>])`
---

**`<Result>: (Cadena de caracteres)`**



- Fecha del archivo especificado (en formato AAAAMMDD),

- Cadena vacía ("") si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Fecha de modificación del último archivo al que se accedió.

**`<Connection identifier>: (Entero)`**

Identificador de conexión, devuelto por la función [FTPConnect](../WDLang3/3040016.md).



**`<File name>: (Cadena de caracteres)`**

Nombre y ruta completa (o relativa) del archivo al que se accede. Las secciones de la ruta están separadas por barras oblicuas ("/").

**`<Type of date>: (Constante opcional)`**

Indica la fecha a manipular:


|   |   |
| --- | --- |
| *ftpAccess* | Fecha en la que se accedió al archivo por última vez. |
| *ftpCreation* | Fecha de creación del archivo. |
| *ftpModification*<br>(Valor predeterminado) | Fecha de la última modificación del archivo. |

![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Este parámetro no está disponible. Solo se devuelve la fecha de la última modificación del archivo.

## Observaciones


### Fecha del archivo
<a name="fecha_del_archivo_ELTPARAGRAPHE000144"></a>

Debido a un error en un componente de Microsoft, el año devuelto por **FTPDate** podría no ser válido para algunas versiones de Internet Explorer 5.0. Si el archivo se creó o modificó el mismo día, puede que la fecha sea de un año anterior a la fecha real.
<a name="NOTE0_2"></a>


### Tiempo de espera
<a name="tiempo_espera_ELTPARAGRAPHE000153"></a>

De forma predeterminada, todas las funciones FTP fallan si no se recibe ninguna respuesta del servidor FTP después de 20 segundos. Este tiempo de espera se puede modificar al conectarse al servidor FTP con la función [FTPConnect](../WDLang3/3040016.md).
<a name="NOTE0_3"></a>
<a name="NOTE0_4"></a>


### Permisos de usuarios
<a name="permisos_usuarios_ELTPARAGRAPHE000181"></a>

Solo un usuario con permisos de lectura en el servidor FTP puede obtener información sobre los archivos o directorios que se encuentran en dicho servidor. En la mayoría de los casos, los usuarios "anónimos" tienen permisos de lectura.
<a name="NOTE0_5"></a>


### Ruta relativa o absoluta
<a name="ruta_relativa_absoluta_ELTPARAGRAPHE000188"></a>

Las nociones de ruta relativa y ruta absoluta son de gran importancia en una aplicación FTP.

- Una ruta que comienza con un slash se considera como una ruta **absoluta**:  es la ruta con respecto a la raíz del servidor FTP (parámetro específico del servidor).
	ej: /publicidad/usuario/JULIA

- Si la ruta no comienza con un slash, se considera como una **ruta relativa**, es decir, una ruta con respecto al directorio en curso. El directorio en curso puede conocerse o modificarse con la función [FTPCurrentDir](../WDLang3/3040003.md).


Al conectarse a un sitio FTP, el directorio inicial ("home directory" del usuario) no necesariamente es la raíz del servidor FTP. Por lo tanto, se recomienda utilizar rutas relativas.




## FTPDateTime( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve las diferentes fechas y horas (creación, modificación o acceso) de associated con un archivo encontrado en un servidor FTP (File Transfer Protocol).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
ResDate = FTPDateTime(ConnectionNum, "/Documents/File.txt")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Averiguar las diferentes fechas y horas de associated con el último archivo visto

`<Result> = FTPDateTime([<Type of date>])`
---

**`<Result>: (Cadena de caracteres o variable de tipo DateTime)`**



- Fecha y hora (formato YYYMMDDHHmmSSCC) associated con el último archivo visualizado a través de una de las siguientes funciones FTP: [FTPAttribute](../WDLang3/3040006.md), [FTPTime](../WDLang3/3040013.md), [FTPSize](../WDLang3/3040007.md), [FTPListFile](../WDLang3/3040018.md).

- Cadena vacía ("") si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**A diferencia de la sintaxis 2, no se envía ninguna solicitud al servidor. Esta sintaxis se recomienda siempre que sea posible**. 

**`<Type of date>: (Constante opcional de tipo Integer)`**

Fecha y hora de uso: 


|   |   |
| --- | --- |
| *ftpAccess* | Fecha y hora del último acceso al archivo. |
| *ftpCreation* | Fecha y hora de creación del archivo. |
| *ftpModification*<br>(Valor predeterminado) | Fecha y hora de la última modificación del archivo. |




<a name="SYNTAX2"></a>

### Averiguar las diferentes fechas y horas de associated con el archivo especificado

`<Result> = FTPDateTime(<Connection identifier> , <File name> [, <Type of date>])`
---

**`<Result>: (Cadena de caracteres o variable de tipo DateTime)`**



- Fecha y hora del archivo especificado (en formato YYYYMMDDHHmmSS),

- Cadena vacía ("") si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Connection identifier>: (Integro)`**

Identificador de conexión, devuelto por la función [FTPConnect](../WDLang3/3040016.md).

**`<File name>: (Cadena de caracteres)`**

Nombre y ruta completa (o relativa) del archivo al que se accede. Las secciones de la ruta están separadas por barras oblicuas ("/").

**`<Type of date>: (Constante opcional de tipo Integer)`**

Fecha y hora de uso: 


|   |   |
| --- | --- |
| *ftpAccess* | Fecha y hora del último acceso al archivo. |
| *ftpCreation* | Fecha y hora de creación del archivo. |
| *ftpModification*<br>(Valor predeterminado) | Fecha y hora de la última modificación del archivo. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Fecha del archivo
<a name="fecha_del_archivo_ELTPARAGRAPHE000117"></a>

Debido a un error en un componente de Microsoft, el año devuelto por **FTPDateTime** podría no ser válido para algunas versiones de Internet Explorer 5.0. Si el archivo se creó o modificó el mismo día, puede que la fecha sea de un año anterior a la fecha real.
<a name="NOTE0_2"></a>


### Timeout
<a name="timeout_ELTPARAGRAPHE000126"></a>

De forma predeterminada, todas las funciones FTP fallan si no se recibe ninguna respuesta del servidor FTP después de 20 segundos. Este tiempo de espera se puede modificar al conectarse al servidor FTP con la función [FTPConnect](../WDLang3/3040016.md).
<a name="NOTE0_3"></a>
<a name="NOTE0_4"></a>


### Permisos de usuarios
<a name="permisos_usuarios_ELTPARAGRAPHE000152"></a>

Solo un usuario con permisos de lectura en el servidor FTP puede obtener información sobre los archivos o directorios que se encuentran en dicho servidor. En la mayoría de los casos, los usuarios "anónimos" tienen permisos de lectura.
<a name="NOTE0_5"></a>


### Ruta relativa o absoluta
<a name="ruta_relativa_absoluta_ELTPARAGRAPHE000159"></a>

Las nociones de ruta relativa y ruta absoluta son de gran importancia en una aplicación FTP.

- Una ruta que comienza con un slash se considera como una ruta **absoluta**:  es la ruta con respecto a la raíz del servidor FTP (parámetro específico del servidor).
	Ejemplo: /publicidad/usuario/JULIA

- Si la ruta no comienza con un slash, se considera como una **ruta relativa**, es decir, una ruta con respecto al directorio en curso. El directorio en curso puede conocerse o modificarse con la función [FTPCurrentDir](../WDLang3/3040003.md).




Al conectarse a un sitio FTP, el directorio inicial ("home directory" del usuario) no necesariamente es la raíz del servidor FTP. Por lo tanto, se recomienda utilizar rutas relativas.

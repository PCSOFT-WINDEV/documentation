
## LDAPRenameAttribute( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Renombra un atributo. El valor de los atributos se almacena durante esta operación.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// "AttributeName" is renamed to "Name2"
LDAPStartModify("SessionName")
LDAPRenameAttribute("SessionName", "AttributeName", "Name2")
LDAPValidateModify("SessionName", "DN")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = LDAPRenameAttribute(<LDAP session> , <Former attribute name> , <New attribute name>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ha realizado la modificación, 

- <u><u><u><u>False</u></u></u></u> en caso contrario. [ErrorInfo](../WDLang1/3013008.md) devuelve más detalles sobre el error. 


La modificación se realizará durante la llamada a [LDAPValidateModify](../WDLang3/3032117.md).

**`<LDAP session>: (Cadena de caracteres)`**

Nombre de la sesión LDAP utilizada.

**`<Former attribute name>: (Cadena de caracteres)`**

Nombre actual del atributo a utilizar.

**`<New attribute name>: (Cadena de caracteres)`**

Nuevo nombre del atributo a utilizar.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Renombrar
<a name="renombrar_ELTPARAGRAPHE000053"></a>

El nuevo nombre del atributo debe ser un atributo válido para el objeto modificado.

Si la operación de renombrado se refiere a un atributo de varios valores, el nuevo nombre debe corresponder a un atributo válido y de varios valores.. De lo contrario, se producirá un error al utilizar [LDAPValidateModify](../WDLang3/3032117.md).
<a name="NOTE0_2"></a>




### Modificación de un objeto existente en un servidor LDAP
<a name="modificacion_objeto_existente_servidor_ldap_ELTPARAGRAPHE000066"></a>

Para modificar un objeto existente en un servidor LDAP, debe definir las diferentes modificaciones a realizar.

Para optimizar la velocidad de modificación de los objetos, las modificaciones se definen localmente: se establece una comunicación única con el servidor al validar la modificación. Los pasos son:

1. Modificación de inicio ([LDAPStartModificar](../WDLang3/3032112.md)).

2. Operaciones de modificación:

	- Añadir atributos y sus valores [Atributo LDAPAddAttribute](../WDLang3/3032110.md)).

	- Borrar atributos ([LDAPDeleteAttribute](../WDLang3/3032115.md)) o sus valores ([LDAPDeleteAttributeValue](../WDLang3/3032116.md)).

	- Sustituir los valores de un atributo ([Atributo LDAPReplaceAttribute](../WDLang3/3032113.md) o [LDAPReplaceAttributeValue](../WDLang3/3032119.md)).

	- Renombrar atributos (**LDAPRenameAttribute**)




3. Validar las modificaciones en el servidor ([LDAPValidateModify](../WDLang3/3032117.md)). Las modificaciones se realizan en el servidor según el orden utilizado en el código de WLanguage.


## OBEXGetFile( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Recupera un archivo de un dispositivo que soporta el protocolo OBEX.

**Observación**: Esta función no se puede utilizar si la transferencia se realiza por infrarrojos.
<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = OBEXGetFile(<Identifier of OBEX connection> , <Remote file path> [, <Local file path>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la transferencia correctamente,

- <u><u><u><u>False</u></u></u></u> en caso contrario. [ErrorInfo](../WDLang1/3013008.md) devuelve más información sobre el error.




**`<Identifier of OBEX connection>: (Integro)`**

Identificador de la conexión OBEX a utilizar. Este identificador es devuelto por [OBEXConnect](../WDLang3/1000017153.md).

**`<Remote file path>: (Cadena de caracteres)`**

Nombre y ruta del archivo a recuperar en el dispositivo remoto.

**Observación**: En un móvil, puedes especificar un directorio en los parámetros de Bluetooth. Este directorio corresponde a la raíz de la estructura del árbol OBEX (que significa el directorio "\\" dado por [OBEXListFile](../WDLang3/1000017148.md)).

**`<Local file path>: (Cadena de caracteres opcional)`**

ruta completa donde se copiará el archivo. Si no se especifica este parámetro, el archivo se crea con el mismo nombre en el directorio actual.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Sólo las pilas de Microsoft bluetooth son soportadas por esta función. Para get la pila utilizada, llame a [BTStack](../WDLang3/1000017229.md).

Para obtener más información, consulte [¿Qué pilas utilizar?](../WDLang3/1000017231.md)


## FTP constants
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


| Constante | description | Utilizado por las funciones |
| --- | --- | --- |
| **ftpAccess** | Tipo de fecha/hora: Fecha/Hora del último acceso al fichero | [FTPDate](../WDLang3/3040015.md), <br>[FTPTime](../WDLang3/3040013.md) |
| **ftpActiveMode** | Conexión de **Active** al servidor FTP. <br><br>Al transferir archivos, el cliente inicia la transferencia. Este tipo de conexión puede ser rechazada por algunos servidores FTP protegidos. | [FTPConnect](../WDLang3/3040016.md) |
| **ftpASCIIMode** | Transferencia de archivos realizada en modo ASCII: Este modo se utiliza para transferir datos entre dos sistemas operativos diferentes, por ejemplo, de UNIX a Windows: el archivo transferido cambia al formato ASCII del sistema de destino. | [FTPSend](../WDLang3/3040014.md), <br>[FTPGet](../WDLang3/3040002.md) |
| **ftpBinaryMode** | Transferencia de archivos realizada en modo binario: el archivo recibido es exactamente idéntico al archivo enviado | [FTPSend](../WDLang3/3040014.md), <br>[FTPGet](../WDLang3/3040002.md) |
| **ftpCreation** | Tipo de fecha/hora: Fecha/Hora de creación del archivo | [FTPDate](../WDLang3/3040015.md), <br>[FTPTime](../WDLang3/3040013.md) |
| **ftpDirectory** | Elemento a buscar: búsqueda realizada sólo en los directorios | [FTPListFile](../WDLang3/3040018.md) |
| **ftpErrorAuthentification** | Notifica un error de conexión causado por un problema de identificación | [FTPConnect](../WDLang3/3040016.md) |
| **ftpExtendedPassiveMode** | **Conexión pasiva extendida** al servidor FTP (modo EPSV). <br><br>Al transferir archivos, el servidor toma la iniciativa para la transferencia. Este tipo de conexión se utiliza para pasar por encima de algunos "firewalls".. <br><br><br><br>Si el servidor no soporta el modo pasivo extendido (el comando FEAT se envía al servidor para averiguar la capacidad), la conexión se interrumpe en modo pasivo simple.. | [FTPConnect](../WDLang3/3040016.md) |
| **ftpFile** | Elemento a buscar: búsqueda realizada sólo en los archivos | [FTPListFile](../WDLang3/3040018.md) |
| **ftpIgnoreExpiredCertificate** | La fecha de Certificate es ignorada. | [FTPConnect](../WDLang3/3040016.md) |
| **ftpIgnoreInvalidCertificate** | El Certificate es ignorado. | [FTPConnect](../WDLang3/3040016.md) |
| **ftpIgnoreInvalidCertificateName** | El nombre del sitio especificado en la Certificate es ignorado. | [FTPConnect](../WDLang3/3040016.md) |
| **ftpIgnoreRevocation** | Se ignora la revocación de Certificate. | [FTPConnect](../WDLang3/3040016.md) |
| **ftpModification** | Tipo de fecha/hora: Fecha/Hora de la última modificación realizada en el fichero | [FTPDate](../WDLang3/3040015.md), <br>[FTPTime](../WDLang3/3040013.md) |
| **ftpParameterMode** | Selecciona el modo de funcionamiento del FTP. Hay dos modos disponibles:<br><br>- las funciones de gestión FTP utilizan el módulo "wininet" de Windows. Este modo corresponde al modo de funcionamiento de Default de las funciones FTP.<br><br>- las funciones para la gestión de FTP utilizan un módulo independiente de Windows y soportan el UTF8. Atención: En este modo, [FTPCommand](../WDLang3/3040017.md) no es compatible.<br><br><br> | [Parámetros FTP](../WDLang3/1000020947.md) |
| **ftpPassiveMode** | **Conexión pasiva** al servidor FTP (modo PASV). <br><br>Al transferir archivos, el servidor toma la iniciativa para la transferencia. Este tipo de conexión se utiliza para pasar por encima de algunos "firewalls".<br><br><br><br>Observación: Si la conexión con el servidor está en IPv6, la conexión se realizará en modo pasivo extendido (porque el modo pasivo no soporta IPv6). | [FTPConnect](../WDLang3/3040016.md) |
| **ftpPassiveModeIgnoreIP** | **Conexión pasiva** al servidor FTP (modo PASV). <br><br>Cuando se transfieren archivos en modo pasivo, el servidor establece el puerto y la IP Address para la transferencia de datos. Con este parámetro, sólo se tiene en cuenta el puerto. La IP Address se ignora.<br>Este tipo de conexión permite el uso de servidores FTP que devuelven IP Address es no enrutables pero que son alcanzables a través de la conexión IP Address | [FTPConnect](../WDLang3/3040016.md) |
| **ftpProxyNone** | Permite a Cancel el uso del proxy FTP. | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxyOpen** | Tipo de proxy a utilizar: Proxy FTP que requiere el comando "OPEN". | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxySite** | Tipo de proxy a utilizar: Proxy FTP que requiere el comando "SITE". | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxySOCKS5** | Proxy FTP que utiliza el protocolo SOCKS5. | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxyUserSeparate** | Tipo de proxy a utilizar: Proxy FTP que requiere el comando "USUARIO" y el comando "PASS" por separado. | [FTPProxy](../WDLang3/1000019366.md) |
| **ftpProxyUserTogether** | Tipo de proxy a utilizar: Proxy FTP que requiere el comando "USUARIO" con el usuario y la contraseña. | [FTPProxy](../WDLang3/1000019366.md) |
| **tpesParameterSecureMode** | Cambia el cifrado FTPES. Hay dos modos disponibles: <br><br>- El cifrado SSL se aplica a los datos y comandos FTP. Este es el modo Default para las funciones en FTPES.<br><br>- El cifrado SSL se aplica sólo a los comandos FTP. Este modo puede ser necesario para algunos servidores FTPES.<br><br><br> | [Parámetros FTP](../WDLang3/1000020947.md) |

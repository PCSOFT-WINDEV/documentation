


## Funciones HTTP
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>






Las siguientes funciones se utilizan para manipular las páginas de INTERNET, para enviar formularios HTTP, ...:



|   |   |
| --- | --- |
| [HTTPAddFile](../WDLang3/1000017191.md) | Añade un archivo al formulario HTTP. |
| [HTTPAddParameter](../WDLang3/1000017189.md) | Añade un parámetro a un formulario HTTP. |
| [HTTPCancelForm](../WDLang3/1000017123.md) | Cancela la declaración de un formulario HTTP y libera todos sus recursos. |
| [HTTPCertificate](../WDLang3/1000018898.md) | Modifica el cliente Certificate utilizado por Default para identificarse en un servidor. |
| [HTTPConfigure](../WDLang3/1000018985.md) | Configura las funciones que utilizan el protocolo HTTP. |
| [HTTPCookieGet](../WDLang3/1000019228.md) | Recupera las cookies leídas por una llamada: <br><br>- a [httpRequest](../WDLang3/3043007.md) para un dominio HTTP.<br><br>- a [HTTPSend](../WDLang3/1000021183.md) para un dominio HTTP.<br><br><br> |
| [HTTPCookieManage](../WDLang3/1000019218.md) | Habilita o deshabilita la administración de cookies en una ejecución de solicitud HTTP.: <br><br>- por [httpRequest](../WDLang3/3043007.md). <br><br>- por [HTTTPSend](../WDLang3/1000021183.md).<br><br><br> |
| [HTTPCookieRead](../WDLang3/1000019233.md) | Devuelve el valor de un Cookie recibido tras una petición HTTP. |
| [HTTPCookieReplace](../WDLang3/1000019231.md) | Reemplaza todas las cookies almacenadas para un dominio: <br><br>- por [httpRequest](../WDLang3/3043007.md).<br><br>- por [HTTTPSend](../WDLang3/1000021183.md).<br><br><br> |
| [HTTPCookieReset](../WDLang3/1000019232.md) | Elimina todas las cookies (globalmente o para un dominio) almacenadas.: <br><br>- por las llamadas a [httpRequest](../WDLang3/3043007.md).<br><br>- por las llamadas a [HTTPSend](../WDLang3/1000021183.md).<br><br><br> |
| [HTTPCookieWrite](../WDLang3/1000019234.md) | Añade o modifica una Cookie en una petición HTTP. |
| [HTTPCreateForm](../WDLang3/1000017119.md) | Crea un formulario HTTP. |
| [HTTPDestination](../WDLang3/1000017094.md) | Indica el archivo de respaldo para el resultado de la siguiente solicitud HTTP, ejecutado en la misma Thread. |
| [HTTPGetResult](../WDLang3/3043001.md) | Recupera el resultado o la cabecera de la última ejecución de la petición HTTP. |
| [HTTPListCertificate](../WDLang3/1000018890.md) | Enumera los certificados de cliente que se encuentran en el equipo para autentificar en un servidor. |
| [HTTPProgressBar](../WDLang3/3043009.md) | Muestra una barra de progreso cuando: <br><br>- recibir la respuesta a una petición HTTP.<br><br>- enviando un formulario HTTP ([HTTPSendForm](../WDLang3/1000017190.md)) o una consulta HTTP.<br><br><br> |
| [HTTPRequest](../WDLang3/3043007.md) | Inicia una petición HTTP en un servidor. |
| [HTTPResetForm](../WDLang3/1000017188.md) | Borra un formulario HTTP que se está editando actualmente. |
| [HTTPSend](../WDLang3/1000021183.md) | Envía una solicitud HTTP y espera la respuesta del servidor HTTP. |
| [HTTPSendForm](../WDLang3/1000017190.md) | Envía un formulario HTTP. |
| [HTTPSimulateNetwork](../WDLang3/1000021595.md) | Simula una red lenta al limitar la velocidad de descarga y carga. |
| [HTTPTimeOut](../WDLang3/3043008.md) | Establece el tiempo de espera para las funciones WLanguage que utilizan el protocolo HTTP (funciones [HTTPRequest](../WDLang3/3043007.md), [SOAPExecute](../WDLang3/3069013.md), [DotNetExecute](../WDLang3/3031016.md), [J2EEExecute](../WDLang3/3047007.md), etc.). |
| [InternetConnected](../WDLang3/3043005.md) | Define si una conexión INTERNET está habilitada en el ordenador actual. En el móvil, también puede recibir una notificación cuando cambie el estado de la conexión de INTERNET. |
| [OpenAPIToHTTP](../WDLang3/1410088947.md) | Construye la solicitud HTTP para llamar a la función de una API. Luego, podrá ser enviada utilizando la función [HTTPSend](../WDLang3/1000021183.md). |
| [Proxy](../WDLang3/3043002.md) | Especifica si las funciones de comunicación que utilizan HTTP o HTTPS deben pasar por un proxy para ejecutar sus peticiones. Se utiliza un proxy para compartir el acceso a INTERNET entre las diferentes computadoras de la red. |
| [RESTSend](../WDLang3/1000021476.md) | Envía una solicitud de REST y espera la respuesta del servidor. |



Los siguientes tipos avanzados se pueden utilizar con las funciones HTTP:



|   |   |
| --- | --- |
| [Galleta](../WDLang3/1000019222.md) | El tipo **Cookie** permite manipular las cookies transmitidas en solicitudes HTTP. |
| [httpProgress](../WDLang3/1000021222.md) | El tipo **httpProgress** permite gestionar el progreso del envío o recepción de una solicitud HTTP. |
| [HTTPRequest](../WDLang3/1000021158.md) | El tipo **httpRequest** se utiliza para recuperar las características avanzadas de una petición HTTP/HTTPS. |
| [httpResponse](../WDLang3/1000021165.md) | El tipo **httpResponse** se utiliza para get las características avanzadas de la respuesta de la solicitud HTTP/HTTPS. |
| [restRequest](../WDLang3/1000021481.md) | El tipo **restResponse** permite definir todas las características avanzadas de una respuesta REST. |
| [restResponse](../WDLang3/1000021477.md) | El tipo **restResponse** permite obtener todas las características avanzadas de una respuesta REST. |


## SocketClose( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cierra un socket o una conexión a un socket del servidor.

**Atención**: La conexión debe estar cerrada para que el socket del servidor detecte un mensaje en modo de lectura de bloqueo.

**Recordatorio**: Un socket es un recurso utilizado por las aplicaciones para comunicarse de un equipo a otro, independientemente del tipo de red.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF SocketCreate("Server", 8000) = False THEN
	Error("Creation error" + ErrorInfo(errMessage))
ELSE
	...
	SocketClose("Server")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`SocketClose(<Socket name>)`
---

**`<Socket name>: (Cadena de caracteres)`**

Nombre del socket que recibió el mensaje. 

**En las aplicaciones WINDEV, este nombre se definió en el servidor**:

- por [SocketCreate](../WDLang3/3070011.md), [SocketCreateSSL](../WDLang3/1000017041.md) o [SocketCreateInfrarrojo](../WDLang3/3070012.md) (para un servidor simplificado),

- por [SocketAccept](../WDLang3/3070004.md) (para un servidor estándar).




**En las aplicaciones WINDEV, este nombre se definía en el cliente** mediante [SocketConnect](../WDLang3/3070016.md), [SocketConnectSSL](../WDLang3/1000017042.md) o [SocketConnectInfrarrojo](../WDLang3/3070018.md).

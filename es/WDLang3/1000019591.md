
## CancelWebserviceContext( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina del contexto del webservice una Variable añadida por [DeclararServicio webContexto](../WDLang3/1000019460.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
GLOBAL
	gsMyGlobal is string

IF DeclareWebserviceContext(gsMyGlobal) = False THEN
	// Initialize with the default value
	gsMyGlobal = InitialValue()
END
...

// Deletes the value if an error occurs
IF bError THEN
	CancelWebserviceContext(gsMyGlobal)
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`CancelWebserviceContext([<Name of variable 1> [... [, <Name of variable N>]]])`
---

**`<Name of variable 1>: (Tipo de Variable (parámetro opcional))`**

Nombre de la primera Variable que debe ser borrado del contexto.



**`<Name of variable N>: (Tipo de Variable (parámetro opcional))`**

Nombre de la Nª Variable que debe ser borrado del contexto. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- No se produce ningún error si la Variable no existe.

- Si no se especifica la Variable, se cancela toda la sesión: todas las variables almacenadas se pierden y el archivo de sesión se destruye.

- Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".


## WebserviceReadMIMEType( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el tipo MIME de la petición recibida durante una llamada a un servicio web REST.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Accepts a JSON or XML content
SWITCH WebserviceReadMIMEType()
	CASE "application/json"
		// Processes the JSON case
		...
	CASE "application/xml"
		// Processes the XML case
		...
	OTHER CASE
		// Processes the other cases (triggers an error)
		// Indicates that the format of received data is in unknown : 
		// 415 Unsupported Media Type
		WebserviceWriteHTTPCode(415, sError)
		RETURN
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<MIME type> = WebserviceReadMIMEType()`
---

**`<MIME type>: (Cadena de caracteres)`**

Tipo MIME recibido en el encabezado HTTP Content-Type.
Cientos de tipos están definidos en el estándar de comunicación MIMI (disponible en INTERNET). Los valores más comunes son:

- "text/html": Página HTML (\*.htm,\*.html)

- "text/plain": archivo de texto (\*.txt)

- "solicitud/pdf": Document en formato PDF (\*.pdf)

- "Image/gif": Image en formato GIF (\*.gif)

- "Image/jpeg": Image en formato JPEG (\*.jpg, \*.jpeg)

- "video/mpeg": vídeo en formato MPEG (\*.mpg,\*.mpeg)

- "aplicación/desconocido": utilizado para mostrar un cuadro de diálogo proponiendo descargar el archivo en la el equipo de usuario de la Web.

- "solicitud / palabra clave": se utiliza para mostrar un archivo de Word.

- "application/vnd.ms-excel": para visualizar un fichero Excel.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Esta función sólo está disponible durante la llamada a una función que procesa una solicitud de un servicio web REST.. En los otros casos, la función devuelve una cadena vacía.

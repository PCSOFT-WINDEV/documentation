
## NetStartServer( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia un servidor FTP (File Transfer Protocol) o RPC (Remote Procedure Call) alimentado por WINDEV.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
ConnectRPC is int
// Connects to an RPC server on port 5010
ConnectRPC = NetStartServer(RPCServer, False, 5010)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = NetStartServer(<Server> [, <Encryption> [, <Port number> [, <Key> [, <Server IP address>]]]])`
---

**`<Result>: (Integro)`**



- Identificador del servidor,

- 0 si la conexión falla.




**`<Server>: (Constante)`**

Tipo de servidor: 


|   |   |
| --- | --- |
| *FTPServer* | Se utiliza un servidor FTP WINDEV. Las funciones para la gestión de archivos están permitidas ([NetSendArchivo](../WDLang3/3056005.md) y [NetGetFile](../WDLang3/3056025.md), por ejemplo). |
| *RPCServer* | Se utiliza un servidor RPC WINDEV. |



**`<Encryption>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> para cifrar los datos transmitidos,

- <u><u><u><u>False</u></u></u></u> (valor predeterminado) para no cifrar.




Si el servidor se inicia con la opción de cifrado en <u><u><u><u>True</u></u></u></u>, los equipos cliente deberán conectarse con la función [NetConnect](../WDLang3/3056004.md) utilizando la opción &lt;Cifrado&gt; = True.

Si se solicita un cifrado, también se cifrará la clave de conexión ().

**`<Port number>: (Entero opcional)`**

Número del puerto en el que se ejecutará el servidor. Por Default, el cliente intenta conectarse al servidor:

- en el puerto 2000 para WINDEV RPC.

- en el puerto 2001 para WINDEV FTP.




**`<Key>: (Cadena de caracteres opcional)`**

Se utiliza para asegurar el acceso al servidor. Cada cliente que intente conectarse al servidor debe utilizar la misma clave definida con la función **NetStartServer**. La conexión se rechaza si las llaves son diferentes.

Si el parámetro &lt;Cifrado&gt; se establece en <u><u><u><u>True</u></u></u></u>, se cifra la clave.

**`<Server IP address>: (Cadena de caracteres opcional)`**

Servidor IP Address. Este parámetro debe especificarse si el servidor tiene varias direcciones IP (por ejemplo, si el servidor utiliza varias tarjetas de red).. Esta IP Address puede tener el siguiente formato:

- IP Address en formato XXX.XXX.XXX.XXX (125.5.110.100 por ejemplo).

- IP Address que contiene el nombre del servidor (pop3.freesbee.eu por ejemplo). Se recomienda esta sintaxis.

- IP Address devuelta por [NetIPAddress](../WDLang3/3056023.md).

## Observaciones




### Parámetros del servidor
<a name="parametros_del_servidor_ELTPARAGRAPHE000096"></a>

Los parámetros del servidor son necesarios antes de establecer una conexión:

- Si se especifica el número de puerto, el cliente deberá utilizar el mismo número de puerto en [NetConnect](../WDLang3/3056004.md).

- Si el servidor se inicia con el parámetro &lt;Cifrado&gt; establecido en <u><u><u><u>True</u></u></u></u>, los clientes deben conectarse con el parámetro &lt;Cifrado&gt; establecido en <u><u><u><u>True</u></u></u></u> en la función [NetConnect](../WDLang3/3056004.md).

- Si el servidor se inicia con una &lt;Clave&gt; específica, los clientes deberán conectarse utilizando la misma cadena de caracteres &lt;Clave&gt; en la función [NetConnect](../WDLang3/3056004.md).



<a name="NOTE0_2"></a>




### Nombre de usuario y contraseña
<a name="nombre_usuario_contrasena_ELTPARAGRAPHE000125"></a>Los parámetros  y  permiten al servidor comprobar si el usuario está autorizado o no. La lista de autorizaciones y contraseñas para los usuarios está disponible en el fichero WDRPCSRV.INI que se encuentra en el directorio del proyecto de servidor.. Ejemplo de contenido de archivo:


```txt
[Passwords]
USERNAME1 = PASS1
USERNAME2 = PASS2
USERNAME3 = PASS3
```

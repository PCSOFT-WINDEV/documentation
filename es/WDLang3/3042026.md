


## grDraw (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Dibuja un gráfico de acuerdo con los parámetros especificados.

**Observaciones**: 

- La función **grDraw** tiene en cuenta los nuevos parámetros especificados desde la última llamada a esta misma función.

- Esta función no debe utilizarse para los controles de gráfico que se encuentran en la editor de reportes.


![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) **Atención**: Se requiere una configuración específica para usar esta función en Linux. Para obtener más información, consulte [Los gráficos](../WDChamp/3042060.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Draw a chart created through programming
grDraw("MyChart")
```
<a name="Example2"></a>

```wl
// Draw a chart in a Chart control
grDraw(CHART_MyChart)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`grDraw(<Chart name>)`
---

**`<Chart name>: (Nombre del control o cadena de caracteres)`**

Este nombre puede corresponder a:

- el nombre del gráfico a dibujar, definido por [grCreate](../WDLang3/3042005.md).

- el nombre de control Gráfico (en una ventana o en una página).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Crear y dibujar un gráfico mediante programación
<a name="crear_dibujar_grafico_mediante_programacion_ELTPARAGRAPHE000133"></a>

Para crear y dibujar un gráfico:

- crear el gráfico ([grCreate](../WDLang3/3042005.md)),

- añadir datos ([grAddData](../WDLang3/3042023.md), [grStockAddData](../WDLang3/3042009.md) y [grScatterAddDataXY](../WDLang3/3042032.md)),

- definir el destino de la carta con las siguientes funciones: 

	- [grDestinoBMP](../WDLang3/3042018.md), 

	- [grDestinoControl](../WDLang3/3042020.md), 

	- [grDestinoEMF](../WDLang3/3042017.md), 

	- [grDestinoWnd](../WDLang3/3042031.md),

	- [grDestinationImage](../WDLang3/1000020952.md),

	- [grDestinoWMF](../WDLang3/3042019.md). 




- dibujar el gráfico (**grDraw**).



<a name="NOTE0_2"></a>


### Gráficos e hilos
<a name="graficos_hilos_ELTPARAGRAPHE000182"></a>

Un solo gráfico puede ser manejado en un Thread.

Consecuencias:

- Dos hilos no pueden hacer manipular el mismo gráfico.

- Un gráfico creado en una Thread se destruye automáticamente al final de esta Thread. El gráfico existe en el hilo que contiene la llamada a [grCreate](../WDLang3/3042005.md).




**Observación**: No se recomienda llamar a la función en un hilo **grDraw**. Si se llama a esta función fuera del hilo principal, se pueden producir interbloqueos (deadlocks) y/o comportamientos inesperados.

## Instalación de un servidor SOAP con Netscape iPlanet 4.1
<a name="instalacion_servidor_soap_con_netscape_iplanet_41_ELTTEXTE000152"></a>


### Para instalar un servidor SOAP con Netscape iPlanet 4.1 (el servidor Netscape iPlanet 4.1 debe estar instalado en el equipo):
<a name="para_instalar_servidor_soap_con_netscape_iplanet_41_servidor_netscape_iplanet_41_debe_estar_instalado_equipo_ELTPARAGRAPHE000027"></a>

**1.** Para encontrar los archivos de configuración:

- Selecciona el directorio de instalación de iPlanet (C:\\Netscape\\Server4 de Default).

- Seleccione el subdirectorio correspondiente al servidor a configurar. El nombre del servidor corresponde a "https-&lt;user name>".

- Seleccione el subdirectorio "config".




**2**. Abra el archivo "mime.types" con WordPad.

ATENCIÓN: si este archivo es abierto por NotePad.EXE, el formato de archivo puede estar dañado durante la copia de seguridad.

**3.** Añade el siguiente Line al final del archivo y guárdalo (WordPad te dice que el diseño se perderá al guardarlo; responde "Sí"):

```txt
type=application/soap exts=soap
```


**Observación**: Use otra extensión si la extensión "jabón" ya es associated con un módulo en su servidor. En este caso, esta nueva extensión debe especificarse en las funciones utilizadas para especificar la URL del servidor SOAP (&lt;URL del servidor SOAP&gt; en [SOAPRun](../WDLang3/3069013.md), por ejemplo).

**4.** Abra el archivo obj.conf.

**5.** Encuentra el "&lt;Nombre del objeto=Default&gt;" Line. Insertar antes de esta Line:

- Las siguientes líneas para las pruebas desde su ordenador:



```txt
Init fn=load-modules shlib="<Setup Directory of WINDEV 15>/Data/wdxxxsns.dll" \
funcs="WinDevHandler,ModuleExit,ModuleInit"
Init fn="ModuleInit"
```




- Las siguientes líneas para instalar el servidor final SOAP:



```txt
Init fn=load-modules shlib="<DLL Location>/wdxxxsns.dll" \
funcs="WinDevHandler,ModuleExit,ModuleInit"
Init fn="ModuleInit"
```


Atención: Las rutas tienen el siguiente formato "d:/mi directorio/mi archivo.dll". Utilice comillas y barras oblicuas '/' (y no barras invertidas'\\').

**6.** Encuentra el primer Line que comienza con "Servicio". Insertar antes de esta Line:

```txt
Service method=(POST) type=application/soap fn=WinDevHandler
```


**7.** Guarde el archivo.

**8.** Para instalar un servidor SOAP estándar, reinicie su servidor. El servidor ya está instalado.

Observación: WDxxxsns.dll es compatible con NSAPI.

## Instalación de un servicio web
<a name="instalacion_servicio_web_ELTTEXTE000176"></a>


### Si desea instalar un servicio web
<a name="desea_instalar_servicio_web_ELTPARAGRAPHE000084"></a>

**9.** Abra el archivo "webpub.conf" y busque la palabra clave "NS-doc-root".

**10.** copy los siguientes elementos en el directorio especificado después de la palabra clave "NS-doc-root":

- El archivo XML que contiene el servicio web description.

- Las páginas HTML que presentan el servicio web.




**Tpara acceder a este archivo XML**, usa el siguiente comando Line:

"http://&lt;Name or IP Address of Soap Server&gt;/&lt;Name of XML File&gt;.XML"

**Tpara ejecutar la prueba de este servicio web**, usa el siguiente comando Line:

"http://&lt;NAme o IP Address de Soap Server&gt;/index.htm"

**Observación**: Este archivo XML también se puede copiar en otro directorio. Para más detalles, consulte la documentación sobre el servidor HTTP utilizado.

**11.** Reinicie su servidor. El Webservice está ahora instalado en el servidor SOAP.

Observación: WDxxxsns.dll es compatible con NSAPI.

### SaaS
<a name="saas_ELTPARAGRAPHE000009"></a>

WEBDEV le permite crear, alojar y gestionar fácilmente sitios SaaS, y convertir los sitios existentes a este modo de distribución.

En el modelo SaaS, la aplicación es accesible a distancia a través de un navegador INTERNET. El usuario no tiene que instalar o mantener nada localmente, ....

En la mayoría de los casos, estas aplicaciones se facturan por suscripción (según el número de usuarios y por mes, por ejemplo), pero pueden utilizarse otros métodos de facturación (según los recursos utilizados, según el espacio ocupado por los datos, etc.).

## Modo de funcionamiento
<a name="modo_funcionamiento_ELTTEXTE000204"></a>


### Modo de funcionamiento y uso de un sitio SaaS desde el punto de vista del usuario
<a name="modo_funcionamiento_uso_sitio_saas_desde_punto_vista_del_usuario_ELTPARAGRAPHE000022"></a>

Un sitio SaaS puede ser utilizado por varias empresas clientes. 

Cada empresa tiene su propia lista de usuarios y cada empresa accede a sus propios datos.. 

Para acceder al sitio SaaS, cada usuario debe estar autentificado (mediante un nombre de usuario y una contraseña).. Es posible:

- para comprobar que el usuario tiene una cuenta,

- comprobar si el usuario ha adquirido una licencia que le permite acceder a todo el sitio o a parte del mismo,

- comprobar si esta licencia sigue siendo válida,

- presentar los datos correspondientes a la empresa del usuario.




La Diagram siguiente presenta el uso de un sitio SaaS por parte de dos empresas distintas:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=SaaS_Principe.gif)


La empresa "Proveedor SaaS" propone un sitio en modo SaaS. Este sitio es utilizado por dos empresas clientes ("Empresa A" y "Empresa B"), cada una de las cuales es propietaria:

- Una base de datos específica,

- Diferentes estilos gráficos,

- Una base de datos diferente de cuentas de usuario.




Por lo tanto, cada empresa puede ser facturada de acuerdo a su uso del sitio Web.

**Modo de funcionamiento y uso de un sitio SaaS desde el punto de vista del administrador del sitio**

La Diagram que figura a continuación presenta los seis elementos principales del modelo SaaS:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=SaaS_Fonctionnement.gif)


Estos elementos son:

| Elemento | Rol |
| --- | --- |
| *sitio SaaS* | El **sitio SaaS** es un sitio WEBDEV desplegado en un servidor de aplicaciones WEBDEV. Se trata de associated con un servicio web SaaS utilizado para control el sitio. |
| *Servicio* | El **Servicios** son las diferentes características de un sitio. |
| *Cuenta de Cliente* | A **Cuenta de Cliente** representa a una empresa que utiliza el **sitio SaaS**.<br>Cada **Cuenta de Cliente** incluye su propia base de datos y su propia versión personalizada de la **sitio SaaS** (logotipo, gráficos, etc.). |
| *drogadicta* | El **Usuarios** son los miembros de la empresa representados por el **Cuenta de Cliente**.<br>Los usuarios son las personas físicas que utilizan el **sitio SaaS**. |
| *Determinación de precios* | Una **Tarificación** representa un conjunto de **Servicios** a los que puede acceder un **Usuario**.<br>A **Determinación de precios** puede incluir uno o más **Servicios**.<br>A **Servicio** puede proponerse en uno o varios **Determinación de precios**. |
| *Suscripción* | Una **Suscripción** representa la disponibilidad de un cierto número de licencias para una **Tarificación** (y por lo tanto de los **Servicios** asociados) para una **Cuenta de Usuario**.<br>Entonces, estas licencias deben ser distribuidas entre los **Usuarios**.<br>El **Suscripciones** se utilizan para supervisar el uso del **sitio SaaS** y la facturación. |

## Requisitos previos
<a name="requisitos_previos_ELTTEXTE000275"></a>
Para desplegar y gestionar un sitio SaaS, se necesitan los siguientes elementos:

- Un servidor de aplicaciones WEBDEV,

- El servicio web para la gestión de los sitios SaaS,

- El sitio de gestión de los sitios SaaS.


Los dos últimos elementos deben seleccionarse al instalar el servidor de aplicaciones WEBDEV.

El servicio web para la gestión de los sitios SaaS se utilizará a través de su alias web (WDSAAS28 0_WEB por ejemplo) que debe corresponder al directorio del mismo nombre en la instalación del servidor de aplicaciones (por Default: C:\\WEBDEV28\\WDSaaS\\WDSaaS28 0\\WDSAAS28 0_WEB).

## Despliegue de un sitio SaaS
<a name="despliegue_sitio_saas_ELTTEXTE000323"></a>
Cuando se habilita SaaS para un sitio, el implementación requiere los siguientes parámetros adicionales:

- **Seleccionar el Webservice asociado al sitio**: tiene la posibilidad de utilizar el servicio web que se encuentra en el mismo servidor de aplicaciones que el que aloja el sitio (caso más común) o un servicio web que se encuentra en otro servidor de aplicaciones WEBDEV (cuando varios sitios SaaS comparten un servicio web).

- **El nombre y la contraseña del administrador del SaaS Webservice**. Esta información es necesaria para realizar algunos pasos de implementación (por ejemplo, la modificación de automatic de los archivos de datos HFSQL).




**Observaciones**:

- En algunos casos, el desarrollador del sitio puede desconocer el nombre de usuario y la contraseña del administrador del servicio web SaaS. En este caso, puede utilizar la instalación por medios físicos o los paquetes de instalación y pedir a una persona con suficientes derechos que realice la instalación.

- Si el Address del servicio web SaaS no se especificó durante el implementación, el alias del servicio web SaaS (WDSAAS28 0_WEB) debe existir en el mismo servidor virtual que el sitio SaaS.

- Si el Address del SaaS Webservice se especificó durante el implementación, el administrador del sitio debe comprobar si el alias del Webservice está realmente disponible en el lugar especificado.

## Gestión de un sitio SaaS
<a name="gestion_sitio_saas_ELTTEXTE000347"></a>
Una vez desplegado, un sitio SaaS debe estar configurado antes de que pueda ser utilizado.. Se deben realizar las siguientes operaciones:

- [Registro del sitio en la interfaz de gestión](../SaaSAdminWeb/1600002.md),

- [Declarar los servicios propuestos por el sitio](../SaaSAdminWeb/1600003.md),

- [Declarar el precio](../SaaSAdminWeb/1600003.md),

- [Creación de las cuentas de las empresas clientes](../SaaSAdminWeb/1600004.md),

- [Creación de las cuentas de usuario para cada empresa cliente](../SaaSAdminWeb/1600005.md),

- [Gestión de las licencias concedidas a las empresas clientes y a sus usuarios](../SaaSAdminWeb/1600006.md).




Todas estas operaciones (así como el acceso a los registros relativos al uso del sitio a efectos de facturación) pueden realizarse a través del Administrador del SaaS.. Para obtener más información, consulte [Administrador SaaS](../SaaSAdminWeb/1600000.md).

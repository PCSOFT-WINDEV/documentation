
## SaaSConnect( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Autentica a un usuario de un sitio SaaS al lado del servicio web SaaS que administra el sitio.. 

**Observación**: Una vez establecida la conexión con **SaaSConnect**, podrá acceder a todos los sitios. [Servicio SaaSCheckService](../WDLang3/1000018961.md) se utiliza para comprobar los derechos de acceso.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Connection to the Webservice in charge of the site
IF SaaSConnect(EDT_USERNAME, EDT_PASSWORD) = False THEN	
	IF ErrorOccurred = True THEN
		sErrorMessage = CR + ErrorInfo(errFullDetails)
		RESULT sErrorMessage
	ELSE
		RESULT "Connection to SaaS site denied."
	END
END

// Pass a specific syntax (without parameter) to specify that we are checking the access to the site
// The client has at least one valid subscription to at least one service of the site
IF SaaSCheckService() = False THEN
	IF ErrorOccurred = True THEN
		sErrorMessage = CR + ErrorInfo(errFullDetails)
		RESULT sErrorMessage
	ELSE
		RESULT "Connection to the SaaS service of the site denied."
	END
END

// Use the connection to the HFSQL Client/Server or Classic database
// defined in the SaaS administrator on the server, for the site data.
IF HChangeConnection("*", SaaSClientConnection()) = False THEN
	sErrorMessage = HErrorInfo(hErrFullDetails)
	RESULT sErrorMessage
END

RESULT "Connection OK"
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = SaaSConnect(<User> , <Password> [, <Webservice>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si la autenticación fue exitosa,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<User>: (Cadena de caracteres)`**

Nombre de conexión del usuario SaaS.

**`<Password>: (Cadena de caracteres)`**

Contraseña del usuario de SaaS.

**`<Webservice>: (Cadena de caracteres opcional)`**

Servicio web a utilizar. Este parámetro corresponde a: 

- el nombre de host del servicio web SaaS (por ejemplo, "Computer", "Computer.MyNetwork.com", "http://Mydomain.com").

- la URL completa del servicio web SaaS (por ejemplo, "http://domain.com/WDSAAS28 0_WEB/WDSaaS28 0.awws").




Por Default, la conexión se establecerá con el servicio web SaaS instalado con el WEBDEV servidor de aplicaciones en el ordenador local.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si la función **SaaSConnect** se ejecuta varias veces, se realiza una desconexión implícita cada vez (función [SaaSDisconnect](../WDLang3/1000018960.md)). 

- Cuando se establece la conexión con la función **SaaSConnect**, el Webservice SaaS prueba periódicamente la validez de la conexión llamando automáticamente a la función [SaaSIsConectado](../WDLang3/1000018962.md) cada 4 minutos.

- Si la autenticación es correcta, las siguientes funciones SaaSxx utilizarán implícitamente la cuenta SaaS especificada en la última llamada a la función **SaaSConnect**. El contexto de ejecución será cerrado por [SaaSDisconnect](../WDLang3/1000018960.md).

- Para aumentar la seguridad, le recomendamos que utilice el protocolo HTTPS, tanto para acceder a la página que contiene la llamada a **SaaSConnect** como en la URL del Webservice. El servidor debe estar configurado para aceptar este protocolo.

- Si se accede al servicio web en HTTPS, el SSL Certificate utilizado no debe producir ninguna excepción de seguridad.

- [SaaSChangeCambiarContraseña](../WDLang3/1000019462.md) se utiliza para modificar la contraseña del usuario conectado.


## NotifPushEnable( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite la gestión de las notificaciones push en una aplicación WINDEV Mobile (Android o iOS).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Enable the push notifications
NotifPushEnable(RegisterPushNotif)
```

```wl
// Manage the registration
PROCEDURE RegisterPushNotif(Token is Buffer, NotifError is string)
IF Length(Token) > 0 THEN
	// Informs the application server about the new identifier
	SendIDOnServer(Token)
ELSE
	// Process the error
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`NotifPushEnable(<WLanguage procedure> [, <Options>])`
---

**`<WLanguage procedure>: (Nombre de Procedure)`**

Nombre del procedimiento WLanguage llamado cuando la aplicación recibe el identificador de registro del servicio Notification.

**`<Options>: (Combinación de constantes opcionales de tipo Integer)`**

Opciones de registro. Puede corresponder a una o más de las siguientes constantes: 


|   |   |
| --- | --- |
| *notifPushAlertType* | La aplicación móvil aceptará las notificaciones push en formato de alerta. <br><br> |
| *notifPushAllNotif*<br>(Valor predeterminado) | La aplicación móvil aceptará todo tipo de notificaciones push. |
| *notifPushBadgeType* | La aplicación móvil aceptará las notificaciones push en formato de tarjeta de identificación. <br><br> |
| *notifPushSoundType* | La aplicación móvil aceptará las notificaciones de sonido push. <br><br> |


Atención: Estas opciones se tienen en cuenta durante la primera aplicación instalación. Entonces, el usuario puede modificar las Notification s aceptadas en el Centro Notification del dispositivo.

## Observaciones


### WLanguage Procedure llamado
<a name="wlanguage_procedure_llamado_ELTPARAGRAPHE000073"></a>

El lenguaje WLanguage Procedure llamado por **NotifPushEnable** debe tener la siguiente sintaxis:


```txt
PROCEDURE RegisterPushNotif(<Identifier> is Buffer, <Error> is string)
```


Este procedimiento se llama cuando se termina la inscripción al servicio de Notification.

Los parámetros de esta Procedure son los siguientes: 

- &lt;Identificador&gt;: Buffer "Token" correspondiente al identificador de registro devuelto por el servicio Notification. Este buffer debe ser transmitido a servidor de aplicaciones para que las notificaciones push puedan ser enviadas a la aplicación.. 
	**Atención**: Este búfer debe ser transmitido a la servidor de aplicaciones:

	- cuando su valor cambia en relación con la ejecución anterior de la aplicación. Para ello, su valor debe almacenarse en un archivo de datos de la aplicación o en un buffer en disco creado por [fSaveBuffer](../WDLang1/1000019412.md) o [SaveParameter](../WDLang1/3025046.md).

	- si la aplicación es iniciada por el usuario (y no por la llegada de un empujón Notification). [ExeInfo](../WDLang1/3035001.md) determina si la aplicación se inicia cuando se recibe un Notification.


 Esta memoria intermedia puede estar vacía si se ha producido un error de registro durante la solicitud de registro.. En este caso, el parámetro &lt;Error&gt; se llenará con el mensaje de error.
	
	

- &lt;Error&gt;: Cadena de caracteres que describe un error de registro. Este parámetro sólo se rellena si se ha producido un error durante la solicitud de registro.


## La estructura de Contact
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000505"></a>
La estructura de Contact es una estructura preestablecida de WLanguage (no se requiere declaración). Esta estructura está acostumbrada:

- crear o modificar una Contact.

- recuperar el contenido de un Contact.








<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Las diferentes variables de la estructura de Contact
<a name="las_diferentes_variables_estructura_contact_ELTTEXTE000529"></a>
La estructura contiene los siguientes miembros:


|   |   |
| --- | --- |
| **mContact.Company** | Cadena de caracteres. <br>Nombre de la empresa donde trabaja el Contact. |
| **mContact.CreationDate** | salir con<br>Fecha de creación de la Contact<br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Este miembro es ignorado. |
| **mContact.Email** | Array de cadenas de caracteres. <br>Email Address de Contact. <br>Para acceder a las diferentes direcciones de correo electrónico del Contact, utilice el siguiente formato: <br>mContacto.Email[&lt;Número de dirección&gt;]<br>Donde &lt;Address Num&gt; corresponde al número del correo electrónico solicitado Address. El número de direcciones de correo electrónico de un Contact es devuelto por mContact.NbEmail. |
| **mContact.FirstName** | Cadena de caracteres. <br>El nombre de pila de la Contact.<br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) <br><br>- Al añadir una Contact ([ContactAdd](../WDLang3/1000006101.md)), el nombre de la Contact en el dispositivo se construirá a partir de los siguientes miembros: mContact.Name y mContact.FirstName.<br><br>- Cuando se lee un Contact, este miembro no se actualiza.<br><br><br> |
| **mContact.ID** | Cadena de caracteres.<br>Identificador único de Contact. |

| **mContact.Name** | Cadena de caracteres. <br>Nombre de Contact.<br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) <br><br>- Al añadir una Contact ([ContactAdd](../WDLang3/1000006101.md)), el nombre de la Contact en el dispositivo se construirá a partir de los siguientes miembros: mContact.Name y mContact.FirstName.<br><br>- Cuando se lee un Contact, contiene el nombre de visualización del Contact (por ejemplo: Paul Smith).<br><br><br> |

| **mContact.Out** | booleano. <br>Indica si hay un Contact actual. |

### Limitación
<a name="limitacion_ELTPARAGRAPHE000456"></a>

La estructura de contacto no maneja un "Otro" Address (diferente del negocio Address y el hogar Address).

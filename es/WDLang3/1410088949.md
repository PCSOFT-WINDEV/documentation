
## SocketConfigure (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Configura el uso de las funciones Socket. 

Observación: Esta función puede ser útil si la versión de Windows en el equipo cliente no soporta los últimos protocolos de seguridad de Schannel (por ejemplo: TLS 1.1 en Windows Vista, o TLS 1.3 antes de Windows 11).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
SocketConfigure(socketConfigureMode, 1)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`SocketConfigure(<Option to configure> , <Value>)`
---

**`<Option to configure>: (Constante de tipo Integer)`**

Opción a configurar: 


|   |   |
| --- | --- |
| *socketConfigureMode* | Modo de uso de los sockets. |



**`<Value>: (Entero)`**

Valor de la opción a configurar. 

Si &lt;Option to configure&gt; se establece como *socketConfigureMode*, los valores posibles son los siguientes: 

- 0 (Valor predeterminado): En este caso, los sockets TCP utilizan el módulo Windows Schannel.

- 1: En este caso, los sockets TCP utilizan OpenSSL (caso común en otras plataformas: Linux, iOS, Android, etc.).

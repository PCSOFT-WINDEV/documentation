
## sEscape( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ejecuta diferentes funciones que afectan directamente a los bits del puerto serie, puerto paralelo o puerto infrarrojo, independientemente del protocolo de comunicación.. Estas funciones se utilizan para gestionar:

- la señal DTR,

- la señal RTS,

- los caracteres XOFF y XON.




**Observaciones**: 

- El puerto debe abrirse previamente ([sOpen](../WDLang3/3062013.md)).

- ![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Los puertos de infrarrojos no son compatibles.







<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Clears the DTR signal
sEscape(PortNum, "CLRDTR")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = sEscape(<Port number> , <Function>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecutó la función,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Si el puerto no fue abierto previamente (por [sOpen](../WDLang3/3062013.md)), **sEscape** no tiene efecto.




**`<Port number>: (Integro)`**

Identificador del puerto utilizado: 

- Número de puerto: 

	- Número del puerto serie: 1, 2, 3, ... 32 para COM1, COM2, COM3, ... COM32.

	- Número del puerto paralelo: -1, -2, -3 o -4 para LPT1, LPT2, LPT3 o LPT4.

	- Número del puerto de infrarrojos: 1, 2, 3, ... 32 para IR1, IR2, IR3, ... IR32.




- Identificador del puerto devuelto por [sOpen](../WDLang3/3062013.md) (si esta función fue llamada con un nombre de puerto). 
	![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Sólo está disponible este tipo de parámetro.




**`<Function>: (Cadena de caracteres)`**

Función a ejecutar:


|   |   |
| --- | --- |
| CLRDTR | Borra la señal DTR. |
| CLRRETOS | Borra la señal RTS. |
| DESCANSO | Despeja la señal de "Romper Line".<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función no es compatible. |
| RESETDEV | Si es posible, reinicie el dispositivo.<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función no es compatible. |
| SETBREAK | Habilita la señal de "Romper Line".<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función no es compatible. |
| SETDTR | Envía la señal DTR. |
| SETRTS | Envía la señal RTS. |
| SETXOFF | La transmisión se comporta como si se hubiera recibido el carácter XOFF.<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función no es compatible. |
| SETXON | La transmisión se comporta como si se hubiera recibido el carácter XON.<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función no es compatible. |








## Observaciones
![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) **Caso especial para Linux**: 

- Sólo se pueden abrir e inicializar los puertos serie. 

- Por Default, los usuarios no tienen acceso a los puertos seriales. Es necesario concederles derechos específicos para la gestión de los puertos serie..

### Utilización de una biblioteca externa: RXTX
<a name="utilizacion_una_biblioteca_externa_rxtx_ELTPARAGRAPHE000328"></a>

En Java, el uso de funciones para el manejo de los puertos serie y paralelo ([sOpen](../WDLang3/3062013.md), [sWrite](../WDLang3/3062009.md), [sRead](../WDLang3/3062012.md), ...) requiere la presencia de una biblioteca externa: RXTX. 

Esta biblioteca incluye un archivo Jar y una o más bibliotecas nativas que dependen del sistema operativo en el que se ejecuta la aplicación..
 
Para utilizar estas funciones: 

- el archivo Jar (RXTXComm.jar) debe: 

	- en el mismo directorio que la aplicación Java generada por WINDEV,

	- en el classpath de ejecución de la aplicación,

	- ser incluido directamente en la aplicación generada (de la asistente para la generación de la Java). 




- se deben encontrar las bibliotecas nativas correspondientes al sistema operativo en el que se ejecuta la aplicación: 

	- en el mismo directorio que la aplicación Java generada por WINDEV,

	- en la ruta de las librerías de aplicaciones (librarypath).







La biblioteca RXTX y su documentación pueden descargarse de: [http://users.frii.com/jarvi/rxtx/index.html](http://users.frii.com/jarvi/rxtx/index.html) (enlace válido en el momento de la publicación de esta documentación). 

**Atención**: La gestión de los puertos de infrarrojos no está disponible en Java.

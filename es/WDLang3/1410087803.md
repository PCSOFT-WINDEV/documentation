
## WebserviceWriteCustomHTTPCode( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Especifica el código HTTP personalizado que se devolverá cuando finalice la ejecución de la función REST Webservice. Esta función permite especificar cualquier código entre 100 y 599, incluyendo los siguientes códigos de retorno: 

- Códigos 2xx para las respuestas **Successful**.

- Códigos 3xx para **Redirects**.

- Códigos 4xx para **Errores del cliente**.

- Códigos 5xx para **Errores del servidor**.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Indicates a token error 
WebserviceWriteCustomHTTPCode("498 Invalid Token")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`WebserviceWriteCustomHTTPCode(<HTTP code and description> [, <Additional information>])`
---

**`<HTTP code and description>: (Cadena de caracteres)`**

Código HTTP y su description, devuelto por la llamada actual al Webservice. El formato previsto es el siguiente: 

```txt
<HTTP code> <Description>
```
Por ejemplo: "498 Invalid Token".. Los códigos admitidos van del 100 al 599.

**`<Additional information>: (Cadena de caracteres opcional)`**

Información adicional requerida por algunos códigos de retorno. Los códigos de retorno afectados son los siguientes:

- Código 201 (Creado) : información opcional : Address de información añadida.

- Código 301 (Movido Permanentemente) : información obligatoria : Redirección Address.

- Código 302 (se mueve temporalmente) : información obligatoria : Redirección Address.

- Códigos 4xx y 5xx : información obligatoria : Texto de error devuelto al cliente. El texto se convertirá en UTF-8.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Esta función es useful cuando la llamada al webservice REST debe devolver un código no estándar no soportado por [Servicio webWriteHTTPCode](../WDLang3/1000022785.md).

- Esta función sólo está disponible durante la llamada a una función que procesa una solicitud de un servicio web REST.. En todos los demás casos, la función provoca un error.

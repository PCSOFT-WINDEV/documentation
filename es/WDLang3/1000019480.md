## Presentación
<a name="presentacion_ELTTEXTE000146"></a>
Android propone varios tipos de notificaciones para informar al usuario: 

- [notificaciones en la barra del sistema](#NOTE2_1). 

- [notificaciones de sonido](#NOTE3_1). 

- [notificaciones por vibración](#NOTE3_1). 

- [notificaciones encendiendo el LED](#NOTE3_1). 

- [mensajes "Toast"](#NOTE3_1). 

- [notificaciones push](../WDLang3/1000021015.md).




WINDEV Mobile le permite utilizar estos diferentes modos de notificaciones.

## Los avisos en la barra del sistema
<a name="los_avisos_barra_del_sistema_ELTTEXTE000170"></a>
Las notificaciones que se encuentran en la parte superior de la pantalla de inicio de Android son notificaciones por iconos. 
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_notif1.gif)


Pueden ser ampliados para mostrar sus detalles. 
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_notif2.gif)


Para crear y manipular estas notificaciones, WINDEV Mobile propone: 

- Funciones de WLanguage: 
	


|   |   |
| --- | --- |
| [NotifAdd](../WDLang3/1000019442.md) | Añade una Notification en la barra de sistema del dispositivo (Android) o en el centro Notification (iOS). |
| [NotifDelete](../WDLang3/1000019444.md) | Borra una Notification que aparece en la barra de sistema del dispositivo. |
| [NotifDeleteCategory](../WDLang3/1000023980.md) | Elimina una categoría de Notification de la solicitud. |
| [NotifListCategory](../WDLang3/1000023979.md) | Enumera las categorías de Notification de la aplicación. |
| [NotifModify](../WDLang3/1000019443.md) | Modifica una Notification que aparece actualmente en la barra de sistema del dispositivo. |

- tipos avanzados utilizados para gestionar las notificaciones: 
	


|   |   |
| --- | --- |
| [Notification](../WDLang3/1000019441.md) | El tipo **Notification** permite manipular notificaciones locales o notificaciones push. |
| [notificationAction](../WDLang3/1000021304.md) | El tipo **notificationAction** se utiliza para describir una acción adicional asociada a una Notification. |
| [notificationCatergory](../WDLang3/1000023981.md) | El tipo **notificationCategory** permite definir todas las características avanzadas de una categoría de notificaciones. |
| [notificationFormat](../WDLang3/1000021301.md) | El tipo **notificationFormat** se utiliza para describir el formato de visualización de un Notification cuando éste se expande. |




Tienes la posibilidad de añadir y eliminar iconos (y sus detalles) a/de la barra de Notification, realizar Notification s sonoras, vibraciones y encender el LED. 

**Nueva conducta de Notification: usando las categorías de Notification**:
A partir de Android 8.0 (Oreo, nivel de API 26), todos los Notification creados por una aplicación Android deben estar asociados a una categoría Notification también conocida como "canal". 
Las categorías le permiten agrupar las notificaciones por "tema" (u otro criterio de agrupación que el desarrollador decida aplicar).. Cada categoría Notification consta de un identificador, un nombre, un description, un nivel de prioridad y un set de set taciones visuales y sonoras (sonido, vibración, led color). Cuando un Notification es associated con una categoría, automáticamente "hereda" sus características.

El usuario final puede modificar localmente la configuración de Notification de una aplicación: activar/desactivar la visualización de Notification para la categoría, modificar la prioridad de Notification, asociar un sonido particular, etc.. Estas modificaciones se hacen desde la pantalla de ajustes de la aplicación Notification: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Categorie_notification.gif)

## Otros tipos de notificaciones
<a name="otros_tipos_notificaciones_ELTTEXTE000261"></a>
WINDEV Mobile propone otros tipos de notificaciones.

**1. Notification por vibración**

El Notification por vibración consiste en hacer vibrar el dispositivo para advertir al usuario. 

Varias funciones WLanguage se utilizan para manejar este tipo de Notification: 



|   |   |
| --- | --- |
| [VibrationStop](../WDLang3/1000019449.md) | Detiene las vibraciones del dispositivo (incluso si han sido provocadas por otra aplicación). |
| [VibrationTrigger](../WDLang3/1000019448.md) | Dispara las vibraciones del dispositivo. |





**2. Notification conectando el LED on**

La Notification al encender el LED consiste en encender el LED del dispositivo (o hacerlo parpadear). En algunos dispositivos, también tiene la posibilidad de cambiar la color del LED. 

Varias funciones WLanguage se utilizan para manejar este tipo de Notification: 



|   |   |
| --- | --- |
| [LEDSwitchOff](../WDLang3/1000019447.md) | Apaga el LED (diodo emisor de luz) del aparato. |
| [LEDSwitchOn](../WDLang3/1000019446.md) | Enciende el LED (diodo emisor de luz) del aparato. |





**3. Notification de Toast mensaje**

Una Toast es un mensaje furtivo que aparece en el centro o en la parte inferior de la pantalla para informar al usuario. El usuario no tiene que validar el mensaje: este se muestra durante un período de tiempo específico.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ToastAffiche.gif)
 La siguiente función de WLanguage gestiona los mensajes de Toast: 



|   |   |
| --- | --- |
| [ToastDisplay](../WDLang1/1000019358.md) | Muestra un mensaje "Toast". |

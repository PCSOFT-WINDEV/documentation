
## SNMPStartSession( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia una sesión SNMP.  
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
SessionID is int
// Start a session
SessionID = SNMPStartSession("TestDoc", "public")
IF SessionID = -1 THEN Error("Error while opening the session " + ErrorInfo())
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = SNMPStartSession(<Agent address> , <SNMP community> [, <Timeout> [, <Port number>]])`
---

**`<Result>: (Integro)`**



- Identificador de la sesión SNMP,

- -1 si se produce un error. [ErrorInfo](../WDLang1/3013008.md) devuelve más información sobre el error.




**`<Agent address>: (Cadena de caracteres)`**

Address del agente con el que se establecerá la comunicación vía SNMP. Esta Address se puede dar en el siguiente formato:

- IP Address en formato XXX.XXX.XXX.XXX (125.5.110.100 por ejemplo).

- URL que contiene el nombre del servidor (www.windev.com por ejemplo). Se recomienda esta sintaxis.




**`<SNMP community>: (Cadena de caracteres)`**

Nombre de la comunidad SNMP utilizada por la conexión. Esta comunidad indica los derechos associated con la sesión del SNMP. En la mayoría de los casos, todos los agentes SNMP soportan la comunidad "pública" en modo de sólo lectura. Pueden existir otras comunidades (privadas por ejemplo) para el acceso en lectura/escritura.

**`<Timeout>: (Entero o duración opcional)`**

Tiempo de espera (en milisegundos) de una solicitud SNMP. Este valor debe estar comprendido entre 1000 y 10000. Este parámetro puede corresponder a: 

- un número entero que corresponde al número de milisegundos,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, 1 s o 10 ms). 


Este parámetro es set a 5000 por Default (5 segundos).

**`<Port number>: (Entero opcional)`**

Número del puerto al que debemos conectarnos. Este valor es set a 161 por Default (puerto estándar SNMP).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>



## Observaciones
<a name="NOTE0_2"></a>
SNMP utiliza el protocolo UDP, que es un protocolo sin conexión.. Por lo tanto, **SNMPStartSession** no permite averiguar si el agente contactado puede unirse realmente. La función **SNMPStartSession** devuelve una sesión una vez que se ha enviado la solicitud SNMP, incluso si el agente no está disponible. Para comprobar si un agente está realmente conectado, tiene la posibilidad de utilizar: 

- [NetMachineName](../WDLang3/3056014.md),

- [Ping](../WDLang1/3014013.md),

- comunicación vía sockets,...


## NotesOpenDatabase( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Da acceso a los documentos gestionados por Lotus Notes.

Es necesario iniciar previamente una conexión con el servidor Domino ([NotasOpenConnection](../WDLang3/1000005102.md)).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Open the connection
SessionID is int
SessionID = NotesOpenConnection("Password", "MarsServer", ...
		"C:\Users\Julia\AppData\Local\Lotus\Notes\Data\user.id")
...
// Specify the Lotus Notes database to use
IF NotesOpenDatabase(SessionID, "email/julia.nsf") <> 0 THEN
	Info("Session started and database specified")
ELSE
	Error(ErrorInfo())
END
...
// Close the database
NotesCloseDatabase(SessionID)
// Close the connection
CloseSession(SessionID)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = NotesOpenDatabase(<Session identifier> , <Lotus Notes database>)`
---

**`<Result>: (Integro)`**



- Identificador de la sesión abierta.

- 0 si se produce un error (o <u><u><u><u>False</u></u></u></u>). Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Session identifier>: (Cadena de caracteres)`**

Identificador de la sesión a iniciar. Este identificador es devuelto por [NotasOpenConnection](../WDLang3/1000005102.md).

**`<Lotus Notes database>: (Cadena de caracteres)`**

Nombre de la base de datos de Lotus Notes que contiene los correos electrónicos del usuario. Este nombre corresponde a la ruta del archivo ".NSF".

Un relativo ruta debe ser usado si la base de datos de Lotus Notes se encuentra en un servidor de Domino.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
El acceso a Lotus Notes está disponible para Lotus Notes versión 6.5 o posterior.


## NotesModifyItem( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica el elemento especificado del actual Document en Lotus Notes. Si el elemento especificado de Lotus Notes no existe, este elemento se agrega a los elementos del actual Document.

Para tener en cuenta esta modificación, el Document debe ser guardado por el [NotasGuardar](../WDLang3/1000005110.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Start a Lotus Notes session
SessionID is int
SessionID = EmailStartNotesSession("Password", ...
			"MarsServer", "email\julia.nsf", ...
			"C:\Users\Julia\AppData\Local\Lotus\Notes\Data\user.id")
...
// Read the first document 
NotesDocumentFirst(SessionID)
// Modify the subject of the document
NotesModifyItem(SessionID, "Subject", "My new subject")
// Save the modifications
NotesSave(SessionID)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Modificar un simple artículo de Lotus Notes

`<Result> = NotesModifyItem(<Session identifier> , <Lotus Notes item> , <New value>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se modificó el elemento Lotus Notes,

- <u><u><u><u>False</u></u></u></u> en caso contrario. 


Si se produce un error, la variable [ErrorOccurred](../WDLang1/3087001.md) se pone en <u><u><u><u>True</u></u></u></u>. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.

**`<Session identifier>: (Integro)`**

Identificación de la sesión que se va a utilizar. Este identificador es devuelto por [EmailStartNotesSession](../WDLang3/3032121.md) o [NotasOpenDatabase](../WDLang3/1000005101.md).

**`<Lotus Notes item>: (Cadena de caracteres)`**

Nombre del artículo de Lotus Notes cuyo valor debe ser modificado. Se añade el elemento Lotus Notes si no existe.
Para conocer el nombre y el tipo de los elementos de las notas actuales Document, utilice [NotesListItem](../WDLang3/1000005112.md).

**`<New value>: (Tipo de elemento de Lotus Notes modificado)`**

Nuevo valor para el artículo especificado de Lotus Notes.  


<a name="SYNTAX2"></a>

### Modificación de un artículo de array Lotus Notes

`<Result> = NotesModifyItem(<Session identifier> , <Lotus Notes item> , <Array of New Values>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se modificó el elemento Lotus Notes,

- <u><u><u><u>False</u></u></u></u> en caso contrario. 


Si se produce un error, la variable [ErrorOccurred](../WDLang1/3087001.md) se pone en <u><u><u><u>True</u></u></u></u>. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.

**`<Session identifier>: (Integro)`**

Identificación de la sesión que se va a utilizar. Este identificador es devuelto por [EmailStartNotesSession](../WDLang3/3032121.md) o [NotasOpenDatabase](../WDLang3/1000005101.md).

**`<Lotus Notes item>: (Cadena de caracteres)`**

Nombre del artículo de Lotus Notes cuyo valor debe ser modificado. Se añade el elemento Lotus Notes si no existe.
Para conocer el nombre y el tipo de los elementos de las notas actuales Document, utilice [NotesListItem](../WDLang3/1000005112.md).

**`<Array of New Values>: (array unidimensional, del mismo tipo que el elemento modificado de Lotus Notes)`**

Array que contiene los nuevos valores para el artículo especificado de Lotus Notes.

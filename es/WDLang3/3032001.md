


## EmailSetTimeOut( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cambia el valor del "tiempo de espera" para conectarse a los servidores de mensajería SMTP, POP3 e IMAP. El valor del "tiempo de espera" es set a 10 minutos por Default. Este valor es devuelto por [EmailGetTimeOut](../WDLang3/3032011.md).

**Recordatorio**: El tiempo de espera es la cantidad máxima de tiempo permitido mientras se espera una respuesta del servidor de mensajería. Si el servidor no responde dentro de este tiempo de espera, la conexión falla.






<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`EmailSetTimeOut(<Timeout Duration>)`
---

**`<Timeout Duration>: (Entero o duración)`**

Valor de tiempo de espera en segundos. Este parámetro puede corresponder a: 

- un número entero correspondiente al número de segundos,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, 1 s). 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Funciones de conexión y tiempo de espera
<a name="funciones_conexion_tiempo_espera_ELTPARAGRAPHE000063"></a>

Las funciones de conexión ([EmailStartSession](../WDLang3/3032028.md), [EmailStartPOP3Session](../WDLang3/3032022.md) y [EmailStartSMTPSession](../WDLang3/3032025.md)) son funciones de bloqueo durante toda la conexión. Si el servidor no responde, estas funciones son funciones de bloqueo durante el tiempo de espera.

**Observación**: El valor de tiempo de espera (timeout) definido con la función **EmailSetTimeOut** representa el tiempo de espera de la respuesta del servidor de mensajería. El tiempo de ejecución real de [EmailStartSession](../WDLang3/3032028.md), [EmailStartPOP3Session](../WDLang3/3032022.md) y [EmailStartSMTPSession](../WDLang3/3032025.md) puede ser mayor que este valor (para un DNS lento, por ejemplo).

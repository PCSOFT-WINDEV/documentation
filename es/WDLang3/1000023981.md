
## notificationCatergory (Tipo de variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **notificationCategory** permite definir todas las características avanzadas de una categoría de notificaciones. Las Notification s de la aplicación pueden asociarse a una categoría de Notification. Las características de esta categoría de Notification pueden definirse y modificarse mediante diferentes propiedades de WLanguage. 

Este tipo de Variable es utilizado por el [Notification](../WDLang3/1000019441.md) Variable s. 

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Description of the notification category to use
categoryNotif is notificationCategory
categoryNotif.Identifier = "ID_NOTIF_COM"
categoryNotif.Name = "Commercial notifications"
categoryNotif.Priority = notifHighPriority

// Add a notification associated with the category
n is Notification
n.Title = "New message"
n.Message = "Order #54781-4551"
n.SecondaryMessage = "customer@company.com"
n.Category = categoryNotif
NotifAdd(n)
```

## Observaciones

### Propiedades específicas de las variables notificationCategory
<a name="propiedades_especificas_las_variables_notificationcategory_ELTPARAGRAPHE000034"></a>

Las siguientes propiedades pueden ser utilizadas para manipular una categoría de Notification:


| Nombre de la propiedad | Tipo utilizado | Efecto |
| --- | --- | --- |
| BadgeEnabled | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> (valor predeterminado) si se deben tener en cuenta las notificaciones de la categoría para mostrar la insignia de la aplicación.<br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario. <br><br><br> |
| ColorLED | Integro | El color del LED cuando se muestra el Notification. <br><br>El valor de esta propiedad se tiene en cuenta solo si el valor de la propiedad **DisplayLED** es <u><u><u><u>True</u></u></u></u>. <br><br>La indicación color del LED es verde por Default. <br><br>Este color puede corresponder a:<br><br>- un color RGB (devuelto por la función [RGB](../WDLang1/3029012.md)),<br><br>- una Component HSL de la color (devuelta por [HSL](../WDLang1/3029057.md)),<br><br>- [un color predefinido de WLanguage](../WDLang5/3010002.md).<br><br><br>**Atención**: Algunos aparatos no permiten modificar el LED color o no proponen todos los color s. Se usará un Color por defecto si el Color especificado no es soportado por el dispositivo. |
| denominar | Cadena de caracteres | Nombre de la categoría. **T Este nombre no debe estar vacío**. |
| description | Cadena de caracteres | Descripción de la categoría Notification (opcional). |
| DisplayLED | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si el LED del dispositivo debe estar encendido cuando se muestra la notificación, <br><br>- <u><u><u><u>False</u></u></u></u> (valor predeterminado) en caso contrario. <br><br><br>La color del LED y la frecuencia de encendido pueden cambiar de un aparato a otro. |
| Identificador | Cadena de caracteres | Identificador de la categoría. **Tsu Property no debe estar vacío**. Este identificador es único para la aplicación (sensible a mayúsculas y minúsculas y acentos). |
| Prioridad | Integro | Prioridad de las notificaciones associated con la categoría. La prioridad de un Notification influye en la forma en que el Notification se muestra al usuario. <br><br>En algunas situaciones, una notificación de baja prioridad puede ocultarse al usuario si se reciben otras notificaciones de mayor prioridad.<br><br>Esta Property puede tomar los siguientes valores: <br><br>- *notifMaxPriority*: máxima prioridad (ejemplo: llamada entrante). <br><br>- *notifHighPriority*: alta prioridad (ejemplo: recibiendo un Email urgente). <br><br>- *notifDefaultPriority* (valor predeterminado): prioridad estándar (ejemplo: recibir un SMS). <br><br>- *notifLowPriority*: baja prioridad (ejemplo: Notification para la actualización).<br><br>- *notifMinPriority*: prioridad mínima (ejemplo: información comercial).<br><br><br> |
| Sonido | Cadena de caracteres | Ruta del archivo de sonido a reproducir cuando se muestra la Notification. <br><br>Para reproducir el sonido predeterminado de las notificaciones, utilice la constante *notifDefaultSound*. Este sonido Default puede cambiar de un dispositivo a otro. <br><br>Tiene la posibilidad de reproducir un sonido emitido por un archivo de audio incluido en la aplicación (a través de la generación asistente): para ello, especifique el nombre del archivo y su extension.<br><br>Para no asociar ningún sonido con la Notification, usa una cuerda vacía ("", valor de Default). |
| Vibración | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si el dispositivo debe vibrar cuando se muestra la notificación,<br><br>- <u><u><u><u>False</u></u></u></u> (valor predeterminado) en caso contrario. <br><br><br>Observación: La duración y la potencia de las vibraciones pueden cambiar de un dispositivo a otro. |
| VisibleOnLockScreen | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si la notificación debe ser visible en la pantalla de bloqueo,<br><br>- <u><u><u><u>False</u></u></u></u> (valor predeterminado) en caso contrario. <br><br><br> |

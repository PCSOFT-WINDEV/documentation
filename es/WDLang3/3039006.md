


## StreamReject( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Rechaza una solicitud de conexión de secuencia. La corriente entrante no será associated con un control Conferencia. Esta función debe utilizarse en la Event "Solicitud de conexión de un nuevo flujo" associated con el flujo.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// -- Event "Connection request of new stream"
// Procedure used to detect a new stream
// nStreamID is an integer variable containing the identifier of the detected stream
PROCEDURE NewStream(nStreamID) 
// Accept the detected stream?
IF YesNo("New call received: " + StreamInfo(nStreamID), "Do you accept this call? ") = Yes THEN
	StreamAccept(nStreamID, MySelf)
ELSE
	StreamReject(nStreamID)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`StreamReject(<Connection identifier>)`
---

**`<Connection identifier>: (Integro)`**

Identificador de la conexión de la transmisión. Este identificador se define en la "Solicitud de conexión del nuevo flujo" Event associated con el flujo.

<a name="NOTE0_1"></a>

## Observaciones


### Procedure para la detección de corrientes
<a name="procedure_para_deteccion_corrientes_ELTPARAGRAPHE000152"></a>

Un flujo está asociado a la Event "Solicitud de conexión de un nuevo flujo". Este Event se ejecuta cuando se detecta una solicitud de conexión a un flujo entrante.

Este Event se ejecuta en uno de los siguientes casos: 

- si el valor de la propiedad [Detección](../Proprietes/2510031.md) utilizada en el flujo es <u><u><u><u>True</u></u></u></u>.

- si la opción "Detección de corrientes entrantes activas" ha sido marcada en la ventana de la corriente description.




**Observación**: Este Event se ejecuta en un thread específico de WLanguage. Por lo tanto, está sujeto a limitaciones específicas de los hilos de rosca. Para más detalles sobre los hilos, véase [Gestión de hilos con WINDEV](../WDLang1/3077026.md).


## inAppCheckSubscription( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Al iniciar la aplicación iOS, comprueba si se han renovado las suscripciones In-App.. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Initialization code of project
inAppCheckSubscription(CheckSubscription)
```

```wl
PROCEDURE CheckSubscription(bSuccess is boolean, Product is inAppProduct)
IF NOT bSuccess THEN
	ToastDisplay(ErrorInfo())
	RETURN
ELSE	
	// Call the inAppListPurchase function 
	// to check the start and end dates of subscription
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`inAppCheckSubscription(<WLanguage procedure>)`
---

**`<WLanguage procedure>: (Nombre de Procedure)`**

El nombre de WLanguage Procedure ("callback" Procedure) se ejecuta si una de las suscripciones fue renovada. Este procedimiento tiene el siguiente formato: 

```txt
PROCEDURE <Procedure name>(<Success> is boolean, <Product> is inAppProduct)
```
donde: 

- &lt;El éxito es booleano: 

	- <u><u><u><u>True</u></u></u></u> si se renovó la suscripción,

	- <u><u><u><u>False</u></u></u></u> en caso contrario.




- &lt;Producto&gt; es un [inAppProduct](../WDLang3/1000020868.md) Variable correspondiente al producto cuya renovación de suscripción se comprueba.

## Observaciones


### Principio y aplicación
<a name="principio_aplicacion_ELTPARAGRAPHE000051"></a>

En iOS, tienes la posibilidad de crear suscripciones autorrenovables para las compras en la aplicación.

Cuando este tipo de suscripción expira, la suscripción es automáticamente renovada por la AppStore.. Una nueva transacción de compra se crea automáticamente y se transmite durante el siguiente inicio de la aplicación.. Por lo tanto, esta transacción debe estar respaldada por la aplicación para tener en cuenta la renovación de la suscripción.

Se debe llamar a la función **inAppCheckSubscription** lo antes posible en el evento "Inicialización" del proyecto para poder procesar las renovaciones de la suscripción. El Procedure será llamado tantas veces como el número de suscripciones a renovar.


## SocketDeny( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ejecutar en el servidor, esta función rechaza una solicitud de conexión a un socket existente en el servidor.

**Observación**:

- Si el ordenador cliente utiliza una aplicación WINDEV para conectarse a este socket, esta aplicación tendrá que utilizar [SocketConnect](../WDLang3/3070016.md).

- La función **SocketDeny** no puede utilizarse con un socket UDP.

- La función **SocketDeny** puede utilizarse con un socket SSL.








**Recordatorio**: Un socket es un recurso utilizado por las aplicaciones para comunicarse de un equipo a otro, independientemente del tipo de red.


<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = SocketDeny(<Socket name>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si la conexión fue rechazada,

- <u><u><u><u>False</u></u></u></u> si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Socket name>: (Cadena de caracteres)`**

Nombre del socket del servidor que rechaza la conexión. Este nombre fue definido:

- por [SocketCreate](../WDLang3/3070011.md), [SocketCreateSSL](../WDLang3/1000017041.md), [SocketCreateInfrarrojo](../WDLang3/3070012.md) (para un servidor simplificado).

- por [SocketAccept](../WDLang3/3070004.md) (para un servidor estándar).




![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Atención: El nombre del socket distingue entre mayúsculas y minúsculas.

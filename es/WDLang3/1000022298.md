


## &lt;emailPOP3Session Variable&gt;.NbMensaje (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el número de mensajes entrantes en el servidor Email (protocolo POP3).
Este número de mensajes se mantiene constante durante toda la sesión de Email, aunque se reciban nuevos Email o se borren Email por [EmailDeleteMessage](../WDLang3/3032027.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MySession is EmailPOP3Session

nbMSG is int = MySession.NbMessage()
FOR I = 1 TO nbMSG
	EmailReadMessage(MySession, I)
	IF Email.Out <> True THEN
		CU.NAME = Email.Sender
		CU.MESSAGE_TXT = Email.Message
		...
	ELSE
		Info("This message was deleted")
	END
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <POP3 session>.NbMessage()`
---

**`<Result>: (Integro)`**



- Número de mensajes recibidos en el servidor al inicio de la sesión actual de Email,

- -1 si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<POP3 session>: (Variable de tipo emailPOP3Session)`**

Nombre de la [emailPOP3Session](../WDLang3/1000018759.md) Variable correspondiente a la sesión Email a utilizar.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Mensajes eliminados (protocolo POP3)
<a name="mensajes_eliminados_protocolo_pop3_ELTPARAGRAPHE000292"></a>

Si un mensaje fue borrado por [EmailDeleteMessage](../WDLang3/3032027.md):

- **&lt;emailPOP3Session Variable&gt;.NbMensaje** devuelve el número de mensajes en el servidor email antes de ser borrados. 

- los números de mensaje no se reasignan.




Por lo tanto, si se encuentran cuatro mensajes en el servidor al principio de la sesión, estos mensajes están numerados del 1 al 4.. Si se borra el mensaje nº 1:

- Al utilizar la función **&lt;emailPOP3Session Variable&gt;.NbMensaje** en la misma sesión, siempre devuelve 4.

- El mensaje 1 no puede ser leído por [EmailReadMessage](../WDLang3/3032016.md) porque el mensaje ya no existe (sólo existen los mensajes 2, 3 y 4).




Los números de mensaje se reasignan durante la siguiente conexión.
<a name="NOTE0_5"></a>


### Nuevos mensajes (protocolo POP3)
<a name="nuevos_mensajes_protocolo_pop3_ELTPARAGRAPHE000322"></a>

El protocolo POP3 no se puede utilizar para descubrir y gestionar los nuevos mensajes recibidos durante la conexión.. Para get los nuevos mensajes, debe desconectarse ([&lt;emailPOP3Session Variable&gt;.CloseSession](../WDLang3/1000022283.md)) y volver a conectarse ([&lt;emailPOP3Session Variable&gt;.StartSession](../WDLang3/1000022305.md)).

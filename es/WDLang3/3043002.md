
## Proxy( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Especifica si las funciones de comunicación que utilizan HTTP o HTTPS deben pasar por un proxy para ejecutar sus peticiones. Se utiliza un proxy para compartir el acceso a INTERNET entre las diferentes computadoras de la red.

Las funciones de comunicación pertinentes son las siguientes: 

- Funciones de [HTTP](../WDLang3/3043004.md),

- Funciones de [DotNet](../WDLang3/3031013.md),

- Funciones de [J2EE](../WDLang3/3047002.md),

- Funciones de [SOAP](../WDLang3/3069006.md),

- Funciones de Google ([Funciones de gestión de Google Calendar](../WDLang5/1000017564.md), [Funciones de gestión de los contactos de Google](../WDLang5/1000017565.md), [Funciones de gestión de Google Maps](../WDLang5/1000017566.md), [Funciones de gestión de Google Document](../WDLang5/1000017380.md), ... ), 

- [Servicios web SOAP importados](../WDLang3/3086001.md), 

- ...




Esta función sólo se tiene en cuenta la próxima vez que se llama a una función que utiliza HTTP.

**Observaciones**: 

- La función **Proxy** configura el acceso a Internet mediante un proxy para toda la aplicación.

- [FTPProxy](../WDLang3/1000019366.md) permite configurar el uso de un proxy para las funciones FTP. 

- [SocketProxy](../WDLang3/1410088761.md) determina si el TCP sockets y los WebSockets deben pasar por un proxy SOCKS5.









<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Indicates that the next connections will go via a proxy
Proxy("192.168.0.1", 8080)

// Performs an HTTP request
sHTMLCode is string = HTTPRequest("http://www.windev.com")
Info("The HTML code is: " + sHTMLCode)

// Cancels the use of proxy
Proxy("")
```

<a name="SYNTAX1"></a>

## Sintaxis

### Definir el proxy

`Proxy(<Proxy address> [, <Proxy port> [, <Username> [, <Password>]]])`
---

**`<Proxy address>: (Cadena de caracteres)`**

Address de proxy en el siguiente formato:

- IP Address en formato XXX.XXX.XXX.XXX (125.5.110.100 por ejemplo).

- URL que contiene el nombre del servidor (www.windev.com por ejemplo). Se recomienda esta sintaxis.

- IP Address devuelta por [NetIPAddress](../WDLang3/3056023.md).

- SOCKS5 proxy Address, en el siguiente formato: 
	```txt
	socks5://<Proxy IP address>
	```





**`<Proxy port>: (Entero opcional)`**

Número de puerto del proxy. 
Este parámetro es set a 80. si no se especifica

**`<Username>: (Cadena de caracteres opcional)`**

Autenticación para el proxy. 
Si este parámetro es una cadena vacía ("" por Default), no se realiza ninguna autenticación.

**`<Password>: (Cadena de caracteres opcional)`**

Contraseña para la autenticación a través del proxy. 
Este parámetro se ignora si &lt;Nombre de usuario&gt; corresponde a una cadena vacía ("").

### Cancelar el uso de la representación

`Proxy("")`

### Uso de un proxy
<a name="uso_proxy_ELTPARAGRAPHE000144"></a>

Con **Proxy**, el usuario final podrá configurar la aplicación WINDEV o WINDEV Mobile para utilizar o no un proxy para el protocolo HTTP. Le aconsejamos que implemente una ventana de configuración en la aplicación WINDEV para entrar en la Address del proxy si es necesario.

Algunos proxies requieren una configuración específica en el Explorador de INTERNET. En este caso, no es necesario llamar a la función **Proxy**: [HTTPConfigurar](../WDLang3/1000018985.md) (**httpConfigureMode** constante) le permitirá ir a través de INTERNET Explorer al enviar peticiones HTTP.


### ¿Cómo comprobar si el proxy existe?
<a name="¿como_comprobar_proxy_existe_ELTPARAGRAPHE000158"></a>

El siguiente código se utiliza para comprobar si el proxy existe a través de [SocketConnect](../WDLang3/3070016.md): 


```wl
IF SocketConnect(lsSocketName, llSocketPort, lsSocketAddr) = False THEN
	RESULT False
ELSE	
	SocketClose(lsSocketName)
END
```





### Protocolo Proxy y SOCKS5
<a name="protocolo_docparampagetitleshort_socks5_ELTPARAGRAPHE000171"></a>

Para utilizar el protocolo SOCKS5 con la función [Proxy](../WDLang3/3043002.md), agregue "socks5://" como prefijo al nombre del servidor proxy y utilice la función con la sintaxis estándar (pasando el puerto y los identificadores).

ATENCIÓN: 

- No se puede utilizar un proxy SOCKS5 si HTTP está configurado para utilizar WinInet (modo que utiliza Internet Explorer). Este modo puede configurarse con la constante *httpParameterMode* de la función [HTTPParameter](../WDLang3/1000018985.md).

- Al llamar a las funciones [HTTPRequest](../WDLang3/3043007.md) y [HTTPSendForm](../WDLang3/1000017190.md) con un proxy que utiliza el protocolo SOCKS5 se forzará el uso del comando cURL. 
	Por lo tanto, se recomienda utilizar una variable de tipo [httpRequest](../WDLang3/1000021158.md) con un proxy SOCKS5.




Ejemplo: 


```wl
// Do not use WinInet to access the SOCKS5 proxy
HTTPParameter(httpParameterMode, 1)
// The following functions will use the SOCKS5 proxy 
// at address 172.12.2.79:1080 with the identifiers USER:PWD 
Proxy("socks5://172.12.2.79", 1080, "USER", "PWD")
req is httpRequest
req.URL = "https://www.google.com"
req is httpResponse = req.Send()
IF ErrorOccurred THEN
	Error(ErrorInfo(errFullDetails))
ELSE
	Info("OK")
END
Proxy("")
```


## StreamConfigure( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Especifica los parámetros de red necesarios para utilizar los controles de la Conferencia en INTERNET. 

Observación: Esta función debe ser llamada antes de abrir la ventana que contiene los controles de Conferencia. De lo contrario, esta función se ignora. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Use the syntax 1
sConfiguration is string = [
{
'url': 'stun:stun.l.google.com:19302'
},
{
'url': 'turn:myturnserver.org'
'username': 'login2'
'credential': password2'
}
]
StreamConfigure(sConfiguration)
Open(WIN_InternetVideoConference)
```



```wl
// Use the syntax 2
StreamConfigure("stun.l.google.com:19302", "myturnserver.org", "user87", "Secret_87")
Open(WIN_InternetVideoConference)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Configuración a través de una cadena específica

`StreamConfigure(<Parameters>)`
---

**`<Parameters>: (Cadena de caracteres)`**

Parámetros utilizados para configurar las direcciones de los servidores de retransmisión a utilizar para que la comunicación sea posible a través de INTERNET. Este string tiene el siguiente formato: 

```txt
{
'url': 'stun:Sturnserver'
},
{
'url': 'turn: Turnservers'
'username': 'username'
'credential': password'
}
```


Atención: No se devolverá ningún error si se produce un error tipográfico en esta cadena. 


<a name="SYNTAX2"></a>

### Configuración a través de diferentes parámetros

`StreamConfigure(<STUN Server> , <TURN Server> , <Login> , <Password>)`
---

**`<STUN Server>: (Cadena de caracteres)`**

Address de un servidor STUN.

**`<TURN Server>: (Cadena de caracteres)`**

Address de un servidor TURN. 

**`<Login>: (Cadena de caracteres)`**

Iniciar sesión para usar en el servidor TURN. 

**`<Password>: (Cadena de caracteres)`**

Contraseña associated con el nombre de usuario del servidor de TURN.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si dos ordenadores se encuentran en la misma red local, pueden comunicarse directamente.: **StreamConfigure** no es necesario.

- Si 2 computadoras están conectadas a INTERNET a través de firewalls (o a través de cajas, por ejemplo), las comunicaciones directas entre estas computadoras están a menudo bloqueadas. En este caso es necesario utilizar la función **StreamConfigure**. 



<a name="NOTE0_2"></a>


### Usando los controles de la Conferencia a través de INTERNET
<a name="usando_los_controles_conferencia_traves_internet_ELTPARAGRAPHE000062"></a>

- Dependiendo de la tipología de la red y de sus parámetros de seguridad (por ejemplo, puertos bloqueados por un cortafuegos), puede que no sea necesario utilizar un servidor TURN y/o un servidor STUN. 

- Algunos servidores STUN están disponibles en acceso libre.

- Los servidores de TURN no están disponibles en acceso libre. Debe alquilar uno o crear uno en INTERNET.

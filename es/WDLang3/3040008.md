
## FTPRemoveDir( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina un directorio y su contenido (archivos y subdirectorios) de un servidor FTP (File Transfer Protocol).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003040008&name=ftpremovedir_funcion&product=WD">Ver ejemplos adicionales</a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Delete the "/Documents/File.DOC" file from the FTP server
RemoveDirResult = FTPRemoveDir(7, "/Documents/File")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = FTPRemoveDir(<Connection identifier> , <Directory to delete>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se eliminó el directorio,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Connection identifier>: (Integro)`**

Identificador de conexión, devuelto por la función [FTPConnect](../WDLang3/3040016.md).



**`<Directory to delete>: (Cadena de caracteres)`**

Nombre y ruta completa (o relativa) del directorio a borrar. Las secciones de la ruta están separadas por barras oblicuas ("/").

## Observaciones


### Timeout
<a name="timeout_ELTPARAGRAPHE000077"></a>

De forma predeterminada, todas las funciones FTP fallan si no se recibe ninguna respuesta del servidor FTP después de 20 segundos. Este tiempo de espera se puede modificar al conectarse al servidor FTP con la función [FTPConnect](../WDLang3/3040016.md).
<a name="NOTE0_2"></a>
<a name="NOTE0_3"></a>


### Permisos de usuarios
<a name="permisos_usuarios_ELTPARAGRAPHE000105"></a>

Sólo un usuario que tenga derechos de escritura en el servidor FTP puede manipular los archivos o directorios encontrados en un servidor FTP.. En la mayoría de los casos, no se conceden derechos de escritura a los usuarios "anónimos".
<a name="NOTE0_4"></a>




### Ruta relativa o absoluta
<a name="ruta_relativa_absoluta_ELTPARAGRAPHE000113"></a>

Las nociones de ruta relativa y ruta absoluta son de gran importancia en una aplicación FTP.

- Una ruta que comienza con un slash se considera como una ruta **absoluta**:  es la ruta con respecto a la raíz del servidor FTP (parámetro específico del servidor).
	ej: /publicidad/usuario/JULIA

- Si la ruta no comienza con un slash, se considera como una **ruta relativa**, es decir, una ruta con respecto al directorio en curso. El directorio en curso puede conocerse o modificarse con la función [FTPCurrentDir](../WDLang3/3040003.md).


Al conectarse a un sitio FTP, el directorio inicial ("home directory" del usuario) no necesariamente es la raíz del servidor FTP. Por lo tanto, se recomienda utilizar rutas relativas.

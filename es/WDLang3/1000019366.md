


## FTPProxy( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Especifica si las funciones de comunicación que utilizan el protocolo FTP deben pasar por un proxy para ejecutar sus peticiones.. 

Esta función se tiene en cuenta en la siguiente llamada a una función que utiliza el protocolo FTP.

**Atención**: 

- Esta función se aplica sólo al protocolo FTP (y no a FTPIS/FTPES/SFTP).. 

- En Linux, Android e iOS, esta función sólo puede utilizarse con el protocolo SOCKS5 (constante *ftpProxySOCKS5*). 





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Defines the FTP proxy
FTPProxy(ftpProxyUserTogether, "test", 2121, "test", "test")

// Opens the connection
ConnectionID is int
ConnectionID = FTPConnect("mytest.doc.com", "test", "test", 21, True, 20)
IF ConnectionID = -1 THEN
	Error(ErrorInfo(errFullDetails))
ELSE
	...
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`FTPProxy(<Proxy Type> [, <Proxy address> [, <Proxy port> [, <Username> [, <Password>]]]])`
---

**`<Proxy Type>: (Constante de tipo Integer)`**

Tipo de proxy FTP a utilizar: 


|   |   |
| --- | --- |
| *ftpProxyNone* | Permite a Cancel el uso del proxy FTP. |
| *ftpProxyOpen* | Proxy FTP que requiere el comando "OPEN".<br><br>Se enviarán las siguientes declaraciones:<br><br>- USUARIO con el login Proxy<br><br>- PASS con la contraseña de Proxy<br><br>- ABIERTO en el siguiente formato: @&lt;Address del servidor FTP&gt;:&lt;Port of FTP Server&gt;<br><br>- USUARIO con el login FTP<br><br>- PASS con la contraseña de FTP<br><br><br><br><br>![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Esta constante no está disponible. |
| *ftpProxySite* | Proxy FTP que requiere el comando "SITE".<br><br>Se enviarán las siguientes declaraciones:<br><br>- USUARIO con el login Proxy<br><br>- PASS con la contraseña de Proxy<br><br>- SITE en el siguiente formato: @&lt;Address del servidor FTP&gt;:&lt;Port of FTP Server&gt;<br><br>- USUARIO con el login FTP<br><br>- PASS con la contraseña de FTP<br><br><br><br><br>![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Esta constante no está disponible. |
| *ftpProxySOCKS5* | Proxy FTP que utiliza el protocolo SOCKS5. |
| *ftpProxyUserSeparate* | Proxy FTP que requiere el comando "USUARIO" y el comando "PASS" por separado.<br><br>Se enviarán las siguientes declaraciones:<br><br>- USUARIO con el login Proxy<br><br>- PASS con la contraseña de Proxy<br><br>- USUARIO en el siguiente formato: &lt;FTP Login&gt;@&lt;Address of FTP Server&gt;:&lt;Port of FTP Server&gt;.<br><br>- PASS con la contraseña de FTP<br><br><br><br><br>![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Esta constante no está disponible. |
| *ftpProxyUserTogether* | Proxy FTP que requiere el comando "USUARIO" con el usuario y la contraseña.<br><br>Se enviarán las siguientes declaraciones:<br><br>- USER en el siguiente formato: &lt;Usuario Proxy&gt;:&lt;Usuario FTP&gt;@&lt;Dirección de servidor FTP&gt;:&lt;Puerto de servidor FTP&gt;<br><br>- PASS en el siguiente formato: &lt;Proxy Password&gt;:&lt;FTP Password&gt;<br><br><br><br><br>![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Esta constante no está disponible. |



**`<Proxy address>: (Cadena de caracteres opcional)`**

Address del proxy FTP en el siguiente formato: 

- IP Address en formato XXX.XXX.XXX.XXX (125.5.110.100 por ejemplo). 

- URL que contiene el nombre del servidor (www.windev.com por ejemplo). Se recomienda esta sintaxis. 

- IP Address devuelta por [NetIPAddress](../WDLang3/3056023.md).




**`<Proxy port>: (Entero opcional)`**

Número de puerto del proxy FTP. Este parámetro es set a 21. si no se especifica

**`<Username>: (Cadena de caracteres opcional)`**

Autenticación a través del proxy FTP. Si este parámetro es una cadena vacía ("" por Default), no se realiza ninguna autenticación.

**`<Password>: (Cadena de caracteres opcional)`**

Contraseña para la autenticación a través del proxy. Este parámetro se ignora si &lt;Nombre de usuario&gt; corresponde a una cadena vacía ("").

## Observaciones


### Protocolo FTPProxy y SOCKS5
<a name="protocolo_docparampagetitleshort_socks5_ELTPARAGRAPHE000148"></a>

Para utilizar el protocolo SOCKS5 con la función [FTPProxy](../WDLang3/1000019366.md), utilice la constante *ftpProxySOCKS5* para especificar el tipo de proxy a utilizar.

ATENCIÓN: 

- No se puede utilizar un proxy SOCKS5 si el protocolo FTP está configurado para utilizar WinInet (modo que utiliza Internet Explorer). Este modo puede configurarse con la constante *httpParameterMode* de la función [FTPParameter](../WDLang3/1000020947.md). Si se llama a la función [FTPProxy](../WDLang3/1000019366.md), se generará un error fatal.

- Los proxies SOCKS5 solo pueden utilizarse con FTP y SFTP. Si se configura un proxy SOCKS5, al llamar a la función [FTPConnect](../WDLang3/3040016.md) en un servidor a través de FTPES o FTPIS se generará un error fatal.




Ejemplo: 


```wl
// Do not use WinInet to access the SOCKS5 proxy
FTPParameter(ftpParameterMode, 1)
// FTP functions will use the SOCKS5 proxy 
// at address 172.12.2.79:1080 with the identifiers USER:PWD 
FTPProxy(ftpProxySOCKS5, "172.12.2.79",1080, "USER", "PWD")
// Connect to ftp://test.rebex.net/
nFTPID is int = FTPConnect("ftp://test.rebex.net/", "demo", "password", 21)
IF nFTPID = -1 THEN
	Error(ErrorInfo(errFullDetails))
ELSE
	Info("OK")
END
FTPDisconnect(nFTPID)
// Remove proxy
FTPProxy(ftpProxyNone)
```


## OLEPlay( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite:

- recuperar la lista de operaciones disponibles (que no sean de edición) de un servidor,

- realizar una operación. En este caso, el servidor debe haber sido iniciado por [OLEActivar](../WDLang3/3057008.md), [OLECreateCopy](../WDLang3/3057005.md) o [OLECreateEmpty](../WDLang3/3057001.md) con &lt;Lock&gt; set a <u><u><u>False</u></u></u>. 



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Creates an object form a .WAV file
OLECreateCopy(OLE_OLE1, "C:\Windows\tada.wav", False)
// Plays the .WAV
OLEPlay(OLE_OLE1, "&Playing")
ListAdd(LIST_List1, OLEPlay(OLE_OLE1))
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Ejecutar una acción específica sobre un objeto

`<Result> = OLEPlay(<OLE control> , <Action to run>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la acción, 

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<OLE control>: (Nombre del control)`**

Nombre del control OLE associated con el objeto OLE. Si este parámetro corresponde a una cadena vacía (""), el control utilizado será el control al que pertenece el actual Event.

**`<Action to run>: (Cadena de caracteres)`**

Texto de la acción (verbo) a ejecutar. Indica al servidor el Process solicitado. Esta información es necesaria para permitir al cliente de OLE a Process el objeto relacionado sin conocer la estructura de los datos incorporados en el objeto OLE.  


<a name="SYNTAX2"></a>

### Recuperar la lista de acciones reconocidas por un servidor OLE

`<Result> = OLEPlay(<OLE control>)`
---

**`<Result>: (Cadena de caracteres)`**

Lista de acciones (verbos) reconocidos por el servidor OLE en el siguiente formato: 

```txt
<Verb1> + CR + <Verb2> + CR + ... + CR + <Verb N>
```
Estos verbos corresponden a opciones de menú y normalmente contienen un carácter '&' que indica el atajo. Este carácter no debe borrarse cuando se llame a **OLEPlay**.

**`<OLE control>: (Nombre del control)`**

Nombre del control OLE associated con el objeto OLE. Si este parámetro corresponde a una cadena vacía (""), el control utilizado será el control al que pertenece el actual Event.

<a name="NOTE0_1"></a>

## Observaciones




### Operaciones permitidas
<a name="operaciones_permitidas_ELTPARAGRAPHE000072"></a>

Algunos servidores OLE permiten a sus clientes realizar operaciones distintas a la edición de los objetos. Por ejemplo, la grabadora de sonido de Windows (SOUNDREC.EXE) también puede utilizarse para leer un objeto (.WAV o.MID.
<a name="NOTE0_2"></a>




### Varios
<a name="varios_ELTPARAGRAPHE000080"></a>

Sólo se admite la versión 1 de OLE.


## awSendInfo( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Envía una información de un iOS Phone a un Apple Watch (o a la inversa).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyBuffer is Buffer = "Text buffer"
IF NOT awSendInfo(MyBuffer) THEN
	Error(ErrorInfo())
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = awSendInfo(<Data>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó el envío

- <u><u><u><u>False</u></u></u></u> en caso contrario. [ErrorInfo](../WDLang1/3013008.md) devuelve más detalles sobre el error.




**`<Data>: (Buffer)`**

Datos por enviar.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- El receptor recibirá los datos enviados a través de la Procedure pasada a [awProcedureGetInfo](../WDLang3/1000022119.md).

- Si los datos no han cambiado desde la llamada anterior, no se llamará al receptor Procedure. 

- El envío de datos o información se considera volátil: cada llamada sucesiva sustituye a los datos enviados por la llamada anterior. 

- Los datos se transfieren en Background para reducir el uso de la batería. Por lo tanto, los datos pueden ser transmitidos bastante tiempo después de la llamada efectiva.

- Los datos enviados deben ser preferentemente pequeños (para evitar sobrecargar la comunicación entre el reloj y el Phone).

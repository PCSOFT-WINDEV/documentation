
## Aplicación Cliente: Ejecutar procedimientos en un servidor de servicios.net 
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000117"></a>
Varias funciones DotNet están disponibles en WINDEV. Estas funciones le permiten gestionar la ejecución de procedimientos en un servidor de servicios Web.Net XML desde sus aplicaciones WINDEV.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principio
<a name="principio_ELTTEXTE000141"></a>
Los parámetros necesarios para ejecutar un Procedure se especifican en la documentación del servidor de los servicios web XML de .Net.

Para ejecutar un Procedure en un servidor de servicios web XML:

1. Inicializar la estructura de parámetros para la ejecución de Procedure ([ver la estructura.net](../WDLang3/3031011.md)).
	Observación: Para realizar comprobaciones adicionales en el servidor de los servicios web XML.Net, añada una cabecera Procedure ([DotNetAddHeader](../WDLang3/3031009.md)).

2. Ejecute el programa Procedure ([DotNetRun](../WDLang3/3031016.md) o [DotNetRunXML](../WDLang3/3031012.md)). Las siguientes operaciones se realizan automáticamente:

	- conectando el ordenador actual y el servidor de los servicios web.net XML.

	- enviando los parámetros de Procedure al servidor de los servicios web XML de .Net.

	- comprobando la cabecera si es necesario.

	- ...dirigiendo el Procedure.

	- devolviendo el resultado o el error de Procedure al ordenador actual.




3. Revisa el resultado de Procedure.




El resultado de [DotNetRun](../WDLang3/3031016.md) o [DotNetRunXML](../WDLang3/3031012.md) se utiliza para saber si la conexión se ha establecido con éxito.

Si la conexión no se ha establecido, utilice [ErrorInfo](../WDLang1/3013008.md) para averiguar la causa del error. 

Si se ha establecido la conexión, compruebe el resultado de [DotNetGetResultado](../WDLang3/3031005.md):

- si el resultado no es una cadena vacía (""), la Procedure se ha ejecutado correctamente.

- si el resultado es una cadena vacía (""), la Procedure no se ha ejecutado y/o se ha devuelto un error. Para conocer el error de Procedure, utilice [DotNetError](../WDLang3/3031004.md).

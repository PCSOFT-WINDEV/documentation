
## OBEXListFile( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Enumera los archivos compartidos por un dispositivo que utiliza el protocolo OBEX.

**Observación**: Esta función no se puede utilizar si la transferencia se realiza por infrarrojos.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
sMACAddress is string
// Retrieve the MAC address of Bluetooth device
// The address is stored in a table column
sMACAddress = TABLE_Devices.COL_MACAddr[TABLE_Devices]
// OBEX connection
EDT_OBEXID = OBEXConnect(obexBluetooth, sMACAddress)
...
// Retrieve the simple list of shared files
FileList is string = OBEXListFile(EDT_OBEXID)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = OBEXListFile(<Identifier of OBEX connection> [, <Home directory> [, <Level of details>]])`
---

**`<Result>: (Cadena de caracteres)`**

Lista de archivos de directorio:

- ***en modo simple***: en el siguiente formato: 
	&lt;Nombre del archivo 1&gt; + TAB + &lt;Tipo 1&gt; + CR + ... &lt;Nombre del archivo N&gt; + TAB + &lt;Tipo N&gt; 
	donde &lt;Tipo&gt; corresponde a "F" si el elemento es un fichero y a "D" si el elemento es un directorio.

- ***en modo detallado***: en el siguiente formato: 
	&lt;Nombre del archivo 1&gt; + TAB + &lt;Tipo 1&gt; + TAB + &lt;Tamaño en bytes 1&gt; + TAB + &lt;Permisos 1&gt; + TAB + &lt;Fecha de modificación 1&gt; + TAB + &lt;Fecha de creación 1&gt; + TAB + &lt;Fecha de acceso 1&gt; + CR + ... 
	donde &lt;Tipo&gt; corresponde a "F" si el elemento es un fichero y a "D" si el elemento es un directorio, 
	&lt;Derechos&gt; es una cadena que puede contener las letras "R" (derechos de lectura), "W" (derechos de escritura) y "D" (derechos de borrado).. 
	Las fechas están en formato "AAAAMMDDHHmmSS".




**Si se utiliza la constante *obexListDirectory***, estas listas también contendrán los subdirectorios. Esta lista puede contener un directorio "..." si la raíz de la estructura de árbol del servidor OBEX no está listada.

**`<Identifier of OBEX connection>: (Integro)`**

Identificador de la conexión OBEX a utilizar. Este identificador es devuelto por [OBEXConnect](../WDLang3/1000017153.md).

**`<Home directory>: (Cadena de caracteres opcional)`**

Nombre del directorio a listar. El directorio de transferencia se utiliza si no se especifica este parámetro.

**`<Level of details>: (Constante o combinación de constantes de tipo Integer (opcional))`**

Nivel de detalle de los resultados:


|   |   |
| --- | --- |
| *obexListDetailed* | &lt;Resultado&gt; contiene la lista de archivos en modo detallado. |
| *obexListDirectory* | &lt;Resultado&gt; contiene la lista de archivos y directorios. Esta constante se puede combinar con *obexListSimple* y *obexListDetailed*. Cuando se utiliza solo, equivale a *obexListSimple* + *obexListDirectory*. |
| *obexListSimple*<br>(Valor predeterminado) | &lt;Resultado&gt; contiene la lista de archivos en modo simple. |

## Observaciones




### Modo de funcionamiento
<a name="modo_funcionamiento_ELTPARAGRAPHE000075"></a>

El nombre del directorio a listar no siempre se conoce. Una primera llamada a la función **OBEXListFile** (sin nombre de directorio) permite listar la raíz de la jerarquía OBEX. Si los dispositivos móviles soportan la creación de subdirectorios, se podrá llamar la función **OBEXListFile** especificando el nombre de un subdirectorio.  
<a name="NOTE0_2"></a>




### Limitación
<a name="limitacion_ELTPARAGRAPHE000087"></a>

Sólo las pilas de Microsoft bluetooth son soportadas por esta función. Para conocer la pila utilizada, llame a [BTStack](../WDLang3/1000017229.md). 

Para obtener más información, consulte [¿Qué pilas utilizar?](../WDLang3/1000017231.md)

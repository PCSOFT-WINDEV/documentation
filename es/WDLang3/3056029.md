
## Gestión de los accesos remotos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000119"></a>
Los accesos remotos se utilizan para crear una conexión de red temporal.

WINDEV le permite gestionar fácilmente los accesos remotos proponiendo varias funciones para abrir, cerrar y listar estos accesos remotos.

Algunos ejemplos "estándar" de uso de los accesos remotos:

- Uso de un módem

- Uso de una VPN (Red Privada Virtual)

- ...




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Funciones de gestión de los accesos remotos
<a name="funciones_gestion_los_accesos_remotos_ELTTEXTE000143"></a>
Las siguientes funciones de WLanguage se utilizan para gestionar los accesos remotos:



|   |   |
| --- | --- |
| [NetCloseRemoteAccess](../WDLang3/3056021.md) | Cierra la Line que fue abierta por [NetOpenRemoteAccess](../WDLang3/3056026.md). |
| [NetInfoRemoteAccess](../WDLang3/3056028.md) | Devuelve información sobre el estado de una conexión, así como las estadísticas sobre la misma. |
| [NetListRemoteAccess](../WDLang3/3056027.md) | Devuelve la lista de accesos remotos disponibles. |
| [NetOpenRemoteAccess](../WDLang3/3056026.md) | Establece una conexión remota. |

## Características específicas

### Conexión para el acceso remoto a la red
<a name="conexion_para_acceso_remoto_red_ELTPARAGRAPHE000034"></a>

Se crea una conexión para el acceso remoto a la red en el panel de Windows control. La siguiente información es necesaria para crear la conexión:

- el título de la conexión (utilizado por [NetOpenRemoteAcceso remoto](../WDLang3/3056026.md)),

- el número de Phone para marcar,

- el nombre de usuario, 

- el módem, el protocolo, etc.


Esta información es proporcionada por el proveedor.

Cuando la conexión se ejecuta a través de un doble clic realizado en el correspondiente Icon, tiene la posibilidad de especificar una contraseña para esta conexión. Esta contraseña puede ser almacenada para ser utilizada cada vez que se inicie esta conexión.
<a name="NOTE3_2"></a>


### Gestionar el cierre de automatic de un acceso remoto
<a name="gestionar_cierre_automatic_acceso_remoto_ELTPARAGRAPHE000053"></a>

[NetOpenRemoteAcceso remoto](../WDLang3/3056026.md) se utiliza para especificar si el acceso remoto debe cerrarse automáticamente al cerrar la aplicación.

Si se solicita el cierre de automatic, la conexión se cerrará automatic al final de la aplicación o durante la llamada a [NetCloseRemoteAccess](../WDLang3/3056021.md) (sin parámetro). En este caso, se cerrará la última conexión abierta por [NetOpenRemoteAcceso remoto](../WDLang3/3056026.md).

Si no se solicita el cierre de automatic, la conexión no se cerrará al final de la solicitud. Esta opción le permite crear una aplicación utilizada para abrir una conexión.
En este caso, la conexión no será cerrada por [NetCloseRemoteAccess](../WDLang3/3056021.md) (sin parámetro). Para cerrar esta conexión, el nombre de acceso se debe pasar en el parámetro.

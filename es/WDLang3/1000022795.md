
## WDTestRest: Ejecute la prueba de sus servicios web REST
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000171"></a>
WDTestRest se utiliza para probar las llamadas a un servicio web HTTP REST o para ejecutar un Web API accesible por HTTP.. 

Esta herramienta se inicia automáticamente al ejecutar la prueba de un servicio web REST desde el editor WINDEV o WEBDEV.. Los métodos de Webservice cuya prueba se ejecutó se proponen automáticamente. 

Observaciones: 

- Se soportan los siguientes métodos: HTTP get, POST, PUT, PATCH, DELETE, copy y HEAD.

- WDTestRest es compatible con los sistemas de autenticación HTTP y OAuth.

- Tiene la capacidad de gestionar tanto las cabeceras transmitidas como las recibidas durante la ejecución.

- El contenido de las solicitudes puede ser personalizado, en formato "Form-Data", "URLEncoded" o binario.

- WINDEV y WEBDEV deben estar instalados.

### URL parameters
<a name="url_parameters_ELTPARAGRAPHE000050"></a>

Usted tiene la posibilidad de especificar los parámetros de URL (botón"Parámetros"):

- Para añadir un parámetro, escriba su nombre y su posible valor.

- Para borrar un parámetro, haga clic en el botón "-" associated con el parámetro.

- Para modificar un parámetro, escriba los nuevos valores en los controles correspondientes.

- Para modificar el orden de los parámetros, mantenga pulsado el botón izquierdo del ratón sobre el botón "" del parámetro y arrastre el ratón hasta la ubicación deseada.. Suelte el botón izquierdo del ratón cuando haya terminado.



<a name="NOTE2_3"></a>


### Encabezados
<a name="encabezados_ELTPARAGRAPHE000062"></a>

El panel "Cabeceras" permite gestionar las cabeceras que se transmitirán durante la solicitud.

La gestión de las cabeceras se realiza como la gestión de los parámetros de la URL.

También tiene la posibilidad de gestionar el contenido de las cabeceras manualmente, a través del botón "Edición aproximada"..
Atención: en este caso, el formato debe respetar el de las cabeceras HTTP.
<a name="NOTE2_4"></a>


### Contenido
<a name="contenido_ELTPARAGRAPHE000075"></a>

Para las solicitudes que no sean de get, se puede transmitir un contenido con la solicitud.

Este contenido puede corresponder a datos de formulario, datos binarios, etc.

En el panel "Contenido", seleccione el tipo de contenido que desea transmitir a través del cuadro combinado "Tipo: la pantalla se adaptará según su elección.

Para los formularios ("form-data" o "urlencoded"), la edición de datos se realiza como la edición de encabezados y parámetros de URL.
<a name="NOTE2_5"></a>


### Autenticación
<a name="autenticacion_ELTPARAGRAPHE000088"></a>

WDTestRest soporta la autenticación básica HTTP, con el nombre de usuario y la contraseña que se transmitirán durante la solicitud.

También tiene la posibilidad de utilizar la autenticación OAuth 2.

WDTestRest le pide que especifique la información de la conexión OAuth y le permite a get el acceso Token ("Token") que se utilizará para ejecutar la solicitud.

Nota: Tienes la capacidad de almacenar los datos de autenticación de OAuth para las futuras llamadas marcando la casilla "Almacenar Token".
<a name="NOTE2_6"></a>


### Transmisión
<a name="transmision_ELTPARAGRAPHE000101"></a>

Una vez definidos los parámetros de petición, haga clic en el botón "Enviar" para transmitirlos a la URL especificada.

Al final de la ejecución de la solicitud, el resultado se muestra en la parte inferior de la ventana:

- Contenido HTTP (la respuesta),

- galletas,

- headers. 




Cada panel se utiliza para ver los datos recibidos. Los códigos XML y JSON pueden ser formateados de acuerdo a sus propias necesidades.

A get el código WL correspondiente a la llamada a la solicitud, pulse el botón "Generar el código WL".

Atención: el código generado está usando las nuevas características de la versión 22.
<a name="NOTE2_7"></a>


### Historial
<a name="historial_ELTPARAGRAPHE000120"></a>

Siempre que se envía una petición HTTP, WDTestRest almacena la configuración completa (URL, parámetros, encabezados, contenido, etc.) en un historial.

Este historial, accesible desde el botón WDTestRest, se utiliza para volver a ejecutar las solicitudes anteriores.

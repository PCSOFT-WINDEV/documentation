
## LDAPValue( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el valor correspondiente a la índice pasada como parámetro para el atributo especificado.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Find out the value of "UserPassword" attribute, subscript 2
AttValue = LDAPValue("MySession", "CN=JP", "UserPassword", 2)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = LDAPValue(<LDAP session> , <Object DN> , <Attribute name> [, <Subscript>])`
---

**`<Result>: (Cadena de caracteres)`**



- Valor del atributo para el índice especificado.

- Cadena vacía si el valor está vacío o si se ha producido un error. 




**`<LDAP session>: (Cadena de caracteres)`**

Nombre de la sesión LDAP utilizada.

**`<Object DN>: (Cadena de caracteres)`**

Nombre Distinguido (DN) del objeto cuyo valor de atributo se busca.

**`<Attribute name>: (Cadena de caracteres)`**

Nombre del atributo cuyo valor se busca. Los atributos del objeto son devueltos por [Atributo LDAPListAttribute](../WDLang3/3032104.md).

**`<Subscript>: (Entero opcional)`**

Índice del valor del atributo. Si no se especifica este parámetro, se devuelve el valor de índice 1 para el atributo especificado..  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Servidor LDAP y objeto LDAP local
<a name="servidor_ldap_objeto_ldap_local_ELTPARAGRAPHE000044"></a>

El servidor LDAP contiene objetos LDAP.

El objeto LDAP local es un objeto LDAP que se obtiene del servidor.
<a name="NOTE0_2"></a>




### Actualización del objeto LDAP local
<a name="actualizacion_del_objeto_ldap_local_ELTPARAGRAPHE000054"></a>

Cuando se ejecuta una función LDAP de lectura  ([Atributo LDAPListAttribute](../WDLang3/3032104.md), [LDAPNbValor](../WDLang3/3032106.md) o **LDAPValue**), el objeto del servidor se devuelve completamente al equipo local para limitar la comunicación con el servidor. Por lo tanto, si estas funciones de lectura se llaman varias veces con el mismo DN, la comunicación con el servidor se establece en la primera llamada.. Las otras llamadas leerán el objeto local.

**LDAPValue** actualiza el objeto LDAP local solo si el DN del objeto pasado como parámetro es diferente del último DN pasado como parámetro a una de las siguientes funciones: [Atributo LDAPListAttribute](../WDLang3/3032104.md), [LDAPNbValor](../WDLang3/3032106.md), **LDAPValue**.

**LDAPValue** no actualiza el objeto LDAP local si el DN pasado como parámetro es el mismo que el último DN pasado como parámetro a una de las siguientes funciones: [Atributo LDAPListAttribute](../WDLang3/3032104.md), [LDAPNbValor](../WDLang3/3032106.md), **LDAPValue**. Sin embargo, si se han utilizado [LDAPValidateAdd](../WDLang3/3032111.md) y [LDAPValidateModify](../WDLang3/3032117.md) con este mismo DN, el objeto local se actualiza automáticamente.
<a name="NOTE0_3"></a>




### Caracteres acentuados
<a name="caracteres_acentuados_ELTPARAGRAPHE000100"></a>Si el servidor utiliza caracteres en UTF8, el valor del atributo puede tener que ser convertido para que get el resultado solicitado. Por ejemplo:


```wl
AttValue = UTF8ToString(LDAPValue("MySession", "CN=JP", "UserPassword", 2), charsetAnsi)
```

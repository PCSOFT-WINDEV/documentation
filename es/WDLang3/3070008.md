
## Gestión de sockets con WINDEV, WEBDEV y WINDEV Mobile
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000183"></a>
WINDEV, WEBDEV y WINDEV Mobile incluyen varias funciones que le permiten realizar una gestión avanzada de sockets.

Un socket es un recurso utilizado por las aplicaciones para comunicarse de un equipo a otro, independientemente del tipo de red.

Este modo de comunicación puede utilizarse, por ejemplo, para establecer una comunicación entre equipos conectados a través de Internet.



WINDEV, WEBDEV y WINDEV Mobile le permiten gestionar:

- El estándar sockets.

- La UDP sockets.

- La sockets por infrarrojos.

- SSL sockets.




Ejemplos de uso de sockets:

- gestionar una mensajería en tiempo real.

- acceso a un servidor de noticias (foro).

## Diferentes modos
<a name="diferentes_modos_ELTTEXTE000207"></a>
Una aplicación WINDEV o WEBDEV puede gestionar la sockets según diferentes modos:

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [Cliente de aplicación WINDEV/WEBDEV](../WDLang3/3070017.md): la aplicación se conecta a cualquier servidor e intercambia datos a través de un socket.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [WINDEV aplicación "Servidor simplificado](../WDLang3/3070001.md): la aplicación WINDEV es un servidor, que intercambia información a través de un socket con **un único ordenador cliente** (Enchufe WINDEV recomendado en el ordenador cliente pero no obligatorio).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [WINDEV Aplicación "Servidor estándar](../WDLang3/3070003.md): la aplicación WINDEV es un servidor que intercambia información a través de sockets con varios ordenadores clientes de ****.

## Modificar el comportamiento estándar de sockets
<a name="modificar_comportamiento_estandar_sockets_ELTTEXTE000231"></a>


### Optimización de los marcos de red
<a name="optimizacion_los_marcos_red_ELTPARAGRAPHE000078"></a>

Por Default, en Windows, los marcos de la red están optimizados según el algoritmo de Nagle.

Para comunicarse con un sistema que no soporta esta optimización (por ejemplo, algunos sistemas operativos industriales embebidos), esta optimización debe estar desactivada.

Para ello, introduzca el siguiente código:


```wl
Socket.Option = SocketNagleOff
```

<a name="NOTE3_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Optimizando las conexiones para una IP Address
<a name="optimizando_las_conexiones_para_una_address_ELTPARAGRAPHE000093"></a>

Cuando se pasa una IP Address a [SocketConnect](../WDLang3/3070016.md), tiene la posibilidad de optimizar la conexión utilizando el siguiente código:  


```wl
Socket.Option = SocketOptimizeIPConnection
```


Esta opción no está habilitada por Default porque la conexión puede fallar en algunas configuraciones. Sin embargo, puede ser useful en algunas aplicaciones para las que las actuaciones son críticas.
<a name="NOTE3_3"></a>


### Reutilizar un Address que ya se utiliza
<a name="reutilizar_address_que_utiliza_ELTPARAGRAPHE000109"></a>

Por defecto, [SocketCreate](../WDLang3/3070011.md) permite reutilizar un Address (ya utilizado por la propia aplicación o por otra aplicación). Para prohibir este modo de funcionamiento, escriba la siguiente Line de código:


```wl
Socket.Option = SocketNoReuseAddress
```


**Observación**: Por extensión, esta opción permite saber si un Address o un puerto ya se utiliza. En este caso, [SocketCreate](../WDLang3/3070011.md) devuelve un error.
<a name="NOTE3_4"></a>


### Usando la configuración de Default
<a name="usando_configuracion_default_ELTPARAGRAPHE000128"></a>

El siguiente código le permite restaurar la configuración de Default:


```wl
Socket.Option = SocketOptionDefault
```

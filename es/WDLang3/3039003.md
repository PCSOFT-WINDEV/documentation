


## StreamAccept( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Asocia una solicitud de conexión de flujo con una control Conferencia. Esta función debe utilizarse en la Event "Solicitud de conexión de un nuevo flujo" associated con el flujo.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// -- Event "Connection request of new stream"
// Procedure used to detect a new stream
// nStreamID is an integer variable containing the identifier of the detected stream
PROCEDURE NewStream(nStreamID)
// Accept the detected stream? CBOX_Auto_Response is a Check Box control
IF CBOX_Auto_Response[1] = True THEN
	StreamAccept(nStreamID, MySelf)
ELSE
	StreamReject(nStreamID)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = StreamAccept(<Connection identifier> , <Conference control>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True STRIMEULB</u></u></u></u> si la secuencia estaba asociada con control Conferencia, 

- <u><u><u><u>False</u></u></u></u> en caso contrario (identificador incorrecto, control inexistente, etc.). Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Connection identifier>: (Integro)`**

Identificador de la conexión de la transmisión. Esta conexión será associated con un control Conferencia y por lo tanto con el flujo al que el control es associated. Este identificador se define en la Event "Solicitud de conexión de un nuevo flujo" associated con el flujo.

**`<Conference control>: (Nombre del control)`**

Nombre de un control Conferencia que acepta la conexión.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### StreamAccept: Operaciones realizadas
<a name="streamaccept_operaciones_realizadas_ELTPARAGRAPHE000044"></a>

El flujo associated con el control es relacionado a esta conexión.

El stream establece enlaces a todos los controles de la Conferencia relacionado a la misma.

Se devuelve el identificador de la conexión (y por lo tanto el identificador de la secuencia):

- con el [Valor](../Proprietes/2510130.md) property.

- usando el nombre del stream directamente.

- utilizando directamente el nombre de la control Conferencia.




Si el control Conferencia ya está conectado a un stream, el stream se desconecta automáticamente.
<a name="NOTE0_2"></a>


### Procedure para la detección de corrientes
<a name="procedure_para_deteccion_corrientes_ELTPARAGRAPHE000192"></a>

Un flujo está asociado a la Event "Solicitud de conexión de un nuevo flujo". Este Event se ejecuta cuando se detecta una solicitud de conexión a un flujo entrante.

Este Event se ejecuta en uno de los siguientes casos: 

- si el valor de la propiedad [Detección](../Proprietes/2510031.md) utilizada en el flujo es <u><u><u><u>True</u></u></u></u>.

- si la opción "Detección de corrientes entrantes activas" ha sido marcada en la ventana de la corriente description.




**Observación**: Este Event se ejecuta en un thread específico de WLanguage. Por lo tanto, está sujeto a limitaciones específicas de los hilos de rosca. Para más detalles sobre los hilos, véase [Gestión de hilos con WINDEV](../WDLang1/3077026.md).

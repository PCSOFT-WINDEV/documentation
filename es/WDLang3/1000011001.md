
## AppointmentAdd( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade una Appointment: 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en un Lotus Notes o Outlook Calendar. La estructura [Appointment](../WDLang3/1000011010.md) se utiliza para definir los datos de esta Contact.
	**Recordatorio**: Esta función no puede utilizarse para añadir una cita a una Calendar de Outlook Express.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000020711&name=appointmentadd_funcion&product=WD">Ver ejemplos adicionales</a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Start a Lotus Notes session
SessionID is int
SessionID = EmailStartNotesSession("Password", "MarsServer", ...
	"email/julia.nsf", "C:\Lotus Notes\julia.id")
// Initialize the variables of the Appointment structure
AppointmentReset()
// Fill the variables for the new appointment
mAppointment.Subject = "Sales meeting"
mAppointment.StartDate ="20070125100000"
mAppointment.Location = "Room 3B"
mAppointment.NbParticipant = 3
mAppointment.Participant[1] = "Robert"
mAppointment.Participant[2] = "Lucas"
mAppointment.Participant[3] = "Stephen"
// Add the appointment
IF AppointmentAdd(SessionID) = True THEN
	Info("Appointment added")
ELSE
	Error("Error during the addition", ErrorInfo())
END
```

## Sintaxis

### Añadir un Appointment a un Calendar de Lotus Notes o Outlook

`<Result> = AppointmentAdd(<Session identifier>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la adición,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Session identifier>: (Integro)`**

Identificación de la sesión que se va a utilizar. Este identificador es devuelto por una de las siguientes funciones: [EmailStartNotesSession](../WDLang3/3032121.md), [EmailStartOutlookSession](../WDLang3/3032128.md), [NotasOpenDatabase](../WDLang3/1000005101.md) o [OutlookStartSession](../WDLang3/3032123.md).

## Observaciones
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 

### Calendar de un mensaje de Lotus Notes o de Outlook
<a name="calendar_mensaje_lotus_notes_outlook_ELTPARAGRAPHE000128"></a>

- Cuando se crea una Appointment con recordatorio en Outlook, el recordatorio se muestra inmediatamente independientemente de la fecha de la Appointment.

- Las Appointment s añadidas son "reunión" de las Appointment s (en lugar de "Appointment" Appointment s). Para cambiar el tipo de cita, modifique el control "AppointmentType" de la nueva cita (con [NotasModificarItem](../WDLang3/1000005113.md), por ejemplo).
	Los diferentes valores posibles para las clases son los siguientes:

	- "0": Appointment

	- "1": aniversario

	- "2": Event

	- "3": reunión

	- "4": recordatorio


 Ejemplo:
	
	```wl
	NotesModifyItem(ID, "AppointmentType", "0")
	```


- En Outlook, se puede especificar el modo de gestión de los contactos/participantes al iniciar la sesión ([EmailStartOutlookSession](../WDLang3/3032128.md) o [OutlookStartSession](../WDLang3/3032123.md)).


## Notificaciones Push: formato de las notificaciones (Android e iOS)
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000366"></a>
Una aplicación escrita en WINDEV Mobile Android o iOS puede recibir notificaciones Push. Estas notificaciones push se pueden enviar desde:

- Una aplicación WINDEV o un sitio WEBDEV de [NotifPushSend](../WDLang3/1000020819.md).

- Una aplicación o un sitio escrito en un lenguaje externo (Java, PHP, C#, etc.).




Cuando se envía un Push Notification desde un idioma externo, se debe identificar y respetar: 

- [el formato esperado por la aplicación WINDEV Mobile Android](#NOTE2_1).

- [el formato esperado por la aplicación WINDEV Mobile iOS](#NOTE3_1).

## Principio de envío de un PUSH Notification a un dispositivo Android desde un lenguaje externo
<a name="principio_envio_push_notification_dispositivo_android_desde_lenguaje_externo_ELTTEXTE000390"></a>


### Empujar a Notification usando la plataforma GCM
<a name="empujar_notification_usando_plataforma_gcm_ELTPARAGRAPHE000032"></a>

Para enviar un Push Notification a un dispositivo Android, la aplicación servidor debe realizar una petición HTTP POST al siguiente Address [https://gcm-http.googleapis.com/gcm/send](https://gcm-http.googleapis.com/gcm/send).

Esta solicitud debe incluir:

- un encabezado.

- un cuerpo que describe la Notification para enviar. 




ATENCIÓN: Desde la actualización 22 de WINDEV Mobile, el envío de notificaciones push para Android utiliza la plataforma Firebase (en lugar de Google Cloud Messaging, GCM). Los proyectos anteriores pueden seguir utilizando la solución GCM, pero los nuevos proyectos necesariamente utilizan Firebase.. WINDEV Mobile soporta las notificaciones push a través de Firebase desde la versión 22 Update 1.


### Empujar a Notification usando la plataforma de la Base de Fuego
<a name="empujar_notification_usando_plataforma_base_fuego_ELTPARAGRAPHE000049"></a>

Para enviar un Push Notification a un dispositivo Android, la aplicación servidor debe realizar una petición HTTP POST al siguiente Address [https://fcm.googleapis.com/fcm/send](https://fcm.googleapis.com/fcm/send).

Esta solicitud debe incluir:

- un encabezado.

- un cuerpo que describe la Notification para enviar. 




ATENCIÓN: A partir de WINDEV Mobile 22 actualización 1, los nuevos proyectos están necesariamente utilizando Firebase para enviar notificaciones push. 
<a name="NOTE2_3"></a>


### Cabecera description
<a name="cabecera_description_ELTPARAGRAPHE000066"></a>

El encabezamiento debe contener al menos los dos elementos siguientes: 

- Contenido-Tipo:aplicación/JSON

- Authorization:key=&lt;Identificador GCM/Firebase&gt;


&lt;Identificador GCM/Firebase&gt; corresponde a: 

- el identificador de la aplicación en el servicio de mensajería en nube de Google. Se requiere una configuración específica para get este identificador. Para obtener más información, consulte [Configuración necesaria para las notificaciones Push en Android (GCM)](../WDLang3/1000021022.md).

- la clave del servidor que se encuentra en los parámetros de la aplicación Firebase. Se requiere una configuración específica para get esta llave. Para obtener más información, consulte [Configuración necesaria para las notificaciones Push en Android (Firebase)](../WDLang3/1000022215.md).



<a name="NOTE2_4"></a>


### Cuerpo description
<a name="cuerpo_description_ELTPARAGRAPHE000089"></a>

El cuerpo de la solicitud debe corresponder a un Document en formato JSON que contenga: 

- La lista de fichas que representan los dispositivos a los que se enviará el Notification.
	
	```txt
	"registration_ids": [<id device 1>,...<id device N>]
	```


- La versión del formato de intercambio de datos (obligatorio).
	
	```txt
	"WX_PUSH_EXT_VERSION":"1.0"
	```


- Los datos de Notification como parejas clave-valor para cada una de las propiedades de [Notification](../WDLang3/1000019441.md) a asignar. No es necesario proporcionar el valor de todas las propiedades de [Notification](../WDLang3/1000019441.md). Las propiedades no asignadas tomarán su valor de Default.


Observación: La versión actual del formato de intercambio es "1.0".

### Ejemplo de JSON Document
<a name="ejemplo_json_document_ELTPARAGRAPHE000268"></a>

Ejemplo de documento de JSON que describe un empuje de Notification para enviar: 


```txt
{
"registration_ids": ["Aki9 ... yXgt","BTh44 ... iQW)"],
"data":
{
"WX_PUSH_EXT_VERSION":"1.0",
"WX_PROP_TITLE":"Title of the notification",
"WX_PROP_MESSAGE":"Message of the notification",
"WX_PROP_ACTIVATEAPPLICATION":true,
"WX_PROP_FORMAT":
{
"WX_PROP_TYPE":1,
"WX_PROP_CONTENT":"Message of the notification\r\non two lines",
}
}
}
```

## Principio para enviar una Notification PUSH a un dispositivo iOS desde un lenguaje externo
<a name="principio_para_enviar_una_notification_push_dispositivo_ios_desde_lenguaje_externo_ELTTEXTE000450"></a>
**PRECAUCIÓN : este formato puede cambiar SIN previo aviso.**

Para enviar un Push Notification a un dispositivo iOS, debe: 

1. Sigue el Procedure para la creación de la carga útil en el sitio de Apple: [https://developer.apple.com/library/content/documentation/NetworkingInternet/Conceptual/RemoteNotificationsPG/BinaryProviderAPI.html](https://developer.apple.com/library/content/documentation/NetworkingInternet/Conceptual/RemoteNotificationsPG/BinaryProviderAPI.html)

2. Proporcione una JSON en el siguiente formato.
	
	```txt
	{
	    "aps": { 
	        "alert": {
	            "body": "Bob wants to play poker",
	            "action-loc-key": "PLAY" 
	            "launch-image": "play.png" 
	        },
	        "badge": 5
		"sound": "bingbong.aiff"
	    },
	    "wd": "bar"
	}
	```





Correspondencia de las propiedades de WLanguage

| Nombre de la clave JSON | Nombre de WLanguage Property |
| --- | --- |
| "body" | Mensaje |
| "action-loc-key" | ActionLabel |
| "launch-image" | Icon |
| "sound" | Sonido |
| "badge" | Insignia |
| "wd" | Contenido |

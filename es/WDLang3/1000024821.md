


## WLanguage Procedure llamado por SocketConnect



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Lenguaje Procedure escrito en el código del navegador ("callback"), llamado por [SocketConnect](../WDLang3/3070016.md). Nombre de la Procedure llamada cuando se establece la conexión con el servidor. Esta Procedure se utiliza para enviar un mensaje al servidor con [SocketWrite](../WDLang3/3070002.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
SocketConnect("client", "ws[s]://<WebSocketServerAddress>", OnSocketEvent)

INTERNAL PROCEDURE OnSocketEvent(nEvent, sMessage)
	
	SWITCH nEvent
		
		CASE SocketOpening
			SocketWrite("client", "Send message from the browser.")
		
		CASE SocketMessage
			Info("Server message received: " + sMessage)
			
		CASE SocketClosing
			Info("Socket closed", sMessage)
			
		CASE SocketError
			Error("Socket error")
	END
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`SocketConnect_Callback(<Event> , <Message>)`
---

**`<Event>: (Constante de tipo Integer)`**

Corresponde a una de las siguientes constantes:



|   |   |
| --- | --- |
| *SocketClosing* | El enchufe está cerrado. El parámetro &lt;Mensaje&gt; contiene un código que indica el motivo del cierre. |
| *SocketError* | Se ha producido un error durante la comunicación con el servidor.. El parámetro &lt;Mensaje&gt; está siempre vacío. |
| *SocketMessage* | La toma recibió un mensaje. El parámetro &lt;Mensaje&gt; contiene datos relativos al mensaje. |
| *SocketOpening* | El socket está conectado al servidor. Usted tiene la capacidad de enviar mensajes.  |



**`<Message>: (Cadena de caracteres)`**

Respuesta del servidor según la &lt;Event&gt;.

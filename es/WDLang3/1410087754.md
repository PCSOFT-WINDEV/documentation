
## WLanguage Procedure llamado por WebSocketClientConnect



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
WLanguage Procedure ("callback") llamado por [WebSocketClientConnect](../WDLang3/1410087751.md) cuando se establece la conexión con el servidor. 

Esta Procedure se llama en los siguientes casos: 

- cuando se establece la conexión con el servidor,

- cuando se reciben los datos,

- cuando el enchufe está cerrado,

- cuando se produce un error. 




ATENCIÓN : el Procedure se llama en el thread principal de la aplicación:

- Puede acceder a los controles de la interfaz de usuario desde la página web de Procedure.

- La Process no debe ser demasiado larga, ya que podría bloquear al usuario. 



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
WebSocketClientConnect("client", WebSocketClientConnect_Callback, "myserver.com")

INTERNAL PROCEDURE WebSocketClientConnect_Callback(nEvent, sMessage)
	SWITCH nEvent
		CASE SocketOpening
			SocketWrite("client", "Send message from the browser.")
		
		CASE SocketMessage
			Info("Server message received: " + sMessage)
			
		CASE SocketClosing
			Info("Socket closed", sMessage)
			
		CASE SocketError
			Error("Socket error")
	END
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`WebSocketClientConnect_Callback(<Event> , <Message>)`
---

**`<Event>: (Constante de tipo Integer)`**

Corresponde a una de las siguientes constantes:



|   |   |
| --- | --- |
| *SocketClosing* | El enchufe está cerrado. El parámetro &lt;Mensaje&gt; contiene un código que indica el motivo del cierre. |
| *SocketError* | Se ha producido un error durante la comunicación con el servidor.. El parámetro &lt;Mensaje&gt; está siempre vacío. |
| *SocketMessage* | La toma recibió un mensaje. El parámetro &lt;Mensaje&gt; contiene datos relativos al mensaje. |
| *SocketOpening* | El socket está conectado al servidor. Usted tiene la capacidad de enviar mensajes.  |



**`<Message>: (Cadena de caracteres)`**

Respuesta del servidor según la &lt;Event&gt;.

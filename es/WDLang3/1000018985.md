
## HTTPConfigure( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Configura las funciones que utilizan el protocolo HTTP ([HTTPRequest](../WDLang3/3043007.md), [HTTPSendForm](../WDLang3/1000017190.md), [SOAPRun](../WDLang3/3069013.md), [SOAPRunXML](../WDLang3/3069014.md), [Webservices importados](../WDLang3/3086001.md), [SFExecuteQuery](../WDLang5/1000018438.md), etc.).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
HTTPConfigure(httpConfigureMode, 1)
HTTPRequest("www.pcsoft.fr")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`HTTPConfigure(<Option to modify> , <New value>)`
---

**`<Option to modify>: (Constante de tipo Integer)`**

Parámetro a modificar: 


|   |   |
| --- | --- |
| *httpConfigureDisableCache* | En modo seguro (HTTPS), usado para administrar la caché del Explorador INTERNET. El  associated con esta constante puede ser:<br><br>- 0 (<u><u><u><u>False</u></u></u></u> por defecto): Se utiliza para volver a activar el caché del Explorador de INTERNET.<br><br>- 1 (<u><u><u><u>True</u></u></u></u>): Se usa para desactivar el caché del Explorador de INTERNET.<br><br><br> |
| *httpConfigureDisableCookie* | En modo seguro (HTTPS), utilizado para configurar el modo de gestión de las cookies HTTP del Explorador INTERNET. El &lt;Nuevo valor&gt; associated con esta constante puede ser:<br><br>- 0 (<u><u><u><u>False</u></u></u></u> por defecto): Se utiliza para volver a activar la gestión de las cookies HTTP del Explorador INTERNET.<br><br>- 1 (<u><u><u><u>True</u></u></u></u>): Se utiliza para desactivar la gestión de las cookies HTTP del Explorador INTERNET. <br><br><br> |
| *httpConfigureMode* | Configura el modo de gestión de las solicitudes. El &lt;Nuevo valor&gt; associated con esta constante puede ser:<br><br>- 0 (<u><u><u><u>False</u></u></u></u> por defecto): Peticiones HTTP gestionadas por framework comunes a los ejecutables WINDEV, sitios WEBDEV y servicios web. En este modo, si una solicitud requiere HTTPS, se utilizará automáticamente el modo 1.<br><br>- 1 (<u><u><u><u>True</u></u></u></u>): Solicitudes HTTP y/o HTTPS gestionadas por la API de Windows que ejecuta Process.<br><br>- 2: Solicitud HTTP y/o HTTPS gestionada por una configuración multiplataforma común a todos los marcos de trabajo. <br>	En este modo, una aplicación WINDEV puede intercambiar datos HTTPS del mismo modo que una aplicación WINDEV Mobile para iOS o Android. Este modo puede utilizarse para sustituir al API de Windows cuando éste no puede cifrar datos HTTPS.<br><br><br> |



**`<New value>: (Integro)`**

Nuevo valor del parámetro modificado.




## restRequest (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **restResponse** permite definir todas las características avanzadas de una respuesta REST. Las características de la solicitud REST pueden definirse y modificarse mediante diferentes propiedades de WLanguage. 

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
cMyRequest is restRequest
cMyRequest.URL = "http://www.windev.com"
cMyResponse is restResponse = RESTSend(cMyRequest)
IF ErrorOccurred THEN
	Error(ErrorInfo(errFullDetails))
ELSE
	Info(cMyResponse.Content)
END
```

### Propiedades específicas de las variables restRequest
<a name="propiedades_especificas_las_variables_restrequest_ELTPARAGRAPHE000042"></a>

Las siguientes propiedades pueden ser utilizadas para hacer una solicitud de DESCANSO a manipular:

| Nombre de la propiedad | Tipo utilizado | Efecto |
| --- | --- | --- |
| AuthToken | Variable de tipo [AuthToken](../WDLang3/1000022220.md) | Token para acceder a un Webservice que se utilizará en caso de autenticación a través del protocolo OAuth 2.0. Este acceso Token es recuperado por [AuthIdentify](../WDLang3/1000022219.md).<br><br>El acceso Token se pasa automáticamente al parámetro **access_Token** en la URL. Si el parámetro debe ser diferente, la URL a ejecutar debe ser construida por usted mismo.<br><br> |
| Cabecera | array asociativa de cadenas de caracteres | Clave/Valor set de cabeceras para enviar.<br><br>Ejemplo de uso de esta Property: <br><br><br><pre><code>cMyRequest is restRequest<br><br>cMyRequest.Header["Authorization"] = ...<br>	" WSSE profile=""UsernameToken"""<br>cMyRequest.Header["X-WSSE"] = ...<br>	"UsernameToken Username=blahblah "<br>cMyRequest.Header["Cache-Control"] = " no-cache"</code></pre><br> |
| ClientCertificate | Cadena o buffer | Corresponde a: <br><br>- una cadena de caracteres con un acceso ruta al archivo .p12 que contiene el Certificate para adjuntar a la solicitud. El Certificate se cargará automáticamente teniendo en cuenta:<br><br>	- el Certificate en la biblioteca ejecutable (si se ha integrado en la aplicación),<br><br>	- el Certificate en el lugar especificado del disco (si el Certificate no ha sido integrado en la biblioteca ejecutable).<br><br><br><br><br>- un buffer con el Certificate ([fLoadBuffer](../WDLang1/1000019410.md)).<br><br><br><br><br>Si esta propiedad corresponde a una cadena vacía (""), el Certificate por defecto se restablece a "&lt;None&gt;". |
| ClientCertificatePassword | Cadena de caracteres | Contraseña asociada al cliente Certificate (cadena vacía de Default) |
| ConnectionTimeout | Integro | Tiempo máximo de espera de la conexión expresado en milisegundos (20 segundos por Default, lo que significa 20000 milisegundos). Esta propiedad puede corresponder a: <br><br>- un número entero que corresponde al número de milisegundos,<br><br>- una variable de tipo Duration,<br><br>- la duración en un formato legible (por ejemplo, "20s" o "20000ms"). <br><br><br><br><br>La conexión falla si no se puede establecer después de este tiempo de espera ([RESTSend](../WDLang3/1000021476.md) devolverá un error, en el resultado de [restResponse](../WDLang3/1000021477.md) Variable). <br><br><br><br> |
| Contenido | Buffer | Mensaje a enviar al servidor. Este Property sólo se tiene en cuenta si el método de envío (**Método** Property) lo permite. <br><br>El mensaje a enviar debe cumplir con el protocolo HTTP utilizado. Por defecto, si se especifica esta Property y si no está vacía, el método utilizado es una solicitud POST; de lo contrario, es una solicitud get. |
| ContentType | Cadena de caracteres | Tipo de contenido del mensaje a enviar al servidor. Este Property sólo se tiene en cuenta si el método de envío (**Método** Property) lo permite. <br><br>Por Default, el tipo de mensaje corresponde a: "application/x-www-form-urlencoded".<br><br><br><br>Sin embargo, tiene la posibilidad de utilizar cualquier valor, por ejemplo: "text/xml", "application/javascript", "application/json", "application/xml", "image/jpeg", ...<br><br><br><br>Para enviar datos en bruto que serán leídos de una vez por el servidor de aplicaciones WEBDEV, utilice los siguientes tipos: <br><br>- "application/octet-stream".<br><br>- "text/xml".<br><br><br> |
| Contraseña | Cadena de caracteres | Contraseña associated con el nombre de usuario (cadena vacía por Default). Permite acceder a una Page con una URL protegida. **Atención**: La contraseña no se encripta cuando se envía a través de INTERNET.<br><br>Observación: Si se especifican las propiedades **UserName** y **Password**, se genera automáticamente la correspondiente cabecera "Authorization:Basic" en la cabecera de la solicitud. |
| Destino | Cadena de caracteres | ruta completa del archivo de copia de seguridad de los resultados de la consulta.<br><br> |
| DownloadProgress | Variable de tipo [httpProgress](../WDLang3/1000021222.md) | Progreso de la recepción de datos Notification. |
| drogadicta | Cadena de caracteres | Nombre utilizado para acceder a una página con una URL protegida (cadena vacía de Default). Este nombre se utiliza para identificar al usuario.<br><br>Observación: Si se especifican las propiedades **UserName** y **Password**, se genera automáticamente la correspondiente cabecera "Authorization:Basic" en la cabecera de la solicitud. |
| IgnoreError | Constante de tipo Integer | Especifica los errores ignorados. Corresponde a una constante o a una combinación de constantes: <br><br>- *httpIgnoreInvalidCertificate*: El Certificate es ignorado.<br><br>- *httpIgnoreInvalidCertificateName*: El nombre del sitio especificado en la Certificate es ignorado.<br><br>- *httpIgnoreExpiredCertificate*: La fecha de Certificate es ignorada.<br><br>- *httpIgnoreRedirection*: La redirección a una Page se ignora.<br><br>- *httpIgnoreRedirectToHTTP*: La redirección a un servidor no seguro está permitida.<br><br>- *httpIgnoreRedirectToHTTPS*: La redirección a un servidor seguro está permitida.<br><br><br><br><br><br><br><br><br> |
| MaxDownloadRate | Integro | Velocidad máxima de descarga de datos en kilobytes por segundo. Esta velocidad se da sólo a título informativo. <br>0 (valor de Default) significa que esta velocidad no está limitada.<br><br> |
| MaxUploadRate | Integro | Velocidad máxima de subida de datos en kilobytes por segundo. Esta velocidad se da sólo a título informativo.<br>0 (valor de Default) significa que esta velocidad no está limitada.<br><br> |
| Método | Constante de tipo Integer | El método HTTP utilizado: <br><br>- *httpCopy*: Método copy (7). <br><br>- *httpDelete*: Método DELETE (4). <br><br>- *httpGet*: Método get (1).<br><br>- *httpHead*: Método HEAD (5).<br><br>- *httpLock*: Método LOCK (12) (protocolo WebDAV). <br><br>- *httpMkCol*: Método MKCOL (10) (protocolo WebDAV).<br><br>- *httpMove*: Método MOVE (11) (protocolo WebDAV).<br><br>- *httpPatch*: Método PATCH (6). <br><br>- *httpPost*: Método Post (2). <br><br>- *httpPropFind*: Método PROPFIND (8) (protocolo WebDAV).<br><br>- *httpPropPatch*: Método PROPPATCH (9) (protocolo WebDAV).<br><br>- *httpPut*: Método PUT (3). <br><br>- *httpUnlock*: UNLOCK (13) (protocolo WebDAV).<br><br><br>De forma predeterminada, si la propiedad **Content** no está vacía, se utiliza el método *httpPost*. De lo contrario, se utiliza el método *httpGet*. |
| ProcedureTrace | Procedure | Nombre del WLanguage Procedure utilizado para identificar los datos y las cabeceras que se han enviado. Este procedimiento tiene el siguiente formato: <br><br><pre><code>PROCEDURE <Procedure name>(<InfoType> is int, <br>	<Data> is Buffer)</code></pre><br> donde: <br><br>- &lt;InfoType&gt; es una constante entera que corresponde al tipo de datos que se están rastreando.: <br><br>	- *httpTraceDataSent*: Fecha de la condena<br><br>	- *httpTraceDataReceived*: Datos recibidos. <br><br>	- *httpTraceHeaderSent*: Cabecera enviada. <br><br>	- *httpTraceHeaderReceived*: Cabecera recibida. <br><br><br><br><br>- &lt;Datos&gt; es un buffer que contiene la información. <br><br><br><br><br>Ejemplo de Procedure: <br><pre><code>PROCEDURE ProcTrace(nInfoType is int, ...<br>		bufByData is Buffer)<br>	SWITCH nInfoType<br>		CASE httpTraceHeaderSent<br>			Trace("Header: ", ...<br>				UTF8ToString(bufByData))<br>		CASE httpTraceDataSent:<br>			Trace("Data: ", ...<br>					bufByData)<br>		CASE httpTraceHeaderReceived:<br>			Trace("Header received: ", ...<br>				UTF8ToString(bufByData))<br>		CASE httpTraceDataReceived:<br>			Trace("Data received: ", ...<br>					bufByData)<br>	END</code></pre><br><br><br><br><br> |
|   | booleano | Indica si la propiedad **Content** corresponde al contenido en bruto:<br><br>- <u><u><u><u>True</u></u></u></u>: la propiedad **Content** corresponde a un buffer.  No se realiza ninguna modificación. <br><br>- <u><u><u><u>False</u></u></u></u> (valor Default): el contenido es modificado por el navegador. En particular, el texto UTF-8 se descodifica automáticamente. la **Contenido** Property devuelve el resultado como una cadena.<br><br><br> |
| SelectedCipher | Cadena de caracteres | Nombre del cifrado elegido para la petición segura. <br><br>**Esta propiedad está disponible en modo de solo lectura.** |
| SelectedSSLVersion | Constante de tipo Integer | Versión SSL elegida para una solicitud segura: <br><br>- *ProtocolDefault*: Default protocol (TLS 1.2). <br><br>- *ProtocolTLS1*: Protocolo TLS1.<br><br>- *ProtocolTLS1_1*: Protocolo TLS1.1.<br><br>- *ProtocolTLS1_2*: TLS1.2 protocol.<br><br>- *ProtocolTLS1_3*: TLS1.3 protocol.<br><br><br><br><br>**Esta propiedad está disponible en modo de solo lectura.** |
| Timeout | Integro | Tiempo máximo de respuesta (en milisegundos). Esta propiedad puede corresponder a: <br><br>- un número entero que corresponde al número de milisegundos,<br><br>- una variable de tipo Duration,<br><br>- la duración en un formato legible (por ejemplo, 1 s o 10 ms). <br><br><br><br><br>Este Property es set a 20 segundos por Default. <br><br><br><br>Observación: El tiempo de espera definido por [HTTPTimeOut](../WDLang3/3043008.md) no influye en este Property.  <br><br><br><br> |
| UploadProgress | Variable de tipo [httpProgress](../WDLang3/1000021222.md) | Notification de estado de progreso al subir los datos de Post. |
| URL | Cadena de caracteres | Dirección del servidor de Contact (URL Address).<br><br>Esta URL puede contener:<br><br>- el número de puerto para conectarse al servidor. <br>	El valor de Default es 80 en HTTP (corresponde a un servidor de páginas Web) y 443 en HTTPS. Para especificar un número de puerto, utilice el siguiente formato: "&lt;URL del servidor&gt;**:**&lt;N° del puerto&gt;". Por ejemplo: http://www.windev.com:80.<br><br>- parámetros adicionales. Estos parámetros se pueden utilizar para realizar una búsqueda o para rellenar un formulario. Por ejemplo, para buscar "windev" en "http://www.google.com", la URL será: "http://www.google.com/search?q=windev". <br><br><br><br><br>**Observaciones**:<br><br>- Para especificar tanto el número de puerto como los parámetros adicionales, utilice el siguiente formato: "&lt;URL del servidor&gt;**:**&lt;N° del puerto>/&lt;Parámetros adicionales>".<br><br>- Para realizar una transacción segura, la URL debe comenzar por "https://". En este caso, el modo de gestión de las solicitudes lo realiza siempre INTERNET Explorer (para más detalles, véase [HTTPConfigurar](../WDLang3/1000018985.md)). <br><br><br> |
| UserAgent | Cadena de caracteres | Identifica al cliente. El valor de Default es "PC SOFT Framework".<br><br>El contenido de la respuesta puede depender del agente usuario (por ejemplo, una solicitud realizada desde un dispositivo móvil y una solicitud realizada desde un navegador de PC pueden requerir páginas diferentes). En este caso, para más detalles, consulte la documentación del agente de usuario.<br><br> |
| VersionHTTP | Constante de tipo Integer | Versión HTTP utilizada por el servidor: <br><br>- *httpVersion2*: HTTP versión 2.0. Si el servidor no admite esta versión, se utiliza una versión anterior. <br><br>- *httpVersion2Only*: Fuerza la versión 2.0 de HTTP: si el servidor no soporta esta versión, se muestra un error fatal. <br><br>- *httpVersion1_1*: Versión HTTP 1.1.<br><br>- *httpVersion1_0*: HTTP versión 1.0. <br><br>- *httpVersionDefault*: HTTP versión 1.0. <br><br><br> |
| VersionSSL | Constante de tipo Integer | Versión SSL mínima y máxima admitida. <br><br>Para establecer las versiones TLS mínima y máxima en 1.1 y 1.3, basta con especificar las constantes *ProtocolTLS1_1* + *ProtocolTLS1_3*<br><br>Se pueden utilizar las siguientes constantes:  <br><br>- *ProtocolDefault*: Default protocol. Versión máxima admitida (o una versión inferior si el servidor no admite este protocolo). <br><br>- *ProtocolTLS1*: Protocolo TLS1.<br><br>- *ProtocolTLS1_1*: Protocolo TLS1.1.<br><br>- *ProtocolTLS1_2*: TLS1.2 protocol.<br><br>- *ProtocolTLS1_3*: TLS1.3 protocol.<br><br><br> |

### Gestionar las redirecciones
<a name="gestionar_las_redirecciones_ELTPARAGRAPHE000500"></a>

Las redirecciones son apoyadas por Default. Para ignorar los redireccionamientos, utilice la constante *httpIgnoreRedirection* en el **IgnoreError** Property.

### Cookies HTTP
<a name="cookies_http_ELTPARAGRAPHE000511"></a>

Las cookies recibidas en respuesta a una llamada a [RESTSend](../WDLang3/1000021476.md) se almacenan a la espera de una posterior llamada a [RESTSend](../WDLang3/1000021476.md) en el mismo dominio: el **restRequest** Variable se actualiza.
Para obtener más información, consulte [HTTPCookieManage](../WDLang3/1000019218.md).

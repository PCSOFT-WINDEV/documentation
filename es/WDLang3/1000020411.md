


## nfcTag (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **nfcTag** se utiliza para definir las características avanzadas de una etiqueta NFC (Near Field Communication). Las características de esta etiqueta NFC pueden ser definidas y modificadas por varias propiedades de WLanguage. 

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).

### Propiedades específicas de las variables nfcTag
<a name="propiedades_especificas_las_variables_nfctag_ELTPARAGRAPHE000042"></a>

Las siguientes propiedades pueden ser utilizadas para manipular una etiqueta NFC:

| **Nombre de la propiedad** | **Tipo utilizado** | **Efecto** |
| --- | --- | --- |
| Datos | Array de [nfcData](../WDLang3/1000020406.md) | Array que contiene: <br><br>- los datos leídos en una etiqueta NFC (tras una llamada a [NFCReadTag](../WDLang3/1000020416.md)).<br><br><br> |
| escribir a máquina | Constante de tipo Integer | Tipo de etiqueta correspondiente a uno de los siguientes valores: <br><br>- *nfcUnknown*: tipo desconocido. Valor de Default si no se realizó ninguna operación de lectura con la etiqueta.<br><br>- *nfcMifareClassic*: NFC Mifare Classic<br><br>- *nfcType1*: Foro NFC Tipo 1<br><br>- *nfcType2*: Foro NFC Tipo 2<br><br>- *nfcType3*: Foro NFC Tipo 3<br><br>- *nfcType4*: Foro NFC Tipo 4<br><br><br>El tipo de etiqueta se llena automáticamente después de una operación de lectura. <br><br>**Este Property está disponible en formato de solo lectura** .<br><br> |
| Identificador | Búfer binario | Identificador de la etiqueta NFC (corresponde al "identificador de hardware" de una etiqueta con placa). Este identificador se rellena automáticamente después de una operación de lectura.<br><br>**Este Property está disponible en formato de solo lectura** .<br><br> |
| MaxSize | Integro | Tamaño máximo (en bytes) de los datos que pueden almacenarse en la etiqueta NFC.<br><br>Esta Property se llena automáticamente después de una operación de lectura.<br><br>**Este Property está disponible en formato de solo lectura** .<br><br> |
| ReadOnly | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si la etiqueta es de solo lectura,<br><br>- <u><u><u><u>False</u></u></u></u> si no es protected en modo escritura. <br><br><br>**Durante una operación de lectura** , este Property se llena automáticamente. <br><br>**Durante una operación de escritura** , si **ReadOnly** Property se establece en <u><u><u><u>True</u></u></u></u> , la etiqueta se escribirá automáticamente: protected: sus datos ya no pueden ser modificados. **Esta operación es irreversible.**. Algunas etiquetas NFC no soportan este modo. Para obtener más información, consulte la ayuda sobre la función [NFCWriteTag](../WDLang3/1000020417.md).<br><br> |


<a name="NOTE0_2"></a>


### Principales observaciones
<a name="principales_observaciones_ELTPARAGRAPHE000160"></a>

- Las funciones NFC admiten las etiquetas correspondientes a los tipos estandarizados 1 a 4 definidos por el Foro NFC ([http://www.nfc-forum.org/specs/spec_list/](http://www.nfc-forum.org/specs/spec_list/)) y sólo pueden leer y escribir datos codificados en el formato NDEF (NFC Data Exchange Service).

- Las funciones NFC no se pueden utilizar en el simulador ni en el emulador.


## SocketAccept( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ejecutar en el servidor, esta función acepta una petición para conectarse a un socket existente en el servidor.. Esta función se utiliza para crear el canal de comunicación entre el socket del servidor y el socket del cliente.

Esto le permite crear un canal de comunicación diferente para cada ordenador cliente.. Cada uno de estos canales de comunicación puede ser manejado en un Thread independiente.

**Observaciones**:

- Si el ordenador cliente utiliza una aplicación WINDEV para conectarse a este socket, esta aplicación tendrá que utilizar [SocketConnect](../WDLang3/3070016.md).

- **SocketAccept** puede utilizarse con el SSL sockets.

- La función **SocketAccept** no puede utilizarse con un socket UDP.








**Recordatorio**: Un socket es un recurso utilizado por las aplicaciones para comunicarse de un equipo a otro, independientemente del tipo de red.

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Creates a socket on the server
IF SocketCreate("Server", 8000) = False THEN
	Error("Creation error " + ErrorInfo(errMessage))
ELSE
	// Manage the connections of the client computers in a thread
	ThreadExecute("Thread1", threadNormal, WaitProcedure)
END
```

```wl
// ------------------------------------------------------
PROCEDURE WaitProcedure()
LOOP
	IF SocketWaitForConnection("Server") = True THEN
		Channel is string
		// If the connection is accepted, 
		// this connection is managed in a specific thread
		Channel = SocketAccept("Server")
		ThreadExecute(Channel, threadNormal, ProcedureManagement, Channel)
	END
END
```

## Sintaxis

`<Result> = SocketAccept(<Socket name>)`
---

**`<Result>: (Cadena de caracteres)`**



- Nombre del canal de comunicación con el socket cliente que solicitó la conexión. Este nombre será utilizado por todas las demás funciones de Socket para especificar este canal de comunicación.. 

- Cadena vacía ("") si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Socket name>: (Cadena de caracteres)`**

Nombre del socket del servidor que acepta la conexión.

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Atención: El nombre del socket distingue entre mayúsculas y minúsculas.

## Observaciones




### Canal de comunicación
<a name="canal_comunicacion_ELTPARAGRAPHE000102"></a>

El canal de comunicación abierto con la función **SocketAccept** debe cerrarse con la función [SocketClose](../WDLang3/3070015.md).

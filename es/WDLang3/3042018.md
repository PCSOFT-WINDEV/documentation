


## grDestinationBMP (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Define un archivo BitMap como destino de un gráfico (BMP significa BitMap). Este archivo se creará durante el siguiente dibujo de la carta ([grDraw](../WDLang3/3042026.md)).

**Observaciones**:

- Esta función no se puede utilizar con los controles gráficos. 

- Esta función no se puede utilizar en la editor de reportes.

- Esta función no se puede utilizar para dibujar un gráfico de cilindro en 3D.






![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) **Atención**: Se requiere una configuración específica para usar esta función en Linux. Para obtener más información, consulte [Los gráficos](../WDChamp/3042060.md). 




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Define the "C:\Charts\Chart.BMP" file 
// as destination of "MyChart"
grDestinationBMP("MyChart", "C:\Charts\Chart.BMP")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`grDestinationBMP(<Chart name> , <Name of BMP file>)`
---

**`<Chart name>: (Cadena de caracteres)`**

Nombre del gráfico a utilizar, definido por [grCreate](../WDLang3/3042005.md). 

**`<Name of BMP file>: (Cadena de caracteres)`**

Nombre del archivo BMP donde se guardará el gráfico. Este archivo se crea durante la llamada a [grDraw](../WDLang3/3042026.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Archivo BMP como destino
<a name="archivo_bmp_como_destino_ELTPARAGRAPHE000078"></a>

- Durante la siguiente llamada a [grDraw](../WDLang3/3042026.md), el gráfico se dibujará en el archivo &lt;Nombre del archivo BMP&gt;.

- Este archivo puede ser leído por una herramienta que reconoce el formato BMP.

- El tamaño de este archivo puede ser definido por [TAMAÑO DE GRINdSize](../WDLang3/3042016.md).

- Le recomendamos que previsualice el gráfico antes de guardarlo en un archivo.



<a name="NOTE0_2"></a>


### Crear y dibujar un gráfico mediante programación
<a name="crear_dibujar_grafico_mediante_programacion_ELTPARAGRAPHE000094"></a>

Para dibujar un gráfico:

- crear el gráfico ([grCreate](../WDLang3/3042005.md)),

- añadir datos ([grAddData](../WDLang3/3042023.md), [grStockAddData](../WDLang3/3042009.md) y [grScatterAddDataXY](../WDLang3/3042032.md)),

- definir el destino de la carta con las siguientes funciones: 

	- **grDestinationBMP**, 

	- [grDestinoControl](../WDLang3/3042020.md), 

	- [grDestinoEMF](../WDLang3/3042017.md), 

	- [grDestinoWnd](../WDLang3/3042031.md),

	- [grDestinationImage](../WDLang3/1000020952.md),

	- [grDestinoWMF](../WDLang3/3042019.md). 




- dibujar el gráfico ([grDraw](../WDLang3/3042026.md)).



<a name="NOTE0_3"></a>


### Diferencias entre grDestinationBMP y grSaveBMP
<a name="diferencias_entre_grdestinationbmp_grsavebmp_ELTPARAGRAPHE000142"></a>

- La función **grDestinationBMP** permite definir el destino del próximo dibujo del gráfico. Cuando se utiliza [grDraw](../WDLang3/3042026.md), el gráfico se dibujará en el archivo especificado y se tendrán en cuenta todos los cambios realizados desde la última vez que se dibujó el gráfico.

- [grSaveBMP](../WDLang3/3042050.md) se utiliza para guardar el gráfico en un archivo en formato BMP. La Image guardada corresponde al gráfico durante la última llamada a [grDraw](../WDLang3/3042026.md). Los cambios realizados en el gráfico desde la última vez que se llamó a [grDraw](../WDLang3/3042026.md) serán ignorados.

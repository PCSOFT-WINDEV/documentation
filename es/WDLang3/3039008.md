


## StreamConnect (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Conecta una corriente (y el control de associated) a un Address específico.

**Observación**: Esta función es equivalente a la [Address](../Proprietes/2510017.md) Property.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Specify the name of the caller
CONF_Conference1.Identifier = UserName
// Connect the Conference control 
StreamConnect(CONF_Conference1, RecipientAddress)
// equivalent to:
// CONF_Conference1.Address = RecipientAddress
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = StreamConnect(<Conference control> , <Address>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se estableció la conexión, 

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Conference control>: (Nombre del control o cadena de caracteres)`**

Nombre de control Conferencia para asociar con el Address especificado.

**`<Address>: (Cadena de caracteres)`**

Address del corresponsal. Este Address puede ser:

- Una IP Address en formato XXX.XXX.XXX.XXX (125.5.110.100 por ejemplo).

- Un nombre de servidor (pop3.serveur.fr por ejemplo). Se recomienda esta sintaxis. 

- Una cadena vacía ("""). En este caso, se utiliza la dirección del equipo actual (permite simplificar las pruebas de una aplicación).

## Observaciones


### Desconexión de una corriente
<a name="desconexion_una_corriente_ELTPARAGRAPHE000053"></a>

Cuando se desconecta un flujo (por el llamador o el llamado), el "Fin de la conexión" Event se ejecuta automáticamente.




## FTPSend( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Transfiere un archivo o directorio a un servidor FTP (File Transfer Protocol).



<a name="Example1"></a>
<a name="sample_code"></a>

## Sintaxis

`<Result> = FTPSend(<Connection identifier> , <File/Directory to Transfer> , <Destination file/directory> [, <WLanguage procedure> [, <Transfer mode>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la transferencia,

- <u><u><u><u>False</u></u></u></u> en caso contrario. El estado de error reporte es devuelto por [ErrorInfo](../WDLang1/3013008.md).




**`<Connection identifier>: (Integro)`**

Identificador de la conexión, devuelto por la función [FTPConnect](../WDLang3/3040016.md).


**`<File/Directory to Transfer>: (Cadena de caracteres)`**

Nombre y ruta completa (o relativa) del archivo (o directorio) a transferir. Se puede utilizar una ruta de tipo UNC. No se puede utilizar ningún carácter comodín (\* o?).
***Si este parámetro corresponde a un directorio***Todos los archivos encontrados en este directorio son transferidos.. Se crea un directorio con el mismo nombre en el servidor FTP en la ubicación especificada por &lt;Destination File/Directory&gt;..
***Si este parámetro corresponde a un archivo***:

- Si &lt;Destino Archivo/Directorio&gt; es un directorio que se encuentra en el servidor FTP, el archivo a recuperar se copia en el directorio de destino.

- Si &lt;Destination File/Directory&gt; es un archivo que se encuentra en el servidor FTP, el archivo a transferir se copia y se le cambia el nombre.




**`<Destination file/directory>: (Cadena de caracteres)`**

Nombre y ruta completa (o relativa) del archivo (o directorio) de destino. Este archivo (o directorio) se encuentra en el servidor FTP. Las secciones de la ruta están separadas por barras oblicuas ("/").
Una ruta completa tiene el siguiente formato: "/&lt;NombreDirectorio&gt;/&lt;NombreArchivo&gt;". La jerarquía obtenida tiene el siguiente formato: "/&lt;NombreDirectorio&gt;/&lt;NombreArchivo&gt;".
Una ruta relativa tiene el siguiente formato: "&lt;NombreDirectorio&gt;/&lt;NombreArchivo&gt;". La estructura de árbol tiene el siguiente formato: "/&lt;CurrentServerDirectory&gt;/&lt;DirectoryName&gt;/&lt;FileName>".

**`<WLanguage procedure>: (Nombre de Procedure)`**

El nombre de Procedure fue llamado automáticamente para comprobar el progreso de la transferencia.


**`<Transfer mode>: (Constante opcional de tipo Integer)`**

Modo de transferencia utilizado:


|   |   |
| --- | --- |
| *ftpASCIIMode* | Transferencia realizada en modo ASCII. Este modo se utiliza para transferir datos entre dos sistemas operativos diferentes, por ejemplo, de UNIX a Windows: el archivo transferido cambia al formato ASCII del sistema de destino. |
| *ftpBinaryMode*<br>(Valor predeterminado) | Transferencia realizada en modo binario: El archivo es idéntico entre el envío y la recepción. |

### Gestión del progreso de la transferencia
<a name="gestion_del_progreso_transferencia_ELTPARAGRAPHE000116"></a>

**FTPSend** es una función de bloqueo: no se puede realizar ninguna otra acción hasta que finalice la transferencia en curso. Para comprobar el progreso de la transferencia, **FTPSend** puede llamar automáticamente a &lt;nombre de Procedure&gt; Procedure a intervalos regulares.

Este procedimiento se declara de la siguiente manera:


```txt
PROCEDURE <Procedure name>(<Total size>, <Transferred size>)
```


- &lt;Tamaño total&gt; es un número entero que corresponde al tamaño total del archivo o directorio actualmente transferido.

- &lt;Tamaño transferido&gt; es un entero que corresponde al número de bytes ya transferidos.




Por ejemplo, es posible:

- mostrar una barra de progreso,

- saber si la transferencia ha finalizado (&lt;Tamaño total&gt; = &lt;Tamaño transferido>).




La función **FTPSend** no se puede interrumpir si no se han transferido todos los archivos. Para forzar la interrupción de la navegación, utilice la siguiente línea de código en el procedimiento &lt;Nombre de procedimiento&gt;:


```wl
RESULT False
```
En los demás casos (para continuar la navegación), el procedimiento &lt;Nombre de procedimiento&gt; DEBE devolver <u><u><u><u>True</u></u></u></u>.

Se genera un error si no se devuelve ningún valor (ni <u><u><u><u>True</u></u></u></u> ni <u><u><u><u>False</u></u></u></u>) por el &lt;nombre de Procedure&gt; Procedure.

### Tiempo límite de transferencia
<a name="tiempo_limite_transferencia_ELTPARAGRAPHE000160"></a>

De forma predeterminada, todas las funciones FTP fallan si no se recibe ninguna respuesta del servidor FTP después de 20 segundos. Este tiempo de espera se puede modificar al conectarse al servidor FTP con la función [FTPConnect](../WDLang3/3040016.md).

<a name="NOTE0_4"></a>


### Permisos de usuarios
<a name="permisos_usuarios_ELTPARAGRAPHE000188"></a>

Sólo un usuario que tenga derechos de escritura en el servidor FTP puede enviar archivos a un servidor FTP.. En la mayoría de los casos, no se conceden derechos de escritura a los usuarios "anónimos".
<a name="NOTE0_5"></a>


### Ruta relativa o absoluta
<a name="ruta_relativa_absoluta_ELTPARAGRAPHE000195"></a>

Las nociones de ruta relativa y ruta absoluta son de gran importancia en una aplicación FTP.

- Una ruta que comienza con un slash se considera como una ruta **absoluta**:  es la ruta con respecto a la raíz del servidor FTP (parámetro específico del servidor).
	ej: /publicidad/usuario/JULIA

- Si la ruta no comienza con un slash, se considera como una **ruta relativa**, es decir, una ruta con respecto al directorio en curso. El directorio en curso puede conocerse o modificarse con la función [FTPCurrentDir](../WDLang3/3040003.md).


Al conectarse a un sitio FTP, el directorio inicial ("home directory" del usuario) no necesariamente es la raíz del servidor FTP. Por lo tanto, se recomienda utilizar rutas relativas.

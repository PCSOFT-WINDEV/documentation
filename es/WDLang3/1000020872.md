


## inAppListProductInfo( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Recupera, de la tienda, información sobre los productos "In-App" associated con la aplicación.  
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
arrIdentifier is array of strings 

IF InAndroidMode() THEN
	arrIdentifier = ["product1", "product2", "product3", "product4", "unknown product"]
ELSE
	arrIdentifier = ["ProductID1", "ProductID2", "ProductID3", ...
		"ProductID4", "ProductID5", "unknown product"]
END

IF inAppListProductInfo(arrIdentifier, ListProductInfo) THEN
	 ToastDisplay("Asks for the list of products sent.")
ELSE
	Error("ERROR: " + ErrorInfo)
END
```

```wl
// --------------------
// ListProductInfo procedure
PROCEDURE ListProductInfo(bSuccess is boolean, ...
		arrProducts is array of inAppProduct)
IF NOT bSuccess THEN
	Error("Error: " + ErrorInfo())
	RETURN
END

LooperDeleteAll(LOOP_ProductInfo)
FOR I = 1 _TO_ arrProducts.Count
	LooperAddLine(LOOP_ProductInfo, ...
		arrProducts[i].Caption, arrProducts[i].Description, ...
		arrProducts[i].Identifier, arrProducts[i].Price)
END
// Copy into the array global to the window
garrProduct <= arrProducts
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = inAppListProductInfo(<Identifiers of products> , <WLanguage procedure>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la solicitud para obtener información sobre el producto,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Identifiers of products>: (Array de cadenas de caracteres)`**

Identificadores de los productos a utilizar. Estos identificadores son los que se definen al crear los productos: 






Observación: Si un identificador no corresponde a un producto existente, se ignora. 

**`<WLanguage procedure>: (Nombre de Procedure)`**

Nombre de la WLanguage Procedure ("callback" Procedure) que se ejecuta al recibir información sobre los productos. Este procedimiento tiene el siguiente formato: 

```txt
PROCEDURE <Procedure name>(<Success> is boolean, 
	<Product information> is array of inAppProduct)
```
donde 


- &lt;El éxito es booleano: 

	- <u><u><u><u>True STRIMEULB</u></u></u></u> si se ha recuperado información sobre los productos, 

	- <u><u><u><u>False</u></u></u></u> en caso contrario. Para get más detalles sobre el error, utilice [ErrorInfo](../WDLang1/3013008.md) antes de cualquier otra llamada a una función de WLanguage (esto reiniciaría el error).




- &lt;Información del producto&gt; es una array de variables de [inAppProduct](../WDLang3/1000020868.md) que contiene información sobre cada uno de los productos existentes cuyo identificador se ha especificado. Si el identificador no corresponde a un producto existente, se ignora. 
	Si &lt;Éxito&gt; = <u><u><u><u>False</u></u></u></u>, el array estará vacío. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- La solicitud de información es asíncrona para no bloquear la aplicación al recuperar información sobre los productos.. Esta operación puede ser bastante larga porque se deben realizar peticiones de red.

- Para conocer la lista de productos comprados por el usuario y no consumidos y la lista de suscripciones a Active, utilice [inAppListPurchase](../WDLang3/1000020888.md).

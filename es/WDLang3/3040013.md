


## FTPTime( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve los diferentes tiempos (creación, modificación o acceso) de associated con un archivo encontrado en un servidor FTP (File Transfer Protocol).
![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Devuelve la hora de modificación de un archivo encontrado en un servidor FTP (File Transfer Protocol).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003040013&name=ftptime_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Find out the time of modification made to
// "/Documents/File.txt" on the FTP server
ResTime = FTPTime(7, "/Documents/File.txt")
```

<a name="SYNTAX1"></a>

## Sintaxis

### Averiguar los diferentes tiempos de associated con el último archivo visto

`<Result> = FTPTime([<Type of time>])`
---

**`<Result>: (Cadena de caracteres)`**



- Hora (en formato HHMMSS) associated con el último archivo accedido por una de las siguientes funciones FTP: [FTPAttribute](../WDLang3/3040006.md), [FTPDate](../WDLang3/3040015.md), [FTPSize](../WDLang3/3040007.md), [FTPListFile](../WDLang3/3040018.md).

- Cadena vacía ("") si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Tiempo de modificación del último archivo visualizado.

A diferencia de la sintaxis 2, no se envía ninguna petición al servidor. Por lo tanto, esta sintaxis se recomienda siempre que sea posible.

**`<Type of time>: (Constante opcional)`**

Indica el tiempo de uso:



|   |   |
| --- | --- |
| *ftpAccess* | Hora del último acceso al fichero. |
| *ftpCreation* | Hora de creación del archivo. |
| *ftpModification*<br>(Valor predeterminado) | Hora de la última modificación realizada en el fichero. |


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Este parámetro no está disponible. Sólo está disponible la hora en que se modificó el último archivo visualizado.


<a name="SYNTAX2"></a>

### Averiguar los diferentes tiempos de associated con el archivo especificado

`<Result> = FTPTime(<Connection identifier> , <File name> [, <Type of time>])`
---

**`<Result>: (Cadena de caracteres)`**



- Hora associated con el archivo especificado (en formato HHMMSS),

- Cadena vacía ("") si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Tiempo de modificación del archivo especificado.

**`<Connection identifier>: (Integro)`**

Identificador de la conexión, devuelto por la función [FTPConnect](../WDLang3/3040016.md).


**`<File name>: (Cadena de caracteres)`**

Nombre y ruta completa (o relativa) del archivo al que se accede. Las secciones de la ruta están separadas por barras oblicuas ("/").

**`<Type of time>: (Constante opcional)`**

Indica el tiempo de uso:



|   |   |
| --- | --- |
| *ftpAccess* | Hora del último acceso al fichero. |
| *ftpCreation* | Hora de creación del archivo. |
| *ftpModification*<br>(Valor predeterminado) | Hora de la última modificación realizada en el fichero. |


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Este parámetro no está disponible. Sólo está disponible la hora de modificación del archivo especificado.

## Observaciones


### Timeout
<a name="timeout_ELTPARAGRAPHE000148"></a>

De forma predeterminada, todas las funciones FTP fallan si no se recibe ninguna respuesta del servidor FTP después de 20 segundos. Este tiempo de espera se puede modificar al conectarse al servidor FTP con la función [FTPConnect](../WDLang3/3040016.md).
<a name="NOTE0_3"></a>


### Permisos de usuarios
<a name="permisos_usuarios_ELTPARAGRAPHE000158"></a>

Solo un usuario con permisos de lectura en el servidor FTP puede obtener información sobre los archivos o directorios que se encuentran en dicho servidor. En la mayoría de los casos, los usuarios "anónimos" tienen permisos de lectura.
<a name="NOTE0_4"></a>


### Ruta relativa o absoluta
<a name="ruta_relativa_absoluta_ELTPARAGRAPHE000165"></a>

Las nociones de ruta relativa y ruta absoluta son de gran importancia en una aplicación FTP.

- Una ruta que comienza con un slash se considera como una ruta **absoluta**:  es la ruta con respecto a la raíz del servidor FTP (parámetro específico del servidor).
	Ejemplo: /publicidad/usuario/JULIA

- Si la ruta no comienza con un slash, se considera como una **ruta relativa**, es decir, una ruta con respecto al directorio en curso. El directorio en curso puede conocerse o modificarse con la función [FTPCurrentDir](../WDLang3/3040003.md).


Al conectarse a un sitio FTP, el directorio inicial ("home directory" del usuario) no necesariamente es la raíz del servidor FTP. Por lo tanto, se recomienda utilizar rutas relativas.




## zipFindFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Encuentra un archivo en un archivo (en formato CAB, ZIP, RAR, WDZ, 7z, TAR o TGZ (TAR.GZ)).. La búsqueda se realiza en la ruta almacenada del archivo.








<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003082002&name=zipfindfile_funcion&product=WD">Ver ejemplos adicionales</a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Create the archive 
MyArchive is zipArchive
ResCreate = zipCreate(MyArchive, "C:\Temp\Archive.zip")
IF ResCreate = 0 THEN
	...
	// Find a file whose stored path contains "Temp"
	ResFileFound = zipFindFile(MyArchive, "Temp", False)
	...
END
```

## Sintaxis

`<Result> = zipFindFile(<Archive> , <Sought path> [, <Search mode> [, <Start>]])`
---

**`<Result>: (Integro)`**



- Índice del fichero buscado en el archivo, 

- -1 si se produce un error o si no se encuentra el archivo.




**`<Archive>: (Cadena de caracteres o zipArchive Variable)`**

Nombre del archivo en el que se debe buscar el archivo.
Este nombre puede corresponder a:

- un nombre de archivo definido con [zipOpen](../WDLang3/3082044.md) o [zipCreate](../WDLang3/3082003.md).

- el nombre de una variable de tipo [zipArchive](../WDLang3/1000018679.md). 
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las variables [zipArchive](../WDLang3/1000018679.md) no están disponibles en Java.




**`<Sought path>: (Cadena de caracteres)`**

Completo o Partial ruta del archivo para encontrar. Este parámetro puede corresponder a:

- un nombre de archivo, 

- la ruta completa de un archivo, 

- el ruta y el nombre de un archivo, 

- ...


Cuando se especifica una ruta, se utiliza una barra invertida (\\) como separador, y no se debe especificar el directorio raíz.

La búsqueda no distingue entre mayúsculas y minúsculas (minúsculas / mayúsculas).

**`<Search mode>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> (valor predeterminado) para una búsqueda exacta,

- <u><u><u><u>False</u></u></u></u> para una búsqueda genérica. La búsqueda no distingue entre mayúsculas y minúsculas (minúsculas / mayúsculas).




**Archivos en formato 7z**: Este parámetro no está disponible. 

**`<Start>: (Entero opcional)`**

Índice del fichero a partir del cual debe realizarse la búsqueda (1 por Default, es decir, el primer fichero incluido en el archivo). 

**Archivos en formato 7z**: Este parámetro no está disponible.

## Observaciones


### Modo de búsqueda
<a name="modo_busqueda_ELTPARAGRAPHE000178"></a>

Dependiendo del  seleccionado, el resultado puede variar:

- Si se realiza una búsqueda exacta, la función **zipFindFile** localizará los archivos cuya ruta almacenada es estrictamente idéntica a la &lt;Ruta buscada&gt;. 
	Por ejemplo: un archivo contiene "Temp.doc", "Temp1.doc" y "Temp2.doc".. Una búsqueda exacta de "Temp.doc" encontrará "Temp.doc".

- Si se realiza una búsqueda genérica, la función **zipFindFile** localizará todos los archivos cuya ruta almacenada contiene &lt;Ruta almacenada&gt;.
	Por ejemplo: un archivo contiene "Temp.doc", "Temp1.doc" y "Temp2.doc".. Una búsqueda genérica de la cadena "Temp" encontrará tres archivos.


**Recordatorio**: El ruta almacenado se define con [zipAddFile](../WDLang3/3082008.md) o [zipAddDirectory](../WDLang3/3082014.md).<a name="NOTE0_2"></a>


### Índice de ficheros en el archivo
<a name="indice_ficheros_archivo_ELTPARAGRAPHE000208"></a>

Cuando se añade un fichero a un archivo, se le asigna automáticamente un índice. Este índice corresponde al orden de inclusión de los ficheros en el archivo. Para seleccionar un fichero en el archivo comprimido, puede utilizar:

- el índice del elemento.

- la ruta almacenada del elemento.

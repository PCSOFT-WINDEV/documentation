## Detalles
<a name="detalles_ELTTEXTE000138"></a>
Para pasar parámetros a un Procedure ejecutado en un servidor de servicios Web XML .Net, WLanguage utiliza la siguiente estructura:

| Variable | escribir a máquina | Detalles |
| --- | --- | --- |
| DotNet.NameSpace | Cadena de caracteres opcional | "NameSpace" del parámetro |
| Nombre DotNet. | Cadena de caracteres | Nombre del parámetro |
| DotNet.XMLParam | Cadena de caracteres opcional | Estructuras de un parámetro expresado en formato XML. Los otros parámetros (Value, Name, Type, NameSpace y EncodingStyle) se ignoran si se especifica este parámetro. |
| Estilo de codificación DotNet. | Cadena de caracteres opcional | Estilo de codificación del parámetro |
| Tipo DotNet. | Constante | Tipo de parámetro |
| Valor DotNet. | Cualquier tipo | Valor del parámetro<br><br>El Variable DotNet.Value se ignora si se usa el DotNet.ExtendedValue. <br><br>**Cadena de caracteres (cadena Unicode)**<br>Esta cadena se convierte automáticamente al formato ANSI cuando se envía.. <br><br>Cadena **Binary**<br>Para transmitir una cadena binaria, use el DotNet.ExtendedValue Variable. |
| DotNet.ExtendedValue | Cualquier tipo | Valor del parámetro<br><br>**Cadena de caracteres (cadena Unicode)**<br>Esta cadena no se modifica cuando se envía. No se realiza ninguna conversión. <br><br>useful cuando se debe transmitir una cadena de caracteres en formato UNICODE por ejemplo. |


Esta estructura debe utilizarse para cada parámetro. Por ejemplo:

```txt
DotNet.Value[1] = 94010
DotNet.Name[1] = "ZipCode"
DotNet.Type[1] = DotNetStringType
```


Esta estructura es equivalente a:

```txt
DotNet.XMLParam[1] = <ZipCode xsi:type="xsd:string">94010</ZipCode>
```


## WidgetDisplay( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Actualiza una ventana de Widget y ejecuta el widget "Refresh" Event. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
WidgetDisplay(WIN_Widget1)
```


<a name="Example2"></a>

```wl
// "Refresh the WIN_Widget widget" event
// Refreshes the APT 
RefreshAPT()

WidgetDisplay(WIN_Details)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`WidgetDisplay([<Window used>])`
---

**`<Window used>: (Nombre de la ventana opcional)`**

Nombre de la ventana del widget que se va a actualizar. 





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Visualización de la ventana Widget
<a name="visualizacion_ventana_widget_ELTPARAGRAPHE000048"></a>

Las ventanas de widgets son mostradas por la Process que gestiona la pantalla de inicio del dispositivo. Los widgets no se actualizan permanentemente, sino sólo en los siguientes casos: 

- Después de añadir una nueva instancia del widget a la pantalla de inicio.

- Después de hacer clic en el widget (o en uno de sus controles). 

- Después de actualizar el contenido del widget. 




**WidgetDisplay** obliga a refrescar la pantalla. Esta función también ejecuta el "widget de actualización" Event en la ventana del widget.

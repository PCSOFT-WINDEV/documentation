
## Gestión del protocolo SNMP
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000145"></a>
SNMP (Simple Network Management Protocol) es el protocolo estándar para la gestión de los dispositivos de red.. Las funciones SNMP le permiten crear aplicaciones utilizadas para monitorizar los elementos de red: PC, servidores, switches, robots, ... Estas funciones también se pueden utilizar para:

- Supervisar la carga de la red

- Optimizar el modo de funcionamiento de la red

- Gestionar los posibles problemas de red.




Observación: Soporta SNMP versión V1. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Activación
<a name="activacion_ELTTEXTE000169"></a>
Para que los PCs que se encuentran en una red puedan proporcionar información, el agente SNMP debe estar habilitado.

En Windows, se deben realizar los siguientes pasos:

1. Muestra el panel de control.

2. Seleccione "Agregar o quitar programas".

3. Seleccione la opción "Agregar componentes de Windows".

4. Añada (si es necesario) el módulo "Herramienta de gestión y supervisión".

5. Inicie el administrador de servicios de Windows y seleccione "Servicio SNMP".

6. En las propiedades del servicio (clic derecho del ratón), en la pestaña "Seguridad", añada dos comunidades: una comunidad "pública" con derechos de sólo lectura y una comunidad "privada" con derechos de lectura/escritura.

## Implementación
<a name="implementacion_ELTTEXTE000193"></a>
Las funciones SNMP se implementan en dos pasos:

1. [Conexión al dispositivo correspondiente](#NOTE3_2).

2. [Recuperación de la información solicitada](#NOTE3_3).



<a name="NOTE3_2"></a>


### Conexión al dispositivo
<a name="conexion_dispositivo_ELTPARAGRAPHE000052"></a>

La conexión la establece [Sesión de inicio SNMPS](../WDLang3/1000017067.md).


```wl
SessionID = SNMPStartSession("TestDoc", "public")
```


Esta función requiere:

- El IP Address del dispositivo.

- El nombre de la comunidad SNMP con la que se implementará el diálogo.




El identificador devuelto se utiliza para dialogar con el dispositivo.
<a name="NOTE3_3"></a>


### Información de lectura
<a name="informacion_lectura_ELTPARAGRAPHE000071"></a>

La información es leída en la estructura de árbol SNMP por [SNMPGet](../WDLang3/1000017069.md).

Esta función requiere:

- el identificador de la sesión,

- el identificador SNMP del valor a recuperar ("1.3.6.1.2.1.1.1.1.1.1.0" para la RAM, por ejemplo).




<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Identificador
<a name="identificador_ELTTEXTE000229"></a>
El identificador del valor a recuperar se llama OID. Este valor se puede encontrar:

- en INTERNET para los valores estándar.

- en la documentación del aparato. Esta documentación se llama MIB y agrupa todos los valores específicos de su dispositivo.




Observación: [SNMPLoadMIB](../WDLang3/1000017074.md) se utiliza para cargar en memoria la información procedente de una MIB con el fin de utilizar el identificador de cadena del OID en lugar de su identificador numérico.

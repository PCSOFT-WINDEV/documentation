<a name="Use"></a>
<a name="description"></a>
Gestiona los derechos de los clientes en el servidor (servidor FTP (File Transfer Protocol) o RPC (Remote Procedure Call) alimentado por WINDEV).. Las autorizaciones concedidas o denegadas se aplican a todos los clientes que se conectarán al servidor.

**Atención**: **Esta función solo se puede utilizar en el servidor.** Por Default , el servidor permite la ejecución de todas las funciones de red.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
FTPServerID is int = NetStartServer(FTPServer)
...
// Forbids the use of NetExecute and NetEraseFile on the server
NetServerOption(FTPServerID, netOptionExecute, False)
NetServerOption(FTPServerID, netOptionEraseFile, False)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`NetServerOption(<Identifier of Server Connection> , <Action Name> , <Authorization>)`
---

**`<Identifier of Server Connection>: (Integro)`**

Identificador de conexión (devuelto por [NetStartServer](../WDLang3/3056013.md)).

**`<Action Name>: (Constante de tipo Integer)`**

A cada función Net que tenga una acción en el servidor RPC/FTP corresponde una constante WLanguage que puede permitir prohibir la acción correspondiente.. El límite de las acciones que se pueden realizar en el servidor se realiza mediante las siguientes constantes:


|   |   |
| --- | --- |
| NetDirList | *netOptionDirList* |
| NetDiskList | *netOptionDiskList* |
| NetEraseFile | *netOptionEraseFile* |
| NetExecute | *netOptionExecute* |
| NetGetFile | *netOptionGetFile* |
| NetRemoteMessageBox | *netOptionRemoteMessageBox* |
| NetRenameFile | *netOptionRenameFile* |
| NetSendArchivo | *netOptionSendFile* |
| NetSendBuffer | *netOptionSendBuffer* |
| NetSendMensaje | *netOptionSendMessage* |



**`<Authorization>: (booleano)`**



- <u><u><u><u>True</u></u></u></u>: Autoriza la acción especificada. 

- <u><u><u><u>False</u></u></u></u>: No autoriza la acción especificada.

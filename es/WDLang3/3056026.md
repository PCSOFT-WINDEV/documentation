


## NetOpenRemoteAccess( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Establece una conexión remota. Esta función puede utilizarse para manipular los correos electrónicos a través del protocolo POP3 en el servidor remoto.

**Observación**: A continuación, la conexión remota puede ser cerrada por [NetCloseRemoteAccess](../WDLang3/3056021.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Starts the Internet connection on a PC
// The name of the connection to use is the one defined in the list of 
// connections of the PC (control panel).
Report is int = NetOpenRemoteAccess("Connection_LiveBox")
IF Report = 0 THEN
	...
	NetCloseRemoteAccess()
ELSE
	Error("Error during the connection", NetMsgError(Report))
END
```
<a name="Example2"></a>

```wl
// Open the connection on a Smartphone
sConnectionName is string 
nCntReturn is int
// The name of the connection is given by the provider
sConnectionName = "Bouygues Internet"
// Open the connection
nCntReturn = NetOpenRemoteAccess(sConnectionName) 
IF nCntReturn = 0 THEN
	// Requested process
	// ...
	NetCloseRemoteAccess(sConnectionName)
ELSE
	Error("Error during the connection:", NetMsgError(nCntReturn))
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = NetOpenRemoteAccess(<Access name> [, <Password>] [, <Automatic closing>])`
---

**`<Result>: (Integro)`**



- 0 si se ha establecido la conexión,

- un código de error de lo contrario. Este código de error puede utilizarse con [NetMsgError](../WDLang3/3056019.md) para get los detalles del error y el mensaje correspondiente.




**`<Access name>: (Cadena de caracteres)`**

Nombre de la conexión "Acceso remoto a la red" definida en Windows. La lista de accesos remotos instalados en un ordenador es devuelta por [NetListRemoteAcceso remoto](../WDLang3/3056027.md).
Si la conexión ya está habilitada (si fue abierta manualmente o por otra aplicación), se recupera automáticamente.

**`<Password>: (Cadena de caracteres opcional)`**

Contraseña associated con la conexión.

**`<Automatic closing>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> (valor predeterminado): la conexión se cerrará automáticamente al final de la aplicación o durante la llamada a [NetCloseRemoteAccess](../WDLang3/3056021.md) sin parámetro (válido para la última conexión abierta por **NetOpenRemoteAccess**).

- <u><u><u><u>False</u></u></u></u>: la conexión no está cerrada cuando la aplicación está cerrada. Se utiliza, por ejemplo, para crear una aplicación que abra una conexión.. Del mismo modo, [NetCloseRemoteAccess](../WDLang3/3056021.md) sin parámetro no cerrará la conexión. Para cerrar esta conexión, hay que pasar el nombre de acceso como parámetro.

<a name="NOTE0_1"></a>

## Observaciones


### Conexión para el acceso remoto a la red
<a name="conexion_para_acceso_remoto_red_ELTPARAGRAPHE000067"></a>

Se crea una conexión para el acceso remoto a la red en el panel de Windows control. La siguiente información es necesaria para crear la conexión:

- el título de la conexión (utilizado por **NetOpenRemoteAccess**),

- el número de Phone para marcar,

- el nombre de usuario, 

- el módem, el protocolo, etc.




Esta información es proporcionada por el proveedor.

Cuando la conexión se ejecuta a través de un doble clic realizado en el correspondiente Icon, tiene la posibilidad de especificar una contraseña para esta conexión. Esta contraseña puede ser almacenada para ser utilizada cada vez que se inicie esta conexión.
<a name="NOTE0_2"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000085"></a>

- Atención: La conexión a una red remota puede llevar bastante tiempo. La ejecución de la aplicación actual se bloquea durante toda la duración de la conexión.

- Se puede abrir una sola conexión en un momento dado.

- Le recomendamos que compruebe si la conexión funciona correctamente en Windows antes de ejecutar **NetOpenRemoteAccess**.

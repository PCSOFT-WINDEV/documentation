
## AndroidActivityResultProcedure (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite especificar el procedimiento a llamar para obtener el resultado devuelto por una actividad de terceros.

**Observación**: En una aplicación WINDEV Mobile, es posible utilizar bibliotecas y APIs externas. En Android, algunas bibliotecas y APIs abren ventanas específicas, denominadas "Actividades", que realizan procesos determinados: pago con dispositivos móviles, alternar la visibilidad de Bluetooth, inicio de sesión con OAuth, etc.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
AndroidActivityResultProcedure((resultCode, requestCode, Uri, extras)=>{
	IF resultCode = -1 _AND_ requestCode = 1024 THEN
		FOR EACH ELEMENT value OF extras 
			...
		END
	END
})
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`AndroidActivityResultProcedure([<WLanguage procedure>])`
---

**`<WLanguage procedure>: (Nombre de procedimiento opcional)`**

Nombre del procedimiento WLanguage ("callback") al que se llama cuando la aplicación necesita obtener el resultado devuelto por una actividad de terceros. 
Para obtener más información sobre este procedimiento, consulte [Parámetros del procedimiento utilizado por la función AndroidActivityResultProcedure](../WDLang3/1410088882.md).
Si no se especifica este parámetro, se cancelará la llamada al procedimiento para la ventana actual.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Esta función puede utilizarse para obtener el resultado de una actividad iniciada mediante una llamada al método **startActivityForResult** desde una biblioteca externa. 

- La función **AndroidActivityResultProcedure** está relacionada con el ciclo de vida de la ventana desde la que se ejecutó. Al cerrar la ventana, el procedimiento se cancela automáticamente. También puede llamar a la función sin pasar el parámetro &lt;Procedure&gt; para forzar la cancelación.

- Se producirá un error si se llama a la función cuando no se está ejecutando ninguna ventana.


## tapiAnswerCall( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Contesta una llamada entrante que fue detectada.

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) El módem contesta y la llamada puede ser procesada por el módem (a través de un contestador automático, por ejemplo).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
PROCEDURE CallDetection(nServiceID, nCallID, nStatus)
// Picks up the line to answer a call
tapiAnswerCall(nCallID)
// Trigger the answerphone
tapiPlay("answerphone.wav", tapiPlayLock, nCallID)
// If the star key was pressed
// the answerphone is stopped
IF Position(tapiKeyPressed(nCallID), "*") = True THEN
	tapiStop(nCallID)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = tapiAnswerCall([<Call identifier>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se contestó la llamada,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Call identifier>: (Entero opcional)`**

Identificador de la llamada que se va a gestionar. Este identificador se define en el lenguaje WLanguage Procedure llamado por [tapiEscuchar](../WDLang3/3076029.md). La llamada actual se gestiona si no se especifica este parámetro.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Limitación
<a name="limitacion_ELTPARAGRAPHE000049"></a>

No se puede llamar a la función **tapiAnswerCall** en el procedimiento de detección de llamadas (definido con la función [tapiEscuchar](../WDLang3/3076029.md)).
<a name="NOTE0_2"></a>


### Configuración necesaria
<a name="configuracion_necesaria_ELTPARAGRAPHE000188"></a>

Las funciones de telefonía utilizan tecnología TAPI 2.0 y/o tecnología TAPI 3.1. Estas tecnologías pueden utilizarse en todos los sistemas.

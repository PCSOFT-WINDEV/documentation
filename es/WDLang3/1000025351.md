
## WLanguage Procedure llamado por FTPListFile



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Procedimiento ("Callback") llamado por la función [FTPListFile](../WDLang3/3040018.md) por cada archivo encontrado. Este procedimiento puede ser local, global o interno.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Connection to an FTP site
nConnection is int
nConnection = FTPConnect("ftp.cdrom.com")

nFile is int  // Number of files found in the directory
nJPG is int		// Number of .JPG files
nJPG = 0

// List ".JPG" files in the current directory
nFile = FTPListFile(nConnection, "", FTPListFile_Callback, ftpFile, nJPG)
Info("There are" + nJPG + ".JPG files in" + nFile + "listed files") 


// Procedure called by FTPListFile
// For each file found, the FTPListFile_Callback procedure 
// allows counting the number of files with ".JPG" extension.
INTERNAL PROCEDURE FTPListFile_Callback(Name, nSize, sAttribute, sDate, sTime, nParam)
	IF Right(Name, 4) ~= ".JPG" THEN
		nParam++
	END
	RETURN True
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = FTPListFile_Callback(<File> , <Size> , <Attribute> , <Date> , <Time> , <Procedure parameter>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> para seguir navegando por los archivos,

- <u><u><u><u>False</u></u></u></u> para dejar de navegar por los archivos. 




**`<File>: (Cadena de caracteres)`**

Nombre del archivo o directorio utilizado.

**`<Size>: (Integro)`**

Tamaño del archivo encontrado en bytes. Este parámetro debe ser ignorado para un directorio.

**`<Attribute>: (Cadena de caracteres)`**

Atributos del archivo o directorio encontrado. El formato utilizado es idéntico al de [FTPAttribute](../WDLang3/3040006.md).

**`<Date>: (Cadena de caracteres)`**

Fecha de modificación del archivo (o directorio) encontrado. El formato utilizado es "AAAAMMDD".

**`<Time>: (Cadena de caracteres)`**

Tiempo de modificación del archivo (o directorio) encontrado. El formato utilizado es "HHMMSS".

**`<Procedure parameter>: (Variant)`**

valor pasado al &lt;Parámetro&gt; de [FTPListFile](../WDLang3/3040018.md). Si &lt;Parámetro&gt; no se especifica, &lt;parámetro Procedure&gt; es set a 0.






## Observaciones


### Característica de la Procedure
<a name="caracteristica_procedure_ELTPARAGRAPHE000057"></a>

**Atención**: En esta Procedure:

- no se debe llamar a ninguna función FTP que utilice la misma conexión que [FTPListFile](../WDLang3/3040018.md).

- puede recuperar las propiedades (nombre, tamaño, atributos, fechas y horas) de los archivos listados por [FTPName](../WDLang3/3040019.md), [FTPSize](../WDLang3/3040007.md), [FTPAttribute](../WDLang3/3040006.md), [FTPDate](../WDLang3/3040015.md) y [FTPTime](../WDLang3/3040013.md). Sin embargo, esta solución lleva mucho tiempo: requiere una petición adicional al servidor FTP. Por lo tanto, es más fácil leer directamente los parámetros opcionales pasados a la Procedure.



<a name="NOTE0_2"></a>


### Interrupción del archivo FTPListFile
<a name="interrupcion_del_archivo_ftplistfile_ELTPARAGRAPHE000085"></a>

[FTPListFile](../WDLang3/3040018.md) no se puede interrumpir mientras haya archivos que recorrer. Para forzar la interrupción de la navegación, use la siguiente línea en la Procedure:


```wl
RESULT False
```


En este caso, la función [FTPListFile](../WDLang3/3040018.md) devuelve el número de archivos encontrados hasta que se llame a "RESULT False".
En los demás casos, (para continuar la navegación), el procedimiento DEBE devolver <u><u><u><u>True</u></u></u></u>.

Si el procedimiento no devuelve ningún valor (ni <u><u><u><u>True</u></u></u></u> ni <u><u><u><u>False</u></u></u></u>), se genera un error.

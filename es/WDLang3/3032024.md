


## EmailNbMessage (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el número de mensajes entrantes que se encuentran actualmente:

- en el servidor Email (protocolo POP3 e IMAP). 

	- Protocolo POP3: Este número de mensajes se mantiene constante durante toda la sesión de Email, aunque se reciban nuevos Email o se borren Email por [EmailDeleteMessage](../WDLang3/3032027.md).

	- protocolo IMAP: Este número de mensajes evoluciona en función de los Email s entrantes o de los Email s efectivamente borrados del servidor Email (a través de un comando "expunge", borrar la papelera de un WebMail por ejemplo). También tiene la posibilidad de forzar la destrucción de todos los correos electrónicos eliminados a través de [EmailExpunge](../WDLang3/1000021553.md).




- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en el software de mensajería Outlook. Este número de mensajes evoluciona en función de los correos electrónicos recibidos o eliminados por [EmailDeleteMessage](../WDLang3/3032027.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en el software de mensajería Notes. Este número de mensajes evoluciona en función de los correos electrónicos recibidos o eliminados por [EmailDeleteMessage](../WDLang3/3032027.md).

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003032024&name=emailnbmessage_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Loop for reading all the incoming messages found
// in the mailbox of POP3 protocol
I is int
nbMSG is int = EmailNbMessage(UserName)
FOR I = 1 TO nbMSG
	EmailReadMessage(UserName)
	IF Email.Out <> True THEN
		CU.NAME = Email.Sender
		CU.MESSAGE_TXT = Email.Message
		...
	ELSE
		Info("This message was deleted")
	END
END
```

## Sintaxis

### Averiguar el número de Email s encontrados en el servidor Email (protocolo POP3)

`<Result> = EmailNbMessage(<POP3 session>)`
---

**`<Result>: (Integro)`**



- Número de mensajes recibidos en el servidor al inicio de la sesión actual de Email,

- -1 si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<POP3 session>: (Cadena de caracteres)`**

Identifica la sesión de usuario (POP3). Este parámetro puede corresponder a:

- el nombre de usuario que se utilizó para iniciar la sesión de email ([EmailStartSession](../WDLang3/3032028.md)).  

- una variable de tipo [emailPOP3Session](../WDLang3/1000018759.md).

## Observaciones


### Mensajes eliminados (protocolo POP3)
<a name="mensajes_eliminados_protocolo_pop3_ELTPARAGRAPHE000272"></a>

Si un mensaje fue borrado por [EmailDeleteMessage](../WDLang3/3032027.md):

- **EmailNbMessage** devuelve el número de mensajes en el servidor email antes de ser borrados. 

- los números de mensaje no se reasignan.




Por lo tanto, si se encuentran cuatro mensajes en el servidor al principio de la sesión, estos mensajes están numerados del 1 al 4.. Si se borra el mensaje nº 1:

- Al utilizar la función **EmailNbMessage** en la misma sesión, siempre devuelve 4.

- El mensaje 1 no puede ser leído por [EmailReadMessage](../WDLang3/3032016.md) porque el mensaje ya no existe (sólo existen los mensajes 2, 3 y 4).




Los números de mensaje se reasignan durante la siguiente conexión.
<a name="NOTE0_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 

### Mensajes eliminados (mensajería de Outlook)
<a name="mensajes_eliminados_mensajeria_outlook_ELTPARAGRAPHE000305"></a>

Si un mensaje fue borrado por [EmailDeleteMessage](../WDLang3/3032027.md) en la misma sesión de correo electrónico, **EmailNbMessage** devuelve el número real de mensajes en la carpeta de entrada.
<a name="NOTE0_4"></a>


### Mensajes eliminados (protocolo IMAP)
<a name="mensajes_eliminados_protocolo_imap_ELTPARAGRAPHE000319"></a>

Si un mensaje fue borrado por [EmailDeleteMessage](../WDLang3/3032027.md), este mensaje es realmente borrado de la sesión: 

- durante el siguiente comando "borrar" (por ejemplo, borrar la papelera de un WebMail).

- al utilizar [EmailExpunge](../WDLang3/1000021553.md) para forzar la destrucción de todos los correos electrónicos eliminados.



<a name="NOTE0_5"></a>


### Nuevos mensajes (protocolo POP3)
<a name="nuevos_mensajes_protocolo_pop3_ELTPARAGRAPHE000339"></a>

El protocolo POP3 no se puede utilizar para descubrir y gestionar los nuevos mensajes recibidos durante la conexión.. Para get los nuevos mensajes, debe desconectarse ([EmailCloseSession](../WDLang3/3032006.md)) y volver a conectarse ([EmailStartSession](../WDLang3/3032028.md)).

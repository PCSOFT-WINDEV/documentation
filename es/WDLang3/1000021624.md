


## &lt;httpRequest Variable&gt;.Send (Function)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Envía una solicitud HTTP y espera la respuesta del servidor HTTP.. 


<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Sintaxis con una httpRequest Variable

`<HTTP response> = <HTTP request>.Sends()`
---

**`<HTTP response>: (httpResponse Variable)`**

Nombre de la [httpResponse](../WDLang3/1000021165.md) Variable que contiene la respuesta recibida del servidor HTTP.

**`<HTTP request>: (Variable de tipo httpRequest)`**

Nombre de la [httpRequest](../WDLang3/1000021158.md) Variable que contiene las características de la petición HTTP que se enviará al servidor.


<a name="SYNTAX3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
### Sintaxis asíncrona con una httpRequest Variable

`Sends(<HTTP request> , <WLanguage procedure>)`
---

**`<HTTP request>: (Variable de tipo httpRequest)`**

Nombre de la [httpRequest](../WDLang3/1000021158.md) Variable que contiene las características de la petición HTTP que se enviará al servidor.

**`<WLanguage procedure>: (Nombre de Procedure)`**

Nombre del lenguaje WL llamado Procedure durante la respuesta del servidor HTTP. Este procedimiento tiene el siguiente formato: 

```txt
PROCEDURE <Procedure name>(<Response> is httpResponse)
```
donde &lt;Respuesta&gt; es una variable de tipo [httpResponse](../WDLang3/1000021165.md) que contiene la respuesta recibida del servidor HTTP.

## Observaciones


### Notas
<a name="notas_ELTPARAGRAPHE000201"></a>

- Para conocer los errores de ejecución de esta función, utilice [ErrorOccurred](../WDLang1/3087001.md) y [ErrorInfo](../WDLang1/3013008.md).

- Si se han implementado cookies para la solicitud especificada ([&lt;httpRequest Variable&gt;.ManageCookie](../WDLang3/1000021608.md)), se actualizan en el [httpRequest](../WDLang3/1000021158.md) Variable después de la llamada a **&lt;httpRequest Variable&gt;.Send**. Así, la próxima vez que se llame a la función **&lt;httpRequest Variable&gt;.Send** con esta solicitud, se tendrán en cuenta las cookies. 

- A partir de la versión 21, las redirecciones se tienen en cuenta automáticamente en las peticiones HTTP.. Para procesar manualmente las redirecciones en las peticiones HTTP, debe utilizar la propiedad **IgnoreError** de la variable [httpRequest](../WDLang3/1000021158.md) con la constante *httpIgnoreRedirection*. 



<a name="NOTE0_2"></a>


### Error de seguridad en una transacción de secure
<a name="error_seguridad_una_transaccion_secure_ELTPARAGRAPHE000243"></a>

Durante una transacción secure, la solicitud puede fallar debido a errores de seguridad:

- Certificate o Certificate inválidos que vienen de una compañía desconocida.

- el nombre del sitio especificado en la Certificate no corresponde a un servidor.

- fecha inválida o caducada de Certificate.

- redirección a un servidor no secure.




Estos errores son devueltos por [ErrorInfo](../WDLang1/3013008.md).

Si se produce uno de estos errores, puede volver a ejecutar la orden ignorando los errores.. Para ello, basta con modificar el **IgnoreError** Property del [httpRequest](../WDLang3/1000021158.md) Variable:

| Error devuelto por [ErrorInfo](../WDLang1/3013008.md)<br> (con la constante *errCode*) | Valor del **IgnorarError** Property del [httpRequest](../WDLang3/1000021158.md) Variable<br>(estos valores se pueden combinar) | description |
| --- | --- | --- |
| *httpErrorInvalidCertificate*<br>Certificate o Certificate inválidos que vienen de una compañía desconocida | *httpIgnoreInvalidCertificate* | El Certificate es ignorado. |
| *httpErrorInvalidCertificateName*<br>El nombre del sitio especificado en la Certificate no corresponde a un servidor | *httpIgnoreInvalidCertificateName* | El nombre del sitio especificado en la Certificate es ignorado. |
| *httpErrorExpiredCertificate*<br>Inválido o caducado Certificate fecha | *httpIgnoreExpiredCertificate* | La fecha de Certificate es ignorada |
| *httpErrorRedirectToHTTP*<br>Redirección a un servidor no secure | *httpIgnoreRedirectToHTTP* | Se permite la redirección a un servidor no secure. |
| *httpIgnoreRedirectToHTTPS*<br>Redirección a un servidor secure | *httpIgnoreRedirectToHTTPS* | Se permite la redirección a un servidor secure. |
|   | *httpIgnoreRevocation* | El Certificate que se encuentra en la lista de Certificate s revocados no se comprueba. |

Observaciones: 

- Cuando las consultas HTTP se ejecutan en varios Thread s, el **HTTP.IgnoreError** Variable tiene un valor específico para cada Thread.


## StreamInfo( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el nombre y/o número del emisor del streaming. Esto le permite a get información sobre la conexión entrante antes de aceptarla.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Retrieves the name of stream issuer
CallerName is string = StreamInfo(StreamID)
```
<a name="Example2"></a>

```wl
// Retrieves the number of the stream issuer
CallerNumber is string = StreamInfo(StreamID, StreamCallerNumber)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = StreamInfo(<Connection identifier> [, <Requested information>])`
---

**`<Result>: (Cadena de caracteres)`**

Nombre o número del emisor del stream.

**`<Connection identifier>: (Integro)`**

Identificador de conexión (se encuentra en la "Solicitud de conexión de la nueva corriente" Event).

**`<Requested information>: (Constante opcional de tipo Integer)`**

Información a devolver:


|   |   |
| --- | --- |
| *StreamCallerName*<br>(valor predeterminado) | Nombre del emisor del stream.<br>En WLanguage, este nombre puede ser devuelto por la función [Identificador](../Proprietes/2510036.md) Property. |
| *StreamCallerNumber* | Número del emisor del flujo (IP Address por ejemplo). |

<a name="NOTE0_1"></a>

## Observaciones


### Procedure para la detección de corrientes
<a name="procedure_para_deteccion_corrientes_ELTPARAGRAPHE000174"></a>

Un flujo está asociado a la Event "Solicitud de conexión de un nuevo flujo". Este Event se ejecuta cuando se detecta una solicitud de conexión a un flujo entrante.

Este Event se ejecuta en uno de los siguientes casos: 

- si el valor de la propiedad [Detección](../Proprietes/2510031.md) utilizada en el flujo es <u><u><u><u>True</u></u></u></u>.

- si la opción "Detección de corrientes entrantes activas" ha sido marcada en la ventana de la corriente description.




**Observación**: Este Event se ejecuta en un thread específico de WLanguage. Por lo tanto, está sujeto a limitaciones específicas de los hilos de rosca. Para más detalles sobre los hilos, véase [Gestión de hilos con WINDEV](../WDLang1/3077026.md).

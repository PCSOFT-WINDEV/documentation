
## WiFiActivate( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Habilita o deshabilita Wi-Fi en el dispositivo.

Observaciones: 

- Para conocer el estado de activación del Wi-Fi en el dispositivo o recibir una notificación cuando el estado cambie, utilice [WiFiStatus](../WDLang3/1000019354.md).

- El acceso Wi-Fi no está disponible en el emulador de Android.







<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF Val(SysAndroidVersion(sysVersionNumber))<10 THEN	  
	IF WiFiStatus() = wifiDisabled THEN
		WiFiActivate(True)
	END
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = WiFiActivate([<Wi-Fi activation>])`
---

**`<Result>: (Constante de tipo Integer)`**

Estado de activación de Wi-Fi después de la modificación:


|   |   |
| --- | --- |
| *wifiActivationInProgress* | El Wi-Fi está siendo habilitado en el dispositivo. |
| *wifiDeactivationInProgress* | El Wi-Fi está siendo desactivado en el dispositivo. |
| *wifiDisabled* | El Wi-Fi está desactivado en el dispositivo. |
| *wifiEnabled* | El Wi-Fi está habilitado en el dispositivo. |
| *wifiError* | No se puede recuperar el estado de activación de Wi-Fi. |



**`<Wi-Fi activation>: (Booleano opcional)`**

Nuevo estado de activación de Wi-Fi: 

- <u><u><u><u>True</u></u></u></u> (valor predeterminado): Habilita el Wi-Fi en el dispositivo. 

- <u><u><u><u>False</u></u></u></u>: Desactiva el Wi-Fi del dispositivo.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Permisos necesarios
<a name="permisos_necesarios_ELTPARAGRAPHE000089"></a>

La llamada a esta función modifica los permisos requeridos por la aplicación.

Permisos necesarios :  

- ACCESS_WIFI_STATE: esta Permission permite a las aplicaciones acceder a la información relativa a las redes Wi-Fi.

- CAMBIAR_WIFI_STATE: esta Permission permite a las aplicaciones modificar el estado de la conectividad Wi-Fi.

- WAKE_LOCK: esta Permission permite utilizar los WakeLocks de PowerManager para evitar que el procesador pase al modo de espera o para evitar que la pantalla se Grayed.

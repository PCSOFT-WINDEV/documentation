
## BackgroundTaskList( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve la lista de tareas de Background para la aplicación.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Returns True if the background task for processing emails was added 
PROCEDURE bTaskEmailProcessActive
sTaskList is string = BackgroundTaskList()
FOR EACH STRING sTask OF sTaskList SEPARATED BY CR
	IF ExtractString(sTask, 2, TAB) = "ProcProcessEmailInBackground" THEN
		RETURN True
	END
END
RESULT False
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = BackgroundTaskList()`
---

**`<Result>: (Cadena de caracteres)`**



- Lista de tareas de Background para la aplicación separadas por caracteres CR (Carriage Return). Esta lista tiene el siguiente formato:
	
	```txt
	<Identifier> + TAB + <Procedure> + TAB + <Frequency> + TAB + <Auto> + CR +  ...
	```
 donde: 

	- &lt;Identifier&gt; es un número entero que corresponde al identificador de la tarea,

	- &lt;Procedure&gt; es una cadena de caracteres que corresponde al nombre del WLanguage Procedure llamado cuando se ejecuta la tarea,

	- &lt;Frecuencia&gt; es un número entero correspondiente a la frecuencia de ejecución de la tarea (en minutos),

	- &lt;Auto&gt; es un booleano: 

		- <u><u><u><u>True</u></u></u></u> si la tarea se agregó automáticamente a través de la configuración de automatización del procedimiento (no disponible en esta versión), 

		- <u><u><u><u>False</u></u></u></u> si fue añadido por [BackgroundTaskAdd](../WDLang3/1000023454.md).




- Vaciar la cadena ("") si no se definió ninguna tarea de Background para la aplicación.

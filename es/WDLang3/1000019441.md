


## Notification (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **Notification** permite manipular

- **las notificaciones push**.




Las características de esta Notification pueden definirse y modificarse mediante diferentes propiedades de WLanguage. 

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Simple notification
// --------------------
notif is Notification
notif.Title = "New incoming message"
notif.Message = "Click here to display the message"
NotifAdd(notif)
```

## Observaciones


### Utilizando el tipo Notification
<a name="utilizando_tipo_notification_ELTPARAGRAPHE000058"></a>

El tipo **Notification** se utiliza para gestionar notificaciones locales y notificaciones push.

Los diferentes elementos de este Notification pueden ser configurados por el tipo Notification: 

- **1.** Icono grande de Notification (**LargeIcon** Property).

- **2.** Notification título (**Título** Property).

### Propiedades específicas de las variables Notification para un push Notification
<a name="propiedades_especificas_las_variables_notification_para_push_notification_ELTPARAGRAPHE000642"></a>

Las siguientes propiedades pueden ser utilizadas para manipular a Notification:

| Nombre de la propiedad | Tipo utilizado | Efecto |
| --- | --- | --- |
| ActionLabel | Cadena de caracteres | Pie de foto para el botón de acción de la Notification. <br><br> |
| ActivateApplication | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si la aplicación debe mostrarse en primer plano al hacer clic en la notificación. <br><br>- <u><u><u><u>False</u></u></u></u> (valor predeterminado) en caso contrario.<br><br><br> |
| Categoría | [notificationCategory](../WDLang3/1000023981.md) | Categoría asociada a la Notification. Las categorías de Notification están disponibles a partir de Android 8.0 (nivel 26 de API). Si la aplicación se ejecuta con una versión anterior, se ignorará la propiedad **Category**.<br>Se recomienda asociar siempre una categoría con un Notification<br><br>- Si la categoría no existe al añadir la Notification, se creará automáticamente,<br><br>- Si la categoría ya existe para la solicitud, el Notification será associated con ella. <br><br>- Si no se ha especificado ninguna categoría, el Notification será automáticamente associated a una categoría preestablecida de acuerdo con su nivel de prioridad. Estas son las categorías preestablecidas: <br><br>	- Prioridad mín.<br><br>	- Prioridad baja<br><br>	- Prioridad normal<br><br>	- Prioridad alta <br><br>	- Prioridad máx.<br><br><br> La categoría preestablecida se creará cuando se añada por primera vez un Notification sin categoría con el nivel de prioridad correspondiente, y utilizará los parámetros visuales y de audio de este Notification (vibración, sonido, indicador luminoso, etc.). Estos parámetros pueden ser modificados posteriormente por el usuario final. Si la categoría predefinida ya existe, la Notification heredará sus propiedades.<br><br><br><br><br> |
| ColorLED | Integro | El color del LED cuando se muestra el Notification. <br><br>El valor de esta propiedad se tiene en cuenta solo si el valor de la propiedad **DisplayLED** es <u><u><u><u>True</u></u></u></u>. <br><br>La indicación color del LED es verde por Default. <br><br>Este color puede corresponder a:<br><br>- un color RGB (devuelto por la función [RGB](../WDLang1/3029012.md)),<br><br>- una Component HSL de la color (devuelta por [HSL](../WDLang1/3029057.md)),<br><br>- [un color predefinido de WLanguage](../WDLang5/3010002.md).**Atención**: Algunos aparatos no permiten modificar el LED color o no proponen todos los color s. Se utilizará un Color por defecto si el Color especificado no es compatible con el dispositivo.<br><br><br><br><br> |
| Contenido | Cadena de caracteres | Datos personalizados de Notification. Estos datos no se muestran en la Notification. |
| Cronómetro | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> para mostrar un cronómetro en lugar de la hora de la notificación. El cronómetro se iniciará automáticamente al recibir la Notification y su pantalla se refrescará automáticamente. <br><br>- <u><u><u><u>False</u></u></u></u> (valor predeterminado) para no mostrar un cronómetro. Se mostrará la hora asociada con el Notification. <br><br><br><br><br>**Atención**: La visualización de un cronómetro en las notificaciones está disponible desde Android 4.1 (Jelly Bean, API nivel 16). En los dispositivos que ejecutan una versión anterior, se ignorará el cronómetro y se mostrará el tiempo asociado al Notification.<br><br> |
| Deletable | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> (valor predeterminado) si el usuario puede eliminar la notificación,<br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario. <br><br><br>Si el valor de esta propiedad es <u><u><u><u>False</u></u></u></u>, la notificación se borrará en los siguientes casos: <br><br>- La solicitud está cerrada. <br><br>- La función [NotifDelete](../WDLang3/1000019444.md) se llama. <br><br>- Si el lenguaje WL Procedure ejecutado en el clic (**ActionClick** Property) devuelve <u><u><u><u>True</u></u></u></u> o no devuelve ningún valor.<br><br><br> |
| DisplayLED | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si el LED del dispositivo debe estar encendido cuando se muestra la notificación, <br><br>- <u><u><u><u>False</u></u></u></u> (valor predeterminado) en caso contrario. <br><br><br>La color del LED y la frecuencia de encendido pueden cambiar de un aparato a otro.<br><br><br><br> |
| DropDownText | Cadena de caracteres | Texto desplegable que se muestra brevemente en la barra de sistema cuando se muestra la Notification. <br><br>No se mostrará ningún mensaje si este Property corresponde a una cadena vacía ("", valor de Default).<br><br> |
| envergadura | [notificationFormat](../WDLang3/1000021301.md) | El formato de visualización del Notification cuando éste se expande. <br><br>Atención: La visualización de las notificaciones en modo "ampliado" está disponible a partir de Android 4.1 (Jelly Bean, API nivel 16). El formato de visualización se ignorará en los dispositivos que ejecutan una versión anterior.. <br><br> |
| Grupo | Cadena de caracteres | Nombre del grupo al que pertenece el push Notification es associated. <br><br>Si se envían varios Notification de empuje pertenecientes al mismo grupo con un contenido idéntico mientras no se puede alcanzar el dispositivo de destino, sólo se enviará el último Notification del grupo cuando el dispositivo pueda ser contactado de nuevo por el servidor de Google Cloud Messaging. <br><br> |
| Icon | Cadena de caracteres | La trayectoria de la imagen correspondiente al icono asociado a la Notification. <br><br>El Icon de la aplicación se utilizará si no se especifica ningún Icon. |
| Insignia | Integro | El número que muestra la Icon de la aplicación. <br><br> |
| LargeIcon | Cadena de caracteres | Ruta de la imagen correspondiente al gran icono asociado a la Notification. Este icono aparecerá a la izquierda del título y del mensaje de Notification. <br><br>Si no se especifica ninguna imagen, se utilizará el icono especificado por **Icon** Property. Si no se especifica esta propiedad, se utilizará el ícono de la aplicación.<br><br>Atención: La visualización de esta Image en las notificaciones está disponible para los dispositivos que ejecutan Android 3.0 (Honeycomb, API nivel 11) o posterior. Este Image será ignorado en los dispositivos que ejecuten una versión anterior. <br><br> |
| Local | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si la notificación es local al dispositivo que la recibe: esta Notification no debe ser transmitida a los objetos que operan en Android Wear conectados al dispositivo (un reloj por ejemplo). <br><br>- <u><u><u><u>False</u></u></u></u> (valor predeterminado) si la notificación debe transmitirse automáticamente a los objetos conectados. <br><br><br><br><br> |
| Mensaje | Cadena de caracteres | Mensaje de la Notification. |
| Prioridad | Constante de tipo Integer | Prioridad de Notification. <br><br>El cambio de prioridad de un Notification se utiliza para modificar su posición en la lista de Notification s del dispositivo para que el usuario vea primero los Notification s más importantes. Un Notification con una prioridad alta aparecerá al principio de la lista mientras que un Notification con una prioridad baja aparecerá al final de la lista. <br><br>En algunas situaciones, una notificación de baja prioridad puede ocultarse al usuario si se reciben otras notificaciones de mayor prioridad.<br><br>Esta Property puede tomar los siguientes valores : <br><br>- *notifMaxPriority*: máxima prioridad (ejemplo: llamada entrante). <br><br>- *notifHighPriority*: alta prioridad (ejemplo: recibiendo un Email urgente). <br><br>- *notifDefaultPriority* (valor predeterminado): prioridad estándar (ejemplo : recibir un SMS). <br><br>- *notifLowPriority*: baja prioridad (ejemplo : Notification para la actualización).<br><br>- *notifMinPriority*: prioridad mínima (ejemplo : información comercial).<br><br><br><br><br>La modificación de las prioridades de las notificaciones está disponible desde Android 4.1 (Jelly Bean, API nivel 16). La prioridad se ignorará en los dispositivos que ejecuten una versión anterior.: todas las notificaciones tienen la misma prioridad.<br><br><br><br> |
| SecondaryMessage | Cadena de caracteres | Mensaje secundario mostrado debajo del mensaje principal de la Notification.<br><br>**Atención**: La visualización de un mensaje secundario en las notificaciones está disponible desde Android 4.1 (Jelly Bean, API nivel 16). El mensaje secundario se ignorará en los dispositivos que ejecuten una versión anterior.<br><br> |
| Sonido | Cadena de caracteres | Ruta del archivo de sonido a reproducir cuando se muestra la Notification. <br><br>Para reproducir el sonido predeterminado de las notificaciones, utilice la constante *notifDefaultSound*. Este sonido Default puede cambiar de un dispositivo a otro. <br><br>Tiene la posibilidad de reproducir un sonido emitido por un archivo de audio incluido en la aplicación (a través de la generación asistente): para ello, especifique el nombre del archivo y su extension.<br><br>Para no asociar ningún sonido con la Notification, usa una cuerda vacía ("", valor de Default).<br><br> |
| Título | Cadena de caracteres | Título de la notificación. <br><br>El nombre de la aplicación se utilizará si no se especifica ningún título.<br><br> |
| Vibración | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si el dispositivo debe vibrar cuando se muestra la notificación,<br><br>- <u><u><u><u>False</u></u></u></u> (valor predeterminado) en caso contrario. <br><br><br>Observación: La duración y la potencia de las vibraciones pueden cambiar de un dispositivo a otro.<br><br><br><br> |

### Funciones que utilizan la clase Notification
<a name="funciones_que_utilizan_clase_notification_ELTPARAGRAPHE001024"></a>Las siguientes funciones manipulan variables de tipo **Notification** para una notificación local: 



|   |   |
| --- | --- |
| [NotifAdd](../WDLang3/1000019442.md) | Añade una Notification en la barra de sistema del dispositivo (Android) o en el centro Notification (iOS). |
| [NotifDelete](../WDLang3/1000019444.md) | Borra una Notification que aparece en la barra de sistema del dispositivo. |
| [NotifModify](../WDLang3/1000019443.md) | Modifica una Notification que aparece actualmente en la barra de sistema del dispositivo. |
| [NotifPushSend](../WDLang3/1000020819.md) | Envía un push Notification a un dispositivo móvil (iOS o Android). |
| [ThreadPersistent](../WDLang1/1000020584.md) | Hace que un Thread sea persistente. |

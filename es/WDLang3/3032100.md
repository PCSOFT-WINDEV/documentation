
## LDAPConnect( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Se utiliza para conectarse a un servidor LDAP a través de las características definidas en el [Estructura de LDAPSession](../WDLang3/3032101.md). La conexión se establece realmente durante el primer acceso ([LDAPEncontrar](../WDLang3/3032108.md) por ejemplo).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Check the "Login/Password" couple

// Initialization
LDAPReset()
LDAPSession.Host = "domain.local"
LDAPSession.Port = 389
LDAPSession.Version = 3
LDAPSession.User = "user@domain.local"
LDAPSession.Password = "pwd"

// Connection
LDAPConnect("MySession")

// Find information about the user.
sDn is string = LDAPFind("MySession", ...
		"cn=users,dc=domain,dc=local","sAMAccountName=tester")

// Other possibility
// sDn is string = LDAPFind("MySession", ...
// "cn=users,dc=domain,dc=local","UserMainName=tester@domain.local")


IF sDn = "" THEN
	// Unable to list the connection DN
	// The login and the password do not match 
	Error("The password of the user " + LDAPSession.User + " is invalid.") 
END

// Disconnection 
LDAPDisconnect("MySession")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = LDAPConnect(<LDAP session>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se establece la conexión,

- <u><u><u><u>False</u></u></u></u> en caso contrario. [ErrorInfo](../WDLang1/3013008.md) devuelve más detalles sobre el error. 


La conexión no se establece si el nombre de la sesión ya existe.

**`<LDAP session>: (Cadena de caracteres)`**

Nombre de la sesión LDAP. Este nombre debe ser único. Este nombre se utilizará para identificar la sesión en todas las funciones LDAP..

<a name="NOTE0_1"></a>

## Observaciones




### Estructura de LDAPSession
<a name="estructura_ldapsession_ELTPARAGRAPHE000052"></a>

La estructura [Sesión LDAP](../WDLang3/3032101.md) se utiliza para especificar las características de la sesión LDAP a utilizar (servidor Address, puerto, etc.). Se debe inicializar esta estructura antes de utilizar la función **LDAPConnect**.

Esta estructura también se utiliza para especificar diferentes opciones como el modo de gestión de SSL o el modo de conexión.. 

**Observación**: Se recomienda reiniciar el [Estructura de LDAPSession](../WDLang3/3032101.md) con [LDAPReset](../WDLang3/3032103.md).




### LDAP y el software de grupo de usuarios
<a name="ldap_software_grupo_usuarios_ELTPARAGRAPHE000073"></a>

Su proyecto puede conectarse automáticamente a un directorio LDAP a través de [Groupware Usuario](../Editeurs/2018001.md).

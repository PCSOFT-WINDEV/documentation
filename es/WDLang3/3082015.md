## Presentación
<a name="presentacion_ELTTEXTE000191"></a>
Hay dos tipos de archivos disponibles:

- archivo de una sola parte.

- archivo de varias partes.




![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Sólo están disponibles los archivos de una sola parte.

<a name="NOTE2_1"></a>


## Archivo de una sola parte
<a name="archivo_una_sola_parte_ELTTEXTE000215"></a>
Un archivo de una sola parte aparece como un único archivo cuyo tipo es ".WDZ", ".ZIP", ".CAB" o ".7z".: un fichero de archivo agrupa todos los ficheros comprimidos.

El archivo ocupa el espacio necesario para contener todos los archivos comprimidos.

**Ejemplo**: Su programa debe guardar varios archivos grandes. Su programa puede crear un archivo que contenga todos los archivos para guardar. Este archivo aumentará el espacio disponible en el disco.

## Archivo de varias partes
<a name="archivo_varias_partes_ELTTEXTE000239"></a>
![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) 

### Presentación
<a name="presentacion_ELTPARAGRAPHE000046"></a>

Un archivo multiarchivo contiene varios archivos (o subarchivos) de tamaño idéntico. El tamaño de cada subarchivo se define al crear el archivo multiarchivo.

El primer subarchivo es un archivo cuyo tipo es ".WDZ", ".ZIP" o "7z".. Los siguientes subarchivos son archivos numerados del 1 al N a través de su extensión.

Para los archivos en formato WINDEV (WDZ), los subarchivos tienen las siguientes extensiones: ".001", ".002", ..., ".NNN".

Para los archivos comprimidos en formato ZIP, los subarchivos tienen las siguientes extensiones: ".Z01", ".Z02", ...., ".ZNN".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=achive_mono_multi3.gif)


**Observación**: Los archivos en formato".CAB" son necesariamente archivos de una sola parte.
<a name="NOTE3_2"></a>
![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) 

### Principio
<a name="principio_ELTPARAGRAPHE000065"></a>

Para crear un archivo de varias partes:

1. Cree un archivo ([zipCreate](../WDLang3/3082003.md)).

2. Agregar archivos a este archivo. Los archivos añadidos se comprimen automáticamente ([zipAddFile](../WDLang3/3082008.md), [zipAddDirectory](../WDLang3/3082014.md)).

3. Dividir este archivo en varios subarchivos ([zipSplit](../WDLang3/3082020.md)). Los subarchivos tienen el mismo tamaño. Se define el tamaño de los subarchivos.




Una vez creado el archivo multiarchivo, los subarchivos se pueden almacenar en diferentes soportes (disquete, CD,...).

Los subarchivos se pueden fusionar ([zipMerge](../WDLang3/3082046.md)): el archivo se convierte en un archivo de una sola parte.

**Observación**: Todas estas operaciones también se pueden realizar a través de [WDZip](../WDZip/3527005.md).

### Atención: Archivos de una y varias partes
<a name="atencion_archivos_una_varias_partes_ELTPARAGRAPHE000102"></a>

Algunas funciones de archivo sólo se pueden utilizar en los archivos comprimidos de una sola parte (agregar archivos, eliminar archivos, crear un ejecutable autoextraíble, ....). Para realizar estas operaciones en un archivo de varias partes, todo lo que tienes que hacer es fusionar los subarchivos ([zipMerge](../WDLang3/3082046.md)) con get un archivo de una sola parte.

Desde un archivo de varias partes, sólo tiene la posibilidad de hacerlo:

- Fusionar las diferentes partes ([zipMerge](../WDLang3/3082046.md)),

- Obtenga información sobre el archivo y sobre los ficheros encontrados en el archivo ([zipExtractPath](../WDLang3/3082023.md), [zipInfoFile](../WDLang3/3082040.md), [zipListFile](../WDLang3/3082031.md), ....),

- Extraer los archivos ([zipExtractFile](../WDLang3/3082019.md), [zipExtractAll](../WDLang3/3082011.md)).

### Almacenamiento en disquetes
<a name="almacenamiento_disquetes_ELTPARAGRAPHE000140"></a>

Su programa debe almacenar una gran cantidad de datos en disquetes.. Su programa puede crear un archivo que contenga los datos solicitados.. Una vez creado, este archivo se puede dividir en varias partes. Estos volúmenes tienen el tamaño de un disquete (1.44 MB). Todo lo que tienes que hacer es copy estas diferentes partes en los disquetes.

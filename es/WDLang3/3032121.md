


## EmailStartNotesSession( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite acceder a los datos manejados por la mensajería de Lotus Notes (correos electrónicos, contactos, grupos de contactos, tareas, citas).

Esta función puede ser:

- se llama directamente para establecer la conexión con el servidor de Domino y para especificar la base de datos de Lotus Notes que se va a utilizar.

- llamado después de [NotasOpenConnection](../WDLang3/1000005102.md). En este caso, esta función sólo se utiliza para especificar la base de datos de Lotus Notes que se va a utilizar.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Open the connection
// and specify the Lotus Notes database to use 
IF EmailStartNotesSession("Password", "MarsServer", ...
		"email/julia.nsf", "C:\Lotus Notes\julia.id") <> 0 THEN
	Info("Session started and database specified")
ELSE
	Error()
END
```
<a name="Example2"></a>

```wl
// Open the connection
SessionID is int 
SessionID = NotesOpenConnection("Password", "MarsServer", ...
		"C:\Lotus Notes\julia.id")
// Specify the Lotus Notes database to use
IF EmailStartNotesSession(SessionID, "email/julia.nsf") <> 0 THEN
	Info("Connection established and database specified")
ELSE
	Error()
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Estableciendo una conexión con un servidor Domino y especificando la base de datos de Lotus Notes a utilizar

`<Result> = EmailStartNotesSession(<Password> , <Domino server> [, <Lotus Notes database> [, <Certificate path>]])`
---

**`<Result>: (Integro)`**



- Identificador de sesión.

- 0 (o <u><u><u><u>False</u></u></u></u>) si se produce un error. La variable [ErrorOccurred](../WDLang1/3087001.md) se establece en <u><u><u><u>True</u></u></u></u>. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Password>: (Cadena de caracteres)`**

Contraseña de usuario. Esta contraseña es proporcionada por el proveedor de servicios o por el administrador de la red.

**`<Domino server>: (Cadena de caracteres)`**



- Nombre del servidor Domino,

- Vaciar la cadena (""") si la base de datos es una base de datos local.




**`<Lotus Notes database>: (Cadena de caracteres opcional)`**

Nombre de la base de datos de Lotus Notes que contiene los datos del usuario. Este nombre corresponde a la ruta del archivo ".NSF".

Si la base de datos de Lotus Notes se encuentra en un servidor de Domino, se debe utilizar un ruta relativo.

Si no se especifica este parámetro, el nombre de la base de datos de Lotus Notes se leerá en el archivo de configuración del cliente de Lotus Notes (archivo"Notes.INI").. La base de datos utilizada corresponderá a la base de datos del usuario del cliente de Lotus Notes para el ordenador actual.

**`<Certificate path>: (Cadena de caracteres opcional)`**

Ruta de Certificate utilizada para autentificar al usuario (archivo "ID"). Este archivo lo proporciona el administrador del servidor de Domino. Este archivo es necesario para establecer la conexión con el servidor.

Si no se especifica este parámetro, la ruta de la Certificate se leerá en el archivo de configuración del cliente de Lotus Notes (archivo "Notes.INI"). El Certificate utilizado corresponderá al Certificate del usuario del cliente de Lotus Notes para el ordenador actual.  


<a name="SYNTAX2"></a>

### Especificación de la base de datos de Lotus Notes que se utilizará

`<Result> = EmailStartNotesSession(<Session identifier> [, <Lotus Notes database>])`
---

**`<Result>: (Integro)`**



- Identificador de sesión.

- 0 (o <u><u><u><u>False</u></u></u></u>) si se produce un error. La variable [ErrorOccurred](../WDLang1/3087001.md) se establece en <u><u><u><u>True</u></u></u></u>. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Session identifier>: (Cadena de caracteres)`**

Identificador de la sesión a iniciar. Este identificador es devuelto por [NotasOpenConnection](../WDLang3/1000005102.md).

**`<Lotus Notes database>: (Cadena de caracteres opcional)`**

Nombre de la base de datos de Lotus Notes que contiene los datos del usuario. Este nombre corresponde a la ruta del archivo ".NSF".

Si la base de datos de Lotus Notes se encuentra en un servidor de Domino, se debe utilizar un ruta relativo.

Si no se especifica este parámetro, el nombre de la base de datos de Lotus Notes se leerá en el archivo de configuración del cliente de Lotus Notes (archivo"Notes.INI").. La base de datos utilizada corresponderá a la base de datos del usuario del cliente de Lotus Notes para el ordenador actual..

## Observaciones


### Versión de Lotus Notes
<a name="version_lotus_notes_ELTPARAGRAPHE000102"></a>

El acceso a Lotus Notes está disponible para Lotus Notes versión 6.5 o posterior.
<a name="NOTE0_2"></a>


### Timeout
<a name="timeout_ELTPARAGRAPHE000109"></a>

El tiempo de espera puede ser set utilizando [EmailSetTimeOut](../WDLang3/3032001.md).

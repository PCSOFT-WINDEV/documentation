
## emailNotesSession (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **emailNotesSession** permite acceder a un servidor de mensajería Lotus Notes para identificar los mensajes que se encuentran en este servidor y enviar mensajes. Las características de la conexión pueden definirse y modificarse mediante diferentes propiedades de WLanguage.

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Start the session 
MyNotesSession is emailNotesSession

MyNotesSession..AddressDominoServer = "domino.mycompany.com"
MyNotesSession..LotusNotesDatabase = "email/accounting.nsf"
MyNotesSession..CertificatePath = SysDir(srMyDocuments) + ["\"] + "certificates\notes.id"
MyNotesSession..Password = EDT_Password

EmailStartSession(MyNotesSession)
```

## Observaciones


### Propiedades específicas de las variables emailNotesSession
<a name="propiedades_especificas_las_variables_emailnotessession_ELTPARAGRAPHE000027"></a>

Las siguientes propiedades se pueden utilizar para manipular una variable de tipo **emailNotesSession**:

| denominar | Tipo utilizado | Efecto |
| --- | --- | --- |
| DirecciónDominoServidor | Cadena de caracteres | <br><br>- Nombre del servidor Domino,<br><br>- Vaciar la cadena (""") si la base de datos es una base de datos local.<br><br><br> |
| Base de datos de LotusNotes | Cadena de caracteres | Nombre de la base de datos de Lotus Notes que contiene los datos del usuario. Este nombre corresponde a la ruta del archivo ".NSF".<br><br>Si la base de datos de Lotus Notes se encuentra en un servidor de Domino, se debe utilizar un ruta relativo.<br><br>Si no se especifica esta propiedad, el nombre de la base de datos Lotus Notes se leerá en el archivo de configuración del cliente Lotus Notes (archivo "Notes.INI"). La base de datos utilizada corresponderá a la base de datos del usuario del cliente de Lotus Notes para el ordenador actual. |
| Ruta de certificado | Cadena de caracteres | Ruta de Certificate utilizada para autentificar al usuario (archivo "ID"). Este archivo lo proporciona el administrador del servidor de Domino. Este archivo es necesario para establecer la conexión con el servidor.<br><br>Si no se especifica esta propiedad, la ruta del certificado se leerá en el archivo de configuración del cliente Lotus Notes (archivo "Notes.INI"). El Certificate utilizado corresponderá al Certificate del usuario del cliente de Lotus Notes para el ordenador actual. |
| Contraseña | Cadena de caracteres | Contraseña de usuario. Esta contraseña es proporcionada por el proveedor de servicios o por el administrador de la red. |

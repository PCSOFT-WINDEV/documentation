


## XMLAddChild( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade una etiqueta de niño en un Document XML. Esta etiqueta secundaria se añade al final de la lista de atributos de la posición actual.






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Add a new order line
XMLParent("XMLOrd")
XMLAddChild("XMLOrd", "orderline", "", True)
XMLAddAttribute("XMLOrd", "number", "2")

// Add the "productcode" tag
XMLAddChild("XMLOrd", "productcode", "sro2125")

// Add the "description" tag
XMLAddChild("XMLOrd", "description", "optical mouse")

// Add the "quantity" tag
XMLAddChild("XMLOrd", "quantity", "15")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`XMLAddChild(<XML document> , <Tag> [, <Tag value> [, <Change of position> [, <Option>]]])`
---

**`<XML document>: (Cadena de caracteres)`**

Nombre del XML Document en el que debe realizarse la adición. Esta Document contiene el código XML y fue creada por [XMLDocument](../WDLang5/3081008.md).

**`<Tag>: (Cadena de caracteres)`**

Nombre de la etiqueta a añadir. 
Si este parámetro corresponde a la constante *CDATA*, el contenido de la etiqueta CDATA no será interpretado por el analizador XML. Tienes la capacidad de incluir una Image en formato binario en esta etiqueta. Atención: la codificación utilizada debe respetar la codificación del XML Document.



**`<Tag value>: (Cadena de caracteres opcional)`**

Valor de la nueva etiqueta. Este parámetro corresponde a una cadena vacía de Default.

**`<Change of position>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> para posicionarse en el elemento agregado. En este caso, el elemento añadido se convierte en la posición actual. 

- <u><u><u><u>False</u></u></u></u> (valor predeterminado) para no modificar la posición en curso al agregar el elemento.




**`<Option>: (Constante opcional de tipo Integer)`**

Opciones de adición. Los valores posibles son:



|   |   |
| --- | --- |
| *XMLEncodingAmpersand* | Si el carácter "&" (ampersand) se encuentra en el valor de la etiqueta añadida, se codificará en formato "&"..<br>Si no se especifica esta constante, el carácter "&" debe estar codificado en la etiqueta XML. No se realizará ninguna codificación. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Gestión de la adición
<a name="gestion_adicion_ELTPARAGRAPHE000089"></a>

- Si la posición actual es un atributo, la nueva etiqueta se añade al final de la lista actual de etiquetas secundarias para la etiqueta que contiene el atributo.

- La posición actual en el documento no cambia al llamar a la función **XMLAddChild** excepto si el parámetro &lt;Cambio de posición&gt; se establece en <u><u><u><u>True</u></u></u></u>.


**Observación**: [XMLDocument](../WDLang5/3081008.md) no crea ninguna raíz para el Document. Para crear un primer elemento hijo en una Document vacía, esta Document debe contener necesariamente una raíz.
<a name="NOTE0_2"></a>


### Guardar las modificaciones en el archivo XML
<a name="guardar_las_modificaciones_archivo_xml_ELTPARAGRAPHE000108"></a>

Para guardar las modificaciones realizadas en un fichero XML, debe hacerlo:

1. Utilice la función [XMLBuildString](../WDLang5/3081016.md). Esta función recupera y da formato al contenido de un Document XML.

2. Guardar la cadena de caracteres generada por [XMLBuildString](../WDLang5/3081016.md) en un archivo XML. Para ello, utilice la función [fSaveText](../WDLang1/3036046.md).




Ejemplo:


```wl
// Once the XML document was modified 
XMLSource = XMLBuildString("XMLDoc")
// Save the XML file
fSaveText("ModifiedExample.xml", XMLSource)
```

## Sintaxis

`<Result> = <Report>.Print([<Parameter 1> [... [, <Parameter N>]]])`
---

**`<Result>: (Tipo de resultado devuelto por el reporte (opcional))`**

Valor devuelto por la impresión de la reporte. Este valor es devuelto: 

- por la palabra clave [RETURN](../Motscles/1510007.md) utilizada en el código de cierre de reporte. 

- por [ReturnedValue](../Proprietes/2510134.md) utilizado antes de cerrar el reporte.




**`<Report>: (Nombre del reporte)`**

Nombre de la reporte, definida en la editor de reportes.

**`<Parameter 1>: (Parámetros opcionales (el tipo corresponde al tipo de parámetro))`**

Parámetro opcional esperado por la reporte. Este parámetro se definió en el código de apertura de reporte. Este parámetro puede ser el nombre de una vista, el nombre de una zona de memoria, etc..
**Atención**: Si se han especificado límites inferiores y superiores para el reporte, los dos primeros parámetros deben corresponder al límite inferior y al límite superior (respectivamente).



**`<Parameter N>: (Parámetros opcionales (el tipo corresponde al tipo de parámetro))`**

Parámetro opcional esperado por la reporte. Este parámetro se definió en el código de apertura de reporte. Este parámetro puede ser el nombre de una vista, el nombre de una zona de memoria, etc..
**Atención**: Si se han especificado límites inferiores y superiores para el reporte, los dos primeros parámetros deben corresponder al límite inferior y al límite superior (respectivamente).

## Observaciones


### Principio para imprimir una reporte
<a name="principio_para_imprimir_una_reporte_ELTPARAGRAPHE000382"></a>

La función **&lt;Reporte&gt;.Print** lee la fuente de datos del reporte (archivos de datos, consulta, control Tabla, etc.). **&lt;Reporte&gt;.Print** imprime el bloque *Body* para cada Record. Los otros bloques se imprimen cuando es necesario:

- El bloque *Inicio de Document* se imprimirá al principio de Document.

- El bloque *Fin de Document* se imprimirá al final del Document.

- El bloque de cabecera *Page* se imprimirá en la parte superior de cada Page.

- El bloque *Page pie* se imprimirá al final de cada Page.

- Si hay una ruptura, el bloque *Body* se ubicará entre el bloque *Break header* y el bloque *Break footer*.



<a name="NOTE0_2"></a>


### Reporte con límites inferior y superior
<a name="reporte_con_limites_inferior_superior_ELTPARAGRAPHE000402"></a>

Si los límites inferior y superior se especificaron cuando se creó el reporte en el editor de reportes, los dos primeros parámetros que se pasan al reporte a través del **&lt;Reporte&gt;.Print** deben corresponder al límite inferior y al límite superior, respectivamente.

Ejemplo: se ha creado una reporte en el fichero Cliente. Esta reporte navega por el archivo en el elemento clave "CustomerNum".. Este reporte se definió como delimitado, con límites mínimos y máximos especificados (pestaña "Datos" de la ventana de descripción del reporte. 

El siguiente código se utiliza para imprimir la reporte para los clientes de 2 a 5: 

```wl
iDestination(iViewer)
RPT_Customers_Prospects.Print("2", "5")
```

<a name="NOTE0_3"></a>


### Ejecución de una consulta y de una reporte basada en esta consulta
<a name="ejecucion_una_consulta_una_reporte_basada_esta_consulta_ELTPARAGRAPHE000424"></a>

Al imprimir una reporte basada en una consulta, si la consulta ya se ha ejecutado, no se vuelve a ejecutar. Para volver a ejecutar la consulta (para añadir registros, por ejemplo), basta con utilizar [&lt;Fuente&gt;.ExecuteQuery](../WDLang4/1000024944.md) o [&lt;Reporte&gt;.InitQuery](../WDLang5/1000025143.md).
<a name="NOTE0_4"></a>


### Destino de la impresión
<a name="destino_impresion_ELTPARAGRAPHE000444"></a>

Puede set el destino de impresión del reporte utilizando [iDestination](../WDLang5/3046074.md).

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [iDestination](../WDLang5/3046074.md) con la constante *iViewer* se utiliza para mostrar la impresión en un visor reporte. El usuario puede entonces seleccionar el modo de impresión (PDF, impresora, etc.). Para saber si el trabajo de impresión se inició desde el visor de reporte, utilice [iDocumentoImpreso](../WDLang5/3046029.md).

### Pasar parámetros
<a name="pasar_parametros_ELTPARAGRAPHE000468"></a>

Los parámetros indicados por la función **&lt;Reporte&gt;.Print** corresponden a los parámetros definidos en el código de apertura del reporte con la siguiente sintaxis:


```txt
PROCEDURE <ReportName> (<NameParam1>, <NameParam2> = "<DefaultValue>", ...
	<NameParamN> = "<DefaultValue>")
```
**Recordatorio**: Algunos de los parámetros pasados a un Procedure pueden ser parámetros opcionales. Al declarar el Procedure, los parámetros opcionales deben ser descritos en la última posición (a la derecha), con un valor de Default precedido por el signo "=".. Para obtener más información, consulte [Parámetros de un procedimiento](../Motscles/1513001.md).

Si la reporte se basa en una consulta, hay que utilizar [&lt;Reporte&gt;.InitQuery](../WDLang5/1000025143.md) para pasar los parámetros de la consulta antes de imprimir la reporte.
<a name="NOTE0_6"></a>


### Impresión de una Component reporte
<a name="impresion_una_component_reporte_ELTPARAGRAPHE000493"></a>Para imprimir un Component reporte, utilice el nombre de Component reporte (el Component que se incluye en el proyecto). Por ejemplo:

```wl
ComponentReport.Print()
```


Si se produce un conflicto con un elemento del proyecto, el nombre del elemento debe ir precedido del nombre del componente. Por ejemplo:

```wl
MyComponent.Report.Print()
```


Para utilizar el nombre reporte en una variable, especifique el nombre Component. Por ejemplo:

```wl
sReport is string = "MyComponent.MyReport"
sReport.Print()
```

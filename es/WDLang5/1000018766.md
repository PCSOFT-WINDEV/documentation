


## &lt;sfConnection Variable&gt;.EjecutarProceso (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Solicita la ejecución de un Process específico. Es posible:

- Presentar los registros a la aprobación de Process.

- Realice una acción de autorización.







<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Salesforce connection>.ExecuteProcess(<Process to perform>)`
---

**`<Result>: (Variable de tipo Array)`**

Array de las variables [sfExecuteProcessResult](../WDLang5/1000018559.md) que contienen los datos del resultado.

**`<Salesforce connection>: (sfConnection Variable)`**

Nombre de la variable de tipo [sfConnection](../WDLang5/1000017579.md) que se utilizará

**`<Process to perform>: (Array de variantes)`**

Array de variantes que contienen los procesos a ejecutar. Esta array puede contener los siguientes tipos de objetos:

- [sfProcessWorkitemRequest](../WDLang5/1000018520.md).

- [sfProcessSubmitRequest](../WDLang5/1000018516.md).






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Posibles casos de error: 
<a name="posibles_casos_error_ELTPARAGRAPHE000175"></a>

- La conexión de INTERNET no es válida.

- La autenticación no se realizó correctamente.

- La respuesta XML del servidor no se puede leer.




Para averiguar si se produjo un error, utilice el [ErrorOccurred](../WDLang1/3087001.md) Variable.

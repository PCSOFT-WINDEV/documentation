
## fbSession (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **fbSession** se utiliza para describir una conexión al servicio de Facebook y para gestionar la autenticación de este servicio. Las características de esta conexión pueden definirse y modificarse mediante diferentes propiedades de WLanguage. 

**Importante: Antes de utilizar este tipo de WLanguage, la aplicación debe ser declarada a Facebook**. Para obtener más información, consulte [Utilizar la autenticación de Facebook](../WDLang5/1000021926.md). 

**Observaciones**: 

- Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).

- Este tipo de Variable está disponible en Linux sólo para sitios WEBDEV. 







<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MySession is fbSession
MySession.AppID = "1702546803104741"
MySession.AppSecret = "badf00ddeadbeefbadc0deoff1ce8ecd"
MySession.Permission = [fbInfoProfile, fbEmail]
IF FBStartSession(MySession) THEN
	Info("Connected")
END
```

## Observaciones
<a name="NOTE0_1"></a>


### Propiedades específicas de la descripción de variables de tipo fbSession
<a name="propiedades_especificas_descripcion_variables_tipo_docparampagetitleshort_ELTPARAGRAPHE000052"></a>

Las siguientes propiedades pueden ser utilizadas para manipular una sesión de Facebook:

| Nombre de la propiedad | Tipo utilizado | Efecto |
| --- | --- | --- |
| AppID | Cadena de caracteres | Identificador de la aplicación suministrada al declarar la aplicación de Facebook. |
| AppSecret | Cadena de caracteres | Código secreto de la aplicación suministrada al declarar la aplicación de Facebook. <br><br> |
| Email | Cadena de caracteres | Email del usuario final (puede ser usado para llenar el diálogo de conexión de antemano). <br><br> |
| PageID | Cadena de caracteres | Identificador del Facebook Page a abrir. Si este parámetro está vacío, la sesión se inicia en un usuario y no en una Page. <br><br> |
| Permission | Array | Derechos necesarios durante la conexión a Facebook. Los derechos (o permisos) de suministro están definidos por Facebook. La lista de derechos disponibles puede consultarse en la página web de Page: [https://developers.facebook.com/docs/facebook-login/permissions/v2.3](https://developers.facebook.com/docs/facebook-login/permissions/v2.3). <br><br>Para algunos derechos (los más comunes), tiene la posibilidad de usar las siguientes constantes: <br><br>- *fbEmail*: "email" Permission: Se utiliza para acceder al primer correo electrónico Address de una persona. <br><br>- *fbInfoFriends*: "usuario_amigos" Permission: Se utiliza para acceder a la lista de amigos de su aplicación.. <br><br>- *fbInfoProfile*: "public_profile" Permission: Se utiliza para acceder al perfil público de una persona. <br><br><br><br><br> |


<a name="NOTE0_3"></a>


### Cierre de la sesión
<a name="cierre_sesion_ELTPARAGRAPHE000128"></a>

Una sesión de Facebook iniciada es una sesión persistente: permanecerá abierta: 

- hasta que se cierre la aplicación. 

- hasta que se llame a la función [FBStopSession](../WDLang5/1000021836.md).


## iSequencingAddPDF( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade una impresión de PDF Document en una secuencia de impresiones reporte. 

Observación: La secuencia de impresiones de reporte le permite imprimir diferentes informes uno tras otro, y verlos como uno solo en el visor de reporte. Puede añadir varios tipos de documentos en esta secuencia. Para obtener más información, consulte [Secuenciación de informes](../WDChamp/9000128.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Print in the report viewer
iDestination(iViewer)
// Add reports into the sequence
iSequencingAdd(RPT_Invoice)
// Add the general conditions in PDF File format
iSequencingAddPDF("GCD.pdf")
iSequencingPrint()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`iSequencingAddPDF(<PDF document> [, <Password> [, <Sizing> [, <List of pages>]]])`
---

**`<PDF document>: (Tipo de elemento)`**

PDF Document para imprimir. Este documento corresponde a: 

- la ruta de un archivo PDF. 

- una nota HFSQL correspondiente a un PDF Document

- un Buffer Variable que corresponde a un PDF Document.




**`<Password>: (Cadena de caracteres)`**



- Contraseña para abrir el archivo PDF,

- Cadena vacía ("") si el fichero no tiene contraseña.




**`<Sizing>: (Constante de tipo Integer)`**

Tamaño de impresión: 


|   |   |
| --- | --- |
| *iHomothetic* | Ampliación o reducción homotética de las páginas del PDF Document para adaptarlas al formato de salida, ignorando los márgenes físicos.  |
| *iHomotheticNoEnlargement* | Reducción homotética de las páginas del PDF Document para adaptarlas al formato de salida, ignorando los márgenes físicos. Las páginas que utilizan un formato menor que el formato de salida mantienen su formato y están centradas.. |
| *iHomotheticNoEnlargementPrintableArea* | Reducción homotética de las páginas del PDF Document para adaptarlas al formato de salida, teniendo en cuenta los márgenes físicos. Las páginas con un formato más pequeño mantienen su formato y están centradas. |
| *iRealSize* | Imprimir en tamaño real. Las páginas que utilizan un formato: <br><br>- más pequeños que el formato de salida se centran en el Page. <br><br>- mayores que el formato de salida se truncan y se centran en el Page. <br><br><br> |



**`<List of pages>: (Cadena de caracteres)`**

Lista de páginas para imprimir. Se puede especificar el intervalo de páginas a imprimir separadas por una coma, cada intervalo está en formato "X-Y", donde X e Y son números de páginas. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Los documentos PDF se añaden realmente durante la llamada a [iSequencingPrint](../WDLang5/1000019903.md). 

- En esta versión, los informes y los documentos PDF secuenciados conservan su propia numeración..

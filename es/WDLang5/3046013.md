
## iPageNum( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve o inicializa el número de la página actualmente impresa.

**Observación**: **iPageNum** se puede utilizar para forzar el número de la primera página a un número distinto de 1.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Prints the page number
iPrint(iXPos(200) + iPageNum())
// Forces the page number to 10
iPageNum(10)
...
iEndPrinting()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = iPageNum([<Page number>])`
---

**`<Result>: (Integro)`**

Número de página actual.

**`<Page number>: (Entero opcional)`**

Nuevo número para la página actual.

## Observaciones




### iPageNum y los informes creados con editor de reportes
<a name="docparampagetitleshort_los_informes_creados_con_editor_reportes_ELTPARAGRAPHE000068"></a>

- **iPageNum** se puede utilizar en los informes, siempre que se inicie la impresión (en el "Después de la impresión" Event de una pausa, por ejemplo). Esta función no debe utilizarse en el código de apertura de reporte. Se debe utilizar la función [PageNumber](../Proprietes/2510106.md).

- **iPageNum** no debe utilizarse en el código de preimpresión de un bloque. De hecho, el tamaño final del bloque no se conoce en esta Process y el número de página en el que se posicionará el bloque puede cambiar.

- El resultado de **iPageNum** no se ve afectado por la reinicialización de los números de página en los informes ("Reiniciar el número de páginas después de imprimir el bloque" en la pestaña "UI" de una ventana de bloque description).

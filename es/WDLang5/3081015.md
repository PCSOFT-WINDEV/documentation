


## XMLExecuteXPath (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ejecuta una consulta del XPATH: 

- en una variable de tipo [xmlDocument](../WDLang5/1000018782.md).
	

- en un XML Document.
	













<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
clDoc is xmlDocument = XMLOpen(n, fromString)
c is xpathResult = XMLExecuteXPath(clDoc, "/root/text()")
If c.IsValue THEN
	Trace(c.Value)
ELSE
	FOR EACH ANode OF c.Node
		Trace("Node: " + ANode..Text)
	END
	FOR EACH attrib OF c.Attribute
		Trace("Node: " + attrib.Name)
	END
END
```
<a name="Example2"></a>

```wl
// Load the XML file
MyDoc is string
MyDoc = fLoadText("example.xml")
// Text of query: Find the Index tag
XPATHQuery is string = "Help/Function/Index"
// Create the document
XMLDocument("Example", MyDoc)
IF ErrorOccurred = True THEN
	Error("The file is not in XML format.")
	RETURN
END
b is boolean
b = XMLExecuteXPath("Example", XPATHQuery)
// Is it a calculation query?
IF b = True AND XMLOut("Example") = True THEN
	 // The calculation result is as follows
	 Trace("XMLResult=" + XMLResult("Example"))
END
// Is it a selection XPath query?
IF XMLFound("Example") = True THEN
	// Browse the result selection
	WHILE XMLFound("Example") = True
		Trace(XMLPath("Example"))
		XMLNext("Example")
	END
	// Browse completed, end the query
	XMLCancelSearch("Example")
END
```

## Sintaxis

### Ejecutando una consulta del XPATH en una xmlDocument Variable

`<Result> = XMLExecuteXPath(<XML document> , <Text of XPath query>)`
---

**`<Result>: (xpathResult Variable)`**

[xpathResult](../WDLang5/1000021967.md) Variable que contiene el resultado de la consulta XPATH. 

Lectura del resultado: 

- El **IsValue** Property permite saber si el resultado es un valor. 

- Para una consulta de selección (el resultado no es un valor), hay que recorrer la [xpathResult](../WDLang5/1000021967.md) Variable. 

- Para una consulta de cálculo (el resultado es un valor), el **Valor** Property lee el resultado.




**`<XML document>: (Variable de tipo xmlDocument)`**

Nombre de la [xmlDocument](../WDLang5/1000018782.md) Variable que contiene el código XML a analizar.

**`<Text of XPath query>: (Cadena de caracteres)`**

Texto de la consulta XPath a ejecutar.  


<a name="SYNTAX2"></a>

### Ejecutando una consulta XPATH en un XML Document (sintaxis guardada para compatibilidad con versiones anteriores)

`<Result> = XMLExecuteXPath(<Name of the XML document> , <Text of XPath query>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si la consulta fue ejecutada,

- <u><u><u><u>False</u></u></u></u> en caso contrario o si ningún elemento corresponde a la consulta.




Lectura del resultado: 

- En el caso de una consulta de selección, si se ha ejecutado la consulta XPATH, se utilizan las funciones de búsqueda XML para examinar el resultado de la consulta XPATH.

- Para una consulta de cálculo, se utiliza [XMLResultado](../WDLang5/3081027.md) para leer el resultado.




**`<Name of the XML document>: (Cadena de caracteres)`**

Nombre del XML Document utilizado. Este Document contiene el código XML a estudiar y fue creado por [XMLDocument](../WDLang5/3081008.md).

**`<Text of XPath query>: (Cadena de caracteres)`**

Texto de la consulta XPath a ejecutar.

### Sintaxis 2: Ejecutando una consulta XPATH en un XML Document: Lectura del resultado de la consulta
<a name="sintaxis_2_ejecutando_una_consulta_xpath_xml_document_lectura_del_resultado_consulta_ELTPARAGRAPHE000184"></a>

Si **XMLExecuteXPath** devuelve <u><u><u><u>True</u></u></u></u>, la consulta se ha ejecutado con éxito.. El resultado de la consulta puede leerse de la siguiente manera:

- **Consulta de cálculo**: La función [XMLFound](../WDLang5/3081020.md) devuelve <u><u><u><u>False</u></u></u></u>.
	El resultado puede ser leído por [XMLResultado](../WDLang5/3081027.md).
	Algunos ejemplos de consultas de cálculo:
	


|   |   |
| --- | --- |
| Contar el número de etiquetas cuyo nombre es "ITEM", lo que significa el número de artículos diferentes que se encuentran en el pedido. | Count(//ITEM) |
| Número de etiquetas que contienen un subelemento llamado &lt;PRICE&gt;. | count(//\*[PRICE]) |
| Total de la factura | suma(//ITEM[ PRECIO])- suma(//COUPON[ PRECIO]) |



- **Consulta para la selección de registros**: [XMLFound](../WDLang5/3081020.md) devuelve <u><u><u><u>True</u></u></u></u>.
	En una consulta de selección, **XMLExecuteXPath** cambia la posición actual en el documento XML a set la posición en el primer elemento del resultado de la selección. A continuación, [XMLNext](../WDLang5/3081030.md) le permite navegar por la selección.
	Algunos ejemplos de consultas de selección:
	


|   |   |
| --- | --- |
| Busque todas las etiquetas cuyo nombre sea "ITEM". | //TEMA |
| Examinar todos los elementos con un atributo "unitario". | //\*[@unit] |


	Para cancelar la consulta y volver a navegar por toda la Document, utilice [XMLCancelSearch](../WDLang5/3081001.md).



<a name="NOTE0_2"></a>


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000285"></a>



- Varias consultas de exploración de XPath no pueden ser anidadas.

- Sintaxis 1: Ejecución de una consulta XPATH en una Variable de tipo [xmlDocument](../WDLang5/1000018782.md): 

	- La lista de nodos no está clasificada. 

	- El estándar utilizado para la consulta XPath es XPath 1.0. 




- Sintaxis 2: Ejecutando una consulta XPATH en un XML Document: 
	[XMLSavePosition](../WDLang5/3081033.md) y [XMLRestorePosition](../WDLang5/3081032.md) no pueden utilizarse para guardar/restablecer una posición cuando se examina el resultado de una consulta select XPath.

### Funciones y subprocesos XML
<a name="funciones_subprocesos_xml_ELTPARAGRAPHE000471"></a>

Si su aplicación utiliza hilos, el XML Document se comparte entre todos estos hilos. Para más detalles sobre los hilos, véase [Gestión de hilos](../WDLang1/3077026.md).
Si la posición actual en un XML Document se modifica en un thread, la posición actual en este XML Document se modifica para todos los thread s.

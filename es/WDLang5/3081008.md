


## XMLDocument( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Crea un nuevo Document XML. Este Document puede ser:

- un Document vacío. Se puede rellenar con [XMLAddAttribute](../WDLang5/3081011.md), [XMLAddChild](../WDLang5/3081002.md), ...

- un Document que contiene el código XML que será utilizado por las funciones XML de WLanguage. Este código puede provenir de un archivo en formato XML o de un Webservice. El elemento actual corresponde automáticamente a la etiqueta raíz (la raíz no se crea si no existe). Esta Document puede ser cerrada por [XMLClose](../WDLang5/3081034.md).




**Observación**: La función **XMLDocument** no manipula un archivo sino un código XML.









<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
XMLSource is string
Res is boolean
// Create the XML document
Res = XMLDocument("XMLDoc", XMLSource)
// Was the document created?
IF Res = False THEN
	Error("The document was not created.")
ELSE
	Info("The document was successfully created. It is ready to be used.")
END
// Modify the document
...
```

```wl
	Error("The following problem was detected: " + ErrorInfo())

	Info("The document was successfully created. It is ready to be used.")
END
// Modify the document
...
// Rebuild the XML string
XMLSource = XMLBuildString("XMLDoc")
// Save the XML document in a file

```

## Sintaxis

`<Result> = XMLDocument(<XML document> [, <XML code> [, <Option>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se crea el documento XML,

- <u><u><u><u>False</u></u></u></u> en caso contrario. En caso de error: 

	- la situación actual permanece inalterada. 

	- la variable [ErrorOccurred](../WDLang1/3087001.md) se establece en <u><u><u><u>True</u></u></u></u>. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).







**`<XML document>: (Cadena de caracteres)`**

Nombre dado al XML Document. Este nombre será utilizado por todas las funciones XML.

**`<XML code>: (Cadena de caracteres opcional)`**

Código XML que será analizado por las funciones XML. Si no se especifica este parámetro, se crea un nuevo Document XML vacío.

Atención: 

- Si la codificación del Document no se especifica en el encabezado XML, la codificación utilizada es "UTF-8".

- Para especificar una codificación diferente, el atributo "encoding" debe añadirse a la etiqueta XML.
	Por ejemplo: &lt;?xml version="1.0" encoding="UCS-2"?&gt;


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Para crear un Document XML vacío, este parámetro debe corresponder a una cadena vacía ("").

**`<Option>: (Constante opcional)`**

Tipo del XML Document creado (ninguna opción es seleccionada por Default): 



|   |   |
| --- | --- |
| *xdHTML* | XHTML Document. Esta constante se utiliza para optimizar el modo de funcionamiento de [XMLBuildString](../WDLang5/3081016.md). El Document se procesa usando el modo xHTML. |
| *xHTML* | HTML Document.<br><br>**Atención**: **Esta constante ya no está disponible**. De hecho, el modo de funcionamiento de esta constante no fue optimizado: el Document se manejaba utilizando el formato XML y luego, en caso de fallo, se manejaba utilizando el formato xHTML. |

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Este parámetro no se tiene en cuenta.

## Observaciones


### Recordatorio: lenguaje XML
<a name="recordatorio_lenguaje_xml_ELTPARAGRAPHE000154"></a>

XML es un lenguaje que contiene etiquetas y atributos (llamados "elementos" en esta ayuda). La estructura de un código XML corresponde a una estructura de árbol: la información está organizada jerárquicamente.

Los siguientes términos se utilizan para describir las diferentes partes de un código XML:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=schema_xml.gif)

<a name="NOTE0_2"></a>


### Funciones y subprocesos XML
<a name="funciones_subprocesos_xml_ELTPARAGRAPHE000303"></a>

Si su aplicación utiliza hilos, el XML Document se comparte entre todos estos hilos. Para más detalles sobre los hilos, véase [Gestión de hilos](../WDLang1/3077026.md).
<a name="NOTE0_3"></a>


### Límite
<a name="limite_ELTPARAGRAPHE000169"></a>

La función **XMLDocument** está adaptada para cargar documentos XML cuyo tamaño de nodos de texto no supere los 9 MB. Para los documentos manipular con un tamaño mayor de nodos, debe utilizar el tipo [XMLDocument](../WDLang5/1000018782.md).


## iSequencingAddDoc( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade una impresión de procesamiento de textos Document en una secuencia de impresiones reporte. 

Observación: La secuencia de impresiones de reporte le permite imprimir diferentes informes uno tras otro, y verlos como uno solo en el visor de reporte. Puede añadir varios tipos de documentos en esta secuencia. Para obtener más información, consulte [Secuenciación de informes](../WDChamp/9000128.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyDocument is Document = "c:\temp\generalconditions.docx"

// Print in the report viewer
iDestination(iViewer)
// Add reports into the sequence
iSequencingAdd(RPT_Report1)
iSequencingAdd(RPT_Report2, 3)
// Add the general conditions in document format
iSequencingAddDoc(MyDocument)
iSequencingPrint()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`iSequencingAddDoc(<Document> [, <Password>])`
---

**`<Document>: (Tipo de elemento)`**

Document para imprimir. Este Document corresponde a : 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) el nombre de un control Procesador de texto.

- una variable de tipo [Document](../WDLang1/1000022461.md).

- la ruta de un fichero de tratamiento de textos (fichero docx).  

- un memorándum HFSQL correspondiente a un procesador de textos Document.  

- un Buffer Variable correspondiente a un procesador de textos Document.




**`<Password>: (Cadena de caracteres opcional)`**



- Contraseña para abrir el archivo "docx" (o Document),

- Vaciar la cadena ("") si el archivo (o Document) no tiene una contraseña.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Los documentos de tratamiento de textos se añaden realmente durante la llamada a [iSequencingPrint](../WDLang5/1000019903.md). 

- En esta versión, los informes y los documentos secuenciados conservan su propia numeración..

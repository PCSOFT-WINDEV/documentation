


## iPrintImage( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Envía el Image para imprimir al buffer de impresión. El trabajo de impresión no se ha iniciado (sólo se puede utilizar [iEndPrinting](../WDLang5/3046053.md) para iniciar el trabajo de impresión). El modo de impresión de la imagen depende de los parámetros pasados a la función **iPrintImage**.

**Observación**: Esta función no debe usarse para imprimir el fondo Image de un reporte. Para definir la imagen de fondo de una reporte, utilice la [BackgroundImage](../Proprietes/2510056.md) Property. El [BackgroundImagePrinted](../Proprietes/1000021350.md) Property especifica si la imagen debe imprimirse con el reporte.

![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Sólo se pueden imprimir las imágenes en formato JPEG.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003046017&name=iprintimage_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Prints Flower.JPG at line 20mm, column 50 mm without stretching or reducing
iPrintImage("Flower.JPG", 20, 50)
iEndPrinting()
```

## Sintaxis

### Imprimir una Image en un área específica

`iPrintImage(<Image name> , <X1> , <Y1> , <X2> , <Y2> [, <Scale>])`
---

**`<Image name>: (Cadena de caracteres o variable de tipo Image)`**



- Nombre del archivo que contiene la Image para imprimir. El formato de este archivo debe ser uno de los formatos reconocidos de Image.

- Nombre de Image Partida pro memoria. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) Nombre de una variable de tipo [Image](../WDLang1/1000019650.md).




![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Este parámetro debe corresponder a un archivo Image en formato JPEG.

**`<X1>: (Real)`**

Coordenada horizontal de la esquina superior izquierda de la posición de Image (en milímetros). 
Valor incluido entre 0 y el ancho de la página (devuelto por [iPageWidth](../WDLang5/3046036.md)).

**`<Y1>: (Real)`**

Coordenada vertical de la esquina superior izquierda de la posición Image (en milímetros). 
Valor incluido entre 0 y la altura de la página (devuelto por [iPageHeight](../WDLang5/3046052.md)).

**`<X2>: (Real)`**

Coordenada horizontal de la esquina inferior derecha de la Border donde debe imprimirse la Image (en milímetros). 
Valor incluido entre 0 y el ancho de la página (devuelto por [iPageWidth](../WDLang5/3046036.md)).

**`<Y2>: (Real)`**

Coordenada vertical de la esquina inferior derecha de la Border donde debe imprimirse la Image (en milímetros). 
Valor incluido entre 0 y la altura de la página (devuelto por [iPageHeight](../WDLang5/3046052.md)).

**`<Scale>: (Entero o constante de tipo integer (opcional))`**

Configura la escala del zoom realizado en la Image.


|   |   |
| --- | --- |
| *iHomoCenteredDisplay* | La imagen se amplía o reduce para ocupar de forma óptima el espacio definido sin distorsionarse. Si está centrado en el área definida por &lt;X1&gt;, &lt;Y1&gt;, &lt;X2&gt; y &lt;Y2&gt;..<br><br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Img_HomoCentre.GIF)<br> |
| *iHomotheticDisplay* | La Image se muestra en &lt;X1&gt; &lt;Y1&gt; sin estar distorsionada. Ocupa toda la altura definida por &lt;X1&gt; y &lt;X2&gt;, o todo el ancho definido por &lt;Y1&gt; y &lt;Y2&gt;..<br><br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Img_Homo.GIF)<br> |
| *iNormalDisplay*<br>(Valor predeterminado) | La Image se muestra en todo el rectángulo definido por &lt;X1&gt;, &lt;Y1&gt;, &lt;X2&gt; y &lt;Y2&gt;. La Image puede ser distorsionada.<br><br><br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Img_Normal.GIF)<br> |

## Observaciones


### Varios
<a name="varios_ELTPARAGRAPHE000252"></a>

- El origen (0,0) se encuentra en el Corner superior izquierdo de la hoja.. Tiene en cuenta los márgenes físicos de la impresora.

- Si la imagen no cabe en el área definido y debe centrarse (&lt;Escala&gt; = 999), no se tiene en cuenta la opción de centrar.



<a name="NOTE0_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 

### Impresión en un archivo HTML
<a name="impresion_archivo_html_ELTPARAGRAPHE000262"></a>

Si se utiliza **iPrintImage** para imprimir un reporte HTML (creado con el editor de reportes), el Image impreso estará distorsionado.
<a name="NOTE0_3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 

### Impresión en PDF
<a name="impresion_pdf_ELTPARAGRAPHE000273"></a>

Si sólo se especifican las coordenadas de la esquina superior izquierda de la Image, el ancho de la Image depende de los PPP de la impresora.

Al imprimir en PDF, el PDF y la impresora actual tienen el mismo DPI: en este caso, el ancho del Image en el PDF corresponderá al ancho del Image en la vista previa.

Cuando se imprime en PDF genérico, el PDF y la pantalla tienen el mismo DPI (para evitar depender de una impresora). El Image puede no tener la anchura solicitada.

Le aconsejamos que especifique las coordenadas de la esquina inferior derecha de la imagen para que get siempre dé el mismo resultado (en PDF, en PDF genérico, e independientemente del DPI de la impresora).

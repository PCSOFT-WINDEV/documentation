
## iSequencingAddDiagram( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade una Diagram Document en una secuencia de impresiones reporte. 

Observación: La secuencia de impresiones de reporte le permite imprimir diferentes informes uno tras otro, y verlos como uno solo en el visor de reporte. Puede añadir varios tipos de documentos en esta secuencia. Para obtener más información, consulte [Secuenciación de informes](../WDChamp/9000128.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyDiagram is Diagram = "c:\temp\Diagram_archi.wddiag"

// Print in the report viewer
iDestination(iViewer)
// Add reports into the sequence
iSequencingAdd(RPT_Report1)
iSequencingAdd(RPT_Report2, 3)
// Add diagram
iSequencingAddDiagram(MyDiagram)
iSequencingPrint()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`iSequencingAddDiagram(<Diagram>)`
---

**`<Diagram>: (Tipo de elemento)`**

Diagram para imprimir. Este documento corresponde a: 

- el nombre de un control Editor de diagramas. 

- una variable de tipo [Diagrama](../WDLang1/1410088055.md).

- a ruta de un archivo Diagram (archivo wddiag).  

- una nota HFSQL correspondiente a un Diagram Document.  

- una variable Buffer correspondiente a una Diagram Document.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Los documentos Diagram se añaden realmente cuando se llama a [iSequencingPrint](../WDLang5/1000019903.md). 

- En esta versión, los informes y los documentos secuenciados conservan su propia numeración.

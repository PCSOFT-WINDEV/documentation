
## WDMap: Viendo los registros encontrados en un archivo de datos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000202"></a>
WDMap permite mostrar los registros de un **archivo de datos**. 

Esto le permite a get una vista general de los diferentes registros encontrados en un archivo de datos mientras desarrolla una aplicación.

Observación: WDMap no le permite ver el resultado de una consulta. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Visualización predeterminada
<a name="visualizacion_predeterminada_ELTTEXTE000226"></a>
Por Default, WDMap muestra los registros encontrados en un archivo de datos usando las siguientes reglas:

- Todos los registros encontrados en el archivo de datos se muestran.

- Los registros se clasifican según su orden de creación en el archivo de datos.

- Todos los elementos que se encuentran en el archivo de datos se muestran (excepto los de "Clave compuesta").

- Las posiciones están organizadas en orden lógico. El orden lógico corresponde al orden de visualización definido en la editor de análisis.

- Se soporta la integridad. 
	**Atención**: Si los archivos relacionado al archivo seleccionado no son encontrados automáticamente por WDMap, sólo podrá ver los registros. Para añadir o modificar estos registros, hay que localizar manualmente los ficheros que faltan (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IconeGestionIntegrite.gif)).

- Los duplicados son compatibles.




Para obtener más información, consulte [Casos especiales](../WDMap/3517005.md).

Sin embargo, usted tiene la capacidad de modificar:

- La selección de la información a mostrar. De hecho, WDMap le permite seleccionar:

	- los registros para ver,

	- los elementos a visualizar.




- Las opciones de visualización de tabla.




Para obtener más información, consulte [Modificar el modo de visualización de los registros](../WDMap/3517011.md).

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000250"></a>
Para ver los registros que se encuentran en un archivo de datos, el modo de funcionamiento puede cambiar según el modo de inicio de WDMap:

- Empieza desde el explorador o desde el editor de análisis.

- Comience desde WINDEV o WEBDEV.

- Empiece directamente desde "WDMAP.EXE".



<a name="NOTE3_2"></a>


### WDMap iniciado desde el explorador (doble clic en un archivo de datos) o desde la editor de análisis ("Editar el archivo de datos con WDMap" en el menú contextual)
<a name="wdmap_iniciado_desde_explorador_doble_clic_archivo_datos_desde_editor_analisis_editar_archivo_datos_con_wdmap_menu_contextual_ELTPARAGRAPHE000067"></a>

Para ver los registros que se encuentran en el archivo de datos seleccionado:

1. Especifique si es necesario:

	- La contraseña del archivo seleccionado (contraseña definida por [HPass](../WDLang4/3044108.md), [HCreation](../WDLang4/3044255.md) o [HCreationIfNotFound](../WDLang4/3044254.md), o durante el [Modificación de datos de automatic](../WDLang4/3044195.md)).

	- La ruta completa del análisis inicial en la versión 5.5 y su contraseña (si el archivo seleccionado proviene de un análisis en la versión 5.5).

	- Los parámetros de conexión (si el archivo se maneja a través de un proveedor de OLE DB, un controlador ODBC o un acceso nativo).




2. El contenido del archivo de datos se muestra en WDMap.



<a name="NOTE3_3"></a>


### WDMap comenzó desde WINDEV o WEBDEV ("Herramientas")
<a name="wdmap_comenzo_desde_windev_webdev_herramientas_ELTPARAGRAPHE000093"></a>

Para ver los registros encontrados en un archivo de datos:

1. Si es necesario, indique la contraseña del análisis (contraseña en tiempo de ejecución definida en la descripción del análisis).
	Observación: La aplicación actual en WINDEV debe ser associated con un análisis. En caso contrario, las operaciones a realizar son las descritas en [WDMAP iniciado directamente por "WDMAP.EXE](#NOTE3_4).

2. Seleccione el fichero lógico que desea visualizar.

3. Especifique si es necesario:

	- La contraseña del archivo seleccionado (contraseña definida por [HPass](../WDLang4/3044108.md), [HCreation](../WDLang4/3044255.md) o [HCreationIfNotFound](../WDLang4/3044254.md), o durante el [Modificación de datos de automatic](../WDLang4/3044195.md)).

	- La ruta completa del análisis inicial en la versión 5.5 y su contraseña (si el archivo seleccionado proviene de un análisis en la versión 5.5).

	- Los parámetros de conexión (si el archivo de datos se maneja a través de un proveedor de OLE DB, un controlador ODBC o un acceso nativo).

	- La ruta del fichero físico correspondiente al fichero lógico seleccionado..
			**Observación**: Si la aplicación actual soporta el fichero ".REP", puede simplificar la selección del directorio del fichero físico: haga clic en el botón ".REP file" y seleccione el archivo ".REP" de la aplicación actual. WDMap listará automáticamente todas las rutas de los archivos físicos correspondientes al archivo lógico a visualizar. Para obtener más información, consulte [Manejando el ".REP"](../WDLang4/3044186.md).




4. El contenido del archivo de datos se muestra en WDMap.



<a name="NOTE3_4"></a>


### WDMAP iniciado directamente por "WDMAP.EXE
<a name="wdmap_iniciado_directamente_por_wdmapexe_ELTPARAGRAPHE000132"></a>

Para ver los registros encontrados en un archivo de datos:

1. En la asistente, seleccione "Ver un archivo" y vaya al siguiente paso. Se muestra un selector de archivos.

2. Seleccione el archivo físico que desea ver (es necesario utilizar el botón "Examinar").

3. Especifique si es necesario:

	- La contraseña del archivo seleccionado (contraseña definida por [Modificación de datos de automatic](../WDLang4/3044195.md), [HPass](../WDLang4/3044108.md) o [HCreation](../WDLang4/3044255.md), o durante el [HCreationIfNotFound](../WDLang4/3044254.md)).

	- La ruta completa del análisis inicial en la versión 5.5 y su contraseña (si el archivo seleccionado proviene de un análisis en la versión 5.5).

	- Los parámetros de conexión (si el archivo se maneja a través de un proveedor de OLE DB, un controlador ODBC o un acceso nativo).




4. El contenido del archivo de datos se muestra en WDMap.

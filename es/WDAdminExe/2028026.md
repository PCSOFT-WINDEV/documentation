


## WDADMINEXE : Controlar la aplicación de forma remota
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000169"></a>
El Centro control de aplicaciones WINDEV permite al gestor de una aplicación en la red: 

- [gestionar la automatic o el cierre forzado de la aplicación](#NOTE2_1) para todos los usuarios actuales. El cierre puede ser necesario cuando se actualiza la aplicación o la base de datos, por ejemplo.
	Para obtener más información, consulte [Principio de control remoto](../WDLang1/3013006.md).
	**Observación**: La gestión de control a distancia puede ser personalizada por [AppControl](../WDLang1/3013003.md).

- [set la frecuencia de comprobación de las actualizaciones](#NOTE4_1).




Estos ajustes se definen en la pestaña "control" de WDADMINEXE.

**Importante**: Si las opciones de control remoto se modifican directamente en "WDADMINEXE" (programa de gestión), estas opciones se reinicializarán en cada instalación con las opciones especificadas al crear el programa instalación.

<a name="NOTE2_1"></a>


## Estado de la aplicación
<a name="estado_aplicacion_ELTTEXTE000193"></a>
La pestaña "control" de WDADMINEXE permite gestionar el estado de la aplicación y los mensajes que se muestran en función de dicho estado: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdminExe%20-%20HC%20N%B0001.gif)


- **Modo de funcionamiento estándar**: la desconexión de los usuarios de la aplicación se gestionará automáticamente. Para obtener más información, consulte [Gestionar la desconexión de automatic](#NOTE2_1).

- **Conexión prohibida**: los usuarios no podrán conectarse a la aplicación. Al ejecutar la aplicación, el mensaje associated con la conexión prohibida se mostrará durante la "duración de visualización de mensajes" especificada..
	Los usuarios que ya están conectados podrán utilizar la aplicación como de costumbre.

- **Advertencia de una interrupción inminente**: el mensaje de advertencia de associated se mostrará en las computadoras de los usuarios actualmente conectados a la aplicación. Este mensaje aparecerá en la pantalla:

	- durante la "duración de visualización de los mensajes" especificada.

	- cada vez que la aplicación comprueba su estado conectándose al servidor. Esta comprobación se realiza a intervalos de tiempo regulares (definidos en "Estado comprobado cada").
			Si los usuarios intentan conectarse a la aplicación, aparecerá el mensaje associated con la conexión prohibida.




- **parada forzada**: el mensaje de apagado se muestra en los ordenadores de los usuarios conectados actualmente a la aplicación. Este mensaje aparecerá en la pantalla:

	- durante la "duración de visualización de los mensajes" especificada.

	- tan pronto como la aplicación compruebe su estado conectándose al servidor. Esta comprobación se realiza a intervalos de tiempo regulares (definidos en "Estado comprobado cada").
			Una vez que se muestre este mensaje, los usuarios se desconectarán automáticamente y no podrán volver a conectarse..
			Si los usuarios intentan conectarse a la aplicación, aparecerá el mensaje associated con la conexión prohibida.







Las opciones relativas a la duración de los controles son las siguientes: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdminExe%20-%20HC%20N%B0001%201.gif)


- **Estado verificado cada**: En una aplicación instalada en una red, la aplicación del usuario comprueba regularmente: 

	- su estado según la aplicación del servidor (aplicación reference).

	- si hay una actualización disponible (después de actualizar la aplicación reference).




- **Mostrar la duración de los mensajes**: Los diferentes mensajes de WDADMINEXE (conexión prohibida, parada inminente y parada forzada) se muestran durante el tiempo especificado.. Los usuarios pueden cerrar la ventana que muestra este mensaje antes de que finalice la duración especificada.




Observación: Los parámetros modificados en WDADMINEXE se mantienen al instalar una actualización de la aplicación. Para restaurar los valores especificados al configurar el instalación, haga clic en "Restaurar valores Default"..

## Gestión automática de la desconexión de usuarios
<a name="gestion_automatica_desconexion_usuarios_ELTTEXTE000217"></a>
Para que la desconexión de usuarios se realice automáticamente, basta con seleccionar "Modo de funcionamiento estándar".

Si se selecciona esta opción, durante la actualización **obligatoria** de la aplicación en el servidor, se realizan automáticamente las siguientes operaciones:

- Se muestran los usuarios conectados actualmente a la aplicación.

- Tan pronto como la aplicación de usuario compruebe su estado conectándose al servidor, el mensaje que indica una parada inminente se mostrará en los ordenadores del usuario.. Este mensaje se mostrará durante la duración de visualización especificada.

- Una vez que todos los usuarios han recibido el mensaje que indica un cierre inminente, la actualización de la aplicación en el servidor pasa a "Parada forzosa" durante dos minutos.

- Durante estos dos minutos, el mensaje de apagado se muestra en los ordenadores de los usuarios.

- Después de estos dos minutos, los usuarios se desconectarán automáticamente y podrán volver a conectarse al final de la actualización de la aplicación en el servidor.

- Una vez que la aplicación ha sido actualizada en el servidor, los usuarios que se vuelvan a conectar a la aplicación tendrán que actualizar su aplicación de usuario.




Si se selecciona esta opción, durante la actualización **opcional** de la aplicación en el servidor, se realizan automáticamente las siguientes operaciones:

- La actualización de la aplicación en el servidor se realiza de forma habitual.

- Los usuarios actualmente conectados a la aplicación no serán desconectados.

- Una vez que la aplicación ha sido actualizada en el servidor, los usuarios que se vuelvan a conectar a la aplicación tendrán que elegir entre instalar la actualización o iniciar la aplicación directamente.

## Comprobación de las actualizaciones
<a name="comprobacion_las_actualizaciones_ELTTEXTE000241"></a>
La pestaña "Control" de WDADMINEXE permite gestionar la verificación de actualizaciones: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdminExe_2%20-%20HC%20N%B0001.gif)


Por Default, la disponibilidad de una actualización en el servidor se comprueba cada vez que se inicia la aplicación. Esta frecuencia de comprobación se puede configurar. Se puede realizar: 

- Cada vez que se inicia la aplicación. 

- Cada X días. 




Cada vez que una aplicación es lanzada (por defecto), una aplicación instalada por medio de una "instalación red" se verifica si se encuentra una actualización disponible en el servidor de referencia. 

- Si una versión más reciente está disponible en el servidor, la actualización de la aplicación se propone al usuario final.. 

- Si al abrir la aplicación no hay conexión de red, Default muestra un mensaje de advertencia . 
	Este mensaje puede ser inútil para una aplicación utilizada en un dispositivo móvil, conectada de forma intermitente.. En este caso, simplemente seleccione "Comprobación silenciosa: No informar al usuario"

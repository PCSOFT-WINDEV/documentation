
## JSInterruptEvent( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Interrumpe la Process de la actual Event.

El proceso actual es el procedimiento específico de WLanguage que fue asociado a la Event por [JSEvent](../WDLang2/1000018924.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Intercepts all clicks on the page
JSInterruptEvent(sJSEvent, jsInterruptEventAction)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`JSInterruptEvent(<Event> [, <Options>])`
---

**`<Event>: (Cadena de caracteres)`**

El actual Event que debe ser interrumpido.
Es el parámetro que recibe la Procedure (para más detalles, véase [JSEvent](../WDLang2/1000018924.md) ).

**`<Options>: (Combinación opcional de constantes de tipo Integer)`**

Indica el nivel de interrupción.
Si no se especifica ninguna constante, la corriente Procedure se interrumpe.

Las posibles constantes son:


|   |   |
| --- | --- |
| *jsInterruptEventAction* | La acción del navegador asociada a la Event se cancela además de la interrupción de la actual Procedure. Por ejemplo, si el actual Event es un clic en una casilla de verificación, el cambio de estado de la casilla se cancela. |
| *jsInterruptEventPropagation* | La propagación del evento se interrumpe además de la interrupción de la corriente Procedure. Si se asocian otros procedimientos con la misma Event sobre el mismo objeto, estos procedimientos seguirán siendo llamados. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
La función **JSInterruptEvent** solo se puede utilizar en un procedimiento WLanguage que procese eventos declarada utilizando la función [JSEvent](../WDLang2/1000018924.md).

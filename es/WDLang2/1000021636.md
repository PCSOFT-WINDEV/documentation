


## PageWriteHeaderHTTP( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade un encabezado HTTP adicional que será devuelto al navegador: 

- durante la siguiente función de visualización de la página ([PageDisplay](../WDLang2/3058008.md), [PageRefresh](../WDLang2/3058011.md), ...),

- durante la vuelta de automatic de la página actual al final de la acción actual.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
PageWriteHeaderHTTP("X-Frame-Options", "deny")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`PageWriteHeaderHTTP(<Header> , <Value>)`
---

**`<Header>: (Cadena de caracteres)`**

Nombre de la cabecera a escribir.

**`<Value>: (Cadena de caracteres)`**

Valor (o contenido) de la cabecera a escribir.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- El servidor de aplicaciones WEBDEV establece el valor de algunas cabeceras HTTP devueltas por el servidor (por ejemplo, "Content-Length"). En este caso, el valor escrito por **PageWriteHeaderHTTP** será ignorado.
	La lista de cabeceras es la siguiente: 

	- Content-xxx (content-length, content-type, content-disposition, etc.)

	- Ubicación

	- Permitir (casos específicos)

	- set-Cookie

	- Access-control-Expose-Headers




- Al llamar por segunda vez a la función **PageWriteHeaderHTTP** con el mismo nombre de cabecera, se sobrescribirá el valor anterior.

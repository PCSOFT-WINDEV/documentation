


## &lt;Upload&gt;.CopyFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Guarda en el servidor un archivo "cargado" por el usuario Web. Este upload puede haberse realizado: 

- a través de un"Upload" control Campo de entrada. 

- a través de un archivo único o de varios archivos control Upload. 




Durante esta copia de seguridad, el archivo copiado se puede renombrar en el servidor.




<a name="sample_code"></a>

## Ejemplo

<a name="Example1"></a>
<a name="3012023_Example1"></a>

```wl
// Get the file sent by the client
refFile is string
refFile = EDT_File.CopyFile("C:\Documents", "REF_3304")
// If the source file is: "/User/Foo.jpg"
// refFile = "REF_3304.jpg" and the file was copied into "C:\documents\REF_3304.jpg"
```
<a name="Example2"></a>

```wl
// Upload performed from a multi-file Upload control
// For each file sent
FOR i = 1 _TO_ UPL_Upload..Count
	// Copies the file into the data directory
	UPL_Upload.CopyFile(fDataDir(), "",  i)	
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Upload control>.CopyFile(<Path on server> , <File name> [, <File index>])`
---

**`<Result>: (Cadena de caracteres)`**



- Nombre (nombre + extensión) del archivo cargado en el servidor (no contiene el archivo ruta),

- Cadena vacía ("") si se produce un error.




**`<Upload control>: (Nombre del control)`**

Nombre de control Upload utilizado. Este control puede corresponder a: 

- un"Upload" control Campo de entrada. Este control contiene el nombre y ruta del archivo que será enviado al servidor. 
	Observación: es posible utilizar el atributo de un control Looper (Valor Property para el atributo).

- un control Upload de un solo archivo o de varios archivos.




**`<Path on server>: (Cadena de caracteres)`**

Ruta del directorio del servidor donde se copiará el archivo. Esta ruta puede corresponder a la ruta devuelta por [fDataDir](../WDLang1/3036001.md). Esta ruta también puede corresponder a [fCurrentDir](../WDLang1/3036022.md) aunque los anfitriones rara vez conceden los derechos de escritura en este directorio.
![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Este parámetro no se tiene en cuenta. El archivo se carga en el directorio "/tmp" del servidor Linux. Este directorio no se puede modificar. 

**`<File name>: (Cadena de caracteres)`**

Nombre (nombre y extensión) del archivo cargado en el servidor. El archivo se sobrescribe si ya existe.

Si este parámetro corresponde a:

- una cadena vacía ("""): el nombre del archivo en el servidor será idéntico al nombre del archivo en el equipo del usuario de la Web.

- sólo el nombre del archivo (sin la extensión), la extensión del archivo en el servidor será idéntica a la extensión del archivo en el equipo del usuario de la Web.

- el nombre y la extensión del archivo, este nombre y esta extensión se utilizarán al copiar el archivo en el servidor.




**`<File index>: (Entero opcional)`**

Índice del archivo a guardar en el servidor. Este parámetro sólo se utiliza para un upload realizado con el multiarchivo control Upload. Por Default, este índice es 1.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Tamaño máximo de los archivos a cargar
<a name="tamano_maximo_los_archivos_cargar_ELTPARAGRAPHE000245"></a>

**By Default,** el tamaño de los archivos a subir no está limitado en el motor WEBDEV. Este tamaño puede ser limitado por el servidor de aplicaciones WEBDEV. 
El archivo no se cargará si su tamaño excede el tamaño límite.
Para modificar el tamaño máximo de los archivos a cargar en WEBDEV servidor de aplicaciones, el registro debe ser modificado en el servidor:  

1. Para modificar el tamaño de los archivos cargados para todas las aplicaciones WEBDEV 28, seleccione la siguiente tecla: HKEY_LOCAL_MACHINE\\SOFTWARE\\PC SOFT\\WEBDEV\\28.0 en un ordenador de 32 bits y HKEY_LOCAL_MACHINE\\SOFTWARE\\WOW6432Node\\PC SOFT\\WEBDEV\\28.0 en un ordenador de 64 bits.

2. Añada la cadena MAX_UPLOAD. El valor de esta cadena será el tamaño máximo de los archivos a cargar (en KB).



<a name="NOTE0_2"></a>


### Nombre del archivo cargado
<a name="nombre_del_archivo_cargado_ELTPARAGRAPHE000277"></a>

[&lt;Subir&gt;.FileName](../WDLang2/1000019064.md) se utiliza para averiguar el nombre de un archivo "cargado".
<a name="NOTE0_3"></a>


### Sesiones preiniciadas
<a name="sesiones_preiniciadas_ELTPARAGRAPHE000293"></a>

Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".

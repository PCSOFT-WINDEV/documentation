


## UploadDelete (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina un archivo de la lista de archivos a cargar: el archivo no se cargará en el servidor. 

**Observación**: Esta función sólo se puede utilizar con una control Upload multiarchivo.. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// -- Browser click code of "BTN_Delete"
nFile is int = ListSelect(LIST_UploadMulti)
IF nFile <> -1 THEN
	UploadDelete(UPL_UploadMulti, nFile)
	// No need to update the list of files to upload
	// it will be automatically done by the modification code of Upload control
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`UploadDelete(<Upload control> , <File index>)`
---

**`<Upload control>: (Nombre del control)`**

Nombre del multi-fichero control Upload utilizado. 

**`<File index>: (Integro)`**

Índice del archivo que se borrará de la lista de archivos a cargar (1 para el primer archivo de la lista, 2 para el segundo, ....). 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Tamaño máximo de los archivos a cargar
<a name="tamano_maximo_los_archivos_cargar_ELTPARAGRAPHE000173"></a>

**By Default,** el tamaño de los archivos a subir no está limitado en el motor WEBDEV. Este tamaño puede ser limitado por el servidor de aplicaciones WEBDEV. 
El archivo no se cargará si su tamaño excede el tamaño límite.
Para modificar el tamaño máximo de los archivos a cargar en WEBDEV servidor de aplicaciones, el registro debe ser modificado en el servidor:  

1. Para modificar el tamaño de los archivos cargados para todas las aplicaciones WEBDEV 28, seleccione la siguiente tecla: HKEY_LOCAL_MACHINE\\SOFTWARE\\PC SOFT\\WEBDEV\\28.0 en un ordenador de 32 bits y HKEY_LOCAL_MACHINE\\SOFTWARE\\WOW6432Node\\PC SOFT\\WEBDEV\\28.0 en un ordenador de 64 bits.

2. Añada la cadena MAX_UPLOAD. El valor de esta cadena será el tamaño máximo de los archivos a cargar (en KB).

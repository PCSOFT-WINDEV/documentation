


## &lt;Arranque (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia el envío de los archivos seleccionados a una control Upload. Al final de la carga, la lista de los archivos a enviar se borra en el navegador (sin importar si el control envía uno o más archivos).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// -- Event "Whenever modifying the list of selected files"
// in the UPL_Upload control
// Starts sending the file and transmits the content of EDT_Family control
UPL_Upload.Start(EDT_Family)
```

```wl
// -- Event "Receiving the uploaded files"
PROCEDURE Reception(FamilyName)
// Copies the file into the directory corresponding to the family
UPL_Upload.CopyFile(CompleteDir(sDestinationDirectory) + FamilyName, "")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Upload control>.Start([<Parameter 1> [... [, <Parameter N>]]])`
---

**`<Upload control>: (Nombre del control)`**

Nombre de la control Upload a utilizar.

**`<Parameter 1>: (Cualquier tipo)`**

Primer parámetro que debe ser pasado al servidor Event "Recepción de los archivos subidos". Para recuperar este parámetro, se debe declarar en la Event "Recepción de los archivos cargados" un procedimiento en el siguiente formato: 

```txt
PROCEDURE ProcedureName(<Parameter 1> [, ... [, <Parameter N>]])
```




**`<Parameter N>: (Cualquier tipo)`**

Nth parámetro que debe ser pasado al servidor Event "Recepción de los archivos subidos". Para recuperar este parámetro, se debe declarar en la Event "Recepción de los archivos cargados" un procedimiento en el siguiente formato: 

```txt
PROCEDURE ProcedureName(<Parameter 1> [, ... [, <Parameter N>]])
```

<a name="NOTE0_1"></a>

## Observaciones


### Modo de funcionamiento de la carga de archivos realizada a través de control Upload
<a name="modo_funcionamiento_carga_archivos_realizada_traves_control_upload_ELTPARAGRAPHE000201"></a>

Varios eventos son associated con el control Upload (de uno o varios archivos): 

- Para un control Upload de un solo archivo, estos eventos son associated con el botón "Enviar".. 

- Para un control Upload de varios archivos, estos eventos son associated con el botón "Add" (Añadir). 




Los eventos utilizados al subir los archivos seleccionados son: 

- Navegador Event **"Progreso de la transferencia"**: 
	Este Event se llama regularmente durante la carga para actualizar una barra de progreso, si está disponible.

- Servidor Event **"Recibiendo los archivos cargados"**: 
	Este Event se ejecuta cuando todos los archivos son recibidos por el servidor. 
	**Observación**: Si se llama a la función **&lt;Arranque** con parámetros adicionales, estos parámetros se transmiten al evento "Recepción de archivos subidos". Para recuperar estos parámetros, se debe declarar un procedimiento en esta Event( a través de la declaración de la Procedure).



<a name="NOTE0_2"></a>


### Límites
<a name="limites_ELTPARAGRAPHE000227"></a>

Las modificaciones realizadas en el servidor Event "Recepción de los archivos cargados" NO se llevan al navegador. Estas modificaciones serán Visible durante la próxima visualización de la página.
<a name="NOTE0_3"></a>


### Tamaño máximo de los archivos a cargar
<a name="tamano_maximo_los_archivos_cargar_ELTPARAGRAPHE000239"></a>

**By Default,** el tamaño de los archivos a subir no está limitado en el motor WEBDEV. Este tamaño puede ser limitado por el servidor de aplicaciones WEBDEV. 
El archivo no se cargará si su tamaño excede el tamaño límite.
Para modificar el tamaño máximo de los archivos a cargar en WEBDEV servidor de aplicaciones, el registro debe ser modificado en el servidor:  

1. Para modificar el tamaño de los archivos cargados para todas las aplicaciones WEBDEV 28, seleccione la siguiente tecla: HKEY_LOCAL_MACHINE\\SOFTWARE\\PC SOFT\\WEBDEV\\28.0 en un ordenador de 32 bits y HKEY_LOCAL_MACHINE\\SOFTWARE\\WOW6432Node\\PC SOFT\\WEBDEV\\28.0 en un ordenador de 64 bits.

2. Añada la cadena MAX_UPLOAD. El valor de esta cadena será el tamaño máximo de los archivos a cargar (en KB).

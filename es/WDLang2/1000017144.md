
## IdentifierAWPContext( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el identificador del contexto de AWP. Este identificador puede ser usado para construir un URL por ejemplo.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Adds the context into a built URL
LINK_Dyn = PageName + ".awp?client=" + Client + "&" + IdentifierAWPContext()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = IdentifierAWPContext()`
---

**`<Result>: (Cadena de caracteres ANSI)`**



- Identificador del contexto de AWP,

- Vaciar la cadena ("") si la función es inútil, en los siguientes casos por ejemplo:

	- La página actual no es una página de AWP.

	- La página actual es una página de AWP sin contexto.

	- La página actual es una página de Active WEBDEV con un contexto que no está almacenado en el disco.  

	- La transmisión es realizada por Cookie. 









<a name="NOTE0"></a>

## Observaciones
Si se utiliza el modo de transmisión Default: 

- Durante la primera visualización de la página, se utilizan los dos modos de transmisión (transmisión por Cookie y por url). La función **IdentifierAWPContext** devuelve el identificador del contexto AWP. 

- Durante las siguientes pantallas: 

	- Si las cookies están activadas, el identificador de contexto AWP se envía a través de las cookies y **IdentifierAWPContext** devuelve una cadena vacía (""). 

	- Si las cookies están desactivadas, el identificador de contexto de AWP se envía a través de las URL. La función **IdentifierAWPContext** devuelve el identificador del contexto AWP. 








### Sesiones preiniciadas
<a name="sesiones_preiniciadas_ELTPARAGRAPHE000053"></a>

Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".

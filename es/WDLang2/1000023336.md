


## LooperDeleteSelect (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina las filas seleccionadas de una control Looper.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Deletes all selected rows
IF YesNo("Do you really want to delete the selected customers?") THEN 
	LooperDeleteSelect(LOOP_Customer_Display)
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`LooperDeleteSelect(<Looper control>)`
---

**`<Looper control>: (Nombre del control)`**

Nombre del control Looper a utilizar.
Si este parámetro es igual a una cadena vacía (""), el borrado se realizará en la control Looper actual .



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000069"></a>

La función **LooperDeleteSelect** puede utilizarse en: 

- a control Looper basada en un archivo de datos, 

- a control Looper poblada programáticamente. 



<a name="NOTE0_2"></a>


### Supresión en una control Looper basada en un archivo de datos
<a name="supresion_una_control_looper_basada_archivo_datos_ELTPARAGRAPHE000083"></a>

La función **LooperDeleteSelect** elimina los registros correspondientes en el archivo de datos asociado al control. Si los elementos son relacionado para otros archivos de datos, no se realiza ninguna modificación automáticamente en estos archivos de datos de relacionado.
<a name="NOTE0_3"></a>


### Integridad referencial y control Looper basada en un archivo de datos
<a name="integridad_referencial_control_looper_basada_archivo_datos_ELTPARAGRAPHE000094"></a>

Si la gestión de integridad referencial está habilitada ( [HSetIntegrity](../WDLang4/3044058.md) ), [HErrorIntegrity](../WDLang4/3044086.md) debe estar llamado después de **LooperDeleteSelect** para verificar la integridad. Si se detecta un error de integridad, el fila no se borra y permanece Visible en el control.
<a name="NOTE0_4"></a>


### Supresión en una control Looper basada en un archivo de datos y Trigger
<a name="supresion_una_control_looper_basada_archivo_datos_trigger_ELTPARAGRAPHE000111"></a>

Se puede activar un disparador al borrar un Record de un control Looper basado en un archivo de datos. Para obtener más información, consulte la documentación de [HDescribeTrigger](../WDLang4/3044074.md).
<a name="NOTE0_5"></a>


### Gestión de errores
<a name="gestion_errores_ELTPARAGRAPHE000123"></a>

La variable [ErrorOccurred](../WDLang1/3087001.md) se establece en <u><u><u><u>True</u></u></u></u> si se produce un error durante la eliminación. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).
<a name="NOTE0_6"></a>


### Bloquear el archivo de datos de relacionado
<a name="bloquear_archivo_datos_relacionado_ELTPARAGRAPHE000142"></a>

Si el archivo de datos está bloqueado en el proceso actual, la función **LooperDeleteSelect** elimina la fila especificada y desbloquea el archivo de datos.

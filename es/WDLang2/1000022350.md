
## AppServerRunScheduledTask( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia inmediatamente la ejecución de una tarea programada en un servidor de aplicaciones WEBDEV. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Programs the task so that it is run every hour
MyScheduledTask is appServerScheduledTask
MyScheduledTask.Name = "MyTask"
MyScheduledTask.Procedure = MyProc
MyScheduledTask.Month = "*"
MyScheduledTask.DayOfMonth = "*"
MyScheduledTask.DayOfWeek = "*"
MyScheduledTask.Hour = "*"
MyScheduledTask.Minute = "0"

// Starts the task immediately
AppServerRunScheduledTask(MyScheduledTask)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = AppServerRunScheduledTask(<Scheduled task>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se inició la ejecución de la tarea programada. 

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Scheduled task>: (Tipo associated)`**

Tarea programada para ejecutar. Este parámetro puede corresponder a: 

- una variable de tipo [appServerScheduledTask](../WDLang2/1000022367.md). En este caso, sólo se tiene en cuenta el nombre de la tarea. Los parámetros de la tarea son los especificados al crear la tarea con [AppServerAddScheduledTask](../WDLang2/1000022347.md).

- un nombre de Procedure. La tarea programada asociada con el nombre de este Procedure se ejecutará. 

- el nombre de una tarea programada. Se ejecutará la tarea programada correspondiente. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- La tarea se ejecuta en una sesión independiente (como todas las demás tareas programadas).

- La tarea no se ejecuta si ya está en marcha.

- La tarea se ejecuta incluso si la tarea está desactivada.

- La tarea programada no puede ejecutarse durante una prueba del proyecto. El sitio debe ser desplegado. 

- Si se produce un error al iniciar o ejecutar la tarea, el error no será transmitido por la función **AppServerRunScheduledTask**. El error tendrá que ser Process ed en el Process, en un archivo LOG por ejemplo.

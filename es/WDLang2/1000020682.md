


## ExecuteDelayedProcedure( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Dirige un procedimiento diferido (en el WEBDEV servidor de aplicaciones).

Atención: Esta función sólo puede ser utilizada para ejecutar los procedimientos retrasados associated con las tareas retrasadas WEBDEV. No se puede utilizar para ejecutar tareas programadas de WEBDEV.. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// The CalcTO procedure was defined 
// as a delayed procedure in the code editor
ExecuteDelayedProcedure(CalcTO, 0)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`ExecuteDelayedProcedure(<WLanguage procedure> [, <Parameter 1> [... [, <Parameter N>]]])`
---

**`<WLanguage procedure>: (Nombre de Procedure)`**

Nombre del procedimiento WLanguage a ejecutar. Este Procedure debe haber sido definido como procedimiento diferido en el editor de código. Para obtener más información, consulte [Retraso en los procedimientos](../WDLang2/1000019455.md).

**`<Parameter 1>: (Tipo de parámetro)`**

Primer parámetro asociado con el Procedure.



**`<Parameter N>: (Tipo de parámetro)`**

El enésimo parámetro asociado a la Procedure.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- La ejecución de la Procedure puede fallar si la Procedure ya se ejecuta en la tarea de Background.

- Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".

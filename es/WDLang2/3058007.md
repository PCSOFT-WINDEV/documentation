


## ChangeAction( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Se utiliza para especificar la acción a realizar cuando la página HTML mostrada en el navegador ya no está sincronizada con el contexto de la página en el servidor.

Por Default, en caso de páginas no sincronizadas, se devuelve la última versión de la página HTML correspondiente al contexto actual en el servidor.

**Atención**: Esta función se ejecuta en el código de sincronización de la página.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000021651&name=changeaction_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Synchronization code of PAGE_Summary
// (The action does not have to be synchronized)
ChangeAction(caContinue)
```

## Sintaxis

`<Result> = ChangeAction(<Action>)`
---

**`<Result>: (Constante de tipo Integer)`**

Anterior modo de gestión de sincronización (antes de llamar a la función **ChangeAction**):



|   |   |
| --- | --- |
| *caContinue* | La aplicación continúa como si se hubiera realizado una sincronización. <br><br>Se ejecutará la acción solicitada por el usuario de la Web. |
| *caError* | Se devuelve la última versión de la página HTML correspondiente al contexto actual en el servidor.. <br>Un cuadro de diálogo informa al usuario de que se ha producido un error de sincronización.. <br>La acción solicitada por el usuario web no se ejecutará.<br><br>Este modo de gestión es el de Default. |
| *caNone* | No se realiza ninguna acción. La acción solicitada por el usuario Web no se ejecuta.<br>Se producirá un error si el Process para la sincronización de páginas utiliza esta opción sin llamar a [PageDisplay](../WDLang2/3058008.md) o [PageRefresh](../WDLang2/3058011.md). Este error indica que el servidor no ha devuelto ninguna página (ERR_NO_CURRENT_PAGE). |



**`<Action>: (Constante de tipo Integer)`**

Acción a realizar si el contexto y la página no están sincronizados: 



|   |   |
| --- | --- |
| *caContinue* | La aplicación continúa como si se hubiera realizado una sincronización. <br><br>Se ejecutará la acción solicitada por el usuario de la Web.<br><br>Observación: Antes de utilizar esta opción, asegúrese de que la Record actual es la esperada para evitar sobrescribir los datos. |
| *caError* | Se devuelve la última versión de la página HTML correspondiente al contexto actual en el servidor.. <br>Un cuadro de diálogo informa al usuario de que se ha producido un error de sincronización.. <br>La acción solicitada por el usuario web no se ejecutará.<br><br>Este modo de gestión es el de Default. |
| *caNone* | No se realiza ninguna acción. La acción solicitada por el usuario Web no se ejecuta.<br>Se recomienda devolver una página con [PageDisplay](../WDLang2/3058008.md) o [PageRefresh](../WDLang2/3058011.md) antes de utilizar **ChangeAction** con la constante *caNone*. De lo contrario, se producirá un error que indica que el servidor no ha devuelto ninguna página (ERR_NO_CURRENT_PAGE). |

## Observaciones


### Sincronización 
<a name="sincronizacion_ELTPARAGRAPHE000104"></a>

Cada vez que el usuario hace clic en una página HTML, se realiza automáticamente una prueba de sincronización.

Esta prueba de sincronización consiste en comprobar que la página mostrada por el navegador corresponde al contexto de la página actual en el servidor.

Los problemas de sincronización pueden ocurrir cuando se utiliza la tecla BACK o la tecla REFRESH en el navegador del usuario de la Web.
La sincronización de páginas se gestiona sólo en uno de los siguientes casos: 

- En la pestaña "UI" de la ventana de la página description, la opción "Uso del botón "Atrás" del navegador" corresponde al valor "Permitido (ejecuta el código de sincronización)". 

- En la pestaña "Avanzado" de la ventana del proyecto description, la opción "Opción predeterminada para utilizar el botón "Atrás" del navegador" corresponde al valor "Permitido (ejecuta el código de sincronización)".

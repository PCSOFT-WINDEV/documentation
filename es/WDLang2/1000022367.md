
## appServerScheduledTask (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **appServerScheduledTask** permite administrar una tarea programada del Servidor de Aplicaciones WEBDEV mediante programación (para un sitio o un Webservice).

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Programs the task so that it is run every hour
MyScheduledTask is appServerScheduledTask
MyScheduledTask.Name = "MyTask"
MyScheduledTask.Procedure = MyProc
MyScheduledTask.Month = "*"
MyScheduledTask.DayOfMonth = "*"
MyScheduledTask.DayOfWeek = "*"
MyScheduledTask.Hour = "*"
MyScheduledTask.Minute = "0"

IF NOT AppServerAddScheduledTask(MyScheduledTask) THEN
	// Error case (if a task with the same name already exists for example)
END
```

## Observaciones
<a name="NOTE0_1"></a>


### Propiedades específicas de las variables appServerScheduledTask
<a name="propiedades_especificas_las_variables_appserverscheduledtask_ELTPARAGRAPHE000029"></a>

Las siguientes propiedades se pueden utilizar para programar una tarea de manipular:

| Nombre de la propiedad | Tipo utilizado | Efecto |
| --- | --- | --- |
| Contraseña | Cadena de caracteres | Contraseña de usuario. |
| DayOfMonth | Cadena de caracteres | Número del día para el que se debe ejecutar la tarea programada (por ejemplo,"31").. Este número se incluye entre 0 y 31.<br><br>Este Property también puede corresponder:<br><br>- una lista de números separados por comas: usado para especificar varios meses.<br><br>- a interval. Por ejemplo, "9-13" (la tarea se programa del 9 al 13 del mes).<br><br>- una estrella ("\*" ") para especificar el rango completo.<br><br>- un intervalo regular, por ejemplo "\*/2" para iniciar la tarea los días impares del mes.<br><br><br> |
| DayOfMonthOrDayOfWeek | booleano | Gestionar la ejecución de una tarea programada especificando el número de día de la semana o el número de día del mes. <br><br>Ejemplo: ejecución cada lunes 1er día del mes.<br><br>Esta propiedad puede corresponder a:<br><br>- <u><u><u><u>True</u></u></u></u> para ejecutar la tarea programada todos los lunes Y el primer día del mes (condiciones acumuladas).<br><br>- <u><u><u><u>False</u></u></u></u> para ejecutar una tarea solo los lunes 1.º (condiciones exclusivas).<br><br><br> |
| DayOfWeek | Cadena de caracteres | Número del día para el que se debe ejecutar la tarea programada (por ejemplo,"2").. Este número se incluye entre 1 y 7 (1 = Lunes, 2 = Martes, ....).<br><br>Este Property también puede corresponder:<br><br>- una lista de números separados por comas: usado para especificar varios días.<br><br>- a interval. Por ejemplo, "1-3" (la tarea está programada para el lunes, martes y miércoles).<br><br>- una estrella ("\*" ") para especificar el rango completo.<br><br>- un intervalo regular, por ejemplo "\*/2" para iniciar la tarea los días pares de la semana.<br><br>- el día de la semana en letras ("lunes", "martes", "miércoles", "jueves", "viernes").<br><br><br> |
| denominar | Cadena de caracteres | Nombre de la tarea. <br><br>Este nombre se utiliza para diferenciar las tareas asociadas con el mismo Procedure. <br><br>Si esta Property no se llena, el nombre de la tarea corresponderá al nombre completo de la Procedure (por ejemplo: MySet.procedureName). |
| drogadicta | Cadena de caracteres | Indica el usuario del sistema operativo que ejecutará la tarea.. Si no se rellena este valor, el usuario que se utilizará es el usuario de Default de la aplicación (sitio o servicio web). |
| En proceso | booleano | Indica si la tarea se está ejecutando actualmente: <br><br>- <u><u><u><u>True</u></u></u></u> si se está ejecutando la tarea.<br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario.<br><br><br>**Esta propiedad es de solo lectura**.<br><br>Esta Property es actualizada únicamente por [AppServerInfoScheduledTask](../WDLang2/1000022789.md) y [AppServerListScheduledTask](../WDLang2/1000022352.md). El valor leído refleja el estado durante la llamada a estas funciones (y no el estado cuando se lee el Property). |
| Habilitado | booleano | Indica si la tarea está habilitada o no: <br><br>- <u><u><u><u>True</u></u></u></u> si la programación de la tarea está activa. <br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario.<br><br><br> |
| Hora | Cadena de caracteres | Tiempo de ejecución de la tarea programada. Si **TimeUTC** es set a <u><u><u><u>True</u></u></u></u>, esta hora estará en UTC.<br><br>Los valores posibles son:<br><br>- un dígito comprendido entre 0 y 23.<br><br>- una lista de números separados por comas: usado para especificar varias horas.<br><br>- a interval. Por ejemplo, "9-13".<br><br>- una estrella ("\*" ") para especificar el rango completo.<br><br>- un intervalo regular, por ejemplo "\*/2" para iniciar la tarea cada hora par.<br><br><br> |
| Mes | Cadena de caracteres | Número del mes de ejecución de la tarea programada (por ejemplo,"11").<br><br>Los valores posibles son:<br><br>- un dígito comprendido entre 1 y 12<br><br>- una lista de números separados por comas: usado para especificar varios meses.<br><br>- a interval. Por ejemplo, "9-11".<br><br>- una estrella ("\*" ") para especificar el rango completo.<br><br>- un intervalo regular, por ejemplo "\*/2" para iniciar la tarea los meses impares.<br><br><br> |
| Minuto | Cadena de caracteres | Minutos del tiempo de ejecución de la tarea programada. Si **TimeUTC** es set a <u><u><u><u>True</u></u></u></u>, esta hora estará en UTC.<br><br>Los valores posibles son:<br><br>- un dígito comprendido entre 0 y 59.<br><br>- una lista de números separados por comas: se utiliza para especificar varios minutos.<br><br>- a interval. Por ejemplo, "9-13".<br><br>- una estrella ("\*" ") para especificar el rango completo.<br><br>- un intervalo regular, por ejemplo "\*/2" para iniciar la tarea cada minuto par.<br><br><br> |
| Nombre del procedimiento | Cadena de caracteres | Nombre del procedimiento a ejecutar. <br><br>Este Property es de sólo lectura y se utiliza para recuperar el nombre del Procedure associated con la tarea (para mostrarlo en una pantalla específica, por ejemplo). |
| paramétrico | Array | Parámetros que se transmitirán a la tarea programada durante su ejecución. <br><br>Cada uno de los parámetros debe ser transformado por [serialize](../WDLang1/3013065.md) (se utiliza la serialización binaria). El servidor de aplicaciones WEBDEV necesita guardar los parámetros para pasarlos en cada llamada. Para ello, se serializará el valor de cada parámetro. <br><br>Observación: Cuando se ejecute la tarea programada, el servidor de aplicaciones WEBDEV pasará automáticamente los parámetros a Procedure. Por lo tanto, un mismo Procedure puede ser associated con múltiples tareas programadas que pasarán diferentes parámetros. |
| Procedure | Nombre de Procedure | Procedure que se ejecutará según la tarea programada. Este nombre tiene el siguiente formato:<br><br>- para un Procedure encontrado en un conjunto de Procedure s: &lt;Nombre del conjunto de Procedure s&gt;.&lt;Nombre de Procedure&gt;.<br><br>- para una Procedure encontrada en una Component: &lt;Nombre del componente&gt;.&lt;Nombre del conjunto de Procedure s&gt;.&lt;Nombre de Procedure&gt;.<br><br><br><br><br>Este Property no acepta:<br><br>- Los procedimientos locales. Un Procedure local sólo existe en el contexto de la llamada a su padre Procedure. Este contexto ya no existirá durante la llamada a la tarea programada.<br><br>- Una variable [Procedure](../Motscles/1000018907.md) (excepto si ésta identifica un Procedure perteneciente a un conjunto de Procedure s).<br><br><br> |
| TiempoUTC | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> (valor Default) si la hora está en UTC (propiedades **Hora** y **Minuto**). <br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario.<br><br><br> |

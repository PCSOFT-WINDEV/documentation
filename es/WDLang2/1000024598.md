
## PageChangeLayout (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Aplica un layout específico a una página. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Forces the use of first layout if the width of screen is set to 320
IF MySelf.Width = 320
	PageChangeLayout(1)
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Forzar el layout en la página actual

`PageChangeLayout(<Layout number>)`
---

**`<Layout number>: (Entero)`**

Número del layout a aplicar. 


<a name="SYNTAX2"></a>

### Forzar el layout en una página específica

`PageChangeLayout(<Page> , <Layout number>)`
---

**`<Page>: (Nombre de la página)`**

Nombre de la página a utilizar. 

**`<Layout number>: (Entero)`**

Número del layout a aplicar. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- El número del layout se indica en la barra de título del layout. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Agencement_Numero.gif&type=thumb)


- Los tamaños, posiciones y visibilidad de los controles se modifican de acuerdo con el layout definido durante la edición. 

- El layout inicial se elige automáticamente en función de la plataforma, orientación y tamaño de la pantalla. 

- El layout utilizado se modifica automáticamente cuando el dispositivo cambia del modo vertical al modo horizontal. 

- Esta función no ejecuta el evento "Cada vez que cambia el layout" de la página. Para forzar la ejecución de este evento, utilice la función [ExecuteProcess](../WDLang1/3013031.md) con la constante *trtChangeLayout*.

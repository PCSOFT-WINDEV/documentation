


## ScriptDisplay( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Llama a un script o página externa (.php,.asp,.mhtml o.mht) y devuelve la página de resultados en la ventana actual del navegador.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Runs the "Script1.php" script found at the root of the Web site
// and returns the result to the Web user
ScriptDisplay("/Script1.php")
```
<a name="Example2"></a>

```wl
// Go back to the ASP initialization page of the site
ScriptDisplay("/localstart.asp")
```
<a name="Example3"></a>

```wl
// Login page in asp. Sends 3 parameters with the GET method
ScriptDisplay("/ASP/Login.asp", "Name=BOB&Pass=My+Secret&Lang=3")
```
<a name="Example4"></a>

```wl
// Displays an external site
ScriptDisplay("http://www.windev.com/")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`ScriptDisplay(<Script name> [, <"Name1=Param1&Name2=Param2&..."> [, <Redirection>]])`
---

**`<Script name>: (Cadena de caracteres)`**

Ruta y nombre del script. La ruta puede ser: 

- una URL completa "http://..."

- una ruta relativa a la raíz del sitio "/...". En este caso, el script y el sitio Web deben encontrarse en el mismo servidor, en uno de los subdirectorios del sitio.




**`<"Name1=Param1&Name2=Param2&...">: (Cadena de caracteres)`**

Parámetros destinados al script. Esta cadena contiene para cada parámetro:

- el nombre del parámetro ("Name1" por ejemplo).

- el signo "=".

- el valor del parámetro (por ejemplo,"Parámetro1").




Dos parámetros están separados por el signo'&'.

**`<Redirection>: (booleano)`**

Permite definir el código de retorno HTTP de la función al navegador.. Este código de retorno se utiliza principalmente para la referenciación por los motores de búsqueda. Este parámetro puede corresponder a: 

- <u><u><u><u>True</u></u></u></u>: redirección permanente (código HTTP 301). La página mostrada por la función se referenciará directamente.

- <u><u><u><u>False</u></u></u></u> (valor predeterminado): redirección temporal (código HTTP 302). Sólo se hará referencia a la página de origen. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Visualización de otros ficheros
<a name="visualizacion_otros_ficheros_ELTPARAGRAPHE000060"></a>

La función **ScriptDisplay** también puede utilizarse en un documento que no sea un script. En este caso, se devuelve la Document (equivalente a la [FileDisplay](../WDLang2/3012005.md), especificando una URL en lugar de una Address física).
<a name="NOTE0_2"></a>


### Codificación de los parámetros de ScriptDisplay
<a name="codificacion_los_parametros_docparampagetitleshort_ELTPARAGRAPHE000072"></a>

El contenido de los parámetros de **ScriptDisplay** se codifica automáticamente para ser interpretado por los navegadores:

- transformación en UTF-8 (según las opciones de la página y del proyecto).

- codificación de caracteres especiales.


Por lo tanto, no es necesario utilizar la función [URLEncode](../WDLang1/3024040.md) para codificar uno de los parámetros de la función **ScriptDisplay**. 
<a name="NOTE0_3"></a>


### Sesiones preiniciadas
<a name="sesiones_preiniciadas_ELTPARAGRAPHE000094"></a>

Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".

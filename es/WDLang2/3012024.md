


## Subir archivos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000170"></a>
La carga consiste en guardar un archivo situado en el ordenador del usuario en el servidor web. El archivo encontrado en el ordenador del usuario se "sube" al servidor: se carga y guarda en el servidor.

Se pueden subir todo tipo de archivos.

Ejemplos de uso de la carga de archivos:

- **Phone Sitio del libro**: cuando se añade una nueva Contact al libro Phone, el usuario puede seleccionar la foto Contact en su ordenador. Para que la foto se muestre independientemente del usuario de la Web, la foto se "sube" al servidor.

- **Site para la gestión de Document**: cada usuario de la Web puede poner a disposición de los demás usuarios de la Web diferentes tipos de documentos: texto, ... Estos documentos se seleccionan en la el equipo del usuario de la Web y se "suben" al servidor para que estén disponibles para todos los usuarios de la Web.

## Elementos disponibles
<a name="elementos_disponibles_ELTTEXTE000194"></a>
Los siguientes elementos se utilizan para gestionar la carga de archivos:

- **un control Campo de entrada de tipo UPLOAD**: En el navegador, un botón "BROWSE" se añade automáticamente a la derecha de la ventana de carga de control Campo de entrada. Este botón permite al usuario de la Web seleccionar el archivo que desea"cargar".. 
	Se muestra un botón gris en el editor. Este botón no se puede modificar (estilo, título, ....).
	**Atención**: esta control Campo de entrada es una control específica: no se deben utilizar propiedades de valor en esta control. Para get el nombre del archivo cargado, utilice [SubirNombreDeArchivo](../WDLang2/3012022.md). El modo de carga con este tipo de control se presenta en [Carga a través de un control Campo de entrada](#NOTE3_1). 

- **an control Upload (archivo único o archivo múltiple)**: Varios controles se insertan automáticamente en el editor, así como el código WLanguage necesario para utilizarlos inmediatamente.. Esta control puede ser generada para una ejecución basada en Flash o en HTML5. Para obtener más información, consulte [Control Upload](../WDChamp/1016330.md).

- **dos funciones de servidor de WLanguage**:
	


|   |   |
| --- | --- |
| [SubirNombreDeArchivo](../WDLang2/3012022.md) | Devuelve el nombre de un archivo "cargado" (nombre de archivo inicial o nombre de archivo guardado en el servidor). |
| [UploadCopyFile](../WDLang2/3012023.md) | Guarda un archivo "cargado" en el servidor. Durante esta copia de seguridad, el archivo guardado se puede renombrar en el servidor. |



- se utilizan varias funciones del navegador WLanguage para manipular the control Upload: 
	


|   |   |
| --- | --- |

| [UploadCurrentFileSizeSizeSent](../WDLang2/1000018796.md) | Devuelve el tamaño (en bytes) ya enviado para el archivo actualmente cargado a través de un control Upload. |
| [UploadDelete](../WDLang2/1000018808.md) | Elimina un archivo de la lista de archivos a cargar: el archivo no se cargará en el servidor. |

## Upload a través de un Upload control Campo de entrada
<a name="upload_traves_upload_control_campo_entrada_ELTTEXTE000260"></a>


### Cargar un fichero en una Page
<a name="cargar_fichero_una_page_ELTPARAGRAPHE000068"></a>

Para cargar un fichero en un Page:

1. Cree un control Campo de entrada de tipo UPLOAD (el tipo del control se define en la pestaña "General" del control). Esta control permite al usuario de la web seleccionar el archivo a "cargar".

2. Añade una control Botón o una control Enlace. Esta control se utilizará para cargar el archivo en el servidor. **Este Botón o control Enlace debe ser de tipo "Submit"** ("Acción del Botón" en la pestaña "General" de la ventana de control de description). Cuando la página se valida con esta control, el archivo se carga en un directorio temporal del servidor. Por lo tanto, este archivo no puede ser visto por los demás usuarios de la Web.

3. En el código de clic del servidor de este Botón o control Enlace, puedes usar: 

	- [SubirNombreDeArchivo](../WDLang2/3012022.md) (UploadFileName(&lt;control Upload&gt;, False)) para comprobar si se ha cargado el archivo y recuperar su nombre. De hecho, durante la carga, el archivo se renombra con un nombre de archivo temporal.

	- [UploadCopyFile](../WDLang2/3012023.md) a copy y cambiar el nombre del archivo cargado en el servidor. Este archivo puede ser copiado en el directorio de Resource (_WEB) o en un directorio accesible a través de un alias.

	- [HLinkMemo](../WDLang4/3044173.md) para cargar el Image en un elemento de memoria de un archivo HFSQL.





**Atención**: Utilizar el nombre del archivo directamente en el ordenador del navegador (valor leído por el [Valor](../Proprietes/2510130.md) Property en el código del servidor o del navegador) en las funciones del archivo es un gran error. De hecho, esta operación funciona en modo de desarrollo (el servidor y el navegador se encuentran en el mismo ordenador) pero NO en implementación. En este caso, el servidor y el navegador son ordenadores diferentes. El servidor no ve los archivos del navegador. Para get los nombres de los archivos a utilizar, utilice [SubirNombreDeArchivo](../WDLang2/3012022.md).

### Mostrando la Image para subir
<a name="mostrando_image_para_subir_ELTPARAGRAPHE000097"></a>

Para mostrar la Image previamente seleccionada con un Upload control Campo de entrada (EDT_UploadControl) en un control Imagen (IMG_Image Control):

1. En el código de la Page, añada el código opcional del navegador "OnMouseMove".

2. Introduzca las siguientes líneas de código en este código:
	
	```wl
	IF EDT_UploadControl <> "" THEN
		IF IMG_ImageControl <> EDT_UploadControl THEN
			IMG_ImageControl = EDT_UploadControl
		END
	END
	```



**Observación**: Esta operación puede no funcionar dependiendo del navegador utilizado y del nivel de seguridad.

## Observación
<a name="observacion_ELTTEXTE000290"></a>


### Tamaño máximo de los archivos a cargar
<a name="tamano_maximo_los_archivos_cargar_ELTPARAGRAPHE000113"></a>

**Por defecto**el tamaño de los archivos a cargar no está limitado en el motor de WEBDEV. Este tamaño puede ser limitado: 

- por el servidor web utilizado. Consulte la documentación sobre el servidor web utilizado para obtener más información.. 

- por WEBDEV servidor de aplicaciones. 




El archivo no se cargará si su tamaño excede el tamaño límite.

**To modificar el tamaño máximo de los archivos a subir en el WEBDEV servidor de aplicaciones**, el registro debe ser modificado en el servidor. Esta modificación se puede realizar para todas las aplicaciones WEBDEV.  

**Para modificar el tamaño de los archivos cargados para todas las aplicaciones WEBDEV**: 

1. Seleccione la siguiente clave de registro: 

	- En un equipo de 64 bits: HKEY_MÁQUINA_LOCAL\\SOFTWARE\\WOW6432Node\\PC SOFT\\WEBDEV\\28.0

	- En una computadora de 32 bits: HKEY_MÁQUINA_LOCAL\\SOFTWARE\\PC SOFT\\WEBDEV\\28.0




2. Añadir la clave MAX_UPLOAD. El valor de esta clave corresponde al tamaño máximo de los archivos a cargar (en KB).


Para una gran Transfer( varios cientos de MB), recomendamos que utilice el protocolo FTP en lugar de la carga. Tiene la posibilidad de realizar una transferencia FTP desde un Page WEBDEV , a través de un applet Java creado con WINDEV.

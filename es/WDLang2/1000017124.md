


## PageDisplayDialog (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Muestra una página en modo modal. Esta función se utiliza para establecer un diálogo con el usuario. La página se muestra en primer plano mientras que la página de apertura se muestra en el fondo, Grayed por el mecanismo OVI. Para más detalles, véase "Visualización del resultado". 

El código de la página está bloqueado hasta que la página sea cerrada por [Diálogo PageCloseDialog](../WDLang2/1000017166.md). La página puede devolver un valor.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Displays a page allowing the user to exit from the site
IF PageDisplayDialog(PAGE_Exit) = True THEN 
	// Displays an end page
	PageDisplay(PAGE_End)
	// Close the session
	EndProgram()
END
```
<a name="Example2"></a>

```wl
// Displays a page to validate the information entered in the form
PageDisplayDialog(PAGE_Info, EDT_LastName, EDT_FirstName, EDT_Email_Address)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = PageDisplayDialog(<Page> [, <Parameter 1> [... [, <Parameter N>]]])`
---

**`<Result>: (Tipo correspondiente al valor recuperado (opcional))`**

Valor recuperado al cerrar la página para abrirla. Este valor es devuelto:

- por la palabra clave [Devuelve](../Motscles/1510007.md) en el código de cierre de la página,

- por la función [Diálogo PageCloseDialog](../WDLang2/1000017166.md),

- por el [ReturnedValue](../Proprietes/2510134.md) Property utilizado antes de cerrar la página,

- por la función [ContextoCerrar](../WDLang2/3058003.md).




&lt;Resultado&gt; corresponde a <u><u><u><u>False</u></u></u></u> y se muestra un error si el usuario presiona el botón "Atrás" del navegador.

**`<Page>: (Nombre de la página)`**

Nombre de la página a abrir. Esta página debe ser una página dinámica no AWP que no utilice código AJAX.

**`<Parameter 1>: (Tipo de valor enviado a la página (opcional))`**

Primer parámetro que se pasará a las "Declaraciones Globales" Event de la página para abrir. Este parámetro es pasado por reference y es considerado como un global de Variable a la página.



**`<Parameter N>: (Tipo de valor enviado a la página (opcional))`**

Nth parámetro que debe ser pasado a las "Declaraciones Globales" Event de la página para abrir. Este parámetro es pasado por reference y es considerado como un global de Variable a la página.

## Observaciones


### Visualización resultante
<a name="visualizacion_resultante_ELTPARAGRAPHE000105"></a>

La pantalla es la siguiente: la página mostrada con la función **PageDisplayDialog** se muestra en primer plano.

Sólo se muestra el Border y el contenido de la página: Esta página no tiene barra de título, ni barra de tareas, ... Independientemente de su Color, la página Background es ignorada.

La página que utilizó **PageDisplayDialog** se encuentra en el Background. Esta página es Grayed de OVI (Dim Disabled Pages). La función OVI puede ser configurada por [PáginaActivarDW](../WDLang2/1000017087.md) y [PáginaTasaDDDDWW](../WDLang2/1000017086.md).

**Observación**: La página abierta no se centra automáticamente: el modo de visualización de la página debe estar definido en sus características. Le aconsejamos que incluya una gran parte de la página en una celda con un fondo Color para evitar los efectos de visualización causados por la función OVI.


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000128"></a>

- La página mostrada debe ser una página dinámica no AWP. 

- La función **PageDisplayDialog** debe utilizarse en una página dinámica que no sea AWP. 

- Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".

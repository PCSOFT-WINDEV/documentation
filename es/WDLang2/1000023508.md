


## &lt;InfoXY (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Se utiliza para averiguar el elemento situado en una posición determinada en la control Looper (X, Y). Este elemento puede ser: 

- a fila,

- un control, 

- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) un descanso.




Esta función se puede utilizar en los controles de Looper basados en un archivo de datos o rellenarlos de forma programada.  






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Click on the left mouse button
x is int = MouseXPos() 
y is int = MouseYPos() 
nRow is int = LOOP_Looper1.InfoXY(liLineNumber, x, y) 
Trace("You've clicked the row: " + nRow)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Looper control>.InfoXY(<Type of information> , <X> , <Y>)`
---

**`<Result>: (Cadena de caracteres o entero)`**



- Información solicitada.

- Una cadena vacía ("") si no hay ningún control en la ubicación especificada.

- -1 si no se encuentra ningún número de fila en la ubicación especificada.




**`<Looper control>: (Nombre del control)`**

Nombre del control Looper a utilizar.

**`<Type of information>: (Constante)`**

Tipo de información solicitada:


|   |   |
| --- | --- |
| *liBreakName* | Nombre de la ruptura en (x,y).<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta constante no está disponible. |
| *liControlName* | Nombre de la control que se encuentra en (x, y). |
| *liLineNumber* | Número de la fila encontrada en (x, y). |
| *liOriginScreen* | Por Default, (0,0) corresponde a fila 1, columna 1 (las barras de desplazamiento se encuentran en el origen).<br>Si la constante *liOriginScreen* se combina con las constantes anteriores, el punto (0,0) corresponde al origen de la pantalla.<br>Esta constante no puede utilizarse por sí sola. |



**`<X>: (Integro)`**

Coordenada X (en pixeles) a analizar. Esta coordenada es relativa al control Looper (si no se especifica la constante *liOriginScreen*).

**`<Y>: (Integro)`**

Coordenada Y (en pixeles) a analizar. Esta coordenada es relativa al control Looper (si no se especifica la constante *liOriginScreen*).




## PreviousPage( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el nombre de la página anterior. La página anterior es la página que abrió la página actual.

En modo AWP, **PreviousPage** siempre devuelve una cadena vacía ("").

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Str is string
Str = PreviousPage()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = PreviousPage()`
---

**`<Result>: (Cadena de caracteres)`**

Nombre de la página anterior en mayúsculas si el código WLanguage que se ejecuta actualmente lo es:

- un código de página (código de apertura, código de cierre o código de sincronización)

- un código control (inicialización, clic)

- un Procedure local a la página




O una cadena vacía (""") si la página es una página estática o si el código que se está ejecutando actualmente lo es:

- el código del proyecto.

- un constructor de clase o código de destructor.

- un método de clase.

- un procedimiento global al proyecto.


![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) El caso del nombre de la página corresponde al caso real del nombre de la página. Por ejemplo, si la página actual corresponde a la página "Home.WWH", el nombre devuelto será "Home".



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- **PreviousPage** puede utilizarse para gestionar la visualización de un botón BACK en una página WEBDEV. Para ello, especifique la siguiente Line en el código de clic del servidor del botón:



```wl
PageDisplay(PreviousPage())
```


- Páginas preparadas para el SEO: Si utiliza páginas preparadas para SEO en el modo de sesión, no debe llamar a **PreviousPage**. Dado que es posible acceder a una página preparada para SEO desde cualquier otra página, la página "anterior" no existirá.




## PageRefresh (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Actualiza una página mostrada en el navegador del usuario a partir del contexto en el servidor. Esta función se utiliza principalmente para los sitios en el modo Sesión. 

**Atención**: El contexto de la página debe existir en el servidor. El contexto de la página es creado por [PageDisplay](../WDLang2/3058008.md) y [ContextoAbrir](../WDLang2/3058004.md). Todos los eventos relacionado a la página o a los controles ya han sido ejecutados. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Refresh PAGE_Page1
PageRefresh(PAGE_Page1)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`PageRefresh([<Page>])`
---

**`<Page>: (Nombre de la página)`**

Nombre de la página de actualización.
Si este parámetro no se especifica o es una cadena vacía (""), el nombre de la página corresponde a la página relacionada con la Event actual.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Utilización de PageRefresh en un Event en modo AJAX
<a name="utilizacion_docparampagetitleshort_event_modo_ajax_ELTPARAGRAPHE000066"></a>

**Recordatorio**: AJAX se utiliza para actualización los datos modificados en una página HTML sin volver a mostrar toda la página. Para obtener más información, consulte [AJAX](../WDLang2/3055101.md).

Si **PageRefresh** se utiliza en un Event en modo AJAX (se muestra "AJAX habilitado" en la barra de Event) y si este Event se utiliza para actualización única o sistemáticamente una página, la tecnología AJAX se vuelve inútil. En el modo AJAX, **PageRefresh** debe reservarse para casos especiales.

Durante una Process larga, [AJAXExecuteAsynchronous](../WDLang2/3055115.md) permite mostrar un mensaje de espera. 


### Sesiones preiniciadas
<a name="sesiones_preiniciadas_ELTPARAGRAPHE000091"></a>

Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".

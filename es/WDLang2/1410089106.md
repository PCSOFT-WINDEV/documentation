
## SessionRequestedPage( función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el nombre de la Page solicitada al inicio de la sesión.

Observación: Esta función es particularmente útil en el código del proyecto, a get el nombre de la Page que se devolverá. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
SWITCH (SessionPageRequested()) 
CASE "PAGE_Admin"
	// Checks the connection parameters on the URL
	IF NOT ConnectUser() THEN
		PageDisplay(PAGE_LoginError)
	END
OTHER CASE
	// Nothing
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = SessionRequestedPage()`
---

**`<Result>: (Cadena de caracteres)`**



- Nombre de la página. A continuación figuran los distintos casos y el nombre Page correspondiente: 

	- Conexión a un sitio en modo Sesión a través de una Page SEO-ready : nombre de la SEO-ready Page.

	- Conexión a un sitio en modo Sesión utilizando [DynamicSiteDisplay](../WDLang2/3058024.md) y especificando un Page: nombre de la Page (sólo si la opción de acceso correspondiente está activada en la Page).

	- Conexión a un sitio en modo Sesión desde una aplicación WINDEV que solicita una Page determinada. : nombre de la Page (sólo si la opción de acceso correspondiente está activada en la Page).

	- Visualización de un Active WEBDEV Page: nombre de la Active WEBDEV Page.




- Cadena vacía ("") en otros casos, o si la función se llama después de la petición de inicio de sesión.

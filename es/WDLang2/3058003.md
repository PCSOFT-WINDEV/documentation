


## ContextClose( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cierra un contexto de página. En algunos casos específicos (User Groupware, etc.), tiene la posibilidad de especificar el valor a devolver.

Atención: No se recomienda cerrar el contexto de la página actual.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
ContextClose(PAGE_Page4)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Cerrando el contexto de la página

`ContextClose(<Page>)`
---

**`<Page>: (Nombre de la página)`**

Nombre de la página cuyo contexto debe ser cerrado. Si este parámetro es igual a una cadena vacía (""), el nombre de la página corresponde a la página a la que pertenece el actual Process.  


<a name="SYNTAX2"></a>

### Cerrando el contexto de la página y devolviendo la información

`ContextClose(<Page> , <Return value>)`
---

**`<Page>: (Nombre de la página)`**

Nombre de la página cuyo contexto debe ser cerrado. Si este parámetro es igual a una cadena vacía (""), el nombre de la página corresponde a la página a la que pertenece el actual Process.

**`<Return value>: (Tipo de valor devuelto)`**

Valor devuelto a la página de llamada. Este parámetro está reservado para el uso de páginas específicas de WEBDEV que se abren automáticamente (páginas de User Groupware, etc.). El valor de este parámetro se ignora si se utiliza para una página que no espera ningún valor de retorno..  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Sesiones preiniciadas
<a name="sesiones_preiniciadas_ELTPARAGRAPHE000050"></a>

Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".

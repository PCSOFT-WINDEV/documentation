


## LooperSort (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite:

- Clasificar un control Looper según uno o más atributos.

- Clasificar una control Looper en todos sus atributos.

- Cancel un tipo que se realizó de antemano (lo que significa ignorar el tipo).













<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Sort according to one of the Looper control attributes selected in a Radio Button control
SWITCH RADIO_RADIO1
	CASE 1: LooperSort("NAME")
	CASE 2: LooperSort("COMPANY")
	CASE 3: LooperSort("CITY")
END
```
<a name="Example2"></a>

```wl
// Sort by name (in ascending order) and by date (in descending order)
LooperSort("NAME" + TAB + "-DATE")
```
<a name="Example3"></a>

```wl
// Sort the LOOP_Customer Looper control on all its attributes (in descending order)
LooperSort("-LOOP_Customer")
```
<a name="Example4"></a>

```wl
// Cancels the sort that was performed in the LOOP_Customer Looper control
LooperSort("LOOP_Customer", False)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Ordenar una control Looper según una lista de atributos 

`<Result> = LooperSort(<List of attributes>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la ordenación,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<List of attributes>: (Cadena de caracteres)`**

Nombre de los atributos según los cuales se clasificará la control Looper (hasta 10 atributos). Este string debe tener el siguiente formato:

```txt
[<Direction>]<Attribute 1> + TAB + [<Direction>]<Attribute 2> + TAB + ...
```
donde:

- &lt;Dirección&gt;: (opcional), puede tomar los siguientes valores:

	- "+": la clasificación se realizará en orden ascendente (valor Default).

	- "-": la ordenación se realizará en orden descendente.




- : nombre del atributo utilizado para la clasificación.




<a name="SYNTAX2"></a>

### Clasificar un control Looper o cancelar un tipo anterior

`<Result> = LooperSort([<Direction>]<Looper> [, <Indicator>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecutó o anuló el orden,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<[]>: (Cadena de caracteres)`**

Nombre de la control Looper a clasificar. 

- Si  es igual a "+", control Looper se ordenará en orden ascendente (por Default).

- Si &lt;Dirección&gt; es igual a "-", el control Looper se ordenará en orden descendente.




**`<Indicator>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> para realizar el orden (valor predeterminado), 

- <u><u><u><u>False</u></u></u></u> para cancelar un orden realizado anteriormente.





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000164"></a>

La función **LooperSort** puede utilizarse en: 

- a control Looper basada en un archivo de datos, 

- a control Looper poblada programáticamente. 



<a name="NOTE0_2"></a>


### Cancelar la clasificación
<a name="cancelar_clasificacion_ELTPARAGRAPHE000173"></a>

- La ordenación no se cancela con [LooperDelete](../WDLang2/3083008.md), [LooperDeleteAll](../WDLang2/3083012.md), [LooperSeek](../WDLang2/3083005.md) y [LooperCount](../WDLang2/3083014.md).

- Si un atributo es modificado por [LooperModify](../WDLang2/3083013.md) y si este atributo se utiliza como atributo de ordenación para la control Looper, la ordenación en este atributo se cancela.



<a name="NOTE0_3"></a>


### clasificar
<a name="clasificar_ELTPARAGRAPHE000198"></a>

La clasificación se tendrá en cuenta durante la siguiente adición de una fila en la control Looper ([LooperAdd](../WDLang2/3083006.md), [LooperAddLine](../WDLang2/3083004.md), [LooperInsert](../WDLang2/3083003.md) y [LooperInsertLine](../WDLang2/3083002.md)).
<a name="NOTE0_4"></a>


### Looper vinculado a una fuente de datos y utilizando una iteración programada
<a name="looper_vinculado_una_fuente_datos_utilizando_una_iteracion_programada_ELTPARAGRAPHE000219"></a>

**LooperSort** no funciona en un control Looper con las siguientes características: 

- control Looper vinculada a una fuente de datos,

- control Looper con la opción control **Recorrer mediante programación** (pestaña "Contenido" de la ventana de descripción del control). 




Para cambiar el orden de visualización de un control Looper con **navegación mediante programación**, se debe adaptar el código de los procesos de lectura del control Looper (proceso Lectura primero registro, anterior registro, siguiente registro, etc.).


## Gestión de cookies
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000178"></a>
Una Cookie es una forma sencilla de almacenar temporalmente información en el ordenador del usuario. Esta información puede ser recuperada más tarde.

Cuando el usuario del Web visita varias veces el mismo sitio, el sitio "identifica" al usuario a partir de sus cookies. Esto le permite evitar solicitar detalles ya proporcionados durante una visita anterior y proponer páginas personalizadas.

Observación: Una cookie tiene una fecha de caducidad (30 días por Default). El navegador del usuario la destruye automáticamente si se supera su vida útil.

**Ejemplo de uso de cookies**
Las cookies pueden utilizarse para almacenar información en el ordenador del usuario, como su nombre, las páginas que ha visitado, la fecha de su última conexión, una copia de seguridad de sus opciones, etc.

Esta información, guardada en forma de cookies, se leerá la próxima vez que el usuario visite el sitio. Por lo tanto, el sitio puede proponer características personalizadas al usuario Web:

- banners publicitarios basados en lo que los usuarios han visto durante su última visita,

- una página de inicio personalizada con el nombre de usuario y la fecha de la última conexión,

- ofertas especiales correspondientes a las búsquedas realizadas durante su última visita, etc.

## ¿Qué se almacena en un Cookie?
<a name="¿que_almacena_cookie_ELTTEXTE000202"></a>
Una cookie es un archivo de texto que se almacena en el ordenador del usuario (en la "caché" de INTERNET del navegador) durante un tiempo determinado. El Cookie es creado por el navegador o por el servidor.

Cada Cookie contiene la siguiente información:

- Nombre de Cookie: utilizado por el sitio para identificar el Cookie

- Texto de Cookie: la información almacenada por el sitio (páginas visitadas, datos personales suministrados por el usuario de la Web, ....)

- Fecha de caducidad después de la cual el Cookie ya no es válido.

- Nombre del dominio INTERNET que creó la Cookie. 
	Recordatorio: El dominio INTERNET corresponde a una sección específica del INTERNET Address: http://computer.domain.com/MyPages/Index.html.

## WEBDEV y la gestión de cookies
<a name="webdev_gestion_cookies_ELTTEXTE000226"></a>


### Elementos disponibles
<a name="elementos_disponibles_ELTPARAGRAPHE000048"></a>

Se pueden utilizar dos funciones de WLanguage para administrar las cookies de sus sitios:


|   |   |
| --- | --- |
| [CookieRead](../WDLang2/3012009.md) | Recupera el valor de una Cookie guardada en la el equipo del usuario de la Web |
| [CookieWrite](../WDLang2/3012008.md) | Envía un Cookie al cliente cuando la página HTML se muestra en el navegador del usuario de la web |

Estas funciones se pueden utilizar en el código del servidor y en el código del navegador.
<a name="NOTE3_2"></a>


### Escribir una Cookie en el ordenador del usuario
<a name="escribir_una_cookie_ordenador_del_usuario_ELTPARAGRAPHE000068"></a>

Escribir una Cookie en la el equipo del usuario de la Web: 


|   |   |
| --- | --- |
| **En código del servidor:** | Utilice [CookieWrite](../WDLang2/3012008.md) y especifique el nombre, el contenido y la fecha de caducidad de la Cookie.<br>La Cookie se creará en la el equipo del usuario de la Web durante la visualización de la siguiente página.<br>**Observación**: un cookie creado en código de servidor puede ser leído de nuevo por [CookieRead](../WDLang2/3012009.md):<br><br>- inmediatamente si se utiliza [CookieRead](../WDLang2/3012009.md) en un código de navegación<br><br>- durante la siguiente conexión del usuario si se utiliza [CookieRead](../WDLang2/3012009.md) en un código de servidor.<br><br><br> |
| **En código de navegador:** | Utilice [CookieWrite](../WDLang2/3012008.md) y especifique el nombre, el contenido y la fecha de caducidad de la Cookie.<br>El Cookie se crea inmediatamente. |


<a name="NOTE3_3"></a>


### Lectura de una Cookie en el ordenador del usuario (código del servidor y código del navegador)
<a name="lectura_una_cookie_ordenador_del_usuario_codigo_del_servidor_codigo_del_navegador_ELTPARAGRAPHE000108"></a>

Para leer un Cookie en el ordenador del usuario (código del servidor y código del navegador), utilice [CookieRead](../WDLang2/3012009.md) y especifique el nombre de Cookie.

**Observaciones**: Durante la conexión al sitio web, el servidor lee automáticamente todas las cookies associated con el dominio actual. [CookieRead](../WDLang2/3012009.md) utilizada:

- en el código del servidor, lee en la memoria y recupera el contenido de un Cookie específico.

- en el código del navegador, lee directamente el contenido de la Cookie en la el equipo del usuario de la Web.

## Comprobación de la gestión de las cookies en un sitio WEBDEV
<a name="comprobacion_gestion_las_cookies_sitio_webdev_ELTTEXTE000262"></a>
La gestión de las cookies en su sitio (en el ordenador de desarrollo) puede ser comprobada: 

- desde la página de prueba del administrador de WEBDEV.

- con una simple prueba por un proyecto "GO" del editor de páginas de WEBDEV.


## BrowserIPAddress( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve la IP Address del ordenador cliente conectado al sitio WEBDEV. 

**Observación**: Si el usuario web está detrás de un router, la IP Address será la del router. 

Para get esta información desde un Webservice, utilice [Servicio webClienteDirección IP](../WDLang3/1000019641.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Returns the IP address of the Web user
sClientAddress is string = BrowserIPAddress()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = BrowserIPAddress([<Accept return address>])`
---

**`<Result>: (Cadena de caracteres)`**

IP Address de el equipo del usuario Web: 

- en formato IPv4: xxx.xxx.xxx.xxx.xxx. Por ejemplo: 195.196.100.1

- en formato IPv6: xxxx:xxxx:xxxx:xxxx:xxxx:xxxx:xxxx:xxxx:xxxx:xxxx




**`<Accept return address>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> para aceptar el Address inicial especificado en la cabecera HTTP, 

- <u><u><u><u>False</u></u></u></u> (valor por defecto) para rechazar este Address. En este último caso, la dirección IP se recupera en el entorno Variable (REMOTE_ADDR) definido por el servidor web. 




Por defecto, el Address especificado en la cabecera HTTP es rechazado por razones de seguridad.

### Aceptar (o no) la devolución Address
<a name="aceptar_devolucion_address_ELTPARAGRAPHE000057"></a>

La dirección devuelta por **BrowserIPAddress** puede no corresponder a la dirección del usuario (si se utiliza un proxy que no sea Transparent o un equilibrador de carga, por ejemplo). En este caso, **BrowserIPAddress** devuelve la IP Address del proxy o equilibrador de carga.

Si se acepta el retorno Address, **BrowserIPAddress** intenta leer el Address original especificado en las cabeceras HTTP (la mayoría de los proxies proporcionan el Address original en las cabeceras HTTP). Sin embargo, esta Address debe utilizarse con precaución porque no es fiable y puede ser completamente errónea.

Observaciones:

- Si el Address original no se encuentra o no es válido (demasiado largo, por ejemplo), se envía el Address "estándar".

- En el modo de prueba, **BrowserIPAddress** devuelve el Address del ordenador de desarrollo.

### Servidor IP Address
<a name="servidor_address_ELTPARAGRAPHE000080"></a>

La IP del servidor Address es devuelta por [NetIPAddress](../WDLang3/3056023.md).


### Sesiones preiniciadas
<a name="sesiones_preiniciadas_ELTPARAGRAPHE000090"></a>

Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio".


## ConfigureAWPContext( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Configura el modo de funcionamiento del contexto AWP. 

Para más detalles sobre los modos disponibles, véase [Gestión de los contextos de AWP](../WDChamp/3539060.md). 

**Observación**: Si no se llama a **ConfigureAWPContext**, los contextos son gestionados a través de cookies por Default. 

**Atención**: Esta función requiere un buen conocimiento de la WEBDEV.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Manage the context of a file on the server
// and transmit the identifier by cookie
ConfigureAWPContext(ctxDisk, ctxIDCookie)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`ConfigureAWPContext(<Content storage mode> [, <Identifier transmission mode>])`
---

**`<Content storage mode>: (Constante de tipo Integer)`**

Modo de almacenar el contenido del contexto:


|   |   |
| --- | --- |
| *ctxCookie* | El contenido se transmite en un Cookie (comportamiento compatible con la versión 11).<br><br>**Atención**: Esta constante no debe utilizarse para almacenar datos sensibles. De hecho, esta información se almacena "sin estar encriptada". |
| *ctxDisk* | El contenido del contexto se guarda en un archivo en el servidor. |



**`<Identifier transmission mode>: (Constante entera (opcional))`**

Define el modo de transmisión del identificador (tenido en cuenta solo si el &lt;Modo de almacenamiento del contenido&gt; corresponde a la constante *ctxDisk*):


|   |   |
| --- | --- |
| *ctxIDCookie* | El identificador es transmitido por un Cookie. |
| *ctxIDCookieURL*<br>(valor predeterminado) | El identificador se transmite por medio de cookies. Si las cookies no funcionan, el identificador se transmite por la URL. Al crear el contexto, se utilizan los dos modos porque no es posible saber si las cookies serán aceptadas o no. |
| *ctxIDURL* | El identificador es transmitido por la URL. |









<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Se recomienda utilizar **ConfigureAWPContext** en la "Inicialización" Event del proyecto. De hecho, este modo no puede modificarse una vez iniciada la sesión.

- **ConfigureAWPContext** sólo se tiene en cuenta en el modo AWP.

- Si no se llama a **ConfigureAWPContext**, los contextos son gestionados a través de cookies por Default. 

- **Recordatorio**: La inicialización Process del proyecto se ejecuta cada vez que se visualiza una página Active WEBDEV.

- Si el proyecto utiliza sesiones preiniciadas, esta función no debe utilizarse en el evento "Inicialización" del proyecto. Debe utilizarse en el evento "Inicialización del proyecto después de la conexión al sitio". 

- Para obtener más información, consulte [Las dinámicas páginas de Active WEBDEV](../WDChamp/3539009.md).

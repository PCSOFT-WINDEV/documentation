


## &lt;Guardar (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Valida la modificación o adición del Record associated con el fila actual en el control Looper.. El Record se actualiza en el el archivo de datos relacionado al control Looper.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Save the data of the Looper control "LOOP_ProductLooper" 
LOOP_ProductLooper.Save()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Looper control>.Save()`
---

**`<Looper control>: (Nombre del control)`**

Nombre del control Looper a utilizar. El actual fila de este control Looper se registrará en el archivo de datos de relacionado.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Actualización de el archivo de datos relacionado a la control Looper
<a name="actualizacion_archivo_datos_relacionado_control_looper_ELTPARAGRAPHE000202"></a>

La función **&lt;Guardar** guarda la fila actual del control Looper en el archivo de datos relacionado con el control Looper. Los elementos se asignan automáticamente con los valores de los controles de associated en el fila actual.

Cuando se realiza una modificación, **&lt;Guardar** reemplaza los valores anteriores por los nuevos. Los elementos que no se encuentran en el control Looper no se modifican.

Atención:

- La función **&lt;Guardar** no soporta el archivo de datos asociado al control Looper. Si algunos atributos/controles son relacionado para elementos encontrados en otros archivos de datos, estos archivos de datos no se gestionan automáticamente.

- Los atributos/controles "Imagen" no se guardan con la función **&lt;Guardar**. Para modificar el contenido de la nota associated con el Image (caso raro), utilice [&lt;Fuente&gt;.LinkMemo](../WDLang4/1000024194.md).



<a name="NOTE0_2"></a>


### Entrada en cascada y actualización del archivo de datos de relacionado
<a name="entrada_cascada_actualizacion_del_archivo_datos_relacionado_ELTPARAGRAPHE000232"></a>

Si la entrada en cascada está activada, los cambios se aplican automáticamente en el archivo de datos relacionado al control Looper cuando el usuario se desplaza al siguiente fila [.Add](../WDLang2/1000023469.md), [&lt;AddLine](../WDLang2/1000023475.md), [&lt;Insertar](../WDLang2/1000023509.md), [&lt;InsertLine](../WDLang2/1000023511.md), [&lt;Modificar](../WDLang2/1000023512.md) y [&lt;ModifyLine](../WDLang2/1000023520.md)). La función **&lt;Guardar** no es útil.

Si la entrada en cascada no está activada, los cambios no se aplican en el archivo de datos relacionado al control Looper. Para escribir la fila agregada o modificada en el archivo de datos relacionado, utilice la función **&lt;Guardar** después de la función utilizada para realizar la adición o la modificación.

Observaciones:

- La propiedad [NewLine](../Proprietes/2510073.md) determina si el fila acaba de ser creado por [.Add](../WDLang2/1000023469.md), [&lt;AddLine](../WDLang2/1000023475.md), [&lt;Insertar](../WDLang2/1000023509.md) o [&lt;InsertLine](../WDLang2/1000023511.md) (..Nuevo = True).

- El [Modified](../Proprietes/2510080.md) Property determina si el usuario ha modificado un fila en el control Looper (..Modificado = True).


<a name="NOTE0_3"></a>


### Visualización de la fila guardada
<a name="visualizacion_fila_guardada_ELTPARAGRAPHE000311"></a>

Después de utilizar la función **&lt;Guardar**, la fila guardada puede "desaparecer". Esta "desaparición" se debe al valor de la clave de búsqueda de la Record añadida o modificada .

Para evitar este problema, vuelva a visualizar la control Looper con [&lt;Pantalla](../WDLang2/1000023451.md).
<a name="NOTE0_4"></a>


### Bloqueo y actualización de el archivo de datos
<a name="bloqueo_actualizacion_archivo_datos_ELTPARAGRAPHE000330"></a>

Si el archivo de datos está bloqueado en el Process actual, **&lt;Guardar** guarda el fila actual 0094 y desbloquea el archivo de datos .
<a name="NOTE0_5"></a>


### Duplicados e integridad referencial
<a name="duplicados_integridad_referencial_ELTPARAGRAPHE000342"></a>

- Si la gestión de duplicados está habilitada ( [&lt;Fuente&gt;.SetDuplicates](../WDLang4/1000024985.md) ), se debe marcar [HErrorDuplicates](../WDLang4/3044077.md) después de la ejecución de **&lt;Guardar** para verificar la unicidad de la clave.

- Si la gestión de integridad referencial está habilitada ( [&lt;Variable Connection&gt;.SetIntegrity](../WDLang4/1000023960.md) ), [HErrorIntegrity](../WDLang4/3044086.md) debe estar verificado después de la ejecución de **&lt;Guardar** para verificar la integridad referencial.



<a name="NOTE0_6"></a>


### Utilice &lt;Guardar para reducir el número de líneas de código
<a name="utilice_docparampagetitleshort_para_reducir_numero_lineas_codigo_ELTPARAGRAPHE000373"></a>

La función **&lt;Guardar** reduce varias líneas de código. Esta única Line permite evitar asignar las variables de los artículos uno por uno.

Por ejemplo, el siguiente código:

```wl
Customer.LastName = ATT_LastName
Customer.FirstName = ATT_FirstName
Customer.Address = ATT_Address
Customer.ZC = ATT_ZC
Customer.City = ATT_City
Customer.Add()
```
 se sustituye por el siguiente Line:

```wl
LOOP_CustomerLooper.Save()
```

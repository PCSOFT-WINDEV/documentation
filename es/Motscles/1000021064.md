
## SecurePassword (Tipo de variable)

*** ***
				

<a name="description"></a>
El tipo **SecurePassword** es un tipo estructurado que permite obtener una contraseña segura, cifrada con un algoritmo de hash y sal. Este tipo de variable se puede utilizar: 

- para aplicaciones de 3 niveles que manipulan campos de tipo Contraseña.  

- para almacenar una contraseña segura (con hash y sal) en una memoria binaria o en un archivo XML. 



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
oSP is SecurePassword
oSP.Hash = MDPS_SHA2_256
oSP.IterationCount = 10000
oSP.Size = 256
oSP = "hello"
```

## Observaciones


### Propiedades WLanguage que se pueden utilizar con el tipo SecurePassword
<a name="propiedades_wlanguage_que_pueden_utilizar_con_tipo_securepassword_ELTPARAGRAPHE000030"></a>

Las siguientes propiedades pueden usarse para manipular una variable de tipo **SecurePassword**: 


| Nombre de la propiedad | Tipo | Efecto |
| --- | --- | --- |
| Age | Duration | Tiempo transcurrido desde la última vez que se cambió la contraseña. |
| Content | Character string | Contraseña. |
| Derivation | Constante de tipo Integer | Función de derivación de claves utilizada para el hash de la contraseña: <br><br>- *SP_PBKDF2*: Se utiliza la función de derivación de claves basada en contraseña 2 (PBKDF2). <br><br><br><br><br>PBKDF2 es una función de derivación de claves que recibe los siguientes parámetros: <br><br>- la contraseña,<br><br>- un algoritmo de hash (SHA256, SHA512, etc.),<br><br>- una sal (SALT) generada aleatoriamente. La sal previene ataques de tipo tablas arcoíris (Rainbow Tables).<br><br>- un número de iteraciones. Este número de iteraciones aumenta el tiempo que se tarda en generar y comparar una contraseña, y evita que los atacantes puedan probar millones de contraseñas en poco tiempo. El tiempo que se tardaría en generar y comparar una contraseña es tan largo que resulta imposible probar muchas combinaciones.<br><br><br> |
| Hash | Constante de tipo Integer | Algoritmo de sal: <br><br>- SP_SHA2_256: Se utiliza el Algoritmo de Hash Seguro 256 (SHA-256).<br><br>- SP_SHA2_512: Se utiliza el Algoritmo de Hash Seguro 512 (SHA-512).<br><br>- SP_SHA3_256: Se utiliza el Algoritmo de Hash Seguro 3-256 (SHA-3 256).<br><br>- SP_SHA3_512: Se utiliza el Algoritmo de Hash Seguro 3-512 (SHA-3 512). <br><br><br> |
| IterationCount | Integer | Número de iteraciones del algoritmo hash utilizado. Se recomienda establecer un valor entre 10 000 y 30 000 |
| Length | Integer | Longitud del hash de salida (en bits). La clave debe ser un múltiplo de 8, mayor de 160 bits y menor de 2048 bits. Se recomienda utilizar una clave de 256 o 512 bits. |

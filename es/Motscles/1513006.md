## Presentación
<a name="presentacion_ELTTEXTE000163"></a>
Para simplificar el mantenimiento o el uso del código existente por parte de otros desarrolladores, WINDEV, WEBDEV y WINDEV Mobile proponen una documentación automatic para los procedimientos (o métodos de clase).

Esta documentación de automatic corresponde a un set de comentarios. Estos comentarios se generan:

- Cuando se crea una nueva Procedure o método.
	Puede elegir si desea generar comentarios en la ventana que se muestra al crear un procedimiento o método (opción "Generar un comentario de encabezado"). Los comentarios generados se ajustarán a la [Parámetros de la documentación de automatic](#NOTE2_1).

- A través de ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IconeCommentaireProc.gif) (que se encuentra en la esquina superior derecha de la ventana de código de Procedure). Este icono se utiliza para generar o eliminar un conjunto de comentarios para la actual Procedure.




También es posible:

- Sincronizar automáticamente los comentarios generados según la sintaxis de los procedimientos. Modificando la sintaxis se actualizan automáticamente los comentarios de associated (durante la copia de seguridad).

- Utilizar la entrada asistida al escribir la llamada a los procedimientos o métodos documentados.. Para obtener más información, consulte [Entrada asistida](#NOTE3_1).

## Documentación automática
<a name="documentacion_automatica_ELTTEXTE000187"></a>


### Parámetros
<a name="parametros_ELTPARAGRAPHE000040"></a>

Para configurar la documentación de automatic: 

1. En la pestaña "Inicio", en el grupo "Entorno", despliegue "Opciones" y seleccione "Opciones del editor de código".  

2. Visualizar la ficha "Doc.

<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Doc_auto.gif)
Importante: Cada bloque de comentarios debe cumplir con el formato que se presenta a continuación. Especialmente, la description de cada parámetro así como la description de valor de retorno debe hacerse en una sola Line. De lo contrario, pueden aparecer desviaciones.

### Observación: borrado de parámetros
<a name="observacion_borrado_parametros_ELTPARAGRAPHE000091"></a>

Si se especifica un parámetro en la sintaxis de Procedure, si se borra, el reference del parámetro sigue siendo Visible en la sección de comentarios "Detalles de los parámetros".

El texto escrito por el desarrollador en relación con el parámetro debe ser borrado manualmente por el desarrollador (limita los riesgos de borrado por "error").

La opción "Sincronizar los comentarios con las sintaxis de los procedimientos" se utiliza para mantener actualizados los comentarios independientemente de las modificaciones realizadas..

## Entrada asistida
<a name="entrada_asistida_ELTTEXTE000223"></a>


### Entrada asistida en los procedimientos y métodos de la clase
<a name="entrada_asistida_los_procedimientos_metodos_clase_ELTPARAGRAPHE000104"></a>

Si los parámetros de un procedimiento han sido documentados y si la entrada asistida está disponible en el editor de código, al escribir la llamada al procedimiento, la sintaxis y la description de los parámetros se muestran en un tooltip. Por ejemplo:

1. Documentación de la Procedure <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Commentaires_proc.gif)


2. Aportación asistida en la Procedure <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Saisieassistee.gif)



El mecanismo de entrada asistida para los procedimientos y métodos es idéntico al mecanismo utilizado para las funciones de WLanguage.. Para más detalles sobre los parámetros de entrada asistida en WINDEV, WEBDEV o WINDEV Mobile, véase [Entrada asistida para las funciones WLanguage](../Editeurs/2013009.md).

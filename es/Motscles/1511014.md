


## STOP (Palabra reservada)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
**PARAR** se usa para llamar al depurador de WLanguage durante una prueba en el editor. Una vez llamada esta palabra clave, la prueba actual se ejecuta en el depurador.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Open a new window
Open(WIN_EditWindow)
// Start the debugger
STOP
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Inicio del depurador

`STOP`
---


<a name="SYNTAX2"></a>

### Iniciar el depurador con la condición

`STOP IF <Condition>`
---

**`<Condition>: ()`**

Condición para iniciar el depurador. El depurador se iniciará si se cumple la condición.. 
Atención: esta condición se evalúa cuando se depura un proyecto solamente. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Modos de inicio del depurador
<a name="modos_inicio_del_depurador_ELTPARAGRAPHE000061"></a>

Para iniciar el depurador durante una prueba, puede utilizar:

- **PARAR** en el editor de código.

- Un punto de interrupción ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_PointArret.gif) en el editor de código: en la pestaña "Código", en el grupo "Puntos de interrupción", despliegue "Punto de interrupción" y seleccione "Agregar un punto de interrupción" (o presione Ctrl + B en el editor de código).

- Las teclas Ctrl + Pausa cuando se ejecuta la prueba.



<a name="NOTE0_2"></a>


### Rastreo/Depuración de un proyecto
<a name="rastreodepuracion_proyecto_ELTPARAGRAPHE000086"></a>

Trazar un proyecto desde el inicio de su ejecución: en la pestaña "Proyecto", en el grupo "Modo de prueba", amplíe "Modo de prueba" y seleccione "Proyecto de depuración" (Ctrl + F9).
<a name="NOTE0_3"></a>


### Ejecutable
<a name="ejecutable_ELTPARAGRAPHE000105"></a>

**PARAR** no tiene ninguna acción en el código de un ejecutable.

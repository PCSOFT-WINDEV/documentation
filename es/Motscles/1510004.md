
## Instrucción LOOP

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El bloque de instrucciones se repite sin parar. El número de iteraciones en el bloque de instrucciones no se comprueba, no hay ninguna expresión que evaluar.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Exit according to an IF condition
LOOP
	// Read a line in the text file
	ALine = fReadLine(FileNum)
	 IF ALine = EOT THEN BREAK
	ProcessLine(ALine)
END
```


<a name="Example2"></a>



```wl
// Exit according to a WHILE condition
LOOP
	// Read a line in the text file
	ALine = fReadLine(FileNum)
	ProcessLine(ALine)
DO WHILE ALine <> EOT
```




<a name="Example3"></a>





```wl
// Exit according to an iteration
LOOP (10)
	// Read a line in the text file
	ALine = fReadLine(FileNum)
	ProcessLine(ALine)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Bucle con salida según una condición "IF".

`LOOP
    ...
    IF <Condition> THEN BREAK
    ...
END
`
---

**`<LOOP>: ()`**

Marca el inicio del bloque de instrucciones.

**`<BREAK>: ()`**

Se utiliza para salir del bloque de instrucciones.

**`<END>: ()`**

Marca el final del bloque de declaraciones.  


<a name="SYNTAX2"></a>

### Bucle con salida según una condición de "MIENTRAS".

`LOOP
...
DO WHILE <Condition>
`
---

**`<LOOP>: ()`**

Marca el inicio del bloque de instrucciones.

**`<DO WHILE>: ()`**

Marca el final del bloque de declaraciones. Se usa para salir del bloque de estado de cuenta. Las líneas del bucle que se encuentran antes de esta sentencia se ejecutan.  


<a name="SYNTAX3"></a>

### Bucle con salida según el número de iteraciones

`LOOP (<Number of iterations>)
...
END
`
---

**`<LOOP>: ()`**

Marca el inicio del bloque de instrucciones.

**`<Number of Iterations>: ()`**

Número de iteraciones a realizar. El programa saldrá del bucle cuando se alcance el número de iteraciones.

**`<END>: ()`**

Marca el final del bloque de declaraciones.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Código a ejecutar
<a name="codigo_ejecutar_ELTPARAGRAPHE000099"></a>

El código a ejecutar se sitúa entre las sentencias **LOOP** y **END**.
<a name="NOTE0_2"></a>




### Salir de un bucle
<a name="salir_bucle_ELTPARAGRAPHE000107"></a>

Hay varias declaraciones disponibles:

- [RETURN](../Motscles/1510009.md): Salir del bucle y de la Process actual (o Procedure).

- [RETURN](../Motscles/1510007.md): Devuelva un estado reporte al llamado Process. Salir del bucle y de la Process actual (o Procedure).

- [BREAK](../Motscles/1510012.md): Salga del bucle y ejecute el resto del actual Process.




[Cerrar](../WDLang1/3038018.md) se utiliza para salir del bucle y cerrar la ventana actual.

**Atención**: [RETURN](../Motscles/1510009.md) y [RETURN](../Motscles/1510007.md) no pueden utilizarse **en el mismo process**.
<a name="NOTE0_3"></a>




### Ejecutar la siguiente iteración
<a name="ejecutar_siguiente_iteracion_ELTPARAGRAPHE000137"></a>

Para ejecutar directamente la siguiente iteración **sin terminar el código de la iteración actual**, utilice la instrucción [CONTINUAR](../Motscles/1510003.md):


```txt
LOOP
	...
	IF <Condition> THEN  CONTINUE		// Go back to the LOOP keyword
	...
END
```

<a name="NOTE0_4"></a>




### Lazo sin final
<a name="lazo_sin_final_ELTPARAGRAPHE000150"></a>

Durante la compilación del proyecto, un bucle sin fin (sin [BREAK](../Motscles/1510012.md), [RETURN](../Motscles/1510009.md) o [RETURN](../Motscles/1510007.md)) es señalado por una advertencia.
<a name="NOTE0_5"></a>




### Variable externa en un bucle
<a name="variable_externa_bucle_ELTPARAGRAPHE000167"></a>

Un Variable declarado por [EXTERN](../Motscles/1511013.md) no puede ser utilizado en un bucle.

## aprovechamiento
<a name="aprovechamiento_ELTTEXTE000205"></a>
Los operadores lógicos son los siguientes:


|   |   |
| --- | --- |
| <br><br>- **Y**<br><br><br> | Multiplicación lógica. Las condiciones hechas de Y son evaluadas completamente (incluso si la primera condición es False). |
| <br><br>- ** _AND_**<br><br><br> | Multiplicación lógica. Las condiciones de _AND_ se evalúan de forma optimizada. Si la primera parte de la expresión es False, el resto de la expresión no se evalúa. |
| <br><br>- **O**<br><br><br> | Adición lógica. Las condiciones hechas de OR son evaluadas completamente (incluso si la primera condición es True). |
| <br><br>- ** _O_**<br><br><br> | Adición lógica. Las condiciones de _OR_ se evalúan de forma optimizada. Si la primera parte de la expresión es True, el resto de la expresión no se evalúa. |
| <br><br>- **NOT**<br><br><br> | Negación lógica. |

Los operadores lógicos se utilizan para realizar operaciones lógicas y crear condiciones..



```wl
IF Customer.City = "Montpellier" AND Customer.Title = "Mr" THEN
	ManMontpellier ++	// Number of men living in Montpellier
END
IF Customer.City = "Montpellier" OR Customer.City = "Lyon" THEN
	MontpellierLyon ++	// Number of customers living in Montpellier,
				// or in Lyon
END
```

## Reglas
<a name="reglas_ELTTEXTE000229"></a>
True **AND** True: Devuelve <u><u><u><u>True</u></u></u></u>

True **AND** False: Devuelve <u><u><u><u>False</u></u></u></u>

True **OR** True: Devuelve <u><u><u><u>True</u></u></u></u>

True **OR** False: Devuelve <u><u><u><u>True</u></u></u></u>

**NOT** True: Devuelve <u><u><u><u>False</u></u></u></u><a name="NOTE3"></a>

## Notas
<a name="notas_ELTTEXTE000253"></a>


### Variable de tipo Numeric
<a name="variable_tipo_numeric_ELTPARAGRAPHE000099"></a>

Si una variable numérica se manipula como un operador lógico (boolean), "0" equivale a <u><u><u><u>False</u></u></u></u>. Cualquier otro valor es equivalente a <u><u><u><u>True</u></u></u></u>.

Por ejemplo, las dos líneas de código siguientes son equivalentes:


```wl
IF TestNum THEN...
```



```wl
IF TestNum <> 0 THEN...
```


La primera sintaxis (IF TestNum THEN) debe ser preferida a la segunda.
<a name="NOTE3_2"></a>


### Cuerda Variable
<a name="cuerda_variable_ELTPARAGRAPHE000122"></a>

Se producirá un error de WLanguage si una cadena Variable se maneja como un operador lógico.

Por ejemplo, la sintaxis: "IF StrTest THEN" devolverá un error en tiempo de ejecución (pero no cuando el proyecto está compilado).
<a name="NOTE3_3"></a>


### Prioridad
<a name="prioridad_ELTPARAGRAPHE000131"></a>

El **Y** y **O, _AND_** y **NO PUEDO CREERLO.** los operadores tienen la misma prioridad. Para dar prioridad a estos operadores, todo lo que tiene que hacer es usar paréntesis.

Por ejemplo:


```txt
IF (A = 2 AND B > 3) OR (A = 2 AND B < 0) THEN ...
```


**Excepciones:**

- En los filtros y consultas SQL, el operador AND tiene prioridad sobre el operador OR.
	Por ejemplo: 
	
	```txt
	Condition1 AND Condition2 OR Condition3
	```

	se evaluará de la siguiente manera:
	
	```txt
	(Condition1 AND Condition2) OR Condition3
	```


- La adición lógica optimizada _OR_ no debe usarse si una de las expresiones a comparar está usando el resultado de una función que puede devolver NULL.
	Por ejemplo, la siguiente comparison:
	
	```txt
	IF CallFunction() = "Value1" _OR_ CallFunction() = "Value2" THEN...
	```

	tendrá que ser sustituido por el siguiente código si CallFunction puede devolver el valor NULL:
	
	```txt
	IF CallFunction() = "Value1" OR CallFunction() = "Value2" THEN...
	```

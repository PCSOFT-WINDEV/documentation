
## Instrucción IF

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La sentencia **IF** condicional permite ejecutar una acción según una condición.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF Customer.CalculatedAge > 60 THEN
	Elderly += 1
ELSE IF Customer.CalculatedAge > 18 THEN
	Adult += 1
ELSE IF Customer.CalculatedAge > 4 THEN
	Child += 1
ELSE 
	Baby += 1
END
```


<a name="Example2"></a>

```wl
// Use of 'IN'
IF Customer.City IN ("PARIS", "MARSEILLE", "LYON") THEN
	Info("You live in one of the 3 biggest cities in France")
END

// Use of 'NOT x IN'
IF NOT Customer.City IN ("PARIS", "MARSEILLE", "LYON") THEN
	Info("You don't live in one of the 3 biggest cities in France")
END

// Use of 'NOT IN' (from version 23)
IF Customer.City NOT IN ("PARIS", "MARSEILLE", "LYON") THEN
	Info("You don't live in one of the 3 biggest cities in France")
END
```

<a name="SYNTAX1"></a>

## Sintaxis

### Sintaxis 1 

`IF <Condition> THEN
   <Action if condition is True>
[ELSE
   <Action if condition is False>]
END`
---


<a name="SYNTAX2"></a>

### Sintaxis 2

`IF <Condition> THEN
   <Action if condition is True>
[ELSE <Action if condition is False>]`
---


<a name="SYNTAX3"></a>

### Sintaxis 3

`IF <Condition> THEN <Action if condition is True> [ELSE <Action if condition is False>]
`
---

**`<IF>: ()`**

Marca el inicio del bloque de instrucciones.

**`<Condition>: ()`**

Condición a verificar.

**`<THEN>: ()`**

Acciones a realizar si la condición es True.

**`<Action if condition is True>: ()`**

Acción a realizar si la condición es True.

**`<ELSE>: ()`**

Acciones a realizar si la condición es False (opcional).

**`<Action if condition false>: ()`**

Acción a realizar si la condición es False (opcional).

**`<END>: ()`**

Marca el final del bloque de declaraciones.  


<a name="SYNTAX4"></a>

### Sintaxis 4

`IF <Condition 1> THEN
   <Action if condition 1 is True>
[ELSE IF <Condition 2> THEN
   <Action if condition 2 is True> 
[ELSE IF <Condition 3> THEN
   <Action if condition 3 is True>
[...]]]
END
`
---

**`<IF>: ()`**

Marca el inicio del bloque de instrucciones.

**`<Condition 1>: ()`**

1ª condición a comprobar.

**`<THEN>: ()`**

Acciones a realizar si la condición previa es True.

**`<Action if condition 1 is True>: ()`**

Acción a realizar si la 1ª condición es True.

**`<ELSE IF>: ()`**

Verifica la condición siguiente si la condición anterior es False (opcional).

**`<Condition 2>: ()`**

2ª condición a verificar (opcional).

**`<Action if condition 2 is True>: ()`**

Acción a realizar si la 2ª condición es True y si la 1ª condición es False (opcional).

**`<Condition 3>: ()`**

Tercera condición a comprobar.

**`<Action if condition 3 is True>: ()`**

Acción a realizar si la 3ª condición es True y si la 1ª y 2ª condiciones son False.

**`<END>: ()`**

Marca el final del bloque de declaraciones.

## Observaciones




### Declaración condicional de Single-Line
<a name="declaracion_condicional_singleline_ELTPARAGRAPHE000114"></a>

También tiene la capacidad de usar la siguiente sintaxis:


```txt
<Result> = <Condition>? <Expression 1> ELSE <Expression 2>
```


Sintaxis detallada:


|   |   |
| --- | --- |
| &lt;Resultado&gt; | Variable para asignar con el resultado de la condición. |
| &lt;Condición&gt; | Condición a verificar. |
| &lt;Expresión 1&gt; | Valor asignado al &lt;Resultado&gt; si la condición es True. |
| &lt;Expresión 2&gt; | Value assigned to &lt;Result&gt; if the condition is False. |

Equivalencia:


```txt
IF <Condition> THEN
	<Result> = <Expression 1>
ELSE
	<Result> = <Expression 2>
END
```


**Observación**: La expresión condicional de Line puede ser usada directamente en una expresión.

**Ejemplo**: Incrementar el número de personas cuando la edad es superior a 60 años, de lo contrario el número de personas sigue siendo el mismo..  


```wl
Elderly = Customer.CalculatedAge > 60? Elderly +1 ELSE Elderly
```

### Condición
<a name="condicion_ELTPARAGRAPHE000153"></a>

La condición puede tener el siguiente formato:


|   |   |
| --- | --- |
| <br><br>- Valor &lt;Valor** =** &lt;Expresión&gt;&gt;Expresión<br><br><br> | Prueba equality |
| <br><br>- &lt;Valor&gt; **&lt;** &lt;Expresión&gt;<br><br><br> | Prueba comparison |
| <br><br>- Valor &lt;Valor **&lt;=** &lt;Expresión&gt;&gt;Expresión<br><br><br> | Prueba comparison |
| <br><br>- &lt;Valor&gt;** &gt;** &lt;Expresión&gt;<br><br><br> | Prueba comparison |
| <br><br>- Valor &lt;Valor **&gt;=** &lt;Expresión&gt;&gt;Expresión<br><br><br> | Prueba comparison |
| <br><br>- Valor &lt;Valor **=** &lt;Expresión Mínima&gt;** A** Expresión Máxima&gt; &lt;Máxima<br><br><br> | El &lt;Valor&gt; debe estar comprendido entre la expresión mínima y la máxima |
| <br><br>- &lt;Expresión mínima&gt; **&lt;=** &lt;Valor&gt; **&lt;=** &lt;Expresión máxima&gt;<br><br><br> |   |
| <br><br>- &lt;Valor&gt; **IN** (&lt;Expresión1&gt;, &lt;Expresión2&gt;..., &lt;Expresión N&gt;)<br><br><br> | El valor es comparado con el resultado de una lista de expresiones. Todas las expresiones encontradas en la lista son evaluadas. El operador devuelve <u><u><u><u>True</u></u></u></u> si &lt;Valor&gt; es igual a al menos una de las expresiones, <u><u><u><u>False</u></u></u></u> si &lt;Valor&gt; no es igual a ninguna de las expresiones. |
| <br><br>- &lt;Valor&gt; **NOT IN** (&lt;Expresión1&gt;, &lt;Expresión2&gt;..., &lt;Expresión N&gt;)<br><br><br> | El valor es comparado con el resultado de una lista de expresiones. Todas las expresiones encontradas en la lista son evaluadas. El operador devuelve <u><u><u><u>False</u></u></u></u> si &lt;Valor&gt; es igual a al menos una de las expresiones, <u><u><u><u>True</u></u></u></u> si &lt;Valor&gt; no es igual a ninguna de las expresiones. |
| <br><br>- &lt;Valor&gt; _IN_ (&lt;Expresión1&gt;****, &lt;Expresión2&gt;, ..., &lt;Expresión N&gt;)<br><br><br> | El valor es comparado con el resultado de una lista de expresiones. Las expresiones se evalúan de izquierda a derecha. Si una expresión es igual a &lt;Valor&gt;, las expresiones restantes no se evalúan y el operador devuelve <u><u><u><u>True</u></u></u></u>. Si &lt;Valor&gt; no es igual a ninguna expresión, el operador devuelve <u><u><u><u>False</u></u></u></u>. |
| <br><br>- &lt;Valor&gt; **NOT _IN_** (&lt;Expresión1&gt;, &lt;Expresión2&gt;..., &lt;Expresión N&gt;)<br><br><br> | El valor es comparado con el resultado de una lista de expresiones. Las expresiones se evalúan de izquierda a derecha. Si una expresión es igual a &lt;Valor&gt;, las expresiones restantes no se evalúan y el operador devuelve <u><u><u><u>False</u></u></u></u>. ### Condición compuesta
<a name="condicion_compuesta_ELTPARAGRAPHE000242"></a>

Las palabras clave **AND** y **OR** se utilizan para realizar operaciones lógicas y crear condiciones compuestas.

Por ejemplo:


```wl
IF Customer.City = "Montpellier" AND Customer.Title = "Mr" THEN
	ManMontpellier ++   // Number of men living in Montpellier
END
 
IF Customer.City = "Montpellier" OR Customer.City = "Lyon" THEN
	MontpellierLyon ++ // Number of customers living in Montpellier or in Lyon
END
```


Las condiciones de **AND** y **OR** se evalúan completamente.

Por ejemplo: 
A &gt; 10 Y B &lt; 20

Si la primera condición (A &gt; 10) es falsa, la segunda condición (B &lt; 20) se verificará.

**Optimización de la evaluación de condiciones compuestas:** Utilice las palabras clave _AND_ y _OR_.. Si la primera condición es falsa (A&gt;10 en el ejemplo), la segunda condición (B&lt;20) no se verificará. Para obtener más información, consulte [Operadores lógicos](../Motscles/1512003.md).

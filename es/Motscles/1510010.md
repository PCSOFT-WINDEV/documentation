
## Instrucción WHILE

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
En un **MIENTRAS** la expresión se evalúa al principio del bloque de sentencia.

El Process se repite siempre y cuando la expresión de la condición sea True. El programa saldrá del bloque de sentencia cuando la condición sea False.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyList = INIRead("Examples", "", "", INIFile)
Keyword = ExtractString(MyList, nb, CR)
WHILE Keyword <> ""
	nb = nb + 1
	ExplName = INIRead("Projects installed", Keyword, "", INIFile)
	Keyword = ExtractString(MyList, nb + 1, CR)
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### La condición de "MIENTRAS"

`WHILE <Condition>
   <Action if condition is True>
END
`
---

**`<WHILE>: ()`**

Marca el inicio del bloque de instrucciones.

**`<Condition>: ()`**

Condición a verificar.

**`<Action if condition is True>: ()`**

Acción a realizar si la condición es True.

**`<END>: ()`**

Marca el final del bloque de declaraciones.  


<a name="SYNTAX2"></a>

### Bucle con salida según una condición de "MIENTRAS".

`LOOP
   ...
   DO WHILE <Condition>
`
---

**`<LOOP>: ()`**

Marca el inicio del bloque de instrucciones.

**`<DO WHILE>: ()`**

Marca el final del bloque de declaraciones. Se usa para salir del bloque de estado de cuenta. Las líneas del bucle que se encuentran antes de esta sentencia se ejecutan.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Código a ejecutar
<a name="codigo_ejecutar_ELTPARAGRAPHE000083"></a>

El código a ejecutar se encuentra entre los botones **MIENTRAS** y la declaración **FIN** declaración.
<a name="NOTE0_2"></a>




### Salir de un bucle MIENTRAS
<a name="salir_bucle_mientras_ELTPARAGRAPHE000091"></a>

Hay varias declaraciones disponibles:

- [RETURN](../Motscles/1510009.md): Salir del bucle **MIENTRAS QUE** y salir del actual Process (o Procedure).

- [RETURN](../Motscles/1510007.md): Devuelva un estado reporte al llamado Process. Salir del bucle **MIENTRAS QUE** y salir del actual Process (o Procedure).

- [BREAK](../Motscles/1510012.md): Salir del bucle **WHILE** y ejecutar el resto del Process actual.




[Cerrar](../WDLang1/3038018.md) se utiliza para salir del bucle **WHILE** y para cerrar la ventana actual.

Atención: [RETURN](../Motscles/1510009.md) y [RETURN](../Motscles/1510007.md) no pueden utilizarse en el mismo process.
<a name="NOTE0_3"></a>




### Lazo sin final
<a name="lazo_sin_final_ELTPARAGRAPHE000121"></a>

Al compilar el proyecto, un bucle **WHILE** sin un final obvio (falta la sentencia [BREAK](../Motscles/1510012.md), [RETURN](../Motscles/1510009.md) o [RETURN](../Motscles/1510007.md)) se señala con una advertencia.
<a name="NOTE0_4"></a>




### Ejecutar la siguiente iteración
<a name="ejecutar_siguiente_iteracion_ELTPARAGRAPHE000138"></a>

Para ejecutar directamente la siguiente iteración **sin terminar el código de la iteración actual**, utilice la instrucción [CONTINUE](../Motscles/1510003.md):


```txt
WHILE <Condition>
	...
	IF <Condition> THEN CONTINUE  // Go back to the WHILE keyword
	...
END
```

<a name="NOTE0_5"></a>




### Condición compuesta
<a name="condicion_compuesta_ELTPARAGRAPHE000151"></a>

Las palabras clave **AND** y **OR** se utilizan para realizar operaciones lógicas y crear condiciones compuestas.

Por ejemplo:


```wl
WHILE Price < 100 AND ProductType = "AA"
	NumProduct ++       // Number of products whose price is less than 
							 // 100 and whose type is "AA"
END

WHILE Price > 100 OR Price < 500
	 NumProduct ++  // Number of products whose price is included between €100 and €500
END
```


Las condiciones de **AND** y **OR** se evalúan completamente.

Por ejemplo:


```wl
A1 > 10 AND B1 < 20
```
Si la primera condición (A1 &gt; 10) es falsa, la segunda condición (B1 &lt; 20) se verificará.

**Optimizar la evaluación de condiciones compuestas:** Utilice las palabras clave **_AND_** y **_OR_**. Si la primera condición es falsa (A1 &gt; 10 en el ejemplo), la segunda condición (B1 &lt; 20) no se verificará.

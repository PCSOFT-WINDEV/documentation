
## UUID (Tipo de Variable)

*** ***
				

<a name="XUse"></a>
<a name="Use"></a>


<a name="description"></a>
El tipo UUID se utiliza para gestionar fácilmente:

- UUIDs o GUIDs,

- Identificadores HFSQL de automatic UUID,

- identificadores para objetos e interfases COM/Automática

- ...




Las variables UUID están disponibles: 

- en 128 bits. Los UUIDs de 128 bits cumplen con RFC 4122. 

- en 256 bits. 





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
v is UUID 	// Declaration of a 128-bit UUID by default
v1 is UUID on 128
v2 is UUID on 256
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Declarar y asignar un tipo de UUID

`<Variable name> is UUID = <Value>
 
OR 

 
  <Variable name> is UUID
  <Variable name> = <Value>
`
---

**`<Variable name>: ()`**

Nombre de la variable a declarar.

**`<Value>: ()`**

Valor que se asignará a la variable. Se puede asignar un UUID Variable con:

- el resultado de [GetGUID](../WDLang1/1000018857.md). 

- el resultado de [GetUUID](../WDLang1/1000023915.md) o [GetUUID256](../WDLang1/1000023916.md).  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### UUID vacío
<a name="uuid_vacio_ELTPARAGRAPHE000060"></a>

Por defecto, un UUID Variable está vacío cuando se crea. Su valor corresponde: 

- a la constante *Nil_UUID* para las variables de tipo UUID de 128 bits. 

- a la constante *Nada_UUUID256* para variables UUID de 256 bits. 



<a name="NOTE0_1B"></a>




### Posibles conversiones
<a name="posibles_conversiones_ELTPARAGRAPHE000071"></a>

El objetivo principal de las conversiones es permitir una depuración fácil mientras se detectan las operaciones que no tienen sentido.. Las conversiones en cadena o en buffer son posibles, pero no es posible hacer una conversión en un Variable numérico.

Un UUID de 128 bits puede convertirse en: 

- cadena hexadecimal estándar "xxxxxxxxxx-xxxxxx-xxxxxx-xxxxxxxxxxxxxxxx".

- búfer"big-endian" (16 bits).




Un UUID de 256 bits puede convertirse en: 

- hexadecimal string "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx".

- búfer"big-endian" (32 bits).




No se permiten otras conversiones. Especialmente, un UUID de 128 bits no puede convertirse en un UUID de 256 bits y viceversa.
<a name="NOTE0_2"></a>




### Operaciones disponibles
<a name="operaciones_disponibles_ELTPARAGRAPHE000091"></a>

El objetivo principal de las operaciones es permitir una depuración fácil mientras se detectan las operaciones que no tienen sentido.. Se permite la concatenación con una cadena, pero no las operaciones aritméticas.

Un UUID (128 o 256 bits) puede concatenarse con una cadena o un buffer.
<a name="NOTE0_3"></a>




### Comparación
<a name="comparacion_ELTPARAGRAPHE000101"></a>

Los UUIDs pueden ser comparados para permitir la ordenación y búsqueda binaria en un array.

- Los UUIDs de 128 bits pueden compararse con los UUIDs, cadenas y búferes de 128 bits.

- Los UUID de 256 bits pueden compararse con los UUID de 256 bits, cadenas y búferes.



<a name="NOTE0_4"></a>


### Generación de un UUID
<a name="generacion_uuid_ELTPARAGRAPHE000111"></a>

Se puede generar un UUID de 128 bits: 

- con la función [GetGUID](../WDLang1/1000018857.md). Ejemplo:  
	
	```wl
	v is UUID = GetGUID()
	```


- con la función [GetUUID](../WDLang1/1000023915.md). Ejemplo: 
	
	```wl
	v is UUID = GetUUID()
	```





Se puede generar un UUID de 256 bits: 

- con la función [GetGUID](../WDLang1/1000018857.md). Ejemplo: 
	
	```wl
	v is UUID on 256 = GetGUID(guidRough256)
	```


- con la función [GetUUID256](../WDLang1/1000023916.md). Ejemplo: 
	
	```wl
	v is UUID on 256 = GetUUID256()
	```




<a name="NOTE0_5"></a>


### Elementos HFSQL
<a name="elementos_hfsql_ELTPARAGRAPHE000146"></a>

- Los elementos UUID de 128 y 256 bits están disponibles en el análisis.

- Los artículos de 128 y 256 bits UUID pueden ser automatic. En este caso, son automatic UUIDs. Se generará automáticamente un UUID cuando se llame a [HAdd](../WDLang4/3044147.md).




Para obtener más información, consulte [Editor de análisis : los diferentes tipos de artículos](../Editeurs/2011026.md). 
<a name="NOTE0_6"></a>


### Gestión de tipos de UUID en los controles
<a name="gestion_tipos_uuid_los_controles_ELTPARAGRAPHE000162"></a>

Para los controles que permiten manejar estos tipos, se dispone de entradas y máscaras de visualización "UUID" y "UUID 256".. 

Es posible enlazar con elementos HFSQL y variables UUID.
<a name="NOTE0_7"></a>


### interfaz COM
<a name="interfaz_com_ELTPARAGRAPHE000171"></a>

Un UUID de 128 bits puede ser usado para almacenar CLSIDs e IIDs de objetos COM e interfaces.. Por tanto, puede utilizarse en [COMCreateInstance](../WDLang1/1000018831.md) y [Interfaz COMQuery](../WDLang1/1000019784.md).
<a name="NOTE0_8"></a>


### Serialización
<a name="serializacion_ELTPARAGRAPHE000184"></a>

Los UUID se pueden serializar ([Serialize](../WDLang1/3013065.md)): 

- UUID 128:

	- Serializable en XML y JSON : cadena estándar: "xxxxxxxxxx-xxxxxx-xxxxxx-xxxxxxxxxxxxxx"

	- Serializable en binario : formato binario"big-endian".




- UUID 256:

	- Serializable en XML y JSON : cadena: "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

	- Serializable en binario : formato binario"big-endian".

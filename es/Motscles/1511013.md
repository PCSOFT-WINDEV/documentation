
## Extern (Palabra reservada)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
**EXTERNO** está acostumbrado:

- Incluir un archivo de texto que contenga comandos de WLanguage en una aplicación (ver sintaxis 1). Esta característica sólo está disponible en el código de inicialización del proyecto, en el código de declaración de las variables globales de una ventana/página y en el código de declaración de una clase.

- Declarar un objeto externo (véase la sintaxis 2).





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Syntax 1
// Include the "WinConst.wl" file that contains the standard Windows constants
EXTERN "WinConst.wl"
```


<a name="Example2"></a>


![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Syntax 2
EXTERN ExampleWindow
LoadWDL("User1.WDL")
Open(ExampleWindow)
```

## Sintaxis
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) 
### Incluir un archivo de texto en una aplicación

`EXTERN <File name>
`
---

**`<File name>: (Cadena de caracteres)`**

Nombre del archivo de texto que contiene el código WLanguage que debe incluirse en una aplicación. Este es un archivo de texto estándar, creado por cualquier editor de texto.

Se puede especificar una ruta completa (por ejemplo: "C:\\WDProject\\MyConst.wl"). Si no se especifica el ruta, se buscará el archivo:

- en el directorio del proyecto,

- en el directorio "Personal\\Extern" de WINDEV, WEBDEV y WINDEV Mobile.




Incluyendo un archivo de texto con **EXTERNO** tiene el mismo efecto que copiar un archivo de texto en el editor de código.  


<a name="SYNTAX2"></a>

### Declarar un objeto externo

`EXTERN <Name of External Object>
`
---

**`<Name of external object>: (Cadena de caracteres)`**

Nombre del objeto externo (Variable, constante, ...) a declarar.

**EXTERN** se utiliza para declarar un Variable que sólo existirá cuando se ejecute la aplicación.

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Por ejemplo, la palabra clave EXTERN puede utilizarse para manipular una ventana que se encuentra en una biblioteca externa (cargada por [LoadWDL](../WDLang1/3013020.md)).

## Observaciones




### Incluyendo constantes
<a name="incluyendo_constantes_ELTPARAGRAPHE000118"></a>

- Incluir archivos es muy useful para describir las constantes comunes a varios proyectos o utilizadas por el sistema operativo. Usar **EXTERNO** para incluir en el código un archivo que contenga las constantes comunes.

- La inclusión de archivos sólo se puede realizar en los siguientes procesos:

	- Código de inicialización del proyecto,

	- Código de declaración de variables globales de ventana o página,

	- Código de declaración de la clase.




- Se suministran varios archivos que definen las constantes (en el subdirectorio "\\Personal\\Extern" de WINDEV, WEBDEV o WINDEV Mobile):

	- "***WinConst.wl***": Constantes estándar de Windows.

	- "***Limites.wl***": Constantes correspondientes a los límites para los tipos de datos WINDEV, WEBDEV y WINDEV Mobile.

	- "***ListDefinitionHF.wl***": Constantes HFSQL utilizadas para el registro Process (ver [Archivos manejados por el registro Process](../WDLang4/3044182.md)).

	- ***"Excepto.wl"***: Constantes utilizadas para gestionar las excepciones.

	- ***"KeyConst.wl"***: Constantes estándar de Windows utilizadas para las teclas del teclado. Estas constantes pueden utilizarse con [KeyPressed](../WDLang1/3014007.md) en los procesos opcionales de WINDEV ("Key down" o "Key up") mediante el parámetro Variable.






<a name="NOTE0_2"></a>




### Implementación de la aplicación
<a name="implementacion_aplicacion_ELTPARAGRAPHE000144"></a>

Al instalar la aplicación en el ordenador cliente:

- ***Si se especifica la ruta del archivo***: El archivo debe estar instalado en el directorio especificado.

- ***Si no se especifica la ruta del archivo***: Por Default, el archivo se instalará automáticamente en el directorio del proyecto.

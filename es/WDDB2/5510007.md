## Principales observaciones
<a name="principales_observaciones_ELTTEXTE000239"></a>


### Versiones necesarias
<a name="versiones_necesarias_ELTPARAGRAPHE000018"></a>

**Atención**: Esta versión del conector DB2 nativo funciona con WINDEV y WEBDEV 28.
<a name="NOTE2_2"></a>


### Elementos Unicode
<a name="elementos_unicode_ELTPARAGRAPHE000027"></a>

El conector DB2 nativo no soporta elementos Unicode. Le aconsejamos que utilice elementos de texto en formato Ansi y que realice la Conversion( si es necesario) con [UnicodeToAnsi](../WDLang1/3024025.md).

[HCreation](../WDLang4/3044255.md) mostrará un mensaje de error al crear archivos que contengan elementos en formato Unicode.
<a name="NOTE2_3"></a>


### Timeout de bloqueo
<a name="timeout_bloqueo_ELTPARAGRAPHE000041"></a>

Los tiempos de bloqueo en una base de datos DB2 no están limitados por Default. La aplicación puede bloquearse si se dan las dos condiciones siguientes al mismo tiempo:

- record bloqueada por [SQLLock](../WDLang4/3072003.md),

- este Record ya está cerrado.




Le aconsejamos encarecidamente que defina un tiempo de bloqueo para cada base de datos DB2 utilizada con el conector nativo DB2. El tiempo de bloqueo suele ser de set a 30 segundos.

Consulte la documentación específica de la DB2 para obtener más detalles.
<a name="NOTE2_4"></a>


### Valores de Default de los artículos
<a name="valores_default_los_articulos_ELTPARAGRAPHE000060"></a>

Los valores Default de los artículos son manejados por el conector nativo DB2. Los valores de Default pueden ser: 

- un valor, 

- una fórmula SQL. En este caso, la expresión SQL debe ser reconocida por el motor HFSQL y por la base de datos DB2.. La única expresión que se puede utilizar es CURRENT_TIMESTAMP.

## Características específicas del conector nativo DB2
<a name="caracteristicas_especificas_del_conector_nativo_db2_ELTTEXTE000281"></a>


### Opciones de análisis
<a name="opciones_analisis_ELTPARAGRAPHE000072"></a>

- Las opciones de análisis relativas a los índices de texto completo no están soportadas por Conector Nativo.

- Las opciones de análisis relativas a la indexación de los memorandos de texto de Partial no están soportadas por el Conector Nativo.



<a name="NOTE3_2"></a>


### Tipos de usuarios
<a name="tipos_usuarios_ELTPARAGRAPHE000080"></a>

El conector nativo DB2 no admite los tipos de usuario definidos en DB2.
<a name="NOTE3_3"></a>


### INSERTAR consultas e identificador de automatic
<a name="insertar_consultas_identificador_automatic_ELTPARAGRAPHE000087"></a>

Los identificadores automatic no son compatibles con las consultas INSERT (creadas mediante programación o en el editor de consultas). Para utilizar un identificador automatic, llame a [HAdd](../WDLang4/3044147.md) en el tabla.
<a name="NOTE3_4"></a>


### Registros idénticos
<a name="registros_identicos_ELTPARAGRAPHE000097"></a>

El conector DB2 nativo no puede diferenciar entre dos registros idénticos (lo que significa registros con valores idénticos para todos los artículos, excluyendo los artículos individuales).

Cuando se modifica o elimina una Record, se modifican o eliminan todas las Record idénticas.
<a name="NOTE3_5"></a>


### Contextos independientes
<a name="contextos_independientes_ELTPARAGRAPHE000106"></a>

El Native DB2 Connector no soporta los contextos independientes. Todas las operaciones se realizan en el mismo contexto HFSQL, incluso si se ha solicitado un contexto independiente para una ventana, una página o un reporte.
<a name="NOTE3_6"></a>


### bloqueos SQL
<a name="bloqueos_sql_ELTPARAGRAPHE000113"></a>

En una navegación realizada por [SQLFetch](../WDLang4/3072011.md) y [SQLGetCol](../WDLang4/3072019.md), el Record que se está leyendo está bloqueado en modo de escritura.

Cuando se utiliza [SQLLock](../WDLang4/3072003.md), la Conector Nativo explora los datos en un nivel de aislamiento de "lectura repetida" (para más detalles, consulte la documentación sobre DB2).

Por lo tanto, varios usuarios pueden bloquear simultáneamente un registro en modo de escritura (la función [SQLLock](../WDLang4/3072003.md) devuelve <u><u><u><u>True</u></u></u></u> para ambos usuarios). Sin embargo, sólo el usuario que bloqueó primero el Record podrá modificarlo o borrarlo. Los demás usuarios podrán acceder al Record en sólo lectura.
<a name="NOTE3_7"></a>


### Bloqueo HFSQL
<a name="bloqueo_hfsql_ELTPARAGRAPHE000140"></a>

Los bloqueos NO PUEDEN ser utilizados con las funciones HFSQL en esta versión.
<a name="NOTE3_8"></a>


### Error de duplicación en el elemento identificador de automatic
<a name="error_duplicacion_elemento_identificador_automatic_ELTPARAGRAPHE000147"></a>

La columna "Identificador automatic" de una tabla creada por [HCreation](../WDLang4/3044255.md) es la siguiente


```txt
BIGINT NOT NULL GENERATED BY DEFAULT 
AS IDENTITY (START WITH 1, INCREMENT BY 1), PRIMARY KEY
```


El valor del identificador automatic es automatic ally gestionado por DB2.

Para ello, un contador se incrementa cada vez que se realiza una adición.

Sin embargo, DB2 no incrementa el contador si se fuerza el valor del identificador automatic en una consulta INSERT.

Cuando se realiza otra adición, puede producirse un error de duplicación si DB2 automatic ally genera un valor que ya existe para el identificador automatic.

**Para evitar este problema**ejecute la siguiente consulta para reinicializar el valor del contador a un valor mayor que el identificador especificado:


```txt
ALTER TABLE <Table name> ALTER COLUMN <Name of auto ID> 
RESTART WITH <Max value of auto ID in file +1>
```


**Observación**: El Conector Nativo DB2 realiza esta operación automáticamente después de llamar a la función **HAdd(hSetIdAuto)**.

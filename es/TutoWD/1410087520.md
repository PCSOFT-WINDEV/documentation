## Presentación
<a name="presentacion_ELTTEXTE000245"></a>
Al crear una aplicación, puede que necesite manipular datos. Para almacenar los datos, debe crear una "base de datos".

En un proyecto WINDEV que manipule datos lo primero que se debe hacer es crear un "análisis".

Un análisis contiene la descripción de los archivos de datos (o tablas) en los que se encuentran los datos de la aplicación.

Al ejecutar la aplicación, se utilizan estas descripciones para crear la base de datos y/o los archivos de datos. Los datos se almacenarán en esta base de datos o en estos archivos.|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | WINDEV proporciona varias herramientas de mantenimiento de bases de datos HFSQL. Se puede acceder a ellas desde el Centro de Control HFSQL. |



WINDEV admite la mayoría de formatos de base de datos (casi todos). Los formatos más comunes son:

- HFSQL, un sistema de base de datos proporcionado con WINDEV. La base de datos HFSQL está disponible en modo Classic o Client/Server.

- Oracle, SQL Server, MySQL, SQL Azure, etc.

- AS/400, Access, Sybase, Informix...

- Cualquier base de datos accesible en lenguaje SQL en Windows.

- Texto (archivos ASCII).




Existen varios métodos para acceder a los datos:

- Conector Nativo (también llamado Acceso Nativo),

- Acceso OLE DB, 

- Acceso ODBC directo,

- Acceso ODBC a través de OLE DB.

<a name="NOTE3_1"></a>


## HFSQL
<a name="hfsql_ELTTEXTE000287"></a>
HFSQL es una base de datos potente, rápida y segura.

HFSQL funciona en Windows y Linux, en dispositivos móviles (iOS, Android), en redes de cualquier tamaño y tipo, y gestiona cientos de accesos simultáneos automáticamente.

HFSQL está disponible en modo Classic y Client/Server.

HFSQL puede distribuirse libremente con sus aplicaciones WINDEV.

HFSQL propone todas las funcionalidades de una base de datos, especialmente:

- [registros](../WDLang4/3044184.md),

- [transacciones](../WDLang4/3044335.md),

- [replicación](../WDLang4/3044266.md),

- [desencadenadores](../WDLang4/3044272.md).




En las diferentes partes de este tutorial, usaremos una base de datos HFSQL Classic, y luego una base de datos HFSQL Client/Server.

## Modos de acceso a las bases de datos
<a name="modos_acceso_las_bases_datos_ELTTEXTE000311"></a>


### Conector Nativo (Acceso Nativo)
<a name="conector_nativo_acceso_nativo_ELTPARAGRAPHE000085"></a>

Un Conector Nativo manipula un formato de base de datos de forma directa y exclusiva. Este tipo de acceso optimizado se desarrolla específicamente para cada formato de base de datos.

En WINDEV, un Conector Nativo está disponible para las siguientes bases de datos:

- HFSQL Classic o Client/Server (estándar)

- Access (estándar)

- SQLite (estándar)

- Oracle (opcional)

- AS/400 (opcional)

- SQL Server (opcional)

- Sybase (opcional)

- Informix (opcional)

- DB2 (opcional)

- Progress (opcional)

- MySQL (opcional y gratuito)

- PostgreSQL (opcional y gratuito)

- MariaDB (opcional y gratuito)

- SQL Azure (opcional y gratuito)


Otros Conectores Nativos pronto estarán disponibles, póngase en contacto con nuestro departamento de ventas para obtener más información.

Las funciones **SQL\*** y **HRead\*** de WLanguage se pueden utilizar con este tipo de acceso. El código es portable e independiente de la base de datos.
<a name="NOTE4_2"></a>


### Acceso ODBC directo
<a name="acceso_odbc_directo_ELTPARAGRAPHE000113"></a>

El acceso a través de ODBC utiliza un estándar de acceso de múltiples bases de datos. La capa ODBC debe estar instalada en su equipo. En la mayoría de los casos, esta capa ya está instalada en las últimas versiones de Windows. 

Puede comprobarlo siguiendo estos pasos: 

- Para sistemas Windows de 32 bits: en el menú de Inicio de Windows, seleccione "Configuración .. Panel de control .. Herramientas administrativas .. Origen de datos".

- Para sistemas Windows de 64 bits: haga doble clic en el siguiente archivo: C:\\Windows\\SysWOW64\\odcbad32.exe.  Aparece un cuadro de diálogo "Administrador de origen de datos" que le permite crear el origen de datos ODBC de 32 bits necesario en el sistema de 64 bits.




Atención: puede que no se pueda acceder a todas las bases de datos a través de este método. Si desea utilizar este tipo de acceso, compruebe si existe un controlador ODBC e instálelo si es necesario.

Solo las funciones **SQL\*** de WLanguage se pueden utilizar con este tipo de acceso.
<a name="NOTE4_3"></a>


### Acceso OLE DB
<a name="acceso_ole_ELTPARAGRAPHE000129"></a>

El acceso a través de OLE DB utiliza un estándar de acceso de múltiples bases de datos. Este tipo de acceso se basa en MDAC (Microsoft Data Access Component).|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=avertissement.png) | Importante | Si utiliza un acceso OLE DB, MDAC debe estar instalado en los equipos de los usuarios (versión 2.6 o posterior). |



Puede que no se pueda acceder a todas las bases de datos a través de este método. Si desea utilizar este tipo de acceso, compruebe si existe un controlador OLE DB.

Las funciones **SQL\*** y **HRead\*** de WLanguage se pueden utilizar con este tipo de acceso.
<a name="NOTE4_4"></a>


### Acceso ODBC a través de OLE DB
<a name="acceso_odbc_traves_ole_ELTPARAGRAPHE000143"></a>

En resumen, es una "mezcla" de OLE DB y ODBC. Este es el método más "pesado" y menos eficiente en términos de rendimiento. No debe utilizarse en bases de datos pequeñas.

Las funciones **SQL\*** y **HRead\*** de WLanguage se pueden utilizar con este tipo de acceso.

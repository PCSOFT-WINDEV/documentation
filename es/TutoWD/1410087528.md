## Crear el ejecutable
<a name="crear_ejecutable_ELTTEXTE000513"></a>
Crear un ejecutable es muy fácil: solo tendrá que pasar por una opción de menú y hacer unos pocos clics.

A continuación, vamos a presentar en detalle las diferentes etapas para crear el ejecutable. Puede hacer clic en "Finalizar" para validar todas las etapas del asistente en cualquier momento.

- Para crear el ejecutable:

	1. Despliegue ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_Icones%20WD%20-%20HC%20N%B0002%201_GAF.jpg) en el área de acceso rápido, y seleccione "Generar ejecutable Windows de 64 bits (.exe)".  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_D%E9ployer%20l'application%20-%20HC%20N%B0001.jpg)

			|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | WINDEV también le permite crear ejecutables de 64 bits, ejecutables Linux, servicios, aplicaciones Java y más. |





	2. El asistente de creación de ejecutables se abre. 

	3. Describa las opciones generales del ejecutable:

		- el nombre del ejecutable: conserve el nombre predeterminado.

		- el nombre del ícono asociado al ejecutable: puede seleccionar un ícono en el catálogo de imágenes.

		- la pantalla de presentación (también conocida como "Splash screen"). Existen varios tipos de pantallas de presentación. 
						<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_D%E9ployer%20l'application%20-%20HC%20N%B0002.jpg&type=thumb)



 Conserve "Sin splash screen" y pase a la etapa siguiente. 

4. Conserve las opciones predeterminadas del modo de funcionamiento del ejecutable. Pase a la etapa siguiente.

5. Puede personalizar el mensaje de error de la aplicación. Conserve el mensaje predeterminado. Pase a la etapa siguiente.

6. El asistente propone activar la telemetría de la aplicación. No usaremos esta funcionalidad. Pase a la etapa siguiente.
			|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | La telemetría permite obtener estadísticas detalladas sobre el uso de la aplicación por parte de los usuarios finales. Además, muestra un reporte de las aplicaciones implementadas, para así poder mejorarlas.<br><br> |





7. Esta etapa permite indicar si el ejecutable tendrá en cuenta las actualizaciones con parches.

Los parches son bibliotecas adicionales.<br><br>Si el ejecutable se crea con la opción "Sí: tener en cuenta las actualizaciones realizadas con patches", al iniciar la aplicación, los elementos del parche reemplazan a los elementos de la biblioteca.<br><br> |



Conserve las opciones predeterminadas. Pase a la etapa siguiente.

8. Esta etapa permite definir los idiomas del ejecutable. Esta función multilingüe se presentará en otra lección. Conserve las opciones predeterminadas. Pase a la etapa siguiente.

9. Esta etapa muestra todos los archivos que se incluirán en la biblioteca del ejecutable. El usuario final podrá manipular estos elementos del proyecto. Conserve las opciones predeterminadas. Pase a la etapa siguiente.

	- en el directorio de datos de la aplicación (si el ejecutable está instalado en "Program files"). 

	- en el directorio del ejecutable.

## Crear la instalación
<a name="crear_instalacion_ELTTEXTE000663"></a>
La instalación se crea mediante un asistente. Este asistente permite definir las principales opciones. Si las opciones propuestas por el asistente no son adecuadas, puede abrir el editor de instalaciones. No lo veremos en detalle en esta lección. Para obtener más información, consulte [Editor de instalaciones WDInst](../Editeurs/2028042.md).

A continuación, veremos las diferentes etapas para crear el programa de instalación. Puede hacer clic en "Finalizar" para validar todas las etapas del asistente en cualquier momento.



- Para crear el programa de instalación:

	1. En la pestaña "Proyecto", en el grupo "Generación", despliegue "Procedimiento de instalación" y seleccione "Crear procedimiento de instalación". El asistente de creación del ejecutable y de la instalación se abre automáticamente.

	2. Ya hemos creado el ejecutable: las opciones seleccionadas se conservan. 

	3. En el asistente, haga clic en "Complementos" en la sección "Ejecutable".

	4. El asistente propone crear una copia de seguridad del proyecto. Seleccione "No: No hacer ninguna copia de seguridad del proyecto" y pase a la etapa siguiente.

	5. El asistente propone crear la página de las nuevas funcionalidades. 
			<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_D%E9ployer%20l'application%20-%20HC%20N%B0005%20b.jpg&type=thumb)
Esta opción permite crear un archivo de ayuda para presentar las nuevas funcionalidades a los usuarios finales. Durante la primera instalación, puede ser la ayuda del software.
			Seleccione "Crear una documentación de las nuevas funcionalidades" y pase a la etapa siguiente.

	6. Puede definir los elementos que se incluirán automáticamente en el archivo de ayuda. Conserve las opciones predeterminadas y pase a la siguiente etapa.
			|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | Si la aplicación ya contiene un sistema de ayuda, puede utilizarlo e incluir la página de las nuevas funcionalidades. |





	7. Valide el mensaje y la ventana para guardar el archivo de ayuda. 

	8. El ejecutable se crea automáticamente (con las opciones definidas al crear el primer ejecutable), así como el sistema de ayuda. Puede introducir información específica en el sistema de ayuda. En este caso, conserve la información predeterminada. Para reanudar la creación del programa de instalación, haga clic en el ícono ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_D%E9ployer%20l'application%20-%20HC%20N%B0005%201.jpg), que corresponde a la opción "Continuar la creación del procedimiento de instalación". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_D%E9ployer%20l'application%20-%20HC%20N%B0005.jpg)
La ayuda se compila automáticamente.

	9. Vamos a pasar a la sección "Instalación" del asistente. En la primera pantalla, seleccione el modo de instalación:

		- Instalación individual para una aplicación independiente, instalada e iniciada en cada equipo. Seleccione esta opción.

		- Instalación con actualización automática, para una instalación en un servidor. Las aplicaciones se instalarán desde el servidor. Si hay una actualización disponible, solo se debe actualizar el servidor. Las aplicaciones instaladas en los equipos se actualizarán automáticamente.

		- Instalación en una tienda privada, para implementar la instalación en una tienda privada definida previamente. Los usuarios podrán instalar la aplicación desde la tienda. 


 Pase a la etapa siguiente.

10. Seleccione una instalación estándar. Pase a la etapa siguiente.

11. No cambiaremos el directorio de instalación predeterminado. Pase a la etapa siguiente.

12. Conserve los archivos que aparecen en la lista de archivos instalados. De forma predeterminada, WINDEV propone el archivo Ejecutable y el archivo de ayuda (creado con la ayuda de las nuevas funcionalidades). Vamos a agregar los archivos de datos:

	- Haga clic en "Agregar". El explorador de Windows muestra el contenido del directorio de generación de la aplicación.

	- Seleccione los archivos de datos Customer, Order, OrderLine, PaymentMode y Product (archivos con extensión ".fic", ".ndx" y ".mmo").

	- Valide. Los archivos se guardan automáticamente en el directorio de datos de la aplicación.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_D%E9ployer%20l'application%20-%20HC%20N%B0006.jpg&type=thumb)

## Instalar la aplicación
<a name="instalar_aplicacion_ELTTEXTE000741"></a>
En la última ventana de creación de la instalación, haga clic en el botón "Probar". Si cerró esta ventana, ejecute el programa "Install.exe" ubicado en el directorio del programa de instalación. La instalación de la aplicación se abre.



- Vamos a seguir las diferentes etapas del programa de instalación de la aplicación "WD Full Application".

	1. En Windows Vista (o posterior), una ventana solicita la autorización para ejecutar el programa. Valide la ventana.

	2. El asistente de instalación se abre.
			|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | Puede cambiar el idioma de instalación mediante el cuadro combinado situado en la esquina superior derecha del asistente.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_D%E9ployer%20l'application%20-%20HC%20N%B0008.jpg)<br> |




			Existen dos modos de instalación: 

		- Instalación rápida: La instalación utiliza los parámetros definidos por el proveedor de la aplicación. 

		- Instalación personalizada: El asistente permite al usuario seleccionar los parámetros de instalación de la aplicación. 




3. Seleccione "Instalación personalizada". El asistente pide el directorio de instalación de la aplicación.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_D%E9ployer%20l'application%20-%20HC%20N%B0009.jpg&type=thumb)

## Diferentes tipos de implementación
<a name="diferentes_tipos_implementacion_ELTTEXTE000783"></a>
Hemos realizado la implementación de una aplicación simple. En el asistente de creación de la instalación, WINDEV también propone instalaciones con actualización automática.
<a name="NOTE6_2"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000303"></a>

Existen varios modos de instalación para una aplicación WINDEV:

- **Instalación independiente**:
	Este tipo de instalación permite crear un único programa de instalación. Los usuarios finales deberán ejecutar el programa de instalación en sus equipos.
	Para actualizar la aplicación, se tendrá que volver a crear una instalación de la aplicación. Luego, el usuario final tendrá que instalar el nuevo programa.
	Este es el tipo de instalación que acabamos de crear.

- **Instalación con actualización automática**:
	Este tipo de instalación detecta las actualizaciones automáticamente al iniciar la aplicación. Si hay una actualización disponible, los usuarios pueden actualizar la aplicación inmediatamente.
	Este tipo de instalación está disponible a través de la red o de internet. También es posible realizar una instalación multisitio. 




Vamos a ver cómo crear una instalación con actualizaciones.
<a name="NOTE6_3"></a>


### Instalación con actualización a través de la red
<a name="instalacion_con_actualizacion_traves_red_ELTPARAGRAPHE000325"></a>

El programa de instalación se puede crear mediante un asistente específico (en la pestaña "Proyecto", en el grupo "Generación", haga clic en "Procedimiento de instalación").

El programa de instalación obtenido al final del asistente contiene dos instalaciones:

- la instalación de la aplicación de referencia. La aplicación de referencia debe estar instalada en la red (en un directorio compartido o en un servidor, por ejemplo), para que todos los usuarios finales de la aplicación puedan acceder a ella.

- la instalación de la aplicación como tal. Esta instalación está incluida en la aplicación de referencia. Solo se puede acceder a ella una vez instalada la aplicación de referencia.




El siguiente diagrama presenta el modo instalación de una aplicación que utiliza una base de datos local HFSQL Classic, con una actualización a través de la red.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P3-Installation%20avec%20mise%20%E0%20jour%20r%E9seau-En%20Ligne.gif&type=thumb)


La versión de la aplicación de referencia se comprueba automáticamente cada vez que el usuario final inicia la aplicación. Si esta versión cambió (por ejemplo, si se actualizó la versión de referencia), se propone automáticamente una actualización de la aplicación.
<a name="NOTE6_4"></a>


### Instalación con actualización por Internet
<a name="instalacion_con_actualizacion_por_internet_ELTPARAGRAPHE000354"></a>

La instalación con la actualización por Internet utiliza el mismo principio.

El programa de instalación se puede crear mediante un asistente específico (en la pestaña "Proyecto", en el grupo "Generación", haga clic en "Procedimiento de instalación").

El programa de instalación obtenido al final del asistente incluye:

- la instalación de la aplicación de referencia y la página Web que permiten descargar la instalación cliente. Estos elementos deben instalarse en un servidor HTTP.

- la instalación de la aplicación como tal. Esta instalación está incluida en la aplicación de referencia. Se puede acceder a ella una vez instalada la aplicación de referencia, a través de la página web de descargas.




El siguiente diagrama presenta el modo instalación de una aplicación que utiliza una base de datos HFSQL Client/Server, con una actualización por Internet.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P3-Installation%20avec%20mise%20%E0%20jour%20internet-En%20Ligne.gif&type=thumb)


Observación: Este tipo de instalación también puede hacerse a través de [PCSCloud](https://pcscloud.net/UK/cloud_development.awp) (si tiene una cuenta). 
<a name="NOTE6_5"></a>


### Instalación multisitio
<a name="instalacion_multisitio_ELTPARAGRAPHE000388"></a>

La instalación multisitio combina:

- una instalación de referencia implementada en una red local.

- una instalación HTTP utilizada para actualizar la instalación de referencia en la red local.

## Presentación
<a name="presentacion_ELTTEXTE000350"></a>
Una aplicación puede ser utilizada por varios colaboradores con perfiles diferentes. Se pueden establecer varios niveles de acceso según el usuario.

Veamos un ejemplo simple: una aplicación de gestión de ventas que propone las siguientes funcionalidades:

- Ver la lista de precios.

- Modificar la lista de precios.

- Agregar pedidos.

- Agregar clientes.


El acceso puede variar según el usuario. Por ejemplo:

- el personal de secretaría puede ver la lista de precios y crear pedidos.

- los vendedores pueden ver la lista de precios, hacer pedidos y crear nuevos clientes.

- los administradores tienen acceso a todas las opciones.




WINDEV le permite administrar fácilmente estos niveles de acceso en sus aplicaciones a través del Groupware Usuario.

Vamos a integrar el Groupware Usuario en nuestra aplicación "WD Full Application" y a configurarlo.



- Abra de nuevo el proyecto en el que trabajó en la lección anterior. 

	1. Vaya a la página de inicio de WINDEV (Ctrl + &lt;).

	2. En la página de inicio, haga clic en "Tutorial", luego en "Part 4 - Full application with data" haga doble clic en "Full application - Exercise".

	3. Un cuadro de diálogo le pide que abra el proyecto en el que trabajó en la lección anterior. Puede abrir la copia local o el proyecto original. Seleccione "Abrir la copia local".

## Integrar el Groupware Usuario
<a name="integrar_groupware_usuario_ELTTEXTE000392"></a>


- Para integrar el Groupware Usuario en el proyecto "WD Full Application":

	1. En la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Groupware Usuario". La ventana de configuración de la plataforma actual se abre en la pestaña "GPU".
			|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | El Groupware Usuario puede configurarse para una sola configuración del proyecto. Al probar el proyecto, la ventana de conexión del Groupware Usuario solo se mostrará cuando se pruebe la plataforma para la que se habilitó el Groupware. |





	2. En "Groupware Usuario", haga clic en "Cambiar". 

	3. La ventana de configuración del Groupware Usuario se abre. Active el Groupware Usuario haciendo clic en "Activo".  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0001.jpg&type=thumb)


	4. Las opciones "Groupware Usuario estándar" e "Inicio automático" están seleccionadas de forma predeterminada. 

	- La opción "Groupware Usuario estándar" integra el Groupware Usuario en la aplicación. El Groupware Usuario solo se asociará a la configuración del proyecto, y los permisos se definirán desde la aplicación. 
						Observación: También es posible utilizar el Groupware Usuario BaaS (Backend as a Service) con administración de permisos mediante un servidor. Para obtener más información, consulte la ayuda en línea:  [Groupware Usuario](../Editeurs/1000021152.md). 

	- La opción "Inicio automático" indica que el Groupware Usuario se inicia al mismo tiempo que la aplicación. 
						Observación: la opción "Inicio manual (mediante programación)" permite definir el momento en el que se debe iniciar el Groupware Usuario.

## Configurar el Groupware Usuario
<a name="configurar_groupware_usuario_ELTTEXTE000470"></a>
La configuración del Groupware Usuario consiste en definir los diferentes usuarios de la aplicación, así como los permisos otorgados sobre las diferentes ventanas y controles.|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | Los usuarios se pueden configurar:<br><br>- al desarrollar la aplicación. Los archivos de datos necesarios (de usuarios y permisos) se pueden instalar con la aplicación.<br><br>- al instalar la aplicación (por el administrador de la aplicación).<br><br><br> |







- Vamos a probar la aplicación para configurar el Groupware Usuario:

	1. Pruebe el proyecto (haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_GO_Projet_WD_GAF.jpg) en los botones de acceso rápido). Una ventana de inicio de sesión se abre.

	2. Inicie sesión como supervisor.

	3. Defina la contraseña y confírmela. Valide.

	4. Un nuevo menú le permite probar o configurar la aplicación.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0007.jpg&type=thumb)


	5. Seleccione la opción "Configurar groupware".

### Crear usuarios y grupos
<a name="crear_usuarios_grupos_ELTPARAGRAPHE000155"></a>

- Para configurar el Groupware Usuario, crearemos un usuario llamado "Test_1" y lo asociaremos al grupo "Tests". Luego, estableceremos los permisos del grupo.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0008.jpg&type=thumb)





- Para crear un usuario:

	1. Haga clic en "Nuevo" debajo del área "Usuarios". La ventana para introducir un nuevo usuario se abre.

	2. Introduzca la siguiente información:

		- Usuario: Test_1

		- Nombre: Test_1

		- Contraseña: Test_1
						|   |   |   |
| --- | --- | --- |
| ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=note.png) | Nota | De forma predeterminada, la contraseña se oculta al introducirla. Para ver la contraseña, haga clic en el ícono de ojo: la contraseña se muestra mientras se mantenga pulsado el botón del ratón. |


<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0009.jpg&type=thumb)







- Para crear un nuevo grupo de usuarios:

	1. Haga clic en "Nuevo" debajo del área "Grupos". La ventana para introducir un nuevo grupo se abre.

	2. Introduzca "Tests" para el nombre del grupo.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0010.jpg&type=thumb)


	3. Haga clic en "Guardar". El grupo "Tests" aparece en la lista de grupos definidos para el Groupware Usuario.




- Para asociar el usuario al grupo:

	1. Seleccione el usuario "Test_1" en la ventana.

	2. Arrastre el usuario "Test_1" y suéltelo en el grupo "Tests".  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0011.jpg&type=thumb)


	3. La asociación se realiza correctamente.



<a name="NOTE4_3"></a>


### Definición de permisos
<a name="definicion_permisos_ELTPARAGRAPHE000193"></a>

Vamos a definir los permisos del grupo "Tests". Estos permisos se concederán a todos los usuarios del grupo. En nuestro ejemplo, los usuarios del grupo "Tests" no podrán:

- Abrir la ventana de envío de correos electrónicos.

- Crear o modificar un producto.




- Para definir los permisos:

	1. Haga clic en "Administrar permisos" en la parte izquierda de la ventana.

	2. Seleccione el grupo "Tests".

	3. Haga clic en "Siguiente".

	4. La ventana que aparece permite seleccionar cada ventana o reporte de la aplicación.

		- Por cada ventana o reporte, es posible especificar si el grupo tiene permisos de acceso.

		- También es posible establecer si los controles de cada ventana se comportarán como la aplicación (modo predeterminado) o si estarán deshabilitados, ocultos o en gris.







- Para impedir el acceso a "WIN_Sending_an_email":

	1. Seleccione la ventana en la lista.

	2. Haga clic en "Prohibir acceso".  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0012.jpg&type=thumb)


	3. Ya no es posible acceder a la ventana.




- Defina los permisos en la ventana "WIN_Menu". Esta ventana contiene una opción de menú que permite abrir la ventana de gestión de correos electrónicos, así como los botones "New" y "Modify".

	1. Seleccione la ventana "WIN_Menu".

	2. Haga clic en "Siguiente".

	3. La ventana para configurar los permisos de los controles aparece en pantalla.

	4. Cambie el estado de los controles "BTN_Modify" y "BTN_New" a "En gris":  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0013.jpg&type=thumb)


	5. Cambie el estado la opción de menú "Send an email" a "Invisible":  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0014.jpg&type=thumb)

### Prueba de la aplicación
<a name="prueba_aplicacion_ELTPARAGRAPHE000233"></a>

- Vamos a probar la aplicación utilizando el nombre de usuario "Test_1".

	1. Pruebe el proyecto (haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_GO_Projet_WD_GAF.jpg) en los botones de acceso rápido). Una ventana de inicio de sesión se abre.

	2. Inicie sesión como "Test_1" con la contraseña "Test_1". Valide.

	3. La aplicación se abre. Si es necesario, seleccione la pestaña "List of products". Los botones "New" y "Modify" aparecen en gris y la opción "Send an email" no se muestra.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P4_Int%E9grer%20le%20GPU%20-%20HC%20N%B0015.jpg&type=thumb)


	4. Salga de la aplicación y vuelva al editor.

## Desactivar el Groupware Usuario
<a name="desactivar_groupware_usuario_ELTTEXTE000548"></a>
**No seguiremos utilizando el Groupware Usuario en este tutorial**. Puede desactivarlo directamente.

1. En la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Groupware Usuario".

2. La ventana de configuración de la plataforma actual se abre en la pestaña "GPU". 

3. En "Groupware Usuario", haga clic en "Cambiar". 

4. En la ventana que aparece, en la pestaña "General", en el área "Activar Groupware Usuario", haga clic en "Inactivo".

5. Valide la ventana de configuración del Groupware Usuario.

6. Valide la ventana de descripción de la configuración del proyecto.

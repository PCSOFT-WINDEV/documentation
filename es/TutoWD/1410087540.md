## Presentación
<a name="presentacion_ELTTEXTE000131"></a>
WINDEV permite crear aplicaciones que pueden acceder a bases de datos HFSQL Client/Server. 

El modo Client/Server permite migrar las bases de datos a un servidor. 

Este modo de funcionamiento: 

- aumenta la seguridad de los datos,

- permite gestionar fácilmente el modo multiusuario,

- simplifica el mantenimiento,

- ... 

<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=P3-HFSQL%20ClientServeur.gif)


WINDEV permite:

- crear una aplicación completa utilizando una base de datos HFSQL Client/Server.

- modificar una aplicación que utiliza una base de datos HFSQL Classic para que pueda utilizar una base de datos HFSQL Client/Server.

## ¿Por qué cambiar una aplicación al modo HFSQL Client/Server?
<a name="¿por_que_cambiar_una_aplicacion_modo_hfsql_clientserver_ELTTEXTE000155"></a>
Las principales ventajas de una aplicación HFSQL Client/Server sobre HFSQL Classic son las siguientes:

- HFSQL Client/Server es más seguro (puede crear un nombre de usuario y una contraseña, y conceder permisos a los usuarios).

- No es necesario gestionar directorios: todos los archivos de la base de datos están en la misma carpeta.

- Los usuarios finales no ven los archivos de datos en el explorador y no pueden acceder a ellos directamente.

- Las bases de datos en modo Client/Server se pueden utilizar a través de una conexión a Internet.

- Gestión del modo multiusuario nativo: El rendimiento se optimiza en el modo multiusuario.

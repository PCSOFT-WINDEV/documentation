<a name="new_controls"></a>

	
		NUEVOS CONTROLES
	

	
		 
	
		Nueva ventana emergente
	

	
		![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WM-Popup-Android.gif)![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WM-Popup-iOS.gif)
Una ventana emergente es una ventana no maximizada que se abre sobre la ventana actual. 
Una ventana emergente puede contener todos los tipos de controles disponibles en 
WINDEV Mobile. 
Una ventana emergente se cierra automáticamente en cuanto el usuario pulsa fuera de ella. 
En la versión 27, crear ventanas emergentes es muy fácil. 
Una ventana emergente es una ventana: simplemente seleccione Ventana emergente en la creación de la ventana asistente. 

	


 
	
		Programación de ventanas emergentes
	

	
		La programación de las ventanas emergentes es muy intuitiva. 
Abrir ventanas emergentes con la función [OpenPopupAsynchronous](../WDLang1/1410087806.md) WLanguage. 
Puede pasar parámetros cuando se abre la ventana emergente y especificar una llamada de retorno para recuperar los valores de retorno cuando se cierra. 

	


 
	
		Popup: Ejemplo de implementación
	

	
		Las ventanas emergentes pueden utilizarse de muchas maneras. 
Un caso de uso común es crear mensajes de ayuda para un determinado control: un menú contextual personalizado, alerta, leyendas ... <br>![Una ventana emergente mostrada en un mapa](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Popup%20exemple%20utilisation.gif)


	


 
	
		Nueva control Tabla TreeView
	

	
		<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WM-Table-Hierarchique.gif)

La control Tabla TreeView ya está disponible en WINDEV Mobile. 
La programación es la misma que en WINDEV. 
Ahora puede mostrar automáticamente los datos en un diseño de vista de árbol en los dispositivos móviles, utilizando este nuevo control. 
La Icon de los nodos y las hojas se puede personalizar para cada fila. 
No olvide que una tabla TreeView también puede contener columnas no jerárquicas. 

	


 
	
		Fijar uno o varios controles en la pantalla, donde quieras
	

	
		<br>![El botón de la parte inferior de la pantalla está anclado](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WM-Epingler-Un-Ou-Des-Champs.gif)

En la versión 27, puede fijar uno o varios controles (botones, principalmente) en una posición fija de la pantalla. 
Por ejemplo: 

- abajo a la derecha,

- abajo en el centro, 

- ... 


Los márgenes con respecto a los bordes de la pantalla son los que se definen al posicionar el control. 
Una opción en la ventana de 7 pestañas permite ocultar la control cuando se desplaza la ventana principal. 

	



	


<a name="camera"></a>

	
		INTEGRE LA CAPTURA DE FOTOS Y VÍDEOS EN SUS APLICACIONES CON LA NUEVA CÁMARA control
	

	
		 
	
		¿Por qué un control Cámara?
	

	
		Todos los dispositivos móviles que tienen un objetivo y pueden hacer fotos o vídeos ya tienen una aplicación de cámara. 
Entonces, ¿por qué incluir este nuevo control Cámara? 
Simplemente para que puedas hacer el tratamiento que quieras sin que el usuario final tenga que salir de la aplicación. 
Con la control Cámara, puede añadir botones, leyendas y campos de entrada; iniciar procesos automáticamente sin salir de la aplicación, sin acciones adicionales, como en una ventana estándar:

- enviar las fotos a la carpeta adecuada con un solo clic,

- fotos de archivo en el archivo HFSQL de un cliente,

- hacer un inventario de las instalaciones, y enviarlo directamente por Email,

- Realice el OCR sobre un texto y Process el texto de forma programada, por ejemplo, para preparar el contenido de un Email o para buscar en una base de datos,

- realizar el reconocimiento de automatic Image a través de un servicio web de IA,

- mostrar la información nutricional de un producto escaneando su código de barras,

- ... 




Para obtener más información, consulte [Control Cámara en una aplicación móvil](../WDChamp/1013349.md).
	


 
	
		Nueva control Cámara
	

	
		WINDEV Mobile 27 incluye un nuevo control "Cámara". 
El control Cámara permite: 


- tomar fotos 
	

- grabar videos 
	

- escanear y descodificar códigos de barras 
	Este control avanzado ofrece una amplia gama de ajustes. 
	En WINDEV Mobile 27, un nuevo control Smart específico permite implementar las funcionalidades del nuevo control Cámara. 




Para obtener más información, consulte [Control Cámara en una aplicación móvil](../WDChamp/1013349.md).
	


 
	
		Principales funcionalidades
	

	
		El control Smart Cámara, basado en el nuevo control Cámara, cuenta con numerosas funcionalidades e incluye todo lo que esperan los usuarios finales: 

- Selección del modo de visualización: homotético, ampliado,... 

- Zoom táctil (o definido mediante programación), 

- Tocar para enfocar (o enfocar mediante programación), 

- Flash y linterna, 

- Cámara frontal y trasera, 

- ... 


Todas estas funciones también están disponibles a través de la programación. 
Su aplicación tiene control completa en el control Cámara. 


Para obtener más información, consulte [Control Cámara en una aplicación móvil](../WDChamp/1013349.md).
	


 
	
		Nuevas funciones de la cámara WLanguage
	

	
		Las funciones control Cámara se han unificado en WLanguage: esta nueva familia de funciones se identifica con el prefijo "Cámara".. 
Las antiguas funciones de vídeo siguen siendo compatibles. 

Para obtener más información, consulte [Control Cámara en una aplicación móvil](../WDChamp/1013349.md).
	


 
	
		Nuevas propiedades WLanguage del control Cámara
	

	
		WLanguage incluye nuevas propiedades para facilitar la programación: 


- [Cámara](../Proprietes/1410087704.md)(anverso, reverso)

- [Flash](../Proprietes/1410087703.md)

- [Torch](../Proprietes/1410087848.md)

- [WorkingDirectory](../Proprietes/1000026169.md)

- ... 


Estas propiedades le permiten manipular fácilmente el control mediante la programación. 

Para obtener más información, consulte [Control Cámara en una aplicación móvil](../WDChamp/1013349.md).
	


 
	
		Agregar procesos a la captura de fotos y videos
	

	
		El control Cámara es totalmente personalizable. 
Al ser un control, puede utilizarse en una ventana con otros controles. 
Por ejemplo, el desarrollador puede crear botones preestablecidos como "Mover a la carpeta", "Cifrar", "Enviar el resultado del OCR en el cuerpo del mensaje" o "Ejecutar el reconocimiento de Image",... 


Para obtener más información, consulte [Control Cámara en una aplicación móvil](../WDChamp/1013349.md).

		WINDEV MOBILE EN EL EDITOR DE WINDEV
	

	
		 
	
		Reminder: WINDEV puede abrir y manipular WINDEV Mobile proyectos
	

	
		Para facilitar el desarrollo multiplataforma, WINDEV permite abrir proyectos de WINDEV Mobile y WEBDEV. 
Esto evita la duplicación de código y objetos y la realización de operaciones innecesarias. Los objetos se comparten de inmediato. 
El mismo proyecto en WINDEV puede contener configuraciones de WINDEV, iOS y Android (e incluso configuraciones de WEBDEV). 
En el modo de edición, puede cambiar sin problemas de una configuración a otra. 
Nota: es necesario tener una licencia de WINDEV Mobile y/o WEBDEV.

		NUEVAS FUNCIONES COMUNES DE IOS/ANDROID
	

	
		 
	
		OpenAsynchronous: La función Open con callback permite un código más lineal
	

	
		La función [OpenAsynchronous](../WDLang1/1410087804.md) acaba de ser añadida a WLanguage. 
Este nuevo método permite abrir ventanas móviles fácilmente. 
Basta con pasar 3 elementos: 

- nombre de la ventana, 

- parámetros, si es necesario: identificador del cliente...

- callback al que se llamará cuando se cierre la ventana.


Esta sintaxis permite escribir código lineal y abrir las ventanas sin bloquear la aplicación. 
El código de cierre de la ventana está justo debajo del código de apertura. 
Ya no es necesario escribir este código en la ventana "Cerrar ventana infantil" Event. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=OuvreAsynchrone.jpg)

		AJUSTES DE TEXTO SOFISTICADOS
	

	
		 
	
		Tamaño de letra: Unidades adicionales para más control
	

	
		En la versión 27, puede utilizar nuevas unidades para establecer el tamaño de la fuente de los controles en dispositivos móviles: Etiquetas control y caracteres introducidos, iconos, botones, cuadros combinados... 
Las unidades disponibles son: 

- unidad heredada (uw)

- punto (pt)

- pixel lógico (px). 


Se admiten valores decimales (2 dígitos después del punto decimal). 
Estas unidades permiten definir con exactitud cualquier tamaño. Las UI son más precisas. 
Si no se especifica ninguna unidad, se utilizará la unidad heredada. 

	


 
	
		Ajustar el espaciado entre caracteres
	

	
		El espaciado de los caracteres determina la distancia entre ellos. 
En la versión 27, puede ajustar el espaciado de los caracteres: Etiqueta control, texto de entrada,... 
Esto le permite comprimir o expandir el texto sin cambiar la fuente. 
	


 
	
		Ajustar el espacio de Line
	

	
		El interlineado define el espacio entre dos líneas. 
En la versión 27, puede ajustar con precisión el interlineado de un texto, eligiendo la unidad y especificando dos decimales si es necesario. 

	


 
	
		UI: Mayor personalización del texto
	

	
		Utilizando estas tres nuevas características: 

- unidad de tamaño de letra, 

- espaciado entre caracteres, 

- Espacio Line.


puede personalizar totalmente los textos y su disposición.

<a name="and_for_android"></a>

	
		Y MÁS PARA ANDROID
	

	
		 
	
		Nuevas características comunes para móviles
	

	
		El desarrollo para Android nunca ha sido tan completo como con WINDEV Mobile 27. 
Aproveche las nuevas y potentes funcionalidades que se suman a las que ya están disponibles. 
Android evoluciona y WINDEV Mobile 27 también, y en especial, su compatibilidad con Android 12 . 
WINDEV Mobile 27 para Android beneficia de las nuevas características comunes de la versión 27, además de nuevas características específicas para dispositivos móviles, más Android características nuevas específicas. 

	


 
	
		Descarga programada de archivos
	

	
		<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WM-TelechargementFichier.jpg)

Android dispone de un "Gestor de descargas", que descarga los archivos de Background. 
WINDEV Mobile 27 le permite control fácilmente este gestor de descargas. 
Los archivos se descargan en el Background, independientemente de la aplicación. 
Las nuevas funciones de descarga permiten definir las opciones de descarga: 


- nombre de los archivos (URL) 
	

- Sólo Wi-Fi 
	

- devolución de llamada al final de la descarga (opción) 
	

- ... 
	



	


 
	
		Manipule fácilmente los archivos en Android 11 y posteriores
	

	
		Android 11 y posteriores no permiten que una aplicación abra directamente archivos (Image, vídeo, texto, etc.) creados por otras aplicaciones. 
Esto significa que las últimas versiones de Android no permiten el acceso a archivos externos utilizando su ruta físico . 
Es necesario manipular estos archivos por su URI. 
Recordatorio: la URI es un identificador único que describe la ubicación del archivo. La URI sólo puede obtenerse una vez que el usuario ha seleccionado manualmente el archivo. 
WINDEV Mobile 27 incluye nuevas funciones de URI WLanguage para leer y escribir archivos externos. 

	


 
	
		WLanguage: Nuevas funciones XML
	

	
		En la versión 27, el manejo de archivos XML en Android es más fácil que nunca . 
Se añaden nuevas funciones de WLanguage a la familia XML . 
En particular, estas funciones permiten crear consultas xPath y firmar documentos XML. 

	


 
	
		Nuevos tipos de lenguaje WLanguage: Border, Corner, Gradient, Background
	

	
		El enfoque WYSIWYG del editor de ventanas permite definir la UI de las aplicaciones Android. 
Puedes editar esta UI en el código: añadir una Border roja alrededor de una control, por ejemplo. 
Realizar estos cambios en el código es ahora más fácil en Android, gracias a los nuevos tipos de variables: Borde, Esquina, Gradient, Background ... 

	


 
	
		Pruebas más rápidas (GO) en el dispositivo
	

	
		En la versión 27, la realización de una serie de pruebas (mediante la opción GO) en un dispositivo móvil es mucho más rápida. 
Las recompilaciones están optimizadas. 
Normalmente, cuando se conectan varios dispositivos al PC (1 Phone y 1 tableta, 1 Phone y el emulador, 2 Phone s, etc.), se le pide que elija el dispositivo a utilizar. 
En la versión 27, puede especificar el dispositivo deseado sin que se le pregunte de nuevo. 

	


 
	
		Tipo Image: Gestionar la información Exif
	

	
		Ahora puedes gestionar de forma nativa los datos EXIF de las fotos en los proyectos Android. 

	


 
	
		Nuevas funciones de WLanguage para gestionar los permisos
	

	
		<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=shutterstock_1850326291.%20PPpsd%20copie.jpg)
En la versión 27, puede solicitar permisos mediante programación. 
Esto permite anticipar las peticiones de Permission que tendrá que hacer una aplicación. 
Las nuevas funciones Permission permiten solicitar un Permission o comprobar si ya se ha concedido un Permission. 

	


 
	
		Funciones de excepción de WLanguage: nuevas funciones
	

	
		Una "excepción" es un error que interrumpe la ejecución de una aplicación en Normal. 
Si esta excepción no es capturada por el código, la aplicación se detiene. 
Un error de WLanguage es un ejemplo de excepción: si el código de su aplicación no manipular la excepción, la aplicación se detiene. 
El manejo de excepciones se hace más rico y personalizable en la versión 27. 
Las familias de lenguajes de excepción y error incluyen ahora nuevas funciones. 
Por ejemplo, la nueva función [ErrorPropagate](../WDLang1/3034015.md) vuelve a arrojar un error. 

	


 
	
		pasar los parámetros del comando Line
	

	
		Puede pasar parámetros a las aplicaciones Android . 
Hay tres formas posibles de pasar parámetros: 


- Al probar la aplicación en el entorno 
	

- Del Centro Android control 
	

- A través de la función ExeRun WLanguage . 
	En la aplicación, los valores de los parámetros pasados pueden recuperarse utilizando la función WLanguage de la línea de comandos, como es habitual . 
	



	


 
	
		Estilos control: compatible con SVG
	

	
		Las imágenes SVG son ahora compatibles con los estilos control para Android. 

	


 
	
		Publica tus aplicaciones en la Amazon Appstore
	

	
		Las aplicaciones de Android desarrolladas con WINDEV Mobile 27 pueden publicarse en la Amazon Appstore. 

	


 
	
		Puertos serie en Android: Dispositivo o equipo industrial inteligente
	

	
		WINDEV Mobile 27 permite gestionar las conexiones RS-232. 
Puede ser un dispositivo móvil, o un dispositivo inteligente integrado en un equipo industrial, como un horno industrial control. 
La familia sxxx de funciones WLanguage para la gestión de puertos serie está disponible en Android. 

	


 
	
		Compras dentro de la aplicación: nuevas API compatibles
	

	
		Android ha implementado una nueva forma de integrar las compras In-App. 
WINDEV Mobile 27 sigue estos cambios. 
No es necesario que modifique su código WLanguage existente. Simplemente recompile la aplicación en la versión 27. 

	


 
	
		iParameterPDF
	

	
		La función [iParameterPDF](../WDLang5/3046007.md) ya está disponible para Android. 
Esta función permite configurar los archivos PDF generados a partir de un reporte.

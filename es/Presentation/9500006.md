<a name="3_new_smart_controls_in_windev_27"></a>

	
		3 NUEVOS CONTROLES INTELIGENTES EN WINDEV 27
	

	
		 
	
		WINDEV 27 incluye 3 nuevos controles inteligentes
	

	
		Los controles inteligentes ahorran un tiempo valioso durante la creación de la interfaz de usuario Process, y hacen que sus aplicaciones sean inmediatamente más sofisticadas. 
Se introducen 3 nuevos controles inteligentes en WINDEV 27: 


- Entrada predictiva de direcciones de correo electrónico con tokens, 

- Formulario con entrada obligatoria no bloqueante, 

- Arrastrar y soltar filas en un control Looper.


Recordatorio: Los controles Smart se proporcionan con su código fuente y, por lo tanto, son completamente modificables y personalizables.

		Nueva control Smart: Formulario con entrada obligatoria no bloqueante
	

	
		El "Formulario con entrada requerida no bloqueante" control Smart está preconfigurado con un modo de entrada no bloqueante. 
Se puede utilizar para crear ventanas de entrada, o para comprender las diferentes posibilidades de entrada requerida no bloqueante. <br>![Los campos de entrada obligatorios que se dejan vacíos y los datos que no coinciden con la máscara de entrada se resaltan en rojo. Puedes navegar entre los controles](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WD-Champ-Metier-Saisie-Obligatoire-Non-Bloquante.gif)


	


<a name="email"></a>
 
	
		Nuevo control Smart: Entrada de correo electrónico tokenizado Address
	

	
		La "entrada de correo electrónico tokenizado Address" control Smart muestra la lista de destinatarios seleccionados como tokens. 
Aparecerá una lista de sugerencias mientras escribes las direcciones de los destinatarios. <br>![Cada destinatario seleccionado aparece como un Token](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WD-Champ-Metier-Saisie-Email-Avec-Jetons.gif)

<a name="table_control_new_features"></a>

	
		NOVEDADES EN EL CONTROL TABLE
	

	
		 
	
		control Tabla: Columna de tipo Atributo
	

	
		Las columnas de "atributos" se introducen en control Tabla en la versión 27. 
Este tipo de columna puede almacenar un valor que no se mostrará en la control Tabla. 
Esto evita la creación de columnas ocultas para almacenar los valores necesarios durante la fase de desarrollo. 
	


 
	
		TableSearchEverywhere: Búsqueda en todas las columnas de la control Tabla
	

	
		La nueva función [TableSearchEverywhere](../WDLang1/1410088362.md) WLanguage busca una cadena en toda la control Tabla. 
Ya no es necesario buscar columna por columna. 
	


 
	
		StoredValue Property: Fuente de vinculación
	

	
		La propiedad [StoredValue](../Proprietes/2510127.md) está ahora disponible para controles Tabla poblada por la vinculación de datos. 
Esta Property permite identificar la fuente de datos de la fila actual en el control Tabla. 
	



	


<a name="new_features_of_controls"></a>

	
		NUEVAS CARACTERÍSTICAS DE LOS CONTROLES
	

	
		 
	
		control Tabla dinámica: Recuerda el último criterio seleccionado
	

	
		En la versión 27, los criterios seleccionados se guardan para la próxima vez que el usuario abra el control: los usuarios finales ya no tienen que especificar los criterios cada vez. 
	


 
	
		Abrir ventanas con pestañas más rápidamente
	

	
		¿Sus aplicaciones contienen ventanas con pestañas? 
Estas ventanas se abren más rápidamente en la versión 27. 
	


 
	
		control Calendario: Muestra el número de la semana
	

	
		La control Calendario en la versión 27 permite visualizar el número de la semana. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WD%20Calendrier%20-%20Affichage%20semaine.jpg)


	


 
	
		Barras de desplazamiento: nuevo estilo "Windows 11" disponible
	

	
		Este estilo está disponible para Windows 10 y 11. 
La versión 27 tiene un nuevo estilo de barra de desplazamiento: el modo "Windows 11". 
En este modo, la barra de desplazamiento aparece como una fina Line cuando el cursor se aleja de ella. 
Al pasar por encima, "recupera" su anchura habitual. 
	


 
	
		Ventana interior: Estilo
	

	
		En la versión 27, se puede definir un estilo para un control Ventana interna.
	


 
	
		Ventana interna y control Disposición: Aplicar plantilla
	

	
		En la versión 27, se pueden aplicar plantillas de tema a los controles internos de Windows y Layout en tiempo de ejecución. 
Esto permite cambiar la apariencia de una aplicación en ejecución, por ejemplo, para cambiar a un tema oscuro. 
	


 
	
		HTML control: Depuración más fácil
	

	
		Un editor HTML o una pantalla HTML control pueden contener código JavaScript. 
En la versión 27, puedes activar el depurador de Chrome y adjuntarlo al HTML de control para depurar el código JavaScript. 

Para más detalles, consulte la ayuda sobre [HTMLEnableDebugger](../WDLang1/1410087849.md).
	


 
	
		Texto: Unidades de tamaño de letra, espaciado entre caracteres y Line
	

	
		Hay nuevos ajustes disponibles para el texto presente en los control s (ya sea la etiqueta del control, el texto introducido o simplemente mostrado). 
Esto facilita el ajuste del texto: 

- se introducen nuevas y más precisas unidades de espacio/tamaño de Line (píxel, punto).

- puede utilizar valores decimales para afinar los ajustes.

- puede ajustar el espaciado de los caracteres.

- puede ajustar el espacio de Line.

- ... 



	


 
	
		Ocultar el título de la ventana
	

	
		<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WD%20-%20Masquage%20titre%20fen.jpg)

Esto hace que los controles de Looper sean más elegantes y legibles. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Rupture%20hauteur%20variable.png)

<a name="windev_install_via_pcscloud"></a>

	
		INSTALACIÓN DE WINDEV A TRAVÉS DE PCSCLOUD
	

	
		 
	
		Instalación a través de PCSCloud: Nombre del dominio en el enlace
	

	
		Puede almacenar los archivos instalación de sus aplicaciones WINDEV en la nube, utilizando PCSCloud. 
El nombre de dominio de Default (PCSCloud.net) puede ser sustituido por su propio nombre de dominio, como el sitio web de su empresa, por ejemplo. 
	


 
	
		Instalación a través de PCSCloud: HTTPS de Default
	

	
		Como se ha mencionado anteriormente, puede almacenar los archivos instalación de sus aplicaciones en la nube, utilizando PCSCloud. 
En ese caso, comparte un enlace con los usuarios para que puedan instalar la aplicación. 
En la versión 27, Default utiliza HTTPS, lo que mejora la seguridad. 
	


 
	
		Su Tienda privada en PCSCloud
	

	
		En la versión 27, puede permitir que los usuarios instalen sus aplicaciones desde un Tienda privada. 
Esta tienda puede alojarse en PCSCloud.net: no tiene que gestionar una infraestructura adicional. 
Existe una solución "todo en uno".
	



	


<a name="windev_private_store
users_install_your_windev_applications_from_your_own_store"></a>

	
		TIENDA PRIVADA DE WINDEV: LOS USUARIOS INSTALAN SUS APLICACIONES WINDEV DESDE SU PROPIA TIENDA
	

	
		 
	
		Distribuya software fácilmente dentro de una empresa
	

	
		La versión 27 incluye una tienda privada. Esta tienda permite agrupar las aplicaciones utilizadas dentro de una empresa. 
Es posible limitar el acceso a la tienda solo desde dentro la empresa, o definir un acceso público . 
Un portal con autenticación opcional permite que los usuarios autorizados instalen aplicaciones. 
Esto permite estandarizar y simplificar la distribución de aplicaciones dentro de una empresa. 
	


 
	
		Implementación fácil con la infraestructura existente.
	

	
		La versión 27 incluye una tienda privada. 
Solo es necesario instalarla en un servidor de la empresa o en la nube (PCSCloud.net, por ejemplo). 
En WINDEV, al crear la instalación de la aplicación, seleccione "Tienda privada". 
La aplicación se instala en el servidor de la tienda de forma automática o manual al final de la generación de la instalación. 
	


 
	
		Aplicación pública, aplicación privada
	

	
		Una tienda puede contener aplicaciones a las que se puede acceder con y sin autenticación. 
Al acceder al servidor con credenciales de inicio de sesión, los usuarios solo ven las aplicaciones que están autorizados a instalar. 
Esto permite mantener ciertas aplicaciones "privadas" y limitar su distribución.
	


 
	
		¿Cómo funciona para los usuarios?
	

	
		Puede proporcionar a los usuarios finales una de las siguientes opciones: 


- el enlace directo a la aplicación, 

- el enlace de su tienda. 
	Si la autenticación es necesaria, se puede implementar: 
	

- mediante el Groupware Usuario 

- mediante oAuth (Microsoft, Google, Facebook, etc.). 


La aplicación se instala automáticamente en el dispositivo: su servicio de asistencia no estará sobrecargado de solicitudes, lo que reduce costos operativos. 
	


 
	
		Actualización automática
	

	
		Cuando se instala una nueva versión de la aplicación en la tienda, los usuarios verán un aviso de actualización disponible al abrir la aplicación.
	


 
	
		Historial de versiones en el servidor
	

	
		El historial de versiones de cada aplicación se guarda en el servidor, lo que permite volver a una versión anterior si es necesario.

		Funciones de escaneo nativas (mejores que las de Twain)
	

	
		Muchos Scanner s ya no proporcionan una interfaz Twain para comunicarse con el Scanner a través de la programación. 
WINDEV y WEBDEV Server 27 introducen un nuevo tipo y nuevas funciones WLanguage para gestionar de forma nativa los escáneres mediante programación. 
El tipo Scanner define el Scanner que se utilizará. Puede definir varias variables [Scanner](../WDLang6/1410087808.md) en un programa para utilizar diferentes escáneres en paralelo. 
[ScanSelect](../WDLang6/1410087835.md), una nueva función de WLanguage, abre la ventana de selección de Scanner por defecto. 
El escaneo puede iniciarse a través de la programación o abriendo la ventana de configuración del controlador específico.


```wl
MyScanner is Scanner = ScanSelect()
// Scan all the pages in the feeder to a PDF file
MyScanner.ToPDF("File.PDF")
```


Para obtener más información, consulte: 

- [Escanear un documento (mediante un controlador WIA)](../WDLang6/1410087869.md),

- [Funciones de escáner](../WDLang6/1000004104.md).

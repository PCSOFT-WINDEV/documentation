
## ThreadStop (Función)

	<IMG style="float:left;margin-right:10px;" src="https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=warning_big.png"><B>Advertencia</B><BR>
	 No se recomienda el uso de esta función porque detiene el hilo actual de manera abrupta. <a name="Use"></a>
<a name="description"></a>
Detiene un hilo secundario.

**Recordatorio**: Un hilo es un proceso que se ejecuta en paralelo con la aplicación actual (hilo principal). Esto le permite iniciar una tarea en segundo plano (hacer una copia de seguridad, etc.).




<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = ThreadStop(<Thread name> [, <Maximum timeout>])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se detuvo el hilo o si el hilo no existe,

- <u><u><u><u>False</u></u></u></u> si el hilo está bloqueado




**`<Thread name>: (Cadena de caracteres)`**

Nombre del hilo a detener. Este nombre se asigna al ejecutar el hilo (con la función [ThreadExecute](../WDLang1/3077024.md)). 

Casos especiales:  

- Si este parámetro corresponde a una cadena vacía (""), el hilo actual se detiene. Si el hilo actual es el hilo principal, la función **ThreadStop** es equivalente a la función [EndProgram](../WDLang1/3013033.md). 
	![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Solo se permite este comportamiento. La función **ThreadStop** solo puede utilizarse en el hilo actual.

- Si el nombre del hilo no existe, la función **ThreadStop** no tiene ningún efecto y devuelve <u><u><u><u>True</u></u></u></u>. 

- Si el nombre del hilo es un hilo hijo o un hilo hermano, **ThreadStop** espera a que la función WLanguage actual (por ejemplo, HReadFirst) termine. En este caso, **ThreadStop** es una función de bloqueo.

- Si este parámetro corresponde a la constante *threadMain*, el hilo principal se detiene.

- De lo contrario, se produce un error WLanguage. 




**`<Maximum timeout>: (Entero opcional)`**

Tiempo máximo de espera en centésimas de segundo. 

- Si al final de la duración especificada, no se ha detenido el hilo &lt;Result&gt; se establece en <u><u><u><u>False</u></u></u></u>. 

- Si no se especifica este parámetro o corresponde a la constante *Infinite*, la espera es infinita.


Este parámetro puede corresponder a: 

- un número entero que corresponde al número de centésimas de segundo,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, 1 s o 10 ms).

## Observaciones




### Caso especial: Detener un hilo que contiene la función SocketWaitForConnection
<a name="caso_especial_detener_hilo_que_contiene_funcion_socketwaitforconnection_ELTPARAGRAPHE000130"></a>

Para detener el hilo correctamente con la función **ThreadStop**, se debe incluir la función [SocketWaitForConnection](../WDLang3/3070006.md) en un bucle:


```wl
Timeout = 30
LOOP
	SocketWaitForConnection("MySocket", Duration)
END
```


## DDEEvent( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Asocia un procedimiento WLanguage con un DDE Event: cuando el evento DDE ocurre, el asociado WLanguage Procedure se ejecuta automáticamente.

**Observación**: Esta función sólo debe utilizarse para el "Nuevo diálogo DDE" de WINDEV o WEBDEV.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Trigger a call to the "DataReception" procedure whenever 
// the program receives a data sent via DDE by another application
DDEEvent("DataReception", DDEtoReceive)
```

```wl
PROCEDURE DataReception
IF (_DDE.Topic = "MySubject") AND (_DDE.Item = "MyItem") THEN
	_DDE.Cr = 1
	Info("Data received: " + _DDE.String)
ELSE
	_DDE.Cr = 0
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`DDEEvent(<Procedure name> , <DDE events>)`
---

**`<Procedure name>: (Cadena de caracteres)`**

Nombre del procedimiento WLanguage a ejecutar. Este Procedure puede ser un Procedure global o un Procedure local de ventana actual. Si este parámetro corresponde a una cadena vacía (""), la asociación "Procedure - DDE Event" se cancela.

**`<DDE events>: (Constante o combinación de constantes de tipo Integer)`**




|   |   |   |
| --- | --- | --- |
| *DDEtoAnswer* | El programa recibe una solicitud de datos de otra aplicación.<br><br>&lt;Procedure&gt; puede averiguar el tema y el elemento afectado por _DDE.Topic y _DDE.Item (_DDE.String no tiene ningún valor significativo). | <br><br>- Si la solicitud de datos es aceptada, &lt;Procedure&gt; debe actualizarse:<br><br>	- _DDE.Cr = 1<br><br>	- DDE.String = &lt;valor a enviar a la aplicación que hizo la petición&gt;<br><br><br><br><br>- Si la solicitud de datos no es aceptada, &lt;Procedure&gt; debe actualizar _DDE.Cr = 0<br><br><br> |
| *DDEtoExecute* | El programa recibe una orden de ejecución que proviene de otra aplicación.<br><br>&lt;Procedure&gt; puede averiguar el tema afectado por _DDE.Topic y los datos recibidos por _DDE.String (_DDE.Item no tiene valor significativo). | <br><br>- Si el comando se ejecuta, &lt;Procedure&gt; debe actualizar _DDE.Cr = 1<br><br>- Si el comando no se ejecuta, &lt;Procedure&gt; debe actualizar _DDE.Cr = 0<br><br><br> |
| *DDEtoLink* | El programa recibe una solicitud de enlace "warm" o "hot" a un dato. Esta solicitud proviene de otra aplicación.<br><br>&lt;Procedure&gt; puede averiguar el tema y el elemento afectado por _DDE.Topic y _DDE.Item (_DDE.String no tiene ningún valor significativo). | <br><br>- Si se crea el enlace, el programa debe llamar a [DDEUpdate](../WDLang1/3028011.md) cada vez que se modifiquen los datos correspondientes al tema y al artículo. &lt;Procedure&gt; debe actualizar _DDE.Cr = 1<br><br>- Si el enlace no se crea, &lt;Procedure&gt; debe actualizar _DDE.Cr = 0<br><br><br> |
| *DDEtoModify* | El programa recibe la señal de que el valor de un dato relacionado ha sido modificado.. Esta señal proviene de otra aplicación.<br><br>Después del evento DDEtoModify, debe llamar a [Enlace DDELink](../WDLang1/3028010.md) o [DDEWarmLink](../WDLang1/3028014.md) para que se llame a &lt;Procedure&gt;.<br><br>&lt;Procedure&gt; puede encontrar el tema y el elemento afectado por _DDE.Topic y _DDE.Item. | <br><br>- Si la modificación es aceptada, &lt;Procedure&gt; debe actualizar _DDE.Cr = 1<br><br>- Si la modificación no es aceptada, &lt;Procedure&gt; debe actualizar _DDE.Cr = 0<br><br>- Si el enlace es un enlace caliente, _DDE.String contiene el nuevo valor de los datos de relacionado; de lo contrario, _DDE.String es set a "" (cadena vacía)<br><br><br> |
| *DDEtoReceive* | El programa recibe datos de otra aplicación.<br><br>&lt;Procedure&gt; puede encontrar el tema y el ítem afectado por _DDE.Topic y _DDE.Item y los datos recibidos por _DDE.String. | <br><br>- Si los datos son aceptados, &lt;Procedure&gt; debe actualizar _DDE.Cr = 1<br><br>- Si los datos no son aceptados, &lt;Procedure&gt; debe actualizar _DDE.Cr = 0<br><br><br> |

<a name="NOTE0_1"></a>

## Observaciones




### Límites
<a name="limites_ELTPARAGRAPHE000097"></a>

- **DDEEvent** no puede ser llamado en el constructor o destructor de la clase.

- [Multitask](../WDLang1/3015004.md) no debe ejecutarse en un procedimiento llamado por **DDEEvent**.

- El depurador no puede ser llamado por Ctrl + Pausa en un Procedure llamado por **DDEEvent**. Debe añadir la palabra clave [detenerse](../Motscles/1511014.md).



<a name="NOTE0_2"></a>




### Valores predeterminados
<a name="valores_predeterminados_ELTPARAGRAPHE000118"></a>

Por defecto, _DDE.Cr es set a 1 (OK) y _DDE.String es set a "" (cadena vacía).
<a name="NOTE0_3"></a>




### Desencadenando el mismo Procedure para varios eventos
<a name="desencadenando_mismo_procedure_para_varios_eventos_ELTPARAGRAPHE000126"></a>

El mismo Procedure puede ser activado por la acción de varios eventos DDE. En este caso, la sintaxis es, por ejemplo:


```wl
DDEEvent("DataReception", DDEtoReceive + DDEtoModify)
```


Para estos dos eventos, debe identificar en &lt;nombre de Procedure&gt; el evento DDE que desencadenó la llamada a Procedure. Para ello, debe comprobar el _DDE.Event Variable.

El _DDE.Event Variable regresa, según el caso:

- *DDEtoAnswer*.

- *DDEtoReceive*.

- *DDEtoExecute*.

- *DDEtoLink*.

- *DDEtoModify*.

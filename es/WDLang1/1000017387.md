
## AnimationPlay( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Reproduce una animación cuando se modifica un control, un grupo de control s o una ventana.

El principio es el siguiente:

1. Llama a [AnimationPrepare](../WDLang1/1000017386.md).

2. Modificar los controles en el código (rellenar, mover, cambiar de plano, etc.). Estas modificaciones no se visualizan. Estas modificaciones se mostrarán cuando se llame a **AnimationPlay**.

3. Utilice **AnimationPlay**. Esta función se utiliza para pasar del estado "antes de la modificación" al estado "después de la modificación" mediante la reproducción de una animación.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Prepare the animation
// The animation will be played in the window
AnimationPrepare(WIN_Customer, WIN_Customer.X, WIN_Customer.Y, ...
		WIN_Customer.Width, WIN_Customer.Height)
// Modify controls 
BTN_Down.Visible = False
IMG_Detail.Height += 50
WIN_Customer.Plane--
// Performs the modifications and plays the animation
AnimationPlay(animFadeIn)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`AnimationPlay(<Type of animation> [, <Animation duration>])`
---

**`<Type of animation>: (Constante de tipo Integer)`**

Tipo de animación a reproducir:


|   |   |
| --- | --- |
| *animCoverDown* | El nuevo elemento aparece desde arriba y cubre el elemento inicial. |
| *animCoverFromCenter* | El nuevo elemento aparece desde el centro y cubre el elemento inicial.<br><br> |
| *animCoverFromCorners* | El nuevo elemento aparece por las esquinas y cubre el elemento inicial.<br><br> |
| *animCoverLeft* | El nuevo elemento aparece por la derecha y cubre el elemento inicial. |
| *animCoverRight* | El nuevo elemento aparece por la izquierda y cubre el elemento inicial. |
| *animCoverUp* | El nuevo elemento aparece desde abajo y cubre el elemento inicial. |
| *animFadeIn* | Animación de fade-in. |
| *animFlipAxisX* | Da la vuelta al eje horizontal.<br><br> |
| *animFlipAxisY* | Da la vuelta alrededor del eje vertical.<br><br> |
| *animFlipDownwardDiagonal* | Da la vuelta a la diagonal que va de la parte superior izquierda a la inferior derecha.<br><br> |
| *animFlipUpwardDiagonal* | Da la vuelta a la diagonal que va de la parte superior derecha a la inferior izquierda.<br><br> |
| *animShrinkAndCoverFromCorners* | El elemento inicial se encoge y el nuevo elemento aparece por las esquinas.<br><br> |
| *animSlideDown* | Desplácese hacia abajo. |
| *animSlideLeft* | Deslizar hacia la izquierda. |
| *animSlideRight* | Deslizar hacia la derecha. |
| *animSlideUp* | Deslizar hacia arriba. |



**`<Animation duration>: (Entero o duración opcional)`**

Duración de la animación en centésimas de segundo. Este parámetro puede corresponder a: 

- un número entero que corresponde al número de centésimas de segundo,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, "1cs" o "10 ms"). 


Recomendamos utilizar una duración corta (hasta 1 segundo)..
La aplicación está bloqueada mientras la animación no esté terminada.. Si el usuario hace clic o pulsa una tecla, la animación se detiene y los controles se muestran en su estado final.
**Observación**: El número de imágenes reproducidas se ajusta según la velocidad de el equipo. Por debajo de un número determinado de Image s por segundo, la Image es demasiado brusca y la animación se cancela. El número de imágenes por segundo lo configura [AnimaciónMinFPS](../WDLang1/1000017400.md). 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si se llama a la función [AnimationPrepare](../WDLang1/1000017386.md) dos veces sin llamar a la función **AnimationPlay**, se produce un error.

- Para evitar ralentizar la apertura de las ventanas, las funciones [AnimationPrepare](../WDLang1/1000017386.md) y **AnimationPlay** no tienen ningún efecto cuando se utilizan en el evento "Inicialización" de la ventana.

- Las funciones [AnimationPrepare](../WDLang1/1000017386.md) y **AnimationPlay** no tienen ningún efecto: 

	- si "Activar animación de controles en el proyecto" no está marcada. Esta opción está disponible en la ventana de description del proyecto, pestaña "Avanzado", botón "Animación de controles".. 

	- o si no se llamó a [AnimaciónPermitida](../WDLang1/1000017399.md) para habilitar las animaciones.

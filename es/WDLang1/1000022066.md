


## InAppleWatchSimulatorMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica si el código se ejecuta o no en el simulador de Apple Watch.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Code run in the Apple Watch simulator? 
IF InAppleWatchSimulatorMode() THEN
	// Specific code for the simulator
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = InAppleWatchSimulatorMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecuta el código en el simulador de Apple Watch,

- <u><u><u><u>False</u></u></u></u> en caso contrario.







## Observaciones


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000184"></a>

La función **InAppleWatchSimulatorMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InAppleWatchSimulatorMode** pueden compartirse entre varios productos sin generar errores de compilación.


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000198"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.

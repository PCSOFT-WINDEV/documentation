
## InvalidInputShowMessage( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Muestra un error de entrada no válido para la control especificada. Esta función no comprueba los datos introducidos.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF NOT bCheckPassword(EDT_Password) THEN
	InvalidInputShowMessage(EDT_Password, ...
		"The password must contain at least one number, one capital letter and one symbol.")
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`InvalidInputShowMessage(<Control used> [, <Text>])`
---

**`<Control used>: (Nombre del control)`**

Nombre de la control afectada por el error. 

**`<Text>: (Cadena de caracteres opcional)`**

Mensaje de error a mostrar. 

Si no se especifica este parámetro, se utilizará [InvalidInputMessage](../Proprietes/1000026245.md) Property de control y si falta este valor, se mostrará un mensaje genérico.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- El estado "Error" desaparecerá una vez que los datos introducidos sean modificados por el usuario o a través de la programación.

- Puede personalizar el modo de visualización de los errores en la ventana de control description.

- Para comprobar los datos introducidos en los controles, utilice [InvalidInputDetect](../WDLang1/1000026180.md) o [InvalidInputListControl](../WDLang1/1000026244.md).


## RunReportsAndQueries( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia informes y consultas y abre el proyecto WINDEV correspondiente a la aplicación actual.. Si se inicia Reports & Queries y el proyecto WINDEV está abierto, también se puede utilizar **RunReportsAndQueries**:

- abrir un reporte. Este reporte debe ser incluido en una librería WINDEV (archivo.WDL) de la aplicación.

- iniciar el asistente para la creación de reporte.

- iniciar el asistente para la creación de consultas.




Para obtener más información, consulte [Informes y Consultas](../Presentation/3088003.md).

**Observaciones**: 

- Esta función no está disponible en el modo de prueba (ventana o proyecto Go).. 

- Si "Reports and Queries" no está instalado en el equipo, la función **RunReportsAndQueries** muestra una ventana específica que permite al usuario buscar e instalar el software.



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=LanceEtatsEtRequ%EAtes%20-%20HC%20N%B0001.gif)

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Opens the WINDEV project corresponding to the current application
// in Reports & Queries
RunReportsAndQueries()
```


<a name="Example2"></a>



```wl
// Opens "RPT_MyStatReport" in Reports & Queries
RunReportsAndQueries(RPT_MyStatReport)
```


<a name="Example3"></a>



```wl
// Starts the wizard for query creation 
// in Reports & Queries
RunReportsAndQueries(CreateQuery)
```

## Sintaxis

### Abrir el proyecto WINDEV correspondiente a la aplicación actual en Reports & Queries

`<Result> = RunReportsAndQueries()`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el proyecto que corresponde a la aplicación se abrió en Reports & Queries,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.  





<a name="SYNTAX2"></a>

### Abrir un reporte en Informes y Consultas

`<Result> = RunReportsAndQueries(<Report to open>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se abrió el reporte especificado en Reports & Queries,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Report to open>: (Cadena de caracteres)`**

Nombre lógico de la reporte que se abrirá en Informes y Consultas. Este reporte debe ser modificable en Informes y Consultas ("Modificable en Reports and Queries" marcado en la pestaña "Detalles" del reporte description).

Cuando se llama a **RunReportsAndQueries** por primera vez, el archivo reporte (archivo ".WDE") se extrae automáticamente de la biblioteca de la aplicación y se copia en el directorio "Custom reports and queries" (definido al crear la aplicación WINDEV instalación Procedure o al instalar la aplicación WINDEV). Este archivo será utilizado por los usuarios finales.

En las siguientes llamadas a **RunReportsAndQueries**, se buscará el reporte que se va a abrir:

- primero, en el directorio de "Informes y consultas personalizadas".

- luego, en el directorio de "Informes y consultas compartidas".


Estos directorios han sido definidos en el instalación asistente de la aplicación. 

Para más detalles sobre Informes y Consultas, véase [Distribución de informes y consultas](../Presentation/3088002.md).  


<a name="SYNTAX3"></a>

### Crear una reporte o una consulta en Informes y Consultas

`<Result> = RunReportsAndQueries(<Type of element to create>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el asistente para crear un reporte o una consulta se ha iniciado en Reports & Queries,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Type of element to create>: (Constante de tipo Integer)`**

Tipo de elemento a crear:


|   |   |
| --- | --- |
| *CreateQuery* | Inicia el asistente para la creación de consultas |
| *CreateReport* | Inicia la creación de asistente para reporte |


## InfoAsynchronous (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Muestra un mensaje personalizado no bloqueante en una ventana de información del sistema.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=fonctions_dialogue%20-%20HC%20N%B0002.gif)


Observaciones: 

- **Esta ventana de información no es bloqueante**. El código WLanguage que sigue la llamada a la función **InfoAsynchronous** continúa ejecutándose. Para cerrar la ventana, el usuario debe hacer clic en el botón OK. Cuando el usuario hace clic en el botón "OK", se puede ejecutar el código de un procedimiento WLanguage.

- En el código WLanguage que sigue la llamada a la función, no se puede abrir otra ventana modal antes de cerrar el cuadro de diálogo. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) WINDEV permite aplicar la plantilla del proyecto al cuadro de diálogo. Para obtener más información, consulte [Personalizar cuadros de diálogo](../WDLang1/3021003.md).



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
User_Name is string
User_Name = "Florence"
InfoAsynchronous("Hello" + User_Name, "Welcome!", MyProcedure_Info)

INTERNAL PROCEDURE MyProcedure_Info()
	ToastDisplay("Close info")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`InfoAsynchronous(<Text> [, <WLanguage procedure>])`
---

**`<Text>: (Cadena de caracteres)`**

Texto a mostrar.

**`<WLanguage procedure>: (Nombre de procedimiento o procedimiento lambda opcional)`**

Nombre del procedimiento WLanguage ("callback") al que se llama cuando el usuario hace clic en "OK" en la ventana de información. 

Este procedimiento WLanguage puede corresponder a: 

- el nombre de un procedimiento global,

- el nombre de un procedimiento interno,

- un procedimiento lambda. 


Si no se especifica este parámetro, no se ejecuta ningún procedimiento cuando el usuario hace clic en el botón "OK" de la ventana de información.

<a name="NOTE0_1"></a>

## Observaciones


### Características del cuadro de diálogo
<a name="caracteristicas_del_cuadro_dialogo_ELTPARAGRAPHE000053"></a>



- El título del cuadro de diálogo corresponde al título de la ventana (o página) actual.
	

- Para modificar o definir el título del cuadro de diálogo, utilice la función [NextTitle](../WDLang1/3038043.md).




- El mensaje está alineado a la izquierda.

- El texto de los botones depende del idioma de ejecución de Windows.

- El número máximo de caracteres está limitado a 4096. Si se pasa una cadena más grande como parámetro, se truncará.
	

- El ícono que se muestra (tooltip de información) no se puede modificar.




![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Para personalizar este cuadro de diálogo** (y todos los cuadros de diálogo del sistema en la aplicación), seleccione la opción "Personalizar ventanas del sistema (Info, YesNo, Confirm, Dialog)" en la pestaña "Estilo" de la descripción del proyecto. Para obtener más información, consulte [Personalizar cuadros de diálogo](../WDLang1/3021003.md).
<a name="NOTE0_2"></a>


### Salto de línea en el cuadro de diálogo
<a name="salto_linea_cuadro_dialogo_ELTPARAGRAPHE000080"></a>La cadena predefinida CR (Carriage Return) se puede utilizar para pasar a la línea siguiente. Por ejemplo:


```wl
InfoAsynchronous("The XXX form was modified." + CR + "This form will be saved.")
```


También se puede usar la sintaxis de las cadenas multilínea. Por ejemplo: 


```wl
InfoAsynchronous(
[
The XXX form has been modified.

This form will be saved.
])
```

<a name="NOTE0_3"></a>


### Casos especiales
<a name="casos_especiales_ELTPARAGRAPHE000093"></a>

- Los timers (función [TimerSys](../WDLang1/3015006.md)) no se detienen al llamar a esta función.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Los eventos de Windows no se detienen al llamar a esta función.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Puede usar el asistente de código para introducir la función en el editor de código. Según las características introducidas en el asistente, la función utilizada puede ser diferente en el código que se genera automáticamente.

- El texto de los botones se muestra en el idioma del sistema.

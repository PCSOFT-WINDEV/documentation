
## AppDetectUPDAvailable( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Desactiva o personaliza la comprobación de actualizaciones disponibles en automatic. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
AppDetectUPDAvailable(UpdateAvaliable)
	INTERNAL PROCEDURE UpdateAvailable(Update, Version)
		IF Update = appUPDAvailMandatory THEN
			ToastDisplay("A mandatory update is available (" + Version + ").")
		END
	END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`AppDetectUPDAvailable(<WLanguage procedure>)`
---

**`<WLanguage procedure>: (Nombre de Procedure)`**

Nombre del WLanguage Procedure ("callback") llamado cuando se detecta una actualización de la aplicación en ejecución. Para más detalles sobre esta Procedure, véase [Parámetros del procedimiento utilizado por la función AppDetectUPDAvailable](../WDLang1/1410087706.md).

## Observaciones


### automatic comprueba las actualizaciones disponibles
<a name="automatic_comprueba_las_actualizaciones_disponibles_ELTPARAGRAPHE000037"></a>

La aplicación comprueba regularmente si hay una actualización de la red disponible. 
Recordatorio: la hora de comprobación puede ser set desde WDAdminExe o con [AppChangeParameter](../WDLang1/1000017260.md). 

Si hay una actualización disponible, se mostrará una Notification de Windows sólo si el usuario está utilizando la aplicación.

- **Para las actualizaciones opcionales**, el mensaje Notification es el siguiente: 
	Una actualización opcional para '&lt;nombre de la aplicación&gt;' está disponible. Para instalar esta actualización, cierre y reinicie la aplicación.

- **Para las actualizaciones obligatorias**, el mensaje Notification es el siguiente: 
	Existe una actualización obligatoria para '&lt;nombre de la aplicación&gt;'.. Para instalar esta actualización, cierre y reinicie la aplicación.


Estos mensajes no se pueden modificar ni configurar. 

La comprobación de las actualizaciones disponibles es siempre Active. 
**Observaciones**: 

- Si el usuario ha elegido no instalar una actualización opcional al iniciar la aplicación, esa versión no será propuesta por el mecanismo de Notification. 

- Las notificaciones de actualización obligatoria se muestran cada 5 minutos si la actualización no se ha instalado. 

- Las notificaciones opcionales de actualización no se vuelven a mostrar si la actualización no está instalada. Sólo aparecerá la próxima vez que se inicie la aplicación.






Gestión de las notificaciones: 

- Si la aplicación tiene un icono en el área de Notification, el icono se utilizará para resaltar el Notification. 

- Si la aplicación no tiene un icono en el área de Notification, se creará un icono para resaltar el Notification. Se utilizará un Icon diferente según si la actualización es opcional u obligatoria. 





### Personalización de la comprobación de actualizaciones de automatic con AppDetectUPDAvailable
<a name="personalizacion_comprobacion_actualizaciones_automatic_con_docparampagetitleshort_ELTPARAGRAPHE000079"></a>

El parámetro &lt;WLanguage Procedure&gt; permite personalizar la acción realizada cuando se detecta la actualización. Esta acción sustituye a la Notification de Windows. Esta Procedure puede, por ejemplo: 

- no realizar ninguna acción: en este caso, no se notificará al usuario que hay una actualización disponible. 

- realizar una acción específica (mostrar un mensaje explicativo como toast, por ejemplo). Esta acción sustituirá a la Notification de Windows.

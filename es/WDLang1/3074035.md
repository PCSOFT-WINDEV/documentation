


## TableSort (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ordena una tabla o control Tabla TreeView en una o más columnas (ordenación lexicográfica). 

**Observación**: Cuando se ordena un control con la función **TableSort**, las imágenes que permiten conocer la dirección del orden no se muestran en las columnas.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Sort "COL_Designation" and "COL_Price" columns in "TABLE_Product"
// "COL_Designation" is sorted in ascending order
// "COL_Price" is sorted in descending order
TableSort(TABLE_Product, "COL_Designation", "-COL_Price")
```
<a name="Example2"></a>

```wl
// Multicolumn sort of Table control
// "COL_Designation" is sorted in descending order
// "COL_Price" is sorted in ascending order 
TableSort("-COL_Designation" + TAB + "+COL_Price")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Clasificar una tabla o control Tabla TreeView en una o más columnas identificadas por su nombre

`<Result> = TableSort(<Table control> , <Column 1> [, <Column 2> [... [, <Column N>]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecutó el orden,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Table control>: (Nombre del control)`**

Nombre del control a manipular. Este control puede corresponder a: 

- un control Tabla.

- un control Tabla TreeView.


Si este parámetro corresponde a una cadena vacía (""), se utilizará el control al que pertenece el evento actual.

**`<Column 1>: (Cadena de caracteres)`**

Nombre de la primera columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:

```txt
"[<Direction>]<Column name>"
```

donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": de tipo descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.




**`<Column 2>: (Cadena de caracteres opcional)`**

Nombre de la segunda columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:

```txt
"[<Direction>]<Column name>"
```

donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": de tipo descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.






**`<Column N>: (Cadena de caracteres opcional)`**

Nombre de la enésima columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:

```txt
"[<Direction>]<Column name>"
```

donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": de tipo descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.





<a name="SYNTAX2"></a>

### Ordenar una tabla o control Tabla TreeView según una lista de nombres de columnas

`<Result> = TableSort(<List of columns>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecutó el orden,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<List of columns>: (Cadena de caracteres)`**

Lista de columnas a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:

```txt
"[<Direction>]<Column1>" + TAB + "[<Direction>]<Column2>" + TAB + ...
```

donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": de tipo descendente.




- &lt;Columna&gt; (cadena de caracteres): nombre de la columna a ordenar. El nombre de una columna es devuelto por [TableEnumColumn](../WDLang1/3074012.md). Si dos control tienen nombres de columna idénticos, el nombre de la columna debe utilizarse como prefijo en el nombre del control (por ejemplo: "+TABLE_Table1.COL_SortColumn"+TAB+"TABLE_Table1.COL_PriceColumn").  





<a name="SYNTAX3"></a>

### Clasificando una tabla o control Tabla TreeView en las primeras diez columnas de Visible de la control

`<Result> = TableSort(<Table control> [, <To sort>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó el orden en las primeras 10 columnas de control,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Table control>: (Nombre del control)`**

Nombre del control a manipular. Este control puede corresponder a: 

- un control Tabla.

- un control Tabla TreeView. 




Este parámetro tiene el siguiente formato:

```txt
"[<Direction>]<Name of Table control>"
```

donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": de tipo descendente.




- &lt;Nombre de control Tabla&gt; (cadena de caracteres): nombre de la Tabla o control Tabla TreeView cuyas primeras 10 columnas serán ordenadas.




**`<To sort>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> si se debe realizar el orden en las primeras 10 columnas del control,

- <u><u><u><u>False</u></u></u></u> si debe cancelarse el orden anterior. Una vez cancelada la ordenación, el orden de las filas no se modifica pero las filas que se añadan por [TableAdd](../WDLang1/3074017.md), [TableAddLine](../WDLang1/3074006.md), [TableInsert](../WDLang1/3074025.md) o [TableInsertLine](../WDLang1/3074026.md) no se ordenarán.  





<a name="SYNTAX4"></a>

### Cancelación de una ordenación previamente definida en una Tabla o control Tabla TreeView poblada programáticamente

`<Result> = TableSort(<Table control> , <False>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se canceló el orden,

- <u><u><u><u>False</u></u></u></u> si se produce un problema.




**`<Table control>: (Nombre del control)`**

Nombre del control a manipular. Este control puede corresponder a: 

- un control Tabla rellenado mediante programación.

- a control Tabla TreeView poblada programáticamente. 




**`<False>: (Constante)`**

*False*: Palabra clave utilizada para eliminar una ordenación previamente definida en una Tabla o control Tabla TreeView poblada programáticamente.

<a name="NOTE0_1"></a>

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000258"></a>

La función **TableSort** puede utilizarse en:

- una Tabla o control Tabla TreeView rellenada programáticamente.

- una Tabla o control Tabla TreeView basada en un archivo de datos con iteración automatic (opción "Examinar automatic aliado" en la pestaña "Contenido" del control description). En este caso, sólo se ordena una columna.

- un control de selección única o de selección múltiple.
	



<a name="NOTE0_2"></a>


### Ordenar una tabla o control Tabla TreeView en función de un archivo de datos
<a name="ordenar_una_tabla_control_tabla_treeview_funcion_archivo_datos_ELTPARAGRAPHE000294"></a>

Una Tabla o control Tabla TreeView basada en un archivo de datos se ordenará según la primera columna especificada (sintaxis 1 y 2). Atención: la columna utilizada para la clasificación debe ser relacionado a un elemento clave en el archivo de datos utilizado para navegar por el control.

**Consejo**: Para ordenar una Tabla o control Tabla TreeView en base a un archivo de datos con varias columnas, utilice una clave compuesta y el [BrowsedItem](../Proprietes/2510111.md) Property:


```txt
TableName..BrowsedItem = NAMECOMPOSITEKEY
```
Para cancelar una ordenación en una Tabla o control Tabla TreeView basada en un archivo de datos, basta con asignar una cadena vacía al [BrowsedItem](../Proprietes/2510111.md) Property.
<a name="NOTE0_3"></a>


### Gestión de una tabla ordenada o control Tabla TreeView
<a name="gestion_una_tabla_ordenada_control_tabla_treeview_ELTPARAGRAPHE000314"></a>

- Cuando se ordena el control con la función **TableSort**, las operaciones realizadas mediante las funciones [TableAdd](../WDLang1/3074017.md), [TableAddLine](../WDLang1/3074006.md), [TableInsert](../WDLang1/3074025.md) y [TableInsertLine](../WDLang1/3074026.md) tienen en cuenta este orden.

- Cuando la control se ordena mediante un clic del ratón, las sumas realizadas por [TableAdd](../WDLang1/3074017.md), [TableAddLine](../WDLang1/3074006.md), [TableInsert](../WDLang1/3074025.md) y [TableInsertLine](../WDLang1/3074026.md) ignoran la ordenación.

- ***La clasificación no se cancela*** por [TableDisplay](../WDLang1/3074003.md), [TableAdjust](../WDLang1/3074004.md), [TableSearch](../WDLang1/3074015.md), [TableCount](../WDLang1/3074021.md), [TablePosition](../WDLang1/3074022.md), [TableSelect](../WDLang1/3074020.md), [TableSelectMinus](../WDLang1/3074029.md), [TableSelectPlus](../WDLang1/3074031.md), [TableDelete](../WDLang1/3074032.md) y [TableDeleteAll](../WDLang1/3074034.md).

- ***La ordenación se cancela*** si se modifica una columna (por entrada directa en la columna o por [TableModify](../WDLang1/3074019.md) y [TableModifyLine](../WDLang1/3074028.md)) y si se ordena.




Recordatorio: [TableSortedColumn](../WDLang1/3074010.md) devuelve la lista de columnas ordenadas.
<a name="NOTE0_4"></a>


### Clasificar en una columna calculada
<a name="clasificar_una_columna_calculada_ELTPARAGRAPHE000393"></a>

No se recomienda utilizar **TableSort** en columnas calculadas. Estas columnas se calculan cuando se muestra cada fila.

Una ordenación definida con **TableSort** en una columna calculada sólo será válida para las filas ya visualizadas.
<a name="NOTE0_5"></a>


### Recorrer mediante un bucle programado una tabla o una vista de árbol controles Tabla
<a name="recorrer_mediante_bucle_programado_una_tabla_una_vista_arbol_controles_tabla_ELTPARAGRAPHE000408"></a>

**TableSort** no tiene ningún efecto sobre los controles que se repiten mediante programación.
<a name="NOTE0_6"></a>


### Realizar una clasificación según el valor ASCII
<a name="realizar_una_clasificacion_segun_valor_ascii_ELTPARAGRAPHE000419"></a>

Una zona de memoria permite realizar una ordenación según el valor ASCII

Ejemplo:

```wl
MemCreate("MZ")
MemAdd("ZM", "abc", "abc")
MemAdd("ZM", "ZZZ", "ZZZ")
MemAdd("ZM", "zbc", "zbc")
MemAdd("MZ", "ééébc", "ééébc")
MemAdd("ZM", "AB", "AB")
MemAdd("ZM", "CD", "CD")
MemAdd("MZ", "çabc", "çabc")
MemAdd("ZM", "bbc", "bbc")
MemSort("MZ")
i is int
MemFirst("MZ")
WHILE NOT MemOut("MZ")
	i = MemCurrent("ZM")
	Trace(MemRetrieve("MZ", i)) // Display in ASCII order
	MemNext("MZ")
END
```

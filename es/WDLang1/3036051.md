
## fRemoveDir( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina un directorio de un disco.

**Observaciones**:

- El usuario debe tener los derechos para eliminar un directorio.

- No se puede borrar un directorio que esté siendo consultado por [fDir](../WDLang1/3036056.md) o [fListFile](../WDLang1/3036058.md).

## Sintaxis

`<Result> = fRemoveDir(<Directory to delete> [, <Deletion indicator>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se eliminó el directorio,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Directory to delete>: (Cadena de caracteres)`**

Nombre y ruta completa (o relativa) del directorio a eliminar. Se puede utilizar una ruta UNC. Los caracteres comodín (\*,?) se ignoran. Este nombre de directorio puede terminar con "\\" o no.

![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Este parámetro puede estar en formato Ansi o Unicode.





**`<Deletion indicator>: (Constante opcional)`**






|   |   |
| --- | --- |
| *frReadOnly* | También elimina los archivos de sólo lectura. Si no se especifica esta constante, la función devolverá <u><u><u><u>False</u></u></u></u> si se deben eliminar los archivos de solo lectura.<br><br><br> |
| *frRecursive* | El contenido del directorio y los subdirectorios también se eliminan. Si no se especifica esta constante, el directorio debe estar vacío para poder ser eliminado. |
| *frToRecycleBin* | Envía el directorio a la papelera de Windows en lugar de eliminarlo..<br><br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta constante no está disponible. |


&lt;Indicador de borrado&gt; está vacío por Default. No se selecciona ninguna de estas opciones.

### Modo de funcionamiento en Windows Vista (y posterior)
<a name="modo_funcionamiento_windows_vista_posterior_ELTPARAGRAPHE000263"></a>

Si esta función no funciona correctamente en Windows Vista (y posterior), compruebe si el archivo o directorio utilizado no está en ninguno de los directorios del sistema (directorio de Windows o directorio "Archivos de programa").

**En Windows Vista (y posteriores), con el [Mecanismo UAC](../Editeurs/2025014.md) (Cuenta de usuario control) habilitado**, debe tener privilegios de administrador para manipular y/o modificar los archivos o directorios en los directorios del sistema (directorio de Windows o directorio "Archivos de programa").

**Consejo de programación**: A manipular y/o modificar los archivos o directorios sin privilegios de administrador, debería:

- evite escribir en el directorio de Windows o en el de "Archivos de programa",

- utilizar el directorio del sistema de la aplicación (devuelto por [SysDir](../WDLang1/3073026.md) con la constante *srAppDataCommun*, por ejemplo).

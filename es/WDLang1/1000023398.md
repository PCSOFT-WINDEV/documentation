


## TestCheckEqual( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Verifica si las expresiones son iguales en la programación asertiva y en el modo de prueba automática.

Observación: Esta función es equivalente a la función [dbgCheckEqual](../WDLang1/1000019776.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// The function should return 0
TestCheckEqual(CalcFunction(), 0)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`TestCheckEqual(<Obtained expression> , <Expected expression> [, <Comments> [, <Parameter 1> [... [, <Parameter N>]]]])`
---

**`<Obtained expression>: (Cadena de caracteres)`**

Expresión correspondiente al resultado obtenido. 

**`<Expected expression>: (Cadena de caracteres)`**

Expresión correspondiente al resultado esperado.

**`<Comments>: (Cadena de caracteres)`**

Cadena de caracteres en el formato definido por [StringBuild](../WDLang1/3024006.md). 

**`<Parameter 1>: (Cualquier tipo)`**

Primer parámetro esperado por &lt;Comentarios&gt;.



**`<Parameter N>: (Cualquier tipo)`**

El enésimo parámetro esperado por &lt;Comentarios&gt;.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Cuando las expresiones no son iguales:

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en modo de prueba automatizado, se añade un error al estado de prueba reporte (equivalente a [Resultado de TestWriteResult](../WDLang1/1000013001.md)).

	- en modo de prueba manual, se muestra una afirmación (equivalente a [dbgAssert](../WDLang1/3014022.md)).

	- en modo de ejecución, se muestra una aserción si se han habilitado las aserciones (equivalente a [dbgAssert](../WDLang1/3014022.md)).




- **TestCheckEqual** se utiliza para comparar el contenido de un array pasado como primer parámetro con un array de valores (sintaxis [ 1, 2, 3]) pasado como segundo parámetro.

- Cuando se ejecuta una aplicación, la expresión no se evalúa si las aserciones no están habilitadas.

- [dbgOnError](../WDLang1/1000020722.md) permite iniciar automáticamente un procedimiento WLanguage utilizado para recuperar el comentario relacionado al error.

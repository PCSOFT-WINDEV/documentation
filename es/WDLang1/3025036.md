
## ControlError( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Personaliza el valor que se muestra en los controles cuando el valor no se puede mostrar.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Define the ceOverflow error
ControlError(ceOverflow, "The control mask is too small")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`ControlError(<Type of error> , <Error message>)`
---

**`<Type of error>: (Constante)`**

Tipo de error:


|   |   |   |
| --- | --- | --- |
| *ceBrokenBinding* | No se encuentra el valor del elemento relacionado (el valor relacionado no existe o se ha producido un error durante su lectura). | Valor mostrado por Default (corresponde a 3 caracteres nítidos): "###" |
| *ceNull* | El elemento relacionado es un elemento HFSQL que contiene NULL. | Valor mostrado por Default (cadena vacía): "" |
| *ceOverflow* | El valor de control es mayor que la máscara de entrada/visualización definida para el control. | Valor mostrado por Default (corresponde a 5 más caracteres): "+++++" |
| *ceReadField* | Error al leer un elemento: el artículo no existe, el archivo de datos está vacía, ... | Valor mostrado por Default (corresponde a 4 caracteres nítidos): "####" |



**`<Error message>: (Cadena de caracteres)`**

Nuevo mensaje de error associated con el tipo de error.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Modificar los valores visualizados
<a name="modificar_los_valores_visualizados_ELTPARAGRAPHE000080"></a>

Los nuevos valores sólo serán efectivos para futuros errores. Los valores asignados a los controles antes de utilizar **ControlError** no cambian.

La modificación es global con respecto al programa actual.

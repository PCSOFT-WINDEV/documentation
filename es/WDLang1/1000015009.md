


## &lt;Xaml&gt;.Event (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Asocia una Procedure escrita en WLanguage con una Event de Xaml control.

Después de ejecutar la función **&lt;Xaml&gt;.Event**, el procedimiento se iniciará cada vez que se active el evento seleccionado.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// - Initialization of the Xaml control DOTNET_Xaml_OK
MyString is string = [
<Button Name="Button1" Width="20">OK</Button>
]
DOTNET_Xaml_OK = MyString
DOTNET_Xaml_OK.Event("Button1", "Click", MyOnClickProcedure)
```

```wl
// -- Procedure MyOnClickProcedure
PROCEDURE MyOnClickProcedure
Info("The OK button was clicked")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Control «xaml»>.Evento([<Name of .Net control>, ] <Event name> , <Procedure>)`
---

**`<Control «xaml»>: (Control name)`**

Nombre de Xaml control.

**`<Name of .Net control>: (Optional character string (with quotes))`**

Nombre del control.net que se asociará a &lt;Procedure&gt;.. Si no se especifica este parámetro, &lt;Procedure&gt; será associated con el Xaml control solamente.

Este nombre distingue entre mayúsculas y minúsculas.

**`<Event name>: (Character string)`**

Nombre de la Event asociada al control Xaml y/o al control .Net que activa la Procedure.

**`<Procedure>: (Character string (with or without quotes))`**

Nombre del procedimiento WLanguage llamado cuando se activa el Event. 
Este procedimiento debe contener el número de parámetros suministrados por la Event y puede eventualmente devolver un resultado (ver la documentación de la .Net control).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Se recomienda llamar a **&lt;Xaml&gt;.Event** en el código de inicialización Xaml control.

- **Importante**: PC SOFT no ofrece soporte técnico sobre el modo de funcionamiento de los controles de red..
	Encontrará los diferentes eventos soportados por los controles de .net en su propia documentación.




## &lt;Variable Queue&gt;.Dequeue (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Elimina un elemento de la cola. El elemento se retira de la cola y se devuelve en la Variable especificada.  




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyQueue is queue of int

// Enqueues the values 1, 2 and 3
MyQueue.Enqueue(1)
MyQueue.Enqueue(2)
MyQueue.Enqueue(3)

// Display the trace: 1, 2, 3
x is int
WHILE MyQueue.Dequeue(x)
	Trace(x)
END
```

<a name="SYNTAX1"></a>

## Sintaxis

### Recuperar el elemento al principio de la cola

`<Result> = <Queue>.Dequeue([<Element>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el elemento fue eliminado de la cola,

- <u><u><u><u>False</u></u></u></u> si la cola está vacía.




**`<Queue>: (Cola)`**

Nombre de la variable de tipo [Cola](../WDLang1/1514021.md) que se utilizará.

**`<Element>: (Tipo de elementos manejados por la cola, parámetro opcional)`**

Nombre de la Variable en la que se devuelve el valor del elemento eliminado. Si no se especifica este parámetro, el elemento se elimina pero no se recupera su valor.


<a name="SYNTAX2"></a>

### Recuperar el elemento al principio de la cola (sintaxis multihilo)

`<Result> = <Queue>.Dequeue([<Element> [, <Options> [, <Wait>]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el elemento fue eliminado de la cola,

- <u><u><u><u>False</u></u></u></u> si se ha agotado el tiempo de espera (parámetro &lt;Espera&gt;) o si la propiedad [AdditionCompleted](../Proprietes/1000018388.md) pasa a <u><u><u><u>True</u></u></u></u>.




**`<Queue>: (Cola)`**

Nombre de la variable de tipo [Cola](../WDLang1/1514021.md) que se utilizará.

**`<Element>: (Tipo de elementos manejados por la cola, parámetro opcional)`**

Nombre de la Variable en la que se devuelve el valor del elemento eliminado. Si no se especifica este parámetro, el elemento se elimina pero no se recupera su valor.

**`<Options>: (Constante opcional de tipo Integer)`**

Opciones de cola:



|   |   |
| --- | --- |
| *WaitIfEmpty* | Espera a que un elemento sea añadido a la cola por otra thread. |



**`<Wait>: (Entero o duración opcional)`**

Tiempo máximo de espera expresado en centésimas de segundo. Este parámetro puede corresponder a: 

- un número entero que corresponde al número de centésimas de segundo,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, 1 s o 10 ms). 


Este parámetro corresponde a la constante *Infinite* de Default (no hay tiempo máximo de espera).

## Observaciones


### Modo de funcionamiento para una gestión multihilo
<a name="modo_funcionamiento_para_una_gestion_multihilo_ELTPARAGRAPHE000272"></a>

Si un elemento está en la cola, se elimina de la cola y es devuelto por el parámetro &lt;Elemento&gt; por reference.

Si la cola está vacía, la función **&lt;Variable Queue&gt;.Dequeue** espera a que se añada un elemento a la cola. El tiempo máximo de espera puede ser especificado por el parámetro &lt;Tiempo de espera&gt;.

La función devuelve <u><u><u><u>True</u></u></u></u> si se ha eliminado un elemento de la cola. En caso contrario, devuelve <u><u><u><u>False</u></u></u></u> si se ha alcanzado el tiempo de espera o si el [AdditionCompleted](../Proprietes/1000018388.md) Property es set a <u><u><u><u>True</u></u></u></u>.

[NbPendingThread](../Proprietes/1000018387.md) devuelve el número de hilos pendientes.

La asignación de [AdditionCompleted](../Proprietes/1000018388.md) a <u><u><u><u>True</u></u></u></u> provoca el fin del tiempo de espera para todos los hilos pendientes. En este caso, la función **&lt;Variable Queue&gt;.Dequeue** devuelve <u><u><u><u>False</u></u></u></u>.




## ExeListProcess( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve la lista de aplicaciones que se están ejecutando actualmente.. Estas aplicaciones pueden corresponder a aplicaciones WINDEV o WINDEV Mobile, o a otras aplicaciones.

Por cada aplicación encontrada, usted tiene la posibilidad de averiguarlo:

- su identificador.

- el identificador del padre Process (el que inició la aplicación). 

- el nombre y la extensión de su ejecutable.

- el nombre y la ruta completa de su ejecutable.

- la cantidad actual de memoria utilizada.

- la cantidad máxima de memoria utilizada.

- el comando Line usado por el ejecutable.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Add the applications currently run into the "LIST_App" control
ListAdd(LIST_App, ExeListProcess(exePID, exeShortName))
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = ExeListProcess([<First information> [... [, <Seventh information>]]])`
---

**`<Result>: (Cadena de caracteres)`**

Lista de aplicaciones que se están ejecutando actualmente. Las diferentes aplicaciones están separadas por caracteres CR (Carriage Return).
La lista tiene el siguiente formato:

```txt
<1st information> + TAB + <2nd information> + CR +...
```

Si no se solicita información, la lista tiene el siguiente formato: &lt;Nombre y extensión del ejecutable de la aplicación>.

**`<First information>: (Constante opcional de tipo Integer)`**

Primera información que se devolverá para cada aplicación actualmente en ejecución:


|   |   |
| --- | --- |
| *exeCommandLine* | Comando Line de aplicación ejecutable. <br><br>**Atención**: <br><br>- El usuario actual debe tener suficientes derechos para leer el contenido de la memoria relacionado al Process. <br><br>- Las aplicaciones de 32 bits no pueden acceder a la información de los procesos de 64 bits.<br><br><br> |
| *exeMemory* | Cantidad de memoria consumida actualmente por la aplicación.<br>Esta constante está disponible para Windows NT, 2000 y XP. En los demás casos, se devuelve una cadena vacía ("""). |
| *exeMaxMemory* | Cantidad máxima de memoria consumida por la aplicación.<br>Esta constante está disponible para Windows NT, 2000 y XP. En los demás casos, se devuelve una cadena vacía ("""). |
| *exeShortName*<br>(Valor predeterminado) | Nombre y extensión del ejecutable de la aplicación. |
| *exeLongName* | Nombre y apellidos y ruta completa del ejecutable de la aplicación.<br> |
| *exePID* | Identificador de la aplicación. |
| *exeParentPID* | Identificador del padre Process (el que inició la aplicación). |





**`<Seventh information>: (Constante opcional de tipo Integer)`**

Séptima información que será devuelta para cada aplicación actualmente en ejecución:


|   |   |
| --- | --- |
| *exeCommandLine* | Comando Line de aplicación ejecutable. <br><br>**Atención**: <br><br>- El usuario actual debe tener suficientes derechos para leer el contenido de la memoria relacionado al Process. <br><br>- Las aplicaciones de 32 bits no pueden acceder a la información de los procesos de 64 bits.<br><br><br> |
| *exeMemory* | Cantidad de memoria consumida actualmente por la aplicación.<br>Esta constante está disponible para Windows NT, 2000 y XP. En los demás casos, se devuelve una cadena vacía ("""). |
| *exeMaxMemory* | Cantidad máxima de memoria consumida por la aplicación.<br>Esta constante está disponible para Windows NT, 2000 y XP. En los demás casos, se devuelve una cadena vacía ("""). |
| *exeShortName*<br>(Valor predeterminado) | Nombre y extensión del ejecutable de la aplicación. |
| *exeLongName* | Nombre y apellidos y ruta completa del ejecutable de la aplicación.<br> |
| *exePID* | Identificador de la aplicación. |
| *exeParentPID* | Identificador del padre Process (el que inició la aplicación). |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
La función **ExeListProcess** no devuelve ningún código de error. Para determinar si esta función ha generado un error, utilice la función [ErrorOccurred](../WDLang1/3087001.md) Variable. Si se produce un error, puede get más detalles sobre el error con [ErrorInfo](../WDLang1/3013008.md).

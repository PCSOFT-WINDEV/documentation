


## CoordinateScreenToEditor( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Convierte un tamaño o una posición especificada en el dpi nativo de la pantalla del dispositivo donde se ejecuta la aplicación al dpi de reference en el editor (160 dpi). 

Esta función se utiliza principalmente para realizar un posicionamiento a nivel de píxel.

![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) Esta función solo está disponible para simplificar la Webificación de proyectos WINDEV.  
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Retrieves the size and position of a control as they have been defined in the editor
nX is int = CoordinateScreenToEditor(MyInputControl.X)
nY is int = CoordinateScreenToEditor(MyInputControl.Y)
nWidth is int = CoordinateScreenToEditor(MyInputControl.Width)
nHeight is int = CoordinateScreenToEditor(MyInputControl.Height)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = CoordinateScreenToEditor(<Value>)`
---

**`<Result>: (Integro)`**

Resultado de la conversión: tamaño o posición convertido en el dpi de reference en el editor (160 dpi). 


**`<Value>: (Integro)`**

Valor a convertir: tamaño o posición especificados en el dpi nativo de la pantalla del dispositivo. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- ![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Al ejecutar la aplicación, estos tamaños y posiciones se convierten automáticamente de acuerdo con el dpi nativo de la pantalla en el dispositivo host.. Esto te permite get un renderizado WYSIWYG independientemente del dpi del dispositivo. Por ejemplo, si el tamaño de un control es igual a 100x100 en el editor, su tamaño en tiempo de ejecución será set a 150x150 en un dispositivo en 240 dpi, a 200x200 en un dispositivo en 320 dpi, etc.

- [Editor de coordenadas a pantalla](../WDLang1/1000021277.md) permite hacer la conversión invertida, es decir, de los ppp de reference en el editor (160 ppp) a los ppp nativos del dispositivo en el que se ejecuta la aplicación.


## HTMLEnableDebugger( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Activa el depurador HTML/JavaScript en los controles HTML basados en CEF: Pantalla HTML o editor HTML control. 

Para depurar los controles, basta con: 

- Abra Chrome y escriba lo siguiente Address: 
	
	```txt
	http://localhost:8088/
	```


- Haga clic en uno de los enlaces que aparecen en Chrome (1 enlace por control). 



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Project initialization code
IF LoadParameter("Debug") = "1" THEN
      HTMLEnableDebugger(8081)
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`HTMLEnableDebugger(<Port number>)`
---

**`<Port number>: (Integro)`**

Número de puerto utilizado para la depuración. Por Default, este número es el 8088. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Esta función debe llamarse antes de crear cualquier control HTML (Display HTML o Editor HTML control), por ejemplo, en la inicialización del proyecto. 

- El depurador HTML/JavaScript (Devtools) puede visualizarse con la siguiente URL: 
	
	```txt
	http://machine_name:port_num/
	```
Por ejemplo: 
	
	```txt
	http://localhost:8088/
	```


- El depurador es habilitado por Default cuando se lanza una prueba a través de GO, pero no está habilitado durante la ejecución de la aplicación. Puede utilizarse para rastrear el código JavaScript añadido con [ExécuteJS](../WDLang1/1000023014.md), por ejemplo.

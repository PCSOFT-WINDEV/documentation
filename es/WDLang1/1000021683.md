


## TraceBuild( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Muestra un mensaje personalizable en la ventana de salida abierta en paralelo a la ventana actual.

**Observación**: Al igual que [Traza](../WDLang1/3013050.md), **TraceBuild** puede utilizarse en la fase de desarrollo para comprobar que el programa se ejecuta correctamente. En el editor, el [Panel "Rastreo del depurador"](../Editeurs/2027028.md) muestra la información encontrada en el ventana de salida así como el texto completo de los errores que se produjeron.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Display an error message in the trace window
TraceBuild("The customer code %1 does not exist.", nCustomerCode)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`TraceBuild(<Initial string> [, <Parameter 1> [... [, <Parameter N>]]])`
---

**`<Initial string>: (Cadena de caracteres)`**

Cadena de caracteres a mostrar. Esta cadena contiene parámetros opcionales (%1 a %n). Para mostrar el carácter %, este carácter debe escribirse dos veces (%%%).

**`<Parameter 1>: (Cadena de caracteres opcional)`**

Primer parámetro que se insertará en la cadena de caracteres.
&lt;Parámetro 1&gt; reemplazará al carácter %1, &lt;Parámetro 2&gt; reemplazará al carácter %2, etc.
El número de parámetros especificados debe ser igual o mayor que el número de parámetros utilizados en &lt;Cadena inicial&gt;.
Si &lt;Cadena inicial&gt; contiene más parámetros, los parámetros adicionales serán reemplazados por cadenas vacías.



**`<Parameter N>: (Cadena de caracteres opcional)`**

Enésimo parámetro que se insertará en la cadena de caracteres.
&lt;Parámetro 1&gt; reemplazará al carácter %1, &lt;Parámetro 2&gt; reemplazará al carácter %2, etc.
El número de parámetros especificados debe ser igual o mayor que el número de parámetros utilizados en &lt;Cadena inicial&gt;.
Si &lt;Cadena inicial&gt; contiene más parámetros, los parámetros adicionales serán reemplazados por cadenas vacías.

<a name="NOTE0_1"></a>

## Observaciones


### Uso de TraceBuild en las aplicaciones multilingües
<a name="uso_tracebuild_las_aplicaciones_multilingues_ELTPARAGRAPHE000073"></a>

En una aplicación multilingüe, se puede utilizar **TraceBuild** para simplificar la traducción Process: 

- el traductor traduce el mensaje utilizando los caracteres "%1", "%2"., ... en la ubicación relevante según el idioma. 

- los caracteres opcionales pueden ser reemplazados en todos los idiomas soportados.

### Ventana de salida
<a name="ventana_salida_ELTPARAGRAPHE000123"></a>

- La ventana para de salida se abre automáticamente desde la primera vez que se llama a la función **TraceBuild** (la función [TraceStart](../WDLang1/3013052.md) se ejecuta automáticamente). De forma predeterminada, esta ventana se abre en la esquina superior izquierda de la pantalla.

- El modo de funcionamiento de la función **TraceBuild** es similar al de la función [InfoBuild](../WDLang1/1000018879.md) pero la visualización se realiza en una ventana paralela, sin esperar la validación del usuario.



<a name="NOTE0_4"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Menú contextual de la ventana de salida
<a name="menu_contextual_ventana_salida_ELTPARAGRAPHE000146"></a>

El menú contextual de la ventana **Trace** permite:

- copiar, cortar o pegar el contenido de la ventana de salida.

- guardar el contenido de la ventana de salida en un archivo de texto. 
	Atención: Esta opción solo está disponible si la biblioteca wdxxxstd.dll se encuentra en el equipo del usuario final.

- borrar el contenido de la ventana de salida.

- abrir la ventana de salida sobre las demás ventanas (incluidas las de otras aplicaciones).

- guardar el tamaño y la posición de la ventana de salida al cerrarla. Cuando esta ventana vuelva a aparecer, mantendrá el tamaño y la posición guardados.

- mostrar una barra de desplazamiento horizontal.

- mostrar los caracteres especiales en gris o utilizarlos directamente: EOT, TAB, etc. Por lo tanto, los caracteres "Carriage Return" (CR) encontrados corresponden a "&lt;\\r&gt;&lt; &gt;"..
	Observación: Esta opción se tendrá en cuenta a partir de la siguiente línea de información que se muestra.


## EmulateConfirm( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Emula la siguiente llamada a [Confirme](../WDLang1/3021007.md) y pulsa automáticamente el botón pasado como parámetro.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Automatically close the next Confirm dialog box
// The answer on which the click will be performed is Yes
EmulateConfirm(Yes)

// Process 
// ...

SWITCH Confirm("Do you want to save the modifications?")
	CASE Yes
	// Save the modifications
	CASE Cancel
	// Cancel the modifications
END
// The message is automatically closed via a click performed on Yes
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`EmulateConfirm(<Button>)`
---

**`<Button>: (Constante de tipo Integer)`**

Indica el botón que se pulsará en la ventana de confirmación:


|   |   |
| --- | --- |
| *Cancel* | Haga clic en [ CANCEL] |
| *No* | Haga clic en [ NO] |
| *YES* | Haga clic en [ YES] |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Comprobación de la función
<a name="comprobacion_funcion_ELTPARAGRAPHE000063"></a>

La existencia de una función de emulación ([EmulateWarning](../WDLang1/1000009001.md), [EmulateError](../WDLang1/1000009005.md), [EmulateInfo](../WDLang1/1000009006.md), ...) se comprueba durante la llamada a la función de diálogo asociada ([Advertencia](../WDLang1/3021009.md), [Error](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ...).
<a name="NOTE0_2"></a>




### Diferencia de clase de mensaje
<a name="diferencia_clase_mensaje_ELTPARAGRAPHE000089"></a>

El orden de las funciones [EmularXXX](../WDLang1/1000009002.md) debe respetar el orden de visualización de [Error](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ... De lo contrario, se producirá un error fatal.

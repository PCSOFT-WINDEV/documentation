


## InWebMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica si se el proceso actual se inicia:

- desde un ejecutable WINDEV.

- desde el motor de ejecución WEBDEV (AWP).




**Observación**: Para los códigos compartidos entre una aplicación WINDEV y un sitio WEBDEV (código de una clase, componente, etc.), esta función permite identificar el modo de ejecución e iniciar el proceso apropiado.

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta función no tiene ningún efecto en esta versión.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Start mode?
IF InWebMode() = True THEN
	// Process run from the runtime engine of WEBDEV: 
	// Call to the "WebSendPage" procedure
	WebSendPage("Info.htm")
ELSE
	// Process run from a WINDEV executable 
	// Display a dialog box
	Info("Action performed")
END
```

<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = InWebMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecuta el código con el motor de ejecución de WEBDEV,

- <u><u><u><u>False</u></u></u></u> en caso contrario.


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Este parámetro siempre corresponde a <u><u><u><u>False</u></u></u></u>.

<a name="NOTE0_1"></a>

## Observaciones


### Compartir componentes y clases entre WINDEV y WEBDEV
<a name="compartir_componentes_clases_entre_windev_webdev_ELTPARAGRAPHE000111"></a>

La función **InWebMode** permite compartir fácilmente clases o componentes desarrollados en WLanguage entre un sitio WEBDEV y una aplicación WINDEV. Dependiendo de si la aplicación se ejecuta en modo Web o no, se pueden utilizar diferentes funciones específicas de la aplicación actual.
<a name="NOTE0_2"></a>


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000252"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.
<a name="NOTE0_3"></a>


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000264"></a>

La función **InWebMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InWebMode** pueden compartirse entre varios productos sin generar errores de compilación.

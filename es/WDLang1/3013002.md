


## ProfilerEnd( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Deja de "recoger datos" para la [analizador de rendimiento](../Editeurs/2030035.md).

Se guarda el archivo WPF que contiene el análisis de todos los procesos realizados.. Este archivo puede ser abierto por el [analizador de rendimiento](../Editeurs/2030035.md) o en el editor, a través del ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_ouvrir.gif) en los botones de acceso rápido.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
ProfilerStart("Test_StartProcedure.wpf")
StartProcedure()
ProfilerEnd()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`ProfilerEnd()`
---



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Si no se llama a la función [ProfilerStart](../WDLang1/3013012.md) antes de la función **ProfilerEnd**, la función **ProfilerEnd** no tiene ningún efecto.

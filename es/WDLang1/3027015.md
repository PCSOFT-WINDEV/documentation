
## DateTimeByDefault (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite definir el valor predeterminado de variables de tipo Date, Time y DateTime.

**Recordatorio**: De forma predeterminada, las variables de tipo [Date](../Motscles/1514066.md), [Time](../Motscles/1514068.md) y [DateTime](../Motscles/1514070.md) se inicializan con la fecha y hora del sistema al ejecutar el programa.






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Initialize all the Date, Time,
// and DateTime variables to January 1st 2020 at 0:00
DateTimeByDefault("20200101", "0000")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Inicialización con la fecha y hora del sistema

`DateTimeByDefault()`
---


<a name="SYNTAX2"></a>

### Inicialización con una hora y fecha específicas

`DateTimeByDefault(<Date> , <Time>)`
---

**`<Date>: (Cadena de caracteres)`**

Fecha utilizada para inicializar variables de tipo [Date](../Motscles/1514066.md) y [DateTime](../Motscles/1514070.md). Esta fecha está en formato "AAAAMMDD" donde AAAA corresponde al año, MM al mes y DD al día.

**`<Time>: (Cadena de caracteres)`**

Hora utilizada para inicializar variables de tipo [DateTime](../Motscles/1514070.md) y [Time](../Motscles/1514068.md). Esta hora está en formato HHMMSSCC (HH corresponde a las horas, MM a los minutos, SS a los segundos y CC a las centésimas de segundo).

### Casos especiales
<a name="casos_especiales_ELTPARAGRAPHE000114"></a>

No se debe llamar a la función **DateTimeByDefault** en el mismo proceso (o evento) en el que se declaran las variables de tiempo (date, time, datetime). 
De lo contrario, la modificación de la fecha y hora predeterminada no se tendrá en cuenta en las variables declaradas en el mismo proceso (o evento).

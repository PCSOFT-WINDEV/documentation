
## TestWriteResult( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Escribe información o un error en el status reporte de una prueba unitaria, en el editor de pruebas.

**Observación**: Esta función sólo está disponible para las pruebas unitarias. En tiempo de ejecución, esta información o estos errores se ignoran para evitar interferir con el funcionamiento de la aplicación o del sitio.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Add an information into the status report of a test
TestWriteResult(twrInfo, "Beginning of test")
```


<a name="Example2"></a>


![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Check the file existence
IF fFileExist("C:\My Documents\MyFile.txt") = False THEN
	// If the file does not exist, an error is triggered in the unit test
	// and "The file does not exist" is displayed in the status report of the test
	TestWriteResult(twrError, "The file does not exist.")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`TestWriteResult(<Type of information> , <Text to Write>)`
---

**`<Type of information>: (Constante de tipo Integer)`**

Tipo de información a escribir:


|   |   |
| --- | --- |
| *twrError* | Se utiliza para escribir un error en el status reporte de la prueba.. La prueba continúa hasta que falla (por ejemplo, error WLanguage)..<br>Si se ha escrito un error, la prueba se muestra como fallida en el administrador de pruebas. |
| *twrInfo* | Se escribe una información en el status reporte de la prueba. |


El tipo de información a escribir puede combinarse con la constante *twrCallingProcess*. 


|   |   |
| --- | --- |
| *twrCallingProcess* | Se utiliza para escribir un cheque personalizado Procedure. <br>Al llamar a la función **TestWriteResult** desde el procedimiento personalizado, sus resultados se referenciarán en la llamada al procedimiento personalizado y no en la llamada a la función **TestWriteResult**). <br>Por lo tanto, un doble clic realizado en los resultados de la prueba abrirá el editor de código en la llamada a la comprobación personalizada Procedure. |



**`<Text to Write>: (Cadena de caracteres)`**

Información para escribir en el estado reporte de la prueba unitaria.

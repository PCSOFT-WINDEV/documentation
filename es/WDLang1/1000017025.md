


## TableSortChild (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Clasifica una rama de una control Tabla TreeView en una o más columnas..

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Sort the "Designation" and "Price" columns in the "TVT_Product" control
// The "Designation" column is sorted in ascending order
// The "Price" column is sorted in descending order
TableSortChild(TVT_Product, NULL, "Designation", "-Price")
```


<a name="Example2"></a>

```wl
// Sort the branch in descending order
TableSortChild(TVT_Product, "Plumbing" + TAB + "Faucets", "Designation", "-Price")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Clasificando una rama en una control Tabla TreeView en una o más columnas identificadas por su nombre.

`<Result> = TableSortChild(<TreeView Table control> , <Branch to sort> , <Name of column 1> [, <Name of column 2> [... [, <Name of column N>]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó el orden,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<TreeView Table control>: (Nombre del control)`**

Nombre del control Tabla TreeView a manipular.

Si este parámetro corresponde a una cadena vacía (""), se utilizará el control al que pertenece la actual Event.

**`<Branch to sort>: (Entero o cadena de caracteres (con o sin comillas))`**

Si este parámetro es un número entero, corresponde a la índice del padre que contiene la rama a ordenar. 
Si este parámetro corresponde a NULL, la clasificación se realiza en los elementos "Raíz".... 
Si este parámetro corresponde a un string, corresponde al elemento padre que contiene la rama a clasificar.

**`<Name of column 1>: (Cadena de caracteres con comillas)`**

Nombre de la primera columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:
"[&lt;Direction&gt;]&lt;Nombre de columna&gt;" donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": orden descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.




**`<Name of column 2>: (Cadena de caracteres opcional con comillas)`**

Nombre de la segunda columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:
"[&lt;Direction&gt;]&lt;Nombre de columna&gt;" donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": orden descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.






**`<Name of column N>: (Cadena de caracteres opcional con comillas)`**

Nombre de la enésima columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:
"[&lt;Direction&gt;]&lt;Nombre de columna&gt;" donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": orden descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.





<a name="SYNTAX2"></a>

### Ordenar una rama en una control Tabla TreeView según una lista de nombres de columnas

`<Result> = TableSortChild(<List of columns> , <Branch to sort>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó el orden,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<List of columns>: (Cadena de caracteres)`**

Lista de columnas a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:

"[&lt;Dirección&gt;]&lt;Nombre de la columna1&gt; + TAB + [&lt;Dirección&gt;]&lt;Nombre de la columna2&gt; + TAB + ..." donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": orden descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar. [TableEnumColumn](../WDLang1/3074012.md) devuelve el nombre de la columna.




**`<Branch to sort>: (Entero o cadena de caracteres (con o sin comillas))`**

Si este parámetro es un número entero, corresponde a la índice del padre que contiene la rama a ordenar. 
Si este parámetro corresponde a NULL, la clasificación se realiza en los elementos "Raíz"....
Si este parámetro corresponde a un string, corresponde al elemento padre que contiene la rama a clasificar.


<a name="SYNTAX3"></a>

### Clasificar una rama en un control Tabla TreeView en las primeras diez columnas de Visible

`<Result> = TableSortChild(<TreeView Table control> , <Branch to sort> [, <To sort>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó el orden,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<TreeView Table control>: (Nombre del control)`**

Nombre de control Tabla TreeView a clasificar. Este parámetro tiene el siguiente formato:

"[&lt;Dirección&gt;]&lt;Control Tabla TreeView&gt;" donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": orden descendente.




- &lt;control Tabla TreeView&gt; (cadena de caracteres): nombre de control Tabla TreeView para clasificar.




**`<Branch to sort>: (Entero o cadena de caracteres (con o sin comillas))`**

Si este parámetro es un número entero, corresponde a la índice del padre que contiene la rama a ordenar. 
Si este parámetro corresponde a NULL, la clasificación se realiza en los elementos "Raíz".... 
Si este parámetro corresponde a un string, corresponde al elemento padre que contiene la rama a clasificar.

**`<To sort>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> si debe realizarse el orden,

- <u><u><u><u>False</u></u></u></u> si debe cancelarse el orden anterior. Una vez que se cancela la clasificación, el orden de las filas de tabla no se modifica, pero las filas que serán añadidas por [TableAdd](../WDLang1/3074017.md), [TableAddLine](../WDLang1/3074006.md), [TableAddChild](../WDLang1/1000010004.md), [TableInsert](../WDLang1/3074025.md), [TableInsertLine](../WDLang1/3074026.md) o [TableInsertChild](../WDLang1/1000010005.md) no serán clasificadas..

## Observaciones


### Gestionar una control Tabla TreeView clasificada
<a name="gestionar_una_control_tabla_treeview_clasificada_ELTPARAGRAPHE000271"></a>

- Al ordenar un control Tabla TreeView con la función **TableSortChild**, los elementos que se agregan al control con las funciones [TableAdd](../WDLang1/3074017.md), [TableAddLine](../WDLang1/3074006.md), [TableAddChild](../WDLang1/1000010004.md), [TableInsert](../WDLang1/3074025.md), [TableInsertLine](../WDLang1/3074026.md) o [TableInsertChild](../WDLang1/1000010005.md) tienen en cuenta el orden.

- Cuando se ordena una control Tabla TreeView haciendo clic, las adiciones hechas por [TableAdd](../WDLang1/3074017.md), [TableAddLine](../WDLang1/3074006.md), [TableAddChild](../WDLang1/1000010004.md), [TableInsert](../WDLang1/3074025.md), [TableInsertLine](../WDLang1/3074026.md) o [TableInsertChild](../WDLang1/1000010005.md) ignoran la ordenación.

- ***El orden no se cancela*** con las funciones [TableDisplay](../WDLang1/3074003.md), [TableAdjust](../WDLang1/3074004.md), [TableSeek](../WDLang1/3074015.md), [TableCount](../WDLang1/3074021.md), [TablePosition](../WDLang1/3074022.md), [TableSelect](../WDLang1/3074020.md), [TableSelectMinus](../WDLang1/3074029.md), [TableSelectPlus](../WDLang1/3074031.md), [TableDelete](../WDLang1/3074032.md) y [TableDeleteAll](../WDLang1/3074034.md).

- ***El orden no se cancela***: 

	- si se modifica una columna (por entrada directa en la columna o por [TableModify](../WDLang1/3074019.md) y [TableModifyLine](../WDLang1/3074028.md)) y si esta columna está clasificada. 

	- por [TableSort](../WDLang1/3074035.md) ([TableSort](../WDLang1/3074035.md)(&lt;control Tabla&gt;, False)).

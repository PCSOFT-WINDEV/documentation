


## TableDeleteSelect (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Borra las filas seleccionadas de una tabla o vista en árbol control Tabla.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Deletes all selected rows
IF YesNo("Do you really want to delete the selected customers?") THEN
	TableDeleteSelect(TABLE_Customer_Display)
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`TableDeleteSelect(<Table control>)`
---

**`<Table control>: (Nombre del control)`**

Nombre del control a manipular. Este control puede corresponder a: 

- un control Tabla.

- un control Tabla TreeView.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000065"></a>

La función **TableDeleteSelect** puede utilizarse en: 

- controles Tabla o Tabla TreeView basados en un archivo de datos.

- controles Tabla o Tabla TreeView rellenados mediante programación.

- controles de selección única o múltiple. 
	



<a name="NOTE0_2"></a>


### Eliminación de registros en una tabla o control Tabla TreeView basada en un archivo de datos
<a name="eliminacion_registros_una_tabla_control_tabla_treeview_basada_archivo_datos_ELTPARAGRAPHE000091"></a>

La función **TableDeleteSelect** elimina los registros correspondientes en el archivo de datos asociado al control. Si los elementos son relacionado para otros archivos de datos, no se realiza ninguna modificación automáticamente en estos archivos de datos de relacionado.
<a name="NOTE0_3"></a>


### Integridad referencial y Tabla o TreeView controles Tabla basado en un archivo de datos
<a name="integridad_referencial_tabla_treeview_controles_tabla_basado_archivo_datos_ELTPARAGRAPHE000102"></a>

Si la gestión de la integridad referencial está activada ([HSetIntegrity](../WDLang4/3044058.md)), se debe llamar a [HErrorIntegrity](../WDLang4/3044086.md) después de **TableDeleteSelect** para comprobar la integridad. Si se detecta un error de integridad, el fila no se borra y permanece Visible en el control.
<a name="NOTE0_4"></a>


### Eliminación de registros en una control Tabla basada en un archivo de datos y Trigger
<a name="eliminacion_registros_una_control_tabla_basada_archivo_datos_trigger_ELTPARAGRAPHE000119"></a>

Tiene la posibilidad de activar un disparador al borrar un Record de un control Tabla basado en un archivo de datos. Para obtener más información, consulte la documentación de [HDescribeTrigger](../WDLang4/3044074.md).
<a name="NOTE0_5"></a>


### Gestión de errores
<a name="gestion_errores_ELTPARAGRAPHE000131"></a>

La variable [ErrorOccurred](../WDLang1/3087001.md) se establece en <u><u><u><u>True</u></u></u></u> si la eliminación falla. Para get los detalles del error, utilice [ErrorInfo](../WDLang1/3013008.md).
<a name="NOTE0_6"></a>


### Bloquear el archivo de datos de relacionado
<a name="bloquear_archivo_datos_relacionado_ELTPARAGRAPHE000150"></a>

Si el archivo de datos está bloqueado en el proceso actual, la función **TableDeleteSelect** elimina la fila especificada y desbloquea el archivo de datos.

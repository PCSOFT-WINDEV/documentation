


## InBackgroundMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica si el código se ejecuta cuando la aplicación está en segundo plano.

Recordatorio: Se considera que una aplicación está en segundo plano cuando ninguna de sus ventanas se muestra en la pantalla.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// A dialog box can be displayed if the application is not in background, 
// otherwise, a a notification is performed
IF NOT InBackgroundMode() THEN
	Info("Process completed")
ELSE
	n is Notification
	n..Message = "Process completed"
	NotifAdd(n)
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = InBackgroundMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecuta el código mientras la aplicación está en segundo plano,

- <u><u><u><u>False</u></u></u></u> en caso contrario.








<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000187"></a>

La función **InBackgroundMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InBackgroundMode** pueden compartirse entre varios productos sin generar errores de compilación.
<a name="NOTE0_2"></a>


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000201"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.

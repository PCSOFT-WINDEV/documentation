
## Constantes de gestión de Thread
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>




| Constante | description | Utilizado por las funciones |
| --- | --- | --- |
| **DestructionWithoutModification** | Por defecto, un Event se modifica automáticamente durante su destrucción. Si el Event se crea con esta constante, no se modificará durante la llamada a EventDestroy o durante la destrucción del automatic del Event al cerrar la aplicación. | [EventCreate](../WDLang1/3077009.md) |
| **Infinite** | Espera infinita | [ThreadWait](../WDLang1/3077020.md), [ThreadStop](../WDLang1/3077019.md), <br>[ThreadWaitSignal](../WDLang1/3077022.md), [SemaphoreStart](../WDLang1/3077008.md), etc<br>[MutexStart](../WDLang1/1000019023.md) |
| **shareNone** | El Event, el semáforo o el mutex es específico de la aplicación. Si no se llama a [EventDestroy](../WDLang1/3077025.md), [SemaphoreDestroy](../WDLang1/3077011.md) o [MutexDestroy](../WDLang1/1000019025.md), el Event, semáforo o mutex se destruye al final de la aplicación. | [SemaphoreCreate](../WDLang1/3077012.md), [EventCreate](../WDLang1/3077009.md), <br>[MutexCreate](../WDLang1/1000019022.md) |
| **shareGlobal** | El Event, el semáforo o el mutex se comparte entre todas las aplicaciones que se encuentran en el equipo. | [SemaphoreCreate](../WDLang1/3077012.md), [EventCreate](../WDLang1/3077009.md), <br>[MutexCreate](../WDLang1/1000019022.md) |
| **shareUser** | El Event, el semáforo o el mutex se comparte entre todas las aplicaciones de la sesión del usuario que ejecuta la aplicación. | [SemaphoreCreate](../WDLang1/3077012.md), [EventCreate](../WDLang1/3077009.md), <br>[MutexCreate](../WDLang1/1000019022.md) |
| **PriorityLow** | thread con prioridad baja. | [Prioridad ThreadPriority](../WDLang1/3077027.md) |
| **PriorityHigh** | thread con alta prioridad. | [Prioridad ThreadPriority](../WDLang1/3077027.md) |
| **PriorityNormal** | thread con prioridad a Normal. | [Prioridad ThreadPriority](../WDLang1/3077027.md) |
| **eventAutomatic** | Una vez abierta, la Event se cierra automáticamente cuando todos los hilos se desbloquean. | [EventCreate](../WDLang1/3077009.md) |
| **eventClose** | La Event está cerrada. | [EventCreate](../WDLang1/3077009.md), [EventChange](../WDLang1/3077017.md) |
| **eventOpen** | El evento se abre. | [EventCreate](../WDLang1/3077009.md), [EventChange](../WDLang1/3077017.md) |
| **eventOpenAndClose** | Abre la Event, desbloquea todos los hilos pendientes y cierra la Event. | [EventChange](../WDLang1/3077017.md) |
| **eventManual** | La Event es modificada por la [EventChange](../WDLang1/3077017.md). | [EventCreate](../WDLang1/3077009.md) |
| **threadStopRequested** | Estado del hilo: Se ha hecho una solicitud de detención para este thread. | [ThreadState](../WDLang1/3077016.md), <br>Tipo [Thread](../WDLang1/1000024879.md) |
| **threadWaitForStart** | Modo de inicio del hilo: Espera a que el hilo inicie realmente antes de continuar con la ejecución. | [ThreadExecute](../WDLang1/3077024.md), <br>Tipo [Thread](../WDLang1/1000024879.md) |
| **threadAuto** | Gestión de hilos automatic. | [ThreadMode](../WDLang1/3077028.md) |
| **threadGlobalContext** | Modo de inicio del hilo: Fuerza el uso del contexto global del proyecto si el hilo se ejecuta desde una ventana. <br>El contexto de la ventana es usado por Default. | [ThreadExecute](../WDLang1/3077024.md), <br>Tipo [Thread](../WDLang1/1000024879.md) |
| **threadRunning** | Se está ejecutando el hilo. | [ThreadState](../WDLang1/3077016.md), <br>Tipo [Thread](../WDLang1/1000024879.md) |
| **threadUnknown** | El Thread no fue creado o fue destruido. | [ThreadState](../WDLang1/3077016.md) |
| **threadMonoProcessor** | Obliga a la aplicación a ejecutarse en un único procesador en los equipos con varios procesadores, hiperhilos o de doble núcleo. | [ThreadMode](../WDLang1/3077028.md) |
| **threadMultiProcessor** | Restaura el uso estándar de los procesadores en los ordenadores multiprocesador, hiperhilos o de doble núcleo. | [ThreadMode](../WDLang1/3077028.md) |
| **threadNotExecuted** | Inicie el hilo en modo Normal. | Tipo [Thread](../WDLang1/1000024879.md) |
| **threadNormal** | Inicie el hilo en modo Normal. | [ThreadExecute](../WDLang1/3077024.md) |
| **threadMain** | Maneja el principal Thread. | [ThreadStop](../WDLang1/3077019.md), <br>[ThreadSendSignal](../WDLang1/3077021.md), <br>Tipo [Thread](../WDLang1/1000024879.md) |
| **threadCriticalSection** | Gestión manual de secciones críticas. | [ThreadMode](../WDLang1/3077028.md) |
| **threadSecure** | Inicia un hilo secundario en modo secure. | [ThreadExecute](../WDLang1/3077024.md), <br>Tipo [Thread](../WDLang1/1000024879.md) |
| **threadSuspended** | El hilo está suspendido. | [ThreadState](../WDLang1/3077016.md)<br>Tipo [Thread](../WDLang1/1000024879.md) |
| **threadEnded** | El Thread se ha terminado. | Tipo [Thread](../WDLang1/1000024879.md) |
| **threadUseHyperFile**<br>**threadUseHFSQL**<br>**threadFullCopyHFSQLContext** | Dispara el inmediato copy del contexto HFSQL. <br>Se recomienda si el hilo debe tener en cuenta las posiciones actuales en los archivos y consultas del contexto llamador. | [ThreadExecute](../WDLang1/3077024.md), <br>Tipo [Thread](../WDLang1/1000024879.md) |
| **threadLightCopyHFSQLContext** | Dispara el inmediato copy de parte del contexto HFSQL.<br>Solo se almacenan los directorios que contienen los archivos de datos en modo HFSQL Classic y/o las conexiones en modo HFSQL Client/Server. | [ThreadExecute](../WDLang1/3077024.md), <br>Tipo [Thread](../WDLang1/1000024879.md) |

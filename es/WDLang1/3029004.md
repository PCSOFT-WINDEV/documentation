


## dArc (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Dibuja un arco de círculo o un arco de elipse:

- en un control Imagen,

- en una variable de tipo [Image](../WDLang1/1000019650.md).
	

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en una variable de tipo [WDPic](../WDLang1/1000023503.md) (en la capa de fondo),

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en una variable de tipo [picLayer](../WDLang1/1000024604.md).




El arco se dibuja en sentido contrario a las agujas del reloj.
![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) **Atención**: Se requiere una configuración específica para usar esta función en Linux. Para obtener más información, consulte [Los dibujos](../WDLang1/3029035.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Initialize the drawing in the Image control IMG_MyDrawing
dStartDrawing(IMG_MyDrawing)

// Draw an arc of circle in light yellow
dArc(10, 10, 50, 50, 20, 20, 40, 50, LightYellow)
```

![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png)


## Sintaxis

### Dibujar un arco

`dArc([<Image>, ] <X1> , <Y1> , <X2> , <Y2> , <X3> , <Y3> , <X4> , <Y4> [, <Line color> [, <Line thickness>]])`
---

**`<Image>: (Nombre de control opcional o variable de tipo Image, WDPic o picLayer opcional)`**

![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) Imagen a manipular. Esta imagen puede corresponder a: 

- el nombre de un control Imagen. 

- el nombre de una variable de tipo [Image](../WDLang1/1000019650.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) el nombre de una variable de tipo [WDPic](../WDLang1/1000023503.md). Solo se manipulará la capa de fondo. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) el nombre de una variable de tipo [picLayer](../WDLang1/1000024604.md).


Si no se especifica este parámetro, se debe definir el destino del dibujo con la función [dStartDrawing](../WDLang1/3029009.md). 

**`<X1>: (Entero)`**

Coordenada X (en pixeles) de la esquina superior izquierda del rectángulo que contiene el círculo.

**`<Y1>: (Entero)`**

Coordenada Y (en pixeles) de la esquina superior izquierda del rectángulo que contiene el círculo.

**`<X2>: (Entero)`**

Coordenada X (en pixeles) de la esquina inferior derecha del rectángulo que contiene el círculo.

**`<Y2>: (Entero)`**

Coordenada Y (en pixeles) de la esquina inferior derecha del rectángulo que contiene el círculo.

**`<X3>: (Entero)`**

Coordenada X (en pixeles) del punto inicial del arco.

**`<Y3>: (Entero)`**

Coordenada Y (en pixeles) del punto inicial del arco.

**`<X4>: (Entero)`**

Coordenada X (en pixeles) del punto final del arco.

**`<Y4>: (Entero)`**

Coordenada Y (en pixeles) del punto final del arco.

**`<Line color>: (Entero o constante (opcional))`**

Color del arco de círculo. Este color puede corresponder a:

- un color RGB (devuelto por la función [RGB](../WDLang1/3029012.md)),

- un color HSL (devuelto por la función [HSL](../WDLang1/3029057.md)),

- [un color predefinido de WLanguage](../WDLang5/3010002.md), 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) una variable de tipo [Color](../WDLang1/1000019661.md).


Si no se especifica este parámetro, el color de la línea:

- es *Transparente* si no se ha utilizado la función [dPen](../WDLang1/3029029.md) previamente, 

- corresponde al color especificado en la última llamada a la función [dPen](../WDLang1/3029029.md). 




**`<Line thickness>: (Entero opcional)`**

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Grosor (en pixeles) del arco de círculo a dibujar. Si no se especifica este parámetro, el valor tenido en cuenta puede ser:

- el último valor especificado en la última llamada a la función [dPen](../WDLang1/3029029.md).

- 1 si no se ha utilizado la función [dPen](../WDLang1/3029029.md) previamente.

## Observaciones


### Coordenadas del arco
<a name="coordenadas_del_arco_ELTPARAGRAPHE000334"></a>Si ***los puntos inicial y final*** no están situados en el borde del círculo (o de la elipse), el punto que se tiene en cuenta es la intersección entre el círculo y la línea que une el punto especificado con el centro del rectángulo (es decir, el centro del círculo).<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=RBArc2.gif)
El origen utilizado para calcular las coordenadas corresponde a la esquina superior izquierda de la imagen (coordenadas: (0,0)).
<a name="NOTE0_2"></a>


### Dibujar una figura con relleno
<a name="dibujar_una_figura_con_relleno_ELTPARAGRAPHE000345"></a>

Para dibujar una figura con relleno (un arco completo), utilice las funciones [dChord](../WDLang1/3029008.md) o [dSlice](../WDLang1/3029021.md).
<a name="NOTE0_3"></a>


### Uso de las funciones de dibujo
<a name="uso_las_funciones_dibujo_ELTPARAGRAPHE000364"></a>

Las funciones de dibujo se pueden utilizar según 2 métodos:

- **Método 1: Uso de la función dStartDrawing**

	- Se debe llamar a la función [dStartDrawing](../WDLang1/3029009.md) antes de utilizar otras funciones de dibujo. La función [dStartDrawing](../WDLang1/3029009.md) permite definir el elemento (control Imagen o variable) en el que se aplicarán las funciones de dibujo.

	- Al dibujar en un control Imagen: 

		- Las funciones de dibujo trabajan en una copia ("bitmap") de la imagen. Se pueden utilizar las funciones de dibujo de Windows (con la función [API](../WDLang1/3014005.md) o [CallDLL32](../WDLang1/3014003.md)), pero estas funciones deben utilizar el DC (Device Context) devuelto por la función [dStartDrawing](../WDLang1/3029009.md).

		- las funciones [dEndDrawing](../WDLang1/3029019.md) y [dStartDrawing](../WDLang1/3029009.md) no deben llamarse en el mismo proceso.
						Si se llama a las funciones [dEndDrawing](../WDLang1/3029019.md) y [dStartDrawing](../WDLang1/3029009.md) en el mismo proceso, no se mostrará el dibujo: se borrará automáticamente.




- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) **Método 2: indicar el destino del dibujo directamente en la sintaxis (mediante un parámetro)**
	El parámetro &lt;Imagen&gt; permite especificar directamente el destino del dibujo. Ya no es necesario utilizar [dStartDrawing](../WDLang1/3029009.md). Esta función debe ser eliminada.
	



<a name="NOTE0_4"></a>
**Limitación de la función dArc**: La opacidad definida con la función [dBackground](../WDLang1/3029002.md) no se tiene en cuenta.




## InJavaMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica si el código se ejecuta o no en Java.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Code run in Java?
IF InJavaMode() = True THEN
	// Java code
	// Process to perform
	...
ELSE
	// Non-Java code
	// Process to perform
	...
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = InJavaMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecuta el código en Java, 

- <u><u><u><u>False</u></u></u></u> en caso contrario.


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Este parámetro siempre corresponde a <u><u><u><u>True</u></u></u></u>.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000229"></a>

La función **InJavaMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InJavaMode** pueden compartirse entre varios productos sin generar errores de compilación.
<a name="NOTE0_2"></a>


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000243"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.

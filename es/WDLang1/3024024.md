
## NumToString (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Convierte un valor numérico (entero, real o monetario) en una cadena según el formato especificado.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
NumToString(123)	// Returns "123"
NumToString(1.23)	// Returns "1.23"
NumToString(12345.5, "+10.2f") // Returns " +12345.50"
NumToString(12345.5, "-+10.2f") // Returns "+12345.50 "
NumToString(12345.5, "+010.2f") // Returns "+012345.50"
NumToString(12345.5, "010.2f") // Returns "0012345.50"

NumToString(12345.5, "10.2fS")
// Returns " 12 345.50" (the thousand separator is " ")

NumToString(12345.5, "10,2fS")
// Returns " 12 345,50" (the decimal separator is ",")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = NumToString(<Number> [, <Format>])`
---

**`<Result>: (Cadena de caracteres)`**

Cadena que representa el número según el formato especificado. 
Si el parámetro &lt;Número&gt; es una cadena, la cadena devuelta es la misma que la cadena inicial. El formato se ignora.



**`<Number>: (Entero o real)`**

Valor numérico a convertir en una cadena (según el formato especificado).

**`<Format>: (Cadena de caracteres opcional o constante)`**



- Formato de la cadena expresado de la siguiente manera: 
	
	```txt
	"[<Align>][<Sign>][<Zero>][<Length>][<.Precision>]<Type>[<Thousand>]"
	```

	
	Para obtener más información, consulte [las observaciones](#NOTE0_1). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Constante que permite definir un formato del sistema:
	


|   |   |
| --- | --- |
| *maskSystemCurrency* | Máscara de moneda utilizada por el sistema. |
| *maskSystemNumeral* | Máscara numérica utilizada por el sistema. |

## Observaciones




### Formato detallado de la cadena que se va a obtener
<a name="formato_detallado_cadena_que_obtener_ELTPARAGRAPHE000112"></a>

Para convertir un valor numérico en una cadena, es necesario especificar el formato de la cadena que se obtendrá. El formato es el siguiente: 


```txt
"[<Align>][<Sign>][<Zero>][<Length>][<.Precision>]<Type>[<Thousand>]"
```




Los valores posibles son: 


|   |   |
| --- | --- |
| &lt;Alineación&gt; | Si se especifica este parámetro, alinea el número a la izquierda (&lt;Alineación&gt; = "-"). <br>De forma predeterminada, el número está alineado a la derecha. |
| &lt;Signo&gt; | Si se especifica este parámetro, muestra el signo para los números positivos (&lt;Signo&gt; = "+"). El signo aparecerá a la izquierda del primer dígito significativo. <br>De forma predeterminada, los números negativos tienen el signo "-" a la izquierda del primer dígito significativo.<br>Caso especial: si se especifica este parámetro y el número es igual a 0, se mostrará el signo "+". Este signo también se mostrará si un número negativo se convierte en 0. |
| &lt;Cero&gt; | Si se especifica este parámetro (&lt;Cero&gt; ="0"), se insertarán ceros a la izquierda del primer dígito significativo si la parte significativa del número es menor que la longitud especificada y el número está alineado a la derecha. <br>De forma predeterminada, no se insertan ceros "0" a la izquierda del número.<br>Por ejemplo: NumToString(123, "05d") devolverá "00123". |
| &lt;Longitud&gt; | Define la longitud total de la cadena resultante. La longitud total corresponde a los siguientes elementos: signo, separador de miles, punto decimal, exponente, parte entera y parte decimal del número. La longitud debe ser inferior a 100. <br>Si la longitud especificada es demasiado pequeña para contener la parte significativa del número, se ignorará. <br>De forma predeterminada, el número tiene la misma longitud que la parte significativa (no se insertan ceros). Lo mismo se aplica si &lt;Longitud&gt; = 0. <br>Si la longitud especificada es mayor que la parte significativa del número:<br><br>- se insertarán espacios a la derecha de la cadena resultante si el número está alineado a la izquierda, <br><br>- se insertarán espacios o ceros a la izquierda de la cadena resultante si el número está alineado a la derecha.<br><br><br><br><br> |
| &lt;.Precisión&gt; | Define el separador decimal (caracteres "." o ",") y el número de posiciones decimales (solo para los &lt;Tipos&gt; flotante ("f") y exponente ("e")). <br><br>**Separador decimal**: <br><br>- Si el separador decimal es una coma, se utilizará el separador decimal que se definió: <br><br>	- en los parámetros lingüísticos de Windows.<br><br><br><br><br>- Si el separador decimal es un punto, el separador decimal utilizado será un punto.  <br><br><br><br><br>Observaciones: <br><br>- La función [ChangeSeparator](../WDLang1/3054004.md) no tiene ningún efecto sobre este separador. Esta función solo afecta la máscara de visualización de los valores numéricos en los controles. <br><br><br><br><br>**Número de decimales**: <br><br>- El número de decimales debe ser inferior a 100. <br><br>- Si el número de decimales en el número es mayor que &lt;Precisión&gt;, la parte decimal se redondea al número especificado de decimales.<br><br>- Si el número de decimales en el número es menor que &lt;Precisión&gt;, la parte decimal se completa con ceros "0" a la derecha.<br><br>- El valor de &lt;Precisión&gt; puede ser mayor que &lt;Longitud&gt;. En este caso, el argumento &lt;Longitud&gt; se ignora.<br><br><br><br><br> |
| &lt;Tipo&gt; | Tipo de resultado, parámetro obligatorio: <br><br>- d: **Entero**: <br>	Los números reales se redondean al número entero más cercano.<br><br>- e: **Notación exponencial**:<br>	Si no se especifica el argumento &lt;Precisión&gt;, la precisión predeterminada es 6.<br><br>- f: **Notación decimal**:<br>	Si no se especifica el argumento &lt;Precisión&gt;, la precisión predeterminada es 6.<br><br>- x y X: **Notación hexadecimal** (X para letras mayúsculas):<br>	Solo se tiene en cuenta la parte entera del número (no se redondea).<br>	Caso particular:<br>	1. Si no se especifica la longitud y el número es negativo: 20 caracteres hexadecimales para un valor monetario, 8 caracteres para los demás tipos numéricos.<br>	2. Si el número es negativo: la longitud de la cadena resultante será al menos la longitud que permita especificar el bit de signo. <br>	Por ejemplo: -32767 en el formato "3X" devolverá "8001"<br><br>- o: **Notación octal**: <br>	Solo se tiene en cuenta la parte entera del número (no se redondea). <br>	Caso particular:<br>	1. Si no se especifica la longitud y el número es negativo: 27 caracteres hexadecimales para un valor monetario, 11 caracteres para los demás tipos numéricos.<br>	2. Si el número es negativo: la longitud de la cadena resultante será al menos la longitud que permita especificar el bit de signo. <br>	Por ejemplo: -32767 en un formato "3o" devolverá "700001"<br><br><br> |
| &lt;Miles&gt; | Separador de miles (&lt;Miles&gt; = "S" o "s"). Los dígitos de la parte entera del número se agrupan por 3 y se separan con el separador de miles definido en Windows. Si hay un signo, este siempre se mostrará a la izquierda del primer dígito significativo. No debe haber separadores de miles entre el signo y el primer dígito.<br><br><br> |

### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000266"></a>

- El formato "f" devuelve un resultado válido para números entre -1e+17 y 1e+17.

- Formatos "x", "d" u "o": la parte entera no debe exceder 99 999 999 999 999 999.



<a name="NOTE0_3"></a>


### Convertir un tamaño expresado en bytes
<a name="convertir_tamano_expresado_bytes_ELTPARAGRAPHE000287"></a>

Para convertir un tamaño expresado en bytes, utilice la función [LengthToString](../WDLang1/1000017038.md).

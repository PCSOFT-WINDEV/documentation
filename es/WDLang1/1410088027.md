
## Notificaciones interactivas
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000145"></a>
Una Notification es una pequeña ventana que aparece en la parte inferior derecha de la pantalla, y que se puede ver en cualquier momento en la zona Notification de Windows.
Este tipo de ventana ha sido estandarizado por Windows (utilizando XML).

En la versión 27, puedes crear fácilmente notificaciones para que se muestren en Windows.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WD%20-%20Notif%20Windows%2011.gif)


Una Notification puede contener:

- imágenes, 

- texto, 

- controles Campo de entrada,

- botones,

- cuadros combinados.




WINDEV incluye múltiples funciones y tipos de WLanguage para crear estas notificaciones.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000169"></a>
Para crear y mostrar una Notification: 

1. Compruebe si el sistema del usuario admite las notificaciones interactivas mediante [SysNotificationCompatible](../WDLang1/1410087932.md). 

2. Cree una variable de tipo [sysNotification](../WDLang1/1410087893.md). Esta variable contiene la descripción completa de la Notification. 

	- Para añadir zonas de entrada a la Notification, cree una o varias variables de tipo [sysNotificationInput](../WDLang1/1410088024.md). La propiedad **Type** de este tipo de variable determina si el área de entrada es un Combo Box o un área de texto. 

	- Para añadir botones a la Notification, cree una o varias variables de tipo [sysNotificationButton](../WDLang1/1410087947.md). La propiedad **Type** de este tipo de variable determina el tipo y la posición del botón a crear.

	- Definir las acciones que se realizarán cuando el usuario haga clic en la Notification o en un botón de la Notification (véase el párrafo siguiente). Estas acciones pueden corresponder a: 

		- un global específico Procedure. 

		- el proyecto opcional "Recibir una notificación" Event( este Event se llama si el procedimiento no existe o no se ha especificado en el **Acción** Property). 




3. Enviar la Notification al sistema con [SysNotificationSend](../WDLang1/1410087906.md).





### Haga clic en una Notification o en un botón de una Notification
<a name="haga_clic_una_notification_boton_una_notification_ELTPARAGRAPHE000061"></a>

La información sobre la interacción del usuario con un Notification se recupera a través de un sistema de procedimientos Callback.

Puede asociar un procedimiento específico a cada acción realizada en un Notification (ya sea en el propio Notification o en sus botones). Sin embargo, dado que una Notification puede reabrir una aplicación, se debe utilizar una Procedure GLOBAL porque debe ser accesible desde el momento en que se abre la aplicación.

El nombre del procedimiento a ejecutar está contenido en el Notification a través de la acción **** Property. Si no se especifica ningún nombre (p. ej., si el usuario utiliza un archivo XML personalizado) o no se encuentra el procedimiento, se ejecuta el evento opcional del proyecto "Recepción de una notificación".

Tanto el procedimiento como el evento opcional del proyecto "Recepción de una notificación" deben tener el siguiente formato: 

```txt
PROCEDURE ProcedureName(arguments is string, data is array of string)
```
 donde: 

- &lt;argumentos&gt; contiene los argumentos asociados a la acción (a través del **Nota** Property del [sysNotification](../WDLang1/1410087893.md) Variable para un clic en el Notification, o a través del **Nota** Property del [sysNotificationButton](../WDLang1/1410087947.md) Variable para un clic en un botón). 

- &lt;datos&gt; contiene los datos introducidos por los usuarios en los controles de la Notification (control Campo de entrada o cuadro combinado). Cada cadena tiene el siguiente formato: 
	
	```txt
	<Control name> + TAB + <Control value>
	```





Observación: Los procedimientos y la Event opcional se ejecutan en la Thread principal de la aplicación (acceso directo a los controles).

## Funciones y tipos de variables associated con notificaciones interactivas
<a name="funciones_tipos_variables_associated_con_notificaciones_interactivas_ELTTEXTE000199"></a>
Las siguientes funciones se utilizan para las notificaciones interactivas de manipular: 


|   |   |
| --- | --- |
| [InSysNotificationMode](../WDLang1/1410087933.md) | Se utiliza para determinar si la aplicación se lanza desde una Notification interactiva en "Windows 10". |
| [SysNotificationClear](../WDLang1/1410087937.md) | Elimina todas las notificaciones interactivas enviadas previamente por la aplicación actual. |
| [SysNotificationCompatible](../WDLang1/1410087932.md) | Comprueba si el sistema operativo soporta notificaciones interactivas. |
| [SysNotificationRemove](../WDLang1/1410087907.md) | Elimina un Notification interactivo enviado anteriormente. |
| [SysNotificationRemoveID](../WDLang1/1410087939.md) | Elimina todas las notificaciones interactivas enviadas anteriormente que coincidan con el identificador (ID) especificado. |
| [SysNotificationSend](../WDLang1/1410087906.md) | Envía una Notification interactiva al sistema para que sea visualizada. |





Los siguientes tipos de variables se utilizan para las notificaciones interactivas de manipular: 



|   |   |
| --- | --- |
| [sysNotification](../WDLang1/1410087893.md) | El tipo **sysNotification** se utiliza para definir todas las características avanzadas de un sistema interactivo Notification. |
| [sysNotificationButton](../WDLang1/1410087947.md) | El tipo **sysNotificationButton** se utiliza para definir todas las características avanzadas de control Botón en un Notification interactivo. |
| [sysNotificationInput](../WDLang1/1410088024.md) | El tipo **sysNotificationInput** se utiliza para definir todas las características avanzadas de un control de entrada, como una entrada de texto o un control de cuadro combinado, en un Notification interactivo. |


## EmulateOKCancel( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Emula la siguiente llamada a [OKCancel](../WDLang1/3021004.md) y pulsa automáticamente el botón pasado como parámetro.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Automatically close the next OKCancel dialog box
// The answer on which the click will be performed is OK
EmulateOKCancel(True)

// Process 
// ...

SWITCH OKCancel("Do you want to save the modifications?")
	CASE True
	// Save the modifications
	CASE False
	// Cancel the modifications
END // The message is automatically closed via a click on OK
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`EmulateOKCancel(<Button>)`
---

**`<Button>: (Constante de tipo Integer)`**

Indica el botón de la ventana de OKCancelar en el que se debe hacer el clic:


|   |   |
| --- | --- |
| *False* | Haga clic en [ CANCEL] |
| *True* | Haga clic en [ OK] |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Comprobación de la función
<a name="comprobacion_funcion_ELTPARAGRAPHE000054"></a>

La existencia de una función de emulación ([EmulateWarning](../WDLang1/1000009001.md), [EmulateError](../WDLang1/1000009005.md), [EmulateInfo](../WDLang1/1000009006.md), ...) se comprueba durante la llamada a la función de diálogo asociada ([Advertencia](../WDLang1/3021009.md), [Error](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ...).
<a name="NOTE0_2"></a>




### Diferencia de clase de mensaje
<a name="diferencia_clase_mensaje_ELTPARAGRAPHE000080"></a>

El orden de las funciones [EmularXXX](../WDLang1/1000009002.md) debe respetar el orden de visualización de [Error](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ... De lo contrario, se producirá un error fatal.




## ParallelTaskRequestCancellation (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Solicita la cancelación de una tarea paralela.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Triggers the execution of a task
t is ParallelTask = ParallelTaskExecute(Proc)
...
// Cancels the execution of the task
ParallelTaskRequestCancellation(t)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Solicitar la cancelación de una tarea paralela

`ParallelTaskRequestCancellation(<Task>)`
---

**`<Task>: (ParallelTask Variable)`**

Nombre de la variable de tipo [ParallelTask](../WDLang1/1000021220.md) que se utilizará. 


<a name="SYNTAX2"></a>

### Solicitar la anulación de tareas paralelas encontradas en una array

`ParallelTaskRequestCancellation(<Array of tasks>)`
---

**`<Array of tasks>: (Array de las variables de la ParallelTask)`**

Nombre de las variables array de [ParallelTask](../WDLang1/1000021220.md) que se van a utilizar. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Se transmite una solicitud de cancelación a la tarea. El código de la tarea debe gestionar la solicitud de cancelación mediante la siguiente sintaxis: 
	
	```txt
	MyParallelTask.Canceled
	```


- La llamada a la función **ParallelTaskRequestCancellation** no es bloqueante. La tarea se considera cancelada cuando se llama a la función.

- La tarea está marcada como *ptsCancellationRequested*. Cualquier llamada a la [ReturnedValue](../Proprietes/2510134.md) Property provocará un error fatal.

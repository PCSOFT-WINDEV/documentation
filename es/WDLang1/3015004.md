
## Multitask( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite:

- definir un timeout,

- Devuelve a control a Windows,

- devuelve a control a Windows y a WLanguage.













<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Calculation loop
FOR i = 1 TO 10000
	CalculationProcedure(i)
	// Give control back to the system in order to display the results
	Multitask()
END
// Wait 1 second 
Multitask(100)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`Multitask([<Timeout>])`
---

**`<Timeout>: (Entero o duración opcional)`**



- **Definir un timeout**: &lt;Timeout&gt; = entero positivo. 
	La ejecución de la aplicación se interrumpe durante &lt;Timeout&gt; centésimas de segundo sin devolver a control al sistema. Los clics del ratón y las acciones realizadas en las teclas del teclado son ignoradas por la aplicación.. Por ejemplo, la ventana se puede mover pero no se puede redimensionar..
	La [procesos opcionales](../WDChamp/1014004.md) de los controles se ejecuta siempre.
	**Observación**: Si &lt;Timeout&gt; = 1, la aplicación devuelve a control a Windows sin esperar durante 1 centésima de segundo.
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) No se ignora la modificación del tamaño de la ventana. Por el contrario, el ratón hace clic y las acciones en las teclas del teclado son ignoradas..
	![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Los valores positivos no están permitidos.

- **Give control volver al sistema**: &lt;Tiempo de espera&gt; = 0 o no especificado. 
	El sistema termina los procesos actuales, devuelve control a la aplicación WINDEV, WEBDEV o WINDEV Mobile, luego tiene en cuenta los clics y las acciones realizadas en las teclas del teclado.
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Si &lt;Tiempo de espera&gt; = 0 la función **Multitask** no tiene ningún efecto.
	

- **Give control volver a Windows y al WLanguage**: &lt;Timeout&gt; = entero negativo. 
	La ejecución de la aplicación se interrumpe durante &lt;Timeout&gt; centésimas de segundo.. Se pueden ejecutar otros procesos durante este período de tiempo (por ejemplo, volver a mostrar o ejecutar un código de clic).. Las interacciones con los usuarios (clic del ratón, pulsación de teclas,...) se procesan como de costumbre.. 
	**Observación**: Si &lt;Timeout&gt; = -1, la aplicación devuelve control a Windows sin esperar una centésima de segundo.


Observación: Este parámetro puede corresponder a: 

- un número entero que corresponde al número de centésimas de segundo,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, "1s" o "10cs").

### Ejemplo de timeout
<a name="ejemplo_timeout_ELTPARAGRAPHE000131"></a>

El usuario presiona la tecla Esc antes o al llamar a la función **Multitask**:

- si &lt;Timeout&gt; es positivo: La tecla Esc se ignora.

- si &lt;Timeout&gt; es igual a 0: La tecla Esc se tiene en cuenta al final de los procesos actuales de WINDEV.

- si &lt;Timeout&gt; es negativo: La tecla Esc se tiene en cuenta inmediatamente.



<a name="NOTE0_2"></a>




### Consejos
<a name="consejos_ELTPARAGRAPHE000145"></a>

- Para actualización el UI, utilice [MultitaskRedraw](../WDLang1/3038004.md).

- Los eventos y timers iniciados antes de ejecutar la función **Multitask** se gestionan durante el tiempo de espera.

- En una aplicación de Servicio, **Multitask** debe sustituirse por [ServicioEspera](../WDLang1/1000018712.md).



<a name="NOTE0_3"></a>




### Nota
<a name="nota_ELTPARAGRAPHE000165"></a>

Se recomienda utilizar [ThreadPause](../WDLang1/3077029.md) en lugar de **Multitask** o [Timeout](../WDLang1/3015012.md) cuando se utilicen varios hilos (incluido el hilo principal de la aplicación) si el temporizador no necesita Process acciones del usuario.




## &lt;Variable DiskFile&gt;.Size (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el tamaño de un archivo (en bytes).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Open an external file
MyFile is DiskFile
MyFile = fOpen("C:\MyDirectories\File.txt", foReadWrite)

// Size of a file
ResSize = MyFile.Size()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Obtener el tamaño de un archivo utilizando la ruta

`<Result> = <File path>.Size()`
---

**`<Result>: (Entero)`**



- Tamaño del archivo en bytes,

- -1 si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*. 


Cuando se manejan archivos de más de 2 GB, &lt;Result&gt; debe corresponder a un entero de 8 bytes.

**`<File path>: (Variable de tipo DiskFile)`**

Nombre de la variable [DiskFile](../WDLang1/1410088883.md) inicializada con [fOpen](../WDLang1/3036036.md), [fCreate](../WDLang1/3036017.md) o [fOpenTempFile](../WDLang1/1000017289.md).

## Observaciones


### Gestión de errores
<a name="gestion_errores_ELTPARAGRAPHE000253"></a>

La función **&lt;Variable DiskFile&gt;.Size** genera un error en los siguientes casos:

- el archivo especificado no existe,

- el nombre del archivo especificado no es válido.




**Observación**: **&lt;Variable DiskFile&gt;.Size** no actúa sobre los archivos ocultos.

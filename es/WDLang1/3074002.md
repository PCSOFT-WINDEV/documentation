
## BuildBrowsingTable (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Crea una control Tabla a partir de los elementos de un archivo de datos (vista o consulta). Cada columna de control está vinculada a un elemento de el archivo de datos. La anchura de las columnas cambia automáticamente en función de los datos del archivo o de la consulta.

Esta función se utiliza para get un control Tabla adaptado a la navegación de un archivo de datos específico.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003074002&name=buildbrowsingtable_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Displays the content of "CUSTOMERS" file in a Table control based on a data file 
BuildBrowsingTable(TABLE_TABLE1, "CUSTOMERS", taWithAutoID)
```


<a name="Example2"></a>

```wl
// Displays the query content in a Table control populated programmatically
IF HExecuteQuery(QRY_LIST_ACTIVE_CUSTOMERS) = True THEN
	BuildBrowsingTable(TABLE_CUSTOMERS, ...
			QRY_LIST_ACTIVE_CUSTOMERS, taFillTable)
END
```





<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`BuildBrowsingTable(<Table control> , <Data file> [, <Options>])`
---

**`<Table control>: (Nombre del control)`**

Nombre de la control Tabla (basada en un archivo de datos o rellenada mediante programación). 

Atención: Esta control debe existir en la ventana. Puede haber sido creado por [ControlCreate](../WDLang1/1000020876.md).

Los registros de el archivo de datos se pueden ver directamente si este parámetro corresponde a un control Tabla basado en un archivo de datos.

**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos, la vista o la consulta utilizados como base para la creación de la control Tabla.

**`<Options>: (Constante o combinación de constantes opcional)`**

Configura el modo de creación de columnas en el control Tabla.

|   | Modo de funcionamiento con la constante | Modo de funcionamiento Default |
| --- | --- | --- |
| *taWithBinary* | Los elementos binarios (clave compuesta de enlace, por ejemplo) se añaden a la control Tabla. | Por Default: Los elementos binarios (clave compuesta de enlace, por ejemplo) no se añaden a la control Tabla. |
| *taWithAutoID* | Se genera una columna para el identificador automatic. | Por Default: No hay columna para el identificador automatic. |
| *taWithDetailsBinding* | Se genera automáticamente un sub-tabla para mostrar los elementos vinculados de los archivos de datos del análisis. | Por Default: No se muestra ninguna ventana "Detalles de fila". |
| *taKeepBrowseKey* | La clave de búsqueda de la control Tabla basada en la un archivo de datos no cambia (esta clave de búsqueda es devuelta por la [BrowsedItem](../Proprietes/2510111.md) Property). | Por Default: El primer elemento clave encontrado se posiciona como clave de búsqueda. |
| *taKeepColumn* | Las columnas existentes en la control Tabla no se borran. | Por Default: Se borran las columnas existentes. |
| *taItemCaption* | Los títulos de los artículos se utilizan para los títulos de las columnas. | Por Default: El nombre del elemento se utiliza como título de columna. |
| *taByLogicalOrder* | Las columnas se crean según el orden lógico de los artículos definidos en el editor de análisis.<br>Para las consultas, el orden lógico y el orden físico de las posiciones son idénticos. | Por Default: Se utiliza el orden físico de los artículos en el archivo de datos.<br>Para las consultas, el orden lógico y el orden físico de las posiciones son idénticos. |
| *taFillTable* | Los controles de la tabla se rellenan mediante programación con el contenido de el archivo de datos (Atención: esta puede ser una operación larga que puede provocar el desbordamiento de la memoria). | Por Default: Los controles de la tabla no se rellenan mediante programación. |
| *taWithAutoID* | Entrada permitida en la columna asociada al identificador automatic (si existe y si la control Tabla está en edición). |   |
| *taNoAdjust* | El ancho de las columnas no se ajusta (los registros no se leen). |   |
| *taNoMemoText* | Los ítems'text memo' no se añaden a la control Tabla. | Por Default: Los elementos "nota de texto" se añaden a la control Tabla. |
| *taNoMemoImage* | Las partidas de "Image memo" no se añaden a la control Tabla. | Por Default: Los elementos de "Image memo" se añaden a la control Tabla. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Características de la control Tabla (por Default)
<a name="caracteristicas_control_tabla_por_default_ELTPARAGRAPHE000141"></a>

- Se borran las columnas de control Tabla definidas en el editor.

- Los modos de entrada definidos en el editor para el control Tabla se mantienen.

- Para cada artículo del archivo de datos se crea una columna cuyo tipo es idéntico al del artículo.

- No se crea ninguna columna para las partidas binarias (se crea una columna para las partidas de texto o para las partidas Image).

- Las propiedades de los controles y columnas de la tabla se actualizan para permitir la navegación por el archivo:
	
	```txt
	<Table control>.BrowsedFile = <Data file name>
	```

	y para cada columna:
	
	```txt
	<Column name>.DataBinding = <Name of the bound item>
	<Column name>.BrowsedItem = <First key in data file>
	```

### Control Tabla rellenado mediante programación
<a name="control_tabla_rellenado_mediante_programacion_ELTPARAGRAPHE000160"></a>

Si &lt;control Tabla&gt; corresponde a un control Tabla poblado programáticamente, no se muestra ningún dato (el control Tabla está vacío) pero se asigna el control Tabla y las propiedades de las columnas como en el caso de un control Tabla basado en un archivo de datos.

Para visualizar los datos en el control Tabla, utilice las funciones [TableAdd](../WDLang1/3074017.md) o **BuildBrowsingTable** con la constante *taFillTable*.

**Observación**: La constante *taFillTable* no puede utilizarse para rellenar programáticamente los controles de la Tabla con memos Image. Para añadir un Image en un control Tabla poblado de forma programática, el archivo correspondiente debe encontrarse en la biblioteca de la aplicación (archivo WDL) o debe ser accesible desde el ordenador actual.
<a name="NOTE0_3"></a>




### Columnas
<a name="columnas_ELTPARAGRAPHE000177"></a>

Las columnas creadas se denominan "_COL1", "_COL2", "_COL3", etc.. Se puede acceder a estas columnas mediante programación a través del operador indirecto { }.

**No puedes escribir**:


```wl
TABLE_TABLE1._COL1.Width = 120 // Generates an error "_COL1 unknown"
```


Debes escribir:


```wl
{"TABLE_TABLE1._COL1"}..Width = 120
```

<a name="NOTE0_4"></a>




### Varios
<a name="varios_ELTPARAGRAPHE000193"></a>

- La función **BuildBrowsingTable** navega por el archivo de datos. La Record actual puede ser modificada una vez utilizada esta función.

- La tabla se completa con la función **BuildBrowsingTable** en un proceso paralelo en segundo plano: si la fuente de datos es una consulta, esta consulta no debe liberarse antes de cerrar la ventana o antes de reiniciar control Tabla.

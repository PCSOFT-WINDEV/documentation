
## dbgConnect (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite conectar un depurador remoto a una aplicación, servicio, Webservice, etc.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Start remote debugger
dbgConnect("1.1.1.1")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = dbgConnect(<Debugger>)`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se estableció la conexión,

- <u><u><u><u>False</u></u></u></u> en caso contrario. 




**`<Debugger>: (Cadena de caracteres)`**

Nombre o dirección IP del equipo de desarrollo.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- El depurador debe esperar a que se establezca una conexión antes de que la aplicación a depurar llame a la función **dbgConnect**. Para set el depurador en modo de escucha, utilice uno de los siguientes métodos: 

	- "Escuchar una conexión a través de dbgConnect" opción en el botón de prueba del proyecto (GO).

	- vaya a la pestaña "Proyecto", grupo "Modo de prueba", expanda "Modo de prueba" y seleccione "Escuchar una conexión a través de dbgConnect".




- Se utiliza el puerto 27280: se debe poder acceder a este puerto desde el equipo remoto.

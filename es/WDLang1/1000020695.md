


## InWindowsStoreAppMode (Función)

	<IMG style="float:left;margin-right:10px;" src="https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=warning_big.png"><B>Advertencia</B><BR>
	A partir de la versión <B>21</B>, esta funcionalidad se conserva por motivos de compatibilidad. Se recomienda utilizar la función [InUniversalWindowsPlatformMode](../WDLang1/1000021972.md).

<a name="Use"></a>
<a name="description"></a>
Indica si el código se ejecuta en modo Windows Store apps.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Code run in Windows Store apps? 
IF InWindowsStoreAppMode() = True THEN
	// Code specific to the Windows Store apps platform
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = InWindowsStoreAppMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecuta el código en modo Windows Store apps (en el equipo con Windows 8 en modo mosaico),

- <u><u><u><u>False</u></u></u></u> en caso contrario. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000181"></a>

La función **InWindowsStoreAppMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InWindowsStoreAppMode** pueden compartirse entre varios productos sin generar errores de compilación.
<a name="NOTE0_2"></a>


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000195"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.

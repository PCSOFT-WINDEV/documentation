


## &lt;Color de fondo (función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Define un fondo Color para una franja horaria en un control Agenda.

Atención: El color de fondo modificado no es el de Appointment sino el del período de tiempo especificado.  




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// October 12, 13 and 14, 2011 will be displayed on a red background
ORG_MyOrganizer.BackgroundColor("20111012", "20111015", iLightRed)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Organizer control>.BackgroundColor(<Start of time period> , <End of time period> , <Color>)`
---

**`<Organizer control>: (Nombre del control)`**

Nombre de la control Agenda que se va a utilizar. Este control puede corresponder a: 

- un control Agenda en una ventana,

- una control Agenda en una reporte.




**`<Start of time period>: (Cadena de caracteres o variable de tipo DateTime)`**

Fecha y hora de inicio del período de tiempo a Color. 

- Si el formato de fecha es 0000000000HHMMSS, todos los días se ven afectados.. 

- Si el formato de la fecha es AAAAMMDDD, todas las horas del día se ven afectadas..




**`<End of time period>: (Cadena de caracteres o variable de tipo DateTime)`**

Fecha y hora de finalización del período de tiempo a Color.

**`<Color>: (Integro)`**

Antecedentes Color de la época. Este color puede corresponder a:

- un color RGB (devuelto por la función [RGB](../WDLang1/3029012.md)),

- un color HSL (devuelto por la función [HSL](../WDLang1/3029057.md)),

- [un color predefinido de WLanguage](../WDLang5/3010002.md).




Observación:  Si este parámetro corresponde a -1 o a la constante *Transparent*, el color de fondo predeterminado se restaura.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
La granularidad del período de tiempo utilizando un fondo específico Color se limita a la granularidad del dibujo.


## Procedimiento WLanguage llamado por la función ConfirmAsynchronous



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Procedimiento ("Callback") llamado por la función [ConfirmAsynchronous](../WDLang1/1000025305.md) cuando el usuario hace clic en uno de los botones del cuadro de diálogo. Este procedimiento permite saber qué botón se pulsó, y ejecutar un proceso específico, si es necesario. 
Observación: Este procedimiento puede ser local, global, interno o lambda. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
ConfirmAsynchronous("Which button do you choose?", Procedure_Confirm)

INTERNAL PROCEDURE Procedure_Confirm(nResponse int)
	SWITCH nResponse
		CASE Yes: ToastDisplay("Yes button")
		CASE No: ToastDisplay("No button")
		CASE Cancel: ToastDisplay("Cancel button")
	END
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`ConfirmAsynchronous_Callback(<Selected button>)`
---

**`<Selected button>: (Entero o constante de tipo integer)`**

Identifica el botón seleccionado por el usuario:



|   |   |   |
| --- | --- | --- |
| *Cancel* | 2 | El usuario seleccionó el botón [CANCELAR]. |
| *No* | 0 | El usuario seleccionó el botón [NO]. |
| *Yes* | 1 | El usuario seleccionó el botón [SÍ]. |

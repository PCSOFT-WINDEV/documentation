


## fCreateLink( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Crea un enlace en un archivo en la ubicación especificada. Este enlace puede ser: 

- un enlace simbólico (archivo lnk). 

- un vínculo físico. Este enlace puede (o no) ser nombrado como el archivo fuente. El enlace físico se utiliza para hacer que el mismo archivo aparezca en diferentes ubicaciones. 




**Observaciones**: 

- Para eliminar un enlace simbólico o físico, utilice [fDelete](../WDLang1/3036029.md). 

- Para crear un acceso directo en el escritorio de Windows, en el menú "Inicio" de Windows o en un directorio específico, utilice [CreateShortcut](../WDLang1/3014008.md). 





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Create a symbolic link (lnk file)
fCreateLink("C:\temp\readme.txt", "C:\readme.lnk", fLinkSymbolic)

// Create a physical link
fCreateLink("C:\temp\readme.txt", "C:\readme.lnk", fLinkPhysical)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = fCreateLink(<Pointed file> , <Link destination> , <Type of link> [, <Command line> [, <Working directory>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se creó el enlace,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Pointed file>: (Cadena de caracteres)`**

ruta completa del archivo hacia el que apuntará el enlace.

**`<Link destination>: (Cadena de caracteres)`**

Ruta y nombre del fichero correspondiente al enlace a crear. Debe especificarse la extensión del enlace.

**`<Type of link>: (Constante de tipo Integer)`**

Tipo de enlace a crear: 


|   |   |
| --- | --- |
| *fLinkPhysical* | Crear un enlace físico. <br>Un enlace físico es un archivo al que se puede acceder a través de varias rutas. La eliminación de archivos sólo se puede realizar cuando se han eliminado todos los enlaces físicos..<br>**Atención**: &lt;Fichero puntual&gt; y &lt;Destino del enlace&gt; deben estar en el mismo disco. |
| *fLinkSymbolic* | Crear un enlace simbólico. <br>Un enlace simbólico es un "puntero" a un archivo. La eliminación del enlace simbólico no afecta al archivo apuntado. La eliminación del archivo apuntado hace que el enlace simbólico sea inválido.<br><br>![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Sólo el propietario del fichero puede crear un enlace simbólico.<br><br><br><br>![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) En Windows 10, para crear un enlace simbólico, el usuario debe tener privilegios de administrador en el equipo. |



**`<Command line>: (Cadena de caracteres opcional)`**

Comando Line del enlace. Este parámetro sólo se tiene en cuenta al crear un enlace simbólico.. 
![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Este parámetro no está disponible.

**`<Working directory>: (Cadena de caracteres opcional)`**

Directorio de trabajo del enlace. Este parámetro sólo se tiene en cuenta al crear un enlace simbólico..
![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Este parámetro no está disponible.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) En Windows, los enlaces sólo pueden ser creados en discos formateados en NTFS.

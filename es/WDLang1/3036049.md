
## fExeDir( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el directorio desde el que se inicia la ejecución.



**Atención**: La función **fExeDir** permite conocer el directorio de ejecución y la función [fCurrentDir](../WDLang1/3036022.md) permite conocer el directorio actual.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Runtime directory
Info("This application is started from " + fExeDir())
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = fExeDir()`
---

**`<Result>: (Cadena de caracteres)`**

Directorio desde el que se ejecuta el ejecutable. Esta ruta no termina con el carácter "\\".. 

- En modo de prueba (GO de WINDEV), corresponde a: 

	- el directorio de generación dado en la description del [configuración del proyecto](../Editeurs/9000030.md) actual. 

	- el subdirectorio EXE del directorio del proyecto. 




- En tiempo de ejecución, corresponde al directorio que contiene el ejecutable actual.

## Observaciones
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### SOAP server
<a name="soap_server_ELTPARAGRAPHE000114"></a>

Cuando se utiliza **fExeDir** en un servidor SOAP, devuelve el ruta del directorio que contiene la biblioteca del servidor SOAP que se está ejecutando.
<a name="NOTE0_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 

### Aplicación utilizada en Windows Vista (y posterior)
<a name="aplicacion_utilizada_windows_vista_posterior_ELTPARAGRAPHE000125"></a>

En Windows Vista (y posteriores), la [Mecanismo UAC](../Editeurs/2025014.md) implica algunas limitaciones de desarrollo.

Por lo tanto, los archivos utilizados por la aplicación (archivos de datos o archivos INI, por ejemplo) deben almacenarse en los directorios específicos de la aplicación (y no en el directorio del ejecutable).. Estos directorios son devueltos por [fDataDirCommon](../WDLang1/3036070.md), [fDataDirUser](../WDLang1/3036071.md), [fGlobalDirCommon](../WDLang1/3036072.md) y [fGlobalDirUser](../WDLang1/3036073.md).
<a name="NOTE0_3"></a>




### Ruta de un directorio de sistema
<a name="ruta_directorio_sistema_ELTPARAGRAPHE000150"></a>

Para conocer la ruta de un directorio del sistema, utilice [SysDir](../WDLang1/3073026.md).


## RegistryCreateKey( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Crea una clave en el registro de Windows.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003065002&name=registrycreatekey_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Create the "HKEY_LOCAL_MACHINE\SOFTWARE\MyApp" key
ResCreateKey = RegistryCreateKey("HKEY_LOCAL_MACHINE\SOFTWARE\MyApp")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = RegistryCreateKey([<Access mode>, ] <Key path>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se creó la clave,

- <u><u><u><u>False</u></u></u></u> en caso contrario. 


La clave no se modifica si ya existe: la función **RegistryCreateKey** devuelve <u><u><u><u>False</u></u></u></u> y la variable [ErrorOccurred](../WDLang1/3087001.md) se establece en <u><u><u><u>False</u></u></u></u>.

**`<Access mode>: (Constante de tipo Integer)`**

Modo de acceso al registro:



|   |   |
| --- | --- |
| *registryMode32* | Modo forzado para acceder al registro como un programa de 32 bits. |
| *registryMode64* | Modo forzado para acceder al registro como un programa de 64 bits. |
| *registryModeAuto*<br>(Valor predeterminado) | Modo de acceso al registro de automatic: <br><br>- una aplicación de 32 bits que se ejecuta en un sistema de 32 bits manipula el registro como un programa de 32 bits.<br><br>- una aplicación de 32 bits que se ejecuta en un sistema de 64 bits manipula el registro desde la siguiente rama:<br>	<br><pre><code>HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node</code></pre><br><br><br>- una aplicación de 64 bits que se ejecuta en un sistema de 64 bits manipula el registro como un programa de 64 bits.<br><br><br> |



**`<Key path>: (Cadena de caracteres)`**

ruta completa de clave para crear.

## Observaciones




### Creación de una clave
<a name="creacion_una_clave_ELTPARAGRAPHE000058"></a>

- La creación de claves es recursiva: si los directorios especificados en &lt;Key ruta&gt; no existen, se crean automáticamente en el registro.

- Sólo se pueden crear claves secundarias (no se puede crear ninguna clave primaria).



<a name="NOTE0_3"></a>




### Manejo del registro en Windows Vista (y posteriores)
<a name="manejo_del_registro_windows_vista_posteriores_ELTPARAGRAPHE000067"></a>

A partir de Windows Vista, si la [Mecanismo UAC](../Editeurs/2025014.md) está activada, debe tener derechos de administrador para escribir en la clave "HKEY_LOCAL_MACHINE\\SOFTWARE".

Si el usuario no tiene los derechos necesarios, el mecanismo UAC realiza una redirección a "HKEY_CURRENT_USER\\Software\\Classes\\VirtualStore\\MACHINE\\SOFTWARE".. El objetivo de esta redirección es garantizar la compatibilidad de las aplicaciones. Las claves escritas en este directorio pueden ser leídas por la aplicación que las ha escrito..  

**Recordatorio**: al crear el ejecutable, se puede incluir un manifiesto en el ejecutable para que se concedan derechos de administrador a la aplicación.

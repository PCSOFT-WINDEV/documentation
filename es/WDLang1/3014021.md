
## dbgEnableAssert( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Habilita o deshabilita la programación asertiva. La programación asertiva fue aplicada por [dbgAssert](../WDLang1/3014022.md).




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
dbgEnableAssert(True)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = dbgEnableAssert([<Activation>])`
---

**`<Result>: (booleano)`**

Si &lt;Activación&gt; no se especifica, &lt;Resultado> corresponde al comportamiento actual: 

- &lt;Resultado&gt; = <u><u><u><u>True</u></u></u></u> si se habilita la programación con aserciones,

- &lt;Resultado&gt; = <u><u><u><u>False</u></u></u></u> si se deshabilita la programación con aserciones. 




Si se especifica el parámetro &lt;Activación&gt;, &lt;Resultado&gt; corresponde al comportamiento anterior: 

- &lt;Resultado&gt; = <u><u><u><u>True</u></u></u></u> si se habilita la programación con aserciones,

- &lt;Resultado&gt; = <u><u><u><u>False</u></u></u></u> si se deshabilita la programación con aserciones. 




**`<Activation>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> para activar la programación con aserciones,

- <u><u><u><u>False</u></u></u></u> para desactivarla (valor predeterminado).






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- En el modo de prueba, la programación asertiva está habilitada por Default.

- En tiempo de ejecución, la programación asertiva es desactivada por Default.

- Si el modo JITc está activado (modo Default), esta función debe ser llamada en el código de inicialización del proyecto. Para obtener más información, consulte [JITc](../Editeurs/2020009.md).

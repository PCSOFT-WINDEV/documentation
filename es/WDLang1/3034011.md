
## ErrorThrow( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Lanzar un error no fatal. Este error puede ser procesado por el código de llamada si la gestión de los errores no fatales fue habilitada para este código (por la función [ventana de tratamiento de errores](../WDLang1/3034010.md) o [ErrorChangeParameter](../WDLang1/3034009.md)). 


<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`ErrorThrow(<Number> [, <Error message>])`
---

**`<Number>: (Integro)`**

Número de error, definido por el desarrollador. Este número debe incluirse entre 1 y 1000.

**`<Error message>: (Cadena de caracteres opcional)`**

Información adicional sobre el error.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Caso especial
<a name="caso_especial_ELTPARAGRAPHE000072"></a>

El mecanismo de tratamiento del error, independientemente de que se gestione de forma automática (por [ventana de tratamiento de errores](../WDLang1/3034010.md)) o a través de la programación (con [ErrorChangeParameter](../WDLang1/3034009.md)), no se activa si la función WLanguage o el usuario Procedure se encuentran en uno de los siguientes casos:


```txt
IF NOT <Function or procedure> THEN
	// Process the error
END
```



```txt
<Function or procedure>
IF ErrorOccurred = True THEN
	// Process the error
END
```


El mecanismo considera que el error es Process ed por la programación y el automatic Process no se activa.

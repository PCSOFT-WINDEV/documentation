
## DDEUnLink( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Interrumpe un enlace entre un elemento y un dato.

**Observación**: Romper un enlace "caliente" o "caliente" con un dato significa que el programa ya no será notificado por el servidor (programa conectado) cuando se modifiquen los datos de relacionado (la celda "L1C1" de la hoja de cálculo "Hoja 1" en el ejemplo siguiente).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyConnection = DDEConnect("Excel", "Worksheet1")
// Same for DDEWarmLink
DDELink(MyConnection, "L1C1", 1)
// Various operations
// ...
DDEUnLink(MyConnection, "L1C1", 1)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = DDEUnLink(<Connection identifier> , <Connection object> [, <Timeout>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se interrumpiera el enlace,

- <u><u><u><u>False</u></u></u></u> si se ha producido un error ([DDEError](../WDLang1/3028006.md) devuelve los detalles del error).




**`<Connection identifier>: (Integro)`**

Identificador de conexión, devuelto por la función [DDEConectar](../WDLang1/3028015.md).

**`<Connection object>: (Cadena de caracteres)`**

Objeto de conexión a manipular.

**`<Timeout>: (Entero o duración opcional)`**

Duración máxima de espera (en segundos). Si el comando no se ejecutaba:

- Si se especifica &lt;Timeout&gt;, hay un delay de &lt;Timeout&gt; segundos antes de salir de **DDEUnLink**.

- Si no se especifica el tiempo de espera, hay una espera interminable.... 


Observación: Este parámetro puede corresponder a: 

- un número entero correspondiente al número de segundos,

- una variable de tipo Duration,

- la duración en claro ('1s' por ejemplo).

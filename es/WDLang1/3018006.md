


## TreeCopyItem (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Copia un elemento (hoja o nodo) en un control TreeView. También se copian todas las sucursales emitidas desde este elemento.

Ejemplo de control TreeView:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Arbre_Vocabulaire.gif)

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Copy the "Pineapple" leaf into the "Starters" node
// in the "TREE_TVRecipe" control
Res = TreeCopyItem(TREE_RecipeTV, ...
	"Recipes" + TAB + "Desserts" + TAB + "Fruit salad", ...
	"Recipes" + TAB + "Starters", tvFirst)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = TreeCopyItem(<TreeView control> , <Source element> , <Destination element> [, <Sort mode>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el elemento fue copiado,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<TreeView control>: (Nombre del control)`**

Nombre de la control TreeView a utilizar.

**`<Source element>: (Cadena de caracteres)`**

Trayectoria del elemento (hoja o nudo) hasta copy. Este elemento y todas las ramas emitidas desde este elemento se moverán. Este parámetro tiene el siguiente formato:

```txt
"<Root name>" + TAB + ["<Name of 1st node>" + TAB + ...
["<Name of 2nd node>" + TAB + [...]]]"<Leaf name>"
```
En caso de duplicados en un elemento ruta, este ruta puede contener el identificador del elemento (especificado al final de su nombre por [TreeID](../WDLang1/3018027.md)).

**`<Destination element>: (Cadena de caracteres)`**

Ruta del elemento (hoja o nodo) correspondiente al nuevo nodo "padre" del elemento copiado. Este elemento debe existir. Este parámetro:

- tiene el siguiente formato:
	
	```txt
	"<Root name>" + TAB + ["<Name of 1st node>" + TAB + ..
	["<Name of 2nd node>" + TAB + [...]]]"<Leaf name>"
	```
En caso de duplicados en un elemento ruta, este ruta puede contener el identificador del elemento (especificado al final de su nombre por [TreeID](../WDLang1/3018027.md)).

- corresponde a la constante *NULL* si el nodo "padre" es la raíz del control TreeView.




**`<Sort mode>: (Constante o combinación de constantes opcional)`**

Posición del elemento copiado en relación con los elementos "hijos" del nodo de destino.


|   |   |
| --- | --- |
| *tvAcceptDuplicate* | Los duplicados son aceptados (dos elementos en el mismo nivel pueden tener el mismo nombre). El elemento copiado se inserta en orden alfabético, entre los elementos del mismo nivel. |
| *tvAlphaSort*<br>(Valor predeterminado) | El elemento copiado se clasifica en orden alfabético, entre los elementos del mismo nivel. |
| *tvFirst* | El elemento se copia en el primer lugar del nivel. |
| *tvLast* | El elemento se copia hasta el último lugar del nivel. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Copiar un elemento en sí mismo
<a name="copiar_elemento_mismo_ELTPARAGRAPHE000103"></a>

Un elemento no puede ser copiado en sí mismo o en uno de sus hijos. En este caso, **TreeCopyItem** devuelve <u><u><u><u>False</u></u></u></u> y el elemento no se copia.
<a name="NOTE0_2"></a>


### Duplicados
<a name="duplicados_ELTPARAGRAPHE000118"></a>

Pueden darse dos casos si el &lt;Elemento destino&gt; ya contiene un hijo con el mismo nombre que el &lt;Elemento fuente&gt;:

- Si se aceptan los duplicados (constante *tvAcceptDuplicate*), el copy se realiza como de costumbre. la función **TreeCopyItem** devuelve <u><u><u><u>True</u></u></u></u>.

- Si no se aceptan duplicados, **TreeCopyItem** devuelve <u><u><u><u>False</u></u></u></u>. El elemento no se copia.




Si el &lt;elemento fuente&gt; contiene duplicados de associated con el identificador, los duplicados se copiarán con el mismo identificador.
<a name="NOTE0_3"></a>


### Orden de prioridad para la clasificación
<a name="orden_prioridad_para_clasificacion_ELTPARAGRAPHE000144"></a>

El orden de prioridad de los parámetros para el modo Añadir es el siguiente:

- *tvFirst*

- *tvLast*

- *tvAlphaSort*




Por ejemplo, en *tvFirst* + *tvLast* + *tvAlphaSort*, sólo se tiene en cuenta y se ejecuta la constante *tvFirst*.

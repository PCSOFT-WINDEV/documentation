


## TableDisableFilter (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Desactiva un filtro de usuario en una columna o en todas las columnas que se encuentran en una Tabla o control Tabla TreeView.. Este filtro fue activado por el usuario o por [TableEnableFilter](../WDLang1/1000017149.md). 








<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Disables the filter on the Company column
TableDisableFilter(TABLE_Customer.COL_Company)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`TableDisableFilter(<Table control or Column>)`
---

**`<Table control or Column>: (Nombre del control)`**

Corresponde a:

- el nombre de la Tabla o control Tabla TreeView. 
	En este caso, todos los filtros de usuario especificados en la control están desactivados.

- el nombre de una columna que se encuentra en una Tabla o control Tabla TreeView. Este nombre tiene el siguiente formato: 
	```txt
	<Name of Table control>.<Column name>
	```

	Por ejemplo: TABLE_CustomerTable.COL_NombreDeLaCompañía. 
	En este caso, el filtro habilitado en esta columna está deshabilitado.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Ventajas de los filtros de usuario
<a name="ventajas_los_filtros_usuario_ELTPARAGRAPHE000091"></a>

The user filters can be modified by the end user (unlike a filter implemented by [HFilter](../WDLang4/3044100.md) or the [filtros](../Proprietes/2510042.md) Property ). 
<a name="NOTE0_2"></a>


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000106"></a>

Esta función está disponible en: 

- una Tabla o control Tabla TreeView basada en un archivo de datos.

- una Tabla o control Tabla TreeView rellenada programáticamente.

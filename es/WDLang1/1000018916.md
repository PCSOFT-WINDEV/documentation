
## fMemExist( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Comprueba la existencia de una zona de memoria compartida.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF fMemExist("ServiceDialogZone", ShareGlobal) = False THEN
	Error("Unable to dialog with the service. " + ...
		"Check whether the service was started.")
	RETURN
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = fMemExist(<Name of memory area> [, <Share>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si existe la zona de memoria compartida,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Name of memory area>: (Cadena de caracteres)`**

Nombre de la zona de memoria a crear o abrir. Este nombre debe ser el mismo en las diferentes aplicaciones que accedan a la zona.
La zona se crea automáticamente si no existe.

**`<Share>: (Constante opcional de tipo Integer)`**

Modo para compartir la zona de memoria entre las sesiones de usuario:


|   |   |
| --- | --- |
| *shareGlobal* | La zona se comparte entre todas las aplicaciones de todas las sesiones de la el equipo. |
| *shareUser*<br>(Valor predeterminado) | La zona se comparte entre todas las aplicaciones de la sesión de usuario. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Gestión del modo compartido
<a name="gestion_del_modo_compartido_ELTPARAGRAPHE000054"></a>

El modo compartido difiere según las versiones de los sistemas operativos:

- Linux, Windows 2000 y versiones anteriores: hay un solo espacio para crear la zona de memoria compartida, el parámetro &lt;Share&gt; es ignorado.

- Windows XP: el parámetro &lt;Share&gt; es efectivo si el servicio de cambio rápido de usuario está habilitado, de lo contrario es ignorado.

- Windows Vista y posterior: se gestiona el parámetro &lt;Share&gt;.. Los servicios y los usuarios están ubicados en un espacio diferente. Para compartir un área de memoria entre un servicio y una aplicación en la sesión de un usuario, debe utilizarse la constante *shareGlobal*.

- En Terminal Server: se gestiona el parámetro &lt;Share&gt;.. Cada sesión abierta en el Terminal Server tiene un espacio de memoria diferente. Las aplicaciones iniciadas en la misma sesión TSE pueden compartir un área de memoria creada con la constante *shareUser*. Las aplicaciones ubicadas en diferentes sesiones pueden compartir una zona de memoria creada con la constante *shareGlobal*.

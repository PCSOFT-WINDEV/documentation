
## ThreadWaitSignal (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Bloquea el hilo actual hasta que reciba una señal de otro hilo. Esta función simplifica la sincronización entre hilos sin tener que implementar una gestión avanzada de eventos. 

Observación: Esta función equivale a la función [EventWait](../WDLang1/3077007.md) utilizada con un evento automático, cuando la función [ThreadSendSignal](../WDLang1/3077021.md) envía una señal. Por lo tanto, el evento se cierra de forma automática.

**Recordatorio**: 

- Un hilo es un proceso que se ejecuta en paralelo con la aplicación actual (hilo principal). Esto le permite iniciar una tarea en segundo plano (hacer una copia de seguridad, etc.).

- Las funciones **ThreadWaitSignal** y [ThreadSendSignal](../WDLang1/3077021.md) permiten sincronizar los hilos de dos en dos.

<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003077022&name=threadwaitsignal_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Create an event
EventCreate("Let's go", eventManual, eventClose)
// Run a first thread
ThreadExecute("Thread1", threadNormal, "Thread_First")
// Locks the current thread until the event is obtained
ThreadWaitSignal()

// Run a second thread
ThreadExecute("Thread2", threadNormal, "Thread_Second")
// Locks the current thread until the event is obtained
ThreadWaitSignal()

// Modifies the event to run the threads
EventChange("Let's go", eventOpen)
```

```wl
//----------------------------------------------
PROCEDURE Thread_First()
Trace(dbgInfo(dbgProcess) + " - ThreadSendSignal(threadMain)")
ThreadSendSignal(threadMain)
EventWait("Let's go")
Trace(dbgInfo(dbgProcess) + " - Event received")
```

```wl
//--------------------------------------------
PROCEDURE Thread_Second()
Trace(dbgInfo(dbgProcess) + " - ThreadSendSignal(threadMain)")
ThreadSendSignal(threadMain)
EventWait("Let's go")
Trace(dbgInfo(dbgProcess) + " - Event received")
```

## Sintaxis

`<Result> = ThreadWaitSignal([<Maximum timeout>])`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si el hilo actual recibió una señal,

- <u><u><u><u>False</u></u></u></u> si el hilo no recibió ninguna señal (se agotó el tiempo de espera).




**`<Maximum timeout>: (Entero, duración o constante opcional)`**

Tiempo máximo de espera en centésimas de segundo. 

- Si al final del tiempo especificado, el hilo no ha recibido una señal, el valor de &lt;Resultado&gt; es <u><u><u><u>False</u></u></u></u>. 

- Si no se especifica este parámetro o corresponde a la constante *Infinite*, la espera es infinita.  


Este parámetro puede corresponder a: 

- un número entero que corresponde al número de centésimas de segundo,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, 1 s o 10 ms).

## Observaciones




### Espera infinita
<a name="espera_infinita_ELTPARAGRAPHE000123"></a>

La ejecución de un hilo se puede detener, incluso si está bloqueado en una espera infinita de un evento. Simplemente utilice [ThreadStop](../WDLang1/3077019.md).
<a name="NOTE0_2"></a>




### Sincronizar hilos
<a name="sincronizar_hilos_ELTPARAGRAPHE000134"></a>

Las funciones **ThreadWaitSignal** y [ThreadSendSignal](../WDLang1/3077021.md) permiten sincronizar los hilos de dos en dos. Para sincronizar varios hilos en el mismo evento, se deben utilizar las funciones de gestión de eventos:

- [EventCreate](../WDLang1/3077009.md): Crea un evento.

- [EventWait](../WDLang1/3077007.md): Bloquea el hilo actual y espera el evento.

- [EventChange](../WDLang1/3077017.md): Modifica el estado de un evento.

- [EventDestroy](../WDLang1/3077025.md): Destruye un evento.




Observación: En los mecanismos internos de gestión de colas y pilas, no es necesario sincronizar los hilos (funciones [Enqueue](../WDLang1/1000018323.md) y [Dequeue](../WDLang1/1000018322.md) para las colas, funciones [Push](../WDLang1/1000017366.md) y [Pop](../WDLang1/1000017372.md) para las pilas). 
<a name="NOTE0_3"></a>




### Varios
<a name="varios_ELTPARAGRAPHE000174"></a>

**ThreadWaitSignal** se puede utilizar en un proceso relacionado a un control (código de clic de un botón, por ejemplo): en este caso, se utiliza en el hilo principal.




## SetFocus (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Enfoque de set:

- a un control (que se encuentra en una ventana, en una página o en un frameset).

- una ventana.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Set focus on the EDT_Edit1 control
SetFocus(EDT_Edit1)
```


<a name="Example2"></a>

```wl
// Set focus on the EDT_Edit1 control in 
// the "Registration" page or window
SetFocus(Registration.EDT_Edit1)
```


<a name="Example3"></a>

```wl
// Set focus on MyControl 
// found in the LOOP_Customer Looper control.
// Select the second row
LOOP_Customer = 2
SetFocus(LOOP_Customer.MyControl)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Fijar el foco en una control en la ventana o página actual

`SetFocus([<Control name>])`
---

**`<Control name>: (Nombre de control opcional)`**

Nombre de la control (o de la columna) que se va a centrar (ver "Observaciones"). Si no se especifica este parámetro, la control actual gana protagonismo. 

Un error de WLanguage ocurre si &lt;control nombre&gt; no corresponde a un control encontrado en la ventana actual o en la página actual.
![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) No se puede enfocar la set en una control Imagen.

### Fijar el foco en una ventana

`SetFocus([<Window>])`
---

**`<Window>: (Nombre de la ventana opcional)`**

Nombre de la ventana que obtendrá el foco. Esta ventana se habilitará y el control que obtenga el foco en esta ventana corresponderá:

- la última control previamente en edición en esta ventana.

- la primera control que puede obtener el foco en esta ventana (si no se han introducido datos previamente).




Si no se especifica este parámetro, la control actual gana protagonismo.

## Observaciones


### Obtención de foco
<a name="obtencion_foco_ELTPARAGRAPHE000097"></a>

Tomar el "enfoque" se representa de diferentes maneras:

- Cursor intermitente (o "Caret") para los controles de edición.

- Rectángulo de puntos para los otros tipos de controles (Botones, List Boxes, Botones de radio, etc.).




**Atención**: Algunos tipos de controles no pueden enfocarse (por ejemplo, los controles"estáticos").
<a name="NOTE0_2"></a>


### ScreenFirst, SetFocus y SetFocusAndReturnToUserInput
<a name="screenfirst_docparampagetitleshort_setfocusandreturntouserinput_ELTPARAGRAPHE000109"></a>

Estas funciones tienen un comportamiento diferente:

- [SetFocusAndReturnToUserInput](../WDLang1/1410088107.md) detiene el process actual para forzar la entrada en el control especificado. 

- [ScreenFirst](../WDLang1/3025039.md) define qué control recibirá el foco de entrada (cuando el código actual se detiene y el foco de entrada se set de nuevo a un control).

- **SetFocus** detiene el process actual, define qué control recibirá el foco de entrada y reanuda el process actual.

### SetFocus en una columna control Tabla
<a name="docparampagetitleshort_una_columna_control_tabla_ELTPARAGRAPHE000130"></a>

**SetFocus** fija el foco en la columna especificada. Si la control Tabla no recibe ninguna entrada, se convierte en editable.

### SetFocus en la actual control
<a name="docparampagetitleshort_actual_control_ELTPARAGRAPHE000142"></a>

**SetFocus** utilizado sin parámetros pone el foco en el control actual. Esta función, llamada en el código de salida de control Campo de entrada, se utiliza para forzar la entrada en el control seleccionado si la información tecleada no corresponde a la información esperada.
<a name="NOTE0_6"></a>
![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) 

### SetFocus en el último control seleccionado
<a name="docparampagetitleshort_ultimo_control_seleccionado_ELTPARAGRAPHE000154"></a>

**SetFocus** utilizado sin parámetros pone el foco en el control actual. Esta función, llamada en el código de salida de control Campo de entrada, se utiliza para forzar la entrada en el control seleccionado si la información tecleada no corresponde a la información esperada.

El tipo control debe ser:

- control Campo de entrada.

- Control Casilla de verificación.

- Control Botón de opción.

- Control List Box.

- Control Combo Box.

- Haga clic en el área control.

### Procesos ejecutados cuando se llama a SetFocus
<a name="procesos_ejecutados_cuando_llama_docparampagetitleshort_ELTPARAGRAPHE000175"></a>

En la mayoría de los casos, la salida Process (del control o ventana actual) y la entrada Process (en el control o ventana especificada) se ejecutan cuando se llama a **SetFocus**.

Excepciones:

- No se ejecuta ningún código si &lt;nombre de control&gt; es el actual control.

- El código de salida de la control seleccionada no se ejecuta si se utiliza **SetFocus** en uno de los eventos de WLanguage associated con esta control.

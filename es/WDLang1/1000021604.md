


## &lt;ParallelTask Variable&gt;.Espera (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Espera al final de la ejecución de una tarea paralela..

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Triggers the execution of a task and waits for the task to be ended
t is ParallelTask = ParallelTaskExecute(Proc)
t.Wait()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = <Task>.esperar([<Maximum duration>])`
---

**`<Result>: (Boolean)`**



- <u><u><u><u>True</u></u></u></u> si la tarea finaliza correctamente. 

- <u><u><u><u>False</u></u></u></u> si la tarea no finaliza correctamente. Por ejemplo:

	- se ha alcanzado la duración máxima de la tarea,

	- se canceló la tarea.







**`<Task>: (ParallelTask variable)`**

Nombre de la variable de tipo [ParallelTask](../WDLang1/1000021220.md) que se utilizará.

**`<Maximum duration>: (Optional integer or optional Duration)`**

Duración máxima para ejecutar la tarea, expresada en centésimas de segundo.

- Si se especifica este parámetro, la función **&lt;ParallelTask Variable&gt;.Espera** bloquea la aplicación hasta que finalice la ejecución de la tarea o hasta que se alcance el tiempo de espera máximo. 

- Si no se especifica este parámetro, la función **&lt;ParallelTask Variable&gt;.Espera** bloquea la aplicación hasta que finalice la ejecución de la tarea.


Este parámetro puede corresponder a: 

- un número entero que corresponde al número de centésimas de segundo,

- una variable de tipo Duration,

- la indicación de la duración en claro (por ejemplo, "1 s" o "10 ms"). 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Si se llama a **&lt;ParallelTask Variable&gt;.Espera** desde la principal Thread, la tarea a esperar no debe ejecutarse en la principal Thread. De lo contrario, se producirá un bloqueo. En efecto, la Thread principal no puede esperar el fin de la tarea y ejecutarla al mismo tiempo. 

Un error de WLanguage señala el problema antes de que ocurra el bloqueo..




## &lt;Treeview&gt;.ItemStatus



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el status de un nodo.: colapsado, expandido o no encontrado.

Ejemplo de control TreeView: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Arbre_Vocabulaire.gif)

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Status of the "Desserts" node in the "TREE_TVRecipe" TreeView control 
ResStatus = TREE_TVRecipe.ItemStatus("Recipes" + TAB + "Desserts")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <TreeView control>.ItemStatus(<Node>)`
---

**`<Result>: (Constante de tipo Integer)`**

Estado del nodo.


|   |   |
| --- | --- |
| *tvCollapse* | Nodo colapsado. |
| *tvError* | Nodo no encontrado. |
| *tvExpand* | Nodo desglosado. |



**`<TreeView control>: (Nombre del control)`**

Nombre de la control TreeView a utilizar.

**`<Node>: (Cadena de caracteres)`**

Ruta del nodo a utilizar. Este parámetro tiene el siguiente formato:

```txt
"<Root name>" + TAB + ["<Name of 1st node>" + TAB + ...
["<Name of 2nd node>" + TAB + [...]]]"<Leaf name>"
```
Si hay dos elementos con el mismo nombre en la ruta, puede añadir los ID de los elementos (después de sus nombres utilizando [TreeID](../WDLang1/3018027.md)).



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Un nodo colapsado o expandido puede ser Visible o no
<a name="nodo_colapsado_expandido_puede_ser_visible_ELTPARAGRAPHE000192"></a>

El status devuelto corresponde al status interno del elemento. Un elemento expandido no es necesariamente Visible porque uno de sus padres puede estar colapsado.

Por ejemplo: Si el nodo "Recetas" + TAB + "Postres" está colapsado, el nodo "Recetas" + TAB + "Postres" + TAB + "Helados" puede ser colapsado o expandido.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ArbreEtat.gif)


## fDateTime( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve o modifica las diferentes fechas y horas de associated con un archivo o directorio (fecha de creación, modificación o acceso).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003036066&name=fdatetime_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
ResDate = fDateTime("C:\MyDirectories\File.txt", "", fModify)
```

## Sintaxis

`<Result> = fDateTime(<Path of file / directory> [, <New date and time> [, <Type of date and time>]])`
---

**`<Result>: (Cadena de caracteres)`**



- Fecha y hora associated con el archivo o directorio (en formato AAAAMMDDHHMMSS),

- Cadena vacía ("") si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Path of file / directory>: (Cadena de caracteres)`**



- Nombre y ruta completa (o relativa) del archivo (hasta 260 caracteres). 

- Ruta completa (o relativa) del directorio (hasta 260 caracteres). Si este parámetro corresponde al nombre de un directorio, este nombre puede (o no) terminar con "\\". Se puede utilizar una ruta de tipo UNC.










**`<New date and time>: (Cadena de caracteres opcional)`**



- Modificar una fecha y hora: nueva fecha y hora del archivo o directorio (en formato AAAAMMDDHHmmSS).

- Leer una fecha y hora: cadena vacía ("").




**`<Type of date and time>: (Constante opcional)`**

Especifica la fecha y la hora de uso:



|   |   |
| --- | --- |
| *fAccess* | Fecha y hora del último acceso al fichero o directorio.<br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta constante no está disponible. |
| *fCreation* | Fecha y hora de creación del archivo/directorio.<br><br>![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) La fecha de creación de un archivo o directorio no se puede modificar.<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Esta constante no está disponible. |
| *fModify*<br>(Valor predeterminado) | Fecha y hora de la última modificación realizada en el fichero/directorio.<br><br> |


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Sólo se pueden modificar los ficheros con fecha posterior al 1 de enero de 1970.

## Observaciones




### Equivalencia y velocidad de ejecución
<a name="equivalencia_velocidad_ejecucion_ELTPARAGRAPHE000141"></a>

La función **fDateTime** es equivalente a las funciones [fDate](../WDLang1/3036019.md) y [fTime](../WDLang1/3036042.md) utilizadas sucesivamente. Sin embargo, la función **fDateTime** es más rápida.
<a name="NOTE0_2"></a>




### Gestión de errores
<a name="gestion_errores_ELTPARAGRAPHE000159"></a>

La función **fDateTime** genera un error en los siguientes casos:

- el archivo especificado se abre o el archivo está bloqueado por otra aplicación,

- el archivo o directorio especificado no existe,

- el nombre del archivo especificado no es válido,

- el año de una de las fechas de archivo es anterior a 1980,

- el formato de &lt;Nueva fecha y hora&gt; es incorrecto.

- ...



<a name="NOTE0_3"></a>




### Modo de funcionamiento en los diferentes sistemas operativos
<a name="modo_funcionamiento_los_diferentes_sistemas_operativos_ELTPARAGRAPHE000176"></a>

La fecha y la hora de un archivo o directorio no se pueden modificar en los siguientes sistemas operativos: Windows 98, Windows ME.

![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) La fecha y hora de creación de un archivo o directorio no se pueden modificar.

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Sólo se pueden modificar los ficheros con fecha posterior al 1 de enero de 1970.
<a name="NOTE0_4"></a>

### Modo de funcionamiento en Windows Vista (y posterior)
<a name="modo_funcionamiento_windows_vista_posterior_ELTPARAGRAPHE000332"></a>

Si esta función no funciona correctamente en Windows Vista (y posterior), compruebe si el archivo o directorio utilizado no está en ninguno de los directorios del sistema (directorio de Windows o directorio "Archivos de programa").

**En Windows Vista (y posteriores), con el [Mecanismo UAC](../Editeurs/2025014.md) (Cuenta de usuario control) habilitado**, debe tener privilegios de administrador para manipular y/o modificar los archivos o directorios en los directorios del sistema (directorio de Windows o directorio "Archivos de programa").

**Consejo de programación**: A manipular y/o modificar los archivos o directorios sin privilegios de administrador, debería:

- evite escribir en el directorio de Windows o en el de "Archivos de programa",

- utilizar el directorio del sistema de la aplicación (devuelto por [SysDir](../WDLang1/3073026.md) con la constante *srAppDataCommun*, por ejemplo).




## FileToMemory( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicializa automáticamente los valores de los miembros encontrados en una clase o en una estructura con los valores de los elementos del actual Record encontrados en un archivo de datos.

También tienes la capacidad de inicializar los valores de los miembros avanzados a partir de los valores de los artículos del actual Record en un archivo de datos.

**Observación**: **FileToMemory** no lee registros en el archivo de datos. El valor de la Record cargada en la memoria se utiliza para inicializar los valores de los miembros. Por lo tanto, un Record debe ser cargado en memoria previamente (con [HRead](../WDLang4/3044047.md), [HReadSeek](../WDLang4/3044050.md), [HReadFirst](../WDLang4/3044051.md), etc.).

En WEBDEV, también tiene la posibilidad de utilizar [ArchivoPágina](../WDLang4/3044148.md).


<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`FileToMemory(<Name of instance of structure or class> , <File name>)`
---

**`<Name of instance of structure or class>: (Estructura o Clase Variable)`**

Nombre de la instancia de la clase o estructura en la que los miembros deben ser inicializados con los valores del actual Record en el archivo de datos. El nombre de la clase o estructura Variable debe ser especificado. 
Observación: Si se llama esta función en una instancia de clase, se debe utilizar la palabra clave "object" para recuperar la instancia actual.

**`<File name>: (Cadena de caracteres)`**

Nombre del archivo de datos al que pertenecen los artículos de relacionado.

<a name="NOTE0_1"></a>

## Observaciones


### Modo de funcionamiento
<a name="modo_funcionamiento_ELTPARAGRAPHE000093"></a>

- Si un miembro de la estructura o clase se nombra como un elemento del archivo de datos HFSQL, el valor del elemento en memoria se copia en el valor del miembro. El nombre del miembro no debe contener ningún prefijo. El caso y los caracteres acentuados se ignoran al comparar el nombre de los elementos y los nombres de los miembros.

- Si un miembro de la estructura o clase no tiene un elemento con el mismo nombre en el archivo de datos HFSQL, su valor no se modifica. Puede utilizar [VariableReset](../WDLang1/3013069.md) para inicializar el valor de los miembros antes de la llamada a **FileToMemory**.





### Array artículos del archivo de datos: cambio de comportamiento
<a name="array_articulos_del_archivo_datos_cambio_comportamiento_ELTPARAGRAPHE000104"></a>

- En la versión 15, si el WLanguage array contenía N elementos (siendo N menor que el tamaño del array de HFSQL), **FileToMemory** copiaba los primeros N elementos del elemento Array del archivo de datos HFSQL. El WLanguage array no se amplió. 

- A partir de la versión 16, al final de la operación, el WLanguage array y el array de los archivos de datos HFSQL contienen el mismo número de elementos.

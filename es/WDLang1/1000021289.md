


## DashMoveWidget (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cambia la posición de un widget que se encuentra en un control Cuadro de mando. Este widget ya está configurado o mostrado. 

Recordatorio: Un widget corresponde a: 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) una ventana interna.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Moves the current widget to the top of Dashboard control
DashMoveWidget(1, 1)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Moviendo el widget actual

`DashMoveWidget(<X> , <Y> [, <Options>])`
---

**`<X>: (Integro)`**

Posición horizontal del widget actual en número de celdas o en píxeles. La primera celda es la celda [ 1, 1].
 El valor en píxeles se redondea para corresponder a un número entero de celdas.

El widget utilizado corresponde a la control a la que pertenece el Event actual (Widget o control perteneciente al widget). 

**`<Y>: (Integro)`**

Posición vertical del widget actual en número de celdas o en píxeles. La primera celda es la celda [ 1, 1]. 
El valor en píxeles se redondea para corresponder a un número entero de celdas.

El widget utilizado corresponde a la control a la que pertenece el Event actual (Widget o control perteneciente al widget). 

**`<Options>: (Constante opcional de tipo Integer)`**

Permite definir la unidad utilizada y el modo de reorganización.: 


|   |   |
| --- | --- |
| *dashNoReorganization* | Mueve el Widget sin reorganizar los widgets adyacentes.<br><br>Atención: El uso de la constante *dashNoReorganization* puede producir una visualización incorrecta (con widgets superpuestos). Al utilizar esta constante, es importante volver a organizar completamente el control Cuadro de mando con  las funciones [DashResizeWidget](../WDLang1/1000021286.md) y **DashMoveWidget**. |
| *dashUnitCell*<br>(Valor predeterminado) | La unidad utilizada al mover el widget es un número de celdas. |
| *dashUnitPixel* | La unidad utilizada para mover el widget es el píxel. |




<a name="SYNTAX2"></a>

### Mover un widget identificado por su índice

`DashMoveWidget(<Dashboard control> , <Widget index> , <X> , <Y> [, <Options>])`
---

**`<Dashboard control>: (Nombre del control)`**

Nombre de la control Cuadro de mando a utilizar.

**`<Widget index>: (Integro)`**

Índice del widget a utilizar. Este índice corresponde a:

- el número de adición del widget definido en el editor de ventanas o en el editor Page.

- el número de adición del widget devuelto por [DashAddWidget](../WDLang1/1000020953.md).




**`<X>: (Integro)`**

Posición horizontal del widget en número de celdas o en píxeles. La primera celda es la celda [ 1, 1]. El valor en píxeles se redondea para corresponder a un número entero de celdas.

**`<Y>: (Integro)`**

Posición vertical del widget en número de celdas o en píxeles. La primera celda es la celda [ 1, 1]. El valor en píxeles se redondea para corresponder a un número entero de celdas.

**`<Options>: (Constante opcional de tipo Integer)`**

Permite definir la unidad utilizada y el modo de reorganización.: 


|   |   |
| --- | --- |
| *dashNoReorganization* | Mueve el Widget sin reorganizar los widgets adyacentes.<br><br>Atención: El uso de la constante *dashNoReorganization* puede producir una visualización incorrecta (con widgets superpuestos). Al utilizar esta constante, es importante volver a organizar completamente el control Cuadro de mando con  las funciones [DashResizeWidget](../WDLang1/1000021286.md) y **DashMoveWidget**. |
| *dashUnitCell*<br>(Valor predeterminado) | La unidad utilizada al mover el widget es un número de celdas. |
| *dashUnitPixel* | La unidad utilizada para mover el widget es el píxel. |

## Observaciones


### Regla utilizada para calcular celdas
<a name="regla_utilizada_para_calcular_celdas_ELTPARAGRAPHE000178"></a>

Veamos algunos ejemplos para un widget cuyo tamaño es igual a 100 px por 100 px con un margen set a 10 px: 


```wl
DashMoveWidget(DASH_TimeZones, 1, 80, 80, dashUnitPixel)
// The widget will be positioned in cell 1 x 1

DashMoveWidget(DASH_TimeZones, 1, 160, 200, dashUnitPixel)
// The widget will be positioned in cell 2 x 2

DashMoveWidget(DASH_TimeZones, 1, 211, 211, dashUnitPixel)
// The widget will be positioned in cell 3 x 3

DashMoveWidget(DASH_TimeZones, 1, 400, 500, dashUnitPixel)
// The widget will be positioned in cell 4 x 5
```

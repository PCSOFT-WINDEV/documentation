
## Definir conjuntos de caracteres específicos en archivos de datos HFSQL
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000165"></a>
Cuando se utiliza una aplicación para mostrar o guardar datos en archivos HFSQL, estos datos se introducen en el idioma actual.

Si la aplicación se utiliza en un país que utiliza un conjunto de caracteres latinos, no se requiere ninguna configuración específica.

Si la aplicación se utiliza en un país que utiliza un conjunto de caracteres específico, este conjunto de caracteres también debe ser compatible con los archivos de datos. Por lo tanto, la mayoría de las operaciones (búsquedas, clasificaciones, ...) se realizarán de acuerdo con el personaje seleccionado set.



<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Definiendo el personaje set de un archivo de datos
<a name="definiendo_personaje_set_archivo_datos_ELTTEXTE000195"></a>


### Definiendo el carácter set utilizado por un archivo de datos
<a name="definiendo_caracter_set_utilizado_por_archivo_datos_ELTPARAGRAPHE000024"></a>

Para definir el carácter set utilizado por un archivo de datos: 

1. Define el carácter set con [ChangeCharset](../WDLang1/3054001.md).

2. Crear el archivo de datos con [HCreation](../WDLang4/3044255.md) (o [HCreationIfNotFound](../WDLang4/3044254.md)). En estas funciones, utilice la constante *hChangeCharset* para tener en cuenta el carácter set especificado por [ChangeCharset](../WDLang1/3054001.md).




Todas las funciones utilizadas para realizar las clasificaciones y búsquedas utilizarán las características específicas del personaje seleccionado set.
<a name="NOTE2_2"></a>


### Notas y limitaciones
<a name="notas_limitaciones_ELTPARAGRAPHE000048"></a>

- El tamaño de los elementos de cadena especificados en el análisis se expresa en bytes y no en caracteres.. Dependiendo del carácter set utilizado, es posible que tenga que aumentar el tamaño del artículo (si los caracteres utilizados están codificados en 2 bytes, por ejemplo).

- El carácter set del archivo de datos debe ser siempre idéntico al carácter set especificado por [ChangeCharset](../WDLang1/3054001.md). De lo contrario, al acceder a un índice de cadena (cadena, carácter, fecha, hora):

	- Leyendo y buscando una Record: no se mostrará ningún error. Sin embargo, los datos leídos no corresponderán a los datos esperados.

	- Modificación de una Record (adición, modificación, supresión de una Record o Record cruzada): se mostrará un error de WLanguage.




- No se puede realizar ninguna consulta con uniones en elementos de texto que utilicen diferentes conjuntos de caracteres.

- El orden de clasificación en los índices HFSQL ignora las letras dobles (por ejemplo, "CH" en español o "LJ" en croata).

- ![HFSQL Client/Server](https://doc.pcsoft.fr/ext/images/us/HFCS.png) En HFSQL Client/Server, sólo se admite el orden de clasificación de los siguientes idiomas: Francés, árabe, chino, coreano, estonio, griego, hebreo, japonés, polaco, ruso, tailandés, turco y vietnamita.

### Ejemplo: Creando un archivo de datos en hebreo
<a name="ejemplo_creando_archivo_datos_hebreo_ELTPARAGRAPHE000067"></a>


```wl
// Language used in the application: Hebrew
Nation(nationHebrew)
// Change the character set
ChangeCharset(charsetHebrew)
// Create the data file 
// and take the change of character set into account
HCreation(Azaa, "", "", "", hChangeCharset)
// Enter a record
Azaa.toto = "ñéâëðéçìòëðâëêðéò"
HAdd(Azaa)
```

## Cambiando el personaje set de un archivo de datos
<a name="cambiando_personaje_set_archivo_datos_ELTTEXTE000231"></a>
En algunos casos, puede querer cambiar el carácter set de un archivo de datos sin volver a crear este archivo. Por ejemplo:

- un archivo de datos fue traducida.

- un archivo de datos se rellena al instalar una aplicación con los datos correspondientes al lenguaje instalación.

- ...



<a name="NOTE3_2"></a>


### Modificando el carácter set utilizado para las búsquedas y para las clases
<a name="modificando_caracter_set_utilizado_para_las_busquedas_para_las_clases_ELTPARAGRAPHE000085"></a>

Modificar el carácter set utilizado para las búsquedas y para los tipos: 

1. Especifique el carácter set a utilizar con [ChangeCharset](../WDLang1/3054001.md).

2. Reindizar el archivo de datos con [HIndex](../WDLang4/3044133.md). Utilice la constante *hChangeCharset* para utilizar el carácter set especificado por [ChangeCharset](../WDLang1/3054001.md).

## Descubrir el personaje set de un archivo de datos
<a name="descubrir_personaje_set_archivo_datos_ELTTEXTE000261"></a>


### ¿Cómo proceder?
<a name="¿como_proceder_ELTPARAGRAPHE000106"></a>

Para identificar el juego de caracteres utilizado por un archivo de datos, utilice el [Charset](../Proprietes/2512008.md) Property en el archivo de datos HFSQL deseado.
<a name="NOTE4_2"></a>


### Ejemplo: Cambiar las fuentes según el carácter set del archivo de datos leído
<a name="ejemplo_cambiar_las_fuentes_segun_caracter_set_del_archivo_datos_leido_ELTPARAGRAPHE000116"></a>


```wl
HOpen(Customer)
FileLanguage is int = Customer.Charset
// Changes the character set according to the data file format
SWITCH FileLanguage
	CASE charsetArabic: ChangeCharset(charsetArabic)
	CASE charsetDefault: ChangeCharset(charsetDefault)
	CASE charsetGreek: ChangeCharset(charsetGreek)
	CASE charsetHebrew: ChangeCharset(charsetHebrew)
	CASE charsetOccidental: ChangeCharset(charsetOccidental)
	CASE charsetRussian: ChangeCharset(charsetRussian)
	CASE charsetTurkish: ChangeCharset(charsetTurkish)
	OTHER CASE
		// Switch to western character set
	  	ChangeCharset(charsetOccidental)
END
```




## Traducir la interfaz a varios idiomas
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000282"></a>
Cuando los diferentes idiomas soportados por la aplicación han sido definidos en las características del proyecto, la información mostrada por la aplicación debe ser traducida a estos diferentes idiomas.

Para traducir la interfaz a varios idiomas, se deben tener en cuenta varios elementos:

- [El modo de traducción seleccionado](#NOTE2_1).

- [Los idiomas soportados (idiomas que utilizan o no un juego de caracteres específico)](#NOTE3_1). 

- [El uso de ventanas o páginas específicas](#NOTE4_1)(Groupware de usuario, Gestión automática de errores de HyperFileSQL, Vista previa de impresión).

- [Los mensajes que se muestran por programación](../Editeurs/2013004.md).






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Modo de traducción seleccionado
<a name="modo_traduccion_seleccionado_ELTTEXTE000312"></a>
Existen varios modos de traducción disponibles:

- **Traducción automática de las interfaces** a través de una herramienta de traducción específica que se encuentra en el ordenador de desarrollo. Esta traducción se realiza en el editor de WINDEV, WEBDEV o WINDEV Mobile.

- **Comprobación y comprobación de la información a traducir** con WDMSG.



<a name="NOTE2_2"></a>


### Traducción automática de las interfaces
<a name="traduccion_automatica_las_interfaces_ELTPARAGRAPHE000043"></a>

Para cada objeto, se muestran diferentes áreas multilingües en la ventana de descripción. Estas áreas multilingües le permiten introducir información en los diferentes idiomas soportados por el proyecto.

Traducir esta información de los diferentes editores:

1. Mostrar las opciones de WINDEV, WEBDEV o WINDEV Mobile: En la pestaña "", en el grupo "", despliegue "" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile".

2. Vaya a la pestaña "Traducción". Esta ficha se utiliza para definir: 

	- la herramienta de traducción por defecto (así como el modo de transmisión del texto a traducir y del texto traducido). 

	- los idiomas de origen y de destino de la traducción.




3. Si se especifica una herramienta de traducción, se mostrará un botón "Traducir" ( ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=BtnTraduire.GIF) ) en cada elemento que contenga información para traducir. Este botón se utiliza para traducir el texto seleccionado a través de la herramienta de traducción especificada.




Para obtener más información, consulte: 

- [Opciones de traducción en los editores](../Editeurs/2036001.md).

- [Introducción de información multilingüe](../WDLang1/1000020822.md).



<a name="NOTE2_3"></a>


### Comprobación y comprobación de la información a traducir
<a name="comprobacion_comprobacion_informacion_traducir_ELTPARAGRAPHE000084"></a>

**WDMSG** (herramienta independiente) se utiliza para comprobar todos los mensajes de un proyecto (subtítulos de los controles, ....) y para volver a comprobarlos una vez traducidos.. Para obtener más información, consulte [la ayuda sobre WDMSG](../WDMsg/3518040.md).

Contacte al Departamento de Ventas de PC SOFT para más detalles sobre las condiciones de uso de este producto.

## Idiomas que utilizan un conjunto de caracteres específico
<a name="idiomas_que_utilizan_conjunto_caracteres_especifico_ELTTEXTE000348"></a>
Si su aplicación admite idiomas que utilizan conjuntos de caracteres específicos (griego, ruso, etc.), debe introducir la conversión de los mensajes multilingües utilizando estos conjuntos de caracteres específicos.

WINDEV, WEBDEV y WINDEV Mobile le permiten administrar automáticamente el uso de conjuntos de caracteres específicos en el editor.

De hecho, tan pronto como el cursor de edición del ratón se encuentra en un área de entrada de un idioma que utiliza un conjunto de caracteres específico, se muestra automáticamente el idioma de entrada correspondiente (conjunto de caracteres utilizado por el teclado).

Por lo tanto, si se introduce un texto en la zona "Ruso" de la ventana de descripción de un control, el juego de caracteres ruso será utilizado automáticamente por el teclado.

Recordatorio: Para utilizar conjuntos de caracteres específicos, los archivos correspondientes a los conjuntos de caracteres deseados deben estar instalados en los ajustes de la región de Windows (panel de control).

Por ejemplo:

- escribir la leyenda de una control Campo de entrada: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=AlphabetLangue.gif)


- introducir un mensaje multilingüe en el editor de código: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=AlphabetMessCode.gif)





**Observación**: Si no desea activar automáticamente el idioma de entrada (idioma utilizado por el teclado) correspondiente al idioma actual:

1. Vaya a la pestaña "Traducción" en las opciones (en el panel "", en el grupo "", expanda "" y seleccione "Opciones generales de WINDEV/WEBDEV/WINDEV Mobile").

2. Desmarque la opción "Activar el cambio automático de la configuración regional de entrada".

### Gestión de los errores de HFSQL
<a name="gestion_los_errores_hfsql_ELTPARAGRAPHE000139"></a>

Por defecto, las ventanas y páginas para la gestión de errores HFSQL se suministran en inglés y francés. Para traducirlos a otro idioma, debe hacerlo:

1. Incluya las ventanas/páginas de error predeterminadas en su proyecto.

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Estas ventanas se suministran (junto con su código WLanguage) en \\Programs\\Data\\Preset windows\\HFSQL - Windows para ayuda automática.




2. Personalizar la gestión de los errores con [HOnError](../WDLang4/3044017.md). Esta personalización consiste en utilizar las ventanas/páginas de ayuda automática que se incluyeron en el proyecto.

3. Traducir los diferentes mensajes (ver párrafo anterior).



<a name="NOTE4_2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Gestión automática de errores
<a name="gestion_automatica_errores_ELTPARAGRAPHE000162"></a>

Por defecto, las ventanas para la gestión automática de los errores se suministran en inglés y francés.. Para traducirlos a otro idioma, debe hacerlo:

1. Incluir las ventanas de error por defecto en su aplicación. Estas ventanas se suministran (así como su código WLanguage) en \\Programs\\Data\\Preset windows\\Assertion - Error - Exception.

2. Traducir los diferentes mensajes (ver párrafo anterior).


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Estas ventanas también están disponibles en aplicaciones Java.

### Groupware usuario
<a name="groupware_usuario_ELTPARAGRAPHE000178"></a>

Por defecto, el groupware usuario se suministra en inglés y en francés.

**Para traducir una aplicación o un sitio que utiliza el groupware usuario**, usted debe elegir el modo "Custom user groupware" cuando el groupware usuario se incluye en su aplicación. Recordatorio: Para integrar el groupware usuario a la aplicación, en la pestaña "", en el grupo "", haga clic en "".

Las diferentes ventanas/páginas de el groupware usuario serán incluidas en su proyecto.

Luego, todo lo que tienes que hacer es traducir estas ventanas/páginas.

**Observación**: Desmarque "AutoRun" si es necesario: esto le permite mostrar una ventana o página para la selección del idioma antes de iniciar la ventana/página de inicio de sesión.

Para obtener más información, consulte: [Característica multilingüe y groupware de usuario](../Editeurs/2018008.md).

### Visor de reportes
<a name="visor_reportes_ELTPARAGRAPHE000212"></a>

Por defecto, la ventana del visor de reporte está disponible en inglés y en francés.

Para traducir el visor de reporte, las ventanas correspondientes deben estar integradas en su aplicación. Estas ventanas se suministran (así como su código WLanguage) en \\Programas\\Datos\\Ventanas predefinidas\\Vista previa. Para obtener más información, consulte [Personalización del viewer de reporte](../WDChamp/2024002.md).

Entonces, todo lo que tienes que hacer es traducir estas ventanas.

### Menús del sistema/Menú desplegable de AAF
<a name="menus_del_sistemamenu_desplegable_aaf_ELTPARAGRAPHE000228"></a>

WINDEV propone un menú emergente por defecto para algunos controles. Este menú permite al usuario realizar fácilmente varias operaciones.

**Para traducir estos menús**debe tener WDINT. Para obtener más información, consulte [Configuración de los menús de los AAFs](../Editeurs/2010040.md).

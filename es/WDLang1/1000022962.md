<a name="Use"></a>
<a name="description"></a>
Cuando se llama a esta función desde el evento "Solicitud para actualizar la visualización" de un supercontrol, modelo de controles, ventana interna o página interna, se ejecuta inmediatamente el evento "Solicitud para actualizar la visualización" del nivel superior, (por lo general el nivel superior es una ventana, pero también puede ser un supercontrol, un modelo de controles o una página interna).

Si se llama a esta función varias veces durante un evento, el evento de actualización se ejecutará varias veces.

**Observación**: Esta función se utiliza principalmente: 

- cuando se actualiza la interfaz mediante hilos. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) cuando se utiliza la programación MVP (Modelo - Vista - Presentador). Para obtener más información, consulte [Presentación MVP: Modelo - Vista - Presentador](../Editeurs/1000021496.md).

<a name="sample_code"></a>

## Ejemplo


```wl
ExecuteRefreshUIParent()
```

## Sintaxis
<a name="SYNTAX1"></a>

`ExecuteRefreshUIParent([<Parameter 1> [... [, <Parameter N>]]])`
---

**`<Parameter 1>: (Tipo de valor esperado por el evento "Solicitud para actualizar la visualización")`**

Primer parámetro que se pasará al evento "Solicitud para actualizar la visualización" de la ventana o página. 



**`<Parameter N>: (Tipo de valor esperado por el evento "Solicitud para actualizar la visualización")`**

Enésimo parámetro que se pasará al evento "Solicitud para actualizar la visualización" de la ventana o página.

## Observaciones


### Evento "Solicitud para actualizar la visualización"
<a name="evento_solicitud_para_actualizar_visualizacion_ELTPARAGRAPHE000061"></a>

El evento "Solicitud para actualizar la visualización" está asociado a las ventanas, ventanas internas, páginas, modelos de páginas, modelos de controles y supercontroles. Este evento permite agrupar el código de actualización de los controles: contenido de los controles, estado (visible, en gris, etc.), título a actualizar, etc.

Este evento puede recibir parámetros. Simplemente declare un procedimiento en el proceso. Por ejemplo: 


```wl
// Request for refreshing the display
PROCEDURE AskForRefresh(RefreshType is Boolean)
```


Este evento se ejecuta mediante solicitud previa con las siguientes funciones:

- [RequestUpdateUI](../WDLang1/1000021378.md): Ejecuta el evento "Solicitud para actualizar la visualización" al final del proceso en curso (ejecución asíncrona).

- [ExecuteRefreshUI](../WDLang1/1000021379.md): Ejecuta inmediatamente el evento "Solicitud para actualizar la visualización". 

- [RequestUpdateUIParent](../WDLang1/1000022961.md): Ejecuta el evento "Solicitud para actualizar la visualización" del nivel superior al final del proceso en curso (ejecución asíncrona).

- **ExecuteUpdateUIParent**: Ejecuta inmediatamente el evento "Solicitud para actualizar la visualización" del nivel superior.


Al llamar a estas funciones, se puede especificar el valor de los parámetros esperados por el evento. 

Observación: El evento "Solicitud para actualizar la visualización" también puede ejecutarse inmediatamente con la función [ExecuteProcess](../WDLang1/3013031.md) utilizando la siguiente sintaxis: 


```txt
ExecuteProcess(<Element name>, trtRequestRefresh)
```

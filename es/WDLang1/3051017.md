
## MCIOpen( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Abre un archivo multimedia.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Open the "C:\MyFiles\File.AVI" file
// in the "ImageAVI" Image control
ResExecuteMCI = MCIOpen("C:\MyFiles\File.AVI")
```


<a name="Example2"></a>



```wl
// Play a video "C:\MyFiles\File.AVI" in repeat mode
// Open the video
MCIOpen("C:\MyFiles\File.AVI", "AVI", MciStyleChild, IMG_Video, MciModeNormal)
// Play the video
MCIExecute("Play AVI repeat")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = MCIOpen(<Media file> [, <Alias name> [, <Type of container object> [, <Container window> [, <Display mode> [, <Parameters> [, <Name of the message window>]]]]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el archivo fue abierto,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para get el número de error, utilice [MCIErr](../WDLang1/3051011.md). 
	Para obtener más detalles sobre el error, utilice [MCIMsgErr](../WDLang1/3051021.md) o [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.
	El estado reporte del comando es devuelto por [MCIRetrieve](../WDLang1/3051022.md).




**`<Media file>: (Cadena de caracteres o constante)`**

Nombre del archivo multimedia a abrir (AVI, MP3, etc.) o tipo de dispositivo a abrir:


|   |   |
| --- | --- |
| *MciAudioCD* | Inicializa el CD de audio |
| *MciDigitalVideo* | Inicializa el reproductor de vídeo digital |
| *MciSequencer* | Inicializar la mezcla de MDI tabla |
| *MciVideoDisc* | Inicializa el reproductor de vídeo |
| *MciWaveAudio* | Inicializa el lector .WAV |

**Observación**: todos los formatos soportados por el sistema están soportados (lo que significa el CODEC instalado en el equipo).

**`<Alias name>: (Cadena de caracteres opcional)`**

Nombre utilizado para manipular el archivo multimedia. Este nombre se puede utilizar en todas las funciones MCI.

**`<Type of container object>: (Cadena de caracteres opcional)`**

Tipo de ventana o control (ventana secundaria, popup, etc.) en la que se mostrará el archivo multimedia.


|   |   |
| --- | --- |
| *MciStyleChild* | Ventana secundaria de la ventana actual |
| *MciStyleOverlapped* | Ventana estándar |
| *MciStylePopup* | Ventana mostrada en modo popup |



**`<Container window>: (Entero o cadena de caracteres opcionales)`**

Nombre o manipular de la ventana o control donde el archivo será mostrado. 
**Observación**: La manipular es devuelta por [Maneje](../WDLang1/3015005.md).

**`<Display mode>: (Constante opcional)`**

Modo de visualización del archivo en la ventana:


|   |   |
| --- | --- |
| *MciModeCentered* | Image centrada en la ventana |
| *MciModeHomotheticCentered*<br>(Valor predeterminado) | Image estirada proporcionalmente |
| *MciModeNormal* | Visualización en 100% |
| *MciModeStretched* | Image estirada para ocupar toda la superficie |



**`<Parameters>: (Cadena de caracteres opcional)`**

Parámetros adicionales si es necesario. Estos parámetros se pueden conocer consultando la documentación de la función "Open" de Windows MCI..
La constante *MciWait* puede ser añadida a esta cadena. Esta constante se utiliza para indicar que el programa debe esperar al final del comando MCI para ejecutar el resto de la aplicación.. 

**`<Name of the message window>: (Cadena de caracteres opcional)`**

Nombre de la ventana utilizada para interceptar y gestionar los diferentes mensajes enviados por MMSYSTEM.DLL. Estos mensajes están en formato MM_xxxxxx y pueden recuperarse con [Event](../WDLang1/3015011.md). Puede por ejemplo mensajes de get sobre la gestión del CD ROM o del Joystick.




## TableModify (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica una fila en: 

- un control Tabla, 

- un control Tabla TreeView, 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) una tabla que se muestra en un control Combo Box.




**Observación**: Se recomienda utilizar la función [TableModifyLine](../WDLang1/3074028.md). La función [TableModifyLine](../WDLang1/3074028.md) permite agregar:

- TAB en las columnas, 

- todo tipo de valores en las columnas.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Modify the "Paul" and "Moore" elements on the fifth
// row of the "TABLE_CustomerTable" control
TableModify(TABLE_CustomerTable, "Paul" + TAB + "Moore", 5)
```
<a name="Example2"></a>

```wl
// Modification in a column of a Table control
TableModify(COL_LASTNAME, "Moore", 5)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`TableModify(<Table control> , <Row> [, <Index>])`
---

**`<Table control>: (Nombre del control)`**

Nombre del control a manipular. Este control puede corresponder a: 

- un control Tabla.

- un control Tabla TreeView. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) un control Combo Box con tabla.


Si este parámetro corresponde a una cadena vacía (""), el control utilizado será el control al que pertenece el actual Event.

**`<Row>: (Cadena de caracteres)`**

Fila modificada que reemplaza a la fila especificada. Esta fila tiene el siguiente formato: 

```txt
<Element of column 1> + TAB + <Element of column 2> + TAB + ...
```


**`<Index>: (Entero opcional)`**

Índice de la fila por modificar. El valor de la primera fila es 1.


|   |   |
| --- | --- |
| Índice no especificado o igual a -1 | Se modifica la fila seleccionada. Esta constante no tiene efecto si no se selecciona fila. |
| Índice mayor que el número de filas de control | Se produce un error del WLanguage. |

<a name="NOTE0_1"></a>

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000139"></a>

La función **TableModify** puede utilizarse en:

- una Tabla o control Tabla TreeView basada en un archivo de datos.

- una Tabla o control Tabla TreeView rellenada programáticamente.

- un control de selección única o de selección múltiple.
	



<a name="NOTE0_2"></a>


### Características de una fila
<a name="caracteristicas_una_fila_ELTPARAGRAPHE000180"></a>

- Cada elemento de fila se cambia al tipo de la columna associated.

- Si uno de los elementos corresponde a una cadena vacía ("") o a 0 (dependiendo del tipo de columna), se sobrescribirá su valor.. 
	Por ejemplo:
	
	```wl
	TableModify(TABLE_CustomerTable, "Moore" + TAB + "Peter" + TAB + "" + ...
			TAB + "06/21/72")
	```


- Si un elemento corresponde al resultado de un cálculo, la expresión numérica debe estar entre paréntesis.. 
	Por ejemplo:
	
	```wl
	TableModify(TABLE_CalcTable, (53+29) + TAB + (83-21))
	```


- Para modificar el valor de un elemento de una fila, utilice la siguiente sintaxis:
	
	```txt
	TableName.ColumnName[RowIndex] = NewValue
	```

### Control Tabla o Tabla TreeView basado en un archivo de datos : Modificaciones en el archivo de datos relacionado
<a name="control_tabla_tabla_treeview_basado_archivo_datos_modificaciones_archivo_datos_relacionado_ELTPARAGRAPHE000214"></a>

**Si la entrada en cascada está habilitada**, el archivo de datos relacionado con el control se modifica automáticamente cuando el usuario pasa a la siguiente fila.

**Si la entrada en cascada no está habilitada**, el archivo de datos relacionado con el control no se modifica. Para escribir el registro en el archivo de datos relacionado, la función [TableSave](../WDLang1/3074008.md) debe utilizarse después de la función **TableModify**.

**Observaciones**: 

- La opción "Entrada en cascada" puede seleccionarse en la pestaña "Detalles" de la ventana del control description.

- ![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) La entrada en cascada no está disponible en controles Tabla basados en un archivo de datos con acceso directo. Esta opción solo está disponible para controles Tabla rellenado mediante programación y controles Tabla basados en un archivo de datos cargado en memoria.



<a name="NOTE0_4"></a>


### Modificaciones realizadas en un control Tabla/Tabla TreeView basado en un archivo de datos
<a name="modificaciones_realizadas_control_tablatabla_treeview_basado_archivo_datos_ELTPARAGRAPHE000242"></a>

Para modificar una fila de un control Tabla/Tabla TreeView, le aconsejamos:

- modificar el registro en el archivo de datos con la función [HModify](../WDLang4/3044042.md),

- utilizar la función [TableDisplay](../WDLang1/3074003.md) para volver a actualizar el control.

### Función TableModify y Property modificada
<a name="funcion_docparampagetitleshort_property_modificada_ELTPARAGRAPHE000276"></a>

**TableModify** no influye en el [Modified](../Proprietes/2510080.md) Property. El [Modified](../Proprietes/2510080.md) Property se utiliza para saber si el fila actual ha sido modificado directamente por el usuario (entrada directa en el control).

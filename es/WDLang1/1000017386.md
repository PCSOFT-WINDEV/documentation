
## AnimationPrepare (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Prepara la animación en un control, un grupo de control s o una ventana. A continuación, la animación será reproducida por [AnimaciónJuego](../WDLang1/1000017387.md).

El principio es el siguiente:

1. Uso de la función **AnimationPrepare**.

2. Modificar los controles en el código (rellenar, mover, cambiar de plano, etc.). Estas modificaciones no se visualizan. Estas modificaciones se mostrarán cuando se utilice [AnimaciónJuego](../WDLang1/1000017387.md).

3. función [AnimaciónJuego](../WDLang1/1000017387.md). Esta función se utiliza para pasar del estado "antes de la modificación" al estado "después de la modificación" mediante la reproducción de una animación.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Prepare the animation
// The animation will be played in the window
AnimationPrepare(WIN_Customer, WIN_Customer.X, WIN_Customer.Y, ...
		WIN_Customer.Width, WIN_Customer.Height)
// Modify controls 
BTN_Down.Visible = False
IMG_Detail.Height += 50
WIN_Customer.Plane--
// Performs the modifications and plays the animation
AnimationPlay(animFadeIn)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`AnimationPrepare(<Element> [, <X> , <Y> , <Width> , <Height>])`
---

**`<Element>: (Cadena de caracteres)`**

Nombre de la control o ventana donde se realizará la animación.

**`<X>: (Entero opcional)`**

Coordenada X (en píxeles) de la Corner superior izquierda del rectángulo donde se realizará la animación.

**`<Y>: (Entero opcional)`**

Coordenada Y (en píxeles) de la Corner superior izquierda del rectángulo donde se realizará la animación.

**`<Width>: (Entero opcional)`**

Ancho (en píxeles) del rectángulo donde se realizará la animación.

**`<Height>: (Entero opcional)`**

Altura (en píxeles) del rectángulo donde se realizará la animación.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si se llama a la función **AnimationPrepare** dos veces sin llamar a la función [AnimaciónJuego](../WDLang1/1000017387.md), se produce un error.

- Para no ralentizar el proceso de apertura de la ventana, **AnimationPrepare** y [AnimaciónJuego](../WDLang1/1000017387.md) no tienen efecto cuando se utilizan en el evento "Inicialización" de la ventana.

- Las funciones **AnimationPrepare** y [AnimaciónJuego](../WDLang1/1000017387.md) no tienen ningún efecto: 

	- si "Activar animación de controles en el proyecto" no está marcada. Esta opción está disponible en la ventana de description del proyecto, pestaña "Avanzado", botón "Animación de controles".. 

	- si no se ha llamado a [AnimaciónPermitida](../WDLang1/1000017399.md) para activar las animaciones.

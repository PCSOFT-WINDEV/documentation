


## &lt;Tabla de vista de árbol&gt;.SortChild (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Clasifica una rama de una control Tabla TreeView en una o más columnas..

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Sort the "Designation" and "Price" columns in the "TVT_Product" control
// The "Designation" column is sorted in ascending order
// The "Price" column is sorted in descending order
TVT_Product.SortChild(NULL, "Designation", "-Price")
```
<a name="Example2"></a>

```wl
// Sort the branch in descending order
TVT_Product.SortChild("Plumbing" + TAB + "Faucets", "Designation", "-Price")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Clasificando una rama en una control Tabla TreeView en una o más columnas identificadas por su nombre.

`<Result> = <TreeView Table control>.SortChild(<Branch to sort> , <Name of column 1> [, <Name of column 2> [... [, <Name of column N>]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó el orden,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<TreeView Table control>: (Nombre del control)`**

Nombre del control Tabla TreeView a manipular.

**`<Branch to sort>: (Entero o cadena de caracteres)`**

Si este parámetro es un número entero, corresponde a la índice del padre que contiene la rama a ordenar. 
Si este parámetro corresponde a NULL, la clasificación se realiza en los elementos "Raíz".... 
Si este parámetro corresponde a un string, corresponde al elemento padre que contiene la rama a clasificar.

**`<Name of column 1>: (Cadena de caracteres)`**

Nombre de la primera columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:
"[&lt;Direction&gt;]&lt;Nombre de columna&gt;" donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": orden descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.




**`<Name of column 2>: (Cadena de caracteres opcional)`**

Nombre de la segunda columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:
"[&lt;Direction&gt;]&lt;Nombre de columna&gt;" donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": orden descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.






**`<Name of column N>: (Cadena de caracteres opcional)`**

Nombre de la enésima columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:
"[&lt;Direction&gt;]&lt;Nombre de columna&gt;" donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": orden descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.

## Observaciones


### Gestionar una control Tabla TreeView clasificada
<a name="gestionar_una_control_tabla_treeview_clasificada_ELTPARAGRAPHE000264"></a>

- Al ordenar un control Tabla TreeView con la función **&lt;Tabla de vista de árbol&gt;.SortChild**, los elementos que se agregan al control con las funciones [&lt;Table&gt;.Add](../WDLang1/1000023600.md), [&lt;Tabla&gt;.AddLine](../WDLang1/1000023404.md), [&lt;Tabla TreeView&gt;.AddChild](../WDLang1/1000021891.md), [.Insert](../WDLang1/1000024260.md), [&lt;Tabla&gt;.InsertLine](../WDLang1/1000023407.md) o [&lt;Tabla TreeView Table&gt;.InsertChild](../WDLang1/1000021934.md) tienen en cuenta el orden.

- Cuando se ordena una control Tabla TreeView haciendo clic, las adiciones hechas por [&lt;Table&gt;.Add](../WDLang1/1000023600.md), [&lt;Tabla&gt;.AddLine](../WDLang1/1000023404.md), [&lt;Tabla TreeView&gt;.AddChild](../WDLang1/1000021891.md), [.Insert](../WDLang1/1000024260.md), [&lt;Tabla&gt;.InsertLine](../WDLang1/1000023407.md) o [&lt;Tabla TreeView Table&gt;.InsertChild](../WDLang1/1000021934.md) ignoran la ordenación.

- ***El orden no se cancela*** con las funciones [&lt;Table&gt;.Display](../WDLang1/1000023505.md), [&lt;Table&gt;.Adjust](../WDLang1/1000023840.md), [TableSeek](../WDLang1/3074015.md), [&lt;Tabla&gt;.cuenta](../WDLang1/1000024277.md), [.Posición](../WDLang1/1000024278.md), [&lt;Tabla&gt;.Select](../WDLang1/1000024284.md), [&lt;Tabla&gt;.SelectMinus](../WDLang1/1000024285.md), [&lt;Tabla&gt;.SelectPlus](../WDLang1/1000024287.md), [.Delete](../WDLang1/1000024289.md) y [&lt;Tabla&gt;.DeleteAll](../WDLang1/1000024292.md).

- ***El orden no se cancela***: 

	- si se modifica una columna (por entrada directa en la columna o por [&lt;Tabla&gt;.Modify](../WDLang1/1000024273.md) y [&lt;Tabla&gt;.ModifyLine](../WDLang1/1000024275.md)) y si esta columna está clasificada. 

	- por [&lt;Tabla&gt;.ordenación](../WDLang1/1000024293.md) (&lt;control Tabla&gt;.Sort(lse)).

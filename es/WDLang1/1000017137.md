


## dbgEnableLog( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Activa el registro de tiempo de ejecución mediante programación.

**Observación**: A partir de la versión 28, el registro de ejecución contiene un archivo ".wxl", un archivo ".ndx" y un archivo ".mmo".. Si el proyecto correspondiente al registro de tiempo de ejecución se abre en el editor, el archivo de registro se puede abrir directamente en el editor.: 

- en la pestaña "Inicio", en el grupo "General", expanda "Abrir", seleccione "Abrir" y, a continuación, seleccione el archivo ".wxl" deseado.

- arrastre el archivo ".wxl" desde el explorador de archivos y suéltelo en el editor. 


Para obtener más información, consulte [Registro de ejecución](../Editeurs/2019030.md).

Nota: En versiones anteriores, el registro de tiempo de ejecución era un archivo ".wlog. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Enables the default runtime log
dbgEnableLog()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = dbgEnableLog([<Log file> [, <Options>]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ha iniciado el registro de ejecución,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Log file>: (Cadena de caracteres opcional)`**

Ubicación y nombre del archivo de registro a crear. El nombre y la ubicación del archivo de registro pueden ser: 

- totalmente personalizado. 

- construido con las siguientes palabras clave: 
	


|   |   |
| --- | --- |
| [%ExeDir%] | Directorio del ejecutable, siempre rellenado con el carácter "\\".. Este directorio es equivalente al resultado de [fExeDir](../WDLang1/3036049.md). |
| [%exeName%] | Nombre del ejecutable. |
| [%Fecha%] | Fecha de inicio del tiempo de ejecución en formato Log( YYYMMDD). |
| [%Hora%] | Hora de inicio del tiempo de ejecución formato Log( HHMMSS). |
| [%NombreUsuario%] | Nombre del usuario actual. |
| [%NombreDeLaComputadora%] | Nombre del ordenador. |
| [%DataDir%] | Directorio de los archivos de datos HFSQL especificados por el usuario al instalar la aplicación. Este directorio es equivalente al resultado de [fDataDir](../WDLang1/3036001.md). |
| [%DirUserData%] | Directorio para los datos del usuario actual de la aplicación actual.  Este directorio es equivalente al resultado de [fDataDirUser](../WDLang1/3036071.md). |





Por Default, el nombre del archivo de registro corresponde a: 
"[%UserDataDir%][%ExeName%]_[%Date%]_[%Time%]". 
Por ejemplo: 
"C:\\Users\\Doc\\AppData\\Roaming\\WINDEV Applications\\MyApp\\Myapp_20071023_130812.wlog".

Atención: Si construye cadenas dinámicamente (opción "Permitir '[% %]' en las cadenas" en la pestaña "Compilación" del proyecto description, aparece un error de compilación (identificador desconocido). En este caso, cada cadena debe ir precedida de "-%".. Ejemplo: 

```wl
dbgEnableLog("[%DirUserData%][%ExeName%]_[%Date%]_[%Time%]")
```
cambia a
```wl
dbgEnableLog(-%"[%DirUserData%][%ExeName%]_[%Date%]_[%Time%]")
```


**`<Options>: (Constante o combinación de constantes de tipo Integer (opcional))`**

Se utiliza para configurar las opciones del registro de tiempo de ejecución:


|   |   |
| --- | --- |
| *LogAll* | Se recupera el log de tiempo de ejecución:<br><br>- las funciones de WLanguage<br><br>- los parámetros de las funciones<br><br>- los procesos<br><br><br> |
| *LogEnabled* | El log de tiempo de ejecución se activa inmediatamente. |
| *LogParameters* | Los parámetros de las funciones se registran en el log de tiempo de ejecución. |
| *LogPause* | El log de tiempo de ejecución no se activa inmediatamente |
| *LogProcessOnly* | El log de tiempo de ejecución sólo recupera información sobre los procesos. |
| *LogWLanguageFunctions* | Las funciones WLanguage se registran en el log de tiempo de ejecución. |


La combinación *LogEnabled* + *LogWLanguageFunctions* es utilizada por Default si este parámetro no se especifica.

## Observaciones


### Modo de funcionamiento del log de tiempo de ejecución
<a name="modo_funcionamiento_del_log_tiempo_ejecucion_ELTPARAGRAPHE000270"></a>

- **dbgEnableLog** debe llamarse solo una vez. Cualquier otra llamada a esta función no tendrá efecto.

- Para activar o desactivar temporalmente el registro de tiempo de ejecución, utilice [dbgLogStatus](../WDLang1/1000017138.md).

- Para modificar las opciones del registro de ejecución, utilice [dbgLogOption](../WDLang1/1000017136.md).


![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Observación: El registro de tiempo de ejecución también puede iniciarse a través del archivo ".wx" que se encuentra junto al ejecutable. Para obtener más información, consulte [Registro de ejecución](../Editeurs/2019030.md).


### Varios
<a name="varios_ELTPARAGRAPHE000292"></a>

- **dbgEnableLog** se puede usar en un servicio web REST. 

- Puede definir cualquier ubicación para el archivo de registro. En este caso, tenga en cuenta que puede haber problemas relacionados con Permission. 
	Recordatorio: Generalmente, el directorio de datos es accesible en modo de escritura.

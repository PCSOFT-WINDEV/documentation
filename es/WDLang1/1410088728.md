
## &lt;Elemento&gt;.RequestRefreshUI (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Solicita la ejecución del evento "Solicitud para actualizar la visualización" del elemento especificado (y devuelve el control **sin esperar la ejecución del evento**).

Esta función se utiliza principalmente: 

- cuando se actualiza la UI mediante subprocesos. 

- cuando se utilizan controles "contenedores": ventana interna, página interna, modelo de controles, supercontrol, etc.

- cuando se utilizan ventanas hijas, etc.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Save process
// The UI of the child window will have to be refreshed
WIN_Child.RequestRefreshUI()
```

```wl
// -----------------------------------------------------------------
// -- Request for refreshing the display of WIN_Child
// Refresh the title of the window
Myself.Title = gpclContactForm.p_sFullname

// Refresh the linked controls of the window
SourceToScreen()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Element>.RequestRefreshUI([<Parameter 1> [... [, <Parameter N>]]])`
---

**`<Element>: (Nombre del control)`**

Elemento para el que debe ejecutarse el evento "Solicitud para actualizar la visualización". Este elemento puede corresponder al nombre de una ventana, página, ventana o página interna, modelo de ventanas o páginas, modelo de controles, supercontrol, etc.

**`<Parameter 1>: (Parámetro opcional que corresponde al tipo esperado por el evento "Solicitud para actualizar la visualización")`**

Primer parámetro que se pasará al evento "Solicitud para actualizar la visualización" de la ventana o control.



**`<Parameter N>: (Tipo de valor esperado por el evento "Solicitud para actualizar la visualización")`**

Enésimo parámetro que se pasará al evento "Solicitud para actualizar la visualización" de la ventana o control.

## Observaciones


### Evento "Solicitud para actualizar la visualización"
<a name="evento_solicitud_para_actualizar_visualizacion_ELTPARAGRAPHE000191"></a>

El evento "Solicitud para actualizar la visualización" contiene el código para actualizar los controles de un elemento determinado: contenido de los controles, estado (visible, en gris, etc.), texto a actualizar, etc, ... Este evento está disponible en ventanas, ventanas internas, páginas, páginas internas, modelos de controles y supercontroles.

Este evento puede recibir parámetros. Solo se debe declarar un procedimiento en el evento. Por ejemplo: 


```wl
// Request for refreshing the display
PROCEDURE AskForRefresh(RefreshType is Boolean)
```


Este evento se ejecuta mediante solicitud previa con las siguientes funciones: 

- **&lt;Elemento&gt;.RequestRefreshUI**: Ejecuta el evento "Solicitud para actualizar la visualización" del elemento especificado al final del proceso actual (ejecución asíncrona).

- [&lt;Elemento&gt;.ExecuteRefreshUI](../WDLang1/1410088729.md): Ejecuta inmediatamente el evento "Solicitud para actualizar la visualización" del elemento especificado. 

- [RequestRefreshUIParent](../WDLang1/1000023900.md): Ejecuta el evento "Solicitud para actualizar la visualización" del nivel superior al final del proceso en curso (ejecución asíncrona).

- [ExecuteRefreshUIParent](../WDLang1/1000023902.md): Ejecuta inmediatamente el evento "Solicitud para actualizar la visualización" del nivel superior.


Al llamar a estas funciones, se puede especificar el valor de los parámetros esperados por el evento "Solicitud para actualizar la visualización".

### Atributo "Presentador"
<a name="atributo_presentador_ELTPARAGRAPHE000257"></a>

El atributo **presentador** permite asociar una clase de la capa del presentador a una vista (ventana, página o reporte). 

Este atributo se utiliza principalmente en la declaración global de las ventanas generadas por el RAD MVP en WINDEV. 

Por ejemplo: 


```wl
PROCEDURE WIN_Table_MyExampleFile(...
	gclPresenter is PTableMyExampleFile dynamic<presenter>=Null)
```


Con este atributo, el evento "Solicitud para actualizar la visualización" de la ventana se activará al realizar:

- una llamada a la función **&lt;Elemento&gt;.RequestRefreshUI** desde una clase declarada como &lt;presentador&gt;,

- una llamada desde cualquier clase asociada a la clase &lt;presentador&gt;. 




Para obtener más información, consulte [RAD MVP: Atributo "Presentador"](../Editeurs/1000021508.md).

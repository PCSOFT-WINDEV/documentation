


## IniOSEmulatorMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Identifica si el código se ejecuta o no en el emulador iOS (iPhone, iPad). 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF IniOSEmulatorMode() = True THEN
	// Code specific to the iOS emulator
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = IniOSEmulatorMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si el código se ejecuta en el emulador iOS,

- <u><u><u><u>False</u></u></u></u> en caso contrario.










<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000212"></a>

La función **IniOSEmulatorMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **IniOSEmulatorMode** pueden compartirse entre varios productos sin generar errores de compilación.
<a name="NOTE0_2"></a>


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000226"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.

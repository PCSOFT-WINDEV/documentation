


## dChangeMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cambia el modo de dibujo utilizado por las funciones de dibujo ([dLine](../WDLang1/3029016.md), [dRectangle](../WDLang1/3029023.md), etc.). El cambio se aplica al elemento que se está dibujando. Este elemento puede ser:  

- un control Imagen, 

- en una variable de tipo [Image](../WDLang1/1000019650.md).
	

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en una variable de tipo [WDPic](../WDLang1/1000023503.md) (en la capa de fondo),

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en una variable de tipo [picLayer](../WDLang1/1000024604.md).





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Initialize the drawing in an Image control
dStartDrawing(IMG_Drawing)

// Draws a rectangle and erases it
dChangeMode(drawXOR)
// Displays the rectangle (in XOR mode)
dRectangle(0, 0, 100, 100)
Multitask(10)
// Erases the rectangle
dRectangle(0, 0, 100, 100)
```


![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Draws a rectangle in an Image control and erases it
dChangeMode(IMG_Drawing, drawXOR)
// Displays the rectangle (in XOR mode)
dRectangle(IMG_Drawing, 0, 0, 100, 100)
Multitask(10)
// Erases the rectangle
dRectangle(IMG_Drawing, 0, 0, 100, 100)
```

## Sintaxis

`dChangeMode([<Image>, ] <New mode>)`
---

**`<Image>: (Nombre de control opcional o variable de tipo Image, WDPic o picLayer opcional)`**

![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) Imagen a manipular. Esta imagen puede corresponder a: 

- el nombre de un control Imagen. 

- el nombre de una variable de tipo [Image](../WDLang1/1000019650.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) el nombre de una variable de tipo [WDPic](../WDLang1/1000023503.md). Solo se manipulará la capa de fondo. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) el nombre de una variable de tipo [picLayer](../WDLang1/1000024604.md).


Si no se especifica este parámetro, se debe definir el destino del dibujo con la función [dStartDrawing](../WDLang1/3029009.md).

**`<New mode>: (Constante de tipo Integer)`**

Modo de dibujo en la imagen:



|   |   |
| --- | --- |
| *drawAntiAliasing* | Utiliza el GDI+ y/o las funcionalidades de la tarjeta gráfica para dibujar líneas y círculos sin efecto de dientes de sierra. Este modo de dibujo es más lento pero el resultado es mejor que con el modo estándar.<br> |
| *drawDefault* | Colores estándar. |
| *drawXOR* | Colores aplicados mediante una operación de "O Exclusivo" con el fondo. Borra un dibujo si esta opción se utiliza dos veces.<br>ATENCIÓN: Este modo de dibujo no es compatible con la gestión de la opacidad (el dibujo no se borra)..<br>**Atención**: <br><br>- El dibujo en modo XOR solo está disponible para controles Imagen. El dibujo en modo XOR no está disponible para variables de tipo [Image](../WDLang1/1000019650.md). <br><br>- La constante *drawXOR* no puede combinarse con la constante *drawAntiAliasing*. <br><br><br> |

### Antialiasing
<a name="antialiasing_ELTPARAGRAPHE000281"></a>El antialiasing consiste en dibujar líneas y círculos sin efecto de dientes de sierra. Esta funcionalidad requiere el framework GDI+ (es decir, el framework de WINDEV).
Ejemplo:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=dchangemode.gif)
Límites:

- La función [dChord](../WDLang1/3029008.md) no está soportada en este modo.

- Para evitar que los bordes tengan un efecto de dientes de sierra, los colores y las líneas de los elementos se suavizan. Algunas funciones de dibujo pueden tener un comportamiento inesperado porque ya no se encuentran los colores especificados. Por ejemplo, si se especifica un color para el borde con la función [dFill](../WDLang1/3029017.md), y este color no se encuentra: se rellena toda el área.

- Para un dibujo con antialiasing, las funciones [dPolygon](../WDLang1/3029006.md) y [dSlice](../WDLang1/3029021.md) tienen el siguiente comportamiento. El contorno se dibuja por encima del interior del polígono: por lo tanto, la línea se combina parcialmente con el interior del polígono. Si la línea es muy gruesa, se mostrará en dos colores.

- Para obtener más información sobre el uso de GDI+, consulte [Framework GDI+](../Editeurs/2020007.md).

### Uso de las funciones de dibujo
<a name="uso_las_funciones_dibujo_ELTPARAGRAPHE000471"></a>

Las funciones de dibujo se pueden utilizar según 2 métodos:

- **Método 1: Uso de la función dStartDrawing**

	- Se debe llamar a la función [dStartDrawing](../WDLang1/3029009.md) antes de utilizar otras funciones de dibujo. La función [dStartDrawing](../WDLang1/3029009.md) permite definir el elemento (control Imagen o variable) en el que se aplicarán las funciones de dibujo.

	- Al dibujar en un control Imagen: 

		- Las funciones de dibujo trabajan en una copia ("bitmap") de la imagen. Se pueden utilizar las funciones de dibujo de Windows (con la función [API](../WDLang1/3014005.md) o [CallDLL32](../WDLang1/3014003.md)), pero estas funciones deben utilizar el DC (Device Context) devuelto por la función [dStartDrawing](../WDLang1/3029009.md).

		- las funciones [dEndDrawing](../WDLang1/3029019.md) y [dStartDrawing](../WDLang1/3029009.md) no deben llamarse en el mismo proceso.
						Si se llama a las funciones [dEndDrawing](../WDLang1/3029019.md) y [dStartDrawing](../WDLang1/3029009.md) en el mismo proceso, no se mostrará el dibujo: se borrará automáticamente.




- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) **Método 2: indicar el destino del dibujo directamente en la sintaxis (mediante un parámetro)**
	El parámetro &lt;Imagen&gt; permite especificar directamente el destino del dibujo. Ya no es necesario utilizar [dStartDrawing](../WDLang1/3029009.md). Esta función debe ser eliminada.

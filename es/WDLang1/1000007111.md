


## HashString (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Calcula el Hash de una cadena de caracteres según un algoritmo específico. Puede calcular: 

- un simple hachís.

- un hash con autenticación de mensajes.

- un hash obtenido con una función de derivación de claves que toma una sal como parámetro (PBKDF2).
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) No disponible.




**Recordatorio**: El hashing se utiliza para resumir un archivo o una cadena de caracteres. Este resumen se llama Hash.

```wl
Hash1 is Buffer = HashString(HA_MD5_128, EDT_Edit1)

IF Hash1 = Hash2 THEN Info("The two strings are identical.")
```

## Sintaxis

### Calculando un simple hash

---

**`<Result>: (Buffer)`**

Resultado del hashing &lt;String&gt; con el &lt;Tipo de algoritmo&gt; algoritmo.

Atención: este resultado puede contener caracteres que no se pueden mostrar. Si desea mostrar el resultado, lo puede convertir con la función [BufferToHexa](../WDLang1/1000019916.md).

**`<Type of algorithm>: (Constante de tipo Integer)`**

Indica el tipo de algoritmo que se utiliza al hacer el hashing de la &lt;String&gt;:



|   |   |
| --- | --- |
| Familia DJB2 (Bernstein) | <br><br>- *HA_DJB2_32*<br><br>- *HA_DJB2_64*<br><br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes de esta familia no están disponibles. |
| MD4 family | *HA_MD4* |
| MD5 family | *HA_MD5_128*<br><br>Atención: este tipo de algoritmo está actualmente en desuso. |
| MURMUR family | <br><br>- *HA_MURMUR_1*: Algoritmo de soplo versión 1: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2*: Algoritmo de soplo versión 2: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_64A*: Algoritmo de soplo versión 2: Resultado hash de 64 bits optimizado para procesadores de 64 bits<br><br>- *HA_MURMUR_2_64B*: Algoritmo de soplo versión 2: Resultado hash de 64 bits optimizado para procesadores de 32 bits<br><br>- *HA_MURMUR_2_A*: Algoritmo de murmullo incremental versión 2: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_BIG_ENDIAN*: Algoritmo de murmullo versión 2 para máquina big-endian: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_ALIGNE*: Algoritmo de soplo versión 2 para máquina alineada: Resultado hash de 32 bits<br><br>- *HA_MURMUR_3_32*: Algoritmo de soplo versión 3: Resultado hash de 32 bits<br><br>- *HA_MURMUR_3_128_X86*: Algoritmo de soplo versión 3: Resultado hash de 128 bits optimizado para procesadores de 32 bits<br><br>- *HA_MURMUR_3_128_X64*: Algoritmo de soplo versión 3: Resultado hash de 128 bits optimizado para procesadores de 64 bits<br><br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes de esta familia no están disponibles. |

- Algoritmos MD4 / MD5 / SHA / RIPE: funciones de hash criptográficas estándar.

- Algoritmos TIGER/WHIRLPOOL: funciones hash con altas propiedades criptográficas optimizadas para el modo de 64 bits (pero también se puede utilizar en el modo de 32 bits).

- Algoritmos CKSUM: funciones de hash sin Property criptográfica, deben utilizarse para realizar comprobaciones de bajo nivel o con tablas de hash.

- Algoritmo DJB2 (ámbar): Las funciones hash reservadas a las cadenas y sin Property criptográfica, deben utilizarse para realizar comprobaciones de bajo nivel o con tablas hashing. Pocos riesgos para get un hachís idéntico. 

- Algoritmo MURMUR: funciones de hash muy rápidas sin propiedades criptográficas. Este algoritmo es usado por el "filtro de florecimiento" de las carteras de bitcoin.




**`<String>: (Cadena de caracteres)`**

String sobre el que se realizará el cálculo. Este parámetro puede contener caracteres binarios.

### Calculando un hash con autenticación de mensajes (algoritmo HMAC/MURMUR)

---

**`<Result>: (Buffer)`**

Atención: este resultado puede contener caracteres que no se pueden mostrar.

**`<Type of algorithm>: (Constante de tipo Integer)`**

Indica el tipo de algoritmo (HMAC o MURMUR) utilizado para el hash de la &lt;Cadena&gt;.:

| --- | --- |
| MD4 family | *HA_HMAC_MD4*<br><br> |
| MD5 family | *HA_HMAC_MD5_128*<br><br>Atención: este tipo de algoritmo está actualmente en desuso. |
| MURMUR family | <br><br>- *HA_MURMUR_1*: Algoritmo de soplo versión 1: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2*: Algoritmo de soplo versión 2: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_64A*: Algoritmo de soplo versión 2: Resultado hash de 64 bits optimizado para procesadores de 64 bits<br><br>- *HA_MURMUR_2_64B*: Algoritmo de soplo versión 2: Resultado hash de 64 bits optimizado para procesadores de 32 bits<br><br>- *HA_MURMUR_2_A*: Algoritmo de murmullo incremental versión 2: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_BIG_ENDIAN*: Algoritmo de murmullo versión 2 para máquina big-endian: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_ALIGNE*: Algoritmo de soplo versión 2 para máquina alineada: Resultado hash de 32 bits<br><br>- *HA_MURMUR_3_32*: Algoritmo de soplo versión 3: Resultado hash de 32 bits<br><br>- *HA_MURMUR_3_128_X86*: Algoritmo de soplo versión 3: Resultado hash de 128 bits optimizado para procesadores de 32 bits<br><br>- *HA_MURMUR_3_128_X64*: Algoritmo de soplo versión 3: Resultado hash de 128 bits optimizado para procesadores de 64 bits<br><br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes de esta familia no están disponibles. |
| RIPEMD family | <br><br>- *HA_HMAC_RIPEMD_128*<br><br>- *HA_HMAC_RIPEMD_160*<br><br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes de esta familia no están disponibles. |
| Familia SHA | <br><br>- *HA_HMAC_SHA_160*<br><br>- *HA_HMAC_SHA_256* <br><br>- *HA_HMAC_SHA_256_DOUBLE*<br><br>- *HA_HMAC_SHA_384*<br><br>- *HA_HMAC_SHA_512*<br><br><br> |

- Algoritmos MD4 / MD5 / SHA / RIPE: funciones de hash criptográficas estándar.

- Algoritmos TIGER/WHIRLPOOL: funciones hash con altas propiedades criptográficas optimizadas para el modo de 64 bits (pero también se puede utilizar en el modo de 32 bits).

- Algoritmo MURMUR: funciones de hash muy rápidas sin propiedades criptográficas. Este algoritmo es usado por el "filtro de florecimiento" de las carteras de bitcoin.




**`<String>: (Cadena de caracteres)`**

String sobre el que se realizará el cálculo. Este parámetro puede contener caracteres binarios.

**`<Secret key>: (Cadena de caracteres o entero)`**

Clave de autenticación del mensaje. 

- Si se utiliza un algoritmo HMAC, este parámetro debe ser un string. 

- Si se utiliza un algoritmo MURMUR, la clave debe ser un número entero.




![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Este parámetro no debe ser una cadena vacía.

### Calcular un hash utilizando una función de derivación de claves que toma una sal como parámetro (PBKDF2)

`<Result> = HashString(<Type of algorithm> , <String> , <Salt> , <Iteration> , <Length>)`
---

**`<Result>: (Buffer)`**

Resultado del hashing &lt;String&gt; con el &lt;Tipo de algoritmo&gt; algoritmo.

Atención: este resultado puede contener caracteres que no se pueden mostrar. Si desea mostrar el resultado, lo puede convertir con la función [BufferToHexa](../WDLang1/1000019916.md).

**`<Type of algorithm>: (Constante de tipo Integer)`**

Especifica el tipo de algoritmo pseudo-aleatorio utilizado por PBKDF2 para el hash de &lt;Cadena&gt;:



|   |   |
| --- | --- |
| MD4 family | *HA_PBKDF2_HMAC_MD4* |
| MD5 family | *HA_PBKDF2_HMAC_MD5_128*<br><br>Atención: este tipo de algoritmo está actualmente en desuso. |
| RIPEMD family | <br><br>- *HA_PBKDF2_HMAC_RIPEMD_128*<br><br>- *HA_PBKDF2_HMAC_RIPEMD_160*<br><br><br> |
| Familia SHA | <br><br>- *HA_PBKDF2_HMAC_SHA_160*<br><br>- *HA_PBKDF2_HMAC_SHA_256*<br><br>- *HA_PBKDF2_HMAC_SHA_256_DOUBLE*<br><br>- *HA_PBKDF2_HMAC_SHA_384*<br><br>- *HA_PBKDF2_HMAC_SHA_512*<br><br><br> |
| Familia SHA-3 | <br><br>- *HA_PBKDF2_HMAC_SHA3_224*<br><br>- *HA_PBKDF2_HMAC_SHA3_256*<br><br>- *HA_PBKDF2_HMAC_SHA3_384*<br><br>- *HA_PBKDF2_HMAC_SHA3_512*<br><br><br> |
| TIGER family | <br><br>- *HA_PBKDF2_HMAC_TIGER_128*<br><br>- *HA_PBKDF2_HMAC_TIGER_160*<br><br>- *HA_PBKDF2_HMAC_TIGER_192*<br><br><br> |
| WHIRLPOOL family | *HA_PBKDF2_HMAC_WHIRLPOOL* |



- Algoritmos MD4 / MD5 / SHA / RIPE: funciones de hash criptográficas estándar.

- Algoritmos TIGER/WHIRLPOOL: funciones hash con altas propiedades criptográficas optimizadas para el modo de 64 bits (pero también se puede utilizar en el modo de 32 bits).




**`<String>: (Cadena de caracteres)`**

String sobre el que se realizará el cálculo. Este parámetro puede contener caracteres binarios.  

**`<Salt>: (Cadena de caracteres)`**

Sal criptográfica utilizada. Cadena (normalmente aleatoria) añadida a la cadena original para hacerla más compleja. Por ejemplo, hace posible que dos usuarios utilicen la misma contraseña. 

**`<Iteration>: (Integro)`**

Número de iteraciones del algoritmo (10000 por Default). Este número depende de la potencia de la máquina de computación y de la complejidad del hash. En general, se recomienda un mínimo de 1000 iteraciones.

**`<Length>: (Integro)`**

Longitud de la clave derivada. Por Default, corresponde al tamaño de la clave generada por la función pseudoaleatoria. Esta longitud es técnicamente ilimitada, pero debido al algoritmo, está limitada a (2^32 - 1) \* (longitud de salida del algoritmo utilizado por PBKDF2).

## Observaciones


### Notas
<a name="notas_ELTPARAGRAPHE000725"></a>

- Dos cadenas de caracteres idénticos tendrán, para el mismo tipo de algoritmo, dos Hashes idénticos.

- El resultado del Hash puede contener caracteres no visualizables. Estos caracteres no serán Visible cuando se utilice [Info](../WDLang1/3021011.md) o [rastro](../WDLang1/3013050.md), por ejemplo.



<a name="NOTE0_2"></a>


### Función HashString y UNICODE
<a name="funcion_docparampagetitleshort_unicode_ELTPARAGRAPHE000739"></a>

- **Atención**: Las funciones Hash operan sobre los bytes que se encuentran en las cadenas de caracteres. Para el mismo tipo de algoritmo, el resultado será diferente en UNICODE y en ANSI.

- **Desarrollo multiplataforma**: Para utilizar el Hash de cadenas entre varias plataformas (un hash generado en iOS y comprobado en Android o Windows por ejemplo), no se debe utilizar ninguna cadena Unicode. En efecto, las cadenas Unicode no tienen el mismo formato según las plataformas. En este caso, le aconsejamos que utilice cadenas en formato ANSI o UTF 8 (y que convierta las cadenas Unicode si es necesario)..




## &lt;Tabla&gt;.InsertColumn (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inserta elementos en forma de columna en ***un control Tabla horizontal***.

**Atención**: Esta función no se puede utilizar para insertar columnas en un control Tabla estándar.. Para añadir dinámicamente columnas en un control Tabla estándar, utilice [ControlClone](../WDLang1/3025017.md).


<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Table control>.InsertColumn([<Subscript> [, <Element column 1> [... [, <Element column N>]]]])`
---

**`<Table control>: (Control name)`**

Nombre de la control Tabla Horizontal a utilizar.

**`<Subscript>: (Optional integer)`**

Índice de la columna en la que se insertarán los elementos. El índice de la primera columna es set a 1.


|   |   |
| --- | --- |
| Índice no especificado | La columna se inserta antes de la columna actual. Si no se selecciona ninguna columna, la columna se inserta al final de control Tabla. |
| Índice mayor que el número de columnas de control Tabla | La columna se inserta al final de control Tabla. |
| Índice igual a 0 | La columna se inserta al principio de control Tabla. |
| Índice igual a -1 | La columna se inserta antes de la columna actual. |



**`<Element column 1>: (Type of associated column (optional))`**

Primer elemento de la columna que se insertará en la control Tabla especificada. Cada elemento corresponde a una fila en el control Tabla. El tipo de elemento insertado debe ser compatible con el tipo de elemento fila relevante..
Si no se especifica ninguno de estos parámetros, se añade una columna vacía a la columna control Tabla. Para una navegación por control Tabla, se tienen en cuenta los valores por defecto definidos en el archivo relacionado.



**`<Element column N>: (Type of associated column (optional))`**

Elemento de la enésima columna que se debe insertar en la control Tabla especificada. Cada elemento corresponde a una fila en el control Tabla. El tipo de elemento insertado debe ser compatible con el tipo de elemento fila relevante..
Si no se especifica ninguno de estos parámetros, se añade una columna vacía a la columna control Tabla. Para una control Tabla basada en un archivo de datos, se tienen en cuenta los valores de Default definidos en el fichero relacionado.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Características de los elementos de columna
<a name="caracteristicas_los_elementos_columna_ELTPARAGRAPHE000215"></a>

- Para especificar ningún valor para uno de los elementos, utilice una cadena vacía ("") o 0 (según el tipo de fila).

- Si un elemento corresponde al resultado de un cálculo, la expresión numérica debe estar entre paréntesis.. Por ejemplo:
	
	```wl
	TABLE_CalculationTable.InsertColumn(3, (53+29), (83-21))
	```




<a name="NOTE0_2"></a>


### Posición de la columna insertada
<a name="posicion_columna_insertada_ELTPARAGRAPHE000233"></a>

Por Default, la columna se inserta en la posición especificada.

- Si la control Tabla está ordenada con [&lt;Tabla&gt;.ordenación](../WDLang1/1000024293.md), la nueva columna se posiciona según la ordenación actual y no según el índice especificado.

- Si el usuario clasifica la control Tabla, la clasificación se ignora cuando se inserta la columna.. La columna se inserta en la posición especificada.



<a name="NOTE0_3"></a>


### Limitaciones: Número máximo de columnas mostradas
<a name="limitaciones_numero_maximo_columnas_mostradas_ELTPARAGRAPHE000251"></a>

El número máximo de columnas que se pueden mostrar en un control Tabla está limitado por la cantidad de memoria disponible (máximo teórico: 2 mil millones de columnas).

Sin embargo, no le aconsejamos que utilice las tablas control con un número importante de filas por razones de rendimiento (tiempo necesario para llenar el control, por ejemplo) y por su facilidad de uso.
<a name="NOTE0_4"></a>


### Control Tabla basado en un archivo de datos: Modificaciones en el archivo de datos relacionado
<a name="control_tabla_basado_archivo_datos_modificaciones_archivo_datos_relacionado_ELTPARAGRAPHE000263"></a>

Si la entrada en cascada está activada, el archivo de datos relacionado a la navegación control Tabla se modifica automáticamente cuando el usuario pasa a la siguiente columna.

Si la entrada en cascada no está habilitada, no se modifica el el archivo de datos relacionado al control Tabla basado en un archivo de datos. Para escribir el registro en el archivo de datos, la función [&lt;Tabla&gt;.Save](../WDLang1/1000024249.md) debe ejecutarse después de la función **&lt;Tabla&gt;.InsertColumn**.

**Recordatorio**: Se puede seleccionar la opción "entrada en cascada":

- Al crear una control Tabla.

- En la pestaña "Detalles" de la ventana de description de control Tabla en el editor de la ventana.



<a name="NOTE0_5"></a>


### Control Tabla basado en un archivo de datos: Inserción de varias columnas
<a name="control_tabla_basado_archivo_datos_insercion_varias_columnas_ELTPARAGRAPHE000287"></a>

Se crea una columna virtual cada vez que se llama a **&lt;Tabla&gt;.InsertColumn** en un control Tabla basado en un archivo de datos. La información se guardará en el archivo de datos al llamar a la función [&lt;Tabla&gt;.Save](../WDLang1/1000024249.md).

Si se llama a la función **&lt;Tabla&gt;.InsertColumn** sucesivamente en el mismo proceso sin llamar a la función [&lt;Tabla&gt;.Save](../WDLang1/1000024249.md), solo la PRIMERA llamada es efectiva (las siguientes llamadas son ignoradas). Sólo se añade una columna a la control Tabla basada en un archivo de datos.
<a name="NOTE0_6"></a>


### Insertar un Image
<a name="insertar_image_ELTPARAGRAPHE000313"></a>

Para insertar un Image en un control Tabla, especifique el nombre y el ruta completo del Image a insertar.

Por ejemplo:

```wl
TABLE_MyTable.InsertColumn(5, "C:\MyDocuments\MyImage.BMP")
```
**Observación**: El fila debe ser un tipo de imagen fila, de lo contrario el Image insertado no se mostrará correctamente.
<a name="NOTE0_7"></a>


### control Tabla en modo edición
<a name="control_tabla_modo_edicion_ELTPARAGRAPHE000332"></a>

Si se llama a la función **&lt;Tabla&gt;.InsertColumn**:

- En un control Event, el cursor se posiciona automáticamente en el primer fila a editar en la columna insertada.

- En un evento que no pertenece al control, (o en otro Process) se debe utilizar [SetFocusAndReturnToUserInput](../WDLang1/1410088107.md) para reanudar la entrada en el control Tabla.

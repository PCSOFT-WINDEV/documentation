
## MultitaskRedraw( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Vuelve a dibujar inmediatamente los controles de la ventana que se actualizarán gráficamente.

**Observación**: Esta función es equivalente a la función [WinRedraw](../WDLang1/3038006.md).



![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Esta función solo está disponible para simplificar la Webificación de proyectos WINDEV/WINDEV Mobile. En un sitio WEBDEV, esta función no tiene ningún efecto.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
i is int
FOR i = 1 TO 1000
	LongProcess()
	MultitaskRedraw()
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`MultitaskRedraw([<Window or page>])`
---

**`<Window or page>: (Nombre de la ventana o página)`**

Nombre de la ventana a redibujar. La ventana actual se redibuja si no se especifica este parámetro. 





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
A diferencia de la función [Multitask](../WDLang1/3015004.md), el código actual no se interrumpe al llamar a la función **MultitaskRedraw** (no se llama a las funciones [Timer](../WDLang1/3015008.md), por ejemplo).




## &lt;Tabla&gt;.Save (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Actualiza o añade la Record vinculada a la fila actual en la Tabla o control Tabla TreeView. El Record se actualiza: 

- en el archivo de datos relacionado a la control.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en la Variable vinculada a la control. 





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Save the data found on the current row 
// for the "TABLE_Product" control
TABLE_Product.Save()
```



```wl
// Save the data found on row 5 
// while modifying a column through programming
TABLE_Product = 5
TABLE_Product[5].COL_Cap = "new value"
TABLE_Product.Save()
```



```wl
// Event Row exit
IF TABLE_Customer.Modified = False THEN
	RETURN 
END
pclFormPresentation is PCustomerForm dynamic
IF TABLE_Customer.NewLine THEN
	pclFormPresentation<-gclPresentation.New()
ELSE
	pclFormPresentation<-gclPresentation.Modify()
END
TABLE_Customer.Save(pclFormPresentation)
```

## Sintaxis

### Ahorro en un archivo de datos

`<Table control>.Save()`
---

**`<Table control>: (Nombre del control)`**

Nombre de la control Tabla basada en un archivo de datos que se va a utilizar.


<a name="SYNTAX2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
### Guardar en un objeto (sintaxis disponible principalmente para el MVP)

`<Table control>.Save(<Object name>)`
---

**`<Table control>: (Nombre del control)`**

Nombre de control Tabla en Variable para usar.

**`<Object name>: (Cadena de caracteres)`**

Nombre del objeto a utilizar. El contenido del actual fila que se encuentra en el control Tabla se guardará en los miembros del objeto associated.

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000261"></a>

La función **&lt;Tabla&gt;.Save** puede utilizarse en: 

- una control Tabla basada en un archivo de datos o una control Tabla TreeView basada en un archivo de datos. 

- un control de selección única o de selección múltiple. Para un control de selección múltiple, sólo se modifica la última Record seleccionada (la Record correspondiente a la última fila actual).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) una control Tabla en la Variable (sintaxis 2). 



<a name="NOTE0_2"></a>


### Actualizando el archivo de datos relacionado a la Mesa o control Tabla TreeView
<a name="actualizando_archivo_datos_relacionado_mesa_control_tabla_treeview_ELTPARAGRAPHE000281"></a>

La función **&lt;Tabla&gt;.Save** guarda la fila actual del control en el archivo de datos relacionado con el control. Los elementos se asignan automáticamente con los valores de las columnas de associated en el fila actual.

Cuando se realiza una modificación, **&lt;Tabla&gt;.Save** sustituye los valores anteriores por los nuevos. Los elementos que no se encuentran en la control no se modifican.

**Atención**:

- La función **&lt;Tabla&gt;.Save** solo gestiona el archivo de datos asociado al control. Si algunas columnas son relacionado a elementos encontrados en otros ficheros de datos, estos ficheros de datos no se gestionan automáticamente.

- Las columnas "imagen" no se guardan con la función **&lt;Tabla&gt;.Save**. Para modificar el contenido de la nota associated con el Image (caso raro), utilice [&lt;Fuente&gt;.LinkMemo](../WDLang4/1000024194.md).

- En las columnas de tipo Casilla de verificación, la función **&lt;Tabla&gt;.Save** se llama de manera implícita y no se puede desactivar. En cualquier caso, la modificación del valor de la casilla de verificación se aplicará a el archivo de datos.



<a name="NOTE0_3"></a>


### Entrada en cascada y actualización del archivo de datos de relacionado
<a name="entrada_cascada_actualizacion_del_archivo_datos_relacionado_ELTPARAGRAPHE000314"></a>

- **Si la entrada en cascada está activada**, el archivo de datos relacionado con el control Tabla o Tabla TreeView se modifica automáticamente cuando el usuario pasa a la siguiente fila (con las funciones [&lt;Table&gt;.Add](../WDLang1/1000023600.md), [&lt;Tabla&gt;.AddLine](../WDLang1/1000023404.md), [.Insert](../WDLang1/1000024260.md), [&lt;Tabla&gt;.InsertLine](../WDLang1/1000023407.md), [&lt;Tabla&gt;.Modify](../WDLang1/1000024273.md) y [&lt;Tabla&gt;.ModifyLine](../WDLang1/1000024275.md)). La función **&lt;Tabla&gt;.Save** no es útil.

- **Si la entrada en cascada no está activada**, el archivo de datos relacionado con el control Tabla o Tabla TreeView no se modifica. Para escribir la fila añadida o modificada en el fichero de datos relacionado, hay que utilizar [&lt;Tabla&gt;.Save](../WDLang1/1000024249.md) después de la función de adición o modificación.




Observaciones:

- El [nuevo](../Proprietes/2510073.md) Property puede utilizarse para determinar si el fila actual acaba de ser creado por [&lt;Table&gt;.Add](../WDLang1/1000023600.md), [&lt;Tabla&gt;.AddLine](../WDLang1/1000023404.md), [.Insert](../WDLang1/1000024260.md) y [&lt;Tabla&gt;.InsertLine](../WDLang1/1000023407.md) (Nuevo = True).

- El [Modified](../Proprietes/2510080.md) Property se utiliza para determinar si un usuario ha introducido datos en un fila (Modificado = True).


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) La entrada en cascada no está disponible en controles Tabla basados en un archivo de datos con acceso directo. Esta opción está disponible para los controles de tabla con fuente de datos en memoria.<a name="NOTE0_4"></a>


### Re-visualización de la fila que se ha grabado
<a name="revisualizacion_fila_que_grabado_ELTPARAGRAPHE000399"></a>

Después de utilizar la función **&lt;Tabla&gt;.Save**, la fila guardada puede "desaparecer". Esta "desaparición" se debe al valor de la clave de búsqueda de la Record añadida o modificada.

Para evitar este problema, vuelva a visualizar la Tabla o control Tabla TreeView con [&lt;Table&gt;.Display](../WDLang1/1000023505.md).
<a name="NOTE0_5"></a>


### Bloqueo y actualización de el archivo de datos
<a name="bloqueo_actualizacion_archivo_datos_ELTPARAGRAPHE000418"></a>

Si el archivo de datos está bloqueada en el proceso actual, **&lt;Tabla&gt;.Save** guarda el fila actual y desbloquea el archivo de datos.
<a name="NOTE0_6"></a>


### Duplicados e integridad referencial
<a name="duplicados_integridad_referencial_ELTPARAGRAPHE000430"></a>

- Si se habilita la gestión de duplicados ([&lt;Fuente&gt;.SetDuplicates](../WDLang4/1000024985.md)), se debe comprobar [HErrorDuplicates](../WDLang4/3044077.md) tras la ejecución de **&lt;Tabla&gt;.Save** para comprobar la unicidad de la clave.

- Si la gestión de la integridad referencial está activada ([&lt;Variable Connection&gt;.SetIntegrity](../WDLang4/1000023960.md)), se debe comprobar [HErrorIntegrity](../WDLang4/3044086.md) después de la ejecución de **&lt;Tabla&gt;.Save** para comprobar la integridad referencial.



<a name="NOTE0_7"></a>


### Utilice &lt;Tabla&gt;.Save para reducir el número de líneas de código
<a name="utilice_docparampagetitleshort_para_reducir_numero_lineas_codigo_ELTPARAGRAPHE000461"></a>

La función **&lt;Tabla&gt;.Save** reduce varias líneas de código. Esta única Line permite evitar asignar las variables de los artículos uno por uno.

Por ejemplo, el siguiente código:


```wl
Customer.LastName = LastNameColumn
Customer.FirstName = FirstNameColumn
Customer.Address = AddressColumn
Customer.ZC = ZCColumn
Customer.City = CityColumn
HAdd(Customer)
```
se sustituye por la siguiente Line:

```wl
TABLE_Customer.Save()
```




## InReportsAndQueriesMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica si el código se ejecuta al probar un reporte o consulta en Reports & Queries.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Code run during the test of a report or 
// query in Reports & Queries?
IF InReportsAndQueriesMode() = True THEN
	// Test mode in Reports & Queries
	// Process to perform
	...
ELSE
	// Different mode
	// Process to perform
	...
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = InReportsAndQueriesMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecuta el código al probar un reporte o consulta en Reports & Queries,

- <u><u><u><u>False</u></u></u></u> en caso contrario.  








<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### GO desde Reports & Queries
<a name="desde_reports_queries_ELTPARAGRAPHE000099"></a>

Cuando se ejecuta la prueba de un reporte desde "Reports & Queries" (botón "GO"), las funciones [InTestMode](../WDLang1/3013011.md) y **InReportsAndQueriesMode** devuelven <u><u><u><u>True</u></u></u></u>.

Se recomienda reemplazar la prueba estándar:


```wl
IF InTestMode() = True THEN
	Trace("...")
END
```
por la siguiente prueba:


```wl
IF InTestMode() = True AND InReportsAndQueriesMode() = False THEN
	Trace("...")
END
```

<a name="NOTE0_2"></a>


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000249"></a>

La función **InReportsAndQueriesMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InReportsAndQueriesMode** pueden compartirse entre varios productos sin generar errores de compilación.

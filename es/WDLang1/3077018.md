
## Sincronizar subprocesos mediante eventos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000244"></a>
Los eventos permiten sincronizar los diferentes subprocesos de una aplicación. Por lo tanto, un subproceso puede esperar a que se ejecute otro subproceso.

WLanguage propone dos modos para gestionar eventos:

- [gestión simple](#NOTE2_1): sincronización entre dos subprocesos.

- [gestión avanzada](#NOTE3_1): sincronización entre varios subprocesos (más de dos).








<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Gestión simple de eventos
<a name="gestion_simple_eventos_ELTTEXTE000268"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000056"></a>

Dos subprocesos se ejecutan en paralelo (un subproceso principal y uno secundario, por ejemplo). Uno de los subprocesos espera una acción específica del segundo subproceso para poder ejecutarse.
<a name="NOTE2_2"></a>


### Ejemplo: Aplicación con corrector ortográfico
<a name="ejemplo_aplicacion_con_corrector_ortografico_ELTPARAGRAPHE000063"></a>

Cuando un usuario pulsa la BARRA ESPACIADORA en un control Campo de entrada, el corrector ortográfico se activa automáticamente para revisar la palabra anterior.

En este caso, la revisión ortográfica se gestiona en un subproceso secundario.

Cada vez que se pulsa la BARRA ESPACIADORA, el subproceso principal envía una señal al subproceso secundario para iniciar el corrector ortográfico.

El código para este tipo de aplicación es el siguiente:

- Código del subproceso principal:
	
	```wl
	IF Right(EDT_Edit1, 1) = " " THEN
		ThreadSendSignal("ThreadSpellCheck")
	END
	```





- Código del subproceso secundario (ThreadSpellCheck):
	
	```wl
	LOOP
		IF ThreadWaitSignal() = True THEN
			 StartSpellCheck()
		END
	END
	```




<a name="NOTE2_3"></a>


### Funciones WLanguage
<a name="funciones_wlanguage_ELTPARAGRAPHE000084"></a>Las siguientes funciones permiten gestionar eventos de forma simple:



|   |   |
| --- | --- |
| [ThreadSendSignal](../WDLang1/3077021.md) | El hilo actual envía una señal al hilo que está en espera para desbloquearlo. |
| [ThreadWaitSignal](../WDLang1/3077022.md) | Bloquea el hilo actual hasta que reciba una señal de otro hilo. |




<a name="NOTE2_4"></a>


### Observación
<a name="observacion_ELTPARAGRAPHE000092"></a>

Los semáforos y los eventos son objetos del sistema identificados por un nombre. Se pueden compartir entre todas las aplicaciones que se ejecutan en un equipo.

Por lo tanto, dos aplicaciones (o dos instancias de la misma aplicación) utilizan los mismos objetos si utilizan el mismo nombre.

Para crear nombres únicos, utilice la función [Instance](../WDLang1/3014019.md) para definir el nombre deseado.

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) En Java, los semáforos, secciones críticas y eventos son únicos para la aplicación que los creó. No se pueden compartir entre varias aplicaciones.

## Gestión avanzada de eventos
<a name="gestion_avanzada_eventos_ELTTEXTE000326"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000112"></a>

La gestión avanzada de eventos se utiliza cuando se establece una comunicación entre varios subprocesos (más de 2). Algunos subprocesos esperan a que el subproceso principal realice una tarea. Cuando el subproceso principal realiza esta tarea, envía una señal a todos los subprocesos secundarios.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=SIGNAUX.gif)

<a name="NOTE3_2"></a>


### Implementación
<a name="implementacion_ELTPARAGRAPHE000121"></a>

Existen dos métodos para implementar una gestión avanzada de eventos: 

- Utilizar las variables AutomaticEvent y ManualEvent. 

- Utilizar eventos con nombre (método recomendado en la gestión de opciones para compartir eventos). 




![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) **Método 1: Utilizar las variables AutomaticEvent y ManualEvent.**
Las siguientes etapas permiten implementar la gestión avanzada de eventos:

1. Crear una variable de tipo [AutomaticEvent](../WDLang1/1000021270.md) o [ManualEvent](../WDLang1/1000021275.md). Este evento está cerrado de forma predeterminada. El evento se destruye automáticamente al salir del alcance de la variable correspondiente.

2. Esperar un evento (función [EventWait](../WDLang1/3077007.md)).

3. Sincronizar subprocesos con las funciones [EventOpen](../WDLang1/1000021271.md) y [EventClose](../WDLang1/1000021272.md).




**Método 2: Utilizar eventos con nombre (método recomendado en la gestión de opciones para compartir eventos)**
Las siguientes etapas permiten implementar la gestión avanzada de eventos:

1. Crear un evento (función [EventCreate](../WDLang1/3077009.md)). Este evento está cerrado de forma predeterminada.

2. Esperar un evento (función [EventWait](../WDLang1/3077007.md)).

3. Sincronizar subprocesos: 

	- con la función [EventChange](../WDLang1/3077017.md): el evento se abre. Todos los subprocesos en espera se desbloquean y el evento se vuelve a cerrar automáticamente (operación predeterminada).

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) con las funciones [EventOpen](../WDLang1/1000021271.md) y [EventClose](../WDLang1/1000021272.md).




4. Destruir el evento (con la función [EventDestroy](../WDLang1/3077025.md)).




## &lt;Tabla&gt;.expandir (función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Se expande:

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) una rama que estaba previamente colapsada en un control Tabla TreeView. Los nodos "niños" son Visible en el control.

- una rotura que previamente se colapsó en un control Tabla. Los elementos de ruptura son Visible en el control. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) una ventana detallada para un fila. Para obtener más información, consulte [Gestionar los detalles de una fila de un control Tabla](../WDChamp/1013200.md). 




**Recordatorio**: En tiempo de ejecución, un nodo se expande cuando el nombre del nodo va precedido de "-".

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las pausas no están disponibles para controles Tabla en esta versión.






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Expands the selected branch in the TreeView Table control
TVT_ScheduleTable.Expand()
```
<a name="Example2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Expands the entire TreeView Table control
TVT_ScheduleTable.Expand(1)
// Equivalent to: TVT_ScheduleTable.ExpandAll()
```
<a name="Example3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Expands the specified branch in the TreeView Table control
TVT_ScheduleTable.Expand("MyRoot" + TAB + "MyBranch")
```

## Sintaxis
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
### Control Tabla TreeView: Expandir una rama mientras se especifica el número de fila

`<TreeView Table control>.Expands([<Row number>])`
---

**`<TreeView Table control>: (Nombre del control)`**

Nombre del control Tabla TreeView a manipular.

**`<Row number>: (Entero opcional)`**

Número de la fila que contiene la rama a desglosar. La rama que se encuentra en la fila actual se expande si no se especifica este parámetro.


<a name="SYNTAX2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
### Control Tabla TreeView: Expandir una rama, especificando su nombre

`<TreeView Table control>.Expands([<Branch name>])`
---

**`<TreeView Table control>: (Nombre del control)`**

Nombre del control Tabla TreeView a manipular.

**`<Branch name>: (Cadena de caracteres opcional)`**

Nombre de la sucursal a expandir. Este parámetro tiene el siguiente formato:


```txt
"<Root name>" + TAB + ["<Name of 1st node>" + TAB + ...
["<Name of 2nd node>" + TAB + [...]]]"<Leaf name>"
```


Si este parámetro no corresponde a una rama existente, se produce un error WLanguage.

La rama que se encuentra en la fila actual se expande si no se especifica este parámetro.


<a name="SYNTAX3"></a>

### Control Tabla con rupturas: Expandiendo una ruptura

`<Table control>.Expands([<Row number> [, <Name of break>]])`
---

**`<Table control>: (Nombre del control)`**

Nombre de control Tabla a manipular.

**`<Row number>: (Entero opcional)`**

Número de la fila cuyo descanso se ampliará. Este número debe corresponder al número de la fila actual, o al número de la primera fila de la pausa a ampliar. Si no se especifica este parámetro, se colapsa la ruptura de la fila actual.

**`<Name of break>: (Cadena de caracteres opcional)`**

Nombre de la pausa a ampliar. Este parámetro debe especificarse en rupturas anidadas. Se usa para definir la ruptura que se expandirá. Por Default, no se expande ninguna ruptura si las rupturas están anidadas.


<a name="SYNTAX4"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
### Tabla y tabla TreeView: Expandiendo una ventana detallada

`<Table control>.Expands([<Row number>])`
---

**`<Table control>: (Nombre del control)`**

Nombre del control Tabla o Tabla TreeView a manipular.

**`<Row number>: (Entero opcional)`**

Número de la fila cuya info detallada debe desglosarse. Si no se especifica este parámetro, se amplían los detalles de la fila actual..


<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000346"></a>

La función **&lt;Tabla&gt;.expandir** puede utilizarse en: 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) un control Tabla TreeView.

- un control Tabla rellenado mediante programación. 

- un control Tabla con fuente de datos en memoria.  




Esta función no se puede utilizar en controles Tabla archivo de datos (acceso directo).
<a name="NOTE0_2"></a>


### Sucursal para expandir
<a name="sucursal_para_expandir_ELTPARAGRAPHE000372"></a>

La función **&lt;Tabla&gt;.expandir** no tiene ningún efecto:


|   |   |
| --- | --- |
| Control Tabla TreeView | Control Tabla con rupturas |
| <br><br>- si no se encuentra la rama que va a colapsar.<br><br>- si la rama que colapsa es una hoja.<br><br>- si la rama que va a colapsar ya está colapsada.<br><br><br> | <br><br>- si el control Tabla no tiene rupturas.<br><br>- si la control Tabla contiene varias rupturas y si no se especifica la ruptura a expandir.<br><br>- si no se selecciona ninguna fila en el control Tabla y no se especifica el parámetro &lt;Número de fila&gt;.<br><br><br> |


**Observaciones para el TreeView controles Tabla**: 

- Si &lt;número de Fila&gt; o &lt;nombre de la rama&gt; corresponde a la rama en el primer fila, se expande toda la jerarquía del control Tabla TreeView (equivalente a [&lt;Tabla&gt;.ExpandAll](../WDLang1/1000021834.md)).

- Cuando se llama a **&lt;Tabla&gt;.expandir**, si la rama especificada tiene un Procedure asociado a través del [AddChildDelayed](../Proprietes/1000020927.md) Property, se ejecutará este Procedure. 



<a name="NOTE0_3"></a>


### Banda de selección
<a name="banda_seleccion_ELTPARAGRAPHE000415"></a>

La función **&lt;Tabla&gt;.expandir** no cambia la posición de la banda de selección.
<a name="NOTE0_4"></a>


### Velocidad de ejecución para controles Tabla TreeView
<a name="velocidad_ejecucion_para_controles_tabla_treeview_ELTPARAGRAPHE000427"></a>

La velocidad de ejecución de la sintaxis 1 es más rápida que la de la sintaxis 2.


## SemaphoreStart( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cierra la actual Thread hasta que se abra el semáforo (es decir, hasta que se disponga de un espacio "libre" en la sección protegida).

**Recordatorio**: Los semáforos se utilizan para definir una sección de código protegida que sólo puede ser ejecutada por un número específico de hilos.




<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = SemaphoreStart(<Semaphore name> [, <Maximum timeout>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el hilo recibió la autorización para entrar en la sección de código,

- <u><u><u><u>False</u></u></u></u> en caso contrario (duración máxima agotada).




**`<Semaphore name>: (Cadena de caracteres)`**

Nombre del semáforo a utilizar. Si el semáforo no existe, se crea con las opciones de Default (ver [SemaphoreCreate](../WDLang1/3077012.md))

**`<Maximum timeout>: (Entero o duración opcional)`**

Tiempo máximo de espera en centésimas de segundo. 

- Si al final de la duración especificada, el hilo no ha recibido la autorización, &lt;Resultado&gt; se establece en <u><u><u><u>False</u></u></u></u>. 

- Si no se especifica este parámetro o corresponde a la constante *Infinite*, la espera es infinita.

- Si este parámetro es set a 0, &lt;Resultado&gt; corresponde al estado del semáforo. El Thread no está pendiente. 


Este parámetro puede corresponder a: 

- un número entero que corresponde al número de centésimas de segundo,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, 1 s o 10 ms).

<a name="NOTE0_1"></a>

## Observaciones




### Equivalencia
<a name="equivalencia_ELTPARAGRAPHE000100"></a>


```wl
SemaphoreStart("SemaphoreName")
```


equivale a: 


```wl
CriticalSectionStart("MyCriticalSection")
```


Mediante Default, **SemaphoreStart** se crea un semáforo limitado a un único thread.

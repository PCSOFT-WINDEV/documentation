
## ExplorerRetrieve( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Recupera el número y el nombre de los archivos "soltados" del explorador.

**Observación**: Para get información del explorador, el diálogo debe haber sido inicializado por [ExplorerAccept](../WDLang1/3030015.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003030009&name=explorerretrieve_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
Nb = ExplorerRetrieve(_eve.wparam)
// Retrieve all the files
FOR i = 1 TO Nb
	Trace(ExplorerRetrieve(_eve.wparam, i))
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Recuperando el número de archivos abandonados

`<Result> = ExplorerRetrieve(<Message>)`
---

**`<Result>: (Integro)`**

Número de archivos seleccionados.

**`<Message>: (Entero del sistema)`**

Primer parámetro del mensaje WM_DROPFILES (devuelto por el explorador a la ventana WINDEV o control), que significa _EVE.wParam.  


<a name="SYNTAX2"></a>

### Recuperando el nombre de los archivos abandonados uno por uno

`<Result> = ExplorerRetrieve(<Message> , <Number>)`
---

**`<Result>: (Cadena de caracteres)`**

Nombre y ruta del archivo eliminado.

**`<Message>: (Entero del sistema)`**

Primer parámetro del mensaje WM_DROPFILES (devuelto por el explorador a la ventana WINDEV o control), que significa _EVE.wParam.

**`<Number>: (Integro)`**

Índice del archivo cuyo nombre debe ser recuperado.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Cuando se realiza un "Drag and Drop" entre el explorador y una ventana WINDEV (o control), el explorador envía el mensaje WM_DROPFILES al destino correspondiente. El primer parámetro de este mensaje (_EVE.wParam) utilizado en **ExplorerRetrieve** devuelve el contenido de la operación "Arrastrar y soltar" (número y nombre de los archivos seleccionados).

Observaciones:

- El número y el nombre de los ficheros seleccionados en el explorador no pueden recuperarse si no se realiza una operación de "arrastrar y soltar" entre el explorador y una ventana WINDEV.

- La constante WM_DROPFILES está definida en el fichero Winconst.wl suministrado en el subdirectorio "Personal\\External" y es set a 0x233.




## ChangeStyle (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica el estilo de un control dinámicamente.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Modify the style of EDT_Edit1 control
ChangeStyle(EDT_Edit1, "Edit_No_caption__Display")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = ChangeStyle(<Control used> , <Style to apply> [, <Skin template to apply>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se cambió el estilo,

- <u><u><u><u>False</u></u></u></u> si se produce un error.




**`<Control used>: (Nombre del control)`**

Nombre de control cuyo estilo debe ser modificado.

**`<Style to apply>: (Cadena de caracteres)`**

Nombre del estilo definido en la edición. Este estilo debe ser aplicable al tipo de control especificado.

**`<Skin template to apply>: (Cadena de caracteres opcional)`**

Nombre del plantilla que se va a aplicar (nombre y ruta del archivo "\*.WDY", si es necesario). La extensión no debe ser especificada. 

Si este parámetro corresponde a una cadena vacía ("""):

- el plantilla definido en el editor de la ventana será aplicado por Default. **Atención**: Si una plantilla ha sido dinámicamente set con [CambiarPlantillaSkin](../WDLang1/3013026.md), tendrá prioridad.

- las ventanas ya abiertas mantendrán la plantilla actual.




Si no se especifica este parámetro, se utilizará el archivo plantilla (\*.WDY) associated con el proyecto actual.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### "Archivo ".WDY
<a name="archivo_wdy_ELTPARAGRAPHE000053"></a>

La description de una plantilla se almacena en un archivo ".WDY".. Los archivos ".WDY" correspondientes a las plantillas tema propuestas por WINDEV se proporcionan con WINDEV.

Para aplicar un estilo personalizado o un estilo de proyecto, se debe utilizar la siguiente sintaxis: 

```txt
ChangeStyle(<Control>, <CustomStyle>, <ProjectName.wdy>)
```
 Excepto si se renombró, el. WDY que contiene los estilos de proyecto se llama como el proyecto mismo. 

Observaciones:

- Al crear una plantilla, el archivo ".WDY" se crea automáticamente.

- Para generar el archivo ".WDY" de una plantilla existente: 

	- Abre el archivo "&lt;NombreDePlantillaDePiel&gt;.STY" en WINDEV: en la pestaña "Inicio", en el grupo "General", haga clic en "Abrir". 

	- Guarda este archivo: en la pestaña "Inicio", en el grupo "General", haga clic en "Guardar".






<a name="NOTE0_2"></a>


### Desplegando una aplicación
<a name="desplegando_una_aplicacion_ELTPARAGRAPHE000097"></a>

Al crear el programa instalación de su aplicación, se deben añadir los siguientes elementos en la lista de elementos a instalar:

- el archivo ".WDY" del proyecto.

- el archivo ".WDY" que corresponde a la nueva plantilla aplicada con la función **ChangeStyle**.

- los archivos de imagen utilizados por la nueva plantilla aplicada con la función **ChangeStyle**. Atención: las imágenes utilizadas por las diferentes plantillas de tema deben tener nombres diferentes, de lo contrario estas imágenes no serán modificadas.




**Observación**: Para añadir uno o más elementos a la lista de elementos del proyecto:

1. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Lista de elementos".

2. Seleccione la opción "Lista de elementos del proyecto".

3. Haga clic en el botón "Agregar".

4. Seleccione los archivos que desea añadir y validar.



<a name="NOTE0_3"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000131"></a>

- También tiene la posibilidad de copy el estilo de un control a otro a través de [Estilo](../Proprietes/1000021472.md). 

- Al cambiar de estilo, la posición de los pies de foto no se modifica. De hecho, cambiar la posición de los pies de foto podría provocar modificaciones en el tamaño de la control. El control podría entrar en conflicto con otros control s. 

- Los controles de los botones: 

	- Si el estilo incluye un Image, este Image sobrescribe el Image inicial del botón. 

	- Si el estilo no incluye Image, se mantiene la inicial Image del botón.

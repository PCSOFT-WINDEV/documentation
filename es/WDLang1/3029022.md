
## LoadImage( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Carga una Image en memoria para que pueda ser utilizada posteriormente por [gImage](../WDLang1/3029042.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// The "C:\MyImages\Image.BMP" image includes
// 20 drawings organized in 10 rows and 2 columns.
// No border will be displayed around them.
LoadImage(ImageID, "C:\MyImages\Image.BMP", 10, 2, "C")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = LoadImage(<Image identifier> , <Image name> [, <Number of rows> [, <Number of columns> [, <Border>]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si la imagen se cargó en memoria, 

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Image identifier>: (Integro)`**

Identificador de Image cargado en la memoria (incluido entre 1 y 9). Si este identificador ya se utiliza, el Image cargado reemplaza al Image anterior. Este identificador puede ser utilizado por la función [gImage](../WDLang1/3029042.md).

**`<Image name>: (Cadena de caracteres)`**

Nombre y apellido (o familiar) ruta de Image para cargar. Se puede utilizar una ruta de tipo UNC.

**`<Number of rows>: (Entero opcional)`**

Número de filas de dibujos que se encuentran en la Image (hasta 255 filas).

**`<Number of columns>: (Entero opcional)`**

Número de columnas de dibujos que se encuentran en la Image (hasta 255 columnas).

**`<Border>: (Carácter opcional)`**

"C" si no se debe mostrar Border alrededor de cada dibujo. Por Default, un Border de 1 píxel se muestra alrededor de cada dibujo.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Dibujos numerados
<a name="dibujos_numerados_ELTPARAGRAPHE000056"></a>

Una Image puede contener varios dibujos organizados en filas y columnas. Todos estos dibujos deben tener el mismo tamaño y pueden ser encerrados en un Border de 1 píxel.

Los dibujos están numerados del 1 al &lt;Número de filas&gt; \* &lt;Número de columnas&gt; de izquierda a derecha y de arriba a abajo.

Cuando se añade un dibujo a la Image, la llamada a **LoadImage** tiene que ser modificada para especificar el nuevo número de dibujos horizontal y verticalmente. 

Observaciones: 

- [CargarImagenSheet](../WDLang1/1000018826.md) simplifica la incorporación de dibujos a image. Se recomienda utilizar esta función. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) La función [gImageExists](../WDLang1/1000021860.md) permite saber si se ha cargado una imagen en memoria con la función **LoadImage** o [CargarImagenSheet](../WDLang1/1000018826.md).



<a name="NOTE0_2"></a>




### Límites
<a name="limites_ELTPARAGRAPHE000091"></a>

**LoadImage** no debe utilizarse en el código del proyecto. De hecho, esta función debe usarse en la ventana donde se debe manejar el Image.




## ThreadSendSignal (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La thread actual envía una Event a la thread especificada para desbloquearla. Esta función simplifica la sincronización entre hilos sin tener que implementar una gestión avanzada de eventos. 

**Recordatorio**: 

- Un hilo es un proceso que se ejecuta en paralelo con la aplicación actual (hilo principal). Esto le permite iniciar una tarea en segundo plano (hacer una copia de seguridad, etc.).

- Las funciones [ThreadWaitSignal](../WDLang1/3077022.md) y **ThreadSendSignal** permiten sincronizar los hilos de dos en dos.

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003077021&name=threadsendsignal_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Create an event
EventCreate("Let's go", eventManual, eventClose)
// Run a first thread
ThreadExecute("Thread1", threadNormal, "Thread_First")
// Locks the current thread until the event is obtained
ThreadWaitSignal()

// Run a second thread
ThreadExecute("Thread2", threadNormal, "Thread_Second")
// Locks the current thread until the event is obtained
ThreadWaitSignal()

// Modifies the event to run the threads
EventChange("Let's go", eventOpen)
```

```wl
//----------------------------------------------
PROCEDURE Thread_First()
Trace(dbgInfo(dbgProcess) + " - ThreadSendSignal(threadMain)")
ThreadSendSignal(threadMain)
EventWait("Let's go")
Trace(dbgInfo(dbgProcess) + " - Event received")
```

```wl
//--------------------------------------------
PROCEDURE Thread_Second()
Trace(dbgInfo(dbgProcess) + " - ThreadSendSignal(threadMain)")
ThreadSendSignal(threadMain)
EventWait("Let's go")
Trace(dbgInfo(dbgProcess) + " - Event received")
```

## Sintaxis

### Enviando una señal a una Thread identificada por su nombre

`ThreadSendSignal(<Thread name>)`
---

**`<Thread name>: (Cadena de caracteres)`**

Nombre de la el hilo a la que debe enviarse la Event. Para notificar al hilo principal, utilice la cadena de caracteres "." o la constante *threadMain*. 
No se puede enviar ningún Event al actual thread.  
Este nombre se asigna al ejecutar el hilo (con la función [ThreadExecute](../WDLang1/3077024.md)).

### Enviar una señal a un hilo identificado por una variable de tipo Thread

`ThreadSendSignal(<Thread>)`
---

**`<Thread>: (Variable de tipo Thread)`**

Nombre de la variable [Thread](../WDLang1/1000024879.md) correspondiente a el hilo a la que se debe enviar la señal. 
Para notificar al hilo principal, utilice la cadena de caracteres "." o la constante *threadMain*. 
No se puede enviar ningún Event al actual thread.

## Observaciones


### Sincronizar hilos
<a name="sincronizar_hilos_ELTPARAGRAPHE000073"></a>

Las funciones [ThreadWaitSignal](../WDLang1/3077022.md) y **ThreadSendSignal** permiten sincronizar los hilos de dos en dos. Para sincronizar varios hilos en el mismo evento, se deben utilizar las funciones de gestión de eventos:

- [EventCreate](../WDLang1/3077009.md): Crea un evento.

- [EventWait](../WDLang1/3077007.md): Bloquea el hilo actual y espera el evento.

- [EventChange](../WDLang1/3077017.md): Modifica el estado de un evento.

- [EventDestroy](../WDLang1/3077025.md): Destruye un evento.




Observación: En los mecanismos internos de gestión de colas y pilas, no es necesario sincronizar los hilos (funciones [Enqueue](../WDLang1/1000018323.md) y [Dequeue](../WDLang1/1000018322.md) para las colas, funciones [Empuje](../WDLang1/1000017366.md) y [Pop](../WDLang1/1000017372.md) para las pilas).




## Funciones de gestión de subprocesos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Las siguientes funciones permiten gestionar subprocesos:



|   |   |
| --- | --- |
| [CriticalSection](../WDLang1/1000024373.md) | Habilita un nombre o una sección crítica en Variable en la declaración [Utilización ... EN](../Motscles/1510025.md). |
| [CriticalSectionEnd](../WDLang1/3077005.md) | Marca el final de la sección crítica: el código será ejecutado por otro Thread. |
| [CriticalSectionStart](../WDLang1/3077001.md) | Marca el inicio de una sección crítica: ningún otro Thread podrá ejecutar el código mientras el Thread actual no salga de la sección crítica. |
| [EventChange](../WDLang1/3077017.md) | Modifica el estado de un evento. |
| [eventClose](../WDLang1/1000021272.md) | Cierra una sincronización Event entre varios hilos. |
| [EventCreate](../WDLang1/3077009.md) | Crea un evento. |
| [EventDestroy](../WDLang1/3077025.md) | Destruye explícitamente un Event. |
| [eventOpen](../WDLang1/1000021271.md) | Abre una sincronización Event entre varios hilos. |
| [EventWait](../WDLang1/3077007.md) | Bloquea el hilo actual mientras espera a que se abra el Event especificado. |
| [ExecuteMainThread](../WDLang1/1000019862.md) | Dispara la ejecución de la Procedure especificada en la aplicación principal thread. |
| [ExecuteMainThreadAsynchronous](../WDLang1/1410088684.md) | Ejecuta un procedimiento en el subproceso principal de la aplicación sin esperar a que termine de ejecutarse. |
| [MutexCreate](../WDLang1/1000019022.md) | Crea explícitamente un mutex. |
| [MutexDestroy](../WDLang1/1000019025.md) | Destruye explícitamente un mutex. |
| [MutexEnd](../WDLang1/1000019024.md) | Señales de que el hilo libera el mutex. |
| [MutexStart](../WDLang1/1000019023.md) | Bloquea el actual Thread mientras espera que se libere el mutex. |
| [SemaphoreCreate](../WDLang1/3077012.md) | Crea un semáforo. |
| [SemaphoreDestroy](../WDLang1/3077011.md) | Destruye explícitamente un semáforo. |
| [SemaphoreEnd](../WDLang1/3077010.md) | Permite que uno o más hilos salgan del área protegida por el semáforo. |
| [SemaphoreStart](../WDLang1/3077008.md) | Cierra la actual Thread hasta que se abra el semáforo (es decir, hasta que se disponga de un espacio "libre" en la sección protegida). |
| [ThreadCurrent](../WDLang1/1000019423.md) | Devuelve el nombre de el hilo actualmente en funcionamiento. |
| [ThreadEnd](../WDLang1/1000020874.md) | Finaliza la ejecución de la Thread actual. |
| [ThreadExecute](../WDLang1/3077024.md) | Inicia la ejecución de un hilo secundario. |
| [ThreadMode](../WDLang1/3077028.md) | Cambia el modo de gestión de los hilos. |
| [ThreadPause](../WDLang1/3077029.md) | Pausa el hilo actual por el tiempo especificado. |
| [ThreadPersistent](../WDLang1/1000020584.md) | Hace que un Thread sea persistente. |
| [ThreadPriority](../WDLang1/3077027.md) | Devuelve o modifica el nivel de prioridad de un thread. |
| [ThreadRequestStop](../WDLang1/1000021218.md) | Envía una solicitud para detener un hilo. |
| [ThreadResume](../WDLang1/3077023.md) | Reanuda la ejecución de una thread que fue interrumpida por [thread Suspender](../WDLang1/3077014.md). **Función no recomendada.** |
| [ThreadSendSignal](../WDLang1/3077021.md) | El hilo actual envía una señal al hilo que está en espera para desbloquearlo. |
| [ThreadState](../WDLang1/3077016.md) | Devuelve el estado actual de un thread. |
| [ThreadStop](../WDLang1/3077019.md) | Detiene un hilo secundario. **Función no recomendada.** |
| [ThreadStopRequested](../WDLang1/1000021219.md) | Verifica si se ha enviado una solicitud al hilo que se está ejecutando. |
| [ThreadSuspend](../WDLang1/3077014.md) | Suspende temporalmente la ejecución de la Thread especificada. **Función no recomendada.** |
| [ThreadWait](../WDLang1/3077020.md) | Espera a que finalice la ejecución del hilo especificado. |
| [ThreadWaitSignal](../WDLang1/3077022.md) | Bloquea el hilo actual hasta que reciba una señal de otro hilo. |



También se pueden utilizar los siguientes tipos de variables: 



|   |   |
| --- | --- |
| [AutomaticEvent](../WDLang1/1000021270.md) | El tipo **AutomaticEvent** permite gestionar un evento automático. |
| [ManualEvent](../WDLang1/1000021275.md) | El tipo **ManualEvent** permite manipular un evento manual. |


## TelemetryConfigure( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite conocer y modificar los parámetros de telemetría. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Forces the telemetry data to be sent every half hour
TelemetryConfigure(tcMaximumPeriodCollection, 1800)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Averiguar los parámetros de telemetría

`<Result> = TelemetryConfigure(<Parameter>)`
---

**`<Result>: (Integro)`**

Valor del parámetro especificado. 

**`<Parameter>: (Constante de tipo Integer)`**

Parámetro a recuperar. Este parámetro corresponde a una de las siguientes constantes: 


|   |   |
| --- | --- |
| *tcEnableLog* | Configurar el registro de telemetría. Puede corresponder a una combinación de las siguientes constantes: <br><br>- *tcFileLog*: La telemetría utiliza un archivo de registro. El archivo correspondiente se guarda en el directorio "&lt;user&gt;\\AppData\\Roaming\\&lt;company&gt;\\&lt;application&gt;".. <br>	<br><br>- *tcMemoryLog*: La telemetría utiliza un registro de memoria. <br><br><br> |
| *tcGetMemoryLog* | Devuelve el contenido del registro de memoria. |
| *tcIdleTimeout* | Se utiliza para contar el tiempo de actividad en una ventana: el cronómetro de conteo se detiene automáticamente después de este tiempo de inactividad. <br>Este tiempo de espera es de set a 1 minuto por Default. |
| *tcMaximumPeriodCollection* | Periodicidad máxima para la recogida de los datos de telemetría (en segundos). Los datos se enviarán al final de este período, independientemente del tamaño de los datos recogidos.. <br>Este período es de set a 1 hora de Default. |
| *tcMinimumPeriodCollection* | Periodicidad mínima para la recogida de los datos de telemetría (en segundos). Los datos pueden enviarse al final de este período si se alcanza el tamaño mínimo.. <br>Este período es set a 5 minutos por Default. |
| *tcMinimumSizeCollection* | Tamaño mínimo de los datos recogidos durante el período mínimo de recogida (en kilobytes). <br>Este tamaño es set a 5 MB por Default. |
| *tcNumberSendAttempts* | Número máximo de intentos de enviar datos de telemetría en modo online: <br><br>- 0. Los intentos no se cuentan. Los datos se envían cada vez, incluso si el servidor no responde. <br><br>- 1. Los datos se envían sólo una vez cuando se inicia la aplicación. Si el servidor no responde, no habrá más intentos. <br><br>- otro valor (3 por Default). Le permite a set un número total de intentos dentro de un período determinado para enviar datos. <br><br><br><br><br>Por defecto, tcNumberSendAttempts = 3 y tcMaximumPeriodCollection = 1 por lo que hay 3 horas de intentos. |
| *tcOfflineMode* | Modo de funcionamiento de la telemetría: <br><br>- <u><u><u><u>True</u></u></u></u> (valor predeterminado). La telemetría funciona en modo desconectado. <br><br>- <u><u><u><u>False</u></u></u></u>. La telemetría funciona en modo online (comportamiento de Default antes de la versión 25).. <br><br><br><br><br>Para más detalles sobre el funcionamiento de la telemetría, véase [Telemetría: Modo de envío de datos](../WDLang1/1410086649.md). |
| *tcStatus* | Estado actual de la telemetría. Este status puede corresponder a una de las siguientes constantes: <br><br>- *tsAbandon*: la telemetría se abandona después de demasiados errores de envío. <br><br>- *tsCollect*: la telemetría está activada y se está realizando una recolección. <br><br>- *tsSend*: la telemetría está activada y la información fue enviada. <br><br>- *tsError*: la telemetría está habilitada pero no se ha enviado información al servidor. <br><br>- *tsInactive*: la telemetría es Inactive.<br><br><br> |

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
La función **TelemetryConfigure** puede llamarse antes de la función [TelemetryEnable](../WDLang1/1000021900.md). 

Si se llama a la función **TelemetryConfigure** después de la función [TelemetryEnable](../WDLang1/1000021900.md), los nuevos parámetros se tendrán en cuenta después de un tiempo de espera que corresponde al tiempo mínimo de recolección anterior.




## dbgStartRecording( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia la grabación de un escenario de prueba (también llamado escenario de reproducción).

El escenario de reproducción es un archivo ".wsct" que contiene el escenario de prueba. 

Este archivo puede ser suministrado al diseñador de la aplicación. Para incluir este escenario en las pruebas automatizadas de la aplicación, vaya a la pestaña "Pruebas automatizadas", grupo "Pruebas", amplíe "Nuevo" y seleccione "Importar un escenario grabado en la aplicación".

Para obtener más información, consulte [Escenario creado por el usuario](../Editeurs/2019031.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
dbgStartRecording()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = dbgStartRecording([<Name of scenario file>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se inició la grabación,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Name of scenario file>: (Cadena de caracteres opcional)`**

Ubicación y nombre del archivo de escenario a crear. El nombre y la ubicación del archivo pueden ser: 

- totalmente personalizado. 

- construido con las siguientes palabras clave: 
	


|   |   |
| --- | --- |
| [%ExeDir%] | Directorio del ejecutable, siempre rellenado con el carácter "\\".. Este directorio es equivalente al resultado de [fExeDir](../WDLang1/3036049.md). |
| [%exeName%] | Nombre del ejecutable. |
| [%Fecha%] | Fecha de inicio del escenario (formato AAAAMMDDD). |
| [%Hora%] | Hora de inicio del escenario (formato HHMMSS). |
| [%NombreUsuario%] | Nombre del usuario actual. |
| [%NombreDeLaComputadora%] | Nombre del ordenador. |
| [%DataDir%] | Directorio de los archivos de datos HFSQL especificados por el usuario al instalar la aplicación. Este directorio es equivalente al resultado de [fDataDir](../WDLang1/3036001.md). |
| [%DirUserData%] | Directorio para los datos del usuario actual de la aplicación actual. Este directorio es equivalente al resultado de [fDataDirUser](../WDLang1/3036071.md). |





Por Default, el nombre del archivo del escenario corresponde a: "[%UserDataDir%][%ExeName%]_[%Date%]_[%Time%]". 
Por ejemplo: "C:\\Usuarios\\Doc\\AppData\\Roaming\\Aplicaciones WINDEV\\MyApp\\Myapp_20161023_130812.wsct".. 

Atención: Si utiliza la construcción dinámica de cadenas ("Permitir "[% %]" en las cadenas" en la pestaña "Compilación" del proyecto description), se produce un error de compilación (identificador desconocido). En este caso, cada cadena debe ir precedida de '-%'.. Ejemplo: 

```wl
dbgStartRecording("[%DirUserData%][%ExeName%]_[%Date%]_[%Time%]")
```
se convierte
```wl
dbgStartRecording(-%"[%DirUserData%][%ExeName%]_[%Date%]_[%Time%]")
```




<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- **dbgStartRecording** no tiene efecto si ya se está grabando un escenario.

- [dbgEndRecording](../WDLang1/1000017161.md) se utiliza para finalizar la grabación del escenario.

- La grabación del escenario también se puede iniciar a través del archivo ".wx" que se encuentra junto al ejecutable. Para obtener más información, consulte [Escenario creado por el usuario](../Editeurs/2019031.md).

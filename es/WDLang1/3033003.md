
## eInitCurrency( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade o cambia las características de una moneda en el cambio de moneda tabla.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Modify the caption of the French Franc
eInitCurrency(eFRF, "Franc", eExchangeRate(eFRF))
```



```wl
// Modify a user currency
// Use the CFA Franc
eInitCurrency(17, "CFA Franc", 655.96)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`eInitCurrency(<Currency> , <Currency caption> , <Exchange rate>)`
---

**`<Currency>: (Entero o constante)`**

Número de moneda o associated constante con la moneda (ver el cambio tabla).

**`<Currency caption>: (Cadena de caracteres)`**

Leyenda de la moneda (devuelta por [eCurrencyName](../WDLang1/3033006.md)).

**`<Exchange rate>: (Moneda)`**

Valor de un euro expresado en la moneda correspondiente (valor comprendido entre 0 y 2147). El tipo de cambio de una moneda es devuelto por [Tipo de cambio electrónico](../WDLang1/3033004.md).

<a name="NOTE0_1"></a>

## Observaciones




### Intercambio tabla
<a name="intercambio_tabla_ELTPARAGRAPHE000070"></a>

Una ***tabla de cambio*** se almacena en la memoria. Cada moneda es con associated:

- Un número de moneda utilizado para identificar la moneda por programación.

- Una constante utilizada para identificar la moneda por programación.

- Una leyenda (hasta 7 caracteres): esta leyenda se muestra en la control "Moneda + Euro" si la máscara de entrada lo permite (se indica el signo $).

- Un tipo de cambio. Este tipo de cambio corresponde al tipo de cambio de la moneda en relación con el euro.

### Redefinición de monedas
<a name="redefinicion_monedas_ELTPARAGRAPHE000370"></a>

Al redefinir monedas, le recomendamos que mantenga el orden de las monedas definidas en la tabla de divisas tabla. De hecho, las listas de monedas definidas en los diferentes editores se basan en este orden.

Por lo tanto, si la moneda nº 3 (correspondiente al franco belga) se redefine a través de la programación para que corresponda a la peseta, el editor de la ventana/página seguirá mostrando el franco belga en la tercera posición de las monedas. Sin embargo, en tiempo de ejecución, la tercera moneda será la peseta.
<a name="NOTE0_3"></a>




### Iniciando la moneda reference
<a name="iniciando_moneda_reference_ELTPARAGRAPHE000380"></a>

No se permite inicializar la divisa de referencia (cuyo identificador es 0) con la función **eInitCurrency**.
<a name="NOTE0_4"></a>




### Nuevas monedas
<a name="nuevas_monedas_ELTPARAGRAPHE000390"></a>

Si se integran nuevas divisas al Euro, la función **eInitCurrency** permite tener en cuenta estas nuevas divisas en las aplicaciones mediante las "Divisas usuario" (enumeradas del 16 al 25).

La utilización de estas "Monedas definidas por el usuario" permite acceder a las nuevas monedas en el editor de ventanas y en el editor de páginas (en "Moneda de visualización" y "Moneda almacenada").

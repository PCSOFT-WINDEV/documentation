
## SchedulerTriggerProperties( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Lee las propiedades de un programa de tareas y actualiza la estructura [Activación de tareas programadas](../WDLang1/3089002.md).

**Recordatorio**: Un programa es uno de los eventos que desencadenarán la tarea.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Read the properties of the schedule 1 for MyTask
SchedulerTriggerProperties("MyTask", 1)
Info("The beginning of MyTask task is " + TriggerScheduledTask.BeginDate)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = SchedulerTriggerProperties(<Task name> , <Schedule identifier>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la operación,

- <u><u><u><u>False</u></u></u></u> si se produce un error.




**`<Task name>: (Cadena de caracteres)`**

Nombre de la tarea associated con el programa de recuperación. Este nombre fue definido cuando la tarea fue creada por [SchedulerAddTask](../WDLang1/3089003.md).

**`<Schedule identifier>: (Integro)`**

Identificador del programa a utilizar. Este identificador se definió en el momento de la creación del calendario por parte de [SchedulerAddTrigger](../WDLang1/3089001.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Lectura de un horario
<a name="lectura_horario_ELTPARAGRAPHE000062"></a>

Al leer un horario, la estructura [Activación de tareas programadas](../WDLang1/3089002.md) se rellena automáticamente. La variable **TriggerScheduledTask.Caption** contiene la descripción del horario (es decir, del evento que desencadenará la tarea). Este Variable es de sólo lectura.

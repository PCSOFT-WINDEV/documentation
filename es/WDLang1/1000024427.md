


## InKioskMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica si la aplicación Android está en modo kiosco. 

En este modo, las acciones de los usuarios están limitadas: 

- los usuarios no pueden salir de la aplicación,

- no pueden ejecutar otras aplicaciones,

- no pueden acceder a la configuración del dispositivo,

- no ven ni reciben notificaciones o llamadas.


Para obtener más información, consulte [Aplicación en modo kiosco](../WDLang3/1000024431.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// If the application is in kiosk mode
IF InKioskMode() THEN
	...
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = InKioskMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si la aplicación está en modo kiosco, 

- <u><u><u><u>False</u></u></u></u> en caso contrario.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Modo kiosco simple o completo
<a name="modo_kiosco_simple_completo_ELTPARAGRAPHE000050"></a>

Hay dos modos kiosco disponibles: 

- Modo kiosco simple: cuando se utiliza la función [KioskEnable](../WDLang3/1000024425.md) para cambiar al modo kiosco, se muestra un mensaje para que el usuario confirme si desea cambiar la aplicación al este modo. 

- Modo kiosco completo: la aplicación también está en modo "Device Owner", no se muestra ningún mensaje al usuario. Para saber si la aplicación usa este modo, utilice la función [InDeviceOwnerMode](../WDLang1/1000024428.md). 




Si la aplicación está en modo kiosco simple, la función **InKioskMode** devuelve <u><u><u><u>True</u></u></u></u> solo cuando el usuario valida el mensaje de confirmación para cambiar al modo kiosco. 
<a name="NOTE0_2"></a>


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000206"></a>

La función **InKioskMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InKioskMode** pueden compartirse entre varios productos sin generar errores de compilación.
<a name="NOTE0_3"></a>


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000220"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.

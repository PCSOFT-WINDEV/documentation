


## &lt;Kanban&gt;.Display (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Actualiza el control Kanban o una tarjeta del control Kanban.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
Task.Add()
KANBAN_Tasks.Display()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Kanban control>.Display([<Card>])`
---

**`<Kanban control>: (Nombre del control)`**

Nombre del control Kanban a actualizar. 

**`<Card>: (Variable de tipo kbCard opcional)`**

Nombre de la variable [kbCard](../WDLang1/1410089167.md) que corresponde a la tarjeta que debe actualizarse. Si no se especifica este parámetro, se actualiza todo el control Kanban.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Caso 1: Solo se especifica el control Kanban
<a name="caso_1_solo_especifica_control_kanban_ELTPARAGRAPHE000180"></a>

- Si el control Kanban está enlazado a una fuente de datos (archivo de datos o array WLanguage), el control se actualiza utilizando los datos de esa fuente. 

- Si el control Kanban se manipula mediante programación, se vuelven a ejecutar los eventos "Solicitud para actualizar la visualización" de cada ventana interna (o página).



<a name="NOTE0_2"></a>


### Caso 2: Se especifican el control Kanban y la tarjeta
<a name="caso_2_especifican_control_kanban_tarjeta_ELTPARAGRAPHE000191"></a>

- Si la tarjeta no está en una lista Kanban, la función no tiene ningún efecto. 

- Si el control Kanban está enlazado a una fuente de datos (archivo de datos o array WLanguage), la tarjeta se actualiza utilizando los datos de dicha fuente. 

- Si el control se manipula mediante programación, se vuelve a ejecutar el evento "Solicitud para actualizar la visualización" de la ventana (o página) interna.

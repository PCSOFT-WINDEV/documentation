
## ThreadPersistent( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Hace que un Thread sea persistente.

La aplicación no se detiene mientras se ejecute al menos una Thread persistente (aunque se hayan cerrado todas sus ventanas).

El uso de un Thread persistente le permite: 

- seguir reproduciendo música aunque la ventana esté cerrada. 

- transmitir coordenadas GPS de forma permanente. 



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Start the thread that triggers the reading of a sound track
ThreadExecute("MyThread", threadGlobalContext, ProcPlayMusic)

// Creates the notification to display while playing the sound track
Notif is Notification
Notif.Title = "Playing..."
Notif.ActionClick = ProcStopMusic

// Make the thread persistent to prevent the application from being closed 
// as long as the thread is not ended
ThreadPersistent("MyThread", notif)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = ThreadPersistent(<Thread name> , <Notification to Display>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el hilo se hizo persistente, 

- <u><u><u><u>False</u></u></u></u> si se produce un error. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Thread name>: (Cadena de caracteres)`**

El nombre de el hilo que debe ser persistente. Este nombre se asigna al ejecutar el hilo (con la función [ThreadExecute](../WDLang1/3077024.md)). Esta Thread debe existir y debe utilizar el contexto global de la aplicación. 

**`<Notification to Display>: (Notification Variable)`**

Nombre de la variable [Notification](../WDLang3/1000019441.md) correspondiente al Notification que debe aparecer durante la ejecución del thread persistente. 
Esta Notification no debe ser ya mostrada. Si se está ejecutando un hilo persistente cuando se llama a **ThreadPersistent**, el Notification asociado a este hilo será sustituido por el Notification especificado. Cuando la ejecución de la el hilo termine, la Notification se ocultará automáticamente si no hay ninguna otra thread persistente en curso.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Se debe mostrar una notificación al ejecutar un hilo persistente. Por lo tanto, se informa al usuario de que la aplicación iniciada sigue funcionando aunque no se abra ninguna ventana.. Puede tomar el control de esta aplicación durante un clic en la Notification. 
	La aplicación también puede ser reiniciada durante un clic en el Notification: para ello, especifique el **ActivarAplicación** Property en el [Notification](../WDLang3/1000019441.md) Variable. 

- **Consejo**: No se debe abrir ninguna ventana de un persistente Thread. Las ventanas pueden abrirse desde el procedimiento llamado por el Notification por ejemplo. 

- A partir de Android 10, la optimización del uso de la batería integrada puede interrumpir un thread hecho persistente con **ThreadPersistent**. En este caso, [SysBatteryOptimBackground](../WDLang1/1000026008.md) permite a la aplicación utilizar la batería y también mantiene la persistencia de Thread.




## InDelayedProcedureMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica si el código se ejecuta desde un procedimiento diferido o una tarea programada de WEBDEV Application Server.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF NOT InDelayedProcedureMode() THEN
	...
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = InDelayedProcedureMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si el código actual: 

	- está en una Procedure iniciada por [ExecuteDelayedProcedure](../WDLang2/1000020682.md) o se inicia como tarea programada por el servidor de aplicaciones WEBDEV

	- es llamado directa o indirectamente por un Procedure iniciado por [ExecuteDelayedProcedure](../WDLang2/1000020682.md) o es iniciado como una tarea programada por el Servidor de Aplicaciones WEBDEV.




- <u><u><u><u>False</u></u></u></u> en caso contrario. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000196"></a>

La función **InDelayedProcedureMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InDelayedProcedureMode** pueden compartirse entre varios productos sin generar errores de compilación.

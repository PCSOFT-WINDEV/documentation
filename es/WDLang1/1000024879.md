
## Thread (Tipo de variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **Thread** permite definir todas las características de un subproceso. Las características del subproceso se pueden definir y cambiar mediante diferentes propiedades WLanguage. 

**Observación**: Para obtener más información sobre la declaración de este tipo de variable y el uso de propiedades WLanguage, consulte [Declarar una variable](../Motscles/1514032.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Declare a variable to handle a thread
t is thread
// Build a thread and trigger its execution on a procedure
t = ThreadExecute(Proc, ("First parameter", 2))
```



```wl
// Build a thread
t is Thread(Proc, ("First parameter", 2))
// Trigger the execution of the thread
ThreadExecute(t)
```





## Declaración
<a name="declaracion_ELTTEXTE000282"></a>
<a name="SYNTAX1"></a>

### Declarar un subproceso

`MyVariable is Thread`
---

En este caso, no se ha descrito el subproceso. Se debe asignar a la variable el valor de un subproceso existente o el resultado de una de las funciones de ejecución de subprocesos.


<a name="SYNTAX2"></a>

### Declarar un subproceso con el tipo Thread

`MyVariable is Thread (<Procedure> [, <Parameters> [, <Options>]])`
---

**`<Procedure>: (Cadena de caracteres o variable de tipo Procedure)`**

Procedimiento a ejecutar. Este parámetro puede corresponder a: 

- el nombre del procedimiento a ejecutar. 

- el nombre de la variable de tipo [Procedure](../Motscles/1000018907.md) que corresponde al procedimiento a ejecutar.




**`<Parameters>: (Lista de valores entre paréntesis, separados por comas)`**

Parámetros del procedimiento a ejecutar. La lista de parámetros tiene el siguiente formato: 


```txt
(<Parameter 1>, ..., <Parameter N>)
```
donde: 

- &lt;Parameter 1&gt;: Primer parámetro del procedimiento.

- ...

- &lt;Parameter N&gt;: Enésimo parámetro de procedimiento.




**`<Options>: (Constante opcional de tipo Integer)`**

Modo de inicio del subproceso.


|   |   |
| --- | --- |
| *threadFullCopyHFSQLContext*<br>(Valor predeterminado) | Activa la copy inmediata del contexto HFSQL actual.<br>Se recomienda si el subproceso debe tener en cuenta las posiciones actuales en los archivos y consultas del contexto llamador. |
| *threadGlobalContext* | Fuerza el uso del contexto global del proyecto si el subproceso se ejecuta desde una ventana. El subproceso continuará ejecutándose hasta que se cierre la aplicación.<br>De forma predeterminada, se utiliza el contexto de la ventana, el hilo se detiene al cerrarla. |
| *threadLightCopyHFSQLContext* | Activa la copy inmediata de una parte del contexto HFSQL actual. <br>Se almacenan los siguientes elementos: <br><br>- los directorios en los que se encuentran los archivos de datos HFSQL Classic. <br><br>- Conexiones de HFSQL Client/Server.<br><br><br> |
| *threadNormal* | Inicia el subproceso en modo normal. El contexto HFSQL se copia la primera vez que se utiliza una función HFSQL. |
| *threadSecure* | Inicia una thread en modo seguro. En este modo: <br><br>- se producirá un error de compilación si se accede a los controles en el hilo (o si el Procedure utiliza el atributo "UI"). <br><br>- se lanzará una excepción: <br><br>	- si el hilo accede a los controles en tiempo de ejecución,<br><br>	- si se llama a la función [ThreadStop](../WDLang1/3077019.md).<br><br><br><br><br>- cuando se cierra la ventana que ha desencadenado el hilo, se genera una petición de parada de el hilo (pero el hilo sigue ejecutándose después de cerrar la ventana). <br><br><br> |
| *threadWaitForStart* | Espera a que el subproceso inicie realmente antes de continuar con la ejecución. |



**En este caso, se describe el subproceso y la descripción no podrá modificarse de nuevo.** 
El subproceso no se ejecuta automáticamente, una declaración por sí sola no hará que se ejecute el subproceso. Se debe llamar a una de las funciones de ejecución de subprocesos para iniciar la ejecución del subproceso.

## Observaciones
<a name="NOTE0_1"></a>


### Propiedades específicas de la descripción de variables de tipo Thread
<a name="propiedades_especificas_descripcion_variables_tipo_thread_ELTPARAGRAPHE000114"></a>

Las siguientes propiedades pueden utilizarse para manipular una variable de tipo **Thread**:

| Nombre de la propiedad | Tipo utilizado | Efecto |
| --- | --- | --- |
| denominar | Cadena de caracteres | Nombre del subproceso. <br><br>**Esta propiedad es de solo lectura**. |
| Estado | Constante de tipo Integer | Estado del subproceso: <br><br>- *threadStopRequested*: Se ha solicitado que se detenga el subproceso. <br><br>- *threadRunning*: Se está ejecutando el subproceso. <br><br>- *threadNotExecuted*: No se ejecutó el subproceso. <br><br>- *threadSuspended*: El subproceso está suspendido. <br><br>- *threadEnded*: El subproceso existe pero ha finalizado. <br><br><br><br><br>**Esta propiedad es de solo lectura**. |
| Prioridad | Constante de tipo Integer | Nivel de prioridad actual del subproceso:<br><br>- *PriorityHigh*: Prioridad alta .<br><br>- *PriorityNormal*: Prioridad normal.<br><br>- *PriorityLow*: Prioridad baja.<br><br><br><br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) No disponible. |
| ReturnedValue | Valor devuelto por el hilo. | Atención: <br><br>- Si el hilo sigue en ejecución, el **ReturnedValue** Property bloquea la ejecución hasta que el hilo termine.<br><br>- Si el hilo termina sin un error fatal, la propiedad devuelve el valor o valores de retorno del procedimiento principal de el hilo.<br><br><br><br><br>**Esta propiedad es de solo lectura**. <br><br>Reminder:<br><br>- La propiedad **State** permite conocer el estado del subproceso. <br><br>- Es posible esperar el final de una thread con [ThreadWait](../WDLang1/3077020.md). <br><br><br> |
| StopRequested | booleano | <br><br>- <u><u><u><u>True</u></u></u></u> si se solicitó detener el subproceso,<br><br>- <u><u><u><u>False</u></u></u></u> en caso contrario. <br><br><br>**Esta propiedad es de solo lectura**. |


<a name="NOTE0_2"></a>


### Funciones WLanguage que manipulan variables de tipo Thread
<a name="funciones_wlanguage_que_manipulan_variables_tipo_thread_ELTPARAGRAPHE000200"></a>




|   |   |
| --- | --- |
| [ThreadExecute](../WDLang1/3077024.md) | Inicia la ejecución de un hilo secundario. |
| [ThreadRequestStop](../WDLang1/1000021218.md) | Envía una solicitud para detener un hilo. |
| [ThreadSendSignal](../WDLang1/3077021.md) | El hilo actual envía una señal al hilo que está en espera para desbloquearlo. |
| [ThreadWait](../WDLang1/3077020.md) | Espera a que finalice la ejecución del hilo especificado. |

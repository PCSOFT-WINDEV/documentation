


## Dialog( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Muestra un cuadro de mensaje y devuelve el valor del botón pulsado por el usuario.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=fonctions_dialogue%20-%20HC%20N%B0009.gif)


Observaciones: 

- **Esta ventana de advertencia es bloqueante**. Para continuar ejecutando la aplicación, el usuario debe validar uno de los botones. 
	![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Para no bloquear la aplicación, utilice la función [DialogAsynchronous](../WDLang1/1000025310.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) WINDEV permite aplicar la plantilla del proyecto al cuadro de diálogo. Para obtener más información, consulte [Personalizar cuadros de diálogo](../WDLang1/3021003.md).





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Caution: This example must not be copied directly.
// The following code is generated by the wizard.
SWITCH Dialog("Do you want to save the modifications in the %1? file", MySource)
	// Yes
	CASE 1: 
		Procedure_Save()
END
```
<a name="Example2"></a>

```wl
// Caution: This example must not be copied directly.
// The following code is generated by the wizard.
IF Dialog(StringBuild("Do you want to save the modifications in %1?", ...
		MySource), ["Yes - Save","No"], 1, 2, dlgIconQuestion) = 1 THEN
	Procedure_Save()
END
```

## Sintaxis

### Sintaxis utilizando el asistente

`<Result> = Dialog(<Message ID> [, <Parameter 1> [, <Parameter N>]])`
---

**`<Result>: (Integro)`**

Identifica la respuesta seleccionada por el usuario. Esta respuesta depende del mensaje seleccionado. Los diferentes valores correspondientes a las diferentes respuestas se incluyen automáticamente en los comentarios en el editor de código al seleccionar el mensaje.

**`<Message ID>: (Integro)`**

Identifica la pregunta formulada al usuario así como las diferentes respuestas posibles.

**`<Parameter 1>: (Cadena de caracteres opcional)`**

Si el mensaje seleccionado contiene elementos con parámetros (identificados por %1, %2, etc.), este parámetro se utiliza para dar el valor deseado al primer elemento. Así, el valor de &lt;Parámetro 1&gt; reemplazará %1.
$~ ... |

**`<Parameter N>: (Cadena de caracteres opcional)`**

Si el mensaje seleccionado contiene elementos con parámetros (identificados por %1, %2, etc.), este parámetro se utiliza para dar el enésimo valor deseado al elemento N. Así, el valor de &lt;Parámetro N&gt; reemplazará %N.


<a name="SYNTAX2"></a>

### Sintaxis directa

`<Result> = Dialog(<Question> , <Captions of buttons> [, <Default button> [, <Cancellation button> [, <Icon>]]])`
---

**`<Result>: (Integro)`**

Identifica la respuesta seleccionada por el usuario. Esta respuesta depende del mensaje seleccionado.

**`<Question>: (Cadena de caracteres)`**

Pregunta hecha al usuario.

**`<Captions of buttons>: (Array)`**

Nombre de la Array Variable que contiene las leyendas de los botones.

**`<Default button>: (Integro)`**

Índice del botón seleccionado por Default. El valor predeterminado de este parámetro es 1.

**`<Cancellation button>: (Integro)`**

Índice del botón de cancelación. En Default, este parámetro corresponde al índice del último botón.

**`<Icon>: (Cadena de caracteres o constante entera)`**

Visualización de Icon. Este parámetro puede corresponder a: 

- el ruta de archivo correspondiente al Icon mostrado. 

- una de las siguientes constantes: 
	


|   |   |
| --- | --- |
| *dlgIconError* | Icon representando un error. |
| *dlgIconInfo*<br>(Valor predeterminado) | Icon representando una información. |
| *dlgIconQuestion* | Icon representando una pregunta. |

## Observaciones


### Uso de una sintaxis con asistente
<a name="uso_una_sintaxis_con_asistente_ELTPARAGRAPHE000155"></a>

Para utilizar esta sintaxis, se debe utilizar [el asistente](../WDLang1/3021020.md) propuesto al escribir el nombre de la función en el editor de código (opción "&lt;Asistente&gt;" propuesta por la [entrada predictiva](../Editeurs/2013001.md)). Este asistente permite especificar los parámetros del cuadro de diálogo que se utilizará. Esta función no se puede utilizar si el asistente no crea la ventana de diálogo.

Para abrir el asistente:

1. En el editor de código, escribe "Dialog(".

2. Haga clic en "Asistente".. El asistente se abre. Pase a la etapa siguiente.

3. Seleccione una pregunta existente o cree una nueva pregunta (opción &lt;Haga clic aquí para crear una nueva pregunta&gt;). Pase a la etapa siguiente.

4. Si ha elegido crear una pregunta, se abre una pantalla. Esta pantalla permite introducir los diferentes parámetros de la ventana de diálogo utilizada para realizar la pregunta. Es posible indicar:

	- El texto de la pregunta,

	- La imagen del cuadro de diálogo,

	- Los botones que permiten al usuario responder.




5. Valide la pantalla en la que se definió la pregunta. El código correspondiente se inserta automáticamente en el editor de código. Pulse de nuevo la tecla ENTRAR. Todo el código se agrega en el editor de código. Las características del cuadro de diálogo se pueden modificar mediante el ícono que se encuentra junto al texto de la pregunta.



<a name="NOTE0_2"></a>


### Características del cuadro de diálogo
<a name="caracteristicas_del_cuadro_dialogo_ELTPARAGRAPHE000182"></a>



- El título del cuadro de diálogo corresponde al título de la ventana (o página) actual.
	

- Para modificar o definir el título del cuadro de diálogo, utilice la función [NextTitle](../WDLang1/3038043.md).




- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) La plantilla del proyecto actual se aplica automáticamente al cuadro de diálogo.
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Para que el plantilla del proyecto se aplique a la ventana de diálogo, es necesario:

	- aplicar la plantilla a los cuadros de diálogo ("Personalizar ventanas del sistema (Info, YesNo, Confirm, Dialog)" en la pestaña "Estilo" de la ventana description del proyecto).

	- asegúrese de que la ventana "WinDevMessageBox" se encuentra en el proyecto.


![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) El cuadro de diálogo que se muestra utiliza el estilo del sistema (no se aplica la plantilla del proyecto). 




![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Para personalizar este cuadro de diálogo** (y todos los cuadros de diálogo del sistema en la aplicación), seleccione la opción "Personalizar ventanas del sistema (Info, YesNo, Confirm, Dialog)" en la pestaña "Estilo" de la descripción del proyecto. Para obtener más información, consulte [Personalizar cuadros de diálogo](../WDLang1/3021003.md).


## EmulateDialog( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Emula la siguiente llamada a [coloquio](../WDLang1/3021015.md) y pulsa automáticamente el botón pasado como parámetro.

**Atención**: Esta función sólo puede utilizarse cuando se ejecuta una prueba automatizada. En este caso, la prueba comprueba si el diálogo se llama y responde automáticamente. Esta función no tiene ninguna acción cuando no se utiliza con las pruebas de automatic.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Automatically close the next Dialog message box
// The answer on which the click will be performed is "Save" (return value = 1)
EmulateDialog(1)

// Process 
// ... 
SWITCH Dialog("Do you want to save the modifications?")
	// Save
	CASE 1
	// ...

	// Don't save
	CASE 2
	// ...
END
// The message is automatically closed via a click performed on "Save"
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`EmulateDialog(<Return value>)`
---

**`<Return value>: (Integro)`**

Valor del botón en la ventana Diálogo en el que se debe realizar el clic.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Comprobación de la función
<a name="comprobacion_funcion_ELTPARAGRAPHE000037"></a>

La existencia de una función de emulación ([EmulateWarning](../WDLang1/1000009001.md), [EmulateError](../WDLang1/1000009005.md), [EmulateInfo](../WDLang1/1000009006.md), ...) se comprueba durante la llamada a la función de diálogo asociada ([Advertencia](../WDLang1/3021009.md), [Error](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ...).
<a name="NOTE0_2"></a>




### Diferencia de clase de mensaje
<a name="diferencia_clase_mensaje_ELTPARAGRAPHE000063"></a>

El orden de las funciones [EmularXXX](../WDLang1/1000009002.md) debe respetar el orden de visualización de [Error](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ... De lo contrario, se producirá un error fatal.

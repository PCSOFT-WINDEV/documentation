


## InPHPSimulatorMode (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica si el código se ejecuta en modo simulador PHP.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Code run in PHP simulator? 
IF InPHPSimulatorMode() = True THEN
	// Specific code
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = InPHPSimulatorMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si la ejecución se realiza desde el modo simulador PHP,

- <u><u><u><u>False</u></u></u></u> en caso contrario.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000222"></a>

La función **InPHPSimulatorMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InPHPSimulatorMode** pueden compartirse entre varios productos sin generar errores de compilación.
<a name="NOTE0_2"></a>


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000236"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.




## Animación de controles
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000189"></a>
Usted tiene la capacidad de animar los controles que se encuentran en una ventana.

Esta animación se puede configurar:

- **en las opciones del proyecto**: Las opciones del proyecto se utilizan para animar algunas operaciones realizadas en algunos tipos de controles. Tienes la habilidad de animar:

	- el desplazamiento realizado en las filas de tabla o en las filas de los garfios,

	- el cambio de mes en un control Calendario,

	- la característica OVI (Oscurecimiento de Ventanas Inactivas),

	- el cambio de avión,

	- el cambio de pestaña.


Si se definió una animación para el proyecto, esta animación se utilizará para todos los controles correspondientes que se encuentren en el proyecto.. En este caso, las animaciones asociadas al control pueden ser modificadas para cada control (pestaña"Estilo" del control).

- **por programación**: Las animaciones avanzadas utilizadas para gestionar el cambio de estado de un control se pueden definir mediante programación. Usted tiene la capacidad de animar la apariencia de los controles en una ventana, por ejemplo.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000213"></a>
![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) 

### Configurar la animación de los controles a nivel de proyecto
<a name="configurar_animacion_los_controles_nivel_proyecto_ELTPARAGRAPHE000051"></a>

Configurar la animación de los controles:

1. Abra la ventana de descripción del proyecto: en la pestaña "", en el grupo "", haga clic en "".

2. En la pestaña "Avanzado", haga clic en "Animación de controles".

3. En la ventana que se muestra:

	- Seleccione la opción "Activar animación de controles en el proyecto" para que se tenga en cuenta la animación de los controles.

	- Para cada tipo de control, defina la animación por defecto utilizada para los diferentes eventos relacionado al control. La duración de la animación se puede configurar para cada evento (botón"Parámetros"). La duración de una animación no debe exceder 1 segundo. 




4. Validar.  



<a name="NOTE2_2"></a>
![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) 

### Modificación de los parámetros de animación para un control específico
<a name="modificacion_los_parametros_animacion_para_control_especifico_ELTPARAGRAPHE000088"></a>

Por defecto, la animación de los controles corresponde a la animación definida a nivel de proyecto.. Sin embargo, tiene la posibilidad de modificar la animación para un control de ventana.

Para modificar la animación de un control: 

1. Abra la ventana de descripción del control a modificar ("Descripción" en el menú contextual).

2. En la pestaña "Estilo", seleccione el elemento "Control animación".

3. Para cada evento que puede ser animado, elija (si es necesario) la animación y los parámetros. La opción "Definido por el proyecto" permite utilizar las opciones de animación definidas en la descripción del proyecto.

4. Validar.

## Configuración de la animación de los controles por programación
<a name="configuracion_animacion_los_controles_por_programacion_ELTTEXTE000243"></a>


### Animar una secuencia de acciones
<a name="animar_una_secuencia_acciones_ELTPARAGRAPHE000104"></a>

Tiene la capacidad de animar una secuencia de acciones realizadas en los controles.. El principio es el siguiente:

1. Usando [AnimationPrepare](../WDLang1/1000017386.md)

2. Modificación de los controles por programación (llenado, desplazamiento, cambio de plano, etc.). Estas modificaciones no se visualizan. Estas modificaciones se mostrarán cuando se utilice [AnimationPlay](../WDLang1/1000017387.md).

3. Usando [AnimationPlay](../WDLang1/1000017387.md). Esta función se utiliza para pasar del estado "antes de la modificación" al estado "después de la modificación" mediante la reproducción de una animación.



<a name="NOTE3_2"></a>


### Funciones para gestionar la animación de los controles
<a name="funciones_para_gestionar_animacion_los_controles_ELTPARAGRAPHE000123"></a>Las funciones son las siguientes:



|   |   |
| --- | --- |
| [AnimationEnabled](../WDLang1/1000017399.md) | Habilita o deshabilita las animaciones en los controles. |
| [AnimationMinFPS](../WDLang1/1000017400.md) | Configura el número mínimo de imágenes por segundo para que las animaciones (efectos de transición) estén habilitadas en los controles. |
| [AnimationPlay](../WDLang1/1000017387.md) | Reproduce una animación cuando se modifica un control, un grupo de control s o una ventana. |
| [AnimationPlayOnControlProperty](../WDLang1/1000017398.md) | Anima la modificación de una propiedad entera entre 2 valores. |
| [AnimationPrepare](../WDLang1/1000017386.md) | Prepara la animación en una control, un grupo de control s o una ventana. |





Se utilizan varias funciones para reproducir una animación cuando cambia el estado de un control (o grupo de controles):



|   |   |
| --- | --- |
| [ControlEnabled](../WDLang1/1000017377.md) | Activa una control o un grupo de control s en una ventana. Una animación puede ser set en los control s durante esta operación. |
| [ControlGrayed](../WDLang1/1000017375.md) | Grises un control o un grupo de control s. Una animación puede ser set en los controles durante esta operación. |
| [ControlInvisible](../WDLang1/1000017369.md) | Hace un control (o un grupo de controles) Invisible en una ventana. Una animación puede ser set en los controles durante esta operación. |
| [ControlVisible](../WDLang1/1000017338.md) | Hace un control (o un grupo de controles) Visible en una ventana. Durante esta operación, se puede realizar una animación en los controles. |




<a name="NOTE3_3"></a>


### Observación
<a name="observacion_ELTPARAGRAPHE000134"></a>

Para evitar la ralentización de la apertura de la ventana, las animaciones de los controles así como de [AnimationPrepare](../WDLang1/1000017386.md) y [AnimationPlay](../WDLang1/1000017387.md) no tienen ningún efecto en el paso de apertura de la ventana (es decir, durante el código de inicialización de la ventana).

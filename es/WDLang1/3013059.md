
## AutomationEvent( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Bifurca una Procedure a la Event de un objeto de automatización.
<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Objeto sin interfaz description

`AutomationEvent(<Procedure or method> , <Automation object> , <Event>)`
---

**`<Procedure or method>: (Cadena de caracteres)`**

Nombre del procedimiento o método a ejecutar cuando se produce el evento.

**`<Automation object>: (Automation object)`**

Nombre de la variable del objeto de automatización que contiene el objeto en el que se ramificará el Event.

**`<Event>: (Cadena de caracteres)`**

Nombre Event.  


<a name="SYNTAX2"></a>

### Objeto con interfaz description

`AutomationEvent(<Procedure or method> , <Automation object> , <Interface> , <Event>)`
---

**`<Procedure or method>: (Cadena de caracteres)`**

Nombre del procedimiento o método a ejecutar cuando se produce el evento.

**`<Automation object>: (Automation object)`**

Nombre de la variable del objeto de automatización que contiene el objeto en el que se ramificará el Event.

**`<Interface>: (Cadena de caracteres)`**

Identificador único de la interfaz de automatización que gestiona el Event. Este identificador tiene el siguiente formato "{XXXXXXXXXXX-XXXXXX-XXXX-XXXXXXXXXXXXXXXXXX}".

**`<Event>: (Integro)`**

Un número entero que corresponde al número de Event en la interfaz. Este número se encuentra en la description del objeto de automatización (archivo IDL): este número de función debe utilizarse en la interfase de eventos.




## fFileName (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el nombre de un archivo externo actualmente abierto. Este archivo fue abierto por una de las siguientes funciones: [fOpen](../WDLang1/3036036.md), [fCreate](../WDLang1/3036017.md) o [fOpenTempFile](../WDLang1/1000017289.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyTestFileID is int 
FileName is string 
MyTestFileID = fCreate("C:\temp\TestCreation.txt")
FileName = fFileName(MyTestFileID)
Info("The file " + fExtractPath(FileName, fFile + fExtension) + ...
	" was created in "+ fExtractPath(FileName, fDrive + fDirectory))
```



```wl
FileName is string 
MyFile is DiskFile
MyFile = fCreate("C:\temp\TestCreation.txt")
FileName = fFileName(MyFile)
Info("The file " + fExtractPath(FileName, fFile + fExtension) + ...
	" was created in "+ fExtractPath(FileName, fDrive + fDirectory))
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = fFileName(<File used>)`
---

**`<Result>: (Cadena de caracteres)`**

Nombre y ruta completa del archivo externo. 
&lt;Resultado&gt; es una cadena de caracteres del mismo tipo que la que se pasó a las funciones [fOpen](../WDLang1/3036036.md), [fCreate](../WDLang1/3036017.md) or [fOpenTempFile](../WDLang1/1000017289.md): 

- Si estas funciones manejan una cadena de caracteres Ansi, &lt;Result&gt; estará en formato Ansi. 

- Si estas funciones manejan una cadena de caracteres Unicode, &lt;Result&gt; estará en formato Unicode.




**`<File used>: (Entero o DiskFile Variable)`**



- Identificador del archivo a utilizar, definido por [fOpen](../WDLang1/3036036.md), [fCreate](../WDLang1/3036017.md) o [fOpenTempFile](../WDLang1/1000017289.md).

- Nombre de la variable [DiskFile](../WDLang1/1410088883.md) inicializada con [fOpen](../WDLang1/3036036.md), [fCreate](../WDLang1/3036017.md) o [fOpenTempFile](../WDLang1/1000017289.md).

### Modo de funcionamiento en Windows Vista (y posterior)
<a name="modo_funcionamiento_windows_vista_posterior_ELTPARAGRAPHE000290"></a>

Si esta función no funciona correctamente en Windows Vista (y posterior), compruebe si el archivo o directorio utilizado no está en ninguno de los directorios del sistema (directorio de Windows o directorio "Archivos de programa").

**En Windows Vista (y posteriores), con el [Mecanismo UAC](../Editeurs/2025014.md) (Cuenta de usuario control) habilitado**, debe tener privilegios de administrador para manipular y/o modificar los archivos o directorios en los directorios del sistema (directorio de Windows o directorio "Archivos de programa").

**Consejo de programación**: A manipular y/o modificar los archivos o directorios sin privilegios de administrador, debería:

- evite escribir en el directorio de Windows o en el de "Archivos de programa",

- utilizar el directorio del sistema de la aplicación (devuelto por [SysDir](../WDLang1/3073026.md) con la constante *srAppDataCommun*, por ejemplo).

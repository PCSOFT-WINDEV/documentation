
## RequestUpdateUI (Función)

	<IMG style="float:left;margin-right:10px;" src="https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=warning_big.png"><B>Advertencia</B><BR>

<a name="Use"></a>
<a name="description"></a>
Solicita la ejecución del evento "Solicitud para actualizar la visualización" (y devuelve el control **sin esperar a que se ejecute el evento**).

**Observaciones**: 

- El evento "Solicitud para actualizar la visualización" está disponible para ventanas, ventanas internas, páginas, páginas internas, modelos de controles y supercontroles.

- Si se llama varias veces a esta función, se ignorarán las llamadas posteriores que tengan los mismos parámetros que las llamadas que aún no se han ejecutado.

- Si se llama a la función en el hilo principal, el evento "Solicitud para actualizar la visualización" se ejecutará después del proceso actual.

- Si se llama a la función en un hilo secundario, el evento "Solicitud para actualizar la visualización" se ejecutará cuando se actualice la ventana.




**Usos**: Esta función se utiliza principalmente: 

- cuando se actualiza la interfaz mediante hilos. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) cuando se utiliza la programación MVP (Modelo - Vista - Presentador). Para obtener más información, consulte [Presentación MVP: Modelo - Vista - Presentador](../Editeurs/1000021496.md).

## Ejemplo


```wl
// Process records
// The graphic interface will have to be refreshed 
RequestRefreshUI()
```

```wl
// ----------------------------------------------------------------
// -- Request for refreshing the display of WIN_Contact_Form
// Refresh the title of the window
Myself..Title = gpclContactForm.p_sFullname

// Refresh the linked controls of the window
SourceToScreen()
```

## Sintaxis
<a name="SYNTAX1"></a>

`RequestUpdateUI([<Parameter 1> [... [, <Parameter N>]]])`
---

**`<Parameter 1>: (Parámetro opcional que corresponde al tipo esperado por el evento "Solicitud para actualizar la visualización")`**

Primer parámetro que se pasará al evento "Solicitud para actualizar la visualización" de la ventana. 



**`<Parameter N>: (Tipo de valor esperado por el evento "Solicitud para actualizar la visualización")`**

Enésimo parámetro que se pasará al evento "Solicitud para actualizar la visualización" de la ventana.

## Observaciones


### Evento "Solicitud para actualizar la visualización"
<a name="evento_solicitud_para_actualizar_visualizacion_ELTPARAGRAPHE000081"></a>

"Solicitud para actualizar la visualización" es uno de los eventos asociados a la ventana. Este evento permite agrupar el código de actualización de los controles de la ventana: contenido de los controles, estado (visible, en gris, etc.), título a actualizar, etc.

Este evento puede recibir parámetros. Solo se debe declarar un procedimiento en el evento. Por ejemplo: 


```wl
// Request for refreshing the display
PROCEDURE AskForRefresh(RefreshType is Boolean)
```


Este evento se ejecuta mediante solicitud previa con las siguientes funciones: 

- **RequestUpdateUI**: Ejecuta el evento "Solicitud para actualizar la visualización" al final del proceso en curso (ejecución asíncrona). 

- [ExecuteUpdateUI](../WDLang1/1000021379.md): Ejecuta inmediatamente el evento "Solicitud para actualizar la visualización". 

- [RequestUpdateUIParent](../WDLang1/1000022961.md): Ejecuta el evento "Solicitud para actualizar la visualización" del nivel superior al final del proceso en curso (ejecución asíncrona).

- [ExecuteUpdateUIParent](../WDLang1/1000022961.md): Ejecuta inmediatamente el evento "Solicitud para actualizar la visualización" del nivel superior.
Al llamar a estas funciones, se puede especificar el valor de los parámetros esperados por el evento "Solicitud para actualizar la visualización". 

Observación: El elemento en el que se ejecuta el evento "Solicitud para actualizar la visualización" depende del código desde el que llama a la función **RequestUpdateUI**. Si es una clase con el atributo **presenter**, el elemento será la ventana asociada a la clase. En caso contrario, será el elemento desde el que se ejecutó el código.



![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) 

### Atributo "Presentador"
<a name="atributo_presentador_ELTPARAGRAPHE000116"></a>

El atributo **presentador** permite asociar una clase de la capa del presentador a una vista (ventana, página o reporte). 

Este atributo se utiliza principalmente en la declaración global de las ventanas generadas por el RAD MVP en WINDEV. 

Por ejemplo: 


```wl
PROCEDURE WIN_Table_MyExampleFile(...
	gclPresenter is PTableMyExampleFile dynamic<presenter>=Null)
```


Con este atributo, el evento "Solicitud para actualizar la visualización" de la ventana se activará al realizar:

- una llamada a la función [RequestUpdateUI](../WDLang1/1000021378.md) desde una clase declarada como &lt;presentador&gt;, 

- una llamada desde cualquier clase asociada a la clase &lt;presentador&gt;. 




Para obtener más información, consulte [RAD MVP: Atributo "Presentador"](../Editeurs/1000021508.md).

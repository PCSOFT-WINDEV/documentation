
## dbgWriteWarningAudit( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Escribe una advertencia de tiempo de ejecución personalizada en la auditoría dinámica. La función **dbgWriteWarningAudit** no tiene ningún efecto si la auditoría no está habilitada.

**Observación**: La auditoría dinámica puede activarse mediante [dbgEnableAudit](../WDLang1/1000018835.md).


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Checks whether a window is opened
IF WinStatus("WindowAlwaysOpened") = NotFound THEN
	dbgWriteWarningAudit(1, "The window named WindowAlwaysOpened was closed")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`dbgWriteWarningAudit(<Code> [, <Warning Text>])`
---

**`<Code>: (Integro)`**

Código de la advertencia a escribir.
Este número (incluido entre 1 y 1000) es definido por el desarrollador.

**`<Warning Text>: (Cadena de caracteres opcional)`**

Mensaje explicativo associated con la advertencia.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
[dbgAuditStatus](../WDLang1/1000018836.md) sirve para saber si la auditoría está habilitada para: 

- realizar las verificaciones sólo en este caso. 

- evitar reducir las prestaciones de la aplicación en ejecución estándar.

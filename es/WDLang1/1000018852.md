
## dbgSaveMemoryDump( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Guarda un volcado de memoria utilizado por la aplicación. Un volcado de la memoria utilizada por la aplicación corresponde a una description detallada de los elementos en memoria así como la memoria consumida por estos elementos. 

A diferencia del archivo de volcado, el volcado de memoria no puede utilizarse para ver el contenido de las variables, sino sólo la huella de memoria.

También se puede crear un volcado de memoria pulsando Ctrl + Pausa en cualquier momento


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Save the memory dump in the current runtime audit
dbgSaveMemoryDump()
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Guardar el volcado de memoria en la actual auditoría dinámica

`dbgSaveMemoryDump()`
---





**Observación**: Esta sintaxis guarda el volcado de memoria en el archivo de auditoría dinámico si la auditoría está habilitada.. La función no tiene ningún efecto si la auditoría dinámica no está activada.
<a name="SYNTAX2"></a>

### Guardar el volcado de memoria en un archivo o en modo interactivo

`dbgSaveMemoryDump(<Name of dump file>)`
---

**`<Name of dump file>: (Cadena de caracteres)`**



- "?": inicia la copia de seguridad en modo interactivo. 
	**Si la auditoría dinámica está habilitada**el volcado de memoria se guarda en el archivo de auditoría dinámico. 
	**Si la auditoría dinámica está desactivada**un selector de archivos permite al usuario seleccionar el nombre del archivo donde se guardará el volcado de memoria..
	En ambos casos, se visualiza un mensaje de confirmación al final de la operación.. 

- Nombre del archivo donde se guarda el volcado. El nombre del archivo se puede construir con los siguientes elementos:
	


|   |   |
| --- | --- |
| [%ExeDir%] | Directorio del ejecutable, siempre rellenado con el carácter "\\".. Este directorio es equivalente al resultado de [fExeDir](../WDLang1/3036049.md). |
| [%exeName%] | Nombre del ejecutable. |
| [%Fecha%] | Fecha de inicio del volcado de memoria (formato AAAAMMDDD). |
| [%Hora%] | Hora de inicio del volcado de memoria (formato HHMMSS). |
| [%NombreUsuario%] | Nombre del usuario actual. |
| [%NombreDeLaComputadora%] | Nombre del ordenador. |
| [%DataDir%] | Directorio de los archivos de datos HFSQL especificados por el usuario al instalar la aplicación. Este directorio es equivalente al resultado de [fDataDir](../WDLang1/3036001.md). |
| [%DirUserData%] | Directorio para los datos del usuario actual de la aplicación actual. Este directorio es equivalente al resultado de [fDataDirUser](../WDLang1/3036071.md). |




Si este parámetro corresponde a una cadena vacía ("""): 

- La extensión del archivo será ".wmem".

- el nombre del archivo creado corresponde a: "[%UserDataDir%][%ExeName%]_[%Date%]_[%Time%]". 
	Por ejemplo: "C:\\Usuarios\\Doc\\AppData\\Roaming\\Aplicaciones WINDEV\\MyApp\\MyApi_20161023_130812.wmem".




Atención: Si utiliza la construcción dinámica de cadenas ("Permitir "[% %]" en las cadenas" en la pestaña "Compilación" del proyecto description), se produce un error de compilación (identificador desconocido). En este caso, cada cadena debe ir precedida de '-%'.. Ejemplo: 

```wl
dbgSaveMemoryDump("[%UserDataDir%][%ExeName%]_[%Date%]_[%Time%]")
```
se convierte
```wl
dbgSaveMemoryDump(-%"[%UserDataDir%][%ExeName%]_[%Date%]_[%Time%]")
```

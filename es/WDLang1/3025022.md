


## ControlFirst( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Indica el nombre del primer campo especificado en la orden de tabulación para la ventana especificada. Esta control corresponde al primer control editable al abrir la ventana.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Resume the input in the first control
Reset()
SetFocusAndReturnToUserInput(ControlFirst())
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = ControlFirst([<Window used>])`
---

**`<Result>: (Cadena de caracteres)`**



- Nombre del primer control en edición en la ventana especificada (este control puede obtener el foco y no puede ser Invisible o de sólo lectura). 

- Cadena vacía ("") si ninguna control puede tomar el foco o si la ventana especificada no existe.




**`<Window used>: (Nombre de la ventana opcional)`**

Nombre de la ventana utilizada. 
Si no se especifica este parámetro o si corresponde a una cadena vacía ("""), se utilizará la ventana actual.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Primera control en la orden de tabulación
<a name="primera_control_orden_tabulacion_ELTPARAGRAPHE000056"></a>

La orden de tabulación de los controles de la ventana es el orden en el que los controles obtienen el foco al pulsar la tecla TAB.

Para set el orden de tabulación en el editor de ventanas, en la pestaña "Ventana", en el grupo "Orden", expanda "Orden tab." y seleccione "Editar".

Los controles que tiene en cuenta el orden de tabulación no son necesariamente controles de edición: Combo Box, botón de radio, controles de la caja de lista, etc.. pueden ser tenidos en cuenta por la orden de tabulación.

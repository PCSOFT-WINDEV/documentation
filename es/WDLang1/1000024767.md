
## AAFEnable (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Vuelve a activar una Funcionalidad Automática de la Aplicación (AAF) desactivada previamente con la función [AAFDisable](../WDLang1/1000022018.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Click "Do not save" Button control
// => the values typed by the user will not be saved
AAFDisable(MyWindow, aafSaveValue)
```

```wl
// Click "Re-enable save" Button control
// => the values entered by the user will be saved
AAFEnable(MyWindow, aafSaveValue)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`AAFEnable([<Control or Window>, ] <Feature to re-enable>)`
---

**`<Control or Window>: (Nombre de control o ventana opcional)`**

Nombre de la ventana o control al que se aplica la funcionalidad automática. También es posible utilizar un nombre de alias de ventana.
Si no se especifica este parámetro, la funcionalidad automática indicada se desactivará en toda la aplicación.

**`<Feature to re-enable>: (Constante de tipo Integer)`**

Acción automática a reactivar. Esta acción depende del tipo de control utilizado. Si el tipo de control no es válido para la constante seleccionada, la función no tiene ningún efecto .

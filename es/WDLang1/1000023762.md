


## &lt;Vista de árbol&gt;.listItem (función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Enumera los "hijos" de un nodo y los elementos "hijo" de estos "hijos" en un control TreeView. Se llama Procedure de WLanguage siempre que se encuentra un elemento "infantil".

Ejemplo de control TreeView:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Arbre_Vocabulaire.gif)

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// List the "children" of the "Desserts" node in the "TREE_TVRecipe" TreeView
// The "ExpandAll" procedure is called
// for each "child" element found in the "Desserts" node
Res = TREE_TVRecipe.ListItem("Recipes" + TAB+ "Desserts", "ExpandAll")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <TreeView control>.ListItem(<Node path> , <WLanguage procedure> [, <Custom parameter> [, <Duplicate identifier>]])`
---

**`<Result>: (Integro)`**

Número de elementos listados.

**`<TreeView control>: (Nombre del control)`**

Nombre de la control TreeView a utilizar.

**`<Node path>: (Cadena de caracteres)`**

ruta completa del nodo desde el que se listarán los elementos. Este parámetro:

- tiene el siguiente formato:
	
	```txt
	"<Root name>" + TAB + ["<Name of 1st node>" + TAB + ...
	["<Name of 2nd node>" + TAB + [...]]]"<Leaf name>"
	```
Si hay dos elementos con el mismo nombre en la ruta, puede añadir los ID de los elementos (después de sus nombres utilizando [TreeID](../WDLang1/3018027.md)).

- corresponde a la constante *NULL* para listar desde la raíz de control TreeView.




**`<WLanguage procedure>: (Nombre de Procedure)`**

Nombre de WLanguage Procedure que se llamará siempre que se encuentre un elemento "infantil".. 
Para obtener más información sobre este procedimiento, consulte [Parámetros del procedimiento utilizado por la función &lt;Vista de árbol&gt;.listItem](../WDLang1/1000024659.md).

**`<Custom parameter>: (Entero opcional)`**

El parámetro pasó a &lt;WLanguage Procedure&gt;.

**`<Duplicate identifier>: (Booleano opcional)`**

Indica el modo de gestión de los duplicados en los elementos:

- <u><u><u><u>False</u></u></u></u> (valor predeterminado): no se realiza ninguna gestión.

- <u><u><u><u>True</u></u></u></u>: los nombres de los elementos ruta se construyen para contener su identificador.

## Observaciones
<a name="NOTE0_2"></a>


### Interrupción de &lt;Vista de árbol&gt;.listItem
<a name="interrupcion_docparampagetitleshort_ELTPARAGRAPHE000211"></a>

La función **&lt;Vista de árbol&gt;.listItem** no se puede interrumpir si no se han navegado todos los elementos "hijo".

Para forzar la interrupción de la navegación realizada por **&lt;Vista de árbol&gt;.listItem**, utilice la siguiente línea de código en &lt;WLanguage Procedure&gt;:

```wl
RESULT False
```


En este caso, la función **&lt;Vista de árbol&gt;.listItem** devuelve el número de elementos encontrados hasta que se llame a "RESULT False".


## RegistrySetValue( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Escribe un valor a una clave en el registro de Windows.

**Atención**: La clave ya debe existir antes de que se pueda escribir un valor. Para crear una clave, utilice [RegistryCreateKey](../WDLang1/3065002.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000003065006&name=registrysetvalue_funcion&product=WD">Ver ejemplos adicionales</a>

```wl
// Write the "English" value into the "Language" value
//  in the "HKEY_CURRENT_USER\SOFTWARE\MyApp" key
ResWrite = RegistrySetValue("HKEY_CURRENT_USER\SOFTWARE\MyApp", ...
		"Language", "English")
```

## Sintaxis

### Escribir un valor identificado por su nombre en un registro

`<Result> = RegistrySetValue([<Access mode>, ] <Key path> , <Value name> , <Value> [, <Length>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se escribió el valor,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md) con la constante *errMessage*.




**`<Access mode>: (Constante de tipo Integer)`**

Modo de acceso al registro:



|   |   |
| --- | --- |
| *registryMode32* | Modo forzado para acceder al registro como un programa de 32 bits. |
| *registryMode64* | Modo forzado para acceder al registro como un programa de 64 bits. |
| *registryModeAuto*<br>(Valor predeterminado) | Modo de acceso al registro de automatic: <br><br>- una aplicación de 32 bits que se ejecuta en un sistema de 32 bits manipula el registro como un programa de 32 bits.<br><br>- una aplicación de 32 bits que se ejecuta en un sistema de 64 bits manipula el registro desde la siguiente rama:<br>	<br><pre><code>HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node</code></pre><br><br><br>- una aplicación de 64 bits que se ejecuta en un sistema de 64 bits manipula el registro como un programa de 64 bits.<br><br><br> |



**`<Key path>: (Cadena de caracteres)`**

ruta completa de llave a utilizar. Esta clave debe existir para que se le pueda dar un valor.

**`<Value name>: (Cadena de caracteres)`**

Nombre del valor a escribir. Si el valor no existe, se creará. 

**`<Value>: (Cadena de caracteres, entero, real o puntero)`**

Valor a introducir. Este valor no debe corresponderse directamente con un control Campo de entrada numérico, por ejemplo. Se debe usar una Variable escrita a máquina (declarada como real, cadena de caracteres, ...).

**`<Length>: (Entero opcional)`**

Longitud del valor a escribir cuando &lt;Valor&gt; corresponde a un puntero.

## Observaciones


### Tipos de datos escritos
<a name="tipos_datos_escritos_ELTPARAGRAPHE000164"></a>

Si no se utiliza &lt;Tipo de valor&gt;, los datos escritos en el registro tienen un tipo específico según &lt;Valor&gt;. Si &lt;Valor&gt; corresponde a:

- una cadena de caracteres, el valor escrito será una clave de cadena (REG_SZ).

- un número entero, el valor escrito será un REF_WORD.

- una moneda real o una moneda, el valor escrito será una clave binaria.

- un puntero, con &lt;Longitud&gt; correspondiente a su longitud, el valor escrito será una clave binaria.



<a name="NOTE0_3"></a>


### Manejo del registro en Windows Vista (y posteriores)
<a name="manejo_del_registro_windows_vista_posteriores_ELTPARAGRAPHE000176"></a>

En Windows Vista, si la [Mecanismo UAC](../Editeurs/2025014.md) está activada, debe tener derechos de administrador para escribir en la clave "HKEY_LOCAL_MACHINE\\SOFTWARE".

Si el usuario no tiene los derechos necesarios, el mecanismo UAC realiza una redirección a "HKEY_CURRENT_USER\\Software\\Classes\\VirtualStore\\MACHINE\\SOFTWARE".. El objetivo de esta redirección es garantizar la compatibilidad de las aplicaciones. Las claves escritas en este directorio pueden ser leídas por la aplicación que las ha escrito..  

**Recordatorio**: Al crear el ejecutable, se puede incluir un manifiesto en el ejecutable para que se concedan derechos de administrador a la aplicación.
<a name="NOTE0_4"></a>


### Limitaciones de la función RegistrySetValue
<a name="limitaciones_funcion_docparampagetitleshort_ELTPARAGRAPHE000190"></a>

**RegistrySetValue** no tiene efecto sobre las entradas "(Default)".. Se trata de entradas específicas. Para llegar a estas entradas, el nombre de la clave debe ser reemplazado por una cadena vacía.

**Ejemplo**:

No lo hagas:


```wl
RegistrySetValue("HKEY_CLASSES_ROOT\.jar", "(default)", "myfile")
```


Hacer:


```wl
RegistrySetValue("HKEY_CLASSES_ROOT\.jar", "myfile")
```

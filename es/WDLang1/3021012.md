
## Interacción con el usuario
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000082"></a>
WINDEV, WEBDEV y WINDEV Mobile le permiten comunicarse con el usuario a través de cuadros de diálogo.

Tiene la posibilidad de utilizar la [cuadros de diálogo estándar](../WDLang1/3021017.md) (que contiene una imagen, un texto y uno o varios botones OK, Cancel, Sí o No). Por ejemplo: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=fonctions_dialogue%20-%20HC%20N%B0004.gif)


![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) WINDEV y WINDEV Mobile también permiten utilizar el sistema [cuadros de diálogo avanzados](../WDLang1/3021019.md). Estos cuadros de diálogo pueden hacerlo:

- mostrar un texto específico en los botones. Por ejemplo:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Dialogue.gif)


- permitir al usuario realizar una entrada. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=fonctions_dialogue%20-%20HC%20N%B0008.gif)





![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Observación: A partir de la versión 25, WINDEV y WINDEV Mobile proponen cuadros de diálogo no bloqueantes. Esto hace posible crear un diálogo no bloqueante con el usuario. Se da información al usuario, se hace una pregunta, pero el código continúa su ejecución, incluso si el usuario no está frente a la pantalla o si necesita tiempo para responder: el programa sigue funcionando. La respuesta se procesará asincrónicamente si es necesario.




## InWindowsStoreAppSimulatorMode (Función)

	<IMG style="float:left;margin-right:10px;" src="https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=warning_big.png"><B>Advertencia</B><BR>
	A partir de la versión <B>21</B>, esta funcionalidad se conserva por motivos de compatibilidad. Se recomienda utilizar la función [InUniversalWindowsPlatformSimulatorMode](../WDLang1/1000021971.md).

<a name="Use"></a>
<a name="description"></a>
Identifica si el código se ejecuta o no en el simulador para aplicaciones en modo Windows Store.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Code run in the Windows Store App simulator?
IF InWindowsStoreAppSimulatorMode() = True THEN
	// Specific code for the simulator
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = InWindowsStoreAppSimulatorMode()`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si el código se ejecuta en el simulador para aplicaciones en modo Windows Store,

- <u><u><u><u>False</u></u></u></u> en caso contrario.







## Observaciones


### Disponibilidad de la función
<a name="disponibilidad_funcion_ELTPARAGRAPHE000176"></a>

La función **InWindowsStoreAppSimulatorMode** se puede utilizar en todos los IDE para escribir código multiplataforma. Las funciones o procedimientos que utilizan la función **InWindowsStoreAppSimulatorMode** pueden compartirse entre varios productos sin generar errores de compilación.


### Ejecución de procesos que contienen errores de compilación
<a name="ejecucion_procesos_que_contienen_errores_compilacion_ELTPARAGRAPHE000190"></a>

De forma predeterminada, el mecanismo de seguridad de WLanguage se activa solo si se ejecuta la línea que contiene el error de compilación.

Por ejemplo, si una función no permitida en el producto actual se encuentra en un bucle IF, el mecanismo de seguridad de WLanguage solo se activará si se ejecuta la línea que contiene esta función.

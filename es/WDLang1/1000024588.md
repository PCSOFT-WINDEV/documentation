


## &lt;Variable picLayer&gt;.Fill (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Colorea un área en una variable de tipo [picLayer](../WDLang1/1000024604.md).

El color se aplica a todos los puntos yuxtapuestos que tienen el mismo color que el punto inicial especificado.

![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) **Atención**: Se requiere una configuración específica para usar esta función en Linux. Para obtener más información, consulte [Los dibujos](../WDLang1/3029035.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyWDPicImage is WDPic = "Test.bmp"
MyLayer is picLayer

// Color an area in light blue
MyLayer.Fill(10, 50, LightBlue)
IMG_MyDrawing = MyWDPicImage
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<picLayer image>.Fill(<X> , <Y> [, <Fill color> [, <Color of limits>]])`
---

**`<picLayer image>: (Variable de tipo picLayer)`**

Nombre de la variable de tipo [picLayer](../WDLang1/1000024604.md) que se utilizará.

**`<X>: (Entero)`**

Coordenada X en pixeles del punto dentro del área a colorear. Esta posición se da con respecto a la esquina superior izquierda del control Imagen (coordenadas (0,0)).

**`<Y>: (Entero)`**

Coordenada Y en pixeles del punto dentro del área a colorear. Esta posición se da con respecto a la esquina superior izquierda del control Imagen (coordenadas (0,0)).

**`<Fill color>: (Entero o constante (opcional))`**

Color de relleno del área. Este color puede corresponder a:

- un color RGB (devuelto por la función [RGB](../WDLang1/3029012.md)),

- un color HSL (devuelto por la función [HSL](../WDLang1/3029057.md)),

- [un color predefinido de WLanguage](../WDLang5/3010002.md).




Si no se especifica este parámetro, el color de relleno:

- es *Transparente* si no se ha utilizado la función [&lt;Variable picLayer&gt;.Background](../WDLang1/1000024480.md) previamente, 

- corresponde al color especificado en la última llamada a la función [&lt;Variable picLayer&gt;.Background](../WDLang1/1000024480.md).




**`<Color of limits>: (Entero o constante (opcional))`**

Color del borde del área. Este color puede corresponder a:

- un color RGB (devuelto por la función [RGB](../WDLang1/3029012.md)),

- un color HSL (devuelto por la función [HSL](../WDLang1/3029057.md)),

- [un color predefinido de WLanguage](../WDLang5/3010002.md).




Si no se especifica este parámetro, el color del borde del área se adapta al color de relleno.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Dibujo en modo antialiasing
<a name="dibujo_modo_antialiasing_ELTPARAGRAPHE000428"></a>

Cuando se dibuja en modo antialiasing (constante *drawAntiAliasing* en la función [&lt;Variable picLayer&gt;.ChangeMode](../WDLang1/1000024432.md)), la tarjeta gráfica suaviza los puntos y colores utilizados en el borde de las formas.

En este caso, el parámetro &lt;Color de borde&gt; no se muestra en el dibujo y el relleno se aplica a todo el dibujo.

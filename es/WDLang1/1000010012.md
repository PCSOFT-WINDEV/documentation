


## TableItemStatus (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Devuelve el estado de un nodo en un control Tabla de Treeview.: colapsado, expandido o no encontrado.

**Recordatorio**: En tiempo de ejecución, un nodo se expande cuando el nombre del nodo va precedido de "-".


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Status of a node
SWITCH TableItemStatus(TVT_Schedule, "Root" + TAB + "Node" + TAB + "Leaf2")
	CASE taError: Trace("The node does not exist.")
	CASE taExpand: Trace("The node is expanded.")
	CASE taCollapse: Trace("The node is collapsed.")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Estado de un nodo especificando el número de fila

`<Result> = TableItemStatus(<TreeView Table control> [, <Row number>])`
---

**`<Result>: (Constante de tipo Integer)`**

Estado del nodo especificado:



|   |   |
| --- | --- |
| *taCollapse* | El nodo está colapsado. |
|  *taError* | Nodo no encontrado |
| *taExpand* | Se desglosa el nodo |



**`<TreeView Table control>: (Nombre del control)`**

Nombre del control Tabla TreeView a manipular.

**`<Row number>: (Entero opcional)`**

Número del fila que contiene el nodo a utilizar. Si no se especifica este parámetro, se devuelve el estado del nodo que se encuentra en el fila actual..  


<a name="SYNTAX2"></a>

### Estado de un nodo especificando su nombre

`<Result> = TableItemStatus(<TreeView Table control> [, <Node name>])`
---

**`<Result>: (Constante de tipo Integer)`**

Estado del nodo especificado:



|   |   |
| --- | --- |
| *taCollapse* | El nodo está colapsado. |
|  *taError* | Nodo no encontrado |
| *taExpand* | Se desglosa el nodo |



**`<TreeView Table control>: (Nombre del control)`**

Nombre del control Tabla TreeView a manipular.

**`<Node name>: (Cadena de caracteres opcional)`**

Nombre del nodo a utilizar. Este parámetro tiene el siguiente formato:

```txt
"<Root name>" + TAB + ["<Name of 1st node>" + TAB + ...
["<Name of 2nd node>" + TAB + [...]]]"<Leaf name>"
```


Si este parámetro no corresponde a una rama existente, se produce un error WLanguage.

Si no se especifica este parámetro, se devuelve el estado del nodo que se encuentra en el fila actual..  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Un nodo colapsado o expandido puede ser Visible o no
<a name="nodo_colapsado_expandido_puede_ser_visible_ELTPARAGRAPHE000127"></a>

El status devuelto corresponde al status interno del elemento. Un elemento expandido no es necesariamente Visible porque uno de sus padres puede estar colapsado.

Por ejemplo: Si el nodo "Recetas" + TAB + "Postres" está colapsado, el nodo "Recetas" + TAB + "Postres" + TAB + "Helados" puede colapsarse o expandirse.
<a name="NOTE0_2"></a>


### Banda de selección
<a name="banda_seleccion_ELTPARAGRAPHE000138"></a>

la función **TableItemStatus** no cambia la posición de la banda de selección.
<a name="NOTE0_3"></a>


### Velocidad de ejecución
<a name="velocidad_ejecucion_ELTPARAGRAPHE000149"></a>

La velocidad de ejecución de la sintaxis 1 es más rápida que la de la sintaxis 2.




## Pruebas automatizadas: Funciones específicas de la prueba
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Las siguientes funciones pueden utilizarse en el código de una prueba automatizada:



|   |   |
| --- | --- |
| [TestAddIteration](../WDLang1/1000017015.md) | Añade una iteración a la prueba actual y da los valores de los parámetros de prueba para esta iteración. |
| [TestCheck](../WDLang1/1000017023.md) | Escribe un resultado en el estado reporte de un test según el resultado de una expresión booleana. |
| [TestCheckDifferent](../WDLang1/1000023397.md) | Verifica si las expresiones especificadas son diferentes en la programación asertiva y en el modo de prueba automática. |
| [TestCheckEqual](../WDLang1/1000023398.md) | Verifica si las expresiones son iguales en la programación asertiva y en el modo de prueba automática. |
| [TestCheckFalse](../WDLang1/1000023399.md) | Verifica si una expresión es falsa en la programación asertiva y en el modo de prueba automática. |
| [TestCheckNotNull](../WDLang1/1000023400.md) | Verifica si una expresión no es nula en la programación asertiva y en el modo de prueba automática. |
| [TestCheckNull](../WDLang1/1000023401.md) | Verifica si una expresión es nula en la programación asertiva y en el modo de prueba automática. |
| [TestCheckTrue](../WDLang1/1000023402.md) | Verifica si una expresión vale True en la programación asertiva y en el modo de prueba automática. |
| [TestError](../WDLang1/1000023506.md) | Provoca: <br><br>- una afirmación durante la prueba de elemento o proyecto (modo Go). <br><br>- un error en el modo de prueba automatizado.<br><br><br> |
| [TestOnError](../WDLang1/1000023507.md) | Provoca: <br><br>- una afirmación durante la prueba de elemento o proyecto (modo Go). <br><br>- un error en el modo de prueba automatizado.<br><br><br> |
| [TestWriteResult](../WDLang1/1000013001.md) | Escribe información o un error en el status reporte de una prueba unitaria, en el editor de pruebas. |



Observación: En el código de una prueba automatizada, también es posible utilizar la declaración [ON](../Motscles/1511020.md).

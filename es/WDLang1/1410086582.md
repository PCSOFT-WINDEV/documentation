


## Telemetría: Configuración de las aplicaciones WINDEV y WINDEV Mobile
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000370"></a>
Para usar la telemetría en una aplicación WINDEV o WINDEV Mobile, debe: 

1. [Habilitar la telemetría en la aplicación](#NOTE2_1). Esta activación se puede hacer: 

	- en la descripción del proyecto. En este caso, esta activación se realiza para todas las configuraciones de proyectos existentes. 

	- en la plataforma description. 

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) al crear el ejecutable. 




2. Desplegar la aplicación: 

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [Crear el programa instalación para la aplicación](#NOTE3_1).

	- Instale la aplicación.   







Observación: Los datos de telemetría son datos anónimos de Default. Tiene la posibilidad de identificar al usuario utilizando [TelemetryIdentifyUser](../WDLang1/1000022160.md).

## Activación de la telemetría
<a name="activacion_telemetria_ELTTEXTE000394"></a>
La telemetría puede ser activada: 

- en la descripción del proyecto. En este caso, esta activación se realiza para todas las configuraciones de proyectos existentes. 

- en la plataforma description. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) al crear el ejecutable. 





### Habilitar la telemetría en el proyecto description
<a name="habilitar_telemetria_proyecto_description_ELTPARAGRAPHE000070"></a>

Para habilitar la telemetría en el proyecto description: 

1. Abra la ventana de descripción del proyecto: en la pestaña "Proyecto", en el grupo "Proyecto", haga clic en "Descripción".

2. En la ventana que aparece, seleccione la pestaña "Telemetría".. 

3. Marque "Activar la telemetría para el proyecto" y especifique los parámetros solicitados: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=T%E9l%E9m%E9trie_Projet%20-%20HC%20N%B0001.gif&type=thumb)


	- Ubicación del servidor: 

	- Servidor local, red o Internet : En este caso, tiene la posibilidad de especificar el protocolo utilizado: HTPP o HTTPS. 
						Observación: Para un protocolo HTTPS, un Certificate debe ser instalado en su servidor Web (IIS o Apache).

	- Plataforma CLOUD. En este caso, todo lo que tiene que hacer es especificar la cuenta CLOUD que desea utilizar. 
						Observación: Se requiere una plataforma de telemetría. 

4. Validar. 




Estos parámetros se definen para el proyecto actual y para todas sus configuraciones. La telemetría puede ser personalizada o desactivada para cada configuración de proyecto.. 


### Habilitando la telemetría en la description de una configuración
<a name="habilitando_telemetria_description_una_configuracion_ELTPARAGRAPHE000119"></a>

Para configurar la telemetría para la configuración actual: 

1. En la pestaña "Proyecto", en el grupo "Configuración del proyecto", haga clic en "Configuración actual". 

2. Seleccione la pestaña "Telemetría" para configurar la telemetría. 

3. Hay varias opciones disponibles: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=telemetrie%20-%20Options%20-%20HC%20N%B0001.gif&type=thumb)


	- **Usar los parámetros de telemetría del proyecto**: Esta opción permite utilizar los parámetros de telemetría especificados en el proyecto description. 
			 Atención: Si la telemetría no está habilitada en el proyecto, la configuración actual no utilizará la telemetría. 

- **Usar parámetros de telemetría específicos**: Esta opción se utiliza para definir parámetros de telemetría específicos para la configuración actual. Por ejemplo, se puede desactivar la telemetría en el proyecto y habilitarla sólo en una configuración. También tiene la posibilidad de utilizar parámetros específicos para un tipo de configuración (por ejemplo, las configuraciones de Windows y móvil no utilizan el mismo servidor). 

- **Deshabilitar la telemetría para esta configuración**: Esta opción le permite desactivar la telemetría para la configuración actual. 




Observación: Si selecciona "Utilizar parámetros de telemetría específicos", deberán definirse todos los parámetros de telemetría.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=telemetrie%20-%20Options%20-%20HC%20N%B0002.gif&type=thumb)


1. Especifique la ubicación del servidor de telemetría: 

	- Servidor local, red o Internet : En este caso, tiene la posibilidad de especificar el protocolo utilizado: HTPP o HTTPS. 
			Observación: Para un protocolo HTTPS, un Certificate debe ser instalado en su servidor Web (IIS o Apache).

	- Plataforma CLOUD. En este caso, todo lo que tiene que hacer es especificar la cuenta CLOUD que desea utilizar. 




2. Especifique la Address del servidor de telemetría a utilizar: 

	- Dirección del servidor, 

	- Nombre de usuario y contraseña de associated. Estos datos se dieron al final de [instalación del servidor de telemetría](../WDLang1/1410086581.md). 




3. Compruebe la conexión con el servidor de telemetría para asegurarse de que los datos son correctos.. 

4. La opción "Telemetría de la aplicación" sirve para configurar: 

	- El nombre de la aplicación. Este nombre será propuesto en la telemetría cuadro de mando. 

	- El modo de activación de la recogida de información. La recopilación de datos la habilita automáticamente Default, pero se puede habilitar mediante programación con [TelemetryEnable](../WDLang1/1000021900.md).

### Habilitar la telemetría al crear el ejecutable
<a name="habilitar_telemetria_crear_ejecutable_ELTPARAGRAPHE000176"></a>

Cuando se crea un ejecutable de Windows, se tiene la capacidad de habilitar y configurar la telemetría. 

Las etapas son las siguientes: 

1. Seleccione (si es necesario) la configuración correspondiente al ejecutable. 

2. Crear el ejecutable: en la pestaña "Proyecto", en el grupo "Generación", haga clic en "Generar".

3. En los diferentes pasos relativos al "EXE Definition", el paso "Telemetría" se utiliza para especificar las opciones de telemetría. 

4. Hay varias opciones disponibles: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=telemetrie%20-%20Exe%20-%20HC%20N%B0004.gif)


	- **Usar los parámetros de telemetría del proyecto**: Esta opción permite utilizar los parámetros de telemetría especificados en el proyecto description. Esta opción no puede utilizarse si la telemetría no está habilitada en el proyecto. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=telemetrie%20-%20Exe%20-%20HC%20N%B0001.gif)




Siga los diferentes pasos de asistente para crear su ejecutable.  

**Observación**: Los datos de telemetría enviados al servidor se procesan cada dos horas. Por lo tanto, no será inmediatamente Visible en la telemetría cuadro de mando.

## Instalación de la aplicación
<a name="instalacion_aplicacion_ELTTEXTE000436"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Instalar una aplicación que soporte la telemetría
<a name="instalar_una_aplicacion_que_soporte_telemetria_ELTPARAGRAPHE000234"></a>

La Procedure para la creación de la instalación de una aplicación que soporte la telemetría es idéntica a la Procedure para la creación de una instalación estándar, con un paso adicional. 

1. Inicie la instalación Procedure de la aplicación: en la pestaña "Proyecto", en el grupo "Generación", haga clic en "Procedimiento de instalación".

2. Crear el ejecutable si es necesario. 

3. En la sección izquierda de asistente, seleccione "Plantilla de instalador".. 

4. Siga los diferentes pasos de asistente hasta llegar al paso "Activar la telemetría".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Tel%E9m%E9trie%20-%20Installation%20-%20HC%20N%B0001.gif&type=thumb)
 En esta pantalla, puede: 

	- **Mostrar un mensaje de información al usuario durante la instalación**. Este mensaje informativo informa al usuario final que la telemetría se utilizará en su aplicación. 
			 **Importante note**: Independientemente de que este mensaje se muestre o no, la telemetría se activará en la aplicación. 
			Si se ha utilizado [TelemetryIdentifyUser](../WDLang1/1000022160.md) en el código de la aplicación, el mensaje Default no especifica que los datos transmitidos son datos anónimos. 

	- **Permitir que el usuario deshabilite la telemetría durante la instalación**. 
			En este caso, el usuario final puede marcar (o no) la opción "Acepto contribuir a la mejora de XXXX". No se recogerá ningún dato si el usuario final se niega a. 
			Observación: Si se marca la opción "Durante una actualización, la telemetría se propondrá automáticamente al usuario aunque estuviera desactivada", la pregunta se planteará al usuario final cada vez que se realice una nueva actualización.







Entonces, la Procedure para la creación de la instalación es la estándar. 

**Recordatorio**: Este programa de instalación se suministra en inglés y en francés. Para traducir el programa instalación a otros idiomas, se debe crear un programa personalizado de instalación. 

Cuando se instala la aplicación, los datos se envían al servidor:  

- cada hora. Esta frecuencia puede ser modificada por [TelemetryConfigure](../WDLang1/1000021913.md). 

- al cerrar la solicitud.

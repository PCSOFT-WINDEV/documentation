
## HaspHLWrite( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Escribe datos (cadenas de caracteres o búferes) en la memoria interna de la tecla Hasp.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Generates an initial activation key
InitialKey is string = KeyGenerateInitialKey()

// Stores the initial activation key in the memory of the Hasp key
HaspHLWrite(EDT_Session_ID, haspMainFile, InitialKey)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = HaspHLWrite(<Session identifier> , <File identifier> , <Position> , <String to write>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se realizó la escritura correctamente,

- <u><u><u><u>False</u></u></u></u> en caso contrario. [ErrorInfo](../WDLang1/3013008.md) devuelve los detalles del error.




**`<Session identifier>: (Integro)`**

Identificador de la sesión Hasp, devuelto por [HaspHLLogin](../WDLang1/1000018891.md).

**`<File identifier>: (Entero o constante de tipo Integer)`**

Identificador del fichero en el que se deben escribir los datos. Este identificador puede corresponder a:

- un entero. 

- una de las siguientes constantes:
	


|   |   |
| --- | --- |
| *haspLicenseFile* | Escribe en el archivo "área de licencia" de la clave. |
| *haspMainFile* | Escribe en el archivo "área de memoria principal" de la llave. |






**`<Position>: (Integro)`**

Posición (en bytes) en la que se escribirán los datos en el fichero.

**Atención**: El inicio del archivo comienza en la posición 0.

**`<String to write>: (Cadena de caracteres o memoria intermedia)`**

Datos que se deben escribir en el archivo.

**Atención**: El tamaño de los archivos en las teclas Hasp es fijo. La función fallará si el tamaño de los datos a escribir excede el tamaño disponible en el archivo.




## HashCheckFile (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Comprueba el Hash de un archivo para un tipo específico de algoritmo.. Tiene la capacidad de comprobar: 

- un simple hachís.

- un hash con autenticación de mensajes.

- un hash obtenido con una función de derivación de claves que toma una sal como parámetro (PBKDF2).
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) No disponible.




**Recordatorio**: El hashing se utiliza para resumir un archivo o una cadena de caracteres. Este resumen se llama Hash.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Calculate the Hash of the "C:\Temp\File.txt" file
HashMyFile is string 
HashMyFile = HashFile(HA_MD5_128, "C:\Temp\File.txt")

// Process on the file
// ...

// Checks whether the file was modified
IF HashCheckFile(HA_MD5_128, "C:\Temp\File.txt", ...
		HashMyFile) = False THEN
	Info("The file was modified.")
END
```

## Sintaxis

### Comprobando un simple hachís

`<Result> = HashCheckFile(<Type of algorithm> , <File path> , <Hash>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si el resultado del hashing del archivo &lt;Ruta de archivo&gt; con el algoritmo &lt;Tipo de algoritmo&gt; corresponde al parámetro &lt;Hash&gt;,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Type of algorithm>: (Constante de tipo Integer)`**

Indica el tipo de algoritmo que se utiliza para obtener el archivo &lt;File ruta&gt;:



|   |   |
| --- | --- |
| MD4 family | *HA_MD4*<br><br> |
| MD5 family | *HA_MD5_128*<br><br>Atención: este tipo de algoritmo está actualmente en desuso. |
| MURMUR family | <br><br>- *HA_MURMUR_1*: Algoritmo de soplo versión 1: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2*: Algoritmo de soplo versión 2: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_64A*: Algoritmo de soplo versión 2: Resultado hash de 64 bits optimizado para procesadores de 64 bits<br><br>- *HA_MURMUR_2_64B*: Algoritmo de soplo versión 2: Resultado hash de 64 bits optimizado para procesadores de 32 bits<br><br>- *HA_MURMUR_2_A*: Algoritmo de murmullo incremental versión 2: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_BIG_ENDIAN*: Algoritmo de murmullo versión 2 para máquina big-endian: Resultado hash de 32 bits<br><br>- *HA_MURMUR_2_ALIGNE*: Algoritmo de soplo versión 2 para máquina alineada: Resultado hash de 32 bits<br><br>- *HA_MURMUR_3_32*: Algoritmo de soplo versión 3: Resultado hash de 32 bits<br><br>- *HA_MURMUR_3_128_X86*: Algoritmo de soplo versión 3: Resultado hash de 128 bits optimizado para procesadores de 32 bits<br><br>- *HA_MURMUR_3_128_X64*: Algoritmo de soplo versión 3: Resultado hash de 128 bits optimizado para procesadores de 64 bits<br><br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes de esta familia no están disponibles. |
| RIPEMD family | <br><br>- *HA_RIPEMD_128*<br><br>- *HA_RIPEMD_160*<br><br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes de esta familia no están disponibles. |
| Familia SHA | <br><br>- HA_SHA_160 (también llamado SHA-1)<br><br>- HA_SHA_256 (especificaciones FIPS PUB 198)<br><br>- HA_SHA_256_DOUBLE<br><br>- HA_SHA_384<br><br>- HA_SHA_512<br><br><br> |
| Familia SHA-3 | <br><br>- *HA_SHA3_224*<br><br>- *HA_SHA3_256*<br><br>- *HA_SHA3_384*<br><br>- *HA_SHA3_512*<br><br><br><br><br> |
| TIGER family | <br><br>- *HA_TIGER_128*<br><br>- *HA_TIGER_160*<br><br>- *HA_TIGER_192*<br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes de esta familia no están disponibles. |
| WHIRLPOOL family | *HA_WHIRLPOOL*<br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) No disponible. |
| CKSUM family | <br><br>- *HA_CKSUM_8*<br><br>- *HA_CKSUM_16*<br><br>- *HA_CKSUM_32*<br><br>- *HA_CKSUM_64*<br><br><br><br><br>![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes de esta familia no están disponibles. |





- Algoritmos MD4 / MD5 / SHA / RIPE: funciones de hash criptográficas estándar.

- Algoritmos TIGER/WHIRLPOOL: funciones hash con altas propiedades criptográficas optimizadas para el modo de 64 bits (pero también se puede utilizar en el modo de 32 bits).

- Algoritmos CKSUM: funciones de hash sin Property criptográfica, deben utilizarse para realizar comprobaciones de bajo nivel o con tablas de hash.

- Algoritmo MURMUR: funciones de hash muy rápidas sin propiedades criptográficas. Este algoritmo es usado por el "filtro de florecimiento" de las carteras de bitcoin.




**`<File path>: (Cadena de caracteres)`**

Ruta del archivo para el que se comprobará el Hash.

**`<Hash>: (Cadena de caracteres)`**

Hash de archivo a comprobar. Este Hash debe ser el resultado de [HashFile](../WDLang1/1000007112.md) llamado previamente para &lt;Fichero ruta&gt; utilizando el &lt;Tipo de algoritmo&gt;..

- HA_HMAC_SHA_256_DOUBLE

- HA_MD4

- HA_MD5_128

- HA_SHA_160, HA_SHA_256, HA_SHA_384, HA_SHA_512

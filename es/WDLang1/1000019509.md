
## fExtractResource( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Extrae un recurso de la aplicación en una ubicación física del dispositivo.

**Observaciones**: 

- ![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) La Resource a extraer debe haber sido incluida en la aplicación generada por la generación asistente (ver Notas). 

- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) En las aplicaciones WINDEV o en los sitios WEBDEV, el Resource a extraer debe ser: 

	- en la categoría "Otros" del panel [Explorador de proyectos](../Editeurs/2027029.md),

	- en la biblioteca principal de la aplicación (archivo WDL). 







Atención: Para evitar la piratería, los elementos de WINDEV, WEBDEV y WINDEV Mobile (informes, ventanas, etc.) no pueden extraerse en tiempo de ejecución.

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = fExtractResource(<Resource name> [, <Destination>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se extrajo el recurso,

- <u><u><u><u>False</u></u></u></u> en caso contrario. Si se produce un error, la variable [ErrorOccurred](../WDLang1/3087001.md) se pone en <u><u><u><u>True</u></u></u></u>. Para obtener más información sobre el error, utilice la función [ErrorInfo](../WDLang1/3013008.md).




**`<Resource name>: (Cadena de caracteres)`**

Nombre del recurso a extraer (nombre del archivo con su extension).



![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) El Resource a extraer debe encontrarse en la biblioteca principal de la aplicación (archivo WDL). La biblioteca no debe haber sido cargada por [LoadWDL](../WDLang1/3013020.md).

**`<Destination>: (Cadena de caracteres opcional)`**





![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) ruta completa del archivo de destino. 
Si el directorio existe, la Resource se extrae directamente a este directorio.
El directorio se crea si no existe. 
Si no se especifica este parámetro o si corresponde a una cadena vacía (""), el Resource se extraerá en el directorio de datos de la aplicación (devuelto por [fDataDir](../WDLang1/3036001.md)) en un archivo llamado como el Resource.
El archivo de destino se sobrescribirá si ya existe antes de la extracción.

## Observaciones
![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) 

### Generación asistente
<a name="generacion_asistente_ELTPARAGRAPHE000181"></a>

El asistente para generar aplicaciones Android, Universal Windows 10 app o iPhone/iPad permite especificar si los archivos y bibliotecas deben incluirse en la aplicación. 

En una aplicación Android, tiene la posibilidad de incluir: 

- recursos específicos. 

- archivos específicos de Java. 




Entonces, estos archivos pueden ser: 

- utilizarse directamente (archivos que solo se van a leer, por ejemplo: archivos de sonido, archivos de imagen, etc.). 

- extraídos con **fExtractResource** (por ejemplo, archivos que serán modificados por la aplicación)




Si se incluye una imagen en la aplicación, puede asignarse directamente a una control especificando su nombre. Por ejemplo: 


```wl
IMG_Image = image1
```




Para obtener más información, consulte: 

- ![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) [Generación de aplicaciones universales para Windows 10 asistente](../Editeurs/1410086577.md).

### Caso especial: Archivo ".WDD" (análisis description)
<a name="caso_especial_archivo_wdd_analisis_description_ELTPARAGRAPHE000255"></a>

No se puede extraer un análisis description (archivo ".WDD") integrado a la biblioteca de una aplicación. 

Para extraer un archivo de una biblioteca que no puede ser extraído por **fExtractResource**, puede utilizar un archivo de datos HFSQL integrado en la biblioteca.

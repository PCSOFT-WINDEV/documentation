
## EmulateInfo( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Emula la siguiente llamada a [Info](../WDLang1/3021011.md) y pulsa automáticamente el botón "OK".
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Automatically close the next Info dialog box
EmulateInfo()

// Process 
// ...

Info("The record was validated.")
// The message is automatically closed
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`EmulateInfo()`
---



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Comprobación de la función
<a name="comprobacion_funcion_ELTPARAGRAPHE000033"></a>

Se puede verificar si existe una función de emulación ([EmulateWarning](../WDLang1/1000009001.md), [EmulateError](../WDLang1/1000009005.md), **EmulateInfo**, etc.) al llamar a la función de diálogo asociada ([Advertencia](../WDLang1/3021009.md), [Error](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), etc.).
<a name="NOTE0_2"></a>




### Diferencia de clase de mensaje
<a name="diferencia_clase_mensaje_ELTPARAGRAPHE000058"></a>

El orden de las funciones [EmularXXX](../WDLang1/1000009002.md) debe respetar el orden de visualización de [Error](../WDLang1/3021013.md), [Info](../WDLang1/3021011.md), ... De lo contrario, se producirá un error fatal.

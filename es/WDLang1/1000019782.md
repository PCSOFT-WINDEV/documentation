


## EvaluateExpression ( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Evalúa el valor de una expresión construida en una cadena de caracteres. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Evaluate a formula
x,y are real
x = 1.5
Y = 5
Trace(EvaluateExpression("cos(x)+sin(y)"))
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = EvaluateExpression(<Expression>)`
---

**`<Result>: (Cualquier tipo)`**



- **Si la expresión compila sin errores**resultado de la expresión.

- **Si la expresión no puede compilar**Si no devuelve un valor o si se produce un error durante la evaluación, se desencadena un error fatal.




**`<Expression>: (Cadena de caracteres)`**

Expresión para evaluar.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Variables locales
<a name="variables_locales_ELTPARAGRAPHE000042"></a>

Las variables locales del actual Process pueden ser usadas directamente en la expresión.
<a name="NOTE0_2"></a>


### Compilación de la expresión
<a name="compilacion_expresion_ELTPARAGRAPHE000049"></a>

El código se recompila cada vez que se llama a la función **EvaluateExpression**. 

Para evitar el paso de compilación, puede utilizar [Compilar](../WDLang1/3013015.md) y [Ejecutar](../WDLang1/3013041.md) para construir el código de un Procedure, compilarlo una vez y ejecutarlo varias veces. En este caso, perderá la capacidad de utilizar directamente las variables locales del proceso actual, que deberán ser sustituidas por parámetros en la Procedure.
<a name="NOTE0_3"></a>


### Usando una Procedure de componente externo
<a name="usando_una_procedure_componente_externo_ELTPARAGRAPHE000066"></a>

El código se recompila cada vez que se llama a la función **EvaluateExpression**. 

La llamada a un Procedure de componente externo debe nombrar a este Procedure con su nombre completo. 


```wl
EvaluateExpression("MyComponent.MySet.MyProcedure()"))
```

<a name="NOTE0_4"></a>


### Código dinámico
<a name="codigo_dinamico_ELTPARAGRAPHE000079"></a>

Las constantes **no pueden utilizarse** en el código dinámico (definido por la palabra clave [CONSTANT](../Motscles/1514012.md)).

Cuando se utilizan constantes en un código, todas las apariciones de las constantes se sustituyen por su valor durante la compilación en el editor, pero la correspondencia entre el nombre de las constantes y su valor no se "incrusta" en la aplicación. Por lo tanto, la compilación dinámica no puede utilizar las constantes.

Veamos dos alternativas:

1 - Usar variables en lugar de constantes: 


```wl
CONSTANT
	CST_Name = 1
END
```
se convierte, por ejemplo


```wl
CST_Name is int = 1
```
o

2 - En la cadena que contiene el código que debe compilarse dinámicamente, sustituir el nombre de la constante por su valor:


```wl
sCode is string
// In the string containing the code that will be compiled dynamically
// instead of leaving the name of the constant:
sCode=[
Info(CST_Name)
]
// Replace the name of the constant by its value
sCode = Replace(sCode, "CST_Name", CST_Name, WholeWord + IgnoreCase)
// The code can be compiled now
IF Compile("DynProc", sCode) <> "" THEN
	Error("Error while compiling the dynamic procedure: ", ...
		ErrorInfo())
ELSE
	// Then it can be run
	WHEN EXCEPTION IN 
		ExecuteProcess("DynProc", trtProcedure)
	DO
		Error("Error while running the dynamic procedure: ", ...
			ExceptionInfo())
	END
END
```

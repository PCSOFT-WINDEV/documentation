
## fReportsAndQueriesDir( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
**fReportsAndQueriesDir** le permite get y cambiar:

- la ruta completa del directorio para los informes y consultas personalizados. Este directorio corresponde al directorio de los informes y consultas de Visible por el usuario que los creó.

- la ruta completa del directorio para los informes y consultas compartidos. Este directorio corresponde al directorio de los informes y consultas de Visible por todos los usuarios de la aplicación.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Directory of custom reports and queries
DirReportsAndQueries is string
DirReportsAndQueries = fReportsAndQueriesDir(dirUser)
// Save the reports and queries found in this
// directory in an archive
zipCreate("MyReportsAndQueries", "C:\MyFiles\ReportsAndQueries.WDZ")
zipAddDirectory("MyReportsAndQueries", DirReportsAndQueries)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Encontrar el directorio de los informes y consultas personalizadas

`<Result> = fReportsAndQueriesDir([<Custom/Shared>])`
---

**`<Result>: (Cadena de caracteres)`**

Ruta del directorio que contiene los informes y consultas personalizados o compartidos: 

- ruta completa (por ejemplo: "C:\\MyReportsQueries\\MyReports")

- UNC ruta (por ejemplo: " ASLASH_\\ServerReportsAndQueries\\MyReports")




**`<Custom/Shared>: (Constante opcional de tipo Integer)`**

Buscaba directorio:


|   |   |
| --- | --- |
| *dirShared* | Directorio de los informes y consultas compartidos. Este directorio corresponde al directorio de los informes y consultas de Visible por todos los usuarios de la aplicación. |
| *dirUser*<br>(Valor predeterminado) | Directorio de informes y consultas personalizados. Este directorio corresponde al directorio de los informes y consultas de Visible por el usuario que los creó. |




<a name="SYNTAX2"></a>

### Modificar el directorio de los informes y consultas personalizados

`<Result> = fReportsAndQueriesDir(<Custom/Shared> [, <New path>])`
---

**`<Result>: (Cadena de caracteres)`**

Ruta del directorio que contiene los informes y consultas personales o compartidos antes de utilizar **fReportsAndQueriesDir**: 

- ruta completa (por ejemplo: "C:\\MyReportsQueries\\MyReports")

- UNC ruta (por ejemplo: " ASLASH_\\ServerReportsAndQueries\\MyReports").




**`<Custom/Shared>: (Constante de tipo Integer)`**

Directorio para modificar:


|   |   |
| --- | --- |
| *dirShared* | Directorio de los informes y consultas compartidos. Este directorio corresponde al directorio de los informes y consultas de Visible por todos los usuarios de la aplicación. |
| *dirUser*<br>(Valor predeterminado) | Directorio de informes y consultas personalizados. Este directorio corresponde al directorio de los informes y consultas de Visible por el usuario que los creó. |



**`<New path>: (Cadena de caracteres)`**

Nueva ruta completa (local o UNC) del directorio. 
Atención: Esta ruta será válida sólo para la instancia actual de la aplicación.: no se guardará en sustitución de la ruta especificada durante la instalación.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Estos directorios se especificaron al crear el programa instalación o al instalar la aplicación. 

- Esta función no le permite encontrar el directorio que contiene los informes y/o consultas guardados a través de "Guardar como" en Informes y Consultas.. Este directorio puede conocerse en la ventana description de la reporte o consulta especificada.

- El posible cambio de directorios se aplica a la instancia actual del ejecutable. Los directorios especificados durante la instalación no son modificados por la función. Por lo tanto, estos directorios no serán modificados en Reports and Queries.

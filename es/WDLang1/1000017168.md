


## TablaAñadirColumna (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade elementos a la derecha de una control Tabla horizontal.

**Atención**: Esta función no se puede utilizar para añadir columnas a una control Tabla estándar (control Tabla vertical).. Para añadir dinámicamente columnas en un control Tabla estándar, utilice [ControlClone](../WDLang1/3025017.md).


<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = TableAddColumn(<Table control> [, <Element row 1> [... [, <Element row N>]]])`
---

**`<Result>: (Integro)`**



- Índice de la nueva columna,

- -1 si se produce un error.




**`<Table control>: (Nombre del control)`**

Nombre de la control Tabla horizontal que se va a utilizar.

Si este parámetro corresponde a una cadena vacía (""), se utilizará el control Tabla al que pertenece el Process actual.

**`<Element row 1>: (Tipo de associated fila (opcional))`**

Elementos de la columna para añadir a la control Tabla especificada. Cada elemento corresponde a un fila. El tipo de elemento añadido debe ser compatible con el tipo de la fila correspondiente..
Si no se especifica ninguno de estos parámetros, se añade una columna vacía a la columna control Tabla. Para una control Tabla basada en un archivo de datos, se tienen en cuenta los valores de Default definidos en el archivo de datos relacionado.



**`<Element row N>: (Tipo de columna associated (opcional))`**

Elementos de la columna para añadir a la control Tabla especificada. Cada elemento corresponde a un fila. El tipo de elemento añadido debe ser compatible con el tipo de la fila correspondiente..
Si no se especifica ninguno de estos parámetros, se añade una columna vacía a la columna control Tabla. Para una control Tabla basada en un archivo de datos, se tienen en cuenta los valores de Default definidos en el archivo de datos relacionado.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Características de los elementos de columna
<a name="caracteristicas_los_elementos_columna_ELTPARAGRAPHE000068"></a>

- Para especificar ningún valor para un elemento, utilice una cadena vacía (""") o 0 (según el tipo de fila).

- Si un elemento corresponde al resultado de un cálculo, la expresión numérica debe estar entre paréntesis.. Por ejemplo:
	
	```wl
	TableAddColumn(TABLE_CalcTable, (53+29), (83-21))
	```




<a name="NOTE0_2"></a>


### Posición de la columna añadida
<a name="posicion_columna_anadida_ELTPARAGRAPHE000085"></a>

Por Default, la columna se añade al final de la control Tabla.

- Si el control Tabla está ordenado por el [TableSort](../WDLang1/3074035.md), la columna añadida se posiciona en el control según la ordenación actual.

- Si el usuario ordena la control Tabla, la ordenación se ignora cuando se añade la columna.. La columna se añade al final de la control.



<a name="NOTE0_3"></a>


### Limitaciones: Número máximo de columnas mostradas
<a name="limitaciones_numero_maximo_columnas_mostradas_ELTPARAGRAPHE000100"></a>

El número máximo de columnas que se pueden mostrar en un control Tabla está limitado por la cantidad de memoria disponible (máximo teórico: 2 mil millones de columnas).

Sin embargo, no recomendamos el uso de tablas control con un gran número de columnas por razones de rendimiento (tiempo necesario para llenar el control, por ejemplo) y de facilidad de uso.
<a name="NOTE0_4"></a>


### Control Tabla basado en un archivo de datos: Modificaciones en el archivo de datos relacionado
<a name="control_tabla_basado_archivo_datos_modificaciones_archivo_datos_relacionado_ELTPARAGRAPHE000111"></a>

Si la entrada en cascada está habilitada, el archivo de datos relacionado a control Tabla se modifica automáticamente cuando el usuario pasa a la siguiente columna.

Si la entrada en cascada no está habilitada, no se modifica el el archivo de datos relacionado al control Tabla basado en un archivo de datos. Para escribir el contenido de la columna agregada al archivo de datos relacionado, utilice la función [TableSave](../WDLang1/3074008.md) después de la función **TablaAñadirColumna**.

**Recordatorio**: Se puede seleccionar la opción "Entrada en cascada (Agregar)":

- Al crear una control Tabla.

- En la pestaña "Detalles" del control description.



<a name="NOTE0_5"></a>


### Control Tabla basado en un archivo de datos: Añadir varias columnas
<a name="control_tabla_basado_archivo_datos_anadir_varias_columnas_ELTPARAGRAPHE000136"></a>

Se crea una columna virtual cada vez que se llama a **TablaAñadirColumna** en un control Tabla basado en un archivo de datos. La información se guardará en el archivo de datos al llamar a la función [TableSave](../WDLang1/3074008.md).

Si se llama a la función **TablaAñadirColumna** sucesivamente en el mismo proceso sin llamar a la función [TableSave](../WDLang1/3074008.md), solo la PRIMERA llamada es efectiva (las siguientes llamadas son ignoradas). Sólo se añade una columna a la control Tabla basada en un archivo de datos.
<a name="NOTE0_6"></a>


### Añadir registros a una control Tabla basada en un archivo de datos y en un Trigger
<a name="anadir_registros_una_control_tabla_basada_archivo_datos_trigger_ELTPARAGRAPHE000157"></a>

Se puede liberar un disparador cuando se añade una Record en una control Tabla basada en un archivo de datos. Para obtener más información, consulte la documentación de [HDescribeTrigger](../WDLang4/3044074.md).
<a name="NOTE0_7"></a>


### Añadiendo un Image
<a name="anadiendo_image_ELTPARAGRAPHE000169"></a>

Para añadir un Image en un control Tabla, especifique el nombre y el ruta completo del Image a añadir.

Por ejemplo:

```wl
TableAddColumn(TABLE_MyTable, "C:\MyDocuments\MyImage.BMP")
```

**Observación**: El fila debe ser un fila de imagen, de lo contrario el Image no se mostrará correctamente.
<a name="NOTE0_8"></a>


### control Tabla en modo edición
<a name="control_tabla_modo_edicion_ELTPARAGRAPHE000188"></a>

Si se llama a la función **TablaAñadirColumna**:

- ***En un evento del control Tabla***, el cursor se posiciona automáticamente en la primera fila de la columna agregada.

- ***En un evento que no pertenece al control Tabla (o en un proceso personalizado)***, utilice la función [SetFocusAndReturnToUserInput](../WDLang1/1410088107.md) para reanudar la entrada en el control Tabla.

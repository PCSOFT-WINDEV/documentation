
## GanttAddTask (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Añade una tarea en una columna del Diagrama de Gantt en una Tabla o control Tabla TreeView.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
NewTask is GanttTask

// Configures the task
NewTask.ID = GetIdentifier()
NewTask.Row = 1
NewTask.Progress = 50
NewTask.StartDate = Today()
NewTask.DurationInDay = 1
NewTask.Title = "New task"

// Adds the task
GanttAddTask(COL_Gantt, NewTask)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Añadiendo una Appointment en un diagrama de Gantt usando el tipo GanttTask

`GanttAddTask(<Gantt column> , <Task>)`
---

**`<Gantt column>: (Nombre del control)`**

Nombre de la columna del Diagrama de Gantt (en una Tabla o control Tabla TreeView) que se va a utilizar.

**`<Task>: (GanttTask Variable)`**

Nombre de la [GanttTask](../WDLang1/1000020881.md) Variable que describe las características de la tarea a añadir.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- La nueva tarea se agrega y luego se muestra.

- La tarea se puede recuperar en una variable [GanttTask](../WDLang1/1000020881.md) utilizando [Tarea GanttGetTask](../WDLang1/1000021026.md): 
	
	```txt
	<Variable name> <- GanttGetTask(<Gantt column>, <Identifier>)
	```

	donde Identificador es el identificador de la tarea a recuperar. 
	Por ejemplo: 
	
	```wl
	// Modify a task
	MyTask is GanttTask
	MyTask <- GanttGetTask(COL_Gantt, "ID1")
	MyTask.Progress = 70
	MyTask.ProgressColor = LightGreen
	```

	A medida que el operador **<-** recupera la tarea, las modificaciones realizadas en la variable de tipo [GanttTask](../WDLang1/1000020881.md) se copiarán automáticamente al diagrama de Gantt. 
	También puede recuperar una tarea utilizando la siguiente sintaxis: 
	
	```txt
	<Variable name> <- <Gantt column>["<Identifier>"]
	```

	Por ejemplo: 
	
	```wl
	t <-COL_Gantt["ID1"]
	t.BackgroundColor = LightBlue
	```


- Un error WLanguage ocurre si la tarea no tiene identificador o si este identificador ya existe.

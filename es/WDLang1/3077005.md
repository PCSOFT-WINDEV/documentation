
## CriticalSectionEnd( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Marca el final de la sección crítica: el código será ejecutado por otro Thread.

Para marcar el inicio de la sección crítica, utilice [CriticalSectionStart](../WDLang1/3077001.md).

![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Cuando las llamadas a [CriticalSectionStart](../WDLang1/3077001.md) y [CriticalSectionEnd](../WDLang1/3077005.md) están en la misma sección del código, se recomienda utilizar el tipo [CriticalSection](../WDLang1/1000023972.md) y la sintaxis [USO ... EN](../Motscles/1510025.md). 
Ejemplo: 

```wl
// Using a Critical Section variable
S is CriticalSection

// Use for a code block
//---------------------------
USING S IN
	// code protected by the critical section
	// only one thread will execute this code at a given moment
END
```





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) 
```wl
// Critical section on a variable
t is array <critical section> of int
CriticalSectionStart(t)
// Operations on the array by one only thread at a time
CriticalSectionEnd(t)
```


<a name="Example2"></a>



```wl
// Use of a named critical section
CriticalSectionStart("MySection")
// Two threads will not be able to run this code at the same time
...
CriticalSectionEnd("MySection")
```

## Sintaxis
<a name="SYNTAX1"></a>
![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) 
### Terminar una sección crítica en una sección de código: usando el tipo CriticalSection

`CriticalSectionStart(<Critical section>)`
---

**`<Critical section>: (CriticalSection Variable)`**

Nombre de la [CriticalSection](../WDLang1/1000023972.md) Variable utilizada para identificar la sección crítica. 


<a name="SYNTAX2"></a>
![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) 
### Terminando una sección crítica de una Variable

`CriticalSectionEnd([<Variable name>])`
---

**`<Variable name>: (Cualquier tipo)`**

Nombre de la lengua de trabajo Variable que fue protegida en una sección crítica por [CriticalSectionStart](../WDLang1/3077001.md). Esta Variable debe definirse con el atributo [&lt;sección crítica&gt;](../WDLang1/1000021292.md).


<a name="SYNTAX3"></a>

### Terminar una sección crítica nombrada en una sección de código

`CriticalSectionEnd([<Section name>])`
---

**`<Section name>: (Cadena de caracteres opcional)`**

Identifica la sección crítica.

## Observaciones
<a name="NOTE0_2"></a>
<a name="NOTE0_3"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000120"></a>

- Para más detalles sobre las secciones críticas, véase [Gestionar secciones críticas](../WDLang1/1000021292.md). 

- Si una Thread es destruida mientras se encuentra en una sección crítica, la sección crítica se libera automáticamente.

- Se permite una sola Thread en todas las secciones con el mismo nombre.

- Si las secciones usan un nombre diferente, una thread puede ejecutar el código de una sección mientras que otra thread ejecuta el código de otra sección.

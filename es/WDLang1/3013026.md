


## ChangeSkinTemplate( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Cambia dinámicamente la plantilla:

- de todas las ventanas que se encuentran en la aplicación actual,

- de las ventanas que se encuentran en los componentes WINDEV utilizados por la aplicación,

- de una ventana específica que se encuentra en el proyecto.




**Atención**: 

- Esta función no tiene ningún efecto en las plantillas de tema aplicadas a un reporte.

- Esta función no tiene ningún efecto sobre las ventanas recortadas. La plantilla no se modifica. 



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Use the "Sea" skin template for all application windows
ChangeSkinTemplate("", "C:\MyApp\Sea.WDY")
```
<a name="Example2"></a>

```wl
// Use the "Emerald" skin template for the current window
ChangeSkinTemplate(MyWindow.Name, "EmeraldWDY")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = ChangeSkinTemplate(<Element to modify> , <New skin template> [, <WLanguage procedure>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se modificó la plantilla,

- <u><u><u><u>False</u></u></u></u> si se produce un problema.




**`<Element to modify>: (Constante o cadena de caracteres)`**

Este parámetro puede corresponder a:


|   |   |
| --- | --- |
| "" (cadena vacía) | La nueva plantilla se aplicará a todas las ventanas encontradas en la aplicación actual (incluyendo las ventanas encontradas en los componentes WINDEV utilizados por la aplicación).. <br>En un ensamblaje.NET generado por WINDEV, la nueva plantilla se aplicará a todas las ventanas encontradas en el ensamblaje actual y a todas las ventanas encontradas en los ensamblajes WINDEV.NET que son cargadas en paralelo por la aplicación. |
| *AllComponents* | La nueva plantilla se aplicará a las ventanas que se encuentran en todos los componentes WINDEV utilizados por la aplicación. |
| "&lt;Nombre de la ventana&gt;" | La nueva plantilla se aplicará a la ventana especificada |



**`<New skin template>: (Cadena de caracteres)`**

Nombre del plantilla que se va a aplicar (nombre y ruta del archivo "\*.WDY", si es necesario). Si este parámetro corresponde a una cadena vacía ("""):

- el plantilla definido en el editor de la ventana será aplicado por Default. **Atención**: Si un plantilla se ha establecido dinámicamente con **ChangeSkinTemplate**, tendrá prioridad.

- las ventanas ya abiertas mantendrán la plantilla actual.  




**`<WLanguage procedure>: (Nombre opcional de Procedure)`**

WLanguage Procedure ("callback" Procedure) llamado cuando el plantilla se aplica a una ventana interna tras la llamada a [ChangeSourceWindow](../WDLang1/3038053.md). 
Este procedimiento tiene el siguiente formato:

```txt
PROCEDURE <Procedure name>(<Destination internal window>, <Skin template>)
```
 donde: 

- &lt;Ventana interna de destino&gt; es un Variable de control correspondiente a la ventana interna a la que se aplica el plantilla. 

- &lt;Plantilla&gt; es una cadena que corresponde al nombre del plantilla utilizado. 






<a name="NOTE0"></a>
<a name="NOTE0_1b"></a>

## Observaciones


### No aplicar el plantilla a una de las ventanas del proyecto
<a name="aplicar_plantilla_una_las_ventanas_del_proyecto_ELTPARAGRAPHE000089"></a>

Para evitar aplicar la plantilla a una de las ventanas del proyecto: 

1. Abra la description de la ventana correspondiente. 

2. En la pestaña "Estilo", seleccione la opción "Conservar siempre el estilo o plantilla inicial". 

3. Validar. 



<a name="NOTE0_1"></a>


### "Archivo ".WDY
<a name="archivo_wdy_ELTPARAGRAPHE000104"></a>

La description de un plantilla se almacena en un archivo ".WDY".

Los archivos ".WDY" correspondientes a las plantillas tema propuestas por WINDEV se proporcionan con WINDEV.

Observaciones:

- Al crear una plantilla, el archivo ".WDY" se crea automáticamente.

- Para generar el archivo ".WDY" de una plantilla existente, abra el archivo "&lt;NombrePlantilla&gt;.STY" en WINDEV y guarde este archivo (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_Enregistrer.gif)).



<a name="NOTE0_2"></a>


### Desplegando una aplicación
<a name="desplegando_una_aplicacion_ELTPARAGRAPHE000120"></a>

Al crear el programa instalación de su aplicación, se deben añadir los siguientes elementos en la lista de elementos a instalar:

- el archivo ".WDY" del nuevo conjunto plantilla con **ChangeSkinTemplate**.

- los archivos de imagen utilizados por la nueva plantilla aplicada con la función **ChangeSkinTemplate**.




Atención: Las imágenes deben tener diferentes nombres en las diferentes plantillas de tema utilizadas, de lo contrario estas imágenes no serán modificadas. En el caso de los botones, el Image del botón no se ve afectado por el cambio, sólo se sustituye el Image del Border por el Image del plantilla.

**Observación**: Para añadir uno o más elementos a la lista de elementos del proyecto:

1. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Lista de elementos" y seleccione "Lista de elementos del proyecto".

2. Haga clic en el botón "Agregar".

3. Seleccione los archivos que desea añadir y validar.



<a name="NOTE0_3"></a>


### tipo de letra
<a name="tipo_letra_ELTPARAGRAPHE000154"></a>

Cargar dinámicamente las plantillas de tema no tiene ningún efecto en la fuente de los controles estáticos. De hecho, el cambio sistemático de la fuente podría desencadenar efectos de visualización inesperados (texto truncado o superpuesto) porque el tamaño de los controles estáticos cambia con la fuente.

Si debe cambiarse la fuente al aplicar una nueva plantilla mediante programación, utilice la propiedad [tipo de letra](../Proprietes/2510095.md) además de la función **ChangeSkinTemplate**.
<a name="NOTE0_4"></a>


### Estilo de los controles
<a name="estilo_los_controles_ELTPARAGRAPHE000168"></a>

La función **ChangeSkinTemplate** cambia dinámicamente la plantilla de TODOS los controles (controles relacionados con un estilo del proyecto o controles disociados). Para aplicar el estilo de una plantilla sólo a algunos controles, aplique [ChangeStyle](../WDLang1/1000017171.md) a los controles correspondientes.

Observación: También tiene la posibilidad de copy el estilo de un control a otro a través de [Estilo](../Proprietes/1000021472.md).




## &lt;HTML Display control&gt;.ExecuteBrowserProcedure (Function)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ejecuta un navegador WLanguage Procedure en un HTML Display control que muestra una página creada con WEBDEV.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Execute the CheckInput() browser procedure
HTM_HTMLDisplay.ExecuteBrowserProcedure("CheckInput")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<HTML Display control>.ExecuteBrowserProcedure(<Browser procedure> [, <Parameter 1> [, <Parameter N>]])`
---

**`<HTML Display control>: (Nombre del control)`**

Nombre de la pantalla HTML utilizada por control.

**`<Browser procedure>: (Cadena de caracteres)`**

Nombre del navegador WEBDEV WLanguage Procedure a ejecutar. Esta Procedure debe ser una Procedure local de la página WEBDEV mostrada en el control.

**`<Parameter 1>: (Cualquier tipo opcional)`**

El primer parámetro opcional que se pasará a la Procedure.
$~ ... |

**`<Parameter N>: (Cualquier tipo opcional)`**

El noveno parámetro opcional que se pasará a la Procedure.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
El nombre real de la Procedure utilizado por WEBDEV para JavaScript Procedure s no corresponde al nombre del navegador Procedure especificado en el editor. **&lt;HTML Display control&gt;.ExecuteBrowserProcedure** vincula automáticamente los diferentes nombres. Esta función debe utilizarse en lugar de [&lt;HTML Display control&gt;.ExecuteJS](../WDLang1/1000024372.md).

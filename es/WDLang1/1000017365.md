


## Seek( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Busca el índice de un elemento en un WLanguage array. Se pueden realizar las siguientes búsquedas:

- Busca un elemento en una array unidimensional de elementos simples.

- Busca un elemento en una array bidimensional de elementos simples según una columna.

- Busca un elemento en una array bidimensional de elementos simples según varias columnas.

- Busca un elemento en una array unidimensional de estructuras o clases.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Busque un elemento en un array de registros.

- Busca un objeto en un array.




**Observaciones**: 

- Esta función es equivalente a la función [ArraySeek](../WDLang1/3075002.md).



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyArray is array of 0 string
// Fill the array
Add(MyArray, "WINDEV")
Add(MyArray, "WEBDEV")
Add(MyArray, "WINDEV and WEBDEV")
// Find "WINDEV" 
ResIndex is int
ResSubscript = Seek(MyArray, asLinearFirst, "WINDEV")
Info("The WINDEV string is found at index: " + ResIndex)
// Returns: "The WINDEV string is found at index: 1"
```

## Sintaxis

### Buscando un elemento en una array unidimensional de elementos simples

`<Result> = Seek(<Array name> , <Type of search> , <Sought value> [, <Start index>])`
---

**`<Result>: (Integro)`**



- Índice del elemento buscado,

- -1 si el elemento no se encuentra en la array.




**`<Array name>: (Array)`**

Nombre de la variable de tipo Array a utilizar. Este array debe ser unidimensional.
Esta array también puede corresponder a una array de elementos simples de una Variable avanzada.

**`<Type of search>: (Constante de tipo Integer)`**

Tipo de búsqueda a realizar:


|   |   |
| --- | --- |
| *asBinary* | Búsqueda binaria. <br>Este modo de búsqueda es rápido, pero debe utilizarse en una array ordenada de forma ascendente, sin duplicados (véase [ArraySort](../WDLang1/3075007.md)). |
| *asLinear* | Búsqueda lineal. Comienza la búsqueda:<br><br>- del primer elemento.<br><br>- a partir del valor del parámetro &lt;índice de inicio&gt;.<br><br><br>La búsqueda se detiene en cuanto se encuentra un elemento. |
| *asLinearFirst* | Búsqueda lineal desde el primer elemento. &lt;Índice de inicio&gt; se ignora cuando se especifica esta constante. |
| *asLinearLast* | Búsqueda lineal desde el último elemento. Esta constante no debe utilizarse si se especifica &lt;índice de inicio&gt;. |
| *asLinearNext* | Búsqueda lineal del siguiente elemento. Esta búsqueda se realiza a partir de la posición actual o del valor de &lt;índice inicial&gt;. |
| *asLinearPrevious* | Búsqueda lineal del elemento anterior. Esta búsqueda se realiza a partir de la posición actual o del valor de &lt;índice inicial&gt;. |

El tipo de búsqueda se puede combinar con una o más opciones de búsqueda:


|   |   |
| --- | --- |
| *tccIgnoreAccent* | Buscar ignorando los caracteres acentuados.<br>Esta opción sólo se puede utilizar en la arrays de los strings de caracteres.<br><br> |
| *tccIgnoreCase* | Buscar ignorando el caso.<br>Esta opción sólo se puede utilizar en la arrays de los strings de caracteres. |
| *tccIgnoreInsideSpace* | Buscar ignorando los espacios dentro de las cadenas.<br>Esta opción sólo se puede utilizar en la arrays de los strings de caracteres. |
| *tccIgnorePonctuationAndSpace* | Buscar ignorando los espacios y la puntuación.<br>Esta opción sólo se puede utilizar en la arrays de los strings de caracteres.<br><br> |
| *tccIgnoreSpace* | Buscar ignorando los espacios al principio y al final de las cadenas.<br>Esta opción sólo se puede utilizar en la arrays de los strings de caracteres. |
| *tccLexicographicOrder* | Array ordenada según el orden lingüístico.<br>Esta opción es useful para una búsqueda dicotómica y no tiene efecto en una búsqueda lineal.<br>Esta opción sólo se puede utilizar en la arrays de los strings de caracteres.<br><br> |
| *tccRespectNumeric* | Array ordenada según el orden de los valores numéricos.<br>Esta opción es useful para una búsqueda dicotómica y no tiene efecto en una búsqueda lineal.<br>Esta opción sólo se puede utilizar en la arrays de los strings de caracteres.<br><br> |

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las constantes utilizadas para definir las opciones de búsqueda no están disponibles.Para una búsqueda dicotómica, las opciones de búsqueda deben ser exactamente las mismas que se utilizaron para ordenar la array (véase [ArraySort](../WDLang1/3075007.md)).

**`<Sought value>: (Cualquier tipo)`**

Valor del elemento cuyo índice se solicita.

**`<Start index>: (Entero opcional)`**

Posición de inicio de la búsqueda (sólo búsqueda lineal). Este parámetro no es necesario para una búsqueda binaria.

## Observaciones


- Esta función no se puede utilizar en la arrays fija.

- Esta función se puede utilizar en la arrays de variables avanzadas. 

- Esta función no se puede utilizar en la array de "Objetos dinámicos".

- Esta función se puede utilizar en la arrays de los objetos escritos.

	- Por ejemplo, no se puede realizar ninguna búsqueda en los siguientes array:
			
		```wl
		t is array of 2 objects dynamic
		```


	- Por ejemplo, se puede realizar una búsqueda en los siguientes array:
			
		```wl
		t is array of 2 MyClass
		```

			
		```wl
		t is array of 2 MyClass dynamic
		```





- No se puede realizar ninguna búsqueda en una array de variantes.

- No se puede realizar ninguna búsqueda en las cadenas de Unicode.
	![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Si el tipo de búsqueda es dicotómico y si el array no está clasificado, no se mostrará ningún error de WLanguage pero el resultado de la búsqueda será indefinido.

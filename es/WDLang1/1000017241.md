
## SysChangeEnvironment( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica las variables de entorno del sistema operativo que se encuentran en el ordenador actual.

**Observación**: [SysEnvironment](../WDLang1/3073014.md) se utiliza para averiguar las variables de entorno del sistema operativo que se encuentra en el ordenador actual.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
VAR is string = "VAR_USER"
IF SysEnvironment(VAR) = "" THEN
	// The variable does not exist, create it
	SysChangeEnvironment(VAR, "VAL_VAR_USER", sysEnvUser)
	Info("The variable has been added," 
		" the changes will be taken into account at the next startup")
ELSE
	// the variable exists, delete it
	SysChangeEnvironment(VAR, Null, sysEnvUser)
	Info("The variable has been deleted,"
		" the changes will be taken into account at the next startup")
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = SysChangeEnvironment(<Name> , <Value> [, <Permanent>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ha realizado la modificación, 

- <u><u><u><u>False</u></u></u></u> si se produce un error.




**`<Name>: (Cadena de caracteres)`**

Nombre del entorno Variable a modificar. El ambiente Variable se crea si no existe.

**`<Value>: (Cadena de caracteres)`**

Nuevo valor para el entorno especificado Variable.

**`<Permanent>: (Constante opcional de tipo Integer)`**

Corresponde a una de las siguientes constantes: 


|   |   |
| --- | --- |
| *sysEnvGlobal* | Cambio global para el sistema: el actual Process así como todos los nuevos Process es del sistema están afectados. El nuevo valor se tendrá en cuenta durante la próxima conexión del usuario..<br><br>Observación: Para utilizar esta constante, el usuario debe ser administrador. |
| *sysEnvProcess* <br>(Valor predeterminado) | Cambio de la actual Process solamente. |
| *sysEnvUser* | Modificación realizada sólo para el usuario actual: el actual Process así como todos los nuevos Process es iniciados por el usuario están afectados.<br><br>El nuevo valor se tendrá en cuenta durante la próxima conexión del usuario. |

![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Este parámetro no se tiene en cuenta.

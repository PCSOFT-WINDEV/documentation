


## TableCollapse (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Colapsos:

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) una sucursal que fue expandida previamente en una control Tabla TreeView. Los nodos "Hijo" no son visibles en el control Tabla TreeView. Los nodos hijo permanecen en su estado anterior (contraído o expandido).

- una pausa que se amplió previamente en un control Tabla. Los elementos de ruptura ya no son Visible en el control Tabla. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) una ventana detallada para un fila. Para obtener más información, consulte [Gestionar los detalles de una fila de un control Tabla](../WDChamp/1013200.md). 




**Recordatorio**: En tiempo de ejecución, un nodo se colapsa cuando el nombre del nodo está precedido por "+".

![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las pausas no están disponibles para controles Tabla en esta versión.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Collapses the selected branch in the TreeView Table control
TableCollapse(TVT_Schedule)
```


<a name="Example2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Collapses the "TVT_Schedule" TreeView Table control
TableCollapse(TVT_Schedule, 1)
// Equivalent to: TableCollapseAll(TVT_Schedule)
```


<a name="Example3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// Collapses the specified branch in the TreeView Table control
TableCollapse(TVT_Schedule, "MyRoot" + TAB + "MyBranch")
```


<a name="Example4"></a>

```wl
// Collapses the break of the current row in the "TABLE_Break" Table control
TableCollapse(TABLE_Break, TABLE_Break)
```

## Sintaxis
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
### Tabla TreeView: Colapsar una rama mientras se especifica el número de fila

`TableCollapse(<TreeView Table control> [, <Row number>])`
---

**`<TreeView Table control>: (Nombre del control)`**

Nombre del control Tabla TreeView a manipular.

**`<Row number>: (Entero opcional)`**

Número de la fila que contiene la rama a colapsar. La rama que se encuentra en la fila actual se colapsa si no se especifica este parámetro.  


<a name="SYNTAX2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
### Tabla TreeView: Colapsar una rama, especificando su nombre

`TableCollapse(<TreeView Table control> [, <Branch name>])`
---

**`<TreeView Table control>: (Nombre del control)`**

Nombre del control Tabla TreeView a manipular.

**`<Branch name>: (Cadena de caracteres opcional)`**

Nombre de la rama a contraer. Este parámetro tiene el siguiente formato:

```txt
"<Root name>" + TAB + ["<Name of 1st node>" + TAB + ...
["<Name of 2nd node>" + TAB + [...]]]"<Branch name>"
```


Si este parámetro no corresponde a una rama existente, se produce un error WLanguage.

La rama que se encuentra en la fila actual se colapsa si no se especifica este parámetro.  

Si dos ramas corresponden a la ruta especificada, sólo se contrae la primera.. 


<a name="SYNTAX3"></a>

### Tabla con rupturas: Colapsando un descanso

`TableCollapse(<Table control> [, <Row number> [, <Name of break>]])`
---

**`<Table control>: (Nombre del control)`**

Nombre de control Tabla a manipular. Este control debe tener una o más rupturas. De lo contrario, esta función no tiene ningún efecto.

**`<Row number>: (Entero opcional)`**

Número de la pausa fila para colapsar. Este número debe corresponder al número de la fila actual o al número de la primera fila de ruptura a colapsar. Si no se especifica este parámetro, se colapsa la ruptura de la fila actual.

**`<Name of break>: (Cadena de caracteres opcional)`**

Nombre de la ruptura al colapso. Este parámetro debe especificarse en rupturas anidadas. Permite definir la ruptura que se colapsará. Por Default, no se colapsa ninguna ruptura si las rupturas están anidadas.  


<a name="SYNTAX4"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
### Tabla y tabla TreeView: contraer una ventana de detalles

`TableCollapse(<Table control> [, <Row number>])`
---

**`<Table control>: (Nombre del control)`**

Nombre del control Tabla o Tabla TreeView a manipular. 

**`<Row number>: (Entero opcional)`**

Número de la fila cuyos detalles deben ser ocultados. Si no se especifica este parámetro, se ocultan los detalles de la fila actual..

<a name="NOTE0_1"></a>

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000205"></a>

La función **TableCollapse** puede utilizarse en: 

- una Tabla o Tabla TreeView con fuente de datos en memoria.

- una Tabla o control Tabla TreeView rellenada programáticamente.   




Esta función no se puede utilizar en controles Tabla archivo de datos (acceso directo).
<a name="NOTE0_2"></a>


### Ramificación hasta el colapso
<a name="ramificacion_hasta_colapso_ELTPARAGRAPHE000227"></a>

La función **TableCollapse** no tiene ningún efecto:


|   |   |
| --- | --- |
| Control Tabla TreeView | Control Tabla con rupturas |
| <br><br>- si no se encuentra la rama que va a colapsar.<br><br>- si la rama que colapsa es una hoja.<br><br>- si la rama que va a colapsar ya está colapsada.<br><br><br> | <br><br>- si el control Tabla no tiene rupturas.<br><br>- si la control Tabla contiene varias rupturas y no se especifica la ruptura que debe colapsarse.<br><br>- si no se selecciona ninguna fila en el control Tabla y no se especifica el parámetro &lt;Número de fila&gt;.<br><br><br> |


**Observación para los controles Tabla TreeView**: Si &lt;número de Fila&gt; o &lt;nombre de la rama&gt; corresponde a la rama en el primer fila, se colapsa toda la jerarquía del control Tabla TreeView (equivalente a [TableCollapseAll](../WDLang1/1000010003.md)).
<a name="NOTE0_3"></a>


### Banda de selección
<a name="banda_seleccion_ELTPARAGRAPHE000257"></a>

**Para los controles Tabla TreeView**:

- Si la banda de selección se posicionó en uno de los elementos del nodo a colapsar, se posiciona en el nodo colapsado después de la llamada a **TableCollapse**.

- El código de modificación se llama cada vez que se mueve la banda de selección. Si dos niveles están contraídos y la banda de selección ser posiciona en el nodo superior, se ejecutan los códigos de modificación de cada nivel.




**Para los controles Tabla con rupturas**, no se modifica la banda de selección. Se mantiene en el elemento actual incluso si la ruptura está contraída.
<a name="NOTE0_4"></a>


### Velocidad de ejecución para controles Tabla TreeView
<a name="velocidad_ejecucion_para_controles_tabla_treeview_ELTPARAGRAPHE000273"></a>

La velocidad de ejecución de la sintaxis 1 es más rápida que la de la sintaxis 2.

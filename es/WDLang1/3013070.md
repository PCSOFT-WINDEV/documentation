
## MemoryToFile( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicializa automáticamente el valor de la memoria de los elementos para el archivo de datos actual Record con los valores de los miembros: 

- de una estructura. 

- de una clase. 

- de un tipo avanzado.




**Observación**: **MemoryToFile** no se puede utilizar para escribir en el archivo de datos: las variables del archivo de datos sólo pueden ser actualizadas (es decir, las variables de la Record cargadas en memoria). Para modificar los registros, debe utilizar [HAdd](../WDLang4/3044147.md), [HModify](../WDLang4/3044042.md), [HWrite](../WDLang4/3044092.md), etc.


<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`MemoryToFile(<Name of Structure or Class> , <File name>)`
---

**`<Name of Structure or Class>: (Estructura o Clase Variable)`**

Nombre de la estructura o nombre de la clase que contiene los valores de las posiciones.

**`<File name>: (Cadena de caracteres)`**

Nombre del archivo de datos al que pertenecen los artículos de relacionado.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si un miembro de la estructura o clase se nombra como un elemento del archivo de datos HFSQL, el valor del miembro se copia en el elemento en memoria. El nombre del miembro no debe contener ningún prefijo. El caso y los caracteres acentuados se ignoran al comparar el nombre de los elementos y los nombres de los miembros.

- Si un elemento HFSQL no tiene ningún miembro en la clase o en la estructura, su valor no se modifica: no se realiza ningún copy. Se puede inicializar el valor de los elementos HFSQL antes de llamar a la función **MemoryToFile**, mediante la función [HReset](../WDLang4/3044118.md).


## RequestUpdateUIParent (Función)

	A partir de la versión <B>25</B>, esta función se conserva por motivos de compatibilidad con versiones anteriores. Se recomienda utilizar la función [RequestRefreshUIParent](../WDLang1/1000023900.md).

<a name="Use"></a>
<a name="description"></a>
Cuando se llama a esta función desde el evento "Solicitud para actualizar la visualización" de un supercontrol, modelo de controles, ventana interna o página interna, se ejecuta el evento "Solicitud para actualizar la visualización" del nivel superior (Por lo general el nivel superior es una ventana, pero también puede ser un supercontrol, un modelo de controles o una ventana interna). Este evento se ejecutará al final del proceso en curso. 

Si se llama a esta función varias veces en el mismo proceso, el evento "Solicitud para actualizar la visualización" solo se ejecutará una vez al final del proceso en curso. 

**Observación**: Esta función se utiliza principalmente: 

- cuando se actualiza la interfaz mediante hilos. 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) cuando se utiliza la programación MVP (Modelo - Vista - Presentador). Para obtener más información, consulte [Presentación MVP: Modelo - Vista - Presentador](../Editeurs/1000021496.md).

<a name="sample_code"></a>

## Ejemplo


```wl
// Process records
// The graphic interface will have to be refreshed
RequestRefreshUIParent()
```

```wl
// -----------------------------------------------------------------
// -- Request for refreshing the display of WIN_Contact_Form
// Refresh the title of the window
Myself..Title = gpclContactForm.p_sFullname

// Refresh the linked controls of the window
SourceToScreen()
```

## Sintaxis
<a name="SYNTAX1"></a>

`RequestUpdateUIParent([<Parameter 1> [... [, <Parameter N>]]])`
---

**`<Parameter 1>: (Parámetro opcional que corresponde al tipo esperado por el evento "Solicitud para actualizar la visualización")`**

Primer parámetro que se pasará al evento "Solicitud para actualizar la visualización" de la ventana o página. 



**`<Parameter N>: (Tipo de valor esperado por el evento "Solicitud para actualizar la visualización")`**

Enésimo parámetro que se pasará al evento "Solicitud para actualizar la visualización" de la ventana o página.

## Observaciones


### Evento "Solicitud para actualizar la visualización"
<a name="evento_solicitud_para_actualizar_visualizacion_ELTPARAGRAPHE000062"></a>

"Solicitud para actualizar la visualización" es uno de los eventos asociados a la ventana o página. Este evento permite agrupar el código de actualización de los controles de la ventana o página: contenido de los controles, estado (visible, en gris, etc.), título a actualizar, etc.

Este evento puede recibir parámetros. Solo se debe declarar un procedimiento en el evento. Por ejemplo: 


```wl
// Request for refreshing the display
PROCEDURE AskForRefresh(RefreshType is Boolean)
```


Este evento se ejecuta mediante solicitud previa con las siguientes funciones: 

- [RequestUpdateUI](../WDLang1/1000021378.md): Ejecuta el evento "Solicitud para actualizar la visualización" al final del proceso en curso (ejecución asíncrona).

- [ExecuteUpdateUI](../WDLang1/1000021379.md): Ejecuta inmediatamente el evento "Solicitud para actualizar la visualización". 

- **RequestRefreshUIParent**: Ejecuta el evento "Solicitud para actualizar la visualización" del nivel superior al final del proceso en curso (ejecución asíncrona).

- [ExecuteUpdateUIParent](../WDLang1/1000022961.md): Ejecuta inmediatamente el evento "Solicitud para actualizar la visualización" del nivel superior.


Al llamar a estas funciones, se puede especificar el valor de los parámetros esperados por el evento.

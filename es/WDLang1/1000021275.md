
## ManualEvent (Tipo de Variable)

*** ***
				



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El tipo **ManualEvent** permite manipular un evento manual. Se utiliza un Event manual para simplificar la sincronización entre varios hilos.

Una Event manual sólo puede ser modificada por [EventChange](../WDLang1/3077017.md).

**Observación**: Le recomendamos utilizar una variable de tipo [AutomaticEvent](../WDLang1/1000021270.md) o **ManualEvent** para crear un evento en lugar de nombrarlo mediante una cadena de caracteres. De hecho, el uso de un Variable le permite: 

- manejar el alcance de Variable,

- evitar tener 2 eventos con el mismo nombre en 2 hilos diferentes.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// MAIN THREAD
//------------------
// Define the event
s is ManualEvent
// Start the thread
ThreadExecute("Thread", threadNormal, ProcThread, s)
// Code in parallel with the thread
...
// Wait for the event triggered by the thread
EventWait("s")
```

```wl
// SECONDARY THREAD
// ----------------------
PROCEDURE ProcThread(s)

// Code in the thread
...
// Unlocks the main thread
EventOpen(s)
```





## Declaración
<a name="declaracion_ELTTEXTE000177"></a>
<a name="SYNTAX1"></a>

### Declarando un manual Event( sintaxis simple)

`MyVariable is ManualEvent`
---

En este caso, el manual Event se cierra cuando se crea.


<a name="SYNTAX2"></a>

### Declarando una sintaxis avanzada de Event( manual)

`MyVariable is ManualEvent(<Initial status>)`
---

**`<Initial status>: (Constante de tipo Integer)`**

Estado inicial de la Event:


|   |   |
| --- | --- |
| *eventOpen* | La Event está abierta cuando se crea. |
| *eventClose* | La Event se cierra cuando se crea. |









<a name="NOTE0"></a>

## Observaciones
<a name="NOTE0_1"></a>


### Utilización de las variables ManualEvent
<a name="utilizacion_las_variables_manualevent_ELTPARAGRAPHE000069"></a>

Las variables de tipo **ManualEvent** pueden utilizarse para sincronizar subprocesos entre ellos en las funciones: 

- sintaxis estándar: 
	


|   |   |
| --- | --- |
| [eventClose](../WDLang1/1000021272.md) | Cierra una sincronización Event entre varios hilos. |
| [eventOpen](../WDLang1/1000021271.md) | Abre una sincronización Event entre varios hilos. |
| [EventWait](../WDLang1/3077007.md) | Bloquea el hilo actual mientras espera a que se abra el Event especificado. |

- sintaxis con prefijo: 
	


|   |   |
| --- | --- |
| [&lt;ManualEvent Variable&gt;.Cerrar](../WDLang1/1000024382.md) | Cierra una sincronización Event entre varios hilos. |
| [&lt;ManualEvent Variable&gt;.Open](../WDLang1/1000024384.md) | Abre una sincronización Event entre varios hilos. |




Durante una llamada a [EventOpen](../WDLang1/1000021271.md) (o [&lt;ManualEvent Variable&gt;.Abrir](../WDLang1/1000024384.md)), el Event permanece abierto: 

- se liberan todos los hilos pendientes,

- todos los hilos que llegarán a la Event más tarde pasarán sin esperar.


Durante una llamada a [EventClose](../WDLang1/1000021272.md) (o [&lt;ManualEvent Variable&gt;.Close](../WDLang1/1000024382.md)), el Event se cierra: todos los hilos que llegarán a la Event más tarde permanecerán bloqueados.

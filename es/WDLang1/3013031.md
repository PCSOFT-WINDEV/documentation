


## ExecuteProcess (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Inicia la ejecución de un Event o Process mediante programación, es decir, sin que el usuario tenga que realizar ninguna acción concreta en la interfaz.. El Event/Process ejecutado puede ser: 

- un Event de un control (en una ventana, un Page o un reporte),

- una Event de una opción de menú,

- una Event de un bloque reporte,

- una Event de una ventana,

- una Event de una Page,

- un Event de un reporte,

- una Process de una Procedure.
	**Observación**: Para ejecutar una Procedure en la Thread principal de la aplicación, utilice: 

	- la función [ExecuteMainThread](../WDLang1/1000019862.md). 

	- la función [ExecuteMainThreadAsynchronous](../WDLang1/1410088684.md).





![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) El Event/Process ejecutado sólo puede ser desde un control en una ventana, desde una opción de menú, desde una ventana o un Procedure.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Execute the click process of a Button control
ExecuteProcess(BTN_Save, trtClick)

// Execute the click process of a control via indirection
ExecuteProcess({NControl}..Value, trtClick)
```
<a name="Example2"></a>

```wl
// Runs a local procedure (test) that belongs
// to another opened window/page (menu)
ExecuteProcess(Menu.Test, trtProcedure)
```
<a name="Example3"></a>

```wl
// Runs the ReadFile procedure
// and passes the file name as parameter
ExecuteProcess(ReadFile, trtProcedure, FileName)
```

## Sintaxis

### Ejecución de un Event associated con un control

`ExecuteProcess(<Control used> , <Process> [, <Parameter 1> [... [, <Parameter N>]]])`
---

**`<Control used>: (Nombre del control)`**

Nombre del control (o de la opción de menú) asociado a la Process (o Event) a ejecutar. 
Para una opción en un menú emergente, utilice la siguiente anotación: &lt;Nombre del menú emergente&gt;.&lt;Nombre de la opción>.

Atención: Si el nombre del control no se especifica directamente (con una indirección, por ejemplo), se debe usar la propiedad **Value**.

### Ejecutar el "colapso/expansión" de Event
<a name="ejecutar_colapsoexpansion_event_ELTPARAGRAPHE001187"></a>


**Para un control TreeView**
La Event "Contraer / Expandir" de un control TreeView espera los dos parámetros siguientes:

- ***&lt;Element ruta&gt;:*** Variable de texto que contiene el ruta completo del elemento actualmente colapsado o expandido. Simplemente especifique el nombre de una Variable. Se inicializará automáticamente en tiempo de ejecución.

- ***&lt;Collapsed&gt;***: Variable booleana, usada para averiguar la acción realizada por el usuario (colapso = True, expansión = False). Simplemente especifique el nombre de una Variable. Se inicializará automáticamente en tiempo de ejecución.




Para ejecutar este Event con **ExecuteProcess**, también es necesario especificar estos dos parámetros. En este caso, se debe utilizar la siguiente sintaxis:


```txt
ExecuteProcess(<TreeView control>, trtTreeviewExpandCollapse, <Element path>, <Collapsed>)
```
Para más detalles sobre el código "Collapse / Expand" de una control TreeView, véase [Eventos del lenguaje associated con un control TreeView](../WDChamp/1013110.md).

**Para un control TreeView**
El lenguaje WL "Collapse / Expand" Event de un control Tabla TreeView no espera ningún parámetro.

Para ejecutar esta Event con **ExecuteProcess**, utilice la siguiente sintaxis:


```txt
ExecuteProcess(<TreeView Table control>, trtCollapseExpandBranch)
```

### Llamando a un componente global Procedure
<a name="llamando_componente_global_procedure_ELTPARAGRAPHE001222"></a>

**ExecuteProcess** se utiliza para llamar a un Component Procedure global. Para ello, utilice la siguiente sintaxis:


```txt
ExecuteProcess(<Component>.<Set of procedures>.<Procedure>, ...
	trtProcedure)
```


**Ejemplo**:


```wl
ExecuteProcess("MyComponent.Set_WDExample.MyProcedure", trtProcedure)
```
En un hilo iniciado por un Component, **ExecuteProcess** no se puede utilizar para llamar a un Procedure local de una ventana desde el proyecto anfitrión.

Debes usar un Procedure global del proyecto anfitrión para llamar al Procedure local de la ventana.




## TableSearchChild (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Busca un valor en una columna de un control Tabla TreeView, en una rama especificada.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Searches for the "Shirt" element in the "COL_Product" column
// Generic search in the rows of level 2
ResSearch = TableSearchChild(COL_Product, "Shirt", False, 2)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Búsqueda de un valor por el índice del elemento padre

`<Result> = TableSearchChild(<Column> , <Search element> [, <Type of search> [, <Index of the parent element> [, <Source row>]]])`
---

**`<Result>: (Integro)`**



- Índice del elemento encontrado,

- -1 si ningún elemento corresponde con la búsqueda.




**`<Column>: (Nombre del control)`**

Nombre de la columna en la que se realizará la búsqueda.

**`<Search element>: (Tipo del elemento buscado)`**

Elemento que debe buscarse en la columna especificada. Si el tipo de elemento buscado es de tipo:

- "Fecha" u "Hora": la búsqueda se realiza en el valor devuelto.

- "Monetario + Euro": la búsqueda se realiza en la divisa almacenada.




**`<Type of search>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> (valor predeterminado) si la búsqueda debe ser exacta, 

- <u><u><u><u>False</u></u></u></u> si la búsqueda debe ser genérica.




**`<Index of the parent element>: (Entero opcional)`**

Índice del padre fila en la jerarquía. 

**`<Source row>: (Entero opcional)`**

Número de la fila de origen de la búsqueda en la columna. Si no se especifica este parámetro, la búsqueda se realiza en toda la columna. Este parámetro se usa para encontrar todas las instancias de una cadena en una columna.  


<a name="SYNTAX2"></a>

### Buscar un valor especificando el elemento padre

`<Result> = TableSearchChild(<Column> , <Search element> [, <Type of search> [, <Parent element> [, <Source row>]]])`
---

**`<Result>: (Integro)`**



- Índice del elemento encontrado,

- -1 si ningún elemento corresponde con la búsqueda.




**`<Column>: (Nombre del control)`**

Nombre de la columna en la que se realizará la búsqueda.

**`<Search element>: (Tipo del elemento buscado)`**

Elemento que debe buscarse en la columna especificada. Si el tipo de elemento buscado es de tipo:

- "Fecha" u "Hora": la búsqueda se realiza en el valor devuelto.

- "Monetario + Euro": la búsqueda se realiza en la divisa almacenada.




**`<Type of search>: (Booleano opcional)`**



- <u><u><u><u>True</u></u></u></u> (valor predeterminado) si la búsqueda debe ser exacta, 

- <u><u><u><u>False</u></u></u></u> si la búsqueda debe ser genérica.




**`<Parent element>: (Entero opcional)`**

Contenido de la fila matriz en la jerarquía. Este parámetro corresponde a NULL de Default: para realizar una búsqueda entre las raíces.

**`<Source row>: (Entero opcional)`**

Número de la fila de origen de la búsqueda en la columna. Si no se especifica este parámetro, la búsqueda se realiza en toda la columna. Este parámetro se usa para encontrar todas las instancias de una cadena en una columna.

## Observaciones


### Tipo de búsqueda
<a name="tipo_busqueda_ELTPARAGRAPHE000138"></a>

- ***Búsqueda exacta***: Solo los elementos cuyo valor es estrictamente igual al &lt;Elemento buscado&gt; son devueltos por la función TableSearchChild. Por ejemplo:
	
	```wl
	// Return all the customers named "Doe" found in "COL_Name"
	TableSearchChild(COL_Name, "Doe")
	```


- ***Búsqueda genérica***: Los elementos que comienzan por &lt;Elemento buscado&gt; son devueltos por la función TableSearchChild. Por ejemplo:
	
	```wl
	// Return all customers whose name starts with "SMI"
	TableSearchChild(COL_Name, "SMI", False)
	```




<a name="NOTE0_2"></a>


### Varios
<a name="varios_ELTPARAGRAPHE000166"></a>

- El elemento encontrado no está seleccionado. [TablePosition](../WDLang1/3074022.md) se utiliza para posicionarse en el fila y [TableSelectPlus](../WDLang1/3074031.md) se utiliza para seleccionar el fila.

- La búsqueda realizada con la función **TableSearchChild** no distingue entre mayúsculas y minúsculas.




## ProfilerStart( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Empieza a "recoger datos" para la [analizador de rendimiento](../Editeurs/2030035.md).

**Recordatorio**: Para iniciar el perfil de rendimiento, pulse: Shift + Windows + F10. 

Una vez que se llama a esta función, todos los procesos/eventos ejecutados se guardan en un archivo WPF.

Este archivo se puede abrir:

- por la [analizador de rendimiento](../Editeurs/2030035.md). 

- en el editor de WINDEV, WEBDEV o WINDEV Mobile: en la pestaña "Inicio", en el grupo "General", despliegue "Abrir" y seleccione "Abrir".





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
ProfilerStart("Test_StartProcedure.wpf")
StartProcedure()
ProfilerEnd()
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`ProfilerStart([<Name of WPF File>])`
---

**`<Name of WPF File>: (Cadena de caracteres opcional)`**

Ubicación y nombre del archivo WPF a crear. El nombre del archivo WPF se puede construir a partir de los siguientes elementos:


|   |   |
| --- | --- |
| [%ExeDir%] | Directorio del ejecutable, siempre rellenado con el carácter "\\".. Este directorio es equivalente al resultado de [fExeDir](../WDLang1/3036049.md). |
| [%exeName%] | Nombre del ejecutable. |
| [%Fecha%] | Fecha de inicio de la recogida de información (formato AAAAMMDDD). |
| [%Hora%] | Hora de inicio de la recogida de información (formato HHMMSS) |
| [%NombreUsuario%] | Nombre del usuario actual. |
| [%NombreDeLaComputadora%] | Nombre de el equipo. |
| [%DataDir%] | Directorio de los archivos de datos HFSQL especificados por el usuario al instalar la aplicación. Este directorio es equivalente al resultado de [fDataDir](../WDLang1/3036001.md). |
| [%DirUserData%] | Directorio para los datos del usuario actual de la aplicación actual. Este directorio es equivalente al resultado de [fDataDirUser](../WDLang1/3036071.md). |

Por Default: 

- la extensión del archivo WPF es ".wpf". 

- el nombre del archivo WPF corresponde a: "[%UserDataDir%][%ExeName%]_[%Date%]_[%Time%]". 
	Por ejemplo: "C:\\Usuarios\\Doc\\AppData\\Roaming\\Aplicaciones WINDEV\\MyApp\\MyApp_20161023_130812.wpf".




Atención: Si utiliza la construcción dinámica de cadenas ("Permitir "[% %]" en las cadenas" en la pestaña "Compilación" del proyecto description), se produce un error de compilación (identificador desconocido). En este caso, cada cadena debe ir precedida de '-%'.. Ejemplo: 

```wl
ProfilerStart("[%DirUserData%][%ExeName%]_[%Date%]_[%Time%]")
```
se convierte
```wl
ProfilerStart(-%"[%DirUserData%][%ExeName%]_[%Date%]_[%Time%]")
```




<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Se produce un error de WLanguage si se utiliza **ProfilerStart** varias veces en el mismo proceso o evento (sin llamar a [ProfilerEnd](../WDLang1/3013002.md)).

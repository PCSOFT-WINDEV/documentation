


## &lt;Tabla&gt;.ordenación (función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Ordena una tabla o control Tabla TreeView en una o más columnas (ordenación lexicográfica). 

**Observación**: Cuando se ordena un control con la función **&lt;Tabla&gt;.ordenación**, las imágenes que permiten conocer la dirección del orden no se muestran en las columnas.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Sort "COL_Designation" and "COL_Price" columns in "TABLE_Product"
// "COL_Designation" is sorted in ascending order
// "COL_Price" is sorted in descending order
TABLE_Product.Sort("COL_Designation", "-COL_Price")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Clasificar una tabla o control Tabla TreeView en una o más columnas identificadas por su nombre

`<Result> = <Table control>.Sort(<Column 1> [, <Column 2> [... [, <Column N>]]])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se ejecutó el orden,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Table control>: (Nombre del control)`**

Nombre del control a manipular. Este control puede corresponder a: 

- un control Tabla.

- un control Tabla TreeView.




**`<Column 1>: (Cadena de caracteres)`**

Nombre de la primera columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:

```txt
"[<Direction>]<Column name>"
```

donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": de tipo descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.




**`<Column 2>: (Cadena de caracteres opcional)`**

Nombre de la segunda columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:

```txt
"[<Direction>]<Column name>"
```

donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": de tipo descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.






**`<Column N>: (Cadena de caracteres opcional)`**

Nombre de la enésima columna a ordenar (hasta 16 columnas). Este parámetro tiene el siguiente formato:

```txt
"[<Direction>]<Column name>"
```

donde:

- &lt;Dirección&gt; (carácter) indica la dirección de la clasificación:

	- "+": tipo ascendente (por Default),

	- "-": de tipo descendente.




- &lt;Nombre de columna&gt; (cadena de caracteres): nombre de la columna a ordenar.





<a name="SYNTAX4"></a>

### Cancelación de una ordenación previamente definida en una Tabla o control Tabla TreeView poblada programáticamente

`<Result> = <Table control>.Sort(<False>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se canceló el orden,

- <u><u><u><u>False</u></u></u></u> si se produce un problema.




**`<Table control>: (Nombre del control)`**

Nombre del control a manipular. Este control puede corresponder a: 

- un control Tabla rellenado mediante programación.

- a control Tabla TreeView poblada programáticamente.




**`<False>: (Constante)`**

*False*: Palabra clave utilizada para eliminar una ordenación previamente definida en una Tabla o control Tabla TreeView poblada programáticamente.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000313"></a>

La función **&lt;Tabla&gt;.ordenación** puede utilizarse en:

- una Tabla o control Tabla TreeView rellenada programáticamente.

- una Tabla o control Tabla TreeView basada en un archivo de datos con iteración automatic (opción "Examinar automatic aliado" en la pestaña "Contenido" del control description). En este caso, sólo se ordena una columna.

- un control de selección única o de selección múltiple.
	



<a name="NOTE0_2"></a>


### Ordenar una tabla o control Tabla TreeView en función de un archivo de datos
<a name="ordenar_una_tabla_control_tabla_treeview_funcion_archivo_datos_ELTPARAGRAPHE000350"></a>

Una Tabla o control Tabla TreeView basada en un archivo de datos se ordenará según la primera columna especificada (sintaxis 1 y 2). Atención: la columna utilizada para la clasificación debe ser relacionado a un elemento clave en el archivo de datos utilizado para navegar por el control.

**Consejo**: Para ordenar una Tabla o control Tabla TreeView en base a un archivo de datos con varias columnas, utilice una clave compuesta y el [BrowsedItem](../Proprietes/2510111.md) Property:


```txt
TableName..BrowsedItem = NAMECOMPOSITEKEY
```
Para cancelar una ordenación en una Tabla o control Tabla TreeView basada en un archivo de datos, basta con asignar una cadena vacía al [BrowsedItem](../Proprietes/2510111.md) Property.
<a name="NOTE0_3"></a>


### Gestión de una tabla ordenada o control Tabla TreeView
<a name="gestion_una_tabla_ordenada_control_tabla_treeview_ELTPARAGRAPHE000371"></a>

- Cuando se ordena el control con la función **&lt;Tabla&gt;.ordenación**, las operaciones realizadas mediante las funciones [&lt;Table&gt;.Add](../WDLang1/1000023600.md), [&lt;Tabla&gt;.AddLine](../WDLang1/1000023404.md), [.Insert](../WDLang1/1000024260.md) y [&lt;Tabla&gt;.InsertLine](../WDLang1/1000023407.md) tienen en cuenta este orden.

- Cuando la control se ordena mediante un clic del ratón, las sumas realizadas por [&lt;Table&gt;.Add](../WDLang1/1000023600.md), [&lt;Tabla&gt;.AddLine](../WDLang1/1000023404.md), [.Insert](../WDLang1/1000024260.md) y [&lt;Tabla&gt;.InsertLine](../WDLang1/1000023407.md) ignoran la ordenación.

- ***La clasificación no se cancela*** por [&lt;Table&gt;.Display](../WDLang1/1000023505.md), [&lt;Table&gt;.Adjust](../WDLang1/1000023840.md), [TableSearch](../WDLang1/3074015.md), [&lt;Tabla&gt;.cuenta](../WDLang1/1000024277.md), [.Posición](../WDLang1/1000024278.md), [&lt;Tabla&gt;.Select](../WDLang1/1000024284.md), [&lt;Tabla&gt;.SelectMinus](../WDLang1/1000024285.md), [&lt;Tabla&gt;.SelectPlus](../WDLang1/1000024287.md), [.Delete](../WDLang1/1000024289.md) y [&lt;Tabla&gt;.DeleteAll](../WDLang1/1000024292.md).

- ***La ordenación se cancela*** si se modifica una columna (por entrada directa en la columna o por [&lt;Tabla&gt;.Modify](../WDLang1/1000024273.md) y [&lt;Tabla&gt;.ModifyLine](../WDLang1/1000024275.md)) y si se ordena.




Recordatorio: [&lt;Tabla&gt;.columna ordenada](../WDLang1/1000024054.md) devuelve la lista de columnas ordenadas.<a name="NOTE0_4"></a>


### Clasificar en una columna calculada
<a name="clasificar_una_columna_calculada_ELTPARAGRAPHE000495"></a>

No se recomienda utilizar **&lt;Tabla&gt;.ordenación** en columnas calculadas. Estas columnas se calculan cuando se muestra cada fila.

Una ordenación definida con **&lt;Tabla&gt;.ordenación** en una columna calculada sólo será válida para las filas ya visualizadas.
<a name="NOTE0_5"></a>


### Recorrer mediante un bucle programado una tabla o una vista de árbol controles Tabla
<a name="recorrer_mediante_bucle_programado_una_tabla_una_vista_arbol_controles_tabla_ELTPARAGRAPHE000511"></a>

**&lt;Tabla&gt;.ordenación** no tiene ningún efecto sobre los controles que se repiten mediante programación.
<a name="NOTE0_6"></a>


### Realizar una clasificación según el valor ASCII
<a name="realizar_una_clasificacion_segun_valor_ascii_ELTPARAGRAPHE000523"></a>

Una zona de memoria permite realizar una ordenación según el valor ASCII

Ejemplo:

```wl
MemCreate("MZ")
MemAdd("ZM", "abc", "abc")
MemAdd("ZM", "ZZZ", "ZZZ")
MemAdd("ZM", "zbc", "zbc")
MemAdd("MZ", "ééébc", "ééébc")
MemAdd("ZM", "AB", "AB")
MemAdd("ZM", "CD", "CD")
MemAdd("MZ", "çabc", "çabc")
MemAdd("ZM", "bbc", "bbc")
MemSort("MZ")
i is int
MemFirst("MZ")
WHILE NOT MemOut("MZ")
	i = MemCurrent("ZM")
	Trace(MemRetrieve("MZ", i)) // Display in ASCII order
	MemNext("MZ")
END
```

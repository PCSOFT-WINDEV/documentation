


## TelemetryIdentifyUser( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Identifica al usuario actual para el que se guardan los datos de telemetría. Los datos de telemetría ya no son datos anónimos.. 

Atención: 

- El usuario solo se identifica para el proyecto donde se utiliza la función **TelemetryIdentifyUser**.

- Cuando se utiliza esta función, los datos ya no son anónimos.. Le aconsejamos que informe al usuario y a get su acuerdo para gestionar estos datos (para cumplir con GDPR por ejemplo). 





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
TelemetryIdentifyUser(sUniqueUserName)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`TelemetryIdentifyUser(<User name>)`
---

**`<User name>: (Cadena de caracteres)`**

**Unique** nombre que se usará para almacenar los datos de telemetría associated con el usuario. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Nombre de usuario
<a name="nombre_usuario_ELTPARAGRAPHE000057"></a>

El nombre único se utiliza para identificar al usuario para todas las aplicaciones.: 

- que llaman a la función **TelemetryIdentifyUser**, 

- en el mismo servidor de telemetría.




El nombre debe ser único para cada usuario:

- para una aplicación interna, hay una forma sencilla de identificar a los usuarios: iniciales, nombre, ....

- para un paquete de software, puede ser necesaria una identificación más global para evitar colisiones: correo electrónico Address, combinación entre un nombre de empresa y un nombre de usuario, ...

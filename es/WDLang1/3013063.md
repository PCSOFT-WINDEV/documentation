## Observaciones
La función **ExecuteAncestor** se inserta automáticamente en el código si se sobrescribe el código de un control de un modelo. Esta función puede ser movida o eliminada del código.

Si existe un Event en la ventana modelo, este Event aparece dos veces en la ventana que utiliza el modelo. De hecho, se muestran los siguientes elementos:

- el Event de la modelo. Este Event aparece en gris y su título va seguido de "(&lt;Nombre de la plantilla&gt; modelo)".

- la Event de la ventana/página.




**Tsu función no debe usarse en los eventos associated con la ventana o página** (código de inicialización, código de apertura, código de cierre, etc.), porque los códigos ancestrales se ejecutan automáticamente en un orden específico. El código introducido en uno de los eventos de la página se ejecutará de la siguiente manera:  

- el Event "Declaraciones Globales" se fusiona entre el modelo y la página (o ventana) que utiliza el modelo. Por lo tanto, las variables con el mismo nombre no deben ser utilizadas en este Event.

- para todas las Event (excepto la página/ventana que cierra Event): 

	- los eventos de las plantillas aplicadas a la página (o ventana) se ejecutan en el orden en que fueron incluidas en la página (o ventana),

	- luego, se ejecuta el evento que corresponde a la página (o ventana).




- para el cierre de la página/ventana Event: se ejecuta el Event, luego se ejecutan los Event s de plantillas aplicadas a la página (o ventana) en el orden inverso al que se incluyeron en la página (o ventana).

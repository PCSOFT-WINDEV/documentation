
## RegistryCopyKey( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Copia una clave del registro con todas sus subclaves y valores. 


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
RegistryCopyKey("HKEY_LOCAL_MACHINE\SOFTWARE\MyApp", ...
	"HKEY_LOCAL_MACHINE\SOFTWARE\MyApp2")
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = RegistryCopyKey([<Access mode>, ] <Path of source key> , <Path of destination key>)`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si se copió la imagen,

- <u><u><u><u>False</u></u></u></u> en caso contrario. 




**`<Access mode>: (Constante de tipo Integer)`**

Modo de acceso al registro:



|   |   |
| --- | --- |
| *registryMode32* | Modo forzado para acceder al registro como un programa de 32 bits. |
| *registryMode64* | Modo forzado para acceder al registro como un programa de 64 bits. |
| *registryModeAuto*<br>(Valor predeterminado) | Modo de acceso al registro de automatic: <br><br>- una aplicación de 32 bits que se ejecuta en un sistema de 32 bits manipula el registro como un programa de 32 bits.<br><br>- una aplicación de 32 bits que se ejecuta en un sistema de 64 bits manipula el registro desde la siguiente rama:<br>	<br><pre><code>HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node</code></pre><br><br><br>- una aplicación de 64 bits que se ejecuta en un sistema de 64 bits manipula el registro como un programa de 64 bits.<br><br><br> |



**`<Path of source key>: (Cadena de caracteres)`**

ruta completo de la clave de copy. 

**`<Path of destination key>: (Cadena de caracteres)`**

ruta completa de la tecla de destino. 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- La función **RegistryCopyKey** realiza una copia recursiva.

- A partir de Windows Vista, las operaciones relacionadas con copy se realizan en una transacción. Si una operación falla, se cancelan todas las operaciones anteriores.

- En las versiones anteriores de Windows, las operaciones relacionado a la copy no se realizan en la transacción. Si una operación falla (por ejemplo, la copia de una subclave), las operaciones anteriores (por ejemplo, la copia de otras subclaves) no se cancelan.

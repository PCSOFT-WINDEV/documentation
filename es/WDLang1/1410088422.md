
## DiagramUngroup( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Disocia las diferentes formas de un grupo en un Diagram Editor control. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Dissociates the selected shapes if they belong to a group
IF DiagramUngroup(DIAGEDT_MyDiagram) THEN
	ToastDisplay("Group dissociated. To undo, press Ctrl+Z.")
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Result> = DiagramUngroup(<Diagram> [, <Group>])`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si los elementos del grupo han sido disociados,

- <u><u><u><u>False</u></u></u></u> en caso contrario. [ErrorInfo](../WDLang1/3013008.md) devuelve más detalles sobre el error.




**`<Diagram>: (Nombre del control)`**

Nombre del editor de Diagram control a utilizar.  

**`<Group>: (Opcional diagGroup Variable)`**

Nombre de la variable de tipo [diagGroup](../WDLang1/1410088127.md) que se utilizará. Si no se especifica este parámetro, se tratará el grupo seleccionado en Diagram Editor control.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Si el parámetro &lt;Group&gt; no es una variable de tipo [diagGroup](../WDLang1/1410088127.md), o si los elementos seleccionados no pertenecen a un grupo, la función **DiagramUngroup** no tiene efecto y devuelve <u><u><u><u>False</u></u></u></u>. 

- Los usuarios finales pueden deshacer la disociación si el Diagram está en modo de edición.

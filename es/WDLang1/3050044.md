
## MatError( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Identifica el tipo del último error causado por una función Matxxx ([funciones para la gestión de matrices](../WDLang1/3050051.md)).




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Identify the type of error
ResError = MatError("MyMatrix")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = MatError(<Matrix>)`
---

**`<Result>: (Integro)`**



- 0 si no se ha producido ningún error en la matriz,

- un código de error (valor superior a 0).




**`<Matrix>: (Cadena de caracteres)`**

Nombre de la matriz utilizada, definida con [MatCreate](../WDLang1/3050058.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Código de error
<a name="codigo_error_ELTPARAGRAPHE000081"></a>

Los códigos de error que pueden aparecer en una matriz son los siguientes:

- 8: Memoria insuficiente.

- 9: La posición solicitada está fuera de la matriz.

- 10: El nombre de la matriz no es válido.

- 11: Las matrices tienen diferentes dimensiones.

- 12: El determinante es nulo. No se puede calcular el inverso de la matriz.




**Atención**: Un error de WLanguage se desencadena por los errores 9, 10 y 11. La aplicación se detiene.


<a name="NOTE0_2"></a>


### Existencia de la matriz
<a name="existencia_matriz_ELTPARAGRAPHE000107"></a>

No se verifica la existencia de la matriz.. Para comprobar la existencia de la matriz, utilice [MatExiste](../WDLang1/3050076.md).

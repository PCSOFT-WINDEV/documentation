


## TableModifyLine (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Modifica una fila en: 

- un control Tabla, 

- un control Tabla TreeView. 





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Modify the "Paul" and "Moore" elements 
// in the 5th row of "TABLE_CustomerTable" control
TableModifyLine(TABLE_CustomerTable, 5, "Paul", "Moore")
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`TableModifyLine(<Table control> , <Index> [, <Element column 1> [... [, <Element column N>]]])`
---

**`<Table control>: (Nombre del control)`**

Nombre del control a manipular. Este control puede corresponder a: 

- un control Tabla.

- un control Tabla TreeView.


Si este parámetro corresponde a una cadena vacía (""), se utilizará el control al que pertenece el evento actual.

**`<Index>: (Integro)`**

Índice de la fila por modificar. El índice de la primera fila es 1.


|   |   |
| --- | --- |
| Índice igual a -1 | Se modifica la fila seleccionada. No tiene ningún efecto si no se selecciona ninguna fila. |
| Índice mayor que el número de filas de control | Se produce un error del WLanguage. |



**`<Element column 1>: (Cadena de caracteres opcional)`**

Primer elemento de la fila modificada. El valor de este elemento sustituye al valor existente. Cada elemento corresponde a una columna. Cada tipo de elemento modificado debe ser compatible con el tipo de columna pertinente..
Si no se especifica ninguno de los parámetros de &lt;columna X&gt;, **TableModifyLine** no tiene ningún efecto.



**`<Element column N>: (Cadena de caracteres opcional)`**

Enésimo elemento de la fila modificada. El valor de este elemento sustituye al valor existente. Cada elemento corresponde a una columna. Cada tipo de elemento modificado debe ser compatible con el tipo de columna pertinente..
Si no se especifica ninguno de los parámetros de &lt;columna X&gt;, **TableModifyLine** no tiene ningún efecto.

## Observaciones


### Condiciones de uso
<a name="condiciones_uso_ELTPARAGRAPHE000120"></a>

La función **TableModifyLine** puede utilizarse en:

- una Tabla o control Tabla TreeView basada en un archivo de datos.

- una Tabla o control Tabla TreeView rellenada programáticamente.

- un control de selección única o de selección múltiple. 
	



<a name="NOTE0_2"></a>


### Características de los elementos fila
<a name="caracteristicas_los_elementos_fila_ELTPARAGRAPHE000161"></a>

- Para modificar el valor de un elemento de una fila, utilice la siguiente sintaxis: 
	
	```txt
	TableControl.ColumnName[RowIndex] = NewValue
	```


- Si un elemento corresponde al resultado de un cálculo, la expresión numérica debe estar entre paréntesis.. 
	Por ejemplo:
	
	```wl
	TableModifyLine(TABLE_CalcTable, (53+29),(83-21))
	```

### Control Tabla o Tabla TreeView basado en un archivo de datos : Modificaciones en el archivo de datos relacionado
<a name="control_tabla_tabla_treeview_basado_archivo_datos_modificaciones_archivo_datos_relacionado_ELTPARAGRAPHE000184"></a>

- **Si la entrada en cascada está habilitada**, el archivo de datos relacionado con el control se modifica automáticamente cuando el usuario pasa a la siguiente fila.

- **Si la entrada en cascada no está habilitada**, el archivo de datos relacionado con el control no se modifica. Para escribir el contenido de la fila modificada en el archivo de datos relacionado, debe utilizarse [TableSave](../WDLang1/3074008.md) después de **TableModifyLine**.




**Observaciones**: 

- La opción "Entrada en cascada" puede seleccionarse en la pestaña "Detalles" de la ventana de descripción del control.

- ![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) La entrada en cascada no está disponible en controles Tabla basados en un archivo de datos con acceso directo. Esta opción sólo está disponible en controles Tabla poblada programáticamente o con fuente de datos en memoria.



<a name="NOTE0_4"></a>


### Modificaciones realizadas en un control Tabla/Tabla TreeView basado en un archivo de datos
<a name="modificaciones_realizadas_control_tablatabla_treeview_basado_archivo_datos_ELTPARAGRAPHE000211"></a>

Para modificar una fila de un control Tabla/Tabla TreeView, le aconsejamos:

- modificar el registro en el archivo de datos con la función [HModify](../WDLang4/3044042.md),

- utilizar la función [TableDisplay](../WDLang1/3074003.md) para volver a actualizar el control.

### Función TableModifyLine y Property modificada
<a name="funcion_docparampagetitleshort_property_modificada_ELTPARAGRAPHE000231"></a>

**TableModifyLine** no influye en el [Modified](../Proprietes/2510080.md) Property. El [Modified](../Proprietes/2510080.md) Property se utiliza para saber si el fila actual ha sido modificado directamente por el usuario (entrada directa en el control).




## TestCheckFalse( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Verifica si una expresión es falsa en la programación asertiva y en el modo de prueba automática. 

Observación: Esta función es equivalente a la función [dbgCheckFalse](../WDLang1/1000019781.md). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// The array should not be empty
TestCheckFalse(ArrayElement.Empty)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`TestCheckFalse(<Expression> [, <Comments> [, <Parameter 1> [... [, <Parameter N>]]]])`
---

**`<Expression>: (Cadena de caracteres)`**

Expresión que se debe verificar. 

**`<Comments>: (Cadena de caracteres)`**

Cadena de caracteres en el formato definido por [StringBuild](../WDLang1/3024006.md). 

**`<Parameter 1>: (Cualquier tipo)`**

Primer parámetro esperado por &lt;Comentarios&gt;.



**`<Parameter N>: (Cualquier tipo)`**

Último parámetro esperado por &lt;Comentarios>.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Modo de funcionamiento
<a name="modo_funcionamiento_ELTPARAGRAPHE000045"></a>

Cuando el valor de la expresión no es <u><u><u><u>False</u></u></u></u>:

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) en modo de prueba automatizado, se añade un error al estado de prueba reporte (equivalente a [Resultado de TestWriteResult](../WDLang1/1000013001.md)).

- en modo de prueba manual, se muestra una afirmación (equivalente a [dbgAssert](../WDLang1/3014022.md)).

- en modo de ejecución, se muestra una aserción si se han habilitado las aserciones (equivalente a [dbgAssert](../WDLang1/3014022.md)).




Cuando se ejecuta una aplicación, la expresión no se evalúa si las aserciones no están habilitadas.

[dbgOnError](../WDLang1/1000020722.md) permite iniciar automáticamente un procedimiento WLanguage utilizado para recuperar el comentario relacionado al error.

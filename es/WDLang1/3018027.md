
## TreeID( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Permite construir la ruta de los elementos que se encuentran en un control TreeView diferenciando los duplicados.. Esta función se puede utilizar en todas las funciones para manejar los elementos que se encuentran en una control TreeView.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
ResultTreeView is Boolean
ResultTreeView = TreeAdd(TREE_TreeView1, "Root" + TAB + "Leaf", ...
		tvDefault, tvDefault, "Id unique leaf")
ResultTreeView = TreeAdd(TREE_TreeView1, "Root" + TAB + "Leaf", ...
		tvDefault, tvDefault, "Id unique leaf 2", tvAcceptDuplicate)
// Delete the 2nd duplicate
TreeDelete(TREE_TreeView1, "Root" + TAB + "Leaf" +  ...
		TreeID("Unique leaf 2 ID"))
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = TreeID(<Identifier>)`
---

**`<Result>: (Cadena de caracteres)`**

Cadena de control que debe ser añadida al nombre del elemento cuando es manejada por las funciones de gestión de control TreeView.

**`<Identifier>: (Cadena de caracteres)`**

Identificador de elemento. Este identificador se utiliza para construir la cadena de control del elemento. Este identificador debe ser idéntico al utilizado en [TreeAdd](../WDLang1/3018012.md), [TreeInsert](../WDLang1/3018023.md) o [TreeModify](../WDLang1/3018013.md).  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones




### Manejo de los elementos duplicados en una control TreeView
<a name="manejo_los_elementos_duplicados_una_control_treeview_ELTPARAGRAPHE000041"></a>

La gestión de duplicados en los controles de TreeView se basa en la gestión de identificadores. Cada sección de la ruta de un elemento control TreeView puede ser associated con un identificador. Esta asociación se realiza mediante [TreeAdd](../WDLang1/3018012.md), [TreeInsert](../WDLang1/3018023.md) o [TreeModify](../WDLang1/3018013.md).

**TreeID** gestiona los duplicados por identificador. Basta con añadir la cadena control correspondiente al elemento ruta deseado.

Ejemplo:


```wl
TreeDelete(TREE_TreeView1, "Root" + TAB + "Leaf" +  ...
		TreeID("Unique leaf 2 ID"))
```


En este ejemplo, hay dos elementos "Hoja". **TreeID** identificó el elemento a eliminar.
<a name="NOTE0_2"></a>


### ¿Cuándo utilizar la función TreeID?
<a name="¿cuando_utilizar_funcion_docparampagetitleshort_ELTPARAGRAPHE000069"></a>

se puede utilizar **TreeID**:

- en todas las sintaxis de las funciones de gestión de control TreeView, tan pronto como se especifique el ruta de un elemento duplicado.

- en la sintaxis **&lt;nombre de control TreeView&gt;[&lt;Elemento ruta&gt;]**, que se utiliza para manipular fácilmente un elemento de un control TreeView.




El elemento utilizado es aquel cuyo ruta e identificador coinciden.

**Observación**: Se puede especificar un identificador para cada sección de la ruta de un elemento.

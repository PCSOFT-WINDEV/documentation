


## &lt;Ventana&gt;.Open (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Abre una ventana modal de WINDEV.

La función **&lt;Ventana&gt;.Open** también permite:

- definir la posición de la ventana,

- pasar parámetros a la ventana.




Cuando se llama a **&lt;Ventana&gt;.Open**, se ejecutan los siguientes eventos y acciones:

- el evento "Declaraciones globales" de la ventana,

- los eventos "Inicialización" de los controles de la ventana (orden no definido),

- el evento "Inicialización" de la ventana,

- la ventana se muestra (se convierte en la ventana actual).




Para obtener más detalles sobre los eventos asociados a una ventana, consulte [Eventos asociados a una ventana](../WDChamp/1010021.md).

**Observación**: Los controles de la ventana anterior se desactivan.



![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) En Java, las ventanas modales aparecen en la barra de tareas.

**Plataformas móviles**: 

- **iOS y UWP**: La función **&lt;Ventana&gt;.Open** no está disponible. Utilice la función [&lt;Ventana&gt;.OpenMobileWindow](../WDLang1/1000023887.md).

- **Android**: No se recomienda utilizar la función **&lt;Ventana&gt;.Open** en Android. Se recomienda utilizar la función [&lt;Ventana&gt;.OpenMobileWindow](../WDLang1/1000023887.md). Si es necesario procesar el valor devuelto por la ventana, utilice el evento "Cierre de una ventana hija" de la ventana llamadora.




![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Esta función solo está disponible para simplificar la Webificación de proyectos WINDEV. En un sitio WEBDEV, esta función tiene el mismo comportamiento que la función [&lt;Página&gt;.DisplayDialog](../WDLang2/1000021663.md).
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Open and reposition the "WIN_Edit" window
// The Value1 and Value2 parameters are passed to the event
// "Global declarations" of "WIN_Edit"
// ReturnedValue will retrieve the returned value
// when the "WIN_Edit" window is closed
ReturnedValue = WIN_Edit.Open(Value1, Value2)
```
<a name="3038035_Example2"></a>

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = <Window>.Open([<Parameter 1> [... [, <Parameter N>]]])`
---

**`<Result>: (Tipo del valor obtenido (opcional))`**

Valor devuelto por la ventana popup al cerrarla. Este valor es devuelto:

- por la palabra clave [RETURN](../Motscles/1510007.md) en el evento "Cierre" de la ventana,

- por la función [Close](../WDLang1/3038018.md),

- por la propiedad [ReturnedValue](../Proprietes/2510134.md) utilizada antes de cerrar la ventana.




**`<Window>: (Nombre de ventana)`**

Nombre de la ventana a abrir.

De forma predeterminada, la ventana se muestra según la posición definida en el editor. 

**`<Parameter 1>: (Tipo de valor enviado a la ventana (opcional))`**

Primer parámetro que se pasará al evento "Declaraciones globales" de la ventana a abrir. Este parámetro se pasa por referencia y se considera como una variable global a la ventana.



**`<Parameter N>: (Tipo de valor enviado a la ventana (opcional))`**

Enésimo parámetro que se pasará al evento "Declaraciones globales" de la ventana que se abrirá. Este parámetro se pasa por referencia y se considera como una variable global a la ventana.



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Parámetros pasados a la ventana a abrir
<a name="parametros_pasados_ventana_abrir_ELTPARAGRAPHE000314"></a>

Los parámetros se obtienen en el evento "Declaraciones globales" de la ventana. Simplemente escriba la siguiente línea de código al inicio del evento:

```txt
PROCEDURE <Window name>(<Parameter1> [, <Parameter2> [, ...]])
```

Para obtener más información, consulte [Ventana con parámetros](../WDChamp/1010026.md).


<a name="NOTE0_2"></a>


### Modo de apertura de la ventana
<a name="modo_apertura_ventana_ELTPARAGRAPHE000340"></a>

La ventana se abre en modo modal:

- el resultado de la función **&lt;Ventana&gt;.Open** se conocerá después de cerrar la ventana,

- el usuario no podrá hacer clic en una de las ventanas padre de la ventana actual.




Para utilizar ventanas no modales, utilice la función [&lt;Page&gt;.OpenChild](../WDLang1/1000023904.md) o [OpenSister](../WDLang1/3038045.md).

Para gestionar el modo MDI, utilice la función [&lt;Ventana&gt;.MDIOpen](../WDLang1/3052007.md).
<a name="NOTE0_3"></a>


### Número de ventanas abiertas
<a name="numero_ventanas_abiertas_ELTPARAGRAPHE000372"></a>

El número de ventanas que pueden abrirse simultáneamente depende de la memoria disponible.
<a name="NOTE0_4"></a>


### Cerrar una ventana
<a name="cerrar_una_ventana_ELTPARAGRAPHE000382"></a>

Una ventana abierta con la función **&lt;Ventana&gt;.Open** puede cerrarse con la función [Close](../WDLang1/3038018.md) (sin parámetros) desde cualquier evento de la ventana o desde un control de la ventana.

Si se cierra una ventana padre, también se cierran las ventanas hijas.
<a name="NOTE0_5"></a>


### Título de la ventana a abrir
<a name="titulo_ventana_abrir_ELTPARAGRAPHE000399"></a>

De forma predeterminada, el título de la ventana es el definido en el editor.

Para cambiar el título de la ventana, utilice la función [NextTitle](../WDLang1/3038043.md) o [CurrentTitle](../WDLang1/3038023.md).

### Ventana minimizada
<a name="ventana_minimizada_ELTPARAGRAPHE000419"></a>

Si se minimiza la ventana actual, se minimiza todo el proyecto.

El ícono que se muestra es:

- el de la ventana actual si la ventana actual tiene un ícono,

- el del proyecto actual si la ventana actual no tiene ícono.



<a name="NOTE0_7"></a>


### Abrir la ventana de un componente
<a name="abrir_ventana_componente_ELTPARAGRAPHE000434"></a>Para abrir la ventana de un componente, solo se debe utilizar el nombre de la ventana del componente (el componente del proyecto). Por ejemplo:

```wl
ComponentWindow.Open()
```
Si se produce un conflicto con un elemento del proyecto, el nombre del elemento debe ir precedido del nombre del componente. Por ejemplo:

```wl
MyComponent.Window.Open()
```
Para utilizar el nombre de la ventana en una variable, especifique el nombre del componente. Por ejemplo:

```wl
sWindow is string = "MyComponent.MyWindow"
sWindow.Open()
```


## TimeValid (Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Comprueba la validez de una hora.




<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
TimeValid("23122222")  // Returns True
TimeValid("2312")      // Returns True
TimeValid("23")        // Returns True
TimeValid("24")        // Returns False
TimeValid("23137099")  // Returns False
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Result> = TimeValid(<Time>)`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si la hora es válida o el parámetro &lt;Hora&gt; corresponde a una cadena vacía (""),

- <u><u><u><u>False</u></u></u></u> si la hora no es válida. Una hora se considera no válida si:

	- "HH" no está comprendido entre "00" y "23".

	- "MM" no está comprendido entre "00" y "59".

	- "SS" no está comprendido entre "00" y "59".

	- "CC" no está comprendido entre "00" y "99".







**`<Time>: (Cadena de caracteres, variable de tipo Date o DateTime)`**

Hora a manipular. Esta hora puede corresponder a: 

- una cadena de caracteres en formato HHMMSSCC (HH corresponde a horas, MM a minutos, SS a segundos y CC a centésimas de segundo). 

- una cadena de caracteres en formato AAAAMMDDHHmmSSCC (AAAA corresponde al año, MM al mes, DD al día, HH a las horas, mm a los minutos, SS a los segundos y CC a las centésimas de segundo). En este caso, solo se tiene en cuenta la hora. 

- una variable de tipo Time. 

- una variable de tipo DateTime. En este caso, solo se tiene en cuenta la hora.






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
**Consejo**: Para introducir una hora en el formato correcto, utilice un control Campo de entrada de tipo Hora. En este caso, la validez de la hora se verifica automáticamente.

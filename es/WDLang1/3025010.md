
## ActiveXEvent( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Asocia una Procedure escrita en WLanguage con una Event de ActiveX control.

Después de ejecutar la función **ActiveXEvent**, el procedimiento se iniciará cada vez que se active el evento seleccionado.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// -- Initialization of the ActiveX control AX_AVIReader
ActiveXEvent("My_Procedure", AX_AVIReader, "OnClick")
```

```wl
// -- Procedure My_Procedure
PROCEDURE My_Procedure(XCoord, YCoord)
Info("Coordinates of the click: " + XCoord + "," + YCoord)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`ActiveXEvent(<WLanguage procedure> , <ActiveX control> , <Event>)`
---

**`<WLanguage procedure>: (Nombre de Procedure)`**

Nombre del procedimiento WLanguage llamado cuando se activa el Event. 
Este procedimiento debe contener el número de parámetro suministrado por la Event y puede devolver un resultado (ver la documentación del ActiveX control).

**`<ActiveX control>: (Nombre del control)`**

Nombre del ActiveX control.

**`<Event>: (Cadena de caracteres)`**

Nombre del evento asociado al ActiveX que activa la Procedure.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- Se recomienda llamar a **ActiveXEvent** en la Event "Inicialización" del ActiveX control.

- **Importante**: PC SOFT no proporciona soporte técnico sobre el modo de funcionamiento de los controles ActiveX..
	Encontrará los diferentes eventos generados por el ActiveX control en su documentación.


## Wait( Función)



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
Detiene temporalmente la ejecución del programa.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Wait during 1s
Wait(100, waitRedraw)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`Wait(<Timeout> [, <Type>])`
---

**`<Timeout>: (Integro)`**

Tiempo de espera en centésimas de segundo. 

- La función no tiene ningún efecto si se utiliza un valor negativo.. 

- Si este parámetro es set a 0, la función procesa todos los eventos pendientes y toma control inmediatamente.


Independientemente del valor del parámetro, la [procesos opcionales](../WDChamp/1014004.md) de los controles se ejecuta siempre.
Observación: Este parámetro puede corresponder a: 

- un número entero que corresponde al número de centésimas de segundo,

- una variable de tipo Duration,

- la duración en un formato legible (por ejemplo, "1s" o "10cs"). 




**`<Type>: (Constante opcional)`**

Tipo de Event que se puede ejecutar durante la espera:


|   |   |
| --- | --- |
| *waitMouseAndKeyboard* | Las ventanas y los controles pueden ser redibujados.<br>Los temporizadores pueden funcionar.<br>Se pueden ejecutar las acciones del ratón (haga clic en los controles, por ejemplo) o los eventos del teclado..<br>**Observación**: Esta constante equivale a [Multitask](../WDLang1/3015004.md) utilizada con un valor negativo. |
| *waitNothing* | No se puede dirigir ningún Event. |
| *waitRedraw*<br>(predeterminado) | Las ventanas y los controles pueden ser redibujados. |
| *waitTimer* | Las ventanas y los controles pueden ser redibujados.<br>Los temporizadores pueden funcionar.<br>**Observación**: Esta constante es equivalente a [Multitask](../WDLang1/3015004.md) utilizada con un valor positivo. |

## Observaciones




### detenerse
<a name="detenerse_ELTPARAGRAPHE000084"></a>

La parada sólo afecta a la actual thread. Los otros hilos siguen funcionando normalmente.
<a name="NOTE0_2"></a>




### Consejos/Notas
<a name="consejosnotas_ELTPARAGRAPHE000092"></a>

- Se recomienda utilizar [ThreadPause](../WDLang1/3077029.md) o [ServicioEspera](../WDLang1/1000018712.md) en lugar de [Multitask](../WDLang1/3015004.md) o **Wait** cuando se utilizan varios hilos (incluso para el hilo principal de la aplicación) si el temporizador no tiene que Process acciones del usuario.

- La función **Wait** impide que el proceso termine si no se ha completado el tiempo de espera solicitado. Le aconsejamos que no utilice una temporización larga sino que realice varias temporizaciones cortas en un bucle.. Por ejemplo, puede reemplazar: 
	
	```wl
	Wait(10000, waitMouseAndKeyboard)
	```

	por:
	
	```wl
	LOOP (100)
		Wait(100, waitMouseAndKeyboard)
	END
	```


- En una aplicación de Servicio, **Wait** debe sustituirse por [ServicioEspera](../WDLang1/1000018712.md).

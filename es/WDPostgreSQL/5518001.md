


## Conector PostgreSQL nativo para WINDEV y WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000145"></a>
WINDEV y WEBDEV 28 incluyen un Conector Nativo PostgreSQL (o Acceso Nativo PostgreSQL), disponible en modo de 32 y 64 bits. Este Conector Nativo le permite acceder a una base de datos PostgreSQL desde un programa WLanguage sin utilizar ningún controlador externo.

**Atención**: Esta versión del Conector Nativo PostgreSQL funciona con WINDEV y WEBDEV versión 28.

**Observación**: El Conector Nativo PostgreSQL es un módulo opcional y gratuito que puede descargarse de [el sitio de PC SOFT](https://www.windev.com).

## Configuración necesaria
<a name="configuracion_necesaria_ELTTEXTE000169"></a>


### Condiciones para usar un conector de progreso nativo
<a name="condiciones_para_usar_conector_progreso_nativo_ELTPARAGRAPHE000029"></a>

Le recomendamos que utilice una capa de cliente PostgreSQL reciente, cuya versión es idéntica a la versión de su servidor si es posible.

Para ello, es necesario:

- copy las DLL de la capa cliente PostgreSQL al directorio de instalación WINDEV o WEBDEV en el ordenador de desarrollo. 
	Observación: También debe copy todas las dependencias proporcionadas con la capa cliente.

- copy las DLL de la capa cliente PostgreSQL al directorio ejecutable o al directorio Windows en el ordenador de desarrollo. 
	Observación: También debe copy todas las dependencias proporcionadas con la capa cliente. 

- Instalar las DLLs de Visual runtime (paquete que puede ser descargado desde el sitio de Microsoft).
	Observación: La versión del tiempo de ejecución visual depende de la capa de cliente de PostgreSQL.




**En el modo de 32 bits**, si no se encuentran estas DLL, el Conector Nativo PostgreSQL utilizará una capa de cliente estándar (PostgreSQL versión 7.4.23). Contiene todas las funciones necesarias para la Conector Nativo y es compatible con servidores hasta la versión 9.6. Si utiliza un servidor de una versión más reciente, instale una versión de la capa de cliente compatible con su servidor. 

![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) La capa cliente de PostgreSQL debe instalarse con el gestor de paquetes de la distribución Linux. También se instalarán las dependencias. 
Ejemplo (Debian): 

```txt
apt-get install libpq5
```


**Observación**: Para usar las conexiones SSL, debes instalar una capa de cliente PostgreSQL versión 9.1 o posterior.
<a name="NOTE3_2"></a>


### Fichero específico distribuido con Native PostgreSQL Connector
<a name="fichero_especifico_distribuido_con_native_postgresql_connector_ELTPARAGRAPHE000060"></a>

Para usar el conector PostgreSQL nativo: 

- en el modo de 32 bits, sólo se requiere ***wd28 0postgresql.DLL*** DLL. Este archivo se proporciona con el Conector Nativo PostgreSQL para WINDEV y WEBDEV.

- en modo de 64 bits, sólo se requiere ***wd28 0postgresql64.DLL*** DLL. Este archivo se proporciona con el Conector Nativo PostgreSQL para WINDEV y WEBDEV.

## aprovechamiento
<a name="aprovechamiento_ELTTEXTE000199"></a>


### Uso del conector PostgreSQL nativo en sus aplicaciones WINDEV y sitios WEBDEV
<a name="uso_del_conector_postgresql_nativo_sus_aplicaciones_windev_sitios_webdev_ELTPARAGRAPHE000078"></a>

Para utilizar PostgreSQL en modo nativo, debe programar en la aplicación WINDEV o en el sitio WEBDEV:

- con las funciones "SQL" de WLanguage (véase [Programación con funciones SQL](../WDPostgreSQL/5518004.md)). 

- con las funciones "HFSQL" de WLanguage (véase [Programación con funciones HFSQL](../WDPostgreSQL/5518003.md)). 




**Observaciones**: 

- No se olvide de leer el documento [limitaciones del Conector Nativo PostgreSQL.](../WDPostgreSQL/5518002.md)

- La opción "Crear automáticamente los archivos de datos si no existen" (en la pestaña "Archivos" del proyecto description) es tenida en cuenta por el Conector Nativo PostgreSQL.

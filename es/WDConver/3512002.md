## Presentación
<a name="presentacion_ELTTEXTE000144"></a>
Antes de empezar a utilizar WDConver, debe tener:

- El análisis WINDEV (archivo".WDD") en el que se deben importar los datos.. Este análisis debe haberse generado al menos una vez.
	Recordatorio: Este análisis debe contener la estructura de los archivos HFSQL en los que se importarán los datos.

- La base de datos que contiene los datos a importar.






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Elegir una conversión
<a name="elegir_una_conversion_ELTTEXTE000174"></a>
Cuando se inicia, WDConver se propone hacerlo:

- **Crear una nueva conversión description**. Esta conversión se ejecutará automáticamente al final de su description.

- **Comprar una conversión existente description** (archivo "WDV") ya creado con WDConver. La conversión se realiza automáticamente.
	Esto le permite realizar una conversión sin tener que volver a describirla..
	Un description de conversión también puede ser usado para iniciar una conversión desde una aplicación WINDEV. Para obtener más información, consulte [Iniciando WDConver en el modo de comando Line](../WDConver/3512003.md).

## Convertir los datos encontrados en un archivo de una base de datos externa en un archivo de datos HFSQL
<a name="convertir_los_datos_encontrados_archivo_una_base_datos_externa_archivo_datos_hfsql_ELTTEXTE000198"></a>
Para describir y convertir los datos encontrados en un archivo de una base de datos externa en un archivo de datos HFSQL:

1. Inicie WDConver y valide las diferentes pantallas hasta llegar al plano "Elegir la fuente de datos".

2. Seleccione la fuente de datos "Base de datos externa".

3. Seleccione el análisis HFSQL Classic (archivo ".WDD") que contiene la estructura de los archivos externos a convertir.

4. Especifique (si es necesario) la contraseña de este análisis en tiempo de ejecución. Esta contraseña se definió en la description del análisis.

5. Seleccionar el modo de acceso a la base de datos externa (opción"Tipo"). La conexión se puede establecer a través de:

	- un proveedor de OLE DB,

	- un controlador ODBC, 

	- un acceso nativo (AS/400, Oracle, SQL Server, ....).
			**Observación**: Para definir los parámetros avanzados de conexión, pulse el botón ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=btn_fleche.gif).




6. Especifique la fuente de datos de la conexión. Esta fuente de datos puede corresponder a:

	- El nombre y la ruta completa de la base de datos. 
			Por ejemplo, con Access: "C:\\MyDirectory\\AccessDatabase.MDB"

	- El nombre del servidor o alias.
			Por ejemplo, con Oracle: "Servidor_Oracle"

	- El directorio que contiene el archivo xBase (".DBF").
			Por ejemplo: "C:\\MyxBaseDirectory"




7. Escriba (si es necesario) el nombre de usuario que se utilizará para conectarse a la base de datos. 
	**Observación**: Este nombre se definió al crear la base de datos. Pueden existir varios usuarios con diferentes derechos para la misma base de datos.

8. Introduzca (si es necesario) la contraseña utilizada para conectarse a la base de datos. Esta contraseña se definió al crear la base de datos.

9. Seleccionar los ficheros (tablas) cuyos datos deben importarse. El description de estos archivos debe ser encontrado tanto en la base de datos de origen como en el análisis WINDEV de destino. 
	**Observación**: Si no se encuentra ninguna tabla, introduzca el nombre de la tabla cuyos datos deben importarse.

10. Especifique el directorio donde se crearán los archivos de datos de HFSQL Classic.

11. Si es necesario, guarde la descripción de la conversación (archivo ".WDV") para volver a utilizarla.

12. Iniciar el botón de validación de datos Conversion( del asistente). Los archivos de datos de HFSQL Classic se crean automáticamente en el directorio especificado.
	**Observación**: Si estos archivos ya existen, los registros convertidos se añadirán después de los registros existentes.

## Conversión de los datos encontrados en un fichero en formato Hyper File 5.5 en un fichero de datos HFSQL Classic
<a name="conversion_los_datos_encontrados_fichero_formato_hyper_file_55_fichero_datos_hfsql_classic_ELTTEXTE000222"></a>
Describir y convertir un archivo en formato Hyper File 5 en un archivo de datos HFSQL Classic:

1. Inicie WDConver y valide las diferentes pantallas hasta llegar al plano "Elegir la fuente de datos".

2. Seleccione la fuente de datos "Hyper File 5".

3. Seleccione el análisis en formato Hyper File 5 asociado a los datos al archivo Convert(".WDD") y especifique (si es necesario) la contraseña de este análisis en tiempo de ejecución. Esta contraseña se definió en la description del análisis.

4. Seleccione el análisis HFSQL Classic (archivo ".WDD") que contiene la estructura de los archivos que contienen los datos a convertir y especifique (si es necesario) la contraseña de este análisis en tiempo de ejecución. Esta contraseña se definió en la description del análisis.

5. Seleccionar los ficheros (ficheros".FIC") cuyos datos deben ser importados. La description de estos archivos debe encontrarse tanto en la base de datos de origen como en el análisis de destino. Especifique su nombre lógico y contraseña si es necesario.

6. Especifique el directorio donde se crearán los archivos de datos de HFSQL Classic.

7. Si es necesario, guarde la descripción de la conversación (archivo ".WDV") para volver a utilizarla.

8. Iniciar el botón de validación de datos Conversion( del asistente). Los archivos HFSQL Classic se crean automáticamente en el directorio especificado.
	**Observación**: Si estos archivos ya existen, los registros convertidos se añadirán después de los registros existentes.

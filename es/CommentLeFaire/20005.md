## Menú desplegable
<a name="menu_desplegable_ELTTEXTE000213"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Los menús desplegables existen desde las primeras versiones de Windows. Es un método fácil de implementar en una aplicación. Este tipo de menú está casi siempre presente en la primera ventana del programa.

**En WINDEV**este tipo de menú se posiciona en la parte superior de la ventana, debajo de la barra de título, y ocupa una barra simple. Sin embargo, este tipo de interfaz ha quedado obsoleto y ha sido sustituido por los menús de cinta.

**En WEBDEV**, este tipo de menú puede situarse en la parte superior del Page o en el lateral (a la izquierda en la mayoría de los casos). Posicionado en una Page modelo, se muestra en cada Page del sitio. El control de la ruta del mapa del sitio es a menudo associated con un menú en la Web para almacenar la progresión del usuario en el sitio Web.

Para obtener más información, consulte: 

- [Principales menús de Page](../WDChamp/1010041.md). 

- [Menús principales de las ventanas](../WDChamp/1010010.md).

## Barra de íconos
<a name="barra_iconos_ELTTEXTE000237"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) La barra de iconos es siempre associated con un menú desplegable. En la mayoría de los casos, una barra de Icon no se utiliza por sí sola.

La barra de Icon representa, mediante pequeños botones gráficos, los atajos para las principales opciones que se encuentran en el menú desplegable. Además, el icono que se dibuja es el mismo que el de associated con la opción equivalente en el menú desplegable.

En la mayoría de los casos, la barra de Icon incluye varias barras pequeñas de Icon, para clasificar las acciones por tema. Las barras de Icon pueden estar flotando, alineadas en la parte superior, en la parte inferior, a la derecha, a la izquierda.

Las barras Icon se utilizan a menudo en aplicaciones multiventana o MDI. En este caso, la ventana principal (que alberga el menú desplegable y las barras de Icon) está en el Background y se utiliza como soporte para las ventanas de los niños.

## Cinta
<a name="cinta_ELTTEXTE000261"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) El control Cinta de opciones es un concepto utilizado para organizar las acciones de una aplicación. Reemplaza a "Menú desplegable/Barra de íconos. Una cinta contiene pestañas y grupos. Una agrupación contiene botones simples o botones de flecha para proponer una acción o varias acciones.

Este tipo de interfaz es moderno pero es más restrictivo porque los usuarios deben get acostumbrados a esta nueva organización.

Para obtener más información, consulte [Control Cinta de opciones](../WDChamp/1000021001.md).

## Botones
<a name="botones_ELTTEXTE000285"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Usted tiene la capacidad de crear menús con botones simples. 

Este tipo de interfaz se adapta bien a un entorno táctil (pantalla táctil - tableta o aplicación para teléfonos inteligentes).. Sin embargo, los botones creados tendrán que ser más grandes (con un tamaño si es mayor que el tamaño estándar) porque se utilizará el dedo en lugar del cursor del ratón.

Ventajas de este método: muy fácil de implementar y crear. Si desea crear varios niveles de botones, todo lo que tiene que hacer es distribuirlos en diferentes ventanas o en diferentes planos.

Para obtener más información, consulte [La control Botón](../WDChamp/1013076.md).

## acoplamientos
<a name="acoplamientos_ELTTEXTE000309"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Los enlaces son una especie de botones rediseñados. Este tipo de control se utiliza principalmente en un sitio web, pero también tiene la capacidad de crear enlaces en una aplicación WINDEV. 

Se utilizan de forma similar a los botones. Los botones y los enlaces son muy similares, excepto por su apariencia.

Para obtener más información, consulte [La control Enlace](../WDChamp/1013166.md).


## ¿Cómo asociar un Image con un elemento de árbol?
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Cuando se rellena una control TreeView, una imagen es associated con cada elemento. Es un pequeño símbolo que representa una carpeta abierta o cerrada.. De hecho, cada nivel y cada elemento de la vista de árbol puede ser:

- colapsado (imagen de la carpeta cerrada de Default). 

- ampliado (imagen de la carpeta abierta por Default). 


Estas imágenes pueden modificarse mediante programación.

Observación: La imagen propuesta por Default para los elementos de un control TreeView también puede modificarse con las propiedades [CollapsedImage](../Proprietes/2513016.md) y [ExpandedImage](../Proprietes/2513015.md). 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Método
<a name="metodo_ELTTEXTE000115"></a>
Para modificar las imágenes que representan el elemento **contraído** o **expandido**, deben especificarse las imágenes deseadas en [TreeAdd](../WDLang1/3018012.md) y [TreeInsert](../WDLang1/3018023.md).

La sintaxis para la función [TreeAdd](../WDLang1/3018012.md) es la siguiente: 


```txt
TreeAdd(<TreeView name>, <Element>, <Name of collapsed image>, <Name of expanded image>)
```


Ejemplo de código:


```wl
TreeAdd(TREE_Product, "Man", "M1.jpg", "M2.jpg")
TreeAdd(TREE_Product, "Woman", "W1.jpg", "W2.jpg")


TreeAdd(TREE_Product, "Man" + TAB + "Trousers", "trousers.JPG", "trousers.JPG")
TreeAdd(TREE_Product, "Man" + TAB + "Jacket", "jacket.JPG", "jacket.JPG")
TreeAdd(TREE_Product, "Man" + TAB + "Shoes", "shoesM.JPG", "shoesM.JPG")
TreeAdd(TREE_Product, "Woman" + TAB + "Dress", "dress.JPG", "dress.JPG")
TreeAdd(TREE_Product, "Woman" + TAB + "Pullover", "pullover.JPG", "pullover.JPG")
TreeAdd(TREE_Product, "Woman" + TAB + "Shoes", "shoesW.JPG", "shoesW.JPG")
```

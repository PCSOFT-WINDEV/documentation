
## ¿Cómo se abre una ventana?
			

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>
Hay diferentes métodos para abrir o vincular ventanas en una aplicación:

- apertura de [modal](#NOTE1_1)

- apertura de [Modelo](#NOTE2_1)

- apertura de [MDI](#NOTE3_1)

- apertura de [Popup](#NOTE4_1)







<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Apertura del modus operandi
<a name="apertura_del_modus_operandi_ELTTEXTE000178"></a>
Este es el modo de apertura más común. 

La ventana que se abre se posiciona Above las ventanas anteriores y bloquea el acceso a las ventanas inferiores. Generalmente, la mayoría de las ventanas de una aplicación (o todas ellas) se abren en modo modal. Esto permite al desarrollador y al usuario manejar fácilmente la secuencia de ventanas.

Modo de funcionamiento: 

- [entablar](../WDLang1/3038035.md) se utiliza para abrir una ventana en modo modal. Se puede acceder a los controles que se encuentran en las ventanas que se encuentran a continuación mediante la programación.

- [Cerrar](../WDLang1/3038018.md) se utiliza para cerrar esta ventana y devolver el foco a la ventana que se encuentra debajo.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Ventana sin modelo
<a name="ventana_sin_modelo_ELTTEXTE000202"></a>
Este modo de apertura es menos común que el modo Modal. 

La ventana que se abre se posiciona Above las ventanas anteriores pero el usuario puede seguir accediendo a las ventanas de abajo. El usuario puede hacer clic en una ventana que esté posicionada debajo y ésta pasará a primer plano. 
Por ejemplo, una persona puede abrir el formulario de un cliente que está esperando en el mostrador y puede abrir el formulario de otro cliente para comparar los resultados, para responder sobre la Phone, ... Por lo tanto, se pueden abrir 2 ventanas de forma.

Generalmente, no todas las ventanas de una aplicación son sin modelo. Sólo algunas ventanas se abren en este modo. Estas ventanas se llaman ventanas de "niño" o ventanas de "hermano".

La programación necesaria para gestionar varias ventanas abiertas al mismo tiempo puede ser compleja, especialmente en lo que respecta a la gestión de los contextos de los archivos o las variables globales. No debería haber más de 10 ventanas abiertas en un momento dado.

Además, al usuario final le puede resultar difícil gestionar varias ventanas abiertas y accesibles al mismo tiempo.

Modo de funcionamiento: 

- [OpenChild](../WDLang1/3038021.md) y [OpenSister](../WDLang1/3038045.md) se utilizan para abrir ventanas sin modelo.

- La primera ventana abierta siempre permanece abajo, esta ventana se llama la ventana "Padre".. Las ventanas sin modelo siguen siendo siempre Above la primera ventana, pero pueden cambiar de posición. Se denominan ventanas "hijas".

- Si se abre una ventana de Above en modo Modal, todas las ventanas de abajo son inaccesibles.

## Apertura en MDI (Interfaz Multi Document)
<a name="apertura_mdi_interfaz_multi_document_ELTTEXTE000226"></a>
Este modo de apertura es menos común que el modo Modal. Este modo de apertura es muy similar al modo sin modelo.

La ventana que se abre se posiciona Above las ventanas anteriores pero el usuario puede seguir accediendo a las ventanas de abajo. El usuario puede hacer clic en una ventana que esté posicionada debajo y ésta pasará a primer plano. 
Por ejemplo, una persona puede abrir el formulario de un cliente que está esperando en el mostrador y puede abrir el formulario de otro cliente para comparar los resultados, para responder sobre la Phone... Por lo tanto, se pueden abrir 2 ventanas de forma.

En la mayoría de los casos, sólo algunas ventanas se abren en modo MDI.  Estas ventanas se llaman ventanas de "niño" o ventanas de "hermano"....
La programación necesaria para gestionar varias ventanas abiertas al mismo tiempo puede ser compleja, especialmente en lo que respecta a la gestión de los contextos de los archivos o las variables globales. No debería haber más de 10 ventanas abiertas en un momento dado.
Además, al usuario final le puede resultar difícil gestionar varias ventanas abiertas y accesibles al mismo tiempo.

Modo de funcionamiento: 

- [MDIOpen](../WDLang1/3052005.md) se utiliza para abrir una ventana en modo MDI.

- Visualmente, la primera ventana abierta siempre permanece debajo. Esta ventana se llama ventana "MDI padre".. Las ventanas abiertas en modo MDI Above la primera ventana siempre permanecen dentro de la primera ventana, pero pueden cambiar de posición. Son las llamadas ventanas "MDI child".

- Para utilizar el modo de apertura en MDI, especifique en las características de la ventana si es "MDI padre" o "MDI hijo".

- Si se abre una ventana de Above en modo modal, todas las ventanas siguientes son inaccesibles

## Apertura de POPUP
<a name="apertura_popup_ELTTEXTE000250"></a>
Una ventana POPUP es una ventana que se muestra Above otra ventana en la posición del ratón. Esta ventana se cierra automáticamente en cuanto se hace un clic fuera de esta ventana.

Este tipo de ventana puede ser muy useful para llenar un formulario mientras se ilumina la ventana principal.
Este modo de visualización también puede ser muy useful para mostrar información adicional en una pantalla.

Modo de funcionamiento: 

- [OpenPopup](../WDLang1/3038046.md) y [AbrirPopupPosición](../WDLang1/3038047.md) se utilizan para abrir una ventana en modo POPUP.


## ¿Cómo crear y ejecutar una consulta SQL mediante programación?
			

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>
Una consulta SQL puede crearse directamente a través del editor de consultas (véase [Creación de una consulta en código SQL](../Editeurs/2032060.md)). Sin embargo, puede ser useful para crear una consulta SQL a través de la programación. 

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Método
<a name="metodo_ELTTEXTE000128"></a>
Para crear una consulta SQL mediante programación: 

1. Declarar una cadena Variable donde se almacenará el código SQL de la consulta. 

2. Declarar una variable de tipo [origen de los datos](../WDLang4/1514053.md). Esta Variable representará la consulta en tiempo de ejecución. 

3. Utilice [HExecuteSQLQuery](../WDLang4/3044084.md) para ejecutar la consulta. 

4. Para leer y recuperar el resultado de la consulta, utilice las funciones de lectura estándar: HReadXXX, PARA CADA UNO, ....

5. No olvides liberar la consulta cuando ya no se utilice ([HFreeQuery](../WDLang4/1000019374.md)). 




**Consejo**
Si la consulta utiliza parámetros procedentes de variables, utilice [StringBuild](../WDLang1/3024006.md) para construir la cadena que representa el código SQL. Este consejo se utiliza en el siguiente ejemplo. 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Ejemplo
<a name="ejemplo_ELTTEXTE000152"></a>

```wl
SQLCode is string
DS is Data Source

SQLCode = [
SELECT CustomerName, City FROM CUSTOMERS
WHERE Country='%1'
]

// List the customers living in France
SQLCode = StringBuild(SQLCode, "FRANCE")

IF HExecuteSQLQuery(DS, cntDatabBase, HQueryWithoutCorrection, SQLCode) THEN
	// Loop through the result
	FOR EACH DS
		// Process the record read
		TRACE(DS.CustomerName, DS.City)
	END

	HFreeQuery(DS)
ELSE
	Info("Error while running the query.")
END
```

## Notas
<a name="notas_ELTTEXTE000176"></a>
Este modo de creación presenta varios inconvenientes:

- no se completan los nombres de los artículos.

- debe conocer e incluso dominar el lenguaje SQL. 

- sin entrada asistida de código SQL.

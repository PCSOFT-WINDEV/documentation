
## ¿Cómo se abre una página?
			

<a name="NOTE0"></a>
<a name="NOTE0_1"></a>
Hay varias funciones para mostrar una página: 

- [ PageDisplay](#NOTE1_1),

- [ PageRefresh](#NOTE2_1),

- [ PageUse](#NOTE3_1),

- [ PageDisplayDialog](#NOTE4_1),

- [ PopupDisplay](#NOTE5_1),

- [CellDisplayDialog](#NOTE6_1).







<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Función PageDisplay
<a name="funcion_pagedisplay_ELTTEXTE000212"></a>
[PageDisplay](../WDLang2/3058008.md) se utiliza para mostrar una página en un navegador. 

- En un primer momento, el contexto de la página se crea o se vuelve a crear en el servidor. 

- Luego, la página se envía al usuario de la web para que la vea en su navegador. 




Esta función se utiliza para gestionar la secuencia de páginas de un sitio web. La página que se muestra reemplaza la página anterior del navegador.

<a name="NOTE2_1"></a>


## Función de actualización de página
<a name="funcion_actualizacion_pagina_ELTTEXTE000236"></a>
[PageRefresh](../WDLang2/3058011.md) se utiliza para mostrar una página en un navegador. 

A diferencia de [PageDisplay](../WDLang2/3058008.md), el contexto de la página no se recrea en el servidor. El contexto que se encuentra en el servidor se reutiliza para preparar la página en el servidor antes de que esta página se envíe al usuario de la web y se muestre en su navegador.

El contexto de la página que ya fue creado en el servidor es reutilizado por [PageDisplay](../WDLang2/3058008.md).

También puede crear un contexto de página en el servidor mediante las funciones de gestión de contextos ([ContextoAbrir](../WDLang2/3058004.md), etc.). 

**C Este método de programación es bastante complejo y requiere un conocimiento profundo de la programación en WLanguage.**.

La página que se muestra reemplaza la página anterior del navegador.

## PáginaFunción de uso
<a name="paginafuncion_uso_ELTTEXTE000260"></a>
[PageUse](../WDLang2/3058016.md) se utiliza para mostrar una página en un navegador. 

- En un primer momento, el contexto de la página se crea o se vuelve a crear en el servidor. 

- Luego, la página se envía al usuario de la web y se muestra en su navegador. 




Esta función elimina todos los contextos de páginas anteriores del servidor. 
Esta función da la ilusión de que el sitio se inició al mostrar la primera página.

## PáginaMostrarFunción de diálogo
<a name="paginamostrarfuncion_dialogo_ELTTEXTE000284"></a>
[PageDisplayDialog](../WDLang2/1000017124.md) se utiliza para mostrar una página en un navegador above una página anterior.

- En un primer momento, el contexto de la página se crea o se vuelve a crear en el servidor. 

- Luego, la página se envía al usuario de la web y se muestra en su navegador Above la página anterior.




En la mayoría de los casos, esta página es más pequeña que las páginas estándar del sitio. Todo lo que se encuentra alrededor de la página es más o menos opaco con el fin de manejar un efecto de transparencia Above la página anterior.

Este tipo de visualización de páginas se utiliza para mantener parte de la página anterior Visible y para..:

- llenar una pantalla,

- realizar una entrada adicional, ...




Observación: Se debe incluir un botón de cierre en esta página para volver a la página anterior.. [Diálogo PageCloseDialog](../WDLang2/1000017166.md) se utiliza para gestionar el cierre de este tipo de páginas.

Este tipo de página se crea en el editor de WEBDEV como cualquier página estándar.

## Función PopupDisplay
<a name="funcion_popupdisplay_ELTTEXTE000308"></a>
[PopupDisplay](../WDLang2/1000019788.md) se utiliza para mostrar una página en un navegador above una página anterior.

- En un primer momento, el contexto de la página se crea o se vuelve a crear en el servidor. 

- Luego, la página se envía al usuario de la web y se muestra en su navegador Above la anterior.




En la mayoría de los casos, esta página es más pequeña que las páginas estándar del sitio. Todo lo que se encuentra alrededor de la página es más o menos opaco con el fin de manejar un efecto de transparencia Above la página anterior.

Este tipo de página se crea en el editor de WEBDEV. Se genera en la misma página que la página en la que se mostrará. 

Se debe crear una página Popup para que pueda ser visualizada por esta función.

## CellDisplayFunción de diálogo
<a name="celldisplayfuncion_dialogo_ELTTEXTE000332"></a>
[Diálogo CellDisplayDialog](../WDLang2/1000017139.md) es el "ancestro" de [PopupDisplay](../WDLang2/1000019788.md). El comportamiento es similar al de una página Popup.

No se crea ninguna página nueva. Se debe crear una celda (control Celda) en la página actual.. Las células están apiladas o no.

Este modo de visualización es menos práctico que las páginas Popup. Le recomendamos que utilice las páginas Popup así como [PopupDisplay](../WDLang2/1000019788.md).

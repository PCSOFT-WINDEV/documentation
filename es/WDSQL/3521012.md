
## WDSQL: Estructura de una base de datos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000208"></a>
WDSQL se utiliza para averiguar la estructura de la base de datos a la que se accede.

Para visualizar la estructura de la base de datos, haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDSQL%20-%20HC%20N%B0006%204.gif).

Esta función está disponible:

- para las bases de datos Oracle o SQL Server,

- para los otros tipos de bases de datos.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Estructura de una base de datos Oracle o SQL Server
<a name="estructura_una_base_datos_oracle_sql_server_ELTTEXTE000232"></a>
Si la base de datos actual es una base de datos Oracle o una base de datos SQL Server, al hacer clic en el botón ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDSQL%20-%20HC%20N%B0006%204.gif) Icon se muestra la siguiente ventana:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=StructureBaseDonnees.gif)


La información se muestra en diferentes pestañas.

**Observación**: La información mostrada en esta ventana concierne a todos los usuarios de la base de datos (y no sólo al usuario conectado).

| lengüeta | Contenido |
| --- | --- |
| Tablas | Listar las tablas de la base de datos creadas por el usuario seleccionado (en el combo de "Usuario")..<br>La siguiente información se muestra para cada tabla:<br><br>- El nombre de las columnas.<br><br>- El tipo de columnas HyperFileSQL. Este tipo corresponde al tipo utilizado en WINDEV.<br><br>- El tipo de columnas Oracle. Este tipo corresponde al tipo de columnas en Oracle.<br><br>- El tamaño de las columnas.<br><br><br> |
| Vistas | Enumerar las vistas de la base de datos creadas por el usuario seleccionado (en el cuadro combinado "Usuario").<br>Se muestra la siguiente información para cada vista:<br><br>- El nombre de las columnas.<br><br>- El tipo de columnas HyperFileSQL. Este tipo corresponde al tipo utilizado en WINDEV.<br><br>- El tipo de columnas Oracle. Este tipo corresponde al tipo de columnas en Oracle.<br><br>- El tamaño de las columnas.<br><br><br> |
| Sinónimos | Enumere los sinónimos de la base de datos Visible por el usuario seleccionado (en el cuadro combinado "Usuario").<br>Se muestra la siguiente información para cada sinónimo:<br><br>- La visibilidad de los sinónimos.<br><br>- El nombre del propietario del sinónimo.<br><br>- El nombre del sinónimo.<br><br>- El nombre de la fuente del objeto.<br><br>- La base de datos remota a la que pertenece el objeto (si es necesario).<br><br><br> |
| Índice | Listar los índices de la base de datos creados por el usuario seleccionado (en el combo de "Usuario")..<br>Cuando se selecciona un índice, se muestra el nombre de las columnas associated con este índice. |
| Procedimientos | Enumerar las funciones de la base de datos, los paquetes y los procedimientos almacenados creados por el usuario seleccionado (en el cuadro combinado "Usuario")..  El icono ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=IconeerreurProcedure.gif) que se encuentra delante del nombre de un Procedure indica que el Procedure contiene al menos un error de compilación.<br>Tienes la capacidad de crear, modificar o borrar un Procedure almacenado, una función o un paquete.<br>Para crear una Procedure almacenada (una función o un paquete), todo lo que tienes que hacer es especificar:<br><br>- El nombre de la Procedure almacenada (función o paquete).<br><br>- El código de la Procedure almacenada (función o paquete).<br><br><br>El Procedure almacenado (función o paquete) se crea automáticamente en la base de datos y se compila. Si es necesario, se muestra la lista de errores de compilación..<br>Ejemplo de uso de una Procedure: la siguiente sintaxis se utiliza para visualizar el resultado de una función:<br><br><pre><code>SELECT <FunctionName>(<Parameters>) from dual</code></pre><br> |
| Enlaces (sólo bases de datos Oracle) | Listar las diferentes bases de datos remotas accesibles desde la base de datos.<br>Se visualiza la siguiente información para cada base de datos:<br><br>- El nombre del propietario de la base de datos remota.<br><br>- El nombre del enlace utilizado para acceder a la base de datos remota.<br><br>- La cadena de conexión utilizada por Oracle para acceder a la base de datos remota.<br><br>- La visibilidad de la base de datos remota.<br><br><br>Ejemplo: la siguiente sintaxis se utiliza para seleccionar los datos encontrados en una tabla de una base de datos remota:<br><br><pre><code>SELECT * from <TableName>@<LinkName></code></pre><br> |
| Usuarios | Listar los usuarios definidos en la base de datos.<br>La fecha y la hora de creación se muestran para cada usuario. |

## Estructura de una base de datos distinta de Oracle y SQL Server
<a name="estructura_una_base_datos_distinta_oracle_sql_server_ELTTEXTE000256"></a>
Si la base de datos actual no es una base de datos Oracle o una base de datos SQL Server, al hacer clic en el botón ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDSQL%20-%20HC%20N%B0006%204.gif) Icon se muestra la siguiente ventana:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDSQL%20-%20HC%20N%B0010.gif)


La información se muestra en diferentes pestañas.

| lengüeta | Contenido |
| --- | --- |

| Vistas | Listar las vistas de la base de datos.<br>Se muestra la siguiente información para cada vista:<br><br>- el nombre de las columnas,<br><br>- el tipo de columnas,<br><br>- el tamaño de las columnas.<br><br><br> |

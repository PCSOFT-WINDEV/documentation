


## WDSQL: Conexión a una base de datos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000165"></a>
La primera operación a realizar para utilizar WDSQL es establecer la conexión a una base de datos. Una vez establecida esta conexión, tiene la posibilidad de crear y ejecutar consultas SQL en la base de datos utilizada.

Existen varios tipos de conexiones disponibles:

- Conexión directa a una base de datos HFSQL Classic.

- Conexión a una base de datos a través de un proveedor OLE DB.

- Conexión a una base de datos mediante un controlador ODBC.

- Conexión a una base de datos a través de un acceso nativo.




**Observación**: WDSQL se utiliza para almacenar los parámetros de varias conexiones. Consulte "Conexión personalizada" para obtener más información.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000195"></a>


### Establecimiento de una conexión con una base de datos
<a name="establecimiento_una_conexion_con_una_base_datos_ELTPARAGRAPHE000029"></a>

Para establecer una conexión con una base de datos:

1. Iniciar WDSQL: a asistente se utiliza para definir los parámetros de conexión. 
	Si WDSQL ya está iniciado, seleccione "Archivo .. Conexión" (o haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDSQL%20-%20HC%20N%B0001%201.gif)). La conexión actual se cierra automáticamente. 
	**Observación**: Los parámetros de la última conexión se proponen automáticamente.

2. Seleccione el modo de acceso a la base de datos (opción "Conexión por").
	**Observación**: WDSQL está acostumbrado a:

	- Definir los parámetros de una conexión. La conexión se puede establecer a través de:

		- un proveedor de OLE DB,

		- un controlador ODBC, 

		- un acceso nativo (AS/400, Oracle, SQL Server, ....).




- Utilizar una conexión almacenada (llamada "Conexión personalizada"). Para obtener más información, consulte [Conexión personalizada](#NOTE3_1).

3. Especifique la fuente de datos de la conexión. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDSQL%20-%20HC%20N%B0002.gif&type=thumb)
 Esta fuente de datos puede corresponder a:

	- el nombre y la ruta completa de la base de datos. 
			Por ejemplo:

		- para HFSQL: "C:\\MyDirectory\\HFDatabase.WDD"

		- para el acceso: "C:\\MyDirectory\\AccessDatabase.MDB"




- el nombre o alias del servidor.
			Por ejemplo, para Oracle: "Servidor_Oracle"

- El directorio que contiene el archivo xBase (".DBF").
			Por ejemplo: "C:\\MyxBaseDirectory"

4. Escriba (si es necesario) el nombre de usuario que se utilizará para conectarse a la base de datos. 
	**Observación**: Este nombre se definió al crear la base de datos. Pueden existir varios usuarios con diferentes derechos para la misma base de datos.

5. Escriba la contraseña si es necesario. Esta contraseña corresponde a:

	- La contraseña de ejecución del análisis WINDEV (para una base de datos HFSQL). Esta contraseña se definió en la description del análisis.

	- La contraseña utilizada para conectarse a la base de datos (para una base de datos Oracle o SQL Server). Esta contraseña se definió al crear la base de datos.
			**Observación**: Para proponer automáticamente esta contraseña la próxima vez que se inicie la conexión asistente, marque "Guardar la contraseña de conexión".




6. Pase a la etapa siguiente. Se muestra la lista de los diferentes archivos (o tablas) que se encuentran en la base de datos.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDSQL%20-%20HC%20N%B0003.gif&type=thumb)


7. Seleccione los archivos (o tablas) a utilizar con SQL en WDSQL. Especifique la contraseña del archivo y ruta para cada archivo seleccionado. 
	**Observación**: Si se utilizó la misma contraseña para todos los archivos seleccionados, escriba esta contraseña para el primer archivo seleccionado y marque "La contraseña es idéntica para todos los archivos".

8. Pase a la etapa siguiente. 
	**Observación**: Si algunos archivos seleccionados provienen de un análisis WINDEV 5.5, especifique el directorio de este análisis.

9. Se establece la conexión. Especifique la operación a realizar:

	- [Creación de una nueva consulta SQL](../WDSQL/3521008.md).

	- Editar una consulta SQL ya creada por WDSQL.

	- Iniciando WDSQL.







**Observación**: Si los parámetros de conexión son incorrectos, la conexión no se establece y la conexión asistente se inicia automáticamente.

## Conexión personalizada
<a name="conexion_personalizada_ELTTEXTE000219"></a>
WDSQL se utiliza para almacenar los parámetros de 20 conexiones. Estas conexiones se denominan "Conexiones personalizadas".

Para almacenar los parámetros de una conexión personalizada:

1. Haga clic en el botón [...] que se encuentra a la derecha de la lista "Conexión por".. Aparece la ventana de conexiones personalizadas.

2. Escriba el nombre de la conexión (cuadro combinado "Nombre de la conexión").

3. Especifique los parámetros de conexión y valide. Estos parámetros se presentan en el párrafo anterior (operaciones 2 a 5).




Para utilizar los parámetros de una conexión personalizada:

1. Haga clic en el botón [...] que se encuentra a la derecha de la lista "Conexión por".. Aparece la ventana de conexiones personalizadas.

2. Seleccione el nombre de la conexión (cuadro combinado "Nombre de la conexión") y valídelo.

## ¿Qué es una página AWP?
<a name="¿que_una_pagina_awp_ELTTEXTE000144"></a>


### Página AWP
<a name="pagina_awp_ELTPARAGRAPHE000032"></a>

Una página WEBDEV Active (archivo .AWP) es una página WEBDEV dinámica **sin contexto**. 
Se puede acceder a esta página **en cualquier momento** a través de un enlace como: 

```txt
http://www.mysite.com/myAWPpage.awp
```


Una página de Active WEBDEV puede recibir parámetros adicionales en su URL.
Ejemplo: En esta página se mostrará el producto #3254 correspondiente a la categoría 23. Incluso con estos parámetros, esta página permanece accesible en cualquier momento..


```txt
http://www.mysite.com/myAWPpage.awp?ProductID=3254&CategoryID=23
```


Las páginas de Active WEBDEV son páginas autónomas (no necesitan ninguna biblioteca (WDL) para funcionar). Sólo las imágenes utilizadas por la página deben ser encontradas en el servidor.

**Tan pronto como se guarda la Página WEBDEV de Active, la información especificada puede ser utilizada con fines de SEO**. Así, el título y el nombre del archivo generado son utilizados por los motores de búsqueda. Para mejorar el SEO, utilice un título y un nombre de archivo (este último se utiliza para construir el nombre del archivo generado con la información mostrada en la página).

### Sitio WEBDEV en modo Sesión con Páginas Active WEBDEV
<a name="sitio_webdev_modo_sesion_con_paginas_active_webdev_ELTPARAGRAPHE000053"></a>

Un sitio WEBDEV en modo Sesión puede contener una o varias Páginas Active WEBDEV. Se puede acceder a cada una de estas páginas de forma directa e independiente (sin pasar por la primera página en modo Sesión en el proyecto).

Es posible:

- **reference cada una de las páginas de Active WEBDEV por los motores de búsqueda**. En el ejemplo anterior, los motores de búsqueda reference el producto mostrado por la página.

- **dar el Address de una o varias páginas Active WEBDEV a un amigo**. En nuestro ejemplo, la página que se muestra será la relacionada con el producto solicitado.

- **incluye una o varias páginas de Active WEBDEV en favoritos**.

- **optimizar la gestión de los recursos del servidor**.



<a name="NOTE2_3"></a>


### ATENCIÓN: El modo AWP está reservado a los desarrolladores que ya están familiarizados con WEBDEV.
<a name="atencion_modo_awp_esta_reservado_los_desarrolladores_que_estan_familiarizados_con_webdev_ELTPARAGRAPHE000067"></a>

Se requiere un buen conocimiento de WEBDEV (incluyendo los conceptos de sesiones y contextos) para utilizar este modo avanzado.. En efecto, los contextos deben gestionarse mediante la programación en este modo.
Algunas características específicas de los contextos de automatic pueden no funcionar más en este modo.
Al desarrollar este tipo de páginas hay que tener en cuenta algunas limitaciones.

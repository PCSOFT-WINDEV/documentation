


## Secuenciación de informes
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000177"></a>
<a name="Sequencing"></a>
En algunos casos, puede ser necesario imprimir una serie de informes, y verlos al mismo tiempo en el visor de reporte. 

Los informes compuestos pueden utilizarse con este fin cuando la secuencia de informes se conoce de antemano. 

Si la secuencia no se conoce de antemano y si se debe programar, se puede utilizar la secuencia de informes a través de la programación.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Enchainement_Etats.gif&type=thumb)
 
![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) Observación: También tiene la capacidad de añadir a la secuencia: 

- un procesador de textos Document.

- un documento PDF.

- un XLS o XLSX Document.

- un documento diagrama.




<a name="NOTE2"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000201"></a>
<a name="NOTE2_1"></a>
Para secuenciar los informes a través de la programación: 

1. Para añadir: 

	- a reporte a la secuencia, utilice [iSequencingAdd](../WDLang5/1000020059.md). Esta función debe utilizarse para cada reporte que se añadirá a la secuencia. Esta función se utiliza para pasar parámetros a la reporte para imprimir.. 

	- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) un tratamiento de palabras Document a la secuencia, utilice [iSequencingAddDoc](../WDLang5/1000023089.md). Esta función se utiliza para especificar el tratamiento de texto Document o impresión.

	- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) a PDF Document a la secuencia, utilice [iSequencingAddPDF](../WDLang5/1000023468.md). Esta función se utiliza para especificar el PDF Document a imprimir.

	- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) un XLS o XLSX Document a la secuencia, utilice [iSequencingAddXLS](../WDLang5/1000023518.md). Esta función se utiliza para especificar el XLS o XLSX Document a imprimir.

	- ![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png)![Linux](https://doc.pcsoft.fr/ext/images/us/LX.png) a Diagram (archivo WDDiag) a la secuencia, utilice [iSequencingAddDiagram](../WDLang5/1410088662.md). Esta función se utiliza para especificar la Diagram Document a imprimir.




2. Para finalizar la secuencia e iniciar la impresión, utilice [iSequencingPrint](../WDLang5/1000019903.md). 




Ejemplo: 


```wl
MyDocument is Document = "c:\temp\generalconditions.docx"
MySpreadsheet is string = "c:\temp\prices.xls"

// Print in the report viewer
iDestination(iViewer)

// Add reports into the sequence
iSequencingAdd(RPT_Report_Portrait)
iSequencingAdd(RPT_Report_Landscape, 3)
// Add the general conditions in document format
iSequencingAddDoc(MyDocument)
// Add prices in the format of an XLS worksheet
iSequencingAddXLS(MySpreadsheet)
iSequencingPrint()
```


**Observaciones**: 

- Las páginas de los informes encontrados en la secuencia se numeran secuencialmente.. La numeración puede ser modificada por [iPageNum](../WDLang5/3046013.md). 

- Los informes que se encuentran en la secuencia pueden tener diferentes orientaciones (vertical y/u horizontal). 

- Los informes no se ejecutan cuando se ejecuta [iSequencingAdd](../WDLang5/1000020059.md): sólo se almacenan su nombre y sus parámetros. 
	Estos informes se ejecutan realmente durante la llamada a [iSequencingPrint](../WDLang5/1000019903.md). 
	Atención: Si los informes encontrados en la secuencia se basan en una consulta, ésta debe ser inicializada por [iInitReportQuery](../WDLang5/1000003046021.md) en el código de inicialización de reporte (y no antes de la llamada a [iSequencingAdd](../WDLang5/1000020059.md)).

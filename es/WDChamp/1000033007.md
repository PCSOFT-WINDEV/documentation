


## Imprimir el contenido de una control Hoja de cálculo (AAF)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000155"></a>
Las hojas de cálculo control s incluyen un menú contextual automatic que permite a los usuarios realizar diferentes acciones sobre los datos mostrados en el control. 

Una de las opciones que se encuentran en este menú contextual permite al usuario imprimir el contenido de control Hoja de cálculo (opción "Imprimir"). 

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Modo de funcionamiento
<a name="modo_funcionamiento_ELTTEXTE000179"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000020"></a>

Pueden darse varios casos:

- Si el desarrollador ha creado un reporte en este control Hoja de cálculo, este reporte se imprime. Para obtener más información, consulte [control Hoja de cálculo en un reporte](../WDChamp/1000033009.md). 

- Si ningún reporte es relacionado a este control Hoja de cálculo, el usuario puede imprimir el contenido del control Hoja de cálculo que se encuentra en la ventana.




**Observaciones**: 

- La opción "Imprimir" del menú contextual de los controles de hoja de cálculo requiere las bibliotecas de impresión de WINDEV framework.

- Mediante programación: 

	- [AAFExecute](../WDLang1/1000022099.md) asociado con el *aafPrintSpreadsheet* La constante 0650065007400 se usa para imprimir el control Hoja de cálculo especificado.

	- [AAFDisable](../WDLang1/1000022018.md) asociado con el *aafPrintSpreadsheet* La constante 0650065007400 elimina la opción "Imprimir" para el control Hoja de cálculo especificado.

	- [AAFChangeCaption](../WDLang1/1000022100.md) con la constante *affPrintSpreadsheet* se usa para cambiar el texto de la opción "Imprimir" para el control Hoja de cálculo especificado.








### Impresión de la reporte creada por el desarrollador
<a name="impresion_reporte_creada_por_desarrollador_ELTPARAGRAPHE000046"></a>

Si el promotor ha creado un reporte en el control Hoja de cálculo, la opción "Imprimir" servirá para imprimir este reporte. No se requiere programación específica. 

Para obtener más información, consulte [control Hoja de cálculo en un reporte](../WDChamp/1000033009.md). 


### Imprimir directamente el contenido de la control Hoja de cálculo
<a name="imprimir_directamente_contenido_control_hoja_calculo_ELTPARAGRAPHE000056"></a>

Si la control Hoja de cálculo contiene varias páginas, aparecerá una ventana con las siguientes opciones: 

- orientación utilizada para la impresión.

- escalada de Document.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=FAA_Imprimer_Tableur%20-%20HC%20N%B0001.gif)



El contenido de la control Hoja de cálculo se visualiza en el visor de reporte.

Si el número de filas encontradas en la control Hoja de cálculo es muy importante (más de 10 000 filas), la aplicación pide al usuario confirmación (la impresión puede ser bastante larga y puede utilizar mucha memoria).

Las características de impresión son las siguientes:

- Se imprime la hoja de trabajo actual que se muestra en el control Hoja de cálculo. 

- El título de la impresión corresponde al título de control Hoja de cálculo. El título de la ventana se utiliza si no se especificó un título para la control.

- Los títulos de las columnas están impresos en cada Page.

- La impresión respeta los colores de Background usados para las diferentes células encontradas en el control Hoja de cálculo. 

- La fecha de hoy está impresa en la parte superior derecha Corner.

- El número de la Page actual y el número total de Page s se imprimen en la parte inferior derecha Corner.





### Personalización de la ventana de configuración de impresión de control Hoja de cálculo
<a name="personalizacion_ventana_configuracion_impresion_control_hoja_calculo_ELTPARAGRAPHE000078"></a>

WINDEV le permite incluir en su aplicación la ventana de configuración de impresión de control Hoja de cálculo utilizada por el AAF. Usted tendrá la posibilidad de personalizar:

- el aspecto de esta ventana (por ejemplo, modificación de plantilla), 

- los idiomas soportados por esta ventana. Esta característica es muy useful a la hora de traducir esta ventana a los diferentes idiomas disponibles en la aplicación.  




Para incluir la ventana para configurar la impresión en su aplicación: 

1. En la pestaña "Proyecto", en el grupo "Proyecto", despliegue "Importar" y seleccione "Elementos WINDEV y dependencias".

2. Seleccione el subdirectorio que contiene los elementos a importar. La ventana de configuración de la impresión se encuentra en el subdirectorio "Programs\\Data\\Preset windows\\AAF" del directorio de instalación de WINDEV.

3. WINDEV hace una lista de los elementos del directorio. Este directorio contiene los elementos que corresponden a las FAA personalizables. La ventana a importar se llama "WinDevAaf_ConfigSpreadsheetPrint.wdw".

4. Validar. La ventana está incluida en el proyecto y se puede modificar. Esta ventana personalizada se tendrá en cuenta automáticamente al ejecutar la aplicación.




Observaciones:

- No se debe cambiar el nombre de la ventana.

- Si agrega esta ventana a la aplicación, es posible que no obtenga las mejoras realizadas en ella durante las diferentes actualizaciones. No olvide comprobar si esta ventana ha evolucionado.

- Atención: Para modificar la ventana, el archivo correspondiente no debe ser de solo lectura. Modifique las características del archivo en Windows si es necesario.

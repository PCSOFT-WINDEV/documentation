


## Visualizador Reporte (Vista previa de impresión)
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000196"></a>
El visor de reporte permite ver el diseño de reporte sin imprimirlo (vista previa de reporte). El diseño y el contenido del reporte se pueden verificar. También puede imprimir el reporte.

El visor reporte está disponible: 

- en la editor de reportes de WINDEV, WEBDEV, WINDEV Mobile. 

- en "Reports and Queries".

- al imprimir informes desde una aplicación WINDEV. 




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Características del visor de reporte
<a name="caracteristicas_del_visor_reporte_ELTTEXTE000220"></a>
**El visor de informes propone varias características**: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Apercu_impression%20-%20HC%20N%B0001.gif&type=thumb)


- Un menú tipo cinta que incluye las siguientes pestañas: 

	- [Vista previa](../WDChamp/1011089.md): se utiliza para gestionar la vista previa actual: Zoom, retícula, mover, mover entre páginas. 

	- [letra de molde](../WDChamp/1011090.md): se utiliza para gestionar la impresión directa en la impresora. 

	- [Exportar](../WDChamp/1011088.md): utilizado para gestionar los diferentes modos de exportación (PDF, Word, Excel, etc.). 

	- [descubrimiento](../WDChamp/1011087.md): se utiliza para gestionar la búsqueda en la vista previa. 

	- [Anotar](../WDChamp/1011086.md): se utiliza para anotar la impresión que se muestra actualmente en la vista previa.


Este menú también está disponible cuando se imprime en WLanguage en el visor de reporte.

- Un menú contextual con la mayoría de las opciones.




**Observación**: Para ver una reporte en el visor de reporte, se debe instalar una impresora en el ordenador actual.

## Otras características
<a name="otras_caracteristicas_ELTTEXTE000244"></a>


- **Personalización**: Su aplicación utiliza un plantilla? ¿Por qué no utilizarlo también para el visor de reporte?. 

- ** Reporte visor e impresión con funciones WLanguage**: Si imprime con las funciones WLanguage, también puede aprovechar las características del visor de reporte. 

- **Configuración de diferentes opciones**: Para mostrar sólo algunas pestañas u opciones:

	- utilizar [iParameterViewer](../WDLang5/1000024634.md). Esta función, utilizada en la "Inicialización" de Event del proyecto, permite elegir las pestañas y opciones que se mostrarán para todos los informes de la aplicación.

	- configurar directamente las diferentes opciones que se mostrarán para cada reporte en la ventana reporte description (pestaña "UI").




- **Impresión desde el visor de reporte**: Para saber si el usuario imprimió desde el visor de reporte, utilice [iDocumentoImpreso](../WDLang5/3046029.md).

## Notas
<a name="notas_ELTTEXTE000268"></a>


### Visualizar los efectos durante un zoom
<a name="visualizar_los_efectos_durante_zoom_ELTPARAGRAPHE000114"></a>

Los efectos de visualización pueden ocurrir cuando se realiza un zoom de reporte en el visor de reporte. Por ejemplo, algunas líneas muy finas pueden no ser mostradas en el visor de reporte. Sin embargo, estas líneas se imprimirán cuando la reporte se imprima en una impresora.
<a name="NOTE4_2"></a>


### Impresión desde el visor de reporte
<a name="impresion_desde_visor_reporte_ELTPARAGRAPHE000121"></a>

Para imprimir **un documento completo** desde el visor de reportes, debe esperar a que se muestren todas las páginas.
<a name="NOTE4_3"></a>


### Reejecución de la reporte
<a name="reejecucion_reporte_ELTPARAGRAPHE000130"></a>

El reporte visualizado en el visor de reporte se vuelve a ejecutar automáticamente cuando el reporte se imprime o se exporta desde el visor de reporte. Los eventos de WLanguage associated con reporte se vuelven a ejecutar. 

En este caso, los datos mostrados en el reporte pueden ser modificados (para tener en cuenta los datos modificados por los otros usuarios, por ejemplo).

Las variables usadas por el reporte deben ser inicializadas en el código de inicialización del reporte (no en el código del control Botón que inicia el visor del reporte).

**Observación**: Las funciones WLanguage utilizadas en su reporte pueden devolver un resultado diferente cuando se vuelve a ejecutar el reporte.. Por ejemplo, si WLanguage funciona en su reporte manipular la ventana actual, cuando se vuelva a ejecutar el reporte, la ventana actual corresponderá a la ventana del visor reporte.
<a name="NOTE4_4"></a>


### Uso de Outlook Express y envío de correos electrónicos desde el visor de reporte
<a name="uso_outlook_express_envio_correos_electronicos_desde_visor_reporte_ELTPARAGRAPHE000143"></a>

Si utiliza Outlook Express y envía correos electrónicos desde el visor de reporte, puede producirse el siguiente problema: la ventana de Email de Outlook Express ya no permite introducir la información de Email. Todos los controles son Grayed.

Esto es causado por un problema de Outlook. 

Asegúrese de tener la última actualización de Outlook.

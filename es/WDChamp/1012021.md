
## Período de validez de las páginas del sitio
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000138"></a>
Al crear una página, se puede definir un período de validez para esta página.

Por ejemplo, si una página está relacionada con un evento específico (una feria de muestras, una promoción de ventas), esta página ya no debe mostrarse a partir de una fecha determinada.

Para evitar que se olvide:

- WEBDEV informará al desarrollador cuando abra el proyecto de acuerdo a una frecuencia configurable.. Se mostrará automáticamente un error de UI.

- El servidor de aplicaciones enviará un correo electrónico a una dirección determinada con un mensaje específico.




Los riesgos de olvidar se minimizan.

<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000162"></a>


### Definición del período de validez de las páginas
<a name="definicion_del_periodo_validez_las_paginas_ELTPARAGRAPHE000027"></a>

Para definir el período de validez de una página:

1. Abra la descripción de la página.

2. En la pestaña "Detalles":

	- Especifique el período de validez de la página.

	- Haga clic en el botón "Configurar los recordatorios. En la ventana que se abre, especifique:

		- la dirección de correo electrónico del destinatario del recordatorio. Si el WEBDEV servidor de aplicaciones está configurado para encontrar las páginas caducadas, se enviará automáticamente un mensaje recordatorio a esta dirección.

		- el texto añadido al mensaje de recordatorio.

		- la fecha de envío del recordatorio (una semana y un día antes por defecto). Esta fecha también se utilizará para mostrar un error de compilación de UI. Este error de compilación indicará si se ha alcanzado el período de validez de la página.




3. Valide



<a name="NOTE2_2"></a>


### Borrar el período de validez
<a name="borrar_periodo_validez_ELTPARAGRAPHE000045"></a>

Para borrar el período de validez de una página:

1. Abra la descripción de la página.

2. En la ficha "Detalles", borre el período de validez de la página.

3. Valide

![Windows](https://doc.pcsoft.fr/ext/images/us/WINDOWS.png) 

### Configurando el Windows servidor de aplicaciones para gestionar el período de validez
<a name="configurando_windows_servidor_aplicaciones_para_gestionar_periodo_validez_ELTPARAGRAPHE000058"></a>

Para que el WEBDEV servidor de aplicaciones compruebe el período de validez de las páginas (versión 140025 y posterior):

1. Inicie el administrador de WEBDEV si es necesario.

2. En la pestaña "Avanzado", marque "Permitir la búsqueda de páginas caducadas".

3. La ventana de configuración se muestra automáticamente.

4. Especifique:

	- la hora de inicio de la búsqueda de las páginas caducadas. A la hora especificada, el servidor de aplicaciones realizará una búsqueda de páginas caducadas en todos los sitios WEBDEV dinámicos gestionados por el servidor.

	- la dirección de correo electrónico del remitente del mensaje de alerta. Los parámetros de los destinatarios de correo electrónico se han definido en la descripción de la página, en el editor.

	- la dirección y el puerto del servidor SMTP utilizado para enviar correos electrónicos. También puede especificar si se debe utilizar el administrador de trabajos de impresión de correo electrónico.

	- el nombre de usuario y la contraseña del usuario del servidor SMTP (para los servidores con autenticación).




5. Valide.

### Configurando el servidor de aplicaciones de Linux para gestionar el período de validez
<a name="configurando_servidor_aplicaciones_linux_para_gestionar_periodo_validez_ELTPARAGRAPHE000078"></a>

Para que el WEBDEV servidor de aplicaciones compruebe el período de validez de las páginas (versión 140025 y posterior):

1. Inicie el administrador remoto de WEBDEV si es necesario.

2. En las opciones de ajuste, muestre los "Parámetros avanzados" y haga clic en la pestaña "Páginas caducadas".

3. Marque "Permitir la búsqueda de páginas caducadas".

4. Especifique:

	- la hora de inicio de la búsqueda de las páginas caducadas. A la hora especificada, el servidor de aplicaciones realizará una búsqueda de páginas caducadas en todos los sitios WEBDEV dinámicos gestionados por el servidor.

	- la dirección de correo electrónico del remitente del mensaje de alerta. Los parámetros de los destinatarios de correo electrónico se han definido en la descripción de la página, en el editor.

	- la dirección y el puerto del servidor SMTP utilizado para enviar correos electrónicos. También puede especificar si se debe utilizar el administrador de trabajos de impresión de correo electrónico.

	- el nombre de usuario y la contraseña del usuario del servidor SMTP (para los servidores con autenticación).




5. Haga clic en el botón "Aplicar" para validar.


## Mandos que se pueden mover en una Page
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000139"></a>
WEBDEV propone dos modos de desplazamiento de los mandos en una Page:

- **Controles que se pueden mover a través de la programación**: hay varias propiedades disponibles para posicionar los controles.

- **Controles que se pueden mover con el ratón**: tiene la posibilidad de especificar que una control puede ser movida por el usuario con el ratón. La nueva posición de la control puede ser guardada.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Controles que se pueden mover a través de la programación
<a name="controles_que_pueden_mover_traves_programacion_ELTTEXTE000163"></a>
Los mandos Page pueden desplazarse dinámicamente mediante programación.

[X](../Proprietes/2510135.md) y [Y](../Proprietes/2510137.md) están disponibles en lectura/escritura, en código de servidor y en código de navegador.

Es posible:

- permiten a los usuarios posicionar controles o grupos de controles.

- modificar dinámicamente la ubicación de los controles en función de algunos procesos y acciones.




Ejemplo: Siguiendo el puntero del ratón: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Deplacement_Souris4.gif)

## Controles que se pueden mover con el ratón
<a name="controles_que_pueden_mover_con_raton_ELTTEXTE000187"></a>
En una Page, los usuarios pueden desplazar directamente los controles con el ratón. Los usuarios podrán reorganizar la Page a su gusto. También puede almacenar la nueva posición de la control definida por el usuario. Así, la próxima vez que se abra el Page, los mandos estarán situados en la posición deseada.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Deplacement_souris3.gif)


Esta característica se utiliza principalmente para mover fácilmente los controles que se encuentran en una celda, en un looper o en un supercontrol, ....

**Para permitir que los controles se muevan con el mouse**: 

1. Abra la ventana de descripción del control.

2. En la pestaña "UI", marque las opciones deseadas en la zona "Superponer y desplazar".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Champ_Deplacable_Page%20-%20HC%20N%B0001.gif&type=thumb)





Si la control se puede apilar, se puede mover con el ratón y se puede almacenar su nueva posición.

**Observación**: puede definir una control que pueda desplazarse con el ratón en una Page interna . Esta control puede moverse en la Page que aloja la Page interna.
<a name="NOTE3_2"></a>


### Controles de looper, supercontrol y controles de célula
<a name="controles_looper_supercontrol_controles_celula_ELTPARAGRAPHE000062"></a>

Para que los Looper control s, los Supercontrol s y los Cell control s se puedan mover con el ratón, también hay que especificar los control s que permitirán mover el control correspondiente. 

Por ejemplo, para control Celda: 

1. Abra la ventana de descripción del control.

2. En la pestaña "UI", en la zona "Superponer y desplazar", seleccione "Superponible y desplazable con el ratón".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Champ_Deplacable_Page%20-%20HC%20N%B0002.gif)


3. Haga clic en "Parámetros" y seleccione los controles que permitirán mover la control Celda con el ratón: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Champ_Deplacable_Page%20-%20HC%20N%B0005.gif)





**Atención**: Los controles iFrame, WebCam, Flash o Java Applet no pueden utilizarse para mover los controles Looper, Supercontrol y Cell con el ratón.
<a name="NOTE3_3"></a>


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000083"></a>

Sólo se pueden mover directamente los siguientes controles:

- Control Campo de entrada, 

- Celda, 

- Looper, 

- Supercontrol, 

- Estático,

- Image.

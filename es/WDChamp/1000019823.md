


## Android: Condiciones de uso del control Mapa
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000141"></a>
**Hasta la versión 180046 de WINDEV Mobile**, se requería la biblioteca de Google Maps API v1 para utilizar un control Mapa en una aplicación Android. 
Esta biblioteca está preinstalada en la mayoría de los teléfonos y tabletas. Todo lo que tienes que hacer es obtener una clave generada a partir del Certificate usado para firmar la aplicación para poder usarla en la aplicación. Para obtener más información, consulte [Condiciones para utilizar un control Mapa antes de la versión 180056](../WDChamp/1000019820.md).

La versión 1 de esta librería fue amortizada por Google el 3 de diciembre de 2012.. A partir del 3 de marzo de 2013, ya no será posible generar nuevas claves para utilizar la biblioteca de la versión 1.. Sin embargo, las aplicaciones existentes seguirán funcionando; tendrá la posibilidad de reutilizar las claves previamente generadas en otras aplicaciones siempre que estén firmadas con el mismo Certificate que el utilizado para generar la clave.

**A partir de la versión 180056 de WINDEV Mobile**: A partir del 3 de marzo de 2013, la versión 2 de la librería deberá utilizarse para las aplicaciones que no incluyan una clave Google Maps API v1..
 
Los pasos para utilizar esta nueva biblioteca en una aplicación Android son diferentes y más complicados. Esta página de ayuda presenta los diferentes pasos.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Pasos necesarios
<a name="pasos_necesarios_ELTTEXTE000165"></a>
Para utilizar la biblioteca API v2 de Google Maps en una aplicación Android, es necesario seguir varios pasos. La mayoría de estos pasos deben realizarse una sola vez. 

1. [Instalación del kit de desarrollo de software de los servicios de Google Play](#NOTE3_1). 

2. [get una llave para usar la biblioteca](#NOTE4_1).

3. copy la clave en la asistente para generar la aplicación Android. 




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Etapa 1: Instalación del kit de desarrollo de software de los servicios de Google Play
<a name="etapa_1_instalacion_del_kit_desarrollo_software_los_servicios_google_play_ELTTEXTE000189"></a>
El SDK de Google Play Services se puede descargar e instalar desde el SDK Manager de Android: 

1. Ejecute el ejecutable SDK Manager.exe que se encuentra en el directorio instalación del SDK de Android (le aconsejamos que lo inicie como administrador).

2. En la lista de paquetes, seleccione "Extra/Google Play services".

3. Haga clic en el botón instalación.




<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Etapa 2: get una llave para usar la biblioteca
<a name="etapa_2_get_una_llave_para_usar_biblioteca_ELTTEXTE000213"></a>
Para acceder a los datos de Google Maps a través de la control Mapa, debes generar una clave que se incluirá en el manifiesto de la aplicación generada. Esta clave es gratuita, sin restricciones en cuanto al número de aplicaciones o al número de usuarios.

La librería Google Maps API v2 utiliza un sistema de gestión de la clave que difiere del utilizado en la versión 1 de la librería.. Las claves generadas para la versión anterior no son compatibles con este nuevo sistema.

La obtención de una clave se realiza desde la consola de gestión de las claves de la API de Google de la Certificate utilizada para firmar la aplicación y desde su nombre de paquete y se realiza en varios pasos.

**1. Recuperando la impresión SHA-1 de Certificate**

La generación asistente para las aplicaciones Android que contienen un control Mapa recupera automáticamente una impresión SHA-1 del Certificate. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_Carte1_Bis.gif&type=thumb)
**2. Creación de un "proyecto API"**

Realice las siguientes operaciones: 

1. Desde un navegador, abra la página de [Consola Google API](https://code.google.com/apis/console/). 

2. Autentificarse con una cuenta de Google.

3. Crear un nuevo proyecto del Proyecto API o utilizar un proyecto existente.

4. En la lista de API s associated con el proyecto, habilitar el servicio de Google Maps Android API v2 colocando el interruptor en ON. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_Google1.gif&type=thumb)


5. Aceptar las condiciones de uso del servicio. 




**3. Generación de la clave**

Realice las siguientes operaciones: 

1. En la consola de Google API, haga clic en "Credenciales".. 

2. Haga clic en "CREAR NUEVA LLAVE".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_Google2.gif&type=thumb)


3. En la ventana emergente que aparece, haga clic en la "tecla Android".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_Google3.gif)


4. Pegue la impresión SHA-1 de la Certificate (generada por WINDEV Mobile). <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_Google4.gif)


5. Recupere la clave generada en la sección "Key for Android apps (with certificates)".<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_Google5.gif&type=thumb)


6. La clave puede copiarse en el asistente para generar la aplicación Android. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Android_Carte1_Bis.gif&type=thumb)

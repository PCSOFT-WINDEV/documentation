
## Visualización de un reporte en formato HTML en el browser del usuario Web
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000178"></a>
Para visualizar un reporte en formato HTML en el browser del usuario Web, debe hacerlo:

- generar el archivo HTML en el servidor Web.

- visualizar el fichero generado en el navegador del usuario Web. La visualización del archivo puede ser:

	- inmediato.

	- solicitado por el usuario de la web (haga clic en un enlace, por ejemplo).




- eliminar el archivo HTML del servidor web.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000202"></a>


### Generación de un archivo HTML en el servidor Web
<a name="generacion_archivo_html_servidor_web_ELTPARAGRAPHE000025"></a>

Para generar un archivo HTML en el servidor Web:

1. Crear un nombre único para el archivo HTML a generar. El directorio de creación de este archivo debe ser accesible en lectura/escritura (el directorio de los archivos de datos o el directorio "&lt;MiProyecto&gt;_WEB" del sitio, por ejemplo).. 
	Para más detalles sobre cómo crear un nombre de archivo único, véase [Definición de un nombre único para el archivo generado](../WDChamp/1011009.md).

2. Si el reporte contiene imágenes, indique que estas imágenes deben generarse en el directorio "&lt;MiProyecto&gt;_WEB" del sitio ([iDirImageHTML](../WDLang5/3046078.md)).

3. Configure el destino de impresión con la función [iDestination](../WDLang5/3046074.md) y a la constante *iHTML* (imprimir en un archivo HTML).

4. Especifique el nombre de la reporte a imprimir en [iPrintReport](../WDLang5/3046032.md). 
	El archivo HTML se crea en el servidor Web.



<a name="NOTE2_2"></a>


### Visualización del fichero HTML
<a name="visualizacion_del_fichero_html_ELTPARAGRAPHE000053"></a>

La visualización del archivo puede ser:

- inmediato.

- solicitado por el usuario de la web (haga clic en un enlace, por ejemplo).




Para visualizar automáticamente el fichero generado en el browser del usuario Web, debe hacerlo:

1. Extrae el nombre y la extensión del archivo generado con [fExtractPath](../WDLang1/3036026.md).

2. Mostrar el archivo generado en el navegador del usuario de la web con [FileDisplay](../WDLang2/3012005.md) o [PageDisplay](../WDLang2/3058008.md).




Para visualizar el fichero generado a petición, debe hacerlo:

1. Extrae el nombre y la extensión del archivo generado con [fExtractPath](../WDLang1/3036026.md).

2. Visualizar el fichero generado en el navegador del usuario web. Puedes, por ejemplo:

	- utilice [FileDisplay](../WDLang2/3012005.md) o [PageDisplay](../WDLang2/3058008.md).

	- modificar la URL del control que muestra el archivo HTML ([URL](../Proprietes/2510132.md) Property de un botón o enlace por ejemplo).

	- ...






<a name="NOTE2_3"></a>


### Eliminación de un archivo HTML
<a name="eliminacion_archivo_html_ELTPARAGRAPHE000098"></a>

Para reducir el tamaño ocupado por los informes en formato HTML en el servidor Web, se recomienda borrar los informes inútiles de forma regular.. Puede crear un Procedure que se utiliza para borrar todos los informes creados durante el día, por ejemplo.

**Atención**: No elimine la reporte inmediatamente después de la [FileDisplay](../WDLang2/3012005.md) o la [PageDisplay](../WDLang2/3058008.md). El archivo HTML generado debe existir para que se pueda visualizar en el navegador del usuario Web.

<a name="NOTE3_1"></a>


## Ejemplo
<a name="ejemplo_ELTTEXTE000238"></a>
La "RPT_InvoiceHTML" reporte se muestra en formato HTML en la el equipo del usuario de la Web.

El botón "Mostrar factura" se utiliza para..:

- generar el archivo HTML en el directorio "&lt;MyProject&gt;_WEB" del sitio.

- ver la reporte en formato HTML.




En este ejemplo, el código de clic del servidor del botón "Mostrar factura" es el siguiente


```wl
// Generate a unique HTML file name
FilePath is string  
FilePath = fWebDir() + "\" + DateSys() + TimeSys() + ".htm"
// Configure the destination of the print 
iDestination(iHTML, FilePath) 
// Print the RPT_InvoiceHTML report 
iPrintReport(RPT_InvoiceHTML) 
// Extract the name and extension of the generated file
FileName is string
FileName = fExtractPath(FilePath, fFileName + fExtension)
// Send the file to the browser 
PageDisplay(FileName)
```


**Observación**: El archivo HTML se puede abrir directamente desde un botón o desde un enlace. Basta con cambiar la acción de control con la [URL](../Proprietes/2510132.md) Property:


```wl
// Generate a unique HTML file name
FilePath is string  
FilePath = fWebDir() + "\" + DateSys() + TimeSys() + ".htm"
// Configure the destination of the print
iDestination(iHTML, FilePath) 
// Print the RPT_InvoiceHTML report
iPrintReport(RPT_InvoiceHTML) 
// Extract the name and extension of the generated file
FileName is string  
FileName = fExtractPath(FilePath, fFileName + fExtension)
// Modify the action of the Link control that will open the HTML file
LINK_HTMLLink.URL = "/" + FolderWeb() + "/" + FileName
```

## Eventos gestionados por Default
<a name="eventos_gestionados_por_default_ELTTEXTE000212"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) WINDEV y WINDEV Mobile gestionan los siguientes eventos para las plantillas de control:

| Event | Condición de ejecución |
| --- | --- |
| Declaraciones globales | Declaración de variables globales al control modelo.<br>Este Event se ejecuta antes de las "Declaraciones globales" Event de la ventana del anfitrión (ventana en la que se integra el control modelo). |
| Inicialización | Corre cuando se abra la ventana.<br>Los eventos se ejecutan en el siguiente orden:<br><br>- Iniciando los controles de la ventana del host<br><br>- Inicialización de la ventana<br><br>- Obtención de foco<br><br><br> |
| partida | Corre cuando el cursor entre en una de las control de control modelo. |
| salida | Corre al cambiar de control (pasando de un control que se encuentra en el control modelo a otra ventana control). |
| Siempre que se modifique | Ejecutar cada vez que se modifique el contenido de control modelo. |
| Asignación de la propiedad Value | Se ejecuta cada vez que se especifica el valor ******' Property para el control modelo (directamente o con el valor Property). Este Event se utiliza para recuperar los parámetros pasados al control modelo, por ejemplo. Consulte [programando las plantillas de control](../WDChamp/9000098.md). |
| Obtención de la propiedad Value | Ejecutar cada vez que se recupere el***Value****** Property LE del control modelo. Este Event permite al control modelo devolver un valor a la ventana que alberga el control Modelo de controles. Consulte [programando las plantillas de control](../WDChamp/9000098.md). |


**Observación**: Las plantillas de control pueden tener sus propios procedimientos locales. Estos procedimientos locales pueden ser creados:

- desde el panel "Explorador de proyectos" (seleccione el control modelo y luego "Procedure s locales", "Nuevo Procedure local" en el menú contextual).

- desde el editor de código (vaya a la pestaña "Código", grupo "Procedimientos", expanda "Nuevo" y seleccione "Nuevo local Procedure").

## Eventos opcionales
<a name="eventos_opcionales_ELTTEXTE000242"></a>
Se pueden utilizar varios eventos opcionales.
Para añadir un evento opcional:

1. Seleccione la control deseada.

2. Mostrar la ventana del código de esta control (tecla F2).

3. Haga clic en el enlace "Añadir otras Event s a xxx" en la parte inferior del código de la ventana, después de la última Event.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Traitements_optionnels_WD_OK%20-%20HC%20N%B0001.gif)


4. Todos los eventos opcionales disponibles para el control se muestran. 

5. Marque la opción Event para agregar y validar. 
	Observación: Puede seleccionar varios eventos opcionales. 

6. El Event opcional seleccionado se agrega automáticamente a los Event s administrados por el control.




Para desactivar un Event opcional, simplemente realice las mismas operaciones para mostrar la lista de Event s opcionales. A continuación, simplemente desmarque los eventos opcionales para eliminar. 
Observación: Si el código desactivado contiene código WLanguage, este código se elimina automáticamente.Se puede gestionar, por ejemplo:

- el desplazamiento del cursor por encima del control,

- botón izquierdo del ratón abajo, arriba, con un doble clic,

- el botón derecho del ratón presionado, liberado o con doble clic 

- la rueda del ratón, etc.




Para más detalles, consulte la página web de [Eventos opcionales](../WDChamp/1014004.md).




## Visualización de una página en un sitio WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000120"></a>
La visualización de una nueva página en un sitio WEBDEV puede realizarse según varios métodos:

- de la description del control que debe abrir la página (botón, enlace, menú, Image pulsable, etc.).

- por programación en WLanguage.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Mostrando una página de la description de un control
<a name="mostrando_una_pagina_description_control_ELTTEXTE000144"></a>
Al describir un botón, un enlace, un Image pulsable, ... tiene la posibilidad de definir la acción realizada así como el destino de esta acción.

Para mostrar una página "xxx" de un botón, un enlace, un Image, ...:

1. Abra la descripción del elemento (opción "Descripción" del menú contextual del elemento).

2. Seleccione la acción a realizar: "Mostrar la página xxx".

3. Seleccione el destino de la acción: página actual, navegador actual, ... (frame específico en el caso de un frameset). La acción que se seleccionó anteriormente se realizará en este cuadro.

4. Valide.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Visualización de una página mediante programación (en WLanguage)
<a name="visualizacion_una_pagina_mediante_programacion_wlanguage_ELTTEXTE000168"></a>
Se pueden utilizar varias funciones WLanguage para abrir una página.


|   |   |
| --- | --- |
| [PageDisplay](../WDLang2/3058008.md) | Abre y muestra una nueva página en el navegador del usuario. |
| [PageRefresh](../WDLang2/3058011.md) | Actualiza la página que se muestra según su contexto. |
| [PageUse](../WDLang2/3058016.md) | Cierra todas las páginas actuales y su contexto, y abre una nueva página. |

Cuando se abre una página, se le pueden pasar parámetros. Para obtener más información, consulte [Página con parámetros](../WDChamp/1012020.md).

Observación: De forma predeterminada, cuando se abre una página mediante programación, el destino seleccionado es:

- el objetivo definido en la description del objeto que desencadena la apertura de la página (botón, enlace, etc.).

- el destino establecido para la página actual.




<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Caso de páginas dinámicas
<a name="caso_paginas_dinamicas_ELTTEXTE000192"></a>
Cuando se muestra una página dinámica, se realizan las siguientes acciones:

1. Se verifica la existencia del contexto de la página en el servidor.

2. El contexto se cierra si existe.

3. Abrir el contexto de la página. Todas las variables y controles relacionado a la página se reinician.

4. Visualización de la página solicitada en el navegador.

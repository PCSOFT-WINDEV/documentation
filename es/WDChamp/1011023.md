## Presentación
<a name="presentacion_ELTTEXTE000167"></a>
Un reporte proporciona una vista **custom de datos**: datos introducidos en la base de datos, datos encontrados en un archivo de texto, datos procedentes de un control Tabla, ....

Dependiendo del producto utilizado, los datos que se encuentran en la reporte se pueden mostrar en diferentes formatos:

- **En WINDEV**: 

	- En una aplicación Windows, se puede visualizar en la pantalla un reporte, guardado en un archivo HTML, impreso en una impresora, ....

	- En una aplicación Linux, una reporte se puede guardar en un archivo PDF. 




- **In WEBDEV**, un reporte puede ser guardado en un archivo (HTML o PDF), enviado por Email, mostrado en el navegador del usuario de la Web o impreso en una impresora conectada al servidor, ...

- **En WINDEV Mobile**: 

	- En una aplicación **Windows Mobile**, un reporte puede imprimirse directamente en una impresora (en formato PCL) o guardarse en un archivo PCL.

	- En una aplicación **iOS**, un reporte puede guardarse en un archivo PDF.. 

	- En una aplicación **in Universal Windows App mode**, se puede guardar un reporte en un archivo PDF.

	- En una aplicación **Android**, un reporte puede ser guardado en un archivo PDF.

## ¿Cuáles son los elementos de una reporte?
<a name="¿cuales_son_los_elementos_una_reporte_ELTTEXTE000191"></a>
Un reporte incluye:

- ***loques*** que definen las diferentes secciones del reporte (ver [Bloques Reporte](../WDChamp/1011040.md) para más detalles).

- ***controla*** en el que se muestran los datos de reporte (ver [Controles de un reporte](../WDChamp/1013211.md) para más detalles).




Se puede utilizar un reporte para integrar y sintetizar datos.. Es posible: 

- datos de grupo,

- clasificar los datos según cualquier criterio,

- realizar cálculos (medios, estadísticas) o crear gráficos,

- visualizar los datos en varias columnas.




Piense en un reporte y el editor de reportes lo creará.

## Simplificado Definition de un reporte
<a name="simplificado_definition_reporte_ELTTEXTE000215"></a>
Los siguientes diagramas presentan un simple Definition de un reporte:

- los datos a imprimir provienen de una fuente de datos (archivo de datos, vista HFSQL, consulta, zona de memoria, WLanguage Variable o archivo de texto, etc.).

- reporte agrupa, clasifica y formatea los datos.

- la ejecución de reporte puede ser dirigida a: 

	- en WINDEV: la pantalla, un archivo (PDF, HTML, RTF), una impresora o un fax, ....

	- en WEBDEV: un archivo (HTML, RTF, PDF, etc.), una impresora o un fax, ... Si el destino reporte es un archivo, este archivo se puede enviar al navegador. 

	- en WINDEV Móvil: un archivo PCL o una impresora PCL (en Windows Mobile) o un archivo PDF (para las aplicaciones iOS o para las aplicaciones en modo Universal Windows App).

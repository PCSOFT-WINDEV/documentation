<a name="NOTE1"></a>

## Presentación
<a name="presentacion_ELTTEXTE000677"></a>
Esta página explica cómo programar controles Tabla TreeView. Existen varias funciones WLanguage para manipular controles Tabla TreeView mediante programación.

Para saber más sobre estas funciones, consulte la documentación. Para obtener más información, consulte [lista de funciones específicas de los controles Tabla TreeView](../WDLang1/1000010002.md).

## Inicializar un control Tabla TreeView
<a name="inicializar_control_tabla_treeview_ELTTEXTE000701"></a>
Para rellenar un control Tabla TreeView mediante programación, utilice las siguientes funciones: 

- [TableAddLine](../WDLang1/3074006.md), [TableAddChild](../WDLang1/1000010004.md), [TableInsertLine](../WDLang1/3074026.md) y [TableInsertChild](../WDLang1/1000010005.md): en cada fila, el valor de cada columna está separado por comas. 

- [TableAdd](../WDLang1/3074017.md) y [TableInsert](../WDLang1/3074025.md): en cada fila, el valor de cada columna está separado por TAB.




Si se agregó una nueva fila en un control Tabla TreeView con la función [TableAdd](../WDLang1/3074017.md), [TableInsert](../WDLang1/3074025.md), [TableAddLine](../WDLang1/3074006.md), [TableAddChild](../WDLang1/1000010004.md), [TableInsertLine](../WDLang1/3074026.md) o [TableInsertChild](../WDLang1/1000010005.md), la propiedad [New](../Proprietes/2510073.md) se establece en <u><u><u><u>True</u></u></u></u> (de lo contrario se establece en <u><u><u><u>False</u></u></u></u>).

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Observación**: Para rellenar un control Tabla TreeView de forma dinámica, puede utilizar la propiedad [AddChildDelayed](../Proprietes/1000020927.md) en las filas del control. 
En este caso, el principio para rellenar el control se simplifica: 

- Al inicializar el control, solo se cargan los elementos del primer nivel. Cada elemento está asociado a la propiedad [AddChildDelayed](../Proprietes/1000020927.md).  

- Cuando el usuario hace clic en el elemento del primer nivel para expandirlo, se ejecuta el procedimiento definido con la propiedad [AddChildDelayed](../Proprietes/1000020927.md). Este procedimiento rellena el nivel a expandir. Los nodos secundarios solo se recuperan a petición del usuario.

## Agregar o modificar filas en un control Tabla TreeView
<a name="agregar_modificar_filas_control_tabla_treeview_ELTTEXTE000725"></a>


### Agregar filas a un control Tabla TreeView
<a name="agregar_filas_control_tabla_treeview_ELTPARAGRAPHE000117"></a>

Las filas no se crean automáticamente en los controles Tabla TreeView. Se debe especificar llamando a las siguientes funciones:

- [TableAdd](../WDLang1/3074017.md), [TableAddLine](../WDLang1/3074006.md) o [TableAddChild](../WDLang1/1000010004.md): agrega una fila al final del control,

- [TableInsert](../WDLang1/3074025.md), [TableInsertLine](../WDLang1/3074026.md) o [TableInsertChild](../WDLang1/1000010005.md): agrega una fila en el índice especificado o en el índice actual. 


![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) **Observación**: Si la opción "Entrada en cascada" no está seleccionada:

- El control Tabla TreeView no contiene ninguna fila cuando se crea. En este caso, no se puede realizar ninguna entrada.. La propiedad [Empty](../Proprietes/2510126.md) se establece en <u><u><u><u>True</u></u></u></u>.

- Para insertar una fila automáticamente en un control Tabla TreeView vacío, utilice la función [TableAdd](../WDLang1/3074017.md) o [TableAddLine](../WDLang1/3074006.md).



<a name="NOTE3_2"></a>


### Modificar una fila o columna
<a name="modificar_una_fila_columna_ELTPARAGRAPHE000165"></a>

El contenido de las filas y columnas de un control Tabla TreeView puede ser modificado:

- por el usuario, introduciendo los datos directamente en las columnas. Los cambios se guardan automáticamente en el control Tabla TreeView (no se requiere ningún código adicional). La propiedad [Modified](../Proprietes/2510080.md) se establece en <u><u><u><u>True</u></u></u></u>.

- mediante programación:

	- con la función [TableModify](../WDLang1/3074019.md) o [TableModifyLine](../WDLang1/3074028.md) para modificar el contenido de la fila actual o de una fila determinada.
			Por ejemplo: 
			
		```wl
		TableModify(TVT_CUSTOMER, "MOORE" + TAB + "Vince" + TAB + "Miami")
		```

			**Observación**: También puede utilizar directamente el nombre del control Tabla TreeView: 
			
		```wl
		// Modify the current row
		TVT_CUSTOMER = "MOORE" + TAB + "Vince" + TAB + "Miami"
		// Modify the COL_DAY coloumn of row 3
		TableModify(COL_DAY, "Wednesday" + TAB + "Off", 3)
		```


	- especificando directamente el nombre de la columna para cambiar el contenido (como para un control Campo de entrada). Para modificar la columna de una fila determinada, se debe especificar el número de fila (índice).
			Por ejemplo:
			
		```wl
		COL_NAME[Index] = CustomerName
		```

			Para modificar la columna de la fila actual, no es necesario especificar el índice. 
			Por ejemplo: 
			
		```wl
		COL_NAME = CustomerName
		```

			La propiedad [Modified](../Proprietes/2510080.md) se establece en <u><u><u><u>False</u></u></u></u> (se establece en <u><u><u><u>True</u></u></u></u> solo cuando se introducen datos en el control Tabla TreeView).

### Reanudar la entrada en una columna
<a name="reanudar_entrada_una_columna_ELTPARAGRAPHE000236"></a>

La función [SetFocusAndReturnToUserInput](../WDLang1/1410088107.md) reanuda la entrada en la columna de la fila actual. Por ejemplo:

```wl
// Event "Entry in" COL_QTY
// COL_QTY cannot be entered if COL_PRODUCT is not entered
IF NoSpace(COL_PRODUCT) = "" THEN
	Error("The Product column must be entered first")
	SetFocusAndReturnToUserInput(COL_PRODUCT)
END
```

### Agregar o eliminar una columna
<a name="agregar_eliminar_una_columna_ELTPARAGRAPHE000252"></a>

Es posible:

- agregar una columna a un control Tabla TreeView con la función [ControlClone](../WDLang1/3025017.md).

- eliminar una columna de un control Tabla TreeView con la función [ControlDelete](../WDLang1/3025037.md).



<a name="NOTE3_5"></a>


### Gestión de columnas calculadas
<a name="gestion_columnas_calculadas_ELTPARAGRAPHE000270"></a>La fórmula de cálculo de una columna calculada debe definirse en el evento "Visualización de una fila" de un control Tabla. Por ejemplo:


```wl
// COL_PBT: calculated column
// COL_UPBT and COL_QTY: columns linked to memory areas
COL_PBT = COL_UPBT * COL_QTY
```

## Operaciones en filas y celdas
<a name="operaciones_filas_celdas_ELTTEXTE000773"></a>


### Eliminar una fila
<a name="eliminar_una_fila_ELTPARAGRAPHE000283"></a>

Para eliminar una fila, debe llamar a la función [TableDelete](../WDLang1/3074032.md) o [TableDeleteChild](../WDLang1/1000010006.md).

- La sintaxis para la función [TableDelete](../WDLang1/3074032.md) es la siguiente: 
	
	```txt
	TableDelete(<TreeView Table control> [, <Index>])
	```

	Si se especifica el índice, la función [TableDelete](../WDLang1/3074032.md) elimina la fila que corresponde al índice. De lo contrario, se elimina la fila actual.
	Al eliminar una fila del control Tabla TreeView también se eliminan todos los valores de las columnas de esa fila.

- La sintaxis para la función [TableDeleteChild](../WDLang1/1000010006.md) es la siguiente: 
	
	```txt
	TableDeleteChild(<TreeView Table control>, <Parent element index>)
	```

	Se eliminarán todos los elementos hijos de la fila que corresponde al índice especificado.



<a name="NOTE4_2"></a>


### Seleccionar una fila
<a name="seleccionar_una_fila_ELTPARAGRAPHE000330"></a>

Una fila se puede seleccionar con la función [TableSelectPlus](../WDLang1/3074031.md).

La sintaxis utilizada es la siguiente:

```txt
TableSelectPlus(<TreeView Table control>[, <Index>])
```

<a name="NOTE4_3"></a>


### Obtener el contenido de una fila o celda
<a name="obtener_contenido_una_fila_celda_ELTPARAGRAPHE000350"></a>

El contenido de un control Tabla TreeView puede obtenerse:

- para toda la fila.

- celda por celda.



<a name="NOTE4_4"></a>


### Valor de la celda en la fila actual
<a name="valor_celda_fila_actual_ELTPARAGRAPHE000362"></a>

Para obtener el valor de una columna (o celda) de la fila actual, la sintaxis es la misma que la de un control Campo de entrada simple: 

```txt
<Value> = <Column_Name>
```
Por ejemplo: 

```wl
// COL_QTY is a column of a TreeView Table control
IF COL_QTY < 10 THEN
	Info("Insufficient quantity")
END
```

<a name="NOTE4_5"></a>


### Valor de una celda en una fila específica
<a name="valor_una_celda_una_fila_especifica_ELTPARAGRAPHE000376"></a>

Para obtener el valor de una columna que no se encuentra en la fila actual, se debe especificar el índice de la fila: 

```txt
<Value> = <Column name>[<Index>]
```
Por ejemplo: 

```wl
// Add the total price before tax (PBT) for all the order lines
TotalPrice = 0
FOR Index = 1 TO TVT_ORDER.Count
	TotalPrice = TotalPrice + COL_PBT[Index]
END
```

<a name="NOTE4_6"></a>


### Contenido de la fila actual
<a name="contenido_fila_actual_ELTPARAGRAPHE000390"></a>

Para obtener el contenido de la fila actual: 

```txt
<Value> = <TreeView Table control>
```

<a name="NOTE4_7"></a>


### Contenido de una fila específica
<a name="contenido_una_fila_especifica_ELTPARAGRAPHE000401"></a>

Para obtener el contenido de la fila cuyo índice es &lt;Index&gt;, utilice la siguiente sintaxis:

```txt
<Value> = <TreeView Table control>[<Index>]
```

**Observación**: [TableSelect](../WDLang1/3074020.md) devuelve el índice de la fila actual. Por ejemplo: 

```wl
// Retrieve the contents of row 10 in the TVT_CUSTOMER TreeView Table control
CurrentRow = TVT_CUSTOMER[10] 
// Name of the selected customer
CustomerName = COL_NAME[TableSelect(TVT_CUSTOMER)]
// Retrieve the contents of the current row in the TVT_CUSTOMER TreeView Table control
CurrentRow = TVT_CUSTOMER
```

## Operaciones en la estructura jerárquica
<a name="operaciones_estructura_jerarquica_ELTTEXTE000833"></a>


### Contraer todos los nodos de un control Tabla TreeView
<a name="contraer_todos_los_nodos_control_tabla_treeview_ELTPARAGRAPHE000426"></a>

Para contraer todos los nodos de un control Tabla TreeView, utilice la función [TableCollapseAll](../WDLang1/1000010003.md). La sintaxis es la siguiente: 

```txt
TableCollapseAll(<TreeView Table control>)
```

<a name="NOTE5_2"></a>


### Expandir todos los nodos de un control Tabla TreeView
<a name="expandir_todos_los_nodos_control_tabla_treeview_ELTPARAGRAPHE000445"></a>

Para expandir todos los nodos de un control Tabla TreeView, utilice la función [TableExpandAll](../WDLang1/1000010001.md). La sintaxis utilizada es la siguiente:

```txt
TableExpandAll(<TreeView Table control>)
```


![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Observación: Si se utiliza la propiedad [AddChildDelayed](../Proprietes/1000020927.md) para rellenar dinámicamente los subniveles en un control Tabla TreeView, el procedimiento de relleno de subniveles se ejecutará por cada nuevo elemento expandido en el lado cliente. 
<a name="NOTE5_3"></a>


### Obtener el número de la fila contraída/expandida
<a name="obtener_numero_fila_contraidaexpandida_ELTPARAGRAPHE000473"></a>

Para obtener el número de la fila expandida o contraída, simplemente utilice el evento "Contracción/Expansión de un nodo" en el control Tabla TreeView: 

```wl
// Which row is used? 
Info(TVT_TreeViewTable[TVT_TreeViewTable])
```

<a name="NOTE5_4"></a>


### Definir el estado de las nuevas filas
<a name="definir_estado_las_nuevas_filas_ELTPARAGRAPHE000484"></a>

Puede definir el estado de la siguiente fila que se agregará a un control Tabla TreeView (contraída o expandida). Para ello, utilice la propiedad [Collapsed](../Proprietes/2513014.md).


```wl
// The next added rows will be automatically collapsed
TVT_TreeviewTable.Collapsed = True
```

<a name="NOTE5_5"></a>


### Obtener y establecer las imágenes de las filas
<a name="obtener_establecer_las_imagenes_las_filas_ELTPARAGRAPHE000498"></a>

Para obtener y establecer las imágenes de las filas de un control Tabla TreeView, utilice las siguientes propiedades:

- [ExpandedImage](../Proprietes/2513015.md): imagen de la fila expandida,

- [CollapsedImage](../Proprietes/2513016.md): imagen de la fila contraída. 





```wl
// Modify the images of rows in a TreeView Table control
TVT_TreeViewTable.ExpandedImage = "OpenFolder.gif"
TVT_TreeViewTable.CollapsedImage = "ClosedFolder.gif"
```

### Columna TreeView con casilla de verificación
<a name="columna_treeview_con_casilla_verificacion_ELTPARAGRAPHE000518"></a>

Una columna de tipo Casilla de verificación puede definirse como "Columna TreeView" en un control Tabla TreeView. En este caso, la columna de tipo Casilla de verificación permite gestionar la estructura jerárquica.

La propiedad [Caption](../Proprietes/2510053.md) de la fila obtiene y establece el texto que aparece junto a la casilla de verificación.
La propiedad [Value](../Proprietes/2510130.md) de la fila obtiene y establece el valor de la casilla de verificación. 
<a name="NOTE5_7"></a>


### Obtener el padre o los hijos de un elemento
<a name="obtener_padre_los_hijos_elemento_ELTPARAGRAPHE000536"></a>

Las siguientes funciones pueden utilizarse para obtener el padre o los hijos de un elemento: 

- [TableGiveParent](../WDLang1/1000010011.md): Devuelve el "padre" de un elemento, es decir, el elemento del nivel superior. 

- [TableGiveChild](../WDLang1/1000010009.md): Devuelve los "hijos" de un elemento, es decir, los elementos del nivel inferior.

## Propiedades específicas de los controles Tabla TreeView
<a name="propiedades_especificas_los_controles_tabla_treeview_ELTTEXTE000893"></a>
Las siguientes propiedades permiten definir las características de los controles Tabla TreeView.


|   |   |
| --- | --- |
| [AddChildDelayed](../Proprietes/1000020927.md) | Define el procedimiento al que se llamará para rellenar una rama dinámicamente. |
| [Collapsed](../Proprietes/2513014.md) | Obtiene y establece el estado (contraído o expandido) de las nuevas filas agregadas a un control Tabla TreeView. |
| [Merge](../Proprietes/1000017095.md) | Determina si las celdas (o títulos de las columnas) de un control Tabla TreeView están combinados y los combina si es necesario. |
| [ExpandedImage](../Proprietes/2513015.md) | Obtiene y establece la imagen predeterminada de una fila expandida en un control Tabla TreeView. |
| [CollapsedImage](../Proprietes/2513016.md) | Obtiene y establece la imagen predeterminada de una fila contraída en un control Tabla TreeView. |
| [Multiselection](../Proprietes/1000017054.md) | Obtiene y establece el modo de selección en un control Tabla TreeView. |
| [TotalNbChildren](../Proprietes/1000021046.md) | Devuelve el número total de hijos de una rama en un control Tabla TreeView. |
| [SortOption](../Proprietes/1000017044.md) | Obtiene y establece las opciones para ordenar una columna en un control Tabla TreeView. |
| [LeftIndent](../Proprietes/2513018.md) | Obtiene y establece el espacio a la izquierda del texto en las columnas de un control Tabla TreeView. |
| [BrowsedItem](../Proprietes/2510111.md) | Obtiene y establece el campo utilizado para recorrer los controles Tabla TreeView automáticamente. |
| [Size](../Proprietes/2510125.md) | Obtiene y establece el número de columnas de un control Tabla TreeView. |
| [TotalsEnabled](../Proprietes/2510122.md) | Determina cómo se realizan los cálculos automáticos en un control Tabla TreeView y desactiva o fuerza los cálculos automáticos en el control. |


Para ver la lista completa de las propiedades WLanguage que se pueden utilizar: 

- con un control Tabla TreeView, consulte [Propiedades asociadas a un control Tabla TreeView](../WDChamp/1015003.md).

- con las filas de un control Tabla TreeView, consulte [Propiedades asociadas a una fila en un control Tabla TreeView](../WDChamp/1015007.md)

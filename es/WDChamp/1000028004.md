## Red social en Facebook
<a name="red_social_facebook_ELTTEXTE000552"></a>
Hay varios tipos disponibles en un Facebook control Red social. Cada tipo puede ser associated con un objetivo específico. 

La tabla que figura a continuación presenta la información utilizada para cada tipo y objetivo. 


| escribir a máquina | Objetivo | Efecto |
| --- | --- | --- |
| **I como o recomendar** | El dominio | Crea en el muro de Facebook del usuario Web conectado un enlace al dominio del sitio actual.<br><br>Atención: Debe existir un índice Default Page en el servidor de alojamiento. |
|   | El sitio | Crea en el muro de Facebook del usuario Web conectado un enlace al sitio actual.<br><br>Atención: Debe existir un índice Default Page en el servidor de alojamiento. |
|   | La Page | Crea en el muro de Facebook del usuario web conectado un enlace a la Page del sitio actual. |
|   | Destino personalizado | <br><br>El URL debe ser escrito en el "Target URL" control.<br><br><br><br>**Caso 1: URL de un Facebook Page**: <br><br>- Incrementa el contador de Page de Facebook que ha gustado.<br><br>- Crea un favorito en la pared de Facebook de un usuario Web conectado.<br><br><br>**Caso 2: URL de cualquier Page o sitio**: <br>Crea en el muro de Facebook del usuario de la Web conectado un enlace a la Page o sitio especificado. |
| **Send** | El dominio | Se utiliza para enviar un enlace en el dominio a otro usuario de Facebook. El mensaje enviado es un mensaje privado. |
|   | El sitio | Se utiliza para enviar un enlace en el sitio a otro usuario de Facebook. El mensaje enviado es un mensaje privado. |
|   | La Page | Sirve para enviar un enlace en la Page a otro usuario de Facebook. El mensaje enviado es un mensaje privado. |
|   | Destino personalizado | Se utiliza para enviar un enlace a otro usuario de Facebook. Este enlace corresponde a un enlace a cualquier Page o sitio. El mensaje enviado es un mensaje privado.<br><br><br><br><br><br>La URL del enlace a enviar debe introducirse en la "URL de destino" control. |
| **Subscribe** | Destino personalizado | <br><br>En "Cuenta de destino", puede introducir:<br><br>- la URL de Facebook Page a la que el internauta desea suscribirse. Por ejemplo: "www.facebook.com/01net" o "http://www.facebook.com/pages/PC-SOFT/247782873866"<br><br>- el identificador de la cuenta de Facebook a la que el usuario web desea suscribirse. Por ejemplo: "01net" o "247782873866".<br><br><br> |
| **SVer perfil de** | Destino personalizado | En "Cuenta de destino", puedes introducir el identificador de la cuenta de Facebook deseada. Por ejemplo: "01net" o "247782873866". |





Observación: Para permitir la publicación en el muro de usuario de la Web, la Page que contiene la control Red social debe cumplir las siguientes condiciones: 

- la Page debe incluir un título. 

- el Page debe incluir al menos un control Imagen. 




Atención: Durante la prueba de sitio (GO en modo localhost), sólo los tooltips son correctos. No se puede publicar en un muro.

Hay varios tipos disponibles en un Twitter control Red social. Cada tipo puede ser associated con un objetivo específico. 

El tabla a continuación resume la información utilizada para cada objetivo. 


| escribir a máquina | Objetivo | Efecto |
| --- | --- | --- |
| **Tweeter** | El dominio | Crea en el muro de Twitter del usuario Web conectado un mensaje que contiene un enlace al dominio del sitio actual. |
|   | El sitio | Crea en el muro de Twitter del usuario Web conectado un mensaje que contiene un enlace al sitio actual. |
|   | La Page | Crea en el muro de Twitter del usuario Web conectado un mensaje que contiene un enlace a la Page del sitio actual. |
|   | Destino personalizado | <br><br>El URL debe ser escrito en el "Target URL" control.<br><br><br><br>Crea en el muro de Twitter del usuario Web conectado un mensaje que contiene un enlace a la URL especificada. |
| **Follow** | Destino personalizado | <br><br>En el "Target URL" control, tienes la capacidad de escribir:<br><br><br><br>- la URL de Twitter Page a la que desea suscribirse el internauta. Por ejemplo: "https://twitter.com/PCSOFT_FR"<br><br>- el identificador de la cuenta de Twitter a la que el usuario de la Web desea suscribirse. Por ejemplo: "PCSOFT_FR".<br><br><br> |
| **SVer perfil de** | Destino personalizado | En "Cuenta de destino", puede introducir el identificador de la cuenta de Twitter deseada. Por ejemplo: "PCSOFT_FR". |


Observaciones: 

- Los mensajes de Twitter están limitados a 140 caracteres. 

- Los controles de la Red Social Twitter funcionan en modo de prueba (pero la URL no se rellena en los mensajes).

## LinkedIn red social
<a name="linkedin_red_social_ELTTEXTE000600"></a>
Puedes compartir el contenido actual en un LinkedIn control Red social. 

La tabla que figura a continuación presenta la información utilizada para cada tipo y objetivo. 


| escribir a máquina | Objetivo | Efecto |
| --- | --- | --- |
| **Compartir** | El dominio | Crea un Post en las noticias del perfil de LinkedIn conectado con un enlace en el dominio.<br><br>Atención: Debe existir un índice Default Page en el servidor de alojamiento. |
|   | El sitio | Crea un Post en las noticias del perfil de LinkedIn conectado con un enlace en el sitio.<br><br>Atención: Debe existir un índice Default Page en el servidor de alojamiento. |
|   | La Page | Crea un post sobre las noticias del perfil LinkedIn conectado con un enlace en el Page. |
|   | Destino personalizado | Crea un Post en las noticias del perfil de LinkedIn conectado con un enlace personalizado.<br><br>La URL del enlace a enviar debe introducirse en la "URL de destino" control. |
| **SVer perfil de** | Destino personalizado | Muestra la Page correspondiente al perfil LinkedIn. <br><br>En "Cuenta de destino", puedes introducir el identificador de la cuenta de LinkedIn deseada. |


**Atención**: Durante la prueba de sitio (GO en modo localhost), no se puede hacer ninguna prueba para este tipo de control. 

Observación: Para especificar la miniatura que se mostrará en las noticias, proporcione la imagen a través de un OpenGraph description. Debe incluirse una etiqueta específica en la cabecera Page. Esta etiqueta debe contener la Image para mostrarla.

Por ejemplo: 


```txt
<meta property="og:image" content="http://mydomain.com/myimage.jpg"/>
```

## Presentación

### Control de página dinámica o estática
<a name="control_pagina_dinamica_estatica_ELTPARAGRAPHE000011"></a>

El control Upload se utiliza para subir archivos al servidor web. 

WEBDEV propone un control Upload avanzado que puede ser usado inmediatamente. Incluye varios controles (que pueden ser borrados si es necesario): 

- Looper utilizado para gestionar los archivos a cargar. Este looper se utiliza para especificar las características del archivo, para mostrar una barra de progreso durante la carga y para proponer un botón para eliminar el archivo de la lista si es necesario.. 

- Leyenda que indica dónde se dejarán caer los archivos que se van a cargar. 

- Botón"Agregar" usado para agregar los archivos. 

- Botón"Enviar" para iniciar la carga del archivo. 


Este control se puede utilizar en el modo de un solo archivo o en el modo multiarchivo.. 

La control Upload gestiona directamente el Drag and Drop de los archivos desde el explorador (si el navegador soporta el estándar HTML 5). Para obtener más información, consulte [Manejo de una control Upload por programación](../WDChamp/1016333.md).

**Observaciones**: 

- En las versiones anteriores (antes de la versión 22), WEBDEV proponía dos controles de carga preestablecidos (control usado para cargar un solo archivo y control usado para cargar varios archivos). 

	- Estos controles siguen funcionando en la versión 22 y posteriores. La única diferencia con el control propuesto en la versión 22 es la composición del control (un looper en la versión 22 y posteriores) y su interfaz.. 

	- Estos controles proponen la generación en modo HTML 5 en la versión 22 (nueva característica de la versión 22, ver [Características del control)](#NOTE1_3). 




- El Upload control Campo de entrada se mantiene para la compatibilidad retroactiva. Para obtener más información, consulte [Subir archivos (Upload)](../WDLang2/3012024.md). 

- El control de preajuste propuesto para el upload es un control asíncrono: Todavía puede seguir utilizando el sitio actual sin estar bloqueado por el upload.



<a name="NOTE1_2"></a>


### Creando un control Upload
<a name="creando_control_upload_ELTPARAGRAPHE000051"></a>

Para crear un control Upload: 

1. En el editor de páginas, en la pestaña "Creación", en el grupo "Controles frecuentes", despliegue "Botón".

2. En la lista que se abre, seleccione el tipo de control "Carga de archivos".

3. Haga clic en la ubicación en la que desea crear el control. El control aparece en la página.




**Para visualizar las características de control**, seleccione "Descripción" en el menú contextual del control Upload ("Enviar" o "Añadir").

**El código WLanguage de gestión del control Upload** se encuentra en los procesos asociados a este control. Este código puede ser modificado de acuerdo a sus requerimientos. 

**Importante**: El proceso de "Recepción de los archivos cargados" debe ser modificado antes de desplegar el sitio.. Un ejemplo de código es suministrado en comentario. Este código puede, por ejemplo: 

- copiar los archivos cargados en un directorio específico del sitio,

- insertar los archivos cargados en una nota binaria, ....


Para obtener más información, consulte [Eventos asociados a un control Upload](../WDChamp/1016331.md).

### Características del control
<a name="caracteristicas_del_control_ELTPARAGRAPHE000088"></a>

En la pestaña "Detalles" de la descripción de control Upload (accesible a través de la descripción del botón "Agregar") hay varias opciones específicas: 

- ****: El código generado para gestionar la carga utiliza HTML5. Este modo de control de generación permite utilizar el control Upload en sitios disponibles en todas las plataformas (incluyendo las plataformas móviles que no soportan el flash). 

- ****: El código generado para gestionar la carga utiliza Flash. En este caso, la carga no estará disponible en plataformas que no soporten el flash.

- **Selección de múltiples archivos**: permite que el selección múltiple en el control. Si no se selecciona esta opción, sólo se podrá cargar un único archivo.. 

- **Incrustar el título en la imagen**: se utiliza para insertar el título del botón en la imagen asociada al botón. 

- **Gestión del Arrastrar/Soltar**: Se usa para permitir "Arrastrar y soltar" desde el explorador de Windows. Para obtener más información, consulte [Gestión de arrastrar y soltar archivos a un control Upload (HTML5)](../WDChamp/1016333.md). 




Notas sobre el tipo de generación del upload: 

- Si el control Upload se crea en la versión 22, la opción "Usar la carga nativa HTML5" se seleccionará automáticamente en una página HTML5. 

- Si el control Upload fue creado con una versión anterior (control Upload preestablecida de un solo archivo o de varios archivos), también tiene la posibilidad de utilizar la generación en formato HTML 5.

### Multi-archivo control Upload en AWP
<a name="multiarchivo_control_upload_awp_ELTPARAGRAPHE000132"></a>

Un control Upload multi-archivo en AWP crea implícitamente un contexto AWP en el disco. Este tipo de control no es compatible con la gestión de contextos a través de cookies.. Para obtener más información, consulte [Gestión de contextos de PTA](../WDChamp/3539060.md). 
<a name="NOTE2_2"></a>


### Notas adicionales
<a name="notas_adicionales_ELTPARAGRAPHE000142"></a>

- Si el control Upload está basado en Flash (opción "" seleccionada en la pestaña "Detalles"), este último debe estar instalado en el equipo del usuario (en la versión 9 o posterior).

- El control Upload no puede ser usado en un looper de Ajax.

### Tamaño máximo de los archivos a cargar
<a name="tamano_maximo_los_archivos_cargar_ELTPARAGRAPHE000154"></a>

**Por defecto**el tamaño de los archivos a cargar no está limitado en el motor de WEBDEV. Este tamaño puede ser limitado: 

- por el servidor web utilizado. Consulte la documentación sobre el servidor web utilizado para obtener más información.. 

- por WEBDEV servidor de aplicaciones. 




El archivo no se cargará si su tamaño excede el tamaño límite.

**To modificar el tamaño máximo de los archivos a subir en el WEBDEV servidor de aplicaciones**, el registro debe ser modificado en el servidor. Esta modificación se puede realizar para todas las aplicaciones WEBDEV.  

**Para modificar el tamaño de los archivos cargados para todas las aplicaciones WEBDEV**: 

1. Seleccione la siguiente clave de registro: 

	- En un equipo de 64 bits: HKEY_MÁQUINA_LOCAL\\SOFTWARE\\WOW6432Node\\PC SOFT\\WEBDEV\\28.0

	- En una computadora de 32 bits: HKEY_MÁQUINA_LOCAL\\SOFTWARE\\PC SOFT\\WEBDEV\\28.0




2. Añadir la clave MAX_UPLOAD. El valor de esta clave corresponde al tamaño máximo de los archivos a cargar (en KB).




Observación: También puede existir un límite relacionado al servidor Web (IIS, Apache, ...). Por ejemplo, para IIS, tiene la posibilidad de editar el archivo XML "%windir%\\system32\\inetsrv\\config\\applicationhost.config" y de crear o modificar el atributo "maxAllowedContentLength" de la etiqueta "requestLimits" especificando el tamaño en bytes. 
Ejemplo para establecer el límite a 50 MB en IIS:


```txt
<system.webServer>
          	<security>
		.....
		<requestFiltering>
		.....
			<requestLimits maxAllowedContentLength="52428800"/>
		</requestFiltering>
....
```

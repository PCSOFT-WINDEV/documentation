
## Máscaras de entrada y máscaras de visualización
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000146"></a>
WINDEV, WEBDEV y WINDEV Mobile proponen dos tipos de máscaras:

- los ***máscaras de entrada*** que definen los caracteres que se pueden escribir y/o mostrar en los controles que se encuentran en una ventana o en una página. A veces, estas máscaras también se pueden usar para modificar el caso (caracteres en mayúscula / minúscula) de los valores asignados a los controles de ventana o a los controles de página mediante la programación.

- ***máscaras de visualización*** que definen los caracteres que pueden imprimirse en los controles de un reporte.

## Las máscaras
<a name="las_mascaras_ELTTEXTE000170"></a>


### Los diferentes tipos de máscaras
<a name="los_diferentes_tipos_mascaras_ELTPARAGRAPHE000021"></a>

Independientemente del tipo de máscara, una máscara puede ser:

- ***preestablecido***. El editor propone diferentes tipos de máscaras según el contenido de la control (texto, numérico, ...).

- ***a medida***. Todo lo que tienes que hacer es introducir la máscara para aplicar. Por ejemplo, "CCC9 LL 9A" para un texto control o "DDD DD MMMM AAAA" para una fecha control. También se puede definir una expresión regular con la función [InputMask](../Proprietes/2510033.md) Property. 
	

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) ***advanced***. Basta con definir la máscara de entrada mediante una [InputMask](../WDLang1/1000024886.md) Variable. Este tipo de variable permite un control personalizado en cada Event relacionado para introducir máscaras en un control Campo de entrada.

- ***NotFound***. Ninguna máscara es associated con el control. En este caso no se realiza ninguna verificación.




Para los controles de tipo Numérico, Monetario, Fecha, Hora y Duración, la máscara también puede ser "***Definida por el proyecto***". En este caso, las características lingüísticas definidas en la pestaña "Language" del proyecto description se tienen en cuenta automáticamente según el idioma de ejecución de la aplicación.

Las opciones lingüísticas seleccionadas por Default son las de Windows.

Para cada tipo de datos numéricos, tiene la posibilidad de definir los parámetros que se aplicarán (separador decimal, separador de mil, símbolo de moneda, formato de hora, símbolo AM o PM, ....).

**Observación**: Esta opción le permite utilizar automáticamente la misma configuración de la función multilingüe en todos los controles numéricos de la aplicación.. La gestión de los formatos multilingües está centralizada en el proyecto description.

La máscara de entrada/visualización de un control puede establecerse en la ventana de su description (pestaña "General" o "Detalles"), o de forma programada (utilizando el [InputMask](../Proprietes/2510033.md) Property).


<a name="NOTE2_2"></a>


### Multilingual masks
<a name="multilingual_masks_ELTPARAGRAPHE000067"></a>

Para un proyecto multilingüe, se puede seleccionar una máscara diferente para cada idioma soportado por la ventana (página o reporte).. Se utilizará la máscara correspondiente según el lenguaje de ejecución del proyecto.

Para obtener más información, consulte [Función multilingüe en los controles de una ventana, página y reporte](../WDChamp/1013031.md).

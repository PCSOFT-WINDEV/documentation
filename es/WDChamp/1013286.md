### Eventos soportados por WINDEV y WINDEV Mobile
<a name="eventos_soportados_por_windev_windev_mobile_ELTPARAGRAPHE000013"></a>

En una ventana, WINDEV y WINDEV Mobile manejan los siguientes eventos por Default (en orden de aparición en el editor de código):


| Evento | Condición de ejecución |
| --- | --- |
| Inicialización | Corre cuando se abra la ventana. (\*) |
| Haga clic | Ejecutar durante un clic del ratón sobre el control Gráfico. |
|   | Ejecutar en una aplicación de iPhone/iPad antes de mostrar el tooltip al presionar un valor del gráfico con un dedo.<br><br>Este Event se utiliza para iniciar un Procedure específico, por ejemplo. |


(\*) Por defecto, la "Inicialización" Event de cada control se ejecuta según el orden en que se crearon las control en la ventana. Para modificar este orden de ejecución: 

1. En la pestaña "Ventana", en el grupo "Orden", haga clic en "Inicialización". 

2. En la ventana mostrada, utilice las flechas de la derecha para modificar el orden en el que se inicializan los controles.





### Eventos disponibles en la editor de reportes
<a name="eventos_disponibles_editor_reportes_ELTPARAGRAPHE000087"></a>

El editor de reportes maneja los siguientes eventos por Default (en orden de aparición en el editor de código):


| Evento | Condición de ejecución |
| --- | --- |
| Inicialización | Ejecutar al inicializar la reporte. Al iniciar una reporte (incluso durante una prueba), los eventos se ejecutan en el siguiente orden:<br><br>- "Apertura" Event de la reporte.<br><br>- "Iniciando" Event de los bloques de reporte.<br><br>- "Iniciando" Event de los controles. <br><br><br> |
| Antes de imprimir | Corre antes de imprimir el control. |
| Haga clic | Corre al hacer clic en la control. El clic en el control está disponible en el visor de reporte solamente. Una estrella permite al usuario identificar fácilmente los controles en los que se puede hacer clic. |

## Eventos opcionales
<a name="eventos_opcionales_ELTTEXTE000262"></a>
Se pueden utilizar varios eventos opcionales.

Para añadir un evento opcional:

1. Seleccione la control deseada.

2. Mostrar la ventana del código de esta control (tecla F2).

3. Haga clic en el enlace "Añadir otras Event s a xxx" en la parte inferior del código de la ventana, después de la última Event.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Traitements_optionnels_WD_OK%20-%20HC%20N%B0001.gif)


4. Todos los eventos opcionales disponibles para el control se muestran. 

5. Marque la opción Event para agregar y validar. 
	Observación: Puede seleccionar varios eventos opcionales. 

6. El Event opcional seleccionado se agrega automáticamente a los Event s administrados por el control.




Para desactivar un Event opcional, simplemente realice las mismas operaciones para mostrar la lista de Event s opcionales. A continuación, simplemente desmarque los eventos opcionales para eliminar. 
Observación: Si el código desactivado contiene código WLanguage, este código se elimina automáticamente. Se puede gestionar, por ejemplo:

- el desplazamiento del cursor por encima del control,

- botón izquierdo del ratón abajo, arriba, con un doble clic,

- el botón derecho del ratón presionado, liberado o con doble clic 

- la rueda del ratón, etc.




Para más detalles, consulte la página web de [Eventos opcionales](../WDChamp/1014004.md).

### Eventos opcionales específicos de los controles del gráfico
<a name="eventos_opcionales_especificos_los_controles_del_grafico_ELTPARAGRAPHE000154"></a>

WINDEV propone dos eventos opcionales específicos para los controles de gráficos: 


| Evento | Condición de ejecución |
| --- | --- |
| Zoom horizontal | Ejecutar durante un zoom en una control Gráfico. Este Event está dirigido:<br><br>- si sólo está activado el zoom horizontal.<br><br>- si el zoom horizontal y el zoom vertical están habilitados.<br><br><br> |
| Zoom vertical | Ejecutar durante un zoom en una control Gráfico. Este Event está dirigido:<br><br>- si sólo está activado el zoom vertical.<br><br>- si el zoom horizontal y el zoom vertical están habilitados.<br><br><br> |


Por ejemplo, estos eventos de zoom pueden ser usados para manejar el zoom en la Image superpuesta en el gráfico.

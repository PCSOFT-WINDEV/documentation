## Visión general de la control
<a name="vision_general_control_ELTTEXTE000121"></a>
El ".NET 2.0 control (WinForm)" se usa para administrar fácilmente los .NET control s en formato WinForm (desde.NET 2.0).

Para usar este control, le recomendamos encarecidamente que revise la documentación de Microsoft.

**Para crear un ".NET 2.0 control (WinForm)"**: 

1. En la pestaña "Creación", en el grupo "Otros controles", haga clic en ".NET".

2. Haga clic en la ubicación en la que desea crear el control. El control aparece en el editor.




Para ver las características del control, seleccione la opción "Descripción" en el menú contextual.

**Observación**: WINDEV también propone usar controles .Net que usan el formato WPF. Para obtener más información, consulte [.Net control (WPF)](../WDChamp/1017001.md).

## Notas
<a name="notas_ELTTEXTE000145"></a>


### Descripción de una ".NET 2.0 control (WinForm)"
<a name="descripcion_una_net_20_control_winform_ELTPARAGRAPHE000044"></a>

Definir las características de la control: 

1. Abra la ventana de descripción del control. 

2. En la pestaña "General", seleccione el ensamblaje utilizado y la clase associated. El botón "Agregar ensamblados .Net al proyecto" le permite integrar los montajes necesarios. Es posible utilizar: 

	- Un ensamblaje de red a medida

	- Un ensamblaje de red suministrado con la red framework.. 







Entonces, la control puede ser manejada a través de la programación. Para obtener más información, consulte [Programación de una ".NET 2.0 control (WinForm)"](../WDChamp/1000019579.md).


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000064"></a>

Las limitaciones de ".NET 2.0 control (WinForm)" son las siguientes:

- La versión 2.0 de framework .Net (o posterior) debe ser instalada en el equipo. Este framework está incluido en Vista y se puede descargar para Windows XP (desde el sitio de Microsoft)..
	El primer uso de un control que requiere este framework (control Carrusel, Cubo control, ".NET 3.0, 4.0, ... control (WPF)") carga este framework, que puede causar delay (fuera de nuestro control).

- La aplicación no se puede iniciar desde la red.

- La función OVI está desactivada para todas las ventanas que contengan un ".NET 2.0 control (WinForm)".

- Este control no opera bajo el TSE.

- Esta control utiliza la biblioteca wdxxxnet.dll. Esta biblioteca no se puede renombrar al crear el ejecutable.

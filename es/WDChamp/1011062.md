## Presentación
<a name="presentacion_ELTTEXTE000201"></a>
La editor de reportes le permite crear informes "Crosstab".. En este caso, la reporte sólo contiene una entrada doble tabla.

**Ejemplo**: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=etatTableaucroise2.gif&type=thumb)


**Fuente de datos del reporte Tabla cruzada**: 
El Crosstab reporte se puede basar en: 

- en uno o más archivos de datos. En este caso, se utiliza una consulta incrustada. Esta consulta se crea a través de la asistente para crear un "Crosstab" reporte. Para obtener más información, consulte [Creación de un Crosstab reporte basado en una consulta](#NOTE2_1). 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) a control Tabla dinámica. Para obtener más información, consulte [reporte Pivote de la mesa](../WDChamp/1011092.md).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) ninguna fuente de datos: esta opción permite especificar el control Tabla dinámica que se utilizará sólo en tiempo de ejecución, a través del [SourceName](../Proprietes/2511035.md) Property.




**Observación**: Si el "Crosstab" reporte se basa en una consulta, tiene la posibilidad de modificar la consulta incrustada en el editor de consultas. Para editar la consulta incrustada en el editor de consultas:

1. Seleccione "description" en el menú contextual de reporte. La ventana de la description de la reporte se muestra.

2. Visualizar la ficha "Data".

3. Haga clic en el botón "Editar".

## Creación de un Crosstab reporte basado en una consulta
<a name="creacion_crosstab_reporte_basado_una_consulta_ELTTEXTE000225"></a>
Para crear una "Crosstab" reporte basada en una consulta: 

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido: 

2. Aparece la ventana de creación: haga clic en "Reporte", y luego en "Reporte".

3. El asistente de creación de reportes se abre. Seleccione "Crosstab".

4. El asistente le pide que seleccione la fuente de datos del reporte: elija "De uno o más archivos". 




En esta asistente, debe especificar:

- la partida correspondiente a los títulos de las columnas.
	**Observación**: Si este artículo es un artículo de "Fecha", la asistente lo propone:

	- guarde el día, el mes o el año sólo para estas fechas.

	- definir el período a tener en cuenta (fecha de inicio y fecha final).




- la partida correspondiente a los subtítulos de las líneas.
	**Observación**: Si este artículo es un artículo de "Fecha", la asistente lo propone:

	- guarde el día, el mes o el año sólo para estas fechas.

	- definir el período a tener en cuenta (fecha de inicio y fecha final).




- la información que se mostrará en las celdas de tabla. Esta información puede corresponder a:

	- cualquiera de los dos **el número de registros correspondientes a la intersección de filas y columnas**.
			Por ejemplo: el número de productos pedidos por producto y por año. El cliente llamado "Smith" ha realizado 2 pedidos en 2000 y 3 pedidos en 2001.

	- los **suma, la media, el máximo o el mínimo de un ítem**.
			Por ejemplo: el número de productos pedidos por producto y por año. En el año 2000 se encargaron 279 sofás.

	- un **punto**.
			Por ejemplo: la puntuación dada por cada juez a un patinador en competición. En el Campeonato de Europa de 2000, el juez italiano otorgó 5,2 a la patinadora "A".




- el cálculo de totales para las líneas y columnas de tabla (si es necesario).

- el formato de la página en la que se imprimirá la reporte. El reporte se imprime en formato A4 por Default.

- la plantilla de la reporte (si es necesario).

- el nombre y el título de la reporte. Este nombre (".WDE". archivo correspondiente a la reporte) se utilizará para identificar la reporte en sus programas.




El nuevo reporte contiene un control Tabla para mostrar los datos. 

Nota: Al imprimir un Crosstab reporte, si el formato reporte no permite mostrar todas las columnas tabla, el tabla se redimensiona automáticamente de acuerdo con el formato de impresión del reporte (la fuente y las columnas se redimensionan en consecuencia).

## Creación de un Crosstab reporte basado en un control Tabla dinámica
<a name="creacion_crosstab_reporte_basado_control_tabla_dinamica_ELTTEXTE000249"></a>
Para crear un "Crosstab" reporte: 

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

2. Aparece la ventana de creación: haga clic en "Reporte", y luego en "Reporte".

3. El asistente de creación de reportes se abre. 

4. En esta asistente, especifique:

	- el tipo de reporte a crear: "Tabla cruzada".

	- el control Tabla dinámica utilizado como fuente de datos.

	- la plantilla de la reporte (si es necesario).

	- el nombre y el título de la reporte. Este nombre (".WDE". archivo correspondiente a la reporte) se utilizará para identificar la reporte en sus programas.







El nuevo reporte contiene un control Tabla dinámica para mostrar los datos. 




### Características de control Tabla dinámica (editor de Reporte)
<a name="caracteristicas_control_tabla_dinamica_editor_reporte_ELTPARAGRAPHE000131"></a>

Puede configurar las siguientes características de un Crosstab reporte que contenga un control Tabla dinámica: 

- La fuente de datos del reporte puede modificarse en la pestaña "Datos" de la ventana description del reporte ("Reporte description" en el menú contextual del reporte). 

- La pestaña "General" de la ventana control Tabla dinámica description sirve para especificar: 

	- **el modo de cálculo de la anchura de las celdas**: las opciones disponibles son: 

		- Utilizar ancho definido en edición:  el ancho de las celdas será el mismo que el definido en la editor de reportes.
						Observación: Las manijas se pueden usar para redimensionar el área de datos en la control Tabla dinámica que se encuentra en el reporte. 

		- Llene la página tanto como sea posible en ancho: las celdas se ampliarán para llenar el espacio disponible en la página.

		- Evitar celdas truncadas: las celdas se ampliarán para mostrar todos los datos si hay suficiente espacio en la página.




- **las dimensiones en expansión**: las opciones disponibles son: 

	- Como el control: el nivel de detalle mostrado en el reporte será el mismo que el mostrado en el control Tabla dinámica utilizado como fuente de datos.

	- Todo expandido: se mostrarán todos los datos encontrados en la tabla pivotante.

	- Todo contraído: solo se mostrará el primer nivel de detalles de la Tabla dinámica.

- La pestaña "Estilo" de la ventana de description del PVT control se utiliza para definir el estilo del control: la opción "Definir el estilo independientemente del control PVT impreso" se utiliza para definir un estilo específico para el control que se encuentra en el reporte. Si esta opción está seleccionada, puede especificar el estilo para las cabeceras de fila y para los datos.

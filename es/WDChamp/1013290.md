
## Los diferentes tipos de controles de la Tabla: se rellena mediante programación, con o sin acceso a la fuente de datos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000314"></a>
WINDEV, WEBDEV y WINDEV Mobile le permiten usar:

- Controles de tabla rellenados mediante programación. 

- Controles de tabla basados en un archivo de datos con acceso directo (llamados control Tabla basados en un archivo de datos). 

- Controles de mesa basados en un archivo de datos cargados en memoria. En este caso, sólo se carga un número limitado de registros en la memoria. Si el archivo de datos contiene más registros que el número definido en la descripción del control Tabla, se muestra un mensaje al usuario: "Hay más de XXX registros. Sólo se mostrarán las primeras filas XXX".




La tabla a continuación presenta las diferentes características disponibles (o no) para los diferentes tipos de controles de la Tabla.

Observación: El concepto de control basado en un "archivo cargado en la memoria" también se puede aplicar a:

- Controles de List Box,

- Controles ListView,

- Controles Combo Box.

Los controles de tabla con fuente de datos en memoria son similares a los controles de tabla estándar. Las principales diferencias entre estos dos tipos de controles se muestran en negrita.

Los controles de tabla con fuente de datos en memoria tienen múltiples ventajas, especialmente la capacidad de ordenar y buscar en cualquier columna. También se utilizan para almacenar el valor de las columnas calculadas para todas las filas.

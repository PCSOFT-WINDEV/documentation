## Las diferentes opciones de exportación en detalle
<a name="las_diferentes_opciones_exportacion_detalle_ELTTEXTE000284"></a>


### Guardar y previsualizar la reporte en formato Word
<a name="guardar_previsualizar_reporte_formato_word_ELTPARAGRAPHE000057"></a>

Se crea un archivo en formato RTF (Rich Text Format). El archivo creado se visualiza directamente en uno de los editores de texto instalados en el ordenador actual (Word, OpenOffice o NotePad, por ejemplo).

Por Default:

- El archivo generado se llama "&lt;ReportName>.RTF".

- En el modo de prueba, este archivo se crea en el subdirectorio EXE del directorio del proyecto.

- En los ordenadores de los usuarios, este archivo se crea en el directorio instalación de la aplicación.




El usuario puede modificar estas características haciendo clic en el botón "Palabra".

Observaciones:

- Un editor de texto debe estar instalado en la computadora actual para poder ver el archivo generado.

- Para guardar un archivo en RTF, el archivo "RICHED20.DLL" debe encontrarse en el ordenador actual. En la mayoría de los casos, el archivo "RICHED20.DLL" se encuentra en el directorio del sistema de Windows.

- No se admite la impresión de textos que no estén orientados horizontalmente (90° por ejemplo).

- Las tablas son compatibles con el formato RTF. La fuente Color y el fondo Color de las células se almacenan.



<a name="NOTE2_EXCEL"></a>


### Guardar y previsualizar la reporte en formato Excel
<a name="guardar_previsualizar_reporte_formato_excel_ELTPARAGRAPHE000079"></a>

Se crea un fichero en formato XLS. El archivo creado se visualiza directamente en una de las aplicaciones de hojas de cálculo instaladas en el equipo (Excel u OpenOffice por ejemplo).

Por Default:

- El archivo generado se llama "&lt;ReportName>.XLS".

- En el modo de prueba, este archivo se crea en el subdirectorio EXE del directorio del proyecto.

- En los ordenadores de los usuarios, este archivo se crea en el directorio instalación de la aplicación.




El usuario podrá modificar estas características haciendo clic en el botón "Excel".

Observaciones:

- Para configurar las opciones de exportación en formato XLS, utilice [iParameterXLS](../WDLang5/1000017224.md). 

- Se debe instalar una aplicación de hoja de cálculo en el ordenador actual para poder ver el archivo generado.

- Los controles Image, Código de barras y Gráfico no se imprimen.



<a name="NOTE2_3"></a>


### Guardar y previsualizar la reporte en formato Document portátil (PDF)
<a name="guardar_previsualizar_reporte_formato_document_portatil_pdf_ELTPARAGRAPHE000103"></a>

El archivo creado se muestra directamente en un visor de PDF (por ejemplo, Adobe Acrobat) instalado en el ordenador actual.

Por Default:

- El archivo generado se llama "&lt;ReportName>.PDF".

- En el modo de prueba, este archivo se crea en el subdirectorio EXE del directorio del proyecto.

- En los ordenadores de los usuarios, este archivo se crea en el directorio instalación de la aplicación.




El usuario podrá modificar estas características haciendo clic en el botón "PDF".

Observaciones: 

- Para configurar las opciones de impresión del PDF, utilice [iParameterPDF](../WDLang5/3046007.md). 

- Se debe instalar un visor de PDF en la computadora actual para poder ver el archivo generado.

- Pueden surgir problemas al imprimir en PDF. Para obtener más información, consulte [Impresión en PDF: Problemas conocidos](../WDChamp/1011016.md).



<a name="NOTE2_2"></a>


### Guardar y previsualizar la reporte en formato HTML
<a name="guardar_previsualizar_reporte_formato_html_ELTPARAGRAPHE000130"></a>

El archivo creado se puede ver en el navegador Default instalado en el ordenador actual.

Por Default:

- El archivo generado se llama "&lt;ReportName>.HTM".

- En el modo de prueba, este archivo se crea en el subdirectorio EXE del directorio del proyecto.

- En los ordenadores de los usuarios, este archivo se crea en el directorio instalación de la aplicación.




El usuario podrá modificar estas características haciendo clic en el botón "HTML".

Observaciones:

- Por Default, el archivo HTML creado está usando las hojas de estilo (CSS). Para que el archivo HTML creado no utilice ninguna hoja de estilo, la impresión debe iniciarse mediante programación.

- Un navegador INTERNET debe ser instalado en el ordenador actual para poder ver el archivo generado.



<a name="NOTE2_4"></a>


### Guardar y previsualizar reporte en formato XML (eXtensible Markup Language)
<a name="guardar_previsualizar_reporte_formato_xml_extensible_markup_language_ELTPARAGRAPHE000150"></a>

El archivo creado se muestra automáticamente en el visor XML (un navegador INTERNET por ejemplo) instalado en el ordenador actual.

Por Default:

- El archivo generado se llama "&lt;ReportName&gt;.XML".. No se genera ninguna hoja de estilo.

- En el modo de prueba, este archivo se crea en el subdirectorio EXE del directorio del proyecto.

- En los ordenadores de los usuarios, este archivo se crea en el directorio instalación de la aplicación.




El usuario podrá modificar estas características haciendo clic en el botón "XML".

Observación: Se debe instalar un visor de archivos XML en el ordenador actual para poder ver el archivo generado.

### Email con contenido integrado
<a name="email_con_contenido_integrado_ELTPARAGRAPHE000167"></a>

El reporte se ve directamente en un nuevo correo electrónico creado con el software de mensajería Default instalado en el ordenador actual.

Los sistemas de mensajería soportados son los siguientes:

- Microsoft Outlook,

- Outlook Express,

- Eudora,

- IncrediMail,

- Netscape,

- Mozilla Thunderbird,

- Mozilla.




Para todos los demás sistemas de mensajería, se crea un archivo.eml. Si el sistema Email soporta este formato, el Email puede ser visto en el ordenador actual.

Observaciones:

- Si el mensaje de Default es Eudora: El reporte se genera en formato de texto y se incluye en el Email.

- Si el mensaje de Default es Outlook: El reporte se genera en RTF y se incluye en el Email.

- Para todos los demás tipos de software de mensajería, el reporte se genera en formato HTML y se incluye en el Email.

- No se admite la impresión de textos que no estén orientados horizontalmente (90° por ejemplo).

- Puede configurar los elementos de la nueva Email con [iParameterExport](../WDLang5/1000017026.md).

<a name="email_con_pdf_adjunto_ELTPARAGRAPHE000197"></a>

El reporte se guarda automáticamente en un archivo PDF.

Este archivo PDF se adjunta directamente a un nuevo Email creado utilizando el cliente MAPI instalado por un software de mensajería del ordenador actual. El cliente MAPI y la aplicación deben instalarse en el mismo modo de compilación (32/64 bits).

Observaciones: 

- Por Default, el archivo generado se llama "&lt;ReportName

- Puede configurar los elementos de la nueva Email con [iParameterExport](../WDLang5/1000017026.md). 

- Pueden surgir problemas al imprimir en PDF. Para obtener más información, consulte [Impresión en PDF: Problemas conocidos](../WDChamp/1011016.md).

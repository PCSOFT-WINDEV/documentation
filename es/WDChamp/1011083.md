


## Tabla TreeView reporte
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000164"></a>
La editor de reportes permite crear informes "TreeView Table".. 
La reporte incluye una tabla (como los informes de tabla). Por defecto, la cabecera se imprime en el bloque Cabecera de página, las líneas de datos en el bloque Cuerpo y el total fila en el bloque Fin de Document.

Esta tabla contiene una columna de vista en árbol: Esta columna mostrará la jerarquía de reporte. 

Ejemplo: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Etat_Tableau_H.gif&type=thumb)


**Fuentes del reporte Tabla TreeView**
Las fuentes de la tabla TreeView reporte pueden ser: 

- uno o más archivos de datos,

- un control Tabla TreeView, 

- ninguno. En este caso, el reporte se programa con las funciones WLanguage.

## Creación de una tabla TreeView reporte
<a name="creacion_una_tabla_treeview_reporte_ELTTEXTE000188"></a>


### Creación
<a name="creacion_ELTPARAGRAPHE000040"></a>

Para crear una tabla TreeView reporte desde el editor de reportes: 

1. Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- Aparece la ventana de creación de elementos: haga clic en "Reporte" y luego en "Reporte".

	- Se inicia el asistente de creación de reporte.




2. Elija una "Tabla TreeView" reporte. El asistente propone las fuentes de datos disponibles: 

	- uno o más archivos de datos

	- un control Tabla TreeView. 

	- por programación. 

	- ninguna fuente de datos. 




3. Si selecciona "De uno o más archivos de datos", el asistente le permite definir: 

	- los archivos que se utilizarán como fuente de reporte: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Etat_TabH_Assistant.gif&type=thumb)


	- los artículos a imprimir, sus leyendas y sus bloques: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Etat_TabH_Assistant_rub.gif&type=thumb)

4. Si selecciona "De una control Tabla TreeView", el asistente le pedirá los siguientes elementos:  

	- el nombre de la control Tabla TreeView en la que se basará la reporte. Sólo se proponen los controles de la tabla TreeView que se encuentran en las ventanas o páginas del proyecto actual.

	- las líneas a imprimir: líneas seleccionadas o todas las líneas de control. 

	- los artículos (columnas de la control Tabla TreeView) a imprimir, sus pies de foto y sus bloques. 

	- los cálculos a realizar (sobre los ítems numéricos).




5. Especificar el formato de la hoja en la que se imprimirá el reporte. El reporte se imprime en formato A4 por Default.

6. Seleccione la plantilla que desea utilizar. 

7. Escriba el nombre y el título de reporte (nombre del archivo ".WDE" correspondiente al reporte). Este nombre se utilizará para identificar la reporte en sus programas..
	![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) También tiene la posibilidad de especificar si el usuario final puede modificar la reporte en "Reports and Queries".. 

8. Validar la creación de reporte. 

9. Si el tabla generado es demasiado grande, usted tiene la capacidad de: 

	- Imprimir la tabla en varias páginas (en ancho). Consulte [Impresión de varias páginas](../WDChamp/1011065.md) para obtener más detalles. 

	- Utilizar el modo horizontal, 

	- Reducir el tamaño de tabla. 




10. La reporte creada actualmente se visualiza en el editor de reportes. 




Observación: Si el reporte se crea por programación o sin fuente de datos, se crea un reporte que contiene un tabla con una columna de árbol..

### Observaciones
<a name="observaciones_ELTPARAGRAPHE000092"></a>

- La pestaña "Datos" en la ventana de description de reporte se usa para encontrar y modificar la fuente de datos. Si la reporte se basa en: 

	- los diferentes niveles de jerarquía son Visible y modificables.. 

	- una control Tabla TreeView, tiene la posibilidad de definir las opciones de impresión de la jerarquía. 




- Si la reporte se basa en un control Tabla TreeView: 

	- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [..SelectionOnly](../Proprietes/2511017.md) se utiliza para averiguar y modificar los registros que se imprimirán en el reporte: imprimir todos los registros o sólo los registros seleccionados.

	- [..KeepVisibleHierarchy](../Proprietes/1000021055.md) se utiliza para averiguar y modificar el modo de impresión de la jerarquía: imprimiendo toda la jerarquía o imprimiendo la jerarquía que es Visible en el control.

	- [..HierarchicalDepth](../Proprietes/1000021027.md) se utiliza para averiguar la profundidad del Item control que define la jerarquía. Puede, por ejemplo, modificar las características de los elementos que se encuentran en un nivel determinado.. 

	- [..ExpandedImage](../Proprietes/2513015.md) y [..CollapsedImage](../Proprietes/2513016.md) se usan para encontrar y modificar la imagen expandida o colapsada associated con el control que define la jerarquía. Ninguna imagen es associated con la jerarquía ser Default. 




- La jerarquía de un control se define en la pestaña "Link" de la ventana description de control.

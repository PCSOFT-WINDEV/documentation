


## Imprimir un reporte en WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Al crear una reporte, la prueba de esta reporte se puede ejecutar desde el editor de reportes.

También tiene la posibilidad de incluir esta reporte en sus aplicaciones o en sus sitios.. El usuario o el usuario de la Web tendrá la posibilidad de imprimir la reporte desde un botón, una opción de menú, ... por ejemplo.

Para un sitio WEBDEV, el servidor preparará el reporte en el formato seleccionado (PDF, HTML, ...). Entonces, tendrás la habilidad de hacerlo:

- visualizar esta reporte en el navegador del usuario Web,

- guardarlo en el servidor y proponer descargarlo o mostrarlo, ....




En un sitio INTRANET, el reporte también puede imprimirse en una impresora local o en una impresora de red conectada al servidor.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Principio para la impresión en WEBDEV
<a name="principio_para_impresion_webdev_ELTTEXTE000168"></a>
Independientemente del tipo de reporte a imprimir, el principio para imprimir un reporte es el siguiente (en código de servidor):

1. Especifique el modo de impresión de la reporte (impresión directa, generación de una página HTML, generación de un archivo PDF, ...) con [iDestination](../WDLang5/3046074.md).

2. Inicie el trabajo de impresión con [iPrintReport](../WDLang5/3046032.md).

3. Entonces, tendrás la habilidad de hacerlo:

	- mostrar el archivo generado en el equipo del usuario web con [FileDisplay](../WDLang2/3012005.md).

	- enviar un Email al usuario de la web permitiéndole descargar el archivo creado.

	- ...







**Atención**: Si el destino de la impresión es un archivo en formato HTML, PDF, XML o RTF, deben comprobarse los siguientes puntos:

- Tienes los derechos para escribir en el directorio donde se generará el Document (el Document es generado en el directorio del sitio por Default).
	Si genera un reporte en formato PDF, este archivo puede guardarse en el directorio de datos (ruta devuelto por [fDataDir](../WDLang1/3036001.md))
	Si genera un reporte en formato HTML, este archivo debe guardarse en el directorio _WEB del sitio (ruta devuelto por [fWebDir](../WDLang1/3036052.md)).

- Se debe utilizar un nombre específico para cada Document generada en el servidor. De lo contrario, varias impresiones simultáneas sobrescribirían las impresiones ya realizadas.. 
	Utilice [Cuenta de conexión](../WDLang2/3012004.md) o [GetIdentifier](../WDLang1/3013028.md) para nombrar sus informes (por ejemplo: NombreArchivo = "ReporteCliente" + **ConnectionCount**() + ".HTML").




Este principio debe ser adaptado en función de la fuente de datos utilizada en cada reporte.

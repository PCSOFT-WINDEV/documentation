## Eventos de Default
<a name="eventos_default_ELTTEXTE000134"></a>
WINDEV y WINDEV Mobile gestionan los siguientes eventos de Default:


| Event | Condición de ejecución |
| --- | --- |
| Inicialización | Se ejecuta al abrir la ventana. (\*) |
|   | Corre al hacer clic en la control.<br><br>Este evento le permite, por ejemplo, tomar una foto o Record un clip (y luego detenerlo) haciendo clic en la vista previa.<br><br><br>	<B><B><B><B>Advertencia</B></B></B></B><br><br>	 Este Event sólo puede ser y ejecutado si el control Cámara está en modo "compatible con la versión 26".<br>
 |
|   | Se ejecuta cuando el usuario se acerca o se aleja en el control Cámara<br><br>**Observación: Este evento sólo está disponible para los controles de cámara creados con la versión 27 o posterior (controles con la opción "Compatible con la versión 26" desmarcada en la pestaña "General" de la ventana de description).** |
|   | Ejecutar cuando la cámara detecta un código de barras. <br><br>**Este evento sólo está disponible si se ha configurado la descodificación del código de barras en la pestaña "Detalles" de la ventana del control description.** <br><br>Este Event tiene el siguiente formato: <br><br><pre><code>PROCEDURE DecodeBarCode(bc is BarCode)</code></pre><br><br><br>La información leída puede ser recuperada por el [BarCode](../WDLang5/1000019451.md) Variable. <br><br>Para obtener más información, consulte [Control Cámara: decodificando los códigos de barras](../WDChamp/1013304.md). |


(\*) De forma predeterminada, el evento "Inicialización" de cada control se ejecuta según el orden en el que se crearon los controles en la ventana. Para modificar este orden de ejecución: 

1. En la pestaña "Ventana", en el grupo "Orden", haga clic en "Inicialización".

2. En la ventana que aparece, utilice las flechas de la derecha para cambiar el orden en el que se inicializan los controles.

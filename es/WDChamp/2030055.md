
## Página compatible con SEO en modo sesión
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000148"></a>
En WEBDEV, hay dos métodos para desarrollar un sitio: 

- el modo "Sesión", que se basa en sesiones automáticas,

- el modo "AWP", que requiere la gestión de contextos mediante programación.




Hasta la versión 27, el modo Sesión no permitía posicionar las páginas de un sitio en los buscadores: solo se podía posicionar la página de inicio. La única manera para posicionar las páginas en los buscadores era mediante la tecnología AWP. 

**A partir de la versión 27**, el modo Sesión permite posicionar las diferentes páginas de un sitio en los buscadores, cada una con su propia URL. El posicionamiento de páginas en buscadores incluye varias ventajas: 

- La página puede aparecer en los resultados de búsqueda de Google, 

- La página tiene una URL fija, que se puede copiar, enviar por correo electrónico y establecer como marcador, 

- No hay tiempo de espera para acceder a la página,

- La reescritura de URL está disponible,

- etc.




Observación: Esta página de ayuda muestra cómo hacer que una página en modo Sesión sea compatible con SEO. Para cambiar un sitio del modo AWP al modo Sesión compatible con SEO, consulte [Cambiar los sitios del modo AWP al modo Sesión compatible con SEO](../WDChamp/2030030.md). 
<a name="NOTE1_2"></a>


### Páginas en modo Sesión compatibles con SEO
<a name="paginas_modo_sesion_compatibles_con_seo_ELTPARAGRAPHE000032"></a>

Cuando un usuario introduce una dirección en el navegador que corresponde a una página WEBDEV, WEBDEV Application Server: 

- Comprueba que no haya sesiones actuales para el usuario. 

- Crea una sesión.

- Ejecuta el código de inicialización del proyecto.

- Abre la página solicitada (sin abrir la "primera página" del proyecto). 


**Observación**: Cuando se crea la sesión, todas las llamadas al servidor utilizan la sesión abierta y son mucho más rápidas. No se requiere ningún procedimiento especial para mostrar las siguientes páginas.

**Atención**: no todas las páginas de un sitio WEBDEV deben ser compatibles con SEO. Algunas páginas (especialmente las que requieren autenticación) no necesitan ser posicionadas en los buscadores.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000178"></a>


### Cómo hacer que una página en modo Sesión sea compatible con SEO
<a name="como_hacer_que_una_pagina_modo_sesion_sea_compatible_con_seo_ELTPARAGRAPHE000050"></a>

Para hacer que una página en modo Sesión sea compatible con SEO: 

1. Abra la ventana de descripción de la página: en la pestaña "Página", en el grupo "Descripción", haga clic en "Descripción". 

2. Vaya a la pestaña "General", sección "Tipo de página": 

	- Seleccione el modo "Sesión" si es necesario y marque la casilla "SEO". 

	- Si es necesario, especifique el nombre que se utilizará en el URI. Este nombre puede ser diferente del nombre de la página en el editor. 
			Observación: Para los sitios multilingües, puede introducir el nombre en los diferentes idiomas del proyecto. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Page_session_reference%20-%20HC%20N%B0002.gif&type=thumb)





3. En la pestaña "Detalles", introduzca expresiones y palabras clave que optimicen el posicionamiento en buscadores (botón "Editar expresiones y palabras clave"). <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Page_session_reference%20-%20HC%20N%B0003.gif&type=thumb)


4. Valide la ventana de descripción de la página. La página ya está preparada para el SEO. 




Observación: Al cambiar las páginas al modo Sesión compatible con SEO, también cambia la extensión del archivo en la URL. Las páginas se convertirán en archivos '.wb'. Los archivos '.wb' no existen físicamente. 


### Puntos importantes a comprobar y cambios a realizar
<a name="puntos_importantes_comprobar_cambios_realizar_ELTPARAGRAPHE000077"></a>

- Como la página es compatible con SEO, debe aceptar como parámetros los elementos que le permiten trabajar de forma independiente a otras páginas. Para ello, se recomienda comprobar los parámetros que recibirá la página. 

- Una página compatible con SEO se puede abrir: 

	- mediante un control Enlace o Botón (recomendado). En la ventana de descripción del control:: 

		- seleccione el tipo de acción "Mostrar una página del sitio". 

		- seleccione el nombre de la página a abrir. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Page_session_reference%20-%20HC%20N%B0004.gif)


		- haga clic en "Parámetros..." para especificar los parámetros que se pasarán a la página. Cada parámetro puede ser un valor fijo, un control o una variable. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Page_session_reference%20-%20HC%20N%B0005.gif)

- mediante la función [PageDisplay](../WDLang2/3058008.md) en código Servidor.

- Caso especial: páginas que utilizan el contexto HFSQL actual: 
	Las páginas compatibles con SEO no deben utilizar el contexto HFSQL actual. Si se utiliza un enlace permanente, se hará referencia a un contexto HFSQL que no existe. Por lo tanto, todos los elementos necesarios para encontrar el contexto se deberán pasar como parámetros a la página. 

- Las variables globales deben utilizarse con precaución, para evitar el uso de una variable global que aún no se ha inicializado al abrir una página compatible con SEO. 

- La reescritura de URL está disponible en páginas compatibles con SEO en modo Sesión.

- La función [PreviousPage](../WDLang2/3058017.md) no puede utilizarse en una página compatible con SEO. La página anterior puede ser cualquier página, incluso una página que no pertenezca al sitio actual.

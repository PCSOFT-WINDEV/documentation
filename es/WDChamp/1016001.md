## Resumen de control Carrusel
<a name="resumen_control_carrusel_ELTTEXTE000286"></a>
La control Carrusel se utiliza para mostrar un menú animado hecho de imágenes.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=CHP_Carroussel.gif)


**Para crear un control Carrusel**: 

1. En la pestaña "Creación", en el grupo "Otros controles", haga clic en "Carrusel".

2. Haga clic en la ubicación en la que desea crear el control.




Para visualizar las características del control, seleccione "description" en el menú contextual del control:

- La pestaña "General" permite definir la orientación, la animación y el fondo Image.

- La pestaña "Detalles" permite definir el modo de visualización del carrusel control.

- La pestaña "Contenido" permite definir las imágenes que aparecen en el carrusel control, así como su leyenda.




**Observación**: WINDEV le permite crear automáticamente un control Carrusel a partir de varios controles:  

1. Selecciona los controles a incluir en el control Carrusel. 

2. En la pestaña "Modificación", en el grupo "Transformaciones", despliegue "Refactorización y permutaciones" y seleccione "Crear un carrusel a partir de la selección".




Sólo los controles Image y los controles Button se tienen en cuenta para crear este carrusel.

## Dos modos para usar la control Carrusel
<a name="dos_modos_para_usar_control_carrusel_ELTTEXTE000310"></a>


### Presentación
<a name="presentacion_ELTPARAGRAPHE000061"></a>

Para optimizar la control Carrusel, WINDEV propone dos modos de funcionamiento:

- Usando el.NET 3.0 framework.

- Utilizando el WINDEV framework (GDI+).



<a name="NOTE2_2"></a>


### .NET 3.0 framework
<a name="net_30_framework_ELTPARAGRAPHE000071"></a>

Para basar el control Carrusel en el framework de .Net 3.0 , basta con marcar la opción "El control se basa en el framework .Net 3.0
(las funciones Xaml también estarán disponibles para este control)" en la pestaña "Detalles" de la ventana del control description.

En este caso:

- Las funciones de la familia Xaml están disponibles para esta control.

- Puede proponer la descarga de la framework.Net 3.0 si ésta no está instalada en el ordenador del usuario final.. Este framework está incluido en Vista y se puede descargar para Windows XP (desde el sitio de Microsoft).

- El primer uso de un control que requiere el framework de .Net 3.0 (control Carrusel, Cube control o Xaml control) carga este framework, lo que puede causar delay (fuera de nuestro control).

- Esta control consume muchos recursos: le recomendamos que lo use sólo en computadoras de alto rendimiento.




Dependiendo de la configuración de el equipo, la visualización de la control Carrusel se hará de la siguiente manera:


|   |   |   |   |
| --- | --- | --- | --- |
|   | **Opción "Descarga" seleccionada** | **Opción "Descarga" no seleccionada** |
| **Equipo con framework .NET 3.0** | **Equipo sin framework .NET 3.0** | **Equipo sin framework .NET 3.0** |
| **Computadora con GDI+ framework** | Uso del framework .NET | Uso del framework GDI + | Uso del framework GDI + |
| **Equipo sin framework GDI+** | x | Enlace para descargar el .NET framework<br>Enlace para mostrar una vista de lista | Usando una vista de lista (modo lisiado) |


**Limitaciones del modo control Carrusel en .Net 3.0**: 

- La versión 3.0 de framework .Net (o posterior) debe ser instalada en el equipo. Este framework está incluido en Vista (y más tarde) y puede ser descargado para Windows XP (desde el sitio de Microsoft).. 

- La aplicación no se puede iniciar desde la red.

- La función OVI (Oscurecimiento de Ventanas Inactivas) está desactivada para todas las ventanas que contienen un control Carrusel.

- Este control no opera bajo el TSE.

- Esta control utiliza la biblioteca wd28 0xaml.dll. Esta biblioteca no se puede renombrar al crear el ejecutable.



<a name="NOTE2_3"></a>


### WINDEV framework (GDI+)
<a name="windev_framework_gdi_ELTPARAGRAPHE000154"></a>

Para basar el control Carrusel en el WINDEV framework, basta con marcar la opción "El control se basa en el framework WINDEV (GDI+)" en la pestaña "Detalles" de la ventana de control description.

En este caso, las funciones de la familia Xaml no están disponibles para esta control.

Dependiendo de la configuración de el equipo, la visualización de la control Carrusel se hará de la siguiente manera:


|   |   |   |
| --- | --- | --- |
|   | **Equipo con framework .NET 3.0** | **Equipo sin framework .NET 3.0** |
| **Computadora con GDI+ framework** | Uso del framework GDI + | Uso del framework GDI + |
| **Equipo sin framework GDI+** | x | Usando una vista de lista (modo lisiado) |


Para obtener más información, consulte [Instalación del marco GDI+](../Editeurs/2020007.md).
<a name="NOTE2_4"></a>


### Modo degradado 
<a name="modo_degradado_ELTPARAGRAPHE000204"></a>

Se requiere la versión 3.0 (o posterior) del framework .Net o del framework WINDEV (GDI+) para utilizar los controles Carrusel. No obstante, WINDEV propone un modo "lisiado" para esta control, de modo que pueda utilizarse independientemente de la configuración del ordenador del usuario final. Si el framework de .Net 3.0 o el framework de WINDEV (GDI+) no están instalados en el ordenador del usuario final, el control Carrusel será sustituido automáticamente por un listview en modo lisiado.

**Para usar este modo lisiado en sus aplicaciones**:

1. Abrir la description de la control Carrusel.

2. En la pestaña "Detalles", puede configurar el control ListView utilizado a través de "Ajustes del control ListView".

3. Validar el control description.




El control ListView se utilizará según el modo de visualización del control Carrusel y según la configuración del ordenador del usuario (véanse los párrafos anteriores).

## Notas
<a name="notas_ELTTEXTE000352"></a>


### Simulando la ausencia de .Net 3.0 framework
<a name="simulando_ausencia_net_30_framework_ELTPARAGRAPHE000225"></a>

Cuando se desarrolla una aplicación que contiene una control Carrusel, puede ser aconsejable ejecutar la prueba de la aplicación en un ordenador en el que no esté instalada la versión 3.0 (o posterior) de framework de.net.

Simular la ausencia de la framework en un ordenador:

- utilice la función [XamlDisable](../WDLang1/1000015004.md).

- cambiar el nombre de la siguiente clave de registro: "HKEY_LOCAL_MACHINE\\SOFTWARE\\Microsoft\\.NETFramework\\v3.0".


**Observaciones**: Un control Carrusel que utiliza el framework de .NET puede ser sustituido automáticamente por un control Carrusel que utiliza el framework de WINDEV (GDI+).

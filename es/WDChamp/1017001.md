## Visión general de la control
<a name="vision_general_control_ELTTEXTE000128"></a>
**Atención**: El Xaml control que se encontraba en las versiones anteriores de WINDEV (hasta la versión 15) se llama ahora ".NET 3.0, 4.0, ... control (WPF)". En esta documentación, utilizaremos el término ".Net control (WPF)".. La control .NET (WPF) (también llamada "XAML") permite gestionar fácilmente las control.NET en formato WPF (para .NET 3.0, 3.5 y 4.0) y especialmente el lenguaje Xaml.

Para usar este control, le recomendamos encarecidamente que revise la documentación de Microsoft.

**Para crear un.Net control (WPF)**: 

1. En la pestaña "Creación", en el grupo "Otros controles", haga clic en "XAML".

2. Haga clic en la ubicación en la que desea crear el control. El control aparece en el editor.




Para ver las características del control, seleccione la opción "Descripción" en el menú contextual.

### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000054"></a>

Las limitaciones del .Net control (WPF) son las siguientes:

- La versión 3.0 de framework .Net (o posterior) debe ser instalada en el equipo. Este framework está incluido en Vista y se puede descargar para Windows XP (desde el sitio de Microsoft)..
	El primer uso de un control que requiere este framework (control Carrusel, Cubo control, ".NET 3.0, 4.0, ... control (WPF)") carga este framework, que puede causar delay (fuera de nuestro control).

- La aplicación no se puede iniciar desde la red.

- La función OVI está desactivada para todas las ventanas que contengan un ".NET 3.0, 4.0, ... control (WPF)".

- Este control no opera bajo el TSE.

- Esta control utiliza la biblioteca wd28 0xaml.dll. Esta biblioteca no se puede renombrar al crear el ejecutable.

### Simulando la ausencia de .Net 3.0 framework
<a name="simulando_ausencia_net_30_framework_ELTPARAGRAPHE000071"></a>

Cuando se desarrolla una aplicación que contiene un control .Net (WPF), puede ser useful para saber cómo funcionará la aplicación cuando se inicie en un ordenador que no esté equipado con el control .Net 3.0 framework (o posterior).

Simular la ausencia de la framework en un ordenador:

- utilice la función [XamlDisable](../WDLang1/1000015004.md).

- cambiar el nombre de la siguiente clave de registro: "HKEY_LOCAL_MACHINE\\SOFTWARE\\Microsoft\\.NETFramework\\v3.0".

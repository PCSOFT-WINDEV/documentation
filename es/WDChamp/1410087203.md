## Presentación del control Kanban
<a name="presentacion_del_control_kanban_ELTTEXTE000212"></a>
Originalmente, Kanban es un método japonés destinado a mejorar los procesos para conseguir un ciclo de producción más eficaz.
Hoy en día, Kanban es un método en el que las tareas se clasifican visualmente, lo que permite una mejor gestión de las mismas.
Cada tarea se escribe en una "tarjeta" que se mueve a una lista (o columna) diferente según el estado de progreso.

Además de las tareas, las tarjetas pueden representar cualquier otro tipo de recurso: historia de usuario, requisito, sugerencia, fase de fabricación, entrega, resolución de problemas, etc.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Kanban%20-%20Gala.png)


Un tablero Kanban suele compartirse entre varios usuarios. Cada miembro del equipo puede ver el estado de avance del proyecto y saber cuáles son sus tareas y las de los demás miembros.

Además de mejorar la comunicación, este método basado en tarjetas ofrece una visión clara de la cadena de producción y del progreso de las tareas, lo que permite a los miembros identificar fácilmente los obstáculos y dar prioridad a determinadas tareas.

Puede utilizar el mismo control Kanban en WINDEV, WEBDEV y WINDEV Mobile. Las principales funcionalidades son las mismas, con algunos ajustes para móvil y web que permiten aprovechar al máximo cada plataforma. Por ejemplo, el control se adapta al modo vertical en los dispositivos móviles.

## Crear un control Kanban
<a name="crear_control_kanban_ELTTEXTE000236"></a>
**Para crear un control Kanban**:

1. En la pestaña "Creación", en el grupo "Controles gráficos", despliegue "Ofimática" y seleccione "Kanban".

2. Haga clic en la ubicación en la que desea crear el control. El control aparece en el editor.




Observación: Las dimensiones del control que se creó se optimizan para ocupar el espacio disponible en la ubicación indicada. Si el tamaño del control no es el indicado, pulse Ctrl+Z: el control volverá al tamaño predeterminado.

Para ver las características del control, seleccione la opción "Descripción" en el menú contextual.

## Características del control Kanban
<a name="caracteristicas_del_control_kanban_ELTTEXTE000264"></a>
Los controles Kanban se componen de listas de tarjetas ("listas Kanban"). 

Las principales opciones del control Kanban son las siguientes: 

- Posibilidad de definir las distintas listas del control (pestaña "General"). 

- Posibilidad de definir las dimensiones y el modo de visualización de las listas (pestaña "Detalles"). 

- Posibilidad de seleccionar ventanas y páginas internas (pestaña "Detalles"): 

	- para mostrar tarjetas. 

	- para agregar y eliminar tarjetas (si esta opción está activada).

### Ventanas internas
<a name="ventanas_internas_ELTPARAGRAPHE000084"></a>

Puede utilizar dos tipos de ventanas internas para personalizar el control Kanban: 

- Ventana interna para mostrar tarjetas. 

- Ventana interna para personalizar el área Agregar/Eliminar. 




**Para personalizar completamente el modo de visualización de las tarjetas** (para realizar una traducción, por ejemplo):

1. Abra la ventana de descripción del control Kanban. 

2. En la pestaña "Detalles", seleccione la ventana interna que se utilizará para mostrar las tarjetas: 

	- "Ninguna": se utilizará el modo de visualización predeterminado. 

	- "Ventana predefinida": Si esta opción está seleccionada, el componente interno WDAAF se agrega inmediatamente al proyecto actual. La ventana interna "IW_KANBAN_CARD" se selecciona automáticamente en "Mostrar tarjetas en".
			Observación: Si el componente WDAAF ya existe, solo se agrega la ventana al componente.  

	- Cualquier ventana interna del proyecto. 




3. Si es necesario, valide la ventana de descripción del control. 




**Para personalizar completamente el área Agregar/Eliminar**:

1. Abra la ventana de descripción del control Kanban. 

2. En la pestaña "Detalles", seleccione la ventana interna que se utilizará para mostrar las tarjetas: 

	- "Ninguna": se utilizará el modo de visualización predeterminado. 

	- "Ventana predefinida": Si esta opción está seleccionada, el componente interno WDAAF se agrega inmediatamente al proyecto actual. La ventana interna "IW_KANBAN_LIST_FOOTER" se selecciona automáticamente en la opción "Área Agregar/Eliminar". 
			Observación: Si el componente WDAAF ya existe, solo se agrega la ventana al componente.  

	- Cualquier ventana interna del proyecto. 




3. Si es necesario, valide la ventana de descripción del control. 




**Observaciones**: 

- Cada ventana interna personalizada está disponible en el panel "Explorador de proyectos", en la carpeta "Componentes internos". Puede editarlas en el editor de ventanas. 
	**Atención**: estas ventanas internas solo permiten personalizar el diseño de los elementos. Solo se ejecuta el evento "Declaraciones globales". No se ejecutan otros eventos. Por lo tanto, no se pueden agregar botones con un código de clic específico. 

- Si utiliza la ventana predefinida, se aplicarán las opciones de estilo de la tarjeta (seleccionadas en la pestaña "Estilo").

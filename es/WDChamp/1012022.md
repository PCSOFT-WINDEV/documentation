
## Contextos de página
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000126"></a>
En una aplicación Windows, la información relativa a una ventana se conserva con la ventana.

En un sitio WEBDEV dinámico, existe un contexto de página en el servidor para cada página visualizada en el navegador del usuario Web.. 

El contexto de página contiene toda la información relacionada con la misma:

- contenido de los controles,

- variables locales, 

- variables globales,

- código WLanguage "servidor", etc.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Modo de funcionamiento
<a name="modo_funcionamiento_ELTTEXTE000150"></a>


### Gestión de automatic
<a name="gestion_automatic_ELTPARAGRAPHE000027"></a>

Por Default, WEBDEV gestiona automáticamente los contextos de las páginas dinámicas:

- Se abre un contexto de página cuando la página dinámica se muestra en el navegador.

- El contexto de una página se actualiza según la información que introduce el usuario en el navegador. El contexto se actualiza cuando el usuario valida la página (mediante un botón "Enviar" o con la función [PageSubmit](../WDLang2/3058020.md)).

- Los contextos de página existentes se cierran cuando se utiliza [PageUse](../WDLang2/3058016.md) o [Uso del juego de marcos](../WDLang2/3058012.md). A continuación, se abren los contextos de página correspondientes a las páginas a mostrar.



<a name="NOTE2_2"></a>


### Gestión avanzada
<a name="gestion_avanzada_ELTPARAGRAPHE000047"></a>

WEBDEV permite realizar una gestión avanzada de los contextos con las funciones ContextXXX.

Es posible:

- abrir el contexto de la página sin mostrarla en el navegador ([ContextoAbrir](../WDLang2/3058004.md)),

- comprobar la existencia de un contexto de página en el servidor ([ContextExist](../WDLang2/3058002.md)),

- cerrar el contexto de una página no utilizada ([ContextoCerrar](../WDLang2/3058003.md)).




Esta gestión avanzada de los contextos de página puede ser utilizada para:

- Guardar memoria en el servidor (cerrando los contextos de las páginas previamente abiertas pero no utilizadas, por ejemplo).

- Preparar las páginas destinadas a comunicarse con otras aplicaciones (por ejemplo, la página que recibe información de un sitio para el pago seguro).

- Utilice las páginas de otras páginas.



<a name="NOTE2_3"></a>


### Caso especial
<a name="caso_especial_ELTPARAGRAPHE000075"></a>

Al utilizar el botón "Atrás" del navegador, puede producirse una desincronización entre la página y su contexto.. Para obtener más información, consulte [Gestión del botón "Atrás" del navegador.](../WDChamp/1012001.md)

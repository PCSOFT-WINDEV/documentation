


## Impresión durante el test reporte en el editor
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000147"></a>
Durante la prueba de un reporte (botón"Go" del editor de reportes), se proponen varias opciones de impresión: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=GO_Etat_choix.gif)


**Observación**: Por Default, el modo de impresión del reporte se solicita automáticamente durante la prueba de cada reporte. 

**Tpara evitar que se muestre esta ventana durante la prueba reporte**, simplemente marque "No mostrar de nuevo".

En este caso, la reporte probada se mostrará directamente en el visor de reporte.

Observación: Puedes mostrar esta ventana de nuevo. Para obtener más información, consulte [No volver a mostrar un mensaje](../Editeurs/2020011.md).

## Detalles de los modos de impresión reporte propuestos durante la prueba
<a name="detalles_los_modos_impresion_reporte_propuestos_durante_prueba_ELTTEXTE000171"></a>


### Visor de reportes
<a name="visor_reportes_ELTPARAGRAPHE000038"></a>

El visor reporte se utiliza para previsualizar el diseño de un reporte sin imprimirlo. El diseño y el contenido del reporte se pueden verificar. También puede imprimir el reporte.

Para obtener más información, consulte [Visor de reportes](../WDChamp/1011085.md).
<a name="NOTE2_2"></a>


### Impresora
<a name="impresora_ELTPARAGRAPHE000050"></a>

El reporte se imprime directamente en la impresora Default del ordenador actual, con las opciones Default de la impresora. 
<a name="NOTE2_3"></a>


### Archivo RTF
<a name="archivo_rtf_ELTPARAGRAPHE000057"></a>

La reporte se guarda y visualiza en formato RTF (Rich Text Format). El archivo creado se visualiza en uno de los editores de texto instalados en el ordenador actual (Word, Open Office o NotePad por ejemplo). 
El archivo generado se llama "&lt;ReportName&gt;.RTF".. Este archivo se crea en el subdirectorio EXE del directorio del proyecto.

Observaciones:

- Un editor de texto debe estar instalado en la computadora actual para poder ver el archivo generado.

- Para guardar un archivo en RTF, el archivo "RICHED20.DLL" debe encontrarse en el ordenador actual. En la mayoría de los casos, el archivo "RICHED20.DLL" se encuentra en el directorio del sistema de Windows.

- No se admite la impresión de textos que no estén orientados horizontalmente (90° por ejemplo).

- Los cuadros son compatibles con RTF. La fuente Color y el fondo Color de las células se almacenan.

- No se admiten elementos superpuestos (imágenes y textos, por ejemplo).. 



<a name="NOTE2_4"></a>


### Archivo HTML
<a name="archivo_html_ELTPARAGRAPHE000074"></a>

El reporte se guarda y se visualiza en formato HTML. El archivo creado se puede ver en el navegador Default instalado en el ordenador actual.
El archivo generado se llama "&lt;ReportName&gt;.HTM".. Este archivo se crea en el subdirectorio EXE del directorio del proyecto.

Observaciones:

- El archivo HTML creado utiliza las hojas de estilo (CSS). 

- Un navegador INTERNET debe ser instalado en el ordenador actual para poder ver el archivo generado.



<a name="NOTE2_5"></a>


### Archivo PDF o archivo PDF genérico
<a name="archivo_pdf_archivo_pdf_generico_ELTPARAGRAPHE000088"></a>

El reporte se guarda y se ve en formato PDF (Portable Document Format). El archivo creado se muestra directamente en un visor de PDF (por ejemplo, Adobe Acrobat) instalado en el ordenador actual.

El archivo generado se llama "&lt;ReportName&gt;.PDF".. Este archivo se crea en el subdirectorio EXE del directorio del proyecto. 

Observaciones: 

- Si se selecciona "Archivo PDF", el archivo PDF creado depende de la impresora actual. Si se selecciona "Archivo PDF genérico", el archivo PDF es independiente de la impresora actual.. 

- Se debe instalar un visor de PDF en la computadora actual para poder ver el archivo generado.

- En esta versión, usted puede encontrar problemas conocidos. Para obtener más información, consulte [Impresión en PDF: Problemas conocidos](../WDChamp/1011016.md)

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000079"></a>
Un control Organigrama se puede: 

- manipular mediante programación. 

- vincular a un archivo de datos. 




Para manipular un control Organigrama mediante programación, WINDEV incluye las [Funciones del organigrama](../WDLang1/1000019986.md). 

Esta página de ayuda explica cómo manipular un control Organigrama mediante programación.



## Manipular un control Organigrama mediante programación
<a name="manipular_control_organigrama_mediante_programacion_ELTTEXTE000115"></a>


### Agregar un elemento raíz
<a name="agregar_elemento_raiz_ELTPARAGRAPHE000138"></a>

El elemento raíz de un organigrama es el elemento a partir del cual se construye toda la estructura jerárquica. Un elemento "raíz" puede agregarse a un control Organigrama con la función [&lt;Organigrama&gt;.add](../WDLang1/1000019975.md). 

Esta función acepta dos sintaxis: 

- sintaxis utilizada para especificar las características del elemento: título, imagen, etc, ...
	Ejemplo: 
	
	```wl
	// Adds a root element 
	ORG_HR.Add("Emma Brown")
	```

	**Observación**: La función [&lt;Organigrama&gt;.add](../WDLang1/1000019975.md) inserta una imagen en el elemento (a la izquierda del texto).

- sintaxis que manipula una variable de tipo [OrgElement](../WDLang1/1000019713.md).
	
	```wl
	// Add elements to the Organization Chart control
	// Root element
	ElementCEO is OrgElement
	ElementCEO..Title = "Big Boss"
	ORG_EXECUTIVE.Add(ElementCEO)
	```

	**Observación**: El tipo [OrgElement](../WDLang1/1000019713.md) puede utilizarse para configurar varias opciones específicas: 

	- tooltip, 

	- color de fondo del elemento,

	- imagen de fondo del elemento,

	- fuente del título y del contenido,

	- orientación del elemento.








### Agregar elementos hijos
<a name="agregar_elementos_hijos_ELTPARAGRAPHE000195"></a>

La adición de elementos hijos en una control Organigrama se realiza mediante [&lt;Organigrama&gt;.AddChild](../WDLang1/1000019976.md). Esta función acepta dos sintaxis: 

- sintaxis utilizada para especificar las características del elemento: título, contenido, ...
	
	```wl
	ORG_NoName1.Add("Emma")
	ORG_NoName1.AddChild(1, "Paul Martin")
	ORG_NoName1.AddChild(1, "Florence Mitchell")
	ORG_NoName1.AddChild(3, "Gary Prince")
	ORG_NoName1.AddChild(3, "Joey Moore")
	```


- sintaxis que manipula una variable de tipo [OrgElement](../WDLang1/1000019713.md).
	
	```wl
	// Add elements to the Organization Chart control
	// Root element
	ElementCEO is OrgElement
	ElementCEO.Title = "Big Boss"
	ORG_EXECUTIVE.Add(ElementCEO)
	
	// Child elements
	Element2 is OrgElement
	Element2.Title = "Production"
	ORG_EXECUTIVE.AddChild(1, Element2)
	
	Element3 is OrgElement
	Element3.Title = "Sales"
	ORG_EXECUTIVE.AddChild(1, Element3)
	```

	El **BackgroundColor** Property del [OrgElement](../WDLang1/1000019713.md) Variable utiliza para definir el color utilizado para mostrar un elemento. Si no se define ningún color de fondo, el control Organigrama utilizará automáticamente el color por defecto.





### Eliminar un elemento
<a name="eliminar_elemento_ELTPARAGRAPHE000237"></a>

[.Delete](../WDLang1/1000019984.md) se utiliza para eliminar un elemento (así como sus hijos) de una control Organigrama. 


```wl
// Deletes the element 15 and its children
ORG_MyOrganizationChart.Delete(15)
```


[&lt;Organigrama&gt;.DeleteAll](../WDLang1/1000019985.md) se utiliza para borrar todos los elementos encontrados en una control Organigrama.


## Manejar un elemento que se encuentra en la control Organigrama
<a name="manejar_elemento_que_encuentra_control_organigrama_ELTTEXTE000260"></a>
Los elementos de una control Organigrama se tratan mediante sus índices. Cada elemento de la control Organigrama es associated con un identificador (índice). Este índice se devuelve: 

- por [&lt;Organigrama&gt;.add](../WDLang1/1000019975.md) y [&lt;Organigrama&gt;.AddChild](../WDLang1/1000019976.md). 

- por [&lt;Organigrama&gt;.InfoXY](../WDLang1/1000019977.md). 




Este índice es utilizado por todas las funciones para el manejo de elementos (selección, borrado, etc.). 

Para manipular o modificar un elemento específico en el control Organigrama, simplemente: 

- defina una variable de tipo [OrgElement](../WDLang1/1000019713.md). 

- asignarle el elemento que desea del control Organigrama. 




Entonces, el elemento puede ser manejado por las propiedades de [OrgElement](../WDLang1/1000019713.md). 

Por ejemplo: 


```wl
MyElement is OrgElement 
MyElement = ORG_MyOrganizationChart[2]

Trace(MyElement.Title)
```



### Cambiar la orientación de un elemento
<a name="cambiar_orientacion_elemento_ELTPARAGRAPHE000328"></a>

Para cambiar la orientación de un elemento en el control Organigrama, utilice el **Orientación90** Property del tipo [OrgElement](../WDLang1/1000019713.md). Por lo tanto, un elemento hijo puede mostrarse a la derecha de su padre. 

Por ejemplo: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Orga_Horizontal.gif)
 <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Orga_Horizontal2.gif)


El siguiente código se utiliza para cambiar la orientación del elemento en el que se ha hecho clic: 


```wl
// Select an element of ORG_NoName1
MyElement is OrgElement 
MyElement = ORG_NoName1[ORG_NoName1]
MyElement.Orientation90 = NOT MyElement.Orientation90
ORG_NoName1[ORG_NoName1] <- MyElement
```




## Propiedades específicas de los controles del Organigrama
<a name="propiedades_especificas_los_controles_del_organigrama_ELTTEXTE000345"></a>
Las siguientes propiedades se utilizan para gestionar un control Organigrama a través de la programación.



|   |   |
| --- | --- |
| [Count](../Proprietes/2510097.md) | La propiedad **Count** devuelve el número de elementos en un conjunto determinado. |
| [ElementHeight](../Proprietes/1000020068.md) | La propiedad **ElementHeight** permite:<br><br>- Descubra o modifique la altura de los elementos de un control Organigrama. <br><br>- Descubre o modifica la altura de las celdas de un control Cuadro de mando.<br><br><br> |
| [ElementOrientation](../Proprietes/1000020055.md) | La propiedad **ElementOrientation** permite obtener y cambiar la orientación de un control Organigrama. |
| [ElementWidth](../Proprietes/1000020066.md) | La propiedad **ElementWidth** permite:<br><br>- Descubra o modifique la anchura de los elementos de un control Organigrama.<br><br>- Descubre o modifica el ancho de las celdas de un control Cuadro de mando.<br><br><br> |
| [MaxLeafPerRow](../Proprietes/1000020103.md) | El **MaxLeafPerRow** Property se utiliza para get y cambiar el número máximo de elementos en un fila en un control Organigrama. |
| [Vertical](../Proprietes/2510133.md) | **Vertical** se utiliza para:<br><br>- averigua si un control está orientado horizontal o verticalmente.<br><br>- modificar la dirección de visualización de los controles de la mesa y del looper (cambiar de vertical a horizontal y viceversa). <br><br><br> |
| [Zoom](../Proprietes/1000017201.md) | La propiedad **Zoom** obtiene o establece el valor del zoom: <br><br>- en un control Imagen.<br><br>- en un control Mapa. <br><br>- en un control Organigrama.<br><br>- en un control Tabla, Tabla TreeView, Tabla dinámica, Hoja de cálculo o List Box. <br><br>- en un control Procesador de texto. <br><br>- en un control Lector PDF.<br><br>- en un control Cámara.<br><br>- en un control Editor de diagramas; <br><br>- en una ventana.<br><br><br> |





Para obtener una lista completa de las propiedades de WLanguage que pueden utilizarse con una control Organigrama, consulte [Propiedades associated con controles de organigrama](../WDChamp/1000019791.md).

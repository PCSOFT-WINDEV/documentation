
## Gestión de la entrada obligatoria en WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000095"></a>
Para evitar que los usuarios dejen un control vacío, puede definirlo como campo obligatorio. Con este mecanismo, los usuarios no podrán validar la página hasta que hayan introducido toda la información requerida.

**Observación**: Solo los controles Campo de entrada pueden definirse como campos obligatorios.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Entrada obligatoria
<a name="entrada_obligatoria_ELTTEXTE000119"></a>
Cuando se valida una página, se realiza una comprobación. Si no se introduce ningún valor en uno o varios controles obligatorios:

- un cuadro de diálogo indica los controles obligatorios.

- se cancela la validación de la página.

- el primer control vacío en el orden de tabulación obtiene el foco automáticamente.




Por ejemplo, la siguiente página contiene varios controles obligatorios:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Formulaire.gif)


El usuario valida la página sin introducir datos en algunos de los controles requeridos. La página no se valida y aparece el siguiente cuadro de diálogo:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=BoiteDialogueSaisieOblig.gif)


Una vez validado este cuadro de diálogo, el primer control vacío en el orden de tabulación ("Postal code" en este caso) obtiene automáticamente el foco:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Formulaire2.gif)

<a name="NOTE3_1"></a>


## Implementación
<a name="implementacion_ELTTEXTE000143"></a>
Para definir un control como obligatorio, simplemente seleccione la opción "Entrada obligatoria" (en la pestaña "Detalles" de la ventana de descripción del control Campo de entrada).
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=SaisieObligatoireOption.gif)

## Recordatorio: Modo de funcionamiento de las páginas dinámicas de WEBDEV
<a name="recordatorio_modo_funcionamiento_las_paginas_dinamicas_webdev_ELTTEXTE000163"></a>


### Sesión WEBDEV
<a name="sesion_webdev_ELTPARAGRAPHE000010"></a>

Cuando se visualiza una página WEBDEV dinámica, se realizan las siguientes operaciones: 

1. Solicitud de visualización de la página.

2. Arranca el motor de WEBDEV. El motor permanece en el servidor hasta que se cierra la aplicación.

3. Crear el contexto de la aplicación. El contexto permanece en el servidor hasta que se cierra la aplicación.

4. El motor WEBDEV ejecuta el código servidor y construye la página HTML (por ejemplo, a partir de los datos de la base de datos).

5. Una vez que la página HTML está completamente construida, el servidor transmite el resultado al cliente (el navegador). 







### Contextos de página
<a name="contextos_pagina_ELTPARAGRAPHE000022"></a>

Por cada página que se muestra en el navegador, se crea automáticamente un contexto de página en el servidor. Este contexto de página contiene todos los elementos necesarios para construir la página que ve el usuario final: 

- variables globales,

- variables locales, 

- procesos del servidor, 

- conexiones a la base de datos,

- los contextos de los archivos, etc.




Los contextos de la página permanecen en el servidor hasta el final de la sesión de WEBDEV. 

Cuando se llama a la misma página varias veces: 

- Si se llama a la función [PageDisplay](../WDLang2/3058008.md) para mostrar la página, el contexto de la página se destruye y se vuelve a crear. 

- Si se llama a la función [PageRefresh](../WDLang2/3058011.md) para mostrar la página, se vuelve a utilizar el mismo contexto de página.



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=PageAffiche_Ctx.gif)





### Programación
<a name="programacion_ELTPARAGRAPHE000049"></a>

De forma predeterminada, la sesión WEBDEV y los contextos de página se gestionan automáticamente. No hay nada por programar.

## Modo de funcionamiento de Active WEBDEV Pages
<a name="modo_funcionamiento_active_webdev_pages_ELTTEXTE000199"></a>


### ¿Qué es una página AWP?
<a name="¿que_una_pagina_awp_ELTPARAGRAPHE000059"></a>

Una página AWP (Active WEBDEV Page) es una página WEBDEV dinámica **sin un contexto persistente** en el servidor. El contexto de página AWP es temporal. Se crea en una sesión temporal. 

Recordatorio: En un sitio WEBDEV dinámico, cada página mostrada posee un contexto de página persistente durante toda la vida útil de la sesión en el servidor.. 


### Modo de funcionamiento de sitios AWP
<a name="modo_funcionamiento_sitios_awp_ELTPARAGRAPHE000066"></a>

Los siguientes elementos se crean automáticamente en el servidor cuando se muestra una página AWP en el navegador: 

- una sesión temporal,

- un contexto de página AWP temporal. 




La sesión temporal contiene todos los elementos que se han utilizado para construir la página visualizada por el usuario Web.: 

- variables locales, 

- procesos del servidor, 

- conexiones a la base de datos,

- los contextos de los archivos, etc.




Cuando se envía la página AWP al usuario, se destruyen estos elementos.

### ¿Cómo compartir información (valores) entre páginas AWP?
<a name="¿como_compartir_informacion_valores_entre_paginas_awp_ELTPARAGRAPHE000086"></a>

Existen dos métodos para compartir información (valores) entre páginas AWP:

- Pasar información en la URL. Este método permite optimizar el SEO.

- Guardar información en los contextos AWP (mediante programación).



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=PageAffiche_AWP2.gif)


**Pasar información (valores) entre dos páginas en la URL**
Se puede pasar información de una página a otra a través de la URL. La URL tiene el siguiente formato: "http:\\\\Servidor Web\\...\\mipagina.awp?NombreParam1=Valor1&NombreParam2=Valor2".
Este método optimiza el SEO de la página porque los robots de SEO pueden ver y analizar la información de la URL.

**Guardar información (valores) en los contextos AWP (mediante programación)**
Tiene la posibilidad de almacenar en el servidor valores comunes a varias páginas de Active WEBDEV, a través de los contextos de AWP. Un contexto AWP se crea en el disco del servidor. Este contexto permanece disponible mientras se muestren las páginas AWP, y siempre y cuando no se exceda el tiempo de espera de dichos contextos. El tiempo de espera de los contextos AWP se define en el administrador WEBDEV (pestaña "Configuración", opción "Duración de contextos AWP").
Para gestionar los contextos AWP, debe utilizar funciones WLanguage como [DeclareAWPContext](../WDLang2/3058028.md), [FreeAWPContext](../WDLang2/1000020429.md), etc, ...
Para obtener más información, consulte [AWP: Gestión de contextos](../WDChamp/3539060.md).

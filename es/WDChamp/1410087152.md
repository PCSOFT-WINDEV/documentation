## Procesos gestionados por Default
<a name="procesos_gestionados_por_default_ELTTEXTE000141"></a>
WINDEV gestiona los siguientes procesos por Default (en el orden en que aparecen en el editor de código):

| Event | Condición de ejecución |
| --- | --- |
| Inicialización | Corre cuando se abra la ventana. \* |
| Modificar selección | Ejecutar cuando se cambia la selección en el Editor HTML control. Este Event acepta parámetros. Este Event tiene el siguiente formato: <br><pre><code>PROCEDURE ModifySelection(HTMLselection is edtHTMLSelection)</code></pre><br> donde &lt;HTMLselection&gt; es una [edtHTMLSelection](../WDLang1/1000026093.md) Variable que corresponde a los elementos seleccionados. |
| Cargar los recursos | Permite que se le notifique e impide que se carguen scripts o imágenes de la aplicación (con fines de seguridad, por ejemplo). <br><br>Este Event acepta parámetros. Este Event tiene el siguiente formato: <br><pre><code>PROCEDURE LoadResources(sURL is string, nType is int)</code></pre><br> donde: <br><br>- &lt;sURL&gt; es una cadena de caracteres que corresponde al URL de la Resource a cargar.<br><br>- nType es una constante utilizada para identificar el tipo de Resource que se va a descargar: <br><br>	- *typehtmlImageRes*: la Resource a descargar es una Image. <br><br>	- *typehtmlScriptRes*: la Resource para descargar es un guión. <br><br>	- *typehtmlCssRes*: la Resource a descargar es un estilo CSS.  <br><br>	- *typehtmlXMLHttpRequestRes*: la Resource a descargar es una Document XML generada como respuesta a una petición HTTP. <br><br>	- *typehtmlOther*: la Resource a descargar es de un tipo desconocido. <br><br><br><br><br><br><br><br>Si este evento devuelve <u><u><u><u>False</u></u></u></u>, el Resource no se cargará. |
| Siempre que se modifique | Se ejecuta cada vez que se modifica el editor HTML control. |


(\*) Por defecto, la "Inicialización" Event de cada control se ejecuta según el orden en que se crearon las control en la ventana. Para modificar este orden de ejecución: 

1. En la pestaña "Ventana", en el grupo "Orden", haga clic en "Inicialización".

2. En la ventana mostrada, utilice las flechas de la derecha para modificar el orden en el que se inicializan los controles.

## Eventos opcionales
<a name="eventos_opcionales_ELTTEXTE000165"></a>
Se pueden utilizar varios eventos opcionales.
Para añadir un evento opcional:

1. Seleccione la control deseada.

2. Mostrar la ventana del código de esta control (tecla F2).

3. Haga clic en el enlace "Añadir otras Event s a xxx" en la parte inferior del código de la ventana, después de la última Event.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Traitements_optionnels_WD_OK%20-%20HC%20N%B0001.gif)


4. Todos los eventos opcionales disponibles para el control se muestran. 

5. Marque la opción Event para agregar y validar. 
	Observación: Puede seleccionar varios eventos opcionales. 

6. El Event opcional seleccionado se agrega automáticamente a los Event s administrados por el control.




Para desactivar un Event opcional, simplemente realice las mismas operaciones para mostrar la lista de Event s opcionales. A continuación, simplemente desmarque los eventos opcionales para eliminar. 
Observación: Si el código desactivado contiene código WLanguage, este código se elimina automáticamente.Se puede gestionar, por ejemplo:

- el desplazamiento del cursor por encima del control,

- botón izquierdo del ratón abajo, arriba, con un doble clic,

- el botón derecho del ratón presionado, liberado o con doble clic 

- la rueda del ratón, etc.




Para más detalles, consulte la página web de [Eventos opcionales](../WDChamp/1014004.md).

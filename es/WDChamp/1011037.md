## Presentación
<a name="presentacion_ELTTEXTE000328"></a>
Los siguientes eventos son associated a los informes (orden de aparición en el editor de código):

- [Apertura de la reporte](#NOTE2_1).

- [Lectura de datos](#NOTE3_1).

- [Filtrado (teniendo en cuenta el Record?)](#NOTE4_1).

- [Cierre de la reporte](#NOTE5_1).

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [Antes de imprimir el reporte](#NOTE6_1).




Varios eventos son associated con los informes. Estos eventos se ejecutan cuando se imprime la reporte.

Para conocer el orden de ejecución de estos eventos, véase [Secuencia de eventos associated con informes](../WDChamp/1011004.md).

## "Apertura" Event de la reporte
<a name="apertura_event_reporte_ELTTEXTE000352"></a>


### Ejecución de la Event
<a name="ejecucion_event_ELTPARAGRAPHE000045"></a>

El evento "Apertura" del reporte se ejecuta al iniciar la impresión del reporte.

**Recordatorio**: Al iniciar una reporte (incluso durante una prueba), los eventos se ejecutan en el siguiente orden:

- "Abrir" el reporte.

- ["Inicialización" de los bloques de reporte](../WDChamp/1011042.md).

- ["Inicialización" de los controles](../WDChamp/1013047.md).



<a name="NOTE2_2"></a>


### Contraseña del archivo de datos utilizada como fuente para el reporte
<a name="contrasena_del_archivo_datos_utilizada_como_fuente_para_reporte_ELTPARAGRAPHE000062"></a>

Si el archivo de datos de origen de reporte está protegido por una contraseña, deberá introducirla antes de abrir el archivo de datos (con [HPass](../WDLang4/3044108.md), por ejemplo).

Si no se abrió el el archivo de datos antes de imprimir el reporte, se puede especificar la contraseña en el Event de "Apertura" del reporte.
<a name="NOTE2_3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 

### Inicialización de variables
<a name="inicializacion_variables_ELTPARAGRAPHE000076"></a>

Cuando se imprime un reporte desde el visor de reporte, el reporte se vuelve a ejecutar automáticamente cuando:

- el usuario cambia de impresora.

- el usuario selecciona una de las opciones de exportación de la cinta (Imprimir en modo PDF, HTML, en un Email, etc.).


En este caso, los datos impresos en el reporte pueden ser diferentes de los datos mostrados en el visor del reporte.

Las variables utilizadas por el reporte deben ser inicializadas en el Event "Opening" en vez de en el código del control Botón que inicia la impresión.
<a name="NOTE2_4"></a>


### Ejemplos de uso
<a name="ejemplos_uso_ELTPARAGRAPHE000090"></a>

Este Event está acostumbrado:

- declarar las variables globales de la reporte.

- cambiar los límites de reporte con las propiedades [MinValue](../Proprietes/2510008.md) y [MaxValue](../Proprietes/2510009.md).

- inicializar las contraseñas de los archivos de datos si es necesario.

- configurar el destino de la impresión ([iDestination](../WDLang5/3046074.md)).

- pasar parámetros a la consulta relacionado a la reporte ([iInitReportQuery](../WDLang5/3046021.md)).

- recuperar los parámetros pasados a la reporte.

- cambiar la fuente de datos de reporte.




**Observación**: [iPrint](../WDLang5/3046031.md) no debe utilizarse en el reporte "Apertura" Event.

## Event "Lectura de datos"
<a name="event_lectura_datos_ELTTEXTE000394"></a>


### Ejecución de la Event
<a name="ejecucion_event_ELTPARAGRAPHE000124"></a>

Este evento se ejecuta cuando se navega por las Record s para imprimir (en el bucle de impresión), una vez que la Record ha sido leída.
<a name="NOTE3_2"></a>


### Informes basados en un archivo de texto
<a name="informes_basados_archivo_texto_ELTPARAGRAPHE000131"></a>

Para los informes basados en un archivo de texto (con lectura programada), la Event "Lectura de datos" se utiliza para leer los registros en el archivo de texto con el fin de imprimirlos.
<a name="NOTE3_3"></a>


### Ejemplos de uso
<a name="ejemplos_uso_ELTPARAGRAPHE000138"></a>

Este Event está acostumbrado:

- realizar sustituciones de bloques (como se conoce al próximo Record a imprimir).

- seleccionar registros.

- realizar cálculos específicos.




**Observación**: Este Event devuelve <u><u><u><u>True</u></u></u></u> si todavía hay registros para imprimir y <u><u><u><u>False</u></u></u></u> si no hay más registros para imprimir.

Si no quieres imprimir un Record, usa el evento "Filtrando (¿teniendo en cuenta el Record?)".

## Evento "Filtrado (¿teniendo en cuenta el Record?)"
<a name="evento_filtrado_¿teniendo_cuenta_record_ELTTEXTE000430"></a>


### Ejecución de la Event
<a name="ejecucion_event_ELTPARAGRAPHE000163"></a>

Este Event está dirigido:

- cuando se inicializa la fuente de datos (informes basados en un archivo de datos, vista o área de memoria).

- en el bucle de impresión, una vez que se lee el Record (informes basados en una consulta o en un control Tabla).




Este Event filtra los registros que se muestran en el reporte.
<a name="NOTE4_2"></a>


### Usando el filtro Event
<a name="usando_filtro_event_ELTPARAGRAPHE000175"></a>

Si hay líneas de código en esta Event, deben devolver un valor:

- <u><u><u><u>Cancelar</u></u></u></u> para detener la lectura de registros (y detener la impresión).

- <u><u><u><u>True</u></u></u></u> para indicar que el registro coincide con el filtro y que debe imprimirse.

- <u><u><u><u>False</u></u></u></u> para indicar que el registro no coincide con el filtro y que no debe imprimirse.



<a name="NOTE4_3"></a>


### Ejemplos de uso
<a name="ejemplos_uso_ELTPARAGRAPHE000195"></a>

Este Event está acostumbrado:

- seleccionar registros.

- realizar cálculos avanzados (subtotal, ...) para los informes basados en archivos de datos, vistas o zonas de memoria.

## "Cerrando" Event de la reporte
<a name="cerrando_event_reporte_ELTTEXTE000466"></a>


### Ejecución de la Event
<a name="ejecucion_event_ELTPARAGRAPHE000207"></a>

Este Event se ejecuta una vez que se ha impreso el reporte, justo antes de cerrar el reporte.

Esta Event no debe contener [funciones de impresión](../WDLang5/3046065.md).

Observación: [iCurrentFile](../WDLang5/1000021859.md) sirve para conocer el nombre del archivo generado para la reporte (para las exportaciones en PDF, HTML, ...).
<a name="NOTE5_2"></a>


### Ejemplos de uso
<a name="ejemplos_uso_ELTPARAGRAPHE000226"></a>

Este Event está acostumbrado:

- conocer el estado de la impresión actual ([iReportPrintingStatus](../WDLang5/3046043.md)).

- abra un cuadro de diálogo.

## Event "Antes de volver a imprimir el reporte"
<a name="event_antes_volver_imprimir_reporte_ELTTEXTE000496"></a>


### Ejecución de la Event
<a name="ejecucion_event_ELTPARAGRAPHE000244"></a>

Este Event se ejecuta cuando se imprime desde el visor de reporte (impresión directa o exportación a PDF, XLS, RTF, etc.). 

Este Event se ejecuta después de la "Apertura" Event del reporte. 

Si este Event regresa:

- <u><u><u><u>True STRIMEULB</u></u></u></u>, la reimpresión se realizará. 

- <u><u><u><u>False</u></u></u></u>, se cancelará la impresión.




Observaciones: 

- El [iDestination](../WDLang5/3046074.md) utilizado en este Event permite conocer el destino del reporte (HTML, PDF, etc.). 

- El [iCurrentFile](../WDLang5/1000021859.md) utilizado en este Event permite conocer el nombre del archivo generado para el reporte (para las exportaciones en PDF, HTML, etc.).



<a name="NOTE6_2"></a>


### Ejemplos de uso
<a name="ejemplos_uso_ELTPARAGRAPHE000275"></a>

Este Event está acostumbrado:

- averiguar la calidad del PDF generado antes de la exportación en formato PDF. 

- añadir una Watermark. 

- pedir una contraseña antes de la impresión o la exportación.

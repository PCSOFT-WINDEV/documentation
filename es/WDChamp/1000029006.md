## ¿Cómo crear la control Tabla dinámica (en una ventana)?
<a name="¿como_crear_control_tabla_dinamica_una_ventana_ELTTEXTE000168"></a>
Para crear una control Tabla dinámica en una ventana: 

1. En la pestaña "Creación", en el grupo "Datos", despliegue "Tabla y List Box" y seleccione "Tabla dinámica". 

2. Haga clic en la ubicación en la que desea crear el control. El asistente para crear el control se abre. Esta asistente se utiliza para definir los principales elementos del pivote tabla. 

3. Al final del asistente, se crea automáticamente el control Tabla dinámica con el control Botón que se utiliza para llenarlo en tiempo de ejecución. 




Observaciones: 

- La control Tabla dinámica funciona: 

	- en las bases de datos HFSQL,

	- en otras bases de datos mediante arrays de estructuras.  




- El contenido de la control Tabla dinámica se puede imprimir a través de su menú emergente. Para obtener más información, consulte [Imprimir el contenido de una control Tabla dinámica (AAF)](../WDChamp/1000029009.md). 

- Puede crear una reporte basada en una control Tabla dinámica. Para obtener más información, consulte [reporte Pivote de la mesa](../WDChamp/1011092.md).

## Asistente para crear una control Tabla dinámica
<a name="asistente_para_crear_una_control_tabla_dinamica_ELTTEXTE000192"></a>
La creación del control Pivot tabla asistente se utiliza para definir los principales elementos del tabla. 

1. En primer lugar, el asistente se utiliza para definir los valores del **que aparecen** en las celdas del pivote tabla. Una celda que se encuentra en un pivote tabla puede mostrar, por ejemplo, el total de ventas, la suma de las cantidades, el número de pedidos, ...
	Por lo tanto, debe definir para cada valor visualizado en la celda: 

	- los **archivo de origen**. Este archivo corresponde al archivo que contiene el valor utilizado para el cálculo. Por ejemplo, el archivo OrdLine para calcular las ventas realizadas. 

	- los **cálculo realizado**. Se dispone de varios cálculos: suma, media, cuenta, ....

	- los **fuente para el cálculo**lo que significa que el elemento del fichero fuente a utilizar. Por ejemplo, para calcular las ventas realizadas, la suma se realizará sobre el precio de venta (Total_Fila item del fichero OrdLine).. 
			El botón "Añadir un valor adicional" sirve para definir un nuevo valor que se mostrará en la celda.. 
			**Observaciones**: 

		- La control Tabla dinámica se puede basar en una consulta. En este caso, el fichero fuente corresponde a la consulta. 

		- La control Tabla dinámica puede basarse en una Array de Estructuras Variable. En este caso, el archivo fuente corresponde al archivo Variable.




2. El siguiente paso consiste en definir la cabecera de la columna **-** de la ventana de diálogo control Tabla dinámica. La cabecera de la columna viene directamente de sus archivos de datos. 
	Se pueden encontrar una o más cabeceras de columna. 
	Para cada cabecera de columna, la cabecera permite definir: 

	- el enlace utilizado para acceder a la posición visualizada en la cabecera. Este enlace se realiza desde el archivo de datos utilizado para la fuente de cálculo. 

	- si la cabecera de la columna corresponde a un elemento Fecha, la asistente propone el modo de visualización de la fecha: Año, Semestre, Trimestre, Mes, Quincena, Semana, Día. Se realizará una nueva agrupación para cada modo de visualización seleccionado.. Por ejemplo, si sólo se muestran los años, los trimestres y los meses, la cabecera de la columna tendrá el siguiente aspecto:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=TCD_Entete_Date.gif)
**Observaciones**: 

		- Si la control Tabla dinámica se basa en una consulta, los datos visualizados en la cabecera de columna proceden directamente de la consulta. 

		- Si la control Tabla dinámica se basa en una Array de Estructuras Variable, los datos mostrados en la cabecera de la columna provienen de los miembros de la estructura. 




3. La siguiente etapa consiste en definir **el encabezado de las filas** del control Tabla dinámica. La cabecera fila proviene de sus ficheros de datos. 
	Se pueden encontrar una o más cabeceras fila. 
	Para cada cabecera fila, el asistente permite definir: 

	- el enlace utilizado para acceder a la posición visualizada en la cabecera. Este enlace se realiza desde el archivo de datos utilizado para la fuente de cálculo. 

	- si la cabecera fila corresponde a un elemento Fecha, el asistente propone el modo de visualización de la fecha: Año, Semestre, Trimestre, Mes, Quincena, Semana, Día. Se realizará una nueva agrupación para cada modo de visualización seleccionado..
			**Observaciones**: 

		- Si control Tabla dinámica se basa en una consulta, los datos visualizados en la cabecera de fila proceden directamente de la consulta. 

		- Si la control Tabla dinámica se basa en una variable Array of Structures, los datos visualizados en la cabecera fila proceden de los miembros de la estructura. 




4. Dar un nombre a la control Tabla dinámica y validar. 
	Observación: Las dimensiones del control que se creó se optimizan para ocupar el espacio disponible en la ubicación indicada. Si el tamaño del control no es el indicado, pulse Ctrl+Z: el control volverá al tamaño predeterminado.




La control Tabla dinámica se crea en la ventana con un "Calcular" control Botón. Esta control Botón se utiliza para iniciar el cálculo del control Tabla dinámica. 

**Atención**: Si la fuente de la control Tabla dinámica es: 

- una consulta, la consulta debe ejecutarse antes de calcular el control Tabla dinámica.

- un array de estructuras, el array de estructuras debe ser rellenado antes de calcular el control Tabla dinámica.




Para modificar las características del control Pivot tabla, visualice el control description ("description" en el menú contextual). Para obtener más información, consulte [description de un control Tabla dinámica](../WDChamp/1000029007.md).

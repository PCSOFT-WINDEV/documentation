
## Asociar un Image con los elementos de un control TreeView
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Imagen associated con un elemento
<a name="imagen_associated_con_elemento_ELTTEXTE000129"></a>
Cada elemento que se encuentra en un ÁrbolVew control está precedido por un Image. Esta Image puede diferir según el estado del elemento (o nivel) actual. Puedes mostrar un Image:

- cuando se desglosa el nivel,

- cuando el nivel está colapsado (o cuando el nivel corresponde a una hoja).




Ejemplo de control TreeView en una aplicación WINDEV:: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Champ_Arbre_1.gif)


Imagen associated con el nivel colapsado: por ejemplo: ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ImageEnroule.gif)

Imagen associated con el nivel ampliado: por ejemplo: ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ImageDeroulee.gif)

**Observación**: También tiene la capacidad de asociar un identificador con un elemento. Para obtener más información, consulte [Asociar un identificador a un elemento](../WDChamp/1013033.md).

## Asociar una o más imágenes a un elemento
<a name="asociar_una_mas_imagenes_elemento_ELTTEXTE000153"></a>
Para asociar uno o varios Image s a un elemento de un control TreeView, basta con especificar el nombre (y el ruta si es necesario) del Image en las siguientes funciones de programación:



|   |   |
| --- | --- |
| [TreeAdd](../WDLang1/3018012.md) | Agrega un elemento "hoja" (y todos los nodos necesarios) en un control TreeView. |
| [TreeInsert](../WDLang1/3018023.md) | Inserta una hoja en una control TreeView en una posición específica. |
| [TreeModify](../WDLang1/3018013.md) | Modifica un elemento (nodo u hoja) en un control TreeView. |





Estas funciones también están disponibles en la sintaxis del prefijo: 



|   |   |
| --- | --- |
| [&lt;Insertar](../WDLang1/1000023761.md) | Inserta una hoja en una control TreeView en una posición específica. |
| [&lt;Modificar](../WDLang1/1000023763.md) | Modifica un elemento (nodo u hoja) en un control TreeView. |
| [&lt;Vista de árbol&gt;.add](../WDLang1/1000023422.md) | Agrega un elemento "hoja" (y todos los nodos necesarios) en un control TreeView. |





Observación: Si no hay una imagen específica associated con los elementos de una control TreeView, se utiliza la "carpeta cerrada" Icon (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ImageEnroule.GIF)) y la "carpeta abierta" Icon (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ImageDeroulee.GIF)).

## Formato de las imágenes
<a name="formato_las_imagenes_ELTTEXTE000217"></a>
Los siguientes formatos de Image están soportados en Windows:

- Mapa de bits (\*.BMP)

- Formato de intercambio de gráficos (\*.gif)

- Grupo Mixto de Expertos en Fotografía (\*.jpg \*;.jpeg)

- CD de fotos Kodak (\*.pcd)

- Pincel de pintura (\*.pcx)

- Formato Adobe Photoshop (\*.psd)

- TrueVision TARGA (\*.tga)

- Etiquetado en el formato de archivo Image (\*.tif ; \*.Tiff)
	Observación: la opción "tiff de mosaico" del estándar tiff 6.0 no está soportada. En este caso, le aconsejamos que guarde el Image en el estándar Tiff 5.0. 

- Gráficos de red portátiles (\*.png)

- MetaFicheros de Windows (\*.emf;\*.wmf)

- Iconos (\*.ico \*;.icw)
	Observaciones: 

	- Pantalla HQ (High Quality) disponible.

	- Manejo de hojas en formato Transparent BMP (32 bits).




- Cursores (\*.cur)

- Formato portátil de Document (\*.pdf)

- Gráficos vectoriales escalables (\*.svg)

- WINDEV Image s (generado por el editor de Image, \*.WDPic).








Las imágenes utilizadas pueden ser:

- Archivos de Image.

- elementos de un archivo de datos HFSQL "Binary Memo" que contiene un archivo Image.

- dibujos creados por la [funciones para dibujar en los list boxes y combo boxes](../WDLang1/3029028.md).






Observaciones:

- La Image especificada se reduce proporcionalmente: la altura de la Image corresponde a la altura de la fila en la control TreeView. El tamaño de las imágenes utilizadas debe ser set a 16x16 píxeles (tamaño de los pequeños iconos WINDEV).

- El color magenta que se encuentra en las imágenes se considera como Transparent en tiempo de ejecución.

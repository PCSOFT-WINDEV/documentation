


## Creación y distribución de las páginas de Active WEBDEV
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Transformación de una página en modo sesión en una página Active WEBDEV
<a name="transformacion_una_pagina_modo_sesion_una_pagina_active_webdev_ELTTEXTE000153"></a>
Para transformar una página en modo Sesión en una Página Active WEBDEV: 

1. Abra la description de la página actual ("description" en el menú emergente de la página).

2. En la pestaña "General", en "Tipo de página", seleccione "AWP". 

3. Valide.




**¿Dónde se guardará la Página WEBDEV de Active?**
Una vez transformada una página dinámica en una Página Active WEBDEV, la página se generará en formato AWP en el directorio &lt;Nombre del proyecto&gt;_WEB del proyecto (en cada subdirectorio correspondiente a los idiomas soportados por el proyecto).

## Consejos de programación
<a name="consejos_programacion_ELTTEXTE000177"></a>


- El código del servidor debe encontrarse en los eventos de inicialización (inicialización de la página o de los controles).

- Los contextos deben ser gestionados a través de la programación. La página debe estar posicionada en la base de datos (según los parámetros recibidos en la URL, por ejemplo).. Para obtener más información, consulte [Visualización de una página de Active WEBDEV desde otra página](../WDChamp/3539047.md).

## Práctico
<a name="practico_ELTTEXTE000201"></a>


- **Refrescar una página de Active WEBDEV**: Al actualizar una página de Active WEBDEV se activan todos los eventos WLanguage de la página. Por ejemplo, si la página de Active WEBDEV permitió una adición, al refrescar la página se hará de nuevo esa adición.

- **Eventos de idiomas**: El orden de ejecución de los eventos de WLanguage al hacer clic en una control Botón, por ejemplo, es el siguiente:

	- "Iniciando" Event del proyecto,

	- "Declaraciones globales" Event de la página,

	- "Iniciando" Event de los controles de la página,

	- "Iniciando" Event de la página,

	- "Click" Event de la control Botón clickada
			Cada vez que se realiza una acción en la página de Active WEBDEV, la página HTML correspondiente se reconstruye por completo en el servidor.




- **acoplamientos**: Puede crear enlaces mediante programación (utilice los controles ***URL*** Property on Link).

- **Enlaces y botones**: Usted debe preferir los enlaces y botones sin"enviar".. Se genera un enlace "&lt;a href&gt;" en la página.

## Limitaciones relativas al contenido y a la programación de las páginas de Active WEBDEV
<a name="limitaciones_relativas_contenido_programacion_las_paginas_active_webdev_ELTTEXTE000225"></a>


- **Primera página de un proyecto**: Una página de Active WEBDEV no puede ser la primera página del proyecto en modo Sesión. Una página de Active WEBDEV puede ser la página de inicio de un proyecto.

- **Funciones de seguimiento de WLanguage**: Solo está disponible la salida en un archivo (constante *trFile*).

- **Cuadros de diálogo**: Los cuadros de diálogo mostrados por las funciones ***Confirm***, ***Error*** e ***Info*** se ignoran si se muestra una página en el mismo proceso con la función ***PageDisplay***.

- **Elementos globales utilizados en el proyecto**: Cuando se modifica un elemento global del proyecto (global Procedure, query, class, ...), se debe recompilar el proyecto para tener en cuenta estas modificaciones en las correspondientes páginas autónomas de Active WEBDEV.

- **Bípers**: Los localizadores generados por el RAD no pueden utilizarse en las páginas de Active WEBDEV. En efecto, estos controles utilizan el contexto del proyecto (que no existe en las páginas de Active WEBDEV). Estos controles deben ser sustituidos por un localizador programado. Cada enlace del buscapersonas debe mostrar una página con datos específicos (página basada en una consulta con parámetros, por ejemplo).. 
	Todos los registros también se pueden mostrar en una celda con barra de desplazamiento.
	El localizador en modo AWP gestiona automáticamente el método de paso de parámetros, sin programación. Este tipo de paginador te permite reference todas las páginas de relacionado y evita tener que usar cookies para almacenar la búsqueda actual. 

- **control Tabla**: Los controles Tabla están limitados (las tablas basadas en un archivo de datos con columnas de enlace ya no funcionan, por ejemplo). Pueden ser sustituidos por un looper (ya que los enlaces se construyen dinámicamente con el ***URL*** Property).

- **control Imagen**: Ninguna imagen generada puede ser utilizada en las páginas de Active WEBDEV.

- **Elemento en memoria (list box, combo box, looper):** El contenido de estos elementos es volátil. El contenido de estos elementos debe rellenarse sistemáticamente en el código de inicialización de la página.

- **El valor de los controles de sólo lectura** se pierde y debe recalcularse cada vez que se muestra la página.

- **Tamaño de Active WEBDEV Páginas**: Con proyectos grandes (que contengan varias clases, procedimientos globales o consultas), el tamaño de las Páginas Active WEBDEV generadas será bastante importante.

- **Variables globales del servidor**: Las variables globales del servidor del proyecto recuperan su valor de Default cada vez que se accede a la página en el caso de las páginas autónomas de Active WEBDEV. Sin embargo, puede utilizar [DeclareAWPContext](../WDLang2/3058028.md) o los controles ocultos associated con el valor deseado.

- **Componentes WINDEV o WEBDEV**: No se encuentran los componentes manejados por una página de Active WEBDEV. Deben ser movidos al directorio &lt;ProjectName&gt;_WEB.

- **Ubicación de los archivos de datos**: La ubicación de los archivos de datos debe realizarse en el código de inicialización del proyecto, a través de la siguiente Line de código por ejemplo:
	
	```wl
	HSubstDir("?", fDataDir())
	```


- **URL relativa**: No se deben utilizar las URLs relativas creadas mediante programación en la página de inicio del proyecto.

## Visualización de una página de Active WEBDEV
<a name="visualizacion_una_pagina_active_webdev_ELTTEXTE000249"></a>
Una Página Active WEBDEV puede visualizarse desde una página estática o desde una página dinámica de un sitio WEBDEV.

Se puede visualizar una página de Active WEBDEV:

- desde la interfaz WEBDEV: simplemente seleccione la página de Active WEBDEV que se mostrará al describir el enlace.

- mediante programación.




Para obtener más información, consulte [Visualización de una página de Active WEBDEV desde otra página](../WDChamp/3539047.md).

## Instalación de Active WEBDEV Pages en un servidor
<a name="instalacion_active_webdev_pages_servidor_ELTTEXTE000273"></a>
Para instalar Active WEBDEV Pages en un servidor, puede: 

- utilizar la instalación estándar de un sitio en modo Sesión. Los archivos AWP se clasifican en la categoría "Archivos HTML estáticos".

- utilizar **WDDeploy** o cualquier otro cliente FTP para instalar el archivo directamente en el sitio deseado.

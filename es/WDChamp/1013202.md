


## Filtros en los controles de la Tabla basados en un archivo de datos
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000195"></a>
WINDEV, WEBDEV y WINDEV Mobile proponen varios métodos para filtrar los registros mostrados en un control Tabla en función de un archivo de datos:

- Filtro en la clave de búsqueda, definido en la ventana de description de la control Tabla.

- propiedad [filtros](../Proprietes/2510042.md).

- función [HFilter](../WDLang4/3044100.md).

- Event opcional para filtrar los registros de un control Tabla. Para obtener más información, consulte [Eventos associated con controles de mesa](../WDChamp/1013189.md).

<a name="NOTE2_1"></a>


## Descripción de la control Tabla
<a name="descripcion_control_tabla_ELTTEXTE000219"></a><br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Champ_Table_contenu.gif)
La opción "Mostrar registros cuyo campo de navegación comienza por" permite establecer un filtro en un control Tabla basado en un archivo de datos.

El filtro tiene las siguientes características:

- "Filtro"Comienza con. La condición se introduce directamente en el editor.

- El filtro sólo se aplica a los controles de la tabla basados en un archivo de datos.

- El filtro no afecta a la navegación de el archivo de datos ([HReadFirst](../WDLang4/3044051.md), etc.)

### La opción "Usar el HFilter para navegar por el archivo" no está marcada.
<a name="opcion_usar_hfilter_para_navegar_por_archivo_esta_marcada_ELTPARAGRAPHE000064"></a>

Si la opción "Use HFilter to browse the file" está desmarcada, el filtro tiene las siguientes características:

- "Filtro"Comienza con. La condición se introduce directamente en el editor.

- El filtro sólo se aplica a los controles de la tabla basados en un archivo de datos. 

- El filtro no afecta a la navegación de el archivo de datos ([HReadFirst](../WDLang4/3044051.md), etc.)

- Sólo se puede ordenar la columna associated con el elemento de búsqueda (opción "Ordenar" en la pestaña "General").

- El usuario no puede realizar ninguna búsqueda en el control Tabla (opción"Buscable" en la pestaña"General").

### La opción "Use HFilter to browse the file" marcada
<a name="opcion_use_hfilter_browse_the_file_marcada_ELTPARAGRAPHE000086"></a>

Si la opción "Use HFilter to browse the file" está marcada, el filtro tiene las siguientes características:

- "Filtro"Comienza con. La condición se introduce directamente en el editor.

- El filtro sólo se aplica a los controles de la tabla basados en un archivo de datos.

- El filtro no afecta a la navegación de el archivo de datos ([HReadFirst](../WDLang4/3044051.md), etc.)

- El usuario podrá ordenar la información mostrada en las columnas vinculadas a los elementos clave (opción "Clasificable" en la pestaña "General").

- El usuario podrá realizar búsquedas en las columnas vinculadas a los elementos clave (opción "Buscar" en la pestaña "General").




**Caso especial**: Antes de abrir la ventana que contiene el control Tabla, se realizó un filtro set sobre el archivo base del control a través de [HFilter](../WDLang4/3044100.md). El control Tabla filtra automáticamente los registros visualizados (filtro fijado directamente en el editor o con el filtro Property). Cuando se cierre la ventana, no se activará ningún filtro en el archivo de datos.

**Atención**: Comprobar el filtro Active al recorrer los archivos de datos. Como no se pueden activar dos filtros al mismo tiempo, sólo se mantiene el último filtro creado por [HFilter](../WDLang4/3044100.md).

## Filtro Property
<a name="filtro_property_ELTTEXTE000255"></a>
El [filtros](../Proprietes/2510042.md) Property puede utilizarse para set dinámicamente las características del filtro aplicado al control Tabla.
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) El filtro aplicado utiliza la opción "Usar HFilter para examinar el archivo" definida en el editor.

## HFilter
<a name="hfilter_ELTTEXTE000279"></a>
[HFilter](../WDLang4/3044100.md) se utiliza para filtrar el archivo base de control Tabla. [HFilter](../WDLang4/3044100.md) permite utilizar todo tipo de filtros (incluidos los filtros "Empieza por").

Si no se utiliza [HFilter](../WDLang4/3044100.md) en el código de inicialización del control Tabla, es necesario volver a mostrar el contenido del control Tabla utilizando [TableDisplay](../WDLang1/3074003.md).

**Observación**: Los controles de tabla basados en un archivo de datos que se rellenan automáticamente (por WINDEV, WEBDEV o WINDEV Mobile) no se rellenan todavía cuando se ejecuta el código de inicialización. Los registros pueden filtrarse por [HFilter](../WDLang4/3044100.md).

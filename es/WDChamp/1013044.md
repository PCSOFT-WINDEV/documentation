## Vista general del "reporte interno" control
<a name="vista_general_del_reporte_interno_control_ELTTEXTE000264"></a>


### control de una reporte
<a name="control_una_reporte_ELTPARAGRAPHE000011"></a>

Los controles "Internal reporte" permiten imprimir varios informes (informes secundarios) en un único reporte (reporte principal) sin necesidad de programación.

Por ejemplo, el siguiente reporte principal se compone de dos informes secundarios impresos uno al lado del otro:

- un reporte que muestra los datos en un tabla.

- a reporte que muestra los datos en un gráfico.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatInterne.gif&type=thumb)





<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Beneficios de los informes internos en comparación con los informes anidados
<a name="beneficios_los_informes_internos_comparacion_con_los_informes_anidados_ELTTEXTE000288"></a>
El [Informes anidados de **:**](../WDChamp/1011001.md) también puede utilizarse para imprimir varios informes secundarios en un reporte principal. Los informes secundarios se imprimen uno debajo del otro.
Para imprimir informes secundarios, utilice [iPrintReport](../WDLang5/3046032.md) en el código del bloque en el que deben imprimirse los informes secundarios.

Los informes **internal** se utilizan para imprimir varios informes **sin programar**:

- uno debajo de otro.

- uno al lado del otro.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000312"></a>


### Creación de una reporte principal compuesta por varios informes secundarios
<a name="creacion_una_reporte_principal_compuesta_por_varios_informes_secundarios_ELTPARAGRAPHE000061"></a>

Para crear una reporte principal compuesta por varios informes secundarios: 

1. Cree y grabe todos los informes secundarios.

2. Cree la reporte principal: 

	- Haga clic en ![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ico_nouveau.gif) en los botones de acceso rápido. 

	- Aparece la ventana de creación: haga clic en "Reporte", y luego en "Reporte". El asistente de creación de reportes se abre.

	- En la asistente, seleccione "Blanco reporte" y valide.




3. En la reporte principal, crear tantos controles "Internal reporte" como el número de informes secundarios a imprimir.. 
	Para crear una reporte interna, vaya a la pestaña "Creación", grupo "Estructura" y haga clic en "Reporte interno"..
	**Observación**: La reporte principal puede contener todos los demás tipos de controles.

4. Asocie cada control "reporte interno" con un reporte secundario ("reporte impreso" en la pestaña "General" del control description). El reporte secundario se imprimirá completamente en el control "Reporte interno".

5. Coloca los controles de "Reporte interno" en el reporte principal.

6. Pruebe el reporte principal (![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ICO_GO_fenetre_WD_bl.GIF) en los botones de acceso rápido) para comprobar la impresión de su reporte.



<a name="NOTE3_2"></a>


### reporte principal no está en blanco
<a name="reporte_principal_esta_blanco_ELTPARAGRAPHE000099"></a>

Si el reporte principal y los informes secundarios tienen la misma fuente de datos, le recomendamos que utilice un contexto HFSQL independiente:

- para la reporte principal.

- para los informes secundarios.




**Observación**: Para definir un contexto HFSQL independiente:

1. Abra la descripción del reporte (seleccione "Descripción del reporte" en el menú contextual).

2. En la pestaña "Datos", marque "La ejecución del reporte no afecta las operaciones de navegación (contexto HFSQL independiente)".



<a name="NOTE3_3"></a>


### reporte secundaria
<a name="reporte_secundaria_ELTPARAGRAPHE000120"></a>

Un reporte secundario puede contener todos los tipos de controles excepto los controles "Internal reporte".
<a name="NOTE3_4"></a>


### reporte secundaria basada en una consulta con parámetros
<a name="reporte_secundaria_basada_una_consulta_con_parametros_ELTPARAGRAPHE000127"></a>

Si la reporte secundaria se basa en una consulta con parámetros, utilice [iInitReportQuery](../WDLang5/3046021.md) para pasar los parámetros a la consulta. Esta función debe utilizarse en el "Antes de imprimir" Event del bloque que contiene el "reporte interno" control.
<a name="NOTE3_5"></a>


### reporte secundaria basada en una consulta que utiliza una conexión específica
<a name="reporte_secundaria_basada_una_consulta_que_utiliza_una_conexion_especifica_ELTPARAGRAPHE000137"></a>

Si la reporte secundaria se basa en una consulta que gestiona una conexión específica, utilice [iInitReportQueryConnection](../WDLang5/3046028.md) para ejecutar esta consulta (y para pasar parámetros si es necesario). Esta función debe utilizarse en el "Antes de imprimir" Event del bloque que contiene el "reporte interno" control.
<a name="NOTE3_6"></a>


### reporte secundario basado en una vista HFSQL
<a name="reporte_secundario_basado_una_vista_hfsql_ELTPARAGRAPHE000147"></a>

Si la consulta secundaria se basa en una vista HFSQL:

1. Declare un Variable global (cadena de caracteres) en el proyecto y asigne el nombre de la vista HFSQL a este Variable ** antes de ejecutar el reporte** principal.

2. Asociar la vista con el secundario reporte (con [ViewName](../Proprietes/2511030.md)) en el código de apertura del secundario reporte:
	
	```txt
	NameSecondaryReport>.ViewName = <NameGlobalVariable
	```




<a name="NOTE3_7"></a>


### Pasar parámetros a una reporte interna
<a name="pasar_parametros_una_reporte_interna_ELTPARAGRAPHE000164"></a>

Los parámetros dados a la reporte (durante la llamada a la [iPrintReport](../WDLang5/3046032.md)) se envían automáticamente a la reporte interna. Atención: en este caso, puede que no sea posible pasar los parámetros por reference. 

Para pasar parámetros a una reporte interna, se recomienda utilizar variables globales. Por ejemplo, un Variable global puede ser declarado en el llamado reporte. Este Variable puede ser manejado en el reporte interno a través de la siguiente sintaxis:


```txt
<NameCallingReport>.<Variable name>
```

## Notas
<a name="notas_ELTTEXTE000384"></a>


### Dimensión de los informes secundarios
<a name="dimension_los_informes_secundarios_ELTPARAGRAPHE000180"></a>

Cuando se imprime el reporte principal, el ancho de los informes secundarios se ajusta en relación con el ancho del reporte "interno" control.

Todos los bloques encontrados en los informes secundarios se imprimirán. La reporte principal se adaptará automáticamente a la altura de los informes secundarios por:

- incrementando (si es necesario) la altura del "reporte interno" control.

- aumentando (si es necesario) la altura del bloque que contiene el "reporte interno" control.

- moviendo hacia abajo los controles que se encuentran debajo del control "Reporte interno".



<a name="NOTE4_2"></a>


### Reduciendo el ancho de un reporte secundario
<a name="reduciendo_ancho_reporte_secundario_ELTPARAGRAPHE000193"></a>

Para reducir el ancho de un reporte secundario:

1. Vaya a la pestaña "Formato" de la description de la reporte secundaria ("Reporte description" en el menú contextual).

2. Modificar el ancho de la página donde se imprimirá la reporte secundaria.

3. Validar. 



<a name="NOTE4_3"></a>


### Salto de página en la secundaria reporte
<a name="salto_pagina_secundaria_reporte_ELTPARAGRAPHE000204"></a>

Los saltos de página definidos en una reporte secundaria se ignorarán al imprimir la reporte principal.
<a name="NOTE4_4"></a>


### Apertura del reporte interno
<a name="apertura_del_reporte_interno_ELTPARAGRAPHE000213"></a>

El menú emergente del "Reporte interno" control en el editor se usa para abrir directamente el reporte interno a través de "Abrir el reporte interno".

## Resumen de control Firma
<a name="resumen_control_firma_ELTTEXTE000149"></a>


### control de una reporte
<a name="control_una_reporte_ELTPARAGRAPHE000011"></a>

El control Firma es un reporte control usado para firmar: 

- los informes generados como PDF. 

- los duplicados de reporte. 




Esta control Firma se utiliza para implementar 2 tipos de firmas: 

- **Firma de aprobación**: Una firma de aprobación es una firma numérica aplicada a una Document. Una Document en PDF puede contener varias firmas de aprobación sucesivas, realizadas con el software correspondiente. Cuando la Document contiene una firma de aprobación, se muestra una barra con una marca verde en Adobe Acrobat. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ChampSignature%20-%20HC%20N%B0001.gif)


- **Firma de certificación**: En relación con la firma de aprobación, la firma de certificación garantiza un nivel más alto de verificación de documentos.. Cuando se certifica el Document, se muestra una barra azul con una certificación Icon en Adobe Acrobat. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=ChampSignature%20-%20HC%20N%B0002.gif)

## Creación de una control Firma
<a name="creacion_una_control_firma_ELTTEXTE000173"></a>
**Para crear un control Firma**:

1. Visualizar (si es necesario) un reporte en el editor de reportes. 

2. En la pestaña "Creación", en el grupo "Certificación", despliegue "Firma" y seleccione el tipo de firma que desea crear: firma de aprobación o firma de certificación.

3. Haga clic en la ubicación en la que desea crear el control.




Para ver las características del control, seleccione la opción "Descripción en el menú contextual.

Observación: se puede crear una sola control Firma en un reporte.

## Características de la control Firma
<a name="caracteristicas_control_firma_ELTTEXTE000197"></a>
El control Firma puede ser configurado en el editor a través de su ventana description.

Las principales características de control están disponibles en la pestaña "General".: 

- **Tipo de firma**: Tipo de firma a generar: firma de aprobación o firma de certificación. 
	Observación: El tipo de firma puede modificarse mediante programación con el programa [SignatureType](../Proprietes/1000021995.md) Property. 

- **Caption**: Leyenda mostrada en la control Firma. Esta leyenda puede contener información sobre la persona que firmó la reporte, por ejemplo. 
	Observación: La leyenda de la firma puede modificarse mediante programación con el [Pie de foto](../Proprietes/2510053.md) Property. 

- **Image**: Image expuesto en el control Firma. 
	Observación: La imagen de la firma puede modificarse mediante programación con el [Image](../Proprietes/2510034.md) Property. 

- **Certificado predeterminado**: Certificate solía firmar el reporte. Se proponen los diferentes certificados instalados en el ordenador actual.
	Observación: El Certificate utilizado para la firma puede modificarse mediante programación con el [Certificate](../Proprietes/1000021684.md) Property. 




Observaciones: 

- La leyenda y la Image son información opcional.

- Si no se muestra el control Firma (opción "Visible" disponible en la pestaña "UI" de la ventana de descripción), el reporte PDF generado (o el duplicado) no se firmará. 

- Si no se encuentra el Certificate associated con el control Firma, el PDF reporte generado (o el duplicado) no será firmado.




- **Atención**: Para que Acrobat reconozca un archivo PDF firmado, la organización que crea la firma Certificate debe estar en la lista de confianza aprobada por Adobe ([Lista de confianza autorizada de Adobe](https://helpx.adobe.com/acrobat/kb/trust-services.html)) o en el programa de servicios certificados Document (CDS). 
	Si no se reconoce la Certificate, Acrobat mostrará el siguiente mensaje "Al menos una firma tiene problemas".

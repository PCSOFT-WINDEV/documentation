## Visión general del editor de HTML control
<a name="vision_general_del_editor_html_control_ELTTEXTE000257"></a>
El editor HTML control permite crear y modificar páginas HTML. Esta control está basada en el Chromium Embedded Framework (CEF). 



La cinta, que se incluye en la control, permite el acceso a todas las funciones necesarias: 

- elección de la fuente, tamaño, color, 

- negrita, cursiva, subrayado, tachado...

- importa estilos CSS, 

- gestión de imágenes: mover, recortar, redimensionar, rotar, etc.

- editar enlaces,

- deshacer/rehacer,

- copiar/pegar texto sin formato, RTF, Docx, HTML..

- pegar sin formato, 

- ahorrar,

- imprimir, 

- exportar a PDF,

- emojis, 

- alineación, 

- sangría, 

- tables,

- corrector ortográfico, 

- buscar/reemplazar, 

- ...

## Creación de un editor HTML control
<a name="creacion_editor_html_control_ELTTEXTE000281"></a>
Para crear un editor HTML control:

1. En la pestaña "Creación", en el grupo "Otros controles", despliegue "HTML" y seleccione "Editor HTML".

2. Haga clic en la ubicación en la que desea crear el control. El control aparece en el editor.




Observación: Las dimensiones del control que se creó se optimizan para ocupar el espacio disponible en la ubicación indicada. Si el tamaño del control no es el indicado, pulse Ctrl+Z: el control volverá al tamaño predeterminado.

Para ver las características del control, seleccione la opción "Descripción en el menú contextual.

## Características del editor HTML control
<a name="caracteristicas_del_editor_html_control_ELTTEXTE000309"></a>
La pestaña "General" de la ventana de description del editor HTML control incluye las siguientes opciones: 

- **Con cinta de opciones**: Permite mostrar una cinta en la parte superior del control. Esta cinta contiene diferentes opciones para manejar la Document HTML que aparece en la control. Es posible: 

	- utilizar la cinta de opciones predeterminada. En este caso, la opción "VI fuente de la cinta" corresponde a "Ninguna". 

	- personalizar la cinta de opciones predeterminada.
			Simplemente seleccione la ventana predefinida en la opción "VI fuente de la cinta". En este caso, la ventana "IW_WinDevAAF_HTMLEditorRibbon" se incluirá automáticamente en su proyecto. Esta ventana interna (disponible en inglés y francés) puede modificarse según sus necesidades. Esta ventana contiene el código necesario para gestionar el Editor HTML control. Le recomendamos comprobar los errores de UI y asegurarse de que la ventana funciona correctamente después de realizar cualquier cambio. 

	- utilizar una ventana interna para crear una cinta de opciones específica.
			Puede utilizar cualquier ventana interna del proyecto para crear la cinta.




- Al utilizar la cinta de opciones, la opción "Aplicar plantilla del proyecto a la cinta de opciones" permite utilizar automáticamente la plantilla del proyecto en la cinta seleccionada. 

- **Adaptar control para escribir correos electrónicos**: Si esta opción está marcada, el editor HTML control está optimizado para escribir emails. 
	**Atención: esta opción cambia la ubicación de las imágenes insertadas en el HTML Document.**

- **Permitir cambiar de documento (Abrir, Nuevo, etc.)**: Si esta opción está marcada, el usuario podrá cargar cualquier Document en el Editor HTML control. De lo contrario, las opciones "Abrir" y "Nuevo" no estarán disponibles. Sólo será posible editar la Document HTML en la control.  

- **Habilitar edición del código fuente HTML**: Si esta opción está marcada, la cinta mostrará una pestaña "Ver" para ver y modificar el código fuente HTML. 

- **Activar [Revisar ortografía] de forma predeterminada**: Especifica si el corrector ortográfico debe estar habilitado. Para obtener más información, consulte [Revisar ortografía con diccionarios Hunspell](../WDChamp/1410087030.md). 







### Ubicación de las imágenes insertadas en el editor HTML control
<a name="ubicacion_las_imagenes_insertadas_editor_html_control_ELTPARAGRAPHE000136"></a>

Cuando se inserta una imagen en un documento HTML, su ubicación variará en función de la opción "Adaptar control para escribir correos electrónicos" de la pestaña "General" del control del editor HTML description.

**Si la opción "Adaptar control para escribir correos electrónicos" está seleccionada:**
En este modo, las imágenes insertadas o pegadas no se incrustarán en el código HTML generado. Sólo se almacenará el nombre de la Image. El código HTML generado tendrá el siguiente formato:

```txt
<img src="nom-image.png">
```
La imagen se copiará en el directorio de trabajo del editor de HTML control. 
El directorio de trabajo del control del Editor HTML se puede configurar usando el [WorkingDirectory](../Proprietes/1000026169.md) Property .

En el caso de un texto HTML destinado a ser enviado por correo electrónico, el directorio de trabajo que contiene los Image s puede utilizarse como directorio base de Image para la función [EmailImportHTML](../WDLang3/3032038.md). Ejemplo: 

```wl
MyEmail is Email
HTMLEditorGet(HTMEDT_New_Email, hegValue, HTMLEditorGet_Callback)

	INTERNAL PROCEDURE HTMLEditorGet_Callback(value)
		EmailImportHTML(MyEmail, value, HTMEDT_New_Email.WorkingDirectory)
	END
```


**Si "Adaptar control para escribir correos electrónicos" no está marcado**:

- Si la imagen se inserta en el control del editor HTML desde el portapapeles, la imagen se incrustará en el código HTML con la codificación automatic base 64. 
	El código HTML generado para la Image tendrá el siguiente formato: 
	```txt
	<img src="data/image/png;base64...">
	```


- Si la Image se inserta usando el botón "Image" en la pestaña "Insertar" de la cinta, hay dos casos posibles: 

	- Si "Incrustar Image en la página" está marcado: El código HTML generado para la Image tendrá el siguiente formato: 
			
		```txt
		<img src="data/image/png;base64 ...">
		```


	- Si "Incrustar Image en la página" no está marcado: el código HTML generado almacenará el nombre de la Image y tendrá el siguiente formato:
			
		```txt
		<img src="nom-image.png">
		```











### Codificación de caracteres
<a name="codificacion_caracteres_ELTPARAGRAPHE000188"></a>

El editor de HTML control codifica los caracteres a UTF-8 para que se muestren de forma coherente en todas las plataformas.

Si se utiliza el control HTML Editor para enviar un Email, [HTMLEditorToEmail](../WDLang1/1000026171.md) insertará automáticamente &lt;meta charset="utf-8"/&gt; en la parte HTML del Email.

Si el código HTML de la control se recupera con [HTMLEditorGet](../WDLang1/1000025981.md) y luego se intercambia con una aplicación de terceros (insertando datos en una base de datos, llamando a un Webservice SOAP o a un API REST, etc.), se puede hacer una de las siguientes cosas: 

- Configurar la aplicación de terceros para manipular el código HTML como UTF-8. 

- Utilice [UTF8ToString](../WDLang1/3024043.md) para modificar la codificación del código HTML.


Si no se aplica ninguna de estas soluciones, algunas características (por ejemplo, los acentos) no se mostrarán correctamente.

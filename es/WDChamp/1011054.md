


## Editor de reportes: Tipos de informes disponibles
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000301"></a>
<a name="Type_Report"></a>
Se pueden crear varios tipos de informes en el editor de reportes. Todos estos tipos de informes están disponibles en editor de reportes (suministrado con WINDEV, WEBDEV y WINDEV Mobile) y en Reports & Queries.

Los siguientes tipos de informes están disponibles:

- Reporte en blanco.

- Forma reporte.

- Mesa reporte.

- Crosstab reporte.

- Tabla TreeView reporte.

- Diagrama de Gantt reporte.

- Organizador reporte (no disponible en WINDEV Mobile).

- Programador reporte (no disponible en WINDEV Mobile).

- Correo personalizado reporte (no disponible en WINDEV Mobile).

- reporte Label.

- Reporte basada en un formulario.

- Reporte multicolumna.

- Reporte compuesto.






<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Detalles sobre los diferentes tipos de informes
<a name="detalles_sobre_los_diferentes_tipos_informes_ELTTEXTE000325"></a>
Veamos las características de cada tipo de reporte así como ejemplos de uso: 

- **Reporte en blanco**:

	- **Lectura de la fuente de datos del reporte**
			Se muestra cada página:

		- un texto específico. 

		- un formulario estándar (rellenado manualmente a partir de entonces). 

		- los datos procedentes de uno o más registros.




- **Ejemplos de uso**: 

	- Formulario para introducir las llamadas entrantes. 

	- Formulario para registrar las ausencias. 

	- Formulario para introducir los productos recibidos.

- **Reporte Formulario**:

	- **Lectura de la fuente de datos del reporte**
			Los datos leídos en la fuente de datos se visualizan en un formulario: cada página muestra datos de uno o varios registros.

	- **Ejemplos de uso**:

		- Lista detallada.

		- Datos agrupados según un criterio.




- **Formulario reporte vista previa**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatFiche2.gif&type=thumb)

- **TABLE reporte**:

	- **Lectura de la fuente de datos del reporte**
			Cada tabla fila corresponde a un registro leído en la fuente de datos de reporte (archivo de datos, consulta, ...).

	- **Ejemplos de uso**:

		- Tabla de resumen.

		- Factura.

		- Lista detallada. 

		- Datos agrupados según un criterio




- **Tabla reporte vista previa**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatListe2.gif&type=thumb)

- **Reporte Tabla cruzada**:

	- **Lectura de la fuente de datos del reporte**
			Los datos leídos en la fuente de datos se visualizan en una entrada doble tabla.

	- **Ejemplo de uso**: Tabla que representa el volumen de negocios por producto y por año.

	- **Crosstab reporte preview** <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=etatTableaucroise2.gif&type=thumb)
.




- **Reporte Tabla TreeView**:

	- **Lectura de la fuente de datos del reporte**
			Los datos leídos en la fuente de datos se visualizan en una vista en árbol tabla.

	- **Ejemplo de uso**: Tabla que representa el volumen de negocios por producto y por año.

	- **Tabla de vista de árbol reporte vista previa** <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Etat_Tableau_H.gif&type=thumb)
.




- **Reporte Planificador**:

	- **Lectura de la fuente de datos del reporte**
			Los datos leídos en la fuente de datos se visualizan en formato de programación. 

	- **Ejemplos de uso**: Impresión de un programa diario o semanal, ... 

	- **Programador reporte vista previa**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatPlanning2.gif&type=thumb)
.




- **Reporte Agenda**:

	- **Lectura de la fuente de datos del reporte**
			Los datos leídos en la fuente de datos se visualizan en formato organizador.. 

	- **Ejemplos de uso**: Impresión de un programa diario o semanal, ... 

	- **Organizador reporte vista previa**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Etat_Agenda%20-%20HC%20N%B0001.gif&type=thumb)
.




- **Reporte Mailing**:

	- **Lectura de la fuente de datos del reporte**
			Cada Record leída en la fuente de datos corresponde a una página.

	- **Ejemplos de uso**:

		- Mailshot.

		- Carta publicitaria.




- **Envío reporte vista previa**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Etat_Mailing.gif&type=thumb)

- **Label reporte**:

	- **Lectura de la fuente de datos del reporte**
			Cada Record leída en la fuente de datos corresponde a una etiqueta. 
			Si selecciona este tipo de reporte, puede definir el número de copias para la misma etiqueta.

	- **Ejemplos de uso**:

		- Etiquetas de precios.

		- Etiquetas Address.




- **Etiqueta reporte vista previa**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatEtiquette3.gif&type=thumb)
.

- **Reporte basado en un formulario**:

	- **Lectura de la fuente de datos del reporte**
			Los datos leídos en la fuente de datos se visualizan en un formulario: cada página muestra datos de uno o varios registros.

	- **Ejemplos de uso**: 

		- Formularios postales. 

		- Comprobado.




- **Vista previa de un reporte basado en un formulario**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatFormulaireExecution.gif&type=thumb)
.

- **Reporte multicolumna**:

	- **Lectura de la fuente de datos del reporte**
			si el reporte multicolumna es un reporte de tabla, cada tabla fila corresponde a un Record leído en la fuente de datos del reporte (archivo de datos, consulta, ...).
			Si la multicolumna reporte es un formulario reporte, los datos leídos en la fuente de datos se visualizan como un formulario.: cada página muestra datos de uno o varios registros.

	- **Ejemplo de uso**: Catálogo de productos

	- **Multicolumna reporte vista previa**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatMulticol.gif&type=thumb)
.




- **Reporte compuesto**:

	- **Lectura de la fuente de datos del reporte**
			La combinación reporte incluye varios subinformes. Cada sub-reporte puede tener un tipo específico. La lectura de la fuente de datos de los subinformes depende del tipo de subinformes. 

	- **Ejemplos de uso**: Impresión de pedidos y acuerdos de venta, ....

	- **Composite reporte preview**<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=EtatComposite.gif&type=thumb)
.

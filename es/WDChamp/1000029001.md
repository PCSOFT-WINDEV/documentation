## Presentación del control Tabla Dinámica
<a name="presentacion_del_control_tabla_dinamica_ELTTEXTE000131"></a>
El control Tabla dinámica muestra dinámicamente datos que provienen de varios archivos diferentes encontrados en una base de datos . 

Esto le permite visualizar el volumen de ventas por familias de productos, productos, regiones, a lo largo del tiempo, con o sin detalles.. 

La mesa pivotante funciona: 

- en las bases de datos HFSQL,

- en otras bases de datos mediante arrays de estructuras.  




Observaciones:

- El contenido de la control Tabla dinámica se puede imprimir a través de su menú emergente. Para obtener más información, consulte [Impresión del contenido de un pivote tabla (AAF)](../WDChamp/1000029009.md). 

- Puede crear una reporte basada en una control Tabla dinámica. Para obtener más información, consulte [reporte Pivote de la mesa](../WDChamp/1011092.md). 





### ATENCIÓN: El tiempo de cálculo depende del tamaño de la base de datos y del número de fila y de las cabeceras de columna.
<a name="atencion_tiempo_calculo_depende_del_tamano_base_datos_del_numero_fila_las_cabeceras_columna_ELTPARAGRAPHE000036"></a>

Cada caso es diferente: sin embargo, si el tiempo de Process parece ser demasiado largo: 

1. Compruebe que sus servidores están en modo de 64 bits (evite las tecnologías anteriores en modo de 32 bits que limitan la memoria a 4 GB).. 

2. Por favor, Contact el soporte técnico gratuito e incluye: 

	- a description de las operaciones realizadas,

	- el pivote tabla,

	- el análisis,

	- la base de datos,

	- el tiempo de respuesta esperado y el tiempo de respuesta observado,

	- las características técnicas del servidor (versión de SE, capacidad de los discos, tecnología de los discos, RAM),

	- comprobar si no se ejecuta ningún otro Process de consumo en el servidor (o indicar estos Process es),





Una prueba de rendimiento será realizada por el Soporte Técnico Gratuito. Si las prestaciones no parecen lo suficientemente buenas, se puede realizar una optimización del pivote tabla (en una próxima versión)..

## Ejemplo de pivote tabla (basado en archivos de datos HFSQL)
<a name="ejemplo_pivote_tabla_basado_archivos_datos_hfsql_ELTTEXTE000161"></a>


### Ejemplo
<a name="ejemplo_ELTPARAGRAPHE000058"></a>

Veamos un ejemplo de pivote tabla basado en archivos de datos HFSQL. Los principales conceptos del pivote tabla se presentan a partir de este ejemplo.  

Análisis asociado: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=TCD_Analyse.gif)


Pivote creado tabla: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=TCD_Vocabulaire.gif)


Esta tabla representa las ventas de productos (organizadas por categorías) por año (años, trimestres y meses).. 

Los valores procedentes del archivo fuente se muestran en el centro (1). Estos valores se calculan automáticamente según los grupos definidos. 

Las cabeceras de fila y las cabeceras de columna (2) corresponden a las posiciones utilizadas para agrupar los datos.: 

- En fila, en nuestro ejemplo, el total de ventas se muestra por producto y por categoría.. Desde el archivo fuente llamado "OrdLine", se realiza una agrupación en el elemento "ProdCap" del archivo "Product", luego se realiza una agrupación en el elemento "CategoryCap" del archivo "Category".. 

- En la columna, en nuestro ejemplo, el total de ventas se muestra por fecha (mes, trimestre, categoría). Desde el archivo fuente llamado "OrdLine", se realiza una agrupación en el elemento "OrderDate" del archivo "Order"..  




Los totales (3) se calculan automáticamente. 

La creación de un pivote tabla se realiza en el editor a través de una asistente. Para obtener más información, consulte [Creación de un pivote tabla](../WDChamp/1000029006.md).

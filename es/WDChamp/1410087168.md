## Eventos predeterminados
<a name="eventos_predeterminados_ELTTEXTE000231"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) WINDEV gestiona los siguientes eventos de forma predeterminada (en orden de aparición en el editor de código):

| Evento | Condición de ejecución |
| --- | --- |
| Inicialización | Se ejecuta al abrir la ventana. \* |
| Modificación del rango (zoom) | Se ejecuta después de acercar o alejar el zoom en el control Editor de diagramas (botón de zoom, Ctrl + rueda, etc.). |
| Antes de cerrar un documento | Se ejecuta antes de cerrar el diagrama en el control Editor de diagramas. |
| Adición de una forma | Se ejecuta cuando se agrega una forma en el control Editor de diagramas. |
| Selección de una forma | Se ejecuta cuando se selecciona una forma en el control Editor de diagramas. <br>Puede obtener las características de la forma seleccionada utilizando la propiedad [Selection](../Proprietes/1000023033.md), que devuelve una variable de tipo [diagSelection](../WDLang1/1410088172.md). <br><br>Observación: cuando se crea una forma a partir de una biblioteca, se selecciona la forma. |
| Desplazamiento de una forma | Se ejecuta cuando se mueve una forma en el control Editor de diagramas. |
| Redimensionamiento de una forma | Se ejecuta cuando se cambia el tamaño de una forma en el control Editor de diagramas. |
| Modificación de una forma | Se ejecuta cuando se modifica una forma en el control Editor de diagramas (por ejemplo, cuando se valida el texto asociado a la forma). |
| Doble clic en una forma | Se ejecuta cuando se hace doble clic en una forma en el control Editor de diagramas. |
| Siempre que se modifique | Se ejecuta cada vez que se modifica el control Editor de diagramas: mover una forma, agregar una forma, eliminar una forma, etc.. |


(\*) De forma predeterminada, el evento "Inicialización" de cada control se ejecuta según el orden en el que se crearon los controles en la ventana. Para cambiar este orden de ejecución: 

1. En la pestaña "Ventana", en el grupo "Orden", haga clic en "Inicialización".

2. En la ventana que aparece, utilice las flechas de la derecha para cambiar el orden en el que se inicializan los controles.

## Eventos opcionales
<a name="eventos_opcionales_ELTTEXTE000261"></a>
Se pueden utilizar varios eventos opcionales.
Para añadir un evento opcional:

1. Seleccione la control deseada.

2. Mostrar la ventana del código de esta control (tecla F2).

3. Haga clic en el enlace "Añadir otras Event s a xxx" en la parte inferior del código de la ventana, después de la última Event.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Traitements_optionnels_WD_OK%20-%20HC%20N%B0001.gif)


4. Todos los eventos opcionales disponibles para el control se muestran. 

5. Marque la opción Event para agregar y validar. 
	Observación: Puede seleccionar varios eventos opcionales. 

6. El Event opcional seleccionado se agrega automáticamente a los Event s administrados por el control.




Para desactivar un Event opcional, simplemente realice las mismas operaciones para mostrar la lista de Event s opcionales. A continuación, simplemente desmarque los eventos opcionales para eliminar. 
Observación: Si el código desactivado contiene código WLanguage, este código se elimina automáticamente.Se puede gestionar, por ejemplo:

- el desplazamiento del cursor por encima del control,

- el botón izquierdo del ratón presionado, liberado o con doble clic,

- el botón derecho del ratón presionado, liberado o con doble clic,

- la rueda del ratón, etc.




Para más detalles, consulte la página web de [Eventos opcionales](../WDChamp/1014004.md).

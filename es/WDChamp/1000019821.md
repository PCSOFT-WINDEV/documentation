
## Organigrama: Mostrando un Process de fabricación
			

<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000076"></a>
Una control Organigrama puede utilizarse para mostrar una Diagram. Se puede utilizar, por ejemplo, para mostrar los diferentes pasos de un Process. 



## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000100"></a>
El control Organigrama es muy adecuado para realizar este tipo de Diagram aunque se requieren ajustes específicos. 

En una control Organigrama, cada elemento puede tener un solo padre. 

En la mayoría de los casos, la representación se invierte en un Process Diagram: un elemento hijo se crea a partir de varios elementos padre. 

Por lo tanto, se debe crear una control Organigrama anulada.. Para ello: 

1. **Definir los elementos en el orden inverso al de Process**.
	El primer elemento "padre" debe ser el último elemento de la Diagram (en nuestro ejemplo, el primer elemento del organigrama es "Absoluto" que es el último elemento de la Diagram). 
	Los elementos "hijos" de Diagram deben definirse como elementos "padres" en el organigrama (en nuestro ejemplo, el paso "Calentar, enfriar, filtrar" se define como hijo de "Absoluto" mientras que es el paso anterior en Diagram). 

2. **Muestra el organigrama de "derecha a izquierda "** (o de "abajo a arriba") para "restablecer" la dirección correcta del Diagram.

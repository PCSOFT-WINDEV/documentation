### Eventos soportados por WINDEV y WINDEV Mobile
<a name="eventos_soportados_por_windev_windev_mobile_ELTPARAGRAPHE000013"></a>

WINDEV y WINDEV Mobile gestionan los siguientes eventos por Default (orden de aparición en el editor de código):

| Event | Condición de ejecución |
| --- | --- |
| Inicialización | Corre cuando se abra la ventana. \* |
| Modificar el panel mostrado | Ejecutar siempre que se modifique el contenido de la control (cambio del panel de pestañas, por ejemplo) |
| ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Creación de un panel | Ejecutar al crear un panel de pestañas dinámico mediante el botón de adición ("+") que se encuentra entre los paneles de pestañas.. <br><br>Si no se ha especificado ninguna ventana interna en la interfaz, utilice [TabOpen](../WDLang1/1000021369.md) en el código para abrir una nueva pestaña dinámica. <br><br>Si se especificó una ventana interna en la interfaz y si esta Event está utilizando [TabOpen](../WDLang1/1000021369.md), esta Event tendrá prioridad sobre la ventana interna especificada en la interfaz. |
| ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Cerrar un panel | Ejecutar al cerrar un panel de tabulación dinámico mediante la cruz de cierre. |


(\*) Por defecto, la "Inicialización" Event de cada control se ejecuta según el orden en que se crearon las control en la ventana. Para modificar este orden de ejecución: 

1. En la pestaña "Ventana", en el grupo "Orden", haga clic en "Inicialización". 

2. En la ventana mostrada, utilice las flechas de la derecha para modificar el orden en el que se inicializan los controles.

## Caso especial: Modificación y cierre de un panel de pestañas
<a name="caso_especial_modificacion_cierre_panel_pestanas_ELTTEXTE000207"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Los eventos "Modificar el panel mostrado" y "Cerrar un panel" pueden aceptar un parámetro, sólo para un control Pestaña dinámico, si el panel de pestañas fue creado dinámicamente.. 

Esta Procedure debe ser declarada de la siguiente manera: 


```txt
PROCEDURE <Procedure name>(<Name of Internal Window>)
```


Esto le permite averiguar el nombre de la ventana interna utilizada. 

Por ejemplo: 


```wl
// Event "Closing a pane" 
PROCEDURE OnEnd(xIW)

MyIntWin is Control <-xIW
IF MyIntWin <> Null THEN
	Trace(MyIntWin.Name)
END
```

## Eventos opcionales
<a name="eventos_opcionales_ELTTEXTE000231"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Se pueden utilizar varios eventos opcionales.
Para añadir un evento opcional:

1. Seleccione la control deseada.

2. Mostrar la ventana del código de esta control (tecla F2).

3. Haga clic en el enlace "Añadir otras Event s a xxx" en la parte inferior del código de la ventana, después de la última Event.  <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Traitements_optionnels_WD_OK%20-%20HC%20N%B0001.gif)


4. Todos los eventos opcionales disponibles para el control se muestran. 

5. Marque la opción Event para agregar y validar. 
	Observación: Puede seleccionar varios eventos opcionales. 

6. El Event opcional seleccionado se agrega automáticamente a los Event s administrados por el control.




Para desactivar un Event opcional, simplemente realice las mismas operaciones para mostrar la lista de Event s opcionales. A continuación, simplemente desmarque los eventos opcionales para eliminar. 
Observación: Si el código desactivado contiene código WLanguage, este código se elimina automáticamente.Se puede gestionar, por ejemplo:

- el desplazamiento del cursor por encima del control,

- botón izquierdo del ratón abajo, arriba, con un doble clic,

- el botón derecho del ratón presionado, liberado o con doble clic 

- la rueda del ratón, etc.

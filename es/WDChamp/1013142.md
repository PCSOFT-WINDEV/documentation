
## Vinculación de una control Barra de progreso a un elemento
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Características del artículo
<a name="caracteristicas_del_articulo_ELTTEXTE000080"></a>
Una control Barra de progreso puede estar vinculada a un elemento de un archivo de datos, consulta, etc.

En este caso, las características de la posición son las siguientes:

- el elemento es de tipo **Integer**.

- el elemento contiene el valor de la posición en la control Barra de progreso. Esta posición corresponde al valor de la control Barra de progreso.

- el valor del artículo debe estar entre el valor mínimo y máximo de la control Barra de progreso.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Manipulación del artículo y de la control Barra de progreso
<a name="manipulacion_del_articulo_control_barra_progreso_ELTTEXTE000104"></a>


### Actualización de la control Barra de progreso con los datos del artículo
<a name="actualizacion_control_barra_progreso_con_los_datos_del_articulo_ELTPARAGRAPHE000024"></a>

**Primer caso: los controles de la ventana o página están enlazados a los campos de un archivo de datos**
Todos los controles enlazados a un campo en el archivo de datos se actualizarán con:

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [FileToScreen](../WDLang4/3044210.md) en WINDEV o WINDEV Mobile.




**Segundo caso: los controles de la ventana o página no están enlazados a los campos de un archivo de datos**
Se debe asignar un valor a cada control utilizando la siguiente sintaxis:


```txt
<Control name> = <Data file>.<Item>
```

<a name="NOTE2_2"></a>


### Actualización de la partida con los datos de la control Barra de progreso
<a name="actualizacion_partida_con_los_datos_control_barra_progreso_ELTPARAGRAPHE000031"></a>

**Primer caso: los controles de la ventana o página están enlazados a los campos de un archivo de datos**
Todos los campos de el archivo de datos enlazados a un control de la ventana o página se actualizarán con:

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) [ScreenToFile](../WDLang4/3044146.md) en WINDEV o WINDEV Mobile.




**Segundo caso: los controles de la ventana o página no están enlazados a los campos de un archivo de datos**
El valor de cada control debe asignarse a cada campo utilizando la siguiente sintaxis:


```txt
<Data file>.<Item> = <Control name>
```

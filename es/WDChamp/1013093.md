


## Los diferentes tipos de botones en una ventana
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000180"></a>
WINDEV y WINDEV Mobile incluyen 5 tipos de botones. Cada uno realiza una acción específica.

| escribir a máquina | description |
| --- | --- |
| Normal | Botón estándar.<br><br>**Secuencia de eventos de WLanguage**: Cuando se pulsa un botón "Normal" (o se utiliza el atajo de teclado associated):<br><br>- se ejecuta el código de salida de la control actual.<br><br>- se ejecuta el código de clic de la control Botón.<br><br><br>**Modo de uso**: " Normal" Los controles de los botones no deben utilizarse en ventanas que contengan controles. Este tipo de botón puede utilizarse en barras de herramientas, barras de Icon o ventanas sin controles de edición. |
| Validar | Botón"Default" de la ventana.<br><br>**Secuencia de eventos de WLanguage**: Durante un clic del ratón en un botón de "Validación" (o cuando se utiliza el acceso directo de associated):<br><br>- se ejecuta el código de salida de la control actual.<br><br>- se ejecuta el código de clic de la control Botón.<br><br><br>El código de clic de este botón se ejecuta cuando se pulsa la tecla Enter, si:<br><br>- ninguna otra ventana control intercepta la tecla Enter.<br><br>- ninguna otra ventana control tiene el foco.<br><br><br>Si dos botones de validación se encuentran en la misma ventana, la ejecución de código es el código del primer botón de "validación" encontrado según el orden de tabulación de la ventana.<br><br>**Modo de uso**: Se debe utilizar un botón VALIDATION para validar la entrada en una ventana.. Un botón VALIDACIÓN debe encontrarse en la ventana tan pronto como se pueda realizar una entrada en un control Campo de entrada. Este es el tipo utilizado para los botones "OK".<br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Este tipo de botón no está disponible. |
| Interrumpir | Botón utilizado para interrumpir la Process en curso.<br><br>**Secuencia de eventos de WLanguage**: Durante un clic del ratón en un botón de "Interrupción" (o cuando se utiliza el acceso directo de associated):<br><br>- se ejecuta el código de clic de la control Botón.<br><br>- el botón no mantiene el enfoque. El foco de atención se devuelve a la control que se utilizó anteriormente.<br><br><br>Observación: El código de salida de control actual no se ejecuta durante la pulsación del botón.<br><br>**Modo de uso**: Un botón "Interrupción" permite llamar a una ventana (para interrumpir la entrada actual sin validar) y reanudar la entrada.. Este es el tipo utilizado para los botones "Ayuda".<br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Este tipo de botón no está disponible. |
| Cancelar | El botón y la tecla Esc tienen la misma acción.<br><br>**Secuencia de eventos de WLanguage**: El código de salida de control actual no se ejecuta durante la pulsación del botón. El código de este botón se ejecuta:<br><br>- al pulsar la tecla Esc siempre que ninguna otra ventana control intercepte la tecla Esc.<br><br>- al pulsar Alt + F4. <br><br>- al cerrar la ventana por la cruz.<br><br><br>Si se encuentran dos botones de cancelación en la misma ventana, el código que se ejecuta es el del primer botón de cancelación encontrado según la orden de tabulación de la ventana.<br><br>**Modo de uso**: Un "Cancel" control Botón servirá para "salir" de la ventana actual sin validar. En una ventana con campos de entrada, se requiere una "Cancel" control Botón para Cancel la entrada actual sin validar. Este es el tipo utilizado para los botones "Cancel".<br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Este tipo de botón no está disponible. |
| Ayuda | La control Botón tiene la misma acción que la tecla F1.. Se ejecuta la Event del "clic" de la control (a diferencia de WINDEV 5.5).. Este tipo de control Botón abre la ventana de ayuda associated con el control actual. Una ayuda control Botón pierde el foco. Se vuelve a centrar en la control utilizada anteriormente. El evento "Salida" y el evento "Entrada" del control no se ejecutan cuando se hace clic en el control Botón "Ayuda".<br><br>![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Este tipo de botón no está disponible. |

## Botón y pestaña
<a name="boton_pestana_ELTTEXTE000204"></a>
Según los estándares de Windows, la tecla de tabulación (Tab) se utiliza para pasar de una control a otra (en modo de edición).

También tiene la posibilidad de pasar de una control (en edición) a otra con la tecla Enter: Para ello, ningún botón activo de la ventana debe ser un botón de validación y ningún control debe ser associated con la tecla Enter.

![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) La gestión a través del tabulador no está disponible.

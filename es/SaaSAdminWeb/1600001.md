
## Administrador de SaaS - Panel de control
			



## Presentación
<a name="presentacion_ELTTEXTE000192"></a>
La cuadro de mando del Administrador SaaS se utiliza para ver rápidamente la información principal relativa a los sitios SaaS de un Servidor de Aplicaciones WEBDEV.

Incluye los siguientes elementos:

- Los enlaces de gestión: estos enlaces proponen las acciones más comunes.

- Un gráfico que muestra los 5 sitios SaaS con el mayor número de licencias asignadas.

- La lista de sitios SaaS. 








## Modo de funcionamiento del modelo SaaS
<a name="modo_funcionamiento_del_modelo_saas_ELTTEXTE000216"></a>
La Diagram que figura a continuación presenta los seis elementos principales del modelo SaaS:
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=SaaS_Fonctionnement.gif)


Estos elementos son:

| Elemento | Rol |
| --- | --- |
| *sitio SaaS* | El **sitio SaaS** es un sitio WEBDEV desplegado en un servidor de aplicaciones WEBDEV. Se trata de associated con un servicio web SaaS utilizado para control el sitio. |
| *Servicio* | El **Servicios** son las diferentes características de un sitio. |
| *Cuenta de Cliente* | A **Cuenta de Cliente** representa a una empresa que utiliza el **sitio SaaS**.<br>Cada **Cuenta de Cliente** incluye su propia base de datos y su propia versión personalizada de la **sitio SaaS** (logotipo, gráficos, etc.). |
| *drogadicta* | El **Usuarios** son los miembros de la empresa representados por el **Cuenta de Cliente**.<br>Los usuarios son las personas físicas que utilizan el **sitio SaaS**. |
| *Determinación de precios* | Una **Tarificación** representa un conjunto de **Servicios** a los que puede acceder un **Usuario**.<br>A **Determinación de precios** puede incluir uno o más **Servicios**.<br>A **Servicio** puede proponerse en uno o varios **Determinación de precios**. |
| *Suscripción* | Una **Suscripción** representa la disponibilidad de un cierto número de licencias para una **Tarificación** (y por lo tanto de los **Servicios** asociados) para una **Cuenta de Usuario**.<br>Entonces, estas licencias deben ser distribuidas entre los **Usuarios**.<br>El **Suscripciones** se utilizan para supervisar el uso del **sitio SaaS** y la facturación. |







## Menú principal del administrador de SaaS
<a name="menu_principal_del_administrador_saas_ELTTEXTE000240"></a>
El menú principal da acceso a todas las operaciones de gestión:

- Tablero de mandos: Volver al resumen general de los sitios saaS de WEBDEV Application Server.

- Sitios

	- Agregar un sitio: Se utiliza para declarar un nuevo sitio SaaS.

	- Administrar un sitio: Permite definir los servicios, los precios y los parámetros de un sitio SaaS.




- Clientes

	- Agregar una cuenta cliente: Se utiliza para declarar una nueva cuenta de cliente SaaS.

	- Administrar una cuenta cliente: Utilizó a manipular los usuarios de associated con la cuenta de cliente así como las Suscripciones.




- Reportes: Permite consultar las estadísticas sobre el uso de los sitios SaaS.. 








## Uso del administrador de SaaS
<a name="uso_del_administrador_saas_ELTTEXTE000264"></a>
Para explotar un sitio SaaS, se deben realizar las siguientes operaciones:

1. Despliegue del sitio SaaS en un servidor de aplicaciones WEBDEV. 
	Para obtener más información, consulte: [Desplegando un sitio](../Editeurs/2028039.md).
	Durante la implementación, usted tiene la capacidad de especificar la Address del Servicio Web SaaS que manipular el sitio.
	El servicio web SaaS puede estar ubicado en el mismo servidor de aplicaciones que el sitio (por defecto) o en un servidor de aplicaciones diferente.
	Las operaciones para la configuración de SaaS se realizarán a partir de entonces a través del Administrador de SaaS situado en el mismo servidor de aplicaciones que el Webservice.

2. Declarar el sitio en el Administrador del SaaS. 
	Para obtener más información, consulte [Adición de un sitio SaaS al administrador](../SaaSAdminWeb/1600002.md).

3. Declarar los servicios propuestos por el sitio en el Administrador del SaaS. 
	Para obtener más información, consulte [Gestión de un sitio SaaS en el administrador](../SaaSAdminWeb/1600003.md).

4. Creación del precio del sitio en el Administrador de SaaS. 
	Para obtener más información, consulte [Gestión de un sitio SaaS en el administrador](../SaaSAdminWeb/1600003.md).

5. Adición de las cuentas de cliente al Administrador de SaaS. 
	Para obtener más información, consulte [Adición de una cuenta de cliente SaaS](../SaaSAdminWeb/1600004.md).

6. Adición de uno o más usuarios para las cuentas de cliente SaaS. 
	Para obtener más información, consulte [Gestión de las cuentas de clientes SaaS](../SaaSAdminWeb/1600005.md).

7. Suscripción de las cuentas de clientes SaaS a los precios del sitio SaaS y asignación de algunas licencias. 
	Para obtener más información, consulte [Gestión de las suscripciones SaaS](../SaaSAdminWeb/1600006.md).




Observación: Si propone enviar el código de seguridad al usuario (contraseña olvidada), debe configurar el servidor de correo electrónico SMTP associated con el administrador SaaS. Esta configuración se puede realizar: 

- mediante la programación, a través de [SaaSAdminConfigureSMTP](../WDLang3/1000022592.md). 

- en el administrador de SaaS, a través del enlace "Configuración SMTP" que se encuentra en la página de cuadro de mando.

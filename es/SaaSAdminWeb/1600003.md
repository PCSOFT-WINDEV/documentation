


## Administrador de SaaS - Gestión de un sitio
			





## Presentación
<a name="presentacion_ELTTEXTE000135"></a>
La gestión de un sitio SaaS se utiliza para modificar las tres características principales del sitio:

- Los servicios SaaS utilizados para definir los permisos de acceso a las diferentes secciones del sitio.

- El precio utilizado para agrupar uno o más servicios con el fin de proporcionarlos a una cuenta de cliente.

- Los parámetros del sitio utilizados para personalizar el sitio según el cliente.






## Gestión de los servicios SaaS
<a name="gestion_los_servicios_saas_ELTTEXTE000159"></a>
Los Servicios SaaS definen los bloques funcionales del sitio. Cuando un cliente utiliza un sitio, los Servicios SaaS se utilizan para limitar el acceso a algunas secciones del sitio o a algunas funciones en función de la identidad del cliente.. 

Esta limitación debe implementarse en el código utilizando [Servicio SaaSCheckService](../WDLang3/1000018961.md).

Un servicio SaaS se define por su nombre. Para definir un nuevo servicio SaaS, sólo tiene que hacer clic en el enlace "Añadir un servicio".

El número de servicios SaaS que puede soportar un sitio SaaS es ilimitado.

Algunos ejemplos de definiciones de servicios SaaS para un sitio de anuncios clasificados:

- Acceso al sitio/lectura de los anuncios

- Escribir anuncios de texto

- Escribir anuncios con imágenes

- Moderar los anuncios






## Gestión de la determinación de precios
<a name="gestion_determinacion_precios_ELTTEXTE000183"></a>
Se utiliza un Pricing para agrupar una set de Servicios SaaS y asociarla a un precio para ponerla a disposición de los usuarios del sitio SaaS (mediante una suscripción).


### Creación de una nueva determinación de precios
<a name="creacion_una_nueva_determinacion_precios_ELTPARAGRAPHE000039"></a>

Para crear un nuevo precio, todo lo que tiene que hacer es hacer clic en el enlace "Añadir un precio".. Una nueva tarificación (cuyo nombre es "New" de Default) se añade a la lista de tarificación del sitio.


### Tratamiento de las propiedades de una determinación de precio
<a name="tratamiento_las_propiedades_una_determinacion_precio_ELTPARAGRAPHE000044"></a>

Para editar los parámetros de una determinación de precios, todo lo que tiene que hacer es hacer clic en su nombre. Una ventana de diálogo le permite introducir:

- El nombre del precio,

- Su coste (la moneda utilizada por los sitios SaaS se puede definir en la página "Panel de control").

- La periodicidad del precio: por mes, por semana, por año o ilimitado (si el costo debe ser pagado de inmediato).

- El modo de facturación: por usuario o por contrato de precio fijo (independientemente del número de usuarios).





### Asociar una tarificación a los servicios
<a name="asociar_una_tarificacion_los_servicios_ELTPARAGRAPHE000054"></a>

Para utilizar un precio, debe ser associated con uno o más servicios. En la lista de precios tabla, verifique las filas correspondientes a los servicios que deben ser relacionado a la lista de precios y haga clic en el botón "Validar".

**Observación**: El administrador de SaaS muestra los diferentes precios de associated con un sitio. Si el centro incluye más de 5 precios, se utiliza una flecha para acceder a los otros precios.. Sin embargo, se pueden proponer más precios mediante la programación con [Precios de SaaSAdminAddPricing](../WDLang3/1000019127.md).



## Gestión de los parámetros del sitio
<a name="gestion_los_parametros_del_sitio_ELTTEXTE000225"></a>
Los parámetros de un sitio SaaS son una set de valores definidos por el desarrollador del sitio para permitir que el sitio sea personalizado de acuerdo a los clientes.

Por defecto, cada sitio declarado en el Administrador SaaS está asociado a un parámetro llamado "Nombre del cliente", lo que permite personalizar la visualización del sitio con el nombre del cliente actual.

A través de la programación, el valor de un parámetro para el cliente actual puede ser leído de nuevo por [SaaSReadSiteParameter](../WDLang3/1000019037.md).

Para definir la lista de parámetros de un sitio, haga clic en el enlace "Definir los parámetros del sitio".

Cada parámetro es associated con un valor de Default que se utilizará para las cuentas de clientes del sitio. El valor de Default puede modificarse para cada cuenta de cliente a través de [SaaSWriteSiteParameter](../WDLang3/1000019039.md).

**Observación**: Aunque todos los usuarios de la cuenta del cliente pueden leer el valor, usted debe ser un usuario con derechos de administración para escribir.

Algunos ejemplos de parámetros:

- Nombre del cliente

- Logotipo

- Imágenes

- Colores

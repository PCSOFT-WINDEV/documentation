### Arrastrar y soltar
<a name="arrastrar_soltar_ELTPARAGRAPHE000010"></a>

Algunos controles de WINDEV y WEBDEV pueden [gestionar automáticamente las operaciones de arrastrar y soltar](../WDLang1/3030012.md): simplemente seleccione la opción correspondiente en la pestaña "Detalles" de la descripción del control. Un control puede definirse como origen o destino de una operación de arrastrar y soltar.

Las operaciones de arrastrar y soltar están disponibles automáticamente: 

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) para los controles Campo de entrada, List Box, ListView y TreeView.


En una [operación de arrastrar y soltar establecida mediante programación](../WDLang1/3030013.md), las propiedades [DndSource](../Proprietes/2510014.md) y **DndTarget** permiten definir los controles de origen y destino.
<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **DndTarget** determina y cambia el comportamiento de un control en una operación de arrastrar y soltar. El control será el destino del arrastre, es decir, el control en el que se soltarán los elementos arrastrados.


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) 
```wl
// -- Initialize the LIST_Source list
// LIST_Source is not a target
LIST_Source.DndTarget = dndInactive

// -- Initialize the LIST_Target list
LIST_Target.DndTarget = dndProgram
DndEvent("Drop", "LIST_Target", dndDrop)
DndEvent("DragOver", "LIST_Target", dndDragOver)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Obtener el comportamiento de un control de destino en una operación de arrastrar y soltar

`<Drag and Drop> = <Control used>.DndTarget`
---

**`<Drag and Drop>: (Constante)`**

Identifica el comportamiento del control de destino en una operación de arrastrar y soltar.


|   |   |
| --- | --- |
| *dndAuto* | Las operaciones de arrastrar y soltar se gestionan automáticamente. |
| *dndInactive* | La control no acepta operaciones de arrastrar y soltar. |
| *dndProgram* | El desarrollador establece la operación de arrastrar y soltar mediante programación. |



**`<Control used>: (Nombre del control)`**

Nombre del control a manipular.  


<a name="SYNTAX2"></a>

### Cambiar el comportamiento de un control de destino en una operación de arrastrar y soltar

`<Control used>.DndTarget = <Drag and Drop>`
---

**`<Control used>: (Nombre del control)`**

Nombre del control a manipular.

**`<Drag and Drop>: (Constante)`**

Identifica el comportamiento del control de destino en una operación de arrastrar y soltar.


|   |   |
| --- | --- |
| *dndAuto* | Las operaciones de arrastrar y soltar se gestionan automáticamente. |
| *dndInactive* | La control no acepta operaciones de arrastrar y soltar. |
| *dndProgram* | El desarrollador establece la operación de arrastrar y soltar mediante programación. |

## Observaciones


### Operaciones de arrastrar y soltar gestionadas automáticamente o establecidas mediante programación
<a name="operaciones_arrastrar_soltar_gestionadas_automaticamente_establecidas_mediante_programacion_ELTPARAGRAPHE000111"></a>

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Las operaciones de arrastrar y soltar automáticas están disponibles para controles Campo de entrada, List Box, ListView, TreeView y Tabla.

Las operaciones de arrastrar y soltar pueden establecerse mediante programación para los siguientes controles:

- Campo de entrada

- Lista Box y control ListView

- control TreeView

- control Tabla

- control Imagen

- Control Código de barras

- Control Botón

- Control Looper

- control Gráfico

- Control Tabla TreeView

- Control Calendario

- Control Agenda

- Control Planificador




![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) Solo están disponibles las operaciones de arrastrar y soltar automáticas.


<a name="NOTE0_2"></a>


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000158"></a>

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) La propiedad **DndTarget** solo puede utilizarse con los siguientes controles:

- Editar control, 

- Control Botón, 

- Control Casilla de verificación, 

- Control Botón de opción, 

- Control Pestaña, 

- Caja de la lista o control ListView,

- Control Combo Box, 

- Control Tabla o Tabla TreeView, 

- Control TreeView,

- Control Looper, 

- Control Imagen, 

- Control de número, 

- Control deslizante, 

- Control Barra de desplazamiento, 

- Control Gráfico,

- Control Calendario, 

- Control deslizante de rango, 

- Control Agenda, 

- Control Planificador, 

- Control .NET 2.0 (WinForm)

- Control Valoración.




![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) El **DndTarget** Property solo se puede usar en los controles Edit, List Box y TreeView.




## Server (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **Server** permite gestionar la fuente de datos de una conexión (HFSQL Cliente/Servidor, OLE DB, etc.). Es posible:

- Averigüe la fuente de datos utilizada para una conexión (HFSQL Client/Server, OLE DB, ....).

- Definir la fuente de datos utilizada para la conexión (HFSQL Client/Server, OLE DB, ....). En este caso, si la conexión está actualmente abierta, se cerrará y se reabrirá con los nuevos parámetros durante el primer acceso a un archivo que utilice esta conexión.




Atención: si se utiliza una base de datos remota y si se utiliza una conexión INTERNET para acceder a ella, el puerto de entrada del router debe ser redirigido al servidor HFSQL. Para obtener más información, consulte [Administrar puertos](../WDLang4/3044334.md).

**Observación**: La conexión puede corresponder a:

- una conexión descrita en la editor de análisis,

- una conexión descrita por [HConnect](../WDLang4/3044263.md), [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- una conexión descrita por un [Conexión](../WDLang4/1514073.md) variable.









<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
MyConnection is Connection
// Describe the connection
MyConnection.User = "USER"
MyConnection.Password = "PASSWORD"
MyConnection.Server = "MYSERVER" 
MyConnection.Database = "Database"
MyConnection.Provider = hAccessHFClientServer
MyConnection.Access = hOReadWrite
MyConnection.ExtendedInfo = "Extended information"
MyConnection.CursorOptions = hClientCursor

HOpenConnection(MyConnection)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Averiguar la fuente de datos de la conexión

`<Data source> = <Connection used>.Server`
---

**`<Data source>: (Cadena de caracteres)`**



- Nombre del servidor (para HFSQL Client/Server, SQL Server, Oracle, MySQL, etc.)

- Nombre del archivo utilizado como fuente de datos (para Access, XML, xBase, etc.)




**`<Connection used>: (Cadena de caracteres o variable de tipo Connection)`**

Nombre de la conexión utilizada. Este nombre fue definido:

- en el editor de análisis, 

- por [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md),

- por el tipo [Conexión](../WDLang4/1514073.md).  





<a name="SYNTAX2"></a>

### Definir la fuente de datos de la conexión

`<Connection used>.Server = <Data source>`
---

**`<Connection used>: (Cadena de caracteres o variable de tipo Connection)`**

Nombre de la conexión utilizada. Este nombre fue definido:

- en el editor de análisis, 

- por [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md),

- por el tipo [Conexión](../WDLang4/1514073.md).




**`<Data source>: (Cadena de caracteres)`**



- Nombre o IP Address del servidor (para HFSQL Client/Server, SQL Server, Oracle, MySQL, ...)

- Nombre del archivo utilizado como fuente de datos (para Access, XML, xBase, etc.)






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
Las propiedades **Server** y [Fuente](../Proprietes/2510147.md) son equivalentes.

Estas dos propiedades se mantienen para que el código sea más fácil de entender. En la conexión HFSQL Client/Server, SQL Server, Oracle o MySQL, se recomienda **Server** Property porque es necesario especificar un nombre de servidor.

Sin embargo, en una conexión Access, XML o xBase, se recomienda utilizar la [Fuente](../Proprietes/2510147.md) Property porque es necesario especificar un archivo de datos.

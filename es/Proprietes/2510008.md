


## MinValue (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **MinValue** Property se utiliza para:

- conocer y modificar el límite mínimo de un **reporte basado en un archivo de datos**. El límite inferior corresponde al valor mínimo permitido para la clave de búsqueda del archivo de datos HFSQL utilizado como fuente de reporte.

- conocer el límite inferior del **filtro actual** (definido con la función [HFilter](../WDLang4/3044100.md)) en un archivo de datos, vista o consulta HFSQL.

- averiguar el valor mínimo de un **serie de gráficos**.

- averiguar y modificar el valor mínimo de **algunos tipos de controles**.

<a name="sample_code"></a>

## Ejemplo

RPT_MyFileReport.MinValue = "DUPONT"

```

```

## Sintaxis

### Averiguar el límite inferior de un control o reporte

`<Current value> = <Element used>.MinValue`
---

**`<Current value>: (Cadena de caracteres)`**

Valor actual del límite inferior:

- para el reporte basado en un archivo de datos.

- para un control encontrado en una ventana o página.




**`<Element used>: (Nombre control o reporte)`**

Este parámetro corresponde a:

- el nombre del reporte utilizado (reporte basado en un archivo de datos).

- el nombre de control cuyo límite inferior se solicita (control en una ventana o en una página). La lista de los controles en cuestión figura en el apartado "Limitaciones. 





<a name="SYNTAX2"></a>

### Modificar el límite inferior de un control o reporte

`<Element used>.MinValue = <New value>`
---

**`<Element used>: (Nombre control o reporte)`**

Este parámetro corresponde a:

- el nombre del reporte utilizado (reporte basado en un archivo de datos).

- el nombre de control cuyo límite inferior debe ser modificado (control en una ventana o en una página).




**`<New value>: (Integro)`**

Nuevo valor del límite inferior:

- para el reporte basado en un archivo de datos.

- para un control encontrado en una ventana o página. La lista de los controles en cuestión figura en el apartado "Limitaciones. 

- para la control Valoración. Sólo se permiten los valores 0 ó 1.

### Averiguar el límite inferior definido para el filtro actual

`<Current value> = <Element used>.MinValue`
---

**`<Current value>: (Tipo correspondiente al filtro utilizado)`**



- Valor actual del límite inferior del filtro utilizado actualmente en el elemento especificado. Este filtro fue definido por [HFilter](../WDLang4/3044100.md). 

- Vaciar la cadena (""") en los siguientes casos:

	- no se ha definido ningún filtro para el elemento especificado

	- el filtro está desactivado

	- no se especificó ningún límite para el filtro







**`<Element used>: (Nombre del elemento)`**

Nombre del archivo de datos, consulta o vista utilizado.

### Averiguar el valor mínimo de una serie de gráficos

`<Minimum value> = <Series>.MinValue`
---

**`<Minimum value>: (Integro)`**

Valor mínimo de la serie. Este valor sólo existe si se ha calculado (por ejemplo, si se ha dibujado la serie).

**`<Series>: (Cadena de caracteres)`**

Series a utilizar. Utilice la siguiente notación: *&lt;Nombre del gráfico&gt;[&lt;Número de serie&gt;]*.

## Observaciones


### Límites para un reporte basado en un archivo de datos
<a name="limites_para_reporte_basado_archivo_datos_ELTPARAGRAPHE000169"></a>

Los límites seleccionan sólo los registros cuya clave de búsqueda está entre el valor del límite inferior (devuelto por el **MinValue** Property) y el valor del límite superior (devuelto por el [MaxValue](../Proprietes/2510009.md) Property).
<a name="NOTE0_2"></a>


### Límite para un reporte
<a name="limite_para_reporte_ELTPARAGRAPHE000181"></a>

El **MinValue** Property sólo puede utilizarse en informes basados en archivos de datos. Al crear la reporte, debe especificar que la navegación está limitada por el valor mínimo. 
<a name="NOTE0_3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 

### Límite para una ventana control
<a name="limite_para_una_ventana_control_ELTPARAGRAPHE000192"></a>

El valor definido para el límite inferior no puede ser mayor que el valor definido para el límite superior.. El valor máximo lo devuelve la [MaxValue](../Proprietes/2510009.md) Property.
<a name="NOTE0_4"></a>


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000202"></a>

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Esta Property puede utilizarse en los siguientes tipos de mandos:

- Control Barra de desplazamiento, 

- Columnas de control Barra de progreso y de la barra de progreso,

- Control deslizante (lineal o circular),

- Control de número, 

- Control Splitter, 

- Control Calendario, 

- control Campo de entrada numérica (no se puede introducir un número inferior a este valor),

- Duración control Campo de entrada (no se puede introducir una duración inferior a este valor),

- Control deslizante de rango, 

- Valoración.




El valor de Default para el límite superior se define en el editor de la ventana cuando se describe el control (pestaña "Detalles").




![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Esta Property puede utilizarse en los siguientes tipos de mandos: 

- controles de edición numérica (no se puede introducir un número mayor que este valor). 

- Controles de la carta.

- Controles Calendar.




El valor de Default para el límite superior se define en el editor de la ventana cuando se describe el control (pestaña "Detalles").

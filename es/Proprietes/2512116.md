


## LogBackupDirectory (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **LogBackupDirectory** permite gestionar el directorio de copia de seguridad de los archivos de registro. Es posible:

- Averigüe el directorio de copia de seguridad del registro para un archivo de datos definido en el editor de análisis o a través de la programación.

- Definir el directorio de copia de seguridad del registro para un archivo de datos (sólo cuando se describe el archivo a través de la programación).




**Observación**: Este ruta se utiliza durante la modificación de los archivos de datos del automatic para archivar los registros. [HHistoryModification](../WDLang4/3044344.md) también buscará el historial de cambios en estos archivos.
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Define the log directory of the Orders data file
Orders.LogBackupDirectory = "C:\MyFiles\MyLogs_BCK"
...
// Validate the file description
HDescribeFile(Orders)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Encontrar el directorio de respaldo de un registro de archivos

`<Current name> = <Data file>.LogBackupDirectory`
---

**`<Current name>: (Cadena de caracteres)`**

Nombre del directorio de respaldo del archivo de registro associated con el archivo de datos. Por Default, este ruta corresponde a: &lt;Directorio de archivos JNL&gt;\\__JNLBackup. Si este parámetro corresponde a un ruta relativo (y comienza con un punto), el ruta será relativo a los archivos de datos.

**`<Data file>: (Cadena de caracteres)`**

Nombre lógico del archivo de datos utilizado. Este nombre fue definido en la editor de análisis o con el tipo [Expediente description](../WDLang4/1514065.md).  


<a name="SYNTAX2"></a>

### Definir el directorio de copia de seguridad del registro para un archivo creado mediante programación

`<Data file>.LogBackupDirectory = <New name>`
---

**`<Data file>: (Cadena de caracteres)`**

Nombre lógico del archivo de datos utilizado. Este nombre fue definido por el tipo [Expediente description](../WDLang4/1514065.md).

**`<New name>: (Cadena de caracteres)`**

Copia de seguridad ruta de los archivos de registro correspondientes al archivo de datos especificado. Por Default, este ruta corresponde a: &lt;Directorio de archivos JNL&gt;\\__JNLBackup. 
Si este parámetro corresponde a una cadena vacía ("""), la posición actual no se modifica.. 
Si este parámetro corresponde a un ruta relativo (y comienza con un punto), el ruta será relativo a los ficheros de datos.

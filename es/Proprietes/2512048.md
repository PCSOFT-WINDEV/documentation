
## CtDescribedAccess (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **CtDescribedAccess** Property se utiliza para set el tipo de acceso a el archivo de datos cuando se conecta a un tabla específico a través de OLE DB. Es posible:

- get el tipo de acceso a el archivo de datos utilizado para conectarse a un tabla vía OLE DB. Este tipo de acceso se describió en la editor de análisis o cuando se describió la el archivo de datos mediante programación.
	Atención: El **CtDescribedAccess** Property ignora cualquier cambio realizado con [HConnect](../WDLang4/3044263.md).
	Este Property puede utilizarse para el un archivo de datos definido en el editor de análisis o mediante programación.

- Definir el tipo de acceso a el archivo de datos utilizado para conectarse a una tabla vía OLE DB (cuando se describe programáticamente sólo el archivo de datos). Esta Property es equivalente a la [CtAccess](../Proprietes/2512015.md) Property. 




**Recordatorio**: [HConnect](../WDLang4/3044263.md) se utiliza para redefinir uno o varios parámetros de la conexión OLE DB a un tabla. Por Default, se definen las características de la conexión OLE DB:

- en la editor de análisis durante la descripción del archivo de datos,

- cuando se describió el archivo de datos a través de la programación.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Find out the type of access to the Customer data file
ResAccessType = Customer.CtDescribedAccess
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Obtención del tipo de acceso a el archivo de datos mediante OLE DB

`<Type of access> = <Data file>.CtDescribedAccess`
---

**`<Type of access>: (Constante)`**

Devuelve el tipo de acceso al archivo de datos especificado. 
Esta información se definió durante el description del archivo de datos (en el editor de análisis o mediante programación).


|   |   |
| --- | --- |
| *hORead* | Se puede acceder al archivo de datos en modo de solo lectura. Solo se pueden leer registros. No se pueden agregar o eliminar. |
| *hOReadWrite* | Se puede acceder al archivo de datos en modo de lectura y escritura. Se pueden leer y escribir nuevos registros. |



**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado. Este nombre fue definido en la editor de análisis o con el tipo [Expediente description](../WDLang4/1514065.md).  


<a name="SYNTAX2"></a>

### Definir el tipo de acceso al archivo a través de la OLE DB

`<Data file>.CtDescribedAccess = <Type of access>`
---

**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado. Este nombre fue definido por el tipo [Expediente description](../WDLang4/1514065.md).

**`<Type of access>: (Constante)`**

Define el tipo de acceso a el archivo de datos.


|   |   |
| --- | --- |
| *hORead* | Se puede acceder al archivo de datos en modo de solo lectura. Solo se pueden leer registros. No se pueden agregar o eliminar. |
| *hOReadWrite* | Se puede acceder al archivo de datos en modo de lectura y escritura. Se pueden leer y escribir nuevos registros. |





<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
El [CtAccess](../Proprietes/2512015.md) Property se utiliza para identificar el tipo de acceso utilizado actualmente para la conexión OLE DB. Este Property también puede utilizarse para definir el tipo de acceso utilizado (cuando el el archivo de datos se describe mediante programación).

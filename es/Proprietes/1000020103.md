


## MaxLeafPerRow (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **MaxLeafPerRow** Property se utiliza para get y cambiar el número máximo de elementos en un fila en un control Organigrama. Cuando se alcanza este número máximo para el último nivel de la jerarquía, los elementos se presentan en varias filas (organigrama vertical) o en varias columnas (organigrama horizontal)..  

**Observación**: El número máximo de hojas por fila para un control Organigrama se define en la pestaña "General" de la ventana de control description ("Moverse por varias filas desde"). 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
IF CBOX_InColumn = True THEN
	ORG_Main.MaxLeafPerRow = 1
ELSE
	ORG_Main.MaxLeafPerRow = 0 // no limit
END
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Encontrar el máximo número de hojas

`<Result> = <Organization Chart control>.MaxLeafPerRow`
---

**`<Result>: (Integro)`**

Número máximo de hojas por fila. Por defecto, este valor se establece en 3. 

No hay límite si este valor corresponde a 0. 

Si este valor corresponde a 1, todas las hojas se muestran una debajo de la otra en lugar de mostrarse una al lado de la otra (organigrama vertical).. 

**`<Organization Chart control>: (Nombre del control)`**

Nombre de control Organigrama a utilizar. 


<a name="SYNTAX2"></a>

### Modificar el número máximo de hojas

`<Organization Chart control>.MaxLeafPerRow = <New value>`
---

**`<Organization Chart control>: (Nombre del control)`**

Nombre de control Organigrama a utilizar. 

**`<New value>: (Integro)`**

Número máximo de hojas por fila. Por defecto, este valor se establece en 3. 

No hay límite si este valor corresponde a 0. 

Si este valor corresponde a 1, todas las hojas se muestran una debajo de la otra en lugar de mostrarse una al lado de la otra (organigrama vertical).

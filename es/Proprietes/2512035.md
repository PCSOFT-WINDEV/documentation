
## KeyExpression (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La **KeyExpression** Property se utiliza para set los diferentes componentes de una clave compuesta. Es posible:

- Averiguar la expresión de una clave compuesta (definida en la editor de análisis o mediante programación).

- Definir la expresión de una clave compuesta (cuando el archivo de datos se describe mediante programación).







<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000002512035&name=keyexpression_property&product=WD">Ver ejemplos adicionales</a>

```wl
MyItem1 is Item Description
MyItem2 is Item Description
CompositeKey is Item Description
...
// Declare the item 1
MyItem1.Name = "Item1"
MyItem1.Type = hItemText
MyItem1.Size = 20
HDescribeItem(MyFile1, MyItem1)
// Declare the item 2
MyItem2.Name = "Item2"
MyItem2.Type = hItemText
MyItem2.Size = 20
HDescribeItem(MyFile1, MyItem2)
// Declare the composite key
CompositeKey.Name = "CompItem"
CompositeKey.KeyExpression = "Item1+Item2"
HDescribeItem(MyFile1, CompositeKey)
HDescribeFile(MyFile1)
HCreation(Temp_File)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Encontrar la expresión de una clave compuesta

`Key expression> = <Data file>.<Item>.KeyExpression`
---

**`<Key expression>: (Cadena de caracteres)`**

Nombre de los diferentes elementos incluidos en la clave compuesta. Estos nombres de elementos están separados por el carácter "+".. 
El nombre del elemento clave se devuelve si la clave no es una clave compuesta.

**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado. Este nombre fue definido en la editor de análisis o con el tipo [Expediente description](../WDLang4/1514065.md).

**`<Item>: (Cadena de caracteres)`**

Nombre de la posición clave utilizada. Este nombre fue definido en la editor de análisis o con el tipo [Artículo description](../WDLang4/1514071.md).  


<a name="SYNTAX2"></a>

### Definir la expresión de una clave compuesta

`<Item>.KeyExpression = <New expression>`
---

**`<Item>: (Cadena de caracteres)`**

Nombre de la posición clave utilizada. Este nombre fue definido por el tipo [Artículo description](../WDLang4/1514071.md).

**`<New expression>: (Cadena de caracteres)`**

Nombre de los diferentes elementos incluidos en la clave compuesta. Estos nombres de elementos están separados por el carácter "+"..

## Observaciones




### Descripción de un elemento clave compuesto mediante programación
<a name="descripcion_elemento_clave_compuesto_mediante_programacion_ELTPARAGRAPHE000080"></a>

Cuando se describe un elemento clave compuesto mediante programación, la [escribir a máquina](../Proprietes/2510131.md) Property es opcional (a diferencia de los elementos simples definidos mediante programación). Sin embargo, la composición de la clave debe describirse utilizando el **KeyExpression** Property.

Para más detalles sobre los elementos descritos mediante programación, véase [Artículo description](../WDLang4/1514071.md).

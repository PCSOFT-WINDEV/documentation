
## DisplayOrphan (Propiedad)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **DisplayOrphan** permite: 

- Determinar si una fila o columna en un control Tabla dinámica se muestra cuando no tiene padre. 

- Cambiar el modo de visualización de una fila o columna en un control Tabla dinámica cuando no tiene padre. 




Observación: Esta propiedad es inútil si la integridad de la base de datos corresponde a la definida en el análisis. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// PVT_Product_Sales is a pivot table 
// displaying the sales of products per year. 
// The products are grouped by category. 

// Displays the products associated with a category only
PVT_Product_Sales.COL_Product.DisplayOrphan = doNever 
PVTCalculateAll(PVT_Product_Sales)
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Obtener el modo de visualización actual de una fila o columna

`<Result> = <Row or column>.DisplayOrphan`
---

**`<Result>: (Constante de tipo Integer)`**

Modo de visualización de elementos huérfanos: 


|   |   |
| --- | --- |
| *doAuto*<br>(Valor predeterminado) | El modo de visualización es automático según la cardinalidad de la relación definida en el análisis. |
| *doNever* | Solo se muestran los elementos que tienen un padre. Por ejemplo, solo se mostrarán los productos asociados a una categoría. |
| *doAlways* | Se mostrarán todos los elementos, incluso los que no tienen padre.. Por ejemplo, se mostrarán todos los productos, incluso los que no están asociados a una categoría. |



**`<Row or column>: (Nombre del control)`**

Nombre de la fila o columna en el control Tabla dinámica. 


<a name="SYNTAX2"></a>

### Cambiar el modo de visualización de una fila o columna

`<Row or column>.DisplayOrphan = <New display mode>`
---

**`<Row or column>: (Nombre del control)`**

Nombre de la fila o columna del control Tabla dinámica a utilizar. 

**`<New display mode>: (Constante de tipo Integer)`**

Modo de visualización de elementos huérfanos: 


|   |   |
| --- | --- |
| *doAuto* | El modo de visualización es automático según la cardinalidad de la relación definida en el análisis. |
| *doNever* | Solo se muestran los elementos que tienen un padre. Por ejemplo, solo se mostrarán los productos asociados a una categoría. |
| *doAlways* | Se mostrarán todos los elementos, incluso los que no tienen padre.. Por ejemplo, se mostrarán todos los productos, incluso los que no están asociados a una categoría. |

## Observaciones


### Caso especial
<a name="caso_especial_ELTPARAGRAPHE000075"></a>

Para cambiar el modo de visualización de los elementos huérfanos, utilice la propiedad **DisplayOrphan** antes de calcular el contenido del control Tabla dinámica (función [PVTCalculateAll](../WDLang1/1000020539.md)).  


### Ejemplo
<a name="ejemplo_ELTPARAGRAPHE000085"></a>

El control Tabla dinámica contiene productos que no están relacionados con una categoría: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=TCD_Orphelin1.gif)


El código para mostrar este control Tabla dinámica es el siguiente: 


```wl
PVT_Sales.COL_ProductName.DisplayOrphan = doAlways
PVTCalculateAll(PVT_Statistics)
```


El mismo control Tabla dinámica puede mostrarse sin los productos que no están relacionados con una categoría: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=TCD_Orphelin2.gif)


En este caso, el código es el siguiente: 


```wl
PVT_Sales.COL_ProductName.DisplayOrphan = doNever
PVTCalculateAll(PVT_Statistics)
```

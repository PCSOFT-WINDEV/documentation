


## NdxEncryption (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
**NdxEncryption** Property se utiliza para configurar el método de cifrado para el archivo de índice (.NDX) associated con el archivo de datos. Es posible:

- Identificar el método de encriptación de un archivo de índice associated con un archivo de datos (archivo definido en el editor de análisis o mediante programación).

- Definición del modo de encriptación de un archivo de índice asociado a un archivo de datos durante su programación en description.







<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Identify the encryption method for the index file associated with the Customer data file
ResEncrypt = Customer.NdxEncryption
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Identificar el método de encriptación de un archivo de índice

`<Current encryption mode> = <Data file>.NdxEncryption`
---

**`<Current encryption mode>: (Constante de tipo Integer)`**

Los modos de encriptación son:


|   |   |
| --- | --- |
| *hEncryptionAES128* | La encriptación se realiza sólo si la contraseña es set para el archivo de datos. En este caso, el archivo índice se cifra utilizando un algoritmo AES de 128 bits con retroalimentación de salida (OFB).. |
| *hEncryptionAES192* | La encriptación se realiza sólo si la contraseña es set para el archivo de datos. En este caso, el archivo de índice se cifra mediante un algoritmo AES de 192 bits con retroalimentación de salida (OFB). |
| *hEncryptionAES256* | La encriptación se realiza sólo si la contraseña es set para el archivo de datos. En este caso, el archivo de índice se cifra mediante un algoritmo AES de 256 bits con retroalimentación de salida (OFB). |
| *hEncryptionAuto* | La encriptación se realiza sólo si la contraseña es set para el archivo de datos. En este caso, el archivo índice se encripta según un algoritmo de encriptación optimizado (RC5 16 redondos a 128 bits). |
| *hEncryptionNO* | No se realiza ninguna encriptación en el archivo índice. |
| *hEncriptaciónRC5_12* | El archivo de índice se cifra mediante un algoritmo de cifrado RC5 optimizado de 12 rondas (128 bits). |
| *hEncriptaciónRC5_16* | El archivo índice se encripta según un algoritmo de encriptación optimizado (RC5 16 rondas sobre 128 bits). |
| *hEncryptionStandard* | El archivo índice se encripta según un algoritmo de encriptación optimizado (en 128 bits). |



**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado. Este nombre se definió con el tipo [Expediente description](../WDLang4/1514065.md).

`<Data file>.NdxEncryption = <New encryption mode>`

## Observaciones


### Cifrado AES de 128, 192 y 256 bits con el modo de funcionamiento OFB
<a name="cifrado_aes_128_192_256_bits_con_modo_funcionamiento_ofb_ELTPARAGRAPHE000130"></a>

- Si el archivo de índice está encriptado con el algoritmo AES, debe utilizarse el mismo método de encriptación para los memos ([MmoCryptMethod](../Proprietes/2512014.md) Property) y los datos ([FicCryptMethod](../Proprietes/2512046.md) Property). 

- Si hay un cifrado que utiliza el algoritmo AES, la opción "Activar alto nivel de seguridad " se aplica automáticamente.

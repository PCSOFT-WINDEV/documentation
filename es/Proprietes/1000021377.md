


## WithNewButton (Propiedad)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **WithNewButton** permite: 

- Determinar si un control Pestaña dinámica tiene un botón Nueva pestaña (+).  

- Configurar un control Pestaña dinámica para que muestre un botón Nueva pestaña. 




Recordatorio: El botón Nueva pestaña permite a los usuarios finales abrir una nueva pestaña. 
<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
TAB_DynTab.WithNewButton = True
TAB_DynTab.WindowIfNew = "IW_Document"
TAB_DynTab.CaptionIfNew = "New tab"
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

### Determinar si un control Pestaña dinámica tiene un botón Nueva pestaña (+)

`<Result> = <Tab control>.WithNewButton`
---

**`<Result>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si todas las pestañas de un control Pestaña dinámica incluyen un botón Nuevo (+),

- <u><u><u><u>False</u></u></u></u> si las pestañas de un control Pestaña dinámica no incluyen un botón Nuevo. 




**`<Tab control>: (Nombre del control)`**

Nombre del control Pestaña a utilizar. 


<a name="SYNTAX2"></a>

### Configurar un control Pestaña dinámica para que muestre un botón Nueva pestaña

`<Tab control>.WithNewButton = <New tab button>`
---

**`<Tab control>: (Nombre del control)`**

Nombre del control Pestaña a utilizar. 

**`<New tab button>: (Booleano)`**



- <u><u><u><u>True</u></u></u></u> si las pestañas de un control Pestaña dinámica deben mostrar un botón Nueva pestaña,

- <u><u><u><u>False</u></u></u></u> si las pestañas de un control Pestaña dinámica no deben mostrar un botón Nueva pestaña. 






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


- El evento "Crear una pestaña" se ejecuta cuando el usuario hace clic en el botón "+". De forma predeterminada, se crea una nueva pestaña. Sin embargo, puede evitar que se creen nuevas pestañas utilizando la instrucción "RESULT False" o la función [SetFocusAndReturnToUserInput](../WDLang1/1410088107.md) en el evento "Crear una pestaña".  

- La ventana interna y el texto predeterminados que se utilizarán para la nueva pestaña se pueden definir: 

	- en la ventana de descripción del control Pestaña (pestaña "Detalles"). 

	- mediante programación con las propiedades [WindowIfNew](../Proprietes/1000021381.md) y [CaptionIfNew](../Proprietes/1000021380.md).




- Si las pestañas no tienen un botón Nueva pestaña, la funcionalidad automática de la aplicación (AAF por sus siglas en inglés) "Nuevo" se desactiva del menú contextual de las pestañas.

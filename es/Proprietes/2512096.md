


## FilterCondition (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **FilterCondition** Property permite conocer la condición de selección set por [HFilter](../WDLang4/3044100.md) sobre un archivo de datos, vista o consulta HFSQL.






<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Enable filter
HFilter(Customer, CustomerName, "D" + hMinVal, "F" + hMaxVal, "City='Paris'")
// Insert your process here
Trace(Customer.MaxValue)
Trace(Customer.MinValue)
Trace(Customer.FilterCondition) // Specifies "City=Paris"

// Disable filter
HDeactivateFilter(Customer)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

`<Selection condition> = <Element used>.FilterCondition`
---

**`<Selection condition>: (Cadena de caracteres)`**



- Condición especificada por [HFilter](../WDLang4/3044100.md),

- Vaciar la cadena (""") si no hay ningún filtro activado o si no se ha especificado ninguna condición de selección.. 


Esta condición de selección puede ser una cadena de caracteres Ansi o Unicode.

**`<Element used>: (Cadena de caracteres)`**

Nombre del archivo de datos HFSQL, consulta o vista en la que se definió el filtro.

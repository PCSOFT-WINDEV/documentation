


## MaxValue (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **MaxValue** Property se utiliza para:

- conocer y modificar el límite máximo de un **reporte basado en un archivo de datos**. El límite superior es el valor máximo permitido para la clave de búsqueda del archivo de datos HFSQL utilizado como fuente de reporte

- conocer el límite superior del **filtro actual** (definido con la función [HFilter](../WDLang4/3044100.md)) en un archivo de datos, vista o consulta HFSQL.

- obtener el valor máximo de la **serie de un gráfico**.

- averiguar y modificar el valor máximo de **algunos tipos de controles**.









<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// Modify the upper bound for row 3 of the Progress Bar column
COL_ProgressBar[3].MaxValue = 230
```


<a name="Example2"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) 
```wl
// Print the customers found between "DUPONT" and "DURANT"
RPT_MyFileReport.MinValue = "DUPONT"
RPT_MyFileReport.MaxValue = "DURANT"
```


<a name="Example3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 
```wl
// According to the upper bound defined for the edit control,
// change this upper bound
IF EDT_Price.MaxValue < 1,000,000,000 THEN
	EDT_Price.MaxValue = 2,000,000,000
END
```

## Sintaxis

### Averiguar el límite superior definido para un control o reporte

`<Current value> = <Element used>.MaxValue`
---

**`<Current value>: (Cadena de caracteres)`**

Valor actual del límite superior:

- para el reporte basado en un archivo de datos.

- para un control encontrado en una ventana o página.




**`<Element used>: (Nombre control o reporte)`**

Este parámetro corresponde a:

- el nombre del reporte utilizado (reporte basado en un archivo de datos).

- el nombre de la control con el valor del límite superior (control en una ventana o página). La lista de los controles en cuestión figura en el apartado "Limitaciones.  





<a name="SYNTAX2"></a>

### Modificar el límite superior de un control o reporte

`<Element used>.MaxValue = <New value>`
---

**`<Element used>: (Nombre control o reporte)`**

Este parámetro corresponde a:

- el nombre del reporte utilizado (reporte basado en un archivo de datos).

- el nombre de control cuyo límite superior debe ser modificado (control en una ventana o en una página).




**`<New value>: (Integro)`**

Nuevo valor del límite superior:

- para el reporte basado en un archivo de datos.

- para una ventana control. La lista de los controles en cuestión figura en el apartado "Limitaciones.

### Averiguar el límite superior definido para el filtro actual

`<Current value> = <Element used>.MaxValue`
---

**`<Current value>: (: Tipo correspondiente al filtro utilizado)`**



- Valor actual del límite superior del filtro utilizado actualmente en el elemento especificado. Este filtro fue definido por [HFilter](../WDLang4/3044100.md). 

- Vaciar la cadena (""") en los siguientes casos:

	- no se ha definido ningún filtro para el elemento especificado.

	- el filtro está desactivado.

	- no se especificó ningún límite para el filtro.







**`<Element used>: (Cadena de caracteres)`**

Nombre del archivo de datos, consulta o vista utilizado.

### Averiguar el valor máximo de una serie de gráficos

`<Maximum value> = <Series>.MaxValue`
---

**`<Maximum value>: (Integro)`**

Valor máximo de la serie. Este valor sólo existe si se ha calculado (por ejemplo, si se ha dibujado la serie).

**`<Series>: (Cadena de caracteres)`**

Series a utilizar. Utilice la siguiente notación: *&lt;Nombre del gráfico&gt;[&lt;Número de serie&gt;]*.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Límites para un reporte basado en un archivo de datos
<a name="limites_para_reporte_basado_archivo_datos_ELTPARAGRAPHE000174"></a>

Los límites seleccionan sólo aquellos registros cuya clave de búsqueda se encuentra entre el valor del límite inferior (devuelto por el [MinValue](../Proprietes/2510008.md) Property) y el valor del límite superior (devuelto por el **MaxValue** Property).
<a name="NOTE0_2"></a>


### Límite para un reporte
<a name="limite_para_reporte_ELTPARAGRAPHE000186"></a>

El **MaxValue** Property sólo puede utilizarse en informes basados en archivos de datos HFSQL. Al crear la reporte, debe especificar que la navegación está limitada por el valor máximo.
<a name="NOTE0_3"></a>
![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png)![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png)![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png)![Código de Usuario (UMC)](https://doc.pcsoft.fr/ext/images/us/MCU.png) 

### Límite para una ventana control
<a name="limite_para_una_ventana_control_ELTPARAGRAPHE000197"></a>

El valor definido para el límite superior no puede ser inferior al valor definido para el límite inferior.. El valor mínimo lo devuelve la [MinValue](../Proprietes/2510008.md) Property.
<a name="NOTE0_4"></a>


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000207"></a>

![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Esta Property puede utilizarse en los siguientes tipos de mandos:

- Control Barra de desplazamiento, 

- Columnas de control Barra de progreso y de la barra de progreso,

- Control deslizante (lineal o circular),

- Control de número, 

- Control Splitter, 

- Control Calendario, 

- control Campo de entrada numérica (no se puede introducir un número mayor que este valor),

- Duración control Campo de entrada (no se puede introducir una duración superior a este valor),

- Control deslizante de rango, 

- Control Valoración.




El valor de Default para el límite superior se define en el editor de la ventana cuando se describe el control (pestaña "Detalles").





![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) Esta Property puede utilizarse en los siguientes tipos de mandos: 

- controles de edición numérica (no se puede introducir un número mayor que este valor). 

- controles de edición de la duración (no se puede introducir una duración superior a este valor),

- Controles de la carta.

- Controles Calendar.

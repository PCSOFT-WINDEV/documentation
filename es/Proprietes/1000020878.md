


## Process (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **Process** Property se utiliza para modificar los Process es que serán ejecutados por los eventos disponibles en un control, ventana, Page, reporte, etc.

Existen varios tipos de operaciones disponibles:  

- El operador "=" borra todos los procesos y Procedure s adjuntos al evento y los reemplaza con la Procedure especificada.

- El operador '+=' añade el Procedure especificado después del Process inicial y después de los Procedure s añadidos con anterioridad.

- Las propiedades [Before](../Proprietes/1000020916.md) y [After](../Proprietes/1000020917.md) sirven para gestionar con mayor precisión los procedimientos a ejecutar.

- Para salir de la secuencia de procedimientos, utilice [SetFocusAndReturnToUserInput](../WDLang1/1410088107.md).





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Specify the process to run on a click
BTN_BUTTON1.Process[trtClick] = myProcedure1

// Add a procedure to run on a click
BTN_BUTTON1.Process[trtClick] += myProcedure2

// Add a procedure to run before the click process
BTN_BUTTON1.Process[trtClick].Before += myProcedure3
```

<a name="XSYNTAX"></a>

## Sintaxis
<a name="SYNTAX1"></a>

`<Element>.Process [<Type of process>] = <Procedure name>`
---

**`<Element>: (Cadena de caracteres)`**

Nombre del elemento a utilizar: 

- Nombre del control o nombre del Control Variable a utilizar. 

- Nombre de la Page que se va a utilizar. 

- Nombre de la ventana a utilizar. 

- Nombre de la reporte que se va a utilizar.

## Observaciones


### Caso de eventos opcionales
<a name="caso_eventos_opcionales_ELTPARAGRAPHE001041"></a>

Si el Event modificado por el **Process** Property es un Event opcional del control (ventana o página), este Event se crea automáticamente: no hay necesidad de añadir esta Event en el editor de código. 
<a name="NOTE0_2"></a>


### Caso de eventos que esperan parámetros
<a name="caso_eventos_que_esperan_parametros_ELTPARAGRAPHE001050"></a>

Algunos eventos utilizan parámetros (por ejemplo, los eventos de los controles del Organizador o del Programador). El procedimiento definido por la propiedad **Process** no debe esperar más parámetros que el asociado Event( sin embargo, puede esperar menos parámetros). 
<a name="NOTE0_3"></a>


### Uso de la palabra clave MySelf
<a name="uso_palabra_clave_myself_ELTPARAGRAPHE001059"></a>

Si la palabra clave [MySelf](../Motscles/1511007.md) se utiliza en un Procedure llamado por la propiedad **Process**, corresponderá al control que ejecuta el Procedure.  
<a name="NOTE0_4"></a>


### Salir de una secuencia de procedimientos
<a name="salir_una_secuencia_procedimientos_ELTPARAGRAPHE001073"></a>

Si se ejecutan varios procedimientos en una secuencia, esta puede interrumpirse desde el código de uno de los procedimientos utilizando la función [SetFocusAndReturnToUserInput](../WDLang1/1410088107.md). 

Ejemplo: 


```wl
// Click on BTN_Button1
BTN_Button1.Process[trtClick] += MyProc1
BTN_Button1.Process[trtClick] += MyProc2
```



```wl
// Click on BTN_Button1
Trace("Click on Button 1")
```



```wl
PROCEDURE MyProc1()
Trace(dbgInfo(dbgProcess))
SetFocusAndReturnToUserInput(MySelf.Name)
```



```wl
PROCEDURE MyProc2()
Trace(dbgInfo(dbgProcess))
```

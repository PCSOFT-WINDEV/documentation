


## FicEncryption (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **FicEncryption** Property se usa para set el método de cifrado de un archivo de datos. Es posible:

- get el método de encriptación de un archivo de datos (.FIC) (archivo definido en el editor de análisis o programáticamente).

- Definir el método de encriptación de un archivo de datos mediante programación.





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Find out the encryption mode of the Customer data file
ResEncrypt = Customer.FicEncryption
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Descubriendo el modo de encriptación de un archivo de datos

`<Current encryption mode> = <Data file>.FicEncryption`
---

**`<Current encryption mode>: (Constante de tipo Integer)`**

Los modos de encriptación son:



|   |   |
| --- | --- |
| *hEncryptionAES128* | La encriptación se realiza sólo si la contraseña es set para el archivo de datos. En este caso, el archivo de datos está encriptado usando un algoritmo AES de 128 bits con Output feedback (OFB). |
| *hEncryptionAES192* | La encriptación se realiza sólo si la contraseña es set para el archivo de datos. En este caso, el archivo de datos se encripta mediante un algoritmo AES de 192 bits con retroalimentación de salida (OFB). |
| *hEncryptionAES256* | La encriptación se realiza sólo si la contraseña es set para el archivo de datos. En este caso, el archivo de datos se encripta mediante un algoritmo AES de 256 bits con retroalimentación de salida (OFB). |
| *hEncryptionAuto* | La encriptación se realiza sólo si la contraseña es set para el archivo de datos. En este caso, el archivo de datos está encriptado según un algoritmo de cifrado optimizado (RC5 16 rondas en 128 bits). |
| *hEncryptionNO* | No se realiza ninguna encriptación en el archivo de datos. |
| *hEncriptaciónRC5_12* | El archivo de datos está encriptado según un algoritmo de cifrado optimizado (RC5 12 rondas en 128 bits). |
| *hEncriptaciónRC5_16* | El archivo de datos está encriptado según un algoritmo de cifrado optimizado (RC5 16 rondas en 128 bits). |
| *hEncryptionStandard* | El archivo de datos está encriptado según un algoritmo de cifrado optimizado en 128 bits. |



**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado. Este nombre se definió en el editor de análisis o con el tipo [Expediente description](../WDLang4/1514065.md).

## Observaciones


### Cifrado AES de 128, 192 y 256 bits con el modo de funcionamiento OFB
<a name="cifrado_aes_128_192_256_bits_con_modo_funcionamiento_ofb_ELTPARAGRAPHE000130"></a>

- Si el archivo de datos se encripta con el algoritmo AES, es necesario utilizar el mismo método para encriptar los memos ([MmoCryptMethod](../Proprietes/2512014.md) Property) y los índices ([NdxCryptMethod](../Proprietes/2512010.md) Property). 

- Si hay un cifrado que utiliza el algoritmo AES, la opción "Activar alto nivel de seguridad " se aplica automáticamente. En este modo, una contraseña debe ser set para el archivo de datos. Esta contraseña será solicitada en cada modificación de la automatic del archivo de datos.




## CacheSize (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **CacheSize** Property se utiliza para:

- get el número máximo de registros que se pueden almacenar en la caché del Conector Nativo de MySQL.

- Cambiar el número máximo de registros que se pueden almacenar en la caché del Conector Nativo de MySQL. En este caso, si la conexión está actualmente abierta, se cerrará y se reabrirá con los nuevos parámetros durante el primer acceso a un archivo que utilice esta conexión.




**Observación**: La conexión puede corresponder a:

- una conexión descrita en la editor de análisis,

- una conexión descrita por [HConnect](../WDLang4/3044263.md), [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md).

- una conexión descrita por un [Conexión](../WDLang4/1514073.md) variable.

## Sintaxis

### Obtención del tamaño de la caché de una conexión

`<Cache size> = <Connection used>.CacheSize`
---

**`<Cache size>: (Integro)`**

Tamaño de la caché del Conector Nativo MySQL.

Si este parámetro corresponde a:

- -2: el tamaño de la caché es el tamaño predeterminado (100 para el Conector Nativo MySQL).

- -1: el tamaño de la caché es ilimitado.

- 0: la caché no almacena ningún registro. No se recupera ningún Record.




**`<Connection used>: (Cadena de caracteres o variable de tipo Connection)`**

Nombre de la conexión utilizada. Este nombre fue definido:

- en el editor de análisis, 

- por [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md),

- por el tipo [Conexión](../WDLang4/1514073.md).  





<a name="SYNTAX2"></a>

### Definir el tamaño de la caché para una conexión

`<Connection used>.CacheSize = <Cache size>`
---

**`<Connection used>: (Cadena de caracteres o variable de tipo Connection)`**

Nombre de la conexión utilizada. Este nombre fue definido:

- en el editor de análisis, 

- por [HDescribeConnection](../WDLang4/3044205.md) o [HOpenConnection](../WDLang4/3044107.md),

- por el tipo [Conexión](../WDLang4/1514073.md).




**`<Cache size>: (Integro)`**

Nuevo tamaño de la caché del Conector Nativo MySQL.

Si este parámetro corresponde a:

- -2: el tamaño de la caché es el tamaño predeterminado (100 para el Conector Nativo MySQL).

- -1: el tamaño de la caché es ilimitado.

- 0: la caché no almacena ningún registro. No se recuperará ningún Record.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Equivalencia
<a name="equivalencia_ELTPARAGRAPHE000101"></a>

[CtCacheSize](../Proprietes/2512027.md) se utiliza para definir el tamaño de la caché de una conexión MySQL. Este Property se mantiene para la compatibilidad retroactiva. Se recomienda utilizar el **CacheSize** Property.
<a name="NOTE0_2"></a>


### Recuperación de los registros
<a name="recuperacion_los_registros_ELTPARAGRAPHE000112"></a>

La velocidad a la que se recuperan los registros es proporcional al tamaño de la caché: cuanto mayor sea el tamaño de la caché, más rápido se recuperarán los registros.

Sin embargo, cuanto menor sea el tamaño de la caché, más rápido se leerá el primer registro.

**Atención**: La memoria utilizada por la caché es proporcional al tamaño de la misma.
<a name="NOTE0_3"></a>

### Gestión del CacheSize Property
<a name="gestion_del_docparampagetitleshort_property_ELTPARAGRAPHE000125"></a>

La propiedad **CacheSize** solo es admitida por el Conector Nativo MySQL.

Otros Conectores Nativos no admiten la propiedad **CacheSize**. En ese caso, se ignorará el uso de esta propiedad y no se generará ningún error.
<a name="NOTE0_4"></a>


### Constante hQueryWithoutCorrection
<a name="constante_hquerywithoutcorrection_ELTPARAGRAPHE000140"></a>

Si se ejecuta la consulta sin verificar el código SQL (constante *hQueryWithoutCorrection*): el tamaño es ilimitado y no se puede cambiar con el **CacheSize** property.

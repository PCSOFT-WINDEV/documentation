


## Detection (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **Detection** se utiliza para determinar y cambiar la detección automatic de los flujos entrantes. Si la detección de flujo está activada, los flujos permiten la detección. 

La detección de corrientes de automatic está habilitada por Default.

Observación: Esta propiedad es equivalente a la opción "Detección de corrientes entrantes activada" que se encuentra en la ventana de corrientes de description. Para abrir la ventana de gestión de flujos, en la pestaña Ventana", en el grupo "Edición", despliegue "Otras acciones" y seleccione "Editar flujos".. Esta opción está disponible si se encuentra una Conferencia en la ventana.<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=Flux_Detection.gif)
A partir de la versión 20, los controles de conferencia son compatibles con Windows Vista y posteriores. Si desea utilizar la control Conferencia con una versión anterior de Windows: 

1. Abra la ventana de descripción del control. 

2. En la pestaña "UI", marque "Usar la TAPI 3.x APIs / Windows XP (para compatibilidad)". 

3. Valide.



<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Allows the detection of incoming streams
MyStream.Detection = Active
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Averiguar el modo de detectar los flujos entrantes

`<Result> = <Stream name>.Detection`
---

**`<Result>: (Constante de tipo Integer)`**

Modo de gestión actual de los flujos entrantes:


|   |   |
| --- | --- |
| *Active* | automatic detección de corrientes entrantes. El procedimiento para la detección de corrientes entrantes de automatic se ejecuta para cada nueva corriente entrante. |
| *Inactive* | No hay detección de automatic de los flujos entrantes. El procedimiento para la detección de corrientes entrantes de automatic se ignora para cada nueva corriente entrante. |



**`<Stream name>: (Cadena de caracteres)`**

Nombre del arroyo o nombre del control Conferencia associated con el arroyo.  


<a name="SYNTAX2"></a>

### Modificando el modo de detección de los flujos entrantes

`<Stream name>.Detection = <New mode>`
---

**`<Stream name>: (Cadena de caracteres)`**

Nombre del arroyo o nombre del control Conferencia associated con el arroyo.

**`<New mode>: (Constante de tipo Integer)`**

Nuevo modo de gestión de los flujos entrantes:


|   |   |
| --- | --- |
| *Active* | automatic detección de corrientes entrantes. El procedimiento para la detección de corrientes entrantes de automatic se ejecuta para cada nueva corriente entrante. |
| *Inactive* | No hay detección de automatic de los flujos entrantes. El procedimiento para la detección de corrientes entrantes de automatic se ignora para cada nueva corriente entrante. |

## Observaciones


### Utilice el ejemplo
<a name="utilice_ejemplo_ELTPARAGRAPHE000082"></a>

Una aplicación permite que el usuario no sea interrumpido durante una videoconferencia.. Esta opción utiliza la propiedad **Detection** para bloquear la detección automatic de las llamadas entrantes.

En este caso, todas las llamadas entrantes se ignoran. El automatic Procedure para detectar las llamadas entrantes no se ejecuta.

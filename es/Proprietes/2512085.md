


## CaseSensitive (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **CaseSensitive** Property se utiliza para determinar si las diferentes búsquedas realizadas en el artículo o en el índice de texto completo distinguen entre mayúsculas y minúsculas (con [HReadSeek](../WDLang4/3044050.md), [HReadSeekFirst](../WDLang4/3044036.md) y [HReadSeekLast](../WDLang4/3044034.md), por ejemplo). Es posible:

- Averigüe si las búsquedas en este artículo (o índice de texto completo) distinguen entre mayúsculas y minúsculas. El artículo o el índice de texto completo se ha definido en la editor de análisis o mediante programación.

- Activar o desactivar la distinción entre mayúsculas y minúsculas en un artículo o índice de texto completo (cuando el artículo o índice de texto completo se describe mediante programación).





<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Define how the case is managed in the Orders.Name item
Orders.Name.CaseSensitive = True
...
// Validate the data file description
HDescribeFile(Orders)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Averiguar si la sensibilidad a las mayúsculas y minúsculas está activada

`<Result> = <Data file>.<Item>.CaseSensitive`
---

**`<Result>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si las mayúsculas y minúsculas se tienen en cuenta para las búsquedas realizadas en este elemento,

- <u><u><u><u>False</u></u></u></u> en caso contrario.




**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado. Este nombre fue definido en la editor de análisis o con el tipo [Expediente description](../WDLang4/1514065.md).

**`<Item>: (Cadena de caracteres)`**



- Nombre del artículo utilizado. Este nombre fue definido en la editor de análisis o con el tipo [Artículo description](../WDLang4/1514071.md).

- Nombre del índice de texto completo utilizado. Este nombre fue definido en la editor de análisis o con el tipo [FullTextIndex Description](../WDLang4/1000017461.md).  





<a name="SYNTAX2"></a>

### Definición del modo de gestión del caso (elemento descrito mediante programación)

`<Item>.CaseSensitive = <New management mode>`
---

**`<Item>: (Cadena de caracteres)`**



- Nombre del artículo utilizado. Este nombre fue definido por el tipo [Artículo description](../WDLang4/1514071.md).

- Nombre del índice de texto completo utilizado. Este nombre fue definido por el tipo [FullTextIndex Description](../WDLang4/1000017461.md).




**`<New management mode>: (booleano)`**



- <u><u><u><u>True</u></u></u></u> si las mayúsculas y minúsculas deben tenerse en cuenta para las búsquedas realizadas en este elemento,

- <u><u><u><u>False</u></u></u></u> en caso contrario.  






<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Búsqueda realizada sobre un elemento "case-sensitive
<a name="busqueda_realizada_sobre_elemento_casesensitive_ELTPARAGRAPHE000107"></a>

Si se realiza una búsqueda en un elemento que distingue entre mayúsculas y minúsculas, solo se encontrarán las palabras que coincidan con los caracteres en minúsculas y mayúsculas.

Por lo tanto, si el nombre del cliente es "Smith":

- una búsqueda realizada en "Smith" encontrará una Record,

- una búsqueda realizada en "SMITH" no encontrará ninguna Record.



<a name="NOTE0_2"></a>


### Configuración de las opciones de búsqueda de un elemento
<a name="configuracion_las_opciones_busqueda_elemento_ELTPARAGRAPHE000119"></a>

- Para que las búsquedas sean sensibles o insensibles a los acentos, utilice el botón [AccentSensitive](../Proprietes/2512077.md) Property.

- Para que las búsquedas distingan entre mayúsculas y minúsculas o no, utilice los códigos **CaseSensitive** Property.

- Para que las búsquedas sean sensibles o insensibles a la puntuación, utilice el botón [PuntuaciónSensible](../Proprietes/2512087.md) Property.

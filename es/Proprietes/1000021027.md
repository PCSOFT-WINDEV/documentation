


## HierarchicalDepth (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
El **HierarchicalDepth** Property se utiliza para:

- Averiguar la profundidad de una control jerárquica en una reporte. 

- Modificar la profundidad de una control jerárquica en una reporte (sólo si la reporte utiliza una fuente de datos mediante programación). 




Esta Property sólo puede ser usada en los eventos de una "Reporte on TreeView Table".


<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// -- Pre-print process of Body block
IF ITEM_Col_Customer.HierarchicalDepth = 1 THEN
	ITEM_Col_Customer.BackgroundColor = iLightGreen
ELSE
	ITEM_Col_Customer.BackgroundColor = iWhite
END
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Averiguar el nivel de una control jerárquica

`<Result> = <Control used>.HierarchicalDepth`
---

**`<Result>: (Integro)`**

Profundidad de la jerárquica control: 0 para la raíz, 1 para el primer nivel, ....

**`<Control used>: (Nombre del control)`**

Nombre de la control jerárquica a utilizar.  


<a name="SYNTAX2"></a>

### Modificar el nivel de una control jerárquica

`<Control used>.HierarchicalDepth = <New depth>`
---

**`<Control used>: (Nombre del control)`**

Nombre de la control jerárquica a utilizar. Esta reporte debe utilizar una fuente de datos mediante programación. 

**`<New depth>: (Integro)`**

Nueva profundidad para la jerárquica control: 0 para la raíz, 1 para el primer nivel, ... 



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones
El **HierarchicalDepth** Property se aplica sólo a los informes basados en controles TreeView.

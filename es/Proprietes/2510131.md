


## Type (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **Type** permite obtener:

- el tipo de elemento (archivo de datos o campo) definido mediante programación o en el editor de análisis, vista o consulta.

- el tipo de objeto.

- el tipo de datos que se almacena en un Variant.
	**Observación**: Para cambiar el tipo de datos almacenado en un Variant, utilice la función [VariantConvert](../WDLang1/3013054.md).




La propiedad **Type** define el tipo de un objeto dinámico.

<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo
<a class="notetitle" target="_blank" href="$DOC$=1000002510131&name=type_property&product=WD">Ver ejemplos adicionales</a>

```wl
MyVariant is variant = "5"
Info(MyVariant..Type) // Return 19
```
<a name="Example2"></a>

```wl
// Find out the type of "Contact" item
ResType = Customer.Contact.Type
// Find out the type of the Customer data file
ResType = Customer.Type
```

### Definir los campos mediante programación
<a name="definir_los_campos_mediante_programacion_ELTPARAGRAPHE002531"></a>

Cuando se describen campos mediante programación (tipo [Artículo description](../WDLang4/1514071.md)), se debe utilizar la propiedad **Type** para todos los campos, excepto para los campos de tipo "Clave compuesta". Para este tipo de campos, utilice la propiedad [KeyExpression](../Proprietes/2512035.md).

En el caso de las partidas de tiempo, hay que utilizar la [envergadura](../Proprietes/2510125.md) Property para definir el formato de la partida (hora de 2, 4, 6 u 8 caracteres).
Para los campos de tipo Combination o Enumeration, utilice la propiedad [TypeDescription](../Proprietes/1000021596.md) para definir el tipo utilizado por el campo.

Los siguientes tipos pueden crearse mediante programación con la función [HDescribeFile](../WDLang4/3044206.md): 


|   |   |
| --- | --- |
| *hFileAS400* | Archivo AS400 |
| *hFileNormal* | Archivo estándar |
| *hFileOLEDB* | Archivo OLE DB |
| *hFileOracle* | Archivo Oracle |
| *hFileSQLServer* | Archivo SQL Server |
| *hFileSybase* | Archivo Sybase |
| *hFileXBase* | Archivo xBase |
| *hFileXML* | Archivo XML |



**Atención**: Para WINDEV Mobile, solo están disponibles los siguientes tipos:


|   |   |
| --- | --- |
| *hFileNormal* | Archivo estándar |
| *hFileOracle* | Archivo Oracle |
| *hFileSQLServerMobile* | Archivo SQL Server Mobile |

### Modo de relleno
<a name="modo_relleno_ELTPARAGRAPHE002614"></a>

Para determinar cómo se rellena un control Tabla, List Box o Combo Box (mediante programación, con datos de archivos de datos o variables), utilice la propiedad [FillType](../Proprietes/1000017723.md).

### Tipo de celda de un control Tabla
<a name="tipo_celda_control_tabla_ELTPARAGRAPHE002626"></a>

Al utilizar la propiedad **Type** en una celda de un control Tabla, se obtiene el tipo de la columna que contiene la celda.
<a name="NOTE0_4"></a>


### Limitación
<a name="limitacion_ELTPARAGRAPHE002635"></a>

Para los controles de una ventana o página, la propiedad **Type** no puede utilizarse en:

- una opción de un control Casilla de verificación o Botón de opción,

- una fila de un control Tabla o Combo Box,

- una fila de un control List Box,

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) una pestaña de un control Pestaña o Barra lateral.


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) En la versión actual, la propiedad **Type** solo se aplica a los controles de una ventana.

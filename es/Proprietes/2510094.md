


## Plane (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **Plane** permite:

- get el plano activo de una ventana (o reporte) o el plano al que pertenece un control.

- Modificar el plano de Active de una ventana (o reporte) o asociar un control con otro plano.




Observaciones: 

- Un plano es una "capa": 

	- de una ventana, 

	- de una reporte.


No se pueden visualizar varios planos al mismo tiempo (en el editor de ventanas, el editor Page, editor de reportes, o en sus aplicaciones).

- En una ventana o en una reporte, cada control puede:

	- no pertenecen a ningún avión. Esta control se mostrará siempre, independientemente del plano actual.

	- pertenecen a un avión. Este control se mostrará cuando el avión correspondiente sea Visible.

	- pertenecen a varios planos. Este control se mostrará cuando uno de los aviones associated con el control esté activado (lo que significa Visible).




- Los aviones de los informes se utilizan principalmente para sustituir los bloques de sustitución.: simplemente posicione los controles en varios planos y active el plano deseado cuando imprima.

## Sintaxis

### get el plano activo de una ventana (o reporte) o el plano al que pertenece un control

`<Plane number> = <Element used>.Plane`
---

**`<Plane number>: (Entero o cadena de caracteres)`**



- Número del avión Active en la ventana especificada.

- Número del avión Active en el reporte especificado.

- Número del avión al que pertenece el control especificado.

- Números de los aviones (separados por una coma) a los que pertenece la control.




**`<Element used>: (Nombre de control, ventana o reporte)`**

Nombre del elemento a utilizar: 

- Nombre del control. 

- Nombre de la ventana. 

- Nombre del reporte. 





<a name="SYNTAX2"></a>

### Modificar el plano activo de una ventana (o reporte) o asociar un control a otro plano

`<Element used>.Plane = <New plane>`
---

**`<Element used>: (Nombre de control, ventana o reporte)`**

Nombre del elemento a utilizar: 

- Nombre del control. 

- Nombre de la ventana. 

- Nombre del reporte. 




**`<New plane>: (Entero o cadena de caracteres)`**



- Número del plano a activar. Si este número es mayor que el número de planos existentes, sólo se mostrarán los controles no asignados a un plano.

- Número del avión associated con el control especificado. El control ya no se mostrará si este número no corresponde al plano actual.  

- Números de los aviones associated con el control. Estos números se suministran en formato de cadena de caracteres y están separados por una coma.

- ![WINDEV](https://doc.pcsoft.fr/ext/images/us/WD.png) Número del plano a activar (sólo para las ventanas). El nombre del avión se definió en la [ventana de gestión de planos](../WDChamp/1013078.md) en el editor de ventanas.

## Observaciones

### Plano y ventana
<a name="plano_ventana_ELTPARAGRAPHE000230"></a>

- Si no se asigna un control a un plano (Opción &lt;Ninguno&gt; seleccionada en la ventana de descripción), el control siempre se mostrará, independientemente del plano activo en la ventana o reporte.

- Si se asigna un control a un avión, este control se mostrará sólo si pertenece al avión Active en la ventana o reporte.

- Si se asigna un control a varios planos, este control se mostrará tan pronto como uno de estos planos se habilite (lo que significa Visible).

- Al abrir una ventana o reporte, el avión de Active es el avión #0.

### Plano y ventana interna
<a name="plano_ventana_interna_ELTPARAGRAPHE000262"></a>

Se puede utilizar el **Plane** Property: 

- directamente en la ventana interior. En este caso, esta Property maneja los controles de la ventana interna. 
	Por ejemplo, para visualizar el plano 2 de la ventana interna desde la ventana host o directamente en la ventana interna, utilice la siguiente Line de código: 
	
	```txt
	<Name of internal window>.Plane = 2
	```


- directamente en la ventana que alberga la control Ventana interna. En este caso, la Property sólo se aplicará a los controles de la ventana anfitriona (incluida la control Ventana interna si es associated con un plano). El contenido de la ventana interna no se modificará. Debe utilizarse la siguiente Line de código: 
	
	```txt
	<Window name>.Plane = 2
	```




<a name="NOTE0_3"></a>


### Avión y reporte interna
<a name="avion_reporte_interna_ELTPARAGRAPHE000282"></a>

Se puede utilizar el **Plane** Property: 

- directamente en el interior de reporte. En este caso, esta Property maneja los controles de la reporte interna . 
	Por ejemplo, para mostrar el plano 2 del reporte interno desde el reporte anfitrión o directamente en el reporte interno, use el siguiente código Line:
	
	```txt
	<Name of internal report>.Plane = 2
	```


- directamente en la ventana que alberga la Reporte control interna . En este caso, la Property sólo se aplicará a los mandos del reporte anfitrión (incluido el mando Reporte interno si es associated con plano). El contenido de la reporte interna no se modificará. Debe utilizarse la siguiente Line de código: 
	
	```txt
	<Report name>.Plane = 2
	```




<a name="NOTE0_4"></a>


### Limitaciones
<a name="limitaciones_ELTPARAGRAPHE000300"></a>

La propiedad **Plane** no se puede utilizar en:

- una opción de un control Casilla de verificación o Botón de opción,

- una fila en una tabla, list box o control Combo Box,

- una celda de un control Tabla,

- una columna de un control Tabla,

- un menú de ventana,

- un panel de pestañas,

- un grupo de controles,


![Java](https://doc.pcsoft.fr/ext/images/us/JAVA.png) La propiedad **Plane** solo se aplica a los siguientes elementos:

- Control Botón.

- Control Estático.

- Control Campo de entrada.

- Control Imagen.

- Control Casilla de verificación.

- Control Botón de opción.

- Control List Box.

- Control Combo Box.

- Control TreeView.

- Control Pestaña.

- Control de ventana.

- Control Tabla.

- Control Supercontrol.


![Universal Windows 10 App](https://doc.pcsoft.fr/ext/images/us/UNIVERSALAPP.png) El **Plane** Property no está disponible para informes.

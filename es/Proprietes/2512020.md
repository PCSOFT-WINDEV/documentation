


## Component (Property)

*** ***
	



<a name="XUse"></a>
<a name="Use"></a>
<a name="description"></a>
La propiedad **Component** configura los diferentes componentes de una clave compuesta. Es posible:

- Averiguar la enésima componente de una clave compuesta (definida en la editor de análisis o mediante programación).

- Definir el componente N de una clave compuesta (Item description Variable).







<a name="Example1"></a>
<a name="sample_code"></a>

## Ejemplo


```wl
// Define the second component of the composite key
LastFirstName.Component[2] = MaidenName
...
// Validate the data file description
HDescribeFile(Customer)
```

<a name="XSYNTAX"></a>
<a name="SYNTAX1"></a>

## Sintaxis

### Descubrir el enésimo componente de una clave compuesta

`<Component name> = <Data file>.<Item>.Component[<N>]`
---

**`<Component name>: (Cadena de caracteres)`**

Nombre del elemento correspondiente al componente buscado de la clave compuesta. 
Si N = 1 y si &lt;Item&gt; corresponde a una clave simple, &lt;Resultado&gt; corresponde al nombre de la clave.

**`<Data file>: (Cadena de caracteres)`**

Nombre del archivo de datos utilizado. Este nombre fue definido en la editor de análisis o con el tipo [Expediente description](../WDLang4/1514065.md).

**`<Item>: (Cadena de caracteres)`**

Nombre de la posición clave utilizada. Este nombre fue definido en la editor de análisis o con el tipo [Artículo description](../WDLang4/1514071.md).

**`<N>: (Integro)`**

Número del componente cuyo nombre se solicita. El número de elementos de una clave compuesta es devuelto por la [NbComponent](../Proprietes/2512058.md) Property.   


<a name="SYNTAX2"></a>

### Definir el enésimo componente de una clave compuesta

`<Item>.Component[<N>] = <New component name>`
---

**`<Item>: (Cadena de caracteres)`**

Nombre de la posición clave utilizada. Este nombre fue definido por el tipo [Artículo description](../WDLang4/1514071.md).

**`<N>: (Integro)`**

Número del componente que se debe modificar o añadir. El número de elementos de una clave compuesta es devuelto por la [NbComponent](../Proprietes/2512058.md) Property.   

**`<New component name>: (Cadena de caracteres)`**

Nombre de la posición que se debe utilizar como componente de la clave compuesta.  



<a name="NOTE0"></a>
<a name="NOTE0_1"></a>

## Observaciones


### Modificación de una clave compuesta
<a name="modificacion_una_clave_compuesta_ELTPARAGRAPHE000089"></a>

La modificación de una llave compuesta puede permitirle añadir nuevos componentes a la llave compuesta durante su description mediante la programación. Los diferentes componentes deben estar numerados en orden ascendente y no se debe encontrar ninguna ranura "vacía" (por ejemplo, no se puede pasar del componente 4 al componente 6).. De lo contrario, se produce un error WLanguage.

Observación: Se recomienda utilizar la [KeyExpression](../Proprietes/2512035.md) Property para construir una clave compuesta en el código.
<a name="NOTE0_2"></a>


### Gestión de errores
<a name="gestion_errores_ELTPARAGRAPHE000101"></a>

Un error fatal de Error( WLanguage) ocurre en los siguientes casos:

- componente solicitado mayor que el número de componentes encontrados en la clave,

- escribir un componente que no siga inmediatamente al último componente de la clave compuesta.

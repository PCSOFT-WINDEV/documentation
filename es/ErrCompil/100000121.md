
## Error 121: Siguiendo una estructura
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>
Se declaró una estructura: ya existe una estructura con el mismo nombre en la ventana o proyecto actual.

Si se mantiene este nombre, la estructura "global" ya no será visible en la actual Process: se utilizará la estructura local.

Atención: Si se redefine una estructura global para el proyecto en el código de inicialización de la ventana, la nueva estructura se utilizará en todos los procesos de los controles de ventana y ventana.

**Recordatorio**: Alcance de las estructuras

- Las estructuras declaradas en el código de inicialización del proyecto son Visible en todo el proyecto. 

- Las estructuras declaradas en el código de apertura de las ventanas son Visible en todos los procesos de la ventana y los controles de la ventana. 

- Las estructuras declaradas en el código de declaración de las clases son Visible en todos los métodos de la clase. 




**Diferencias con la versión 5.5**:

En WINDEV 5.5, una estructura permanece pública tan pronto como se declara. Si se declara una estructura en la apertura de Process de una ventana, ésta permanece pública y es reconocida por todos los demás Process es aunque la ventana donde fue declarada esté cerrada.

De WINDEV 11, esta afirmación ya no es True: una estructura declarada en el código de apertura de la ventana es accesible mientras la ventana esté abierta.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Consejo
<a name="consejo_ELTTEXTE000104"></a>
Evite utilizar los mismos identificadores para estructuras diferentes.

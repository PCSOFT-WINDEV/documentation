
## Error 116: Una dimensión array debe ser un valor entero
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Motivo
<a name="motivo_ELTTEXTE000103"></a>
Está utilizando un array: las dimensiones de array no son válidas. Se debe utilizar un valor entero para definir la dimensión.

**Recordatorio**: Al declarar una array, se deben utilizar los valores INTEGER para definir las dimensiones de array.

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Corrección
<a name="correccion_ELTTEXTE000127"></a>
Modificar el tipo utilizado para definir las dimensiones de array. Utilice un número entero.

<a name="NOTE4"></a>
<a name="NOTE4_1"></a>


## Ejemplos
<a name="ejemplos_ELTTEXTE000151"></a>


### Usar un valor numérico incorrecto: este valor no corresponde al tipo esperado
<a name="usar_valor_numerico_incorrecto_este_valor_corresponde_tipo_esperado_ELTPARAGRAPHE000027"></a>

**Código que desencadena el error** 


```wl
MyArray is array of 5.5 int
```


**Posibles correcciones**

Para utilizar una array bidimensional, separe las dimensiones de array con una coma: sustitúyase el punto 5.5 por el texto siguiente **5,5**.


```wl
MyArray is array of 5,5 int
```


Para utilizar una array unidimensional, modificar el valor numérico: sustitúyase el punto 5.5 por el texto siguiente **5**.


```wl
MyArray is array of 5 int
```

<a name="NOTE4_2"></a>


### Utilizar un string para declarar la dimensión array
<a name="utilizar_string_para_declarar_dimension_array_ELTPARAGRAPHE000046"></a>

**Código que desencadena el error** 


```wl
NbLines is string
MyArray is array of NbLines int
```


**Posible corrección**

Modificar el tipo de la NbLines Variable para usar un número entero.


```wl
NbLines is int
MyArray is array of NbLines int
```

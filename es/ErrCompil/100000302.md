
## Información 302: Ambigüedad entre un Variable y un control
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Motivo
<a name="motivo_ELTTEXTE000083"></a>
El mismo nombre se le dio a un control y a un local de Variable. En el código actual, este nombre representará a la Variable.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Consejo
<a name="consejo_ELTTEXTE000107"></a>
Para evitar cualquier ambigüedad, le recomendamos que utilice nombres diferentes para sus controles, ventanas y variables.

<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Ejemplo
<a name="ejemplo_ELTTEXTE000131"></a>


### Usando un control y un Variable local con el mismo nombre
<a name="usando_control_variable_local_con_mismo_nombre_ELTPARAGRAPHE000025"></a>**Código que desencadena el mensaje**

La ventana contiene una control Edit1 y un botón Button1. En el código de clic del botón, Edit1 se redefine y corresponde a una Variable booleana.


```wl
// -- Click on button1
 
Edit1 is boolean = True
```


**Posible corrección**
La línea de código puede mantenerse como está o, si Edit1 corresponde a 2 elementos diferentes, se puede cambiar el nombre de Edit1 Variable. Esta operación de renombrar evita confusiones.

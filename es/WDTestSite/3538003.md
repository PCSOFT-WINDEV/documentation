## Presentación
<a name="presentacion_ELTTEXTE000163"></a>


### WDTestSite
<a name="wdtestsite_ELTPARAGRAPHE000011"></a>

WDTestSite se utiliza para ejecutar diferentes pruebas en un sitio WEBDEV.

Las diferentes pruebas son las siguientes:

- Prueba de carga: 
	La prueba de estrés consiste en simular la conexión de varios usuarios de la Web a un sitio WEBDEV. Cada usuario de la Web ejecuta una set de operaciones (escenario) simultáneamente.

- Prueba de regresión:
	El test de regresión consiste en comprobar el modo de funcionamiento de un sitio WEBDEV entre dos actualizaciones. La prueba de regresión consiste en comprobar si un escenario realizado con una versión anterior del sitio sigue funcionando correctamente una vez que se ha actualizado el sitio.

- Prueba de un sitio en modo multiusuario: 
	La prueba de un sitio en modo multiusuario se utiliza para comprobar si los accesos simultáneos a los archivos de datos se gestionan correctamente.. Esta prueba consiste en simular la conexión simultánea de varios usuarios Web a un sitio WEBDEV. Cada usuario de la Web ejecuta una set de operaciones (escenario) simultáneamente.

- comparison de diferentes servidores:
	WDTestSite se utiliza para comparar la velocidad de diferentes servidores. Para ello, ejecute el mismo escenario en servidores diferentes y compare el tiempo de ejecución de este escenario.

- Optimizar los procesos creados en WLanguage:
	WDTestSite se utiliza para comparar el tiempo de ejecución de un escenario antes y después de optimizar el código de WLanguage.

## Iniciando WDTestSite
<a name="iniciando_wdtestsite_ELTTEXTE000187"></a>
Para iniciar WDTestSite:

- en la pestaña "Herramientas", en el grupo "Utilidades", haga clic en "WDTestSite" (de WEBDEV).

- ejecute "WDTestSite.EXE" directamente.




<a name="NOTE3"></a>
<a name="NOTE3_1"></a>


## Condiciones de uso
<a name="condiciones_uso_ELTTEXTE000211"></a>
WDTestSite no puede ser redistribuido.

Sin embargo, para ejecutar pruebas en modo multiusuario, WDTestSite puede instalarse en varios ordenadores.

## Uso de WDTestSite
<a name="uso_wdtestsite_ELTTEXTE000235"></a>


### Principio
<a name="principio_ELTPARAGRAPHE000069"></a>

WDTestSite está acostumbrado a:

- [crear un escenario](../WDTestSite/3538004.md). Este escenario contiene una set de acciones que deben realizarse en un sitio WEBDEV.

- [ejecutar el test de un escenario directamente](../WDTestSite/3538004.md).

- [iniciar sucesivamente varias ejecuciones del mismo escenario desde el mismo ordenador o desde diferentes ordenadores](../WDTestSite/3538004.md).
	El escenario se ejecuta en el servidor.

- [ejecutar el mismo escenario (usuarios simultáneos) desde el mismo ordenador o desde ordenadores diferentes](../WDTestSite/3538001.md).
	El escenario se ejecuta en el servidor.

## Instalación de WDTestSite en diferentes ordenadores
<a name="instalacion_wdtestsite_diferentes_ordenadores_ELTTEXTE000259"></a>
Para ejecutar pruebas de tensión en un servidor o para probar la rapidez del servidor, se recomienda utilizar ordenadores diferentes para ejecutar la ejecución de un escenario.. El escenario siempre se ejecuta en el mismo servidor Web.

Los siguientes archivos deben copiarse en cada ordenador de pruebas:

- el ejecutable: WDTestSite.exe

- las diferentes bibliotecas: httputil.dll, wd28 0hf.dll, wd28 0mat.dll, wd28 0obj.dll, wd28 0pnt.dll, wd28 0std.dll y wd28 0vm.dll.

- el archivo de ayuda de WDTestSite: WDTestSite.pdf

- el escenario (archivo".WCN"): este archivo debe copiarse en un subdirectorio "Scenario" del directorio instalación de WDTestSite.exe. 




Una vez copiados estos archivos en los ordenadores de prueba, basta con ejecutar el escenario o la prueba de carga en cada uno de los ordenadores de prueba.


## WDTestSite: Creación de un escenario
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000101"></a>
El escenario es un archivo de texto (extensión".WCN") que contiene todas las operaciones realizadas al registrar el escenario.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Creación de un escenario
<a name="creacion_escenario_ELTTEXTE000125"></a>
Para crear un escenario, los siguientes elementos deben estar instalados en el ordenador actual:

- Versión de Desarrollo WEBDEV o Servidor de Aplicaciones WEBDEV.

- el sitio WEBDEV utilizado para crear el escenario.




Le aconsejamos que cree este escenario en el servidor web donde está instalado el sitio WEBDEV (ordenador implementación).
<a name="NOTE2_2"></a>


### Creación de un escenario en el ordenador implementación
<a name="creacion_escenario_ordenador_implementacion_ELTPARAGRAPHE000028"></a>

Para crear un escenario en el ordenador implementación: 

1. Inicie WDTestSite (haga doble clic en WDTestSite.EXE, por ejemplo).

2. Compruebe si el servidor WEBDEV utilizado corresponde al "Ordenador local".

3. Compruebe si el administrador de WEBDEV está iniciado ("Programas .. WEBDEV 28.. Administrador de WEBDEV" del menú "Inicio").

4. Haga clic en el botón "Nuevo".

5. Especifique el nombre y la description del escenario. La description puede corresponder a un resumen de las características comprobadas en el escenario. Esta description servirá para identificar fácilmente el escenario y sus características.
	Atención:

	- El escenario description no puede ser modificado posteriormente.

	- Si el nombre del escenario corresponde a un escenario que ya existe en este ordenador, el escenario existente será sustituido por el nuevo.




6. Haga clic en "Save".: se muestra la página de prueba del administrador de WEBDEV.

7. Seleccione el sitio WEBDEV utilizado para crear el escenario.

8. Realizar todas las operaciones necesarias en el sitio WEBDEV.
	Observaciones:

	- Para comprobar la capacidad de carga del servidor con respecto a un número importante de conexiones simultáneas, le aconsejamos que deje el sitio conectado después de la última acción.: cuando grabe el escenario, no utilice ningún botón o enlace utilizado para cerrar el sitio.

	- Para probar la regresión de un sitio o la gestión de accesos concurrentes, puede salir del sitio (mediante un enlace o un botón) al registrar el escenario.




9. Para finalizar la grabación del escenario, haga clic en el botón "Listo" de WDTestSite. 
	El escenario (archivo".WCN") se crea en el subdirectorio "Escenario" del directorio que contiene "WDTestSite.EXE".

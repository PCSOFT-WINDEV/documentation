
## WDTestSite: Prueba de carga
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000115"></a>
Las pruebas de esfuerzo se utilizan para comprobar:

- el número de usuarios de la Web que pueden conectarse simultáneamente a un sitio WEBDEV.

- la gestión de los accesos concurrentes a un sitio WEBDEV.




<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Realizar una prueba de esfuerzo
<a name="realizar_una_prueba_esfuerzo_ELTTEXTE000139"></a>
Para realizar una prueba de esfuerzo desde un ordenador:

1. Configurar el administrador WEBDEV de el equipo donde se ejecutará el escenario (servidor Web).
	Verifique los parámetros del sitio correspondientes al escenario:

	- el número máximo de conexiones al sitio:
			este número debe corresponder al número máximo de conexiones simultáneas (o conexiones sucesivas sin cerrar el sitio).

	- el número máximo de veces que un usuario puede volver a conectarse al sitio:
			Para un escenario que se ejecuta varias veces desde un ordenador, este número debe coincidir con el número de ejecuciones de escenario.

	- el tiempo de inactividad antes de desconectar a los usuarios de Inactive:
			Al comprobar la capacidad del servidor, el tiempo de inactividad antes de desconectar a los usuarios de Inactive debe ser bastante largo (más de 30 minutos por ejemplo).




2. Inicie WDTestSite en el ordenador de pruebas (ordenador de desarrollo, por ejemplo).

3. Seleccione el servidor Web donde se debe ejecutar el escenario.. Este servidor debe ser accesible desde su ordenador y el administrador de WEBDEV (versión de desarrollo o versión implementación) debe ser ejecutado en este ordenador.
	En la lista "Ordenador de destino" puede introducir..:

	- el nombre de un ordenador accesible a través de la red. Ejemplo: "Probar servidor"

	- año IP Address. Ejemplo: 123.3.456.12

	- un INTERNET Address. Ejemplo: www.webdev.eu
			El botón "Vecindario" se utiliza para listar los equipos a los que se puede acceder directamente a través de la red (no disponible en Windows 98 o Me).




4. Seleccione su escenario de la lista de escenarios disponibles.

5. Haga clic en el botón "Cargar".

6. Especificar el número de usuarios (que significa usuarios de la Web) simulados de el equipo.
	Se creará un "Robot" de prueba en el ordenador actual para cada usuario..
	Atención: el número de usuarios simulados debe ser elegido de acuerdo a los recursos de el equipo utilizados para ejecutar la prueba. Cada robot consume recursos de memoria en el equipo (RAM).

7. Indicar el tiempo de espera (en centésimas de segundo). Este tiempo muerto corresponde al tiempo entre cada acción de escenario.

8. Crear los diferentes usuarios (botón"Crear los robots").

9. Ejecutar la prueba (botón"Iniciar los robots"). Se ejecutan los tests seleccionados.

10. Para finalizar las pruebas o para ejecutar una nueva prueba con más usuarios simulados, haga clic en el botón "Matar los robots".

## Notas
<a name="notas_ELTTEXTE000163"></a>


- Cuando se ejecuta el escenario, se muestran todas las acciones realizadas en el sitio.. Si se produce un problema (error, ....), el escenario se detiene.

- Si se produce un error, el archivo HTML que lo contiene se guarda en el directorio "C:\\TEMP" de el equipo.

- Para comprobar la capacidad del servidor durante las conexiones simultáneas, le recomendamos que realice pruebas de esfuerzo desde diferentes ordenadores. Para obtener más información, consulte [Instalación de WDTestSite en diferentes ordenadores](../WDTestSite/3538003.md).

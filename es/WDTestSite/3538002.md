<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000143"></a>
El escenario es un archivo de texto (extensión".WCN") que contiene todas las operaciones realizadas al registrar el escenario.

Este escenario es ejecutado por WDTestSite.

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Ejecutar un escenario
<a name="ejecutar_escenario_ELTTEXTE000167"></a>
Ejecutar un escenario le permite ejecutar pruebas de regresión en un sitio WEBDEV. De hecho, un escenario ejecutado con una versión anterior del sitio debe funcionar correctamente con un sitio actualizado.

Ejecutar el mismo escenario varias veces consecutivas se utiliza para probar la capacidad del servidor para múltiples conexiones., ... En este caso, recomendamos que utilice un escenario que no cierre el sitio.

En todos los casos, el escenario debería ejecutarse:

- desde cualquier ordenador (ordenador de desarrollo, por ejemplo).

- en el servidor Web para implementar el sitio WEBDEV.



<a name="NOTE2_2"></a>


### Ejecutar un escenario
<a name="ejecutar_escenario_ELTPARAGRAPHE000032"></a>

Para ejecutar un escenario:

1. Configurar el administrador WEBDEV de el equipo donde se ejecutará el escenario (servidor Web). Verifique los parámetros del sitio correspondientes al escenario:

	- el número máximo de conexiones al sitio:
			Este número debe corresponder al número máximo de conexiones simultáneas (o conexiones sucesivas si los emplazamientos no están cerrados).

	- el número máximo de veces que un usuario puede volver a conectarse al sitio:
			Para un escenario que se ejecuta varias veces desde un ordenador, este número debe coincidir con el número de ejecuciones de escenario.

	- el tiempo de inactividad antes de desconectar a los usuarios de Inactive: 
			Al comprobar la capacidad del servidor, el tiempo de inactividad antes de desconectar a los usuarios de Inactive debe ser bastante largo (más de 30 minutos por ejemplo).




2. Inicie WDTestSite en el ordenador de pruebas (ordenador de desarrollo, por ejemplo).

3. Seleccione el servidor Web en el que se ejecutará el escenario.. Este servidor debe ser accesible desde el ordenador de pruebas. El administrador de WEBDEV (versión de desarrollo o versión de implementación) debe iniciarse en el servidor..
	En la lista "Ordenador de destino" puede introducir..:

	- el nombre de un ordenador accesible a través de la red. Ejemplo: "Probar servidor"

	- año IP Address. Ejemplo: 123.3.456.12

	- un INTERNET Address. Ejemplo: www.webdev.eu
			El botón "Vecindario" se utiliza para listar los equipos a los que se puede acceder directamente a través de la red (no disponible en Windows 98 o Me).




4. Seleccione su escenario de la lista de escenarios disponibles.

5. Haga clic en "Ejecutar". Se muestra la ventana de tiempo de ejecución y la prueba se ejecuta automáticamente.

6. Para ejecutar el escenario varias veces consecutivas, especifique un número de turnos superior a 1 y haga clic en el botón "Re-ejecutar".



<a name="NOTE2_3"></a>


### Observaciones
<a name="observaciones_ELTPARAGRAPHE000064"></a>

- Cuando se ejecuta el escenario, se muestran todas las acciones realizadas en el sitio.. Si se produce un problema (error, ...), el escenario se detiene y el último Line muestra el error que se produjo.

- Si se produce un error, se guarda un archivo HTML que contiene el error en el directorio "C:\\TEMP" del ordenador actual.. Atención: el usuario de INTERNET debe tener derechos de lectura/escritura en "C:\\Temp".

- Para probar la capacidad del servidor durante las conexiones simultáneas, se debe ejecutar el mismo escenario desde diferentes ordenadores. Consulte "Instalación de WDTestSite en diferentes ordenadores" para más detalles.

## Ejecutando un escenario en el comando Line
<a name="ejecutando_escenario_comando_line_ELTTEXTE000203"></a>
Para ejecutar un escenario en el comando Line:

1. Configurar el administrador WEBDEV de el equipo donde se ejecutará el escenario (servidor Web). Verifique los parámetros del sitio correspondientes al escenario:

	- el número máximo de conexiones al sitio:
			Este número debe corresponder al número máximo de conexiones simultáneas (o conexiones sucesivas si los emplazamientos no están cerrados).

	- el número máximo de veces que un usuario puede volver a conectarse al sitio:
			Para un escenario que se ejecuta varias veces desde un ordenador, este número debe coincidir con el número de ejecuciones de escenario.

	- el tiempo de inactividad antes de desconectar a los usuarios de Inactive: 
			Al comprobar la capacidad del servidor, el tiempo de inactividad antes de desconectar a los usuarios de Inactive debe ser bastante largo (más de 30 minutos por ejemplo).




2. Inicie WDTestSite en el ordenador de prueba utilizando la siguiente sintaxis:





```txt
WDTestSite /AUTO = <Scenario name>
```


Por ejemplo: "WDTestSite /AUTO = MyScenario.wcn". El escenario se ejecuta automáticamente y WDTestSite se cierra.

El escenario debe encontrarse en el directorio de escenarios. 
Recordatorio: El escenario (archivo".WCN") se crea en el subdirectorio "Escenario" del directorio que contiene "WDTestSite.EXE".

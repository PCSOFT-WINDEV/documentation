## Presentación
<a name="presentacion_ELTTEXTE000343"></a>
La pestaña "Avanzado" sirve para configurar las funciones avanzadas a nivel de servidor:<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0006.gif)

	- Especifique el nombre o la IP Address del ordenador actual.

	- Muestra en el navegador una Page utilizada para iniciar todos los sitios WEBDEV instalados en el ordenador actual.

	- Gestionar los correos electrónicos en modo asíncrono.

	- Gestionar el servidor sockets.

	- Prohibir el cambio de IP mientras se navega.

	- No permitir el acceso a los identificadores de contexto AWP desde JavaScript.

	- Administre las funciones fCopyFileWebFolder y fDeleteFileWebFolder.

	- Exigir una conexión segura (https) para los sitios de administración remota.

## Detalles de la pestaña "Avanzado

### Servidor utilizado
<a name="servidor_utilizado_ELTPARAGRAPHE000052"></a>

El botón "Servidor" permite elegir uno de los servidores Web instalados en el ordenador actual para ejecutar los sitios WEBDEV. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0006%201.gif)
 Precaución: Este servidor se utilizará para todos los sitios WEBDEV y servicios web WEBDEV instalados en este ordenador.

Si el servidor utilizado no se encuentra en esta lista, seleccione "Otro".. Tendrá que configurar manualmente el servidor Web utilizado. Para obtener más información, consulte [Configuración de servidores específicos](../Editeurs/2037005.md).

**Observaciones**:

- El botón [ Servidores virtuales] aparece si se selecciona un servidor IIS o Apache. Este botón se utiliza para seleccionar los servidores virtuales en los que se creará el alias requerido por WEBDEV (WD28 0AWP).

- El botón de parámetros de [ Apache] aparece si se selecciona un servidor Apache. Este botón permite seleccionar el tipo de servidor Apache utilizado así como sus parámetros. Esto le permite administrar los servidores Apache instalados por EasyPHP o WAMP.

### Diagnóstico
<a name="diagnostico_ELTPARAGRAPHE000085"></a>

El botón "Diagnóstico" se utiliza para comprobar la configuración del ordenador actual o la configuración del ordenador especificado.. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0006%201.gif)
 Este diagnóstico se utiliza para comprobar:

- si el protocolo FTP/IP está instalado

- si se ha iniciado un servidor web (servidor HTTP)

- si el administrador de protocolo de AWP está configurado correctamente

- si se encuentran los ejecutables necesarios de WEBDEV.




Los sitios WEBDEV dinámicos o los servicios web WEBDEV no funcionarán si no se cumple una de estas condiciones. Para obtener más información, consulte [Configuración del ordenador](../Editeurs/2037005.md).

### Búsqueda de páginas caducadas
<a name="busqueda_paginas_caducadas_ELTPARAGRAPHE000131"></a>

Esta opción se utiliza para habilitar y configurar la búsqueda de páginas caducadas en los sitios WEBDEV gestionados por el Servidor de Aplicaciones WEBDEV.
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0006%203.gif)


Gestionar la búsqueda de páginas caducadas:

1. Marque "Permitir la búsqueda de páginas caducadas".

2. La ventana de configuración aparece automáticamente.

3. Especifique: 

	- la hora de inicio de la búsqueda de las páginas caducadas. En el momento especificado, servidor de aplicaciones realizará una búsqueda de páginas caducadas en todos los sitios WEBDEV dinámicos gestionados por el servidor.. 

	- el Email Address del remitente de la alerta Email. Los parámetros de los destinatarios del correo electrónico se han definido en la página description, en el editor. Para obtener más información, consulte [Gestión de las fechas de caducidad de las páginas](../WDChamp/1012021.md).

	- el Address y el puerto del servidor SMTP utilizado para enviar correos electrónicos. También puede especificar si debe usarse el spooler de Email. 

	- el nombre de usuario y la contraseña del usuario del servidor SMTP (para los servidores con autenticación).




4. Valide.

### Cola de correo electrónico
<a name="cola_correo_electronico_ELTPARAGRAPHE000180"></a>

Si la opción "Desactivar cola de correo" está desmarcada, sus sitios podrán enviar correos electrónicos sin bloquear la ejecución de los diferentes procesos (modo asíncrono). 
El modo asíncrono debe activarse al iniciar la sesión de Email (con [EmailStartSMTPSession](../WDLang3/3032025.md) o [EmailStartSession](../WDLang3/3032028.md)).

Si se activa el modo asíncrono, todos los correos electrónicos salientes se redirigirán a un "Spooler". Los correos electrónicos se ponen en cola antes de ser enviados.

La ejecución de las funciones de Email ya no bloquea el resto del programa. [EmailStatus](../WDLang3/3032009.md) se utiliza para determinar el estado de un Email.

**Observación**: Si el administrador de WEBDEV está cerrado, el spooler de Email se borra.: los correos electrónicos pendientes no se envían y se eliminan del spooler.

Si se marca "Desactivar cola de correo" mientras hay correos pendientes, estos correos no se perderán: el administrador continúa enviándolos, pero el spooler no aceptará nuevos correos electrónicos.

**Atención**: El modo asíncrono solo puede utilizarse al iniciar una sesión en un servidor SMTP (función [EmailStartSMTPSession](../WDLang3/3032025.md) para enviar correos electrónicos o función [EmailStartSession](../WDLang3/3032028.md)). El modo asíncrono es ignorado en todos los demás casos.

### No permitir cambiar de IP durante la navegación
<a name="permitir_cambiar_durante_navegacion_ELTPARAGRAPHE000234"></a>

Si "No permitir cambiar de IP durante la navegación" está marcada, la dirección IP associated con la sesión no puede cambiar mientras está en línea. Se utiliza para protegerse de los ataques de "secuestro de sesión" (ataque que consiste en hacerse pasar por un usuario legítimo conectado al servidor).. 


<a name="NOTE2_12"></a>


### No permitir el acceso a los identificadores de contexto AWP desde JavaScript
<a name="permitir_acceso_los_identificadores_contexto_awp_desde_javascript_ELTPARAGRAPHE000248"></a>

En un sitio de AWP, el contexto del sitio se almacena en el servidor. El identificador de este contexto es enviado y almacenado en el navegador a través de un Cookie. 

Si la opción "No permitir el acceso a los identificadores de contexto AWP desde JavaScript" está seleccionada, las cookies utilizadas serán de tipo "HTTPOnly", el cual no puede ser leído desde el código JavaScript. Este modo protege contra ataques XSS (Cross-Site scripting). 
Por Default, el acceso a los identificadores de contexto de AWP no está permitido desde JavaScript.

### Administrar las funciones fCopyFileWebFolder y fDeleteFileWebFolder
<a name="administrar_las_funciones_fcopyfilewebfolder_fdeletefilewebfolder_ELTPARAGRAPHE000267"></a>

La opción "Permitir las funciones fCopyFileWebFolder y fDeleteFileWebFolder." debe estar marcada si se utilizan estas funciones en el sitio web. 

Estas funciones se utilizan principalmente para incluir imágenes que se encuentran en el directorio de datos en el directorio de imágenes del sitio (por ejemplo, imágenes subidas y puestas a disposición de los usuarios del sitio web).

**Atención**: La copy es tenida en cuenta por el administrador de WEBDEV en el servidor (WD28 0ADMIN.EXE). La cuenta de Windows que lo ejecute debe tener suficientes derechos sobre la ubicación de destino de copy.
<a name="NOTE2_13"></a>


### Exigir una conexión segura (https) para los sitios de administración remota
<a name="exigir_una_conexion_segura_https_para_los_sitios_administracion_remota_ELTPARAGRAPHE000286"></a>

La opción "Exigir una conexión segura (https) para los sitios de administración remota." permite utilizar los sitios de administración remota proporcionados con WEBDEV en modo seguro. Los sitios de manejo afectados son: 

- el administrador WEBDEV remoto, 

- el administrador remoto de HFSQL, 

- el administrador remoto de SaaS.

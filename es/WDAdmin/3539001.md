
## Personalización de los errores
			



<a name="NOTE1"></a>
<a name="NOTE1_1"></a>


## Presentación
<a name="presentacion_ELTTEXTE000202"></a>
El administrador de WEBDEV (versión de desarrollo) se utiliza para personalizar los mensajes de error de un sitio WEBDEV.

Los mensajes personalizados de un sitio se guardan y se instalan automáticamente cuando se despliega el sitio WEBDEV.

Observación: La página de errores del sitio también se puede personalizar a partir del proyecto WEBDEV. Para obtener más información, consulte [Gestión de los errores del sitio](../WDChamp/9000134.md).

<a name="NOTE2"></a>
<a name="NOTE2_1"></a>


## Personalización de los errores
<a name="personalizacion_los_errores_ELTTEXTE000226"></a>
Se puede personalizar:

- los mensajes de error para todos los centros.

- los mensajes de error para un sitio específico.



<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0007.gif&type=thumb)

<a name="NOTE2_2"></a>


### Personalización de los mensajes de error para todos los centros
<a name="personalizacion_los_mensajes_error_para_todos_los_centros_ELTPARAGRAPHE000037"></a>

Para personalizar los mensajes de error de todos los sitios WEBDEV instalados en el ordenador actual (y en el servidor Web durante la instalación), elija "Todos los sitios" en el cuadro combinado "Sitio".

Esta personalización se aplica a:

- el texto de los mensajes de error.

- la página que muestra el mensaje de error.



<a name="NOTE2_3"></a>


### Personalización de los mensajes de error para un sitio específico
<a name="personalizacion_los_mensajes_error_para_sitio_especifico_ELTPARAGRAPHE000049"></a>

Para personalizar el mensaje de error para un sitio específico instalado en el ordenador actual (y en el servidor Web durante la instalación), elija el nombre de su sitio en el cuadro combinado "Sitio".

Esta personalización se aplica a:

- el texto de los mensajes de error.

- la página que muestra el mensaje de error.



<a name="NOTE2_4"></a>


### Orden de prioridad para la personalización
<a name="orden_prioridad_para_personalizacion_ELTPARAGRAPHE000061"></a>

El orden de prioridad para la personalización es:

1. personalizar todos los mensajes para todos los sitios instalados en el servidor.

2. personalizar los mensajes para un sitio específico.




**Atención**: No se recomienda personalizar los mensajes de error para todos los sitios cuando se utiliza un servidor compartido.. En efecto, esta modificación se aplicará a todos los sitios WEBDEV instalados en el servidor.

## ¿Cómo proceder?
<a name="¿como_proceder_ELTTEXTE000268"></a>
Para personalizar los mensajes de error: 

1. Seleccione el sitio que debe personalizarse (sitio específico o todos los sitios).

2. Para personalizar ***la página utilizada para todos los errores***:
	Por Default, los errores se muestran en una página específica. Para personalizar esta página:

	- Elija la opción "Personalizada":<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0007%202.gif&type=thumb)


	- Seleccione una página HTML accesible desde su ordenador.
			Esta página debe ser una página HTML estática que pueda contener imágenes, ... 
			En esta página se requieren dos controles estáticos: Estos controles deben tener respectivamente las siguientes etiquetas para el valor: [%MSG%] y [%SYS%]. Estos controles serán sustituidos por el identificador y el título del error. 
			Esta página HTML debe estar ubicada en el directorio del proyecto &lt;NombredelProyecto&gt;_WEB.

3. Para personalizar ***el mensaje de error y/o la página de error***:

	- Seleccione el identificador de error: aparece la leyenda Default. 
			Para get una explicación sobre estos mensajes, véase [Mensajes de error que se pueden visualizar en el navegador](../WDAdmin/3539002.md).<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0007%201.gif&type=thumb)


	- Seleccione "Pie de foto personalizado" e introduzca el nuevo pie de foto associated con la opción. 
			Observación: Los caracteres especiales utilizados en los mensajes:

	- la etiqueta {RECONNECT} significa que el usuario Web se conectará automáticamente una vez que aparezca el mensaje de error.

	- los caracteres %x serán reemplazados automáticamente cuando se muestre el mensaje de error.

4. Para ***aplicar los cambios***:

	- Haga clic en "Aplicar" para aplicar inmediatamente los cambios en el ordenador actual.

	- Haga clic en "Guardar" para guardar los cambios en un archivo ".REG".. 
			Si el sitio seleccionado es un sitio específico, el archivo de configuración se guarda en el directorio del proyecto..
			Si el sitio seleccionado es "Todos los sitios", el archivo de configuración se guarda en el directorio WEBDEV.




5. Para ***instalar los archivos modificados***:

	- Si la personalización se aplica a un sitio específico, la instalación remota seleccionará automáticamente el archivo de configuración y lo ejecutará en el servidor. Si el archivo HTML para personalizar la página de error está en el directorio de imágenes del proyecto (&lt;Nombre_del_proyecto&gt;_Web), se instalará automáticamente.

	- Si la personalización afecta a todos los sitios del servidor, debe hacerlo:

		- Paso opcional: copiar (si es necesario) los archivos HTML de las páginas de error y las imágenes de associated al servidor Web. 
						Estos archivos deben estar ubicados en un directorio del servidor para el que se haya definido un alias.

		- copy y ejecute el archivo ERR_MSG_WEBDEV.REG en el servidor web.







**Atención**: Esta personalización es una modificación avanzada que requiere conocimientos en la gestión de servidores Web.

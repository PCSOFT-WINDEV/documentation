## Detalles de la pestaña "Ajustes"
<a name="detalles_pestana_ajustes_ELTTEXTE000160"></a>
La pestaña "Ajustes" le permite:

- Configure el servidor para las configuraciones y actualizaciones del sitio.

- Gestionar las cuentas WEBDEV.

- Gestionar el archivo de registro de las configuraciones.



<a name="NOTE2_2"></a>


### Instalaciones/actualizaciones
<a name="instalacionesactualizaciones_ELTPARAGRAPHE000030"></a>

Estas opciones están disponibles en la versión implementación. Estas opciones permiten que el host permita y configure configuraciones y actualizaciones remotas.

Las diferentes opciones son las siguientes: 

- La opción **"Bloquear el servidor (para actualización)"** simplifica las actualizaciones del sitio impidiendo que los nuevos usuarios se conecten al servidor. Cada vez que los usuarios intentan conectarse a uno de los sitios del servidor, reciben un mensaje que les indica que el sitio no está disponible temporalmente y les pide que lo vuelvan a intentar más tarde. Los usuarios de la Web que ya están conectados pueden seguir utilizando el sitio WEBDEV actual..
	**Observación**: Durante una actualización por FTP, el sitio WEBDEV actualizado se bloquea automáticamente.

- La opción **"Permitir actualización de los parámetros de conexión"** permite al administrador del servidor permitir la personalización de los parámetros de conexión cuando se instala un sitio vía FTP. Si esta opción está seleccionada, se tendrán en cuenta las opciones de conexión definidas por el creador del sitio.. Si esta opción no está seleccionada, sólo se tendrán en cuenta las opciones de conexión definidas en esta pestaña por el administrador del servidor.. Las opciones personalizadas serán ignoradas.

- La opción **"Permitir instalación remota de sitios"** permite al administrador del servidor autorizar la instalación de sitios WEBDEV dinámicos en el servidor a través de FTP.

- Con la opción **"Permitir actualización remota de sitios"**, el administrador del servidor puede autorizar las actualizaciones de los sitios instalados en el servidor por FTP.

- La opción **"Permitir instalación de sitios con el mismo nombre para usuarios diferentes"** permite al administrador autorizar el implementación de varios sitios con el mismo nombre en diferentes cuentas WEBDEV en el mismo servidor de alojamiento. Esta opción es muy useful para el alojamiento compartido. 

- Con la opción **"Permitir visualización de la página http[s]:/// para ver la lista de aplicaciones instaladas"**, el administrador puede autorizar el uso de la sintaxis "http[s]://&lt;servidor#XGTXLT#servidor&gt;/" muestra una página con un enlace a cada sitio instalado en el servidor.  


<a name="NOTE2_5"></a>


### Archivo de registro de las instalaciones
<a name="archivo_registro_las_instalaciones_ELTPARAGRAPHE000085"></a>

El registro de las configuraciones se utiliza para rastrear los elementos instalados de un sitio.. Este registro es muy útil antes de que el soporte técnico de Contact si se produjeron errores durante el implementación.

Para utilizar el registro de configuraciones, basta con especificar la ruta completa del archivo correspondiente (archivo "WDInstall_log.txt" de Default).

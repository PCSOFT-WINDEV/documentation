## Mensaje de error
<a name="mensaje_error_ELTTEXTE000902"></a>
Los mensajes de error que se pueden visualizar en el navegador son los siguientes:

| Errores | Designación |
| --- | --- |
| Error al abrir el registro de acceso. Revisa instalación | Este mensaje aparece al iniciar un sitio WEBDEV.<br><br>El servidor OmniHTTPd muestra este mensaje. Este mensaje puede tener varias causas:<br><br>- el servidor OmniHTTPd está instalado en un directorio que utiliza nombres largos. <br>	**Solución**: Vuelva a instalar el OmniHTTPd en un directorio que no utilice nombres largos.<br><br>- se abren varios servidores OmniHTTPd. <br>	**Solución**: Cierre todos los servidores abiertos y reinicie una única instancia de OmniHTTPd.<br><br>- el directorio "Log file" no existe. <br>	**Solución**: En el panel control del servidor, desactive todos los directorios de "Archivos de registro" y reinicie el servidor.<br><br><br> |
| HTTP/1.0 404 Objeto no encontrado o Resource no encontrado | Este mensaje se muestra en el navegador de INTERNET cuando se inicia el sitio.<br><br>Este mensaje es causado por una mala configuración del servidor de INTERNET. Esto significa que el servidor no puede encontrar el gestor de protocolos CGI..<br><br>**Solución**: Compruebe la configuración del servidor de INTERNET utilizado, especialmente:<br><br>- el directorio virtual (o alias) que apunta al directorio que contiene el gestor del protocolo CGI, WD28 0AWP.EXE<br><br>- la clave de registro del gestor del protocolo CGI debe ser set a WD28 0AWP.EXE.<br><br><br>Recordatorio: Los detalles de configuración de WEBDEV se guardan en el registro, en la siguiente entrada "HKEY_LOCAL_MACHINE\\SOFTWARE\\PC SOFT\\WEBDEV\\28.0". |
| El Explorador de INTERNET no puede abrir el sitio de INTERNET... | Este mensaje es mostrado por el navegador INTERNET cuando se ejecuta un sitio WEBDEV si no se inicia ningún servidor INTERNET al ejecutar el sitio.<br><br>**Solución**: Inicie un servidor de INTERNET antes de iniciar el sitio WEBDEV. |

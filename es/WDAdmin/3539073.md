## Presentación
<a name="presentacion_ELTTEXTE000210"></a>
La pestaña "Logs" sirve para: 

- ver los detalles de los registros de un período específico. 

- configurar el envío de informes de error por parte de Email.

<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0011.gif)


**Atención**: Para utilizar esta función, los registros deben estar habilitados para el sitio. Los registros pueden ser habilitados: 

- en la pestaña "Logs" para crear un log general,

- en la pestaña "Sitios" (botón "Parámetros") y en la pestaña "Servicios web" (botón "Parámetros") para gestionar un registro específico. 


El registro de errores se crea automáticamente cuando se activa el registro de estadísticas.

## Detalles de la ficha "Logs
<a name="detalles_ficha_logs_ELTTEXTE000234"></a>


### Gestión del archivo de registro
<a name="gestion_del_archivo_registro_ELTPARAGRAPHE000032"></a>
<br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDADMIN_Onglets%20-%20HC%20N%B0011%201.gif&type=thumb)
El administrador de WEBDEV se utiliza para gestionar un archivo de registro específico que le permite seguir las estadísticas de tráfico.

- ***Generar un archivo de registro (.log)***:
	Esta opción le permite generar un archivo de registro (extensión".LOG") utilizado para ver las estadísticas de tráfico de los sitios WEBDEV a través de WDStatistic.. Para más detalles, consulte la ayuda sobre WDStatistic.
	Se puede especificar la ubicación del archivo de registro. 
	Esta opción puede modificarse para cada sitio en la pestaña "Sitios".

- ***Guardar en el fichero global y en el fichero de cada aplicación***: 
	Esta opción se utiliza para guardar las estadísticas en los archivos de registro tanto de la aplicación como del servidor. De hecho, si el archivo de registro se define tanto a nivel de servidor como a nivel de aplicación (pestaña"Sitios"), la información puede "perderse".. La tabla de abajo presenta los diferentes casos posibles:
	


|   |   |   |   |
| --- | --- | --- | --- |
| **Log de aplicación** | **Registro del servidor** | **La opción"Guardar en el archivo global y en el archivo de cada aplicación" está marcada.** | **Resultado** |
| No | No | Cualquier | Sin registro |
| No | Sí | Cualquier | Copia de seguridad en el registro del servidor |
| Sí | No | Cualquier | Copia de seguridad en el log de aplicación |
| Sí | Sí | No | Copia de seguridad en el log de aplicación |
| Sí | Sí | Sí | Copia de seguridad en el log del servidor y en el log de aplicación |







### Informe de errores por correo electrónico
<a name="informe_errores_por_correo_electronico_ELTPARAGRAPHE000119"></a>

El WEBDEV servidor de aplicaciones examina periódicamente los registros y puede Email informes de error. 

Para permitir la generación de informes de error: 

1. Habilitar los registros si es necesario. 

2. En la pestaña "Registros" del administrador WEBDEV, haga clic en el botón "Informe de errores por correo electrónico". <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdmin%20-%20Journaux%20-%20notification%20Erreur%20-%20HC%20N%B0001.gif&type=thumb)


3. En la ventana para la gestión de los informes de error, marque "Habilitar los informes de error".. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdmin%20-%20Journaux%20-%20notification%20Erreur%20-%20HC%20N%B0002.gif&type=thumb)
Es posible: 

	- Configurar los destinatarios de reporte general. El reporte general afecta a todas las aplicaciones WEBDEV instaladas en el servidor Web. Esta configuración se realiza en la ventana de gestión de informes de error. 

	- Configure los destinatarios del error reporte para cada sitio o servicio web (pestaña "Sitios" o "Servicios web", botón "Parámetros").







Para configurar los destinatarios: <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdmin%20-%20Journaux%20-%20notification%20Erreur%20-%20HC%20N%B0002%201.gif&type=thumb)


1. Haga clic en "+". 

2. Especifique: 

	- el receptor Address. 

	- la frecuencia de Email: diario, semanal, mensual. 

	- las condiciones de envío: 

		- sólo si se ha producido un error (opción seleccionada),

		- sistemáticamente (opción no seleccionada). 







**Observaciones**: 

- Los informes de error se crean todos los días a medianoche y al iniciar el servidor.. Se envían de acuerdo a la frecuencia solicitada. 

- La configuración de los parámetros SMTP necesarios para enviar los informes de error se realiza en la ventana de gestión de informes de error.. <br>![](https://doc.pcsoft.fr/en-US/images/image.awp?langid=7&name=WDAdmin%20-%20Journaux%20-%20notification%20Erreur%20-%20HC%20N%B0003.gif&type=thumb)


- Si no se reciben los informes de error, compruebe los parámetros SMTP especificados.. Si el envío a través de SMTP falla, la información se especifica en los registros del sistema utilizado.





### Ver los registros
<a name="ver_los_registros_ELTPARAGRAPHE000156"></a>

Para ver los detalles de los registros: 

1. Seleccione el periodo de estudio. 

2. Seleccione el tipo de registro a estudiar. 

3. Haga clic en el botón "Actualizar. 

4. Sólo se visualizan los elementos para los que se han producido errores durante el período seleccionado..

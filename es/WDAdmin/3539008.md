## Detalles de la pestaña "Sitios"
<a name="detalles_pestana_sitios_ELTTEXTE000168"></a>
La pestaña "Sitios" devuelve la lista de sitios WEBDEV dinámicos instalados en el equipo y gestionados por el administrador de WEBDEV. Para cada sitio, la siguiente información se muestra en un tabla:

- el nombre del sitio.

- el directorio completo de instalación del sitio en el equipo.

- nombre del archivo de "proyecto" correspondiente.

- el status del centro (bloqueado o no). Puede marcar la casilla para bloquear directamente el acceso al sitio durante la actualización. 

- el número de tareas programadas utilizadas por el centro.




La pestaña "Sitios" también se puede utilizar para:

- **Borrar el sitio o los sitios seleccionados de la tabla** (botón "Borrar").
	Se utiliza para eliminar el sitio del administrador.: el sitio WEBDEV todavía se encuentra en el disco.

- **RRefresca la visualización** de los sitios encontrados en el tabla (botón"Refrescar").
	Esta operación de actualización sirve para comprobar la presencia de un sitio en el administrador después de haberlo instalado.

- **Añade un site** en el tabla (botón "Añadir").

- **Configurar cada sitio que se encuentra en el equipo**.
	La configuración utilizada por Default es la definida en [la pestaña "Configuración](../WDAdmin/3539046.md).
	También puede crear una configuración específica para el sitio seleccionado: 

	- el botón "Tareas programadas" se utiliza para gestionar las tareas programadas associated con el sitio seleccionado. En el modo de desarrollo, este botón es Grayed out. De hecho, el establecimiento y la ejecución de tareas programadas en un sitio actualmente desarrollado puede desencadenar errores o comportamientos inesperados.. Esta configuración está disponible cuando el sitio se despliega.

	- el botón "Parámetros" sirve para especificar los parámetros del sitio. Las opciones que se pueden configurar aparecen en una nueva ventana. Cada opción tiene una Combo Box con dos opciones: "General" y "Específico". La opción "General" indica que el parámetro corresponde al valor seleccionado en la pestaña "Configuración", la opción "Específico" permite personalizar el parámetro.. Se puede personalizar: 

		- el tiempo de inactividad autorizado,

		- la duración de los contextos de AWP,

		- la duración máxima de la tarea,

		- el número de conexiones autorizadas,

		- el número máximo de conexiones permitidas para el mismo usuario,

		- el número máximo de sesiones en caché,

		- las opciones de seguridad. 
						Las opciones de seguridad del servidor se definen en la pestaña ["Avanzado"](../WDAdmin/3539066.md) del administrador WEBDEV (sección "Varios"). 

		- la generación de un fichero log específico. 

		- los destinatarios de los informes de error para este sitio. Esta opción sólo está disponible si la generación de informes de error está habilitada en la pestaña "Log" del administrador.. Para obtener más información, consulte [Administrador WEBDEV: Visualización de los registros](../WDAdmin/3539073.md). 

		- la URL del servicio web SaaS que se va a utilizar. Este parámetro corresponde a: 

			- el nombre del host del servicio web SaaS (por ejemplo, "Computer", "Computer.MyNetwork", "http://Mydomain.com").

			- la URL completa del servicio web SaaS (por ejemplo, "http://domain.com/WDSAAS28 0_WEB/WDSaaS28 0.awws").




- la ubicación de los archivos de datos de User Groupware. Si el sitio utiliza el programa de grupo de usuarios: 

	- versión 18 y posteriores: especifique el directorio de ubicación del archivo de derechos.

	- versión compatible: especifique el directorio de ubicación del archivo de derechos y usuarios.




Para obtener más información, consulte [la pestaña "Configuración](../WDAdmin/3539046.md).

Al confirmar la configuración del sitio, los cambios se tienen en cuenta inmediatamente para el sitio seleccionado.
